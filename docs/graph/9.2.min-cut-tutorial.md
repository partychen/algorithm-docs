## 一、什么是最小割

**最小割**（Minimum Cut），指的是在一个有向带权网络 $G = (V, E)$ 中，给定源点 $s$ 和汇点 $t$，找到一个边集 $C \subseteq E$，使得移除 $C$ 后 $s$ 与 $t$ 不再连通，且 $C$ 中所有边的容量之和最小。

$$\text{最小割} = \min_{C} \sum_{e \in C} c(e) \quad \text{s.t. 移除 } C \text{ 后 } s \not\rightsquigarrow t$$

用一句话概括：

> **以最小的代价切断从源到汇的所有路径 —— 这就是最小割。**

### 最大流最小割定理（Max-Flow Min-Cut Theorem）

这是网络流理论中最核心的定理：

$$\boxed{\text{最大流} = \text{最小割}}$$

直觉理解：最大流被网络中"最窄的瓶颈"限制，而最小割恰好就是这个瓶颈。流量不可能超过任何一个割的容量，而最小割给出了最紧的上界 —— 并且这个上界可以达到。

### 最小割建模的核心价值

最小割之所以重要，不仅因为它在理论上等价于最大流，更因为它提供了一种强大的**建模视角**：

| 模型 | 含义 | 典型应用 |
|------|------|---------|
| 二选一模型 | 每个元素必须归属 $s$ 或 $t$，选错付出代价 | 善意的投票、黑白染色 |
| 黑白染色 | 网格图中相邻节点归属不同阵营需付代价 | 狼和羊的故事 |
| 闭合子图对偶 | 权值有正有负的选取问题 | 方格取数、最大权闭合子图 |
| 最小点割 | 删点 → 拆点转边割 | Telecowmunication |
| 最小割方案输出 | BFS 残量网络染色 | AHOI2009 最小割 |

## 二、核心思想与前置知识

### 2.1 最大流最小割定理

**定理**：在任意网络中，从 $s$ 到 $t$ 的最大流等于 $s$-$t$ 最小割的容量。

**证明思路**（三步）：
1. **任何流 $\le$ 任何割**：流必须穿过割，流量不超过割的容量。
2. **存在一个流 $=$ 一个割**：当最大流跑完后，在残量网络中从 $s$ 出发 BFS 能到达的点集为 $S$，其余为 $T$。此时 $S$ 到 $T$ 的边在原网络中全部满流，$T$ 到 $S$ 的边流量为 0。因此该割的容量 $=$ 最大流值。
3. 综合 1 和 2，最大流 $=$ 最小割。

```
证明的直觉：
  跑完最大流后，从 s 出发沿残量 > 0 的边 BFS，
  能到的点集合 S，不能到的点集合 T（包含 t）。
  S → T 的边残量 = 0（即满流），构成一个割。
  这个割的容量 = 流过这些边的流量 = 最大流。
  所以：最大流 = 这个割的容量 ≥ 最小割 ≥ 最大流。
  三者相等。
```

### 2.2 最小割的求解（跑最大流）

由最大流最小割定理，**求最小割只需要跑一遍最大流**。常用 Dinic 算法，时间复杂度 $O(V^2 E)$。

```cpp
// Dinic 最大流核心框架（前置板子）
struct Edge { int to, rev; ll cap; };
vector<Edge> graph[N];
int level[N], iter[N];

void add_edge(int from, int to, ll cap) {
    graph[from].push_back({to, (int)graph[to].size(), cap});
    graph[to].push_back({from, (int)graph[from].size()-1, 0});
}

bool bfs(int s, int t) {
    memset(level, -1, sizeof level);
    queue<int> q; q.push(s); level[s] = 0;
    while (!q.empty()) {
        int v = q.front(); q.pop();
        for (auto &e : graph[v])
            if (e.cap > 0 && level[e.to] < 0)
                { level[e.to] = level[v]+1; q.push(e.to); }
    }
    return level[t] >= 0;
}

ll dfs(int v, int t, ll f) {
    if (v == t) return f;
    for (int &i = iter[v]; i < (int)graph[v].size(); i++) {
        Edge &e = graph[v][i];
        if (e.cap > 0 && level[v] < level[e.to]) {
            ll d = dfs(e.to, t, min(f, e.cap));
            if (d > 0) { e.cap -= d; graph[e.to][e.rev].cap += d; return d; }
        }
    }
    return 0;
}

ll max_flow(int s, int t) {
    ll flow = 0;
    while (bfs(s, t)) {
        memset(iter, 0, sizeof iter);
        ll d;
        while ((d = dfs(s, t, LLONG_MAX)) > 0) flow += d;
    }
    return flow;
}
```

**最小割值 $=$ `max_flow(s, t)`。**

### 2.3 最小割方案的输出（残量网络 BFS 染色）

跑完最大流后，从 $s$ 出发，在残量网络中沿残量 $> 0$ 的边做 BFS，能到达的点染为 $S$ 集，其余点染为 $T$ 集。

- 一条原始边 $(u, v)$ 属于最小割 $\iff$ $u \in S$ 且 $v \in T$ 且该边满流。

```cpp
// 最小割方案输出：BFS 残量网络
bool vis[N];
void find_cut(int s) {
    memset(vis, false, sizeof vis);
    queue<int> q; q.push(s); vis[s] = true;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (auto &e : graph[u])
            if (e.cap > 0 && !vis[e.to])
                { vis[e.to] = true; q.push(e.to); }
    }
    // vis[u] = true 表示 u ∈ S 集
    // 边 (u,v) 满流且 vis[u] && !vis[v] → 该边在最小割中
}
```

### 2.4 常见建模套路

#### 二选一模型

每个元素必须选择归属 $s$ 阵营或 $t$ 阵营。若选 $s$ 付出代价 $a$，选 $t$ 付出代价 $b$，则：

- $s \to u$，容量 $b$（割掉此边意味着 $u$ 不选 $s$，即选 $t$，代价 $b$）
- $u \to t$，容量 $a$（割掉此边意味着 $u$ 不选 $t$，即选 $s$，代价 $a$）

```
二选一模型的理解：
  s ──b──→ u ──a──→ t

  若 u ∈ S 集（归 s 阵营）：割掉 u→t 的边，代价 a
  若 u ∈ T 集（归 t 阵营）：割掉 s→u 的边，代价 b

  额外约束：若 u,v 选择不同阵营需付额外代价 w
  → u ↔ v 连容量 w 的双向边
```

#### 黑白染色（方格图）

方格图上相邻格子天然形成二分图。将黑格连 $s$，白格连 $t$（或反之），相邻格之间连边表示"选择不同阵营的代价"。

#### 闭合子图 / 最大权闭合子图

选取一些有权值的节点（权值有正有负），满足：若选了 $u$ 且 $u \to v$ 有边，则 $v$ 也必须选。

$$\text{最大权闭合子图} = \text{正权值之和} - \text{最小割}$$

建模方法：
- $s$ 连所有正权点，容量 $=$ 权值
- 所有负权点连 $t$，容量 $=$ |权值|
- 原图的依赖边，容量 $= +\infty$

## 三、分类与变体详解

### 3.1 基本最小割

最基础的形式：给定网络，求 $s$-$t$ 最小割的值。直接跑最大流即可。

**关键点**：
- 无向图中每条边要建两条方向相反的边，各自容量为原边权。
- 最小割的值等于最大流的值。

### 3.2 最小点割（拆点）

当需要删除点而非边时，使用经典的**拆点**技巧：将每个点 $v$ 拆为入点 $v_{\text{in}}$ 和出点 $v_{\text{out}}$，之间连一条容量等于删除该点的代价的边。

```
拆点操作：
  原图中的点 v → v_in 和 v_out
  v_in → v_out：容量 = 删除 v 的代价（通常为 1）
  原图中边 (u, v) → u_out → v_in：容量 = +∞

  这样割掉 v_in → v_out 的边就等价于删掉点 v。
```

**注意**：源点和汇点不能被删除，因此 $s_{\text{in}} \to s_{\text{out}}$ 和 $t_{\text{in}} \to t_{\text{out}}$ 的容量设为 $+\infty$。

### 3.3 二选一模型（最小权闭合子图的对偶）

这是最小割建模中最常见、最强大的套路。核心思想是：

- 每个元素要做一个二选一的决策（选 $A$ 还是选 $B$）；
- 选不同方案有不同代价；
- 相邻元素选择不同还有额外惩罚。

最小割恰好模拟了这种**全局最优二分决策**。

**典型特征**：
- 题目出现"每个人/物品有两种选择"；
- "不同选择之间有冲突代价"；
- 求最小总代价。

### 3.4 方格图最小割

方格图中，将格子按 $(i+j)$ 的奇偶性分为黑白两部分，天然形成二分图。常见套路：

- 黑格连源点 $s$，白格连汇点 $t$；
- 相邻黑白格之间连边；
- 最小割 = 需要放弃的最小收益。

在方格取数问题中：**最大收益 = 所有格子权值之和 - 最小割**。

### 3.5 最小割边数最少

有时题目要求：在所有最小割中，找**包含边数最少**的一个。

**方法**：先跑一遍最大流得到最小割值 $f$。然后将所有满流的边（残量 $= 0$）的容量改为 $1$，其余边容量改为 $+\infty$（比如 $m+1$ 倍边数以上的大值）。再跑一遍最大流，第二次的最大流值就是最小割中的最少边数。

另一种等价做法：将原来每条边的容量 $c$ 替换为 $c \times (m+1) + 1$，然后跑一遍最大流。设结果为 $F$，则最小割值为 $\lfloor F / (m+1) \rfloor$，最少边数为 $F \bmod (m+1)$。

```
最小割边数最少的技巧：
  原边容量 c → c * (m+1) + 1
  跑最大流得 F
  最小割值 = F / (m+1)
  最少边数 = F % (m+1)

  原理：+1 让算法在割值相同时优先选边数少的方案。
```

## 四、7 道经典例题详解

### 题目 1：[USACO4.4] Pollutant Control（洛谷 P1344）

> 给定一个 $n$ 个节点 $m$ 条边的有向网络，每条边有容量。求从 $1$ 到 $n$ 的最小割的值，以及最小割中最少包含多少条边。$n \le 32$，$m \le 1000$。

**难度**：🟡

**分析**：

```
第一问直接求最大流即可。
第二问要求在所有最小割中找边数最少的。
使用经典技巧：将每条边的容量 c 替换为 c * (m+1) + 1。
这样跑出的最大流 F 中：
  F / (m+1) = 最小割值
  F % (m+1) = 最小割的最少边数
原理：每条边被割时额外贡献 1，在割值相同时优先选边数少的。
```

**C++ 代码**：

```cpp
// 将每条边容量 c 改为 c * (m+1) + 1
// 跑一遍 Dinic 最大流
void solve(int n, int m) {
    // 读入建图时：add_edge(u, v, c * (ll)(m+1) + 1)
    ll F = max_flow(1, n);
    ll cut_val = F / (m + 1);
    ll cut_cnt = F % (m + 1);
    printf("%lld %lld\n", cut_val, cut_cnt);
}
```

**核心启示**：最小割边数最少的经典技巧是将容量乘以一个大常数后加 1，将两个目标编码在同一个值中。这是一种通用的"双目标编码"方法。

### 题目 2：[USACO5.4] Telecowmunication（洛谷 P1345）

> 给定一个 $n$ 个节点 $m$ 条无向边的图，求从 $s$ 到 $t$ 的最小点割（最少删多少个点使 $s$ 和 $t$ 不连通）。不能删除 $s$ 和 $t$。$n \le 100$，$m \le 600$。

**难度**：🟡

**分析**：

```
这是最小点割问题。删点而非删边，需要使用拆点技巧。

拆点建图：
  每个点 v 拆为 v_in 和 v_out。
  v_in → v_out：容量 1（删掉该点的代价为 1）。
  特殊：s 和 t 不能删，s_in → s_out 和 t_in → t_out 容量 INF。
  原图无向边 (u,v)：
    u_out → v_in，容量 INF
    v_out → u_in，容量 INF

源点 = s_in，汇点 = t_out。
跑最大流即为最小点割。
```

**C++ 代码**：

```cpp
// 拆点：v_in = v, v_out = v + n
void build(int n, int m, int s, int t) {
    for (int v = 1; v <= n; v++) {
        ll cap = (v == s || v == t) ? INF : 1;
        add_edge(v, v + n, cap); // v_in → v_out
    }
    // 对每条无向边 (u, v)：
    // add_edge(u + n, v, INF);  // u_out → v_in
    // add_edge(v + n, u, INF);  // v_out → u_in
    printf("%lld\n", max_flow(s, t + n));
}
```

**核心启示**：最小点割 = 拆点后的最小边割。每个点拆为入点和出点，中间连容量为代价的边，原图边用 INF 容量连接出点到入点。这是将"删点"转化为"删边"的标准手法。

### 题目 3：[SHOI2007] 善意的投票（洛谷 P2057）

> $n$ 个小朋友投票，每人有初始意见 0 或 1。有 $m$ 对好朋友关系。每个人可以改变自己的意见，但改变会产生 1 的代价。每对好朋友如果最终意见不同，也会产生 1 的代价。求最小总代价。$n \le 300$，$m \le n(n-1)/2$。

**难度**：🟡

**分析**：

```
经典的二选一模型。
每个小朋友最终的意见只有两种：0（归 t 阵营）或 1（归 s 阵营）。

建模：
  初始意见为 1 的人：s → u，容量 1（割掉 = 改主意为 0，代价 1）
  初始意见为 0 的人：u → t，容量 1（割掉 = 改主意为 1，代价 1）
  每对好朋友 (u, v)：u ↔ v 双向边，容量 1
    （若最终选择不同阵营，这条边被割，代价 1）

最小割 = 最小总代价。
```

**C++ 代码**：

```cpp
void build(int n, int m, int a[]) {
    int S = 0, T = n + 1;
    for (int i = 1; i <= n; i++) {
        if (a[i] == 1) add_edge(S, i, 1);
        else add_edge(i, T, 1);
    }
    // 每对好朋友 (u, v)：
    // add_edge(u, v, 1); add_edge(v, u, 1);
    printf("%lld\n", max_flow(S, T));
}
```

**核心启示**：二选一模型是最小割建模的核心。当每个元素有两种选择、改变选择有代价、不同选择的相邻元素有冲突代价时，直接套用二选一模型：初始属 $s$ 则 $s \to u$，属 $t$ 则 $u \to t$，冲突连双向边。

### 题目 4：[ZJOI2009] 狼和羊的故事（洛谷 P2598）

> $n \times m$ 的方格图上，每个格子是狼 (1)、羊 (2) 或空地 (0)。狼和羊之间需要建篱笆隔开（篱笆建在相邻格子的公共边上）。求最少建多少道篱笆。$n, m \le 100$。

**难度**：🟡

**分析**：

```
狼和羊必须被切断，空地可以归属任一阵营。
这正是经典的最小割模型。

建模：
  源点 s 代表狼阵营，汇点 t 代表羊阵营。
  s → 每个狼格子，容量 INF（狼必须属于 s 阵营）
  每个羊格子 → t，容量 INF（羊必须属于 t 阵营）
  相邻格子之间：双向边，容量 1（在公共边建篱笆的代价为 1）

最小割 = 最少篱笆数。

注意：狼格和狼格之间也要连边，羊和羊之间也要连边，
因为空地需要被分配阵营。所有相邻对都连双向容量 1 的边。
```

**C++ 代码**：

```cpp
int dx[]={0,0,1,-1}, dy[]={1,-1,0,0};
void build(int n, int m, int g[][MAXM]) {
    int S = 0, T = n*m + 1;
    for (int i = 0; i < n; i++)
      for (int j = 0; j < m; j++) {
        int u = i*m + j + 1;
        if (g[i][j] == 1) add_edge(S, u, INF);
        if (g[i][j] == 2) add_edge(u, T, INF);
        for (int d = 0; d < 4; d++) {
            int ni = i+dx[d], nj = j+dy[d];
            if (ni>=0 && ni<n && nj>=0 && nj<m)
                add_edge(u, ni*m+nj+1, 1);
        }
    }
    printf("%lld\n", max_flow(S, T));
}
```

**核心启示**：方格图上两类对象的分割问题是最小割的经典应用。强制属于某阵营的节点用 INF 容量连源/汇，可选节点通过相邻边的割来决定归属。空地的存在不影响建模，它们会被最小割自动分配到最优阵营。

### 题目 5：方格取数问题（洛谷 P2774）

> $n \times m$ 的方格中每个格子有一个正整数。取出若干个数，使得任意两个取出的数所在格子不相邻（上下左右），求取出数的最大和。$n, m \le 100$。

**难度**：🟠

**分析**：

```
方格图中"不能取相邻格子"是经典的独立集问题。
方格图按 (i+j) 奇偶性黑白染色后形成二分图。

方法一（最大独立集 → 最小顶点覆盖 → 最小割）：
  最大权独立集 = 总权值 - 最小权顶点覆盖。
  在二分图中，最小权顶点覆盖 = 最小割。

建模：
  黑格（(i+j) 为偶）：s → 黑格，容量 = 格子权值
  白格（(i+j) 为奇）：白格 → t，容量 = 格子权值
  相邻黑白格：黑格 → 白格，容量 = INF

最小割切断了"冲突的取法"，保留下来的是最大权独立集。
答案 = 所有格子权值之和 - 最小割。
```

**C++ 代码**：

```cpp
void build(int n, int m, int a[][MAXM]) {
    int S = 0, T = n*m + 1;
    ll sum = 0;
    for (int i = 0; i < n; i++)
      for (int j = 0; j < m; j++) {
        int u = i*m + j + 1;
        sum += a[i][j];
        if ((i+j) % 2 == 0) { // 黑格
            add_edge(S, u, a[i][j]);
            for (int d = 0; d < 4; d++) {
                int ni=i+dx[d], nj=j+dy[d];
                if (ni>=0&&ni<n&&nj>=0&&nj<m)
                    add_edge(u, ni*m+nj+1, INF);
            }
        } else { // 白格
            add_edge(u, T, a[i][j]);
        }
    }
    printf("%lld\n", sum - max_flow(S, T));
}
```

**核心启示**：方格图最大权独立集 = 总权值 - 最小割。黑白染色后，黑格连 $s$，白格连 $t$，相邻冲突连 INF。这是"选最多 → 去掉最少"思维的典型体现。割掉 $s \to u$ 表示不选该黑格，割掉 $v \to t$ 表示不选该白格。

### 题目 6：[AHOI2009] 最小割（洛谷 P4126）

> 给定一个有向网络，求最小割后，判断每条边是否**可能**属于某个最小割，以及是否**一定**属于每个最小割。$n \le 4000$，$m \le 60000$。

**难度**：🔴

**分析**：

```
先跑一遍最大流。然后在残量网络上用 Tarjan 求强连通分量（SCC）。

对于原图中的一条边 (u, v, c)：

1. 该边可能在某个最小割中 ↔ 该边满流（残量 = 0）
   且 u 和 v 不在残量网络的同一个 SCC 中。

2. 该边一定在每个最小割中 ↔ 该边满流
   且 u 与 s 在同一 SCC 中，v 与 t 在同一 SCC 中。

   更准确地说：u 所在 SCC 能从 s 的 SCC 到达，
   且 v 所在 SCC 能到达 t 的 SCC。
   简化判定：scc[u] == scc[s] 且 scc[v] == scc[t]。

原理：
  满流且不在同一 SCC → 不存在替代增广路 → 可以属于最小割。
  u 与 s 同 SCC、v 与 t 同 SCC → 割掉此边是"必须的"。
```

**C++ 代码**：

```cpp
// 1. Dinic 求最大流
// 2. 残量网络上 Tarjan 求 SCC
// 3. 对每条边 (u, v)：
void solve() {
    max_flow(s, t);
    tarjan(); // 在残量网络（cap > 0 的边）上跑 SCC
    for (int i = 0; i < m; i++) {
        int u = edges[i].from, v = edges[i].to;
        bool full = (edges[i].cap == 0); // 满流
        bool possible = full && (scc[u] != scc[v]);
        bool necessary = full && (scc[u]==scc[s]) && (scc[v]==scc[t]);
        printf("%d %d\n", possible, necessary);
    }
}
```

**核心启示**：判断最小割边的"可能性"和"必然性"需要在残量网络上做 SCC 分析。满流是前提条件，SCC 关系决定了边的可替代性。这是最小割理论的高级应用，将割的结构与图的连通性深度结合。

### 题目 7：[SHOI2010] 最小生成树（洛谷 P5039）

> 给定一个 $n$ 个节点 $m$ 条边的无向连通加权图，指定一条边 $e_{\text{lab}}$。问至少修改多少条边的权值（每次修改可以将某条边权值 $+1$ 或 $-1$），使得 $e_{\text{lab}}$ 一定出现在所有最小生成树中。$n \le 500$，$m \le 800$。

**难度**：🟣

**分析**：

```
设指定边为 (a, b, w)。
在最小生成树中，边 (a, b, w) 一定被选 ↔
  删掉该边后图不连通，或者：
  所有与 (a, b) 形成环的边的权值都严格大于 w。

与 (a, b) 形成环的边：连接 a-b 路径上不同连通块的边，
即：原图中所有权值 ≤ w 且与 (a, b) 在某棵生成树中形成环的边。

更精确地：考虑原图中所有权值 ≤ w 的边（不含 e_lab 本身），
这些边中连接 a 所在连通块和 b 所在连通块（通过权值 < w 的边形成的连通块）
的边，需要把它们的权值都改到 > w。

但直接枚举太复杂。关键观察：
  将 e_lab 必须在 MST 中 ↔ 对于任何包含 e_lab 的环，
  e_lab 不是环中权值严格最大的边。
  即：任何权值 ≤ w 的边如果与 e_lab 形成环，就形成竞争。

建最小割模型：
  取出所有权值 ≤ w 的非 e_lab 边（设其权值为 wi）。
  这些边如果能连通 a 和 b，就意味着存在不经过 e_lab 的
  权值 ≤ w 的路径。我们需要让这些边的权值全部 > w，
  每条边的修改代价为 w - wi + 1。

  或者：只需让用这些边无法从 a 到达 b 即可！
  这就是以 a 为源、b 为汇、容量为 (w - wi + 1) 的最小割。

建图：
  对原图中所有权值 ≤ w 的边 (u, v, wi)（不含 e_lab）：
    u ↔ v 双向边，容量 = w - wi + 1
  源 = a，汇 = b
  最小割 = 最少总修改量。
```

**C++ 代码**：

```cpp
void solve(int n, int m, int lab) {
    int a = e[lab].u, b = e[lab].v, w = e[lab].w;
    for (int i = 1; i <= m; i++) {
        if (i == lab) continue;
        if (e[i].w <= w) {
            int cost = w - e[i].w + 1;
            add_edge(e[i].u, e[i].v, cost);
            add_edge(e[i].v, e[i].u, cost);
        }
    }
    printf("%lld\n", max_flow(a, b));
}
```

**核心启示**：让某条边必须在 MST 中 → 消除所有竞争环 → 让权值 $\le w$ 的边无法连通 $a$ 和 $b$ → 最小割。代价为 $w - w_i + 1$ 表示将边权从 $w_i$ 改到 $w + 1$ 所需的最小修改量。将生成树条件转化为网络流，是一种深刻的建模能力。

## 五、常见技巧与思路总结

### 1. 建模技巧

| 技巧 | 说明 | 典型应用 |
|------|------|---------|
| 二选一模型 | 每个元素归 $s$ 或 $t$，代价不同 | 善意的投票 |
| 黑白染色 | 方格按奇偶分组，冲突连 INF | 狼和羊、方格取数 |
| 拆点转边割 | 删点问题转删边 | Telecowmunication |
| 容量编码双目标 | $c \times M + 1$ 同时优化割值和边数 | Pollutant Control |
| INF 强制归属 | 不允许被割的边设为 INF | 源汇到固定阵营节点 |
| 竞争边建模 | 需要消除的竞争边设为容量 | 最小生成树 |

### 2. 常见思维套路

- **"两种选择，选错有代价"** → 二选一模型，最小割
- **"方格图不相邻选最多"** → 黑白染色 + 最大权独立集 = 总和 - 最小割
- **"删最少的点使不连通"** → 拆点 + 最小割
- **"判断边是否在最小割中"** → 残量网络 + SCC
- **"最小割边数最少"** → 容量 $\times (m+1) + 1$
- **"两类对象分割"** → 源汇分别连两类，中间连分割代价

### 3. 常见陷阱

- **无向图建边要双向**：无向边 $(u,v,c)$ 需要建 $u \to v$ 容量 $c$ 和 $v \to u$ 容量 $c$（不是反向边，是两条正向边各带反向边）。
- **拆点编号不要冲突**：通常 $v_{\text{in}} = v$，$v_{\text{out}} = v + n$，注意源汇编号不要与之重叠。
- **INF 不能太大**：多条 INF 边可能溢出，建议 INF $= 10^{18} / n$ 左右，或者根据题目数据范围适当设置。
- **忘记建反向边**：Dinic 算法的反向边容量初始为 0，这是增广路退流的基础，绝对不能漏。
- **SCC 判必须边时必须先判满流**：不满流的边绝对不在最小割中。

## 六、优化方法 / 解题框架

### 解题框架

```
1. 识别最小割模型：
   - 是否有"两种选择 / 两个阵营"的结构？
   - 是否要"切断某种连通性"？
   - 是否是"选最多/最大但有冲突约束"？（转最小割的对偶）

2. 确定源点和汇点：
   - 二选一：s = 一种选择，t = 另一种选择
   - 分割：s = 一类对象，t = 另一类对象
   - 点割：记得拆点

3. 建图：
   - 强制归属边：INF
   - 选择代价边：具体代价值
   - 冲突边：冲突代价（双向）
   - 拆点内部边：删除代价

4. 求解：
   - 跑 Dinic 最大流 = 最小割值
   - 需要方案：BFS 残量网络
   - 需要判断边的性质：Tarjan SCC

5. 输出答案：
   - 直接输出最小割值
   - 或"总收益 - 最小割"
```

### 算法选择指南

```
问题类型                    算法                        备注
──────────────────────────────────────────────────────────────
基本最小割                  Dinic 最大流                O(V^2 E)
最小点割                    拆点 + Dinic                标准套路
最小割方案输出              Dinic + BFS 残量网络         染色法
边是否在最小割中            Dinic + Tarjan SCC           满流 + SCC
最小割边数最少              容量编码 / 两次最大流         c*(m+1)+1
二选一 / 黑白染色           最小割建模                  核心是建图
最大权独立集（方格图）       总和 - 最小割                对偶转化
```

### 复杂度一览

| 算法 | 时间复杂度 | 适用场景 |
|------|-----------|---------|
| Dinic | $O(V^2 E)$ | 通用最大流 / 最小割 |
| Dinic（单位容量） | $O(E\sqrt{V})$ | 二分图匹配、单位网络 |
| ISAP | $O(V^2 E)$ | 实践中常数较小 |
| HLPPS（预流推进） | $O(V^2 \sqrt{E})$ | 大规模稠密图 |
| BFS 染色 | $O(V + E)$ | 输出最小割方案 |
| Tarjan SCC | $O(V + E)$ | 判断边的割属性 |

## 七、终极总结与归纳

### 最小割知识图谱

```
最小割
├── 基础理论
│   ├── 最大流 = 最小割（Ford-Fulkerson 定理）
│   ├── 求解方法：跑最大流（Dinic / ISAP）
│   └── 方案输出：残量网络 BFS 染色
├── 经典模型
│   ├── 二选一模型（每个元素二选一，代价最小化）
│   ├── 黑白染色（方格图分割）
│   ├── 最大权闭合子图（正权连 s，负权连 t，依赖连 INF）
│   └── 方格图最大权独立集 = 总和 - 最小割
├── 变体
│   ├── 最小点割（拆点转边割）
│   ├── 最小割边数最少（容量编码双目标）
│   └── 判断边的割属性（残量网络 + SCC）
└── 高级应用
    ├── MST 中指定边必须选 → 消除竞争环 → 最小割
    ├── 最大权闭合子图 → 正权和 - 最小割
    └── 最小割树 / 全局最小割（Gomory-Hu Tree）
```

### 一句话记忆

| 概念 / 算法 | 一句话 |
|-------------|-------|
| 最大流最小割定理 | 最大流等于最小割，跑完最大流就得到了最小割 |
| 二选一模型 | 归 $s$ 还是归 $t$——割哪条边代价小就割哪条 |
| 拆点 | 删点变删边——$v_{\text{in}} \to v_{\text{out}}$ 容量为代价 |
| 方格图独立集 | 黑白染色连源汇，总和减最小割就是答案 |
| 割方案输出 | BFS 残量网络，能到标 $S$，不能到标 $T$ |
| 必须边判定 | 满流 $+$ 源端同 $s$ 的 SCC $+$ 汇端同 $t$ 的 SCC |
| 容量编码 | $c \times (m+1) + 1$，一次跑出割值和边数 |

### 做题心法

1. **先判断是否是割模型**：题目中出现"分成两组""切断""隔离""二选一"等关键词，优先考虑最小割。
2. **建图是核心**：最小割问题 80% 的难度在建图。想清楚"割掉这条边意味着什么"，就能建出正确的图。
3. **善用对偶**：最大权独立集 = 总和 - 最小割；最大权闭合子图 = 正权和 - 最小割。"求最大"常常转化为"减去最小"。
4. **拆点是万能钥匙**：只要涉及"删点"，就拆点。拆点后一切回归边割。
5. **残量网络藏信息**：最小割的方案、边的必要性、可替代性，都藏在残量网络的结构中。
