# 费用流（MCMF）— 最小费用最大流完全指南

## 一、什么是费用流

在普通最大流问题中，每条边只有一个属性——**容量**。而在费用流问题中，每条边同时拥有两个属性：

| 属性 | 含义 |
|------|------|
| **容量** $c(u,v)$ | 该边最多能通过多少流量 |
| **费用** $w(u,v)$ | 每单位流量经过该边需要付出的代价 |

**目标**：在从源点 $s$ 到汇点 $t$ 的所有可行流中，找到一个**最大流**，并且在所有最大流中，使得**总费用最小**。

总费用的计算公式为：

$$
\text{Cost} = \sum_{(u,v) \in E} f(u,v) \times w(u,v)
$$

其中 $f(u,v)$ 是边 $(u,v)$ 上的实际流量。

**直觉理解**：想象你要通过一个管道网络运送货物，每条管道有运输上限（容量），也有运输单价（费用）。你需要运尽可能多的货物，同时在运最多的前提下花最少的钱。

**与最大流的关系**：

- 最大流：只关心"运多少"
- 费用流：既关心"运多少"，又关心"花多少"
- 费用流 = 最大流 + 最短路

## 二、核心思想与前置知识

### 2.1 费用流的形式化定义

给定网络 $G = (V, E)$，源点 $s$，汇点 $t$，每条边 $(u,v)$ 有容量 $c(u,v)$ 和单位费用 $w(u,v)$。

**最小费用最大流 (MCMF)** 求解的是：

$$
\min \sum_{(u,v) \in E} f(u,v) \cdot w(u,v) \quad \text{s.t.} \quad |f| = f_{\max}
$$

即在流量最大的前提下，使总花费最小。

**反向边的费用**：建边 $(u,v,c,w)$ 时，反向边为 $(v,u,0,-w)$。反向边费用取反的含义是"退流 = 退钱"。

### 2.2 SSP（Successive Shortest Path）算法

SSP 是最经典的费用流算法，核心思想：

> **每次沿费用最短的增广路增广，直到没有增广路为止。**

这等价于把 EK 算法中的 BFS 换成 SPFA（因为存在负权边）。

**算法步骤**：

1. 用 SPFA 在残量网络上找 $s \to t$ 的最短路（以费用为边权）
2. 沿最短路增广，更新残量网络
3. 累加流量和费用
4. 重复直到找不到增广路

**正确性**：每次沿最短路增广，保证了每单位流量的费用递增，因此总费用最小。

**SPFA 费用流核心代码模板**：

```cpp
const int INF = 1e9;
int n, S, T, maxflow, mincost;
int head[N], to[M], nxt[M], cap[M], cost[M], tot = 1;
int dis[N], cur[N], incf[N], pre[N];
bool vis[N];

void addEdge(int u, int v, int c, int w) {
    to[++tot] = v; cap[tot] = c; cost[tot] = w;  nxt[tot] = head[u]; head[u] = tot;
    to[++tot] = u; cap[tot] = 0; cost[tot] = -w; nxt[tot] = head[v]; head[v] = tot;
}

bool spfa() {
    memset(dis, 0x3f, sizeof dis);
    memset(vis, 0, sizeof vis);
    queue<int> q;
    dis[S] = 0; vis[S] = 1; incf[S] = INF; q.push(S);
    while (!q.empty()) {
        int u = q.front(); q.pop(); vis[u] = 0;
        for (int i = head[u]; i; i = nxt[i]) {
            int v = to[i];
            if (cap[i] && dis[v] > dis[u] + cost[i]) {
                dis[v] = dis[u] + cost[i];
                incf[v] = min(incf[u], cap[i]);
                pre[v] = i;
                if (!vis[v]) { vis[v] = 1; q.push(v); }
            }
        }
    }
    return dis[T] < INF;
}

void mcmf() {
    maxflow = mincost = 0;
    while (spfa()) {
        maxflow += incf[T];
        mincost += incf[T] * dis[T];
        int u = T;
        while (u != S) {
            cap[pre[u]] -= incf[T];
            cap[pre[u] ^ 1] += incf[T];
            u = to[pre[u] ^ 1];
        }
    }
}
```

**复杂度**：$O(V \cdot E \cdot f)$，其中 $f$ 为最大流量。实际中通常远快于理论上界。

### 2.3 Primal-Dual 原始对偶算法

当图中没有负环时，可以用 Johnson 技术 + Dijkstra 替代 SPFA，获得更优的复杂度。

**核心思想**：

1. 先用一次 Bellman-Ford / SPFA 求出初始势函数 $h(v)$
2. 将边权修改为 $w'(u,v) = w(u,v) + h(u) - h(v) \geq 0$
3. 每次用 Dijkstra 求最短路
4. 增广后更新势函数 $h(v) \mathrel{+}= d(v)$

**优势**：避免了 SPFA 在稠密图上的退化，复杂度更稳定。

**适用场景**：需要更高效率的费用流，尤其是边权非负或可通过势函数转化的场景。

### 2.4 最大费用 = 取反

若题目要求**最大费用最大流**，只需将所有边的费用**取反**，跑最小费用最大流后，将得到的总费用**再取反**即可。

$$
\text{MaxCost} = -\text{MCMF}(-w)
$$

这个技巧在很多建模题中非常实用。

## 三、分类与变体

### 3.1 SSP（连续最短路）

- **实现**：SPFA + EK 增广
- **优点**：代码简短，容易理解
- **缺点**：存在被卡的可能（SPFA 退化）
- **适用**：竞赛中绝大多数费用流题目

### 3.2 Primal-Dual（原始对偶）

- **实现**：势函数 + Dijkstra + 多路增广
- **优点**：复杂度稳定，适合稠密图
- **缺点**：代码较长，需要处理势函数
- **适用**：对效率要求高的场景

### 3.3 最大费用最大流

- **方法**：所有费用取反 → 跑 MCMF → 结果取反
- **适用**：题目要求最大化某种收益

### 3.4 常见建模套路

| 建模场景 | 建边方式 |
|----------|---------|
| 每个点有流量限制 | 拆点：$v \to v_{in}, v_{out}$，中间连边 $(v_{in}, v_{out}, \text{cap}, \text{cost})$ |
| 恰好选 $k$ 个 | 源点流量设为 $k$ |
| 每条路径费用不同 | 对应边设不同费用 |
| 网格图取数 | 每格拆点，经过有收益则费用为负 |
| 最大权匹配 | 二分图费用流建模 |
| 分层建图 | 按时间/阶段分层，层间连边 |

### 题目 1：模板 MCMF（洛谷 P3381）🟢

> **题意**：给定一个有 $n$ 个点、$m$ 条边的网络，每条边有容量和费用。求从 $s$ 到 $t$ 的最小费用最大流。

**分析**

这是费用流的模板题，直接套用 SSP 算法即可。

```
建模：
- 直接按题意建边 (u, v, cap, cost)
- 每条边同时建反向边 (v, u, 0, -cost)
- 运行 SPFA + 增广 循环
```

```cpp
// 核心：直接套用 SSP 模板
// addEdge(u, v, c, w) 建边
// 读入每条边后调用 addEdge
// 最后调用 mcmf() 得到 maxflow 和 mincost
void solve() {
    int n, m;
    scanf("%d%d%d%d", &n, &m, &S, &T);
    for (int i = 0; i < m; i++) {
        int u, v, c, w;
        scanf("%d%d%d%d", &u, &v, &c, &w);
        addEdge(u, v, c, w);
    }
    mcmf();
    printf("%d %d\n", maxflow, mincost);
}
```

**核心启示**：费用流模板是所有后续题目的基础，务必熟练掌握 SSP 的 SPFA 增广写法。

### 题目 2：负载平衡（洛谷 P4016）🟡

> **题意**：$n$ 个仓库排成一圈，第 $i$ 个仓库有 $a_i$ 个货物。每次可以在相邻仓库之间搬运一个货物（花费 1）。求使所有仓库货物相等的最小花费。

**分析**

设平均值 $\bar{a} = \frac{\sum a_i}{n}$。每个仓库要么多出货物（供应），要么缺少货物（需求）。这是一个经典的供需匹配问题。

```
建模：
- 超级源 S，超级汇 T
- 若 a[i] > avg：S → i，容量 a[i]-avg，费用 0（供应节点）
- 若 a[i] < avg：i → T，容量 avg-a[i]，费用 0（需求节点）
- 相邻仓库 i ↔ i+1（含环 n↔1）：双向边，容量 INF，费用 1
- 跑最小费用最大流
```

```cpp
void solve() {
    int avg = sum / n;
    S = 0; T = n + 1;
    for (int i = 1; i <= n; i++) {
        if (a[i] > avg) addEdge(S, i, a[i] - avg, 0);
        if (a[i] < avg) addEdge(i, T, avg - a[i], 0);
        int j = i % n + 1;
        addEdge(i, j, INF, 1);
        addEdge(j, i, INF, 1);
    }
    mcmf();
    printf("%d\n", mincost);
}
```

**核心启示**：环形供需问题的经典费用流建模——多余的连 S，不足的连 T，相邻节点间双向边费用为运输代价。

### 题目 3：方格取数加强版（洛谷 P2045）🟡

> **题意**：$n \times n$ 的网格，每格有一个非负整数。从左上到右下走 $k$ 次，每次只能向右或向下，每个格子的数只能被取一次。求 $k$ 次取数的最大值之和。

**分析**

每个格子可经过多次但只取一次值，因此需要拆点。第一次经过时获得收益，之后经过收益为 0。

```
建模（最大费用最大流，费用取反）：
- 每个格 (i,j) 拆为入点 in 和出点 out
- in → out：容量 1，费用 -val[i][j]（取数，只取一次）
- in → out：容量 INF，费用 0（后续经过不取数）
- out → 右邻 in：容量 INF，费用 0
- out → 下邻 in：容量 INF，费用 0
- S → (1,1)_in：容量 k，费用 0
- (n,n)_out → T：容量 k，费用 0
- 答案 = -mincost
```

```cpp
void solve() {
    S = 0; T = 2 * n * n + 1;
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= n; j++) {
            int id = (i - 1) * n + j;
            addEdge(id, id + n*n, 1, -val[i][j]); // 取一次
            addEdge(id, id + n*n, INF, 0);         // 后续不取
            if (j < n) addEdge(id + n*n, id + 1, INF, 0);       // 右
            if (i < n) addEdge(id + n*n, id + n, INF, 0);       // 下
        }
    addEdge(S, 1, k, 0);
    addEdge(2 * n * n, T, k, 0);
    mcmf();
    printf("%d\n", -mincost);
}
```

**核心启示**：拆点是费用流中处理"只能取一次"约束的标准技巧。第一条边容量 1 带费用，第二条边容量 INF 不带费用，完美分离"取数"与"经过"。

### 题目 4：修车（洛谷 P2053）🟠

> **题意**：$n$ 个顾客、$m$ 个技师。第 $i$ 个顾客由第 $j$ 个技师修车需要 $t_{i,j}$ 时间。每个技师按顺序修车，后面的顾客需要等待前面所有车修完。求最小总等待时间。

**分析**

关键在于拆技师节点。如果技师 $j$ 修了 $p$ 辆车，那么他倒数第 $k$ 辆车的等待时间会被后面 $k$ 个顾客都等到，贡献为 $k \times t_{i,j}$。

```
建模：
- 将每个技师 j 拆成 n 个节点 (j, k)，表示"技师 j 倒数第 k 个修的位置"
- S → 每个顾客 i：容量 1，费用 0
- 顾客 i → 技师节点 (j, k)：容量 1，费用 k * t[i][j]
  （选这条边意味着顾客 i 是技师 j 倒数第 k 个修的，贡献 k*t 的等待）
- 技师节点 (j, k) → T：容量 1，费用 0
- 跑 MCMF，mincost 即为答案
```

```cpp
void solve() {
    S = 0; T = n + m * n + 1;
    for (int i = 1; i <= n; i++)
        addEdge(S, i, 1, 0);
    for (int j = 1; j <= m; j++)
        for (int k = 1; k <= n; k++) {
            int node = n + (j - 1) * n + k;
            addEdge(node, T, 1, 0);
            for (int i = 1; i <= n; i++)
                addEdge(i, node, 1, k * t[i][j]);
        }
    mcmf();
    printf("%.2f\n", 1.0 * mincost / n);
}
```

**核心启示**：对"顺序产生叠加等待"的问题，拆成"倒数第 k 个位置"是经典建模手法。贡献乘以 $k$ 巧妙地将排队等待转化为费用。

### 题目 5：网络扩容（洛谷 P2604）🟠

> **题意**：给定网络，第一问求最大流。第二问：可以花费代价扩容边的容量（每条边扩容单位容量有单位费用），求在最大流基础上再增加 $k$ 单位流量的最小费用。

**分析**

第一问直接跑最大流。第二问在残量网络上新增"扩容边"，然后跑费用流增广 $k$ 单位流量。

```
建模：
- 第一问：普通最大流（Dinic / EK）
- 第二问（在第一问的残量网络上）：
  - 对每条原始边 (u, v)，新增扩容边：容量 INF，费用 cost[i]
  - 新增 S' → s：容量 k，费用 0（限制增广量为 k）
  - 或直接在残量网络上限制增广流量为 k
  - 跑 MCMF，mincost 即为扩容的最小代价
```

```cpp
void solve() {
    // 第一问：跑最大流
    int flow1 = dinic();
    printf("%d ", flow1);

    // 第二问：在残量网络上加扩容边
    for (int i = 0; i < m; i++)
        addEdge(eu[i], ev[i], INF, ec[i]); // 扩容边

    // 限制额外流量为 k：新建超级源 S' → 原源
    int S2 = n + 1;
    addEdge(S2, S, k, 0);
    S = S2;
    mcmf();
    printf("%d\n", mincost);
}
```

**核心启示**：最大流 + 费用流的混合建模。第一问确定基础流量，第二问在残量网络上做增量扩容，体现了网络流层层推进的思想。

### 题目 6：最长 k 可重区间集（洛谷 P3358）🟠

> **题意**：给定 $n$ 个开区间，每个点最多被 $k$ 个区间覆盖。求满足约束的最大区间长度之和。

**分析**

将数轴上的点离散化后，用费用流控制每个点的覆盖次数。

```
建模（最大费用最大流）：
- 离散化所有端点，设共 p 个不同值
- 相邻离散点 i → i+1：容量 k，费用 0（每个位置最多被覆盖 k 次）
- 对每个区间 [l, r]：l → r，容量 1，费用 -(r-l)（取反求最小费用）
- S → 第一个点：容量 k，费用 0
- 最后一个点 → T：容量 k，费用 0
- 答案 = -mincost
```

```cpp
void solve() {
    // 离散化端点
    sort(vals.begin(), vals.end());
    vals.erase(unique(vals.begin(), vals.end()), vals.end());
    int p = vals.size();
    S = 0; T = p + 1;
    addEdge(S, 1, k, 0);
    addEdge(p, T, k, 0);
    for (int i = 1; i < p; i++)
        addEdge(i, i + 1, k, 0);
    for (int i = 0; i < n; i++) {
        int l = getid(a[i]), r = getid(b[i]);
        addEdge(l, r, 1, -(b[i] - a[i]));
    }
    mcmf();
    printf("%d\n", -mincost);
}
```

**核心启示**：区间覆盖问题的费用流建模——离散化后用"流量"控制覆盖次数，用"费用"表示区间长度收益。这是一类非常经典的建模范式。

### 题目 7：数字梯形（洛谷 P4013）🟠

> **题意**：$m$ 列的数字梯形（第 $i$ 行有 $m+i-1$ 个数），从顶行到底行走 $m$ 条路径。三种规则：(1) 路径不相交（点不重复）；(2) 仅边不相交；(3) 无限制。分别求最大路径数值之和。

**分析**

三种规则对应不同的容量设置，核心思想一致：拆点 + 最大费用最大流。

```
建模（三种规则的区别仅在容量）：
- 每格拆为入点和出点
规则 1（点不相交）：
  - in → out：容量 1，费用 -val
  - out → 下一行相邻 in：容量 1，费用 0
规则 2（边不相交）：
  - in → out：容量 INF，费用 -val
  - out → 下一行相邻 in：容量 1，费用 0
规则 3（无限制）：
  - in → out：容量 INF，费用 -val
  - out → 下一行相邻 in：容量 INF，费用 0
- S → 顶行每格 in：容量 1，费用 0
- 底行每格 out → T：容量对应规则，费用 0
- 答案 = -mincost
```

```cpp
void solve(int rule) {
    int capNode = (rule == 1) ? 1 : INF;
    int capEdge = (rule == 3) ? INF : 1;
    int capOut  = (rule == 1) ? 1 : INF;
    S = 0; T = totNode + 1;
    for (int i = 1; i <= rows; i++)
        for (int j = 1; j <= m + i - 1; j++) {
            int id = getId(i, j);
            addEdge(id, id + offset, capNode, -val[i][j]);
            if (i < rows) {
                addEdge(id + offset, getId(i+1, j),   capEdge, 0);
                addEdge(id + offset, getId(i+1, j+1), capEdge, 0);
            }
            if (i == 1)    addEdge(S, id, 1, 0);
            if (i == rows)  addEdge(id + offset, T, capOut, 0);
        }
    mcmf();
    printf("%d\n", -mincost);
}
```

**核心启示**：同一道题的不同约束只需调整容量参数——点不重复 → 点容量 1，边不重复 → 边容量 1，无限制 → 容量 INF。体会容量与约束的对应关系。

### 题目 8：交换棋子（洛谷 P3159）🔴

> **题意**：$n \times m$ 的棋盘，初始状态和目标状态各有若干棋子。每次可以交换相邻两格的棋子。每个格子最多参与 $c_{i,j}$ 次交换。求最少交换次数使初始变为目标状态，或判断无解。

**分析**

这道题的难点在于"每个点参与交换次数有限"。一次交换涉及两个格子，每个格子用掉一次机会。拆点需要更精细地处理。

```
建模（三层拆点法）：
- 每格拆成三个点：A（入点）、B（中间点）、C（出点）
- A → B → C，通过容量限制交换次数
- 设 c = limit[i][j]，src = 初始有棋子，dst = 目标有棋子

  若 src && dst：A→B 容量 (c-1)/2，B→C 容量 c/2（棋子留在原地不消耗次数不变）
                  但由于棋子要"经过"此格，B 上可能驻留一个流量
  若 src && !dst：A→B 容量 (c+1)/2，B→C 容量 c/2（初始棋子流出需 1 次出）
  若 !src && dst：A→B 容量 c/2，B→C 容量 (c+1)/2（目标棋子流入需 1 次入）
  若 !src && !dst：A→B 容量 c/2，B→C 容量 c/2

- 相邻格子的 C → 邻格 A：容量 INF，费用 1
- S → 初始有棋子的 B：容量 1，费用 0
- 目标有棋子的 B → T：容量 1，费用 0
- 若 maxflow != 棋子数，无解
- 否则答案 = mincost / 2（每次交换被两个格子各计一次）
```

```cpp
void solve() {
    S = 0; T = 3 * n * m + 1;
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= m; j++) {
            int A = id(i,j), B = A + n*m, C = A + 2*n*m;
            int c = lim[i][j], s = src[i][j], d = dst[i][j];
            int ab = c / 2, bc = c / 2;
            if (s && !d) ab = (c + 1) / 2;
            if (!s && d) bc = (c + 1) / 2;
            if (s && d)  ab = (c - 1) / 2, bc = c / 2;
            // 特殊处理：s&&d 时需多给 B 一个进出配额
            addEdge(A, B, ab, 0);
            addEdge(B, C, bc, 0);
            if (s) addEdge(S, B, 1, 0);
            if (d) addEdge(B, T, 1, 0);
            // 相邻连边
            for (auto [di, dj] : dir) {
                int ni = i+di, nj = j+dj;
                if (valid(ni, nj))
                    addEdge(C, id(ni,nj), INF, 1);
            }
        }
    mcmf();
    if (maxflow != total) puts("-1");
    else printf("%d\n", mincost / 2);
}
```

**核心启示**：当限制涉及"进出都消耗次数"时，需要三层拆点精细分配容量。根据棋子的存在情况（初始/目标）调整入边和出边的容量分配，是本题最核心的建模难点。

## 五、技巧总结

### 技巧 1：反向边费用取反

建边 $(u, v, c, w)$ 时，反向边为 $(v, u, 0, -w)$。这保证了退流时同时退费用，是 MCMF 正确性的基础。初始化边编号 `tot = 1`，使得正反边通过 `^1` 互相访问。

### 技巧 2：拆点控制通过次数

当某个节点有流量限制（最多经过 $k$ 次）时，将其拆为入点 $v_{in}$ 和出点 $v_{out}$，中间连边容量为 $k$。这是费用流中最常用的技巧之一。

### 技巧 3：最大费用 = 费用取反

求最大费用最大流时，将所有费用取反后跑 MCMF，结果再取反。不需要修改算法本身，只需修改建边时的费用符号。

### 技巧 4：源汇流量控制全局约束

通过 $S$ 到起点的边容量可以控制"总共选多少条路径"或"总共分配多少流量"。这是费用流将组合优化转化为流问题的关键接口。

### 技巧 5：分层图建模时间维度

当问题涉及时间顺序时（如修车、排队），可以按时间拆层，同一对象在不同时间层对应不同节点，层间连边表示状态转移。

## 六、解题框架

遇到费用流题目时，按以下步骤思考：

```
Step 1：识别问题类型
    └─ 是否存在"在某种约束下最大化/最小化某个值"？
    └─ 是否可以抽象为"多条路径"的选择问题？

Step 2：确定网络结构
    └─ 节点：什么是供应方？什么是需求方？
    └─ 源汇：S 连供应方，需求方连 T
    └─ 中间节点：是否需要拆点？

Step 3：确定边的属性
    └─ 容量：表示什么约束？（次数、数量、限制）
    └─ 费用：表示什么代价/收益？
    └─ 最大费用还是最小费用？

Step 4：建图 & 验证
    └─ 画小规模样例手动验证建模正确性
    └─ 检查节点数和边数是否在承受范围内

Step 5：选择算法
    └─ 一般用 SSP（SPFA）
    └─ 卡 SPFA 时用 Primal-Dual
```

## 七、总结

### 7.1 费用流的本质

费用流的本质是在**流量守恒**和**容量约束**的基础上，加入了**费用优化**维度。它将许多看似复杂的组合优化问题，统一归约为网络流模型来求解。

### 7.2 核心要点

1. **建模是灵魂**：算法本身是固定的模板，难点在于如何把题意转化为费用流模型
2. **拆点是利器**：遇到点的限制就拆点，遇到复杂约束就分层
3. **费用是关键**：费用的设定直接决定了优化目标是否正确
4. **容量是约束**：容量不仅限制流量，更编码了题目中的各种限制条件
5. **反向边是保证**：反向边的费用取反保证了贪心增广的正确性

### 7.3 学习路线图


### 7.4 算法总览表

| 算法 | 核心操作 | 复杂度 | 适用场景 |
|------|---------|--------|---------|
| SSP (SPFA) | SPFA 找最短路 + 增广 | $O(VEf)$ | 通用，竞赛首选 |
| Primal-Dual | Dijkstra + 势函数 + 增广 | $O(VE \log V \cdot f)$ | 稠密图、卡 SPFA |
| 取反技巧 | 费用取反 → MCMF → 结果取反 | 同上 | 最大费用最大流 |

### 7.5 费用流口诀

```
费用流，建模难，容量约束费用算。
SPFA 跑最短，沿路增广把流传。
拆点控制经过数，反向退流费用反。
最大费用怎么办？取反跑完再取反。
供需匹配连源汇，分层建图解时间。
```

> 掌握费用流，你就掌握了网络流建模中最强大的武器。从模板到建模，从简单到复杂，坚持练习，必有所成。
