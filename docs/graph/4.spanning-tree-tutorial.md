## 一、什么是生成树

生成树（Spanning Tree）是图论中的基础概念：对于一个有 $n$ 个顶点的**连通无向图**，从中选出 $n-1$ 条边，使得所有顶点仍然连通且不形成环，这 $n-1$ 条边与 $n$ 个顶点构成的子图就是一棵**生成树**。

直观理解：假设你有一张城市地图，城市之间有很多道路。你想保留尽可能少的道路，使得任意两个城市之间仍然互通——保留下来的道路网络就是一棵生成树。

**最小生成树（Minimum Spanning Tree, MST）** 是所有生成树中，边权总和最小的那棵。它在实际工程中有广泛应用：网络布线、管道铺设、电路设计等，核心目标都是"用最小的代价连通所有节点"。

围绕最小生成树，还延伸出许多重要变体：

| 概念 | 含义 | 核心算法/思想 |
|------|------|---------------|
| 最小生成树 | 边权和最小的生成树 | Kruskal / Prim |
| 次小生成树 | 边权和第二小的生成树 | MST + 树上倍增枚举替换边 |
| 瓶颈生成树 | 最大边权最小的生成树 | 等价于 MST |
| Kruskal 重构树 | Kruskal 过程构建的加权二叉树 | 路径最大/最小边权查询 |

## 二、核心思想与前置知识

### 2.1 生成树定义与性质

**定义**：给定连通无向图 $G = (V, E)$，$|V| = n$，生成树 $T$ 是 $G$ 的一个子图，满足：
1. $T$ 包含 $G$ 的所有 $n$ 个顶点
2. $T$ 恰好有 $n - 1$ 条边
3. $T$ 是连通的（等价于 $T$ 无环）

**最小生成树的关键性质**：

**切割性质（Cut Property）**：对于图的任意一个切割（将顶点分成两个非空集合 $S$ 和 $V \setminus S$），横跨切割的**最小权边**一定属于某棵 MST。

**环性质（Cycle Property）**：对于图中任意一个环，环上的**最大权边**一定不属于某棵 MST（若边权互不相同则不属于唯一的 MST）。

```
切割性质直觉理解：
  假设两群城市需要一条桥连通
  选最便宜的那条桥一定是最优策略
  → Kruskal 和 Prim 都基于此性质

环性质直觉理解：
  如果形成了环，环上最重的那条边是冗余的
  去掉它不影响连通性，且总权值一定不增
  → 次小生成树利用此性质替换边
```

### 2.2 Kruskal 算法框架（并查集）

Kruskal 算法是最常用的 MST 算法，思路简洁：**将边按权值从小到大排序，依次加入，若不成环则选中**。用并查集判断两端点是否已连通。

```
Kruskal 流程：
1. 将所有边按权值升序排序
2. 初始化并查集，每个顶点为独立集合
3. 依次遍历每条边 (u, v, w)：
   - 若 find(u) ≠ find(v)（不在同一集合）
     → 选中该边，合并 u, v 所在集合
   - 否则跳过（加入会形成环）
4. 选够 n-1 条边时，MST 构建完成

时间复杂度：O(E log E)（排序为瓶颈）
适用场景：稀疏图（E 较小时高效）
```

### 2.3 Prim 算法框架

Prim 算法思路类似 Dijkstra，从一个起点出发，**每次选择连接已选集合与未选集合的最小边**，逐步扩展生成树。

```
Prim 流程：
1. 选取任意顶点作为起点，加入已选集合 S
2. 用优先队列（小根堆）维护所有连接 S 与 V\S 的边
3. 每次取出最小权边 (u, v, w)，其中 u ∈ S, v ∉ S：
   - 将 v 加入 S，将边 (u,v) 加入 MST
   - 将 v 的所有邻边加入堆
4. 重复直到 |S| = n

时间复杂度：O(E log V)（堆优化版）
            O(V²)（朴素版，适合稠密图）
适用场景：稠密图（E 接近 V² 时朴素版更优）
```

### 2.4 并查集模板

并查集是 Kruskal 算法的核心数据结构，支持快速合并集合与查询连通性。

```cpp
const int MAXN = 200005;
int fa[MAXN], rnk[MAXN];

void init(int n) {
    for (int i = 1; i <= n; i++) fa[i] = i, rnk[i] = 0;
}

int find(int x) {
    return fa[x] == x ? x : fa[x] = find(fa[x]); // 路径压缩
}

bool unite(int x, int y) {
    x = find(x); y = find(y);
    if (x == y) return false;
    if (rnk[x] < rnk[y]) swap(x, y);
    fa[y] = x;
    if (rnk[x] == rnk[y]) rnk[x]++;
    return true; // 合并成功
}
```

## 三、分类与变体详解

### 3.1 Kruskal 算法

**核心思想**：贪心地按边权从小到大选边，用并查集维护连通性。

Kruskal 算法的正确性由**切割性质**保证：每次选中的边一定是某个切割的最小横切边。

```cpp
struct Edge { int u, v, w; } edges[MAXM];

long long kruskal(int n, int m) {
    sort(edges, edges + m, [](const Edge& a, const Edge& b) {
        return a.w < b.w;
    });
    init(n);
    long long ans = 0;
    int cnt = 0;
    for (int i = 0; i < m && cnt < n - 1; i++) {
        if (unite(edges[i].u, edges[i].v)) {
            ans += edges[i].w;
            cnt++;
        }
    }
    return cnt == n - 1 ? ans : -1; // -1 表示不连通
}
```

**关键点**：
- 时间复杂度 $O(E \log E)$，排序是瓶颈
- 并查集操作近似 $O(\alpha(n))$，可视为常数
- 适合边数较少的稀疏图

### 3.2 Prim 算法

**核心思想**：从一个起点出发，不断扩展已选顶点集合，每次选择连接已选集合与未选集合的**最短边**。

```cpp
const int INF = 0x3f3f3f3f;
vector<pair<int,int>> adj[MAXN]; // adj[u] = {(v, w)}
int dis[MAXN], vis[MAXN];

long long prim(int n) {
    memset(dis, 0x3f, sizeof(dis));
    memset(vis, 0, sizeof(vis));
    dis[1] = 0;
    priority_queue<pair<int,int>, vector<pair<int,int>>, greater<>> pq;
    pq.push({0, 1});
    long long ans = 0;
    int cnt = 0;
    while (!pq.empty() && cnt < n) {
        auto [d, u] = pq.top(); pq.pop();
        if (vis[u]) continue;
        vis[u] = 1; ans += d; cnt++;
        for (auto [v, w] : adj[u]) {
            if (!vis[v] && w < dis[v]) {
                dis[v] = w;
                pq.push({w, v});
            }
        }
    }
    return cnt == n ? ans : -1;
}
```

**关键点**：
- 堆优化版时间复杂度 $O(E \log V)$
- 朴素版 $O(V^2)$，在稠密图上优于 Kruskal
- 不需要存所有边再排序，适合邻接表存储

### 3.3 次小生成树

**定义**：边权和第二小的生成树。**严格次小生成树**要求权值严格大于 MST。

**核心思路**：
1. 先求出 MST
2. 枚举每条**非树边** $(u, v, w)$
3. 在 MST 上找 $u \to v$ 路径上的最大边权 $\text{max1}$ 和严格次大边权 $\text{max2}$
4. 用 $w$ 替换路径上的最大边（若 $w = \text{max1}$ 则替换次大边）
5. 所有替换方案中取最小值

**树上倍增**预处理路径最大值与严格次大值，可以做到 $O(E \log V)$。

```
次小生成树流程：
1. Kruskal 求 MST，记录哪些边是树边
2. 在 MST 上建树，倍增预处理：
   - mx1[u][k]: u 向上走 2^k 步路径上的最大边权
   - mx2[u][k]: u 向上走 2^k 步路径上的严格次大边权
3. 枚举每条非树边 (u, v, w)：
   - LCA 查询 u→v 路径的 max1, max2
   - 若 w > max1: delta = w - max1
   - 若 w == max1 且 max2 存在: delta = w - max2
   - 否则: 该边无法产生合法替换
4. ans = MST权值 + min(delta)
```

```cpp
// 倍增预处理（MST上）
int dep[MAXN], fa[MAXN][20];
long long mx1[MAXN][20], mx2[MAXN][20]; // 最大、严格次大

void dfs(int u, int f, int d) {
    dep[u] = d; fa[u][0] = f;
    for (int k = 1; k < 20; k++) {
        fa[u][k] = fa[fa[u][k-1]][k-1];
        long long vals[4] = {mx1[u][k-1], mx2[u][k-1],
                             mx1[fa[u][k-1]][k-1], mx2[fa[u][k-1]][k-1]};
        mx1[u][k] = *max_element(vals, vals + 4);
        mx2[u][k] = -1;
        for (int i = 0; i < 4; i++)
            if (vals[i] != mx1[u][k] && vals[i] > mx2[u][k])
                mx2[u][k] = vals[i];
    }
    for (auto [v, w] : tree[u]) if (v != f) {
        mx1[v][0] = w; mx2[v][0] = -1;
        dfs(v, u, d + 1);
    }
}

// 查询路径上的最大和严格次大边权
pair<long long,long long> query(int u, int v) {
    long long r1 = -1, r2 = -1;
    // ... LCA + 倍增收集路径上 max1, max2（标准倍增LCA流程）
    return {r1, r2};
}
```

### 3.4 Kruskal 重构树

**Kruskal 重构树**是在执行 Kruskal 算法的过程中构建的一棵**有根二叉树**。每次合并两个连通分量时，新建一个虚拟节点作为它们的父亲，虚拟节点的权值为当前合并边的边权。

```
Kruskal 重构树构建流程：
1. 边按权值排序（升序→最小生成树；降序→最大生成树）
2. 对每条边 (u, v, w)，若 find(u) ≠ find(v)：
   a. 新建虚拟节点 p，权值为 w
   b. 令 find(u), find(v) 的父亲都指向 p
   c. 合并并查集
3. 最终得到一棵 2n-1 个节点的二叉树

重要性质：
- 叶子是原图的 n 个顶点
- 非叶节点是虚拟节点，权值为对应合并边权
- 从叶子 u 到叶子 v 的 LCA 的权值 = 原图 u→v 所有路径中「最大边权的最小值」
  （即瓶颈路径的瓶颈值）
```

```cpp
int tot; // 初始化为 n
int val[MAXN * 2], par[MAXN * 2][20];
int ch[MAXN * 2][2]; // 左右儿子

void build_kruskal_tree(int n, int m) {
    sort(edges, edges + m, [](const Edge& a, const Edge& b) {
        return a.w < b.w;
    });
    init(2 * n); // 并查集初始化到 2n
    tot = n;
    for (int i = 0; i < m; i++) {
        int fu = find(edges[i].u), fv = find(edges[i].v);
        if (fu == fv) continue;
        ++tot;
        val[tot] = edges[i].w;
        fa[fu] = fa[fv] = tot;
        ch[tot][0] = fu; ch[tot][1] = fv;
    }
    // tot 为根，在重构树上用倍增/DFS回答查询
}
```

**应用场景**：
- 查询两点之间所有路径中，最大边权的最小值（瓶颈路径）
- 查询从某点出发，只经过边权 $\le k$ 的边能到达哪些点
- 等价于 MST 上的路径最大边权 = LCA 的权值

### 3.5 瓶颈路径问题

**瓶颈路径**：两点之间所有路径中，**最大边权的最小值**（minimax 路径）。

**定理**：**最小生成树一定是瓶颈生成树**。即：MST 上 $u \to v$ 的路径最大边权，就是所有路径中最大边权的最小值。

```
瓶颈路径的求解方法：
方法 1: MST + 树上倍增
  - 建 MST，倍增预处理路径最大值
  - 查询 u→v 路径上的最大边权
  - O(n log n) 预处理，O(log n) 单次查询

方法 2: Kruskal 重构树
  - 建 Kruskal 重构树
  - u, v 的 LCA 的权值即为答案
  - O(n log n) 预处理，O(log n) 单次查询
```

这一性质在"货车运输"等题目中直接应用：将 minimax 转化为 MST 上的路径查询。

## 四、6 道经典例题详解

### 题目 1：P3366 【模板】最小生成树

> 给定一个 $n$ 个点 $m$ 条边的无向连通图，每条边有一个权值。求最小生成树的边权之和。

**分析**：最小生成树的模板题。直接使用 Kruskal 算法：将所有边按权值排序，依次加入并查集，选够 $n-1$ 条边即可。

```
算法选择：Kruskal（边数不多，排序高效）
时间复杂度：O(m log m)
核心步骤：排序 → 遍历边 → 并查集判环 → 累加权值
```

**C++ 代码**：

```cpp
const int MAXN = 5005, MAXM = 200005;
struct Edge { int u, v, w; } e[MAXM];
int fa[MAXN];

int find(int x) { return fa[x] == x ? x : fa[x] = find(fa[x]); }

long long kruskal(int n, int m) {
    for (int i = 1; i <= n; i++) fa[i] = i;
    sort(e, e + m, [](const Edge& a, const Edge& b) { return a.w < b.w; });
    long long ans = 0; int cnt = 0;
    for (int i = 0; i < m && cnt < n - 1; i++) {
        int fu = find(e[i].u), fv = find(e[i].v);
        if (fu != fv) {
            fa[fu] = fv;
            ans += e[i].w;
            cnt++;
        }
    }
    return cnt == n - 1 ? ans : -1;
}
```

**核心启示**：
- Kruskal 是最小生成树的首选算法，代码简洁、理解直观
- 并查集的路径压缩使合并与查询近似 $O(1)$
- 注意判断图是否连通（选中边数是否达到 $n-1$）

### 题目 2：P4180 【模板】严格次小生成树[BJWC2010]

> 给定一个 $n$ 个点 $m$ 条边的无向连通图，求其**严格次小**生成树的边权之和。严格次小意味着权值必须严格大于最小生成树。

**分析**：先求 MST，然后枚举每条非树边 $(u, v, w)$，尝试将其加入树中，替换掉 $u \to v$ 路径上的最大边。若 $w$ 等于路径最大边权，则需替换**严格次大边**。使用**树上倍增**预处理每段路径的最大值和严格次大值。

```
算法流程：
1. Kruskal 求 MST，记录树边和非树边
2. MST 上 DFS 建树，倍增预处理 mx1[u][k], mx2[u][k]
3. 枚举非树边 (u,v,w)：
   - 查询 u→v 在 MST 路径上的 max1, max2
   - 若 w > max1: 替换增量 = w - max1
   - 若 w == max1 且 max2 存在: 替换增量 = w - max2
4. 答案 = MST 权值 + 最小替换增量
时间复杂度：O(m log m + m log n)
```

**C++ 代码**：

```cpp
const int MAXN = 100005, LOG = 17;
const long long INF = 1e18;
vector<pair<int,int>> tree[MAXN];
int dep[MAXN], anc[MAXN][LOG];
long long mx1[MAXN][LOG], mx2[MAXN][LOG];

void dfs(int u, int f) {
    anc[u][0] = f;
    for (int k = 1; k < LOG; k++) {
        anc[u][k] = anc[anc[u][k-1]][k-1];
        long long a = mx1[u][k-1], b = mx2[u][k-1];
        long long c = mx1[anc[u][k-1]][k-1], d = mx2[anc[u][k-1]][k-1];
        mx1[u][k] = max(a, c);
        mx2[u][k] = -INF;
        long long arr[4] = {a, b, c, d};
        for (long long v : arr)
            if (v != mx1[u][k] && v > mx2[u][k]) mx2[u][k] = v;
    }
    for (auto [v, w] : tree[u]) if (v != f) {
        dep[v] = dep[u] + 1;
        mx1[v][0] = w; mx2[v][0] = -INF;
        dfs(v, u);
    }
}
```

**核心启示**：
- 严格次小生成树的关键在于同时维护**最大值和严格次大值**
- 当非树边权等于路径最大边权时，必须退而替换次大边
- 倍增预处理使得单次路径查询为 $O(\log n)$，总复杂度可控

### 题目 3：P2872 [USACO07DEC]Building Roads

> 有 $n$ 个农场，坐标为 $(x_i, y_i)$。已有 $m$ 条道路连接了部分农场。现在需要新修一些道路，使所有农场连通，求新修道路的最小总长度。

**分析**：先将已有道路对应的农场在并查集中合并，再将所有农场两两之间的欧几里得距离作为候选边，运行 Kruskal 算法，只对尚未连通的农场对选边。

```
算法思路：
1. 初始化并查集
2. 读入已有 m 条道路，执行 unite
3. 生成所有 n*(n-1)/2 条候选边（欧几里得距离）
4. 按距离排序，Kruskal 选边（跳过已连通的）
5. 累加选中边的距离

注意：距离用浮点数，n ≤ 1000 所以 O(n²) 建边可行
```

**C++ 代码**：

```cpp
const int MAXN = 1005;
double x[MAXN], y[MAXN];
int fa[MAXN];
struct Edge { int u, v; double w; } e[MAXN * MAXN / 2];

int find(int a) { return fa[a] == a ? a : fa[a] = find(fa[a]); }

double solve(int n, int m) {
    for (int i = 1; i <= n; i++) fa[i] = i;
    // 先合并已有道路（读入时执行 unite）
    int tot = 0;
    for (int i = 1; i <= n; i++)
        for (int j = i + 1; j <= n; j++) {
            double d = sqrt((x[i]-x[j])*(x[i]-x[j]) + (y[i]-y[j])*(y[i]-y[j]));
            e[tot++] = {i, j, d};
        }
    sort(e, e + tot, [](const Edge& a, const Edge& b) { return a.w < b.w; });
    double ans = 0;
    for (int i = 0; i < tot; i++) {
        int fu = find(e[i].u), fv = find(e[i].v);
        if (fu != fv) { fa[fu] = fv; ans += e[i].w; }
    }
    return ans;
}
```

**核心启示**：
- **已有边 = 预合并**：在 Kruskal 之前先处理已有连接，并查集天然支持这一操作
- 坐标转距离后，本质仍是标准 MST
- $n$ 较小时可以 $O(n^2)$ 建全部边，$n$ 大时需要其他优化（如 Delaunay 三角剖分）

### 题目 4：P1991 无线通讯网

> 有 $n$ 个哨所，坐标为 $(x_i, y_i)$。有 $k$ 部卫星电话可分配给 $k$ 个哨所，拥有卫星电话的哨所间通信不受距离限制。其余哨所需要用无线电通信，通信距离 $D$ 以内才能互通。求最小的 $D$，使得所有哨所都能互相通信。

**分析**：将所有哨所两两之间的距离作为边，按边权升序排序。执行 Kruskal 算法，当连通分量数减少到 $k$ 时停止——此时剩余的 $k$ 个分量可以用 $k$ 部卫星电话连通，最后一条选中的边权就是答案 $D$。

```
核心观察：
- k 部卫星电话可以免费连通 k 个连通分量
- 因此只需将 n 个点用 MST 连到 k 个连通分量即可
- 需要选 n-k 条 MST 边，第 n-k 条边的权值就是 D

等价理解：求 MST 后，去掉最大的 k-1 条边，剩余最大边权就是 D
```

**C++ 代码**：

```cpp
const int MAXN = 505;
double x[MAXN], y[MAXN];
int fa[MAXN];
struct Edge { int u, v; double w; } e[MAXN * MAXN / 2];

int find(int a) { return fa[a] == a ? a : fa[a] = find(fa[a]); }

double solve(int n, int k) {
    for (int i = 1; i <= n; i++) fa[i] = i;
    int tot = 0;
    for (int i = 1; i <= n; i++)
        for (int j = i + 1; j <= n; j++) {
            double d = sqrt((x[i]-x[j])*(x[i]-x[j]) + (y[i]-y[j])*(y[i]-y[j]));
            e[tot++] = {i, j, d};
        }
    sort(e, e + tot, [](const Edge& a, const Edge& b) { return a.w < b.w; });
    int comp = n; // 当前连通分量数
    double ans = 0;
    for (int i = 0; i < tot && comp > k; i++) {
        int fu = find(e[i].u), fv = find(e[i].v);
        if (fu != fv) { fa[fu] = fv; comp--; ans = e[i].w; }
    }
    return ans;
}
```

**核心启示**：
- **"免费连接 k 个分量"等价于 MST 去掉最大的 k-1 条边**
- Kruskal 过程天然按边权递增选边，在连通分量减到 $k$ 时停止即可
- 答案是最后选中的边的权值，而非总权值和

### 题目 5：P1967 货车运输

> 有 $n$ 个城市 $m$ 条道路，每条道路有一个限重。有 $q$ 次询问，每次问从城市 $u$ 到城市 $v$ 的货车最大载重是多少（即路径上最小边权的最大值）。

**分析**：最大载重 = 路径上最小边权的最大值（maximin 路径）。这正是**最大生成树**上的路径最小边权。建最大生成树，然后用**倍增**或 **Kruskal 重构树**回答路径最小边权查询。

```
核心定理：最大生成树上 u→v 的路径最小边权 = 所有路径中最小边权的最大值
等价于：最大瓶颈生成树的路径查询

方法 1: 最大生成树 + 倍增求路径最小值
方法 2: Kruskal 重构树（按边权降序建），LCA 权值即为答案

注意：图可能不连通，不连通时答案为 -1
```

**C++ 代码**：

```cpp
const int MAXN = 100005, LOG = 17;

}

int query(int u, int v) {
    // 倍增 LCA，沿路收集 min(mn[u][k])

}
```

**核心启示**：
- **最大载重问题 = 最大生成树上的瓶颈路径查询**
- 按边权**降序**跑 Kruskal 即得最大生成树
- 倍增预处理后单次查询 $O(\log n)$，可高效处理多次询问
- 不连通的点对需特判，答案为 $-1$

### 题目 6：P4047 [JSOI2010]部落划分

> 平面上有 $n$ 个点（部落），将其划分为 $k$ 个组，使得**不同组的最近点对的距离**最大。求这个最大距离。

**分析**：与 P1991（无线通讯网）思路完全一致。将所有点对按距离排序，执行 Kruskal 算法，当连通分量数减为 $k$ 时停止。此时**下一条要加入的边**的权值就是答案——它是"不同组之间的最短距离"。

```
核心思路：
- Kruskal 按距离升序合并，每次合并最近的两个分量
- 当连通分量减到 k 个时停止
- 停止后，下一条要选的边 = 不同组之间最近的两个点
- 该边权即为答案

等价理解：MST 第 n-k+1 小的边权
（MST 的前 n-k 条边分出 k 个组，第 n-k+1 条边就是组间最近距离）
```

**C++ 代码**：

```cpp
const int MAXN = 1005;
double x[MAXN], y[MAXN];
int fa[MAXN];
struct Edge { int u, v; double w; } e[MAXN * MAXN / 2];

int find(int a) { return fa[a] == a ? a : fa[a] = find(fa[a]); }

double solve(int n, int k) {
    for (int i = 1; i <= n; i++) fa[i] = i;
    int tot = 0;
    for (int i = 1; i <= n; i++)
        for (int j = i + 1; j <= n; j++) {
            double d = sqrt((x[i]-x[j])*(x[i]-x[j]) + (y[i]-y[j])*(y[i]-y[j]));
            e[tot++] = {i, j, d};
        }
    sort(e, e + tot, [](const Edge& a, const Edge& b) { return a.w < b.w; });
    int comp = n;
    for (int i = 0; i < tot; i++) {
        if (comp <= k) return e[i].w; // 下一条跨组边
        int fu = find(e[i].u), fv = find(e[i].v);
        if (fu != fv) { fa[fu] = fv; comp--; }
    }
    return 0;
}
```

**核心启示**：
- **部落划分 = MST 删去最大的 k-1 条边**，与无线通讯网本质相同
- 答案是"第一条被拒绝/跨组的边"，而非最后一条加入的边
- 这类"分 k 组使组间距离最大"的问题，都可以用 Kruskal 的中间状态求解

## 五、常见技巧与思路总结

### 5.1 Kruskal vs Prim 的选择

| 维度 | Kruskal | Prim（堆优化） | Prim（朴素） |
|------|---------|----------------|-------------|
| 时间复杂度 | $O(E \log E)$ | $O(E \log V)$ | $O(V^2)$ |
| 适用图类型 | 稀疏图 | 通用 | 稠密图 |
| 数据结构 | 并查集 | 优先队列 | 数组 |
| 实现难度 | 简单 | 中等 | 简单 |
| 竞赛首选 | 是 | 否（除非稠密图） | 特殊情况 |

**经验法则**：竞赛中 90% 的题目用 Kruskal 即可。

### 5.2 "MST 提前终止"的模型

许多看似与 MST 无关的题目，本质上都是 Kruskal 的提前终止：

```
通用模型：
  有 n 个元素，额外有 k 个"免费连接"名额
  求用最小代价连通所有元素
  → Kruskal 选 n-k 条边后停止

变体：
  P1991 无线通讯网 → k 部卫星电话 = k 个免费连接
  P4047 部落划分   → 分成 k 组 = 保留 k 个连通分量
```

### 5.3 瓶颈路径的等价转化

```
最小瓶颈路径（minimax）：路径最大边权的最小值
  → 最小生成树上的路径最大边权

最大瓶颈路径（maximin）：路径最小边权的最大值
  → 最大生成树上的路径最小边权

求解方法：
  1. MST/MaxST + 倍增  → O(n log n) 预处理, O(log n) 查询
  2. Kruskal 重构树    → O(n log n) 预处理, O(log n) 查询
```

### 5.4 次小生成树的注意点

```
严格 vs 非严格：
  - 非严格次小：w ≥ MST 权值，只需替换路径最大边
  - 严格次小：  w > MST 权值，当非树边权 = 最大边权时需替换次大边

实现要点：
  1. 倍增数组需同时维护最大值和严格次大值
  2. 合并两段路径时，从四个候选值中取最大和严格次大
  3. 初始化次大值为 -INF，表示不存在
```

### 5.5 Kruskal 重构树的应用场景

```
适用问题类型：
  1. "只经过边权 ≤ k 的边，u 能到达哪些点？"
     → 从 u 向上爬到权值 ≤ k 的最高祖先，其子树内所有叶子即可达

  2. "u 到 v 的所有路径中，最大边权的最小值？"
     → LCA(u, v) 的权值

  3. 动态连通性查询（离线）
     → 按边权排序后建重构树
```

## 七、终极总结与归纳

### 7.1 生成树的本质

**最小生成树的本质是：用最小代价保持全局连通性。** 它是图论中贪心思想最完美的体现——无论是 Kruskal 的"最小边优先"还是 Prim 的"最近点优先"，都在每一步做出局部最优选择，最终达到全局最优，其正确性由切割性质和环性质严格保证。

### 7.2 核心要点

1. **Kruskal 是竞赛首选**：排序 + 并查集，代码短、思路清、适用广
2. **切割性质和环性质是理论基础**：理解这两条性质才能理解为什么贪心有效
3. **"MST 提前终止"是高频模型**：分组、通信距离、划分等问题都可归结于此
4. **瓶颈路径 = MST 路径查询**：这个等价关系是很多进阶题的关键
5. **Kruskal 重构树将边权问题转化为点权**：使得 LCA 查询就能回答路径极值
6. **次小生成树需维护路径最大值和严格次大值**：倍增预处理是标准手段
