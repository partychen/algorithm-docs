## 一、什么是最短路问题

**最短路问题**，指的是在一张带权图中，求从某个源点到某个目标点（或到所有点）的最短距离。

最短路问题是图论中最基础、也是应用最广泛的问题之一。大量的竞赛题和实际问题都可以转化为最短路模型：网络路由、地图导航、差分约束、分层状态转移……

用一句话概括：

> **在带权图上，找到从起点到终点的权值总和最小的路径。**

### 最短路问题的分类

| 问题类型 | 算法 | 时间复杂度 | 适用场景 |
|---------|------|-----------|---------|
| 单源最短路（正权） | Dijkstra（优先队列） | $O((n+m)\log n)$ | 正权图，最常用 |
| 单源最短路（负权） | Bellman-Ford / SPFA | $O(nm)$ / 平均 $O(km)$ | 含负权边 |
| 全源最短路（稠密图） | Floyd-Warshall | $O(n^3)$ | $n$ 较小 |
| 全源最短路（稀疏图） | Johnson | $O(nm\log n)$ | 含负权边的稀疏图 |

### 最短路的本质

不管哪种最短路算法，核心都是同一个操作 —— **松弛（Relaxation）**：

> 如果经过中间点 $k$ 能使 $u \to v$ 的距离更短，就更新 $\text{dist}[v]$。

所有算法的区别仅仅在于：**以什么顺序、用什么策略去执行松弛操作。**

## 二、核心思想与前置知识

### 2.1 松弛操作

松弛操作是所有最短路算法的核心。对于一条从 $u$ 到 $v$、权值为 $w$ 的边，松弛操作为：

$$
\text{if } \text{dist}[v] > \text{dist}[u] + w, \quad \text{then } \text{dist}[v] = \text{dist}[u] + w
$$

```cpp
// 松弛操作
void relax(int u, int v, int w) {
    if (dist[v] > dist[u] + w)
        dist[v] = dist[u] + w;
}
```

松弛操作的直觉很简单：如果走 $u$ 这条"弯路"反而更短，那就更新答案。
所有最短路算法都在反复执行这个操作，直到不能再松弛为止。

### 2.2 Dijkstra 算法框架

**适用条件**：边权非负。

**核心思想**：贪心。每次从未确定最短距离的点中，选取 $\text{dist}$ 最小的点 $u$，确定其最短距离，然后用 $u$ 去松弛所有邻居。

**为什么正确**：当所有边权非负时，$\text{dist}$ 最小的未确定点不可能再被更新（因为经过其他未确定点只会更远）。

```
Dijkstra 算法流程：
1. 初始化 dist[s] = 0，其余为 INF
2. 将 (0, s) 放入优先队列（小根堆）
3. 每次取出堆顶 (d, u)：
   a. 若 d > dist[u]，跳过（已过期）
   b. 遍历 u 的所有邻边 (v, w)，执行松弛
   c. 若松弛成功，将 (dist[v], v) 入堆
4. 重复直到堆空
```

```cpp
// Dijkstra 优先队列版
typedef pair<ll,int> pli;
priority_queue<pli, vector<pli>, greater<pli>> pq;
void dijkstra(int s) {
    memset(dist, 0x3f, sizeof dist);
    dist[s] = 0;
    pq.push({0, s});
    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (d > dist[u]) continue;
        for (auto [v, w] : G[u])
            if (dist[v] > dist[u] + w) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
    }
}
```

### 2.3 Bellman-Ford / SPFA 框架

**适用条件**：可以处理负权边。能检测负环。

**Bellman-Ford 核心思想**：暴力松弛所有边，重复 $n-1$ 轮。如果第 $n$ 轮仍能松弛，说明存在负环。

**SPFA**：Bellman-Ford 的队列优化。只有 $\text{dist}$ 被更新的点，才有可能更新其他点。用队列维护"待处理"的点。

```
SPFA 流程：
1. 初始化 dist[s] = 0，将 s 入队
2. 取出队头 u，遍历 u 的所有邻边 (v, w)
3. 若松弛成功，且 v 不在队中，将 v 入队
4. 重复直到队空
5. 如果某个点入队超过 n 次，说明有负环
```

```cpp
// SPFA
bool inq[N];
int cnt[N]; // 入队次数，用于判负环
void spfa(int s) {
    memset(dist, 0x3f, sizeof dist);
    dist[s] = 0; inq[s] = 1;
    queue<int> q;
    q.push(s);
    while (!q.empty()) {
        int u = q.front(); q.pop(); inq[u] = 0;
        for (auto [v, w] : G[u])
            if (dist[v] > dist[u] + w) {
                dist[v] = dist[u] + w;
                if (!inq[v]) {
                    q.push(v); inq[v] = 1;
                    if (++cnt[v] > n) return; // 负环
                }
            }
    }
}
```

**注意**：SPFA 在一般随机图上很快（平均 $O(km)$），但在特殊构造数据上会退化到 $O(nm)$。竞赛中若边权非负，务必使用 Dijkstra。

### 2.4 Floyd 全源最短路框架

**适用条件**：求所有点对之间的最短距离。$n$ 较小（$n \le 500$ 左右）。

**核心思想**：区间 DP 的思想。$\text{dist}[i][j]$ 表示从 $i$ 到 $j$、经过编号不超过 $k$ 的中间点的最短距离。

$$
\text{dist}[i][j] = \min(\text{dist}[i][j],\;\text{dist}[i][k] + \text{dist}[k][j])
$$

```cpp
// Floyd
void floyd() {
    for (int k = 1; k <= n; k++)
        for (int i = 1; i <= n; i++)
            for (int j = 1; j <= n; j++)
                dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]);
}
```

**关键**：三重循环中 $k$ 必须在最外层。

### 2.5 Johnson 算法框架

**适用条件**：含负权边的稀疏图上求全源最短路。

**核心思想**：通过势函数 $h[v]$ 把所有负权边转化为非负权边，然后对每个源点跑 Dijkstra。

**算法步骤**：

1. 新增虚拟源点 $0$，向所有点连边权为 $0$ 的边。
2. 从 $0$ 出发跑 Bellman-Ford（或 SPFA），得到 $h[v] = \text{dist}_0[v]$。若有负环则无解。
3. 对每条边 $(u,v,w)$，将权值重置为 $w' = w + h[u] - h[v] \ge 0$。
4. 对每个点 $s$ 用 Dijkstra 跑最短路，得到 $d'[v]$，真实距离为 $d'[v] - h[s] + h[v]$。

```cpp
// Johnson 算法核心
// 1. spfa(0) 求出 h[]
// 2. 重新赋权：w'(u,v) = w(u,v) + h[u] - h[v]
// 3. 对每个 s 跑 dijkstra
// 4. 真实距离 = d'[v] - h[s] + h[v]
```

**时间复杂度**：$O(nm\log n)$，在稀疏图上远优于 Floyd 的 $O(n^3)$。

## 三、分类与变体详解

### 3.1 单源最短路（正权）— Dijkstra

这是最短路中最常见的类型。题目给出一张正权图和一个源点，求源点到所有点的最短距离。

**算法选择**：优先队列版 Dijkstra，时间复杂度 $O((n+m)\log n)$。

**注意事项**：
- 开 `long long` 防止溢出。
- 初始化 `dist` 为 `0x3f3f3f3f3f3f3f3f`（`long long` 级别）。
- 重边和自环不影响正确性。
- 不要对 Dijkstra 做 `vis` 标记然后松弛——直接用 `d > dist[u]` 判断即可。

### 3.2 单源最短路（负权）— Bellman-Ford / SPFA

当图中存在负权边时，Dijkstra 的贪心不再正确，必须使用 Bellman-Ford 或 SPFA。

**判断负环**：如果 Bellman-Ford 在第 $n$ 轮仍然能松弛，或者 SPFA 中某个点入队超过 $n$ 次，则存在负环。

**适用场景**：差分约束系统、含负权边的图。

### 3.3 全源最短路 — Floyd

Floyd 适用于 $n$ 较小的情况，可以一次求出所有点对之间的最短距离。

**适用场景**：
- $n \le 400 \sim 500$。
- 需要多次查询不同点对之间的距离。
- 传递闭包问题（将 `min` 换成 `|`、`+` 换成 `&`）。

### 3.4 全源最短路 — Johnson

当 $n$ 较大但图是稀疏图（$m$ 远小于 $n^2$），且含负权边时，使用 Johnson 算法。

### 3.5 分层图最短路

**模型**：在原图基础上，允许最多 $k$ 次"特殊操作"（如将某条边权值变为 $0$、免费通行等）。

**建模方法**：将图复制 $k+1$ 层。第 $i$ 层表示已经使用了 $i$ 次特殊操作。在第 $i$ 层执行特殊操作时，连一条跨层边到第 $i+1$ 层的对应节点。

**状态**：$\text{dist}[v][j]$ 表示到达 $v$、已使用 $j$ 次特殊操作的最短距离。可以合并成一维，令节点编号为 $v + j \cdot n$。

```cpp
// 分层图：最多 k 次免费通行
// 建图时，对每条边 (u, v, w)：
// 层内正常边：add(u + i*n, v + i*n, w)  对所有 0 <= i <= k
// 跨层免费边：add(u + i*n, v + (i+1)*n, 0)  对所有 0 <= i < k
// 答案：min(dist[t + i*n]) 对所有 0 <= i <= k
```

### 3.6 最短路计数

**问题**：在最短路的基础上，求从源点到每个点的最短路条数。

**方法**：在 Dijkstra/SPFA 松弛时，维护一个额外数组 `cnt[]`：
- 若 $\text{dist}[v] > \text{dist}[u] + w$：更新 $\text{dist}[v]$，且 $\text{cnt}[v] = \text{cnt}[u]$。
- 若 $\text{dist}[v] = \text{dist}[u] + w$：$\text{cnt}[v] += \text{cnt}[u]$。

```cpp
// 最短路计数（在 Dijkstra 内）
if (dist[v] > dist[u] + w) {
    dist[v] = dist[u] + w;
    cnt[v] = cnt[u];
    pq.push({dist[v], v});
} else if (dist[v] == dist[u] + w) {
    cnt[v] = (cnt[v] + cnt[u]) % MOD;
}
```

### 3.7 二分 + 最短路

**模型**：有些题目并不直接求最短路，而是要求"在满足某个条件下的最优解"。条件具有单调性时，可以二分答案，然后将判定问题转化为最短路。

**典型题型**：
- 限制经过的点/边的某个属性不超过阈值，求最短路。
- 二分阈值，删掉不满足条件的点/边，跑最短路判断可达性。

## 四、11 道经典例题详解

### 题目 1：单源最短路径（弱化版）（洛谷 P3371）

> 给定一张 $n$ 个点 $m$ 条边的有向图，求从源点 $s$ 出发到所有点的最短距离。边权可能为 $0$。$n \le 10^4$，$m \le 5 \times 10^5$。

**难度**：🟢

**分析**：

```
这是最短路的入门模板题。数据范围较弱，Dijkstra / SPFA 均可。
直接套用优先队列 Dijkstra 即可。
注意初始化 dist 为 INF，最后不可达的点输出 2^{31} - 1。
```

**C++ 代码**：

```cpp
const int INF = 0x7fffffff;
ll dist[N];
bool vis[N];
vector<pair<int,int>> G[N];

void dijkstra(int s) {
    memset(dist, 0x3f, sizeof dist);
    dist[s] = 0;
    priority_queue<pair<ll,int>, vector<pair<ll,int>>, greater<>> pq;
    pq.push({0, s});
    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (d > dist[u]) continue;
        for (auto [v, w] : G[u])
            if (dist[v] > dist[u] + w) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
    }
    // 输出时：dist[i] >= 0x3f3f3f3f3f3f3f3f 则输出 INF = 2^31-1
}
```

**核心启示**：Dijkstra 优先队列版是正权图单源最短路的标准写法。跳过过期状态（`d > dist[u]`）是效率关键。

### 题目 2：单源最短路径（标准版）（洛谷 P4779）

> 与 P3371 相同题意，但 $n \le 10^5$，$m \le 2 \times 10^5$，数据经过加强，SPFA 会被卡。

**难度**：🟢

**分析**：

```
数据加强后，SPFA 在特殊构造的数据上会 TLE。
必须使用优先队列版 Dijkstra。
代码与 P3371 本质相同，但要注意开 long long。
```

**C++ 代码**：

```cpp
ll dist[N];
vector<pair<int,int>> G[N];

void dijkstra(int s) {
    memset(dist, 0x3f, sizeof dist);
    dist[s] = 0;
    priority_queue<pair<ll,int>, vector<pair<ll,int>>, greater<>> pq;
    pq.push({0, s});
    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (d > dist[u]) continue;
        for (auto [v, w] : G[u])
            if (dist[v] > dist[u] + w) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
    }
}
```

**核心启示**：在正权图上，永远优先使用 Dijkstra 而非 SPFA。"SPFA 已死"在竞赛中是铁律——出题人会刻意卡 SPFA。

### 题目 3：Johnson 全源最短路（洛谷 P5905）

> 给定一张 $n$ 个点 $m$ 条边的有向图，可能有负权边。求所有点对之间的最短距离。若存在负环输出 $-1$。$n \le 3000$，$m \le 6000$。

**难度**：🟡

**分析**：

```
含负权边，不能直接用 Dijkstra。Floyd 复杂度 O(n^3) = 2.7×10^10，太慢。
Johnson 算法：
1. 加虚拟源点 0，向所有点连权 0 的边。
2. 从 0 跑 SPFA 得势函数 h[]；若有负环返回 -1。
3. 重新赋权：w'(u,v) = w + h[u] - h[v]，保证所有边权非负。
4. 对每个源点跑 Dijkstra，最后还原真实距离。
复杂度 O(nm log n)，可以通过。
```

**C++ 代码**：

```cpp
ll h[N], d[N];
bool inq[N]; int cnt[N];

bool spfa_init() { // 从虚拟源 0 跑 SPFA 求 h[]
    memset(h, 0x3f, sizeof h);
    queue<int> q; h[0] = 0; q.push(0); inq[0] = 1;
    while (!q.empty()) {
        int u = q.front(); q.pop(); inq[u] = 0;
        for (auto [v, w] : G[u])
            if (h[v] > h[u] + w) {
                h[v] = h[u] + w;
                if (!inq[v]) { q.push(v); inq[v] = 1; if (++cnt[v] > n) return false; }
            }
    }
    return true;
}

void johnson() {
    for (int i = 1; i <= n; i++) G[0].push_back({i, 0});
    if (!spfa_init()) { /* 负环 */ return; }
    // 重赋权
    for (int u = 1; u <= n; u++)
        for (auto &[v, w] : G[u]) w += h[u] - h[v];
    // 对每个源点跑 Dijkstra
    for (int s = 1; s <= n; s++) {
        dijkstra(s); // 结果在 d[] 中
        // 真实距离: d[v] - h[s] + h[v]
    }
}
```

**核心启示**：Johnson 算法是含负权稀疏图全源最短路的最优选择。势函数重赋权的技巧非常经典，本质是利用三角不等式将负权消除。

### 题目 4：最短路计数（洛谷 P1144）

> 给定一张 $n$ 个点 $m$ 条边的无向无权图，求从 $1$ 出发到每个点的最短路条数（对 $100003$ 取模）。$n \le 10^6$，$m \le 2 \times 10^6$。

**难度**：🟢

**分析**：

```
无权图的最短路用 BFS 即可（边权都是 1）。
在 BFS 松弛时额外维护 cnt[v]：
- dist[v] 被首次更新时：cnt[v] = cnt[u]
- dist[v] == dist[u] + 1 时（等距路径）：cnt[v] += cnt[u]
```

**C++ 代码**：

```cpp
int dist[N], cnt[N];
vector<int> G[N];

void bfs() {
    memset(dist, -1, sizeof dist);
    queue<int> q;
    dist[1] = 0; cnt[1] = 1; q.push(1);
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int v : G[u]) {
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                cnt[v] = cnt[u];
                q.push(v);
            } else if (dist[v] == dist[u] + 1) {
                cnt[v] = (cnt[v] + cnt[u]) % 100003;
            }
        }
    }
}
```

**核心启示**：最短路计数的关键在于松弛时多一个"等距则累加"的判断。无权图用 BFS 即可，不需要 Dijkstra。

### 题目 5：通往奥格瑞玛的道路（洛谷 P1462）

> $n$ 个点 $m$ 条边的无向图，每个点有费用 $f_i$，每条边有血量消耗 $w$。从 $1$ 到 $n$，初始血量为 $b$，每经过一条边消耗对应血量（降到 $0$ 或以下死亡），经过一个点就要缴纳该点的费用。最小化经过的所有点中费用的最大值，使得血量足够到达终点。

**难度**：🟡

**分析**：

```
"最小化最大值"→ 二分答案。
二分"允许经过的最大费用"mid：
- 只保留 f[v] <= mid 的点
- 在这些点构成的子图上跑 Dijkstra，求 1 到 n 的最短距离
- 若 dist[n] < b，则 mid 可行
注意：f[1] 和 f[n] 也受约束，若 f[1] 或 f[n] > mid 则不可行。
```

**C++ 代码**：

```cpp
int f[N]; // 每个点的费用
ll dist[N];
vector<pair<int,int>> G[N];

bool check(int mid) {
    if (f[1] > mid || f[n] > mid) return false;
    memset(dist, 0x3f, sizeof dist);
    priority_queue<pair<ll,int>, vector<pair<ll,int>>, greater<>> pq;
    dist[1] = 0; pq.push({0, 1});
    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (d > dist[u]) continue;
        for (auto [v, w] : G[u]) {
            if (f[v] > mid) continue; // 费用超标，跳过
            if (dist[v] > dist[u] + w) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }
    return dist[n] < b;
}
// 外层二分：对 f[] 排序去重后二分，找最小可行的 mid
```

**核心启示**：当目标是最小化"路径上的最大值"时，优先考虑二分答案 + 最短路判定。在松弛时跳过不合法的点即可实现条件过滤。

### 题目 6：Cow Tours（洛谷 P1522）

> $n$ 个点的平面图，用邻接矩阵给出连通性。图可能不连通（多个连通分量）。可以在任意两个不同连通分量的点之间加一条边（欧几里得距离），求加边后整个图直径的最小值。$n \le 150$。

**难度**：🟡

**分析**：

```
n <= 150，可以用 Floyd 求全源最短路。
1. Floyd 求出所有点对最短距离（同一连通分量内）。
2. 对每个点 u，求 far[u] = max(dist[u][v])，即 u 在其连通分量中的最远距离。
3. 对每个连通分量，求其直径 dia[c] = max(far[u])。
4. 枚举不同连通分量的点 (u, v)，连接后新直径为：
   max(dia[cu], dia[cv], far[u] + euclidean(u,v) + far[v])
5. 答案 = min(上述值)，还要和所有 dia[c] 取 max。
```

**C++ 代码**：

```cpp
double d[N][N], far[N], dia[N]; // dia 按连通分量
int comp[N]; // 连通分量编号
double x[N], y[N];

void solve() {
    // 初始化 d[][]，同连通分量 Floyd
    for (int k = 1; k <= n; k++)
        for (int i = 1; i <= n; i++)
            for (int j = 1; j <= n; j++)
                if (d[i][k] + d[k][j] < d[i][j])
                    d[i][j] = d[i][k] + d[k][j];
    // 求 far[] 和 dia[]
    for (int u = 1; u <= n; u++)
        for (int v = 1; v <= n; v++)
            if (comp[u] == comp[v] && d[u][v] < INF)
                far[u] = max(far[u], d[u][v]);
    // 求每个分量直径
    for (int u = 1; u <= n; u++) dia[comp[u]] = max(dia[comp[u]], far[u]);
    // 枚举加边
    double ans = 1e18;
    for (int u = 1; u <= n; u++)
        for (int v = u+1; v <= n; v++)
            if (comp[u] != comp[v]) {
                double val = far[u] + hypot(x[u]-x[v], y[u]-y[v]) + far[v];
                val = max({val, dia[comp[u]], dia[comp[v]]});
                ans = min(ans, val);
            }
}
```

**核心启示**：合并连通分量时，新直径 = max(两个分量各自的直径, 跨分量的最长路径)。Floyd + 枚举加边是经典套路。

### 题目 7：速度限制（洛谷 P1266）

> 有向图，每条边有限速 $v_i$ 和长度 $l_i$。若 $v_i = 0$ 表示该段没有限速，可以使用任意之前设定的速度。从起点出发，初始速度需要自选。求到达终点的最短时间。

**难度**：🟡

**分析**：

```
这是一个二指标最短路问题。状态不仅取决于"当前在哪个点"，还取决于"当前速度是多少"。
将状态扩展为 (节点, 速度)，在扩展图上跑 Dijkstra。
- 边有限速：速度被强制设为 v_i，时间 = l_i / v_i
- 边无限速（v_i=0）：保持当前速度 spd，时间 = l_i / spd
由于速度是离散且范围有限，可以直接开二维 dist[节点][速度]。
```

**C++ 代码**：

```cpp
double dist[N][MAXV]; // dist[u][spd]
struct Edge { int v, lim, len; };
vector<Edge> G[N];

void dijkstra() {
    // 初始化全部为 INF
    priority_queue<tuple<double,int,int>, vector<tuple<double,int,int>>, greater<>> pq;
    // 枚举起始速度
    for (int s = 1; s <= maxv; s++) {
        dist[0][s] = 0;
        pq.push({0, 0, s});
    }
    while (!pq.empty()) {
        auto [d, u, spd] = pq.top(); pq.pop();
        if (d > dist[u][spd] + 1e-9) continue;
        for (auto &[v, lim, len] : G[u]) {
            int ns = lim ? lim : spd; // 有限速用限速，否则沿用
            double nd = d + (double)len / ns;
            if (nd < dist[v][ns] - 1e-9) {
                dist[v][ns] = nd;
                pq.push({nd, v, ns});
            }
        }
    }
    // 答案 = min(dist[t][spd]) 对所有 spd
}
```

**核心启示**：当路径上有影响后续决策的"状态量"时，将该状态量加入最短路节点定义中，扩展状态空间再跑 Dijkstra — 这就是"二指标最短路"的核心思想。

### 题目 8：[ICPC-Beijing 2006] 狼抓兔子（洛谷 P4001）

> $n \times m$ 的网格图，有横向、纵向和对角线三种边。求从左上角 $(1,1)$ 到右下角 $(n,m)$ 的最小割。$n,m \le 1000$。

**难度**：🟠

**分析**：

```
平面图最小割 = 对偶图最短路。
对偶图构建：
- 将网格中的每个三角形视为对偶图的一个节点。
- 原图一条边对应对偶图中相邻两个三角形之间的一条边。
- 源点 S 连接上边界和右边界外侧的对偶点。
- 汇点 T 连接下边界和左边界外侧的对偶点。
在对偶图上跑 Dijkstra 求 S 到 T 的最短路，即为原图的最小割。
注意特判 n=1 或 m=1 的情况。
```

**C++ 代码**：

```cpp
// 对偶图节点编号：每个格子拆成上三角和下三角
// 上三角 id = (i*m + j)*2，下三角 = (i*m + j)*2 + 1
// S = tot, T = tot+1
ll dist[MAXNODE];
vector<pair<int,int>> G[MAXNODE];

void solve() {
    if (n == 1 || m == 1) { /* 特判：取边权最小值 */ return; }
    int S = 2*(n-1)*(m-1), T = S+1;
    // 对每条原图边，在对偶图的两个相邻三角形之间连边
    // 横边：上下三角之间
    // 纵边：左右三角之间
    // 对角边：同一格子的上下三角之间
    // 边界边：连 S 或 T
    dijkstra(S);
    // 答案 = dist[T]
}
```

**核心启示**：平面图最小割可以通过对偶图转化为最短路问题。这是一个重要的图论技巧。网格图的对偶图构建需要仔细处理边界。

### 题目 9：[JLOI2011] 飞行路线（洛谷 P4568）

> $n$ 个点 $m$ 条边的无向图，有 $k$ 次免费通行的机会（将某条边的权值视为 $0$）。求从 $s$ 到 $t$ 的最短距离。$n \le 10^4$，$m \le 5 \times 10^4$，$k \le 10$。

**难度**：🟠

**分析**：

```
经典分层图最短路模型。
建 k+1 层图，第 i 层表示已经使用了 i 次免费机会。
对每条边 (u, v, w)：
- 层内边：第 i 层的 u → 第 i 层的 v，权为 w
- 跨层边：第 i 层的 u → 第 i+1 层的 v，权为 0（使用一次免费）
在分层图上跑 Dijkstra，答案 = min(dist[t + i*n]) 对 0 <= i <= k。
```

**C++ 代码**：

```cpp
// 节点编号：第 i 层的 v = v + i * n
ll dist[N * 12]; // N * (k+1)
vector<pair<int,int>> G[N * 12];

void build(int n, int k) {
    // 对每条原边 (u, v, w):
    for (int i = 0; i <= k; i++) {
        add(u + i*n, v + i*n, w); // 层内
        add(v + i*n, u + i*n, w);
        if (i < k) {
            add(u + i*n, v + (i+1)*n, 0); // 跨层
            add(v + i*n, u + (i+1)*n, 0);
        }
    }
}
// Dijkstra(s)，答案 = min(dist[t + i*n], 0 <= i <= k)
```

**核心启示**：分层图是处理"最多 $k$ 次特殊操作"的通用模型。将操作次数编码进节点编号，就能将带操作次数约束的最短路转化为普通最短路。

### 题目 10：[GXOI/GZOI2019] 旅行者（洛谷 P5304）

> 给定 $n$ 个点 $m$ 条边的有向图，$k$ 个关键点。求所有关键点对之间的最短距离的最小值。$n \le 10^5$，$m \le 5 \times 10^5$。

**难度**：🟠

**分析**：

```
暴力：对每个关键点跑 Dijkstra，O(k(n+m)log n)，k 可达 n，太慢。
技巧：将 k 个关键点随机二分为两组 A 和 B。
- 建虚拟源点 S 向 A 中所有点连权 0 的边，跑 Dijkstra 得 distA[]。
- 建虚拟源点 T 向 B 中所有点连权 0 的边，跑 Dijkstra 得 distB[]。
- 答案候选 = min(distA[v]) 其中 v ∈ B（或反过来）。
随机多轮取最小值。实际上只需要：
    按二进制位分组——对每一位，按该位为 0/1 分成两组，
    正反各跑一次 Dijkstra，即可保证任意两个不同关键点
    在某一位上不同，从而被分入不同组。
总共 O(log k) 轮，每轮两次 Dijkstra，复杂度 O((n+m)log n * log k)。
```

**C++ 代码**：

```cpp
ll distA[N], distB[N], ans;
int key[N]; // 关键点

void solve() {
    ans = LLONG_MAX;
    for (int bit = 0; (1 << bit) <= k; bit++) {
        // A = key[i] 的第 bit 位为 0, B = 为 1
        // 从 A 出发跑正图 Dijkstra -> distA
        // 从 B 出发跑反图 Dijkstra -> distB（等价于反图上多源）
        // 更新 ans = min(ans, distA[v]) for v in B
        // 再反过来
        multiSourceDijkstra(A, distA); // 正图
        multiSourceDijkstra(B, distB); // 反图
        for (int v : B) ans = min(ans, distA[v]);
        for (int v : A) ans = min(ans, distB[v]);
    }
}
```

**核心启示**：多关键点对最短路问题，按二进制位分组是经典技巧。每一位将关键点分为两组，保证任意不同的两个点至少在某一位被分开，从而以 $O(\log k)$ 轮 Dijkstra 覆盖所有点对。

### 题目 11：Indecisive Taxi Fee（Codeforces CF1163F）

> 一张 $n$ 个点 $m$ 条边的无向图。给定 $1$ 到 $n$ 的某条最短路 $P$。$q$ 次询问，每次将第 $e_i$ 条边的权值改为 $w_i$，问修改后 $1$ 到 $n$ 的最短路长度。每次询问独立。

**难度**：🟣

**分析**：

```
先求 1 到 n 的一条最短路 P = (v_0=1, v_1, ..., v_t=n)。
设原最短路长度为 D。修改边 e 时：
1. 若 e 不在 P 上：新最短路 = min(D, 不经过 P 上的边而经过修改后 e 的路径)。
   具体地，新最短路 = min(D, dist1[u] + w_new + distn[v])（其中 e=(u,v)）
   和 min(D, dist1[v] + w_new + distn[u])。
2. 若 e 在 P 上：需要求"不经过 e 的最短路"。
   - 删掉 P 上第 i 条边后的最短路 = 绕行值 bypass[i]。
   - 预处理：对 P 外的每条边 (u,v,w)，它能替代 P 上从 l 到 r 的一段，
     贡献为 dist1[u] + w + distn[v]。
   - 这等价于区间赋值取 min，用线段树维护。
   - 最终答案 = min(D - w_old + w_new, bypass[i])。

预处理 dist1[]（从 1 出发）和 distn[]（从 n 出发）。
对 P 上每条边 (v_{i-1}, v_i)，bypass[i] = 不经过该边的最短路。
用线段树维护 bypass[]。
```

**C++ 代码**：

```cpp
ll dist1[N], distn[N]; // 从 1/ 从 n 的最短路
int path[N]; int plen; // 最短路上的点
ll bypass[N]; // bypass[i] = 删掉第 i 条最短路边后的最短距离

// 线段树区间取 min
struct SegTree {
    ll mn[N*4];
    void build(int o,int l,int r) { mn[o]=LLONG_MAX; if(l==r)return; int m=(l+r)/2; build(o*2,l,m); build(o*2+1,m+1,r); }
    void upd(int o,int l,int r,int ql,int qr,ll v) {
        if(ql>qr||l>qr||r<ql)return;
        if(ql<=l&&r<=qr){mn[o]=min(mn[o],v);return;}
        int m=(l+r)/2; upd(o*2,l,m,ql,qr,v); upd(o*2+1,m+1,r,ql,qr,v);
    }
    ll qry(int o,int l,int r,int p) {
        if(l==r)return mn[o];
        int m=(l+r)/2; ll res=mn[o];
        return min(res, p<=m?qry(o*2,l,m,p):qry(o*2+1,m+1,r,p));
    }
} seg;

void preprocess() {
    dijkstra_from_1(); dijkstra_from_n(); // 求 dist1[], distn[]
    // 还原最短路 path[]
    // 对每条非最短路边 (u,v,w)：
    //   找到 u 在 path 上最晚能从 1 到达的位置 l（即 dist1[u] 经过 path[0..l]）
    //   找到 v 在 path 上最早能到 n 的位置 r
    //   seg.upd(l+1, r, dist1[u] + w + distn[v])
    // 查询时：bypass[i] = seg.qry(i)
}

void answer_query(int edge_id, ll new_w) {
    if (edge_on_path(edge_id)) {
        int idx = pos_on_path(edge_id);
        ll old_w = weight(edge_id);
        // 答案 = min(D - old_w + new_w, bypass[idx])
    } else {
        auto [u, v] = endpoints(edge_id);
        ll alt = min(dist1[u] + new_w + distn[v], dist1[v] + new_w + distn[u]);
        // 答案 = min(D, alt)
    }
}
```

**核心启示**：最短路树 + 线段树维护绕行值，是处理"删边/改边后最短路"问题的高级技巧。对最短路上每条边预处理其"绕行最短路"，利用线段树区间取 min 来高效维护，是这类问题的核心框架。

## 五、常见技巧与思路总结

### 1. 建图技巧

| 技巧 | 说明 | 典型应用 |
|------|------|---------|
| 反向建图 | 建反向边，从终点出发跑最短路 | 求所有点到某终点的最短距离 |
| 虚拟源/汇点 | 新增超级源/汇连接多个起点/终点 | 多源最短路 |
| 分层图 | 将状态分解到多层图中 | 限次免费、分阶段决策 |
| 拆点 | 将一个节点拆成多个状态节点 | 经过点有代价、状态依赖 |
| 对偶图 | 平面图最小割转最短路 | 网格图最小割 |

### 2. 常见思维套路

- **"最大值最小"或"最小值最大"** → 二分答案 + 最短路判定
- **"恰好 k 次操作"** → 分层图最短路
- **"最短路上有多少条路径"** → 最短路计数
- **"删边/改边后的最短路"** → 最短路树 + bypass 预处理
- **"多个关键点对之间的最短路"** → 二进制分组 + 多源 Dijkstra
- **"负权边"** → Bellman-Ford / SPFA / Johnson（重赋权）

### 3. 常见陷阱

- **SPFA 被卡**：正权图请务必使用 Dijkstra。
- **溢出**：$\text{dist}$ 开 `long long`，INF 不要设太大导致加法溢出。
- **重边与自环**：Dijkstra 不受影响，但 Floyd 初始化要取 min。
- **0 权边**：Dijkstra 仍然正确（非负即可）。
- **无向图跑 SPFA**：同一条无向边松弛会在两个方向都起作用，注意。

## 六、优化方法 / 解题框架

### 解题框架

```
1. 分析图的性质：
   - 有无负权边？→ 有则用 Bellman-Ford/SPFA/Johnson，无则用 Dijkstra
   - 单源还是全源？→ 单源用 Dijkstra/SPFA，全源用 Floyd/Johnson
   - n 的大小？→ n <= 500 可 Floyd，n 较大用 Dijkstra

2. 分析题目变体：
   - 最短路计数？→ 松弛时维护 cnt
   - 分层图？→ 建多层图 or 多维 dist
   - 二分 + 最短路？→ 二分答案 + 过滤后跑 Dijkstra
   - 平面图最小割？→ 对偶图最短路

3. 建图：
   - 确定节点和边的含义
   - 是否需要虚拟源/汇、分层、拆点、反向建图

4. 实现：
   - 优先队列 Dijkstra 是最核心的板子
   - 注意开 long long、INF 的选取、过期状态的跳过
```

### 各算法选择指南

```
问题类型            算法              条件
──────────────────────────────────────────────────
单源正权            Dijkstra (PQ)     边权 >= 0
单源负权            SPFA / BF         有负权边
全源小 n            Floyd             n <= 500
全源负权稀疏        Johnson           有负权，m << n²
判负环              BF / SPFA         第 n 轮仍松弛 / 入队 > n 次
无权图最短路        BFS               边权全为 1
```

### 复杂度一览

| 算法 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| Dijkstra（优先队列） | $O((n+m)\log n)$ | $O(n+m)$ |
| Bellman-Ford | $O(nm)$ | $O(n+m)$ |
| SPFA（平均） | $O(km)$，最坏 $O(nm)$ | $O(n+m)$ |
| Floyd | $O(n^3)$ | $O(n^2)$ |
| Johnson | $O(nm\log n)$ | $O(n+m)$ |

## 七、终极总结与归纳

### 最短路知识图谱

```
最短路问题
├── 单源最短路
│   ├── 正权 → Dijkstra（优先队列版）
│   └── 负权 → Bellman-Ford / SPFA
├── 全源最短路
│   ├── 稠密图 / 小 n → Floyd
│   └── 稀疏图 / 负权 → Johnson
└── 变体
    ├── 分层图最短路 → 复制 k+1 层 + 跨层边
    ├── 最短路计数 → 松弛时维护 cnt
    ├── 二分 + 最短路 → 二分阈值 + 过滤 + 判定
    ├── 二指标最短路 → 扩展状态维度
    ├── 多源多汇 → 虚拟源/汇点
    ├── 平面图最小割 → 对偶图最短路
    └── 动态删边/改边 → 最短路树 + 线段树
```

### 一句话记忆

| 算法 | 一句话 |
|------|-------|
| Dijkstra | 贪心：每次取最近的未确定点，正权图首选 |
| Bellman-Ford | 暴力：松弛所有边 $n-1$ 轮，能判负环 |
| SPFA | BF 的队列优化，正权图会被卡，慎用 |
| Floyd | DP：枚举中间点 $k$，$O(n^3)$ 求全源，$k$ 在最外层 |
| Johnson | 势函数消负权，然后跑 $n$ 次 Dijkstra |

### 做题心法

1. **先判图的性质**：正权 / 负权 / 无权 / 平面图 — 决定算法选择。
2. **再看问题类型**：单源 / 全源 / 计数 / 限制操作次数 — 决定建模方式。
3. **最后想变形**：二分答案、分层图、拆点、反向建图 — 决定建图技巧。
4. **代码层面**：优先队列 Dijkstra 是最基础的板子，写熟即可应对大部分题目。
5. **SPFA 只在必须处理负权边时使用**，正权图一律 Dijkstra。
