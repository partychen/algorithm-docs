## 一、什么是最大流

在图论中，**最大流问题**（Maximum Flow Problem）是网络流理论的核心问题之一。给定一个**流网络**（Flow Network），它是一个有向图 $G = (V, E)$，其中每条边 $(u, v) \in E$ 都有一个非负的**容量** $c(u, v) \geq 0$。网络中存在两个特殊节点：**源点** $s$（source）和**汇点** $t$（sink）。我们的目标是找到从源点 $s$ 到汇点 $t$ 的最大流量。

**流**（Flow）是一个定义在边集上的函数 $f(u, v)$，它必须满足两个基本约束：（1）**容量限制**：对于所有边 $(u, v)$，有 $0 \leq f(u, v) \leq c(u, v)$，即流量不能超过容量；（2）**流量守恒**：对于除源点和汇点之外的所有节点 $v$，流入 $v$ 的总流量等于流出 $v$ 的总流量，即 $\sum_{u} f(u, v) = \sum_{w} f(v, w)$。最大流就是在满足这两个约束下，从 $s$ 流出的总流量的最大值。

最大流问题有着极为广泛的应用场景：交通网络中的最大运输量、通信网络中的最大带宽、二分图匹配、项目选择、图像分割等问题都可以建模为最大流问题。著名的**最大流最小割定理**（Max-Flow Min-Cut Theorem）指出，网络的最大流等于最小割的容量，这一定理将最大流与最小割这两个看似不同的问题联系在了一起，是组合优化中最深刻的结果之一。

## 二、核心思想与前置知识

### 2.1 增广路定理（Ford-Fulkerson 方法）

**增广路定理**是所有最大流算法的理论基础。在**残量网络**（Residual Graph）$G_f$ 中，如果不存在从 $s$ 到 $t$ 的增广路径，则当前流 $f$ 就是最大流。

残量网络的定义：对于原图中的每条边 $(u, v)$，如果 $f(u, v) < c(u, v)$，则在残量网络中存在一条**前向边** $(u, v)$，残量为 $c(u, v) - f(u, v)$；同时存在一条**反向边** $(v, u)$，残量为 $f(u, v)$。反向边的存在是增广路方法的关键——它允许算法"撤销"之前的错误决策。

Ford-Fulkerson 方法的基本框架：

1. 初始化所有边的流量为 0。
2. 在残量网络中寻找一条从 $s$ 到 $t$ 的增广路径。
3. 沿增广路径增加流量（增加量为路径上最小残量）。
4. 重复步骤 2-3，直到没有增广路径为止。

> **注意**：Ford-Fulkerson 是一种方法（Method），而非算法（Algorithm），因为它没有指定如何寻找增广路。不同的寻路策略产生了不同的具体算法。

### 2.2 EK 算法（Edmonds-Karp，BFS 找增广路）

EK 算法是 Ford-Fulkerson 方法的一种具体实现，它使用 **BFS** 在残量网络中寻找**最短增广路**（边数最少的增广路）。

- **时间复杂度**：$O(VE^2)$
- **核心思想**：每次 BFS 找到一条最短增广路并沿路增广。由于 BFS 保证了增广路的长度单调不减，因此总增广次数不超过 $O(VE)$ 次。
- **优点**：实现简单，适用于小规模网络。
- **缺点**：对于大规模网络效率不足，每次只增广一条路径。

### 2.3 Dinic 算法（BFS 分层 + DFS 阻塞流）

Dinic 算法是实际竞赛中最常用的最大流算法，它在 EK 算法的基础上做了重要优化：先用 BFS 构建**分层图**（Level Graph），再用 DFS 在分层图中寻找**阻塞流**（Blocking Flow），一次 DFS 可以找到多条增广路。

- **时间复杂度**：$O(V^2 E)$，对于单位容量图为 $O(E\sqrt{V})$
- **关键优化**：**当前弧优化**（current arc optimization）——在 DFS 过程中，记录每个节点已经访问到哪条出边，避免重复遍历已经无法增广的边。

**Dinic 算法 C++ 核心模板**：

```cpp
struct MaxFlow {
    struct Edge { int to, rev; long long cap; };
    vector<vector<Edge>> graph;
    vector<int> level, iter;
    int n;

    MaxFlow(int n) : n(n), graph(n), level(n), iter(n) {}

    void add_edge(int from, int to, long long cap) {
        graph[from].push_back({to, (int)graph[to].size(), cap});
        graph[to].push_back({from, (int)graph[from].size() - 1, 0});
    }

    bool bfs(int s, int t) {
        fill(level.begin(), level.end(), -1);
        queue<int> q;
        level[s] = 0;
        q.push(s);
        while (!q.empty()) {
            int v = q.front(); q.pop();
            for (auto& e : graph[v]) {
                if (e.cap > 0 && level[e.to] < 0) {
                    level[e.to] = level[v] + 1;
                    q.push(e.to);
                }
            }
        }
        return level[t] >= 0;
    }

    long long dfs(int v, int t, long long f) {
        if (v == t) return f;
        for (int& i = iter[v]; i < (int)graph[v].size(); i++) {
            Edge& e = graph[v][i];
            if (e.cap > 0 && level[v] < level[e.to]) {
                long long d = dfs(e.to, t, min(f, e.cap));
                if (d > 0) {
                    e.cap -= d;
                    graph[e.to][e.rev].cap += d;
                    return d;
                }
            }
        }
        return 0;
    }

    long long max_flow(int s, int t) {
        long long flow = 0;
        while (bfs(s, t)) {
            fill(iter.begin(), iter.end(), 0);
            long long d;
            while ((d = dfs(s, t, LLONG_MAX)) > 0)
                flow += d;
        }
        return flow;
    }
};
```

### 2.4 ISAP 算法简介

ISAP（Improved Shortest Augmenting Path）是对 Dinic 算法的进一步优化。它的核心改进在于：只做**一次 BFS**（从汇点反向 BFS），之后在 DFS 增广过程中**动态维护**距离标号。当某个距离标号出现**断层**（gap）时，可以立即判定不存在增广路，从而提前终止。

- **时间复杂度**：理论上与 Dinic 相同，均为 $O(V^2 E)$，但实际运行通常更快。
- **GAP 优化**：维护每个距离标号的节点数。当某个距离标号的节点数变为 0 时，说明图已断开，算法终止。
- **适用场景**：在稠密图上往往比 Dinic 快，是部分竞赛选手的首选。

### 2.5 预流推进简介

**预流推进**（Push-Relabel / Preflow-Push）是一种与增广路方法完全不同的思路。它不再寻找从 $s$ 到 $t$ 的完整路径，而是维护一个**预流**（preflow）——允许中间节点流入量大于流出量（产生**溢出**），然后通过 push（推流）和 relabel（重标号）操作逐步将多余的流量推回源点或推向汇点。

- **基本预流推进**：$O(V^2 E)$
- **HLPPS（最高标号预流推进）**：$O(V^2 \sqrt{E})$，理论和实际表现都非常优秀。
- **核心操作**：push（将溢出节点的多余流量推向邻居）和 relabel（当无法 push 时提高节点的高度标号）。
- **适用场景**：大规模稠密图，洛谷加强版模板题需要此算法。

### 2.6 常见建模技巧

**1. 拆点**：当节点有容量限制时（如每个节点最多经过 $k$ 次），将节点 $v$ 拆为入点 $v_{in}$ 和出点 $v_{out}$，中间连一条容量为 $k$ 的边。

**2. 分层图**：当问题涉及时间、阶段等概念时，将同一节点在不同时间/阶段复制为不同的节点，构成分层图。相邻层之间连边表示状态转移。

**3. 超级源汇**：当有多个源点或多个汇点时，新建一个超级源点 $S$ 连向所有源点，所有汇点连向一个超级汇点 $T$，容量根据题意设定。

**4. 无向边建模**：将无向边 $(u, v, c)$ 拆成两条有向边 $(u, v, c)$ 和 $(v, u, c)$。

**5. 最大权闭合子图**：利用最小割建模，选择权值为正的节点则从源点连边，权值为负的节点则连向汇点，依赖关系用无穷大容量边表示。答案 = 正权值之和 - 最小割。

## 三、分类与变体详解

### 3.1 EK 算法

| 特性 | 说明 |
|------|------|
| **核心方法** | BFS 寻找最短增广路 |
| **时间复杂度** | $O(VE^2)$ |
| **空间复杂度** | $O(V + E)$ |
| **增广路长度** | 单调不减 |
| **适用场景** | 小规模网络，$V \leq 500, E \leq 5000$ |
| **实现难度** | 简单 |

EK 算法的核心循环：每次 BFS 找到从 $s$ 到 $t$ 的最短路径，记录路径上的最小残量 $\delta$，然后沿路径更新残量网络（前向边减 $\delta$，反向边加 $\delta$）。当 BFS 无法到达 $t$ 时，算法结束。

### 3.2 Dinic 算法

| 特性 | 说明 |
|------|------|
| **核心方法** | BFS 分层 + DFS 阻塞流 |
| **时间复杂度** | $O(V^2 E)$，单位容量图 $O(E\sqrt{V})$ |
| **空间复杂度** | $O(V + E)$ |
| **关键优化** | 当前弧优化、多路增广 |
| **适用场景** | 中等规模网络，竞赛首选 |
| **实现难度** | 中等 |

Dinic 算法的优势在于每个阶段（一次 BFS 分层 + 多次 DFS）可以找到所有最短增广路构成的阻塞流。分层图保证了 DFS 只沿着层次递增的方向前进。当前弧优化保证每条边至多被访问常数次。分层图的层数最多为 $V-1$，且每个阶段后层数严格增加，这保证了最多 $V-1$ 个阶段。

### 3.3 ISAP

| 特性 | 说明 |
|------|------|
| **核心方法** | 动态维护距离标号 + GAP 优化 |
| **时间复杂度** | $O(V^2 E)$（与 Dinic 相同） |
| **空间复杂度** | $O(V + E)$ |
| **关键优化** | GAP 断层优化、只需一次反向 BFS |
| **适用场景** | 稠密图，追求常数优化 |
| **实现难度** | 中等偏高 |

ISAP 的工作流程：（1）从 $t$ 反向 BFS，初始化所有节点的距离标号；（2）从 $s$ 出发 DFS 寻找增广路，沿距离标号递减的方向前进；（3）当节点无法推进时，将其距离标号更新为所有可达邻居中最小标号加 1；（4）当某个标号值的节点数变为 0（GAP）时，算法终止。

### 3.4 预流推进

| 特性 | 说明 |
|------|------|
| **核心方法** | Push + Relabel |
| **时间复杂度** | 基本 $O(V^2 E)$，HLPPS $O(V^2 \sqrt{E})$ |
| **空间复杂度** | $O(V + E)$ |
| **关键优化** | 最高标号选取、GAP 优化 |
| **适用场景** | 大规模图、加强版模板 |
| **实现难度** | 较高 |

预流推进的核心思想：不再寻找完整的增广路径，而是"局部操作"。初始时从源点向所有邻居推送满流，产生大量溢出节点。然后反复选取溢出节点进行 push（将多余流量推向更低的邻居）和 relabel（提高自身高度以使 push 可行）。最终所有溢出流量要么到达汇点（贡献最大流），要么退回源点。

### 3.5 建模技巧

| 技巧 | 使用场景 | 建模方法 |
|------|---------|---------|
| **拆点** | 节点有容量限制 | $v \to v_{in}, v_{out}$，中间连容量边 |
| **分层图** | 涉及时间/阶段 | 按时间展开，层间连边 |
| **超级源汇** | 多源多汇 | 添加 $S, T$ 连接所有源/汇 |
| **二分图匹配** | 匹配问题 | $S \to$ 左部 $\to$ 右部 $\to T$，容量为 1 |
| **最大权闭合子图** | 项目选择/收益最大化 | 正权连 $S$，负权连 $T$，依赖连 $\infty$ |
| **最小割应用** | 分割/覆盖问题 | 利用最大流最小割定理转化 |

## 四、10 道经典例题详解

### 题目 1：【模板】网络最大流（洛谷 P3376） 🟢

> **题意简述**：给定一个 $n$ 个点 $m$ 条边的有向图，每条边有容量，求从源点 $s$ 到汇点 $t$ 的最大流。

**分析**：这是最大流的裸模板题，直接使用 Dinic 算法即可。数据范围较小，EK 也能通过。

```
算法：Dinic
建模：直接建图
时间复杂度：O(V²E)
```

**C++ 代码**：

```cpp
// 使用上述 MaxFlow 模板
MaxFlow mf(n);
for (int i = 0; i < m; i++) {
    int u, v; long long w;
    scanf("%d%d%lld", &u, &v, &w);
    mf.add_edge(u - 1, v - 1, w);
}
printf("%lld\n", mf.max_flow(s - 1, t - 1));
```

**核心启示**：
- **模板掌握**：熟练掌握 Dinic 模板是解决所有网络流问题的基础。
- **反向边**：建图时每条正向边必须同时建反向边（初始容量为 0），这是增广路方法的核心。
- **邻接表编号**：正向边和反向边在邻接表中成对存储，通过异或或记录 rev 索引快速定位反向边。

### 题目 2：【模板】最大流 加强版 / 预流推进（洛谷 P4722） 🟡

> **题意简述**：同 P3376，但数据范围大幅增强（$n \leq 1200, m \leq 120000$），Dinic 算法无法通过，需要预流推进算法。

**分析**：本题专门卡 Dinic 和 ISAP，需要使用 HLPPS（最高标号预流推进）算法。维护一个按高度组织的桶，每次选取最高标号的溢出节点进行操作。

```
算法：HLPPS（最高标号预流推进）
建模：直接建图
时间复杂度：O(V²√E)
关键：GAP优化 + 最高标号选取
```

**C++ 代码**：

```cpp
// HLPPS 核心片段
void push(int u, Edge& e) {
    long long d = min(excess[u], e.cap);
    e.cap -= d; graph[e.to][e.rev].cap += d;
    excess[u] -= d; excess[e.to] += d;
}
void relabel(int u) {
    int mn = 2 * n;
    for (auto& e : graph[u])
        if (e.cap > 0) mn = min(mn, height[e.to]);
    cnt[height[u]]--;
    height[u] = mn + 1;
    cnt[height[u]]++;
}
void discharge(int u) {
    for (auto& e : graph[u]) {
        if (excess[u] == 0) break;
        if (e.cap > 0 && height[u] == height[e.to] + 1)
            push(u, e);
    }
    if (excess[u] > 0) relabel(u);
}
```

**核心启示**：
- **算法选择**：当数据范围较大时，预流推进优于增广路类算法。
- **HLPPS 策略**：最高标号优先选取溢出节点能有效减少操作次数。
- **GAP 优化**：当某个高度值的节点数降为 0 时，更高的节点都无法到达汇点，可以直接标记退回源点。

### 题目 3：[TJOI2011] 卡片（洛谷 P2065） 🟡

> **题意简述**：有 $n$ 张蓝色卡片和 $m$ 张红色卡片，每张卡片上有一个数字。如果蓝色卡片的数字与红色卡片的数字不互质（即 $\gcd > 1$），则它们可以配对。每张卡片最多使用一次，求最大配对数。

**分析**：经典的二分图最大匹配问题，可转化为最大流。建立超级源点连向所有蓝色卡片（容量 1），所有红色卡片连向超级汇点（容量 1），满足条件的蓝色和红色卡片之间连容量 1 的边。

```
算法：Dinic（二分图匹配转最大流）
建模：S → 蓝色卡片 → 红色卡片 → T
时间复杂度：O(E√V)（单位容量网络）
判定条件：gcd(a[i], b[j]) > 1
```

**C++ 代码**：

```cpp
MaxFlow mf(n + m + 2);
int S = n + m, T = n + m + 1;
for (int i = 0; i < n; i++) mf.add_edge(S, i, 1);
for (int j = 0; j < m; j++) mf.add_edge(n + j, T, 1);
for (int i = 0; i < n; i++)
    for (int j = 0; j < m; j++)
        if (__gcd(a[i], b[j]) > 1)
            mf.add_edge(i, n + j, 1);
printf("%lld\n", mf.max_flow(S, T));
```

**核心启示**：
- **二分图匹配**：二分图最大匹配可以直接转化为最大流问题，且 Dinic 在单位容量网络上效率极高。
- **建模思路**：源点到左部、左部到右部、右部到汇点，容量均为 1。
- **数论判断**：gcd 不互质条件是建边的关键。

### 题目 4：试题库问题（洛谷 P2763） 🟡

> **题意简述**：有 $k$ 种类型的试题，每种类型需要选出 $r_i$ 道题。题库中有 $n$ 道题，每道题可能属于多种类型。问能否选出满足要求的试卷，若能则输出方案。

**分析**：多类型需求对应多个汇点需求，用最大流建模。源点连向每道题（容量 1，每题只能选一次），每道题连向它所属的类型（容量 1），每种类型连向汇点（容量 $r_i$）。若最大流等于 $\sum r_i$，则有解。

```
算法：Dinic
建模：S → 题目 → 类型 → T
时间复杂度：O(V²E)
方案输出：检查满流边
```

**C++ 代码**：

```cpp
MaxFlow mf(n + k + 2);
int S = n + k, T = n + k + 1;
for (int i = 0; i < n; i++) mf.add_edge(S, i, 1);
for (int j = 0; j < k; j++) mf.add_edge(n + j, T, r[j]);
for (int i = 0; i < n; i++)
    for (int tp : types[i])
        mf.add_edge(i, n + tp, 1);
long long res = mf.max_flow(S, T);
// 检查满流边输出方案
if (res == total) { /* 遍历边找cap==0的正向边 */ }
```

**核心启示**：
- **需求建模**：不同类型的需求量体现为对应节点到汇点的边容量。
- **方案还原**：最大流算法结束后，通过检查哪些边满流（残量为 0）来还原具体方案。
- **可行性判定**：最大流是否等于总需求量决定了问题是否有解。

### 题目 5：[SCOI2007] 蜥蜴（洛谷 P2472） 🟠

> **题意简述**：一个 $r \times c$ 的网格，部分格子上有石柱（有高度限制，每次经过高度减 1），部分格子上有蜥蜴。蜥蜴可以跳到距离不超过 $d$ 的石柱上。问最少有多少蜥蜴无法逃离网格。

**分析**：石柱高度限制 = 节点容量限制，需要**拆点**。将每个石柱拆为入点和出点，中间边容量为石柱高度。蜥蜴所在位置由超级源点连入（容量 1），能跳出网格的出点连向超级汇点（容量 $\infty$）。答案 = 蜥蜴总数 - 最大流。

```
算法：Dinic + 拆点
建模：S → 蜥蜴入点 → 蜥蜴出点 → ... → 边界出点 → T
拆点：每个石柱拆为(入点, 出点)，容量=高度
时间复杂度：O(V²E)
```

**C++ 代码**：

```cpp
// 节点编号：(i*c+j)*2 为入点，(i*c+j)*2+1 为出点
int in(int i, int j) { return (i * c + j) * 2; }
int out(int i, int j) { return (i * c + j) * 2 + 1; }
// 拆点边
if (h[i][j] > 0) mf.add_edge(in(i,j), out(i,j), h[i][j]);
// 蜥蜴 → 源点连接
if (has_lizard[i][j]) mf.add_edge(S, in(i,j), 1);
// 石柱间连边（距离 <= d）
if (dis(i1,j1,i2,j2) <= d)
    mf.add_edge(out(i1,j1), in(i2,j2), INF);
// 边界石柱 → 汇点
if (can_escape(i,j,d)) mf.add_edge(out(i,j), T, INF);
```

**核心启示**：
- **拆点技巧**：节点容量限制是拆点最典型的应用场景，将一个节点拆为入点和出点。
- **逆向思维**：答案 = 总数 - 最大流，求"最少逃不出"等价于求"最多逃出"。
- **距离建边**：两个石柱间距离不超过 $d$ 就建边，注意用欧几里得距离而非曼哈顿距离。

### 题目 6：[CTSC1999] 家园（洛谷 P2754） 🟠

> **题意简述**：有 $n$ 个太空站（编号 1~$n$），地球编号 0，月球编号 -1。有 $m$ 艘太空船，每艘有容量 $h_i$ 和周期性路线。求所有人从地球到月球的最短时间，若不可能输出 0。

**分析**：涉及时间维度，使用**分层图**建模。按时间 $t = 0, 1, 2, \ldots$ 展开，每一层代表一个时刻的所有太空站。每艘飞船在时刻 $t$ 从站 $a$ 到站 $b$，就从第 $t$ 层的 $a$ 连向第 $t+1$ 层的 $b$（容量为 $h_i$）。同一站点相邻时间层之间连无穷容量边（人可以等待）。逐层扩展，直到最大流达到总人数。

```
算法：Dinic + 分层图（按时间展开）
建模：按时刻建层，飞船航线为层间边
终止条件：最大流 = 总人数 或 超过时间上界
时间复杂度：O(T × V²E)，T为枚举时间
```

**C++ 代码**：

```cpp
// 枚举时间 t
for (int t = 1; t <= MAXTIME; t++) {
    // 每个站点 t-1 层 → t 层（等待边）
    for (int i = 0; i <= n; i++)
        mf.add_edge(id(i,t-1), id(i,t), INF);
    // 每艘飞船在时刻 t-1 → t 的航段
    for (int k = 0; k < m; k++) {
        int from = route[k][(t-1) % len[k]];
        int to = route[k][t % len[k]];
        mf.add_edge(id(from,t-1), id(to,t), cap[k]);
    }
    // 源点连地球第0层已处理，汇点连月球第t层
    mf.add_edge(id(MOON, t), T, INF);
    if (mf.max_flow(S, T) >= total) { printf("%d\n", t); return; }
}
```

**核心启示**：
- **分层图**：时间维度的问题通过按时间展开构建分层图来解决。
- **增量建图**：每扩展一个时间层，就在原图上添加新的节点和边，然后继续增广（利用之前的流）。
- **周期性路线**：飞船路线是周期性的，用取模来确定当前时刻的航段。

### 题目 7：魔术球问题（洛谷 P2765） 🟠

> **题意简述**：有 $n$ 根柱子，按 $1, 2, 3, \ldots$ 的顺序将球放入柱子。每次放球只能放在某根柱子的顶端。要求相邻两个球（柱子上下相邻）的编号之和为完全平方数。求最多能放多少个球。

**分析**：将每个球拆为入点和出点。如果球 $i$ 和球 $j$（$i < j$）的编号之和为完全平方数，则从 $i$ 的出点连向 $j$ 的入点（容量 1）。这构成了一个 DAG 上的路径覆盖问题，最小路径覆盖 = 节点数 - 最大匹配。枚举球数，当所需路径数超过 $n$ 时停止。

```
算法：Dinic（最小路径覆盖 = 总数 - 最大匹配）
建模：拆点 + 二分图匹配，枚举球数
判定条件：i+j 为完全平方数 (i<j)
终止：最小路径覆盖 > n
```

**C++ 代码**：

```cpp
// 枚举新增球 ball
for (int ball = 1; ; ball++) {
    mf.add_edge(S, ball * 2, 1);      // 源点→入点
    mf.add_edge(ball * 2 + 1, T, 1);  // 出点→汇点
    for (int pre = 1; pre < ball; pre++) {
        int sum = pre + ball;
        int sq = (int)sqrt(sum);
        if (sq * sq == sum)
            mf.add_edge(pre * 2 + 1, ball * 2, 1);
    }
    // 增量求最大流
    total_match += mf.max_flow(S, T);
    int paths = ball - total_match; // 最小路径覆盖
    if (paths > n) { printf("%d\n", ball - 1); break; }
}
```

**核心启示**：
- **最小路径覆盖**：DAG 上最小路径覆盖数 = 节点数 - 最大匹配数，这是经典结论。
- **增量建图**：随着球数增加逐步向图中添加节点和边，利用已有流增量求解。
- **完全平方数判断**：预处理或实时判断两数之和是否为完全平方数。

### 题目 8：最长不下降子序列问题（洛谷 P2766） 🔴

> **题意简述**：给定序列 $a_1, a_2, \ldots, a_n$。（1）求最长不下降子序列长度 $L$；（2）求最多能取出多少个长度为 $L$ 的不相交的不下降子序列；（3）若 $a_1$ 和 $a_n$ 可重复使用，再求一次。

**分析**：第一问用 DP 求 LIS 长度 $L$。第二问用网络流：先 DP 求出每个位置 $i$ 的 $f[i]$（以 $i$ 结尾的 LIS 长度）。拆点，每个位置拆为入点和出点（容量 1，保证每个元素只用一次）。若 $f[i] = 1$，源点连入点；若 $f[i] = L$，出点连汇点。若 $i < j$ 且 $a[i] \leq a[j]$ 且 $f[j] = f[i] + 1$，则出点 $i$ 连入点 $j$。最大流即为答案。第三问将 $a_1$ 和 $a_n$ 的容量改为无穷。

```
算法：DP + Dinic + 拆点
建模：f[i]=1 从S连入，f[i]=L 出点连T
转移边：f[j]=f[i]+1 且 a[i]<=a[j] 时 out(i)→in(j)
第三问：节点1和n的拆点边容量改为INF
```

**C++ 代码**：

```cpp
// 第一问：DP 求 f[i]
for (int i = 1; i <= n; i++) {
    f[i] = 1;
    for (int j = 1; j < i; j++)
        if (a[j] <= a[i]) f[i] = max(f[i], f[j] + 1);
    L = max(L, f[i]);
}
// 第二问：建最大流图
for (int i = 1; i <= n; i++) {
    mf.add_edge(in(i), out(i), 1); // 拆点
    if (f[i] == 1) mf.add_edge(S, in(i), 1);
    if (f[i] == L) mf.add_edge(out(i), T, 1);
    for (int j = i + 1; j <= n; j++)
        if (a[i] <= a[j] && f[j] == f[i] + 1)
            mf.add_edge(out(i), in(j), 1);
}
```

**核心启示**：
- **DP + 网络流**：先用 DP 确定结构，再用网络流求最优解，这是经典的组合技巧。
- **拆点保证唯一性**：每个元素只能使用一次，通过拆点容量为 1 来限制。
- **分层建边**：只在 DP 值恰好差 1 的位置之间建边，保证了子序列的合法性。
- **灵活调整**：第三问只需修改首尾节点的容量，体现了网络流模型的灵活性。

### 题目 9：[NOI2009] 植物大战僵尸（洛谷 P2805） 🔴

> **题意简述**：$n \times m$ 的网格上有植物，每个植物有收益值（可正可负）。僵尸从右向左进攻，要吃掉一个植物必须先吃掉它右边同行的所有植物以及保护它的植物。求僵尸的最大收益。

**分析**：这是一个**最大权闭合子图**问题。"吃 A 之前必须吃 B"构成依赖关系。首先需要用拓扑排序去掉无法到达的植物（存在环形保护则不可攻击）。然后建最大权闭合子图模型：正权节点从源点连容量为权值的边，负权节点向汇点连容量为权值绝对值的边，依赖关系用无穷容量边表示。答案 = 正权值之和 - 最小割。

```
算法：拓扑排序 + Dinic（最大权闭合子图）
建模：正权连S，负权连T，依赖连INF
预处理：拓扑排序去除环中不可达节点
答案：sum(正权) - 最大流
```

**C++ 代码**：

```cpp
// 拓扑排序去除不可达节点
queue<int> q;
for (int i = 0; i < n * m; i++)
    if (indeg[i] == 0) q.push(i);
while (!q.empty()) {
    int u = q.front(); q.pop(); valid[u] = true;
    for (int v : dep[u]) if (--indeg[v] == 0) q.push(v);
}
// 建最大权闭合子图
for (int i = 0; i < n * m; i++) {
    if (!valid[i]) continue;
    if (w[i] > 0) { mf.add_edge(S, i, w[i]); sum += w[i]; }
    else mf.add_edge(i, T, -w[i]);
    for (int v : dep[i])
        if (valid[v]) mf.add_edge(i, v, INF);
}
printf("%lld\n", sum - mf.max_flow(S, T));
```

**核心启示**：
- **最大权闭合子图**：当问题涉及"选取节点获得收益但有依赖关系"时，首先考虑此模型。
- **拓扑排序预处理**：存在循环依赖的节点是不可达的，必须先排除。
- **正权连源、负权连汇**：这是最大权闭合子图建模的固定模式。
- **答案公式**：$\text{最大收益} = \sum \text{正权} - \text{最小割}$。

### 题目 10：[六省联考 2017] 寿司餐厅（洛谷 P3749） 🟣

> **题意简述**：有 $n$ 种寿司排成一排，选择若干个连续区间 $[i, j]$，获得收益 $d_{i,j}$。若选了 $[i, j]$，则其所有子区间也必须选。代价方面，每种寿司只收一次费，编号为 $a$ 的寿司费用为 $a$（代号 $> 0$）时还需额外交 $m \cdot a^2$。求最大净收益。

**分析**：区间的包含关系构成依赖：选 $[i, j]$ 必须选 $[i, j-1]$ 和 $[i+1, j]$。这又是一个**最大权闭合子图**问题。将每个区间 $[i, j]$ 视为节点，收益为 $d_{i,j}$。对于每个寿司代号 $a$，创建一个代号节点，费用为 $a + m \cdot a^2$。区间 $[i, i]$ 向寿司代号 $a_i$ 连依赖边。建模后求最小割。

```
算法：Dinic（最大权闭合子图）
建模：区间节点 + 代号节点，依赖关系建边
区间依赖：[i,j] → [i,j-1], [i,j] → [i+1,j]
代号费用：a + m*a²
答案：sum(正权) - 最大流
```

**C++ 代码**：

```cpp
// 区间节点编号 id[i][j], 代号节点另编号
for (int i = 1; i <= n; i++)
    for (int j = i; j <= n; j++) {
        int u = id[i][j];
        if (d[i][j] > 0) { mf.add_edge(S, u, d[i][j]); sum += d[i][j]; }
        else mf.add_edge(u, T, -d[i][j]);
        if (j > i) {
            mf.add_edge(u, id[i][j-1], INF);
            mf.add_edge(u, id[i+1][j], INF);
        } else {
            mf.add_edge(u, code_id[a[i]], INF); // 连代号节点
        }
    }
// 代号节点连T，费用 = a + m*a*a
for (int x : all_codes)
    mf.add_edge(code_id[x], T, x + (long long)m * x * x);
printf("%lld\n", sum - mf.max_flow(S, T));
```

**核心启示**：
- **区间依赖**：选大区间必选子区间的依赖关系是此题建模的关键。
- **代号节点去重**：同一代号的寿司费用只算一次，通过建立代号节点来处理。
- **闭合子图模板**：正权连源、负权连汇、依赖连 $\infty$ 的标准模式再次适用。
- **综合建模能力**：需要同时处理区间依赖和代号费用两种约束。

## 五、常见技巧与思路总结

### 技巧 1：识别网络流模型

| 问题特征 | 对应模型 |
|---------|---------|
| 匹配、分配、选择 | 二分图匹配 → 最大流 |
| 节点/边有容量限制 | 直接建流网络 |
| 选取有依赖关系的项获最大收益 | 最大权闭合子图 |
| 最小代价划分为两组 | 最小割 |
| DAG 上路径覆盖 | 最小路径覆盖 = n - 最大匹配 |

### 技巧 2：拆点的通用范式

| 场景 | 拆法 | 中间边容量 |
|------|------|----------|
| 节点容量限制 | $v \to (v_{in}, v_{out})$ | 节点容量 |
| 节点只能用一次 | $v \to (v_{in}, v_{out})$ | 1 |
| 节点可无限使用 | 不拆或中间边 $\infty$ | $\infty$ |

### 技巧 3：增量建图法

在某些枚举类问题中（如魔术球、家园），无需每次从头建图重跑最大流。利用 Dinic 等算法的性质，在已有流的基础上添加新节点和边，然后继续增广即可。这大幅降低了时间复杂度。

### 技巧 4：最大流最小割定理的应用

$$\text{最大流} = \text{最小割}$$

这一定理使得我们可以从"割"的角度理解问题：选择代价最小的边集将源点和汇点分开。在最大权闭合子图中，最小割恰好对应不选某些节点的代价。

### 技巧 5：方案还原

最大流结束后，残量网络中从源点 BFS/DFS 可以到达的节点属于源点侧（$S$-集），不可到达的属于汇点侧（$T$-集）。对于二分图匹配，检查哪些匹配边满流即可还原方案。

### 技巧 6：判断无解与上界

| 场景 | 判断方法 |
|------|---------|
| 最大流 < 需求量 | 无法满足所有需求 |
| 最大流 = 需求量 | 恰好满足 |
| 枚举时间超过上界 | 不可能完成（如家园问题） |
| 最小割 = 0 | 源汇已天然断开 |

### 6.1 算法选择决策树

```
问题规模？
├── V ≤ 500, E ≤ 5000 → EK 算法即可
```

### 6.2 建模五步法

1. **识别源汇**：谁是起点？谁是终点？是否需要超级源汇？
2. **确定节点**：问题中的实体分别对应哪些节点？是否需要拆点？
3. **确定边与容量**：实体间的关系如何用有向边和容量表达？
4. **确定目标**：求最大流？最小割？方案？
5. **验证模型**：手动模拟小样例检验建模正确性。

### 6.3 Dinic 常见优化

| 优化 | 效果 | 实现 |
|------|------|------|
| **当前弧优化** | 避免重复遍历死边 | `iter[v]` 记录当前搜索位置 |
| **多路增广** | 一次 DFS 增广多条路 | DFS 返回后继续搜索 |
| **剪枝：流量用尽** | 提前返回 | `if (f == 0) return 0` |
| **剪枝：无法到达汇点** | 标记层次为 -1 | DFS 返回 0 时设 `level[v] = -1` |

- [ ] 是否正确建立了反向边？
- [ ] 拆点时入点和出点是否连接正确？
- [ ] 超级源汇的容量是否设置合理？
- [ ] 无穷大 INF 的值是否足够大且不会溢出？
- [ ] 节点编号是否冲突？总节点数是否开够？
- [ ] 是否需要预处理（如拓扑排序去环）？

## 七、终极总结与归纳

### 7.1 最大流的本质

最大流的本质是在**容量约束和流量守恒**下，寻找从源到汇的最大输送量。通过最大流最小割定理，它同时给出了网络的"瓶颈"在哪里——即最小割所对应的关键边集。最大流不仅是一个算法问题，更是一种强大的**建模工具**：通过巧妙地设计节点、边和容量，大量看似无关的组合优化问题都可以转化为最大流来求解。

### 7.2 核心要点

1. **增广路方法**是最大流的理论基础，反向边机制允许算法自我纠错。
2. **Dinic 算法**是竞赛中的主力算法，掌握 BFS 分层 + DFS 阻塞流 + 当前弧优化三大要素。
3. **建模能力**比算法本身更重要——拆点、分层图、超级源汇、最大权闭合子图是四大核心建模技巧。
4. **最大流 = 最小割**是连接"流"与"割"的桥梁，很多问题从割的角度更容易理解。
5. **增量建图**技巧在枚举类问题中至关重要，可以避免重复计算。
