## 一、什么是倍增法求 LCA

**最近公共祖先（Lowest Common Ancestor, LCA）**：在一棵有根树中，节点 $u$ 和 $v$ 的 LCA 定义为它们所有公共祖先中**深度最大**的那个节点。

**倍增法（Binary Lifting）** 是求 LCA 最经典、最通用的算法。核心思想是：预处理每个节点的 $2^0, 2^1, 2^2, \dots$ 级祖先，查询时通过"倍增跳跃"在 $O(\log n)$ 时间内找到 LCA。

```
示例树：
        1
       / \
      2   3
     / \   \
    4   5   6
   /
  7

LCA(7, 5) = 2    （7 → 4 → 2，5 → 2）
LCA(7, 6) = 1    （7 → 4 → 2 → 1，6 → 3 → 1）
LCA(4, 5) = 2    （4 → 2，5 → 2）
```

### 与其他 LCA 算法的对比

| 算法 | 预处理 | 单次查询 | 空间 | 特点 |
|------|--------|---------|------|------|
| 倍增法 | $O(n \log n)$ | $O(\log n)$ | $O(n \log n)$ | 最通用，支持路径查询 |
| 树链剖分 | $O(n)$ | $O(\log n)$ | $O(n)$ | 可结合线段树做路径修改 |
| Tarjan 离线 | $O(n + q)$ | 离线 $O(1)$ | $O(n)$ | 仅支持离线查询 |
| Euler Tour + ST 表 | $O(n \log n)$ | $O(1)$ | $O(n \log n)$ | 查询 $O(1)$，但不易扩展路径信息 |

倍增法的优势在于：**代码简洁、易于扩展**，可以在跳跃过程中同时维护路径上的最大值、最小值、路径和等信息。

## 二、核心思想与前置知识

### 2.1 倍增思想

倍增的核心：任何正整数都可以表示为若干个 $2$ 的幂次之和（二进制分解）。因此，从节点 $u$ 向上跳 $k$ 步，可以分解为若干次"跳 $2^i$ 步"的操作。

```
跳 13 步 = 跳 8 步 + 跳 4 步 + 跳 1 步
13 = 1101₂ = 2³ + 2² + 2⁰
```

### 2.2 倍增数组的定义与预处理

定义 `fa[u][k]` 表示节点 $u$ 的第 $2^k$ 级祖先。

```
递推关系：
fa[u][0] = parent(u)              // 父节点
fa[u][k] = fa[fa[u][k-1]][k-1]    // 先跳 2^(k-1)，再跳 2^(k-1)

预处理复杂度：O(n log n)
```

```
可视化：u 的第 2^k 级祖先

u ---(2^(k-1) 步)---> fa[u][k-1] ---(2^(k-1) 步)---> fa[u][k]
|<-----------------  2^k 步  ----------------------->|
```

### 2.3 LCA 查询流程

```
LCA(u, v) 查询步骤：
1. 调整深度：让 u, v 处于同一深度（深的那个向上跳）
2. 特判：如果此时 u == v，则 LCA = u
3. 同步跳跃：从高位到低位枚举 k，若 fa[u][k] != fa[v][k] 则同时跳
4. 返回结果：LCA = fa[u][0]
```

### 2.4 倍增 LCA 模板代码

```cpp
const int MAXN = 500005;
const int LOG = 20;  // log2(MAXN)

int fa[MAXN][LOG], dep[MAXN];
vector<int> adj[MAXN];

void dfs(int u, int p, int d) {
    fa[u][0] = p;
    dep[u] = d;
    for (int k = 1; k < LOG; k++)
        fa[u][k] = fa[fa[u][k-1]][k-1];
    for (int v : adj[u])
        if (v != p) dfs(v, u, d + 1);
}

int lca(int u, int v) {
    if (dep[u] < dep[v]) swap(u, v);
    int diff = dep[u] - dep[v];
    for (int k = 0; k < LOG; k++)
        if ((diff >> k) & 1) u = fa[u][k];
    if (u == v) return u;
    for (int k = LOG - 1; k >= 0; k--)
        if (fa[u][k] != fa[v][k]) {
            u = fa[u][k];
            v = fa[v][k];
        }
    return fa[u][0];
}

// 调用：dfs(root, 0, 0); 之后即可查询 lca(u, v)
```

### 2.5 树上两点距离

LCA 的经典应用之一：$\text{dist}(u, v) = \text{dep}[u] + \text{dep}[v] - 2 \times \text{dep}[\text{LCA}(u,v)]$

若边带权，将 `dep[]` 替换为到根的距离 `dis[]` 即可。

## 三、分类与变体详解

### 3.1 基础 LCA 查询

最基本的形式：给定树和若干查询 $(u, v)$，返回 LCA。

```
数据结构：fa[u][k] — 倍增祖先数组
支持操作：LCA 查询
关键性质：预处理 O(n log n)，查询 O(log n)
时间复杂度：O((n + q) log n)
```

### 3.2 路径信息维护（倍增 + 路径聚合）

在倍增跳跃过程中，同时维护路径上的信息（最大值、最小值、边权和、GCD 等）。

```
扩展数组：
  fa[u][k]   — u 的第 2^k 级祖先
  maxw[u][k] — u 到 fa[u][k] 路径上的边权最大值
  minw[u][k] — u 到 fa[u][k] 路径上的边权最小值
  sumw[u][k] — u 到 fa[u][k] 路径上的边权之和

递推：
  maxw[u][k] = max(maxw[u][k-1], maxw[fa[u][k-1]][k-1])
  minw[u][k] = min(minw[u][k-1], minw[fa[u][k-1]][k-1])
  sumw[u][k] = sumw[u][k-1] + sumw[fa[u][k-1]][k-1]
```

```cpp
int maxw[MAXN][LOG];

void dfs(int u, int p, int d) {
    fa[u][0] = p;
    dep[u] = d;
    for (int k = 1; k < LOG; k++) {
        fa[u][k] = fa[fa[u][k-1]][k-1];
        maxw[u][k] = max(maxw[u][k-1], maxw[fa[u][k-1]][k-1]);
    }
    for (auto [v, w] : adj[u]) {
        if (v != p) {
            maxw[v][0] = w;
            dfs(v, u, d + 1);
        }
    }
}

int query_max(int u, int v) {
    int res = 0;
    if (dep[u] < dep[v]) swap(u, v);
    int diff = dep[u] - dep[v];
    for (int k = 0; k < LOG; k++)
        if ((diff >> k) & 1) {
            res = max(res, maxw[u][k]);
            u = fa[u][k];
        }
    if (u == v) return res;
    for (int k = LOG - 1; k >= 0; k--)
        if (fa[u][k] != fa[v][k]) {
            res = max(res, max(maxw[u][k], maxw[v][k]));
            u = fa[u][k]; v = fa[v][k];
        }
    res = max(res, max(maxw[u][0], maxw[v][0]));
    return res;
}
```

### 3.3 Kruskal 重构树 + LCA

将 Kruskal 算法过程中每次合并边构建成一棵新树，使得两点在重构树上的 LCA 权值 = 原图中连通两点所需的最小瓶颈边权。

```
数据结构：Kruskal 重构树（二叉树）+ 倍增 LCA
支持操作：两点间最小瓶颈路查询
关键性质：LCA 的权值就是瓶颈边权
时间复杂度：O(m log m + n log n) 预处理，O(log n) 查询
```

### 3.4 倍增求第 k 级祖先

倍增数组的直接应用：求节点 $u$ 的第 $k$ 级祖先。

```cpp
int kth_ancestor(int u, int k) {
    for (int i = 0; i < LOG; i++)
        if ((k >> i) & 1) u = fa[u][i];
    return u;  // 若 u == 0 表示不存在
}
```

### 3.5 变体对比

| 变体 | 额外维护 | 典型应用 |
|------|---------|---------|
| 基础 LCA | 无 | 纯 LCA 查询、树上距离 |
| 路径最值 | `maxw[u][k]` / `minw[u][k]` | 瓶颈路径、路径最值查询 |
| 路径聚合 | `sumw[u][k]` / `gcdw[u][k]` | 路径和、路径 GCD |
| Kruskal 重构树 | 重构树节点权值 | 最小瓶颈路、离线连通性 |
| 第 k 级祖先 | 无额外 | 树上跳跃、定位祖先 |

### 题目 1：【模板】最近公共祖先（洛谷 P3379）

> 给定一棵 $n$ 个节点的有根树（根为 $s$），$m$ 次查询两个节点的最近公共祖先。$n, m \le 5 \times 10^5$。

**分析**：LCA 倍增法裸模板。DFS 预处理深度和倍增数组，每次查询 $O(\log n)$。

```
数据结构：fa[u][k] — 倍增祖先数组，dep[u] — 深度
支持操作：LCA(u, v) 查询
关键性质：fa[u][k] = fa[fa[u][k-1]][k-1]
时间复杂度：O((n + m) log n)
```

**C++ 代码**：

```cpp
const int MAXN = 500005, LOG = 20;
int fa[MAXN][LOG], dep[MAXN];
vector<int> adj[MAXN];

void dfs(int u, int p, int d) {
    fa[u][0] = p; dep[u] = d;
    for (int k = 1; k < LOG; k++)
        fa[u][k] = fa[fa[u][k-1]][k-1];
    for (int v : adj[u])
        if (v != p) dfs(v, u, d + 1);
}

int lca(int u, int v) {
    if (dep[u] < dep[v]) swap(u, v);
    for (int k = LOG - 1; k >= 0; k--)
        if (dep[u] - (1 << k) >= dep[v]) u = fa[u][k];
    if (u == v) return u;
    for (int k = LOG - 1; k >= 0; k--)
        if (fa[u][k] != fa[v][k]) { u = fa[u][k]; v = fa[v][k]; }
    return fa[u][0];
}
// 调用：dfs(s, 0, 0); 然后每次查询 lca(u, v)
```

**核心启示**：
- **倍增递推** `fa[u][k] = fa[fa[u][k-1]][k-1]` 是整个算法的基石
- **先调平深度再同步跳** 是 LCA 查询的标准两步流程
- DFS 栈深度可能爆栈时需改用 BFS 预处理

### 题目 2：树上两点距离（洛谷 P3379 变体 / CSES Distance Queries）

> 给定一棵 $n$ 个节点的无权树，$q$ 次查询两点之间的距离。

**分析**：$\text{dist}(u,v) = \text{dep}[u] + \text{dep}[v] - 2 \times \text{dep}[\text{LCA}(u,v)]$。

```
数据结构：fa[u][k]、dep[u]
支持操作：LCA + 深度计算距离
关键性质：树上路径唯一，距离可通过 LCA 拆解
时间复杂度：O((n + q) log n)
```

**C++ 代码**：

```cpp
// 复用题目 1 的 dfs 和 lca
int dist(int u, int v) {
    return dep[u] + dep[v] - 2 * dep[lca(u, v)];
}
```

**核心启示**：
- **LCA 拆分路径** 是树上距离计算的核心公式，几乎所有树上路径问题都以此为基础
- 带权树只需将 `dep[]` 替换为到根距离 `dis[]`

### 题目 3：第 K 级祖先（LeetCode 1483 / CSES Company Queries I）

> 给定一棵有根树，多次查询节点 $u$ 的第 $k$ 级祖先。若不存在返回 $-1$。

**分析**：直接利用倍增数组，将 $k$ 二进制分解逐位跳跃。

```
数据结构：fa[u][k] — 倍增祖先数组
支持操作：kth_ancestor(u, k) 查询
关键性质：k 的二进制分解 → 多次 2^i 跳跃的组合
时间复杂度：O(n log n) 预处理，O(log n) 查询
```

**C++ 代码**：

```cpp
int kth_ancestor(int u, int k) {
    for (int i = 0; i < LOG && u; i++)
        if ((k >> i) & 1) u = fa[u][i];
    return u ? u : -1;  // 0 表示不存在
}
```

**核心启示**：
- **倍增数组本身就能求第 k 级祖先**，不需要额外结构
- 这是倍增思想最纯粹的体现：将一个大步分解为若干个 $2^i$ 小步

### 题目 4：货车运输（洛谷 P1967 / NOIP 2013 提高组）

> 给定 $n$ 个城市 $m$ 条道路（边权为载重限制），$q$ 次查询从 $u$ 到 $v$ 的路径上，载重限制的最大值（取路径上边权的最小值，所有路径中取最大的那条路径）。$n \le 10^4, m \le 5 \times 10^4$。

**分析**：最大瓶颈路问题。先求最大生成树（瓶颈路一定在最大生成树上），然后在树上用倍增维护路径最小值。

```
数据结构：最大生成树 + fa[u][k] + minw[u][k]（路径最小边权）
支持操作：路径最小值查询
关键性质：最大瓶颈路一定在最大生成树上
时间复杂度：O(m log m + n log n + q log n)
```

**C++ 代码**：

```cpp
const int MAXN = 10005, LOG = 14;
int fa[MAXN][LOG], dep[MAXN], minw[MAXN][LOG];
vector<pair<int,int>> adj[MAXN];  // {to, weight}
int par[MAXN];  // 并查集

int find(int x) { return par[x] == x ? x : par[x] = find(par[x]); }

// Kruskal 求最大生成树后建树（省略 Kruskal 过程）

void dfs(int u, int p, int d) {
    fa[u][0] = p; dep[u] = d;
    for (int k = 1; k < LOG; k++) {
        fa[u][k] = fa[fa[u][k-1]][k-1];
        minw[u][k] = min(minw[u][k-1], minw[fa[u][k-1]][k-1]);
    }
    for (auto [v, w] : adj[u]) if (v != p) {
        minw[v][0] = w;
        dfs(v, u, d + 1);
    }
}

int query(int u, int v) {
    if (find(u) != find(v)) return -1;  // 不连通
    int res = INT_MAX;
    if (dep[u] < dep[v]) swap(u, v);
    int diff = dep[u] - dep[v];
    for (int k = 0; k < LOG; k++)
        if ((diff >> k) & 1) { res = min(res, minw[u][k]); u = fa[u][k]; }
    if (u == v) return res;
    for (int k = LOG - 1; k >= 0; k--)
        if (fa[u][k] != fa[v][k]) {
            res = min(res, min(minw[u][k], minw[v][k]));
            u = fa[u][k]; v = fa[v][k];
        }
    res = min(res, min(minw[u][0], minw[v][0]));
    return res;
}
```

**核心启示**：
- **最大/最小生成树 + 倍增 LCA** 是解决瓶颈路径问题的经典套路
- 倍增数组可以**同时维护多种路径信息**，递推公式与 LCA 一致
- 注意不连通的情况需要用并查集判断

### 题目 5：次小生成树（洛谷 P4180 / 严格次小生成树）

> 给定一个带权无向连通图，求**严格次小生成树**的边权和。$n \le 10^5, m \le 3 \times 10^5$。

**分析**：先求 MST，然后枚举每条非树边 $(u, v, w)$，用它替换 $u \to v$ 路径上的最大边（若等于 $w$ 则替换次大边）。用倍增维护路径最大值和严格次大值。

```
数据结构：MST + fa[u][k] + max1[u][k] + max2[u][k]
支持操作：路径上最大值和严格次大值查询
关键性质：替换一条非树边 = 删树上最大边 + 加非树边
时间复杂度：O(m log m + n log n)
```

**C++ 代码**：

```cpp
const int MAXN = 100005, LOG = 17;
const long long INF = 1e18;
int fa[MAXN][LOG], dep[MAXN];
long long max1[MAXN][LOG], max2[MAXN][LOG];  // 最大、严格次大

void dfs(int u, int p, int d) {
    fa[u][0] = p; dep[u] = d;
    for (int k = 1; k < LOG; k++) {
        fa[u][k] = fa[fa[u][k-1]][k-1];
        // 合并两段的最大和次大
        long long vals[4] = {max1[u][k-1], max2[u][k-1],
                             max1[fa[u][k-1]][k-1], max2[fa[u][k-1]][k-1]};
        max1[u][k] = -INF; max2[u][k] = -INF;
        for (long long x : vals) {
            if (x > max1[u][k]) { max2[u][k] = max1[u][k]; max1[u][k] = x; }
            else if (x != max1[u][k] && x > max2[u][k]) max2[u][k] = x;
        }
    }
    for (auto [v, w] : adj[u]) if (v != p) {
        max1[v][0] = w; max2[v][0] = -INF;
        dfs(v, u, d + 1);
    }
}

// 查询 u→v 路径上最大值和严格次大值
pair<long long, long long> path_query(int u, int v) {
    long long m1 = -INF, m2 = -INF;
    auto upd = [&](long long x) {
        if (x > m1) { m2 = m1; m1 = x; }
        else if (x != m1 && x > m2) m2 = x;
    };
    if (dep[u] < dep[v]) swap(u, v);
    int diff = dep[u] - dep[v];
    for (int k = 0; k < LOG; k++) if ((diff >> k) & 1) {
        upd(max1[u][k]); upd(max2[u][k]); u = fa[u][k];
    }
    if (u == v) return {m1, m2};
    for (int k = LOG - 1; k >= 0; k--) if (fa[u][k] != fa[v][k]) {
        upd(max1[u][k]); upd(max2[u][k]);
        upd(max1[v][k]); upd(max2[v][k]);
        u = fa[u][k]; v = fa[v][k];
    }
    upd(max1[u][0]); upd(max2[u][0]);
    upd(max1[v][0]); upd(max2[v][0]);
    return {m1, m2};
}

// 枚举非树边求严格次小生成树：
// ans = mst_weight + min(w - (w == m1 ? m2 : m1)) 对所有非树边
```

**核心启示**：
- **严格次小需要维护"最大值 + 严格次大值"两个信息**，这是比单最值更复杂的路径聚合
- 合并两段区间的最大/次大需要用 4 元素排序，避免重复计入相同值
- 这是倍增维护路径信息的极致应用之一

### 题目 6：暗的连锁（洛谷 P3950 / NOIP 2017）

> 树上有 $n$ 个节点和 $n-1$ 条树边以及 $m$ 条非树边。问砍掉某条树边后、再砍掉某条非树边，使图不连通的方案数。

**分析**：每条非树边 $(u,v)$ 会覆盖 $u \to v$ 路径上的所有树边。用树上差分统计每条树边被覆盖的次数 $cnt$。砍一条树边时：
- $cnt = 0$：任意一条非树边都行，贡献 $m$
- $cnt = 1$：只有覆盖它的那条非树边，贡献 $1$
- $cnt \ge 2$：无法通过砍一条非树边断开，贡献 $0$

```
数据结构：倍增 LCA + 树上差分
支持操作：求每条树边被非树边覆盖的次数
关键性质：非树边 (u,v) 覆盖 u→LCA(u,v)→v 路径上所有树边
时间复杂度：O((n + m) log n)
```

**C++ 代码**：

```cpp
const int MAXN = 100005, LOG = 17;
int fa[MAXN][LOG], dep[MAXN], diff[MAXN];
vector<int> adj[MAXN];
long long ans;
int n, m;

void dfs(int u, int p, int d) {
    fa[u][0] = p; dep[u] = d;
    for (int k = 1; k < LOG; k++)
        fa[u][k] = fa[fa[u][k-1]][k-1];
    for (int v : adj[u]) if (v != p) dfs(v, u, d + 1);
}

int lca(int u, int v) {
    if (dep[u] < dep[v]) swap(u, v);
    for (int k = LOG - 1; k >= 0; k--)
        if (dep[u] - (1 << k) >= dep[v]) u = fa[u][k];
    if (u == v) return u;
    for (int k = LOG - 1; k >= 0; k--)
        if (fa[u][k] != fa[v][k]) { u = fa[u][k]; v = fa[v][k]; }
    return fa[u][0];
}

// 树上差分：对非树边 (u, v)，diff[u]++, diff[v]++, diff[lca(u,v)] -= 2
// DFS 求子树和得到每条边的覆盖次数
int dfs_sum(int u, int p) {
    int cnt = diff[u];
    for (int v : adj[u]) if (v != p) {
        int sub = dfs_sum(v, u);
        // sub 是边 (u, v) 的覆盖次数
        if (sub == 0) ans += m;
        else if (sub == 1) ans += 1;
        cnt += sub;
    }
    return cnt;
}
```

**核心启示**：
- **LCA + 树上差分** 是处理"非树边覆盖树边"问题的标准方法
- 差分操作 `diff[u]++, diff[v]++, diff[lca]-=2` 是核心套路，一次 DFS 求子树和即可还原每条边的覆盖次数

### 题目 7：聚会 / 三点 LCA（洛谷 P4281 / JLOI 2015）

> 给定一棵 $n$ 个节点的树，$m$ 次查询：给出三个节点 $a, b, c$，找出一个点 $p$ 使得 $\text{dist}(p,a) + \text{dist}(p,b) + \text{dist}(p,c)$ 最小，输出 $p$ 和最小总距离。

**分析**：三个点的最优聚会点是三个两两 LCA 中深度最大的那个。总距离 = 三条两两距离之和的一半。

```
数据结构：倍增 LCA + dep[]
支持操作：三点聚会的最优点和距离
关键性质：最优点 = 三个 LCA 中深度最大者
时间复杂度：O((n + m) log n)
```

**C++ 代码**：

```cpp
// 复用 dfs 和 lca 函数

void solve(int a, int b, int c) {
    int ab = lca(a, b), bc = lca(b, c), ac = lca(a, c);
    // 最优点是三个 LCA 中深度最大的
    int p = ab;
    if (dep[bc] > dep[p]) p = bc;
    if (dep[ac] > dep[p]) p = ac;
    int total = dep[a] + dep[b] + dep[c]
              - dep[ab] - dep[bc] - dep[ac];
    // 输出 p 和 total
}
```

**核心启示**：
- **三点 LCA 性质**：$\text{LCA}(a,b), \text{LCA}(b,c), \text{LCA}(a,c)$ 中必有两个相同，第三个深度更大
- 最优聚会点 = 深度最大的那个 LCA，总距离有简洁公式
- 这是 LCA 性质的经典推论，在虚树等高级结构中也很常用

### 题目 8：树上路径边权最值查询（CSES Path Queries II 变体）

> 给定一棵 $n$ 个节点的带权树，$q$ 次查询路径 $u \to v$ 上的边权最大值。$n, q \le 2 \times 10^5$。

**分析**：倍增数组同时维护路径最大值 `maxw[u][k]`。查询时在调平和同步跳跃过程中取 max。

```
数据结构：fa[u][k] + maxw[u][k]
支持操作：路径最大边权查询
关键性质：maxw[u][k] = max(maxw[u][k-1], maxw[fa[u][k-1]][k-1])
时间复杂度：O((n + q) log n)
```

**C++ 代码**：

```cpp
const int MAXN = 200005, LOG = 18;
int fa[MAXN][LOG], dep[MAXN], maxw[MAXN][LOG];
vector<pair<int,int>> adj[MAXN];

void dfs(int u, int p, int d) {
    fa[u][0] = p; dep[u] = d;
    for (int k = 1; k < LOG; k++) {
        fa[u][k] = fa[fa[u][k-1]][k-1];
        maxw[u][k] = max(maxw[u][k-1], maxw[fa[u][k-1]][k-1]);
    }
    for (auto [v, w] : adj[u]) if (v != p) {
        maxw[v][0] = w;
        dfs(v, u, d + 1);
    }
}

int query(int u, int v) {
    int res = 0;
    if (dep[u] < dep[v]) swap(u, v);
    int diff = dep[u] - dep[v];
    for (int k = 0; k < LOG; k++) if ((diff >> k) & 1) {
        res = max(res, maxw[u][k]); u = fa[u][k];
    }
    if (u == v) return res;
    for (int k = LOG - 1; k >= 0; k--) if (fa[u][k] != fa[v][k]) {
        res = max(res, max(maxw[u][k], maxw[v][k]));
        u = fa[u][k]; v = fa[v][k];
    }
    return max(res, max(maxw[u][0], maxw[v][0]));
}
```

**核心启示**：
- **倍增维护路径信息**是 LCA 最强大的扩展，只需在递推和查询时"顺便"合并信息
- 可推广到任何满足**结合律**的运算（max、min、gcd、xor 等）

### 题目 9：异象石 / 动态虚树（洛谷 P4103 / HEOI 2014）

> 维护一个节点集合 $S$（支持插入和删除），每次操作后查询连接 $S$ 中所有节点的最短路径长度（即虚树的边权和）。

**分析**：用 set 按 DFS 序维护节点集合，每次插入/删除时动态计算贡献变化。关键观察：将 $S$ 中节点按 DFS 序排列为 $v_1, v_2, \dots, v_k$，虚树边权和 = $\frac{1}{2} \sum_{i=1}^{k} \text{dist}(v_i, v_{i+1})$（其中 $v_{k+1} = v_1$，环形求和）。

```
数据结构：倍增 LCA + DFS 序 + set
支持操作：动态插入/删除节点，查询虚树边权和
关键性质：虚树边权和 = DFS 序相邻节点距离之和 / 2
时间复杂度：O(n log n) 预处理，O(log^2 n) 每次操作
```

**C++ 代码**：

```cpp
const int MAXN = 100005, LOG = 17;
int fa[MAXN][LOG], dep[MAXN], dfn[MAXN], timer;
set<pair<int,int>> s;  // {dfn, node}
long long total;

int dist(int u, int v) {
    return dep[u] + dep[v] - 2 * dep[lca(u, v)];
}

void insert(int u) {
    auto it = s.insert({dfn[u], u}).first;
    auto nxt = next(it), prv = (it == s.begin()) ? prev(s.end()) : prev(it);
    if (s.size() == 1) { total = 0; return; }
    if (s.size() == 2) { total = 2 * dist(it->second, nxt == s.end() ? s.begin()->second : nxt->second); return; }
    int p = prv->second, n = (nxt == s.end()) ? s.begin()->second : nxt->second;
    total += dist(p, u) + dist(u, n) - dist(p, n);
}

void erase(int u) {
    auto it = s.find({dfn[u], u});
    auto nxt = next(it), prv = (it == s.begin()) ? prev(s.end()) : prev(it);
    if (s.size() == 1) { s.erase(it); total = 0; return; }
    if (s.size() == 2) { s.erase(it); total = 0; return; }
    int p = prv->second, n = (nxt == s.end()) ? s.begin()->second : nxt->second;
    total -= dist(p, u) + dist(u, n) - dist(p, n);
    s.erase(it);
}
// 查询答案：total / 2
```

**核心启示**：
- **DFS 序 + LCA 距离公式** 可以动态维护虚树信息，无需真正建虚树
- 环形相邻和公式是动态虚树题的核心技巧
- 插入/删除只影响相邻两个节点的距离贡献

### 题目 10：树上差分 + LCA（洛谷 P3128 / USACO 2015 Dec Platinum）

> 给定一棵 $n$ 个节点的树，$m$ 次操作，每次将路径 $u \to v$ 上的所有节点权值加 $1$。求操作结束后权值最大的节点。$n, m \le 10^5$。

**分析**：经典树上点差分。对每次路径 $(u,v)$，令 `diff[u]++, diff[v]++, diff[lca(u,v)]--, diff[fa[lca(u,v)]]--`，最后 DFS 求子树和。

```
数据结构：倍增 LCA + 树上点差分
支持操作：路径加 1 + 查询最大值
关键性质：点差分公式：diff[u]++, diff[v]++, diff[lca]--, diff[fa[lca]]--
时间复杂度：O((n + m) log n)
```

**C++ 代码**：

```cpp
const int MAXN = 100005, LOG = 17;
int fa[MAXN][LOG], dep[MAXN], diff[MAXN];
vector<int> adj[MAXN];
int ans, ans_node;

// 预处理 dfs 和 lca 同题目 1

// 差分标记
void mark(int u, int v) {
    int l = lca(u, v);
    diff[u]++; diff[v]++;
    diff[l]--; if (fa[l][0]) diff[fa[l][0]]--;
}

// DFS 求子树和（即每个点的实际权值）
int dfs_sum(int u, int p) {
    int sum = diff[u];
    for (int v : adj[u]) if (v != p) sum += dfs_sum(v, u);
    if (sum > ans) { ans = sum; ans_node = u; }
    return sum;
}
```

**核心启示**：
- **点差分和边差分的公式不同**：点差分 `diff[lca]--` 和 `diff[fa[lca]]--`；边差分 `diff[lca]-=2`
- 树上差分 + LCA 是处理大量路径修改问题的利器，只需 $O(1)$ 标记每条路径

### 题目 12：Kruskal 重构树 + LCA（洛谷 P4768 / NOI 2018 归程）

> 给定一个 $n$ 个点 $m$ 条边的带权无向图，每条边有长度 $l$ 和海拔 $a$。$q$ 次查询：从 $s$ 出发，水位线为 $p$（海拔 $> p$ 的边可以开车，否则步行），问到达节点 $1$ 的最短步行距离。

**分析**：
1. 先用 Dijkstra 求节点 $1$ 到所有点的最短路 $d[v]$
2. 将边按海拔从大到小排序，建 Kruskal 重构树
3. 查询时在重构树上倍增找到 $s$ 能到达的最高祖先（海拔 $> p$），答案为该子树内 $d[v]$ 的最小值

```
数据结构：Kruskal 重构树 + 倍增 LCA + Dijkstra
支持操作：倍增跳到满足条件的最高祖先，子树最值查询
关键性质：重构树上 LCA 的权值 = 两点间最小瓶颈边的海拔
时间复杂度：O(m log m + n log n + q log n)
```

**C++ 代码**：

```cpp
const int MAXN = 400005, LOG = 20;  // 重构树节点数 ≤ 2n-1
int fa[MAXN][LOG], val[MAXN];       // val[u] 重构树节点的海拔值
long long mn[MAXN];                  // mn[u] 子树内 d[] 最小值

// 建 Kruskal 重构树（边按海拔降序）
int tot;  // 重构树节点编号从 n+1 开始
int find(int x) { return par[x] == x ? x : par[x] = find(par[x]); }

void build(vector<tuple<int,int,int>>& edges) {
    // edges: {altitude, u, v}，已按海拔降序排
    tot = n;
    for (auto [a, u, v] : edges) {
        int fu = find(u), fv = find(v);
        if (fu == fv) continue;
        ++tot;
        val[tot] = a;
        fa[fu][0] = fa[fv][0] = tot;
        par[fu] = par[fv] = tot;
        mn[tot] = min(mn[fu], mn[fv]);
    }
    // 倍增预处理
    for (int k = 1; k < LOG; k++)
        for (int u = 1; u <= tot; u++)
            fa[u][k] = fa[fa[u][k-1]][k-1];
}

long long query(int s, int p) {
    int u = s;
    for (int k = LOG - 1; k >= 0; k--)
        if (fa[u][k] && val[fa[u][k]] > p)
            u = fa[u][k];
    return mn[u];
}
```

**核心启示**：
- **Kruskal 重构树**将图上问题转化为树上问题，用倍增跳到满足条件的最高祖先
- 重构树的核心性质：两点在重构树上的 LCA 的权值 = 原图连通两点的最小瓶颈边权
- NOI 级别的综合题，融合了最短路 + Kruskal + 倍增

### 题目 13：树上路径 GCD 查询

> 给定一棵 $n$ 个节点的带权树，$q$ 次查询路径 $u \to v$ 上所有边权的 GCD。$n, q \le 10^5$。

**分析**：倍增维护路径 GCD，与路径最值类似，只是合并操作换成 gcd。

```
数据结构：fa[u][k] + gcdw[u][k]
支持操作：路径 GCD 查询
关键性质：gcdw[u][k] = gcd(gcdw[u][k-1], gcdw[fa[u][k-1]][k-1])
时间复杂度：O((n + q) log n · log W)，W 为边权上限
```

**C++ 代码**：

```cpp
const int MAXN = 100005, LOG = 17;
int fa[MAXN][LOG], dep[MAXN], gcdw[MAXN][LOG];
vector<pair<int,int>> adj[MAXN];

void dfs(int u, int p, int d) {
    fa[u][0] = p; dep[u] = d;
    for (int k = 1; k < LOG; k++) {
        fa[u][k] = fa[fa[u][k-1]][k-1];
        gcdw[u][k] = __gcd(gcdw[u][k-1], gcdw[fa[u][k-1]][k-1]);
    }
    for (auto [v, w] : adj[u]) if (v != p) {
        gcdw[v][0] = w;
        dfs(v, u, d + 1);
    }
}

int query(int u, int v) {
    int res = 0;
    if (dep[u] < dep[v]) swap(u, v);
    int diff = dep[u] - dep[v];
    for (int k = 0; k < LOG; k++) if ((diff >> k) & 1) {
        res = __gcd(res, gcdw[u][k]); u = fa[u][k];
    }
    if (u == v) return res;
    for (int k = LOG - 1; k >= 0; k--) if (fa[u][k] != fa[v][k]) {
        res = __gcd(res, __gcd(gcdw[u][k], gcdw[v][k]));
        u = fa[u][k]; v = fa[v][k];
    }
    return __gcd(res, __gcd(gcdw[u][0], gcdw[v][0]));
}
```

**核心启示**：
- **GCD 满足结合律**，可以直接套倍增路径查询模板
- 类似地，XOR、AND、OR 等位运算都可以这样维护
- 唯一的区别在于合并函数：`max` → `__gcd` → `^` → `&`

### 题目 14：路径上第 k 大边权（倍增 + 路径提取 / 主席树变体）

> 给定一棵 $n$ 个节点的带权树，$q$ 次查询路径 $u \to v$ 上第 $k$ 大的边权。

**分析**：倍增求 LCA 定位路径后，用主席树（可持久化线段树）或在线归并来回答第 $k$ 大。这里用倍增提取路径 + 排序的简单做法（适合 $n$ 较小时）。对大规模数据，需树上主席树。

```
数据结构：倍增 LCA + 路径提取 + 排序（简单做法），或树上主席树（高效做法）
支持操作：路径第 k 大查询
关键性质：LCA 将路径拆为两段：u→lca 和 v→lca
时间复杂度：简单做法 O(L log L)，L 为路径长度；主席树 O(log n)
```

**C++ 代码**：

```cpp
// 简单做法：提取路径上所有边权后排序
vector<int> get_path_weights(int u, int v) {
    vector<int> res;
    int l = lca(u, v);
    // u 到 lca 的路径
    int x = u;
    while (x != l) {
        res.push_back(edge_weight(x, fa[x][0]));  // x 到父亲的边权
        x = fa[x][0];
    }
    // v 到 lca 的路径
    x = v;
    while (x != l) {
        res.push_back(edge_weight(x, fa[x][0]));
        x = fa[x][0];
    }
    sort(res.rbegin(), res.rend());
    return res;
}
// 第 k 大 = res[k-1]
```

**核心启示**：
- **倍增 LCA 定位路径两端**是所有路径查询的第一步，具体怎么回答取决于查询类型
- 简单的提取+排序适合路径短或查询少的情况，大规模需要主席树

### 题目 15：树上两条路径交集（CF 519E / Hedgehog and Tree）

> 给定一棵树和两条路径 $(a, b)$ 与 $(c, d)$，求两条路径的交集长度。

**分析**：两条路径的交集也是一条路径（可以为空或一个点）。设 $L_1 = \text{LCA}(a,b)$，路径 $(a,b)$ 经过 $L_1$。交集的两端点可以通过求四个 LCA 来确定：$\text{LCA}(a,c), \text{LCA}(a,d), \text{LCA}(b,c), \text{LCA}(b,d)$，取深度最大的两个（且都在两条路径上）作为交集端点。

```
数据结构：倍增 LCA + dep[]
支持操作：两条路径交集长度
关键性质：交集端点 = 四个跨路径 LCA 中深度最大的两个
时间复杂度：O(log n) 每次查询
```

**C++ 代码**：

```cpp
// 判断 x 是否在路径 (u, v) 上
bool on_path(int x, int u, int v) {
    return dep[lca(x, u)] + dep[lca(x, v)] - 2 * dep[lca(u, v)] == dep[x] - dep[lca(u, v)];
    // 等价于 dist(u, x) + dist(x, v) == dist(u, v)
}

int path_intersection(int a, int b, int c, int d) {
    // 收集 4 个跨路径 LCA
    int pts[] = {lca(a,c), lca(a,d), lca(b,c), lca(b,d)};
    // 取深度最大的两个不同点作为交集端点
    sort(pts, pts + 4, [](int x, int y) { return dep[x] > dep[y]; });

    int u = pts[0], v = pts[1];
    // 验证 u, v 同时在两条路径上
    if (on_path(u, a, b) && on_path(u, c, d) &&
        on_path(v, a, b) && on_path(v, c, d)) {
        return dep[u] + dep[v] - 2 * dep[lca(u, v)];  // 交集长度
    }
    return 0;  // 无交集
}
```

**核心启示**：
- **树上两条路径的交集仍是路径**，这是树结构的优美性质
- 巧用多次 LCA 求交集端点，是 LCA 组合应用的典范
- `on_path` 判断 = `dist(u,x) + dist(x,v) == dist(u,v)`

### 题目 16：运输计划（洛谷 P2680 / NOIP 2015 提高组）

> 给定一棵 $n$ 个节点的带权树和 $m$ 条运输路径 $(u_i, v_i)$。可以将一条边的权值变为 $0$，求所有路径长度的最大值的最小值。$n, m \le 3 \times 10^5$。

**分析**：二分答案 $mid$，check 时统计长度 $> mid$ 的路径集合，用树上差分找被所有这些路径共同覆盖的边，检查是否存在一条这样的边使得删去它后所有路径长度 $\le mid$。

```
数据结构：倍增 LCA + 二分答案 + 树上边差分
支持操作：路径长度计算 + 差分覆盖次数 + 二分检验
关键性质：删去的边必须被所有"超标"路径覆盖，且边权足够大
时间复杂度：O((n + m) log n · log W)
```

**C++ 代码**：

```cpp
const int MAXN = 300005, LOG = 19;
int fa[MAXN][LOG], dep[MAXN], diff[MAXN];
long long dis[MAXN];  // 到根的距离
vector<pair<int,int>> adj[MAXN];
int n, m;
int path_u[MAXN], path_v[MAXN], path_lca[MAXN];
long long path_len[MAXN];

long long get_dist(int u, int v) {
    return dis[u] + dis[v] - 2 * dis[lca(u, v)];
}

// DFS 求子树和，返回子树差分和，同时记录最大可删边权
long long max_edge;
int dfs_check(int u, int p) {
    int cnt = diff[u];
    for (auto [v, w] : adj[u]) if (v != p) {
        cnt += dfs_check(v, u);
    }
    // cnt 是边 (fa[u], u) 被覆盖的次数，该边权 = dis[u] - dis[fa[u][0]]
    // 如果 cnt == total_bad（被所有超标路径覆盖），更新 max_edge
    return cnt;
}

bool check(long long mid) {
    memset(diff, 0, sizeof(int) * (n + 1));
    int total_bad = 0;
    long long max_path = 0;
    for (int i = 0; i < m; i++) {
        if (path_len[i] > mid) {
            total_bad++;
            diff[path_u[i]]++; diff[path_v[i]]++;
            diff[path_lca[i]] -= 2;
            max_path = max(max_path, path_len[i]);
        }
    }
    if (total_bad == 0) return true;
    // DFS 找被 total_bad 条路径全覆盖的边中权值最大的
    max_edge = 0;
    // ... DFS 求差分和，若 cnt == total_bad 且边权 >= max_path - mid，则可行
    return max_edge >= max_path - mid;
}
```

**核心启示**：
- **二分答案 + LCA + 树上差分** 三者联合是处理"最大值最小化"路径问题的标准方案
- 差分统计"被所有超标路径覆盖的公共边"是此题的精髓
- NOIP 提高组的压轴题级别，综合性很强

### 🟣 进阶 Level 5：倍增 LCA 的高级扩展

### 题目 17：树上倍增 + 二分（在路径上二分跳跃）

> 给定一棵带权有根树，查询从节点 $u$ 出发向根走，最多走总距离 $W$ 能到达的最浅节点（即最远祖先）。

**分析**：利用倍增从高位到低位贪心跳跃：如果跳 $2^k$ 步后总距离 $\le W$，则跳。

```
数据结构：fa[u][k] + sumw[u][k]（到第 2^k 级祖先的路径权值和）
支持操作：路径上的"倍增二分"
关键性质：从高位到低位贪心，类似于二进制分解
时间复杂度：O(log n) 每次查询
```

**C++ 代码**：

```cpp
const int MAXN = 200005, LOG = 18;
int fa[MAXN][LOG], dep[MAXN];
long long sumw[MAXN][LOG];

void dfs(int u, int p, int d) {
    fa[u][0] = p; dep[u] = d;
    for (int k = 1; k < LOG; k++) {
        fa[u][k] = fa[fa[u][k-1]][k-1];
        sumw[u][k] = sumw[u][k-1] + sumw[fa[u][k-1]][k-1];
    }
    for (auto [v, w] : adj[u]) if (v != p) {
        sumw[v][0] = w;
        dfs(v, u, d + 1);
    }
}

int farthest_ancestor(int u, long long W) {
    long long used = 0;
    for (int k = LOG - 1; k >= 0; k--) {
        if (fa[u][k] && used + sumw[u][k] <= W) {
            used += sumw[u][k];
            u = fa[u][k];
        }
    }
    return u;
}
```

**核心启示**：
- **倍增 + 贪心跳跃** = 在路径上做"二分搜索"，可以在 $O(\log n)$ 内找到满足条件的最远/最近位置
- 这个技巧广泛用于"预算有限，能走多远"类问题
- 关键是从高位到低位，"能跳则跳"的贪心策略

### 题目 18：边权树上的 LCA 路径异或和

> 给定一棵 $n$ 个节点的带权树，$q$ 次查询路径 $u \to v$ 上所有边权的异或和。$n, q \le 10^5$。

**分析**：异或的特殊性质：$a \oplus a = 0$。因此，$\text{xor}(u,v) = \text{xor\_to\_root}[u] \oplus \text{xor\_to\_root}[v]$（LCA 到根的部分被异或两次抵消）。甚至不需要倍增维护路径异或，只需前缀异或。

但此题仍然需要 LCA：若查询的是"路径上异或和最大的某一段"，则需要倍增维护更复杂的信息。

```
数据结构：fa[u][k] + xor_to_root[u]（到根的异或前缀）
支持操作：路径异或和查询
关键性质：xor(u,v) = xor_root[u] ^ xor_root[v]（LCA 部分自动抵消）
时间复杂度：O(n log n) 预处理，O(log n) 查询（含 LCA），但异或和本身 O(1)
```

**C++ 代码**：

```cpp
const int MAXN = 100005, LOG = 17;
int fa[MAXN][LOG], dep[MAXN];
long long xr[MAXN];  // xr[u] = 根到 u 路径上所有边权的异或和
vector<pair<int,int>> adj[MAXN];

void dfs(int u, int p, int d, long long x) {
    fa[u][0] = p; dep[u] = d; xr[u] = x;
    for (int k = 1; k < LOG; k++)
        fa[u][k] = fa[fa[u][k-1]][k-1];
    for (auto [v, w] : adj[u]) if (v != p)
        dfs(v, u, d + 1, x ^ w);
}

long long query_xor(int u, int v) {
    return xr[u] ^ xr[v];
    // 注意：这里甚至不需要显式求 LCA，因为 LCA 到根的部分异或两次抵消
}
```

**核心启示**：
- **异或的自逆性**使得路径异或和可以用前缀异或直接求，不需要在倍增中维护
- 但更复杂的异或查询（如路径上最大异或对）仍需倍增 + Trie 等高级结构
- 区分"可以用前缀性质简化"和"必须路径维护"的运算很重要

## 五、常见技巧与思路总结

### 5.1 倍增数组的初始化细节

```cpp
// LOG 的选取：LOG = __lg(n) + 1 或固定值
// 节点编号从 1 开始时，fa[root][k] = 0 表示不存在
// 重要：fa[0][k] = 0 保证越界安全
```

### 5.2 BFS 预处理替代 DFS（防爆栈）

当树退化为链（$n \le 5 \times 10^5$）时，DFS 递归可能爆栈。可用 BFS 预处理：

```cpp
void bfs(int root) {
    queue<int> q;
    dep[root] = 0; fa[root][0] = 0;
    q.push(root);
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int k = 1; k < LOG; k++)
            fa[u][k] = fa[fa[u][k-1]][k-1];
        for (int v : adj[u]) if (v != fa[u][0]) {
            dep[v] = dep[u] + 1;
            fa[v][0] = u;
            q.push(v);
        }
    }
}
```

### 5.3 倍增维护的信息类型

| 信息类型 | 合并操作 | 示例 |
|---------|---------|------|
| 最大值 | `max(a, b)` | 瓶颈路径 |
| 最小值 | `min(a, b)` | 最小边权 |
| 和 | `a + b` | 路径总权 |
| GCD | `gcd(a, b)` | 路径 GCD |
| XOR | `a ^ b` | 路径异或和 |
| 最大+次大 | 四元素排序 | 严格次小生成树 |

### 5.4 树上差分的两种形式

```
点差分（给路径上所有点 +1）：
  diff[u]++, diff[v]++, diff[lca]--, diff[fa[lca]]--

边差分（给路径上所有边 +1）：
  diff[u]++, diff[v]++, diff[lca] -= 2
```

### 5.5 LCA 与 DFS 序结合

```
判断 u 是 v 的祖先：dfn[u] <= dfn[v] && dfn[v] < dfn[u] + sz[u]
虚树构建时排序依据：DFS 序
```

## 七、终极总结与归纳

### 7.1 倍增 LCA 的本质

**用 $O(n \log n)$ 的预处理空间换取 $O(\log n)$ 的查询效率，将树上的"逐步爬升"加速为"二进制跳跃"。**

倍增法不仅是求 LCA 的工具，更是一种通用的"树上路径快速访问"框架——在跳跃过程中，可以同时聚合路径上的任意满足结合律的信息。

### 7.2 核心要点

1. **倍增递推** `fa[u][k] = fa[fa[u][k-1]][k-1]` 是一切的基石
2. **调平深度 + 同步跳跃** 是 LCA 查询的标准两步
3. **路径信息维护**只需在递推和查询时"顺便"合并
4. **树上差分 + LCA** 处理批量路径修改
5. **Kruskal 重构树** 将图上瓶颈问题转化为树上倍增问题
6. **倍增贪心跳跃** 实现路径上的"二分搜索"
