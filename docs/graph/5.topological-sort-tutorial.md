## 一、什么是拓扑排序

**拓扑排序（Topological Sort）** 是针对 **有向无环图（DAG, Directed Acyclic Graph）** 的一种线性排序方式：
将所有节点排成一条线，使得对于图中每一条有向边 $u \to v$，$u$ 在排序中都出现在 $v$ 之前。

### DAG 概念

```
  1 → 2 → 4
  ↓ ↗     ↑
  3 ——→ 5 ─┘
```

- **有向图**：每条边有明确方向
- **无环**：不存在从某个节点出发经过若干条边又回到自身的路径
- **只有 DAG 才存在拓扑排序**；反过来，若一个有向图能够被拓扑排序，则它一定是 DAG

### 偏序与全序

| 概念 | 含义 | 例子 |
|------|------|------|
| **偏序** | 只有 **部分** 元素之间有先后关系 | 课程依赖关系：A→B, A→C，但 B 与 C 之间没有确定顺序 |
| **全序** | **所有** 元素之间都有先后关系 | 拓扑序就是一种将偏序扩展为全序的方式 |

拓扑排序将偏序关系扩展为全序；如果偏序中缺少约束，拓扑序可能 **不唯一**。

### 直观理解

想象你有一堆待完成的任务，某些任务必须在其他任务之前完成。拓扑排序就是给这些任务排一个合法的执行顺序。

## 二、核心思想与前置知识

### 2.1 入度与 BFS 法（Kahn）

**核心思想**：反复选择入度为 0 的节点加入排序，删除其所有出边，更新相邻节点的入度。

```
步骤：
1. 统计所有节点的入度
2. 将所有入度为 0 的节点加入队列
3. 取出队首节点 u，将 u 加入拓扑序列
4. 遍历 u 的所有出边 u→v，将 v 的入度减一；若 v 入度变为 0，入队
5. 重复 3-4 直到队列为空
6. 若拓扑序列长度 = n，则排序成功；否则图中存在环
```

**时间复杂度**：$O(V + E)$

### 2.2 DFS 逆后序法

**核心思想**：对图做 DFS，当一个节点的所有后继都已访问完毕时（即回溯时），将该节点压入栈。最终栈的弹出顺序就是拓扑序。

```
步骤：
1. 对每个未访问的节点调用 dfs
2. dfs(u)：标记 u 为访问中，遍历所有出边 u→v
   - 若 v 未访问，递归 dfs(v)
   - 若 v 正在访问中（vis[v] == 1），说明存在环
3. u 的所有后继处理完毕后，标记 u 为已完成，将 u 压入栈
4. 栈中元素依次弹出即为拓扑排序结果
```

**时间复杂度**：$O(V + E)$

### 2.3 判环

拓扑排序天然具备判环能力：

| 方法 | 判环依据 |
|------|----------|
| **BFS (Kahn)** | 最终拓扑序列长度 $< n$，说明有节点始终入度不为 0，图中有环 |
| **DFS** | 访问到一个"正在访问中"状态的节点，说明存在环（后向边） |

## 三、分类与变体详解

### 3.1 BFS 拓扑排序（Kahn's Algorithm）

这是最常用的拓扑排序算法，思路清晰，便于与 DP 结合。

**C++ 代码**：
```cpp
// adj[u]: u 的邻接表, deg[v]: v 的入度, n: 节点总数
// 返回拓扑序列, 若为空则图中有环
vector<int> topoSort() {
    queue<int> q;
    vector<int> order;
    for (int i = 1; i <= n; i++)
        if (deg[i] == 0) q.push(i);
    while (!q.empty()) {
        int u = q.front(); q.pop();
        order.push_back(u);
        for (int v : adj[u])
            if (--deg[v] == 0) q.push(v);
    }
    return order.size() == n ? order : vector<int>();
}
```

**适用场景**：

- 标准拓扑排序 / 判断 DAG
- 配合 DP 逐层推进（如最长路、最短路）
- 需要逐层处理的场景（类似 BFS 分层）

### 3.2 DFS 拓扑排序

利用 DFS 的回溯顺序（逆后序）获得拓扑序列。

**C++ 代码**：
```cpp
// vis[u]: 0=未访问, 1=访问中, 2=已完成
// order: 存储拓扑排序结果（逆序压入）
bool hasCycle = false;
vector<int> order;

void dfs(int u) {
    vis[u] = 1;
    for (int v : adj[u]) {
        if (vis[v] == 1) { hasCycle = true; return; }
        if (vis[v] == 0) dfs(v);
    }
    vis[u] = 2;
    order.push_back(u);  // 回溯时记录
}
// 调用后 reverse(order.begin(), order.end()) 即为拓扑序
```

**适用场景**：

- 需要同时判环的场景
- 与强连通分量（Tarjan / Kosaraju）结合使用
- 偏好递归写法时

### 3.3 拓扑排序 + DP

DAG 上的 DP 是拓扑排序最经典的应用之一。由于 DAG 天然具有无后效性，按照拓扑序做 DP 可以保证每个节点被计算时，其所有前驱都已处理完毕。

**DAG 最长路**：
```cpp
// dist[u]: 从源点到 u 的最长距离
// w[u]: 节点 u 的权值（或边权）
void dagDP() {
    vector<int> order = topoSort();
    fill(dist + 1, dist + n + 1, 0);
    for (int u : order)
        for (auto [v, w] : adj[u])
            dist[v] = max(dist[v], dist[u] + w);
}
```

**典型应用**：

- 关键路径（项目管理中的最早/最晚完成时间）
- DAG 上最长路 / 最短路
- 任务调度中的最短总工期

### 3.4 字典序最小拓扑序

当存在多个合法拓扑序时，要求输出 **字典序最小** 的那一个。

**核心改动**：将 BFS 中的普通队列替换为 **小根堆（优先队列）**，每次优先选择编号最小的入度为 0 的节点。

**C++ 代码**：
```cpp
vector<int> topoSortMinLex() {
    priority_queue<int, vector<int>, greater<int>> pq;
    vector<int> order;
    for (int i = 1; i <= n; i++)
        if (deg[i] == 0) pq.push(i);
    while (!pq.empty()) {
        int u = pq.top(); pq.pop();
        order.push_back(u);
        for (int v : adj[u])
            if (--deg[v] == 0) pq.push(v);
    }
    return order;
}
```

**注意**：使用优先队列后时间复杂度为 $O((V + E) \log V)$，比普通 Kahn 多一个 $\log$ 因子。

## 四、例题详解

### 题目 1：拓扑排序模板 / 判断 DAG

> 给定一个 $n$ 个节点、$m$ 条边的有向图，判断它是否是 DAG，若是则输出一个合法的拓扑序。

**分析**：

这是拓扑排序的最基础应用。直接使用 Kahn 算法即可同时完成排序和判环。

```
思路：
1. 建图，统计入度
2. 执行 Kahn 拓扑排序
3. 若拓扑序长度 == n → 输出拓扑序
4. 否则 → 图中存在环，不是 DAG
```

**C++ 代码**：
```cpp
vector<int> topoSort() {
    queue<int> q;
    vector<int> order;
    for (int i = 1; i <= n; i++)
        if (deg[i] == 0) q.push(i);
    while (!q.empty()) {
        int u = q.front(); q.pop();
        order.push_back(u);
        for (int v : adj[u])
            if (--deg[v] == 0) q.push(v);
    }
    if ((int)order.size() < n)
        return {};  // 有环
    return order;
}
```

**复杂度**：$O(V + E)$

### 题目 2：P1113 杂务

> 有 $n$ 项杂务，第 $i$ 项杂务需要耗时 $t_i$，且必须在某些前驱杂务全部完成后才能开始。
> 求完成所有杂务的最短总时间。

**分析**：

经典的 **关键路径问题**，即 DAG 上的最长路。每个杂务的最早开始时间 = 所有前驱杂务的最早完成时间的最大值。

```
状态定义：f[u] = 完成杂务 u 的最早时间
转移方程：f[u] = max(f[v] for v in predecessors(u)) + t[u]
初始条件：无前驱的杂务 f[u] = t[u]
答案：max(f[1], f[2], ..., f[n])
```

**C++ 代码**：
```cpp
// t[u]: 杂务 u 的耗时, f[u]: 最早完成时间
void solve() {
    queue<int> q;
    for (int i = 1; i <= n; i++) {
        f[i] = t[i];
        if (deg[i] == 0) q.push(i);
    }
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int v : adj[u]) {
            f[v] = max(f[v], f[u] + t[v]);
            if (--deg[v] == 0) q.push(v);
        }
    }
    cout << *max_element(f + 1, f + n + 1);
}
```

**关键点**：

- 这里求的是"最长路"而非最短路——所有前驱都完成后才能开始，因此取 max
- 答案是所有 $f[u]$ 中的最大值，因为要等最后一个杂务完成

### 题目 3：P1983 车站分级

> 一条铁路线上有 $n$ 个车站。每趟车停靠若干个站，要求：如果某趟车在车站 $a$ 和 $b$ 都停靠（$a < b$），
> 则 $a, b$ 之间所有 **未停靠** 的站的级别必须严格低于 $a$ 和 $b$ 的级别。
> 求最少需要多少个不同的级别。

**分析**：

将"级别低于"的关系转化为 **有向边**：若站 $x$ 未停靠而站 $y$ 停靠（且 $x$ 在 $a, b$ 之间），则连一条边 $x \to y$，表示 $x$ 的级别 < $y$ 的级别。最终答案就是 **DAG 最长链长度**。

```
建图：
- 对于每趟车，在停靠区间 [a, b] 内，将所有未停靠站向所有停靠站连边
- 连边含义：未停靠站的级别 < 停靠站的级别

求解：
- 在 DAG 上求最长链（拓扑排序 + DP）
- 最长链长度即为最少级别数
```

**C++ 代码**：
```cpp
// 对每趟车建边后，拓扑排序 + DP 求最长链
void solve() {
    queue<int> q;
    for (int i = 1; i <= n; i++) {
        dist[i] = 1;
        if (deg[i] == 0) q.push(i);
    }
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int v : adj[u]) {
            dist[v] = max(dist[v], dist[u] + 1);
            if (--deg[v] == 0) q.push(v);
        }
    }
    cout << *max_element(dist + 1, dist + n + 1);
}
```

**注意**：暴力建边的边数可能很大（$O(n^2)$ 级别），可以通过虚拟节点优化建边来降低复杂度。对每趟车建一个虚拟节点 $p$，所有未停靠站向 $p$ 连权为 0 的边，$p$ 向所有停靠站连权为 1 的边。

### 题目 4：P1038 神经网络

> 给定一个 DAG 表示的神经网络。输入层节点（入度为 0）有初始值 $c_i$，
> 其他节点的状态为 $c_j = \sum_{i \to j} w_{ij} \cdot c_i - u_j$（其中 $u_j$ 是阈值），
> 若 $c_j \leqslant 0$ 则该节点不会向后传递信号。
> 输出所有输出层节点（出度为 0）的编号和状态值。

**分析**：

这是一道拓扑排序的直接模拟题。按照拓扑序依次计算每个节点的状态值。

```
思路：
1. 建图，统计入度和出度
2. 输入层节点（入度 = 0）初始值已知，加入队列
3. 按拓扑序处理：对于节点 u 的每条出边 u→v（边权 w），累加 c[v] += c[u] * w
4. 当节点 v 入度减为 0 时，减去阈值：c[v] -= threshold[v]
   若 c[v] <= 0，则 c[v] = 0，该节点不传递信号
5. 输出所有出度为 0 的节点
```

**C++ 代码**：
```cpp
// edge: {to, weight}, c[u]: 节点状态, th[u]: 阈值
void solve() {
    queue<int> q;
    for (int i = 1; i <= n; i++)
        if (deg[i] == 0) q.push(i);
    while (!q.empty()) {
        int u = q.front(); q.pop();
        if (indeg_orig[u] != 0 && c[u] <= 0)
            { c[u] = 0; continue; }  // 非输入层且 <= 0，不传递
        for (auto [v, w] : adj[u]) {
            c[v] += c[u] * w;
            if (--deg[v] == 0) { c[v] -= th[v]; q.push(v); }
        }
    }
    for (int i = 1; i <= n; i++)
        if (outdeg[i] == 0 && c[i] > 0)
            cout << i << " " << c[i] << "\n";
}
```

**关键点**：

- 输入层节点（入度为 0）不需要减阈值，直接使用初始值
- 非输入层节点在收到所有前驱信号后才计算（自然由拓扑序保证）
- 只有状态 $> 0$ 的非输入层节点才会向后传递信号

### 题目 5：字典序最小的拓扑序列

> 给定一个 $n$ 个节点、$m$ 条边的 DAG，输出字典序最小的拓扑序列。

**分析**：

将 Kahn 算法中的普通队列替换为小根堆，保证每次选出编号最小的入度为 0 节点。

```
思路：
1. 统计入度，所有入度为 0 的节点加入小根堆
2. 每次取堆顶（最小编号），加入结果
3. 更新邻接节点入度，入度为 0 则入堆
```

**C++ 代码**：
```cpp
void solve() {
    priority_queue<int, vector<int>, greater<int>> pq;
    for (int i = 1; i <= n; i++)
        if (deg[i] == 0) pq.push(i);
    while (!pq.empty()) {
        int u = pq.top(); pq.pop();
        cout << u << " ";
        for (int v : adj[u])
            if (--deg[v] == 0) pq.push(v);
    }
}
```

**复杂度**：$O((V + E) \log V)$

**易错点**：使用 `greater<int>` 构造小根堆，C++ 默认优先队列是大根堆。

### 题目 6：DAG 最长路（拓扑排序 + DP）

> 给定一个带权 DAG，求从任意源点出发的最长路径长度。

**分析**：

在 DAG 上，最长路可以用拓扑排序 + DP 在 $O(V + E)$ 时间内解决（不同于一般图上的 NP-hard 问题）。

```
状态定义：dist[v] = 到达 v 的最长路径长度
转移方程：dist[v] = max(dist[v], dist[u] + w(u,v))  对所有边 u→v
初始条件：dist[u] = 0（或者指定单源 dist[s] = 0, 其余 -∞）
答案：max(dist[1..n])
```

**C++ 代码**：
```cpp
void dagLongestPath() {
    queue<int> q;
    for (int i = 1; i <= n; i++)
        if (deg[i] == 0) { dist[i] = val[i]; q.push(i); }
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (auto [v, w] : adj[u]) {
            dist[v] = max(dist[v], dist[u] + w);
            if (--deg[v] == 0) q.push(v);
        }
    }
    int ans = *max_element(dist + 1, dist + n + 1);
}
```

**与最短路算法对比**：

| 算法 | 适用图 | 时间复杂度 | 能否处理负权 |
|------|--------|-----------|-------------|
| Dijkstra | 一般图 | $O((V+E)\log V)$ | 不能 |
| Bellman-Ford | 一般图 | $O(VE)$ | 能 |
| **拓扑排序 + DP** | **DAG** | $O(V+E)$ | **能** |

### 题目 7：Kahn's BFS vs DFS 方法对比

> 对同一个 DAG，分别用 BFS（Kahn）和 DFS 两种方法求拓扑序，对比两者的差异。

**BFS（Kahn）方法**：
```cpp
vector<int> kahnBFS() {
    queue<int> q;
    vector<int> order;
    for (int i = 1; i <= n; i++)
        if (deg[i] == 0) q.push(i);
    while (!q.empty()) {
        int u = q.front(); q.pop();
        order.push_back(u);
        for (int v : adj[u])
            if (--deg[v] == 0) q.push(v);
    }
    return order;
}
```

**DFS 方法**：
```cpp
vector<int> order;
int vis[MAXN];

void dfs(int u) {
    vis[u] = 1;
    for (int v : adj[u])
        if (!vis[v]) dfs(v);
    order.push_back(u);
}

vector<int> dfsTopo() {
    for (int i = 1; i <= n; i++)
        if (!vis[i]) dfs(i);
    reverse(order.begin(), order.end());
    return order;
}
```

**两种方法对比**：

| 维度 | BFS（Kahn） | DFS 逆后序 |
|------|------------|-----------|
| 实现方式 | 队列 + 入度 | 递归 + 回溯压栈 |
| 判环方式 | 检查结果长度是否 $= n$ | 检查是否遇到"访问中"节点 |
| 适合场景 | DP 结合、分层处理 | SCC 算法、递归风格 |
| 字典序控制 | 换用优先队列即可 | 不易直接实现 |
| 空间 | $O(V)$ 队列 | $O(V)$ 递归栈 |
| 易出错点 | 忘记统计入度 | 忘记 reverse |

## 五、常见技巧与思路总结

### 技巧 1：虚拟节点优化建边

当需要从一组节点向另一组节点连边时（如 P1983），暴力连边的复杂度为 $O(|A| \times |B|)$。
引入虚拟节点 $p$，让 $A$ 中所有节点连向 $p$，$p$ 连向 $B$ 中所有节点，边数从 $O(|A| \times |B|)$ 降为 $O(|A| + |B|)$。

### 技巧 2：拓扑排序 + DP 万金油框架

几乎所有 DAG 上的最优化问题都可以用以下框架解决：

```
1. 建图并统计入度
2. 将所有入度为 0 的节点加入队列, 初始化 DP 值
3. 按拓扑序处理, 对每条出边做状态转移
4. 答案在汇点或所有节点中取最优
```

### 技巧 3：逆向拓扑排序

有时候正向拓扑不好做，可以 **反向建图**：将所有边的方向取反，然后从汇点开始做拓扑排序。适用于从终点逆推起点的问题。

### 技巧 4：度数统计法快速判环

如果只需判断有向图是否为 DAG（不需要求出拓扑序），可以直接运行 Kahn 算法，检查最终入队的节点数是否等于 $n$。

### 技巧 5：多源拓扑排序

如果 DAG 有多个入度为 0 的"源点"，Kahn 算法天然支持——初始时将所有源点入队即可。无需添加虚拟源点（除非 DP 需要统一的初始值）。

## 六、优化方法 / 解题框架

### 6.1 通用解题步骤

```
第一步：识别问题 → 是否存在"先后顺序"或"依赖关系"？→ 考虑拓扑排序
第二步：建图 → 明确节点和边的含义，确定边的方向
第三步：选择方法 → BFS (Kahn) or DFS？是否需要字典序？
第四步：是否需要 DP → 若求最值/计数，在拓扑序上做 DP
第五步：检查特殊情况 → 是否可能有环？多个连通分量？
```

### 6.2 什么时候想到拓扑排序

| 问题特征 | 拓扑排序的作用 |
|---------|---------------|
| 任务有先后依赖 | 确定合法执行顺序 |
| 求最早/最晚完成时间 | 拓扑序 + DP（关键路径） |
| 判断是否有循环依赖 | 拓扑排序是否能完成 |
| DAG 上求最值（最长路/最短路） | 拓扑序保证 DP 无后效性 |
| 要求字典序最小的排列 | 小根堆 + Kahn |

### 6.3 BFS vs DFS 选择指南

```
需要分层处理 / 结合 DP / 字典序 → BFS (Kahn)
需要判断后向边 / 配合 SCC → DFS
两者都可以时 → 推荐 BFS (更直观、不会栈溢出)
```

### 6.4 复杂度速查

| 操作 | 时间复杂度 |
|------|-----------|
| Kahn 拓扑排序 | $O(V + E)$ |
| DFS 拓扑排序 | $O(V + E)$ |
| 字典序最小拓扑排序 | $O((V + E) \log V)$ |
| 拓扑排序 + DP | $O(V + E)$ |
| 虚拟节点优化建边 | 额外 $O(V)$ |

## 七、终极总结与归纳

### 核心知识点回顾

| 知识点 | 要点 |
|--------|------|
| DAG | 有向无环图，是拓扑排序的前提 |
| Kahn 算法 | BFS + 入度表，最常用的拓扑排序方法 |
| DFS 逆后序 | 递归回溯时记录顺序，适合配合 SCC |
| 判环 | Kahn 检查序列长度，DFS 检查后向边 |
| 拓扑排序 + DP | DAG 上最值问题的标准解法 |
| 字典序最小 | 将队列替换为小根堆 |

### 做题思维导图

```
        遇到依赖关系 / 先后顺序
                ↓
         建有向图, 考虑拓扑排序
        ↙               ↘
   只需排序/判环      需要求最值/计数
      ↓                  ↓
  标准 Kahn           拓扑排序 + DP
      ↓                  ↓
  需要字典序？       DAG 最长路/最短路
    ↓    ↓
   是    否
   ↓     ↓
 小根堆  普通队列
```

### 易错清单

| 错误 | 后果 | 修正 |
|------|------|------|
| 忘记统计入度 | 所有节点都入队，排序无意义 | 建图时同步计算 `deg[v]++` |
| BFS 队列用错 | 字典序不对 | 明确题意后选队列或堆 |
| DFS 忘记 reverse | 拓扑序反了 | 回溯记录的是逆序，必须翻转 |
| 没有判环 | WA / TLE | 检查拓扑序长度或 DFS 环标记 |
| 边的方向建反 | 拓扑序完全错误 | 牢记 $u \to v$ 表示 $u$ 先于 $v$ |
| 多组数据不清空 | 数据残留导致 WA | 每组数据重置 `deg[]`, `adj[]`, `vis[]` |

### 一句话总结

> **拓扑排序 = 在 DAG 上找一个"不违背依赖关系"的线性顺序，BFS (Kahn) 最常用，配合 DP 能解决 DAG 上几乎所有最优化问题。**
