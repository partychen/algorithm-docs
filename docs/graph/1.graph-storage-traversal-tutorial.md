## 一、什么是图的存储与遍历

**图（Graph）** 是由 **顶点（Vertex）** 和 **边（Edge）** 组成的数据结构，用来描述事物之间的关系。图论是算法竞赛中最庞大的分支之一，而图的存储与遍历则是图论的地基——几乎所有高级图论算法（最短路、连通性、网络流……）都建立在正确地存储图和高效地遍历图之上。

用一句话概括：

> **选择合适的方式把图存下来，然后用 DFS 或 BFS 把图走一遍，提取出需要的信息。**

### 图的基本分类

| 分类依据 | 类型 | 说明 |
|---------|------|------|
| 边的方向 | **无向图** / **有向图** | 无向边 $(u,v)$ 等价于双向；有向边 $u \to v$ 单方向 |
| 边的权值 | **无权图** / **有权图** | 无权图可视为所有边权都为 $1$ |
| 特殊结构 | **树** / **DAG** / **功能图** / **基环树** | 树是无环连通图；功能图每个点恰好一条出边 |

### 三种主流存储方式对比

| 存储方式 | 空间复杂度 | 加边 | 遍历邻居 | 适用场景 |
|---------|-----------|------|---------|---------|
| 邻接矩阵 | $O(n^2)$ | $O(1)$ | $O(n)$ | $n$ 小（$\le 5000$），稠密图，需快速查询两点是否有边 |
| vector 邻接表 | $O(n+m)$ | $O(1)$ 均摊 | $O(\deg)$ | **最常用**，代码简洁，足以应对绝大多数题目 |
| 链式前向星 | $O(n+m)$ | $O(1)$ | $O(\deg)$ | 需要极致常数优化或支持删边操作 |

### 存储方式的本质

不管哪种存储方式，核心都是同一件事——**记录每个点出发能到达哪些邻居（以及边权）**。所有区别仅仅在于：**用什么数据结构来组织这些邻居信息。**

## 二、核心思想与前置知识

### 2.1 图的基本概念

**度（Degree）**：无向图中一个点连接的边数。有向图中分为 **入度** 和 **出度**。

**连通性**：无向图中任意两点可以互相到达，称为 **连通图**。连通图的极大连通子图称为 **连通分量**。

**环（Cycle）**：从某个点出发，沿边走回到自身的路径。无向图中长度 $\ge 3$ 的简单回路即为环；有向图中沿有向边走回起点即为有向环。

**功能图（Functional Graph）**：每个点的出度恰好为 $1$ 的有向图。其结构必然是若干 **$\rho$ 形**（一条链接入一个环），又称"内向基环树森林"。

$$
\text{功能图的结构} = \text{若干棵树的根挂在环上（基环树）}
$$

### 2.2 三种存储方式详解

#### 邻接矩阵

用二维数组 `g[u][v]` 存储 $u$ 到 $v$ 的边信息（边权或是否有边）。

```cpp
// 邻接矩阵
int g[N][N]; // g[u][v] = w 表示 u->v 有一条权为 w 的边；0 表示无边
void addEdge(int u, int v, int w) {
    g[u][v] = w;
    // 无向图还需 g[v][u] = w;
}
```

**优点**：$O(1)$ 查询任意两点是否有边。**缺点**：$O(n^2)$ 空间，$n$ 大时不可用。

#### vector 邻接表

每个点 $u$ 维护一个 `vector`，存储所有从 $u$ 出发的边。

```cpp
// vector 邻接表
vector<pair<int,int>> G[N]; // G[u] = {(v1,w1), (v2,w2), ...}
void addEdge(int u, int v, int w) {
    G[u].push_back({v, w});
    // 无向图还需 G[v].push_back({u, w});
}
```

**这是竞赛中最推荐的写法**：代码简洁、调试方便、性能足够。

#### 链式前向星

用数组模拟链表，边按照加入顺序存储在数组中，每个点维护一个"链表头"。

```cpp
// 链式前向星
int head[N], nxt[M], to[M], wt[M], ecnt;
void addEdge(int u, int v, int w) {
    to[++ecnt] = v;
    wt[ecnt] = w;
    nxt[ecnt] = head[u];
    head[u] = ecnt;
}
// 遍历 u 的所有邻边
// for (int i = head[u]; i; i = nxt[i]) { int v = to[i], w = wt[i]; ... }
```

**优点**：内存连续、常数小，支持通过边编号直接操作边（如网络流中反向边 `i^1`）。**缺点**：代码较长，不易调试。

### 2.3 DFS 遍历框架

**深度优先搜索（DFS）** 从起点出发，沿一条路尽可能深入，直到走不动再回溯。

```
DFS 遍历流程：
1. 标记起点已访问
2. 遍历当前点的所有邻居：
   a. 若邻居未访问，递归进入
   b. 递归返回后继续遍历下一个邻居
3. 所有邻居遍历完毕，回溯
```

```cpp
// DFS 基本框架
bool vis[N];
void dfs(int u) {
    vis[u] = true;
    for (auto [v, w] : G[u]) {
        if (!vis[v]) dfs(v);
    }
}
```

**DFS 的核心性质**：
- **时间复杂度**：$O(n + m)$，每个点和每条边恰好访问一次。
- **空间复杂度**：$O(n)$ 递归栈。注意递归深度可能达到 $n$，极端情况下需改为非递归或增大栈空间。
- **DFS 序**：按照访问顺序给每个节点编号，是许多高级算法（Tarjan、树链剖分）的基础。

### 2.4 BFS 遍历框架

**广度优先搜索（BFS）** 从起点出发，按层扩展——先访问距离为 $1$ 的点，再访问距离为 $2$ 的点……

```
BFS 遍历流程：
1. 将起点入队，标记已访问
2. 取出队头 u，遍历 u 的所有邻居：
   a. 若邻居 v 未访问，标记并入队
3. 重复直到队空
```

```cpp
// BFS 基本框架
int dist[N];
void bfs(int s) {
    memset(dist, -1, sizeof dist);
    queue<int> q;
    dist[s] = 0; q.push(s);
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (auto [v, w] : G[u]) {
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
}
```

**BFS 的核心性质**：
- 在 **无权图** 中，BFS 求出的 $\text{dist}[]$ 就是最短距离。
- **时间复杂度**：$O(n + m)$。
- BFS 天然按层遍历，非常适合"最少步数"类问题。
- **多源 BFS**：将所有源点同时入队，一次 BFS 求出每个点到最近源点的距离。

### 2.5 找环算法

#### DFS 染色法（有向图找环）

用三种颜色标记节点状态：
- **白色（0）**：未访问
- **灰色（1）**：正在递归栈中（当前路径上）
- **黑色（2）**：已回溯完成

若 DFS 过程中遇到灰色节点，说明找到了一个 **有向环**。

```cpp
// DFS 染色法判有向图是否有环
int color[N]; // 0=白, 1=灰, 2=黑
bool hasCycle;
void dfs(int u) {
    color[u] = 1;
    for (int v : G[u]) {
        if (color[v] == 1) { hasCycle = true; return; }
        if (color[v] == 0) dfs(v);
    }
    color[u] = 2;
}
```

#### 功能图找环（每个点出度为 1）

在功能图中，每个连通分量恰好包含一个环。可以用"沿边走 + 标记"的方式高效找到环及其长度。

```cpp
// 功能图找环：nxt[u] 为 u 的唯一后继
// vis[u]: 0 未访问, 其他为访问批次编号
// 当碰到同一批次已访问的点时，说明找到环
int vis[N], step[N]; // step[u] 记录到 u 时走了几步
void findCycle(int s, int batch) {
    int u = s, cnt = 0;
    while (!vis[u]) {
        vis[u] = batch;
        step[u] = cnt++;
        u = nxt[u];
    }
    if (vis[u] == batch) { // 找到环，环长 = cnt - step[u]
        int cycleLen = cnt - step[u];
    }
}
```

## 三、分类与变体详解

### 3.1 邻接矩阵存储

邻接矩阵最适合 $n$ 较小的稠密图，例如 Floyd 全源最短路要求的输入格式。

**使用时机**：
- $n \le 5000$（通常 $n \le 500$ 才会在 Floyd 等算法中使用矩阵）。
- 需要频繁查询 $(u, v)$ 之间是否有边。
- 涉及矩阵运算（如矩阵快速幂求路径数）。

### 3.2 邻接表存储（vector / 链式前向星）

邻接表是绝大多数图论题的首选存储方式。vector 版本代码简洁，链式前向星在需要"反向边编号"等特殊操作时更方便。

**使用时机**：
- 几乎所有情况。
- 无权图可以用 `vector<int> G[N]`，有权图用 `vector<pair<int,int>> G[N]`。

### 3.3 DFS 遍历的典型应用

| 应用 | 说明 |
|------|------|
| 连通分量计数 | 对每个未访问点启动一次 DFS，启动次数即为连通分量数 |
| 有向图判环 | DFS 染色法，遇到灰色节点即存在环 |
| 二分图判定 | DFS 染色（黑白交替），遇到同色邻居则不是二分图 |
| 求树的深度/子树大小 | 在树上 DFS 时维护深度和子树节点数 |
| 功能图找环 | 沿唯一出边走，碰到当前路径上的点即找到环 |

### 3.4 BFS 遍历的典型应用

| 应用 | 说明 |
|------|------|
| 无权图最短路 | BFS 天然求出最短距离 |
| 多源 BFS | 多个起点同时入队，求每个格子到最近源点的距离 |
| 0-1 BFS | 边权只有 $0$ 和 $1$，用双端队列代替普通队列 |
| Flood Fill | 从某点出发扩展到所有同色/满足条件的相邻格子 |
| 拓扑排序（BFS 版） | 入度为 $0$ 的点入队，逐步删边 |

### 3.5 0-1 BFS 详解

当图中边权只有 $0$ 和 $1$ 两种时，可以用 **双端队列（deque）** 代替优先队列实现 $O(n+m)$ 的最短路。

**核心思想**：权为 $0$ 的边松弛后将邻居放到队首（优先处理），权为 $1$ 的边放到队尾。

```cpp
// 0-1 BFS
int dist[N];
void bfs01(int s) {
    memset(dist, 0x3f, sizeof dist);
    deque<int> dq;
    dist[s] = 0; dq.push_back(s);
    while (!dq.empty()) {
        int u = dq.front(); dq.pop_front();
        for (auto [v, w] : G[u]) {
            if (dist[v] > dist[u] + w) {
                dist[v] = dist[u] + w;
                if (w == 0) dq.push_front(v);
                else dq.push_back(v);
            }
        }
    }
}
```

### 3.6 多源 BFS 详解

多源 BFS 用于求"每个点到最近的一组特殊点的距离"。只需在 BFS 初始化时将所有源点同时入队。

```cpp
// 多源 BFS
void multiBFS(vector<int>& sources) {
    memset(dist, -1, sizeof dist);
    queue<int> q;
    for (int s : sources) {
        dist[s] = 0;
        q.push(s);
    }
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int v : G[u]) {
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
}
```

**本质**：等价于建一个虚拟超级源点 $S$，从 $S$ 向所有真实源点连权为 $0$ 的边，然后从 $S$ 跑 BFS。多源 BFS 只是省略了建虚拟点的步骤。

## 四、10 道经典例题详解

### 题目 1：连通分量计数（基础练习）

> 给定一张 $n$ 个点 $m$ 条边的无向图，求连通分量的个数。$n, m \le 10^5$。

**难度**：🟢

**分析**：

```
最基础的图遍历应用。
对每个未访问的点启动一次 DFS（或 BFS），启动次数即为连通分量数。
时间复杂度 O(n + m)。
```

**C++ 代码**：

```cpp
vector<int> G[N];
bool vis[N];
int ans;

void dfs(int u) {
    vis[u] = true;
    for (int v : G[u])
        if (!vis[v]) dfs(v);
}

void solve() {
    for (int i = 1; i <= n; i++)
        if (!vis[i]) { dfs(i); ans++; }
}
```

**核心启示**：DFS/BFS 求连通分量数是图遍历的最基本应用——"启动几次遍历，就有几个连通分量"。

### 题目 2：二分图判定（洛谷 P1330 封锁阳光大学）

> 给定一张 $n$ 个点 $m$ 条边的无向图，判断能否将所有点分成两组，使得每条边的两个端点在不同组中。如果可以，求一种方案使某一组点数尽量少。$n \le 10^4$，$m \le 10^5$。

**难度**：🟢

**分析**：

```
经典的二分图判定（二染色法）问题。
对每个连通分量做 DFS 染色：起点染白色，邻居染黑色，黑色的邻居染白色……
如果发现某个邻居已经被染了和自己相同的颜色，则该连通分量不是二分图。
每个连通分量独立判断。若某个连通分量的两色点数为 a 和 b，选 min(a, b) 加入答案。
```

**C++ 代码**：

```cpp
vector<int> G[N];
int col[N]; // -1 未染色, 0/1 两种颜色
int cnt[2];
bool ok;

void dfs(int u, int c) {
    col[u] = c; cnt[c]++;
    for (int v : G[u]) {
        if (col[v] == -1) dfs(v, c ^ 1);
        else if (col[v] == c) ok = false;
    }
}

void solve() {
    memset(col, -1, sizeof col);
    int ans = 0;
    for (int i = 1; i <= n; i++) {
        if (col[i] == -1) {
            ok = true; cnt[0] = cnt[1] = 0;
            dfs(i, 0);
            if (!ok) { puts("Impossible"); return; }
            ans += min(cnt[0], cnt[1]);
        }
    }
    printf("%d\n", ans);
}
```

**核心启示**：二分图判定 = DFS/BFS 二染色。遇到冲突（同色邻居）即非二分图。这也是二分图匹配等高级算法的前置知识。

### 题目 3：Flood Fill — 湖计数（洛谷 P1596 Lake Counting）

> 给定一个 $N \times M$ 的字符矩阵，`W` 表示水，`.` 表示地。水的八连通域算作一个湖。求湖的个数。$N, M \le 100$。

**难度**：🟢

**分析**：

```
经典 Flood Fill 问题。
将矩阵看成隐式图：每个格子是一个节点，相邻的 W 格子之间有边。
对每个未访问的 W 格子启动一次 DFS/BFS（八方向扩展），将整个连通块标记为已访问。
启动次数即为湖的个数。
```

**C++ 代码**：

```cpp
int n, m, ans;
char grid[105][105];
bool vis[105][105];
int dx[] = {-1,-1,-1,0,0,1,1,1};
int dy[] = {-1,0,1,-1,1,-1,0,1};

void dfs(int x, int y) {
    vis[x][y] = true;
    for (int d = 0; d < 8; d++) {
        int nx = x + dx[d], ny = y + dy[d];
        if (nx >= 0 && nx < n && ny >= 0 && ny < m
            && !vis[nx][ny] && grid[nx][ny] == 'W')
            dfs(nx, ny);
    }
}

void solve() {
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            if (!vis[i][j] && grid[i][j] == 'W') { dfs(i, j); ans++; }
}
```

**核心启示**：Flood Fill 本质就是在网格图上做连通分量计数。网格图的"邻居"由方向数组定义（四方向或八方向）。

### 题目 4：多源 BFS — 矩阵距离（洛谷 P1747 好奇怪的游戏 变体 / CF1520G）

> 给定一个 $n \times m$ 的 01 矩阵，求每个格子到最近的 $1$ 格子的曼哈顿距离。$n, m \le 1000$。

**难度**：🟢

**分析**：

```
经典多源 BFS 问题。
将所有值为 1 的格子作为源点同时入队，初始距离为 0。
BFS 逐层扩展，第一次到达某个格子时的距离即为该格子到最近源点的距离。
时间复杂度 O(nm)，每个格子恰好入队一次。
```

**C++ 代码**：

```cpp
int n, m, dist[1005][1005];
int dx[] = {-1, 0, 1, 0}, dy[] = {0, 1, 0, -1};

void solve() {
    queue<pair<int,int>> q;
    memset(dist, -1, sizeof dist);
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            if (grid[i][j] == 1) { dist[i][j] = 0; q.push({i, j}); }
    while (!q.empty()) {
        auto [x, y] = q.front(); q.pop();
        for (int d = 0; d < 4; d++) {
            int nx = x + dx[d], ny = y + dy[d];
            if (nx >= 0 && nx < n && ny >= 0 && ny < m && dist[nx][ny] == -1) {
                dist[nx][ny] = dist[x][y] + 1;
                q.push({nx, ny});
            }
        }
    }
}
```

**核心启示**：多源 BFS 的关键是"所有源点同时入队"。这等价于从虚拟超级源点出发的单源 BFS，但实现更简洁，无需真正建虚拟点。

### 题目 5：0-1 BFS（CF1263D / 洛谷 P4667 Switch the Lamp On）

> 给定一个 $n \times m$ 的网格，每个格子有一条对角线（`\` 或 `/`）。从左上角 $(0,0)$ 走到右下角 $(n,m)$，每步沿格子对角线走。可以花费 $1$ 的代价旋转某个格子的对角线。求到达终点的最小代价。

**难度**：🟡

**分析**：

```
每个格点是图中的节点，格子的对角线连接两对对角格点。
如果走的方向与格子对角线方向一致，边权为 0（不用旋转）；否则边权为 1（需要旋转）。
边权只有 0 和 1，经典 0-1 BFS。
用双端队列：权 0 的边放队首，权 1 的边放队尾。
时间复杂度 O(nm)。
```

**C++ 代码**：

```cpp
int dist[N][N]; // N > max(n,m)+1
int dx[] = {-1, -1, 1, 1}, dy[] = {-1, 1, -1, 1}; // 四个对角方向对应的格点偏移
int gx[] = { 0,  0, 0, 0}, gy[] = { 0,  0, 0, 0}; // 对应的格子坐标（需根据方向推导）
char need[] = {'\\', '/', '/', '\\'}; // 各方向需要的对角线类型

void solve() {
    memset(dist, 0x3f, sizeof dist);
    deque<pair<int,int>> dq;
    dist[0][0] = 0; dq.push_back({0, 0});
    while (!dq.empty()) {
        auto [x, y] = dq.front(); dq.pop_front();
        for (int d = 0; d < 4; d++) {
            int nx = x + dx[d], ny = y + dy[d];
            int cx = min(x, nx), cy = min(y, ny); // 格子坐标
            if (nx < 0 || nx > n || ny < 0 || ny > m) continue;
            int w = (grid[cx][cy] == need[d]) ? 0 : 1;
            if (dist[nx][ny] > dist[x][y] + w) {
                dist[nx][ny] = dist[x][y] + w;
                if (w == 0) dq.push_front({nx, ny});
                else dq.push_back({nx, ny});
            }
        }
    }
    printf("%d\n", dist[n][m] >= 0x3f3f3f3f ? -1 : dist[n][m]);
}
```

**核心启示**：0-1 BFS 是边权只有 0/1 时的最短路利器，时间复杂度 $O(n+m)$，比 Dijkstra 的 $O((n+m)\log n)$ 更优。核心技巧是用双端队列保证处理顺序的正确性。

### 题目 6：有向图判环（洛谷 P4782 前置 / 基础练习）

> 给定一张 $n$ 个点 $m$ 条边的有向图，判断图中是否存在环。$n, m \le 10^5$。

**难度**：🟢

**分析**：

```
DFS 染色法判环：
- 白色(0)：未访问
- 灰色(1)：在当前递归路径上
- 黑色(2)：已完成回溯
DFS 时遇到灰色节点，说明当前路径形成了环。
也可以用拓扑排序：若拓扑排序结束后仍有未入队的节点，说明有环。
```

**C++ 代码**：

```cpp
vector<int> G[N];
int color[N]; // 0 白 1 灰 2 黑
bool hasCycle;

void dfs(int u) {
    color[u] = 1;
    for (int v : G[u]) {
        if (color[v] == 1) { hasCycle = true; return; }
        if (color[v] == 0) { dfs(v); if (hasCycle) return; }
    }
    color[u] = 2;
}

void solve() {
    for (int i = 1; i <= n; i++)
        if (color[i] == 0) { dfs(i); if (hasCycle) break; }
    puts(hasCycle ? "Yes" : "No");
}
```

**核心启示**：有向图判环的标准方法是 DFS 三色标记法。灰色代表"正在当前路径上"，遇灰即有环。这也是拓扑排序是否可行的等价判定。

### 题目 7：信息传递（洛谷 P2661）

> $n$ 个人站成一圈，每人恰好将信息传给一个人（不能传给自己）。第 $i$ 个人将信息传给第 $T_i$ 个人。每轮每人同时传递。问最少经过几轮后，存在某人收到了自己发出的信息？$n \le 2 \times 10^5$。

**难度**：🟡

**分析**：

```
每个人恰好传给一个人 → 每个节点出度为 1 → 这是一个功能图。
"最少几轮后某人收到自己的信息" = 找到功能图中的最小环长度。
因为信息沿边传递，某人收到自己的信息等价于从该人出发沿边走回到自己，即该人在一个环上，
环的长度就是轮数。

方法：对每个未访问的节点，沿边走并记录步数。当走到一个本轮已访问的节点时，
环长 = 当前步数 - 该节点的步数。对所有环取最小值即为答案。

也可以使用拓扑排序删去所有非环节点（入度为0的点不断删），剩下的就是环，
对每个环 DFS 求长度，取最小值。
```

**C++ 代码**：

```cpp
int nxt[N]; // nxt[i] = T[i]，即功能图的后继
int vis[N], step[N];
int ans;

void solve() {
    ans = n + 1;
    int batch = 0;
    for (int i = 1; i <= n; i++) {
        if (vis[i]) continue;
        ++batch;
        int u = i, cnt = 0;
        while (!vis[u]) {
            vis[u] = batch;
            step[u] = cnt++;
            u = nxt[u];
        }
        if (vis[u] == batch) // 找到的环属于本轮
            ans = min(ans, cnt - step[u]);
    }
    printf("%d\n", ans);
}
```

**核心启示**：功能图（每个点出度为 $1$）的结构是"若干链汇入若干个环"。找最小环只需沿边走并记录步数，$O(n)$ 即可完成。这是基环树/功能图问题的经典入门。

### 题目 8：[USACO08DEC] Trick or Treat on the Farm（洛谷 P2921）

> $n$ 个农场，每个农场 $i$ 恰好有一条路通向农场 $\text{nxt}[i]$（功能图）。对每个农场 $i$，FJ 从 $i$ 出发沿路走，直到到达一个之前已经到过的农场为止。在停下之前（含起点），经过的不同农场数即为答案 $\text{ans}[i]$。求所有 $\text{ans}[i]$。$n \le 10^5$。

**难度**：🟡

**分析**：

```
功能图遍历问题。关键观察：
1. 功能图中每个连通分量恰好有一个环，其余节点形成若干链（树枝）挂在环上。
2. 环上的节点：ans = 环长（走完整个环回到自己停下）。
3. 链上的节点：ans = 链长（到环入口的距离） + 环长。

做法：
1. 先找出所有环：拓扑排序去掉非环节点，或用功能图找环法。
2. 环上节点直接赋值 ans = 环长。
3. 链上节点（拓扑排序删掉的点）：ans[u] = ans[nxt[u]] + 1。
   按拓扑逆序（从环出发往外推）处理即可。

也可以用"路径标记"法：从每个点出发沿边走，标记访问批次和步数，
碰到已处理的点或本轮已访问的点时停下，反推出所有经过节点的答案。
```

**C++ 代码**：

```cpp
int nxt[N], ans[N], deg[N];
bool onCycle[N];

void solve() {
    // 拓扑排序找环
    queue<int> q;
    for (int i = 1; i <= n; i++) deg[nxt[i]]++;
    for (int i = 1; i <= n; i++) if (deg[i] == 0) q.push(i);
    while (!q.empty()) {
        int u = q.front(); q.pop();
        if (--deg[nxt[u]] == 0) q.push(nxt[u]);
    }
    // deg[i] > 0 的点在环上
    for (int i = 1; i <= n; i++) {
        if (deg[i] == 0) continue;
        onCycle[i] = true;
        int len = 1, u = nxt[i];
        while (u != i) { len++; onCycle[u] = true; u = nxt[u]; }
        // 环上所有点 ans = len
        ans[i] = len; u = nxt[i];
        while (u != i) { ans[u] = len; u = nxt[u]; }
    }
    // 非环节点：按拓扑逆序处理
    // 重新拓扑排序
    for (int i = 1; i <= n; i++) deg[i] = 0;
    for (int i = 1; i <= n; i++) if (!onCycle[i]) deg[nxt[i]]++;
    for (int i = 1; i <= n; i++) if (!onCycle[i] && deg[i] == 0) q.push(i);
    // 注意：非环节点的 nxt 可能指向环上节点（已有 ans）
    // 我们需要从叶子往环方向推
    // 但由于链上的点需要反向处理，用 DFS 更方便
    // 改用简洁做法：对非环节点递归
    // ans[u] = ans[nxt[u]] + 1 （因为环上的 ans 已知）
    // 但需小心顺序，用记忆化
    // 此处用简洁的 DFS 记忆化
}
```

更简洁的完整做法：

```cpp
int nxt[N], ans[N], vis[N], stk[N];

void solve() {
    for (int i = 1; i <= n; i++) {
        if (ans[i]) continue;
        // 沿路径走，记录路径
        int top = 0, u = i;
        while (!vis[u] && !ans[u]) {
            vis[u] = i; // 标记属于第 i 次遍历
            stk[++top] = u;
            u = nxt[u];
        }
        if (ans[u]) {
            // 走到了已计算答案的点
            while (top && stk[top] != u) {
                ans[stk[top]] = ans[u] + (top - 0); // 不对，需逐个赋值
                top--;
            }
            // 逐个回退赋值
        }
        // 完整方法：先处理环部分，再处理链部分
    }
    // 更清晰的实现：
    memset(vis, 0, sizeof vis);
    memset(ans, 0, sizeof ans);
    for (int i = 1; i <= n; i++) {
        if (ans[i]) continue;
        vector<int> path;
        int u = i;
        while (!vis[u] && !ans[u]) { vis[u] = i; path.push_back(u); u = nxt[u]; }
        if (ans[u]) {
            // u 已有答案，链上节点逆序赋值
            for (int j = (int)path.size()-1; j >= 0; j--)
                ans[path[j]] = ans[u] + ((int)path.size() - j);
        } else if (vis[u] == i) {
            // 找到环：从 u 开始到路径末尾是环
            int cyc = 0;
            for (int j = (int)path.size()-1; ; j--) {
                cyc++;
                if (path[j] == u) break;
            }
            // 环上节点赋值
            for (int j = (int)path.size()-1; ; j--) {
                ans[path[j]] = cyc;
                if (path[j] == u) {
                    // 链上节点
                    for (int k = j - 1; k >= 0; k--)
                        ans[path[k]] = ans[path[k+1]] + 1;
                    break;
                }
            }
        }
    }
    for (int i = 1; i <= n; i++) printf("%d\n", ans[i]);
}
```

**核心启示**：功能图的经典处理模式是"找环 + 拓扑排序/链上回推"。环上的点行为相同（答案等于环长），链上的点答案递推自环方向。本题是功能图遍历的典型模板。

### 题目 9：迷宫问题 — BFS 求最短路（洛谷 P1141 01迷宫）

> 给定一个 $n \times n$ 的 01 矩阵。可以从某个格子走到相邻（上下左右）的格子，前提是两个格子的值不同（0 走到 1，或 1 走到 0）。给定 $m$ 个询问，每次问从 $(x, y)$ 出发能到达多少个格子。$n \le 1000$，$m \le 10^5$。

**难度**：🟡

**分析**：

```
两个相邻格子值不同才能走 → 将"值不同的相邻格子"连边，求连通分量。
同一连通分量内的所有格子互相可达 → 答案就是所在连通分量的大小。

BFS/DFS 预处理所有连通分量及其大小，然后对每个询问 O(1) 回答。
注意连边条件是"值不同"，所以合法路径必然 0-1-0-1 交替。
```

**C++ 代码**：

```cpp
int n, grid[1005][1005], id[1005][1005], sz[N];
int dx[] = {-1,0,1,0}, dy[] = {0,1,0,-1};
int comp;

void bfs(int sx, int sy) {
    ++comp;
    queue<pair<int,int>> q;
    q.push({sx, sy}); id[sx][sy] = comp; sz[comp] = 1;
    while (!q.empty()) {
        auto [x, y] = q.front(); q.pop();
        for (int d = 0; d < 4; d++) {
            int nx = x + dx[d], ny = y + dy[d];
            if (nx < 1 || nx > n || ny < 1 || ny > n) continue;
            if (id[nx][ny] || grid[nx][ny] == grid[x][y]) continue;
            id[nx][ny] = comp; sz[comp]++;
            q.push({nx, ny});
        }
    }
}

void solve() {
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= n; j++)
            if (!id[i][j]) bfs(i, j);
    // 对每个询问 (x, y)，答案为 sz[id[x][y]]
}
```

**核心启示**：大量询问"可达性"或"连通块大小"时，先预处理连通分量再 $O(1)$ 回答，避免对每个询问都做一次 BFS。这是"离线预处理"思想在图遍历中的经典应用。

### 题目 10：电话网络 — 树的 DFS 应用（洛谷 P3916 图的遍历）

> 给定一张 $n$ 个点 $m$ 条边的有向图，求每个点能到达的编号最大的点。$n, m \le 10^5$。

**难度**：🟡

**分析**：

```
正向思考：从每个点出发 DFS 找最大编号，复杂度可能 O(n(n+m))，太慢。
反向思考：建反图！从编号最大的点 n 开始 DFS（沿反向边），所有能到达的点的答案都是 n。
然后从 n-1 开始，跳过已有答案的点……以此类推。

从大到小遍历，在反图上 DFS，第一次到达某个点时，当前源点就是它能到达的最大编号。
时间复杂度 O(n + m)。
```

**C++ 代码**：

```cpp
vector<int> rG[N]; // 反图
int ans[N];

void dfs(int u, int val) {
    ans[u] = val;
    for (int v : rG[u])
        if (!ans[v]) dfs(v, val);
}

void solve() {
    // 读入边 (u, v)，在反图中加边 rG[v].push_back(u)
    for (int i = n; i >= 1; i--)
        if (!ans[i]) dfs(i, i);
    for (int i = 1; i <= n; i++) printf("%d ", ans[i]);
}
```

**核心启示**："正难则反"是图论中极为重要的思想。正向从每个点出发代价太高时，试试建反图、从目标出发反向搜索。

## 五、常见技巧与思路总结

### 1. 建图技巧

- **反向建图**：当需要求"谁能到达我"时，在反图上做"我能到达谁"。
- **隐式图**：网格、状态空间等不需要显式建图，用方向数组 / 状态转移代替邻接表。
- **虚拟节点**：多源 BFS 中的超级源点、分层图中的跨层边等。
- **拆点**：一个节点有多种状态时，将其拆为多个节点。

### 2. DFS 技巧

- **记忆化**：对已经处理过的节点记录答案，避免重复计算。
- **回溯与剪枝**：在搜索问题中常用，但在图遍历中通常不需要回溯。
- **非递归 DFS**：当递归深度可能达到 $10^5$ 以上时，改用栈模拟 DFS 防止栈溢出。

### 3. BFS 技巧

- **双端队列 BFS（0-1 BFS）**：边权 $\{0, 1\}$ 时使用，$O(n+m)$。
- **多源 BFS**：多个源点同时入队，一次 BFS 解决。
- **BFS 按层处理**：有些问题需要知道当前是第几层，可以记录层号或用 $\text{nullptr}$ 分隔。

### 4. 功能图 / 基环树技巧

- **找环**：沿唯一出边走，记录步数，碰到已访问的点即找到环。
- **拓扑排序去链**：将入度为 $0$ 的点反复删去，剩下的就是环上的点。
- **链上回推**：环上的点赋值后，链上的点通过 $\text{ans}[u] = f(\text{ans}[\text{nxt}[u]])$ 递推。

## 六、优化方法 / 解题框架

### 解题框架：拿到图论遍历题的思考流程

```
1. 识别图的类型
   ├─ 无向图 / 有向图？
   ├─ 有权 / 无权？
   ├─ 特殊结构？（树、DAG、功能图、网格图）
   └─ 数据规模？（决定存储方式和算法）

2. 确定存储方式
   ├─ n ≤ 5000 且需查边 → 邻接矩阵
   ├─ 一般情况 → vector 邻接表
   └─ 需要边编号操作 → 链式前向星

3. 确定遍历方式
   ├─ 求连通性 / 判环 / 拓扑结构 → DFS
   ├─ 求最短距离（无权图）→ BFS
   ├─ 边权 0/1 → 0-1 BFS
   ├─ 多源最近距离 → 多源 BFS
   └─ 功能图 → 沿边走 / 拓扑排序

4. 考虑优化方向
   ├─ 正难则反 → 建反图
   ├─ 离线预处理 → 先求连通分量再回答询问
   ├─ 记忆化 → 避免重复计算
   └─ 虚拟节点 / 拆点 → 简化建模
```

### 各存储方式的代码模板总结

| 操作 | 邻接矩阵 | vector 邻接表 | 链式前向星 |
|------|----------|--------------|-----------|
| 声明 | `int g[N][N]` | `vector<pair<int,int>> G[N]` | `int head[N], to[M], nxt[M], wt[M]` |
| 加边 | `g[u][v] = w` | `G[u].push_back({v,w})` | `to[++ec]=v; wt[ec]=w; nxt[ec]=head[u]; head[u]=ec;` |
| 遍历 | `for(int v=1;v<=n;v++) if(g[u][v])` | `for(auto [v,w]:G[u])` | `for(int i=head[u];i;i=nxt[i])` |

### 时间复杂度速查

| 算法 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| DFS / BFS | $O(n + m)$ | $O(n + m)$ |
| 多源 BFS | $O(n + m)$ | $O(n + m)$ |
| 0-1 BFS | $O(n + m)$ | $O(n + m)$ |
| 功能图找环 | $O(n)$ | $O(n)$ |
| 二分图判定 | $O(n + m)$ | $O(n + m)$ |

## 七、终极总结与归纳

### 7.1 图的存储与遍历的本质

图的存储与遍历解决的是两个最基本的问题：

1. **存储**：如何高效地表示图的结构（邻居关系）。
2. **遍历**：如何系统地访问图中的每个节点，不遗漏不重复。

所有高级图论算法都是在这两个基础之上构建的。Dijkstra 是"带权 BFS"、Tarjan 是"加了时间戳的 DFS"、拓扑排序是"BFS + 入度"……核心永远是遍历框架。

### 7.2 五大要点

1. **存储方式的选择**：绝大多数情况用 vector 邻接表；$n$ 小且需查边用邻接矩阵；需要边编号用链式前向星。
2. **DFS vs BFS**：DFS 擅长探索连通性和结构（判环、连通分量、DFS 树）；BFS 擅长求最短距离和层次信息。
3. **功能图的特殊性**：每个点出度为 $1$，结构是"链 + 环"。找环和链上回推是核心操作模式。
4. **建图的灵活性**：反向建图、虚拟节点、隐式图、拆点——建图方式决定了问题是否可解。
5. **预处理与离线**：面对大量查询，先遍历一次预处理出连通分量 / 距离信息，再 $O(1)$ 回答。

### 7.3 学习路线图

```

    ├── BFS 基础 ──→ 最短路 ──→ 分层图 ──→ 网络流
    │                   │
    │                   └──→ 差分约束
    │
    ├── 树的遍历 ──→ 树上问题 ──→ LCA ──→ 点分治 ──→ 虚树
```
