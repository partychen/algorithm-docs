## 一、什么是上下界网络流

在普通的网络流问题中，每条边的流量 $f(e)$ 只需满足 $0 \le f(e) \le c(e)$（容量约束）和流量守恒。而在**上下界网络流**中，每条边 $(u, v)$ 同时具有**下界** $l(e)$ 和**上界**（容量） $c(e)$，流量必须满足：

$$
l(e) \le f(e) \le c(e)
$$

也就是说，某些边**至少**要流过 $l(e)$ 单位的流量，不能为零。这使得问题比普通最大流复杂得多——甚至可能**无可行解**。

**应用场景：**
- 工作排班中每人至少工作若干小时、至多工作若干小时
- 供应链中某条路线运输量有最低保障要求
- 竞赛建模中将"恰好"、"至少"类约束转化为下界

## 二、核心思想与前置知识

### 2.1 无源汇上下界可行流

**问题描述：** 给定有向图，每条边有下界 $l$ 和上界 $c$，没有源点和汇点，要求每个节点流量守恒，求一组可行流。

**核心构造方法：**

1. **变量替换：** 对每条边 $(u, v, l, c)$，令实际流量 $f(e) = l(e) + f'(e)$，则 $0 \le f'(e) \le c(e) - l(e)$。在新图中，该边容量变为 $c - l$。

2. **处理多余/不足流量：** 对于节点 $v$，定义：

$$
d(v) = \sum_{(u,v)} l(u,v) - \sum_{(v,w)} l(v,w)
$$

即所有**流入** $v$ 的下界之和减去所有**流出** $v$ 的下界之和。

3. **建立附加源汇：**
   - 新建超级源点 $S'$ 和超级汇点 $T'$。
   - 若 $d(v) > 0$（流入的下界多，需要额外补充流出），加边 $S' \to v$，容量为 $d(v)$。
   - 若 $d(v) < 0$（流出的下界多，需要额外补充流入），加边 $v \to T'$，容量为 $-d(v)$。

4. **判定可行性：** 对新图跑 $S' \to T'$ 的最大流。若最大流等于所有 $S'$ 出边容量之和（即 $\sum_{d(v)>0} d(v)$），则可行流存在。每条原边的实际流量为 $l(e) + f'(e)$。

### 2.2 有源汇上下界可行流

**问题描述：** 在有源点 $s$ 和汇点 $t$ 的网络中，每条边有上下界，求一组可行流。

**方法：** 加一条从 $t$ 到 $s$ 的边，容量为 $+\infty$，下界为 $0$。这样源汇之间的流量通过这条回边"循环"，整个图变成**无源汇问题**，然后按 2.1 的方法求解。

$$
\text{加边 } t \xrightarrow{[0, +\infty]} s
$$

转化后跑无源汇可行流。若可行，则 $t \to s$ 这条回边上的流量就是 $s$ 到 $t$ 的可行流量。

### 2.3 有源汇上下界最大流

**步骤：**

1. 按 2.2 的方法，先求出一组**有源汇可行流**，设可行流量为 $f_0$。
2. 删除 $t \to s$ 的回边。
3. 在**残量网络**上，再从 $s$ 到 $t$ 跑一次普通最大流，设增量为 $\Delta f$。
4. 最终最大流 $= f_0 + \Delta f$。

**原理：** 可行流保证了所有下界约束被满足，残量网络上的增广只在上下界范围内调整流量，不会破坏下界约束。

### 2.4 有源汇上下界最小流

**步骤：**

1. 按 2.2 的方法，先求出一组**有源汇可行流**，设可行流量为 $f_0$。
2. 删除 $t \to s$ 的回边。
3. 在**残量网络**上，从 $t$ 到 $s$ 跑一次普通最大流（注意方向反过来），设退流量为 $\Delta f$。
4. 最终最小流 $= f_0 - \Delta f$。

**原理：** 从 $t$ 到 $s$ 跑最大流相当于在残量网络上"退回"尽可能多的流量，使得 $s \to t$ 的净流量最小。

## 三、分类与变体详解

### 3.1 无源汇可行流

- **输入：** 有向图 $G$，每条边 $(u,v)$ 有下界 $l$ 和上界 $c$。
- **输出：** 满足所有下界约束且节点流量守恒的可行流，或判断无解。
- **算法：** 构建附加图 + 超级源汇 $S', T'$，跑最大流判满流。
- **时间复杂度：** $O(V^2 E)$（Dinic 算法）。

### 3.2 有源汇可行流

- **额外操作：** 加边 $t \to s$（容量 $\infty$，下界 $0$），转化为无源汇问题。
- **可行流量：** $t \to s$ 回边上的流即为 $s \to t$ 流量。
- **时间复杂度：** 同上。

### 3.3 有源汇最大流

- **方法：** 可行流 $f_0$ + 残量网络上 $s \to t$ 最大流 $\Delta f$。
- **答案：** $f_0 + \Delta f$。
- **注意：** 第二次跑最大流时要先把 $t \to s$ 回边删除（或容量置零）。

### 3.4 有源汇最小流

- **方法：** 可行流 $f_0$ - 残量网络上 $t \to s$ 最大流 $\Delta f$。
- **答案：** $f_0 - \Delta f$。
- **注意：** 退流方向是从 $t$ 到 $s$。

### 3.5 上下界最小费用流

- **思路：** 与可行流构造类似，但在附加图中使用**最小费用最大流**（MCMF）算法。
- **建图：**
  - 原边 $(u, v, l, c, w)$：新边容量 $c - l$，费用 $w$。
  - 附加边根据 $d(v)$ 连向 $S'$ 或 $T'$，费用为 $0$。
  - 下界流量的固定费用 $\sum l(e) \cdot w(e)$ 需要额外加入答案。
- **应用：** 当需要在满足上下界约束的前提下最小化总费用时使用。

## 四、4 道经典例题详解

### 题目 1：志愿者招募（洛谷 P3980）

> **题意简述：** 有 $n$ 天的活动，每天至少需要 $a_i$ 个志愿者。有 $m$ 类志愿者，第 $i$ 类可在 $[s_i, t_i]$ 天工作，雇佣费用为 $c_i$。求满足每天需求的最小总费用。

**分析：**

将每天看作一个节点，建立流量网络。第 $i$ 天至少需要 $a_i$ 个志愿者，转化为下界约束。对于每类志愿者 $(s_i, t_i, c_i)$，建一条从 $s_i$ 到 $t_i + 1$ 的边，容量 $\infty$，费用 $c_i$。相邻天之间连边 $i \to i+1$，下界 $a_i$，上界 $\infty$，费用 $0$。最终转化为**上下界最小费用可行流**问题。

- **建模：** 上下界最小费用流
- **时间复杂度：** $O(\text{MCMF})$，取决于 SPFA 增广次数

```cpp
// 建图核心
for (int i = 1; i <= n; i++)
    addedge(i, i + 1, INF, 0, a[i]); // 下界 a[i]
for (int i = 1; i <= m; i++) {
    int s, t, c;
    scanf("%d%d%d", &s, &t, &c);
    addedge(s, t + 1, INF, c, 0); // 志愿者类型边
}
// 处理下界，建附加源汇，跑最小费用最大流
build_bounded_mcmf();
ans += mcmf(S2, T2);
```

**核心启示：** "每天至少 $a_i$ 人" 天然对应下界约束，上下界最小费用流是处理此类"至少"约束 + 最优化的标准模型。

### 题目 2：支线剧情（洛谷 P4043）

> **题意简述：** 一个 DAG 表示游戏的剧情图，从节点 $1$ 出发，每条边有固定时间花费，要求经过所有边至少一次，每次必须从节点 $1$ 重新开始（可在任意终点结束后瞬间回到 $1$）。求最小总时间。

**分析：**

每条边至少经过一次，对应下界为 $1$。而且可以多次经过，上界为 $\infty$。每次从 $1$ 出发到某个节点结束后"免费"回到 $1$，等价于加边 $v \to 1$（每个节点 $v$），容量 $\infty$，费用 $0$。最终转化为**有源汇上下界最小费用流**，源点为 $1$，汇点为附加虚拟节点（或利用回边转无源汇）。

- **建模：** 有源汇上下界最小费用流
- **时间复杂度：** $O(\text{MCMF})$

```cpp
// 建图核心
for (int i = 1; i <= m; i++) {
    int u, v, w;
    scanf("%d%d%d", &u, &v, &w);
    addedge(u, v, INF, w, 1); // 下界1, 上界INF, 费用w
}
for (int v = 2; v <= n; v++)
    addedge(v, 1, INF, 0, 0); // 免费返回起点
// 转化为上下界最小费用流
build_bounded_mcmf();
ans = fixed_cost + mcmf(S2, T2);
```

**核心启示：** "每条边至少走一次" 直接转化为下界 $1$ 的约束。"免费返回起点" 通过加零费用回边实现，是上下界费用流的经典建模技巧。

### 题目 3：80人环游世界（洛谷 P4553）

> **题意简述：** 有 $n$ 个国家，$m$ 个人要进行环游。每个国家恰好被访问 $a_i$ 次。国家之间有有向边，带有旅行费用。求 $m$ 个人访问所有国家恰好规定次数的最小总费用。

**分析：**

"恰好被访问 $a_i$ 次" 意味着流过节点 $i$ 的流量**上下界均为** $a_i$。将每个节点拆为入点 $i_{in}$ 和出点 $i_{out}$，连边 $(i_{in}, i_{out})$，上下界均为 $a_i$，费用 $0$。源点向每个入点连边，出点向汇点连边。国家间的边按题意建立。转化为**上下界最小费用流**。

- **建模：** 上下界最小费用流 + 拆点
- **时间复杂度：** $O(\text{MCMF})$

```cpp
// 拆点：i -> i_in, i+n -> i_out
for (int i = 1; i <= n; i++)
    addedge(i, i + n, a[i], 0, a[i]); // 上下界均为 a[i]
addedge(S, T_orig, m, 0, 0); // 源点到汇点控制总人数
for (int i = 1; i <= n; i++) {
    addedge(S, i, a[i], 0, 0);
    addedge(i + n, T_orig, a[i], 0, 0);
}
// 加入国家间的旅行边，跑上下界最小费用流
build_bounded_mcmf();
ans = fixed_cost + mcmf(S2, T2);
```

**核心启示：** "恰好访问 $a_i$ 次" 等价于上下界相等的约束。拆点是处理节点流量约束的标准技巧——将节点约束转化为边约束。

### 题目 4：清理雪道（洛谷 P4843）

> **题意简述：** 一个 DAG 表示雪道网络，要求从山顶出发（可多次），经过所有边至少一次。每次可以从任意起点出发，在任意终点结束。求最少总次数（即最少需要走几趟）。

**分析：**

每条边至少经过一次，下界为 $1$，上界为 $\infty$。建立超级源点 $S$ 连向所有可能的起点，超级汇点 $T$ 连接所有可能的终点。目标是最小化总流量，即**有源汇上下界最小流**。

- **建模：** 有源汇上下界最小流
- **时间复杂度：** $O(V^2 E)$（Dinic）

```cpp
// 建图核心
for (int u = 1; u <= n; u++)
    for (each edge (u, v))
        addedge(u, v, INF, 1); // 下界1, 上界INF
// S连所有点, 所有点连T
for (int i = 1; i <= n; i++) {
    addedge(S, i, INF, 0);
    addedge(i, T, INF, 0);
}
// 先求可行流 f0，再在残量网络上跑 T->S 最大流退流
build_bounded();
int f0 = feasible_flow();
int delta = dinic(T, S); // 退流
ans = f0 - delta;
```

**核心启示：** "最少走几趟"对应最小流。先通过可行流满足所有下界，再通过退流尽量减少总流量，是上下界最小流的标准套路。

## 五、常见技巧与思路总结

1. **下界转化：** 遇到"至少"、"恰好"约束，立刻想到上下界网络流。"至少 $k$" 对应下界 $k$，"恰好 $k$" 对应上下界均为 $k$。

2. **拆点技巧：** 当约束在**节点**上时（如"节点恰好经过 $a$ 次"），将节点拆为入点和出点，中间连一条带上下界的边。

3. **免费回边：** 当题目允许"免费返回起点"或"从任意位置开始"时，添加费用为 $0$ 的回边将问题转化为标准上下界流。

4. **最小费用流结合：** 当目标是最小化费用且存在下界约束时，使用上下界最小费用流。固定费用（下界自带的费用）要单独累加。

5. **无源汇转化：** 有源汇问题统一通过加回边 $t \to s$（容量 $\infty$）转化为无源汇问题，简化思考。

6. **判无解：** 如果附加图上 $S' \to T'$ 的最大流未满流，则原问题无可行解。

## 六、优化方法 / 解题框架

**解题框架（上下界网络流四步法）：**

1. **识别约束类型：** 判断是"至少"（下界）、"至多"（上界）还是"恰好"（上下界相等）。

2. **确定问题类型：**

| 目标 | 有无源汇 | 问题类型 |
|------|----------|----------|
| 判断可行性 | 无源汇 | 无源汇可行流 |
| 判断可行性 | 有源汇 | 有源汇可行流 |
| 最大化流量 | 有源汇 | 有源汇最大流 |
| 最小化流量 | 有源汇 | 有源汇最小流 |
| 最小化费用 | 任意 | 上下界最小费用流 |

3. **建图与转化：**
   - 对每条边 $(u, v, l, c)$：新容量 $c - l$，计算 $d(v)$。
   - 建超级源汇 $S', T'$，连接附加边。
   - 有源汇时加 $t \to s$ 回边。

4. **求解与还原：**
   - 跑最大流/最小费用流。
   - 判定满流（可行性）。
   - 根据问题类型做后续增广或退流。
   - 实际流量 = 下界 + 残量网络中对应边的流。

**优化技巧：**

- **Dinic 算法：** 上下界网络流中通常使用 Dinic 算法，时间复杂度 $O(V^2 E)$。
- **当前弧优化：** Dinic 多路增广时务必使用当前弧优化，避免重复遍历。
- **缩减节点数：** 如果很多节点的 $d(v) = 0$，与 $S', T'$ 无连边，可以减少不必要的计算。
- **先跑 $s \to t$：** 在有源汇最小流中，有一种优化：先不加 $t \to s$ 回边，先跑 $S' \to T'$，再加回边跑第二次 $S' \to T'$，最后退流。这在某些情况下更快。

## 七、终极总结与归纳

### 7.1 上下界网络流的本质

上下界网络流的本质是**将带有下界约束的问题转化为普通网络流问题**。核心思想是：
- 通过变量替换 $f' = f - l$ 消去下界。
- 通过超级源汇 $S', T'$ 补偿因下界引入的流量不平衡。
- 满流条件保证所有下界约束被满足。

它将"困难的约束满足问题"优雅地归约为"标准的最大流/最小费用流问题"。

### 7.2 核心要点

1. **下界处理公式：** $d(v) = \sum_{\text{in}} l - \sum_{\text{out}} l$，正值连 $S'$，负值连 $T'$。
2. **可行性判定：** 附加图满流 $\Leftrightarrow$ 存在可行流。
3. **最大流 = 可行流 + 增广：** 先保证下界，再在残量网络上贪心增广。
4. **最小流 = 可行流 - 退流：** 先保证下界，再在残量网络上退回多余流量。
5. **费用流结合：** 下界自带的固定费用 $\sum l \cdot w$ 要单独计算，附加边费用为 $0$。

### 7.3 难度分级路线图

| 阶段 | 内容 | 推荐题目 |
|------|------|----------|
| 入门 | 无源汇可行流 | 模板题 |
| 基础 | 有源汇可行流 + 最大流 | 基础练习 |
| 进阶 | 有源汇最小流 | P4843 清理雪道 |
| 高级 | 上下界最小费用流 | P3980 志愿者招募 |
| 综合 | 复杂建模 + 拆点 | P4553 80人环游世界 |

**建议路线：** 无源汇可行流 → 有源汇可行流 → 最大流/最小流 → 最小费用流 → 综合建模

### 7.5 口诀

```
下界约束莫慌张，变量替换减下方。
超级源汇补差额，满流可行是关键。
最大再跑正向流，最小退流反方向。
费用固定单独加，建模转化是核心。
```

以上就是上下界网络流的完整知识体系。掌握了**变量替换 + 超级源汇 + 满流判定**这三个核心工具，再结合**可行流 → 最大流/最小流/最小费用流**的递进关系，就能应对绝大多数上下界网络流问题。
