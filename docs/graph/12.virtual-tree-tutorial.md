## 一、什么是虚树

**虚树（Virtual Tree / Auxiliary Tree）** 是一种从原树中提取少量关键节点、保留它们之间拓扑关系的"压缩树"。

### 直观理解

```
原树（10 个节点）：                 虚树（仅保留关键节点 3, 7, 9 及必要的 LCA）：
        1                                   1
       / \                                 / \
      2   3*                              3   5
     / \   \                                  |
    4   5   6                                 7
   /   / \                                    |
  8   7*  9*                                  9
     /
   10
```

虚树只保留了关键节点 $\{3, 7, 9\}$、它们两两之间的 LCA $\{1, 5\}$，以及根节点 $1$。节点总数从 10 下降到 5，但关键节点之间的祖先-后代关系完全保留。

### 为什么需要虚树

很多树上问题有这样的特征：

- 给定一棵 $n$ 个节点的树，$q$ 次询问
- 每次询问给出 $k_i$ 个关键节点，需要在树上做 DP / 统计
- 保证 $\sum k_i = O(n)$，但 $n$ 很大（$10^5$ 级别）

如果每次询问都在整棵原树上做 DP，总复杂度为 $O(nq)$，无法接受。但如果每次只在关键节点构成的虚树上做 DP，每次复杂度降为 $O(k_i)$，总复杂度为 $O(\sum k_i) = O(n)$，加上预处理后即可通过。

### 与原树的关系

| 性质 | 原树 | 虚树 |
|------|------|------|
| 节点数 | $n$ | $O(k)$（关键点数量级） |
| 边权 | 原始边权 | 压缩路径上的权值和 |
| 祖先关系 | 完整 | 仅保留关键节点间的祖先关系 |
| 适用场景 | 单次询问 | 多次询问，每次涉及少量关键点 |

## 二、核心思想与前置知识

### 2.1 虚树的定义（关键点 + 两两 LCA）

给定原树 $T$ 和一个关键点集合 $S = \{v_1, v_2, \ldots, v_k\}$，虚树 $T'$ 的节点集定义为：

$$V(T') = S \cup \{\text{lca}(v_i, v_j) \mid v_i, v_j \in S\} \cup \{root\}$$

即：关键点集合 $S$，加上 $S$ 中所有点对的 LCA，再加上根节点。

**关键定理**：虚树的节点数 $|V(T')| \le 2k - 1$（其中 $k = |S|$）。

证明思路：将关键点按 DFS 序排序后，所有"新增"的 LCA 恰好是相邻关键点的 LCA。相邻 $k$ 对最多产生 $k - 1$ 个不同的 LCA，因此总节点数不超过 $2k - 1$。

### 2.2 虚树构建算法（DFS 序 + 单调栈）

构建虚树的经典方法利用 **DFS 序** 和一个维护从根到当前节点路径的 **单调栈**。

**算法流程**：

```
输入：关键点集合 S = {v_1, v_2, ..., v_k}
预处理：原树的 DFS 序 dfn[]、LCA（倍增或树剖）

1. 将 S 中的节点按 dfn 排序
2. 初始化栈，压入根节点 root
3. 依次插入排序后的每个关键点 u：
   a. 计算 p = lca(u, stk.top())
   b. 若 p != stk.top()：
      - 不断弹栈，直到 stk[top-1] 的 dfn <= dfn[p]
      - 每次弹栈时，连边 stk[top-1] → stk[top]（或 p → stk[top]）
      - 若 p != stk.top()，将 p 压入栈
   c. 将 u 压入栈
4. 弹出栈中剩余元素，依次连边 stk[top-1] → stk[top]
```

**时间复杂度**：排序 $O(k \log k)$ + 构建 $O(k \log n)$（LCA 查询），总计 $O(k \log n)$。

**C++ 实现框架**：

```cpp
// 预处理：dfn[], dep[], fa[][], lca() 已经求好
// adj2[] 是虚树的邻接表
int stk[MAXN], top;
bool isKey[MAXN];  // 标记是否为关键点

void addEdge(int u, int v) {
    adj2[u].push_back(v);
}

void build(vector<int>& keyNodes) {
    sort(keyNodes.begin(), keyNodes.end(),
         [](int a, int b) { return dfn[a] < dfn[b]; });
    // 去重
    keyNodes.erase(unique(keyNodes.begin(), keyNodes.end()), keyNodes.end());
    stk[top = 1] = root;  // 根入栈
    for (int u : keyNodes) {
        if (u == root) continue;
        int p = lca(u, stk[top]);
        if (p != stk[top]) {
            while (top > 1 && dfn[stk[top - 1]] >= dfn[p])
                addEdge(stk[top - 1], stk[top]), top--;
            if (stk[top] != p)
                addEdge(p, stk[top]), stk[top] = p;
        }
        stk[++top] = u;
    }
    while (top > 1)
        addEdge(stk[top - 1], stk[top]), top--;
}
```

### 2.3 虚树的大小分析（O(k) 级别）

**定理**：对于 $k$ 个关键点构成的虚树，节点数最多为 $2k - 1$。

| 关键点数 $k$ | 虚树节点数上界 | 虚树边数上界 |
|-------------|-------------|-------------|
| 1 | 1（仅根或关键点本身） | 0 或 1 |
| 2 | 3 | 2 |
| $k$ | $2k - 1$ | $2k - 2$ |

这意味着：如果所有询问的关键点总数为 $\sum k_i = O(n)$，则建虚树和在虚树上 DP 的总复杂度为 $O(n \log n)$（瓶颈是 LCA 查询）。

### 2.4 虚树上 DP

虚树构建完毕后，可以在虚树上执行与原树上相同的 DP。需要注意的是：

- **虚树上的边权**：虚树上 $(u, v)$ 这条边对应原树上 $u \to v$ 的一条链，边权通常取原树上这段路径的某种统计值（如权值和、最小值等）
- **清理**：每次询问后需要清空虚树的邻接表（只清理用到的节点，保证复杂度）
- **标记关键点**：DP 时需要区分哪些节点是当前询问的关键点

**虚树上 DP 的通用模式**：

```cpp
void dp(int u) {
    // 初始化 dp[u]
    for (int v : adj2[u]) {
        dp(v);
        // 利用 dp[v] 更新 dp[u]
        // 边权 = 原树上 u 到 v 路径的统计值
    }
    adj2[u].clear();  // 用完即清
}
```

## 三、分类与变体详解

### 3.1 基本虚树构建

最基本的用法是：给定关键点集合，构建虚树，然后在虚树上做一次简单的树形 DP。

**典型场景**：
- 求关键节点到根路径上的某些信息
- 关键节点之间的距离统计
- 断开关键节点与根的最小代价

**要点**：
1. 预处理 DFS 序和 LCA
2. 每次询问按 DFS 序排序关键点
3. 单调栈构建虚树
4. 在虚树上 DFS/DP
5. 清理虚树

### 3.2 虚树 + 树形 DP

**核心思想**：在虚树上做自底向上的 DP，利用虚树边权携带原树压缩路径的信息。

**常见 DP 模式**：
- **最小代价切割**：虚树上每条边的权值取路径最小值，DP 求最小割
- **路径统计**：虚树上每条边的权值取路径长度，统计关键节点间的距离

**关键技巧**：虚树上两点 $(u, v)$ 之间的边权需要快速求出原树上 $u \to v$ 路径的统计值。常用方法：
- 利用 LCA + 深度差求路径长度
- 利用倍增求路径最小/最大值
- 利用前缀和求路径权值和

### 3.3 虚树 + 换根 DP

**核心思想**：有些问题需要考虑每个关键节点"作为根"或"控制区域"时的答案。此时需要在虚树上做换根 DP（二次扫描法）。

**典型场景**：
- 每个关键节点需要知道原树上哪些节点离它最近
- 需要对虚树上每个节点分别求出以它为根的信息

**做法**：
1. 在虚树上做第一遍 DFS（向上收集信息）
2. 做第二遍 DFS（向下分发信息）
3. 结合虚树边对应的压缩路径长度，计算每个节点的控制范围

### 3.4 多组询问的虚树优化

**核心思想**：对于 $q$ 次询问、每次 $k_i$ 个关键点、$\sum k_i = O(n)$ 的问题，每次建虚树 + DP 可将总复杂度从 $O(nq)$ 降到 $O(n \log n)$。

**实现要点**：
- 每次建虚树后立即做 DP，然后清理虚树邻接表
- 清理时只清理虚树中实际出现的节点（不能 `memset` 整个数组）
- 使用标记数组区分关键点和 LCA 辅助点

```cpp
// 高效清理模式
void solve(int u) {
    for (int v : adj2[u]) {
        solve(v);
        // 合并 dp
    }
    adj2[u].clear();        // 清邻接表
    isKey[u] = false;       // 清标记
    // dp[u] = 初始值;      // 清 DP 值
}
```

### 题目 1：P2495 [SDOI2011]消耗战 🟡

**题意**：给定一棵 $n$ 个节点的带边权树，根为 $1$。有 $m$ 次询问，每次给出 $k$ 个关键节点，要求切断一些边使得所有关键节点都不与根节点连通，求最小的切边代价之和。保证 $\sum k_i \le 5 \times 10^5$。

**分析**：

```
经典的虚树入门题。
若在原树上直接 DP，每次询问 O(n)，总 O(nm) 超时。
观察到 sum(k_i) 较小，考虑每次只对关键节点建虚树。
在虚树上做最小割 DP：
  - 对于关键节点 u：必须在 u 到根的路径上切一刀，代价 = 路径最小边权
  - 对于非关键节点 u：取 min(路径最小边权, 所有子树 DP 值之和)
虚树上的边权 = 原树 u 到 v 路径上的最小边权（可用倍增预处理）。
```

> **算法**
> 1. 预处理 DFS 序、LCA（倍增）、路径最小边权 $minW[u][k]$
> 2. 每次询问：按 DFS 序排序关键点，单调栈建虚树
> 3. 虚树上 DP：$dp[u] = \begin{cases} w(u, fa) & u \text{ 是关键点} \\ \min(w(u, fa),\; \sum_{v \in child(u)} dp[v]) & \text{otherwise}\end{cases}$
> **复杂度**：预处理 $O(n \log n)$，每次询问 $O(k_i \log n)$，总 $O(n \log n)$

**C++ 代码（核心）**：

```cpp
// minE(u, v): 原树上 u 到 v 路径最小边权（倍增预处理）
// isKey[u]: 是否为当前询问的关键点
long long dp(int u) {
    long long sum = 0;
    for (int v : adj2[u]) {
        long long w = minE(u, v);
        if (isKey[v]) sum += w;
        else sum += min(w, dp(v));
    }
    adj2[u].clear();
    return sum;
}
```

**核心启示**：虚树将多次小规模询问的总复杂度从 $O(nm)$ 降至 $O(n \log n)$。虚树上 DP 的边权需要根据题意从原树路径中提取（本题取路径最小边权）。

### 题目 2：P3233 [HNOI2014]世界树 🔴

**题意**：给定一棵 $n$ 个节点的树。$q$ 次询问，每次给出 $k$ 个关键节点（"议事处"），树上每个节点归属于离它最近的议事处管辖（距离相同取编号小的）。对每次询问，输出每个议事处管辖的节点数。

**分析**：

```
本题的核心难度在于：需要求出原树上每个节点被哪个关键点管辖，
而不仅仅是虚树上的节点。
思路：
1. 对关键点建虚树
2. 在虚树上做两遍 DP（换根DP），确定虚树上每个节点最近的议事处
3. 对虚树的每条边 (u, v)（对应原树上一段链），
   找到链上的"分界点"——链上某个节点到 u 最近议事处更近 vs 到 v 最近议事处更近
4. 利用原树子树大小信息，计算每个议事处管辖的节点数
```

> **算法**
> 1. 预处理 DFS 序、LCA、子树大小 $sz[]$
> 2. 建虚树，虚树上两遍 DFS（换根 DP）求每个虚树节点的最近议事处
> 3. 对虚树每条边 $(u, v)$，二分找链上分界点，将链上节点分配给 $u$ 或 $v$ 的最近议事处
> 4. 未在虚树中出现的子树整体归属于其在虚树中最近的祖先的议事处
> **复杂度**：$O(n \log n + \sum k_i \log n)$

**C++ 代码（核心）**：

```cpp
// bel[u]: 虚树节点 u 的最近议事处
// 两遍 DFS 求 bel[]
void dfs1(int u) {  // 自底向上
    if (isKey[u]) bel[u] = u;
    for (int v : adj2[u]) {
        dfs1(v);
        if (bel[v] && (!bel[u] || closer(bel[v], u, bel[u])))
            bel[u] = bel[v];
    }
}
void dfs2(int u) {  // 自顶向下
    for (int v : adj2[u]) {
        if (bel[u] && (!bel[v] || closer(bel[u], v, bel[v])))
            bel[v] = bel[u];
        dfs2(v);
    }
}
// 对虚树边 (u,v) 二分分界点，分配管辖节点数
```

**核心启示**：虚树 + 换根 DP 可以处理"每个节点归属最近关键点"类问题。关键技巧是在虚树的压缩边上二分找分界点，将链上节点正确分配。

### 题目 3：P5360 [SDOI2019]世界地图 🟣

**题意**：给定一个 $n \times m$ 的网格图，每次询问给出列的范围 $[l, r]$，求该列范围内所有节点构成的最小生成树的边权和。$n$ 较小（$\le 200$），$m$ 较大（$\le 10^5$）。

**分析**：

```
本题将虚树思想与最小生成树相结合。
关键观察：
1. 网格图按列划分，每列只有 n 个节点
2. 预处理前缀 MST（第 1~j 列）和后缀 MST（第 j~m 列）
3. 每个前缀/后缀 MST 可以只保留"边界列"上的节点之间的连通信息
   —— 这就是虚树的思想：只保留关键节点（边界列上的点）
4. 回答询问 [l, r] 时，合并后缀 MST[l] 和前缀 MST[r] 即可
合并两个 MST 的方法：
- 两个 MST 各自保留了边界上 n 个关键点的连通信息（用 Kruskal 重构树 / 虚树维护）
- 加入两个边界之间的连边，再做一次 MST 合并
```

> **算法**
> 1. 预处理：从左到右依次加入每一列，维护当前 MST 中左边界节点的连通信息（虚树压缩）
> 2. 同理从右到左预处理后缀
> 3. 查询 $[l, r]$：取后缀左边界信息 + 前缀右边界信息，合并得到答案
> **复杂度**：预处理 $O(nm \cdot \alpha(n))$，单次查询 $O(n^2 \cdot \alpha(n))$

**C++ 代码（核心）**：

```cpp
// 合并两个边界 MST 信息
// edges: 两个边界之间的连边 + 两个 MST 的虚树边
// 用 Kruskal 合并
long long merge(vector<Edge>& e1, vector<Edge>& e2,
                vector<Edge>& cross) {
    vector<Edge> all;
    all.insert(all.end(), e1.begin(), e1.end());
    all.insert(all.end(), e2.begin(), e2.end());
    all.insert(all.end(), cross.begin(), cross.end());
    sort(all.begin(), all.end());
    long long res = 0;
    for (auto& [w, u, v] : all)
        if (find(u) != find(v))
            unite(u, v), res += w;
    return res;
}
```

**核心启示**：虚树的思想不局限于"树上 DP"，也可以用于压缩图的连通结构。本题利用虚树压缩 MST 中非关键点的信息，实现了高效的区间 MST 查询。

### 题目 4：P5439 【XR-2】永恒 🟣

**题意**：给定两棵 $n$ 个节点的有根树 $T_1, T_2$。对于 $T_1$ 上的一个点集 $S$ 和 $T_2$ 上对应的点集 $S$，定义权值为 $\sum_{u \in S} \sum_{v \in S} \text{dep}_{T_1}(\text{lca}_{T_1}(u,v)) \times \text{dep}_{T_2}(\text{lca}_{T_2}(u,v))$。求某种统计量。

**分析**：

```
本题涉及两棵树上的 LCA 深度乘积统计。
核心思路：
1. 枚举 T1 上的节点 x，考虑 x 作为 T1 中某些点对的 LCA 的贡献
2. 对于 T1 上以 x 为 LCA 的点对 (u, v)，它们在 T2 上的贡献与 T2 上的 lca(u,v) 有关
3. 利用虚树：将 x 的子树中的关键点在 T2 上建虚树，统计贡献
4. 通过 DFS 序 + 子树合并的方式枚举 T1 上的 LCA

关键技巧：
- 在 T1 上 DFS，维护当前子树内节点在 T2 上的虚树
- 合并子树时，小集合插入大集合（启发式合并），每次插入一个点重新维护虚树
- 利用虚树上的 DFS 统计深度乘积和
```

> **算法**
> 1. 预处理两棵树的 DFS 序、LCA、深度
> 2. 在 $T_1$ 上 DFS，对每个节点维护其子树内的节点在 $T_2$ 上的虚树
> 3. 子树合并时采用启发式合并（小集合插入大集合），动态维护虚树
> 4. 每次合并时计算新增的贡献
> **复杂度**：$O(n \log^2 n)$

**C++ 代码（核心）**：

```cpp
// 在 T1 上 DFS，维护子树在 T2 上的虚树
void dfs_T1(int u) {
    int heavy = -1;  // 最大子树
    for (int v : ch1[u]) {
        dfs_T1(v);
        if (heavy == -1 || sz[v] > sz[heavy]) heavy = v;
    }
    // 继承最大子树的虚树，将其他子树的节点逐个插入
    for (int v : ch1[u]) {
        if (v == heavy) continue;
        for (int x : subtreeNodes[v])
            ans += dep1[u] * insertAndCalc(x);
            // insertAndCalc: 将 x 插入 T2 虚树，返回新增的 T2 贡献
    }
}
```

**核心启示**：两棵树上的统计问题可以通过"一棵树上 DFS + 另一棵树上建虚树"的方式解决。启发式合并保证了总插入次数为 $O(n \log n)$。

### 题目 5：虚树 + 树上距离统计 🟠

**题意（抽象模型）**：给定一棵 $n$ 个节点的带权树，$q$ 次询问，每次给出 $k$ 个关键节点，求关键节点两两之间的距离之和。$\sum k_i \le 5 \times 10^5$。

**分析**：

```
朴素做法：每次 O(k^2 log n) 枚举所有点对求距离，太慢。
虚树优化：
1. 对关键点建虚树
2. 在虚树上统计：对每条边 (u,v)（边权 w），
   计算有多少对关键点的路径经过这条边
3. 边 (u,v) 被经过的次数 = v 子树内关键点数 * v 子树外关键点数
4. 答案 = sum(w * cnt_in * cnt_out)
```

> **算法**
> 1. 建虚树
> 2. DFS 求每个虚树节点子树内包含的关键点数 $cnt[u]$
> 3. 对每条虚树边 $(u, v)$，贡献 = $w(u,v) \times cnt[v] \times (k - cnt[v])$
> **复杂度**：$O(k \log n)$

**C++ 代码（核心）**：

```cpp
// cnt[u]: u 子树内的关键点数
long long ans;
void dp(int u, int k) {
    cnt[u] = isKey[u] ? 1 : 0;
    for (int v : adj2[u]) {
        dp(v, k);
        long long w = dep[v] - dep[u]; // 边权 = 深度差（带权树用加权深度）
        ans += w * cnt[v] * (k - cnt[v]);
        cnt[u] += cnt[v];
    }
    adj2[u].clear();
}
```

**核心启示**：树上路径统计问题可以转化为"每条边被多少对点的路径经过"，虚树上同样适用这一经典技巧。

### 题目 6：虚树 + 子树染色统计 🟠

**题意（抽象模型）**：给定一棵 $n$ 个节点的树，$q$ 次操作，每次将 $k$ 个关键节点染色，求染色后每个子树内的染色节点数。$\sum k_i \le 5 \times 10^5$。

**分析**：

```
朴素做法：每次在原树上打标记后 DFS 统计，O(n) 每次。
虚树优化：
1. 只对关键点建虚树
2. 虚树上 DFS，统计每个虚树节点子树内的关键点数
3. 虚树上非关键点的子树中，不在虚树中的部分没有任何染色节点
4. 如果需要还原到原树上，利用虚树的压缩边与原树子树大小即可
```

> **算法**
> 1. 建虚树
> 2. DFS 统计每个虚树节点子树内的关键点数
> 3. 虚树上父子关系对应原树上祖先-后代关系，利用 DFS 序快速映射
> **复杂度**：$O(k \log n)$

**C++ 代码（核心）**：

```cpp
// 虚树上统计子树内关键点数
int dfs(int u) {
    int res = isKey[u] ? 1 : 0;
    for (int v : adj2[u]) {
        res += dfs(v);
    }
    keyCount[u] = res;
    adj2[u].clear();
    return res;
}
```

**核心启示**：虚树天然保留了子树包含关系，子树内关键点数量的统计可以直接在虚树上完成，避免遍历整棵原树。

## 五、常见技巧与思路总结

### 技巧 1：虚树建完后及时清理

```
每次询问结束后，必须清理虚树的邻接表和标记数组。
关键：只清理虚树中实际出现的节点，不要 memset 全部数组。
```

### 技巧 2：虚树边权的快速计算

```
虚树上 (u, v) 对应原树上 u 到 v 的一条链。
- 路径长度：dep[v] - dep[u]
- 路径最小边权：倍增维护 minW[u][k]，查询时沿 LCA 倍增路径取 min
- 路径权值和：前缀和 sum[v] - sum[u]
```

### 技巧 3：虚树 + 启发式合并

```
当问题涉及两棵树时（如题目 4），可以在一棵树上 DFS 枚举，
另一棵树上维护虚树。合并子树信息时采用启发式合并（小并入大），
保证总插入次数 O(n log n)。
```

### 技巧 4：关键点按 DFS 序排序后相邻点的 LCA

```
将 k 个关键点按 DFS 序排序为 a_1, a_2, ..., a_k。
则所有"必要的 LCA"恰好是 {lca(a_i, a_{i+1}) | 1 <= i < k}。
这是虚树节点集 O(k) 大小的关键保证。
```

### 技巧 5：虚树上的贡献拆分

```
对于路径统计类问题，经典技巧是将答案拆分到每条边上：
ans = sum_{边 (u,v)} w(u,v) * f(u,v)
其中 f(u,v) 是这条边被多少对关键点的路径经过。
在虚树上，f(u,v) = cnt[v] * (k - cnt[v])。
```

## 六、优化方法 / 解题框架

### 虚树解题通用框架

```
步骤 1：预处理原树
  - DFS 求 dfn[]、dep[]、sz[]（子树大小）
  - 预处理 LCA（倍增 / 树链剖分）
  - 根据题意预处理边权相关信息（路径最小值、前缀和等）

步骤 2：读入询问，构建虚树
  - 关键点按 dfn 排序
  - 单调栈建虚树
  - 标记关键点

步骤 3：虚树上 DP
  - 根据题意设计 DP 转移
  - 注意虚树边权 ≠ 原树边权
  - 区分关键点和 LCA 辅助点

步骤 4：输出答案 + 清理虚树
  - 只清理虚树相关数组（不要 memset）
```

### 虚树构建的完整模板

```cpp
// === 预处理部分 ===
int dfn[MAXN], dep[MAXN], fa[MAXN][20], timer;
int sz[MAXN]; // 子树大小
vector<int> adj[MAXN];  // 原树
vector<int> adj2[MAXN]; // 虚树

void dfs_init(int u, int f) {
    dfn[u] = ++timer; dep[u] = dep[f] + 1;
    fa[u][0] = f; sz[u] = 1;
    for (int k = 1; k < 20; k++)
        fa[u][k] = fa[fa[u][k-1]][k-1];
    for (int v : adj[u]) {
        if (v == f) continue;
        dfs_init(v, u);
        sz[u] += sz[v];
    }
}

int lca(int u, int v) {
    if (dep[u] < dep[v]) swap(u, v);
    for (int k = 19; k >= 0; k--)
        if (dep[fa[u][k]] >= dep[v]) u = fa[u][k];
    if (u == v) return u;
    for (int k = 19; k >= 0; k--)
        if (fa[u][k] != fa[v][k]) u = fa[u][k], v = fa[v][k];
    return fa[u][0];
}

bool isAnc(int u, int v) { // u 是否是 v 的祖先
    return dfn[u] <= dfn[v] && dfn[v] < dfn[u] + sz[u];
}

// === 虚树构建 ===
int stk[MAXN], top;

void build(vector<int>& key) {
    sort(key.begin(), key.end(),
         [](int a, int b) { return dfn[a] < dfn[b]; });
    key.erase(unique(key.begin(), key.end()), key.end());

    stk[top = 1] = 1;
    for (int u : key) {
        if (u == 1) continue;
        int p = lca(u, stk[top]);
        if (p != stk[top]) {
            while (top > 1 && dfn[stk[top-1]] >= dfn[p])
                adj2[stk[top-1]].push_back(stk[top]), top--;
            if (stk[top] != p)
                adj2[p].push_back(stk[top]), stk[top] = p;
        }
        stk[++top] = u;
    }
    while (top > 1)
        adj2[stk[top-1]].push_back(stk[top]), top--;
}
```

### 复杂度分析速查表

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 预处理 DFS 序 + LCA | $O(n \log n)$ | 一次性 |
| 建虚树（单次） | $O(k \log n)$ | 排序 + LCA 查询 |
| 虚树上 DP（单次） | $O(k)$ | |
| 所有询问总计 | $O(n \log n + \sum k_i \log n)$ | $\sum k_i \le O(n)$ 时为 $O(n \log n)$ |

## 七、终极总结与归纳

### 虚树的本质

虚树是 **"只保留有用节点"** 思想在树结构上的体现。当树上问题满足以下条件时，虚树几乎是必选方案：

1. **多次询问**，每次涉及少量关键点
2. **$\sum k_i$ 有保证**（通常 $O(n)$ 级别）
3. 答案只与关键点之间的拓扑关系有关

### 核心知识清单

| 知识点 | 要点 |
|--------|------|
| 虚树节点集 | 关键点 + 两两 LCA + 根，大小 $\le 2k-1$ |
| 构建方法 | DFS 序排序 + 单调栈，$O(k \log n)$ |
| 虚树边权 | 对应原树上压缩路径的统计值 |
| 虚树上 DP | 与原树 DP 形式相同，需区分关键点与辅助点 |
| 清理 | 只清理虚树中出现的节点，保证总复杂度 |

### 难度递进路线

```
入门：理解虚树定义 → 手动构建小规模虚树
       ↓
基础：P2495 消耗战（虚树 + 最小割 DP）
       ↓
进阶：P3233 世界树（虚树 + 换根 DP + 边上二分）
       ↓

     P5439 永恒（双树 + 虚树 + 启发式合并）
```

### 解题思维导图

```
题目特征分析
├── 多次询问、每次少量关键点、sum(k_i) 有界 → 虚树
├── 树上 DP 类
│   ├── 最小割 / 最小代价 → 虚树 + 最小割 DP（消耗战）
│   ├── 最近归属 → 虚树 + 换根 DP（世界树）
│   └── 路径统计 → 虚树 + 边贡献拆分
├── 图结构压缩类

└── 双树类
    └── 一棵树枚举 + 另一棵树建虚树 + 启发式合并（永恒）
```

掌握虚树，本质上是掌握了 **"压缩无关信息、聚焦关键节点"** 的算法设计范式。这一思想在竞赛中应用广泛，远不止于树上 DP 一隅。
