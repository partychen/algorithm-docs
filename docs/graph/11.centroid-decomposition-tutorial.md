## 一、什么是点分治

**点分治** 是一种高效统计树上路径信息的分治算法。它的核心思路是：对于一棵树，选择一个合适的"分治中心"将问题拆成若干个规模更小的子问题，递归求解。

### 解决什么问题

点分治通常用于处理以下类型的树上路径统计问题：

- 树上是否存在长度恰好为 $k$ 的路径？
- 树上满足某种条件的路径有多少条？
- 树上路径的某种最优值是什么？

这类问题的共性是：需要考虑 **所有** $O(n^2)$ 条路径，而暴力枚举显然太慢。

### 与普通分治的对比

| 维度 | 普通分治（序列） | 点分治（树） |
|------|----------------|-------------|
| 分治对象 | 数组 / 区间 | 树 |
| 分治中心 | 区间中点 | 树的重心 |
| 子问题划分 | 左半段 / 右半段 | 删除重心后的各子树 |
| 子问题规模 | 每次恰好减半 | 每个子树大小 $\le n/2$ |
| 跨中心贡献 | 合并左右两半的信息 | 统计经过重心的路径 |
| 时间复杂度 | 通常 $O(n \log n)$ | 通常 $O(n \log^2 n)$ 或 $O(n \log n)$ |

**关键区别**：序列的中点天然将区间等分，但树的形态各异，直接选根可能导致退化（如链）。点分治选择 **树的重心** 作为分治中心，保证递归深度为 $O(\log n)$。

## 二、核心思想与前置知识

### 2.1 树的重心

**定义**：树的重心是这样一个节点 $c$，删除 $c$ 后，剩余各连通分量的大小的最大值最小。

**性质**：
- 对于一棵大小为 $n$ 的树，其重心 $c$ 满足：删除 $c$ 后，每个连通块大小 $\le n/2$
- 重心最多有 2 个，且若有 2 个则它们相邻
- 所有节点到重心的距离之和最小

**求法**：DFS 统计每个子树大小 $sz[v]$，一个节点 $u$ 删除后最大连通块大小为 $\max(n - sz[u], \max_{v \in \text{son}(u)} sz[v])$，取该值最小的节点即为重心。

```cpp
// 求以 rt 为根的树（大小 tot）的重心
int centroid, minMaxSub;
void getCentroid(int u, int fa, int tot) {
    sz[u] = 1;
    int maxSub = 0;
    for (auto [v, w] : adj[u]) {
        if (v == fa || del[v]) continue;
        getCentroid(v, u, tot);
        sz[u] += sz[v];
        maxSub = max(maxSub, sz[v]);
    }
    maxSub = max(maxSub, tot - sz[u]);
    if (maxSub < minMaxSub) {
        minMaxSub = maxSub;
        centroid = u;
    }
}
```

### 2.2 点分治流程

点分治的核心流程分为四步：

```
1. 找重心：在当前连通块中求出重心 c
2. 统计贡献：统计所有 经过 c 的路径 的答案
3. 删重心：标记 c 已删除
4. 递归子树：对删除 c 后的每个连通块递归执行上述过程
```

**为什么选重心**？选重心保证每次递归后子树大小不超过原来的一半，因此递归深度为 $O(\log n)$。每层递归遍历所有节点的总时间为 $O(n)$，所以总复杂度为 $O(n \log n)$（不含统计贡献部分的额外开销）。

```
              原始树
                |
          [找到重心 c]
           /    |    \
     子树1    子树2   子树3    ← 每棵大小 ≤ n/2
      |        |       |
  [找重心]  [找重心] [找重心]
    / \       |      / \
  ...  ...   ...   ...  ...    ← 递归深度 O(log n)
```

**整体框架代码**：

```cpp
void solve(int u, int tot) {
    minMaxSub = tot;
    getCentroid(u, 0, tot);
    int c = centroid;
    del[c] = true;

    // 统计所有经过 c 的路径贡献
    calc(c);

    for (auto [v, w] : adj[c]) {
        if (del[v]) continue;
        int subSz = sz[v] > sz[c] ? tot - sz[c] : sz[v];
        solve(v, subSz);
    }
}
```

### 2.3 容斥原理在点分治中的应用

统计"经过重心 $c$ 的路径"时，一种常用思路是 **容斥**：

1. 先统计从 $c$ 出发到其所有子树节点的信息
2. 将来自 **不同子树** 的两端配对，得到经过 $c$ 的路径信息

具体做法有两种：

**方法一：逐子树合并（推荐）**

依次处理重心 $c$ 的每棵子树。处理第 $i$ 棵子树时，先用已有的前 $i-1$ 棵子树的信息和第 $i$ 棵子树配对统计答案，然后将第 $i$ 棵子树的信息加入全局。最后清空。

```
处理子树 1：
  → 统计子树 1 与 c 本身的配对 → 加入子树 1 的信息
处理子树 2：
  → 统计子树 2 与前面（子树 1）的配对 → 加入子树 2 的信息
处理子树 3：
  → 统计子树 3 与前面（子树 1+2）的配对 → 加入子树 3 的信息
...
清空全部信息
```

这样天然避免了同一子树内部路径被错误统计的问题。

**方法二：全局统计 - 子树内减去（容斥）**

先将从 $c$ 出发到所有子树节点的信息全部收集起来，统计所有配对的贡献 $S_{\text{all}}$，再对每棵子树单独统计内部配对的贡献 $S_i$，最终答案为 $S_{\text{all}} - \sum S_i$。

$$\text{经过 c 的路径贡献} = S_{\text{all}} - \sum_{i} S_i$$

两种方法各有优劣。逐子树合并的写法更直观且不需要减法操作，而容斥法在某些需要排序后双指针的场景下更方便。

### 2.4 点分树（动态点分治）

**点分树** 是在点分治过程中自然形成的一棵树：每次找到的重心作为节点，递归调用时上一层重心是下一层重心的父节点。

```
点分树的建立过程：
原始树 → 找到重心 c0                     c0 (根)
          ↓ 递归子树1 → 找到重心 c1       / \
          ↓ 递归子树2 → 找到重心 c2     c1  c2
          ↓ 递归子树3 → 找到重心 c3     /  |  \
          ...                         ... ... ...
```

**点分树的性质**：
- 树高为 $O(\log n)$——因为每次重心的子树大小不超过原来的一半
- 对于原树上任意两点 $u, v$，它们的 LCA（在点分树上）一定在 $u \to v$ 的路径上
- 点分树的边不对应原树的边，但节点集合相同

**核心应用**：将点分治"离线化"变为"在线化"。

当问题需要 **带修改** 或 **在线查询** 时，预先建出点分树，然后对每个节点维护一些辅助数据结构。查询时沿着点分树从节点向根跳，利用每层的数据结构回答贡献。

```cpp
// 建点分树：在点分治过程中记录父子关系
int cdFa[MAXN]; // 点分树上的父亲
void buildCDTree(int u, int tot, int fa) {
    minMaxSub = tot;
    getCentroid(u, 0, tot);
    int c = centroid;
    cdFa[c] = fa;
    del[c] = true;
    for (auto [v, w] : adj[c]) {
        if (del[v]) continue;
        int subSz = sz[v] > sz[c] ? tot - sz[c] : sz[v];
        buildCDTree(v, subSz, c);
    }
}
```

**查询模式**：对于查询点 $u$，从 $u$ 出发沿点分树向根逐级跳跃，在每一层的重心处累计贡献。由于点分树高度为 $O(\log n)$，每次查询最多经过 $O(\log n)$ 层。

## 三、分类与变体详解

### 3.1 路径计数/判定

**典型问题**：统计树上满足某种条件的路径数目（如长度为 $k$、长度是 3 的倍数等），或判断树上是否存在满足条件的路径。

**做法**：
1. 以重心 $c$ 为中心，DFS 收集 $c$ 到各子树节点的距离信息
2. 用排序 + 双指针 / 桶 / 数组计数等方式，统计经过 $c$ 的满足条件的路径数
3. 容斥或逐子树合并以排除同一子树内部的路径

**关键代码模式**：
```cpp
void calc(int c) {
    // 清空桶 / 初始化
    // 逐子树合并：
    for (auto [v, w] : adj[c]) {
        if (del[v]) continue;
        // DFS 收集子树 v 中所有点到 c 的距离 dist[]
        // 与已有信息配对统计答案
        // 将子树 v 的信息加入全局桶
    }
    // 清空桶
}
```

### 3.2 路径最值

**典型问题**：求树上路径权值和的最大/最小值（可能附带约束），或求满足约束条件下的最优路径。

**做法**：
- 收集经过重心的路径信息后，用单调队列 / 二分查找 / 线段树等数据结构求最值
- 有时需要二分答案 + 点分治判定

**关键点**：最值问题往往需要枚举一端，在另一端的集合中高效查询最优配对，因此常与二分、排序、数据结构结合。

### 3.3 点分树（在线查询）

**典型问题**：多次在线查询树上满足条件的信息，可能带修改。

**做法**：
1. 预处理建出点分树，每个节点维护辅助数据结构（如排序数组、线段树、vector）
2. 查询时从查询节点沿点分树往根跳，在每层累计贡献
3. 修改时也沿点分树往根跳，更新每层的辅助数据结构

**典型空间开销**：每个节点在点分树中的深度为 $O(\log n)$，因此每个节点的信息被存储 $O(\log n)$ 次，总空间 $O(n \log n)$。

### 3.4 点分治 + 数据结构

**典型问题**：统计路径信息时，单纯的排序或桶无法满足需求，需要更强的数据结构辅助。

**常见组合**：
| 数据结构 | 应用场景 |
|---------|---------|
| 树状数组 | 路径权值范围统计 |
| 线段树 | 路径权值区间查询 |
| 单调队列 | 路径长度有上下界时求最值 |
| Trie | 路径上字符串匹配 |
| 哈希表/桶 | 路径长度恰好为某值的计数 |

### 题目 1：P3806 【模板】点分治1 🟢

**题意**：给定一棵 $n$ 个节点的带权树，有 $m$ 次询问，每次询问树上是否存在路径长度恰好为 $k$ 的路径。

**分析**：
- 这是点分治的模板题
- 对于重心 $c$，收集其到所有子树节点的距离，检查是否存在两个来自不同子树的距离之和恰好为 $k$
- 用桶（布尔数组）记录已处理子树中出现过的距离值，逐子树合并

> **算法**
> 1. 找到当前连通块的重心 $c$
> 2. 逐子树 DFS，收集子树中各点到 $c$ 的距离
> 3. 对每棵子树中的距离 $d$，检查桶中是否存在 $k - d$
> 4. 将当前子树的距离加入桶，处理完后清空桶
> 5. 递归处理删除 $c$ 后的各子树
> **复杂度**：$O(n \log n \cdot m)$

**C++ 代码（核心）**：
```cpp
bool exist[10000001]; // 桶
vector<int> cur, all; // 当前子树距离、所有距离（用于清空）
void getDist(int u, int fa, int d) {
    cur.push_back(d);
    for (auto [v, w] : adj[u])
        if (v != fa && !del[v]) getDist(v, u, d + w);
}
void calc(int c) {
    exist[0] = true; all.push_back(0);
    for (auto [v, w] : adj[c]) {
        if (del[v]) continue;
        cur.clear(); getDist(v, c, w);
        for (int d : cur)
            for (int i = 0; i < m; i++)
                if (q[i] >= d) ans[i] |= exist[q[i] - d];
        for (int d : cur) exist[d] = true, all.push_back(d);
    }
    for (int d : all) exist[d] = false;
    all.clear();
}
```

**核心启示**：点分治的基本套路——找重心、逐子树统计经过重心的路径、递归。桶（布尔数组）是处理"是否存在"类问题的利器。

### 题目 2：P2634 [国家集训队]聪聪可可 🟢

**题意**：给定一棵 $n$ 个节点的带权树，求满足路径长度为 3 的倍数的点对 $(u, v)$ 的数量占总点对数的比值（最简分数形式）。

**分析**：
- 路径长度模 3 余 0 的点对数量
- 只需要关注距离模 3 的余数（0, 1, 2），用桶 $cnt[0..2]$ 记录
- 余数为 0 的配对余数为 0 的，余数为 1 的配对余数为 2 的

> **算法**
> 1. 点分治框架，对每个重心 $c$
> 2. 收集子树中各点到 $c$ 的距离模 3 的余数
> 3. 逐子树合并：$cnt[r_1] \times$ 当前子树中余数为 $(3 - r_1) \bmod 3$ 的个数
> 4. 统计总的合法点对数，最后除以 $n^2$ 并化简
> **复杂度**：$O(n \log n)$

**C++ 代码（核心）**：
```cpp
int cnt[3]; // 已处理子树中，到重心距离 mod 3 的计数
long long ans = 0;
void calc(int c) {
    cnt[0] = 1; // 重心本身距离为 0
    for (auto [v, w] : adj[c]) {
        if (del[v]) continue;
        cur.clear(); getDist(v, c, w);
        for (int d : cur) ans += cnt[(3 - d % 3) % 3];
        for (int d : cur) cnt[d % 3]++;
    }
    cnt[0] = cnt[1] = cnt[2] = 0;
}
```

**核心启示**：当路径条件只涉及模某个数的余数时，桶的大小仅为常数，统计效率极高。逐子树合并天然保证配对来自不同子树。

### 题目 3：P2664 树上游戏 🟡

**题意**：给定一棵 $n$ 个节点的树，每个节点有一个颜色 $c_i$。对每个节点 $u$，求 $s(u) = \sum_{v=1}^{n} \text{col}(u, v)$，其中 $\text{col}(u, v)$ 表示 $u$ 到 $v$ 路径上不同颜色的数量。

**分析**：
- 直接枚举每对点计算颜色数是 $O(n^2)$ 的，需要更巧妙的方法
- 点分治框架下，考虑经过重心 $c$ 的路径对各节点 $s$ 值的贡献
- 关键转化：将"每条路径的颜色数"转化为"每种颜色对多少条路径有贡献"
- 一种颜色 $col$ 对路径 $u \to v$（经过 $c$）有贡献，当且仅当 $col$ 出现在 $u \to c$ 或 $c \to v$ 上
- 需要仔细分析每种颜色在经过重心路径上的贡献，利用"总贡献 - 不含该颜色的路径数"来计算

> **算法**
> 1. 点分治，以 $c$ 为重心
> 2. DFS 时维护 $c \to u$ 路径上的颜色集合
> 3. 对于颜色 $col$，设子树中首次出现 $col$ 的子树大小为 $s$，则 $col$ 对另一端所有点的贡献为子树外的点数
> 4. 利用"总贡献"减去"同子树内部贡献"来完成容斥
> **复杂度**：$O(n \log n)$

**C++ 代码（核心）**：
```cpp
// sum = 从 c 出发到所有点路径颜色数之和
// 对于子树 v 中的点 u，s[u] += sum - sub_sum[v] + new_colors(c->u) * (tot - sub_size[v])
//   其中 new_colors(c->u) 是 c->u 路径上不在 c->v 之外的新颜色数
long long sum; int col[MAXN], vis[MAXN], tot;
void addColor(int c, int delta) {
    // 第一次出现该颜色时，对 sum 贡献 delta * (关联子树大小)
    if (vis[c]++ == 0) sum += delta;
}
void calcSub(int u, int fa, int cnt, long long base) {
    if (!vis[col[u]]) cnt++; vis[col[u]]++;
    ans[u] += base + (long long)cnt * (tot - subSz);
    for (auto [v, w] : adj[u])
        if (v != fa && !del[v]) calcSub(v, u, cnt, base);
    vis[col[u]]--;
}
```

**核心启示**：颜色计数类问题的经典转化——将"路径上的颜色种数"拆成"每种颜色是否出现"再求和。点分治中要特别注意颜色在不同子树间的贡献计算。

### 题目 4：P3714 [BJOI2017]树的难题 🟠

**题意**：给定一棵 $n$ 个节点的树，每条边有一个颜色和权值。经过一段连续相同颜色的边时，收益为这些边权值之和。求长度（边数）在 $[L, R]$ 范围内的路径中，收益的最大值。

**分析**：
- 点分治框架下，要统计经过重心的路径中边数在 $[L, R]$ 内且收益最大的路径
- 难点在于"连续相同颜色"的收益定义——经过重心 $c$ 时，从 $c$ 两侧拼接路径可能导致相邻边颜色相同，需要特殊处理
- 将从重心出发的链按"第一条边颜色"分组，同色组内的链拼接不产生额外合并，异色组间的链拼接需要独立计算
- 使用线段树维护不同链长对应的最大收益，按颜色分组用单调队列 / 线段树查询

> **算法**
> 1. 点分治，对重心 $c$ 的各子树按第一条边的颜色排序
> 2. 同色子树内部合并时，两条链拼接后在 $c$ 处的颜色相同，收益直接相加
> 3. 不同颜色子树间合并时，需要在线段树上查询长度互补区间的最大值
> 4. 用线段树维护已处理链的"链长 → 最大收益"映射
> **复杂度**：$O(n \log^2 n)$

**C++ 代码（核心）**：
```cpp
// seg: 线段树，维护链长 -> 最大收益
// chains: 从 c 出发的链，按第一条边颜色分组
void calc(int c) {
    // 将子树按第一条边颜色排序分组
    for (每个颜色组 group) {
        // 组内链之间互相匹配（同色拼接）
        for (链 ch : group)
            ans = max(ans, ch.val + seg.query(L - ch.len, R - ch.len));
        // 组内链加入临时线段树 tmpSeg
        // 组结束后，tmpSeg 合并到 seg
    }
    seg.clear();
}
```

**核心启示**：当路径拼接时存在"边界效应"（如相邻边颜色相同导致合并），需要对子树按影响因素（颜色）分组，组内和组间分别处理。线段树是处理区间最值查询的有力工具。

### 题目 5：P4149 [IOI2011]Race 🟡

**题意**：给定一棵 $n$ 个节点的带权树，求长度（边权之和）恰好为 $K$ 的路径中，经过边数最少的那条路径的边数。

**分析**：
- 类似模板题 P3806，但这里不是判定存在性，而是求最小边数
- 用桶 $best[d]$ 记录距离为 $d$ 的路径中最少的边数
- 逐子树合并：对当前子树中的距离 $d$、边数 $e$，查询 $best[K - d]$，更新 $ans = \min(ans, e + best[K - d])$

> **算法**
> 1. 点分治框架，维护桶 $best[0..K]$，$best[d]$ = 已处理子树中距离为 $d$ 的最少边数
> 2. 初始 $best[0] = 0$
> 3. 逐子树合并：DFS 子树，对每个点检查 $best[K - d]$ 并更新答案，然后更新桶
> 4. 用 vector 记录修改过的桶位置，处理完一个重心后逐个还原
> **复杂度**：$O(n \log n)$

**C++ 代码（核心）**：
```cpp
int best[MAXK], ans;
vector<int> toReset;
void getDist(int u, int fa, int d, int e) {
    if (d > K) return;
    if (best[K - d] >= 0) ans = min(ans, e + best[K - d]);
    cur.push_back({d, e});
    for (auto [v, w] : adj[u])
        if (v != fa && !del[v]) getDist(v, u, d + w, e + 1);
}
void calc(int c) {
    best[0] = 0; toReset.push_back(0);
    for (auto [v, w] : adj[c]) {
        if (del[v]) continue;
        cur.clear(); getDist(v, c, w, 1);
        for (auto [d, e] : cur)
            if (best[d] < 0 || e < best[d])
                best[d] = e, toReset.push_back(d);
    }
    for (int x : toReset) best[x] = -1;
    toReset.clear();
}
```

**核心启示**：当桶需要记录的不是"是否存在"而是"最优值"时，只需将布尔桶换成记录最优值的桶。注意桶的清空要用记录式清空（不能 memset），否则会退化为 $O(K)$。

### 题目 6：P3241 [HNOI2015]开店 🔴

**题意**：给定一棵 $n$ 个节点的带权树，每个节点有一个值 $a_i$。多次在线询问：给定节点 $u$ 和区间 $[L, R]$，求所有满足 $a_v \in [L, R]$ 的节点 $v$ 到 $u$ 的距离之和。强制在线。

**分析**：
- 需要在线回答"到某点的距离之和"，且有值域约束，自然想到点分树
- 在点分树上，$u$ 到任意点 $v$ 的路径一定经过它们在点分树上的 LCA
- 对点分树上每个节点 $c$，维护其管辖范围内所有节点按 $a$ 值排序后到 $c$ 的距离前缀和
- 查询时从 $u$ 沿点分树向根跳，每层利用二分查找统计 $a_v \in [L, R]$ 的距离和，并减去重复计算的部分

> **算法**
> 1. 建点分树，对每个节点 $c$，收集管辖区域内所有点，按 $a$ 值排序
> 2. 对每个 $c$，预处理两个前缀和数组：到 $c$ 的距离前缀和、到 $c$ 在点分树上父亲的距离前缀和
> 3. 查询 $(u, L, R)$：从 $u$ 向点分树根跳，每层用二分求 $a \in [L, R]$ 的距离和，累加后减去到上一层的重复
> **复杂度**：$O(n \log^2 n)$

**C++ 代码（核心）**：
```cpp
// dist1[c]: 管辖区域内各点到 c 的距离（按 a 排序后前缀和）
// dist2[c]: 管辖区域内各点到 cdFa[c] 的距离（同上）
long long query(int u, int L, int R) {
    long long res = 0;
    for (int c = u; c; c = cdFa[c]) {
        int cnt = cntInRange(c, L, R); // 二分求 a∈[L,R] 的个数
        long long s1 = sumInRange(c, L, R); // 这些点到 c 的距离和
        res += s1 + (long long)dist(u, c) * cnt;
        if (cdFa[c]) {
            long long s2 = sumInRange2(c, L, R);
            res -= s2 + (long long)dist(u, cdFa[c]) * cnt;
        }
    }
    return res;
}
```

**核心启示**：点分树将"离线的点分治"转化为"在线的层级数据结构"。查询时沿点分树向根跳跃累加贡献，用"到当前层 - 到上一层"来容斥消除重复。这是动态点分治的核心模式。

### 题目 7：P4075 [SDOI2016]模式字符串 🔴

**题意**：给定一棵 $n$ 个节点的树，每个节点有一个字符。给定模式串 $S$，求有多少有序点对 $(u, v)$ 满足 $u \to v$ 的路径拼出的字符串恰好是 $S$ 重复若干次。

**分析**：
- 路径上字符串 = $S^k$（$S$ 重复 $k$ 次），等价于路径长度是 $|S|$ 的倍数，且路径字符串的每个前缀/后缀都与 $S$ 的对应前缀/后缀一致
- 点分治框架：经过重心 $c$ 的路径 $u \to c \to v$，$u \to c$ 的字符串是 $S$ 的某个后缀循环，$c \to v$ 是 $S$ 的某个前缀循环
- 用哈希判断匹配：从 $c$ 向下走，维护到当前位置在 $S$ 中的匹配位置和哈希值

> **算法**
> 1. 点分治，对重心 $c$
> 2. DFS 收集从 $c$ 出发向下的路径，判断它是否是 $S$ 前缀的循环重复（正向）和 $S$ 后缀的循环重复（反向）
> 3. 用哈希验证匹配，桶统计正向长度 $\bmod |S|$ 余 $r$ 的合法路径数
> 4. 反向路径与正向路径配对：正向长度 $\bmod |S| = r$ 的配反向长度 $\bmod |S| = |S| - r$ 的
> **复杂度**：$O(n \log n)$

**C++ 代码（核心）**：
```cpp
// len = |S|, 用哈希判断路径前缀/后缀是否匹配 S 的循环
int fCnt[MAXLEN], rCnt[MAXLEN]; // 正/反向匹配到位置 r 的路径数
void dfs(int u, int fa, int pos, bool ok) {
    // pos: 当前在 S 中匹配到的位置
    // ok: 到目前为止是否全部匹配
    if (!ok) return;
    ok &= (ch[u] == S[pos]);
    if (ok && pos == len - 1) fCnt[0]++; // 完整匹配了 S 的倍数
    else if (ok) fCnt[pos + 1]++;
    for (auto [v, w] : adj[u])
        if (v != fa && !del[v]) dfs(v, u, (pos + 1) % len, ok);
}
```

**核心启示**：字符串匹配 + 点分治的结合，关键在于将路径字符串的匹配条件转化为在 $S$ 上的位置对齐。哈希是高效验证匹配的工具，逐字符 DFS 时维护匹配状态。

### 题目 8：P4183 [USACO18JAN]Cow at Large P 🟠

**题意**：一棵 $n$ 个节点的树，所有叶子节点上有农夫。奶牛从某个节点 $s$ 出发试图到达某个叶子逃脱，农夫可以移动拦截。求对于每个起点 $s$，至少需要多少个农夫才能保证拦截奶牛。

**分析**：
- 对于单次查询，观察可知：一个叶子 $l$ 能"有效覆盖"某个节点，当且仅当 $l$ 到该节点的距离不超过 $s$ 到该节点的距离
- 定义 $d_s(u)$ 为 $s$ 到 $u$ 的距离，$d_l(u)$ 为离 $u$ 最近的叶子到 $u$ 的距离
- 答案 = 满足 $d_l(u) \le d_s(u)$ 且 $d_l(\text{fa}(u)) > d_s(\text{fa}(u))$ 的节点 $u$ 的个数（即"分界面"上的点数）
- 多次查询需要对所有 $s$ 求答案，用点分治优化

> **算法**
> 1. 先预处理每个节点到最近叶子的距离 $g_u$
> 2. 对于每个节点定义 $f_u = [g_u \le d_s(u)] - [g_{\text{fa}(u)} \le d_s(\text{fa}(u))]$（用容斥表示分界面）
> 3. 用点分治统计 $\sum_u f_u$ 对每个起点 $s$ 的贡献
> 4. 等价于统计子树中满足 $g_u \le \text{dist}(s, u)$ 的点数，利用点分治 + 排序 + 二分实现
> **复杂度**：$O(n \log^2 n)$

**C++ 代码（核心）**：
```cpp
// g[u] = u 到最近叶子的距离（BFS 预处理）
// 点分治中，统计经过重心 c 的路径的贡献
void calc(int c) {
    // 收集子树中各点 v 到 c 的距离 d(c,v) 和 g[v]
    // 对起点 s：需要 d(s,c) + d(c,v) >= g[v]，即 d(c,v) - g[v] >= -d(s,c)
    // 将 d(c,v) - g[v] 排序，对每个 s 二分统计满足条件的点数
    sort(info.begin(), info.end()); // 按 d-g 排序
    for (auto [v, dv] : subNodes)
        ans[v] += upperBound(info, dv); // 二分统计
}
```

**核心启示**：当问题要求对每个节点都求一次答案时，点分治可以"反过来用"——不是枚举路径的两端，而是枚举一端并统计另一端的贡献。排序 + 二分是高效统计的关键。

### 题目 9：P4292 [WC2010]重建计划 🟣

**题意**：给定一棵 $n$ 个节点的带权树，求一条包含 $L$ 到 $U$ 条边的路径，使得路径上边权的平均值最大。

**分析**：
- $\max \frac{\sum w_i}{|E|}$，其中 $L \le |E| \le U$，这是经典的 **分数规划** 问题
- 二分答案 $\text{mid}$，判断是否存在路径满足 $\sum (w_i - \text{mid}) \ge 0$ 且边数在 $[L, U]$ 内
- 二分中使用点分治来判定：对重心的子树收集路径信息，查询是否存在合法配对
- 需要在边数 $[L, U]$ 的约束下查询最大权值和——使用 **单调队列** 优化

> **算法**
> 1. 二分答案 $\text{mid}$
> 2. 将边权减去 $\text{mid}$，问题转化为：是否存在边数 $\in [L, U]$ 的路径使得边权和 $\ge 0$
> 3. 点分治判定：对重心逐子树合并，维护 $best[l]$ = 已处理子树中边数为 $l$ 的最大边权和
> 4. 对当前子树中边数为 $e$ 的链，需查询 $best[L-e..U-e]$ 的最大值，用单调队列实现
> **复杂度**：$O(n \log^2 n)$（点分治 $O(n \log n)$ + 二分 $O(\log n)$）

**C++ 代码（核心）**：
```cpp
double best[MAXN]; // best[l] = 已处理子树中长度为 l 的链的最大权值和
int maxLen; // 当前已处理的最大链长
bool check(double mid) {
    // 点分治判定：边权全部减去 mid
    // 对重心 c 逐子树合并
    // 当前子树 DFS 得到链长 e, 权值和 val
    // 需要查询 best[max(L-e,0) .. min(U-e, maxLen)] 的最大值
    // 若 val + maxBest >= 0，则存在合法路径
    // 用单调队列维护 best 在滑动窗口 [L-e, U-e] 上的最大值
    deque<int> dq;
    for (int e = 按链长从大到小) {
        int lo = L - e, hi = U - e;
        while (右端新增) { /* 单调队列入队 */ }
        while (dq.front() < lo) dq.pop_front();
        if (!dq.empty() && val[e] + best[dq.front()] >= 0) return true;
    }
}
```

**核心启示**：分数规划 + 点分治 + 单调队列是三者结合的经典范式。二分将最优化问题转化为判定问题，点分治处理树上路径，单调队列在区间约束下高效求最值。

### 题目 10：P5306 [COCI2019]Transport 🔥

**题意**：给定一棵 $n$ 个节点的带权树，每个节点有一个 **油量** $f_i$，每条边有长度 $w$。从 $u$ 走到 $v$ 时，经过节点会加上该节点的油量，经过边会消耗边权的油量。初始油量为 0。求有多少有序点对 $(u, v)$ 满足从 $u$ 到 $v$ 过程中油量始终 $\ge 0$。

**分析**：
- 路径 $u \to c \to v$（经过重心），需要 $u \to c$ 的前缀油量始终非负，且 $c \to v$ 的后缀油量始终非负（从拼接处看）
- 从 $u$ 到 $c$：维护"到 $c$ 的净油量"和"路径上前缀油量的最小值"
- 从 $c$ 到 $v$：维护"从 $c$ 到 $v$ 的净油量"和"路径上后缀油量的最小值"
- 两条链能拼接的条件：$u \to c$ 的最小前缀油量 $\ge 0$，且 $c \to v$ 的油量加上 $u \to c$ 的净油量后后缀最小值 $\ge 0$
- 转化为二维偏序问题，用排序 + 树状数组解决

> **算法**
> 1. 点分治，对重心 $c$ DFS 收集正向（$u \to c$）和反向（$c \to v$）链信息
> 2. 正向链：$(net_u, minPre_u)$，要求 $minPre_u \ge 0$
> 3. 反向链：$(net_v, minSuf_v)$，要求 $minSuf_v + net_u \ge 0$
> 4. 即 $net_u \ge -minSuf_v$，排序后用树状数组或归并统计满足条件的对数
> 5. 逐子树合并或容斥，注意方向性（有序对需正反各做一次）
> **复杂度**：$O(n \log^2 n)$

**C++ 代码（核心）**：
```cpp
// info: {net, minPre} 正向链信息
// rinfo: {net, minSuf} 反向链信息
void calc(int c) {
    // 收集子树正向和反向信息
    // 正向要求 minPre >= 0 才合法
    // 配对条件: net_forward + minSuf_reverse >= 0
    //        即 net_forward >= -minSuf_reverse
    sort(rinfo.begin(), rinfo.end()); // 按 -minSuf 排序
    sort(finfo.begin(), finfo.end()); // 按 net 排序
    // 双指针 / 树状数组统计满足条件的对数
    int j = 0;
    for (int i = rinfo.size() - 1; i >= 0; i--) {
        while (j < finfo.size() && finfo[j].net >= -rinfo[i].minSuf)
            BIT.add(finfo[j++].net);
        ans += BIT.query(...);
    }
}
```

**核心启示**：涉及路径上"前缀/后缀"约束（如油量始终非负）的问题，需要分别维护两个方向的约束信息。拼接时的合法性判定往往转化为二维偏序，排序 + 树状数组是标准处理方法。

## 五、常见技巧与思路总结

### 技巧 1：桶的清空方式

**绝对不要** 在每次处理完一个重心后用 `memset` 清空桶。正确做法是用 vector 记录所有修改过的位置，逐个还原。

```cpp
// 错误做法：memset(bucket, 0, sizeof bucket); → O(MAXVAL) 每次
// 正确做法：
vector<int> modified;
// 修改时：bucket[x] = val; modified.push_back(x);
// 清空时：for (int x : modified) bucket[x] = 0; modified.clear();
```

### 技巧 2：子树大小的正确计算

在递归子树时，子树大小不能直接用 `sz[v]`，因为 `getCentroid` 过程中 `sz` 数组可能以错误的节点为根。正确做法：

```
若 sz[v] < sz[c]：子树大小 = sz[v]
若 sz[v] >= sz[c]：子树大小 = tot - sz[c]
```

### 技巧 3：逐子树合并 vs. 容斥

| 场景 | 推荐方式 |
|------|---------|
| 需要排序 + 双指针 | 容斥（先全部收集再减去） |
| 桶计数类问题 | 逐子树合并（更简洁） |
| 无法定义"减法"操作 | 逐子树合并 |
| 统计代码复用性强 | 容斥（正反都调同一函数） |

### 技巧 4：点分治中距离的剪枝

当问题存在距离上限 $K$ 时，DFS 收集距离的过程中一旦超过 $K$ 就立即返回，可以显著减少实际遍历的节点数。

### 技巧 5：点分树的空间管理

点分树中每个节点存储其管辖范围内所有节点的信息。由于每个节点恰好属于 $O(\log n)$ 个管辖范围，总空间为 $O(n \log n)$。实现时通常用 `vector` 存储，利用"所有 vector 大小之和为 $O(n \log n)$"的性质。

## 六、优化方法 / 解题框架

### 6.1 点分治通用框架

```
输入 → 建树 → solve(根, n)
  solve(u, tot):
    1. getCentroid(u, 0, tot) → 得到重心 c
    2. del[c] = true
    3. calc(c) → 统计经过 c 的路径贡献
    4. for (v ∈ adj[c], !del[v]):
         solve(v, subSize(v))
```

### 6.2 点分树通用框架

```
预处理阶段：
  buildCDTree(根, n, 0) → 建出点分树 cdFa[]
  对每个节点 c，收集其管辖区域的信息并排序/建数据结构

查询阶段（查询点 u）：
  res = 0
  for (c = u; c != 0; c = cdFa[c]):
    res += 从 c 的数据结构中查询与 u 相关的贡献
    res -= 从 c 的数据结构中查询"到 cdFa[c]"的重复贡献  // 容斥
  return res

修改阶段（修改点 u）：
  for (c = u; c != 0; c = cdFa[c]):
    更新 c 的数据结构
```

### 6.3 常见复杂度分析

| 做法 | 统计贡献方式 | 总时间复杂度 |
|------|------------|-------------|
| 点分治 + 桶 | $O(n)$ 每层 | $O(n \log n)$ |
| 点分治 + 排序 | $O(n \log n)$ 每层 | $O(n \log^2 n)$ |
| 点分治 + 树状数组 | $O(n \log n)$ 每层 | $O(n \log^2 n)$ |
| 二分 + 点分治 | 外层二分 $O(\log V)$ | $O(n \log n \cdot \log V)$ |
| 点分树 + 二分 | 查询 $O(\log^2 n)$ | $O(n \log^2 n)$ 预处理 + $O(q \log^2 n)$ 查询 |

### 6.4 调试技巧

1. **单独测试重心求法**：先对小树打印重心，验证是否正确
2. **验证递归总规模**：在 `solve` 开头累加 `tot`，最终应为 $O(n \log n)$
3. **桶越界检查**：确保距离不超过桶的大小（加上 `if (d > MAXD) return;` 的剪枝）
4. **容斥正确性**：用小数据暴力验证，确保不多统计也不遗漏
5. **清空彻底性**：确保每个重心处用过的桶都被正确还原

### 做题时的思考流程

1. **识别**：题目涉及树上路径统计 → 考虑点分治
2. **确认**：路径信息可以从两端拼接 → 分治中心选重心
3. **设计 calc**：
   - 从重心出发 DFS 收集什么信息？
   - 两条链如何配对？配对条件是什么？
   - 用什么数据结构高效统计？
4. **选择合并方式**：逐子树合并 or 容斥
5. **分析复杂度**：每层统计的开销 $\times O(\log n)$ 层
6. **是否需要点分树**：在线 / 带修 → 预建点分树

### 一句话记忆

> **点分治 = 选重心做分治中心 + 统计经过重心的路径 + 递归子树。点分树 = 把点分治的递归结构建成树，支持在线查询。**
