## 一、什么是 2-SAT

**SAT 问题**（Boolean Satisfiability Problem）是计算机科学中最基础的判定问题之一：给定一个由布尔变量组成的**合取范式（CNF）**，问是否存在一组变量赋值使得整个公式为**真**。

### SAT 问题定义

一个 CNF 公式由若干**子句（clause）**的合取（AND）组成，每个子句是若干**文字（literal）**的析取（OR）。文字是变量 $x_i$ 或其否定 $\lnot x_i$。

$$\varphi = (l_{1,1} \lor l_{1,2} \lor \cdots) \land (l_{2,1} \lor l_{2,2} \lor \cdots) \land \cdots$$

当每个子句中**恰好含 $k$ 个文字**时，称为 **$k$-SAT** 问题。

### 2-SAT vs 3-SAT

| 性质 | 2-SAT | 3-SAT |
|------|-------|-------|
| 每个子句的文字数 | 恰好 2 | 恰好 3 |
| 计算复杂度 | **多项式时间** $O(n + m)$ | **NP 完全**（目前无多项式算法） |
| 求解方法 | 蕴含图 + Tarjan SCC | 回溯搜索 / SAT Solver |
| 实用性 | 可高效解决大规模实例 | 仅能处理较小规模 |

**为何 2-SAT 可以多项式时间求解？** 关键在于：每个子句恰好有两个文字时，一个子句 $(a \lor b)$ 可以等价转化为两条**蕴含关系** $\lnot a \Rightarrow b$ 和 $\lnot b \Rightarrow a$。所有蕴含关系构成一张有向图（蕴含图），利用强连通分量即可判定可满足性并构造解。而 3-SAT 没有这种优美的图论转化。

### 布尔变量与子句

- **变量**：$x_1, x_2, \ldots, x_n$，取值为 $\text{true}$ 或 $\text{false}$
- **文字**：$x_i$（取真）或 $\lnot x_i$（取假）
- **子句**：两个文字的析取，如 $(x_1 \lor \lnot x_3)$
- **公式**：所有子句的合取，要求同时满足

```
例：n=3, 子句为 (x1 ∨ ¬x2) ∧ (¬x1 ∨ x3) ∧ (x2 ∨ x3)

赋值 x1=T, x2=T, x3=T：
  (T ∨ F) ∧ (F ∨ T) ∧ (T ∨ T) = T ∧ T ∧ T = T  ✓ 可满足
```

## 二、核心思想与前置知识

### 2.1 蕴含图建模

2-SAT 的核心是将每个子句转化为**蕴含（implication）**关系，然后建立**蕴含图（Implication Graph）**。

对于每个布尔变量 $x_i$，在图中建两个节点：
- 节点 $2i$：表示 $x_i = \text{false}$
- 节点 $2i+1$：表示 $x_i = \text{true}$

> 也可写作节点 $i$ 表示 $x_i$，节点 $i + n$ 表示 $\lnot x_i$，具体看实现习惯。

**核心等价关系**：子句 $(a \lor b)$ 等价于"若 $\lnot a$ 则必须 $b$"且"若 $\lnot b$ 则必须 $a$"：

$$
(a \lor b) \iff (\lnot a \Rightarrow b) \land (\lnot b \Rightarrow a)
$$

每条蕴含关系对应蕴含图中的一条有向边。

```
变量 x_i 对应两个节点：
  i*2     → 代表 x_i 为假 (¬x_i)
  i*2+1   → 代表 x_i 为真 (x_i)

子句 (x1 ∨ ¬x2) 的蕴含：
  ¬x1 → ¬x2   即  节点 2*1+1=3 的否定(2) → 节点 2*2(=4)
  x2  → x1     即  节点 2*2+1(=5) → 节点 2*1+1(=3)

  简写: 1*2 → 2*2,  2*2+1 → 1*2+1
```

### 2.2 逻辑转化规则表（OR/XOR/NAND 等转建边）

设 $a, b$ 为文字（可以是 $x_i$ 或 $\lnot x_i$），$\bar{a}$ 表示 $a$ 的否定。

| 约束类型 | 逻辑含义 | 等价蕴含 | 建边（有向） |
|---------|---------|---------|-------------|
| $a \lor b$ | 至少一个为真 | $\bar{a} \Rightarrow b$，$\bar{b} \Rightarrow a$ | $\bar{a} \to b$，$\bar{b} \to a$ |
| $\lnot a \lor \lnot b$ (NAND) | 不同时为真 | $a \Rightarrow \lnot b$，$b \Rightarrow \lnot a$ | $a \to \bar{b}$，$b \to \bar{a}$ |
| $a \oplus b$ (XOR，恰好一真) | 恰好选一 | $(a \Rightarrow \lnot b) \land (\lnot a \Rightarrow b)$ | $a \to \bar{b}$，$b \to \bar{a}$，$\bar{a} \to b$，$\bar{b} \to a$ |
| $a \Leftrightarrow b$（同真同假） | 等价 | $(a \Rightarrow b) \land (b \Rightarrow a)$ | $a \to b$，$b \to a$，$\bar{a} \to \bar{b}$，$\bar{b} \to \bar{a}$ |
| $a$ 必须为真 | 强制赋值 | $\bar{a} \Rightarrow a$ | $\bar{a} \to a$ |
| 至多一个为真 | $\lnot(a \land b)$ | 同 NAND | $a \to \bar{b}$，$b \to \bar{a}$ |

**建边的统一口诀**：

> **"若选了 $a$ 会导致矛盾，就从 $a$ 连边到矛盾的推论"**。每条蕴含 $p \Rightarrow q$ 对应一条有向边 $p \to q$。

### 2.3 Tarjan SCC 判定可满足性

建好蕴含图后，运行 **Tarjan 算法** 求出所有强连通分量（SCC）。

**判定规则**：

$$
\text{可满足} \iff \forall i,\ x_i \text{ 与 } \lnot x_i \text{ 不在同一个 SCC 中}
$$

若 $x_i$ 和 $\lnot x_i$ 在同一个 SCC 中，意味着 $x_i \Rightarrow \lnot x_i$ 且 $\lnot x_i \Rightarrow x_i$，矛盾。

**Tarjan 求 SCC 模板**：

```cpp
// n: 变量个数, 图中共 2*n 个节点 (0..2n-1)
// scc_id[u]: 节点 u 所在 SCC 的编号（拓扑逆序）
int dfn[MAXN], low[MAXN], stk[MAXN], scc_id[MAXN];
bool in_stk[MAXN];
int ts, top, scc_cnt;

void tarjan(int u) {
    dfn[u] = low[u] = ++ts;
    stk[++top] = u; in_stk[u] = true;
    for (int v : adj[u]) {
        if (!dfn[v]) { tarjan(v); low[u] = min(low[u], low[v]); }
        else if (in_stk[v]) low[u] = min(low[u], dfn[v]);
    }
    if (dfn[u] == low[u]) {
        scc_cnt++;
        int x;
        do { x = stk[top--]; in_stk[x] = false; scc_id[x] = scc_cnt; } while (x != u);
    }
}
```

**判定**：

```cpp
bool satisfiable(int n) {
    for (int i = 0; i < n; i++)
        if (scc_id[i * 2] == scc_id[i * 2 + 1])
            return false;
    return true;
}
```

### 2.4 输出方案（SCC 编号比较）

当 2-SAT 可满足时，还需输出一组具体的赋值方案。利用 Tarjan 算法的一个关键性质：**SCC 编号的逆序就是拓扑序**（Tarjan 缩点后的 DAG 上，编号越大的 SCC 拓扑序越靠前）。

**赋值规则**：

$$
x_i = \begin{cases} \text{true} & \text{if } \operatorname{scc\_id}(x_i) > \operatorname{scc\_id}(\lnot x_i) \\ \text{false} & \text{otherwise} \end{cases}
$$

**直觉理解**：在缩点后的 DAG 上，SCC 编号大意味着拓扑序靠前。若 $x_i$ 所在 SCC 的拓扑序在 $\lnot x_i$ 前面，意味着从 $\lnot x_i$ 出发可以推导出 $x_i$，因此 $x_i$ 被"蕴含"为真。

```cpp
// 输出方案
for (int i = 0; i < n; i++)
    val[i] = scc_id[i * 2] < scc_id[i * 2 + 1];
    // scc_id 小 → 拓扑序靠后 → 被蕴含
    // 若 ¬x_i 的 scc_id 更小（拓扑序更后），则选 x_i 为真
```

## 三、分类与变体详解

### 3.1 基本 2-SAT

最标准的形式：给定 $n$ 个布尔变量和 $m$ 个形如 $(a \lor b)$ 的子句，判断是否可满足并输出方案。

**完整求解流程**：

```
1. 对每个变量 xi 建两个节点: 2i (¬xi) 和 2i+1 (xi)
2. 对每个子句 (a ∨ b) 建两条蕴含边: ¬a→b, ¬b→a
3. 跑 Tarjan 求 SCC
4. 检查: xi 和 ¬xi 不在同一 SCC → 可满足
5. 输出方案: 比较 scc_id 大小
```

**时间复杂度**：$O(n + m)$，其中 $n$ 为变量数，$m$ 为子句数。

### 3.2 至多选一 (at most one)

**问题**：给定一组文字 $a_1, a_2, \ldots, a_k$，要求其中**至多一个为真**。

**朴素方法**：对所有 $i \ne j$，添加子句 $(\lnot a_i \lor \lnot a_j)$，即建边 $a_i \to \bar{a_j}$ 和 $a_j \to \bar{a_i}$。边数为 $O(k^2)$。

**优化方法（前缀辅助变量）**：引入 $k-1$ 个辅助变量 $y_1, \ldots, y_{k-1}$，其中 $y_j$ 表示"$a_1, \ldots, a_j$ 中是否已经有一个为真"。

```
约束：
  若 a_j 为真，则 y_j 必须为真:    a_j → y_j
  若 y_j 为真，则 y_{j+1} 也为真:  y_j → y_{j+1}  （前缀传递）
  若 y_j 已为真，则 a_{j+1} 不能为真: y_j → ¬a_{j+1}

边数从 O(k^2) 降到 O(k)
```

这是 **前缀优化建图** 在 2-SAT 中的典型应用。

### 3.3 恰好选一 (XOR)

**问题**：$a$ 和 $b$ 中恰好有一个为真（异或关系 $a \oplus b = 1$）。

**建边**：需要 4 条蕴含边：

$$a \Rightarrow \lnot b, \quad b \Rightarrow \lnot a, \quad \lnot a \Rightarrow b, \quad \lnot b \Rightarrow a$$

这等价于 $(a \lor b) \land (\lnot a \lor \lnot b)$ 的合取，即"至少一个为真"加上"至多一个为真"。

### 3.4 2-SAT + 二分答案

很多竞赛题将 2-SAT 与二分答案结合：二分一个参数 $d$，根据 $d$ 的值建立不同的 2-SAT 约束，判断是否可满足。

**典型模式**：

```
二分答案 d:
  1. 根据 d 建立 2-SAT 蕴含图
  2. Tarjan SCC 判定可满足性
  3. 可满足 → 调整边界（d 可行）
     不可满足 → 调整边界（d 不可行）
```

**常见场景**：两个选择之间的"距离"或"差值"不超过 $d$，二分 $d$ 的最小值/最大值。

### 3.5 2-SAT + 前缀/后缀优化建图

当约束涉及"在一组元素中选择了某个，则另一组中某个范围内的元素都不能选"时，暴力连边会导致 $O(n^2)$ 条边。

**前缀/后缀优化**：建立前缀节点 $P_j$（表示"$a_1 \sim a_j$ 中至少一个为真"）和后缀节点 $S_j$（表示"$a_j \sim a_k$ 中至少一个为真"），通过辅助节点的链式连接将边数降至 $O(n)$。

```
前缀节点 P_j 的蕴含链：
  P_j → P_{j+1}                   （前缀单调性）
  a_j → P_j                       （选了 a_j 则前缀 j 以后都被激活）

后缀节点 S_j 的蕴含链：
  S_j → S_{j-1}                   （后缀单调性）

实际约束 "选 a_i 则 a_1..a_{i-1} 全不选"：
  a_i → ¬P_{i-1}                  （一条边代替 i-1 条边）
```

这种技巧在处理区间约束的 2-SAT 问题中非常常用。

## 四、经典例题详解

### 题目 1：P4782 【模板】2-SAT 问题

**难度**：🟢

**题意**：有 $n$ 个布尔变量 $x_1, \ldots, x_n$，给出 $m$ 个形如"$x_i$ 为 $a$ 或 $x_j$ 为 $b$"的约束（$a, b \in \{0, 1\}$）。判断是否有解，有则输出方案。

**分析**：

这是 2-SAT 的裸题。每个约束"$x_i$ 为 $a$ 或 $x_j$ 为 $b$"就是一个子句。按照标准方法建蕴含图、跑 Tarjan、判定并输出方案。

```
算法流程：
1. 每个变量 xi 建两个节点: 2i 和 2i+1
2. 对约束 "xi=a 或 xj=b":
   - 文字 a 对应节点 2i+a, 文字 b 对应节点 2j+b
   - 建边: (2i+!a) → (2j+b), (2j+!b) → (2i+a)
3. Tarjan SCC
4. 判定 + 输出
```

**C++ 代码**：

```cpp
// add_clause(u, a, v, b): 添加子句 (xu=a ∨ xv=b)
// u,v: 变量编号(0-indexed), a,b ∈ {0,1}
void add_clause(int u, int a, int v, int b) {
    adj[u*2+!a].push_back(v*2+b);
    adj[v*2+!b].push_back(u*2+a);
}
// 建图后跑 Tarjan，检查 scc_id[2i] != scc_id[2i+1]
// 方案: val[i] = (scc_id[2i] < scc_id[2i+1])
```

**核心启示**：这是 2-SAT 的基础骨架，所有后续题目都建立在此之上。关键是 **"或"约束到蕴含边的转化** 以及 **SCC 编号比较确定赋值方案**。

### 题目 2：P4171 [JSOI2010]满汉全席

**难度**：🟡

**题意**：有 $n$ 种食材，每种可以做成满式或汉式（两种状态选其一）。给出 $m$ 个评委的要求，每个评委指定两道菜：如果第一道满足不了，第二道必须满足（至少满足一道）。判断是否能同时满足所有评委。多组数据。

**分析**：

每种食材是一个布尔变量（满式=true / 汉式=false）。每位评委的要求恰好是一个 2-SAT 子句"$x_i$ 为 $a$ 或 $x_j$ 为 $b$"。直接套模板判定可满足性。

```
算法流程：
1. xi: 0=汉式, 1=满式
2. 评委要求 "(xi 为 a) 或 (xj 为 b)" → 标准子句
3. 建蕴含图, Tarjan SCC, 判定
4. 注意多组数据需清空
```

**C++ 代码**：

```cpp
// 对每位评委 (i,a) (j,b): 至少一个满足
void add_clause(int i, int a, int j, int b) {
    adj[i*2+!a].push_back(j*2+b);
    adj[j*2+!b].push_back(i*2+a);
}
// Tarjan 后检查每个变量的两个节点是否在同一 SCC
// 是 → "IMPOSSIBLE", 否 → "POSSIBLE"
```

**核心启示**：2-SAT 的 **"判定可满足"** 是最基础的应用。只要每个约束可以表达为"二选一"的形式，就可以建成 2-SAT 模型。注意多测时要彻底清空所有数组。

### 题目 3：P3825 [NOI2017]游戏

**难度**：🟠

**题意**：有 $n$ 场游戏，每场在三种地图 A/B/C 中选一种。有些场次地图**不能**选某一种（类型 $x$），其余场次可以任选三种之一。给出 $m$ 条约束"若第 $i$ 场选地图 $h_i$，则第 $j$ 场必须选地图 $h_j$"。求一种合法方案。

**分析**：

表面看是 3-SAT（三选一），但关键观察是：**类型 $x$ 的场次只有 2 种合法选择**，天然是 2-SAT。问题在于"可以三选一"的场次（最多 $d \le 8$ 个）。

对于这 $d$ 个"自由场次"，可以枚举它们的"禁用选项"（每个场次枚举禁用 A 或 B 或 C 中的一种，使其变为"二选一"）。枚举量为 $3^d$，但 $d \le 8$，$3^8 = 6561$，可以接受。实际只需枚举 $2^d$ 种（禁用 A 或禁用 B 即可涵盖所有情况，因为如果答案选 C，无论禁 A 还是禁 B 都不影响）。

```
算法流程：
1. 类型 x 的场次: 只有 2 种选择 → 直接 2-SAT 变量
2. 自由场次 (≤8个): 枚举禁用哪种 → 变成 2 种选择
3. 枚举 2^d 种方案, 对每种:
   a. 建 2-SAT 蕴含图
   b. 约束 "若第 i 场选 hi, 则第 j 场选 hj"
      → 若 hi 是 i 的某个选项 a, hj 是 j 的某个选项 b
      → 加蕴含: 选a → 选b (以及 ¬b → ¬a)
   c. Tarjan 判定
   d. 可满足则输出方案, 结束
4. 所有枚举都不可满足 → -1
```

**C++ 代码**：

```cpp
// map_choice[i][0], map_choice[i][1]: 场次 i 的两种合法地图
// 约束 (i,hi) → (j,hj): 若 i 选 hi, 则 j 必须选 hj
void add_constraint(int i, int hi, int j, int hj) {
    int a = -1, b = -1;
    for (int k = 0; k < 2; k++) {
        if (map_choice[i][k] == hi) a = k;
        if (map_choice[j][k] == hj) b = k;
    }
    if (a == -1) return;     // hi 不是 i 的合法选项, 约束自动满足
    if (b == -1) {           // hj 不是 j 的合法选项, i 不能选 hi
        adj[i*2+a].push_back(i*2+!a);  // 强制 i 不选 a
        return;
    }
    adj[i*2+a].push_back(j*2+b);       // 选a → 选b
    adj[j*2+!b].push_back(i*2+!a);     // ¬b → ¬a
}
```

**核心启示**：**三选一不一定是 3-SAT**。当"自由变量"数量有限时，可以枚举将其转化为 2-SAT。本题的关键是 $d \le 8$ 这个条件，使得枚举 $2^d$ 种"禁用方案"成为可行策略。

### 题目 4：P5332 [JSOI2019]精准预测

**难度**：🔴

**题意**：有 $n$ 个个体，初始状态为"存活"。在 $T$ 个时间步中，给出 $m$ 条约束，每条形如：

- "若时刻 $t$ 个体 $x$ 存活，则时刻 $t$ 个体 $y$ 死亡"
- "若时刻 $t$ 个体 $x$ 死亡, 则时刻 $t$ 个体 $y$ 存活"

个体一旦死亡就不会复活。对于每个个体 $i$，计算在时刻 $T$ 时，有多少个其他个体**可能**与 $i$ **同时存活**。

**分析**：

将每个个体在每个时刻的状态建为 2-SAT 变量。由于"死亡不可逆"，有隐含约束：若时刻 $t$ 死亡则时刻 $t+1$ 也死亡（$\text{dead}_t \Rightarrow \text{dead}_{t+1}$），等价于"若时刻 $t+1$ 存活则时刻 $t$ 也存活"。

直接 $n \times T$ 个变量太大，需要观察到实际约束只涉及有限的时刻，可以只对有约束的时刻建立变量。更关键的是，统计"可以同时存活的对数"需要高效的方法。

**核心思路**：利用 2-SAT 蕴含图的可达性——若从 $\text{alive}(i, T)$ 可以到达 $\text{dead}(j, T)$，则 $i$ 和 $j$ 不可能同时存活。配合前缀/后缀优化建图来处理时间维度上的传递约束，以及位集（bitset）加速可达性统计。

```
算法框架：
1. 对每个个体 i 在 T 时刻建 2-SAT 节点: alive_i, dead_i
2. 死亡不可逆: 通过时间链连边
3. 约束转蕴含边
4. Tarjan SCC 缩点
5. 在 DAG 上传播可达性 (bitset 优化)
6. 对每个 i: 统计 alive_i 能到达 dead_j 的 j 的数量 → 不能共存
   ans[i] = n - 1 - (不能共存的个数)
```

**C++ 代码**：

```cpp
// 核心: 约束建边 + 前缀优化
// 约束"时刻t x存活 → 时刻t y死亡":
//   alive(x,t) → dead(y,t)
//   alive(y,t) → dead(x,t) (逆否)
// 死亡不可逆: dead(x,t) → dead(x,t+1), alive(x,t+1) → alive(x,t)
// SCC 缩点后 DAG 上传播 bitset 统计可达性
void add_implication(int u, int v) {  // u → v
    adj[u].push_back(v);
}
// 统计答案时用 bitset 在 DAG 上逆拓扑序 DP
```

**核心启示**：本题是 2-SAT 的高级应用。关键技巧包括：(1) 时间维度上的链式建图处理"不可逆"约束；(2) 前缀/后缀优化避免 $O(nT)$ 的建图规模；(3) bitset 优化传递闭包来统计可达对数。这体现了 **2-SAT 不仅用于判定，还可以利用蕴含图的结构做更精细的分析**。

### 题目 5：2-SAT + 二分答案（经典模型）

**难度**：🟣

**题意**：在数轴上有 $n$ 个区间，每个区间可以放在位置 $a_i$ 或 $b_i$。两个放置后的区间如果重叠则冲突。求最小的缩放因子 $d$，使得可以为每个区间选择一个位置且不发生冲突；或者等价地，二分最大的间距/覆盖长度使得可行。

**分析**：

每个区间有两种选择（$a_i$ 或 $b_i$），天然对应布尔变量。两个选择产生冲突当且仅当"区间重叠"，这取决于参数 $d$。二分 $d$ 后，冲突关系确定，构成 2-SAT 约束。

```
算法框架 (二分 + 2-SAT):
1. 二分参数 d
2. 对于每对区间 (i, j), 枚举 4 种放置组合:
   - 若 (i选a, j选a) 冲突: 加子句 (¬a_i ∨ ¬a_j)
   - 若 (i选a, j选b) 冲突: 加子句 (¬a_i ∨ ¬b_j) 即 (¬a_i ∨ a_j)
   - ...（共 4 种组合）
3. Tarjan SCC 判定
4. 可满足 → d 可行, 扩大 d
   不可满足 → d 不可行, 缩小 d
```

**C++ 代码**：

```cpp
// 二分 d, 对每个 d 建 2-SAT 并判定
bool check(double d) {
    // 清空邻接表
    for (int i = 0; i < n; i++)
        for (int j = i + 1; j < n; j++) {
            // 枚举 (i 选 0/1, j 选 0/1) 四种组合
            for (int a = 0; a < 2; a++)
                for (int b = 0; b < 2; b++)
                    if (conflict(pos[i][a], pos[j][b], d))
                        add_clause(i, !a, j, !b);  // NAND
        }
    // Tarjan + 判定
    return satisfiable(n);
}
```

**核心启示**：**2-SAT + 二分答案** 是一种强力组合。二分参数控制冲突的严格程度，2-SAT 判定当前参数下是否存在合法方案。这种模式适用于"每个元素有两种选择 + 某些选择对之间会冲突 + 冲突由某个可二分的参数决定"的问题。

### 题目 6：至多选一优化建图（附加模型）

**难度**：🟠

**题意**：有 $n$ 个节点，每个节点可以选或不选。给出 $k$ 组约束，每组包含 $s$ 个节点，要求组内**至多选一个**。判断能否满足所有约束。

**分析**：

朴素做法对组内每对节点加 NAND 约束，边数 $O(s^2)$。用前缀辅助变量优化到 $O(s)$。

```
至多选一优化建图:
  对组 {a1, a2, ..., as}:
  引入辅助变量 y1, y2, ..., y_{s-1}
  yi 表示 "a1..ai 中已经选了一个"

  蕴含边:
    ai → yi          (选了 ai 则 yi 为真)
    yi → yi+1        (单调递增)
    yi ∧ a_{i+1} 矛盾: yi → ¬a_{i+1}
                      a_{i+1} → ¬yi

  等价于: 选了前 i 个中的某一个后, 后面都不能选
  总边数: O(s) 而非 O(s^2)
```

**C++ 代码**：

```cpp
// 至多选一: nodes[] 中至多一个为真
// base: 辅助变量的起始编号
void at_most_one(int nodes[], int sz, int &base) {
    if (sz <= 1) return;
    // y[j] 对应变量编号 base+j, 共 sz-1 个
    for (int j = 0; j < sz - 1; j++) {
        int yj = base + j;
        add_edge(nodes[j]*2+1, yj*2+1);     // a_j → y_j
        if (j > 0) add_edge((base+j-1)*2+1, yj*2+1); // y_{j-1} → y_j
        add_edge(yj*2+1, nodes[j+1]*2);     // y_j → ¬a_{j+1}
        add_edge(nodes[j+1]*2+1, yj*2);     // a_{j+1} → ¬y_j
    }
    base += sz - 1;
}
```

**核心启示**：当 2-SAT 中出现"一组中至多/恰好选一"的约束时，**前缀辅助变量**可以将 $O(k^2)$ 的边数优化到 $O(k)$。这是 2-SAT 建图优化的核心技巧之一。

## 五、常见技巧与思路总结

### 技巧 1：蕴含边的逆否命题

每条蕴含 $a \Rightarrow b$ 都有逆否命题 $\lnot b \Rightarrow \lnot a$。建边时**必须同时建两条边**（原命题和逆否命题），否则 SCC 判定会出错。

```
子句 (a ∨ b) 建边：
  ¬a → b     (原)
  ¬b → a     (逆否)
两条边缺一不可！
```

### 技巧 2：强制赋值

当需要强制某个变量 $x_i = \text{true}$ 时，添加蕴含 $\lnot x_i \Rightarrow x_i$，即建边 $\bar{x_i} \to x_i$。这保证了若假设 $x_i$ 为假会立刻推出 $x_i$ 为真，形成矛盾，从而强制 $x_i$ 为真。

### 技巧 3：三选一转二选一

遇到"三种选择"的场景不要直接放弃 2-SAT。如果自由度较小（如 P3825），可以**枚举**将三选一限制为二选一。枚举量 $2^d$ 在 $d$ 较小时可以接受。

### 技巧 4：利用可达性做进一步分析

2-SAT 蕴含图 / 缩点后 DAG 的可达性蕴含了丰富的信息：

- 若 $x_i$ 的真节点可达 $x_j$ 的假节点，则 $x_i$ 为真和 $x_j$ 为真**不能同时成立**
- 这可以用来统计"有多少对变量可以同时取某个值"（如 P5332）

### 技巧 5：注意节点编号方式

常见的两种编号方式：

| 方式 | $x_i$ 为真的节点 | $x_i$ 为假的节点 | 取反操作 |
|------|-----------------|-----------------|---------|
| $2i$ / $2i+1$ | $2i+1$ | $2i$ | `u ^ 1` |
| $i$ / $i+n$ | $i$ | $i+n$ | `u < n ? u+n : u-n` |

推荐第一种，因为取反只需异或 1，代码更简洁。

## 六、优化方法 / 解题框架

### 6.1 通用解题步骤

```
Step 1: 识别变量
  ├── 哪些东西有"二选一"的性质？
  └── 每个变量的两种状态分别是什么？

Step 2: 分析约束
  ├── 约束能否写成 (a ∨ b) 的形式？
  ├── 是否有 XOR / NAND / 强制赋值 等变体？
  └── 是否需要配合其他算法（二分、前缀优化等）？

Step 3: 建蕴含图
  ├── 每个变量建两个节点
  ├── 每条约束转为蕴含边（注意逆否命题）
  └── 辅助变量优化大规模约束

Step 4: Tarjan SCC + 判定
  ├── 跑 Tarjan
  ├── 检查 xi 与 ¬xi 是否同 SCC
  └── 比较 scc_id 输出方案

Step 5: 根据题意提取答案
  ├── 输出方案 / 判定可行
  └── 可能需要进一步分析蕴含图结构
```

### 6.2 什么时候想到 2-SAT

| 问题特征 | 2-SAT 的角色 |
|---------|-------------|
| 每个元素有两种选择 | 布尔变量 |
| 某些选择之间有互斥/依赖 | 蕴含边 |
| "若 A 则 B" 的约束 | 直接蕴含 |
| "A 和 B 至少/至多选一" | 标准子句 / NAND |
| "A 和 B 恰好选一" | XOR（4 条边） |
| 判定是否存在合法赋值 | 2-SAT 判定 |
| 配合二分答案 | 二分参数控制约束 |

### 6.3 复杂度速查

| 操作 | 时间复杂度 |
|------|-----------|
| 基本 2-SAT（$n$ 变量，$m$ 子句） | $O(n + m)$ |
| 2-SAT + 二分 | $O((n + m) \log W)$ |
| 至多选一朴素建边（$k$ 个元素） | $O(k^2)$ 条边 |
| 至多选一优化建边 | $O(k)$ 条边 |
| 2-SAT + bitset 可达性（$n$ 个变量） | $O(n^2 / w)$（$w$ 为字长） |
| 枚举三选一转二选一（$d$ 个自由变量） | 额外 $O(2^d)$ 次 2-SAT |

| 信号词 / 特征 | 对应方法 |
|--------------|---------|
| "两种选择 / 选或不选" | 2-SAT 变量建模 |
| "若...则..." 条件约束 | 蕴含边 |
| "不能同时选" | NAND 建边 |
| "恰好选一个" | XOR（4 条蕴含边） |
| "至少选一个" | OR（2 条蕴含边） |
| "强制选某个" | 强制赋值边 |
| "三选一但自由变量少" | 枚举 + 2-SAT |
| "最大化/最小化某参数使得方案存在" | 二分 + 2-SAT |

### 易错清单

| 错误 | 后果 | 修正 |
|------|------|------|
| 只建原命题边，漏了逆否命题 | SCC 判定错误 | 每条蕴含**必须同时建**原命题和逆否命题两条边 |
| 节点编号搞混（真/假对应哪个节点） | 全部建反 | 统一用 $2i$/$2i+1$，取反用 `^1` |
| 输出方案时 scc_id 比较方向反了 | 方案全错 | $\text{scc\_id}(\lnot x_i) < \text{scc\_id}(x_i)$ 时 $x_i = \text{true}$ |
| 至多选一用朴素 $O(k^2)$ 建边 | TLE | 使用前缀辅助变量优化到 $O(k)$ |
| 三选一直接当 3-SAT 放弃 | 漏解 | 检查自由变量数，枚举转 2-SAT |
| 多组数据不清空 | WA | 清空邻接表、dfn/low/scc_id 等所有数组 |

> **2-SAT = 把"二选一"约束建成蕴含图，Tarjan SCC 一遍判定可满足性并输出方案，配合二分答案、前缀优化建图等技巧可以解决大量组合约束问题。**
