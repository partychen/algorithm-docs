## 一、什么是二分图

**二分图**（Bipartite Graph），指的是能将所有顶点分成两个不相交的集合 $X$ 和 $Y$，使得图中每条边的两个端点分别属于 $X$ 和 $Y$ 的图。

二分图是图论中一类极其重要的结构。大量的匹配、覆盖、独立集问题都建立在二分图模型之上；同时，二分图上的很多问题都可以通过网络流高效求解。

用一句话概括：

> **把节点分成两组，所有边都"跨组"连接 —— 这就是二分图。**

### 二分图的核心定理总览

| 定理 / 结论 | 内容 | 前提 |
|-------------|------|------|
| 二分图判定 | 图是二分图 $\iff$ 不含奇环 | 无向图 |
| 最大匹配 | 匈牙利算法 / 网络流求解 | 二分图 |
| König 定理 | 最小顶点覆盖 $=$ 最大匹配 | 二分图 |
| 最大独立集 | $=\; n -$ 最大匹配 | 二分图 |
| 最小路径覆盖 | $=\; n -$ 最大匹配 | DAG |
| KM 算法 | 带权最大匹配（最优匹配） | 完备二分图 |
| 最大权匹配 | 等价于最小费用最大流 | 二分图 |

### 二分图匹配的本质

不管用哪种算法求二分图匹配，核心都基于同一个概念 —— **增广路（Augmenting Path）**：

> 从一个未匹配点出发，交替经过非匹配边和匹配边，最终到达另一个未匹配点。沿增广路取反（匹配边变非匹配、非匹配边变匹配），匹配数 $+1$。

所有匹配算法的区别仅在于：**如何高效地寻找增广路。**

## 二、核心思想与前置知识

### 2.1 二分图判定

**方法**：染色法（BFS / DFS）。尝试用两种颜色（如黑白）对图进行染色，使得每条边的两端颜色不同。如果能成功染色，则是二分图；如果某条边两端颜色相同，则不是二分图（存在奇环）。

**判定定理**：图是二分图 $\iff$ 图中不存在奇数长度的环。

```cpp
// 染色法判二分图（BFS）
int color[N]; // 0 未染色, 1 黑, 2 白
bool is_bipartite(int n) {
    memset(color, 0, sizeof color);
    for (int i = 1; i <= n; i++) {
        if (color[i]) continue;
        queue<int> q; q.push(i); color[i] = 1;
        while (!q.empty()) {
            int u = q.front(); q.pop();
            for (int v : G[u]) {
                if (!color[v]) { color[v] = 3 - color[u]; q.push(v); }
                else if (color[v] == color[u]) return false;
            }
        }
    }
    return true;
}
```

**时间复杂度**：$O(n + m)$。

### 2.2 匈牙利算法框架

**适用条件**：求二分图最大匹配。

**核心思想**：依次尝试为左部每个点找匹配。对于当前点 $u$，遍历其邻居 $v$：若 $v$ 未被匹配，则直接匹配 $(u, v)$；若 $v$ 已匹配给 $w$，则递归尝试为 $w$ 找新的匹配（即寻找增广路）。若找到增广路，匹配数 $+1$。

```
匈牙利算法流程：
1. 初始化所有点为未匹配
2. 对左部每个点 u：
   a. 清空本轮访问标记
   b. 若 dfs(u) 成功找到增广路，匹配数 +1
3. dfs(u)：
   a. 遍历 u 的邻居 v
   b. 若 v 本轮未访问，标记访问
   c. 若 v 未匹配 或 dfs(match[v]) 成功，则 match[v] = u，返回 true
   d. 所有邻居都失败，返回 false
```

```cpp
// 匈牙利算法
int match_r[N]; // 右部点的匹配对象，0 表示未匹配
bool vis[N];    // 本轮 DFS 的访问标记

bool dfs(int u) {
    for (int v : G[u]) {
        if (vis[v]) continue;
        vis[v] = true;
        if (!match_r[v] || dfs(match_r[v])) {
            match_r[v] = u;
            return true;
        }
    }
    return false;
}

int hungarian(int n_left) {
    int ans = 0;
    memset(match_r, 0, sizeof match_r);
    for (int u = 1; u <= n_left; u++) {
        memset(vis, false, sizeof vis);
        if (dfs(u)) ans++;
    }
    return ans;
}
```

**时间复杂度**：$O(nm)$，其中 $n$ 为左部点数，$m$ 为边数。

### 2.3 König 定理与推论

**König 定理**：在二分图中，**最小顶点覆盖 $=$ 最大匹配**。

- **顶点覆盖**：选择一个点集 $S$，使得每条边至少有一个端点在 $S$ 中。
- **最小顶点覆盖**：最小的这样的集合 $S$ 的大小。

**推论 1 —— 最大独立集**：

$$\text{最大独立集} = n - \text{最大匹配}$$

- **独立集**：选择一个点集 $S$，使得 $S$ 中任意两点之间没有边。
- 独立集与顶点覆盖互补：$V \setminus S$ 是覆盖 $\iff$ $S$ 是独立集。

**推论 2 —— 最小边覆盖**（无孤立点时）：

$$\text{最小边覆盖} = n - \text{最大匹配}$$

**直觉理解**：

```
König 定理的直觉：
  每条匹配边需要至少一个端点来"覆盖"它。
  最大匹配确定了最少需要多少个覆盖点 —— 恰好每条匹配边选一个端点。
  而非匹配边一定被选中的端点"顺带"覆盖了。
  因此：最小覆盖 = 最大匹配。
```

### 2.4 KM 算法框架

**适用条件**：求完备二分图的**带权最大（或最小）匹配**。即左右各 $n$ 个点，每对之间有权值，找到一个完美匹配使权值之和最大。

**核心思想**：为每个左部点维护**顶标** $lx[i]$，每个右部点维护顶标 $ly[j]$，满足 $lx[i] + ly[j] \ge w[i][j]$。在满足 $lx[i] + ly[j] = w[i][j]$ 的边（相等子图）上寻找完美匹配。若找不到，则调整顶标（降低左部顶标、升高右部顶标），扩大相等子图，直到找到完美匹配。

```
KM 算法流程：
1. 初始化：lx[i] = max(w[i][j])，ly[j] = 0
2. 对每个左部点 u：
   a. 设 slack[j] = INF
   b. 反复尝试在相等子图上寻找增广路
   c. 若失败，取 delta = min(slack[j])（未匹配的右部点）
   d. 调整顶标：访问过的左部 lx -= delta，访问过的右部 ly += delta
   e. 更新 slack，重复直到成功
3. 答案 = sum(lx[i] + ly[i])
```

```cpp
// KM 算法（O(n^3)）
int w[N][N], match_l[N], match_r[N];
int lx[N], ly[N], slack[N];
bool vl[N], vr[N];
int n;

bool dfs_km(int u) {
    vl[u] = true;
    for (int v = 1; v <= n; v++) {
        if (vr[v]) continue;
        int gap = lx[u] + ly[v] - w[u][v];
        if (gap == 0) {
            vr[v] = true;
            if (!match_r[v] || dfs_km(match_r[v])) {
                match_r[v] = u; match_l[u] = v;
                return true;
            }
        } else slack[v] = min(slack[v], gap);
    }
    return false;
}

ll km() {
    memset(match_r, 0, sizeof match_r);
    memset(ly, 0, sizeof ly);
    for (int i = 1; i <= n; i++) {
        lx[i] = *max_element(w[i]+1, w[i]+n+1);
    }
    for (int u = 1; u <= n; u++) {
        while (true) {
            memset(vl, false, sizeof vl);
            memset(vr, false, sizeof vr);
            fill(slack+1, slack+n+1, INT_MAX);
            if (dfs_km(u)) break;
            int d = INT_MAX;
            for (int j = 1; j <= n; j++)
                if (!vr[j]) d = min(d, slack[j]);
            for (int i = 1; i <= n; i++) if (vl[i]) lx[i] -= d;
            for (int j = 1; j <= n; j++)
                if (vr[j]) ly[j] += d;
                else slack[j] -= d;
        }
    }
    ll res = 0;
    for (int i = 1; i <= n; i++) res += w[i][match_l[i]];
    return res;
}
```

**时间复杂度**：$O(n^3)$。

## 三、分类与变体详解

### 3.1 二分图最大匹配

这是二分图问题中最基础的类型。给定一个二分图，求最大匹配数。

**算法选择**：
- 匈牙利算法：$O(nm)$，实现简单，适合 $n, m \le 10^3 \sim 10^4$。
- Hopcroft-Karp 算法：$O(m\sqrt{n})$，更高效，适合数据较大的情况。
- 网络流（Dinic）：$O(m\sqrt{n})$（二分图上），通用性更强。

**注意事项**：
- 匈牙利算法每轮 DFS 必须清空当轮的 `vis` 标记。
- 匹配数组 `match_r[v]` 记录右部点 $v$ 匹配的左部点。
- 左右两部的编号互不重叠。

### 3.2 最小顶点覆盖

**问题**：选最少的点，覆盖所有的边。

由 König 定理：**最小顶点覆盖 $=$ 最大匹配**。

因此只需求最大匹配即可得到答案。若需要输出具体方案，可以在最大匹配的基础上，通过交替标记法构造覆盖集：

```
构造最小顶点覆盖：
1. 求最大匹配
2. 从所有未匹配的左部点出发，沿"非匹配边→右部→匹配边→左部"交替 BFS
3. 将访问到的点标记
4. 覆盖集 = 未标记的左部匹配点 ∪ 已标记的右部匹配点
```

### 3.3 最大独立集

**问题**：选最多的点，使得选出的点之间没有边相连。

$$\text{最大独立集} = n - \text{最大匹配}$$

直觉：独立集和顶点覆盖互补。不在覆盖集中的点就构成了独立集。

### 3.4 最小路径覆盖（DAG）

**问题**：在一个 DAG 中，用最少的不相交路径覆盖所有点。

**方法**：将每个点 $u$ 拆为出点 $u_{\text{out}}$ 和入点 $u_{\text{in}}$。对于原图的每条边 $(u, v)$，在二分图中连 $u_{\text{out}} \to v_{\text{in}}$。然后求最大匹配。

$$\text{最小路径覆盖} = n - \text{最大匹配}$$

**直觉理解**：
- 初始时每个点自成一条路径，共 $n$ 条路径。
- 每找到一条匹配边 $(u_{\text{out}}, v_{\text{in}})$，就将点 $u$ 和点 $v$ 所在的路径合并为一条，路径数 $-1$。
- 最大匹配使得合并次数最多，路径数最少。

```
最小路径覆盖建模：
  原图 DAG：n 个点
  二分图：左部 {1_out, 2_out, ..., n_out}，右部 {1_in, 2_in, ..., n_in}
  对原图每条边 (u, v)：连 u_out → v_in
  答案 = n - 最大匹配
```

### 3.5 带权二分图匹配（KM）

**问题**：在完备二分图中（左右各 $n$ 个点，每对之间有权值），找一个完美匹配使得权值之和最大（或最小）。

**算法**：KM 算法，时间复杂度 $O(n^3)$。

**求最小权匹配**的方法：将所有边权取反（$w[i][j] \to -w[i][j]$），用 KM 求最大权匹配，结果取反即为最小权匹配。

**注意**：
- 若左右部大小不等，用虚拟节点补齐，虚拟边权 $= 0$。
- KM 要求存在完美匹配。若不存在完美匹配但仍需求最大权匹配，可将不存在的边权设为 $-\infty$。

### 3.6 二分图与网络流

二分图匹配问题可以转化为网络流问题：

```
二分图 → 网络流建模：
1. 建超级源 S 和超级汇 T
2. S → 左部每个点，容量 1
3. 左部 → 右部（按原图边），容量 1
4. 右部每个点 → T，容量 1
5. 最大匹配 = 最大流
```

**网络流的优势**：
- 更通用：可以处理多重匹配（容量 > 1）、带权匹配（费用流）等变体。
- Dinic 算法在二分图上复杂度为 $O(m\sqrt{n})$。
- 费用流可以求带权最大匹配（等价于 KM，但更灵活）。

| 二分图问题 | 网络流等价 |
|-----------|-----------|
| 最大匹配 | 最大流 |
| 最大权匹配 | 最小费用最大流（费用取反） |
| 最小顶点覆盖 | 最小割 $=$ 最大流 |
| 最大独立集 | $n -$ 最大流 |

## 四、11 道经典例题详解

### 题目 1：【模板】二分图匹配（洛谷 P3386）

> 给定一个二分图，左部 $n_1$ 个点，右部 $n_2$ 个点，$m$ 条边。求最大匹配数。$n_1, n_2 \le 500$，$m \le 5 \times 10^4$。

**难度**：🟢

**分析**：

```
二分图最大匹配的标准模板题。
直接使用匈牙利算法。
对左部每个点尝试增广，统计成功次数即为最大匹配。
时间复杂度 O(n*m)，对于 n=500, m=50000 完全足够。
```

**C++ 代码**：

```cpp
int match_r[N];
bool vis[N];
vector<int> G[N];

bool dfs(int u) {
    for (int v : G[u]) {
        if (vis[v]) continue;
        vis[v] = true;
        if (!match_r[v] || dfs(match_r[v])) {
            match_r[v] = u;
            return true;
        }
    }
    return false;
}

int solve(int n1) {
    int ans = 0;
    for (int u = 1; u <= n1; u++) {
        memset(vis, false, sizeof vis);
        if (dfs(u)) ans++;
    }
    return ans;
}
```

**核心启示**：匈牙利算法是二分图最大匹配的基础板子。每次增广前必须清空 `vis` 数组，保证同一右部点在一轮中只被访问一次。

### 题目 2：飞行员配对方案问题（洛谷 P2756）

> $n$ 个飞行员分为外籍和英国籍两组。给出若干可配对关系，求最大配对数，并输出配对方案。

**难度**：🟢

**分析**：

```
经典二分图最大匹配 + 输出方案。
两组飞行员分别作为左部和右部，可配对关系为边。
用匈牙利算法求最大匹配，匹配数即为答案。
方案直接从 match_r[] 数组中读取。
```

**C++ 代码**：

```cpp
int match_r[N];
bool vis[N];
vector<int> G[N];

bool dfs(int u) {
    for (int v : G[u]) {
        if (vis[v]) continue;
        vis[v] = true;
        if (!match_r[v] || dfs(match_r[v])) {
            match_r[v] = u; return true;
        }
    }
    return false;
}
// 求完最大匹配后，遍历 match_r[] 输出配对方案
```

**核心启示**：匈牙利算法天然记录匹配方案 —— `match_r[v] = u` 表示右部点 $v$ 与左部点 $u$ 匹配。输出方案只需遍历该数组。

### 题目 3：[ZJOI2007] 矩阵游戏（洛谷 P1129）

> 给定一个 $n \times n$ 的 01 矩阵，可以交换任意两行或任意两列。问能否通过若干次操作使主对角线全为 1。$n \le 200$。

**难度**：🟡

**分析**：

```
主对角线全为 1 意味着：存在一种行列排列，使得 a[i][i] = 1 对所有 i 成立。
这等价于：对于每一行 i，都能找到唯一的一列 j，使得原矩阵 a[i][j] = 1，且 j 各不相同。
这正是一个完美匹配问题！
将行看作左部、列看作右部，若 a[i][j] = 1 则连边 i → j。
问题转化为：是否存在完美匹配。
```

**C++ 代码**：

```cpp
// 建图：若 a[i][j] == 1，连边 i → j
// 匈牙利算法求最大匹配
// 若最大匹配 == n，输出 Yes；否则 No
int match_r[N]; bool vis[N];
bool dfs(int u) {
    for (int v : G[u]) {
        if (vis[v]) continue;
        vis[v] = true;
        if (!match_r[v] || dfs(match_r[v]))
            { match_r[v] = u; return true; }
    }
    return false;
}
bool check(int n) {
    int cnt = 0;
    for (int i = 1; i <= n; i++) {
        memset(vis, false, sizeof vis);
        if (dfs(i)) cnt++;
    }
    return cnt == n;
}
```

**核心启示**：矩阵行列交换使对角线全 1 $\iff$ 二分图完美匹配。将"排列"问题转化为匹配问题是经典的建模技巧。遇到"每行恰好选一列"的结构，就要想到二分图匹配。

### 题目 4：运动员最佳匹配问题（洛谷 P1559）

> 有 $n$ 个男运动员和 $n$ 个女运动员，给出两个 $n \times n$ 矩阵 $P$ 和 $Q$，第 $i$ 个男运动员和第 $j$ 个女运动员配对的价值为 $P[i][j] \times Q[j][i]$。求最优配对使总价值最大。$n \le 20$。

**难度**：🟡

**分析**：

```
这是一个带权二分图最大匹配问题。
左部为男运动员，右部为女运动员。
边权 w[i][j] = P[i][j] * Q[j][i]。
直接使用 KM 算法求最大权完美匹配。
n ≤ 20，KM 的 O(n^3) 绰绰有余。
```

**C++ 代码**：

```cpp
int w[N][N]; // w[i][j] = P[i][j] * Q[j][i]
int lx[N], ly[N], match_r[N], slack[N];
bool vl[N], vr[N];

bool dfs_km(int u) {
    vl[u] = true;
    for (int v = 1; v <= n; v++) {
        if (vr[v]) continue;
        int gap = lx[u] + ly[v] - w[u][v];
        if (gap == 0) {
            vr[v] = true;
            if (!match_r[v] || dfs_km(match_r[v]))
                { match_r[v] = u; return true; }
        } else slack[v] = min(slack[v], gap);
    }
    return false;
}
// 外层循环：对每个左部点调用 KM 的增广 + 调标流程
```

**核心启示**：带权二分图匹配就是 KM 算法的直接应用。注意边权的计算可能是两个矩阵的乘积，建图时预处理即可。

### 题目 5：[HEOI2012] 朋友圈（洛谷 P2423）

> 两个国家 A 和 B，A 国内部同性别不是朋友、异性别是朋友；B 国内部同性别是朋友、异性别不是朋友。两国之间给出一些朋友关系。求所有人中的最大朋友圈（最大团）大小。

**难度**：🔴

**分析**：

```
关键观察：
1. A 国内部：同性别互不为友，异性别互为友 → A 国内部是一个二分图（男/女两部分）
2. B 国内部：同性别互为友，异性别互不为友 → B 国可以看成两个团（男团+女团），最大独立集最多选 2 人（一男一女）
3. 因此 B 国中最多选 2 个人进入朋友圈。

枚举 B 国选 0/1/2 人的情况：
- B 选 0 人：A 国的最大团。但 A 国内部是二分图的补图，最大团 = 补图最大独立集。
  由于 A 内部二分图较特殊，需要进一步分析。
- B 选 1 人 b：A 中与 b 为朋友的人构成集合 S，在 S 中求最大团。
- B 选 2 人 b1, b2（必须异性别）：A 中同时与 b1, b2 为朋友的人构成集合 S，在 S 中求最大团。

A 国中某子集的最大团 = 该子集在 A 国"非友关系图"上的最大独立集。
A 国非友关系图是二分图（同性别之间有边），所以最大独立集 = |S| - 最大匹配。

算法：枚举 B 国选的 2 人（或更少），对 A 国求二分图最大独立集。
优化：B 国同性别人互为朋友，所以团中最多 1 男 1 女。
枚举 B 中的一对(b1, b2)，求出 A 中与二者均为友的同性别子集之间的最大匹配。
```

**C++ 代码**：

```cpp
// 枚举 B 国选的两个人（一男一女），或 0/1 个
// 对 A 国中与所选 B 国人都是朋友的人建二分图
// 求最大独立集 = |子集| - 匈牙利最大匹配
// 答案 = max(B选的人数 + 最大独立集)

int solve() {
    int ans = 0;
    // 枚举 B 选 0 人
    ans = max(ans, (int)A.size() - hungarian(A));
    // 枚举 B 选 1 人 b
    for (int b : B) {
        auto S = friends_in_A(b);
        ans = max(ans, 1 + (int)S.size() - hungarian(S));
    }
    // 枚举 B 选 2 人 (b1, b2)，b1 b2 异性别且互为朋友
    for (auto [b1,b2] : B_pairs) {
        auto S = friends_in_A(b1) ∩ friends_in_A(b2);
        ans = max(ans, 2 + (int)S.size() - hungarian(S));
    }
    return ans;
}
```

**核心启示**：最大团问题一般是 NP-hard，但利用特殊图结构（二分图的补图）可以转化为最大独立集问题，进而用 König 定理求解。解题关键在于分析图的结构性质，将难问题拆解为可解的子问题。

### 题目 6：最小路径覆盖问题（洛谷 P2764）

> 给定一个 DAG，求用最少的不相交路径覆盖所有顶点。输出路径方案和最小路径数。$n \le 150$，$m \le 6000$。

**难度**：🟡

**分析**：

```
经典的"DAG 最小路径覆盖"模型。
将每个点 u 拆为出点 u（左部）和入点 u'（右部）。
对原图每条边 (u, v)，在二分图中连 u → v'。
最小路径覆盖 = n - 最大匹配。

输出方案：
  从每条路径的起点出发，沿匹配边依次输出路径上的点。
  路径起点 = 左部中未被匹配的点。
```

**C++ 代码**：

```cpp
// 拆点建二分图，左部 1..n（出点），右部 1..n（入点）
// 对原图每条边 (u,v)：G[u].push_back(v)（左部 u 连右部 v）
// 匈牙利求最大匹配
int match_r[N]; // match_r[v] = u 表示入点 v 与出点 u 匹配
bool vis[N];

bool dfs(int u) {
    for (int v : G[u]) {
        if (vis[v]) continue;
        vis[v] = true;
        if (!match_r[v] || dfs(match_r[v]))
            { match_r[v] = u; return true; }
    }
    return false;
}
// 答案 = n - 匹配数
// 输出方案：按 match 链向后走输出每条路径
```

**核心启示**：DAG 最小路径覆盖是二分图匹配的经典应用。拆点建模的核心思想是：一条匹配边 $(u, v')$ 表示在路径覆盖中 $u$ 的后继是 $v$，从而将路径合并转化为匹配。

### 题目 7：[HEOI2016/TJOI2016] 游戏（洛谷 P2825）

> $n \times m$ 的网格地图上有空地、软石头和硬石头。在空地上放炸弹，炸弹可以炸同行同列的所有空地，遇硬石头阻挡。软石头可以被穿透（但不能放炸弹）。同一格不能同时被两颗炸弹炸到。求最多能放多少炸弹。

**难度**：🟠

**分析**：

```
经典的"棋盘放置"转"二分图匹配"问题。
关键观察：每颗炸弹影响所在的"行段"和"列段"。
  - "行段"：同一行中被硬石头分隔的连续区域（空地+软石头）。
  - "列段"：同一列中被硬石头分隔的连续区域。
每个空地恰好属于一个行段和一个列段。
放了一颗炸弹，该行段和该列段就各被占用了（不能再放）。
因此：
  左部 = 所有行段，右部 = 所有列段。
  对每个空地 (i,j)，连边：其所属行段 → 其所属列段。
  最大匹配 = 最多能放的炸弹数。

注意：软石头不阻挡爆炸传播，但不能放炸弹。
硬石头阻挡爆炸传播，用于分割行段/列段。
```

**C++ 代码**：

```cpp
int row_id[N][N], col_id[N][N]; // 每个格子所属的行段/列段编号
int match_r[MAXSEG]; bool vis[MAXSEG];
vector<int> G[MAXSEG]; // 行段 → 列段

// 预处理行段和列段编号
// 对每个空地 (i,j)，连边 row_id[i][j] → col_id[i][j]
// 匈牙利求最大匹配

bool dfs(int u) {
    for (int v : G[u]) {
        if (vis[v]) continue;
        vis[v] = true;
        if (!match_r[v] || dfs(match_r[v]))
            { match_r[v] = u; return true; }
    }
    return false;
}
// 答案 = 最大匹配数
```

**核心启示**：网格放置问题转二分图匹配的经典套路是：将行约束和列约束分别作为二分图的两部，每个可放置位置对应一条边。最大匹配保证了每个行段、列段最多被占用一次。注意软/硬石头对"段"划分的不同影响。

### 题目 8：[USACO11NOV] Cow Steeplechase（洛谷 P3033）

> 给定 $n$ 条线段，每条线段要么水平要么竖直。任意两条平行线段不交。求最多能保留多少条线段使得它们互不相交。$n \le 250$。

**难度**：🟡

**分析**：

```
水平线段和竖直线段天然构成二分图的两部。
两条线段有交叉当且仅当：一水平一竖直，且它们在几何上相交。
"最多保留不相交的线段" = 最大独立集。
由 König 定理：最大独立集 = n - 最大匹配。

建二分图：
  左部 = 水平线段，右部 = 竖直线段。
  若水平线段 h 与竖直线段 v 相交，连边 h → v。
  最大匹配 = 最小顶点覆盖 = 最少需要删掉的线段数。
  答案 = n - 最大匹配。
```

**C++ 代码**：

```cpp
// 判断水平线段 h(x1,y,x2,y) 与竖直线段 v(x,y1,x,y2) 是否相交
bool intersect(Seg &h, Seg &v) {
    return h.x1 <= v.x && v.x <= h.x2
        && v.y1 <= h.y && h.y <= v.y2;
}

// 左部 = 水平线段，右部 = 竖直线段
// 若相交则连边
// 匈牙利求最大匹配
// 答案 = n - 最大匹配
```

**核心启示**：水平/竖直线段相交问题天然具有二分图结构。"最多保留不冲突的元素"就是最大独立集问题。遇到两类对象间的冲突关系，优先考虑二分图模型。

### 题目 9：[HAOI2017] 新型城市化（洛谷 P3731）

> $n$ 个城市，给出 $m$ 对"不和谐"关系（即这些城市对之间不能同属一个商业集团）。已知不和谐关系构成一个二分图。一个商业集团是一个完全子图（团）。求每对不和谐关系中，删掉哪些对关系后最大团会增大（即该关系是"关键"的）。

**难度**：🟣

**分析**：

```
不和谐关系构成二分图。最大团 = 补图的最大团 = 原图（二分图）的最大独立集 = n - 最大匹配。
删掉一条边后最大独立集增大 ↔ 删掉该边后最大匹配减小。
即：该边是所有最大匹配中的"必须边"（出现在每个最大匹配中）。

判断一条边是否是最大匹配的必须边：
1. 先求一个最大匹配。
2. 对于匹配边 (u, v)：若在残量网络中 u 和 v 不在同一个强连通分量中，
   则该边是必须边。
3. 非匹配边一定不是必须边。

具体方法：
  建网络流图（S → 左部 → 右部 → T），求最大流（= 最大匹配）。
  在残量网络上缩点（Tarjan 求 SCC）。
  匹配边 (u, v) 是必须边 ↔ u 和 v 在残量网络中不属于同一个 SCC。
```

**C++ 代码**：

```cpp
// 1. 建网络流，求最大流（= 最大匹配）
// 2. 在残量网络图上跑 Tarjan 求 SCC
// 3. 对每条匹配边 (u, v)：
//    若 scc[u] != scc[v]，则该边是必须边，输出

// Dinic 最大流 + Tarjan SCC
// 最终输出所有必须边

void solve() {
    int maxflow = dinic(S, T);
    tarjan_on_residual();
    for (auto &[u, v] : edges) {
        // 判断是否是匹配边且 scc[u] != scc[v]
        if (is_matched(u, v) && scc[u] != scc[v])
            output(u, v);
    }
}
```

**核心启示**：判断二分图最大匹配的"必须边"需要结合网络流和强连通分量。残量网络中，若匹配边两端点在不同 SCC 中，则该匹配边不可替代。这是网络流高级应用的经典问题。

### 题目 10：分配问题（洛谷 P4014）

> $n$ 个人分配 $n$ 件工作，第 $i$ 个人做第 $j$ 件工作的费用为 $c[i][j]$。求总费用最小和最大的分配方案。$n \le 100$。

**难度**：🟡

**分析**：

```
这是带权二分图匹配的经典题。
每个人恰好做一件工作，每件工作恰好给一个人 → 完美匹配。
求最小/最大权完美匹配。

方法一：KM 算法。
  最大权：直接 KM。
  最小权：边权取反后 KM，结果取反。

方法二：费用流。
  S → 人（容量 1，费用 0）。
  人 → 工作（容量 1，费用 c[i][j]）。
  工作 → T（容量 1，费用 0）。
  最小费用最大流 = 最小权匹配。
  求最大权：费用取反跑最小费用最大流，结果取反。
```

**C++ 代码**：

```cpp
// 费用流写法：
// S = 0, T = 2*n+1
// S → i (cap=1, cost=0)
// i → n+j (cap=1, cost=c[i][j])
// n+j → T (cap=1, cost=0)
// 最小费用最大流 → 最小总费用

void build(int n) {
    S = 0; T = 2*n+1;
    for (int i = 1; i <= n; i++) {
        addedge(S, i, 1, 0);
        addedge(n+i, T, 1, 0);
        for (int j = 1; j <= n; j++)
            addedge(i, n+j, 1, c[i][j]);
    }
}
// 跑 MCMF 得到最小费用
// 求最大费用：将 cost 取反，跑 MCMF，结果取反
```

**核心启示**：分配问题是带权二分图匹配的原型。KM 和费用流都能求解。费用流更通用（能处理不等大小、多重匹配等），KM 在完备二分图上效率更高。

### 题目 11：[COCI2017-2018#5] Planinarenje（洛谷 P4617）

> 给定一个二分图，左右各 $n$ 个点，$m$ 条边。对于每个左部点 $u$，判断：是否存在一个最大匹配使得 $u$ 是未匹配的。$n \le 1000$，$m \le  10^5$。

**难度**：🟠

**分析**：

```
问题等价于：对每个左部点 u，判断 u 是否是"非必须点"（即存在不包含 u 的最大匹配）。

方法：
1. 先求一个最大匹配。
2. 对于未匹配的左部点：显然存在不包含它的最大匹配（当前的就是），答案为 Yes。
3. 对于已匹配的左部点 u（匹配了 v = match_l[u]）：
   删掉 u 后最大匹配是否仍等于原来的值？
   等价于：在当前匹配中去掉 (u, v) 后，能否找到一条从 v 出发的增广路
   （不经过 u），把 v 重新匹配。

更高效的做法：构建残量网络，用 Tarjan 求 SCC。
  左部匹配点 u 是非必须的 ↔ u 与某个未匹配的左部点在同一 SCC 中，
  或 match_l[u] 与某个未匹配的右部点在同一 SCC 中。

实际上判断条件：左部点 u（已匹配 v）不是必须的 ↔
  在残量网络中，v 可以从某个未匹配右部点到达（通过交替路径）。
```

**C++ 代码**：

```cpp
// 1. 匈牙利求最大匹配
// 2. 建残量网络：
//    匹配边 u→v（右到左方向为残量边）
//    非匹配边 v→u（左到右方向为残量边）
//    未匹配的左部点为"自由点"
// 3. Tarjan 求 SCC
// 4. 左部点 u 可以不在最大匹配中 ↔
//    u 未匹配，或 u 与某个未匹配左部点在同一 SCC

void solve() {
    hungarian();
    build_residual_graph();
    tarjan();
    for (int u = 1; u <= n; u++) {
        if (!matched[u]) { puts("Yes"); continue; }
        // 检查 u 是否与未匹配左部点在同一 SCC
        // （或通过交替路径可达未匹配点）
        puts(can_be_free[u] ? "Yes" : "No");
    }
}
```

**核心启示**：判断某点是否必须在最大匹配中，需要分析残量网络的连通结构。通过 SCC 缩点可以高效判断：若匹配点能通过交替路径到达自由点，则该点可以不在匹配中。

## 五、常见技巧与思路总结

### 1. 建模技巧

| 技巧 | 说明 | 典型应用 |
|------|------|---------|
| 行列建模 | 行作左部、列作右部 | 矩阵放置、矩阵游戏 |
| 行段/列段 | 被障碍分隔的连续区域 | 棋盘放炸弹、放车 |
| 拆点建二分图 | DAG 中每点拆为出点和入点 | 最小路径覆盖 |
| 补图转化 | 最大团 $\to$ 补图最大独立集 | 朋友圈、冲突关系 |
| 网络流建模 | 加源汇转为流问题 | 多种匹配变体 |

### 2. 常见思维套路

- **"每行选一个、每列选一个"** → 二分图匹配
- **"互不冲突地最多选多少"** → 最大独立集 $= n -$ 最大匹配
- **"最少选多少覆盖所有关系"** → 最小顶点覆盖 $=$ 最大匹配
- **"DAG 最少路径覆盖所有点"** → 拆点 + 二分图匹配
- **"带权一一对应"** → KM 算法或费用流
- **"删边后匹配是否变化"** → 残量网络 + SCC 分析

### 3. 常见陷阱

- **左右部编号重叠**：匈牙利算法中左右部的编号空间要分开，或者 `match_r[]` 只记录右部点。
- **忘记清空 vis 数组**：匈牙利算法每轮 DFS 前必须清空。
- **KM 的 slack 更新**：每次增广失败后，除了调标还要更新 `slack` 数组，否则会死循环或得到错误答案。
- **最小路径覆盖要求路径不相交**：若允许相交，则需要用 Floyd 传递闭包后再建二分图。
- **二分图判定**：若图不连通，需要对每个连通分量分别染色。

## 六、优化方法 / 解题框架

### 解题框架

```
1. 判断是否为二分图问题：
   - 是否存在天然的两类对象？
   - 约束是否只在两类对象之间？
   - 是否可以转化为匹配/覆盖/独立集？

2. 确定问题类型：
   - 最大匹配？→ 匈牙利 / Hopcroft-Karp / Dinic
   - 最小覆盖？→ König 定理（= 最大匹配）
   - 最大独立集？→ n - 最大匹配
   - 最小路径覆盖（DAG）？→ 拆点 + 二分图匹配
   - 带权匹配？→ KM / 费用流
   - 判断边/点的必要性？→ 残量网络 + SCC

3. 建图：
   - 确定左部和右部的含义
   - 确定边的含义（何时连边）
   - 是否需要拆点、补虚拟节点

4. 求解：
   - n ≤ 500：匈牙利 O(nm)
   - n ≤ 10^5：Dinic O(m√n) / Hopcroft-Karp
   - 带权 n ≤ 300：KM O(n^3)
   - 带权 n 较大：费用流
```

### 算法选择指南

```
问题类型                算法                适用范围
────────────────────────────────────────────────────────
无权最大匹配            匈牙利              n ≤ 1000
无权最大匹配            Hopcroft-Karp       n ≤ 10^5
无权最大匹配            Dinic               通用
最小顶点覆盖            König (= 最大匹配)  二分图
最大独立集              n - 最大匹配        二分图
最小路径覆盖            拆点 + 匹配         DAG
带权最大匹配            KM                  n ≤ 300
带权最大匹配            费用流              通用
必须边/必须点判断       最大匹配 + SCC       二分图
```

### 复杂度一览

| 算法 | 时间复杂度 | 适用场景 |
|------|-----------|---------|
| 匈牙利 | $O(nm)$ | 小规模二分图匹配 |
| Hopcroft-Karp | $O(m\sqrt{n})$ | 大规模无权匹配 |
| Dinic（二分图） | $O(m\sqrt{n})$ | 通用二分图匹配 |
| KM | $O(n^3)$ | 带权完美匹配 |

### 二分图知识图谱

│   └── 最小边覆盖 = n - 最大匹配（无孤立点）
├── 最小路径覆盖（DAG）
│   └── 拆点 + 二分图匹配，答案 = n - 最大匹配

| König 定理 | 最小覆盖 $=$ 最大匹配：每条匹配边选一个端点就够了 |
| 最大独立集 | $n -$ 最大匹配：覆盖的补集就是独立集 |
| 最小路径覆盖 | 拆点匹配：每条匹配边合并一次路径，$n -$ 匹配 $=$ 路径数 |
| KM 算法 | 顶标 + 增广：维护相等子图，调标扩图，直到完美匹配 |
| 二分图与网络流 | 加源汇、容量 1，最大流 $=$ 最大匹配 |

### 做题心法

1. **先识别二分图结构**：两类对象 + 跨类约束 → 二分图。行与列、人与工作、时间与资源等都是常见的二分结构。
2. **再确定问题类型**：最大匹配 / 最小覆盖 / 最大独立集 / 带权匹配 / 路径覆盖 — 决定算法和定理。
3. **善用 König 定理**：很多看似复杂的覆盖、独立集问题，通过 König 定理都能归结为最大匹配。
4. **网络流是万能后盾**：当匈牙利/KM 不够用时（如多重匹配、复杂约束），转为网络流建模。
5. **高级分析用 SCC**：必须边、必须点等问题需要在残量网络上做 Tarjan 分析。
