## 一、什么是图的连通性

**图的连通性** 是图论中最基础也最重要的概念之一。它描述了图中节点之间"可达"的关系，不同类型的连通性反映了图的不同结构特征。

### 强连通

在 **有向图** 中，若从任意节点 $u$ 到任意节点 $v$ 都存在一条有向路径，则称该图是 **强连通** 的。一个有向图的 **极大强连通子图** 称为 **强连通分量（SCC, Strongly Connected Component）**。

```
示例：
  1 → 2
  ↑   ↓
  4 ← 3 → 5

{1, 2, 3, 4} 构成一个 SCC
{5} 单独为一个 SCC
```

### 弱连通

在有向图中，若忽略边的方向后，任意两点间存在路径，则称该图是 **弱连通** 的。弱连通是比强连通更弱的条件。

### 双连通

在 **无向图** 中，双连通有两种定义：

| 类型 | 定义 | 直观含义 |
|------|------|----------|
| **边双连通** | 任意删去一条边后，图仍然连通 | 图中不存在桥（割边） |
| **点双连通** | 任意删去一个点（及其关联边）后，图仍然连通 | 图中不存在割点 |

```
边双连通分量示例：
  1 - 2
  |   |     5 - 6
  4 - 3 --- |   |
            7 - 8

{1,2,3,4} 是一个边双连通分量
{5,6,7,8} 是一个边双连通分量
边 3-5 是桥
```

### 各类连通性对比

| 性质 | 适用图类型 | 核心概念 | 关键算法 |
|------|-----------|---------|---------|
| 强连通分量 | 有向图 | SCC | Tarjan / Kosaraju |
| 割点 | 无向图 | 删去后图不连通的点 | Tarjan |
| 桥（割边） | 无向图 | 删去后图不连通的边 | Tarjan |
| 边双连通分量 | 无向图 | 不含桥的极大连通子图 | 删桥求连通分量 |
| 点双连通分量 | 无向图 | 不含割点的极大二连通子图 | 栈维护 Tarjan |

## 二、核心思想与前置知识

### 2.1 DFS 树与时间戳

在图上进行 DFS 时，会产生一棵 **DFS 树**。对每个节点记录两个时间戳：

- **$dfn[u]$**（发现时间）：节点 $u$ 第一次被访问的时刻
- **$low[u]$**（追溯值）：从 $u$ 出发，通过子树中的边和**至多一条非树边**所能到达的最小 $dfn$ 值

```
DFS 树中的边分类（有向图）：
- 树边（tree edge）：DFS 过程中经过的边
- 返祖边（back edge）：指向祖先的边 → 形成环路
- 前向边（forward edge）：指向后代的非树边
- 横叉边（cross edge）：指向已完成节点的边

DFS 树中的边分类（无向图）：
- 树边（tree edge）：DFS 过程中经过的边
- 返祖边（back edge）：连接后代到祖先的非树边
```

**$low$ 值的更新规则**：

初始令 $low[u] = dfn[u]$，然后：
1. 若 $(u, v)$ 是树边，则 $low[u] = \min(low[u], low[v])$
2. 若 $(u, v)$ 是返祖边（$v$ 已在栈中 / 是祖先），则 $low[u] = \min(low[u], dfn[v])$

### 2.2 Tarjan 求 SCC 框架

Tarjan 算法是求解有向图强连通分量的经典算法，利用 DFS 树和栈来识别 SCC。

**核心思想**：
- 维护一个栈，DFS 时将节点入栈
- 当某节点 $u$ 满足 $dfn[u] = low[u]$ 时，栈中 $u$ 及其上方所有节点构成一个 SCC
- 因为 $dfn[u] = low[u]$ 意味着 $u$ 是这个 SCC 中最早被发现的节点（SCC 的"根"）

```
Tarjan 求 SCC 流程：
1. 初始化时间戳 timer = 0
2. 对每个未访问的节点调用 tarjan(u)
3. tarjan(u):
   a. dfn[u] = low[u] = ++timer，将 u 入栈
   b. 遍历 u 的所有出边 u→v：
      - v 未访问：递归 tarjan(v)，low[u] = min(low[u], low[v])
      - v 在栈中：low[u] = min(low[u], dfn[v])
   c. 若 dfn[u] == low[u]：弹栈直到弹出 u，弹出的所有节点构成一个 SCC
```

**C++ 代码**：
```cpp
// dfn[], low[], stk[], inStk[], sccId[], sccCnt, timer
void tarjan(int u) {
    dfn[u] = low[u] = ++timer;
    stk[++top] = u; inStk[u] = true;
    for (int v : adj[u]) {
        if (!dfn[v]) {
            tarjan(v);
            low[u] = min(low[u], low[v]);
        } else if (inStk[v]) {
            low[u] = min(low[u], dfn[v]);
        }
    }
    if (dfn[u] == low[u]) {
        ++sccCnt;
        int x;
        do {
            x = stk[top--];
            inStk[x] = false;
            sccId[x] = sccCnt;
        } while (x != u);
    }
}
```

**时间复杂度**：$O(V + E)$

**注意**：Tarjan 算法中 SCC 的编号顺序恰好是 **反拓扑序**（最后发现的 SCC 编号最小），这在缩点后做 DP 时非常有用。

### 2.3 缩点建 DAG

将每个 SCC 缩为一个"超级节点"，原图中跨越不同 SCC 的边保留（去重），得到一个 **DAG（有向无环图）**。

```
缩点流程：
1. 用 Tarjan 求出所有 SCC，sccId[u] 记录 u 所在 SCC 编号
2. 遍历原图每条边 (u, v)：
   若 sccId[u] ≠ sccId[v]，在新 DAG 中加入边 sccId[u] → sccId[v]
3. 可对新 DAG 做拓扑排序、DP 等操作
```

**C++ 代码**：
```cpp
// 缩点后建新图 dag[], 合并 SCC 权值 sccW[]
for (int u = 1; u <= n; u++)
    sccW[sccId[u]] += w[u];         // 合并节点权值
for (int u = 1; u <= n; u++)
    for (int v : adj[u])
        if (sccId[u] != sccId[v])
            dag[sccId[u]].push_back(sccId[v]);
```

### 2.4 Tarjan 求割点与桥

在 **无向图** 上使用 Tarjan 算法，可以求出割点和桥。

**割点判定**：
对于非根节点 $u$：若存在子节点 $v$ 使得 $low[v] \ge dfn[u]$，则 $u$ 是割点。
对于根节点：若 DFS 树中有 $\ge 2$ 个子节点，则根节点是割点。

**直觉理解**：$low[v] \ge dfn[u]$ 意味着 $v$ 的子树不经过 $u$ 就无法到达 $u$ 的祖先，因此删除 $u$ 会使 $v$ 的子树与图的其余部分断开。

**桥（割边）判定**：
若树边 $(u, v)$ 满足 $low[v] > dfn[u]$，则 $(u, v)$ 是桥。

**注意区别**：割点是 $\ge$，桥是 $>$。

**C++ 代码（割点）**：
```cpp
void tarjan(int u, int fa) {
    dfn[u] = low[u] = ++timer;
    int child = 0;
    for (int v : adj[u]) {
        if (!dfn[v]) {
            child++;
            tarjan(v, u);
            low[u] = min(low[u], low[v]);
            if (fa == 0 && child >= 2) isCut[u] = true;
            if (fa != 0 && low[v] >= dfn[u]) isCut[u] = true;
        } else if (v != fa) {
            low[u] = min(low[u], dfn[v]);
        }
    }
}
```

**C++ 代码（桥）**：
```cpp
// 注意：重边需用边编号去重，不能只用 v != fa
void tarjan(int u, int inEdge) {
    dfn[u] = low[u] = ++timer;
    for (auto [v, eid] : adj[u]) {
        if (!dfn[v]) {
            tarjan(v, eid);
            low[u] = min(low[u], low[v]);
            if (low[v] > dfn[u]) isBridge[eid] = true;
        } else if (eid != inEdge) {
            low[u] = min(low[u], dfn[v]);
        }
    }
}
```

### 2.5 边双/点双连通分量

**边双连通分量（e-BCC）**：
- 删除所有桥后，每个连通块就是一个边双连通分量
- 也可在 Tarjan 过程中用栈维护

**点双连通分量（v-BCC）**：
- 在 DFS 过程中维护边栈，当发现割点条件 $low[v] \ge dfn[u]$ 时，弹出边栈直到弹出边 $(u, v)$，弹出的边和端点构成一个 v-BCC
- 注意：一个割点可能属于多个点双连通分量

```
边双 vs 点双的区别：
- 边双中的任意两点间至少存在两条边不相交的路径
- 点双中的任意两点间至少存在两条点不相交的路径（除端点外）
- 割点是多个点双的公共节点
- 桥是连接不同边双的边
```

**边双连通分量 C++ 代码**：
```cpp
// 先求桥，再 DFS 标记分量
void markEBCC(int u, int id) {
    ebccId[u] = id;
    for (auto [v, eid] : adj[u])
        if (!ebccId[v] && !isBridge[eid])
            markEBCC(v, id);
}
```

**点双连通分量 C++ 代码**：
```cpp
// 维护边栈
void tarjan(int u, int fa) {
    dfn[u] = low[u] = ++timer;
    int child = 0;
    for (int v : adj[u]) {
        if (!dfn[v]) {
            child++;
            edgeStk.push({u, v});
            tarjan(v, u);
            low[u] = min(low[u], low[v]);
            if ((fa == 0 && child >= 2) || (fa != 0 && low[v] >= dfn[u])) {
                bccCnt++;              // 新的点双
                while (edgeStk.top() != make_pair(u, v)) {
                    auto [a, b] = edgeStk.top(); edgeStk.pop();
                    bcc[bccCnt].insert(a); bcc[bccCnt].insert(b);
                }
                edgeStk.pop();
                bcc[bccCnt].insert(u); bcc[bccCnt].insert(v);
            }
        } else if (v != fa && dfn[v] < dfn[u]) {
            edgeStk.push({u, v});
            low[u] = min(low[u], dfn[v]);
        }
    }
}
```

## 三、分类与变体详解

### 3.1 强连通分量 + 缩点

**问题类型**：给出有向图，要求对 SCC 进行缩点，然后在 DAG 上求解（最长路 / 统计信息等）。

**标准做法**：
1. Tarjan 求 SCC
2. 缩点建 DAG，合并 SCC 内部信息
3. 在 DAG 上拓扑排序 + DP

**关键代码模式**：
```cpp
// 缩点后 DAG 上求最长路（按反拓扑序即 SCC 编号从大到小）
for (int u = sccCnt; u >= 1; u--) {
    if (!dp[u]) dp[u] = sccW[u];
    for (int v : dag[u])
        dp[v] = max(dp[v], dp[u] + sccW[v]);
}
```

### 3.2 割点

**问题类型**：求无向图中的所有割点，或分析删除某个节点后图的连通性变化。

**关键性质**：
- 非根节点 $u$ 是割点 $\iff$ 存在子节点 $v$ 使得 $low[v] \ge dfn[u]$
- 根节点是割点 $\iff$ DFS 树上有至少 2 个子节点
- 割点与点双连通分量密切相关

### 3.3 桥（割边）

**问题类型**：求无向图中的所有桥，或判断删除某条边后图是否仍连通。

**关键性质**：
- 树边 $(u,v)$（$u$ 是 $v$ 的父亲）是桥 $\iff$ $low[v] > dfn[u]$
- 桥不在任何简单环上
- 需要注意重边的处理：两条端点相同的边不会同时为桥

**重边处理方法**：用 **边编号** 而非父节点来判断是否走了反向边。

### 3.4 边双连通分量

**问题类型**：求无向图的边双连通分量，或缩边双后分析树结构。

**做法**：
1. 先用 Tarjan 求出所有桥
2. 删除桥后，对每个连通块标记边双连通分量

**性质**：
- 边双内部任意两点至少存在 2 条边不相交路径
- 缩边双后，原图变成一棵树（或森林）
- 树上的每条边对应原图的一条桥

### 3.5 点双连通分量

**问题类型**：求无向图的点双连通分量，分析割点对连通性的影响。

**做法**：
- 在 Tarjan 过程中用边栈维护，当触发割点条件时弹栈收集 v-BCC

**性质**：
- 点双内部任意两点至少存在 2 条内部点不相交路径
- 一个割点属于多个点双
- 缩点双后得到的是一棵**块割树**（Block-Cut Tree）：割点和点双交替连接

### 3.6 缩点后 DAG 上 DP

**问题类型**：有向图上的最优化问题，先用 SCC 缩点化为 DAG，再在 DAG 上做 DP。

**常见 DP 目标**：
- DAG 最长路 / 最大点权和路径
- DAG 上的路径计数
- DAG 上的可达性分析

**通用框架**：
```cpp
// step 1: Tarjan 缩点
// step 2: 建 DAG，合并 SCC 权值
// step 3: DAG 上拓扑排序 + DP
// 因为 SCC 编号的反拓扑序性质，可直接倒序枚举
for (int u = sccCnt; u >= 1; u--)
    for (int v : dag[u])
        dp[v] = max(dp[v], dp[u] + sccW[v]);
int ans = *max_element(dp + 1, dp + sccCnt + 1);
```

### 题目 1：P3387 【模板】缩点 🟢

**题意**：给定一个 $n$ 个点 $m$ 条边的有向图，每个点有一个权值。求一条路径使得路径上经过的点权值之和最大（每个点权值只算一次，可以经过同一个点多次）。

**分析**：
- 可以经过同一个点多次 → 同一个 SCC 内的点都可以取到
- 缩点后变成 DAG，每个超级节点权值 = SCC 内所有点权之和
- 问题转化为 DAG 最长路

> **算法**
> 1. Tarjan 求 SCC，缩点建 DAG
> 2. 合并每个 SCC 的点权和
> 3. DAG 上拓扑排序 + DP 求最长路
> **复杂度**：$O(V + E)$

**C++ 代码（核心）**：
```cpp
// 缩点后求 DAG 最长路
for (int u = 1; u <= n; u++)
    sccW[sccId[u]] += w[u];
for (int u = 1; u <= n; u++)
    for (int v : adj[u])
        if (sccId[u] != sccId[v])
            dag[sccId[u]].push_back(sccId[v]);

fill(dp + 1, dp + sccCnt + 1, 0);
for (int i = 1; i <= sccCnt; i++) dp[i] = sccW[i];
// SCC 编号本身是反拓扑序
for (int u = sccCnt; u >= 1; u--)
    for (int v : dag[u])
        dp[v] = max(dp[v], dp[u] + sccW[v]);
```

**核心启示**：Tarjan 缩点 + DAG DP 是处理有向图路径优化问题的标准模式。SCC 编号天然具有反拓扑序性质，可以直接按编号倒序 DP。

### 题目 2：P3388 【模板】割点（割顶） 🟢

**题意**：给定一个 $n$ 个点 $m$ 条边的无向图，求所有割点。

**分析**：
- 直接使用 Tarjan 算法求割点
- 非根节点 $u$ 是割点当且仅当存在子节点 $v$ 使得 $low[v] \ge dfn[u]$
- 根节点是割点当且仅当 DFS 树中有不少于 2 个子节点

> **算法**
> 1. 对每个连通分量跑 Tarjan
> 2. 根据割点判定条件标记割点
> **复杂度**：$O(V + E)$

**C++ 代码（核心）**：
```cpp
void tarjan(int u, int fa) {
    dfn[u] = low[u] = ++timer;
    int child = 0;
    for (int v : adj[u]) {
        if (!dfn[v]) {
            child++;
            tarjan(v, u);
            low[u] = min(low[u], low[v]);
            if (fa && low[v] >= dfn[u]) isCut[u] = true;
            if (!fa && child >= 2) isCut[u] = true;
        } else if (v != fa) {
            low[u] = min(low[u], dfn[v]);
        }
    }
}
```

**核心启示**：割点判定的关键在于理解 $low[v] \ge dfn[u]$ 的含义——子树 $v$ 无法"绕过"$u$ 到达更早的祖先。根节点需要单独判定。

### 题目 3：P2341 [HAOI2006]受欢迎的牛 🟡

**题意**：有 $n$ 头牛，给定 $m$ 对"崇拜"关系（$A$ 崇拜 $B$），崇拜关系具有传递性。求有多少头牛被所有其他牛崇拜（即"明星牛"）。

**分析**：
- 崇拜关系构成有向图，传递性 → 同一个 SCC 中的牛互相崇拜
- 缩点后变成 DAG，"被所有牛崇拜"意味着该 SCC 在 DAG 中是所有节点都能到达的
- 在 DAG 中，所有节点都能到达的点只可能是 **出度为 0** 的节点
- 如果出度为 0 的节点恰好只有一个，则该 SCC 中的牛就是明星牛；否则无解

> **算法**
> 1. Tarjan 缩点建 DAG
> 2. 统计 DAG 中出度为 0 的节点数
> 3. 若恰好一个，答案为该 SCC 的大小；否则答案为 0
> **复杂度**：$O(V + E)$

**C++ 代码（核心）**：
```cpp
// 统计缩点后的出度
for (int u = 1; u <= n; u++)
    for (int v : adj[u])
        if (sccId[u] != sccId[v])
            outDeg[sccId[u]]++;

int zeroOut = 0, ans = 0;
for (int i = 1; i <= sccCnt; i++)
    if (outDeg[i] == 0) { zeroOut++; ans = sccSize[i]; }
if (zeroOut != 1) ans = 0;
```

**核心启示**：DAG 中"所有节点可达的唯一汇点"这一性质是缩点后的经典分析模式。出度为 0 的节点仅一个时，它才是全局可达的。

### 题目 4：P2863 [USACO06JAN]The Cow Prom 🟢

**题意**：给定一个 $n$ 个点 $m$ 条边的有向图，求包含至少 2 个节点的强连通分量的个数。

**分析**：
- 直接用 Tarjan 求出所有 SCC
- 统计大小 $\ge 2$ 的 SCC 个数

> **算法**
> 1. Tarjan 求所有 SCC
> 2. 遍历每个 SCC，统计大小 $\ge 2$ 的个数
> **复杂度**：$O(V + E)$

**C++ 代码（核心）**：
```cpp
// Tarjan 求 SCC 后统计
int ans = 0;
for (int i = 1; i <= sccCnt; i++)
    if (sccSize[i] >= 2) ans++;
```

**核心启示**：这是 Tarjan 求 SCC 的基础应用题。确保正确实现 Tarjan 算法是关键，注意不要遗漏初始化和栈的维护。

### 题目 5：P2746 [USACO5.3]Network of Schools 🟡

**题意**：给定 $n$ 所学校之间的有向通信网络。
- 任务 A：至少给几所学校发送软件，能让所有学校最终都收到？
- 任务 B：至少添加几条边，能使整个网络变成强连通的？

**分析**：
- 缩点后变成 DAG
- 任务 A：DAG 上入度为 0 的节点数（这些"源头"必须各自初始拥有软件）
- 任务 B：要把 DAG 变成强连通的，至少需要添加 $\max(\text{入度为 0 的节点数}, \text{出度为 0 的节点数})$ 条边
- 特殊情况：如果整个图本身就是强连通的（只有一个 SCC），任务 A 答案为 1，任务 B 答案为 0

> **算法**
> 1. Tarjan 缩点建 DAG
> 2. 统计 DAG 中入度为 0 和出度为 0 的节点个数
> 3. 任务 A = max(1, 入度 0 的数量)，任务 B = max(入度 0 的数量, 出度 0 的数量)
> 4. 特判：sccCnt == 1 时，A = 1, B = 0
> **复杂度**：$O(V + E)$

**C++ 代码（核心）**：
```cpp
for (int u = 1; u <= n; u++)
    for (int v : adj[u])
        if (sccId[u] != sccId[v]) {
            inDeg[sccId[v]]++;
            outDeg[sccId[u]]++;
        }
int a = 0, b = 0;
for (int i = 1; i <= sccCnt; i++) {
    if (inDeg[i] == 0) a++;
    if (outDeg[i] == 0) b++;
}
if (sccCnt == 1) printf("1\n0\n");
else printf("%d\n%d\n", a, max(a, b));
```

**核心启示**：缩点后 DAG 的入度/出度分析是非常经典的思路。将 DAG 补成强连通图的最少边数 = $\max$（源点数, 汇点数）是一个重要结论。

### 题目 6：P1407 [国家集训队]稳定婚姻 🟠

**题意**：有 $n$ 对夫妇和若干对恋人关系（恋人分属不同夫妇）。判断每对婚姻是否"稳定"——即若拆散这对夫妇，是否所有人仍能配对（原配偶可拆开与恋人配对，具有传递效应）。

**分析**：
- 建有向图：对于每对夫妻 $(h_i, w_i)$，连边 $h_i \to w_i$；对于每对恋人 $(h_a, w_b)$，连边 $w_b \to h_a$
- 边的含义：$h_i \to w_i$ 表示丈夫"绑定"妻子，$w_b \to h_a$ 表示如果 $w_b$ 单身则可以找到 $h_a$
- 如果某对夫妻 $(h_i, w_i)$ 在同一个 SCC 中，说明存在一条从 $w_i$ 回到 $h_i$ 的路径，即拆散后可以通过恋人关系形成新的配对链，因此该婚姻不稳定
- 反之若不在同一 SCC 中，婚姻安全

> **算法**
> 1. 按上述规则建图
> 2. Tarjan 求 SCC
> 3. 对每对夫妻，检查 $sccId[h_i]$ 是否等于 $sccId[w_i]$
> **复杂度**：$O(V + E)$

**C++ 代码（核心）**：
```cpp
// 建图：夫妻 h[i] -> w[i]，恋人 w[b] -> h[a]
for (int i = 1; i <= n; i++)
    addEdge(h[i], w[i]);
for (int i = 1; i <= m; i++) {
    int a, b; // 恋人: h[a] 和 w[b]
    addEdge(w[b], h[a]);
}
// Tarjan 后判断
for (int i = 1; i <= n; i++)
    puts(sccId[h[i]] == sccId[w[i]] ? "Unsafe" : "Safe");
```

**核心启示**：将配对关系建模为有向图，利用 SCC 判断"可替代性"。技巧在于正确设计边的方向，使得 SCC 表示"循环替代链"。

### 题目 7：P2272 [ZJOI2007]最大半连通子图 🔴

**题意**：给定有向图，求最大的"半连通子图"的大小及方案数。半连通子图是指：对于其中任意两个节点 $u, v$，存在从 $u$ 到 $v$ 或从 $v$ 到 $u$ 的路径。

**分析**：
- 首先 SCC 缩点，因为 SCC 内部任意两点互相可达，显然是半连通的
- 缩点后变成 DAG，DAG 中的一条链（路径）上的所有节点构成一个半连通子图
- 因此问题转化为：在 DAG 上求最长链（权值为 SCC 大小之和），以及最长链的方案数
- 注意要对 DAG 的边去重（同一对 SCC 之间可能有多条边）

> **算法**
> 1. Tarjan 缩点
> 2. 建 DAG 并去重边
> 3. 拓扑排序 + DP，维护最长路及最长路方案数
> **复杂度**：$O(V + E)$

**C++ 代码（核心）**：
```cpp
// DAG 上 DP：dp[u] 为以 u 结尾的最长链权值，cnt[u] 为方案数
for (int i = 1; i <= sccCnt; i++)
    dp[i] = sccSize[i], cnt[i] = 1;
// 按拓扑序遍历
for (int u : topoOrder)
    for (int v : dag[u]) {
        long long nv = dp[u] + sccSize[v];
        if (nv > dp[v]) { dp[v] = nv; cnt[v] = cnt[u]; }
        else if (nv == dp[v]) cnt[v] = (cnt[v] + cnt[u]) % MOD;
    }
int ans = *max_element(dp + 1, dp + sccCnt + 1);
```

**核心启示**：半连通子图 = DAG 最长链。注意去重边是此题的关键细节，否则方案数会计算错误。

### 题目 8：P3225 [HNOI2012]矿场搭建 🔴

**题意**：给定一个无向图（矿场的巷道连接图），需要在若干点上建设救援出口。要求：如果任意一个点发生坍塌（删去该点及其关联边），剩下的所有工人都能到达至少一个出口。求最少建设几个出口，以及方案数。

**分析**：
- 关键观察：需要抵御"任意一个点被删除"的风险 → 与 **割点** 和 **点双连通分量** 密切相关
- 求出所有点双连通分量（v-BCC），分析每个 v-BCC 与割点的关系：
  - 若某个 v-BCC 中不包含任何割点 → 它是一个"孤立"的连通块，需要建 2 个出口（防止其中一个坍塌）
  - 若某个 v-BCC 中恰好包含 1 个割点 → 需要在非割点的位置建 1 个出口（割点坍塌时才需要出口）
  - 若某个 v-BCC 中包含 $\ge 2$ 个割点 → 不需要建出口（无论哪个割点坍塌，都能通过另一个割点逃到其他分量）
- 答案 = 各 v-BCC 的出口数之和；方案数 = 各 v-BCC 可选位置数之积

> **算法**
> 1. Tarjan 求割点和点双连通分量
> 2. 统计每个 v-BCC 中割点数量
> 3. 按上述规则计算最少出口数和方案数
> **复杂度**：$O(V + E)$

**C++ 代码（核心）**：
```cpp
// 对每个 v-BCC 统计割点数 cutCnt 和非割点数 sz
long long ans = 0, ways = 1;
for (int i = 1; i <= bccCnt; i++) {
    int cutCnt = 0, sz = bcc[i].size();
    for (int u : bcc[i])
        if (isCut[u]) cutCnt++;
    if (cutCnt == 0) { ans += 2; ways *= (long long)sz * (sz - 1) / 2; }
    else if (cutCnt == 1) { ans += 1; ways *= (sz - 1); }
    // cutCnt >= 2: 不需要建出口
}
```

**核心启示**：点双连通分量在处理"删除任意一点后的连通性"问题中非常关键。分析每个 v-BCC 与割点的关系是解题的核心思路。

### 题目 9：P5058 [ZJOI2004]嗅探器 🟡

**题意**：给定一个无向连通图以及两个指定节点 $a$ 和 $b$，求编号最小的割点 $c$（$c \ne a, c \ne b$），使得删除 $c$ 后 $a$ 和 $b$ 不连通。

**分析**：
- 首先求出所有割点
- 但不是所有割点都满足条件，需要进一步判断删除该割点后 $a$ 和 $b$ 是否真的不连通
- 方法：以 $a$ 为根做 DFS，求 Tarjan。对于割点 $u$（$u \ne a, u \ne b$），如果 $b$ 在 $u$ 的子树中（即 $u$ 是 $a$ 到 $b$ 路径上的必经点），且 $low[v] \ge dfn[u]$（$v$ 是 $u$ 的包含 $b$ 的子节点），则 $u$ 是答案候选
- 实际操作：以 $a$ 为根跑 Tarjan，若某非根节点 $u$ 满足 $low[v] \ge dfn[u]$ 且 $b$ 在 $v$ 的子树中，则 $u$ 满足条件

> **算法**
> 1. 以 $a$ 为根跑 Tarjan
> 2. 枚举所有割点，检查是否在 $a$ 到 $b$ 的必经路径上
> 3. 输出编号最小的满足条件的割点
> **复杂度**：$O(V + E)$

**C++ 代码（核心）**：
```cpp
// 以 a 为根跑 Tarjan
void tarjan(int u, int fa) {
    dfn[u] = low[u] = ++timer;
    for (int v : adj[u]) {
        if (!dfn[v]) {
            tarjan(v, u);
            low[u] = min(low[u], low[v]);
            // u 非 a 且非 b，low[v]>=dfn[u] 且 b 在 v 子树中
            if (u != a && u != b && low[v] >= dfn[u]
                && dfn[b] >= dfn[v] && dfn[b] <= tout[v])
                candidates.insert(u);
        } else if (v != fa)
            low[u] = min(low[u], dfn[v]);
    }
    tout[u] = timer;
}
// 答案为 candidates 中的最小值
```

**核心启示**：割点问题的变体——不仅要找割点，还需判断删除后是否影响两个特定节点的连通性。结合 DFS 子树判断是关键。

### 题目 10：P2515 [HAOI2010]软件安装 🟣

**题意**：有 $n$ 个软件，每个软件有占用空间 $w_i$、价值 $v_i$、以及最多一个依赖 $d_i$（必须安装 $d_i$ 后才能安装 $i$）。总空间为 $m$。求最大价值。

**分析**：
- 依赖关系构成一个有向图（可能有环，因为 A 依赖 B，B 也可能间接依赖 A）
- 有环意味着环内软件要么全装要么全不装 → SCC 缩点
- 缩点后变成 DAG（实际上是森林/树形结构），每个 SCC 合并成一个节点
- 建虚拟根节点 $0$ 连接所有入度为 0 的节点
- 问题转化为 **树上依赖背包**（树形 DP + 分组背包）

> **算法**
> 1. Tarjan 缩点，合并 SCC 的空间和价值
> 2. 建树（DAG 为森林，添加虚拟根）
> 3. 树形 DP：$dp[u][j]$ 表示以 $u$ 为根的子树中，花费空间 $j$ 能获得的最大价值
> 4. 子树合并用分组背包
> **复杂度**：$O(n \cdot m^2)$（树上背包）

**C++ 代码（核心）**：
```cpp
// 树形背包 DP
void dfs(int u) {
    for (int j = sccW[u]; j <= m; j++) dp[u][j] = sccV[u];
    for (int v : children[u]) {
        dfs(v);
        for (int j = m; j >= sccW[u]; j--)
            for (int k = 0; k <= j - sccW[u]; k++)
                dp[u][j] = max(dp[u][j], dp[u][j - k] + dp[v][k]);
    }
}
// 从虚拟根 0 开始 dfs(0)，答案为 dp[0][m]
```

**核心启示**：当依赖关系可能存在环时，先 Tarjan 缩点消除环，再做树形 DP。这是"SCC 缩点 + 树形 DP"的经典组合模式。

## 五、常见技巧与思路总结

### 技巧 1：Tarjan SCC 编号的反拓扑序性质

Tarjan 算法中，SCC 的编号顺序天然是 **反拓扑序**。这意味着：
- 编号小的 SCC 在 DAG 中更靠后
- 可以直接按编号从大到小做 DP，无需额外拓扑排序

### 技巧 2：割点判定中根节点的特殊处理

根节点的割点判定条件与非根节点不同：
- 非根：$\exists$ 子节点 $v$ 使 $low[v] \ge dfn[u]$
- 根：DFS 树上子节点数 $\ge 2$

**常见错误**：忘记对根节点单独判定，或将两种判定条件搞混。

### 技巧 3：桥的重边处理

当无向图存在重边时，不能简单用 `v != fa` 来避免走反向边。应使用 **边的编号** 来判断：
- 链式前向星中，成对存储的边编号为 $i$ 和 $i \oplus 1$
- DFS 时记录进入边编号 `inEdge`，跳过 `inEdge ^ 1`

### 技巧 4：缩点后的 DAG 去重

缩点后建 DAG 时，同一对 SCC 之间可能产生多条平行边。在统计路径数等场景下，需要去重：
- 使用 `set` 或排序去重
- 或者在 DP 转移时用边的编号标记避免重复

### 技巧 5：点双与割点的关系

- 每个割点属于多个 v-BCC
- 每个非割点恰好属于一个 v-BCC
- 建立 **块割树**（Block-Cut Tree）可以快速查询两点间必经的割点

### 技巧 6：有向图依赖关系可能有环

在处理依赖关系问题时（如软件安装、课程选修），如果依赖关系是有向图且可能有环，必须先 Tarjan 缩点再处理。

## 七、终极总结与归纳

### 核心算法一览

| 算法 | 用途 | 时间复杂度 | 核心判定条件 |
|------|------|-----------|-------------|
| Tarjan SCC | 有向图强连通分量 | $O(V+E)$ | $dfn[u] = low[u]$ → SCC 根 |
| Tarjan 割点 | 无向图割点 | $O(V+E)$ | $low[v] \ge dfn[u]$（非根）/ 子节点数 $\ge 2$（根） |

| 边双连通分量 | 无向图 e-BCC | $O(V+E)$ | 删桥后连通块 |
| 点双连通分量 | 无向图 v-BCC | $O(V+E)$ | 割点条件触发时弹边栈 |

### 做题口诀

1. **有向图求分量 → Tarjan SCC → 缩点建 DAG**
