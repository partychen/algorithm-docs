## 一、什么是凸包

凸包（Convex Hull）可以理解为“把所有点用一根橡皮筋紧紧包起来”后得到的最外层边界。凡是出现在这条边界上的点，才可能影响点集的整体轮廓；严格位于内部的点不会改变外壳。

在算法竞赛里，凸包本身往往不是终点，而是很多几何题的**标准预处理**：先把原始点集压缩成一个按逆时针排列的凸多边形，再去处理最远点对、宽度、点内判定、旋转卡壳、半平面交前置等问题。

| 场景 | 凸包提供的价值 |
|------|----------------|
| 围栏/外轮廓 | 直接得到最外层边界 |
| 最远点对 | 把候选点压缩到凸包顶点 |
| 面积/周长 | 直接在凸多边形上计算 |
| 多次询问点内关系 | 可转成凸多边形判定 |
| 更复杂几何算法 | 作为旋转卡壳等算法的输入 |

如果题目里出现“最外层点”“围住全部点”“只与边界有关”“删掉内部点不影响答案”等信号词，通常就应该优先联想到凸包。

## 二、核心思想与前置知识

### 2.1 叉积与转向判断

凸包算法的核心判定是“三点是否左转”。若按顺序经过点 `O -> A -> B`，那么：

- 叉积 `> 0`：左转，说明当前边界保持凸性
- 叉积 `< 0`：右转，说明中间点不该留在凸包上
- 叉积 `= 0`：三点共线，是否保留中间点取决于题目要求

```cpp
struct Point {
    long long x, y;
    bool operator<(const Point& other) const {
        return x < other.x || (x == other.x && y < other.y);
    }
    bool operator==(const Point& other) const {
        return x == other.x && y == other.y;
    }
};

long long cross(const Point& o, const Point& a, const Point& b) {
    return (a.x - o.x) * (b.y - o.y) - (a.y - o.y) * (b.x - o.x);
}
```

### 2.2 Graham Scan 与 Andrew 单调链

两种经典凸包算法都能在 `O(n log n)` 内完成，但竞赛里更常用 Andrew 单调链。

| 算法 | 核心思路 | 优点 | 注意点 |
|------|----------|------|--------|
| Graham Scan | 先选基准点，再按极角排序，用栈维护左转 | 理解直观 | 极角排序与共线处理略麻烦 |
| Andrew 单调链 | 先按坐标排序，再分别构造上下凸壳 | 实现短、稳定、易调试 | 需要明确是否保留共线点 |

如果题目没有特殊说明，推荐默认写 Andrew 模板；它更像“排序 + 两遍扫描”，容错率高。

### 2.3 共线点与去重策略

凸包题最容易错的地方，不是主算法，而是“共线点到底留不留”。

- **严格凸包**：边上的中间点不保留，只保留端点
- **包含边界点的凸包**：边上的所有点都要保留
- **重复点**：排序后必须先去重，否则容易在退化情形下出错

一个非常实用的判断规则是：

- 若题目只关心面积、周长、直径，通常使用**严格凸包**即可
- 若题目要求“输出所有在围栏上的点”，则必须**保留共线边界点**

### 2.4 Andrew 模板骨架

```cpp
vector<Point> convexHull(vector<Point> pts, bool keepCollinear = false) {
    sort(pts.begin(), pts.end());
    pts.erase(unique(pts.begin(), pts.end()), pts.end());
    int n = pts.size();
    if (n <= 2) return pts;

    auto shouldPop = [&](const Point& a, const Point& b, const Point& c) {
        long long v = cross(a, b, c);
        return keepCollinear ? v < 0 : v <= 0;
    };

    vector<Point> lower, upper;
    for (const auto& p : pts) {
        while (lower.size() >= 2 && shouldPop(lower[lower.size() - 2], lower.back(), p)) {
            lower.pop_back();
        }
        lower.push_back(p);
    }
    for (int i = n - 1; i >= 0; --i) {
        while (upper.size() >= 2 && shouldPop(upper[upper.size() - 2], upper.back(), pts[i])) {
            upper.pop_back();
        }
        upper.push_back(pts[i]);
    }

    lower.pop_back();
    upper.pop_back();
    lower.insert(lower.end(), upper.begin(), upper.end());
    return lower;
}
```

这段骨架里最值得记住的是 `shouldPop`：把“是否保留共线点”的差异集中到一个判断里，整套模板就会非常稳定。

### 2.5 凸包后的常见二次利用

得到凸包后，常见后续操作包括：

1. 计算周长或面积
2. 在凸包上做旋转卡壳求直径/宽度
3. 对凸多边形做点内判定
4. 反复剥离最外层形成多层凸包
5. 作为多个凸集求交、切线或支持函数问题的输入

也就是说，**凸包负责把无序点集整理成有序边界**，后续大多数技巧都建立在这个“有序”之上。

## 三、分类与变体详解

### 3.1 严格凸包与包含边界点凸包

这两类写法只差一个不等号，但答案集合可能完全不同。

- 严格凸包：弹栈条件用 `<= 0`
- 保留边界点：弹栈条件用 `< 0`

如果题目要求返回所有边界点，除了改不等号，还要注意最后输出顺序不要重复端点。

### 3.2 凸包周长与面积

对于按逆时针顺序排列的凸包点 `p[0..m-1]`：

- 周长：累加相邻点距离与首尾距离
- 面积：用多边形面积公式（叉积求和）

```cpp
long double polygonArea2(const vector<Point>& hull) {
    long double sum = 0;
    int m = hull.size();
    for (int i = 0; i < m; ++i) {
        int j = (i + 1) % m;
        sum += 1.0L * hull[i].x * hull[j].y - 1.0L * hull[i].y * hull[j].x;
    }
    return fabsl(sum);  // 返回面积的 2 倍
}
```

### 3.3 点在凸多边形内判定

若有大量询问“点是否在凸包内”，可以利用凸多边形的有序性，将一次判定降为 `O(log n)`。

常见做法是：

1. 固定一个顶点 `p0`
2. 先判断询问点是否在扇形 `p0-p1-p[n-1]` 内
3. 再在扇形里二分找到相邻三角形，做一次叉积判定

这类题的关键不是重写一套新算法，而是意识到“凸包后的点序可二分”。

### 3.4 多层凸包（洋葱剥壳）

有些题会反复删除最外层凸包，问能剥多少层、某点属于第几层，或者最后剩下什么结构。这类问题通常叫 **Onion Decomposition**。

做法很直接：

1. 对剩余点集求一次凸包
2. 删除该凸包上的点
3. 重复直到点数不足以形成新一层

单次凸包是 `O(n log n)`，总复杂度视层数而定。

### 3.5 凸包作为后续算法前置

许多题表面不是“求凸包”，但第一步仍然要先上凸包，例如：

- 最远点对：先上凸包，再做旋转卡壳
- 最小包围矩形：先上凸包，再维护对踵点
- 两凸包距离：先分别求凸包，再做双指针或支持线处理

所以看到点集几何题时，应该先问自己一句：**内部点有没有资格影响答案？** 如果没有，凸包大概率就是第一步。

### 模型对比总表

| 问题类型 | 是否先求凸包 | 后续常用技巧 | 典型复杂度 |
|----------|--------------|-------------|-----------|
| 周长/面积 | 是 | 多边形遍历 | `O(n log n)` |
| 最远点对 | 是 | 旋转卡壳 | `O(n log n)` |
| 点内判定（多询问） | 是 | 扇形二分 | `O(n log n + q log n)` |
| 分层剥壳 | 是 | 重复构造凸包 | `O(k · n log n)` |
| 输出边界点 | 是 | 保留共线点 | `O(n log n)` |

### 🟢 入门级 Level 1：模板与边界处理

### 题目 1：洛谷 P2742【模板】二维凸包 / [USACO5.1] 圈奶牛 Fencing the Cows

> 给定若干平面点，求围住所有点的最短围栏长度。

**分析**：这是二维凸包周长模板题。排序去重后用 Andrew 单调链得到逆时针凸包，再顺着边界累加距离即可。退化为单点或双点时要单独处理。

```
建模关键词：最短围栏、点集外轮廓、凸包周长
核心步骤：排序去重 -> 构造凸包 -> 累加边长
```

**C++ 代码**：
```cpp
long double perimeter(const vector<Point>& hull) {
    long double ans = 0;
    for (int i = 0; i < (int)hull.size(); ++i) {
        int j = (i + 1) % hull.size();
        ans += hypotl(hull[i].x - hull[j].x, hull[i].y - hull[j].y);
    }
    return ans;
}
```

答案位置：`perimeter(hull)`

**代码讲解**：
- `for` 循环逐条累加凸包相邻顶点的边长，等价于沿外轮廓走一圈。
- `j = (i + 1) % hull.size()` 负责补上最后一个点到第一个点的闭合边，避免漏算收尾。

**核心启示**：
- “最短围住所有点”几乎就是凸包周长的同义信号。
- 这题最适合用来检查你的 Andrew 模板是否稳定。

### 题目 2：LeetCode 587 安装栅栏（Erect the Fence）

> 返回恰好位于最外层围栏上的所有树，边上的共线点也要保留。

**分析**：和普通凸包模板只差一个细节：边界共线点不能丢。也就是说，弹栈条件要从 `<= 0` 改成 `< 0`，让共线点继续留在边界上。

```
建模关键词：边界点保留、共线点、凸包点集输出
核心区别：严格凸包 vs 包含边界点凸包
```

**C++ 代码**：
```cpp
auto shouldPop = [&](const Point& a, const Point& b, const Point& c) {
    return cross(a, b, c) < 0;  // 保留共线点
};
```

答案位置：`convexHull(...)` 返回的边界点集合 `hull`

**代码讲解**：
- `cross(a, b, c) < 0` 时才弹栈，表示只删除形成右拐的点，从而把边上的共线点保留下来。
- 这道题真正的差异不在建壳流程，而在“不等号是否严格”这个边界策略。

**核心启示**：
- 题目要求“边界上的所有点”时，最先改的就是不等号。
- 同一套模板，答案集合可能因为共线策略完全不同。

### 题目 3：POJ 1113 Wall

> 已知城堡顶点和安全距离 `L`，求最省材料的围墙长度。

**分析**：先对所有顶点求凸包周长，再额外补上一圈半径为 `L` 的圆周长 `2πL`。因为距离约束等价于在凸包外做一次 Minkowski 膨胀。

```
建模关键词：凸包周长、圆角补偿、外扩一圈
答案形式：perimeter(hull) + 2 * pi * L
```

**C++ 代码**：
```cpp
long double answer = perimeter(hull) + 2.0L * acosl(-1.0L) * L;
```

答案位置：`answer`

**代码讲解**：
- `perimeter(hull)` 负责直线边段部分，表示最外层凸包的总边长。
- `2πL` 是把所有拐角统一补成一整圈圆弧后的长度，所以直接与周长相加即可。

**核心启示**：
- 凸包常和“圆角/缓冲距离”一起出现。
- 一旦看懂“外扩边界”的几何意义，公式会非常自然。

### 🟡 进阶级 Level 2：面积、点序与基础综合

### 题目 4：洛谷 P2116 [NEERC 2001] 城墙

> 给出城堡多边形顶点和最小安全距离，求最短城墙长度并四舍五入。

**分析**：本质与 `POJ 1113 Wall` 一致：真正需要围住的是顶点集合的凸包，而不是原多边形折线本身。求出凸包周长后，再加上 `2πL` 即可。

```
建模关键词：原图未必凸、先压成凸包、偏移边界
核心公式：凸包周长 + 2πL
```

**C++ 代码**：
```cpp
long long rounded = llround(perimeter(hull) + 2.0L * acosl(-1.0L) * L);
```

答案位置：`rounded`

**代码讲解**：
- 内部公式仍然是“凸包周长 + `2πL`”，区别只在最后输出要求四舍五入。
- `llround(...)` 把浮点答案转成题目要求的整数格式，避免手写取整细节。

**核心启示**：
- 题面给的是多边形，不代表答案必须沿原边走。
- 只要目标是“最短围住全部点”，内部凹陷都没有存在价值。

### 题目 5：POJ 3348 Cows

> 给出若干棵树的位置，问能围出的最大牧场能养多少头牛。

**分析**：最大牧场面积就是点集凸包面积。题目最后再用每头牛所需的固定面积做一次换算，因此关键还是先把无序点集整理成凸包点序。

```
建模关键词：最大可围面积、凸包面积、鞋带公式
核心步骤：求 hull -> 求面积 -> 按题目规则换算
```

**C++ 代码**：
```cpp
long double polygonArea(const vector<Point>& hull) {
    long double sum = 0;
    for (int i = 0; i < (int)hull.size(); ++i) {
        int j = (i + 1) % hull.size();
        sum += 1.0L * hull[i].x * hull[j].y - 1.0L * hull[i].y * hull[j].x;
    }
    return fabsl(sum) / 2.0L;
}
```

答案位置：`polygonArea(hull)`（再按题目单位规则换算）

**代码讲解**：
- `sum += x_i y_{i+1} - y_i x_{i+1}` 就是鞋带公式的标准写法，顺着凸包点序一圈累加有向面积。
- 最后取 `fabsl(sum) / 2` 把有向面积转回普通面积，顺逆时针点序都能兼容。

**核心启示**：
- 面积类题只要答案由最外层决定，几乎都要先求凸包。
- 鞋带公式是凸包后最常见的第二步。

### 题目 6：UVa 11626 Convex Hull

> 给出带筛选条件的点集，要求输出凸包顶点序列。

**分析**：这类题比模板题多了一层输入过滤与输出格式要求，但核心仍是“排序 -> 构壳 -> 按逆时针输出”。当题目要求输出点序时，去重和首尾重复处理尤其容易出错。

```
建模关键词：输出顶点序、逆时针顺序、格式细节
注意事项：去重、首点是否重复、共线点保留策略
```

**C++ 代码**：
```cpp
vector<Point> hull = convexHull(pts, false);
for (const auto& p : hull) {
    cout << p.x << ' ' << p.y << '
';
}
```

答案位置：`hull`（按逆时针顺序输出）

**代码讲解**：
- `convexHull(pts, false)` 这里显式要求不要重复首点，便于直接按序输出。
- 后面的遍历只是把已经排好的凸包顶点序打印出来，所以重点其实在模板返回值的约定。

**核心启示**：
- 只会“算答案”不够，很多题还会专门卡输出序。
- 凸包模板最好默认返回“不重复首点”的逆时针序列。

### 🟠 提高级 Level 3：凸包与组合优化

### 题目 7：POJ 1873 The Fortified Forest

> 砍掉一部分树获得木材，用来围住剩余的树，要求代价最小。

**分析**：这是“子集枚举 + 凸包”的经典综合题。枚举哪些树被砍掉，再对剩余树求凸包周长，检查木材总长是否足够；若足够，就按价值与数量更新最优方案。

```
建模关键词：子集枚举、剩余点集凸包、围栏可行性
核心判定：cutLength >= perimeter(remainingHull)
```

**C++ 代码**：
```cpp
if (woodSum + 1e-9 >= perimeter(remainingHull)) {
    // 更新最小价值、最少砍树数
}
```

答案位置：外层枚举维护的最优方案（最小总代价、最少砍树数）

**代码讲解**：
- `woodSum + 1e-9 >= perimeter(remainingHull)` 是当前子集是否可行的核心判定：木材总长必须至少覆盖剩余点集凸包。
- 一旦可行，外层就可以按“总代价更小、同价时砍树更少”的规则更新最优解。

**核心启示**：
- 凸包经常不是整题主体，而是嵌在搜索/状压里的判定器。
- 当 `n` 很小、代价定义又在点上时，要敢于枚举子集再套凸包。

### 题目 8：洛谷 P3099 [USACO14JAN] Cow Curling G

> 两队石头互相“捕获”，若某石头落在对手三个石头形成三角形内则被记分。

**分析**：任意三点构成的三角形并不需要逐个枚举，因为一个点只要落在对手凸包内部，就一定落在对手某个三角剖分里。于是问题转成“两组点分别求凸包，再统计对方点有多少落在其中”。

```
建模关键词：点在三角形内 -> 点在凸包内、两组点、批量判定
核心步骤：分别求 hullA/hullB -> 统计 B 在 hullA 中的点数与反向点数
```

**C++ 代码**：
```cpp
int scoreA = 0;
for (const auto& p : teamB) scoreA += insideConvex(hullA, p);
```

答案位置：`scoreA` 与对称统计得到的 `scoreB`

**代码讲解**：
- `insideConvex(hullA, p)` 把“落在某三个石头形成的三角形内”整体提升成“落在对手凸包内”。
- 循环是在批量计数 Team B 有多少点被 Hull A 覆盖，反向再做一次即可得到两边得分。

**核心启示**：
- “被某三个边界点围住”通常可以整体提升成“落在凸包内”。
- 先做几何抽象，再做批量点内判定，会比枚举三元组清晰得多。

### 题目 9：洛谷 P3122 [USACO15FEB] Fencing the Herd G

> 点集动态加入新牛，多次询问某条直线是否能把所有牛严格分在同一侧。

**分析**：一条直线是否“有用”，本质是在检查点集在线性函数 `Ax+By` 下的最小值和最大值是否都落在 `C` 的同一侧，且不能有点恰在直线上。维护点集凸包后，只需查询该方向上的极值即可。

```
建模关键词：线性函数极值、支持函数、动态点集
核心判定：min(Ax+By) > C 或 max(Ax+By) < C
```

**C++ 代码**：
```cpp
long long value(const Point& p, long long A, long long B) {
    return A * p.x + B * p.y;
}
```

答案位置：每次询问比较得到的方向极值 `min(Ax+By)` / `max(Ax+By)`

**代码讲解**：
- `value(p, A, B)` 把点映射到当前查询方向上的线性函数值，后续所有“同侧”判断都围绕它展开。
- 真正的回答不靠枚举所有点，而是查询凸包在这个方向上的最小值和最大值。

**核心启示**：
- “所有点在同侧”其实是一个线性函数极值问题。
- 凸包的价值不只在周长面积，还在支持函数与极值查询。

### 🔴 困难级 Level 4：凸包作为更高阶算法前置

### 题目 10：洛谷 P3829 [SHOI2012] 信用卡凸包

> 若干张圆角矩形信用卡叠在平面上，求它们外轮廓的周长。

**分析**：圆角矩形的直边部分可以转成四个圆角圆心的凸包，最后再统一补上整圈圆弧长度。也就是说，真正要做的仍然是“构造关键点 -> 求凸包周长”。

```
建模关键词：圆角矩形、关键点抽取、凸包 + 圆周补偿
答案形式：perimeter(centersHull) + 2πr
```

**C++ 代码**：
```cpp
long double answer = perimeter(hullOfCornerCenters) + 2.0L * acosl(-1.0L) * r;
```

答案位置：`answer`

**代码讲解**：
- `hullOfCornerCenters` 先把每张圆角矩形转成“圆心骨架”的凸包，离散部分都落在这条外轮廓上。
- 再加上 `2πr`，就把所有圆角补成完整一圈弧长，得到最终周长。

**核心启示**：
- 复杂轮廓题经常能被“抽关键点 + 补连续曲线”拆开。
- 凸包和圆/线段混合出现时，先分离离散骨架与连续补偿。

### 题目 11：洛谷 P1452【模板】旋转卡壳 / [USACO03FALL] Beauty Contest G

> 给定平面点集，求最远点对距离的平方。

**分析**：最远点对一定出现在凸包上，所以第一步必须先求凸包；第二步再用旋转卡壳在凸包上找直径。这题是“凸包是高级几何预处理”的最佳示范。

```
建模关键词：先求凸包、缩候选集合、再做卡壳
核心流程：points -> hull -> diameter(hull)
```

**C++ 代码**：
```cpp
vector<Point> hull = convexHull(pts);
long long ans = diameter2(hull);
```

答案位置：`ans`

**代码讲解**：
- `convexHull(pts)` 先删掉所有不可能成为最远点对的内部点，把候选集合压成凸包顶点。
- `diameter2(hull)` 再在凸包上做旋转卡壳，外层返回的就是最远点对距离平方。

**核心启示**：
- 很多题真正的第一问不是“怎么求答案”，而是“内部点还重要吗”。
- 会先压缩候选集合，是凸包题最核心的思维收益。

### 题目 12：洛谷 P3187 [HNOI2007] 最小矩形覆盖

> 求覆盖所有点的最小面积矩形，并输出四个顶点。

**分析**：同样先求凸包，再在凸包上做四卡壳维护最优矩形。题面看起来是在求矩形，但若不先做凸包，后续根本谈不上旋转扫描。

```
建模关键词：最小包围矩形、凸包预处理、四卡壳
关键前置：原点集先压成逆时针凸包
```

**C++ 代码**：
```cpp
vector<Point> hull = convexHull(pts);
// 再在 hull 上维护最远点、最左投影点、最右投影点
```

答案位置：外层维护的最小面积及其对应矩形四个顶点

**代码讲解**：
- 这两行强调的是“先凸包，后四卡壳”的顺序：没有逆时针凸包，多指针单调性就不成立。
- 真正的答案变量在后续扫描里更新，包括当前最小面积和恢复出的矩形顶点。

**核心启示**：
- 凸包与旋转卡壳经常是成套出现的。
- 有时题目的难点其实只是看出“先做凸包”这一步。

### 🟣 专家级 Level 5：空间与复杂变体

### 题目 13：洛谷 P4724【模板】三维凸包

> 给出空间中的点，求三维凸包表面积。

**分析**：二维 Andrew 模板已经不够用了，需要转到增量法维护可见面与新面。虽然实现复杂得多，但核心思想没变：仍然是在求“包住所有点的最小凸体边界”。

```
建模关键词：空间凸包、可见面、增量法
核心步骤：初始化四面体 -> 删除可见面 -> 补新面
```

**C++ 代码**：
```cpp
struct Face {
    int a, b, c;
};
// 维护当前凸包表面的三角面集合
```

答案位置：当前三角面集合面积之和

**代码讲解**：
- `Face {a, b, c}` 表示三维凸包表面的一个有向三角面，是后续可见面删除与补面的基本单元。
- 外层算法持续维护这批面，最后把所有面的面积求和就是表面积答案。

**核心启示**：
- 三维凸包是二维凸包思想在空间中的直接升级。
- 这题更适合当“眼界题”，帮助你理解凸包的本质定义。

### 题目 14：POJ 2079 Triangle

> 在给定点集中选三个点，使三角形面积最大。

**分析**：先求凸包，再在凸包上寻找最大面积三角形。虽然最终用到的是旋转卡壳思想，但第一步依然是把候选点缩到凸包顶点集合。

```
建模关键词：最大面积三角形、候选点压缩、凸包前置
核心流程：convex hull -> rotating calipers for triangle
```

**C++ 代码**：
```cpp
vector<Point> hull = convexHull(pts);
// 在 hull 上枚举 i, j，并单调推进 k
```

答案位置：外层三指针维护的最大三角形面积

**代码讲解**：
- 这里先做 `convexHull(pts)`，是为了把三角形候选点缩到凸包顶点上。
- 后续枚举 `i, j` 并单调推进 `k` 时，所有面积比较都只在 `hull` 上完成。

**核心启示**：
- 许多“选若干点最优”的几何题，候选点最后都会落到凸包上。
- 学会先削掉内部点，能大幅降低后续复杂度。

### 题目 15：洛谷 P7401 [COCI 2020/2021 #5] Planine

> 山谷照明问题里，真正决定可见性的往往是若干峰顶形成的可视包络。

**分析**：这题不是“直接求凸包”，但核心仍是维护单调地形上的上凸壳/可视包络：凹进去的峰谷不会改变远处的切线关系。构出可视凸壳后，再做区间覆盖或贪心。

```
算法思路：单调地形上的可视性，本质上仍然依赖“谁留在最外层包络上”。
关键步骤：抽取关键峰点；维护单调凸壳；按切线可见区间做扫描。
时间复杂度：常见做法为 O(n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
vector<Point> hull;
for (const auto& p : peaks) {
    while (hull.size() >= 2 && cross(hull[hull.size() - 2], hull.back(), p) >= 0) hull.pop_back();
    hull.push_back(p);
}
```

答案位置：外层扫描或贪心维护的最优可见区间/最少灯数

**代码讲解**：
- `while (...) hull.pop_back()` 会删掉使包络不再保持单调凸性的峰点，留下来的才是真正决定远处视线的关键峰。
- `hull.push_back(p)` 把新峰并入当前可视包络，后续无论做覆盖还是切线扫描，都是围绕这条外层壳展开。

**核心启示**：
- **单调壳也是凸包思想**：哪怕题面是山、灯和视线，关键仍是维护最外层包络。
- **切线决定区间**：一旦切线端点可单调移动，就能把可见性转成扫描问题。


## 五、常见技巧与思路总结

### 5.1 是否保留共线点，先从题意倒推

最稳妥的判断方式不是背模板，而是反问自己：题目最终需要的是“最少顶点集合”，还是“整条边界上的所有点”。前者多用严格凸包，后者必须保留共线点。

### 5.2 整数几何优先，浮点只留到最后

只要输入坐标是整数，叉积与转向判断都尽量用整数完成。浮点比较只在最终输出距离、面积时再参与，这样能最大限度避免精度误差。

### 5.3 去重是模板的一部分，不是可选项

排序后去重看似不起眼，但它会直接影响：

- 点数过少时的特判
- 共线点数量
- 上下凸壳拼接时是否产生重复端点

### 5.4 输出顺序与封闭方式要统一

常见模板返回的是“不重复起点”的逆时针点序；如果你在后续又手动把首点补到末尾，就不要再用 `% size` 做环，否则容易多算一条边。

### 5.5 凸包题的识别信号

下面这些描述经常对应凸包：

- 求包住所有点的最小围栏
- 只和最外层轮廓有关
- 问最远点对、最小包围形状
- 给很多点，但内部点明显不重要

一旦识别出这些信号，后面的题就会顺很多。

## 六、解题思路框架

### 6.1 点集题如何判断先上凸包

```
看到平面点集题
  -> 问答案是否只取决于最外层边界？
      -> 是：先求凸包
      -> 否：继续看是否需要最近点对、扫描线、KD-Tree 等其他模型
  -> 求出凸包后
      -> 若求周长/面积：直接遍历
      -> 若求最远点对/宽度：接旋转卡壳
      -> 若多次点内查询：接二分判定
```

### 6.2 凸包模板落地 checklist

| 检查项 | 说明 |
|--------|------|
| 是否排序去重 | 没去重容易退化出错 |
| 是否确认共线策略 | `<= 0` 还是 `< 0` |
| 是否处理 `n <= 2` | 单点/双点时不能直接套完整公式 |
| 是否统一点序 | 一般使用逆时针且不重复首点 |
| 是否分离“构壳”和“算答案” | 模板更容易复用 |

### 6.3 复杂度判断模板

- 求凸包：`O(n log n)`，瓶颈几乎总在排序
- 凸包后遍历：通常是 `O(m)`，其中 `m` 为凸包顶点数
- 多次询问：常见为 `O(n log n + q log m)`

因此只要题目允许先做一次排序预处理，凸包通常就是一条非常稳的路线。

## 七、终极总结与归纳

### 7.1 凸包的本质

**一句话**：凸包就是把“无序点集”压缩成“有序外边界”。

一旦外边界被提取出来，很多原本看似需要处理所有点的问题，都会立刻降维：内部点被过滤，边界顺序被建立，后续算法因此变得更快、更稳定、更可复用。

### 7.2 核心要点

1. **叉积判转向是灵魂**：左转保留、右转弹出，共线点再按题意处理
2. **Andrew 单调链最实战**：实现短、稳定、便于修改是否保留共线点
3. **排序去重不可省**：这是模板稳定性的前提
4. **内部点常常无贡献**：会先缩候选集合，是几何题的重要思维
5. **凸包后还有很多二次操作**：面积、周长、点内判定、旋转卡壳都依赖它
6. **退化情形要单独看**：单点、双点、全共线往往是 WA 来源
