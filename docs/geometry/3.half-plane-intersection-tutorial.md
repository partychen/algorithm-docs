## 一、什么是半平面交

半平面交（Half-Plane Intersection）研究的是：给定若干个“有向直线左侧”的合法区域，把这些区域全部同时满足后，最后剩下什么形状。

因为半平面本身是凸集，而凸集的交仍然是凸集，所以半平面交的结果一定是一个**凸区域**：它可能是有界凸多边形，也可能退化成线段、点，甚至为空。

这类题的真正价值在于建模能力。很多题表面在讲“多条限制”“可行域”“每条边左侧”“是否存在满足全部条件的位置”，本质都可以转成半平面交。

| 表面题意 | 半平面交视角 |
|----------|--------------|
| 一堆线性不等式 | 每个不等式对应一个半平面 |
| 凸多边形求交 | 每条边都给出一个半平面 |
| 多边形 Kernel | 每条边的内侧共同构成可见区域 |
| 二分答案判定 | 判定某个可行域是否非空 |

如果你已经会凸包，那么半平面交可以理解为“从点集边界问题，升级到线性约束交集问题”。

## 二、核心思想与前置知识

### 2.1 用有向直线表示半平面

竞赛里通常规定：一条有向直线 `p + t * d` 的**左侧**是合法半平面。这样做的好处是统一，只要判断某个点是否在直线左侧即可。

```cpp
const double EPS = 1e-9;

struct Point {
    double x, y;
    Point operator-(const Point& other) const { return {x - other.x, y - other.y}; }
};

double cross(const Point& a, const Point& b) {
    return a.x * b.y - a.y * b.x;
}

struct Line {
    Point p, d;  // p 为直线上一点，d 为方向向量
    double angle() const { return atan2(d.y, d.x); }
    bool onLeft(const Point& q) const {
        return cross(d, q - p) >= -EPS;
    }
};
```

只要方向统一，后面的所有“合法/不合法”判断都会变得非常自然。

### 2.2 直线交点与 `onLeft` 判定

半平面交算法的候选顶点，来自相邻边界直线的交点。因此你必须稳定写出“两直线求交点”。

```cpp
Point lineIntersection(const Line& a, const Line& b) {
    Point u = a.p - b.p;
    double t = cross(b.d, u) / cross(a.d, b.d);
    return {a.p.x + a.d.x * t, a.p.y + a.d.y * t};
}
```

一旦交点能稳定求出，剩下的问题就是：这个交点是否还在新加入半平面的左侧。

### 2.3 按极角排序与平行线去重

半平面交最经典的做法是：

1. 按直线方向的极角排序
2. 对极角相同的平行线，只保留“更靠内”的那条
3. 用双端队列维护当前有效边界

如果平行线不先去重，队列里会混入互相冲突或冗余的边，后面非常容易炸。

### 2.4 双端队列维护交集边界

排完序后，算法并不是“把所有交点都算出来”，而是维护一组当前还有效的边界线：

- 新半平面若把队尾交点裁掉，就弹队尾
- 新半平面若把队首交点裁掉，就弹队首
- 新半平面通过检查后，再入队

这和单调队列有点像：不合法的老元素会被新限制淘汰。

### 2.5 加大矩形边界处理无界情况

很多竞赛题希望最终交集是有界多边形。这时常见技巧是先加入一个足够大的矩形边界，把原问题限制在一个大框内。这样：

- 无界区域也能转成有界多边形输出
- 判空与求面积时更方便
- 数值上更稳定，代码结构更统一

## 三、分类与变体详解

### 3.1 基础半平面交（求交集多边形）

这是原型问题：若干个半平面，求它们交集形成的凸多边形顶点序列或面积。模板掌握后，其余题型大多只是换一种建模来源。

### 3.2 凸多边形求交

一个逆时针凸多边形的每条边，都天然对应“边的左侧”这个半平面。因此多个凸多边形求交，本质就是把所有边丢进同一个半平面交模板里。

### 3.3 多边形 Kernel（核）问题

简单多边形内部，能看到全部边界的点集叫作 Kernel。对每条边取其内侧半平面，全部求交，得到的就是 Kernel。

这类题的关键不是算法，而是看出“看得到所有边”其实就是“同时落在每条边的内侧”。

### 3.4 可行域判定与判空

当题目问“是否存在一个点满足全部条件”时，通常根本不需要把完整多边形都输出，只要判断半平面交是否为空即可。

### 3.5 二分答案 + 半平面交判定

有些优化题会把答案二分成一个参数 `mid`，然后把“参数合法”转成一组线性限制。此时每次 `check(mid)` 都是一次半平面交判空。

### 模型对比总表

| 问题类型 | 半平面来源 | 目标 | 复杂度 |
|----------|------------|------|--------|
| 基础模板 | 直接给定直线 | 求交点/面积 | `O(n log n)` |
| 凸多边形求交 | 多边形边界 | 求交区域 | `O((n+m) log (n+m))` |
| Kernel | 简单多边形边 | 求可见区域 | `O(n log n)` |
| 可行域判定 | 线性不等式 | 判空 | `O(n log n)` |
| 二分答案 | 参数化不等式 | 可行性检查 | `O(log V · n log n)` |

## 四、15 道经典例题详解

### 🟢 入门级 Level 1：模板与凸多边形求交

### 题目 1：洛谷 P4196【模板】半平面交 / [CQOI2006] 凸多边形

> 给出若干逆时针凸多边形，求它们交集的面积。

**分析**：每条边都天然对应“边左侧”的合法半平面。把所有边丢进统一模板，极角排序后用双端队列维护有效边界，最后收集交点求面积。

```
算法思路：多凸多边形求交，本质就是一次大号半平面交。
关键步骤：枚举所有边；极角排序和平行去重；队列维护后求交多边形面积。
时间复杂度：O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
vector<Line> lines;
for (const auto& poly : polys) {
    for (int i = 0; i < (int)poly.size(); ++i) {
        int j = (i + 1) % poly.size();
        lines.push_back(makeLine(poly[i], poly[j]));
    }
}
vector<Point> inter = halfPlaneIntersection(lines);
```

答案位置：`polygonArea(inter)`

**代码讲解**：
- 前面的双重循环把每个凸多边形的每条边都转成有向直线，统一汇总成一个半平面集合。
- `halfPlaneIntersection(lines)` 求出公共交集顶点序列后，面积问题就退化成普通凸多边形面积。

**核心启示**：
- **边就是半平面**：凸多边形内部永远在每条边的同一侧。
- **模板统一**：多个多边形求交没必要两两裁剪，直接一次性丢进 HPI。

### 题目 2：POJ 2451 Uyuw's Concert

> 若干线性边界围出一块可行区域，要求最终多边形面积。

**分析**：这类题通常会额外补一个大矩形，把无界情况关进盒子里。之后就是标准半平面交：极角排序、平行线去重、双端队列维护交界。

```
算法思路：很多“求区域面积”的 HPI 题，第一步其实是处理无界。
关键步骤：加入大边界框；排序与去重；求交点并套面积公式。
时间复杂度：O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
addBoundingBox(lines, LIM);
sort(lines.begin(), lines.end(), byAngle);
vector<Point> poly = halfPlaneIntersection(lines);
double ans = polygonArea(poly);
```

答案位置：`ans = polygonArea(poly)`

**代码讲解**：
- `addBoundingBox(lines, LIM)` 先把可能无界的可行域关进足够大的边界框中。
- 排序后做半平面交得到 `poly`，最后用 `polygonArea(poly)` 才能稳定输出舞台面积。

**核心启示**：
- **先封边界框**：无界区域不先裁掉，后面的求面积就没有意义。
- **平行线去重**：同角度半平面里只留最靠内的一条。

### 题目 3：POJ 1279 Art Gallery

> 给定一个简单多边形，求内部能看到全部墙面的区域面积。

**分析**：这就是经典 Kernel。对每条边取房间内部所在的半平面，全部求交后得到的区域就是警卫可站立的位置。

```
算法思路：“看得到所有边”翻成几何语言，就是“在每条边的内侧”。
关键步骤：统一边方向；把每条边转成半平面；求交后算面积。
时间复杂度：O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
vector<Line> lines;
for (int i = 0; i < n; ++i) {
    int j = (i + 1) % n;
    lines.push_back(makeLine(poly[i], poly[j]));
}
vector<Point> kernel = halfPlaneIntersection(lines);
```

答案位置：`polygonArea(kernel)`

**代码讲解**：
- `makeLine(poly[i], poly[j])` 把房间每条边的内侧写成一个合法半平面。
- 全部边同时成立时得到的 `kernel`，正是能够看见全部墙面的站立区域。

**核心启示**：
- **Kernel 招牌题**：只要题目提“一个点能看见全部边界”，就该想到它。
- **边方向统一**：顺逆时针错了，合法半平面就会翻到外面。

### 🟡 进阶级 Level 2：Kernel 与可视区域

### 题目 4：洛谷 P2600 [ZJOI2008] 瞭望塔

> 在折线地形上选建塔位置，使塔高最小且能看到整条轮廓。

**分析**：把塔顶坐标视作变量 `(x, h)`，地形上每一段都会给出一条“塔顶必须在其上方”的线性约束。最终就是在参数平面里求若干半平面的交，并取其中 `h` 最小的位置。

```
算法思路：看地形的视线约束，一旦升维到参数平面，就会变成标准线性半平面。
关键步骤：把每段轮廓转成关于 `(x,h)` 的约束；求交区域；取最低点。
时间复杂度：O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
vector<Line> cons;
for (int i = 0; i + 1 < n; ++i) cons.push_back(buildConstraint(seg[i], seg[i + 1]));
vector<Point> poly = halfPlaneIntersection(cons);
double ans = lowestY(poly);
```

答案位置：外层在可行域上计算出的最优位置或最优值

**代码讲解**：
- `if (cross(...) < 0) swap(a, b)` 的作用是统一直线方向，保证“左侧就是合法侧”。
- `lines.push_back(makeLine(a, b))` 之后，所有原始几何约束都会被压成同一种半平面格式。

**核心启示**：
- **先换坐标系**：参数平面建模是这题最关键的一步。
- **最优在边界**：线性目标函数最优值通常落在交集边界顶点。

### 题目 5：洛谷 P5696 [CTSC1998] 监视摄像机

> 判断一个房间能否放置一台摄像机，使其看到房间全部角落。

**分析**：本质仍是 Kernel 判空。和 Art Gallery 的区别只是这里只问存在性，因此不必关心面积，最终交集非空即可。

```
算法思路：同一套 HPI 模板，问面积还是问判空，只差最后一步。
关键步骤：将每条墙转为内侧半平面；求交；判断是否为空。
时间复杂度：O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
vector<Point> kernel = halfPlaneIntersection(lines);
bool ok = !kernel.empty();
```

答案位置：`ok = !kernel.empty()`

**代码讲解**：
- `buildVisibilityConstraints(cameras, room)` 先把监视器视野和房间边界一起翻成半平面。
- 只要 `halfPlaneIntersection(lines)` 的结果非空，就说明至少存在一个点同时满足全部视野限制。

**核心启示**：
- **先识别成 Kernel**：房间 + 一台摄像机 = 典型单点可视区域。
- **判空更常见**：很多 HPI 题根本不要求输出完整多边形。

### 题目 6：POJ 3130 How I Mathematician Wonder What You Are!

> 判断给定多边形是否存在非空 Kernel。

**分析**：这是半平面交判空的另一道经典题。模型和 Art Gallery 几乎一样，但更强调数值稳定与“空、点、线段”这些退化交集的处理。

```
算法思路：在 HPI 里，“非空”不只意味着面积大于零，退化到点和线段也可能算合法。
关键步骤：按边建半平面；做 HPI；用 EPS 判断交集是否存在。
时间复杂度：O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
bool nonEmpty(const vector<Line>& lines) {
    auto poly = halfPlaneIntersection(lines);
    return !poly.empty();
}
```

答案位置：`nonEmpty(lines)` 的布尔返回值

**代码讲解**：
- `auto poly = halfPlaneIntersection(lines)` 得到的结果即便退化成点或线段，也可能仍然表示“存在可行解”。
- 因此这里用 `!poly.empty()` 判存在性，而不是强行要求面积必须大于零。

**核心启示**：
- **退化也要认**：有些题只要存在一点可行，就不要求面积正。
- **EPS 管理**：判平行、判在左侧都要统一误差策略。

### 🟠 提高级 Level 3：参数化判定器

### 题目 7：洛谷 P3256 [JLOI2013] 赛车

> 若一辆赛车曾在某时刻领跑，就算得奖，求所有可能得奖的赛车。

**分析**：把位置函数 `k + vt` 看成平面上的直线，曾领跑等价于它出现在上包络线上。常见做法会用对偶/半平面视角理解：保留下来的就是若干约束共同形成的可见边界。

```
算法思路：半平面交与凸包/上包络在对偶意义下属于同一类外层边界问题。
关键步骤：按速度排序去重；维护上包络；输出曾经可见的线。
时间复杂度：O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
sort(cars.begin(), cars.end(), bySpeed);
for (auto car : cars) {
    while (stk.size() >= 2 && bad(stk[stk.size() - 2], stk.back(), car)) stk.pop_back();
    stk.push_back(car);
}
```

答案位置：栈 `stk` 中保留下来的赛车编号

**代码讲解**：
- `sort(cars.begin(), cars.end(), bySpeed)` 先按斜率/速度排序，便于维护上包络的单调性。
- `while (bad(...)) stk.pop_back()` 会删掉永远不可能出现在外层包络上的直线，只保留真正可能领跑的车。

**核心启示**：
- **对偶思想**：HPI 不只会出现在“显式直线左侧”，也会躲在线性函数包络里。
- **同斜率去重**：同速度时只保留更有机会出现在外层的一条。

### 题目 8：洛谷 P3222 [HNOI2012] 射箭

> 前 k 个靶子能否被同一条抛物线全部穿过，问最多能通过多少关。

**分析**：把抛物线参数当成平面中的一个点后，每个靶子都会转成若干线性不等式。于是“前 k 关可行”就能用半平面交判空，再配合二分答案。

```
算法思路：看似曲线拟合的问题，很多时候能在参数空间里线性化。
关键步骤：把靶子约束转成参数平面的半平面；二分关数；HPI 判空。
时间复杂度：O(log n · m log m)
空间复杂度：O(m)
```

**C++ 代码**：
```cpp
bool check(int mid) {
    vector<Line> lines = buildConstraints(mid);
    return !halfPlaneIntersection(lines).empty();
}
```

答案位置：`check(mid)`

**代码讲解**：
- `buildConstraints(mid)` 把前 `mid` 个靶子的要求翻成参数平面中的半平面组。
- `!halfPlaneIntersection(lines).empty()` 就是在问这些参数限制是否还有公共可行区，供外层二分使用。

**核心启示**：
- **参数化再几何化**：不是在原平面射箭，而是在参数平面判断可行。
- **二分 + 判空**：优化题里，HPI 常作为 `check(mid)` 出现。

### 题目 9：洛谷 P11264 [COTS 2018] 逃遁 Bijeg

> 劫匪选择逃跑方向，判断能否永远逃脱；若不能，求最远还能逃多远。

**分析**：把“某个方向会被某警察抓住”翻成关于方向/速度参数的线性约束后，可行逃脱方向形成一个凸区域。判能否逃脱看半平面交是否为空；若不可逃，再沿边界找最优值。

```
算法思路：方向选择题里，关键是把连续角度空间转成凸可行域。
关键步骤：建立每个警察对应的限制半平面；判是否存在可行方向；必要时在边界优化距离。
时间复杂度：常见为 O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
vector<Line> lines = buildDirectionConstraints(police);
auto poly = halfPlaneIntersection(lines);
if (!poly.empty()) return -1;
double ans = farthestCaptureDistance(lines);
```

答案位置：存在合法逃跑方向时返回 `-1`，否则返回 `ans`

**代码讲解**：
- `halfPlaneIntersection(lines)` 先判断是否还存在任何能一直逃脱的方向。
- 若交集为空才继续调用 `farthestCaptureDistance(lines)`，说明问题从判空切换到了边界最优值计算。

**核心启示**：
- **先判存在性**：只要还有一个合法方向，后面的距离优化都不用做。
- **边界最优**：不可逃时，最优值通常落在可行域边界顶点上。

### 🔴 挑战级 Level 4：对偶化与概率建模

### 题目 10：洛谷 P4642 [BJWC2008] 方程

> 在两条线性等式与非负约束下，最大化第三个线性函数，多次询问 `(S,T)`。

**分析**：把每个物品看成参数平面中的限制后，查询 `(S,T)` 的最优值就变成在某个凸可行域上做线性规划；本质仍是半平面交/凸包边界查询。

```
算法思路：二维线性规划和 HPI 是天然一对：可行域由半平面交出，最优值落在顶点。
关键步骤：对偶化约束；构造外层凸边界；查询时在线性函数上取最值。
时间复杂度：预处理 O(n log n)，单询问 O(log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
buildEnvelope(items);
for (auto [S, T] : queries) {
    int id = bestVertex(S, T);
    answerQuery(id, S, T);
}
```

答案位置：每次询问的最优值

**代码讲解**：
- `buildEnvelope(items)` 先把所有约束转成一个外层凸边界，后续查询都只在这个边界上找最优点。
- `bestVertex(S, T)` 给出当前线性目标函数最优的顶点编号，再由 `answerQuery(...)` 输出答案。

**核心启示**：
- **二维 LP**：变量维数低时，可行域就是几何对象。
- **最优在顶点**：线性目标函数不会在凸多边形内部取独特最优。

### 题目 11：洛谷 P4250 [SCOI2015] 小凸想跑步

> 在凸多边形内随机选点，求某个固定三角形面积最小的概率。

**分析**：条件 `S(p,0,1)` 最小可以整理成一组关于点 `p` 的线性不等式，因此满足条件的点集仍是一个凸多边形。求出这个区域面积，再除以原多边形面积即可。

```
算法思路：概率几何题一旦把事件翻成线性约束，就回到了 HPI。
关键步骤：把面积比较式化成半平面；求交区域面积；除以原面积。
时间复杂度：O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
vector<Line> cons;
for (int i = 1; i + 1 < n; ++i) cons.push_back(buildAreaConstraint(v[0], v[1], v[i], v[i + 1]));
vector<Point> good = halfPlaneIntersection(cons);
double prob = polygonArea(good) / polygonArea(poly);
```

答案位置：`prob`

**代码讲解**：
- `buildAreaConstraint(...)` 把“某个三角形面积不超过阈值”的条件整理成一条线性边界。
- 求得满足条件的交集 `good` 后，`polygonArea(good) / polygonArea(poly)` 就是所求概率。

**核心启示**：
- **面积比较线性化**：把叉积展开后，关于点坐标往往就是一次式。
- **先找事件区域**：概率题常先算满足条件区域面积占比。

### 题目 12：洛谷 P3297 [SDOI2013] 逃考

> 每个位置只受最近亲戚监控，求逃到边界时最少会被多少人监控。

**分析**：“离谁最近”天然对应 Voronoi 风格的半平面交：某个亲戚的监控区域就是若干垂直平分线半平面的交。先把每个区域凸化，再在区域邻接图上做最短路。

```
算法思路：最近点划分本质上是半平面交出来的凸区域拼图。
关键步骤：对每个亲戚构造监控单元；建立区域邻接；图上求最优路径。
时间复杂度：O(n^2 log n) 量级
空间复杂度：O(n^2)
```

**C++ 代码**：
```cpp
for (int i = 0; i < n; ++i) {
    vector<Line> cell;
    for (int j = 0; j < n; ++j) if (i != j) cell.push_back(perpendicularBisectorHalfPlane(i, j));
    region[i] = halfPlaneIntersection(cell);
}
```

答案位置：`region[i]` 以及后续图论答案

**代码讲解**：
- 内层枚举 `j` 时不断加入 `i` 相比 `j` 更近的垂直平分线半平面，最终交集就是亲戚 `i` 的 Voronoi 单元。
- `region[i] = halfPlaneIntersection(cell)` 先把每个人的监控区域求出来，后续才能建邻接图跑最短路。

**核心启示**：
- **Voronoi 单元**：“比别人更近”就是一串半平面同时成立。
- **先分区再图论**：很多综合题都是 HPI 建图的套路。

### 题目 13：洛谷 P3493 [POI 2009] WSP-Island

> 复杂道路与禁区共同决定合法区域，再在其上寻找最短安全路线。

**分析**：这类综合题的难点在于先把复杂图形拆成若干凸单元。凸岛、道路和禁区的交互区域常通过半平面裁剪得到，再在这些单元和交点组成的图上跑最短路。

```
算法思路：HPI 有时不是终点，而是把几何场景离散化成图结构的工具。
关键步骤：裁出合法凸单元；抽取端点与交点；图上求最短路径。
时间复杂度：核心裁剪常为 O(n log n)
空间复杂度：O(n^2)
```

**C++ 代码**：
```cpp
vector<Point> cell = island;
for (const auto& ban : banned) cell = polygonClip(cell, ban);
buildGraphFromCells(cell);
```

答案位置：由 `buildGraphFromCells(...)` 产出的最短路答案

**代码讲解**：
- `polygonClip(cell, ban)` 反复把岛屿区域裁掉禁区，保留下来的就是合法凸单元。
- 几何裁剪完成后再 `buildGraphFromCells(cell)`，说明 HPI/裁剪在这里只负责准备图结构。

**核心启示**：
- **先裁再图**：复杂路线题里，几何部分常负责产出图节点与边。
- **凸单元更友好**：一旦区域被裁成凸形，后续距离和连通性都更清晰。

### 题目 14：洛谷 P7397 雨水收集系统（2021 CoE-I E）

> 移动的凸多边形雨云在一段时间内覆盖若干屋顶，求总受雨面积。

**分析**：把时间区间内的平移效果看成一次 Minkowski 扩张后，单个屋顶是否被覆盖、覆盖到多少，都可转成凸多边形与矩形的裁剪面积问题。

```
算法思路：移动中的凸体，经常可以化成“静态对象 + 额外平移向量”的裁剪问题。
关键步骤：求时间段对应的云轨迹外包；与每个屋顶矩形裁剪；累加面积。
时间复杂度：O((n+m) log(n+m)) 级别
空间复杂度：O(n + m)
```

**C++ 代码**：
```cpp
vector<Point> cover = movingCloudEnvelope(cloud, s, e, v, dt);
for (const auto& rect : roofs) {
    vector<Point> inter = convexClip(cover, rect);
    ans += polygonArea(inter);
}
```

答案位置：`ans`

**代码讲解**：
- `movingCloudEnvelope(...)` 先把随时间平移的雨云打包成一个静态覆盖区域。
- 对每个屋顶做 `convexClip` 后，把交集面积累加到 `ans`，就得到了总受雨面积。

**核心启示**：
- **动变静**：把平移过程打包成一个静态覆盖区域，题就容易很多。
- **矩形也是半平面组**：轴对齐矩形本质上就是四个简单半平面。

### 题目 15：UVa 11265 The Sultan's Problem

> 从一个大矩形宫殿出发，被多条墙线不断裁剪，保留与喷泉同侧的部分。

**分析**：这是最直观的“反复裁剪凸多边形”题。你既可以直接做多边形裁剪，也可以把每条保留侧统一写成半平面，最终交集就是喷泉仍能到达的区域。

```
算法思路：半平面交与逐次多边形裁剪，在凸区域上是等价的两种写法。
关键步骤：以宫殿矩形为初始区域；逐条取与喷泉同侧；输出剩余面积。
时间复杂度：O(n log n) 或 O(n^2)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
vector<Point> poly = palace;
for (const auto& wall : walls) {
    Line keep = orientedByFountain(wall, fountain);
    poly = polygonClip(poly, keep);
}
```

答案位置：剩余区域面积 `polygonArea(poly)`

**代码讲解**：
- `orientedByFountain(wall, fountain)` 先把每面墙的方向统一成“喷泉所在侧为保留侧”。
- `poly = polygonClip(poly, keep)` 表示用这条半平面连续裁剪宫殿区域，最终剩下的就是答案区域。

**核心启示**：
- **裁剪法直观**：初始区域已知且边数不大时，逐次裁剪往往更好写。
- **同侧别弄反**：保留喷泉所在侧，是整题最关键的几何判断。

## 五、常见技巧与思路总结

### 5.1 方向统一比公式本身更重要

同一套 `onLeft` 判定之所以好用，是因为所有线都约定“左侧合法”。如果有的边按顺时针，有的边按逆时针，结果会立刻乱掉。

### 5.2 平行线去重必须保留“更靠内”的那条

极角相同的两条线若方向一致，只保留限制更强的那条。否则：

- 冗余线会让队列变脏
- 冲突线会让结果不稳定
- 最终交点序列可能出现重复或非法

### 5.3 EPS 要集中管理

浮点几何最忌讳这里写 `1e-8`、那里写 `1e-10`。把 `EPS` 统一放在模板顶部，所有比较都走同一标准，调试会轻松很多。

### 5.4 加边界框是处理无界区域的常用工程技巧

有些题理论上允许无界交集，但程序里往往更希望拿到一个真正封闭的多边形。这时先套一层足够大的矩形，是非常常见也非常有效的做法。

### 5.5 判空、退化与输出目标要提前想清楚

半平面交可能得到：空集、点、线段、有界多边形。不同题目对这些情况的处理完全不同，所以模板外侧一定要先明确“我要的到底是什么”。

## 六、解题思路框架

### 6.1 看到哪些题面信号要联想到半平面交

```
如果题目出现：
  - 多条线性不等式
  - 每条边的内侧/左侧/合法侧
  - 求多个凸约束的公共区域
  - 二分答案后得到一组线性限制
那么就应优先尝试半平面交
```

### 6.2 半平面交模板检查表

| 检查项 | 说明 |
|--------|------|
| 方向是否统一 | 默认左侧合法 |
| 是否按极角排序 | 队列维护的前提 |
| 平行线是否去重 | 只保留更紧约束 |
| 是否统一 `EPS` | 避免浮点判定混乱 |
| 是否需要加大矩形 | 看题目是否允许无界 |

### 6.3 调试顺序建议

先拿一个简单矩形或三角形做样例，确认：

1. 每条边的左侧确实指向内部
2. 平行线去重后仍保留正确约束
3. 最终交点按顺序围成闭环

只要这三步走通，大多数模板 bug 都能很快定位。

## 七、终极总结与归纳

### 7.1 半平面交的本质

**一句话**：半平面交就是把“多条线性几何限制”统一压成一个凸可行域。

它的强大之处不在于公式多，而在于建模统一：不管原题讲的是多边形、约束、视野范围还是二分判定，只要最后都能写成“在若干条有向直线左侧”，就能复用同一套模板。

### 7.2 核心要点

1. **所有限制先统一成“左侧合法”**：方向一乱，模板必错
2. **极角排序是入口**：不排序，就无法稳定维护边界顺序
3. **平行线去重是关键工程细节**：决定结果是否干净可用
4. **双端队列维护的是有效边界线，不是所有交点**
5. **半平面交最常作为建模与判定器**：很多题只需要判空，不需要完整多边形
6. **加大矩形是高频技巧**：让无界情况变成有界输出，提升实现稳定性
