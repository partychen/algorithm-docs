## 一、什么是旋转卡壳

旋转卡壳（Rotating Calipers）是一类建立在**凸多边形有序边界**上的线性扫描技巧。它并不是某一个固定题目模板，而是一整套“随着边界方向单调旋转、让若干指针同步前进”的方法论。

你可以把它想成一把沿着凸包旋转的卡尺：当一条支撑线的方向发生变化时，与之对应的“最远点”“最左点”“最右点”等极值位置也会沿着凸包单调移动，于是很多原本看起来需要 `O(n^2)` 枚举的几何量，都能被压到 `O(n)`。

| 常见应用 | 典型答案 |
|----------|----------|
| 凸包直径 | 最远点对 |
| 凸包宽度 | 最小支撑线距离 |
| 最小面积/周长包围矩形 | 最优外接矩形 |
| 两凸包距离 | 最近/最远距离 |
| 最大面积三角形 | 凸包上三点最优 |

旋转卡壳几乎总是和凸包配套出现：**先求凸包，再在线性时间里旋转扫描**。

## 二、核心思想与前置知识

### 2.1 对踵点与支撑线

对踵点（Antipodal Pair）是旋转卡壳里最常见的关键词。直观地说，就是存在一对平行支撑线分别接触凸包的两个顶点或边，这两个接触位置就是一对“相对”的极值点。

对凸包直径而言，最远点对一定出现在某组对踵点里，所以不必枚举所有顶点对。

```cpp
long long area2(const Point& a, const Point& b, const Point& c) {
    return llabs((b.x - a.x) * (c.y - a.y) - (b.y - a.y) * (c.x - a.x));
}
```

旋转卡壳里最常见的“推进条件”就是：比较相邻状态下的面积（或投影）是否继续增大。

### 2.2 凸包直径模板

设 `hull` 为逆时针凸包。枚举边 `(i, i+1)` 时，让指针 `j` 保持为当前边的最优对踵点；若 `j+1` 让面积更大，就继续前进。

```cpp
long long dist2(const Point& a, const Point& b) {
    long long dx = a.x - b.x;
    long long dy = a.y - b.y;
    return dx * dx + dy * dy;
}

long long diameter2(const vector<Point>& hull) {
    int n = hull.size();
    if (n <= 1) return 0;
    if (n == 2) return dist2(hull[0], hull[1]);

    long long ans = 0;
    int j = 1;
    for (int i = 0; i < n; ++i) {
        int ni = (i + 1) % n;
        while (area2(hull[i], hull[ni], hull[(j + 1) % n]) > area2(hull[i], hull[ni], hull[j])) {
            j = (j + 1) % n;
        }
        ans = max(ans, dist2(hull[i], hull[j]));
        ans = max(ans, dist2(hull[ni], hull[j]));
    }
    return ans;
}
```

关键在于：`j` 在整轮扫描里只会绕凸包走一圈，所以总复杂度是 `O(n)`，不是 `O(n^2)`。

### 2.3 凸包宽度与最小距离

宽度问题本质上是：对每条边，找与它最“对立”的支撑线位置。判断最优指针是否前进，通常还是借助面积（等价于底乘高）变化。

```cpp
long double pointLineDistance(const Point& a, const Point& b, const Point& p) {
    return (long double)area2(a, b, p) / hypotl(b.x - a.x, b.y - a.y);
}
```

宽度与直径最大的区别在于：

- 直径要取**最大距离**
- 宽度要对所有边取“最远点到边距离”的**最小值**

### 2.4 最小面积/周长包围矩形

当我们枚举凸包的一条边作为矩形底边时，矩形的另外三个极值位置分别对应：

- 与底边最远的点：决定高度
- 在底边方向投影最小的点：决定左边界
- 在底边方向投影最大的点：决定右边界

这就是经典的“四卡壳”模型。多个指针都会随边方向单调前进，因此总复杂度仍然是线性的。

### 2.5 多指针同步旋转的本质

旋转卡壳好写的根本原因，是极值点随方向变化具备**单调性**。只要你能证明“最优指针不会回退”，就可以尝试用双指针或多指针把问题从二次枚举压成线性扫描。

这也是旋转卡壳最重要的思维价值：它不是背模板，而是学会识别“方向单调变化 + 极值位置单调移动”的结构。

## 三、分类与变体详解

### 3.1 凸包直径（最远点对）

这是最经典也最常见的卡壳应用。先求凸包，再枚举边并维护对踵点。若题目只要求最远距离，通常比较平方距离即可。

### 3.2 凸包宽度（最小支撑线距离）

仍然是围绕“当前边对应的最远对踵点”展开，但最终答案不是“全局最大”，而是“所有边对应最远距离中的最小值”。

### 3.3 最小面积/周长包围矩形

这是旋转卡壳里最具代表性的多指针问题。你需要同时维护高度、左右投影，常数较大，但本质仍然是“各极值指针单调前进”。

### 3.4 最大面积三角形

在凸包上枚举两个点后，第三个点的最优位置也具有单调性，因此可以把本来三重循环的枚举压缩。很多人第一次接触这个问题时，会意识到“卡壳并不只服务于两点问题”。

### 3.5 两个凸多边形之间的距离

如果两个对象都已经是凸包，那么最近/最远距离也能利用支撑线方向与极值点单调性来处理。虽然实现细节更繁琐，但本质依旧是旋转卡壳。

### 模型对比总表

| 问题类型 | 指针数量 | 判定核心 | 复杂度 |
|----------|----------|----------|--------|
| 直径 | 2 | 面积是否继续增大 | `O(n)` |
| 宽度 | 2 | 点到边距离是否更远 | `O(n)` |
| 最小包围矩形 | 4 | 投影极值同步维护 | `O(n)` |
| 最大面积三角形 | 3 | 第三个点单调推进 | `O(n^2)` 或更优实现 |
| 两凸包距离 | 2~4 | 支撑线与投影变化 | `O(n + m)` |

## 四、15 道经典例题详解

### 🟢 入门级 Level 1：对踵点与基础三指针

### 题目 1：洛谷 P1452【模板】旋转卡壳 / [USACO03FALL] Beauty Contest G

> 给定平面点集，求最远点对距离平方。

**分析**：这是旋转卡壳的起点题。先求凸包，再枚举边 `(i,i+1)`，让对踵点 `j` 只沿面积增大的方向前进。

```
算法思路：先把候选压到凸包上，再用对踵点单调推进求直径。
关键步骤：凸包预处理；比较相邻面积；更新最远距离平方。
时间复杂度：O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
long long ans = 0;
int j = 1;
for (int i = 0; i < n; ++i) {
    int ni = (i + 1) % n;
    while (area2(hull[i], hull[ni], hull[(j + 1) % n]) > area2(hull[i], hull[ni], hull[j])) j = (j + 1) % n;
    ans = max(ans, max(dist2(hull[i], hull[j]), dist2(hull[ni], hull[j])));
}
```

答案位置：`ans`

**代码讲解**：
- `while (...)` 固定当前边后，只要 `j+1` 让对应三角形面积更大，就继续推进对踵点 `j`。
- `ans = max(...)` 同时比较 `i`、`ni` 到 `j` 的距离平方，避免漏掉边切换时的最优点对。

**核心启示**：
- **面积驱动**：很多卡壳推进都能转成“下一点面积是否更大”。
- **先比平方**：距离题先比平方，最后再开方更稳。

### 题目 2：POJ 2187 Beauty Contest

> 与 P1452 同源的经典直径题，更强调边界特判。

**分析**：思路不变，依旧是“凸包 + 卡壳”，但这题更容易暴露全共线、点数过少与整型范围等工程细节。

```
算法思路：同一模型换平台，重点往往变成模板的鲁棒性。
关键步骤：构造严格凸包；处理二点与共线；再做直径卡壳。
时间复杂度：O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
if (hull.size() == 2) return dist2(hull[0], hull[1]);
long long ans = diameter2(hull);
```

答案位置：`ans`

**代码讲解**：
- `if (hull.size() == 2)` 先处理退化成线段的情况，这时直径就是两点间距离。
- `diameter2(hull)` 把真正的卡壳逻辑封装起来，外层只需要拿到最远距离平方作为答案。

**核心启示**：
- **模板落地**：算法会写不代表能过边界数据。
- **退化处理**：直径题一定要先把 1 点、2 点、全共线说清楚。

### 题目 3：HDU 2202 最大三角形

> 从点集中选三个点，使三角形面积最大。

**分析**：先求凸包，再固定前两个点，让第三个点沿着凸包单调前进。它是从双指针过渡到三指针卡壳的代表题。

```
算法思路：最大面积三角形只会出现在凸包顶点上。
关键步骤：凸包预处理；外层枚举两点；第三点按面积单调推进。
时间复杂度：O(n log n + m^2)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
for (int i = 0; i < m; ++i) {
    int k = (i + 2) % m;
    for (int j = i + 1; j < m; ++j) {
        while (area2(hull[i], hull[j], hull[(k + 1) % m]) > area2(hull[i], hull[j], hull[k])) k = (k + 1) % m;
        ans = max(ans, area2(hull[i], hull[j], hull[k]));
    }
}
```

答案位置：`ans`

**代码讲解**：
- 外层枚举前两个顶点 `i, j`，把第三个点 `k` 作为单调推进的指针，不会为每个 `j` 从头重扫。
- `ans = max(ans, area2(...))` 负责记录当前找到的最大三角形面积的两倍值，最后按题意再换算即可。

**核心启示**：
- **三指针思想**：卡壳并不只服务于两点问题。
- **叉积比较**：面积题里叉积就是最自然的比较量。

### 🟡 进阶级 Level 2：宽度与最小包围矩形

### 题目 4：POJ 2079 Triangle

> 另一道最大面积三角形经典题，重点在环形实现。

**分析**：模型和 HDU 2202 一致，但更强调把凸包看成环。常见做法会复制一遍顶点或统一用取模，避免跨过首尾时漏解。

```
算法思路：同一模型最容易挂掉的地方，不是思想而是环的实现。
关键步骤：把凸包视作环；三指针推进；维护面积最大值。
时间复杂度：O(n log n + m^2)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
for (int i = 0; i < m; ++i) {
    int k = i + 2;
    for (int j = i + 1; j < m; ++j) {
        while (k + 1 < i + m && area2(h[i], h[j], h[k + 1]) >= area2(h[i], h[j], h[k])) ++k;
        ans = max(ans, area2(h[i], h[j], h[k]));
    }
}
```

答案位置：`ans`

**代码讲解**：
- `k + 1 < i + m` 这类条件是在把环形凸包摊平成链后，防止第三指针越过合法区间。
- `ans = max(...)` 仍然是最大三角形模板，只是这版实现更强调首尾衔接的细节。

**核心启示**：
- **先处理环**：卡壳很多 bug 都来自首尾衔接。
- **双倍数组**：把环摊成链，往往比频繁取模更稳。

### 题目 5：洛谷 P3187 [HNOI2007] 最小矩形覆盖

> 求覆盖所有点的最小面积矩形，并输出矩形四顶点。

**分析**：这是最典型的四卡壳：底边、最远点、最左投影点和最右投影点一起单调前进。真正难点是维护每个指针的几何含义。

```
算法思路：多个极值点随着边方向同步移动，就是多指针卡壳的核心。
关键步骤：求凸包；枚举底边；同步维护高度和左右投影。
时间复杂度：O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
for (int i = 0, j = 1, l = 1, r = 1; i < n; ++i) {
    int ni = (i + 1) % n;
    advanceFarPoint(i, ni, j);
    advanceLeftProj(i, ni, l);
    advanceRightProj(i, ni, r);
    ans = min(ans, rectangleArea(i, ni, j, l, r));
}
```

答案位置：`ans` 以及对应矩形四个顶点

**代码讲解**：
- `advanceFarPoint / advanceLeftProj / advanceRightProj` 分别维护高度、左边界和右边界三个极值指针。
- `rectangleArea(...)` 用这四个支撑点恢复当前矩形面积，外层再用 `min` 保留全局最优。

**核心启示**：
- **四指针分工**：先想清楚谁维护高度、谁维护左右边界。
- **极值单调**：只要不回退，总复杂度就能压在线性。

### 题目 6：UVa 10173 Smallest Bounding Rectangle

> 给定凸多边形，求最小面积包围矩形。

**分析**：它比 P3187 更“纯模板”，因为输入已经是凸多边形。你可以更专注地观察方向变化如何带动四个支持点单调移动。

```
算法思路：当凸包前置被省掉时，四卡壳本体会更清楚。
关键步骤：按边方向旋转；更新四个支持点；计算当前矩形面积。
时间复杂度：O(n)
空间复杂度：O(1) 额外空间
```

**C++ 代码**：
```cpp
long double base = hypotl(hull[ni].x - hull[i].x, hull[ni].y - hull[i].y);
long double h = pointLineDistance(hull[i], hull[ni], hull[j]);
long double w = proj(hull[r], hull[i], hull[ni]) - proj(hull[l], hull[i], hull[ni]);
ans = min(ans, h * w);
```

答案位置：`ans`

**代码讲解**：
- `base` 是当前底边长度，`h` 是对应方向上的最大高度，`w` 是左右投影之差得到的矩形宽度。
- `ans = min(ans, h * w)` 表示对每个方向都算一遍包围矩形面积，再取最小值。

**核心启示**：
- **方向优先**：四卡壳处理的是方向变化，不是坐标轴上的扫描。
- **纯模板题**：非常适合拿来练包围矩形的完整实现。

### 🟠 提高级 Level 3：双凸包与多指针同步

### 题目 7：2011 ACM-ICPC World Finals Problem K Trash Removal

> 求凸多边形在所有方向上的最小宽度。

**分析**：宽度题和直径题推进逻辑很像，都是为每条边寻找最远对踵点。区别只在于答案更新从 `max` 变成了 `min`。

```
算法思路：推进规则常常不变，真正变化的是最后怎么取答案。
关键步骤：枚举边；维护对应对踵点；更新全局最小宽度。
时间复杂度：O(n)
空间复杂度：O(1) 额外空间
```

**C++ 代码**：
```cpp
long double width = 1e100;
for (int i = 0, j = 1; i < n; ++i) {
    int ni = (i + 1) % n;
    while (area2(hull[i], hull[ni], hull[(j + 1) % n]) > area2(hull[i], hull[ni], hull[j])) j = (j + 1) % n;
    width = min(width, pointLineDistance(hull[i], hull[ni], hull[j]));
}
```

答案位置：`width`

**代码讲解**：
- `while (...)` 的推进条件和直径题几乎相同，说明“最远对踵点”仍然由面积比较驱动。
- `width = min(width, pointLineDistance(...))` 把当前方向的最坏厚度记下来，再在所有方向里取最小。

**核心启示**：
- **同推不同取**：面积比较负责推进，`min/max` 才决定具体题目。
- **宽度视角**：本质是所有方向上最坏厚度的最小值。

### 题目 8：POJ 3608 Bridge Across Islands

> 求两个凸多边形之间的最近距离。

**分析**：双凸包最近距离可以用卡壳维护两条支撑边的相对方向。实现时通常要在“点到边”和“边到边”几个候选之间切换。

```
算法思路：双对象也能卡壳，只要相对方向具备单调性。
关键步骤：分别求凸包；找到初始支持点；按边法向同步推进。
时间复杂度：O((n + m) log(n + m))
空间复杂度：O(n + m)
```

**C++ 代码**：
```cpp
long double ans = 1e100;
for (int i = ia, j = ib, cnt = 0; cnt < n + m; ++cnt) {
    ans = min(ans, segmentDistance(a[i], a[(i + 1) % n], b[j], b[(j + 1) % m]));
    if (cross(edgeA(i), edgeB(j)) >= 0) i = (i + 1) % n;
    else j = (j + 1) % m;
}
```

答案位置：`ans`

**代码讲解**：
- 先分别对两组点求凸包、找到初始支撑边，之后按法向旋转同步推进两个指针。
- 每一步都要在点到边、边到边等候选距离里更新 `ans`，因为最近距离未必来自两个顶点。

**核心启示**：
- **候选要全**：最近距离未必是点对，也可能是点到边。
- **点序统一**：双凸包问题开写前先把顺逆时针方向统一好。

### 题目 9：ICPC WF Moscow Invitational Contest Problem F Framing Pictures

> 多幅图片/框架需要按方向对齐，代价由多个支持函数共同决定。

**分析**：这类题的共同特征是：答案只与若干方向上的极值点有关。先把对象凸化，再用多指针跟踪左右/上下支持点即可。

```
算法思路：旋转卡壳的普适性，在于它处理的是“方向变化下的极值轨迹”。
关键步骤：把对象凸化；枚举方向；同步更新多个支持点。
时间复杂度：常见为 O(n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
while (dot(dir, hull[(r + 1) % n]) > dot(dir, hull[r])) r = (r + 1) % n;
while (dot(perp(dir), hull[(u + 1) % n]) > dot(perp(dir), hull[u])) u = (u + 1) % n;
ans = min(ans, evaluateFrame(r, u));
```

答案位置：外层维护的最优框架值

**代码讲解**：
- `advanceFramePointers(...)` 负责把与当前边方向对应的多个极值点同时推进到最佳位置。
- `updateByCurrentSupports(...)` 再用这些支撑点计算当前框架代价，形成完整的多指针模板。

**核心启示**：
- **多指针一体化**：无非是同时维护多个方向上的极值点。
- **先问依赖哪些极值**：一旦只依赖支持点，就很可能能上卡壳。

### 🔴 挑战级 Level 4：排名、面积与形态优化

### 题目 10：洛谷 P4357 [CQOI2016] K 远点对

> 求第 K 远点对距离，`K` 很小但点数很多。

**分析**：先把点压到凸包上，再围绕每个顶点的对踵区间生成高质量候选。常见做法是“卡壳定位区间 + 堆扩展次优解”。

```
算法思路：卡壳不仅能求一个最值，也能帮你高效枚举最值附近的候选。
关键步骤：构造凸包；定位最优对踵区间；优先队列扩展前 K 大。
时间复杂度：O(n log n + K log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
struct Node {
    long long d;
    int i, l, r;
    bool operator<(const Node& other) const { return d < other.d; }
};
priority_queue<Node> pq;
for (int i = 0; i < m; ++i) pq.push(bestRange(i));
```

答案位置：`ans`

**代码讲解**：
- `heap.push(makeNode(...))` 先枚举候选远点对状态，把“第 k 大”问题转成按距离排序的状态扩展。
- `if (vis.insert(...).second)` 用于去重，防止同一状态被重复加入堆中导致复杂度失控。

**核心启示**：
- **候选生成**：先找到每个点最强的对踵区间，再从局部往外扩。
- **K 小突破**：不必枚举全部点对，只维护前若干个高质量候选。

### 题目 11：洛谷 P11441 [Code+#6] 进阶法师

> 由若干法师两两形成的保护区域，求整体覆盖面积。

**分析**：最终边界只会由极少数极值点对决定，因此常先求凸包，再随着方向旋转追踪最远对或支持点。题面问面积，但控制答案的仍是外层极点。

```
算法思路：面积题也能用卡壳，只要边界随方向单调变化。
关键步骤：提取外层点；按方向追踪极值点对；累加外层贡献。
时间复杂度：常见为 O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
for (int i = 0, j = 1; i < m; ++i) {
    while (dist2(hull[i], hull[(j + 1) % m]) >= dist2(hull[i], hull[j])) j = (j + 1) % m;
    addContribution(hull[i], hull[j]);
}
```

答案位置：`area`

**代码讲解**：
- `diameter2(hull)` 先找出决定最大直径圆的最远点对，说明控制答案的还是凸包直径。
- 半径由直径直接除以 `2` 得到，最后 `πr²` 就是题目要求的保护区域面积。

**核心启示**：
- **边界只看极值对**：先想清楚谁可能上外轮廓，再决定是否卡壳。
- **公式可变**：最终面积公式会变，但控制答案的点对常常是同一批。

### 题目 12：洛谷 P14802 [CCPC 2024 哈尔滨站] 凹包

> 从给定点中选出一些点组成面积最大的凹多边形。

**分析**：高层思路通常是“先拿到凸包最大面积，再寻找一次最小面积的内凹损失”。这些损失项往往可在凸包上借助三指针或切线单调性维护。

```
算法思路：卡壳不一定直接给答案，也可以用来找“从凸解退一步”的最小代价。
关键步骤：先求凸包面积；枚举凹折位置；用单调指针找最小损失。
时间复杂度：常见为 O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
long long loss = (1LL << 62);
for (int i = 0, j = 2; i < m; ++i) {
    while (betterLoss(i, (i + 1) % m, (j + 1) % m, j)) j = (j + 1) % m;
    loss = min(loss, triangleLoss(i, (i + 1) % m, j));
}
long long ans = hullArea2 - loss;
```

答案位置：基于 `hull` 继续计算出的最优值

**代码讲解**：
- `rebuildUsefulBoundary(points)` 的职责是把原本非纯凸的边界重构成真正有用的外层壳。
- 只有先把“有效边界”提取出来，后面的卡壳推进条件才会重新恢复单调性。

**核心启示**：
- **先凸再凹**：很多“最大凹形”题是从凸包答案往回扣一点。
- **最小损失单调**：一旦损失项可单调维护，就能继续套卡壳。

### 🟣 大师级 Level 5：综合旋转建模

### 题目 13：洛谷 P14707 [ICPC 2023 Tehran R] Largest Triangle

> 在单调地形或特殊边界结构中寻找最大三角形。

**分析**：虽然题面不是标准凸包，但常见做法依旧会先把候选边界压成若干凸链，再在凸链上做三指针或两链同步扫描。

```
算法思路：卡壳依赖的不是整块凸包，而是边界上的单调极值结构。
关键步骤：抽取有效凸链；固定一条边；第三点或另一条链单调前进。
时间复杂度：常见为 O(n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
for (int i = 1, k = 2; i + 1 < m; ++i) {
    while (k + 1 < m && area2(chain[0], chain[i], chain[k + 1]) >= area2(chain[0], chain[i], chain[k])) ++k;
    ans = max(ans, area2(chain[0], chain[i], chain[k]));
}
```

答案位置：`best`

**代码讲解**：
- `orderedBoundary(poly)` 先把题面的复杂边界整理成可按顺序扫描的点列。
- `best = maxTriangle(boundary)` 表示真正的求值阶段仍然是标准最大三角形卡壳，只是前置建模更重。

**核心启示**：
- **单调边界也够用**：不必拘泥于整张凸包，只要候选边界有序即可。
- **先筛有效链**：真正参与最优解的边界通常远少于原始输入。

### 题目 14：洛谷 P13335 [GCJ 2012 Finals] Twirling Towards Freedom

> 不断绕某些点做 90 度旋转，问若干步后离原点最远能到哪里。

**分析**：把“旋转一次后的可达位置”看成对当前点集做仿射变换，再只保留其外层极值点。随着方向变化，真正有贡献的状态总落在支持函数的极点上。

```
算法思路：卡壳的深层用法，是在状态转移里只保留每个方向上的最优极点。
关键步骤：维护可达集凸包；按四个方向旋转支持函数；转移极值状态。
时间复杂度：核心转移常为线性扫描
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
for (int s = 0; s < 4; ++s) {
    while (dot(dir[s], hull[(ptr[s] + 1) % m]) > dot(dir[s], hull[ptr[s]])) ptr[s] = (ptr[s] + 1) % m;
    nxt[s] = rotate90(hull[ptr[s]], center);
}
```

答案位置：外层维护的最优支撑点或最优值

**代码讲解**：
- `cross(edge, cand[next]) >= cross(edge, cand[cur])` 表示在当前旋转方向下，下一候选不比当前差。
- 只要比较结果仍然单调，`cur = next` 就能持续前进，保持整轮扫描的摊还线性。

**核心启示**：
- **状态也能凸化**：很多可达状态里，真正重要的只是每个方向上的极值。
- **四向分组**：旋转 90 度的题常天然分成四个方向状态。

### 题目 15：洛谷 P8222 [WFOI - 02] I wanna escape the shadow（阴影）

> 半径趋于无穷大时，只关心被照亮区域的极限方向覆盖。

**分析**：当半径趋于无穷大，面积问题会退化成“哪些方向被覆盖”。于是关键对象不再是普通长度，而是凸包上的极角区间；卡壳负责找最大可覆盖方向段。

```
算法思路：有些看似面积题，极限化之后会变成纯粹的方向覆盖问题。
关键步骤：先求外层极角序；维护覆盖区间右端；贪心或 DP 统计最少操作。
时间复杂度：常见为 O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：
```cpp
int j = 0;
for (int i = 0; i < m; ++i) {
    while (j + 1 < i + m && covered(i, j + 1)) ++j;
    far[i] = j;
}
```

答案位置：`shadow`

**代码讲解**：
- `bestRight(...)` 和 `bestLeft(...)` 分别找出当前方向上投影最右与最左的支撑点。
- `shadow = proj(..., hull[r]) - proj(..., hull[l])` 直接把两个投影极值相减，得到阴影长度。

**核心启示**：
- **极限先看方向**：半径无穷大时，很多几何量都会只剩角度信息。
- **右端单调**：区间右端只前进不回退，就是卡壳信号。

## 五、常见技巧与思路总结

### 5.1 没有凸包，就谈不上稳定的旋转卡壳

旋转卡壳默认输入是按逆时针排列的凸多边形。若原始点集还没过凸包，很多“单调前进”的性质根本不成立。

### 5.2 `while` 判定式只比较“下一步是否更优”

卡壳代码看起来像魔法，其实只有一句：**若下一位置更优，就继续前进**。至于“更优”是面积更大、投影更大，还是夹角更合适，要看具体问题。

### 5.3 平行边与相等值要提前定好策略

当两种选择同样优时，代码必须保证至少有一种推进方式，否则可能死循环。实践里通常写成：

- `>`：只在严格变好时前进
- `>=`：允许并列前进，但要确保不会错过候选答案

### 5.4 距离比较尽量先用平方或叉积

开方是最后一步。只要题目还在“比较谁更优”的阶段，优先用平方距离、叉积面积、投影大小等不会引入精度误差的量。

### 5.5 多指针能否摊还 `O(n)`，取决于是否不回退

只要每个指针在整轮中都只绕多边形走一圈，就能得到线性总复杂度。这是判断题目能否上旋转卡壳的核心标准。

## 六、解题思路框架

### 6.1 如何识别一道题能否用旋转卡壳

```
看到凸多边形/先能求凸包
  -> 目标是否与某个方向上的极值有关？
      -> 是：继续判断极值点会不会单调移动
          -> 会：尝试旋转卡壳
          -> 不会：考虑别的几何工具
```

### 6.2 旋转卡壳模板检查表

| 检查项 | 说明 |
|--------|------|
| 输入是否已是逆时针凸包 | 否则单调性不成立 |
| 初始指针是否合理 | 常取与第一条边最对立的位置 |
| 推进条件是否只看“下一步” | 卡壳的本质就是局部比较 |
| 更新答案位置是否完整 | 常要同时看 `i`、`i+1` 与 `j` |
| 是否处理 `n <= 2` | 退化时不必进入主循环 |

### 6.3 调试顺序建议

先在纸上画一个简单凸四边形，手动模拟 `i` 和 `j` 的移动顺序。只要你能把这张图走通，代码通常就不会离谱。

## 七、终极总结与归纳

### 7.1 旋转卡壳的本质

**一句话**：旋转卡壳就是在凸多边形上追踪“随方向旋转而单调移动的极值点”。

它的价值不只是快，而是把“看似全局枚举”的问题转写成“局部推进”的过程：每次只判断下一步要不要走，却能在全局上覆盖全部候选状态。

### 7.2 核心要点

1. **先有凸包，后有卡壳**：点集题通常先压到凸包上
2. **单调性决定一切**：没有“指针只前进不回退”的性质，就谈不上卡壳
3. **面积比较最常见**：很多推进条件都可转成叉积大小比较
4. **局部最优推进覆盖全局候选**：这是复杂度从平方降到线性的根本原因
5. **多指针仍然是同一思想**：矩形、三角形、双凸包距离都只是变体
6. **并列情况要小心**：平行边与相等值若处理不当，很容易漏解或死循环
