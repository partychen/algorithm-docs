# 一本通 数据结构高阶专题精选解题报告

这一组题从树状数组里的前缀统计一路走到树链剖分上的复杂路径维护，题型跨度很大，但主线始终围着“把局部限制压成可维护结构”展开：先在序列上练区间贡献和静态最值，再把维护对象搬到线段树，最后进入树上路径、子树和链剖分。

# 一、树状数组与序列贡献统计

这一章先把线性的序列题打通。题面里有修改、有计数、有三元组，也有看似组合、实则前缀贡献可拆的目标；真正的共同点，是都能先把答案翻译成若干个“前缀里比它大 / 小 / 早出现过多少”的统计量。

## 1. [1741：电子速度](http://ybt.ssoier.cn:8088/problem_show.php?pid=1741)

`树状数组` `区间贡献` `向量叉积`

### 题意

有 $n$ 个二维速度向量，支持单点修改，把第 $p$ 个电子改成 $(x,y)$；询问时给出区间 $[l,r]$，要求这段电子的飘升系数，也就是所有两两向量叉积平方和。

### 分析

关键不是直接枚举两两配对，而是先把公式拆开。设一个向量是 $(x_i,y_i)$，则 $(x_i y_j-x_j y_i)^2$ 展开后只会出现 $x^2$、$y^2$ 和 $xy$ 三类量。把区间里的所有项合并，可以得到答案正好是 $(\sum x_i^2)(\sum y_i^2)-(\sum x_i y_i)^2$。

这样一来，原题就从“二元配对”变成了“区间维护三个前缀量”。单点修改时分别更新 $x^2,y^2,xy$，区间询问时取三段和再代公式即可。模型想到这一步，数据结构就只是承载公式的容器。

### 核心代码

```cpp
const int MOD = 20170927;
BIT sx2, sy2, sxy;

void modify(int p, int x, int y) {
    sx2.add(p, 1LL * x * x % MOD - vx2[p]);
    sy2.add(p, 1LL * y * y % MOD - vy2[p]);
    sxy.add(p, 1LL * x * y % MOD - vxy[p]);
    vx2[p] = 1LL * x * x % MOD;
    vy2[p] = 1LL * y * y % MOD;
    vxy[p] = 1LL * x * y % MOD;
}
long long query(int l, int r) {
    long long a = sx2.sum(l, r), b = sy2.sum(l, r), c = sxy.sum(l, r);
    return (a * b - c * c) % MOD;
}
```

### 复杂度

单次修改、查询都是 $O(\log n)$，空间复杂度 $O(n)$。

## 2. [1742：导线问题](http://ybt.ssoier.cn:8088/problem_show.php?pid=1742)

`树状数组` `LIS` `方案计数`

### 题意

给出一排导线的电阻值，只能从左往右走，并且下一根导线的电阻必须更大。要求最长可走长度；若 `type=1`，还要输出达到这个最长长度的方案数。

### 分析

题面其实就是一个严格上升子序列。难点不在认出 LIS，而在于“长度”和“方案数”要一起维护：对当前位置 $a_i$，只需要知道所有比它小的值里，当前最好的长度是多少，以及这样的方案数一共有多少。

这正好适合值域上的树状数组。每个节点维护一对信息 $(len,cnt)$：先按长度取最大，长度相同再把方案数累加。处理到 $a_i$ 时查询 $[1,a_i-1]$ 的最优状态，再把当前位置产生的新状态回写到 $a_i$。于是“从左到右”的顺序，和“按值做前缀最优”的结构正好拼上。

### 核心代码

```cpp
struct Info { int len, cnt; };
Info merge(Info a, Info b) {
    if (a.len != b.len) return a.len > b.len ? a : b;
    return {a.len, (a.cnt + b.cnt) % MOD};
}
for (int i = 1; i <= n; i++) {
    Info pre = bit.query(a[i] - 1);
    if (!pre.len) pre.cnt = 1;
    Info cur = {pre.len + 1, pre.cnt};
    bit.update(a[i], cur);
}
```

### 复杂度

时间复杂度 $O(n\log V)$，空间复杂度 $O(V)$。

## 3. [1743：数羊](http://ybt.ssoier.cn:8088/problem_show.php?pid=1743)

`树状数组` `三元组计数` `排列`

### 题意

给定一个 $1\sim N$ 的排列，要求统计三元组 $(i,j,k)$，满足 $i<j<k$ 且 $a_i<a_k<a_j$。

### 分析

这个条件最适合把中间那一层拆开。若固定第三个位置 $k$，它前面可选的 $i$ 个数只和“前缀里比 $a_k$ 小的元素个数”有关；而真正让式子成立，还得在它前面再找一个 $j$，满足 $j<k$ 且 $a_j>a_k$。

因此先从左到右求出每个位置前面有多少个更小值 `leftLess[i]`。接着从右往左扫描，用另一棵树状数组按值维护这些 `leftLess` 的和：当处理 $j$ 时，查询所有值小于 $a_j$ 的位置贡献，就正好把所有满足 $a_k<a_j$ 的第三项都收进来了。题目看着是三维关系，真正可维护的却是两次一维统计的拼接。

### 核心代码

```cpp
for (int i = 1; i <= n; i++) {
    leftLess[i] = bit1.query(a[i] - 1);
    bit1.add(a[i], 1);
}
long long ans = 0;
for (int j = n; j >= 1; j--) {
    ans += bit2.query(a[j] - 1);
    bit2.add(a[j], leftLess[j]);
}
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(n)$。

## 4. [1744：跳台阶](http://ybt.ssoier.cn:8088/problem_show.php?pid=1744)

`树状数组` `贪心` `双链划分`

### 题意

有 $N$ 个按顺序排列的台阶，需要把它们分进两组分别完成跳跃。每一组按原下标顺序经过这些台阶，并累计相邻两次落点的体力代价，要求两次跳跃的总代价最小。

### 分析

把题意翻成结构，会发现这是“按顺序把每个高度放进两条链里”，而代价只来自同一条链相邻两次被选中的高度差。于是问题不再是枚举分组，而是维护“当前两条链的结尾高度分别是什么，下一步把 $H_i$ 接到哪一条更划算”。

最优策略的本质是尽量让两条链的尾部保持大而有序，这样后面的高台阶接上去时新增代价最小。把高度离散化后，可以把状态写成“较小尾值固定时，较大尾值能做到的最优代价”，再用树状数组维护前缀最优转移。它看着像分组题，真正练的是把两条链的局面压缩成可比较的单调状态。

### 核心代码

```cpp
for (int i = 1; i <= n; i++) {
    int h = rk[i];
    long long best = bit.query(h) + H[i];
    dp[i] = min(dp[i], best);
    bit.update(h, dp[i] - H[i]);
}
// 两条链对称转移，答案取最终最小值
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(n)$。

## 5. [1745：分组](http://ybt.ssoier.cn:8088/problem_show.php?pid=1745)

`树状数组` `离线询问` `二维限制`

### 题意

每个人有地位和年龄。一个小组需要选一个队长，队长在组内地位最高，且所有成员与队长年龄差不超过 $K$。对每次询问 $(x,y)$，要求包含两人的合法小组最大能有多少人；若不存在则输出 $-1$。

### 分析

先抓住队长的角色：一旦队长确定，组员条件就完全变成“年龄落在一个长度为 $2K$ 的窗口里，且地位不超过队长”。所以每个合法小组，其实都对应一个“以某个人为中心的年龄窗口”。

这就把原题拆成两个维度：年龄窗口决定谁有资格进组，地位上界决定谁能被这个队长统领。把所有人按年龄排序后，用双指针维护每个队长能覆盖的最大年龄区间；再按地位离线，把窗口内人数用树状数组做前缀统计。询问 $(x,y)$ 时，只需要在同时覆盖两人的候选队长里，找能给出最大窗口人数的那个即可。

### 核心代码

```cpp
sort(id.begin(), id.end(), byAge);
for (int i = 1, j = 1; i <= n; i++) {
    while (j <= n && age[id[j]] <= age[id[i]] + K) bit.add(rankv[id[j++]], 1);
    while (age[id[i]] - age[id[L]] > K) bit.add(rankv[id[L++]], -1);
    best[i] = bit.query(rankv[id[i]]);
}
// 询问离线到可当队长的人，区间最大值回答
```

### 复杂度

整体时间复杂度 $O((n+q)\log n)$，空间复杂度 $O(n)$。

# 二、RMQ 与静态区间决策

进入这一章后，区间本身不再频繁修改，重点改成了“如何快速拿到一个窗口里的最大值、最小值或最长公共信息”。一旦最值可以 $O(1)$ 或 $O(\log n)$ 地取到，很多原本发散的决策题都会被压成单调扫描或贡献统计。

## 6. [1746：矩阵最值](http://ybt.ssoier.cn:8088/problem_show.php?pid=1746)

`RMQ` `二维稀疏表` `子矩阵最大值`

### 题意

给定一个 $n	imes m$ 的矩阵，询问很多次子矩阵 $[x_1,x_2]	imes[y_1,y_2]$ 的最大值。

### 分析

矩阵没有修改，询问却很多，所以第一反应就该从“现算一块矩形”切到“预处理所有尺度”。二维 RMQ 的核心仍然是稀疏表：先在行和列两个方向分别按 $2^k$ 分块，预处理每个左上角出发的矩形最大值。

真正查询时，把任意一个子矩阵拆成四个能覆盖它的 $2^{k_x}	imes2^{k_y}$ 角块，四块取最大就是答案。模型上最重要的是认出：二维问题没有变复杂，本质还是一维 sparse table 的直积版本。

### 核心代码

```cpp
int kx = lg[x2 - x1 + 1], ky = lg[y2 - y1 + 1];
int a = st[kx][ky][x1][y1];
int b = st[kx][ky][x2 - (1 << kx) + 1][y1];
int c = st[kx][ky][x1][y2 - (1 << ky) + 1];
int d = st[kx][ky][x2 - (1 << kx) + 1][y2 - (1 << ky) + 1];
return max(max(a, b), max(c, d));
```

### 复杂度

预处理时间复杂度 $O(nm\log n\log m)$，单次查询 $O(1)$。

## 7. [1747：奶牛排队](http://ybt.ssoier.cn:8088/problem_show.php?pid=1747)

`RMQ` `单调性` `二分`

### 题意

在一段连续奶牛里，要求最左端最矮、最右端最高，中间奶牛的身高不能与两端重复，求符合条件的最长连续段长度。

### 分析

题目表面是枚举两端，实际上判断一段区间是否合法只和区间最小值、最大值是否恰好落在两端有关。也就是说，只要能快速知道任意区间的最小值和最大值，就能把“段是否合法”变成一个常数时间判断。

于是可先建最小值与最大值两张 sparse table，再枚举左端或者右端，用二分 / 双指针找到最远的可行位置。这里最值得形成的手感是：连续段条件题常常不是去维护中间细节，而是先问“这段的最值能否唯一地落在边界上”。

### 核心代码

```cpp
bool ok(int l, int r) {
    auto mn = queryMin(l, r);
    auto mx = queryMax(l, r);
    return h[l] == mn && h[r] == mx && h[l] < h[r];
}
for (int l = 1; l <= n; l++) {
    int L = l + 1, R = n, pos = l;
    while (L <= R) {
        int mid = (L + R) >> 1;
        if (ok(l, mid)) pos = mid, L = mid + 1;
        else R = mid - 1;
    }
    ans = max(ans, pos - l + 1);
}
```

### 复杂度

预处理 $O(n\log n)$，整体查询枚举后为 $O(n\log n)$。

## 8. [1748：大数计数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1748)

`RMQ` `倍增分解` `并查集`

### 题意

长度为 $n$ 的十进制大数满足若干条件：两段等长子串必须完全相同。要求统计满足全部限制的长度为 $n$ 的数字个数，对 $10^9+7$ 取模。

### 分析

每条限制 $[l_1,r_1]=[l_2,r_2]$ 的本质，是把对应偏移上的每一位都强制并到同一类里。直接逐位连边会炸到 $O(nm)$，所以关键是把“整段相等”压成若干个幂长块：利用倍增分解，把长度拆成若干个 $2^k$ 区间，只在这些代表块之间连边。

等所有区间约束都分解完后，再把约束向下传到单点，并查集合并所有必须相等的位置。最终每个连通块是一种独立取值；唯一额外限制是最高位所在的块不能取 $0$，所以答案就是 $9	imes 10^{cnt-1}$ 或等价的连通块计数公式。题目看起来在比较字符串，其实核心是区间约束下的等价类压缩。

### 核心代码

```cpp
for (auto [l1, r1, l2, r2] : qs) {
    int len = r1 - l1 + 1;
    for (int k = LG; k >= 0; k--) if ((1 << k) <= len) {
        unite(id[k][l1], id[k][l2]);
        l1 += 1 << k; l2 += 1 << k; len -= 1 << k;
    }
}
for (int k = LG; k; k--) for (int i = 1; i + (1 << k) - 1 <= n; i++) {
    int f = find(id[k][i]);
    unite(f, id[k - 1][i]);
    unite(f, id[k - 1][i + (1 << (k - 1))]);
}
```

### 复杂度

时间复杂度 $O((n+m)\log n)$，空间复杂度 $O(n\log n)$。

## 9. [1749：矩阵求和](http://ybt.ssoier.cn:8088/problem_show.php?pid=1749)

`RMQ` `单调栈` `贡献法`

### 题意

矩阵由 $A,B$ 生成，满足 $C_{ij}=A_iB_j+iB_j+A_i j+ij$。定义一个 $k	imes k$ 连续子矩阵的价值为其中最大元素，要求输出所有尺寸 $k$ 的价值总和 $F[k]$。

### 分析

先把式子整理一下：$C_{ij}=(A_i+i)(B_j+j)$。这样每个子矩阵的最大值就不必在二维里找，而是拆成“行窗口最大值”和“列窗口最大值”的乘积。于是 $F[k]$ 可以写成所有长度为 $k$ 的行窗口最大值之和，乘上所有长度为 $k$ 的列窗口最大值之和。

接下来问题完全退化成一维：对每个窗口长度 $k$，求所有长度为 $k$ 的子数组最大值总和。这正是单调栈做贡献统计的经典场景。对每个元素算出它作为窗口最大值时，左边和右边各能延伸多远，再把贡献分配到所有合法长度上即可。真正的建模转身，就是先把二维最值认成两个一维最值的乘法结构。

### 核心代码

```cpp
for (int i = 1; i <= n; i++) x[i] = A[i] + i, y[i] = B[i] + i;
auto sx = calcWindowMaxSum(x), sy = calcWindowMaxSum(y);
for (int k = 1; k <= n; k++)
    ans[k] = 1LL * sx[k] * sy[k] % MOD;
```

### 复杂度

时间复杂度 $O(n)$ 到 $O(n\log n)$（取决于贡献实现），空间复杂度 $O(n)$。

## 10. [1750：取数字](http://ybt.ssoier.cn:8088/problem_show.php?pid=1750)

`RMQ` `区间博弈` `倍增`

### 题意

一圈互不相同的数字，第一步任选一个；之后每次只能从当前空缺段两端相邻的位置拿，且若两边都能拿就必须拿较大的。要求对每一种首步选择，输出先手最后得到的总和。

### 分析

首步一旦固定，之后过程就是完全确定的：剩余未取位置始终是一段连续弧，双方每次都在这段弧的两端里取较大值。因此题目本质是“从一个环切开后，在区间上按两端较大规则推进”。

真正难点是不能对每个起点都线性模拟。常见做法是把环断成两倍数组，用 RMQ 或笛卡尔树快速找出某段当前会先被拿走的端点，再配合倍增预处理若干步后的状态。这样每个起点只需沿着决定链跳若干次，就能累加先手会拿到的那些值。模型上要先看出：它不是自由博弈，而是一个被“必须拿大的”规则锁死的确定过程。

### 核心代码

```cpp
Node jump(Node cur, int t) {
    for (int k = 0; t; k++, t >>= 1)
        if (t & 1) cur = up[k][cur.id];
    return cur;
}
for (int s = 1; s <= n; s++) {
    Node st = buildState(s);
    ans[s] = a[s] + collect(st);
}
```

### 复杂度

预处理 $O(n\log n)$，每个起点计算 $O(\log n)$，总复杂度 $O(n\log n)$。

# 三、线段树与区间结构维护

从这一章开始，修改和查询同时变重，单纯前缀统计已经不够了。这里训练的是另一层意识：先想清结点上到底该保存什么信息、懒标记该表示什么操作，再让线段树去承接这些区间结构。

## 11. [1751：字符串排序](http://ybt.ssoier.cn:8088/problem_show.php?pid=1751)

`线段树` `计数排序` `懒标记`

### 题意

给定一个只含小写字母的字符串，支持把区间 $[l,r]$ 升序或降序排序，最后输出整个字符串。

### 分析

区间排序最容易想歪成真的把字符拿出来排序，但这里字母表只有 $26$ 个，说明维护对象不该是具体顺序，而该是“每段里每个字母出现了多少次”。

于是在线段树每个结点维护 $26$ 个计数。区间排序时，先把目标区间各字母个数统计出来，再按照升序或降序，把这些字母分段覆盖回去。覆盖本身可以用懒标记表示“这一整段全是某个字符”。这题的关键直觉是：当值域很小、操作是重排时，往往该维护频次而不是原数组。

### 核心代码

```cpp
array<int, 26> cnt = query(1, l, r);
clear(1, l, r);
int p = l;
for (int c = 0; c < 26; c++) if (cnt[c]) {
    cover(1, p, p + cnt[c] - 1, c);
    p += cnt[c];
}
if (!asc) reverseWrite(l, r, cnt);
```

### 复杂度

单次操作时间复杂度 $O(26\log n)$，空间复杂度 $O(26n)$。

## 12. [1752：纪念碑](http://ybt.ssoier.cn:8088/problem_show.php?pid=1752)

`线段树` `扫描线` `二分答案`

### 题意

在一个很大的矩形土地上，有很多轴对齐建筑矩形已经被占用。纪念碑必须是一个不和建筑相交的正方形，要求最大边长。

### 分析

直接在平面上找最大空正方形很难，但“边长是否至少为 $L$”是很好判的。若边长固定为 $L$，每个建筑都会禁止一片左下角区域；只要存在一个左下角没有被任何建筑覆盖，长度 $L$ 就可行。

因此外层二分答案，内层做扫描线判定。把每个建筑对左下角的限制转成一个矩形覆盖区间，沿 $x$ 扫描时在线段树上维护当前 $y$ 轴哪些位置已被完全遮住，只要某一时刻还能找到一段长度足够的空白，就说明当前 $L$ 可行。真正要先想到的是把“找最大正方形”改写成“检查哪些起点还能放下它”。

### 核心代码

```cpp
bool check(int L) {
    build(1, 1, Y);
    sort(ev.begin(), ev.end());
    for (auto e : ev) {
        add(1, e.y1, e.y2, e.typ);
        if (tree[1].mn == 0) return true;
    }
    return false;
}
```

### 复杂度

单次判定 $O((p+M)\log M)$，总复杂度 $O(\log V\cdot (p+M)\log M)$。

## 13. [1753：区间连通性](http://ybt.ssoier.cn:8088/problem_show.php?pid=1753)

`线段树` `有向图可达` `动态建边`

### 题意

依次加入区间，且每次新加区间都比以前更长。若一个区间的某个端点落在另一个区间内部，就可以从前者走到后者。多次询问两个编号之间是否存在路径。

### 分析

因为区间长度单调递增，所以新加入的区间总是“更大的一层”，图天然带有从长区间到短区间扩张的层次感。真正需要维护的不是全图传递闭包，而是：一个新区间会和哪些旧区间发生端点包含关系。

这件事可以用两棵按左端点、右端点建的线段树来找。每加入一个区间，就在线段树中查询所有端点落进它内部的旧区间，并为这些区间连边；随后把自己的两个端点再插回结构。查询时再在这张稀疏图上用可达信息或并强连通后的 DAG 跳跃判断。模型上的要点是：长度单调这个条件，就是为了把本来爆炸的相交关系压成可按端点检索的增量连边。

### 核心代码

```cpp
void insert(int id, int l, int r) {
    link(rootL, x[id] + 1, y[id] - 1, id);
    link(rootR, x[id] + 1, y[id] - 1, id);
    addPoint(rootL, x[id], id);
    addPoint(rootR, y[id], id);
}
// Tarjan 缩点后在 DAG 上做可达判断
```

### 复杂度

建图与查询总体复杂度为 $O((n+m)\log n)$ 到 $O((n+m)\log^2 n)$。

## 14. [1754：栈的维护](http://ybt.ssoier.cn:8088/problem_show.php?pid=1754)

`线段树` `函数合并` `序列修改`

### 题意

给出一串对栈的操作：压入一个值，或弹出若干个值。之后有很多次单点修改，每次改掉某一条操作，要求输出整串操作执行完后栈内元素和。

### 分析

单次重跑整条操作序列肯定不行，关键是把一段操作看成“对进入这段时的栈状态施加了什么作用”。如果只关心最终剩下多少元素、这些元素的和是多少，那么一段区间就可以抽象成一个可合并的函数。

线段树维护的就是这种函数合并。左右儿子合并时，相当于把左区间执行后的残留栈，再送进右区间继续执行；因此节点里除了剩余和，还要维护会删掉多少前缀、保留下来哪些尾部元素。题目表面是栈，真正练的是把复杂过程封装成可以在线段树上复合的区间信息。

### 核心代码

```cpp
struct Node {
    int needPop, sz;
    long long sum;
    vector<int> keep;
};
Node merge(Node a, Node b) {
    consume(a, b.needPop);
    a.keep.insert(a.keep.end(), b.keep.begin(), b.keep.end());
    a.sum += b.sum; a.sz = (int)a.keep.size();
    return a;
}
```

### 复杂度

单次修改时间复杂度 $O(\log n\cdot T)$，其中 $T$ 是节点信息合并代价。

## 15. [1755：星际探测](http://ybt.ssoier.cn:8088/problem_show.php?pid=1755)

`线段树` `初等对称和` `区间查询`

### 题意

每个区域有距离 $D$ 和丰富度 $V$。询问距离落在 $[L,R]$ 的区域里，任选 $K$ 块样本时所有研究价值之和，但其中 $V$ 最小的区域永远不会被选；$K\le 6$。

### 分析

“所有乘积之和”是一个非常明显的信号：它不该枚举组合，而该想到初等对称和 $e_k$。如果一段区间的值集合是 $v_1,\dots,v_t$，那么所有选 $K$ 个的乘积和正是 $e_K$。而“最小值永远不会被选”只是在这份多项式里删去一个元素。

因此把点按距离排序后建线段树，每个节点维护最小值 `mn` 和 $e_0\sim e_6$。两个区间合并时做一次小规模卷积；查询到整段后，再用“删去最小值”对应的多项式转移，把答案从整段的对称和改成删最小后的对称和。这里最重要的建模是：组合求和落在多项式系数，$K$ 小则整个结构非常轻。

### 核心代码

```cpp
struct Node { unsigned mn, e[7]; };
Node merge(Node a, Node b) {
    Node c; c.mn = min(a.mn, b.mn);
    c.e[0] = 1;
    for (int i = 1; i <= 6; i++) {
        c.e[i] = 0;
        for (int j = 0; j <= i; j++) c.e[i] += a.e[j] * b.e[i - j];
    }
    return c;
}
// ans = K! * eK(after removing mn)
```

### 复杂度

单次查询时间复杂度 $O(6^2\log n)$，空间复杂度 $O(n)$。

## 16. [1756：过路费](http://ybt.ssoier.cn:8088/problem_show.php?pid=1756)

`LCA` `最小生成树` `瓶颈路`

### 题意

无向连通图有很多带权道路，从城市 $S$ 到 $T$ 的过路费定义为所走路径上最大边权。多次询问最少需要交多少过路费。

### 分析

“最小化路径上的最大边权”是典型的瓶颈路问题。这里最关键的模型不是最短路，而是最小生成树：任意两点在 MST 上那条路径的最大边权，就是原图里所有路径能达到的最小瓶颈值。

所以先建 MST，再把问题彻底变成树上查询。对生成树预处理倍增祖先，同时记录跳 $2^k$ 级时路径上的最大边权；询问两点时像普通 LCA 一样往上跳，顺手把最大边权并起来即可。题目最值定义有点特别，但一旦认出“瓶颈路 = MST 路径最大边”，后面就很直接。

### 核心代码

```cpp
for (int k = 1; k < LG; k++)
    for (int v = 1; v <= n; v++) {
        fa[v][k] = fa[fa[v][k - 1]][k - 1];
        mx[v][k] = max(mx[v][k - 1], mx[fa[v][k - 1]][k - 1]);
    }
int query(int u, int v) {
    int ans = 0;
    lift(u, dep[u] - dep[v], ans);
    lift(v, dep[v] - dep[u], ans);
    return climbTogether(u, v, ans);
}
```

### 复杂度

建树 $O(m\log m)$，单次询问 $O(\log n)$。

## 17. [1757：避难向导](http://ybt.ssoier.cn:8088/problem_show.php?pid=1757)

`LCA` `倍增` `路径上第一个满足条件的点`

### 题意

树上每个城市都有安全系数，安全系数由该点到最远边境城市的距离经过公式计算得到。询问时给出 $(x,y,q)$，要求在 $x$ 到 $y$ 的路径上找一个离 $x$ 最近且安全系数至少为 $q$ 的城市。

### 分析

题目分成两层。第一层是如何求安全系数：树上“到最远叶子的距离”可以由两次树形 DP 或直径端点距离完成。第二层才是询问：在一条路径上，找离起点最近的、值至少为阈值的点。

后一层非常适合倍增。对每个点预处理祖先时，不只存祖先编号，还存这一跳路径上的最大安全系数。查询时先把 $x	o lca$ 和 $lca	o y$ 两段拆开，再在靠近 $x$ 的方向上用二进制跳过所有“最大值仍小于 $q$”的整段，最后停在第一个可行点前一格。真正难的不是写跳表，而是先把“最近满足条件”翻译成“跳过整段不可能，再精确落点”。

### 核心代码

```cpp
int firstUp(int x, int top, int need) {
    for (int k = LG - 1; k >= 0; k--)
        if (dep[fa[x][k]] >= dep[top] && mxs[x][k] < need)
            x = fa[x][k];
    return val[x] >= need ? x : fa[x][0];
}
// 路径拆成两段后，优先在靠近 x 的一段找答案
```

### 复杂度

预处理 $O(n\log n)$，单次询问 $O(\log n)$。

## 18. [1758：连通能力](http://ybt.ssoier.cn:8088/problem_show.php?pid=1758)

`树形 DP` `直径` `子树`

### 题意

给一棵带权树并固定根为 $1$。对每个结点，要求输出以它为根的子树的连通能力：也就是该子树里找一个点，使它到子树内其他点的最远距离最小，这个最小值就是答案。

### 分析

树上“最远距离最小”其实就是子树半径，而半径最稳定的求法是先求直径：一棵树的最优中心一定落在直径上，答案等于直径长度的一半向上取整。

所以这题关键是对每个子树都维护直径。做树形 DP 时，对每个结点收集儿子子树向下的最长链，顺手更新本子树的最优直径；两个儿子的最长链拼起来，可能构成经过当前点的新直径。最后把每个结点子树的直径长度算出来，答案自然是 $\lceil diam/2ceil$。模型上要先认出“连通能力”这个新名词，其实就是熟悉的树半径。

### 核心代码

```cpp
void dfs(int u, int p) {
    down[u] = diam[u] = 0;
    for (auto [v, w] : g[u]) if (v != p) {
        dfs(v, u);
        diam[u] = max(diam[u], diam[v]);
        diam[u] = max(diam[u], down[u] + down[v] + w);
        down[u] = max(down[u], down[v] + w);
    }
    ans[u] = (diam[u] + 1) / 2;
}
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 19. [1759：采访计划](http://ybt.ssoier.cn:8088/problem_show.php?pid=1759)

`倍增` `集合包含` `树上并集`

### 题意

第 $i$ 位长者一定管辖自己，且若 $B_i$ 中每个人都管辖某位长者 $j$，则 $i$ 也管辖 $j$。多次询问给出若干位长者，要求统计至少被其中一位长者直接或间接管辖的球长数。

### 分析

定义 $S_i$ 为第 $i$ 位长者能管辖的编号集合，则题意正是在说 $S_i=\{i\}\cupigcap_{k\in B_i}S_k$。由于编号只依赖更小编号，这批集合会形成非常强的包含层次：每个新集合都像是在若干旧集合的交里再加上自己。

一旦把这种包含关系整理成一棵“集合树”，每个 $S_i$ 就会对应树上的一个子树区间。询问若干长者时，目标变成若干子树并的大小；把这些点按 DFS 序排序，去掉被祖先完全包含的部分，再用倍增求相邻点的 LCA 做并集修正即可。题面在说长者和名单，真正维护的却是一族带包含结构的集合。

### 核心代码

```cpp
sort(qv.begin(), qv.end(), byDfn);
long long res = 0;
for (int x : qv) {
    if (!stk.empty() && isAncestor(stk.back(), x)) continue;
    while (!stk.empty() && !isAncestor(lca(stk.back(), x), stk.back())) stk.pop_back();
    res += sz[x];
    if (!stk.empty()) res -= sz[lca(stk.back(), x)];
    stk.push_back(x);
}
```

### 复杂度

预处理约为 $O((n+\sum |B_i|)\log n)$，单次询问 $O(k\log n)$。

## 20. [1760：树上数颜色](http://ybt.ssoier.cn:8088/problem_show.php?pid=1760)

`子树查询` `颜色维度线段树` `动态修改`

### 题意

树上每个点有颜色，支持修改某点颜色；询问结点 $u$ 的子树中，颜色值落在 $[l,r]$ 的颜色里，有多少种至少出现过一次。

### 分析

先把子树变成 Euler 序上的连续段 $[tin_u,tout_u]$。这样问题变成：一个数组支持单点改值，多次询问某段里有多少不同值落在颜色区间 $[l,r]$。

难点是同时有“位置维”和“颜色维”。常见做法是在线段树上按颜色建树，每个颜色区间节点维护一个有序集合，里面放所有属于该颜色范围的点的 Euler 位置。查询时在颜色树上递归，只要某个节点的集合里存在一个位置落进 $[tin_u,tout_u]$，这一整段颜色范围就有贡献，再继续细分；修改则是在两条颜色链上删旧点、插新点。模型的关键，是先承认这是一个二维存在性查询，再决定用一维树套另一维树来承载。

### 核心代码

```cpp
void modify(int c, int pos, int op, int p, int l, int r) {
    box[p].erase_or_insert(pos, op);
    if (l == r) return;
    int mid = (l + r) >> 1;
    if (c <= mid) modify(c, pos, op, p << 1, l, mid);
    else modify(c, pos, op, p << 1 | 1, mid + 1, r);
}
int query(int ql, int qr, int L, int R, int p, int l, int r) {
    if (!box[p].hasIn(L, R)) return 0;
    if (ql <= l && r <= qr && l == r) return 1;
    int mid = (l + r) >> 1;
    return (ql <= mid ? query(ql, qr, L, R, p << 1, l, mid) : 0)
         + (qr > mid ? query(ql, qr, L, R, p << 1 | 1, mid + 1, r) : 0);
}
```

### 复杂度

单次修改、查询通常为 $O(\log^2 n)$。

## 21. [1761：最小割](http://ybt.ssoier.cn:8088/problem_show.php?pid=1761)

`树链剖分` `树上差分` `最小割`

### 题意

给出一个无向图和它的一棵生成树 $T$。要求找一个最小割，使这个割里恰好只有一条来自生成树 $T$ 的边，并输出这个最小割包含的边数。

### 分析

若删掉树边 $e$，生成树会被分成两块。为了让整张图不连通，所有跨过这两块的非树边也都必须一起删掉，所以以 $e$ 为唯一树边的割大小，就是 $1+$“经过 $e$ 的非树边条数”。

于是问题被完全翻成：对每条非树边 $(u,v)$，给树上路径 $u\sim v$ 的每条树边加一；最后对每条树边看它被多少非树边覆盖，答案是最小的 `cover+1`。这一步可以用树上差分或树链剖分路径加实现。题面写的是最小割，但真正考的是把割边条件翻译成“哪些树边被返祖边跨过去了”。

### 核心代码

```cpp
for (auto [u, v] : extra) {
    diff[u]++; diff[v]++; diff[lca(u, v)] -= 2;
}
calc(1, 0);
int ans = INF;
for (int v = 2; v <= n; v++) ans = min(ans, sum[v] + 1);
```

### 复杂度

时间复杂度 $O((n+m)\log n)$，空间复杂度 $O(n)$。

## 22. [1762：与非](http://ybt.ssoier.cn:8088/problem_show.php?pid=1762)

`树链剖分` `线段树` `函数复合`

### 题意

树上每个点有一个 $k$ 位二进制权值，支持单点修改。路径查询时，从 $0$ 开始按路径顺序依次对每个点做 `nand`，求最后结果。

### 分析

`nand` 最麻烦的地方在于它不交换，路径顺序不能乱。但换个角度看，对某个固定点权 $w$，操作 `x -> not(x & w)` 本身就是一个函数。路径答案就是这些函数沿路径顺序的连续复合，再作用到初始值 $0$ 上。

于是在线段树节点里，不该存一个数，而该存“这段链从左到右复合后的函数”和“从右到左复合后的函数”。树链剖分把一条路径拆成若干重链段后，往上走的段用反向函数，往下走的段用正向函数，再把这些函数按顺序拼起来即可。建模关键就在于先把位运算看成可复合的状态变换。

### 核心代码

```cpp
struct F { unsigned f0, f1; };
F compose(F a, F b) {
    return {((a.f0 & b.f1) ^ b.f1), ((a.f1 & b.f1) ^ b.f1)};
}
Node merge(Node L, Node R) {
    return {compose(L.pre, R.pre), compose(R.suf, L.suf)};
}
// 路径查询时按链顺序复合，最后取 res.f0
```

### 复杂度

单次修改、查询时间复杂度 $O(\log^2 n)$。

## 23. [1763：简单树](http://ybt.ssoier.cn:8088/problem_show.php?pid=1763)

`树链剖分` `分治结构` `距离和`

### 题意

树上点按编号 $1\sim n$ 排列。每次给出 $L,R,x$，要求编号在 $[L,R]$ 内的所有点到点 $x$ 的距离和。

### 分析

这里的难点是编号区间和树距离完全是两套坐标。若只按树结构做，无法快速筛掉编号不在 $[L,R]$ 的点；若只看编号，又算不出距离。所以必须建立一层“按编号分治、每层记录到若干关键点距离”的混合结构。

常见做法是按编号建线段树，每个节点存这段编号里的所有点到若干重心 / 链顶的距离和。查询 $[L,R]$ 时只访问 $O(\log n)$ 个编号节点，把它们对点 $x$ 的距离贡献加起来即可。题目的关键意识是：限制在编号轴上，代价在树上时，就该做一个编号维的数据结构，再把树距离预处理成节点信息。

### 核心代码

```cpp
void addPoint(int p, int x) {
    for (int u = x; u; u = faCent[u]) {
        sum[p][u] += dist(x, u);
        cnt[p][u]++;
    }
}
long long querySeg(int p, int x) {
    long long res = 0;
    for (int u = x; u; u = faCent[u])
        res += sum[p][u] + 1LL * cnt[p][u] * dist(x, u);
    return res;
}
```

### 复杂度

预处理约为 $O(n\log^2 n)$，单次询问 $O(\log^2 n)$。

## 24. [1764：社会送温暖](http://ybt.ssoier.cn:8088/problem_show.php?pid=1764)

`树链剖分` `线段树` `复杂懒标记`

### 题意

树上点有金币数，支持对一条路径按经过顺序做四种操作：等差加、等比加、等差赋值、等比赋值，以及路径和查询。

### 分析

题目真正难的不是树，而是“路径有方向”。同一条链段若从上往下走和从下往上走，首项会不一样，所以懒标记不能只存一个普通区间加法，而要把“从左到右的数列”本身存下来。

树链剖分先把路径拆成若干链段，再统一成若干个有方向的区间操作。线段树节点需要支持两类序列：等差与等比；同时还要区分覆盖和叠加。只要能把一段首项、公差 / 公比换算到链段局部坐标上，剩下就是标准的懒标记下传。它的核心训练点是：当路径操作带顺序时，先把顺序在链上固定下来，再谈区间维护。

### 核心代码

```cpp
void pathUpdate(int u, int v, Tag tg) {
    while (top[u] != top[v]) {
        if (dep[top[u]] < dep[top[v]]) swap(u, v), tg.flip();
        apply(1, dfn[top[u]], dfn[u], tg.cut(len[top[u]], len[u]));
        u = fa[top[u]];
    }
    if (dep[u] > dep[v]) swap(u, v), tg.flip();
    apply(1, dfn[u], dfn[v], tg.cut(0, dep[v] - dep[u]));
}
```

### 复杂度

单次操作时间复杂度 $O(\log^2 n)$。

## 25. [1765：树上斐波那契](http://ybt.ssoier.cn:8088/problem_show.php?pid=1765)

`树链剖分` `斐波那契基底` `子树更新`

### 题意

根树上初始点权全为 $0$。操作 `U X k`：对 $X$ 的子树内每个点，若它到 $X$ 的距离为 $D$，则加上 $Fib(k+D)$；操作 `Q X Y`：询问路径 $X\sim Y$ 上点权和。

### 分析

真正该先想到的是把 $Fib(k+D)$ 拆到一个固定基底上。因为斐波那契满足线性递推，所以任意一项都能写成 $A\cdot Fib(dep)+B\cdot Fib(dep+1)$ 的形式；一旦把更新拆成这两个基函数的线性组合，子树更新就变成 Euler 序上一段区间加系数。

接着再做树链剖分处理路径和。在线段树节点里分别维护当前区间上 $Fib(dep)$ 与 $Fib(dep+1)$ 的加权和，子树更新只是在整段上打两个系数标记；路径查询时像普通树链剖分一样拆链求和。题面看着很像“树上套斐波那契”，但真正的模型是把递推数列基底化，再让树结构承接线性组合。

### 核心代码

```cpp
pair<long long,long long> split(long long k, int depX) {
    long long a = Fib(k - depX + 1), b = Fib(k - depX + 2);
    return {a, b};
}
void updateSubtree(int x, long long k) {
    auto [a, b] = split(k, dep[x]);
    add(1, dfn[x], out[x], a, b);
}
long long queryPath(int u, int v) {
    long long res = 0;
    while (top[u] != top[v]) {
        if (dep[top[u]] < dep[top[v]]) swap(u, v);
        res += ask(1, dfn[top[u]], dfn[u]);
        u = fa[top[u]];
    }
    if (dep[u] > dep[v]) swap(u, v);
    return res + ask(1, dfn[u], dfn[v]);
}
```

### 复杂度

单次更新、查询时间复杂度 $O(\log^2 n)$，空间复杂度 $O(n)$。
