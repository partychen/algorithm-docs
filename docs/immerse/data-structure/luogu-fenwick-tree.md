# 洛谷 树状数组专题精选解题报告

这一组题从最基础的前缀和维护一路走到二维偏序与离线统计，核心始终是把复杂关系改写成“若干次前缀加、前缀查”的组合。树状数组本身很轻，但一旦坐标、时间或值域能被重新排序，它就能接住很多原本看似不该由它解决的问题。

# 一、树状数组基础模板

这一章先把单点、区间和逆序类问题打牢，后面的离线和偏序都建立在同一套前缀维护思维上。

## 1. [P3374 【模板】树状数组 1](https://www.luogu.com.cn/problem/P3374)
`树状数组` `单点修改` `区间查询`

### 题意

维护一个数组，支持单点加和区间求和两种操作。

### 分析

树状数组最适合解决的，不是任意区间操作，而是“前缀和相关”的问题。因为它把数组拆成若干个长度为 `lowbit(x)` 的块，每个位置 `x` 负责维护一个固定区间的和。

所以单点修改时，要把所有覆盖到这个位置的块都更新到；这就是为什么沿着 `x += lowbit(x)` 往上跳。反过来，查询前缀和时，就不断拿当前块的贡献并跳到它前面一个块去，也就是 `x -= lowbit(x)`。

因此这题真正该学的是树状数组的维护对象：**它维护的不是原数组本身，而是一组能恰好拼出任意前缀的块和。**

### 核心代码

```cpp
const int N = 500010;
int n, tr[N];

void add(int x, int v) {
    for (; x <= n; x += x & -x) tr[x] += v;
}

int sum(int x) {
    int s = 0;
    for (; x; x -= x & -x) s += tr[x];
    return s;
}
```

### 复杂度

单次修改和查询都是 `O(log n)`。

## 2. [P3368 【模板】树状数组 2](https://www.luogu.com.cn/problem/P3368)
`树状数组` `差分` `区间修改`

### 题意

支持区间加法，并查询某个位置的当前值。

### 分析

这题表面是“区间修改、单点查询”，但树状数组本身更擅长“单点修改、前缀查询”，所以关键是先换一个表达方式。

把原数组做成差分数组后，区间 `[l,r]` 加上 `v`，就只会让差分数组的两个位置发生变化：`d[l] += v`，`d[r+1] -= v`。而某个位置 `x` 的真实值，正好等于差分数组前缀和 `d[1..x]`。

于是题目就被完全翻译成了树状数组擅长的形式：端点单点修改 + 前缀和查询。所以这题最值得记住的是一个套路：**当操作长在区间上而查询落在单点上时，优先看看差分能不能把它翻译成端点问题。**

### 核心代码

```cpp
const int N = 500010;
int n, tr[N];

void add(int x, int v) {
    for (; x <= n; x += x & -x) tr[x] += v;
}

void range_add(int l, int r, int v) {
    add(l, v);
    add(r + 1, -v);
}

int ask(int x) {
    int s = 0;
    for (; x; x -= x & -x) s += tr[x];
    return s;
}
```

### 复杂度

单次操作都是 `O(log n)`。

## 3. [P1908 逆序对](https://www.luogu.com.cn/problem/P1908)
`树状数组` `逆序对` `离散化`

### 题意

给定一个序列，求满足 `i < j` 且 `a[i] > a[j]` 的数对数量。

### 分析

树状数组做逆序对，核心不是“会统计频次”，而是先把题目改写成前缀计数。对当前位置 `a[i]` 来说，它产生多少逆序对，只取决于前面已经出现了多少个比它大的数。

如果值域很大或有负数，就先离散化，把“比它大”变成排名意义上的区间。然后从左到右扫描，树状数组维护前面各个值已经出现了多少次；当前答案就是“已出现总数 - 不大于它的个数”。

所以这题最该建立的是一种树状数组视角：**当你需要动态维护某个值域上的频次前缀，并据此统计“大于 / 小于当前值的个数”时，它往往就是树状数组题。**

### 核心代码

```cpp
const int N = 500010;
int n, tr[N];
long long ans;

void add(int x) {
    for (; x <= n; x += x & -x) tr[x]++;
}

int sum(int x) {
    int s = 0;
    for (; x; x -= x & -x) s += tr[x];
    return s;
}
```

### 复杂度

时间复杂度 `O(n log n)`，空间复杂度 `O(n)`。

## 4. [P1966 [NOIP 2013 提高组] 火柴排队](https://www.luogu.com.cn/problem/P1966)
`树状数组` `逆序对` `排序映射`

### 题意

两排火柴长度分别给定，允许交换相邻位置，求把两排火柴按对应关系排齐的最少交换次数。

### 分析

这题本身不是树状数组题，而是“最少相邻交换”题。真正关键的一步，是先把两排火柴的最优对应关系固定下来。

分别按长度排序后，第 `i` 根对应第 `i` 根，这样总代价最优；再把第一排中的每根火柴映射成它在第二排里的目标位置，原题就变成了：把一个排列通过相邻交换排到有序，最少需要多少次。

而这个答案正是逆序对数量，所以树状数组在这里只承担“动态统计排列前缀里已有多少元素”的工作。也就是说，这题最值得记的是：**树状数组常常是某个更大转化链条的最后一步，而不是第一眼就出现。**

### 核心代码

```cpp
const int N = 200010;
int n, tr[N], rk[N];
long long ans;

void add(int x) {
    for (; x <= n; x += x & -x) tr[x]++;
}

int sum(int x) {
    int s = 0;
    for (; x; x -= x & -x) s += tr[x];
    return s;
}
```

### 复杂度

时间复杂度 `O(n log n)`。

## 5. [P1774 最接近神的人](https://www.luogu.com.cn/problem/P1774)
`树状数组` `逆序对` `稳定排序`

### 题意

给定一个序列，求把它变成非降序时所需的最少相邻交换次数。

### 分析

这题和普通逆序对只有一个关键差别：序列里可能有相等元素。如果直接按值映射位置，相等元素之间的对应关系一乱，就会平白制造出不存在的交换。

所以正确翻译方式不是“把值排序后随便对应”，而是做稳定映射：按值排序时再按原下标排序，让相等元素保持原有先后次序。这样得到的目标排列才真的对应“变成非降序后，每个元素应该去哪里”。

一旦映射完成，后面就回到经典模型：最少相邻交换次数等于排列逆序对数，树状数组负责动态统计前面已有多少目标位置已经出现。也就是说，这题真正教的是：**遇到重复元素时，先想清楚目标排列是否需要稳定化。**

### 核心代码

```cpp
const int N = 500010;
int n, tr[N];
long long ans;

void add(int x) {
    for (; x <= n; x += x & -x) tr[x]++;
}

int sum(int x) {
    int s = 0;
    for (; x; x -= x & -x) s += tr[x];
    return s;
}
```

### 复杂度

时间复杂度 `O(n log n)`。

# 二、区间维护与二维树状数组

这一章从一维前缀扩展到二维矩阵，核心还是把“区间贡献”拆成若干个前缀贡献。

## 6. [P4514 上帝造题的七分钟](https://www.luogu.com.cn/problem/P4514)
`二维树状数组` `矩形加` `矩形求和`

### 题意

维护一个二维矩阵，支持矩形加法和矩形求和。

### 分析

一维里“区间加 + 区间求和”已经需要把差分和前缀关系拆开，到了二维同样如此，只是容斥项更多。矩形加法本质上是二维差分，而矩形求和本质上是二维前缀和。

如果把这些式子完全展开，会发现一次矩形修改最终会影响常数项、`x` 项、`y` 项和 `x*y` 项四部分，所以需要 4 棵二维树状数组分别维护，查询时再按二维容斥组合回来。

所以这题最值得举一反三的地方不是“记住 4 棵树状数组”，而是知道它们为什么出现：**因为二维区间操作在前缀展开后会自然拆成四类系数项。**

### 核心代码

```cpp
const int N = 2050;
long long t1[N][N], t2[N][N], t3[N][N], t4[N][N];
int n, m;

void add(long long t[][N], int x, int y, long long v) {
    for (int i = x; i <= n; i += i & -i)
        for (int j = y; j <= m; j += j & -j)
            t[i][j] += v;
}
```

### 复杂度

单次修改和查询都是 `O(log n log m)`。

## 7. [P4054 [JSOI2009] 计数问题](https://www.luogu.com.cn/problem/P4054)
`二维树状数组` `颜色计数` `离散值域`

### 题意

给定一个二维矩阵，支持单点修改，并查询某个子矩形里某个值出现了多少次。

### 分析

这题难点不在二维，而在“既要按位置查，又要按颜色筛”。如果值域很大，颜色这一维很难直接并进状态；但题目颜色范围很小，这就给了最直接的做法：每种颜色各维护一棵二维树状数组。

这样一来，位置维度仍然通过二维前缀和处理，而颜色筛选则退化成“你查哪种颜色，就去对应那棵树里问一次矩形和”。修改时也只需在旧颜色的树里删掉，再在新颜色的树里加上。

所以这题最值得记住的是一个工程化思路：**当某一维值域足够小，可以直接按这一维拆多棵数据结构，换来更简单清晰的主逻辑。**

### 核心代码

```cpp
const int V = 105;
int n, m;
int bit[V][305][305];

void add(int c, int x, int y, int v) {
    for (int i = x; i <= n; i += i & -i)
        for (int j = y; j <= m; j += j & -j)
            bit[c][i][j] += v;
}
```

### 复杂度

单次修改和查询都是 `O(log n log m)`，额外乘一个小值域常数。

## 8. [P2161 [SHOI2009] 会场预约](https://www.luogu.com.cn/problem/P2161)
`树状数组` `区间占用` `离散化`

### 题意

按时间顺序处理会场预约，判断某段时间是否还能被安排，或者当前时刻的占用情况。

### 分析

时间区间题一看到“预约一段区间、查询某时刻或某段是否冲突”，就该先想时间轴差分。因为一段预约对中间每个时刻的影响都相同，真正发生变化的只是在左右端点。

把时间离散化后，一次预约 `[l,r]` 可以翻译成差分数组上的两次端点修改；再做前缀和，就能知道每个时刻当前被覆盖了多少次。于是“是否还能安排”“当前占用几次”这类问题，都变成了查询某个前缀或区间上的覆盖信息。

所以这题不是单纯套树状数组模板，而是在训练一个改写习惯：**当操作长在时间段上时，先问自己能不能只在端点记录变化。**

### 核心代码

```cpp
const int N = 200010;
int n, tr[N];

void add(int x, int v) {
    for (; x <= n; x += x & -x) tr[x] += v;
}

int ask(int x) {
    int s = 0;
    for (; x; x -= x & -x) s += tr[x];
    return s;
}
```

### 复杂度

单次操作通常是 `O(log n)`。

## 9. [P4113 [HEOI2012] 采花](https://www.luogu.com.cn/problem/P4113)
`树状数组` `离线查询` `前两次出现`

### 题意

给定花的颜色序列，查询区间内有多少种颜色至少出现两次。

### 分析

扫到某个颜色的新出现位置时，只需要维护它“倒数第二次出现”的位置。若某颜色在当前位置前已经出现两次，就把贡献从旧位置挪到新位置。这样每个颜色在树状数组里只保留一个有效贡献点，查询时直接做区间和即可。

### 核心代码

```cpp
const int N = 2000010;
int last[N], prelast[N];
long long tr[N];

void add(int x, int v) {
    for (; x < N; x += x & -x) tr[x] += v;
}

long long sum(int x) {
    long long s = 0;
    for (; x; x -= x & -x) s += tr[x];
    return s;
}
```

### 复杂度

时间复杂度 `O((n+m) log n)`。

动态修改若干数值，并询问把每个数截断到 `s` 之后的总和是否达到要求。

```cpp
const int N = 2000010;

void add(long long tr[], int x, long long v) {

}

long long sum(long long tr[], int x) {
    long long s = 0;
    for (; x; x -= x & -x) s += tr[x];
    return s;
}
```

### 复杂度

单次修改和查询都是 `O(log n)`。

# 三、离线统计与答案分治

这一章把答案放到值域或时间轴上整体二分，再借助树状数组批量判定。

## 11. [P3527 [POI 2011] MET-Meteors](https://www.luogu.com.cn/problem/P3527)
`整体二分` `树状数组` `区间加单点查`

### 题意

每次流星雨会给一段环形区间增加贡献。每个国家都有若干扇区，问最早经过多少次流星雨后，各国累计贡献能达到目标。

### 分析

这题难点不在树状数组，而在“每个国家的答案不同，却都在问最早经过多少次流星雨”。这种“很多对象都在找最小可行前缀”的结构，最自然就是整体二分。

把答案猜成前 `mid` 次流星雨后，判定函数实际在问：每个国家控制的那些扇区，总累计贡献是否已经达到目标。流星雨对扇区是区间加，国家验收时是若干点求和，所以刚好能用“区间加、单点查”的树状数组来批量完成一次判定。

因此这题最该形成的认识是：**整体二分并不是把每个国家各做一次二分，而是把同一轮 `mid` 的判定贡献共享掉。**树状数组只是用来高效承接这轮共享判定。

### 核心代码

```cpp
const int N = 300010;
long long bit[N];

void add(int x, long long v) {
    for (; x < N; x += x & -x) bit[x] += v;
}

long long sum(int x) {
    long long s = 0;
    for (; x; x -= x & -x) s += bit[x];
    return s;
}
```

### 复杂度

整体二分配合树状数组，通常是 `O((n+m) log^2 m)`。

## 12. [P3332 [ZJOI2013] K 大数查询](https://www.luogu.com.cn/problem/P3332)
`整体二分` `树状数组` `区间加`

### 题意

维护若干位置上的数值插入，支持区间插入和区间第 `k` 大查询。

### 分析

区间第 `k` 大通常不好直接在线维护，但如果只问答案值是多少，就可以把“第 `k` 大”翻译成一个值域判定：区间里有多少数 `>= mid`。

这样答案就能在值域上整体二分。每轮固定 `mid` 后，把所有值至少为 `mid` 的修改视作有效贡献，批量加入树状数组；对一个查询来说，只要统计当前区间内有效元素个数，就知道第 `k` 大是在左半边还是右半边。

所以这题真正要学会的是判定函数的落点：**第 `k` 大不是直接维护顺序，而是转成“至少有多少个数不小于某阈值”。**一旦判定对象变成计数，树状数组就接得住了。

### 核心代码

```cpp
const int N = 50010;
int bit[N];

void add(int x, int v) {
    for (; x < N; x += x & -x) bit[x] += v;
}

int sum(int x) {
    int s = 0;
    for (; x; x -= x & -x) s += bit[x];
    return s;
}
```

### 复杂度

时间复杂度通常是 `O(m log^2 V)`。

## 13. [P3605 [USACO17JAN] Promotion Counting P](https://www.luogu.com.cn/problem/P3605)
`树状数组` `欧拉序` `子树统计`

### 题意

给定一棵树和每个点的权值，求每个点子树中比它权值更大的点有多少个。

### 分析

这题表面是树上统计，但真正关心的是：每个点的子树里，有多少节点的等级比它高。对子树这类范围，最自然的第一步就是 DFS 序展开，把每棵子树压成序列上的一个连续区间。

一旦子树变成区间，问题就退化成离线值域统计：按等级从大到小处理节点，把已经出现的高等级节点在它们的 DFS 位置上打 `1`。当处理到点 `u` 时，它子树区间里的前缀和，就是已经出现的、等级比它高的节点数。

所以这题把树状数组真正的长处展现得很典型：**树上的子树范围先压成 DFS 序区间，再用值域排序把“比当前大/小”的条件变成一次离线扫描。**

### 核心代码

```cpp
const int N = 100010;
int tin[N], tout[N], ord[N], idx;
long long bit[N];

void add(int x, int v) {
    for (; x <= idx; x += x & -x) bit[x] += v;
}

long long sum(int x) {
    long long s = 0;
    for (; x; x -= x & -x) s += bit[x];
    return s;
}
```

### 复杂度

时间复杂度 `O(n log n)`。

## 14. [P5677 [GZOI2017] 配对统计](https://www.luogu.com.cn/problem/P5677)
`树状数组` `离线排序` `区间计数`

### 题意

先预处理出所有“配对”关系，再回答很多区间询问：区间里有多少对端点都落在其中。

### 分析

这类“很多区间询问，问有多少对完整落在区间里”的题，最关键的是给每一对找到一个统一的计数时刻。若把配对按右端点排序，那么当扫描线走到位置 `r` 时，所有右端点不超过 `r` 的配对都已经可以被考虑。

接下来树状数组只需要记录这些已激活配对的左端点。这样对于询问 `[l,r]`，扫描到 `r` 时，答案就是“左端点也不少于 `l` 的配对个数”，也就是一次区间计数。

所以这题的核心不是树状数组本身，而是离线顺序的选择：**把一对元素中较难控制的那个端点交给扫描线，另一个端点交给树状数组做前缀统计。**

### 核心代码

```cpp
const int N = 200010;
int bit[N];

void add(int x, int v) {
    for (; x < N; x += x & -x) bit[x] += v;
}

int sum(int x) {
    int s = 0;
    for (; x; x -= x & -x) s += bit[x];
    return s;
}
```

### 复杂度

时间复杂度 `O((n+m) log n)`。

# 四、高维偏序与综合应用

这一章把树状数组放到更高维的分治框架里，用来处理偏序、矩阵和异或类问题。

## 15. [P1527 [国家集训队] 矩阵乘法](https://www.luogu.com.cn/problem/P1527)
`整体二分` `二维树状数组` `子矩形第 k 小`

### 题意

给定一个矩阵，很多询问要求子矩形中的第 `k` 小数。

### 分析

把矩阵元素和询问一起离线整体二分。判定某个值 `mid` 时，把所有 `<= mid` 的格子加入二维树状数组，再统计每个询问子矩形里有多少个数满足条件。若数量不足 `k`，答案就往右半边走。

### 核心代码

```cpp
const int N = 505;
int bit[N][N];

void add(int x, int y, int v) {
    for (int i = x; i < N; i += i & -i)
        for (int j = y; j < N; j += j & -j)
            bit[i][j] += v;
}

int sum(int x, int y) {
    int s = 0;
    for (int i = x; i; i -= i & -i)
        for (int j = y; j; j -= j & -j)
            s += bit[i][j];
    return s;
}
```

### 复杂度

时间复杂度通常是 `O(n^2 log^2 n)` 级别的离线判定。

## 16. [P3810 【模板】三维偏序 / 陌上花开](https://www.luogu.com.cn/problem/P3810)
`CDQ 分治` `树状数组` `三维偏序`

### 题意

给定三维点，统计每个点有多少个点在它的三维偏序范围内，然后按答案分组输出。

### 分析

三维偏序难在三个条件不能同时在线处理，但它其实很适合分层拆解。先按第一维排序后，CDQ 分治负责保证“只统计左半边对右半边的贡献”，也就自动满足了第一维不超过。

进入某一层合并时，再按第二维从小到大归并；这样当右侧点准备计算答案时，所有第二维不超过它的左侧点都已经可以加入树状数组。最后第三维不超过多少，就只剩一次前缀和查询。

所以整套结构的理解重点是：**CDQ 不是在神秘优化，它只是把三维条件拆成了‘第一维靠分治、第二维靠归并顺序、第三维靠树状数组前缀’。**

### 核心代码

```cpp
struct Node {
    int a, b, c, cnt, ans;
} p[N], t[N];
int bit[N];

void add(int x, int v) {
    for (; x < N; x += x & -x) bit[x] += v;
}

int sum(int x) {
    int s = 0;
    for (; x; x -= x & -x) s += bit[x];
    return s;
}
```

### 复杂度

时间复杂度 `O(n log^2 n)`。

## 17. [P2487 [SDOI2011] 拦截导弹](https://www.luogu.com.cn/problem/P2487)
`CDQ 分治` `DP` `序列偏序`

### 题意

导弹有高度和速度两个属性，要求求出最优拦截链的长度，并统计每个导弹出现在最优方案中的概率。

### 分析

这题先别被“概率”吓到，第一步仍然是最长链。每枚导弹有两个属性，能否接在前面某枚后面，本质上是二维偏序上的 DP 转移，所以先要解决“如何高效找所有合法前驱中的最优值”。

这里和三维偏序模板很像：按一维排序，另一维在 CDQ 过程中归并，树状数组里维护当前能提供的最长链长度与方案数。这样就能求出每个点作为结尾时的最优长度和条数；再反向做一遍，就知道它作为起点往后还能接出多长。

当一个点满足“前向最长链长度 + 反向最长链长度 - 1 = 全局最优长度”时，它才可能出现在某条最优方案中；再把前后方案数相乘除以总方案数，就是所求概率。所以这题最值得迁移的是：**先把复杂附加量建立在主 DP 结构上，而不是一开始就直接算概率。**

### 核心代码

```cpp
struct Node {
    int h, v, id;
} p[N];
int f[N], g[N], cnt1[N], cnt2[N];
int bit[N];

void add(int x, int v) {
    for (; x < N; x += x & -x) bit[x] = max(bit[x], v);
}
```

### 复杂度

时间复杂度一般是 `O(n log^2 n)`。

## 18. [CF341D Iahub and Xors](https://www.luogu.com.cn/problem/CF341D)
`二维树状数组` `异或` `矩形修改`

### 题意

维护一个二维矩阵，支持矩形异或修改和矩形异或查询。

### 分析

异或版二维区间题和加法版最大的不同，在于异或没有普通乘法展开，但它有一个更好用的性质：同一个贡献出现偶数次会抵消。所以矩形修改仍然可以转成若干个前缀角点的影响。

真正麻烦的是，当你把二维前缀异或展开后，某个贡献会不会被算进答案，取决于查询点横纵坐标的奇偶性。于是不能只维护一棵二维树状数组，而要按 `(xmod2,ymod2)` 分成四类分别记录。

所以这题不要死记“四棵树状数组”这个结论，更该理解它的来源：**前缀展开后，贡献是否保留只和坐标奇偶有关，因此状态自然按奇偶分类。**

### 核心代码

```cpp
const int N = 1010;
int bit[2][2][N][N];

void add(int x, int y, int v) {
    for (int i = x; i < N; i += i & -i)
        for (int j = y; j < N; j += j & -j)
            bit[x & 1][y & 1][i][j] ^= v;
}

int sum(int x, int y) {
    int s = 0;
    for (int i = x; i; i -= i & -i)
        for (int j = y; j; j -= j & -j)
            s ^= bit[x & 1][y & 1][i][j];
    return s;
}
```

### 复杂度

单次操作和查询都是 `O(log n log m)`。
