# 洛谷 并查集专题精选解题报告

这一组题从最基本的连通性判定一路走到带权关系、离线删点和生成树建模，主线非常鲜明：当题目只关心“是否属于同一类”以及“类与类之间怎样合并”时，并查集往往就是最稳的底层结构。难点通常不在合并本身，而在把约束翻译成可合并的关系。

# 一、离线删点与约束建模

先从删点倒序、集合打包和矛盾判定这些最典型的并查集建模入手。

## 1. [P1197 [JSOI2008] 星球大战](https://www.luogu.com.cn/problem/P1197)

`逆序操作` `离线处理` `并查集`

### 题意

按顺序摧毁若干点，并在每次摧毁后询问连通块数量。

### 分析

并查集不支持删点，所以反过来做：先把所有会被摧毁的点都删掉，只保留最后还在图中的点；再按逆序一个个加回来。每次加点时把它和邻居合并，连通块数也随之更新，这样就把删点问题变成了加点问题。

### 核心代码

```cpp
for (int i = 1; i <= n; i++) if (!dead[i]) init_add(i);
for (int i = q; i >= 1; i--) {
    ans[i] = cnt;
    int x = op[i];
    alive[x] = 1;
    cnt++;
    for (auto y : g[x]) if (alive[y] && unite(x, y)) cnt--;
}
```

### 复杂度

时间复杂度 $O((n+m+q)\alpha(n))$。

## 2. [P1455 搭配购买](https://www.luogu.com.cn/problem/P1455)

`并查集` `缩点` `01背包`

### 题意

若干物品之间存在必须一起购买的关系，给定预算，求最大价值。

### 分析

先用并查集把必须一起买的物品缩成一组，再把每组的价格和价值分别累加，最后把每个连通块当成一个物品做 01 背包。并查集负责把约束关系变成若干独立组，背包负责在预算内选最优组合。

### 核心代码

```cpp
for (int i = 1; i <= m; i++) unite(a[i], b[i]);
for (int i = 1; i <= n; i++) {
    int r = find(i);
    cost[r] += c[i];
    val[r] += w[i];
}
// 对所有根节点做 01 背包
```

### 复杂度

时间复杂度 $O(nV)$，其中背包部分是主导。

## 3. [P1955 [NOI2015] 程序自动分析](https://www.luogu.com.cn/problem/P1955)

`并查集` `离散化` `先合并再检验`

### 题意

给出若干等式和不等式约束，判断是否存在一组赋值同时满足所有约束。

### 分析

先把所有变量离散化，再先处理所有等式，把相等的变量合并到一起，最后逐条检查不等式是否和当前集合冲突。顺序不能反过来，否则会漏判。

### 核心代码

```cpp
for (auto &q : eq) unite(id(q.x), id(q.y));
for (auto &q : ne) {
    if (find(id(q.x)) == find(id(q.y))) {
        cout << "NO";
        return;
    }
}
cout << "YES";
```

### 复杂度

时间复杂度 $O(n\alpha(n))$。

## 4. [P5937 [CEOI 1999] Parity Game](https://www.luogu.com.cn/problem/P5937)

`扩展域并查集` `离散化` `奇偶性`

### 题意

给定若干区间奇偶性条件，判断从第几条开始出现矛盾。

### 分析

把区间和转成前缀和差，`[l, r]` 的奇偶性等价于 `s[l-1]` 和 `s[r]` 的奇偶关系。因为端点很大，先离散化，再用两倍域并查集维护“同奇偶”和“异奇偶”关系；一旦发现冲突，就输出当前之前的最后一条。

### 核心代码

```cpp
int x = get(l - 1), y = get(r);
if (odd) {
    if (find(x) == find(y)) break;
    unite(x, y + tot);
    unite(x + tot, y);
} else {
    if (find(x) == find(y + tot)) break;
    unite(x, y);
    unite(x + tot, y + tot);
}
```

### 复杂度

时间复杂度 $O(m\alpha(m))$。

# 二、关系维护与扩展域并查集

这一章处理同类、异类、距离和敌对关系等带额外信息的并查集。

## 5. [P1196 [NOI2002] 银河英雄传说](https://www.luogu.com.cn/problem/P1196)

`带权并查集` `集合大小` `偏移量`

### 题意

支持把一整列战舰接到另一列后面，并查询两艘战舰之间隔了多少艘。

### 分析

`d[x]` 维护 `x` 到所在集合根的偏移量，也就是它前面有多少艘。合并时把整棵树接到另一棵树末尾，根的偏移量直接加上目标集合的大小。查询时只要两点在同一集合，答案就是偏移量差减一。

### 核心代码

```cpp
int find(int x) {
    if (p[x] == x) return x;
    int t = find(p[x]);
    d[x] += d[p[x]];
    return p[x] = t;
}
// 合并：p[fx] = fy, d[fx] = siz[fy]
```

### 复杂度

时间复杂度 $O(q\alpha(n))$。

## 6. [P2024 [NOI2001] 食物链](https://www.luogu.com.cn/problem/P2024)

`扩展域并查集` `三类关系` `模 3`

### 题意

判断若干句“同类”或“吃”的陈述里有多少是假话。

### 分析

把每个动物拆成三个域，分别表示自己、被自己吃、吃自己的对象。这样三类关系就能用并查集统一维护。若当前陈述和已有关系冲突，就记为假话；否则把对应的三个域合并。

### 核心代码

```cpp
// 同类：x,y 以及它们的三层对应都合并
unite(x, y); unite(x + n, y + n); unite(x + 2 * n, y + 2 * n);
// x 吃 y
unite(x, y + n); unite(x + n, y + 2 * n); unite(x + 2 * n, y);
```

### 复杂度

时间复杂度 $O(k\alpha(n))$。

## 7. [P1892 [BalticOI 2003] 团伙 (Day 2)](https://www.luogu.com.cn/problem/P1892)

`扩展域并查集` `敌人的敌人是朋友`

### 题意

朋友和敌人关系同时存在，朋友的朋友是朋友，敌人的敌人也是朋友，求最终有多少个团伙。

### 分析

把每个人拆成“自己”和“敌人”两份。朋友关系直接合并本体，敌对关系则把一个人的本体和另一个人的敌人域互相合并。最后统计本体域中不同根的数量，就是团伙数。

### 核心代码

```cpp
void enemy(int a, int b) {
    unite(a, b + n);
    unite(b, a + n);
}
void friend(int a, int b) {
    unite(a, b);
    unite(a + n, b + n);
}
```

### 复杂度

时间复杂度 $O(m\alpha(n))$。

## 8. [P1525 [NOIP 2010 提高组] 关押罪犯](https://www.luogu.com.cn/problem/P1525)

`扩展域并查集` `贪心排序` `二分图`

### 题意

给出若干对罪犯之间的仇恨值，要求把罪犯分到两个监狱里，使矛盾尽量晚出现。

### 分析

按仇恨值从大到小处理。对于一对罪犯 `x, y`，如果他们已经被放到同一组，就说明当前仇恨值就是最早无法满足的答案；否则把 `x` 放进 `y` 的对立域，把 `y` 放进 `x` 的对立域。它本质上是在动态维护二分图约束。

### 核心代码

```cpp
sort(e + 1, e + m + 1, [](auto &a, auto &b) { return a.w > b.w; });
for (int i = 1; i <= m; i++) {
    int x = e[i].u, y = e[i].v;
    if (find(x) == find(y)) {
        cout << e[i].w;
        break;
    }
    unite(x, y + n);
    unite(y, x + n);
}
```

### 复杂度

时间复杂度 $O(m\alpha(n))$。

# 三、模板题与静态连通维护

把模板、亲属关系、祖先映射和静态连通块统计放在一起，便于集中复习基础操作。

## 9. [P3367 【模板】并查集](https://www.luogu.com.cn/problem/P3367)

`并查集模板` `路径压缩` `按秩合并`

### 题意

维护若干个集合，支持合并两个集合和查询两个元素是否在同一集合中。

### 分析

这是并查集最基础的模板题。用 `fa[x]` 记录父节点，`find` 负责找到根并在回溯时压缩路径，`unite` 负责把两个根合并到一起。只要把这两个操作写对，后面的很多题都能直接套用。

### 核心代码

```cpp
int fa[N], siz[N];
int find(int x) { return fa[x] == x ? x : fa[x] = find(fa[x]); }
void unite(int x, int y) {
    x = find(x), y = find(y);
    if (x == y) return;
    if (siz[x] < siz[y]) swap(x, y);
    fa[y] = x;
    siz[x] += siz[y];
}
```

### 复杂度

时间复杂度 $O(m\alpha(n))$，空间复杂度 $O(n)$。

## 10. [P1551 亲戚](https://www.luogu.com.cn/problem/P1551)

`并查集` `关系传递`

### 题意

给出若干亲戚关系和询问，判断两个人是否属于同一家族。

### 分析

亲戚关系具有传递性，所以只要把每对亲戚合并到同一个集合里，最后判断两个询问点的祖先是否相同即可。这题就是并查集模板在“关系连通性”上的最直接应用。

### 核心代码

```cpp
for (int i = 1; i <= n; i++) fa[i] = i;
for (int i = 1; i <= m; i++) {
    int a, b;
    cin >> a >> b;
    unite(a, b);
}
cout << (find(x) == find(y) ? "Yes" : "No");
```

### 复杂度

时间复杂度 $O((n+m+p)\alpha(n))$。

## 11. [P2078 朋友](https://www.luogu.com.cn/problem/P2078)

`并查集` `连通块统计`

### 题意

维护朋友关系，朋友的朋友还是朋友，最后按题意统计能够归到同一朋友圈的规模。

### 分析

这类题本质还是把所有直接或间接认识的人放进同一个集合。处理完所有关系后，再按根节点统计每个连通块的大小，答案通常就是某个特定集合的规模或由此导出的配对数。

### 核心代码

```cpp
for (每条朋友关系) unite(a, b);
map<int, int> cnt;
for (int i = 1; i <= n; i++) cnt[find(i)]++;
```

### 复杂度

时间复杂度 $O((n+m)\alpha(n))$。

## 12. [P2814 家谱](https://www.luogu.com.cn/problem/P2814)

`并查集` `字符串映射` `祖先查询`

### 题意

按输入顺序维护家谱关系，并查询某个名字对应的最早祖先。

### 分析

名字是字符串，先用 `map` 把名字映射成编号。每次读到新的父子关系，就让孩子指向当前父亲；查询时一路找到根节点，这个根就是最早祖先。这里的“合并”更像是在维护一棵有向祖先树。

### 核心代码

```cpp
map<string, string> fa;
string find(string x) {
    return fa[x] == x ? x : fa[x] = find(fa[x]);
}
// 读到 #name 就更新当前父亲
// 读到 +name 就 fa[name] = cur
```

### 复杂度

时间复杂度 $O(q\alpha(n))$，其中字符串映射还要加上哈希或 `map` 的代价。

## 13. [P1536 村村通](https://www.luogu.com.cn/problem/P1536)

`并查集` `连通块计数`

### 题意

给定若干村庄和已有道路，问最少还要修几条路才能让所有村庄连通。

### 分析

先把已有道路全部合并，然后统计当前有多少个连通块。若有 `k` 个连通块，最少只需要再修 `k-1` 条路。这是并查集最经典的“数集合”用法。

### 核心代码

```cpp
for (int i = 1; i <= n; i++) fa[i] = i;
for (int i = 1; i <= m; i++) {
    int a, b;
    cin >> a >> b;
    unite(a, b);
}
int cnt = 0;
for (int i = 1; i <= n; i++) if (find(i) == i) cnt++;
cout << cnt - 1;
```

### 复杂度

时间复杂度 $O((n+m)\alpha(n))$。

# 四、Kruskal 与最小瓶颈生成树

最后整理边按权值排序、边合并边判定的最小生成树与最小瓶颈套路。

## 14. [P1111 修复公路](https://www.luogu.com.cn/problem/P1111)

`并查集` `排序` `Kruskal`

### 题意

每条公路有修复时间，问最早什么时候所有村庄连通；若始终不能连通则输出 `-1`。

### 分析

把公路按时间从小到大排序，依次合并两端。如果某一时刻所有点已经在同一个连通块里，那么当前这条路的时间就是答案。这和 Kruskal 的过程完全一致，只是目标从“最小生成树”变成了“最早连通时刻”。

### 核心代码

```cpp
sort(e + 1, e + m + 1, [](auto &a, auto &b) { return a.t < b.t; });
int cnt = n;
for (int i = 1; i <= m; i++) {
    if (unite(e[i].u, e[i].v)) {
        if (--cnt == 1) {
            cout << e[i].t;
            break;
        }
    }
}
```

### 复杂度

时间复杂度 $O(m\log m + m\alpha(n))$。

## 15. [P1195 口袋的天空](https://www.luogu.com.cn/problem/P1195)

`并查集` `Kruskal` `最小生成森林`

### 题意

给定若干点和边，要把图分成恰好 `k` 个连通块，并让总代价最小。

### 分析

还是 Kruskal，但停止条件从“合成一棵树”改成“剩下 `k` 个连通块”。每次选最小边合并两个块，直到连通块数变成 `k` 为止。若边不够，说明无解。

### 核心代码

```cpp
sort(e + 1, e + m + 1, cmp);
int cnt = n, ans = 0;
for (int i = 1; i <= m; i++) {
    if (unite(e[i].u, e[i].v)) {
        ans += e[i].w;
        if (--cnt == k) break;
    }
}
```

### 复杂度

时间复杂度 $O(m\log m + m\alpha(n))$。

## 16. [P1396 营救](https://www.luogu.com.cn/problem/P1396)

`并查集` `最小瓶颈路` `Kruskal`

### 题意

从起点到终点，要求路径上最大边权尽量小。

### 分析

按边权从小到大合并连通块，直到起点和终点第一次连通，这条边的权值就是答案。因为在这个时刻之前都无法连通，而当前这条边已经提供了最小可能的瓶颈。

### 核心代码

```cpp
sort(e + 1, e + m + 1, cmp);
for (int i = 1; i <= m; i++) {
    unite(e[i].u, e[i].v);
    if (find(s) == find(t)) {
        cout << e[i].w;
        break;
    }
}
```

### 复杂度

时间复杂度 $O(m\log m + m\alpha(n))$。

## 17. [P1991 无线通讯网](https://www.luogu.com.cn/problem/P1991)

`并查集` `最小生成树` `聚类`

### 题意

有 `s` 架卫星电话，可以把哨所分成 `s` 个连通块，问无线电的最小最大传输距离。

### 分析

把所有点两两连边并按距离排序，做 Kruskal。最终只保留 `n-s` 条边时，最后加入的那条边权就是答案，也就是把 MST 切成 `s` 段后剩下的最大边。

### 核心代码

```cpp
sort(e + 1, e + cnt + 1, cmp);
int need = n - s, ans = 0;
for (int i = 1; i <= cnt; i++) {
    if (unite(e[i].u, e[i].v)) {
        ans = e[i].w;
        if (--need == 0) break;
    }
}
```

### 复杂度

时间复杂度 $O(n^2\log n)$。

## 18. [P4047 [JSOI2010] 部落划分](https://www.luogu.com.cn/problem/P4047)

`并查集` `Kruskal` `聚类分割`

### 题意

把平面点划分成若干部落，使最近两个部落之间的距离尽量大。

### 分析

本质和上一题很像，还是先把所有边按距离排序，再用并查集合并。假如要分成 `k` 个部落，那么当连通块数减少到 `k` 时，下一条本来会连上的边权就是部落之间的最小距离，也就是答案。

### 核心代码

```cpp
sort(e + 1, e + cnt + 1, cmp);
int comp = n;
for (int i = 1; i <= cnt; i++) {
    if (unite(e[i].u, e[i].v) && --comp == k) {
        cout << e[i].w;
        break;
    }
}
```

### 复杂度

时间复杂度 $O(n^2\log n)$。

## 19. [P1967 [NOIP 2013 提高组] 货车运输](https://www.luogu.com.cn/problem/P1967)

`并查集` `最大生成树` `LCA`

### 题意

查询两点之间能运输的最大载重，若不连通则输出 `-1`。

### 分析

先用并查集按边权从大到小建最大生成树。因为最大生成树能让路径上的最小边尽量大，所以两点在树上的路径瓶颈就是最优答案。建树后再用 LCA 维护倍增数组，查询路径最小边。

### 核心代码

```cpp
sort(e + 1, e + m + 1, [](auto &a, auto &b) { return a.w > b.w; });
for (int i = 1; i <= m; i++) {
    if (unite(e[i].u, e[i].v)) add_tree(e[i].u, e[i].v, e[i].w);
}
// dfs 预处理倍增表，再做 LCA 查询
```

### 复杂度

时间复杂度 $O(m\log m + n\log n + q\log n)$。
