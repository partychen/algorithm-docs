# 洛谷 离线、树上与动态数据结构专题精选解题报告

这一组题从莫队与离线统计一路走到虚树、K-D Tree、可并堆、点分治、LCT 和平衡树，表面上分散，实际上都在处理“标准模板已经装不下”的维护问题。它们共同训练的是另一种能力：先找清真正该维护的对象，再决定用离线重排、树上拆解还是动态结构去承接。

# 一、莫队与离线统计

这一组的共同点是把询问重新排序，再把区间增删或时间回退压成可维护状态。

## 1. [P4688 \[Ynoi Easy Round 2016\] 掉进兔子洞](https://www.luogu.com.cn/problem/P4688)

`莫队` `值域计数` `多区间交`

### 题意

每次给出序列上的三个区间，要求把三个区间中能三方同时配对掉的数按出现次数一一删去，求删完后三个区间剩余元素个数之和。

### 分析

核心是把答案写成 `len1+len2+len3-3*sum(min(c1[x],c2[x],c3[x]))`。做法是用莫队维护三个区间的值频，同时维护每个值在三段中的公共出现次数，这样每次区间端点移动时只需改一个值的贡献。

### 核心代码

```cpp
long long common;

void upd(int id, int x, int d) {
    int old = min(c1[x], min(c2[x], c3[x]));
    cnt[id][x] += d;
    common += min(c1[x], min(c2[x], c3[x])) - old;
}
// ans = len1 + len2 + len3 - 3 * common;
```

### 复杂度

时间复杂度约为 $O((n+m)\sqrt n)$，空间复杂度 $O(n)$。

## 2. [P3674 小清新人渣的本愿](https://www.luogu.com.cn/problem/P3674)

`莫队` `bitset` `存在性判定`

### 题意

给定序列，多次询问区间 `[l,r]` 内是否能选出两个数，使它们的差、和或积恰好等于给定的 `x`。

### 分析

正解是莫队配合 bitset。区间内只维护“某个值是否出现”，差与和都能转成 bitset 平移后的按位与，积则枚举 `x` 的因子判断两个因子是否同时出现。这样三类判定都落在同一套区间维护上。

### 核心代码

```cpp
bitset<V> vis, rev;

bool ok1(int x) { return (vis & (vis << x)).any(); }
bool ok2(int x) { return (vis & (rev >> (MAXV - x))).any(); }
bool ok3(int x) {
    for (int d = 1; d * d <= x; d++)
        if (x % d == 0 && vis[d] && vis[x / d]) return true;
    return false;
}
```

### 复杂度

时间复杂度约为 $O((n+m)\sqrt n\cdot \frac{V}{w} + \sum \sqrt x)$，空间复杂度 $O(V)$。

## 3. [P5355 \[Ynoi Easy Round 2017\] 由乃的玉米田](https://www.luogu.com.cn/problem/P5355)

`莫队` `bitset` `因子枚举`

### 题意

每次询问一个区间，判断其中是否存在两个数，使它们的差、和、积或整除意义下的商等于给定的 `x`。

### 分析

这题仍然是莫队维护值域出现情况。差和和继续用 bitset 平移，积靠枚举因子，商则枚举较小的数 `d`，检查 `d` 与 `d*x` 是否同在区间里。四类判定都围绕“某个值是否出现过”展开。

### 核心代码

```cpp
bool ok4(int x) {
    if (x == 0) return vis[0];
    for (int d = 1; d * x <= MAXV; d++)
        if (vis[d] && vis[d * x]) return true;
    return false;
}
// 其余三种判定与 P3674 同理
```

### 复杂度

时间复杂度约为 $O((n+m)\sqrt n\cdot \frac{V}{w} + \sum (\sqrt x + \frac{V}{x}))$，空间复杂度 $O(V)$。

## 4. [P5313 \[Ynoi2011\] WBLT](https://www.luogu.com.cn/problem/P5313)

`莫队` `根号分治` `bitset`

### 题意

对每个区间 `[l,r]` 和公差 `b`，求最大的 `x`，使得存在某个首项 `a`，让 `a,a+b,...,a+(x-1)b` 这些值都至少在区间中出现一次。

### 分析

区间本质上只关心“哪些值出现过”。做法是莫队维护出现集合，再对 `b` 做根号分治：`b` 小时按模 `b` 分组维护连续段长度，`b` 大时可行长度很短，直接枚举首项即可。这样把长等差链查询拆成小公差预处理与大公差暴力。

### 核心代码

```cpp
int solve_large(int b) {
    int best = 0;
    for (int a = 0; a < b; a++) {
        int len = 0;
        for (int x = a; x <= MAXV; x += b)
            len = exist[x] ? len + 1 : 0, best = max(best, len);
    }
    return best;
}
```

### 复杂度

时间复杂度约为 $O((n+m)\sqrt n + m\sqrt V)$，空间复杂度 $O(V)$。

## 5. [P4887 【模板】莫队二次离线 / 第十四分块(前体)](https://www.luogu.com.cn/problem/P4887)

`莫队二次离线` `异或计数` `分块`

### 题意

多次询问区间 `[l,r]` 中有多少对下标 `(i,j)` 满足 `i<j` 且 `a_i xor a_j` 的二进制里恰好有 `k` 个 `1`。

### 分析

直接莫队会在每次增删时枚举所有异或目标，代价太高。正解把询问按左端点块分类，块内部分用普通移动维护，跨整块的贡献二次离线处理，再用预处理好的 `popcount` 合法转移表统计答案。

### 核心代码

```cpp
for (int x = 0; x < (1 << 14); x++)
    for (int y : trans[x]) cur += cnt[y];

void add(int x) {
    for (int y : trans[x]) ans_now += cnt[y];
    cnt[x]++;
}
```

### 复杂度

时间复杂度约为 $O((n+m)\sqrt n + 2^{14}\binom{14}{k})$，空间复杂度 $O(2^{14})$。

## 6. [P5047 \[Ynoi2019 模拟赛\] Yuno loves sqrt technology II](https://www.luogu.com.cn/problem/P5047)

`莫队` `树状数组` `逆序对`

### 题意

给定一个序列，回答许多区间 `[l,r]` 内的逆序对数量。

### 分析

用普通莫队维护当前区间，再用树状数组按值域统计“小于当前值”和“大于当前值”的数量。右端加入一个数时新增若干前面比它大的元素，左端加入一个数时新增若干后面比它小的元素，删除时反向减掉即可。

### 核心代码

```cpp
void add_right(int x) {
    cur += query(MAXV) - query(x);
    bit.add(x, 1);
}
void add_left(int x) {
    cur += query(x - 1);
    bit.add(x, 1);
}
```

### 复杂度

时间复杂度 $O((n+m)\sqrt n\log V)$，空间复杂度 $O(V)$。

## 7. [P5501 \[LnOI2019\] 来者不拒，去者不追](https://www.luogu.com.cn/problem/P5501)

`莫队` `树状数组` `排名贡献`

### 题意

每次询问一个区间，把区间元素从小到大排序后，若某个值的排名是 `k`，就贡献 `k*a_i`；要求整个区间所有元素的 Abbi 值之和。

### 分析

插入一个值 `x` 时，它自身贡献 `x*(区间内严格小于它的个数+1)`，同时所有严格大于它的值排名都会加一，各自再多贡献一个自身值。于是用莫队维护区间，再用两个树状数组分别维护“数量”和“值之和”即可在线更新总答案。

### 核心代码

```cpp
void add(int x) {
    long long less = bit_cnt.sum(x - 1);
    long long greater_sum = bit_val.sum(MAXV) - bit_val.sum(x);
    cur += 1LL * x * (less + 1) + greater_sum;
    bit_cnt.add(x, 1), bit_val.add(x, x);
}
```

### 复杂度

时间复杂度 $O((n+m)\sqrt n\log V)$，空间复杂度 $O(V)$。

## 8. [P4074 \[WC2013\] 糖果公园](https://www.luogu.com.cn/problem/P4074)

`树上莫队` `带修` `欧拉序`

### 题意

树上每个点有颜色和权值，操作包含修改某个点的颜色，以及询问一条路径上按颜色出现次数计权后的总收益。

### 分析

正解是树上带修莫队。先把树转成欧拉序，路径询问改成区间加一个 LCA 特判；时间维上的颜色修改也一起排序。维护时用“点是否在当前路径内”的翻转技巧更新颜色计数，再按 `V[color] * W[cnt[color]]` 维护答案。

### 核心代码

```cpp
void toggle(int x) {
    int c = col[x];
    cur -= 1LL * val[c] * w[cnt[c]];
    cnt[c] += vis[x] ? -1 : 1;
    cur += 1LL * val[c] * w[cnt[c]];
    vis[x] ^= 1;
}
```

### 复杂度

时间复杂度约为 $O(q\,n^{2/3})$，空间复杂度 $O(n)$。

## 9. [AT_joisc2014_c 歴史の研究](https://www.luogu.com.cn/problem/AT_joisc2014_c)

`回滚莫队` `最大值维护` `离线询问`

### 题意

给定序列，多次询问区间 `[l,r]` 内 `值 × 出现次数` 的最大值。

### 分析

若只是普通莫队，难点会卡在删除：一个值被删掉后，`值 × 出现次数` 的最大值可能来自谁，很难在线回退。可这个目标在“只加入元素”时却很好维护，因为某个值的出现次数只会增加，贡献随之单调更新。

这正是回滚莫队的适用信号。固定左端点块后，让右端点只向右扩张，于是大区间部分永远只做加入；对于块内零散的左端点，再临时向左补几个元素算答案，算完后把这部分整体回滚。这样整个框架里，真正长期维护的数据结构从来不需要支持困难的删除操作。

所以这题最该学的是选莫队变种的判断标准：**如果加入容易、删除难，而询问又很多，就优先考虑回滚莫队把删除变成“临时补、算完撤销”。**

### 核心代码

```cpp
void add(int x) {
    cnt[x]++;
    cur = max(cur, 1LL * val[x] * cnt[x]);
}
for (auto q : bucket[id]) {
    while (R < q.r) add(a[++R]);
    Snapshot bak = save();
    while (L > q.l) add(a[--L]);
    ans[q.id] = cur, load(bak), L = block_right + 1;
}
```

### 复杂度

时间复杂度约为 $O((n+m)\sqrt n)$，空间复杂度 $O(n)$。

## 10. [P5906 【模板】回滚莫队&不删除莫队](https://www.luogu.com.cn/problem/P5906)

`回滚莫队` `最远相同值距离` `首尾位置`

### 题意

每次询问区间 `[l,r]` 中，相同数字之间最远的下标差是多少；若没有重复数则输出 `0`。

### 分析

这题要维护的是每个值在当前区间里的最左、最右出现位置，从而得到最远距离。问题在于一旦删除左端点或右端点，某个值新的最左 / 最右是谁并不好快速恢复，所以普通莫队不顺手。

回滚莫队正好绕开这个痛点。把询问按左端点分块后，右端点统一向右扫，这样长期状态里每个值的首尾位置只会增加信息；块内左端点不足的那一小段，再临时向左补进去更新答案，算完询问后整段撤销即可。

因此这题和上一题一样，核心不是“会用回滚莫队模板”，而是先识别出：**维护量对加入友好、对删除不友好。**一旦看出这一点，做法就比较自然了。

### 核心代码

```cpp
void add(int pos) {
    int x = a[pos];
    if (!Lft[x]) Lft[x] = pos;
    Rgt[x] = pos;
    cur = max(cur, Rgt[x] - Lft[x]);
}
```

### 复杂度

时间复杂度约为 $O((n+m)\sqrt n)$，空间复杂度 $O(n)$。

## 11. [P1903 【模板】带修莫队 / \[国家集训队\] 数颜色 / 维护队列](https://www.luogu.com.cn/problem/P1903)

`带修莫队` `颜色种类数` `时间维`

### 题意

在一个颜色序列上支持单点改色，并询问区间 `[l,r]` 内不同颜色的种类数。

### 分析

一旦区间询问之间夹着修改，普通莫队的二维排序就不够了，因为同一个 `[l,r]` 在不同修改时刻对应的是不同数组。于是每个询问实际上都带着第三维：它发生在第几次修改之后。

这就是带修莫队的本质：把询问看成三维点 `(l,r,t)`，排序时同时控制左右端点和时间指针的移动。左右端移动和普通莫队一样，时间指针前进 / 后退则表示应用或撤销某次改色；如果被改位置当前就在维护区间内，就要同步删掉旧颜色、加入新颜色。

所以这题最关键的不是背排序公式，而是理解模型升级：**有修改的区间询问 = 在普通莫队上再加一根时间轴。**

### 核心代码

```cpp
void apply(int t, int dir) {
    int p = chg[t].pos, &x = a[p];
    if (L <= p && p <= R) del(x), add(chg[t].to);
    swap(x, chg[t].to);
}
```

### 复杂度

时间复杂度约为 $O(n^{5/3})$，空间复杂度 $O(n)$。

## 12. [P1494 \[国家集训队\] 小 Z 的袜子](https://www.luogu.com.cn/problem/P1494)

`莫队` `组合计数` `概率`

### 题意

每次询问区间 `[l,r]` 内随机取两只袜子，它们颜色相同的概率是多少。

### 分析

这题看起来在问概率，其实本质还是计数。若把“随机取两只袜子”看成按顺序选两个位置，那么分母就是区间内所有有序二元组数 `len*(len-1)`。

而分子只需要统计颜色相同的有序二元组。某种颜色若出现了 `cnt` 次，就能贡献 `cnt*(cnt-1)`；于是整个问题被完全翻译成了区间内各颜色计数的组合和。这样莫队就能直接维护，每次加一个颜色时，分子增加的正是它与已有同色袜子能配成的有序对数量。

所以这题值得记住的不是“概率题也能莫队”，而是一个常见翻译：**先把概率拆成分子分母，再看分子是不是一个可动态维护的计数式。**

### 核心代码

```cpp
long long num;

void add(int x) { num += 2LL * cnt[x]; cnt[x]++; }
void del(int x) { cnt[x]--; num -= 2LL * cnt[x]; }
// numerator = num, denominator = 1LL * len * (len - 1)
```

### 复杂度

时间复杂度 $O((n+m)\sqrt n)$，空间复杂度 $O(n)$。

## 13. [P2709 【模板】莫队 / 小 B 的询问](https://www.luogu.com.cn/problem/P2709)

`莫队` `平方和` `计数维护`

### 题意

多次询问区间 `[l,r]` 中各个值出现次数平方和，也就是 `sum(cnt_x^2)`。

### 分析

这题是莫队最好的入门模型之一，因为答案虽然看着像一个整体式子，但真正会变的只有被增删的那个值对应的平方项。

假设当前加入一个值 `x`，原来它出现了 `cnt_x` 次，那么答案变化量就是 `(cnt_x+1)^2-cnt_x^2=2cnt_x+1`；删除时同理反向更新。也就是说，只要维护每个值在当前区间的出现次数，整个区间答案就能在 `O(1)` 里改动。

所以这题真正训练的是莫队的基本眼光：**先问区间答案能不能写成“很多值各自贡献的和”，再看一次增删是否只影响其中一个局部项。**

### 核心代码

```cpp
long long cur;

void add(int x) { cur += 2LL * cnt[x] + 1, cnt[x]++; }
void del(int x) { cnt[x]--, cur -= 2LL * cnt[x] + 1; }
```

### 复杂度

时间复杂度 $O((n+m)\sqrt n)$，空间复杂度 $O(n)$。

# 二、单调结构与序列贪心

这一组都靠“最近更大/更小”或“维护单调候选集”来一次扫描解决。

## 14. [P1901 发射站](https://www.luogu.com.cn/problem/P1901)

`单调栈` `最近更高点` `贡献累加`

### 题意

每个发射站有高度和能量，它会把能量发给左右两侧离它最近且比它更高的发射站，求最后收到总能量最大的站点。

### 分析

题目真正的计算对象不是“往哪发能量”，而是每个位置左右两侧的**最近更高站**。一旦这个关系确定，每个站的贡献只会落到至多两个目标上。

单调栈之所以适合，是因为从左到右扫描时，栈里保留下来的正是一条“可能成为后面某人最近更高站”的高度轮廓。若栈顶不比当前站高，它以后也不可能再成为任何更右侧位置的最近更高站，因为当前站更靠右且高度不低，已经把它挡死了。

所以扫描到位置 `i` 时，弹完以后留下的栈顶，就是 `i` 左边最近更高站；反向再做一遍，就能找到右边最近更高站。这题最该形成的模板感是：**最近更大 / 更高关系，本质上是在维护一条不断淘汰劣候选的单调轮廓。**

### 核心代码

```cpp
while (!st.empty() && h[st.top()] <= h[i]) st.pop();
if (!st.empty()) sum[st.top()] += v[i];
st.push(i);
// 从左到右、从右到左各做一次
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 15. [P1823 \[COI 2007\] Patrik 音乐会的等待](https://www.luogu.com.cn/problem/P1823)

`单调栈` `可见对数` `相同高度压缩`

### 题意

一排人按顺序站立，若两人之间没有比他们更高的人就互相看得见，要求统计所有可见的人对数量。

### 分析

经典单调栈维护“从栈底到栈顶严格递减的高度”。遇到新高度时，所有不高于它的栈顶都会被它看到；对于相同高度，需要把连续相同的人数压成一组，既能一次统计同高对数，也能继续判断组后面是否还有更高的人。

### 核心代码

```cpp
while (!st.empty() && st.top().h < x) ans += st.top().cnt, st.pop();
if (!st.empty() && st.top().h == x) {
    ans += st.top().cnt;
    if (st.size() > 1) ans++;
    st.top().cnt++;
} else {
    if (!st.empty()) ans++;
    st.push({x, 1});
}
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 16. [P5788 【模板】单调栈](https://www.luogu.com.cn/problem/P5788)

`单调栈` `下一个更大元素` `模板`

### 题意

对每个位置 `i`，求它右侧第一个比 `a_i` 大的位置编号，不存在则输出 `0`。

### 分析

这题是单调栈最标准的原型。难点不在实现，而在想清楚栈里到底存谁：它存的是“右侧第一个更大元素还没找到”的那些位置。

从左到右扫描到 `i` 时，若 `a[i]` 比栈顶位置更大，那么 `i` 一定就是栈顶的答案。因为栈顶之后还没有出现过更大的数，否则它早就该出栈了；而 `i` 又是第一个把它压倒的位置，所以答案可以立刻确定。

于是栈中值始终保持递减，保证每个位置只会进栈出栈一次。这道模板题最重要的不是背代码，而是理解这个句式：**当前元素一旦足够强，就能替一批“尚未找到答案”的旧元素结案。**

### 核心代码

```cpp
for (int i = 1; i <= n; i++) {
    while (!st.empty() && a[st.top()] < a[i]) ans[st.top()] = i, st.pop();
    st.push(i);
}
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 18. [P2866 \[USACO06NOV\] Bad Hair Day S](https://www.luogu.com.cn/problem/P2866)

`单调栈` `可见数量` `递减栈`

### 题意

每头奶牛只向右看，直到被第一头不比自己矮的奶牛挡住，要求统计所有奶牛能看到的奶牛总数。

### 分析

如果直接站在每头奶牛的视角往右扫，会把题做成平方级。更自然的想法是反过来：当第 `i` 头奶牛走到场上时，左边哪些奶牛还能看见它？

维护一个严格递减栈后，栈里留下的都是一条“从近到远逐渐变高的挡板链”。当前奶牛会把所有不高于它的栈顶弹掉，因为这些奶牛以后既挡不住它，也挡不住任何更右侧奶牛；弹完后，栈中剩余的每一头奶牛都比它高，并且中间没有更高者遮挡，因此都能看见它。

于是把当前栈大小加入答案即可。这题很适合帮助理解另一类单调栈视角：**不是给当前元素找答案，而是统计当前元素能一次性解决多少旧元素之间的可见关系。**

### 核心代码

```cpp
for (int i = 1; i <= n; i++) {
    while (!st.empty() && h[st.top()] <= h[i]) st.pop();
    ans += st.size();
    st.push(i);
}
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 20. [P8082 \[COCI 2011/2012 #4\] KEKS](https://www.luogu.com.cn/problem/P8082)

`单调栈` `贪心` `删位最大化`

### 题意

给定一个数字串，必须删除恰好 `k` 位，使剩下的数字按原相对顺序组成的数尽量大。

### 分析

决定数值大小时，越靠前的位权越高，所以贪心重点一定落在“尽量让前面的数字更大”。若某一位前面放着更小的数字，而后面来了更大的数字，并且我们还有删除次数，那么删掉前面的较小数字永远不亏，因为它能把更大的数字提到更高位。

于是从左到右扫描时，只要当前字符 `c` 比栈顶更大且还能删除，就持续弹栈；这等价于不断修正前缀，使当前保留下来的前缀在所有可行方案里字典序最大。若扫完整串后还没删够，说明整个序列已经单调不增，这时只能从末尾删掉最低位。

所以这题最该记住的不是“单调栈删数字”这个结论，而是背后的交换逻辑：**高位优先，大数尽量前移。**

### 核心代码

```cpp
for (char c : s) {
    while (!st.empty() && k && st.back() < c) st.pop_back(), k--;
    st.push_back(c);
}
while (k--) st.pop_back();
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

这一章收拢虚树、圆方树、树剖矩阵、树上差分以及一类图上结构压缩题。它们的共同点不是某个单一模板，而是都要先把原图或原树提炼成更稳定的结构，再在那层结构上做统计或转移。

## 21. [P8820 \[CSP-S 2022\] 数据传输](https://www.luogu.com.cn/problem/P8820)

`树链剖分` `矩阵DP` `k<=3`

### 题意

树上每台主机有处理时间，若两点树距不超过 `k` 就能直接传输；对每次 `(s,t)` 询问，求从 `s` 到 `t` 完成转发所需的最小总时间。

### 分析

这题看上去像树上最短路变形，但真正特殊的地方是 `k<=3`。因为一次传输最远只能跨 `3` 条边，所以在沿路径推进时，后续决策只会依赖“离当前末端最近的少数几个候选中转点”，根本不需要记整段历史。

这就把问题压成了一个很小的状态机：走过一段链后，末端保留哪些最近点、对应最小代价是多少。树链剖分负责把任意路径拆成若干重链段，而每段链都可以预处理成一个小矩阵，描述“入口状态 -> 出口状态”的最优转移。

查询时按路径顺序把这些矩阵连乘，就等于把整条路径上的局部转移拼起来。所以这题最该学的是：**当路径 DP 的记忆窗口很小，就可以把链段压成矩阵，再靠树剖做快速拼接。**

### 核心代码

```cpp
Mat merge(const Mat &A, const Mat &B) {
    Mat C;
    for (int i = 0; i < S; i++)
        for (int j = 0; j < S; j++)
            for (int k = 0; k < S; k++)
                C[i][j] = min(C[i][j], A[i][k] + B[k][j]);
    return C;
}
```

### 复杂度

预处理 $O(n\log n)$，单次询问 $O(\log n)$ 到 $O(\log^2 n)$，空间复杂度 $O(n)$。

```cpp
void dfs(int x) {

        dp[x] += min(1LL * w, dp[y]);

    if (key[x]) dp[x] = INF;
}
```

单次询问建树与 DP 都是 $O(k\log k)$，空间复杂度 $O(k)$。

## 30. [CF19E Fairy](https://www.luogu.com.cn/problem/CF19E)

### 题意
给定无向图，求删掉哪一条边后整张图会变成二分图。
### 分析
先别直接想“删边后重判二分图”，那会把问题做成很多次判定。真正该抓的是：一张图不是二分图，当且仅当里面存在奇环；所以能删的边，必须把**所有奇环**同时打掉。

DFS 染色后，只有连向同色点的返祖边才会制造奇环，而且这条奇环在 DFS 树上对应一条确定的树链。于是每条奇环都能转成“给这条树链打一次标记”，再用树上差分统计每条树边被多少个奇环经过。若某条树边恰好被全部奇环覆盖，删掉它就能一次切断所有奇环。

非树边也要单独考虑：如果整张图只有这一条同色返祖边制造奇环，那么直接删它也成立。所以这题真正的转化链是：**删边变二分图 -> 删边打掉全部奇环 -> 统计每条边覆盖了多少条奇环。**
### 核心代码
```cpp
void dfs(int u, int p) {
    for (auto [v, id] : g[u]) if (id != p) {
        if (!dep[v]) dep[v] = dep[u] + 1, fa[v] = u, dfs(v, id);
        else if (dep[v] < dep[u]) {
            if ((dep[u] - dep[v] + 1) & 1) odd++, tag[u]++, tag[v]--, mark[id] = 1;
        }
    }
}
```

## 31. [P5227 [AHOI2013] 连通图](https://www.luogu.com.cn/problem/P5227)
`边双连通` `桥树` `小集合删边`
### 题意
给定连通图和很多个小集合，每次独立删除集合中的若干条边，判断删完后图是否仍然连通。
### 分析
这题最容易误入的方向，是把每次询问都当成一次动态删边连通性。其实题目有个很强的限制：每次只删很少几条边，而且询问彼此独立。

先抓一个根本事实：删掉非桥永远不会让图断开，所以真正有杀伤力的只有桥。把原图做边双连通缩点后，会得到一棵桥树；原题每次删边，其实只是问“删掉桥树上的少数几条树边后，这棵树是否还连通”。

因此难点已经从一般图动态连通，降成了桥树上的小规模判定。因为每次只涉及最多 `4` 条边，可以把这些桥映射到桥树上，再利用 DFS 序、祖先关系或虚树视角判断它们是否把树切成多块。核心迁移点是：**多次独立删边题，先问哪些边真的可能改变连通性。**
### 核心代码
```cpp
build_bridge_tree();
for (auto &qry : queries) {
    vector<int> cut;
    for (int id : qry.edges) if (isBridge[id]) cut.push_back(treeEdgeId[id]);
    puts(disconnected_by(cut) ? "Disconnected" : "Connected");
}
```
### 复杂度
预处理 `O(n+m)`，单次询问 `O(c log n)`，其中 `c<=4`。

## 32. [P1600 [NOIP 2016 提高组] 天天爱跑步](https://www.luogu.com.cn/problem/P1600)
`树上差分` `深度统计` `LCA`
### 题意
每个玩家沿树上最短路从 `s` 跑到 `t`，第 `j` 个观察员只在第 `w[j]` 秒观察一次，问每个点恰好能看到多少名玩家。
### 分析
这题最难的地方不是树上差分本身，而是先把“第 `w[u]` 秒刚好经过点 `u`”翻译成可统计的式子。对一条路径 `s -> t` 而言，玩家到达路径上某点的时刻，分成经过 `lca(s,t)` 之前和之后两种公式。

若点 `u` 在从 `s` 往 `lca` 走的这一段上，那么到达 `u` 的时刻就是 `dep[s] - dep[u]`，等价于 `dep[s] = w[u] + dep[u]`；这说明对当前点 `u`，只要统计经过它的玩家里有多少起点深度等于这个值即可。若 `u` 落在 `lca` 到 `t` 这一段，时刻公式会改写成另一种“固定的深度差 / 路程差”形式，因此要再开一套桶。

于是每条路径都被拆成两类贡献，DFS 回溯时把属于子树的信息累积上来，就能在每个点读取两套桶的增量。这题真正要学的是：**先把“某时刻经过某点”化成只依赖当前点和路径端点的等式，再决定用差分还是桶去承接。**
### 核心代码
```cpp
void dfs(int u, int p) {
    int a = bucket1[w[u] + dep[u]];
    int b = bucket2[w[u] - dep[u] + OFFSET];
    for (int v : g[u]) if (v != p) dfs(v, u);
    for (auto id : startAt[u]) bucket1[dep[s[id]]]++;
    for (auto id : endAt[u]) bucket2[dep[s[id]] - dist[t[id]] + OFFSET]++;
    ans[u] += bucket1[w[u] + dep[u]] - a;
    ans[u] += bucket2[w[u] - dep[u] + OFFSET] - b;
}
```
### 复杂度
时间复杂度 `O((n+m)log n)` 或 `O(n+m)`（配合桶）。

## 33. [CF1681F Unique Occurrences](https://www.luogu.com.cn/problem/CF1681F)
`边权分组` `虚树` `计数`
### 题意
对树上所有点对 `(u,v)`，统计路径上“出现次数恰好为一次的边权值个数”的总和。
### 分析
总答案可以按边权拆开算。固定某个权值 `c`，我们只需要知道：有多少点对的路径上，权值为 `c` 的边恰好出现一次。把所有权值混在一起看很乱，但逐个权值拆开后就只剩一类局部结构。

```cpp
for (auto &vec : byColor) {
```

## 34. [P4148 简单题](https://www.luogu.com.cn/problem/P4148)

`动态 K-D Tree` `二维数点` `矩形求和`

### 题意

在一个很大的棋盘上支持单点加值，并反复询问某个矩形区域内所有格子的权值和。

### 分析

如果坐标小且静态，二维树状数组或二维前缀和会更自然；但这题坐标范围极大、点又是动态插入的，直接开网格或事先完整离散化都不现实。于是更合适的思路是只围绕“真的出现过的点”建空间结构。

动态 K-D Tree 正是这样做：每个结点存一个点，同时维护整棵子树的包围盒和权值和。查询矩形时，若某个包围盒被询问矩形完全包含，就能把这棵子树的总和整块拿走；若完全不交，则整棵剪掉。只有部分相交时，才继续递归下探。

因此这题最值得掌握的不是实现细节，而是选型思路：**二维动态稀疏点集 + 矩形统计，优先考虑能按包围盒整块剪枝的空间划分结构。**

### 核心代码

```cpp
int query(int p, Rect q) {
    if (!p || disjoint(box[p], q)) return 0;
    if (inside(box[p], q)) return sum[p];
    int ret = in(node[p], q) ? val[p] : 0;
    return ret + query(ls[p], q) + query(rs[p], q);
}
```

### 复杂度

均摊时间复杂度约为 $O(\sqrt n)$ 到 $O(n^{2/3})$，空间复杂度 $O(n)$。

## 35. [P2093 \[国家集训队\] JZPFAR](https://www.luogu.com.cn/problem/P2093)

`K-D Tree` `第k远点` `优先队列`

### 题意

给定平面点集，许多询问给出查询点 `(px,py)` 和 `k`，要求离它第 `k` 远的点编号；距离相同时编号较小者算更远。

### 分析

K-D Tree 结点维护包围盒，对一个查询点可以快速算出某棵子树可能达到的最远距离上界。搜索时总是优先进入上界更大的儿子，并用一个大小为 `k` 的最小堆维护当前最优候选，明显不可能更优的子树直接剪掉。

### 核心代码

```cpp
void solve(int p) {
    if (!p || up_dist(box[p], q) < heap.top()) return;
    push_answer(point_dist(pt[p], q), id[p]);
    int l = up_dist(box[ls[p]], q), r = up_dist(box[rs[p]], q);
    if (l < r) swap(ls[p], rs[p]);
    solve(ls[p]), solve(rs[p]);
}
```

### 复杂度

单次询问期望复杂度约为 $O(\log n + k\log k)$，空间复杂度 $O(n)$。

## 37. [P6247 \[SDOI2012\] 最近最远点对](https://www.luogu.com.cn/problem/P6247)

`最近点对分治` `凸包` `旋转卡壳`

### 题意

给定平面点集，要求同时求出欧几里得距离最近的点对距离和最远的点对距离。

### 分析

这题看起来像一个综合题，其实是把两个经典结论拼在一起：最近点对和最远点对虽然都问“点对距离”，但它们依赖的几何结构完全不同。

最近点对用分治，是因为若左右两边各自最优距离已经算出，跨中线的更优答案只可能出现在宽度为 `d` 的窄条里；再按 `y` 排序后，每个点只需比较后面常数个候选。最远点对则完全相反：内部点不可能成为直径端点，所以先缩到凸包，再在凸包上用旋转卡壳找最远点对。

因此这题真正值得总结的是：**同样是点对问题，最近靠局部稠密性剪枝，最远靠边界性缩掉内部点。**把这两个事实分开记，会比记一道“综合模板题”更有迁移价值。

### 核心代码

```cpp
double mn = solve_closest(1, n);
vector<Point> hull = convex_hull(p);
for (int i = 0, j = 1; i < hull.size(); i++) {
    while (area_inc(i, j, hull)) j = (j + 1) % hull.size();
    mx = max(mx, dist2(hull[i], hull[j]));
}
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(n)$。

## 38. [P1429 平面最近点对（加强版）](https://www.luogu.com.cn/problem/P1429)

`分治` `平面最近点对` `经典模板`

### 题意

给定平面上 `n` 个点，求所有点对中的最小欧几里得距离。

### 分析

暴力枚举所有点对会是 `O(n^2)`，最近点对分治的关键是把“跨左右两半的候选”压缩到极少。先按 `x` 排序递归后，左右两边内部的最优值设为 `d`，那么真正可能更优的跨区点对，横坐标差一定小于 `d`，所以只会落在中线附近的窄条里。

接下来再利用平面几何的稀疏性：把窄条中的点按 `y` 排序后，对于每个点，只需要检查它后面 `y` 距离小于 `d` 的少数几个点，因为更远的点不可能形成更近距离，过密则会与当前 `d` 矛盾。

所以最近点对分治最该理解的是两个压缩：**先用 `x` 把问题分半，再用当前最优值把跨区候选缩到一条窄条和常数个邻居。**

### 核心代码

```cpp
double solve(int l, int r) {
    if (l == r) return INF;
    int mid = (l + r) >> 1;
    double d = min(solve(l, mid), solve(mid + 1, r));
    merge_by_y(l, mid, r);
    return check_strip(l, r, mid, d);
}
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(n)$。

## 45. [P3377 【模板】可并堆 1](https://www.luogu.com.cn/problem/P3377)

`左偏树` `堆合并` `模板`

### 题意

维护若干个小根堆，支持合并两个堆以及删除并输出某个堆的最小值。

### 分析

可并堆和普通堆最大的区别，是“合并”必须足够快。左偏树就是专门为 `meld` 设计的：它强制右链尽量短，因此递归合并时只会沿着右链往下走，复杂度才能压到对数级。

具体地说，合并两棵堆时先保留较小根作为新根，再把另一棵堆和它的右子树继续合并；回溯时若发现左儿子的空路径长小于右儿子，就交换左右儿子，维持“左边更重、右边更轻”的左偏性。这样右链长度始终受控。

删除最小值则更自然：堆顶就是最小值，删掉后把左右子树再 meld 回去即可。所以这道模板题真正该记住的是：**左偏树的所有操作几乎都在为 merge 服务，pop 只是 merge 的副产品。**

### 核心代码

```cpp
int merge(int x, int y) {
    if (!x || !y) return x | y;
    if (val[x] > val[y]) swap(x, y);
    rs[x] = merge(rs[x], y);
    if (dist[ls[x]] < dist[rs[x]]) swap(ls[x], rs[x]);
    dist[x] = dist[rs[x]] + 1;
    return x;
}
```

### 复杂度

单次合并或删除最小值时间复杂度 $O(\log n)$，空间复杂度 $O(n)$。

每个人属于一个部落，操作包括合并两个部落，以及弹出某个部落当前战斗力最小的人。

### 分析

这题把两个最经典的职责拆得很清楚：并查集只负责回答“这个人现在属于哪个部落”，左偏树只负责维护“这个部落里当前谁最小、两堆如何合并”。

```cpp
int fx = find(x), fy = find(y);
if (fx != fy) fa[fx] = fy, rt[fy] = merge(rt[fx], rt[fy]);
int t = rt[find(x)];
print(val[t]);
rt[find(x)] = merge(ls[t], rs[t]);
```

单次操作时间复杂度 $O(\log n)$，并查集额外均摊 $O(\alpha(n))$。

## 47. [P1456 Monkey King](https://www.luogu.com.cn/problem/P1456)

`左偏树` `并查集` `对战模拟`

### 题意

每只猴子有战斗力，两只猴子所在的部落决斗时，各自拿出堆顶最强者并把战斗力减半后再放回，最后输出胜者所在部落当前最强值。

### 分析

题目的战斗规则只会动到每个部落当前最强的猴子，所以部落内部没有必要维护完整顺序，只要能高效拿堆顶、删堆顶、再把两边合并回去即可，因此最自然的数据结构就是大根可并堆。

具体流程是：先用并查集找到两只猴子所在部落，若已经同属一部落就无事可做；否则分别弹出两边堆顶，把这两只猴子的战斗力减半后重新插回，再把两个部落合并。由于之后部落还会继续参与战斗，必须支持反复 meld，这也是普通优先队列不够用、左偏树刚好合适的原因。

所以这题并不只是“并查集 + 左偏树”模板，而是在训练一个判断：**当操作总是围绕集合最值展开，且集合之间会频繁合并时，就该上可并堆。**

### 核心代码

```cpp
int x = pop_max(rt[fx]), y = pop_max(rt[fy]);
val[x] >>= 1, val[y] >>= 1;
rt[fx] = merge(rt[fx], x);
rt[fy] = merge(rt[fy], y);
rt[fx] = merge(rt[fx], rt[fy]), fa[fy] = fx;
```

### 复杂度

单次操作时间复杂度 $O(\log n)$，空间复杂度 $O(n)$。

## 48. [P3273 [SCOI2011] 棘手的操作](https://www.luogu.com.cn/problem/P3273)
`并查集` `可并堆` `懒标记`
### 题意
图从若干孤点开始，支持连边合并连通块、单点加、整块加、全局加，并查询单点值、块内最大值和全局最大值。
### 分析
这题一眼看上去操作很多，但其实值只分三层来源：点自己的基值、所在连通块的整块加、以及全局统一加。把这三层拆开后，每个点真实值都能写成 `base[x] + tagComp[find(x)] + tagAll`。

这样单点加只改 `base[x]`，整块加只改块代表元的懒标记，全局加只改一个总标记；真正麻烦的是“块内最大值”和“连边合并”。为此需要给每个连通块配一个能快速取最大、还能 meld 的结构，于是可并堆正好接住：块内每个点只存去掉块标记和全局标记后的相对值，合并连通块时直接并堆即可。

所以这题的关键不是记操作，而是先拆值的层次：**谁是点私有的，谁是块共享的，谁是全局共享的。**层次一旦拆开，数据结构就自然变成“并查集 + 可并堆 + 两层懒标记”。
### 核心代码
```cpp
int find(int x) { return fa[x] == x ? x : fa[x] = find(fa[x]); }
void unite(int x, int y) {
    x = find(x); y = find(y);
    if (x == y) return;
    fa[x] = y;
    heap[y] = merge(heap[y], heap[x]);
    tagComp[y] += 0;
    mxAll = max(mxAll, top(heap[y]) + tagComp[y] + tagAll);
}
```
### 复杂度
单次操作均摊 `O(log n)`。

## 49. [P4331 [BalticOI 2004] Sequence (Day1)](https://www.luogu.com.cn/problem/P4331)
`slope trick` `可并堆` `单调回归`
### 题意
给定序列 `t[i]`，要求构造一个严格递增序列 `z[i]`，使 `sum |t[i]-z[i]|` 最小，并输出最小代价与构造结果。
### 分析
“严格递增”这种约束很难直接和绝对值放在一起优化，所以第一步要先把约束变平：令 `y[i] = z[i] - i`，那么 `z` 严格递增就等价于 `y` 非降。题目于是变成：找一个非降序列 `y`，最小化 `sum |(t[i]-i) - y[i]|`。

这就是经典的 `L1` 单调回归。若一段位置最终被压成同一个值，那么这段的最优代表值一定是中位数；因此可以把每个位置先看成一个块，维护块内中位数。只要发现前一个块的中位数大于后一个块，就说明单调性被破坏，这两个块在最优解中不可能分开，必须合并。

可并堆在这里负责维护块内中位数与合并过程，最后再把每个块的最优值回填成整段答案。所以这题最值得迁移的是：**遇到“绝对值 + 单调性”时，先试着把它翻译成单调回归，再想中位数和块合并。**
### 核心代码
```cpp
for (int i = 1; i <= n; i++) {
    heap[++tot].push(t[i] - i);
    L[tot] = R[tot] = i;
    while (tot > 1 && top(heap[tot - 1]) > top(heap[tot])) {
        heap[tot - 1] = merge(heap[tot - 1], heap[tot]);
        R[tot - 1] = R[tot];
        tot--;
    }
}
assign_by_block_median();
```
### 复杂度
时间复杂度 `O(n log n)`，空间复杂度 `O(n)`。

```cpp
for (int x = u; x; x = fa[x]) {

```

### 题意

给定带权树和若干个长度 `k`，判断树上是否存在一对点，其距离恰好等于每个给定的 `k`。

```cpp
vis[0] = true;
for (int y : g[c]) {

}
```

时间复杂度约为 $O(n\log n + m\,n\log n / w)$，常见实现可视为 $O(n\log n)$ 级别，空间复杂度 $O(n)$。

## 52. [P4178 Tree](https://www.luogu.com.cn/problem/P4178)
`点分治` `树上距离` `点对统计`
### 题意
给定一棵带边权树，统计距离不超过 `k` 的点对数量。
### 分析
树上点对统计最怕的是每条路径都去单独看一遍。点分治的切入点是：任意一条路径，要么完全落在某个重心儿子子树里，要么一定经过当前重心。

因此处理重心 `c` 时，只需要统计“经过 `c` 的合法点对”。把每个儿子子树到 `c` 的距离收集出来后，先把所有距离放在一起排序，用双指针统计距离和不超过 `k` 的点对数；这一步会把同一儿子子树内部的点对也算进去，所以还要对每个儿子子树各自做一次同样统计并减掉。

这样留下来的就恰好是经过重心的点对，再递归处理各个子树即可。也就是说，这题里点分治真正做的事是：**把“所有点对”按路径经过哪个最高层重心分摊掉。**
### 核心代码
```cpp
void solve(int u) {
    vis[u] = 1;
    vector<int> all = {0};
    for (auto [v, w] : g[u]) if (!vis[v]) {
        vector<int> cur;
        get_dis(v, u, w, cur);
        ans -= count_pair(cur);
        all.insert(all.end(), cur.begin(), cur.end());
    }
    ans += count_pair(all);
    for (auto [v, _] : g[u]) if (!vis[v]) solve(get_rt(v));
}
```
### 复杂度
时间复杂度 `O(n log n)`，空间复杂度 `O(n)`。

## 53. [P2056 [ZJOI2007] 捉迷藏](https://www.luogu.com.cn/problem/P2056)
`点分治` `动态最远点对`
### 题意
树上房间的灯会开关切换，关灯房间才可能藏人。每次询问当前所有关灯房间中最远两点的距离。
### 分析
静态最远点对可以靠树形 DP，但这里点会动态开关，所以需要一个能局部更新、全局取最优的结构。点分治的优势在于：每个点只会落在一条长度 `O(log n)` 的重心祖先链上。

对于某个重心 `c`，所有“路径经过 `c`”的候选答案，只取决于来自不同儿子块的两条最大距离。因此可以给每个儿子块维护一个可删堆，记录该块内所有激活点到 `c` 的距离；再用一个总堆维护各儿子块的堆顶，重心 `c` 的最佳答案就是其中最大的两项之和。

切换一个点时，只需沿它的重心祖先链，把这个点到各级重心的距离插入或删除；查询时再从全局堆里取所有重心答案的最大值。这题的核心理解是：**动态最远点对不是直接维护点对，而是维护每个重心作为“拐点”时能提供的最好答案。**
### 核心代码
```cpp
void toggle(int x) {
    on[x] ^= 1;
    for (int u = x; u; u = fa[u]) {
        int d = dist(x, u);
        if (!on[x]) ins(subHeap[childBelong(x, u)], d), ins(allHeap[u], top(subHeap[childBelong(x, u)]));
        else del(subHeap[childBelong(x, u)], d), refresh(u);
    }
}
```
### 复杂度
单次修改、查询都是 `O(log^2 n)`。

`LCT` `动态森林` `路径异或`

### 题意

维护一个动态森林，支持连边、断边、修改点权，以及查询两点路径上的点权异或和。

### 分析

Link-Cut Tree 通过伸展树维护实链。`makeroot(x)` 后再 `access(y)`，`y` 的辅助树就正好对应 `x` 到 `y` 的整条路径；路径异或和作为 splay 的区间信息维护即可，连边断边则分别用 `link` 和 `cut` 完成。

### 核心代码

```cpp
void split(int x, int y) { makeroot(x); access(y); splay(y); }
void link(int x, int y) { makeroot(x); if (findroot(y) != x) fa[x] = y; }
void cut(int x, int y) {
    split(x, y);
    if (ch[y][0] == x && !ch[x][1]) ch[y][0] = fa[x] = 0, pushup(y);
}
```

### 复杂度

单次操作时间复杂度均摊 $O(\log n)$，空间复杂度 $O(n)$。

## 55. [P2147 [SDOI2008] 洞穴勘测](https://www.luogu.com.cn/problem/P2147)
`Link-Cut Tree` `动态连通`
### 题意
动态维护一片森林，支持连边、断边以及询问两个点当前是否连通。
### 分析
并查集只能处理加边不能删边，所以这题必须维护一棵真正会变化的森林结构。Link-Cut Tree 的切入点不是“维护整棵树”，而是维护一组首选路径，把树操作化成若干次 splay。

当我们执行 `makeroot(x)` 后，再去 `findroot(y)` 或 `access(y)`，就能把 `x`、`y` 所在树的结构拉到可直接判断或修改的形态上：若两点根相同则连通；连边时先判不在同一棵树里，再把一方根挂过去；断边时则要先把这条边对应的路径提到一棵辅助树上，确认它确实是一条直接相连的树边后再切开。

所以这道模板题最该学的不是函数名，而是这个动作链：**换根 -> 暴露路径 -> 在辅助树里读信息或改结构。**以后看大多数 LCT 题，核心都还是这三步。
### 核心代码
```cpp
makeroot(x);
if (findroot(y) != x) link(x, y);

makeroot(x); access(y); splay(y);
if (ch[y][0] == x && !ch[x][1]) ch[y][0] = fa[x] = 0, pushup(y);
```
### 复杂度
单次操作均摊 `O(log n)`。

## 56. [P4219 [BJOI2014] 大融合](https://www.luogu.com.cn/problem/P4219)
`Link-Cut Tree` `动态树` `边负载`
### 题意
森林中不断加边成树，并询问某条已存在边当前被多少条简单路径经过。
### 分析
先把“被多少条简单路径经过”翻译清楚。固定一条边 `e`，如果把它删掉，树会被分成大小分别为 `s` 和 `tot-s` 的两块；任取一端一个点，二者间的简单路径都会经过 `e`，反之不会，所以答案就是 `s * (tot - s)`。

真正难的是动态加边后，如何快速得到某条边两侧的点数。LCT 的常见技巧是把“边”也拆成一个点，再把它连向原来的两个端点。这样查询一条边时，只要把对应边点旋到合适位置，就能从它两边挂着的子树大小里读出被分开的两部分规模。

所以这题的关键不只是会用 LCT，而是知道为什么要**边转点**：因为题目问的是边的信息，而 LCT 更擅长维护点在动态树中的子树 / 路径信息。
### 核心代码
```cpp
makeroot(x);
access(y); splay(y);
int s = siz[x];
int tot = siz[y];
long long ans = 1LL * s * (tot - s);
```
### 复杂度
单次加边、查询均摊 `O(log n)`。
