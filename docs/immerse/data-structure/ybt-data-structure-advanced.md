# 一本通 数据结构提高专题精选解题报告

这一组题从前缀型结构一路走到树上剖分与平衡树，表面上横跨树状数组、RMQ、线段树、LCA、树链剖分和 Treap，但主线其实很统一：先把询问对象压成能维护的结构，再决定是用前缀、区间、路径还是有序集合去承接。前半段练的是“怎样把答案改写成可减的前缀或静态块”，后半段则逐步转向“怎样让动态修改仍然落在一套稳定骨架里”。

# 一、树状数组与前缀维护

这一章先把“前缀信息可加可减”这条主线走顺：有时是普通前缀和，有时是差分后的区间翻转，有时再往上长成二维统计。关键不在模板，而在先把询问改写成前缀可减的形式。

## 1. [1535：【例 1】数列操作](http://ybt.ssoier.cn:8088/problem_show.php?pid=1535)

`树状数组` `单点修改` `区间求和`

### 题意

给定一个长度为 $n$ 的数列，操作只有两类：单点加上一个值，或者查询区间 $[a,b]$ 的连续和。数据和操作次数都到 $10^5$ 级，必须在线回答。

### 分析

这题最该先想到的是：区间和如果能写成两个前缀和之差，就没有必要直接维护所有区间。于是问题被压成“支持单点增量、查询前缀和”，这正是树状数组最顺手的形态。

树状数组里 `c[x]` 存的是一段长度为 `lowbit(x)` 的块和，所以单点修改时只沿着包含它的那些块往上跳，前缀查询时则把若干块拆开累加。把 `sum(b)-sum(a-1)` 一减，区间和就出来了。

### 核心代码

```cpp
int c[N];

void add(int x,int v){
    for(;x<=n;x+=x&-x) c[x]+=v;
}
int sum(int x){
    int r=0;
    for(;x;x-=x&-x) r+=c[x];
    return r;
}
// query(a,b)=sum(b)-sum(a-1);
```

### 复杂度

单次修改、查询都是 $O(\log n)$，空间复杂度 $O(n)$。

## 2. [1536：【例 2】数星星 Stars](http://ybt.ssoier.cn:8088/problem_show.php?pid=1536)

`树状数组` `前缀计数` `二维偏序`

### 题意

每颗星星有一个坐标，若某颗星星左下方（含正左、正下）的星星有 $k$ 颗，就称它是 $k$ 级。输入已经按 $y$ 递增、同 $y$ 时按 $x$ 递增给出，要统计每个等级分别有多少颗星星。

### 分析

排序条件把二维关系直接压平了。因为输入已经保证当前读到的点，其下方和同一水平线左侧的点都已经处理过，所以“左下方有多少颗星”就只剩下一个问题：之前出现过多少个 $x\le x_i$ 的点。

于是把横坐标当成树状数组下标，每读到一个点就先查询 `sum(x_i)` 得到它的等级，再把这个横坐标出现次数加一。二维偏序在这里被转成了单维前缀计数。

### 核心代码

```cpp
for(int i=1;i<=n;i++){
    int x,y; cin>>x>>y; x++;
    int lv=sum(x);
    ans[lv]++;
    add(x,1);
}
```

### 复杂度

总时间复杂度 $O(n\log V)$，空间复杂度 $O(V)$，其中 $V$ 是横坐标范围。

## 3. [1537：【例 3】校门外的树](http://ybt.ssoier.cn:8088/problem_show.php?pid=1537)

`树状数组` `区间覆盖` `区间计数`

### 题意

道路上的每次种树操作都会在一个区间里种上一种全新的树种，询问时要统计区间 $[l,r]$ 里当前一共出现了多少种树。位置可以被反复覆盖，但任一时刻同一种树不会分裂成两段。

### 分析

既然同一种树在任意时刻都只对应一段连续区间，那么查询“有多少种树出现在 $[l,r]$”就等价于问：当前有多少个活动区间与 $[l,r]$ 相交。这个计数可以拆成两部分：左端点不超过 $r$ 的区间数，减去右端点小于 $l$ 的区间数。

所以真正的结构分工是两层：用有序区间结构维护当前被哪些整段颜色覆盖，再用两棵树状数组分别统计活动区间的左端点与右端点。新区间插入时，把被覆盖掉的旧区间从两棵树状数组里删掉，再把新区间加进去即可。

### 核心代码

```cpp
void cover(int l,int r,int id){
    auto segs=split_out(l,r);
    for(auto [L,R,col]:segs) addL(L,-1), addR(R,-1);
    insert_seg(l,r,id);
    addL(l,1); addR(r,1);
}
int ask(int l,int r){
    return preL(r)-preR(l-1);
}
```

### 复杂度

若配合有序区间拆分，单次操作均摊为 $O(k\log n)$，其中 $k$ 是本次真正被覆盖并删除的区间数。

## 4. [1538：清点人数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1538)

`树状数组` `前缀和` `在线统计`

### 题意

火车每节车厢的人数会动态上下车，主任只会不断向后走，并询问前 $m$ 节车厢一共有多少人。需要按事件顺序输出每次提问的答案。

### 分析

虽然题目强调提问位置单调递增，但真正决定做法的还是“前 $m$ 节车厢共有多少人”这句话：它本质就是一个前缀和。上车、下车都是单点增减，于是问题又被压成了树状数组模板。

单调递增的提问只是在提醒你不必回头看历史状态；而数据结构层面，维护所有车厢当前人数后，每次 `A m` 直接输出 `sum(m)` 就够了。

### 核心代码

```cpp
if(op=='B') add(pos,delta);
if(op=='C') add(pos,-delta);
if(op=='A') cout<<sum(pos)<<"\n";
```

### 复杂度

每个事件处理 $O(\log n)$，空间复杂度 $O(n)$。

## 5. [1539：简单题](http://ybt.ssoier.cn:8088/problem_show.php?pid=1539)

`树状数组` `差分` `区间翻转`

### 题意

长度为 $n$ 的 $01$ 数组初始全为 $0$。操作一把一段区间全部翻转，操作二询问某个位置当前是 $0$ 还是 $1$。

### 分析

如果直接维护原数组，区间翻转会很别扭；但换成差分视角后，一次翻转只会影响区间两端：在 $l$ 处加一，在 $r+1$ 处再加一，表示后面的前缀奇偶性发生变化。

于是某个位置最终的值，只取决于它之前一共被翻转了多少次。把差分增量放进树状数组，查询 `sum(x) % 2` 就是当前位置的当前值。

### 核心代码

```cpp
void flip(int l,int r){
    add(l,1);
    if(r+1<=n) add(r+1,1);
}
int query(int x){
    return sum(x)&1;
}
```

### 复杂度

单次翻转和查询都是 $O(\log n)$，空间复杂度 $O(n)$。

## 6. [1540：打鼹鼠_二维树状数组](http://ybt.ssoier.cn:8088/problem_show.php?pid=1540)

`二维树状数组` `单点修改` `子矩阵求和`

### 题意

给定一个初始全零的 $n\times m$ 矩阵，支持单点加值，以及询问任意子矩阵元素和。

### 分析

一维树状数组维护的是前缀和，二维版本只是把这个想法沿两个坐标轴各做一次。只要能求出二维前缀和 `S(x,y)`，任意矩形和就能用四个前缀做容斥。

所以关键仍是“把目标答案改写成前缀信息”：更新时沿着两个维度同时按 `lowbit` 往上跳，查询时则沿着两个维度往回拆。

### 核心代码

```cpp
void add(int x,int y,int v){
    for(int i=x;i<=n;i+=i&-i)
        for(int j=y;j<=m;j+=j&-j) c[i][j]+=v;
}
long long sum(int x,int y){
    long long r=0;
    for(int i=x;i;i-=i&-i)
        for(int j=y;j;j-=j&-j) r+=c[i][j];
    return r;
}
```

### 复杂度

单次修改、查询都是 $O(\log n\log m)$，空间复杂度 $O(nm)$。

# 二、静态区间查询与 RMQ 变形

当数据基本静止时，重心就从“如何修改”转到“如何把很多询问一次性吃掉”。这一章既有标准 RMQ，也有滑窗最值、最长不重复段这类把区间答案改写成预处理信息的问题。

## 7. [1541：【例 1】数列区间最大值](http://ybt.ssoier.cn:8088/problem_show.php?pid=1541)

`ST表` `RMQ` `区间最大值`

### 题意

给定静态数列，多次询问区间 $[X,Y]$ 内的最大值，数列本身不会发生修改。

### 分析

静态 RMQ 的第一反应就该是预处理而不是在线维护。因为数组不变，所以可以预先把所有长度为 $2^k$ 的区间最大值都算好，查询时再用两个重叠块把目标区间盖住。

之所以两个块就够，是因为求最大值满足幂等性：重复覆盖不会出错。于是 ST 表的本质不是“倍增模板”，而是把任意区间拆到两个固定长度块上。

### 核心代码

```cpp
for(int j=1;(1<<j)<=n;j++)
    for(int i=1;i+(1<<j)-1<=n;i++)
        st[i][j]=max(st[i][j-1],st[i+(1<<(j-1))][j-1]);
int ask(int l,int r){
    int k=lg[r-l+1];
    return max(st[l][k],st[r-(1<<k)+1][k]);
}
```

### 复杂度

预处理 $O(n\log n)$，单次查询 $O(1)$，空间复杂度 $O(n\log n)$。

## 8. [1542：【例 2】最敏捷的机器人](http://ybt.ssoier.cn:8088/problem_show.php?pid=1542)

`单调队列` `滑动窗口` `区间最值`

### 题意

对长度为 $n$ 的序列，要求依次输出每个长度为 $k$ 的连续子段中的最大值和最小值。

### 分析

这里所有询问的区间长度都相同，而且是按左端点从小到大滑过去的，所以没有必要上 RMQ。更自然的模型是滑动窗口：窗口右端加入新元素时，队尾把不可能再成为最值的元素弹掉；左端离开窗口时，再把过期下标从队首删掉。

一条维护递增、一条维护递减，最小值和最大值就都能在队首读出。真正要建立的感觉是：区间顺序固定滑动时，应该优先考虑“单调队列接管窗口”。

### 核心代码

```cpp
for(int i=1;i<=n;i++){
    while(!qmax.empty()&&a[qmax.back()]<=a[i]) qmax.pop_back();
    while(!qmin.empty()&&a[qmin.back()]>=a[i]) qmin.pop_back();
    qmax.push_back(i); qmin.push_back(i);
    while(qmax.front()<=i-k) qmax.pop_front();
    while(qmin.front()<=i-k) qmin.pop_front();
    if(i>=k) mx.push_back(a[qmax.front()]), mn.push_back(a[qmin.front()]);
}
```

### 复杂度

总时间复杂度 $O(n)$，空间复杂度 $O(k)$。

## 9. [1543：【例 3】与众不同](http://ybt.ssoier.cn:8088/problem_show.php?pid=1543)

`RMQ` `最长不重复子段` `离线预处理`

### 题意

定义连续一段数互不相同为“完美序列”。对于很多组区间 $[L,R]$，要求回答其中最长完美连续子段的长度。

### 分析

先不要直接想每个询问怎么做，而要先把“以某个位置结尾的最长不重复段”预处理出来。设 `left[i]` 是以 `i` 结尾时能延伸到的最左端，它可以用上一次相同数字出现位置在线扫出来，而且 `left[i]` 单调不减。

对一个询问 $[L,R]$，答案来自两部分：那些 `left[i]<L` 的位置，长度被截成 `i-L+1`；其余位置则可以直接用预处理的 `i-left[i]+1`。借助 `left` 的单调性二分分界点，再在右半段上做 RMQ，就把复杂询问压平了。

### 核心代码

```cpp
for(int i=1;i<=n;i++){
    left[i]=max(left[i-1],last[a[i]]+1);
    val[i]=i-left[i]+1;
    last[a[i]]=i;
}
int solve(int L,int R){
    int p=upper_bound(left+1,left+n+1,L-1)-left;
    int ans=max(0,p<=R? p-L : 0);
    if(p<=R) ans=max(ans,rmq(p,R));
    return ans;
}
```

### 复杂度

预处理 $O(n\log n)$，单次查询 $O(\log n)$，空间复杂度 $O(n\log n)$。

## 10. [1544：天才的记忆](http://ybt.ssoier.cn:8088/problem_show.php?pid=1544)

`ST表` `RMQ` `区间最大值`

### 题意

静态数组给定后会被询问很多次，每次要求立即回答区间 $[A,B]$ 内的最大数。

### 分析

这题和标准 RMQ 完全同构，重点仍是认出“数组不会变”。一旦没有修改，所有在线数据结构都显得过重，最省事的就是把长度为 $2^k$ 的块最大值全部预处理。

查询时取两段长度相同的块覆盖整个询问区间即可，因为最大值允许重叠统计。这个思路把“很多询问”转成了“一次预处理加常数时间回答”。

### 核心代码

```cpp
int ask(int l,int r){
    int k=lg[r-l+1];
    return max(st[l][k],st[r-(1<<k)+1][k]);
}
```

### 复杂度

预处理 $O(n\log n)$，单次查询 $O(1)$，空间复杂度 $O(n\log n)$。

## 11. [1545：Balanced Lineup](http://ybt.ssoier.cn:8088/problem_show.php?pid=1545)

`ST表` `RMQ` `最大最小差`

### 题意

每次给出一段连续的牛，要求输出这段里最高和最低身高的差值。数组静态不变，询问很多。

### 分析

题目并不是直接问最大值，而是问“最大值减最小值”。但这并没有改变结构本质：如果能分别快速取到区间最大值和最小值，答案就是它们的差。

因此只需建两张 ST 表，一张维护最大值，一张维护最小值。真正的建模动作，是先把复合目标拆回两个标准 RMQ。

### 核心代码

```cpp
int askMax(int l,int r){
    int k=lg[r-l+1];
    return max(mx[l][k],mx[r-(1<<k)+1][k]);
}
int askMin(int l,int r){
    int k=lg[r-l+1];
    return min(mn[l][k],mn[r-(1<<k)+1][k]);
}
```

### 复杂度

预处理 $O(n\log n)$，每次询问 $O(1)$，空间复杂度 $O(n\log n)$。

## 12. [1546：NOIP2011 选择客栈](http://ybt.ssoier.cn:8088/problem_show.php?pid=1546)

`双指针` `前缀统计` `计数`

### 题意

每家客栈有色调和咖啡最低消费。要求统计有多少对同色客栈，且它们之间至少存在一家最低消费不超过 $p$ 的咖啡店。

### 分析

难点在于把“区间里至少有一个消费不超过 $p$”翻成可在线统计的条件。顺着位置从左到右扫描时，记 `last` 为最近一个满足消费不超过 $p$ 的位置，那么对当前客栈 `i` 来说，合法的另一端 `j` 必须满足 `j\le last`，并且颜色与 `i` 相同。

于是可以让一个指针把所有位置不超过 `last` 的客栈按颜色计入桶中。扫到第 `i` 家时，答案增加的就是“颜色与它相同、且位置已经进入桶中的客栈数”。条件里那个“存在一个便宜咖啡店”，就这样被压成了一个最右合法前缀。

### 核心代码

```cpp
for(int i=1,j=0;i<=n;i++){
    if(cost[i]<=p) last=i;
    while(j<last) ++cnt[color[++j]];
    ans+=cnt[color[i]]-(i<=last);
}
```

### 复杂度

总时间复杂度 $O(n+k)$，空间复杂度 $O(k)$。

# 三、线段树的区间维护骨架

到了这里，问题开始要求在线修改、在线查询，而且维护量不再只是一维前缀。线段树的关键是先确定结点该存什么，再决定修改如何整段生效、何时才需要向下展开。

## 13. [1547：【 例 1】区间和](http://ybt.ssoier.cn:8088/problem_show.php?pid=1547)

`线段树` `单点修改` `区间和`

### 题意

初始全零序列，支持单点增加和区间求和。虽然问题本身很基础，但这一章的重点是把线段树的结点维护方式先搭起来。

### 分析

与树状数组相比，这题在线段树里更像是练“区间如何递归分治”。每个结点维护一段区间的和，单点修改只沿着包含该位置的链一路下去，回溯时重新合并左右儿子的答案。

它的价值在于把线段树最基本的三件事连起来：建树、递归下探、回收信息。后面所有复杂版本，都是在这副骨架上继续加信息与标记。

### 核心代码

```cpp
void upd(int p,int l,int r,int x,int v){
    if(l==r){ tr[p]+=v; return; }
    int m=(l+r)>>1;
    if(x<=m) upd(p<<1,l,m,x,v);
    else upd(p<<1|1,m+1,r,x,v);
    tr[p]=tr[p<<1]+tr[p<<1|1];
}
long long ask(int p,int l,int r,int L,int R){
    if(L<=l&&r<=R) return tr[p];
    int m=(l+r)>>1; long long ans=0;
    if(L<=m) ans+=ask(p<<1,l,m,L,R);
    if(R>m) ans+=ask(p<<1|1,m+1,r,L,R);
    return ans;
}
```

### 复杂度

单次修改、查询都是 $O(\log n)$，空间复杂度 $O(n)$。

## 14. [1548：【例 2】A Simple Problem with Integers](http://ybt.ssoier.cn:8088/problem_show.php?pid=1548)

`线段树` `懒标记` `区间加`

### 题意

在静态初值数组上支持区间加法和区间求和，数据范围很大，要求依次在线处理。

### 分析

一旦修改从单点扩展到整段，真正该想的是：如果一个结点代表的区间被完全覆盖，能不能直接更新它的区间和，而不立刻改到底层每个元素。对求和来说答案是可以，所以懒标记自然出现。

这里的 `tag` 记录的是“这整段都还欠孩子一个统一增量”。只要后面暂时不继续细分子区间，这个增量就可以一直挂在当前结点上。

### 核心代码

```cpp
void apply(int p,int l,int r,long long v){
    sum[p]+=v*(r-l+1);
    tag[p]+=v;
}
void push(int p,int l,int r){
```

## 16. [1550：花神游历各国](http://ybt.ssoier.cn:8088/problem_show.php?pid=1550)

`线段树` `区间开方` `最大值剪枝`

### 题意

数列支持两类操作：查询区间和，或者把一段区间里每个数都变成其整数平方根。数值非负，操作次数很多。

### 分析

区间开方最麻烦的地方在于它既不像加法，也不像赋值，没有统一的懒标记可打。真正救这题的不是合并操作，而是数值会快速稳定：当一个区间里的最大值已经不超过 $1$ 时，再开方也不会变化。

于是线段树应维护区间和与区间最大值。更新时先看 `mx<=1` 能不能整段剪掉，只有仍可能变化的区间才递归下去。这样依靠“值域收敛”而不是懒标记，把复杂操作压住了。

### 核心代码

```cpp
void change(int p,int l,int r,int L,int R){
    if(mx[p]<=1) return;
    if(l==r){ sum[p]=mx[p]=(long long)sqrt(sum[p]); return; }
    int m=(l+r)>>1;
    if(L<=m) change(p<<1,l,m,L,R);
    if(R>m) change(p<<1|1,m+1,r,L,R);
    pull(p);
}
```

### 复杂度

单次查询 $O(\log n)$，区间开方在均摊意义下接近 $O(\log n)$，空间复杂度 $O(n)$。

## 17. [1551：维护序列](http://ybt.ssoier.cn:8088/problem_show.php?pid=1551)

`线段树` `懒标记` `乘加同存`

### 题意

需要在模 $P$ 意义下维护数列，支持区间乘、区间加和区间求和三种操作。

### 分析

这题最关键的不是记两个标记，而是先认出区间上的每个元素都在做同一种仿射变换：`x -> x*mul + add`。一旦把乘法和加法合并到同一个函数里，结点和标记的更新规则就顺了。

区间和受到仿射变换后会整体变成 `sum*mul + add*len`，而儿子原有标记再叠父亲标记时，本质是函数复合。顺序写对了，这题就只剩模板实现。

### 核心代码

```cpp
void apply(int p,int l,int r,long long mul,long long add){

    tagAdd[p]=(tagAdd[p]*mul+add)%mod;
}
void push(int p,int l,int r){

    apply(p<<1,l,m,tagMul[p],tagAdd[p]);
```

每次操作都是 $O(\log n)$，空间复杂度 $O(n)$。

### 分析

遇到这类线段树题，第一步永远不是急着写模板，而是先确定结点到底要存什么：是和、最值、颜色段，还是若干量的组合。只要结点信息满足“左右儿子能合并成父亲”，骨架就已经成了。

```cpp
void pull(int p){
```

```cpp
void apply(int p,int l,int r,long long v){
```

# 四、LCA、树上距离与结构化统计

树上题真正难的通常不是写倍增，而是先看清查询到底依赖哪种树上关系：是距离、祖先、路径覆盖次数，还是若干关键点在 DFS 序里的相对位置。模型一旦翻对，后面只是在固定骨架里填答案。

## 21. [1552：【例 1】点的距离](http://ybt.ssoier.cn:8088/problem_show.php?pid=1552)

`LCA` `树上距离` `倍增`

### 题意

给定一棵树，多次询问两个点之间的距离。树无权时，距离就是路径上的边数。

### 分析

两点距离本质上依赖它们分叉前的最后一个公共祖先。只要先求出 `lca(x,y)`，答案立刻就是 `dep[x]+dep[y]-2*dep[lca]`。

所以这题的建模核心不是“树上路径”，而是“路径可以拆成两段到公共祖先的深度差”。倍增预处理祖先表后，LCA 查询就能稳定在对数时间。

### 核心代码

```cpp
int lca(int x,int y){
    if(dep[x]<dep[y]) swap(x,y);
    for(int k=LOG;k>=0;k--) if(dep[fa[x][k]]>=dep[y]) x=fa[x][k];
    if(x==y) return x;
    for(int k=LOG;k>=0;k--) if(fa[x][k]!=fa[y][k]) x=fa[x][k],y=fa[y][k];
    return fa[x][0];
}
int dist(int x,int y){ return dep[x]+dep[y]-2*dep[lca(x,y)]; }
```

### 复杂度

预处理 $O(n\log n)$，单次询问 $O(\log n)$，空间复杂度 $O(n\log n)$。

## 22. [1553：【例 2】暗的连锁](http://ybt.ssoier.cn:8088/problem_show.php?pid=1553)

`LCA` `树上差分` `统计`

### 题意

图由一棵主边构成的树和若干附加边组成。先切一条主边，再切一条附加边，要求统计总共有多少种办法能让整张图最终不连通。

### 分析

先盯住第一刀：如果切断某条主边后，仍有多少条附加边跨过这条边，就决定了第二刀还能怎么选。设一条主边被跨过的附加边数为 `cnt`，那么 `cnt=0` 时第二刀可以任意切一条附加边，贡献 `m`；`cnt=1` 时只能切那唯一一条，贡献 `1`；否则无解。

于是问题转成统计每条树边被多少条附加路径经过。每条附加边 $(u,v)$ 在树上对应一条路径，可用树上差分：`d[u]++ , d[v]++ , d[lca]-=2`，最后 DFS 汇总到边。

### 核心代码

```cpp
for(each extra edge u,v){
    int p=lca(u,v);
    d[u]++; d[v]++; d[p]-=2;
}
void dfs2(int u,int f){
    for(int v:g[u]) if(v!=f){
        dfs2(v,u);
        d[u]+=d[v];
        if(d[v]==0) ans+=m;
        else if(d[v]==1) ans+=1;
    }
}
```

### 复杂度

总时间复杂度 $O((n+m)\log n)$，空间复杂度 $O(n\log n)$。

## 23. [1554：【例 3】异象石](http://ybt.ssoier.cn:8088/problem_show.php?pid=1554)

`LCA` `DFS序` `动态集合`

### 题意

树上会动态出现或删除异象石，询问时要计算把当前所有异象石所在点连通起来所需边集的最小总长度。

### 分析

这道题真正要认出的不是最小斯坦纳树，而是一个更适合动态维护的等价量：把所有关键点按 DFS 序放进一个环，相邻点树上距离之和的一半，恰好就是连通这些点所需边集的总长度。

因此只要把所有当前存在异象石的点按 DFS 序放进有序集合，插入或删除一个点时，只会影响它与前驱、后继两段距离。每次局部修补总环长，询问时除以二即可。

### 核心代码

```cpp
void add_node(int x){
    auto it=s.insert(dfn[x]).first;
    int a=id[pre(it)], b=id[nxt(it)];
    cyc+=dist(a,x)+dist(x,b)-dist(a,b);
}
void del_node(int x){
    auto it=s.find(dfn[x]);
    int a=id[pre(it)], b=id[nxt(it)];
    cyc-=dist(a,x)+dist(x,b)-dist(a,b);
    s.erase(it);
}
```

### 复杂度

每次插入、删除、查询都是 $O(\log n)$，预处理 LCA 为 $O(n\log n)$。

## 24. [1555：【例 4】次小生成树](http://ybt.ssoier.cn:8088/problem_show.php?pid=1555)

`最小生成树` `LCA` `严格次小生成树`

### 题意

给定带权无向图，要求求出权值严格大于最小生成树，但又尽量小的那棵生成树权值。

### 分析

严格次小生成树的标准转身，是先求一棵 MST，再考虑把某条非树边加进去。这样会在树上形成一个环，为了仍然保持生成树，就必须删掉环上一条边。想让新树尽量小，就该删掉路径上尽可能大的边；但因为题目要求“严格大于 MST”，若非树边权值刚好等于路径最大边，还要继续考虑次大边。

于是倍增表里不能只存路径最大值，还要同时存次大值。枚举每条非树边时，在它连接的两点路径上取出前两大的边权，再决定能否形成合法的严格次优替换。

### 核心代码

```cpp
for(each non_tree_edge u,v,w){
    auto [mx1,mx2]=query_max2(u,v);
    if(w>mx1) ans=min(ans,mst+w-mx1);
    else if(w>mx2) ans=min(ans,mst+w-mx2);
}
```

### 复杂度

Kruskal 为 $O(m\log m)$，每条非树边查询 $O(\log n)$，总复杂度 $O(m\log m)$。

### 分析

祖先关系最适合用 DFS 序去判断。因为在一棵有根树上，某个点的整棵子树在 DFS 序中一定对应一段连续区间，所以只要知道 `tin` 和 `tout`，就能常数时间判断一个点是否落在另一个点的子树里。

### 核心代码

```cpp
bool is_anc(int x,int y){
    return tin[x]<=tin[y]&&tout[y]<=tout[x];
}
if(is_anc(x,y)) cout<<1;
else if(is_anc(y,x)) cout<<2;
else cout<<0;
```

`树链剖分` `路径查询` `点修改`

### 题意

树上每个点有权值，支持单点改值、查询两点路径上的最大值，以及查询两点路径上的权值和。

### 分析

树上路径不好直接维护，难点是它不是连续区间。树链剖分的想法就是先把树切成若干条重链，让任意一条路径都能拆成 $O(\log n)$ 段连续区间。

拆完以后，问题就退化成线段树上的区间最大值与区间和查询。真正该先建立的模型是：树路径先分段，再把每段扔回一维结构处理。

### 核心代码

```cpp
while(top[x]!=top[y]){
    if(dep[top[x]]<dep[top[y]]) swap(x,y);
    ansSum+=query_sum(dfn[top[x]],dfn[x]);
    ansMax=max(ansMax,query_max(dfn[top[x]],dfn[x]));
    x=fa[top[x]];
}
if(dep[x]>dep[y]) swap(x,y);
ansSum+=query_sum(dfn[x],dfn[y]);
ansMax=max(ansMax,query_max(dfn[x],dfn[y]));
```

### 复杂度

单次修改、查询都是 $O(\log^2 n)$，空间复杂度 $O(n)$。

## 30. [1561：「HAOI2015」树上操作](http://ybt.ssoier.cn:8088/problem_show.php?pid=1561)

### 题意

### 分析

这题同时出现了子树和根路径两个对象，它们一个在 DFS 序里天然连续，一个适合树链剖分拆段，所以最自然的统一办法是：先做树链剖分，再在线段树上同时支持区间加和区间求和。

子树加就是 DFS 序上一段连续区间加；根到点的路径和则是树链剖分标准路径查询。看似三种操作不统一，其实都落回了“一维区间加 + 区间和”的骨架。

```cpp
void add_subtree(int x,int v){
```

## 31. [1562：「NOI2015」软件包管理器](http://ybt.ssoier.cn:8088/problem_show.php?pid=1562)

`树链剖分` `路径赋值` `子树赋值`

### 题意

软件包依赖关系构成一棵树。安装某个包时要把根到该点路径上所有包都设为已安装，卸载某个包时要把它整棵子树都设为未安装，并输出状态真正发生变化的包数。

### 分析

依赖关系天然是一棵根树：安装一个包要把祖先全部点亮，卸载一个包要把后代全部熄灭。于是操作对象分别是“根到点路径”和“某点子树”，这正适合树链剖分加线段树。

线段树里维护区间已安装包数量，再支持区间赋值为 $0/1$。安装时把若干条链赋成 $1$，卸载时把一个 DFS 序连续段赋成 $0$，赋值前后的计数差就是答案。

### 核心代码

```cpp
int install(int x){
    int old=query_path(1,x);
    assign_path(1,x,1);
    return query_path(1,x)-old;
}
int uninstall(int x){
    int old=query_sum(dfn[x],dfn[x]+siz[x]-1);
    assign_seg(dfn[x],dfn[x]+siz[x]-1,0);
    return old;
}
```

## 32. [1563：染色](http://ybt.ssoier.cn:8088/problem_show.php?pid=1563)

`树链剖分` `路径染色` `颜色段`

### 题意

树上支持把两点路径上的所有节点染成同一种颜色，并询问两点路径上的颜色段数量。相邻同色节点算同一段。

### 分析

这里路径上的答案已经不是简单求和，而是带边界效应的“颜色段数”。所以线段树结点除了段数，还得维护区间左端颜色和右端颜色，合并时若左右端颜色相同就把段数减一。

树链剖分仍负责把路径拆成若干一维区间，但由于路径可能从两头向中间拼，需要分别收集两边的信息，最后再按方向合并。模型核心是：答案不仅依赖内部统计，还依赖区间边界。

### 核心代码

```cpp
Node merge(Node a,Node b){
    Node c;
    c.lc=a.lc; c.rc=b.rc;
    c.seg=a.seg+b.seg-(a.rc==b.lc);
    return c;
}
// 路径拆链后按方向依次 merge
```

### 复杂度

每次染色与查询都是 $O(\log^2 n)$，空间复杂度 $O(n)$。

### 分析

## 34. [1565：【例 1】营业额统计](http://ybt.ssoier.cn:8088/problem_show.php?pid=1565)

`Treap` `前驱后继` `最小差值`

### 题意

顺序读入每天营业额。第 $i$ 天的波动值定义为它与此前某一天营业额差值绝对值的最小值，第一天的波动值等于自身，要求求出总和。

### 分析

对当前营业额 `x` 来说，最接近它的历史值一定只可能是它在有序集合里的前驱或后继，不可能来自更远的位置。所以每天只要把 `x` 插进一棵支持前驱后继查询的平衡树里，就能立刻得到当天贡献。

这题的建模关键，是把“和所有历史值比较”缩成“只和相邻两个候选比较”。一旦有序性被抓住，复杂度就从线性比对降到了对数级。

### 核心代码

```cpp
insert(rt,x);
int a=pre(rt,x), b=suc(rt,x);
if(i==1) ans+=x;
else ans+=min(abs(x-a),abs(b-x));
```

### 复杂度

每次插入与查询都是 $O(\log n)$，总时间复杂度 $O(n\log n)$。

## 35. [1566：宠物收养所](http://ybt.ssoier.cn:8088/problem_show.php?pid=1566)

`Treap` `前驱后继` `贪心匹配`

### 题意

收养所里会交替到来宠物和领养者。若当前两边人数不平衡，新来的对象会与等待集合里特点值最接近的一方立即匹配，并累加不满意度。

### 分析

题目虽然故事很长，但真正的动作只有一个：在当前等待集合中找与新值最接近的数，若左右距离相同取较小者。这正是前驱、后继的经典用法。

所以只要维护“当前人数较多的一方”的特点值有序集合。新来的是同一类就插入；新来的是另一类，就查前驱后继，选更近的那个匹配并删除。贪心之所以成立，是因为题目规定每次都立即按最近原则配对。

### 核心代码

```cpp
int match(int x){
    int a=pre(rt,x), b=suc(rt,x);
    int y=(x-a<=b-x)?a:b;
    erase(rt,y);
    return abs(x-y);
}
```

### 复杂度

每次事件 $O(\log n)$，总时间复杂度 $O(n\log n)$。

## 36. [1567：郁闷的出纳员](http://ybt.ssoier.cn:8088/problem_show.php?pid=1567)

`Treap` `全局偏移` `第k大`

### 题意

员工工资支持插入、整体加、整体减，工资低于下界的人会立刻离开；还要回答当前第 $k$ 高工资是多少，并统计总共离开了多少人。

### 分析

如果每次整体加减都真的改所有节点，复杂度会直接炸掉。真正该想到的是给整棵树挂一个全局偏移 `delta`：树里保存“真实工资减去当前偏移”的值，整体涨跌工资只改 `delta`。

### 核心代码

```cpp
if(op=='A') delta+=x;
if(op=='S'){
    delta-=x;
    split(rt,limit-delta-1,a,b);
    removed+=siz[a]; rt=b;
}
if(op=='F') cout<<(k>siz[rt]?-1:kth(rt,siz[rt]-k+1)+delta)<<"\n";
```

## 37. [1568：普通平衡树](http://ybt.ssoier.cn:8088/problem_show.php?pid=1568)

`Treap` `排名` `前驱后继`

### 题意

维护一个可重集合，支持插入、删除、查询某值排名、查询某排名的值、查询前驱、查询后继六种标准操作。

### 分析

这题就是把平衡树最核心的一套接口一次性练全。Treap 的好处在于既保持二叉搜索树的有序性，又用随机优先级维护近似平衡，所以所有操作都能递归地落在左右子树里。

真正需要形成的统一视角是：排名来自左子树大小，前驱后继来自二叉搜索树的顺序性，删除则是把目标节点旋到容易删的位置。六个操作本质上都围绕这两类信息展开。

### 核心代码

```cpp
int rank(int p,int x){
    if(!p) return 1;
    if(x<=tr[p].val) return rank(tr[p].l,x);
    return siz[tr[p].l]+tr[p].cnt+rank(tr[p].r,x);
}
int kth(int p,int k){
    if(k<=siz[tr[p].l]) return kth(tr[p].l,k);
    if(k<=siz[tr[p].l]+tr[p].cnt) return tr[p].val;
    return kth(tr[p].r,k-siz[tr[p].l]-tr[p].cnt);
}
```

### 复杂度

各项操作期望时间复杂度均为 $O(\log n)$，空间复杂度 $O(n)$。
