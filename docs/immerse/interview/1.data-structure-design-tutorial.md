## 一、概述

面试中的数据结构设计题，本质是：**根据场景需求，选择或组合数据结构，满足特定的复杂度约束**。

本文按难度从低到高，覆盖 5 类面试高频结构：

| 难度 | 数据结构 | 典型场景 | 核心操作复杂度 |
|------|---------|---------|--------------|
| ⭐ | MinStack | O(1) 查询栈的最小值 | 全部 O(1) |
| ⭐⭐ | Heap | 求第 K 大、数据流中位数 | push/pop O(log n) |
| ⭐⭐ | Union-Find | 朋友圈、判环、最小生成树 | 近 O(1) |
| ⭐⭐⭐ | Trie + AutoComplete | 前缀匹配、搜索自动补全 | O(L) |
| ⭐⭐⭐⭐ | LRU Cache | 缓存淘汰策略 | get/put O(1) |

## 二、MinStack / MinQueue — O(1) 查询极值 ⭐

### 2.1 MinStack

**场景**：设计一个栈，支持 O(1) 的 push、pop、top 和 getMin（LeetCode 155）。

**核心思想**：每个栈帧绑定"截至当前的最小值"，pop 时最小值自然恢复。

```
push(3): [(3, min=3)]
push(5): [(3,3), (5,3)]          ← min 不变
push(2): [(3,3), (5,3), (2,2)]   ← min 更新为 2
pop():   [(3,3), (5,3)]          ← min 自动恢复为 3
```

```cpp
class MinStack {
    vector<pair<int,int>> stk; // {val, curMin}
public:
    void push(int val) {
        int curMin = stk.empty() ? val : min(val, stk.back().second);
        stk.push_back({val, curMin});
    }
    void pop() { stk.pop_back(); }
    int top() { return stk.back().first; }
    int getMin() { return stk.back().second; }
};
```

所有操作 O(1)。思路简单但面试出现率很高。

**写法二：双栈写法**（更经典，面试中更常见）：

```cpp
class MinStack {
    stack<int> s1;      // 数据栈
    stack<int> s2;      // 最小值栈，栈顶始终是当前最小值
public:
    void push(int val) {
        s1.push(val);
        s2.push(s2.empty() ? val : min(val, s2.top()));
    }
    void pop() { s1.pop(); s2.pop(); }
    int top() { return s1.top(); }
    int getMin() { return s2.top(); }
};
```

两种写法本质一样：pair 写法把值和最小值绑在一起，双栈写法拆成两个栈同步操作。面试中双栈写法更直观，容易讲清楚。

### 2.2 MinQueue（单调队列）

**场景**：滑动窗口最大值（LeetCode 239）——给定数组和窗口大小 K，求每个窗口的最大值。

**核心思想**：队列两端都要操作，不能像栈那样绑定最小值。用一个**单调 deque** 维护"有可能成为极值"的候选元素。

```
为什么叫"单调"？
  deque 中元素从前到后严格递增（最小队列）或递减（最大队列）
  新元素入队时，把 deque 尾部所有不如它的踢掉
  → 它们比新元素差且更早出队，永远不可能成为答案
```

```cpp
// 滑动窗口最大值
vector<int> maxSlidingWindow(vector<int>& nums, int k) {
    deque<int> dq; // 存下标，对应值单调递减
    vector<int> res;
    for (int i = 0; i < (int)nums.size(); i++) {
        if (!dq.empty() && dq.front() <= i - k) dq.pop_front();
        while (!dq.empty() && nums[dq.back()] <= nums[i]) dq.pop_back();
        dq.push_back(i);
        if (i >= k - 1) res.push_back(nums[dq.front()]);
    }
    return res;
}
```

每个元素最多入 deque 一次、出一次，整体 O(n)。

### 2.3 MinStack vs MinQueue 对比

| 维度 | MinStack | MinQueue |
|------|----------|----------|
| 核心差异 | 栈只从一端操作，最小值随栈帧自然维护 | 队列两端操作，需要单调 deque 追踪候选值 |
| 面试题 | LeetCode 155 | LeetCode 239, 907 |

**面试延伸**：用两个 MinStack 可以实现 MinQueue。

### 2.4 用两个 MinStack 实现 MinQueue

这是 LeetCode 232（栈实现队列）和 LeetCode 155（最小栈）的经典组合题。

**核心思想**：两个 MinStack 分别充当"入口"和"出口"，倒腾一次就把栈的 LIFO 变成了队列的 FIFO，而各自的 getMin 能力不受影响。

```
push(1,2,3):  in=[1,2,3]   out=[]
pop():        out 空 → 把 in 全倒进 out
              in=[]         out=[3,2,1]  ← 顺序反转，队头在栈顶
              pop out → 返回 1 ✓（FIFO）

getMin：min(in.getMin(), out.getMin())
  两个 MinStack 各自维护自己那半的最小值，取较小的即可
```

```cpp
class MinQueue {
    MinStack in, out;  // 用前面任意一种 MinStack 写法

    void transfer() {
        if (out.empty())  // 只在 out 空时倒腾，保证均摊 O(1)
            while (!in.empty()) {
                out.push(in.top());
                in.pop();
            }
    }
public:
    void push(int val) { in.push(val); }

    void pop() { transfer(); out.pop(); }

    int front() { transfer(); return out.top(); }

    int getMin() {
        if (in.empty()) return out.getMin();
        if (out.empty()) return in.getMin();
        return min(in.getMin(), out.getMin());
    }
};
```

**复杂度**：push O(1)，pop 均摊 O(1)（每个元素最多倒腾一次），getMin O(1)。

**面试时怎么讲清楚这个思路**：
1. 先说"栈实现队列"的倒腾原理（LeetCode 232）
2. 再说"如果两个栈都是 MinStack，那倒腾之后各自的 getMin 依然有效"
3. 整体的 getMin 就是两半取 min

## 三、Heap — 动态维护极值 ⭐⭐

### 3.1 场景

- **求第 K 大**：维护大小为 K 的小顶堆，堆顶即第 K 大（LeetCode 215）
- **数据流中位数**：对顶堆（LeetCode 295）
- **合并 K 个有序链表**：堆做多路归并（LeetCode 23）

### 3.2 核心实现

堆的本质：用数组存完全二叉树，通过 up（上浮）和 down（下沉）维护堆性质。

```
parent(i) = (i-1) / 2
left(i)   = 2*i + 1
right(i)  = 2*i + 2
```

```cpp
// 小顶堆（改 > 为 < 就是大顶堆）
class Heap {
    vector<int> data;
    void up(int i) {
        int p = (i - 1) / 2;
        if (i > 0 && data[i] < data[p]) { swap(data[i], data[p]); up(p); }
    }
    void down(int i) {
        int best = i, l = 2*i+1, r = 2*i+2, n = data.size();
        if (l < n && data[l] < data[best]) best = l;
        if (r < n && data[r] < data[best]) best = r;
        if (best != i) { swap(data[i], data[best]); down(best); }
    }
public:
    void push(int v) { data.push_back(v); up(data.size()-1); }
    void pop() { swap(data[0], data.back()); data.pop_back(); down(0); }
    int top() { return data[0]; }
    int size() { return data.size(); }
    bool empty() { return data.empty(); }
    // O(n) 建堆
    void build(vector<int> arr) {
        data = move(arr);
        for (int i = data.size()/2-1; i >= 0; i--) down(i);
    }
};
```

| 操作 | 复杂度 | 说明 |
|------|--------|------|
| push / pop | O(log n) | |
| top | O(1) | |
| build | O(n) | 自底向上，$\sum 2^k(h-k) \approx 2n$ |

### 3.3 面试实战：求第 K 大

两种方案，面试必须能对比讲清楚：

| 方案 | 复杂度 | 适用场景 |
|------|--------|---------|
| 小顶堆（size=K） | O(n log K) | 数据流、在线、多次查询 |
| Quick Select | 平均 O(n)，最坏 O(n²) | 离线、一次性查询 |

```cpp
// 堆解法（小顶堆，直接用上面的 Heap）
int findKthLargest(vector<int>& nums, int k) {
    Heap h;
    for (int x : nums) {
        h.push(x);
        if (h.size() > k) h.pop();
    }
    return h.top();
}

// Quick Select
int quickSelect(vector<int>& a, int l, int r, int k) {
    int pivot = a[l + rand() % (r-l+1)];
    int i = l, j = r;
    while (i <= j) {
        while (a[i] > pivot) i++;
        while (a[j] < pivot) j--;
        if (i <= j) { swap(a[i], a[j]); i++; j--; }
    }
    if (k <= j-l+1) return quickSelect(a, l, j, k);
    return quickSelect(a, i, r, k-(i-l));
}
```

### 3.4 面试实战：对顶堆求中位数

大顶堆存较小一半，小顶堆存较大一半，堆顶对接处就是中位数。

```cpp
// 小顶堆存较大一半（上面的 Heap 就是小顶堆）
// 大顶堆存较小一半（把比较符 < 改成 > 即可）
Heap lo; // 大顶堆（改 < 为 >）：左半，堆顶是左半最大
Heap hi; // 小顶堆：右半，堆顶是右半最小

void addNum(int num) {
    lo.push(num);
    hi.push(lo.top()); lo.pop(); // 保证左 ≤ 右
    if (lo.size() < hi.size()) { lo.push(hi.top()); hi.pop(); }
}
double findMedian() {
    return lo.size() > hi.size() ? lo.top() : (lo.top()+hi.top()) / 2.0;
}
```

## 四、Union-Find — 动态连通性 ⭐⭐

### 4.1 场景

- **社交网络**：判断两人是否在同一个朋友圈（LeetCode 547）
- **无向图判环**：加边时两端已连通 = 有环
- **最小生成树**：Kruskal 算法 = 排序 + 并查集

### 4.2 核心实现

两个优化缺一不可：**路径压缩** + **按秩合并** → 均摊 $O(\alpha(n)) \approx O(1)$

```
路径压缩：find 时把沿途节点直接挂到根
    1              1
    |            / | \
    2    →     2   3   5
    |          |
    3          4
    |
    4 → 5
```

```cpp
class UnionFind {
    vector<int> fa, rank;
public:
    UnionFind(int n) : fa(n), rank(n, 0) { iota(fa.begin(), fa.end(), 0); }
    int find(int x) { return fa[x] == x ? x : (fa[x] = find(fa[x])); }
    void unite(int x, int y) {
        int rx = find(x), ry = find(y);
        if (rx == ry) return;
        if (rank[rx] < rank[ry]) swap(rx, ry);
        fa[ry] = rx;
        if (rank[rx] == rank[ry]) rank[rx]++;
    }
    bool connected(int x, int y) { return find(x) == find(y); }
};
```

### 4.3 面试实战

**朋友圈数量**（LeetCode 547）：

```cpp
int findCircleNum(vector<vector<int>>& g) {
    int n = g.size();
    UnionFind uf(n);
    for (int i = 0; i < n; i++)
        for (int j = i+1; j < n; j++)
            if (g[i][j]) uf.unite(i, j);
    int cnt = 0;
    for (int i = 0; i < n; i++) if (uf.find(i) == i) cnt++;
    return cnt;
}
```

**无向图判环**：

```cpp
bool hasCycle(int n, vector<pair<int,int>>& edges) {
    UnionFind uf(n);
    for (auto& [u, v] : edges) {
        if (uf.connected(u, v)) return true;
        uf.unite(u, v);
    }
    return false;
}
```

**Kruskal 最小生成树**：边按权排序，依次加入不产生环的边。

```cpp
int kruskal(int n, vector<tuple<int,int,int>>& edges) {
    sort(edges.begin(), edges.end());
    UnionFind uf(n);
    int total = 0, cnt = 0;
    for (auto& [w, u, v] : edges) {
        if (!uf.connected(u, v)) {
            uf.unite(u, v);
            total += w;
            if (++cnt == n-1) break;
        }
    }
    return total;
}
```

**面试追问：并查集不能做什么？**
- 不能删边（只能合并，不能分裂）
- 不能处理有向图连通性（需要 Tarjan SCC）
- 不能遍历同一集合的所有成员

## 五、Trie — 前缀匹配与自动补全 ⭐⭐⭐

### 5.1 场景

**搜索引擎 AutoComplete**：用户输入 "he"，推荐 "hello"、"heaven"、"heavy"。

```
为什么不用 HashMap？
  精确查找 O(1)，但前缀查询必须遍历所有 key → O(N)
  Trie 沿 h→e 走两步到前缀节点，DFS 收集 → O(L + 结果数)
```

### 5.2 核心实现

静态数组写法（比指针写法更快、更省心），直接内置 AutoComplete 能力：

```cpp
class Trie {
    static const int MAXN = 1e5 + 5;
    int ch[MAXN][26], cnt[MAXN], idx = 0;

    void dfs(int u, string& cur, vector<string>& res, int limit) {
        if ((int)res.size() >= limit) return;
        if (cnt[u] > 0) res.push_back(cur);
        for (int c = 0; c < 26; c++) {
            if (ch[u][c]) {
                cur.push_back('a' + c);
                dfs(ch[u][c], cur, res, limit);
                cur.pop_back();
            }
        }
    }
public:
    Trie() { memset(ch, 0, sizeof(ch)); memset(cnt, 0, sizeof(cnt)); }

    void insert(const string& s) {
        int u = 0;
        for (char c : s) {
            int v = c - 'a';
            if (!ch[u][v]) ch[u][v] = ++idx;
            u = ch[u][v];
        }
        cnt[u]++;
    }
    bool search(const string& s) {
        int u = 0;
        for (char c : s) {
            int v = c - 'a';
            if (!ch[u][v]) return false;
            u = ch[u][v];
        }
        return cnt[u] > 0;
    }
    bool startsWith(const string& s) {
        int u = 0;
        for (char c : s) {
            int v = c - 'a';
            if (!ch[u][v]) return false;
            u = ch[u][v];
        }
        return true;
    }
    // AutoComplete：沿前缀走到节点，DFS 收集完整单词
    vector<string> suggest(const string& prefix, int limit = 10) {
        int u = 0;
        for (char c : prefix) {
            int v = c - 'a';
            if (!ch[u][v]) return {};
            u = ch[u][v];
        }
        string cur = prefix;
        vector<string> res;
        dfs(u, cur, res, limit);
        return res;
    }
};
```

### 5.3 AutoComplete 工作流程

定位到前缀节点后，DFS 按字典序收集所有完整单词：

```
         root
          |
         h
          |
         e  ← suggest("he") 走到这里
        / \
       a   l
       |   |
       v   l ← cnt>0 → "hell" ✓
      / \   |
     e   y  o ← cnt>0 → "hello" ✓
     |   |
     n   ← cnt>0 → "heavy" ✓
     |
     ← cnt>0 → "heaven" ✓

DFS 字典序 → ["heaven", "heavy", "hell", "hello"]
```

### 5.4 面试延伸

| 变体 | 思路 |
|------|------|
| 支持通配符 `.` 搜索（LeetCode 211） | 遇到 `.` 时 DFS 遍历所有 26 个子节点 |
| 按热度排序的 AutoComplete | cnt 存搜索频次，DFS 时用堆取 Top-K |
| 大规模数据 | 分布式 Trie / 倒排索引 |

## 六、LRU Cache — O(1) 缓存淘汰 ⭐⭐⭐⭐

### 6.1 场景

设计一个缓存，支持 O(1) 的 get 和 put，容量满时自动淘汰最久未使用的（LeetCode 146）。

```
需要两种能力：
  O(1) 按 key 查找      → 哈希表
  O(1) 维护使用顺序      → 双向链表
  组合 → 哈希表存 key → 链表迭代器
```

### 6.2 核心实现

精髓在于 `list::splice`——O(1) 把链表节点从原位置搬到头部，无拷贝、无内存分配。

```cpp
template <typename K, typename V>
class LRUCache {
    int cap;
    list<pair<K, V>> cache;
    unordered_map<K, typename list<pair<K,V>>::iterator> mp;
public:
    LRUCache(int capacity) : cap(capacity) {}

    V get(const K& key) {
        auto it = mp.find(key);
        if (it == mp.end()) return V(-1); // or throw
        cache.splice(cache.begin(), cache, it->second); // 移到头部
        return it->second->second;
    }

    void put(const K& key, const V& val) {
        auto it = mp.find(key);
        if (it != mp.end()) {
            cache.splice(cache.begin(), cache, it->second);
            it->second->second = val;
            return;
        }
        if ((int)mp.size() >= cap) {
            mp.erase(cache.back().first); // 淘汰尾部
            cache.pop_back();
        }
        cache.emplace_front(key, val);
        mp[key] = cache.begin();
    }
};
```

### 6.3 数据流图解

```
capacity = 3

put(A,1)  →  [A:1]                    mp: {A}
put(B,2)  →  [B:2] ⇄ [A:1]           mp: {A,B}
put(C,3)  →  [C:3] ⇄ [B:2] ⇄ [A:1]  mp: {A,B,C}

get(A)    →  [A:1] ⇄ [C:3] ⇄ [B:2]  ← splice 移到头部

put(D,4)  →  淘汰尾部 B
           →  [D:4] ⇄ [A:1] ⇄ [C:3]  mp: {A,C,D}
```

### 6.4 面试追问

**为什么选 list 而不是 vector？**
→ vector 中间删除 O(n)，list 通过迭代器 O(1) 删除

**为什么 map 存 iterator？**
→ 需要 O(1) 定位链表节点才能 O(1) splice

**LRU → LFU 怎么改？**（LeetCode 460）

LFU = Least **Frequently** Used，淘汰**使用次数最少**的；如果次数相同，再淘汰最久未使用的（退化为 LRU）。

```
LRU vs LFU 的区别：
  LRU：淘汰"最久没访问的"    → 只看时间
  LFU：淘汰"访问次数最少的"  → 先看频率，频率相同再看时间

例：容量 2，操作序列 put(A), put(B), get(A), get(A), get(B), put(C)
  LRU 淘汰 A（最久没 put/get 的）
  LFU 淘汰 B（freq=1），A 被访问了 2 次不会被淘汰
```

```
关键设计：按频率分桶，每桶内按时间排序
  kv:      key → {value, freq}
  freq:    freq → list<key>（同频率的 key，尾部最旧）
  pos:     key → 在 list 中的迭代器
  minFreq: 当前最小频率（O(1) 定位淘汰桶）
```

```cpp
template <typename K, typename V>
class LFUCache {
    int cap, minFreq;
    unordered_map<K, pair<V,int>> kv;                    // key → {value, freq}
    unordered_map<int, list<K>> freq;                     // freq → list<key>（尾部最旧）
    unordered_map<K, typename list<K>::iterator> pos;     // key → 在 freq list 中的位置

    void touch(const K& key) {
        int f = kv[key].second;
        freq[f].erase(pos[key]);
        if (freq[f].empty()) {
            freq.erase(f);
            if (minFreq == f) minFreq++;
        }
        kv[key].second = f + 1;
        freq[f+1].push_front(key);
        pos[key] = freq[f+1].begin();
    }
public:
    LFUCache(int capacity) : cap(capacity), minFreq(0) {}

    V get(const K& key) {
        if (!kv.count(key)) return V(-1);
        touch(key);
        return kv[key].first;
    }

    void put(const K& key, const V& value) {
        if (cap <= 0) return;
        if (kv.count(key)) {
            kv[key].first = value;
            touch(key);
            return;
        }
        if ((int)kv.size() >= cap) {
            K evict = freq[minFreq].back();  // 最低频率中最旧的
            freq[minFreq].pop_back();
            if (freq[minFreq].empty()) freq.erase(minFreq);
            kv.erase(evict);
            pos.erase(evict);
        }
        kv[key] = {value, 1};
        freq[1].push_front(key);
        pos[key] = freq[1].begin();
        minFreq = 1;  // 新插入的频率一定是 1
    }
};
```

核心是 `touch` 函数：把 key 从旧频率桶移到新频率桶，同时维护 `minFreq`。新插入时 `minFreq` 一定重置为 1。

**系统设计追问**：
- 分布式 LRU → 一致性哈希分片
- 缓存穿透 → 布隆过滤器
- 缓存雪崩 → 过期时间加随机抖动

---

## 七、总结

### 7.1 选型指南

```
需要 O(1) 查询栈/队列极值？    → MinStack / 单调队列
需要动态 Top-K / 极值？        → Heap
需要动态合并+连通性判断？       → Union-Find
需要前缀匹配 / 自动补全？      → Trie
需要 O(1) 读写 + 淘汰策略？    → LRU Cache（链表 + 哈希表）
```

### 7.2 核心设计原则

1. **从需求推导结构**：先明确操作和复杂度要求，再选择数据结构
2. **组合互补**：LRU = 链表 + 哈希表，对顶堆 = 大顶堆 + 小顶堆，单一结构往往不够
3. **善用 STL**：`splice`、`deque`、`unordered_map` 是面试利器，理解原理的前提下用 STL 更加分

### 7.3 面试速查表

| 面试问题 | 数据结构 | 关键思路 |
|---------|---------|---------|
| O(1) 最小栈 | MinStack | 栈帧绑定当前最小值 |
| 滑动窗口最值 | 单调 deque | 候选值单调性维护 |
| 求第 K 大 | Heap / Quick Select | 堆适合流式，QS 适合一次性 |
| 数据流中位数 | 对顶堆 | 大顶堆 + 小顶堆，堆顶对接 |
| 朋友圈 / 连通分量 | Union-Find | 路径压缩 + 按秩合并 |
| 无向图判环 | Union-Find | 两端已连通 + 加边 = 环 |
| 最小生成树 | Union-Find + 排序 | Kruskal |
| 前缀匹配 / AutoComplete | Trie + DFS | 定位前缀节点后 DFS 收集 |
| 通配符搜索 | Trie + 回溯 | `.` 遍历所有子节点 |
| 设计 LRU Cache | list + HashMap | splice O(1) 移动 |
| 设计 LFU Cache | 分桶链表 + HashMap | minFreq 追踪 |
