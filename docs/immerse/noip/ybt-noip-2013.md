# 一本通 NOIP 2013赛题解题报告

这一组题从普及组的模拟、前缀最优和简单图建模，一路走到提高组的排序逆序对、最大生成树、贪心构造与状态最短路。读的时候可以始终盯住同一条主线：题目条件到底在限制什么，它应该被压成怎样的状态、不变量或转移，最后答案又该从哪里提取。

# 一、普及组赛题

这一部分先练最基础也最关键的能力：把题面条件翻成可计算的结构。四道题分别对应数位拆分、线性扫描、前缀最优和偏序建模，难度不高，但每道都很适合练“先看限制，再定状态”。

## 1. [【13NOIP普及组】计数问题](http://ybt.ssoier.cn:8088/problem_show.php?pid=1961)
`NOIP 2013` `普及组`

### 题意

给定 `1` 到 `n` 的所有整数，统计某个数字 `x` 在十进制表示里一共出现多少次。题目只问总次数，不要求逐个数展开。

数据范围已经大到不适合直接枚举，所以关键在于把“出现次数”拆到每一位上分别计算。

### 分析

一看到“统计 `1..n` 中某个数字出现多少次”，就应该把注意力放到十进制位上。个位、十位、百位的贡献彼此独立：只要知道某一位左边有多少完整循环、当前位是什么、右边还剩多少尾巴，这一位对答案的贡献就能一次算出来。于是把 `n` 在某个权值 `base` 上拆成 `high`、`cur`、`low` 三部分。

对 `x\neq0`，这一位每经过一个长度为 `10\times base` 的完整循环，数字 `x` 会连续出现 `base` 次，所以完整循环贡献是 `high\times base`。剩下的不完整部分里，如果 `cur>x`，说明这一整段 `base` 都出现了；如果 `cur=x`，只多出 `low+1` 次；如果 `cur<x`，就没有额外贡献。

数字 `0` 要单独处理，因为最高位不能出现前导零。做法是把完整循环从 `high\times base` 改成 `(high-1)\times base`，等价于先扣掉那些“最高位取零”的非法情况，再按 `cur` 和 `low` 讨论尾巴。把所有位的贡献累加起来就是答案，这里的不变量就是“每一位都只按自己的循环节结算，互不干扰”。

### 核心代码

```cpp
long long solve(long long n, int x){
  long long ans = 0;
  for(long long base = 1; base <= n; base *= 10){
    long long high = n / (base * 10), cur = n / base % 10, low = n % base;
    if(x){
      ans += high * base;
      if(cur > x) ans += base;
      else if(cur == x) ans += low + 1;
    }else if(high){
      ans += (high - 1) * base;
      if(cur) ans += base;
      else ans += low + 1;
    }
  }
  return ans;
}
```

### 复杂度

时间复杂度 $O(\log_{10} n)$，空间复杂度 $O(1)$。

## 2. [【13NOIP普及组】表达式求值](http://ybt.ssoier.cn:8088/problem_show.php?pid=1962)
`NOIP 2013` `普及组`

### 题意

表达式里只有非负整数、加号和乘号，没有括号。要求按照正常优先级算值，并且最后只保留末四位。

真正困难的不在高精度，而在于怎样在线扫描时既尊重乘法优先级，又把结果始终控制在模 `10000` 之内。

### 分析

题目条件已经把模型说得很清楚：没有括号，所以运算顺序只由“先乘后加”决定。那就没必要真的建语法树，只要把整个式子看成“若干乘法块的和”即可。比如 `2*3+4*5*6+7`，本质上就是三段乘积再相加。

顺着字符串扫一遍，维护两个状态：`mul` 表示当前这个乘法块已经算到哪里，`sum` 表示前面所有完整加法块的和。每读完一个数，就根据前一个运算符决定它该并入 `mul` 还是另起一段：如果前一个符号是 `*`，就把它乘进当前块；如果前一个符号是 `+`，说明前一块已经结束，先把 `mul` 加进 `sum`，再用当前数重开一个乘法块。

因为加法和乘法都满足取模分配律，所以整个过程中始终对 `10000` 取模不会影响末四位答案。扫描结束时，最后一个乘法块还没并到 `sum`，因此答案是 `(sum+mul)\bmod 10000`。这里的关键不变量就是：任何时刻，`sum+mul` 都等于已经读入前缀的真实值模 `10000`。

### 核心代码

```cpp
const int MOD = 10000;
int sum = 0, mul = 1, num = 0;
char last = '*';
for(char c : s + "+"){
  if(isdigit(c)) num = (num * 10 + c - '0') % MOD;
  else{
    if(last == '*') mul = mul * num % MOD;
    else sum = (sum + mul) % MOD, mul = num;
    last = c; num = 0;
  }
}
cout << (sum + mul) % MOD;
```

### 复杂度

时间复杂度 $O(|s|)$，空间复杂度 $O(1)$。

## 3. [【13NOIP普及组】小朋友的数字](http://ybt.ssoier.cn:8088/problem_show.php?pid=1963)
`NOIP 2013` `普及组`

### 题意

先定义每个小朋友的特征值：在他之前的前缀里，任选一段连续区间，最大和是多少。再定义分数：第一个人的分数等于自己的特征值，其余人的分数来自某个前面小朋友的“分数加特征值”。

最后要找最大分数，并按题意保留符号后对 `p` 取模。题面很绕，但只要把两个定义拆开，都是线性扫描能维护的前缀最优。

### 分析

先看特征值。题目说的是“在前面若干个小朋友中连续选一段”，这就是典型最大子段和，只不过要求的是每个前缀各自的答案。于是用一遍 Kadane：`end` 维护“必须以当前人结尾”的最大子段和，`bestSeg` 维护当前前缀里的最大子段和，那么第 `i` 个人的特征值 `c_i` 就是这个 `bestSeg`。

再看分数定义。对第 `i` 个人，题目要在所有前面的人里选一个 `j`，让“那个小朋友的分数加上那个小朋友的特征值”最大，所以 `f_i=\max_{j<i}(f_j+c_j)`，并不是把当前人的 `c_i` 加进去。这样一来，`f_i` 只取决于前缀最优值 `best=\max_{j<i}(f_j+c_j)`，于是扫描到 `i` 时直接令 `f_i=best`，再用当前人的 `f_i+c_i` 去更新 `best`。

整个过程的关键状态只有四个：`end`、`c_i`、`best`、`ans`。`end` 负责子段和转移，`c_i` 是当前前缀特征值，`best` 表示所有合法前驱里的最优“分数+特征值”，`ans` 记录出现过的最大分数。最后答案就是 `ans`；由于数值可能到 $10^{20}$ 量级，用 `__int128` 做比较即可，再输出 `ans\bmod p`，负数在 C++ 里天然保留负号。

### 核心代码

```cpp
using i128 = __int128_t;
i128 end = a[1], c = a[1], f = a[1], best = f + c, ans = f;
for(int i = 2; i <= n; i++){
  end = max<i128>(a[i], end + a[i]);
  c = max(c, end);
  f = best;
  ans = max(ans, f);
  best = max(best, f + c);
}
cout << (long long)(ans % p);
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(1)$。

## 4. [【13NOIP普及组】车站分级](http://ybt.ssoier.cn:8088/problem_show.php?pid=1964)
`NOIP 2013` `普及组`

### 题意

每趟车给出一个停靠站集合，并保证它满足“如果停靠了某站，那么起点到终点间所有级别不低于它的站都必须停靠”。要求反推所有车站最少需要分成多少级。

题目表面像模拟，真正要做的是把“谁的级别必须严格小于谁”整理成偏序，再在这个偏序上求最少层数。

### 分析

抓住一趟车的起点和终点后，区间内部所有“没停靠”的站一定都比这趟车停靠的所有站级别低；否则如果某个没停靠站级别不低于某个停靠站，它就应该被这趟车顺带停下。于是对一趟车，我们得到一批关系：区间内未停站 `<` 区间内已停站。

如果把这些关系两两连边，最坏会炸成平方级。更自然的做法是给每趟车新建一个辅助点 `T`：所有未停站向 `T` 连一条权值 `0` 的边，`T` 再向所有停靠站连一条权值 `1` 的边。这样 `dp[T]` 继承的是“这趟车阈值之前的最低要求”，真正的级别加一发生在从 `T` 走到停靠站时，于是精确表达了“未停站级别必须严格小于停靠站”。

整张图是有向无环的偏序图，拓扑序上做最长路即可。状态 `dp[v]` 表示点 `v` 至少要处在第几级；遇到边 `(u,v,w)` 时转移 `dp[v]=max(dp[v],dp[u]+w)`。所有原始车站初始最低级别都是 `1`，最后在前 `n` 个车站点里取 `dp` 最大值，就是最少需要划出的级别数。

### 核心代码

```cpp
struct E{int v, w;};
for(each train){
  int id = ++tot;
  for(int x = l; x <= r; x++) if(!stop[x]) g[x].push_back({id, 0}), indeg[id]++;
  for(int x : sta) g[id].push_back({x, 1}), indeg[x]++;
}
for(int i = 1; i <= tot; i++) if(!indeg[i]) q.push(i), dp[i] = (i <= n);
while(!q.empty()){
  int u = q.front(); q.pop();
  for(auto [v, w] : g[u]){
    dp[v] = max(dp[v], dp[u] + w);
    if(--indeg[v] == 0) q.push(v);
  }
}
for(int i = 1; i <= n; i++) ans = max(ans, dp[i]);
```

### 复杂度

设所有车次区间总长度为 $L$，时间复杂度 $O(L+\sum s_i+n+m)$，空间复杂度 $O(L+\sum s_i)$。

# 二、提高组赛题

到了提高组，题目开始明显强调结构选择：有的先用数学化简把轮数压缩掉，有的要先定唯一最优匹配再数逆序对，有的则需要在生成树、贪心和状态图之间选对工具。读这一节时，可以重点看“为什么这个模型一旦选对，后面的转移就会自然很多”。

## 5. [【13NOIP提高组】转圈游戏](http://ybt.ssoier.cn:8088/problem_show.php?pid=1875)
`NOIP 2013` `提高组`

### 题意

一轮操作等价于每个人都顺时针整体平移 `m` 个位置。问进行了 $10^k$ 轮后，编号为 `x` 的人落在哪个位置。

本质上这是一个模 `n` 的线性同余，不需要真的模拟轮数。

### 分析

先把一轮的效果看清：第 `i` 个位置的人会去 `i+m`，因此对任意一个固定小伙伴来说，每轮位置都是“当前编号加 `m` 再模 `n`”。做很多轮以后，效果就是总共加了 `m\times` 轮数。

题目真正卡人的地方在 $10^k$ 非常大，但我们只关心它模 `n` 的值，所以直接计算 $10^k\bmod n$ 即可。因为模意义下乘法可以提前取模，所以用快速幂在 $O(\log k)$ 时间内得到这个值。

设 `t=10^k\bmod n`，那么总位移就是 `m\times t\bmod n`，最后位置为 `(x+m\times t)\bmod n`。这里的不变量很简单：无论做多少轮，位置变化始终只取决于累计位移对 `n` 的余数。

### 核心代码

```cpp
long long qpow(long long a, long long b, long long mod){
  long long r = 1 % mod;
  while(b){
    if(b & 1) r = r * a % mod;
    a = a * a % mod; b >>= 1;
  }
  return r;
}
long long t = qpow(10, k, n);
cout << (x + m * t) % n;
```

### 复杂度

时间复杂度 $O(\log k)$，空间复杂度 $O(1)$。

## 6. [【13NOIP提高组】火柴排队](http://ybt.ssoier.cn:8088/problem_show.php?pid=1876)
`NOIP 2013` `提高组`

### 题意

允许在两列火柴内部做相邻交换，希望最后两列按位置一一对应时距离最小，并求达到这个最小距离所需的最少交换次数。

题目把“最小平方距离”和“最少相邻交换”绑在一起，正确顺序是先由距离确定最终匹配，再把交换次数转成逆序对。

### 分析

平方差和最小这一部分，可以直接想到重排不等式：两列火柴最终如果都按高度从小到大排列，同排名的火柴彼此对应时，$\sum (a_i-b_i)^2$ 最小。由于同一列里高度互不相同，这个最优匹配是唯一的——第 `k` 小的火柴只能和另一列第 `k` 小的火柴配对。

最优匹配确定后，问题就只剩“各自最少交换多少次才能走到这个排名”。把第一列排序后第 `k` 个元素的原位置记作 `pa_k`，第二列对应配对元素的原位置记作 `pb_k`，于是我们得到一个排列 `to[pa_k]=pb_k`。只要让第一列按原顺序经过相邻交换变成这个目标顺序，所需交换次数恰好就是排列 `to[1..n]` 的逆序对数。

为什么逆序对就是答案？因为一次相邻交换只能消去一个逆序，而稳定地把一个排列排成升序，最少交换次数正等于它的逆序对总数。于是用树状数组顺扫 `to`：`sum(pos)` 表示已经放好的元素里有多少目标位置不超过 `pos`，当前新增的逆序数就是“前面已处理元素数减去这个数量”。把它们累加并对模数取模即可。

### 核心代码

```cpp
sort(a.begin() + 1, a.end());
sort(b.begin() + 1, b.end());
for(int i = 1; i <= n; i++) to[a[i].id] = b[i].id;
long long ans = 0;
for(int i = 1; i <= n; i++){
  ans = (ans + i - 1 - bit.sum(to[i])) % MOD;
  bit.add(to[i], 1);
}
cout << ans % MOD;
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(n)$。

## 7. [【13NOIP提高组】货车运输](http://ybt.ssoier.cn:8088/problem_show.php?pid=1877)
`NOIP 2013` `提高组`

### 题意

每条路有一个限重，询问两城之间能运输的最大载重。也就是要在所有路径里，把路径上的最小边权尽量做大。

这种“最大化最小值”的问法，通常意味着先抽出一棵最大生成森林，再在树上回答路径最小边权。

### 分析

把一条运输路线的承载能力定义为这条路上最小的限重，那么题目要找的是两点之间“瓶颈值”最大的路径。经典结论是：在最大生成森林里，两点之间树上路径的最小边权，恰好就是原图中的最大瓶颈值。理由和最小生成树类似：如果树上这条瓶颈边还能变大，就会和 Kruskal 选边顺序矛盾。

所以先把所有边按限重从大到小排序，用 Kruskal 建出最大生成森林。这样原图可能有多个连通块，对不连通的询问直接输出 `-1`。连通块内部再做倍增 LCA：`fa[k][u]` 表示 `u` 的 $2^k$ 级祖先，`mn[k][u]` 表示从 `u` 跳到这个祖先途中最小的边权。

查询时先把较深的点抬到同一深度，并一路维护经过边的最小值；如果还没相遇，再从高位到低位同时向上跳，直到它们的父亲相同。最后再把两条最后一步的边一起并入答案。整个过程的不变量是：答案始终等于已经走过那段树路径上的最小边权，最后取完整条路径的最小值即可。

### 核心代码

```cpp
sort(e.begin(), e.end(), [](auto a, auto b){ return a.w > b.w; });
for(auto [u, v, w] : e) if(dsu.unite(u, v)) g[u].push_back({v, w}), g[v].push_back({u, w});
for(int r = 1; r <= n; r++) if(!dep[r]) dfs(r, 0, 1e9);
int query(int u, int v){
  if(dsu.find(u) != dsu.find(v)) return -1;
  int ans = 1e9; if(dep[u] < dep[v]) swap(u, v);
  for(int k = LOG - 1; k >= 0; k--) if(dep[fa[k][u]] >= dep[v]) ans = min(ans, mn[k][u]), u = fa[k][u];
  if(u == v) return ans;
  for(int k = LOG - 1; k >= 0; k--) if(fa[k][u] != fa[k][v]) ans = min(ans, min(mn[k][u], mn[k][v])), u = fa[k][u], v = fa[k][v];
  return min(ans, min(mn[0][u], mn[0][v]));
}
```

### 复杂度

建树与预处理时间复杂度 $O(m\log m+n\log n)$，单次查询时间复杂度 $O(\log n)$，空间复杂度 $O(n\log n+m)$。

## 8. [【13NOIP提高组】积木大赛](http://ybt.ssoier.cn:8088/problem_show.php?pid=1878)
`NOIP 2013` `提高组`

### 题意

每次操作可以把一个连续区间整体加一，要求从全零高度出发，用最少操作次数造出目标高度序列。

只要把每次操作看成“开启一层连续的平台”，最优策略会立刻变成统计每个位置新冒出来了多少层。

### 分析

从左到右看高度变化。假设前一个位置已经堆到了 `h_{i-1}`，当前要到 `h_i`。如果 `h_i\le h_{i-1}`，说明之前开启的那些区间里，有一部分在这里自然结束就行，不需要新增操作；如果 `h_i>h_{i-1}`，那多出来的 `h_i-h_{i-1}` 层只能在位置 `i` 这里新开这么多段区间。

这就说明最优策略完全由相邻差分决定：每当高度上升多少，就必须额外做多少次操作；高度下降部分不用付费，因为那只是旧区间停止延伸。这个结论成立的关键不变量是“每一次操作对应一层连续区间，它一旦开启，就可以尽量向右延伸，直到遇到高度不够的位置再结束”。

于是把 `h_0` 视为 `0`，答案就是 $\sum_{i=1}^n \max(0,h_i-h_{i-1})$。这既是必要的，也是可构造达到的，所以直接输出这个和即可。

### 核心代码

```cpp
long long ans = 0, pre = 0;
for(int i = 1; i <= n; i++){
  ans += max(0, h[i] - pre);
  pre = h[i];
}
cout << ans;
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(1)$。

## 9. [【13NOIP提高组】花匠](http://ybt.ssoier.cn:8088/problem_show.php?pid=1879)
`NOIP 2013` `提高组`

### 题意

从原序列里删去一些花，保留尽量多的花，使剩下高度序列交替呈现“峰谷峰谷”或“谷峰谷峰”的形状。

因为只关心相邻大小关系，不关心具体数值，所以这题最核心的是提炼出“趋势改变时才值得保留拐点”。

### 分析

如果连续几朵花始终在上升，那么中间那些点无论留下哪个，都不会比留下最右端更优：最右端更高，更有利于下一次向下形成峰值。同理，连续下降时只保留最右端也不会吃亏。也就是说，在同一段单调趋势里，真正有价值的只有端点。

于是问题就化成最长摆动子序列。扫描相邻差值 `d_i=h_i-h_{i-1}`，忽略零差值；一旦当前非零差值和上一个有效差值符号不同，说明出现了新的峰或谷，这个拐点就必须被保留，答案加一。最开始保留第一朵花，第一次出现非零趋势时再把第二个端点计入。

这里的不变量是：`pre` 始终记录最近一段保留序列的趋势方向，当前答案对应某个最优摆动子序列长度。当新趋势没有反向时，直接用更靠后的端点替换旧端点；当趋势反向时，说明形成了一个合法转折，答案就能安全加一。扫描结束后的 `ans` 就是最多能留下的花数。

### 核心代码

```cpp
int ans = 1, pre = 0;
for(int i = 2; i <= n; i++){
  long long d = 1ll * h[i] - h[i - 1];
  int cur = (d > 0) - (d < 0);
  if(cur && cur != pre){
    ans++;
    pre = cur;
  }
}
cout << ans;
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(1)$。

## 10. [【13NOIP提高组】华容道](http://ybt.ssoier.cn:8088/problem_show.php?pid=1880)
`NOIP 2013` `提高组`

### 题意

棋盘上有固定格与可移动格，每次给定空白格、目标棋子起点和终点，要求把这枚指定棋子移到目标位置的最短时间。

难点不在单次移动，而在于“要移动目标棋子之前，空白格得先绕到它的某一侧”，所以状态必须同时描述目标棋子位置和空白格相对它站在哪一边。

### 分析

先把一次“推动目标棋子”的动作拆开。若目标棋子当前在格子 `u`，想把它推到相邻格 `v`，那么推动之前空白格必须已经站在 `v` 上；而在这段准备过程中，目标棋子 `u` 不能被别的棋子穿过去，所以对空白格来说，相当于在“封锁 `u`”的棋盘上做一次最短路。这样题目自然变成状态图：状态是“目标棋子在 `u`，空白格在 `u` 的第 `k` 个相邻位置”。

状态之间怎样转移？假设当前空白格在 `u` 的相邻位置 `a`，想把目标棋子推向另一个相邻位置 `b`。那就先在封锁 `u` 的前提下，用 BFS 求出空白格从 `a` 绕到 `b` 的最短距离 `turn[u][a][b]`；准备完成后再做一次真正的交换，代价再加 `1`。于是可以预处理每个可移动格 `u` 周围四个方向两两之间的绕行代价，再在这些状态上跑 Dijkstra。

单次询问时，先以初始空白格为起点、目标棋子起点 `S` 为封锁点做一遍 BFS，把“空白格先站到 `S` 周围哪一侧”作为 Dijkstra 的初始状态。之后每弹出一个状态 `(u,k)`，就尝试把目标棋子从 `u` 推向四个可行邻格，转移到新状态 `(v,back)`。状态距离的含义始终是“已经把目标棋子移动到当前位置所需的最少时间”，所以当某个出队状态满足 `u=T` 时，这个最短路值就是答案；若始终到不了，则输出 `-1`。

### 核心代码

```cpp
void build(int u){
  for(int a = 0; a < 4; a++) if(go[u][a]){
    bfs(go[u][a], u, dist);
    for(int b = 0; b < 4; b++) if(go[u][b]) turn[u][a][b] = dist[go[u][b]];
  }
}
int solve(int E, int S, int T){
  if(S == T) return 0;
  bfs(E, S, dist0);
  memset(dis, 0x3f, sizeof dis);
  for(int a = 0; a < 4; a++) if(go[S][a] && dist0[go[S][a]] < INF) pq.push({dis[S][a] = dist0[go[S][a]], S, a});
  while(!pq.empty()){
    auto [w, u, a] = pq.top(); pq.pop(); if(w != dis[u][a]) continue;
    if(u == T) return w;
    for(int b = 0; b < 4; b++) if(go[u][b] && turn[u][a][b] < INF){
      int v = go[u][b], na = rev[b], nw = w + turn[u][a][b] + 1;
      if(nw < dis[v][na]) pq.push({dis[v][na] = nw, v, na});
    }
  }
  return -1;
}
```

### 复杂度

预处理时间复杂度 $O(nm\cdot 4\cdot nm)$，单次询问时间复杂度约为 $O(nm\log(nm))$，空间复杂度 $O(nm\cdot 4^2)$。
