# 一本通 NOIP 2006赛题解题报告

# 一、普及组赛题

这一组题目的重心是把题面条件翻译成清晰模型：要么抓住数据范围直接设计状态，要么抓住“按顺序的下一个”与“互不相等的幂和”这样的结构性质。做这几题时，最重要的不是背模板，而是先问自己：题目真正限制了什么，答案又该从哪里读出来。

## 1. [【06NOIP普及组】明明的随机数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1934)
`NOIP 2006` `普及组`

### 题意
给出若干个 1 到 1000 之间的随机整数，要求先去掉重复数字，再按从小到大的顺序输出不同数字的个数和这些数字。

### 分析
这题最容易先想到“排序后去重”，但题面还有一个更关键的条件：**所有数字都落在 1 到 1000 的小范围内**。一旦看到值域很小，就要想到“出现过没有”可以直接用桶或布尔数组记录，不必真的把所有数拿去比较。

设 `vis[x]` 表示数字 `x` 是否出现过。顺次读入每个数时，只做一件事：把 `vis[x]` 记成真。这个状态的**不变量**是：处理完前 `i` 个数后，`vis[x]=1` 当且仅当 `x` 在前 `i` 个数里至少出现过一次。这样重复元素天然被折叠掉了。

接下来再从 1 扫到 1000。因为扫描顺序本身就是升序，所以凡是 `vis[x]=1` 的数字，加入答案数组时就已经自动排好序。最后答案提取也很直接：**答案个数**就是结果数组长度，**答案序列**就是按桶序收集出的所有数字。

### 核心代码
```cpp
bool vis[1001];
vector<int> ans;

void solve(int n) {
    for (int i = 1, x; i <= n; i++) {
        read(x);
        vis[x] = true;
    }
    for (int x = 1; x <= 1000; x++) {
        if (vis[x]) ans.push_back(x);
    }
}
```

### 复杂度
时间复杂度 `O(n + 1000)`，空间复杂度 `O(1000)`。

## 2. [【06NOIP普及组】开心的金明](http://ybt.ssoier.cn:8088/problem_show.php?pid=1935)
`NOIP 2006` `普及组`

### 题意
给定总预算 `N` 和若干件物品，每件物品有价格 `v` 与重要度 `p`，选择若干件且总价不超过 `N`，使 `v*p` 的总和最大。

### 分析
题面里每件物品只有“选”或“不选”两种状态，且这一题没有主件、附件依赖，所以它就是非常标准的 **0/1 背包**。真正要做的，是把“满意度”翻译成价值：每件物品的贡献不是 `p`，而是 `v*p`。

令 `dp[j]` 表示**预算不超过 `j` 时，当前已经处理过的物品能够取得的最大总价值**。当我们处理一件价格为 `v`、价值为 `v*p` 的物品时，如果选它，就从 `dp[j-v]` 转移到 `dp[j]`；如果不选，`dp[j]` 保持原值。因此转移是 `dp[j] = max(dp[j], dp[j-v] + v*p)`。

这题最关键的细节是容量必须**倒序枚举**。倒序的意义不是形式要求，而是状态不变量：在处理当前物品时，`dp[j-v]` 仍然代表“上一层”的结果，这样当前物品最多只会被使用一次。最后答案直接从 `dp[N]` 读取，因为它就是总预算限制下的最优值。

### 核心代码
```cpp
int dp[30005];

void solve(int n, int m) {
    for (int i = 1, v, p; i <= m; i++) {
        read(v, p);
        int val = v * p;
        for (int j = n; j >= v; j--) {
            dp[j] = max(dp[j], dp[j - v] + val);
        }
    }
}
```

### 复杂度
时间复杂度 `O(mN)`，空间复杂度 `O(N)`。

## 3. [【06NOIP普及组】Jam的计数法](http://ybt.ssoier.cn:8088/problem_show.php?pid=1936)
`NOIP 2006` `普及组`

### 题意
在给定字母范围 `[s, t]` 内，一个 Jam 数字是长度固定、从左到右严格递增且字母互不相同的字符串。输入一个合法 Jam 数字，要求按顺序输出它后面的最多 5 个 Jam 数字。

### 分析
这题的核心不是暴力枚举所有字符串，而是把题目看成“**在一个严格递增序列上求下一个字典序组合**”。设当前串对应的编号数组为 `a[0..w-1]`，其中每个位置都严格递增。

如果想得到紧邻的下一个串，就应该尽量让前缀不变，只在最靠右还能变大的位置动手。设位置 `i` 右边还剩 `w-1-i` 个位置，那么 `a[i]` 最大只能增到 `t-(w-1-i)`；否则就没有足够大的数给后缀保持严格递增。因此我们从右往左找第一个满足 `a[i] < t-(w-1-i)` 的位置，这就是本次必须调整的位置。

把 `a[i]` 加一以后，为了得到**紧接着的**下一个而不是跳过中间答案，后缀必须恢复成最小合法形态，即令 `a[i+1]=a[i]+1`，`a[i+2]=a[i+1]+1`，依次类推。这里的**不变量**是：修改后前缀尽量不变，后缀保持严格递增且字典序最小，所以它恰好就是当前串的直接后继。每成功构造一次就输出，再从新串继续求下一个，最多重复 5 次；如果某次根本找不到可增加的位置，说明后继不存在，过程结束。

### 核心代码
```cpp
int a[30];

bool next_jam(int w, int t) {
    for (int i = w - 1; i >= 0; i--) {
        int up = t - (w - 1 - i);
        if (a[i] == up) continue;
        a[i]++;
        for (int j = i + 1; j < w; j++) a[j] = a[j - 1] + 1;
        return true;
    }
    return false;
}
```

### 复杂度
每次求后继的时间复杂度 `O(w)`，最多输出 5 个，因此总时间复杂度 `O(5w)`，空间复杂度 `O(w)`。

## 4. [【06NOIP普及组】数列](http://ybt.ssoier.cn:8088/problem_show.php?pid=1937)
`NOIP 2006` `普及组`

### 题意
给定 `k`，把所有 `k` 的不同次幂之和组成一个递增序列，要求求出这个序列的第 `N` 项。

### 分析
先别急着列出很多项，而是要看清这个数列里的每个数都长什么样：它们都可以写成
`b0*k^0 + b1*k^1 + b2*k^2 + ...`，其中每个 `bi` 只能是 `0` 或 `1`。也就是说，本题本质上是在枚举“只含 0/1 系数的 k 进制表示”。

关键观察是：`k>=3` 时，高位幂一旦出现，影响会压过所有更低位幂的任意组合。因此比较两个这样的和时，决定大小的就是**最高的那个不同系数位**。这和比较二进制数的大小完全一致：如果把系数串 `bi` 当成一个二进制整数的各位，那么这些和按从小到大排列的顺序，就恰好对应正整数 `1,2,3,...` 的二进制表示顺序。

所以第 `N` 项根本不用搜索。只要把 `N` 写成二进制：若第 `i` 位为 1，就把 `k^i` 加入答案；若为 0，就跳过。这里的**状态含义**非常直接：扫描二进制位时，当前幂 `pw` 始终表示 `k^i`，而 `ans` 始终表示已经处理过的那些 1 位对应的幂和。位扫完以后，`ans` 就是第 `N` 项。

### 核心代码
```cpp
int solve(int k, int n) {
    int ans = 0, pw = 1;
    while (n) {
        if (n & 1) ans += pw;
        pw *= k;
        n >>= 1;
    }
    return ans;
}
```

### 复杂度
时间复杂度 `O(log N)`，空间复杂度 `O(1)`。

# 二、提高组赛题

提高组的四题开始强调“结构化状态设计”。环要拉直成区间，依赖关系要压成分组决策，调度过程要维护不变量，计数问题要先化成组合模型再做高精度。做题时如果能先把对象抽象对，后面的式子和代码都会自然很多。

## 5. [【06NOIP提高组】能量项链](http://ybt.ssoier.cn:8088/problem_show.php?pid=1843)
`NOIP 2006` `提高组`

### 题意
有一串环形能量珠，相邻珠子可以合并，若两颗珠子的连接点标记相同，合并时会释放对应乘积能量。要求选择合并顺序，使整串项链最后合成一颗珠子时释放的总能量最大。

### 分析
一看到“不同合并顺序导致总收益不同”，就要想到区间 DP；一看到“项链是环”，就要先想办法把环拆成链。设第 `i` 颗珠子的头标记为 `a[i]`，那么它的尾标记就是 `a[i+1]`，最后一颗的尾标记回到 `a[1]`。把数组复制一遍后，任意长度为 `n` 的连续段都对应项链断开后的一个展开方案。

接下来考虑链上区间 `[l, r]`。如果这段珠子最终合成一颗珠子，那么它的头标记一定是 `a[l]`，尾标记一定是 `a[r+1]`，这件事与具体合并顺序无关，是整个状态设计里最重要的**不变量**。于是设 `dp[l][r]` 表示把区间 `[l, r]` 全部合并后的最大能量。

最后一次合并一定是把 `[l, k]` 与 `[k+1, r]` 两段合成，所以转移自然是：
`dp[l][r] = max(dp[l][k] + dp[k+1][r] + a[l]*a[k+1]*a[r+1])`。
前两项是左右子区间各自最优，最后一项是最后一步合并释放的能量。枚举区间长度从小到大，保证子问题先算完。所有长度为 `n` 的展开区间里取最大值，就是原环的最优答案。

### 核心代码
```cpp
int a[205], dp[205][205];

int solve(int n) {
    for (int i = 1; i <= n; i++) a[i + n] = a[i];
    a[2 * n + 1] = a[1];
    for (int len = 2; len <= n; len++) {
        for (int l = 1; l + len - 1 <= 2 * n; l++) {
            int r = l + len - 1;
            for (int k = l; k < r; k++) {
                dp[l][r] = max(dp[l][r], dp[l][k] + dp[k + 1][r] + a[l] * a[k + 1] * a[r + 1]);
            }
        }
    }
    int ans = 0;
    for (int l = 1; l <= n; l++) ans = max(ans, dp[l][l + n - 1]);
    return ans;
}
```

### 复杂度
时间复杂度 `O(n^3)`，空间复杂度 `O(n^2)`。

## 6. [【06NOIP提高组】金明的预算方案](http://ybt.ssoier.cn:8088/problem_show.php?pid=1844)
`NOIP 2006` `提高组`

### 题意
在总预算限制下选择主件和附件，使价格与重要度乘积之和最大。附件必须依附于自己的主件购买，每个主件最多有两个附件。

### 分析
和普及组那题相比，本题多出来的关键条件是**附件不能单独选**。这意味着它不再是“每件物品独立地选或不选”，而是“每个主件带着若干种合法购买方案一起参加决策”。一旦看清这一点，题目就从普通 0/1 背包变成了**分组背包**。

对于每个主件，合法方案其实非常少。若它有两个附件 `a1`、`a2`，那么可选组合只有四种：主件单独买、主件加 `a1`、主件加 `a2`、主件加两个附件。因为附件不能脱离主件，所以所有非法情况在枚举方案时就被彻底排除了。

设 `dp[j]` 表示处理完前若干个主件后，预算不超过 `j` 时的最大收益。处理当前主件组时，仍然倒序枚举预算 `j`，再枚举这个组里所有合法组合 `(cost, val)`，转移为 `dp[j] = max(dp[j], dp[j-cost] + val)`。这里的**状态不变量**是：每个主件组最多只会贡献一种买法，因此不会把同一主件重复购买。最终答案直接读取 `dp[N]`。

### 核心代码
```cpp
struct Group {
    int c[4], v[4], cnt;
};
int dp[32005];
Group g[65];

void add_plan(Group &x, int cost, int val) {
    x.c[++x.cnt] = cost;
    x.v[x.cnt] = val;
}

int solve(int n, int tot) {
    for (int i = 1; i <= tot; i++) {
        for (int j = n; j >= 0; j--) {
            for (int k = 1; k <= g[i].cnt; k++) {
                if (j >= g[i].c[k]) dp[j] = max(dp[j], dp[j - g[i].c[k]] + g[i].v[k]);
            }
        }
    }
    return dp[n];
}
```

### 复杂度
设主件数为 `G`，时间复杂度 `O(GN)`，每组只枚举常数个方案；空间复杂度 `O(N)`。

## 7. [【06NOIP提高组】作业调度方案](http://ybt.ssoier.cn:8088/problem_show.php?pid=1845)
`NOIP 2006` `提高组`

### 题意
有若干工件和机器，每个工件有固定工序顺序，每道工序对应指定机器和加工时间。已给出操作的安排顺序，要求按题目规定的“尽量靠前插入、若可插多个空档则选最前空档”规则，求最终完成全部作业所需的总时间。

### 分析
这题最容易乱在“安排顺序”和“实际加工顺序”不是一回事。真正该维护的是：当我们按给定顺序处理到一个操作时，前面已经安排好的所有操作都不能改动，所以当前操作只能插入某台机器现有时间轴上的最早可行空档。

因此我们顺着安排顺序模拟即可。设 `step[j]` 表示工件 `j` 已经安排到第几道工序，那么当前读到工件 `j` 时，它要安排的就是下一道工序 `k=++step[j]`。这道工序有两个硬约束：一是必须等前一道工序完成，所以最早开工时刻是 `endt[j]`；二是目标机器在加工区间内必须空闲。

最直接的实现是给每台机器开一条时间轴布尔数组。对于当前操作，从 `endt[j]` 开始往后试最早起点 `t`，检查 `[t, t+cost)` 是否全空；若中间撞上已占用时刻，就把起点跳到冲突点后面继续试。找到后立即把这段区间标记为已占用，并更新 `endt[j]=t+cost`。这里的**不变量**是：时间轴上所有已标记区间，恰好对应题目要求下已经固定的安排结果；因此每次找到的第一个可行位置，就是题目规定的唯一插入位置。所有工件的完成时间最大值就是最终答案。

### 核心代码
```cpp
bool used[25][8005];
int step[25], endt[25];

int solve(int m, int n) {
    int ans = 0;
    for (int id = 1; id <= n * m; id++) {
        int job = seq[id], op = ++step[job];
        int mch = mac[job][op], cost = tim[job][op], t = endt[job];
        while (true) {
            bool ok = true;
            for (int p = t; p < t + cost; p++) {
                if (used[mch][p]) {
                    t = p + 1;
                    ok = false;
                    break;
                }
            }
            if (ok) break;
        }
        for (int p = t; p < t + cost; p++) used[mch][p] = true;
        endt[job] = t + cost;
        ans = max(ans, endt[job]);
    }
    return ans;
}
```

### 复杂度
设最终总时间为 `T`，时间复杂度约为 `O(nmT)`，在题目给定数据范围内可以通过；空间复杂度 `O(mT)`。

## 8. [【06NOIP提高组】2^k进制数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1846)
`NOIP 2006` `提高组`

### 题意
把一个二进制串从右向左每 `k` 位划成一段，就能对应一个 `2^k` 进制数。要求统计满足“至少两位、最高位非零、各位严格递增、转换回二进制后总位数不超过 `w`”的不同 `2^k` 进制数个数。

### 分析
这题表面看像进制转换，真正难点却在**计数模型**。设 `B = 2^k`，若一个 `B` 进制数有 `len` 位，记为 `d1 < d2 < ... < dlen`。后面 `len-1` 位一定各占满 `k` 个二进制位，只有最高位 `d1` 可能因为总长度不超过 `w` 而受更紧的上界限制。

固定长度 `len` 后，留给最高位的二进制位数是 `rem = w - (len-1)k`，但一个 `B` 进制位最多只需要 `k` 个二进制位，所以真正能使用的是 `bits = min(k, rem)`。于是最高位必须满足 `1 <= d1 < 2^bits`，其余各位只要严格递增且小于 `B` 即可。

接下来把它化成组合计数。若最高位定成 `x`，后面 `len-1` 位要从 `x+1` 到 `B-1` 里选，方案数是 `C(B-1-x, len-1)`。把 `x` 从 `1` 累加到 `2^bits-1`，得到固定长度的总方案数。利用组合数的“错位求和”恒等式，可以把这一串和压成：
`C(B-1, len) - C(B-2^bits, len)`。
这样题目就转成了纯组合求和。因为答案可能非常大，我们用高精度整数预处理杨辉三角 `C[n][k]`，再按长度累加即可。这里的**状态**就是组合数表本身，`C[i][j]` 始终表示从 `i` 个数中选 `j` 个的方案数；最后对所有合法 `len` 求和，就是所求答案。

### 核心代码
```cpp
using boost::multiprecision::cpp_int;

cpp_int C[513][513];

cpp_int solve(int k, int w) {
    int B = 1 << k;
    for (int i = 0; i <= B; i++) {
        C[i][0] = C[i][i] = 1;
        for (int j = 1; j < i; j++) C[i][j] = C[i - 1][j - 1] + C[i - 1][j];
    }
    cpp_int ans = 0;
    int lim = min(B, (w + k - 1) / k);
    for (int len = 2; len <= lim; len++) {
        int bits = min(k, w - (len - 1) * k);
        int up = 1 << bits;
        ans += C[B - 1][len];
        if (B - up >= len) ans -= C[B - up][len];
    }
    return ans;
}
```

### 复杂度
时间复杂度 `O((2^k)^2)`，空间复杂度 `O((2^k)^2)`。
