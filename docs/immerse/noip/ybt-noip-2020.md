# 一本通 2020赛题解题报告

这一年的题目从最朴素的二进制拆分、计数桶与表达式依赖，一路走到历法换算、位运算计数、DAG 复合、博弈贪心、字符串周期、高维盒子计数与构造分治。表面上模型跨度很大，但真正贯穿全篇的主线始终没变：先把题面条件压成稳定的状态结构，再利用这个结构去判断哪些信息必须保留、哪些过程其实可以整体打包。

# 一、CSP-J 赛题

这一章更像在训练“第一步建模”的手感：有的题一眼要翻成二进制，有的题要把动态排名改写成值域计数，有的题要看出表达式里真正会影响根的那条依赖链，还有的题要把走格子的路线压成按列推进的状态。

## 1. [【20CSPJ普及组】优秀的拆分](http://ybt.ssoier.cn:8088/problem_show.php?pid=2004)

`CSP 2020` `CSP-J`

### 题意
给定一个正整数 `n`，要求判断它能否拆成若干个两两不同的 $2$ 的正整数次幂之和；如果可以，就按从大到小输出这组拆分，否则输出 `-1`。

### 分析
这题真正要认出的结构，其实就是二进制表示。一个数拆成互不相同的 $2$ 的幂，本质上就是把它写成若干个不同二进制位的和，而这种写法在固定顺序后是唯一的。

题目的额外限制是“正整数次幂”，也就是只能用 $2^1,2^2,2^3,\dots$，不能用 $2^0=1$。这一下就把判定条件压得非常直接：如果 `n` 是奇数，它的二进制最低位一定是 $1$，想凑出这个最低位就必须用到 $1$，因此无解；如果 `n` 是偶数，最低位本来就是 $0$，其余为 $1$ 的二进制位恰好对应一组合法拆分。

所以答案既不用搜索，也不用动态规划，直接从高位往低位扫描二进制位即可。某一位为 $1$，就把对应的 $2^i$ 输出出来；位为 $0$，就跳过。题目要求从大到小输出，这和从高位到低位扫描完全一致。

### 核心代码
```cpp
if(n & 1) cout << -1;
else{
  bool first = true;
  for(int i = 60; i >= 1; i--){
    if((n >> i) & 1){
      if(!first) cout << ' ';
      cout << (1LL << i);
      first = false;
    }
  }
}
```

### 复杂度
时间复杂度 $O(\log n)$，空间复杂度 $O(1)$。

## 2. [【20CSPJ普及组】直播获奖](http://ybt.ssoier.cn:8088/problem_show.php?pid=2005)

`CSP 2020` `CSP-J`

### 题意
比赛成绩会按顺序公布。设当前已经公布了 `p` 个选手，计划获奖人数是 $\max(1,\lfloor p\times w/100\rfloor)$，分数线定义为当前前若干名中的最低分。要求在每次公布新成绩后，立刻输出当前分数线。

### 分析
题面看着像“动态维护排名”，但关键数据范围把题目压得很简单：每个分数都在 $0$ 到 $600$ 之间。既然值域这么小，就没必要上平衡树或堆，直接维护一个计数桶 `cnt[s]` 表示分数 `s` 出现了多少次。

当第 `i` 个成绩到来后，先把它扔进桶里，然后算出当前计划获奖人数 $need = \max(1,\lfloor iw/100 \rfloor)$。接下来只要从高分 `600` 往低分扫，维护已经数过多少人。第一次使累计人数达到或超过 `need` 的分数，就是分数线。

为什么并列情况会被自动处理？因为题目要的是“前 `need` 名里的最低分”。如果某个分数段一加进去就让累计人数跨过了 `need`，那说明这个分数的所有人都和分数线并列，自然都应该算作获奖者，而我们输出的也正是这个分数。

所以这题最值得记住的点是：**动态排名不一定要维护有序序列，只要值域足够小，计数桶就是最稳的有序结构。**

### 核心代码
```cpp
int cnt[601] = {};
for(int i = 1; i <= n; i++){
  cnt[a[i]]++;
  int need = max(1, i * w / 100), sum = 0, cut = 600;
  while(cut >= 0 && sum + cnt[cut] < need) sum += cnt[cut--];
  cout << cut << " \n"[i == n];
}
```

### 复杂度
时间复杂度 $O(600n)$，空间复杂度 $O(600)$。

## 3. [【20CSPJ普及组】表达式](http://ybt.ssoier.cn:8088/problem_show.php?pid=2006)

`CSP 2020` `CSP-J`

### 题意
给出一个只含变量、`!`、`&`、`|` 的逻辑表达式，输入形式是后缀表达式。每个变量只出现一次，先给出所有变量当前取值，再回答若干次询问：把某个变量取反后，整个表达式的值会变成多少。

### 分析
如果每次询问都真的把变量改掉再重算整棵表达式树，复杂度会变成 $O(q|s|)$，显然扛不住。题目的关键不是“如何多次求值”，而是先搞清楚：**一个变量翻转后，到底有没有机会影响到根的值。**

先用栈把后缀表达式还原成表达式树，再自底向上求出每个结点当前的布尔值。接着从根往下做第二遍 DFS，标记“哪些叶子在当前赋值下真正有影响力”。这个判断完全由运算符和兄弟结点的当前值决定：

- 对于 `!`，子结点一定有影响，因为取反会直接翻转结果。
- 对于 `a & b`，某个孩子想影响父亲，另一个孩子当前必须是 $1$。因为只有在“另一边不拖后腿”的时候，这一边的翻转才会改变 `&` 的值。
- 对于 `a | b`，某个孩子想影响父亲，另一个孩子当前必须是 $0$。道理同上，只有另一边没把结果锁死为 $1$ 时，这一边才有发言权。

于是根到叶子上真正“可传递影响”的路径都能被一次 DFS 标出来。若询问变量 `x` 被标记过，答案就是把根值翻转；若没被标记过，说明它无论怎么变都传不到根，答案保持不变。

### 核心代码
```cpp
int calc(int u){
  if(isVar[u]) return val[u];
  if(op[u] == '!') return val[u] = !calc(ls[u]);
  int a = calc(ls[u]), b = calc(rs[u]);
  return val[u] = (op[u] == '&' ? (a & b) : (a | b));
}
void mark(int u){
  if(isVar[u]){ can[id[u]] = 1; return; }
  if(op[u] == '!') mark(ls[u]);
  else if(op[u] == '&'){
    if(val[rs[u]] == 1) mark(ls[u]);
    if(val[ls[u]] == 1) mark(rs[u]);
  }else{
    if(val[rs[u]] == 0) mark(ls[u]);
    if(val[ls[u]] == 0) mark(rs[u]);
  }
}
cout << (can[x] ? (val[root] ^ 1) : val[root]) << '\n';
```

### 复杂度
建树与两次 DFS 的时间复杂度都是 $O(|s|)$，回答全部询问的时间复杂度是 $O(q)$，空间复杂度 $O(|s|)$。

## 4. [【20CSPJ普及组】方格取数](http://ybt.ssoier.cn:8088/problem_show.php?pid=2007)

`CSP 2020` `CSP-J`

### 题意
在一个 $n\times m$ 的网格里，从左上角走到右下角，每步只能向上、向下或向右，而且不能重复经过格子。路径上所有格子的权值都会被拿走，要求最大化总和。

### 分析
“不能向左”是这题最关键的结构。它意味着路径一旦离开某一列，就永远不会回来，所以整条路径可以按列来理解。进一步看，在同一列里如果出现两段分离的访问区间，中间就必须靠左右绕回来才能连起来，但题目又禁止向左和重复经过，因此每一列中被走到的格子一定形成**一段连续区间**。

于是可以做“按列推进”的动态规划。设 `f[i]` 表示处理完前一列后，路径停在当前列左侧、并准备从第 `i` 行进入新列时的最优值。进入新列后，路径可能从某个更上面的行一路向下走到 `i`，也可能从某个更下面的行一路向上走到 `i`，所以同一列内需要两种转移：

- `down[i]`：从上往下扫，表示最后一段在当前列里向下走到第 `i` 行的最优值；
- `up[i]`：从下往上扫，表示最后一段在当前列里向上走到第 `i` 行的最优值。

这两个数组都只需要线性递推：`down[i]` 要么继承 `down[i-1]` 继续往下，要么直接从上一列的 `f[i]` 横着进来；`up[i]` 同理。最后当前列停在第 `i` 行的最优值就是 `max(down[i], up[i])`。

这种做法本质上是把“这一列里走哪一段”压成了两次单调扫描，不需要显式枚举进入行和离开行，因此能把复杂度压到 $O(nm)$。

### 核心代码
```cpp
const long long NEG = -(1LL << 60);
vector<long long> f(n + 1, NEG), g(n + 1), up(n + 2), down(n + 2);
f[1] = 0;
for(int col = 1; col <= m; col++){
  down[1] = f[1] + a[1][col];
  for(int i = 2; i <= n; i++) down[i] = max(down[i - 1], f[i]) + a[i][col];
  up[n] = f[n] + a[n][col];
  for(int i = n - 1; i >= 1; i--) up[i] = max(up[i + 1], f[i]) + a[i][col];
  for(int i = 1; i <= n; i++) g[i] = max(down[i], up[i]);
  f.swap(g);
}
cout << f[n];
```

### 复杂度
时间复杂度 $O(nm)$，空间复杂度 $O(n)$。

# 二、CSP-S 赛题

这一章的题型跨度一下拉开，但共同难点仍然是结构提炼：有些题要把复杂规则压成公式或直接映射，有些题要把嵌套调用改写成 DAG 传播，有些题则要把构造、计数和博弈都先转成一个更稳定的中间模型，再从那个模型里把答案提出来。

## 5. [【20CSPS提高组】儒略日](http://ybt.ssoier.cn:8088/problem_show.php?pid=2008)

`CSP 2020` `CSP-S`

### 题意
输入若干个儒略日编号 `r`，要求把它们还原成对应的公历日期。日期在 $1582$ 年 $10$ 月 $15$ 日之前采用儒略历，在这一天及之后采用格里高利历，同时不存在公元 $0$ 年。

### 分析
这题如果按“从起点一天一天往后推”来想，肯定会被年份范围直接卡死。真正适合本题的视角是：儒略日本来就是一条连续编号的整数轴，我们只需要把这个编号**直接映射回日期**。

关键分界点是 `2299161`。它对应公历的 $1582$ 年 $10$ 月 $15$ 日；再前一天 `2299160` 对应的是儒略历的 $1582$ 年 $10$ 月 $4$ 日，中间那十天天然就被跳过去了。所以做法非常清楚：

- 若 `r < 2299161`，按儒略历公式还原年月日；
- 否则，按格里高利历公式还原年月日。

这两套公式本质上都在做同一件事：先把天数压成年循环、月循环，再从剩余量里拆出日。好处是它们已经把闰年规则和历法边界编码进去了，不需要我们手工分类讨论每一年。最后只剩一个输出细节：公式里用的是天文学记年，`0` 表示公元前 `1` 年，所以如果算出的年份 `y<=0`，输出时要转成 `1-y BC`。

这题最值得记住的是：**当题目给出的对象本身就是标准编号体系时，优先想“编号到答案”的直接变换，而不是模拟时间流逝。**

### 核心代码
```cpp
void solve(long long z){
  long long a, b, c, d, e, m, y, mon, day;
  if(z >= 2299161){
    a = z + 32044; b = (4 * a + 3) / 146097; c = a - 146097 * b / 4;
    d = (4 * c + 3) / 1461; e = c - 1461 * d / 4; m = (5 * e + 2) / 153;
    day = e - (153 * m + 2) / 5 + 1; mon = m + 3 - 12 * (m / 10);
    y = 100 * b + d - 4800 + m / 10;
  }else{
    c = z + 32082; d = (4 * c + 3) / 1461; e = c - 1461 * d / 4; m = (5 * e + 2) / 153;
    day = e - (153 * m + 2) / 5 + 1; mon = m + 3 - 12 * (m / 10); y = d - 4800 + m / 10;
  }
  if(y <= 0) cout << day << ' ' << mon << ' ' << 1 - y << " BC\n";
  else cout << day << ' ' << mon << ' ' << y << '\n';
}
```

### 复杂度
每次询问的时间复杂度 $O(1)$，空间复杂度 $O(1)$。

## 6. [【20CSPS提高组】动物园](http://ybt.ssoier.cn:8088/problem_show.php?pid=2009)

`CSP 2020` `CSP-S`

### 题意
有 $2^k$ 种动物编号，动物园当前饲养了其中 `n` 种。每条规则形如：如果某只已饲养动物的第 `p` 个二进制位是 $1$，就必须购买某种饲料。现在问还可以额外加入多少种动物，使得饲料清单完全不发生变化。

### 分析
题面里有很多关于饲料编号的描述，但真正决定答案的其实只有“哪些二进制位会不会触发新规则”。把所有当前动物编号按位看，设 `has` 为它们的按位或：若 `has` 的第 `p` 位是 $1$，说明现有动物里已经有人触发过第 `p` 位对应的所有规则。

于是某一位 `p` 只分两种情况：

- 这一位根本没有任何规则约束，那它取 $0$ 还是 $1$ 都无所谓；
- 这一位有规则。如果当前 $has[p]=1$，那么相关饲料本来就已经买了，新动物在这一位上也可以自由取值；如果当前 $has[p]=0$，那新动物一旦把这一位置成 $1$，就会新增饲料，违反题意，所以这一位只能取 $0$。

这样一来，问题就被压成了“有多少位是自由位”。设自由位个数为 `freeBit`，那么所有合法新动物的编号总数就是 $2^{freeBit}$。题目问的是“还能饲养多少种”，所以最后再减去已经存在的 `n` 种动物即可。

这题的关键是不要被“饲料种类”本身带跑偏。真正的约束落点是二进制位，而不是饲料编号。只要把规则翻成“这一位能不能随便填”，计数就一眼看穿了。

### 核心代码
```cpp
unsigned long long has = 0;
for(auto x : a) has |= x;
vector<int> need(64, 0);
for(auto [p, q] : rule) need[p] = 1;
int freeBit = 0;
for(int b = 0; b < k; b++)
  if(!need[b] || ((has >> b) & 1ULL)) freeBit++;
__int128 ans = ((__int128)1 << freeBit) - n;
print(ans);
```

### 复杂度
时间复杂度 $O(n+m+k)$，空间复杂度 $O(k)$。

## 7. [【20CSPS提高组】函数调用](http://ybt.ssoier.cn:8088/problem_show.php?pid=2010)

`CSP 2020` `CSP-S`

### 题意
有三类函数：单点加法、全局乘法、按顺序调用若干函数。调用关系保证无递归。给定初始数组和最终要执行的一串函数编号，要求输出全部调用结束后的数组，答案对 $998244353$ 取模。

### 分析
直接模拟最大的障碍，是第三类函数会把很多操作套在一起，而且“加法在乘法前后”的效果完全不同。要把它做快，得先认出一个事实：**任意一个函数执行完，对整个数组的影响都可以拆成两部分——先整体乘上某个系数，再在若干位置额外加一些量。**

第一步先求每个函数的乘法贡献 $mul[u]$。对于单点加法，$mul=1$；对于全局乘法，$mul=v$；对于复合函数，因为它按顺序依次调用若干子函数，所以总乘法系数就是这些子函数 $mul$ 的连乘。

难点在第二步：统计每个原子“单点加法函数”到底等价执行了多少次。这里要抓住“后缀积”——如果某个复合函数依次调用 `g1,g2,...,gt`，那么 `g1` 产生的加法，最终还要被后面的 `g2..gt` 的乘法效应整体放大；`g2` 的加法会被 `g3..gt` 放大；依此类推。所以从一个父函数向子函数分发贡献时，必须从右往左扫，用后缀乘积把权重传下去。

于是可以建一张调用 DAG：先用逆拓扑序求出所有 `mul[u]`，再把最终查询序列看成一个超级源点 `0`，同样按顺序连向它调用的函数。随后按拓扑序做一次“权值下推”，把每个函数的调用权重 `cnt[u]` 传给子函数。最后：

- 先把原数组整体乘上超级源点的 `mul[0]`；
- 再把每个第一类函数的贡献 `cnt[u] * add[u]` 加到对应位置。

这样就把“嵌套调用”改写成了 DAG 上的两次传播：一次传乘法系数，一次传加法权重。

### 核心代码
```cpp
for(int u : revTopo){
  if(type[u] == 1) mul[u] = 1;
  else if(type[u] == 2) mul[u] = v[u];
  else{
    mul[u] = 1;
    for(int x : son[u]) mul[u] = mul[u] * mul[x] % MOD;
  }
}
son[0] = qry; mul[0] = 1;
for(int x : son[0]) mul[0] = mul[0] * mul[x] % MOD;
auto push = [&](int u, long long base){
  long long suf = 1;
  for(int i = (int)son[u].size() - 1; i >= 0; i--){
    int x = son[u][i];
    cnt[x] = (cnt[x] + base * suf) % MOD;
    suf = suf * mul[x] % MOD;
  }
};
cnt[0] = 1; push(0, 1);
for(int u : topo) if(type[u] == 3) push(u, cnt[u]);
for(int i = 1; i <= n; i++) a[i] = a[i] * mul[0] % MOD;
for(int u = 1; u <= m; u++) if(type[u] == 1)
  a[pos[u]] = (a[pos[u]] + cnt[u] * addv[u]) % MOD;
```

### 复杂度
设所有第三类函数调用总长度为 $E$，时间复杂度 $O(n+m+E)$，空间复杂度 $O(n+m+E)$。

## 8. [【20CSPS提高组】贪吃蛇](http://ybt.ssoier.cn:8088/problem_show.php?pid=2011)

`CSP 2020` `CSP-S`

### 题意
若干条蛇按体力值比较强弱；每轮最强的蛇可以选择吃掉最弱的蛇，吃完后自身体力变成“原体力减去对方体力”，也可以选择立刻停止。所有蛇都足够聪明，会在保证自己不被吃的前提下尽量多吃。问最终会剩多少条蛇。

### 分析
这题最难的地方不在模拟，而在先找出“什么情况下最强蛇一定会吃”。关键结论有两个：

- 如果最强蛇吃掉最弱蛇后，**不会变成新的最弱蛇**，那它一定会吃。因为它仍然足够强，后续不会因为这一步立刻送命。
- 如果它一吃就会变成新的最弱蛇，那么局面进入“危险链”。这时是否敢吃，要看后面经过多少轮才能回到上一种“必吃”局面；这个轮数的奇偶性，决定了当前这条蛇会不会成为那条最终被反吃的蛇。

所以整场博弈可以拆成两段：前面是一段确定会持续吃掉弱者的贪心过程；一旦第一次进入“吃完就变成最弱”的区域，后面只剩一个奇偶性判断。题解里常把这叫成第二阶段。

为了在线性时间里不断取全局最小、全局最大、再插入新生成的蛇，可以用两个单调队列：`q1` 放原始序列，`q2` 放后来生成的新蛇。由贪心过程可证明，新生成的蛇按产生顺序也是单调的，因此两个队列都能保持有序。每轮只要比较两边队头取最小、比较队尾取最大即可。

当出现“新蛇比当前最小还小”时，说明进入了第二阶段；此后只需要记录危险链的奇偶性。若之后又回到了安全局面，就可以根据奇偶性直接推出最终剩余条数，不必真的把整段博弈全部展开。

### 核心代码
```cpp
int solve(){
  initTwoQueues();
  int alive = n, sum = n, danger = 0, parity = 0;
  while(sum >= 2){
    auto mx = popMax(), mn = popMin();
    mx.first -= mn.first;
    sum--;
    if(sum >= 2 && mx < getMin()){
      if(danger) parity ^= 1;
      else danger = 1;
    }else{
      if(danger) return alive - parity;
      alive--;
    }
    pushNew(mx);
  }
  return alive;
}
```

### 复杂度
每组数据的时间复杂度 $O(n)$，空间复杂度 $O(n)$。

# 三、NOIP 赛题

这一章开始是 2020 年的 NOIP 提高组赛题。和前面的 CSP 相比，这几题更强调把抽象出来的模型继续往前推：有的题要把奇偶结构拆成可贪心的两部分，有的题要围绕“第一次失误”反向计数，有的题要抓住树上唯一传递链或把任意长度范围拆成少量可预处理的块。

## 9. [【20NOIP提高组】排水系统](http://ybt.ssoier.cn:8088/problem_show.php?pid=2012)

`NOIP 2020` `NOIP 提高组`

### 题意
给出一个没有环的排水系统。前 `m` 个结点是进水口，每个都注入 $1$ 吨水；每个结点会把当前收到的水平均分到所有出边上。要求输出所有出度为 $0$ 的结点最终流出的水量，结果以最简分数形式表示。

### 分析
因为整张图是 DAG，所以“水量传播”最自然的顺序就是拓扑序。一个结点只有等所有前驱都处理完，才会拿到完整的入流水量；而拓扑排序恰好保证这一点。

每个点的水量都要表示成分数。设 $water[u] = p/q$，如果 `u` 有 `d` 条出边，那么流向每个儿子的部分就是 $p/(qd)$。这批水要和儿子原来已经收到的分数做加法，因此需要写一个分数加法函数：先通分，再相加，最后用 `gcd` 约分。

这里最重要的不变量是：当一个点 `u` 出队时，`water[u]` 已经等于它最终会收到的全部水量。于是若它不是汇点，就把这份水平均分给所有后继，然后把这些后继的入度减一；某个后继入度减到 $0$ 时，说明它的所有上游都处理完了，可以安全出队。

整个过程和 DAG 上做 DP 非常像，只不过状态从整数和最值换成了分数。题目最后只要求输出汇点，所以那些出度为 $0$ 的结点在拓扑传递结束时，保存的就是最终答案。

### 核心代码
```cpp
struct Frac{ __int128 p, q; };
Frac add(Frac a, Frac b){
  __int128 g = gcd(a.q, b.q), l = a.q / g * b.q;
  Frac c{a.p * (l / a.q) + b.p * (l / b.q), l};
  __int128 d = gcd(c.p, c.q);
  return {c.p / d, c.q / d};
}
for(int i = 1; i <= m; i++) water[i] = {1, 1};
while(!q.empty()){
  int u = q.front(); q.pop();
  if(outdeg[u]){
    Frac part{water[u].p, water[u].q * outdeg[u]};
    for(int v : g[u]){
      water[v] = add(water[v], part);
      if(--indeg[v] == 0) q.push(v);
    }
    water[u] = {0, 1};
  }
}
```

### 复杂度
时间复杂度 $O(n+m)$，空间复杂度 $O(n+m)$。

## 10. [【20NOIP提高组】字符串匹配](http://ybt.ssoier.cn:8088/problem_show.php?pid=2013)

`NOIP 2020` `NOIP 提高组`

### 题意
给定字符串 `S`，要求统计形如 $S=(AB)^tC$ 的拆分方案数，其中 `A`、`B`、`C` 都非空，并且 `A` 中出现奇数次的字符种数不超过 `C` 中出现奇数次的字符种数。

### 分析
这题表面上是三段拆分，真正的核心却是两件事要分开做：

1. 前缀部分能不能写成若干个相同的 `AB` 块；
2. 一旦某个拆分点合法，满足 $F(A)\le F(C)$ 的 `A` 有多少种。

先看第二件事。因为字符集只有 `26` 个小写字母，所以“奇数字符种数”只会落在 $0$ 到 $26$。我们从左往右预处理 `pre[i]` 表示前缀 `S[1..i]` 的奇数字符种数，从右往左预处理 `suf[i]` 表示后缀 `S[i..n]` 的奇数字符种数。这样一来，如果当前 $|AB|=len$，那么所有可能的 `A` 长度就是 `1..len-1`，它们满足条件的个数只需要统计有多少个 $pre[x] \le suf[pos]$。

再看第一件事。设当前块长为 `len`，如果前缀要写成 $(AB)^t$，那就要求每一个长度为 `len` 的块都和开头那一块完全相同。这个“相同”可以用字符串哈希、Z 函数或 LCP 来 $O(1)$ 判断。于是我们固定 `len` 后，按 `len` 为步长往后跳；一旦某一块和首块不同，就立刻停止，因为更长的重复前缀已经不可能成立了。

枚举 `len` 时，把 `A` 的候选长度按顺序加入桶里；由于桶大小只有 `27`，查询 `pre[x] <= suf[pos]` 只要扫一遍小常数即可。总复杂度来自调和级数 $\sum_{len=1}^{n} n/len$，因此是 $O(n\log n)$ 级别。

### 核心代码
```cpp
long long ans = 0;
int cnt[27] = {};
for(int len = 2; len < n; len++){
  cnt[pre[len - 1]]++;
  for(int end = len; end < n; end += len){
    if(!same(1, end - len + 1, len)) break;
    for(int c = 0; c <= suf[end + 1]; c++) ans += cnt[c];
  }
}
```

### 复杂度
若 `same` 能在 $O(1)$ 判断，则总时间复杂度 $O(n\log n)$，空间复杂度 $O(n)$。

## 11. [【20NOIP提高组】移球游戏](http://ybt.ssoier.cn:8088/problem_show.php?pid=2014)

`NOIP 2020` `NOIP 提高组`

### 题意
有 `n+1` 根柱子，前 `n` 根一开始各装着 `m` 个球，第 `n+1` 根为空。球有 `n` 种颜色，每种颜色恰好出现 `m` 次。每次只能把某根柱子顶部的球移到另一根未满柱子的顶部。要求构造一组合法操作，把同色球最终归并到同一根柱子里。

### 分析
多出来的那根空柱不是“附赠条件”，而是整个构造能成立的关键缓冲区。因为每次只能动栈顶，如果没有额外空间，就很难把压在下面的目标颜色翻出来，更别说同时保证每根柱子不超容量。

一种非常稳的做法是分治。设当前要处理的颜色区间是 $[l,r]$，取中点 `mid`，把颜色分成左半段 $[l,mid]$ 和右半段 $[mid+1,r]$。这一层的任务不是直接把每种颜色都摆好，而是先完成更粗的一步：让左半段颜色全部落到左边那批柱子里，右半段颜色全部落到右边那批柱子里。

怎么分？不断观察各柱顶部的球：若它属于左半段，就把它倒向左侧预留区域；若属于右半段，就倒向右侧预留区域；当某个目标球被别的颜色压住时，就借助那根空柱先把挡路的球临时挪开。因为整层只区分“落在左边还是右边”，所以缓冲策略始终能维持，不会陷入死锁。等这一层完成后，左右两半颜色已经被彻底隔离开，再分别递归处理即可。

这类构造题最重要的不是把每一步都记死，而是先认出“层层划分颜色区间”的主框架。额外空柱负责局部腾挪，分治负责全局收束，合起来就能把总操作数压在限制内。

### 核心代码
```cpp
void moveTop(int x, int y){
  int c = tube[x].back();
  tube[x].pop_back();
  tube[y].push_back(c);
  ans.push_back({x, y});
}
void solve(int l, int r, int spare){
  if(l == r) return;
  int mid = (l + r) >> 1;
  partitionColor(l, r, mid, spare);
  solve(l, mid, spare);
  solve(mid + 1, r, spare);
}
```

### 复杂度
分治层数是 $O(\log n)$，每层做一次线性搬运，时间复杂度通常写作 $O(nm\log n)$，空间复杂度 $O(nm)$（含操作记录）。

## 12. [【20NOIP提高组】微信步数](http://ybt.ssoier.cn:8088/problem_show.php?pid=2015)

`NOIP 2020` `NOIP 提高组`

### 题意
在一个 $k$ 维整数网格盒子中，每个起点都会反复执行长度为 `n` 的移动序列，直到第一次走出边界。要求统计所有起点总共会走多少步；如果存在某个起点永远走不出去，则输出 `-1`。

### 分析
这题如果按“枚举起点再模拟”去做，会被 $\prod w_i$ 直接击穿。正确视角是看**路径对边界盒子的挤压**：对任意一个前缀，我们只关心每一维的前缀最小位移和最大位移，因为一个起点在走到这一步后仍然合法，当且仅当它平移出来的整段轨迹都还落在盒子里。

先处理“在第一轮内就出界”的起点。随着步骤推进，当前仍然合法的起点集合始终是一个高维长方体 $[L_1,R_1]\times\cdots\times[L_k,R_k]$。只有当某一步把某一维的前缀最小值或最大值刷新时，这个长方体才会真正缩小，也才会有一批新起点恰好在这一步出界。若缩小发生在第 $c_i$ 维，那么这批起点数量就是其余各维长度的乘积，因此贡献可以直接算出来。

接着看“能活过第一轮”的起点。设一整轮结束后第 `i` 维总位移为 $\Delta_i$。如果所有 $\Delta_i=0$，并且第一轮结束后长方体仍非空，那么这些起点以后每轮都会回到原状，必然死循环，答案就是 `-1`。否则，随着轮数增加，仍然合法的长方体在第 `i` 维的宽度会按 $|\Delta_i|$ 线性缩小。

于是后续统计就变成了计数问题：某个同余类步数在第 `t` 轮产生的贡献，是“步数 $(tn+offset)$”乘上“若干个关于 `t` 的一次式的乘积”。因为 $k\le 10$，这个乘积最多只是 $k$ 次多项式。把它展开后，就可以把贡献化成若干项 $\sum t^p$ 的线性组合，再用预处理好的幂和公式（通常用拉格朗日插值）一次性求完。

整题最难的地方，是把“高维路径”改写成“不断收缩的盒子”。一旦盒子视角建立起来，第一轮是直接计数，后续轮次是多项式求和，两部分逻辑就都清楚了。

### 核心代码
```cpp
int work(int a[]){
  int lim = INF;
  for(int i = 1; i <= k; i++) if(delta[i]) lim = min(lim, (a[i] + delta[i] - 1) / delta[i]);
  int poly[11] = {1};
  for(int i = 1; i <= k; i++)
    for(int j = i - 1; j >= 0; j--){
      poly[j + 1] = (poly[j + 1] - 1LL * delta[i] * poly[j]) % MOD;
      poly[j] = 1LL * a[i] * poly[j] % MOD;
    }
  int res = 0;
  for(int i = 0; i <= k; i++) res = (res + 1LL * poly[i] * sumPow(i, lim - 1)) % MOD;
  return (res + MOD) % MOD;
}
```

### 复杂度
预处理幂和后，整体时间复杂度 $O(nk^2)$，空间复杂度 $O(k^2)$（不计输入存储）。
