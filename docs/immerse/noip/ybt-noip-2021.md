# 一本通 2021赛题解题报告

这一年的题从取模与稳定排序这样直接的模型，一路走到区间 DP、双端构造、最小割与状态重排。前半段重点在把题面条件翻成可维护的判定或模拟，后半段则更强调先认出隐藏结构，再决定贪心、最短路、离线或动态规划应该落在哪一层。

# 一、CSP-J 赛题

这一组题虽然都在普及难度范围里，但每题都要求先把文字条件压成一个很干净的模型：有的落成余数最大化，有的落成稳定排序排名，有的落成严格格式判定，还有的要把整段水果改写成块链表上的逐轮消去。

## 1. [【21CSPJ普及组】分糖果（candy）](http://ybt.ssoier.cn:8088/problem_show.php?pid=2074)

`CSP 2021` `CSP-J`

### 题意

给出 $n,L,R$，在区间 $[L,R]$ 内选一个带回去的糖果数 $k$。分糖结束后剩下的 $k\bmod n$ 块就是奖励，要求把这个奖励尽量做大。

### 分析

题目一旦把发糖过程看成“每轮减去 $n$”，本质就只剩一句话：最后的奖励就是 $k\bmod n$。所以我们真正要做的，是在区间 $[L,R]$ 里找一个数，让它对 $n$ 取模后的余数尽量大。

如果 $L$ 和 $R$ 落在同一个长度为 $n$ 的商块里，也就是 $\lfloor L/n\rfloor=\lfloor R/n\rfloor$，那么区间内的余数是单调递增的，直接取最右端 $R$ 就最好，答案是 $R\bmod n$。

如果区间跨过了新的商块，就一定能碰到某个整除点前面的数，它的余数正好是 $n-1$，这已经是余数的理论上界了。于是只要判断区间有没有跨块，答案就立刻确定。

### 核心代码

```cpp
long long solve(long long n, long long L, long long R){
  if(L / n != R / n) return n - 1;
  return R % n;
}
```

### 复杂度

时间复杂度 $O(1)$，空间复杂度 $O(1)$。

## 2. [【21CSPJ普及组】插入排序（sort）](http://ybt.ssoier.cn:8088/problem_show.php?pid=2075)

`CSP 2021` `CSP-J`

### 题意

维护一个支持单点修改的数组。若对当前数组执行题面里的插入排序，询问原来的第 $x$ 个元素在稳定排序后的最终位置。

### 分析

插入排序这段代码只有在 `a[j] < a[j-1]` 时才交换，因此它排出来的其实就是一个**稳定升序序列**。这一步想清楚以后，询问“原来的第 $x$ 个元素最后在哪”就不再和排序过程本身有关了，而是变成：有多少元素严格小于它，再加上多少与它相等但原下标不超过 $x$。

如果没有修改，直接把所有元素按二元组 $(a_i,i)$ 排序，位置就是答案。难点只在于有修改操作，但题目又给了一个很关键的限制：类型 $1$ 的操作总数不超过 $5000$，而数组长度只有 $8000$。这意味着我们完全可以把每次修改后的整张排名表重建一次。

具体做法就是：每次修改后，把所有二元组 `(值, 原下标)` 重新排序，得到新的稳定顺序，再把每个下标对应的排名写入 `rk[i]`。之后每个询问直接输出 `rk[x]` 即可。这里真正利用的是“修改少”，而不是去硬撑一个复杂数据结构。

### 核心代码

```cpp
void rebuild(){
  vector<pair<int,int>> ord;
  for(int i = 1; i <= n; i++) ord.push_back({a[i], i});
  sort(ord.begin(), ord.end());
  for(int i = 0; i < n; i++) rk[ord[i].second] = i + 1;
}
if(op == 1){
  a[x] = v;
  rebuild();
}else{
  cout << rk[x] << '\n';
}
```

### 复杂度

设修改次数为 $U$，总时间复杂度 $O(U n\log n+Q)$，单次询问 $O(1)$，额外空间复杂度 $O(n)$。

## 3. [【21CSPJ普及组】网络连接(network)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2076)

`CSP 2021` `CSP-J`

### 题意

依次处理若干台机器的 `Server/Client` 操作。地址串必须严格符合 `a.b.c.d:e` 规范，服务机成功建连后还要支持客户机按地址查找最早的服务机编号。

### 分析

这题的主体不是哈希，而是**格式判定必须做细**。地址串合法需要同时满足四件事：分隔符顺序必须是 `a.b.c.d:e`，五段都非空；四个 IP 段在 $[0,255]$ 内，端口在 $[0,65535]$ 内；每段只能由数字组成；除单独的 `0` 外不允许前导零。只要其中一条不满足，这次操作就应直接输出 `ERR`。

格式过关以后，服务机和客户机的逻辑都很直接。服务机如果地址第一次出现，就建立成功并把“地址 $\to$ 编号”记下来；若已有成功服务机占用了同地址，就输出 `FAIL`。客户机则只需要在映射表里查这个地址是否已经被某台服务机成功建立。

所以整题最值得形成的习惯是：先把字符串校验函数单独写干净，再把“合法地址的后续业务逻辑”交给字典处理。这样不会把判错、重名和查找三种情况搅在一起。

### 核心代码

```cpp
bool okField(const string& t, int lim){
  if(t.empty() || (t.size() > 1 && t[0] == '0')) return false;
  long long x = stoll(t);
  return x <= lim;
}
bool parse(const string& s){
  vector<int> p;
  for(int i = 0; i < (int)s.size(); i++) if(s[i] == '.' || s[i] == ':') p.push_back(i);
  if(p.size() != 4 || s[p[3]] != ':' || s[p[0]] != '.' || s[p[1]] != '.' || s[p[2]] != '.') return false;
  vector<string> seg = {s.substr(0,p[0]), s.substr(p[0]+1,p[1]-p[0]-1), s.substr(p[1]+1,p[2]-p[1]-1), s.substr(p[2]+1,p[3]-p[2]-1), s.substr(p[3]+1)};
  return okField(seg[0],255) && okField(seg[1],255) && okField(seg[2],255) && okField(seg[3],255) && okField(seg[4],65535);
}
```

### 复杂度

设地址串长度上界为 $C$，总时间复杂度 $O(nC)$，额外空间复杂度 $O(nC)$。

## 4. [【21CSPJ普及组】小熊的果篮(fruit)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2077)

`CSP 2021` `CSP-J`

### 题意

把一排苹果和桔子按轮装入果篮：每轮都从每个当前同色块中同时取出最左边一个水果，空块删除后若左右同色则继续合并，要求输出每轮取出的原下标。

### 分析

直接按数组删点会反复搬移元素，真正应该抓住的是“块”这个层次。把整串压成若干个同色块后，一轮操作等价于：从当前链表上的每个块里弹出一个最左位置，组成这一轮的果篮。

关键细节在于**合并发生在一整轮之后**。因为题目说的是“每一个块中最左边的水果同时挑出”，所以本轮内即使某个块被取空，也不能立刻让左右新合并出的块再多取一次；必须等本轮全部块都弹完，才删除空块并检查相邻同色块是否需要合并。

实现时，把每个块的所有位置存在一个链表里，再用双向链表维护块与块之间的左右关系。每轮从左到右扫描活动块、弹出块首元素即可；若某块空了，就把它从块链表中删掉；若删完后左右颜色相同，就用 `splice` 把右块剩余位置接到左块后面。每个水果只会被弹出一次，整体就是线性的。

### 核心代码

```cpp
while(head){
  vector<int> basket, dead;
  for(int u = head; u; u = nxt[u]){
    basket.push_back(pos[u].front());
    pos[u].pop_front();
    if(pos[u].empty()) dead.push_back(u);
  }
  print(basket);
  for(int u : dead) if(alive[u]){
    int L = pre[u], R = nxt[u];
    eraseBlock(u);
    if(L && R && color[L] == color[R]){
      pos[L].splice(pos[L].end(), pos[R]);
      eraseBlock(R);
    }
  }
}
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

# 二、CSP-S 赛题

提高组部分开始明显转向结构化建模：调度题要先把资源分配拆成两个独立函数，计数题要先认出文法或进位，构造与图论题则要先找到真正的状态空间，再决定是做区间 DP、对偶最短路，还是离线并查集维护。

## 5. [【21CSPS提高组】廊桥分配(airport)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2078)

`CSP 2021` `CSP-S`

### 题意

一共有 $n$ 个廊桥，要在国内区和国际区之间分配。每类航班都按先到先得占桥，要求最大化最终停靠廊桥的飞机总数。

### 分析

第一步要把“大问题”拆成两个互不干扰的小问题：如果给国内区分到 $x$ 个廊桥、给国际区分到 $n-x$ 个廊桥，那么答案就是“国内最多停多少”加“国际最多停多少”。所以整题变成预处理两个数组 `f[x]`、`g[x]`，最后枚举切分点取 `f[x]+g[n-x]` 的最大值。

怎么求一类航班在有 $x$ 个廊桥时的最优值？题目已经把策略固定成“先到先得”：飞机到达时，只要有空桥就必须立刻分配，不存在更高层的选择。因此我们只需要忠实模拟。把航班按到达时间排序，维护一个“正在使用的廊桥”小根堆和一个“空闲廊桥编号”小根堆；每来一架飞机，先释放所有离开时间不晚于当前到达时刻的廊桥，再尝试分给编号最小的空闲桥。

真正精妙的是为什么一次模拟就能得到所有 `f[x]`。给足 $n$ 个廊桥去跑时，每架成功停靠的航班都会落到某个桥编号上，记 `cnt[i]` 为第 $i$ 座桥服务了多少架飞机。由于每次都优先使用最小可用编号，所以只保留前 $x$ 座桥时，能接住的恰好就是编号不超过 $x$ 的那些航班，总数就是 `cnt` 的前缀和。这样两个方向各算一遍，再拼起来即可。

### 核心代码

```cpp
void calc(vector<Flight> f, vector<int>& pre){
  sort(f.begin(), f.end());
  priority_queue<pair<int,int>, vector<pair<int,int>>, greater<pair<int,int>>> busy;
  priority_queue<int, vector<int>, greater<int>> freeId;
  for(int i = 1; i <= n; i++) freeId.push(i);
  vector<int> cnt(n + 1);
  for(auto [l, r] : f){
    while(!busy.empty() && busy.top().first <= l) freeId.push(busy.top().second), busy.pop();
    if(freeId.empty()) continue;
    int id = freeId.top();
    freeId.pop();
    cnt[id]++;
    busy.push({r, id});
  }
  for(int i = 1; i <= n; i++) pre[i] = pre[i - 1] + cnt[i];
}
```

### 复杂度

时间复杂度 $O((m_1+m_2+n)\log n)$，空间复杂度 $O(n)$。

## 6. [【21CSPS提高组】括号序列(bracket)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2079)

`CSP 2021` `CSP-S`

### 题意

给定一个含 `(`、`)`、`*`、`?` 的字符串，`?` 可任意替换，要求统计能变成合法超级括号序列的方案数，其中每段连续 `*` 的长度不能超过 $k$。

### 分析

这题最难的不是区间 DP 这四个字，而是**怎么拆状态才能既不重又不漏**。因为合法串的文法里既有外层包一对括号，也有串与串的拼接，还允许中间夹一段星号串 `S`。如果只写一个 `f[l][r]` 表示“整段是否合法”，做拼接时很容易把 `AB` 和 `ASB` 混在一起重复计数。

比较自然的拆法是把“星号块”和“外层括号块”分开。设 `star[l][r]` 表示这一段全是 `*` 且长度不超过 $k$；`wrap[l][r]` 表示这一段恰好由一对最外层括号包起来；`left[l][r]` 表示“合法串后面接一段星号块”；`right[l][r]` 对称；`full[l][r]` 表示任意合法超级括号串。这样每种状态都对应一个明确的文法片段，转移就不会互相踩。

`wrap` 的来源最直接：两端能填成括号时，里面可以是空串、纯星号块，或者一个已经合法的片段。`left/right` 则是把一个 `wrap` 与一个 `star` 拼起来。至于 `full`，可以把“最左边那对最外层括号”钦定出来：前半段必须是一个 `wrap`，后半段则是一个完整合法串，或者一个“合法串后跟星号块”的形态。这样分裂点是唯一的，就不会重复。

所以这题真正要练的不是记模板，而是把文法改写成若干个可维护的子语言。只要状态含义清楚，剩下就是标准的按区间长度递推。

### 核心代码

```cpp
for(int len = 1; len <= n; len++){
  for(int l = 1, r = l + len - 1; r <= n; l++, r++){
    if(len <= k && canStar(r)) star[l][r] = (len == 1 ? 1 : star[l][r - 1]);
    if(canL(l) && canR(r)){
      long long mid = (l + 1 == r ? 1 : (star[l + 1][r - 1] + full[l + 1][r - 1] + left[l + 1][r - 1] + right[l + 1][r - 1]) % MOD);
      wrap[l][r] = mid % MOD;
    }
    for(int p = l; p < r; p++){
      left[l][r] = (left[l][r] + wrap[l][p] * star[p + 1][r]) % MOD;
      right[l][r] = (right[l][r] + star[l][p] * wrap[p + 1][r]) % MOD;
      full[l][r] = (full[l][r] + wrap[l][p] * (full[p + 1][r] + left[p + 1][r])) % MOD;
    }
    full[l][r] = (full[l][r] + wrap[l][r]) % MOD;
  }
}
```

### 复杂度

时间复杂度 $O(n^3)$，空间复杂度 $O(n^2)$。

## 7. [【21CSPS提高组】回文（palin）](http://ybt.ssoier.cn:8088/problem_show.php?pid=2080)

`CSP 2021` `CSP-S`

### 题意

序列里每个数恰好出现两次。每次只能从两端取数并追加到新序列末尾，要求最后得到回文，并输出字典序最小的 `L/R` 操作串。

### 分析

这题最关键的观察是：回文的第一个数一旦确定，它在原序列里的另一个副本就必须去对称位置，也就是最后一个数。于是全局上只有两种起手：第一步取左端，或者第一步取右端。除此之外，不需要再枚举更深的分支。

假设先取左端 $a_1$，设它的另一个位置是 $p$。这样最外层这一对已经被钉死，剩下的序列会被切成两个区间 $[2,p-1]$ 和 $[p+1,2n]$。之后我们每走一步，其实都是在这两个区间的四个端点里做匹配：若下一次放进回文左半边的是某个端点值，那么回文右半边对应位置就必须从另外三个端点里找同值项。于是每轮只剩四种配对关系要试。

模拟时始终优先尝试能让答案当前位置写成 `L` 的方案，这样若整条路径能走通，就自动得到当前起手下的字典序最小解。把“左起手”和“右起手”各跑一次，取其中较小的可行串即可；若两边都失败，答案就是 $-1$。题面看上去像构造搜索，实际上靠的是把第一对对称元素先钉死，之后就退化成线性模拟。

### 核心代码

```cpp
bool buildLeft(int p, string& ans){
  int l1 = 2, r1 = p - 1, l2 = p + 1, r2 = 2 * n;
  string L = "L", R = "L";
  while(l1 <= r1 || l2 <= r2){
    if(l1 <= r1 && a[l1] == a[r1]) L += 'L', R += 'L', l1++, r1--;
    else if(l1 <= r1 && l2 <= r2 && a[l1] == a[l2]) L += 'L', R += 'R', l1++, l2++;
    else if(l2 <= r2 && l1 <= r1 && a[l2] == a[r1]) L += 'R', R += 'L', l2++, r1--;
    else if(l2 <= r2 && a[l2] == a[r2]) L += 'R', R += 'R', l2++, r2--;
    else return false;
  }
  ans = L + string(R.rbegin(), R.rend());
  return true;
}
```

### 复杂度

单次尝试是 $O(n)$，总时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 8. [【21CSPS提高组】交通规划(traffic)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2081)

`CSP 2021` `CSP-S`

### 题意

给网格边和若干边界附加点赋权。每次询问给定一些边界点的黑白颜色，要求给所有格点染黑白，使异色边权和最小。

### 分析

把“异色边权和最小”翻译成图论语言，就是要求一个最小割。之所以这题不像普通最小割那样直接建源汇，是因为每次询问固定颜色的点不止两个，而且它们都落在外边界。这里真正该认出的结构是**平面图对偶**：在平面图里，一条割边集合对应对偶图里的一组路径。

如果只有两个边界端点，那么答案就是对偶图上连接它们的最短路长度。多于两个端点时，外边界上的黑白点按环形顺序排开。先把相邻同色的点缩掉之后，剩下的颜色一定交替出现；任意可行的割在对偶图里对应若干条**互不相交的配对路径**，而这种配对在圆周上一定是非交叉的。于是问题就被压成了：端点之间的代价是对偶图最短路，整体方案是一个非交叉配对。

这样就可以先对本次询问真正出现的边界点各跑一次 Dijkstra，得到两两代价，再按圆周顺序做区间 DP。设 `dp[l][r]` 表示解决这一段端点的最小代价，转移要么让 `l` 和某个与它异色的位置直接配对，要么把区间拆成两半。由于所有询问的端点总数不超过 $50$，这种“最短路 + 小规模区间 DP”的组合完全可行。

### 核心代码

```cpp
for(int i = 0; i < k; i++) dijkstra(dualId[i], dist[i]);
for(int len = 2; len <= k; len += 2){
  for(int l = 0; l + len - 1 < k; l++){
    int r = l + len - 1;
    dp[l][r] = INF;
    if(color[l] != color[r]) dp[l][r] = min(dp[l][r], dp[l + 1][r - 1] + dist[l][dualId[r]]);
    for(int m = l + 1; m < r; m += 2)
      dp[l][r] = min(dp[l][r], dp[l][m] + dp[m + 1][r]);
  }
}
```

### 复杂度

设单次询问的附加点数为 $k$，则时间复杂度约为 $O(k\,nm\log(nm)+k^3)$，空间复杂度 $O(nm)$。

# 三、NOIP 赛题

这一章开始是 2021 年的 NOIP 提高组赛题。和前面的 CSP 相比，这几题更强调把抽象出来的模型继续往前推：有的题要把奇偶结构拆成可贪心的两部分，有的题要围绕“第一次失误”反向计数，有的题要抓住树上唯一传递链或把任意长度范围拆成少量可预处理的块。

## 9. [【21NOIP提高组】报数](http://ybt.ssoier.cn:8088/problem_show.php?pid=2082)

`NOIP 2021` `NOIP 提高组`

### 题意

一个数不能报出，当且仅当它含有某个“十进制里带 7 的数”作为因子。给出很多个 $x$，若 $x$ 本身非法输出 $-1$，否则输出下一个合法数。

### 分析

如果按询问逐个判断因子，会被 $2\times 10^5$ 组询问直接拖垮。这题应该反过来想：先把整个值域里哪些数“不能报”一次性筛出来，再用查询去读答案。

核心是一个类埃氏筛的过程。顺序枚举 `i` 时，若它已经被标成非法，就不用再管；若它还没被标记，而且十进制里含有 $7$，那它就会让所有倍数都非法，于是把 $i,2i,3i,\dots$ 全部标掉。为什么这样不会漏？因为任何非法数都至少有一个含 $7$ 的因子，枚举到那个最小因子时它就会被覆盖。

预处理完“非法表”后，再倒着扫一遍，维护每个位置后面的下一个合法数 `nxt[i]`。这样回答询问时只分两种情况：`bad[x]` 为真就输出 `-1`，否则直接输出 `nxt[x]`。这正是典型的“重预处理、轻查询”。

### 核心代码

```cpp
for(int i = 1; i <= MAXV; i++){
  if(bad[i]) continue;
  if(has7(i)){
    for(int j = i; j <= MAXV; j += i) bad[j] = true;
  }
}
int last = -1;
for(int i = MAXV; i >= 1; i--){
  nxt[i] = last;
  if(!bad[i]) last = i;
}
```

### 复杂度

预处理时间复杂度约为 $O(M\log M)$，单次询问 $O(1)$，空间复杂度 $O(M)$，其中 $M$ 是预处理上界。

## 10. [【21NOIP提高组】数列](http://ybt.ssoier.cn:8088/problem_show.php?pid=2083)

`NOIP 2021` `NOIP 提高组`

### 题意

长度为 $n$ 的序列每个元素都在 $[0,m]$ 内，权值是对应 $v_{a_i}$ 的乘积。要求统计所有满足 $\sum 2^{a_i}$ 的二进制中 $1$ 的个数不超过 $k$ 的序列权值和。

### 分析

直接按序列做 DP 会被排列数淹没。这题真正该数的是：每个指数 $i$ 被选了多少次。若某一位选了 $t$ 次，它对总和 $S=\sum 2^{a_i}$ 的贡献就是往第 $i$ 位加上 $t$ 个 $1$，而序列内部有多少种排法，则由组合数 $C(\text{剩余位置},t)$ 统一处理。

难点在于“二进制里 $1$ 的个数”并不等于各位出现次数之和，因为会产生进位。所以状态必须把进位带着走。设 `dp[i][used][ones][carry]` 表示处理到权值 $v_i$ 之前，已经放了 `used` 个数，已经确定的低位中有 `ones` 个 `1`，当前还向更高位留下了 `carry`。若当前位置再选 $t$ 个值为 $i$ 的元素，那么这一位最终贡献出的新 $1$ 个数是 $(carry+t)\bmod 2$，新的进位是 $\lfloor(carry+t)/2\rfloor$。

同时别忘了把“选法数”和“权值积”乘进去：新增贡献要乘 $C(n-used,t)\cdot v_i^t$。位数全部处理完以后，`carry` 里还残留着更高位的信息，再额外看一次 `popcount(carry)` 是否让总 $1$ 数超过 $k$。这题最值得掌握的，就是把“进位”从麻烦事改写成 DP 里的一个显式状态。

### 核心代码

```cpp
dp[0][0][0][0] = 1;
for(int i = 0; i <= m; i++)
  for(int used = 0; used <= n; used++)
    for(int ones = 0; ones <= k; ones++)
      for(int carry = 0; carry <= n; carry++) if(dp[i][used][ones][carry])
        for(int t = 0; used + t <= n; t++){
          int nOnes = ones + ((carry + t) & 1);
          int nCarry = (carry + t) >> 1;
          add(dp[i + 1][used + t][nOnes][nCarry], dp[i][used][ones][carry] * C[n - used][t] % MOD * pw[i][t]);
        }
```

### 复杂度

时间复杂度约为 $O(mn^4)$，空间复杂度约为 $O(n^3)$；由于 $n\le 30$，这样的进位 DP 可以承受。

## 11. [【21NOIP提高组】方差](http://ybt.ssoier.cn:8088/problem_show.php?pid=2084)

`NOIP 2021` `NOIP 提高组`

### 题意

给定一个非降序列，操作可把某个中间项变成左右邻居之和减去自身。要求经过任意次操作后，使方差最小，并输出最小方差乘 $n^2$ 的结果。

### 分析

这题最核心的转化是差分。设 $d_i=a_{i+1}-a_i$，那么把 $a_i$ 变成 $a_{i-1}+a_{i+1}-a_i$，会让相邻两个差分 $d_{i-1},d_i$ 直接交换位置。也就是说，题目的所有操作本质上只是把差分数组做任意重排。

于是我们不再盯着原序列，而是盯着重排后的前缀和。把常数平移去掉以后，可以把最终序列看成 $0,p_1,p_2,\dots,p_{n-1}$，其中 $p_j$ 是某个差分排列的前缀和。要最小化的量正是 $n\sum p_i^2-(\sum p_i)^2$。进一步可以证明，最优差分排列一定是单谷形：大的差分应该尽量放到两端，小的差分放到中间。

有了这个结构，就可以先把所有正差分从小到大排序，然后依次决定“当前这个差分放到谷形的左端，还是右端”。设 $f[i][s]$ 表示已经放了前 $i$ 个差分、当前所有前缀和的总和为 $s$ 时，前缀平方和的最小值。若新差分 $x$ 放到左端，那么现有的 $i$ 个前缀和都会整体加上 $x$；若放到右端，只会新添一个值为当前总差分和的前缀。这两个动作都能写出显式转移。

还有一个关键省状态的点：零差分怎么放都不影响答案，可以直接丢掉。再利用题目给的 $a_i\le 600$，就知道有效差分个数和总差分和都很小，DP 状态空间是可控的。

### 核心代码

```cpp
sort(d.begin() + 1, d.begin() + m + 1);
for(int i = 1; i <= m; i++) pre[i] = pre[i - 1] + d[i];
f[0][0] = 0;
for(int i = 0; i < m; i++){
  for(int s = 0; s <= LIM; s++) if(f[i][s] < INF){
    long long x = d[i + 1];
    chmin(f[i + 1][s + i * x], f[i][s] + 1LL * i * x * x + 2LL * s * x);
    chmin(f[i + 1][s + pre[i + 1]], f[i][s] + 1LL * pre[i + 1] * pre[i + 1]);
  }
}
for(int s = 0; s <= LIM; s++) ans = min(ans, 1LL * n * f[m][s] - 1LL * s * s);
```

### 复杂度

去掉零差分后，设总差分和为 $V=a_n-a_1$，时间复杂度约为 $O(V^3)$，空间复杂度约为 $O(V^2)$。

## 12. [【21NOIP提高组】棋局](http://ybt.ssoier.cn:8088/problem_show.php?pid=2085)

`NOIP 2021` `NOIP 提高组`

### 题意

棋盘边有三种可走道路：一步路、直行路、互通路。每次新放一枚棋子，要求统计只用这枚新棋子走一步完整合法走子，能到达多少个位置。

### 分析

这题最先要做的不是上数据结构，而是把“走法必须全用同一种道路”拆开来看。道路类型 $1,2,3$ 的规则完全不同，所以答案也应该分三部分求并集：普通道路最多只走一条边，直行道路只能在同一方向连走，互通道路则是在一个连通块里任意转弯。

类型 $1$ 最简单，四个方向各判一次即可。类型 $2$ 的难点在于不能每次都沿一整条线扫过去，否则最坏会炸到平方级。这里通常按行和按列维护“下一个仍可继续走的位置”或并查集跳指针，让新棋子沿着某个方向直接跳到下一个关键点：要么遇到第一枚挡路棋子，要么遇到道路断开。这样一条直线上的处理就接近均摊线性。

类型 $3$ 才是真正的大头。因为题目是不断**加棋子**，这意味着空点构成的互通块会被切碎，正向维护很难；标准做法是把所有操作倒过来做，把“加棋子”变成“删棋子”，于是互通块只会不断合并。这样就能用并查集维护类型 $3$ 的连通块，再给每个块配一棵线段树或集合，维护其中当前仍为空的位置，以及按颜色和等级还能否吃到的边界点。查询时先分别统计三类道路的可达点，再用时间戳或哈希去重。

所以这题的教学价值不在某个单一技巧，而在于先按规则分模型，再针对三类模型分别选最合适的维护方式：一步直接判，直线用跳指针，互通块用离线并查集。

### 核心代码

```cpp
int solve(int id){
  vis.clear();
  for(int d = 0; d < 4; d++) if(type1(id, d) == 1) relax(id, step(id, d), vis);
  for(int d = 0; d < 4; d++) if(type1(id, d) == 2) walkStraight(id, d, vis);
  collectType3(find3(id), col[id], lv[id], vis);
  place(id);
  return vis.size();
}
```

### 复杂度

结合跳指针、离线并查集与集合维护后，总时间复杂度可做到 $O((nm+q)\log(nm))$ 量级，空间复杂度 $O(nm)$。
