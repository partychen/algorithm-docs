# 一本通 NOIP 2015赛题解题报告

这一组题从普及组的顺手模拟、规律观察一路走到提高组的树上二分、函数图成环搜索与多维动态规划，跨度很大，但主线很清楚：先把题面条件翻译成能直接维护的量，再围绕状态、单调性或结构性质把复杂决策压成可计算的模型。

# 一、普及组赛题

普及组四题都不靠重型算法，关键在于先把叙述拆成可执行的规则。读题时要先分清哪些条件决定了枚举方式，哪些条件决定了统计公式，这样代码自然会顺下来。

## 1. [【15NOIP普及组】金币](http://ybt.ssoier.cn:8088/problem_show.php?pid=1969)
`NOIP 2015` `普及组`

### 题意

工资按分段规律发放：先连续 `1` 天每天拿 `1` 枚，再连续 `2` 天每天拿 `2` 枚，再连续 `3` 天每天拿 `3` 枚，依次类推。给出前 `K` 天，求总金币数。

### 分析

题面已经把规律说得很完整，第一反应就应该是“按块模拟”。第 `x` 段会持续 `x` 天，每天贡献 `x` 枚金币，所以这一整段一共贡献 `x*x`。但最后一段未必能完整取完，因此更稳妥的写法是维护当前该发多少金币 `v`，以及这一档还要发几天 `cnt`。

这里的关键状态其实只有两个：当前档位 `v`，以及还剩多少天 `k` 没处理。每次拿出 `min(k,v)` 天，这一段对答案的贡献就是 `take*v`，然后把 `k` 减掉，再让 `v` 自增进入下一档。这样做对，是因为工资规则天然按连续区间分层，任意一天都恰好属于唯一一档，不会重不漏。

答案提取也很直接：把每一档的贡献累加到 `ans`，直到 `k=0` 为止，最终 `ans` 就是前 `K` 天的总金币数。

### 核心代码

```cpp
long long solve(int k){
  long long ans = 0;
  for(long long v = 1; k > 0; v++){
    long long take = min<long long>(k, v);
    ans += take * v;
    k -= take;
  }
  return ans;
}
```

### 复杂度

时间复杂度 `O(√K)`，空间复杂度 `O(1)`。

## 2. [【15NOIP普及组】扫雷游戏](http://ybt.ssoier.cn:8088/problem_show.php?pid=1970)
`NOIP 2015` `普及组`

### 题意

给出一个 `n×m` 的雷区，`*` 表示地雷，`?` 表示空格。要求输出整个棋盘：地雷位置仍输出 `*`，其余位置输出周围八个方向相邻格中的地雷个数。

### 分析

题目本质是局部统计。每个非雷格子的值只和自己周围八个邻居有关，因此建模时不需要全局算法，只要把“八个方向”翻译成方向数组即可。看到这种“上下左右加四个对角”的描述，最稳的思路就是准备 `dx, dy`，对每个格子循环八次。

不变量是：当我们处理到格子 `(i,j)` 时，只统计边界内且确实是地雷的邻居。边界判断是这题最容易漏的地方，所以要把“是否合法坐标”写成统一条件。对于地雷格，不做任何统计，直接保留 `*`；对于非雷格，从八个方向累计得到数字字符即可。

方法成立的原因在于题目要求的数字定义就是“相邻八格中地雷数”，没有连锁展开、没有搜索传播，所以每个答案都可以独立计算。答案提取就是把每个位置算出来后直接写入结果矩阵并输出。

### 核心代码

```cpp
int dx[8]={-1,-1,-1,0,0,1,1,1};
int dy[8]={-1,0,1,-1,1,-1,0,1};
for(int i=0;i<n;i++) for(int j=0;j<m;j++){
  if(g[i][j]=='*'){ out[i][j]='*'; continue; }
  int cnt=0;
  for(int k=0;k<8;k++){
    int x=i+dx[k], y=j+dy[k];
    if(0<=x&&x<n&&0<=y&&y<m&&g[x][y]=='*') cnt++;
  }
  out[i][j]=char('0'+cnt);
}
```

### 复杂度

时间复杂度 `O(nm)`，空间复杂度 `O(nm)`。

## 3. [【15NOIP普及组】求和](http://ybt.ssoier.cn:8088/problem_show.php?pid=1971)
`NOIP 2015` `普及组`

### 题意

纸带上有 `n` 个位置，每个位置有颜色 `color_i` 和数值 `number_i`。要统计所有满足 `x<y<z`、`y-x=z-y` 且 `color_x=color_z` 的三元组分数之和，分数为 `(x+z)(number_x+number_z)`，答案对 `10007` 取模。

### 分析

先从条件出发。`y-x=z-y` 说明 `x` 和 `z` 关于 `y` 对称，所以 `x` 与 `z` 的奇偶性必然相同；换句话说，我们真正需要在“同颜色、同奇偶”的位置集合里做配对。把题目从三元组改写成“选两个端点，中点自然确定”，模型就清晰了。

设当前处理位置为右端点 `z=i`。若左端点 `x` 与它同色且同奇偶，那么中点 `y=(x+i)/2` 一定是整数，而且自动满足 `x<y<i`。于是每一对合法端点都会贡献 `(x+i)(number_x+number_i)`。把式子展开：`x*number_x + x*number_i + i*number_x + i*number_i`，其中对固定的 `i` 来说，只需要知道此前同颜色同奇偶位置的个数、下标和、数值和、下标乘数值和。

这就是本题的核心状态：按“颜色 + 奇偶”分类维护四个前缀量 `cnt、sumPos、sumNum、sumPosNum`。扫描到 `i` 时，先用这些统计量一次算出所有以 `i` 为右端点的贡献，再把 `i` 自己加入对应桶里。之所以可行，是因为每一对端点只会在较右端点被处理时计算一次，既不遗漏也不重复。

答案提取就是把每个位置作为右端点得到的贡献累加并随时对 `10007` 取模，最后输出总和。

### 核心代码

```cpp
int p = i & 1, c = color[i], v = number[i] % MOD;
ans = (ans + sumPosNum[c][p]) % MOD;
ans = (ans + 1LL * v * sumPos[c][p]) % MOD;
ans = (ans + 1LL * i * sumNum[c][p]) % MOD;
ans = (ans + 1LL * i * v % MOD * cnt[c][p]) % MOD;
cnt[c][p]++;
sumPos[c][p] = (sumPos[c][p] + i) % MOD;
sumNum[c][p] = (sumNum[c][p] + v) % MOD;
sumPosNum[c][p] = (sumPosNum[c][p] + 1LL * i * v) % MOD;
```

### 复杂度

时间复杂度 `O(n)`，空间复杂度 `O(m)`。

## 4. [【15NOIP普及组】推销员](http://ybt.ssoier.cn:8088/problem_show.php?pid=1972)
`NOIP 2015` `普及组`

### 题意

一条死胡同里有 `N` 家住户，第 `i` 家距离入口为 `S_i`，推销它可获得疲劳值 `A_i`。若要向若干住户推销，阿明从入口出发，走到所选住户中最远的一家后原路返回，因此走路疲劳值固定为最远距离的两倍。对每个 `X=1..N`，求恰好向 `X` 家住户推销时的最大总疲劳值。

### 分析

先把目标拆开：若最远访问到第 `i` 家，那么走路贡献就是 `2S_i`，而被选住户里必须包含第 `i` 家，其余还要从前 `i-1` 家中再选 `X-1` 家，纯粹为了让 `A` 之和最大。这样题目就从“路线”转成了“枚举最远点 + 取若干最大收益”。

进一步看，最优方案其实只会落在两类情形里。第一类是“最远那家本身就在全局收益最大的前 `X` 家里”，那就直接取这 `X` 家，总代价是“这 `X` 家的 `A` 总和 + 其中最远下标的 `2S`”。第二类是“最远那家不在全局前 `X` 大里”，那它只能作为额外补进来的那一户，其余 `X-1` 家就一定选全局前 `X-1` 大收益；此时为了让它真的是最远住户，它的下标必须在这 `X-1` 家的最远位置之后。

于是核心状态就是三组量：按 `A` 降序取前缀得到的收益和 `sum[k]`，这些前缀元素中的最远位置 `mx[k]`，以及从每个位置开始的后缀最大值 `suf[i]=max_{j≥i}(2S_j+A_j)`。第一类方案贡献 `sum[X] + 2S_{mx[X]}`；第二类方案贡献 `sum[X-1] + suf[mx[X-1]+1]`。贪心成立的根本原因是：一旦区分出“最远住户是否属于收益前 `X` 大”，其余住户的选择就只剩下按 `A` 取最大值。

答案提取时，对每个 `X` 直接取上述两类方案的较大值即可，逐个输出 `ans[X]`。

### 核心代码

```cpp
sort(id+1,id+n+1,[](int x,int y){ return a[x]>a[y]; });
for(int k=1;k<=n;k++){
  sum[k]=sum[k-1]+a[id[k]];
  mx[k]=max(mx[k-1], id[k]);
}
for(int i=n;i>=1;i--) suf[i]=max(suf[i+1], 2LL*s[i]+a[i]);
for(int k=1;k<=n;k++){
  long long v1=sum[k]+2LL*s[mx[k]];
  long long v2=sum[k-1]+suf[mx[k-1]+1];
  ans[k]=max(v1,v2);
}
```

### 复杂度

时间复杂度 `O(n log n)`，空间复杂度 `O(n)`。

# 二、提高组赛题

提高组开始，难点不再是把规则照着写出来，而是先识别隐藏结构：树上路径、函数图中的环、可行性单调、搜索状态压缩、二维到三维的动态规划。只要模型认准，后面的状态设计就有据可依。

## 5. [【15NOIP提高组】运输计划](http://ybt.ssoier.cn:8088/problem_show.php?pid=1892)
`NOIP 2015` `提高组`

### 题意

给一棵带边权的树和 `m` 个运输计划，每个计划都走两点间最短路。可以把恰好一条边改成长度 `0`，所有计划同时开始，问所有计划完成时间的最大值最小能变成多少。

### 分析

这题第一步要从“优化一条边”想到“二分答案”。设最终最大完成时间不超过 `mid`，那么原本长度大于 `mid` 的计划必须都经过那条被改成虫洞的边，而且这条边的边权至少要覆盖它们超出的部分。于是问题变成：是否存在一条边，同时被所有超标路径经过，并且其边权不少于 `max(dist-mid)`。

为了判断“哪些边被所有超标路径经过”，要把路径覆盖统计压到树上。先用 `LCA` 求出每个计划原始路径长度 `dist`。对所有 `dist>mid` 的计划，在两端点各加一，在 `lca` 处减二，随后做一次树上后序累加。这样某条父边对应的子树计数 `cnt[x]`，就表示有多少条超标路径经过这条边。这正是树上差分的核心不变量。

若共有 `tot` 条超标路径，那么只要存在某个结点 `x` 满足 `cnt[x]=tot`，说明父边 `(fa[x],x)` 被所有超标路径共同经过；再检查这条边权 `w[x]` 是否至少为 `mx-mid`，其中 `mx` 是超标计划中的最大原长度。如果满足，就能把这条边变成虫洞，让每条超标路径至少减少这条边权，从而全部压到 `mid` 以内。方法成立的原因正是“所有计划的最短路唯一”，树上每条路径的经过关系可被差分准确还原。

答案提取是对 `mid` 二分最小可行值。每次调用检查函数，最终左端点就是最短完成时间。

### 核心代码

```cpp
bool check(int lim){
  fill(tag+1, tag+n+1, 0); int tot=0, need=0;
  for(auto [u,v,d]:q) if(d>lim){
    int p=lca(u,v); tag[u]++; tag[v]++; tag[p]-=2;
    tot++; need=max(need,d-lim);
  }
  int ok=0;
  dfs2(1,0,tot,need,ok);
  return ok;
}
void dfs2(int u,int f,int tot,int need,int &ok){
  for(auto [v,w]:g[u]) if(v!=f){
    dfs2(v,u,tot,need,ok); tag[u]+=tag[v];
    if(tag[v]==tot && w>=need) ok=1;
  }
}
```

### 复杂度

时间复杂度 `O((n+m)log n+(n+m)log W)`，空间复杂度 `O(n+m)`。

## 6. [【15NOIP提高组】神奇的幻方](http://ybt.ssoier.cn:8088/problem_show.php?pid=1887)
`NOIP 2015` `提高组`

### 题意

给定奇数 `N`，按题目规定的四条放置规则，从 `1` 到 `N×N` 依次填入矩阵，输出最终构造出的幻方。

### 分析

虽然这题挂在提高组，但本质还是规则模拟。关键不是记四句话，而是把它们统一成“优先走右上，越界时环绕；如果右上已占，就改走正下方”。题目列出的四种情况，其实就是这个统一规则拆开的边界版。

所以建模时维护当前数字 `k` 所在位置 `(x,y)`。下一步先尝试候选位置 `nx=x-1, ny=y+1`，若越过上边界就回到最后一行，若越过右边界就回到第一列，这就是环绕。若这个候选格还没填数，就把 `k+1` 放进去；否则只能走到 `(x+1,y)`。这里的不变量是：每一步恰好选择一个空格，且已填数字始终保持题目要求的生成顺序。

之所以这样统一后仍正确，是因为题面四条规则已经把所有位置关系分完了，而“右上优先，否则下移”正好覆盖这四类。答案提取无需额外处理，矩阵填完后按行输出即可。

### 核心代码

```cpp
int x=0,y=n/2; a[x][y]=1;
for(int v=2;v<=n*n;v++){
  int nx=(x-1+n)%n, ny=(y+1)%n;
  if(!a[nx][ny]) x=nx,y=ny;
  else x=(x+1)%n;
  a[x][y]=v;
}
```

### 复杂度

时间复杂度 `O(N^2)`，空间复杂度 `O(N^2)`。

## 7. [【15NOIP提高组】信息传递](http://ybt.ssoier.cn:8088/problem_show.php?pid=1888)
`NOIP 2015` `提高组`

### 题意

有 `n` 个人，每人只把自己当前知道的信息传给固定的一个对象 `T_i`。问最少经过多少轮，第一次有人听到自己的生日信息。

### 分析

每个点只有一条出边，这就是典型函数图。信息一轮一轮沿出边传播，某个人第一次听到自己的生日，等价于某条信息沿着若干次传递后回到起点。因此答案其实就是函数图中最短有向环长度。

从条件到模型的关键一步是：树枝部分的信息只会一路流向环，永远不可能先回到自己；真正决定答案的只有环。于是我们只要找出所有环，并取最小长度。常见做法有拓扑删叶，也可以直接用时间戳 DFS。对每个未访问点沿着出边走，记录它在本轮搜索中的进入时间 `step[u]` 与所属轮次 `bel[u]`。若走到一个同属当前轮次的点，就说明找到了一个新环，环长是当前深度减去它第一次出现的深度。

这里的不变量是：在同一次链式追踪中，路径上的点顺序就是信息传播顺序；一旦遇到本轮已出现点，闭合部分恰好构成一个简单环。方法成立，是因为函数图每点出度为一，路径结构只能是“链 + 环”。

答案提取就是把所有检测到的环长取最小值，最后输出这个最小环长度。

### 核心代码

```cpp
for(int s=1;s<=n;s++) if(!vis[s]){
  int u=s, dep=0;
  while(!vis[u]){
    vis[u]=1; bel[u]=s; step[u]=dep++;
    u=to[u];
  }
  if(bel[u]==s) ans=min(ans, dep-step[u]);
}
```

### 复杂度

时间复杂度 `O(n)`，空间复杂度 `O(n)`。

## 8. [【15NOIP提高组】斗地主](http://ybt.ssoier.cn:8088/problem_show.php?pid=1889)
`NOIP 2015` `提高组`

### 题意

给出若干手牌。允许按题目规定的牌型出牌，目标是用最少的出牌次数把整手牌打完。需要对每组数据输出最少步数。

### 分析

这题最难的不是写牌型，而是先抓住状态规模。牌面种类有限，真正重要的是每个点数剩几张，所以可以把手牌压成计数数组 `cnt[]`。之后问题变成：每一步从若干合法牌型里选一种，把对应计数减掉，最少多少步清空。这就是标准的深搜最优化模型。

为什么要优先搜顺子、连对、飞机这类连续牌型？因为它们覆盖面大，最能减少总步数，也是搜索树里最有价值的决策。实际做法通常是迭代加深或普通 DFS + 当前最优上界：先枚举所有可连续伸展的牌型进行递归；当这些“大块”都处理完后，再对剩余牌做一个贪心估价，例如统计四张、三张、对子、单牌的最少附带打法，得到当前状态还能用几步收尾。

本题的关键不变量是计数数组始终准确描述“还没打掉的牌”，每次递归只做合法减法；剪枝条件 `step + heuristic >= ans` 保证无望分支直接停止。之所以剪枝有效，是因为估价函数给出的是剩余状态的乐观下界，而不会把真实最优值估大。

答案提取就是在搜索过程中不断更新全局最优 `ans`，当计数数组清空或只能被收尾函数处理时，用当前步数加收尾代价更新答案。

### 核心代码

```cpp
void dfs(int step){
  if(step + calc() >= ans) return;
  for(int l=3;l<=14;l++){
    for(int r=l;r<=14&&cnt[r];r++) if(r-l+1>=5){
      for(int i=l;i<=r;i++) cnt[i]--;
      dfs(step+1);
      for(int i=l;i<=r;i++) cnt[i]++;
    }
  }
  ans=min(ans, step + calc());
}
```

### 复杂度

时间复杂度与状态分布相关，最坏为指数级；空间复杂度 `O(牌面种类数)`。

## 9. [【15NOIP提高组】跳石头](http://ybt.ssoier.cn:8088/problem_show.php?pid=1890)
`NOIP 2015` `提高组`

### 题意

河道长度为 `L`，中间有 `N` 块石头，最多移走 `M` 块。希望移走后，从起点到终点相邻落脚点之间的最小跳跃距离尽量大，求这个最大值。

### 分析

一看到“最大化最小值”，就要优先考虑二分答案。设我们猜最小跳跃距离至少为 `mid`，那就要判断能否通过删除不超过 `M` 块石头，使所有相邻保留点间距都不少于 `mid`。

检查函数的思路是贪心：从起点开始，记最后一块保留石头位置为 `last`。顺次扫描每块石头，如果当前石头到 `last` 的距离小于 `mid`，那么它必须删掉；否则就保留并把 `last` 更新为它。这个贪心为什么正确？因为当间距不够时，在当前石头和之前保留点之间，删掉当前石头一定不比删掉之前保留点更差：保留更靠后的石头只会让后续可用空间更大。

关键状态是不变量 `last`：它始终表示在当前前缀内、为了满足最小间距要求而保留下来的最后一块石头位置。全部扫描后，还要检查终点与 `last` 的距离；若也小于 `mid`，说明还得再删一块。若总删除数 `<=M`，则 `mid` 可行。

答案提取就是对 `mid` 做二分，找到最大的可行值，这个值就是最优最小跳跃距离。

### 核心代码

```cpp
bool check(int d){
  int last=0, del=0;
  for(int i=1;i<=n;i++){
    if(pos[i]-last<d) del++;
    else last=pos[i];
  }
  if(L-last<d) del++;
  return del<=m;
}
```

### 复杂度

时间复杂度 `O(N log L)`，空间复杂度 `O(N)`。

## 10. [【15NOIP提高组】子串](http://ybt.ssoier.cn:8088/problem_show.php?pid=1891)
`NOIP 2015` `提高组`

### 题意

从字符串 `A` 中取出 `k` 个互不重叠的非空子串，保持它们在 `A` 中的相对顺序后拼接，要求结果恰好等于字符串 `B`。求方案数，答案对 `1000000007` 取模。

### 分析

题目同时限制了“已经匹配到 `B` 的哪里”和“已经开了多少段”，所以模型一定是多维 DP。难点在于子串必须连续，因此光记“前 `i` 个字符匹配前 `j` 个字符、用了 `t` 段”还不够，还要知道当前位置是不是正处在一段被选中的子串内部。

于是设计两个状态：`f[i][j][t]` 表示考虑 `A` 前 `i` 个字符，拼出 `B` 前 `j` 个字符，用了 `t` 段，且第 `i` 个字符不一定被选作当前段结尾的总方案数；`g[i][j][t]` 表示同样条件下，并且第 `i` 个字符必须被选中、作为某一段的结尾。若 `A[i]==B[j]`，那么 `g` 可以由两种方式转移来：一是把前一个被选字符继续延长，来自 `g[i-1][j-1][t]`；二是从当前位置新开一段，来自 `f[i-1][j-1][t-1]`。随后 `f[i][j][t]=f[i-1][j][t]+g[i][j][t]`，表示当前字符可以不用，也可以恰好用到这里结束。

这里的关键转移就体现了“连续子串”这个约束：只有 `g→g` 才表示段内延续，`f→g` 才表示新开一段。也正因为把“是否处于段尾”单独拆出来，互不重叠和保持顺序两个条件才都被状态准确记录。

答案提取是 `f[n][m][k]`，因为处理完整个 `A` 后，只要已经拼出整个 `B` 且总共用了 `k` 段，无论最后一段是否正好在末尾结束，都已经包含在总状态里。

### 核心代码

```cpp
for(int i=1;i<=n;i++){
  f[i][0][0]=1;
  for(int j=1;j<=m;j++) for(int t=1;t<=k;t++){
    if(A[i]==B[j])
      g[i][j][t]=(g[i-1][j-1][t]+f[i-1][j-1][t-1])%MOD;
    else g[i][j][t]=0;
    f[i][j][t]=(f[i-1][j][t]+g[i][j][t])%MOD;
  }
}
```

### 复杂度

时间复杂度 `O(nmk)`，空间复杂度 `O(nmk)`，滚动数组可降到 `O(mk)`。
