# 一本通 NOIP 2000赛题解题报告

这一年的题从一元方程解析、价格决策一路走到负进制转换、双路径动态规划，前半段更强调把题面条件拆成可计算的式子，后半段则进一步要求把路径、划分与搜索状态压成稳定的转移。阅读时可以先抓“式子怎么立”，再看“状态怎么定”。

# 一、普及组赛题

这一组题更像建模入门：有的把字符串方程翻成系数与常数，有的把利润比较改写成不等式区间，也有把划分、接龙压成动态规划与深度优先搜索状态。重点不是技巧多，而是先把题面语言翻成统一的计算对象。

## 1. [【00NOIP普及组】计算器的改良](http://ybt.ssoier.cn:8088/problem_show.php?pid=1910)
`NOIP 2000` `普及组`

### 题意

给出一个只含整数、加减号、等号和单个未知数的一元一次方程，要求求出这个未知数的实数解，并按三位小数输出。

### 分析

这题最关键的一步，是把整条等式统一看成 $a\times x+b=0$。题面虽然写成普通字符串，但因为没有括号、没有除法，所有项都只是“常数项”或“系数乘未知数”，所以扫描一遍就能把左边、右边的量全部折到同一个线性表达式里。
做法上，把等号左侧记为正、右侧记为负；再把每一段按 `+`、`-` 切成项。若一项里带字母，它就贡献到未知数系数 $a$；若不带字母，它就贡献到常数项 $b$。像 `x`、`-y` 这种没写数字的项，系数要当成 $1$ 或 $-1$。整个过程中不需要真的“移项”，因为扫描累计本身就在完成移项。
当所有项都处理完以后，就得到标准形式 $a\times x+b=0$，答案直接是 $x=-\frac{b}{a}$。这里的核心不变量就是：任意时刻累计出的 $a$ 和 $b$，都等价于“已扫描部分整理到左边以后”的系数与常数。

### 核心代码

```cpp
pair<long long,long long> parse(string t){
  int p = 0, sgn = 1;
  if(t[p] == '+') p++;
  else if(t[p] == '-') sgn = -1, p++;
  long long num = 0;
  while(p < t.size() && isdigit(t[p])) num = num * 10 + t[p++] - '0';
  if(p < t.size() && isalpha(t[p])) return {sgn * (num ? num : 1), 0};
  return {0, sgn * num};
}

void addSide(string s, int side, long long& a, long long& b){
  int st = 0;
  for(int i = 1; i <= s.size(); i++) if(i == s.size() || s[i] == '+' || s[i] == '-') {
    auto [x, c] = parse(s.substr(st, i - st));
    a += side * x; b += side * c; st = i;
  }
}
addSide(left, 1, a, b);
addSide(right, -1, a, b);
printf("%.3f", -1.0 * b / a);
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(1)$。

## 2. [【00NOIP普及组】税收与补贴问题](http://ybt.ssoier.cn:8088/problem_show.php?pid=1911)
`NOIP 2000` `普及组`

### 题意

已知成本、若干整数价位的销量以及高价区间的线性递减规律，要求判断最少收税或补贴多少，才能让政府预期价成为总利润最大的价位。

### 分析

题目表面在问“收多少税、补多少贴”，真正难点却是先把利润比较写成同一个变量控制的不等式。设政府给每件商品的调节额为 $s$，取正表示补贴、取负表示收税，那么商家在价格 $p$ 处的总利润就是 $((p-c)+s)\times q(p)$，其中 $q(p)$ 是该价位的销量。
因此，想让预期价格 $t$ 成为最优，只需要让它对所有其他整数价格 $p$ 都满足 $((t-c)+s)q(t)\ge((p-c)+s)q(p)$。整理以后得到 $s\,(q(t)-q(p))\ge (p-c)q(p)-(t-c)q(t)$。这说明每个候选价格都会给出一个关于 $s$ 的下界、上界或无解判断，整题就转化成求所有不等式交出来的整数区间。
剩下的工作是先把每个整数价格对应的销量补全出来：已知点之间按线性变化插值，超过最高价后按固定速度递减，直到销量降到零为止。若最终区间为空，则不存在方案；否则在区间里取绝对值最小的整数即可。这里的核心状态不是某个动态规划数组，而是当前可行调节额区间 $[L,R]$。

### 核心代码

```cpp
for(int p = cost; p <= lastPrice && sale[p] > 0; p++){
  long long A = sale[target] - sale[p];
  long long B = 1LL * (p - cost) * sale[p] - 1LL * (target - cost) * sale[target];
  if(A == 0){
    if(B > 0) ok = false;
  }else if(A > 0){
    L = max(L, ceil_div(B, A));
  }else{
    R = min(R, floor_div(B, A));
  }
}
if(!ok || L > R) puts("NO SOLUTION");
else if(L <= 0 && 0 <= R) cout << 0;
else if(R < 0) cout << R;
else cout << L;
```

### 复杂度

若补全后的整数价位共有 $m$ 个，时间复杂度 $O(m)$，空间复杂度 $O(m)$。

## 3. [【00NOIP普及组】乘积最大](http://ybt.ssoier.cn:8088/problem_show.php?pid=1912)
`NOIP 2000` `普及组`

### 题意

给定一个长度为 $N$ 的数字串，要插入 $K$ 个乘号把它分成 $K+1$ 段，求这些整数段乘积的最大值。

### 分析

乘号要放在什么位置，看起来像在枚举所有切法，实际上更适合改成“前缀怎么分”。设 `f[i][k]` 表示前 `i` 位数字里已经放了 `k` 个乘号时，能够得到的最大乘积，那么最后一段一定是某个区间 `j+1..i`，前面则对应 `f[j][k-1]`。
于是转移就很自然：`f[i][k]=max(f[j][k-1]×val[j+1][i])`。这里 `val[l][r]` 表示数字串第 `l` 到第 `r` 位组成的整数，可以预处理出来，这样每次转移都只是在“最后一刀切在哪里”之间取最优。题目真正要练的是：把一个看似全局的乘法最大化问题，拆成“前缀最优值 + 最后一段”的结构。
因为数字串长度能到 `40`，最终答案会远超普通整型范围，所以状态值要用高精度整数保存。核心状态就是 `f[i][k]`：它始终代表“前 `i` 位恰好切成 `k+1` 段后的最优乘积”，答案直接取 `f[n][K]`。

### 核心代码

```cpp
cpp_int val[45][45], f[45][7];
for(int l = 1; l <= n; l++)
  for(int r = l; r <= n; r++)
    val[l][r] = val[l][r - 1] * 10 + s[r] - '0';
for(int i = 1; i <= n; i++) f[i][0] = val[1][i];
for(int k = 1; k <= K; k++)
  for(int i = k + 1; i <= n; i++)
    for(int j = k; j < i; j++)
      f[i][k] = max(f[i][k], f[j][k - 1] * val[j + 1][i]);
cout << f[n][K];
```

### 复杂度

时间复杂度 $O(Kn^2)$，空间复杂度 $O(n^2+Kn)$。

## 4. [【00NOIP普及组】单词接龙](http://ybt.ssoier.cn:8088/problem_show.php?pid=1913)
`NOIP 2000` `普及组`

### 题意

给定若干单词和一个起始字母，要求在每个单词最多使用两次、且相邻部分不能互相包含的条件下，求最长单词龙的长度。

### 分析

题目要求的是“接出来的总长度最大”，所以先别急着真的拼字符串，而是先想清楚一次连接会增加多少长度。若单词 `a` 的后缀和单词 `b` 的前缀能重合 `k` 个字符，那么把 `b` 接到后面时，长度只会增加 `|b|-k`。于是问题被压成一张有向图：点是单词，边表示可以相接，边权表示新增长度。
这里要特别注意两条限制：每个单词最多用两次；相邻两部分不能包含关系。第二条其实正好体现在重合长度的枚举里——重合长度必须严格小于两个单词长度的较小值，这样就不会出现一个单词完整包含在另一边的非法拼接。
预处理出任意两个单词间最大的合法重合长度后，直接深度优先搜索即可。搜索状态是“当前停在哪个单词、已经得到的总长度、每个单词用了几次”。因为每个点最多进两次，树虽然会分叉，但规模仍可控。答案在整棵搜索树上维护最大长度。

### 核心代码

```cpp
for(int i = 1; i <= n; i++)
  for(int j = 1; j <= n; j++)
    for(int k = 1; k < min(w[i].size(), w[j].size()); k++)
      if(w[i].substr(w[i].size() - k) == w[j].substr(0, k)) ov[i][j] = k;

void dfs(int u, int len){
  ans = max(ans, len);
  used[u]++;
  for(int v = 1; v <= n; v++)
    if(used[v] < 2 && ov[u][v])
      dfs(v, len + (int)w[v].size() - ov[u][v]);
  used[u]--;
}
for(int i = 1; i <= n; i++) if(w[i][0] == st) dfs(i, w[i].size());
```

### 复杂度

预处理重合长度为 $O(n^2L^2)$，搜索在题目数据范围内可通过，额外空间复杂度 $O(n^2)$。

# 二、提高组赛题

提高组延续同样的题型，但要求更稳定的状态设计与更完整的推导：既要处理负进制这种不直观表示，也要把两次走路改写成双人同步的三维动态规划。读这部分时，建议重点体会“原题怎么重述成更适合转移的过程”。

## 5. [【00NOIP提高组】进制转换](http://ybt.ssoier.cn:8088/problem_show.php?pid=1820)
`NOIP 2000` `提高组`

### 题意

给定一个十进制整数和一个负进制基数 $-R$，要求把这个整数转换成对应的负进制表示，并在大于 $9$ 的数码处使用大写字母。

### 分析

负进制最容易卡住的地方，是“除法余数怎么定义”。如果照普通进制那样直接不断除以基数，遇到负基数时余数可能会变成负数，就不再是合法数码了。正确思路是仍然做“除基取余”，但每一步都把余数调整回 $0$ 到 $|R|-1$ 的范围。
设基数为 `b(<0)`。先取 `r=n%b`，若 `r` 已经非负，它就能直接作为当前最低位；若 `r<0`，就把它加上 `|b|`，同时把商多补 `1`，因为我们实际上把 `n=qb+r` 改写成了 `n=(q+1)b+(r-b)`，而 `-b=|b|`。这样处理后，每一位都合法，且不会破坏原数值。
整个过程和普通进制一样，按从低位到高位依次生成数码，最后逆序输出即可。这里的不变量是：当前 `n` 始终等于“尚未处理的高位部分”，而已经收集的余数序列就是原数在负进制下的低位到高位表示。

### 核心代码

```cpp
string convert(int n, int b){
  if(n == 0) return "0";
  string s;
  while(n){
    int r = n % b;
    n /= b;
    if(r < 0) r -= b, n++;
    s += (r < 10 ? char('0' + r) : char('A' + r - 10));
  }
  reverse(s.begin(), s.end());
  return s;
}
```

### 复杂度

时间复杂度 $O(\log_{|R|}|N|)$，空间复杂度 $O(\log_{|R|}|N|)$。

## 6. [【00NOIP提高组】乘积最大](http://ybt.ssoier.cn:8088/problem_show.php?pid=1821)
`NOIP 2000` `提高组`

### 题意

给定长度为 $N$ 的数字串，要放入 $K$ 个乘号，使其被分成 $K+1$ 个整数段，并让这些整数段的乘积最大。

### 分析

虽然题面和普及组同名，但提高组里更值得强化的是状态含义。把数字串前 `i` 位切成 `k+1` 段后得到的最大乘积，天然就是一个前缀最优值；而最后一段只可能是 `j+1..i` 这一整块，所以决策点只有“最后一次切在什么位置”。
一旦认出这一点，转移式就固定下来：前面取 `f[j][k-1]`，最后一段取 `val[j+1][i]`，两者相乘即可。预处理所有子串对应的数值以后，三重循环枚举 `k、i、j` 就能把答案完整推出来。这个模型最重要的不是公式，而是理解“最后一刀”把全局方案拆成两个独立部分。
由于答案可能非常大，状态必须用高精度整数维护。于是 `f[i][k]` 的意义、`val[l][r]` 的含义以及最终输出 `f[n][K]` 这三件事要完全对齐，程序才不会在实现时跑偏。

### 核心代码

```cpp
cpp_int val[45][45], f[45][7];
for(int l = 1; l <= n; l++)
  for(int r = l; r <= n; r++)
    val[l][r] = val[l][r - 1] * 10 + s[r] - '0';
for(int i = 1; i <= n; i++) f[i][0] = val[1][i];
for(int k = 1; k <= K; k++)
  for(int i = k + 1; i <= n; i++)
    for(int j = k; j < i; j++)
      f[i][k] = max(f[i][k], f[j][k - 1] * val[j + 1][i]);
cout << f[n][K];
```

### 复杂度

时间复杂度 $O(Kn^2)$，空间复杂度 $O(n^2+Kn)$。

## 7. [【00NOIP提高组】单词接龙](http://ybt.ssoier.cn:8088/problem_show.php?pid=1822)
`NOIP 2000` `提高组`

### 题意

给定一组单词和起始字母，要求在每个单词最多用两次、且相邻部分不能互相包含的条件下，求出最长单词龙的长度。

### 分析

这题的关键不是“把龙真的拼出来”，而是抓住每次接一个新单词时，答案只会增加“新单词长度减去重合部分”。因此只要预处理出每一对单词之间能形成的最大合法重合长度，就已经把题目抽成了图上的搜索。
合法重合有两层含义：后一个单词前缀要和前一个单词后缀相同；重合长度不能等于某个单词的全长，否则就会出现题目禁止的包含关系。于是边是否存在、边的增量是多少，都能在预处理阶段一次性算出来。
之后从所有首字母满足要求的单词出发做深度优先搜索。状态中最重要的是每个单词已被使用的次数，因为题目只允许每个单词最多出现两次。搜索过程中维护当前总长度，扩展一条边时只加上新增部分，整棵搜索树里的最大值就是答案。

### 核心代码

```cpp
for(int i = 1; i <= n; i++)
  for(int j = 1; j <= n; j++)
    for(int k = 1; k < min(w[i].size(), w[j].size()); k++)
      if(w[i].substr(w[i].size() - k) == w[j].substr(0, k)) ov[i][j] = k;

void dfs(int u, int len){
  ans = max(ans, len);
  used[u]++;
  for(int v = 1; v <= n; v++)
    if(used[v] < 2 && ov[u][v])
      dfs(v, len + (int)w[v].size() - ov[u][v]);
  used[u]--;
}
for(int i = 1; i <= n; i++) if(w[i][0] == st) dfs(i, w[i].size());
```

### 复杂度

预处理重合长度为 $O(n^2L^2)$，搜索在题目范围内可以接受，额外空间复杂度 $O(n^2)$。

## 8. [【00NOIP提高组】方格取数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1823)
`NOIP 2000` `提高组`

### 题意

在一个 $N\times N$ 方格图中，某些格子有权值。要求从左上到右下走两次，每次只能向右或向下，并让两条路径取得的数字总和最大。

### 分析

“从左上走到右下两次”这一说法，很容易让人去想第一次走完后再算第二次，但那样会把路径间的相互影响拆散。更好的重述方式是：看成两个人同时从左上走到右下，每一步都只能向右或向下，并且如果两人走到同一个格子，这个格子的数只能算一次。这样两次路径的耦合关系就被放进了同一个状态里。
设 `f[k][x1][x2]` 表示两个人都恰好走了 `k-2` 步时，第一个人在 `(x1, y1)`、第二个人在 `(x2, y2)` 的最大取数和，其中 `y1=k-x1`、`y2=k-x2`。因为每个人本步只可能从上边或左边走来，所以转移只需看四种前驱组合。
加分时要小心重合：若两人落在同一格，只能加一次该格权值；否则加两个格子的值。整个动态规划的核心不变量是：`f[k][x1][x2]` 永远代表“在相同步数层上，两条前缀路径已经共同取得的最大价值”。最后取 `f[2n][n][n]` 就是答案。

### 核心代码

```cpp
memset(f, -0x3f, sizeof f);
f[2][1][1] = a[1][1];
for(int k = 3; k <= 2 * n; k++)
  for(int x1 = 1; x1 <= n; x1++)
    for(int x2 = 1; x2 <= n; x2++){
      int y1 = k - x1, y2 = k - x2;
      if(y1 < 1 || y1 > n || y2 < 1 || y2 > n) continue;
      int add = a[x1][y1] + (x1 == x2 && y1 == y2 ? 0 : a[x2][y2]);
      f[k][x1][x2] = max(max(f[k-1][x1][x2], f[k-1][x1-1][x2]),
                         max(f[k-1][x1][x2-1], f[k-1][x1-1][x2-1])) + add;
    }
cout << f[2 * n][n][n];
```

### 复杂度

时间复杂度 $O(n^3)$，空间复杂度 $O(n^3)$；若滚动数组可降到 $O(n^2)$。
