# 一本通 NOIP 2009赛题解题报告

这一套题很适合按“先把条件翻清楚，再决定工具”的顺序来练。普及组先把格式模拟、排序规则、质因数拆解和时间分段决策讲透；提高组再把一一映射、数论逆向、图上最优转移和强约束搜索接起来，正好形成一条从读题建模到综合实现的完整训练线。

# 一、普及组赛题

普及组四题看上去类型不同，但真正反复出现的动作很统一：先抓住题面里那个必须始终正确的量，再围绕它写转移。格式题维护“当前项该怎么输出”，排序题维护“同分时谁更靠前”，数论题维护“每个质因子还差多少”，动态规划题维护“这一段时间选哪个机器人最值”。

## 1. [【09NOIP普及组】多项式输出](http://ybt.ssoier.cn:8088/problem_show.php?pid=1945)
`NOIP 2009` `普及组`

### 题意

给出一个 $n$ 次多项式从高次到低次的各项系数，按题目规定输出标准形式：零系数项跳过，首项与后续项的正负号规则不同，次数为 $0$、$1$、大于 $1$ 时写法也不同。

### 分析

这题表面是输出，其实核心是把“每一项什么时候省略、什么时候补符号”拆成几条互不冲突的规则。最稳的想法不是整串字符串一起讨论，而是把每一项单独格式化：给定系数 $c$ 和次数 $e$，只回答“这一项该长什么样”。

先看哪些项根本不该出现：系数为 $0$ 的项直接跳过。剩下的项再分三层处理。第一层是**符号**：如果它是整串里的第一项，正号不能写，负号要写；如果不是第一项，正号写 `+`，负号写 `-`。第二层是**系数绝对值**：当次数大于 $0$ 且系数绝对值为 $1$ 时，数字 `1` 要省掉，只保留符号和变量；常数项不能省。第三层是**次数部分**：$e=0$ 只写常数，$e=1$ 写成 `x`，$e>1$ 写成 `x^e`。

这样做为什么稳？因为整道题没有全局优化，只有局部格式规则；把每一项独立转成字符串，再按次数从大到小拼接，天然就满足输出顺序。这里的不变量是：答案串始终等于“已经处理过的所有非零项的正确拼接结果”。

### 核心代码

```cpp
string build(int c,int e,bool first){
    if(c==0) return "";
    string s;
    if(c<0) s+='-';
    else if(!first) s+='+';
    int v=abs(c);
    if(e==0) return s+to_string(v);
    if(v!=1) s+=to_string(v);
    s+='x';
    if(e>1) s+="^"+to_string(e);
    return s;
}
string ans;
for(int i=0;i<=n;i++){
    string cur=build(a[i],n-i,ans.empty());
    if(!cur.empty()) ans+=cur;
}
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(1)$，若把输出串计入则为 $O(L)$。

## 2. [【09NOIP普及组】分数线划定](http://ybt.ssoier.cn:8088/problem_show.php?pid=1946)
`NOIP 2009` `普及组`

### 题意

有 $n$ 个选手，每人有报名号和分数。先按成绩从高到低、同分按报名号从小到大排序；面试线取排序后第 `m*3/2` 名的分数，其中乘法后按整数除法向下取整；所有分数不低于面试线的人都进入面试。输出分数线、人数和名单。

### 分析

这题最关键的是把“排名第 `m*3/2` 名”翻成**排完序以后数组中的一个位置**。一旦顺序固定，分数线就不再模糊：它就是第 `k=m*3/2` 个选手的成绩。这里的 `/2` 是整数除法，所以天然就是向下取整。后面之所以人数可能变多，只是因为这个分数线会把所有同分选手一起放进来。

所以整道题只需要两步。第一步，把所有人按题目要求排序。这里排序规则就是不变量：任何时刻都必须满足“分高的在前，同分号小的在前”，因为后面所有判断都建立在这个顺序上。第二步，取出第 $k$ 名的分数 `line`，然后从前往后扫到最后一个 `score >= line` 的位置为止。

为什么这样一定对？因为排序后，分数不低于分数线的人一定形成一个前缀；一旦遇到第一个低于分数线的人，后面的人只会更低，不可能再被录取。答案提取也就非常直接：第一行输出 `line` 和这个前缀长度，随后原样输出这个前缀。

### 核心代码

```cpp
sort(a+1,a+n+1,[](Node x,Node y){
    if(x.s!=y.s) return x.s>y.s;
    return x.id<y.id;
});
int k=m*3/2;
int line=a[k].s;
int cnt=k;
while(cnt<n && a[cnt+1].s==line) cnt++;
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(n)$ 或排序所需辅助空间。

## 3. [【09NOIP普及组】细胞分裂](http://ybt.ssoier.cn:8088/problem_show.php?pid=1947)
`NOIP 2009` `普及组`

### 题意

有 $N$ 种细胞，第 $i$ 种细胞每秒会把数量乘成 $S_i$ 倍。试管总数是 $M=m_1^{m_2}$。从一个细胞开始培养，问最少多少秒后，细胞总数能被 $M$ 整除；若所有细胞都做不到，输出 $-1$。

### 分析

“能平均分入 $M$ 个试管”本质上就是**细胞总数能被 $M$ 整除**。如果选第 $i$ 种细胞，培养 $t$ 秒后总数是 $S_i^t$，于是问题变成：最小的 $t$ 使得 $M\mid S_i^t$。

整除问题最适合拆到质因数层面。把 $M$ 分解成
`M = p_1^{c_1} p_2^{c_2} ... p_k^{c_k}`，其中每个指数其实是“$m_1$ 中该质因子的次数，再整体乘上 $m_2$”。再看某个 $S_i$，设它含有质因子 $p_j$ 的次数为 $d_j$，那么 $S_i^t$ 中这个质因子的次数就是 $t d_j$。为了让 $M\mid S_i^t$，必须对每个 $j$ 都满足 `t * d_j >= c_j`。

于是每个质因子都会给出一个下界 `t >= ceil(c_j / d_j)`，真正可行的最小时间就是这些下界的最大值。如果某个必须的质因子在 $S_i$ 里根本不存在，也就是 `d_j = 0`，那无论培养多久都补不出来，这种细胞直接判无解。

这里的关键状态就是“每个必要质因子还缺多少指数”，关键不变量是：只要所有质因子的指数需求都被满足，整除关系就已经成立；不需要真的去算巨大的 $S_i^t$。最后在所有细胞的最小可行时间里取最小值即可。

### 核心代码

```cpp
vector<pair<int,long long>> need;
void factor(int x,long long mul){
    for(int p=2;p*p<=x;p++) if(x%p==0){
        long long c=0;
        while(x%p==0) x/=p,c++;
        need.push_back({p,c*mul});
    }
    if(x>1) need.push_back({x,mul});
}
long long calc(long long s){
    long long t=0;
    for(auto [p,req]:need){
        long long c=0,x=s;
        while(x%p==0) x/=p,c++;
        if(c==0) return INF;
        t=max(t,(req+c-1)/c);
    }
    return t;
}
```

### 复杂度

设 $m_1$ 的不同质因子个数为 $k$。预处理复杂度为 $O(\sqrt{m_1})$，每种细胞判断复杂度约为 $O(k\log S_i)$，总复杂度约为 $O(\sqrt{m_1}+Nk\log S_i)$。

## 4. [【09NOIP普及组】道路游戏](http://ybt.ssoier.cn:8088/problem_show.php?pid=1948)
`NOIP 2009` `普及组`

### 题意

环上有 $n$ 个工厂和 $n$ 段路。第 $i$ 段路在第 $t$ 个单位时间会出现给定数量的金币；在工厂 $i$ 买一个机器人要花给定代价。任意时刻路上至多有一个机器人，每个机器人要预先设定走的步数，范围是 $1$ 到 $p$。机器人会从购买工厂出发顺时针走，每走一步就收走当前经过那段路在这一时刻的全部金币。问经过恰好 $m$ 个单位时间后，净收益最大是多少。

### 分析

先把题面拆成两层决策。外层决策是：把总共 $m$ 个时间单位切成若干段，每一段交给一个机器人去走，段长在 $1$ 到 $p$ 之间。内层决策是：如果这一段从时间 $l$ 开始、持续 `len` 步，那么应该从哪个工厂买机器人，才能让这 `len` 步的金币和减去购买代价最大。

于是自然得到时间动态规划。记 `f[t]` 为刚好用掉前 $t$ 个时间单位后能得到的最大净收益。那么只要我们预处理出 `best[l][len]`——表示“从第 $l$ 个时间开始，连续走 `len` 步时，单个机器人能做到的最大净收益”——就有转移
`f[t+len] = max(f[t+len], f[t] + best[t+1][len])`。

难点在 `best` 的计算。若从工厂 `i` 出发，在时间 `l` 开始走，依次会经过路 `i, i+1, ...`，同时对应时间是 `l, l+1, ...`。这正是一条沿着“路编号和时间一起加一”的对角线。于是可以先做一张对角线后缀和表 `diag[i][t]`，表示从“第 `i` 段路、第 `t` 时刻”出发，沿顺时针对角线一直走到游戏结束能收到多少金币。这样某个起点、某个长度的收益就能用两次后缀和相减在 $O(1)$ 内拿到，再减去购买工厂代价。

这套做法为什么成立？因为机器人一旦买下，中途不会转向、不会停顿，所以一段方案唯一对应一条环上的时间对角线；而总收益又正好等于各段收益之和减去各次购买花费，所以“分段最优值 + 时间 DP”就是完整模型。答案提取是 `f[m]`。

### 核心代码

```cpp
for(int t=m;t>=1;t--)
    for(int i=1;i<=n;i++)
        diag[i][t]=gold[i][t]+diag[i%n+1][t+1];
for(int l=1;l<=m;l++){
    for(int i=1;i<=n;i++){
        for(int len=1;len<=p && l+len-1<=m;len++){
            int r=l+len-1;
            int nx=(i+len-1)%n+1;
            int sum=diag[i][l]-diag[nx][r+1];
            best[l][len]=max(best[l][len],sum-cost[i]);
        }
    }
}
for(int t=0;t<m;t++)
    for(int len=1;len<=p && t+len<=m;len++)
        f[t+len]=max(f[t+len],f[t]+best[t+1][len]);
```

### 复杂度

预处理 `best` 的时间复杂度为 $O(nmp)$，时间 DP 为 $O(mp)$，总空间复杂度为 $O(mp+nm)$。

# 二、提高组赛题

提高组四题开始把“局部规则”升级成“全局结构”。你要么证明一个映射已经覆盖全部字母，要么把公约数与公倍数条件同时压到候选约数上，要么在有向图里同时维护可达路径上的最小买价和最大卖价，要么在搜索中维护行、列、宫三个约束的交集。读题时如果先找全局结构，后面的实现会顺很多。

## 5. [【09NOIP提高组】潜伏者](http://ybt.ssoier.cn:8088/problem_show.php?pid=1855)
`NOIP 2009` `提高组`

### 题意

已知一条加密信息以及它对应的原信息。加密规则是大写字母到大写字母的一一映射。利用这组对应关系判断能否唯一确定整张密码表；如果可以，再把第三行给出的新加密信息翻译成原信息，否则输出 `Failed`。

### 分析

题面的本质不是“模拟替换”，而是“检查一张 26 个字母的双射是否已经被完整确定”。设原字母为 $x$，密字为 $y$，那么我们既要维护 `x -> y`，也要同时维护逆映射 `y -> x`。少任何一边都不够，因为题目明确要求“每个字母只对应一个唯一密字，不同字母对应不同密字”。

扫描已知的两条对应信息时，每一对字符都会带来两个约束：原字母这一侧不能前后映到两个不同的密字；密字这一侧也不能被两个不同的原字母抢占。只要出现任何冲突，说明题面信息自相矛盾，直接失败。

如果扫描结束仍无冲突，还要继续检查“是否已经唯一确定”。这里很多人容易漏掉：不是无冲突就够了，必须 26 个原字母都出现过，整张映射表才完整。由于映射是一一对应的，只要原字母一侧 26 个都被确定，再加上前面的逆向不冲突检查，密字一侧也会完整。此时第三行的新电报才能按逆映射逐字翻译。

关键不变量是：扫描到当前位置时，`原字母 -> 密字` 与 `密字 -> 原字母` 两张表始终互相一致。答案提取也很直接：若有冲突或未覆盖 26 个字母，输出 `Failed`；否则把第三行每个密字替换成对应原字母。

### 核心代码

```cpp
for(int i=0;i<len;i++){
    int c=s1[i]-'A';
    int p=s2[i]-'A';
    if(enc[p] && enc[p]!=s1[i]) fail=1;
    if(dec[c] && dec[c]!=s2[i]) fail=1;
    enc[p]=s1[i];
    dec[c]=s2[i];
}
for(int i=0;i<26;i++)
    if(!enc[i] || !dec[i]) fail=1;
for(char ch:t) ans+=dec[ch-'A'];
```

### 复杂度

时间复杂度 $O(L)$，空间复杂度 $O(1)$，其中 $L$ 是字符串长度。

## 6. [【09NOIP提高组】Hankson的趣味题](http://ybt.ssoier.cn:8088/problem_show.php?pid=1856)
`NOIP 2009` `提高组`

### 题意

给定 $a_0,a_1,b_0,b_1$，统计有多少个正整数 $x$ 同时满足 `gcd(x, a0) = a1` 和 `lcm(x, b0) = b1`。

### 分析

这题最容易卡住的地方，是同时看见最大公约数和最小公倍数后想直接构造 $x$。更顺的思路是先缩小候选范围。因为 `lcm(x, b0) = b1`，所以 $x$ 一定是 $b_1$ 的约数；否则最小公倍数不可能正好等于 $b_1$。这样原题就从“在所有正整数里找”缩成了“在 $b_1$ 的全部约数里筛”。

筛选标准也很直接：对每个候选约数 $d$，分别检查两个条件是否同时成立。最大公约数条件控制的是“和 $a_0$ 共有的质因子次数恰好等于 $a_1$”；最小公倍数条件控制的是“和 $b_0$ 合并后的质因子次数恰好顶到 $b_1$”。虽然可以逐个质因子分析，但在数据范围下，直接枚举约数再用 `gcd`、`lcm` 判断就已经够快，而且实现更稳。

为什么只枚举到 $\sqrt{b_1}$ 即可？因为约数总是成对出现：如果 $d$ 是约数，那么 `b1 / d` 也是约数。于是每次找到一个约数就可以同时检查两边，注意平方根只算一次。

这里的关键不变量是：当前统计到的答案，恰好等于已经检查过的所有约数中满足两个条件的个数。答案提取就是最终计数值。

### 核心代码

```cpp
auto ok=[&](long long x){
    return gcd(x,a0)==a1 && x/gcd(x,b0)*b0==b1;
};
int ans=0;
for(long long d=1;d*d<=b1;d++) if(b1%d==0){
    if(ok(d)) ans++;
    if(d*d!=b1 && ok(b1/d)) ans++;
}
```

### 复杂度

单组数据时间复杂度 $O(\sqrt{b_1}\log V)$，空间复杂度 $O(1)$。

## 7. [【09NOIP提高组】最优贸易](http://ybt.ssoier.cn:8088/problem_show.php?pid=1857)
`NOIP 2009` `提高组`

### 题意

有 $n$ 个城市和若干单向或双向道路，每个城市有一个商品价格。必须从 $1$ 号城市出发，最后到 $n$ 号城市，途中最多进行一次买卖，可以不交易。问最大利润。

### 分析

“只交易一次”意味着我们只需要决定两个点：在哪个经过的城市买入，在哪个之后还能到达终点的城市卖出。于是整题可以拆成两个独立信息：

- `mn[u]`：从 $1$ 号城市走到 $u$ 的所有可行路线里，沿途见过的最低价格是多少；
- `mx[u]`：从 $u$ 出发最终走到 $n$ 的所有可行路线里，之后还能遇到的最高价格是多少。

如果这两个量都知道，那么把交易点放在“经过 $u$ 这件事”的前后两侧，能得到的最大利润就是 `mx[u] - mn[u]`。再对所有城市取最大值即可。

为什么可以分别求？因为题目允许重复经过城市，所以我们不关心具体是哪条简单路径，只关心“能不能到达”和“在所有可达路线里最优的买价或卖价是多少”。这正是图上松弛最擅长维护的量。`mn` 在原图上从 `1` 出发传播：沿边走到邻点时，历史最低价会变成 `min(mn[u], price[v])`。`mx` 则在反图上从 `n` 出发传播：从反向邻点转移时，未来最高卖价更新为 `max(mx[u], price[v])`。

关键不变量是：队列中每次弹出的点，都会尝试把“目前已知更优的最低买价 / 最高卖价”继续传给邻居；一旦某个点的值再也无法改进，它对答案的贡献也就稳定了。最后答案是 `max(0, max_u(mx[u] - mn[u]))`。

### 核心代码

```cpp
queue<int> q;
mn[1]=price[1], q.push(1), in1[1]=1;
while(!q.empty()){
    int u=q.front(); q.pop(); in1[u]=0;
    for(int v:g[u]) if(min(mn[u],price[v])<mn[v]){
        mn[v]=min(mn[u],price[v]);
        if(!in1[v]) q.push(v), in1[v]=1;
    }
}
mx[n]=price[n], q.push(n), in2[n]=1;
while(!q.empty()){
    int u=q.front(); q.pop(); in2[u]=0;
    for(int v:rg[u]) if(max(mx[u],price[v])>mx[v]){
        mx[v]=max(mx[u],price[v]);
        if(!in2[v]) q.push(v), in2[v]=1;
    }
}
for(int i=1;i<=n;i++) ans=max(ans,mx[i]-mn[i]);
```

### 复杂度

时间复杂度约为 $O(n+m)$ 到 $O(k(n+m))$ 的松弛复杂度，空间复杂度 $O(n+m)$。

## 8. [【09NOIP提高组】靶形数独](http://ybt.ssoier.cn:8088/problem_show.php?pid=1858)
`NOIP 2009` `提高组`

### 题意

给一个 $9	imes 9$ 的数独，空位用 $0$ 表示。普通数独约束依旧成立，同时每个格子有靶形权值，最终得分是“格子权值乘填入数字”的总和。求可行解中的最大得分；若无解输出 $-1$。

### 分析

普通数独的本质是约束满足问题，而这题多了一层“在所有可行解里取最大得分”。因此模型仍然是回溯搜索，只是搜索成功时不立刻返回，而是更新最优答案。

真正决定效率的，是如何快速知道某个空格能填哪些数。最常见也最稳的做法是三个九位二进制集合：`row[i]`、`col[j]`、`box[k]` 分别表示这一行、这一列、这一宫当前还允许哪些数字。某个空格 `(i,j)` 的候选集合就是三者按位与。这样一次合法性判断就从“扫一整行一整列一整宫”降成了常数级位运算。

接着要解决“先填哪个格子”。如果随便找一个空格暴搜，分支会非常大。更好的策略是始终选**候选数最少**的那个格子，也就是最强约束优先。因为它最容易尽早发现冲突，能把无效分支提早剪掉。为了让高分答案更早出现，还可以把候选数字按从大到小尝试，使中心高权格更容易先放入大数，便于尽快抬高当前最优值。

这套搜索为什么正确？因为每一步都只在当前候选集合内尝试，保证搜索树覆盖了全部合法填法；回溯时再把行、列、宫状态恢复，不会漏也不会重。关键状态是三组可选掩码与当前得分，关键不变量是：递归进入下一层前，棋盘与三组掩码始终完全对应。所有空格填完时，当前得分就是一个完整解的分值，用它更新答案即可。

### 核心代码

```cpp
int w(int x,int y){ return 10-max(abs(x-4),abs(y-4)); }
void dfs(int cur){
    int x=-1,y=-1,mask=0,best=10;
    for(int i=0;i<9;i++) for(int j=0;j<9;j++) if(!a[i][j]){
        int m=row[i]&col[j]&box[i/3*3+j/3];
        if(cnt[m]<best) best=cnt[m],x=i,y=j,mask=m;
    }
    if(x==-1){ ans=max(ans,cur); return; }
    for(int d=9;d>=1;d--) if(mask>>(d-1)&1){
        int bit=1<<(d-1), b=x/3*3+y/3;
        a[x][y]=d, row[x]^=bit, col[y]^=bit, box[b]^=bit;
        dfs(cur+w(x,y)*d);
        a[x][y]=0, row[x]^=bit, col[y]^=bit, box[b]^=bit;
    }
}
```

### 复杂度

最坏时间复杂度是指数级搜索，空间复杂度为递归深度 $O(81)$。借助位运算和“最少候选优先”剪枝，实际可行性远好于朴素暴搜。
