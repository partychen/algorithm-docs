# 一本通 NOIP 2004赛题解题报告

# 一、普及组赛题

普及组这一部分从最直接的日程模拟起步，逐步过渡到带有贪心选择、递归构造和排列后继的题目。阅读时可以特别留意一个共同动作：先把题面里的自然语言条件压成一个可比较、可维护的状态量，再决定是顺着过程模拟，还是把操作抽象成标准模型。

## 1. [【04NOIP普及组】不高兴的津津](http://ybt.ssoier.cn:8088/problem_show.php?pid=1926)
`NOIP 2004` `普及组`

### 题意

给出周一到周日每天的上课时间与额外补习时间。若某天总时长超过 8 小时，津津就会不高兴，而且总时长越长越不高兴。求下周里最不高兴的是哪一天；如果每天都不超过 8 小时，就输出 0。

### 分析

这题本质上没有隐藏结构，核心就是把题面里的“情绪”翻成一个可比较的量：一天的总课时 `a+b`。当 `a+b>8` 时，这一天会不高兴，而且不高兴程度随着总课时单调增加，所以我们根本不需要额外设计评分函数，直接比较总课时即可。

扫描 7 天时维护两个量：当前见过的最大总课时 `mx`，以及对应的最早日期 `ans`。只有当今天的总课时严格大于 `mx` 且又超过 8 小时时，答案才更新。这样天然保证了并列时保留更早的那一天。整个过程里的不变量是：扫到第 `i` 天后，`ans` 一定是前 `i` 天里最早出现的“最大不高兴日”。最后直接输出它即可。

### 核心代码

```cpp
int ans=0,mx=8;
for(int i=1;i<=7;i++){
    int s=a[i]+b[i];
    if(s>mx){
        mx=s;
        ans=i;
    }
}
```

### 复杂度

时间复杂度 $O(1)$，空间复杂度 $O(1)$。

## 2. [【04NOIP普及组】花生采摘](http://ybt.ssoier.cn:8088/problem_show.php?pid=1927)
`NOIP 2004` `普及组`

### 题意

给定花生田中每个位置的花生数量，多多可以从路边进入第一行、在相邻格之间移动、在某格采摘一次，并且最后必须回到路边。问在限定时间内最多能采到多少花生。

### 分析

先把题目拆成两个部分：**选哪些植株**，以及**按什么顺序走**。题面说明所有有花生的植株数量互不相同，而每采一棵都要额外花 1 个单位时间，所以当我们决定“下一棵还要不要摘”时，最值得优先考虑的一定是剩余花生数最多的植株。于是可以先把所有非零位置按花生数从大到小排序。

之后就只剩路径时间的精确核算。若当前在 `(x,y)`，下一棵在 `(nx,ny)`，走过去要花 `|x-nx|+|y-ny|`，采摘要 1，最后还必须从所在行跳回路边，因此在尝试摘下一棵前，必须检查 `已用时间+过去的路程+1+nx` 是否不超过 `K`。这个检查维护了一个关键不变量：**只要当前步骤被接受，就保证整条路径在摘完当前植株后仍然来得及回到路边**。一旦某棵也来不及摘，它后面就不必继续模拟了，答案就是已经累计的花生总数。

### 核心代码

```cpp
sort(p.begin(),p.end(),[](auto &u,auto &v){ return u.w>v.w; });
int x=0,y=0,t=0,ans=0;
for(auto &u:p){
    int d=(x?abs(x-u.x)+abs(y-u.y):u.x);
    if(t+d+1+u.x>K) break;
    t+=d+1;
    ans+=u.w;
    x=u.x; y=u.y;
}
```

### 复杂度

设有花生的植株数为 $P$，时间复杂度 $O(P\log P)$，空间复杂度 $O(P)$。

## 3. [【04NOIP普及组】FBI树](http://ybt.ssoier.cn:8088/problem_show.php?pid=1928)
`NOIP 2004` `普及组`

### 题意

给定一个长度为 $2^N$ 的 01 串。把全 0 串记为 B，全 1 串记为 I，混合串记为 F，并按二分递归规则构造 FBI 树。要求输出整棵树的后序遍历序列。

### 分析

题目的构造规则已经把模型直接告诉我们了：每个区间先看自己属于 B、I 还是 F，再把它平分成左右两半继续递归。因此真正要抓住的是递归函数的含义：`dfs(l,r)` 表示“构造子串 `s[l..r]` 对应的整棵子树，并输出这棵子树的后序遍历”。

后序遍历的顺序是“左子树、右子树、根”，所以递归时先处理左右区间，最后再根据当前区间是否全 0、全 1 或混合来输出 `B/I/F`。这里的不变量很清楚：当 `dfs(l,r)` 返回时，`s[l..r]` 这段对应子树的后序序列已经完整输出。区间长度缩到 1 时，结点类型由单个字符直接决定，这就是递归边界。

### 核心代码

```cpp
void dfs(int l,int r){
    if(l==r){
        putchar(s[l]=='0'?'B':'I');
        return;
    }
    int m=(l+r)>>1;
    dfs(l,m);
    dfs(m+1,r);
    int has0=0,has1=0;
    for(int i=l;i<=r;i++) has0|=s[i]=='0',has1|=s[i]=='1';
    putchar(has0&&has1?'F':(has0?'B':'I'));
}
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(\log n)$，其中 $n=2^N$。

## 4. [【04NOIP普及组】火星人](http://ybt.ssoier.cn:8088/problem_show.php?pid=1929)
`NOIP 2004` `普及组`

### 题意

火星人的手指排列对应一个字典序中的排列编号。给出当前排列以及要加上的一个很小的整数 $M$，要求输出加完之后对应的排列。

### 分析

题目表面是在“做加法”，本质却是在**排列的字典序序号上前进 `M` 次**。如果直接把当前排列换算成排名再加上 `M`，会牵扯阶乘数系统；但这里 `M<=100` 非常小，而 `N` 再大也只是排列长度大，因此最自然的做法就是把“加 1”实现成一次 `next_permutation`，连续做 `M` 次。

`next_permutation` 维护的核心不变量是：每做一次，都得到当前排列的**紧邻后继**。它先找到从右往左第一个破坏下降趋势的位置，把这里视为“还能变大的一位”；再和右侧刚好比它大的元素交换，并把后缀翻成最小字典序。于是循环 `M` 次后，恰好走到了“原排列加上 `M`”的位置。

### 核心代码

```cpp
for(int i=1;i<=m;i++)
    next_permutation(a+1,a+n+1);
```

### 复杂度

每次求下一个排列是 $O(n)$，共执行 $M$ 次，所以时间复杂度 $O(Mn)$，空间复杂度 $O(1)$。

# 二、提高组赛题

提高组题目明显更强调模型抽象：有的要把现金流过程拆成状态维护，有的要把“总代价最小”改写成经典贪心结构，有的则要在动态规划和按列搜索里维护全局合法性。做这一组题时，关键是学会先找不变量，再写转移。

## 5. [【04NOIP提高组】津津的储蓄计划](http://ybt.ssoier.cn:8088/problem_show.php?pid=1835)
`NOIP 2004` `提高组`

### 题意

已知津津 12 个月的预算。每月月初先拿到 300 元，再按计划花钱；若手头剩余达到整百，就把整百存给妈妈，年末连本带息返还 120%。若中途某个月钱不够花，输出第一次出问题的月份；否则输出年末总钱数。

### 分析

这题的关键不是“怎么存最优”，因为存钱规则已经固定；真正要做的是按月份把现金流一笔笔算清楚。我们维护两个状态：`cash` 表示手里随时可用的零钱，`save` 表示已经交给妈妈、年末才能取出的整百金额。

每个月先让 `cash+=300`，再支付预算。如果这一步后 `cash<0`，说明当月就已经无法覆盖开销，答案必须立刻定为负的月份编号，因为题目要找第一次出问题的月份。若还能支付，则把 `cash` 中可以整百存走的部分加入 `save`，只留下不足 100 的零头。这里的不变量是：**每个月处理结束后，`cash<100`，而 `save` 恰好等于此前所有被存走的整百总额**。最后把 `save` 按 120% 返还，再加上手里的零钱即可。

### 核心代码

```cpp
int cash=0,save=0;
for(int i=1;i<=12;i++){
    cash+=300-a[i];
    if(cash<0){
        ans=-i;
        break;
    }
    save+=cash/100*100;
    cash%=100;
}
if(ans==0) ans=cash+save*12/10;
```

### 复杂度

时间复杂度 $O(1)$，空间复杂度 $O(1)$。

## 6. [【04NOIP提高组】合并果子](http://ybt.ssoier.cn:8088/problem_show.php?pid=1836)
`NOIP 2004` `提高组`

### 题意

有若干堆果子，每次可以合并任意两堆，代价等于两堆重量之和。经过 $n-1$ 次合并后只剩一堆，求总代价最小值。

### 分析

一旦某两堆被合并，它们的和以后还会继续参与后续合并，所以越早形成的大堆，往后被重复计算的次数越多。为了让“被反复带着走”的重量尽量小，最优策略一定是每次先合并当前最轻的两堆，这正是 Huffman 合并的经典模型。

证明思路可以抓住交换观点：如果某一步没有选最小的两堆，而是让更重的果子更早进入后续流程，那么这些更重的部分会在之后的若干次合并里被额外累计，只会更差不会更优。因此维护一个小根堆，每次弹出最小的两堆 `x,y`，把代价 `x+y` 累加到答案，再把新堆压回去。堆里的不变量是：**始终保存当前尚未完成合并的所有堆重量**。

### 核心代码

```cpp
priority_queue<int,vector<int>,greater<int>> pq;
for(int x:a) pq.push(x);
long long ans=0;
while(pq.size()>1){
    int x=pq.top(); pq.pop();
    int y=pq.top(); pq.pop();
    ans+=x+y;
    pq.push(x+y);
}
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(n)$。

## 7. [【04NOIP提高组】合唱队形](http://ybt.ssoier.cn:8088/problem_show.php?pid=1837)
`NOIP 2004` `提高组`

### 题意

给出 $N$ 位同学的身高，要求删去尽量少的人，使剩下的人能构成先严格上升后严格下降的合唱队形。

### 分析

“删最少”通常等价于“留最多”，所以先把问题改写成：最多能留下多少人构成一个山峰形序列。若把某个位置 `i` 看成山顶，那么它左边必须是以 `i` 结尾的最长严格上升子序列，右边必须是以 `i` 开始的最长严格下降子序列。于是这题自然拆成两个 DP。

定义 `up[i]` 为以 `i` 结尾的最长上升子序列长度，`down[i]` 为以 `i` 开始的最长下降子序列长度。转移都只依赖更前或更后的状态：当 `h[j]<h[i]` 时，`up[i]=max(up[i],up[j]+1)`；从右往左同理可求 `down[i]`。这样 `up[i]+down[i]-1` 就是把 `i` 当峰顶时能保留的人数，减掉的最少人数便是 `n-max(up[i]+down[i]-1)`。这里的核心状态含义非常稳定：**每个状态都表示“必须选当前位置作为端点”时的最优长度**。

### 核心代码

```cpp
for(int i=1;i<=n;i++){
    up[i]=1;
    for(int j=1;j<i;j++)
        if(h[j]<h[i]) up[i]=max(up[i],up[j]+1);
}
for(int i=n;i>=1;i--){
    down[i]=1;
    for(int j=n;j>i;j--)
        if(h[j]<h[i]) down[i]=max(down[i],down[j]+1);
    best=max(best,up[i]+down[i]-1);
}
```

### 复杂度

时间复杂度 $O(n^2)$，空间复杂度 $O(n)$。

## 8. [【04NOIP提高组】虫食算](http://ybt.ssoier.cn:8088/problem_show.php?pid=1838)
`NOIP 2004` `提高组`

### 题意

给定一个 $N$ 进制加法竖式，三个长度都为 $N$ 的字符串由前 $N$ 个大写字母组成。不同字母代表不同数字，且恰有唯一解。求每个字母对应的数字。

### 分析

这题最容易迷失在“整体有很多字母”里，但竖式加法真正起作用的是**按列约束**。从最低位往最高位看，当前列只和三件事有关：两个加数字母、结果字母，以及从低一位传来的进位 `carry`。也就是说，只要低位已经确定，高位根本不用提前考虑。

于是搜索状态可以设成 `dfs(col, carry)`：表示第 `col` 列到最高列还没处理，低于它的列都已经满足加法关系，并且向这一列带来一个进位 `carry`。在这一列里，只要两个加数字母的取值确定，结果位就被 `sum mod N` 唯一确定，新的进位也被 `sum / N` 唯一确定，所以搜索的核心不是盲枚举三位，而是围绕列约束做即时剪枝：一旦当前列无法满足同余关系，整棵子树立刻返回。整个递归的不变量是：**已经处理过的低位列全部合法，且已赋值字母之间始终保持一一对应**。到最高位之后只需检查最终进位是否为 0，唯一解也就被还原出来了。

### 核心代码

```cpp
bool dfs(int col,int carry){
    if(col<0) return carry==0;
    int x=A[col]-'A',y=B[col]-'A',z=C[col]-'A';
    for(int vx=0;vx<n;vx++){
        if(val[x]!=-1&&val[x]!=vx) continue;
        if(val[x]==-1&&used[vx]) continue;
        bool addx=val[x]==-1;
        if(addx) val[x]=vx,used[vx]=1;
        for(int vy=0;vy<n;vy++){
            if(val[y]!=-1&&val[y]!=vy) continue;
            if(val[y]==-1&&used[vy]) continue;
            bool addy=val[y]==-1;
            if(addy) val[y]=vy,used[vy]=1;
            int sum=val[x]+val[y]+carry;
            int vz=sum%n,nc=sum/n;
            if((val[z]==-1&&!used[vz])||val[z]==vz){
                bool addz=val[z]==-1;
                if(addz) val[z]=vz,used[vz]=1;
                if(dfs(col-1,nc)) return true;
                if(addz) used[vz]=0,val[z]=-1;
            }
            if(addy) used[vy]=0,val[y]=-1;
        }
        if(addx) used[vx]=0,val[x]=-1;
    }
    return false;
}
```

### 复杂度

最坏情况下时间复杂度为指数级，但按列递归并结合取值唯一与同余约束后，实际剪枝非常强；空间复杂度为 $O(n)$。
