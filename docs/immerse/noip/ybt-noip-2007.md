# 一本通 NOIP 2007赛题解题报告

# 一、普及组赛题
普及组四题从规则实现、排序贪心一路走到递推与状态设计。读这一部分时，重点不是背模板，而是学会把题目里的“比较规则”“每秒动作”“最少次数”翻译成可以逐步维护的不变量。

## 1. [【07NOIP普及组】奖学金](http://ybt.ssoier.cn:8088/problem_show.php?pid=1938)
`NOIP 2007` `普及组`

### 题意
先为每个学生计算三科总分，再按“总分降序、语文降序、学号升序”的优先级排序，输出前五名的学号和总分。

### 分析
题目已经把排名规则说得非常完整，本质上就是把每个学生压成一个待比较的记录。这里最关键的不是“会不会排序”，而是能不能把比较顺序写成严格一致的比较器：先看总分，若总分相同再看语文，若还相同再看学号。这个比较器一旦确定，排序后的序列就天然满足题意要求。

可以把 `id`、`cn`、`sum` 当成状态。排序过程保持的核心不变量是：任意排在前面的学生，按照题目给定的三层规则都不比后面的学生差。排序完成后，答案提取就非常直接，取前五个元素输出即可。

### 核心代码
```cpp
struct Node {
    int id, cn, sum;
};

bool cmp(const Node& a, const Node& b) {
    if (a.sum != b.sum) return a.sum > b.sum;
    if (a.cn != b.cn) return a.cn > b.cn;
    return a.id < b.id;
}

sort(a + 1, a + n + 1, cmp);
for (int i = 1; i <= 5; i++) {
    cout << a[i].id << ' ' << a[i].sum << '\n';
}
```

### 复杂度
时间复杂度 `O(n log n)`，空间复杂度 `O(n)`。

## 2. [【07NOIP普及组】纪念品分组](http://ybt.ssoier.cn:8088/problem_show.php?pid=1939)
`NOIP 2007` `普及组`

### 题意
每组最多放两件纪念品，且价格和不超过上限 `w`。要求把全部纪念品分完，并让分组总数最少。

### 分析
“每组最多两件”是这题最强的结构信息，它提示我们只需要考虑“最轻的能不能和最重的配成一组”。先把价格从小到大排序，设左右指针 `l`、`r` 分别指向当前最轻和最重的纪念品。若 `p[l] + p[r] <= w`，说明最重的这一件可以和当前最轻的一件一起走；若不能配对，那么最重的这一件无论和谁配都会超限，只能单独成组。

这个贪心之所以成立，是因为每一步都先固定最难安排的“最重物品”。能和它搭配时，选最轻的最省资源；不能和最轻搭配时，它和任何别的物品都更不可能搭配。于是每次决策都不会破坏最优性。循环结束时，组数累加值就是答案。

### 核心代码
```cpp
sort(p + 1, p + n + 1);
int l = 1, r = n, ans = 0;
while (l <= r) {
    if (l == r) {
        ans++;
        break;
    }
    if (p[l] + p[r] <= w) l++, r--;
    else r--;
    ans++;
}
```

### 复杂度
时间复杂度 `O(n log n)`，空间复杂度 `O(1)`（不计排序栈空间）。

## 3. [【07NOIP普及组】守望者的逃离](http://ybt.ssoier.cn:8088/problem_show.php?pid=1940)
`NOIP 2007` `普及组`

### 题意
守望者每秒只能做一件事：跑步 `17m`、闪烁 `60m`、或者原地休息恢复 `4` 点魔法。闪烁要消耗 `10` 点魔法。给定初始魔法、目标距离和沉没时间，求能否在时限内逃离；若能，求最短时间，否则输出最多能走多远。

### 分析
题目里的难点不是动作多，而是“休息本身不前进，却能让后面更快”。一个好做法是把“纯魔法方案”与“最终最优距离”分开维护。设 `f[i]` 表示恰好经过 `i` 秒，只考虑“闪烁或休息”时能到达的最远距离；同时记录此时剩余魔法。再设 `dp[i]` 表示经过 `i` 秒后，允许把跑步和前面的最优魔法方案混合起来，所能达到的最远距离。

转移时，如果当前魔法至少有 `10`，这一秒可以闪烁，`f[i] = f[i-1] + 60`；否则只能休息，`f[i] = f[i-1]` 且魔法加 `4`。而 `dp[i]` 有两种来源：一是上一秒已经达到最优，这一秒继续跑步，得到 `dp[i-1] + 17`；二是前 `i` 秒全部按最优的闪烁或休息策略执行，得到 `f[i]`。于是 `dp[i] = max(dp[i-1] + 17, f[i])`。这个状态的核心不变量是：`dp[i]` 始终表示 `i` 秒内所有合法动作序列的最远距离。每一秒更新后立刻检查是否达到 `S`，第一次达到的时刻就是最短逃离时间；若直到 `T` 秒都没到，输出 `dp[T]`。

### 核心代码
```cpp
int mana = M;
for (int i = 1; i <= T; i++) {
    f[i] = f[i - 1];
    if (mana >= 10) {
        f[i] += 60;
        mana -= 10;
    } else {
        mana += 4;
    }
    dp[i] = max(dp[i - 1] + 17, f[i]);
    if (dp[i] >= S) {
        cout << "Yes\n" << i << '\n';
        return;
    }
}
cout << "No\n" << dp[T] << '\n';
```

### 复杂度
时间复杂度 `O(T)`，空间复杂度 `O(T)`。

## 4. [【07NOIP普及组】Hanoi双塔问题](http://ybt.ssoier.cn:8088/problem_show.php?pid=1941)
`NOIP 2007` `普及组`

### 题意
有 `n` 种大小、每种各两个相同圆盘，共 `2n` 个盘子在 `A` 柱上。每次只能移动一个盘，且三根柱子始终保持上层不大于下层。求全部移到 `C` 柱的最少次数。

### 分析
题目已经提示要找 `A_n` 与 `A_{n-1}` 的关系。思考最大的一对圆盘：要把第 `n` 种盘子移到 `C`，必须先把前面 `n-1` 种盘子整体挪开；这一步需要 `A_{n-1}` 次。接着把两个最大的盘子依次移到 `C`，共 `2` 次。最后再把前面 `n-1` 种盘子搬到这两个大盘上，又是 `A_{n-1}` 次。

于是得到递推 `A_n = 2A_{n-1} + 2`，边界是 `A_1 = 2`。这里的状态非常纯粹：`A_n` 就是前 `n` 对圆盘的最优解。递推的含义也很直观：小问题搬开一次、最大盘处理两次、小问题再覆盖一次。因为 `n` 可到 `200`，答案会很大，要用高精度。按递推从小到大算到 `A_n`，最后输出当前值即可。

### 核心代码
```cpp
using boost::multiprecision::cpp_int;

cpp_int ans = 2;
for (int i = 2; i <= n; i++) {
    ans = ans * 2 + 2;
}
cout << ans << '\n';
```

### 复杂度
时间复杂度 `O(n)`，空间复杂度 `O(1)`（高精度位数另计）。

# 二、提高组赛题
提高组四题开始明显强调“模型抽象”：有的要把字符串规则拆成局部模拟，有的要把每一行拆成区间 DP，有的甚至要先找直径、再做窗口优化。读这一部分时，重点是把题目条件重写成状态、转移和可验证的不变量。

## 5. [【07NOIP提高组】统计数字](http://ybt.ssoier.cn:8088/problem_show.php?pid=1847)
`NOIP 2007` `提高组`

### 题意
给出 `n` 个自然数，要求统计每个不同数字出现的次数，并按数字从小到大输出。

### 分析
题目要求“按数值升序输出”，这意味着相同数字最好先被放到一起。最自然的做法就是先排序，再线性扫描计数。排序完成后，相同数字会形成连续段；我们只需维护当前段的值 `a[i]` 和这一段的出现次数 `cnt`。一旦发现下一个数不同，就说明当前连续段结束，可以立刻输出这一段的数字和次数。

这里的关键不变量是：扫描到位置 `i` 时，`cnt` 始终表示当前这个数字已经连续出现了多少次。因为排序后不会再出现“同一个数被隔开”的情况，所以每个连续段输出一次即可。全部扫完后，别忘了把最后一段补输出。

### 核心代码
```cpp
sort(a + 1, a + n + 1);
for (int i = 1, j; i <= n; i = j) {
    j = i;
    while (j <= n && a[j] == a[i]) j++;
    cout << a[i] << ' ' << (j - i) << '\n';
}
```

### 复杂度
时间复杂度 `O(n log n)`，空间复杂度 `O(1)`（不计排序栈空间）。

## 6. [【07NOIP提高组】字符串的展开](http://ybt.ssoier.cn:8088/problem_show.php?pid=1848)
`NOIP 2007` `提高组`

### 题意
字符串中若出现形如 `a-d`、`3-7` 的合法区间，就按参数 `p1`、`p2`、`p3` 把中间字符展开；若两端类型不同、顺序不升或本来就是相邻字符，则按题目规则保留或仅删除减号。

### 分析
这道题最怕一上来就硬写分支。更稳的思路是把每个减号都看成一个“局部替换点”，只关注它左边字符 `s[i-1]` 与右边字符 `s[i+1]` 能不能生成一个合法区间。合法条件只有三步：两边都存在、同为小写字母或同为数字、且左字符严格小于右字符。若不合法，直接保留减号；若相邻，只删去减号；若中间有字符需要填，就把待填字符序列先生成出来，再统一处理大小写、重复次数和逆序。

这个做法的关键状态是当前减号两侧的端点，以及由它们决定的“中间字符列表”。只要这个列表正确，后续的 `p1`、`p2`、`p3` 都只是对列表做映射和重排，不会互相干扰。答案提取也很简单：从左到右扫描原串，普通字符直接加入，遇到减号则把展开结果拼到答案串中。

### 核心代码
```cpp
auto ok = [&](char l, char r) {
    bool same_digit = isdigit(l) && isdigit(r);
    bool same_lower = islower(l) && islower(r);
    return (same_digit || same_lower) && l < r;
};

for (int i = 0; i < n; i++) {
    if (s[i] != '-' || i == 0 || i == n - 1 || !ok(s[i - 1], s[i + 1])) {
        ans += s[i];
        continue;
    }
    if (s[i + 1] == s[i - 1] + 1) continue;
    string t;
    for (char c = s[i - 1] + 1; c < s[i + 1]; c++) {
        char x = (p1 == 3 ? '*' : c);
        if (p1 == 2 && islower(c)) x = char(c - 'a' + 'A');
        t.append(p2, x);
    }
    if (p3 == 2) reverse(t.begin(), t.end());
    ans += t;
}
```

### 复杂度
时间复杂度 `O(|s| + 展开后长度)`，空间复杂度 `O(|s|)`。

## 7. [【07NOIP提高组】矩阵取数游戏](http://ybt.ssoier.cn:8088/problem_show.php?pid=1849)
`NOIP 2007` `提高组`

### 题意
矩阵共有 `n` 行 `m` 列。每一轮要从每一行取走一个数，且只能取当前行的行首或行尾。第 `i` 轮取到的数会乘上权值 `2^i`。要求总得分最大。

### 分析
先抓住题目的“每次从每行都取一个”这一条件。因为第 `i` 轮每一行乘的权值都相同，所以总分等于“各行最优得分之和”，行与行之间互不影响，可以逐行求解后累加。于是原问题被拆成若干个一维子问题：对一行长度为 `m` 的序列，反复从两端取数，让加权和最大。

对单行做区间 DP。设 `dp[l][r]` 表示当前还剩区间 `[l, r]` 没取时，从这一段继续取到结束所能得到的最大得分。若当前剩余长度为 `len = r - l + 1`，说明已经取了 `m - len` 个数，下一次取数对应的权值就是 `2^{m-len+1}`。转移时只有两个选择：取左端 `a[l]` 或取右端 `a[r]`。因此有 `dp[l][r] = max(dp[l+1][r] + a[l] * w, dp[l][r-1] + a[r] * w)`。

状态的核心不变量是：`dp[l][r]` 永远代表“这一段还没动，后续最优能拿多少分”。按区间长度从小到大递推，单行答案就是 `dp[1][m]`。由于 `2^m` 很大，需要高精度；全部行处理完后把每行答案累加，就是整张矩阵的最优值。

### 核心代码
```cpp
using boost::multiprecision::cpp_int;

for (int i = 1; i <= m; i++) pw[i] = pw[i - 1] * 2;
for (int len = 1; len <= m; len++) {
    int step = m - len + 1;
    for (int l = 1; l + len - 1 <= m; l++) {
        int r = l + len - 1;
        cpp_int w = pw[step];
        if (l == r) dp[l][r] = row[l] * w;
        else dp[l][r] = max(dp[l + 1][r] + row[l] * w,
                            dp[l][r - 1] + row[r] * w);
    }
}
ans += dp[1][m];
```

### 复杂度
时间复杂度 `O(nm^2)`，空间复杂度 `O(m^2)`。

## 8. [【07NOIP提高组】树网的核](http://ybt.ssoier.cn:8088/problem_show.php?pid=1850)
`NOIP 2007` `提高组`

### 题意
给一棵带权树和长度上限 `S`，要求在某条直径上选一段长度不超过 `S` 的路径 `F`，使全树所有点到 `F` 的最大距离最小，输出这个最小偏心距。

### 分析
这题看上去定义很多，但真正有用的信息只有一句：核一定选在某条直径上。于是第一步先求出一条直径，并把直径上的点按顺序拉成一个数组。记 `dis[i]` 为直径起点到第 `i` 个点的距离。接下来还要处理“离开直径的支路”：设 `dep[i]` 表示从第 `i` 个直径点出发，不走直径边时，能进入的最大支路长度。

如果我们枚举直径上的一段窗口 `[l, r]` 作为候选核，长度限制就是 `dis[r] - dis[l] <= S`。此时最远点有三类来源。第一类在窗口内部，它们到核的最远距离就是窗口内某个直径点的支路深度，贡献 `max(dep[k])`。第二类在窗口左边，它们最近只能贴到 `l`，距离形如 `dep[k] + dis[l] - dis[k]`。第三类在窗口右边，对称地是 `dep[k] + dis[k] - dis[r]`。因此只要预处理前缀最大值 `pre[i] = max(dep[k] - dis[k])` 和后缀最大值 `suf[i] = max(dep[k] + dis[k])`，就能把左右两侧的最坏情况 `O(1)` 算出来。

整个模型的关键状态是直径序列上的 `dis[i]` 与 `dep[i]`，它们分别刻画“沿主干走多远”和“离主干还能伸多远”。枚举所有合法窗口时，取三类贡献的最大值就是该窗口的偏心距，再在所有窗口中取最小值，就是最终答案。

### 核心代码
```cpp
for (int i = 1; i <= m; i++) {
    pre[i] = max(pre[i - 1], dep[i] - dis[i]);
}
for (int i = m; i >= 1; i--) {
    suf[i] = max(suf[i + 1], dep[i] + dis[i]);
}
for (int l = 1; l <= m; l++) {
    int mid = 0;
    for (int r = l; r <= m && dis[r] - dis[l] <= S; r++) {
        mid = max(mid, dep[r]);
        int left = (l > 1 ? dis[l] + pre[l - 1] : 0);
        int right = (r < m ? suf[r + 1] - dis[r] : 0);
        ans = min(ans, max(mid, max(left, right)));
    }
}
```

### 复杂度
时间复杂度 `O(n^2)`，空间复杂度 `O(n)`。
