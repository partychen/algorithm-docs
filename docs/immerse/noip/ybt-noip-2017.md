# 一本通 NOIP 2017赛题解题报告

这一组题从最直接的算式模拟、后缀检索与最短路状态建模，一路走到复杂度判定、最短路计数、空间连通、状态压缩与动态序列维护。前半段练的是把题面规则翻成清晰状态，后半段则不断把“合法方案”压进图、并查集、记忆化搜索和数据结构里。

# 一、普及组赛题

这一部分看起来更贴近直觉，但真正要练的是同一件事：把题面里的规则翻成可执行的状态，再判断该直接模拟、做最短路，还是把判定问题转成单调性加动态规划。

## 1. [【17NOIP普及组】成绩](http://ybt.ssoier.cn:8088/problem_show.php?pid=1414)

`普及组` `模拟` `加权求和`

### 题意

输入作业、小测、期末三项成绩，按 `20%`、`30%`、`50%` 的权重计算总成绩，并输出最终整数结果。

### 分析

这题最重要的判断是：题面没有隐藏状态，也没有优化目标，本质就是把评分规则原样翻译成式子。只要把三部分贡献分别算出再相加即可。

为了避免浮点误差，可以把式子改写成 `(2A+3B+5C)/10`。因为题目保证三项成绩都是 `10` 的整数倍，所以最终结果一定是整数，这样直接做整数运算最稳。

状态其实只有三个输入量 `A,B,C`，不需要额外数组，也不存在中间决策。答案提取就是把公式值直接输出。

### 核心代码

```cpp
int ans = (2 * a + 3 * b + 5 * c) / 10;
cout << ans;
```

### 复杂度

时间复杂度 `O(1)`，空间复杂度 `O(1)`。

## 2. [【17NOIP普及组】图书管理员](http://ybt.ssoier.cn:8088/problem_show.php?pid=1415)

`普及组` `哈希` `后缀匹配`

### 题意

给出 `n` 本书的编码。每次询问给出需求码长度 `len` 和需求码 `x`，要求找到所有“最后 `len` 位恰好等于 `x`”的图书里编码最小的一本；如果不存在就输出 `-1`。

### 分析

题面真正有用的条件是“以需求码结尾”以及“额外给了长度”。这说明不能只比较数字大小，而是要做“固定长度后缀匹配”。把它翻成运算就是：一本书 `v` 能回答询问 `(len,x)`，当且仅当 `v % 10^len == x`。

既然书的编码最多只有 `7` 位，我们完全可以把每本书所有可能的后缀都枚举出来。对每个长度 `len`，维护一个键值对 `(len, suffix) -> 最小编码`。这样查询时就不必再扫所有书，而是直接去表里取答案。

这个模型的关键状态就是哈希表里的最小值不变量：它始终表示“目前见过的、具有这个后缀的最小编码”。答案提取时查一下对应键是否存在，存在就输出最小值，不存在就输出 `-1`。

### 核心代码

```cpp
unordered_map<long long,int> mp;
int p10[8] = {1,10,100,1000,10000,100000,1000000,10000000};
for(int v : book)
  for(int len = 1; len <= 7; len++){
    long long key = 1LL * len * 10000000 + v % p10[len];
    if(!mp.count(key) || v < mp[key]) mp[key] = v;
  }
long long key = 1LL * len * 10000000 + need;
cout << (mp.count(key) ? mp[key] : -1) << endl;
```

### 复杂度

预处理时间复杂度 `O(7n)`，查询时间复杂度 `O(1)`，空间复杂度 `O(7n)`。

## 3. [【17NOIP普及组】棋盘](http://ybt.ssoier.cn:8088/problem_show.php?pid=1416)

`普及组` `最短路` `状态设计`

### 题意

在 `m × m` 棋盘上从左上角走到右下角。走到相同颜色格子不花钱，走到不同颜色格子花 `1`。还可以花 `2` 金币把相邻无色格子临时染成任意颜色并走进去，但魔法不能连续使用。求最小花费；若无法到达则输出 `-1`。

### 分析

看到“最小花费”加上移动代价非负，第一反应就是最短路；但这题不能只把格子当点，因为能不能继续用魔法，取决于你当前所在格子是不是刚刚用魔法造出来的。于是状态至少要补上一位“上一脚是不是魔法”的信息。

更完整地说，我们把状态记成 `(x,y,c,t)`：当前位置是 `(x,y)`，当前脚下格子的颜色是 `c`，`t=1` 表示这个格子是临时染色得到的，所以这一回合不能再施法；`t=0` 表示它本来就有颜色，下一步既可以正常走，也可以对无色格子施法。这样一来，题面里那句“魔法不能连续使用”就被准确翻成了状态限制。

转移分两类。若相邻格子本来就有颜色 `nc`，代价增加 `c!=nc`，并转到 `(nx,ny,nc,0)`；若相邻格子无色，则只有 `t=0` 时才能进入，而且可以把它染成 `0` 或 `1`，代价增加 `2 + (c!=nc)`，转到 `(nx,ny,nc,1)`。所有边权都非负，所以用 Dijkstra 能保证第一次弹出的最短距离就是最优值。

关键不变量是 `dist[x][y][c][t]`：它表示到达该状态的最小金币数。最后答案不是看某一个固定颜色，而是取所有到达终点 `(m,m)` 的状态最小值；如果这些状态都不可达，就输出 `-1`。

### 核心代码

```cpp
while(!pq.empty()){
  auto [d,x,y,c,t] = pq.top(); pq.pop();
  if(d != dist[x][y][c][t]) continue;
  for(auto [nx,ny] : nbr(x,y)){
    if(col[nx][ny] != -1){
      int nc = col[nx][ny], nd = d + (c != nc);
      relax(nx, ny, nc, 0, nd);
    }else if(!t){
      for(int nc = 0; nc < 2; nc++)
        relax(nx, ny, nc, 1, d + 2 + (c != nc));
    }
  }
}
int ans = INF;
for(int c = 0; c < 2; c++)
  for(int t = 0; t < 2; t++) ans = min(ans, dist[m][m][c][t]);
cout << (ans == INF ? -1 : ans);
```

### 复杂度

状态数为 `O(m^2)` 级别，时间复杂度 `O(m^2 log m)`，空间复杂度 `O(m^2)`。

## 4. [【17NOIP普及组】跳房子](http://ybt.ssoier.cn:8088/problem_show.php?pid=1417)

`普及组` `二分` `单调队列动态规划`

### 题意

一条直线上有若干格子，第 `i` 个格子在位置 `x_i`，得分为 `s_i`。机器人每次跳跃距离原本固定为 `d`，花 `g` 个金币改造后，每次可跳距离落在 `[max(1,d-g), d+g]`。要求至少得到 `k` 分，问最少需要多少金币；若无论如何都做不到则输出 `-1`。

### 分析

题面问“最少改造多少”，这通常先找单调性：如果某个 `g` 已经可行，那么更大的 `g` 只会让跳距区间更宽，不会更差，所以答案可以二分。

接着要解决“固定 `g` 时是否可行”。把跳距区间记成 `[L,R]`，问题就变成：从起点 `0` 出发，只能从某个位置跳到右侧距离在 `[L,R]` 的格子，能拿到的最大分数是多少。于是自然定义 `dp[i]` 为“最后落在第 `i` 个格子时的最大得分”，转移是 `dp[i] = s_i + max(dp[j])`，其中 `x_i - x_j` 必须落在 `[L,R]`。

由于位置 `x_i` 递增，满足条件的前驱集合会随着 `i` 向右滑动。我们用双指针维护哪些 `j` 已经进入窗口，再用单调队列维护这些候选里最大的 `dp[j]`，就能把每次转移从暴力扫描压到均摊 `O(1)`。这里的关键不变量是：队首始终是当前合法窗口中 `dp` 最大的位置。

判定函数里只要看是否存在某个 `dp[i] >= k` 即可；二分时取第一个可行的 `g` 作为答案。若把 `g` 放到足够大仍然不可行，就输出 `-1`。

### 核心代码

```cpp
bool check(int g){
  int L = max(1, d - g), R = d + g, p = 0;
  fill(dp, dp + n + 1, NEG); dp[0] = 0; deque<int> q;
  for(int i = 1; i <= n; i++){
    while(p < i && x[i] - x[p] >= L){
      while(!q.empty() && dp[q.back()] <= dp[p]) q.pop_back();
      q.push_back(p++);
    }
    while(!q.empty() && x[i] - x[q.front()] > R) q.pop_front();
    if(!q.empty()) dp[i] = dp[q.front()] + s[i];
    if(dp[i] >= k) return true;
  }
  return false;
}
```

### 复杂度

单次判定时间复杂度 `O(n)`，二分后总时间复杂度 `O(n log V)`，空间复杂度 `O(n)`。

# 二、提高组赛题

提高组开始，题面已经不再只是把规则翻成代码，而是要先找到真正的数学结构：有的题把限制压成判定栈，有的题把“多走一点”压成最短路松弛量，有的题则要把整队人的变化压成若干条可维护序列。

## 5. [【17NOIP提高组】小凯的疑惑](http://ybt.ssoier.cn:8088/problem_show.php?pid=1899)

`提高组` `数论` `裴蜀定理`

### 题意

有两种面值分别为 `a,b` 的金币，数量无限，且 `a,b` 互质。问不能被这两种金币凑出的最大价值是多少。

### 分析

看到“两个互质正整数的线性组合能表示哪些数”，就要想到经典的 Frobenius 问题。互质这个条件非常关键：它保证从某个界开始，所有整数都能表示出来，所以“最大不能表示值”一定存在。

为什么公式是 `ab-a-b`？一种常见理解是：模 `a` 意义下，`0,b,2b,...,(a-1)b` 会把所有余数恰好走一遍。于是从 `0,b,2b,...` 分别往上每次再加若干个 `a`，最终所有足够大的数都会被覆盖。而最后一个没被覆盖到的整数，恰好就是 `ab-a-b`。

这题没有额外状态，结论就是模型本身。答案提取直接输出 `a*b-a-b` 即可，注意要用 `long long` 防止乘法溢出。

### 核心代码

```cpp
long long ans = 1LL * a * b - a - b;
cout << ans;
```

### 复杂度

时间复杂度 `O(1)`，空间复杂度 `O(1)`。

## 6. [【17NOIP提高组】时间复杂度](http://ybt.ssoier.cn:8088/problem_show.php?pid=1900)

`提高组` `模拟` `栈` `语法判定`

### 题意

给出若干段只含 `F` 和 `E` 的伪代码，同时给出期望复杂度 `O(1)` 或 `O(n^w)`。需要判断程序是否语法错误；若无语法错误，再判断真实复杂度是否与给定复杂度一致。

### 分析

这题最容易踩坑的地方是：它不是让你真的执行程序，而是让你根据循环上下界推断“这一层会不会贡献一个 `n`”。所以第一步要把题面里的每个循环翻成三种属性：变量名是否合法、这一层是否会进入、这一层是否会让复杂度指数加一。

判断是否贡献一个 `n` 的关键在于上下界形式。只有“下界是常数、上界是 `n`”这一类循环，会把指数加一；“常数到常数”只是常数层；“`n` 到常数”或“常数但下界大于上界”则是根本不会执行的死循环。死循环内部虽然不参与复杂度计算，但语法错误依然要照样检查。

因此我们用一个栈维护当前未结束的循环，每层记录变量名、它是否让指数 `+1`、它是否把整段后继代码压进死循环。再用 `dead` 表示当前有多少层“不会执行的循环”包着你：只有 `dead==0` 时，`+1` 的贡献才真正计入当前指数。栈顶进出时同步更新 `cur` 和 `mx`，其中 `mx` 始终表示所有可执行位置上的最大指数。

最后先看语法是否出错：包括 `F/E` 不匹配和变量重复定义。若无错误，再把 `mx` 与题目给定的指数比较，相同输出 `Yes`，否则输出 `No`。

### 核心代码

```cpp
for(auto line : prog){
  if(line[0] == 'F'){
    auto [v,l,r] = parse(line);
    if(used.count(v)) err = 1;
    bool dead_loop = bad_range(l, r), add = (!dead && is_num(l) && r == "n" && !dead_loop);
    st.push({v, add, dead_loop}); used.insert(v);
    if(dead_loop) dead++;
    if(add) mx = max(mx, ++cur);
  }else{
    if(st.empty()) err = 1;
    else{
      auto t = st.top(); st.pop(); used.erase(t.v);
      if(t.add) cur--;
      if(t.dead_loop) dead--;
    }
  }
}
if(!st.empty()) err = 1;
```

### 复杂度

设程序行数为 `L`，时间复杂度 `O(L)`，空间复杂度 `O(L)`。

## 7. [【17NOIP提高组】逛公园](http://ybt.ssoier.cn:8088/problem_show.php?pid=1901)

`提高组` `最短路` `记忆化搜索` `环判定`

### 题意

有向图中从 `1` 到 `N` 的最短路长度为 `d`。只统计总长度不超过 `d+K` 的路径条数，答案对 `P` 取模；如果这样的合法路径有无穷多条，输出 `-1`。

### 分析

题面限制不是“路径长度恰好是多少”，而是“比最短路多走不超过 `K`”。这说明普通计数不够，我们必须把“多走出来的那一点代价”也放进状态里。

先从 `1` 跑最短路，记 `dis[u]` 为 `1` 到 `u` 的最短距离。对于一条边 `v -> u`，定义它带来的额外代价 `delta = dis[v] + w - dis[u]`。由最短路三角不等式可知 `delta >= 0`，它恰好表示：如果某条到 `v` 的路径已经比最短路多走了 `k-delta`，再接这条边到 `u`，就会变成“比 `u` 的最短路多走了 `k`”。

于是定义 `f[u][k]` 为“从 `1` 走到 `u`，总长度等于 `dis[u] + k` 的路径数”。转移只需要沿反图枚举前驱：`f[u][k] += f[v][k-delta]`。这里状态设计的好处是，所有额外代价都非负，所以只会从小状态转到大状态，不会凭空制造负环问题。

无穷多条合法路径只会出现在 `delta=0` 的状态环里：你可以在不增加额外代价的前提下绕圈，于是同一个 `(u,k)` 会在递归栈中再次出现。用记忆化搜索时给状态打上“正在访问”的标记，一旦回到栈内状态，就说明存在可达且可贡献答案的零代价环，直接判成 `-1`。否则把 `f[N][0..K]` 求和就是答案。

### 核心代码

```cpp
int dfs(int u,int k){
  if(u == 1) return k == 0;
  int &res = dp[u][k];
  if(res != -1) return res;
  if(vis[u][k] == 1){ inf = 1; return 0; }
  vis[u][k] = 1; res = 0;
  for(auto [v,w] : rev[u]){
    int d = dis[v] + w - dis[u];
    if(d <= k) res = (res + dfs(v, k - d)) % P;
  }
  vis[u][k] = 2;
  return res;
}
```

### 复杂度

最短路部分时间复杂度 `O(M log N)`，动态规划部分时间复杂度 `O((N+M)K)`，空间复杂度 `O(NK)`。

## 8. [【17NOIP提高组】奶酪](http://ybt.ssoier.cn:8088/problem_show.php?pid=1902)

`提高组` `并查集` `空间连通`

### 题意

奶酪里有若干半径相同的球形空洞。如果两个空洞相交或相切，就能从一个走到另一个；如果空洞与下表面或上表面相交或相切，就能从对应表面进出。问能否从下表面一路走到上表面。

### 分析

题面虽然是三维几何，但目标只有“能不能连通”，所以不必真的模拟老鼠运动。只要两个空洞之间有交集，它们就属于同一个可达连通块；下表面和上表面也可以各自看成一个虚拟节点。

于是建模就很直接：每个空洞是一个点，若两球心距离不超过 `2r`，说明两空洞相交或相切，连边并合并；若某个空洞满足 `z <= r`，它和下表面连通；若满足 `h - z <= r`，它和上表面连通。几何条件一旦翻成这些判定，不变量就变成了并查集里的代表元：同一集合中的点一定互相可达。

最后只需判断“下表面虚拟点”和“上表面虚拟点”是否落在同一个集合里。若是，就输出 `Yes`；否则输出 `No`。

### 核心代码

```cpp
for(int i = 1; i <= n; i++){
  if(z[i] <= r) unite(S, i);
  if(h - z[i] <= r) unite(T, i);
  for(int j = 1; j < i; j++){
    long long dx = x[i] - x[j], dy = y[i] - y[j], dz = z[i] - z[j];
    if(dx * dx + dy * dy + dz * dz <= 4LL * r * r) unite(i, j);
  }
}
cout << (find(S) == find(T) ? "Yes" : "No") << endl;
```

### 复杂度

时间复杂度 `O(n^2 α(n))`，空间复杂度 `O(n)`。

## 9. [【17NOIP提高组】宝藏](http://ybt.ssoier.cn:8088/problem_show.php?pid=1903)

`提高组` `状态压缩动态规划` `分层建树`

### 题意

有 `n` 个宝藏屋和若干道路。可以免费任选一个宝藏屋从地面打通作为根，然后逐步开发道路把其他宝藏屋接进来。若一条新道路的起点离根经过了 `d` 个宝藏屋，那么这条道路的代价是 `长度 × d`。要求把所有宝藏屋都挖到，总代价最小。

### 分析

真正决定代价的不是“连了哪些边”，而是“每个点在树里的深度”。因为一条边的系数只取决于它父节点离根有多深，所以这题本质是在找一棵根树，并让边长按层数加权。

既然 `n ≤ 12`，就该考虑状态压缩。把整棵树按深度分层：第 `1` 层只有根，第 `2` 层由根直接连到的点组成，第 `3` 层由第 `2` 层扩展出来，依此类推。这样一来，新加入的一整层 `U` 只能从上一层 `T` 连过去，而且这一层里每个点 `v` 最优的父亲一定是 `T` 中与它边长最短的那个点，所以该层总代价就是 `depth × Σ min(w(u,v)) for u∈T`。

于是定义 `dp[depth][S][T]`：已经挖出的点集是 `S`，最深一层恰好是 `T`，并且 `T` 处在第 `depth` 层时的最小代价。转移时枚举下一层 `U`，把所有 `v∈U` 都从 `T` 中挑最便宜的入口接进来。这个状态设计把“树结构怎么选”压成了“每一层选哪些点”。

关键不变量是：`T` 始终表示最后一层，所以再往外扩展时的乘数一定是当前层号 `depth`。当 `S` 变成全集时，所有层构造完毕，取这些终态里的最小值就是答案。

### 核心代码

```cpp
for(int r = 0; r < n; r++) dp[1][1 << r][1 << r] = 0;
for(int dep = 1; dep < n; dep++)
  for(int S = 0; S < (1 << n); S++)
    for(int T = S; T; T = (T - 1) & S) if(dp[dep][S][T] < INF){
      int R = ((1 << n) - 1) ^ S;
      for(int U = R; U; U = (U - 1) & R)
        if(cost[T][U] < INF)
          dp[dep + 1][S | U][U] = min(dp[dep + 1][S | U][U],
            dp[dep][S][T] + 1LL * dep * cost[T][U]);
    }
```

### 复杂度

预处理和转移合计时间复杂度约为 `O(n^2 3^n + n 4^n)`，空间复杂度 `O(n 3^n)`。

## 10. [【17NOIP提高组】列队](http://ybt.ssoier.cn:8088/problem_show.php?pid=1904)

`提高组` `数据结构` `动态序列` `线段树`

### 题意

一个 `n × m` 方阵反复发生“第 `x` 行第 `y` 列的学生离队”事件。离队后先做一次“向左看齐”，再做一次“向前看齐”，最后离队学生回到右下角空位。要求输出每次离队学生原本的编号。

### 分析

这题最难的不是删人，而是先看清整体运动规律。把每一行拆开看：在“向左看齐”之后，第 `x` 行只有前 `m-1` 个位置会发生左移，第 `m` 列留下一个空位。接着“向前看齐”时，每一行的最后一列会整体向前补位。于是整张方阵可以被压成两类序列：

- 每一行维护一个长度为 `m-1` 的横向序列；
- 所有行的最后一列按从上到下组成一个纵向序列。

这样查询 `(x,y)` 时，若 `y<m`，先从第 `x` 行横向序列里弹出第 `y` 个元素，这就是离队学生；然后从纵向序列里弹出第 `x` 个元素补到该行尾部；最后把离队学生追加到纵向序列尾部，对应“回到右下角”。若 `y==m`，则直接从纵向序列里弹出第 `x` 个元素，再把它追加到尾部即可。

因此整题的核心不变量是：任意时刻，第 `i` 行前 `m-1` 个位置恰好对应第 `i` 个横向序列，最后一列恰好对应纵向序列的第 `i` 个元素。剩下的问题就变成“动态序列上删除第 `k` 个元素并在末尾追加”。由于初始编号是等差形式，再加上只会发生删除和追加，可以用动态开点线段树维护“哪些位置已经被删掉”，通过第 `k` 个未删除元素定位真实编号。

答案提取发生在每次弹出操作本身：第一次弹出的那个编号，就是这次离队事件真正离开的学生编号。

### 核心代码

```cpp
long long ask(int x,int y){
  if(y == m){
    long long v = pop_col(x);
    col.push_back(v);
    return v;
  }
  long long v = pop_row(x, y);
  long long tail = pop_col(x);
  row[x].push_back(tail);
  col.push_back(v);
  return v;
}
long long pop_row(int x,int k){
  int pos = kth(rtRow[x], 1, m - 1 + row[x].size(), k);
  erase(rtRow[x], 1, m - 1 + row[x].size(), pos);
  if(pos < m) return 1LL * (x - 1) * m + pos;
  return row[x][pos - m];
}
```

### 复杂度

每次查询只做若干次线段树第 `k` 小与删除，时间复杂度 `O(log(n+q) + log(m+q))`，总空间复杂度 `O(n+q)`。
