# 一本通 CSP 2019赛题解题报告

这一年的 CSP 从最朴素的字符串统计、规则模拟出发，很快就把思路推进到时间轴决策、图上奇偶可达与树上括号状态；到了提高组，题目又继续往递归构造、结构贪心、组合计数和单调优化延伸。读这组题时，最值得反复体会的，是题面限制如何被压成一个足够稳定的状态：一旦状态立住，后面的转移、贪心和数据结构就都顺了。

# 一、CSP-J 赛题

CSP-J 这一组题的共同训练点，不是机械模拟，而是把优惠规则、买卖收益和生产层数分别翻译成队列、背包和奇偶最短路；先完成这一步，后面的实现才会自然。

## 1. [【19CSPJ普及组】数字游戏](http://ybt.ssoier.cn:8088/problem_show.php?pid=1982)

`CSP 2019` `CSP-J`

### 题意

给出一个长度固定为 8 的 01 串，要求输出其中字符 `1` 出现的次数。

### 分析

这题最重要的是先把题面压成一个最简单的状态：设 `ans` 表示已经扫描过的前缀里有多少个 `1`。题目没有任何跨位置的关联条件，所以每一位是否贡献答案，只取决于这一位自己是不是 `1`。

因此从左到右扫一遍字符串即可。若当前字符是 `'1'`，就把 `ans` 加一；若是 `'0'`，直接跳过。因为字符串长度固定且字符种类也已经限定，整个过程就是一次纯线性的计数。

扫描结束时，`ans` 已经累计了全部有效贡献，直接输出就是答案。这类题训练的不是技巧，而是看到叙述后能立刻落到“逐字符独立判定”的模型。

### 核心代码

```cpp
int ans = 0;
for(char ch : s){
  if(ch == '1') ans++;
}
cout << ans;
```

### 复杂度

时间复杂度 $O(|s|)$，空间复杂度 $O(1)$。

## 2. [【19CSPJ普及组】公交换乘](http://ybt.ssoier.cn:8088/problem_show.php?pid=1983)

`CSP 2019` `CSP-J`

### 题意

按时间顺序给出若干次乘车记录。乘一次地铁会得到一张持续 $45$ 分钟、且票价上限为该次地铁票价的优惠票；坐公交时若存在可用优惠票，就必须使用最早获得的那一张。要求求出总花费。

### 分析

题面的关键对象其实不是“乘车记录”，而是“地铁产生的优惠票”。每次地铁都对应一张带两个属性的券：获得时间 `t` 和可免费乘坐的最高公交票价 `price`。公交记录到来时，我们只需要判断当前手里是否存在一张同时满足“没过期”和“票价够高”的券。

真正容易写乱的地方是“如果能用，就一定用获得最早的那张”。这意味着我们不能只看是否存在可用券，还要在所有可用券里选时间最小的那个。由于输入已经按时间顺序给出，同一票价的优惠票天然会按获得顺序进入，因此可以把每种地铁票价单独开一个队列，队首永远是这种票价里最早且尚未处理的那张券。

当处理一辆票价为 `p` 的公交、时刻为 `t` 时，先把所有票价区间 `[p,1000]` 的队首中过期的券弹掉；随后在这些队列的队首里找时间最小的一张，如果存在就把它消费掉，否则只能自己付钱。因为票价上界只有 $1000$，逐票价扫描是完全可行的，而且它恰好把“全局最早可用券”的规则翻译得很直接。

### 核心代码

```cpp
deque<int> q[1001];
int ans = 0;
for(auto [tp, price, t] : rec){
  if(tp == 0){ ans += price; q[price].push_back(t); continue; }
  int best = -1, tim = 1e9;
  for(int v = price; v <= 1000; v++){
    while(!q[v].empty() && t - q[v].front() > 45) q[v].pop_front();
    if(!q[v].empty() && q[v].front() < tim) tim = q[v].front(), best = v;
  }
  if(best == -1) ans += price;
  else q[best].pop_front();
}
```

### 复杂度

时间复杂度 $O(1000n)$，空间复杂度 $O(n)$。

## 3. [【19CSPJ普及组】纪念品](http://ybt.ssoier.cn:8088/problem_show.php?pid=1984)

`CSP 2019` `CSP-J`

### 题意

已知未来 $T$ 天里 $N$ 种纪念品每天的价格，初始有 `M` 枚金币。每天都可以无限次买卖任意纪念品，最后一天必须清仓，要求最大化最终金币数。

### 分析

这题如果盯着“未来 $T$ 天整体怎么操作”会很乱，但题目其实给了一个很强的分层信号：每天都允许先卖再买，而且第二天以后真正影响未来的只有“你手里还剩多少金币”。所以可以把问题拆成若干个相邻两天之间的财富增长：已知今天手里有 `money`，如何通过今天买入、明天卖出，让明天的金币尽量多。

对某一种纪念品，若今天买入价是 `a`、明天卖出价是 `b`，买一件的净收益就是 `b-a`。若这个值不正，就完全没有买它的理由；若它为正，而且同一种纪念品可以无限次购买，那么这就是一个“花费 `a`，收益 `b-a`”的完全背包物品。

于是每天只要做一次完全背包：`dp[c]` 表示今天花掉 `c` 枚金币后，最多还能额外赚到多少。最后把 `money` 加上 `dp` 的最大值，就得到下一天开盘前能拥有的最多金币。这样日与日之间通过一个数字传递，整个模型就稳定了。

### 核心代码

```cpp
for(int day = 1; day < T; day++){
  fill(dp, dp + money + 1, 0);
  for(int k = 1; k <= N; k++) if(a[day + 1][k] > a[day][k]){
    int cost = a[day][k], gain = a[day + 1][k] - a[day][k];
    for(int c = cost; c <= money; c++)
      dp[c] = max(dp[c], dp[c - cost] + gain);
  }
  money += *max_element(dp, dp + money + 1);
}
```

### 复杂度

时间复杂度 $O(TNM)$，空间复杂度 $O(M)$。

## 4. [【19CSPJ普及组】加工零件](http://ybt.ssoier.cn:8088/problem_show.php?pid=1985)

`CSP 2019` `CSP-J`

### 题意

给出一张无向图和若干询问 `(a,L)`。若工人 `a` 想生产一个第 `L` 阶段零件，问工人 `1` 是否会在递归展开后成为提供原材料的人。

### 分析

先把递归定义翻译成图上的运动：生产一个第 $1$ 阶段零件，会向所有相邻点要原材料；生产一个第 $2$ 阶段零件，会先让邻点去生产第 $1$ 阶段，于是原材料需求又沿边往外扩一层。继续展开后就会发现，工人 `1` 需要出原料，当且仅当存在一条从 `a` 到 `1`、长度恰好为 `L` 的走法。

但询问里的 `L` 很大，不能按长度一层层扩。无向图里有一个很有用的事实：如果已经找到一条到达 `1` 的走法，之后随时可以在某条边上来回走两步，所以长度可以额外增加任意偶数。因此真正决定答案的，只是“到 `1` 的最短偶数长度走法”和“到 `1` 的最短奇数长度走法”。

这正好对应一个奇偶分层最短路。把状态写成 `(u,0/1)`，表示走到点 `u` 时路径长度的奇偶性；沿一条边转移时奇偶性翻转。用 BFS 求出从 `1` 出发到每个点两种奇偶状态的最短长度后，查询只需检查 `dist[a][L&1]` 是否存在且不超过 `L`。

### 核心代码

```cpp
queue<pair<int,int>> q;
memset(dist, -1, sizeof dist);
dist[1][0] = 0, q.push({1, 0});
while(!q.empty()){
  auto [u, p] = q.front(); q.pop();
  for(int v : g[u]) if(dist[v][p ^ 1] == -1){
    dist[v][p ^ 1] = dist[u][p] + 1;
    q.push({v, p ^ 1});
  }
}
puts(dist[a][L & 1] != -1 && dist[a][L & 1] <= L ? "Yes" : "No");
```

### 复杂度

时间复杂度 $O(n+m+q)$，空间复杂度 $O(n+m)$。

# 二、CSP-S 赛题

CSP-S 更强调结构抽象：有的题把递归定义压成位公式，有的题把括号串搬到树上维护，有的题则要用贪心、计数 DP 或单调队列去守住全局最优。读的时候可以多问一句：当前真正需要维护的全局约束到底是什么。

## 5. [【19CSPS提高组】格雷码](http://ybt.ssoier.cn:8088/problem_show.php?pid=1986)

`CSP 2019` `CSP-S`

### 题意

按题目给定的反射构造法，求 `n` 位格雷码中编号为 `k` 的那个二进制串。

### 分析

题面虽然把构造过程描述得很长，但真正要抓住的是“前半段加前缀 `0`，后半段把前一层倒过来再加前缀 `1`”这件事。也就是说，第 `k` 个串在每一位取什么，不必真的把整层序列生成出来，只要知道它在这一层落在前半还是后半即可。

继续按位拆下去，会得到一个标准结论：反射格雷码中第 `k` 个数的数值，正是 `k` 与 `k` 右移一位后的异或，也就是 $g = k \oplus (k \gg 1)$。这个公式本质上就是把“是否翻转这一位”浓缩成了高位区间编号的奇偶性。

因此做法就只剩两步：先算出 `g`，再从高位到低位依次输出它的二进制表示。这样既和递归构造完全一致，又避免了指数级展开。

### 核心代码

```cpp
unsigned long long g = k ^ (k >> 1);
for(int i = n - 1; i >= 0; i--)
  cout << ((g >> i) & 1);
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(1)$。

## 6. [【19CSPS提高组】括号树](http://ybt.ssoier.cn:8088/problem_show.php?pid=1987)

`CSP 2019` `CSP-S`

### 题意

树上每个点放一个括号。对每个点 `i`，考虑根到 `i` 的括号串 $s_i$，求其中有多少个不同位置的子串是合法括号串，最后输出 $\bigoplus_{i=1}^n i\cdot k_i$。

### 分析

题目的“不同子串”是按位置区分的，所以不需要去重结构，只要把根到当前点这条路径上所有合法区间都数出来即可。对一条普通括号序列，常见做法是令 $f[pos]$ 表示“以当前位置结尾的合法括号串个数”。一旦这个状态能维护，整条前缀里的合法子串总数就只是这些 $f$ 的前缀和。

树上路径看似变了，实际上 DFS 的栈序正好就是当前根到点的括号序列。沿着 DFS 维护一个“尚未匹配的左括号栈”：遇到 `'('` 就压栈；遇到 `')'` 且栈非空时，把栈顶左括号 `x` 与当前点 `u` 配成一对。此时区间 `(x,u)` 自己贡献一个新的合法串，而任何一个“恰好以 $fa[x]$ 结尾”的合法串，都能把这一对接在后面继续变成以 `u` 结尾的合法串，所以有 $f[u] = f[fa[x]] + 1$。

再令 $sum[u] = sum[fa[u]] + f[u]$，表示整条根到 `u` 路径上的合法子串总数。原因是每个合法子串都有唯一的结束位置，所有结束在不同点的贡献正好可以分开累加。DFS 到 `u` 时用 $u \cdot sum[u]$ 更新异或答案，回溯时把栈恢复，就完成了整棵树的统计。

### 核心代码

```cpp
void dfs(int u){
  int x = 0;
  if(s[u] == '(') st.push_back(u);
  else if(!st.empty()){
    x = st.back(); st.pop_back();
    f[u] = f[fa[x]] + 1;
  }
  sum[u] = sum[fa[u]] + f[u];
  ans ^= 1LL * u * sum[u];
  for(int v : ch[u]) dfs(v);
  if(s[u] == '(') st.pop_back();
  else if(x) st.push_back(x);
}
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 7. [【19CSPS提高组】树上的数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1988)

`CSP 2019` `CSP-S`

### 题意

树上初始放着 $1$ 到 $n$ 的一个排列。每次删去一条尚未删除的边，先交换它两端的数字，再删边；所有边删完后，要求最终按数字顺序得到的结点编号排列字典序最小。

### 分析

字典序最小的第一反应一定是贪心：先让数字 `1` 尽量去编号最小的结点，再在不破坏这一决定的前提下处理数字 `2`，如此类推。难点不在“想不想过去”，而在“这条搬运路径能否对应某个合法的删边顺序”。

若一个数字想沿着一条路径从起点走到终点，那么起点那条路径边必须是它所有 incident 边里最先删掉的，终点那条路径边必须是最后删掉的，而中间点的入边和出边则必须在本点的删边次序里相邻。也就是说，每个点周围的边会逐渐被拼成若干条局部链，合法路径就是把一条新链接到这些局部约束里，而且不能产生冲突或提前成环。

于是状态维护的核心就变成了“每个点当前链的头尾是谁”，以及“哪些边已经被并到同一条链里”。用并查集维护边链连通性，用 `head`、`tail` 记录某点当前删边序列的首尾，再配合 DFS 去尝试把当前数字接到某个目标点的路径上；一旦沿路所有点都能满足起点、终点或中转点的角色，就把这条路径对应的链合并操作真正提交。

这题最难的地方，是把“数字移动”转换成“删边顺序存在性”的判定。外层贪心只负责字典序，真正保住正确性的，是路径上每个点的局部顺序约束都能同时成立。

### 核心代码

```cpp
bool connect(int u, int inE, int outE){
  if(inE && tail[u] && find(tail[u]) != find(inE)) return false;
  if(outE && head[u] && find(head[u]) != find(outE)) return false;
  if(inE && outE && find(inE) == find(outE)) return false;
  if(inE && outE) nxt[rep(inE)] = rep(outE), merge(inE, outE);
  if(inE) tail[u] = rep(inE);
  if(outE) head[u] = rep(outE);
  return true;
}
for(int x = 1; x <= n; x++)
  for(int t = 1; t <= n; t++) if(!used[t] && dfs(pos[x], 0, t))
    used[t] = 1, ans[x] = t, apply(), t = n;
```

### 复杂度

时间复杂度 $O(n^2\alpha(n))$，空间复杂度 $O(n)$。

## 8. [【19CSPS提高组】Emiya家今天的饭](http://ybt.ssoier.cn:8088/problem_show.php?pid=1989)

`CSP 2019` `CSP-S`

### 题意

有一个 $n\times m$ 的菜品计数矩阵 `a`。每种方案至多从每一行选一道菜，且某种主食材出现次数不能超过所选总菜数的一半，求合法方案数。

### 分析

限制里最难处理的是“某种食材不能超过总数一半”。直接正着数很别扭，但补集却很自然：先算出“每行选零道或一道菜”的总方案数，再减去不合法方案。由于若某一种食材超过一半，其他食材就不可能同时也超过一半，所以不合法方案可以按“哪一列超标”分别统计，最后直接相减。

固定一列 `t` 作为超标食材。此时每一行只有三类选择：不选、选一份食材 `t` 的菜、选一份其他食材的菜。真正影响是否合法的不是绝对选了多少道，而是差值 $cnt_t-cnt_{other}$：最后如果这个差值大于 $0$，就说明食材 `t` 的出现次数严格多于其余食材总和，也就是超过了一半。

因此可以做一个差值 DP。把状态写成 `dp[i][d]`，表示处理完前 `i` 行后，差值为 `d-n` 的方案数；选 `t` 就让差值加一，选其他食材就减一，不选则不变。每一列单独跑一遍，累加所有 `d>n` 的状态就是“这一列超标”的方案数。用总方案数减掉这些非法方案，就是答案。

### 核心代码

```cpp
long long ans = 1;
for(int i = 1; i <= n; i++) ans = ans * (row[i] + 1) % MOD;
ans = (ans - 1 + MOD) % MOD;
for(int t = 1; t <= m; t++){
  fill(dp, dp + 2 * n + 1, 0), dp[n] = 1;
  for(int i = 1; i <= n; i++){
    fill(ndp, ndp + 2 * n + 1, 0);
    for(int d = n - i + 1; d <= n + i - 1; d++) if(dp[d]){
      add(ndp[d], dp[d]);
      add(ndp[d + 1], dp[d] * a[i][t] % MOD);
      add(ndp[d - 1], dp[d] * (row[i] - a[i][t] + MOD) % MOD);
    }
    swap(dp, ndp);
  }
  for(int d = n + 1; d <= 2 * n; d++) ans = (ans - dp[d] + MOD) % MOD;
}
```

### 复杂度

时间复杂度 $O(mn^2)$，空间复杂度 $O(n)$。

## 9. [【19CSPS提高组】划分](http://ybt.ssoier.cn:8088/problem_show.php?pid=1990)

`CSP 2019` `CSP-S`

### 题意

把正整数序列划分成若干连续段，要求段和单调不降，并最小化所有段和平方的总和。数据极大，还可能通过生成器在线产生。

### 分析

先设前缀和为 `s`。如果 `pre[i]` 是最优划分里第 `i` 个位置之前的最后一个断点，那么最后一段的和就是 `s[i]-s[pre[i]]`。为了让段和不降，上一段的和 `s[pre[i]]-s[pre[pre[i]]]` 必须不超过它，于是可以整理出判定式 $2s[pre[i]]-s[pre[pre[i]]]\le s[i]$。这一步把“单调不降”的全局约束压成了只和前一个断点有关的局部条件。

接下来要想清楚为什么只选最大的合法断点。因为所有 $a_i$ 都是正数，断点越靠右，最后一段越短、段和越小，而平方函数是凸的，让最后一段尽量短只会更优，不会更差。所以对每个 `i`，最优决策就是满足条件的最大 `j`。

于是状态就从枚举所有 `j` 变成了“找最大的合法 `j`”。令 $key[j]=2s[j]-s[pre[j]]$，随着 `j` 推进它是单调的，可以用单调队列维护候选断点；处理到 `i` 时，把所有 $key$ 不超过 $s[i]$ 的队首弹过去，最后留下的队首就是最大的合法断点。这样整题就从一个看似高维的 DP 压成了线性扫描。

### 核心代码

```cpp
q[0] = 0;
for(int i = 1; i <= n; i++){
  while(l < r && key[q[l + 1]] <= s[i]) l++;
  pre[i] = q[l];
  f[i] = f[pre[i]] + sqr(s[i] - s[pre[i]]);
  key[i] = 2 * s[i] - s[pre[i]];
  while(l <= r && key[q[r]] >= key[i]) r--;
  q[++r] = i;
}
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 10. [【19CSPS提高组】树的重心](http://ybt.ssoier.cn:8088/problem_show.php?pid=1991)

`CSP 2019` `CSP-S`

### 题意

对树的每一条边，单独删去它后会得到两棵子树。要求把这两棵子树各自所有重心的编号和累加起来，并对所有边求总和。

### 分析

如果每删一条边都重新 DFS 求两棵子树的重心，复杂度显然爆炸。真正要抓住的是重心的移动规律：在一棵大小为 `tot` 的树里，若当前点存在某个连通块大小大于 `tot/2`，那么重心一定要朝那个连通块继续走；反过来，一旦所有连通块都不超过一半，当前点就是重心。也就是说，寻找重心本质上是在沿“唯一过大”的方向不断前进。

对于断边后得到的子树，这个“过大方向”只可能落在若干重儿子链上，所以可以先预处理子树大小、父子关系和重儿子，再用倍增或跳链快速找到第一个满足条件的点。对边 `(u,v)` 而言，若 `v` 是 `u` 的儿子，一侧就是 `v` 的整棵子树，另一侧则是把这棵子树从原树里切掉后的补树；两边都可以用同样的“沿过大块跳到重心”思路处理。

还要注意重心可能有两个。当某个方向的最大连通块大小恰好等于 `tot/2` 时，当前点和那个相邻点都会成为重心，所以贡献要把两个编号都算上。于是整题的关键就从“每次重算整棵树”转成了“如何快速定位断边后两边的过大块”，这正是树上预处理能发挥作用的地方。

### 核心代码

```cpp
int climb(int u, int tot){
  while(heavyPart(u, tot) > tot / 2) u = heavySon(u, tot);
  return u;
}
long long calc(int root, int ban, int tot){
  int c = climb(root, tot);
  long long res = c;
  int v = nextPart(c, ban, tot);
  if(v && partSize(c, v, ban, tot) == tot / 2) res += v;
  return res;
}
for(auto [u, v] : edges)
  ans += fa[v] == u ? calc(v, 0, sz[v]) + calc(u, v, n - sz[v])
                    : calc(u, 0, sz[u]) + calc(v, u, n - sz[u]);
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(n\log n)$。
