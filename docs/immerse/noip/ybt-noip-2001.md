# 一本通 NOIP 2001赛题解题报告

这一组题从普及组的递推、数论、树形重建与背包起步，逐步过渡到提高组的方程求根、分段动态规划和图上最短路。读的时候可以一直抓住同一条主线：先把题目条件翻成状态、结构或图，再顺着不变量把答案从状态里提出来。

# 一、普及组赛题

这一部分的题都很适合练“把叙述条件翻译成标准模型”的基本功：有的要把构造过程改写成递推，有的要先拆掉公因子，有的要从遍历序列倒推树结构。方法不难，但每题都有一个决定成败的建模拐点。

## 1. [【01NOIP普及组】数的计数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1914)
`NOIP 2001` `普及组`

### 题意

给定自然数 `n`。从 `n` 开始，可以选择不再处理，也可以在当前数左边添上一个不超过当前数一半的自然数，再对添好后的整体继续重复同样规则。问最终一共能构造出多少个不同的数，且把 `n` 本身也算进去。

### 分析

这题最容易卡住的地方，是把“不断往左添数”的构造过程当成字符串搜索去想。其实题目真正关心的不是长什么样，而是**最后一步左边添了多少**。如果最后一次添的是 `j`，那么右边那一整段已经是一种合法构造，而 `j` 只需要满足 `j <= 当前整体的一半`，于是问题自然会回到“比它小的那些数有多少种合法构造”。

设 `f[i]` 表示以 `i` 作为当前整体时，能得到的合法数的个数。什么都不添是一种方案，所以先有 `f[i]=1`；如果最后一次在左边添了 `j`，那么右侧部分必须是一种以 `j` 为起点继续扩展出的合法结果，且 `j<=i/2`，因此 `f[i]=1+\sum_{j=1}^{\lfloor i/2 
floor} f[j]`。状态含义一旦立住，答案就是 `f[n]`，因为题目要的正是从 `n` 出发能形成的全部结果。

### 核心代码

```cpp
int f[1005] = {};
f[1] = 1;
for(int i = 2; i <= n; i++){
  f[i] = 1;
  for(int j = 1; j <= i / 2; j++) f[i] += f[j];
}
cout << f[n];
```

### 复杂度

时间复杂度 `O(n^2)`，空间复杂度 `O(n)`。

## 2. [【01NOIP普及组】最大公约数与最小公倍数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1915)
`NOIP 2001` `普及组`

### 题意

给定 `x0` 和 `y0`，要求统计有多少组正整数有序对 `(P,Q)`，满足 `gcd(P,Q)=x0` 且 `lcm(P,Q)=y0`。

### 分析

看到最大公约数和最小公倍数同时给出，第一步应该先把公共部分拆掉。令 `P=x0*a`、`Q=x0*b`，那么 `gcd(a,b)=1`；再由 `P*Q=gcd(P,Q)*lcm(P,Q)` 得到 `ab=y0/x0`。这样原题就变成：把 `m=y0/x0` 的所有质因子幂次分给 `a` 和 `b`，并且还要保证二者互质。

互质这个条件非常关键，它意味着同一个质因子的整段幂次只能完整地放进一边，不能拆开。于是如果 `m` 有 `k` 个不同的质因子，每个质因子都只有“给 `a`”或“给 `b`”两种选择，所以答案就是 `2^k`。如果 `y0` 不能被 `x0` 整除，那连 `m` 都不存在，答案直接为 `0`。最终只要分解 `m`，统计不同质因子的个数即可。

### 核心代码

```cpp
if(y0 % x0){
  cout << 0;
  return;
}
int m = y0 / x0, cnt = 0;
for(int p = 2; p * p <= m; p++) if(m % p == 0){
  cnt++;
  while(m % p == 0) m /= p;
}
if(m > 1) cnt++;
cout << (1 << cnt);
```

### 复杂度

时间复杂度 `O(\sqrt{y0/x0})`，空间复杂度 `O(1)`。

## 3. [【01NOIP普及组】求先序排列](http://ybt.ssoier.cn:8088/problem_show.php?pid=1916)
`NOIP 2001` `普及组`

### 题意

输入一棵二叉树的中序序列和后序序列，结点字符互不相同，要求恢复它的先序序列。

### 分析

后序遍历的最后一个字符一定是整棵树的根，这是这题最核心的突破口。根在中序序列里会把整棵树切成“左子树部分”和“右子树部分”，而后序序列除了最后一个根之外，前面那一段也正好能按同样长度切成左、右两块。也就是说，只要知道根在哪，左右子树的区间就全定了。

于是递归就顺下来了：先从后序末尾拿根，再用它在中序里的位置划分左右子树，然后分别递归求左、右子树的先序。先序的提取方式也很直接——根一定最先输出，所以返回值就是“根 + 左子树先序 + 右子树先序”。这正好对应了答案的提取顺序。

### 核心代码

```cpp
string build(string in, string post){
  if(in.empty()) return "";
  char rt = post.back();
  int k = in.find(rt);
  string L = build(in.substr(0, k), post.substr(0, k));
  string R = build(in.substr(k + 1), post.substr(k, in.size() - k - 1));
  return string(1, rt) + L + R;
}
```

### 复杂度

时间复杂度 `O(n^2)`，空间复杂度 `O(n)`，其中 `n<=8`。

## 4. [【01NOIP普及组】装箱问题](http://ybt.ssoier.cn:8088/problem_show.php?pid=1917)
`NOIP 2001` `普及组`

### 题意

给定箱子容量 `V` 和 `n` 个物品体积。每个物品最多选一次，要求选出若干个物品放入箱中，使剩余空间尽量小，输出这个最小剩余空间。

### 分析

“剩余最小”这句话，通常要先反过来读成“装进去的总体积尽量大，但不能超过 `V`”。这样题目立刻就落回最经典的 `0/1` 背包：每个物品只有“拿”与“不拿”两种决策，状态就是当前容量下最多能装多少体积。

设 `dp[j]` 表示容量恰好限制为 `j` 时，最多能装入的总体积。枚举一个体积为 `x` 的物品时，倒序更新 `dp[j]=max(dp[j],dp[j-x]+x)`，倒序的含义是不让同一件物品被重复使用。所有物品处理完后，`dp[V]` 就是容量 `V` 下能装到的最大总体积，因此答案直接取 `V-dp[V]`。

### 核心代码

```cpp
vector<int> dp(V + 1, 0);
for(int x : a)
  for(int j = V; j >= x; j--)
    dp[j] = max(dp[j], dp[j - x] + x);
cout << V - dp[V];
```

### 复杂度

时间复杂度 `O(nV)`，空间复杂度 `O(V)`。

# 二、提高组赛题

进入提高组之后，题目不再停留在直接套模板，而是更强调先拆层次：先找可二分的区间，再把划分条件压成动态规划，最后把路线问题完整落成带权图。读这一组时，重点看“状态为什么这样定义”和“答案最后从哪里取”。

## 5. [【01NOIP提高组】一元三次方程求解](http://ybt.ssoier.cn:8088/problem_show.php?pid=1824)
`NOIP 2001` `提高组`

### 题意

给出一元三次方程 `ax^3+bx^2+cx+d=0` 的四个系数，已知它在 `[-100,100]` 内有三个互不相同的实根，且任意两根之差的绝对值至少为 `1`。要求按从小到大的顺序输出这三个实根。

### 分析

题目已经把做法暗示得很明显：如果在两个点 `x1<x2` 之间函数值异号，那么这个区间里一定有根。真正该抓住的是“根与根之间至少相距 `1`”，这说明把区间按整数拆成 `[i,i+1]` 之后，每个小段里至多只会藏一个根，扫描时不会把两个根混在同一次二分里。

因此先从 `-100` 到 `100` 逐段检查：若左端点就是根，直接记下；若 `f(i)` 与 `f(i+1)` 异号，就在这个长度为 `1` 的区间里二分逼近。二分过程维持的不变量是“当前区间两端函数值异号”，每次保留仍然异号的那一半，区间就会稳定缩到唯一实根附近。所有找到的根天然按扫描顺序递增，直接依次输出即可。

### 核心代码

```cpp
auto f = [&](double x){ return ((a * x + b) * x + c) * x + d; };
vector<double> ans;
for(int i = -100; i < 100; i++){
  double l = i, r = i + 1, fl = f(l), fr = f(r);
  if(fabs(fl) < 1e-8) ans.push_back(l);
  else if(fl * fr < 0){
    for(int t = 0; t < 60; t++){
      double m = (l + r) / 2;
      if(f(l) * f(m) <= 0) r = m;
      else l = m;
    }
    ans.push_back((l + r) / 2);
  }
}
if(fabs(f(100)) < 1e-8) ans.push_back(100);
```

### 复杂度

时间复杂度 `O(区间数 × 二分轮数)`，这里可视为 `O(200 × 60)`，空间复杂度 `O(1)`。

## 6. [【01NOIP提高组】数的划分](http://ybt.ssoier.cn:8088/problem_show.php?pid=1825)
`NOIP 2001` `提高组`

### 题意

把整数 `n` 划分成 `k` 份，每份都不能为空，并且不考虑顺序，也就是把相同数字的重排视为同一种方案。要求输出不同划分的总数。

### 分析

“不考虑顺序”这句话，等价于把每种划分都按非降顺序写出来。这样每一种合法答案只会对应一种表示方式，重复就自然消掉了。接下来要找一个能把方案分开的分类标准，最顺手的是看最小值是不是 `1`。

设 `f[i][j]` 表示把 `i` 划成 `j` 份的方案数。若某个划分里有一份是 `1`，去掉这个 `1` 后，就变成把 `i-1` 划成 `j-1` 份，对应 `f[i-1][j-1]`；若所有份数都至少是 `2`，那么每份都减去 `1`，就变成把 `i-j` 划成 `j` 份，对应 `f[i-j][j]`。于是有 `f[i][j]=f[i-1][j-1]+f[i-j][j]`。这个转移正好把所有情况无重无漏地分完，最终答案就是 `f[n][k]`。

### 核心代码

```cpp
int f[205][10] = {};
for(int i = 1; i <= n; i++) f[i][1] = 1;
for(int i = 1; i <= n; i++)
  for(int j = 2; j <= k && j <= i; j++){
    f[i][j] = f[i - 1][j - 1];
    if(i >= j) f[i][j] += f[i - j][j];
  }
cout << f[n][k];
```

### 复杂度

时间复杂度 `O(nk)`，空间复杂度 `O(nk)`。

## 7. [【01NOIP提高组】统计单词个数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1826)
`NOIP 2001` `提高组`

### 题意

给定一个总长度不超过 `200` 的小写字母串，需要把它切成 `k` 段。每一段里，用给定字典中的单词去匹配，单词可以相互重叠，但一旦选了某个单词，就不能再选与它拥有同一个起始位置的另一个单词。要求让所有段内能选到的单词总数最大。

### 分析

这题有两层决策：第一层是在哪里切段，第二层是某一段内部最多能贡献多少个单词。先看第二层。因为限制只落在“同一个起始位置不能重复取词”，所以对固定区间 `[l,r]` 来说，真正要数的是：有多少个起点 `p`，存在至少一个字典单词能从 `p` 开始并完整落在 `[l,r]` 内。只要这个起点能匹配至少一个词，它就能贡献 `1`。于是可以先预处理 `cnt[l][r]`。

预处理完一段的价值之后，整题就变成标准分段动态规划。设 `dp[i][j]` 表示前 `i` 个字符分成 `j` 段时的最大单词数，最后一刀切在 `t` 之后，就有 `dp[i][j]=max(dp[t][j-1]+cnt[t+1][i])`。这里的状态含义很清楚：前一部分已经分完，最后一段把剩下字符整段吃掉。枚举完最后一个断点，`dp[n][k]` 就是最终答案。

### 核心代码

```cpp
for(int l = 1; l <= n; l++)
  for(int r = l; r <= n; r++)
    for(int i = l; i <= r; i++)
      for(string w : dic)
        if(i + (int)w.size() - 1 <= r && s.substr(i - 1, w.size()) == w){
          cnt[l][r]++;
          break;
        }
for(int i = 1; i <= n; i++) dp[i][1] = cnt[1][i];
for(int i = 1; i <= n; i++)
  for(int j = 2; j <= k; j++)
    for(int t = j - 1; t < i; t++)
      dp[i][j] = max(dp[i][j], dp[t][j - 1] + cnt[t + 1][i]);
```

### 复杂度

设串长为 `n`、字典大小为 `s`、单词最大长度为 `L`，时间复杂度约为 `O(n^3sL)`，空间复杂度 `O(n^2)`。

## 8. [【01NOIP提高组】Car的旅行路线](http://ybt.ssoier.cn:8088/problem_show.php?pid=1827)
`NOIP 2001` `提高组`

### 题意

有若干城市，每个城市有四个机场，分别位于同一个矩形的四个顶点上；已知其中任意三个顶点坐标和该城市铁路的单位距离费用。城市内部任意两机场之间可以走铁路，不同城市任意两机场之间可以坐飞机，飞机单位距离费用统一为 `t`。要求从城市 `A` 的任意机场出发，到达城市 `B` 的任意机场，总花费最小。

### 分析

先别急着想路线，第一步是把题目完全翻译成图。每个机场都是一个点；同城两机场之间连铁路边，边权是欧氏距离乘本城 `T_i`；不同城市两机场之间连航线边，边权是欧氏距离乘统一票价 `t`。这样“从城市 `A` 到城市 `B` 的最小花费”就变成了一个标准最短路问题。题目说出发机场和到达机场都可以任取，因此源点其实是城市 `A` 的四个机场，终点是城市 `B` 的四个机场中的最小距离。

还差一个细节：每城只给了三个顶点，第四个顶点要先补出来。矩形的三个点里，总有一个点是直角点；找到这个直角点后，第四点就是“另外两点坐标和减去直角点坐标”。点集补全后，整张图的状态就固定了。最短路过程中保持的不变量是：`dist[u]` 始终表示目前已知到机场 `u` 的最小花费估计，松弛边时只会把更优方案继续向外传播。Dijkstra 结束后，取城市 `B` 四个机场中的最小 `dist`，就是答案落点。

### 核心代码

```cpp
Point fourth(Point a, Point b, Point c){
  if(dot(b - a, c - a) == 0) return b + c - a;
  if(dot(a - b, c - b) == 0) return a + c - b;
  return a + b - c;
}
for(int i = 1; i <= S; i++){
  p[i][3] = fourth(p[i][0], p[i][1], p[i][2]);
  for(int u = 0; u < 4; u++) for(int v = 0; v < 4; v++) if(u != v)
    g[id(i, u)].push_back({id(i, v), dist(p[i][u], p[i][v]) * T[i]});
}
for(int i = 1; i <= S; i++) for(int j = i + 1; j <= S; j++)
  for(int u = 0; u < 4; u++) for(int v = 0; v < 4; v++) {
    double w = dist(p[i][u], p[j][v]) * t;
    g[id(i, u)].push_back({id(j, v), w});
    g[id(j, v)].push_back({id(i, u), w});
  }
for(int u = 0; u < 4; u++) dista[id(A, u)] = 0, pq.push({0, id(A, u)});
dijkstra();
for(int u = 0; u < 4; u++) ans = min(ans, dista[id(B, u)]);
```

### 复杂度

若共有 `S` 个城市，则点数为 `4S`。建图复杂度 `O(S^2)`，Dijkstra 复杂度 `O(S^2 \log S)` 量级，空间复杂度 `O(S^2)`。
