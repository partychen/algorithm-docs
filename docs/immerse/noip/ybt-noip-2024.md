# 一本通 2024赛题解题报告

这一年的题目从最直接的统计、模拟与小范围状态出发，很快就转向区间极值、赛制递推和树上结构。读这份专题时，可以一直抓住同一条主线：先把题面里真正约束答案的边界抽出来，再看它最终会落成计数、动态规划、贪心覆盖，还是一类更稳定的树上模型。

# 一、CSP-J 赛题

这一组题表面更贴近直觉，但真正训练的是“把规则压成状态”的第一步：有的题只要把对象去重，有的题需要把过程忠实模拟，有的题则要在位数、轮数和最后一个操作者这些隐含条件上补足状态。

## 1. [【24CSPJ普及组】扑克牌（poker）](http://ybt.ssoier.cn:8088/problem_show.php?pid=2111)

`CSP 2024` `CSP-J`

### 题意

给出小 P 目前拿到的若干张扑克牌。每张牌由“花色 + 点数”唯一确定，完整牌组一共需要 $4 \times 13 = 52$ 种牌各一张。题目要求最少还要再借多少张牌，才能从现有牌和借来的牌里挑出一副完整牌组。

### 分析

这题最关键的翻译只有一句：**完整牌组只在乎某种牌“有没有出现过”，并不在乎它出现了几次。** 如果一张方片 $Q$ 已经出现过，那么后面再来多少张方片 $Q$，都不会减少还缺的牌种数。

因此状态根本不用复杂。把每张牌映射到 $(花色, 点数)$ 这个二元类别，用一个布尔数组记录这一类牌是否已经出现。扫描所有输入后，`have` 中为真的格子个数就是当前已经拥有的不同牌种数。

最后答案就很自然了：完整牌组总共有 $52$ 种牌，所以最少还要借的张数就是 $52 - \text{distinct}$。这类题最值得形成的手感是：**题面里虽然说的是“若干张牌”，真正需要维护的却是“若干种牌”。**

### 核心代码

```cpp
string suit = "DCHS", rank = "A23456789TJQK";
bool vis[4][13] = {};
int kind = 0;
for(string s : cards){
  int x = suit.find(s[0]), y = rank.find(s[1]);
  if(!vis[x][y]){
    vis[x][y] = true;
    kind++;
  }
}
cout << 52 - kind;
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(1)$。

## 2. [【24CSPJ普及组】地图探险（explore）](http://ybt.ssoier.cn:8088/problem_show.php?pid=2112)

`CSP 2024` `CSP-J`

### 题意

给定一张有障碍和空地的地图，以及机器人的起点和朝向。机器人连续执行 $k$ 次操作：若正前方在地图内且是空地，就前进一步；否则原地右转。要求统计执行完后，机器人一共经过了多少个不同位置，起点也算。

### 分析

这题最容易想多，但题面过程其实已经把状态定义得非常完整了：当前状态只有两个量——**位置**和**朝向**。每一步都严格按照“先看前方，再决定前进还是右转”的规则转移，不存在额外决策，也没有隐藏最优性。

因此做法就是直接模拟。当前位置记为 `(x,y)`，朝向记为 `d`，每次先根据 `d` 算出前方格子 `(nx,ny)`。如果它合法且是空地，就把位置改成 `(nx,ny)`；否则只需要原地右转，把朝向改成 $(d+1) \bmod 4$。与此同时，用一个二维布尔数组记录某个位置是否曾经到达过，第一次到达就把答案加一。

这题真正该建立的是“过程题就按过程写”的直觉。虽然状态总数至多只有 $4nm$ 个，但题目给出的 $k$ 本身就可以直接承受，所以没必要再把它硬拐成复杂的找环问题；把模拟写干净，反而最稳。

### 核心代码

```cpp
int dx[4] = {0, 1, 0, -1}, dy[4] = {1, 0, -1, 0};
vis[x][y] = true;
int ans = 1;
while(k--){
  int nx = x + dx[d], ny = y + dy[d];
  if(1 <= nx && nx <= n && 1 <= ny && ny <= m && g[nx][ny] == '.') x = nx, y = ny;
  else d = (d + 1) & 3;
  if(!vis[x][y]) vis[x][y] = true, ans++;
}
cout << ans;
```

### 复杂度

时间复杂度 $O(k)$，空间复杂度 $O(nm)$。

## 3. [【24CSPJ普及组】小木棍（sticks）](http://ybt.ssoier.cn:8088/problem_show.php?pid=2113)

`CSP 2024` `CSP-J`

### 题意

用若干根长度相同的小木棍去拼一个十进制正整数。每个数字消耗的木棍数固定，要求恰好用掉 $n$ 根，不能有前导零，并且拼出的整数尽量小；若无解则输出 $-1$。

### 分析

要让一个正整数尽量小，第一优先级一定是**位数尽量少**。因为任何一个没有前导零的两位数都比任何一位数大，三位数也一定比两位数大。所以先别急着想每一位填什么，而是先确定最少需要多少位。

每个数字最少用 $2$ 根，最多用 $7$ 根。若最后决定用 `len` 位，那么必须满足 $2\,len \le n \le 7\,len$。因此最优位数就是满足这个条件的最小 `len`，也就是从 $\lceil n/7 \rceil$ 开始找的第一个可行值。如果连某个 `len` 都找不到，就说明无解。

位数定下来以后，再从高位到低位贪心填最小数字。当前位枚举最小可能的数字 `d`（首位不能取 $0$），扣掉它消耗的木棍数后，只要剩余木棍还能被后面剩余位数承接，即满足 $2\times \text{restPos} \le \text{remain} \le 7\times \text{restPos}$，这个选择就是安全的。这样一位一位往右填，得到的正是字典序最小、也就是数值最小的方案。

### 核心代码

```cpp
int cost[10] = {6,2,5,5,4,5,6,3,7,6};
int len = 1;
while(2 * len > n || 7 * len < n) len++;
if(2 * len > n || 7 * len < n) return cout << -1, void();
for(int pos = 1; pos <= len; pos++){
  for(int d = (pos == 1); d <= 9; d++){
    int rem = n - cost[d], rest = len - pos;
    if(2 * rest <= rem && rem <= 7 * rest){
      cout << d;
      n = rem;
      break;
    }
  }
}
```

### 复杂度

设答案长度为 $L$，时间复杂度 $O(10L)$，空间复杂度 $O(1)$。

## 4. [【24CSPJ普及组】接龙（chain）](http://ybt.ssoier.cn:8088/problem_show.php?pid=2114)

`CSP 2024` `CSP-J`

### 题意

有 $n$ 个人，每个人有一个整数序列词库。每一轮要从某个人的词库里选一个长度在 $[2,k]$ 的连续子序列：第一轮必须以 $1$ 开头，之后每轮必须以上一轮的末尾元素开头，且不能连续两轮都由同一个人出题。对于很多询问 $(r,c)$，判断能否恰好进行 $r$ 轮并让最后一个元素等于 $c$。

### 分析

这题表面像图论，但真正的难点是“**上一轮是谁出的**”这个限制。若只记“第 $t$ 轮能否以值 $x$ 结尾”，就没法判断下一轮是否还能继续用同一个人；可如果把“上一次的人”完整并进状态，状态又会太大。

一个很巧的压缩方式是：对每个值 `x`，只记录它在当前轮的可达性属于哪三类：不可达、只可能由某一个固定的人结尾、或者已经能由至少两个人结尾。因为下一轮只需要判断“能不能换一个人继续”，所以知道“唯一的人是谁”或者“至少有两个人都行”就够了。于是可以把状态记成 `f[t][x]`，取值为 `-1 / 具体编号 / 0(表示多人可达)`。

转移时按“人”去扫词库。设现在要考虑第 `id` 个人的一整条序列。若某个位置 `p` 上的值 `S[p]` 在上一轮可达，并且上一轮不是唯一由 `id` 这个人得到的，那么它就能作为本轮某个连续子段的开头，于是区间 $(p+1,\min(len,p+k-1))$ 里的所有位置都能作为本轮终点。这个“从一个起点向后覆盖一段”的结构特别适合差分：对每个人每一轮只要扫一遍序列，就能把所有可达终点打出来。轮数上界只有 $100$，所以滚动做完所有轮，再按询问直接查表即可。

### 核心代码

```cpp
auto upd = [&](int &cur, int id){
  if(cur == -1) cur = id;
  else if(cur != id) cur = 0;
};
for(int step = 1; step <= 100; step++){
  fill(g.begin(), g.end(), -1);
  for(int id = 1; id <= n; id++){
    fill(diff.begin(), diff.begin() + len[id] + 2, 0);
    for(int p = 1; p <= len[id]; p++) if(f[step - 1][a[id][p]] != id){
      int L = p + 1, R = min(len[id], p + k - 1);
      if(L <= R) diff[L]++, diff[R + 1]--;
    }
    for(int p = 1, cover = 0; p <= len[id]; p++){
      cover += diff[p];
      if(cover > 0) upd(g[a[id][p]], id);
    }
  }
  f[step].swap(g);
}
```

### 复杂度

设单组数据所有序列长度和为 $S$，最大轮数为 $R$，时间复杂度 $O(RS)$，空间复杂度 $O(S+V)$，其中 $V$ 是值域大小。

# 二、CSP-S 赛题

提高组开始之后，题目就不再只是把过程写出来，而是要先识别真正决定答案的结构：有的题把生存关系压成偏序匹配，有的题把物理过程压成区间，有的题则要把赛制、不确定性和树上区间极值重新编码成更稳定的模型。

## 5. [【24CSPS提高组】决斗（duel）](http://ybt.ssoier.cn:8088/problem_show.php?pid=2115)

`CSP 2024` `CSP-S`

### 题意

有 $n$ 只怪兽，第 $i$ 只怪兽的攻击力和防御力都等于 $r_i$。每只怪兽整场游戏至多主动攻击一次；若它攻击的对象防御严格更小，对方退出游戏，否则无事发生。要求安排攻击顺序，使游戏结束时存活怪兽数尽量少。

### 分析

先看一条本质限制：**一次成功攻击，本质上是在一只更强怪兽和一只更弱怪兽之间做一次匹配。** 同值怪兽之间永远互相杀不掉，因为条件要求严格大于；而每只怪兽又只能主动攻击一次，所以你不可能让一只更强怪兽反复清空一大群同值怪兽。

把所有怪兽按能力值分组之后，问题就变成：能否把尽可能多的怪兽放进“严格递增链”里，让链上的前者去杀后者。依据偏序上的最小链覆盖结论，最后不得不保留下来的数量，正好等于这个偏序里的最大反链大小；而在一维严格大小关系下，最大反链就是某个相同能力值出现次数的最大值。

直观上也很好理解：出现次数最多的那一组值彼此之间无法互杀，而比它们强的怪兽数量再多，每只也只够匹配掉一个更弱者，不可能把这组值全部消掉。反过来，只要把其他值尽量按从大到小去贪心匹配，确实能把除这组“众数”以外的怪兽都消到最优。因此答案就是所有能力值出现次数的最大值。

### 核心代码

```cpp
unordered_map<int,int> cnt;
int ans = 0;
for(int x : r){
  ans = max(ans, ++cnt[x]);
}
cout << ans;
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 6. [【24CSPS提高组】超速检测（detect）](http://ybt.ssoier.cn:8088/problem_show.php?pid=2116)

`CSP 2024` `CSP-S`

### 题意

一条长度为 $L$ 的道路上有若干辆车，每辆车从位置 $d_i$ 出发，以初速度 $v_i$、加速度 $a_i$ 向北运动，直到驶离道路或者速度降到 $0$。道路上有 $m$ 个按位置排序的测速仪。需要求出：当所有测速仪都开启时，有多少辆车会被判超速；以及在不漏掉任何一辆会被抓到的超速车的前提下，最多能关闭多少个测速仪。

### 分析

题目虽然披着物理外衣，但算法核心其实是一个**区间问题**。对某一辆车来说，真正重要的不是它每一时刻的速度，而是：在道路上的哪些位置上，它的瞬时速度满足 $v(x) > V$。利用匀加速公式 $v(x)^2 = v_0^2 + 2a(x-d)$，再结合它真实会行驶到的最远位置，就能把每辆车的超速部分化成道路上的一个区间，可能为空，也可能是一整段前缀或后缀。

有了道路位置上的超速区间以后，第一问就很直接：只要这个区间里至少落着一个测速仪，这辆车就一定会被抓。因为测速仪位置本身已经排好序，所以把超速区间左右端点映射到测速仪下标区间 $[l,r]$，只要 $l \le r$，这辆车就计入答案。

第二问则是经典的“区间最少点覆盖”。每辆会被抓的车都对应一个测速仪下标区间，要求保留尽量少的测速仪，使每个区间至少被击中一次。按右端点从小到大排序，若当前区间还没被上一个已选测速仪覆盖，就把它的右端点对应测速仪保留下来，这是标准且最优的贪心。设最少需要保留 `keep` 个测速仪，那么最多能关闭的数量就是 $m-\text{keep}$。

### 核心代码

```cpp
vector<pair<int,int>> seg;
for(auto car : cars){
  auto [L, R] = overspeed_interval(car);
  int l = lower_bound(p + 1, p + m + 1, L) - p;
  int r = upper_bound(p + 1, p + m + 1, R) - p - 1;
  if(l <= r) seg.push_back({l, r});
}
sort(seg.begin(), seg.end(), [](auto a, auto b){ return a.second < b.second; });
int keep = 0, last = 0;
for(auto [l, r] : seg) if(last < l) keep++, last = r;
cout << seg.size() << ' ' << m - keep;
```

### 复杂度

时间复杂度 $O((n+m)\log m)$，空间复杂度 $O(n)$。

## 7. [【24CSPS提高组】染色（color）](http://ybt.ssoier.cn:8088/problem_show.php?pid=2117)

`CSP 2024` `CSP-S`

### 题意

给定正整数数组 $A$。你要把每个位置染成红色或蓝色。对每个位置 $i$，若它左侧最近的同色位置的值恰好也等于 $A_i$，就产生贡献 $A_i$；否则该位置贡献为 $0$。要求最大化总得分。

### 分析

这题真正难的地方，是“最近的同色位置”这个条件让贡献不是简单地按值独立。先看最容易的一类：如果 $A_i = A_{i-1}$，那么无论怎么染，只要把这两个位置染成同色，就一定能让第 $i$ 个位置拿到 $A_i$ 这一分。这类贡献是确定会赚到的，可以先用前缀和 $pre[i]$ 累计下来。

除去这些“相邻相等”的必得项以后，剩下的额外收益只能来自某个值在更远处的再次配对。设 $j = last[A_i]$ 是 $A_i$ 上一次出现的位置。如果我们想让位置 $i$ 靠位置 $j$ 拿到贡献，那么在区间 $(j,i)$ 里，就不能再出现一个更近的同色位置把它挡住。换句话说，这一段的颜色结构基本被“卡死”了，能继续保留下来的，只有我们前面已经单独统计过的那些相邻相等必得项。

于是可以做线性 DP。设 $dp[i]$ 表示处理到前 $i$ 个位置时，除去 $pre$ 这部分以后，最多还能额外拿多少分。若不让位置 $i$ 作为某次远距离配对的右端点，则 $dp[i]=dp[i-1]$；若让它和 $j=last[A_i]$ 配对，则新增收益是 $A_i + (pre[i-1]-pre[j+1])$，前缀部分接 $dp[j+1]$。这就得到转移：

$$
dp[i]=\max\bigl(dp[i-1],\ dp[j+1]+A_i+pre[i-1]-pre[j+1]\bigr).
$$

最后答案就是 $pre[n] + dp[n]$。

### 核心代码

```cpp
for(int i = 1; i <= n; i++){
  pre[i] = pre[i - 1] + (i > 1 && a[i] == a[i - 1] ? a[i] : 0);
  dp[i] = dp[i - 1];
  int j = last[a[i]];
  if(j) dp[i] = max(dp[i], dp[j + 1] + a[i] + pre[i - 1] - pre[j + 1]);
  last[a[i]] = i;
}
cout << pre[n] + dp[n];
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 8. [【24CSPS提高组】擂台游戏（arena）](http://ybt.ssoier.cn:8088/problem_show.php?pid=2118)

`CSP 2024` `CSP-S`

### 题意

给定按报名顺序出现的选手能力值，以及每一轮每一场比赛的擂主方向。对任意前缀 $c_i$，需要把人数补到不小于 $c_i$ 的最小 $2$ 的幂，多出来的选手能力值可以任意指定。问在这场完整比赛中，所有可能成为总冠军的选手编号之和是多少。

### 分析

这题看起来像在问很多前缀，但真正的突破口是：**比赛结果只关心某名选手的能力值是否达到当前轮数。** 设整棵赛树高度为 $K$，因为 $K \le 17$，所以每个选手的能力值实际上只需要截断到 $[0,K]$ 这几个层级，超过 $K$ 再大也不会带来新信息。

对于赛树上的一个节点 `u`，我们维护两个量。`win[u]` 表示当这个子树里的叶子能力都已经确定时，最后的确定赢家是谁；`lim[u]` 表示当前缀逐步增大时，这棵子树“最后一次仍然自由”的位置，也就是在只固定前若干个报名选手时，这棵子树还可以通过给未出现选手任意赋值来改变冠军。自由性会随着前缀增大单调消失，所以 `lim` 可以自底向上递推。

有了 $win$ 和 $lim$ 以后，再看一名具体选手 `x`。它能成为冠军，当且仅当从叶子走到根的整条路径上，没有哪一层已经被另一侧的确定赢家彻底堵死；而所有“不确定还没消失”的层，只会给出一个“前缀长度不能超过某值”的限制。把这些限制取交，就得到一个前缀区间 $[L_x,R_x]$：在这段前缀范围内，选手 `x` 恰好可能成为冠军。于是对每个选手在这个区间上做一次差分，前缀和之后就能得到每个 $c_i$ 的答案。多组数据只是异或修改能力值，所以每次重算一遍整棵赛树即可。

### 核心代码

```cpp
for(int i = base; i < base + (1 << K); i++)
  win[i] = leaf_value(i), lim[i] = birth_time(i);
for(int i = base - 1; i; i--){
  int need = K - __lg(i);
  if(dir[i] == 0){
    lim[i] = (win[i << 1] < need ? lim[i << 1 | 1] : lim[i << 1]);
    win[i] = (win[i << 1] < need ? win[i << 1 | 1] : win[i << 1]);
  }else{
    lim[i] = lim[i << 1 | 1];
    win[i] = (win[i << 1 | 1] >= need ? win[i << 1 | 1] : win[i << 1]);
  }
}
for(each leaf x){
  auto [L, R] = champion_interval(x);
  if(L <= R) diff[L] += id[x], diff[R + 1] -= id[x];
}
```

### 复杂度

设整棵补齐后的赛树叶子数为 $2^K$，单组数据时间复杂度 $O(2^K)$，空间复杂度 $O(2^K)$。

# 三、NOIP 赛题

这一章开始是 2024 年的 NOIP 提高组赛题。和前面的 CSP 相比，这几题更强调把抽象出来的模型继续往前推：有的题要把奇偶结构拆成可贪心的两部分，有的题要围绕“第一次失误”反向计数，有的题要抓住树上唯一传递链或把任意长度范围拆成少量可预处理的块。

## 9. [【24NOIP提高组】编辑字符串（edit）](http://ybt.ssoier.cn:8088/problem_show.php?pid=2119)

`NOIP 2024` `NOIP 提高组`

### 题意

给定两个 $01$ 串 $s_1,s_2$。每个位置还有一个标记，表示该字符能否参与相邻交换。允许在两个串内部各自进行任意多次合法交换，要求最大化最终两个串对应位置相同的个数。

### 分析

这题最重要的观察是：**一个连续的“可交换”区间内部，字符可以任意重排；而不能交换的位置，会把字符串切成若干独立的段。** 所以我们没必要真的模拟交换过程，只要知道每个位置属于哪一段，以及这段里还剩多少个 `0`、多少个 `1` 没被用掉即可。

接下来从左到右贪心处理每个位置 `i`。设 $s_1[i]$ 所在段为 `A`，$s_2[i]$ 所在段为 `B`。如果 `A` 和 `B` 里都还剩未使用的 `0`，那就在这个位置匹配一个 `0`；否则若两边都还剩 `1`，就匹配一个 `1`；若两种都做不到，说明这个位置无论如何都不能相等，只能各自消耗掉一枚字符。这样做的理由是：当前位置是两个当前段最靠左的未决位置，如果此时已经能匹配出相等字符，拖到后面并不会创造更多机会，反而可能浪费一次本可以立刻拿到的贡献。

因此整题只要先线性预处理“每个位置属于哪个可交换段”，再统计每段还剩多少个 `0/1`，最后按位置扫一遍做贪心消耗即可。不可交换的位置也自然被视为长度为 $1$ 的单独一段，不需要额外分类讨论。

### 核心代码

```cpp
for(int i = 0; i < n; i++){
  if(cnt0a[segA[i]] && cnt0b[segB[i]]) ans++, cnt0a[segA[i]]--, cnt0b[segB[i]]--;
  else if(cnt1a[segA[i]] && cnt1b[segB[i]]) ans++, cnt1a[segA[i]]--, cnt1b[segB[i]]--;
  else if(cnt0a[segA[i]]) cnt0a[segA[i]]--, cnt1b[segB[i]]--;
  else cnt1a[segA[i]]--, cnt0b[segB[i]]--;
}
cout << ans;
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 10. [【24NOIP提高组】遗失的赋值(assign)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2120)

`NOIP 2024` `NOIP 提高组`

### 题意

有 $n$ 个变量，每个变量取值在 $[1,v]$。相邻变量之间有形如“若 $x_i=a_i$，则 $x_{i+1}=b_i$”的二元限制，同时还有若干一元限制 $x_{c_j}=d_j$。现在只记得所有一元限制，忘掉了所有 $a_i,b_i$。要求统计有多少种二元限制的取值方案，能保证整组限制至少存在一个合法赋值。

### 分析

先看最直接的矛盾：如果同一个位置被一元限制成两个不同的值，那么无论怎么选二元限制，都不可能存在合法赋值，答案立刻是 $0$。除此之外，一元限制之间其实把整条链切成了若干独立区间，答案就是各段方案数的乘积。

考虑两条相邻的一元限制，位置分别是 $l,r$。这中间一共有 $r-l$ 条二元限制，每条限制有两个参数，所以总方案数是 $v^{2(r-l)}$。什么情况下会不合法？只有一种：从左端给定值一路“强制传递”到右端，形成一条完全串起来的链，但最后传到右端时偏偏不等于右端给定值。左端起点被固定，中间每一跳的衔接值可以任意选，共有 $v^{r-l-1}(v-1)$ 种坏方案。

于是这段的合法方案数就是

$$
v^{2(r-l)}-v^{r-l-1}(v-1).
$$

左右两端没有被一元限制卡住的部分更简单，因为不会和固定值发生冲突，直接任意选二元限制即可：若前缀或后缀含有 $len$ 条边，贡献就是 $v^{2\,len}$。把所有段的贡献乘起来，就是最终答案。

### 核心代码

```cpp
sort(lim.begin(), lim.end());
for(int i = 1; i < m; i++)
  if(lim[i - 1].first == lim[i].first && lim[i - 1].second != lim[i].second) return cout << 0, void();
long long ans = qpow(v, 2LL * (firstPos - 1)) * qpow(v, 2LL * (n - lastPos)) % MOD;
for(int i = 1; i < tot; i++){
  int gap = pos[i] - pos[i - 1];
  long long all = qpow(v, 2LL * gap);
  long long bad = qpow(v, gap - 1) * (v - 1) % MOD;
  ans = ans * ((all - bad + MOD) % MOD) % MOD;
}
cout << ans;
```

### 复杂度

设去重后的一元限制数量为 $m$，时间复杂度 $O(m\log m)$，空间复杂度 $O(m)$。

## 11. [【24NOIP提高组】树的遍历(traverse)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2121)

`NOIP 2024` `NOIP 提高组`

### 题意

把原树的边当成新图里的点，两条原边若有公共端点，就视作新图中相邻。现在从若干条关键边中的任意一条出发，按 DFS 的边遍历规则走完整个线图，并把访问顺序生成一棵新的树。题目要求统计：从任意关键边出发，最终可能生成多少种不同的新树。

### 分析

先看只有一条起始边的情况。原树中某个点的所有邻边，在线图里会形成一个团；而一个团的 DFS 树一定是一条链。于是当进入这个原点的那条边已经确定后，其余邻边只剩“按什么顺序串成链”这一种自由度，所以这个点贡献 $(deg_u-1)!$。把所有点乘起来，就得到单起点时的总方案数。

难点在于有多条关键边时会重复计数。什么时候一棵新树会同时被两条甚至多条关键边作为起点生成？答案是：这些关键边在原树里必须落在同一条链上。因为只要某个点处有三条不同方向的关键边分叉，DFS 生成的新树就不可能同时满足它们都作为“起始链”的要求，贡献直接变成 $0$。所以真正有贡献的关键边集合，只可能是一条链。

这就把题目带到“链上的容斥”。把基础因子 $\prod (deg_u-1)!$ 提出来之后，链上每经过一个内部点，贡献就要额外乘一个 $(deg_u-1)^{-1}$，也就是把原本的 $(deg_u-1)!$ 降成 $(deg_u-2)!$。设 $c_u=(deg_u-1)^{-1}$，再做一次树形 DP：`f[u]` 表示关键边链经过 `u`，且恰有一个端点在 `u` 子树内时的带权方案和。若边 `(u,fa)` 本身是关键边，就把它当作一条新的链端点；否则只能把子树里的链通过 `u` 往上延。合并不同儿子时顺手做容斥扣重，最后再乘回基础因子即可。

### 核心代码

```cpp
void dfs(int u, int fa, int upKey){
  long long s = upKey ? MOD - 1 : 0;
  f[u] = s;
  if(upKey) ans = (ans + 1) % MOD;
  long long c = inv[deg[u] - 1];
  for(auto [v, id] : g[u]) if(v != fa){
    dfs(v, u, key[id]);
    ans = (ans - c * s % MOD * f[v] % MOD + MOD) % MOD;
    s = (s + f[v]) % MOD;
    if(!upKey) f[u] = (f[u] + c * f[v]) % MOD;
  }
}
for(int u = 1; u <= n; u++) ans = ans * fac[deg[u] - 1] % MOD;
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 12. [【24NOIP提高组】树上查询（query）](http://ybt.ssoier.cn:8088/problem_show.php?pid=2122)

`NOIP 2024` `NOIP 提高组`

### 题意

给定一棵根为 $1$ 的树。记 $LCA^*(l,r)$ 为编号在 $[l,r]$ 内所有结点的最近公共祖先。每次询问给出 $(l,r,k)$，要求在所有长度至少为 $k$ 的连续子区间里，求 $dep_{LCA^*}$ 的最大值。

### 分析

这题第一步一定要把“区间里很多个点的 LCA”压成一维数组。令

$$
c_i = dep_{LCA(i,i+1)}.
$$

那么任意一个结点区间 $[L,R]$ 的公共祖先深度，恰好等于 $\min(c_L,c_{L+1},\dots,c_{R-1})$。原因是：区间公共祖先一定会在某一对相邻编号跨出不同子树的地方被“卡住”，而那一对相邻点的 LCA 深度正好就是整个区间的 LCA 深度。

于是原问题化成了：在数组 $c$ 上，给定区间 $[l,r-1]$，求所有长度至少为 $k-1$ 的子段中，“子段最小值”的最大值。这里还可以再走一步：长度更长的子段只会让最小值更小，所以最优答案一定出现在**长度恰好等于 $k-1$** 的子段里。这样问题就变成了经典的“定长窗口最大最小值”。

接下来用单调栈求每个位置 $i$ 作为子段最小值时，能向左右扩展到的极大区间 $[L_i,R_i]$。为了避免相等值重复计数，左右边界采用左侧用 $\le$、右侧用 $<$ 的规范拆分。这样 $c_i$ 会对一批合法窗口产生贡献，这批窗口在平面上对应几个规则矩形。把所有贡献矩形离线下来，再按查询长度做三遍扫描线 / 线段树维护，就能在 $O((n+q)\log n)$ 内回答全部询问；而 $k=1$ 时答案直接是区间深度最大值，用 RMQ 单独处理即可。

### 核心代码

```cpp
for(int i = 1; i < n; i++) c[i] = dep[lca(i, i + 1)];
for(int i = 1; i < n; i++){
  L[i] = prev_leq(i) + 1;
  R[i] = next_less(i) - 1;
  build_rectangles(L[i], R[i], i, c[i]);
}
offline_sweep_left(rects, queries, ans);
offline_sweep_right(rects, queries, ans);
offline_sweep_inside(rects, queries, ans);
if(k == 1) ans[id] = max(ans[id], rmq_depth(l, r));
```

### 复杂度

时间复杂度 $O((n+q)\log n)$，空间复杂度 $O(n+q)$。
