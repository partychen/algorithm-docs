# 一本通 NOIP 2002赛题解题报告

这一年的题目从最直接的枚举、递推一路走到分割与最优划分，前半段重在把题面条件翻成可计算的状态，后半段则不断追问“局部调整为什么会导向全局最优”。阅读时可以先抓住每题最核心的不变量，再看状态如何推进、答案如何落地。

# 一、普及组赛题

普及组四题都不追求复杂数据结构，但很考验把条件拆开后的建模力：有的题靠单调性顺推，有的题靠搜索枚举，有的题要把“任意次变换”压成可达关系，还有的题直接把棋盘路径转成经典递推。

## 1. [【02NOIP普及组】级数求和](http://ybt.ssoier.cn:8088/problem_show.php?pid=1918)
`NOIP 2002` `普及组`

### 题意

给定一个整数 `K`，要求找到最小的 `n`，使得调和级数前缀和 `1+1/2+...+1/n` 严格大于 `K`。

### 分析

这题最重要的观察是：前缀和随着 `n` 增大严格递增，所以一旦某个位置第一次超过 `K`，它就一定是答案。题目只要求最小下标，并没有让我们做高精度估计，因此直接顺着定义累加就是最自然的做法。

状态非常简单，只需要维护当前枚举到的项数 `n` 和当前前缀和 `sum`。每次把 `1/n` 加进去，检查是否已经超过 `K`。由于 `K≤15`，需要的项数并不大，用 `double` 做累加就足够稳定。

答案的提取也很直接：循环第一次满足 `sum>K` 时，对应的 `n` 就是题目要的最小值，因为前面所有位置都还没有越过这个阈值。

### 核心代码

```cpp
double sum = 0;
int n = 0;
while(sum <= k){
  ++n;
  sum += 1.0 / n;
}
cout << n;
```

### 复杂度

时间复杂度 `O(n)`，空间复杂度 `O(1)`。

## 2. [【02NOIP普及组】选数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1919)
`NOIP 2002` `普及组`

### 题意

给出 `n` 个整数，要从中恰好选出 `k` 个，统计所有选法里“元素和为素数”的方案数。

### 分析

题目真正的难点不是判素数，而是“从 `n` 个数里恰好选 `k` 个”这一层组合枚举。由于 `n≤20`，可以把每个数看成“选或不选”的决策点，用深度优先搜索沿着下标往后做选择。只要把“已经选了多少个数”和“当前和是多少”带着往下递归，就能完整覆盖所有合法组合。

搜索状态可以记成 `dfs(pos, cnt, sum)`：表示当前考虑到第 `pos` 个数，已经选了 `cnt` 个，已有和为 `sum`。转移分成两种：跳过当前数，或者选上当前数。为了避免无效分支，还可以加一个常见剪枝：如果“剩余元素数量 + 已选数量”仍然凑不够 `k`，这条路就不用再走。

当 `cnt==k` 时，说明已经形成一个完整组合，此时只需检查 `sum` 是否为素数，若是就把答案加一。这样统计到的每条成功路径都恰好对应一种选法，答案自然正确。

### 核心代码

```cpp
bool isPrime(int x){
  if(x < 2) return false;
  for(int i = 2; i * i <= x; i++) if(x % i == 0) return false;
  return true;
}
void dfs(int pos, int cnt, int sum){
  if(cnt == k){ ans += isPrime(sum); return; }
  if(pos == n || cnt + (n - pos) < k) return;
  dfs(pos + 1, cnt, sum);
  dfs(pos + 1, cnt + 1, sum + a[pos]);
}
```

### 复杂度

时间复杂度最坏为 `O(2^n\sqrt{S})`，其中 `S` 是组合和的大小量级；空间复杂度 `O(n)`。

## 3. [【02NOIP普及组】产生数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1920)
`NOIP 2002` `普及组`

### 题意

给出一个整数 `n` 和若干位数字变换规则。一位数字可以经过任意次变换变成别的数字，原数各位也都允许独立变换。要求统计最终能产生多少个不同整数，原数本身也算。

### 分析

“任意次变换”这四个字，意味着我们不能只看题面给出的直接规则，而要看一位数字经过多次跳转后最终能到哪些数字。因为数字只有 `0~9` 十个点，所以先把规则看成一张有向图，再求每个数字的传递闭包：`reach[i][j]=1` 表示数字 `i` 经过若干次变换后能变成 `j`。

闭包求出来后，整题会一下子变得很清楚。原数的每一位互不影响：某一位如果原来是 `d`，它最终可以取的值个数就是 `reach[d][*]` 中可达点的数量；不同位置的选择彼此独立，所以总方案数就是每一位可选数量的乘积。这一步的核心不变量是“某一位怎么变，只由这一位原本的数字决定，不会影响其他位”。

因此答案提取分两步：先用 Floyd 或 DFS 求出十个数字之间的可达关系，再顺着原数每一位做乘法累积。由于结果可能很大，用高精度整数承接乘积即可。

### 核心代码

```cpp
for(int i = 0; i < 10; i++) reach[i][i] = 1;
for(auto [u, v] : rule) reach[u][v] = 1;
for(int k = 0; k < 10; k++)
  for(int i = 0; i < 10; i++)
    for(int j = 0; j < 10; j++)
      reach[i][j] |= reach[i][k] && reach[k][j];
cpp_int ans = 1;
for(char ch : s){
  int d = ch - '0', cnt = 0;
  for(int j = 0; j < 10; j++) cnt += reach[d][j];
  ans *= cnt;
}
cout << ans;
```

### 复杂度

时间复杂度 `O(10^3+10|n|)`，空间复杂度 `O(10^2)`。

## 4. [【02NOIP普及组】过河卒](http://ybt.ssoier.cn:8088/problem_show.php?pid=1921)
`NOIP 2002` `普及组`

### 题意

棋盘上一个卒从 `(0,0)` 走到 `(n,m)`，每步只能向右或向下。同时给出一匹马的位置，马所在点和它一步能控制到的八个点都不能经过。要求计算合法路径条数。

### 分析

只允许向右、向下时，从左上走到某个格子 `(i,j)` 的最后一步只可能来自 `(i-1,j)` 或 `(i,j-1)`。这正是一个非常标准的网格递推模型。题目额外加入的马控制点，本质上只是把某些格子改成“不可达”，在转移时把这些位置当成障碍即可。

因此状态设为 `dp[i][j]`：表示走到坐标 `(i,j)` 的方案数。若当前位置被马控制，那么 `dp[i][j]=0`；否则它就等于上方和左方方案数之和。整个递推过程中保持的不变量是：`dp[i][j]` 只统计不经过任何禁点的路径，所以一旦某格被封锁，它不会向后贡献任何方案。

起点 `(0,0)` 若不受控制，就令 `dp[0][0]=1` 作为递推源头。最终答案直接读取 `dp[n][m]`，因为所有合法路径都会且只会在这里汇总。

### 核心代码

```cpp
int dx[9] = {0, 1, 2, 2, 1, -1, -2, -2, -1};
int dy[9] = {0, 2, 1, -1, -2, -2, -1, 1, 2};
for(int t = 0; t < 9; t++){
  int nx = hx + dx[t], ny = hy + dy[t];
  if(nx >= 0 && nx <= n && ny >= 0 && ny <= m) ban[nx][ny] = 1;
}
if(!ban[0][0]) dp[0][0] = 1;
for(int i = 0; i <= n; i++)
  for(int j = 0; j <= m; j++) if(!ban[i][j]){
    if(i) dp[i][j] += dp[i - 1][j];
    if(j) dp[i][j] += dp[i][j - 1];
  }
cout << dp[n][m];
```

### 复杂度

时间复杂度 `O(nm)`，空间复杂度 `O(nm)`。

# 二、提高组赛题

提高组开始，题目不再满足于“把状态列出来就结束”，而是要继续判断局部操作与整体最优的关系：有的题把一次移动解释成前缀失衡的修正，有的题把几何过程转成时间区间，有的题则要利用矩形互不相交这一结构做递归划分。

## 5. [【02NOIP提高组】均分纸牌](http://ybt.ssoier.cn:8088/problem_show.php?pid=1828)
`NOIP 2002` `提高组`

### 题意

有 `N` 堆纸牌，只能把某堆纸牌移到相邻的一堆。总牌数一定是 `N` 的倍数，要求用最少的移动次数让每堆都变成同样多。

### 分析

先设目标张数为平均值 `avg=sum/N`。如果我们从左到右看，位置 `i` 最终是否平衡，其实只取决于前 `i` 堆总共多了多少或少了多少。因为牌只能在相邻堆之间移动，所以当前堆和目标值的差额，最终一定要通过边 `(i,i+1)` 传给右边。

这就带出一个关键不变量：当我们处理完第 `i` 堆后，前 `i` 堆必须已经全部变成 `avg`，它们和后面的联系只剩下一次“把盈余或亏空传给右边”。于是只要 `a[i] != avg`，就必须发生一次跨边移动，把 `a[i]-avg` 这一差额并到 `a[i+1]` 上，同时答案加一；若 `a[i]==avg`，这一位已经平衡，不需要额外动作。

从左扫到右时，每条边最多被作为“修正当前前缀失衡”使用一次，因此这样做既能完成目标，也不会多算操作。最后统计到的次数就是最少移动次数。

### 核心代码

```cpp
int avg = sum / n;
for(int i = 1; i < n; i++){
  if(a[i] == avg) continue;
  a[i + 1] += a[i] - avg;
  ans++;
}
cout << ans;
```

### 复杂度

时间复杂度 `O(n)`，空间复杂度 `O(1)`。

## 6. [【02NOIP提高组】自由落体](http://ybt.ssoier.cn:8088/problem_show.php?pid=1829)
`NOIP 2002` `提高组`

### 题意

高为 `H` 的天花板上有 `n` 个小球，水平位置分别是 `0,1,2,...`。地面上一辆长度为 `L`、高度为 `K` 的小车从距原点 `S1` 的位置出发，以速度 `V` 前进。小球和小车同时开始运动，问小车最终能接住多少个小球。

### 分析

把每个小球单独看，它的水平坐标始终不变，变化的只有竖直高度 `H-5t^2`。要被小车接住，小球必须在还没落地前进入小车所占据的矩形区域。于是关键不是模拟每个时刻，而是先求出“小球处在小车高度范围内”的时间段。

若 $H>K$，小球从高度 $H$ 落到车顶高度 $K$ 的时刻是 $t_1=\sqrt{(H-K)/5}$；落到地面的时刻是 $t_2=\sqrt{H/5}$。在这段时间里，小车的水平覆盖区间从 $[S_1+Vt_1,S_1+L+Vt_1]$ 扩展到 $[S_1+Vt_2,S_1+L+Vt_2]$，所以能被接住的小球横坐标，恰好是所有这些区间的并，也就是连续区间 $[S_1+Vt_1,S_1+L+Vt_2]$。若 $H\le K$，说明一开始小球就在车高范围内，此时把 $t_1$ 视为 `0` 即可。

因此答案就变成“整数点 `0..n-1` 中有多少个落在这个连续区间里”。这一步只需处理好浮点误差，分别取左端上取整、右端下取整，再和球的编号范围求交即可。

### 核心代码

```cpp
double t1 = H <= K ? 0.0 : sqrt((H - K) / 5.0);
double t2 = sqrt(H / 5.0);
double left = S1 + V * t1, right = S1 + L + V * t2;
int l = max(0, (int)ceil(left - 1e-9));
int r = min(n - 1, (int)floor(right + 1e-9));
cout << max(0, r - l + 1);
```

### 复杂度

时间复杂度 `O(1)`，空间复杂度 `O(1)`。

## 7. [【02NOIP提高组】矩形覆盖](http://ybt.ssoier.cn:8088/problem_show.php?pid=1830)
`NOIP 2002` `提高组`

### 题意

平面上有 `n` 个点，要用 `k` 个边平行于坐标轴且彼此完全分开的矩形覆盖所有点，要求矩形面积和最小。单点或同一直线上的点允许用面积为 `0` 的矩形覆盖。

### 分析

题目的结构关键词是“矩形彼此完全分开”。两个互不相交的轴对齐矩形之间，一定存在一条竖直线或水平线把它们彻底隔开。于是最优方案虽然看起来像在平面上同时摆 `k` 个矩形，本质上却可以递归地理解成：先把点集用一条横线或竖线切成两部分，再分别覆盖左右两边或上下两边。

这样就可以定义递归状态 `solve(p, t)`：用 `t` 个矩形覆盖点集 `p` 的最小面积。若 `t==1`，答案就是这一组点的最小包围矩形面积；若 `t>1`，就把当前点集按 `x` 排序尝试所有合法前缀/后缀切分，再按 `y` 排序尝试所有上下切分。每次切分都对应“一条分界线把两个子问题彻底分开”，这正好符合题目对矩形互不接触的要求。

转移时取 `solve(left,1)+solve(right,t-1)`、`solve(left,2)+solve(right,t-2)` 等所有分配方式的最小值即可。因为 `k<4`，矩形个数极小，递归枚举的层数很浅；而答案提取就是所有切分方案中的最小面积和。

### 核心代码

```cpp
int area(vector<Point> p){
  int minx = INF, maxx = -INF, miny = INF, maxy = -INF;
  for(auto &pt : p) minx = min(minx, pt.x), maxx = max(maxx, pt.x), miny = min(miny, pt.y), maxy = max(maxy, pt.y);
  return (maxx - minx) * (maxy - miny);
}
int solve(vector<Point> p, int t){
  if(p.empty()) return 0;
  if(t == 1) return area(p);
  int ans = area(p);
  sort(p.begin(), p.end(), cmpX);
  for(int i = 1; i < (int)p.size(); i++) if(p[i - 1].x != p[i].x){
    vector<Point> l(p.begin(), p.begin() + i), r(p.begin() + i, p.end());
    for(int s = 1; s < t; s++) ans = min(ans, solve(l, s) + solve(r, t - s));
  }
  sort(p.begin(), p.end(), cmpY);
  for(int i = 1; i < (int)p.size(); i++) if(p[i - 1].y != p[i].y){
    vector<Point> l(p.begin(), p.begin() + i), r(p.begin() + i, p.end());
    for(int s = 1; s < t; s++) ans = min(ans, solve(l, s) + solve(r, t - s));
  }
  return ans;
}
```

### 复杂度

时间复杂度取决于递归切分的枚举，`k<4` 时可视为 `O(n^3)` 量级，空间复杂度为递归栈 `O(kn)`。
