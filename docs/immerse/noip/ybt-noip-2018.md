# 一本通 NOIP 2018赛题解题报告

这一年的题目从普及组的直接统计、线段势力与时间安排，一路走到提高组的背包判冗、树上贪心、基环树搜索、数学计数与动态树形规划。读的时候可以先抓住题面条件如何压成状态，再看转移为什么不丢最优，最后看答案究竟落在哪个状态上。

# 一、普及组赛题

这一组题的难点都不在模板，而在第一步建模：有的题把文字条件直接压成计数，有的题把气势差改写成带符号的量，有的题要顺着时间轴做调度，还有的题要把“镜像相同”变成递归可判的结构。

## 1. [【18NOIP普及组】标题统计](http://ybt.ssoier.cn:8088/problem_show.php?pid=1978)

`普及组` `字符串`

### 题意

给出一行标题字符串，要求统计其中真正算作标题内容的字符个数。空格和行末换行都不计入答案，其他大写字母、小写字母、数字字符都要计数。

### 分析

这题的关键信息只有一句：**空格和换行符不算，其余字符都算。** 一旦把条件翻译成“逐字符判定是否有效”，题目就从文字描述变成了最直接的线性扫描。

维护一个计数器 `ans` 作为状态，顺着字符串从左到右看。当前字符如果不是空格、也不是换行，就把 `ans` 加一；否则直接跳过。这个状态始终表示“已经扫描过的前缀里，有多少个有效字符”，不变量非常清楚。

整串扫完以后，`ans` 已经把所有该算的字符都累计进来了，最后直接输出它，就是题目要求的字符数。

### 核心代码

```cpp
int ans = 0;
for(char ch : s){
  if(ch != ' ' && ch != '\n' && ch != '\r') ans++;
}
cout << ans;
```

### 复杂度

时间复杂度 $O(|s|)$，空间复杂度 $O(1)$。

## 2. [【18NOIP普及组】龙虎斗](http://ybt.ssoier.cn:8088/problem_show.php?pid=1979)

`普及组` `模拟` `枚举`

### 题意

棋盘是一条以 `m` 号兵营为分界的线段，左边兵营贡献龙方气势，右边兵营贡献虎方气势，贡献值等于“兵营人数 × 到分界点的距离”。题目先额外在 `p1` 放入 `s1` 个工兵，再让你把手里的 `s2` 个工兵全部放到某个兵营 `p2`，使龙虎双方气势差的绝对值尽量小；若最优位置不唯一，取编号最小的。

### 分析

题面看起来像两边来回算和，但真正该抓住的是：**无论怎么放兵，变化都只体现在“虎方总气势减去龙方总气势”的一个带符号差值上。** 设 `diff = 虎势 - 龙势`，那么原有兵营和天降神兵 `s1` 都可以一次性算进 `diff`。之后我们只需要研究把 `s2` 放到不同位置时，`diff` 会怎么改。

如果把 `s2` 放在 `m` 左边，它会让龙方气势增加 `s2 * (m-p2)`，所以 `diff` 要减去这部分；如果放在右边，就让虎方气势增加 `s2 * (p2-m)`，所以 `diff` 加上这部分；放在 `m` 本身则完全不影响。于是每个候选位置都能在 $O(1)$ 时间得到新的差值 `cur`。

状态就变成了“当前枚举位置得到的最小 `|cur|` 和对应最小编号”。因为题目只让我们追加一次神兵，没有连锁决策，所以把所有 `p2` 线性扫一遍就是完整搜索。扫描结束后，第一次达到最小绝对差且编号最小的位置，就是答案提取的位置。

### 核心代码

```cpp
long long diff = 0;
for(int i = 1; i <= n; i++){
  if(i < m) diff -= 1LL * c[i] * (m - i);
  else if(i > m) diff += 1LL * c[i] * (i - m);
}
if(p1 < m) diff -= 1LL * s1 * (m - p1);
else if(p1 > m) diff += 1LL * s1 * (p1 - m);
int ans = 1;
long long best = (1LL << 62);
for(int p = 1; p <= n; p++){
  long long cur = diff;
  if(p < m) cur -= 1LL * s2 * (m - p);
  else if(p > m) cur += 1LL * s2 * (p - m);
  if(llabs(cur) < best || (llabs(cur) == best && p < ans))
    best = llabs(cur), ans = p;
}
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(1)$。

## 3. [【18NOIP普及组】摆渡车](http://ybt.ssoier.cn:8088/problem_show.php?pid=1980)

`普及组` `动态规划` `时间轴`

### 题意

有 `n` 名同学分别在时刻 `t_i` 到站等车，一辆摆渡车每次出发后完成“送到终点再返回起点”的整个往返要花 `m` 分钟，容量无限。你可以自由安排每次发车时刻，要求把所有同学都送走，并使总等待时间最小。

### 分析

题目条件里最有价值的是“容量无限”和“往返固定花 `m` 分钟”。这意味着一次发车其实对应时间轴上的一个切分点：凡是在上一次发车之后、当前时刻之前到站的同学，都可以被这一趟一起带走。所以这题可以直接压成时间轴动态规划，而不是去想复杂调度。

设 `cnt[i]` 表示时刻 `i` 之前到站的总人数，`sum[i]` 表示这些人的到站时间和。再设 `dp[i]` 表示**最后一趟车恰好在时刻 `i` 出发，并且所有到站时间不超过 `i` 的同学都已经被送走**时的最小等待总和。若上一趟车在 `j` 时刻发出，那么中间必须满足 `j <= i-m`；这时所有在区间 `(j,i]` 到站的同学都等到 `i` 才上车，新增加的等待就是 `(cnt[i]-cnt[j]) * i - (sum[i]-sum[j])`。

于是转移写成 `dp[i] = min(dp[j] + 这段等待)`。为什么只枚举 `j` 在 `[i-2m+1, i-m]` 之间就够了？因为如果两次真正发车之间空得更久，前一趟车完全可以向右平移到 `i-m` 而不破坏可行性，只会让之前那批同学等得更少，不会更差。这样每个 `i` 只看不超过 `m` 个前驱，正好利用了题目给出的 `m<=100`。

最后一名同学到站时刻记为 `mx`。最后一趟车一定会落在 `[mx, mx+m-1]` 之间，所以在这一段里取 `dp[i]` 的最小值，就是最终答案。

### 核心代码

```cpp
for(int i = 1; i <= mx + m - 1; i++){
  cnt[i] += cnt[i - 1];
  sum[i] += sum[i - 1];
}
fill(dp, dp + mx + m, INF);
dp[0] = 0;
for(int i = 1; i <= mx + m - 1; i++){
  for(int j = max(0, i - 2 * m + 1); j <= i - m; j++){
    long long w = 1LL * (cnt[i] - cnt[j]) * i - (sum[i] - sum[j]);
    dp[i] = min(dp[i], dp[j] + w);
  }
}
long long ans = INF;
for(int i = mx; i <= mx + m - 1; i++) ans = min(ans, dp[i]);
```

### 复杂度

设 `T = max(t_i) + m`，时间复杂度 $O(Tm)$，空间复杂度 $O(T)$。

## 4. [【18NOIP普及组】对称二叉树](http://ybt.ssoier.cn:8088/problem_show.php?pid=1981)

`普及组` `树哈希` `递归`

### 题意

给出一棵带点权的二叉树。要求在所有以某个节点为根的子树中，找出一棵“左右镜像后结构不变且对应点权相等”的对称二叉子树，并输出它的最大节点数。

### 分析

直接对每个点暴力判断左右子树是否互为镜像，会把大量相同结构重复比较，规模到 `10^6` 显然撑不住。题目条件真正提示我们的，是“结构相同且点权相等”本质上就是一个**可递归定义的同构判定**：一棵树是否对称，只取决于它的左子树和右子树是否互为镜像。

因此可以对每个节点同时算两种哈希：`h[u]` 表示按“左孩子在前、右孩子在后”编码出来的子树哈希，`rh[u]` 表示按“右孩子在前、左孩子在后”编码出来的镜像哈希；再顺带维护 `sz[u]` 表示子树大小。空节点用固定常量表示。这样递归回溯时，`h[u]` 和 `rh[u]` 都能由孩子一趟合成出来。

关键不变量是：`h[u]` 唯一刻画原子树，`rh[u]` 唯一刻画镜像后的子树。于是 `h[u] == rh[u]` 就等价于“整棵以 `u` 为根的子树与自己的镜像完全一致”，也就是题目要求的对称。每次回溯到一个节点时都检查一次，如果成立就用 `sz[u]` 更新答案；最终留下来的最大值就是要输出的节点数。

### 核心代码

```cpp
using ull = unsigned long long;
const ull B1 = 131, B2 = 233;
void dfs(int u){
  if(u == -1) return;
  dfs(lc[u]);
  dfs(rc[u]);
  int L = lc[u] == -1 ? 0 : lc[u];
  int R = rc[u] == -1 ? 0 : rc[u];
  sz[u] = sz[L] + sz[R] + 1;
  h[u] = (h[L] * B1 + val[u]) * B2 + h[R];
  rh[u] = (rh[R] * B1 + val[u]) * B2 + rh[L];
  if(h[u] == rh[u]) ans = max(ans, sz[u]);
}
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

# 二、提高组赛题

提高组五题把建模继续往深处推：先是用可达性判断面额是否冗余，再把答案二分到树上可行性，再把唯一环拆掉求最优深搜序，最后落到数学归纳与动态规划这种“先认结构、再写转移”的题型。

## 5. [【18NOIP提高组】货币系统](http://ybt.ssoier.cn:8088/problem_show.php?pid=1905)

`提高组` `完全背包` `贪心判冗`

### 题意

给出一个货币系统的若干面额，每种面额张数无限。要删掉尽量多的面额，但删完后的系统与原系统在“哪些金额可表示、哪些不可表示”上完全等价，求最少保留多少种面额。

### 分析

题目要的不是怎么拼出某个金额，而是哪些面额其实“可有可无”。判断一个面额 `a[i]` 是否冗余，唯一要问的是：**在只使用更小面额的前提下，`a[i]` 本身能不能被表示出来？** 如果能，那么以后所有包含 `a[i]` 的方案，都可以把这一张面额替换成若干张更小面额，整套可表示金额集合不会变；反之它必须保留。

所以先把面额从小到大排序，用一个完全背包可达数组 `f` 维护“当前已经保留的面额能拼出哪些金额”。枚举到 `a[i]` 时，如果 `f[a[i]]` 已经为真，说明它早就能被更小面额凑出来，是冗余面额；否则它是新的基础面额，答案加一，并用它去更新后续的可达状态。

这里的状态不变量非常明确：处理到第 `i` 个面额时，`f[x]` 只记录“由所有已保留、且不大于 `a[i]` 的面额能否拼出金额 `x`”。扫描结束后，被判为“必须保留”的个数，就是最小等价系统的大小。

### 核心代码

```cpp
sort(a + 1, a + n + 1);
memset(f, 0, sizeof f);
f[0] = 1;
int ans = 0;
for(int i = 1; i <= n; i++){
  if(f[a[i]]) continue;
  ans++;
  for(int s = a[i]; s <= a[n]; s++)
    f[s] |= f[s - a[i]];
}
```

### 复杂度

设最大面额为 `A`，时间复杂度 $O(nA)$，空间复杂度 $O(A)$。

## 6. [【18NOIP提高组】赛道修建](http://ybt.ssoier.cn:8088/problem_show.php?pid=1906)

`提高组` `二分答案` `树上贪心`

### 题意

给一棵带边权的树，要选出 `m` 条互不共边的简单路径作为赛道。每条赛道的长度是路径上边权和。要求把这 `m` 条赛道里最短的一条尽量做长，输出这个最大值。

### 分析

“让最短赛道尽量长”几乎就是在提醒我们做二分答案。设当前二分值为 `L`，问题就变成：能不能在树上找出至少 `m` 条互不共边、长度都不少于 `L` 的路径？如果可行，说明答案至少是 `L`；否则 `L` 太大。

检查 `L` 时要做的是一趟树形贪心。对每个节点，把每个儿子子树里“还没有凑成一条合法赛道、准备继续向上延伸”的最长残链长度取回来，再加上连到当前节点的边权。若某条残链本身已经达到 `L`，它可以独立成为一条赛道，计数加一；其余不足 `L` 的残链只能尝试在当前节点两两配对，因为两条来自不同儿子的链拼在一起，正好形成一条经过当前节点的简单路径。

贪心的关键不变量是：当前节点至多只能向父亲继续送出一条残链，因为父边只能继续接一条路径。因此我们把所有不足 `L` 的残链排序后，用“最大的链优先找能配上的最小伙伴”去尽量多配对；配不上的那些里，只保留最长的一条向上返回，其他都作废。这样既不会浪费能配成合法赛道的组合，也给父亲留下了最有希望继续变长的残链。

整棵树检查完后，若合法赛道数量不少于 `m`，说明 `L` 可行。二分结束时最后一个可行的 `L`，就是题目要求的最大最小长度。

### 核心代码

```cpp
int cnt;
int dfs(int u, int fa, int lim){
  vector<int> v;
  for(auto [to, w] : g[u]) if(to != fa){
    int t = dfs(to, u, lim) + w;
    if(t >= lim) cnt++;
    else v.push_back(t);
  }
  sort(v.begin(), v.end());
  multiset<int> s(v.begin(), v.end());
  int ret = 0;
  while(!s.empty()){
    auto it = prev(s.end());
    int x = *it; s.erase(it);
    auto jt = s.lower_bound(lim - x);
    if(jt != s.end()) s.erase(jt), cnt++;
    else ret = max(ret, x);
  }
  return ret;
}
```

### 复杂度

单次检查时间复杂度 $O(n \log n)$，总时间复杂度 $O(n \log n \log W)$，其中 `W` 是边权和上界；空间复杂度 $O(n)$。

## 7. [【18NOIP提高组】旅行](http://ybt.ssoier.cn:8088/problem_show.php?pid=1907)

`提高组` `基环树` `字典序`

### 题意

给出一个连通无向图，满足 `m=n-1` 或 `m=n`。从某个起点开始，每次只能沿边走到一个没访问过的新点，或者按第一次进入该点的边原路回退；把每次首次到达的新点编号按顺序记下来，得到长度为 `n` 的访问序列。要求输出字典序最小的合法序列。

### 分析

题目允许“走到新点”与“沿原路回退”，这正是深度优先遍历的行为模型。所以先把题面翻译掉：我们真正要找的是**某种深度优先遍历访问顺序里，首次访问序列的字典序最小值**。

当 `m=n-1` 时图是一棵树，没有选择回路。此时把每个点的邻接表从小到大排序，再从 `1` 开始做深度优先遍历，每一步总是先走编号最小的未访问邻点，就已经是字典序最小的首访序列了。因为树上没有额外回边，越早选更小的点就一定更优。

难点在 `m=n`，图里恰好多出一条环。此时真正影响字典序的，是“环上哪一条边被当成无效边跳过”，因为删去环上的任意一条边后，整张图就重新变回一棵树。于是先找出唯一环，再枚举删去环上的每一条边，分别在排序后的图上做一次深度优先遍历，得到一条合法首访序列；最后按字典序取最小的那一条。状态不变量是：每次试删一条环边后，我们都在一棵树上跑标准最小字典序深搜，所以得到的是该删边方案下的最优序列。

枚举完所有环边后，字典序最小的那条序列就是答案。因为原图只有一个环，所有合法深搜方案的差异都已经被这一步完整覆盖了。

### 核心代码

```cpp
void dfs(int u, int bu, int bv){
  vis[u] = 1;
  seq.push_back(u);
  for(int v : g[u]){
    if(vis[v]) continue;
    if((u == bu && v == bv) || (u == bv && v == bu)) continue;
    dfs(v, bu, bv);
  }
}
// 树：直接 dfs(1, -1, -1)
// 单环：枚举每条环边 (bu, bv)，取字典序最小的 seq
```

### 复杂度

时间复杂度 $O(n^2)$，空间复杂度 $O(n)$。

## 8. [【18NOIP提高组】填数游戏](http://ybt.ssoier.cn:8088/problem_show.php?pid=1908)

`提高组` `数学归纳` `快速幂`

### 题意

在一个 $n \times m$ 的棋盘里填 `0/1`。对任意两条从左上到右下、只会向右或向下走的合法路径，如果路径动作串 $w(P_1)$ 的字典序大于 $w(P_2)$，就要求对应读出的数字串 $s(P_1)$ 的字典序不大于 $s(P_2)$。求满足条件的填法总数，对 $10^9+7$ 取模。

### 分析

这题最难的不是写代码，而是先把全局路径比较压成可归纳的结构性质。把格子按“左下到右上”的对角线来看，可以推出两个关键结论：第一，每条这样的对角线上的数字必须单调；第二，一旦某个位置的上方和左方取值相同，右下角那一整块子矩阵里的对角线形态就会被连锁固定。也就是说，合法方案并不是任意自由填，而是被对角线结构强烈约束。

有了这个结构，再把矩阵旋转对称性用起来，可以先令 $n \le m$。接下来按较小维度分类：$n=1$ 时每个格子独立，答案是 $2^m$；$n=2$ 时每新增一列只带来三种延续方式，答案是 $4 \cdot 3^{m-1}$；$n=3$ 时同理得到 $112 \cdot 3^{m-3}$。真正麻烦的是 $n \ge 4$，这时先对前面几条对角线做分类讨论，能推出两个基值：

- $f(n,n) = \dfrac{83 \cdot 8^n + 5 \cdot 2^{n+7}}{384}$

- $f(n,n+1) = \dfrac{83 \cdot 8^n + 2^{n+8}}{128}$

接下来利用前面的“相同即锁死”性质，可以证明只要 $m \ge n+1$，再往右多扩一列时，每种合法形态都恰好只会分裂出 $3$ 种合法后继，因此 $f(n,m+1) = 3f(n,m)$。这就把大范围计数彻底压成了快速幂：当 $m=n$ 时直接取 $f(n,n)$；当 $m>n$ 时先取 $f(n,n+1)$，再乘上 $3^{m-n-1}$。答案提取时把分母换成模逆即可。

### 核心代码

```cpp
const long long MOD = 1000000007;
const long long INV128 = 570312504;
const long long INV384 = 190104168;
if(n > m) swap(n, m);
if(n == 1) ans = qpow(2, m);
else if(n == 2) ans = 4 * qpow(3, m - 1) % MOD;
else if(n == 3) ans = 112 * qpow(3, m - 3) % MOD;
else if(n == m)
  ans = (83 * qpow(8, n) + 5 * qpow(2, n + 7)) % MOD * INV384 % MOD;
else
  ans = (83 * qpow(8, n) + qpow(2, n + 8)) % MOD * qpow(3, m - n - 1) % MOD * INV128 % MOD;
```

### 复杂度

时间复杂度 $O(\log \max(n,m))$，空间复杂度 $O(1)$。

## 9. [【18NOIP提高组】保卫王国](http://ybt.ssoier.cn:8088/problem_show.php?pid=1909)

`提高组` `树形动态规划` `动态规划` `重链剖分`

### 题意

一棵树上每个点驻军有代价，要求每条边至少有一个端点驻军，也就是带权树上点覆盖。接着有 `m` 次询问，每次强制指定两个点“必须驻军”或“不得驻军”，要求在这一附加条件下给出最小总代价；若无解输出 `-1`。

### 分析

先不看询问，这就是标准树形动态规划。设 $f[u][0]$ 表示 $u$ 不驻军时覆盖整棵 $u$ 子树的最小代价，$f[u][1]$ 表示 $u$ 驻军时的最小代价，那么有

- $f[u][0] = \sum f[v][1]$

- $f[u][1] = p[u] + \sum \min(f[v][0], f[v][1])$

这一步已经把题目条件翻成了最核心的状态与转移。但如果每次询问都重跑整棵树，$n,m$ 都到 $3 \times 10^5$ 肯定不行。

真正可做的关键在于：一次“强制某点取 0 或 1”只会影响它到根这条链上的动态规划值。于是用重链剖分把树拆成若干重链，再把每个点对其父亲的转移写成一个 $2 \times 2$ 的最小加法矩阵。若把轻儿子的贡献先揉进 $g[u][0/1]$，那么点 $u$ 的局部矩阵可以写成

$A_u = \begin{bmatrix} \infty & g[u][1] \\ g[u][0] & g[u][1] \end{bmatrix}$

这里行表示父节点是否驻军，列表示 $u$ 是否驻军：如果父亲不驻军而 $u$ 也不驻军，就会留下未覆盖边，所以对应项是 $\infty$。强制 $u$ 取某个状态时，只需把不允许的那一列改成 $\infty$。线段树维护整条重链上的矩阵乘积，点修改后沿树剖路径向上更新，最终根所在链的总矩阵就给出了全树答案。

这个做法的核心不变量是：矩阵乘积始终表示“沿着这条重链往下走，父状态如何转成链尾状态，并累计最小代价”。每次询问临时修改两个点的矩阵、查询根答案，再恢复原矩阵即可。若根矩阵对应的最优值仍是无穷大，就说明这一组强制条件无解；否则最小值就是答案。

### 核心代码

```cpp
struct Mat{
  long long a[2][2];
  Mat operator * (const Mat& o) const {
    Mat c = {{{INF, INF}, {INF, INF}}};
    for(int i = 0; i < 2; i++)
      for(int k = 0; k < 2; k++)
        for(int j = 0; j < 2; j++)
          c.a[i][j] = min(c.a[i][j], a[i][k] + o.a[k][j]);
    return c;
  }
};
Mat make(int u){
  Mat t = {{{INF, g1[u]}, {g0[u], g1[u]}}};
  if(force[u] == 0) t.a[0][1] = t.a[1][1] = INF;
  if(force[u] == 1) t.a[1][0] = INF;
  return t;
}
// 线段树维护重链矩阵积，单点改矩阵后重新查询根链乘积
```

### 复杂度

预处理时间复杂度 $O(n)$，单次询问时间复杂度 $O(\log^2 n)$，空间复杂度 $O(n)$。
