# 一本通 2023赛题解题报告

这一年的题目从普及组的直接建模一路推到提高组的状态设计、判定路径与区间优化，表面上分散，底层却很统一：先把题面里的“限制”翻成可维护的状态，再看哪些量能单调、哪些代价能前缀化，最后把答案落到最稳的那一层状态上。读的时候不妨先盯住“什么信息必须一直带着走”，很多看起来难的地方，往往就是这一步想清之后自然展开的。

# 一、CSP-J 赛题

这一章更像是在练“第一步翻译题面”的基本功：有的题把过程压成递推，有的题把代价改写成前缀需求，有的题虽然披着图论外衣，本质仍然是在维护一个不丢信息的状态。

## 1. [【23CSPJ普及组】小苹果(apple)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2098)

`CSP 2023` `CSP-J`

### 题意

桌上从左到右摆着编号为 $1$ 到 $n$ 的苹果。每天都会从当前序列的第 $1$ 个苹果开始，每隔 $2$ 个再拿走 $1$ 个，也就是删去当前位置为 $1,4,7,\dots$ 的苹果，然后把剩下的苹果重新并成一列。题目要求求出：所有苹果被拿完一共需要多少天，以及原来编号为 $n$ 的苹果会在第几天被拿走。

### 分析

这题最关键的不是真的去模拟整列苹果，而是先看清一轮操作到底改变了什么。若当前有 $m$ 个苹果，被删掉的位置正好是所有满足 $pos\equiv 1\pmod 3$ 的位置，因此一轮一共删掉 $\lceil m/3\rceil$ 个，留下来的苹果数就是 $m-\lceil m/3\rceil=\lfloor 2m/3\rfloor$。所以“总共还剩多少个苹果”满足一个非常短的递推。

接下来想原来编号为 $n$ 的那个苹果。它在当前序列里如果恰好落在位置 $1,4,7,\dots$，也就是当前位置满足 $pos\equiv 1\pmod 3$，那么这一天就会被拿走；否则它虽然没被删，但因为它本来就是最右边那个苹果，删完之后它仍然会是新序列里最右边那个苹果。于是它的新位置恰好就是删完后的总长度，也同样变成 $\lfloor 2m/3\rfloor$。

这就把两个问题统一起来了：总天数和“编号为 $n$ 的苹果当前所处位置”都按同一个递推往下走，只是后者在第一次出现 $m\equiv 1\pmod 3$ 时记录答案即可。整个过程只会做 $O(\log n)$ 轮缩小。

### 核心代码

```cpp
long long days = 0, hit = 0, cur = n;
while(cur > 0){
  days++;
  if(!hit && cur % 3 == 1) hit = days;
  cur -= (cur + 2) / 3;
}
cout << days << ' ' << hit;
```

### 复杂度

时间复杂度 $O(\log n)$，空间复杂度 $O(1)$。

## 2. [【23CSPJ普及组】公路(road)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2099)

`CSP 2023` `CSP-J`

### 题意

有 $n$ 个站点，站点 $i$ 到 $i+1$ 的距离是 $v_i$。在站点 $i$ 买油的价格是 $a_i$ 元每升，油箱无限大，但每次买的升数必须是整数。一开始车在 $1$ 号站点且油箱为空，每升油能跑 $d$ 公里，要求求出从 $1$ 号站开到 $n$ 号站的最小花费。

### 分析

很多人第一反应会去想“在每个站点买多少升”，但这题真正该维护的是“到当前为止一共至少买了多少升”。假设前缀总路程是 $S$，因为每升油最多跑 $d$ 公里，所以走完这段路前，总购油量至少要达到 $\lceil S/d\rceil$。这是一个只随着前缀路程增长而单调增加的量。

于是可以把整题理解成：随着公路往前扫描，当“前缀至少需要的总升数”从 $need$ 增加到 $need'$ 时，新增出来的这几升油到底该在哪买。由于油箱没有容量限制，这几升完全可以提前在之前的某个更便宜站点买好，所以最优策略一定是：**每一升新增的必要油量，都按此前出现过的最低油价购买。**

扫描到第 $i$ 段路时，先把站点 $i$ 的油价纳入前缀最小值，再根据新的前缀距离算出总共至少需要多少升油。如果所需升数增加了 $\Delta$，就说明这 $\Delta$ 升必须在前 $i$ 个站点里买出来，而最划算的办法就是全部按前缀最低价结算。这样一次线性扫描就够了。

### 核心代码

```cpp
long long ans = 0, dist = 0, need = 0, mn = (long long)4e18;
for(int i = 1; i < n; i++){
  mn = min(mn, 1LL * a[i]);
  dist += v[i];
  long long cur = (dist + d - 1) / d;
  ans += (cur - need) * mn;
  need = cur;
}
cout << ans;
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(1)$。

## 3. [【23CSPJ普及组】一元二次方程(uqe)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2100)

`CSP 2023` `CSP-J`

### 题意

给出 $T$ 个一元二次方程 $ax^2+bx+c=0$，要求对每个方程输出较大的实数根；如果没有实数根输出 `NO`。若答案是有理数，要化成最简分数；若答案含根号，还要写成题面要求的标准形式：把平方因子从判别式里提出，并把分子分母约到最简。

### 分析

整题的骨架其实非常固定：先看判别式 $\Delta=b^2-4ac$。如果 $\Delta<0$，直接无实根；如果 $\Delta$ 是完全平方数，那么较大根就是 $\dfrac{-b+\sqrt{\Delta}}{2a}$，只要把分子分母同除最大公因数即可。

难点在 $\Delta$ 不是完全平方数时的格式化。此时要先把 $\Delta$ 写成 $k^2r$，其中 $r$ 不含平方因子，于是较大根可以写成 $\dfrac{-b+k\sqrt r}{2a}$。接下来要做的其实只有三步：先把分母调成正数，再把有理部分和根号部分能约掉的公因数都约掉，最后按照题面分类讨论输出——有理部分是否为 $0$、根号系数是否为 $1$、分母是否为 $1$，每个分支都对应一种固定模板。

因此这题真正练的是“把一个数学表达式规整成唯一标准形”。只要始终围绕“提平方因子、统一符号、整体约分、最后格式化”这条线写，代码虽然分支多，但逻辑是顺着表达式一步步收紧的。

### 核心代码

```cpp
string solve(long long a,long long b,long long c){
  long long D=b*b-4*a*c;
  if(D<0) return "NO";
  long long s=sqrtl(D);
  if(s*s==D) return fmtRat(-b+s,2*a);
  long long r=D,k=1;
  for(long long p=2;p*p<=r;p++) while(r%(p*p)==0) r/=p*p,k*=p;
  long long num=-b, den=2*a, coef=k;
  if(den<0) den=-den, num=-num, coef=-coef;
  long long g=gcd(gcd(llabs(num), llabs(coef)), den);
  num/=g; coef/=g; den/=g;
  return fmtMix(num,coef,r,den);
}
```

### 复杂度

单个方程若直接试除提平方因子，时间复杂度为 $O(\sqrt{\Delta})$，额外空间复杂度 $O(1)$。

## 4. [【23CSPJ普及组】旅游巴士(bus)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2101)

`CSP 2023` `CSP-J`

### 题意

有一张有向图，游客只能从 $1$ 号点进入，从 $n$ 号点离开。巴士在时刻 $0,k,2k,\dots$ 到达入口并从出口发车，所以进入和离开的时刻都必须是 $k$ 的非负整数倍。每条边走过都恰好花 $1$ 单位时间，但只能在不早于该边开放时间 $a_i$ 时通过，而且游客在景区里不能停留，只能一直沿边移动。问最早能在什么时刻离开；若无解输出 $-1$。

### 分析

这题看起来最麻烦的是“不能等待”，但只要换个角度，等待其实被“推回到起点”了。设你已经固定了一条走法，那么把进入景区的时间整体推迟 $k$ 的整数倍，整条路径上每一步到达各点的时间模 $k$ 都不会变，只是所有时刻一起整体后移。因此一条路径是否能满足某条边的开放时间，本质只和“到达这条边前的时刻模 $k$”有关。

于是自然想到做分层最短路。状态设成 $(u,r)$，表示到达点 $u$ 且当前时刻对 $k$ 取模等于 $r$。`dist[u][r]` 记录达到这个状态的最小真实时刻。若当前时刻是 $t$，要走一条开放时间为 $a$ 的边：当 $t\ge a$ 时可以立刻走；当 $t<a$ 时，虽然途中不能停，但我们完全可以把一开始乘车进景区的时刻整体推迟若干个 $k$，直到这一步恰好满足开放要求，所以最早可行出发时刻就是不小于 $a$ 且与 $t$ 同余模 $k$ 的最小值。

这样每次转移都变成“把 $t$ 向上取整到某个模类，再加一条边的时间”，边权非负，直接用 Dijkstra 即可。最终只有在时刻模 $k$ 为 $0$ 到达终点，才能刚好赶上离开景区的巴士，所以答案就是 `dist[n][0]`。

### 核心代码

```cpp
using State = pair<long long,int>;
vector<vector<long long>> dist(n+1, vector<long long>(k, INF));
priority_queue<State, vector<State>, greater<State>> pq;
dist[1][0] = 0;
pq.push({0, 1 * k});
while(!pq.empty()){
  auto [t, code] = pq.top(); pq.pop();
  int u = code / k, r = code % k;
  if(t != dist[u][r]) continue;
  for(auto [v, a] : g[u]){
    long long depart = t;
    if(depart < a) depart += (a - depart + k - 1) / k * k;
    long long nt = depart + 1;
    int nr = nt % k;
    if(nt < dist[v][nr]) dist[v][nr] = nt, pq.push({nt, v * k + nr});
  }
}
cout << (dist[n][0] == INF ? -1 : dist[n][0]);
```

### 复杂度

设状态数为 $nk$，时间复杂度 $O((nk+m k)\log(nk))$，空间复杂度 $O(nk)$。

# 二、CSP-S 赛题

提高组这一章开始，题目更强调“信息如何压缩”与“答案怎样落点”：有的题把复杂过程压成候选集合，有的题把前缀过程抽成等价状态，有的题要先把题面翻译成区间、函数图或单调路径，再做真正的算法设计。

## 5. [【23CSPS提高组】密码锁(lock)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2102)

`CSP 2023` `CSP-S`

### 题意

一把五位密码锁的正确密码未知。每次上锁时，只会从正确密码出发做一次操作：要么转动某一个拨圈若干格，要么同时转动两个相邻拨圈同样多的格数。现在给出若干个“上锁后的状态”，并保证这些状态都不是正确密码。问一共有多少种正确密码，能够通过题目规定的上锁方式生成给出的全部状态。

### 分析

这里最值钱的信息是：密码锁只有 $5$ 个拨圈，而且每次只转一次。于是与其正着猜正确密码，不如从某个观测状态反推“它可能是由哪些正确密码一步转过来的”。对于一个固定状态，如果只转一个拨圈，那么有 $5$ 个位置、每个位置有 $9$ 种非零转动幅度，总共 $45$ 个前驱；如果转两个相邻拨圈，则有 $4$ 对相邻位置、每对也有 $9$ 种非零幅度，总共 $36$ 个前驱。一共正好是 $81$ 个候选。

既然 $n\le 8$，那最划算的做法就是：只从第一条观测状态出发，枚举这 $81$ 个候选正确密码，然后拿这些候选去验证所有其他状态。验证时也很直接——把候选密码和目标状态逐位比较：若恰好只有一个位置不同，而且差值是某个非零循环位移，就对应“转一个拨圈”；若恰好有两个相邻位置不同，且这两个位置的循环差值完全相同，也对应“转两个相邻拨圈”；除此之外都不合法。

所以这题不是在大范围搜索密码，而是在利用“一次操作”的强限制，把全体答案先压成一个很小的候选集合，再逐个过滤。候选集足够小，整体就变成了非常稳定的暴力判定题。

### 核心代码

```cpp
set<string> cand;
for(int i = 0; i < 5; i++) for(int d = 1; d <= 9; d++){
  auto t = s0;
  t[i] = char((t[i] - '0' - d + 10) % 10 + '0');
  cand.insert(t);
}
for(int i = 0; i < 4; i++) for(int d = 1; d <= 9; d++){
  auto t = s0;
  t[i] = char((t[i] - '0' - d + 10) % 10 + '0');
  t[i + 1] = char((t[i + 1] - '0' - d + 10) % 10 + '0');
  cand.insert(t);
}
int ans = 0;
for(auto pwd : cand){
  bool ok = true;
  for(auto st : obs) ok &= oneMove(pwd, st);
  ans += ok;
}
```

### 复杂度

候选数是常数 $81$，时间复杂度 $O(81\cdot n)$，空间复杂度 $O(81)$。

## 6. [【23CSPS提高组】消消乐(game)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2103)

`CSP 2023` `CSP-S`

### 题意

给定一个长度为 $n$ 的字符串。每次允许删除两个相邻且相同的字符，删除后左右拼接。定义一个连续子串是“可消除的”，当且仅当经过若干次这样的操作后能变成空串。要求统计所有非空连续子串中，可消除的子串个数。

### 分析

这题不能直接枚举子串，关键是先把“能否消成空”变成一个前缀状态相等的判定。对任意字符串，若我们从左到右维护一个栈：当前字符和栈顶相同就弹栈，否则压栈，那么最后留下的栈，正好就是这条消除规则下的唯一“最简剩余形态”。这一步很重要，因为它告诉我们：题目的操作虽然顺序很多，但最后的归约结果是唯一的。

设 `rt[i]` 表示前缀 $s_1\dots s_i$ 归约后的栈状态。如果某个子串 $s_{l..r}$ 可以完全消空，那么把它插在前缀 $s_1\dots s_{l-1}$ 后面，并不会改变归约结果，因此一定有 $rt[l-1]=rt[r]$。反过来，如果这两个前缀状态相等，也说明中间这段子串在这个消除体系里等价于“什么都没发生”，也就能消成空。于是问题就变成：有多少对前缀状态完全相同。

剩下的工作是把栈状态高效表示出来。由于每一步只会在上一步状态上做一次“压一个字符”或“弹一个字符”，我们可以把每次压栈看成新建一个结点，结点里存字符和父指针；弹栈就回到父结点。这样每个前缀都只对应一个“栈顶结点编号”。扫描字符串时统计每个编号出现了多少次，当前编号再次出现，就能和之前所有同编号前缀配成可消除子串。

### 核心代码

```cpp
cnt[0] = 1;
for(char ch : s){
  if(top && val[top] == ch) top = fa[top];
  else{
    val[++tot] = ch;
    fa[tot] = top;
    top = tot;
  }
  ans += cnt[top];
  cnt[top]++;
}
cout << ans;
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 7. [【23CSPS提高组】结构体（struct）](http://ybt.ssoier.cn:8088/problem_show.php?pid=2104)

`CSP 2023` `CSP-S`

### 题意

需要模拟一种类似 C++ 的结构体系统。操作包括：定义结构体类型、定义变量、通过 `a.b.c` 形式访问成员地址，以及给一个内存地址反查它对应的最内层基本类型元素。规则里包含完整的对齐要求：基本类型按自身大小对齐，结构体按其成员对齐要求的最大值对齐，成员偏移和结构体总大小都要向上取整到相应倍数。

### 分析

这题的核心不是语法，而是两个永远不变的量：**类型大小**和**对齐要求**。只要先把每种类型的这两个量算对，后面无论是定义变量还是递归访问，都只是拿着这两个量做地址运算。定义一个结构体时，顺着成员依次放：当前偏移先向上取整到该成员的对齐要求，再把成员塞进去；全部成员放完后，整个结构体总大小还要再向上取整到该结构体自身的对齐要求。

定义变量时同理。把“当前内存尾部地址”向上取整到该类型的对齐要求，就是这个变量的起始地址，然后尾部再加上类型大小。这样所有根变量在内存里就是一段一段连续排开的区间。

正向查询 `a.b.c` 很直接：先找到根变量 `a` 的基址，再按成员表一层层往下跳，把每个成员偏移量累加上去。逆向查询地址更值得想清：先在线性扫描里找到地址落在哪个根变量的占用区间内，再递归向内层结构体下降。若当前类型是基本类型且地址真的落在它的实际占用字节里，就输出路径；若地址落进某层 padding，说明它不属于任何基本类型元素，答案就是 `ERR`。题目规模很小，直接模拟就足够稳定。

### 核心代码

```cpp
long long up(long long x,long long a){ return (x + a - 1) / a * a; }
Type build(vector<pair<string,string>> mems){
  Type cur{0,1,{}};
  for(auto [tp,name] : mems){
    auto &sub = types[tp];
    cur.sz = up(cur.sz, sub.al);
    cur.mem.push_back({name,tp,cur.sz});
    cur.sz += sub.sz;
    cur.al = max(cur.al, sub.al);
  }
  cur.sz = up(cur.sz, cur.al);
  return cur;
}
bool locate(string tp,long long base,long long addr,string path){
  if(isBasic(tp)) return base <= addr && addr < base + types[tp].sz ? (out = path, true) : false;
  for(auto &m : types[tp].mem){
    long long l = base + m.off, r = l + types[m.tp].sz;
    if(l <= addr && addr < r && locate(m.tp, l, addr, path + "." + m.name)) return true;
  }
  return false;
}
```

### 复杂度

设单次查询涉及的成员总数为 $L$，则定义与查询都在 $O(L)$ 内完成；总空间复杂度与定义出的类型、变量数量线性相关。

## 8. [【23CSPS提高组】种树（tree）](http://ybt.ssoier.cn:8088/problem_show.php?pid=2105)

`CSP 2023` `CSP-S`

### 题意

给定一棵树，第 $1$ 天必须在 $1$ 号点种树，此后每天最多在一个与已种点相邻的未种点再种一棵。节点 $i$ 从被种下那天开始，在第 $x$ 天会增长 $\max(b_i+x c_i,1)$ 的高度。要求所有点最终都被种上树，且到任务结束时第 $i$ 个点的树高都至少为 $a_i$，求最少需要多少天。

### 分析

这题最自然的总思路是二分答案。假设总天数固定为 $T$，问题就变成：每个点最晚能在第几天种下，才能在第 $T$ 天之前长到目标高度？如果这个“最晚种植日”能算出来，那么原题就会退化成一个带先修关系的调度问题。

先看单个点。若节点 $i$ 在第 $p$ 天种下，它从第 $p$ 天一直长到第 $T$ 天，累计高度是 $\sum_{x=p}^{T}\max(b_i+x c_i,1)$。这个序列要么一直是等差数列，要么先按等差走一段、再被下界 $1$ 截平，所以把拐点找出来以后，`gain(i,p,T)` 可以在 $O(1)$ 时间算出。于是对每个点都能二分出最大的可行种植日 $lim_i$。

接下来才是图上的本质：节点必须按“父亲先于儿子”的顺序被种下，而且每天只能种一个点，要求每个点都在自己的截止日 $lim_i$ 之前完成。这就是单位时间任务、树形先修关系、带截止日的可行性判定。做法是把树根定在 $1$，维护当前已经解锁、也就是父亲已经种过的所有节点，在这些可选节点里总是优先种截止日最早的那个；如果某一天可选集合里最小截止日已经小于今天，说明它已经来不及了，任何更晚的安排都不可能补救。

所以完整算法就是：二分总天数 $T$，对每个点算 $lim_i$，再用“小根堆 + 截止日最早优先”判断能否完成种植。二分通过的最小 $T$ 就是答案。

### 核心代码

```cpp
long long latestDay(int i,long long T){
  long long l = 1, r = min<long long>(T, n), ans = 0;
  while(l <= r){
    long long mid = (l + r) >> 1;
    if(gain(i, mid, T) >= a[i]) ans = mid, l = mid + 1;
    else r = mid - 1;
  }
  return ans;
}
bool check(long long T){
  for(int i = 1; i <= n; i++) lim[i] = latestDay(i, T);
  priority_queue<pair<long long,int>, vector<pair<long long,int>>, greater<pair<long long,int>>> pq;
  pq.push({lim[1], 1});
  for(int day = 1; day <= n; day++){
    if(pq.empty() || pq.top().first < day) return false;
    int u = pq.top().second; pq.pop();
    for(int v : g[u]) if(v != fa[u]) fa[v] = u, pq.push({lim[v], v});
  }
  return true;
}
```

### 复杂度

若 `gain` 能在 $O(1)$ 计算，则一次可行性检查时间复杂度为 $O(n\log n)$，整体二分后为 $O(n\log n\log T)$，空间复杂度 $O(n)$。

# 三、NOIP 赛题

这一章开始是 2023 年的 NOIP 提高组赛题。和前面的 CSP 相比，这几题更强调把抽象出来的模型继续往前推：有的题要把奇偶结构拆成可贪心的两部分，有的题要围绕“第一次失误”反向计数，有的题要抓住树上唯一传递链或把任意长度范围拆成少量可预处理的块。

## 9. [【23NOIP提高组】词典](http://ybt.ssoier.cn:8088/problem_show.php?pid=2106)

`NOIP 2023` `NOIP 提高组`

### 题意

有 $n$ 个长度都为 $m$ 的不同单词。每个单词都允许任意交换内部字符。对于每个单词 $w_i$，判断是否存在一组重排结果，使得重排后的 $w_i$ 字典序严格小于其他所有单词的重排结果。

### 分析

题面虽然说“每个单词都可以任意交换字符”，但这恰好意味着每个单词的最优形态是固定的。若想让某个单词 $w_i$ 尽量小，显然应该把它的字符升序排列；若想让别的单词尽量大，从而更容易被 $w_i$ 压住，就应该把它们降序排列。也就是说，关于 $w_i$ 能否成为全场最小，最有利的选择已经被唯一确定了。

于是判定条件就变成：设 $mn_i$ 为 $w_i$ 升序后的字符串，$mx_j$ 为 $w_j$ 降序后的字符串，那么 $w_i$ 可行，当且仅当 $mn_i < mx_j$ 对所有 $j\ne i$ 都成立。因为若连“自己最小、别人最大”都压不过某个对手，那再怎么交换也不可能成功；反之，如果在这组最有利形态下已经比所有人小，直接采用它们就是一组可行方案。

剩下只要把“对所有 $j\ne i$”压成一个全局最小值比较。预处理所有 $mx_j$ 中最小和次小的两个字符串：当 $i$ 不是全局最小者时，用全局最小去比较；当 $i$ 恰好就是那个最小者时，就改用次小值。这样每个单词只需一次比较就能得出答案。

### 核心代码

```cpp
for(int i = 1; i <= n; i++){
  mn[i] = mx[i] = w[i];
  sort(mn[i].begin(), mn[i].end());
  sort(mx[i].rbegin(), mx[i].rend());
}
string best1 = "{", best2 = "{";
int id1 = -1;
for(int i = 1; i <= n; i++){
  if(mx[i] < best1) best2 = best1, best1 = mx[i], id1 = i;
  else if(mx[i] < best2) best2 = mx[i];
}
for(int i = 1; i <= n; i++){
  string lim = (i == id1 ? best2 : best1);
  ans[i] = (n == 1 || mn[i] < lim) ? '1' : '0';
}
```

### 复杂度

排序部分时间复杂度 $O(nm\log m)$，比较与汇总部分时间复杂度 $O(nm)$，空间复杂度 $O(nm)$。

## 10. [【23NOIP提高组】三值逻辑](http://ybt.ssoier.cn:8088/problem_show.php?pid=2107)

`NOIP 2023` `NOIP 提高组`

### 题意

有 $n$ 个变量，取值属于 $T,F,U$。给定若干赋值语句，语句只可能是把某个变量赋成常量 $T/F/U$、另一个变量，或另一个变量的逻辑非。你需要选择一组初值，使得整段程序执行完后，每个变量的最终值都和初值完全相同；在所有满足条件的方案里，要求初值中取 $U$ 的变量尽量少。

### 分析

这题最重要的观察是：程序里没有与、或，只有常量赋值、拷贝和取反，所以无论执行多少步，一个变量最终都只会变成四种东西之一：某个常量 $T/F/U$，某个初始变量 $x_j$，或者它的相反值 $\lnot x_j$。因此我们完全可以把整段程序“符号化执行”一遍，最后得到每个变量的最终表达式。

然后把题意改写成固定点条件：对每个变量 $x_i$，要求 $\text{初值}=\text{最终表达式}$。若最终表达式是常量，就等价于把这个变量钉死；若最终表达式是另一个变量或其相反值，就得到一条形如 $x_i=x_j$ 或 $x_i=\lnot x_j$ 的关系。这样整张图每个点至多向外连一条边，天然形成函数图结构。

接下来按连通块讨论即可。若一个块最终流向常量 $T$ 或 $F$，整块取值都会被唯一确定，不需要任何 $U$；若流向常量 $U$，则沿着拷贝或取反往回推都仍然只能得到 $U$，整块都被迫取 $U$。更有意思的是纯变量环：如果环上一圈取反次数为偶数，相当于绕一圈回到自己，给它们赋一组 $T/F$ 就能自洽；如果为奇数，就会推出 $x=\lnot x$，在三值逻辑里只有 $U$ 能满足，所以这个环及所有指向它的点都必须取 $U$。统计这些被迫为 $U$ 的点数，就是最优答案。

### 核心代码

```cpp
for(int i = 1; i <= n; i++) expr[i] = {VAR, i, 1};
for(auto op : ops){
  if(op.t == 'T') expr[op.x] = {CONST, T, 1};
  else if(op.t == 'F') expr[op.x] = {CONST, F, 1};
  else if(op.t == 'U') expr[op.x] = {CONST, U, 1};
  else if(op.t == '+') expr[op.x] = expr[op.y];
  else expr[op.x] = negate(expr[op.y]);
}
for(int i = 1; i <= n; i++) buildFunctionalEdge(i, expr[i]);
for(each component){
  if(reachUConst(component) || oddNegCycle(component)) markAllBad(component);
}
cout << count(bad + 1, bad + n + 1, true);
```

### 复杂度

时间复杂度 $O(n+m)$，空间复杂度 $O(n)$。

## 11. [【23NOIP提高组】双序列拓展](http://ybt.ssoier.cn:8088/problem_show.php?pid=2108)

`NOIP 2023` `NOIP 提高组`

### 题意

给定两个序列 $X,Y$。允许把某个元素复制若干次，从而把原序列拓展成更长的序列，但元素相对顺序不能改变。题目问能否把两列都拓展到同一长度，使得对应位置上的差值始终同号，也就是始终满足 $f_i>g_i$ 或始终满足 $f_i<g_i$。除此之外还有若干独立询问，每次只在原序列上做一些修改后重新判断。

### 分析

这题最应该先建立的是“公共拓展”与网格路径的等价。设当前配对到了 $x_i$ 和 $y_j$，那么下一位要么继续复制 $x_i$，要么继续复制 $y_j$，要么两边同时往后走一步，所以整个过程可以看成在一个 $n\times m$ 的网格上，从左上角走到右下角的一条单调路径。路径经过格子 $(i,j)$，就表示某一段公共拓展里把 $x_i$ 和 $y_j$ 配在了一起。

于是“始终同号”就变成：这条路径经过的所有格子，都必须满足同一个严格不等式。比如我们尝试让 $X$ 全程压住 $Y$，那就要求路径只走在满足 $x_i>y_j$ 的格子里。把一段连续路径压成块来看，就是在拿一段连续的 $X$ 去覆盖一段连续的 $Y$；只要这整个矩形里的格子都合法，就等价于这段 $X$ 的最小值仍然大于这段 $Y$ 的最大值。

因此可以用双指针贪心地按块推进。固定当前块后，维护 `X` 这边的块最小值和 `Y` 这边的块最大值。若已经满足 `minX > maxY`，说明当前矩形整体合法，可以尝试结束这一块；若不满足，就必须把导致冲突的一侧继续并入当前块，因为不把它吃进来，路径根本绕不过这个坏格子。整个检查只需线性扫描一次；若要判断全程 `X<Y`，交换两列再做一遍即可。每个询问只在修改后的原序列上重新线性判定，不需要更重的结构。

### 核心代码

```cpp
bool check(vector<int> A, vector<int> B){
  if(A[1] <= B[1] || A.back() <= B.back()) return false;
  int i = 1, j = 1;
  while(i <= n && j <= m){
    int mnA = A[i], mxB = B[j];
    expandBlock(i, j, mnA, mxB, A, B);
    if(mnA <= mxB) return false;
    i++, j++;
  }
  return i == n + 1 && j == m + 1;
}
ans = check(X, Y) || check(Y, X);
```

### 复杂度

单次询问在线性扫描下时间复杂度为 $O(n+m)$，空间复杂度 $O(1)$（不计存储修改后的序列）。

## 12. [【23NOIP提高组】天天爱打卡（run）](http://ybt.ssoier.cn:8088/problem_show.php?pid=2109)

`NOIP 2023` `NOIP 提高组`

### 题意

试运行有 $n$ 天。若某天跑步打卡，能量值减少 $d$；并且不能连续打卡超过 $k$ 天。每个挑战给出 $(x_i,y_i,v_i)$，表示若到第 $x_i$ 天时连续打卡长度至少为 $y_i$，就额外获得 $v_i$。要求最大化试运行结束后的能量值。

### 分析

把挑战改写一下会更清楚：若在第 $x_i$ 天连续打卡长度至少为 $y_i$，等价于区间 $[x_i-y_i+1,\,x_i]$ 这几天全都打卡。于是每个挑战其实对应一个带权区间，只有当当前连续跑步段完整覆盖它时，奖励才会被触发。

这样做 DP 时，最自然的视角就不是“今天跑不跑”，而是“当前这一段连续跑步是从哪一天开始的”。设某一段连续跑步从 $s$ 开始、当前扫到右端点 $r$，那么这段的代价是长度乘上 $d$；而所有右端点等于 $r$、左端点不小于 $s$ 的挑战，都会在这一天一次性贡献奖励。于是对每个可能的起点 $s$，我们都在维护“让当前连续段从 $s$ 开始并延伸到今天，能得到多少收益”。

扫右端点 $r$ 时会发生三类统一更新：第一，所有仍在延伸的连续段都多跑了一天，所以统统减去当天代价；第二，所有以 $r$ 为右端点的挑战，会给所有满足 $s\le l_i$ 的起点同时加上奖励；第三，只允许连续跑 $k$ 天，所以真正可选的起点只能落在最近的长度窗口里。把这些“对一整段起点同时加减”的操作交给线段树做区间加，再维护区间最大值，就能在每个右端点快速得到“以今天结尾的最优连续段”。

由于 $n$ 可以很大，但真正会影响答案的位置只出现在挑战端点和长度窗口边界附近，所以先把所有可能成为起点、终点的关键日子离散化，再在离散后的轴上做同样的区间加与取最大值，就能把整题压到可做的规模。

### 核心代码

```cpp
compressDays();
seg.build(1, tot, NEG);
seg.modify(1, 1, dp[0]);
for(int p = 1; p <= tot; p++){
  seg.rangeAdd(1, 1, p, -1LL * d * (day[p] - day[p - 1]));
  for(auto [L, v] : addAt[p]) seg.rangeAdd(1, 1, L, v);
  int left = lower_bound(day + 1, day + tot + 1, day[p] - k + 1) - day;
  long long best = seg.query(1, left, p);
  dp[p] = max(dp[p - 1], best);
  seg.modify(1, p + 1, dp[p]);
}
cout << dp[tot];
```

### 复杂度

离散后设关键位置数为 $M$，则时间复杂度 $O(M\log M)$，空间复杂度 $O(M)$。
