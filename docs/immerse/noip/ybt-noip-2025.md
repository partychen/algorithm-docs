# 一本通 2025赛题解题报告

这一组题从 CSP-J 的局部选择与前缀建模出发，经过 CSP-S 的结构化 DP、图论与数据结构，最后落到 NOIP 提高组里的贪心计数、树形 DP 与区间最优决策。表面上题型跨度很大，真正反复出现的却是同一条赛场主线：先把题意压成一个可比较、可转移的状态，再判断贪心能不能直接成立；若不能，就继续把失败的边界补成计数、数据结构或结构化 DP。

# 一、CSP-J 赛题

这一章更像是在练“把规则翻译成判定式”的基本功：有的题只要看清比较顺序，有的题要把区间条件改写成前缀信息，还有的题则是在计数前先找到唯一负责某个方案的那根木棍。

## 1. [【25CSPJ普及组】拼数](http://ybt.ssoier.cn:8088/problem_show.php?pid=2129)

`CSP 2025` `CSP-J`

### 题意

给出一个只含小写字母和数字的字符串，可以从中挑出任意多个数字、重新排列，并且每个数字字符至多使用一次。要求拼出的结果是一个正整数，并且数值尽可能大。

### 分析

这题最容易多想成“挑哪些数字”和“怎么排顺序”两个决策，但其实先抓住整数比较规则就够了：先比位数，再比字典序。只要已经有一个非零数字能放在最前面，多拿一个数字就会让位数增加，因此一定不会更差。也就是说，字母全部丢掉，所有数字都应该保留。

剩下的问题只是在固定数字 multiset 的前提下怎样排得最大。答案当然是把数字按从大到小排。因为题目保证至少存在一个 $1\sim 9$ 的数字，所以降序输出时第一位一定非零，不会出现前导零问题。数据范围到 $10^6$，直接计数 $0\sim 9$ 再输出，比把所有数字收集后排序更稳。

### 核心代码

```cpp
int cnt[10] = {};
for(char ch : s) if(isdigit(ch)) cnt[ch - '0']++;
string ans;
for(int d = 9; d >= 0; d--)
  ans.append(cnt[d], char('0' + d));
cout << ans;
```

### 复杂度

时间复杂度 $O(|s|)$，空间复杂度 $O(1)$。

## 2. [【25CSPJ普及组】座位](http://ybt.ssoier.cn:8088/problem_show.php?pid=2130)

`CSP 2025` `CSP-J`

### 题意

把考场座位按列蛇形编号：第 $1$ 列从上到下，第 $2$ 列从下到上，如此交替。已知所有人成绩互不相同，且会按成绩从高到低依次入座，要求求出小 R 的列号和行号。

### 分析

题目表面在描述一个二维摆放过程，但真正决定座位的只有“小 R 的成绩排第几”。因为成绩互不相同，只要统计有多少人的成绩严格大于 $a_1$，就能得到小 R 的排名 $rk$。

排名转成坐标时，按每列容纳 $n$ 个人处理即可。列号是 $c=\lfloor (rk-1)/n \rfloor+1$，列内偏移是 $t=(rk-1)\bmod n$。如果这一列是奇数列，就从上往下编号，行号是 $t+1$；如果是偶数列，就从下往上编号，行号是 $n-t$。题目给的蛇形规则，其实就被这个公式一次性吃掉了。

### 核心代码

```cpp
int rk = 1;
for(int i = 2; i <= n * m; i++)
  if(a[i] > a[1]) rk++;
int c = (rk - 1) / n + 1;
int t = (rk - 1) % n;
int r = (c & 1) ? t + 1 : n - t;
```

### 复杂度

时间复杂度 $O(nm)$，空间复杂度 $O(1)$。

## 3. [【25CSPJ普及组】异或和](http://ybt.ssoier.cn:8088/problem_show.php?pid=2131)

`CSP 2025` `CSP-J`

### 题意

给定序列和目标值 $k$，要从中选出尽可能多的两两不相交区间，使得每个区间的异或和都等于 $k$。输出最多能选多少段。

### 分析

“区间异或和”几乎都会把人带到前缀异或。设 $pre_i=a_1\oplus a_2\oplus\cdots\oplus a_i$，那么区间 $[l,r]$ 的异或和等于 $k$，当且仅当 $pre_r\oplus pre_{l-1}=k$，也就是 $pre_{l-1}=pre_r\oplus k$。这样一来，若我们想让某一段恰好以 $r$ 结尾，只需要知道前面哪个前缀值等于 $pre_r\oplus k$。

真正的难点在“不相交”。这里最自然的状态是 $dp[i]$：只看前 $i$ 个数时，最多能选多少段。转移有两种：不把第 $i$ 个位置当作新区间结尾，那么 $dp[i]=dp[i-1]$；或者选一段 $[j+1,i]$，这时贡献是 $dp[j]+1$。于是我们只需要对每种前缀异或值维护“出现过它时最大的 $dp[j]$”，扫描一遍就能完成转移。这个维护本身就把“不相交”编码进去了，因为被拿来转移的永远是某个前缀位置之前的最优答案。

### 核心代码

```cpp
unordered_map<int,int> best;
best[0] = 0;
for(int i = 1, pre = 0; i <= n; i++){
  pre ^= a[i];
  dp[i] = dp[i - 1];
  if(best.count(pre ^ k)) dp[i] = max(dp[i], best[pre ^ k] + 1);
  best[pre] = max(best[pre], dp[i]);
}
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 4. [【25CSPJ普及组】多边形](http://ybt.ssoier.cn:8088/problem_show.php?pid=2132)

`CSP 2025` `CSP-J`

### 题意

从 $n$ 根木棍里选出若干根，要求所选长度能拼成一个多边形。题目已经给出了判定条件：若最大长度为 $M$，总和为 $S$，则当且仅当所选木棍数不少于 $3$ 且 $S>2M$。现在要求统计所有合法选法。

### 分析

把木棍从小到大排序后，考虑把第 $i$ 根木棍当作所选集合里“最靠右”的最大值。这样每一种方案只会被计算一次，即由它的最大下标负责。设这根木棍长度是 $a_i$，其余木棍来自前 $i-1$ 根。判定条件 $S>2M$ 就化成了“其余木棍长度和大于 $a_i$”。

于是问题变成：前 $i-1$ 根木棍的子集中，有多少个子集的和大于 $a_i$？总子集数是 $2^{i-1}$，所以只要减掉“和不超过 $a_i$ 的子集数”即可。这里有个很关键的小观察：由于前面的木棍长度都不超过 $a_i$，若一个子集的和已经严格大于 $a_i$，它不可能只选 $0$ 根或 $1$ 根木棍，因此“至少三条边”这个限制已经自动满足了。

而 $a_i\le 5000$ 给了我们一个很舒服的容量上界：只需做一维背包，维护前缀木棍凑出和为 $s$ 的方案数。查询时求出 $\sum_{s=0}^{a_i}dp[s]$ 就是坏方案数，再把当前木棍加入背包即可。因为以后只会问到不超过 $5000$ 的阈值，所以更大的和根本不用存。

### 核心代码

```cpp
sort(a + 1, a + n + 1);
dp[0] = 1;
for(int i = 1; i <= n; i++){
  long long bad = 0;
  for(int s = 0; s <= a[i]; s++) bad = (bad + dp[s]) % MOD;
  ans = (ans + pw2[i - 1] - bad + MOD) % MOD;
  for(int s = 5000; s >= a[i]; s--) dp[s] = (dp[s] + dp[s - a[i]]) % MOD;
}
```

### 复杂度

时间复杂度 $O(nV)$，其中 $V=5000$；空间复杂度 $O(V)$。

# 二、CSP-S 赛题

从这里开始，题目明显更强调建模的转身：贪心往往只是起点，真正拿分的关键，是看出哪里会失效、哪些信息必须保留，以及怎样把巨大的状态压成还能转移的骨架。

## 5. [【25CSPS提高组】社团招新](http://ybt.ssoier.cn:8088/problem_show.php?pid=2133)

`CSP 2025` `CSP-S`

### 题意

每个新成员都要被分到三个部门之一，满意度按所去部门决定。要求总满意度最大，并且任何一个部门的人数都不能超过 $n/2$。

### 分析

先把约束忘掉，只看总满意度，显然每个人都应该去自己最喜欢的部门。设这样得到的总和是 `base`。如果三个部门人数都没有超过 $n/2$，答案已经结束。真正的重点在于：由于总人数就是 $n$，所以不可能有两个部门同时超过 $n/2$，也就是说至多只有一个“超载部门”需要修。

假设部门 $x$ 超了。那另外两个部门最终能再接收多少人，其实是固定的：如果当前人数分别是 $cnt_y,cnt_z$，那么必须恰好再塞入 $need_y=n/2-cnt_y$ 和 $need_z=n/2-cnt_z$ 个人。于是问题立刻降成一个经典的“最小损失调整”：对所有当前在 $x$ 的人，记改去 $y$ 的损失为 $loss_y$，改去 $z$ 的损失为 $loss_z$。先假装他们全去 $z$，再从中选出 $need_y$ 个人改去 $y$，新增损失就是 $loss_y-loss_z$。把这批差值最小的挑出来即可。

这个写法的好处在于，它把三分类约束压成了一次排序。很多人会担心“最初给某个人选哪个最大值部门”会不会影响答案，但如果存在并列最大值，被你后续调走时损失就是 $0$，所以这个贪心修正依然成立。

### 核心代码

```cpp
for(int i = 1; i <= n; i++){
  int x = bestDept(i), y = (x + 1) % 3, z = (x + 2) % 3;
  base += a[i][x], cnt[x]++;
  people[x].push_back({a[i][x] - a[i][y], a[i][x] - a[i][z]});
}
int x = overloaded();
int y = (x + 1) % 3, need = n / 2 - cnt[y];
long long loss = 0; vector<int> diff;
for(auto [ly, lz] : people[x]) loss += lz, diff.push_back(ly - lz);
nth_element(diff.begin(), diff.begin() + need, diff.end());
for(int i = 0; i < need; i++) loss += diff[i];
cout << base - loss;
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(n)$。

## 6. [【25CSPS提高组】道路修复](http://ybt.ssoier.cn:8088/problem_show.php?pid=2134)

`CSP 2025` `CSP-S`

### 题意

原来的城市之间有 $m$ 条双向道路可以修复，另外还有至多 $k\le 10$ 个乡镇可以选择城市化。若城市化某个乡镇，需要先付改造费，再决定是否把它和哪些原城市连边。目标是让原有的 $n$ 座城市两两连通，总代价最小。

### 分析

看到 $k\le 10$，第一反应一定是状压枚举哪些乡镇被改造。可如果每个状态都把原图的 $m$ 条边重新丢进 Kruskal，复杂度会被 $10^6$ 级别的边数直接压垮。要想继续做，必须先把原图压瘦。

关键观察是：在只看原有城市时，原图里真正可能有用的道路，其实只需要保留原图最小生成树上的 $n-1$ 条边。原因是其余非树边在原城市之间已经被一条不更贵的树上路径替代，就算后来又加入了若干乡镇，这些边也不会突然比树边更有竞争力。于是我们先对原图做一次 Kruskal，把这 $n-1$ 条边抽出来。

之后每个乡镇只会带来一组“乡镇到所有城市”的星形边，数量是 $n$。因为每组边、以及原树边都可以预先按权值排好序，所以枚举一个乡镇子集时，不必重新排序，只要把这些有序表归并起来，再做一次 Kruskal 即可。这样总复杂度就落在 $2^k\cdot kn$ 这一层，可以稳稳吃下。

### 核心代码

```cpp
sort(roads.begin(), roads.end());
vector<Edge> base = kruskal_keep_mst_edges(roads, n);
for(int j = 0; j < k; j++) sort(star[j].begin(), star[j].end());
for(int mask = 0; mask < (1 << k); mask++){
  auto cur = base; long long cost = 0;
  for(int j = 0; j < k; j++) if(mask >> j & 1){
    cost += c[j];
    cur = merge_sorted(cur, star[j]);
  }
  ans = min(ans, cost + kruskal(cur, n + __builtin_popcount(mask)));
}
```

### 复杂度

时间复杂度 $O(m\log m+kn\log n+2^kkn\alpha(n))$，空间复杂度 $O(kn)$。

## 7. [【25CSPS提高组】谐音替换](http://ybt.ssoier.cn:8088/problem_show.php?pid=2135)

`CSP 2025` `CSP-S`

### 题意

给出很多等长替换规则 $(s_{i,1},s_{i,2})$，每次询问给两个不同字符串 $(t_1,t_2)$，问有多少种“选一个位置、套一条规则、只替换一次”的做法，能把 $t_1$ 变成 $t_2$。

### 分析

一次替换最本质的限制，是两串的差异必须被一个连续区间完整包住。对询问串先找出第一处和最后一处不同的位置 $[l,r]$；对每条规则也做同样的事情，得到它自己的差异核心 $[l',r']$。如果一条规则真能在某个询问里生效，那么这两对“核心子串”必须完全相同：规则左串在核心里的内容等于 $t_1[l,r]$，规则右串在核心里的内容等于 $t_2[l,r]$。这一步先把海量字符串对按“本质变化”分组。

分组之后，剩下的条件就不再是字符串整体相等，而是上下文可扩展：规则左侧保留下来的公共前缀，必须是询问公共前缀的后缀；规则右侧保留下来的公共后缀，必须是询问公共后缀的前缀。把左侧字符串反过来看，问题就变成了“前缀匹配 + 前缀匹配”的二维计数。

因此同一组内可以把规则映射成一个点：横坐标是反转后左侧串在 Trie 上的子树区间，纵坐标是右侧串在另一棵 Trie 上的子树区间；询问则对应一个矩形计数。实现时常见写法是：两棵 Trie 求出 dfn 区间，再用离线扫描线配树状数组做二维数点。这样总复杂度基本跟字符串总长同阶。

### 核心代码

```cpp
auto [key, L, R] = split_core(s1, s2);
rules[key].push_back({rev(L), R});
...
for(auto &[key, grp] : rules){
  build_left_trie(grp); build_right_trie(grp);
  for(auto &x : grp) add_point(tinL[x.u], tinR[x.v]);
  for(auto &q : asks[key])
    ans[q.id] = rect_query(subL[q.u], subR[q.v]);
}
```

### 复杂度

时间复杂度 $O(L\log L)$ 或 $O(L)$（取决于组内实现），空间复杂度 $O(L)$，其中 $L$ 是总字符串长度。

## 8. [【25CSPS提高组】员工招聘](http://ybt.ssoier.cn:8088/problem_show.php?pid=2136)

`CSP 2025` `CSP-S`

### 题意

要给 $n$ 个人安排面试顺序。第 $i$ 天题目难度由串 $s$ 给出，若当天是简单题则来的人一定录用，否则一定不录用；而每个人有耐心上限 $c_i$，若在他之前已经有不少于 $c_i$ 个人被拒或放弃，他就直接不来。求最终录用人数至少为 $m$ 的排列数。

### 分析

这题最难的地方在于“人是谁”并不重要，真正有影响的是当前已经有多少人没有被录用。记这个数为 $j$。一个人的耐心只和 $j$ 比较，所以先按耐心值分桶：`cnt[x]` 表示有多少人满足 $c_i=x$。

接下来要做的是一种很典型的“贡献延迟”。如果我们直接在 DP 里记录已经用了哪些耐心值，状态会炸；但当 $j$ 固定时，真正关键的是还有多少个已经被安排到前缀、且满足 $c_i>j$ 的人没有被结算。于是设 `dp[i][j][k]` 表示处理了前 $i$ 天、当前已有 $j$ 人未录用、并且有 $k$ 个“耐心仍然大于 $j$ 的待结算人”时的方案数。

当下一天是简单题时，未录用人数不变，只能从仍愿意来的人里挑一个放进来；当下一天会让未录用人数增为 $j+1$ 时，所有耐心恰好等于 $j+1$ 的人都会在这一层被迫结算，因此需要枚举有多少个这类人藏在当前的 $k$ 里，再用组合数和阶乘补上“他们在之前那些空位里怎么排”的方案数。看起来状态很抽象，但它本质上是在按阈值逐层清算。最终只要把未录用人数不超过 $n-m$ 的状态汇总即可。

### 核心代码

```cpp
for(int i = 0; i < n; i++) for(int j = 0; j <= i; j++) for(int k = 0; k <= i; k++){
  long long cur = dp[i][j][k]; if(!cur) continue;
  if(s[i + 1] == '1') add(dp[i + 1][j][k + 1], cur * alive(j, i, k) % MOD);
  for(int l = 0; l <= min(k, cnt[j + 1]); l++){
    long long ways = C[k][l] * C[cnt[j + 1]][l] % MOD * fac[l] % MOD;
    add(dp[i + 1][j + 1][k - l + 1], cur * ways % MOD * good(j, i, k, l) % MOD);
    add(dp[i + 1][j + 1][k - l], cur * ways % MOD * bad(j, i, k, l) % MOD);
  }
}
```

### 复杂度

时间复杂度 $O(n^3)$，空间复杂度 $O(n^2)$（滚动数组后）。

# 三、NOIP 赛题

这一章开始是 2025 年的 NOIP 提高组赛题。和前面的 CSP 相比，这几题更强调把抽象出来的模型继续往前推：有的题要把奇偶结构拆成可贪心的两部分，有的题要围绕“第一次失误”反向计数，有的题要抓住树上唯一传递链或把任意长度范围拆成少量可预处理的块。

## 9. [【25NOIP提高组】糖果店(candy)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2137)

`NOIP 2025` `NOIP 提高组`

### 题意

第 $i$ 种糖果买第奇数颗要花 $x_i$，买第偶数颗要花 $y_i$，循环交替。每种糖果无限颗，给定预算 $m$，求最多能买多少颗。

### 分析

先把同一种糖果买若干颗的费用写开：买 $2t$ 颗要花 $t(x_i+y_i)$，买 $2t+1$ 颗要花 $t(x_i+y_i)+x_i$。这说明一整个解法天然分成两部分：若干个“只买第一颗”的奇数贡献，以及大量“再补一对”的偶数贡献。

一旦这样拆，就能看出结构很硬：所有偶数部分都应该交给 $x_i+y_i$ 最小的那一种糖果去承担，因为每一对都是独立的两颗；而奇数部分则应该优先选择 $x_i$ 最小的若干种糖果，各买第一颗。于是只需要枚举最终有多少种糖果被买成奇数次，记为 $k$。

若我们已经决定买 $k$ 个单颗，那么最优选择一定是 $x_i$ 最小的前 $k$ 种，花费就是这些 $x_i$ 的前缀和；剩余预算全部砸到最便宜的“成对包”上即可。这个枚举没有遗漏，因为任何方案里，奇偶性的决定一旦确定，偶数部分和奇数部分都会各自走向贪心最优。

### 核心代码

```cpp
long long bestPair = INF;
for(int i = 1; i <= n; i++) bestPair = min(bestPair, x[i] + y[i]);
sort(x + 1, x + n + 1);
for(int k = 0; k <= n; k++){
  if(pre[k] > m) break;
  ans = max(ans, 1LL * k + 2 * ((m - pre[k]) / bestPair));
}
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(n)$。

## 10. [【25NOIP提高组】清仓甩卖(sale)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2138)

`NOIP 2025` `NOIP 提高组`

### 题意

每颗糖果只能定价成 $1$ 元或 $2$ 元。顾客会按性价比 $a_i/w_i$ 从大到小贪心购买，钱不够就跳过继续往后看。要求统计：有多少种定价方案能让这个贪心过程本身就是最优选择。

### 分析

正面数“好方案”很难，因为贪心何时正确不太好刻画；反过来数“坏方案”就清楚得多。把原价从大到小排序后，若贪心失效，一定能找到一个最早被跳过的高价值 $2$ 元糖果，以及后面某个已经买走的较低价值 $1$ 元糖果。更进一步，若后面还存在一颗 $1$ 元糖果，使两颗 $1$ 元糖果的总原价都不如那颗被跳过的 $2$ 元糖果，那么把这两颗换成那一颗，价值就会更大，贪心必错。

于是坏方案可以被一个“最早冲突对”唯一刻画：固定前面被跳过的 $2$ 元糖果位置 $i$，以及后面被买走的 $1$ 元糖果位置 $j$。在这两个位置确定后，前缀部分要恰好花到还剩 $1$ 元，这个条件会压成一个简单的组合数 $\binom{j-2}{m-i-1}$；而后缀里只要从某个最早位置 $z$ 开始随便定价，都能保住这次失败，于是再乘一个 $2^{n-z+1}$。

随着 $j$ 后移，满足 $a_j+a_z<a_i$ 的最早位置 $z$ 只会单调左移，所以可以用双指针把三重枚举压成 $O(n^2)$。最后总答案就是 $2^n$ 减去所有坏方案数。这个题最该记住的不是公式，而是“用唯一的第一次失误去反向计数”。

### 核心代码

```cpp
sort(a + 1, a + n + 1, greater<int>());
long long ans = pw2[n];
for(int i = 1; i <= n; i++){
  int z = n + 1;
  for(int j = i + 1; j <= n && 2 * a[j] > a[i]; j++){
    while(z > 1 && a[z - 1] + a[j] < a[i]) --z;
    if(0 <= m - i - 1 && m - i - 1 <= j - 2)
      ans = (ans - C[j - 2][m - i - 1] * pw2[n - z + 1]) % MOD;
  }
}
```

### 复杂度

时间复杂度 $O(n^2)$，空间复杂度 $O(n^2)$（若预处理组合数）。

## 11. [【25NOIP提高组】树的价值(tree)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2139)

`NOIP 2025` `NOIP 提高组`

### 题意

在一棵高度不超过 $m$ 的有根树上给每个点赋非负整数，定义每个结点价值为其子树权值集合的 $mex$，要求最大化全树价值和。

### 分析

想做这题，第一步必须先接受一个事实：对某个结点来说，子树里那些大于当前 $mex$ 的值暂时没有直接作用，它们只是以后还能拿去给祖先补连续段的“垫脚石”。所以一个结点想把自己的 $mex$ 继续做大，最合理的方式不是同时继承多个儿子的结构，而是沿着某一个儿子已经形成的连续段往上延伸，再把别处攒出来的垫脚石塞进缺口里。

这就引出了“传递边”的观点：每个非叶子结点只会选一个儿子，把那条连续链继续往上送。所有传递边合起来是一组链剖分。对任意一个垫脚石来说，它被某个祖先利用后，能贡献几次，恰好等于它往根走时所经过的最长传递链长度。因此当我们按链长做 DP 时，不必关心具体给哪些点填了什么数，只要关心“这条链还能向上延伸多长”。

官方做法把状态压成两类：`g(u,x)` 表示一条长度为 $x$ 的有效链经过 $u$ 并继续向上；`f(u,x)` 表示在 $u$ 子树里，面向祖先时能提供的最佳总收益。外层按 $x$ 从大到小枚举，再用长链剖分复用深度数组，把“找距离恰好为 $x-1$ 的后代里最优者”压成线性转移，最终做到 $O(nm)$。这题最难的是状态设计，一旦接受“只有一条链在传递”，后面的式子就顺了。

### 核心代码

```cpp
for(int x = m + 1; x >= 1; x--){
  swap_layer(g);
  dfs2(1, x);
}
void dfs2(int u, int x){
  f[u] = g[u][0] = 1LL * x * siz[u];
  for(int v : son[u]) dfs2(v, x), sum += f[v];
  for(int v : son[u]) g[u][0] = max(g[u][0], g[v][1] + x + sum - f[v]);
  sh[u][1] = g[u][0] - delta[u];
  if(hgt[u] >= x) f[u] = max(f[u], sh[u][x] + delta[u] + 1LL * x * (x - 1));
}
```

### 复杂度

时间复杂度 $O(nm)$，空间复杂度 $O(n)$ 到 $O(nm)$（取决于实现压缩程度）。

## 12. [【25NOIP提高组】传统型(query)](http://ybt.ssoier.cn:8088/problem_show.php?pid=2140)

`NOIP 2025` `NOIP 提高组`

### 题意

每次询问给一个长度范围 $[L,R]$。对每个位置 $i$，要找所有包含 $i$、且区间长度落在 $[L,R]$ 内的子数组里，区间和最大的那个，再把这些答案按题目要求异或起来。

### 分析

先固定一个长度范围 $[l,r]$，只想求“每个位置被某个合法区间覆盖时的最大区间和”。用前缀和写区间和后，若我们把右端点固定在 $p$，合法左端点对应的是一个滑动窗口，窗口里前缀和最小的位置就是最优左端点，所以可以用单调队列在线维护。把原数组和反转数组各跑一遍，就能分别得到“以某点结尾的最优区间”和“以某点开头的最优区间”。再做一次滑动窗口取最大，就能把这些值扩散成“覆盖每个位置的最优值”。

当 $2L\ge R$ 时，合法长度区间比较窄，一次 `Solve(L,R)` 就够了。难的是一般情形。这里的经典技巧是把长度按倍增块拆开，例如 $[2^b-1,2^{b+1}-2]$ 为一块。对每一对完整块区间预处理一个答案数组 `Bans[p][q]`，表示长度落在这些完整块里的最优覆盖值。

这样一来，一次询问最多只剩“左边残块 + 若干完整块 + 右边残块”三部分：两端残块各调用一次 `Solve`，中间完整块直接取预处理结果，最后按位置取最大即可。核心思想其实很统一：单调队列负责解决一个连续长度段，倍增分块负责把任意长度范围拆成少量连续长度段。

### 核心代码

```cpp
Solve(L, min(R, br[bel[L]]));
Solve(max(L, bl[bel[R]]), R);
if(bel[L] + 1 <= bel[R] - 1)
  for(int i = 1; i <= n; i++)
    ans[i] = max(ans[i], Bans[bel[L] + 1][bel[R] - 1][i]);
unsigned long long out = 0;
for(int i = 1; i <= n; i++)
  out ^= (unsigned long long)i * (unsigned long long)ans[i];
```

### 复杂度

预处理时间复杂度 $O(n\log^2 n)$，单次询问复杂度 $O(n)$，空间复杂度 $O(n\log^2 n)$。
