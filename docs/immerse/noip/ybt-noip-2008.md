# 一本通 NOIP 2008赛题解题报告

这一组题从普及组的公式模拟、局部计数和图形输出，一路走到提高组的枚举、动态规划与结构判定。前半段重在把题面规则准确翻译成过程，后半段更强调把“来回路径、资源分配、栈操作限制”压成可维护的状态或不变量；只要状态立住，答案往往就会顺着转移自然浮出来。

# 一、普及组赛题

这一部分更像是在练“把规则看清楚”。有的题是把校验公式直接落成程序，有的题是把局部影响拆开计数，也有题目要求把几何外观还原成可覆盖的字符画；共同点都是先把题面里的每一条限制翻译成清楚的过程。

## 1. [【08NOIP普及组】ISBN号码](http://ybt.ssoier.cn:8088/problem_show.php?pid=1942)
`NOIP 2008` `普及组`

### 题意

给定一个形如 `x-xxx-xxxxx-x` 的 ISBN 串，前面 9 个数字参与校验，权值依次是 `1` 到 `9`。把这 9 个数字按顺序乘权求和后对 `11` 取模，得到识别码；若余数是 `10`，识别码写成大写字母 `X`。

题目要求判断输入中的最后一位识别码是否正确。若正确就输出 `Right`，否则输出修正后的完整 ISBN 串，连同原有的分隔符一起保留。

### 分析

这题最关键的不是技巧，而是把“哪些字符参与计算”看准：横杠只是分隔符，真正参与加权和的只有前面 9 个数字。于是我们一边扫描字符串，一边跳过横杠，把读到的数字依次乘上当前位置权值即可。

算出余数后，再把它翻译回题目要求的字符。这里有一个唯一需要单独处理的分支：当余数等于 `10` 时，答案不是数字字符，而是 `X`。这就是题目的核心不变量：**前 9 位数字唯一决定最后一位识别码**。

答案提取也很直接。若算出的识别码与原串最后一位相同，就说明原 ISBN 完全合法；否则只需要把最后一位替换成正确识别码，前面的所有字符原样输出即可。

### 核心代码

```cpp
int sum = 0, k = 0;
for(char ch : s){
    if(ch == '-') continue;
    if(k == 9) break;
    sum += (ch - '0') * (++k);
}
char chk = sum % 11 == 10 ? 'X' : char('0' + sum % 11);
if(chk == s.back()) cout << "Right";
else{
    s.back() = chk;
    cout << s;
}
```

### 复杂度

时间复杂度 $O(|s|)$，空间复杂度 $O(1)$。

## 2. [【08NOIP普及组】排座椅](http://ybt.ssoier.cn:8088/problem_show.php?pid=1943)
`NOIP 2008` `普及组`

### 题意

教室是一个 `M` 行 `N` 列的座位网格，现在已知有 `D` 对会交头接耳的学生，而且每一对都只发生在上下相邻或左右相邻的位置之间。老师准备设置 `K` 条横向通道和 `L` 条纵向通道，只要一条通道正好隔在这对学生中间，他们就不会再交头接耳。

要求给出唯一的最优方案：应该在哪些相邻行之间开横向通道、在哪些相邻列之间开纵向通道，使剩余会交头接耳的对数最少。

### 分析

看到“整体最优”时，先别急着想复杂搜索。题目里每一对学生只会被一条横向边界或一条纵向边界影响：若两人上下相邻，那么只有他们中间那条“行间边界”能隔开他们；若两人左右相邻，那么只有对应那条“列间边界”有作用。

这就把原问题拆成了两个完全独立的子问题。对于每条行间边界，统计它能隔开的上下相邻耳语对数；对于每条列间边界，统计它能隔开的左右相邻耳语对数。因为横向通道只影响前者，纵向通道只影响后者，所以最佳方案就是分别选出贡献最大的 `K` 条行边界和 `L` 条列边界。

为什么这样选一定对？因为每一对学生至多被一条候选边界覆盖，不存在“两条通道一起配合才多赚一点”的联动收益，所以每条边界的贡献可以独立累加。最后把选中的边界编号升序输出，就是题目要求的答案提取方式。

### 核心代码

```cpp
for(int t = 0; t < D; t++){
    cin >> x >> y >> p >> q;
    if(x != p) row[min(x, p)]++;
    else col[min(y, q)]++;
}
auto pick = [&](vector<int> cnt, int need){
    vector<int> id(cnt.size() - 1);
    iota(id.begin(), id.end(), 1);
    sort(id.begin(), id.end(), [&](int a, int b){ return cnt[a] > cnt[b]; });
    id.resize(need);
    sort(id.begin(), id.end());
    return id;
};
```

### 复杂度

时间复杂度 $O(D + M\log M + N\log N)$，空间复杂度 $O(M + N)$。

## 3. [【08NOIP普及组】传球游戏](http://ybt.ssoier.cn:8088/problem_show.php?pid=1944)
`NOIP 2008` `普及组`

### 题意

`n` 个同学围成一个圆圈，球从小蛮手里出发。每次传球时，拿球的人只能把球传给左边或右边相邻的同学。问恰好传了 `m` 次之后，球重新回到小蛮手里的不同传球方案数。

两种方案只要“每一步接到球的人序列”不同，就算不同方案，因此这是一个标准的按步数计数问题，而不是只问能否回到起点。

### 分析

题目同时给出了“阶段数”与“当前位置”，这正是动态规划最自然的信号。设 `f[t][i]` 表示传了 `t` 次后，球在第 `i` 个同学手中的方案数，那么从某个位置出发，下一次只会走到左右两个邻居，于是状态转移只和相邻位置有关。

由于队形是圆圈，左右邻居要写成环形下标：编号 `1` 的左边是 `n`，编号 `n` 的右边是 `1`。于是有 `f[t][i] = f[t-1][left(i)] + f[t-1][right(i)]`。这个转移为什么成立？因为最后一步传到 `i` 时，前一时刻一定在它的两个邻居之一，而且两种来源没有重叠，直接相加即可。

初始状态是 `f[0][1] = 1`，表示还没传球时，球就在小蛮手里。题目最后问“传了 `m` 次又回到小蛮”，所以答案提取就是 `f[m][1]`。

### 核心代码

```cpp
f[0][1] = 1;
for(int t = 1; t <= m; t++){
    for(int i = 1; i <= n; i++){
        int l = (i == 1 ? n : i - 1);
        int r = (i == n ? 1 : i + 1);
        f[t][i] = f[t - 1][l] + f[t - 1][r];
    }
}
cout << f[m][1];
```

### 复杂度

时间复杂度 $O(nm)$，空间复杂度 $O(nm)$；若滚动数组优化，可降到 $O(n)$。

## 4. [【08NOIP普及组】立体图](http://ybt.ssoier.cn:8088/problem_show.php?pid=1977)
`NOIP 2008` `普及组`

### 题意

给出一个 `m \times n` 的网格，每个格子里堆着若干个单位积木。每个积木的画法固定，不能旋转；多个积木在左右、前后、上下相邻时，要按照题目给出的透视规则拼接成一张完整字符画，空白处用 `.` 填充。

要求输出包含整幅立体图的最小字符矩阵。这意味着不仅要会画单个积木，还要把所有积木放到统一坐标系里，并处理前后遮挡后的最终覆盖结果。

### 分析

这题最容易乱在“看图写图”，真正该先做的是建立坐标。把单个积木视为固定模板，再给每个小方块分配一个左下角基准点：同一格里往上多一层，高度方向就整体上移 `3` 行；网格里往右一列，图形向右平移 `4` 列；往后一个格子，则同时上移 `2` 行、右移 `2` 列。这样每个小方块都能落到统一画布上。

接下来要想清楚覆盖顺序。远处和底层的方块应该先画，近处和前面的方块后画，因为字符画本质上就是“后画的字符覆盖先画的字符”。因此按“从后到前、从左到右、从下到上”的顺序把模板盖到画布上，就能保证最后留下的是可见部分。

整个过程里的关键不变量是：**画布上每个位置始终保存当前最前面那块积木在该点应显示的字符**。所有积木都盖完后，直接把整张画布输出，就是答案提取；由于画布初值全是 `.`，空白背景也自动处理好了。

### 核心代码

```cpp
vector<string> g(H, string(W, '.'));
auto draw = [&](int x, int y){
    static const vector<string> s = {
        "  +---+", " /   /|", "+---+ |",
        "|   | +", "|   |/ ", "+---+  "
    };
    for(int i = 0; i < 6; i++)
        for(int j = 0; j < 7; j++)
            if(s[i][j] != ' ') g[H - 1 - (y + 5 - i)][x + j] = s[i][j];
};
for(int i = 1; i <= m; i++)
    for(int j = 1; j <= n; j++)
        for(int k = 0; k < a[i][j]; k++)
            draw(2 * (m - i) + 4 * (j - 1), 2 * (m - i) + 3 * k);
```

### 复杂度

设积木总数为 $S$，时间复杂度 $O(S)$，空间复杂度 $O(HW)$，其中 $H,W$ 是输出画布大小。

# 二、提高组赛题

这一部分开始明显要求“先建模再实现”。有的题要把方程可行性压成枚举条件，有的题要把来回两条路改写成同步行走的状态，还有的题需要先证明某些元素绝不能落在同一栈里；核心已经从规则照搬，转向状态与结构的设计。

## 5. [【08NOIP提高组】笨小猴](http://ybt.ssoier.cn:8088/problem_show.php?pid=1851)
`NOIP 2008` `提高组`

### 题意

给定一个只含小写字母的单词，统计每个出现过的字母出现次数。设最大出现次数是 `maxn`，最小正出现次数是 `minn`，若 `maxn-minn` 是质数，则这是一个 `Lucky Word`；否则输出 `No Answer`，并在第二行输出 `0`。

换句话说，题目只关心“出现过的字母”之间的频次差值，不关心没有出现的字母，因此最小值不能把频次为 `0` 的字母算进去。

### 分析

这题的信息量很集中：只有 26 个小写字母，所以先做频次数组统计就是最自然的第一步。随后在线性扫描里维护两个量：出现次数的最大值，以及所有正频次中的最小值。

真正容易出错的点有两个。第一，`minn` 只能在 `cnt[i] > 0` 时更新，否则没出现的字母会把最小值错误压成 `0`；第二，质数判定时要记得 `0` 和 `1` 都不是质数。把这两个边角处理好，题目就退化成了一个非常直接的分类讨论。

为什么答案就是 `maxn-minn`？因为题目对幸运单词的定义完全建立在这个差值是否为质数上。于是算出差值后，若它是质数就原样输出；否则按照题意固定输出 `No Answer` 和 `0`。

### 核心代码

```cpp
for(char ch : s) cnt[ch - 'a']++;
int mx = 0, mn = 1e9;
for(int x : cnt) if(x){
    mx = max(mx, x);
    mn = min(mn, x);
}
int d = mx - mn;
auto isPrime = [&](int x){
    if(x < 2) return false;
    for(int i = 2; i * i <= x; i++) if(x % i == 0) return false;
    return true;
};
cout << (isPrime(d) ? "Lucky Word\n" : "No Answer\n");
cout << (isPrime(d) ? d : 0);
```

### 复杂度

时间复杂度 $O(|s| + \sqrt d)$，空间复杂度 $O(1)$。

## 6. [【08NOIP提高组】火柴棒等式](http://ybt.ssoier.cn:8088/problem_show.php?pid=1852)
`NOIP 2008` `提高组`

### 题意

给定 `n` 根火柴棒，要求统计有多少个形如 `A+B=C` 的等式可以恰好用完所有火柴。数字 `0` 到 `9` 各自消耗的火柴数固定，加号和等号各消耗 `2` 根；若 `A \ne B`，则 `A+B=C` 与 `B+A=C` 视作不同答案。

题目要求的是“方案总数”，因此只要一个三元组满足火柴数恰好用完，就应该计入答案，不需要真的输出这些等式。

### 分析

先把最固定的部分拿掉：加号和等号总共一定消耗 `4` 根，所以真正分配给 `A、B、C` 三个整数的只有 `n-4` 根。接着问题就变成：有多少对有序整数 `(A,B)` 满足 `cost(A)+cost(B)+cost(A+B)=n-4`。

这里的关键是把“一个整数要多少根火柴”预处理出来。设 `need[x]` 表示拼出整数 `x` 的火柴数，那么它可以由十进制各位的代价累加得到，特别地 `0` 要单独定义成 `6` 根。于是枚举 `A` 和 `B` 时，`C` 被唯一确定为 `A+B`，检查三者代价之和是否匹配即可。

为什么上界可以直接枚举到四位数？因为五位数至少要 `10` 根火柴，而等式另一边还至少需要再摆出一个数和结果，总火柴数不够。这样搜索空间就被压到了可接受范围内。最后把所有满足条件的有序对计数，得到的就是答案提取结果。

### 核心代码

```cpp
int seg[10] = {6,2,5,5,4,5,6,3,7,6};
auto calc = [&](int x){
    if(x == 0) return 6;
    int s = 0;
    while(x) s += seg[x % 10], x /= 10;
    return s;
};
for(int i = 0; i <= 19998; i++) need[i] = calc(i);
int rest = n - 4, ans = 0;
for(int a = 0; a <= 9999; a++)
    for(int b = 0; b <= 9999; b++)
        if(need[a] + need[b] + need[a + b] == rest)
            ans++;
cout << ans;
```

### 复杂度

时间复杂度 $O(U^2)$，空间复杂度 $O(U)$，其中 $U=10^4$ 量级。

## 7. [【08NOIP提高组】传纸条](http://ybt.ssoier.cn:8088/problem_show.php?pid=1853)
`NOIP 2008` `提高组`

### 题意

在一个 `m \times n` 的网格里，小渊要把纸条从左上角传到右下角，只能向下或向右；小轩回信时从右下角回到左上角，只能向上或向左。除起点和终点外，每位同学最多帮忙一次，每个格子还有一个好心值，要求让来回两条路径经过的好心值总和最大。

把回程反过来看，就等价于有两个人同时从左上角走到右下角，每次都只向下或向右，且除端点外不能重复经过同一个格子。

### 分析

这题的突破口就是上面的“反向等价”。如果直接分别求两条路，会遇到“第一条路怎么影响第二条路”的耦合；但把回程翻转后，两个人每走一步，总步数始终相同，于是可以把“已经走了多少步”当成第一维状态。

设 `f[k][i][p]` 表示两个人都走了 `k` 步后，第一个人在 `(i, k-i)`，第二个人在 `(p, k-p)` 时能取得的最大好心值。这样一来，列坐标不需要单独存，因为它被总步数唯一决定。每个人上一步只可能来自“上方”或“左方”，所以总共有四种前驱组合，取最大值即可。

状态转移时最关键的不变量是：**同一时刻两个人代表的是两条前缀路径，它们除了允许在起点和终点重合外，其他格子不能重复计分。** 所以若当前位置不同，就把两个格子的权值都加上；若相同，只加一次。最后当两个人都走到 `(m,n)` 时，也就是 `k=m+n` 的终态，这个状态值就是答案。

### 核心代码

```cpp
memset(f, -0x3f, sizeof(f));
f[2][1][1] = 0;
for(int k = 3; k <= m + n; k++)
    for(int i = 1; i <= m; i++){
        int j = k - i;
        if(j < 1 || j > n) continue;
        for(int p = 1; p <= m; p++){
            int q = k - p;
            if(q < 1 || q > n) continue;
            int add = a[i][j] + (i == p && j == q ? 0 : a[p][q]);
            f[k][i][p] = max({f[k-1][i][p], f[k-1][i-1][p],
                              f[k-1][i][p-1], f[k-1][i-1][p-1]}) + add;
        }
    }
cout << f[m + n][m][m];
```

### 复杂度

时间复杂度 $O((m+n)m^2)$，空间复杂度 $O((m+n)m^2)$；滚动数组可把空间压到 $O(m^2)$。

## 8. [【08NOIP提高组】双栈排序](http://ybt.ssoier.cn:8088/problem_show.php?pid=1854)
`NOIP 2008` `提高组`

### 题意

给定一个 `1` 到 `n` 的排列，元素会依次从输入端出现。每个元素到来时，只能压入栈 `S1` 或栈 `S2`；之后可以在合适的时候从两个栈顶弹出，输出目标序列 `1,2,\dots,n`。如果这个排列无法借助两个栈完成升序输出，就输出 `0`；否则输出字典序最小的操作序列。

这里的难点不是单纯模拟，而是先判断“哪些元素绝不能进同一个栈”，再在所有可行操作中找字典序最小的一条。

### 分析

先想单个栈为什么会失败。若 `a[i]` 先入栈、`a[j]` 后入栈且 `i<j`，那么 `a[j]` 会压在 `a[i]` 上面；如果后面还存在一个更小的数 `x`，满足 `x<a[i]<a[j]`，那么想按升序输出时，`a[i]` 必须先于 `a[j]` 出栈，但它已经被 `a[j]` 卡住了，所以 `a[i]` 和 `a[j]` 不能放进同一个栈。

于是我们可以据此建冲突图：若存在 `k>j` 使得 `a[k]<a[i]<a[j]`，就给 `i,j` 连边。两种栈恰好对应两种颜色，所以原排列可双栈排序，当且仅当这张冲突图是二分图。这个判定本身就把“结构能否成立”说清楚了。

判定可行后，再做字典序最小的构造。颜色已经决定每个元素只能进哪一个栈，接下来只需按 `a,b,c,d` 的优先级贪心尝试当前合法操作：能压 `S1` 就优先压，接着优先弹 `S1`，再考虑 `S2` 的压入与弹出。过程中维持的不变量是两个栈顶都必须大于将要压入的元素，否则较小元素会被较大元素压住，未来就不可能按升序弹出。所有元素处理完后，得到的操作串就是答案提取结果。

### 核心代码

```cpp
for(int i = n; i >= 1; i--) mn[i] = min(mn[i + 1], a[i]);
for(int i = 1; i <= n; i++)
    for(int j = i + 1; j <= n; j++)
        if(mn[j + 1] < a[i] && a[i] < a[j]) addEdge(i, j);
if(!bipartite()) return cout << 0, void();
for(int i = 1, need = 1; i <= n || !s1.empty() || !s2.empty(); ){
    if(i <= n && col[i] == 0 && (s1.empty() || s1.top() > a[i])) s1.push(a[i++]), ans.push_back('a');
    else if(!s1.empty() && s1.top() == need) s1.pop(), need++, ans.push_back('b');
    else if(i <= n && col[i] == 1 && (s2.empty() || s2.top() > a[i])) s2.push(a[i++]), ans.push_back('c');
    else if(!s2.empty() && s2.top() == need) s2.pop(), need++, ans.push_back('d');
}
```

### 复杂度

时间复杂度 $O(n^2)$，空间复杂度 $O(n^2)$。
