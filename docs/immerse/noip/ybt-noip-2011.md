# 一本通 NOIP 2011赛题解题报告

这一组题从普及组的直接模拟与计数，一路走到提高组的搜索、组合、贪心和参数化分析。读的时候可以一直盯住同一件事：题面给了什么限制，就把它翻成什么状态；状态一旦站稳，转移、维护和最后的答案位置都会自然清楚。

# 一、普及组赛题

普及组这几题重在把规则直接落成过程：有的只要盯住数字和单词边界，有的要抓住“每轮结束后仍有序”这样的结构性质，有的则要把表达式改写成子表达式计数。难点不在技巧多，而在第一步建模是否准确。

## 1. [【11NOIP普及组】数字反转](http://ybt.ssoier.cn:8088/problem_show.php?pid=1953)
`NOIP 2011` `普及组`

### 题意

给定一个整数，把它的十进制数字顺序翻过来后输出。翻转后的结果仍要满足正常整数书写习惯，也就是除了原数本身就是 `0` 以外，答案最高位不能带前导零；如果原数是负数，负号要保留在最前面。

### 分析

这题最容易做复杂的地方，是把“负号”和“数位翻转”搅在一起。其实题面条件可以先拆成两个互不干扰的部分：符号单独记住，真正需要翻转的只是绝对值的十进制数位。这样题目马上就变成了一个逐位模拟。

把绝对值不断取模 `10`，就能从低位到高位依次弹出原数的数字；而这些被弹出的顺序，恰好就是答案从高位到低位的顺序。于是维护不变量：`ans` 始终等于“已经弹出的那一段数位翻转后组成的整数”。每次执行 `ans=ans*10+n%10`，就是把新弹出的一位接到答案末尾。循环结束后，`ans` 已经是绝对值部分的最终结果，再把一开始保存的符号乘回去就是答案；若原数是 `0`，直接输出 `0` 即可。

### 核心代码

```cpp
long long n,ans=0,sgn=1;
cin>>n;
if(n<0) sgn=-1,n=-n;
if(n==0) cout<<0;
while(n){
    ans=ans*10+n%10;
    n/=10;
}
if(ans) cout<<sgn*ans;
```

### 复杂度

时间复杂度 $O(\log_{10}|n|)$，空间复杂度 $O(1)$。

## 2. [【11NOIP普及组】统计单词数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1954)
`NOIP 2011` `普及组`

### 题意

输入一个目标单词和一整行文章，要求在忽略大小写的前提下，统计这个单词作为“独立单词”在文章里出现了多少次，并输出它第一次出现的位置。这里的“独立单词”很关键：只能和完整单词匹配，不能作为别的单词的一部分。

### 分析

题目的核心条件有两个：一是不区分大小写，二是必须按单词边界完整匹配。顺着这两个条件建模，最自然的办法就是先把目标单词和整篇文章都转成小写，然后按空格把文章切成一个个独立单词去检查。这样就不会把 `to` 和 `today` 混在一起。

扫描文章时，不变量是：当前处理区间 `[l,i)` 正好对应一个完整单词，`l` 记录这个单词在原文章中的起点。每遇到空格或字符串末尾，就说明一个单词边界已经闭合，可以把这段拿出来与目标比较。若匹配成功，就让计数加一；第一次成功时顺手记下 `l`，它就是题目要求的位置。整篇文章扫完后，若计数仍为 `0`，输出 `-1`；否则输出总次数和第一次出现的位置。

### 核心代码

```cpp
string w,s;
getline(cin,w),getline(cin,s);
for(char &c:w) c=tolower(c);
for(char &c:s) c=tolower(c);
int cnt=0,first=-1,n=s.size();
for(int i=0,l=0;i<=n;i++) if(i==n||s[i]==' '){
    if(s.substr(l,i-l)==w){ if(first==-1) first=l; cnt++; }
    l=i+1;
}
if(!cnt) cout<<-1; else cout<<cnt<<" "<<first;
```

### 复杂度

时间复杂度 $O(|s|)$，空间复杂度 $O(|w|)$。

## 3. [【11NOIP普及组】瑞士轮](http://ybt.ssoier.cn:8088/problem_show.php?pid=1955)
`NOIP 2011` `普及组`

### 题意

有 `2N` 名选手进行 `R` 轮比赛。每轮开始前都要按总分从高到低、编号从小到大排名，然后相邻两人对战，实力值大的必胜且获胜者加一分。问全部轮次结束后，排名第 `Q` 的选手编号是多少。

### 分析

如果每轮都把整张表重新排序，复杂度会落在 $O(RN\log N)$，大数据会吃紧。真正该抓的条件是“赛前已经有序，赛后配对方式固定为相邻两人”。对每一对选手来说，胜者分数加一，负者分数不变；把所有胜者按比赛顺序收集起来，会发现它们内部的相对顺序没有被破坏，所有败者也是同理。也就是说，一轮结束后我们天然得到两条有序序列。

这时题目就从“反复排序”变成了“反复归并”。状态是当前按排名排好的数组；转移是按相邻位置比赛，分别放进 `win` 和 `lose` 两个数组，再按“分数降序、编号升序”的规则归并回主数组。不变量是：每轮开始前主数组一定整体有序，因此相邻配对是合法的；每轮结束后归并又恢复整体有序，下一轮可以继续。最终所有轮次做完后，第 `Q` 个位置上的编号就是答案。

### 核心代码

```cpp
struct P{int id,s,w;} a[N],win[N],lose[N];
auto cmp=[&](P x,P y){ return x.s!=y.s?x.s>y.s:x.id<y.id; };
sort(a,a+2*n,cmp);
while(r--){
    for(int i=0,j=0,k=0;i<2*n;i+=2){
        P x=a[i],y=a[i+1];
        if(x.w>y.w) x.s++,win[j++]=x,lose[k++]=y;
        else y.s++,win[j++]=y,lose[k++]=x;
    }
    merge(win,win+n,lose,lose+n,a,cmp);
}
cout<<a[q-1].id;
```

### 复杂度

初始排序时间复杂度 $O(N\log N)$，每轮归并时间复杂度 $O(N)$，总时间复杂度 $O(N\log N+RN)$，空间复杂度 $O(N)$。

## 4. [【11NOIP普及组】表达式的值](http://ybt.ssoier.cn:8088/problem_show.php?pid=1956)
`NOIP 2011` `普及组`

### 题意

给出一个只含括号和运算符 `+`、`*` 的表达式骨架，每个空位都要填 `0` 或 `1`。其中 `+` 表示“或”，`*` 表示“与”，并且 `*` 的优先级高于 `+`。要求统计有多少种填法能让整个表达式的值为 `0`，结果对 `10007` 取模。

### 分析

题目真正问的不是某一种填法的值，而是“一个子表达式值为 `0` 或 `1` 各有多少种填法”。一旦这样看，模型就固定了：对每个已经解析出来的子表达式，维护一个状态对 `(f0,f1)`，分别表示它结果为 `0` 和结果为 `1` 的方案数。每个空位本身可以填 `0` 也可以填 `1`，所以初始状态就是 `(1,1)`。

接下来只剩下如何按优先级把子表达式正确合并。对 `+` 来说，只有 `0+0` 会得到 `0`；对 `*` 来说，只有 `1*1` 会得到 `1`。因此两个子状态的转移可以直接由运算表写出。括号和优先级则交给运算符栈维护：每次准备弹出一个运算符时，数栈顶部的两个状态一定正好对应它的左右子表达式，这就是整个算法的关键不变量。所有字符处理完后，数栈栈顶就是整棵表达式树的状态，答案直接取其中“结果为 `0` 的方案数”。

### 核心代码

```cpp
const int MOD=10007;
struct Node{int z,o;};
stack<char> op; stack<Node> st;
auto calc=[&](){
    auto b=st.top(); st.pop(); auto a=st.top(); st.pop();
    char c=op.top(); op.pop();
    if(c=='+') st.push({a.z*b.z%MOD,(a.z*b.o+a.o*b.z+a.o*b.o)%MOD});
    else st.push({(a.z*b.z+a.z*b.o+a.o*b.z)%MOD,a.o*b.o%MOD});
};
st.push({1,1});
for(char c:s){
    if(c=='(') op.push(c);
    else if(c==')'){ while(op.top()!='(') calc(); op.pop(); }
    else{ while(!op.empty()&&op.top()!='('&&(op.top()=='*'||c=='+')) calc(); op.push(c); st.push({1,1}); }
}
while(!op.empty()) calc();
cout<<st.top().z;
```

### 复杂度

时间复杂度 $O(L)$，空间复杂度 $O(L)$。

# 二、提高组赛题

提高组开始，题面规则不再只是照抄模拟，而是要进一步提炼结构：有时要抓住“最后铺上的一定在最上面”，有时要把“区间里存在便宜咖啡店”压成可维护前缀，有时还要从单调性或边际收益里做贪心。做题时最重要的是先判断，哪些信息真的需要维护，哪些信息只是干扰。

## 5. [【11NOIP提高组】铺地毯](http://ybt.ssoier.cn:8088/problem_show.php?pid=1863)
`NOIP 2011` `提高组`

### 题意

按编号从小到大依次铺 `n` 张轴对齐矩形地毯，后铺的会盖在前面的上面。最后给出一个点 `(x,y)`，要求判断它最上面覆盖的是哪一张地毯；如果没有任何地毯覆盖，输出 `-1`。

### 分析

这题最关键的建模动作，是把“最上面”直接翻译成“最后一个满足覆盖条件的编号”。因为铺设顺序已经给定，而且后铺必定压住先铺，所以根本不需要维护平面覆盖关系，也不需要离散化；谁最后出现，谁就更靠上。

于是状态几乎不需要保存，只需从编号 `n` 倒着扫到 `1`。扫描时的不变量是：凡是已经看过但没有命中的那些地毯，都在当前编号之上且不能覆盖查询点，因此它们不可能成为答案。只要第一次遇到一个包含 `(x,y)` 的矩形，这张地毯就是最上面的那一张，立即输出即可；若全部扫完仍未命中，答案就是 `-1`。

### 核心代码

```cpp
for(int i=n;i>=1;i--)
    if(a[i].x<=x&&x<=a[i].x+a[i].w&&a[i].y<=y&&y<=a[i].y+a[i].h){
        cout<<i;
        return;
    }
cout<<-1;
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 6. [【11NOIP提高组】选择客栈](http://ybt.ssoier.cn:8088/problem_show.php?pid=1864)
`NOIP 2011` `提高组`

### 题意

一排客栈各有颜色和咖啡店最低消费。需要统计有多少对不同位置的客栈满足：两家客栈颜色相同，并且在它们之间至少存在一家最低消费不超过 `p` 的咖啡店。

### 分析

题面里的难点不是“同色配对”，而是“区间里存在一家便宜咖啡店”这个条件。把它翻成线性扫描语言后，会发现只需要维护 `last`：表示到当前位置为止，最近一个最低消费不超过 `p` 的位置。对于当前右端点 `i`，若左端点 `j` 与它配对合法，那么区间 `[j,i]` 必须包含 `last`，等价于 `j<=last`。这样一个看似区间存在性的问题，就被压成了一个前缀条件。

于是我们让指针 `j` 把前缀 `[1,last]` 中的客栈按颜色加入计数桶 `cnt`。这里的不变量是：任意时刻 `cnt[c]` 恰好等于颜色为 `c` 且位置不超过 `last` 的客栈数。扫到第 `i` 家时，与它同色且能形成合法区间的左端点个数就是 `cnt[color[i]]`；如果 `i` 自己已经落在这个前缀里，还要减去自己这一项。把这一步对所有 `i` 累加，得到的就是最终答案。

### 核心代码

```cpp
for(int i=1,j=0;i<=n;i++){
    if(cost[i]<=p) last=i;
    while(j<last) ++cnt[color[++j]];
    ans+=cnt[color[i]]-(i<=last);
}
```

### 复杂度

时间复杂度 $O(n+k)$，空间复杂度 $O(k)$。

## 7. [【11NOIP提高组】Mayan 游戏](http://ybt.ssoier.cn:8088/problem_show.php?pid=1865)
`NOIP 2011` `提高组`

### 题意

在一个 `5` 列 `7` 行的棋盘上做至多 `n` 次横向移动，每次移动后都要立即触发“连续三个及以上同色块同时消除、上方方块下落、可能继续连消”的规则。要求找到一组字典序最小的操作序列，使得在规定步数内把棋盘清空；若无解输出 `-1`。

### 分析

题目的规模其实已经把方法写出来了：步数上限只有 `5`，真正困难的是每一步之后的局面演化非常复杂。因此模型不是宽搜最短路，而是“深度有限的 DFS + 精确局面模拟”。状态由当前棋盘和已经走了多少步组成，搜索树上的一次转移就是“移动一个块，然后把所有连消过程完整跑到稳定”。

稳定态是不变量里的关键。每次进入下一层 DFS 之前，棋盘都必须满足两件事：一是所有块已经下落到不能再落，二是当前局面中已经不存在可继续消除的三连。只有把这一整套物理过程都模拟干净，下一步枚举才是合法的。为了保证找到的第一组解就是字典序最小，枚举顺序固定为 `x` 从小到大、`y` 从小到大、方向先 `1` 后 `-1`。

剪枝则来自题面结构本身。若某种颜色在当前局面里剩下 `1` 个或 `2` 个，它以后无论如何都不可能凑成三连，直接判死；某些左右交换会与之前枚举过的右移完全等价，也可以跳过。搜索深度达到 `n` 时，如果棋盘恰好被清空，那么当前记录下来的操作序列就是答案。

### 核心代码

```cpp
bool clear_drop(){
    bool del[5][7]={},any=0;
    for(int x=0;x<5;x++) for(int y=0;y<7;y++) if(g[x][y]){
        if(x+2<5&&g[x][y]==g[x+1][y]&&g[x][y]==g[x+2][y]) del[x][y]=del[x+1][y]=del[x+2][y]=any=1;
        if(y+2<7&&g[x][y]==g[x][y+1]&&g[x][y]==g[x][y+2]) del[x][y]=del[x][y+1]=del[x][y+2]=any=1;
    }
    for(int x=0;x<5;x++){ int t=0; for(int y=0;y<7;y++) if(g[x][y]&&!del[x][y]) g[x][t++]=g[x][y]; while(t<7) g[x][t++]=0; }
    return any;
}
bool dfs(int dep){
    if(dep==n) return empty();
    if(!color_ok()) return 0;
    int bak[5][7]; memcpy(bak,g,sizeof g);
    for(int x=0;x<5;x++) for(int y=0;y<7&&g[x][y];y++) for(int d:{1,-1}) if(movable(x,y,d)&&!same_skip(x,y,d)){
        move_one(x,y,d); while(clear_drop()); path[dep]={x,y,d};
        if(dfs(dep+1)) return 1;
        memcpy(g,bak,sizeof g);
    }
    return 0;
}
```

### 复杂度

状态数取决于搜索树与剪枝效果；在题目给定的步数上限下，时间复杂度可记为指数级搜索，空间复杂度为 $O(\text{状态深度}+\text{棋盘大小})$。

## 8. [【11NOIP提高组】计算系数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1866)
`NOIP 2011` `提高组`

### 题意

给定 `(ax+by)^k`，要求求出展开式里 `x^n y^m` 这一项的系数，并对 `10007` 取模。题目保证 `n+m=k`。

### 分析

从条件走到模型的关键一步，是先把“指定项系数”翻译成二项式展开。要得到 `x^n y^m`，就必须在 `k` 次乘法里选出恰好 `n` 次取 `ax`、其余 `m` 次取 `by`。因此系数不是靠模拟乘法去算，而是直接写成

$C(k,n)\cdot a^n\cdot b^m$。

接下来只需把三个部分分别算出来。组合数用杨辉三角递推最稳：`c[i][j]=c[i-1][j-1]+c[i-1][j]`；它的不变量是 `c[i][j]` 始终表示 `C(i,j)` 对 `10007` 取模后的结果。幂部分用快速幂。最后把 `C(k,n)`、`a^n`、`b^m` 三项相乘，就是所求答案。

### 核心代码

```cpp
for(int i=0;i<=k;i++){
    c[i][0]=c[i][i]=1;
    for(int j=1;j<i;j++) c[i][j]=(c[i-1][j-1]+c[i-1][j])%10007;
}
auto qp=[&](int x,int y){ long long r=1; for(;y;y>>=1,x=1LL*x*x%10007) if(y&1) r=r*x%10007; return r; };
cout<<1LL*c[k][n]*qp(a,n)%10007*qp(b,m)%10007;
```

### 复杂度

时间复杂度 $O(k^2+\log n+\log m)$，空间复杂度 $O(k^2)$。

## 9. [【11NOIP提高组】聪明的质监员](http://ybt.ssoier.cn:8088/problem_show.php?pid=1867)
`NOIP 2011` `提高组`

### 题意

每块矿石有重量和价值。选定阈值 `W` 后，在每个区间里只保留重量不小于 `W` 的矿石，该区间的检验值等于“保留下来的矿石数量”乘“这些矿石价值和”，所有区间检验值再求总和得到 `Y`。要求让 `|Y-S|` 尽量小。

### 分析

题目里唯一能调的量是阈值 `W`。一旦 `W` 变大，能被统计进去的矿石只会更少，因此无论区间里的数量还是价值和都会不增，总检验值 `Y(W)` 也是单调不增的。看到这种“答案随阈值单调变化”的结构，就应该把模型定成二分答案，而不是在值域里暴力枚举。

固定一个 `W` 以后，真正要高效维护的是两个前缀量：`cnt[i]` 表示前 `i` 个位置里满足 `w>=W` 的矿石个数，`sum[i]` 表示它们的价值总和。它们的前缀差正好给出任意区间 `[l,r]` 内的“数量”和“价值和”，于是该区间的检验值就是这两个差值的乘积。把所有区间累加便得到 `Y(W)`。二分过程中，每次算出 `Y(W)` 都顺手更新当前的 `|Y-S|` 最小值；最后保留下来的最小差值就是答案。

### 核心代码

```cpp
long long calc(int W){
    for(int i=1;i<=n;i++)
        cnt[i]=cnt[i-1]+(w[i]>=W), sum[i]=sum[i-1]+(w[i]>=W?v[i]:0);
    long long y=0;
    for(auto [l,r]:q){
        long long c=cnt[r]-cnt[l-1], s=sum[r]-sum[l-1];
        y+=c*s;
    }
    return y;
}
for(int l=0,r=maxw+1;l<=r;){
    int mid=(l+r)>>1; long long y=calc(mid);
    ans=min(ans,llabs(y-S));
    if(y>=S) l=mid+1; else r=mid-1;
}
```

### 复杂度

时间复杂度 $O((n+m)\log V)$，空间复杂度 $O(n)$。

## 10. [【11NOIP提高组】观光公交](http://ybt.ssoier.cn:8088/problem_show.php?pid=1868)
`NOIP 2011` `提高组`

### 题意

公交按 `1` 到 `n` 的顺序单向行驶，边 `i` 的行驶时间为 `D_i`。每个乘客在时刻 `T_i` 到达站点 `A_i`，要坐到 `B_i`。公交在每一站都必须等到该站所有要出发的乘客都上车后才能离开。现在有 `k` 个加速器，每次可让某一条边的行驶时间减一，问怎样分配这些加速器，才能使所有乘客旅行时间总和最小。

### 分析

先别急着分配加速器，第一步要把“当前时刻表”写出来。设 `last[i]` 为在站点 `i` 出发的乘客里最晚到站的时刻，`arr[i]` 为公交到达站点 `i` 的时刻，那么递推就是 `arr[i+1]=max(arr[i],last[i])+D_i`。这一步把题面里的“可能等待乘客”明确翻成了状态。乘客总旅行时间则等于所有 `arr[B_i]-T_i` 的总和，因此只要知道每个站的到达时刻，就知道当前答案。

难点在于一次“把某条边减一”到底能让多少乘客受益。它并不会让所有后续乘客都少一分种，因为这份节省可能在后面某一站被等待重新吃掉。若边 `i` 减一后，公交到达 `i+1` 站仍然晚于 `last[i+1]`，说明这 `1` 分钟还能继续传递到更后面的站；反之，如果到达时间已经不晚于 `last[i+1]`，公交会在这里重新等人，收益就到此为止。于是可以从后往前求出每条边的最远受益站 `far[i]`，再用 `down` 的前缀和统计有多少乘客的终点落在 `(i,far[i]]` 中，这个数量就是当前对边 `i` 再减一的边际收益。

因为减完一次以后，整张时刻表和各边收益都会变化，所以策略是“每次重算后贪心选当前收益最大的边”。不变量是：当前 `arr` 永远对应当前的 `D`，而 `far[i]` 永远准确表示边 `i` 这一次减一能影响到的最远站。每次把答案减去最大收益并把那条边的 `D_i` 减一，重复直到没有收益或加速器用完，得到的就是最优总旅行时间。

### 核心代码

```cpp
auto build=[&](){
    arr[1]=0;
    for(int i=1;i<n;i++) arr[i+1]=max(arr[i],last[i])+d[i];
    pre[0]=0;
    for(int i=1;i<=n;i++) pre[i]=pre[i-1]+down[i];
};
while(k--){
    build(); far[n-1]=n;
    for(int i=n-2;i>=1;i--) far[i]=arr[i+1]>last[i+1]?far[i+1]:i+1;
    int best=0,pos=0;
    for(int i=1;i<n;i++) if(d[i]){
        int gain=pre[far[i]]-pre[i];
        if(gain>best) best=gain,pos=i;
    }
    if(!best) break;
    ans-=best; --d[pos];
}
```

### 复杂度

每次重算需要 $O(n)$，最多进行 `k` 次，因此时间复杂度 $O(nk+m)$，空间复杂度 $O(n+m)$。
