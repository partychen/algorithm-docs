# 一本通 NOIP 2012赛题解题报告

这一组题正好沿着 NOIP 2012 的比赛层次展开：普及组先练怎样把题面条件直接翻译成枚举、模拟、递推和状态限制，提高组再把同样的读题动作推进到贪心排序、倍增跳跃、数论构造、二分判定与树上调度。阅读时可以一直盯住同一条主线：题目到底在要求我们维护什么状态，为什么这样的状态足够，而答案又会从哪里自然落下。

# 一、普及组赛题

这一部分几乎每题都把模型直接写在题面上：有的要顺着定义做模拟，有的只差一步就能落到 DP 或状态最短路。读这一段时，重点不是记结论，而是练习把限制条件翻译成状态和转移。

## 1. [【12NOIP普及组】质因数分解](http://ybt.ssoier.cn:8088/problem_show.php?pid=1957)
`普及组` `枚举` `质数`

### 题意

给定正整数 $n$，已知它恰好是两个不同质数的乘积。题目要求输出这两个质数里较大的那个。

### 分析

题面已经把结构说得很直白：$n=p\times q$，而且 $p,q$ 都是质数、彼此不同。于是建模重点不是“分解成很多因子”，而是找到这一对因子中的较小者。只要枚举到 $\sqrt n$，第一个能整除 $n$ 的数一定就是那对质因子里的较小值，因为若两个因子都大于 $\sqrt n$，乘积就会超过 $n$。

这里的不变量是：枚举到位置 $i$ 时，我们已经确认区间 $[2,i-1]$ 中没有任何因子；一旦出现 $n\bmod i=0$，配对因子就是 $n/i$。题目保证答案一定由两个不同质数构成，所以不需要再做复杂判定。最终答案直接取 $\max(i,n/i)$，也就是较大的那个质因数。

### 核心代码

```cpp
for(long long i = 2; i * i <= n; ++i){
    if(n % i == 0){
        cout << max(i, n / i);
        break;
    }
}
```

### 复杂度

时间复杂度 $O(\sqrt n)$，空间复杂度 $O(1)$。

## 2. [【12NOIP普及组】寻宝](http://ybt.ssoier.cn:8088/problem_show.php?pid=1958)
`普及组` `模拟` `环形计数`

### 题意

藏宝楼有 $N$ 层普通楼层，每层有 $M$ 个围成一圈的房间。进入某层的某个房间后，要先记下该房间指示牌上的数字 $x$，再从这个房间开始按逆时针寻找第 $x$ 个有楼梯的房间，上楼后会到达上一层同编号的房间。求一路记下的数字总和，对 $20123$ 取模。

### 分析

题目描述的是一个完全确定的过程，所以先把状态抓成“当前在第几层、进入这一层时所在的房间编号是多少”。处理当前楼层时，先把这个房间的指示牌数字加入答案，然后按照规则找到本层要走的楼梯房间；由于上楼后编号不变，这个楼梯房间的编号就会成为下一层的起点。

真正要想清楚的是“第 $x$ 个有楼梯的房间”怎么找。设本层一共有 $cnt$ 个楼梯房间，那么沿环多转一整圈只会把楼梯顺序重复一遍，所以只需要找第 $((x-1)\bmod cnt)+1$ 个楼梯。这样每层最多扫一圈房间就够了。这里的不变量是：处理完第 $i$ 层后，`cur` 正是该层选中的上楼房间编号。最终答案就是所有楼层首次进入房间的指示牌数字之和，再按题意取模。

### 核心代码

```cpp
int ans = 0, cur = start;
for(int i = 0; i < n; ++i){
    ans = (ans + val[i][cur]) % 20123;
    int need = (val[i][cur] - 1) % stairCnt[i] + 1, cnt = 0;
    for(int p = cur; ; p = (p + 1) % m) if(has[i][p]){
        if(++cnt == need){ cur = p; break; }
    }
}
cout << ans;
```

### 复杂度

时间复杂度 $O(NM)$，空间复杂度 $O(NM)$。

## 3. [【12NOIP普及组】摆花](http://ybt.ssoier.cn:8088/problem_show.php?pid=1959)
`普及组` `动态规划` `计数`

### 题意

共有 $n$ 种花，要摆出总计 $m$ 盆。第 $i$ 种花至多使用 $a_i$ 盆，而且同种花必须放在一起、不同种花按编号从小到大依次出现。求不同摆法数量，对 $1000007$ 取模。

### 分析

“同种花放在一起且按编号顺序出现”这一句很关键，它把原题直接压成了“每种花选几盆”的计数问题。只要决定每个 $x_i$，满足 $0\le x_i\le a_i$ 且 $\sum x_i=m$，摆放方案就唯一确定，因此自然想到做容量型 DP。

设 `f[i][j]` 表示只看前 $i$ 种花，恰好摆出 $j$ 盆的方案数。转移时枚举第 $i$ 种花放了多少盆：`k=0..min(a[i],j)`，把 `f[i-1][j-k]` 累加过来即可。这里的状态含义就是不变量：前 $i$ 种花的贡献已经完全确定，后面的花还没有参与。最后答案直接取 `f[n][m]`。

### 核心代码

```cpp
f[0][0] = 1;
for(int i = 1; i <= n; ++i){
    for(int j = 0; j <= m; ++j){
        for(int k = 0; k <= min(a[i], j); ++k){
            f[i][j] = (f[i][j] + f[i - 1][j - k]) % 1000007;
        }
    }
}
cout << f[n][m];
```

### 复杂度

时间复杂度 $O\!\left(nm\max a_i\right)$，空间复杂度 $O(nm)$。

## 4. [【12NOIP普及组】文化之旅](http://ybt.ssoier.cn:8088/problem_show.php?pid=1960)
`普及组` `状态压缩` `最短路`

### 题意

国家之间有带权无向道路，每个国家对应一种文化。旅行者一旦学过某种文化，就不能再去其他同文化国家；而且如果某个国家的文化会排斥他已经学过的任意一种文化，他也不能进入这个国家。求从起点到终点的最短路。

### 分析

这题难点不是路，而是“已经学过哪些文化”会影响以后还能不能走。于是最自然的建模是把“所在国家”与“已学文化集合”绑在一起，形成状态 `(u,mask)`。从 `u` 走到相邻国家 `v` 时，只有在 `v` 的文化还没学过、并且 `v` 的文化不排斥 `mask` 中任何文化时，转移才合法。

因为边权都是正数，状态图上跑 Dijkstra 最稳妥。这里的不变量是：`dist[u][mask]` 始终表示到达国家 `u` 且已学文化集合恰为 `mask` 的最短距离，一旦它被最短路算法弹出，就已经是最终最优值。终点可能对应多个合法集合，所以答案不是单个固定状态，而是所有 `dist[T][mask]` 的最小值。

### 核心代码

```cpp
int s0 = 1 << (c[s] - 1);
dist[s][s0] = 0;
pq.push({0, s, s0});
while(!pq.empty()){
    auto [d, u, mask] = pq.top(); pq.pop();
    if(d != dist[u][mask]) continue;
    for(auto [v, w] : g[u]){
        int cv = c[v] - 1;
        if(mask >> cv & 1) continue;
        if(ban[cv][mask]) continue;
        int nmask = mask | (1 << cv);
        if(dist[v][nmask] > d + w)
            dist[v][nmask] = d + w, pq.push({dist[v][nmask], v, nmask});
    }
}
```

### 复杂度

时间复杂度 $O\!\left((N+M)2^K\log(N2^K)\right)$，空间复杂度 $O(N2^K)$。

# 二、提高组赛题

进入提高组以后，题目的表层叙述开始和真正模型拉开距离：要么先证明排序准则，要么把反复行走压成倍增，要么把可行性判定塞进二分。共同目标都是先抓住不变量，再让复杂过程变成可维护的结构。

## 5. [【12NOIP提高组】Vigenère密码](http://ybt.ssoier.cn:8088/problem_show.php?pid=1869)
`提高组` `字符串` `模拟`

### 题意

给定密钥串和密文，按 Vigenère 密码的规则把密文还原成明文。运算忽略字母大小写，但输出时要保持原有大小写形式；当密钥长度不够时需要循环使用。

### 分析

题目虽然包装成密码学背景，真正的模型却是逐位独立的字符变换。密钥第 $i$ 位决定了当前位置要向前回退多少个字母，所以先把密钥字符统一转成 $0\sim25$ 的位移量，再让它按长度循环即可。

这里的关键不变量是：处理到密文第 $i$ 位时，只依赖 `key[i % len]` 和当前字符本身，不会和其他位置互相影响。因此直接线性扫描即可。大小写也不用分开讨论，只要先根据当前字符确定基准是 `A` 还是 `a`，最后把回退后的结果重新加回这个基准，答案串自然就出来了。

### 核心代码

```cpp
string ans = c;
for(int i = 0; i < (int)c.size(); ++i){
    int sh = tolower(k[i % k.size()]) - 'a';
    char base = isupper(c[i]) ? 'A' : 'a';
    ans[i] = char((c[i] - base - sh + 26) % 26 + base);
}
cout << ans;
```

### 复杂度

时间复杂度 $O(|C|)$，空间复杂度 $O(|C|)$。

## 6. [【12NOIP提高组】国王游戏](http://ybt.ssoier.cn:8088/problem_show.php?pid=1870)
`提高组` `贪心` `排序`

### 题意

国王在队首，每位大臣有左手数 $a_i$、右手数 $b_i$。某位大臣得到的金币数等于他前面所有人左手数字乘积除以他自己右手数字后向下取整。要求重新排列大臣顺序，使“获得金币最多的人”的金币数尽量小。

### 分析

先把题意改写成更容易比较的形式：如果当前前缀左手乘积是 `prod`，那么某位大臣排在这里时造成的风险值就是 `prod / b_i`，而他排完后又会把前缀乘积放大成 `prod * a_i`。所以一个人的位置既影响自己的分母，也影响后面所有人的分子，这正是需要排序贪心的信号。

比较相邻两位大臣 $x,y$ 的交换。固定前缀乘积为 $P$，若 $x$ 在前，局部最坏值是 $\max(P/b_x, P a_x/b_y)$；若 $y$ 在前，则是 $\max(P/b_y, P a_y/b_x)$。用交换论证可以推出：把 `a_i b_i` 较小的人放前面不会更劣，因此按 `a_i*b_i` 升序排序即可。扫描时维护不变量 `prod` 为国王和已经排好的所有大臣左手数乘积，当前大臣的金币数就是 `prod / b_i`，取整个扫描过程中的最大值便是答案。由于乘积会非常大，核心实现需要高精度整数。

### 核心代码

```cpp
using boost::multiprecision::cpp_int;
sort(p.begin(), p.end(), [](auto x, auto y){ return x.a * x.b < y.a * y.b; });
cpp_int prod = king.a, ans = 0;
for(auto [a, b] : p){
    ans = max(ans, prod / b);
    prod *= a;
}
cout << ans;
```

### 复杂度

排序部分时间复杂度 $O(n\log n)$，扫描为 $O(n)$；空间复杂度 $O(n)$，另加高精度整数的存储开销。

## 7. [【12NOIP提高组】开车旅行](http://ybt.ssoier.cn:8088/problem_show.php?pid=1871)
`提高组` `倍增` `有序集合`

### 题意

城市从西到东编号为 $1\sim N$，海拔互不相同，两城距离等于海拔差绝对值。旅行时小 A 先开，每次去东边还未到达过的“第二近”城市；随后小 B 从当前城市去东边“最近”城市；如此交替，若下一步会超出总里程上限 $X$ 或者不存在可选城市就停止。题目先给一个 $X_0$，要求找出使 $A/B$ 最小的出发城市，再回答若干组 $(S_i,X_i)$ 下两人的总里程。

### 分析

先从单步规则建模。对每个城市 $i$，真正会用到的只有东边城市里按“距离最小、若并列则海拔较低者更近”的前两名：最近城市给小 B，次近城市给小 A。因为这个排名只和东边所有城市的海拔有序关系有关，所以可以从东向西扫描，用按海拔有序的平衡树维护已经出现的城市；每次只检查前驱、后继附近的少数候选点，就能确定 `bTo[i]` 和 `aTo[i]`。

接下来把两个人的一整轮行动压成一个跳跃。设 `up[i][k]` 表示从城市 `i` 出发，连续完成 $2^k$ 轮“先 A 后 B”后会停在哪个城市；同时记录对应的小 A 里程和小 B 里程 `da[i][k]`、`db[i][k]`。这样查询某个 $(S,X)$ 时，就能像倍增跳祖先一样，从大到小尝试加入整块轮次，只要当前累计里程不超过 $X$ 就接受这个跳跃。所有整轮跳完后，再额外判断能不能补走最后一步 A。

这里的不变量是：倍增表里的每一项都精确对应若干完整轮次的终点和两人累计里程，因此贪心拼块不会漏掉合法方案。第一问只要对每个起点都跑一次查询，用交叉相乘比较 $A/B$，若比值相同再按题意选择海拔更高的起点；其余询问直接输出查询得到的两段里程。

### 核心代码

```cpp
for(int k = 1; k < LOG; ++k) if(up[i][k - 1]){
    int mid = up[i][k - 1];
    up[i][k] = up[mid][k - 1];
    da[i][k] = da[i][k - 1] + da[mid][k - 1];
    db[i][k] = db[i][k - 1] + db[mid][k - 1];
}
pair<long long,long long> calc(int s, long long lim){
    long long A = 0, B = 0;
    for(int k = LOG - 1; k >= 0; --k)
        if(up[s][k] && da[s][k] + db[s][k] <= lim)
            lim -= da[s][k] + db[s][k], A += da[s][k], B += db[s][k], s = up[s][k];
    if(aTo[s] && distA[s] <= lim) A += distA[s];
    return {A, B};
}
```

### 复杂度

预处理时间复杂度 $O(N\log N)$，单次查询时间复杂度 $O(\log N)$，空间复杂度 $O(N\log N)$。

## 8. [【12NOIP提高组】同余方程](http://ybt.ssoier.cn:8088/problem_show.php?pid=1872)
`提高组` `数论` `扩展欧几里得`

### 题意

求同余方程 $ax\equiv1\pmod b$ 的最小正整数解，题目保证一定有解。

### 分析

同余式 $ax\equiv1\pmod b$ 等价于存在整数 $y$ 使 $ax+by=1$。这已经把问题直接翻译成了贝祖等式，所以核心模型就是扩展欧几里得：只要求出一组 $(x,y)$ 满足 $ax+by=\gcd(a,b)$，而题目保证有解，也就意味着 $\gcd(a,b)=1$。

扩欧返回的 `x` 可能是负数，也可能不是最小正解，但同余类只有一个，所以把它规范到模 $b$ 的范围里即可。这里的不变量是：递归每一层都保持“当前返回值是一组贝祖系数”。答案提取就是 `x mod b` 的最小正代表，即 `((x % b) + b) % b`。

### 核心代码

```cpp
long long exgcd(long long a, long long b, long long &x, long long &y){
    if(!b) return x = 1, y = 0, a;
    long long d = exgcd(b, a % b, y, x);
    y -= a / b * x;
    return d;
}
long long x, y;
exgcd(a, b, x, y);
cout << (x % b + b) % b;
```

### 复杂度

时间复杂度 $O(\log b)$，空间复杂度 $O(\log b)$。

## 9. [【12NOIP提高组】借教室](http://ybt.ssoier.cn:8088/problem_show.php?pid=1873)
`提高组` `二分答案` `差分`

### 题意

学校未来 $n$ 天每天有固定数量的教室，订单按顺序依次处理。第 $j$ 份订单要求在区间 $[s_j,t_j]$ 内每天借走 $d_j$ 间教室。若某一份订单加入后出现任意一天供给不足，就必须立刻停止并输出最早出问题的订单编号；若全部可行则输出 $0$。

### 分析

题目看似在线处理，但真正要找的是“第一份失败订单”的位置。若前 $k$ 份订单都能满足，那么更少的前缀显然也能满足；反过来，一旦前 $k$ 份已经失败，更长前缀也不可能重新变可行。这个单调性一出现，就应该把问题转成二分最早失败位置。

固定检查前 $k$ 份订单时，只需要知道每一天总共被借走多少教室，因此用差分数组最合适：对订单 $(d,s,t)$ 做 `diff[s]+=d, diff[t+1]-=d`，再扫一遍前缀和得到每天需求量。扫描时的不变量是：当前位置的前缀和就是当天总需求，只要它第一次超过 `r[i]`，前 $k$ 份订单就不可行。若二分后找到最小坏前缀 `ans`，输出 `-1` 和 `ans` 即可；若所有前缀都可行，答案就是单独输出 `0`。

### 核心代码

```cpp
auto check = [&](int k){
    fill(diff.begin(), diff.end(), 0);
    for(int i = 1; i <= k; ++i) diff[s[i]] += d[i], diff[t[i] + 1] -= d[i];
    long long cur = 0;
    for(int i = 1; i <= n; ++i){
        cur += diff[i];
        if(cur > r[i]) return false;
    }
    return true;
};
int l = 1, r0 = m, ans = -1;
while(l <= r0){
    int mid = (l + r0) >> 1;
    if(check(mid)) l = mid + 1;
    else ans = mid, r0 = mid - 1;
}
```

### 复杂度

单次检查时间复杂度 $O(n+m)$，整体时间复杂度 $O((n+m)\log m)$，空间复杂度 $O(n)$。

## 10. [【12NOIP提高组】疫情控制](http://ybt.ssoier.cn:8088/problem_show.php?pid=1874)
`提高组` `树上贪心` `二分答案`

### 题意

首都位于树根 $1$，若想阻止疫情传到所有叶子，就要让从根到每个叶子的路径上至少有一个检查点。若干支军队分布在树上，每支军队可以沿树移动，总耗时等于边权和，并且最终只能在一个非根节点建立一个检查点。求最少需要多少时间才能完成全部布防。

### 分析

看到“最少多少时间”且所有军队可以并行移动，第一反应应当是二分答案。固定一个时间上限 $T$ 后，问题变成：每支军队最多走这么远，能否让所有根到叶子的路径都被拦住。树结构告诉我们，任何根到叶子的路径都会先经过根的某个儿子，所以真正需要被逐一保证的，其实是根的每个儿子子树。

检查时，先用倍增把每支军队尽量往上跳到时间 $T$ 内能到达的最高位置。若它还到不了根儿子这一层，那它无论怎么布置都只能在当前子树里设点，于是把它记作“内部覆盖资源”；这些资源沿 DFS 向上合并，能够直接判定哪些根儿子子树已经被自己内部的军队封住。若某支军队已经能顶到根儿子甚至更高，它就变成“机动资源”：在到达某个根儿子位置后，还会剩下一段可再分配的时间。

剩下的就是贪心匹配。把还没被内部资源覆盖的根儿子子树按从根进入该子树所需的边权排序，再把机动资源按剩余时间排序，用最小可用时间去覆盖当前最容易补上的子树即可。这个贪心成立的原因是需求和资源都只有一个维度——时间。只要某个 $T$ 能通过检查，更大的时间也一定能通过，所以二分得到的第一个可行值就是最终答案；若连足够大的时间都无法覆盖全部根儿子子树，则无解输出 $-1$。

### 核心代码

```cpp
bool check(long long T){
    clear_tag(); vector<long long> rest, need;
    for(int u : army){
        int x = climb_highest(u, T);
        if(dep[x] > 1) tag[x]++;
        else rest.push_back(T - dist_to_root_son(u, T));
    }
    dfs_cover(1, 0);
    for(auto [v, w] : rootSon) if(!covered[v]) need.push_back(w);
    sort(rest.begin(), rest.end());
    sort(need.begin(), need.end());
    int j = 0;
    for(long long t : rest) if(j < (int)need.size() && t >= need[j]) ++j;
    return j == (int)need.size();
}
```

### 复杂度

单次检查时间复杂度 $O((n+m)\log n)$，整体时间复杂度 $O((n+m)\log n\log W)$，空间复杂度 $O(n\log n)$。
