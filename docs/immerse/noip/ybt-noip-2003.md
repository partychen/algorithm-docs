# 一本通 NOIP 2003赛题解题报告

这一年赛题从普及组的规则模拟、环形区间 DP 与组合计数，一路走到提高组的 DAG 传播、逻辑枚举、区间树形 DP 和层次搜索。前半段练的是怎样把题面流程压成清晰状态，后半段则更强调先认清结构，再决定该做动态规划、拓扑传播还是按传播层递归决策。

# 一、普及组赛题

普及组几题看似分散，其实都在练“把过程写成状态”。有的题直接按规则推进，有的题先断环成链再做区间 DP，有的题则把合法操作序列改写成经典计数模型。

## 1. [【03NOIP普及组】乒乓球](http://ybt.ssoier.cn:8088/problem_show.php?pid=1922)
`NOIP 2003` `普及组`

### 题意

给出按时间顺序记录的胜负串，遇到 `E` 结束。你要分别按照 11 分制和 21 分制，把已经结束的每一局比分以及最后一局未结束时的当前比分依次输出。

### 分析

这题最重要的观察是：无论用 11 分制还是 21 分制，真正会随扫描过程变化的只有“当前这一局的比分” `(a,b)`。每读到一个 `W` 或 `L`，只是在当前局里给一方加一分；一旦有人达到目标分且领先至少 2 分，这一局就立刻封口，答案里新增一行，然后状态清零进入下一局。

所以题面里那串很长的比赛记录，其实可以压成一个非常小的自动机：扫描字符、更新当前分数、满足封局条件就结算。这里的核心不变量是：`(a,b)` 始终表示“在当前计分规则下，最后一局尚未结束时的比分”。

同一份记录要输出两套结果，只是封局阈值不同，因此把规则写成 `solve(limit)` 最自然。最后别忘了把扫描结束时的 `(a,b)` 也输出出来；如果记录恰好停在一局结束后，这个状态就是 `0:0`，它正是题目要求的“下一局刚开始”的比分。

### 核心代码

```cpp
vector<pair<int,int>> solve(const string& s, int lim){
  vector<pair<int,int>> res;
  int a = 0, b = 0;
  for(char ch : s){
    if(ch == 'W') a++;
    else if(ch == 'L') b++;
    if((a >= lim || b >= lim) && abs(a - b) >= 2){
      res.push_back({a, b});
      a = b = 0;
    }
  }
  res.push_back({a, b});
  return res;
}
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$，其中 $n$ 是有效比赛记录长度。

## 2. [【03NOIP普及组】数字游戏](http://ybt.ssoier.cn:8088/problem_show.php?pid=1923)
`NOIP 2003` `普及组`

### 题意

有一个首尾相接的整数环，要按顺序切成 `m` 段。每段先求和，再对 10 取成非负余数，最后把这 `m` 个余数相乘。要求分别求这个乘积的最小值和最大值。

### 分析

麻烦点有两个：一是“环”，二是“既要最小又要最大”。先看环形条件。因为每一段都必须连续，环上切 `m` 刀后一定可以选一个断点把它摊平成长度为 `n` 的链，所以标准处理就是把数组复制一遍，枚举断点，把问题转成所有长度为 `n` 的区间上的线性划分。

摊平之后，真正的状态就很清楚了：设 `mn[l][r][k]`、`mx[l][r][k]` 表示区间 `[l,r]` 被分成 `k` 段时，乘积最小值和最大值。若最后一段是 `[t+1,r]`，那前面 `k-1` 段只能落在 `[l,t]`，转移就是“前一部分最优值 × 最后一段的余数”。其中段权 `val(l,r)` 可以用前缀和在 $O(1)$ 求出，再做一次非负取模。

这个状态之所以成立，是因为题目目标是分段后各段函数值的乘积，而最后一刀一旦确定，左右两边就完全独立。答案提取时，只需在复制后的数组里枚举起点 `s`，考察区间 `[s,s+n-1]` 的 `m` 段最值，再把所有起点的结果汇总即可。

### 核心代码

```cpp
int val(int l, int r){
  return ((pre[r] - pre[l - 1]) % 10 + 10) % 10;
}
for(int len = 1; len <= n; len++)
  for(int l = 1; l + len - 1 <= 2 * n; l++){
    int r = l + len - 1;
    mn[l][r][1] = mx[l][r][1] = val(l, r);
    for(int k = 2; k <= min(m, len); k++){
      mn[l][r][k] = INF, mx[l][r][k] = -INF;
      for(int t = l + k - 2; t < r; t++){
        mn[l][r][k] = min(mn[l][r][k], mn[l][t][k - 1] * val(t + 1, r));
        mx[l][r][k] = max(mx[l][r][k], mx[l][t][k - 1] * val(t + 1, r));
      }
    }
  }
```

### 复杂度

时间复杂度 $O(n^3m)$，空间复杂度 $O(n^2m)$。

## 3. [【03NOIP普及组】栈](http://ybt.ssoier.cn:8088/problem_show.php?pid=1924)
`NOIP 2003` `普及组`

### 题意

输入序列固定为 `1,2,...,n`，你可以反复执行“把下一个数压栈”与“把栈顶弹出到输出序列”两种操作。题目要求统计最终可能得到多少种不同的输出序列。

### 分析

与其直接数输出排列，不如回到操作过程本身。因为输入顺序已经固定，真正的自由度只有两种选择：当前是继续压入下一个数，还是把栈顶弹出。于是每一种合法输出，其实都唯一对应一条合法的压栈/出栈操作序列。

把压栈记成上一步，出栈记成下一步，整个过程就变成了长度为 `2n` 的括号序列：总共要压 `n` 次、弹 `n` 次，而且任何前缀里都不能出现“弹出次数多于压入次数”，否则就等价于从空栈里弹元素。于是状态可以写成 `f[i][j]`：做了 `i` 次操作后，当前栈里还有 `j` 个元素的方案数。

转移时，如果已经压入的元素还没到 `n` 个，就能再压一次；如果 `j>0`，就能弹一次。这里的核心不变量是“栈内元素个数 = 已压入次数 - 已弹出次数”，而题目要的答案就是走完 `2n` 步后回到空栈的方案数，也就是 Catalan 结构对应的终点值。

### 核心代码

```cpp
f[0][0] = 1;
for(int i = 0; i < 2 * n; i++)
  for(int j = 0; j <= n; j++) if(f[i][j]){
    int pushed = (i + j) / 2;
    if(pushed < n) f[i + 1][j + 1] += f[i][j];
    if(j > 0) f[i + 1][j - 1] += f[i][j];
  }
cout << f[2 * n][0];
```

### 复杂度

时间复杂度 $O(n^2)$，空间复杂度 $O(n^2)$。

## 4. [【03NOIP普及组】麦森数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1925)
`NOIP 2003` `普及组`

### 题意

给定指数 `P`，要求输出高精度整数 $2^P-1$ 的十进制位数，以及它十进制表示的最后 500 位，不需要判断这个数是否真的为素数。

### 分析

位数部分可以直接用对数：十进制位数是 $\lfloor P\log_{10}2\rfloor+1$。真正的难点只在“最后 500 位”，也就是只要求 $2^P-1$ 对 $10^{500}$ 取模后的结果。这样一来，题目就从“算一个巨大的整数”转成了“在模 $10^{500}$ 意义下做快速幂”。

为了把这个模数落到程序里，可以把大整数按十进制每 5 位压成一个块，用数组保存低位在前的 100 个块。乘法时只保留前 100 个块，等价于始终在模 $10^{500}$ 下运算；这就是整个高精度过程的核心不变量。因为我们只关心末 500 位，更高的块天然可以截断。

快速幂让乘法次数从 `P` 次降到 `O(log P)` 次。最后把结果减一，再按固定 500 位补前导零输出即可。答案提取分成两步：位数由对数公式给出，末 500 位由被截断的大整数数组直接格式化输出。

### 核心代码

```cpp
const int BASE = 100000, LEN = 100;
using Big = array<int, LEN>;
Big mul(Big a, Big b){
  Big c{};
  for(int i = 0; i < LEN; i++) if(a[i])
    for(int j = 0; i + j < LEN; j++){
      long long t = c[i + j] + 1LL * a[i] * b[j];
      c[i + j] = t % BASE;
      if(i + j + 1 < LEN) c[i + j + 1] += t / BASE;
    }
  for(int i = 0; i + 1 < LEN; i++) c[i + 1] += c[i] / BASE, c[i] %= BASE;
  return c;
}
Big qpow(int p){
  Big ans{}, x{}; ans[0] = 1; x[0] = 2;
  while(p){
    if(p & 1) ans = mul(ans, x);
    x = mul(x, x);
    p >>= 1;
  }
  return ans;
}
```

### 复杂度

时间复杂度约为 $O(L^2\log P)$，空间复杂度 $O(L)$，其中 $L=100$ 表示 500 位十进制按 5 位一组后的块数。

# 二、提高组赛题

提高组明显更强调建模深度：有的题要先把网络看成 DAG 再传播状态，有的题要在真假话与日期之间做一致性检验，有的题则要在树的层次推进里枚举控制策略。

## 5. [【03NOIP提高组】神经网络](http://ybt.ssoier.cn:8088/problem_show.php?pid=1831)
`NOIP 2003` `提高组`

### 题意

给定一张分层有向图形式的神经网络。输入层已有初始状态，非输入层初始为 0，每个点有阈值，每条边有权。只有当前状态大于 0 的神经元才会向下一层传递信号，要求求出输出层神经元最后的非零状态。

### 分析

题目虽然叫“神经网络”，但结构已经被限定成分层 DAG：边只从上一层指向下一层，所以状态传播不会回头。于是关键不是图算法多复杂，而是先把题面公式翻译成一趟拓扑传播。对每个非输入层，真正起作用的是“收到的总输入减去阈值”，因此可以先把它的状态初始化为 `-U_i`，后面只不断累加来自前驱的贡献。

传播时要抓住一个核心条件：只有当前 `C_j>0` 的点才会继续向外发送强度为 `C_j` 的信号，所以拓扑序处理到某个点 `u` 时，若 `C_u<=0`，它对后继的贡献就是 0；若 `C_u>0`，就把 `C_u * w` 加到每条出边终点上。这个规则本身就是状态转移。

因为图无环，等一个点所有前驱都处理完，它的最终状态也就确定了。最后只需在出度为 0 的点里筛出状态仍然为正的神经元并按编号输出；如果一个都没有，说明输出层没有被真正激活，答案就是 `NULL`。

### 核心代码

```cpp
for(int i = 1; i <= n; i++){
  if(indeg[i] == 0) q.push(i);
  else c[i] -= u[i];
}
while(!q.empty()){
  int u0 = q.front(); q.pop();
  if(c[u0] > 0)
    for(auto [v, w] : g[u0]) c[v] += c[u0] * w;
  for(auto [v, w] : g[u0])
    if(--indeg[v] == 0) q.push(v);
}
for(int i = 1; i <= n; i++)
  if(outdeg[i] == 0 && c[i] > 0) cout << i << ' ' << c[i] << "\n";
```

### 复杂度

时间复杂度 $O(n+p)$，空间复杂度 $O(n+p)$。

## 6. [【03NOIP提高组】侦探推理](http://ybt.ssoier.cn:8088/problem_show.php?pid=1832)
`NOIP 2003` `提高组`

### 题意

有 `M` 个同学、其中恰好 `N` 个永远说假话。每条证词只在几种固定句式中才参与推理，并且证词还可能涉及“今天星期几”。要求判断罪犯是谁，若无法唯一确定则输出相应结论。

### 分析

题目的不确定量其实只有两类：真正的罪犯是谁，以及“Today is ...”里的星期几是哪一天。把这两个量枚举掉之后，每一句可识别证词的真假就都能直接判定，因此整题会从“复杂逻辑题”变成“对一个假设做一致性检查”。

对固定的 `(killer, day)`，我们逐个人看他的所有有效证词。如果某人的证词里既有真话又有假话，那他不可能属于“永远真”或“永远假”中的任何一种，这个假设立刻作废；如果全是真，他被强制为真话者；如果全是假，他被强制为假话者；如果一句有效证词都没有，他的身份仍然悬空。

于是状态检查就压成了计数问题：设当前假设下被强制成假的人数为 `lie`，没有被判定的人数为 `unk`，那么只要 `lie <= N <= lie + unk`，就说明还能补出恰好 `N` 个说谎者，这个假设可行。某个名字只要在至少一个星期几下可行，就可能是罪犯。最后看可能罪犯集合的大小：0 个是 `Impossible`，1 个就唯一确定，多个则是 `Cannot Determine`。

### 核心代码

```cpp
int eval(int who, const string& s, int killer, int day){
  if(s == "I am guilty.") return who == killer;
  if(s == "I am not guilty.") return who != killer;
  for(int i = 1; i <= m; i++){
    if(s == name[i] + " is guilty.") return i == killer;
    if(s == name[i] + " is not guilty.") return i != killer;
  }
  for(int d = 0; d < 7; d++)
    if(s == "Today is " + week[d] + ".") return d == day;
  return -1;
}
bool check(int killer, int day){
  int lie = 0, unk = 0;
  for(int i = 1; i <= m; i++){
    int state = -1;
    for(auto& s : say[i]){
      int t = eval(i, s, killer, day);
      if(t == -1) continue;
      if(state == -1) state = t;
      else if(state != t) return false;
    }
    if(state == -1) unk++;
    else if(state == 0) lie++;
  }
  return lie <= nLiar && nLiar <= lie + unk;
}
```

### 复杂度

时间复杂度 $O(7M(M+P))$，空间复杂度 $O(M+P)$。

## 7. [【03NOIP提高组】加分二叉树](http://ybt.ssoier.cn:8088/problem_show.php?pid=1833)
`NOIP 2003` `提高组`

### 题意

已知二叉树的中序遍历固定为 `1,2,...,n`，第 `i` 个点有权值 `d_i`。一棵子树的得分定义为“左子树得分 × 右子树得分 + 根权值”，空子树得分记为 1。要求求出整棵树的最大得分，并输出对应的前序遍历。

### 分析

中序遍历已经固定成连续编号，这意味着一旦根选成 `k`，左子树就只能用区间 `[l,k-1]`，右子树就只能用区间 `[k+1,r]`。所以题目本质上不是在任意二叉树里搜索，而是在一个区间上决定谁做根，典型就是区间 DP。

设 `f[l][r]` 表示用中序区间 `[l,r]` 这些点构成子树时能取得的最大得分，`rt[l][r]` 记录最优根。若根选 `k`，左、右两边的最优得分互不干扰，当前得分就是 `f[l][k-1] * f[k+1][r] + d[k]`；边界上如果某一侧为空，就按题意把它当作 1。这里的核心状态与转移都直接来自“中序固定后，根把区间切成左右两段”。

区间 DP 求完后，答案就是 `f[1][n]`。而前序遍历也不需要真的把树建出来，只要递归读取 `rt[l][r]`：先输出根，再输出左区间和右区间对应的根序列，这正好就是前序。

### 核心代码

```cpp
for(int i = 1; i <= n; i++) f[i][i] = a[i], rt[i][i] = i;
for(int len = 2; len <= n; len++)
  for(int l = 1; l + len - 1 <= n; l++){
    int r = l + len - 1;
    for(int k = l; k <= r; k++){
      long long L = (k == l ? 1 : f[l][k - 1]);
      long long R = (k == r ? 1 : f[k + 1][r]);
      long long cur = L * R + a[k];
      if(cur > f[l][r]) f[l][r] = cur, rt[l][r] = k;
    }
  }
void print(int l, int r){
  if(l > r) return;
  int k = rt[l][r];
  cout << k << ' ';
  print(l, k - 1), print(k + 1, r);
}
```

### 复杂度

时间复杂度 $O(n^3)$，空间复杂度 $O(n^2)$。

## 8. [【03NOIP提高组】传染病控制](http://ybt.ssoier.cn:8088/problem_show.php?pid=1834)
`NOIP 2003` `提高组`

### 题意

传播图是一棵以 1 号点为源头的树。每个传播周期里，病毒只会从当前已感染的一层继续感染下一层，而你在每个周期只能切断一条传播边。要求安排切边顺序，使最终被感染的人数尽量少。

### 分析

题目真正卡人的地方，是要先把“传播有周期性”读成“感染按层推进”。既然一个周期内只会新增下一代患者，那么在任意时刻，真正会参与下一轮传播的只有当前最深那一层感染者。也正因为如此，最优方案里没有必要提前去切更深的边：如果某条边要发挥作用，等它的父亲刚准备把病毒传过去时再切，不会比现在更差。

于是状态就可以写成“当前这一层已经感染的点集 `cur`”。把它们所有孩子收集出来得到 `nxt`，这就是下一轮可能被感染的人。你本轮只能切一条边，等价于在 `nxt` 里选一个孩子 `ban` 不让它被感染；其余 `|nxt|-1` 个孩子全部进入下一层，形成新的 `cur`。这说明整题可以用按层 DFS：枚举每一轮到底保住哪一棵子树。

搜索成立的核心不变量是：递归参数 `infected` 始终等于到当前层为止已经确定会被感染的人数，`cur` 始终是下一轮唯一可能继续外扩的节点集合。每展开一层，新增感染数就是新集合的大小。为了让分支尽快收敛，可以预处理子树大小并优先尝试切掉大子树，这样更容易尽早得到较优答案用于剪枝。最终搜到没有孩子可扩散时，`infected` 就是这一方案下的总感染人数。

### 核心代码

```cpp
void dfs(vector<int> cur, int infected){
  if(infected >= ans) return;
  vector<int> nxt;
  for(int u : cur)
    for(int v : son[u]) nxt.push_back(v);
  if(nxt.empty()){
    ans = infected;
    return;
  }
  sort(nxt.begin(), nxt.end(), [&](int a, int b){ return siz[a] > siz[b]; });
  for(int ban : nxt){
    vector<int> go;
    for(int v : nxt) if(v != ban) go.push_back(v);
    dfs(go, infected + (int)go.size());
  }
}
```

### 复杂度

最坏时间复杂度呈指数级，但由于每轮只在同一层里选一个分支并配合子树大小剪枝，在题目的数据范围内可以通过；空间复杂度为递归深度与层集合存储，记作 $O(n)$。
