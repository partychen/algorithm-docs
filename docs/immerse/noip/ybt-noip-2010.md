# 一本通 NOIP 2010赛题解题报告

这一组题从普及组的直接模拟、调度贪心一路走到提高组的状态压缩、二分图约束与区间覆盖，难点不是公式突然变多，而是要学会把题面条件一步步翻成可计算的状态。读的时候可以先盯住“当前局面到底该记什么”，再看“这个转移为什么没有漏情况”，这样从前半段的直观题走到后半段的综合建模会更顺。

# 一、普及组赛题

这一节的题面都不绕，但每一道都在训练最基本的建模判断：数据范围小就老老实实枚举，过程有固定顺序就抓住当前最关键的量，几何题也先别急着上复杂工具，而是先看半径和覆盖条件到底怎样落成一个切分点。

## 1. [【10NOIP普及组】数字统计](http://ybt.ssoier.cn:8088/problem_show.php?pid=1949)
`NOIP 2010` `普及组`

### 题意

给定整数区间 `[L,R]`，要统计从 `L` 到 `R` 的所有整数里，数字 `2` 总共出现了多少次。这里统计的是十进制表示中每一位上的出现次数，同一个数里如果有多个 `2`，都要分别计入答案。

### 分析

先看数据范围，`R` 只有 `10000`，这就说明题目并不要求我们去设计数位动态规划之类的大模型，而是直接把区间里的每个数拿出来检查即可。真正要做的事只有一件：把一个整数不断除以 `10`，逐位判断当前末位是不是 `2`。

这个建模最朴素，但也最贴合条件。因为题目要的是“区间内所有数的每一位贡献之和”，所以把答案拆成“每个数贡献多少个 `2`”再累加，天然不会重不漏。这里的关键状态就是当前正在处理的数 `x`；循环不变的是：已经弹掉的低位都统计完了，剩下的高位还没看。每次做 `x % 10` 得到当前位，做 `x /= 10` 进入下一位。

答案提取也很直接：对每个 `x` 求出它内部数字 `2` 的个数，再把这些值累加起来就是最终答案。

### 核心代码

```cpp
int count2(int x){
  int s = 0;
  while(x){
    s += (x % 10 == 2);
    x /= 10;
  }
  return s;
}
int ans = 0;
for(int x = l; x <= r; x++) ans += count2(x);
cout << ans;
```

### 复杂度

时间复杂度是 `O((R-L+1)\log_{10}R)`，空间复杂度是 `O(1)`。

## 2. [【10NOIP普及组】接水问题](http://ybt.ssoier.cn:8088/problem_show.php?pid=1950)
`NOIP 2010` `普及组`

### 题意

有 `m` 个水龙头和按顺序排好的 `n` 名同学，第 `i` 名同学接满水需要 `w_i` 秒。开始时前 `m` 名同学同时接水，以后谁先接完，队伍中的下一名同学就立刻补上去。要求所有同学都完成接水的最早时刻。

### 分析

题目表面是模拟排队，真正的模型其实是“固定顺序的并行机器调度”：水龙头是机器，同学是按给定顺序到来的任务。顺序不能改，所以每来一个新同学，我们唯一能决定的，只是把他放到哪个最先空出来的水龙头上。

为什么一定要接到“当前最早空闲”的水龙头？因为下一名同学的开工时间完全取决于某个水龙头什么时候释放。如果明明有更早空出的龙头却故意不用，只会让这个同学结束得更晚，而且不会给后面带来任何补偿，所以这是一个很稳的局部最优选择。于是问题就被压成一个简单状态：每个水龙头当前会在什么时候空出来。

用小根堆维护这些“空闲时刻”即可。堆顶永远是最早空闲的龙头，弹出后加上当前同学的接水时间，再放回去，就是新的结束时刻。整个过程中堆始终保持“不遗漏任何一个正在工作的龙头的结束时间”这个不变性。最后所有同学分配完后，所有龙头上的最大结束时刻就是总用时。

### 核心代码

```cpp
priority_queue<int, vector<int>, greater<int>> q;
for(int i = 1; i <= m; i++) q.push(w[i]);
for(int i = m + 1; i <= n; i++){
  int t = q.top(); q.pop();
  q.push(t + w[i]);
}
int ans = 0;
while(!q.empty()) ans = max(ans, q.top()), q.pop();
cout << ans;
```

### 复杂度

时间复杂度是 `O(n\log m)`，空间复杂度是 `O(m)`。

## 3. [【10NOIP普及组】导弹拦截](http://ybt.ssoier.cn:8088/problem_show.php?pid=1951)
`NOIP 2010` `普及组`

### 题意

平面上有两套导弹拦截系统，位置固定。每套系统只能设定一个工作半径，只要导弹到系统的距离不超过半径，就能被该系统拦截。要求两套系统合起来覆盖全部导弹，并让当天代价 `r_1^2+r_2^2` 最小。

### 分析

题目一看到“半径平方和最小”，先不要被几何外表带偏。两套系统的位置固定，所以每颗导弹对系统一、系统二分别对应两个确定的值：`d_1` 和 `d_2`，也就是到两个系统的距离平方。真正要决定的是：每颗导弹交给哪一套系统负责。

接下来要抓住一个关键翻译：如果系统一的半径平方定成某个值 `R`，那么所有 `d_1 > R` 的导弹就绝不可能交给系统一，只能交给系统二；而所有 `d_1 <= R` 的导弹都可以顺手交给系统一，因为这样只会让系统二的压力更小，不会让 `r_1^2` 变大。于是最优划分一定可以表示成：按 `d_1` 从小到大排序后，某个前缀给系统一，剩下后缀给系统二。

这样模型就完全清楚了。设前缀最后一个导弹的 `d_1` 为系统一需要的半径平方，后缀里最大的 `d_2` 为系统二需要的半径平方，两者相加就是这个切分点的总代价。前缀值天然由排序后的当前位置给出，后缀最大值可以用一个后缀数组维护。枚举切分点时，状态含义非常明确：`i` 之前都由系统一覆盖，`i+1` 之后都由系统二覆盖。把所有切分点的代价取最小值，就是答案。

### 核心代码

```cpp
struct Node{ long long d1, d2; } a[N];
sort(a + 1, a + n + 1, [](const Node& x, const Node& y){ return x.d1 < y.d1; });
suf[n + 1] = 0;
for(int i = n; i >= 1; i--) suf[i] = max(suf[i + 1], a[i].d2);
long long ans = suf[1];
for(int i = 1; i <= n; i++) ans = min(ans, a[i].d1 + suf[i + 1]);
cout << ans;
```

### 复杂度

时间复杂度是 `O(n\log n)`，空间复杂度是 `O(n)`。

## 4. [【10NOIP普及组】三国游戏](http://ybt.ssoier.cn:8088/problem_show.php?pid=1952)
`NOIP 2010` `普及组`

### 题意

有 `N` 个武将，两人一组时有对应的默契值。你和计算机轮流选将，你先手；计算机每次都会拿走“当前最能破坏你下一步最强组合”的那个自由武将。最后双方各自从自己的武将里挑出默契值最大的二人组比大小。问你能否必胜；如果能，最大的可保证默契值是多少。

### 分析

这题最重要的不是搜索选将顺序，而是先把计算机策略对你第一步的影响看透。假设你第一手选了武将 `i`，此时你最想和 `i` 搭配的人，当然是与 `i` 默契值最大的那个武将。可计算机的规则正好会把这个“当前最危险的搭档”抢走，所以你不可能再保证拿到 `i` 的第一搭档。

那还能保证什么？因为你和计算机交替选，计算机抢走第一搭档之后，立刻轮到你行动。此时你只要把 `i` 剩余搭档里最强的那个人拿到手，这一对就已经锁在你自己手里，后面无论计算机怎么选，都不可能拆掉。所以若第一手选 `i`，你至少能保证得到“`i` 的第二大默契值”这么好的一个组合，同时也不可能保证超过它。

于是整题被压成一个非常简洁的结论：对每个武将 `i`，算出它连向其他人的第二大边权；你真正能保证的最优值就是这些“第二大值”里的最大者，因此答案一定存在，第一行恒输出 `1`，第二行输出这个最大值即可。

### 核心代码

```cpp
long long ans = 0;
for(int i = 1; i <= n; i++){
  long long mx1 = 0, mx2 = 0;
  for(int j = 1; j <= n; j++) if(i != j){
    long long x = w[i][j];
    if(x >= mx1) mx2 = mx1, mx1 = x;
    else if(x > mx2) mx2 = x;
  }
  ans = max(ans, mx2);
}
cout << 1 << "\\n" << ans;
```

### 复杂度

时间复杂度是 `O(n^2)`，空间复杂度是 `O(1)`（若边权边读边维护）。

# 二、提高组赛题

进入提高组后，题目的共同特点就变成了“局面不能只靠肉眼模拟”，必须把顺序、限制和可达性压成状态或结构。读这一节时，重点看每题里那个最核心的状态是怎样被定义出来的，以及为什么一旦这个状态成立，后面的转移就足以代表全局最优。

## 5. [【10NOIP提高组】机器翻译](http://ybt.ssoier.cn:8088/problem_show.php?pid=1859)
`NOIP 2010` `提高组`

### 题意

翻译软件有容量为 `M` 的内存，按文章顺序处理 `N` 个单词。若当前单词已在内存中，就直接翻译；否则要查一次外存词典，并把这个单词放进内存。若内存已满，就淘汰最早进入内存的单词。要求总共需要查词典多少次。

### 分析

题目已经把替换规则写得非常具体：不是最近最少使用，也不是按频率，而是“最早进入内存的那个先出去”。这说明模型本质上就是一个先进先出的缓存模拟。只要把“当前内存里有哪些单词”和“这些单词进入内存的先后顺序”维护好，整个过程就完全确定。

因此需要两个量。第一个是集合状态，表示某个单词现在是否还在内存里；第二个是队列顺序，表示一旦内存满了，下一个应该被淘汰的是谁。循环不变性是：队列从头到尾正好对应当前内存中单词的进入顺序，队首就是最早进入且尚未被淘汰的单词。

扫描文章时，若单词已在内存中，答案不变；若不在内存中，答案加一。若还有空位就直接入队，否则先弹出队首，再把新单词压入队尾。整篇文章处理完后，累积的缺页次数就是答案。

### 核心代码

```cpp
queue<int> q;
bool in[1005] = {};
int used = 0, ans = 0;
for(int i = 1; i <= n; i++){
  int x = a[i];
  if(in[x]) continue;
  ans++;
  if(used == m) in[q.front()] = 0, q.pop();
  else used++;
  q.push(x), in[x] = 1;
}
cout << ans;
```

### 复杂度

时间复杂度是 `O(n)`，空间复杂度是 `O(m)`。

## 6. [【10NOIP提高组】乌龟棋](http://ybt.ssoier.cn:8088/problem_show.php?pid=1860)
`NOIP 2010` `提高组`

### 题意

棋盘是一条长度为 `N` 的链，起点和每个格子上都有分数。你手里有若干张步长为 `1`、`2`、`3`、`4` 的卡片，每张卡片只能用一次，并且总步长保证刚好能走到终点。要求安排使用顺序，使沿途经过格子的分数总和最大。

### 分析

如果直接把“卡片使用顺序”当成状态，排列数会非常大，显然不可做。真正该观察的是：因为步长只有 `1` 到 `4` 四种，所以一个局面并不需要记录完整顺序，只需要知道每种卡片已经用了多少张。只要这四个数确定，乌龟当前位置也就唯一确定了。

这就是本题从条件到模型的关键翻译。设 `f[a][b][c][d]` 表示已经用了 `a` 张 `1` 卡、`b` 张 `2` 卡、`c` 张 `3` 卡、`d` 张 `4` 卡时，能够取得的最大得分。当前位置不是额外状态，而是由不变量 `pos = 1 + a + 2b + 3c + 4d` 唯一决定。于是转移就变成：当前状态的最后一步来自哪一类卡片，就从对应的前驱状态转移过来，再加上当前位置格子的分数。

因为所有卡片最终都会被用完，所以答案提取也非常干净：`f[cnt1][cnt2][cnt3][cnt4]` 就是终点状态的最优值。

### 核心代码

```cpp
f[0][0][0][0] = score[1];
for(int a = 0; a <= c1; a++)
for(int b = 0; b <= c2; b++)
for(int c = 0; c <= c3; c++)
for(int d = 0; d <= c4; d++){
  int pos = 1 + a + 2 * b + 3 * c + 4 * d;
  if(a) f[a][b][c][d] = max(f[a][b][c][d], f[a - 1][b][c][d] + score[pos]);
  if(b) f[a][b][c][d] = max(f[a][b][c][d], f[a][b - 1][c][d] + score[pos]);
  if(c) f[a][b][c][d] = max(f[a][b][c][d], f[a][b][c - 1][d] + score[pos]);
  if(d) f[a][b][c][d] = max(f[a][b][c][d], f[a][b][c][d - 1] + score[pos]);
}
cout << f[c1][c2][c3][c4];
```

### 复杂度

时间复杂度是 `O(c_1c_2c_3c_4)`，空间复杂度也是 `O(c_1c_2c_3c_4)`。

## 7. [【10NOIP提高组】关押罪犯](http://ybt.ssoier.cn:8088/problem_show.php?pid=1861)
`NOIP 2010` `提高组`

### 题意

有 `N` 名罪犯和 `M` 对仇恨关系，每对关系有一个怨气值。要把所有罪犯分到两座监狱里，使同一监狱内发生的冲突事件中，影响力最大的那个尽量小。要求输出这个最小可能值。

### 分析

题目说的是“把最大的冲突压到尽量小”，这类表述通常要先想阈值。假设我们规定：所有怨气值大于某个 `X` 的关系都绝不能落在同一监狱，那么这些关系就都变成了“两个端点必须分属两侧”的二分图约束。只要这些高于 `X` 的边还能同时满足，说明最大冲突还能继续往下压；一旦不能满足，答案就被卡在当前这条边上。

因此最自然的做法不是二分答案，而是直接把边按怨气值从大到小处理。处理到当前边 `(u,v,w)` 时，前面所有更大的边都已经被强制成“异侧”。此时若发现 `u` 和 `v` 已经被推到了同一边，就说明怨气值为 `w` 的这条边无论如何都躲不开，它就是最小可能的最大冲突值。

为了维护“异侧”关系，可以用扩展域并查集：`x` 和 `x+n` 分别表示罪犯 `x` 在两种对立阵营中的身份。加入约束“`u` 与 `v` 异侧”时，就把 `u` 并到 `v+n`，把 `v` 并到 `u+n`。整个结构的不变性是：并查集始终维护着当前所有已处理高权边的二分图关系。第一个产生矛盾的边权就是答案；若直到最后都不矛盾，答案就是 `0`。

### 核心代码

```cpp
sort(e + 1, e + m + 1, [](const Edge& a, const Edge& b){ return a.w > b.w; });
for(int i = 1; i <= 2 * n; i++) fa[i] = i;
for(int i = 1; i <= m; i++){
  int x = e[i].u, y = e[i].v;
  if(find(x) == find(y)){
    cout << e[i].w;
    return;
  }
  unite(x, y + n);
  unite(y, x + n);
}
cout << 0;
```

### 复杂度

时间复杂度是 `O(m\\alpha(n))`，空间复杂度是 `O(n)`。

## 8. [【10NOIP提高组】引水入城](http://ybt.ssoier.cn:8088/problem_show.php?pid=1862)
`NOIP 2010` `提高组`

### 题意

给定一个 `N×M` 的高程网格，只有第一行城市能建蓄水厂。水可以从有设施的高处沿公共边流向更低的相邻城市。要求第 `N` 行所有城市最终都能得到水：如果做不到，输出无法得到水的干旱区城市数；如果做得到，输出最少需要建多少个蓄水厂。

### 分析

先把题面条件翻成图：每个城市是一个点，只能向海拔更低的相邻城市连边。这样一来，水流方向是一个有向无环关系，题目就变成了“从第一行若干源点出发，能否覆盖最后一行全部点；若能，最少选几个源点”。所以整题其实分成两个子问题：可达性判断，以及最小源点覆盖。

先说可达性。只要从第一行所有城市出发，沿着“向更低处走”的边做一次搜索，就能知道最后一行哪些城市根本碰不到。若存在不可达城市，题目已经结束，因为无论建多少蓄水厂，它们都不可能被供水。

若全部可达，关键性质才出现：从同一个第一行城市出发，最终能到达的最后一行城市一定构成一个连续区间。原因在于路径只能在平面网格里向下扩展，不会出现“能到左边和右边，却永远跨不过中间列”的断裂。于是我们可以用记忆化深搜求出每个格子能覆盖到的最后一行区间 `[L,R]`：底边城市自己的区间就是 `[j,j]`，普通格子的区间则是所有更低相邻格子区间的并。

这样模型就彻底落成了区间覆盖。把第一行每个城市对应的区间取出来，问题变成：用最少的区间覆盖 `[1,M]`。经典贪心做法是每次在所有左端点不超过当前待覆盖位置的区间里，选右端点最远的那个。这里的关键状态是不变的“`cur` 左边已经全部覆盖”，每次把 `cur` 推到最远位置之后再继续，最终使用的区间数就是最少蓄水厂数。

### 核心代码

```cpp
void mark(int x, int y){
  if(reach[x][y]) return;
  reach[x][y] = 1;
  for(int k = 0; k < 4; k++){
    int nx = x + dx[k], ny = y + dy[k];
    if(in(nx, ny) && h[nx][ny] < h[x][y]) mark(nx, ny);
  }
}
void dfs(int x, int y){
  if(vis[x][y]) return;
  vis[x][y] = 1, L[x][y] = 1e9, R[x][y] = 0;
  if(x == n) L[x][y] = R[x][y] = y;
  for(int k = 0; k < 4; k++){
    int nx = x + dx[k], ny = y + dy[k];
    if(in(nx, ny) && h[nx][ny] < h[x][y]) dfs(nx, ny), L[x][y] = min(L[x][y], L[nx][ny]), R[x][y] = max(R[x][y], R[nx][ny]);
  }
}
for(int j = 1; j <= m; j++) mark(1, j), dfs(1, j);
int bad = 0;
for(int j = 1; j <= m; j++) bad += !reach[n][j];
if(bad){ cout << 0 << "\\n" << bad; return; }
for(int j = 1; j <= m; j++) seg.push_back({L[1][j], R[1][j]});
sort(seg.begin(), seg.end());
for(int i = 0, cur = 1; cur <= m; ans++){
  int far = 0;
  while(i < (int)seg.size() && seg[i].first <= cur) far = max(far, seg[i++].second);
  cur = far + 1;
}
cout << 1 << "\\n" << ans;
```

### 复杂度

时间复杂度是 `O(NM+M\log M)`，空间复杂度是 `O(NM)`。
