# 一本通 NOIP 2016赛题解题报告

这一组题从普及组的直接模拟、枚举与窗口维护一路走到提高组的树上统计、期望动态规划和状态压缩，表面上跨度很大，但读题主线其实很统一：先把条件翻成可操作的状态，再抓住那个真正决定复杂度的结构。前半段重在把规则写对，后半段则重在把路径、概率和集合压成可维护的模型。

# 一、普及组赛题

这一部分更像在练“条件翻译”的基本功：题面规则都不算深，但只有先把约束压成明确的公式、窗口或枚举对象，代码才会自然变短。

## 1. [【16NOIP普及组】买铅笔](http://ybt.ssoier.cn:8088/problem_show.php?pid=1973)

`模拟` `向上取整`

### 题意

给定需要的铅笔支数以及三种包装的“每包支数+每包价格”，并且只能整包购买且只能选一种包装，求买到不少于 `n` 支时的最小花费。

### 分析

题目条件先翻成决策模型：真正要选的不是“买几支”，而是“选哪一种包装”。因为包装种类只有三种，而且一旦选定某种包装，最少需要买多少包就是确定的，所以这题没有复杂搜索，核心是把“至少买到 `n` 支”改写成一个向上取整。

设某种包装每包有 `c` 支、每包价格为 `p`。为了覆盖至少 `n` 支，最少要买 `ceil(n / c)` 包，总花费就是 `ceil(n / c) * p`。这个式子成立的关键不变式是：对固定包装而言，包数越少一定不够，包数一旦达到这个上界就是最优。三种包装各算一次，最后取最小值，就是答案提取。

### 核心代码

```cpp
long long ans = (1LL << 62);
for (int i = 0; i < 3; ++i) {
    long long c, p;
    cin >> c >> p;
    long long need = (n + c - 1) / c;
    ans = min(ans, need * p);
}
cout << ans;
```

### 复杂度

时间复杂度 $O(1)$，空间复杂度 $O(1)$。

## 2. [【16NOIP普及组】回文日期](http://ybt.ssoier.cn:8088/problem_show.php?pid=1974)

`枚举` `日期判断`

### 题意

给出两个真实存在的日期 `date1` 和 `date2`，统计闭区间内有多少个日期写成 `yyyymmdd` 的八位数字后是回文串。

### 分析

先从回文条件倒推模型。八位日期是回文，说明后四位一定是前四位年份的倒序，所以一个候选日期一旦确定年份，月份和日子就已经被唯一确定。于是“枚举所有日期”可以缩成“枚举区间内所有年份，再把年份反过来拼出月份和日期”。

为什么这样不会漏？因为任意一个合法回文日期都必须满足这个镜像结构；为什么不会多算？因为拼出来的 `mmdd` 还要经过真实日期校验，闰年二月、大小月都要检查。状态只有当前年份，转移就是年份加一，答案提取则是在范围内且日期合法时把计数加一。

### 核心代码

```cpp
bool leap(int y){ return y % 400 == 0 || (y % 4 == 0 && y % 100); }
bool ok(int y,int m,int d){
    int days[] = {0,31,28,31,30,31,30,31,31,30,31,30,31};
    if (m < 1 || m > 12) return false;
    if (m == 2) days[m] += leap(y);
    return 1 <= d && d <= days[m];
}
for (int y = l / 10000; y <= r / 10000; ++y) {
    int t = y, md = 0;
    for (int k = 0; k < 4; ++k) md = md * 10 + t % 10, t /= 10;
    int x = y * 10000 + md, m = md / 100, d = md % 100;
    if (l <= x && x <= r && ok(y, m, d)) ++ans;
}
```

### 复杂度

时间复杂度 $O(Y)$，其中 $Y$ 是区间覆盖的年份数；空间复杂度 $O(1)$。

## 3. [【16NOIP普及组】海港](http://ybt.ssoier.cn:8088/problem_show.php?pid=1975)

`双指针` `计数维护`

### 题意

按时间顺序给出每艘船到港时刻和船上乘客国籍，要求在每艘船到达后，统计最近 `86400` 秒内所有到港乘客来自多少个不同国家。

### 分析

条件里最重要的是“时间递增”和“窗口长度固定为 24 小时”。这说明我们不必每次重新统计整段历史，而应把问题建成一个滑动窗口：右端随着当前船只推进，左端不断删掉已经离开窗口的乘客。

窗口里真正需要维护的是“每个国家当前出现了多少次”。某国计数从 `0` 变成 `1` 时，不同国家数加一；从 `1` 变成 `0` 时，不同国家数减一。队列里存的是还在窗口内的乘客 `(time, nation)`，队头始终是不变式中的最早乘客。每处理完一艘船并把过期乘客弹出后，当前不同国家总数就是这一时刻的答案。

### 核心代码

```cpp
queue<pair<int,int>> q;
for (int i = 1; i <= n; ++i) {
    cin >> t >> k;
    while (k--) {
        cin >> x;
        q.push({t, x});
        if (++cnt[x] == 1) ++tot;
    }
    while (!q.empty() && q.front().first <= t - 86400) {
        auto [tm, id] = q.front(); q.pop();
        if (--cnt[id] == 0) --tot;
    }
    cout << tot << '\n';
}
```

### 复杂度

时间复杂度 $O(\sum k_i)$，空间复杂度 $O(\sum k_i)$。

## 4. [【16NOIP普及组】魔法阵](http://ybt.ssoier.cn:8088/problem_show.php?pid=1976)

`值域统计` `前后缀和`

### 题意

每个物品有一个魔法值，若四个物品的数值满足 `x_a < x_b < x_c < x_d`、`x_b - x_a = 2(x_d - x_c)` 且 `x_b - x_a < (x_c - x_b) / 3`，就构成一个魔法阵。要求对每个物品分别统计它作为 `A/B/C/D` 出现的次数。

### 分析

这题先别盯着“每个物品”发慌，先把条件翻译成按数值的组合。设 `d = x_d - x_c`，则 `x_b - x_a = 2d`，再由不等式得到 `x_c >= x_b + 6d + 1`。于是一个合法四元组只要确定了差值 `d`，就变成：`A` 和 `B` 的值相隔 `2d`，`C` 和 `D` 的值相隔 `d`，并且 `C` 的起点落在某个后缀区间里。

因此真正可维护的状态不是“第几个物品”，而是“某个魔法值出现了多少次”。先统计每个值的出现次数 `cnt[v]`，再枚举 `d`。对于 `A/B`，需要的是后缀和 `suf[pos] = sum cnt[c] * cnt[c+d]`，它表示从某个起点往后能接多少组 `(C,D)`；对于 `C/D`，则用一个前缀累加量 `pre = sum cnt[a] * cnt[a+2d]`，表示当前位置之前能提供多少组 `(A,B)`。这样四个角色都能在一次线性扫描里拿到贡献。最后把按“魔法值”算出的四个答案，回填给拥有该值的每个物品，就是题目的输出。

### 核心代码

```cpp
for (int d = 1; d <= V; ++d) {
    for (int c = 1; c + d <= V; ++c) g[c] = 1LL * cnt[c] * cnt[c + d];
    for (int c = V; c >= 1; --c) suf[c] = suf[c + 1] + g[c];
    for (int a = 1; a + 2 * d <= V; ++a) {
        int b = a + 2 * d, lim = a + 8 * d + 1;
        if (lim <= V - d) {
            long long ways = suf[lim];
            A[a] += 1LL * cnt[b] * ways;
            B[b] += 1LL * cnt[a] * ways;
        }
    }
    long long pre = 0;
    for (int c = 1; c + d <= V; ++c) {
        int a = c - 8 * d - 1;
        if (a >= 1) pre += 1LL * cnt[a] * cnt[a + 2 * d];
        C[c] += pre * cnt[c + d];
        D[c + d] += pre * cnt[c];
    }
}
```

### 复杂度

设魔法值上界为 $V$。时间复杂度 $O(V^2)$，空间复杂度 $O(V)$。

# 二、提高组赛题

提高组开始不再满足于直接模拟，很多题都要先抓住树路径、概率状态、值域结构或集合覆盖，再把原题改写成能批量统计的形式。

## 5. [【16NOIP提高组】玩具谜题](http://ybt.ssoier.cn:8088/problem_show.php?pid=1893)

`环形模拟` `方向判断`

### 题意

有 `n` 个玩具小人围成一圈，已知每个人朝内还是朝外以及职业名称。指令依次给出“向左数/向右数多少人”，但左右方向会随当前小人的朝向改变。求从第一个小人出发，执行完全部指令后停在哪个职业。

### 分析

先把文字规则压成一个“符号判断”。题目按逆时针给出小人顺序，所以我们只需要决定当前指令对应的是顺着数组下标加，还是减。若当前小人朝内，那么它的左边其实是顺时针，右边是逆时针；若朝外则相反。把这四种情况列成表，就能发现：`朝向 xor 指令方向` 为 `1` 时沿逆时针正向走，否则反向走。

这样模型就从“读懂左右”变成了一个简单的环形数组移动。状态只有当前所处编号，转移是 `(pos ± step) mod n`。整个过程中职业字符串只在最后答案提取时使用：循环处理完所有指令后，输出当前位置对应的名称即可。

### 核心代码

```cpp
int pos = 0;
for (int i = 1; i <= m; ++i) {
    int op, s;
    cin >> op >> s;
    if ((dir[pos] ^ op) == 1) pos = (pos + s) % n;
    else pos = (pos - s % n + n) % n;
}
cout << name[pos];
```

### 复杂度

时间复杂度 $O(m)$，空间复杂度 $O(n)$。

## 6. [【16NOIP提高组】天天爱跑步](http://ybt.ssoier.cn:8088/problem_show.php?pid=1894)

`树上差分` `最近公共祖先` `桶计数`

### 题意

树上有 `m` 个玩家同时从 `S_i` 沿最短路跑向 `T_i`，每个结点 `x` 的观察员会在时刻 `W_x` 观察，问每个结点能看到多少玩家恰好在这一秒到达这里。

### 分析

条件先拆成“路径经过”与“到达时刻匹配”两部分。若结点 `x` 在路径 `S -> T` 的 `S -> lca` 段上，那么到达 `x` 的时间是 `dep[S] - dep[x]`，也就是 `dep[S] = dep[x] + W[x]`；若 `x` 在 `lca -> T` 的下半段上，那么到达时间是 `dep[S] + dep[x] - 2dep[lca]`，可改写成 `dep[S] - 2dep[lca] = W[x] - dep[x]`。这一步就是从条件到模型的关键翻译。

接下来用两套树上差分分别维护两种路径段。第一套把每条路径的上半段转成“在 `S` 加，在 `fa(lca)` 减”，并用桶统计键值 `dep[S]`；第二套把下半段转成“在 `T` 加，在 `lca` 减”，并统计键值 `dep[S] - 2dep[lca]`。后序 DFS 时，子树汇总后的桶里保存的就是“会经过当前点的所有路径键值”，这是整个算法的不变式。当前点只需查询两只桶中与自己条件匹配的位置，再把两部分相加，就是该观察员的答案。

### 核心代码

```cpp
void dfs(int x, int f) {
    int old1 = c1[dep[x] + w[x]];
    int old2 = c2[w[x] - dep[x] + OFF];
    for (int y : g[x]) if (y != f) dep[y] = dep[x] + 1, dfs(y, x);
    for (int v : add1[x]) ++c1[v];
    for (int v : sub1[x]) --c1[v];
    for (int v : add2[x]) ++c2[v + OFF];
    for (int v : sub2[x]) --c2[v + OFF];
    ans[x] = c1[dep[x] + w[x]] - old1 + c2[w[x] - dep[x] + OFF] - old2;
}
int l = lca(s, t);
add1[s].push_back(dep[s]);
if (fa[l]) sub1[fa[l]].push_back(dep[s]);
int key = dep[s] - 2 * dep[l];
add2[t].push_back(key), sub2[l].push_back(key);
```

### 复杂度

预处理最近公共祖先为 $O(n\log n)$，整套统计为 $O((n+m)\log n)$，空间复杂度 $O(n+m)$。

## 7. [【16NOIP提高组】换教室](http://ybt.ssoier.cn:8088/problem_show.php?pid=1895)

`最短路` `概率期望` `动态规划`

### 题意

每个时间段原定在 `c_i` 上课，也可申请换到 `d_i`，第 `i` 次申请独立地以概率 `k_i` 成功，且总共最多申请 `m` 次。要求最小化整学期在相邻课程之间移动体力的期望值。

### 分析

先看图上移动：不论怎么申请，真正影响期望的都只是教室之间的最短距离，所以第一步要用 Floyd 预处理任意两间教室的最短路。之后题目才回到“在哪些课上提交申请”的决策。

动态规划的关键状态是 `f[i][j][0/1]`：处理完前 `i` 节课、用了 `j` 次申请后，第 `i` 节课是否提交了申请。之所以只要这两个状态，是因为从第 `i-1` 节转移到第 `i` 节时，期望只依赖于前一节和当前节可能落在哪两间教室。若前后两节都申请了，就把四种成功/失败组合按独立概率展开；若某一节没申请，对应的教室就是确定的。转移完成后，在 `i=n` 的所有 `j\le m` 状态里取最小值，就是最终答案。

### 核心代码

```cpp
for (int k = 1; k <= v; ++k)
  for (int i = 1; i <= v; ++i)
    for (int j = 1; j <= v; ++j)
      dis[i][j] = min(dis[i][j], dis[i][k] + dis[k][j]);
for (int i = 2; i <= n; ++i)
  for (int j = 0; j <= min(i, m); ++j) {
    f[i][j][0] = min(f[i-1][j][0] + dis[c[i-1]][c[i]],
                     f[i-1][j][1] + (1-p[i-1]) * dis[c[i-1]][c[i]] + p[i-1] * dis[d[i-1]][c[i]]);
    if (j) f[i][j][1] = min(
        f[i-1][j-1][0] + (1-p[i]) * dis[c[i-1]][c[i]] + p[i] * dis[c[i-1]][d[i]],
        f[i-1][j-1][1] + (1-p[i-1]) * (1-p[i]) * dis[c[i-1]][c[i]]
        + (1-p[i-1]) * p[i] * dis[c[i-1]][d[i]]
        + p[i-1] * (1-p[i]) * dis[d[i-1]][c[i]]
        + p[i-1] * p[i] * dis[d[i-1]][d[i]]);
  }
```

### 复杂度

时间复杂度 $O(v^3 + nm)$，空间复杂度 $O(nm)$。

## 8. [【16NOIP提高组】组合数问题](http://ybt.ssoier.cn:8088/problem_show.php?pid=1896)

`杨辉三角` `前缀和`

### 题意

给定常数 `k` 和多组询问 `(n,m)`，每次要求统计所有 `0 \le i \le n`、`0 \le j \le min(i,m)` 中，有多少个组合数 `C_i^j` 是 `k` 的倍数。

### 分析

多组询问提示我们必须先把“是否被 `k` 整除”预处理出来。组合数最自然的模型就是杨辉三角，因为 `C_i^j = C_{i-1}^{j-1} + C_{i-1}^j`，而我们只关心它对 `k` 的余数，所以整张三角形可以直接在模 `k` 意义下递推。

得到 `c[i][j] mod k` 后，再把“是 `k` 的倍数”记成一个 `0/1` 矩阵。询问区间其实是一个左上角矩形，只是每一行超过 `i` 的位置本来就不存在，我们把这些位置视作 `0` 即可。于是再做一遍二维前缀和，`sum[n][m]` 就正好是题目要的计数。状态是杨辉三角上的余数，转移是两格相加取模，答案提取是直接查前缀和表。

### 核心代码

```cpp
c[0][0] = 1 % k;
for (int i = 1; i <= N; ++i) {
    c[i][0] = 1 % k;
    for (int j = 1; j <= i; ++j) c[i][j] = (c[i-1][j-1] + c[i-1][j]) % k;
}
for (int i = 0; i <= N; ++i)
  for (int j = 0; j <= N; ++j) {
    int bad = (j <= i && c[i][j] == 0);
    s[i+1][j+1] = s[i][j+1] + s[i+1][j] - s[i][j] + bad;
  }
cout << s[n+1][min(n, m)+1] << '\n';
```

### 复杂度

预处理时间复杂度 $O(N^2)$，单次询问 $O(1)$，空间复杂度 $O(N^2)$。

## 9. [【16NOIP提高组】蚯蚓](http://ybt.ssoier.cn:8088/problem_show.php?pid=1897)

`三个队列` `懒增量`

### 题意

每秒取当前最长蚯蚓切成两段，其余蚯蚓统一增长 `q`。要求输出每隔 `t` 秒被切断前的长度，以及 `m` 秒后所有蚯蚓按从大到小排序后每隔 `t` 个位置的长度。

### 分析

朴素大根堆会被每秒“全体加 `q`”拖垮，所以要先把这条条件抽成一个全局增量 `delta`。把所有蚯蚓都视为“存储值 + delta”，每过一秒只需让 `delta += q`，就等价于给所有旧蚯蚓统一加长，而不用真的逐个修改。

切开的两段为什么还能用队列维护？因为每次被切的是当前最大值，而按同一个比例切开后得到的两段长度形成两条各自单调不增的序列。这样初始数组排好序后，再配合两条“新生成队列”，每次只要比较三个队首就能得到当前最大蚯蚓。状态是不含公共增量的真实基值，转移是弹出最大值、按比例拆成两段并减去新的 `delta` 后入队。第一问在第 `t`、`2t` 秒输出刚弹出的值；第二问则在模拟结束后继续三路归并，按排名取答案。

### 核心代码

```cpp
sort(a + 1, a + n + 1, greater<int>());
for (int i = 1; i <= n; ++i) q0.push(a[i]);
long long delta = 0;
auto top = [&](queue<long long>& q){ return q.empty() ? -(1LL<<60) : q.front() + delta; };
for (int i = 1; i <= m; ++i) {
    long long x = max({top(q0), top(q1), top(q2)}); pop_max(x);
    if (i % t == 0) print(x);
    long long a = x * u / v, b = x - a;
    delta += q;
    q1.push(a - delta), q2.push(b - delta);
}
while (!q0.empty() || !q1.empty() || !q2.empty()) {
    long long x = max({top(q0), top(q1), top(q2)}); pop_max(x);
    if (++rk % t == 0) print(x);
}
```

### 复杂度

时间复杂度 $O(n + m)$，空间复杂度 $O(n + m)$。

## 10. [【16NOIP提高组】愤怒的小鸟](http://ybt.ssoier.cn:8088/problem_show.php?pid=1898)

`状态压缩` `抛物线枚举`

### 题意

每只小鸟的轨迹必须是 `y=ax^2+bx` 且 `a<0` 的抛物线。给定最多 `18` 只小猪的坐标，求最少发射多少只小鸟才能消灭全部小猪。

### 分析

`n \le 18` 直接提醒我们往状态压缩想。真正的建模关键是：一条合法抛物线只要经过原点，再额外知道两只小猪，就能唯一确定；若这条抛物线开口不向下，就根本不合法。因此可以先枚举两只小猪，解出对应的 `a,b`，再把所有落在这条轨迹上的小猪打成一个二进制集合 `mask`。

预处理完“发一只鸟最多能打掉哪些猪”后，原题就变成集合覆盖的最少步数。设 `f[S]` 为已经消灭集合 `S` 时的最少小鸟数，每次从 `S` 中找第一只还没被消灭的小猪 `i`，枚举所有经过 `i` 的有效抛物线，把新集合更新为 `S | hit[i][j]`。这个转移之所以正确，是因为最优解里第一只尚未消灭的小猪总得被某一发小鸟覆盖。最终 `f[(1<<n)-1]` 就是答案。

### 核心代码

```cpp
for (int i = 0; i < n; ++i) hit[i][i] = 1 << i;
for (int i = 0; i < n; ++i) for (int j = i + 1; j < n; ++j) {
    double a = (y[i] / x[i] - y[j] / x[j]) / (x[i] - x[j]);
    if (a >= -eps) continue;
    double b = y[i] / x[i] - a * x[i];
    for (int k = 0; k < n; ++k)
        if (fabs(a * x[k] * x[k] + b * x[k] - y[k]) < eps) hit[i][j] |= 1 << k;
    hit[j][i] = hit[i][j];
}
int ALL = (1 << n) - 1;
fill(f, f + (1 << n), INF), f[0] = 0;
for (int S = 0; S < ALL; ++S) {
    int i = 0; while (S >> i & 1) ++i;
    f[S | (1 << i)] = min(f[S | (1 << i)], f[S] + 1);
    for (int j = 0; j < n; ++j) f[S | hit[i][j]] = min(f[S | hit[i][j]], f[S] + 1);
}
```

### 复杂度

预处理时间复杂度 $O(n^3)$，状态压缩动态规划复杂度 $O(n^2 2^n)$，空间复杂度 $O(n2^n)$。
