# 一本通 字符串算法提高专题精选解题报告

这一组题从字符串哈希一路走到 KMP、Trie 与 AC 自动机，题面里有查重、周期、前缀冲突、删词和多模式匹配，真正贯穿始终的却是一件事：把“字符比较”翻译成可复用的状态。前半段更偏向局部等价与区间判断，后半段则把单串信息折进树和自动机里，开始一次性处理整批模式串。

# 一、哈希与等价关系建模

这一章先把“两个片段是不是一样”这件事练熟。很多题的难点都不在实现，而在于先把原问题改写成前后缀相等、循环节成立、窗口内元素互异或状态是否重复。

## 1. [1455：【例题1】Oulipo](http://ybt.ssoier.cn:8088/problem_show.php?pid=1455)

`字符串哈希` `子串计数`

### 题意

给定多组字符串 $s_1,s_2$，要求统计模式串 $s_1$ 在文本串 $s_2$ 中一共出现多少次，重叠出现也要计算。

### 分析

题目本质是在长串里反复判断很多个长度固定的窗口是否等于同一个模式串。只要把“整段是否相等”压成哈希值，问题就从逐字符比较变成了区间取值。

因此先求出模式串的整串哈希，再枚举文本串里每个长度为 $|s_1|$ 的起点，用前缀哈希在 $O(1)$ 时间取出该窗口的哈希值。窗口和模式哈希相等，就说明这里出现了一次。重叠匹配不会带来额外麻烦，因为所有起点都被完整枚举了一遍。

### 核心代码

```cpp
using ull = unsigned long long;
ull h[N], pw[N];
ull get(int l, int r){ return h[r] - h[l - 1] * pw[r - l + 1]; }
int solve(string p, string s){
    int m = p.size(), n = s.size(), ans = 0;
    ull hp = 0;
    for(char c : p) hp = hp * 131 + c;
    for(int i = 1; i <= n; ++i) h[i] = h[i - 1] * 131 + s[i - 1], pw[i] = pw[i - 1] * 131;
    for(int l = 1; l + m - 1 <= n; ++l) if(get(l, l + m - 1) == hp) ++ans;
    return ans;
}
```

### 复杂度

时间复杂度 $O(|s_1|+|s_2|)$，空间复杂度 $O(|s_2|)$。

## 2. [1456：【例题2】图书管理](http://ybt.ssoier.cn:8088/problem_show.php?pid=1456)

`哈希表` `字符串集合`

### 题意

维护一个图书系统，支持 `add(s)` 把书名加入馆藏，`find(s)` 查询当前是否存在这本书。书名区分大小写。

### 分析

题目没有顺序关系，也不需要前缀匹配，真正要维护的只是“某个字符串是否出现过”。这时就不该上复杂字符串结构，而是直接把书名放进哈希表。

`add` 就是插入，`find` 就是查询是否存在。因为操作数只有 $3\times 10^4$，而单次查询只关心精确相等，所以用哈希集合正好把题意完整落地。

### 核心代码

```cpp
unordered_set<string> st;
while(n--){
    string op, s;
    cin >> op >> s;
    if(op == "add") st.insert(s);
    else cout << (st.count(s) ? "yes" : "no") << '\n';
}
```

### 复杂度

时间复杂度均摊 $O(n)$，空间复杂度 $O(n\cdot L)$，其中 $L$ 为书名长度上界。

## 3. [1457：Power Strings](http://ybt.ssoier.cn:8088/problem_show.php?pid=1457)

`字符串哈希` `循环节`

### 题意

对每个输入字符串，求它最多能写成多少个相同子串首尾相接的形式，例如 `ababab` 可以写成 $3$ 个 `ab`。

### 分析

要判断一个串是不是由某个短块重复得到，核心不是去模拟拼接，而是先猜一个块长 $d$，再验证所有长度为 $d$ 的块是否都和第一块一样。题目问“最多几段”，等价于问“最短循环节有多长”。

因此只需要枚举长度 $n$ 的因子 $d$，用哈希在 $O(1)$ 时间比较每一段和首段是否相等。第一个通过检查的最小块长就是最短循环节，答案便是 $n/d$。

### 核心代码

```cpp
bool ok(int d, int n){
    for(int l = d + 1; l <= n; l += d)
        if(get(l, l + d - 1) != get(1, d)) return false;
    return true;
}
int solve(int n){
    for(int d = 1; d <= n; ++d){
        if(n % d) continue;
        if(ok(d, n)) return n / d;
    }
    return 1;
}
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(n)$。

## 4. [1458：Seek the Name, Seek the Fame](http://ybt.ssoier.cn:8088/problem_show.php?pid=1458)

`字符串哈希` `border`

### 题意

对每个字符串，输出所有既是前缀又是后缀的子串长度，并按从小到大排列。

### 分析

这题要先认出要找的是整串的所有 `border`。如果还在想“把前缀和后缀逐个拿出来比较”，就会停在 $O(n^2)$；而一旦意识到它只是大量前缀与后缀的等价判断，哈希就能把比较压到常数时间。

于是枚举长度 $l$，判断前缀 $[1,l]$ 和后缀 $[n-l+1,n]$ 的哈希是否相等。相等的长度全部收集起来即可。题目只要求长度，不涉及出现位置，所以这种按长度直接扫的建模最自然。

### 核心代码

```cpp
vector<int> ans;
for(int l = 1; l <= n; ++l){
    if(get(1, l) == get(n - l + 1, n)) ans.push_back(l);
}
for(int x : ans) cout << x << ' ';
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 5. [1459：friends](http://ybt.ssoier.cn:8088/problem_show.php?pid=1459)

`字符串哈希` `构造判定`

### 题意

给定一个大写字母串 $U$，它来自两个相同串拼接后在某一半里多插入了一个字符。要求恢复原串 $S$；若不存在输出 `NOT POSSIBLE`，若不唯一输出 `NOT UNIQUE`。

### 分析

长度关系先给出第一层筛选：$|U|$ 必须是奇数。设 $m=\lfloor |U|/2 \rfloor$，额外字符只可能落在左半边或右半边，所以候选解只会来自这两个方向。

关键不在暴力删点，而在把“删掉一个字符后两半是否相等”改成两段拼接后的哈希比较。比如假设多出的字符在左边，就枚举左半要删的位置，检查“左半删掉该字符后的串”是否等于右半；右边同理。最终把所有能成立的候选串去重后判断是零个、一个还是多个。

### 核心代码

```cpp
bool same_after_erase_left(int p){
    return cat(get(1, p - 1), get(p + 1, m + 1)) == get(m + 2, 2 * m + 1);
}
set<string> cand;
for(int p = 1; p <= m + 1; ++p) if(same_after_erase_left(p)) cand.insert(build_left(p));
for(int p = m + 1; p <= 2 * m + 1; ++p) if(same_after_erase_right(p)) cand.insert(build_right(p));
```

### 复杂度

时间复杂度 $O(n)$ 到 $O(n\log n)$，空间复杂度 $O(n)$。

## 6. [1460：A Horrible Poem](http://ybt.ssoier.cn:8088/problem_show.php?pid=1460)

`字符串哈希` `循环节`

### 题意

给定一个长串 $S$ 和大量询问，每次询问子串 $S[a..b]$ 的最短循环节长度。

### 分析

单次询问若从 $1$ 开始枚举循环节，会被 $2\times 10^6$ 次询问直接压垮。这题真正要想的是：一个区间的最短循环节一定是区间长度的因子，所以需要缩的不是字符比较，而是候选长度个数。

设当前区间长度为 $len$。先把 $len$ 分解质因数，再尝试把答案从 $len$ 往小除：如果当前候选周期为 $d/p$，只需检查区间前后错开这个长度后的两段是否完全相等，即可判断能不能继续缩。哈希负责做区间相等判断，因子分解负责减少尝试次数。

### 核心代码

```cpp
bool check(int l, int r, int d){
    return get(l, r - d) == get(l + d, r);
}
int solve(int l, int r){
    int ans = r - l + 1;
    for(int p : fac[ans]){
        while(ans % p == 0 && check(l, r, ans / p)) ans /= p;
    }
    return ans;
}
```

### 复杂度

单次询问时间复杂度为区间长度质因子个数的线性量级，整体空间复杂度 $O(n)$。

## 7. [1461：Beads](http://ybt.ssoier.cn:8088/problem_show.php?pid=1461)

`字符串哈希` `分块统计`

### 题意

给定一串珠子颜色。选择一个块长 $k$，把序列按长度 $k$ 切成若干块，最后不足 $k$ 的尾块舍去。若两个块互为翻转也算同一种，求能得到不同块种数最多的所有 $k$。

### 分析

题目表面在找“最优 $k$”，但一旦固定 $k$，任务就变成了：把每个长度为 $k$ 的整块映射成一个可比较的签名，再统计不同签名个数。由于翻转视为同一种，单块的代表值应该取“正向哈希”和“反向哈希”中的较小者。

接着再看复杂度：对每个 $k$，块数是 $\lfloor n/k \rfloor$，于是总工作量是 $\sum_{k=1}^n \lfloor n/k \rfloor = O(n\log n)$。也就是说，不需要更玄的结构，直接枚举 $k$ 并用哈希统计就够了。

### 核心代码

```cpp
int best = 0;
vector<int> ks;
for(int k = 1; k <= n; ++k){
    unordered_set<ull> st;
    for(int l = 1; l + k - 1 <= n; l += k){
        ull a = get_fwd(l, l + k - 1);
        ull b = get_rev(l, l + k - 1);
        st.insert(min(a, b));
    }
    int cur = st.size();
    if(cur > best) best = cur, ks = {k};
    else if(cur == best) ks.push_back(k);
}
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(n)$。

## 8. [1462：Antisymmetry](http://ybt.ssoier.cn:8088/problem_show.php?pid=1462)

`字符串哈希` `二分半径`

### 题意

给定一个 $01$ 串，统计有多少个子串满足“翻转后再把每一位取反”仍与原串相同，也就是反对称子串的个数。

### 分析

反对称的定义看着别扭，但翻译一下就很顺：子串左半段必须等于右半段反转取反后的结果，所以合法长度只能是偶数。于是每个答案都围绕某个相邻字符之间的中心展开。

对一个中心，设半径为 $r$，只要左边长度为 $r$ 的串和右边长度为 $r$ 的串在“反转取反”后相等，这个半径就成立。把原串反转并取反后再做一份哈希，就能用二分找出每个中心的最大半径，所有半径之和就是答案。

### 核心代码

```cpp
long long ans = 0;
for(int c = 1; c < n; ++c){
    int l = 0, r = min(c, n - c);
    while(l < r){
        int mid = (l + r + 1) >> 1;
        if(ok(c - mid + 1, c, c + 1, c + mid)) l = mid;
        else r = mid - 1;
    }
    ans += l;
}
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(n)$。

## 9. [1463：门票](http://ybt.ssoier.cn:8088/problem_show.php?pid=1463)

`哈希表` `函数迭代`

### 题意

定义数列 $a_0=1$，$a_{i+1}=(A\times a_i+a_i\bmod B)\bmod C$。求第一次出现重复项的位置；若超过 $2\times 10^6$ 还没重复，输出 $-1$。

### 分析

虽然它不再是字符串题，但建模核心和哈希表一模一样：状态只有当前值，而下一项由当前值唯一决定，所以一旦某个值再次出现，后面序列就会完全循环。

因此直接顺着递推往下生成，同时用哈希表记录每个值第一次出现的位置。新值第一次见就写入表，若已经见过，当前下标就是题目要的“第一次出现重复项的位置”。

### 核心代码

```cpp
unordered_map<long long, int> pos;
long long x = 1;
pos[x] = 0;
for(int i = 1; i <= 2000000; ++i){
    x = (A * x + x % B) % C;
    if(pos.count(x)){ cout << i; return; }
    pos[x] = i;
}
cout << -1;
```

### 复杂度

时间复杂度均摊 $O(\min(2\times 10^6,\text{答案}))$，空间复杂度同阶。

## 10. [1464：收集雪花](http://ybt.ssoier.cn:8088/problem_show.php?pid=1464)

`双指针` `哈希表`

### 题意

给出每个时刻落下的雪花形状，要求选一个连续时间段，使得其中没有重复形状，并让收集到的雪花数量最大。

### 分析

题目问的是“连续段里元素两两不同”的最长长度，这种限制天然对应滑动窗口。真正要维护的不是具体集合，而是每种雪花最近一次出现的位置。

枚举右端点时，若当前形状在窗口里已经出现过，就把左端点直接跳到它上次出现位置的后一位。这样每个元素只会进窗、出窗各一次，哈希表负责记最近位置，窗口长度随时都能更新。

### 核心代码

```cpp
unordered_map<int, int> last;
int ans = 0;
for(int l = 1, r = 1; r <= n; ++r){
    if(last.count(a[r])) l = max(l, last[a[r]] + 1);
    last[a[r]] = r;
    ans = max(ans, r - l + 1);
}
cout << ans;
```

### 复杂度

时间复杂度均摊 $O(n)$，空间复杂度 $O(n)$。

# 二、KMP 与前后缀结构

这一章的共同点是都在利用“失配后还能保留多少已经匹配的信息”。一旦把前后缀关系抽成 failure 链，很多看似要重扫的过程都能在线完成。

## 11. [1465：【例题1】剪花布条](http://ybt.ssoier.cn:8088/problem_show.php?pid=1465)

`KMP` `不重叠匹配`

### 题意

给定花布条和小饰条，要求从前者中尽可能剪出多少块后者。每次剪下一块后不能和别的块重叠。

### 分析

如果只问出现次数，那就是普通匹配；这题多出的条件是“不重叠”。不过贪心很好理解：从左到右扫描时，只要当前位置已经形成一个完整匹配，立刻剪掉它一定不亏，因为任何再往后的方案都不可能让这个匹配和之前部分产生更好重叠。

于是还是先建模式串的前缀函数，再跑 KMP。唯一的区别是匹配成功后把模式指针直接清零，而不是退到 `pi[m-1]`，这样后续匹配就不会和当前这一段交叠。

### 核心代码

```cpp
build_pi(p);
int ans = 0;
for(int i = 0, j = 0; i < n; ++i){
    while(j && s[i] != p[j]) j = pi[j - 1];
    if(s[i] == p[j]) ++j;
    if(j == m){
        ++ans;
        j = 0;
    }
}
cout << ans;
```

### 复杂度

时间复杂度 $O(n+m)$，空间复杂度 $O(m)$。

## 12. [1466：【例题2】Power Strings](http://ybt.ssoier.cn:8088/problem_show.php?pid=1466)

`KMP` `循环节`

### 题意

对每个字符串，求它最多能由多少个相同子串拼接而成。

### 分析

KMP 在这里最关键的不是匹配文本，而是 `pi[n-1]` 这条信息：它给出了整串最长的相等真前后缀长度。若设最短循环节长度为 $p$，就有 $p=n-\pi[n-1]$。

但能否真的由这个周期整除，还要看 $n$ 是否是 $p$ 的整数倍。如果可以，答案就是 $n/p$；否则说明虽然有 border，但没法整齐铺满整串，只能把整个串当成一段。

### 核心代码

```cpp
build_pi(s);
int n = s.size();
int p = n - pi[n - 1];
if(n % p == 0) cout << n / p;
else cout << 1;
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 13. [1467：Radio Transmission](http://ybt.ssoier.cn:8088/problem_show.php?pid=1467)

`KMP` `最小周期`

### 题意

给定一个长度为 $L$ 的字符串，它是某个更短字符串不断自我连接后的一个子串。求这个原始循环串可能的最短长度。

### 分析

这里和整齐重复不同，末尾不要求刚好对齐，所以不再需要整除条件。真正要抓住的是：如果一个串来自某个循环串，那么它自己的前后缀重合部分越长，说明能支撑它的基础周期越短。

于是仍然先求前缀函数。最长 border 长度为 `pi[n-1]`，则最短可行周期长度就是 $n-\pi[n-1]$。这一回即使 $n$ 不是它的倍数也没有关系，因为题目允许当前串只是无限循环串的一个截断片段。

### 核心代码

```cpp
build_pi(s);
cout << (int)s.size() - pi[s.size() - 1];
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 14. [1468：OKR-Periods of Words](http://ybt.ssoier.cn:8088/problem_show.php?pid=1468)

`KMP` `failure链`

### 题意

给定一个字符串，设每个前缀的最大周期是它最长的周期串长度。要求求出所有前缀的最大周期长度之和。

### 分析

先把“周期”翻译成 border：若一个长度为 $i$ 的前缀存在某个 border 长度为 $b$，那么长度 $i-b$ 的前缀就能作为它的周期串。题目要最大周期，也就是希望 $i-b$ 尽量大，等价于让正 border 长度 $b$ 尽量短。

而所有 border 正好沿着 failure 链一层层往上跳。所以对每个前缀，只要知道它 failure 链上最短的正 border 是谁，答案就是 `i - shortest_border`；若不存在正 border，贡献为 $0$。这个“最短正 border”可以沿着 failure 链递推压出来，不需要每次都重新往上爬。

### 核心代码

```cpp
build_pi(s);
long long ans = 0;
for(int i = 1; i <= n; ++i){
    int f = pi[i - 1];
    if(!f) continue;
    mn[i] = mn[f] ? mn[f] : f;
    ans += i - mn[i];
}
cout << ans;
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 15. [1469：似乎在梦中见过的样子](http://ybt.ssoier.cn:8088/problem_show.php?pid=1469)

`LCP` `字符串计数`

### 题意

给定字符串 $S$ 和下界 $k$，统计有多少个子串能写成 $A+B+A$ 的形式，其中 $|A|\ge k, |B|\ge 1$。

### 分析

这题最容易卡在“怎么枚举所有子串”。真正该枚举的不是整段，而是两份相同的 $A$ 之间的相对位置：只要知道左边一份 $A$ 和右边一份 $A$ 的起点差，就能用它们周围还能延伸多远来统一统计一大批答案。

因此常见做法是对一对起点求它们的最长公共前缀和最长公共后缀。前者告诉我们右边还能向后扩多少，后者告诉我们左边还能向前补多少，于是所有满足 $|A|\ge k$ 且中间至少留出一位给 $B$ 的方案都会落成一段连续计数。这里的核心不是暴力找每个 $A$，而是把“两个相同块”压成 LCP/LCS 的区间问题。

### 核心代码

```cpp
long long ans = 0;
for(int i = 1; i <= n; ++i)
    for(int j = i + k + 1; j <= n; ++j){
        int lcp = LCP(i, j), lcs = LCS(i, j);
        int L = max(k, j - i - lcs + 1);
        int R = min(lcp, j - i - 1);
        if(L <= R) ans += R - L + 1;
    }
cout << ans;
```

### 复杂度

配合哈希或扩展匹配后，时间复杂度通常为 $O(n^2)$ 量级，空间复杂度 $O(n)$。

## 16. [1470：Censoring](http://ybt.ssoier.cn:8088/problem_show.php?pid=1470)

`在线删除` `自动机`

### 题意

给定原串 $S$ 和若干屏蔽词，从左到右扫描字符串；每当当前串里出现某个屏蔽词，就立刻删掉它，再继续重复，直到不存在任何屏蔽词。

### 分析

这题的难点是“删完以后会不会又在新边界处形成新的匹配”。所以不能先把所有匹配位置离线找完，而要在线维护当前保留下来的答案串。

把所有屏蔽词建成自动机，然后扫描原串时把“当前自动机状态”和“当前答案长度”一起压栈。每加一个字符都沿转移走一步；如果走到某个单词结尾，就把这段长度直接从答案栈里弹掉，同时把状态回退到弹完后的栈顶。这样删除后的继续匹配就自然接上了。

### 核心代码

```cpp

    out.push_back(c), st.push_back(u);
    if(len[u]){
        for(int k = 0; k < len[u]; ++k) out.pop_back(), st.pop_back();
    }
}
for(char c : out) cout << c;
```

### 复杂度

时间复杂度 $O(|S|+\sum |t_i|)$，空间复杂度同阶。

## 18. [1472：【例题2】The XOR Largest Pair](http://ybt.ssoier.cn:8088/problem_show.php?pid=1472)

`Trie` `异或最值`

### 题意

从给定的 $N$ 个整数中选两个，令它们按位异或，求能得到的最大结果。

### 分析

异或最大值的思路要从二进制高位出发：高位一旦做成 `1`，收益永远比低位更重要。因此最自然的结构不是排序，而是按二进制位往下分叉的 01-Trie。

把所有数依次插入 Trie。查询某个数时，从最高位往最低位贪心走“相反位”，因为那会让当前位异或成 `1`；如果相反位不存在，才退而求其次走相同位。这样每个数都能在线找到与自己异或值最大的配对。

### 核心代码

```cpp
int query(int x){
    int u = 0, res = 0;
    for(int b = 30; b >= 0; --b){
        int t = (x >> b) & 1;
        if(ch[u][t ^ 1]) u = ch[u][t ^ 1], res |= 1 << b;
        else u = ch[u][t];
    }
    return res;
}
for(int x : a) insert(x), ans = max(ans, query(x));
```

### 复杂度

时间复杂度 $O(N\log V)$，空间复杂度 $O(N\log V)$。

## 19. [1473：【例题3】Codechef REBXOR](http://ybt.ssoier.cn:8088/problem_show.php?pid=1473)

### 题意

在数组中选两个互不相交的子段，使两段异或和之和最大。

### 分析

单段最大异或子段的经典模型是前缀异或：区间异或和等于两个前缀异或值的异或。于是“以某个位置结尾的最好答案”可以在 01-Trie 中查询当前前缀异或的最佳配对。

### 核心代码

```cpp
for(int i = 1, x = 0; i <= n; ++i){
    x ^= a[i];

    trie.insert(x);
}
for(int i = n, x = 0; i >= 1; --i){
    x ^= a[i];
    R[i] = max(R[i + 1], trie2.query(x));
    trie2.insert(x);
}
for(int i = 1; i < n; ++i) ans = max(ans, L[i] + R[i + 1]);
```

时间复杂度 $O(n\log V)$，空间复杂度 $O(n\log V)$。

## 22. [1476：Secret Message 秘密信息](http://ybt.ssoier.cn:8088/problem_show.php?pid=1476)

### 题意

### 分析

“较短长度范围内完全相同”意味着匹配分两类：一类是信息更短，它必须正好在密码路径上结束；另一类是密码更短，它只要是某些信息的前缀即可。所以仅有终点计数还不够，还需要整条路径的通过计数。

把所有截获信息插入 Trie，每个节点维护 `pass` 表示有多少信息经过它，`ed` 表示有多少信息恰好在此结束。查询一条密码时，沿路径往下走的同时，把沿途的 `ed` 累加进去；若密码整条走完，再把当前节点的 `pass` 加上，表示所有以它为前缀的更长信息也匹配。

### 核心代码

```cpp
int query(string s){
    int u = 0, ans = 0;
    for(char c : s){
        ans += ed[u];
        int v = c - '0';
        if(!ch[u][v]) return ans;
        u = ch[u][v];
    }
    return ans + pass[u];
}
```

时间复杂度 $O(\sum |s|)$，空间复杂度 $O(\sum |s|)$。

### 分析

dfs(0);
for(int i = 0; i < ord.size(); ++i) pos[ord[i]] = i + 1;
for(int id : ord){

}

        dfs(v, u);

最后这几题把“很多模式串”继续往前推进：有的要在字符串集合之间做状态压缩，有的要在自动机删掉危险点后判环或计数。到这里，字符串结构已经彻底变成图上的状态系统了。

## 29. [1483：最短母串](http://ybt.ssoier.cn:8088/problem_show.php?pid=1483)

`状态压缩DP` `最短母串`

### 题意

给定 $n$ 个字符串，求一个最短字符串 $T$，使得所有给定串都是 $T$ 的子串；若最短解不唯一，输出字典序最小者。

### 分析

这题的关键不是直接构造母串，而是先把模式串之间的重叠关系算清楚。若某个串本来就是另一个串的子串，它对答案没有独立贡献，可以先删掉；剩下的串之间只需要关心“前一个串的后缀和后一个串的前缀最多能重合多少”。

一旦重叠长度预处理出来，问题就变成了旅行商式的状态压缩 DP：`dp[S][i]` 表示覆盖集合 $S$ 且最后一个放的是第 $i$ 个串时，得到的最优母串。转移时把新串接到末尾，只补上没有重叠的那一截；长度优先，长度相同再比较字典序。

### 核心代码

```cpp
for(int S = 1; S < 1 << n; ++S)
    for(int i = 0; i < n; ++i) if(S >> i & 1){
        if(S == (1 << i)) dp[S][i] = s[i];
        else for(int j = 0; j < n; ++j) if((S >> j & 1) && j != i)
            relax(dp[S][i], dp[S ^ (1 << i)][j] + s[i].substr(overlap[j][i]));
    }
for(int i = 0; i < n; ++i) relax(ans, dp[(1 << n) - 1][i]);
```

### 复杂度

时间复杂度 $O(n^2 2^n + n^2L)$，空间复杂度 $O(n2^n)$。

## 30. [1484：病毒](http://ybt.ssoier.cn:8088/problem_show.php?pid=1484)

`AC自动机` `环判定`

### 题意

给定若干二进制病毒串。若一个无限长二进制串不包含任何病毒串作为子串，就称它是安全的。要求判断是否存在无限长安全串。

### 分析

“是否存在无限长安全串”翻译成自动机语言，就是：在把所有病毒串插入 AC 自动机后，删掉所有会落到病毒结尾的危险状态，剩下的安全状态图里是否还存在一个可以无限走下去的环。

因为只要安全图中有环，就能在环上无限绕，生成无限长安全串；反之若安全图是 DAG，任何安全串长度都有限。于是问题从字符串构造直接转成了图上判环。

### 核心代码

```cpp
build_fail();
for(int u = 0; u <= tot; ++u)
    bad[u] |= bad[fail[u]];
for(int u = 0; u <= tot; ++u) if(!bad[u])
    for(int c = 0; c < 2; ++c){
        int v = ch[u][c];
        if(!bad[v]) g[u].push_back(v), indeg[v]++;
    }
cout << (has_cycle_on_safe_graph() ? "TAK" : "NIE");
```

### 复杂度

时间复杂度 $O(\sum |s_i|)$，空间复杂度同阶。

## 31. [1485：文本生成器](http://ybt.ssoier.cn:8088/problem_show.php?pid=1485)

`AC自动机` `计数DP`

### 题意

给定若干个已知单词和固定文本长度 $M$。把每一位都看成随机生成的大写字母，要求统计长度恰为 $M$ 的文章中，有多少篇至少包含一个给定单词。

### 分析

直接数“可读文章”很难，因为它要求至少命中一个模式串。更自然的反面建模是先数“完全不含任何单词”的安全文章，再用总数减掉它。

把所有单词建成 AC 自动机后，所有终点及其 fail 后继都视为危险状态。令 `dp[i][u]` 表示生成了前 $i$ 个字符、目前处在安全状态 $u$ 的方案数；枚举下一个大写字母做转移，只允许走向安全状态。最后答案就是 $26^M - \sum_u dp[M][u]`。

### 核心代码

```cpp
build_fail();
for(int u = 0; u <= tot; ++u) bad[u] |= bad[fail[u]];
dp[0][0] = 1;
for(int i = 0; i < m; ++i)
    for(int u = 0; u <= tot; ++u) if(dp[i][u] && !bad[u])
        for(int c = 0; c < 26; ++c){
            int v = ch[u][c];
            if(!bad[v]) dp[i + 1][v] = (dp[i + 1][v] + dp[i][u]) % 10007;
        }
ans = (qpow(26, m) - sum_safe(m) + 10007) % 10007;
```

### 复杂度

时间复杂度 $O(M\cdot |\text{AC状态}|\cdot 26)$，空间复杂度 $O(M\cdot |\text{AC状态}|)$。
