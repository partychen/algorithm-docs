# 一本通 字符串算法高阶专题精选解题报告

这一组题从固定长度子串判重一路走到 Trie、AC 自动机与状态图计数，表面上像是在轮流切换工具，真正的主线却一直没变：先把“字符串之间什么算同一种结构”说清楚，再把这种结构压成能线性维护的状态。前半段更强调哈希与前后缀关系，后半段则逐步过渡到多串自动机、矩阵和图上的综合建模。

# 一、哈希视角下的等价类与结构识别

这一章先把“两个对象什么时候算一样”这件事打透：有时是子串完全相同，有时是带约束的字符重命名，有时甚至要把整棵树的局部结构也哈希成可比较的指纹。

## 1. [1691：文章评分](http://ybt.ssoier.cn:8088/problem_show.php?pid=1691)

`高手训练` `哈希表`

### 题意

给定一个长度为 $n$ 的小写字母串，以及一个固定长度 $m$。题目要求统计所有长度恰好为 $m$ 的子串里，互不相同的有多少个。

### 分析

固定长度子串去重，最自然的建模就是把每个窗口压成一个可比较的指纹。因为所有比较都发生在同一长度上，所以并不需要后缀结构，直接做滚动哈希即可。

先预处理前缀哈希与幂数组，再把每个区间 $[l,l+m-1]$ 的哈希值丢进集合里。窗口右移时只是常数时间取区间哈希，最后集合大小就是答案。这里真正该形成的手感是：**当题目只问固定长度的“有多少种”时，哈希比逐个比较子串更像第一反应。**

### 核心代码

```cpp
using ull = unsigned long long;
vector<ull> h(n + 1), pw(n + 1);
pw[0] = 1;
for (int i = 1; i <= n; ++i) {
    h[i] = h[i - 1] * 131 + s[i];
    pw[i] = pw[i - 1] * 131;
}
auto get = [&](int l, int r) {
    return h[r] - h[l - 1] * pw[r - l + 1];
};
unordered_set<ull> st;
for (int l = 1; l + m - 1 <= n; ++l) st.insert(get(l, l + m - 1));
cout << st.size();
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 2. [1692：字符串编码](http://ybt.ssoier.cn:8088/problem_show.php?pid=1692)

`高手训练` `哈希表`

### 题意

给定字符串 $S,T$。允许先选若干对互不相交的字母对，再把串里落在这些字母对中的字符两两互换。题目要求找出 $S$ 中哪些长度为 $|T|$ 的子串，能够通过这样的编码关系从 $T$ 变来。

### 分析

这题的关键不是“替换字符”，而是先认出这里的映射其实是一种**只含 1-环和 2-环的对合置换**：某个字符要么不变，要么和另一个字符互换，而且一个字母最多参与一条边。

于是每个候选窗口都要同时满足两件事。第一，`T` 中相同字符出现的位置，在窗口里也必须落到同一种字符上；这类“相等模式”可以用每个字符出现位置的哈希指纹去比。第二，若 `T` 的字符 `a` 被映到窗口字符 `b`，那么 `b` 也必须反向映回 `a`，这就是对合约束。把 26 个字母之间的映射关系在窗口上建出来，再用哈希快速验证各类位置模式，就能在线性扫描里筛出所有合法起点。

### 核心代码

```cpp
for (int l = 1; l + m - 1 <= n; ++l) {
    fill(mp, mp + 26, -1);
    bool ok = true;
    for (int i = 0; i < m && ok; ++i) {
        int a = t[i] - 'a', b = s[l + i] - 'a';
        if (mp[a] == -1) mp[a] = b;
        else ok &= (mp[a] == b);
    }
    for (int a = 0; a < 26 && ok; ++a) if (mp[a] != -1) {
        int b = mp[a];
        ok &= (mp[b] == -1 || mp[b] == a);
        ok &= same_occ_hash(a, b, l);
    }
    if (ok) ans.push_back(l);
}
```

### 复杂度

时间复杂度 $O(26(n+m))$，空间复杂度 $O(n+m)$。

## 3. [1693：正确答案](http://ybt.ssoier.cn:8088/problem_show.php?pid=1693)

`高手训练` `哈希表`

### 题意

有 $n$ 份判断题作答记录，每份长度为 $m$，且已知其中恰有 $p$ 份是满分、$q$ 份是零分。要求还原标准答案；若有多解，输出字典序最小的那个，否则输出 $-1$。

### 分析

“满分”和“零分”这两个条件把候选答案压得很死：标准答案串若为 `A`，那么满分的人答案必须正好等于 `A`，零分的人答案必须正好等于 `rev(A)`，这里的 `rev` 指把 `Y/N` 全部翻转。于是判断一个候选串是否合法，本质上只是在问：它本身出现了多少次，它的按位取反串又出现了多少次。

所以建模重点不是做复杂搜索，而是把所有答案串和它们的反串都哈希起来。若 $p>0$ 或 $q>0$，候选一定来自某份作答或其补串；把这些候选去重后逐个检查计数即可。若需要字典序最小，就按字典序扫第一个满足条件的串。整题的转身在于：**约束不是按列独立的，而是整串相等或整串相反。**

### 核心代码

```cpp
for (auto &s : rec) ++cnt[get_hash(s)];
for (auto &s : rec) cand.push_back(s), cand.push_back(flip(s));
sort(cand.begin(), cand.end());
cand.erase(unique(cand.begin(), cand.end()), cand.end());
for (auto &s : cand) {
    if (cnt[get_hash(s)] == p && cnt[get_hash(flip(s))] == q) {
        cout << s;
        return;
    }
}
cout << -1;
```

### 复杂度

时间复杂度 $O(nm)$，空间复杂度 $O(n)$。

## 4. [1694：回文串](http://ybt.ssoier.cn:8088/problem_show.php?pid=1694)

`高手训练` `哈希表`

### 题意

给定两个串 $A,B$，其中 $A_x$ 表示 $A$ 的长度为 $|A|-x+1$ 的后缀，$B_y$ 表示 $B$ 的长度为 $y$ 的前缀。每次询问给定 $x,y$，要求计算从 $A_x$ 里选一个非空前缀、从 $B_y$ 里选一个非空后缀后，二者拼起来是回文串的方案数。

### 分析

拼接回文最重要的是盯住分界线。设左半段来自 $A_x$ 的前缀、右半段来自 $B_y$ 的后缀，那么跨过分界线向两边扩展时，能匹配多远完全由两侧的最长公共延伸决定；而若一边先耗尽，剩下那一截还必须本身就是回文前缀或回文后缀。

因此这题通常拆成两层预处理：先用正反哈希把“某个后缀和某个前缀在边界处能对上多长”算出来，再用 Manacher 或回文半径数组预处理每个位置向内剩余的一段是否可独立成为回文。查询时只要按照公共匹配长度把可行切法分成“左边先用完”和“右边先用完”两类求和即可。真正的建模核心是：**回文不是整串做，而是围绕拼接点做。**

### 核心代码

```cpp
int solve(int x, int y) {
    int lim = min(na - x + 1, y);
    int k = lcp_suffix_prefix(x, y);
    k = min(k, lim);
    int ans = cntA[x][k] + cntB[y][k];
    if (all_pal(k, x, y)) --ans;
    return ans;
}
```

### 复杂度

预处理时间复杂度 $O(|A|+|B|)$，单次查询时间复杂度 $O(1)$，空间复杂度 $O(|A|+|B|)$。

## 5. [1695：独特的树叶](http://ybt.ssoier.cn:8088/problem_show.php?pid=1695)

`高手训练` `哈希表`

### 题意

给出两棵无根树 $A,B$。已知 $B$ 恰好是由 $A$ 添加一个叶子后再重新编号得到的。要求找出 $B$ 里那个多出来的叶子；若有多个可行答案，输出编号最小的叶子。

### 分析

虽然题面说的是树，但真正需要比较的还是“删去某个叶子后，剩下的结构是否和另一棵树完全一样”。无根树同构最稳妥的做法就是把树根定在重心附近，用 AHU 风格的树哈希把每棵树压成规范表示。

对树 $A$ 先求出重心并算出整树哈希。对树 $B$，显然只有叶子才可能是候选；枚举叶子时，等价于看它的父亲删掉这一枝后的整树哈希是否能和 $A$ 的规范哈希对上。借助子树哈希与换根哈希，可以把“删除一片叶子后的整树签名”在线得到。题目的思考顺序应该是：**先把树变成可比较的字符串，再去枚举那个多余点。**

### 核心代码

```cpp
using ull = unsigned long long;
ull dfs(int u, int p, vector<int> g[]) {
    vector<ull> son;
    for (int v : g[u]) if (v != p) son.push_back(dfs(v, u, g));
    sort(son.begin(), son.end());
    ull h = 146527;
    for (ull x : son) h = h * 19260817 ^ (x + 911382323);
    return h * 19260817 + 7;
}
for (int leaf : leavesB) {
    if (hash_after_cut(leaf) == target_hash) ans = min(ans, leaf);
}
cout << ans;
```

### 复杂度

时间复杂度 $O(n \log n)$，空间复杂度 $O(n)$。

# 二、前后缀、边界与环上匹配

从这里开始，焦点从“整体长得像不像”转向“当前位置和前缀到底能对齐多长”。KMP、前缀函数和滑动窗口虽然表面不同，实际上都在维护一种局部边界。

## 6. [1696：前缀和](http://ybt.ssoier.cn:8088/problem_show.php?pid=1696)

`高手训练` `KMP算法`

### 题意

给定一个字符串，要求统计所有长度为偶数的前缀，在整串中出现次数之和。

### 分析

题目要的是“某个前缀作为子串出现了多少次”，这正是前缀函数最擅长维护的量。对整串求出 `pi` 后，每个位置贡献一次“以该位置结尾的最长 border”，再沿着 `pi` 链把出现次数向短 border 累加，就能得到每个前缀在整串中的出现次数。

等所有前缀的出现次数都求出来以后，只把长度为偶数的那些加起来即可。这题最值得记住的是：**前缀函数不只会做匹配，它的 border 树还能顺手统计每个前缀出现了多少次。**

### 核心代码

```cpp
for (int i = 1, j = 0; i < n; ++i) {
    while (j && s[i] != s[j]) j = pi[j - 1];
    if (s[i] == s[j]) ++j;
    pi[i] = j;
}
for (int i = 0; i < n; ++i) ++cnt[pi[i]];
for (int i = n; i > 0; --i) cnt[pi[i - 1]] += cnt[i];
for (int len = 1; len <= n; ++len) ++cnt[len];
long long ans = 0;
for (int len = 2; len <= n; len += 2) ans += cnt[len];
cout << ans;
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 7. [1697：深意](http://ybt.ssoier.cn:8088/problem_show.php?pid=1697)

`高手训练` `KMP算法`

### 题意

给定文本串 $s$ 和模式串 $t$。因为单词 $t$ 有“两层意思”，题目要求统计整段话 $s$ 一共有多少种可能的理解方式，答案对模数取余。

### 分析

真正需要先认出的，是“深意”发生的地方一定对应于 $t$ 在 $s$ 中的出现区间。于是第一步并不是直接做计数，而是先用 KMP 找出所有匹配终点，把每个可触发深意的区间定位出来。

有了这些区间后，问题会自然转成前缀 DP：扫到位置 `i` 时，既可以继承前面的理解方式，也可以让某个以 `i` 结尾的出现区间作为一次新的“意义切换”。重叠出现不能独立暴力枚举，所以要把所有以同一位置结尾的贡献用前缀和压到 $O(1)$ 转移。思维上的关键，是把语义变化翻译成区间选择，而不是在字符层面重复讨论。 

### 核心代码

```cpp
vector<int> occ = kmp_match(s, t);
for (int r : occ) end_at[r].push_back(r - m + 1);
dp[0] = pre[0] = 1;
for (int i = 1; i <= n; ++i) {
    dp[i] = dp[i - 1];
    for (int l : end_at[i]) dp[i] = (dp[i] + pre[l - 1]) % MOD;
    pre[i] = (pre[i - 1] + dp[i]) % MOD;
}
cout << dp[n];
```

### 复杂度

时间复杂度 $O(|s|+|t|)$，空间复杂度 $O(|s|)$。

## 8. [1698：字符串匹配](http://ybt.ssoier.cn:8088/problem_show.php?pid=1698)

`高手训练` `KMP算法`

### 题意

两个串若能通过任意次交换位置互相得到，就称为匹配。给定整数串 $s,t$，要求找出 $s$ 中所有与 $t$ 匹配的连续子串位置。

### 分析

允许任意交换位置，其实就把“顺序”全部抹掉了：两个串匹配，当且仅当它们每个字符值的出现次数完全相同。看清这一点以后，题目就从字符串匹配题退化成了定长窗口的频率比较题。

于是对模式串统计一份频次数组，再在文本串上开一个长度为 $m$ 的滑动窗口，维护当前窗口与模式串的频率差。只要差分全部为零，这个起点就是合法答案。真正的建模动作，是把“交换任意位置”识别成“只剩多重集信息”。

### 核心代码

```cpp
for (int x : t) ++need[x];
int diff = nonzero_kind(need);
for (int i = 1; i <= n; ++i) {
    diff += modify(s[i], -1);
    if (i > m) diff += modify(s[i - m], +1);
    if (i >= m && diff == 0) ans.push_back(i - m + 1);
}
cout << ans.size() << '\n';
for (int x : ans) cout << x << ' ';
```

### 复杂度

时间复杂度 $O(n+C)$，空间复杂度 $O(C)$。

## 9. [1699：石环](http://ybt.ssoier.cn:8088/problem_show.php?pid=1699)

`高手训练` `KMP算法`

### 题意

给定一个环形颜色串。对每个 $k$，要判断是否能删去一段连续长度为 $k$ 的石头，使剩下的环满足任意相邻两块颜色都不同。

### 分析

删去一段连续区间，等价于在环上保留一段连续长度为 $n-k$ 的区间，再把它首尾接起来看是否仍然“相邻不同”。因此只要在 `s+s` 上枚举长度为 `len` 的窗口，判断窗口内部相邻字符都不同，且首尾字符也不同，就得到对应的可行性。

为了不把所有长度和所有起点都暴力扫一遍，可以先在双倍串上预处理“从某个位置开始，最长能延伸多远而始终相邻不同”，这样每个起点对应哪些保留长度合法就一目了然，再差分标记到答案数组里。思考顺序应是：**先把环切开成区间，再把条件改写成窗口内部与窗口端点的约束。**

### 核心代码

```cpp
string t = s + s;
for (int i = 1, l = 1; i <= 2 * n; ++i) {
    if (i > 1 && t[i] == t[i - 1]) l = i;
    good[i] = i - l + 1;
}
for (int st = 1; st <= n; ++st) {
    int mx = min(n, good[st + n - 1]);
    cover[1] = 1, cover[mx + 1]--;
    if (t[st] == t[st + n - 1]) ban[n]++;
}
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

# 三、Trie 上的约束计数与状态合并

这一章开始，不再只盯着单串的前后缀，而是把大量字符串一起塞进树形状态里。Trie 的价值不只是查找，更在于把“前缀关系”直接变成一棵可以做 DP、合并与换根的结构树。

## 10. [1700：PFS集合](http://ybt.ssoier.cn:8088/problem_show.php?pid=1700)

`高手训练` `Trie树`

### 题意

给定若干互不相同的字符串，要求统计它们有多少个子集满足“任意一个字符串都不是另一个字符串的前缀”。空集也算。

### 分析

“不能互为前缀”这条约束，一旦放到 Trie 上就会变得非常直观：如果某个终止节点对应的字符串被选了，那么它的整棵子树里任何终止节点都不能再选；反过来，如果它不选，就可以把所有儿子的方案数独立相乘。

因此直接在 Trie 上做树形 DP 即可。设 `f[u]` 表示在 `u` 的子树里能选出多少个合法子集。若 `u` 不是字符串结尾，就只能把各儿子方案相乘；若 `u` 是字符串结尾，则多出一种“直接选当前串并且整棵子树到此为止”的方案。这题最重要的转化就是：**前缀禁选关系，天然就是 Trie 上的祖先-后代冲突。**

### 核心代码

```cpp
int dfs(int u) {
    int ways = 1;
    for (int v : ch[u]) ways = ways * dfs(v) % MOD;
    if (end[u]) ways = (ways + 1) % MOD;
    return ways;
}
cout << dfs(0);
```

### 复杂度

时间复杂度 $O(\sum |s|)$，空间复杂度 $O(\sum |s|)$。

## 11. [1701：最大值](http://ybt.ssoier.cn:8088/problem_show.php?pid=1701)

`高手训练` `Trie树`

### 题意

先任选一个 $n$ 位二进制数 $x$，再依次把它与 $a_1\sim a_m$ 做异或。对手可以在恰好一个时刻把当前值做一次循环左移一位。你想让最终结果尽量大，而对手会让它尽量小。要求输出最优最终值以及能达到它的初值个数。

### 分析

这题最关键的一步，是把“在任意时刻左移一次”改写成一批固定的对手候选值。设前缀异或为 $p_i$，如果对手在第 $i$ 次异或后动手，最终值就是
$\operatorname{rot}(x\oplus p_i)\oplus (p_m\oplus p_i)$。
因为循环左移对异或是线性的，所以它又等于
$\operatorname{rot}(x)\oplus (\operatorname{rot}(p_i)\oplus p_m\oplus p_i)$。

这样问题就变成：给定一堆数 $b_i$，选择一个数 $y=\operatorname{rot}(x)$，最大化 $\min_i (y\oplus b_i)$。放到二进制 Trie 上按位做极大极小即可：若当前位两侧都有数，那么无论 `y` 当前位取什么，最小值都会被落在同侧的那一批数卡成 `0`，只能继续比较低位；若某一侧为空，就可以把这一位稳定地做成 `1`。这就是整题真正的模型。 

### 核心代码

```cpp
pair<int, long long> solve(vector<int> &v, int b) {
    if (b < 0) return {0, 1};
    vector<int> L, R;
    for (int x : v) (((x >> b) & 1) ? R : L).push_back(x);
    if (L.empty()) {
        auto t = solve(R, b - 1);
        return {t.first | (1 << b), t.second};
    }
    if (R.empty()) {
        auto t = solve(L, b - 1);
        return {t.first | (1 << b), t.second};
    }
    auto a = solve(L, b - 1), c = solve(R, b - 1);
    if (a.first != c.first) return a.first > c.first ? a : c;
    return {a.first, a.second + c.second};
}
```

### 复杂度

时间复杂度 $O(mn)$，空间复杂度 $O(mn)$。

## 12. [1702：异或运算](http://ybt.ssoier.cn:8088/problem_show.php?pid=1702)

`高手训练` `Trie树`

### 题意

给定两列非负整数 $X,Y$，定义矩阵 $A_{ij}=x_i\operatorname{xor} y_j$。每次询问指定一个子矩形和一个整数 $k$，要求求出该矩形里的第 $k$ 大值。

### 分析

矩形里每个元素都是“某个固定的 $x_i$ 去和一段 $Y$ 做异或”，所以按位贪心时，核心操作就是：对某个 `x_i`，统计区间 `Y[l..r]` 里有多少个数能让当前位异或成 `1`。这恰好是可持久化二进制 Trie 的标准能力。

于是每个询问都能从高位到低位做“第 $k$ 大”的贪心。到了某一位，先把行区间内每个 `x_i` 在 `Y[l..r]` 上能贡献的 `1` 的个数加起来；如果总数不少于 `k`，答案这一位就是 `1`，并把所有行转到对应的优先儿子；否则这一位只能是 `0`，同时把 `k` 减掉这些更大的方案数。难点不在矩阵，而在把矩形拆成“若干行 + 区间 Trie 查询”。

### 核心代码

```cpp
int kth(int u, int d, int l, int r, long long k) {
    int ans = 0;
    for (int b = 30; b >= 0; --b) {
        long long cnt = 0;
        for (int i = u; i <= d; ++i)
            cnt += query(rt[r], rt[l - 1], x[i], b, 1);
        if (cnt >= k) ans |= 1 << b;
        else {
            k -= cnt;
            for (int i = u; i <= d; ++i) flip_pref(i, b);
        }
    }
    return ans;
}
```

### 复杂度

单次询问时间复杂度 $O((d-u+1)\log V)$，空间复杂度 $O(m\log V)$。

## 13. [1703：量子纠缠](http://ybt.ssoier.cn:8088/problem_show.php?pid=1703)

`高手训练` `Trie树`

### 题意

维护一个数字串集合，支持加入字符串、询问某串是否属于集合，以及让两个串“纠缠”。所谓纠缠，是指它们后面接任意后缀后，属于集合的结果必须完全等价。

### 分析

题目里最重要的一句话其实是“对任意后缀都保持等价”。这意味着被纠缠的不是两个完整字符串本身，而是**Trie 上表示这两个前缀的状态**：从这两个状态继续往下走，能否到达被加入集合的终止状态，必须完全同步。

所以正确的模型是把所有出现过的字符串先离线建出一棵 Trie，然后把每次纠缠看成两个 Trie 状态的并查集合并。合并时不仅要把当前节点并起来，还要把 10 个数字儿子递归对应合并，因为“后面再接任何串”都要保持同构。加入操作只是给某个状态打上可达标记，查询时看询问串对应状态所在并查集是否已经被激活即可。整题的本质，是**状态等价类维护**，不是普通字符串集合。 

### 核心代码

```cpp
int find(int x) { return fa[x] == x ? x : fa[x] = find(fa[x]); }
void merge(int x, int y) {
    x = find(x), y = find(y);
    if (x == y) return;
    fa[x] = y, tag[y] |= tag[x];
    for (int d = 0; d < 10; ++d) {
        if (ch[x][d] && ch[y][d]) merge(ch[x][d], ch[y][d]);
        else if (ch[x][d]) ch[y][d] = ch[x][d];
    }
}
```

### 复杂度

总时间复杂度 $O(\sum |s| \alpha(N))$，空间复杂度 $O(\sum |s|)$。

## 14. [1704：奇特的猫](http://ybt.ssoier.cn:8088/problem_show.php?pid=1704)

`高手训练` `Trie树`

### 题意

三天的按键记录会生成三个字符串集合，集合里的元素是屏幕在各个时刻出现过的所有串。题目先定义普通编辑距离，再通过选定若干“特殊串”在人为地连接不同集合后，要求输出三天对应结果的最小值和最大值。

### 分析

因为键盘只能在串尾插入字符或退格，所以一天里所有出现过的字符串天然构成一棵树：父节点就是删掉最后一个字符后的字符串。两个字符串之间的编辑距离，正好等于这棵树上的距离，也就是深度和减去两倍 LCA 深度。

看清这层以后，后两天所谓“变换编辑”其实就是在不同集合的两棵树之间额外连若干条权值为 `1` 的桥。题目要的最小值与最大值，就转成了在树上选择桥端点，使全体点对距离和最小或最大。做法上通常是先把每一天的操作序列建成 Trie/祖先树，再用子树大小、换根 DP 和树上距离和公式求出各候选特殊点的贡献。核心模型不是字符串编辑，而是**树上距离 + 人工连边后的重心/直径型优化**。

### 核心代码

```cpp
void dfs1(int u) {
    sz[u] = mark[u];
    for (int v : g[u]) {
        dep[v] = dep[u] + 1;
        dfs1(v);
        sz[u] += sz[v];
        down[u] += down[v] + 1LL * sz[v];
    }
}
void dfs2(int u) {
    for (int v : g[u]) {
        up[v] = up[u] + (tot - 2LL * sz[v]);
        dfs2(v);
    }
}
```

### 复杂度

时间复杂度 $O(S)$ 或 $O(S \log S)$，空间复杂度 $O(S)$。

## 15. [1705：DNA序列](http://ybt.ssoier.cn:8088/problem_show.php?pid=1705)

`高手训练` `AC自动机`

### 题意

字母表只有 `A,C,T,G`。给定长度 $n$ 以及若干禁止出现的模式串，要求统计长度为 $n$ 的合法序列个数，对模数取余。

### 分析

这类“长度很大、禁止若干子串”的计数题，标准转身就是把“当前已经匹配到哪些危险前缀”压进 AC 自动机状态。自动机里凡是能通过失配链走到某个模式串结尾的状态，都属于非法状态，后续不能再转移进去。

因为 $n$ 可以大到 $2\times10^9$，线性 DP 已经不可能，只能把状态转移写成矩阵。矩阵的第 `u,v` 项表示合法状态 `u` 经过一个字符转移到 `v` 的方案数，然后做矩阵快速幂，求出从根出发走 `n` 步的总方案数即可。整题的关键，是先把“不能出现某串”翻译成“不能进入某些自动机节点”。

### 核心代码

```cpp
for (int u = 0; u < tot; ++u) if (!bad[u]) {
    for (int c = 0; c < 4; ++c) {
        int v = ch[u][c];
        if (!bad[v]) mat[u][v] = (mat[u][v] + 1) % MOD;
    }
}
Matrix res = qpow(mat, n);
int ans = 0;
for (int v = 0; v < tot; ++v) ans = (ans + res[0][v]) % MOD;
cout << ans;
```

### 复杂度

时间复杂度 $O(s^3\log n)$，空间复杂度 $O(s^2)$，其中 $s$ 为自动机状态数。

## 16. [1706：串包含问题](http://ybt.ssoier.cn:8088/problem_show.php?pid=1706)

`高手训练` `AC自动机`

### 题意

要求统计长度为 $2m$ 的反对称 $01$ 串个数，并且这些串必须同时包含给定的所有模式串作为子串。

### 分析

反对称条件说明整个长度为 $2m$ 的串并不是自由的：前半段一旦确定，后半段就被“反向取反”唯一决定。所以真正需要枚举的只有前 $m$ 位，而子串是否出现则要同时照顾前半段和后半段对模式串的影响。

因此做法通常是把原模式串和“反向取反”后的模式串一起压进 AC 自动机，再让 DP 状态同时记录从左往右扫前半段时，原串自动机所在状态、镜像部分对应自动机所在状态，以及当前已经覆盖到哪些模式串的集合。每填一位，两个状态同步转移，一个按原字符走，一个按互补字符走。难点不在反对称本身，而在**把左右两半的影响放进同一次状态转移里**。

### 核心代码

```cpp
for (int i = 0; i < m; ++i)
    for (auto [a, b, mask] : states)
        for (int c = 0; c < 2; ++c) {
            int na = go[a][c];
            int nb = rgo[b][c ^ 1];
            int nm = mask | out[na] | rout[nb];
            add(dp[i + 1][na][nb][nm], dp[i][a][b][mask]);
        }
```

### 复杂度

时间复杂度 $O(m \cdot s^2 \cdot 2^n)$，空间复杂度 $O(s^2 \cdot 2^n)$。

`高手训练` `AC自动机`

### 题意

给定若干模式串。每次等概率随机敲一个小写字母，直到当前串包含任意一个模式串为止。题目要求停止时串长的期望值。

### 分析

当“出现任意一个模式串就停止”时，AC 自动机里的终止状态天然就是吸收态，期望为 `0`。其余状态设期望为 `E[u]`，那么从 `u` 走一步一定会多打一个字符，然后以均匀概率转到 26 个后继状态，于是有
$E[u]=1+\frac{1}{26}\sum E[v]$。

这已经不是字符串题，而是标准的线性方程组。把所有非终止状态列方程，终止状态直接赋值为 `0`，最后在模意义下做高斯消元即可。真正该形成的模型感是：**随机输入直到命中模式串，本质就是自动机上的吸收马尔可夫链。**

### 核心代码

```cpp
for (int u = 0; u < tot; ++u) {
    if (bad[u]) a[u][u] = 1;
    else {
        a[u][u] = 26;
        for (int c = 0; c < 26; ++c)
            a[u][go[u][c]] = (a[u][go[u][c]] - 1 + MOD) % MOD;
        a[u][tot] = 26;
    }
}
gauss(a, tot);
cout << a[0][tot];
```

### 复杂度

时间复杂度 $O(s^3)$，空间复杂度 $O(s^2)$。

### 题意

用前 $t$ 个小写字母构造双向无限长的链，要求其中不能出现给定的任何禁串。把平移后完全相同的无限链视为同一种，要求统计合法无限链个数；若无限多则输出 $-1$。

### 分析

双向无限串在自动机上对应的，其实是**可以无限延展的合法循环结构**。先在 AC 自动机里删去所有会命中禁串的坏状态，只保留合法转移；之后问题就变成了：这个有向图里有多少种本质不同的循环行为。

若某个强连通分量里同时容纳了不止一条独立环，那么沿着这些环可以拼出无限多种不同的双向无限链，答案直接是 $-1$。否则每个贡献答案的强连通分量都只能退化成一个简单环，平移同构正好对应环上的循环位移等价类，只需按 SCC 统计即可。整题的建模核心，是把“无限串”理解成自动机合法子图上的循环。 

### 核心代码

```cpp
tarjan();
for (int c = 1; c <= scc_cnt; ++c) {

    for (int u : comp[c])
        for (int k = 0; k < t; ++k)
            if (!bad[go[u][k]] && id[go[u][k]] == c) ++e;
    if (e > v) return void(cout << -1);
    if (e == v) ans += count_cycle_class(comp[c]);
}
cout << ans;
```

时间复杂度 $O(s\cdot t)$，空间复杂度 $O(s)$。

### 题意

给定一个字符串集合 $S$。若一个字符串能被拆成两段非空串，且这两段都分别是集合里某个字符串的前缀，则称它为“好串”。要求统计不同好串的总数。

### 分析

这题先别急着拼串，先看“前缀集合”本身。把所有给定串插入 Trie 后，Trie 上每个非根节点都代表一个可选前缀。题目要数的，其实就是“两段前缀串拼起来以后，最终能形成多少个不同结果”。

做法上可以先把所有前缀的哈希按长度分组。对任意拆分长度 $i+j$，第一段长度为 $i$、第二段长度为 $j$ 的所有前缀对，都能组合出一批候选串；把拼接后的双哈希丢进全局集合去重即可。这里最重要的不是枚举哪两个原串，而是认清：**题目真正关心的是前缀全集的两两拼接结果。**

### 核心代码

```cpp
for (auto &s : dict) {
    ull h = 0;
    for (char c : s) {
        h = h * BASE + (c - 'a' + 1);
        pref[len_of_prefix].push_back(h);
    }
}
for (int i = 1; i <= 30; ++i) sort_unique(pref[i]);
for (int i = 1; i <= 30; ++i)
    for (int j = 1; j <= 30; ++j)
        for (ull a : pref[i])
            for (ull b : pref[j]) st.insert(a * pw[j] + b);
cout << st.size();
```

### 复杂度

时间复杂度与不同前缀数的乘积有关，空间复杂度 $O(P)$，其中 $P$ 为不同前缀总数。
