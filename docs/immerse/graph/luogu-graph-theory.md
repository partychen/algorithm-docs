# 洛谷 图论综合专题精选解题报告

这一组题从 2-SAT 与 Tarjan 一路走到匹配、网络流、最小生成树和 DAG 优化，表面上模型很多，但核心都在“先认图结构，再选正确的抽象层”：有时要先缩点，有时要把可行性改写成流量或匹配，有时则要把原图提炼成更稳定的 DAG。真正的门槛通常不是实现，而是建模转身的那一下。

# 一、图论综合建模

这几题外表常像最短路，但真正决定做法的是最小割、缩点 DAG 或更一般的图论模型。

## 1. [P2046 \[NOI2010\] 海拔](https://www.luogu.com.cn/problem/P2046)

`平面图` `最小割` `Dijkstra`

### 题意

城市路网是规则网格，道路代价已知。要求求出把城市从一侧切到另一侧所需付出的最小总代价。

### 分析

这题和“狼抓兔子”是同一类平面图转化。原问题想切断两侧连通，看上去像最小割；但在平面图里，割集本身可以看成对偶图里一条把两块外部区域连起来的路径。

因此更自然的做法是直接建对偶图：每个面是一个点，每条道路变成连接左右两个面的对偶边，权值就是切掉这条道路的代价。最后在两侧外部面之间跑最短路，就等价于求原图最小割。

所以这题要形成的直觉是：**平面网格里若目标是“隔开两边”，优先想对偶图最短路，而不是先上流。**

### 核心代码

```cpp
for(each road){
    int a=face_left(road), b=face_right(road);
    add_edge(a,b,cost), add_edge(b,a,cost);
}
dijkstra(outer_up);
cout<<dist[outer_down];
```

### 复杂度

$O(n^2\log n)$。

## 2. [P4001 \[ICPC 2006 Beijing R\] 狼抓兔子](https://www.luogu.com.cn/problem/P4001)

`平面图` `最小割` `Dijkstra`

### 题意

兔子和狼位于网格图两端，三类道路都带有通过代价。要求求出阻断两端连通所需移除道路权值和的最小值。

### 分析

题目表面在问“最便宜地切断哪批路”，本质却是一个平面图最小割。真正该想到的不是网络流暴力建图，而是平面图里那条很重要的对偶关系：**原图的割，对应对偶图里一条把两侧外部面连起来的路径。**

在网格图里，这个转化尤其自然。每条道路都夹在两个面之间，于是它在对偶图中变成连接这两个面的边，权值仍是切掉这条路的代价。最后在源点和汇点对应的外部面之间跑最短路，得到的正好就是最小割代价。

所以这题最值得记住的是平面图建模的转身：**阻断连通性的最小代价，常先翻到对偶图里做最短路。**

### 核心代码

```cpp
for(each edge in grid){
    int u=left_face(edge), v=right_face(edge);
    dual[u].push_back({v,w});
    dual[v].push_back({u,w});
}
dijkstra(Sface);
ans=dist[Tface];
```

### 复杂度

$O(nm\log(nm))$。

## 3. [P1073 \[NOIP 2009 提高组\] 最优贸易](https://www.luogu.com.cn/problem/P1073)

`SCC` `DP` `最短路建模`

### 题意

城市有买卖价格和单双向道路，要求从 `1` 走到 `n` 的过程中先买后卖，求最大利润。

### 分析

在同一强连通分量里可以任意往返，所以先缩点。对 DAG 正向 DP 维护从 `1` 到该点途中见过的最低价格，反向 DP 维护从该点到 `n` 能遇到的最高卖价，两者差值取最大就是答案。

### 核心代码

```cpp
tarjan(); build_dag();
for(int x:topo){
    mn[x]=min(mn[x],low_price[x]);
    for(int y:dag[x]) mn[y]=min(mn[y],mn[x]);
}
for(int x:rtopo){
    mx[x]=max(mx[x],high_price[x]);
    for(int y:rdag[x]) mx[y]=max(mx[y],mx[x]);
}
```

### 复杂度

$O(n+m)$。

## 4. [P3008 \[USACO11JAN\] Roads and Planes G](https://www.luogu.com.cn/problem/P3008)

`Dijkstra` `拓扑排序` `最短路`

### 题意

城镇之间有双向非负道路和单向可能为负的航线，并保证航线不会形成可回到原点的环。要求求出源点到所有城镇的最短路。

### 分析

道路连通块内部没有负环，可以在每个块里正常跑 Dijkstra；航线只会把答案从前面的块传到后面的块，因此连通块缩成点后是一张 DAG。按拓扑序处理块：块内跑 Dijkstra，块间用航线松弛。

### 核心代码

```cpp
build_road_cc();
topo(comp_dag);
for(int c:ord){
    priority_queue<Node> pq;
    for(int x:ver[c]) if(dist[x]<INF) pq.push({x,dist[x]});
    run_inner_dijkstra(c,pq);
    for(int u:ver[c]) for(auto [v,w]:plane[u])
        if(dist[u]+w<dist[v]) dist[v]=dist[u]+w;
}
```

### 复杂度

$O((R+P)\log T)$。

# 二、2-SAT

只要出现二元选择、互斥与蕴含约束，就把变量和否定拆成点，转成强连通判定。

## 5. [AT_arc069_d \[ARC069F\] Flags](https://www.luogu.com.cn/problem/AT_arc069_d)

`2-SAT` `二分` `线段树`

### 题意

第 $i$ 面旗子只能放在直线上的 $x_i$ 或 $y_i$ 位置，要求让任意两面旗子之间的最小距离尽可能大。

### 分析

题目要求“最大化最小距离”，这类字眼先触发二分答案。固定一个候选距离 `d` 后，问题就变成：能不能给每面旗子在 `x_i / y_i` 中选一个位置，使所有被选位置两两距离都不少于 `d`。

判定里每面旗子都只有两种落点，所以天然是布尔变量。若“旗子 `i` 取方案 `a`”与“旗子 `j` 取方案 `b`”之间距离小于 `d`，那这两个选择不能同时成立，就是一条标准 2-SAT 冲突。由于一段连续位置都会同时冲突，才需要用线段树把“向一个区间连边”压到对数级。

所以这题真正该记住的是三段翻译：**最小值最大化 $\to$ 二分，二选一摆放 $\to$ 2-SAT，批量冲突区间 $\to$ 线段树优化连边。**

### 核心代码

```cpp
bool check(int d){
    init_sat();
    for(int i=1;i<=n;i++) for(int t=0;t<2;t++){
        auto [L,R]=conflict_range(i,t,d);
        seg_link(L,R,node(i,t^1));
    }
    tarjan();
    for(int i=1;i<=n;i++) if(scc[i<<1]==scc[i<<1|1]) return false;
    return true;
}
```

### 复杂度

$O(n\log^2 n\log V)$。

## 6. [UVA1146 Now or later](https://www.luogu.com.cn/problem/UVA1146)

`2-SAT` `二分`

### 题意

每个事件都有“现在做”或“稍后做”两种备选时刻，要求安排所有事件并最大化任意两次操作之间的最小时间间隔。

### 分析

看到“每个事件二选一，并且若两个具体选择太近就冲突”，第一反应就该是把每个候选时刻视作布尔变量的一个取值。题目要求最大化最小间隔，因此自然二分答案 $d$，转而判断“能否让所有已选时刻两两至少相距 $d$”。

判定时最关键的翻译是：若事件 $i$ 选方案 $a$、事件 $j$ 选方案 $b$ 会导致 $\lvert t_{i,a}-t_{j,b}\rvert<d$，那么这两个选择不能同时成立，也就是一个标准的 2-SAT 冲突子句。把所有这类冲突都写成蕴含边后，只要整张蕴含图可满足，说明当前 $d$ 可行。

所以这题要记住的不是“2-SAT + 二分”六个字，而是这个常见套路：最大化最小距离，先二分答案；每个对象只有少数离散选择，再把冲突关系改写成 2-SAT。

### 核心代码

```cpp
bool check(int d){
    clear_graph();
    for(int i=1;i<=n;i++) for(int j=i+1;j<=n;j++)
        for(int a=0;a<2;a++) for(int b=0;b<2;b++)
            if(abs(tim[i][a]-tim[j][b])<d)
                add(id(i,a),id(j,b^1)), add(id(j,b),id(i,a^1));
    return satisfiable();
}
```

### 复杂度

$O(n^2\log V)$。

## 8. [P6378 \[PA 2010\] Riddle](https://www.luogu.com.cn/problem/P6378)

`2-SAT` `图论建模`

### 题意

无向图的点被划分成若干部分，每个部分必须恰好选一个关键点，并要求每条边至少有一个端点被选中。

### 分析

这题难点不是“边覆盖”四个字，而是“每个部分恰选一个点”看起来不像 2-SAT。真正的转身是：把一个部分里的多个候选点改写成一串有序布尔变量，表示“是否已经选到这里之前”，于是“恰选一个”就能拆成若干相邻变量之间的蕴含关系。

另一类条件就很熟悉了。每条边至少覆盖一个端点，直接翻译成 `(u\lor v)` 这样的二元子句。于是整题虽然外表是分组选择，其实仍然可以统一落到 2-SAT 的蕴含图上。

所以这题最值得学的是建模心法：**当原约束不是天然二选一时，可以先造辅助布尔变量，把“多选一”拆成一串局部的二元逻辑关系。**

### 核心代码

```cpp
for(each part){
    for(int i=1;i<sz;i++) add(prefix[i],prefix[i+1]);
    add_exactly_one(part_nodes);
}
for(auto [u,v]:edges){
    add(not_choose(u),choose(v));
    add(not_choose(v),choose(u));
}
```

### 复杂度

$O(n+m)$。

```cpp
for(auto [i,hi,j,hj]:rule){
```

## 12. [P3209 \[HNOI2010\] 平面图判定](https://www.luogu.com.cn/problem/P3209)

`2-SAT` `平面图` `区间相交`

### 题意

无向图已知含有一条包含所有顶点的哈密顿回路，要求判断其余弦边能否分别画在环内或环外而互不相交，从而判断图是否平面。

### 分析

已知一条哈密顿回路后，所有非环边都可以看成圆上的弦。平面性在这里不再抽象：你只需要决定每条弦画在环内还是环外，并保证同侧的弦互不相交。

于是冲突条件就非常具体了：若两条弦在圆周顺序上交叉，那它们绝不能画在同一侧；若不交叉，则放哪边都无所谓。这说明每条弦只有两种状态，而每对交叉弦提供一个“必须异色”的约束，所以问题自然落成弦交图二分，或等价地写成 2-SAT。

这题的迁移点也很明确：**只要题目已给出一圈固定顺序，剩下边只能分到两侧避免交叉，就要立刻想到交叉对建冲突图。**

### 核心代码

```cpp
for(int i=1;i<=m;i++) for(int j=i+1;j<=m;j++)
    if(cross(ch[i],ch[j])){
        add(in(i),out(j)); add(in(j),out(i));
        add(out(i),in(j)); add(out(j),in(i));
    }
cout<<(satisfiable()?"YES":"NO");
```

### 复杂度

$O(m^2)$。

## 13. [CF27D Ring Road 2](https://www.luogu.com.cn/problem/CF27D)

`2-SAT` `环上弦` `构造`

### 题意

城市在一个环上，新修的道路是连接两点的弦，每条弦只能画在环内或环外，要求使任意两条道路除了端点外都不相交，并输出一种内外分配方案。

### 分析

环上弦题的核心观察是：真正会出事的不是“弦很多”，而是**两条弦在圆周顺序上交叉时，若还被画在同一侧就一定相交。**

于是每条弦天然只有“画在环内”或“画在环外”两种状态，是一个布尔变量；而任意一对交叉弦都必须分居两侧，这就变成了 `x_i≠x_j` 的异色约束。写成图论语言可以做二分染色，写成逻辑语言就是 2-SAT。

所以这类题下次遇到时，优先先问自己：**冲突是不是只发生在交叉弦对之间，且每条弦只有两种摆放面。** 若答案是，那就已经非常接近 2-SAT 了。

### 核心代码

```cpp
for(int i=1;i<=m;i++) for(int j=i+1;j<=m;j++) if(cross(i,j)){
    add(i_in,j_out); add(j_in,i_out);
    add(i_out,j_in); add(j_out,i_in);
}
solve_2sat();
```

### 复杂度

$O(m^2)$。

每头奶牛对两个议案分别给出赞成/反对偏好，要求最终结果至少满足她的一个愿望。若存在合法方案，还要判断每个议案是必定通过、必定否决还是两者皆可。

## 15. [P5782 \[POI 2001 R2\] 和平委员会](https://www.luogu.com.cn/problem/P5782)

`2-SAT` `模板`

### 题意

每个党派有两名代表，和平委员会要求每个党派恰好选一人，同时互相厌恶的两名代表不能同时入选。要求输出一组可行代表名单。

### 分析

每个党派“二选一”这一点，已经把变量形式送到你面前了：选代表 A 还是代表 B，本身就是一个布尔变量。

剩下的厌恶关系则是在说“某两个人不能同时被选”，这正好是 2-SAT 里最典型的冲突约束。把党派内部互斥和跨党派厌恶关系都统一写成蕴含边后，问题就被完全翻译成可满足性判定。

所以这道模板题最重要的不是实现，而是识别结构：**一组组二选一对象，再叠加“这两项不能同取”，就是标准 2-SAT。**

### 核心代码

```cpp
for(int i=1;i<=n;i++){
    add(rep(i,0),not_rep(i,1));
    add(rep(i,1),not_rep(i,0));
}
for(auto [a,b]:hate){
    add(a,opp(b)); add(b,opp(a));
}
solve_2sat();
```

### 复杂度

$O(n+m)$。

## 16. [P4782 【模板】2-SAT](https://www.luogu.com.cn/problem/P4782)

`2-SAT` `模板`

### 题意

给定若干形如“变量 $x_i$ 取某真值或变量 $x_j$ 取某真值”的条件，要求判断整个 2-SAT 公式是否有解，并给出一组赋值。

### 分析

2-SAT 模板最重要的不是背公式，而是理解这两条蕴含边从哪来。子句 `(a∨b)` 的意思是：如果 `a` 不成立，那你就必须让 `b` 成立；同理如果 `b` 不成立，就必须让 `a` 成立，所以才会连出 `¬a→b`、`¬b→a`。

建完蕴含图后，再看强连通分量。若某变量和它的否定能互相推出，说明你既必须选真又必须选假，当然无解；否则按 SCC 的反拓扑序贪心赋值，就能保证先确定“更强制”的那一边。

所以模板真正该掌握的是：**子句不是直接拿来判，而是先改写成“一个不成立会逼出另一个成立”的蕴含关系。**

### 核心代码

```cpp
void add_or(int a,int b){
    add(a^1,b); add(b^1,a);
}
tarjan();
for(int i=0;i<n;i++){
    if(scc[i<<1]==scc[i<<1|1]) bad();
    val[i]=scc[i<<1]<scc[i<<1|1];
}
```

### 复杂度

$O(n+m)$。

## 18. [P3386 【模板】二分图最大匹配](https://www.luogu.com.cn/problem/P3386)

`二分图` `最大匹配`

### 题意

给定一个左右部大小分别为 `n,m` 的二分图，要求输出最大匹配边数。

### 分析

最大匹配最朴素的想法是不断找增广路：若能把一条“未匹配边、已匹配边”交替的路径翻转，就能让匹配数加一。模板题真正该学会的，是 Hopcroft-Karp 为什么比逐条 DFS 增广更快。

它先用 BFS 在“交替图”里分层，找出当前所有最短增广路的长度；然后 DFS 只在这张分层图里走，把同一轮能找到的最短增广路尽量一起翻完。这样每一轮都会把最短增广路长度整体抬高，因此总轮数不多。

所以这题的迁移点是：**二分图最大匹配 = 交替路翻转；HK 的优化本质是按层批量处理最短增广路。**

### 核心代码

```cpp
bool bfs(){
    queue<int> q;
    for(int i=1;i<=n;i++) if(!mx[i]) dep[i]=0,q.push(i); else dep[i]=-1;
    while(!q.empty()){
        int u=q.front(); q.pop();
        for(int v:g[u]) if(my[v]&&dep[my[v]]==-1) dep[my[v]]=dep[u]+1,q.push(my[v]);
    }
    return true;
}
```

### 复杂度

$O(m\sqrt n)$。

## 20. [U248878 染色法判定二分图](https://www.luogu.com.cn/problem/U248878)

`二分图` `染色`

### 题意

给定一个可能含重边和自环的无向图，要求判断它是否是二分图。

### 分析

二分图判定最核心的不是“染两种颜色”这件事本身，而是理解它在排除什么结构。若一张图能被染成两色且每条边两端颜色不同，那么沿着任意一条边走，颜色都会来回翻转，因此绝不可能出现奇环。

反过来，只要某个连通块里出现了奇环，你沿环交替染色绕一圈后一定会撞色，所以二分图判定与“图中是否存在奇环”是完全等价的。这就是为什么只需要做 BFS/DFS 二染色。

实现时还要顺手注意自环：自环等于一个长度为 `1` 的奇环，直接判坏。于是整题真正的思路链就是：**能否分成两个独立集合 ⇔ 能否二染色 ⇔ 是否不存在奇环。**

### 核心代码

```cpp
bool dfs(int u,int c){
    col[u]=c;
    for(int v:adj[u]){
        if(!col[v]&&!dfs(v,3-c)) return false;
        if(col[v]==c) return false;
    }
    return true;
}
```

### 复杂度

$O(n+m)$。

## 23. [P3381 【模板】最小费用最大流](https://www.luogu.com.cn/problem/P3381)

`费用流` `最小费用最大流`

### 题意

给定带容量和单位费用的网络，要求求出从源点到汇点的最大流以及在该最大流下的最小费用。

### 分析

最小费用最大流模板最容易被背成“反复 SPFA 增广”，但真正要先想清楚的是目标分成两层：先尽量多送流量，再在这些最大流方案里把总费用压到最低。

因此每一轮寻找的都不是任意增广路，而是**当前残量网络里单位费用最小的增广路**。沿这条路尽量多送一些流量后，你既保持了“流量继续增长”，又保证新增这部分流量付出的代价最划算。

直到汇点再也不可达时，说明最大流已经送满；而一路都按最短费用增广，最终费用也就最小。所以模板里真正该理解的是：**增广的是流量，最短路负责控制每次新增流量的边际成本。**

### 核心代码

```cpp
while(spfa(s,t)){
    int f=INF;
    for(int v=t;v!=s;v=pre[v]) f=min(f,e[preid[v]].cap);
    flow+=f; cost+=1LL*f*dis[t];
    for(int v=t;v!=s;v=pre[v]){
        e[preid[v]].cap-=f;
        e[preid[v]^1].cap+=f;
    }
}
```

### 复杂度

$O(Fm\cdot SPFA)$。

## 25. [P3376 【模板】网络最大流](https://www.luogu.com.cn/problem/P3376)

### 题意

给定网络及源汇点，要求求出网络最大流。

最大流模板题最容易被写成“背算法”，但真正要先看懂的是网络流在做什么：每条边有容量限制，源点往汇点送的总流量想尽可能大，而所有中间点都必须满足流量守恒。

Dinic 为什么有效，核心在于它不是盲目增广，而是先用 BFS 把当前残量网络分成层，确保之后 DFS 只沿着“离汇点更近一层”的方向推流。这样每次 DFS 都在找阻塞流：把当前分层图里还能送的流尽量一次送满。

### 核心代码

```cpp
while(bfs()){
    memcpy(cur,head,sizeof head);
    while(int f=dfs(s,t,INF)) maxflow+=f;
}
int dfs(int u,int t,int f){
    if(u==t||!f) return f;
    for(int &i=cur[u];i;i=e[i].nxt) if(e[i].cap&&dep[e[i].to]==dep[u]+1){
        int w=dfs(e[i].to,t,min(f,e[i].cap));

    }
    return 0;
}
```

## 27. [P8436 【模板】边双连通分量](https://www.luogu.com.cn/problem/P8436)

### 题意

给定无向图，要求输出全部边双连通分量的个数以及每个分量包含的点。

### 分析

边双的定义是：任意两点之间至少有两条边不相交路径。真正会破坏这个性质的，恰恰就是桥，因为桥一删，整张图就被切断了。

所以求边双最自然的步骤就是先找桥。把所有桥删掉后，剩下的每个连通块内部都不再含桥，也就是说块内任意两点都处在同一个边双连通分量里；不同块之间则只能靠桥相连，显然不能混在一起。

因此这题最值得记住的是边双的本质翻译：**边双分量 = 删光所有桥后剩下的连通块。**

```cpp
void tarjan(int u,int in){

    for(int i=head[u];i;i=e[i].nxt){
        int v=e[i].to;
        if(!dfn[v]) tarjan(v,i),low[u]=min(low[u],low[v]),bridge[i]=bridge[i^1]=(low[v]>dfn[u]);
        else if(i!=(in^1)) low[u]=min(low[u],dfn[v]);

}
```

$O(n+m)$。

## 29. [T103481 【模板】割边](https://www.luogu.com.cn/problem/T103481)

`Tarjan` `割边`

### 题意

给定无向图，要求输出割边数量。

### 分析

桥模板最该理解的不是“删了会断”这句结论，而是 Tarjan 为什么能在线发现它。`dfn[u]` 记录首次访问时间，`low[v]` 则表示：从 `v` 子树往下走若干树边，再借助至多一条返祖边，最早还能回到哪个祖先。

于是对 DFS 树边 `(u,v)` 来说，若 `low[v] > dfn[u]`，就说明 `v` 子树根本没有别的路能绕回 `u` 或 `u` 上面。那这条边一断，下面整块就必然与上面脱离，它当然就是桥。

所以这类题真正要记住的是 `low` 的物理含义：**我这棵子树最早还能绕回哪里。** 一旦连父亲都绕不回，这条父子边就成了唯一通道。

### 核心代码

```cpp

    for(int i=head[u];i;i=e[i].nxt){
        int v=e[i].to;
        if(!dfn[v]) tarjan(v,i),low[u]=min(low[u],low[v]),ans+=low[v]>dfn[u];
        else if(i!=(in^1)) low[u]=min(low[u],dfn[v]);
    }
}
```

$O(n+m)$。

## 30. [U582665 【模板】割边](https://www.luogu.com.cn/problem/U582665)

### 题意

给定无向图，要求按字典序输出所有割边。

### 分析

桥的本质，不是“删了会断”，而是“这条 DFS 树边下面那整个子树，再也没有别的路能绕回上面”。Tarjan 里 $dfn[u]$ 记录首次到达时间，$low[v]$ 记录 $v$ 子树借助返祖边最早能回到的祖先时间。

于是对树边 $(u,v)$ 来说，若 $low[v] > dfn[u]$，就说明 $v$ 子树无法通过任何非父子边绕回 $u$ 或 $u$ 以上的位置，这条边一断，图就真的被切成两块，所以它就是桥。

因此桥模板真正要理解的是 `low` 的含义：**我这棵子树最早还能绕回哪里。** 一旦这条“绕回能力”上不去父亲，当前边就成了关键连接。

### 核心代码

```cpp
if(low[v]>dfn[u]){
    int a=min(u,v), b=max(u,v);
    bridge.push_back({a,b});
}
sort(bridge.begin(),bridge.end());
```

### 复杂度

$O(n+m)$。

## 32. [P3388 【模板】割点（割顶）](https://www.luogu.com.cn/problem/P3388)

`Tarjan` `割点`

### 题意

给定无向图，要求求出所有割点。

### 分析

割点和桥的差别在于：桥删的是边，割点删的是点，所以判断条件也从 `>` 变成了 `>=`。对 DFS 树边 `(u,v)`，若 `low[v] >= dfn[u]`，说明 `v` 子树最多只能绕回 `u` 本人，却再也回不到 `u` 的祖先。

这意味着一旦删掉 `u`，`v` 这整块就会和上方彻底断开，因此 `u` 是割点。只有根节点要单独讨论，因为它没有祖先，必须至少分出两个 DFS 子树，删掉后图才真的被拆成多块。

所以这题别只死记判定式，更要把它和桥模板对照着记：**桥看子树能否绕回父亲以上，割点看子树能否绕回当前点以上。**

### 核心代码

```cpp
void tarjan(int u,int fa){
    dfn[u]=low[u]=++idx; int child=0;
    for(int v:adj[u]){
        if(!dfn[v]){
            tarjan(v,u); low[u]=min(low[u],low[v]); child++;
            if(fa&&low[v]>=dfn[u]) cut[u]=1;
        }else if(v!=fa) low[u]=min(low[u],dfn[v]);
    }
    if(!fa&&child>=2) cut[u]=1;
}
```

### 复杂度

$O(n+m)$。

## 33. [P3469 \[POI 2008\] BLO-Blockade](https://www.luogu.com.cn/problem/P3469)

`Tarjan` `割点` `统计`

### 题意

对于每个点 `i`，删去与 `i` 相连的所有边后，要求统计有多少个有序点对 `(x,y)` 不再连通。

### 分析

删边后，$i$ 自己会与其他 $n-1$ 个点全部断开，贡献 $2(n-1)$。若 $i$ 是割点，它把图拆成若干块，设这些块大小为 $s_1,s_2,\ldots$，额外断开的有序点对数就是 $\sum s_j\cdot(\text{rest}-s_j)$。

### 核心代码

```cpp
void tarjan(int u,int fa){
    dfn[u]=low[u]=++idx; sz[u]=1; long long sum=0;
    for(int v:adj[u]) if(v!=fa){
        if(!dfn[v]){
            tarjan(v,u); sz[u]+=sz[v]; low[u]=min(low[u],low[v]);
            if(low[v]>=dfn[u]) ans[u]+=1LL*sz[v]*(n-sz[v]), sum+=sz[v];
        }else low[u]=min(low[u],dfn[v]);
    }
    ans[u]+=1LL*(n-1-sum)*(sum+1);
}
```

### 复杂度

$O(n+m)$。

## 34. [P2812 校园网络【\[USACO\]Network of Schools加强版】](https://www.luogu.com.cn/problem/P2812)

`SCC` `缩点`

### 题意

学校网络是有向图。要求先求最少选多少所学校作为软件源点能让所有学校都能用上软件，再求最少加多少条线路能让任意学校都能到达所有学校。

### 分析

原图里只要在同一个强连通分量内，软件拿到后就能互相传播，所以先缩点是第一反应。缩完后得到一张 DAG，问题就从“点对点传播”变成了“怎样让整张 DAG 都被覆盖并最终强连通”。

第一个问题为什么看入度为 `0` 的 SCC？因为这些块没有任何外来来源，若不主动在其中某个学校放软件，它们永远拿不到。第二个问题为什么是 `max(入度0块数, 出度0块数)`？因为想把一张 DAG 补成强连通，至少要给每个“只出不进”的源块补入口，也要给每个“只进不出”的汇块补出口，最少新增边数就是两者较大值；若整图本来就是一个 SCC，则不必加边。

### 核心代码

```cpp
tarjan();
for(auto [u,v]:edges) if(scc[u]!=scc[v]){
    indeg[scc[v]]++; outdeg[scc[u]]++;
}
for(int i=1;i<=sc;i++) in0+=!indeg[i], out0+=!outdeg[i];

```

$O(n+m)$。

## 36. [P3387 【模板】缩点](https://www.luogu.com.cn/problem/P3387)

### 题意

有向图每个点有权值，允许重复走边但点权只算一次。要求求出能取得的最大点权和。

### 分析

题目允许重复走边，但点权只算一次，这说明真正麻烦的不是路径长短，而是“哪些点能被打包一起白拿”。在一个强连通分量里，你既然能从任意点走到任意点，就可以把这个分量里的所有点权都拿满，而且不会因为进入顺序不同而改变结果。

这就是为什么第一步一定是缩点：先把每个 SCC 的总权值并成一个超级点。缩完后图变成 DAG，原题就从“允许反复绕圈的有向图”变成了“在 DAG 上选一条路径，使经过点权和最大”的标准最长路 DP。

所以这题最重要的不是 Tarjan 本身，而是这个建模转身：**重复可达的一坨点先压成一个价值包，再在 DAG 上做线性 DP。**

### 核心代码

```cpp
tarjan(); build_dag();
queue<int> q;
for(int i=1;i<=sc;i++) if(!indeg[i]) dp[i]=sum[i], q.push(i);
while(!q.empty()){
    int u=q.front(); q.pop();
    for(int v:dag[u]){
        dp[v]=max(dp[v],dp[u]+sum[v]);
        if(--indeg[v]==0) q.push(v);
    }
}
```

### 复杂度

$O(n+m)$。

## 38. [B3609 \[图论与代数结构 701\] 强连通分量](https://www.luogu.com.cn/problem/B3609)

`SCC` `Tarjan`

### 题意

给定一张可能含重边和自环的有向图，要求输出全部强连通分量。

### 分析

SCC 模板真正要抓住的是“什么时候能确认一整个强连通块已经闭合”。Tarjan 用栈维护当前 DFS 路径上还没被分配出去的点，`low[u]` 则表示从 `u` 出发还能通过回边回到的最早祖先时间戳。

当某个点满足 `dfn[u] = low[u]` 时，就说明以它为根的这一段 DFS 栈再也回不到更早的祖先了；换句话说，这一段正好构成一个完整的强连通分量。于是不断弹栈直到弹回 `u`，就得到了一个 SCC。

所以 SCC 模板别只记“等号时弹栈”，而要记它背后的意义：**这一坨点已经对外封口，可以整体缩成一个点了。**

### 核心代码

```cpp
void tarjan(int u){

    for(int v:adj[u]){
        if(!dfn[v]) tarjan(v), low[u]=min(low[u],low[v]);
        else if(ins[v]) low[u]=min(low[u],dfn[v]);
    }
    if(dfn[u]==low[u]) pop_scc(u);
}
```

$O(n+m)$。

## 40. [P3366 【模板】最小生成树](https://www.luogu.com.cn/problem/P3366)

`最小生成树` `Kruskal`

### 题意

给定无向图，要求求出最小生成树权值；如果图不连通则输出 `orz`。

### 分析

最小生成树真正要解决的是：在保证全图连通的前提下，尽量便宜地选出 `n-1` 条边。Kruskal 的贪心之所以自然，是因为如果当前有两块尚未连通，那么连接这两块的最便宜边迟早都值得优先考虑。

具体做法就是按边权从小到大扫描。若一条边两端已经在同一连通块里，再选它只会成环，纯属浪费；若它连接的是两个不同块，那把它加入答案一定不吃亏，因为任何生成树都必须想办法把这两块连起来，而你已经拿到了当前最便宜的候选。

所以模板里并查集扮演的角色不是“为了写板子”，而是在实时回答：**这条边是在补连通，还是只会制造环。**

### 核心代码

```cpp
sort(edges.begin(),edges.end());
for(auto [u,v,w]:edges) if(find(u)!=find(v)){
    unite(u,v); ans+=w; cnt++;
}
if(cnt!=n-1) cout<<"orz";
else cout<<ans;
```

### 复杂度

$O(m\log m)$。

# 六、拓扑排序与 DAG

当依赖关系决定了边方向，就要先固定拓扑序，再在线性顺序上做 DP 或构造。

## 41. [B3644 【模板】拓扑排序 / 家谱树](https://www.luogu.com.cn/problem/B3644)

`拓扑排序` `模板`

### 题意

给定一张有向无环图，要求输出一个合法的拓扑序；题面也可理解为按照家谱关系排出先后顺序。

### 分析

拓扑序的本质不是“某种神奇排列”，而是不断找出当前**没有任何前驱依赖**的点。一个点入度为 `0`，就表示在剩余图里没有点必须排在它前面，于是它现在出队是合法的。

Kahn 算法正是把这个逻辑做成迭代：每次取出一个入度为 `0` 的点，相当于把它放进答案并从图中删除；被它指向的点可能因此失去最后一个前驱，新的入度 `0` 点又变成下一批可放位置。

所以以后看到拓扑排序时，要先意识到它在维护的是“当前已经满足所有先修条件的点集合”，而不是单纯背一个队列模板。

### 核心代码

```cpp
queue<int> q;
for(int i=1;i<=n;i++) if(!indeg[i]) q.push(i);
while(!q.empty()){
    int u=q.front(); q.pop(); ord.push_back(u);
    for(int v:adj[u]) if(--indeg[v]==0) q.push(v);
}
```

### 复杂度

$O(n+m)$。

## 42. [P1807 最长路](https://www.luogu.com.cn/problem/P1807)

`拓扑排序` `DAG DP`

### 题意

给定一个 DAG，要求求出从 `1` 到 `n` 的最长路径长度；若不可达则输出 `-1`。

### 分析

一般图上的最长路是困难问题，但题目一旦明确是 DAG，难点就被“无环”这个条件彻底化解了。因为无环意味着你总能给所有点排出一个拓扑序，保证每条边都从前指向后。

这时最长路不再需要回溯或判环，而是变成了最朴素的 DP：设 `dp[u]` 表示从起点到 `u` 的最长路，按拓扑序遍历时，`u` 的值一旦确定，就可以放心地拿去更新所有后继点。

所以这题最值得迁移的是：**只要图是 DAG，很多原本会有环依赖的图论优化问题，都可以直接改写成拓扑序上的线性 DP。**

### 核心代码

```cpp
topo_sort();
fill(dp,dp+n+1,-INF); dp[1]=0;
for(int u:ord) if(dp[u]>-INF)
    for(auto [v,w]:g[u]) dp[v]=max(dp[v],dp[u]+w);
cout<<(dp[n]<0?-1:dp[n]);
```

### 复杂度

$O(n+m)$。

## 43. [P4017 最大食物链计数](https://www.luogu.com.cn/problem/P4017)

`拓扑排序` `DAG DP` `计数`

### 题意

食物链关系构成 DAG，要求统计从所有入度为 0 的生物到所有出度为 0 的生物的路径条数。

### 分析

食物链是 DAG，这意味着“从上游到下游”的依赖不会回头，因此路径计数可以放心按拓扑序推进。题目要数的是从任意源点走到任意汇点的路径条数，所以最自然的状态就是 `dp[u]` 表示到达 `u` 的路径数。

为什么入度为 `0` 的点初值设成 `1`？因为每个这样的点本身就是一条合法路径的起点。之后按拓扑序遍历时，把 `dp[u]` 沿所有出边分发给后继点，相当于把所有以 `u` 结尾的链统一向后延长一步。

最后只统计出度为 `0` 的点，是因为只有走到它们时食物链才真正结束。于是这题完整思路就是：**源点给初值，拓扑序上传播计数，汇点收答案。**

### 核心代码

```cpp
for(int i=1;i<=n;i++) if(!indeg[i]) dp[i]=1,q.push(i);
while(!q.empty()){
    int u=q.front(); q.pop();
    for(int v:adj[u]){
        dp[v]=(dp[v]+dp[u])%MOD;
        if(--indeg[v]==0) q.push(v);
    }
}
```

### 复杂度

$O(n+m)$。

## 44. [CF1385E Directing Edges](https://www.luogu.com.cn/problem/CF1385E)

`拓扑排序` `构造`

### 题意

部分边方向已确定，部分边仍未定。要求判断能否给所有未定边定向，使整张图无环；若可以则输出一种定向方案。

### 分析

题目里真正危险的只有一件事：先验已定向边如果自己就形成环，那后面怎么给自由边定向都救不回来。

所以第一步一定是只看已定向边做拓扑排序。若它们无环，就拿到了一份“所有点从前到后”的合法顺序。接下来未定向边其实就很好处理了：无论原来两端是谁，只要统一从拓扑序更靠前的点指向更靠后的点，就绝不可能制造回边。

这题最该形成的构造意识是：**先用已知边固定一个合法偏序，再让所有自由边顺着这个偏序定向。**

### 核心代码

```cpp
if(!topo(directed_edges)) return puts("NO"),0;
cout<<"YES\n";
for(auto [u,v]:fixed) cout<<u<<" "<<v<<"\n";
for(auto [u,v]:freee){
    if(pos[u]<pos[v]) cout<<u<<" "<<v<<"\n";
    else cout<<v<<" "<<u<<"\n";
}
```

### 复杂度

$O(n+m)$。
