# 一本通 图论高阶专题精选解题报告

这一组题从生成树的“怎么把边权补到刚刚好”出发，一路走到最短路上的计数、均值与期望，再转进差分约束、缩点判定、桥树与欧拉结构。题型跨度很大，但主线始终一致：先把题面条件翻成图上的可比较对象，再决定是做最优生成、可行性判定，还是做结构压缩后的动态规划。

# 一、最优生成与参数化生成树

这一章先练“树为什么是最优的”，再练“最优树怎样随着额外条件变化”。同样是生成树，有的题在补边权，有的题在强制一条边出现，有的题则把整棵树看成平面上的一个点集去做参数化查询。

## 1. [1711：最小花费](http://ybt.ssoier.cn:8088/problem_show.php?pid=1711)

`最小生成树` `前缀异或` `建模`

### 题意

设 $s_i=x_1\oplus x_2\oplus\cdots\oplus x_i$，并令 $s_0=0$。一次询问区间 $[l,r]$ 的奇偶性，本质上得到的是 $s_r\oplus s_{l-1}$ 的值，并且每种询问都有一个费用。题目要求选出若干次询问，使得能唯一确定全部 $x_i$，且总费用最小。

### 分析

这题最关键的一步，是把“区间奇偶”改写成“两个前缀点之间的关系”。一旦把每个 $s_i$ 看成图上的一个点，那么询问 $[l,r]$ 就是一条连接 $l-1$ 与 $r$ 的边，边权是这次询问的代价。

为什么最后会落成最小生成树？因为 $s_0=0$ 已知，只要这 $n+1$ 个前缀点在询问图里连成一个连通块，所有前缀异或值就都能顺着边被推出；反过来，只要图不连通，就总有一块前缀值整体翻转而不影响已知信息，答案不唯一。所以题目等价于：在 $0\sim n$ 这 $n+1$ 个点上选最便宜的连通边集，直接就是 MST。

### 核心代码

```cpp
long long prim(){
    vector<long long> dis(n + 1, INF);
    vector<int> vis(n + 1, 0);
    dis[0] = 0;
    long long ans = 0;
    for(int i = 0; i <= n; ++i){
        int x = -1;
        for(int j = 0; j <= n; ++j)
            if(!vis[j] && (x == -1 || dis[j] < dis[x])) x = j;
        vis[x] = 1; ans += dis[x];
        for(int y = 0; y <= n; ++y) dis[y] = min(dis[y], w[x][y]);
    }
    return ans;
}
```

### 复杂度

时间复杂度 $O(n^2)$，空间复杂度 $O(n^2)$。

## 2. [1710：构造完全图](http://ybt.ssoier.cn:8088/problem_show.php?pid=1710)

`最小生成树` `Kruskal` `计数`

### 题意

给定一棵带权树 $T$，要求给整张完全图补上其余边的权值，使得这张完全图的唯一最小生成树恰好是 $T$，并让完全图边权总和最小。

### 分析

把一条非树边 $(u,v)$ 放回去，会和树上 $u\to v$ 的路径形成唯一一个环。若想让 $T$ 仍然是最小生成树，就必须让这条补边不优于这条路径里最重的边；若还要求“唯一”，就必须严格更大。因为边权是整数，所以最省的设法就是令它等于“路径最大边权 $+1$”。

于是问题变成：对每一对点，求树上路径最大边权。直接对所有点对做会很慢，但从 Kruskal 的视角就简单了。按树边权从小到大合并连通块时，第一次把两个块连起来的边权 $w$，恰好就是跨这两个块任意点对的路径最大边权。设两块大小为 $a,b$，其中一对点由当前树边本身贡献，另外 $ab-1$ 对都要补边，各自贡献 $w+1$，于是可以并查集统计。

### 核心代码

```cpp
sort(e.begin(), e.end(), [](auto a, auto b){ return a.w < b.w; });
for(auto [u, v, w] : e){
    int fu = find(u), fv = find(v);
    long long a = sz[fu], b = sz[fv];
    ans += w;                    // 树边本身
    ans += (a * b - 1) * (w + 1);
    fa[fu] = fv;
    sz[fv] += sz[fu];
}
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(n)$。

## 3. [1712：汉堡店](http://ybt.ssoier.cn:8088/problem_show.php?pid=1712)

`最小生成树` `树上路径最大边` `几何图`

### 题意

平面上有 $N$ 个点，每两点之间都有一条边，边权是欧氏距离。要从完全图中选一棵生成树，并再指定树中的一条边 $(a,b)$ 作为“吃掉的两家店”，最大化 $(P_a+P_b)/(\text{生成树总权}-w(a,b))$。

### 分析

如果先固定要保留的边 $(a,b)$，问题就变成“在必须包含这条边的前提下，最小生成树总权是多少”。这正是强制边进入 MST 的经典模型：先求整图 MST，若 $(a,b)$ 已在树上，就不用改；若不在树上，把它加进 MST 会形成一个环，删掉这条环上最重的边才最优。因此最小总权是
$\text{mst}+w(a,b)-\max\_T(a,b)$。

而题目的分母还要再减去 $w(a,b)$，于是竟然只剩下 $\text{mst}-\max\_T(a,b)$。这一步很重要：真正影响答案的不是边本身长度，而是它在 MST 路径上能顶掉多重的一条边。所以先建出 MST，再预处理任意两点路径最大边，最后枚举点对取最优比值即可。

### 核心代码

```cpp
kruskal();
for(int s = 1; s <= n; ++s) dfs(s, 0, 0, s);
for(int i = 1; i <= n; ++i)
    for(int j = i + 1; j <= n; ++j){
        double den = mst - mx[i][j];
        ans = max(ans, (p[i] + p[j]) / den);
    }

void dfs(int x, int f, double best, int s){
    mx[s][x] = best;
    for(auto [y, w] : tree[x]) if(y != f)
        dfs(y, x, max(best, w), s);
}
```

### 复杂度

时间复杂度 $O(n^2)$，空间复杂度 $O(n^2)$。

## 4. [1713：生成树](http://ybt.ssoier.cn:8088/problem_show.php?pid=1713)

`生成树` `约数` `并查集`

### 题意

给定一张无向图。每一棵生成树都有一个“边权最大公约数”，题目要求所有生成树这些 gcd 的最小公倍数。

### 分析

先不要直接盯着“某棵树的 gcd 是多少”，而要先问：一个数 $d$ 能否整除某棵生成树的所有边？这等价于“只保留边权能被 $d$ 整除的边之后，图是否仍然连通”。只要这件事成立，就存在一棵生成树所有边都是 $d$ 的倍数，于是它的 gcd 至少含有 $d$ 的全部质因子指数。

而最小公倍数只关心每个质因子的最高次幂，所以把所有满足“整除后仍连通”的 $d$ 都并到答案里，最终得到的 lcm 正好就是所求。做法上可以预处理每条边权的全部约数，把边挂到这些约数名下，再对每个约数做一遍并查集判断是否连通。

### 核心代码

```cpp
for(auto [u, v, w] : edges)
    for(int d : divisors[w]) bag[d].push_back({u, v});
for(int d = 1; d <= W; ++d){
    init_dsu(n);
    for(auto [u, v] : bag[d]) unite(u, v);
    if(comp_cnt == 1) ans = lcm(ans, 1ULL * d);
}
```

### 复杂度

时间复杂度约为 $O(\sum d(w_i)+W\alpha(n))$，空间复杂度 $O(m+W)$。

## 5. [1714：地壳运动](http://ybt.ssoier.cn:8088/problem_show.php?pid=1714)

`参数最小生成树` `凸包` `分治`

### 题意

每条边有两个属性 $(u_i,v_i)$，一次询问给定 $(k_1,k_2)$ 后，这条边代价变成 $u_i k_1+v_i k_2$。要对很多次询问求最小生成树总长度。

### 分析

把一棵生成树看成平面上的一个点 $(\sum u,\sum v)$，那么一次询问实际上是在所有这些点上最小化线性函数 $k_1 x+k_2 y$。线性函数在一个点集上的最优值，一定出现在下凸壳的极点上，所以题目不需要真的找出所有生成树，只需要找出“可能成为某个方向最优解”的那一小批候选树。

怎么找这些候选树？先用 $(1,0)$ 和 $(0,1)$ 各跑一遍 MST，得到两个端点。如果两棵树对应的点之间线段上方还能被某棵新树撑起来，就说明这里还有新的凸壳点；于是用法向量继续跑一次 MST，把找到的新树递归插入。这样就能枚举出全部候选极点。对每次询问，只要在候选树里取最小线性值即可。

### 核心代码

```cpp
Tree mst(long long a, long long b){
    for(auto &e : edges) e.key = a * e.u + b * e.v;
    sort(edges.begin(), edges.end(), by_key);
    return kruskal_point();
}
void solve(Tree L, Tree R){
    Tree M = mst(L.v - R.v, R.u - L.u);
    if(cross(L, R, M) <= 0) return;
    hull.push_back(M);
    solve(L, M); solve(M, R);
}
```

### 复杂度

设候选树数量为 $K$，时间复杂度约为 $O(K\,m\log m)$，空间复杂度 $O(m)$。

# 二、最短路上的状态扩张

这一章的共同点，是“路”本身已经不够描述决策了。你要么把时间相位挂到公交线路上，要么把“是否正好比最短多一”挂到节点上，要么把允许等待这件事也纳入状态，最短路模型才会闭合。

## 6. [1715：公交旅行](http://ybt.ssoier.cn:8088/problem_show.php?pid=1715)

`时间扩张图` `Dijkstra` `循环序列`

### 题意

每条公交线按固定循环路线每秒前进一步。你初始在站台 $1$，只允许乘公交，问最早何时能到达每个站台。

### 分析

这题不能只把站台当点，因为“同一个站台在不同时间能上到的车”完全不同。真正稳定的对象，是“某条线路上的某个位置”，因为一旦上车，下一秒就只能到这条线路的下一个位置。

于是可以建立两类状态：站台状态与线路位置状态。站台到线路位置的代价是等待到该车下一次来到此站的时间；线路位置到下一位置的代价恒为 $1$；在某个位置下车则回到当前站台。这样整张图边权非负，直接 Dijkstra。题目的难点不在最短路，而在于先把周期运动翻译成静态图上的“等待边”。

### 核心代码

```cpp
while(!pq.empty()){
    auto [d, x] = pq.top(); pq.pop();
    if(d != dist[x]) continue;
    if(is_stop(x)){
        for(auto id : occ[stop_id(x)]){
            long long nd = d + wait_time(id, d);
            relax(node(id), nd);
        }
    }else{
        relax(next_pos(x), d + 1);
        relax(stop_node(cur_stop(x)), d);
    }
}
```

### 复杂度

设所有线路长度之和为 $S$，时间复杂度 $O(S\log S)$，空间复杂度 $O(S)$。

## 7. [1716：次短路计数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1716)

`最短路` `分层状态` `计数`

### 题意

要求统计从 $s$ 到 $t$ 的所有最短路，以及长度恰好比最短路多 $1$ 的路径，总方案数是多少。

### 分析

“多一条边”或者“多一单位长度”这类题，一般都要把“偏离最优多少”挂到状态里。这题最自然的状态就是每个点保留两种距离：最短距离 `dis[0]` 和次短候选 `dis[1]`。一条新边松弛进来时，要么更新最短，要么更新次短，要么把与最短同长的方案数累加进去。

因为题目只关心“最短”和“最短加一”两层，所以不必维护完整的第 $k$ 短路集合。最后若 `dis1[t] == dis0[t] + 1`，就把两层计数相加，否则只取最短层计数。

### 核心代码

```cpp
while(!pq.empty()){
    auto [d, x, k] = pq.top(); pq.pop();
    if(d != dis[x][k]) continue;
    for(auto [y, w] : g[x]){
        int nd = d + w;
        if(nd < dis[y][0]){
            dis[y][1] = dis[y][0], cnt[y][1] = cnt[y][0];
            dis[y][0] = nd, cnt[y][0] = cnt[x][k];
        }else if(nd == dis[y][0]) cnt[y][0] += cnt[x][k];
        else if(nd < dis[y][1]) dis[y][1] = nd, cnt[y][1] = cnt[x][k];
        else if(nd == dis[y][1]) cnt[y][1] += cnt[x][k];
    }
}
```

### 复杂度

时间复杂度 $O(m\log n)$，空间复杂度 $O(n+m)$。

## 8. [1717：负环](http://ybt.ssoier.cn:8088/problem_show.php?pid=1717)

`最小负环` `min-plus` `矩阵快速幂`

### 题意

在带权图中找一个点数最少、且边权和为负的环；若不存在负环则输出 $0$。

### 分析

这题问的不是“负环是否存在”，而是“最短需要多少步才出现负对角线”。于是最合适的工具不是普通 SPFA，而是 min-plus 矩阵乘法：若矩阵 $A$ 记录一条边的最小代价，那么 $A^k$ 的对角线就表示“走恰好 $k$ 条边回到自己”的最小代价。只要某个对角线元素小于 $0$，就存在长度为 $k$ 的负环。

接下来就可以做二进制提升。预处理 $A^{2^i}$，然后从大到小尝试把当前步数再加上 $2^i$；如果加完之后所有对角线仍然非负，就说明答案还更长。最后第一次使对角线变负的位置，就是最小负环长度。

### 核心代码

```cpp
Mat mul(const Mat& A, const Mat& B){
    Mat C(n, vector<long long>(n, INF));
    for(int i = 0; i < n; ++i)
        for(int k = 0; k < n; ++k) if(A[i][k] < INF)
            for(int j = 0; j < n; ++j)
                C[i][j] = min(C[i][j], A[i][k] + B[k][j]);
    return C;
}
for(int i = LOG; i >= 0; --i){
    Mat nxt = mul(cur, pw[i]);
    if(no_neg_diag(nxt)) cur = nxt, ans += 1 << i;
}
cout << (ans == n ? 0 : ans + 1);
```

### 复杂度

时间复杂度 $O(n^3\log n)$，空间复杂度 $O(n^2\log n)$。

## 9. [1718：益智游戏](http://ybt.ssoier.cn:8088/problem_show.php?pid=1718)

`最短路 DAG` `偏序 DP` `公共节点`

### 题意

两个人分别从 $A\to B$ 与 $C\to D$ 按最短路移动，每回合也可以原地等待。若同一时刻在同一节点相遇，可获得一次奖励，每个节点最多计一次。问最多能获得多少次奖励。

### 分析

先把“必须走最短路”拆开。对小 P 跑两次最短路，所有满足 $dist_A[x]+dist'_B[x]=dist_A[B]$ 的点和边，构成一张最短路 DAG；小 R 同理。一个节点若想被计分，就必须同时落在两张 DAG 上。

允许等待这件事，意味着两个人并不需要在同一步来到一个点，只要各自都能按自己的最短路顺序到达这里，就可以把时间对齐。因此问题变成：在这些公共点里找一条最长链，使得链上顺序同时符合两张最短路 DAG 的拓扑顺序。把公共点按两套距离形成的偏序连边，再做 DAG 上最长路即可。

### 核心代码

```cpp
dij(A, g, da); dij(B, rg, db);
dij(C, g, dc); dij(D, rg, dd);
for(auto [u, v, w] : edges){
    if(da[u] + w + db[v] == da[B]) dag1[u].push_back(v);
    if(dc[u] + w + dd[v] == dc[D]) dag2[u].push_back(v);
}
for(int x : common_nodes)
    for(int y : nxt[x])
        dp[y] = max(dp[y], dp[x] + 1);
```

### 复杂度

时间复杂度 $O((n+m)\log n)$ 外加公共 DAG 的转移复杂度，空间复杂度 $O(n+m)$。

## 10. [1719：过河](http://ybt.ssoier.cn:8088/problem_show.php?pid=1719)

`最短路` `节点费用` `几何建图`

### 题意

河中有木桩，可在木桩上放不同半径、不同价格的圆盘。人只能在两岸和圆盘上移动，相切或相交即可连通。求从 $y=0$ 到 $y=W$ 的最小花费。

### 分析

虽然题面在说“修一座桥”，但真正被选择的是一串能首尾相接的圆盘。只要能从下岸走到上岸，中间并不需要分叉，所以最优方案一定是一条链，而不是一张网。这样一来，每个“木桩 + 选择某种半径”都可以看成一个带费用的节点。

若两个圆盘相交或相切，就能从一个走到另一个；若圆盘碰到下岸或上岸，就分别与源点、汇点连通。节点有费用时，可以把“走到这个节点”理解为支付它的价格，于是问题直接变成一张节点带权图上的最短路。

### 核心代码

```cpp
for(int i = 0; i < N; ++i)
    for(int t = 0; t < M; ++t){
        int a = id(i, t);
        if(y[i] <= R[t]) add(S, a, C[t]);
        if(W - y[i] <= R[t]) add(a, T, 0);
        for(int j = 0; j < N; ++j)
            for(int s = 0; s < M; ++s)
                if(dis(i, j) <= R[t] + R[s]) add(a, id(j, s), C[s]);
    }
dijkstra(S);
```

### 复杂度

若状态数为 $NM$，时间复杂度约为 $O((NM)^2)$ 建图加 $O((NM)^2\log (NM))$ 最短路，空间复杂度 $O((NM)^2)$。

# 三、均值、单调与路长函数

这一章里，最短路不再只给一个固定权值。你会遇到“把平均值减掉再判负环”“边权必须单调不降”“代价是拥堵系数的线性函数”这类题。核心都是先把题目改写成一个可比较的函数，再用判定或递归把最优值抠出来。

## 11. [1720：均值最小环](http://ybt.ssoier.cn:8088/problem_show.php?pid=1720)

`最小平均环` `二分答案` `负环判定`

### 题意

在有向图中找平均边权最小的环；若图中没有环则输出指定字符串。

### 分析

平均值问题最经典的做法，就是把“平均值不超过 $mid$”改写成“减掉 $mid$ 以后是否存在负环”。若某个环原本平均边权小于 $mid$，那么它减掉 $mid$ 后总权就会变负；反之亦然。

于是题目被拆成两层：外层二分答案，内层做负环检测。由于边权都为正、图规模中等，直接用 SPFA 或 DFS 版的松弛判负环都可以。若原图无环，则任何 $mid$ 都不会出现负环，需要单独先判断一次有无有向环。

### 核心代码

```cpp
bool check(double mid){
    fill(dis + 1, dis + n + 1, 0);
    fill(cnt + 1, cnt + n + 1, 0);
    for(int s = 1; s <= n; ++s) if(dfs(s, mid)) return true;
    return false;
}
bool dfs(int x, double mid){
    vis[x] = 1;
    for(auto [y, w] : g[x]) if(dis[y] > dis[x] + w - mid){
        dis[y] = dis[x] + w - mid;
        if(vis[y] || dfs(y, mid)) return true;
    }
    vis[x] = 0; return false;
}
```

### 复杂度

时间复杂度约为 $O(Km)$，其中 $K$ 为二分轮数，空间复杂度 $O(n+m)$。

## 12. [1721：最小边权和](http://ybt.ssoier.cn:8088/problem_show.php?pid=1721)

`DP` `边权单调` `离线询问`

### 题意

多次询问：从 $a$ 到 $b$ 走不超过 $c$ 条边，且边权必须不下降，要求边权和最小。

### 分析

边权“互不相同”很关键，它让“不下降”直接变成“严格按权值排序向后选”。于是可以把一条路径看成若干条边组成的递增序列，状态自然落在“走了几条边、最后一条边是谁”。

如果对每个询问暴力做 DP 会很慢，更好的办法是按起点离线。固定起点 $a$ 后，按边权从小到大扫描边，用 `f[k][e]` 表示恰好走 $k$ 条边并以边 $e$ 结尾的最小代价。转移时需要从所有终点是 `u`、且边权比当前边小的状态里取最小值，这可以对每个点维护一个按权值下标更新的 Fenwick / 线段树前缀最小值。

### 核心代码

```cpp
sort(edges.begin(), edges.end(), [](auto a, auto b){ return a.w < b.w; });
for(int k = 1; k <= C; ++k){
    for(int v = 1; v <= n; ++v) bit[v].clear();
    for(int i = 0; i < m; ++i){
        auto [u, v, w] = edges[i];
        if(k == 1 && u == S) f[k][i] = w;
        else f[k][i] = bit[u].query(i - 1) + w;
        bit[v].add(i, f[k][i]);
    }
}
```

### 复杂度

设同一起点需要做到的最大边数为 $C$，时间复杂度 $O(Cm\log m)$，空间复杂度 $O(Cm)$。

## 13. [1722：骑士游戏](http://ybt.ssoier.cn:8088/problem_show.php?pid=1722)

`最短路思想` `不动点` `反图松弛`

### 题意

杀死一种怪兽有两种方法：法术攻击直接杀死，普通攻击会花费 $S_i$ 并生成若干新怪兽。问清空所有怪兽的最小体力消耗。

### 分析

把 `dp[i]` 记作“彻底消灭一只 $i$ 号怪兽的最小代价”，那么它满足一个非常直接的方程：
$dp[i]=\min\bigl(K_i,S_i+\sum dp[son]\bigr)$。真正困难在于怪兽生成关系可能有环，不能简单拓扑。

这时就要把它看成一组单调松弛方程。先把所有 `dp[i]` 设成法术代价 `K_i`，因为这是显然可行的上界；若某个儿子的代价下降了，那么所有把它当作普通攻击后继的怪兽，其“普通攻击总代价”也可能随之下降。于是沿反图做类似 SPFA 的松弛，直到收敛到最小不动点即可。

### 核心代码

```cpp
for(int i = 1; i <= n; ++i) dp[i] = K[i], q.push(i);
while(!q.empty()){
    int x = q.front(); q.pop(); inq[x] = 0;
    for(int p : rev[x]){
        long long sum = S[p];
        for(int y : son[p]) sum += dp[y];
        if(sum < dp[p]){
            dp[p] = sum;
            if(!inq[p]) q.push(p), inq[p] = 1;
        }
    }
}
```

### 复杂度

时间复杂度与总生成边数同阶，约为 $O(\sum R_i)$，空间复杂度 $O(n+\sum R_i)$。

## 14. [1723：交通](http://ybt.ssoier.cn:8088/problem_show.php?pid=1723)

`参数最短路` `线性函数` `分治`

### 题意

每条边在拥堵系数 $a\in[0,1]$ 时的代价为 $ax_i+(1-a)y_i$。要求随机均匀选取 $a$ 时，从 $S$ 到 $T$ 的最短路期望。

### 分析

固定一条路径后，它关于 $a$ 的总代价就是一条直线。题目真正要积分的是“所有这些直线的下包络”。所以不要想着对 $a$ 离散采样，而要想：哪几条路径真的有可能成为某段区间上的最优解？

做法和参数 MST 很像。先在区间端点 $a=0,1$ 各跑一次最短路，得到两条候选直线。若它们在某点相交，就到交点再跑一次最短路：若出现了第三条更低的直线，说明下包络在这里还会折，继续递归；否则整个区间上只需在两条直线里取较小者。最后对每一段由同一条直线控制的区间直接积分即可。

### 核心代码

```cpp
Line solve(double l, double r, Line L, Line R){
    double p = intersect(L, R);
    if(p <= l || p >= r) return add_seg(l, r, min_line(L, R));
    Line M = shortest_path_line(p);
    if(M == L || M == R) return split_by_two(l, r, L, R, p);
    solve(l, p, L, M);
    solve(p, r, M, R);
}
```

### 复杂度

设下包络上共有 $K$ 条有效路径，时间复杂度约为 $O(K(m\log n))$，空间复杂度 $O(n+m)$。

# 四、差分约束与可行性构造

这一章的共同语言是“不直接求数值，而是先写出变量之间必须满足的不等式或等式”。有的题只要判断有无解，有的题要把可行解真的构造出来，但第一步都是把题意翻成边，再看图里有没有矛盾。

## 15. [1724：小K的农场](http://ybt.ssoier.cn:8088/problem_show.php?pid=1724)

`差分约束` `SPFA` `可行性`

### 题意

给出若干“至少多 $c$”“至多多 $c$”“相等”三类关系，判断是否存在一组农场种植量满足全部条件。

### 分析

把每个农场的种植量记为 $d_i$。那么“$a$ 至少比 $b$ 多 $c$”就是 $d_a-d_b\ge c$，可写成 $d_b\le d_a-c$；“至多多 $c$”则是 $d_a-d_b\le c$。这就是标准的差分约束系统。

建图时把形如 $x\le y+w$ 的不等式写成边 $y\to x$、权值 $w$。题目只问可行性，所以不需要真的求最优值，只要给所有点加一个超级源、跑一遍最长路 / 最短路判负环即可。出现矛盾时，本质上就是图中存在一圈不可能同时满足的约束。

### 核心代码

```cpp
if(op == 1) add(a, b, -c);      // d_b <= d_a - c
if(op == 2) add(b, a, c);       // d_a <= d_b + c
if(op == 3) add(a, b, 0), add(b, a, 0);
for(int i = 1; i <= n; ++i) add(0, i, 0);
cout << (spfa_has_neg_cycle() ? "No" : "Yes");
```

### 复杂度

时间复杂度均摊约为 $O(m)$ 到 $O(nm)$，空间复杂度 $O(n+m)$。

## 16. [1725：账本核算](http://ybt.ssoier.cn:8088/problem_show.php?pid=1725)

`前缀和` `图上赋值` `判矛盾`

### 题意

每条信息给出某段月份区间的总收入。要判断这些信息是否能对应同一本真实账本。

### 分析

设前缀和 $S_i=A_1+\cdots+A_i$，则区间和条件
$A_x+\cdots+A_y=w$ 等价于 $S_y-S_{x-1}=w$。这是一个“两个变量差值固定”的约束，不再是不等式，而是等式。

等式最适合建成带权无向关系：从 $x-1$ 到 $y$ 连一条权值 $w$ 的边，同时反向连 $-w$。之后在每个连通块里任选一个点赋值为 $0$，沿边 DFS/BFS 推出其余前缀和；若某条边要求的差值与已经推出的值冲突，就说明账本必假。

### 核心代码

```cpp
add(x - 1, y, w);
add(y, x - 1, -w);
for(int s = 0; s <= n; ++s) if(!vis[s]){
    q.push(s); vis[s] = 1; val[s] = 0;
    while(!q.empty()){
        int x = q.front(); q.pop();
        for(auto [y, w] : g[x]){
            if(!vis[y]) val[y] = val[x] + w, vis[y] = 1, q.push(y);
            else if(val[y] != val[x] + w) ok = 0;
        }
    }
}
```

### 复杂度

时间复杂度 $O(n+m)$，空间复杂度 $O(n+m)$。

## 17. [1726：矩阵](http://ybt.ssoier.cn:8088/problem_show.php?pid=1726)

`二分图赋值` `等式约束` `DFS`

### 题意

初始全零矩阵，可对整行或整列做 $\pm1$ 操作。给出若干格子的最终值，判断能否实现。

### 分析

若把第 $i$ 行累计操作量记为 $R_i$，第 $j$ 列累计操作量记为 $C_j$，那么格子 $(i,j)$ 的值永远都能写成 $R_i+C_j$。于是每条限制都变成一个等式：$R_x+C_y=c$。

这已经不是矩阵题了，而是一张二分图上的赋值题：左边是行变量，右边是列变量，每条限制边规定两端和为某个常数。任选一个未定的行值作为起点，就能顺着边把整块中的其余变量都推出；若某个变量被推出两次且值不同，就是无解。

### 核心代码

```cpp
for(auto [x, y, c] : cons){
    g[row(x)].push_back({col(y), c});
    g[col(y)].push_back({row(x), c});
}
for(int s = 1; s <= n + m; ++s) if(!vis[s]){
    val[s] = 0; dfs(s);
}
void dfs(int x){
    vis[x] = 1;
    for(auto [y, c] : g[x]){
        long long need = is_row(x) ? c - val[x] : c - val[x];
        if(!vis[y]) val[y] = need, dfs(y);
        else if(val[y] != need) ok = 0;
    }
}
```

时间复杂度 $O(k)$，空间复杂度 $O(k+n+m)$。

## 18. [1727：魔棒](http://ybt.ssoier.cn:8088/problem_show.php?pid=1727)

`二分答案` `区间 DP` `生存判定`

### 题意

英雄每秒受一次伤害，魔棒每秒多一格能量，释放时恢复 $15\times$ 当前能量，且两次释放间隔至少为 $cd$。问在始终存活的前提下，$cd$ 最大能是多少。

### 分析

一眼看上去像模拟，但真正要判的是“某个施法间隔是否可行”，于是先二分 $cd$。固定 $cd$ 后，关键观察是：如果上一次施法在时刻 $j$，下一次施法在时刻 $i$，那么这整段的净血量变化只和区间伤害和与长度有关，是否能撑住则取决于区间中最危险的前缀。

因此可以做区间 DP。令 `dp[i]` 表示处理完前 $i$ 秒后、且最优安排下还能留下的最大生命值。枚举最后一次施法覆盖的区间 $[j,i]`，只要长度满足间隔限制、并且这段过程中任意前缀都不会把生命压到零以下，就能从 `dp[j-1]` 转移过来。因为 $n\le 500$，直接 $O(n^2)$ 判定足够，再套一层二分即可。

### 核心代码

```cpp
bool check(int cd){
    fill(dp.begin(), dp.end(), -INF); dp[0] = hp;
    for(int i = 1; i <= n; ++i){
        long long sum = 0, worst = 0;
        for(int j = i; j >= 1; --j){
            sum += a[j];
            worst = min(worst, -sum);
            if(i - j + 1 >= cd && dp[j - 1] + worst > 0)
                dp[i] = max(dp[i], dp[j - 1] - sum + 15LL * (i - j + 1));
        }
    }
    return dp[n] > -INF;
}
```

### 复杂度

时间复杂度 $O(n^2\log n)$，空间复杂度 $O(n)$。

## 19. [1728：构造序列](http://ybt.ssoier.cn:8088/problem_show.php?pid=1728)

`差分约束` `线段树优化建图` `构造`

### 题意

给定部分已知位置的数值，以及若干限制：区间 $[l,r]$ 中给出的 $k$ 个位置都严格大于其余位置。要求构造任意一组满足条件的正整数序列，或判断无解。

### 分析

“这些位置都比那些位置大”最适合先抽象成等级关系。若把每个位置的值视为一个层级，那么题目的限制就是：对每个被点名的位置 $x$ 和区间内未被点名的位置 $y$，都有 $a_x\ge a_y+1$。这正是带权有向边 `y -> x (1)` 的最长路约束。

难点在于一条限制里会出现大量“未被点名的位置”，不能一对一连边。标准做法是用线段树把区间拆成 $O(\log n)$ 个节点，让“区间所有普通位置”统一向“被点名点”连权值为 $1$ 的边，再把已知数值也转成上下界约束。若图中有正环则无解；否则在 DAG / 缩点图上跑最长路，最后按层级回填成正整数即可。

### 核心代码

```cpp
for(auto rule : rules){
    auto segs = split_other_pos(rule.l, rule.r, rule.pick);
    for(int x : rule.pick)
        for(int p : segs) add(p, id[x], 1);
}
for(auto [pos, val] : fixed) add(S, id[pos], val), add(id[pos], T, -val);
topo();
for(int x : ord)
    for(auto [y, w] : g[x]) dis[y] = max(dis[y], dis[x] + w);
```

`二分图匹配` `交替路` `强连通分量`

### 题意

## 22. [1731：最大流](http://ybt.ssoier.cn:8088/problem_show.php?pid=1731)

`Gomory-Hu 树` `单位容量` `全点对最小割`

### 题意

无向图每条边容量都是 $1$，要求所有点对间最大流之和。

### 分析

无向图里最大流等于最小割，而“所有点对最小割”最适合被一棵 Gomory-Hu 树统一表示。树上任意两点路径上的最小边权，就是原图中这对点的最小割，也就是最大流。

为什么这题能过？关键在“度数不超过 $3$、容量全为 $1$”。这意味着任意一次 s-t 最大流的值最多只有 $3$，Dinic / ISAP 每次只需要很少的增广轮数。于是用 $n-1$ 次单位容量最大流建 Gomory-Hu 树完全可做。树建好以后，把边按权值从大到小并查集合并，统计每条树边作为路径最小值时贡献了多少点对即可。

### 核心代码

```cpp

    int t = fa[s];

    tree.push_back({s, t, cut});
    for(int v = s + 1; v <= n; ++v)
        if(fa[v] == t && side[v]) fa[v] = s;
    if(side[fa[t]]) swap(fa[s], fa[t]);
}
sort(tree.begin(), tree.end(), [](auto a, auto b){ return a.w > b.w; });
for(auto [u, v, w] : tree) ans += 1LL * sz[find(u)] * sz[find(v)] * w, unite(u, v);
```

### 复杂度

## 23. [1732：情报传递](http://ybt.ssoier.cn:8088/problem_show.php?pid=1732)

`可达性构造` `缩点` `偏序`

### 题意

给出若干“必须能到达”和“必须不能到达”的要求，要求构造一张有向图满足全部条件，或判断无解。

### 分析

先只看“必须能到达”。若同时要求 $a\to b$ 与 $b\to a$，这两个人最终一定在同一个强连通分量里，所以先把这些互相可达关系缩起来。缩点之后，剩下的“必须能到达”关系是一张 DAG 上的偏序。

接着检查“必须不能到达”是否与这个偏序冲突：一旦某个禁止对在缩点后已经存在路径，那就无解。若无冲突，只需在 DAG 上保留一组能实现全部正向可达性的覆盖边即可。换句话说，题目不是让你还原唯一原图，而是让你构造一张 reachability 相同、边数尽量省的偏序图。

### 核心代码

```cpp
build_pos_graph();
tarjan();
for(auto [u, v] : neg)
    if(bel[u] == bel[v] || reachable(bel[u], bel[v])) return no();
for(auto [u, v] : pos)
    if(bel[u] != bel[v]) dag[bel[u]].push_back(bel[v]);
transitive_reduce(dag);
output_edges();
```

### 复杂度

缩点部分时间复杂度 $O(n+m+t)$；后续在 DAG 上做可达性检查与约简，取决于具体实现。

## 24. [1733：连续数字区间](http://ybt.ssoier.cn:8088/problem_show.php?pid=1733)

`区间树` `排列` `LCA`

### 题意

给定一个排列。对每次询问区间 $[x,y]$，要找一个最短连续区间 $[l,r]$ 包含它，并且其中的数值集合排完序后恰好是连续整数。

### 分析

一个区间在排列里“连续”，等价于区间内最大值减最小值正好等于长度减一。对于排列而言，这类最小包围区间会形成一套天然的包含关系：两个最小连续区间不是相离，就是一方包含另一方，因此可以组织成一棵区间树。

建树时可以从相邻值的位置关系出发，不断把必须合并的区间往上提；每个树节点代表一个最小连续段。这样一来，询问 $[x,y]$ 的答案就是“同时覆盖位置 $x$ 与 $y$ 的最小树节点”，也就是区间树上的 LCA。

### 核心代码

```cpp
build_interval_tree();
dfs(root, 0);
while(m--){
    int x, y; cin >> x >> y;
    int p = lca(posNode[x], posNode[y]);
    cout << L[p] << ' ' << R[p] << '
';
}
```

### 复杂度

建树与预处理复杂度 $O(n\log n)$，每次询问 $O(\log n)$，空间复杂度 $O(n\log n)$。

## 25. [1734：删边问题](http://ybt.ssoier.cn:8088/problem_show.php?pid=1734)

`桥` `树形 DP` `连通块`

### 题意

多次询问删除某条边后，会新增多少对不能互相到达的点对。

### 分析

非桥边删掉以后，原连通性不变，答案只等于原图本来的不可达点对数。真正会改变答案的只有桥。于是先按连通块统计一遍原图本来就互不可达的点对数，作为所有询问的基底。

然后在每个连通块里找桥。若一条桥把这个块切成大小为 $s$ 和 $tot-s$ 的两部分，那么删掉它以后，恰好多出 $s(tot-s)$ 对新不可达点。树上 DFS 出每条桥子树大小即可，询问时 O(1) 回答。

### 核心代码

```cpp
tarjan_bridge();
for(int i = 0; i < m; ++i) if(isBridge[i]){
    int s = min(sub[edge[i].u], sub[edge[i].v]);
    extra[i] = 1LL * s * (compSize[cc[i]] - s) % 1000;
}
for(int x : query) cout << (base + extra[x]) % 1000 << '
';
```

### 复杂度

时间复杂度 $O(n+m+Q)$，空间复杂度 $O(n+m)$。

## 26. [1735：环上计数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1735)

`边双连通分量` `桥` `环计数`

### 题意

统计多少条边不在任何简单环上，以及多少条边至少在两个简单环上。

### 分析

第一问最直接：不在任何简单环上的边就是桥。因为一条边若处在环上，就能绕另一侧回去，不可能是桥；反过来，非桥必在某个简单环里。

```cpp
tarjan_bridge();
shrink_edcc();
for(int c = 1; c <= scc; ++c){

    else multiCycle += E[c];
}
cout << bridgeCnt << ' ' << multiCycle << '
';
```

### 复杂度

时间复杂度 $O(n+m)$，空间复杂度 $O(n+m)$。

## 27. [1736：最大连通块](http://ybt.ssoier.cn:8088/problem_show.php?pid=1736)

`点双` `因数分解` `割点`

### 题意

两点间若点权 gcd 为合数就连边。上帝删去一个点后，要让剩余图中的最大连通块尽量小，求这个最小值。

### 分析

先别真按点对建图。`gcd` 为合数，说明两数至少共享一个“足够强”的公共因子：要么某个质因子次数至少为 $2$，要么共享两个不同质因子。把每个数分解质因子后，可以把这类“能导致合数 gcd 的因子模式”当作中介节点，把原图稀疏化成“数字点 - 因子模式点”的关系图，再据此得到真实连通结构。

之后题目就回到经典的“删一个点后最大连通块最小”。对每个连通块建块割树，删除某个原图点时，会把与它相邻的若干块分裂成多部分；枚举这个点，取被切出的最大部分大小，再和块外剩余点数比较即可。真正的难点不是 Tarjan，而是先把 gcd 连边压缩到可处理的规模。

### 核心代码

```cpp
for(int i = 1; i <= n; ++i){
    auto fac = factor(a[i]);
    for(auto key : useful_keys(fac)) link(i, id[key]);
}
tarjan_block_cut();
for(int x = 1; x <= n; ++x){
    int mx = total - compSize[x];
    for(int y : cutSon[x]) mx = max(mx, sub[y]);
    ans = min(ans, mx);
}
```

### 复杂度

时间复杂度由分解与 Tarjan 组成，约为 $O(n\log A + E)$，空间复杂度 $O(E)$。

## 28. [1737：邮递员](http://ybt.ssoier.cn:8088/problem_show.php?pid=1737)

`欧拉回路` `强制片段` `Hierholzer`

### 题意

要求找一条经过每条有向边恰好一次、并回到起点的欧拉回路，同时必须包含若干给定的路径片段。

### 分析

若没有片段限制，这就是标准欧拉回路。但“必须包含一段连续片段”意味着某些边的先后次序被固定了：在片段内部，一个点的某条出边必须紧跟在某条入边之后。于是可以先把这些被强制连接的边链压成超级边。

压缩后，只要每个点的入度等于出度、并且没有出现“同一个入口被要求接两条不同后继”这类冲突，就还能在压缩图上跑 Hierholzer。最后把超级边再按原片段展开，得到完整答案。也就是说，片段限制并没有改变欧拉本质，只是先把局部顺序钉死了。

### 核心代码

```cpp
for(auto path : paths)
    for(int i = 0; i + 1 < path.size(); ++i)
        force(edge(path[i], path[i + 1]));
contract_forced_chains();
if(!check_degree()) return puts("NIE"), 0;
hierholzer(start);
reverse(ans.begin(), ans.end());
```

### 复杂度

时间复杂度 $O(n+m+\sum k)$，空间复杂度同阶。

## 29. [1738：圆桌聚会](http://ybt.ssoier.cn:8088/problem_show.php?pid=1738)

`欧拉回路` `构造` `de Bruijn`

### 题意

给定编号规则，要求把 $2n$ 个人排成一圈，使得每个人顺时针的下一个编号满足倍增式约束。

### 分析

这种“下一个元素由当前元素按二进制规则决定”的题，第一反应就该往 de Bruijn 图上靠。把一个编号看成状态，允许的后继正好对应向左移位再补一位 $0/1$。于是题目要找的不是一般 Hamilton 环，而是某张二出二入有向图上的欧拉回路，再把走过的边序列解码成座位顺序。

这样想的好处是，欧拉回路天然线性可构造，而且不会被“每个人只出现一次”这件事卡住，因为唯一性已经由边访问次数保证了。真正的构造只是在模 $2n$ 的编号系里把这条欧拉回路翻译回人名次序。

### 核心代码

```cpp
void dfs(int x){
    for(int b = 0; b < 2; ++b){
        int y = trans(x, b);
        int id = edge_id(x, b);
        if(vis[id]) continue;
        vis[id] = 1;
        dfs(y);
        seq.push_back(y);
    }
}
dfs(0);
output_decode(seq);
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 30. [1739：观光巴士](http://ybt.ssoier.cn:8088/problem_show.php?pid=1739)

`欧拉回路` `局部顺序 DP` `时刻表`

### 题意

要从 $1$ 号站出发，恰好乘坐每条巴士线路一次并回到 $1$ 号站。每条线路只有固定发车时刻和行驶时间，等待也算观光时间，求总停留时间最小。

### 分析

边都必须恰好走一次，所以骨架仍然是一条欧拉回路。不同之处在于：经过某个站时，两条出边谁先走、谁后走，会改变到达后续站点时的模 $24$ 时刻，从而改变等待时间。也就是说，这题不是在问“有没有欧拉回路”，而是在问“每个点的局部出边顺序怎样排最优”。

由于每个点只有两条出边，局部选择只有两种。把整张图按欧拉结构拆成若干环 / 链以后，每个块只需记录首尾时刻模 $24$ 下的最优代价，最后做一次环上 DP 合并即可。核心思路是：欧拉约束确定了必须走什么，DP 只负责优化局部顺序带来的等待损失。

### 核心代码

```cpp
for(int u = 1; u <= n; ++u){
    trans[u][0] = cost(first[u], second[u]);
    trans[u][1] = cost(second[u], first[u]);
}
for(auto cyc : cycles){
    init_dp(cyc[0]);
    for(int u : cyc) push(u);
    ans = min(ans, close_cycle(cyc));
}
```

### 复杂度

时间复杂度 $O(n\cdot 24^2)$ 或同数量级，空间复杂度 $O(n\cdot 24)$。

## 31. [1740：星空穿越](http://ybt.ssoier.cn:8088/problem_show.php?pid=1740)

`欧拉化` `路径分解` `树上配对`

### 题意

若一条边当前被经过了奇数次，就需要安排若干个人各走一条路径，使得所有边最终都被经过偶数次，并让人数尽量少。

### 分析

一条路径会把它经过的边奇偶性整体翻转一次，所以题目本质上是在问：如何用最少条路径，把当前“奇边集合”表示出来。把所有奇边拿出来看成一张子图，其中度数为奇的点必须作为某些路径的端点，因此最少路径数至少是奇度点数的一半。

而在连通块内，这个下界总能达到：把奇边子图不断做欧拉分解，或在一棵 DFS 树上自底向上把未配对端点往上送，遇到两端就合并成一条路径。若某个连通块里没有奇度点但仍有奇边，那它自身需要贡献一条闭合路径。于是整题就是“按连通块做最小路径分解”。

### 核心代码

```cpp
void dfs(int x, int f){
    for(auto [y, id] : g[x]) if(id != f){
        dfs(y, id);
        if(rem[y] != -1) link_path(rem[y], x), rem[y] = -1;
    }
    if(degOdd[x]){
        if(rem[x] == -1) rem[x] = x;
        else output_path(rem[x], x), rem[x] = -1;
    }
}
for(int s = 1; s <= n; ++s) if(!vis[s] && hasOddEdge[s]) dfs(s, 0);
```

### 复杂度

时间复杂度 $O(n+m)$，空间复杂度 $O(n+m)$。
