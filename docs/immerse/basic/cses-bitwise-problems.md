# CSES 位运算专题精选解题报告

这一组题从最基础的按位计数一路走到线性基、异或金字塔、SOS DP 和 XOR 卷积，题型跨度不小，但核心一直没变：先把十进制题面翻回二进制结构，再决定是逐位统计、在异或线性空间里做贪心，还是把整张掩码空间当成状态整体推进。前半段偏线性空间与前缀异或，后半段则转向杨辉三角奇偶、高维变换和按位反演。

# 一、按位贡献、前缀异或与线性空间

这一章先把最基础的按位统计、前缀异或和线性基串起来。它们看上去题型不同，但共同点都在于：先把原问题压成“二进制位的独立贡献”或“异或线性空间里的组合”。

## 1. [Counting Bits](https://cses.fi/problemset/task/1146)

`按位统计` `循环节`

### 题意

给定整数 $n$，要求统计从 $1$ 到 $n$ 的所有整数二进制表示中，数字 $1$ 一共出现了多少次。

### 分析

把答案拆到每一位单独统计最稳。对于第 $b$ 位，它会以长度 $2^{b+1}$ 为一个完整循环：前 $2^b$ 个数这一位为 $0$，后 $2^b$ 个数这一位为 $1$。

所以只要计算完整循环的个数，再处理最后一个残缺区间，就能直接得到这一位的贡献。把所有位的贡献累加起来，整个题就从“逐个数二进制”变成了一个按位分块公式。

### 核心代码

```cpp
long long ans = 0;
for (int b = 0; (1LL << b) <= n; ++b) {
    long long len = 1LL << (b + 1);
    long long full = (n + 1) / len, rem = (n + 1) % len;
    ans += full * (1LL << b) + max(0LL, rem - (1LL << b));
}
cout << ans;
```

### 复杂度

时间复杂度 $O(\log n)$，空间复杂度 $O(1)$。

## 2. [Maximum Xor Subarray](https://cses.fi/problemset/task/1655)

`前缀异或` `01-Trie`

### 题意

给定一个数组，要求找出某个连续子数组的异或和最大值。

### 分析

子数组异或和天然要想到前缀异或：设 $px_i=a_1\oplus a_2\oplus\cdots\oplus a_i$，那么区间 $[l,r]$ 的异或和就是 $px_r\oplus px_{l-1}$。于是问题就变成：对每个当前前缀异或值，去历史前缀里找一个与它异或后尽量大的数。

这正是 01-Trie 的经典用途。把已经出现过的前缀异或逐个插入 Trie，查询时从高位到低位优先走相反的比特，就能贪心地把答案的高位尽量变成 $1$。

### 核心代码

```cpp
insert(0);
int px = 0, ans = 0;
for (int x : a) {
    px ^= x;
    ans = max(ans, query_max(px));
    insert(px);
}
cout << ans;
```

### 复杂度

时间复杂度 $O(n\log V)$，空间复杂度 $O(n\log V)$。

## 3. [Maximum Xor Subset](https://cses.fi/problemset/task/3191)

`线性基` `贪心`

### 题意

给定一个数组，要求从中任选若干个数，使它们的异或和最大。

### 分析

“子集异或最大值”几乎就是线性基的招牌题。把每个数依次插入按最高位维护的基底里，等价于对这些数在 $\mathrm{GF}(2)$ 上做高斯消元。

基底建好后，再从高位往低位贪心尝试：如果当前答案与某个基向量异或后更大，就把它并进来。因为高位的优先级最高，而线性基保证了每个主元位都能独立控制，所以这个贪心就是正确的。

### 核心代码

```cpp
for (int x : a) insert_basis(x);
int ans = 0;
for (int b = LOG - 1; b >= 0; --b) {
    if ((ans ^ basis[b]) > ans) ans ^= basis[b];
}
cout << ans;
```

### 复杂度

时间复杂度 $O(n\log V)$，空间复杂度 $O(\log V)$。

## 4. [Number of Subset Xors](https://cses.fi/problemset/task/3211)

`线性基` `秩`

### 题意

给定一个数组，要求统计所有子集异或值中，不同结果一共有多少个。

### 分析

这里不是问某个具体最值，而是问“异或空间有多大”。线性基一旦建好，所有子集异或值恰好就是这个基底张成的整个线性空间。

若线性基的秩为 $r$，那就意味着有 $r$ 个独立方向可以自由选择，每个方向选或不选都对应一种组合，因此不同的子集异或值个数就是 $2^r$。原数组里那些能被消掉的相关向量，只会制造重复，不会增加新结果。

### 核心代码

```cpp
int rk = 0;
for (int x : a) rk += insert_basis(x);
long long ans = 1LL << rk;
cout << ans;
```

### 复杂度

时间复杂度 $O(n\log V)$，空间复杂度 $O(\log V)$。

## 5. [K Subset Xors](https://cses.fi/problemset/task/3192)

`线性基` `第 k 小`

### 题意

考虑数组所有 $2^n$ 个子集的异或值（包括空集得到的 $0$），要求输出其中最小的 $k$ 个，且重复值也要按出现次数输出。

### 分析

先用线性基把所有不同的子集异或值压成一个秩为 $r$ 的空间。关键观察是：若原数组长度为 $n$，那么每个不同的异或值都会被恰好 $2^{n-r}$ 个子集产生，因为那些被消掉的相关向量只负责“选或不选都不改变结果”。

接下来要做的是把这些不同异或值按从小到大枚举出来。把线性基进一步化成最简形并按主元位从低到高排序后，每个基向量都会引入一个比前面所有向量都更高的新二进制位，于是按二进制顺序枚举组合，得到的正好就是从小到大的不同异或值。最后再把每个值重复输出 $2^{n-r}$ 次即可。

### 核心代码

```cpp
build_basis(a);
normalize_basis();
long long rep = (n - rk >= 60 ? k : min<long long>(k, 1LL << (n - rk)));
for (long long id = 0, used = 0; used < k; ++id) {
    long long val = kth_xor(id);
    for (long long t = 0; t < rep && used < k; ++t, ++used) out.push_back(val);
}
```

### 复杂度

时间复杂度 $O(n\log V + k\log V)$，空间复杂度 $O(\log V)$。

# 二、异或金字塔与杨辉三角奇偶

这几题的主角都是异或金字塔。真正的核心不是模拟每一层，而是认出系数来自杨辉三角，再用 Lucas 定理把“奇数项留下来”这件事翻成位运算。

## 6. [Xor Pyramid Peak](https://cses.fi/problemset/task/2419)

`Lucas` `组合奇偶`

### 题意

给定异或金字塔的最底层，要求求出不断向上异或之后的塔顶数值。

### 分析

把塔顶对底层每个位置的贡献展开，会得到帕斯卡三角形第 $n-1$ 行的组合系数，只不过这里一切都在模 $2$ 下计算，因为“加法”已经变成了异或。于是塔顶就是所有满足 $C(n-1,i)$ 为奇数的位置的异或和。

而组合数奇偶最经典的判定就是 Lucas 定理的模 $2$ 版本：$C(n-1,i)$ 为奇数，当且仅当 $i$ 的每个二进制 $1$ 位都出现在 $n-1$ 的对应位置里。这样一来，问题就从一层层往上推，变成了直接扫底层并判断哪些位置会留下来。

### 核心代码

```cpp
int ans = 0;
for (int i = 0; i < n; ++i) {
    if ((i & (n - 1 - i)) == 0) ans ^= a[i];
}
cout << ans;
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(1)$。

## 7. [Xor Pyramid Diagonal](https://cses.fi/problemset/task/3194)

`SOS DP` `子集变换`

### 题意

给定异或金字塔底层，要求输出从底到顶每一层最左边的那个数。

### 分析

若从底层往上走了 $t$ 步，那么这一层最左端的值会由底层若干位置异或得到，其系数仍然是第 $t$ 行杨辉三角在模 $2$ 下的奇偶。也就是说，第 $t$ 个答案恰好等于所有满足 $s\subseteq t$ 的 $a_{s+1}$ 的异或和。

这已经不再是普通前缀，而是一个标准的“子集和型”变换。把 $f[t]$ 初始设为底层第 $t+1$ 个元素，然后对每一位做一次子集 DP：若当前掩码这一位为 $1$，就把去掉这一位的值异或进来。处理完之后，$f[t]$ 就正好是第 $t$ 层左端答案。

### 核心代码

```cpp
for (int i = 0; i < n; ++i) f[i] = a[i];
for (int b = 0; b < LOG; ++b) {
    for (int mask = 0; mask < M; ++mask) if (mask >> b & 1) {
        f[mask] ^= f[mask ^ (1 << b)];
    }
}
for (int t = 0; t < n; ++t) print(f[t]);
```

### 复杂度

时间复杂度 $O(M\log M)$，空间复杂度 $O(M)$，其中 $M$ 是不小于 $n$ 的二次幂。

## 8. [Xor Pyramid Row](https://cses.fi/problemset/task/3195)

`Lucas` `位分解`

### 题意

给定异或金字塔底层和从顶往下数的第 $k$ 行，要求输出这一整行的所有数字。

### 分析

若目标行有 $k$ 个数，那么它距离底层还差 $d=n-k$ 次向上合并。直接做 $d$ 轮显然不行，但在异或金字塔里，走 $2^b$ 层之后的转移特别整齐：新的第 $i$ 个数就是旧数组的 $a_i\oplus a_{i+2^b}$。

因此只要把 $d$ 拆成二进制。对于 $d$ 的每一个置位 $2^b$，就对当前数组做一次“与右边相隔 $2^b$ 的元素异或”的批量更新。等所有置位都处理完，前面的 $k$ 个数就正好是目标行。这个过程本质上仍然来自 Lucas 定理，只是把组合奇偶改写成了可执行的位分解更新。

### 核心代码

```cpp
int d = n - k;
for (int b = 0; (1 << b) <= d; ++b) if (d >> b & 1) {
    for (int i = 0; i + (1 << b) < n; ++i) {
        a[i] ^= a[i + (1 << b)];
    }
}
for (int i = 0; i < k; ++i) print(a[i]);
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(1)$（原地更新时）。

# 三、SOS DP、卷积与按位反演

最后三题已经不再是局部贪心或单个线性基，而是要把整张掩码空间当成状态整体推进：有的做 XOR 卷积，有的做 SOS DP，有的还要在超集格上反演回来。

## 9. [All Subarray Xors](https://cses.fi/problemset/task/3233)

`前缀异或` `FWHT`

### 题意

给定一个数组，要求找出所有能作为某个连续子数组异或和出现的整数，并按升序输出。

### 分析

子数组异或仍然要先压成前缀异或差：任何区间答案都可以写成两个前缀异或值的异或。因此题目等价于：给定所有前缀异或值的多重集合，求哪些数能写成其中两个元素的异或。

这已经是标准的 XOR 卷积模型。统计每个前缀异或值出现次数后，对频次数组做一次 FWHT，点值平方，再逆变换回来，得到的就是每个异或结果对应的有序对数量。数量大于 $0$ 的值都可达；只有 $0$ 需要额外判断某个前缀异或是否出现了至少两次，以排除“同一个位置和自己配对”的伪贡献。

### 核心代码

```cpp
cnt[0]++;
for (int x : a) px ^= x, cnt[px]++;
fwht(cnt, 1);
for (int i = 0; i < M; ++i) cnt[i] *= cnt[i];
fwht(cnt, -1);
for (int x = 0; x < M; ++x) if (ok(x, cnt[x], freq)) ans.push_back(x);
```

### 复杂度

时间复杂度 $O(M\log M)$，空间复杂度 $O(M)$。

## 10. [SOS Bit Problem](https://cses.fi/problemset/task/1654)

`SOS DP` `掩码统计`

### 题意

对每个数 $x$，分别要求统计数组中有多少个 $y$ 满足 $x\mid y=x$、$x\mathbin{\&}y=x$ 以及 $x\mathbin{\&}y\neq 0$。

### 分析

三问看起来不同，其实都在问“某些数是否是当前掩码的子集 / 超集 / 非互斥集合”。这正是 SOS DP 最适合的场景：把每个值出现次数记到频次数组上，然后批量求出每个掩码的所有子掩码和、所有超掩码和。

设 `sub[mask]` 表示值是 `mask` 子掩码的元素个数，`sup[mask]` 表示值是 `mask` 超掩码的元素个数。那么第一问就是 `sub[x]`，第二问就是 `sup[x]`。第三问则改成补集更好算：与 $x$ 按位与为 $0$ 的那些数，恰好都是 `ALL ^ x` 的子掩码，所以答案是 $n-\mathrm{sub}[ALL\oplus x]$。

### 核心代码

```cpp
sub = sup = freq;
for (int b = 0; b < B; ++b)
    for (int mask = 0; mask < (1 << B); ++mask) {
        if (mask >> b & 1) sub[mask] += sub[mask ^ (1 << b)];
        else sup[mask] += sup[mask | (1 << b)];
    }
for (int x : a) print(sub[x], sup[x], n - sub[ALL ^ x]);
```

### 复杂度

时间复杂度 $O(B2^B)$，空间复杂度 $O(2^B)$。

## 11. [And Subset Count](https://cses.fi/problemset/task/3141)

`按位反演` `莫比乌斯反演`

### 题意

给定数组，要求对每个 $k=0,1,\ldots,n$，统计按位与恰好等于 $k$ 的非空子集个数。

### 分析

直接按“与值恰好等于某个掩码”统计会很别扭，更顺手的方向是先数“与值至少包含某个掩码”。若一个元素满足 $a_i\mathbin{\&}mask=mask$，就说明它可以出现在所有按位与包含 `mask` 的子集里。

于是先求出每个 `mask` 被多少个数组元素完整覆盖，记为 `cover[mask]`，那么这些元素形成的非空子集一共有 $2^{\mathrm{cover}[mask]}-1$ 个，它们的按位与都至少是 `mask`。最后再在超集格上做一次莫比乌斯反演，把“至少是”还原成“恰好是”，就能得到每个答案。

### 核心代码

```cpp
cover = freq;
for (int b = 0; b < B; ++b)
    for (int mask = 0; mask < (1 << B); ++mask)
        if (!(mask >> b & 1)) cover[mask] += cover[mask | (1 << b)];
for (int mask = 0; mask < (1 << B); ++mask) f[mask] = pw2[cover[mask]] - 1;
for (int b = 0; b < B; ++b)
    for (int mask = 0; mask < (1 << B); ++mask) if (!(mask >> b & 1)) f[mask] = (f[mask] - f[mask | (1 << b)] + MOD) % MOD;
```

### 复杂度

时间复杂度 $O(B2^B)$，空间复杂度 $O(2^B)$。
