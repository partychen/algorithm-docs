# 洛谷 DP优化专题精选解题报告

这一组题从分治优化、斜率优化一路走到单调队列与二维滑窗，做的都是同一件事：把原本要枚举的一大段决策压成单调性、凸性或窗口最值。比起背模板，更重要的是先认出“朴素转移慢在哪里”，再决定该借数学性质还是数据结构砍掉枚举。

# 一、分治决策与区间代价

这一章先把“代价可快速维护、最优断点会顺着终点移动”这一类基础优化建立起来。

## 1. [CF868F Yet Another Minimization Problem](https://www.luogu.com.cn/problem/CF868F)

`分治优化` `区间代价` `决策单调性`

### 题意

把序列划成 `k` 段，每段费用是段内相同元素形成的数对数，求最小总代价。

### 分析

这种“把序列切成 `k` 段”的题，第一反应就是枚举最后一段从哪里开始。于是设 `dp[t][i]` 表示前 `i` 个数分成 `t` 段的最小代价，那么最后一段若是 `j+1..i`，就有

`dp[t][i] = min(dp[t-1][j] + cost(j+1, i))`。

真正的难点在 `cost(l,r)`：一段里的费用不是区间和，而是“相同元素形成的数对数”。这提示我们不能只靠前缀和，而要维护一个会随左右端点变化的区间代价。好在当左右端点每次只移动一格时，新增或删除一个数对总费用的影响是可增量维护的，所以可以像莫队那样用双指针维护当前窗口费用。

此时朴素 DP 仍然是 `O(kn^2)`，瓶颈只剩“每个 `i` 都要枚举全部 `j`”。而这类代价函数满足分治优化常见的决策单调性：右端点越往右，最优断点不会往左跳。于是每一层 DP 都可以改成“在一段区间内求中点最优断点，再递归处理左右两半”，把整行转移压到 `O(n log n)`。

### 核心代码

```cpp
long long cost;
void add(int x){ cost += cnt[a[x]]; cnt[a[x]]++; }
void del(int x){ cnt[a[x]]--; cost -= cnt[a[x]]; }
void solve(int l, int r, int ql, int qr, int t) {
    int mid = (l + r) >> 1, best = -1;
    pair<long long, int> res = {INF, -1};
    for (int j = ql; j <= min(mid - 1, qr); j++) {
        move_window(j + 1, mid);
        res = min(res, {pre[j] + cost, j});
    }
    dp[t][mid] = res.first; best = res.second;
    if (l < mid) solve(l, mid - 1, ql, best, t);
    if (mid < r) solve(mid + 1, r, best, qr, t);
}
```

### 复杂度

用可移动区间费用配合分治优化后，常见实现复杂度为 `O(k n log n)`，空间复杂度 `O(n)`。

## 2. [P4767 [IOI 2000] 邮局 加强版](https://www.luogu.com.cn/problem/P4767)

`四边形不等式` `区间代价` `决策单调性`

### 题意

有序村庄上建立若干邮局，使每个村庄到最近邮局的距离和最小。

### 分析

先别急着想 `m` 个邮局怎么放，先想一个邮局负责一整段村庄时最优放哪。对一段有序坐标来说，把邮局放在中位数位置一定最优，因此 `w[l][r]` 可以预处理成“第 `l..r` 个村庄共用一个邮局时的最小代价”。

这样原题就自然变成分段 DP：设 `dp[p][i]` 表示前 `i` 个村庄用 `p` 个邮局服务时的最小总代价，若最后一个邮局负责的是 `j+1..i`，就有

`dp[p][i] = min(dp[p-1][j] + w[j+1][i])`。

如果这一步直接枚举断点，复杂度还是太高。关键观察是：这个代价函数满足四边形不等式，最优断点会随着 `i` 的增大单调移动。于是我们不需要每次都从头枚举，而可以把枚举范围限制在前后相邻状态的最优决策之间，这就是这题能做 Knuth 型优化的根本原因。

### 核心代码

```cpp
for (int l = 1; l <= n; l++)
    for (int r = l; r <= n; r++) {
        int mid = (l + r) >> 1;
        w[l][r] = pre[r] - pre[mid] - x[mid] * (r - mid)
                + x[mid] * (mid - l + 1) - (pre[mid] - pre[l - 1]);
    }
for (int p = 1; p <= m; p++) {
    opt[p][n + 1] = n - 1;
    for (int i = n; i >= 1; i--) {
        dp[p][i] = INF;
        for (int j = opt[p - 1][i]; j <= opt[p][i + 1]; j++) {
            long long v = dp[p - 1][j] + w[j + 1][i];
            if (v < dp[p][i]) dp[p][i] = v, opt[p][i] = j;
        }
    }
}
```

### 复杂度

预处理区间代价为 `O(n^2)`，Knuth 型优化后的 DP 为 `O(mn)`，空间复杂度 `O(mn)`。

## 3. [P1880 [NOI1995] 石子合并](https://www.luogu.com.cn/problem/P1880)

`区间 DP` `环形展开` `四边形不等式`

### 题意

有 `n` 堆石子围成一个环，每次只能把相邻两堆合并，代价等于这次合并后新堆的石子总数。要求同时求出把它们合成一堆时的最小得分和最大得分。

### 分析

环形区间题第一步几乎总是“断环成链”。把序列复制一遍后，任取长度为 `n` 的连续区间，就对应原环上选择一个断点后的展开结果；这样原题就从“环上怎么合并”变成了“所有可能断点里哪条链最优”。

接着做经典区间 DP。设 `mn[l][r]`、`mx[l][r]` 分别表示把区间 `[l,r]` 合并成一堆的最小和最大代价，那么最后一次合并一定把它切成 `[l,k]` 和 `[k+1,r]` 两部分，再加上整段区间和 `sum(l,r)`。这一步其实就是区间 DP 最核心的套路：最后一次操作把大区间拆成两个更小的独立子问题。

真正的区别在于最小值和最大值能不能继续优化。最小值部分满足四边形不等式，所以最优断点会单调移动，可以用 `opt[l][r-1]..opt[l+1][r]` 缩小枚举范围；而最大值部分通常不具备这个性质，只能老老实实做普通区间转移。

所以这题最该建立的判断是：**同一个区间 DP，求最小和求最大未必共享同一种优化性质**。不要因为最小值能套 Knuth/四边形不等式，就下意识认为最大值也能一起优化。

### 核心代码

```cpp
for (int len = 2; len <= n; len++) {
    for (int l = 1; l + len - 1 <= 2 * n; l++) {
        int r = l + len - 1;
        mn[l][r] = INF, mx[l][r] = -INF;
        for (int k = opt[l][r - 1]; k <= opt[l + 1][r]; k++) {
            long long s = pre[r] - pre[l - 1];
            long long v = mn[l][k] + mn[k + 1][r] + s;
            if (v < mn[l][r]) mn[l][r] = v, opt[l][r] = k;
        }
        for (int k = l; k < r; k++)
            mx[l][r] = max(mx[l][r], mx[l][k] + mx[k + 1][r] + pre[r] - pre[l - 1]);
    }
}
```

### 复杂度

最小值部分可做到 `O(n^2)`，若最大值保留朴素区间转移则总体为 `O(n^3)`，空间复杂度 `O(n^2)`。

# 二、斜率优化的线性决策

当转移式能展开成“截距 + 斜率 × 横坐标”时，就可以把 DP 决策变成凸包上的直线查询。

## 4. [CF311B Cats Transport](https://www.luogu.com.cn/problem/CF311B)

`斜率优化` `前缀和` `分段 DP`

### 题意

把所有猫按能够被接走的“有效时刻”排序后，分成 `p` 段连续区间分别交给不同饲养员处理，要求总等待时间最小。

### 分析

原题里猫所在山丘不同、饲养员行走距离也不同，看起来信息很多，但真正会影响等待时间的，其实是“猫准备好时刻 - 饲养员赶到该山丘所需时间”。把这一步预处理完以后，每只猫就只剩一个一维的“有效时刻”。

接下来再把这些有效时刻排序，题目就被压成了纯序列问题：若一位饲养员负责若干只猫，它一定会接走排好序后的一段连续区间，而不会跳着接。因为如果区间中间断开，就等于把同一条时间线上的人硬拆成两段，不会更优。

于是设 `dp[t][i]` 表示前 `i` 只猫由 `t` 位饲养员负责时的最小等待和，那么最后一位饲养员一定接走一段连续区间 `j+1..i`。这段区间的代价用前缀和展开后，会变成“与 `i` 有关的固定项 + 关于断点 `j` 的一条直线在 `a[i]` 处的取值”。

所以这题最该学会的是两步压缩：先把山路信息吸收到“有效时刻”里，再把多人接猫改写成“有序序列分段”。完成这两步之后，它才会自然落成最标准的斜率优化分段 DP。

### 核心代码

```cpp
for (int t = 1; t <= p; t++) {
    int hh = 1, tt = 0;
    q[++tt] = 0;
    for (int i = 1; i <= m; i++) {
        while (hh < tt && slope(q[hh], q[hh + 1]) <= a[i]) hh++;
        int j = q[hh];
        dp[t][i] = dp[t - 1][j] + 1LL * a[i] * (i - j) - (pre[i] - pre[j]);
        while (hh < tt && bad(q[tt - 1], q[tt], i)) tt--;
        q[++tt] = i;
    }
}
```

### 复杂度

排序后每层 DP 都是线性凸包，整体复杂度 `O(pm)`，空间复杂度 `O(m)`。

## 5. [P4072 [SDOI2016] 征途](https://www.luogu.com.cn/problem/P4072)

`斜率优化` `前缀和` `方差转化`

### 题意

整条路线被分成若干段路，必须在恰好 `m` 天内走完，并且每天都只能走若干完整路段。希望每天行走长度尽量平均，从而让总方差最小。

### 分析

题面说最小化方差，但真正可操作的是把式子改写。方差看着像“整体平均值”问题，不方便直接做 DP；把它乘上 `m^2` 并展开后，和分段方式无关的常数项可以被剥掉，问题就等价于：把整条路切成 `m` 段，最小化每段长度与平均长度的偏差平方和。

这一步很关键，因为它把“统计学目标”翻译成了“序列分段代价”。于是设 `s[i]` 为前缀和，`dp[t][i]` 表示前 `i` 段路在 `t` 天内走完的最小代价。若最后一天走的是 `j+1..i`，转移就是 `dp[t-1][j] + (s[i]-s[j]-avg)^2`，这已经是非常标准的二次费用分段 DP。

朴素瓶颈自然就是枚举最后一天从哪里开始。平方项一展开后，关于断点 `j` 的部分会变成一条直线，而当前终点 `i` 只提供一个查询横坐标 `s[i] - avg`。也就是说，原题最后仍然在回答“若干候选断点里谁对当前终点最优”。

所以这题最值得学的是：遇到“最小化方差、均衡程度、离平均值偏差”这类目标，先别被整体统计量吓住，优先尝试把它改写成分段平方和，很多时候就会自然走到斜率优化。

### 核心代码

```cpp
for (int t = 1; t <= m; t++) {
    int hh = 1, tt = 0;
    q[++tt] = t - 1;
    for (int i = t; i <= n; i++) {
        while (hh < tt && slope(q[hh], q[hh + 1]) <= 2.0 * (s[i] - avg)) hh++;
        int j = q[hh];
        dp[t][i] = dp[t - 1][j] + sqr(s[i] - s[j] - avg);
        while (hh < tt && bad(q[tt - 1], q[tt], i, avg)) tt--;
        q[++tt] = i;
    }
}
```

### 复杂度

每一层用单调凸包线性转移，整体复杂度 `O(mn)`，空间复杂度 `O(n)`。

## 6. [P2120 [ZJOI2007] 仓库建设](https://www.luogu.com.cn/problem/P2120)

`斜率优化` `前缀和` `建仓决策`

### 题意

工厂沿山坡从高到低排成一列，产品只能往山下运输。可以在若干工厂建仓库，付出建仓费后，由这些仓库接收上方工厂的产品，要求总建仓费与运输费最小。

### 分析

把状态定成“最后一个仓库建在哪里”会很顺。因为货物只能往山下运，所以如果第 `i` 个工厂建仓，那么它负责接收的一定是上方一整段连续工厂 `j+1..i` 的产品；中间不可能跳着分配。

这一步其实已经把题目的结构看穿了：不是在很多仓库之间自由连边，而是在一条有方向的链上做连续分段。于是设 `dp[i]` 表示前 `i` 个工厂都处理完，并且最后一个仓库建在 `i` 的最小代价。

借助 `P[i]`、`Q[i]` 两组前缀和，可以把“上游产品统一运到 `i`”的总代价整段展开，最终整理成“与 `i` 有关的固定项 + 一批前驱直线在 `x[i]` 处的最小值”。也就是说，朴素转移里枚举上一个仓库位置 `j` 的那一步，被改写成了直线最优查询。

所以这题的迁移价值在于：**一条线上、货流只能单向前进、服务范围天然连续**，往往都能先写成分段 DP，再看费用能否整理成斜率优化。

### 核心代码

```cpp
int hh = 1, tt = 0;
q[++tt] = 0;
for (int i = 1; i <= n; i++) {
    while (hh < tt && slope(q[hh], q[hh + 1]) <= x[i]) hh++;
    int j = q[hh];
    dp[i] = dp[j] + c[i] + x[i] * (P[i] - P[j]) - (Q[i] - Q[j]);
    while (hh < tt && bad(q[tt - 1], q[tt], i)) tt--;
    q[++tt] = i;
}
```

### 复杂度

单次扫描即可完成，时间复杂度 `O(n)`，空间复杂度 `O(n)`。

## 7. [P3628 [APIO2010] 特别行动队](https://www.luogu.com.cn/problem/P3628)

`斜率优化` `二次函数` `前缀和`

### 题意

把士兵划成若干连续小队，每队收益是区间和代入二次函数后的结果，求最大总收益。

### 分析

这题也是标准的“最后一队从哪里开始”。若最后一支特别行动队是 `j+1..i`，它的战斗力只取决于这段区间和，所以一旦记下前缀和 `s[i]`，区间贡献就能写成 `a(s[i]-s[j])^2+b(s[i]-s[j])+c`。

于是设 `dp[i]` 表示前 `i` 名士兵划分完毕的最大收益，转移自然是从所有断点 `j` 里选一个最优的。朴素枚举每个 `j` 会是 `O(n^2)`，真正要做的是把这个二次式展开，看看前驱 `j` 对当前 `s[i]` 的影响能不能整理成更规则的形式。

展开后会发现：与当前终点 `i` 有关的部分可以单独提出，剩下的前驱部分正好是“若干条直线在横坐标 `s[i]` 处的最大值”。又因为前缀和 `s[i]` 单调递增，所以查询点顺序天然有序，这就把题目直接送进了单调上凸包模板。

### 核心代码

```cpp
int hh = 1, tt = 0;
q[++tt] = 0;
for (int i = 1; i <= n; i++) {
    while (hh < tt && slope(q[hh], q[hh + 1]) >= 2.0 * a * s[i]) hh++;
    int j = q[hh];
    dp[i] = dp[j] + a * sqr(s[i] - s[j]) + b * (s[i] - s[j]) + c;
    while (hh < tt && bad_max(q[tt - 1], q[tt], i)) tt--;
    q[++tt] = i;
}
```

### 复杂度

前缀和单调时凸包可线性维护，时间复杂度 `O(n)`，空间复杂度 `O(n)`。

## 8. [P5504 [JSOI2011] 柠檬](https://www.luogu.com.cn/problem/P5504)

`斜率优化` `按值分类` `分段 DP`

### 题意

把一串贝壳切成若干连续段。每段选一种大小 `s0` 作为“主值”，若这段里该大小出现了 `t` 次，就能贡献 `s0 * t^2`，要求总贡献最大。

### 分析

这题乍看像“每段选一种值”的复杂分段 DP，但真正关键的是：如果一段的主值是 `v`，那这段收益只取决于 `v` 在这段里出现了多少次，和段里其他值的分布没有直接关系。

这意味着题目不必按“段”来想，反而更适合按“值”来拆。固定某个值 `v` 后，只需要盯住它的出现位置序列；若当前位置是它第 `k` 次出现，那么把某一段的主值定成 `v`，收益里真正有用的信息其实只是“这一段包含了 `v` 的第几次到第几次出现”。

这样一来，原本杂乱的分段选择就被压缩成“同值出现次数之间怎么接”。平方项展开以后，关于前一个断点的部分会变成线性函数，于是每个值 `v` 都能独立维护一套自己的凸包。

扫描到一次新的出现位置时，先用该值对应的凸包查询“如果当前段主值取 `v`，最优能到多少”，再把新的候选状态作为一条直线插回去；最后别忘了和 `dp[i-1]` 合并，表示当前位置也可以什么都不新开、直接沿用之前最优答案。

### 核心代码

```cpp
for (int i = 1; i <= n; i++) {
    int v = a[i], k = ++cnt[v];
    auto &q = hull[v];
    while (q.size() > 1 && value(q[0], k) <= value(q[1], k)) q.pop_front();
    dp[i] = max(dp[i - 1], value(q.front(), k) + 1LL * v * k * k);
    Line cur = make_line(dp[i - 1], k, v);
    while (q.size() > 1 && bad(q[q.size() - 2], q.back(), cur)) q.pop_back();
    q.push_back(cur);
}
```

### 复杂度

每个值的凸包都只会进出一次，整体复杂度 `O(n)` 到 `O(n log V)`，空间复杂度 `O(n)`。

## 9. [P4027 [NOI2007] 货币兑换](https://www.luogu.com.cn/problem/P4027)

`CDQ 分治` `斜率优化` `凸包`

### 题意

每天都能按给定比例把资产在两种券之间转换，要求最大化最终资金。

### 分析

这题最不容易一下看出的地方，是状态其实不是“一维资金”，而是“持有两种券的资产组合”。如果把第 `i` 天结束后的最优状态看成平面上的一个点 `(X_i, Y_i)`，那么第 `i+1` 天能得到多少总价值，本质上就是拿一条权值直线 `A_i x + B_i y` 去问：之前所有可达点里，谁在这条直线方向上的投影最大。

这一步翻译非常重要，因为它把“每天换券”从金融叙事，改写成了“带时间顺序的二维点集最优线性查询”。一旦看成几何问题，为什么要建凸包就很自然了：只有凸包上的点才可能在某个方向上成为最优。

麻烦在于：这些点必须按时间先后转移，但对它们做凸包查询时，又需要按斜率顺序组织。如果直接在线做，两种顺序会打架；而 CDQ 分治正好擅长把这两件事拆开。

分治时，左半边代表更早的可用状态，右半边代表等待被更新的状态；在归并过程中，把左半边点集建成凸包，再批量回答右半边的查询，就同时满足了“只能从过去转移”和“凸包按序查询”这两个要求。

### 核心代码

```cpp
void cdq(int l, int r) {
    if (l == r) return;
    int mid = (l + r) >> 1;
    cdq(l, mid);
    build_hull(l, mid);
    for (int i = mid + 1; i <= r; i++) {
        int j = query_best(line[i]);
        dp[i] = max(dp[i], A[i] * X[j] + B[i] * Y[j]);
    }
    inplace_merge(id + l, id + mid + 1, id + r + 1, cmp_k);
    cdq(mid + 1, r);
}
```

### 复杂度

CDQ 每层做一次凸包归并，时间复杂度 `O(n log n)`，空间复杂度 `O(n)`。

## 10. [P6302 [NOI2019] 回家路线 加强版](https://www.luogu.com.cn/problem/P6302)

`斜率优化` `按站点建凸包` `最短路式 DP`

### 题意

每班列车都有出发站、到达站和对应时刻，可以在同站等待后换乘下一班车。等待时间会产生二次烦躁值，要求最小化到终点的总烦躁值。

### 分析

如果把“人停在车站里等下一班车”当状态会很乱，因为你不仅要记录在哪个站，还要记录等到了什么时刻。更顺手的做法是反过来：把“准备乘坐某一班列车”当成状态，这样上一段行程结束的位置和当前等待的起点都会自动确定。

一旦这样建模，转移就很清楚：上一班列车把你送到某站，接着你在这个站等待到下一班列车发车，中间这段等待时间会贡献一个二次烦躁值。把这个二次式展开后，会发现对于当前发车时刻来说，不同历史到达状态的贡献正是一批直线。

而且这些直线不是全局混在一起的，只会在“同一个出发站”内部竞争。也就是说，每个站点都在维护自己的一套“历史最优直线集合”，当前列车只去自己的出发站那套集合里查询最优前驱。

按发车时刻从小到大处理列车时，某站上可供转移的历史状态也在按时间单调增加，所以可以给每个站点各维护一个下凸包。查询当前列车时，只需在出发站的凸包上取最优；处理完这班列车后，再把它作为到达站的新候选状态插进去。

### 核心代码

```cpp
sort(train + 1, train + m + 1, cmp_depart);
for (int i = 1; i <= m; i++) {
    int x = train[i].from, y = train[i].to;
    while (hull[x].size() > 1 && hull[x][0].value(train[i].p) >= hull[x][1].value(train[i].p))
        hull[x].pop_front();
    dp[i] = hull[x].front().value(train[i].p);
    Line cur = make_line(train[i].q, dp[i]);
    while (hull[y].size() > 1 && bad(hull[y][hull[y].size() - 2], hull[y].back(), cur))
        hull[y].pop_back();
    hull[y].push_back(cur);
}
```

### 复杂度

若每个站点的查询顺序单调，整体可做到 `O(m)` 到 `O(m log m)`，空间复杂度 `O(m)`。

## 11. [P1721 [NOI2016] 国王饮水记](https://www.luogu.com.cn/problem/P1721)

`斜率优化` `平均值模型` `前缀和`

### 题意

有若干城市的蓄水高度互不相同，可以进行不超过 `k` 次连通操作，把若干城市的水重新平均。目标是让首都最终水位尽量高。

### 分析

“重新平均”这种操作最怕直接在原图上想，信息太多反而抓不到核心。真正该先问的是：如果只关心最后首都能被抬到多高，那么一次操作最可能把哪些城市放在一起？答案通常不会是零散若干点，而是按水位排好序后的一段前缀。

所以先把所有城市按高度排序，再考虑前缀合并。设 `dp[t][i]` 表示做了 `t` 次操作、处理到前 `i` 个高度层时的最优结果，那么最后一次操作如果从某个断点 `j` 开始，把 `j+1..i` 这一段拿来平均，收益就能用前缀和整体写出。

朴素做法会卡在“对每个 `i` 都枚举全部断点 `j`”。继续把平均值公式展开后，会发现前驱 `j` 对当前 `i` 的影响可以整理成线性比较式，也就是说又回到了“当前状态给出横坐标、候选断点对应直线”的斜率优化模型。

这题最值得迁移的点是：遇到“合并 / 平均 / 拉平一段元素”的题，先尝试排序并把操作改写成前缀决策，很多表面像图论或贪心的问题，最后都会落成序列 DP。

### 核心代码

```cpp
for (int t = 1; t <= k; t++) {
    int hh = 1, tt = 0;
    q[++tt] = 0;
    for (int i = 1; i <= n; i++) {
        while (hh < tt && slope(q[hh], q[hh + 1]) <= h[i]) hh++;
        int j = q[hh];
        dp[t][i] = trans(dp[t - 1][j], j, i);
        while (hh < tt && bad(q[tt - 1], q[tt], i)) tt--;
        q[++tt] = i;
    }
}
```

### 复杂度

按高度顺序转移时，每层都是单调凸包，常见复杂度为 `O(kn)`，空间复杂度 `O(n)`。

## 12. [P2305 [NOI2014] 购票](https://www.luogu.com.cn/problem/P2305)

`树形 DP` `李超线段树` `斜率优化`

### 题意

每个城市可以买一张票跳到某个满足距离限制的祖先，要求各城市到根的最小总票价。

### 分析

这题先不要被“树”和“买票”吓住，核心仍然是：对每个城市 `u`，从一批合法前驱里挑一个最便宜的。不同的是，这批前驱不再是序列里的前缀，而是 `u` 到根路径上的一段祖先。

如果把某个祖先 `v` 作为转移来源，那么票价可以整理成

`dp[v] - p[u] * dist[v] + p[u] * dist[u] + q[u]`。

对当前城市 `u` 来说，`p[u] * dist[u] + q[u]` 是固定项，真正要比较的是不同祖先 `v` 带来的 `dp[v] - p[u] * dist[v]`。这已经非常像“拿横坐标 `p[u]` 去查询若干条直线的最小值”了：每个祖先 `v` 都对应一条直线。

难点在于合法前驱还带着路径限制：只有那些与 `u` 的距离差不超过 `l[u]` 的祖先才能用。而当我们 DFS 一棵树时，当前递归栈恰好就是从根到 `u` 的祖先链。于是可以一边 DFS，一边把祖先对应的直线插入李超线段树；离开子树时再回滚，这样线段树里永远只保留“当前路径上的可候选祖先”。查询时再结合距离限制裁掉不合法部分，就能得到当前城市的最优转移。

### 核心代码

```cpp
void dfs(int u) {
    long long best = query(1, 0, maxd, dist[u]);
    dp[u] = best + 1LL * p[u] * dist[u] + q[u];
    auto bak = snapshot();
    insert(1, 0, maxd, Line(-p[u], dp[u]));
    for (auto [v, w] : g[u]) {
        dist[v] = dist[u] + w;
        dfs(v);
    }
    rollback(bak);
}
```

### 复杂度

每个节点做一次插入和查询，复杂度通常为 `O(n log D)`，空间复杂度 `O(n log D)`。

## 13. [P2497 [SDOI2012] 基站建设](https://www.luogu.com.cn/problem/P2497)

`CDQ 分治` `斜率优化` `区间限制`

### 题意

在线上有若干基站，开启某站并设置接收半径需要付费。要求通过一系列合法接力把信号传到终点，并使总代价最小。

### 分析

按坐标排序后，每个基站能从哪些前驱接收到信号，不是任意的，而会形成一个连续合法区间 `[L[i], R[i]]`。这说明状态转移并不是“全局挑最优前驱”，而是“在限定区间里挑最优前驱”。

把费用公式展开后，前驱状态对当前点的影响仍然可以视作直线查询，只不过这次查询不能在全部历史状态里做，而只能在特定下标范围内做。这也是本题比普通斜率优化更难的地方：不仅要维护凸包，还要处理“只有某一段前驱能用”。

直接在线维护这种“区间受限的最优直线”会很麻烦，所以常见思路是换个角度：不用一次次在线回答，而是用 CDQ 分治把时间顺序拆成很多批。左半边统一提供候选直线，右半边统一接收查询，再结合各自的 `[L[i], R[i]]` 把不合法来源过滤掉。

这样一来，原本缠在一起的“顺序关系、区间限制、凸包最优”三件事就被拆开处理了。这题最值得记住的是：**当转移仍像斜率优化，但可用前驱只落在一个区间里时，要想到 CDQ 分治分批做贡献。**

### 核心代码

```cpp
void cdq(int l, int r) {
    if (l == r) return;
    int mid = (l + r) >> 1;
    cdq(l, mid);
    build_hull(l, mid);
    for (int i = mid + 1; i <= r; i++)
        if (L[i] <= mid)
            dp[i] = min(dp[i], query(x[i], max(L[i], l), min(R[i], mid)) + add[i]);
    cdq(mid + 1, r);
}
```

### 复杂度

分治每层处理一次候选直线，时间复杂度约为 `O(n log n)`，空间复杂度 `O(n)`。

## 14. [P2900 [USACO08MAR] Land Acquisition G](https://www.luogu.com.cn/problem/P2900)

`斜率优化` `去劣矩形` `凸包`

### 题意

要购买若干块矩形土地，可以把相邻若干块合并成一组购买；一组的费用等于组内最大长乘最大宽。要求把所有土地分组后总费用最小。

### 分析

这题最关键的不是一上来写 DP，而是先把“永远不会成为答案关键边界”的矩形删掉。若某块地的长和宽都不优于另一块，那么无论你怎么分组，它都不会成为组内最大长或最大宽的决定者，可以直接视为无效状态。

把矩形按长排序、宽逆序后删去这些被支配的矩形，剩下的会自动形成“长度递增、宽度递减”的结构。这个结构非常重要，因为它把原题从乱糟糟的二维分组，压成了一个有序序列上的分段问题。

这时设 `dp[i]` 表示前 `i` 块有效矩形处理完的最小费用。若最后一组从 `j+1` 开始，那么这一组的最大长一定是当前的 `w[i]`，最大宽一定是 `h[j+1]`，于是转移变成 `dp[j] + w[i] * h[j+1]`。

也就是说，难点已经从“怎么分组”转成了“对每个 `i` 枚举哪个 `j` 最好”。而这个式子正好能看成若干条直线在单调查询点上的最小值，所以前面的删劣预处理，本质就是在把题目修整成能直接套单调凸包的形状。

### 核心代码

```cpp
int hh = 1, tt = 0;
q[++tt] = 0;
for (int i = 1; i <= m; i++) {
    while (hh < tt && slope(q[hh], q[hh + 1]) <= w[i]) hh++;
    int j = q[hh];
    dp[i] = dp[j] + 1LL * w[i] * h[j + 1];
    while (hh < tt && bad(q[tt - 1], q[tt], i)) tt--;
    q[++tt] = i;
}
```

### 复杂度

去劣后线性扫一遍即可，时间复杂度 `O(n log n)`（排序）加 `O(n)`（DP），空间复杂度 `O(n)`。

## 15. [P5017 [NOIP 2018 普及组] 摆渡车](https://www.luogu.com.cn/problem/P5017)

`斜率优化` `前缀和` `分批调度`

### 题意

所有同学在不同时间到达起点等车，摆渡车每次往返都需要固定时间。要求自行安排每一趟的出发时刻，使所有同学等待时间总和最小。

### 分析

这题先不要去想“每一趟车该几点开”，而要先想最后一趟车负责了哪些人。因为同学按到达时刻排序后，最后一趟车一定带走一段连续区间 `j+1..i`，所以原题本质上仍然是“枚举最后一段”的分段 DP。

设 `dp[i]` 表示前 `i` 个同学都已经被送走的最小等待和。若最后一趟负责 `j+1..i`，那么这一整段人的等待时间都由同一个发车时刻决定，因此代价可以整体打包，而不是一个个去加。

把这段代价用前缀和展开后，关于终点 `i` 的部分和关于断点 `j` 的部分可以拆开，最后仍会变成“当前 `i` 提供查询横坐标，前驱 `j` 对应一条直线”的形式。由于到达时刻已经排好序，查询顺序天然单调，因此可以直接用下凸包维护最优断点。

所以这题可迁移的结论是：**把一批人/任务按顺序分成若干批，且每批共享某个统一时刻或统一代价**，就应先从“最后一批是一段连续区间”这个角度下手。

### 核心代码

```cpp
int hh = 1, tt = 0;
q[++tt] = 0;
for (int i = 1; i <= n; i++) {
    while (hh < tt && slope(q[hh], q[hh + 1]) <= t[i] + m) hh++;
    int j = q[hh];
    dp[i] = dp[j] + 1LL * (i - j) * (t[i] + m) - (pre[i] - pre[j]);
    while (hh < tt && bad(q[tt - 1], q[tt], i)) tt--;
    q[++tt] = i;
}
```

### 复杂度

排序后做一次凸包 DP，时间复杂度 `O(n log n)`，空间复杂度 `O(n)`。

## 16. [P3195 [HNOI2008] 玩具装箱](https://www.luogu.com.cn/problem/P3195)

`斜率优化` `二次 DP` `前缀和`

### 题意

玩具必须按顺序装进若干容器。每个容器装下一段连续玩具后，若总长度偏离给定值 `L`，就会产生平方代价，要求总费用最小。

### 分析

这题最容易漏掉的细节，是玩具之间还有固定空隙。如果只记玩具本身长度，转移式会一直很难看；更顺的做法是把这些固定空隙也一起并进前缀和，这样一段 `j+1..i` 放进同一个容器时，总长度就能直接表示成两个前缀和之差。

设 `dp[i]` 表示前 `i` 件玩具装完的最小代价，那么最后一个容器若装的是 `j+1..i`，代价就是 `(x[i] - x[j] - L)^2`。这说明原题的瓶颈不是状态设计，而是每次都要枚举最后一个容器从哪里开始，朴素会是 `O(n^2)`。

平方项展开后，前驱 `j` 对当前 `x[i]` 的影响可以视作一条直线，而 `x[i]` 本身单调递增，所以查询点和候选直线顺序都满足单调凸包要求。这就是为什么这题能从“平方代价分段 DP”顺滑地落到斜率优化。

以后再遇到**分段 DP + 平方代价**的组合，第一反应就该是：先展开平方项，看看能不能把前驱整理成直线。

### 核心代码

```cpp
int hh = 1, tt = 0;
q[++tt] = 0;
for (int i = 1; i <= n; i++) {
    while (hh < tt && slope(q[hh], q[hh + 1]) <= 2.0 * x[i]) hh++;
    int j = q[hh];
    dp[i] = dp[j] + sqr(x[i] - x[j] - L);
    while (hh < tt && bad(q[tt - 1], q[tt], i)) tt--;
    q[++tt] = i;
}
```

### 复杂度

单调凸包优化后是 `O(n)`，空间复杂度 `O(n)`。

## 17. [P2365 [IOI 2002] 任务安排](https://www.luogu.com.cn/problem/P2365)

`斜率优化` `任务分批` `前缀和`

### 题意

有一串必须按顺序处理的任务，可以把它们分成若干连续批次。每批开始前都要额外花费固定启动时间 `s`，每个任务的费用等于完成时刻乘上自己的系数，要求总费用最小。

### 分析

这题的本质仍然是“最后一批从哪里开始”。若最后一批是 `j+1..i`，那么这一整批里的所有任务会在同一时刻完成，因此它们的费用并不是一个个单独决定，而是可以整体打包计算。

把时间前缀和 `T[i]`、费用系数前缀和 `F[i]` 记出来后，转移就能写成 `dp[j] + (T[i]-T[j]+s) * (F[n]-F[j])`。这一步最值得注意的是：当前终点 `i` 和前驱断点 `j` 的信息，已经被成功拆成了两部分。

继续展开后，当前终点 `i` 只负责提供一个横坐标 `T[i]`，而每个前驱 `j` 都对应一条直线。于是问题就从“分批最优决策”变成了“若干候选直线里谁在当前横坐标最优”。

又因为任务顺序固定，`T[i]` 是单调递增的，所以查询顺序天然满足单调凸包条件。这题真正该形成条件反射的是：**顺序任务分批、且一整批共享同一个完成时刻**，很容易就会推到斜率优化。

### 核心代码

```cpp
int hh = 1, tt = 0;
q[++tt] = 0;
for (int i = 1; i <= n; i++) {
    while (hh < tt && slope(q[hh], q[hh + 1]) <= T[i]) hh++;
    int j = q[hh];
    dp[i] = dp[j] + (T[i] - T[j] + s) * (F[n] - F[j]);
    while (hh < tt && bad(q[tt - 1], q[tt], i)) tt--;
    q[++tt] = i;
}
```

### 复杂度

利用单调性后时间复杂度 `O(n)`，空间复杂度 `O(n)`。

## 18. [P5785 [SDOI2012] 任务安排](https://www.luogu.com.cn/problem/P5785)

`李超线段树` `斜率优化` `在线查询`

### 题意

仍是任务分批模型，但数据范围更强，不能再依赖完全单调的凸包顺序。

### 分析

这一题和上一题的本质完全同源：仍然是“最后一批从哪里开始”，仍然能把每个前驱状态看成一条直线，然后在当前横坐标上查询最小值。

但这里真正值得学的是：什么时候**不能**再用普通斜率优化。上一题之所以能用单调队列，是因为候选直线的加入顺序、查询点的访问顺序都满足单调性；而这题数据范围更强、转移顺序更乱，这两个单调条件不再同时成立。

一旦发现“式子已经是直线查询了，但查询/插入次序不单调”，就该从静态凸包切换到动态结构。李超线段树正是干这个的：把横坐标范围建成线段树，每次插入一条直线，再在某个横坐标上查询最优值，都能在线完成。

所以这题真正的举一反三点不是公式本身，而是识别信号：**斜率优化的式子已经出现，但普通队列维护凸包所需的单调性消失了**，那就该考虑李超线段树或平衡树。

### 核心代码

```cpp
insert(1, xmin, xmax, Line(0, 0));
for (int i = 1; i <= n; i++) {
    long long best = query(1, xmin, xmax, T[i]);
    dp[i] = best + T[i] * F[i] + s * (F[n] - F[i]);
    Line cur(-F[i], dp[i] + T[i] * F[i] - s * F[i]);
    insert(1, xmin, xmax, cur);
}
```

### 复杂度

每次操作 `O(log X)`，总复杂度 `O(n log X)`，空间复杂度 `O(n log X)`。

# 四、单调队列优化状态转移

如果合法决策只会落在一个滑动窗口里，单调队列往往就是把枚举压到线性的关键。

## 19. [P2569 [SCOI2010] 股票交易](https://www.luogu.com.cn/problem/P2569)

`单调队列` `状态压缩` `多阶段转移`

### 题意

已知未来 `T` 天的股票买入价、卖出价、单日最多买入股数、单日最多卖出股数，以及交易冷却期 `W`。问在这些限制下最多能赚多少钱。

### 分析

这题最自然的状态不是“今天赚了多少”，而是“今天结束时手里还剩多少股”。所以设 `dp[i][j]` 表示第 `i` 天结束后持有 `j` 股时的最大收益。

如果今天什么都不做，那就直接从前一天继承；如果今天买入若干股，那么前一次允许影响今天的状态必须来自第 `i-W-1` 天，因为中间还隔着冷却期。同理，今天卖出若干股时，也只能从那个更早的状态转移过来。

关键在于：当我们固定今天的持股数 `j` 时，买入转移和卖出转移都不是从任意股数来，而是各自落在一段连续区间里。比如买入时，前驱股数必须在 `[j-AS[i], j]` 之类的窗口中；把式子展开后，会变成“当前 `j` 的固定项 + 窗口里某个表达式的最大值”。

这就把一个看似二维的大 DP，拆成了每天两次一维滑窗最值：一次处理买入，一次处理卖出。单调队列的作用，不是优化时间轴，而是优化“对每个持股数去枚举所有可能前驱股数”这层枚举。

### 核心代码

```cpp
for (int i = 1; i <= T; i++) {
    copy(dp[i - 1], dp[i - 1] + MaxP + 1, dp[i]);
    deque<int> q;
    for (int j = 0; j <= MaxP; j++) {
        while (!q.empty() && q.front() < j - AS[i]) q.pop_front();
        while (!q.empty() && base_buy(i, q.back()) <= base_buy(i, j)) q.pop_back();
        q.push_back(j);
        dp[i][j] = max(dp[i][j], base_buy(i, q.front()) - 1LL * j * AP[i]);
    }
}
```

### 复杂度

每一天对股数做常数次线性扫描，总复杂度 `O(T * MaxP)`，空间复杂度 `O(T * MaxP)` 或滚动到 `O(MaxP)`。

## 20. [P2564 [SCOI2009] 生日礼物](https://www.luogu.com.cn/problem/P2564)

`双指针` `覆盖计数` `最短区间`

### 题意

彩带上挂着若干彩珠，每颗彩珠有坐标和颜色，一共 `K` 种颜色。现在要截下一段连续区间当礼物，要求这段区间至少包含每一种颜色各一颗，并且总长度尽量短。

### 分析

这题最重要的第一步，是把“彩带上的珠子”抽象成一条按坐标排序的序列。排完序后，问题就不再是礼物故事，而是：在一条线性序列里，找一个最短区间，使它覆盖全部 `K` 种颜色。

一旦看到“覆盖全部种类 + 区间尽量短”，就该想到双指针。因为区间一旦已经包含全部颜色，再继续往右扩张只会让答案更差；真正该做的是把左端点尽量右移，看看能不能在不破坏覆盖性的前提下缩短区间。

所以整个过程就是两个动作反复交替：右端点负责“把缺的颜色补齐”，左端点负责“把多余的珠子删掉”。计数数组记录每种颜色在窗口中的出现次数，`kind` 记录当前已经覆盖了几种颜色。这样每次窗口合法时，都能立刻尝试更新答案。

这类题最值得记住的不是代码，而是识别信号：**目标是最短 / 最长区间，约束是覆盖某些类别或满足某个计数条件**，那大概率就是“扩右补条件，缩左卡极值”的双指针模型。

### 核心代码

```cpp
int kind = 0, ans = INF;
for (int l = 1, r = 0; l <= n; l++) {
    while (r < n && kind < K) {
        ++r;
        if (++cnt[col[r]] == 1) kind++;
    }
    if (kind == K) ans = min(ans, x[r] - x[l]);
    if (--cnt[col[l]] == 0) kind--;
}
```

### 复杂度

双指针线性推进，时间复杂度 `O(n)`，空间复杂度 `O(K)`。

## 21. [P2219 [HAOI2007] 修筑绿化带](https://www.luogu.com.cn/problem/P2219)

`二维单调队列` `子矩形最值` `前缀和`

### 题意

公园里要先选一个 `A×B` 的外框，再在里面放一个 `C×D` 的花坛。绿化带的肥沃度等于外框总和减去内层花坛总和，要求这个差值最大。

### 分析

先用二维前缀和把任意子矩形的权值和压到 `O(1)`。这样外框一旦固定，绿化带的值就等于“外框和 - 外框内部某个合法 `C×D` 子矩形的和”，所以问题核心立刻变成：对每个外框，快速找出其中所有合法内框里的最小矩形和。

注意这里真正难的不是算某一个内框，而是“对每个外框都重新枚举所有内框位置”会爆炸。好在合法内框的左上角范围本身也是一个矩形窗口，所以完全可以先把所有 `C×D` 小矩形的和预处理成一张表，再在这张表上做二维滑窗最小值。

二维滑窗的思路和一维单调队列是同一个模板的两次叠加：先在每一行上维护横向窗口最小值，把宽度限制处理掉；再在每一列上维护纵向窗口最小值，把高度限制处理掉。这样得到的就是每个外框对应区域里的最小内框和。

所以这题最值得迁移的是：**外层枚举大矩形，内层要在一个可滑动范围里找最优小矩形**，通常先把小矩形值预处理出来，再做二维窗口最值，而不是双层套暴力。

### 核心代码

```cpp
for (int i = 1; i <= n; i++) {
    deque<int> q;
    for (int j = 1; j <= m; j++) {
        while (!q.empty() && q.front() < j - (B - D)) q.pop_front();
        while (!q.empty() && inner[i][q.back()] >= inner[i][j]) q.pop_back();
        q.push_back(j);
        rowMin[i][j] = inner[i][q.front()];
    }
}
for (int j = 1; j <= m; j++)
    solve_col(j);
```

### 复杂度

两次二维滑窗后总复杂度 `O(nm)`，空间复杂度 `O(nm)`。

## 22. [P3957 [NOIP 2017 普及组] 跳房子](https://www.luogu.com.cn/problem/P3957)

`二分答案` `单调队列` `可行性检查`

### 题意

跳房子的基础步长是 `d`，如果花费 `g` 个金币，就能把每一步的实际跳跃距离放宽到 `[d-g, d+g]`。要求总得分至少达到 `k`，问最少需要多少金币。

### 分析

“最少金币”这种问法通常先看单调性：如果花 `g` 个金币能做到，那么花更多金币只会让允许的跳跃区间更宽，不会更差，所以答案可以二分。

固定某个金币数 `g` 后，问题就变成一次判定。这里最重要的不是“怎么跳得远”，而是把合法前驱描述清楚：设 `dp[i]` 表示落到第 `i` 个格子时最多能拿到多少分，那么它只能从那些与 `x[i]` 的距离落在 `[d-g, d+g]` 的前驱转移过来。

一旦坐标排好序，这个合法前驱集合就不再是乱的，而是在数轴上形成一个整体右移的窗口。于是 `check(g)` 的瓶颈，实质上就是：对每个 `i`，在当前窗口里找一个 `dp` 最大的前驱。

所以这题和很多“二分答案 + 判定 DP”题一样，真正的优化点不在二分本身，而在于把判定式里的合法来源集合看成滑动窗口。不断把新进入下界的点加进队列，再把超出上界的点从队头踢掉，队头就始终是当前最优前驱。

### 核心代码

```cpp
bool check(int g) {
    int L = max(1, d - g), R = d + g, p = 1;
    deque<int> q;
    fill(dp, dp + n + 1, -INF);
    for (int i = 1; i <= n; i++) {
        while (p < i && x[p] <= x[i] - L) {
            while (!q.empty() && dp[q.back()] <= dp[p]) q.pop_back();
            q.push_back(p++);
        }
        while (!q.empty() && x[q.front()] < x[i] - R) q.pop_front();
        if (!q.empty()) dp[i] = dp[q.front()] + score[i];
        if (dp[i] >= k) return true;
    }
    return false;
}
```

### 复杂度

二分外层是 `O(log V)`，每次检查线性，整体复杂度 `O(n log V)`，空间复杂度 `O(n)`。

## 23. [P3229 [HNOI2013] 旅行](https://www.luogu.com.cn/problem/P3229)

`单调队列` `分段 DP` `前缀和`

### 题意

给定一条已经排好的旅行路线，需要把整段行程切成恰好 `m` 个月。每个月走一段连续区间，要求总快乐值最大。

### 分析

这种“把序列切成若干连续段”的题，先想最后一段从哪开始。设 `dp[t][i]` 表示前 `i` 个位置分成 `t` 个月时的最大收益，那么最后一个月一定是某个断点 `j` 之后的一段连续区间。

关键不只是写出转移，而是看清“断点价值”怎么比较。把收益写成前缀和后，`j -> i` 的贡献可以拆成两部分：一部分只和终点 `i` 有关，另一部分只和断点 `j` 有关。也就是说，当我们固定月份 `t`、从左到右扫描终点 `i` 时，本质上一直在一批候选断点里选“断点价值”最大的那个。

如果候选断点是乱的，那还是得暴力；但本题的合法断点范围会随 `i` 单调移动，而且比较两个断点谁更优时，也能整理成单调淘汰关系。这就意味着我们不需要保留所有历史断点，只要用队列维护当前仍有竞争力的候选者。

所以这题最值得迁移的是：有些分段 DP 不一定会化成凸包直线，但只要“候选集合是滑动窗口、优劣关系可单调维护”，依然能用单调队列把一层转移压成线性。

### 核心代码

```cpp
for (int t = 1; t <= m; t++) {
    deque<int> q;
    for (int i = t; i <= n; i++) {
        int j = i - 1;
        while (!q.empty() && key(t - 1, q.back()) <= key(t - 1, j)) q.pop_back();
        q.push_back(j);
        while (!q.empty() && q.front() < t - 1) q.pop_front();
        dp[t][i] = val(i) + key(t - 1, q.front());
    }
}
```

### 复杂度

每层状态都只做一次入队和出队，时间复杂度 `O(mn)`，空间复杂度 `O(n)`。

## 24. [P2254 [NOI2005] 瑰丽华尔兹](https://www.luogu.com.cn/problem/P2254)

`单调队列` `网格 DP` `分方向处理`

### 题意

钢琴位于带障碍的网格里，每个时段船体倾斜方向固定；若不施法，钢琴会沿该方向滑动一格，施法则原地不动。要求在全过程中累计滑行距离最大。

### 分析

难点不在状态本身，而在“很多步朝同一方向移动”怎么快速转移。因为题目把操作按时间段给出，而每个时间段里的方向固定，所以可以把整道题拆成若干次一维 DP：如果当前段水平移动，就逐行处理；如果竖直移动，就逐列处理。

这样想的关键是把二维网格先忘掉，只盯住“同一行或同一列上的一条线”。在这一条线上，当前位置 `pos` 只能从前面不超过 `len` 步的位置转移过来，而且收益里还会多出一个和坐标差成正比的滑行距离。

一看到“窗口范围限制 + 线性偏移量”，就该想到把这个偏移量提前扣掉。扣完以后，原式就退化成在最近 `len` 个位置里取最大值，恰好是单调队列能维护的对象。

障碍物则会把一整条线切成若干互不影响的小段；每遇到障碍，就把队列清空重新开始。于是每个时间段都能在线性扫描整张网格的过程中完成。这题最值得迁移的是：**固定方向移动的网格 DP，往往能拆成很多条一维线性 DP。**

### 核心代码

```cpp
void work_line(vector<int> cells) {
    deque<int> q;
    for (int i = 0; i < (int)cells.size(); i++) {
        if (block[cells[i]]) { q.clear(); continue; }
        while (!q.empty() && q.front() < i - len) q.pop_front();
        while (!q.empty() && f[cells[q.back()]] - q.back() <= g[cells[i]] - i) q.pop_back();
        q.push_back(i);
        f[cells[i]] = max(f[cells[i]], g[cells[q.front()]] + i - q.front());
    }
}
```

### 复杂度

每个时间段只会遍历全部格子一次，整体复杂度 `O(KNM)` 中的常数被队列优化压到线性，空间复杂度 `O(NM)`。

## 25. [CF372C Watching Fireworks is Fun](https://www.luogu.com.cn/problem/CF372C)

`单调队列` `分层 DP` `滑动窗口最大值`

### 题意

烟花按时间顺序依次燃放，你站在一条长度为 `n` 的街道上，相邻两场之间最多移动一定距离。每场烟花在不同位置观看会得到不同幸福值，要求总幸福值最大。

### 分析

这题的状态很直接：`dp[i][x]` 表示看完第 `i` 场烟花后站在位置 `x` 的最大幸福值。真正的瓶颈不在状态数，而在于转移时，上一层哪些位置能够在规定时间内走到当前的 `x`。

如果第 `i-1` 场和第 `i` 场之间相隔 `Δt`，那么最大移动距离就是 `d * Δt`。也就是说，计算 `dp[i][x]` 时，只需要在上一层的区间 `[x-lim, x+lim]` 里找一个最大值，再加上当前这场烟花带来的贡献 `b_i-|a_i-x|`。

于是整层转移的瓶颈就完全暴露了：对每个位置 `x`，都去邻域里扫一遍最大值会变成 `O(n^2)`。但当 `x` 从左往右扫时，这个可达区间其实只是一个不断平移的窗口，所以可以用单调队列维护上一层数组在当前窗口内的最大值。

所以这题可迁移的模式非常清晰：**分层 DP + 相邻两层之间的可达范围是一个长度受限的区间**，就要优先想到滑动窗口最值，而不是继续做二维暴力转移。

### 核心代码

```cpp
for (int i = 1; i <= m; i++) {
    int lim = d * (t[i] - t[i - 1]);
    deque<int> q;
    for (int x = 1; x <= n; x++) {
        while (!q.empty() && q.front() < x - lim) q.pop_front();
        while (!q.empty() && pre[q.back()] <= pre[x]) q.pop_back();
        q.push_back(x);
        cur[x] = pre[q.front()] + b[i] - abs(a[i] - x);
    }
    swap(pre, cur);
}
```

### 复杂度

每场烟花做一次线性滑窗，时间复杂度 `O(mn)`，空间复杂度 `O(n)`。

# 五、环形前缀与判定型队列

当题目转成前缀和、环形展开或二分判定时，队列维护的通常是窗口内的最值或可行性边界。

## 26. [U230380 [POI2004] 旅行问题](https://www.luogu.com.cn/problem/U230380)

`环形前缀和` `单调队列` `最小前缀`

### 题意

环形公路上每个站点都有可获得的油量，相邻站点之间有固定路程消耗。对每个站点判断：如果从这里出发，能否顺时针或逆时针完整跑回起点。

### 分析

判断一个起点能不能跑完整圈，真正该看的不是“总油量够不够”，而是“中途会不会先没油”。所以这题和很多环形判定题一样，核心都在检查：从某个起点开始的一整段前缀和会不会中途跌到负数以下。

把环复制成链后，若从起点 `s` 顺时针出发，那么区间 `[s, s+n-1]` 上每一步累计净油量都必须不小于 `0`。换成前缀和，就是这段区间里的最小前缀和必须至少为 `pre[s-1]`。

这样原题就被翻译成了固定长度窗口最小值判定：每个起点都对应后面长度为 `n` 的一个窗口，而窗口最小前缀和正好可以用递增队列在线维护。逆时针方向只是把同样的判定在反方向序列上再做一遍。

所以这题真正该记住的是一条通用套路：**环上能否从某点出发完整走一圈**，往往都要转成“复制成链 + 固定长度窗口里的最小前缀和是否合法”。

### 核心代码

```cpp
for (int i = 1; i <= 2 * n; i++) pre[i] = pre[i - 1] + gas[i] - dist[i];
deque<int> q;
for (int i = 1; i <= 2 * n; i++) {
    while (!q.empty() && q.front() < i - n) q.pop_front();
    while (!q.empty() && pre[q.back()] >= pre[i]) q.pop_back();
    q.push_back(i);
    if (i >= n) ok[i - n + 1] |= pre[q.front()] - pre[i - n] >= 0;
}
```

### 复杂度

顺逆两个方向各一次线性扫描，总复杂度 `O(n)`，空间复杂度 `O(n)`。

## 27. [U126668 【一本通 5.5 例 4】 旅行问题](https://www.luogu.com.cn/problem/U126668)

`环形展开` `单调队列` `可行起点`

### 题意

这是上一题的同模型版本：环形公路上每站提供一定油量，行驶到下一站会消耗油，要求求出所有能够顺利绕行一圈的起点。

### 分析

虽然题面更口语化，但判断逻辑和上一题完全一样：从一个站点出发能否顺利回到原点，关键不在于最后总油量，而在于途中任意时刻油量余额都不能掉成负数。

一旦把每一段写成“得到油量 - 走到下一站的消耗”，问题就立刻变成前缀和判定。把环复制成链后，起点 `s` 是否可行等价于：从 `s` 开始长度为 `n` 的这段里，最小前缀和是否始终不低于起点前一个位置的前缀和。

这说明真正该枚举的不是起点本身，而是一个个固定长度为 `n` 的窗口。只要知道这个窗口里的最小前缀和，就能立刻判断该起点是否合法；而窗口右端点单调移动，正适合用递增队列维护最小值。

所以这题真正训练的是一种常见转换：遇到“环上每个起点都要判是否合法”，不要逐个起点暴力模拟，而是把环摊平成链，再问“每个固定长度窗口里的最小前缀和是什么”。

### 核心代码

```cpp
for (int i = 1; i <= 2 * n; i++) sum[i] = sum[i - 1] + a[i] - b[i];
deque<int> q;
for (int i = 1; i <= 2 * n; i++) {
    while (!q.empty() && q.front() < i - n) q.pop_front();
    while (!q.empty() && sum[q.back()] >= sum[i]) q.pop_back();
    q.push_back(i);
    if (i >= n) ans[i - n + 1] = (sum[q.front()] >= sum[i - n]);
}
```

### 复杂度

时间复杂度 `O(n)`，空间复杂度 `O(n)`。

## 28. [P1725 琪露诺](https://www.luogu.com.cn/problem/P1725)

`单调队列` `线性 DP` `窗口最大值`

### 题意

从 `0` 号格子出发，每次只能向右跳，且步长必须落在 `[L, R]`。每个格子有一个冰冻指数，问在到达终点前最多能累计多少收益。

### 分析

一旦把“最后一步跳到哪里”想清楚，这题的状态就很自然。设 `dp[i]` 表示落到格子 `i` 时的最大收益，那么它的前驱只能是那些与 `i` 的距离在 `[L, R]` 之间的位置，也就是区间 `[i-R, i-L]`。

这时朴素瓶颈也非常明确：对每个 `i`，都去枚举这一整段合法步长，复杂度就会被窗口宽度拖住。而当终点 `i` 从左往右增加时，这个合法前驱区间并不会乱跳，它只是整体右移。

于是问题直接退化成“右移窗口里的最大值”。队列里维护当前窗口里 `dp` 最大的候选位置，队头就是最优前驱；新状态进窗口时按大小淘汰劣势状态，旧状态出窗口时从队头弹走。

因此整道题虽然本质是 DP，但优化点完全落在“窗口里取最大值”上：不再为每个 `i` 暴力枚举所有步长，而是让队列替我们记住这一段里最好的状态。

### 核心代码

```cpp
deque<int> q;
dp[0] = 0;
for (int i = 1; i <= n; i++) {
    int p = i - L;
    if (p >= 0) {
        while (!q.empty() && dp[q.back()] <= dp[p]) q.pop_back();
        q.push_back(p);
    }
    while (!q.empty() && q.front() < i - R) q.pop_front();
    if (!q.empty()) dp[i] = dp[q.front()] + a[i];
}
```

### 复杂度

每个位置只进出队一次，时间复杂度 `O(n)`，空间复杂度 `O(n)`。

## 29. [U301134 绿色通道](https://www.luogu.com.cn/problem/U301134)

`二分答案` `单调队列` `最小代价 DP`

### 题意

总共有 `n` 道题，每题抄写耗时已知，总耗时不能超过 `t`。允许留空一些题，但想让最长连续空题段尽量短，问这个最优值是多少。

### 分析

“最长空题段最小”这种问法，一眼就该先看单调性：如果允许的最长空段长度是 `x` 时能做到，那么放宽成更大的值显然也能做到，所以答案可以二分。

接下来关键是把判定写成好做的形式。若当前假设最长空题段不能超过 `x`，那就等价于：任意连续 `x+1` 道题里，至少要抄一道。这样原题立刻从“情绪值最小化”变成了一个带窗口约束的最小代价问题。

于是设 `dp[i]` 表示“第 `i` 题必须抄，并且前 `i` 题都已经合法”时的最小耗时。那么它只能从前面最近 `x+1` 个位置中的某个已抄题转移过来，因为再远就会产生长度大于 `x` 的空段。

所以 `check(x)` 的瓶颈，正是“在最近 `x+1` 个前驱里找最小 `dp`”，这就是单调队列维护滑动窗口最小值的典型场景。如果最后一段后缀里存在某个可作结束点的位置总代价不超过 `t`，就说明这个 `x` 可行。

### 核心代码

```cpp
bool check(int x) {
    deque<int> q;
    dp[0] = 0; q.push_back(0);
    for (int i = 1; i <= n; i++) {
        while (!q.empty() && q.front() < i - x - 1) q.pop_front();
        dp[i] = dp[q.front()] + a[i];
        while (!q.empty() && dp[q.back()] >= dp[i]) q.pop_back();
        q.push_back(i);
    }
    return *min_element(dp + max(1, n - x), dp + n + 1) <= T;
}
```

### 复杂度

外层二分 `O(log n)`，每次检查线性，整体复杂度 `O(n log n)`，空间复杂度 `O(n)`。

## 30. [P2627 [USACO11OPEN] Mowing the Lawn G](https://www.luogu.com.cn/problem/P2627)

`单调队列` `线性 DP` `前缀和`

### 题意

一排奶牛各有一个效率值，可以选若干头参加工作，但任意连续被选中的奶牛不能超过 `K` 头。要求总效率最大。

### 分析

正面做“哪些牛选、哪些牛不选”不太好下手，比较顺的思路是把状态定在“哪里断开”。设 `f[i]` 表示前 `i` 头奶牛已经处理完，并且第 `i` 头奶牛不选时的最优值，那么最后一段连续被选的奶牛长度一定在 `1..K` 之间。

如果最后一段连续被选的是 `j+1..i`，那么前面必须在 `j` 处断开，也就是从 `f[j]` 转移过来，再加上这段的前缀和贡献。这样一来，朴素做法的瓶颈就暴露得很清楚：对每个 `i`，都要枚举最后一段到底有多长。

把式子整理后会得到一个“前缀和 + 窗口内最大值”的形态，而这个窗口宽度正好就是 `K`。也就是说，题目虽然表面上在限制“连续选中个数”，实际优化点却是把“枚举最后一段长度”压成一个滑动窗口最大值。

所以这题可迁移的想法是：遇到“连续选择长度受限”的问题，可以先把状态放在断开点上，再看最后一段长度是否刚好对应一个滑动窗口。

### 核心代码

```cpp
deque<int> q;
q.push_back(0);
for (int i = 1; i <= n; i++) {
    while (!q.empty() && q.front() < i - K) q.pop_front();
    f[i] = pre[i] + g[q.front()];
    g[i] = f[i - 1] - pre[i];
    while (!q.empty() && g[q.back()] <= g[i]) q.pop_back();
    q.push_back(i);
}
```

### 复杂度

时间复杂度 `O(n)`，空间复杂度 `O(n)`。

## 31. [T557186 烽火传递](https://www.luogu.com.cn/problem/T557186)

`单调队列` `线性 DP` `滑动窗口最小值`

### 题意

一排共有 `n` 座烽火台，第 `i` 座被点亮的代价是 `a[i]`。要求任意连续 `m` 座烽火台里都至少有一座被点亮，求满足条件时的最小总代价。

### 分析

这题最关键的是把“连续 `m` 个里至少选一个”改写成“相邻两座被点亮的烽火台之间，空着的位置不能超过 `m-1` 个”。于是如果最后一个点亮的位置是 `i`，那么上一个点亮的位置 `j` 只能落在区间 `[i-m, i-1]`。

设 `f[i]` 表示“点亮第 `i` 座烽火台，且前 `i` 个位置已经全部合法”时的最小代价，就有

`f[i] = a[i] + min(f[j]) (i-m <= j < i)`。

这已经是标准的滑动窗口最小值模型：随着 `i` 从左往右扫描，候选决策区间始终是一个长度为 `m` 的窗口。用单调递增队列维护窗口内 `f[j]` 最小的位置，就能把每次转移从 `O(m)` 降到均摊 `O(1)`。最后答案不是 `f[n]`，而是最后一段长度不足 `m` 的后缀里任选一个点亮位置，因此取 `i in [n-m+1, n]` 的最小 `f[i]`。

### 核心代码

```cpp
deque<int> q;
q.push_back(0);
for (int i = 1; i <= n; i++) {
    while (!q.empty() && q.front() < i - m) q.pop_front();
    f[i] = f[q.front()] + a[i];
    while (!q.empty() && f[q.back()] >= f[i]) q.pop_back();
    q.push_back(i);
}
int ans = INF;
for (int i = max(1, n - m + 1); i <= n; i++)
    ans = min(ans, f[i]);
```

### 复杂度

每个下标至多进出队一次，时间复杂度 `O(n)`，空间复杂度 `O(n)`。

## 32. [U162981 最大连续和](https://www.luogu.com.cn/problem/U162981)

`前缀和` `单调队列` `最长度限制`

### 题意

给定一个长度为 `n` 的序列，要求从中选一段连续子段，长度不能超过 `m`，并让子段和最大。

### 分析

这题和普通最大子段和唯一的差别，就是左端点不能离右端点太远。只要看到“连续子段和 + 长度上限”，第一步都应该先写前缀和。

若答案区间是 `(j, i]`，它的和就是 `pre[i] - pre[j]`，同时必须满足 `i - j <= m`。也就是说，当右端点固定为 `i` 时，我们只需要在最近 `m` 个合法前缀和里，找一个尽量小的 `pre[j]`。

这就把问题彻底改写成了“滑动窗口最小值”：右端点每往右走一步，合法的前缀和窗口也随之右移一格。维护一个递增队列后，队头始终就是当前窗口里的最小前缀和，因此能立刻求出以 `i` 结尾的最优答案。

以后再遇到“长度不超过 / 至少 / 恰好”的子段和问题，都应该先问自己：能不能把约束变成“右端点扫描时，左端点只在某个窗口里选”，如果能，单调队列往往就不远了。

### 核心代码

```cpp
deque<int> q;
q.push_back(0);
for (int i = 1; i <= n; i++) {
    while (!q.empty() && q.front() < i - m) q.pop_front();
    ans = max(ans, pre[i] - pre[q.front()]);
    while (!q.empty() && pre[q.back()] >= pre[i]) q.pop_back();
    q.push_back(i);
}
```

### 复杂度

时间复杂度 `O(n)`，空间复杂度 `O(n)`。

## 33. [P1714 切蛋糕](https://www.luogu.com.cn/problem/P1714)

`前缀和` `单调队列` `最大子段和`

### 题意

有一串连续蛋糕块，每块有幸运值。现在只能吃一段连续区间，且长度不能超过 `m`，要求幸运值总和最大。

### 分析

这题表面是“切蛋糕”，其实和最大子段和只有一步之差：多了一个“长度不能超过 `m`”的限制。遇到“连续区间和 + 长度限制”，最自然的动作就是先写前缀和。

设 `pre[i]` 为前缀和，若答案区间是 `(j, i]`，那么它的和就是 `pre[i] - pre[j]`，同时必须满足 `i - j <= m`。这说明右端点固定后，真正要做的不是找某个神奇断点，而是在最近 `m` 个合法前缀里挑一个最小的 `pre[j]`。

于是原题被彻底翻译成了一个窗口问题：扫描右端点 `i` 时，动态维护“最近 `m` 个前缀和里的最小值”。队头最小，这就是单调队列；而且这个模板不只适用于本题，几乎所有“长度受上限约束的最大子段和”都会走同一条路。

所以以后再看到“长度不超过 `m` 的最优连续区间”，别先想 DP，先问自己：能不能写成 `pre[i] - pre[j]`，然后把合法 `j` 看成一个滑动窗口。

### 核心代码

```cpp
deque<int> q;
q.push_back(0);
for (int i = 1; i <= n; i++) {
    while (!q.empty() && q.front() < i - m) q.pop_front();
    ans = max(ans, pre[i] - pre[q.front()]);
    while (!q.empty() && pre[q.back()] >= pre[i]) q.pop_back();
    q.push_back(i);
}
```

### 复杂度

时间复杂度 `O(n)`，空间复杂度 `O(n)`。

## 34. [P2629 好消息，坏消息](https://www.luogu.com.cn/problem/P2629)

`环形前缀和` `单调队列` `区间最小值`

### 题意

可以把消息序列做一次循环移位后再依次汇报。每条消息都有好坏度，要求在整个汇报过程中老板心情始终不低于 `0`，问哪些起点是合法的。

### 分析

“老板心情始终不为负”这种叙述，翻译成算法语言就是：从某个起点开始往后走时，累计和的每一个前缀都不能小于 `0`。也就是说，本质问题不是枚举起点，而是判断一整段前缀和会不会下穿。

因为允许循环移位，所以先把序列复制一遍，把环摊成链。若以 `k` 为起点，那么接下来长度为 `n` 的那一段过程中，任意时刻的累计和都必须至少为 `0`。写成前缀和，就是区间 `[k, k+n-1]` 里的最小前缀和要不小于 `pre[k-1]`。

于是原题被翻译成了一个非常标准的窗口判定：对每个起点 `k`，询问后面长度为 `n` 的窗口里最小前缀和是多少。既然窗口长度固定、右端点单调右移，就可以用单调队列在线维护窗口最小值。

所以这题真正该学会的是一种套路：**环上每个起点做合法性判定**，通常都要先断环成链，再把“全过程合法”改写成“某个固定长度窗口里的最值条件”。

### 核心代码

```cpp
deque<int> q;
for (int i = 1; i <= 2 * n; i++) {
    while (!q.empty() && q.front() < i - n + 1) q.pop_front();
    while (!q.empty() && pre[q.back()] >= pre[i]) q.pop_back();
    q.push_back(i);
    if (i >= n) ok[i - n + 1] = pre[q.front()] >= pre[i - n];
}
```

### 复杂度

时间复杂度 `O(n)`，空间复杂度 `O(n)`。

## 35. [P1886 【模板】单调队列 / 滑动窗口](https://www.luogu.com.cn/problem/P1886)

`单调队列` `滑动窗口` `模板`

### 题意

给定长度为 `n` 的序列和窗口大小 `k`，窗口每次向右平移一格。要求同时输出每个窗口里的最小值和最大值。

### 分析

这题虽然是模板题，但真正该学的不是代码，而是“为什么队列里只剩这些元素”。窗口向右滑动时，一个元素之所以还能留在队列里，只可能是因为它还没过期，而且未来仍有机会成为某个窗口的最值。

因此队列要同时维护两种正确性：位置上不能过期，数值上不能被后来的更优元素支配。维护最小值时，队列里的下标必须按对应数值递增排列；只要新元素更小，那么它左边那些更大、还更靠前的元素以后无论窗口怎么滑，都不可能再成为最小值了。

最大值同理维护递减队列。两边写法几乎完全对称，所以这题其实是在训练你把“候选人淘汰条件”说清楚，而不是背两份模板。更进一步讲，队列里为什么一定存下标，也正是因为你既要比值，又要判断是否过期。

所以单调队列的本质不是“记住整个窗口”，而是“把窗口里所有已经失去竞争力的元素提前淘汰掉”。后面你看到的所有滑窗 DP、前缀和判定、二维窗口，本质都在重复这个思路。

### 核心代码

```cpp
for (int i = 1; i <= n; i++) {
    while (!qmin.empty() && qmin.front() <= i - k) qmin.pop_front();
    while (!qmax.empty() && qmax.front() <= i - k) qmax.pop_front();
    while (!qmin.empty() && a[qmin.back()] >= a[i]) qmin.pop_back();
    while (!qmax.empty() && a[qmax.back()] <= a[i]) qmax.pop_back();
    qmin.push_back(i), qmax.push_back(i);
    if (i >= k) mn.push_back(a[qmin.front()]), mx.push_back(a[qmax.front()]);
}
```

### 复杂度

每个下标只会进出队一次，时间复杂度 `O(n)`，空间复杂度 `O(k)`。

## 36. [P2698 [USACO12MAR] Flowerpot S](https://www.luogu.com.cn/problem/P2698)

`单调队列` `双指针` `窗口极差`

### 题意

每滴水有横坐标 `x` 和下落高度 `y`。要把花盆放在 `x` 轴上，使同一个花盆接到的最早和最晚水滴时间差至少为 `D`，并让花盆宽度最小。

### 分析

先抓住真正决定答案的量：花盆放在 `x` 轴上，本质上是在选一个横坐标区间。这个区间一旦固定，最早和最晚接到水滴的时间差，其实只由区间内最高和最低的水滴高度之差决定。

于是原题马上变成：在按 `x` 排序后的点集里，找一个最短连续区间，使其中 `max(y) - min(y) >= D`。这里“连续区间”四个字很重要，因为花盆在数轴上覆盖的是一段实实在在的区间；只要按 `x` 排序后，它接住的点一定对应一个连续下标段。

看到“最短区间 + 区间内最大值最小值约束”，就应该想到双指针配两条单调队列。右端点负责把窗口扩到第一次满足条件，左端点负责在保持条件成立的前提下尽量缩短区间。与此同时，一条递减队列维护窗口内最大高度，一条递增队列维护最小高度，这样每次都能 `O(1)` 判断当前窗口是否合法。

所以这题可迁移的思路是：**当区间合法性只由窗口内极值决定时，双指针往往要和“最大值队列 + 最小值队列”一起使用。**

### 核心代码

```cpp
for (int l = 1, r = 0; l <= n; l++) {
    while (r < n && maxY - minY < D) {
        ++r;
        push_max(r); push_min(r);
    }
    if (maxY - minY >= D) ans = min(ans, x[r] - x[l]);
    if (qmax.front() == l) qmax.pop_front();
    if (qmin.front() == l) qmin.pop_front();
    maxY = qmax.empty() ? -INF : y[qmax.front()];
    minY = qmin.empty() ? INF : y[qmin.front()];
}
```

### 复杂度

排序后双指针线性推进，时间复杂度 `O(n log n)`，空间复杂度 `O(n)`。

## 37. [P1440 求m区间内的最小值](https://www.luogu.com.cn/problem/P1440)

`单调队列` `滑动窗口最小值` `在线输出`

### 题意

对序列中的每个位置，输出它前面最近 `m` 个数里的最小值；如果前面不足 `m` 个，就把已有的那些全算上；如果前面一个数都没有，则输出 `0`。

### 分析

这题和模板题几乎一模一样，但恰恰因为太像，最容易把窗口边界写错。这里问的是“当前位置之前最近 `m` 个数”的最小值，所以当前元素 `a[i]` 本身并不属于这一轮要查询的窗口。

因此虽然维护方式仍然是递增队列，代码顺序却必须先想清：先删掉已经离开最近 `m` 个范围的过期元素，此时队头就是当前位置该输出的答案；等输出完之后，才能把 `a[i]` 自己按单调性插进去，供后面的点使用。

这里还顺带说明了为什么队列里必须存“下标”而不是“值”。因为我们不仅要比较大小，还要知道一个元素有没有过期离开窗口；只有下标才能同时处理这两件事。

所以这题最该学会的不是模板本身，而是一个判断方法：**窗口是“到当前位置为止”还是“当前位置之前”**？这个问题会直接决定你代码里到底是“先查后插”还是“先插后查”。以后看到类似的在线滑窗题，先把数学上的窗口区间写出来，再去落代码顺序，错误率会低很多。

### 核心代码

```cpp
deque<int> q;
for (int i = 1; i <= n; i++) {
    while (!q.empty() && q.front() < i - m) q.pop_front();
    cout << (q.empty() ? 0 : a[q.front()]) << '\n';
    while (!q.empty() && a[q.back()] >= a[i]) q.pop_back();
    q.push_back(i);
}
```

### 复杂度

时间复杂度 `O(n)`，空间复杂度 `O(m)`。

## 38. [P2032 扫描](https://www.luogu.com.cn/problem/P2032)

`单调队列` `滑动窗口最大值` `模板应用`

### 题意

有一块长度为 `k` 的木板覆盖在序列上，木板每次向右移动一格。要求在每个位置上输出当前被木板覆盖区间中的最大值。

### 分析

这题本身就是“滑动窗口最大值”最直接的应用，所以真正该学的不是结论，而是单调队列到底在维护什么。它从来不是把整个窗口原样存下来，而是在不断筛掉“以后再也不可能成为最大值”的元素。

窗口长度固定为 `k`，每次只向右移动一格，于是每个元素只会经历进入窗口、参与竞争、最后过期离开这几个阶段。若新来的元素更大，那么队尾那些更小、而且位置更靠左的元素以后无论窗口怎么滑，都不可能再打败它，自然可以立刻删除。

这里同样要注意，队列里存的是下标不是值，因为窗口在移动，过期判断必须靠位置完成。值只负责比较谁更优，下标负责判断谁还在场，这两件事缺一不可。

所以队列里实际留下来的，是“当前窗口中仍有资格竞争最大值的候选人”，而且它们按数值递减排列，队头永远是真正的最大值。这也是单调队列最值得迁移的思维：不是记住全部元素，而是**提前淘汰所有已经失去竞争力的候选项**。

### 核心代码

```cpp
deque<int> q;
for (int i = 1; i <= n; i++) {
    while (!q.empty() && q.front() <= i - k) q.pop_front();
    while (!q.empty() && a[q.back()] <= a[i]) q.pop_back();
    q.push_back(i);
    if (i >= k) cout << a[q.front()] << '\n';
}
```

### 复杂度

时间复杂度 `O(n)`，空间复杂度 `O(k)`。

## 39. [P2216 [HAOI2007] 理想的正方形](https://www.luogu.com.cn/problem/P2216)

`二维单调队列` `窗口最值` `矩阵处理`

### 题意

给定一个矩阵，要求找一个边长固定为 `n` 的正方形子矩阵，使这个正方形里的最大值与最小值之差最小。

### 分析

如果直接对每个 `n×n` 正方形暴力求最大值和最小值，复杂度会非常大。看到这种“固定大小子矩阵里的最值”问题，最自然的思路就是把二维拆成两次一维，因为窗口大小在两个方向上都是固定的。

第一步，按行做长度为 `n` 的滑窗，求出每一行上所有横向窗口的最小值和最大值。这样原矩阵就被压成了两张中间表：一张记录“这个横向宽度为 `n` 的条带里最小是多少”，另一张记录最大是多少。

第二步，再把这两张中间表按列做一次同样长度为 `n` 的滑窗。这样每个 `n×n` 正方形的最小值和最大值就都已经被处理出来，查询一个正方形时不再需要重新扫描整块区域。

所以二维单调队列最值得举一反三的地方，就是这种“先沿一个维度滑，再沿另一个维度滑”的分解方式。很多固定大小子矩阵最值题，本质都不是新算法，而是一维滑窗模板在二维上的两次串联。

### 核心代码

```cpp
for (int i = 1; i <= a; i++) {
    solve_row_min(i, rowMin[i]);
    solve_row_max(i, rowMax[i]);
}
for (int j = 1; j + n - 1 <= b; j++) {
    solve_col_min(j, rowMin, mn);
    solve_col_max(j, rowMax, mx);
}
for (int i = n; i <= a; i++)
    for (int j = n; j <= b; j++)
        ans = min(ans, mx[i][j] - mn[i][j]);
```

### 复杂度

两次二维滑窗后总体复杂度 `O(ab)`，空间复杂度 `O(ab)`。
