# 一本通 动态规划高阶专题精选解题报告

这一组题从连续段划分一路走到树上换根、数位统计、状态压缩、单调结构与斜率优化。题型跨度很大，但阅读主线很统一：先把题目的决策边界翻译成状态，再判断该用朴素转移、局部合并，还是把转移式继续压到队列或凸包里。

# 一、区间划分与连续段建模

这一章先把最经典的 DP 直觉练扎实：只要题目要求“按顺序处理”“每次选连续一段”“最后一次操作切开左右两部分”，状态往往就应该落在前缀或区间上。难点不在公式，而在认出哪一段信息必须保留到下一次合并。

## 1. [1766：成绩单](http://ybt.ssoier.cn:8088/problem_show.php?pid=1766)

`区间DP` `分段DP`

### 题意

有一叠按编号排好的成绩单，每次只能抽出当前序列中的一段连续区间发给对应同学。每一批的代价由这一段内部的分数波动决定，整道题要求把全部成绩单分成若干批后，使总代价最小。

### 分析

因为每一批必须是连续段，所以最后一步一定是“前面若干份已经发完，最后一批发了区间 $(j+1\sim i)$”。这就把问题直接压成前缀划分：设 `dp[i]` 表示前 `i` 份成绩单发完的最小代价，那么转移就是枚举最后一批从哪里开始。

真正需要先想清楚的是“一个固定区间作为一批时，它的代价能不能快速得到”。把区间内最大值、最小值以及题目给出的批次代价规则预处理出来后，`dp[i]=min(dp[j]+cost[j+1][i])` 就顺理成章了。这类题的核心手感就是：**连续分段问题，先把段代价做成可查表，再做前缀 DP。**

### 核心代码

```cpp
for (int l = 1; l <= n; l++) {
  int mx = w[l], mn = w[l];
  for (int r = l; r <= n; r++) {
    mx = max(mx, w[r]);
    mn = min(mn, w[r]);
    cost[l][r] = 1LL * (mx - mn) * (mx - mn);
  }
}
fill(dp, dp + n + 1, INF);
dp[0] = 0;
for (int i = 1; i <= n; i++)
  for (int j = 0; j < i; j++)
    dp[i] = min(dp[i], dp[j] + cost[j + 1][i]);
```

### 复杂度

时间复杂度 $O(n^2)$，空间复杂度 $O(n^2)$。

## 2. [1767：字符合并](http://ybt.ssoier.cn:8088/problem_show.php?pid=1767)

`区间DP` `结果状态`

### 题意

给定一个字符序列和一套合并规则。每次必须把一段相邻字符按规则合成一个新字符并获得分数，最终把整段串不断缩成一个字符，要求总得分最大。

### 分析

这题最容易漏掉的信息是：一个区间不仅有“最优得分”，还要知道它最后会被合成成什么字符。因为外层继续合并时，能不能触发哪条规则完全取决于这个结果字符，所以状态必须写成 `f[l][r][c]`，表示区间 `[l,r]` 最终合成字符 `c` 的最大得分。

接着就用“最后一次合并”来拆区间。无论规则要求把区间拆成几块，本质上都是先把若干子区间各自合成特定字符，再按规则把它们并起来。只要先把小区间的结果枚举出来，大区间的转移就是标准的区间合并。这里真正练的是：**区间 DP 不只保存数值最优，还可能要保存区间最终的类型。**

### 核心代码

```cpp
memset(f, -0x3f, sizeof(f));
for (int i = 1; i <= n; i++) f[i][i][a[i]] = 0;
for (int len = 2; len <= n; len++)
  for (int l = 1, r; (r = l + len - 1) <= n; l++)
    for (int mid = l; mid < r; mid++)
      for (int x = 0; x < SIG; x++)
        for (int y = 0; y < SIG; y++) {
          int z = to[x][y], add = score[x][y];
          f[l][r][z] = max(f[l][r][z], f[l][mid][x] + f[mid + 1][r][y] + add);
        }
```

### 复杂度

时间复杂度 $O(n^3\cdot \Sigma^2)$，空间复杂度 $O(n^2\cdot \Sigma)$。

## 3. [1768：交换](http://ybt.ssoier.cn:8088/problem_show.php?pid=1768)

`区间DP` `计数DP`

### 题意

从初始排列 $0,1,2,\dots,n-1$ 出发，要按某个长度为 $n-1$ 的相邻交换序列操作，最终得到给定排列 $p$。题目要求统计这样的交换序列有多少种。

### 分析

长度恰好是 $n-1$ 次相邻交换，意味着每一步都在把一个新元素“接”到已经形成的目标相对顺序旁边。把每个值在目标排列中的位置记成 `pos[x]`，会发现已经就位的一批最小元素在目标排列里始终对应一个连续区间，于是状态自然变成 `f[l][r]`：当前已经构成了目标排列中的区间 `[l,r]`。

下一步要加入的元素是更大的那个值，它在目标里的位置只能恰好落在当前区间左边或右边，否则就不可能只靠一次新的相邻交换接进来。因此转移只需要看 `pos[len]` 是否等于 `l-1` 或 `r+1`。这题想法的关键是先看值域增长，而不是直接盯着交换过程本身。

### 核心代码

```cpp
for (int i = 0; i < n; i++) pos[p[i]] = i;
memset(f, 0, sizeof(f));
f[pos[0]][pos[0]] = 1;
for (int len = 1; len < n; len++)
  for (int l = 0; l + len - 1 < n; l++) {
    int r = l + len - 1;
    if (!f[l][r]) continue;
    if (l && pos[len] == l - 1) add(f[l - 1][r], f[l][r]);
    if (r + 1 < n && pos[len] == r + 1) add(f[l][r + 1], f[l][r]);
  }
ans = f[0][n - 1];
```

### 复杂度

时间复杂度 $O(n^2)$，空间复杂度 $O(n^2)$。

## 4. [1769：景中人](http://ybt.ssoier.cn:8088/problem_show.php?pid=1769)

`分段DP` `区间覆盖`

### 题意

平面上给出若干点，需要用底边贴在 $y=0$ 上的矩形把所有点覆盖。每个矩形面积不能超过上限，问最少需要多少个矩形。

### 分析

先按横坐标排序。因为矩形底边都在 $y=0$ 上，一个矩形一旦覆盖了一批点，这些点在排序后必然是一段连续区间，不会出现“隔着一个点去覆盖两边”的情况。于是又回到了典型的前缀划分：`dp[i]` 表示前 `i` 个点的最少矩形数。

那段代价也很好判断。若区间 `[j+1,i]` 由一个矩形覆盖，那么宽度取左右端点的横坐标差，高度取区间中的最大纵坐标，只要面积不超过限制就能一块解决。于是从右端点固定、向左扩展时不断维护最大高度，就能判断哪些前驱合法。识别出“一个矩形对应一个连续点段”，整题就直接落回线性分段模型。

### 核心代码

```cpp
sort(p + 1, p + n + 1);
fill(dp, dp + n + 1, INF);
dp[0] = 0;
for (int i = 1; i <= n; i++) {
  int mx = 0;
  for (int j = i; j >= 1; j--) {
    mx = max(mx, p[j].y);
    long long area = 1LL * (p[i].x - p[j].x) * mx;
    if (area <= S) dp[i] = min(dp[i], dp[j - 1] + 1);
  }
}
```

### 复杂度

时间复杂度 $O(n^2)$，空间复杂度 $O(n)$。

## 5. [1770：跳跳棋](http://ybt.ssoier.cn:8088/problem_show.php?pid=1770)

`博弈DP` `区间状态`

### 题意

Alice 从第一行任选一个格子出发，每轮先由 Bob 在若干上下相邻格子间加墙，但不能切断她到最后一行的连通性；然后 Alice 走一步并累加新格子的分值。双方都最优，要求最终 Alice 到达第 $n$ 行时的最小可能罚分。

### 分析

Bob 只能加竖向墙，而且不能完全堵死，所以对 Alice 来说，每走到某一行时，未来仍然可达的列集合会被压成若干连续段；而 Bob 真正能控制的是把可行区域继续收窄，让 Alice 只能在更差的位置中选路。于是状态不再只是“人在哪个点”，还要包含“这一层还能活动到哪些列”，也就是区间极小极大 DP。

从思考顺序上，应该先站在 Bob 的视角：他不是任意改图，而是在不破坏终点可达的前提下，选择一个最坏的可达区间。再站回 Alice 的视角，她在这个区间里选一步最小化后续总代价。把两层选择顺序写清楚，转移就自然是 `max` 包 `min` 的形式。

### 核心代码

```cpp
for (int c = 1; c <= m; c++) f[n][c][c] = a[n][c];
for (int i = n - 1; i >= 1; i--)
  for (int l = 1; l <= m; l++)
    for (int r = l; r <= m; r++) {
      int worst = 0;
      for (auto seg : nextSeg[i][l][r]) {
        int best = INF;
        for (int c = seg.first; c <= seg.second; c++)
          best = min(best, a[i][c] + trans(i, c, seg));
        worst = max(worst, best);
      }
      f[i][l][r] = worst;
    }
ans = min { f[1][c][c] }   // c = 1..m
```

### 复杂度

时间复杂度约为 $O(nm^4)$，空间复杂度约为 $O(nm^2)$。

# 二、树上合并、换根与期望

树形 DP 的共同难点，是你不能只看一个儿子怎么转，而要始终想清“子树信息如何向父亲汇总”“父侧信息如何反推给儿子”。这一章从子树统计一路走到换根、树上背包和随机游走，练的就是信息在树上的双向流动。

## 6. [1771：仓库选址](http://ybt.ssoier.cn:8088/problem_show.php?pid=1771)

`树形DP` `换根DP`

### 题意

给定一棵带边权的树。两点之间的贡献由距离与固定常数做按位异或得到，要求对每个点都求出它与所有点的贡献和。

### 分析

异或最适合按位拆开。对某一位来说，只要知道从当前点出发到所有点的距离这一位有多少个是 $0$、多少个是 $1$，这一位对答案的贡献就能一次算完。因此可以把问题拆成若干个互不干扰的按位树形统计。

做法上先在子树内统计：`down[u][b][0/1]` 表示 `u` 子树中与 `u` 的距离第 `b` 位为 `0/1` 的点数。再做一次换根，把父侧的统计传给儿子，就能得到整棵树范围内相对每个点的计数。真正的模型不是“异或很怪”，而是“树上全点对贡献 + 按位可分”，于是自然进入换根框架。

### 核心代码

```cpp
void dfs1(int u, int fa) {
  down[u][0][0] = 1;
  for (auto [v, w] : g[u]) if (v != fa) {
    dfs1(v, u);
    for (int b = 0; b < B; b++)
      for (int t = 0; t < 2; t++)
        down[u][b][t ^ ((w >> b) & 1)] += down[v][b][t];
  }
}
void dfs2(int u, int fa) {
  for (int b = 0; b < B; b++) ans[u] += calcBit(u, b);
  for (auto [v, w] : g[u]) if (v != fa) {
    reroot(u, v, w);
    dfs2(v, u);
    rollback(u, v, w);
  }
}
```

### 复杂度

时间复杂度 $O(n\log V)$，空间复杂度 $O(n\log V)$。

## 7. [1772：动漫排序](http://ybt.ssoier.cn:8088/problem_show.php?pid=1772)

`树形DP` `组合计数`

### 题意

树上每个结点代表一部作品，要求一个结点必须排在其所有后继之前；同时同一父节点的若干儿子还带有给定的优先顺序。问满足这些限制的线性排序有多少种。

### 分析

把一个结点看成必须先于整棵子树出现，那么它的子树内部顺序数就是标准的树上合并计数。对每个儿子先递归求出 `f[v]` 和 `sz[v]`，再把这些儿子的子树按题目给定顺序依次并入当前结点。

为什么合并时会出现组合数？因为当某个儿子子树整体相对次序已经固定后，它只需要和当前已经拼好的序列做“穿插”，而父亲必须最先出现、儿子之间的先后关系又有限制，所以合法穿插数恰好就是若干个组合数连乘。这类题的关键不是公式本身，而是先认出“每个子树先独立排，再做受限插空”。

### 核心代码

```cpp
void dfs(int u) {
  f[u] = 1; sz[u] = 1;
  for (int v : son[u]) {
    dfs(v);
    f[u] = 1LL * f[u] * f[v] % MOD;
    f[u] = 1LL * f[u] * C[sz[u] + sz[v] - 1][sz[v]] % MOD;
    sz[u] += sz[v];
  }
}
```

### 复杂度

时间复杂度 $O(n^2)$（组合数预处理后为树上一次合并），空间复杂度 $O(n^2)$ 或 $O(n)$。

## 8. [1773：消息传递](http://ybt.ssoier.cn:8088/problem_show.php?pid=1773)

`树形DP` `换根DP`

### 题意

消息从某个人开始传播。每个已经知道消息的人每单位时间只能通知一个直接相邻的人，要求求出最短传遍整棵树的时间，并找出所有能达到这个最优时间的起点。

### 分析

从某个点开始时，它要把消息发向多个分支。若一个分支本身传播很慢，就应尽量先通知它，所以儿子子树要按“完成时间较大者优先”排序。这就是典型的树上广播 DP：对子树求出 `down[u]`，表示从 `u` 出发只覆盖自己子树所需的最短时间。

但题目要枚举所有起点，因此必须把父侧的影响也算进去。换根时，把“来自父亲以及兄弟子树的信息”当成一个额外分支并入儿子的排序中，就能求出 `all[u]`。思维关键是：这题不是普通最长路，而是**多个分支按顺序依次启动**，所以转移里会出现“第几次通知”这一层偏移。

### 核心代码

```cpp
void dfs1(int u, int fa) {
  vector<int> t;
  for (int v : g[u]) if (v != fa) dfs1(v, u), t.push_back(down[v]);
  sort(t.begin(), t.end(), greater<int>());
  for (int i = 0; i < (int)t.size(); i++) down[u] = max(down[u], t[i] + i + 1);
}
void dfs2(int u, int fa, int up) {
  all[u] = mergeChildren(u, fa, up);
  for (int v : g[u]) if (v != fa)
    dfs2(v, u, contributionWithout(u, v, fa, up));
}
```

### 复杂度

时间复杂度 $O(n\log n)$，空间复杂度 $O(n)$。

## 9. [1774：大逃杀](http://ybt.ssoier.cn:8088/problem_show.php?pid=1774)

`树上背包` `树形DP`

### 题意

树上每个点可能有敌人和资源，移动与战斗都会消耗时间，资源会提升武力值。在总时间不超过限制的前提下，可以从任意点空降开始，求最终武力值最大是多少。

### 分析

时间受限、收益可累积、图又是一棵树，这基本就是树上背包。把树定根后，`f[u][t][0/1]` 可以表示在 `u` 子树内花费 `t` 时间时的最大武力，其中最后一维区分“处理完后是否回到 `u`”。因为走进子树、打完、拿资源、是否返回，都会影响后面还能不能继续并其他儿子。

从建模角度看，这题的难点不是细节很多，而是先把“树上行走”翻译成可卷积的子树物品。每个儿子子树都像一个带花费和收益的组，父亲逐个把它们合并即可；如果允许从任意点开始，可以加超级根或在转移里额外维护“起点已经选过”的状态统一处理。

### 核心代码

```cpp
void dfs(int u, int fa) {
  initState(u);
  for (auto [v, w] : g[u]) if (v != fa) {
    dfs(v, u);
    for (int i = T; i >= 0; i--)
      for (int j = 0; j + i + 2 * w <= T; j++)
        f[u][i + j + 2 * w][1] = max(f[u][i + j + 2 * w][1], f[u][i][1] + f[v][j][1]);
  }
  applyFightAndReward(u);
}
```

### 复杂度

时间复杂度 $O(nT^2)$，空间复杂度 $O(nT)$。

## 10. [1775：梦中漫步](http://ybt.ssoier.cn:8088/problem_show.php?pid=1775)

`树上期望` `换根DP`

### 题意

在树上做随机游走：每到一个点都会等概率走向一条相邻边。对于很多组 $(u,v)$ 询问，要求从 $u$ 出发直到第一次到达 $v$ 为止，期望经过多少条边。

### 分析

树上 hitting time 的关键在于“目标点固定后，往目标方向走”和“走向其他分支后再绕回来”贡献不同。若先固定根，可以求出每条有向边 `u->v` 的期望代价 `E(u,v)`：从 `u` 出发，第一次走进包含目标的那个方向前，需要额外在其他分支里来回绕多少步。

树是无环的，所以任意 `u` 到 `v` 的期望都能拆到唯一路径上，答案就是路径上若干条有向边期望的和。也就是说，先做树上的局部期望 DP，再把询问变成路径求和，模型就清楚了：**不是直接对每个询问做随机过程，而是预处理每条朝向边的贡献。**

### 核心代码

```cpp
void dfs1(int u, int fa) {
  for (int v : g[u]) if (v != fa) dfs1(v, u);
  for (int v : g[u]) if (v != fa)
    down[v] = 1 + sumOther(u, v);
}
void dfs2(int u, int fa) {
  for (int v : g[u]) if (v != fa) {
    up[v] = 1 + sumExclude(u, v);
    dfs2(v, u);
  }
}
long long query(int u, int v) {
  int w = lca(u, v);
  return pathSum(u, w, down, up) + pathSumRev(v, w, down, up);
}
```

### 复杂度

预处理时间复杂度 $O(n\log n)$，每次询问时间复杂度 $O(\log n)$，空间复杂度 $O(n\log n)$。

# 三、数位统计、二进制构造与递归分治

这一章的共同点，是答案不再靠“最后一刀切区间”，而是靠“按位往下压状态”。有的题按二进制高位到低位构造，有的题按数位和做计数，有的题把巨大坐标递归拆成标准块，本质上都是把全局对象拆成一层一层可控的小决策。

## 11. [1776：最大值](http://ybt.ssoier.cn:8088/problem_show.php?pid=1776)

`位运算DP` `按位贪心`

### 题意

给定一个数组和一个位运算类型，要在所有 $i<j$ 的二元组中求 `Ai op Aj` 的最大值，其中 `op` 可能是 `and`、`xor` 或 `or`。

### 分析

这类题最自然的思路不是枚举二元组，而是从高位到低位尝试“答案这一位能不能是 1”。一旦高位能取到 1，低位再怎么补都不会更差，所以问题变成：在已经固定了更高位前缀的前提下，是否还存在两个数满足当前位的要求。

于是状态本质上是“哪些数还保留在候选集合里”。对于 `or`，只要有一边这一位为 1；对于 `and`，两边都要为 1；对于 `xor`，两边这一位必须不同。把这层可行性判断写成按位 DP 或位集筛选，就能一路贪心出答案。重点是先把“最大值”翻译成“高位优先的可行性”。

### 核心代码

```cpp
long long ans = 0;
vector<int> cand(a + 1, a + n + 1);
for (int b = 30; b >= 0; b--) {
  vector<int> nxt;
  for (int x : cand) if (good(x, ans, b)) nxt.push_back(x);
  if (canMakeOne(nxt, ans, b)) {
    ans |= 1LL << b;
    cand.swap(nxt);
  }
}
```

### 复杂度

时间复杂度 $O(n\log V)$，空间复杂度 $O(n)$。

## 12. [1777：寻找整数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1777)

`数位DP` `构造`

### 题意

给定整数 $m,k$，要求找到正整数 $n$，使区间 $(n,2n]$ 中恰好有 $m$ 个整数的二进制表示里含有恰好 $k$ 个 $1$；同时还要统计所有可行的 $n$ 的个数。

### 分析

先把问题转成计数函数。设 `F(x)` 表示 `1..x` 中 popcount 等于 `k` 的数的个数，那么题目条件就是 `F(2n)-F(n)=m`。这就说明真正的工具是二进制数位 DP：我们要能快速算任意上界的 `F(x)`。

有了 `F` 之后，再从高位到低位构造 `n`。当前缀确定时，可以算出如果这一位填 `0` 或 `1`，后面还能得到多少个合法解，于是既能求字典序最小的可行 `n`，也能累计方案总数。思维主线是：**先把区间条件改写成前缀计数差，再把构造问题交给数位 DP。**

### 核心代码

```cpp
long long calc(long long x) {
  long long res = 0;
  for (int i = 60, ones = 0; i >= 0; i--)
    if (x >> i & 1) {
      res += C[i][k - ones];
      ones++;
    }
  return res + (popcount(x) == k);
}
long long countIn(long long n) { return calc(2 * n) - calc(n); }
for (int i = 60; i >= 0; i--)
  if (can(ans | (1LL << i))) ans |= 1LL << i;
```

### 复杂度

单次计数时间复杂度 $O(\log n)$，整体构造时间复杂度 $O(\log^2 n)$，空间复杂度 $O(\log n)$。

## 13. [1778：好数字](http://ybt.ssoier.cn:8088/problem_show.php?pid=1778)

`计数DP` `数位和`

### 题意

用给定数字集合构造长度为 $2n$ 的数字串。题目定义两类“好”条件，例如前半和后半数位和相等，或奇偶位置上的数位和相等，要求统计满足条件的方案数。

### 分析

只要限制只跟“某一半的数位和”有关，最直接的状态就是 `f[i][s]`：长度为 `i` 的前缀，数位和为 `s` 的方案数。先把长度为 `n` 时每个和的方案数算出来，前后半相等的答案就是 `sum f[n][s]^2`。

如果题目还要求另一种和式条件，就再做一层更细的状态，或者用容斥把“同时满足两种条件”的部分扣掉。重点是先看出：字符串本身很长，但真正影响合法性的只有总和这一维，所以可以把整个串压缩成“长度 + 和”的计数 DP。

### 核心代码

```cpp
f[0][0] = 1;
for (int i = 0; i < n; i++)
  for (int s = 0; s <= 9 * i; s++)
    for (int d : S)
      f[i + 1][s + d] = (f[i + 1][s + d] + f[i][s]) % MOD;
for (int s = 0; s <= 9 * n; s++)
  ans = (ans + 1LL * f[n][s] * f[n][s]) % MOD;
```

### 复杂度

时间复杂度 $O(n\cdot \Sigma\cdot |S|)$，空间复杂度 $O(n\Sigma)$。

## 14. [1779：第K小数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1779)

`状压DP` `余数DP`

### 题意

给出一个数字串，要求重新排列这些数字，不能有前导零，并且新数必须能被 $17$ 整除。需要输出按数值从小到大排序后的第 $K$ 个合法结果。

### 分析

因为每一位数字都要恰好使用一次，最自然的状态是 `f[mask][rem]`：已经用了 `mask` 这些位置后，当前前缀对 $17$ 的余数为 `rem` 的方案数。这样所有后缀方案数都能先预处理出来。

接着从高位到低位贪心构造答案。当前位尝试从小到大放某个尚未使用的数字，只要这条选择对应的后缀方案数不少于 `K`，就说明第 `K` 小答案在这条分支里；否则把这部分方案数减掉继续试更大的数字。这种“先状压计数，再按字典序构造”的套路非常常见。

### 核心代码

```cpp
f[0][0] = 1;
for (int mask = 0; mask < (1 << n); mask++)
  for (int r = 0; r < 17; r++) if (f[mask][r])
    for (int i = 0; i < n; i++) if (!(mask >> i & 1))
      add(f[mask | (1 << i)][(r * 10 + d[i]) % 17], f[mask][r]);
for (int pos = 0, mask = 0, r = 0; pos < n; pos++)
  for (int c = 0; c <= 9; c++) if (tryDigit(pos, mask, r, c, K)) {
    takeDigit(pos, mask, r, c);
    break;
  }
```

### 复杂度

时间复杂度 $O(2^n\cdot n\cdot 17)$，空间复杂度 $O(2^n\cdot 17)$。

## 15. [1780：修墙](http://ybt.ssoier.cn:8088/problem_show.php?pid=1780)

`递归DP` `分形分治`

### 题意

无限黑白矩阵按 `A -> [[A,A],[A,B]]` 的规则不断迭代生成，其中 `B` 是全黑块。每次询问取出一个子矩形，要求统计其中白格形成了多少个四连通块。

### 分析

坐标很大，显然不能真的展开矩阵；但生成规则非常自相似，所以任何一个大矩形都能递归拆成若干个标准分形块。于是每个块需要保存的不只是“内部有多少连通块”，还要保存“上下左右边界上哪些白格彼此连通”，这样父块在合并四个子块时才能判断需要把多少个连通块拼成一个。

因此状态的本质是块级轮廓信息。对标准块预处理 `cnt + boundary mask + union relation`，询问矩形时递归切分并按同样规则合并即可。这题看似是图论，真正落脚点却是一个典型的“递归结构 + 边界状态 DP”。

### 核心代码

```cpp
Node solve(int k, long long x1, long long y1, long long x2, long long y2) {
  if (x1 > x2 || y1 > y2) return empty();
  if (k == 0) return singleWhite();
  long long len = 1LL << (k - 1);
  auto A = solvePart(k - 1, x1, y1, x2, y2, 0, 0);
  auto B = solvePart(k - 1, x1, y1, x2, y2, 0, len);
  auto C = solvePart(k - 1, x1, y1, x2, y2, len, 0);
  auto D = blackPart(k - 1, x1, y1, x2, y2, len, len);
  return merge4(A, B, C, D);
}
```

### 复杂度

单次询问时间复杂度约为 $O(\log \max(x_2,y_2))$，空间复杂度 $O(\log \max(x_2,y_2))$。

# 四、状态压缩与局部结构刻画

当规模已经小到可以枚举子集、轮廓或局部边界时，真正要练的是“最少保存哪些信息就够了”。这一章里，状态有时是一批点的集合，有时是一层上的可达奇偶，有时是一条轮廓线，但本质都一样：把局部结构压成一个可滚动的状态。

## 16. [1781：死亡之树](http://ybt.ssoier.cn:8088/problem_show.php?pid=1781)

`状态压缩DP` `子集DP`

### 题意

给定一个无向图，要求统计其中有多少个树形子图，并且这些树形子图的叶子个数恰好为给定值。

### 分析

点数很小，最自然的做法就是对点集做子集 DP。对一个子集而言，树有两个判定条件：连通且边数等于点数减一；而叶子数又只和子图内各点的度数有关，所以状态里再带上叶子数量即可。

转移可以理解为“在一个已知连通无环的子集外再接入一个点”。若新点与当前子集恰好有若干连边，就能更新已有叶子的度变化，并得到更大的树。模型重点在于：**规模小就不要硬上图算法，直接把连通结构压成集合状态。**

### 核心代码

```cpp
f[1 << s][1] = 1;
for (int mask = 1; mask < (1 << n); mask++)
  for (int leaf = 0; leaf <= n; leaf++) if (f[mask][leaf])
    for (int v = 0; v < n; v++) if (!(mask >> v & 1)) {
      int deg = countEdge(mask, v);
      if (!deg) continue;
      int nleaf = leaf + (deg == 1) - changedLeaf(mask, v);
      add(f[mask | (1 << v)][nleaf], f[mask][leaf]);
    }
```

### 复杂度

时间复杂度 $O(2^n n^2)$，空间复杂度 $O(2^n n)$。

## 17. [1782：分层图](http://ybt.ssoier.cn:8088/problem_show.php?pid=1782)

`状态压缩DP` `线性代数`

### 题意

给定一个分层有向图。每一层之间的边可以选择是否执行一次“取反”操作，题目只关心源点到汇点的路径数奇偶性，要求统计最终路径数为偶数的操作方案数。

### 分析

既然只关心奇偶，那所有路径计数都可以在模 $2$ 意义下做。于是每一层之间的转移，不是一个庞大的路径数问题，而只是把当前层的可达奇偶向量乘上“原转移矩阵”或“取反后的矩阵”。

这样状态就非常紧：`f[i][mask]` 表示处理到第 `i` 层时，各点路径奇偶组成的二进制向量为 `mask` 的方案数。下一层只需枚举这一步是否翻转。题目看起来像图论构造，真正建模时却被压成了一个按层滚动的小状态 DP。

### 核心代码

```cpp
f[0][startMask] = 1;
for (int i = 0; i < L - 1; i++)
  for (int mask = 0; mask < (1 << w[i]); mask++) if (f[i][mask]) {
    int a = trans(mask, mat[i]);
    int b = trans(mask, revMat[i]);
    add(f[i + 1][a], f[i][mask]);
    add(f[i + 1][b], f[i][mask]);
  }
for (int mask = 0; mask < (1 << w[L]); mask++)
  if (((mask >> target) & 1) == 0) add(ans, f[L][mask]);
```

### 复杂度

时间复杂度 $O(\sum 2^{w_i})$，空间复杂度 $O(2^{\max w_i})$。

## 18. [1783：矩阵填数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1783)

`状态压缩DP` `计数DP`

### 题意

给一个矩阵填入 $1\sim m$ 之间的数，并给出若干子矩形限制：某些子矩形的最大值必须等于指定数。要求统计满足全部限制的填法数。

### 分析

矩阵本身可能不小，但限制个数很小。于是不能按格子暴力，而要按“一个格子属于哪些限制矩形”来分类：属于同一组限制的格子，在合法性上是等价的。这样原矩阵会被压成若干类格子，每类有固定数量。

再按值从大到小考虑。某个值放进哪些格子，会决定哪些限制第一次被“打到最大值”，因此状态可以写成“已经满足了哪些最大值限制”。本质上是把空间维度压掉，只保留与约束有关的布尔状态。

### 核心代码

```cpp
for (int s = 0; s < (1 << q); s++) dp[s] = 0;
dp[0] = 1;
for (int v = m; v >= 1; v--) {
  memset(ndp, 0, sizeof(ndp));
  for (int s = 0; s < (1 << q); s++) if (dp[s])
    for (int t : placeState[v]) {
      int ns = updateMask(s, t, v);
      add(ndp[ns], 1LL * dp[s] * ways[v][t] % MOD);
    }
  memcpy(dp, ndp, sizeof(dp));
}
```

### 复杂度

时间复杂度约为 $O(m\cdot 2^q\cdot S)$，空间复杂度 $O(2^q)$。

## 19. [1784：山谷](http://ybt.ssoier.cn:8088/problem_show.php?pid=1784)

`轮廓线DP` `状态压缩`

### 题意

在 $n\times m$ 的网格中填入 $1\sim nm$ 的一个排列。某个格子是山谷，当且仅当它比八连通邻居都小。给定哪些位置必须是山谷、哪些不能是山谷，求合法填数方案数。

### 分析

山谷条件只和相邻格子的相对大小有关，所以不必真的关心每个格子填了哪个具体数字，只要关心“当前已经放过的较小数字形成了什么局部关系”。按数字从小到大填时，某个格子一旦被放入，它是否可能成为山谷，取决于周围格子未来还能不能更小——而未来已经不可能了。

这就把题目压成一个局部边界状态问题：顺着网格扫描，状态维护当前轮廓线附近格子的相对大小关系以及山谷要求是否已被满足。由于 $n$ 很小，可以把一行或两行的局部结构整体状压。这里的关键手感是：排列题不一定存绝对值，很多时候只存**相对次序已经决定的局部事实**。

### 核心代码

```cpp
dp[0][initMask] = 1;
for (int pos = 0; pos < n * m; pos++) {
  memset(ndp, 0, sizeof(ndp));
  for (int mask = 0; mask < LIM; mask++) if (dp[pos][mask])
    for (int st : placeWays(mask, pos)) {
      int nmask = trans(mask, pos, st);
      add(ndp[nmask], dp[pos][mask]);
    }
  memcpy(dp[pos + 1], ndp, sizeof(ndp));
}
ans = sumValid(dp[n * m]);
```

### 复杂度

时间复杂度约为 $O(nm\cdot S)$，空间复杂度 $O(S)$，其中 $S$ 为轮廓线状态数。

## 20. [1785：简单的期望](http://ybt.ssoier.cn:8088/problem_show.php?pid=1785)

`概率DP` `状态压缩`

### 题意

初始值为 $x$，连续进行 $n$ 次随机操作：以概率 $p$ 把当前值变成 `x/2`，以概率 $1-p$ 变成 `x+1`。最后记结果中质因数 $2$ 的个数为 $d$，要求 $d$ 的期望。

### 分析

最终只关心二进制尾零个数，也就是 $v_2(w)$。因此没有必要把整个数值完整保存下来，真正重要的是当前数对 $2^k$ 的低位形态以及已经有多少个 $2$ 因子。特别是执行 `+1` 时，尾部连续的 $1$ 会全部翻成 $0$，这正是一个低位局部结构问题。

于是可以把状态压成“当前尾零个数 / 若干低位模式”的概率 DP。每一步按两种操作更新状态概率，最后把每个状态的尾零个数乘上概率求和即可。建模核心是先识别目标只与 2-adic 结构有关，而不是与完整数值有关。

### 核心代码

```cpp
dp[0][encode(x)] = 1.0;
for (int i = 0; i < n; i++) {
  clear(ndp);
  for (auto [st, pr] : dp[i]) {
    add(ndp[half(st)], pr * p);
    add(ndp[inc(st)], pr * (1.0 - p));
  }
  dp[i + 1].swap(ndp);
}
for (auto [st, pr] : dp[n]) ans += pr * val2(st);
```

### 复杂度

时间复杂度约为 $O(nS)$，空间复杂度 $O(S)$，其中 $S$ 为压缩状态数。

# 五、单调结构优化线性转移

到了这一步，状态本身已经不难写，真正麻烦的是“前驱很多”。如果某个状态只会从一个滑动窗口、一个前缀区间里转移过来，那就该开始想单调队列、单调栈或窗口最优维护，而不是老老实实枚举所有前驱。

## 21. [1786：01背包](http://ybt.ssoier.cn:8088/problem_show.php?pid=1786)

`单调队列优化` `背包DP`

### 题意

有大量 0/1 物品，物品体积较小但数量极多。要求同时求出背包容量为 $1\sim m$ 时的最大总价值。

### 分析

如果直接把每件物品都做一次 0/1 背包，数量一大就会爆掉。题目的突破口在于体积很小，所以可以按体积分组：同一体积的物品只要按价值从大到小排序，取前若干件时的收益就能预处理出来。

接着对于固定体积 `v` 的一组物品，转移只会发生在同一个余数类里：`dp[j]` 从 `dp[j-kv]` 过来。把这个式子展开，会发现需要在一个滑动窗口中维护 `dp[old] - prefixCntCost` 的最大值，正好可以用单调队列优化。思路重点是先把“很多物品”压成“少量体积分组”。

### 核心代码

```cpp
for (int v = 1; v <= V; v++) {
  for (int r = 0; r < v; r++) {
    deque<int> q;
    for (int k = 0, j = r; j <= m; k++, j += v) {
      while (!q.empty() && q.front() < k - cnt[v]) q.pop_front();
      while (!q.empty() && val(q.back(), k, v) <= val(k, k, v)) q.pop_back();
      q.push_back(k);
      ndp[j] = best(q.front(), k, v);
    }
  }
  copy(ndp, ndp + m + 1, dp);
}
```

### 复杂度

时间复杂度 $O(Vm)$，空间复杂度 $O(m)$。

## 22. [1787：保龄球](http://ybt.ssoier.cn:8088/problem_show.php?pid=1787)

`单调队列优化` `区间DP`

### 题意

一排球瓶有对应得分。每次可以用一个球打掉一段长度不超过 $w$ 的连续区间，共有 $k$ 个球，要求最大化总得分。

### 分析

标准状态是 `dp[t][i]`：前 `i` 个位置里，用了 `t` 个球能得到的最大得分。最后一个球若覆盖区间 `[j+1,i]`，贡献就是 `sum(i)-sum(j)`，于是转移为 `dp[t][i]=max(dp[t][i-1], max(dp[t-1][j]-sum(j))+sum(i))`，其中 `j` 落在一个固定窗口内。

一旦写出这个式子，就能看出 `sum(i)` 对当前 `i` 是常数，真正需要维护的是窗口里的 `dp[t-1][j]-sum(j)` 最大值，所以单调队列直接上。重点不是记模板，而是把“最后一段的区间贡献”拆成前缀和差。

### 核心代码

```cpp
for (int t = 1; t <= k; t++) {
  deque<int> q;
  for (int i = 1; i <= n; i++) {
    while (!q.empty() && q.front() < i - w) q.pop_front();
    while (!q.empty() && pre[t - 1][q.back()] - sum[q.back()] <= pre[t - 1][i - 1] - sum[i - 1]) q.pop_back();
    q.push_back(i - 1);
    pre[t][i] = max(pre[t][i - 1], pre[t - 1][q.front()] + sum[i] - sum[q.front()]);
  }
}
```

### 复杂度

时间复杂度 $O(kn)$，空间复杂度 $O(kn)$ 或 $O(n)$。

## 23. [1788：爬山](http://ybt.ssoier.cn:8088/problem_show.php?pid=1788)

`单调栈` `线性DP`

### 题意

给出一列山峰的高度。从任意一座山开始，每次都会走向相邻方向上“目前能看到的最高峰”所在的一侧，直到抵达全局最高峰。题目要求对每个起点求最终会经过多少座山。

### 分析

如果能先求出每个位置下一步一定走向哪里，整题就退化成 `dp[i]=dp[to[i]]+1` 的简单链式 DP。难点在于快速找这个 `to[i]`。由于“能看到的最高峰”满足典型的可见性单调性质，可以用单调栈预处理每个点左右两边第一个更高、以及由此决定的下一跳。

于是原题其实分成两段：先用单调结构把几何/视线关系压成一个唯一后继，再在这个函数图上做线性 DP。很多看起来像模拟的题，真正突破口都是先把动态过程变成静态后继关系。

### 核心代码

```cpp
for (int i = 1; i <= n; i++) {
  while (!st.empty() && h[st.top()] <= h[i]) st.pop();
  L[i] = st.empty() ? 0 : st.top();
  st.push(i);
}
for (int i = n; i >= 1; i--) {
  while (!st.empty() && h[st.top()] <= h[i]) st.pop();
  R[i] = st.empty() ? 0 : st.top();
  st.push(i);
}
for (int i = ordTopDown; i; i--) dp[i] = dp[to[i]] + 1;
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 24. [1789：分班](http://ybt.ssoier.cn:8088/problem_show.php?pid=1789)

`分段DP` `单调队列优化`

### 题意

学生已经按成绩排好序。现在要把他们分成若干个连续班级，每个班人数必须落在给定区间内，并按题目给出的评价式最小化总成本；若最优值相同，还要进一步比较班级数与最后一班人数。

### 分析

仍然是连续划分，所以状态自然写成 `dp[c][i]`：前 `i` 个学生分成 `c` 个班的最优结果。最后一班的人数在一个固定长度窗口里，于是前驱 `j` 只会落在 `[i-B,i-A]` 之间。

若一段班级的代价能用前缀和在 $O(1)$ 求出，转移就会变成“窗口内某个表达式的最小值 + 当前常数”。因此每个班数层都可以开一个单调队列维护候选前驱，同时把题目要求的二级、三级比较项一起打包成可比较的三元组。这题练的是：**当段长被限制在区间里时，前驱窗口天然适合队列维护。**

### 核心代码

```cpp
for (int c = 1; c <= N; c++) {
  deque<int> q;
  for (int i = 1; i <= M; i++) {
    if (i - A >= 0) push(q, i - A, dp[c - 1][i - A]);
    while (!q.empty() && q.front() < i - B) q.pop_front();
    if (!q.empty())
      dp[c][i] = best(dp[c][i], trans(dp[c - 1][q.front()], q.front() + 1, i));
  }
}
```

### 复杂度

时间复杂度 $O(NM)$，空间复杂度 $O(NM)$ 或 $O(M)$。

## 25. [1790：序列划分](http://ybt.ssoier.cn:8088/problem_show.php?pid=1790)

`二分答案` `单调优化DP`

### 题意

把序列划成若干连续段，既要满足跨段之间的大小关系约束，又要求各段某个代价的最大值尽量小，同时另一项总成本不能超过给定上限。

### 分析

“最小化最大段代价”这句话首先提示我们二分答案。假设已经猜出最大段代价上限 `lim`，问题就变成：能否在每段代价都不超过 `lim` 的前提下完成合法划分，并且附加成本不超标。

检查阶段仍然是线性 DP。对每个右端点，合法前驱会随着位置移动而整体右移，因此可以用单调结构维护当前窗口内最优的 `dp[j] + extra(j)`。也就是说，这题真正的模型是“外层二分，内层做一个窗口型可行性 DP”，把两个层次分清楚就不会乱。

### 核心代码

```cpp
bool check(long long lim) {
  deque<int> q;
  fill(dp, dp + n + 1, INF);
  dp[0] = 0;
  for (int i = 1; i <= n; i++) {
    updateWindow(q, i, lim);
    while (!q.empty() && bad(q.front(), i, lim)) q.pop_front();
    if (!q.empty()) dp[i] = cost(dp[q.front()], q.front() + 1, i);
  }
  return dp[n] <= m;
}
```

### 复杂度

时间复杂度 $O(n\log V)$，空间复杂度 $O(n)$。

# 六、斜率优化与凸包决策

当转移式里开始稳定出现平方项、前缀和平方差、或者“枚举断点 $j$ 与当前点 $i$ 的代价可拆成直线对点查询”时，就该进入斜率优化。最后这一章就是把朴素 DP 再往前推一步：不是改状态，而是把决策集合维护成一条凸包。

## 26. [1791：太空飞船](http://ybt.ssoier.cn:8088/problem_show.php?pid=1791)

`斜率优化DP` `划分DP`

### 题意

环形飞船有 $N$ 个舱室，要切成 $K$ 段。题目给出的目标等价于让各段和尽量均衡，最终要求最小化一个关于段和方差的量。

### 分析

方差展开后，常数部分固定，真正需要最小化的是各段和平方和。把环断开后枚举起点，线性版本里设 `dp[t][i]` 为前 `i` 个舱室分成 `t` 段的最小代价，则转移含有 `dp[t-1][j] + (sum[i]-sum[j])^2`。

把平方展开：`sum[i]^2 - 2sum[i]sum[j] + sum[j]^2`。对当前 `i` 来说，只有 `sum[i]` 是变量，所以每个前驱 `j` 都对应一条斜率为 `-2sum[j]` 的直线，查询的是在 `x=sum[i]` 处的最小值。这就是最标准的凸包 DP 形态。

### 核心代码

```cpp
for (int t = 1; t <= K; t++) {
  deque<int> q;
  q.push_back(0);
  for (int i = 1; i <= n; i++) {
    while (q.size() > 1 && value(t - 1, q[0], sum[i]) >= value(t - 1, q[1], sum[i])) q.pop_front();
    int j = q.front();
    dp[t][i] = dp[t - 1][j] + sqr(sum[i] - sum[j]);
    while (q.size() > 1 && bad(t - 1, q[q.size() - 2], q.back(), i)) q.pop_back();
    q.push_back(i);
  }
}
```

### 复杂度

时间复杂度 $O(KN)$，空间复杂度 $O(KN)$ 或 $O(N)$。

## 27. [1792：小P的牧场](http://ybt.ssoier.cn:8088/problem_show.php?pid=1792)

`斜率优化DP` `前缀和`

### 题意

线性牧场上要建设若干控制站。建站有固定代价，被某个站控制的牧场还会按距离和放养量产生花费，要求最小化总成本。

### 分析

按位置从左到右考虑。若最后一个覆盖当前点的控制站建在 `j`，那么 `j+1..i` 这段的服务成本可以用前缀和展开成“与 `i` 有关的常数 + 一项 `x_i * slope(j) + intercept(j)`”。于是 `dp[i]` 会变成从若干条直线里取最小值。

为什么能做斜率优化？因为前缀坐标是单调的，查询点也是按位置递增的，所以候选断点维护成下凸包即可。识别这类题的关键是：先把一整段的费用彻底前缀和化，把 `j` 和 `i` 分离开。

### 核心代码

```cpp
deque<int> q;
q.push_back(0);
for (int i = 1; i <= n; i++) {
  while (q.size() > 1 && calc(q[0], i) >= calc(q[1], i)) q.pop_front();
  int j = q.front();
  dp[i] = dp[j] + cost(j + 1, i) + build[i];
  while (q.size() > 1 && cross(q[q.size() - 2], q.back(), i)) q.pop_back();
  q.push_back(i);
}
```

### 复杂度

时间复杂度 $O(n)$，空间复杂度 $O(n)$。

## 28. [1793：鏖战字符串](http://ybt.ssoier.cn:8088/problem_show.php?pid=1793)

`斜率优化DP` `前缀DP`

### 题意

字符串每个位置有删除难度。删去一个子串有两种方式：一种代价是该段难度和的二次函数，另一种在线段内出现次数最多的字符个数落在给定区间时生效，代价是一次函数。需要对每个前缀求最小删除时间。

### 分析

设 `dp[i]` 表示删掉前缀 `[1,i]` 的最小代价，最后一段为 `[j+1,i]`。若用第一种删除方式，代价是 `a*(sum[i]-sum[j])^2+b`，这是标准的平方型转移，可以直接拆成斜率优化。于是对每个 `i`，只要在所有前驱 `j` 中找一条最优直线。

第二种删除方式则多了一条可行性限制：子串众数出现次数必须落在 `[l,r]`。这部分不能靠凸包本身解决，但可以在扫右端点时，用频次结构维护哪些左端点对当前 `i` 合法，再把这些合法前驱提供的线性转移值一起取最小。思考顺序应该是先把两种代价拆开：一种靠几何优化，一种靠合法区间维护。

### 核心代码

```cpp
for (int i = 1; i <= n; i++) {
  while (q.size() > 1 && line(q[0], sum[i]) >= line(q[1], sum[i])) q.pop_front();
  dp[i] = line(q.front(), sum[i]) + a * sum[i] * sum[i] + b;
  updateFreqWindow(i);
  if (validL <= validR) dp[i] = min(dp[i], seg.query(validL, validR) + c * sum[i] + d);
  while (q.size() > 1 && bad(q[q.size() - 2], q.back(), i)) q.pop_back();
  q.push_back(i);
  seg.modify(i, dp[i] - c * sum[i]);
}
```

### 复杂度

时间复杂度 $O(n\log n)$ 或 $O(n\Sigma)$，空间复杂度 $O(n)$。

## 29. [1794：分数](http://ybt.ssoier.cn:8088/problem_show.php?pid=1794)

`斜率优化DP` `分段最优`

### 题意

从序列里任选一些位置标记。得分等于“全部被标记的连续子数组个数”减去“被标记数字之和”。每次询问会单独把一个位置改成新值，要求在这个修改下重新求最大得分。

### 分析

如果把一段连续被标记的区间长度记作 `len`，它对第一部分贡献的是 $len(len+1)/2$；再减去这段元素和，就得到一个关于区间左右端点的二次型收益。因此整题就是：选择若干互不相交的连续段，使总收益最大。

把收益用前缀和展开后，枚举最后一段左端点 `j+1` 时，会得到 `dp[j] + A(j) + B(j) * i + const(i)` 的形式，仍然是直线最优决策。大量询问的处理，则是把受修改影响的部分拆出来维护。建模关键在于先把“被标记位置的集合”理解成若干连续段，而不是单点选择。

### 核心代码

```cpp
deque<int> q;
q.push_back(0);
for (int i = 1; i <= n; i++) {
  while (q.size() > 1 && eval(q[0], i) <= eval(q[1], i)) q.pop_front();
  dp[i] = max(dp[i - 1], eval(q.front(), i));
  while (q.size() > 1 && worse(q[q.size() - 2], q.back(), i)) q.pop_back();
  q.push_back(i);
}
answerQuery(pos, x) = rebuildAround(pos, x);
```

### 复杂度

单次线性 DP 的时间复杂度为 $O(n)$，配合询问维护后总复杂度可做到 $O((n+m)\log n)$ 或相近量级，空间复杂度 $O(n)$。
