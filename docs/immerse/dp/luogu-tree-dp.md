# 洛谷 树形DP专题精选解题报告

这一组题从树的直径、重心与树心一路走到换根、树上背包和覆盖类状态，几乎把树形 DP 的主干套路全走了一遍。它们的共同点不在 DFS，而在于你能否把整棵树拆成若干子树答案，再把来自父侧和子侧的贡献严丝合缝地接起来。

# 一、树的经典量：直径、重心与中心

树形 DP 最基础的一类题，是把“整棵树的全局性质”拆成每个点的局部信息，再在一次或两次 DFS 中拼起来。直径关注两条最长向下链，重心关注最大连通块，树心与深度和则常常借助换根完成全局转移。

## 1. [U104609 【模板】树的重心](https://www.luogu.com.cn/problem/U104609)

`DP` `树形DP` `图论` `树形数据结构`

### 题意

给定一个 n 个节点的无根树，节点按照 1∼n 编号，并且无自环和重边。求它的重心，如果有多个重心，按照编号从小到大的顺序依次输出。

### 分析

树的重心不是“离大家最近”的直觉点，而是一个更稳定的量：删掉它以后，剩下所有连通块中最大的那一块尽量小。因为只要最大连通块最小，树就被切得最均衡。

所以关键不在枚举删点后的所有连通块，而在于发现这些连通块其实只有两类：每个儿子子树是一块，剩余的“父亲方向”部分是一块。做一次 DFS 求出每个子树大小后，这些块的大小就都知道了。

于是对每个点 `u`，只要取 `max(各儿子子树大小, n - siz[u])`，就得到了删掉 `u` 后最大的那块。整个题的本质，其实就是把“删点后的连通块大小”翻译成子树大小统计。

所以这题最值得形成的模板感是：**一旦题目问“删掉某点后树被分成什么样”，第一反应就是用子树大小把所有分裂出来的连通块写出来。**

### 核心代码

```cpp
const int N = 200005;
int n, siz[N], best = 1e9, ans;
vector<int> g[N];

void dfs(int u, int fa) {
    siz[u] = 1;
    int mx = 0;
    for (int v : g[u]) {
        if (v == fa) continue;
        dfs(v, u);
        siz[u] += siz[v];
        mx = max(mx, siz[v]);
    }
    mx = max(mx, n - siz[u]);
    if (mx < best) best = mx, ans = u;
}
```

### 复杂度

$O(n)$。

## 2. [P1395 会议](https://www.luogu.com.cn/problem/P1395)

`DP` `树形DP` `树的重心` `福建省历届夏令营`

### 题意

有一个村庄居住着 n 个村民，有 n−1 条路径使得这 n 个村民的家联通，每条路径的长度都为 1。

### 分析

会议点这题虽然故事不同，但核心和树的重心完全一致：你想选一个点，使得把整棵树“拉向它”时最不平衡的那部分尽量小。

因此仍然先看删去某个点后的连通块。对当前点来说，这些连通块只可能来自各个儿子子树，以及父亲方向剩下的那一大块，所以一次 DFS 求子树大小后就能全部刻画出来。

这样每个点的评价值就变成“删掉它以后最大的那块有多大”。选这个值最小的点，就是最平衡的会议点；若题目还要求编号最小，只需在取最优时顺手处理字典序即可。

所以它虽然是应用题，底层想法却非常纯粹：**把“最平衡的点”翻译成“删点后最大连通块最小的点”。**

### 核心代码

```cpp
const int N = 200005;
int n, siz[N], best = 1e9, ans;
vector<int> g[N];

void dfs(int u, int fa) {
    siz[u] = 1;
    int mx = 0;
    for (int v : g[u]) {
        if (v == fa) continue;
        dfs(v, u);
        siz[u] += siz[v];
        mx = max(mx, siz[v]);
    }
    mx = max(mx, n - siz[u]);
    if (mx < best) best = mx, ans = u;
}
```

### 复杂度

$O(n)$。

## 3. [CF685B Kay and Snowflake](https://www.luogu.com.cn/problem/CF685B)

`DP` `树形DP` `深度优先搜索 DFS` `树的重心`

### 题意

当魔镜碎片击中凯的眼睛后，他对玫瑰的美再也不感兴趣了。现在他喜欢观察雪花。 有一天，他发现了一片巨大的雪花，这片雪花的结构是一棵有 n 个结点的树（即连通无环图）。

### 分析

这题比普通重心模板多走了一步：不是求整棵树的重心，而是对每个节点都要求“它那棵子树的重心”。如果每次都重新做一次删点统计，显然会太慢。

真正的突破口在于：一棵子树的重心不会乱跳，它只可能沿着这棵子树的重儿子链移动。因为只有最大的那块可能导致失衡，而往重儿子方向走，才有机会继续减小“最大连通块”。

于是做法就变成：先 DFS 求出每棵子树大小和重儿子，再把重儿子的重心当作候选答案向上调整。若这个候选点在当前子树里仍然让某一块超过一半，就继续往父亲跳；否则它就是当前子树的重心。

所以这题最值得学的是一个很强的结构性质：**对子树重心这种局部最优点，答案常常沿某条重路径单调移动，而不是在整棵子树里乱找。**

### 核心代码

```cpp
int siz[N], son[N], cen[N];
vector<int> g[N];

void dfs(int u) {
    siz[u] = 1, cen[u] = u;
    for (int v : g[u]) {
        dfs(v);
        siz[u] += siz[v];
        if (siz[v] > siz[son[u]]) son[u] = v;
    }
    int x = cen[son[u]];
    while (x && max(siz[son[x]], siz[u] - siz[x]) * 2 > siz[u]) x = fa[x];
    cen[u] = x ? x : u;
}
```

### 复杂度

$O(n)$ 预处理，单点查询 $O(1)$。

## 4. [U81904 【模板】树的直径](https://www.luogu.com.cn/problem/U81904)

`DP` `树形DP`

### 题意

给定一棵树，树中每条边都有一个权值， 树中两点之间的距离定义为连接两点的路径边权之和。

### 分析

树上最长路这类题，先不要想着“枚举两个端点”，而要先问：一条经过 `u` 的最优路径由什么组成？答案很自然——如果路径经过 `u`，它一定是从某个儿子子树里往上走到 `u`，再拐到另一个儿子子树去，所以本质上由两条“从 `u` 向下走的最长链”拼出来。

于是定义 `down[u]` 表示从 `u` 出发、只往子树里走时能到达的最长距离。做后序 DFS 时，每处理完一个儿子 `v`，就已经知道了“经过 `v` 往下走”的最优长度 `down[v] + w`。

这时要做两件事：一是尝试用“当前儿子提供的链 + 之前见过的最好链”更新全局答案；二是顺手更新 `down[u]`。也就是说，直径并不是额外再做一次复杂合并，而是在维护最长向下链的过程中顺便被枚举到了。

所以这题最该形成的直觉是：**树上最长路径问题，往往先维护每个点向下的最优链，再看能否在某个点把两条链拼成全局答案。**

### 核心代码

```cpp
long long down[N], ans;
vector<pair<int,int>> g[N];

void dfs(int u, int fa) {
    for (auto [v, w] : g[u]) {
        if (v == fa) continue;
        dfs(v, u);
        ans = max(ans, down[u] + down[v] + w);
        down[u] = max(down[u], down[v] + w);
    }
}
```

### 复杂度

$O(n)$。

## 5. [B4016 树的直径](https://www.luogu.com.cn/problem/B4016)

`DP` `树形DP` `深度优先搜索 DFS` `树形 DP`

### 题意

给定一棵 n 个结点的树，树没有边权。请求出树的直径是多少，即树上最长的不重复经过一个点的路径长度是多少。

### 分析

和带权模板题完全同构，只是这里每条边权都等于 `1`。因此真正该看的仍然不是“哪两个点最远”，而是“对每个点来说，往下延伸的最长链和次长链分别是多少”。

若直径经过当前点，那么它一定由两条向下链首尾相接组成；若不经过当前点，那它已经在某个儿子子树内部被算出来了。于是一次 DFS 就够：从孩子那里拿到各自的向下深度，更新本点最好的两条链，并尝试刷新全局答案。

这类题的关键不是记住“求直径”三个字，而是记住一种树形 DP 模式：**全局最优路径 = 某点处两条局部最优向下路径的拼接**。以后遇到很多“树上最长链 / 最大路径和”题，第一反应都可以从这里出发。

### 核心代码

```cpp
long long down[N], ans;
vector<pair<int,int>> g[N];

void dfs(int u, int fa) {
    for (auto [v, w] : g[u]) {
        if (v == fa) continue;
        dfs(v, u);
        ans = max(ans, down[u] + down[v] + w);
        down[u] = max(down[u], down[v] + w);
    }
}
```

### 复杂度

$O(n)$。

## 6. [U392706 【模板】树的中心](https://www.luogu.com.cn/problem/U392706)

`DP` `树形DP`

### 题意

给定一棵 n 个节点的树，求树的所有中心。

### 分析

树心和直径不一样，它关注的不是“最长路径有多长”，而是“站在这个点时，离最远点还能有多远”。也就是说，每个点真正的评价值是它的离心率：从它出发到全树最远点的距离。

这个最远点可能在自己子树里，也可能在父亲方向，所以只算向下信息还不够。第一遍 DFS 求出每个点子树中的最长链 `d1[u]` 和次长链 `d2[u]`，相当于回答“最远点如果在我下面，会有多远”；第二遍 DFS 再把来自父侧的最优距离 `up[u]` 传下来，补上“最远点如果不在我子树里”的情况。

对子节点 `v` 来说，父侧给它的最好向下链不能直接拿 `d1[u]`，因为如果 `d1[u]` 正好来自 `v` 自己，就等于把同一条子树链重复用了，所以要在 `d1[u]` 和 `d2[u]` 之间做排除。这正是树心题最容易写错的地方。

最后每个点的最远距离就是 `max(d1[u], up[u])`，取这个值最小的点即可。所以这题真正练的是：**当答案同时受子侧和父侧影响时，要用两遍 DFS 把“向下信息”和“换根传下来的父侧信息”拼起来。**

### 核心代码

```cpp
long long d1[N], d2[N], up[N];
int son[N];

void dfs1(int u, int fa) {
    for (auto [v, w] : g[u]) {
        if (v == fa) continue;
        dfs1(v, u);
        long long t = d1[v] + w;
        if (t > d1[u]) d2[u] = d1[u], d1[u] = t, son[u] = v;
        else if (t > d2[u]) d2[u] = t;
    }
}

void dfs2(int u, int fa) {
    for (auto [v, w] : g[u]) {
        if (v == fa) continue;
        up[v] = max(up[u], son[u] == v ? d2[u] : d1[u]) + w;
        dfs2(v, u);
    }
}
```

### 复杂度

$O(n)$。

## 7. [U261056 树的中心](https://www.luogu.com.cn/problem/U261056)

`DP` `树形DP`

### 题意

给定一棵树，树中有n个结点（结点编号为1~n），请求出该树的中心结点的编号。树的中心指的是，该结点离树中的其他结点，最远距离最近。

### 分析

这题与树心模板完全同源，但更适合作为“为什么要记次大值”的练习题。因为你想把父亲方向的最优链传给儿子时，最容易犯的错就是把“本来就来自这个儿子”的最长链又传回给它。

所以第一遍 DFS 不只是求一个最大值，而是要同时维护最长和次长两条向下链。这样当我们处理儿子 `v` 时，如果 `u` 的最长链正是从 `v` 这棵子树贡献的，就必须改用次长链去代表“除了你之外，父亲这一侧还能给你的最好支持”。

第二遍 DFS 的意义，也因此不只是“换根”，而是把“来自父亲和兄弟子树的最优远端信息”准确地下传。等这一步想清楚后，树心的状态其实一点都不复杂，难的是信息去重。

所以这题最值得记住的是一句话：**父亲给儿子的补充信息，必须排除儿子自己已经提供过的贡献。** 这在换根 DP 里是反复出现的核心细节。

### 核心代码

```cpp
long long d1[N], d2[N], up[N];
int son[N];

void dfs1(int u, int fa) {
    for (auto [v, w] : g[u]) {
        if (v == fa) continue;
        dfs1(v, u);
        long long t = d1[v] + w;
        if (t > d1[u]) d2[u] = d1[u], d1[u] = t, son[u] = v;
        else if (t > d2[u]) d2[u] = t;
    }
}

void dfs2(int u, int fa) {
    for (auto [v, w] : g[u]) {
        if (v == fa) continue;
        up[v] = max(up[u], son[u] == v ? d2[u] : d1[u]) + w;
        dfs2(v, u);
    }
}
```

### 复杂度

$O(n)$。

# 二、换根 DP 与全局贡献转移

这一组题的共同点，是“以某点为根”的答案可以从父亲向儿子递推得到。先在子树内做一次聚合，再把父侧贡献二次下传，就能把每个点都当作根来计算答案。

## 8. [AT_dp_v Subtree](https://www.luogu.com.cn/problem/AT_dp_v)

`DP` `树形 DP`

### 题意

有一棵包含 N 个顶点的树。顶点编号为 1,2,…,N。对于每个 i（1≤i≤N−1），第 i 条边连接顶点 x i ​ 和 y i ​ 。

### 分析

设 $down[u]$ 表示只看 $u$ 子树、且必须选到 $u$ 的合法方案数。换根时利用前缀积和后缀积，把“去掉某个儿子后的其余贡献”在线性时间内传给它。

### 核心代码

```cpp
long long down[N], up[N], pre[N], suf[N];

void dfs1(int u, int fa) {
    down[u] = 1;
    for (int v : g[u]) if (v != fa) {
        dfs1(v, u);
        down[u] = down[u] * (down[v] + 1) % mod;
    }
}

void dfs2(int u, int fa) {
    int m = g[u].size();
    pre[0] = suf[m + 1] = 1;
    for (int i = 1; i <= m; i++) {
        int v = g[u][i - 1];
        long long cur = (v == fa ? up[u] : down[v]) + 1;
        pre[i] = pre[i - 1] * cur % mod;
    }
    for (int i = m; i >= 1; i--) {
        int v = g[u][i - 1];
        long long cur = (v == fa ? up[u] : down[v]) + 1;
        suf[i] = suf[i + 1] * cur % mod;
    }
    for (int i = 1; i <= m; i++) {
        int v = g[u][i - 1];
        if (v == fa) continue;
        up[v] = pre[i - 1] * suf[i + 1] % mod;
        dfs2(v, u);
    }
}
```

### 复杂度

$O(n)$。

## 9. [CF1187E Tree Painting](https://www.luogu.com.cn/problem/CF1187E)

`DP` `动态规划 DP` `树形 DP`

### 题意

给定一棵包含 n 个顶点的树（无向连通无环图）。你要在这棵树上玩一个游戏。 最开始所有顶点都是白色的。

### 分析

第一次 DFS 计算以 1 为根时的总得分和子树大小；第二次 DFS 令根从父亲移到儿子，答案变化量正好是 $n-2\cdot siz[v]$。这是换根 DP 的经典式子。

### 核心代码

```cpp
long long siz[N], f[N];

void dfs1(int u, int fa, int dep) {
    siz[u] = 1;
    f[1] += dep;
    for (int v : g[u]) if (v != fa) {
        dfs1(v, u, dep + 1);
        siz[u] += siz[v];
    }
}

void dfs2(int u, int fa) {
    for (int v : g[u]) if (v != fa) {
        f[v] = f[u] + n - 2 * siz[v];
        dfs2(v, u);
    }
}
```

### 复杂度

$O(n)$。

## 10. [CF1324F Maximum White Subtree](https://www.luogu.com.cn/problem/CF1324F)

`DP` `深度优先搜索 DFS` `树形 DP`

### 题意

给定一棵包含 n 个顶点的树。树是一个包含 n−1 条边的连通无向图。树上的每个顶点 v 都被赋予了一种颜色（如果顶点 v 是白色，则 a v ​ =1，如果顶点 v 是黑色，则 a v ​ =0）。

### 分析

把白点看作 $+1$、黑点看作 $-1$。先在子树内保留所有正贡献，再把父侧的正贡献向下传，最终每个点都能得到“必须包含它”的最优连通子树值。

### 核心代码

```cpp
long long down[N], up[N], val[N];

void dfs1(int u, int fa) {
    down[u] = val[u];
    for (int v : g[u]) if (v != fa) {
        dfs1(v, u);
        down[u] += max(0LL, down[v]);
    }
}

void dfs2(int u, int fa) {
    for (int v : g[u]) if (v != fa) {
        up[v] = max(0LL, up[u] + down[u] - max(0LL, down[v]));
        dfs2(v, u);
    }
}
```

### 复杂度

$O(n)$。

## 11. [P2986 [USACO10MAR] Great Cow Gathering G](https://www.luogu.com.cn/problem/P2986)

`DP` `动态规划 DP` `树形 DP` `USACO`

### 题意

Bessie 正在计划一年一度的奶牛大集会，来自全国各地的奶牛将来参加这一次集会。

### 分析

先固定一个根求出“所有奶牛到根的总代价”，同时维护每个子树中的奶牛数。换根到儿子时，只需把这条边两侧的牛群整体平移即可完成转移。

### 核心代码

```cpp
long long siz[N], w[N], f[N], tot;

void dfs1(int u, int fa, long long dep) {
    siz[u] = w[u];
    f[1] += dep * w[u];
    for (auto [v, c] : g[u]) if (v != fa) {
        dfs1(v, u, dep + c);
        siz[u] += siz[v];
    }
}

void dfs2(int u, int fa) {
    for (auto [v, c] : g[u]) if (v != fa) {
        f[v] = f[u] + (tot - 2 * siz[v]) * c;
        dfs2(v, u);
    }
}
```

### 复杂度

$O(n)$。

## 12. [P3478 [POI 2008] STA-Station](https://www.luogu.com.cn/problem/P3478)

`DP` `动态规划 DP` `树形数据结构` `树形 DP`

### 题意

给定一个 n 个点的树，请求出一个结点，使得以这个结点为根时，所有结点的深度之和最大。 一个结点的深度之定义为该节点到根的简单路径上边的数量。

### 分析

深度和最大值本质也是换根：根从 $u$ 移到 $v$ 时，$v$ 子树内点深度减一，其余点深度加一，所以答案变化仍是 $n-2\cdot siz[v]$。

### 核心代码

```cpp
long long siz[N], f[N];

void dfs1(int u, int fa, int dep) {
    siz[u] = 1;
    f[1] += dep;
    for (int v : g[u]) if (v != fa) {
        dfs1(v, u, dep + 1);
        siz[u] += siz[v];
    }
}

void dfs2(int u, int fa) {
    for (int v : g[u]) if (v != fa) {
        f[v] = f[u] + n - 2 * siz[v];
        dfs2(v, u);
    }
}
```

$O(n)$。

for (int u = 1; u <= n; u++) f[u][0] = cow[u];

    for (int u = 1; u <= n; u++)

## 16. [P10974 Accumulation Degree](https://www.luogu.com.cn/problem/P10974)

`DP` `树形DP` `动态规划 DP` `树形 DP`

### 题意

树在许多世界神话中也扮演着亲密的角色。许多学者对树的一些特殊属性感兴趣，例如树的中心、树的计数、树的着色等。

### 分析

这题最难的地方不是状态多，而是看懂每条边到底在做什么决策。对于父子边 `(u,v)` 来说，若从 `u` 方向把信息送进 `v` 的子树，那么这条边要么真的走进去继续覆盖子树内部，要么干脆把代价截在这条边上，不再深入。

因此对子树做 DFS 时，儿子 `v` 对父亲 `u` 的贡献自然就是 `min(w, down[v])`：要么只付这条边本身的代价，要么付子树内部已经算好的累积代价，取更优的那个。这就是 `down[u]` 的来源。

但仅有子侧信息还不够，因为把根换到别处后，某个点的答案还会受到父亲方向贡献的影响，所以还要再做一遍换根，把“除了这棵子树以外，外面世界能给你的最优贡献”传给儿子。

所以这题本质上是在做一类很常见的树形 DP：**边的贡献可以在局部截断，也可以继续向下展开；先自底向上求子树最优，再自顶向下补齐父侧信息。**

### 核心代码

```cpp
long long down[N], up[N];

void dfs1(int u, int fa) {
    for (auto [v, w] : g[u]) if (v != fa) {
        dfs1(v, u);
        down[u] += min((long long)w, down[v]);
    }
}

void dfs2(int u, int fa) {
    for (auto [v, w] : g[u]) if (v != fa) {
        up[v] = down[v] + min((long long)w, up[u] + down[u] - min((long long)w, down[v]));
        dfs2(v, u);
    }
}
```

### 复杂度

$O(n)$。

## 20. [P1131 [ZJOI2007] 时态同步](https://www.luogu.com.cn/problem/P1131)

`DP` `树形DP` `动态规划 DP` `树形 DP`

### 题意

小 Q 在电子工艺实习课上学习焊接电路板。一块电路板由若干个元件组成，我们不妨称之为节点，并将其用数字 1,2,3⋯ 进行标号。

### 分析

题目表面在说“同步”，实质上是在问：对每个分叉点，若想让它下面所有叶子最终同时完成，应该对哪些较短分支补时间。

所以先从子树角度想。设 `f[v] + w` 表示从当前点 `u` 沿儿子 `v` 走下去，到该子树中最晚完成叶子的时间。那么若想让 `u` 的所有儿子分支同步，目标时间一定只能取这些值里的最大者 `mx`，因为你不可能让最长那条变短，只能把短的补齐。

于是转移就非常自然：第一遍先找出所有儿子分支中的最大完成时间 `mx`；第二遍把每个儿子离 `mx` 还差多少全部加进答案里，这些差值就是必须额外补的长度。与此同时，`u` 自己向父亲汇报的最晚完成时间就是这个 `mx`。

所以这题值得迁移的思路是：**当一棵树的多个分支要“对齐”到同一完成时刻时，往往先找最长分支，再把其余分支补到同一高度。**

### 核心代码

```cpp
long long f[N];

void dfs(int u, int fa) {
    long long mx = 0;
    for (auto [v, w] : g[u]) if (v != fa) {
        dfs(v, u);
        mx = max(mx, f[v] + w);
    }
    for (auto [v, w] : g[u]) if (v != fa)
        f[u] += mx - (f[v] + w);
}
```

### 复杂度

$O(n)$。

## 25. [P1270 “访问”美术馆](https://www.luogu.com.cn/problem/P1270)

`背包` `动态规划 DP` `搜索` `树形数据结构`

### 题意

经过数月的精心准备，Peer Brelstet，一个出了名的盗画者，准备开始他的下一个行动。

### 分析

这题虽然包装成“偷画路线规划”，但真正结构非常规整：每个分叉点只会分成左右两个分支，所以整张图天然就是一棵二叉决策树。

时间限制则直接扮演背包容量。设 `dp[u][t]` 表示在 `u` 这棵子树中花费不超过 `t` 时间时最多能拿到多少价值，那么左右子树就是两组必须共享总时间预算的决策来源。

因此转移本质就是二叉树背包：先递归求出左右子树各自的“时间 - 收益”表，再枚举给左边多少时间、给右边多少时间，把两边的最优收益拼起来。叶子节点则是最简单的物品型状态：时间够就能拿到对应画作。

所以这题最值得迁移的是：**当搜索空间本身就是一棵二叉结构，而资源上限又是总时间 / 总步数时，常常可以把 DFS 路径选择改写成二叉树背包。**

### 核心代码

```cpp
void dfs(int u) {
    if (leaf[u]) {
        for (int t = cost[u]; t <= T; t++) dp[u][t] = value[u];
        return;
    }
    dfs(ls[u]), dfs(rs[u]);
    for (int t = 0; t <= T; t++)
        for (int k = 0; k <= t; k++)
            dp[u][t] = max(dp[u][t], dp[ls[u]][k] + dp[rs[u]][t - k]);
}
```

### 复杂度

$O(nT^2)$。

## 26. [P3360 偷天换日](https://www.luogu.com.cn/problem/P3360)

`背包` `动态规划 DP` `背包 DP` `树形 DP`

`背包` `动态规划 DP` `树形数据结构` `递归`

### 题意

某收费有线电视网计划转播一场重要的足球比赛。他们的转播网和用户终端构成一棵树状结构，这棵树的根结点位于足球比赛的现场，树叶为各个用户终端，其他中转站为该树的内部节点。

### 分析

这题看似是网络建设，实质上还是一棵树上的资源分配。每个叶子用户都可能带来收益，而经过一条边去连接某棵子树则要支付代价，所以父节点面对的是一个很典型的问题：要不要接这棵子树进来，接进来后准备服务多少个叶子。

因此状态自然落在“服务了多少用户”上。设 `dp[u][j]` 表示在 `u` 的子树中恰好服务 `j` 个用户时的最大利润，那么每个儿子子树都会提供一整列选择：服务它那边 `0/1/2/...` 个用户分别值多少。

合并儿子时之所以要减去边权 `w`，是因为只要你决定从 `u` 把这棵儿子子树接进方案里，就必须先付出连接这条支路的成本。只有叶子那边的收益足以覆盖这些连接代价，整条分支才值得保留。

所以这题和树上背包的很多经典题一样，本质都在做一件事：**子树提供收益表，父节点决定哪些子树值得接入，并在总服务数量维度上做分配。**

### 核心代码

```cpp
void dfs(int u) {
    dp[u][0] = 0;
    if (leaf[u]) dp[u][1] = pay[u];
    for (auto [v, w] : g[u]) {
        dfs(v);
        for (int i = siz[u]; i >= 0; i--)
            for (int j = 1; j <= siz[v]; j++)
                dp[u][i + j] = max(dp[u][i + j], dp[u][i] + dp[v][j] - w);
        siz[u] += siz[v];
    }
}
```

### 复杂度

$O(n^2)$。

## 29. [P2015 二叉苹果树](https://www.luogu.com.cn/problem/P2015)

`背包` `动态规划 DP` `树形数据结构` `背包 DP`

### 题意

有一棵苹果树，如果树枝有分叉，一定是分二叉（就是说没有只有一个儿子的结点） 这棵树共有 N 个结点（叶子点或者树枝分叉点），编号为 1∼N，树根编号一定是 1。

### 分析

这题看似是在“选边摘苹果”，但真正的结构是：保留的边必须连成一棵从根出发的连通子树。也就是说，某条边想被保留，它上面的连接关系就必须已经成立，所以这不是普通背包，而是带连通性约束的树上背包。

于是状态自然落在“这棵子树里准备保留多少条边”上。设 `dp[u][i]` 表示在 `u` 的子树内保留 `i` 条边所能得到的最大苹果数，那么每个儿子 `v` 都提供一列选择：给它分多少条边，能换来多少收益。

合并时之所以有 `dp[u][i - j - 1] + dp[v][j] + w`，正是因为如果你想从 `u` 连接到 `v` 的子树去拿那边的苹果，就必须额外花掉 `1` 条边去保留连接边 `(u,v)`，然后才能在 `v` 子树里继续分配 `j` 条边。

所以这题最该学会的是树上背包里非常典型的一点：**子树资源不是直接白拿的，很多题都要先付出“把这棵子树接进来”的连接成本。**

### 核心代码

```cpp
void dfs(int u, int fa) {
    for (auto [v, w] : g[u]) if (v != fa) {
        dfs(v, u);
        for (int i = q; i >= 1; i--)
            for (int j = 0; j < i; j++)
                dp[u][i] = max(dp[u][i], dp[u][i - j - 1] + dp[v][j] + w);
    }
}
```

### 复杂度

$O(nq^2)$。

## 30. [P3177 [HAOI2015] 树上染色](https://www.luogu.com.cn/problem/P3177)

`DP` `树形DP` `动态规划 DP` `树形 DP`

### 题意

有一棵点数为 n 的树，树边有边权。给你一个在 0∼n 之内的正整数 k ，你要在这棵树中选择 k 个点，将其染成黑色，并将其他的 n−k 个点染成白色。

### 分析

这题最容易卡住的地方，是不知道边权到底什么时候结算。其实对一条边 `(u,v)` 来说，一旦确定 `v` 子树里染黑了多少个点，这条边对答案的贡献就已经完全确定了，因为跨过这条边的黑白点对数量都能直接算出来。

这说明状态不需要记录复杂结构，只要记录“某棵子树里选了多少个黑点”。设 `dp[u][k]` 表示在 `u` 子树中恰好染黑 `k` 个点时的最大贡献，那么合并儿子时，除了做普通的名额分配，还要顺手把边 `(u,v)` 的跨边贡献加进去。

为什么只知道数量就够？因为题目里的价值来自点对路径经过边的次数，而一条边被多少黑点和多少白点跨越，只取决于它两侧分别有多少黑白点，不取决于这些点的具体位置。这正是这题能压成树上背包的核心原因。

所以这题的迁移提醒是：**若树边贡献只由子树两侧“选了多少个”决定，而不关心具体选了谁，就优先考虑“按数量计数”的树上背包。**

### 核心代码

```cpp
void dfs(int u, int fa) {
    siz[u] = 1;
    for (auto [v, w] : g[u]) if (v != fa) {
        dfs(v, u);
        for (int i = min(siz[u], k); i >= 0; i--)
            for (int j = 1; i + j <= k; j++)
                dp[u][i + j] = max(dp[u][i + j], dp[u][i] + dp[v][j] + 1LL * j * (k - j) * w);
        siz[u] += siz[v];
    }
}
```

### 复杂度

$O(nk^2)$。

## 31. [P8677 [蓝桥杯 2018 国 A] 采油](https://www.luogu.com.cn/problem/P8677)

`DP` `树形DP` `树形 DP` `蓝桥杯国赛`

### 题意

LQ 公司是世界著名的石油公司，为世界供应优质石油。 最近，LQ 公司又在森林里发现了一大片区域的油田，可以在这个油田中开采 n 个油井。

### 分析

采油问题本质上不是“在树上走”，而是“预算固定时，怎么在若干子树之间分配名额”。一旦题目要求从一棵树的不同分支里挑若干个对象，而且每个分支内部还有自己的最优收益，基本就该往树上背包想。

设 `dp[u][k]` 表示在 `u` 的子树中分配 `k` 个名额时的最优收益，那么每个儿子 `v` 实际上都向父亲提供了一整列选择：给它 `0` 个、`1` 个、`2` 个……分别能换来多少收益。父节点要做的，就是把这些“子树收益表”一个一个并进来。

这就是为什么转移会写成双层枚举：`i` 表示当前已经在前面若干儿子里分掉了多少名额，`j` 表示准备再分给新儿子多少。看起来像背包，其实也确实就是背包，只不过物品不是一维物品，而是整棵子树打包后的“分配曲线”。

所以这题的迁移结论非常直接：**树上背包的核心不是树，而是“每个儿子都提供一组可选收益，父亲要在总容量限制下做资源分配”。**

### 核心代码

```cpp
void dfs(int u, int fa) {
    siz[u] = 1;
    dp[u][0] = 0;
    for (int v : g[u]) {
        if (v == fa) continue;
        dfs(v, u);
        for (int i = siz[u]; i >= 0; i--)
            for (int j = 1; j <= siz[v]; j++)
                dp[u][i + j] = max(dp[u][i + j], dp[u][i] + dp[v][j]);
        siz[u] += siz[v];
    }
}
```

### 复杂度

$O(nm^2)$。

在大学里每个学生，为了达到一定的学分，必须从很多课程里选择一些课程来学习，在课程里有些课程必须在某些课程之前学习，如高等数学总是在其它课程之前学习。

# 四、覆盖、染色与局部状态设计

这类题往往不是单纯求和，而是要设计“选或不选、被覆盖或未覆盖、能否向父亲转移”的有限状态。状态虽小，但合法性约束强，转移时一定要先想清父子之间的信息边界。

## 33. [P2899 [USACO08JAN] Cell Phone Network G](https://www.luogu.com.cn/problem/P2899)

`DP` `树形DP` `排序` `树形 DP`

### 题意

Farmer John 想让他的所有牛用上手机以便相互交流。他需要建立几座信号塔在 N 块草地中。

### 分析

这类“用最少点覆盖整棵树”的题，最难的从来不是转移式，而是先把状态语义想准。因为对一个点来说，它可能自己放站，也可能不放但已经被下面某个儿子覆盖，还可能当前没人管、只能等父亲来覆盖。

所以三态的价值就在这里：`本点放站 / 本点已被子树覆盖 / 本点等待父亲覆盖`。一旦把这三个身份分清，儿子该取哪些状态就变得很机械了：若本点放站，儿子怎么合法都行；若本点等父亲覆盖，儿子就不能也等自己往上甩锅；若本点要被儿子覆盖，则至少得有一个儿子真的放站。

这也是这题比树上点覆盖稍微难一点的地方：并不是简单地对每个儿子独立取最优，而是“本点被儿子覆盖”这个状态还带着一个整体约束——必须存在至少一个负责覆盖它的儿子。

所以这题最该记住的是：**树上支配集类问题，先把“我覆盖谁、谁覆盖我、我是否还欠父亲一次覆盖”这些语义列清，再写状态。**

### 核心代码

```cpp
const int INF = 1e9;
void dfs(int u, int fa) {
    dp[u][0] = 1; dp[u][1] = dp[u][2] = 0;
    for (int v : g[u]) if (v != fa) {
        dfs(v, u);
        ndp0 = min({dp[v][0], dp[v][1], dp[v][2]});
        ndp1 = min(dp[v][0], dp[v][2]);
        ndp2 = dp[v][1];
    }
}
```

### 复杂度

$O(n)$。

## 34. [P2458 [SDOI2006] 保安站岗](https://www.luogu.com.cn/problem/P2458)

`DP` `树形DP` `动态规划 DP` `树形数据结构`

### 题意

五一来临，某地下超市为了便于疏通和指挥密集的人员和车辆，以免造成超市内的混乱和拥挤，准备临时从外单位调用部分保安来维持交通秩序。

### 分析

保安站岗和上一题同属树上最小支配集，只不过这里每个点放人的代价不同，所以不能再只数个数，而要真的做最小费用。

状态设计仍然是那三个经典身份：本点自己放保安、本点已经被儿子覆盖、本点暂时没人管只能等父亲覆盖。难点集中在第二种状态——“被儿子覆盖”并不是随便把儿子最优值加起来就完了，因为你必须保证至少有一个儿子真的放了保安来照顾当前点。

因此常见写法会先把“默认所有儿子各自最优地合法”这一层合出来，再单独枚举哪个儿子承担“覆盖父亲”的职责，用一次替换把这个全局约束补上。代码里 `f[u][1]` 的那轮枚举，本质上就在做这件事。

所以这类题最值得迁移的不是三态名字，而是一种处理方式：**先合并局部最优，再为“至少一个儿子承担特殊职责”补一层额外枚举。**

### 核心代码

```cpp
const long long INF = 4e18;
void dfs(int u, int fa) {
    f[u][0] = cost[u];
    f[u][1] = 0;
    f[u][2] = 0;
    for (int v : g[u]) if (v != fa) {
        dfs(v, u);
        f[u][0] += min({f[v][0], f[v][1], f[v][2]});
        f[u][2] += min(f[v][0], f[v][1]);
    }
    f[u][1] = INF;
    for (int v : g[u]) if (v != fa)
        f[u][1] = min(f[u][1], f[u][2] - min(f[v][0], f[v][1]) + f[v][0]);
}
```

### 复杂度

$O(n)$。

## 35. [P2016 [SEERC 2000] 战略游戏](https://www.luogu.com.cn/problem/P2016)

`DP` `树形DP` `动态规划 DP` `贪心`

### 题意

他要建立一个古城堡，城堡中的路形成一棵无根树。他要在这棵树的结点上放置最少数目的士兵，使得这些士兵能瞭望到所有的路。

### 分析

这题约束只落在边上：每条边至少要有一个端点放士兵。看到这种描述，就应该先把原题翻译成图论语言——这就是树上的最小点覆盖。

一旦目标变成“覆盖所有边”，状态就非常干净。设 `dp[u][1]` 表示选 `u` 时，覆盖 `u` 子树所有边的最小代价；`dp[u][0]` 表示不选 `u` 时的最小代价。

为什么转移会这么简单？因为边只连接父子。若当前点不选，那么和它相连的每条子边都只能靠儿子去覆盖，所以所有儿子都必须选；若当前点选了，这条父子边已经被覆盖，儿子就可以各自独立决定选不选。

所以这题和“没有上司的舞会”正好形成一对对照：一个是边覆盖，一个是点互斥。它们都只受父子边约束，因此固定父亲状态后，儿子子树就能独立合并。

### 核心代码

```cpp
void dfs(int u, int fa) {
    dp[u][1] = 1;
    for (int v : g[u]) if (v != fa) {
        dfs(v, u);
        dp[u][0] += dp[v][1];
        dp[u][1] += min(dp[v][0], dp[v][1]);
    }
}
```

$O(n)$。

void dfs(int u, int fa) {

## 37. [P1352 没有上司的舞会](https://www.luogu.com.cn/problem/P1352)

`DP` `树形DP` `动态规划 DP` `树形 DP`

### 题意

某大学有 n 个职员，编号为 1…n。 他们之间有从属关系，也就是说他们的关系就像一棵以校长为根的树，父结点就是子结点的直接上司。

### 分析

这题是树上最大独立集的标准原型。真正要抓住的，不是“舞会”这个故事，而是限制只有一条：如果当前员工参加，那么所有直接下属都不能参加。

一旦约束只发生在父子之间，状态就会非常自然。设 `dp[u][1]` 表示选 `u` 时，这棵子树能得到的最大快乐值；`dp[u][0]` 表示不选 `u` 时的最大快乐值。

为什么这两个状态就够了？因为当父亲的选或不选确定后，各个儿子子树之间就彼此独立了。若选 `u`，孩子全部只能走“不选”状态；若不选 `u`，每个孩子都可以在“选”和“不选”里各取最优。

所以这题最值得迁移的是树形 DP 里最常见的一种切分方式：**只要约束只压在父子边上，固定父节点状态后，各儿子子树通常就能独立求最优。**

### 核心代码

```cpp
void dfs(int u) {
    dp[u][1] = val[u];
    for (int v : son[u]) {
        dfs(v);
        dp[u][0] += max(dp[v][0], dp[v][1]);
        dp[u][1] += dp[v][0];
    }
}
```

### 复杂度

$O(n)$。

## 39. [P3554 [POI 2013] LUK-Triumphal arch](https://www.luogu.com.cn/problem/P3554)

`DP` `树形DP` `二分` `动态规划 DP`

### 题意

给一颗 n 个节点的树，初始时 1 号节点被染黑，其余是白的。两个人轮流操作，一开始 B 在 1 号节点。

### 分析

这题表面上是博弈，但解法核心仍然是“二分某个阈值，然后做树上判定”。真正难的不是二分本身，而是想清楚 `check(lim)` 到底在判什么：限制每个点向父亲方向最多还残留多少个“没解决的需求”，看整棵树能否在该阈值内被消化。

后序遍历时，最优策略一定是让子树内部先尽量自给自足，只把实在消化不了的信息往父亲上传。因为一旦某个问题能在更低层解决，就没必要拖到更高层再付代价。

所以 DFS 的状态通常不是复杂方案数，而是“这棵子树向上还要带多少麻烦”。当某个儿子上传的信息已经逼近阈值，就意味着再往上拖会超限，必须在当前层做一次处理；否则就继续把它合并进父节点状态。

这类题最值得记住的信号是：**答案看起来像最小最大值，且子树里的需求能局部消化，就很适合二分阈值后做树上贪心判定。**

### 核心代码

```cpp
bool check(int lim) {
    dfs(1, 0);
    return need[1] <= m;
}

void dfs(int u, int fa) {
    for (int v : g[u]) if (v != fa) {
        dfs(v, u);
        if (dis[v] == lim - 1) need[u]++, dis[u] = max(dis[u], -1);
        else dis[u] = max(dis[u], dis[v] + 1);
    }
}
```

### 复杂度

$O(n\log n)$。

## 40. [CF767C Garland](https://www.luogu.com.cn/problem/CF767C)

`DP` `树形DP` `动态规划 DP` `剪枝`

### 题意

新年时，Dima 做了一个梦，梦中他收到了一串神奇的彩灯。这串彩灯由若干个灯泡组成，其中一些灯泡两两之间由导线直接相连。

### 分析

这题最关键的观察是：如果最后要切成三块且三块权值和相等，那么每块的目标和就已经被总和唯一决定了。若总权值不能被 `3` 整除，当然直接无解。

一旦目标和固定为 `target`，问题就变成：能不能找到两棵互不包含、且都不是整棵树本身的子树，它们的子树和都等于 `target`。因为只要切掉这两棵子树与父亲相连的边，剩下那一大块的和也会自动等于 `target`。

所以 DFS 求子树和时，一旦发现某棵子树正好达到 `target`，就可以把它视作已经独立切出的一块，并向上返回 `0`，表示这部分不再参与更高层的累加。代码里返回 `0` 的动作，本质上就是在做这次“切断”。

因此这题最值得迁移的是：**树上切成若干等和连通块时，往往先定目标和，再用子树和 DFS 在遍历过程中直接贪心地截出完整块。**

### 核心代码

```cpp
int cnt = 0, a = -1, b = -1;

int dfs(int u, int fa) {
    int sum = val[u];
    for (int v : g[u]) if (v != fa) sum += dfs(v, u);
    if (sum == target && cnt < 2) {
        if (cnt == 0) a = u;
        else b = u;
        cnt++;
        return 0;
    }
    return sum;
}
```

### 复杂度

$O(n)$。

## 41. [P8744 [蓝桥杯 2021 省 A] 左孩子右兄弟](https://www.luogu.com.cn/problem/P8744)

`DP` `树形DP` `树形 DP` `蓝桥杯省赛`

### 题意

对于一棵多叉树，我们可以通过“左孩子右兄弟”表示法，将其转化成一棵二叉树。 如果我们认为每个结点的子结点是无序的，那么得到的二叉树可能不唯一。

### 分析

左孩子右兄弟表示法的关键，不是记住转换形式，而是理解“重新排列儿子顺序”会影响什么。因为多叉树被拉成二叉树后，同一层兄弟会变成沿右指针排开的一条链，所以儿子越多，横向层数就越深。

因此对某个节点 `u` 来说，若它有若干个儿子，那么无论怎么排，最终高度都至少要支付“兄弟链长度”这部分代价；与此同时，你还可以把最深的那棵儿子子树放在最有利的位置，让它贡献纵向深度。

这就得到一个非常自然的递推：当前节点的最优高度 = 儿子个数 + 所有儿子子树最优高度中的最大值。前半项来自兄弟链，后半项来自最深那棵被接在左孩子位置上的子树。

所以这题最值得记的是一个建模动作：**当题目允许重排儿子，而目标和“兄弟链长度 + 某个最深分支”有关时，先把答案拆成横向代价与纵向代价两部分。**

### 核心代码

```cpp
int dfs(int u) {
    int mx = 0;
    for (int v : son[u]) mx = max(mx, dfs(v));
    return mx + (int)son[u].size();
}
```

### 复杂度

$O(n)$。
