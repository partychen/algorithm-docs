## 一、什么是单调队列优化动态规划

很多 DP 的瓶颈，不在于状态不会定义，而在于每次转移都要在一大段候选决策里找最优值。若你每次都老老实实枚举这段区间，复杂度往往会多出一层 `O(n)`，直接从可做题变成不可做题。

**单调队列优化 DP** 做的事情，就是在“候选决策区间会滑动”的前提下，把那些**以后再也不可能成为最优**的决策提前删除，只保留一小撮还可能上场的候选。这样，原本每个状态都要枚举整段区间，现在常常只看队头就够了。

它的本质不是“会写 deque”，而是：

- 候选决策有一个**单调滑动的可行区间**；
- 两个候选之间能判断谁被谁**永久支配**；
- 所以可以用队列把劣决策持续弹掉。

```text
朴素转移：dp[i] = min / max { value(k) }，k ∈ [L(i), R(i)]
优化后：  队头始终保留当前窗口里最优的 k
         窗口右移时，过期的弹掉
         新候选更优时，把队尾劣解弹掉
```

和它最容易混淆的几个优化如下：

| 优化方式 | 处理对象 | 关键条件 | 常见形态 |
|----------|----------|----------|----------|
| 单调队列优化 DP | 滑动窗口中的最值 | 可行区间滑动、劣决策可删 | `max/min` over interval |
| 单调栈 | 邻近更大/更小元素 | 比较关系局部成立 | next greater / previous smaller |
| 决策单调性分治优化 | 决策点整体单调 | 最优决策位置单调 | `opt[i] <= opt[i+1]` |
| 斜率优化 | 线性函数集合 | 上凸壳 / 下凸壳 | `dp[i] = min/max (ax+b)` |

一句话说清楚：**单调队列优化 DP 是“滑动窗口最值”在状态转移里的算法化表达。**

## 二、核心思想与前置知识

### 2.1 什么样的 DP 适合单调队列优化

最典型的形态是：

```
dp[i] = best_{k ∈ [L(i), R(i)]} { F(k) } + G(i)
```

其中：

- `best` 是 `max` 或 `min`；
- `L(i)`、`R(i)` 随 `i` 单调移动；
- `F(k)` 只和候选决策 `k` 有关，或者能被拆成“只和 `k` 有关 + 只和 `i` 有关”；
- 过期决策和劣决策都能在队头/队尾被删除。

只要长成这副样子，单调队列就很可能能上。

### 2.2 通用窗口最值骨架

```cpp
deque<int> q;
for (int i = 1; i <= n; i++) {
    while (!q.empty() && q.front() < L(i)) q.pop_front();

    while (!q.empty() && val(q.back()) <= val(i)) q.pop_back();
    q.push_back(i);

    dp[i] = calc(q.front(), i);
}
```

这里要注意两件事：

1. **谁先进队、谁后查询** 不一定固定。  
   有的题是“先插入候选，再计算当前状态”；有的题是“先用旧候选转移，再把当前点作为未来候选插入”。顺序要和状态定义严格对齐。
2. **队列里存的是下标，不是值。**  
   因为我们不仅要比较优劣，还要判断它有没有过期。

### 2.3 多重背包同余分组骨架

多重背包是单调队列优化 DP 的标志性模型。按体积 `w` 分组后，同一组里的状态下标会变成一条等差序列：

```cpp
for (int r = 0; r < w; r++) {
    deque<int> q;
    for (int k = 0, j = r; j <= m; k++, j += w) {
        while (!q.empty() && q.front() < k - s) q.pop_front();
        while (!q.empty() && old[r + q.back() * w] - q.back() * v <= old[j] - k * v)
            q.pop_back();
        q.push_back(k);
        dp[j] = old[r + q.front() * w] + (k - q.front()) * v;
    }
}
```

其中：

- `old` 是上一层物品的 DP，不能原地乱覆盖；
- `s` 是该物品最多可取个数；
- `old[r + k*w] - k*v` 是把转移式拆项后，只和候选决策 `k` 相关的部分。

### 2.4 队列里到底在维护什么

单调队列优化 DP 的所有实现，本质都在做同三件事：

| 队列操作 | 含义 | 常见原因 |
|----------|------|----------|
| `pop_front()` | 过期决策出队 | 已经不在可行区间 |
| `pop_back()` | 劣决策出队 | 新候选以后总不会更差 |
| `front()` | 当前最优决策 | 当前窗口最值所在 |

判断“谁能删掉谁”时，最常见的两种依据是：

- **直接维护 `F(k)` 的单调性**；
- **先把转移式拆项**，把与 `i` 无关的部分单独拿出来比较。

### 2.5 什么时候不该强上单调队列

如果题目不满足下面任一条件，就不要硬套：

1. 可行决策区间不会单调滑动；
2. 队尾劣决策无法被永久淘汰；
3. 转移值和 `i`、`k` 缠在一起，拆不出统一比较式；
4. 其实更适合分治优化、斜率优化或线段树。

## 三、子模型与分类详解

### 3.1 固定窗口最值转移型

```
状态定义：`dp[i]` 通常表示到位置 i 的最优值
转移关键：`dp[i] = G(i) + best_{k ∈ [i-L, i-R]} F(k)`
适用信号：每次只在最近一段区间里选最优前驱
实现要点：先弹出过期下标，再维护候选单调性，最后用队头转移
```

```cpp
while (!q.empty() && q.front() < left_bound) q.pop_front();
while (!q.empty() && key(q.back()) <= key(cur)) q.pop_back();
q.push_back(cur);
dp[i] = extra(i) + key(q.front());
```

### 3.2 同余分组多重背包型

```
状态定义：`dp[j]` 表示容量 j 的最优值
转移关键：把 `j = r + k*w` 按模 w 分组，每组内做滑动窗口最值
适用信号：多重背包，单件体积固定、件数有限
实现要点：必须保留 `old` 数组；队列里存的是“第几个该组元素”而不是原容量
```

```cpp
for (int r = 0; r < w; r++) {
    deque<int> q;
    for (int k = 0, j = r; j <= m; k++, j += w) {
        while (!q.empty() && q.front() < k - s) q.pop_front();
        while (!q.empty() && old[r + q.back() * w] - q.back() * v <= old[j] - k * v)
            q.pop_back();
        q.push_back(k);
        dp[j] = old[r + q.front() * w] + (k - q.front()) * v;
    }
}
```

### 3.3 线性拆项型

```
状态定义：`dp[i]` 由某个前驱区间转移而来
转移关键：把式子整理成 `dp[i] = H(i) + best_{k ∈ window}(A[k])`
适用信号：区间和、前缀和、买卖收益等可以拆成“候选部分 + 当前部分”
实现要点：先写出朴素转移，再把和 i 无关的候选值单独抽出来
```

```cpp
long long val(int k) {
    return dp[k] - pre[k];
}
while (!q.empty() && q.front() < left_bound) q.pop_front();
while (!q.empty() && val(q.back()) <= val(i)) q.pop_back();
q.push_back(i);
dp[i] = pre[i] + val(q.front());
```

### 3.4 双向扫描 / 分层队列型

```
状态定义：常见为 `dp[layer][pos]` 或 `f[i][j]`
转移关键：一层状态只从上一层某个滑动窗口转移
适用信号：多轮事件、双方向跳跃、每层都要做一次区间最值
实现要点：每一层都重新开队列；若有左右两个方向，通常要正反各扫一次
```

```cpp
for (int t = 1; t <= m; t++) {
    deque<int> q;
    for (int i = 1; i <= n; i++) {
        while (!q.empty() && q.front() < L(i, t)) q.pop_front();
        while (!q.empty() && prev[q.back()] <= prev[i]) q.pop_back();
        q.push_back(i);
        cur[i] = gain(i, t) + prev[q.front()];
    }
}
```

### 3.5 复杂决策维护型

```
状态定义：可能带“最后一段”“已选次数”“前驱断点”等附加信息
转移关键：候选决策仍然落在一个单调滑动区间里，但比较式更复杂
适用信号：需要维护前缀和、方案数、特殊约束、甚至额外证明贪心性质
实现要点：先证明可行决策区间单调，再写出队头/队尾删除条件，不要反过来猜公式
```

```cpp
while (!q.empty() && q.front() < left_bound(i)) q.pop_front();
while (!q.empty() && worse(q.back(), cur, i)) q.pop_back();
q.push_back(cur);
dp[i] = transfer(q.front(), i);
```

### 3.6 模型对比总表

| 模型 | 核心对象 | 典型题 | 常见复杂度 |
|------|----------|--------|------------|
| 固定窗口最值型 | 最近一段前驱 | 烽火传递、Little Bird | `O(n)` / `O(nq)` |
| 多重背包分组型 | 同余组上的滑窗 | 宝物筛选、股票交易 | `O(nm)` |
| 线性拆项型 | `A[k] + B(i)` | 修剪草坪、Buying Feed | `O(n)` / `O(nm)` |
| 双向扫描型 | 分层窗口最值 | Pogo-Cow、CF372C | `O(n^2)` / `O(nm)` |
| 复杂决策维护型 | 特殊断点 / 约束 | 划分、嘉年华、choukapai | 依题而定 |

### 题目 1：烽火传递（AcWing 1089）

> 选择若干个烽火台，使得任意连续 `m` 个位置里至少有一个被选中，代价最小。

**分析**：设 `dp[i]` 表示最后一个被选的烽火台是 `i` 的最小代价，则它只能从前面最近 `m` 个位置转移过来。于是转移就是一个标准的窗口最小值。

```
状态定义：`dp[i]` 表示最后一次点燃位置为 i 的最小总代价
转移方程：`dp[i] = a[i] + min(dp[j])`，其中 `j ∈ [i-m, i-1]`
边界条件：`dp[0] = 0`，其余初值为 `INF`
答案位置：最后 `m` 个位置中的 `min(dp[i])`
```

**C++ 代码**：
```cpp
deque<int> q;
dp[0] = 0;
q.push_back(0);
for (int i = 1; i <= n; i++) {
    while (!q.empty() && q.front() < i - m) q.pop_front();
    dp[i] = dp[q.front()] + a[i];
    while (!q.empty() && dp[q.back()] >= dp[i]) q.pop_back();
    q.push_back(i);
}
```

**核心启示**：
- **窗口最小值** 是单调队列优化 DP 最基础的入口。
- **先删过期，再取队头** 是最常见的执行顺序。
- **答案不一定在 `dp[n]`**，要回到题目条件重新看最后取哪一段。

### 题目 2：宝物筛选（洛谷 P1776）

> 每种物品有重量、价值和件数限制，求容量不超过 `m` 时的最大价值。

**分析**：这是多重背包单调队列优化模板题。按体积 `w` 的余数分组后，每组里的转移都会变成“长度至多为 `s` 的滑动窗口最大值”。

```
状态定义：`dp[j]` 表示容量 j 的最大价值
转移方程：`dp[j] = max(old[j-t*w] + t*v)`，其中 `0 ≤ t ≤ s`
边界条件：使用 `old` 保存上一层物品的 DP，避免原地覆盖
答案位置：`max(dp[0..m])` 或按题意取 `dp[m]`
```

**C++ 代码**：
```cpp
memcpy(old, dp, sizeof(dp));
for (int r = 0; r < w; r++) {
    deque<int> q;
    for (int k = 0, j = r; j <= m; k++, j += w) {
        while (!q.empty() && q.front() < k - s) q.pop_front();
        while (!q.empty() && old[r + q.back() * w] - q.back() * v <= old[j] - k * v)
            q.pop_back();
        q.push_back(k);
        dp[j] = old[r + q.front() * w] + (k - q.front()) * v;
    }
}
```

**核心启示**：
- **同余分组** 是多重背包单调队列优化的核心视角。
- **拆项后比较候选值**，不要直接在原式上死看。
- **`old` 数组不能省**，否则会把同一物品重复用超次数。

### 题目 3：修剪草坪（洛谷 P2627）

> 选择若干位置获得收益，但限制不能出现超过 `k` 个连续被选位置，求最大总收益。

**分析**：把“最后一段连续选中的区间”当作结尾来枚举，会得到一个前缀和拆项公式。拆开以后，转移就变成窗口最大值。

```
状态定义：`dp[i]` 表示前 i 个位置的最大收益，`pre[i]` 表示前缀和
转移方程：`dp[i] = pre[i] + max(dp[j-1] - pre[j])`，其中 `j ∈ [i-k, i]`
边界条件：`dp[0] = 0`
答案位置：`dp[n]`
```

**C++ 代码**：
```cpp
long long val(int j) {
    return dp[j - 1] - pre[j];
}
for (int i = 1; i <= n; i++) {
    while (!q.empty() && q.front() < i - k) q.pop_front();
    while (!q.empty() && val(q.back()) <= val(i)) q.pop_back();
    q.push_back(i);
    dp[i] = pre[i] + val(q.front());
}
```

**核心启示**：
- **前缀和拆项** 常常是把 DP 变成单调队列题的关键一步。
- **队列里比较的是转换后的候选值**，不一定是原始 `dp[j]`。
- **这类题很像“选一段区间作为最后操作”**。

### 题目 4：TEM-Temperature（洛谷 P3522）

> 第 `i` 天温度可任取区间 `[l_i, r_i]` 内的值。求最长连续区间，使得能选出一个单调不下降的温度序列。

**分析**：这题不一定显式写出二维 DP，但本质仍是“合法窗口维护”。一个区间合法，当且仅当窗口内所有下界的最大值不超过最后一天的上界，于是可以用单调队列维护窗口内 `l` 的最大值。

```
状态定义：`left` 表示当前合法窗口左端，队列维护窗口内 `l` 的最大值下标
转移方程：若当前窗口 `max(l) > r[i]`，则不断右移左端直到重新合法
边界条件：窗口初始为空
答案位置：枚举右端点 i 时的最大窗口长度
```

**C++ 代码**：
```cpp
for (int i = 1; i <= n; i++) {
    while (!q.empty() && l[q.back()] <= l[i]) q.pop_back();
    q.push_back(i);
    while (!q.empty() && l[q.front()] > r[i]) {
        if (q.front() == left) q.pop_front();
        left++;
    }
    ans = max(ans, i - left + 1);
}
```

**核心启示**：
- **单调队列优化 DP** 有时会退化成“直接维护合法窗口”。
- **窗口是否合法** 也可以看成状态转移条件的一部分。
- **最大值 / 最小值维护** 是这一类题的底层统一语言。

### 题目 5：PTA-Little Bird（洛谷 P3572）

> 一只鸟从第 1 棵树跳到第 n 棵树，每次最多跳 `k` 格。若跳到更高的树就多耗 1 点体力，问最少体力。

**分析**：朴素转移是从最近 `k` 个位置里找最优前驱，但“最优”不只看 `dp[j]`，同值时还要兼顾高度。于是队列维护的是一个带次关键字的最优候选集合。

```
状态定义：`dp[i]` 表示跳到 i 的最小体力
转移方程：`dp[i] = min(dp[j] + [h[j] <= h[i]])`，其中 `j ∈ [i-k, i-1]`
边界条件：`dp[1] = 0`
答案位置：`dp[n]`
```

**C++ 代码**：
```cpp
q.clear();
q.push_back(1);
for (int i = 2; i <= n; i++) {
    while (!q.empty() && q.front() < i - k) q.pop_front();
    dp[i] = dp[q.front()] + (h[q.front()] <= h[i]);
    while (!q.empty() && (dp[q.back()] > dp[i] || (dp[q.back()] == dp[i] && h[q.back()] <= h[i])))
        q.pop_back();
    q.push_back(i);
}
```

**核心启示**：
- **队列比较规则可以是多关键字**，不一定只比较一个 `dp` 值。
- **队头是当前最优前驱**，但“最优”的定义要完全按转移式写。
- **多询问题** 往往会把同一个单调队列过程重复多次。

### 题目 6：股票交易（洛谷 P2569）

> 第 `i` 天可买入/卖出股票，数量有上限，且相邻两次操作间要隔 `W` 天。求最大收益。

**分析**：把第 `i` 天持有 `j` 股的最优收益记为 `dp[i][j]`。买入和卖出的转移都对应一个长度有限的区间最值，因此可以正着扫一次、倒着扫一次，各用一个单调队列。

```
状态定义：`dp[i][j]` 表示第 i 天结束后持有 j 股时的最大收益
转移方程：买入、卖出都可整理成某个区间上的最大值转移
边界条件：`dp[i][j]` 初始为 `-INF`，且前 W 天只能直接买入
答案位置：`max(dp[T][0..maxp])`
```

**C++ 代码**：
```cpp
for (int j = 0; j <= maxp; j++) dp[i][j] = max(dp[i][j], dp[i - 1][j]);
if (i > W) {
    deque<int> q;
    for (int j = 0; j <= maxp; j++) {
        while (!q.empty() && q.front() < j - as[i]) q.pop_front();
        while (!q.empty() && pre[q.back()] + q.back() * ap[i] <= pre[j] + j * ap[i]) q.pop_back();
        q.push_back(j);
        dp[i][j] = max(dp[i][j], pre[q.front()] - (j - q.front()) * ap[i]);
    }
}
```

**核心启示**：
- **一题里可以开两次队列**，分别处理买入和卖出。
- **正向扫 / 反向扫** 是股票、区间买卖类题的常见技巧。
- **先写朴素式，再拆成窗口最值**，比直接猜队列维护式可靠得多。

### 题目 7：Pogo-Cow（洛谷 P3089）

> 在数轴上的若干点之间跳跃，只能一直向同一方向跳，且每次跳跃距离不能小于上一次。问最大得分。

**分析**：常见做法是设 `f[i][j]` 表示最后一步从 `j` 跳到 `i` 的最优得分。对固定的 `(i, j)`，合法前驱 `k` 会形成一个单调扩张的区间，于是可以边移动指针边维护区间最优值。

```
状态定义：`f[i][j]` 表示最后一步从 j 跳到 i 的最大得分
转移方程：`f[i][j] = p[i] + max(f[j][k])`，满足 `x[i]-x[j] >= x[j]-x[k]`
边界条件：每个点都可作为起点
答案位置：所有 `f[i][j]` 的最大值，并正反各做一遍
```

**C++ 代码**：
```cpp
for (int j = 1; j <= n; j++) {
    int k = j, best = 0;
    for (int i = j + 1; i <= n; i++) {
        while (k >= 1 && x[i] - x[j] >= x[j] - x[k]) {
            best = max(best, f[j][k]);
            k--;
        }
        f[i][j] = best + p[i];
        ans = max(ans, f[i][j]);
    }
}
```

**核心启示**：
- **单调队列思想不一定非得显式写 `deque`**，有时一个滑动区间最大值变量就够。
- **二维 DP** 里固定一维、扫描另一维，是很常见的降维方式。
- **正反各做一次** 往往是处理“向左 / 向右对称”题目的标准套路。

### 题目 8：Watching Fireworks is Fun（CF 372C）

> 一共有多场烟花表演，每场有收益与位置约束。相邻两场之间可移动的距离有限，求最大总收益。

**分析**：设 `dp[t][i]` 表示处理到第 `t` 场时站在位置 `i` 的最大收益。它只会从上一层某个可达区间转移来，因此每一层都能用一个单调队列做窗口最大值。

```
状态定义：`dp[t][i]` 表示第 t 场结束后站在位置 i 的最大收益
转移方程：`dp[t][i] = gain(t, i) + max(prev[j])`，其中 `|i-j| <= limit`
边界条件：第一层按起始位置初始化
答案位置：最后一层 `max(dp[m][i])`
```

**C++ 代码**：
```cpp
for (int t = 1; t <= m; t++) {
    deque<int> q;
    for (int i = 1, p = 1; i <= n; i++) {
        while (p <= min(n, i + lim[t])) {
            while (!q.empty() && prev[q.back()] <= prev[p]) q.pop_back();
            q.push_back(p++);
        }
        while (!q.empty() && q.front() < i - lim[t]) q.pop_front();
        dp[i] = prev[q.front()] + b[t] - abs(a[t] - i);
    }
    swap(dp, prev);
}
```

**核心启示**：
- **分层 DP + 每层一个窗口最值** 是单调队列优化的高频模型。
- **窗口左右边界可能同时移动**，要把“插入新区间”和“删除过期区间”都写清楚。
- **按层滚动数组** 可以把空间压下来。

### 题目 9：Buying Feed（洛谷 P4544）

> 沿路经过若干商店，买入饲料并运到终点。购买有库存限制，运输成本与携带量有关，问最小总花费。

**分析**：设 `f[i][j]` 为处理到第 `i` 个商店、已买 `j` 吨时的最小费用。转移可整理成 `c[i] * j + min(g(k))` 的形式，而 `k` 的可行区间正好是一个滑动窗口。

```
状态定义：`f[i][j]` 表示到第 i 家店时已买 j 吨的最小费用
转移方程：`f[i][j] = c[i] * j + min(g_i(k))`，其中 `k ∈ [j-s_i, j]`
边界条件：把起点和终点也视作库存为 0 的虚拟商店
答案位置：终点层的 `f[last][K]`
```

**C++ 代码**：
```cpp
for (int i = 1; i <= n + 1; i++) {
    deque<int> q;
    for (int j = 0; j <= K; j++) {
        while (!q.empty() && q.front() < j - s[i]) q.pop_front();
        while (!q.empty() && g(i, q.back()) >= g(i, j)) q.pop_back();
        q.push_back(j);
        f[i][j] = c[i] * j + g(i, q.front());
    }
}
```

**核心启示**：
- **最小值队列** 和最大值队列完全对称，只是比较号反过来。
- **先拆项，再看可行区间**，能把看似复杂的费用式子变成标准模板。
- **虚拟起点 / 终点** 往往能大幅简化状态定义。

### 题目 10：划分（洛谷 P5665）

> 把序列划分成若干段，要求段和单调不降，目标最小化段和平方和。

**分析**：这题的难点已经不在“会不会 deque”，而在于先把转移式和贪心性质证明出来。确定了每个决策点的生效区间以后，才可以用队列维护“当前最优断点”。

```
状态定义：`dp[i]` 表示前 i 个数划分后的最小代价，另需维护最后一段和的辅助信息
转移方程：从某个最优断点 j 转移，代价为 `dp[j] + (pre[i]-pre[j])^2`
边界条件：`dp[0] = 0`
答案位置：`dp[n]`
```

**C++ 代码**：
```cpp
while (head <= tail && pre[i] >= lim[q[head]]) head++;
int j = q[head];
dp[i] = dp[j] + sqr(pre[i] - pre[j]);
seg[i] = pre[i] - pre[j];
lim[i] = pre[i] + seg[i];
while (head <= tail && lim[q[tail]] >= lim[i]) tail--;
q[++tail] = i;
```

**核心启示**：
- **高级题先证明单调性，再写队列**，顺序不能反。
- **队列里维护的不一定只是下标优劣，还可能是“生效右端点”**。
- **单调队列优化 DP** 的上限很高，不只是模板题。

### 题目 11：yyf hates choukapai（洛谷 P4852）

> 需要选出若干次长度恰为 `c` 的连抽和若干次单抽，并限制连续单抽长度，求最大收益并恢复方案。

**分析**：固定连抽次数后，`dp[i][j]` 的转移区间是 `[i-c-d, i-c]`。把和 `i` 无关的部分抽出来以后，就得到一个标准的窗口最大值模型，外加前驱记录。

```
状态定义：`dp[i][j]` 表示处理到前 i 张牌、做了 j 次连抽的最大收益
转移方程：`dp[i][j] = pre[i] + max(dp[k][j-1] + a[k+1] - pre[k+c])`
边界条件：`dp[0][0] = 0`，非法状态置为 `-INF`
答案位置：最后一层中的最大值，并用 `pre_id` 反推方案
```

**C++ 代码**：
```cpp
for (int j = 1; j <= n_pack; j++) {
    deque<int> q;
    for (int i = j * c; i <= N; i++) {
        int k = i - c;
        while (!q.empty() && q.front() < i - c - d) q.pop_front();
        while (!q.empty() && val(q.back(), j - 1) <= val(k, j - 1)) q.pop_back();
        q.push_back(k);
        dp[i][j] = pre[i] + val(q.front(), j - 1);
        pre_id[i][j] = q.front();
    }
}
```

**核心启示**：
- **单调队列和方案恢复并不冲突**，只要额外记前驱即可。
- **二维 DP + 固定窗口** 是这类题最常见的组合。
- **非法状态要先清干净**，否则队列会把脏值带进来。

### 题目 12：Noi 嘉年华（洛谷 P1973）

> 有很多活动区间，要分配到两个场馆，并最大化“较少的那个场馆”的活动数量；还要回答某个活动必须被选时的答案。

**分析**：这题已经不是常规模板能直接套的了。常见做法是先离散化时间，再做区间 DP / 前后缀 DP，最后利用转移中的单峰性或决策单调性，把内层枚举压到队列或指针上。

```
状态定义：`f[i][j]`、`g[i][j]` 分别表示前缀/后缀区间里，在一个场馆选了 j 个活动时另一个场馆的最大值
转移方程：固定某个覆盖区间后，把左右两边的 DP 拼起来，并在一维上维护最优决策
边界条件：离散化后按区间包含关系预处理 `w(l, r)`
答案位置：枚举必须包含的区间后取 `max(min(...))`
```

**C++ 代码**：
```cpp
for (int l = 1; l <= cnt; l++)
    for (int r = l; r <= cnt; r++) {
        int y = n;
        for (int x = 0; x <= n; x++) {
            if (f[l][x] < 0) continue;
            while (y && worse(l, r, x, y - 1, y)) y--;
            ans = max(ans, min(x + w[l][r] + y, f[l][x] + g[r][y]));
        }
    }
```

**核心启示**：
- **压轴题里的“单调队列优化”** 往往藏在更大的 DP 框架内部。
- **先有状态设计，再有单调性证明**，绝不能本末倒置。
- **当内层最优决策沿外层单调移动** 时，队列、指针、分治优化常常能互相联动。

## 五、常见技巧与思路总结

### 5.1 一定先写出朴素转移

单调队列优化最怕“凭感觉硬套模板”。正确顺序永远是：

1. 写出朴素 DP；
2. 明确候选决策区间；
3. 拆出只和 `k` 有关的部分；
4. 再写队列比较规则。

### 5.2 过期条件和优劣条件不要混

```cpp
while (!q.empty() && q.front() < left_bound) q.pop_front();   // 过期
while (!q.empty() && value(q.back()) <= value(cur)) q.pop_back(); // 劣解
```

一个管“合法性”，一个管“最优性”，职责完全不同。

### 5.3 `old` 数组和滚动数组要分清

- 多重背包往往需要 `old` 保存上一层；
- 分层 DP 往往要 `prev / cur` 滚动；
- 如果直接原地覆盖，很容易把“本层状态”错误拿去继续转移。

### 5.4 最值方向别写反

- 求最大值：队列通常维护**单调递减**候选值；
- 求最小值：队列通常维护**单调递增**候选值。

一旦比较号写反，整题就会悄悄变成错解。

### 5.5 二维 DP 里，常常是“固定一维开队列”

像 `股票交易`、`Pogo-Cow`、`CF372C` 这类题，不是整体二维都进队列，而是：

- 固定物品 / 天数 / 最后一步；
- 在另一维上扫一遍并维护窗口。

### 5.6 方案恢复通常只要多记一个前驱

若题目要求输出方案，不要害怕单调队列。只要在转移时额外记录：

```cpp
pre_id[i] = q.front();
```

最后照常倒推即可。

## 六、优化方法与解题框架

### 6.1 看到单调队列优化 DP 时的思考流程

```text
读题
 ├─ 朴素转移是否在枚举一段连续决策区间？
 │   └─ 否：大概率不能直接用单调队列
 ├─ 这个区间是否随 i 单调滑动？
 │   └─ 否：继续找别的优化
 ├─ 能否把转移式拆成 “候选部分 + 当前部分”？
 │   └─ 能：候选部分进队列比较
 ├─ 队头何时过期？
 ├─ 队尾何时被新候选永久淘汰？
 ├─ 是求 max 还是 min？
 └─ 最后再决定：
     ├─ 一维一次扫描
     ├─ 同余分组
     ├─ 每层开一个队列
     └─ 正反各扫一次
```

### 6.2 信号词 → 题型判断表

| 题面信号词 | 往往对应的模型 | 第一反应 |
|------------|----------------|----------|
| “从最近 k 个状态里转移” | 固定窗口最值型 | 想队头最优前驱 |
| “物品件数有限，多重背包” | 同余分组型 | 按 `mod w` 分组 |
| “转移式里有前缀和 / 线性项” | 线性拆项型 | 先整理成 `A[k] + B(i)` |
| “每一轮事件都从上一轮某个可达区间转移” | 分层队列型 | 每层开新队列 |
| “向左做一遍、向右做一遍” | 双向扫描型 | 正反各扫 |
| “内层枚举区间很规整，但比较式复杂” | 高级决策维护型 | 先证单调性再写队列 |

### 6.3 常见复杂度估算

| 模型 | 朴素复杂度 | 优化后复杂度 |

| 分层窗口型 | `O(mnk)` | `O(mn)` |

## 七、终极总结与归纳

### 7.1 单调队列优化动态规划的本质

**一句话概括：把“当前窗口中不可能再成为最优决策”的状态提前删掉，于是每次转移只看队头。**

再展开一点：

- 它优化的不是状态数量，而是**每个状态的决策枚举成本**；
- 它依赖的不是玄学，而是**窗口滑动 + 劣决策可删**；
- 它常和前缀和、滚动数组、同余分组、双向扫描一起出现。

### 7.2 核心要点

1. 先写朴素转移，再谈单调队列。
2. 队头负责合法性，队尾负责最优性，两个条件不能混写。
3. 多重背包要按模分组，别直接在原容量维上乱推。
4. 二维 DP 往往是固定一维、扫描另一维。
5. 方案恢复只需多记前驱，不会破坏单调队列。
6. 高级题先证明单调性，再决定到底用队列、指针还是别的优化。
