## 一、什么是概率 DP

**概率 DP**，就是把“下一步会随机发生什么”也纳入动态规划状态里。

普通 DP 里，转移往往是确定的：从 $dp[i-1]$、$dp[i][j-1]$、$dp[u]$ 推到新状态；而在概率 DP 里，**同一个状态会以不同概率流向多个后继状态**，所以 $dp$ 数组里存放的值，常常不再是“最大值 / 最小值 / 方案数”，而是：

- **某件事发生的概率**
- **走到终点所需的期望步数 / 期望代价**
- **最终落到某类结局的概率分布**

你可以把它先记成一句话：

> **概率 DP = 状态设计 + 随机转移 + 概率论公式。**

一个最简单的图景是：

```text
当前状态 S
  ├─ 以 p1 转到 A
  ├─ 以 p2 转到 B
  └─ 以 p3 转到 C
```

如果你想求“最后到 A 的概率”，那就是**概率分布 DP**；
如果你想求“平均还要走多少步”，那就是**期望 DP**；
如果状态之间还有“留在原地”的自环，甚至会出现 $dp[u]$ 依赖 $dp[u]$ 自己，这时就要做**移项**，甚至进一步写成**方程组**。

### 与其他常见 DP 的对比

| DP 类型 | $dp$ 里存什么 | 转移特征 | 常见问题 |
|---------|---------------|----------|----------|
| 计数 DP | 方案数 | 把可行方案加起来 | 走格子、背包计数 |
| 最值 DP | 最优值 | 取 $\max / \min$ | LIS、背包、最短路型 DP |
| 概率 DP | 某状态出现概率 | 按概率加权转移 | 抛硬币、随机游走、锦标赛 |
| 期望 DP | 平均代价 / 平均步数 | 用期望线性性列式 | 收集问题、随机图、迷宫 |

### 概率 DP 的本质

概率 DP 真正难的地方，其实不是“会不会概率论”，而是下面这三个判断：

1. **状态该描述谁**：是第 $i$ 次操作后，还是当前还剩多少个对象？
2. **该顺推还是逆推**：概率通常顺推，期望通常逆推。
3. **有没有自环 / 后效性**：若 $dp[u]$ 会再次依赖 $dp[u]$，就必须先移项。

把这三点想清楚，概率 DP 的大方向就不会错。

## 二、核心思想与前置知识

### 2.1 概率型 DP 与期望型 DP 的区别

这是概率 DP 最重要的分水岭。

#### 情况一：求“发生的概率”

这时状态值是概率，最常见写法是：

```
dp[state] = 到达该状态的概率
```

转移时，把当前状态的概率乘上某条边的概率，再分配给后继状态。

#### 情况二：求“平均还要多久 / 花多少钱”

这时状态值是期望，常见写法是：

```
dp[state] = 从该状态出发到目标的期望代价
```

它和概率最大的不同在于：

- **概率**：常从起点顺推到终点
- **期望**：常从终点逆推回起点

因为期望的公式通常长这样：

```text
dp[u] = 当前代价 + Σ(转移概率 × dp[v])
```

如果所有 $v$ 都是“更靠后”的状态，那逆推就会非常自然。

举个具体例子：假设你想求“掷骰子直到出现 6 的期望次数”。设 $dp[i]$ 为“当前点数为 $i$，还需要多少次才能掷到 6”，那么终点 $dp[6]=0$ 是已知的，而起点处 $dp[\text{start}] = 1 + \frac{1}{6}dp[1] + \frac{1}{6}dp[2] + \cdots + \frac{1}{6}dp[6]$——每个状态都依赖“更靠近终点”的状态，所以从终点往回推是最自然的方向。如果试图从起点顺推，你会发现终点的期望值反而是未知的，公式无法展开。

### 2.2 三个最常用的概率论工具

#### 工具一：全概率公式

如果一个状态会分裂成若干互斥情况：

```text
P(答案) = Σ P(走这条分支) × P(该分支下答案成立)
```

这正是概率 DP 的直接来源。

#### 工具二：期望的线性性

期望最常用的不是定义，而是：

```text
E(X + Y) = E(X) + E(Y)
```

所以“平均总代价”经常可以拆成“当前这一步的代价 + 后续期望代价”。

#### 工具三：自环移项

如果你发现状态会回到自己，比如：

```text
dp[u] = stay * dp[u] + other
```

一定要立刻移项：

```text
dp[u] = other / (1 - stay)
```

这类题非常高频，尤其在**随机迷宫、原地停留、抽不到有效物品**这些模型里。

### 2.3 三种通用框架代码

#### 框架一：顺推概率 DP

```cpp
const int MAXN = 3005;
double dp[MAXN][MAXN];

void solveProb(int n) {
    dp[0][0] = 1.0;
    for (int i = 1; i <= n; ++i) {
        for (int j = 0; j <= i; ++j) {
            dp[i][j] += dp[i - 1][j] * lose[i];
            if (j > 0) dp[i][j] += dp[i - 1][j - 1] * win[i];
        }
    }
}
// 按题意读取 dp[n][...] 即可
```

适用信号：
- “进行了前 $i$ 次操作之后”
- “恰好出现 $j$ 次成功的概率”
- “最后落在某个分布上的概率”

#### 框架二：逆推期望 DP

```cpp
const int MAXN = 100005;
double dp[MAXN];

void solveExpect(int n) {
    dp[n] = 0;
    for (int i = n - 1; i >= 1; --i) {
        dp[i] = cost[i];
        for (auto [to, p] : trans[i]) dp[i] += p * dp[to];
    }
}
// 答案常是 dp[start]
```

适用信号：
- “从这里出发还要走多久”
- “从终点反推更自然”
- “目标状态一眼就知道答案是 0” 

#### 框架三：带自环的期望转移

```cpp
const int MAXN = 1005;
double dp[MAXN];

void solveLoop(int n) {
    for (int i = n - 1; i >= 1; --i) {
        double stay = pStay[i];
        double rhs = baseCost[i];
        for (auto [to, p] : trans[i]) rhs += p * dp[to];
        dp[i] = rhs / (1.0 - stay);
    }
}
// 当状态可能回到自己时先移项
```

适用信号：
- “有概率原地不动”
- “本轮失败后还在当前状态”
- “抽空、抽重复、卡在同一个格子”

### 2.4 精度、边界与建模陷阱

做概率 DP 时，最容易挂掉的不是大思路，而是细节。

#### 陷阱一：把“概率”和“方案数”混用

同一个 $dp[i][j]$，到底是“恰好 $j$ 次成功的概率”，还是“恰好 $j$ 次成功的方案数”，一定要写清楚。

#### 陷阱二：漏掉停留概率

只要存在“什么都没发生”的情况，就要小心：
- 概率题里，它会留在当前分布上
- 期望题里，它会把 $dp[u]$ 留在等式右边

#### 陷阱三：边界状态没初始化干净

例如：
- 没有硬币时，正面数只能是 $0$
- 已经到终点时，期望代价是 $0$
- 概率和应始终在 $[0,1]$ 附近

#### 陷阱四：浮点误差

建议统一使用：
- `double` / `long double`
- 比较时加 `EPS`
- 输出时按题目保留位数

## 三、分类与变体详解

### 3.1 概率分布 DP

**简要说明**：状态值直接表示某个事件发生的概率，最常见于“抛硬币 / 走棋盘 / 比赛晋级 / 多轮随机操作”。

```
状态定义：dp[i][j] = 进行到第 i 步、处于状态 j 的概率
转移方程：dp[next] += dp[cur] * p(cur -> next)
边界条件：初始状态概率为 1，其余为 0
答案位置：某个终态概率或终态概率之和
```

```cpp
const int MAXN = 3005;
double dp[MAXN][MAXN];

void solveProb(int n) {
    dp[0][0] = 1.0;
    for (int i = 1; i <= n; ++i) {
        for (int j = 0; j <= i; ++j) {
            dp[i][j] += dp[i - 1][j] * lose[i];
            if (j > 0) dp[i][j] += dp[i - 1][j - 1] * win[i];
        }
    }
}
```

### 3.2 期望倒推 DP

**简要说明**：状态值是“从当前状态到结束的平均代价”，终点通常最容易定边界，因此更适合倒推。

```
状态定义：dp[u] = 从状态 u 出发到目标的期望代价
转移方程：dp[u] = cost[u] + Σ p(u -> v) * dp[v]
边界条件：目标状态 dp[target] = 0
答案位置：dp[start]
```

```cpp
const int MAXN = 100005;
double dp[MAXN];

void solveExpect(int n) {
    dp[n] = 0;
    for (int u = n - 1; u >= 1; --u) {
        dp[u] = cost[u];
        for (auto [v, p] : trans[u]) dp[u] += p * dp[v];
    }
}
```

### 3.3 图上 / DAG 概率 DP

**简要说明**：若随机过程发生在 DAG、比赛树、拓扑有序图上，就能按拓扑序或比赛轮次推进。

```
状态定义：dp[u] = 到达 u 的概率 / 从 u 出发的期望
转移方程：沿有向边按概率传播或按出边平均反推
边界条件：起点概率为 1 或终点期望为 0
答案位置：终点、冠军、或 start 状态
```

```cpp
double dp[MAXN];

for (int u : topo) {
    for (auto [v, p] : adj[u]) dp[v] += dp[u] * p;
}
```

### 3.4 带自环的期望 DP

**简要说明**：若某一步可能“留在当前状态”，就会出现 $dp[u]$ 依赖 $dp[u]$。这时先移项，再做正常 DP。

```
状态定义：dp[u] = 从 u 出发完成任务的期望代价
转移方程：dp[u] = stay * dp[u] + other
边界条件：终点 / 吸收态期望为 0
答案位置：移项后得到的 dp[start]
```

```cpp
double solveOne(double stay, double rhs) {
    return rhs / (1.0 - stay);
}
```

### 3.5 多维 / 状压概率 DP

**简要说明**：有些题不只关心“走了几步”，还要记录“还剩几类对象”“哪些奖励已解锁”“三种生物各剩多少只”。这时维度会明显升高。

```
状态定义：dp[a][b][c] / dp[mask] = 复合状态下的概率或期望
转移方程：根据随机事件改变多个维度
边界条件：空集、全集、单一物种、全清空等终态
答案位置：初始状态或所有终态之和
```

```cpp
double dp[305][305][305];

for (int a = A; a >= 0; --a)
    for (int b = B; b >= 0; --b)
        for (int c = C; c >= 0; --c) {
            // 按题意更新三维状态
        }
```

### 模型对比总表

| 模型 | 常见状态 | 推进方向 | 代表题 |
|------|----------|----------|--------|
| 概率分布 DP | $dp[i][j]$ | 顺推 | Coins、Knight Probability |
| 期望倒推 DP | $dp[i]$、$dp[u]$ | 逆推 | Discovering Gold、绿豆蛙 |
| 带自环期望 | $dp[u]$ | 逆推 + 移项 | LOOPS、Sushi |
| 图上概率 / 期望 | $dp[u]$ | 拓扑序 / 层次推进 | Football、DAG 期望 |
| 多维 / 状压概率 DP | $dp[a][b][c]$、$dp[mask]$ | 视状态单调性而定 | Bad Luck Island、奖励关 |

## 四、15 道经典例题详解

### 🟢 入门级 Level 1：基础概率分布

### 题目 1：Toss Strange Coins（LeetCode 1230）

> 给定每枚硬币朝上的概率，求恰好出现 $target$ 次正面的概率。

**分析**：这是最标准的“前 $i$ 次操作 + 恰好 $j$ 次成功”的概率分布 DP。每一枚硬币只有正反两种结果，状态转移完全平行于 0/1 背包的“选 / 不选”结构。

```
状态定义：dp[i][j] = 前 i 枚硬币中恰好出现 j 次正面的概率
转移方程：dp[i][j] = dp[i-1][j] * (1-p[i]) + dp[i-1][j-1] * p[i]
边界条件：dp[0][0] = 1，其余初始为 0
答案位置：dp[n][target]
```

**C++ 代码**：
```cpp
const int MAXN = 1005;
double dp[MAXN];

void solve(int n, int target) {
    dp[0] = 1.0;
    for (int i = 1; i <= n; ++i) {
        for (int j = i; j >= 0; --j) {
            dp[j] = dp[j] * (1.0 - prob[i]);
            if (j > 0) dp[j] += dp[j - 1] * prob[i];
        }
    }
}
```

答案位置：$dp[target]$

**核心启示**：
- **概率分布就是加权转移**：本质仍是把上一步状态加权汇总过来。
- **倒序滚动**：这一类二维概率 DP 往往都能压成一维。

### 题目 2：Coins（AtCoder DP Contest I）

> 有 $n$ 枚偏硬币，第 $i$ 枚正面概率为 $p_i$，求正面数严格多于反面的概率。

**分析**：和题 1 完全同模，只是答案不再是“恰好 $target$ 次”，而是“所有 $j > n/2$ 的概率之和”。这题非常适合用来练“同一模型只改答案读取方式”。

```
状态定义：dp[i][j] = 前 i 枚硬币中恰好 j 次正面的概率
转移方程：dp[i][j] = dp[i-1][j] * (1-p[i]) + dp[i-1][j-1] * p[i]
边界条件：dp[0][0] = 1
答案位置：Σ dp[n][j] (j > n/2)
```

**C++ 代码**：
```cpp
const int MAXN = 3005;
double dp[MAXN];

void solve(int n) {
    dp[0] = 1.0;
    for (int i = 1; i <= n; ++i) {
        for (int j = i; j >= 0; --j) {
            dp[j] = dp[j] * (1.0 - p[i]);
            if (j > 0) dp[j] += dp[j - 1] * p[i];
        }
    }
}
```

答案位置：$\sum dp[j]$，其中 $j > n / 2$

**核心启示**：
- **同一张分布表可以回答不同问题**：恰好、至少、至多，都只是读取区间不同。
- **先求分布，再求答案** 是概率 DP 的常见套路。

### 题目 3：Knight Probability in Chessboard（LeetCode 688）

> 骑士从棋盘某格出发，随机走 $k$ 步，求仍留在棋盘内的概率。

**分析**：这题把“一维成功次数”扩展成了“二维位置 + 步数”。每一步从当前格子等概率尝试八个骑士方向，出界的概率直接丢弃即可。

```
状态定义：dp[step][x][y] = 走了 step 步后位于 (x, y) 的概率
转移方程：dp[step+1][nx][ny] += dp[step][x][y] / 8
边界条件：dp[0][row][column] = 1
答案位置：第 k 步后所有合法格子的概率和
```

**C++ 代码**：
```cpp
double cur[30][30], nxt[30][30];

void solve(int n, int k, int row, int col) {
    cur[row][col] = 1.0;
    for (int step = 1; step <= k; ++step) {
        memset(nxt, 0, sizeof(nxt));
        for (int x = 0; x < n; ++x) {
            for (int y = 0; y < n; ++y) {
                if (cur[x][y] == 0) continue;
                for (auto [dx, dy] : dirs) {
                    int nx = x + dx, ny = y + dy;
                    if (0 <= nx && nx < n && 0 <= ny && ny < n) nxt[nx][ny] += cur[x][y] / 8.0;
                }
            }
        }
        memcpy(cur, nxt, sizeof(cur));
    }
}
```

答案位置：$\sum cur[x][y]$

**核心启示**：
- **概率沿状态图流动**：本质就是在图上做带权传播。
- **滚动数组** 很适合“只依赖上一步”的概率转移。

### 🟡 进阶级 Level 2：基础期望与比赛树

### 题目 4：Bag of mice（Codeforces 148D）

> 袋子里有白鼠和黑鼠，公主与龙轮流抓，问公主最终赢的概率。

**分析**：关键不在“谁抓了什么”，而在**轮到公主行动时，袋中还剩多少白鼠和黑鼠**。由于题意只关心公主获胜概率，所以状态应直接定义成“当前局面对公主有利的概率”。

```
状态定义：dp[w][b] = 轮到公主时，袋中有 w 只白鼠、b 只黑鼠时公主获胜的概率
转移方程：按公主先抓、龙再抓、老鼠逃跑的题意分情况加权
边界条件：dp[0][b] = 0，dp[w][0] = 1
答案位置：dp[W][B]
```

**C++ 代码**：
```cpp
double dp[1005][1005];

void solve(int W, int B) {
    for (int w = 1; w <= W; ++w) dp[w][0] = 1.0;
    for (int w = 1; w <= W; ++w) {
        for (int b = 1; b <= B; ++b) {
            dp[w][b] += 1.0 * w / (w + b);
            if (b >= 3) dp[w][b] += 1.0 * b / (w + b) * (b - 1) / (w + b - 1) * (b - 2) / (w + b - 2) * dp[w][b - 3];
            if (b >= 2) dp[w][b] += 1.0 * b / (w + b) * (b - 1) / (w + b - 1) * w / (w + b - 2) * dp[w - 1][b - 2];
        }
    }
}
```

答案位置：$dp[W][B]$

**核心启示**：
- **复杂题面先压状态**：一旦状态定义对了，剩下只是分情况列概率。
- **按轮次定义状态** 往往比记录每个玩家的动作更清楚。

### 题目 5：Discovering Gold（LightOJ 1030）

> 给定一个带分值的棋盘，从第 $i$ 格掷骰子走到后面 $1 \sim 6$ 格，求走到终点时获得金币总数的期望。

**分析**：这是最经典的一维期望 DP。设 $dp[i]$ 表示从 $i$ 出发最终能得到的期望金币数，那么当前格金币一定拿到，后面只是在若干可走位置上平均分配期望。

```
状态定义：dp[i] = 从第 i 格出发走到终点的期望金币总和
转移方程：dp[i] = val[i] + avg(dp[i+1..min(n, i+6)])
边界条件：dp[n] = val[n]
答案位置：dp[1]
```

**C++ 代码**：
```cpp
double dp[110];

void solve(int n) {
    dp[n] = val[n];
    for (int i = n - 1; i >= 1; --i) {
        int cnt = min(6, n - i);
        dp[i] = val[i];
        for (int j = 1; j <= cnt; ++j) dp[i] += dp[i + j] / cnt;
    }
}
```

答案位置：$dp[1]$

**核心启示**：
- **当前收益先拿，后续期望再接**，这是期望 DP 最常见的写法。
- **可走分支数不固定** 时，别忘了按真实 $cnt$ 平均。

### 题目 6：Football（POJ 3071）

> $2^n$ 支队伍打淘汰赛，已知任意两队对战胜率，求最终冠军是谁的概率最大。

**分析**：比赛结构天然分层，所以状态按“打完第 $r$ 轮后，队伍 $i$ 仍存活的概率”来定义最自然。每一轮的对手范围都能由分组大小直接算出。

```
状态定义：dp[r][i] = 打完第 r 轮后，队伍 i 仍存活的概率
转移方程：dp[r][i] = dp[r-1][i] * Σ(dp[r-1][j] * win[i][j])，其中 j 在本轮可能对手集合里
边界条件：dp[0][i] = 1
答案位置：max(dp[n][i])
```

**C++ 代码**：
```cpp
double dp[12][1 << 7];

void solve(int n) {
    int m = 1 << n;
    for (int i = 0; i < m; ++i) dp[0][i] = 1.0;
    for (int r = 1; r <= n; ++r) {
        int block = 1 << r, half = block >> 1;
        for (int i = 0; i < m; ++i) {
            int base = i / block * block;
            int l = (i < base + half ? base + half : base);
            int rr = l + half;
            for (int j = l; j < rr; ++j) dp[r][i] += dp[r - 1][i] * dp[r - 1][j] * win[i][j];
        }
    }
}
```

答案位置：$\arg\max(dp[n][i])$

**核心启示**：
- **树形赛制 = 分层 DP**，每一轮就是一个天然阶段。
- **概率 DP 不只做序列题**，比赛树、图、网格都能做。

### 🟠 提高级 Level 3：倒推期望与自环

### 题目 7：Soup Servings（LeetCode 808）

> 两种汤 A、B 每次随机按四种方案分配，求 A 先空或同时空的概率。

**分析**：这题最关键的不是 DP 本身，而是**状态压缩**：把毫升数统一除以 $25$，问题规模立刻降下来。之后就是标准记忆化概率 DP。

```
状态定义：dp[a][b] = 当前 A 还剩 a 单位、B 还剩 b 单位时，A 先空或同时空的概率
转移方程：dp[a][b] = 1/4 * Σ dp[nextA][nextB]
边界条件：A、B 同时空返回 0.5；仅 A 空返回 1；仅 B 空返回 0
答案位置：dp[m][m]
```

**C++ 代码**：
```cpp
double memo[205][205];
bool vis[205][205];

double dfs(int a, int b) {
    if (a <= 0 && b <= 0) return 0.5;
    if (a <= 0) return 1.0;
    if (b <= 0) return 0.0;
    if (vis[a][b]) return memo[a][b];
    vis[a][b] = true;
    return memo[a][b] = 0.25 * (dfs(a - 4, b) + dfs(a - 3, b - 1) + dfs(a - 2, b - 2) + dfs(a - 1, b - 3));
}
```

答案位置：`dfs(m, m)`，其中 $m = (n + 24) / 25$

**核心启示**：
- **先做状态缩放**，常常比盲目套 DP 更重要。
- **概率题 + 记忆化搜索** 是非常自然的组合。

### 题目 8：Sushi（AtCoder DP Contest J）

> 有若干盘寿司，每盘初始有 $1 \sim 3$ 个，随机选盘吃一个，问吃完所有寿司的期望操作次数。

**分析**：如果直接记每一盘还剩几个寿司，状态会爆炸；真正有用的信息只有：当前有几盘 1 个、几盘 2 个、几盘 3 个。这就是经典的“三维计数态期望 DP”。

```
状态定义：dp[a][b][c] = 当前有 a 盘 1 个寿司、b 盘 2 个、c 盘 3 个时的期望操作数
转移方程：dp[a][b][c] = (n + a*dp[a-1][b][c] + b*dp[a+1][b-1][c] + c*dp[a][b+1][c-1]) / (a+b+c)
边界条件：dp[0][0][0] = 0
答案位置：dp[cnt1][cnt2][cnt3]
```

**C++ 代码**：
```cpp
double dp[305][305][305];
bool vis[305][305][305];

int n;

double dfs(int a, int b, int c) {
    if (!a && !b && !c) return 0.0;
    if (vis[a][b][c]) return dp[a][b][c];
    vis[a][b][c] = true;
    double s = a + b + c, res = n;
    if (a) res += a * dfs(a - 1, b, c);
    if (b) res += b * dfs(a + 1, b - 1, c);
    if (c) res += c * dfs(a, b + 1, c - 1);
    return dp[a][b][c] = res / s;
}
```

答案位置：`dfs(cnt1, cnt2, cnt3)`

**核心启示**：
- **状态压缩靠“统计量”**，不是机械开更高维数组。
- **期望里出现 $+n$**，本质是在补上“抽到空盘的浪费次数”。

### 题目 9：Collecting Bugs（POJ 2096）

> 有 $n$ 种 bug 类型、$s$ 个子系统，每天随机发现一个 bug，问集齐所有类型且覆盖所有子系统的期望天数。

**分析**：这题是双维 coupon collector。状态不是“具体收集了哪些”，而是“已经见过多少种类型、多少个子系统”。当前重复抽到旧类型与旧系统时，就形成了原地停留的自环。

```
状态定义：dp[i][j] = 已见过 i 种 bug 类型、j 个子系统时到目标状态的期望天数
转移方程：按“类型新旧 × 子系统新旧”四种情况列期望并移项
边界条件：dp[n][s] = 0
答案位置：dp[0][0]
```

**C++ 代码**：
```cpp
double dp[1010][1010];

void solve(int n, int s) {
    for (int i = n; i >= 0; --i) {
        for (int j = s; j >= 0; --j) {
            if (i == n && j == s) continue;
            dp[i][j] = (dp[i][j + 1] * i * (s - j)
                      + dp[i + 1][j] * (n - i) * j
                      + dp[i + 1][j + 1] * (n - i) * (s - j)
                      + 1.0 * n * s) / (1.0 * n * s - 1.0 * i * j);
        }
    }
}
```

答案位置：$dp[0][0]$

**核心启示**：
- **多维收集问题** 往往天然对应多维期望 DP。
- **先把自环移掉**，式子就会重新变成可递推形式。

### 🔴 困难级 Level 4：综合建模与期望最优

### 题目 10：LOOPS（HDU 3853）

> 在一个网格迷宫中，每个格子有概率向右、向下或停在原地，使用一次传送门要花 $2$ 点代价，问从左上到右下的期望代价。

**分析**：这题最经典的点就是“原地不动”。如果直接写期望转移，$dp[i][j]$ 会出现在等式两边；把停留概率移到左边之后，就得到标准的倒推公式。

```
状态定义：dp[i][j] = 从格子 (i, j) 走到终点的期望代价
转移方程：dp[i][j] = (2 + pr*dp[i][j+1] + pd*dp[i+1][j]) / (1-ps)
边界条件：dp[R][C] = 0
答案位置：dp[1][1]
```

**C++ 代码**：
```cpp
double dp[1010][1010];

void solve(int R, int C) {
    for (int i = R; i >= 1; --i) {
        for (int j = C; j >= 1; --j) {
            if (i == R && j == C) continue;
            double ps = stay[i][j], pr = right[i][j], pd = down[i][j];
            if (1.0 - ps < 1e-12) continue;
            dp[i][j] = (2.0 + pr * dp[i][j + 1] + pd * dp[i + 1][j]) / (1.0 - ps);
        }
    }
}
```

答案位置：$dp[1][1]$

**核心启示**：
- **一看到“留在原地”就先移项**，别硬推。
- **期望的单位要统一**：这题每次代价是 $2$，不是 $1$。

### 题目 11：New 21 Game（LeetCode 837）

> 从 $0$ 分开始，每次随机加上 $1 \sim maxPts$，当分数达到 $k$ 时停止，求最终分数不超过 $n$ 的概率。

**分析**：如果直接暴力枚举所有转移，复杂度会是 $O(n \times maxPts)$。关键优化在于：每个 $dp[i]$ 都是前面一段窗口平均值，于是可以用滑动窗口维护。

```
状态定义：dp[i] = 最终过程中恰好到达分数 i 的概率
转移方程：dp[i] = (dp[i-1] + ... + dp[i-maxPts]) / maxPts
边界条件：dp[0] = 1
答案位置：Σ dp[i] (k <= i <= n)
```

**C++ 代码**：
```cpp
double dp[10005];

void solve(int n, int k, int maxPts) {
    dp[0] = 1.0;
    double win = 1.0, ans = 0.0;
    for (int i = 1; i <= n; ++i) {
        dp[i] = win / maxPts;
        if (i < k) win += dp[i];
        else ans += dp[i];
        if (i - maxPts >= 0) win -= dp[i - maxPts];
    }
}
```

答案位置：`ans`

**核心启示**：
- **概率 DP 也能做前缀和优化**，不只是期望题会优化。
- **窗口平均** 是这题真正的信号词。

### 题目 12：换教室（UOJ #262 / NOIP2016）

> 一共有 $n$ 节课，每节课申请换教室有成功概率，最多能申请 $m$ 次，问总移动路程期望最小值。

**分析**：这题把最短路、概率和 DP 三件事揉到了一起。最短路先用 Floyd 预处理；然后用 $dp[i][j][0/1]$ 记录上到第 $i$ 节、用了 $j$ 次申请、当前这节是否申请换教室时的最小期望代价。

```
状态定义：dp[i][j][0/1] = 上完前 i 节课、用了 j 次申请、当前第 i 节课换或不换时的最小期望路程
转移方程：按前一节课换/不换、当前课换/不换四种情况做期望加权
边界条件：dp[1][0][0] = 0，dp[1][1][1] = 0
答案位置：min(dp[n][j][0], dp[n][j][1])，0 <= j <= m
```

**C++ 代码**：
```cpp
double dp[2005][2005][2];

double transBoth(int a, int b) {
    return p[a] * p[b] * dis[d[a]][d[b]]
         + p[a] * (1.0 - p[b]) * dis[d[a]][c[b]]
         + (1.0 - p[a]) * p[b] * dis[c[a]][d[b]]
         + (1.0 - p[a]) * (1.0 - p[b]) * dis[c[a]][c[b]];
}

void relax(int i, int j) {
    dp[i][j][0] = min(dp[i - 1][j][0] + dis[c[i - 1]][c[i]],
                      dp[i - 1][j][1] + p[i - 1] * dis[d[i - 1]][c[i]] + (1.0 - p[i - 1]) * dis[c[i - 1]][c[i]]);
    if (j > 0) {
        dp[i][j][1] = min(dp[i - 1][j - 1][0] + p[i] * dis[c[i - 1]][d[i]] + (1.0 - p[i]) * dis[c[i - 1]][c[i]],
                          dp[i - 1][j - 1][1] + transBoth(i - 1, i));
    }
}
```

答案位置：$\min(dp[n][j][0], dp[n][j][1])$

**核心启示**：
- **概率只是代价权重**，本质仍然是在做最优决策 DP。
- **先把图论部分做干净**，后面的概率 DP 才容易写。

### 🟣 专家级 Level 5：图上 / 多维 / 状压高阶

### 题目 13：绿豆蛙的归宿（洛谷 P4316）

> 在一个 DAG 上，从起点出发每次等概率走出边，求到终点路径长度的期望。

**分析**：DAG 让这个题特别干净。设 $dp[u]$ 为从 $u$ 到终点的期望长度，那么它就是所有出边的"边权 + 后继期望"的平均值；终点的期望自然为 $0$。

```
状态定义：dp[u] = 从点 u 到终点 n 的路径长度期望
转移方程：dp[u] = avg(w(u,v) + dp[v])
边界条件：dp[n] = 0
答案位置：dp[1]
```

**C++ 代码**：
```cpp
double dp[100005];

void solve() {
    dp[n] = 0.0;
    for (int u = n - 1; u >= 1; --u) {
        for (auto [v, w] : adj[u]) dp[u] += (dp[v] + w) / outdeg[u];
    }
}
```

答案位置：$dp[1]$

**核心启示**：
- **DAG 上的期望 DP** 往往比一般图简单很多，因为没有环。
- **平均出边** 是图上随机游走题最直接的模型。

### 题目 14：Bad Luck Island（Codeforces 540D）

> 岛上有石头、剪刀、布三类生物，随机两两相遇并淘汰一方，求三类生物各自最终存活的概率。

**分析**：这题不能只看“还剩总数”，必须同时记录三类各剩多少只，所以状态天然是三维概率 DP。每次相遇都让某一类数量减一，过程始终单调下降。

```
状态定义：dp[r][s][p] = 当前石头、剪刀、布分别剩 r、s、p 只的概率
转移方程：按 R-S、S-P、P-R 三种相遇的相对概率，把当前概率分配到更小状态
边界条件：初始状态 dp[R][S][P] = 1
答案位置：所有单一物种终态概率之和
```

**C++ 代码**：
```cpp
double dp[105][105][105];

void solve(int R, int S, int P) {
    dp[R][S][P] = 1.0;
    for (int r = R; r >= 0; --r) {
        for (int s = S; s >= 0; --s) {
            for (int p = P; p >= 0; --p) {
                double tot = 1.0 * r * s + 1.0 * s * p + 1.0 * p * r;
                if (tot == 0) continue;
                if (r && s) dp[r][s - 1][p] += dp[r][s][p] * r * s / tot;
                if (s && p) dp[r][s][p - 1] += dp[r][s][p] * s * p / tot;
                if (p && r) dp[r - 1][s][p] += dp[r][s][p] * p * r / tot;
            }
        }
    }
}
```

答案位置：$\sum dp[r][0][0]$、$\sum dp[0][s][0]$、$\sum dp[0][0][p]$

**核心启示**：
- **多物种随机消耗** 往往天然对应多维概率分布。
- **状态单调减小** 是三维 DP 还能做下去的关键。

### 题目 15：奖励关（洛谷 P2473）

> 系统会随机抛出 $k$ 次宝物，每次你可以吃或不吃；宝物有前置依赖和得分，问采取最优策略时的期望总得分。

**分析**：这题同时有“随机出现什么宝物”和“你如何最优选择”的双重结构，所以状态既要记录当前轮次，又要记录已经吃过哪些宝物。由于宝物种类只有 $15$，状态压缩 DP 正合适。

```
状态定义：dp[t][mask] = 从第 t 次抛物开始、已吃宝物集合为 mask 时的最大期望得分
转移方程：对每个随机出现的宝物 i，若依赖满足则在“吃/不吃”中取较优，再对所有 i 取平均
边界条件：dp[k+1][mask] = 0
答案位置：dp[1][0]
```

**C++ 代码**：
```cpp
double dp[105][1 << 15];

void solve(int k, int n) {
    for (int t = k; t >= 1; --t) {
        for (int mask = 0; mask < (1 << n); ++mask) {
            for (int i = 0; i < n; ++i) {
                double best = dp[t + 1][mask];
                if ((need[i] & mask) == need[i]) best = max(best, score[i] + dp[t + 1][mask | (1 << i)]);
                dp[t][mask] += best / n;
            }
        }
    }
}
```

答案位置：$dp[1][0]$

**核心启示**：
- **随机过程 + 最优决策** 很适合“期望 DP + 状压”的混合建模。
- **期望里照样能取 $\max$**，因为你在随机事件发生后仍然可以做最优选择。

## 五、常见技巧与思路总结

### 5.1 先问：$dp$ 里到底存“概率”还是“期望”

这是第一性问题。若状态含义都没定对，后面所有公式都会错。

### 5.2 一看到“平均还要多久”就优先想期望倒推

终点常常天然是 $0$，逆推写起来会非常干净。

### 5.3 一看到“可能原地不动 / 抽到无效物品”就检查是否要移项

这是概率 DP 最经典的坑点之一。

```cpp
dp[u] = (rhs) / (1.0 - stay);
```

### 5.4 大状态别硬记“具体对象”，尽量转成“统计量”

例如：
- 寿司题只记 $1/2/3$ 个寿司的盘数
- 奖励关只记已经解锁的集合 $mask$
- 石头剪刀布只记三类剩余数量

### 5.5 概率 DP 也能做优化

常见优化包括：
- **滚动数组**：Coins、Knight Probability
- **滑动窗口**：New 21 Game
- **状态压缩**：奖励关
- **拓扑序 / DAG**：绿豆蛙的归宿

### 5.6 精度输出别掉以轻心

很多题对误差要求是 $10^{-6}$、$10^{-9}$。统一用 `double`，并按题目要求 `setprecision` 输出。

## 六、优化方法 / 解题思路框架

### 6.1 概率 DP 做题流程图

```text
先读题目问什么
    |
    +-- 求某事发生概率？ ----> 概率分布 DP
    |
    +-- 求平均步数 / 平均代价？ -> 期望 DP
                |
                +-- 有自环 / 原地不动？ -> 先移项
                |
                +-- 无环 / DAG？ -------> 逆推或拓扑 DP
```

### 6.2 信号词 -> 题型 判断表

| 信号词 | 优先考虑模型 |
|--------|--------------|
| “恰好出现 k 次” | 概率分布 DP |
| “最终留在棋盘内的概率” | 步数 + 位置概率 DP |
| “平均还要多少次操作” | 期望 DP |
| “有概率什么都不发生” | 自环期望 DP |
| “随机走图 / DAG” | 图上期望 DP |
| “哪些物品已解锁 / 已拿到” | 状压概率 / 期望 DP |
| “三类对象数量同时变化” | 多维概率 DP |

### 6.3 四步建模法

1. **状态**：当前剩什么 / 进行了多少步 / 已收集什么。
2. **随机事件**：这一轮随机发生哪些分支，各自概率是多少。
3. **是否自环**：若能回当前状态，先把 $dp[u]$ 移到左边。
4. **答案读取**：是单个状态、若干终态求和，还是初始状态本身。

### 6.4 常见复杂度优化思路

| 场景 | 优化方法 |
|------|----------|
| 二维分布只依赖前一层 | 滚动数组 |
| 转移是固定长度平均 | 前缀和 / 滑动窗口 |
| 图是 DAG | 逆拓扑序 |
| 状态是集合 | 位压缩 |
| 三维但状态数量实际有限 | 统计量压缩 + 记忆化 |

## 七、终极总结与归纳

### 7.1 概率 DP 的本质

**把“随机过程”翻译成“状态之间按概率流动”的 DP。**

换句话说，概率 DP 从来不是单独的一门神秘技巧；它只是把普通 DP 的“确定转移”改成了“带概率的加权转移”，再配上期望线性性、自环移项、图上拓扑这些工具。

### 7.2 核心要点

1. **先分清概率和期望**：这决定了 $dp$ 的语义和推进方向。
2. **概率题多顺推，期望题多逆推**：别硬来。
3. **能停在原地就先移项**：这是最常见的公式化处理。
4. **状态尽量压成统计量**：别直接记每个对象的具体情况。
5. **图上随机过程优先看 DAG / 层次结构**：能省掉很多方程。
6. **答案读取方式很关键**：单点、终态和、初始状态本身都可能是答案。

### 7.5 口诀

> 🎲 **概率 DP 口诀**：
>
> **先问状态装啥值，概率期望要分开；**
> **顺推常看分布表，逆推多从终点来；**
> **若能原地再停住，赶紧移项别硬挨；**
> **对象太多记数量，集合就用 bitmask 摆；**
> **图上随机看拓扑，窗口平均前缀裁；**
> **本质还是旧状态，只是转移带权来。**
