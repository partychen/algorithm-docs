## 一、什么是树形 DP

**树形 DP** 是在 **树结构** 上进行的动态规划。利用树天然的 **递归子结构**（每个节点的子树本身也是一棵树），
自底向上（或自顶向下）合并子问题的解，从而得到全局最优解。

### 为什么树适合 DP？

```
        1
       / \
      2   3
     / \   \
    4   5   6
```

- 树 **无环**，天然具有拓扑序 → 不会出现循环依赖
- 树的 **子结构性**：以任意节点为根的子树，都是一个独立的子问题
- 合并方式：父节点的答案 = f(所有子节点的答案)

### 与线性 DP 的对比

| 维度       | 线性 DP              | 树形 DP               |
|-----------|----------------------|----------------------|
| 结构       | 数组/序列             | 树/图                 |
| 遍历方式    | for 循环             | DFS 递归              |
| 状态定义    | $dp[i]$ = 前 $i$ 个元素   | $dp[u]$ = 以 $u$ 为根的子树 |
| 转移方向    | 从左到右/从小到大      | 自底向上（后序遍历）     |

## 二、核心思想与前置知识

### 2.1 三个核心问题

做任何树形 DP 题目，必须回答三个问题：

1. **状态怎么定义？** → $dp[u][...]$ 表示以 $u$ 为根的子树中的什么？
2. **转移怎么写？** → 父节点如何从子节点的答案合并得到自己的答案？
3. **答案在哪里？** → 最终答案在 $dp[root]$ 还是需要遍历所有节点？

### 2.2 前置知识

- **图的邻接表表示**
- **DFS（深度优先搜索）** 的递归实现
- **基础 DP** 思想（最优子结构、无后效性）

### 2.3 通用框架代码 (C++)

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 100005;
vector<int> adj[MAXN];  // 邻接表
int dp[MAXN][2];        // DP 数组（状态数视题目而定）

void dfs(int u, int parent) {
    // Step 1: 初始化当前节点
    dp[u][0] = ...; 
    dp[u][1] = ...;
    
    // Step 2: 遍历所有子节点
    for (int v : adj[u]) {
        if (v == parent) continue;  // 避免回到父节点
        dfs(v, u);                  // 先递归处理子树
        
        // Step 3: 状态转移——合并子节点的信息
        dp[u][0] = merge(dp[u][0], dp[v]);
        dp[u][1] = merge(dp[u][1], dp[v]);
    }
    
    // Step 4: 可选的后处理
}

int main() {
    // 建树 + 调用 dfs(root, -1)
    // 输出 dp[root][...]
}
```

### 题目 1：树的最大深度（LeetCode 104）

> 给定一棵二叉树，求其最大深度。

**分析**：最简单的树形 DP，一个状态即可。

```
状态定义：dp[u] = 以 u 为根的子树的最大深度
转移方程：dp[u] = max(dp[v] for v in children(u)) + 1
边界条件：叶节点 dp[leaf] = 1
答案位置：dp[root]
```

**C++ 代码**：
```cpp
int dfs(int u, int parent) {
    int maxDepth = 0;
    for (int v : adj[u]) {
        if (v == parent) continue;
        maxDepth = max(maxDepth, dfs(v, u));
    }
    return maxDepth + 1;
}
```

**核心启示**：树形 DP 的最简形式 —— 单状态，自底向上合并。

### 题目 2：子树节点数 / 子树权值和

> 给定一棵 $n$ 个节点的树，每个节点有权值 $w[u]$，求每个节点子树的权值和。

```
状态定义：sum[u] = u 的子树（含 u）的权值和
转移方程：sum[u] = w[u] + Σ sum[v]  (v 是 u 的子节点)
```

**C++ 代码**：
```cpp
int sz[MAXN];   // 子树大小
long long sum[MAXN]; // 子树权值和

void dfs(int u, int fa) {
    sz[u] = 1;
    sum[u] = w[u];
    for (int v : adj[u]) {
        if (v == fa) continue;
        dfs(v, u);
        sz[u] += sz[v];
        sum[u] += sum[v];
    }
}
```

**核心启示**：这是所有树形 DP 的"砖块"——几乎所有复杂问题都需要先统计子树信息。

### 题目 3：树的直径（LeetCode 543 / CSES Tree Diameter）

> 求树上最远两点之间的距离（边数）。

**关键洞察**：树的直径 = 某个节点的 **最长链 + 次长链**。

```
状态定义：depth[u] = 从 u 出发向子树走的最长链长度
更新直径：diameter = max(diameter, 最长链 + 次长链)
转移方程：depth[u] = max(depth[v] + 1)  (v 是 u 的子节点)
```

**C++ 代码**：
```cpp
int ans = 0;

int dfs(int u, int fa) {
    int first = 0, second = 0;  // 最长链、次长链
    for (int v : adj[u]) {
        if (v == fa) continue;
        int d = dfs(v, u) + 1;
        if (d >= first) { second = first; first = d; }
        else if (d > second) { second = d; }
    }
    ans = max(ans, first + second);  // 经过 u 的最长路径
    return first;                     // 返回最长链给父节点
}
```

**核心启示**：
- **返回值 $\neq$ 答案**：返回给父节点的是"最长链"，但答案是"最长链+次长链"
- 这种"递归返回部分信息 + 全局变量记录最终答案"的模式非常常见

### 题目 4：没有上司的舞会 / 打家劫舍 III（洛谷 P1352 / LeetCode 337）

> 树形结构中每个节点有权值，相邻节点不能同时选取，求选取的最大权值和。
> （经典最大权独立集问题）

**状态设计**：对每个节点考虑"选"或"不选"两种状态。

```
dp[u][0] = 不选 u 时，u 的子树能获得的最大权值
dp[u][1] = 选 u 时，u 的子树能获得的最大权值

转移方程：
  dp[u][1] = w[u] + Σ dp[v][0]           // 选 u，则子节点都不能选
  dp[u][0] = Σ max(dp[v][0], dp[v][1])   // 不选 u，子节点可选可不选

答案：max(dp[root][0], dp[root][1])
```

**C++ 代码**：
```cpp
void dfs(int u, int fa) {
    dp[u][0] = 0;
    dp[u][1] = w[u];
    for (int v : adj[u]) {
        if (v == fa) continue;
        dfs(v, u);
        dp[u][0] += max(dp[v][0], dp[v][1]);
        dp[u][1] += dp[v][0];
    }
}
// ans = max(dp[root][0], dp[root][1])
```

**核心启示**：
- **"选/不选"二状态** 是树形 DP 最经典的模式
- 两个状态之间有 **约束关系**（选了父就不能选子）
- 这个模板可以解决大量类似问题

### 题目 5：最小点覆盖（POJ 1463 / 洛谷 P2016 战略游戏）

> 选最少的节点，使得每条边至少有一个端点被选中。

**状态设计**：与最大独立集对偶。

```
dp[u][0] = 不选 u，覆盖 u 子树所有边所需的最少节点数
dp[u][1] = 选 u，覆盖 u 子树所有边所需的最少节点数

转移方程：
  dp[u][0] = Σ dp[v][1]                  // 不选 u → 子节点必须全选（否则 u-v 边无法覆盖）
  dp[u][1] = 1 + Σ min(dp[v][0], dp[v][1]) // 选 u → 子节点可选可不选

答案：min(dp[root][0], dp[root][1])
```

**C++ 代码**：
```cpp
void dfs(int u, int fa) {
    dp[u][0] = 0;
    dp[u][1] = 1;
    for (int v : adj[u]) {
        if (v == fa) continue;
        dfs(v, u);
        dp[u][0] += dp[v][1];              // 子节点必选
        dp[u][1] += min(dp[v][0], dp[v][1]); // 子节点可选可不选
    }
}
```

**核心启示**：
- 与最大独立集的转移 **恰好反转**
- 理解 **约束来源**：边的覆盖要求决定了选与不选的依赖关系

### 题目 6：最大子树和（洛谷 P1122）

> 给定一棵 $n$ 个节点的带权树（节点有正负权值），你可以删去若干条边，使得剩下的某个连通块的权值和最大。求这个最大值。

**分析**：基础树形 DP，核心是"负贡献子树直接剪掉"。

```
状态定义：f[u] = 包含 u 的最大连通子图权值和
转移方程：f[u] = w[u] + Σ max(0, f[v])   (v 是 u 的子节点)
答案位置：max(f[u]) over all u
```

**思路**：对每个子树，如果子树的贡献 $f[v] > 0$ 就加上，否则不连这条边（等价于剪掉）。

**C++ 代码**：
```cpp
const int N = 16005;
vector<int> adj[N];
int w[N], n, ans;
int f[N];

void dfs(int u, int fa) {
    f[u] = w[u];
    for (int v : adj[u]) {
        if (v == fa) continue;
        dfs(v, u);
        if (f[v] > 0) f[u] += f[v]; // 正贡献才连接
    }
    ans = max(ans, f[u]);
}
// 调用 dfs(1, 0)，ans 初始化为 -INF
```

**核心启示**：
- $\max(0, f[v])$ 是"选择性连接"的经典技巧
- 这个思想和"最大子段和"完全类似，只是从序列推广到了树

### 题目 7：最小支配集（三状态树形 DP）

> 选最少的节点，使得每个节点要么被选中，要么与一个被选中的节点相邻。

**状态设计**：需要三个状态！

```
dp[u][0] = u 被选中
dp[u][1] = u 未被选中，但被某个子节点支配
dp[u][2] = u 未被选中，也未被子节点支配（需要父节点支配）

转移方程：
  dp[u][0] = 1 + Σ min(dp[v][0], dp[v][1], dp[v][2])
  dp[u][1] = 需要至少一个子节点被选中 + 其余取 min(dp[v][0], dp[v][1])
  dp[u][2] = Σ min(dp[v][0], dp[v][1])

答案：min(dp[root][0], dp[root][1])  // root 不能依赖父节点
```

**$dp[u][1]$ 的精确处理**：
```cpp
void dfs(int u, int fa) {
    dp[u][0] = 1;
    dp[u][1] = 0;
    dp[u][2] = 0;
    int minExtra = INF; // 额外代价：强制选一个子节点的最小增量
    bool hasSelectedChild = false;
    
    for (int v : adj[u]) {
        if (v == fa) continue;
        dfs(v, u);
        dp[u][0] += min({dp[v][0], dp[v][1], dp[v][2]});
        dp[u][2] += min(dp[v][0], dp[v][1]);
        
        // 对于 dp[u][1]：至少需要一个孩子被选
        int best = min(dp[v][0], dp[v][1]);
        dp[u][1] += best;
        minExtra = min(minExtra, dp[v][0] - best);
        if (dp[v][0] <= dp[v][1]) hasSelectedChild = true;
    }
    if (!hasSelectedChild) dp[u][1] += minExtra;
}
```

**核心启示**：
- 当二状态不够用时，增加状态维度
- "至少选一个"的约束处理是一个经典技巧
- 状态之间的约束越复杂，DP 维度越高

### 题目 8：二叉树最大路径和（LeetCode 124）

> 找一条路径（不一定过根），使得路径上的节点值之和最大。路径中至少包含一个节点。

**关键洞察**：路径 = 左链 + 当前节点 + 右链（可以为空）。

```
状态定义：maxChain(u) = 从 u 出发向下走的最大链和（可以不走，即只取 u）
全局答案更新：ans = max(ans, 左链 + u + 右链)
                     其中左链右链可以为 0（不走）

转移方程：maxChain(u) = u.val + max(0, max(maxChain(left), maxChain(right)))
```

**C++ 代码**：
```cpp
int ans = INT_MIN;

int dfs(TreeNode* u) {
    if (!u) return 0;
    int left  = max(0, dfs(u->left));   // 负数链不如不走
    int right = max(0, dfs(u->right));
    ans = max(ans, left + u->val + right);  // 经过 u 的完整路径
    return u->val + max(left, right);       // 返回最佳单链
}
```

**核心启示**：
- 与"树的直径"本质相同，但加入了 **权值** 和 **负数处理**
- $\max(0, ...)$ 的技巧：负收益的链不如不走
- **返回单链，更新双链** —— 经典的"拆分递归返回值与答案"

### 题目 9：监控二叉树（LeetCode 968）

> 在二叉树节点上放置摄像头，每个摄像头可以监控其父节点、自身和子节点。求最少摄像头数。

**状态设计**：三状态！

```
a = u 上放摄像头，覆盖 u 子树的最少摄像头数
b = u 上不放，但 u 被子节点覆盖
c = u 上不放，u 也未被覆盖（等待父节点覆盖）

转移（设子节点返回 la,lb,lc 和 ra,rb,rc）：
  a = 1 + min(la,lb,lc) + min(ra,rb,rc)    // u 放摄像头，子节点任意状态
  b = min(la+min(ra,rb), min(la,lb)+ra)     // 至少一个子放了摄像头
  c = lb + rb                                // 两个子都被覆盖但不能帮 u

答案：min(a, b)  // root 不能指望父节点
```

**C++ 代码**：
```cpp
struct State { int a, b, c; };

State dfs(TreeNode* u) {
    if (!u) return {1e9, 0, 0};  // 空节点：不能放摄像头，已覆盖，代价0
    auto [la, lb, lc] = dfs(u->left);
    auto [ra, rb, rc] = dfs(u->right);
    int a = 1 + min({la, lb, lc}) + min({ra, rb, rc});
    int b = min({la + rb, lb + ra, la + ra});
    int c = lb + rb;
    return {a, b, c};
}
// ans = min(root.a, root.b)
```

**核心启示**：
- 这道题综合了"选/不选"与"覆盖约束"
- 三状态设计：**主动放、被动被覆盖、等待覆盖**
- 空节点的边界值设计很巧妙

### 题目 10：三色二叉树（洛谷 P2585 [ZJOI2006]）

> 一棵二叉树（用递归序列描述），每个节点涂绿/红/蓝三色，要求相邻节点颜色不同。求绿色节点的 **最多** 和 **最少** 个数。

**分析**：多色约束的树形 DP，需要枚举当前节点的颜色。

```
状态定义：f[u][c] = 节点 u 涂颜色 c 时，子树内绿色节点最大数
          g[u][c] = 节点 u 涂颜色 c 时，子树内绿色节点最小数
          其中 c ∈ {0=绿, 1=红, 2=蓝}

转移方程（以最大值为例）：
  若 u 有两个孩子 l, r：
    f[u][c] = (c==0 ? 1 : 0) + max over (a,b) where a≠c, b≠c, a≠b of (f[l][a] + f[r][b])
  若 u 只有一个孩子 l：
    f[u][c] = (c==0 ? 1 : 0) + max over a where a≠c of f[l][a]
```

**关键**：二叉树 + 三色 + 相邻不同 → 枚举颜色组合，两孩子也不能同色。

**C++ 代码**：
```cpp
const int N = 500005;
int f[N][3], g[N][3]; // 最大/最小绿色数
int ls[N], rs[N], tot;
string s;

int build(int &pos) {
    int u = ++tot;
    int type = s[pos++] - '0'; // 0=叶, 1=一个孩子, 2=两个孩子
    ls[u] = rs[u] = 0;
    if (type >= 1) ls[u] = build(pos);
    if (type >= 2) rs[u] = build(pos);
    return u;
}

void dfs(int u) {
    if (!ls[u] && !rs[u]) { // 叶节点
        f[u][0] = g[u][0] = 1;
        f[u][1] = g[u][1] = 0;
        f[u][2] = g[u][2] = 0;
        return;
    }
    if (ls[u]) dfs(ls[u]);
    if (rs[u]) dfs(rs[u]);

    for (int c = 0; c < 3; c++) {
        int base = (c == 0) ? 1 : 0;
        if (ls[u] && rs[u]) { // 两个孩子
            f[u][c] = -1; g[u][c] = 1e9;
            for (int a = 0; a < 3; a++) if (a != c)
                for (int b = 0; b < 3; b++) if (b != c && b != a) {
                    f[u][c] = max(f[u][c], base + f[ls[u]][a] + f[rs[u]][b]);
                    g[u][c] = min(g[u][c], base + g[ls[u]][a] + g[rs[u]][b]);
                }
        } else { // 一个孩子
            f[u][c] = -1; g[u][c] = 1e9;
            for (int a = 0; a < 3; a++) if (a != c) {
                f[u][c] = max(f[u][c], base + f[ls[u]][a]);
                g[u][c] = min(g[u][c], base + g[ls[u]][a]);
            }
        }
    }
}
// build 建树后 dfs(1)；max/min(f[1][c], g[1][c]) for c=0,1,2
```

**核心启示**：
- 多色约束 → 枚举所有合法颜色组合
- 同时求最大和最小：开两个 DP 数组，并行计算
- 二叉树的输入方式特殊（递归序列），需要用递归建树

### 题目 11：Gem 气垫车（洛谷 P4395 [BOI2003]）

> 给一棵 $n$ 个节点的树，给每个节点赋一个 **正整数** 权值，要求相邻节点权值不同，求权值总和最小值。

**分析**：和"三色"类似，但颜色数不固定。关键观察：**最优解只需用到 $O(\log n)$ 种权值**。

```
状态定义：f[u][i] = 节点 u 赋值 i 时，子树最小权值和
转移方程：f[u][i] = i + Σ min(f[v][j], j ≠ i)   (v 是 u 的子节点)
答案：min(f[root][i])
```

**优化**：枚举上界 $K$ 取 $\log_2 n + 1$（通常取 15 即可），复杂度 $O(nK^2)$。

**C++ 代码**：
```cpp
const int N = 10005, K = 15, INF = 0x3f3f3f3f;
vector<int> e[N];
int f[N][K + 1], n;

void dfs(int u, int fa) {
    for (int i = 1; i <= K; i++) f[u][i] = i; // 初始化：单节点权值
    for (int v : e[u]) {
        if (v == fa) continue;
        dfs(v, u);
        for (int i = 1; i <= K; i++) {
            int mn = INF;
            for (int j = 1; j <= K; j++)
                if (j != i) mn = min(mn, f[v][j]);
            f[u][i] += mn;
        }
    }
}
// ans = min(f[1][i]) for i = 1..K
```

**核心启示**：
- 状态维度的"枚举上界"可以通过数学分析大幅缩减
- 树上相邻不同值 → 至多 $\log n + 1$ 种就够了（直觉：二叉树层层交替）
- 和图着色问题的联系：树是二部图，2-着色足够，但最小权值和需要更多种

### 题目 12：小Q的棋盘（洛谷 P3698 [CQOI2017]）

> 一棵 $n$ 个节点的树（根为 1），从节点 1 出发，最多走 $N$ 步（每步走一条边，可以回头），最多能经过多少个不同节点？

**分析**：经典的**树上有限步数遍历**问题。关键洞察是区分"最终停留位置"的两种状态。

**状态设计**：
```
dp[u][i][0] = 从节点u出发，用恰好i步，最终停留在u的某个子树中，能访问的最大节点数
dp[u][i][1] = 从节点u出发，用恰好i步，最终回到节点u，能访问的最大节点数
```

**转移分析**：对于节点 $u$ 的每个子节点 $v$，有三种策略：
1. **往返子树 $v$**：用 $j+2$ 步（去1步+回1步+在 $v$ 子树用 $j$ 步），剩余步数继续探索
2. **去子树 $v$ 不回头**：用 $j+1$ 步直接到 $v$ 的子树并停留
3. **组合策略**：先处理部分子树，再去另一个子树

**转移方程**：
```
// 往返v子树，剩余步数继续在u探索
dp[u][i][1] = max(dp[u][i][1], dp[u][i-(j+2)][1] + dp[v][j][1])

// 先在u用一些步数，再去v子树不回来
dp[u][i][0] = max(dp[u][i][0], dp[u][i-(j+2)][0] + dp[v][j][1])

// 直接去v子树不回来
dp[u][i][0] = max(dp[u][i][0], dp[u][i-(j+1)][1] + dp[v][j][0])
```

**C++ 代码**：
```cpp
vector<int> adj[110];
int dp[110][110][2];
int n, m;

void dfs(int u, int fa) {
    for (int i = 0; i <= m; i++) {
        dp[u][i][0] = dp[u][i][1] = 1;
    }

    for (int v : adj[u]) {
        if (v == fa) continue;
        dfs(v, u);
        for (int i = m; i >= 0; i--) {
            for (int j = 0; j < i; j++) {
                if (i - j - 2 >= 0) {
                    dp[u][i][1] = max(dp[u][i][1], dp[u][i - (j + 2)][1] + dp[v][j][1]);
                    dp[u][i][0] = max(dp[u][i][0], dp[u][i - (j + 2)][0] + dp[v][j][1]);
                }
                dp[u][i][0] = max(dp[u][i][0], dp[u][i - (j + 1)][1] + dp[v][j][0]);
            }
        }
    }
}
// ans = max(dp[0][m][1], dp[0][m][0])
```

**核心启示**：
- **状态设计的精髓**：通过区分"最终位置"，精确处理有限步数约束
- **转移的完整性**：考虑所有可能的探索策略（往返、单向、组合）
- **树形DP经典技巧**：倒序枚举避免状态污染，逐子树合并信息
- 相比贪心方法，DP能处理更复杂的约束和优化目标

### 题目 13：树中距离之和（LeetCode 834）

> 给一棵 $n$ 个节点的无根树，对每个节点 $i$，求所有其他节点到 $i$ 的距离之和。

**暴力做法**：对每个节点做一次 DFS → $O(n^2)$，太慢！

**换根 DP 核心思想**：
1. 第一遍 DFS（自底向上）：以节点 0 为根，计算 $dp[0]$ 和每个子树的大小 $sz[u]$
2. 第二遍 DFS（自顶向下）：把根从 $u$ "换"到其子节点 $v$，利用已知信息 $O(1)$ 推出 $dp[v]$

```
第一遍：
  sz[u] = 1 + Σ sz[v]
  dp[u] = Σ (dp[v] + sz[v])    // 子树中每个节点到 u 的距离 = 到 v 的距离 + 1

第二遍（换根递推）：
  dp[v] = dp[u] - sz[v] + (n - sz[v])
  
  含义：u→v 换根时
    - v 的子树中 sz[v] 个节点，距离各减 1 → 总距离减 sz[v]
    - 非 v 子树的 (n - sz[v]) 个节点，距离各加 1 → 总距离加 (n - sz[v])
```

**C++ 代码**：
```cpp
int n;
vector<int> adj[MAXN];
long long dp[MAXN];
int sz[MAXN];

// 第一遍：自底向上
void dfs1(int u, int fa) {
    sz[u] = 1;
    dp[u] = 0;
    for (int v : adj[u]) {
        if (v == fa) continue;
        dfs1(v, u);
        sz[u] += sz[v];
        dp[u] += dp[v] + sz[v];
    }
}

// 第二遍：自顶向下换根
void dfs2(int u, int fa) {
    for (int v : adj[u]) {
        if (v == fa) continue;
        dp[v] = dp[u] - sz[v] + (n - sz[v]);
        dfs2(v, u);
    }
}

// main: dfs1(0, -1); dfs2(0, -1);
```

**核心启示**：
- **两遍 DFS** 是换根 DP 的标准范式
- 第一遍收集子树信息，第二遍传播全局信息
- 换根公式的推导关键：想清楚"哪些节点距离+1，哪些-1"

### 题目 14：树的重心

> 找一个节点，删除它后，最大连通分量的节点数最小。

```
状态定义：
  sz[u] = u 的子树大小
  maxPart[u] = 删除 u 后最大连通分量大小
             = max(n - sz[u], max(sz[v] for v in children(u)))

答案：使 maxPart[u] 最小的节点 u
```

**C++ 代码**：
```cpp
int centroid, minMax = INT_MAX;

void dfs(int u, int fa) {
    sz[u] = 1;
    int maxChild = 0;
    for (int v : adj[u]) {
        if (v == fa) continue;
        dfs(v, u);
        sz[u] += sz[v];
        maxChild = max(maxChild, sz[v]);
    }
    int maxPart = max(maxChild, n - sz[u]);  // "上方"的部分
    if (maxPart < minMax) {
        minMax = maxPart;
        centroid = u;
    }
}
```

**核心启示**：
- $n - sz[u]$ 巧妙地表示了"u 上方的节点数"
- 不需要换根也能看到全局信息，因为 $sz[u]$ + 上方 = $n$
- 树的重心性质：最大分量 $\leq n/2$

### 题目 15：Nearby Cows（洛谷 P3047 [USACO12FEB]）

> 给一棵 $n$ 个节点的带权树（节点权值 $c[u]$），给定距离 $K$。对每个节点 $u$，求距离 $u$ 不超过 $K$ 的所有节点的权值和。

**分析**：这是 **二维换根 DP**，在经典换根 DP 的基础上增加了距离维度。

```
第一遍 DFS（自底向上）：
  down[u][j] = u 子树内距离 u 恰好为 j 的节点权值和
  转移：down[u][j] = Σ down[v][j-1]   (v 是 u 的子节点)
  初始：down[u][0] = c[u]

第二遍 DFS（换根，自顶向下）：
  up[u][j] = u 子树外距离 u 恰好为 j 的节点权值和
  转移：up[v][j] = up[u][j-1] + down[u][j-1] - down[v][j-2]
  （从 u 的全局信息中去掉 v 子树的贡献）

最终答案：
  ans[u] = Σ (down[u][j] + up[u][j])  for j = 0 to K
```

**C++ 代码**：
```cpp
const int N = 100005;
vector<int> adj[N];
int c[N], n, K;
long long dp[N][21]; // dp[u][j] = 距离 u 恰好为 j 的权值和（合并后）

void dfs1(int u, int fa) {
    dp[u][0] = c[u];
    for (int v : adj[u]) {
        if (v == fa) continue;
        dfs1(v, u);
        for (int j = 1; j <= K; j++)
            dp[u][j] += dp[v][j - 1];
    }
}

void dfs2(int u, int fa) {
    for (int v : adj[u]) {
        if (v == fa) continue;
        // 把 u 的信息（去掉 v 子树的部分）传给 v
        for (int j = K; j >= 1; j--) {
            dp[v][j] += dp[u][j - 1] - (j >= 2 ? dp[v][j - 2] : 0);
        }
        dfs2(v, u);
    }
}
// dfs1(1, 0); dfs2(1, 0); ans[u] = Σ dp[u][j] for j=0..K
```

**核心启示**：
- 换根 DP 可以推广到二维：增加距离维度 j
- 换根时要"去掉子树贡献"，这是换根 DP 的核心操作
- 注意 $K \leq 20$，否则复杂度会爆

### 题目 16：树的中心（换根 DP 求最远距离最小的节点）

> 给定一棵 $n$ 个节点的带权树。定义节点 $u$ 的"偏心距" $ecc(u) = \max(dist(u, v))$（到最远节点的距离）。  
> 求 **偏心距最小** 的节点（即树的中心），以及该最小偏心距。

**辨析**：树的中心 $\neq$ 树的重心
- **重心**：最小化最大子树大小（删根后），用于分治
- **中心**：最小化到最远节点的距离，用于选址/最优服务点

**分析**：对每个节点 $u$，需要求 $u$ 到全树最远节点的距离。只看子树内是不够的，还需要"子树外"的最远距离 → 换根 DP。

```
第一遍 DFS（自底向上）：
  d1[u] = u 子树内到 u 的最远距离（最长链）
  d2[u] = u 子树内到 u 的次远距离（经过不同子节点的次长链）
  ch1[u] = d1[u] 经过的子节点编号（用于换根时判断）

  转移：遍历 u 的子节点 v，边权 w
    如果 d1[v] + w > d1[u]：
      d2[u] = d1[u]
      d1[u] = d1[v] + w
      ch1[u] = v
    否则如果 d1[v] + w > d2[u]：
      d2[u] = d1[v] + w

第二遍 DFS（换根，自顶向下）：
  up[u] = u 往子树外方向走的最远距离

  转移：对 u 的子节点 v，边权 w
    如果 ch1[u] == v：
      up[v] = max(up[u], d2[u]) + w    // u 的最长链经过 v，用次长链
    否则：
      up[v] = max(up[u], d1[u]) + w    // u 的最长链不经过 v，用最长链

最终答案：
  ecc(u) = max(d1[u], up[u])
  树的中心 = argmin(ecc(u))
```

**C++ 代码**：
```cpp
const int N = 100005;
struct Edge { int to, w; };
vector<Edge> adj[N];
long long d1[N], d2[N], up[N]; // 最长链、次长链、向上最远
int ch1[N]; // d1 经过的子节点
int n;

void dfs1(int u, int fa) {
    d1[u] = d2[u] = 0;
    ch1[u] = 0;
    for (auto [v, w] : adj[u]) {
        if (v == fa) continue;
        dfs1(v, u);
        long long dist = d1[v] + w;
        if (dist > d1[u]) {
            d2[u] = d1[u];
            d1[u] = dist;
            ch1[u] = v;
        } else if (dist > d2[u]) {
            d2[u] = dist;
        }
    }
}

void dfs2(int u, int fa) {
    for (auto [v, w] : adj[u]) {
        if (v == fa) continue;
        if (ch1[u] == v)
            up[v] = max(up[u], d2[u]) + w; // 最长链经过 v，用次长
        else
            up[v] = max(up[u], d1[u]) + w; // 最长链不经过 v，用最长
        dfs2(v, u);
    }
}
// up[1]=0; dfs1(1,0); dfs2(1,0); center = argmin max(d1[u], up[u])
```

**核心启示**：
- 换根 DP 的经典应用：子树内信息（$d1, d2$）+ 子树外信息（$up$）
- **记录次长链 $d2$** 是关键技巧：换根时如果最长链经过当前子节点，就只能用次长链
- 树的中心一定在树的直径的中点位置（可以用两次 BFS 验证）
- 与树的重心对比学习，两者都是"最优节点"问题，但优化目标不同

### 题目 17：积蓄程度（AcWing 1073）

> 一棵 $n$ 个节点的树，每条边有容量。选一个节点为源点，水从源点流向所有叶子（度为 1 的非源节点）。每条边流量不超过容量，内部节点流量守恒。求选哪个源点使总流量最大。

**分析**：换根 DP 的经典应用。与"树的中心"类似的两遍 DFS，但操作从"距离取最值"变为"流量受容量约束的求和"。

```
核心观察：
1. 叶节点（汇点）吸收能力无限 → 对父节点的贡献 = 边容量
2. 内部节点 v 对父节点的贡献 = min(边容量, d[v])（受子树容量限制）
3. 换根时减去 v 对 u 的贡献，得到 v 的上流量

状态定义：
  d[u] = 以 u 为根时，u 子树内的最大下流量
  f[u] = u 作为源点时的最大总流量（下流量 + 上流量）

答案：max(f[u]) over all u
```

**C++ 代码**：
```cpp
struct Edge { int to; long long w; };
vector<Edge> adj[N];
long long d[N], f[N];
int deg[N]; // 每个节点在原树中的度数

void dfs1(int u, int fa) { // 自底向上：计算下流量
    d[u] = 0;
    for (auto [v, w] : adj[u]) {
        if (v == fa) continue;
        dfs1(v, u);
        d[u] += (deg[v] == 1) ? w : min(w, d[v]);
        // 叶节点贡献边容量，内部节点贡献 min(边容量, 子树容量)
    }
}

void dfs2(int u, int fa) { // 自顶向下：换根求全流量
    for (auto [v, w] : adj[u]) {
        if (v == fa) continue;
        long long used = (deg[v] == 1) ? w : min(w, d[v]); // u→v 的实际流量
        if (deg[u] == 1) // u 是叶节点，换根后变为汇点
            f[v] = d[v] + w;
        else
            f[v] = d[v] + min(w, f[u] - used);
        dfs2(v, u);
    }
}
// deg[u]=度数; dfs1(1,0); f[1]=d[1]; dfs2(1,0); ans = max(f[u])
```

**核心启示**：
- 换根 DP 不仅适用于距离问题，**流量/容量约束**同样适用
- 叶节点特殊处理（贡献 = 边容量）：叶子是汇点，吸收能力无限
- 换根时"减去子树贡献"的通用技巧：$f[u] - \min(w, effective\_d[v])$
- 与"树的中心"对比学习：同一换根框架，不同合并操作

### 题目 18：括号树（洛谷 P5658 [CSP-S 2019]）

> 给一棵 $n$ 个节点的有根树，每个节点上有一个括号（`(` 或 `)`）。  
> 对每个节点 $u$，考虑根到 $u$ 的路径上的括号序列，求其中有多少个 **非空合法括号子串**。  
> 输出所有节点答案的异或和。

**分析**：树上 DFS + 栈维护匹配。

```
思路：
1. DFS 遍历树，维护一个栈记录未匹配的左括号位置
2. 到达节点 u 时：
   - 如果 u 是 '('：入栈
   - 如果 u 是 ')' 且栈非空：弹出栈顶（匹配成功）
     设匹配到的左括号在节点 p 上，则 u 的答案增量 = f[p的父亲] + 1
3. f[u] = 从路径上以 u 结尾的合法括号串的个数
4. 回溯时恢复栈状态
```

**C++ 代码**：
```cpp
const int N = 500005;
vector<int> adj[N];
char ch[N];
int fa[N], n;
long long f[N], ans;
stack<int> stk; // 记录未匹配左括号的节点编号

void dfs(int u) {
    int pushed = 0; // 标记本层对栈的操作
    int old_top = stk.empty() ? 0 : stk.top();

    if (ch[u] == '(') {
        stk.push(u);
        pushed = 1;
        f[u] = 0;
    } else { // ')'
        if (!stk.empty()) {
            int p = stk.top(); stk.pop();
            pushed = -1;
            f[u] = f[fa[p]] + 1; // p 的父亲位置的答案 + 1
        } else {
            f[u] = 0;
        }
    }

    ans ^= (long long)u * f[u]; // 题目要求异或和
    for (int v : adj[u]) dfs(v);

    // 回溯恢复栈
    if (pushed == 1) stk.pop();
    else if (pushed == -1) stk.push(old_top); // 恢复弹出的左括号
}
// 调用 dfs(1)，答案为 ans（异或和）
```

**核心启示**：
- 树上路径问题 → DFS + 维护路径信息
- 括号匹配的"栈"可以和 DFS 结合，回溯时恢复状态
- $f[u]$ 的含义精妙：以 $u$ 结尾的合法子串数 = 匹配点父亲处的答案 + 1

### 题目 19：选课问题（洛谷 P2014 / 树上背包）

> $n$ 门课程形成树形依赖关系，每门课有学分。选 $m$ 门课（选了课必须先选其先修课），求最大学分。

**状态定义**：
```
dp[u][j] = 以 u 为根的子树中，选 j 门课（包含 u 自身）能获得的最大学分
```

**转移**：对 $u$ 的每个子节点 $v$ 做一次"分组背包"合并：
```
// 逐个合并子树
for (int v : children(u)):
    for j = 当前总容量 downto 1:
        for k = 1 to sz[v]:  // 从 v 的子树选 k 门
            dp[u][j+k] = max(dp[u][j+k], dp[u][j] + dp[v][k])
```

**C++ 代码**：
```cpp
int dp[MAXN][MAXM];

int dfs(int u, int fa) {
    dp[u][1] = score[u];
    int sz = 1;  // u 子树中已合并的节点数（初始只有 u 自己）
    for (int v : adj[u]) {
        if (v == fa) continue;
        int sv = dfs(v, u);  // sv = v 的子树大小
        sz += sv;
        // j 上界：已合并总大小 sz，不超过 m
        // k 上界：从 v 子树最多选 sv 个，且不超过 j
        for (int j = min(sz, m); j >= 1; j--)
            for (int k = 1; k <= min(sv, j); k++)
                dp[u][j] = max(dp[u][j], dp[u][j - k] + dp[v][k]);
    }
    return sz;
}
```

**核心启示**：
- 树上背包 = 树形 DP + 分组背包
- $sz$：已合并的子树总大小，控制 $j$ 的枚举上界，避免无效循环
- $sv$：当前子树 $v$ 的大小，控制 $k$ 的枚举上界
- 时间复杂度 $O(n \cdot m^2)$，通过 $sz$ 和 $sv$ 限制循环范围可优化到 $O(n \cdot m)$

### 题目 20：二叉树中的最大 BST 子树（LeetCode 333）

> 给定一棵二叉树，找到其中最大的二叉搜索树（BST）子树，返回该子树的节点数。

**状态设计**：每个节点返回多个信息。

```
对每个节点 u，DFS 返回：
  (isBST, size, minVal, maxVal)

转移：
  u 的子树是 BST $\iff$ 左子树是 BST $\wedge$ 右子树是 BST $\wedge$ leftMax < u.val < rightMin
  
  如果是 BST：size = leftSize + rightSize + 1
  如果不是：这个子树不构成 BST，但记录子问题中最大的 BST
```

**C++ 代码**：
```cpp
int ans = 0;

// 返回 {isBST, size, min, max}
tuple<bool, int, int, int> dfs(TreeNode* u) {
    if (!u) return {true, 0, INT_MAX, INT_MIN};
    
    auto [lBST, lSize, lMin, lMax] = dfs(u->left);
    auto [rBST, rSize, rMin, rMax] = dfs(u->right);
    
    if (lBST && rBST && lMax < u->val && u->val < rMin) {
        int sz = lSize + rSize + 1;
        ans = max(ans, sz);
        return {true, sz, min(lMin, u->val), max(rMax, u->val)};
    }
    return {false, 0, 0, 0};
}
```

**核心启示**：
- 有时需要返回 **多维信息**（不只是一个数）
- 元组/结构体是处理多返回值的好方式
- 合法性检查 + 信息合并可以一起做

### 题目 21：有线电视网（洛谷 P1273）

> 一棵有根树，内部节点是中继站，叶子节点是用户。每条边有费用（正整数），每个用户有愿付的钱数。  
> 选择若干用户接入网络，使得 **总收入 - 总费用 $\geq$ 0** 的前提下，接入的用户数最多。

**分析**：树上分组背包。和 P2014（选课）思路类似，但多了边权费用。

```
状态定义：dp[u][j] = u 子树内选 j 个用户接入时的最大利润（收入 - 费用）
转移方程（子树合并，分组背包）：
  dp[u][j] = max over 分配方案 of (dp[u][j-k] + dp[v][k] - w(u,v))
  其中 w(u,v) 是边 u→v 的费用，k 个用户分配给子树 v
初始化：
  叶节点 u：dp[u][1] = val[u]（用户愿付的钱），dp[u][0] = 0
答案：max(j) such that dp[root][j] >= 0
```

**C++ 代码**：
```cpp
const int N = 3005, INF = 0x3f3f3f3f;
struct Edge { int to, w; };
vector<Edge> adj[N];
int dp[N][N], sz[N], val[N];
int n, m;

void dfs(int u) {
    if (u > n - m) { // 叶节点（用户）
        sz[u] = 1;
        dp[u][0] = 0;
        dp[u][1] = val[u];
        return;
    }
    sz[u] = 0;
    dp[u][0] = 0;
    for (auto [v, w] : adj[u]) {
        dfs(v);
        for (int j = sz[u]; j >= 0; j--)
            for (int k = sz[v]; k >= 0; k--)
                if (dp[u][j] > -INF && dp[v][k] > -INF)
                    dp[u][j + k] = max(dp[u][j + k], dp[u][j] + dp[v][k] - w);
        sz[u] += sz[v];
    }
}
// dp 初始化为 -INF; dfs(1); ans = max j where dp[1][j] >= 0
```

**核心启示**：
- 和 P2014（选课）的树上背包框架完全相同
- 区别在于：边有费用，合并时需要减去边权
- 答案条件是 $dp[root][j] \geq 0$，而不是求最大值

### 题目 22：树上染色（洛谷 P3177 [HAOI2015]）

> 给定一棵 $n$ 个节点的带权无向树，选 $m$ 个点染黑，其余染白。  
> 求 **黑点对距离之和 + 白点对距离之和** 的最大值。

**分析**：核心突破点是 **按边计算贡献**，而非按点对。

```
关键转化：
  对于边 (u, v)（v 是 u 的子节点，边权 w）：
  设 v 子树内有 k 个黑点，sz[v] 个节点
  - 黑点对贡献 = w × k × (m - k)
  - 白点对贡献 = w × (sz[v] - k) × (n - m - sz[v] + k)

状态定义：dp[u][j] = u 子树内选 j 个黑点时的最大贡献
转移方程：
  dp[u][j] = max over k of {
    dp[u][j-k] + dp[v][k] +
    w × k × (m - k) +
    w × (sz[v] - k) × (n - m - sz[v] + k)
  }
```

**C++ 代码**：
```cpp
const int N = 2005;
const long long NEG = -1e18;
struct Edge { int to; long long w; };
vector<Edge> adj[N];
long long dp[N][N];
int sz[N], n, m;

void dfs(int u, int fa) {
    sz[u] = 1;
    dp[u][0] = dp[u][1] = 0;
    for (auto [v, w] : adj[u]) {
        if (v == fa) continue;
        dfs(v, u);
        vector<long long> tmp(sz[u] + sz[v] + 1, NEG);
        for (int j = min(sz[u], m); j >= 0; j--)
            for (int k = min(sz[v], m - j); k >= 0; k--) {
                if (dp[u][j] == NEG || dp[v][k] == NEG) continue;
                long long val = w * k * (m - k) +
                    w * (sz[v] - k) * ((long long)(n - m) - sz[v] + k);
                tmp[j + k] = max(tmp[j + k], dp[u][j] + dp[v][k] + val);
            }
        sz[u] += sz[v];
        for (int j = 0; j <= min(sz[u], m); j++) dp[u][j] = tmp[j];
    }
}
// dp 初始化为 -1e18; dfs(1, 0); ans = dp[1][m]
```

**核心启示**：
- **边贡献分析** 是树上计数/求和问题的万能武器
- 将"所有点对距离"拆解成"每条边被多少对经过"
- 树上背包 + 贡献计算 = 高阶组合，注意用临时数组防止覆盖

### 题目 23：骑士（洛谷 P2607 [ZJOI2008]）

> $n$ 个骑士各有战斗力 $w[i]$，每个骑士恰好厌恶另一个骑士（形成基环树森林）。  
> 选出一些骑士，要求互相厌恶的两人不能同时选，求战斗力之和最大值。

**分析**：这是 **基环树 DP** 的经典题。图的结构是：每个连通分量恰好有 $n$ 条边和 $n$ 个节点（基环树 = 树 + 一条多余边）。

```
核心思路：
1. 找到环上的一条边 (a, b)
2. 断开这条边，分两种情况做树形 DP：
   情况 1：强制不选 a → 以 a 为根做"没有上司的舞会"
   情况 2：强制不选 b → 以 b 为根做"没有上司的舞会"
3. 两种情况取最大值
```

**C++ 代码**：
```cpp
const int N = 1000005;
struct Edge { int to, next; } e[N * 2];
int head[N], cnt;
long long w[N], f[N][2];
int hate[N]; // hate[i] = i 厌恶的人
bool vis[N], in_ring[N];
int n;
long long ans;

void add(int u, int v) {
    e[++cnt] = {v, head[u]}; head[u] = cnt;
}

int ring_u, ring_v; // 断开的边

void dfs(int u, int fa, int ban) {
    f[u][0] = 0;
    f[u][1] = w[u];
    for (int i = head[u]; i; i = e[i].next) {
        int v = e[i].to;
        if (v == fa) continue;
        if ((u == ring_u && v == ring_v) || (u == ring_v && v == ring_u))
            continue; // 跳过断开的边
        dfs(v, u, ban);
        f[u][0] += max(f[v][0], f[v][1]);
        f[u][1] += f[v][0];
    }
    if (u == ban) f[u][1] = -1e18; // 强制不选
}

void solve(int root) {
    // 找环上一条边
    // 使用 DFS 或直接用 hate 关系找环
    // 简化：用并查集或拓扑排序找环
    // 这里用标记法
    ring_u = root; ring_v = hate[root]; // 断开 root -> hate[root] 的边

    dfs(root, 0, ring_u); // 强制不选 ring_u
    long long res = max(f[root][0], f[root][1]);
    dfs(root, 0, ring_v); // 强制不选 ring_v
    res = max(res, max(f[root][0], f[root][1]));
    ans += res;
}
// 对每个基环树连通分量调用 solve(root)，累加到 ans
```

**核心启示**：
- **基环树 = 树 + 一条边**：断掉环上一条边变成树
- 断边后分两种情况做树形 DP，取最优
- 基环树上的问题几乎都能转化为"断环为链/树 + 枚举约束"

### 题目 24：潜入行动（洛谷 P4516 [JSOI2018]）

> 给一棵 $n$ 个节点的树，放恰好 $k$ 个监听装置在不同节点上。  
> 监听装置可以监听所有邻居，但 **不能监听自身所在节点**。  
> 要求每个节点都被监听到。求方案数（模 $10^9+7$）。

**分析**：四维状态的树形背包计数 DP。

```
状态定义：dp[u][i][a][b]
  u：以 u 为根的子树
  i：子树内放了 i 个监听器
  a：u 是否放了监听器（0/1）
  b：u 是否被监听到（0/1）

初始化（叶节点 u）：
  dp[u][0][0][0] = 1  （不放，没被监听）
  dp[u][1][1][0] = 1  （放了，但自己没被监听）

转移（合并子节点 v 到 u）：
  需要枚举 u 和 v 的 (a, b) 状态组合，同时累加 i 维度（背包卷积）。
  关键约束：
  - 如果 v 放了监听器(a_v=1)，则 u 被监听(b_u 可以变为 1)
  - 如果 u 放了监听器(a_u=1)，则 v 被监听(b_v 必须为 1，但不是由 u 直接保证)
  - 必须保证 v 被监听（b_v=1），否则 v 不合法

最终答案：dp[root][k][0][1] + dp[root][k][1][1]
  （根节点必须被监听，装或不装监听器均可）
```

**转移方程详细**（合并子节点 v 到 u，用临时数组 tmp）：

```
// u不放，u未被监听 + v的贡献
tmp[i+j][0][0] += dp[u][i][0][0] * (dp[v][j][0][1])
  // v不放但已被监听 → u 依然未被监听

// u不放，u被监听 + v的贡献
tmp[i+j][0][1] += dp[u][i][0][1] * (dp[v][j][0][1] + dp[v][j][1][1])
  // u 已被监听，v 合法即可
tmp[i+j][0][1] += dp[u][i][0][0] * dp[v][j][1][1]
  // u 之前未被监听，v 放了监听器且 v 合法 → u 被监听

// u放，u未被监听（自己监听不了自己）
tmp[i+j][1][0] += dp[u][i][1][0] * (dp[v][j][0][1] + dp[v][j][1][1])
  // v 合法即可（u 放了 → v 一定被监听，但这里 v 的 b=1 已保证）

// u放，u被监听
tmp[i+j][1][1] += dp[u][i][1][1] * (dp[v][j][0][1] + dp[v][j][1][1])
tmp[i+j][1][1] += dp[u][i][1][0] * dp[v][j][1][1]
  // u 之前未被监听，v 放了监听器 → u 被监听
```

**C++ 代码**：
```cpp
const int N = 100005, MOD = 1e9 + 7;
vector<int> adj[N];
long long dp[N][105][2][2]; // dp[u][i][a][b]
int sz[N], n, K;

void dfs(int u, int fa) {
    sz[u] = 1;
    dp[u][0][0][0] = dp[u][1][1][0] = 1;
    for (int v : adj[u]) {
        if (v == fa) continue;
        dfs(v, u);
        // 临时数组 tmp
        long long tmp[sz[u]+sz[v]+1][2][2];
        memset(tmp, 0, sizeof(long long)*(sz[u]+sz[v]+1)*4);
        for (int i = min(sz[u], K); i >= 0; i--)
            for (int j = min(sz[v], K-i); j >= 0; j--) {
                // 按上述转移方程合并
                // ... (完整实现较长，此处省略重复代码)
                auto &t = tmp[i+j];
                auto &du = dp[u][i], &dv = dp[v][j];
                // 合并所有 (a,b) 组合
                (t[0][0] += du[0][0] % MOD * (dv[0][1]) % MOD) %= MOD;
                (t[0][1] += du[0][1] % MOD * ((dv[0][1]+dv[1][1])%MOD) % MOD) %= MOD;
                (t[0][1] += du[0][0] % MOD * (dv[1][1]) % MOD) %= MOD;
                (t[1][0] += du[1][0] % MOD * ((dv[0][1]+dv[1][1])%MOD) % MOD) %= MOD;
                (t[1][1] += du[1][1] % MOD * ((dv[0][1]+dv[1][1])%MOD) % MOD) %= MOD;
                (t[1][1] += du[1][0] % MOD * (dv[1][1]) % MOD) %= MOD;
            }
        sz[u] += sz[v];
        for (int i = 0; i <= min(sz[u], K); i++)
            for (int a = 0; a < 2; a++)
                for (int b = 0; b < 2; b++)
                    dp[u][i][a][b] = tmp[i][a][b];
    }
}
// dfs(1, 0); ans = (dp[1][K][0][1] + dp[1][K][1][1]) % MOD
```

**核心启示**：
- 状态设计的精确性至关重要：4 维状态 (节点, 个数, 放/不放, 被监听/未监听)
- "不能监听自己"这个条件使得 a 和 b 必须分开
- 背包合并时，转移方程要逐项推导，不能含糊
- 计数 DP 中合并用乘法（方案数相乘），优化用加法（取模累加）

## 四、常用思考方法总结

### 4.1 五步思考法

```
┌─────────────────────────────────────────────────┐
│  Step 1: 识别子问题                              │
│  → "以 u 为根的子树"是否构成独立子问题？           │
├─────────────────────────────────────────────────┤
│  Step 2: 设计状态                                │
│  → dp[u][状态] 中"状态"是什么？选/不选？容量？     │
├─────────────────────────────────────────────────┤
│  Step 3: 推导转移方程                            │
│  → 父节点的答案如何从子节点的答案合并得到？         │
├─────────────────────────────────────────────────┤
│  Step 4: 确定边界条件                            │
│  → 叶节点/空节点的 DP 值是什么？                  │
├─────────────────────────────────────────────────┤
│  Step 5: 确定答案位置                            │
│  → 答案是 dp[root]，还是全局变量，还是需要换根？   │
└─────────────────────────────────────────────────┘
```

### 4.2 常见状态设计模式

| 模式 | 状态维度 | 典型问题 |
|------|---------|---------|
| 单状态 | $dp[u]$ | 子树大小、最大深度 |
| 选/不选 | $dp[u][0/1]$ | 最大独立集、最小点覆盖 |
| 三状态 | $dp[u][0/1/2]$ | 最小支配集、监控二叉树 |
| 背包型 | $dp[u][j]$ | 选课、树上背包 |
| 多返回值 | $(val_1, val_2, ...)$ | 最大BST子树、直径 |

### 4.3 转移方程的三种模式

**模式一：累加型**
```
dp[u] = 初始值 + Σ f(dp[v])
例：dp[u][1] = w[u] + Σ dp[v][0]
```

**模式二：取极值型**
```
dp[u] = 初始值 + max/min over children
例：depth[u] = 1 + max(depth[v])
```

**模式三：合并型（背包）**
```
逐个子树合并，类似背包问题
例：dp[u][j] = max(dp[u][j], dp[u][j-k] + dp[v][k])
```

## 五、通用代码结构模板

### 模板 1：基础树形 DP（选/不选）

```cpp
void dfs(int u, int fa) {
    dp[u][0] = 0;       // 不选 u
    dp[u][1] = w[u];    // 选 u
    for (int v : adj[u]) {
        if (v == fa) continue;
        dfs(v, u);
        dp[u][0] += max(dp[v][0], dp[v][1]);
        dp[u][1] += dp[v][0];
    }
}
```

### 模板 2：路径型（返回单链，更新全局）

```cpp
int ans = 0;
int dfs(int u, int fa) {
    int best = 0;
    for (int v : adj[u]) {
        if (v == fa) continue;
        int d = dfs(v, u) + 1;
        ans = max(ans, best + d);  // 合并两条链
        best = max(best, d);       // 保留最优单链
    }
    return best;
}
```

### 模板 3：换根 DP（两遍 DFS）

```cpp
void dfs1(int u, int fa) {   // 第一遍：自底向上
    sz[u] = 1; dp[u] = 0;
    for (int v : adj[u]) {
        if (v == fa) continue;
        dfs1(v, u);
        sz[u] += sz[v];
        dp[u] += dp[v] + sz[v];
    }
}

void dfs2(int u, int fa) {   // 第二遍：自顶向下
    for (int v : adj[u]) {
        if (v == fa) continue;
        dp[v] = dp[u] - sz[v] + (n - sz[v]);
        dfs2(v, u);
    }
}
```

### 模板 4：树上背包

```cpp
int dfs(int u, int fa) {
    dp[u][1] = val[u];
    int sz = 1;
    for (int v : adj[u]) {
        if (v == fa) continue;
        int sv = dfs(v, u);
        sz += sv;
        for (int j = min(sz, m); j >= 1; j--)
            for (int k = 1; k <= min(sv, j); k++)
                dp[u][j] = max(dp[u][j], dp[u][j-k] + dp[v][k]);
    }
    return sz;
}
```

## 七、终极总结与归纳

### 7.1 树形 DP 的本质

> **树形 DP = 树的递归结构 + 最优子结构 + DFS 遍历**

树形 DP 之所以自然，是因为树本身就是递归定义的数据结构。
每棵子树都是一个独立的子问题，而父节点只需要 **合并** 子节点的答案。

### 7.2 六大核心要点

```
1. 状态设计是灵魂
   → 能用 1 维别用 2 维，能用 2 维别用 3 维
   → 状态要完整：能唯一确定子问题的解

2. 转移方程是骨架
   → 父节点从子节点合并信息
   → 三种模式：累加、取极值、背包合并

3. DFS 是引擎
   → 后序遍历 = 自底向上
   → 前序遍历 = 自顶向下（换根的第二遍）

4. 返回值 ≠ 答案
   → 递归返回的可能是"部分信息"（如单链）
   → 真正的答案可能需要全局变量

5. 边界条件是基石
   → 叶节点、空节点的初始值必须仔细设计
   → 特别是涉及 min/max 时，INF 和 -INF 的使用

6. 换根是扩展
   → 需要每个节点答案时 → 两遍 DFS
   → 第二遍的关键：推导"换根公式"
```

### 7.5 一句话记忆法

> 🌲 **树形 DP 口诀**：
> 
> **子树为状态，DFS 做遍历；**
> **选与不选分两类，路径要拆单双链；**
> **换根两遍全搞定，背包逐子来合并；**
> **多色枚举看约束，边贡献拆点对算；**
> **基环断边做两遍，四维状态细推转。**
