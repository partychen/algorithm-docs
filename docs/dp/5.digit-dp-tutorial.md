## 一、什么是数位 DP

**数位 DP**，顾名思义，就是把一个整数拆成一位一位的数字来做动态规划。

它最经典的题型只有一句话：

> **求区间 `[L, R]` 内，有多少个整数满足某种“数位性质”。**

这里的“数位性质”通常只和数字本身有关，例如：
- 不含数字 `9`
- 数位和等于某个值
- 相邻数位之差至少为 `2`
- 不含重复数字
- 能被 `13` 整除
- 同时满足“出现子串 + 取模 + 计数”这类组合条件

为什么不能暴力枚举？因为很多题目的范围会直接给到：
- `R <= 10^18`
- 甚至 `R` 以字符串形式给出，长度达到 `22` 位、`100` 位

这时逐个检查每个数显然不现实。数位 DP 的核心价值就在于：

1. **不去枚举每个整数，而是枚举“每一位填什么”**
2. **利用上界约束 `tight` 剪枝**
3. **把重复子问题记忆化**
4. **把区间问题转成前缀问题**

### 区间计数的标准转化

绝大多数数位 DP 都会先做这一步：

```text
设 f(n) = [0, n] 中满足条件的整数个数
那么答案就是：
  [L, R] 的答案 = f(R) - f(L - 1)
```

这样一来，你只需要解决：

> **如何计算“从 0 到 n”之间的合法数量？**

而这恰恰非常适合“从高位到低位逐位决策”的写法。

### 数位 DP 的本质

你可以把它理解成一种“带上界限制的搜索树压缩”。

例如上界是 `325`，你在第 0 位只能填 `0..3`；
如果这一位填了 `1`，那后面就完全自由；
如果这一位填了 `3`，那下一位还得继续受 `325` 的约束。

于是“前缀是否贴着上界”就成了最重要的信息之一。

### 与其他常见 DP 的对比

| DP 类型 | 常见状态 | 决策对象 | 典型顺序 | 代表问题 |
|---------|----------|----------|----------|----------|
| 线性 DP | `dp[i]` | 第 `i` 个位置怎么选 | 从左到右 | LIS、打家劫舍 |
| 区间 DP | `dp[l][r]` | 区间如何划分 | 按区间长度递增 | 石子合并 |
| 树形 DP | `dp[u][state]` | 子树如何合并 | DFS 后序 | 没有上司的舞会 |
| 背包 DP | `dp[i][j]` / `dp[j]` | 当前物品取不取 | 枚举物品与容量 | 0/1 背包 |
| 数位 DP | `dfs(pos, state, tight, lead)` | 当前位填什么数字 | 从高位到低位 | Windy 数、不要 62 |

### 一句话记住它

> **数位 DP = 按位枚举 + 上界约束 + 状态记录 + 记忆化搜索。**

## 二、核心思想与前置知识

### 2.1 数位 DP 的四个关键概念

做数位 DP 时，几乎所有题都绕不开下面四件事。

#### 1）逐位枚举

我们不是枚举整数，而是枚举每一位填什么。

以 `n = 325` 为例：

```text
n = 325
Position 0: d ∈ [0, 3]
  如果 d < 3：Position 1 就可以取 [0, 9]（已经不贴上界）
  如果 d = 3：Position 1 只能取 [0, 2]（仍然贴上界）
    如果这一位 d < 2：Position 2 就可以取 [0, 9]
    如果这一位 d = 2：Position 2 只能取 [0, 5]（仍然贴上界）
```

这就是数位 DP 搜索树的本质：
**每一层处理一位，每个分支选择当前位数字。**

#### 2）`tight` 约束

`tight` 是数位 DP 里最灵魂的布尔量之一。

它表示：

```text
tight = true  -> 当前前缀和上界 n 的前缀完全相同
tight = false -> 当前前缀已经严格小于上界，后面可自由填 0..9
```

因此当前位的上限写法固定是：

```cpp
int up = tight ? s[pos] - '0' : 9;
```

下一层是否仍然贴边，写法也固定是：

```cpp
bool new_tight = tight && (d == up);
```

这里很多人会写错成 `d == s[pos] - '0'`，严格来说虽然在 `tight=true` 时等价，但写成 `d == up` 更统一、更不容易出错。

#### 3）前导零 `leading zero`

前导零是数位 DP 里最容易把题写错的地方。

例如我们在处理上界 `507` 时，长度固定成 3 位，那么整数 `57` 在搜索树里会被看成：

```text
057
```

这时前两个状态要分清：
- `0` 作为“前导零”时，通常**不参与性质判断**
- 真正进入数字后出现的 `0`，才是这个数的一部分

比如：
- “相邻数字之差至少为 2” —— 前导零不应该和后面的首个非零数字比较
- “数字 0 出现了几次” —— 前导零不能算作出现的 `0`
- “数字必须由给定集合组成” —— 前导零只是占位，不代表真的选了 `0`

所以很多题都要多带一个参数：

```text
lead = true  -> 到当前为止还没放过非零数字
lead = false -> 数字已经开始构造
```

#### 4）记忆化搜索

数位 DP 最常见的实现方式不是“表格 DP”，而是：

> **记忆化搜索 `dfs(pos, state, tight, lead)`**

原因很简单：
- 逐位枚举天然像 DFS
- 有上界限制，递归写法最直观
- 很多题的状态是“当前位置 + 若干条件”，递归更自然

记忆化时有两个常见流派：

1. **只缓存 `tight=false` 的状态**
   - 因为一旦不贴边，后续可选范围就是固定的 `0..9`
   - 这是竞赛中最常见的写法

2. **把 `tight` 也放进记忆化维度**
   - 更统一
   - 但有时空间稍大，且 `tight=true` 的状态通常只会访问很少次

大多数情况下，推荐优先掌握第一种。

### 2.2 万能模板

下面给出一份 **可直接运行的数位 DP 通用骨架**。

这份代码并不是针对某一道具体题，而是把数位 DP 最通用的框架完整展示出来。
你真正需要根据题目修改的，主要是两处：
- `transition(state, d)`：状态如何转移
- `terminal(state, lead)`：到末尾时这个状态贡献什么答案

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXLEN = 20;
const int STATE = 205;

string s;
long long memo[MAXLEN][STATE][2];
bool vis[MAXLEN][STATE][2];
int initState = 0;

int transition(int state, int d) {
    return state + d;  // 按题意修改
}

long long terminal(int state, bool lead) {
    return !lead;      // 按题意修改
}

long long dfs(int pos, int state, bool tight, bool lead) {
    if (pos == (int)s.size()) return terminal(state, lead);
    if (!tight && vis[pos][state][lead]) return memo[pos][state][lead];

    int up = tight ? s[pos] - '0' : 9;
    long long res = 0;

    for (int d = 0; d <= up; d++) {
        bool newLead = lead && d == 0;
        int newState = newLead ? initState : transition(state, d);
        res += dfs(pos + 1, newState, tight && d == up, newLead);
    }

    if (!tight) {
        vis[pos][state][lead] = true;
        memo[pos][state][lead] = res;
    }
    return res;
}

long long solve(long long n) {
    if (n < 0) return 0;
    s = to_string(n);
    memset(vis, 0, sizeof vis);
    return dfs(0, initState, true, true);
}

int main() {
    long long L, R;
    cin >> L >> R;
    cout << solve(R) - solve(L - 1) << '\n';
    return 0;
}
```

你可以把它拆成五层理解：

1. **`pos`**：当前处理到第几位
2. **`state`**：题目真正关心的状态
3. **`tight`**：是否仍受上界约束
4. **`lead`**：是否仍处于前导零阶段
5. **返回值**：这个状态往后能产生多少合法方案 / 多少统计量

#### 参数含义速查表

| 参数 | 含义 | 是否几乎必备 |
|------|------|--------------|
| `pos` | 当前处理到哪一位 | 是 |
| `state` | 题目的附加状态，如数位和、mask、余数 | 是 |
| `tight` | 当前前缀是否贴着上界 | 是 |
| `lead` | 当前是否还没开始构造有效数字 | 很常见 |

#### 数字拆分方式

数位 DP 通常有两种取数位方式：

1. **转成字符串**
   - 最直观
   - 适合 `10^18` 以内，也适合超大整数

2. **先拆到数组，再翻转**
   - 更接近传统竞赛模板
   - 常见于早期 OI/ACM 代码

这里推荐字符串写法，因为更不容易在下标上出错。

#### `f(R) - f(L-1)` 为什么成立

因为 `solve(n)` 统计的是：

```text
[0, n] 中的合法整数个数
```

那么：

```text
[0, R] - [0, L-1] = [L, R]
```

这是数位 DP 区间计数的固定套路。只要题目问的是某个区间内满足性质的数量、总和、平方和，几乎都先想这一步。

### 2.3 状态设计的思路

数位 DP 最难的部分，通常不是 `tight`，而是：

> **到底要把什么信息塞进 `state`？**

一个非常重要的原则是：

> **状态里只保留“会影响后续决策”的信息。**

也就是说，如果某个信息不会影响未来能不能选、怎么选、能贡献多少答案，就不要放进状态。

#### 常见状态设计表

| 题目特征 | 常见状态 | 说明 |
|----------|----------|------|
| 只看数位和 | `sum` / `sum % k` | 统计数位和、数位和取模 |
| 看相邻关系 | `last` | 记录上一位数字 |
| 看是否出现某种模式 | `flag` / 自动机状态 | 如是否出现子串 `13` |
| 看某个数字出现次数 | `cnt` / `has` | 统计 `1` 出现了多少次、是否出现过 `7` |
| 看是否重复使用数字 | `mask` | 10 位 bitmask 记录哪些数字用过 |
| 看整除性 | `mod` | 记录当前前缀对 `k` 的余数 |

#### 三个判断问题

拿到题目后，先问自己：

1. **未来转移会依赖过去哪部分信息？**
   - 若依赖上一位，就记 `last`
   - 若依赖整个已用集合，就记 `mask`
   - 若依赖整体余数，就记 `mod`

2. **状态规模会不会爆炸？**
   - `mask` 最多 `2^10 = 1024`，通常可以接受
   - `sum` 若上界 18 位，则最多 `9*18=162`
   - `mod` 若模数过大，就要考虑是否真的适合数位 DP

3. **前导零会不会影响这个状态？**
   - 比如 `last` 在前导零阶段通常没有意义
   - 比如统计 `0` 的出现次数时，前导零不能计入

#### 一个实战口诀

```text
看总量 -> 记 sum / cnt
看相邻 -> 记 last
看集合 -> 记 mask
看整除 -> 记 mod
看模式 -> 记 flag / 自动机状态
```

## 三、常见数位 DP 状态类型

在真正刷题时，数位 DP 的状态设计大致可以归纳为下面五类。

### 3.1 数位和相关

这类题最常见，通常形如：
- 数位和等于某个值
- 数位和在某个区间内
- 数位和对 `k` 取模满足条件

```text
典型状态：
  dfs(pos, sum, tight, lead)
  dfs(pos, sum_mod_k, tight, lead)
```

**设计要点**：
- 若题目只关心“数位和是否等于/不超过某值”，记录 `sum` 即可
- 若只关心“数位和 % k”，记录 `sum % k` 更省空间
- 若 `sum` 已经超过上限，可提前剪枝

```cpp
int dfs(int pos, int sum, bool tight, bool lead) {
    if (sum > limit) return 0;
    ...
}
```

### 3.2 相邻数位关系

这类题会关心：
- 相邻位差至少为 `2`
- 单调不减 / 单调不增
- 不出现连续 `62`

```text
典型状态：
  dfs(pos, last, tight, lead)
```

**设计要点**：
- 只要条件和“上一位”有关，就记录 `last`
- 若前导零阶段还没开始构造数字，`last` 往往应设成哨兵值，如 `10` 或 `-1`

```cpp
if (last != 10 && abs(d - last) < 2) continue;
```

### 3.3 特定数位出现次数

这类题问的是：
- 数字 `1` 一共出现多少次
- 是否包含某个数字
- 某个数字恰好出现 `k` 次

```text
典型状态：
  dfs(pos, cnt, tight, lead)
  dfs(pos, has, tight, lead)
```

**设计要点**：
- 如果只关心“有没有出现过”，`bool has` 就够了
- 如果关心“出现了几次”，就记录 `cnt`
- 对数字 `0` 特别小心：前导零通常不能算作出现次数

```cpp
int add = (!newLead && d == target);
```

### 3.4 数位集合（bitmask）

这类题一般问：
- 每位数字互不相同
- 某些数字是否已经用过
- 使用过哪些数字

```text
典型状态：
  dfs(pos, mask, tight, lead)
```

**设计要点**：
- 十进制只有 `0..9`，因此 `mask` 只需要 10 位
- `mask >> d & 1` 判断数字 `d` 是否已出现
- 新状态通常是 `mask | (1 << d)`

```cpp
if (mask >> d & 1) continue;
res += dfs(pos + 1, mask | (1 << d), ...);
```

### 3.5 整除性（取模）

这类题会出现：
- 数本身能被 `k` 整除
- 前缀/整体对某个数取模满足条件
- 同时带子串、计数等组合限制

```text
典型状态：
  dfs(pos, mod, tight, lead)
```

**设计要点**：
- 若当前前缀的余数是 `mod`，填入数字 `d` 后的新余数是：

```cpp
new_mod = (mod * 10 + d) % k;
```

- 这是数位 DP 里处理整除性的标准公式
- 若还要同时记录“是否出现某个模式”，就把 `flag` 一起放进状态

### 状态类型总览表

| 状态类型 | 常见状态 | 典型例题 | 难点 |
|----------|----------|----------|------|
| 数位和 | `sum` / `sum % k` | 数位和区间、LeetCode 2719 | 剪枝与范围控制 |
| 相邻关系 | `last` | Windy 数、不要 62 | 前导零下的 `last` 处理 |
| 出现次数 | `cnt` / `has` | LeetCode 233、计数问题 | 前导零是否计入 |
| 数位集合 | `mask` | LeetCode 1012、2376 | bitmask 转移 |
| 整除性 | `mod` | 含 `13` 且能被 `13` 整除 | `mod` 与其他条件组合 |

建议学习顺序是：

### 🟢 Level 1：入门基础

### 题目 1：不含 9 的数（模板入门）

> 题意：给定正整数 `n`，求 `[1, n]` 中不含数字 `9` 的正整数个数。

**分析/状态设计**：这是最朴素的数位 DP 模板题。题目只要求“某一位不能取 9”，并不依赖数位和、上一位、余数等附加信息，所以状态里连额外 `state` 都不需要。

```text
状态定义：
  dfs(pos, tight, lead) = 从第 pos 位开始构造，能组成多少个合法正整数

转移：
  枚举当前位 d ∈ [0, up]
  若 d == 9，跳过
  其余正常递归

结束条件：
  到达末位时，若整个数不是空串（即 !lead），返回 1
```

**C++ 代码**：
```cpp
string s;
long long memo[20][2];

long long dfs(int pos, bool tight, bool lead) {
    if (pos == (int)s.size()) return !lead;
    if (!tight && memo[pos][lead] != -1) return memo[pos][lead];
    int up = tight ? s[pos] - '0' : 9;
    long long res = 0;
    for (int d = 0; d <= up; d++) {
        if (d == 9) continue;
        res += dfs(pos + 1, tight && d == up, lead && d == 0);
    }
    if (!tight) memo[pos][lead] = res;
    return res;
}

long long solve(long long n) {
    if (n <= 0) return 0;
    s = to_string(n);
    memset(memo, -1, sizeof memo);
    return dfs(0, true, true);
}
// 调用方式/答案：solve(n) 即 [1, n] 中不含 9 的正整数个数
```

**核心启示**：
- 最简单的数位 DP，往往连附加状态都不需要。
- 这题的重点不是状态，而是先把 `tight + lead + 逐位枚举` 这一套框架写顺手。

### 题目 2：数字 1 的个数（LeetCode 233）

> 题意：给定整数 `n`，计算 `[1, n]` 中数字 `1` 出现的总次数。

**分析/状态设计**：这题不是问“有多少个数含 1”，而是问“所有数里一共出现了多少个 1”。因此到达结尾时，返回值不再是 `0/1`，而是当前路径上累积的 `1` 的个数。

```text
状态定义：
  dfs(pos, cnt, tight, lead) = 从第 pos 位开始构造，最终能贡献多少个数字 1

转移：
  若当前位填 d，且它不是前导零阶段的 0，
  那么当 d == 1 时，cnt 增加 1

结束条件：
  到达末位时，直接返回当前 cnt
```

**C++ 代码**：
```cpp
string s;
long long memo[20][20][2];

long long dfs(int pos, int cnt, bool tight, bool lead) {
    if (pos == (int)s.size()) return cnt;
    if (!tight && memo[pos][cnt][lead] != -1) return memo[pos][cnt][lead];
    int up = tight ? s[pos] - '0' : 9;
    long long res = 0;
    for (int d = 0; d <= up; d++) {
        bool newLead = lead && d == 0;
        int newCnt = cnt + (!newLead && d == 1);
        res += dfs(pos + 1, newCnt, tight && d == up, newLead);
    }
    if (!tight) memo[pos][cnt][lead] = res;
    return res;
}

long long countDigitOne(long long n) {
    if (n <= 0) return 0;
    s = to_string(n);
    memset(memo, -1, sizeof memo);
    return dfs(0, 0, true, true);
}
// 调用方式/答案：countDigitOne(n) 即 [1, n] 中数字 1 出现总次数
```

**核心启示**：
- 数位 DP 的返回值不一定是“是否合法的个数”，也可以是某种统计量。
- 只要当前状态能把未来贡献完整描述出来，返回值就可以是“总出现次数”。

### 题目 3：最大为 N 的数字组合（LeetCode 902）

> 题意：给定一个数字集合 `digits`（如 `{1,3,5,7}`）和整数 `n`，求 `[1, n]` 中只由 `digits` 中数字组成的正整数个数。

**分析/状态设计**：这题的关键点在于：当前位并不是从 `0..9` 枚举，而是从给定集合里选。同时，为了表示“位数更短的数”，前导零阶段仍要允许“跳过当前位”。

```text
状态定义：
  dfs(pos, tight, lead) = 从第 pos 位开始构造，能组成多少个合法正整数

转移：
  1) 若仍在前导零阶段，可以选择“这一位不放数字”
  2) 否则只能从 digits 集合中枚举 d
  3) 只有 d <= up 时才能转移

结束条件：
  到达末位时，若至少放过一位数字，则返回 1
```

**C++ 代码**：
```cpp
vector<int> a;
string s;
long long memo[12][2];

long long dfs(int pos, bool tight, bool lead) {
    if (pos == (int)s.size()) return !lead;
    if (!tight && memo[pos][lead] != -1) return memo[pos][lead];
    int up = tight ? s[pos] - '0' : 9;
    long long res = 0;
    if (lead) res += dfs(pos + 1, tight && 0 == up, true);
    for (int d : a) {
        if (d > up) break;
        res += dfs(pos + 1, tight && d == up, false);
    }
    if (!tight) memo[pos][lead] = res;
    return res;
}

long long atMostNGivenDigitSet(vector<string>& digits, int n) {
    a.clear();
    for (auto& x : digits) a.push_back(x[0] - '0');
    sort(a.begin(), a.end());
    s = to_string(n);
    memset(memo, -1, sizeof memo);
    return dfs(0, true, true);
}
// 调用方式/答案：atMostNGivenDigitSet(digits, n)
```

**核心启示**：
- 数位 DP 的枚举范围不一定非得是 `0..9`。
- 只要“当前位有哪些合法候选”能明确列出，就能套同一套框架。

### 🟡 Level 2：相邻与约束

### 题目 4：Windy 数（洛谷 P2657 / SCOI2009）

> 题意：不含前导零且相邻两个数字之差至少为 `2` 的正整数被称为 Windy 数。求 `[A, B]` 内有多少 Windy 数。

**分析/状态设计**：题目的限制只和“当前位”和“上一位”有关，所以状态中必须记录上一位数字 `last`。但在前导零阶段，上一位其实还不存在，因此通常用 `10` 或 `-1` 当作哨兵值。

```text
状态定义：
  dfs(pos, last, tight) = 从第 pos 位开始构造，在上一位为 last 时的合法数量
  其中 last = 10 表示还没开始构造数字

转移：
  1) 若 last = 10 且 d = 0，说明仍在前导零阶段
  2) 否则若 last != 10，需要满足 |d - last| >= 2

结束条件：
  到达末位时，必须已经构成了一个正整数（last != 10）
```

**C++ 代码**：
```cpp
string s;
long long memo[20][11];

long long dfs(int pos, int last, bool tight) {
    if (pos == (int)s.size()) return last != 10;
    if (!tight && memo[pos][last] != -1) return memo[pos][last];
    int up = tight ? s[pos] - '0' : 9;
    long long res = 0;
    for (int d = 0; d <= up; d++) {
        if (last == 10 && d == 0) {
            res += dfs(pos + 1, 10, tight && d == up);
        } else {
            if (last != 10 && abs(d - last) < 2) continue;
            res += dfs(pos + 1, d, tight && d == up);
        }
    }
    if (!tight) memo[pos][last] = res;
    return res;
}

long long solve(long long n) {
    if (n <= 0) return 0;
    s = to_string(n);
    memset(memo, -1, sizeof memo);
    return dfs(0, 10, true);
}
// 调用方式/答案：solve(B) - solve(A - 1)
```

**核心启示**：
- 只要题目涉及“相邻数位关系”，状态里大概率就要记录上一位。
- 前导零阶段的“上一位不存在”必须用哨兵值单独处理。

### 题目 5：不降数（单调不减数位）

> 题意：求 `[1, n]` 中各位数字从高位到低位单调不减的数的个数，如 `123`、`112`、`2339`。

**分析/状态设计**：这题和 Windy 数几乎是同一种模型，只不过相邻限制从“差至少为 2”改成了“后一位不能小于前一位”。本质仍是“上一位状态”。

```text
状态定义：
  dfs(pos, last, tight) = 从第 pos 位开始构造，且上一位数字为 last 的合法数量

转移：
  若已经开始构造数字，则当前位必须满足 d >= last
  若还在前导零阶段，则任何 d 都可以尝试

结束条件：
  到达末位时，必须已经放过至少一位非零数字
```

**C++ 代码**：
```cpp
string s;
long long memo[20][11];

long long dfs(int pos, int last, bool tight) {
    if (pos == (int)s.size()) return last != 10;
    if (!tight && memo[pos][last] != -1) return memo[pos][last];
    int up = tight ? s[pos] - '0' : 9;
    long long res = 0;
    for (int d = 0; d <= up; d++) {
        if (last == 10 && d == 0) {
            res += dfs(pos + 1, 10, tight && d == up);
        } else {
            if (last != 10 && d < last) continue;
            res += dfs(pos + 1, d, tight && d == up);
        }
    }
    if (!tight) memo[pos][last] = res;
    return res;
}

long long solve(long long n) {
    if (n <= 0) return 0;
    s = to_string(n);
    memset(memo, -1, sizeof memo);
    return dfs(0, 10, true);
}
// 调用方式/答案：solve(n)
```

**核心启示**：
- 单调性约束，本质就是相邻关系约束的一种特例。
- 一旦你会了 `last` 状态，很多“相邻位限制题”都只是改一行判断条件。

### 题目 6：计数问题（洛谷 P1239 / NOIP2014 改）

> 题意：给定两个正整数 `a` 和 `b (a ≤ b)`，求 `[a, b]` 中 `0-9` 每个数字各出现了多少次。

**分析/状态设计**：这是“出现次数统计”的标准题。对每个数字 `tar = 0..9` 分别做一次数位 DP 即可。注意：前导零不能算作数字 `0` 的出现，这正是这题最容易出错的点。

```text
状态定义：
  dfs(pos, cnt, tight, lead) = 从第 pos 位开始构造，最终能贡献多少次 tar

转移：
  若当前位填 d：
    - 若它仍是前导零，不计入 cnt
    - 否则当 d == tar 时，cnt + 1

结束条件：
  到达末位时，返回当前 cnt
```

**C++ 代码**：
```cpp
string s;
int tar;
long long memo[20][20][2];

long long dfs(int pos, int cnt, bool tight, bool lead) {
    if (pos == (int)s.size()) return cnt;
    if (!tight && memo[pos][cnt][lead] != -1) return memo[pos][cnt][lead];
    int up = tight ? s[pos] - '0' : 9;
    long long res = 0;
    for (int d = 0; d <= up; d++) {
        bool newLead = lead && d == 0;
        int add = (!newLead && d == tar);
        res += dfs(pos + 1, cnt + add, tight && d == up, newLead);
    }
    if (!tight) memo[pos][cnt][lead] = res;
    return res;
}

long long solve(long long n, int d) {
    if (n <= 0) return 0;
    s = to_string(n);
    tar = d;
    memset(memo, -1, sizeof memo);
    return dfs(0, 0, true, true);
}

vector<long long> countDigits(long long a, long long b) {
    vector<long long> ans(10);
    for (int d = 0; d <= 9; d++) ans[d] = solve(b, d) - solve(a - 1, d);
    return ans;
}
// 调用方式/答案：countDigits(a, b)[d] 即数字 d 的出现次数
```

**核心启示**：
- 前导零不算数字 `0`，这是数位 DP 里最经典的坑之一。
- 这种“分别统计每个数字贡献”的题，通常就是把目标数字当参数跑 10 次。

### 题目 7：数字游戏 II（AcWing 1085 / 不要 62）

> 题意：杭州人认为不含数字 `4` 且不含连续 `62` 的号码是吉利号码。求 `[L, R]` 中吉利号码的个数。

**分析/状态设计**：这题同时有两类约束：
- 单点约束：某一位不能是 `4`
- 相邻约束：不能出现连续 `62`

只要记录“上一位是不是 6”即可。这个状态比记录完整上一位更省。

```text
状态定义：
  dfs(pos, lastIs6, tight) = 从第 pos 位开始构造的合法数量

转移：
  1) 若 d == 4，直接跳过
  2) 若上一位是 6 且当前位是 2，也跳过
  3) 否则正常转移，更新 lastIs6 = (d == 6)

结束条件：
  到达末位时返回 1
```

**C++ 代码**：
```cpp
string s;
long long memo[20][2];

long long dfs(int pos, bool lastIs6, bool tight) {
    if (pos == (int)s.size()) return 1;
    if (!tight && memo[pos][lastIs6] != -1) return memo[pos][lastIs6];
    int up = tight ? s[pos] - '0' : 9;
    long long res = 0;
    for (int d = 0; d <= up; d++) {
        if (d == 4) continue;
        if (lastIs6 && d == 2) continue;
        res += dfs(pos + 1, d == 6, tight && d == up);
    }
    if (!tight) memo[pos][lastIs6] = res;
    return res;
}

long long solve(long long n) {
    if (n < 0) return 0;
    s = to_string(n);
    memset(memo, -1, sizeof memo);
    return dfs(0, false, true);
}
// 调用方式/答案：solve(R) - solve(L - 1)
```

**核心启示**：
- 有些题不必记完整上一位，只记“是否为某个关键值”就够了。
- 数位 DP 的状态设计目标不是“全记下来”，而是“记最少但够用的信息”。

### 🟠 Level 3：计数与统计

### 题目 8：至少有 1 位重复的数字（LeetCode 1012）

> 题意：给定正整数 `n`，返回 `[1, n]` 中至少有一位数字重复出现的正整数个数。

**分析/状态设计**：正着数“至少有重复”的情况很麻烦，但反过来数“所有位互不相同”的整数就很自然。因此经典做法是：

```text
答案 = n - [1, n] 中每位互不相同的整数个数
```

而“哪些数字已经用过”最适合用 bitmask 记录。

```text
状态定义：
  dfs(pos, mask, tight, lead) = 从第 pos 位开始构造，且已用数字集合为 mask 时的合法数量

转移：
  1) 若还在前导零阶段且 d = 0，可继续保持 mask 不变
  2) 否则若 d 已在 mask 中，跳过
  3) 若未使用过，则转移到 mask | (1 << d)

结束条件：
  到达末位时，必须已经构成正整数（!lead）
```

**C++ 代码**：
```cpp
string s;
long long memo[20][1 << 10][2];

long long dfs(int pos, int mask, bool tight, bool lead) {
    if (pos == (int)s.size()) return !lead;
    if (!tight && memo[pos][mask][lead] != -1) return memo[pos][mask][lead];
    int up = tight ? s[pos] - '0' : 9;
    long long res = 0;
    for (int d = 0; d <= up; d++) {
        bool newLead = lead && d == 0;
        if (newLead) {
            res += dfs(pos + 1, mask, tight && d == up, true);
        } else {
            if (mask >> d & 1) continue;
            res += dfs(pos + 1, mask | (1 << d), tight && d == up, false);
        }
    }
    if (!tight) memo[pos][mask][lead] = res;
    return res;
}

long long countDistinct(long long n) {
    if (n <= 0) return 0;
    s = to_string(n);
    memset(memo, -1, sizeof memo);
    return dfs(0, 0, true, true);
}

long long numDupDigitsAtMostN(int n) {
    return n - countDistinct(n);
}
// 调用方式/答案：numDupDigitsAtMostN(n)
```

**核心启示**：
- 很多数位 DP 题都适合“正难则反”。
- 十进制数字只有 10 个，用 bitmask 记录使用情况是天然选择。

### 题目 9：统计特殊整数（LeetCode 2376）

> 题意：返回 `[1, n]` 中特殊整数的数目，其中“特殊整数”指每位数字都不同的正整数。

**分析/状态设计**：这题与上一题本质完全相同，只是这次要求的正好就是“无重复数字”的数量，因此直接返回 bitmask 数位 DP 的结果即可。

```text
状态定义：
  dfs(pos, mask, tight, lead) = 从第 pos 位开始构造，已用数字集合为 mask 时的合法数量

转移：
  与题 8 完全一致：
    - 前导零可跳过
    - 非前导零时不能使用已在 mask 中的数字
```

```cpp
string s;

long long dfs(int pos, int mask, bool tight, bool lead) {

    long long res = 0;
    for (int d = 0; d <= up; d++) {

        if (newLead) {

        }

    return res;
}

int countSpecialNumbers(int n) {
    s = to_string(n);
    memset(memo, -1, sizeof memo);
    return (int)dfs(0, 0, true, true);
}
// 调用方式/答案：countSpecialNumbers(n)
```

**核心启示**：
- 同一个状态模型，常常只换一下题意就能复用。
- 真正应该背的不是题目，而是“bitmask 记录数字集合”这个状态套路。

### 题目 10：Round Numbers（POJ 3252 / 洛谷 P6218）

> 题意：求 `[L, R]` 中 Round Number 的个数。Round Number 定义为二进制表示中 `0` 的个数不少于 `1` 的个数。

**分析/状态设计**：这题的关键不是十进制，而是 **二进制数位 DP**。也就是说，数位 DP 并不局限于十进制，只要是“按位构造 + 上界限制”的问题，任何进制都能做。

```text
状态定义：
  dfs(pos, zero, one, tight, lead)
  = 从二进制第 pos 位开始构造，当前已统计 zero 个 0、one 个 1 的合法数量

转移：
  1) 若仍在前导零阶段且 d = 0，则 zero / one 都不增加
  2) 若数字已经开始：
       d = 0 -> zero + 1
       d = 1 -> one + 1

结束条件：
  必须构成正整数，且 zero >= one
```

**C++ 代码**：
```cpp
string s;
long long memo[65][65][65][2];

long long dfs(int pos, int zero, int one, bool tight, bool lead) {
    if (pos == (int)s.size()) return !lead && zero >= one;
    if (!tight && memo[pos][zero][one][lead] != -1) return memo[pos][zero][one][lead];
    int up = tight ? s[pos] - '0' : 1;
    long long res = 0;
    for (int d = 0; d <= up; d++) {
        bool newLead = lead && d == 0;
        int nz = zero, no = one;
        if (!newLead) {
            if (d == 0) nz++;
            else no++;
        }
        res += dfs(pos + 1, nz, no, tight && d == up, newLead);
    }
    if (!tight) memo[pos][zero][one][lead] = res;
    return res;
}

string toBinary(long long n) {
    string t;
    while (n) t.push_back(char('0' + (n & 1))), n >>= 1;
    reverse(t.begin(), t.end());
    return t.empty() ? "0" : t;
}

long long solve(long long n) {
    if (n <= 0) return 0;
    s = toBinary(n);
    memset(memo, -1, sizeof memo);
    return dfs(0, 0, 0, true, true);
}
// 调用方式/答案：solve(R) - solve(L - 1)
```

**核心启示**：
- 数位 DP 的核心是“按位决策”，不是“十进制”。
- 只要能拆成某个进制下的位序列，模板照样成立。

### 题目 11：能被 13 整除且含 "13" 子串的数（经典）

> 题意：求 `[1, n]` 中既能被 `13` 整除，又包含连续子串 `"13"` 的数的个数。

**分析/状态设计**：这题是数位 DP 组合状态的经典模板。
它同时需要记录：
- 当前前缀对 `13` 的余数 `mod`
- 是否已经出现过子串 `13`
- 上一位数字 `last`

```text
状态定义：
  dfs(pos, mod, has13, last, tight, lead)
  = 从第 pos 位开始构造，当前余数为 mod，是否已出现 13，上一位为 last 的合法数量

转移：
  1) 新余数：new_mod = (mod * 10 + d) % 13
  2) 若上一位是 1 且当前位是 3，则 has13 变为 true
  3) 前导零阶段时，last 用 10 表示“尚不存在”

结束条件：
  到达末位时，必须是正整数，且 has13 = true，mod = 0
```

**C++ 代码**：
```cpp
string s;
long long memo[20][13][2][11];

long long dfs(int pos, int mod, bool has13, int last, bool tight, bool lead) {
    if (pos == (int)s.size()) return !lead && has13 && mod == 0;
    if (!tight && memo[pos][mod][has13][last] != -1) return memo[pos][mod][has13][last];
    int up = tight ? s[pos] - '0' : 9;
    long long res = 0;
    for (int d = 0; d <= up; d++) {
        bool newLead = lead && d == 0;
        int newLast = newLead ? 10 : d;
        bool newHas13 = has13;
        if (!newLead && last == 1 && d == 3) newHas13 = true;
        int newMod = newLead ? 0 : (mod * 10 + d) % 13;
        res += dfs(pos + 1, newMod, newHas13, newLast, tight && d == up, newLead);
    }
    if (!tight) memo[pos][mod][has13][last] = res;
    return res;
}

long long solve(long long n) {
    if (n <= 0) return 0;
    s = to_string(n);
    memset(memo, -1, sizeof memo);
    return dfs(0, 0, false, 10, true, true);
}
// 调用方式/答案：solve(n)
```

**核心启示**：
- `mod`、`last`、`flag` 这三类状态经常需要组合使用。
- 一旦会了“余数转移 + 子串标记”，很多综合约束题都能拆开处理。

### 题目 12：Balanced Number（HDU 3709）

> 题意：求 `[L, R]` 中平衡数的个数。平衡数定义为：存在某一位作为支点，使得左右两边力矩之和相等。

**分析/状态设计**：这题的特殊点在于，除了普通数位 DP 状态外，还要额外枚举一个“支点位置 `pivot`”。固定支点后，某一位数字 `d` 对总力矩的贡献就是：

```text
d * (当前位置 - pivot)
```

因此我们可以：
1. 先枚举支点 `pivot`
2. 再做一次记录“当前力矩和”的数位 DP

```text
状态定义：
  dfs(pos, balance, tight, lead)
  = 在支点 pivot 固定时，从第 pos 位开始构造，当前总力矩为 balance 的合法数量

转移：
  若当前位放 d，且不是前导零阶段的空位，
  则力矩变化：balance += d * (pos - pivot)

结束条件：
  到达末位时，必须构成正整数，且总力矩为 0
```

**C++ 代码**：
```cpp
const int OFFSET = 4000;
string s;
int pivot;
long long memo[20][8005][2];

long long dfs(int pos, int balance, bool tight, bool lead) {
    if (pos == (int)s.size()) return !lead && balance == OFFSET;
    if (!tight && memo[pos][balance][lead] != -1) return memo[pos][balance][lead];
    int up = tight ? s[pos] - '0' : 9;
    long long res = 0;
    for (int d = 0; d <= up; d++) {
        bool newLead = lead && d == 0;
        int newBalance = balance;
        if (!newLead) newBalance += d * (pos - pivot);
        res += dfs(pos + 1, newBalance, tight && d == up, newLead);
    }
    if (!tight) memo[pos][balance][lead] = res;
    return res;
}

long long solve(long long n) {
    if (n <= 0) return 0;
    s = to_string(n);
    long long ans = 0;
    for (pivot = 0; pivot < (int)s.size(); pivot++) {
        memset(memo, -1, sizeof memo);
        ans += dfs(0, OFFSET, true, true);
    }
    return ans;
}
// 调用方式/答案：solve(R) - solve(L - 1)
```

**核心启示**：
- 有时题目里的某个“特殊位置”无法直接并入状态，就先枚举它。
- “枚举额外参数 + 数位 DP”是处理复杂结构约束的常见套路。

### 题目 13：恨 7 不成妻（HDU 4507）

> 题意：求 `[L, R]` 中与 `7` 无关的数的平方和。与 `7` 无关是指：数字不含 `7`、数位和不是 `7` 的倍数、本身也不是 `7` 的倍数。答案通常对 `1e9+7` 取模。

**分析/状态设计**：这题是数位 DP 里非常经典的“多返回值统计”题。
仅仅返回个数不够，我们还要同时维护：
- `cnt`：合法数的个数
- `sum`：合法数之和
- `sq`：合法数平方和

状态则需要记录：
- `digit_sum % 7`
- `number % 7`

```text
状态定义：
  dfs(pos, sumMod, numMod, tight) = 一个结构体 {cnt, sum, sq}

转移：
  若当前位放 d：
    newSumMod = (sumMod + d) % 7
    newNumMod = (numMod * 10 + d) % 7

平方和合并公式：
  设当前位贡献前缀值为 p，子问题返回 {cnt, sum, sq}
  则：
    new_sum += sum + p * cnt
    new_sq  += sq + 2 * p * sum + p^2 * cnt
```

**C++ 代码**：
```cpp
const long long MOD = 1000000007;

struct Node {
    long long cnt, sum, sq;
};

string s;
Node memo[20][7][7];
bool vis[20][7][7];
long long pw[20];

Node dfs(int pos, int sumMod, int numMod, bool tight) {
    if (pos == (int)s.size()) {
        if (sumMod != 0 && numMod != 0) return {1, 0, 0};
        return {0, 0, 0};
    }
    if (!tight && vis[pos][sumMod][numMod]) return memo[pos][sumMod][numMod];
    int up = tight ? s[pos] - '0' : 9;
    Node res{0, 0, 0};
    int len = (int)s.size();
    for (int d = 0; d <= up; d++) {
        if (d == 7) continue;
        Node t = dfs(pos + 1, (sumMod + d) % 7, (numMod * 10 + d) % 7, tight && d == up);
        long long p = d * pw[len - pos - 1] % MOD;
        res.cnt = (res.cnt + t.cnt) % MOD;
        res.sum = (res.sum + t.sum + p * t.cnt) % MOD;
        res.sq = (res.sq + t.sq + 2LL * p % MOD * t.sum + p * p % MOD * t.cnt) % MOD;
    }
    if (!tight) vis[pos][sumMod][numMod] = true, memo[pos][sumMod][numMod] = res;
    return res;
}

long long solve(long long n) {
    if (n <= 0) return 0;
    s = to_string(n);
    pw[0] = 1;
    for (int i = 1; i < 20; i++) pw[i] = pw[i - 1] * 10 % MOD;
    memset(vis, 0, sizeof vis);
    return dfs(0, 0, 0, true).sq;
}
// 调用方式/答案：(solve(R) - solve(L - 1) + MOD) % MOD
```

**核心启示**：
- 数位 DP 不只会“计数”，也能“求和”“求平方和”“求多种统计量”。
- 一旦返回值是结构体，核心就变成“如何正确合并子问题贡献”。

### 题目 14：手机号码（洛谷 P4124 / CQOI2016）

> 题意：11 位手机号码必须以 `1` 开头，必须包含连续 3 个相同数字，但不能同时含 `4` 和 `8`。求 `[L, R]` 中满足条件的号码个数。

**分析/状态设计**：这题是“多约束组合”的典型代表。我们需要同时记录：
- 最近两位是什么（判断是否出现连续 3 个相同数字）
- 是否已经出现过三连
- 是否出现过 `4`
- 是否出现过 `8`

由于号码长度固定为 `11` 位，且首位必须为 `1`，所以这里不需要 `lead` 状态，直接从高位往低位填满 11 位即可。

```text
状态定义：
  dfs(pos, last1, last2, hasTriple, has4, has8, tight)

转移：
  1) pos = 0 时，只允许填 1
  2) 若 last2 == last1 == d，则 hasTriple 置 true
  3) has4 / has8 根据 d 是否等于 4 / 8 更新
  4) 一旦同时出现 4 和 8，直接剪枝

结束条件：
  到达第 11 位时，只要 hasTriple 为 true 即合法
```

**C++ 代码**：
```cpp
string s;
long long memo[11][11][11][2][2][2];
bool vis[11][11][11][2][2][2];

long long dfs(int pos, int last1, int last2, bool hasTriple, bool has4, bool has8, bool tight) {
    if (has4 && has8) return 0;
    if (pos == 11) return hasTriple;
    if (!tight && vis[pos][last1][last2][hasTriple][has4][has8])
        return memo[pos][last1][last2][hasTriple][has4][has8];
    int up = tight ? s[pos] - '0' : 9;
    long long res = 0;
    for (int d = 0; d <= up; d++) {
        if (pos == 0 && d != 1) continue;
        bool newTriple = hasTriple || (last2 != 10 && last2 == last1 && last1 == d);
        res += dfs(pos + 1, d, last1, newTriple, has4 || d == 4, has8 || d == 8, tight && d == up);
    }
    if (!tight) {
        vis[pos][last1][last2][hasTriple][has4][has8] = true;
        memo[pos][last1][last2][hasTriple][has4][has8] = res;
    }
    return res;
}

long long solve(long long n) {
    if (n < 10000000000LL) return 0;
    if (n > 99999999999LL) n = 99999999999LL;
    s = to_string(n);
    memset(vis, 0, sizeof vis);
    return dfs(0, 10, 10, false, false, false, true);
}
// 调用方式/答案：solve(R) - solve(L - 1)
```

**核心启示**：
- 多条件题看起来复杂，但仍然只是把多个“小状态”并在一起。
- 只要每个状态都明确对应某一条约束，整体框架并不会改变。

### 题目 15：Count of Integers（LeetCode 2719）

> 题意：给定 `num1`、`num2`（字符串，长度可达 `10^22` 量级）以及 `min_sum`、`max_sum`，求 `[num1, num2]` 中数位和位于 `[min_sum, max_sum]` 的整数个数。

**分析/状态设计**：这题最重要的点不是状态本身，而是：

> **当数值已经超过 `long long` 范围时，必须直接对字符串做数位 DP。**

由于性质只与“数位和”有关，所以状态非常简单，只需要记录当前累计和 `sum`。此外，为了求区间答案，需要写一个字符串版的 `minusOne`。

```text
状态定义：
  dfs(pos, sum, tight) = 从第 pos 位开始构造，当前数位和为 sum 时的合法数量

转移：
  枚举 d ∈ [0, up]
  递归到 dfs(pos + 1, sum + d, ...)

结束条件：
  到达末位时，检查 min_sum <= sum <= max_sum
```

**C++ 代码**：
```cpp
const int MOD = 1000000007;

string s;
int minSum, maxSum;
int memo[25][405];
bool vis[25][405];

int dfs(int pos, int sum, bool tight) {
    if (sum > maxSum) return 0;
    if (pos == (int)s.size()) return minSum <= sum && sum <= maxSum;
    if (!tight && vis[pos][sum]) return memo[pos][sum];
    int up = tight ? s[pos] - '0' : 9;
    long long res = 0;
    for (int d = 0; d <= up; d++) {
        res += dfs(pos + 1, sum + d, tight && d == up);
        res %= MOD;
    }
    if (!tight) vis[pos][sum] = true, memo[pos][sum] = (int)res;
    return (int)res;
}

int solve(string num) {
    s = num;
    memset(vis, 0, sizeof vis);
    return dfs(0, 0, true);
}

string minusOne(string x) {
    int i = (int)x.size() - 1;
    while (i >= 0 && x[i] == '0') x[i--] = '9';
    if (i < 0) return "";
    x[i]--;
    int p = 0;
    while (p + 1 < (int)x.size() && x[p] == '0') p++;
    return x.substr(p);
}

int count(string num1, string num2, int lo, int hi) {
    minSum = lo, maxSum = hi;
    string left = minusOne(num1);
    int a = solve(num2);
    int b = left.empty() ? 0 : solve(left);
    return (a - b + MOD) % MOD;
}
// 调用方式/答案：count(num1, num2, min_sum, max_sum)
```

**核心启示**：
- 数值一旦大到无法放进整数类型，就立刻转成字符串数位 DP。
- 当题目是区间 `[num1, num2]` 时，别忘了准备字符串版的 `L-1` 操作。

## 五、调试技巧与常见陷阱

### 5.1 前导零的正确处理

前导零是数位 DP 最容易错、也最容易“以为自己没错”的地方。

#### 场景一：统计“是否构成正整数”

最常见的结束写法是：

```cpp
if (pos == len) return !lead;
```

意思是：
- 如果一路都没放过非零数字，那其实构造的是“空数”或数字 `0`
- 若题目只统计正整数，就不能算进去

#### 场景二：统计数字 `0` 出现次数

这一题型必须牢记：

> **前导零不是数字 `0` 的出现。**

正确写法通常是：

```cpp
bool newLead = lead && d == 0;
int add = (!newLead && d == 0);
```

如果你写成 `add = (d == 0)`，那么像 `0057` 的前两个 `0` 也会被错误统计。

#### 场景三：涉及相邻数位关系

例如 Windy 数、单调数、不要 `62` 这类题。

这时在前导零阶段：
- `last` 还没有意义
- 不能拿“前导零”去和首个非零数字做相邻判断

所以必须用哨兵值区分：

```cpp
last = 10   // 表示尚未开始构造
```

### 5.2 `tight` 传递的正确写法

数位 DP 里有一行几乎是固定模板：

```cpp
int up = tight ? s[pos] - '0' : 9;
```

然后下一层：

```cpp
bool newTight = tight && (d == up);
```

很多错误都出在这里，常见问题有两个：

1. **忘了 `tight &&`**
   - 写成 `d == up`
   - 这样当当前已经不贴边时，也会错误地把某些状态当成“继续贴边”

2. **上界写错对象**
   - 在二进制数位 DP 中，上界不是 `9`，而是 `1`
   - 在给定集合枚举中，仍要先判断候选数字 `d <= up`

一个很好的自检问题是：

> 当前前缀已经比上界小了吗？如果小了，后面应该完全自由。

只要你拿这个逻辑反推，大多数 `tight` 错误都能及时发现。

### 5.3 记忆化的维度设计

数位 DP 里有个非常重要的原则：

> **所有会影响未来选择和答案的量，都必须进入状态键。**

例如：
- Windy 数必须记 `last`
- 无重复数字必须记 `mask`
- 能被 `k` 整除必须记 `mod`
- 统计出现次数必须记 `cnt`

反过来，如果某个量不会影响未来，就不要塞进去，否则状态会无意义膨胀。

#### 一个判断标准

假设有两个中间状态：

```text
A: pos 相同，但某个信息不同
B: pos 相同，但某个信息不同
```

如果从 A 和 B 往后走，未来合法转移集合或最终答案会不同，那么这个信息就必须放进状态。

#### 只缓存 `tight=false` 时的状态

这是最常见的写法，因为：
- `tight=true` 的状态依赖上界具体前缀
- 一般只会访问很少次
- `tight=false` 后才是真正的大量重复子树

所以常见代码会写成：

```cpp
if (!tight && memo[...] != -1) return memo[...];
...
if (!tight) memo[...] = res;
```

### 5.4 大数范围的处理（string-based digit DP）

当上界超过 `10^18`，甚至题目直接给你一个字符串时，就不要再尝试转 `long long` 了。

标准做法是：

1. 直接把上界当字符串 `s`
2. 用 `pos` 在字符串上下标移动
3. `up = s[pos] - '0'`
4. 如果要算区间答案，写一个字符串减一函数 `minusOne`

```cpp
string minusOne(string x) {
    int i = x.size() - 1;
    while (i >= 0 && x[i] == '0') x[i--] = '9';
    if (i < 0) return "";
    x[i]--;
    return x;
}
```

这类题的难点不是 DP 本身，而是：
- 字符串减一是否写对
- 前导零裁剪是否正确
- 区间端点是否考虑周全

## 六、终极总结与归纳

### 6.1 数位 DP 的本质

数位 DP 的本质，可以压缩成一句话：

> **在“不能超过上界”的前提下，按位构造所有可能数字，并把与题意相关的信息压进状态。**

无论题面包装成：
- 不含某些数字
- 相邻位满足限制
- 数位和落在区间内
- 数字是否重复
- 是否整除
- 是否包含某种模式

本质都还是这四步：
- 从高位往低位填
- 维护 `tight`
- 处理 `lead`
- 在 `state` 里保留未来需要的信息

### 6.2 解题模板速查

```text
第 1 步：把区间题转成 solve(R) - solve(L - 1)
第 2 步：写 dfs(pos, state, tight, lead)
第 3 步：枚举当前位 d ∈ [0, up]
第 4 步：根据题意更新状态，递归到下一位
第 5 步：在 pos == len 时返回合法性 / 统计量
第 6 步：对 !tight 的状态做记忆化
```

一个最常见的骨架如下：

```cpp
long long dfs(int pos, int state, bool tight, bool lead) {
    if (pos == len) return check(state, lead);
    if (!tight && memo[pos][state][lead] != -1) return memo[pos][state][lead];
    int up = tight ? digits[pos] : 9;
    long long res = 0;
    for (int d = 0; d <= up; d++) {
        bool newLead = lead && d == 0;
        int newState = newLead ? initState : transition(state, d);
        res += dfs(pos + 1, newState, tight && d == up, newLead);
    }
    if (!tight) memo[pos][state][lead] = res;
    return res;
}
```

### 6.5 口诀

> 🔢 **数位 DP 口诀**：
>
> **数位从高往低填，tight 标记贴上界；**
> **前导零要分清楚，状态设计看条件；**
> **相邻记 last，重复开 mask，整除余数随位变；**
> **记忆搜索存子树，`f(R)` 减去 `f(L-1)`；**
> **十进制是主战场，二进制也能照样练。**
