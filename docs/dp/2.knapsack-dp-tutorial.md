## 一、什么是背包 DP

**背包 DP** 的核心，是在“资源受限”的条件下做最优决策。
最常见的资源是 **容量 / 重量 / 体积**，但在实际题目中，它也可能是 **金额、时间、人数、0 的个数、1 的个数、利润下界** 等。

你可以把它理解为一句话：

> **面对一批可选对象，在不超过约束的前提下，求最优值、方案数或可行性。**

为什么它叫“背包”？因为最初问题就是：
有一个容量为 `W` 的背包，每件物品有重量和价值，问怎么装最划算。
后来，这种“容量受限 + 选择决策”的模型被不断抽象，形成了背包 DP 家族。

### 与其他常见 DP 的对比

| DP 类型 | 常见状态 | 决策对象 | 典型枚举顺序 | 代表问题 |
|---------|----------|----------|--------------|----------|
| 线性 DP | `dp[i]` | 第 `i` 个位置选什么 | 从左到右 | LIS、打家劫舍 |
| 区间 DP | `dp[l][r]` | 在区间内如何分割 | 按区间长度递增 | 石子合并 |
| 树形 DP | `dp[u][state]` | 如何合并子树 | DFS 后序 | 没有上司的舞会 |
| 背包 DP | `dp[j]` / `dp[i][j]` | 当前物品取几件 | 枚举物品 + 枚举容量 | 0/1 背包、完全背包 |

### 背包 DP 的本质

背包题看起来千变万化，但本质非常统一：

1. **阶段**：当前处理到第几件物品 / 第几组物品 / 第几类选择
2. **资源**：还剩多少容量 / 已用了多少费用
3. **目标**：最大价值、最小代价、方案数、是否可达

只要你能把题目翻译成这三层结构，十有八九就能落到背包 DP。

## 二、核心思想与前置知识

### 2.1 背包问题的三要素

做背包 DP 时，先不要急着写代码，先把下面三件事写出来：

1. **物品是什么？**
   - 真正的“物品”可能是一件商品、一个数字、一种硬币、一段时间、一种主件组合。
   - 有时题目会先让你做一层转化，再得到“物品”。

2. **容量是什么？**
   - 可能是体积 `V`
   - 可能是金额 `money`
   - 可能是人数 `n`
   - 可能是两个维度，例如 `(0 的个数, 1 的个数)`

3. **目标是什么？**
   - 求最大值：`max`
   - 求最小值：`min`
   - 求方案数：`+=`
   - 判断可行性：`bool`

可以用下面这张通用思考卡片来套：

```
阶段：枚举到第 i 件 / 第 i 组 / 第 i 类物品
状态：dp[...] 表示某种容量条件下的最优值 / 方案数 / 可行性
决策：当前物品选 0 件、1 件、若干件，或组内选哪一个
转移：从“上一阶段”或“当前阶段较小容量”推到当前状态
答案：dp[目标容量] 或 max(dp[0..容量])
```

还有一个背包题最容易出错的隐藏关键：

> **循环顺序决定“同一件物品会不会被重复使用”。**

- `j` **逆序**：防止本轮刚更新的状态再次被用到 → 适合 **0/1 背包**
- `j` **正序**：允许同一物品在本轮被继续使用 → 适合 **完全背包**

### 2.2 通用框架代码

下面给出最经典的 **0/1 背包通用模板**。这一版保留完整 `#include` 与 `main`，方便你直接运行理解。

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 1005;
const int MAXV = 10005;

int n, m;
int w[MAXN], v[MAXN];
int dp[MAXV];

int main() {
    cin >> n >> m;
    for (int i = 1; i <= n; i++) cin >> w[i] >> v[i];

    for (int i = 1; i <= n; i++) {
        for (int j = m; j >= w[i]; j--) {
            dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
        }
    }

    cout << dp[m] << '\n';
    return 0;
}
```

你可以把它拆成四步：

1. 读入物品信息
2. 枚举物品
3. 枚举容量并做状态转移
4. 输出答案

后面所有背包模型，本质上都只是改动这四步中的某一部分：
- 改循环顺序
- 改转移公式
- 改状态维度
- 改物品组织方式

## 三、五大背包模型

背包 DP 最核心的五个模型，分别是：
**0/1 背包、完全背包、多重背包、分组背包、混合背包**。
它们之间不是孤立的，而是一层一层扩展出来的。

### 3.1 0/1 背包

0/1 背包是所有背包题的起点。每件物品只能选 **0 次或 1 次**。
当题目里出现“每个元素只能用一次”“每个数只能选或不选”“每个字符串最多取一次”时，优先联想到它。

```
状态定义：
  dp[i][j] = 只看前 i 件物品，容量不超过 j 时的最大价值

转移方程：
  dp[i][j] = dp[i-1][j]
  dp[i][j] = max(dp[i][j], dp[i-1][j-w[i]] + v[i])   (j >= w[i])

一维优化：
  dp[j] = max(dp[j], dp[j-w[i]] + v[i])
  j 必须从大到小枚举
```

**关键洞察**：与“只做一次决策”的线性 DP 很像，但这里的第二维是容量；逆序枚举 `j` 的目的，是保证每件物品只被用一次。

```cpp
const int MAXV = 10005;
int dp[MAXV];

void solve() {
    for (int i = 1; i <= n; i++)
        for (int j = m; j >= w[i]; j--)
            dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
}
// 调用方式/答案：solve(); dp[m] 即容量不超过 m 的最大价值
```

### 3.2 完全背包

完全背包和 0/1 背包几乎长得一模一样，唯一差别是：
**每件物品可以选无限次**。

例如：硬币无限使用、完全平方数可重复取、某种商品库存无限。

```
状态定义：
  dp[i][j] = 前 i 类物品，容量不超过 j 的最大价值

转移方程：
  dp[i][j] = max(dp[i-1][j], dp[i][j-w[i]] + v[i])   (j >= w[i])

一维优化：
  dp[j] = max(dp[j], dp[j-w[i]] + v[i])
  j 必须从小到大枚举
```

**关键洞察**：和 0/1 背包唯一真正不同的地方，就是第二项来自 `dp[i][...]` 而不是 `dp[i-1][...]`，因此一维优化时容量要正序。

```cpp
const int MAXV = 10005;
int dp[MAXV];

void solve() {
    for (int i = 1; i <= n; i++)
        for (int j = w[i]; j <= m; j++)
            dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
}
// 调用方式/答案：solve(); dp[m] 即完全背包最优值
```

### 3.3 多重背包

多重背包介于 0/1 与完全之间：每件物品最多选 `s[i]` 次。
如果直接枚举选几件，朴素做法是三重循环；当 `s[i]` 很大时，要用 **二进制拆分** 把它化成若干件 0/1 物品。

```
朴素状态：
  dp[i][j] = max(dp[i-1][j-k*w[i]] + k*v[i])
  其中 0 <= k <= s[i] 且 k*w[i] <= j

优化思路：
  s[i] = 1 + 2 + 4 + ... + rest
  将每一块看成一件新的 0/1 物品
```

**关键洞察**：数量有限，就说明“不能无限滚”；而二进制拆分把“最多 s 件”压缩成 `O(log s)` 个选择块，是最常用优化。

```cpp
const int MAXM = 12005;
int dp[MAXM], ww[MAXM], vv[MAXM], tot;

void solve() {
    for (int i = 1; i <= n; i++) {
        int c = s[i];
        for (int k = 1; k <= c; k <<= 1) {
            ww[++tot] = k * w[i], vv[tot] = k * v[i];
            c -= k;
        }
        if (c) ww[++tot] = c * w[i], vv[tot] = c * v[i];
    }
    for (int i = 1; i <= tot; i++)
        for (int j = m; j >= ww[i]; j--)
            dp[j] = max(dp[j], dp[j - ww[i]] + vv[i]);
}
// 调用方式/答案：solve(); dp[m] 即二进制优化后的最优值
```

### 3.4 分组背包

分组背包的特点是：物品被分成若干组，**每组最多选一个**。
它经常出现在“主件搭配”“每门课选一本教材”“每个类别选一种方案”这类题里。

```
状态定义：
  dp[i][j] = 前 i 组中选，容量不超过 j 的最大价值

转移方程：
  dp[i][j] = dp[i-1][j]
  dp[i][j] = max(dp[i][j], dp[i-1][j-w[i][k]] + v[i][k])
  其中 k 枚举第 i 组中的物品

一维优化：
  对每一组，j 从大到小枚举；组内再枚举具体物品
```

**关键洞察**：分组背包不是“每件物品单独决策”，而是“每一组只做一次选择”。因此外层按组枚举，容量仍然逆序，保证同组内不会取多件。

```cpp
const int MAXV = 10005;
int dp[MAXV];

void solve() {
    for (int i = 1; i <= n; i++)
        for (int j = m; j >= 0; j--)
            for (int k = 1; k <= sz[i]; k++)
                if (j >= w[i][k])
                    dp[j] = max(dp[j], dp[j - w[i][k]] + v[i][k]);
}
// 调用方式/答案：solve(); dp[m] 即分组背包最优值
```

### 3.5 混合背包

混合背包是综合版：有些物品只能选一次，有些能无限选，有些只能选若干次。
本质上不是新模型，而是 **把不同物品交给不同的转移方式处理**。

```
分类处理：
  s[i] = -1  -> 0/1 背包
  s[i] = 0   -> 完全背包
  s[i] > 0   -> 多重背包（二进制拆分后做 0/1）
```

**关键洞察**：看到“物品类型不同”不要慌，先分类，再分别套模板；混合背包考察的不是新公式，而是模板组合能力。

```cpp
const int MAXV = 10005;
int dp[MAXV];

void zeroOne(int c, int val) {
    for (int j = m; j >= c; j--) dp[j] = max(dp[j], dp[j - c] + val);
}
void complete(int c, int val) {
    for (int j = c; j <= m; j++) dp[j] = max(dp[j], dp[j - c] + val);
}
void solve() {
    for (int i = 1; i <= n; i++) {
        if (s[i] == -1) zeroOne(w[i], v[i]);
        else if (s[i] == 0) complete(w[i], v[i]);
        else {
            int c = s[i];
            for (int k = 1; k <= c; k <<= 1) zeroOne(k * w[i], k * v[i]), c -= k;
            if (c) zeroOne(c * w[i], c * v[i]);
        }
    }
}
// 调用方式/答案：solve(); dp[m] 即混合背包最优值
```

**五种模型对比表**

| 模型 | 每件物品可选次数 | 容量循环方向 | 常见目标 | 典型题型 |
|------|------------------|--------------|----------|----------|
| 0/1 背包 | 0 或 1 次 | 逆序 | 最大值 / 可达性 / 方案数 | 子集和、选或不选 |
| 完全背包 | 无限次 | 正序 | 最小个数 / 方案数 / 最大值 | 硬币、完全平方数 |
| 多重背包 | 最多 `s[i]` 次 | 朴素三重 / 拆分后逆序 | 最大值 / 方案数 | 有库存上限 |
| 分组背包 | 每组最多 1 个 | 按组逆序 | 最大值 | 分类选择、主件附件 |
| 混合背包 | 视类型而定 | 按类型决定 | 综合最优 | 多模型同场景 |

### 题目 1：0/1 背包模板（AcWing 2）

> 题意：有 `N` 件物品，每件物品有重量 `w[i]` 和价值 `v[i]`，背包容量为 `W`。每件物品最多选一次，求最大总价值。

**分析/状态设计**：这是最标准的 0/1 背包。二维状态最好理解，一维优化则是必须会默写的模板。

```
dp[i][j] = 只看前 i 件物品、容量不超过 j 的最大价值

转移：
  不选第 i 件：dp[i-1][j]
  选第 i 件：dp[i-1][j-w[i]] + v[i]

故：
  dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])

一维优化：
  dp[j] = max(dp[j], dp[j-w[i]] + v[i])
  j 从 W 到 w[i] 逆序枚举
```

**C++ 代码**：
```cpp
const int MAXV = 1010;
int dp[MAXV];

void solve() {
    for (int i = 1; i <= n; i++)
        for (int j = m; j >= w[i]; j--)
            dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
}
// 调用方式/答案：solve(); dp[m] 即最大价值
```

**核心启示**：
- 0/1 背包的精髓不是公式，而是 **逆序枚举容量**。
- 一维 `dp[j]` 表示“容量不超过 `j` 时的最优值”，这是最常用写法。

### 题目 2：分割等和子集（LeetCode 416）

> 题意：给定一个正整数数组，问能否把它分成两个子集，使得两个子集元素和相等。

**分析/状态设计**：总和若为奇数，必然无解；否则目标就是能否选出若干数，使和恰好为 `sum / 2`。于是它变成了 **布尔型 0/1 背包**。

```
设 target = sum / 2

dp[j] = 是否能用若干个数恰好凑出和 j

初始化：
  dp[0] = true，其余为 false

转移：
  dp[j] = dp[j] || dp[j - nums[i]]

由于每个数只能用一次，j 必须逆序
```

**C++ 代码**：
```cpp
bool dp[10005];

bool canPartition(vector<int>& nums) {
    int sum = accumulate(nums.begin(), nums.end(), 0);
    if (sum & 1) return false;
    int target = sum / 2;
    dp[0] = true;
    for (int x : nums)
        for (int j = target; j >= x; j--)
            dp[j] = dp[j] || dp[j - x];
    return dp[target];
}
// 调用方式/答案：返回 canPartition(nums)
```

**核心启示**：
- “能否拆成两半”几乎总能想到 **子集和 = 总和一半**。
- 这是把“求最大值”改成“判断可达性”的经典变形。

### 题目 3：目标和（LeetCode 494）

> 题意：给定一个数组，在每个数前面添加 `+` 或 `-`，问有多少种方法使表达式结果等于 `target`。

**分析/状态设计**：设加号集合为 `P`，减号集合为 `N`，则 `sum(P) - sum(N) = target`，又有 `sum(P) + sum(N) = total`，两式相加得 `sum(P) = (total + target) / 2`。于是转化成：有多少个子集和等于该值。

```
若 |target| > total 或 total + target 为奇数，则无解

设 bag = (total + target) / 2

dp[j] = 凑出和 j 的方案数

初始化：
  dp[0] = 1

转移：
  dp[j] += dp[j - nums[i]]

这是“计数版 0/1 背包”，j 仍然逆序
```

**C++ 代码**：
```cpp
long long dp[1005];

int findTargetSumWays(vector<int>& nums, int target) {
    int total = accumulate(nums.begin(), nums.end(), 0);
    if (abs(target) > total || ((total + target) & 1)) return 0;
    int bag = (total + target) / 2;
    dp[0] = 1;
    for (int x : nums)
        for (int j = bag; j >= x; j--)
            dp[j] += dp[j - x];
    return (int)dp[bag];
}
// 调用方式/答案：返回 findTargetSumWays(nums, target)
```

**核心启示**：
- 很多“加减号”“正负选择”问题，都能通过列方程转成子集和。
- 从 `max` 变成 `+=`，就是“求最优值”到“求方案数”的关键跳跃。

### 题目 4：最后一块石头的重量 II（LeetCode 1049）

> 题意：每次选两块石头相撞，最终剩下一块或零块，求最后剩余石头的最小重量。

**分析/状态设计**：本质上是把石头分成两堆，设总和为 `sum`，两堆和分别为 `x` 与 `sum - x`，最终差值为 `|sum - 2x|`。因此目标是找一个尽量接近 `sum/2` 的可达子集和。

```
设 bag = sum / 2

dp[j] = 不超过 j 时，能够凑出的最大子集和

转移：
  dp[j] = max(dp[j], dp[j - stone] + stone)

答案：
  sum - 2 * dp[bag]
```

**C++ 代码**：
```cpp
int dp[1505];

int lastStoneWeightII(vector<int>& stones) {
    int sum = accumulate(stones.begin(), stones.end(), 0);
    int bag = sum / 2;
    for (int x : stones)
        for (int j = bag; j >= x; j--)
            dp[j] = max(dp[j], dp[j - x] + x);
    return sum - 2 * dp[bag];
}
// 调用方式/答案：返回 lastStoneWeightII(stones)
```

**核心启示**：
- “最小差值”常常等价于“尽量靠近一半”。
- 这是 0/1 背包里非常典型的“值=重量本身”的建模方式。

### 🟡 基础级 Level 2：完全背包

### 题目 5：完全背包模板（AcWing 3）

> 题意：有 `N` 种物品，每种物品有重量和价值，每种可以选无限次，背包容量为 `W`，求最大总价值。

**分析/状态设计**：和 0/1 背包唯一不同的是：当前物品还能继续用。因此一维优化时，容量必须从小到大枚举。

```
dp[i][j] = 前 i 类物品、容量不超过 j 的最大价值

转移：
  不选第 i 类：dp[i-1][j]
  再选一件第 i 类：dp[i][j - w[i]] + v[i]

一维优化：
  dp[j] = max(dp[j], dp[j - w[i]] + v[i])
  j 从 w[i] 到 W 正序枚举
```

**C++ 代码**：
```cpp
const int MAXV = 1010;
int dp[MAXV];

void solve() {
    for (int i = 1; i <= n; i++)
        for (int j = w[i]; j <= m; j++)
            dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
}
// 调用方式/答案：solve(); dp[m] 即最大价值
```

**核心启示**：
- 只改一行循环方向，模型就从 0/1 变成了完全背包。
- 记忆口诀：**0/1 逆序，完全正序**。

### 题目 6：零钱兑换（LeetCode 322）

> 题意：给定若干硬币面值，每种硬币可无限使用，问凑成 `amount` 的最少硬币数；若无解返回 `-1`。

**分析/状态设计**：这是 **最小值版完全背包**。物品是硬币，容量是金额，目标是最少使用多少枚。

```
dp[j] = 凑成金额 j 所需的最少硬币数

初始化：
  dp[0] = 0
  其余 dp[j] = INF

转移：
  dp[j] = min(dp[j], dp[j - coin] + 1)

因为每种硬币可重复使用，j 正序枚举
```

**C++ 代码**：
```cpp
const int INF = 1e9;
int dp[10005];

int coinChange(vector<int>& coins, int amount) {
    fill(dp, dp + amount + 1, INF);
    dp[0] = 0;
    for (int c : coins)
        for (int j = c; j <= amount; j++)
            dp[j] = min(dp[j], dp[j - c] + 1);
    return dp[amount] == INF ? -1 : dp[amount];
}
// 调用方式/答案：返回 coinChange(coins, amount)
```

**核心启示**：
- 最大值、最小值、方案数、可达性，本质都能套同一个背包框架。
- 求最小值时，初始化尤其重要：不可达状态必须设成 `INF`。

### 题目 7：零钱兑换 II（LeetCode 518）

> 题意：给定若干硬币面值，每种硬币可无限使用，问有多少种组合可以凑成 `amount`。

**分析/状态设计**：这题不是求最少硬币数，而是求 **组合数**。注意“组合”和“排列”不同：`1+2` 和 `2+1` 算同一种，因此必须让 **硬币在外层循环**。

```
dp[j] = 凑成金额 j 的组合数

初始化：
  dp[0] = 1

转移：
  dp[j] += dp[j - coin]

循环顺序：
  外层枚举硬币，内层 j 正序
  这样统计的是组合，不会重复计算顺序
```

**C++ 代码**：
```cpp
long long dp[5005];

int change(int amount, vector<int>& coins) {
    dp[0] = 1;
    for (int c : coins)
        for (int j = c; j <= amount; j++)
            dp[j] += dp[j - c];
    return (int)dp[amount];
}
// 调用方式/答案：返回 change(amount, coins)
```

**核心启示**：
- 计数题最常见的坑不是公式，而是 **枚举顺序导致重复计数**。
- 记住：**组合数 → 物品在外层；排列数 → 容量在外层**。

### 题目 8：完全平方数（LeetCode 279）

物品：1², 2², 3², ... , k² <= n

### 题目 9：多重背包 I（AcWing 4）

> 题意：有 `N` 种物品，第 `i` 种体积 `w[i]`、价值 `v[i]`、数量上限 `s[i]`。容量为 `V`，求最大价值。数据范围允许朴素做法。

**分析/状态设计**：这是最直接的多重背包。由于数据较小，可以在每个物品处枚举取 `0..s[i]` 件。

```
dp[i][j] = 前 i 种物品、容量不超过 j 的最大价值

转移：
  dp[i][j] = max(dp[i-1][j-k*w[i]] + k*v[i])
  其中 0 <= k <= s[i] 且 k*w[i] <= j

一维写法：
  先逆序枚举 j，再枚举当前物品取 k 件
```

**C++ 代码**：
```cpp
int dp[105];

void solve() {
    for (int i = 1; i <= n; i++)
        for (int j = m; j >= 0; j--)
            for (int k = 1; k <= s[i] && k * w[i] <= j; k++)
                dp[j] = max(dp[j], dp[j - k * w[i]] + k * v[i]);
}
// 调用方式/答案：solve(); dp[m] 即最大价值
```

**核心启示**：
- 多重背包朴素版就是“在 0/1 背包外再套一层件数枚举”。
- 当 `s[i]` 小时，朴素三重循环完全够用。

### 题目 10：多重背包 II（AcWing 5）

> 题意：仍是多重背包，但数据范围更大，朴素 `O(N*V*S)` 会超时，需要优化。

**分析/状态设计**：核心技巧是 **二进制拆分**。把 `s[i]` 拆成 `1, 2, 4, ..., rest`，每一块都视作一件新的 0/1 物品，这样总复杂度降为 `O(N*V*logS)`。

```
例如：
  13 = 1 + 2 + 4 + 6

表示成四个“打包物品”：
  1 件、2 件、4 件、6 件

每个打包物品只能选一次，因此整体转化成 0/1 背包
```

**C++ 代码**：
```cpp
int dp[2005], ww[20005], vv[20005], tot;

void solve() {
    for (int i = 1; i <= n; i++) {
        int c = s[i];
        for (int k = 1; k <= c; k <<= 1) {
            ww[++tot] = k * w[i], vv[tot] = k * v[i];
            c -= k;
        }
        if (c) ww[++tot] = c * w[i], vv[tot] = c * v[i];
    }
    for (int i = 1; i <= tot; i++)
        for (int j = m; j >= ww[i]; j--)
            dp[j] = max(dp[j], dp[j - ww[i]] + vv[i]);
}
// 调用方式/答案：solve(); dp[m] 即最大价值
```

**核心启示**：
- 二进制拆分是多重背包最常见、最稳定的优化。
- 一旦拆完，它就重新回到了你最熟悉的 0/1 背包。

### 题目 11：分组背包（AcWing 9）

> 题意：物品被分成若干组，每组中最多选择一个物品，背包容量为 `W`，求最大价值。

**分析/状态设计**：这题的关键不是“物品”，而是“组”。每一组只能做一次选择：不选，或选其中一个。

```
dp[i][j] = 前 i 组中选择，容量不超过 j 的最大价值

转移：
  1) 不选第 i 组任何物品：dp[i-1][j]
  2) 选第 i 组第 k 件：dp[i-1][j-w[i][k]] + v[i][k]

一维优化：
  按组枚举；对每组让 j 逆序，再枚举组内物品
```

**C++ 代码**：
```cpp
int dp[1005];

void solve() {
    for (int i = 1; i <= n; i++)
        for (int j = m; j >= 0; j--)
            for (int k = 1; k <= sz[i]; k++)
                if (j >= w[i][k])
                    dp[j] = max(dp[j], dp[j - w[i][k]] + v[i][k]);
}
// 调用方式/答案：solve(); dp[m] 即最大价值
```

**核心启示**：
- 分组背包的核心限制是：**同一组不能拿两件**。
- 所以必须先按组处理，而不能把所有物品摊平成普通 0/1 背包。

### 题目 12：摆花（洛谷 P1077）

> 题意：有 `n` 种花，第 `i` 种最多放 `a[i]` 盆，要求总共恰好放 `m` 盆，且摆放顺序按花种编号非递减。求方案数，答案对 `p` 取模。

**分析/状态设计**：因为最终序列是非递减的，所以方案只由“每种花选了几盆”决定。于是它就是一个 **有上限的计数型背包**。朴素转移要枚举 `k`，可用前缀和优化。

```
dp[i][j] = 前 i 种花恰好摆 j 盆的方案数

转移：
  dp[i][j] = Σ dp[i-1][j-k]
  其中 0 <= k <= a[i] 且 k <= j

前缀和优化：
  设 pre[j] = dp[i-1][0] + ... + dp[i-1][j]
  则 dp[i][j] = pre[j] - pre[j-a[i]-1]
```

**C++ 代码**：
```cpp
int dp[105][105], pre[105];

void solve() {
    dp[0][0] = 1;
    for (int i = 1; i <= n; i++) {
        pre[0] = dp[i - 1][0];
        for (int j = 1; j <= m; j++) pre[j] = (pre[j - 1] + dp[i - 1][j]) % mod;
        for (int j = 0; j <= m; j++) {
            int l = max(0, j - a[i]);
            dp[i][j] = (pre[j] - (l ? pre[l - 1] : 0) + mod) % mod;
        }
    }
}
// 调用方式/答案：solve(); dp[n][m] 即方案数
```

**核心启示**：
- “非递减顺序”意味着不用真的排列，只需要统计每种花取多少盆。
- 计数型多重背包如果直接枚举件数太慢，前缀和是很常见的优化。

### 题目 13：一和零（LeetCode 474）

> 题意：给定若干只含 `0/1` 的字符串，最多有 `m` 个 `0` 和 `n` 个 `1`，求最多能选多少个字符串。

**分析/状态设计**：每个字符串只能选一次，但它同时消耗两种资源：`0` 的数量与 `1` 的数量。所以这是 **二维费用 0/1 背包**。

```
dp[j][k] = 最多使用 j 个 0、k 个 1 时，最多能选多少个字符串

对每个字符串统计：
  z = 0 的个数，o = 1 的个数

转移：
  dp[j][k] = max(dp[j][k], dp[j-z][k-o] + 1)

因为每个字符串只能用一次，所以两维都逆序
```

**C++ 代码**：
```cpp
int dp[105][105];

int findMaxForm(vector<string>& strs, int m, int n) {
    for (string s : strs) {
        int z = count(s.begin(), s.end(), '0'), o = (int)s.size() - z;
        for (int j = m; j >= z; j--)
            for (int k = n; k >= o; k--)
                dp[j][k] = max(dp[j][k], dp[j - z][k - o] + 1);
    }
    return dp[m][n];
}
// 调用方式/答案：返回 findMaxForm(strs, m, n)
```

**核心启示**：
- 背包的“容量”不一定只有一维，任何受限资源都可以成为费用维。
- 二维费用背包最重要的是：**每一维都要按模型要求决定顺序**。

### 题目 14：背包问题求方案数（AcWing 11）

> 题意：标准 0/1 背包，要求在最大总价值的前提下，统计最优方案数。

**分析/状态设计**：这题要同时维护两类信息：
- `dp[j]`：容量不超过 `j` 时的最大价值
- `cnt[j]`：达到这个最大价值的方案数

```
dp[j] = 容量不超过 j 的最大价值
cnt[j] = 达到 dp[j] 的方案数

初始化：
  dp[j] = 0
  cnt[j] = 1    // 什么都不选也是一种方案

转移：
  val = dp[j-w[i]] + v[i]
  若 val > dp[j]：
      dp[j] = val, cnt[j] = cnt[j-w[i]]
  若 val == dp[j]：
      cnt[j] += cnt[j-w[i]]
```

**C++ 代码**：
```cpp
const int MOD = 1000000007;
int dp[1005], cnt[1005];

void solve() {
    for (int j = 0; j <= m; j++) cnt[j] = 1;
    for (int i = 1; i <= n; i++)
        for (int j = m; j >= w[i]; j--) {
            int val = dp[j - w[i]] + v[i];
            if (val > dp[j]) dp[j] = val, cnt[j] = cnt[j - w[i]];
            else if (val == dp[j]) cnt[j] = (cnt[j] + cnt[j - w[i]]) % MOD;
        }
}
// 调用方式/答案：solve(); dp[m] 为最大价值，cnt[m] 为最优方案数
```

**核心启示**：
- 当题目既要“最优值”又要“方案数”时，常用做法是 **双数组并行维护**。
- 先比较优劣，再决定是覆盖方案数还是累加方案数。

### 题目 15：背包问题求具体方案（AcWing 12）

> 题意：标准 0/1 背包，不仅要求最大价值，还要输出字典序最小的一组选择方案。

**分析/状态设计**：要输出路径，就不能只保留一维最值。经典做法是建立 **后缀 DP**：`f[i][j]` 表示从第 `i` 件到第 `n` 件中选，容量为 `j` 时的最大价值。然后从小编号往大编号贪心回溯。

```
f[i][j] = 只看 i..n 这些物品，容量不超过 j 的最大价值

转移：
  f[i][j] = f[i+1][j]
  f[i][j] = max(f[i][j], f[i+1][j-w[i]] + v[i])

回溯：
  若 j >= w[i] 且 f[i][j] == f[i+1][j-w[i]] + v[i]
  则选择第 i 件，并令 j -= w[i]
```

**C++ 代码**：
```cpp
int f[1005][1005];
vector<int> path;

void solve() {
    for (int i = n; i >= 1; i--)
        for (int j = 0; j <= m; j++) {
            f[i][j] = f[i + 1][j];
            if (j >= w[i]) f[i][j] = max(f[i][j], f[i + 1][j - w[i]] + v[i]);
        }
    for (int i = 1, j = m; i <= n; i++)
        if (j >= w[i] && f[i][j] == f[i + 1][j - w[i]] + v[i])
            path.push_back(i), j -= w[i];
}
// 调用方式/答案：solve(); path 即字典序最小方案
```

**核心启示**：
- “输出方案”通常意味着你要么记决策，要么保留二维状态用于回溯。
- 想要字典序最小，通常要让小编号物品尽可能先被判断与选中。

### 题目 16：金明的预算方案（洛谷 P1064 / 有依赖的背包）

> 题意：有主件和附件，附件只能在购买对应主件后才能购买，每个主件最多有两个附件。预算固定，求最大满意度。

**分析/状态设计**：依赖关系使它不能直接做普通背包。关键是把每个主件视作一组，枚举“主件单买 / 主+附件1 / 主+附件2 / 主+双附件”这几种合法组合，然后转化为 **分组背包**。

```
每个主件形成一个组，组内合法选择最多 4 种：
  1) 主件
  2) 主件 + 附件1
  3) 主件 + 附件2
  4) 主件 + 附件1 + 附件2

非法情况（只买附件不买主件）不加入组内选项

之后直接做分组背包：
  dp[j] = max(dp[j], dp[j-cost] + value)
```

**C++ 代码**：
```cpp
vector<pair<int, int>> group[65];
int dp[32005];

void build(int i) {
    group[i].push_back({mc[i], mv[i]});
    if (a1[i]) group[i].push_back({mc[i] + c[a1[i]], mv[i] + v[a1[i]]});
    if (a2[i]) group[i].push_back({mc[i] + c[a2[i]], mv[i] + v[a2[i]]});
    if (a1[i] && a2[i]) group[i].push_back({mc[i] + c[a1[i]] + c[a2[i]], mv[i] + v[a1[i]] + v[a2[i]]});
}
void solve() {
    for (int i = 1; i <= gcnt; i++) {
        build(i);
        for (int j = budget; j >= 0; j--)
            for (auto [cost, val] : group[i])
                if (j >= cost) dp[j] = max(dp[j], dp[j - cost] + val);
    }
}
// 调用方式/答案：先整理主件和附件，再调用 solve(); dp[budget] 即最大满意度
```

**核心启示**：
- “有依赖”不一定要上树形 DP；当依赖结构很浅时，先合法组合再分组，往往更简单。
- 这类题的关键是 **建模层面的预处理**，不是 DP 本身有多复杂。

### 题目 17：混合背包（AcWing 7）

> 题意：同一个问题里同时出现 0/1 物品、完全物品和多重物品，求最大价值。

**分析/状态设计**：没有新状态，仍然是一维 `dp[j]`。难点在于识别每种物品属于哪一类，然后调用对应模板；多重物品先二进制拆分。

```
分类规则：
  s = -1：0/1 物品
  s = 0 ：完全物品
  s > 0 ：多重物品

转移：
  0/1  -> j 逆序
  完全 -> j 正序
  多重 -> 先拆分，再按 0/1 处理
```

**C++ 代码**：
```cpp
int dp[10005];

void zeroOne(int c, int val) { for (int j = m; j >= c; j--) dp[j] = max(dp[j], dp[j - c] + val); }
void complete(int c, int val) { for (int j = c; j <= m; j++) dp[j] = max(dp[j], dp[j - c] + val); }
void solve() {
    for (int i = 1; i <= n; i++) {
        if (s[i] == -1) zeroOne(w[i], v[i]);
        else if (s[i] == 0) complete(w[i], v[i]);
        else {
            int c = s[i];
            for (int k = 1; k <= c; k <<= 1) zeroOne(k * w[i], k * v[i]), c -= k;
            if (c) zeroOne(c * w[i], c * v[i]);
        }
    }
}
// 调用方式/答案：solve(); dp[m] 即最大价值
```

**核心启示**：
- 混合背包考的不是新公式，而是你能不能把几个模板熟练拼接。
- 真正的难点通常在“识别类型”与“拆分多重物品”。

### 题目 18：盈利计划（LeetCode 879）

> 题意：有若干犯罪计划，第 `i` 个需要 `group[i]` 个成员，能带来 `profit[i]` 的利润。总人数不超过 `n`，总利润至少达到 `minProfit`，求方案数。

**分析/状态设计**：每个计划只能选一次，所以是 0/1 背包；但资源是“人数”，收益维是“利润下界”。由于利润只要求“至少”，可以把利润维压到 `0..minProfit`，超出的统一截断到 `minProfit`。

```
dp[j][k] = 恰好用了 j 个人，且利润至少达到 k 的方案数

初始化：
  dp[0][0] = 1

转移：
  dp[j][k] += dp[j-group[i]][max(0, k-profit[i])]

其中：
  max(0, k-profit[i]) 表示“补齐到至少 k 还差多少利润”
```

**C++ 代码**：
```cpp
const int MOD = 1000000007;
int dp[105][105];

int profitableSchemes(int n, int minProfit, vector<int>& group, vector<int>& profit) {
    dp[0][0] = 1;
    for (int i = 0; i < (int)group.size(); i++)
        for (int j = n; j >= group[i]; j--)
            for (int k = minProfit; k >= 0; k--)
                dp[j][k] = (dp[j][k] + dp[j - group[i]][max(0, k - profit[i])]) % MOD;
    int ans = 0;
    for (int j = 0; j <= n; j++) ans = (ans + dp[j][minProfit]) % MOD;
    return ans;
}
// 调用方式/答案：返回 profitableSchemes(n, minProfit, group, profit)
```

**核心启示**：
- “至少达到”这类条件，常见技巧是 **截断状态上界**。
- 二维计数背包里，一个维度是费用，一个维度是“已达到多少目标”。

### 题目 19：货币系统（洛谷 P5020 / NOIP2018）

> 题意：给定 `n` 种货币面值，求最小等价货币集合大小。若某个面值能由更小面值组合得到，则它是冗余的。

**分析/状态设计**：先按面值升序排序。如果当前面值 `a[i]` 已经能被之前面值组合出来，那它就是冗余币种；否则它必须保留，并把它加入完全背包的可达性转移。

```
dp[j] = 是否能由“已保留的更小面值”凑出金额 j

流程：
  1) 排序
  2) 若 dp[a[i]] == true，说明 a[i] 冗余
  3) 否则保留它，并做完全背包更新

完全背包更新：
  for j = a[i]..maxA
      dp[j] |= dp[j - a[i]]
```

**C++ 代码**：
```cpp
bool dp[25005];

int solve() {
    sort(a + 1, a + n + 1);
    int ans = 0;
    dp[0] = true;
    for (int i = 1; i <= n; i++) {
        if (dp[a[i]]) continue;
        ans++;
        for (int j = a[i]; j <= a[n]; j++) dp[j] = dp[j] || dp[j - a[i]];
    }
    return ans;
}
// 调用方式/答案：solve() 返回最小等价集合大小
```

**核心启示**：
- 这题表面像数学题，实质是完全背包的可达性判重。
- “排序 + 可达性 DP”是处理冗余元素的经典套路。

### 题目 20：能量石（AcWing 734）

> 题意：每块能量石吃掉需要 `S[i]` 秒，初始能量 `E[i]`，每过 1 秒减少 `L[i]`。一次只能吃一块，求最大总能量。

**分析/状态设计**：这题有两层关键：
1. **先确定吃石头的顺序**：应按 `S[i] * L[j] < S[j] * L[i]` 排序。
2. 排序后再做 0/1 背包：时间是容量，吃第 `i` 块石头会在某个开始时刻损失能量。

```
排序规则：
  若 i 在 j 前更优，则 S[i] * L[j] < S[j] * L[i]

排序后：
  dp[t] = 总耗时恰为 t 时的最大能量

转移：
  dp[t] = max(dp[t], dp[t-S[i]] + max(0, E[i] - (t-S[i]) * L[i]))

解释：
  若第 i 块在时刻 t-S[i] 开始吃，则此时能获得的能量为
  max(0, E[i] - (t-S[i]) * L[i])
```

**C++ 代码**：
```cpp
struct Stone {
    int s, e, l;
    bool operator<(const Stone& other) const { return s * other.l < other.s * l; }
} a[105];
int dp[10005], T;

int solve() {
    sort(a + 1, a + n + 1);
    for (int i = 1; i <= n; i++) T += a[i].s;
    for (int i = 1; i <= n; i++)
        for (int t = T; t >= a[i].s; t--)
            dp[t] = max(dp[t], dp[t - a[i].s] + max(0, a[i].e - (t - a[i].s) * a[i].l));
    return *max_element(dp, dp + T + 1);
}
// 调用方式/答案：solve() 返回最大总能量
```

**核心启示**：
- 有些背包题不是直接上 DP，而是要先做一层 **贪心排序 + 再背包**。
- 看到“价值随时间衰减”，就要立刻想到：顺序很可能会影响状态价值。

## 五、常见变体与技巧总结

### 5.1 恰好装满 vs 不超过

这是初学者最容易忽略、却极其关键的区别。

#### 情况一：容量 **不超过** `m`

最常见的最大值写法：

```cpp
for (int j = 0; j <= m; j++) dp[j] = 0;
```

解释：容量没用满也没关系，因此“什么都不选”的价值是 0，所有状态一开始都合法。

#### 情况二：要求 **恰好装满** `m`

如果要求容量必须正好用完，那么很多状态初始时其实是 **不可达** 的：

```cpp
dp[0] = 0;
for (int j = 1; j <= m; j++) dp[j] = -INF;
```

这样只有从 `dp[0]` 推出来的状态才有效。

#### 情况三：布尔可达性

```cpp
dp[0] = true;
其余为 false
```

#### 一句话总结

| 问法 | 初始化 |
|------|--------|
| 不超过容量，求最大值 | `dp[0..m] = 0` |
| 恰好装满，求最大值 | `dp[0]=0`，其余 `-INF` |
| 恰好装满，问能否做到 | `dp[0]=true`，其余 `false` |

### 5.2 求方案数 vs 求最值

很多人会在这里突然“断档”：
模板明明会写最大值，一换成方案数就不会了。

其实本质只是在问：**状态里存的是什么？**

#### 求最值

```cpp
dp[j] = max(dp[j], dp[j - w] + v);
// 或 min(dp[j], dp[j - w] + cost)
```

#### 求方案数

```cpp
dp[j] += dp[j - w];
```

#### 求是否可达

```cpp
dp[j] = dp[j] || dp[j - w];
```

#### 关键区别

| 目标 | 状态含义 | 常用操作 |
|------|----------|----------|
| 最大值 | 最优价值 | `max` |
| 最小值 | 最小代价/最少件数 | `min` |
| 方案数 | 方案数量 | `+=` |
| 可达性 | 能否凑出 | `||` |

所以不要死记题目，而要记住：

> **背包 DP = 状态定义 + 初始化 + 循环顺序 + 转移运算。**

### 5.3 求具体方案（路径回溯）

当题目不只要最优值，还要输出“选了哪些物品”时，通常有两种方法：

1. **开二维数组记录决策**
2. **保留二维 DP，事后回溯**

最常见的是第二种，也就是题目 15 的套路：

```cpp
if (j >= w[i] && f[i][j] == f[i + 1][j - w[i]] + v[i]) {
    // 说明最优方案里可以选第 i 件
    path.push_back(i);
    j -= w[i];
}
```

如果题目要求 **字典序最小 / 最大**，就要格外注意：
- 你是从前往后回溯，还是从后往前回溯
- 当“选”和“不选”都最优时，优先哪一个

### 5.4 二维费用背包

当一个物品会同时消耗两类资源时，就要把容量扩成二维。

常见场景：
- 最多 `m` 个 `0` 和 `n` 个 `1`
- 最多 `T` 时间且最多 `K` 金额
- 最多 `x` 体积且最多 `y` 重量

通用写法：

```cpp
dp[j][k] = 某种二维容量限制下的最优值

for (每个物品) {
    for (int j = cap1; j >= cost1; j--)
        for (int k = cap2; k >= cost2; k--)
            dp[j][k] = max(dp[j][k], dp[j - cost1][k - cost2] + value);
}
```

如果是完全背包，就把对应维度的枚举改成正序。二维背包本质没有变，只是多加了一层容量维。

## 六、优化方法

### 6.1 滚动数组（空间优化）

很多二维背包状态：

```cpp
dp[i][j] 只依赖 dp[i-1][...] 或 dp[i][j-w]
```

这意味着第一维 `i` 往往可以滚掉，变成一维 `dp[j]`。

#### 什么时候能滚？

- 只依赖上一层：可以滚
- 只依赖当前层更小容量：也可以滚
- 但要通过 **正确的循环顺序** 保证语义不变

#### 记忆法

- **0/1 背包**：逆序滚动
- **完全背包**：正序滚动
- **二维费用 0/1 背包**：两维都逆序

空间优化不是“高端技巧”，而是背包 DP 的基本操作。

### 6.2 二进制拆分（多重背包）

当某种物品最多可用 `s` 次时，朴素写法要枚举 `0..s`，复杂度太高。

可以把 `s` 拆成二进制块：

```
s = 13
=> 1 + 2 + 4 + 6
```

对应成若干件 0/1 物品：
- 重量：`1*w, 2*w, 4*w, 6*w`
- 价值：`1*v, 2*v, 4*v, 6*v`

这样就能覆盖 `1..13` 的任意选取数量，而且新物品个数只有 `O(log s)`。

#### 适用场景

- 多重背包求最值
- 多重背包求可达性
- 混合背包中的有限件物品

这是最常见、最通用的多重背包优化。

### 6.3 单调队列优化

当多重背包的数据更大时，二进制拆分也未必够快，这时会用到 **单调队列优化**。

核心思想是：把容量按照模 `w` 的余数分类。
对固定余数 `r`，只看如下状态：

```
r, r + w, r + 2w, r + 3w, ...
```

设当前物品体积为 `w`、价值为 `v`、数量上限为 `s`，则有：

```
dp_new[r + k*w] = max(dp_old[r + t*w] + (k - t) * v)
其中 k - s <= t <= k
```

移项后得到：

```
dp_new[r + k*w] = max(dp_old[r + t*w] - t*v) + k*v
```

这就变成了一个滑动窗口取最大值的问题，可以用单调队列维护。

#### 什么时候考虑它？

- 多重背包
- 容量很大
- 数量上限也很大
- 二进制拆分仍然不够快

#### 你需要记住的不是代码，而是思路

> **按余数分组，把“枚举件数”改造成“窗口最值”。**

竞赛里这是非常高频的高级优化，但在日常刷题中，二进制拆分通常已经够用。

## 七、终极总结与归纳

### 7.1 背包 DP 的本质

背包 DP 的本质，可以压缩成一句话：

> **在资源限制下，对一组候选决策做阶段性最优合并。**

不管题目包装成硬币、石头、字符串、利润、人数、附件还是时间，本质都在问：
- 资源有多少？
- 每个选择消耗多少资源？
- 选择后带来什么收益 / 方案数 / 可达性？

只要能回答这三问，模型通常就已经出来了。

### 7.2 五大核心要点

1. **先认模型，再写代码**
   - 0/1、完全、多重、分组、混合，先分类，后转移。

2. **循环顺序决定语义**
   - 0/1 逆序，完全正序；顺序错了，答案往往全错。

3. **初始化决定“是否可达”**
   - 不超过容量通常全 0
   - 恰好装满常常要 `-INF`
   - 可达性则用 `true/false`

4. **目标不同，运算不同**
   - 最大值 `max`
   - 最小值 `min`
   - 方案数 `+=`
   - 可达性 `||`

5. **很多题先转化，再背包**
   - 等和划分、目标和、货币系统、预算方案、能量石，真正难点都在建模。

### 7.5 口诀

> 🎒 **背包 DP 口诀**：
>
> **先定物品与容量，状态目标写清爽；**
> **0/1 逆序防重复，完全正序可连装；**
> **多重拆成二进块，分组同层只选一项；**
> **求值用 max / min，计数就把加法上；**
> **恰装先设负无穷，二维费用再开一框；**
> **若见依赖先建组，若见衰减先排顺；**
> **顺序一错全盘乱，本质就是资源分配场。**
