## 一、什么是区间 DP

**区间 DP**，就是把状态定义在一个区间上：`dp[i][j]` 表示区间 `[i, j]` 的某种最优解、最少代价、最大收益、最多匹配数或方案数。

你可以把它理解为一句话：

> **当题目的操作对象是“一段连续区间”，并且大区间的答案可以由小区间合并出来时，就很可能是区间 DP。**

### 为什么区间适合 DP

```
区间 [1, 5] 的拆分方式：

[1,5] → [1,1] + [2,5]    在 k=1 处断开
     → [1,2] + [3,5]    在 k=2 处断开
     → [1,3] + [4,5]    在 k=3 处断开
     → [1,4] + [5,5]    在 k=4 处断开

每种拆分方式都产生两个更小的子问题 → 取最优
```

区间之所以适合做 DP，核心在于三点：

1. **最优子结构**：大区间的最优解可以由小区间的最优解组合出来。
2. **无后效性**：一旦某个小区间的最优值确定，它不会因为外部区间变化而被推翻。
3. **可枚举决策点**：常见决策是“在哪个位置切一刀”“最后处理哪个点”“两端是否匹配”。

### 与其他常见 DP 的对比

| DP 类型 | 常见状态 | 决策对象 | 典型枚举顺序 | 代表问题 |
|---------|----------|----------|--------------|----------|
| 线性 DP | `dp[i]` | 第 `i` 个位置怎么选 | 从左到右 | LIS、打家劫舍 |
| 区间 DP | `dp[i][j]` | 区间内如何分割/匹配 | 按区间长度递增 | 石子合并、回文问题 |
| 树形 DP | `dp[u][state]` | 如何合并子树 | DFS 后序 | 没有上司的舞会 |
| 背包 DP | `dp[j]` / `dp[i][j]` | 当前物品取几件 | 枚举物品 + 枚举容量 | 0/1 背包、完全背包 |

区间 DP 的本质可以概括成一句话：

> **把“分治拆区间”的过程，变成“按长度递推区间”的动态规划。**

## 二、核心思想与前置知识

### 2.1 三个核心问题

做任何区间 DP 题，都先强迫自己写出下面三句话：

1. **状态怎么定义？**
   - `dp[i][j]` 到底表示区间 `[i, j]` 上的什么量？
   - 是最小代价、最大收益、最长长度，还是方案数？

2. **转移怎么写？**
   - 是枚举分割点 `k`：`dp[i][k] + dp[k+1][j]`？
   - 还是看两端关系：`s[i]` 和 `s[j]` 能不能匹配？
   - 还是逆向考虑“最后一步操作”？

3. **答案在哪里？**
   - 是 `dp[1][n]`、`dp[0][n-1]`，还是所有长度为 `n` 的区间中的最优值？

把这三问先答出来，代码基本就只剩模板套壳了。

### 2.2 两种经典模型

区间 DP 最常见的模型只有两大类。

**模型一：合并型（Merge）**

把区间 `[i, j]` 在某个位置 `k` 切开，左边和右边分别求解，再把两部分合并。

```
dp[i][j] = opt { dp[i][k] + dp[k+1][j] + cost(i, j) }    (i ≤ k < j)
```

典型题目：石子合并、矩阵链乘法、最优三角剖分。

**模型二：消去/匹配型（Eliminate / Match）**

关注区间两端元素之间的关系：如果能匹配，就缩掉两端；如果不能匹配，就尝试舍弃一端或在中间分割。

```
若 s[i] == s[j]：dp[i][j] = dp[i+1][j-1] + 某种贡献
否则：           dp[i][j] = opt { dp[i+1][j], dp[i][j-1], ... }
```

典型题目：最长回文子序列、最少插入回文、括号匹配、奇怪的打印机。

### 2.3 枚举顺序（按区间长度从小到大）

区间 DP 最容易错的点，不在公式，而在 **枚举顺序**。

因为 `dp[i][j]` 往往依赖更短的区间，所以必须先算短区间，再算长区间：

```cpp
for (int len = 2; len <= n; len++) {          // 区间长度
    for (int i = 1; i + len - 1 <= n; i++) {  // 左端点
        int j = i + len - 1;                  // 右端点
        // 用更短区间更新 dp[i][j]
    }
}
```

记忆方法很简单：

> **长度是阶段，左端点是位置，分割点是决策。**

### 2.4 通用框架代码

下面给出最标准的区间合并模板。这一版保留完整 `#include` 与 `main`，方便你直接运行理解。

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 505;
int n;
int a[MAXN], pre[MAXN];
int dp[MAXN][MAXN];

int main() {
    cin >> n;
    for (int i = 1; i <= n; i++) {
        cin >> a[i];
        pre[i] = pre[i - 1] + a[i];
    }

    for (int i = 1; i <= n; i++) dp[i][i] = 0;

    for (int len = 2; len <= n; len++) {
        for (int i = 1; i + len - 1 <= n; i++) {
            int j = i + len - 1;
            dp[i][j] = INT_MAX;
            int cost = pre[j] - pre[i - 1];
            for (int k = i; k < j; k++) {
                dp[i][j] = min(dp[i][j], dp[i][k] + dp[k + 1][j] + cost);
            }
        }
    }

    cout << dp[1][n] << '\n';
    return 0;
}
```

你可以把它拆成 4 步：

1. 定义 `dp[i][j]` 的语义。
2. 确定 base case（单点区间 / 空区间）。
3. 按长度递增枚举区间。
4. 在每个区间内枚举决策点并更新答案。

## 三、两大区间 DP 模型

### 3.1 合并型（Merge）

合并型最典型的思考方式是：

> **最后一次操作把区间切成两段。**

只要题目出现下面这些信号，优先想合并型：

- 合并相邻两段
- 选择分割点
- 拆成左右两个子区间
- 代价与整个区间有关（如区间和、矩阵维度、端点权值）

```
状态定义：
  dp[i][j] = 区间 [i, j] 的最优解

典型转移：
  dp[i][j] = opt { dp[i][k] + dp[k+1][j] + cost(i, j) }

答案：
  dp[1][n] 或 dp[0][n-1]
```

**简版模板**：

```cpp
for (int len = 2; len <= n; len++)
    for (int i = 1; i + len - 1 <= n; i++) {
        int j = i + len - 1;
        dp[i][j] = INF;
        for (int k = i; k < j; k++)
            dp[i][j] = min(dp[i][j], dp[i][k] + dp[k + 1][j] + cost(i, j));
    }
```

### 3.2 消去/匹配型（Eliminate / Match）

消去型最典型的思考方式是：

> **先看区间两端能不能一起处理。**

如果两端能配对、能消去、能共用一次操作，就把问题缩成中间区间；否则就舍弃一端，或者再补一个分割转移。

```
状态定义：
  dp[i][j] = 区间 [i, j] 的最优解

典型转移：
  若两端匹配：dp[i][j] = dp[i+1][j-1] + gain
  若两端不匹配：dp[i][j] = opt { dp[i+1][j], dp[i][j-1], ... }
```

**简版模板**：

```cpp
for (int len = 2; len <= n; len++)
    for (int i = 0; i + len - 1 < n; i++) {
        int j = i + len - 1;
        if (s[i] == s[j])
            dp[i][j] = dp[i + 1][j - 1] + gain;
        else
            dp[i][j] = opt(dp[i + 1][j], dp[i][j - 1]);
    }
```

有三种常见变体值得单独记住：

1. **开区间**：如戳气球，状态定义成 `dp[i][j]` 表示开区间 `(i, j)` 的答案。
2. **混合型**：如括号匹配，既有“两端匹配”，又要“枚举分割点”。
3. **环形问题**：如环形石子合并、多边形游戏，需要先 **破环成链**。

### 题目 1：石子合并（直线版，经典模板）

> 题意：有 `n` 堆石子排成一行，每次可以合并 **相邻** 的两堆，代价为两堆石子数量之和。求将所有石子合并为一堆的 **最小代价**。

**分析**：这是区间 DP 最经典的入门题。关键观察是：**最后一次合并**，一定是把某个位置 `k` 左边的一整堆和右边的一整堆合起来。

```
状态定义：
  dp[i][j] = 将第 i 堆到第 j 堆合并为一堆的最小代价

转移方程：
  dp[i][j] = min { dp[i][k] + dp[k+1][j] + sum(i, j) }    (i ≤ k < j)

base case：
  dp[i][i] = 0

答案：
  dp[1][n]
```

其中 `sum(i, j)` 是区间 `[i, j]` 的石子总数，用前缀和可以 `O(1)` 求出。

```
石子: [3, 4, 2, 5]

dp[1][1]=0  dp[2][2]=0  dp[3][3]=0  dp[4][4]=0

长度 2:
  dp[1][2] = dp[1][1] + dp[2][2] + (3+4) = 7
  dp[2][3] = dp[2][2] + dp[3][3] + (4+2) = 6
  dp[3][4] = dp[3][3] + dp[4][4] + (2+5) = 7

长度 3:
  dp[1][3] = min(
    dp[1][1] + dp[2][3] + 9 = 0+6+9 = 15,   // k=1
    dp[1][2] + dp[3][3] + 9 = 7+0+9 = 16    // k=2
  ) = 15
  dp[2][4] = min(
    dp[2][2] + dp[3][4] + 11 = 0+7+11 = 18, // k=2
    dp[2][3] + dp[4][4] + 11 = 6+0+11 = 17  // k=3
  ) = 17

长度 4:
  dp[1][4] = min(
    dp[1][1] + dp[2][4] + 14 = 0+17+14 = 31, // k=1
    dp[1][2] + dp[3][4] + 14 = 7+7+14  = 28, // k=2
    dp[1][3] + dp[4][4] + 14 = 15+0+14 = 29  // k=3
  ) = 28

答案: dp[1][4] = 28
```

**C++ 代码**：
```cpp
const int MAXN = 305;
int a[MAXN], pre[MAXN];
int dp[MAXN][MAXN];

void solve(int n) {
    for (int len = 2; len <= n; len++)
        for (int i = 1; i + len - 1 <= n; i++) {
            int j = i + len - 1;
            dp[i][j] = 0x3f3f3f3f;
            int sum = pre[j] - pre[i - 1];
            for (int k = i; k < j; k++)
                dp[i][j] = min(dp[i][j], dp[i][k] + dp[k + 1][j] + sum);
        }
}
// pre[i] = a[1] + ... + a[i]; solve(n); 答案：dp[1][n]
```

**核心启示**：
- 石子合并的本质，不是“每次怎么合”，而是 **最后一刀切在哪里**。
- 合并型区间 DP 往往都要枚举分割点 `k`。
- 只要代价和整个区间有关，前缀和通常就是标配。

### 题目 2：矩阵链乘法（经典模型）

> 题意：给定 `n` 个矩阵的维度序列 `p[0..n]`，第 `i` 个矩阵的维度为 `p[i-1] × p[i]`。求将所有矩阵相乘的 **最少标量乘法次数**。

**分析**：这题和石子合并的区间结构几乎一模一样，区别只在于“合并代价”不再是区间和，而是由左右两边结果矩阵的维度决定。

```
状态定义：
  dp[i][j] = 计算矩阵 i 到矩阵 j 的乘积所需的最少乘法次数

转移方程：
  dp[i][j] = min { dp[i][k] + dp[k+1][j] + p[i-1] * p[k] * p[j] }    (i ≤ k < j)

base case：
  dp[i][i] = 0

答案：
  dp[1][n]
```

在分割点 `k` 处：
- 左边结果矩阵的维度是 `p[i-1] × p[k]`
- 右边结果矩阵的维度是 `p[k] × p[j]`
- 把它们乘起来的代价就是 `p[i-1] * p[k] * p[j]`

**C++ 代码**：
```cpp
const int MAXN = 505;
int p[MAXN];
long long dp[MAXN][MAXN];

void solve(int n) {
    for (int len = 2; len <= n; len++)
        for (int i = 1; i + len - 1 <= n; i++) {
            int j = i + len - 1;
            dp[i][j] = (1LL << 62);
            for (int k = i; k < j; k++) {
                long long cost = dp[i][k] + dp[k + 1][j]
                               + 1LL * p[i - 1] * p[k] * p[j];
                dp[i][j] = min(dp[i][j], cost);
            }
        }
}
// p[0..n] 为维度序列；solve(n); 答案：dp[1][n]
```

**核心启示**：
- 这题说明：**模型不变，代价函数变了**，仍然是合并型区间 DP。
- 关键不是记公式，而是看懂“分割后左右矩阵的维度是什么”。
- 乘法次数可能很大，实战里要优先想到 `long long`。

### 题目 3：最长回文子序列（LeetCode 516）

> 题意：给定字符串 `s`，求其 **最长回文子序列** 的长度。

**分析**：这是消去型区间 DP 的典型代表。我们只需要盯住区间两端：如果两端字符相等，就把它们一起纳入答案；否则，必须舍弃一端。

```
状态定义：
  dp[i][j] = 字符串 s[i..j] 中最长回文子序列的长度

转移方程：
  若 s[i] == s[j]：dp[i][j] = dp[i+1][j-1] + 2
  若 s[i] != s[j]：dp[i][j] = max(dp[i+1][j], dp[i][j-1])

base case：
  dp[i][i] = 1

答案：
  dp[0][n-1]
```

```
s = "bbbab"

base: dp[i][i] = 1  对所有 i

长度 2:
  dp[0][1]: s[0]='b'==s[1]='b' → dp[1][0](=0) + 2 = 2
  dp[1][2]: s[1]='b'==s[2]='b' → 2
  dp[2][3]: s[2]='b'!=s[3]='a' → max(dp[3][3], dp[2][2]) = 1
  dp[3][4]: s[3]='a'!=s[4]='b' → max(dp[4][4], dp[3][3]) = 1

长度 3:
  dp[0][2]: s[0]='b'==s[2]='b' → dp[1][1] + 2 = 3
  dp[1][3]: s[1]='b'!=s[3]='a' → max(dp[2][3], dp[1][2]) = 2
  dp[2][4]: s[2]='b'==s[4]='b' → dp[3][3] + 2 = 3

长度 4:
  dp[0][3]: s[0]='b'!=s[3]='a' → max(dp[1][3], dp[0][2]) = 3
  dp[1][4]: s[1]='b'==s[4]='b' → dp[2][3] + 2 = 3

长度 5:
  dp[0][4]: s[0]='b'==s[4]='b' → dp[1][3] + 2 = 4

答案: dp[0][4] = 4  （回文子序列为 "bbbb"）
```

**C++ 代码**：
```cpp
int longestPalindromeSubseq(string s) {
    int n = s.size();
    vector<vector<int>> dp(n, vector<int>(n, 0));

    for (int i = 0; i < n; i++) dp[i][i] = 1;

    for (int len = 2; len <= n; len++)
        for (int i = 0; i + len - 1 < n; i++) {
            int j = i + len - 1;
            if (s[i] == s[j])
                dp[i][j] = dp[i + 1][j - 1] + 2;
            else
                dp[i][j] = max(dp[i + 1][j], dp[i][j - 1]);
        }

    return dp[0][n - 1];
}
// 答案：longestPalindromeSubseq(s)
```

**核心启示**：
- 回文类区间 DP，第一反应永远是：**看两端能不能配**。
- 若两端匹配，问题立刻缩成内部区间；若不匹配，常常是“舍左”或“舍右”。
- 这是“按长度递推二维表”的标准训练题。

### 题目 4：最少插入次数构成回文串（LeetCode 1312）

> 题意：给定字符串 `s`，每次可以在任意位置插入一个字符，求使 `s` 成为回文串的 **最少插入次数**。

**分析**：这题和最长回文子序列密切相关，因为
`最少插入次数 = n - 最长回文子序列长度`。

不过更推荐你直接写区间 DP，这样更能锻炼“状态语义”能力。

```
状态定义：
  dp[i][j] = 使 s[i..j] 变为回文串的最少插入次数

转移方程：
  若 s[i] == s[j]：dp[i][j] = dp[i+1][j-1]
  若 s[i] != s[j]：dp[i][j] = min(dp[i+1][j], dp[i][j-1]) + 1

base case：
  dp[i][i] = 0

答案：
  dp[0][n-1]
```

当 `s[i] != s[j]` 时有两种补法：
- `dp[i+1][j] + 1`：在右端插入 `s[i]`，让左端字符找到配对
- `dp[i][j-1] + 1`：在左端插入 `s[j]`，让右端字符找到配对

**C++ 代码**：
```cpp
int minInsertions(string s) {
    int n = s.size();
    vector<vector<int>> dp(n, vector<int>(n, 0));

    for (int len = 2; len <= n; len++)
        for (int i = 0; i + len - 1 < n; i++) {
            int j = i + len - 1;
            if (s[i] == s[j])
                dp[i][j] = dp[i + 1][j - 1];
            else
                dp[i][j] = min(dp[i + 1][j], dp[i][j - 1]) + 1;
        }

    return dp[0][n - 1];
}
// 答案：minInsertions(s)
```

**核心启示**：
- 和回文有关的问题，很多都能落到“看两端是否匹配”。
- 这题的 `+1` 不是删字符，而是 **补一个匹配字符**。
- 同一类区间结构，只要改变“状态含义”，就能解决完全不同的问题。

### 题目 5：括号匹配（最多匹配数，经典括号 DP）

> 题意：给定一个只包含 `(`、`)`、`[`、`]` 的字符串，求最多能匹配的括号数。

**分析**：这题非常经典，因为它同时包含两种转移：
一方面，若区间两端能匹配，可以直接配成一对；另一方面，最优答案也可能来自于把区间拆成两段分别匹配后再相加。

```
状态定义：
  dp[i][j] = 区间 [i, j] 中最多能匹配的括号数

转移方程：
  1. 若 s[i] 和 s[j] 匹配：
     dp[i][j] = dp[i+1][j-1] + 2

  2. 枚举分割点：
     dp[i][j] = max(dp[i][j], dp[i][k] + dp[k+1][j])    (i ≤ k < j)

base case：
  dp[i][i] = 0
  空区间也视为 0

答案：
  dp[0][n-1]
```

也就是说，这题属于 **消去型 + 合并型** 的混合写法。

**C++ 代码**：
```cpp
bool match(char a, char b) {
    return (a == '(' && b == ')') || (a == '[' && b == ']');
}

int maxMatched(string s) {
    int n = s.size();
    if (n == 0) return 0;
    vector<vector<int>> dp(n, vector<int>(n, 0));

    for (int len = 2; len <= n; len++)
        for (int i = 0; i + len - 1 < n; i++) {
            int j = i + len - 1;
            if (match(s[i], s[j]))
                dp[i][j] = (i + 1 <= j - 1 ? dp[i + 1][j - 1] : 0) + 2;
            for (int k = i; k < j; k++)
                dp[i][j] = max(dp[i][j], dp[i][k] + dp[k + 1][j]);
        }

    return dp[0][n - 1];
}
// 答案：maxMatched(s)
```

**核心启示**：
- 区间 DP 不一定只有一种转移，很多题是 **端点匹配 + 区间分割** 同时存在。
- “最优解不一定从两端开始”，这是括号类题的关键警报。
- 只要出现“拼起来更优”的可能，就要想到枚举分割点。

### 题目 6：戳气球（LeetCode 312）

> 题意：有 `n` 个气球排成一行，编号 `0` 到 `n-1`。每次戳破气球 `i`，获得 `nums[i-1] * nums[i] * nums[i+1]` 枚硬币（边界视为 `1`）。求戳破所有气球获得的 **最大硬币数**。

**分析**：这题正着想很难，因为你先戳谁，会影响后面的左右邻居。正确思路是 **逆向思考**：不要想“先戳哪个”，而要想“最后戳哪个”。

假设在开区间 `(i, j)` 中，最后戳破的是 `k`：
- 那么在最后这一刻，`k` 的左邻居一定是 `i`
- 右邻居一定是 `j`
- 所得硬币就是 `a[i] * a[k] * a[j]`

```
状态定义：
  dp[i][j] = 戳破开区间 (i, j) 内所有气球的最大硬币数

转移方程：
  dp[i][j] = max { dp[i][k] + dp[k][j] + a[i] * a[k] * a[j] }    (i < k < j)

base case：
  dp[i][i+1] = 0

答案：
  dp[0][n+1]
```

为什么用 **开区间**？因为这样一来，当我们固定“最后戳 `k`”时，`k` 的左右邻居必然就是边界 `i` 和 `j`，子问题 `dp[i][k]` 与 `dp[k][j]` 才会相互独立。

**C++ 代码**：
```cpp
int maxCoins(vector<int>& nums) {
    int n = nums.size();
    vector<int> a(n + 2, 1);
    for (int i = 0; i < n; i++) a[i + 1] = nums[i];

    vector<vector<int>> dp(n + 2, vector<int>(n + 2, 0));

    for (int len = 3; len <= n + 2; len++)
        for (int i = 0; i + len - 1 <= n + 1; i++) {
            int j = i + len - 1;
            for (int k = i + 1; k < j; k++)
                dp[i][j] = max(dp[i][j], dp[i][k] + dp[k][j] + a[i] * a[k] * a[j]);
        }

    return dp[0][n + 1];
}
// 首尾补 1；答案：maxCoins(nums)
```

**核心启示**：
- 这题最值钱的不是代码，而是思维转化：**先想最后一步**。
- 开区间是为了让“最后戳哪个”的子问题独立，这是区间 DP 的经典技巧。
- 如果正向状态彼此纠缠，逆向定义通常能把问题拆开。

### 题目 7：石子合并（环形版，经典环形变体）

> 题意：有 `n` 堆石子排成一个 **环**，每次可以合并相邻的两堆，代价为两堆石子数量之和。求最小代价。

**分析**：环形问题最大的麻烦是“起点不固定”。标准处理方法叫 **破环成链**：把长度为 `n` 的环复制一份，变成长度为 `2n` 的链。然后在链上做区间 DP，再枚举所有长度为 `n` 的连续区间取最优。

```
状态定义：
  dp[i][j] = 合并链上第 i 堆到第 j 堆的最小代价

转移方程：
  与直线版完全相同：
  dp[i][j] = min { dp[i][k] + dp[k+1][j] + sum(i, j) }

答案：
  min { dp[i][i+n-1] }    (1 ≤ i ≤ n)
```

```
原始环:  1 - 2 - 3 - 4 - 1 (首尾相连)

破环成链: 1 - 2 - 3 - 4 - 1 - 2 - 3 - 4

所有长度为 n 的连续子区间:
  [1,4], [2,5], [3,6], [4,7]
    ↓      ↓      ↓      ↓
  从1开始  从2开始  从3开始  从4开始
```

**C++ 代码**：
```cpp
const int MAXN = 605;
int a[MAXN], pre[MAXN];
int dp[MAXN][MAXN];

int solveRing(int n) {
    int m = 2 * n;
    for (int len = 2; len <= n; len++)
        for (int i = 1; i + len - 1 <= m; i++) {
            int j = i + len - 1;
            dp[i][j] = 0x3f3f3f3f;
            int sum = pre[j] - pre[i - 1];
            for (int k = i; k < j; k++)
                dp[i][j] = min(dp[i][j], dp[i][k] + dp[k + 1][j] + sum);
        }

    int ans = 0x3f3f3f3f;
    for (int i = 1; i <= n; i++)
        ans = min(ans, dp[i][i + n - 1]);
    return ans;
}
// 先把 a[1..n] 复制到 a[n+1..2n]，重算 pre；答案：solveRing(n)
```

**核心启示**：
- 环形区间题最常见套路就是：**复制一份，破环成链**。
- 破环之后，转移公式通常不变，变的是答案位置。
- 环没有天然起点，所以要枚举所有长度为 `n` 的候选区间。

### 题目 8：最优三角剖分（LeetCode 1039）

> 题意：给定凸多边形的顶点值 `values[0..n-1]`，将其三角剖分，每个三角形的得分为三个顶点值的乘积。求所有三角形得分之和的 **最小值**。

**分析**：固定一条边 `(i, j)` 后，必须选择一个顶点 `k`（`i < k < j`）和它组成三角形 `(i, k, j)`。这样整个多边形就被拆成左右两个更小的多边形。

```
状态定义：
  dp[i][j] = 顶点 i 到顶点 j 构成的多边形的最优三角剖分得分

转移方程：
  dp[i][j] = min { dp[i][k] + dp[k][j] + values[i] * values[k] * values[j] }    (i < k < j)

base case：
  dp[i][i+1] = 0

答案：
  dp[0][n-1]
```

这题本质上和矩阵链乘法极其相似，都是“固定两端，再选中间点分治”。

**C++ 代码**：
```cpp
int minScoreTriangulation(vector<int>& values) {
    int n = values.size();
    vector<vector<int>> dp(n, vector<int>(n, 0));

    for (int len = 3; len <= n; len++)
        for (int i = 0; i + len - 1 < n; i++) {
            int j = i + len - 1;
            dp[i][j] = 0x3f3f3f3f;
            for (int k = i + 1; k < j; k++)
                dp[i][j] = min(dp[i][j], dp[i][k] + dp[k][j] + values[i] * values[k] * values[j]);
        }

    return dp[0][n - 1];
}
// 答案：minScoreTriangulation(values)
```

**核心启示**：
- 当题目是“固定区间两端，再在中间挑一个点”，几乎就是区间合并模型。
- 三角剖分、矩阵链乘法、最优二叉搜索树，本质都很像。
- 看到“凸多边形 + 划分 + 最优值”，要优先往区间 DP 上靠。

### 题目 9：奇怪的打印机（LeetCode 664）

> 题意：打印机每次可以从位置 `i` 到位置 `j` 打印同一个字符（会覆盖原有内容）。求打印出目标字符串 `s` 所需的 **最少打印次数**。

**分析**：先想一个朴素上界：把 `s[j]` 单独打印一次，那么
`dp[i][j] = dp[i][j-1] + 1`。

但如果存在某个 `k`（`i ≤ k < j`）满足 `s[k] == s[j]`，那么打印 `s[k]` 的那次操作有机会顺便把 `s[j]` 也覆盖出来，从而省掉一次打印。

```
状态定义：
  dp[i][j] = 打印 s[i..j] 所需的最少次数

转移方程：
  初始：dp[i][j] = dp[i][j-1] + 1

  若存在 i ≤ k < j 且 s[k] == s[j]：
  dp[i][j] = min(dp[i][j], dp[i][k] + dp[k+1][j-1])

base case：
  dp[i][i] = 1

答案：
  dp[0][n-1]
```

这里最难理解的一点是：
当 `s[k] == s[j]` 时，我们可以把 `s[j]` 放到“打印 `s[k]` 的那一轮”里一起完成；
而中间的 `s[k+1..j-1]` 以后再覆盖打印即可。

**C++ 代码**：
```cpp
int strangePrinter(string s) {
    int n = s.size();
    if (n == 0) return 0;
    vector<vector<int>> dp(n, vector<int>(n, 0));

    for (int i = 0; i < n; i++) dp[i][i] = 1;

    for (int len = 2; len <= n; len++)
        for (int i = 0; i + len - 1 < n; i++) {
            int j = i + len - 1;
            dp[i][j] = dp[i][j - 1] + 1;
            for (int k = i; k < j; k++) {
                if (s[k] == s[j]) {
                    int mid = (k + 1 <= j - 1 ? dp[k + 1][j - 1] : 0);
                    dp[i][j] = min(dp[i][j], dp[i][k] + mid);
                }
            }
        }

    return dp[0][n - 1];
}
// 答案：strangePrinter(s)
```

**核心启示**：
- 这题提醒你：区间 DP 的“匹配”不一定是回文式的两端相配，也可以是 **共享一次操作**。
- 先给一个朴素上界，再用特殊关系优化，是很常见的写法。
- 当相同字符能复用一次操作时，要格外警惕这种“合并打印”转移。

### 题目 10：多边形游戏（NOIP 经典）

> 题意：一个 `n` 边形，每个顶点有一个整数，每条边有一个运算符（加或乘）。第一步删除一条边，之后每步选一条边，对两端顶点做运算，结果替代两端。求最终结果的 **最大值**。

**分析**：这道题是综合题中的经典：它同时用到了 **破环成链** 和 **最大值 / 最小值同时维护**。

为什么要同时维护最小值？因为一旦有乘法和负数，最大值可能来自“两个最小值相乘”。

```
状态定义：
  dpmax[i][j] = 区间 [i, j] 能得到的最大值
  dpmin[i][j] = 区间 [i, j] 能得到的最小值

转移方程：
  枚举分割点 k，对应运算符 op[k+1]

  若为加法：
    dpmax[i][j] = max(dpmax[i][j], dpmax[i][k] + dpmax[k+1][j])
    dpmin[i][j] = min(dpmin[i][j], dpmin[i][k] + dpmin[k+1][j])

  若为乘法：
    从四种组合中同时更新最大最小：
    max*max, max*min, min*max, min*min

答案：
  枚举起点 i，取 max { dpmax[i][i+n-1] }
```

这题还要先破环成链：把顶点和运算符都复制一份，再在长度为 `2n` 的链上做 DP。

**C++ 代码**：
```cpp
const int MAXN = 205;
const long long NEG_INF = -(1LL << 60);
const long long POS_INF = 1LL << 60;

long long val[MAXN];
char op[MAXN];
long long dpmax[MAXN][MAXN], dpmin[MAXN][MAXN];

void solve(int n) {
    int m = 2 * n;
    for (int i = 1; i <= m; i++) {
        dpmax[i][i] = dpmin[i][i] = val[i];
        for (int j = i + 1; j <= m; j++) {
            dpmax[i][j] = NEG_INF;
            dpmin[i][j] = POS_INF;
        }
    }

    for (int len = 2; len <= n; len++)
        for (int i = 1; i + len - 1 <= m; i++) {
            int j = i + len - 1;
            for (int k = i; k < j; k++) {
                if (op[k + 1] == 't') {
                    dpmax[i][j] = max(dpmax[i][j], dpmax[i][k] + dpmax[k + 1][j]);
                    dpmin[i][j] = min(dpmin[i][j], dpmin[i][k] + dpmin[k + 1][j]);
                } else {
                    long long a = dpmax[i][k], b = dpmin[i][k];
                    long long c = dpmax[k + 1][j], d = dpmin[k + 1][j];
                    long long x1 = a * c, x2 = a * d, x3 = b * c, x4 = b * d;
                    dpmax[i][j] = max(dpmax[i][j], max(max(x1, x2), max(x3, x4)));
                    dpmin[i][j] = min(dpmin[i][j], min(min(x1, x2), min(x3, x4)));
                }
            }
        }
}
// 先把 val、op 都复制一份；答案：max(dpmax[i][i+n-1])
```

**核心启示**：
- 只要题目里同时出现 **乘法 + 负数**，就要警惕“最大值来自最小值”的情况。
- 环形题先破环成链，几乎是固定动作。
- 综合题往往不是新模型，而是把多个区间 DP 技巧叠在一起。

## 五、常用思考方法总结

### 5.1 五步思考法

遇到区间 DP 题，建议固定按下面五步走：

```
Step 1：定义状态
  └─ dp[i][j] 表示区间 [i, j]（或 (i, j)）上的什么最优解？

Step 2：判断模型
  ├─ 合并型：枚举分割点 k
  ├─ 消去型：看两端能否匹配
  └─ 混合型：端点匹配 + 分割点枚举

Step 3：写出边界
  ├─ 单元素区间 dp[i][i] = ?
  └─ 空区间 dp[i][i-1] = ?（有些题会用到）

Step 4：确定顺序
  └─ 按区间长度从小到大递推

Step 5：找答案位置
  ├─ 直线问题：dp[1][n] / dp[0][n-1]
  └─ 环形问题：枚举所有长度为 n 的区间
```

### 5.2 常见状态设计模式

| 状态设计 | 含义 | 典型题目 | 备注 |
|----------|------|----------|------|
| `dp[i][j]` | 闭区间 `[i, j]` 的最优解 | 石子合并、矩阵链、回文类 | 最常见 |
| `dp[i][j]` | 开区间 `(i, j)` 的最优解 | 戳气球 | 常用于“最后一步” |
| `dpmax[i][j] / dpmin[i][j]` | 同时维护最大值和最小值 | 多边形游戏 | 处理负数乘法 |
| `dp[i][j] + 枚举 k` | 两段合并后取最优 | 三角剖分、括号匹配 | 合并型核心 |
| `dp[i][j] + 两端关系` | 端点匹配/不匹配 | 最长回文、最少插入 | 消去型核心 |

补充记忆：
- **闭区间** 适合描述“这一整段都要处理完”。
- **开区间** 适合描述“边界留着不动，中间才是待处理对象”。
- 如果结果可能受正负号影响，不要只维护一个最值。

### 5.3 转移方程的典型形式

把区间 DP 常见转移压缩成下面这张表，基本就够应对大多数题了：

| 类型 | 典型转移 | 代表题 |
|------|----------|--------|
| 区间合并 | `dp[i][j] = opt{dp[i][k] + dp[k+1][j] + cost}` | 石子合并、矩阵链 |
| 两端匹配 | `s[i] == s[j] -> dp[i+1][j-1] + val` | 最长回文子序列 |
| 两端不匹配 | `dp[i][j] = opt{dp[i+1][j], dp[i][j-1]} + cost` | 最少插入回文 |
| 匹配 + 分割 | 端点匹配后再和枚举分割点取最优 | 括号匹配 |
| 最后一步操作 | `dp[i][j] = opt{dp[i][k] + dp[k][j] + f(i,k,j)}` | 戳气球 |
| 复合最值 | 同时维护 `max/min` | 多边形游戏 |

几个高频技巧也一起记住：
- **前缀和**：快速算区间代价。
- **破环成链**：处理环形区间。
- **逆向思维**：正着难想时，考虑“最后一步”。
- **开区间**：避免子问题互相干扰。

### 常见信号词速查表

| 题面信号词 | 大概率模型 | 你该立刻想到什么 |
|------------|------------|------------------|

| 回文、对称、两端 | 消去型 | 看 `s[i]` 与 `s[j]` |
| 最后一次、最后一步 | 逆向区间 DP | 试着改成“最后处理谁” |
| 气球、覆盖、打印 | 开区间 / 特殊匹配 | 子问题是否独立 |

| 乘法、负数 | 双最值 | 同时维护最大值和最小值 |
| 区间代价、区间和 | 合并型 + 前缀和 | `cost(i, j)` 能否 O(1) 求 |
| `n ≤ 500` 左右 | 区间 DP 可行 | `O(n³)` 通常能过 |

## 七、终极总结与归纳

### 7.1 区间 DP 的本质

区间 DP 的本质，可以压缩成一句话：

> **把一个大区间的问题，拆成若干更小区间的问题，并按照区间长度从小到大完成最优合并。**

很多题表面上看是石子、矩阵、字符串、括号、气球、多边形，实际上都在问同一件事：
- 这个区间的答案能否由更小区间拼出来？
- 拼的时候应该枚举哪一种“最后一步”？
- 这一步的代价或收益是什么？

### 7.2 核心要点

1. **先定语义，再写公式**
   - `dp[i][j]` 表示什么，比转移本身更重要。

2. **长度优先，是区间 DP 的生命线**
   - 顺序错了，子问题就还没算出来。

3. **最后一步视角非常重要**
   - 最后一刀、最后一对、最后一个气球、最后一次打印，往往就是建模入口。

4. **开区间 / 闭区间必须分清**
   - 戳气球这类题，一旦区间定义错，整个转移就会崩掉。

5. **特殊结构单独处理**
   - 环形 → 破环成链
   - 负数乘法 → 同时维护最大最小值
   - 区间代价 → 前缀和预处理

6. **大多数朴素区间 DP 是 `O(n³)`**
   - 这通常对应 `n ≤ 500` 的数据范围。
   - 再往上就要考虑四边形不等式等高级优化。
