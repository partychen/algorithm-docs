## 一、什么是斜率优化动态规划

很多 DP 的瓶颈，并不是状态太多，而是每个状态都要枚举一大堆前驱。若转移式长成二次项、前缀和乘积、线性组合这类样子，朴素做法往往是 `O(n^2)`，甚至多阶段时会膨胀到 `O(kn^2)`。

**斜率优化 DP** 的核心，是把“枚举前驱”的转移式整理成“在若干条直线里，询问某个 `x` 处的最优值”。一旦直线集合的加入顺序和询问顺序带有单调性，就可以用**单调队列维护一个凸壳**，让每次查询和插入都均摊 `O(1)`。

它的本质不是“会背公式”，而是：

- 先把 DP 转移拆成 `y = kx + b` 的形式；
- 再把每个历史决策看成一条直线；
- 最后用几何意义删掉永远不可能成为最优的直线。

```text
朴素：dp[i] = min / max { transfer(j, i) }
直线化：dp[i] = G(i) + min / max { Y(j) - K(i) * X(j) }
几何化：决策 j 变成平面上的点 / 直线
优化后：只维护凸壳上的候选，队头或二分求最优
```

和它常被一起提及的几个优化如下：

| 技巧 | 维护对象 | 关键条件 | 常见实现 |
|------|----------|----------|----------|
| 单调队列优化 DP | 滑动窗口最值 | 决策区间滑动 | `deque` |
| 斜率优化 DP | 直线集合 / 凸壳 | 转移可直线化 | `deque` / 二分 |
| 李超线段树 | 动态直线集合 | 查询 x 无单调性 | 线段树 |
| CDQ 分治 + CHT | 分治区间内的直线转移 | 离线可排序 | CDQ |

一句话概括：**斜率优化 DP 是“把 DP 转移画成几何对象，再用凸壳删掉劣决策”。**

## 二、核心思想与前置知识

### 2.1 如何把 DP 转移直线化

最典型的形式是：

```
dp[i] = G(i) + min / max_{j < i} { A(j) * B(i) + C(j) }
```

如果把：

- `m(j) = A(j)` 当作斜率；
- `b(j) = C(j)` 当作截距；
- `x = B(i)` 当作查询横坐标；

那么每个 `j` 就对应一条直线：

```
y = m(j) * x + b(j)
```

于是，`dp[i]` 就是在横坐标 `x = B(i)` 处，问哪条直线给出的值最优。

### 2.2 为什么可以删掉凸壳内的点

若三条候选直线 `a, b, c` 满足：

- 在插入顺序下，`b` 永远不会比 `a` 和 `c` 更优；
- 那么 `b` 以后就再也不会成为答案；
- 所以可以直接弹出。

这就是“维护凸壳”的几何本质。

常用判劣式不要写成除法，直接交叉乘：

```cpp
bool bad(int a, int b, int c) {
    return (__int128)(Y[c] - Y[a]) * (X[a] - X[b])
         <= (__int128)(Y[b] - Y[a]) * (X[a] - X[c]);
}
```

这里的 `X[i]`、`Y[i]` 代表第 `i` 个决策点在斜率优化中的几何坐标，而不是原题输入。

### 2.3 单调队列维护凸壳骨架

当两件事同时成立时：

1. 插入直线的斜率单调；
2. 询问的 `x` 单调；

就能用双端队列维护凸壳：

```cpp
int q[MAXN], head = 0, tail = -1;

for (int i = 1; i <= n; i++) {
    while (head + 1 <= tail && value(q[head], K(i)) >= value(q[head + 1], K(i)))
        head++;
    dp[i] = calc(q[head], i);

    while (head + 1 <= tail && bad(q[tail - 1], q[tail], i))
        tail--;
    q[++tail] = i;
}
```

这里要注意：

- `value(line, x)` 是该候选在查询点 `x` 处的函数值；
- 有些题是先查询再插入，有些题是先插入当前直线供后面使用；
- `bad()` 比较的是几何坐标，不是原始 `dp` 数组。

### 2.4 什么时候队列不够用了

如果：

- 斜率不单调；
- 查询点不单调；
- 直线是动态加入且顺序混乱；
- 或者候选集合受树上、时间、可达性等复杂限制；

那么仅靠“单调队列维护凸壳”就不够了，常见升级方向有：

| 场景 | 常见升级方案 |
|------|--------------|
| 查询 `x` 不单调 | 凸壳上二分 |
| 插线 / 查询都乱序 | 李超线段树 |
| 离线分治转移 | CDQ + CHT |
| 树上祖先受限 | 点分治 / 树上分治 + CHT |

## 三、子模型与分类详解

### 3.1 查询单调 + 斜率单调型

```
状态定义：`dp[i]` 由一段前缀里的某个决策 j 转移而来
转移关键：把式子化成 `dp[i] = G(i) + min/max (Y[j] - K(i) * X[j])`
适用信号：前缀和单调、直线斜率单调、查询点也单调
实现要点：队头弹劣，队尾弹废，均摊 `O(1)` 查询与插入
```

```cpp
while (head + 1 <= tail && value(q[head], x) >= value(q[head + 1], x)) head++;
dp[i] = calc(q[head], i);
while (head + 1 <= tail && bad(q[tail - 1], q[tail], i)) tail--;
q[++tail] = i;
```

### 3.2 多阶段分层型

```
状态定义：`dp[t][i]` 表示做完第 t 层决策、终点为 i 的最优值
转移关键：每一层都要在上一层的若干直线里查询一次
适用信号：分段 DP、k 段划分、每层同一公式重复出现
实现要点：每层单独建壳；若要恢复方案，记录 `pre[t][i]`
```

```cpp
for (int t = 1; t <= K; t++) {
    head = 0, tail = -1;
    for (int i = t; i <= n; i++) {
        while (head + 1 <= tail && value(q[head], key(i)) <= value(q[head + 1], key(i))) head++;
        dp[t][i] = trans(q[head], i);
        pre[t][i] = q[head];
        while (head + 1 <= tail && bad(q[tail - 1], q[tail], i)) tail--;
        q[++tail] = i;
    }
}
```

### 3.3 查询不单调型

```
状态定义：仍然是“历史决策对应直线，当前状态对应查询点”
转移关键：凸壳仍可维护，但队头不再单调前进
适用信号：`x_i` 乱序，或斜率 / 查询顺序不匹配
实现要点：壳建好后在壳上二分，或改用李超线段树 / CDQ
```

```cpp
int l = head, r = tail;
while (l < r) {
    int mid = (l + r) >> 1;
    if (value(q[mid], x) >= value(q[mid + 1], x)) l = mid + 1;
    else r = mid;
}
dp[i] = calc(q[l], i);
```

### 3.4 动态插线 / 时间排序型

```
状态定义：每个状态点会在某个时间或某个节点上生成一条新直线
转移关键：查询某时刻/某位置可用的直线集合
适用信号：事件按时间推进、每个点维护局部凸壳
实现要点：常按时间排序；一边查询，一边把新线推入对应数据结构
```

```cpp
while (ptr < cnt && event[ptr].time <= cur_time) {
    add_line(event[ptr].belong, event[ptr].line_id);
    ptr++;
}
if (!hull[u].empty()) dp[i] = query(hull[u], x_i);
```

### 3.5 树上 / 分治扩展型

```
状态定义：树上某点 u 的答案由一批受祖先/距离限制的点转移而来
转移关键：候选直线不再是简单前缀，而是某棵子树或某条可达祖先链
适用信号：树形 DP + 一次函数转移 + 祖先限制
实现要点：单调队列往往不再够用，常与点分治、CDQ、可撤销凸壳联动
```

```cpp
solve(centroid) {
    collect_candidates(subtree);
    build_hull(candidates);
    answer_queries(subtree);
    recurse(children);
}
```

### 3.6 模型对比总表

| 模型 | 典型维护方式 | 代表题 | 常见复杂度 |
|------|--------------|--------|------------|
| 单调查询型 | `deque` 维护凸壳 | 玩具装箱、特别行动队 | `O(n)` |
| 多阶段分层型 | 每层一个凸壳 | 序列分割 | `O(kn)` |
| 查询不单调型 | 壳上二分 / 李超树 | 货币兑换 | `O(n log n)` |
| 动态插线型 | 按时间/节点维护局部壳 | 回家路线 | `O(m)` / `O(m log m)` |
| 树上扩展型 | 点分治 / 分治 + CHT | 购票 | `O(n log^2 n)` 左右 |

### 题目 1：Kalila and Dimna in the Logging Industry（CF 319C）

> 给定两个序列 `a[i]`、`b[i]`，要求 `dp[i] = min_{j<i}(dp[j] + b[j] * a[i])`。

**分析**：这是最经典的 CHT 入门式子。每个 `j` 对应一条斜率为 `b[j]`、截距为 `dp[j]` 的直线，在 `x = a[i]` 处取最小值即可。

```
状态定义：`dp[i]` 表示做到 i 的最小代价
转移方程：`dp[i] = min(dp[j] + b[j] * a[i])`
边界条件：`dp[1] = 0`
答案位置：`dp[n]`
```

**C++ 代码**：
```cpp
long long value(int j, long long x) {
    return dp[j] + b[j] * x;
}
for (int i = 2; i <= n; i++) {
    while (head + 1 <= tail && value(q[head], a[i]) >= value(q[head + 1], a[i])) head++;
    dp[i] = value(q[head], a[i]);
    while (head + 1 <= tail && bad(q[tail - 1], q[tail], i)) tail--;
    q[++tail] = i;
}
```

**核心启示**：
- **每个前驱是一条线**，这就是斜率优化的起点。
- **公式越短，越适合先拿来练手感**。
- **先会画图，再会写 `bad()`**，理解会快很多。

### 题目 2：Land Acquisition（洛谷 P2900）

> 有若干矩形地块，购买一块地的代价与前面某些地块的宽高组合有关，求最小总代价。

**分析**：先按一维排序并删去被支配的矩形，问题就会变成典型的前缀 DP。删完冗余地块以后，剩下的高和宽具有单调性，正好能用单调队列维护凸壳。

```
状态定义：`dp[i]` 表示处理前 i 块有效地后的最小代价
转移方程：`dp[i] = min(dp[j] + h[i] * w[j+1])` 一类前缀式
边界条件：先排序并删除被完全支配的地块
答案位置：`dp[m]`，其中 m 是删冗余后的地块数
```

**C++ 代码**：
```cpp
sort(rect + 1, rect + n + 1);
shrink_dominated();
for (int i = 1; i <= m; i++) {
    while (head + 1 <= tail && value(q[head], h[i]) >= value(q[head + 1], h[i])) head++;
    dp[i] = value(q[head], h[i]);
    while (head + 1 <= tail && bad(q[tail - 1], q[tail], i)) tail--;
    q[++tail] = i;
}
```

**核心启示**：
- **很多斜率优化题第一步不是上壳，而是先删冗余状态**。
- **排序 + 去支配** 常常决定后面能不能单调维护。
- **有效状态数缩小后，CHT 才真正有用武之地。**

### 题目 3：玩具装箱（洛谷 P3195）

> 把若干玩具分成若干段装箱，代价是一个平方项，求最小总代价。

**分析**：这题是斜率优化最常见的“前缀和 + 平方代价”模板题。把平方展开后，会得到一个标准的 `Y(j) - K(i) * X(j)` 形式。

```
状态定义：`dp[i]` 表示前 i 个玩具的最小代价，`s[i]` 为改造后的前缀和
转移方程：`dp[i] = min(dp[j] + (s[i] - s[j] - L)^2)`
边界条件：`dp[0] = 0`
答案位置：`dp[n]`
```

**C++ 代码**：
```cpp
long long X(int j) { return s[j]; }
long long Y(int j) { return dp[j] + s[j] * s[j]; }
for (int i = 1; i <= n; i++) {
    while (head + 1 <= tail && slope(q[head], q[head + 1]) <= 2 * (s[i] - L)) head++;
    int j = q[head];
    dp[i] = dp[j] + sqr(s[i] - s[j] - L);
    while (head + 1 <= tail && bad(q[tail - 1], q[tail], i)) tail--;
    q[++tail] = i;
}
```

**核心启示**：
- **平方项展开** 是最常见的直线化入口。
- **`2 * x` 形式的查询斜率** 经常在这类题里出现。
- **前缀和几乎是斜率优化的固定搭档。**

### 题目 4：特别行动队（洛谷 P3628）

> 对前缀和序列做分段，收益/代价中带有二次项，求最优值。

**分析**：和玩具装箱非常像，只是这题通常是取 `max`，因此维护的是**上凸壳**而不是下凸壳。写代码时最容易错的，就是把比较号照抄成最小值版本。

```
状态定义：`dp[i]` 表示前 i 个位置的最大收益
转移方程：`dp[i] = max(dp[j] + a(s[i]-s[j])^2 + b(s[i]-s[j]) + c)`
边界条件：`dp[0] = 0`
答案位置：`dp[n]`
```

**C++ 代码**：
```cpp
for (int i = 1; i <= n; i++) {
    while (head + 1 <= tail && value(q[head], s[i]) <= value(q[head + 1], s[i])) head++;
    dp[i] = calc(q[head], i);
    while (head + 1 <= tail && bad_upper(q[tail - 1], q[tail], i)) tail--;
    q[++tail] = i;
}
```

**核心启示**：
- **min 壳和 max 壳只差几个符号，但整题方向会完全反过来。**
- **写 `value()` 时先明确是求最小还是求最大。**
- **玩具装箱和特别行动队是一对非常适合对着学的题。**

### 题目 5：仓库建设（洛谷 P2120）

> 在若干村庄中建设仓库，运输费用与距离和人口有关，求最小总成本。

**分析**：这题也是典型的前缀和线性化模型。把人口前缀和、加权前缀和记出来后，转移可以整理成“斜率为人口前缀和，查询点为坐标”的形式。

```
状态定义：`dp[i]` 表示前 i 个村庄处理完的最小代价
转移方程：`dp[i] = c[i] + min(dp[j] + x[i](W[i]-W[j]) - (S[i]-S[j]))`
边界条件：按村庄坐标递增处理
答案位置：`dp[n]`
```

**C++ 代码**：
```cpp
for (int i = 1; i <= n; i++) {
    while (head + 1 <= tail && value(q[head], x[i]) >= value(q[head + 1], x[i])) head++;
    dp[i] = calc(q[head], i);
    while (head + 1 <= tail && bad(q[tail - 1], q[tail], i)) tail--;
    q[++tail] = i;
}
```

**核心启示**：
- **人口前缀和 / 距离前缀和** 这类统计量很容易进 CHT。
- **别被题目背景吓到**，真正落到 DP 上常常还是一条线性式。
- **同类“建站 / 仓库 / 选址”题经常互相同构。**

### 题目 6：锯木厂选址（洛谷 P4360）

> 在若干位置中选址，使总运输或加工代价最小，转移里含前缀和与坐标乘积。

**分析**：这类题和仓库建设是同一大类，但常常多一个站点固定费用或更复杂的前缀项。真正难点不在推壳，而在把原式整理到标准形态。

```
状态定义：`dp[i]` 表示处理到第 i 个工厂的最小代价
转移方程：可整理成 `dp[i] = G(i) + min(Y[j] - X[j] * K(i))`
边界条件：前缀和、坐标都按输入顺序递增
答案位置：`dp[n]`
```

**C++ 代码**：
```cpp
while (head + 1 <= tail && value(q[head], key(i)) >= value(q[head + 1], key(i))) head++;
dp[i] = extra(i) + value(q[head], key(i));
while (head + 1 <= tail && bad(q[tail - 1], q[tail], i)) tail--;
q[++tail] = i;
```

**核心启示**：
- **一旦看出是“前缀和乘坐标”的味道，就要警觉是不是 CHT。**
- **推式子比维护壳更重要**，壳本身通常是模板。
- **背景千变万化，式子高度统一。**

### 题目 7：序列分割（洛谷 P3648）

> 把序列分成 `k+1` 段，每次分割获得一部分乘积收益，求最大总分并恢复分割点。

**分析**：这题是多阶段斜率优化 DP 的经典代表。每一层都是一次完整的 CHT，外加前驱记录恢复答案。

```
状态定义：`dp[t][i]` 表示前 i 个数分成 t 段的最大得分
转移方程：`dp[t][i] = max(dp[t-1][j] + s[j](s[i]-s[j]))`
边界条件：`dp[1][i]` 可直接从前缀和得到
答案位置：`dp[k+1][n]`，并通过 `pre[t][i]` 倒推断点
```

**C++ 代码**：
```cpp
for (int t = 1; t <= K; t++) {
    head = 0, tail = -1;
    for (int i = t; i <= n; i++) {
        while (head + 1 <= tail && value(q[head], s[i]) <= value(q[head + 1], s[i])) head++;
        dp[t][i] = calc(q[head], i);
        pre[t][i] = q[head];
        while (head + 1 <= tail && bad_upper(q[tail - 1], q[tail], i)) tail--;
        q[++tail] = i;
    }
}
```

**核心启示**：
- **多阶段分割 DP** 和斜率优化是天作之合。
- **队列要一层一层重建**，不能把上一层的壳直接拿来复用。
- **前驱数组** 是恢复方案的关键。

### 题目 8：货币兑换（洛谷 P4027）

> 每天可以在两种券之间按比例兑换，目标最大化最终现金。转移本质是一堆直线上的最优查询。

**分析**：这题非常经典，但也正好提醒你：**不是所有斜率优化题都能直接用队头指针线性扫。** 因为查询斜率和插线顺序不再天然单调，常见做法是凸壳二分、CDQ 分治或李超树。

```
状态定义：`dp[i]` 表示到第 i 天结束时的最大现金
转移方程：`dp[i] = max(A_i * x_j + B_i * y_j)`，历史状态 j 可看作直线/点
边界条件：`dp[1] = S`
答案位置：`dp[n]`
```

**C++ 代码**：
```cpp
sort_by_key(lines);
for (int i = 1; i <= n; i++) {
    int pos = binary_search_hull(query_key(i));
    dp[i] = max(dp[i], value(hull[pos], query_key(i)));
    insert_line(i);
}
```

**核心启示**：
- **“斜率优化”不等于“只能用 deque”**。
- **单调性一丢，查询方式就要升级。**
- **货币兑换** 是学习“CHT 进一步升级”的分水岭。

### 题目 9：回家路线（洛谷 P5468）

> 有若干条路线，每次等待会产生二次代价。按时间衔接路线，求最小总花费。

**分析**：把等待代价展开后，仍然是标准的二次型 DP。但这题的候选直线不是全局统一维护，而是按**站点 / 时间**分桶维护，每个位置都有自己的局部凸壳。

```
状态定义：`dp[i]` 表示最后乘第 i 条路线时的最小总花费
转移方程：`dp[i] = min(dp[j] + A(p_i-q_j)^2 + B(p_i-q_j) + C)`
边界条件：只允许从终点和时间都合法的历史路线转移
答案位置：所有到达终点路线的最小值
```

**C++ 代码**：
```cpp
while (ptr < tot && add_evt[ptr].time <= cur_time) {
    add_line(add_evt[ptr].to, add_evt[ptr].id);
    ptr++;
}
if (!hull[from].empty()) {
    int j = query_best(hull[from], 2LL * A * p[i]);
    dp[i] = calc(j, i);
}
```

**核心启示**：
- **局部维护多个凸壳** 是高级 CHT 题的常见套路。
- **时间顺序本身就是一种单调性资源**。
- **把复杂图题化成“事件 + 局部壳”**，会比硬做图 DP 清楚很多。

### 题目 10：购票（洛谷 P2305）

> 树上每个点向某个祖先转移，代价是一次函数形式，但祖先还受距离限制，求每个点的最优值。

**分析**：这题说明了斜率优化的最高阶形态：候选不再是“前缀里的所有点”，而是“满足限制的一批祖先”。因此通常要把 CHT 和点分治、CDQ 或树上分治结合起来。

```
状态定义：`dp[u]` 表示到达节点 u 的最小代价
转移方程：`dp[u] = q_u + min(dp[v] + (dep[u]-dep[v]) * p_u)`，其中 v 是合法祖先
边界条件：只允许满足路径限制的祖先参与转移
答案位置：所有节点各自的 `dp[u]`
```

**C++ 代码**：
```cpp
void solve(int rt) {
    collect(rt);
    build_hull(cand);
    for (int u : qry[rt])
        dp[u] = min(dp[u], query_hull(cand, p[u]) + q[u]);
    for (int v : child[rt]) solve(v);
}
```

**核心启示**：
- **树上斜率优化** 往往已经超出了“单调队列模板题”的范畴。
- **限制条件决定数据结构**，而不是反过来。
- **购票** 是把“DP、CHT、树上分治”揉在一起的代表题。

## 五、常见技巧与思路总结

### 5.1 一定先把式子整理到标准形态

斜率优化最怕“见平方就想上壳”。正确顺序永远是：

1. 写出朴素转移；
2. 把和 `i` 无关的部分、和 `j` 无关的部分拆开；
3. 确认历史决策是否能变成直线；
4. 再决定是队列、二分、李超还是 CDQ。

### 5.2 `bad()` 一律用交叉乘，别用浮点除法

```cpp
return (__int128)(Y[c] - Y[a]) * (X[a] - X[b])
     <= (__int128)(Y[b] - Y[a]) * (X[a] - X[c]);
```

这样更稳，也不会被精度坑死。

### 5.3 min 壳和 max 壳千万别写混

- 求最小值：维护**下凸壳**；
- 求最大值：维护**上凸壳**；
- 查询时队头弹出的比较号也会反过来。

### 5.4 单调查询时，队头是可以只前进不后退的

这正是用 `deque` 能做到 `O(n)` 的根本原因：

```cpp
while (head + 1 <= tail && value(q[head], x) >= value(q[head + 1], x)) head++;
```

一旦查询点不单调，这一步就不能再硬写成前进指针了。

### 5.5 多阶段题要重建凸壳，别跨层复用

像 `序列分割` 这类 `dp[t][i]` 的题，每层的直线集合来自上一层 `dp[t-1][j]`，所以每一层都要重新建壳。

### 5.6 复杂题里，CHT 常常只是大框架中的一环

- `货币兑换`：CHT + CDQ / 二分；
- `回家路线`：按时间维护局部壳；
- `购票`：树上分治 + CHT。

别把“斜率优化”理解成一道独立模板，它常常是组合拳。

## 六、优化方法与解题框架

### 6.1 看到斜率优化 DP 时的思考流程

```text
读题
 ├─ 朴素转移是否在枚举历史决策 j？
 ├─ 式子能否整理成 A(j) * B(i) + C(j) + D(i)？
 │   └─ 否：大概率不是标准 CHT
 ├─ 历史决策能否看成直线？
 ├─ 插线顺序是否单调？
 ├─ 查询 x 是否单调？
 │   ├─ 都单调      → deque 维护凸壳
 │   ├─ 查询不单调  → 壳上二分 / 李超树
 │   ├─ 区间离线    → CDQ + CHT
 │   └─ 树上限制    → 分治 / 点分治 + CHT
 └─ 最后确认：是求 min 还是 max？
```

### 6.2 信号词 → 题型判断表

| 题面信号词 | 往往对应的模型 | 第一反应 |
|------------|----------------|----------|
| “平方代价 / 二次代价” | 前缀和二次式 | 先展开平方 |
| “前缀和乘当前值” | 标准单调查询型 | 尝试 `Y(j)-K(i)X(j)` |
| “分成 k 段” | 多阶段分层型 | 每层重建壳 |
| “查询顺序乱” | 非单调查询型 | 准备二分 / 李超 |
| “路线按时间衔接” | 动态插线型 | 时间排序 + 局部壳 |
| “祖先 / 树上限制” | 树上扩展型 | 分治 + CHT |

### 6.3 常见复杂度估算

| 模型 | 朴素复杂度 | 优化后复杂度 |
|------|------------|--------------|
| 单调查询型 | `O(n^2)` | `O(n)` |
| 多阶段分层型 | `O(kn^2)` | `O(kn)` |
| 查询不单调型 | `O(n^2)` | `O(n log n)` |
| 动态插线型 | 依题而定 | 常为 `O(m log m)` 或摊还线性 |
| 树上扩展型 | `O(n^2)` 甚至更高 | 常降到 `O(n log^2 n)` 左右 |

## 七、终极总结与归纳

### 7.1 斜率优化动态规划的本质

**一句话概括：把“枚举前驱”改写成“在凸壳上查询最优直线”。**

再展开一点：

- DP 在做的是状态转移；
- 几何上做的是直线比较；
- 数据结构做的是删除永远不可能再最优的候选。

### 7.2 核心要点

1. 斜率优化首先是**推式子**，其次才是维护凸壳。
2. `bad()` 一律交叉乘，尽量别碰浮点。
3. min / max 两套比较号一定要分清。
4. 若插线和查询都单调，`deque` 往往就是最优实现。
5. 若单调性断掉，就要升级到二分、李超树或 CDQ。
6. 高级题往往是“大框架 + CHT”的组合，而不是纯模板。
