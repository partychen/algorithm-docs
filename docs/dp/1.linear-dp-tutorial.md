## 一、什么是线性 DP

**线性 DP**，指的是状态围绕“线性结构”展开的动态规划。
这里的线性结构，通常是 **数组、字符串、序列、前缀、后缀、双序列对齐过程** 等。它们的共同特点是：状态可以沿着下标顺序逐步推进。

你可以把它理解成一句话：

> **在线性序列上，利用前面已经算出的最优信息，逐步推出后面的答案。**

最典型的三类问法是：

1. **前缀型**：只看前 `i` 个元素时，最优值 / 可行性 / 方案数是什么。
2. **结尾型**：必须以 `i` 结尾时，最优值是什么。
3. **双序列型**：两个序列分别走到 `i`、`j` 时，最优匹配结果是什么。

### 与其他常见 DP 的对比

| DP 类型 | 常见状态 | 关注对象 | 典型推进方式 | 代表问题 |
|---------|----------|----------|--------------|----------|
| 线性 DP | `dp[i]`、`dp[i][j]` | 前缀 / 结尾 / 双序列 | 按下标顺序推进 | LIS、LCS、编辑距离 |
| 区间 DP | `dp[l][r]` | 一个区间如何合并 | 按区间长度递增 | 石子合并、回文划分 |
| 树形 DP | `dp[u][state]` | 子树信息如何汇总 | DFS 后序 | 没有上司的舞会 |
| 背包 DP | `dp[j]`、`dp[i][j]` | 资源限制下如何选择 | 枚举物品 + 资源 | 0/1 背包、完全背包 |
| 网格 DP | `dp[i][j]` | 走到某个格子的最优值 | 按行列推进 | 不同路径、最小路径和 |

### 线性 DP 的本质

线性 DP 看起来题型很多，但本质非常统一：

1. **阶段**：当前推进到哪个位置 / 哪两个位置。
2. **状态**：当前这个位置下，已经掌握了什么信息。
3. **转移**：最后一步从哪里来，为什么能从旧状态推出新状态。

很多题最后都会落回下面三种视角之一：

- **前缀视角**：`dp[i]` 表示前 `i` 个元素的答案。
- **结尾视角**：`dp[i]` 表示以 `i` 结尾的答案。
- **子序列/子串视角**：`dp[i][j]` 表示两个下标边界之间的答案。

只要你能先想清楚“这个状态究竟描述的是谁”，线性 DP 就已经做对了一大半。

## 二、核心思想与前置知识

### 2.1 线性 DP 的三种经典模式

做线性 DP 时，先判断你属于哪一类模型，通常就能立刻确定状态形式。

#### 模式一：单序列 DP

最常见，状态只围绕一个数组或字符串展开。
例如：爬楼梯、最大子数组和、LIS、打家劫舍。

```
dp[i] 依赖 dp[0..i-1] 或 dp[i-1], dp[i-2]
```

这类题的关键一般是：
- 是“前 `i` 个”还是“以 `i` 结尾”
- 最后一步是“选当前”还是“不选当前”

#### 模式二：双序列 DP

当题目同时给出两个字符串 / 数组时，往往要开二维状态。
例如：LCS、编辑距离、交错字符串、通配符匹配。

```
dp[i][j] 表示第一个序列走到 i，第二个序列走到 j 时的答案
```

这类题最常见的三种转移来源是：
- 从上面来：`dp[i-1][j]`
- 从左边来：`dp[i][j-1]`
- 从左上角来：`dp[i-1][j-1]`

#### 模式三：区间 / 子串型

如果状态写成 `dp[i][j]`，表示子串 `s[i..j]`、区间 `[i..j]` 的答案，它常常是线性序列上的“双端向内”问题。
例如：最长回文子序列。

```
dp[i][j] = 区间 [i..j] 内的答案
```

不过要注意：**当转移依赖“把区间拆成两段”时，它更接近区间 DP。**
如果你想系统学习这一类，请配合 `interval-dp-tutorial.md` 一起看。

### 2.2 状态设计的核心思路

线性 DP 最难的不是写公式，而是 **把题目翻译成正确状态**。

#### 视角一：“以 i 结尾”

当题目问的是：
- 以某个位置收尾的最优值
- 最后一步必须落在当前位置
- 当前字符必须被选入答案

就要优先考虑这种写法。

典型例子：
- LIS：`dp[i] = 以 i 结尾的最长递增子序列长度`
- 最大子数组和：`dp[i] = 以 i 结尾的最大子数组和`
- 最长有效括号：`dp[i] = 以 i 结尾的最长有效括号长度`

#### 视角二：“前 i 个”

当题目问的是：
- 只看前 `i` 个元素时的最优决策
- 前缀是否可行
- 前缀能否拆分 / 匹配 / 达成

就更适合前缀型写法。

典型例子：
- 爬楼梯：`dp[i] = 到第 i 阶的方法数`
- 打家劫舍：`dp[i] = 前 i 个房子的最大收益`
- 单词拆分：`dp[i] = 前 i 个字符能否被合法切分`

#### 什么时候需要额外维度

如果一个下标不够完整描述状态，就必须加维度。

| 额外维度 | 含义 | 代表题 |
|----------|------|--------|
| `j` 表示第二个序列位置 | 两个序列同时推进 | LCS、编辑距离 |
| `j` 表示区间右端点 | 子串 / 子区间 | 最长回文子序列 |
| `state` 表示状态机 | 当前是否持有、是否冷冻 | 股票系列 |
| `k` 表示次数 / 已用资源 | 限制交易次数、切分次数 | 股票 IV、分段 DP |

可以用下面这张思考卡片来做题：

```
1. 状态描述的是“前缀整体”还是“必须以当前位置结尾”？
2. 最后一步做了什么选择？
3. 这个选择需要看几个历史位置？
4. 一个下标够不够，不够就加维度。
5. 答案是在 dp[n]、max(dp[...])，还是某个区间状态里？
```

### 2.3 通用框架代码

下面用“最大子数组和”演示一个最标准的 **单序列线性 DP 框架**。实际做题时，你只需要替换“状态含义”和“转移逻辑”。

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 200005;
int n, a[MAXN], dp[MAXN];

int main() {
    cin >> n;
    for (int i = 1; i <= n; i++) cin >> a[i];

    dp[1] = a[1];
    int ans = dp[1];
    for (int i = 2; i <= n; i++) {
        dp[i] = max(a[i], dp[i - 1] + a[i]);
        ans = max(ans, dp[i]);
    }

    cout << ans << '\n';
    return 0;
}
```

你可以把所有线性 DP 都拆成四步：

1. **确定状态定义**：`dp[i]` / `dp[i][j]` 到底表示什么。
2. **写初始化**：空串、空前缀、单个元素时是什么。
3. **枚举顺序**：从小到大？从大到小？区间要不要倒着枚举左端点？
4. **写转移**：最后一步来自哪里。

## 三、六大线性 DP 模型

线性 DP 最常见、最值得反复练习的六个模型分别是：
**LIS、LCS、编辑距离、最大子数组和、打家劫舍、股票状态机**。
它们几乎覆盖了刷题里最常见的线性 DP 思维骨架。

### 3.1 最长递增子序列（LIS）

LIS 是线性 DP 的招牌模型之一。它既能训练你掌握“以 `i` 结尾”的经典设计，也能让你接触从 `O(n²)` 到 `O(n log n)` 的优化思想。

#### 模板一：`O(n²)` DP

```
状态定义：
  dp[i] = 以 a[i] 结尾的最长递增子序列长度

转移方程：
  dp[i] = 1
  若存在 j < i 且 a[j] < a[i]
  则 dp[i] = max(dp[i], dp[j] + 1)

答案：
  max(dp[0..n-1])
```

**关键洞察**：LIS 的核心不是“前 i 个”，而是“必须以 i 结尾”。因为只有这样，最后一步接谁才是明确可枚举的。

```cpp
int dp[2505];

int lengthOfLIS(vector<int>& a) {
    int n = a.size(), ans = 0;
    for (int i = 0; i < n; i++) {
        dp[i] = 1;
        for (int j = 0; j < i; j++)
            if (a[j] < a[i]) dp[i] = max(dp[i], dp[j] + 1);
        ans = max(ans, dp[i]);
    }
    return ans;
}
// 调用方式/答案：返回 lengthOfLIS(a)
```

#### 模板二：`O(n log n)` 贪心 + 二分

```
状态含义：
  tails[len] = 长度为 len+1 的递增子序列，结尾最小值是多少

转移方式：
  对每个 a[i]，用 lower_bound 找到第一个 >= a[i] 的位置替换
  若找不到，就把它接到 tails 末尾

答案：
  tails.size()
```

**关键洞察**：`tails` 不是一个真实子序列，而是“每种长度下最优的最小结尾”。结尾越小，后面越容易接新数。

```cpp
int lengthOfLIS2(vector<int>& a) {
    vector<int> tails;
    for (int x : a) {
        auto it = lower_bound(tails.begin(), tails.end(), x);
        if (it == tails.end()) tails.push_back(x);
        else *it = x;
    }
    return (int)tails.size();
}
// 调用方式/答案：返回 lengthOfLIS2(a)
```

### 3.2 最长公共子序列（LCS）

LCS 是最经典的双序列 DP。两个字符串各走一步，问“匹配上了没有”，这正是二维前缀 DP 最标准的场景。

```
状态定义：
  dp[i][j] = s 的前 i 个字符与 t 的前 j 个字符的 LCS 长度

转移方程：
  若 s[i-1] == t[j-1]：dp[i][j] = dp[i-1][j-1] + 1
  否则：dp[i][j] = max(dp[i-1][j], dp[i][j-1])

答案：
  dp[n][m]
```

**关键洞察**：LCS 里“字符相等”时一定来自左上角；“字符不等”时，本质是在讨论“删掉谁更优”。

```cpp
int dp[1005][1005];

int longestCommonSubsequence(string s, string t) {
    int n = s.size(), m = t.size();
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= m; j++)
            if (s[i - 1] == t[j - 1]) dp[i][j] = dp[i - 1][j - 1] + 1;
            else dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);
    return dp[n][m];
}
// 调用方式/答案：返回 longestCommonSubsequence(s, t)
```

### 3.3 编辑距离

编辑距离描述的是“把一个字符串变成另一个字符串最少要多少步”。它本质上也是双序列前缀 DP，只不过转移不再是 `max`，而是三种操作取 `min`。

```
状态定义：
  dp[i][j] = s[0..i-1] 变成 t[0..j-1] 的最少操作数

转移方程：
  若 s[i-1] == t[j-1]：dp[i][j] = dp[i-1][j-1]
  否则：
    插入：dp[i][j-1] + 1
    删除：dp[i-1][j] + 1
    替换：dp[i-1][j-1] + 1
  取三者最小

答案：
  dp[n][m]
```

**关键洞察**：编辑距离的三种操作，一定要从“当前前缀已经处理到哪里”来理解；不是背公式，而是想象最后一步做了哪种编辑。

```cpp
int dp[505][505];

int minDistance(string s, string t) {
    int n = s.size(), m = t.size();
    for (int i = 0; i <= n; i++) dp[i][0] = i;
    for (int j = 0; j <= m; j++) dp[0][j] = j;
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= m; j++)
            if (s[i - 1] == t[j - 1]) dp[i][j] = dp[i - 1][j - 1];
            else dp[i][j] = min({dp[i - 1][j], dp[i][j - 1], dp[i - 1][j - 1]}) + 1;
    return dp[n][m];
}
// 调用方式/答案：返回 minDistance(s, t)
```

### 3.4 最大子数组和（Kadane）

这类题只盯住一个问题：**以当前位置结尾的最优答案是多少。** 一旦状态这样定义，转移就非常自然。

```
状态定义：
  dp[i] = 以 nums[i] 结尾的最大子数组和

转移方程：
  dp[i] = max(nums[i], dp[i-1] + nums[i])

答案：
  max(dp[0..n-1])
```

**关键洞察**：前面的和如果是负贡献，就果断断开重开；这就是 Kadane 算法的本质。

```cpp
int maxSubArray(vector<int>& nums) {
    int cur = nums[0], ans = nums[0];
    for (int i = 1; i < nums.size(); i++) {
        cur = max(nums[i], cur + nums[i]);
        ans = max(ans, cur);
    }
    return ans;
}
// 调用方式/答案：返回 maxSubArray(nums)
```

### 3.5 打家劫舍系列

打家劫舍是“选 / 不选当前位置”模型的代表题。它看起来像贪心，但本质是前缀 DP：前 `i` 个房子的最优解，要么不偷当前，要么偷当前并接上 `i-2` 的答案。

```
状态定义：
  dp[i] = 前 i 个房子的最大收益

转移方程：
  不偷第 i 个：dp[i-1]
  偷第 i 个：dp[i-2] + nums[i]

故：
  dp[i] = max(dp[i-1], dp[i-2] + nums[i])
```

**关键洞察**：这类题不是“局部看哪个值大”，而是“当前选择会不会影响相邻位置能否再选”。

```cpp
int rob(vector<int>& nums) {
    int prev2 = 0, prev1 = 0;
    for (int x : nums) {
        int cur = max(prev1, prev2 + x);
        prev2 = prev1;
        prev1 = cur;
    }
    return prev1;
}
// 调用方式/答案：返回 rob(nums)
```

### 3.6 股票买卖系列

股票题是线性 DP 里最重要的“状态机模型”。状态不再只由下标决定，而是要额外记录：**今天结束后你手里有没有股票、已经进行了几次交易、是否处于冷冻期。**

```
基础状态：
  hold = 今天结束后持股时的最大利润
  cash = 今天结束后空仓时的最大利润

一次交易 / 无限交易的本质：
  hold = max(hold, cash - price)
  cash = max(cash, hold + price)

若限制最多 K 次交易：
  buy[t]  = 完成 t-1 次卖出后，再买入一股的最大利润
  sell[t] = 完成 t 次卖出后的最大利润
```

**关键洞察**：股票题的“状态”描述的不是数组本身，而是“你当前所处的交易身份”。这就是状态机 DP 的典型味道。

```cpp
int maxProfitK(vector<int>& prices, int K) {
    vector<int> buy(K + 1, -1000000000), sell(K + 1, 0);
    for (int p : prices) {
        auto nbuy = buy, nsell = sell;
        for (int t = 1; t <= K; t++) {
            nbuy[t] = max(buy[t], sell[t - 1] - p);
            nsell[t] = max(sell[t], buy[t] + p);
        }
        buy.swap(nbuy), sell.swap(nsell);
    }
    return sell[K];
}
// 调用方式/答案：返回 maxProfitK(prices, K)
```

**六种模型对比表**

| 模型 | 常见状态 | 核心转移 | 代表题 | 典型优化 |
|------|----------|----------|--------|----------|
| LIS | `dp[i]` / `tails` | 从更小元素接到当前位置 | 最长递增子序列 | 二分优化到 `O(n log n)` |
| LCS | `dp[i][j]` | 上 / 左 / 左上 | 最长公共子序列 | 滚动数组 |
| 编辑距离 | `dp[i][j]` | 插入 / 删除 / 替换取最小 | 编辑距离 | 滚动数组 |
| 最大子数组和 | `dp[i]` | 断开或继续 | LeetCode 53 | 空间压缩到常数 |
| 打家劫舍 | `dp[i]` | 偷 / 不偷当前位置 | LeetCode 198/213 | 两变量滚动 |
| 股票状态机 | `dp[i][state]` | 买入 / 卖出 / 休息 | LeetCode 121/309/714 | 状态压缩 |

### 🟢 入门级 Level 1：单序列基础

### 题目 1：爬楼梯（LeetCode 70）

> 题意：每次可以爬 `1` 阶或 `2` 阶，问到第 `n` 阶一共有多少种不同方法。

**分析/状态设计**：这是最经典的前缀型线性 DP。到第 `i` 阶，只可能从 `i-1` 或 `i-2` 转移而来，本质上就是 Fibonacci 变体。

```
dp[i] = 到达第 i 阶的方法数

转移：
  dp[i] = dp[i-1] + dp[i-2]

初始化：
  dp[0] = 1, dp[1] = 1
```

**C++ 代码**：
```cpp
int climbStairs(int n) {
    int a = 1, b = 1;
    for (int i = 2; i <= n; i++) {
        int c = a + b;
        a = b, b = c;
    }
    return b;
}
// 调用方式/答案：返回 climbStairs(n)
```

**核心启示**：
- “方法数”常常对应 `+=` 型转移。
- 只依赖前两项时，空间可以压到常数。

### 题目 2：最大子数组和（LeetCode 53）

> 题意：给定一个整数数组，求一个连续子数组的最大和。

**分析/状态设计**：关键不在“前 `i` 个”，而在“必须以 `i` 结尾”。因为连续子数组一旦断开，就必须重新开始。

```
dp[i] = 以 nums[i] 结尾的最大子数组和

转移：
  dp[i] = max(nums[i], dp[i-1] + nums[i])

答案：
  max(dp[i])
```

**C++ 代码**：
```cpp
int maxSubArray(vector<int>& nums) {
    int cur = nums[0], ans = nums[0];
    for (int i = 1; i < nums.size(); i++) {
        cur = max(nums[i], cur + nums[i]);
        ans = max(ans, cur);
    }
    return ans;
}
// 调用方式/答案：返回 maxSubArray(nums)
```

**核心启示**：
- 连续段问题，常常优先想“以 i 结尾”。
- 前缀和为负时，保留它只会拖后腿。

### 题目 3：打家劫舍（LeetCode 198）

> 题意：一排房子中，每个房子有一定金额，相邻房子不能同时偷，求最大收益。

**分析/状态设计**：前 `i` 个房子的最优解，只可能来自两种决策：不偷当前房子，或偷当前房子并接上前 `i-2` 个房子的最优解。

```
dp[i] = 前 i 个房子的最大收益

转移：
  dp[i] = max(dp[i-1], dp[i-2] + nums[i-1])

初始化：
  dp[0] = 0
```

**C++ 代码**：
```cpp
int rob(vector<int>& nums) {
    int prev2 = 0, prev1 = 0;
    for (int x : nums) {
        int cur = max(prev1, prev2 + x);
        prev2 = prev1;
        prev1 = cur;
    }
    return prev1;
}
// 调用方式/答案：返回 rob(nums)
```

**核心启示**：
- 这类题的本质是“选当前，就必须舍弃相邻”。
- 当转移只依赖 `i-1`、`i-2` 时，滚动变量最自然。

### 题目 4：最小花费爬楼梯（LeetCode 746）

> 题意：每个台阶有一个花费，可以从第 `0` 或第 `1` 阶开始，每次爬 `1` 阶或 `2` 阶，问到楼顶的最小代价。

**分析/状态设计**：楼顶可以看成第 `n` 阶，最后一步只可能从 `n-1` 或 `n-2` 跨上来，因此仍然是标准前缀 DP，只是目标从求最大 / 求方案改成了求最小。

```
dp[i] = 到达第 i 阶（含楼顶）的最小代价

转移：
  dp[i] = min(dp[i-1] + cost[i-1], dp[i-2] + cost[i-2])

初始化：
  dp[0] = dp[1] = 0
```

**C++ 代码**：
```cpp
int dp[1005];

int minCostClimbingStairs(vector<int>& cost) {
    int n = cost.size();
    dp[0] = dp[1] = 0;
    for (int i = 2; i <= n; i++)
        dp[i] = min(dp[i - 1] + cost[i - 1], dp[i - 2] + cost[i - 2]);
    return dp[n];
}
// 调用方式/答案：返回 minCostClimbingStairs(cost)
```

**核心启示**：
- 线性 DP 里，`max / min / += / bool` 往往只是“目标不同”，骨架不变。
- 初始化要紧扣题意：起点不花钱，所以 `dp[0] = dp[1] = 0`。

### 题目 5：最长递增子序列（LeetCode 300）

> 题意：给定一个整数数组，求最长严格递增子序列的长度。

**分析/状态设计**：这是“以 `i` 结尾”模型的代表题。朴素做法是枚举所有前驱 `j`，进阶做法是维护每种长度下最小可能结尾 `tails`。

```
O(n²) DP：
  dp[i] = 以 nums[i] 结尾的 LIS 长度
  dp[i] = max(dp[j] + 1), j < i 且 nums[j] < nums[i]

O(n log n)：
  tails[len] = 长度 len+1 的递增子序列的最小结尾
  用 lower_bound 替换第一个 >= nums[i] 的位置
```

**C++ 代码**：
```cpp
int dp[2505];

int lengthOfLIS(vector<int>& nums) {
    int n = nums.size(), ans = 0;
    for (int i = 0; i < n; i++) {
        dp[i] = 1;
        for (int j = 0; j < i; j++)
            if (nums[j] < nums[i]) dp[i] = max(dp[i], dp[j] + 1);
        ans = max(ans, dp[i]);
    }
    return ans;
}

int lengthOfLIS2(vector<int>& nums) {
    vector<int> tails;
    for (int x : nums) {
        auto it = lower_bound(tails.begin(), tails.end(), x);
        if (it == tails.end()) tails.push_back(x);
        else *it = x;
    }
    return (int)tails.size();
}
// 调用方式/答案：返回 lengthOfLIS(nums) 或 lengthOfLIS2(nums)
```

**核心启示**：
- LIS 是理解“以 i 结尾”最好的入门题。
- `tails` 不是答案本身，而是为了让“后面更容易接”而维护的最优中间态。

### 题目 6：最长公共子序列（LeetCode 1143）

> 题意：给定两个字符串，求它们的最长公共子序列长度。

**分析/状态设计**：两个字符串分别推进到 `i`、`j`，如果当前字符相等，就能从左上角多匹配一个；否则只能删掉其中一个字符继续比较。

```
dp[i][j] = s 前 i 个字符与 t 前 j 个字符的 LCS 长度

转移：
  若 s[i-1] == t[j-1]：dp[i][j] = dp[i-1][j-1] + 1
  否则：dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```

**C++ 代码**：
```cpp
int dp[1005][1005];

int longestCommonSubsequence(string s, string t) {
    int n = s.size(), m = t.size();
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= m; j++)
            if (s[i - 1] == t[j - 1]) dp[i][j] = dp[i - 1][j - 1] + 1;
            else dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);
    return dp[n][m];
}
// 调用方式/答案：返回 longestCommonSubsequence(s, t)
```

**核心启示**：
- 子序列允许跳过元素，所以“不相等”时要考虑删左还是删上。
- LCS 和最长重复子数组最像，但一个允许跳，一个必须连续。

### 题目 7：最长回文子序列（LeetCode 516）

> 题意：给定一个字符串，求其最长回文子序列长度。

**分析/状态设计**：这题的状态是子串 `s[i..j]`。如果两端相等，就把中间答案包起来；否则只能舍弃左端或右端其中之一。

```
dp[i][j] = 子串 s[i..j] 中最长回文子序列长度

转移：
  若 s[i] == s[j]：dp[i][j] = dp[i+1][j-1] + 2
  否则：dp[i][j] = max(dp[i+1][j], dp[i][j-1])

初始化：
  dp[i][i] = 1
```

**C++ 代码**：
```cpp
int dp[1005][1005];

int longestPalindromeSubseq(string s) {
    int n = s.size();
    if (!n) return 0;
    for (int i = n - 1; i >= 0; i--) {
        dp[i][i] = 1;
        for (int j = i + 1; j < n; j++)
            if (s[i] == s[j]) dp[i][j] = dp[i + 1][j - 1] + 2;
            else dp[i][j] = max(dp[i + 1][j], dp[i][j - 1]);
    }
    return dp[0][n - 1];
}
// 调用方式/答案：返回 longestPalindromeSubseq(s)
```

**核心启示**：
- 这是“线性序列上的双端 DP”代表题。
- 枚举顺序必须让 `i+1` 与 `j-1` 已经算过，所以左端点要倒着枚举。

### 题目 8：最长重复子数组（LeetCode 718）

> 题意：给定两个整数数组，求它们最长公共连续子数组的长度。

**分析/状态设计**：和 LCS 最大区别在于这里要求 **连续**。因此状态不是“前缀最优值”，而是“必须同时以 `i-1`、`j-1` 结尾的公共后缀长度”。

```
dp[i][j] = 以 a[i-1] 与 b[j-1] 结尾的最长公共后缀长度

转移：
  若 a[i-1] == b[j-1]：dp[i][j] = dp[i-1][j-1] + 1
  否则：dp[i][j] = 0

答案：
  max(dp[i][j])
```

**C++ 代码**：
```cpp
int dp[1005][1005];

int findLength(vector<int>& a, vector<int>& b) {
    int n = a.size(), m = b.size(), ans = 0;
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= m; j++)
            if (a[i - 1] == b[j - 1])
                ans = max(ans, dp[i][j] = dp[i - 1][j - 1] + 1);
    return ans;
}
// 调用方式/答案：返回 findLength(a, b)
```

**核心启示**：
- 子序列和子数组只差一个字，但状态设计完全不同。
- 一旦要求“连续”，不匹配时通常就必须归零。

### 题目 9：编辑距离（LeetCode 72）

> 题意：给定两个单词，求把一个单词转换成另一个单词所需的最少操作数。

**分析/状态设计**：三种操作分别对应三种转移来源：插入来自左边，删除来自上边，替换来自左上角；若当前字符相等，则什么都不做，直接继承左上角。

```
dp[i][j] = s 的前 i 个字符变成 t 的前 j 个字符的最少操作数

转移：
  若 s[i-1] == t[j-1]：dp[i][j] = dp[i-1][j-1]
  否则：
    dp[i][j] = min(dp[i-1][j] + 1, dp[i][j-1] + 1, dp[i-1][j-1] + 1)
```

**C++ 代码**：
```cpp
int dp[505][505];

int minDistance(string s, string t) {
    int n = s.size(), m = t.size();
    for (int i = 0; i <= n; i++) dp[i][0] = i;
    for (int j = 0; j <= m; j++) dp[0][j] = j;
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= m; j++)
            if (s[i - 1] == t[j - 1]) dp[i][j] = dp[i - 1][j - 1];
            else dp[i][j] = min({dp[i - 1][j], dp[i][j - 1], dp[i - 1][j - 1]}) + 1;
    return dp[n][m];
}
// 调用方式/答案：返回 minDistance(s, t)
```

**核心启示**：
- 双序列 `min` 型 DP 的关键，是把“最后一步操作”说清楚。
- 初始化同样是转移的一部分：空串变成长度为 `j` 的串，至少要插入 `j` 次。

### 题目 10：不同的子序列（LeetCode 115）

> 题意：给定字符串 `s` 和 `t`，问 `s` 的多少个不同子序列恰好等于 `t`。

**分析/状态设计**：前 `i` 个字符形成前 `j` 个字符的方案数，要么不用 `s[i-1]`，要么在 `s[i-1] == t[j-1]` 时使用它。因为方案数可能很大，这里用 `long long` 更稳妥。

```
dp[i][j] = 用 s 的前 i 个字符组成 t 的前 j 个字符的方案数

转移：
  先继承不选 s[i-1]：dp[i][j] = dp[i-1][j]
  若 s[i-1] == t[j-1]：dp[i][j] += dp[i-1][j-1]

初始化：
  dp[i][0] = 1
```

**C++ 代码**：
```cpp
long long dp[1005][1005];

int numDistinct(string s, string t) {
    int n = s.size(), m = t.size();
    for (int i = 0; i <= n; i++) dp[i][0] = 1;
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= m; j++) {
            dp[i][j] = dp[i - 1][j];
            if (s[i - 1] == t[j - 1]) dp[i][j] += dp[i - 1][j - 1];
        }
    return (int)dp[n][m];
}
// 调用方式/答案：返回 numDistinct(s, t)
```

**核心启示**：
- 这题是“LCS 思维”向“计数 DP”迈出的关键一步。
- 遇到“用前缀构造前缀”的方案数问题，先想“选当前 / 不选当前”。

### 题目 11：交错字符串（LeetCode 97）

> 题意：判断 `s3` 是否能由 `s1` 和 `s2` 按相对顺序交错组成。

**分析/状态设计**：走到 `dp[i][j]` 时，说明 `s1` 用了前 `i` 个字符、`s2` 用了前 `j` 个字符，于是 `s3` 已经匹配了前 `i+j` 个字符。最后一个字符只能来自 `s1` 或 `s2` 之一。

```
dp[i][j] = s1 前 i 个字符与 s2 前 j 个字符能否组成 s3 前 i+j 个字符

转移：
  若 s1[i-1] == s3[i+j-1]：dp[i][j] |= dp[i-1][j]
  若 s2[j-1] == s3[i+j-1]：dp[i][j] |= dp[i][j-1]
```

**C++ 代码**：
```cpp
bool isInterleave(string s1, string s2, string s3) {
    int n = s1.size(), m = s2.size();
    if (n + m != s3.size()) return false;
    vector<vector<char>> dp(n + 1, vector<char>(m + 1, 0));
    dp[0][0] = 1;
    for (int i = 0; i <= n; i++)
        for (int j = 0; j <= m; j++) {
            if (i && s1[i - 1] == s3[i + j - 1]) dp[i][j] |= dp[i - 1][j];
            if (j && s2[j - 1] == s3[i + j - 1]) dp[i][j] |= dp[i][j - 1];
        }
    return dp[n][m];
}
// 调用方式/答案：返回 isInterleave(s1, s2, s3)
```

**核心启示**：
- 这题是“两个来源拼成一个目标串”的标准模型。
- 看到“保持各自相对顺序”这句话，通常就该联想到双序列前缀 DP。

### 题目 12：通配符匹配（LeetCode 44）

> 题意：给定字符串 `s` 和模式串 `p`，其中 `?` 匹配任意单个字符，`*` 匹配任意长度字符串，判断是否匹配。

**分析/状态设计**：`?` 只吃掉一个字符，因此来自左上角；`*` 可以匹配空串，也可以继续吞掉 `s` 的字符，因此分别来自左边与上边。

```
dp[i][j] = s 前 i 个字符是否能被 p 前 j 个字符匹配

转移：
  若 p[j-1] == '*':
    dp[i][j] = dp[i][j-1] || dp[i-1][j]
  若 p[j-1] == '?' 或 s[i-1] == p[j-1]:
    dp[i][j] = dp[i-1][j-1]

初始化：
  dp[0][0] = true
  只有前缀全是 '*' 时，dp[0][j] 才为 true
```

**C++ 代码**：
```cpp
bool isMatch(string s, string p) {
    int n = s.size(), m = p.size();
    vector<vector<char>> dp(n + 1, vector<char>(m + 1, 0));
    dp[0][0] = 1;
    for (int j = 1; j <= m; j++)
        if (p[j - 1] == '*') dp[0][j] = dp[0][j - 1];
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= m; j++)
            if (p[j - 1] == '*') dp[i][j] = dp[i][j - 1] || dp[i - 1][j];
            else if (p[j - 1] == '?' || s[i - 1] == p[j - 1]) dp[i][j] = dp[i - 1][j - 1];
    return dp[n][m];
}
// 调用方式/答案：返回 isMatch(s, p)
```

**核心启示**：
- 字符串匹配题最重要的是把每个特殊字符的“语义”翻译成转移。
- `*` 是状态机味道最浓的一类字符，因为它既能停，也能继续吞字符。

### 题目 13：打家劫舍 II（LeetCode 213）

> 题意：房子首尾相连，仍然要求不能偷相邻房子，求最大收益。

**分析/状态设计**：因为首尾相邻，所以不能同时选第一间和最后一间。于是把问题拆成两次线性 DP：一次只考虑 `[0, n-2]`，一次只考虑 `[1, n-1]`，答案取最大。

```
环形数组 -> 两个线性子问题：
  1) 不选最后一个：做区间 [0..n-2]
  2) 不选第一个：做区间 [1..n-1]

每个子问题仍是：
  dp[i] = max(dp[i-1], dp[i-2] + nums[i])
```

**C++ 代码**：
```cpp
int solveRange(vector<int>& nums, int l, int r) {
    int prev2 = 0, prev1 = 0;
    for (int i = l; i <= r; i++) {
        int cur = max(prev1, prev2 + nums[i]);
        prev2 = prev1, prev1 = cur;
    }
    return prev1;
}

int rob(vector<int>& nums) {
    int n = nums.size();
    if (n == 1) return nums[0];
    return max(solveRange(nums, 0, n - 2), solveRange(nums, 1, n - 1));
}
// 调用方式/答案：返回 rob(nums)
```

**核心启示**：
- 环形结构常见套路：拆成两个互斥的线性区间。
- 一旦拆开，本质仍然回到最熟悉的线性 DP 模板。

### 题目 14：买卖股票的最佳时机（LeetCode 121/122/123）

> 题意：分别考虑最多做 `1` 次交易、可做无限次交易、最多做 `2` 次交易时的最大利润。

**分析/状态设计**：股票题最重要的是“状态身份”。
- `121`：只有一次买入和一次卖出。
- `122`：每天都在“持股 / 空仓”之间切换。
- `123`：把状态扩成 `hold1 / sold1 / hold2 / sold2`。

```
121：
  hold = max(hold, -price)
  cash = max(cash, hold + price)

122：
  newHold = max(hold, cash - price)
  newCash = max(cash, hold + price)

123：
  newHold1 = max(hold1, -price)
  newSold1 = max(sold1, hold1 + price)
  newHold2 = max(hold2, sold1 - price)
  newSold2 = max(sold2, hold2 + price)
```

**C++ 代码**：
```cpp
int maxProfit121(vector<int>& prices) {
    int hold = -prices[0], cash = 0;
    for (int i = 1; i < prices.size(); i++) {
        int preHold = hold, preCash = cash;
        hold = max(preHold, -prices[i]);
        cash = max(preCash, preHold + prices[i]);
    }
    return cash;
}

int maxProfit122(vector<int>& prices) {
    int hold = -prices[0], cash = 0;
    for (int i = 1; i < prices.size(); i++) {
        int preHold = hold, preCash = cash;
        hold = max(preHold, preCash - prices[i]);
        cash = max(preCash, preHold + prices[i]);
    }
    return cash;
}

int maxProfit123(vector<int>& prices) {
    int hold1 = -1000000000, sold1 = 0, hold2 = -1000000000, sold2 = 0;
    for (int p : prices) {
        int nhold1 = max(hold1, -p);
        int nsold1 = max(sold1, hold1 + p);
        int nhold2 = max(hold2, sold1 - p);
        int nsold2 = max(sold2, hold2 + p);
        hold1 = nhold1, sold1 = nsold1, hold2 = nhold2, sold2 = nsold2;
    }
    return sold2;
}
// 调用方式/答案：分别返回 maxProfit121 / 122 / 123
```

**核心启示**：
- 股票题不是记题，而是记“状态机”。
- 限制越多，通常只是把“身份状态”扩展开来。

### 题目 15：买卖股票含冷冻期（LeetCode 309）

> 题意：卖出股票后第二天不能立刻买入，求最大利润。

**分析/状态设计**：需要区分三种身份：持股 `hold`、今天刚卖出 `sold`、空仓且不在冷冻 `rest`。买入只能从 `rest` 转来，不能直接从 `sold` 转来。

int maxProfit(vector<int>& prices) {

### 题目 17：俄罗斯套娃信封（LeetCode 354）

> 题意：每个信封有宽和高，只有当宽和高都严格更大时，才能把一个信封套进另一个，求最多能套多少层。

**分析/状态设计**：先按宽升序、宽相同时按高降序排序。这样一来，宽度约束已经通过排序处理掉，剩下只需要在高度上做一次 LIS。

```
排序规则：
  宽升序；若宽相同，则高降序

原因：
  宽相同时不能互相嵌套，若高也升序，会错误地被 LIS 同时选中

之后：
  对高度数组求 LIS
```

**C++ 代码**：
```cpp
int maxEnvelopes(vector<vector<int>>& e) {
    sort(e.begin(), e.end(), [](auto& a, auto& b) {
        return a[0] == b[0] ? a[1] > b[1] : a[0] < b[0];
    });
    vector<int> tails;
    for (auto& x : e) {
        auto it = lower_bound(tails.begin(), tails.end(), x[1]);
        if (it == tails.end()) tails.push_back(x[1]);
        else *it = x[1];
    }
    return (int)tails.size();
}
// 调用方式/答案：返回 maxEnvelopes(e)
```

**核心启示**：
- 很多高阶线性 DP 题，真正难点在“先排序转化”。
- 二维偏序问题常通过“排序掉一维，另一维做 LIS”来解决。

### 题目 18：最长有效括号（LeetCode 32）

> 题意：给定只包含 `(` 和 `)` 的字符串，求最长有效括号子串长度。

**分析/状态设计**：定义 `dp[i]` 为“以 `i` 结尾”的最长有效括号长度，只在 `s[i] == ')'` 时才可能更新。关键是找到能和 `s[i]` 配对的那个左括号位置。

```
dp[i] = 以 s[i] 结尾的最长有效括号长度

若 s[i] == ')'：
  令 j = i - dp[i-1] - 1
  若 j >= 0 且 s[j] == '('：
    dp[i] = dp[i-1] + 2 + (j > 0 ? dp[j-1] : 0)
```

**C++ 代码**：
```cpp
int dp[30005];

int longestValidParentheses(string s) {
    int n = s.size(), ans = 0;
    for (int i = 1; i < n; i++)
        if (s[i] == ')') {
            int j = i - dp[i - 1] - 1;
            if (j >= 0 && s[j] == '(') {
                dp[i] = dp[i - 1] + 2 + (j ? dp[j - 1] : 0);
                ans = max(ans, dp[i]);
            }
        }
    return ans;
}
// 调用方式/答案：返回 longestValidParentheses(s)
```

**核心启示**：
- 这是“以 i 结尾”模型中最值得反复体会的一题。
- 看到括号匹配，不一定非要栈；DP 的优势是能直接把前面连成更长答案。

### 题目 19：单词拆分（LeetCode 139）

> 题意：给定字符串 `s` 和单词字典，判断 `s` 能否被拆成若干个字典中的单词。

**分析/状态设计**：把问题看成“前缀是否可行”。如果前 `j` 个字符已经能拆分，且 `s[j..i-1]` 恰好是一个单词，那么前 `i` 个字符也能拆分。

```
dp[i] = s 的前 i 个字符能否被拆分

转移：
  dp[i] = 存在某个 j < i
          使得 dp[j] = true 且 s[j..i-1] 在字典中

初始化：
  dp[0] = true
```

**C++ 代码**：
```cpp
bool dp[305];

bool wordBreak(string s, vector<string>& wordDict) {
    unordered_set<string> st(wordDict.begin(), wordDict.end());
    int n = s.size();
    dp[0] = true;
    for (int i = 1; i <= n; i++)
        for (int j = 0; j < i; j++)
            if (dp[j] && st.count(s.substr(j, i - j))) {
                dp[i] = true;
                break;
            }
    return dp[n];
}
// 调用方式/答案：返回 wordBreak(s, wordDict)
```

**核心启示**：
- “前缀能否成立”是布尔型线性 DP 的高频写法。
- 只要最后一段合法，前面是否合法就能由 `dp[j]` 接上。

### 题目 20：最大正方形（LeetCode 221）

> 题意：给定一个只包含 `0/1` 的矩阵，求只包含 `1` 的最大正方形面积。

**分析/状态设计**：虽然这是二维矩阵题，但它的转移仍然是“沿行列线性推进”的典型 DP。定义 `dp[i][j]` 为以 `(i, j)` 为右下角的最大正方形边长，那么它取决于上、左、左上三者的最小值。

```
dp[i][j] = 以 (i, j) 为右下角的最大全 1 正方形边长

若 matrix[i][j] == '1'：
  dp[i][j] = min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) + 1

答案：
  max(dp[i][j])²
```

**C++ 代码**：
```cpp
int dp[305][305];

int maximalSquare(vector<vector<char>>& matrix) {
    int n = matrix.size(), m = matrix[0].size(), ans = 0;
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= m; j++)
            if (matrix[i - 1][j - 1] == '1') {
                dp[i][j] = min({dp[i - 1][j], dp[i][j - 1], dp[i - 1][j - 1]}) + 1;
                ans = max(ans, dp[i][j]);
            }
    return ans * ans;
}
// 调用方式/答案：返回 maximalSquare(matrix)
```

**核心启示**：
- 有些二维题本质仍是“线性推进 + 邻接转移”。
- 只要状态定义成“以某点为右下角”，正方形扩展条件就会自然出现。

## 五、常见技巧与思路总结

### 5.1 “以 i 结尾” vs “前 i 个” 的选择

这是线性 DP 最核心、也最容易混淆的地方。

#### 什么时候选“以 i 结尾”

当答案必须包含当前位置，或者最后一步必须落在当前位置时，用“以 `i` 结尾”最自然。

典型题：
- LIS
- 最大子数组和
- 最长有效括号
- 最长重复子数组（二维版“以双下标结尾”）

通用味道：

```cpp
dp[i] = 某种以 i 结尾的最优值
```

#### 什么时候选“前 i 个”

当题目关注前缀整体是否成立、最优值是多少、能不能拆分时，用“前 `i` 个”更合适。

典型题：
- 爬楼梯
- 打家劫舍
- 单词拆分
- 编辑距离 / LCS（二维前缀）

通用味道：

```cpp
dp[i] = 前 i 个元素的答案
```

#### 一句话判断法

| 问法 | 常见状态 |
|------|----------|
| “必须以当前位置收尾” | `以 i 结尾` |
| “只看前缀整体答案” | `前 i 个` |
| “两个前缀如何匹配” | `dp[i][j]` |
| “一个子串 / 区间内部答案” | `dp[i][j]` |

### 5.2 状态压缩与空间优化

很多线性 DP 的第一维，本质只是“时间轴”。如果 `dp[i]` 只依赖上一层，就可以压缩空间。

最典型的三种压缩方式：

1. **二维压一维**：LCS、编辑距离可用滚动数组。
2. **一维压常数**：爬楼梯、打家劫舍、最大子数组和。
3. **状态机压变量**：股票题把 `dp[i][state]` 压成若干变量。

例如 LCS 的滚动思想：

```cpp
dp[j] 只保留当前行
额外用一个变量保存左上角旧值
```

压缩前一定先问自己：

> **当前状态会不会读到“本轮刚更新过”的值？**

如果会，就必须重新设计更新顺序或额外保存旧值。

### 5.3 LIS 的贪心二分优化

LIS 是最经典的“线性 DP + 数据结构优化”题。

朴素 DP 的瓶颈在于：
- 对每个 `i` 都要找所有 `j < i`
- 复杂度 `O(n²)`

贪心 + 二分的核心替代品是：

```
tails[len] = 长度为 len+1 的递增子序列的最小结尾
```

为什么它成立？
- 同样长度下，结尾越小，越有利于后面接更大的数
- 所以我们不必保存所有序列，只保存“每种长度最优的最小结尾”

这一招非常重要，因为它体现了动态规划里常见的思想：
**只保留对未来最有用的信息，而不是保留整个历史。**

### 5.4 状态机模型的通用思路

股票系列、字符串匹配系列，本质都可以看成状态机 DP。

做这类题，建议固定四步：

1. **列状态**：今天结束后我可能处于哪些身份？
2. **画转移**：哪些状态之间允许转移？
3. **写收益**：转移时利润 / 代价 / 可行性怎么变化？
4. **定初值**：第 0 天每个状态是否合法？

例如股票：
- `hold`：持股
- `cash`：空仓
- `sold`：今天刚卖
- `rest`：空仓且不在冷冻

例如通配符：
- 普通字符：只走左上角
- `?`：也只走左上角
- `*`：可以留在模式当前位置，持续吞字符

所以状态机 DP 的本质是：

> **把题目的限制翻译成“合法状态”和“合法边”。**

## 六、优化方法

### 6.1 滚动数组

当 `dp[i][j]` 只依赖上一行或上一列时，就可以滚动掉第一维。

常见场景：
- LCS
- 编辑距离
- 最长重复子数组
- 最大正方形

#### 什么时候能滚？

- 只依赖 `i-1` 层：通常可以滚
- 依赖左上角旧值：需要额外保存一个 `pre`
- 依赖当前行左侧：更新顺序要特别小心

#### 记忆法

- **只看上一层**：放心滚
- **还要看左上角**：多存一个旧值
- **看当前层左边**：检查顺序是否会污染状态

### 6.2 单调栈 / 单调队列优化

有些线性 DP 的转移长这样：

```
dp[i] = max(dp[j] + value(i, j))
其中 j 只能在一个滑动窗口里选
```

这时就很适合用 **单调队列** 维护窗口最值，复杂度从 `O(nk)` 降到 `O(n)`。

典型场景：
- 跳跃游戏 VI
- 受限子序列和
- 某些“前 k 个位置内转移”的 DP

而 **单调栈** 常用于先预处理“左边第一个更小 / 更大位置”，把原本需要枚举的大量状态边界快速确定下来，再配合 DP 做转移。

你要记住的不是具体代码，而是这类信号：
- 转移区间是滑动窗口
- 只关心窗口内最大 / 最小值
- 或者需要快速找到最近满足单调关系的位置

### 6.3 二分优化（LIS）

LIS 的 `O(n log n)` 写法，是线性 DP 中最经典的二分优化。

核心不在二分本身，而在于先找到可维护的中间态：

```
tails[len] = 长度为 len+1 的递增序列的最小结尾
```

一旦这个数组天然有序，就能立刻想到用二分替代线性查找。

#### 什么时候考虑二分优化

- 状态只和“某种有序最优边界”有关
- 你只关心“第一个 >= x”或“第一个 > x”的位置
- 每次更新都能保持数组整体有序

### 6.4 前缀和优化

当前缀型 DP 的转移需要枚举一整个区间时，前缀和经常能把“区间求和 / 区间方案数累加”优化掉。

常见形式：

```
dp[i] = sum(dp[j])，其中 j 落在某个连续区间
```

如果直接枚举 `j`，复杂度可能是 `O(n²)`；
但如果维护前缀和 `pre[i] = dp[0] + ... + dp[i]`，就能把一次区间和变成 `O(1)`。

典型应用：
- 计数型 DP
- 限长切分 DP
- 某些多重背包 / 线性序列计数问题

一句话概括：

> **转移在枚举一段连续区间时，优先想前缀和；转移在枚举一段滑动窗口最值时，优先想单调队列。**

## 七、终极总结与归纳

### 7.1 线性 DP 的本质

线性 DP 的本质，可以压缩成一句话：

> **在一个按下标推进的结构上，利用更短前缀或更小区间的答案，合并出更长前缀或更大区间的答案。**

不管题目包装成台阶、房子、字符串、括号、股票、信封还是矩阵，本质都逃不开三问：
- 状态到底描述谁？
- 最后一步从哪里来？
- 答案落在哪个状态里？

### 7.2 五大核心要点

1. **先定状态，再谈转移**
   - 线性 DP 最怕一上来就背公式，不先想“这个状态在描述什么”。

2. **“以 i 结尾”和“前 i 个”要分清**
   - 一个强调最后一步必须落在 `i`，一个强调前缀整体最优。

3. **双序列题先想上、左、左上**
   - LCS、编辑距离、匹配问题几乎都绕不开这三个来源。

4. **限制越多，越像状态机**
   - 冷冻期、手续费、交易次数限制，本质都在加状态。

5. **优化来自“只保留最有用的信息”**
   - LIS 的 `tails`、滚动数组、单调队列，本质都在压缩无用历史。
