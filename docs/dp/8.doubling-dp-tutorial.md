## 一、什么是倍增优化动态规划

很多动态规划的慢点，不在于“状态不会定义”，而在于“转移一次太便宜，转很多次太昂贵”。例如从一个点连续走 `k` 步、沿父链往上跳很多层、重复执行某个固定状态转移很多轮，这时朴素做法往往要一下一下跳，复杂度是 `O(k)` 甚至更高。

**倍增**做的事情，是把“连续做很多次的一步转移”压缩成“少量的 `2^0, 2^1, 2^2, ...` 大步转移”。这样，原本要做 `k` 次的小转移，可以通过二进制拆分改成 `O(log k)` 次大转移。

所以，所谓**倍增优化动态规划**，并不是“发明一种新的 DP”，而是：

- 先定义好“一步转移”；
- 再预处理“做 `2^j` 次转移后会变成什么”；
- 最后用二进制拆解，把多步转移快速拼出来。

这套思想最常出现于三类问题：

| 场景 | 一步转移是什么 | 倍增后维护什么 |
|------|----------------|----------------|
| 跳点/祖先 | 走一步到下一个点 | `to[u][j]` |
| 路径信息 | 走一步并附带权值 | `to/sum/min/max` |
| 线性递推 / 固定转移 | 做一次状态变换 | 转移矩阵 `A^(2^j)` |

把话说得更直白一点：**倍增优化 DP 的核心，不是“倍增”，而是“重复转移可合并”。**

```text
朴素：走 13 步 = 1 + 1 + 1 + ... + 1
倍增：13 = 8 + 4 + 1
      只要会走 1、4、8 步，就能拼出 13 步
```

## 二、核心思想与前置知识

### 2.1 什么是倍增

倍增的数学基础非常简单：任意正整数都可以拆成若干个二次幂之和。

```text
13 = 8 + 4 + 1 = 2^3 + 2^2 + 2^0
25 = 16 + 8 + 1 = 2^4 + 2^3 + 2^0
```

因此，只要你能快速知道：

- 从状态 `x` 出发，做 `2^0` 次转移后去哪；
- 做 `2^1` 次转移后去哪；
- 做 `2^2` 次转移后去哪；
- ...

那么任意次多步转移，都能用二进制拆分拼出来。

### 2.2 倍增的常规做法

最基础的倍增表只有一个含义：

```text
to[x][j] = 从状态 x 出发，做 2^j 次转移后到达的状态
```

递推方式极其固定：

```text
to[x][j] = to[ to[x][j-1] ][j-1]
```

含义就是：

- 先做 `2^(j-1)` 次；
- 再从中间状态继续做 `2^(j-1)` 次；
- 合起来就是做了 `2^j` 次。

这就是所有倍增表的核心递推原型。

### 2.3 跳点型倍增骨架

```cpp
const int LOG = 60;
const int NIL = 0;
int to[MAXN][LOG + 1];

for (int u = 1; u <= n; u++)
    to[u][0] = nxt[u];

for (int j = 1; j <= LOG; j++)
    for (int u = 1; u <= n; u++) {
        int mid = to[u][j - 1];
        to[u][j] = (mid == NIL ? NIL : to[mid][j - 1]);
    }
```

这里的 `nxt[u]` 就是“一步转移”。若题目里可能“跳空”，通常约定 `0` 或 `-1` 为哨兵并保留这层 guard；若每个状态都有唯一后继，则可以省略判空版本。它可以是：

- 父亲节点；
- 函数图下一点；
- 某个确定策略下下一座城市；
- 某种自动机上的下一个状态。

### 2.4 带信息倍增骨架

很多题不只是问“走到哪”，还问：

- 路径和是多少；
- 最小值是多少；
- 最大值是多少；
- A 走了多少、B 走了多少。

这时除了 `to`，还要维护“跟着这段跳跃一起被合并的信息”。

```cpp
const int LOG = 60;
const long long INF = 4e18;
const int NIL = 0;
int to[MAXN][LOG + 1];
long long sum[MAXN][LOG + 1];
long long mn[MAXN][LOG + 1];

for (int u = 1; u <= n; u++)
    for (int j = 0; j <= LOG; j++)
        mn[u][j] = INF;

for (int u = 1; u <= n; u++) {
    to[u][0] = nxt[u];
    if (to[u][0] != NIL) {
        sum[u][0] = w[to[u][0]];
        mn[u][0] = w[to[u][0]];
    }
}

for (int j = 1; j <= LOG; j++)
    for (int u = 1; u <= n; u++) {
        int mid = to[u][j - 1];
        if (mid == NIL || to[mid][j - 1] == NIL) continue;
        to[u][j] = to[mid][j - 1];
        sum[u][j] = sum[u][j - 1] + sum[mid][j - 1];
        mn[u][j] = min(mn[u][j - 1], mn[mid][j - 1]);
    }
```

这类倍增的本质是：**位置和信息同时倍增**。若题目统计的是边权而不是到达点的点权，只需把 `sum[u][0]`、`mn[u][0]` 改成“一步转移本身带来的代价”即可。

### 2.5 矩阵快速幂为什么也属于倍增

如果一个 DP 的转移可以写成：

```text
state_i = A × state_{i-1}
```

那么做很多轮转移后就是：

```text
state_k = A^k × state_0
```

而矩阵快速幂的递推：

```text
A^(2^j) = A^(2^(j-1)) × A^(2^(j-1))
```

本质上和 `to[x][j]` 的倍增完全一样，只不过：

- 跳点型是在“状态空间”里倍增；
- 矩阵快速幂是在“转移算子”上倍增。

因此，矩阵快速幂可以看成“倍增优化 DP”在**线性递推 / 固定转移**场景下的标准形态。

### 2.6 什么时候该想到倍增优化 DP

如果题目里出现了以下信号，就要警觉：

1. 有一个**确定的一步转移**。
2. 需要执行这个转移很多次，比如 `k` 次，且 `k` 很大。
3. 除了最终位置，还要维护路径上的一些可合并信息。
4. 题目本质是“固定转移做很多轮”，例如线性递推、固定步数路径。

一句话判断：

> **当你发现“我其实是在重复做同一类转移很多次”时，倍增通常就是第一选择。**

## 三、子模型与分类详解

### 3.1 跳点型

```
状态定义：常写成 `to[u][j]`
转移关键：`to[u][j] = to[to[u][j-1]][j-1]`
适用信号：只关心多步之后到哪，不关心中间附加信息
实现要点：先定义一步转移；若可能跳空，要保留 `0/-1` 哨兵，再递推出 `2^j` 步转移
```

```cpp
const int NIL = 0;
for (int u = 1; u <= n; u++)
    to[u][0] = nxt[u];
for (int j = 1; j <= LOG; j++)
    for (int u = 1; u <= n; u++) {
        int mid = to[u][j - 1];
        to[u][j] = (mid == NIL ? NIL : to[mid][j - 1]);
    }
```

### 3.2 带信息合并型

```
状态定义：常写成 `to[u][j] + info[u][j]`
转移关键：位置按倍增跳，信息按可结合的规则合并
适用信号：查询多步跳跃后的位置、和、最小值、最大值、贡献值
实现要点：信息的合并律要稳定，通常是加法、min/max 或矩阵乘法；若可能跳空，还要同步处理空跳 guard
```

```cpp
const int NIL = 0;
int mid = to[u][j - 1];
if (mid == NIL || to[mid][j - 1] == NIL) continue;
to[u][j] = to[mid][j - 1];
sum[u][j] = sum[u][j - 1] + sum[mid][j - 1];
mn[u][j] = min(mn[u][j - 1], mn[mid][j - 1]);
```

### 3.3 固定步数转移型

```
状态定义：`f[j][u][v]` 或 `reach[u][v][j]`
转移关键：把“恰好 2^j 步”的可达/代价从两个 `2^(j-1)` 合并出来
适用信号：题目要求恰好走固定步数，或一步能走 2^k 长度
实现要点：先做 powers-of-two 可达性，再挂到最短路/DP 主体上
```

```cpp
for (int j = 1; j <= LOG; j++)
    for (int u = 1; u <= n; u++)
        for (int v = 1; v <= n; v++)
            for (int mid = 1; mid <= n; mid++)
                if (f[j - 1][u][mid] && f[j - 1][mid][v])
                    f[j][u][v] = true;
```

### 3.4 线性递推 / 矩阵转移型

```
状态定义：把若干个 DP 状态压成列向量 `state`
转移关键：一步转移写成矩阵 `A`，多步转移变成 `A^k`
适用信号：递推式固定、阶数固定、步数很大
实现要点：矩阵乘法必须和问题的“合并运算”一致
```

```cpp
Matrix qpow(Matrix base, long long exp) {
    Matrix res = identity();
    while (exp) {
        if (exp & 1) res = mul(res, base);
        base = mul(base, base);
        exp >>= 1;
    }
    return res;
}
```

### 3.5 树上转移矩阵型（相关扩展）

这类题不一定显式维护 `to[u][j]`，但和倍增共享同一底层思想：**把大量重复出现的局部转移压成可快速合并的算子**。

```
状态定义：把树形 DP 的局部转移写成小矩阵，再沿链合并
转移关键：单点修改影响局部矩阵，再用乘法把整条链的信息拼回去
适用信号：树形 DP + 多次带限制询问
实现要点：矩阵表示局部状态转移，树剖/链分治负责快速合并
```

```cpp
struct Mat {
    long long a[2][2];
};

Mat mul(const Mat& A, const Mat& B) {
    Mat C;
    // min-plus 或普通乘法，视题目而定
    return C;
}
```

### 3.6 模型对比总表

| 模型 | 核心对象 | 典型信息 | 代表题 |
|------|----------|----------|--------|
| 跳点型 | `to[u][j]` | 位置 | 二叉提升、祖先问题 |
| 带信息合并型 | `to + sum/min/max` | 路径统计 | 开车旅行、函数图 |
| 固定步数转移型 | `f[j][u][v]` | 可达 / 最短路 | 跑路、Cow Relays |
| 矩阵转移型 | `A^(2^j)` | 线性递推 | 斐波那契数列 |
| 树上矩阵型 | 链上转移矩阵 | DP 状态合并 | 保卫王国 |

### 题目 1：最近公共祖先（洛谷 P3379）

> 给定一棵树和多次询问，每次询问两个点的最近公共祖先。

**分析**：这题不是传统 DP 题，但它是最标准的跳点型倍增练习。先把“走一步”定义成“走到父亲”，再把若干次上跳压成 `2^j` 层祖先，几乎所有 `to[u][j]` 类题都从这里起步。

```
状态定义：`up[u][j]` 表示点 u 向上走 `2^j` 步到达的祖先，`dep[u]` 表示深度
转移方程：`up[u][j] = up[up[u][j-1]][j-1]`
边界条件：`up[root][0] = 0`，根的深度从 1 开始
答案位置：先把更深的点提升到同层，再同步上跳直到父亲相同
```

**C++ 代码**：
```cpp
const int LOG = 20;
int up[MAXN][LOG + 1], dep[MAXN];

void dfs(int u, int p) {
    up[u][0] = p;
    dep[u] = dep[p] + 1;
    for (int j = 1; j <= LOG; j++)
        up[u][j] = up[up[u][j - 1]][j - 1];
    for (int v : adj[u])
        if (v != p) dfs(v, u);
}

int lca(int u, int v) {
    if (dep[u] < dep[v]) swap(u, v);
    int d = dep[u] - dep[v];
    for (int j = LOG; j >= 0; j--)
        if ((d >> j) & 1) u = up[u][j];
    if (u == v) return u;
    for (int j = LOG; j >= 0; j--)
        if (up[u][j] != up[v][j]) u = up[u][j], v = up[v][j];
    return up[u][0];
}
```

**核心启示**：
- **倍增表的第一性原理** 就是“把一步转移扩成 `2^j` 步转移”。
- **先补齐到同一层，再同步跳**，是查询时二进制拆分的典型用法。
- **虽然不是传统 DP**，但它是理解跳点型倍增最好的门槛题。

### 题目 2：斐波那契数列（洛谷 P1962）

> 求第 `n` 项斐波那契数对 `1e9+7` 取模，`n` 很大，朴素递推无法承受。

**分析**：这题是矩阵快速幂的标准入门题。核心不是“数列”，而是“固定线性转移做很多轮”，这正是倍增优化的最经典场景。

```
状态定义：`[F_n, F_{n-1}]^T` 作为状态向量
转移方程：`state_n = A × state_{n-1}`，其中 `A = [[1,1],[1,0]]`
边界条件：`F_1 = 1, F_2 = 1`
答案位置：`fib(n)`，等价于 `A^(n-2)` 乘上初始向量后的第一维
```

**C++ 代码**：
```cpp
const long long MOD = 1000000007;
struct Mat { long long a[2][2]; };

Mat mul(Mat A, Mat B) {
    Mat C = {};
    for (int i = 0; i < 2; i++)
        for (int j = 0; j < 2; j++)
            for (int k = 0; k < 2; k++)
                C.a[i][j] = (C.a[i][j] + A.a[i][k] * B.a[k][j]) % MOD;
    return C;
}

Mat qpow(Mat base, long long exp) {
    Mat res = {{{1, 0}, {0, 1}}};
    while (exp) {
        if (exp & 1) res = mul(res, base);
        base = mul(base, base);
        exp >>= 1;
    }
    return res;
}

long long fib(long long n) {
    if (n <= 2) return 1;
    Mat A = {{{1, 1}, {1, 0}}};
    Mat P = qpow(A, n - 2);
    return (P.a[0][0] + P.a[0][1]) % MOD;
}
```

**核心启示**：
- **矩阵快速幂本质上就是倍增**。
- **固定线性递推** 优先考虑矩阵化。
- **状态向量 + 转移矩阵** 是把 DP 压成 `O(log n)` 的标准套路。

### 题目 3：Cow Relays（POJ 3613）

> 给一张图，要求从 `S` 到 `E`，**恰好经过 `N` 条边**，求最短路。

**分析**：这是“限步数最短路”的经典模型。普通最短路算法不适用，因为题目要求“恰好走 `N` 步”。把一步转移写成 min-plus 矩阵后，就能用矩阵快速幂倍增步数。

```
状态定义：`A[i][j]` 表示从 i 到 j 恰好走 1 条边的最短代价
转移方程：`A^k[i][j]` 表示恰好走 k 条边的最短代价（min-plus 乘法）
边界条件：无边记为 `INF`
答案位置：`A^N[S][E]`
```

**C++ 代码**：
```cpp
const long long INF = 4e18;
struct Mat { long long a[MAXV][MAXV]; };

Mat mul(Mat A, Mat B) {
    Mat C;
    for (int i = 1; i <= V; i++)
        for (int j = 1; j <= V; j++)
            C.a[i][j] = INF;
    for (int i = 1; i <= V; i++)
        for (int k = 1; k <= V; k++)
            for (int j = 1; j <= V; j++) {
                if (A.a[i][k] == INF || B.a[k][j] == INF) continue;
                C.a[i][j] = min(C.a[i][j], A.a[i][k] + B.a[k][j]);
            }
    return C;
}
```

**核心启示**：
- **矩阵乘法不一定是普通乘法**，也可以是 min-plus。
- **固定步数路径问题** 很适合矩阵倍增。
- **当“步数”是约束时**，优先想“恰好 k 步”的转移结构。

### 题目 4：跑路（洛谷 P1613）

> 一秒钟可以选择跑 `2^k` 千米，图上每条边长 1，问从 1 到 n 最少需要多少秒。

**分析**：这题的关键不是直接算最短路，而是先求“能否在 1 秒内从 i 到 j”。因为一秒能跑 `2^k`，所以只要存在一条长度恰为 `2^k` 的路径，就能把 `i → j` 看成新图中的一条边，边权为 1。

```
状态定义：`reach[j][u][v]` 表示从 u 到 v 是否能恰好走 `2^j` 步
转移方程：若存在 mid 使 `reach[j-1][u][mid]` 和 `reach[j-1][mid][v]` 都为真，则 `reach[j][u][v] = true`
边界条件：原图中的有向边对应 `reach[0][u][v] = true`
答案位置：在新图上跑 Floyd / 最短路后取 `dist[1][n]`
```

**C++ 代码**：
```cpp
const int INF = 0x3f3f3f3f;
bool reach[LOG + 1][MAXN][MAXN];
int dist[MAXN][MAXN];

for (int j = 1; j <= LOG; j++)
    for (int u = 1; u <= n; u++)
        for (int mid = 1; mid <= n; mid++)
            if (reach[j - 1][u][mid])
                for (int v = 1; v <= n; v++)
                    if (reach[j - 1][mid][v]) reach[j][u][v] = true;

for (int u = 1; u <= n; u++)
    for (int v = 1; v <= n; v++) {
        dist[u][v] = (u == v ? 0 : INF);
        for (int j = 0; j <= LOG; j++)
            if (reach[j][u][v]) dist[u][v] = 1;
    }

for (int k = 1; k <= n; k++)
    for (int u = 1; u <= n; u++)
        for (int v = 1; v <= n; v++)
            if (dist[u][k] < INF && dist[k][v] < INF)
                dist[u][v] = min(dist[u][v], dist[u][k] + dist[k][v]);
```

**核心启示**：
- **powers-of-two 可达性** 也可以先预处理出来。
- **先倍增建新图，再做最短路** 是很典型的组合拳。
- **题目里只要出现“每次可选 `2^k`”**，就要立刻想到倍增。

### 题目 5：在传球游戏中最大化函数值（LeetCode 2836）

> 给定函数图 `receiver` 和传球次数 `k`，可以任选起点。要求最大化所有被传到球的点编号之和。

**分析**：这题和函数图倍增高度贴合。一步转移就是“把球传给 `receiver[u]`”，难点在于不仅要知道 `k` 步后到哪，还要维护这 `k` 次传球过程中新增的编号和。

```
状态定义：`to[u][j]` 表示从 u 出发传 `2^j` 次后停在哪，`sum[u][j]` 表示这 `2^j` 次传球新增的编号和
转移方程：`to[u][j] = to[to[u][j-1]][j-1]`，`sum[u][j] = sum[u][j-1] + sum[to[u][j-1]][j-1]`
边界条件：`to[u][0] = receiver[u]`，`sum[u][0] = receiver[u]`
答案位置：对每个起点 `u`，计算 `calc(u, k)`，取最大值
```

**C++ 代码**：
```cpp
for (int u = 0; u < n; u++) {
    to[u][0] = receiver[u];
    sum[u][0] = receiver[u];
}
for (int j = 1; j <= LOG; j++)
    for (int u = 0; u < n; u++) {
        int mid = to[u][j - 1];
        to[u][j] = to[mid][j - 1];
        sum[u][j] = sum[u][j - 1] + sum[mid][j - 1];
    }

long long calc(int start, long long k) {
    long long res = start;
    int cur = start;
    for (int j = 0; j <= LOG; j++)
        if ((k >> j) & 1) res += sum[cur][j], cur = to[cur][j];
    return res;
}
```

**核心启示**：
- **函数图 + 大步数** 是倍增的天然主场。
- **sum 的定义要和查询严格对齐**，这里维护的是“新增贡献”，所以起点编号要单独加上。
- **位置和答案贡献同时倍增**，就是“带信息倍增”的标准写法。

### 题目 6：Analysis of Pathes in Functional Graph（CF 702E）

> 给一个函数图，每个点只有一条出边。对每个起点，求走 `k` 步后的路径和与路径最小值。

**分析**：函数图上的路径是确定的，因此天然适合倍增。除了位置 `to`，还要同时维护路径和 `sum` 与最小值 `mn`。

```
状态定义：`to[u][j]`、`sum[u][j]`、`mn[u][j]`
转移方程：位置按倍增跳，和用加法合并，最小值用 min 合并
边界条件：`j = 0` 对应走 1 步
答案位置：把 k 按二进制拆开，逐位累加结果
```

**C++ 代码**：
```cpp
for (int j = 1; j <= LOG; j++)
    for (int u = 1; u <= n; u++) {
        int mid = to[u][j - 1];
        to[u][j] = to[mid][j - 1];
        sum[u][j] = sum[u][j - 1] + sum[mid][j - 1];
        mn[u][j] = min(mn[u][j - 1], mn[mid][j - 1]);
    }
```

**核心启示**：
- **位置和信息一起倍增**，是倍增优化 DP 的标准升级版。
- **函数图** 是倍增最天然的应用场景之一。
- **多种统计量同时维护** 时，要确认它们都满足稳定的合并方式。

### 🟠 提升级 Level 3：真正开始“像 DP 一样思考倍增” 

### 题目 7：ZAB-Frog（洛谷 P3509）

> 河床上有很多石头。每次青蛙都会跳到“第 k 近的石头”，问从每块石头出发连跳 `m` 次后停在哪。

**分析**：这题先要把“一步转移”预处理出来。借助双指针/滑动窗口，可以在线性时间求出每块石头的“第 k 近目标”，随后问题就退化成标准的“函数图上跳 `m` 次到哪”，直接上倍增即可。

```
状态定义：`to[i][j]` 表示从第 i 块石头出发跳 `2^j` 次后停在哪
转移方程：`to[i][j] = to[to[i][j-1]][j-1]`
边界条件：`to[i][0] = nxt[i]`，其中 `nxt[i]` 由双指针预处理
答案位置：对每个起点把 `m` 按二进制拆开后不断跳转
```

**C++ 代码**：
```cpp
for (int i = 1; i <= n; i++)
    to[i][0] = nxt[i];
for (int j = 1; j <= LOG; j++)
    for (int i = 1; i <= n; i++)
        to[i][j] = to[to[i][j - 1]][j - 1];

int jump(int u, long long step) {
    for (int j = 0; j <= LOG; j++)
        if ((step >> j) & 1) u = to[u][j];
    return u;
}
```

**核心启示**：
- **很多倍增题的真正难点在于找出一步转移**，倍增本身反而是后半段。
- **预处理 `nxt[i]` 后，整题就变成了标准跳点模板**。
- **“先建确定转移，再做二进制跳跃”** 是很常见的出题套路。

### 题目 8：开车旅行（洛谷 P1081）

> 城市按编号从西到东排列。A 和 B 轮流开车，A 总去第二近城市，B 总去最近城市。给定很多询问，问在总路程不超过 x 时 A、B 各走多远。

**分析**：这题的一步转移不是“图上的任意边”，而是在预处理出“最近 / 次近城市”后形成的**确定转移**。由于驾驶人轮流变化，所以倍增状态里不仅要记“跳到哪”，还要记 A、B 在这段 `2^j` 跳跃里各自累计了多少路程。

```
状态定义：`to[u][j]` 表示从 u 出发执行 `2^j` 天后的城市，`da[u][j]` / `db[u][j]` 表示这段过程中 A、B 的总路程
转移方程：把两个 `2^(j-1)` 段拼起来，同时合并去向和路程信息
边界条件：`j = 0` 由“开一天”得到
答案位置：对每组 `(s, x)` 输出二进制拆分查询得到的 `(ansA, ansB)`
```

**C++ 代码**：
```cpp
for (int j = 1; j <= LOG; j++)
    for (int u = 1; u <= n; u++) {
        int mid = to[u][j - 1];
        if (!mid) continue;
        to[u][j] = to[mid][j - 1];
        da[u][j] = da[u][j - 1] + da[mid][j - 1];
        db[u][j] = db[u][j - 1] + db[mid][j - 1];
    }

pair<long long, long long> query(int s, long long lim) {
    long long ansA = 0, ansB = 0;
    int cur = s;
    for (int j = LOG; j >= 0; j--)
        if (cur && da[cur][j] + db[cur][j] <= lim) {
            lim -= da[cur][j] + db[cur][j];
            ansA += da[cur][j];
            ansB += db[cur][j];
            cur = to[cur][j];
        }
    return {ansA, ansB};
}
```

**核心启示**：
- **一步转移只要是确定的，就能倍增。**
- **倍增表里可以挂多个统计量**，不止一个 `sum`。
- **很多“模拟很多天/很多轮”的题**，本质都是倍增优化的查询题。

### 🔴 高级 Level 4：从二进制跳跃扩展到“转移算子”的快速合并 

### 题目 9：国旗计划（洛谷 P4155）

> 给定环形边境上的若干可覆盖区间。对于每个战士，若必须选他，求至少共选多少人才能覆盖整整一圈。

**分析**：这题先把环断成链，并把区间复制一遍。排序后可用贪心/双指针预处理“从区间 i 出发，再选一个人时应该跳到哪个区间”，随后就变成“最少跳多少次能把右端点推进至少一圈”，非常适合倍增。

```
状态定义：`go[i][j]` 表示从区间 i 出发，贪心地再选 `2^j` 个区间后跳到的位置
转移方程：`go[i][j] = go[go[i][j-1]][j-1]`
边界条件：`go[i][0] = nxt[i]`，其中 `nxt[i]` 是再选 1 个区间后的贪心去向
答案位置：从起点 s 出发，倍增找到覆盖长度第一次达到 `seg[s].l + m` 的最小人数
```

**C++ 代码**：
```cpp
for (int i = 1; i <= 2 * n; i++)
    go[i][0] = nxt[i];
for (int j = 1; j <= LOG; j++)
    for (int i = 1; i <= 2 * n; i++)
        go[i][j] = go[go[i][j - 1]][j - 1];

int solve(int s) {
    int cur = s, used = 1;
    for (int j = LOG; j >= 0; j--) {
        int nxt_id = go[cur][j];
        if (seg[nxt_id].r < seg[s].l + m) cur = nxt_id, used += 1 << j;
    }
    return used + 1;
}
```

**核心启示**：
- **区间覆盖也能抽象成“确定跳转”**，于是就能套倍增。
- **断环成链 + 复制区间** 是环形覆盖问题的经典预处理。
- **倍增不只用在点上跳，也可以用在区间状态上跳。**

### 题目 10：保卫王国（洛谷 P5024）

> 树上每条边两端至少有一个点选中。多次询问中，两个点会被强制选 / 不选，要求输出最小总代价。

**分析**：这题的基础是树上最小权点覆盖 DP。难点在于多次带限制询问。经典做法是把局部树形 DP 转移写成小矩阵，再通过链上合并快速得到整体结果。虽然这里最核心的实现常配合树剖，但底层仍然是“把多次固定转移快速合并”的思想，和倍增 / 快速幂一脉相承。

```
状态定义：单个节点或一条链的 DP 转移写成 `2×2` 小矩阵
转移方程：链上矩阵按 min-plus 方式合并，表示把局部 DP 连起来
边界条件：强制选 / 不选可通过把非法状态赋成 `INF` 实现
答案位置：整棵树对应的合并矩阵作用到根后得到最优值
```

**C++ 代码**：
```cpp
const long long INF = 4e18;
struct Mat {
    long long a[2][2];
};

Mat mul(const Mat& A, const Mat& B) {
    Mat C;
    for (int i = 0; i < 2; i++)
        for (int j = 0; j < 2; j++) {
            C.a[i][j] = INF;
            for (int k = 0; k < 2; k++)
                if (A.a[i][k] < INF && B.a[k][j] < INF)
                    C.a[i][j] = min(C.a[i][j], A.a[i][k] + B.a[k][j]);
        }
    return C;
}
```

**核心启示**：
- **倍增思想的本质是“快速合并重复转移”**，不一定非得显式写出 `to[u][j]`。
- **树形 DP + 小矩阵** 是“转移算子化”的代表。
- **当多次询问只改局部约束** 时，要考虑把 DP 转移抽象成可合并对象。

### 🟣 拓展级 Level 5：把倍增推进到对象接口、自动机与高维转移

### 题目 11：树节点的第 K 个祖先（LeetCode 1483）

> 给定一棵根树的 `parent` 数组，多次询问某个节点的第 `k` 个祖先，不存在则返回 `-1`。

**分析**：这题本质上是“树上跳点型倍增”的 API 版本。相比 `P3379` 的 LCA，它更直接地暴露了“单点上跳 `k` 步”的接口，因此很适合作为树上倍增的独立巩固题。

```
状态定义：`up[u][j]` 表示节点 u 的 `2^j` 级祖先
转移方程：若 `up[u][j-1] != -1`，则 `up[u][j] = up[up[u][j-1]][j-1]`
边界条件：`up[u][0] = parent[u]`，不存在祖先时保持 `-1`
答案位置：把 k 按二进制拆开，逐位把 u 往上跳；若途中变成 `-1` 就提前结束
```

**C++ 代码**：
```cpp
const int LOG = 17;
int up[MAXN][LOG + 1];

for (int u = 0; u < n; u++)
    up[u][0] = parent[u];
for (int j = 1; j <= LOG; j++)
    for (int u = 0; u < n; u++) {
        int mid = up[u][j - 1];
        up[u][j] = (mid == -1 ? -1 : up[mid][j - 1]);
    }

int get_kth(int u, int k) {
    for (int j = 0; j <= LOG && u != -1; j++)
        if ((k >> j) & 1) u = up[u][j];
    return u;
}
```

**核心启示**：
- **树上倍增的最纯粹接口** 就是“第 k 级祖先”。
- **空祖先要显式保留为 `-1`**，不能默认继续二跳。
- **对象题 / 查询题** 里，倍增经常以“预处理 + 多次接口调用”的形态出现。

### 题目 12：DNA Sequence（POJ 2778）

> 给定若干禁止出现的 DNA 片段，问长度为 `n` 的 DNA 串中，有多少个不会包含任一禁串，答案对 `100000` 取模。

**分析**：单纯按长度做自动机 DP 是 `O(n · 状态数)`，但这里 `n` 很大，必须把“在 AC 自动机上走一步”的转移压成矩阵，再用快速幂一次性完成 `n` 轮转移。这是“自动机 DP + 矩阵快速幂”的经典模板题。

```
状态定义：AC 自动机的每个安全节点是一个 DP 状态，`T[i][j]` 表示从状态 i 走 1 步到状态 j 的方案数
转移方程：长度为 n 的总转移就是 `T^n`
边界条件：含禁串后缀的节点记为坏点，相关转移一律不连；长度 0 对应单位矩阵
答案位置：`T^n` 中从根节点出发到所有安全节点的方案数之和
```

**C++ 代码**：
```cpp
const int MOD = 100000;
struct Mat { int a[MAXS][MAXS]; };

for (int u = 0; u < sz; u++)
    if (!bad[u])
        for (int c = 0; c < 4; c++) {
            int v = tr[u][c];
            if (!bad[v]) T.a[u][v] = (T.a[u][v] + 1) % MOD;
        }

Mat P = qpow(T, n);
for (int v = 0; v < sz; v++)
    if (!bad[v]) ans = (ans + P.a[rt][v]) % MOD;
```

**核心启示**：
- **自动机上的一步转移** 也能被矩阵化。
- **禁串计数题** 一旦长度很大，通常要把“按位 DP”升级成“转移矩阵快速幂”。
- **坏点必须在建图时就剪掉**，不能留到最后再排除。

### 题目 13：Treasure Island（CF 352E）

> 给定带权有向图，要求从起点到终点**恰好走 k 条边**，求最小总代价，其中 `k` 很大。

**分析**：这题是高维 min-plus 转移矩阵的代表。普通最短路无法处理“恰好 k 条边”，朴素 `dp[步数][点]` 又扛不住超大的 `k`。把一步转移写成邻接矩阵后，对它做 min-plus 快速幂即可。

```
状态定义：`G[i][j]` 表示从 i 到 j 恰好走 1 条边的最小代价
转移方程：在 min-plus 乘法下，`G^k[i][j]` 表示恰好走 k 条边的最小代价
边界条件：无边记为 `INF`；min-plus 单位矩阵满足对角线为 0、其余为 `INF`
答案位置：`qpow(G, k).a[S][T]`
```

**C++ 代码**：
```cpp
const long long INF = 4e18;
struct Mat { long long a[MAXV][MAXV]; };

Mat mul(const Mat& A, const Mat& B) {
    Mat C;
    for (int i = 1; i <= V; i++)
        for (int j = 1; j <= V; j++)
            C.a[i][j] = INF;
    for (int i = 1; i <= V; i++)
        for (int k = 1; k <= V; k++) {
            if (A.a[i][k] == INF) continue;
            for (int j = 1; j <= V; j++) {
                if (B.a[k][j] == INF) continue;
                C.a[i][j] = min(C.a[i][j], A.a[i][k] + B.a[k][j]);
            }
        }
    return C;
}
```

**核心启示**：
- **min-plus 快速幂** 是“固定步数最短路”最标准的高阶武器。
- **`INF` guard 不能省**，否则骨架代码会把不可达态错误参与转移。
- **矩阵快速幂的本质不是线代，而是重复转移的快速合并。**

## 五、常见技巧与思路总结

### 5.1 先找“一步转移”

倍增的第一步永远不是开 `LOG` 数组，而是先想清楚：

> **题目中的“一步”到底是什么？**

- 祖先问题：一步是“走到父亲”；
- 函数图：一步是“走到唯一出边”；
- 开车旅行：一步是“按当前规则开一天”；
- 矩阵快速幂：一步是“做一次状态变换”。

### 5.2 确认信息能不能合并

如果除了位置，还要维护路径上的信息，那么这些信息必须满足可稳定合并的规律。

```cpp
sum = left_sum + right_sum;
mn = min(left_mn, right_mn);
mx = max(left_mx, right_mx);
```

一旦合并规则不稳定，倍增就很难直接使用。

### 5.3 查询时按二进制拆分

```cpp
for (int j = LOG; j >= 0; j--)
    if ((k >> j) & 1) {
        ans += sum[cur][j];
        cur = to[cur][j];
    }
```

这段逻辑是所有倍增查询的核心：

- 看 `k` 的二进制；
- 哪一位是 1，就跳哪一段。

### 5.4 矩阵快速幂其实也是倍增

如果你已经会写快速幂：

```cpp
while (exp) {
    if (exp & 1) res = mul(res, base);
    base = mul(base, base);
    exp >>= 1;
}
```

那你其实已经掌握了“对转移算子做倍增”的思想。

### 5.5 复杂题常常是“预处理 + 倍增 + 主算法”组合

- `P1613`：先倍增可达，再最短路；
- `P1081`：先找最近 / 次近，再倍增查询；
- `P5024`：先树上 DP，再矩阵合并处理询问。

所以不要把倍增看成孤立技巧，它很常和其它算法协作。

## 六、优化方法与解题框架

### 6.1 看到倍增优化 DP 时的思考流程

```text
读题
 ├─ 是否存在“确定的一步转移”？
 │   └─ 否：大概率不能直接倍增
 ├─ 是否需要重复做很多轮转移？
 │   └─ 是：考虑 powers-of-two 预处理
 ├─ 查询的目标是什么？
 │   ├─ 只问到哪        → to[u][j]
 │   ├─ 还问和/最值     → to + info
 │   ├─ 固定步数路径     → reach / min-plus
 │   └─ 线性递推很多轮   → 矩阵快速幂
 ├─ 信息能否稳定合并？
 │   ├─ 能：直接倍增
 │   └─ 不能：考虑重新定义状态或改用别的结构
 └─ 最后判断是否需要叠加别的算法
     ├─ 最短路
     ├─ 树剖
     ├─ 线段树
     └─ Floyd / Dijkstra
```

### 6.2 信号词 → 题型判断表

| 题面信号词 | 往往对应的模型 | 第一反应 |
|------------|----------------|----------|
| “执行 k 次操作，k 很大” | 纯倍增 / 带信息倍增 | 建 `to[][j]` |
| “每次都按确定规则走下一步” | 确定转移型 | 先找一步转移 |
| “走 2^k 步 / 每秒可选 2^k” | 固定步数转移型 | 预处理 `2^j` 可达性 |
| “线性递推，n 很大” | 矩阵快速幂 | 把状态压成向量 |
| “函数图上走很多步” | 带信息倍增 | 同时维护位置和统计量 |
| “树形 DP + 多询问” | 转移矩阵型 | 把局部 DP 写成矩阵 |

### 6.3 常见复杂度估算

| 状态形式 | 预处理复杂度 | 单次查询复杂度 | 备注 |
|----------|--------------|----------------|------|
| `to[u][j]` | `O(n log k)` | `O(log k)` | 最基础倍增 |
| `to + sum/min` | `O(n log k)` | `O(log k)` | 常见函数图 / 跳点题 |
| `reach[j][u][v]` | `O(log k · n^3)` | 取决于后续主算法 | 常配 Floyd |
| 矩阵快速幂 | `O(s^3 log k)` | `O(1)` 或 `O(s^2)` | `s` 为状态数 |
| 树上转移矩阵 | `O(n log n)` | `O(log^2 n)` 左右 | 常配树剖 |

## 七、终极总结与归纳

### 7.1 倍增优化动态规划的本质

**一句话概括：把“重复很多次的一步转移”预处理成“若干个 powers-of-two 大转移”，再用二进制拆分把它们快速拼起来。**

更进一步地说，倍增优化 DP 并不是一个孤立招式，而是一种“把转移算子压缩”的思想：

- 如果转移发生在点与点之间，就得到跳点表；
- 如果转移附带路径信息，就得到带信息倍增；
- 如果转移是线性的，就得到矩阵快速幂；
- 如果转移要在链或树上反复合并，就会走向矩阵化、算子化的 DP 结构。

### 7.2 核心要点

1. 倍增的起点永远是：先定义清楚“一步转移”。
2. 只要一步转移可以稳定拼接，通常就能递推出 `2^j` 步转移。
3. 如果除了位置还要维护信息，必须确认这些信息可以稳定合并。
4. 矩阵快速幂本质上就是“对转移矩阵做倍增”。
5. 很多倍增优化 DP 不是单独出现，而是和最短路、树剖、线段树一起出现。
6. 查询时要把目标次数按二进制拆开，这是所有倍增题的统一落点。

### 7.5 口诀

> ⏫ 一步先定清，二进制来拼。  
> 跳点看去向，信息顺手并。  
> 固定步数想 powers，线性递推看矩阵。  
> 若见重复多轮转，别再一下一下奔。  
> 倍增不是多开表，而是把转移压成层。  
> 想清“一步是什么”，整题就会顺起来。
