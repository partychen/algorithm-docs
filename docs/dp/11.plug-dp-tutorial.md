### 题目 2：插头 DP 模板（洛谷 P5056）

> 给一个有障碍的网格，要求所有可行格子恰好被一条闭合回路覆盖，求方案数。

**分析**：这是最标准的括号表示法模板题。当前格只看左插头和上插头，再分类讨论是新开一对括号、延伸、合并还是闭环。

```
状态定义：dp[pos][mask] = 扫到当前位置时，轮廓线括号状态为 mask 的方案数
转移方程：按 left / up 的 0、1、2 分类讨论插入、延伸、匹配、闭环
边界条件：初始只有空状态；障碍格必须 left = up = 0 才能通过
答案位置：最后一个可行格闭环时累加到 ans
```

**C++ 代码**：
```cpp
unordered_map<int, long long> cur, nxt;

void expand(int i, int j) {
    nxt.clear();
    for (auto [mask, ways] : cur) {
        int left = get(mask, j), up = get(mask, j + 1);
        if (blocked[i][j]) {
            if (!left && !up) push(nxt, clear_slot(mask, j), ways);
            continue;
        }
        // 新建一对括号、延伸、merge_left、merge_right、闭环特判
    }
    cur.swap(nxt);
}
```

**核心启示**：
- **P5056** 几乎就是整个插头 DP 的“语法课”。
- **左 / 上两个插头** 是所有分类讨论的入口。
- **最后一个合法闭环点** 往往要单独特判，不能让局部小环提前闭死。

### 🟡 进阶级 Level 2：障碍、整图覆盖与高精度计数

### 题目 3：Eat the Trees（HDU 1693）

> 有障碍的棋盘上，要用若干不相交回路覆盖所有树格，问方案数。

**分析**：和模板题很像，但它通常允许**多个不相交回路**，这一点和“恰好一条闭合回路”的 P5056 不同。对障碍格，插头必须全清空；对树格，则要求它被合法覆盖，但局部闭环未必一定非法，关键看题目是否要求唯一回路。

```
状态定义：dp[pos][mask] = 扫到当前位置时，轮廓线状态为 mask 的方案数
转移方程：障碍格只允许空过；树格必须在“不放 / 放边”中选合法方案
边界条件：初始空状态；所有树格都要被有效覆盖
答案位置：处理完整张图后的空状态方案数
```

**C++ 代码**：
```cpp
for (auto [mask, ways] : cur) {
    int left = get(mask, j), up = get(mask, j + 1);
    if (!tree[i][j]) {
        if (!left && !up) push(nxt, clear_slot(mask, j), ways);
        continue;
    }
    if (!left && !up) push(nxt, put_pair(mask, j), ways);
    if (left && !up) push(nxt, extend(mask, j, RIGHT), ways), push(nxt, extend(mask, j, DOWN), ways);
    if (!left && up) push(nxt, extend(mask, j, RIGHT), ways), push(nxt, extend(mask, j, DOWN), ways);
    if (left == 2 && up == 1) push(nxt, clear_slot(mask, j), ways);  // 允许在当前格闭掉一个局部回路
    if (left == 1 && up == 2) push(nxt, clear_slot(mask, j), ways);  // 是否允许取决于题目的“多环 / 单环”口径
    if (left == 1 && up == 1) push(nxt, merge_right(mask, j), ways);
    if (left == 2 && up == 2) push(nxt, merge_left(mask, j), ways);
}
```

**核心启示**：
- **障碍格** 会极大改变状态合法性。
- **“当前格必须被覆盖”** 和 **“当前格可以空着”** 是两类完全不同的题。
- **“单环模板”和“多环覆盖”** 的核心差别，正是局部闭环能不能提前出现。

### 题目 4：邮递员（洛谷 P2289 / HNOI2004）

> 在 `m×n` 网格点上，从起点出发走遍所有邮筒且恰好一次，最后回到起点，问最短回路数量。

**分析**：这题本质是整图哈密顿回路计数。状态仍然是单回路插头 DP，只是答案会很大，通常需要高精度或自定义大整数。

```
状态定义：dp[pos][mask] = 扫到当前位置时，轮廓线括号状态为 mask 的方案数（高精度）
转移方程：按照单回路模型处理 left / up 的组合，保证每个点度数为 2
边界条件：初始空状态；整张图必须形成唯一回路
答案位置：在最后一个合法闭环点把方案数累加到 ans，再按题目计数口径输出
```

**C++ 代码**：
```cpp
unordered_map<int, BigInt> cur, nxt;

for (auto [mask, ways] : cur) {
    int left = get(mask, j), up = get(mask, j + 1);
    if (!left && !up) push(nxt, put_pair(mask, j), ways);
    if (left && !up) push(nxt, extend(mask, j, RIGHT), ways), push(nxt, extend(mask, j, DOWN), ways);
    if (!left && up) push(nxt, extend(mask, j, RIGHT), ways), push(nxt, extend(mask, j, DOWN), ways);
    if (left == 1 && up == 2 && clear_slot(mask, j) == 0) ans += ways;
    if (left == 1 && up == 1) push(nxt, merge_right(mask, j), ways);
    if (left == 2 && up == 2) push(nxt, merge_left(mask, j), ways);
}
```

**核心启示**：
- **单回路插头 DP** 可以自然处理哈密顿回路。
- **大整数** 有时不是附属，而是主实现难点之一。
- **“所有点度数为 2”** 是把题面翻译成插头转移的关键；若题目把顺逆时针视为不同，再按题意后处理。

### 🟠 提升级 Level 3：从单回路到单路径，再到附加限制

### 题目 5：Formula 1（Ural 1519）

> 在有障碍的网格中，从起点走到终点，要求经过所有可行格恰好一次，问哈密顿路径数量。

**分析**：和闭环题最大的不同是：起点和终点的度数应为 `1`，其他可行格的度数应为 `2`。所以它是“单路径插头 DP”，不是单回路。

```
状态定义：dp[pos][mask] = 扫到当前位置时，轮廓线括号状态为 mask 的方案数
转移方程：端点格允许度数为 1；若该格是可行普通格，则最终度数必须为 2；障碍格度数为 0
边界条件：起点、终点的度数规则单独处理；障碍格仍要求无插头穿过
答案位置：处理完整张图后的空状态方案数
```

**C++ 代码**：
```cpp
bool endpoint = (i == sx && j == sy) || (i == tx && j == ty);
int left = get(mask, j), up = get(mask, j + 1);

if (blocked[i][j]) {
    if (!left && !up) push(nxt, clear_slot(mask, j), ways);
} else if (endpoint) {
    if (!left && up) push(nxt, extend(mask, j, RIGHT), ways);
    if (left && !up) push(nxt, extend(mask, j, DOWN), ways);
    if (!left && !up) push(nxt, start_one(mask, j), ways);
} else {
    // 普通格走单回路 / 单路径的标准四类转移
}
```

**核心启示**：
- **起点终点的度数差异**，就是“回路题”变“路径题”的根本。
- **端点特判** 必须写进当前格转移，不能最后补。
- **同一套括号表示法** 可以同时处理回路与路径。

### 题目 6：俄罗斯方块（洛谷 P5347 / XR-1）

> 在有障碍的网格上画若干线段，统计本质不同的方案数；还可能带颜色与旋转等价判定。

**分析**：这题的结构仍然是轮廓线推进，但状态里除了连通关系，还要放颜色或对称规范化结果。属于“插头 DP + 额外维度”的综合题。

```
状态定义：dp[pos][mask][sig] = 扫到当前位置时，轮廓线结构为 mask、附加签名为 sig 的方案数
转移方程：枚举当前格是否画线、线的方向、颜色及其规范化表示
边界条件：障碍格不能穿线；若题目把旋转视为等价，则状态或答案要做规范化
答案位置：处理完整张图后的合法终态方案数
```

**C++ 代码**：
```cpp
long long pack(int mask, int sig) {
    return (1LL * sig << 32) | (unsigned)mask;
}

void push(int mask, int sig, int ways) {
    sig = normalize_sig(sig);
    long long key = pack(mask, sig);
    nxt[key] = (nxt[key] + ways) % MOD;
}

for (auto [key, ways] : cur) {
    int mask = (int)key;
    int sig = (int)(key >> 32);
    // 枚举当前格画横线 / 竖线 / 不画，并维护 sig
}
```

**核心启示**：
- **轮廓线负责结构，sig 负责附加限制**，分工要明确。
- **规范化** 是这种题的生命线，否则同一个方案会被重复算很多次。
- **P5347** 很适合训练“结构状态 + 额外状态”的打包能力。

### 🔴 困难级 Level 4：多连通块与高维状态爆炸

### 题目 7：Black and White（UVA 10572）

> 给棋盘染黑白，要求黑块与白块都连通，还要满足局部 `2×2` 约束，问方案数。

**分析**：这题已经不能只用括号表示，因为同时存在两种颜色、多个连通块。最常见做法是轮廓线 + 最小表示法：状态记录每个位置当前属于哪个黑 / 白连通块，再做重编号。

```
状态定义：dp[pos][state] = 扫到当前位置时，轮廓线上黑白连通块状态为 state 的方案数
转移方程：枚举当前格染黑或白，合并相邻同色连通块，并检查 2×2 约束
边界条件：状态需最小表示；最后黑白各自恰为一个连通块
答案位置：处理完整张图后的合法终态方案数
```

**C++ 代码**：
```cpp
vector<int> st = decode(mask);
for (int color = 0; color < 2; color++) {
    if (!valid_2x2(i, j, color)) continue;
    apply_color(st, j, color);
    merge_same_color(st, j, j + 1);
    int next_mask = encode(st);
    push(nxt, next_mask, ways);
}
```

**核心启示**：
- **最小表示法** 是从“单回路”迈向“多连通块”的关键升级。
- **结构合法性 + 局部约束** 常常要同时检查。
- **这类题最难的不是 DP 名字，而是状态设计。**

### 题目 8：喵星人的入侵（洛谷 P2337 / SCOI2012）

> 网格中有障碍、起终点、可放炮台的位置以及最多 `K` 个炮台限制，要让路径存在并尽量提高伤害。

**分析**：这是非常典型的“轮廓线 + 额外维度 + 最值 DP”综合题。结构状态要维护路径连通，额外维度要维护已放炮台数和当前价值，分类讨论极多，但核心框架仍然是当前格只看左口和上口。

```
状态定义：dp[pos][mask][k] = 扫到当前位置时，轮廓线状态为 mask、已用炮台数为 k 的最大收益
转移方程：枚举当前格作为障碍 / 路径 / 炮台时的合法转移，并更新收益
边界条件：k 不能超过 K；起点终点、障碍格、路径度数都要单独合法性检查
答案位置：所有合法终态中的最大收益
```

**C++ 代码**：
```cpp
void push(int mask, int used, int val) {
    if (used > K) return;
    long long key = (1LL * used << 32) | (unsigned)mask;
    nxt[key] = max(nxt[key], val);
}

for (auto [key, val] : cur) {
    int used = (int)(key >> 32);
    int mask = (int)key;
    int left = get(mask, j), up = get(mask, j + 1);
    // 当前格作为障碍 / 路径 / 炮台 的分类转移
}
```

**核心启示**：
- **P2337** 体现了插头 DP 的“状态工程学”本质。
- **路径结构** 和 **收益维度** 可以并存，但必须层次分清。
- **哈希 + 滚动数组 + 强合法性判断** 是这类题的基本配置。

## 五、常见技巧与思路总结

### 5.1 当前格永远只看左口和上口

```cpp
int left = get(mask, j, BITS);
int up = get(mask, j + 1, BITS);
```

不要一上来把整个轮廓线都脑补成一团。绝大多数转移，其实只围绕这两个口展开。

### 5.2 行末平移是 broken profile 的固定动作

```cpp
unordered_map<int, long long> shifted;
for (auto [mask, ways] : cur) push(shifted, (mask << 1) & limit_mask, ways);
cur.swap(shifted);
```

如果你用的是“`m+1` 槽位”写法，那么走完一行后通常都要整体平移一格，让下一行继续复用同一套下标语义。

### 5.3 括号匹配一定要写成独立函数

```cpp
int match_left(int mask, int p);
int match_right(int mask, int p, int m);
```

`(1,1)`、`(2,2)` 的合并最容易写挂。把匹配逻辑拆成函数，比把它们糊在转移里安全得多。

### 5.4 最小表示法的本质是“去同构”

```cpp
for (int& x : st) {
    if (x && !id[x]) id[x] = ++tot;
    if (x) x = id[x];
}
```

编号本身没有意义，关系才有意义。任何时候只要出现“连通块标号”，就要警惕等价状态重复。

### 5.5 哈希存状态，滚动存层数

```cpp
unordered_map<long long, long long> cur, nxt;

nxt.clear();
for (auto [key, val] : cur) {
    // 根据当前格转移到 next_key
    push(nxt, next_key, next_val);
}
cur.swap(nxt);
```

插头 DP 很少直接开满数组，因为合法状态远少于理论状态。真正需要优化的，是**活跃状态集合**。

### 5.6 先判断“当前格可不可以空着”

| 题型 | 当前格能否空着 |
| --- | --- |
| 骨牌覆盖 | 不一定，取决于是否最终能铺满 |
| 单回路覆盖 | 通常不能漏掉要求覆盖的格子 |
| 路径题 | 非路径格可以空，路径格必须满足度数限制 |
| 混合最值题 | 由题目角色（障碍 / 路径 / 炮台）决定 |

### 6.2 常见信号词 → 题型映射表

| 信号词 | 往往意味着什么 |
| --- | --- |
| “逐格扫描网格” | 可以尝试轮廓线 DP |

### 6.3 轮廓线 DP 的 checklist

1. 先决定轮廓线有多少槽位，通常是 `m+1`。
2. 写清楚每个槽位的编码意义：0/1，还是 0/1/2，还是连通块编号。
3. 明确当前格能否空、能否放、放完后是否要留下右 / 下插头。
4. 如果有连通块，决定用括号表示法还是最小表示法。
5. 最后再考虑哈希、滚动数组和附加维度压缩。

## 七、终极总结与归纳

### 7.1 轮廓线动态规划的本质

**一句话本质**：把整张网格的历史压缩成“当前边界上还没处理完的信息”，然后逐格推进。

展开来说，轮廓线 DP 并不是神秘魔法，它只是在提醒你：对很多网格题而言，真正影响未来的不是整块前缀，而只是那条分界线。如果问题只问覆盖，边界上记 0/1 就够；如果问题问连通性，就要记插头如何匹配；如果问题还带颜色、数量、收益，就再把这些维度绑在轮廓线上一起维护。

### 7.2 核心要点

1. **先想轮廓线，再想格子**；当前格只看左口和上口。
2. **覆盖型与连通型要区分开**；不是所有轮廓线 DP 都是括号插头。
3. **单回路 / 单路径** 常用括号表示法，多连通块常用最小表示法。
4. **状态规范化** 是高阶题是否可做的分水岭。
5. **哈希表只存活跃状态**，几乎是插头 DP 的标配。
6. **大题往往是“结构状态 + 额外维度”一起维护**。
