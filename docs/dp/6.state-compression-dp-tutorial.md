### 题目 2：互不侵犯（洛谷 P1896）

> 在 `N × N` 棋盘上放 `K` 个国王，任意两个国王不能互相攻击，求方案数。

**分析**：和玉米田类似，但这里不仅有“上下同列冲突”，还有“斜对角冲突”。此外需要额外记录已经放了多少个国王，因此再加一维计数。

```
状态定义：实现时常写 dp[i][id][k]，表示前 i 行处理完，第 i 行采用第 id 个合法状态，共放了 k 个国王
转移方程：若 s = st[id]、t = st[pre] 兼容，则 `dp[i][id][k] += dp[i-1][pre][k-cnt[id]]`
边界条件：dp[0][0][0] = 1
答案位置：sum(dp[n][id][K])
```

**C++ 代码**：
```cpp
vector<int> st, cnt;
long long dp[10][1 << 9][82];

for (int s = 0; s < (1 << n); s++)
    if (!(s & (s << 1))) {
        st.push_back(s);
        cnt.push_back(__builtin_popcount(s));
    }
dp[0][0][0] = 1;
for (int i = 1; i <= n; i++)
    for (int a = 0; a < (int)st.size(); a++)
        for (int b = 0; b < (int)st.size(); b++) {
            int s = st[a], t = st[b];
            if (s & t) continue;
            if ((s << 1) & t) continue;
            if ((s >> 1) & t) continue;
            for (int k = cnt[a]; k <= K; k++)
                dp[i][a][k] += dp[i - 1][b][k - cnt[a]];
        }

```

这里 `cnt[a]` 表示合法状态 `st[a]` 中国王个数；代码里用的是“合法状态编号”，不是直接拿 bitmask 当数组下标。

**核心启示**：
- **状态上再挂一个计数维**，是棋盘状压里最常见的扩展。
- **斜向限制** 往往就是左右移一位再与。
- **`popcount` 预处理** 很常用，别在循环里反复算。

### 题目 3：蒙德里安的梦想（AcWing 291 / POJ 2411）

> 用 `1×2` 的骨牌覆盖 `n×m` 棋盘，问完全覆盖的方案数。

**分析**：这是“按列/按行轮廓推进”的基础题。转移时常把“当前列遗留到下一列的凸出格子”作为状态，关键不是冲突，而是空位能否被横放骨牌恰好补满。

```
状态定义：dp[i][s] = 前 i 列处理完，伸向下一列的轮廓状态为 s 的方案数
转移方程：dp[i][s] += dp[i-1][t]，其中 s 与 t 互补后可被横牌填满
边界条件：dp[0][0] = 1
答案位置：dp[m][0]
```

**C++ 代码**：
```cpp
bool full[1 << 11];
long long dp[12][1 << 11];

dp[0][0] = 1;
for (int s = 0; s < (1 << n); s++) {
    int cnt = 0; full[s] = true;
    for (int i = 0; i < n; i++) {
        if (s >> i & 1) {
            if (cnt & 1) full[s] = false;
            cnt = 0;
        } else cnt++;
    }
    if (cnt & 1) full[s] = false;
}
for (int i = 1; i <= m; i++)
    for (int s = 0; s < (1 << n); s++)
        for (int t = 0; t < (1 << n); t++)
            if (!(s & t) && full[s | t])
                dp[i][s] += dp[i - 1][t];

```

**核心启示**：
- **状压不只用于“摆点”**，也可以用于表示“未完成轮廓”。
- **合法性可以预处理成一个布尔数组**。
- **完全覆盖题** 经常是“当前状态 + 上一状态互补”。

### 题目 4：炮兵阵地（洛谷 P2704）

> 地图上有平原和山地，炮兵只能放在平原。炮兵会攻击同一行左右两格、同一列上下两格，问最多能放多少炮兵。

**分析**：行内限制从“不能相邻”升级成“距离 1 和 2 都不行”，且纵向会影响到前两行，所以 DP 需要记两层历史：当前行、上一行、上上行。

```
状态定义：实现时常写 dp[i][id1][id2]，表示处理到第 i 行，当前行采用第 id1 个合法状态、上一行采用第 id2 个合法状态
转移方程：若 `s = st[id1]`、`t = st[id2]`、`u = st[id3]` 彼此兼容，则 `dp[i][id1][id2] = max(dp[i][id1][id2], dp[i-1][id2][id3] + cnt[id1])`
边界条件：dp[0][0][0] = 0，其余为 `-INF`
答案位置：max(dp[n][id1][id2])
```

**C++ 代码**：
```cpp
vector<int> st, cnt;
const int NEG_INF = -1e9;
int dp[105][70][70];

for (int s = 0; s < (1 << m); s++)
    if (!(s & (s << 1)) && !(s & (s << 2))) {
        st.push_back(s);
        cnt.push_back(__builtin_popcount(s));
    }
for (int i = 0; i <= n; i++)
    for (int a = 0; a < 70; a++)
        for (int b = 0; b < 70; b++)
            dp[i][a][b] = NEG_INF;
dp[0][0][0] = 0;

for (int i = 1; i <= n; i++)
    for (int a = 0; a < (int)st.size(); a++) {
        int s = st[a];
        if (s & ban[i]) continue;
        for (int b = 0; b < (int)st.size(); b++) {
            int t = st[b];
            if (s & t) continue;
            for (int c = 0; c < (int)st.size(); c++) {
                int u = st[c];
                if (s & u) continue;
                if (dp[i - 1][b][c] == NEG_INF) continue;
                dp[i][a][b] = max(dp[i][a][b], dp[i - 1][b][c] + cnt[a]);
            }
        }
    }
// ban[i] 为山地掩码
```

**核心启示**：
- **影响范围多一行**，就多记一层历史。
- **最大化问题** 很适合把不合法状态初始化为 `-INF`。
- **合法状态个数远小于 `2^m`**，预处理后复杂度会显著下降。

### 题目 5：旅行商问题 TSP（经典模型）

> 给定 `n` 个点和两两距离，从 0 号点出发，经过每个点恰好一次后回到起点，求最小总代价。

**分析**：这题不是棋盘，而是纯集合型状压的代表。`mask` 表示已经访问过哪些点，再用一个末尾点 `i` 表示路径最后停在哪儿。

```
状态定义：dp[S][i] = 经过点集 S，且最后停在 i 的最小代价
转移方程：dp[S∪{j}][j] = min(dp[S][i] + w[i][j])
边界条件：dp[1<<0][0] = 0
答案位置：min(dp[(1<<n)-1][i] + w[i][0])
```

**C++ 代码**：
```cpp
const int INF = 0x3f3f3f3f;
int dp[1 << 16][16];

memset(dp, 0x3f, sizeof(dp));
dp[1][0] = 0;
for (int S = 1; S < (1 << n); S++)
    for (int i = 0; i < n; i++) {
        if (!(S >> i & 1)) continue;
        if (dp[S][i] == INF) continue;
        for (int j = 0; j < n; j++) {
            if (S >> j & 1) continue;
            int T = S | (1 << j);
            dp[T][j] = min(dp[T][j], dp[S][i] + w[i][j]);
        }
    }

```

**核心启示**：
- **`dp[mask][last]`** 是集合型最重要的模板之一。
- **集合里带一个“末尾”维度**，本质是把路径信息补回来。
- **状压 DP ≠ 棋盘专属**。

### 题目 6：No Change（洛谷 P3092）

> 有 `K` 枚硬币，每枚硬币可从当前未买位置起连续购买若干件物品，不能找零。问能否买完所有物品；若能，剩余硬币总面值最大是多少。

**分析**：`K ≤ 16`，自然要记录“已经用过哪些硬币”。关键是 `dp[S]` 不存价值，而存“用掉硬币集合 `S` 后，最多买到了第几个物品”。这样状态既单调，也便于继续扩展。

```
状态定义：dp[S] = 使用硬币集合 S 后，最多能买完前多少件物品
转移方程：dp[S] = max(dp[S\{j}] 经第 j 枚硬币扩展后的最远位置)
边界条件：dp[0] = 0
答案位置：所有 dp[S] ≥ n 的状态中，未使用硬币总和最大值
```

**C++ 代码**：
```cpp
int dp[1 << 16];
fill(dp, dp + (1 << k), -1);
dp[0] = 0;
for (int S = 1; S < (1 << k); S++)
    for (int j = 0; j < k; j++) {
        if (!(S >> j & 1)) continue;
        int p = dp[S ^ (1 << j)];
        if (p < 0) continue;
        int q = upper_bound(pre, pre + n + 1, pre[p] + coin[j]) - pre - 1;
        dp[S] = max(dp[S], q);
    }
// 枚举 dp[S] >= n 的状态，统计未使用硬币和的最大值
```

**核心启示**：
- **状态里存“最远能做到哪”**，比直接存花费更顺。
- **二分 + 状压** 很常见。
- **设计状态时要让转移尽量单调**。

### 题目 7：邦邦的大合唱站队（洛谷 P3694）

> 有 `M` 个乐队、`N` 个人，想通过“出列再插入”让同乐队的人连续站在一起，问最少需要出列多少人。

**分析**：本质是决定“各个乐队块的排列顺序”。`S` 表示已经安排了哪些乐队，`dp[S]` 表示这些乐队按某个顺序放在前缀里时，最多有多少人能留在原位。

```
状态定义：dp[S] = 已安排集合 S 中所有乐队时，最多有多少人不用动
转移方程：dp[S∪{j}] = max(dp[S] + 乐队 j 放在当前位置能贡献的原地人数)
边界条件：dp[0] = 0
答案位置：N - dp[(1<<M)-1]
```

**C++ 代码**：
```cpp
int dp[1 << 20], sum[1 << 20];

for (int S = 1; S < (1 << m); S++) {
    int b = __builtin_ctz(S);
    sum[S] = sum[S ^ (1 << b)] + sz[b];
}
for (int S = 0; S < (1 << m); S++)
    for (int j = 0; j < m; j++) {
        if (S >> j & 1) continue;
        int T = S | (1 << j);
        int l = sum[S], r = sum[T];
        dp[T] = max(dp[T], dp[S] + cnt[j][r] - cnt[j][l]);
    }
// cnt[j][i] 为前缀里乐队 j 的人数
```

**核心启示**：
- **状压可以枚举“分组顺序”**。
- **前缀和负责快速算贡献，状压负责枚举顺序。**
- **有时答案不是 `dp[full]`，而是总量减去 `dp[full]`。**

### 题目 8：学校食堂（洛谷 P2157）

> 每位同学最多只能容忍后面 `b_i` 个人先打饭，且 `b_i ≤ 7`。相邻两次打饭的代价与两人的口味异或有关，求最小总代价。

**分析**：虽然总人数可很大，但“插队影响”只在未来 7 个人之内，因此可以把窗口 `[i, i+7]` 的处理状态压成 8 位二进制。这类题的本质不是整段压，而是**局部未来压缩**。

```
状态定义：dp[i][S][last] = 当前最前未打饭者是 i，窗口状态为 S，上一个打饭者相对 i 的偏移为 last 的最小代价
转移方程：若 S 最低位为 1，则右移窗口；否则枚举当前窗口内此刻允许先打饭的人
边界条件：dp[0][0][NONE] = 0，其中 NONE 表示“还没有上一个人”；窗口右移后对 last 的维护交给辅助函数
答案位置：min(dp[n][0][*])
```

**C++ 代码**：
```cpp
const int INF = 0x3f3f3f3f;
const int NONE = 16;              // 0..15 表示偏移 -8..7，16 表示不存在前驱
int dp[1005][1 << 8][17];

int shift_last(int last) {
    if (last == NONE || last == 0) return NONE;
    // 统一处理窗口右移后的前驱编码
    return last - 1;
}

memset(dp, 0x3f, sizeof(dp));
dp[0][0][NONE] = 0;
for (int i = 0; i < n; i++)
    for (int S = 0; S < (1 << 8); S++)
        for (int last = 0; last < 17; last++) {
            if (dp[i][S][last] == INF) continue;
            if (S & 1) {
                int next_last = shift_last(last);
                dp[i + 1][S >> 1][next_last] = min(dp[i + 1][S >> 1][next_last], dp[i][S][last]);
                continue;
            }
            for (int k = 0; k < 8 && i + k < n; k++) {
                if (S >> k & 1) continue;
                if (!can_choose(i, S, k)) continue;      // 结合 b_i 判断当前能否让第 k 个人先打饭
                int add = (last == NONE ? 0 : calc_cost(last - 8, i + k));
                dp[i][S | (1 << k)][k + 8] = min(dp[i][S | (1 << k)][k + 8], dp[i][S][last] + add);
            }
        }
// can_choose 负责忍耐度合法性判断
```

**核心启示**：
- **窗口宽度小** 是这类题能状压的根本原因。
- **当前位置向前推进时，状态整体右移** 是局部窗口题的典型动作。
- **多出来的维度往往记录“最近一次选择”**。

### 题目 9：Bill 的挑战（洛谷 P2167）

> 给若干个带 `?` 的模式串，问恰好匹配其中 `K` 个模式串的字符串个数。

**分析**：这题最核心的不是普通 DP，而是**对子集做计数，再通过容斥还原答案**。`S` 表示“要求同时匹配的模式串集合”，先算 `f(S)`，再按组合系数恢复“恰好匹配 `K` 个”的数量。

```
状态定义：f(S) = 同时满足集合 S 内所有模式串的字符串数
转移方程：逐位统计该位可填字符数并相乘
边界条件：单点和空集按自然定义处理
答案位置：sum((-1)^{|S|-K} * C(|S|,K) * f(S))
```

**C++ 代码**：
```cpp
long long ans = 0;
for (int S = 0; S < (1 << n); S++) {
    int c = __builtin_popcount(S);
    if (c < K) continue;
    long long ways = 1;
    for (int p = 0; p < len; p++) {
        int ch = -1; bool ok = true;
        for (int i = 0; i < n; i++) {
            if (!(S >> i & 1) || pat[i][p] == '?') continue;
            if (ch == -1) ch = pat[i][p];
            else if (ch != pat[i][p]) ok = false;
        }
        if (!ok) ways = 0;
        else if (ch == -1) ways = ways * 26 % MOD;
    }
    long long term = ways * C[c][K] % MOD;
    if ((c - K) & 1) ans = (ans - term + MOD) % MOD;
    else ans = (ans + term) % MOD;
}

```

**核心启示**：
- **状压不一定非得有“逐步 DP”**，对子集做统一计数也很常见。
- **“恰好匹配 K 个”** 这种话术往往在暗示容斥。
- **子集枚举 + 组合数** 是很多高级状压题的固定搭档。

### 题目 10：寿司晚宴（洛谷 P2150）

> 两个人分别从 `2..n` 中挑一些数，要求任意一对分别属于两人的数都互质，求方案数。

**分析**：这题很经典，难点在于“互质限制”是质因子层面的。把小质数（如前 8 个质数）压成 `mask`，`dp[a][b]` 记录两人各自占用了哪些小质数；大质因子则按分组处理，避免互相污染。

```
状态定义：dp[s1][s2] = 处理完若干数字后，A 占小质数集合 s1，B 占小质数集合 s2 的方案数
转移方程：同一大质因子分组内，整组只向 A 转、向 B 转或都不取
边界条件：dp[0][0] = 1
答案位置：sum(dp[s1][s2])，其中 s1 & s2 = 0
```

**C++ 代码**：
```cpp
int dp[1 << 8][1 << 8], f[1 << 8][1 << 8], g[1 << 8][1 << 8];

dp[0][0] = 1;
for (auto grp : groups) {
    memcpy(f, dp, sizeof(dp));
    memcpy(g, dp, sizeof(dp));
    for (auto x : grp) {
        int m = small_mask[x];
        for (int a = 0; a < (1 << 8); a++)
            for (int b = 0; b < (1 << 8); b++) {
                if (a & b) continue;
                if (!(m & b)) f[a | m][b] = (f[a | m][b] + dp[a][b]) % MOD;
                if (!(m & a)) g[a][b | m] = (g[a][b | m] + dp[a][b]) % MOD;
            }
    }
    for (int a = 0; a < (1 << 8); a++)
        for (int b = 0; b < (1 << 8); b++)
            dp[a][b] = (f[a][b] + g[a][b] - dp[a][b] + MOD) % MOD;
}
// groups 按大质因子分组
```

**核心启示**：
- **真正需要压缩的是“限制来源”而不是数字本身。**
- **分组处理大质因子** 是把不可压的信息拆开的关键。
- **双集合状压** 常写成 `dp[a][b]`。

### 题目 11：一双木棋（洛谷 P4363）

> 两人轮流在一个满足前置关系的棋盘上落子。先手想最大化 `A` 分之和减去后手 `B` 分之和，后手想最小化该值。

**分析**：这是一个轮廓线 + 极大极小搜索的综合题。更统一的说法是：记录**每一行当前已经填了多少格**，于是得到一组单调不增的行前缀长度；把这一组长度按 `(m+1)` 进制压起来，就是整个局面。轮到谁走由已放置格子数的奇偶决定。

```
状态定义：f(state) = 从当前轮廓 state 出发，最终先手分 - 后手分 的最优值
转移方程：先手取 max，后手取 min；每次选择一个当前可落子位置进入新状态
边界条件：棋盘填满时 f(state) = 0
答案位置：f(初始状态)
```

**C++ 代码**：
```cpp
unordered_map<long long, int> memo;
int row_len[10];

long long encode() {
    long long s = 0;
    for (int i = 0; i < n; i++) s = s * (m + 1) + row_len[i];
    return s;
}
int dfs() {
    long long s = encode();
    if (memo.count(s)) return memo[s];
    int used = 0;
    for (int i = 0; i < n; i++) used += row_len[i];
    if (used == n * m) return memo[s] = 0;
    int best = (used & 1) ? INT_MAX : INT_MIN;
    for (int i = 0; i < n; i++) {
        if (row_len[i] == m) continue;
        if (i && row_len[i] >= row_len[i - 1]) continue;
        int j = row_len[i];
        row_len[i]++;
        int v = dfs();
        row_len[i]--;
        if (used & 1) best = min(best, v - B[i][j]);
        else best = max(best, v + A[i][j]);
    }
    return memo[s] = best;
}
// 初始化 row_len[i]=0
```

**核心启示**：
- **广义状压** 可以不是二进制，而是“一组行前缀长度”的多进制编码。
- **博弈 + 状压** 经常写成记忆化搜索。
- **单调性** 能让状态数量从指数爆炸降到组合数量级。

### 题目 12：中国象棋 - 摆上马（洛谷 P5005）

> 在 `X × Y` 的棋盘上任意摆放中国象棋中的“马”，要求任意两匹马都不能互相攻击，问方案数。

**分析**：`Y ≤ 6`，所以一行仍能状压；但中国象棋的马有“蹩马腿”规则，因此当前行和前两行之间的兼容关系比炮兵阵地更复杂。整体流程仍然是：预处理所有行状态，再预处理三行之间的兼容关系。

```
状态定义：dp[i][s][t] = 处理到第 i 行，第 i 行位状态为 s，第 i-1 行位状态为 t 的合法方案数
转移方程：dp[i][s][t] += dp[i-1][t][u]，其中 (s,t,u) 三者整体合法
边界条件：可统一写成 `dp[0][0][0] = 1`，其余状态从这一空局面推出来
答案位置：sum(dp[x][s][t])
```

**C++ 代码**：
```cpp
long long dp[105][1 << 6][1 << 6];
dp[0][0][0] = 1;

for (int i = 1; i <= x; i++)
    for (int s = 0; s < (1 << y); s++)
        for (int t = 0; t < (1 << y); t++) {
            if (!ok1[s][t]) continue;
            for (int u = 0; u < (1 << y); u++) {
                if (!ok1[t][u]) continue;
                if (!ok2[s][u]) continue;
                dp[i][s][t] = (dp[i][s][t] + dp[i - 1][t][u]) % MOD;
            }
        }
// ok1 / ok2 分别表示相邻两行、隔一行的攻击兼容性
```

**核心启示**：
- **先想“影响到几行”**，再决定要记几层历史。
- **复杂攻击规则也能预处理成兼容表。**
- **宽度小、长度大** 的棋盘题，几乎天然适合按行状压。

### 题目 13：Scarlet 的字符串不可能这么可爱（洛谷 P4925）

> 用 `k` 种字符构造长度为 `L` 的字符串，要求不能出现形如 `aa` 或 `aba` 的回文连续子串，部分数据还固定某一位字符，求方案数。

**分析**：这题很适合作为“反向思考”的例子。最直观的做法是把“最后两位是什么”压成小状态做 DP；但进一步观察会发现，从第三位开始，每一位只要不等于前两位即可，状态直接退化成公式。**状压是起点，不一定是终点。**

```
状态定义：最初可设 dp[i][x][y] 表示长度 i、最后两位为 x,y 的方案数
转移方程：枚举新字符 z，满足 z != x 且 z != y
边界条件：长度 1、2 特判
答案位置：进一步化简后直接变成公式
```

**C++ 代码**：
```cpp
long long qpow(long long a, long long b, long long mod) {
    long long r = 1 % mod;
    while (b) {
        if (b & 1) r = r * a % mod;
        a = a * a % mod;
        b >>= 1;
    }
    return r;
}
// 若无固定位置：ans = k * (k - 1) % mod * qpow(k - 2, L - 2, mod) % mod
// 若固定一位：ans = (k - 1) * qpow(k - 2, L - 2, mod) % mod
```

**核心启示**：
- **先想状态，再看能不能消掉状态。**
- **有些题是“状压思维”而不是“状压 DP”本体。**
- **发现转移稳定后，要敢于推公式。**

### 题目 14：yyy loves Maths VII（洛谷 P2396）

> 给定 `n ≤ 20` 个数和若干个“禁止和”，统计和不落在禁止集合中的非空子集个数。

**分析**：这题甚至不需要传统 DP，单纯枚举所有子集就够了。但它非常适合拿来提醒：只要 `n ≤ 20`，**先把子集视角建立起来**，再决定要不要 DP。很多时候，状压枚举本身已经足够强。

```
状态定义：sum[S] = 子集 S 的元素和
转移方程：sum[S] = sum[S-lowbit] + a[pos(lowbit)]
边界条件：sum[0] = 0
答案位置：统计所有非空且 sum[S] 不在禁集中的 S
```

**C++ 代码**：
```cpp
long long sum[1 << 20], ans = 0;
for (int S = 1; S < (1 << n); S++) {
    int b = __builtin_ctz(S);
    sum[S] = sum[S ^ (1 << b)] + a[b];
    if (!ban.count(sum[S])) ans++;
}
// ban 为禁止和集合
```

**核心启示**：
- **状压枚举** 是很多高级算法的前置能力。
- **不是所有 `2^n` 题都需要写 `dp`**。
- **先想“子集怎么维护”再想“值怎么转移”**，思路会更清楚。

## 五、常见技巧与思路总结

### 5.1 先预处理合法状态，再做 DP

很多棋盘题里，合法状态数量远少于 `2^m`。比如 `m=10` 时，所有状态有 1024 个，但满足“不相邻”“不隔一位相邻”的合法状态往往只有几十个。

```cpp
for (int s = 0; s < (1 << m); s++)
    if (!(s & (s << 1)) && !(s & (s << 2))) st.push_back(s);
```

### 5.2 预处理兼容表，减少转移常数

不要在三重循环里每次都判断 `s & t`。可以把兼容关系提前存成邻接表。

```cpp
for (int i = 0; i < V; i++)
    for (int j = 0; j < V; j++)
        if (!(st[i] & st[j])) to[i].push_back(j);
```

### 5.3 子集枚举要记住这句模板

```cpp
for (int sub = S; ; sub = (sub - 1) & S) {
    // 处理 sub
    if (sub == 0) break;
}
```

它会在 `O(子集数)` 的复杂度内遍历 `S` 的所有子集，是子集 DP、容斥、集合划分题的核心工具。

### 5.4 滚动数组与层数压缩

若转移只依赖上一层或前两层，就没必要保留全部历史。

```cpp
int cur = i & 1, pre = cur ^ 1;
memset(dp[cur], 0, sizeof(dp[cur]));
// 然后只从 dp[pre] 向 dp[cur] 转移
```

### 5.5 判断“应该压哪一层信息”

- 行状态题：压**当前行**。
- TSP 类：压**已访问点集**。
- 局部窗口题：压**未来几个人/几位的占用情况**。
- 质因子题：压**会造成冲突的小质因子集合**。
- 轮廓线题：压**每列当前高度/边界形状**。

### 5.6 复杂题里，状压常常只是外壳

真正的难点往往在：
- 如何把题目限制翻译成兼容关系；
- 如何把不可压的部分剥离出去；
- 如何让状态定义支持单调转移。

## 六、优化方法与解题框架

### 6.1 看到状压题时的思考流程

```text
读题
 ├─ 是否有“小而固定”的维度？(n≤20 / m≤10 / b_i≤7)
 │   └─ 是：考虑压成 mask
 ├─ 状态代表什么？
 │   ├─ 集合     → dp[mask]
 │   ├─ 路径末尾 → dp[mask][last]
 │   ├─ 当前一行 → dp[row][mask]
 │   ├─ 两行历史 → dp[row][a][b]
 │   └─ 局部窗口 → dp[i][mask][extra]
 ├─ 合法性由谁决定？
 │   ├─ 行内结构 → 位运算
 │   ├─ 行间关系 → 兼容表
 │   ├─ 子集关系 → 枚举 sub
 │   └─ 分组冲突 → 排序 / 分块 / 容斥
 └─ 最后再判断能否继续化简
     ├─ 能滚动数组？
     ├─ 能预处理兼容？
     └─ 能推成公式？
```

### 6.2 信号词 → 题型 判断表

| 题面信号词 | 往往对应的模型 | 第一反应 |
|------------|----------------|----------|
| `n ≤ 20` | 集合型状压 | 想 `dp[mask]` 或 `dp[mask][i]` |
| `m ≤ 10` 的网格 | 逐行棋盘型 | 枚举合法行状态 |
| “相邻/隔一格不能同时选” | 行内位约束 | `s & (s<<1)` / `s & (s<<2)` |
| “上一行、上上行影响当前行” | 多行历史 | `dp[row][id1][id2]` |
| `b_i ≤ 7`、窗口很窄 | 局部窗口型 | `dp[i][mask][extra]` |
| “恰好满足 K 个条件” | 子集计数/容斥 | 先算 `f(S)` |
| “小质数冲突” | 质因子集合型 | 压小质数集合 |
| “边界呈阶梯形” | 轮廓线状压 | 多进制 / 编码 |

### 6.3 常见复杂度估算

| 状态形式 | 复杂度大致量级 | 备注 |
|----------|----------------|------|
| `dp[mask]` | `O(n 2^n)` | `n ≤ 20` 常见 |
| `dp[mask][i]` | `O(n^2 2^n)` | TSP 标配 |
| `dp[row][mask]` | `O(n V^2)` | `V` 为合法状态数 |
| `dp[row][id1][id2]` | `O(n V^3)` | 需靠 `V` 很小支撑 |
| `dp[i][mask][extra]` | `O(n 2^k poly(k))` | `k` 必须很小 |
| 轮廓线记忆化 | `O(状态数 × 转移数)` | 状态编码最重要 |

## 七、终极总结与归纳

### 7.1 状态压缩动态规划的本质

**一句话概括：把“本来很难存、很难判、很难转”的离散结构状态，编码成一个可枚举整数，再围绕这个整数设计转移。**

展开来说，状压 DP 真正厉害的地方不在于“用了位运算”，而在于它逼着你把题目限制抽象成**集合关系、局部结构、兼容关系**。一旦抽象正确，很多题就从“没有头绪”变成“套一个小状态枚举框架”。

### 7.2 核心要点

1. 先判断小参数在哪：元素个数、棋盘宽度、窗口大小、质数个数。
2. 先想“压什么”，再想“怎么转”。
3. 合法状态、兼容关系、`popcount` 等信息应尽量预处理。
4. `dp[mask]`、`dp[mask][last]`、`dp[row][mask]` 是最常见的三类基础模板。
5. 看到“恰好 K 个”“两边不能冲突”“顺序待定分组”时，要警惕容斥和双集合状压。
6. 高级题里，状压常和前缀和、二分、博弈、容斥、数论分组一起出现。

### 7.5 口诀

> 🧠 小参数，先别慌，二进制里把它装。  
> 行压一层看本行，两层历史也常见。  
> 集合顺序看 `mask`，末尾位置要记清。  
> 容斥、分组、前缀和，复杂限制也能拆。  
> 若觉状态还能消，先别急着写表，先看能否继续化简。  
> 状压不是炫位运算，而是先把结构想清楚，再把转移落下来。
