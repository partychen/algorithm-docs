## 一、什么是决策单调性优化动态规划

很多 DP 的朴素转移都长这样：为了求当前状态 `dp[i]` 或 `dp[t][i]`，你要在一整段候选决策里枚举最优断点 `k`。如果每个状态都暴力扫一遍，那么复杂度往往是 `O(n^2)`、`O(kn^2)` 甚至 `O(n^3)`。

所谓**决策单调性优化**，本质就是发现：随着状态下标变大，最优决策点 `opt` 往往也会**单调地向右移动**，不会来回跳。既然最优点不会回头，那么我们就没有必要每次都从头枚举整段决策区间。

这类优化最常见的主线，其实只有两条：

1. **区间断点型**：`dp[l][r] = min(dp[l][k] + dp[k+1][r] + w(l, r))`，常对应 Knuth / 四边形不等式优化。
2. **分段划分型**：`dp[t][i] = min(dp[t-1][k] + w(k+1, i))`，常对应分治优化。

除此以外，还有一批**相关扩展**：它们不一定是标准意义上的“`opt` 单调”DP，但也会利用“候选决策的优劣只切换一次”这类递变规律来减少枚举，例如决策生存区间维护、候选集合的单调合并等。

一个非常重要的观念是：**你优化的不是状态本身，而是“决策集合的搜索方式”**。状态定义没变，变化的是“我如何更快地找到最优转移来源”。

```text
朴素做法：
状态 i      : 枚举所有 k < i
状态 i + 1  : 再枚举所有 k < i + 1

若存在决策单调性：
opt[i] <= opt[i+1] <= opt[i+2] <= ...

那就意味着：
- 搜索区间会右移
- 被淘汰的决策不会重新变优
- 一部分候选只需要维护一次
```

| 方法 | 典型转移形式 | 关键保证 | 常见复杂度 | 代表题 |
| --- | --- | --- | --- | --- |
| Knuth 优化 | `dp[l][r]` 区间断点 | `opt[l][r-1] <= opt[l][r] <= opt[l+1][r]` | `O(n^3) -> O(n^2)` | 石子合并等区间合并题 |
| 分治优化 | `dp[t][i] = min(dp[t-1][k] + w(k+1, i))` | `opt[t][i] <= opt[t][i+1]` | `O(kn^2) -> O(knlogn)` | 邮局、Ciel and Gondolas、CF868F |
| 决策区间维护（相关扩展） | 比较两个决策谁在位置 `x` 更优 | 胜负分界点单调 | `O(n^2) -> O(nlogn)` 或 `O(n)` | 诗人小 G、Lightning Conductor |

## 二、核心思想与前置知识

### 2.1 最优决策点为什么会右移

设一类 DP 的转移是：

```
dp[t][i] = min_{0 <= k < i} { prev[k] + cost(k + 1, i) }
opt[t][i] = 取得最小值的 k
```

如果我们能够证明 `opt[t][i] <= opt[t][i+1]`，那么求 `dp[t][i+1]` 时就不必再从 `0` 开始枚举；因为左边那一大段候选已经不可能重新变优了。

这类单调性常见的来源有两个：

- **代价函数满足四边形不等式 / Monge 性质**。
- **比较两个候选决策时，优劣关系只会切换一次**。

下面是最常用的分治优化骨架：

```cpp
const long long INF = 4e18;
long long prev_dp[MAXN], cur_dp[MAXN];
int best_k[MAXN];

long long cost(int l, int r);

void solve(int l, int r, int ql, int qr) {
    if (l > r) return;
    int mid = (l + r) >> 1;
    int upper = min(mid - 1, qr);
    cur_dp[mid] = INF;
    best_k[mid] = ql;
    for (int k = ql; k <= upper; k++) {
        if (prev_dp[k] == INF) continue;
        long long val = prev_dp[k] + cost(k + 1, mid);
        if (val < cur_dp[mid]) {
            cur_dp[mid] = val;
            best_k[mid] = k;
        }
    }
    if (l == r) return;
    solve(l, mid - 1, ql, best_k[mid]);
    solve(mid + 1, r, best_k[mid], qr);
}
```

这个骨架的核心不是递归，而是：**左半区间的最优决策不会跑到右边去，右半区间的最优决策也不会跑回左边去**。

### 2.2 区间 DP 中的 Knuth / 四边形不等式优化

如果转移形如：

```
dp[l][r] = min_{l <= k < r} { dp[l][k] + dp[k+1][r] + w(l, r) }
```

并且 `w` 满足区间单调性和四边形不等式，那么最优断点会满足：

```
opt[l][r-1] <= opt[l][r] <= opt[l+1][r]
```

这就能把原本枚举整个 `[l, r-1]` 的过程，缩到一个很小的区间里：

```cpp
const long long INF = 4e18;
long long dp[MAXN][MAXN];
int opt[MAXN][MAXN];

long long seg_cost(int l, int r);

void solve(int n) {
    for (int i = 1; i <= n; i++) {
        dp[i][i] = 0;
        opt[i][i] = i;
    }
    for (int len = 2; len <= n; len++) {
        for (int l = 1; l + len - 1 <= n; l++) {
            int r = l + len - 1;
            int L = opt[l][r - 1];
            int R = min(r - 1, opt[l + 1][r]);
            dp[l][r] = INF;
            for (int k = L; k <= R; k++) {
                long long val = dp[l][k] + dp[k + 1][r] + seg_cost(l, r);
                if (val < dp[l][r]) {
                    dp[l][r] = val;
                    opt[l][r] = k;
                }
            }
        }
    }
}
```

注意这里不能省掉 `min(r - 1, ...)` 这样的边界保护，否则会直接越界或把无效断点拿去转移。

### 2.3 当“比较两个决策”的胜负区间是连续段时

有些题并不长成标准的二维 DP，但你仍然会遇到这种结构：

```
dp[i] = min value(j, i)   或   best[i] = max value(j, i)
```

如果对固定的两个候选决策 `a`、`b` 而言，`value(a, x)` 和 `value(b, x)` 的优劣只会切换一次，那么就可以维护：

- 决策 `a` 在 `[L, R]` 上最优；
- 决策 `b` 从某个分界点之后开始更优。

这时常见写法是“队列 + 生存区间”：

```cpp
struct Node {
    int id;
    int l, r;
};

long long dp[MAXN];
long long value(int j, int i);
int first_better(int old_id, int new_id, int L, int R);

void solve(int n) {
    deque<Node> q;
    dp[0] = 0;
    q.push_back({0, 1, n});
    for (int i = 1; i <= n; i++) {
        while (!q.empty() && q.front().r < i) q.pop_front();
        dp[i] = value(q.front().id, i);
        while (!q.empty() && value(i, q.back().l) <= value(q.back().id, q.back().l)) {
            q.pop_back();
        }
        if (i == n) continue;
        if (q.empty()) {
            q.push_back({i, i + 1, n});
            continue;
        }
        int pos = first_better(q.back().id, i, max(i + 1, q.back().l), q.back().r);
        if (pos <= q.back().r) {
            q.back().r = pos - 1;
            q.push_back({i, pos, n});
        }
    }
}
```

这套东西和斜率优化很像，但不要求一定是直线；只要**“两个候选的比较结果有单次交替”**，就能尝试维护决策区间。

### 2.4 三种常见“可用信号”

| 信号 | 说明 | 常见优化 |
| --- | --- | --- |
| `dp[l][r]` 断点合并，且代价只跟区间有关 | 多半是区间 DP | Knuth / 四边形不等式优化 |
| `dp[t][i] = min(prev[k] + w(k+1, i))` | 多层分段 DP | 分治优化 |
| 比较两个候选时，优劣只切换一次 | 候选集合可以按区间维护 | 队列维护决策生存区间 |
| `cost` 可用前缀和、二维前缀和、移动指针维护 | 转移代价本身不再是瓶颈 | 更容易把重点放到“优化枚举”上 |

## 三、子模型与分类详解

### 3.1 区间断点型：Knuth / 四边形不等式优化

这类题通常是“把区间 `[l, r]` 切成左右两半”，代价里额外带一个只与区间整体相关的 `w(l, r)`。

```
状态定义：dp[l][r] = 解决区间 [l, r] 的最优值
转移关键：枚举断点 k，把 [l, r] 拆成 [l, k] 和 [k+1, r]
适用信号：w(l, r) 只与区间有关，且满足四边形不等式与区间单调性
实现要点：维护 opt[l][r]，把 k 的搜索区间缩成 [opt[l][r-1], opt[l+1][r]]
```

```cpp
long long dp[MAXN][MAXN];
int opt[MAXN][MAXN];

for (int i = 1; i <= n; i++) {
    dp[i][i] = 0;
    opt[i][i] = i;
}
for (int len = 2; len <= n; len++) {
    for (int l = 1; l + len - 1 <= n; l++) {
        int r = l + len - 1;
        int L = opt[l][r - 1];
        int R = min(r - 1, opt[l + 1][r]);
        dp[l][r] = INF;
        for (int k = L; k <= R; k++) {
            long long val = dp[l][k] + dp[k + 1][r] + w(l, r);
            if (val < dp[l][r]) {
                dp[l][r] = val;
                opt[l][r] = k;
            }
        }
    }
}
```

### 3.2 多阶段分段型：分治优化

这一型是最常见的“前缀分段”模型：前 `i` 个元素分成 `t` 段，最后一段从 `k+1` 开始。

```
状态定义：dp[t][i] = 前 i 个元素分成 t 段的最优值
转移关键：dp[t][i] = min(dp[t-1][k] + cost(k+1, i))
适用信号：最优断点 opt[t][i] 随 i 不减
实现要点：整层一起算，对中点求最优决策后递归左右半边
```

```cpp
long long prev_dp[MAXN], cur_dp[MAXN];
int opt[MAXN];

void solve(int l, int r, int ql, int qr) {
    if (l > r) return;
    int mid = (l + r) >> 1;
    int upper = min(mid - 1, qr);
    cur_dp[mid] = INF;
    int best = ql;
    for (int k = ql; k <= upper; k++) {
        if (prev_dp[k] == INF) continue;
        long long val = prev_dp[k] + cost(k + 1, mid);
        if (val < cur_dp[mid]) {
            cur_dp[mid] = val;
            best = k;
        }
    }
    opt[mid] = best;
    solve(l, mid - 1, ql, best);
    solve(mid + 1, r, best, qr);
}
```

### 3.3 决策生存区间型：比较两个候选的优劣切换点

这类题的重点不是 `opt[i]` 表，而是：

- 旧决策 `a` 当前在某段区间最优；
- 新决策 `b` 插入后，只会从某个位置开始把 `a` 打败；
- 于是每个候选都对应一个“自己负责的区间”。

```
状态定义：dp[i] = 第 i 个位置的最优值
转移关键：dp[i] = min/max value(j, i)
适用信号：对任意 a,b，value(a, x) 与 value(b, x) 的优劣只切换一次
实现要点：维护候选队列，每个候选带一个生效区间 [l, r]
```

```cpp
struct Node { int id, l, r; };
long long dp[MAXN];

void solve(int n) {
    deque<Node> q;
    dp[0] = 0;
    q.push_back({0, 1, n});
    for (int i = 1; i <= n; i++) {
        while (!q.empty() && q.front().r < i) q.pop_front();
        dp[i] = value(q.front().id, i);
        while (!q.empty() && value(i, q.back().l) <= value(q.back().id, q.back().l)) {
            q.pop_back();
        }
        if (i == n) continue;
        if (q.empty()) {
            q.push_back({i, i + 1, n});
            continue;
        }
        int pos = first_better(q.back().id, i, max(i + 1, q.back().l), q.back().r);
        if (pos <= q.back().r) {
            q.back().r = pos - 1;
            q.push_back({i, pos, n});
        }
    }
}
```

### 3.4 相关扩展：先做 DP，再利用单调性合并候选

还有一类题，DP 本身只是第一步。你先用 DP / BFS 算出“若干可行候选对”，然后在最终合并时利用有序性或单调指针降低复杂度。**它们不属于标准的“DP 转移决策点单调”主线**，但常常和本专题一起出现，因此可以当作边界扩展来理解。

```
状态定义：dp[...] 负责生成候选；cand[i] = (收益, 代价)
转移关键：先求出所有有效候选，再排序合并
适用信号：随着收益变大，可选的另一半候选范围单调收缩
实现要点：排序后双指针、前缀最值、前后缀最优配合使用
```

```cpp
sort(cand.begin(), cand.end());
int j = (int)cand.size() - 1;
int best = -INF;
for (int i = 0; i < (int)cand.size(); i++) {
    while (j >= 0 && cand[i].gain + cand[j].gain > limit) {
        best = max(best, cand[j].gain - cand[j].cost);
        j--;
    }
    answer = max(answer, cand[i].gain + best);
}
```

### 3.5 模型对比总表

| 模型 | 你维护的对象 | 典型复杂度 | 难点 |
| --- | --- | --- | --- |
| Knuth 优化 | `opt[l][r]` | `O(n^2)` | 证明四边形不等式 |
| 分治优化 | 每层 `opt[i]` | `O(knlogn)` | 证明 `opt` 随 `i` 单调 |
| 决策区间维护 | 候选的生效区间 | `O(nlogn)` / `O(n)` | 写对分界点二分 |
| 可行性合并 | 候选集 + 双指针 | 依题而定 | 先把候选状态设计对 |

### 题目 1：石子合并（洛谷 P1880）

> 把一排石子不断合并成一堆，每次只能合并相邻区间，代价是区间和，求最小总代价。

**分析**：这是最经典的区间 DP 断点模型。`w(l, r)` 就是区间和，它满足区间单调性和四边形不等式，因此最优断点具有单调性，可以用 Knuth 优化把 `O(n^3)` 压到 `O(n^2)`。若你做的是 `P1880` 原题的环形版本，通常先断环成链，再在链上处理这个线性子问题。

```
状态定义：dp[l][r] = 合并区间 [l, r] 的最小代价
转移方程：dp[l][r] = min(dp[l][k] + dp[k+1][r] + sum[r] - sum[l-1])
边界条件：dp[i][i] = 0，opt[i][i] = i
答案位置：dp[1][n]
```

**C++ 代码**：
```cpp
const long long INF = 4e18;
long long dp[MAXN][MAXN], sum[MAXN];
int opt[MAXN][MAXN];

void solve(int n) {
    for (int i = 1; i <= n; i++) {
        dp[i][i] = 0;
        opt[i][i] = i;
    }
    for (int len = 2; len <= n; len++) {
        for (int l = 1; l + len - 1 <= n; l++) {
            int r = l + len - 1;
            int L = opt[l][r - 1];
            int R = min(r - 1, opt[l + 1][r]);
            dp[l][r] = INF;
            for (int k = L; k <= R; k++) {
                long long val = dp[l][k] + dp[k + 1][r] + sum[r] - sum[l - 1];
                if (val < dp[l][r]) {
                    dp[l][r] = val;
                    opt[l][r] = k;
                }
            }
        }
    }
}
```

**核心启示**：
- **区间总代价独立于断点** 时，最容易出现 Knuth 优化。
- **opt 表** 是这类题的灵魂，不要只记 `dp`。
- **断点右移** 的含义是：更长的区间，其最优切法不会突然跳回很左边。

### 题目 2：邮局（洛谷 P4767 / IOI2000）

> 一条直线上有若干村庄，建 `k` 个邮局，使所有村庄到最近邮局的距离和最小。

**分析**：先把村庄位置排序。若最后一个邮局负责区间 `[l, r]`，那么最优位置在中位数。于是问题变成“前 `i` 个村庄分成 `t` 段”的 DP，且 `cost(l, r)` 只与区间有关，决策点单调，可用分治优化。

```
状态定义：dp[t][i] = 前 i 个村庄建 t 个邮局的最小总代价
转移方程：dp[t][i] = min(dp[t-1][k] + cost(k+1, i))
边界条件：dp[0][0] = 0，其余无效状态为 INF
答案位置：dp[k][n]
```

**C++ 代码**：
```cpp
const long long INF = 4e18;
long long prev_dp[MAXN], cur_dp[MAXN], pre[MAXN];
int x[MAXN], best_k[MAXN];

long long cost(int l, int r) {
    int m = (l + r) >> 1;
    long long left = 1LL * x[m] * (m - l + 1) - (pre[m] - pre[l - 1]);
    long long right = (pre[r] - pre[m]) - 1LL * x[m] * (r - m);
    return left + right;
}

void solve(int l, int r, int ql, int qr) {
    if (l > r) return;
    int mid = (l + r) >> 1;
    cur_dp[mid] = INF;
    int best = ql;
    for (int k = ql; k <= min(mid - 1, qr); k++) {
        if (prev_dp[k] == INF) continue;
        long long val = prev_dp[k] + cost(k + 1, mid);
        if (val < cur_dp[mid]) {
            cur_dp[mid] = val;
            best = k;
        }
    }
    best_k[mid] = best;
    solve(l, mid - 1, ql, best);
    solve(mid + 1, r, best, qr);
}
```

**核心启示**：
- **中位数最优** 是把几何题翻译成 DP 的关键。
- **先把 `cost(l, r)` 压成 `O(1)`**，再谈决策优化。
- **分治优化** 的典型手感，就是“一层一层算整行 DP”。

### 题目 3：诗人小 G（洛谷 P1912 / NOI2009）

> 把若干单词分行，每一行的代价与“这一行长度和理想长度的偏差”有关，求最小总难看度。

**分析**：朴素转移是枚举上一行从哪里结束，`O(n^2)`。但对两个候选断点 `a`、`b` 来说，谁更优只会在某个位置切换一次，所以可以维护“每个断点在哪一段区间负责最优”，用队列管理决策生存区间。

```
状态定义：dp[i] = 前 i 个单词排版后的最小代价
转移方程：dp[i] = min(dp[j] + cost(j+1, i))
边界条件：dp[0] = 0；候选队列初始放入决策 0
答案位置：dp[n]
```

**C++ 代码**：
```cpp
struct Node { int id, l, r; };
long long dp[MAXN], s[MAXN];

long long value(int j, int i) {
    return dp[j] + pow_cost(llabs(s[i] - s[j] - L));
}

int first_better(int old_id, int new_id, int L, int R);

void solve(int n) {
    deque<Node> q;
    dp[0] = 0;
    q.push_back({0, 1, n});
    for (int i = 1; i <= n; i++) {
        while (!q.empty() && q.front().r < i) q.pop_front();
        dp[i] = value(q.front().id, i);
        while (!q.empty() && value(i, q.back().l) <= value(q.back().id, q.back().l)) {
            q.pop_back();
        }
        if (i == n) continue;
        if (q.empty()) {
            q.push_back({i, i + 1, n});
            continue;
        }
        int pos = first_better(q.back().id, i, max(i + 1, q.back().l), q.back().r);
        if (pos <= q.back().r) {
            q.back().r = pos - 1;
            q.push_back({i, pos, n});
        }
    }
}
```

**核心启示**：
- **不一定要有二维 `opt` 表**，决策单调性也能用。
- **比较两个候选的分界点**，是很多神题的通用套路。
- **“谁在哪一段区间负责最优”**，比“当前谁最优”更重要。

### 题目 4：Ciel and Gondolas（Codeforces 321E）

> 把 `n` 个人分成 `k` 组，每组内部有不愉快值，要求总不愉快值最小。

**分析**：先用二维前缀和把一组人的内部代价 `cost(l, r)` 算成 `O(1)`。之后就是标准的“前缀分 `k` 段”模型，满足决策单调性，直接套分治优化。

```
状态定义：dp[t][i] = 前 i 个人分成 t 组的最小不愉快值
转移方程：dp[t][i] = min(dp[t-1][k] + cost(k+1, i))
边界条件：dp[0][0] = 0；cost 用二维前缀和预处理
答案位置：dp[k][n]
```

**C++ 代码**：
```cpp
long long prev_dp[MAXN], cur_dp[MAXN];
long long pre2[MAXN][MAXN];

long long cost(int l, int r) {
    return (pre2[r][r] - pre2[l - 1][r] - pre2[r][l - 1] + pre2[l - 1][l - 1]) >> 1;
}

void solve(int l, int r, int ql, int qr) {
    if (l > r) return;
    int mid = (l + r) >> 1;
    int upper = min(mid - 1, qr);
    cur_dp[mid] = INF;
    int best = ql;
    for (int k = ql; k <= upper; k++) {
        if (prev_dp[k] == INF) continue;
        long long val = prev_dp[k] + cost(k + 1, mid);
        if (val < cur_dp[mid]) {
            cur_dp[mid] = val;
            best = k;
        }
    }
    solve(l, mid - 1, ql, best);
    solve(mid + 1, r, best, qr);
}
```

**核心启示**：
- **二维前缀和 + 分治 DP** 是一个固定搭配。
- **只优化枚举，不修改状态定义**，是决策单调性的共性。
- **把代价函数变成 `O(1)`** 往往比证明单调性更先手。

### 题目 5：Subarray Squares（CSES）

> 把数组分成 `k` 个连续子段，每段代价是该段元素和的平方，求总代价最小。

**分析**：状态还是标准分段 DP，`cost(l, r) = (pre[r] - pre[l-1])^2`。平方代价带来凸性，最优断点单调，因此仍可用分治优化。

```
状态定义：dp[t][i] = 前 i 个数分成 t 段的最小代价
转移方程：dp[t][i] = min(dp[t-1][k] + sq(pre[i] - pre[k]))
边界条件：dp[0][0] = 0；其余无效状态为 INF
答案位置：dp[k][n]
```

**C++ 代码**：
```cpp
long long prev_dp[MAXN], cur_dp[MAXN], pre[MAXN];

long long cost(int l, int r) {
    long long s = pre[r] - pre[l - 1];
    return s * s;
}

void solve(int l, int r, int ql, int qr) {
    if (l > r) return;
    int mid = (l + r) >> 1;
    int upper = min(mid - 1, qr);
    cur_dp[mid] = INF;
    int best = ql;
    for (int k = ql; k <= upper; k++) {
        if (prev_dp[k] == INF) continue;
        long long val = prev_dp[k] + cost(k + 1, mid);
        if (val < cur_dp[mid]) {
            cur_dp[mid] = val;
            best = k;
        }
    }
    solve(l, mid - 1, ql, best);
    solve(mid + 1, r, best, qr);
}
```

**核心启示**：
- **凸代价函数** 经常孕育出决策单调性。
- **分治优化与具体题意无关**，只和转移结构有关。
- **同一骨架能横着打很多题**，这是专题学习最该积累的东西。

### 题目 6：Lightning Conductor（洛谷 P3515 / POI2011）

> 对每个位置，要求它最终至少满足来自左右两侧某种“影响高度”约束，求每个位置需要补多少高度。

**分析**：这题不长成传统 `dp[t][i]`，但核心依然是“候选决策谁更优”。固定两个候选塔 `a`、`b` 后，它们对未来位置的支配关系只会切换一次，因此可维护每个候选在哪一段区间最优，左右各扫一次即可。

```
状态定义：best[i] = 来自某一侧的最强约束值
转移方程：best[i] = max(value(j, i))，j 是候选制高点
边界条件：左右各做一次扫描，最后与原高度合并
答案位置：ans[i] = max(left[i], right[i]) - h[i]
```

**C++ 代码**：
```cpp
struct Node { int id, l, r; };
long long h[MAXN], best[MAXN];

long long value(int j, int i) {
    return h[j] + ceil_sqrt(i - j);
}

int first_better(int old_id, int new_id, int L, int R);

void scan(int n) {
    deque<Node> q;
    q.push_back({1, 1, n});
    for (int i = 1; i <= n; i++) {
        while (!q.empty() && q.front().r < i) q.pop_front();
        best[i] = max(best[i], value(q.front().id, i));
        while (!q.empty() && value(i, q.back().l) >= value(q.back().id, q.back().l)) {
            q.pop_back();
        }
        if (q.empty()) {
            q.push_back({i, i, n});
            continue;
        }
        int pos = first_better(q.back().id, i, max(i, q.back().l), q.back().r);
        if (pos <= q.back().r) {
            q.back().r = pos - 1;
            q.push_back({i, pos, n});
        }
    }
}
```

**核心启示**：
- **决策单调性不等于“必须有 DP 表”**。
- **左右各扫一次** 是处理“绝对值 / 双向影响”的常见套路。
- **维护候选负责区间**，比维护“当前最大值”更稳定。

### 题目 7：Yet Another Minimization Problem（Codeforces 868F）

> 把数组分成 `k` 段，每段代价是相同数对的数量，求总代价最小。

**分析**：DP 结构是标准分段型，但 `cost(l, r)` 很难预处理成二维表。好消息是它可以通过移动左右端点在线维护，于是整题变成“分治优化 + 莫队式区间维护”。

```
状态定义：dp[t][i] = 前 i 个数分成 t 段的最小代价
转移方程：dp[t][i] = min(dp[t-1][k] + pair_cost(k+1, i))
边界条件：dp[0][0] = 0；当前区间代价由移动指针维护
答案位置：dp[k][n]
```

**C++ 代码**：
```cpp
long long prev_dp[MAXN], cur_dp[MAXN], cur_cost;
int a[MAXN], cnt[MAXA], L = 1, R = 0;

void add(int x) { cur_cost += cnt[x]; cnt[x]++; }
void del(int x) { cnt[x]--; cur_cost -= cnt[x]; }

void move_to(int l, int r) {
    while (L > l) add(a[--L]);
    while (R < r) add(a[++R]);
    while (L < l) del(a[L++]);
    while (R > r) del(a[R--]);
}

void solve(int l, int r, int ql, int qr) {
    if (l > r) return;
    int mid = (l + r) >> 1;
    int upper = min(mid - 1, qr);
    cur_dp[mid] = INF;
    int best = ql;
    for (int k = ql; k <= upper; k++) {
        if (prev_dp[k] == INF) continue;
        move_to(k + 1, mid);
        long long val = prev_dp[k] + cur_cost;
        if (val < cur_dp[mid]) {
            cur_dp[mid] = val;
            best = k;
        }
    }
    solve(l, mid - 1, ql, best);
    solve(mid + 1, r, best, qr);
}
```

**核心启示**：
- **先优化决策枚举，再优化代价维护**，两层优化经常同时出现。
- **分治优化不要求 `cost` 是静态表**，只要能高效算当前枚举到的代价即可。
- **`move_to(k+1, mid)`** 这种写法，是这类题的标准姿势。

### 题目 8：任务分配问题（洛谷 P5574 / CmdOI2019）

> 把一个排列分成 `k` 段，使每段逆序对数量之和最小。

**分析**：还是标准分段 DP，但区间代价变成逆序对数。它同样具有决策单调性，于是用分治优化枚举断点，再用树状数组配合移动区间维护当前逆序对代价。

```
状态定义：dp[t][i] = 前 i 个任务分成 t 段的最小逆序对和
转移方程：dp[t][i] = min(dp[t-1][k] + inv_cost(k+1, i))
边界条件：dp[0][0] = 0；逆序对代价通过数据结构动态维护
答案位置：dp[k][n]
```

**C++ 代码**：
```cpp
long long prev_dp[MAXN], cur_dp[MAXN], inv_cost;
Fenwick bit;
int a[MAXN], L = 1, R = 0;

void move_to(int l, int r);  // 调整当前区间并同步 inv_cost

void solve(int l, int r, int ql, int qr) {
    if (l > r) return;
    int mid = (l + r) >> 1;
    int upper = min(mid - 1, qr);
    cur_dp[mid] = INF;
    int best = ql;
    for (int k = ql; k <= upper; k++) {
        if (prev_dp[k] == INF) continue;
        move_to(k + 1, mid);
        long long val = prev_dp[k] + inv_cost;
        if (val < cur_dp[mid]) {
            cur_dp[mid] = val;
            best = k;
        }
    }
    solve(l, mid - 1, ql, best);
    solve(mid + 1, r, best, qr);
}
```

**核心启示**：
- **逆序对代价** 仍然能塞进“分段 DP + 决策单调性”框架。
- **代价维护由树状数组负责，最优断点由分治负责**，职责要分清。
- **看见 `dp[t][i] = min(dp[t-1][k] + cost(k+1, i))`** 时，先问自己有没有单调性。

### 题目 9：Noi 嘉年华（洛谷 P1973 / NOI2011，专题边界扩展）

> 一批活动区间要安排到两个会场中，要求两个会场都不冲突，并最大化较少一侧的活动数；还有“某个活动必须举办”的加强询问。

**分析**：这题更适合作为**专题边界扩展**来读，而不是标准模板题。核心仍然是离散化后的区间 / 前后缀 DP：若把一整段 `[l, r]` 的活动全部交给某个会场，那么这一段能提供的活动数量是固定的。竞赛做法会继续利用断点性质压缩枚举，并结合前后缀 DP 处理“某个活动必须举办”的加强询问，但它已经明显重于本专题主线里的 Knuth / 分治模板。

```
状态定义：f[i][a] = 到第 i 个离散点为止，会场 A 已放 a 个活动时，会场 B 最多能放多少个
转移方程：枚举上一个断点 k，把 [k+1, i] 整段分给 A 或 B
边界条件：f[0][0] = 0；不可达状态记为负无穷
答案位置：max_a min(a, f[m][a])
```

**C++ 代码**：
```cpp
const int NEG = -1000000000;
int f[MAXM][MAXN], cnt[MAXM][MAXM];

void solve(int m, int n) {
    for (int i = 0; i <= m; i++) {
        for (int a = 0; a <= n; a++) f[i][a] = NEG;
    }
    f[0][0] = 0;
    for (int r = 1; r <= m; r++) {
        for (int a = 0; a <= n; a++) {
            for (int l = 0; l < r; l++) {
                int take = cnt[l + 1][r];
                if (a >= take && f[l][a - take] != NEG) {
                    f[r][a] = max(f[r][a], f[l][a - take]);
                }
                if (f[l][a] != NEG) {
                    f[r][a] = max(f[r][a], f[l][a] + take);
                }
            }
        }
    }
}
```

**核心启示**：
- **先把题意压成“区间整体交给某一方”**，结构就出来了。
- **断点性质** 有时只负责提示你“可以继续优化”，却未必能沉淀成统一模板。
- **前后缀 DP 合并** 是处理“强制选择某个元素”问题的常用补刀。

### 题目 10：大佬（洛谷 P3724 / AH2017/HNOI2017，相关单调性扩展）

> 你需要先保证自己能活下来，再利用有限天数制造尽可能高的伤害，看能否击败大佬。

**分析**：这题也不属于标准的“DP 转移决策点单调”模型，更适合作为**相关单调性扩展**。先用 DP 求出最多能省下多少攻击天数，再 BFS / 状态扩展枚举若干 `(伤害, 耗时)` 候选，最后在有序候选上做双指针与前缀最值合并，避免暴力枚举两次技能组合。这里的单调性主要体现在**候选集合的合并阶段**，而不是主转移本身。

```
状态定义：dp[i][x] = 前 i 天后自信值为 x 时，最多能空出的攻击天数
转移方程：根据题目三类操作更新下一天状态；再对候选攻击方式做有序合并
边界条件：不可达状态记为负无穷；候选集合需去重
答案位置：检查平 A / 一次技能 / 两次技能能否在 M 天内凑够目标伤害
```

**C++ 代码**：
```cpp
const int NEG = -1000000000;
int dp[MAXN][MAXV], M;
vector<pair<int, int>> cand;

void build_days(int n) {
    for (int i = 0; i <= n; i++) {
        for (int x = 0; x <= limit; x++) dp[i][x] = NEG;
    }
    dp[0][init_conf] = 0;
    for (int i = 0; i < n; i++) {
        for (int x = 0; x <= limit; x++) {
            if (dp[i][x] < 0) continue;
            // 按题意三种操作更新 dp[i+1][...] 或额外攻击天数
        }
    }
    for (int x = 0; x <= limit; x++) M = max(M, dp[n][x]);
}

bool can_kill(int C) {
    sort(cand.begin(), cand.end());
    int j = (int)cand.size() - 1;
    int best = NEG;
    for (int i = 0; i < (int)cand.size(); i++) {
        while (j >= 0 && cand[i].first + cand[j].first > C) {
            best = max(best, cand[j].first - cand[j].second);
            j--;
        }
        if (cand[i].first + M - cand[i].second >= C) return true;
        if (best != NEG && cand[i].first + M - cand[i].second + best >= C) return true;
    }
    return M >= C;
}
```

**核心启示**：
- **单调性不只会出现在 DP 转移里，也会出现在候选合并阶段**。
- **先做状态生成，再做单调合并**，是很多综合题的套路。
- **双指针 + 前缀最值**，本质上是在利用“另一半候选范围单调收缩”。

## 五、常见技巧与思路总结

### 5.1 先证明“最优断点不会回头”

如果你没有把 `opt` 的单调性说清楚，那么后面的所有优化都没有理论基础。做题时至少要问自己：

- 是 `opt[t][i] <= opt[t][i+1]`，还是 `opt[l][r-1] <= opt[l][r] <= opt[l+1][r]`？
- 单调性来自 `cost` 的四边形不等式，还是来自两个决策的优劣只切换一次？

### 5.2 代价函数先压到 `O(1)` 或“可移动维护”

```cpp
long long seg_sum(int l, int r) { return pre[r] - pre[l - 1]; }
long long square_cost(int l, int r) {
    long long s = seg_sum(l, r);
    return s * s;
}
```

如果 `cost` 每次还要 `O(n)` 算，那你就算把决策优化了也救不了复杂度。

### 5.3 无效状态一定要保留 guard

```cpp
if (prev_dp[k] == INF) continue;
if (a >= take && f[l][a - take] != NEG) {
    f[r][a] = max(f[r][a], f[l][a - take]);
}
```

决策单调性优化非常怕“把不可达状态拿去更新”，因为一旦 `opt` 被污染，整层递归都会错。

### 5.4 决策区间维护时，要明确“谁负责哪一段”

```cpp
while (!q.empty() && q.front().r < i) q.pop_front();
int pos = first_better(old_id, new_id, L, R);
```

这类题的思维不是“当前最优是谁”，而是“每个候选从哪个位置开始接管答案”。

### 5.5 分治优化的递归边界不要偷懒

```cpp
if (l > r) return;
int upper = min(mid - 1, qr);
```

很多 WA 不是因为单调性没证明，而是因为 `mid` 左边没决策可选、`qr` 越界、`best` 初始化错了。

### 5.6 三种套路怎么选

| 看到的结构 | 优先考虑 |
| --- | --- |
| `dp[l][r]` 断点型区间 DP | Knuth / 四边形不等式优化 |
| `dp[t][i] = min(prev[k] + cost(k+1, i))` | 分治优化 |
| 两个候选的胜负只会切一次 | 决策生存区间维护 |
| 先生成候选，再做两两配对 | 排序 + 双指针 + 前缀最值 |

| “前 i 个分成 k 段” | 分治优化 DP |
| “区间合并 / 断点划分” | 区间 DP，优先检查 Knuth |
| “代价只与区间整体有关” | `cost(l, r)` 可能具备 Monge 性 |
| “比较两个候选谁更优” | 决策区间维护 |
| “区间代价能随左右端点移动更新” | 分治优化 + Mo / BIT / 双指针 |
| “最后还要强制选某元素 / 某操作” | 前后缀 DP 合并 |

### 6.3 解题 checklist

1. 先写出朴素状态和转移，别一上来就想优化。
2. 看 `cost` 是否能压成 `O(1)` 或滑动维护。
3. 试着定义 `opt`，并观察它是否右移。
4. 如果没有显式 `opt`，就比较两个候选的优劣区间是否连续。
5. 最后再决定是 Knuth、分治，还是队列维护候选区间。

## 七、终极总结与归纳

### 7.1 决策单调性优化的本质

**一句话本质**：不是所有决策都值得被反复枚举，因为最优决策的位置本身往往带有“只会向前走”的结构。

展开来说，DP 优化的关键不是背模板，而是识别：**随着状态推进，哪些候选已经永久失效，哪些候选只会在一个连续区间内负责最优**。一旦看清这一点，复杂度就能从“每次全扫”降到“只维护必要的那部分候选”。不过也要分清主次：**标准主线是 `opt` 单调的 DP 转移优化；若单调性只出现在候选合并或后处理阶段，更适合作为相关扩展理解。**

### 7.2 核心要点

1. **先有正确状态，再谈优化**；优化从来不替代状态设计。
2. **区间 DP 看 Knuth，分段 DP 看分治**，这是最常见的两条主线。
3. **比较两个候选只切一次** 时，可以维护决策生存区间。
4. **代价函数的预处理或在线维护** 往往和决策优化同等重要。
5. **无效状态 guard 不能省**，否则 `opt` 会被污染。
6. **重型综合题里，单调性常作为扩展性武器出现，但不一定属于标准决策单调性 DP。**
