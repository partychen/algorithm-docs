## 一、什么是轮廓线动态规划

**轮廓线动态规划**，常被简称为 **Contour Line DP**。它是一个大类：凡是把“已处理区与未处理区的分界线”压成状态、再逐格推进的网格 DP，都可以算作轮廓线 DP。**插头 DP（Plug DP）** 则是其中更强调**边界连通关系**的一支。

为什么这么做？因为网格题里最麻烦的往往不是“这个格子选不选”，而是“这条路径从哪儿进、往哪儿出、当前边界上有哪些尚未闭合的连通关系”。把这些信息全留在整张棋盘里太大；而如果只保留轮廓线上的必要信息，状态就会小很多。

```text
处理顺序：从上到下、从左到右

已经处理完的区域   |   尚未处理的区域
#################   |   .............
####### 当前格 ####  |   .............
-----------轮廓线-----------
      ^ 只保留这条线上的状态
```

具体到网格上，轮廓线并不是一条简单的水平线，而是一条**阶梯形折线**，它恰好把已处理的格子和未处理的格子分隔开来。假设我们正在处理第 2 行第 3 列的格子（标记为 `*`），则轮廓线的形状如下：

```text
  列:  0   1   2   3   4
     +---+---+---+---+---+
  0  | # | # | # | # | # |   # = 已处理
     +---+---+---+---+---+
  1  | # | # | # | # | # |   . = 未处理
     +---+===+===+===+---+
  2  | # | # | * ‖ . | . |   * = 当前格
     +---+---+---+---+---+
  3  | . | . | . | . | . |   === 和 ‖ 连起来
     +---+---+---+---+---+       就是轮廓线
```

轮廓线从当前行的左端沿着当前格上方水平走，到当前格处转而竖直下行，再沿下方水平走到行末——形成一个阶梯。DP 只需要保存这条阶梯线上每个槽位的信息，就足以完成所有转移。

在宽度为 $m$ 的网格里，轮廓线通常会开到 $m+1$ 个槽位。这样当前格 $(i, j)$ 在转移时，只要关心：

- 当前格左边伸出来的插头；
- 当前格上边伸下来的插头；
- 处理完后，要不要往右 / 往下留下新的插头。

和普通按行状压 DP 相比，轮廓线 DP 更擅长处理这三类问题：

1. **覆盖型**：例如骨牌覆盖，状态只记录某些格子是否已被“跨格”占用。
2. **单回路 / 单路径型**：例如哈密顿回路、唯一闭合回路，状态要记录插头如何匹配。
3. **多连通块 / 混合限制型**：例如颜色连通、炮台数量、额外权值，这时状态里还要带更多信息。

| 方法 | 状态里记录什么 | 适合问题 | 难点 |
| --- | --- | --- | --- |
| 普通行压 DP | 一行哪些位置被占用 | 骨牌覆盖、独立格合法性 | 容易，但连通性能力弱 |
| broken profile DP | 当前轮廓线上哪些口被占 | 覆盖型、障碍铺砖 | 行尾平移与当前格转移 |
| 插头 DP | 当前轮廓线的连通关系 | 回路、路径、连通块 | 状态编码与合并最难 |
| 插头 DP + 额外维度 | 连通关系 + 颜色 / 数量 / 权值 | 复杂综合题 | 状态爆炸与哈希优化 |

## 二、核心思想与前置知识

### 2.1 轮廓线是什么，为什么常开成 $m+1$

在处理格子 $(i, j)$ 时，最常见的做法是把轮廓线理解成“当前行前缀 + 下一行入口”的交界。于是长度不是 $m$，而是 $m+1$：

- 第 $j$ 位表示当前格左边的槽位；
- 第 $j+1$ 位表示当前格上方的槽位；
- 转移完成后，你会把新信息写回这两个位置，再继续扫下一个格子。

```cpp
int get(int mask, int p, int bits) {
    return (mask >> (p * bits)) & ((1 << bits) - 1);
}

int setv(int mask, int p, int bits, int v) {
    int clear = ((1 << bits) - 1) << (p * bits);
    return (mask & ~clear) | (v << (p * bits));
}

void step_cell(int i, int j, int mask) {
    int left = get(mask, j, BITS);
    int up = get(mask, j + 1, BITS);
    // 只围绕 left / up 讨论当前格的所有合法转移
}
```

这里的 `BITS` 取多少，取决于你要记录的信息：

- 覆盖型常常只要 $1$ bit；
- 括号表示法常用 $2$ bit 或 $3$ 个值；
- 最小表示法可能不再直接用固定 bit，而是用数组状态再压回整数或哈希键。

### 2.2 最基础的 broken profile 骨架

如果问题只关心“当前位置有没有被前面的放置方式占掉”，而不关心复杂连通性，那么可以把轮廓线简化成 0/1 状态：

```cpp
unordered_map<int, long long> cur, nxt;

void push(unordered_map<int, long long>& mp, int mask, long long val) {
    mp[mask] += val;
}

void solve(int n, int m) {
    cur[0] = 1;
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) {
            nxt.clear();
            for (auto [mask, ways] : cur) {
                int left = (mask >> j) & 1;
                int up = (mask >> (j + 1)) & 1;
                // 根据当前格能否放置，讨论 left / up 的转移
            }
            cur.swap(nxt);
        }
        unordered_map<int, long long> shifted;
        for (auto [mask, ways] : cur) push(shifted, mask << 1, ways);
        cur.swap(shifted);
    }
}
```

这一类题的关键不是括号匹配，而是：

- 行末如何平移到下一行；
- 当前格空着是否允许；
- 横放 / 竖放是否会和已有占用冲突。

像“蒙德里安的梦想”这种经典题，经常会把“逐格扫描 + 行末平移”改写成“按行 DP + $ok[mask]$ 合法性预处理”。写法不同，但它们维护的其实是同一种**占用型边界信息**。

### 2.3 单回路问题里的括号表示法

如果你要求的是“恰好形成一条回路”或“一条路径”，那么 0/1 就不够了，因为你还得知道：轮廓线上这些口彼此是怎么连着的。最常见的表示法是：

- `0`：没有插头；
- `1`：左括号型插头；
- `2`：右括号型插头。

这样整条轮廓线就像一个括号序列，匹配关系表示连通关系。

```cpp
int get(int mask, int p) { return (mask >> (p << 1)) & 3; }

int setv(int mask, int p, int v) {
    return (mask & ~(3 << (p << 1))) | (v << (p << 1));
}

int match_right(int mask, int p, int m) {
    int dep = 1;
    for (int q = p + 1; q <= m; q++) {
        if (get(mask, q) == 1) dep++;
        if (get(mask, q) == 2) dep--;
        if (dep == 0) return q;
    }
    return -1;
}
```

这套表示最适合：

- 单回路覆盖；
- 哈密顿回路 / 路径；
- “最后只能剩一对合法匹配口”的问题。

### 2.4 多连通块时的最小表示法

一旦问题不止一条回路、或者需要同时区分多个连通块，括号表示法就不够自然了。这时更常用的是：轮廓线上每个位置记一个“连通块编号”，再做**最小表示法**消去等价状态。

```cpp
int encode(vector<int>& st) {
    int id[MAXM] = {0};
    int tot = 0;
    int mask = 0;
    for (int x : st) {
        if (x && !id[x]) id[x] = ++tot;
        int y = x ? id[x] : 0;
        mask = mask * BASE + y;
    }
    return mask;
}
```

这里的含义是：

- `st[p] = 0` 表示这一位没有连通块；
- 非零编号只表示“谁和谁连着”，不表示编号本身有意义；
- 所以每次转移后都要重新编号，避免同构状态重复统计。

### 2.5 轮廓线 DP 为什么几乎都要哈希

插头 DP 的状态数非常不稳定：虽然理论上很多，但真正合法的状态只占一小部分。所以写法上通常不会开 $dp[pos][mask]$ 的完整数组，而是用哈希表存活跃状态。

```cpp
unordered_map<long long, long long> cur, nxt;

void push(unordered_map<long long, long long>& mp, long long key, long long val) {
    mp[key] += val;
}

void next_cell() {
    nxt.clear();
    for (auto [key, ways] : cur) {
        // 解包状态 -> 讨论转移 -> 压回 key
    }
    cur.swap(nxt);
}
```

有些题还需要把“轮廓线状态 + 额外维度”一起打包成 key，例如：

- 已用炮台数；
- 颜色规范化结果；
- 当前收益 / 最优值。

## 三、子模型与分类详解

### 3.1 覆盖型 broken profile：只记录占用，不记录连通块

这是最适合入门的模型。你只关心某个格子有没有被前面的横放 / 竖放骨牌占掉，不关心更深的路径连通结构。

```
状态定义：dp[pos][mask] = 扫到当前位置时，轮廓线占用状态为 mask 的方案数
转移关键：若当前格已被左/上之一占用，则只能消耗该占用；若 left 与 up 同时存在则状态非法；若都为空，再按题意决定能否留空 / 横放 / 竖放
适用信号：问题只问覆盖是否合法，不问路径怎样连起来
实现要点：行末要统一平移，mask 的每一位只表示“这个口有没有被占”，并明确第 j / j+1 位分别对应当前格右/下出口
```

```cpp
for (auto [mask, ways] : cur) {
    int left = (mask >> j) & 1;
    int up = (mask >> (j + 1)) & 1;
    if (left && up) continue;
    if (left || up) {
        int next_mask = mask ^ (left << j) ^ (up << (j + 1));
        push(nxt, next_mask, ways);
        continue;
    }
    if (cell_can_be_empty(i, j)) push(nxt, mask, ways);
    if (j + 1 < m) push(nxt, mask | (1 << j), ways);        // 往右留下占用
    if (i + 1 < n) push(nxt, mask | (1 << (j + 1)), ways);  // 往下留下占用
}
```

### 3.2 单回路 / 单路径型：括号表示法最常见

一旦题目开始要求“恰好形成一个回路”或“起点到终点只有一条哈密顿路径”，就要知道：轮廓线上的两个口到底是不是一对。

```
状态定义：dp[pos][mask] = 当前轮廓线上括号状态为 mask 的方案数
转移关键：讨论 left / up 是 0、1、2 的各种组合，决定插入、延伸、合并或闭环
适用信号：只允许一个连通结构，且插头匹配关系很关键
实现要点：写对括号匹配、闭环判定，以及最后一个合法闭环格子的特判
```

```cpp
int left = get(mask, j), up = get(mask, j + 1);
if (blocked[i][j]) {
    if (!left && !up) push(nxt, clear_slot(mask, j), ways);
    continue;
}
if (!left && !up) {
    if (cell_can_be_empty(i, j)) push(nxt, clear_slot(mask, j), ways);
    if (can_turn_right(i, j) && can_turn_down(i, j)) push(nxt, put_pair(mask, j), ways);
    continue;
}
if (!!left ^ !!up) {
    if (can_turn_right(i, j)) push(nxt, extend(mask, j, RIGHT), ways);
    if (can_turn_down(i, j)) push(nxt, extend(mask, j, DOWN), ways);
    continue;
}
if (left == 1 && up == 1) push(nxt, merge_right(mask, j), ways);
if (left == 2 && up == 2) push(nxt, merge_left(mask, j), ways);
if (left == 2 && up == 1) push(nxt, clear_slot(mask, j), ways);
if (left == 1 && up == 2 && clear_slot(mask, j) == 0 && is_last(i, j)) ans += ways;
```

### 3.3 多连通块最小表示法：比括号更泛化

如果你要同时维护多个连通块，或者括号表示已经不够自然，那么可以直接记录轮廓线上每个位置属于哪个连通块，再做规范化。

```
状态定义：dp[pos][st] = 当前轮廓线连通块编号数组为 st 的方案数
转移关键：当前格可能新建连通块、合并两个已有连通块、或删除一个端点
适用信号：题目允许 / 需要同时存在多个连通块，且还要区分颜色或类别
实现要点：每次转移后都把编号重标成最小表示，避免等价状态爆炸
```

```cpp
vector<int> st = decode(mask);
if (new_component) st[j] = st[j + 1] = ++tot;
if (merge_component) {
    int a = st[j], b = st[j + 1];
    for (int& x : st) if (x == b) x = a;
}
int next_mask = encode(st);
push(nxt, next_mask, ways);
```

### 3.4 轮廓线 + 额外维度：数量、颜色、收益一起记

真正的大题往往不只问方案数，还会额外卡你：

- 最多放 $K$ 个炮台；
- 每条线段有颜色；
- 旋转 / 翻转视为等价；
- 还要求最大收益而不是计数。

```
状态定义：dp[pos][mask][extra] = 当前轮廓线状态为 mask，额外信息为 extra 的最优值 / 方案数
转移关键：轮廓线负责结构，extra 负责附加限制
适用信号：连通关系之外还有数量、颜色、收益等维度
实现要点：把结构状态与 extra 一起打包进哈希键，必要时做规范化判重
```

```cpp
struct Node {
    int mask, used;
    int val;
};

void push(int mask, int used, int val) {
    if (used > K) return;
    long long key = (1LL * used << 32) | mask;
    nxt[key] = max(nxt[key], val);
}
```

### 3.5 模型对比总表

| 模型 | 记录内容 | 适合题型 | 常见技巧 |
| --- | --- | --- | --- |
| 覆盖型 broken profile | 占用与否 | 骨牌覆盖、障碍铺砖 | 行末平移、DFS 补格 |
| 括号表示法 | 单回路 / 单路径 | 哈密顿回路、单环覆盖 | 括号匹配、闭环特判 |
| 最小表示法 | 多连通块编号 | 双色连通、复杂连通块 | 重新编号、状态规范化 |
| 轮廓线 + 额外维度 | 结构 + 限制 | 炮台、颜色、最值 | 哈希键打包、滚动优化 |

## 四、8 道经典例题详解

### 🟢 入门级 Level 1：先学会“轮廓线只记必要信息”

### 题目 1：蒙德里安的梦想（AcWing 291）

> 用 $1 \times 2$ 骨牌覆盖 $n \times m$ 棋盘，问铺满方案数。

**分析**：这是最基础的 broken profile DP。虽然它常被写成按行状压，但本质上已经在做轮廓线转移：我们只保留当前边界上哪些位置还需要和下一行配对。

```
状态定义：dp[i][mask] = 处理完前 i 行后，下一行入口状态为 mask 的方案数
转移方程：dp[i][cur] += dp[i-1][pre]，前提是 pre 与 cur 不冲突且空位可在本行内配完
边界条件：dp[0][0] = 1；合法性数组 ok[mask] 预处理
答案位置：dp[n][0]
```

**C++ 代码**：
```cpp
long long dp[MAXN][1 << MAXM];
bool ok[1 << MAXM];

void solve(int n, int m) {
    dp[0][0] = 1;
    for (int i = 1; i <= n; i++) {
        for (int cur = 0; cur < (1 << m); cur++) {
            for (int pre = 0; pre < (1 << m); pre++) {
                if ((cur & pre) == 0 && ok[cur | pre]) {
                    dp[i][cur] += dp[i - 1][pre];
                }
            }
        }
    }
}
```

**核心启示**：
- **覆盖型轮廓线 DP** 可以不涉及连通块。
- **合法性预处理** 常常能把格内 DFS 融进一个 $ok[mask]$。
- **$dp[n][0]$** 这种“边界清空”结尾，是轮廓线 DP 的标志之一。

### 题目 2：插头 DP 模板（洛谷 P5056）

> 给一个有障碍的网格，要求所有可行格子恰好被一条闭合回路覆盖，求方案数。

**分析**：这是最标准的括号表示法模板题。当前格只看左插头和上插头，再分类讨论是新开一对括号、延伸、合并还是闭环。

```
状态定义：dp[pos][mask] = 扫到当前位置时，轮廓线括号状态为 mask 的方案数
转移方程：按 left / up 的 0、1、2 分类讨论插入、延伸、匹配、闭环
边界条件：初始只有空状态；障碍格必须 left = up = 0 才能通过
答案位置：最后一个可行格闭环时累加到 ans
```

**C++ 代码**：
```cpp
unordered_map<int, long long> cur, nxt;

void expand(int i, int j) {
    nxt.clear();
    for (auto [mask, ways] : cur) {
        int left = get(mask, j), up = get(mask, j + 1);
        if (blocked[i][j]) {
            if (!left && !up) push(nxt, clear_slot(mask, j), ways);
            continue;
        }
        // 新建一对括号、延伸、merge_left、merge_right、闭环特判
    }
    cur.swap(nxt);
}
```

**核心启示**：
- **P5056** 几乎就是整个插头 DP 的“语法课”。
- **左 / 上两个插头** 是所有分类讨论的入口。
- **最后一个合法闭环点** 往往要单独特判，不能让局部小环提前闭死。

### 🟡 进阶级 Level 2：障碍、整图覆盖与高精度计数

### 题目 3：Eat the Trees（HDU 1693）

> 有障碍的棋盘上，要用若干不相交回路覆盖所有树格，问方案数。

**分析**：和模板题很像，但它通常允许**多个不相交回路**，这一点和“恰好一条闭合回路”的 P5056 不同。对障碍格，插头必须全清空；对树格，则要求它被合法覆盖，但局部闭环未必一定非法，关键看题目是否要求唯一回路。

```
状态定义：dp[pos][mask] = 扫到当前位置时，轮廓线状态为 mask 的方案数
转移方程：障碍格只允许空过；树格必须在“不放 / 放边”中选合法方案
边界条件：初始空状态；所有树格都要被有效覆盖
答案位置：处理完整张图后的空状态方案数
```

**C++ 代码**：
```cpp
for (auto [mask, ways] : cur) {
    int left = get(mask, j), up = get(mask, j + 1);
    if (!tree[i][j]) {
        if (!left && !up) push(nxt, clear_slot(mask, j), ways);
        continue;
    }
    if (!left && !up) push(nxt, put_pair(mask, j), ways);
    if (left && !up) push(nxt, extend(mask, j, RIGHT), ways), push(nxt, extend(mask, j, DOWN), ways);
    if (!left && up) push(nxt, extend(mask, j, RIGHT), ways), push(nxt, extend(mask, j, DOWN), ways);
    if (left == 2 && up == 1) push(nxt, clear_slot(mask, j), ways);  // 允许在当前格闭掉一个局部回路
    if (left == 1 && up == 2) push(nxt, clear_slot(mask, j), ways);  // 是否允许取决于题目的“多环 / 单环”口径
    if (left == 1 && up == 1) push(nxt, merge_right(mask, j), ways);
    if (left == 2 && up == 2) push(nxt, merge_left(mask, j), ways);
}
```

**核心启示**：
- **障碍格** 会极大改变状态合法性。
- **“当前格必须被覆盖”** 和 **“当前格可以空着”** 是两类完全不同的题。
- **“单环模板”和“多环覆盖”** 的核心差别，正是局部闭环能不能提前出现。

### 题目 4：邮递员（洛谷 P2289 / HNOI2004）

> 在 $m \times n$ 网格点上，从起点出发走遍所有邮筒且恰好一次，最后回到起点，问最短回路数量。

**分析**：这题本质是整图哈密顿回路计数。状态仍然是单回路插头 DP，只是答案会很大，通常需要高精度或自定义大整数。

```
状态定义：dp[pos][mask] = 扫到当前位置时，轮廓线括号状态为 mask 的方案数（高精度）
转移方程：按照单回路模型处理 left / up 的组合，保证每个点度数为 2
边界条件：初始空状态；整张图必须形成唯一回路
答案位置：在最后一个合法闭环点把方案数累加到 ans，再按题目计数口径输出
```

**C++ 代码**：
```cpp
unordered_map<int, BigInt> cur, nxt;

for (auto [mask, ways] : cur) {
    int left = get(mask, j), up = get(mask, j + 1);
    if (!left && !up) push(nxt, put_pair(mask, j), ways);
    if (left && !up) push(nxt, extend(mask, j, RIGHT), ways), push(nxt, extend(mask, j, DOWN), ways);
    if (!left && up) push(nxt, extend(mask, j, RIGHT), ways), push(nxt, extend(mask, j, DOWN), ways);
    if (left == 1 && up == 2 && clear_slot(mask, j) == 0) ans += ways;
    if (left == 1 && up == 1) push(nxt, merge_right(mask, j), ways);
    if (left == 2 && up == 2) push(nxt, merge_left(mask, j), ways);
}
```

**核心启示**：
- **单回路插头 DP** 可以自然处理哈密顿回路。
- **大整数** 有时不是附属，而是主实现难点之一。
- **“所有点度数为 2”** 是把题面翻译成插头转移的关键；若题目把顺逆时针视为不同，再按题意后处理。

### 🟠 提升级 Level 3：从单回路到单路径，再到附加限制

### 题目 5：Formula 1（Ural 1519）

> 在有障碍的网格中，从起点走到终点，要求经过所有可行格恰好一次，问哈密顿路径数量。

**分析**：和闭环题最大的不同是：起点和终点的度数应为 $1$，其他可行格的度数应为 $2$。所以它是“单路径插头 DP”，不是单回路。

```
状态定义：dp[pos][mask] = 扫到当前位置时，轮廓线括号状态为 mask 的方案数
转移方程：端点格允许度数为 1；若该格是可行普通格，则最终度数必须为 2；障碍格度数为 0
边界条件：起点、终点的度数规则单独处理；障碍格仍要求无插头穿过
答案位置：处理完整张图后的空状态方案数
```

**C++ 代码**：
```cpp
bool endpoint = (i == sx && j == sy) || (i == tx && j == ty);
int left = get(mask, j), up = get(mask, j + 1);

if (blocked[i][j]) {
    if (!left && !up) push(nxt, clear_slot(mask, j), ways);
} else if (endpoint) {
    if (!left && up) push(nxt, extend(mask, j, RIGHT), ways);
    if (left && !up) push(nxt, extend(mask, j, DOWN), ways);
    if (!left && !up) push(nxt, start_one(mask, j), ways);
} else {
    // 普通格走单回路 / 单路径的标准四类转移
}
```

**核心启示**：
- **起点终点的度数差异**，就是“回路题”变“路径题”的根本。
- **端点特判** 必须写进当前格转移，不能最后补。
- **同一套括号表示法** 可以同时处理回路与路径。

### 题目 6：俄罗斯方块（洛谷 P5347 / XR-1）

> 在有障碍的网格上画若干线段，统计本质不同的方案数；还可能带颜色与旋转等价判定。

**分析**：这题的结构仍然是轮廓线推进，但状态里除了连通关系，还要放颜色或对称规范化结果。属于“插头 DP + 额外维度”的综合题。

```
状态定义：dp[pos][mask][sig] = 扫到当前位置时，轮廓线结构为 mask、附加签名为 sig 的方案数
转移方程：枚举当前格是否画线、线的方向、颜色及其规范化表示
边界条件：障碍格不能穿线；若题目把旋转视为等价，则状态或答案要做规范化
答案位置：处理完整张图后的合法终态方案数
```

**C++ 代码**：
```cpp
long long pack(int mask, int sig) {
    return (1LL * sig << 32) | (unsigned)mask;
}

void push(int mask, int sig, int ways) {
    sig = normalize_sig(sig);
    long long key = pack(mask, sig);
    nxt[key] = (nxt[key] + ways) % MOD;
}

for (auto [key, ways] : cur) {
    int mask = (int)key;
    int sig = (int)(key >> 32);
    // 枚举当前格画横线 / 竖线 / 不画，并维护 sig
}
```

**核心启示**：
- **轮廓线负责结构，sig 负责附加限制**，分工要明确。
- **规范化** 是这种题的生命线，否则同一个方案会被重复算很多次。
- **P5347** 很适合训练“结构状态 + 额外状态”的打包能力。

### 🔴 困难级 Level 4：多连通块与高维状态爆炸

### 题目 7：Black and White（UVA 10572）

> 给棋盘染黑白，要求黑块与白块都连通，还要满足局部 $2 \times 2$ 约束，问方案数。

**分析**：这题已经不能只用括号表示，因为同时存在两种颜色、多个连通块。最常见做法是轮廓线 + 最小表示法：状态记录每个位置当前属于哪个黑 / 白连通块，再做重编号。

```
状态定义：dp[pos][state] = 扫到当前位置时，轮廓线上黑白连通块状态为 state 的方案数
转移方程：枚举当前格染黑或白，合并相邻同色连通块，并检查 2×2 约束
边界条件：状态需最小表示；最后黑白各自恰为一个连通块
答案位置：处理完整张图后的合法终态方案数
```

**C++ 代码**：
```cpp
vector<int> st = decode(mask);
for (int color = 0; color < 2; color++) {
    if (!valid_2x2(i, j, color)) continue;
    apply_color(st, j, color);
    merge_same_color(st, j, j + 1);
    int next_mask = encode(st);
    push(nxt, next_mask, ways);
}
```

**核心启示**：
- **最小表示法** 是从“单回路”迈向“多连通块”的关键升级。
- **结构合法性 + 局部约束** 常常要同时检查。
- **这类题最难的不是 DP 名字，而是状态设计。**

### 题目 8：喵星人的入侵（洛谷 P2337 / SCOI2012）

> 网格中有障碍、起终点、可放炮台的位置以及最多 $K$ 个炮台限制，要让路径存在并尽量提高伤害。

**分析**：这是非常典型的“轮廓线 + 额外维度 + 最值 DP”综合题。结构状态要维护路径连通，额外维度要维护已放炮台数和当前价值，分类讨论极多，但核心框架仍然是当前格只看左口和上口。

```
状态定义：dp[pos][mask][k] = 扫到当前位置时，轮廓线状态为 mask、已用炮台数为 k 的最大收益
转移方程：枚举当前格作为障碍 / 路径 / 炮台时的合法转移，并更新收益
边界条件：k 不能超过 K；起点终点、障碍格、路径度数都要单独合法性检查
答案位置：所有合法终态中的最大收益
```

**C++ 代码**：
```cpp
void push(int mask, int used, int val) {
    if (used > K) return;
    long long key = (1LL * used << 32) | (unsigned)mask;
    nxt[key] = max(nxt[key], val);
}

for (auto [key, val] : cur) {
    int used = (int)(key >> 32);
    int mask = (int)key;
    int left = get(mask, j), up = get(mask, j + 1);
    // 当前格作为障碍 / 路径 / 炮台 的分类转移
}
```

**核心启示**：
- **P2337** 体现了插头 DP 的“状态工程学”本质。
- **路径结构** 和 **收益维度** 可以并存，但必须层次分清。
- **哈希 + 滚动数组 + 强合法性判断** 是这类题的基本配置。

## 五、常见技巧与思路总结

### 5.1 当前格永远只看左口和上口

```cpp
int left = get(mask, j, BITS);
int up = get(mask, j + 1, BITS);
```

不要一上来把整个轮廓线都脑补成一团。绝大多数转移，其实只围绕这两个口展开。

### 5.2 行末平移是 broken profile 的固定动作

```cpp
unordered_map<int, long long> shifted;
for (auto [mask, ways] : cur) push(shifted, (mask << 1) & limit_mask, ways);
cur.swap(shifted);
```

如果你用的是“$m+1$ 槽位”写法，那么走完一行后通常都要整体平移一格，让下一行继续复用同一套下标语义。

### 5.3 括号匹配一定要写成独立函数

```cpp
int match_left(int mask, int p);
int match_right(int mask, int p, int m);
```

$(1,1)$、$(2,2)$ 的合并最容易写挂。把匹配逻辑拆成函数，比把它们糊在转移里安全得多。

### 5.4 最小表示法的本质是“去同构”

```cpp
for (int& x : st) {
    if (x && !id[x]) id[x] = ++tot;
    if (x) x = id[x];
}
```

编号本身没有意义，关系才有意义。任何时候只要出现“连通块标号”，就要警惕等价状态重复。

### 5.5 哈希存状态，滚动存层数

```cpp
unordered_map<long long, long long> cur, nxt;

nxt.clear();
for (auto [key, val] : cur) {
    // 根据当前格转移到 next_key
    push(nxt, next_key, next_val);
}
cur.swap(nxt);
```

插头 DP 很少直接开满数组，因为合法状态远少于理论状态。真正需要优化的，是**活跃状态集合**。

### 5.6 先判断“当前格可不可以空着”

| 题型 | 当前格能否空着 |
| --- | --- |
| 骨牌覆盖 | 不一定，取决于是否最终能铺满 |
| 单回路覆盖 | 通常不能漏掉要求覆盖的格子 |
| 路径题 | 非路径格可以空，路径格必须满足度数限制 |
| 混合最值题 | 由题目角色（障碍 / 路径 / 炮台）决定 |

### 6.3 轮廓线 DP 的 checklist

1. 先决定轮廓线有多少槽位，通常是 $m+1$。
2. 写清楚每个槽位的编码意义：0/1，还是 0/1/2，还是连通块编号。
3. 明确当前格能否空、能否放、放完后是否要留下右 / 下插头。
4. 如果有连通块，决定用括号表示法还是最小表示法。
5. 最后再考虑哈希、滚动数组和附加维度压缩。

## 七、终极总结与归纳

### 7.1 轮廓线动态规划的本质

**一句话本质**：把整张网格的历史压缩成“当前边界上还没处理完的信息”，然后逐格推进。

展开来说，轮廓线 DP 并不是神秘魔法，它只是在提醒你：对很多网格题而言，真正影响未来的不是整块前缀，而只是那条分界线。如果问题只问覆盖，边界上记 0/1 就够；如果问题问连通性，就要记插头如何匹配；如果问题还带颜色、数量、收益，就再把这些维度绑在轮廓线上一起维护。

### 7.2 核心要点

1. **先想轮廓线，再想格子**；当前格只看左口和上口。
2. **覆盖型与连通型要区分开**；不是所有轮廓线 DP 都是括号插头。
3. **单回路 / 单路径** 常用括号表示法，多连通块常用最小表示法。
4. **状态规范化** 是高阶题是否可做的分水岭。
5. **哈希表只存活跃状态**，几乎是插头 DP 的标配。
6. **大题往往是“结构状态 + 额外维度”一起维护**。
