## 一、什么是树链剖分

树链剖分（Heavy-Light Decomposition，简称 HLD）是一种将**树上路径问题**转化为**序列区间问题**的经典技术。其核心思想是：通过对树进行特殊的 DFS 遍历，将树上的节点重新编号，使得**任意一条从 u 到 v 的路径**都可以分解为 $O(\log n)$ 段**连续的区间**，从而借助线段树等数据结构高效维护。

**适用场景：**
- 树上路径求和/求最值/修改
- 树上子树求和/修改
- 树上 LCA 查询
- 结合各种数据结构（线段树、树状数组、平衡树等）处理树上问题

**时间复杂度：** 单次路径操作 $O(\log^2 n)$，子树操作 $O(\log n)$。

## 二、核心思想与前置知识

### 2.1 重儿子与轻儿子

对于树上每个非叶节点 $u$，定义：

| 概念 | 定义 |
|------|------|
| **重儿子（heavy child）** | $u$ 的所有儿子中，子树大小最大的那个儿子 |
| **轻儿子（light child）** | $u$ 的除重儿子外的其他儿子 |
| **重边（heavy edge）** | 连接 $u$ 与其重儿子的边 |
| **轻边（light edge）** | 连接 $u$ 与其轻儿子的边 |
| **重链（heavy chain）** | 由重边首尾相连形成的极长路径 |

**关键性质：** 从任意节点到根的路径上，轻边的数量不超过 $O(\log n)$ 条。这保证了路径分解后的区间段数为 $O(\log n)$。

### 2.2 两遍 DFS

树链剖分需要进行两遍 DFS：

**第一遍 DFS（`dfs1`）：** 求出每个节点的深度 `dep`、父节点 `fa`、子树大小 `sz`、重儿子 `son`。

**第二遍 DFS（`dfs2`）：** 优先遍历重儿子，求出每个节点的 DFS 序 `dfn`（即新编号）、所在重链的链顶 `top`。

这样做的关键在于：**同一条重链上的节点在 DFS 序中是连续的**，从而可以用线段树区间操作。

### 2.3 DFS 序映射到线段树

经过 `dfs2` 后，每个节点 $u$ 获得一个新编号 `dfn[u]`。我们将节点权值按 `dfn` 顺序排列，建立线段树。

- **路径操作**：将路径拆分为若干连续区间，对线段树进行区间操作
- **子树操作**：节点 $u$ 的子树对应 DFS 序中 $[\text{dfn}[u],\ \text{dfn}[u] + \text{sz}[u] - 1]$ 的连续区间

### 2.4 路径查询与修改

查询/修改 $u \to v$ 路径的流程：

1. 若 $u$ 和 $v$ 不在同一条重链上，选取链顶较深的那个节点（设为 $u$），对区间 $[\text{dfn}[\text{top}[u]],\ \text{dfn}[u]]$ 进行操作，然后令 $u \leftarrow \text{fa}[\text{top}[u]]$
2. 重复步骤 1，直到 $u$ 和 $v$ 在同一条重链上
3. 对区间 $[\text{dfn}[u],\ \text{dfn}[v]]$（保证 $\text{dfn}[u] \le \text{dfn}[v]$）进行最后一次操作

每次循环至少跳过一条轻边，因此总共操作 $O(\log n)$ 次，每次线段树操作 $O(\log n)$，总复杂度 $O(\log^2 n)$。

### 2.5 C++ 模板

```cpp
int dep[N], fa[N], sz[N], son[N];
int dfn[N], top[N], rnk[N], cnt;

void dfs1(int u, int f, int d) {
    dep[u] = d; fa[u] = f; sz[u] = 1; son[u] = 0;
    int maxsz = 0;
    for (int v : adj[u]) {
        if (v == f) continue;
        dfs1(v, u, d + 1);
        sz[u] += sz[v];
        if (sz[v] > maxsz) { maxsz = sz[v]; son[u] = v; }
    }
}

void dfs2(int u, int t) {
    top[u] = t; dfn[u] = ++cnt; rnk[cnt] = u;
    if (son[u]) dfs2(son[u], t);
    for (int v : adj[u]) {
        if (v == fa[u] || v == son[u]) continue;
        dfs2(v, v);
    }
}

void pathUpdate(int u, int v, int val) {
    while (top[u] != top[v]) {
        if (dep[top[u]] < dep[top[v]]) swap(u, v);
        segUpdate(1, dfn[top[u]], dfn[u], val);
        u = fa[top[u]];
    }
    if (dep[u] > dep[v]) swap(u, v);
    segUpdate(1, dfn[u], dfn[v], val);
}

long long pathQuery(int u, int v) {
    long long res = 0;
    while (top[u] != top[v]) {
        if (dep[top[u]] < dep[top[v]]) swap(u, v);
        res += segQuery(1, dfn[top[u]], dfn[u]);
        u = fa[top[u]];
    }
    if (dep[u] > dep[v]) swap(u, v);
    res += segQuery(1, dfn[u], dfn[v]);
    return res;
}

void subtreeUpdate(int u, int val) {
    segUpdate(1, dfn[u], dfn[u] + sz[u] - 1, val);
}
long long subtreeQuery(int u) {
    return segQuery(1, dfn[u], dfn[u] + sz[u] - 1);
}
```

## 三、分类与变体详解

### 3.1 重链剖分（Heavy-Light Decomposition）

最经典、最常用的树链剖分方式。按子树大小选择重儿子，保证任意路径拆分为 $O(\log n)$ 段连续区间。

**特点：**
- 轻边跳跃次数 $O(\log n)$
- 支持路径操作和子树操作
- 与线段树结合，单次操作 $O(\log^2 n)$

### 3.2 长链剖分（Long-Path Decomposition）

按子树深度（而非大小）选择"重儿子"——选深度最大的儿子为长儿子。

**特点：**
- 从任意节点到根经过的轻边数为 $O(\sqrt{n})$（最坏情况）
- 主要用于优化与深度相关的 DP，如合并儿子的深度信息
- 可以实现 $O(n)$ 求 $k$ 级祖先

**典型应用：** $O(n\log n)$ 预处理后 $O(1)$ 查询 $k$ 级祖先。

### 3.3 欧拉序（Euler Tour）

将树展开为 DFS 进出序列，每个节点出现两次（进入和离开）。

**特点：**
- 子树对应连续区间
- 路径操作需要配合差分或其他技巧
- 常用于离线算法和树上差分

**对比总结：**

| 方法 | 重儿子选择依据 | 路径拆分段数 | 主要应用 |
|------|---------------|-------------|---------|
| 重链剖分 | 子树大小 | $O(\log n)$ | 路径/子树修改与查询 |
| 长链剖分 | 子树深度 | $O(\sqrt{n})$ | 深度相关 DP、$k$ 级祖先 |
| 欧拉序 | — | — | 子树操作、树上差分 |

### 3.4 树链剖分 + 各类数据结构

树链剖分的本质是将树上问题转化为序列问题。因此它可以与任何序列数据结构结合：

| 搭配数据结构 | 典型应用 |
|-------------|---------|
| 线段树（基础） | 路径/子树加、求和、最值 |
| 线段树（复杂合并） | 颜色段数、函数复合 |
| 动态开点线段树 | 多类别分别维护 |
| 李超线段树 | 一次函数最值 |
| 可持久化 Trie | 区间异或最大值 |
| 树状数组 | 代码量小，配合差分 |

### 题目 1：轻重链剖分（洛谷 P3384）

🟢 入门

> 给定一棵 $n$ 个节点的树，每个节点有一个权值。支持四种操作：路径加、路径求和、子树加、子树求和。所有操作对 $p$ 取模。

**分析：**

- **数据结构**：树链剖分 + 带懒标记的线段树
- **支持操作**：路径加、路径求和、子树加、子树求和
- **关键性质**：两遍 DFS 使重链连续，路径跳链 + 线段树区间操作
- **时间复杂度**：路径操作 $O(\log^2 n)$，子树操作 $O(\log n)$

**C++ 代码：**

```cpp
long long pathSum(int u, int v) {
    long long res = 0;
    while (top[u] != top[v]) {
        if (dep[top[u]] < dep[top[v]]) swap(u, v);
        res = (res + query(1, dfn[top[u]], dfn[u])) % mod;
        u = fa[top[u]];
    }
    if (dep[u] > dep[v]) swap(u, v);
    res = (res + query(1, dfn[u], dfn[v])) % mod;
    return res;
}
long long subtreeSum(int u) {
    return query(1, dfn[u], dfn[u] + sz[u] - 1) % mod;
}
```

**核心启示：**
- 这是树链剖分的模板题，掌握两遍 DFS + 线段树框架是一切树链剖分题目的基础
- 路径操作的核心循环：跳链顶较深的一方，直到同链
- 子树操作直接利用 DFS 序的连续性

### 题目 2：树的统计（洛谷 P2590）

🟡 进阶

> 给定一棵 $n$ 个节点的树，每个节点有一个权值。支持三种操作：修改单点权值、查询路径最大值、查询路径求和。

**分析：**

- **数据结构**：树链剖分 + 线段树（同时维护和与最值）
- **支持操作**：单点修改、路径求和、路径最大值
- **关键性质**：线段树节点同时维护 sum 和 max 两个信息
- **时间复杂度**：$O(\log^2 n)$

**C++ 代码：**

```cpp
int pathMax(int u, int v) {
    int res = -INF;
    while (top[u] != top[v]) {
        if (dep[top[u]] < dep[top[v]]) swap(u, v);
        res = max(res, queryMax(1, dfn[top[u]], dfn[u]));
        u = fa[top[u]];
    }
    if (dep[u] > dep[v]) swap(u, v);
    res = max(res, queryMax(1, dfn[u], dfn[v]));
    return res;
}
```

**核心启示：**
- 线段树可以同时维护多种信息（和、最大值等）
- 路径查询时只需在跳链过程中分别合并对应信息
- 单点修改比区间修改简单，无需懒标记

### 题目 3：松鼠的新家（洛谷 P3258）

🟡 进阶

> 松鼠要按照给定顺序依次访问 $n$ 个节点。对于每条路径 $a_i \to a_{i+1}$，路径上的所有节点都需要放一个糖果（起点除外，终点包含）。求每个节点最终需要放置多少糖果。

**分析：**

- **数据结构**：树链剖分（求 LCA）+ 树上差分
- **支持操作**：多次路径加 1、最后一次汇总
- **关键性质**：树上差分 $d[u]++, d[v]++, d[\text{lca}]--, d[\text{fa}[\text{lca}]]--$ 后 DFS 求子树和
- **时间复杂度**：$O(n \log n)$

**C++ 代码：**

```cpp
// 树上差分
for (int i = 1; i < n; i++) {
    int u = a[i], v = a[i + 1];
    int l = lca(u, v);
    d[u]++; d[v]++;
    d[l]--; if (fa[l]) d[fa[l]]--;
}
// 每条路径起点多算了一次，减去
for (int i = 2; i <= n; i++) d[a[i]]--;
// DFS 求子树和
```

**核心启示：**
- 多次路径加操作可以用树上差分 $O(1)$ 处理每次修改，最后一遍 DFS 汇总
- 树链剖分在此题中主要用于高效求 LCA
- 树上差分是"离线批量路径操作"的有力武器

### 题目 4：旅行（洛谷 P3313）

🟡 进阶

> 给定一棵树，每个节点有一个宗教信仰（颜色）和评级值。支持修改节点的信仰、修改评级值、查询路径上同一信仰的评级和与最大值。

**分析：**

- **数据结构**：树链剖分 + 动态开点线段树（每种信仰一棵）
- **支持操作**：单点修改信仰/评级、路径按信仰查询和/最大值
- **关键性质**：信仰种类数有限，为每种信仰开一棵动态开点线段树
- **时间复杂度**：$O(c \log^2 n)$ per query，$c$ 为信仰种类数

**C++ 代码：**

```cpp
void modify(int &o, int l, int r, int pos, int val) {
    if (!o) o = ++tot;
    if (l == r) { mx[o] = sm[o] = val; return; }
    int mid = (l + r) >> 1;
    if (pos <= mid) modify(ls[o], l, mid, pos, val);
    else modify(rs[o], mid + 1, r, pos, val);
    mx[o] = max(mx[ls[o]], mx[rs[o]]);
    sm[o] = sm[ls[o]] + sm[rs[o]];
}
// 修改信仰：旧树删除 + 新树插入
```

**核心启示：**
- 当节点带有类别属性时，可以对每类开一棵动态开点线段树
- 修改信仰时先从旧树删除再插入新树
- 动态开点节省空间，避免为空类别浪费内存

### 题目 5：旅游（洛谷 P1505）

🟠 提高

> 给定一棵带边权的树，支持修改边权、路径求和、路径求最大值、路径求最小值、路径取反（所有边权变为相反数）。

**分析：**

- **数据结构**：树链剖分 + 线段树（维护 sum/max/min + 取反标记）
- **支持操作**：单边修改、路径求和/最大/最小、路径取反
- **关键性质**：边权下放到子节点；取反后 max 和 min 互换并取反；路径查询排除 LCA
- **时间复杂度**：$O(\log^2 n)$

**C++ 代码：**

```cpp
void negate(int o) {
    t[o].sum = -t[o].sum;
    int tmp = t[o].mx;
    t[o].mx = -t[o].mn;
    t[o].mn = -tmp;
    t[o].rev ^= 1;
}
// 路径查询排除 LCA:
// 最后一段 if (dfn[u] + 1 <= dfn[v])
//   query(1, dfn[u] + 1, dfn[v]);
```

**核心启示：**
- 边权树链剖分的关键在于"边权下放到子节点"
- 路径查询最后一段需排除 LCA 节点
- 取反操作的 lazy 设计：swap(max, min) 后各取反

### 题目 6：染色（洛谷 P2486）

🟠 提高

> 给定一棵树，每个节点有一个颜色。支持两种操作：将路径上所有节点染成同一颜色，查询路径上有多少段不同的颜色。

**分析：**

- **数据结构**：树链剖分 + 线段树（维护颜色段数和端点颜色）
- **支持操作**：路径区间赋值、路径颜色段数查询
- **关键性质**：合并两个区间时若左右端颜色相同则段数减一；路径跳链时需维护拼接处颜色
- **时间复杂度**：$O(\log^2 n)$

**C++ 代码：**

```cpp
struct Node { int cnt, lc, rc; };
Node merge(Node a, Node b) {
    if (!a.cnt) return b;
    if (!b.cnt) return a;
    return {a.cnt + b.cnt - (a.rc == b.lc), a.lc, b.rc};
}
// 路径查询：u 侧和 v 侧分别累积
// 最终 merge(reverse(ans_u), ans_v) 得到答案
```

**核心启示：**
- 颜色段数问题的核心在于区间合并时的拼接处理
- 路径跳链时要额外处理两段衔接处颜色是否相同
- 这是线段树复杂合并的经典题，也是树链剖分的经典难点

### 题目 7：LCA 查询（洛谷 P4211）

🟠 提高

> 给定一棵以 $1$ 为根的树，多次询问：$\sum_{i=l}^{r} \text{dep}[\text{lca}(i, z)]$，其中 $l, r, z$ 为给定参数。

**分析：**

- **数据结构**：树链剖分 + 线段树（离线处理）
- **支持操作**：路径加 1、路径查询和、离线差分
- **关键性质**：$\text{dep}[\text{lca}(u,v)]$ 等于 $u$ 到根路径与 $v$ 到根路径的公共节点数；用前缀和差分处理 $[l,r]$
- **时间复杂度**：$O(n \log^2 n)$

**C++ 代码：**

```cpp
void addToRoot(int u) {
    while (u) {
        update(1, dfn[top[u]], dfn[u], 1);
        u = fa[top[u]];
    }
}
long long queryToRoot(int u) {
    long long res = 0;
    while (u) {
        res += query(1, dfn[top[u]], dfn[u]);
        u = fa[top[u]];
    }
    return res;
}
// 离线：将询问挂在 r 和 l-1 处，用差分处理
```

**核心启示：**
- $\text{dep}[\text{lca}]$ 的求和可以通过"路径加、路径查"的方式离线处理
- 这一转化技巧在树上计数问题中非常常见
- 离线 + 扫描线 + 差分是处理此类问题的标准套路

### 题目 8：树上操作（洛谷 P3178）

🟡 进阶

> 给定一棵树，每个节点有权值。支持三种操作：单点加、子树加、路径求和。

**分析：**

- **数据结构**：树链剖分 + 线段树
- **支持操作**：单点加、子树加、路径求和

void pointAdd(int u, long long val) {

### 题目 9：异或（洛谷 P4592）

🔴 困难

> 给定一棵带权树，支持两类查询：给定 $x$ 和路径/子树，求路径/子树中某个节点权值与 $x$ 异或的最大值。

**分析：**

- **数据结构**：树链剖分 + 可持久化 01-Trie
- **支持操作**：路径异或最大值、子树异或最大值
- **关键性质**：按 DFS 序建可持久化 01-Trie，路径/子树对应连续区间，在 Trie 上做前缀差
- **时间复杂度**：$O(\log n \cdot \log V)$ per query（$V$ 为值域）

**C++ 代码：**

```cpp
int queryTrie(int l, int r, int x) {
    int u = root[r], v = root[l - 1], res = 0;
    for (int i = LOG - 1; i >= 0; i--) {
        int bit = (x >> i) & 1, want = bit ^ 1;
        if (cnt[ch[u][want]] - cnt[ch[v][want]] > 0) {
            res |= (1 << i);
            u = ch[u][want]; v = ch[v][want];
        } else {
            u = ch[u][bit]; v = ch[v][bit];
        }
    }
    return res;
}
// 路径查询：跳链拆成多个区间，每个区间在 Trie 上查询取 max
```

**核心启示：**
- 可持久化 01-Trie 按 DFS 序建立后，天然支持区间异或最大值查询
- 树链剖分的 DFS 序为各种可持久化数据结构提供了"序列化"基础
- 路径查询拆成多段，每段在 Trie 上查询后取最大值

### 题目 10：游戏（洛谷 P4069）

🔴 困难

> 给定一棵带边权的树，支持两种操作：将路径 $u \to v$ 上每条边的权值修改为 $\min(w_e,\ k \cdot \text{dist}(u, e) + b)$（其中 $k, b$ 为给定参数），以及查询路径上边权最小值。

**分析：**

- **数据结构**：树链剖分 + 李超线段树
- **支持操作**：路径上插入一次函数、路径最值查询
- **关键性质**：每次修改相当于在每个区间上插入一条线段（一次函数）；注意正反方向斜率不同
- **时间复杂度**：$O(\log^2 n \cdot \log V)$

**C++ 代码：**

```cpp
void insert(int o, int l, int r, int ql, int qr, Line seg) {
    int mid = (l + r) >> 1;
    if (ql <= l && r <= qr) {
        if (seg.eval(mid) < t[o].eval(mid)) swap(seg, t[o]);
        if (seg.eval(l) < t[o].eval(l))
            insert(ls, l, mid, ql, qr, seg);
        if (seg.eval(r) < t[o].eval(r))
            insert(rs, mid+1, r, ql, qr, seg);
        return;
    }
    if (ql <= mid) insert(ls, l, mid, ql, qr, seg);
    if (qr > mid) insert(rs, mid+1, r, ql, qr, seg);
}
```

**核心启示：**
- 当修改操作涉及一次函数（线段）时，李超线段树是理想选择
- 树链剖分将路径拆分为区间后，可以与李超线段树无缝结合
- 注意路径正反方向的线段斜率正负不同

### 题目 11：旧词（洛谷 P5305）

🔴 困难

> 给定一棵以 $1$ 为根的树和参数 $k$，多次询问：$\sum_{i=1}^{y} \text{dep}[\text{lca}(i, z)]^k$，其中 $y, z$ 为给定参数。

**分析：**

- **数据结构**：树链剖分 + 线段树（离线处理）
- **支持操作**：路径加差分值、路径查询和
- **关键性质**：将 $\text{dep}^k$ 差分为 $d^k - (d-1)^k$，加入节点时在路径上每层加对应差分值
- **时间复杂度**：$O(n \log^2 n)$

**C++ 代码：**

```cpp
void addNode(int u) {
    while (u) {
        int t = top[u];
        // 这一段重链深度从 dep[t] 到 dep[u]
        // 每个深度 d 加上 pw[d] - pw[d-1]
        segAdd(1, dfn[t], dfn[u], /* 差分值 */);
        u = fa[t];
    }
}
// 离线排序，顺序插入节点并回答询问
```

**核心启示：**
- 将 $\text{dep}^k$ 通过差分拆解为逐层贡献，是处理深度幂次问题的通用技巧
- 本质上利用 $d^k = \sum_{i=1}^{d} [i^k - (i-1)^k]$ 的恒等式
- 与 P4211 思路相似但需要额外的差分处理

### 题目 12：DFS 序特性：子树加与路径查（洛谷 P3833）

🟡 进阶

> 给定一棵树，支持两种操作：对以 $u$ 为根的子树所有节点加 $v$，以及查询 $u$ 到根的路径上所有节点权值之和。

**分析：**

- **数据结构**：树链剖分（DFS 序）+ 树状数组差分
- **支持操作**：子树加、路径查和
- **关键性质**：子树加可以用 DFS 序区间加实现；路径查和可以用差分转化为子树操作
- **时间复杂度**：$O(\log^2 n)$ 或 $O(\log n)$ 取决于实现

**C++ 代码：**

```cpp
// 子树加: 对 DFS 序区间 [dfn[u], dfn[u]+sz[u]-1] 加 v
void subtreeAdd(int u, long long v) {
    bitUpdate(dfn[u], v);
    bitUpdate(dfn[u] + sz[u], -v);
}
// 路径查和 u->root: 通过差分转化
// 或直接用树链剖分跳链查询
long long queryToRoot(int u) {
    long long res = 0;
    while (u) {
        res += bitPrefixQuery(dfn[u]);
        u = fa[top[u]]; // 也可直接前缀和
    }
    return res;
}
```

**核心启示：**
- 子树操作天然利用 DFS 序的连续性
- 路径查询到根可以通过差分技巧巧妙转化
- 树状数组在不需要区间修改+区间查询时是更简洁的选择

### 题目 13：树上路径最大值（SPOJ-QTREE）

🟠 提高

> 给定一棵带边权的树，支持两种操作：修改某条边的权值、查询两点路径上的最大边权。

**分析：**

- **数据结构**：树链剖分 + 线段树（维护最大值）
- **支持操作**：单边修改、路径最大边权查询
- **关键性质**：边权下放到子节点；路径查询最后一段排除 LCA
- **时间复杂度**：$O(\log^2 n)$

**C++ 代码：**

```cpp
// 边 (u, v) 的权值存在深度较大的端点处
// 修改边 (u, v) 权值为 w
void changeEdge(int u, int v, int w) {
    if (dep[u] < dep[v]) swap(u, v);
    // u 是较深的端点，边权存在 u 上
    segUpdate(1, dfn[u], dfn[u], w);
}
// 路径最大值查询（排除 LCA）
int pathMaxEdge(int u, int v) {
    int res = -INF;
    while (top[u] != top[v]) {
        if (dep[top[u]] < dep[top[v]]) swap(u, v);
        res = max(res, segQueryMax(1, dfn[top[u]], dfn[u]));
        u = fa[top[u]];
    }
    if (u == v) return res;
    if (dep[u] > dep[v]) swap(u, v);
    res = max(res, segQueryMax(1, dfn[u] + 1, dfn[v]));
    return res;
}
```

**核心启示：**
- 边权转点权是树链剖分处理边权问题的标准技巧
- 查询时最后一段区间为 $[\text{dfn}[u]+1, \text{dfn}[v]]$，排除 LCA 自身
- 这是树上边权问题的经典模板

### 题目 14：树上换根查询（洛谷 P3979）

🔴 困难

> 给定一棵树，支持三种操作：换根（将某个节点设为新的根）、路径区间加、子树查询最小值。

**分析：**

- **数据结构**：树链剖分 + 线段树
- **支持操作**：换根、路径加、子树最小值
- **关键性质**：不真正换根，而是根据当前根与查询点的关系判断实际子树范围。设当前根为 $rt$，查询 $u$ 的子树时分三种情况讨论
- **时间复杂度**：$O(\log^2 n)$

**C++ 代码：**

```cpp
// 换根后查询 u 的子树最小值
long long subtreeMinWithRoot(int u, int rt) {
    if (u == rt) return segQueryMin(1, 1, n); // 整棵树
    if (!isAncestor(u, rt))
        return segQueryMin(1, dfn[u], dfn[u] + sz[u] - 1);
    // rt 在 u 的子树内，找 u->rt 路径上 u 的儿子 v
    int v = jumpTo(rt, u); // u 朝 rt 方向走一步
    // 答案 = 全树减去 v 的子树
    return min(segQueryMin(1, 1, dfn[v] - 1),
               segQueryMin(1, dfn[v] + sz[v], n));
}
bool isAncestor(int u, int v) {
    return dfn[u] <= dfn[v] && dfn[v] < dfn[u] + sz[u];
}
```

**核心启示：**
- 换根问题不需要真正重新剖分，而是通过分类讨论确定实际子树范围
- 三种情况：查询点是根、查询点不是根的祖先、查询点是根的祖先
- 这是树链剖分中经典的"虚拟换根"技巧

### 题目 15：由乃的 OJ（洛谷 P5354）

🟣 竞赛

> 给定一棵树，每个节点有一个 $k$ 位二进制数和一个操作符（AND/OR/XOR 之一）。从 $u$ 到 $v$ 的路径上，将一个初始值依次经过每个节点的操作后得到结果。支持单点修改和路径查询。

**分析：**

- **数据结构**：树链剖分 + 线段树（维护 $\{0,1\} \to \{0,1\}$ 函数复合）
- **支持操作**：单点修改、路径函数复合查询
- **关键性质**：位运算按位独立；每个操作抽象为 (val0, val1) 二元组；正向和反向复合需分别维护
- **时间复杂度**：$O(k \cdot \log^2 n)$

**C++ 代码：**

```cpp
struct Func {
    long long v0, v1;
};
Func compose(Func f, Func g) {
    return {(~f.v0 & g.v0) | (f.v0 & g.v1),
            (~f.v1 & g.v0) | (f.v1 & g.v1)};
}
// 线段树维护正向和反向复合
// 路径查询时 u 侧用反向，v 侧用正向
// 最后按位贪心选择初始值
```

**核心启示：**
- 位运算问题的精髓在于"按位独立 + 函数复合"
- 将每个操作抽象为 $\{0,1\} \to \{0,1\}$ 的映射后，即可用线段树维护复合结果
- 路径查询时注意方向：u 侧反向、v 侧正向

### 题目 16：边上的颜色段（洛谷 P5499）

🟣 竞赛

> 给定一棵树，每条边有一个颜色和权值。支持修改边的颜色和权值、查询路径上同一颜色的连续段的权值之和的最大值。

**分析：**

- **数据结构**：树链剖分 + 线段树（复杂信息合并）
- **支持操作**：单边修改颜色/权值、路径最大同色段权值和
- **关键性质**：线段树维护 ans/lc/lw/rc/rw 五个量；合并时若颜色相同可拼接；需维护正反向
- **时间复杂度**：$O(\log^2 n)$

**C++ 代码：**

```cpp
struct Info {
    long long ans, lw, rw;
    int lc, rc;
};
Info merge(Info a, Info b) {
    Info res;
    res.lc = a.lc; res.rc = b.rc;
    res.lw = a.lw; res.rw = b.rw;
    res.ans = max(a.ans, b.ans);
    if (a.rc == b.lc) {
        res.ans = max(res.ans, a.rw + b.lw);
        if (a.lc == a.rc) res.lw = a.lw + b.lw;
        if (b.lc == b.rc) res.rw = a.rw + b.rw;
    }
    return res;
}
```

**核心启示：**
- 复杂的区间合并问题需要在线段树节点中维护足够的信息以支持拼接
- 路径查询在这种场景下需要格外注意合并顺序和方向
- 这是树链剖分高级应用的巅峰，综合考察信息合并与方向处理

## 五、实用技巧与注意事项

### 技巧 1：边权转点权

树链剖分天然处理点权。对于边权问题，将每条边的权值**下放到深度较大的端点**。查询路径 $u \to v$ 时，最后一段区间需排除 LCA 节点，即区间为 $[\text{dfn}[u]+1, \text{dfn}[v]]$。

### 技巧 2：路径方向处理

当维护的信息不满足交换律时（如颜色段数、函数复合），需要注意路径方向。跳链时 $u$ 侧是反向的（从深处跳到浅处），$v$ 侧是正向的。可以分别维护两侧的结果，最后拼接。

### 技巧 3：线段树信息合并

树链剖分的路径查询会产生多段区间，需要将它们合并。若合并操作需要考虑相邻段的衔接（如颜色段数），建议封装线段树查询返回结构体，支持 `merge` 操作。

### 技巧 4：DFS 序与子树操作

节点 $u$ 的子树对应 DFS 序中 $[\text{dfn}[u],\ \text{dfn}[u] + \text{sz}[u] - 1]$ 的连续区间，可以直接用线段树区间操作。

### 技巧 5：利用树链剖分求 LCA

```cpp
int lca(int u, int v) {
    while (top[u] != top[v]) {
        if (dep[top[u]] < dep[top[v]]) swap(u, v);
        u = fa[top[u]];
    }
    return dep[u] < dep[v] ? u : v;
}
```

时间复杂度 $O(\log n)$，常数极小，实际效率往往优于倍增法。

### 技巧 6：虚拟换根

换根问题不需要重新剖分。设原根为 $1$，当前虚拟根为 $rt$，查询 $u$ 的子树时分三种情况：
- $u = rt$：子树为整棵树
- $u$ 不是 $rt$ 的祖先：子树不变
- $u$ 是 $rt$ 的祖先：子树为全树减去 $u$ 朝 $rt$ 方向的儿子的子树

```
Step 1: 判断是否适用树链剖分
├── 涉及路径修改/查询？ → 很可能适用
├── 涉及子树修改/查询？ → 适用（DFS序即可）
├── 需要高效 LCA？ → 适用
└── 结合序列数据结构？ → 适用

├── 点权 → 直接处理
└── 边权 → 下放到子节点，路径查询排除 LCA

Step 3: 确定线段树需要维护的信息
├── 简单的和/最值 → 基础线段树
├── 区间赋值/加 → 带懒标记的线段树
├── 颜色段数 → 需要维护端点信息的线段树

├── 满足交换律/结合律 → 直接累加
└── 不满足交换律 → 分别维护 u 侧和 v 侧，注意方向

Step 5: 是否需要离线处理
├── 强制在线 → 在线维护
└── 可以离线 → 考虑差分、排序等优化
```

## 七、终极总结与归纳

### 7.1 树链剖分的本质

**树链剖分 = 巧妙的 DFS 序 + 重链连续性。** 通过优先遍历重儿子，使得同一条重链上的节点 DFS 序连续，从而将树上路径问题转化为至多 $O(\log n)$ 段的序列区间问题，再借助线段树等数据结构高效维护。

### 7.2 核心要点

1. **两遍 DFS** 是树链剖分的基础：第一遍求重儿子和子树信息，第二遍求 DFS 序和链顶
2. **重链连续性** 是关键性质：同一条重链上的节点 DFS 序连续
3. **路径拆分** 保证 $O(\log n)$ 段：每跳过一条轻边，子树大小至少翻倍
4. **总复杂度** $O(\log^2 n)$：$O(\log n)$ 次跳链，每次线段树操作 $O(\log n)$
5. **边权转点权** 是处理边权问题的标准技巧，查询时排除 LCA
6. **路径方向** 在非交换律操作中必须分别维护
