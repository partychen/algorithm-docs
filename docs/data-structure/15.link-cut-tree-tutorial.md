## 一、什么是 LCT

**Link-Cut Tree（LCT）** 是 Sleator 和 Tarjan 于 1983 年提出的一种用于维护**动态森林**的数据结构。它支持在 $O(\log n)$ 均摊时间内完成以下操作：

| 操作 | 含义 |
|------|------|
| `link(u, v)` | 在 $u$ 和 $v$ 之间连一条边 |
| `cut(u, v)` | 删除 $u$ 和 $v$ 之间的边 |
| `findroot(u)` | 查询 $u$ 所在树的根 |
| `split(u, v)` | 提取 $u$ 到 $v$ 的路径信息 |
| `access(u)` | 打通 $u$ 到根的路径 |
| `makeroot(u)` | 将 $u$ 设为所在树的根 |

**适用场景：** 当树的形态会动态变化（加边、删边），需要在线维护路径信息时，LCT 是最强大的工具之一。

与树链剖分（HLD）相比：
- HLD 只能处理**静态树**上的路径查询与修改
- LCT 能处理**动态森林**中的连边、断边、路径查询

## 二、核心思想与前置知识

### 2.1 前置知识

学习 LCT 之前需要牢固掌握以下内容：

1. **Splay 树**：旋转、伸展操作、均摊分析
2. **树链剖分**：重链/轻链的概念，路径分解
3. **懒标记机制**：翻转标记的下传
4. **树的基本操作**：DFS、LCA、路径查询

### 2.2 辅助 Splay 树

原树中的每个节点都存在于某棵**辅助 Splay 树**中。一条从根到叶子的路径上，被选中的"偏爱路径"用一棵 Splay 来维护。

**关键性质：**
- 每棵 Splay 内部按**深度**为关键字排序（中序遍历即从浅到深）
- **实边（preferred edge）**：父子之间的 Splay 内部边，`ch[fa][0/1] == x`
- **虚边（non-preferred edge）**：`fa[x]` 指向 Splay 的根的父节点，但父节点**不认**这个子节点

```
原树:          辅助 Splay 表示:
    1              Splay1: {1, 2, 5}  (实链)
   / \             Splay2: {3}
  2   3            Splay3: {4}
  |   |
  5   4
```

判断节点 $x$ 是否为某棵 Splay 的根：

$$
\text{isRoot}(x) = (ch[fa[x]][0] \neq x) \land (ch[fa[x]][1] \neq x)
$$

### 2.3 access 操作

`access(x)` 是 LCT 最核心的操作——将 $x$ 到其所在树的根节点的路径全部变成**实边**，并且 $x$ 成为该实链 Splay 中**深度最大**的节点（即没有右子树）。

```
access(x) 的过程：
1. 从 x 出发，不断向上跳到 Splay 的根
2. 将当前 Splay 根与上一层 Splay 通过"实边"连接
3. 原来的实儿子变为虚边（断开右子树）
4. 最终 x 被 splay 到整条链的 Splay 根
```

**伪代码流程：**

```
access(x):
    last = 0
    while x != 0:
        splay(x)
        ch[x][1] = last   // 原右子树变虚，last 变实
        pushup(x)
        last = x
        x = fa[x]
    // 结束后 last 是整条链的 Splay 根
```

**时间复杂度：** 均摊 $O(\log n)$（利用 Splay 的势能分析）。

### 2.4 makeroot / link / cut / findroot

#### makeroot(x)：换根

将 $x$ 设为其所在原树的根：

```
makeroot(x):
    access(x)
    splay(x)
    // 此时 x 是 Splay 根且无右子树
    // 翻转整棵 Splay，使 x 从最深变最浅（即成为根）
    reverse(x)  // 打翻转标记
```

核心原理：access 后 $x$ 是深度最大的节点，翻转后变成深度最小，即成为根。

#### link(x, y)：连边

```
link(x, y):
    makeroot(x)
    if findroot(y) == x: return  // 已连通，不操作
    fa[x] = y  // x 作为 y 的虚儿子
```

#### cut(x, y)：断边

```
cut(x, y):
    makeroot(x)
    access(y)
    splay(y)
    if ch[y][0] == x and fa[x] == y and ch[x][1] == 0:
        ch[y][0] = 0
        fa[x] = 0
        pushup(y)
```

#### findroot(x)：找根

```
findroot(x):
    access(x)
    splay(x)
    while ch[x][0]:
        pushdown(x)
        x = ch[x][0]
    splay(x)  // 保证均摊复杂度
    return x
```

### 2.5 C++ 完整模板

```cpp
const int N = 100005;
int ch[N][2], fa[N], val[N], s[N], rev[N], stk[N];

bool isRoot(int x) {
    return ch[fa[x]][0] != x && ch[fa[x]][1] != x;
}

void pushup(int x) {
    s[x] = s[ch[x][0]] ^ s[ch[x][1]] ^ val[x];
}

void pushrev(int x) {
    swap(ch[x][0], ch[x][1]);
    rev[x] ^= 1;
}

void pushdown(int x) {
    if (rev[x]) {
        pushrev(ch[x][0]);
        pushrev(ch[x][1]);
        rev[x] = 0;
    }
}

void rotate(int x) {
    int y = fa[x], z = fa[y];
    int k = (ch[y][1] == x), w = ch[x][!k];
    if (!isRoot(y)) ch[z][ch[z][1] == y] = x;
    ch[x][!k] = y; ch[y][k] = w;
    if (w) fa[w] = y;
    fa[y] = x; fa[x] = z;
    pushup(y); pushup(x);
}

void splay(int x) {
    int top = 0, u = x;
    stk[++top] = u;
    while (!isRoot(u)) stk[++top] = u = fa[u];
    while (top) pushdown(stk[top--]);
    while (!isRoot(x)) {
        int y = fa[x], z = fa[y];
        if (!isRoot(y))
            rotate((ch[y][0]==x)^(ch[z][0]==y) ? x : y);
        rotate(x);
    }
    pushup(x);
}

void access(int x) {
    for (int y = 0; x; y = x, x = fa[x]) {
        splay(x);
        ch[x][1] = y;
        pushup(x);
    }
}

void makeroot(int x) {
    access(x); splay(x); pushrev(x);
}

int findroot(int x) {
    access(x); splay(x);
    while (ch[x][0]) pushdown(x), x = ch[x][0];
    splay(x);
    return x;
}

void split(int x, int y) {
    makeroot(x); access(y); splay(y);
}

void link(int x, int y) {
    makeroot(x);
    if (findroot(y) != x) fa[x] = y;
}

void cut(int x, int y) {
    makeroot(x); access(y); splay(y);
    if (ch[y][0] == x && !ch[x][1]) {
        ch[y][0] = fa[x] = 0;
        pushup(y);
    }
}
```

## 三、分类与变体详解

### 3.1 基本 LCT（路径信息维护）

维护**路径上的信息**，如路径异或和、路径最值、路径求和等。这是最常见的 LCT 用法。

- 只需在 `pushup` 中维护相关信息
- 利用 `split(x, y)` 将 $x$ 到 $y$ 的路径提取出来后，直接查询 Splay 根节点的汇总值

### 3.2 维护子树信息的 LCT

基本 LCT 只能维护路径信息。若需要维护**子树大小、子树和**等信息，需要额外记录"虚子树"的贡献。

核心思想：对每个节点额外维护 `si[x]` 表示所有**虚儿子**子树的信息汇总。

$$
\text{total}(x) = \text{si}(x) + \text{total}(\text{ch}[x][0]) + \text{total}(\text{ch}[x][1]) + \text{val}(x)
$$

需要在 `access` 和 `link` 中维护 `si` 的变化：
- `access` 中替换实儿子时，旧实儿子的贡献加入 `si`，新实儿子的贡献从 `si` 中减去
- `link` 时将 $x$ 的贡献加入 `fa[x]` 的 `si`

### 3.3 LCT 维护边权

LCT 的 Splay 以节点为单位维护，无法直接处理边权。标准做法：

- 为每条边 $(u,v)$ 建立一个**虚拟节点** $e$
- 将 `link(u, e)` 和 `link(e, v)` 代替原本的 `link(u, v)`
- 边权存储在虚拟节点上
- pushup 时虚拟节点参与最值/求和等统计

### 3.4 Euler Tour Tree（ETT）

ETT 是另一种维护动态森林的数据结构，基于欧拉游览序。与 LCT 的对比：

| 特性 | LCT | ETT |
|------|-----|-----|
| 路径操作 | 擅长 | 不擅长 |
| 子树操作 | 需额外维护 | 天然支持 |
| 实现难度 | 较高 | 较高 |
| 常数 | 较大 | 中等 |

在实际竞赛中，LCT 使用频率远高于 ETT。

### 3.5 LCT + 其他数据结构

- **LCT + DFS 序线段树**：如 P3703 树点涂色，LCT 管理染色链，DFS 序线段树维护全局统计
- **LCT + 并查集**：动态连通性的线段树分治方案中，可撤销并查集是 LCT 的轻量替代
- **LCT + 最小生成树**：按一个维度排序，LCT 动态维护另一个维度的最优生成森林

### 题目 1：LCT 模板（洛谷 P3690）

> 给定 $n$ 个节点，每个节点有权值。支持操作：连边、断边、修改点权、查询路径异或和。

**分析**：路径异或和是 LCT 最经典的应用。`pushup` 中维护子树异或和，`split(x,y)` 后直接读取 Splay 根的信息。

```
数据结构：LCT（辅助 Splay 树）
支持操作：link、cut、修改点权、查询路径异或和
关键性质：pushup 维护异或和，split 提取路径
时间复杂度：均摊 O(log n) 每次操作
```

**C++ 代码**：

```cpp
void modify(int x, int v) {
    splay(x);
    val[x] = v;
    pushup(x);
}

int query(int x, int y) {
    split(x, y);
    return s[y];
}
```

**核心启示**：
- 这是 LCT 的标准模板题，掌握此题即掌握 LCT 的基本框架
- 路径异或和只需在 pushup 中完成
- link/cut 前需检查连通性和边的存在性

### 题目 2：OTOCI（洛谷 P4312）

> 给定 $n$ 个城市，每个有权值。支持连边、修改点权、查询两点路径权值和（若不连通输出 impossible）。

**分析**：基础 LCT 加上连通性判断。pushup 维护路径和，用 findroot 判断连通性。

```
数据结构：LCT
支持操作：link、修改点权、查询路径和 + 连通性判断
关键性质：findroot 天然判断连通性
时间复杂度：均摊 O(log n)
```

**C++ 代码**：

```cpp
long long query(int x, int y) {
    makeroot(x);
    if (findroot(y) != x) return -1;
    split(x, y);
    return s[y];
}

void modify(int x, long long v) {
    splay(x);
    val[x] = v;
    pushup(x);
}
```

**核心启示**：
- LCT 天然支持连通性判断（findroot）
- 对于"动态加边 + 路径查询"问题非常方便
- 注意此题只有连边没有断边

### 题目 3：弹飞绵羊（洛谷 P3203）

> $n$ 个弹力装置排成一排，第 $i$ 个装置会弹飞到 $i + k_i$。支持修改某个装置的弹力系数，查询从某个装置出发经过多少次弹射飞出序列。

**分析**：每个装置 $i$ 向 $i + k_i$ 连边（若超出范围则连向虚拟节点 $n+1$），形成一棵树。查询即为 $i$ 到根的路径长度。修改即为断旧边连新边。

```
数据结构：LCT（维护子树大小）
支持操作：cut + link（修改弹力系数）、路径长度查询
关键性质：弹射链建模为树边，动态修改等价于 cut + link
时间复杂度：均摊 O(log n)
```

**C++ 代码**：

```cpp
void pushup(int x) {
    sz[x] = sz[ch[x][0]] + sz[ch[x][1]] + 1;
}

int query(int x) {
    split(x, n + 1);
    return sz[n + 1] - 1;
}

void modify(int x, int newk) {
    cut(x, min(x + k[x], n + 1));
    k[x] = newk;
    link(x, min(x + k[x], n + 1));
}
```

**核心启示**：
- 将"弹射链"建模为树边，动态修改弹力系数等价于 cut + link
- LCT 天然适合这种"链式跳跃 + 动态修改"问题
- 虚拟节点 $n+1$ 作为所有弹出序列的汇聚点

### 题目 4：大融合（洛谷 P4219）

> 动态加边（保证始终是森林），查询一条边两端分别有多少节点（即删掉这条边后两个连通块的大小之积）。

**分析**：需要维护**子树大小**，这是 LCT 维护子树信息的经典应用。需要额外维护虚子树大小 `si[x]`。

```
数据结构：LCT（维护子树信息）
支持操作：link、查询边两端子树大小之积
关键性质：si[x] 记录虚儿子子树大小之和，access/link 中更新
时间复杂度：均摊 O(log n)
```

**C++ 代码**：

```cpp
void pushup(int x) {
    sz[x] = sz[ch[x][0]] + sz[ch[x][1]] + si[x] + 1;
}

void access(int x) {
    for (int y = 0; x; y = x, x = fa[x]) {
        splay(x);
        si[x] += sz[ch[x][1]];
        si[x] -= sz[y];
        ch[x][1] = y;
        pushup(x);
    }
}

long long query(int u, int v) {
    makeroot(u); access(v); splay(v);
    return (long long)sz[u] * (sz[v] - sz[u]);
}
```

**核心启示**：
- 维护子树信息的关键在于额外记录"虚子树"贡献 si[x]
- 在 access 和 link 中正确更新 si 是核心难点

### 🟡 进阶级 Level 2：LCT 经典应用

### 题目 5：魔法森林（洛谷 P2387）

> 给定一个无向图，每条边有两个权值 $a_i, b_i$。求从 $1$ 到 $n$ 的路径，使路径上 $\max(a_i) + \max(b_i)$ 最小。

**分析**：将边按 $a$ 排序后依次加入。每加入一条边时，若两端点已连通，检查路径上 $b$ 的最大值是否大于当前边的 $b$，若是则替换。这是 LCT 维护最小生成树的经典应用。

```
数据结构：LCT（维护路径最大值，边权转点权）
支持操作：动态加边/替换边、路径最大值查询
关键性质：按 a 排序逐步加入，LCT 动态维护 b 的最小瓶颈生成树
时间复杂度：O(m log m)
```

**C++ 代码**：

```cpp
sort(edges, edges + m, [](auto& a, auto& b) { return a.a < b.a; });
int ans = INF;
for (int i = 0; i < m; i++) {
    int u = e[i].u, v = e[i].v, id = n + i + 1;
    val[id] = e[i].b;
    if (findroot(u) == findroot(v)) {
        split(u, v);
        int mx = mx_id[v];
        if (val[mx] <= e[i].b) continue;
        cut(mx, edges[mx-n-1].u); cut(mx, edges[mx-n-1].v);
    }
    link(u, id); link(id, v);
    if (findroot(1) == findroot(n)) {
        split(1, n); ans = min(ans, e[i].a + val[mx_id[n]]);
    }
}
```

**核心启示**：
- LCT 维护动态最小生成树：按一个维度排序，用 LCT 动态维护另一个维度的最优值
- 边权转点权是处理边权问题的标准技巧

### 题目 6：Tree II（洛谷 P1501）

> 给定一棵树，支持：路径加、路径乘、查询路径和（模意义下），以及连边、断边。

**分析**：在 LCT 上维护路径和，且有加法和乘法两种懒标记。标记合并的优先级：先乘后加。

```
数据结构：LCT（双重懒标记：乘法 + 加法 + 翻转）
支持操作：路径加、路径乘、路径和查询、link、cut
关键性质：先乘后加的标记合并优先级
时间复杂度：均摊 O(log n)
```

**C++ 代码**：

```cpp
void pushdown(int x) {
    if (rev[x]) { pushrev(ch[x][0]); pushrev(ch[x][1]); rev[x] = 0; }
    if (mul[x] != 1 || add[x] != 0) {
        for (int c : {ch[x][0], ch[x][1]}) if (c) {
            s[c] = (s[c] * mul[x] + add[x] * sz[c]) % P;
            val[c] = (val[c] * mul[x] + add[x]) % P;
            mul[c] = mul[c] * mul[x] % P;
            add[c] = (add[c] * mul[x] + add[x]) % P;
        }
        mul[x] = 1; add[x] = 0;
    }
}
```

**核心启示**：
- 当有多种懒标记时，必须明确优先级（先乘后加）
- LCT 上的懒标记设计与线段树完全类似，只是多了翻转标记的处理

### 题目 7：水管局长（洛谷 P4172）

> 给定一个无向图，边有权值。支持删边操作，每次查询两点间路径上边权最大值的最小值（即最小瓶颈路）。

**分析**：最小瓶颈路在最小生成树上取得。删边不好处理，考虑**离线倒序**变成加边。用 LCT 动态维护最小生成树。

```
数据结构：LCT（维护路径最大值，边权转点权）
支持操作：动态加边/替换边、路径最大值查询
关键性质：离线倒序将"删边"变为"加边"
时间复杂度：O(m log m)
```

**C++ 代码**：

```cpp
void add_edge(int u, int v, int w, int id) {
    int eid = n + id;
    val[eid] = w;
    if (findroot(u) != findroot(v)) {
        link(u, eid); link(eid, v);
    } else {
        split(u, v);
        if (val[mx_id[v]] > w) {
            int old = mx_id[v];
            cut(old, e[old-n].u); cut(old, e[old-n].v);
            link(u, eid); link(eid, v);
        }
    }
}

int query(int u, int v) {
    split(u, v);
    return val[mx_id[v]];
}
```

**核心启示**：
- "删边"操作通常通过**离线倒序变加边**来处理
- LCT 动态维护最小生成树是解决最小瓶颈路问题的标准方法

### 题目 8：二分图（洛谷 P4234 / BZOJ 4025）

> 给定一个动态图，边有存在的时间区间 $[l_i, r_i]$。对每个时刻判断当前图是否为二分图。

**分析**：使用线段树分治 + 可撤销并查集（或 LCT）。将每条边按时间区间挂到线段树节点上，分治过程中加入/撤销边，用带权并查集判断奇偶环。

```
数据结构：线段树分治 + 可撤销带权并查集（或 LCT）
支持操作：动态加边/撤销、二分图判定
关键性质：二分图等价于无奇环，带权并查集维护奇偶性
时间复杂度：O(m log m log n)
```

**C++ 代码**：

```cpp
int fa[MAXN], rnk[MAXN], dist[MAXN]; // dist: 到根的距离(mod 2)
struct StackItem { int u, v, old_fa, old_rnk, old_dist; };
vector<StackItem> stk;
bool is_bipartite = true;

int find(int x, int &d) {
    d = 0;
    while (fa[x] != x) { d ^= dist[x]; x = fa[x]; }
    return x;
}

void unite(int u, int v) {
    int du, dv;
    int fu = find(u, du), fv = find(v, dv);
    if (fu == fv) {
        if (du == dv) is_bipartite = false; // 奇环
        return;
    }
    if (rnk[fu] < rnk[fv]) swap(fu, fv), swap(du, dv);
    stk.push_back({fv, fu, fa[fv], rnk[fu], dist[fv]});
    fa[fv] = fu;
    dist[fv] = du ^ dv ^ 1;
    if (rnk[fu] == rnk[fv]) rnk[fu]++;
}
```

**核心启示**：
- 线段树分治框架将"边有时间区间"的问题优雅地分解
- 可撤销并查集（按秩合并、不路径压缩）是 LCT 在此场景的轻量替代
- 二分图判定等价于无奇环，通过带权并查集维护奇偶性

### 题目 11：最小差值生成树（洛谷 P4234 / BZOJ 3669）

> 给定一个无向图，求一棵生成树使得最大边权与最小边权之差最小。

**分析**：将边按权值排序后用 LCT 维护。依次加入边，若成环则删除环上权值最小的边。维护当前树中最小边权，答案为当前最大边权 - 当前最小边权。

```
数据结构：LCT（维护路径最小值，边权转点权）
支持操作：动态加边/替换边、路径最小值查询
关键性质：按权值排序加入，删除最小边保持连通性
时间复杂度：O(m log m)
```

**C++ 代码**：

```cpp
sort(edges, edges + m);
int ans = INF;
multiset<int> edge_set; // 当前树中所有边的权值
for (int i = 0; i < m; i++) {
    int u = e[i].u, v = e[i].v, w = e[i].w, id = n + i + 1;
    val[id] = w; mn_id[id] = id;
    if (findroot(u) == findroot(v)) {
        split(u, v);
        int old = mn_id[v]; // 路径上权值最小的边
        edge_set.erase(edge_set.find(val[old]));
        cut(old, edges[old-n-1].u); cut(old, edges[old-n-1].v);
    }
    link(u, id); link(id, v);
    edge_set.insert(w);
    if ((int)edge_set.size() == n - 1)
        ans = min(ans, *edge_set.rbegin() - *edge_set.begin());
}
```

**核心启示**：
- 最小差值生成树与最小瓶颈路类似，核心是 LCT 动态维护生成树

```

支持操作：换根、路径赋值、子树最小值查询
关键性质：换根后子树查询需要分类讨论（根在子树内/外）
时间复杂度：O(m log^2 n)
```

**C++ 代码**：

```cpp
int cur_root = 1;

int subtree_min(int u) {
    if (u == cur_root) return seg_query(1, 1, n, 1, n);
    int lca = LCA(u, cur_root);
    if (lca != u)
        return seg_query(1, 1, n, dfn[u], dfn[u] + sz[u] - 1);
    // 根在 u 的子树内，需要找到根方向的儿子 v
    int v = jump(cur_root, dep[cur_root] - dep[u] - 1);
    // 答案 = 全局 - v 的子树
    return min(seg_query(1, 1, n, 1, dfn[v] - 1),
               seg_query(1, 1, n, dfn[v] + sz[v], n));
}
```

**核心启示**：
- 换根操作的核心是分类讨论：根在查询节点的子树内还是子树外
- LCT 的 makeroot 可以真正换根，但在某些题目中树链剖分 + 虚拟换根更高效

### 题目 14：动态图连通性 线段树分治（洛谷 P5489 / CF100551A）

> 给定一个动态图，支持加边、删边，查询两点是否连通。边的删除是任意的。

**分析**：动态图连通性是经典难题。使用离线线段树分治 + LCT（或可撤销并查集）。将每条边的存在时间拆成线段树上的若干区间，在线段树上分治处理。

```
数据结构：线段树分治 + LCT / 可撤销并查集
支持操作：动态加边/删边、连通性查询
关键性质：每条边的存在区间挂到时间线段树上，分治处理
时间复杂度：O(m log m log n)
```

**C++ 代码**：

```cpp
void solve(int node, int l, int r) {
    int prev_size = stk.size();
    for (auto [u, v] : seg[node]) {
        if (findroot(u) != findroot(v))
            link(u, v), stk.push({u, v, 1});
        else
            stk.push({u, v, 0});
    }
    if (l == r) {
        for (auto [u, v] : queries[l])
            ans[qid] = (findroot(u) == findroot(v));
    } else {
        int mid = (l + r) / 2;
        solve(2*node, l, mid);
        solve(2*node+1, mid+1, r);
    }
    while ((int)stk.size() > prev_size) {
        auto [u, v, linked] = stk.top(); stk.pop();
        if (linked) cut(u, v);
    }
}
```

**核心启示**：
- 动态图连通性用线段树分治框架解决最为优雅
- LCT 的 link/cut 天然支持可撤销操作，与分治框架完美配合

### 题目 15：维护树上路径最值（洛谷 P2542 / BZOJ 2594 航线规划）

> 给定一个无向图，支持删边操作，每次查询两点间所有路径中必须经过的边的数量（即"桥"的数量）。

```
数据结构：LCT + 缩点（边双连通分量维护）
支持操作：动态加边、路径上桥的计数
关键性质：离线倒序加边，加边成环则路径上所有边不再是桥
时间复杂度：O(m log n)
```

**C++ 代码**：

```cpp
// 加入边 (u, v)，若已连通则缩路径为一点
void add_edge(int u, int v) {
    u = find(u); v = find(v);
    if (u == v) return;
    if (findroot(u) != findroot(v)) {
        link(u, v); // 新桥
    } else {
        // 路径上的所有边和点缩成一个超级节点
        split(u, v);
        compress_path(v); // 路径上所有节点合并到 v
    }
}

int query(int u, int v) {
    u = find(u); v = find(v);
    split(u, v);
    return sz[v] - 1; // 路径上的边数 = 桥的数量
}
```

**核心启示**：
- 边双连通分量的动态维护需要 LCT + 并查集缩点配合
- "桥"的数量等于缩点后树上路径的边数

### 题目 16：共价大爷游长沙（洛谷 P1/UOJ 207）

> 给定一棵 $n$ 个节点的树和 $m$ 条路径（起终点对）。支持动态修改树的结构（cut + link）和修改路径集合（加入/删除路径），每次操作后查询某条特定边是否被所有路径经过。

**分析**：对每条路径随机赋一个权值，路径的端点各异或上该权值。一条边 $(u,v)$ 被所有路径经过，等价于断开该边后某一侧子树的异或和等于所有路径权值的异或和。使用 LCT 维护子树异或和。

```
数据结构：LCT（维护子树异或和 + 随机化验证）
支持操作：cut、link、路径加入/删除、边的"全覆盖"判定
关键性质：随机权值 + 子树异或和判定，正确率极高
时间复杂度：均摊 O(log n)
```

**C++ 代码**：

```cpp
// 每条路径 (s, t) 随机赋权 w
// val[s] ^= w, val[t] ^= w
// 查询边 (u,v) 是否被所有路径经过：
//   cut(u,v) 后看 u 子树的异或和 == 所有路径权值的异或总和
void add_path(int s, int t) {
    unsigned long long w = rng();
    path_weight[path_id] = w;
    total_xor ^= w;
    splay(s); val[s] ^= w; pushup(s);
    splay(t); val[t] ^= w; pushup(t);
}

bool check_edge(int u, int v) {
    // 使 u 成为 v 的子树
    makeroot(u); access(v); splay(v);
    // u 子树的总异或和（含虚子树）
    unsigned long long sub_xor = si_xor[u] ^ val[u];
    return sub_xor == total_xor;
}
```

**核心启示**：
- 随机化技巧（随机赋权 + 异或和判定）在动态树上有广泛应用
- LCT 维护子树异或和配合随机化，可以高效判定"所有路径是否经过某边"
- 这是 LCT 维护子树信息的极致应用

## 五、实战技巧总结

### 5.1 边权转点权

LCT 的 Splay 以节点为单位维护，无法直接处理边权。解决方法：为每条边 $(u,v)$ 建立一个虚拟节点 $e$，将 `link(u, e)` 和 `link(e, v)` 代替原本的 `link(u, v)`。边权存储在虚拟节点上。

### 5.2 pushdown 的正确时机

`splay(x)` 之前，必须将**从 Splay 根到 $x$ 路径上所有节点的标记下传**。标准做法是先用栈收集路径，再从上到下依次 `pushdown`。

### 5.3 findroot 后要 splay

`findroot` 中一路走到最左节点后，必须将该节点 `splay` 到根，否则会破坏 Splay 的均摊复杂度。

### 5.4 cut 的安全检查

断边前需要验证边确实存在：`makeroot(x)` 后 `access(y)` 再 `splay(y)`，检查 $x$ 是 $y$ 的左儿子且 $x$ 没有右子树且 $fa[x] == y$。

### 5.5 多组标记的合并顺序

当有多种懒标记（如加法、乘法、翻转）时，必须明确优先级。通常：翻转独立处理，乘法优先于加法。下传时严格按照优先级顺序。

### 5.6 调试技巧

- 在关键操作前后打印 Splay 的结构（中序遍历）验证正确性
- 单独测试 splay、access、makeroot 是否正确
- 注意 $0$ 号节点（空节点）的初始化：`sz[0] = s[0] = 0`

```

Step 5: 实现与调试
  - 套模板，修改 pushup 和 pushdown
  - 注意边界：0 号节点、虚拟节点编号
  - 大数据下注意常数优化
```

### 7.5 与其他数据结构的对比

| 数据结构 | 适用场景 | 动态连边 | 路径操作 | 子树操作 | 常数 |
|----------|----------|----------|----------|----------|------|
| 树链剖分 | 静态树路径/子树操作 | 不支持 | 支持 | 支持 | 小 |
| LCT | 动态森林路径操作 | 支持 | 支持 | 需额外维护 | 较大 |
| ETT | 动态森林子树操作 | 支持 | 不擅长 | 支持 | 中等 |
| 树分块 | 静态树暴力优化 | 不支持 | 部分支持 | 部分支持 | 中等 |

> 当树会动态变化且需要维护路径信息时，LCT 是唯一选择。掌握 access 的本质（虚实边切换）就掌握了 LCT 的灵魂。
