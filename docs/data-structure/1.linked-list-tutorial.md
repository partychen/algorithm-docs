## 一、什么是链表

链表是一种**线性数据结构**，其中每个元素（称为**节点**）包含两部分：

1. **数据域**：存储实际的数据值
2. **指针域**：存储下一个（或上一个）节点的地址

与数组不同，链表中的元素在内存中**不需要连续存放**。每个节点通过指针"链接"到下一个节点，形成一条链式结构。

```
数组（连续内存）：
┌───┬───┬───┬───┬───┐
│ 1 │ 2 │ 3 │ 4 │ 5 │
└───┴───┴───┴───┴───┘

链表（分散内存，指针串联）：
[1|→] ──→ [2|→] ──→ [3|→] ──→ [4|→] ──→ [5|∅]
```

**链表 vs 数组 核心对比：**

| 操作 | 数组 | 链表 |
|------|------|------|
| 随机访问 | $O(1)$ | $O(n)$ |
| 头部插入/删除 | $O(n)$ | $O(1)$ |
| 中间插入/删除 | $O(n)$ | $O(1)$（已知位置） |
| 内存分配 | 连续 | 非连续 |
| 缓存友好性 | 好 | 差 |

链表的核心优势在于：**在已知位置的情况下，插入和删除操作的时间复杂度为 $O(1)$**，无需像数组那样移动大量元素。

## 二、核心思想与前置知识

链表的核心思想是：**用指针将分散的内存块串联起来，以牺牲随机访问能力换取高效的动态插入与删除。**

### 2.1 单链表

单链表是最基本的链表结构。每个节点只有一个指针，指向下一个节点。最后一个节点的指针指向 `NULL`。

```
[data|next] -> [data|next] -> [data|next] -> NULL
```

**基本操作思路：**

- **插入**：修改前驱节点的 `next` 指针，使其指向新节点，新节点的 `next` 指向原来的后继节点
- **删除**：将前驱节点的 `next` 直接指向被删除节点的后继节点

> 注意：单链表只能**从前往后**遍历，无法反向查找前驱节点。因此删除操作通常需要维护前驱指针。

### 2.2 双链表

双链表的每个节点有两个指针：`prev` 指向前驱节点，`next` 指向后继节点。

```
NULL <- [prev|data|next] <-> [prev|data|next] <-> [prev|data|next] -> NULL
```

**优势：**

- 可以双向遍历
- 删除当前节点时，不需要额外维护前驱指针，因为可以直接通过 `prev` 访问

**代价：** 每个节点多占用一个指针的空间，插入/删除时需要维护两个方向的指针。

### 2.3 循环链表

循环链表是将链表的尾节点指向头节点，形成一个**环形结构**。

```
[data|next] -> [data|next] -> [data|next] --+
  ^                                          |
  +------------------------------------------+
```

- **单向循环链表**：尾节点的 `next` 指向头节点
- **双向循环链表**：尾节点的 `next` 指向头节点，头节点的 `prev` 指向尾节点

循环链表常用于需要**循环遍历**的场景，例如约瑟夫问题。

### 2.4 数组模拟链表（竞赛首选）

在算法竞赛中，为了避免动态内存分配的开销（`new`/`delete` 较慢），通常使用**数组模拟链表**（静态链表）。核心思想：用数组下标代替指针。

**数组模拟单链表：**

```cpp
const int N = 100010;
int e[N], ne[N], head, idx;
// e[i]: 节点 i 的数据
// ne[i]: 节点 i 的下一个节点编号（-1 表示尾部）
// head: 头节点编号, idx: 当前已分配的节点编号

void init() { head = -1; idx = 0; }

void insert_head(int x) {
    e[idx] = x; ne[idx] = head; head = idx++;
}

void insert_after(int k, int x) {
    e[idx] = x; ne[idx] = ne[k]; ne[k] = idx++;
}

void remove_after(int k) { ne[k] = ne[ne[k]]; }
```

**数组模拟双链表：**

```cpp
const int N = 100010;
int e[N], l[N], r[N], idx;
// l[i]/r[i]: 节点 i 的左/右邻节点
// 用 0 表示左端点，1 表示右端点（哨兵）

void init() { r[0] = 1; l[1] = 0; idx = 2; }

void insert_right(int k, int x) {
    e[idx] = x;
    l[idx] = k; r[idx] = r[k];
    l[r[k]] = idx; r[k] = idx++;
}

void remove(int k) { r[l[k]] = r[k]; l[r[k]] = l[k]; }
```

> 竞赛中推荐使用数组模拟链表，速度快且不容易出错。

## 三、分类与变体详解

### 3.1 按指针方向分类

| 类型 | 特点 | 典型应用 |
|------|------|----------|
| 单链表 | 单向遍历，结构简单 | 基础链表操作、邻接表 |
| 双链表 | 双向遍历，操作更灵活 | LRU 缓存、文本编辑器 |

### 3.2 按是否成环分类

| 类型 | 特点 | 典型应用 |
|------|------|----------|
| 非循环链表 | 有明确的头和尾 | 大多数通用场景 |
| 循环链表 | 首尾相连形成环 | 约瑟夫问题、轮转调度 |

### 3.3 常见变体与扩展

- **带头节点的链表（哨兵节点）**：在链表头部增加一个不存储有效数据的虚拟节点，简化边界处理
- **跳表（Skip List）**：在链表基础上增加多级索引，支持 $O(\log n)$ 的查找，是 Redis 有序集合的底层实现之一
- **邻接表**：图论中用链表存储每个顶点的邻居，本质是多个单链表的组合
- **链式前向星**：用数组模拟的邻接表，竞赛中存图的主流方式
- **Dancing Links（舞蹈链）**：基于双向十字链表的精确覆盖算法，用于数独求解等问题

### 3.4 变体对比总表

| 变体 | 核心思想 | 时间复杂度（查找） | 适用场景 |
|------|---------|-------------------|---------|
| 普通链表 | 节点+指针 | $O(n)$ | 频繁插入/删除 |
| 跳表 | 多级索引 | $O(\log n)$ | 有序集合 |
| 邻接表/前向星 | 多链表组合 | $O(\text{度})$ | 图的存储 |
| Dancing Links | 十字交叉双链表 | $O(1)$ 恢复/删除 | 精确覆盖 |

### 题目 1：约瑟夫问题（洛谷 P1996）

> $n$ 个人围成一圈，从第 1 个人开始报数，报到 $m$ 的人出列，然后从下一个人重新开始报数，直到所有人都出列。输出出列顺序。

**分析**：循环链表的经典应用。用数组模拟循环链表，每次向前走 $m-1$ 步后删除当前节点。

```
数据结构：数组模拟循环链表
支持操作：向后遍历、删除节点
关键性质：环形结构天然适配"围成一圈"的场景
时间复杂度：O(n * m)
```

**C++ 代码**：

```cpp
int ne[110];

void solve(int n, int m) {
    for (int i = 1; i <= n; i++) ne[i] = i % n + 1;
    int cur = n; // 从 n 开始，走到 1 的前驱
    for (int i = 0; i < n; i++) {
        for (int j = 1; j < m; j++) cur = ne[cur];
        // ne[cur] 就是第 m 个人
        cout << ne[cur] << " ";
        ne[cur] = ne[ne[cur]]; // 删除
    }
}
```

**核心启示**：
- **循环链表建模**：当问题涉及"围成一圈"，用 `ne[n] = 1` 即可构成环
- **删除即跳过**：链表删除不需要真正释放，只需修改前驱的 `next` 指针

### 题目 2：反转链表（LeetCode 206）

> 给定一个单链表的头节点，将链表反转，返回新的头节点。

**分析**：链表最经典的基础题。维护 prev/cur/nxt 三个指针，逐个翻转指针方向。

```
数据结构：单链表
支持操作：指针翻转
关键性质：翻转前必须先保存 next，否则链断裂后丢失后续节点
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
ListNode* reverseList(ListNode* head) {
    ListNode* prev = nullptr;
    ListNode* cur = head;
    while (cur) {
        ListNode* nxt = cur->next;
        cur->next = prev;
        prev = cur;
        cur = nxt;
    }
    return prev;
}
```

**核心启示**：
- **先保存再修改**：修改指针前，必须先保存即将丢失的引用
- **三指针法**：prev/cur/nxt 是链表原地操作的万能模板
- **返回 prev**：循环结束时 cur 为 NULL，prev 指向新头节点

### 题目 3：合并两个有序链表（LeetCode 21）

> 给定两个升序排列的单链表，将它们合并为一个新的升序链表。

**分析**：经典的双指针归并。使用哨兵节点（dummy）简化头部特判。

```
数据结构：单链表 + 哨兵节点
支持操作：尾部追加、双指针归并
关键性质：哨兵节点消除头部为空的特殊判断
时间复杂度：O(n + m)
```

**C++ 代码**：

```cpp
ListNode* mergeTwoLists(ListNode* l1, ListNode* l2) {
    ListNode dummy;
    ListNode* tail = &dummy;
    while (l1 && l2) {
        if (l1->val <= l2->val) { tail->next = l1; l1 = l1->next; }
        else { tail->next = l2; l2 = l2->next; }
        tail = tail->next;
    }
    tail->next = l1 ? l1 : l2;
    return dummy.next;
}
```

**核心启示**：
- **哨兵节点**：dummy node 在链表构造题中极其常用，避免对空头的特判
- **归并思想复用**：与归并排序的合并步骤完全一致，思路可迁移

### 题目 4：删除链表的倒数第 N 个节点（LeetCode 19）

> 给定一个链表，删除链表的倒数第 $n$ 个节点，返回链表的头节点。要求一趟扫描实现。

**分析**：快慢指针的经典应用。让快指针先走 $n$ 步，然后快慢指针同步前进，快指针到末尾时慢指针恰好在倒数第 $n+1$ 个节点。

```
数据结构：单链表 + 哨兵节点
支持操作：快慢指针定位、删除节点
关键性质：快指针领先 n 步 → 慢指针到达倒数第 n+1 个位置
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
ListNode* removeNthFromEnd(ListNode* head, int n) {
    ListNode dummy; dummy.next = head;
    ListNode* fast = &dummy;
    ListNode* slow = &dummy;
    for (int i = 0; i <= n; i++) fast = fast->next;
    while (fast) { fast = fast->next; slow = slow->next; }
    ListNode* del = slow->next;
    slow->next = del->next;
    return dummy.next;
}
```

**核心启示**：
- **快慢指针间距法**：让两个指针保持固定间距，可以一趟找到"倒数第 k 个"
- **哨兵处理边界**：删除头节点时（倒数第 n 个恰好是头），哨兵节点让逻辑统一

### 题目 5：队列安排（洛谷 P1160）

> $n$ 个同学依次入列。第 $i$ 个同学入列时，选择站在第 $k_i$ 个同学的左边或右边。最后有 $m$ 个同学离开队列。输出最终队列从左到右的编号。

**分析**：需要频繁在任意位置的左边或右边插入，然后删除若干元素。典型的**数组模拟双链表**场景。

```
数据结构：数组模拟双链表 + 哨兵节点
支持操作：在 k 的左/右边插入、按编号删除、顺序遍历
关键性质：左边插入 = 在 l[k] 的右边插入，统一为一种操作
时间复杂度：O(n + m)
```

**C++ 代码**：

```cpp
const int N = 100010;
int l[N], r[N];
bool removed[N];

void init() { r[0] = 1; l[1] = 0; } // 0 左哨兵, 1 右哨兵

void insert_right(int k, int x) {
    l[x] = k; r[x] = r[k];
    l[r[k]] = x; r[k] = x;
}

void solve(int n) {
    // 第 i 个人编号 i+2（0,1 是哨兵），左插 = insert_right(l[k], x)
    for (int i = r[0]; i != 1; i = r[i])
        if (!removed[i]) cout << i - 1 << " ";
}
```

**核心启示**：
- **操作统一化**："在 k 左边插入"转化为"在 l[k] 右边插入"，减少代码分支
- **标记删除**：删除时不真正修改链表，用 `removed[]` 标记，遍历时跳过

### 题目 6：环形链表 II（LeetCode 142）

> 给定一个链表，判断是否有环。如果有，返回入环的第一个节点。

**分析**：Floyd 判圈算法。快慢指针相遇后，将一个指针移回起点，两指针同速前进，再次相遇即为环的入口。

```
数据结构：单链表
支持操作：快慢指针遍历
关键性质：设起点到环入口距离 a，环入口到相遇点距离 b，环长 c → a = c - b（同余）
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
ListNode* detectCycle(ListNode* head) {
    ListNode* slow = head;
    ListNode* fast = head;
    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) {
            ListNode* p = head;
            while (p != slow) { p = p->next; slow = slow->next; }
            return p;
        }
    }
    return nullptr;
}
```

**核心启示**：
- **Floyd 判圈**：快慢指针是检测链表环的标准方法，空间 $O(1)$
- **数学推导**：相遇后"回到起点同速走"能找到入口，依赖 $a \equiv c - b \pmod{c}$

### 题目 7：寻找重复数（LeetCode 287）

> 给定一个包含 $n + 1$ 个整数的数组 `nums`，其中每个整数都在 $[1, n]$ 范围内，证明至少存在一个重复的整数。找出这个重复数。要求不修改数组且空间 $O(1)$。

**分析**：将数组看作隐式链表！`nums[i]` 是节点 $i$ 的 `next` 指针。由于有重复数，必然存在环，环的入口就是重复数。直接套用 Floyd 判圈。

```
数据结构：隐式链表（数组下标 → 值 = next 指针）
支持操作：Floyd 判圈
关键性质：重复数 → 两个节点指向同一后继 → 形成环 → 环入口 = 重复数
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
int findDuplicate(vector<int>& nums) {
    int slow = 0, fast = 0;
    do { slow = nums[slow]; fast = nums[nums[fast]]; } while (slow != fast);
    int p = 0;
    while (p != slow) { p = nums[p]; slow = nums[slow]; }
    return p;
}
```

**核心启示**：
- **隐式链表**：数组 `a[i]` 可以看作"节点 i 的 next"，将数组问题转化为链表问题
- **算法迁移**：同一个 Floyd 判圈算法，换一个建模视角就解决了完全不同的问题

### 题目 8：链表的中间节点（LeetCode 876）

> 给定一个非空单链表，返回中间节点。如果有两个中间节点，返回第二个。

**分析**：快慢指针的另一经典用途。快指针每次走 2 步，慢指针每次走 1 步，快指针到末尾时慢指针恰在中间。

```
数据结构：单链表
支持操作：快慢指针遍历
关键性质：快指针速度为慢指针 2 倍 → 慢指针走过的距离恰为总长度的一半
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
ListNode* middleNode(ListNode* head) {
    ListNode* slow = head;
    ListNode* fast = head;
    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
    }
    return slow;
}
```

**核心启示**：
- **快慢指针找中点**：这是链表归并排序、判断回文链表等高级题的前置技巧
- **边界细节**：`while (fast && fast->next)` 同时处理了奇偶长度的情况

### 🟠 提高级 Level 3：链表综合应用

### 题目 9：重排链表（LeetCode 143）

> 给定链表 $L_0 \to L_1 \to \cdots \to L_n$，重排为 $L_0 \to L_n \to L_1 \to L_{n-1} \to L_2 \to L_{n-2} \to \cdots$。

**分析**：三步走：找中点 → 反转后半部分 → 交替合并。综合运用了前面三道题的技巧。

```
数据结构：单链表
支持操作：快慢指针找中点、反转链表、交替合并
关键性质：将三个基础操作组合解决复杂问题
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
void reorderList(ListNode* head) {
    if (!head || !head->next) return;
    // 1. 找中点
    ListNode* slow = head, *fast = head;
    while (fast->next && fast->next->next) { slow = slow->next; fast = fast->next->next; }
    // 2. 反转后半部分
    ListNode* prev = nullptr, *cur = slow->next;
    slow->next = nullptr;
    while (cur) { ListNode* nxt = cur->next; cur->next = prev; prev = cur; cur = nxt; }
    // 3. 交替合并
    ListNode* l1 = head, *l2 = prev;
    while (l2) {
        ListNode* n1 = l1->next, *n2 = l2->next;
        l1->next = l2; l2->next = n1;
        l1 = n1; l2 = n2;
    }
}
```

**核心启示**：
- **组合基础操作**：复杂链表题往往是多个基础操作的拼装
- **原地重排**：找中点 + 反转 + 合并是链表"从两端向中间"操作的标准范式

### 题目 10：排序链表（LeetCode 148）

> 给定链表头节点，将其按升序排序。要求时间 $O(n \log n)$，空间 $O(1)$。

**分析**：链表归并排序。自顶向下递归：找中点分割 → 递归排序左右 → 合并有序链表。

```
数据结构：单链表
支持操作：快慢指针分割、递归归并、有序合并
关键性质：归并排序天然适配链表——合并操作 O(1) 空间，分割用快慢指针
时间复杂度：O(n log n)
```

**C++ 代码**：

```cpp
ListNode* sortList(ListNode* head) {
    if (!head || !head->next) return head;
    // 找中点并断开
    ListNode* slow = head, *fast = head->next;
    while (fast && fast->next) { slow = slow->next; fast = fast->next->next; }
    ListNode* mid = slow->next;
    slow->next = nullptr;
    // 递归排序 + 合并
    return mergeTwoLists(sortList(head), sortList(mid));
}
// mergeTwoLists 同题目 3
```

**核心启示**：
- **归并排序最适合链表**：不需要随机访问，合并时无需额外空间
- **分割要断开**：`slow->next = nullptr` 是关键，否则递归不会终止

### 题目 11：K 个一组翻转链表（LeetCode 25）

> 给定一个链表，每 $k$ 个节点一组进行翻转。如果剩余节点不足 $k$ 个，则保持原样。

**分析**：先检查剩余长度是否 $\geq k$，然后对当前 $k$ 个节点做局部反转，再递归处理剩余部分。

```
数据结构：单链表
支持操作：计数、局部反转、递归连接
关键性质：每组反转后的尾节点需要连接下一组反转后的头节点
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
ListNode* reverseKGroup(ListNode* head, int k) {
    ListNode* cur = head;
    int cnt = 0;
    while (cur && cnt < k) { cur = cur->next; cnt++; }
    if (cnt < k) return head; // 不足 k 个，不翻转
    // 翻转前 k 个
    ListNode* prev = nullptr;
    cur = head;
    for (int i = 0; i < k; i++) {
        ListNode* nxt = cur->next;
        cur->next = prev; prev = cur; cur = nxt;
    }
    head->next = reverseKGroup(cur, k); // head 变成了尾部
    return prev;
}
```

**核心启示**：
- **递归处理分组问题**：每次只处理一组 $k$ 个节点，剩余交给递归
- **反转后的连接**：原来的 head 变成组内最后一个，连接后续递归结果

### 题目 12：合并 K 个升序链表（LeetCode 23）

> 给定一个链表数组，每个链表已按升序排列，将所有链表合并为一个升序链表。

**分析**：使用小根堆（优先队列）维护各链表的当前最小节点，每次取出全局最小值追加到结果。

```
数据结构：小根堆 + 链表
支持操作：堆中取最小、追加到结果链表
关键性质：堆中始终维护 k 个元素（每条链表的当前头），每次 O(log k)
时间复杂度：O(N log k)，N 为总节点数
```

**C++ 代码**：

```cpp
ListNode* mergeKLists(vector<ListNode*>& lists) {
    auto cmp = [](ListNode* a, ListNode* b) { return a->val > b->val; };
    priority_queue<ListNode*, vector<ListNode*>, decltype(cmp)> pq(cmp);
    for (auto h : lists) if (h) pq.push(h);
    ListNode dummy;
    ListNode* tail = &dummy;
    while (!pq.empty()) {
        ListNode* cur = pq.top(); pq.pop();
        tail->next = cur; tail = cur;
        if (cur->next) pq.push(cur->next);
    }
    return dummy.next;
}
```

**核心启示**：
- **堆优化多路归并**：将 $O(kN)$ 的暴力复杂度优化到 $O(N \log k)$
- **通用范式**：任何"从多个有序源中选最小"的问题都可以用堆解决

### 题目 13：LRU 缓存（LeetCode 146）

> 设计一个 LRU（最近最少使用）缓存，支持 `get(key)` 和 `put(key, value)` 操作，两者均 $O(1)$。超过容量时淘汰最久未使用的键。

**分析**：双链表 + 哈希表。双链表维护访问顺序（头部最新，尾部最旧），哈希表实现 $O(1)$ 的键值查找。

```
数据结构：双链表 + 哈希表
支持操作：O(1) 查找、O(1) 移动到头部、O(1) 删除尾部
关键性质：双链表两端操作 O(1)；哈希表定位节点 O(1)
时间复杂度：get/put 均 O(1)
```

**C++ 代码**：

```cpp
struct Node { int key, val; Node* prev; Node* next; };

struct LRUCache {
    int cap;
    Node head, tail; // 哨兵
    unordered_map<int, Node*> mp;

    LRUCache(int capacity) : cap(capacity) { head.next = &tail; tail.prev = &head; }

    void remove(Node* node) {
        node->prev->next = node->next;
        node->next->prev = node->prev;
    }

    void insertFront(Node* node) {
        node->next = head.next; node->prev = &head;
        head.next->prev = node; head.next = node;
    }

    int get(int key) {
        if (!mp.count(key)) return -1;
        Node* node = mp[key];
        remove(node); insertFront(node);
        return node->val;
    }

    void put(int key, int value) {
        if (mp.count(key)) {
            Node* node = mp[key];
            node->val = value;
            remove(node); insertFront(node);
        } else {
            if ((int)mp.size() == cap) {
                Node* old = tail.prev;
                remove(old); mp.erase(old->key); delete old;
            }
            Node* node = new Node{key, value, nullptr, nullptr};
            insertFront(node); mp[key] = node;
        }
    }
};
```

**核心启示**：
- **双链表 + 哈希表**：这是实现 $O(1)$ LRU 缓存的标准组合
- **哨兵节点**：head/tail 哨兵消除了链表为空时的边界判断
- **工程价值**：LRU 是操作系统页面置换、数据库缓存的核心算法

### 题目 14：复制带随机指针的链表（LeetCode 138）

> 链表每个节点除 `next` 外还有一个 `random` 指针指向链表中任意节点或 null。深拷贝整个链表。

**分析**：三步走法（原地复制）：1) 在每个原节点后插入其拷贝；2) 利用位置关系设置拷贝的 random；3) 拆分出新链表。

```
数据结构：单链表（带额外随机指针）
支持操作：原地插入、指针映射、链表拆分
关键性质：copy 紧跟在 original 后面 → original->next->random = original->random->next
时间复杂度：O(n)，空间 O(1)（不计输出）
```

**C++ 代码**：

```cpp
Node* copyRandomList(Node* head) {
    if (!head) return nullptr;
    // 1.在每个节点后插入拷贝
    for (Node* cur = head; cur; cur = cur->next->next) {
        Node* copy = new Node(cur->val);
        copy->next = cur->next; cur->next = copy;
    }
    // 2.设置拷贝的 random
    for (Node* cur = head; cur; cur = cur->next->next)
        if (cur->random) cur->next->random = cur->random->next;
    // 3.拆分
    Node* newHead = head->next;
    for (Node* cur = head; cur; cur = cur->next) {
        Node* copy = cur->next;
        cur->next = copy->next;
        if (copy->next) copy->next = copy->next->next;
    }
    return newHead;
}
```

**核心启示**：
- **原地映射**：将拷贝节点插在原节点后面，巧妙解决"原节点 → 拷贝节点"的映射问题
- **三步拆分法**：插入 → 连接 → 拆分的模式可以推广到其他"带额外指针"的链表深拷贝

### 题目 15：邻接表建图与遍历（洛谷 B3643）

> 给定一个 $n$ 个点 $m$ 条边的有向图，使用邻接表存储并进行 DFS 遍历，输出 DFS 序。

**分析**：链式前向星（数组模拟邻接表）是竞赛中最常用的存图方式，本质是多个单链表的组合。

```
数据结构：链式前向星（数组模拟多链表）
支持操作：添加边、遍历某顶点的所有邻居
关键性质：head[u] 是节点 u 的边链表头，每条边的 ne 指向同一节点的下一条边
时间复杂度：建图 O(m)，遍历 O(n + m)
```

**C++ 代码**：

```cpp
const int N = 100010, M = 200010;
int head[N], e[M], ne[M], idx;

void init() { memset(head, -1, sizeof head); idx = 0; }

void add(int u, int v) { e[idx] = v; ne[idx] = head[u]; head[u] = idx++; }

bool vis[N];
void dfs(int u) {
    vis[u] = true;
    cout << u << " ";
    for (int i = head[u]; i != -1; i = ne[i]) {
        int v = e[i];
        if (!vis[v]) dfs(v);
    }
}
```

**核心启示**：
- **链式前向星**：本质是对每个顶点维护一个单链表，头插法添加边
- **竞赛标配**：几乎所有图论题都以此为存图基础，必须熟练掌握

### 题目 16：Dancing Links 精确覆盖（洛谷 P4929）

> 给定一个 01 矩阵，选出若干行使得每列恰好有一个 1（精确覆盖问题）。用 Dancing Links（DLX）算法求解。

**分析**：DLX 基于**双向十字链表**。删除某行/列时修改指针跳过它，回溯时恢复指针。这种"可撤销删除"是 DLX 的精髓。

```
数据结构：双向十字链表（Dancing Links）
支持操作：覆盖列（删除）、恢复列（撤销）、递归搜索
关键性质：双向链表删除后可 O(1) 恢复（节点本身未释放，邻居指针还原即可）
时间复杂度：指数级（精确覆盖是 NP 完全），但剪枝高效
```

**C++ 代码**：

```cpp
const int MAXN = 100010;
int L[MAXN], R[MAXN], U[MAXN], D[MAXN]; // 四向指针
int col[MAXN], row[MAXN], sz[MAXN];      // 所属列、所属行、每列节点数
int tot, ans[MAXN], cnt;

void cover(int c) { // 覆盖第 c 列
    L[R[c]] = L[c]; R[L[c]] = R[c];
    for (int i = D[c]; i != c; i = D[i])
        for (int j = R[i]; j != i; j = R[j]) {
            U[D[j]] = U[j]; D[U[j]] = D[j]; sz[col[j]]--;
        }
}

void uncover(int c) { // 恢复第 c 列（操作顺序严格反转）
    for (int i = U[c]; i != c; i = U[i])
        for (int j = L[i]; j != i; j = L[j]) {
            U[D[j]] = j; D[U[j]] = j; sz[col[j]]++;
        }
    L[R[c]] = c; R[L[c]] = c;
}

bool dance(int dep) {
    if (R[0] == 0) { cnt = dep; return true; } // 所有列已覆盖
    int c = R[0];
    for (int i = R[0]; i != 0; i = R[i]) // 选节点数最少的列（优化）
        if (sz[i] < sz[c]) c = i;
    cover(c);
    for (int i = D[c]; i != c; i = D[i]) {
        ans[dep] = row[i];
        for (int j = R[i]; j != i; j = R[j]) cover(col[j]);
        if (dance(dep + 1)) return true;
        for (int j = L[i]; j != i; j = L[j]) uncover(col[j]);
    }
    uncover(c);
    return false;
}
```

**核心启示**：
- **可撤销删除**：双向链表的节点被"摘除"后仍保存着邻居信息，恢复只需反向操作
- **Dancing Links**：链表"跳舞"般地删除和恢复，是回溯搜索的极致优化
- **应用广泛**：数独、N 皇后、拼图覆盖等问题都可以建模为精确覆盖

## 五、常见技巧与思路总结

### 5.1 哨兵节点（Dummy Node）

在链表头部添加虚拟节点，统一处理"头部插入/删除"和"中间插入/删除"的逻辑，消除大量 `if (head == nullptr)` 的边界判断。

```cpp
ListNode dummy;
dummy.next = head;
// 所有操作基于 dummy 进行，最后返回 dummy.next
```

### 5.2 快慢指针

| 用途 | 快指针速度 | 慢指针速度 | 结果 |
|------|-----------|-----------|------|
| 找中点 | 2 步 | 1 步 | 慢指针停在中间 |
| 判环 | 2 步 | 1 步 | 相遇 → 有环 |
| 找环入口 | 相遇后回起点，1 步 | 1 步 | 再次相遇 → 入口 |
| 找倒数第 k 个 | 先走 k 步，再 1 步 | 1 步 | 快到终点时慢在目标 |

### 5.3 先保存再修改

链表指针操作的第一原则。修改 `cur->next` 之前，务必用临时变量保存 `cur->next`，否则链断裂后丢失后续节点。

### 5.4 画图！画图！画图！

链表题最重要的调试方法。在纸上画出指针变化过程，比在脑子里模拟快得多，也不容易出错。

### 5.5 竞赛中优先用数组模拟

数组模拟链表比动态分配内存（`new`/`delete`）快数倍，代码也更短。竞赛中几乎不用指针实现链表。

### 5.6 操作顺序决定正确性

双链表插入时四条指针的修改顺序至关重要。安全的顺序是：**先设置新节点的指针，再修改邻居的指针**。

```cpp
// 安全的插入顺序（在 k 右边插入 x）
l[x] = k; r[x] = r[k];   // 先设置新节点
l[r[k]] = x; r[k] = x;    // 再修改邻居
```

## 七、终极总结与归纳

### 7.1 链表的本质

**链表 = 用指针（或数组下标）将分散的节点串联成线性结构。**

它的价值在于：在已知位置时，$O(1)$ 完成插入/删除，且无需移动其他元素。这种灵活性使链表成为树、图、哈希表等复杂数据结构的"基础零件"。

### 7.2 核心要点

1. **单链表**适用于大多数场景，**双链表**在需要双向遍历或快速删除当前节点时使用
2. **快慢指针**是链表的核心技巧，可以找中点、判环、找倒数第 k 个
3. **哨兵节点**能消除头部边界判断，是链表题的"万金油"
4. **数组模拟链表**是竞赛首选，比指针实现快且不易出错
5. **先保存再修改**：修改指针前务必保存即将丢失的引用
6. **链表的应用远超链表本身**：邻接表、哈希表拉链、DLX、LRU 缓存都以链表为基础
