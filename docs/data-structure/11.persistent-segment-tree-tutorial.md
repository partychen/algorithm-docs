## 一、什么是主席树

主席树，又称**可持久化权值线段树**（Persistent Weighted Segment Tree），因发明者黄嘉泰（HJT）的姓名缩写谐音"主席"而得名。

**本质定义：** 主席树是在权值线段树的基础上，通过**共享节点**技术实现可持久化，从而能够高效地维护多个历史版本的数据结构。

**核心应用：** 静态区间第 $k$ 小问题。给定长度为 $n$ 的序列，每次询问区间 $[l, r]$ 中第 $k$ 小的元素，要求在线回答。

**为什么需要主席树？**

| 方法 | 时间复杂度 | 缺陷 |
|------|-----------|------|
| 暴力排序 | $O(qn\log n)$ | 太慢 |
| 归并树 | $O(n\log^2 n)$ | 常数大、实现复杂 |
| 整体二分 | $O(n\log^2 n)$ | 离线 |
| **主席树** | $O(n\log n)$ 建树 + $O(\log n)$ 查询 | **在线、高效** |

## 二、核心思想与前置知识

### 2.1 权值线段树

权值线段树是以**值域**为下标建立的线段树，每个节点维护的是某个值域范围内元素的**出现次数**。

例如值域为 $[1, 5]$，插入序列 $\{2, 3, 2, 5\}$ 后：

```
          [1,5]: 4
        /          \
    [1,3]: 3       [4,5]: 1
    /     \        /     \
 [1,1]:0 [2,3]:3 [4,4]:0 [5,5]:1
         /    \
      [2,2]:2 [3,3]:1
```

在权值线段树上求第 $k$ 小：从根开始，若左子树的 $cnt \ge k$，则答案在左子树中，递归进入左子树；否则令 $k = k - cnt_{left}$，递归进入右子树。

### 2.2 可持久化思想（共享节点）

如果对序列中每个前缀 $a[1..i]$ 都建一棵权值线段树，就能通过**版本差分**得到任意区间的信息。但朴素地建 $n$ 棵树空间为 $O(n^2)$，不可接受。

**关键观察：** 从第 $i-1$ 个版本到第 $i$ 个版本，只有从根到某个叶子的一条路径（$O(\log n)$ 个节点）发生了变化。

**解决方案：** 每次只新建这条路径上的节点，其余节点直接指向上一个版本中对应的节点（共享）。

```
版本 i-1 的树          版本 i 的树（仅新建灰色节点）

      A                     A'    ← 新建
     / \                   / \
    B   C                 B   C'  ← 新建，左孩子指向 B（共享）
   / \   \               / \   \
  D   E   F             D   E   F' ← 新建
```

这样每个版本只增加 $O(\log n)$ 个节点，总空间 $O(n \log n)$。

### 2.3 前缀和版本差分

设 $T_i$ 为前缀 $a[1..i]$ 对应的权值线段树版本。

对于区间 $[l, r]$ 中值域 $[vl, vr]$ 内的元素个数，利用前缀和：

$$cnt_{[l,r]}[vl, vr] = T_r.count(vl, vr) - T_{l-1}.count(vl, vr)$$

查询区间第 $k$ 小时，同步在 $T_r$ 和 $T_{l-1}$ 上走，比较差值与 $k$ 的关系即可。

### 2.4 代码模板

```cpp
const int MAXN = 200005;
const int MAXM = MAXN * 40;

int ls[MAXM], rs[MAXM], cnt[MAXM], tot = 0;
int root[MAXN];

int build(int l, int r) {
    int p = ++tot;
    cnt[p] = 0;
    if (l == r) return p;
    int mid = (l + r) >> 1;
    ls[p] = build(l, mid);
    rs[p] = build(mid + 1, r);
    return p;
}

int update(int prev, int l, int r, int x) {
    int p = ++tot;
    ls[p] = ls[prev]; rs[p] = rs[prev]; cnt[p] = cnt[prev] + 1;
    if (l == r) return p;
    int mid = (l + r) >> 1;
    if (x <= mid) ls[p] = update(ls[prev], l, mid, x);
    else          rs[p] = update(rs[prev], mid + 1, r, x);
    return p;
}

int query(int u, int v, int l, int r, int k) {
    if (l == r) return l;
    int mid = (l + r) >> 1;
    int d = cnt[ls[v]] - cnt[ls[u]];
    if (k <= d) return query(ls[u], ls[v], l, mid, k);
    else        return query(rs[u], rs[v], mid + 1, r, k - d);
}
```

**使用方式：**

```cpp
root[0] = build(1, m);
for (int i = 1; i <= n; i++)
    root[i] = update(root[i - 1], 1, m, a[i]);

int ans = query(root[l - 1], root[r], 1, m, k);
```

## 三、分类与变体详解

### 3.1 静态区间第 $k$ 小

最经典的应用。对原序列按前缀建 $n$ 个版本，利用版本差分回答区间查询。

- 时间复杂度：建树 $O(n\log n)$，单次查询 $O(\log n)$
- 空间复杂度：$O(n\log n)$

### 3.2 树上路径第 $k$ 小

将序列推广到树上。对每个节点 $u$，令 $T_u$ 为从根到 $u$ 路径上所有点权构成的权值线段树版本。

利用树上前缀和与 LCA：

$$cnt_{path(u,v)} = T_u + T_v - T_{lca} - T_{fa(lca)}$$

在四棵树上同步行走即可求路径第 $k$ 小。

### 3.3 带修改（树状数组套主席树）

当序列存在单点修改时，朴素主席树的前缀和性质失效。

**解决方案：** 用树状数组管理主席树版本。树状数组的每个位置维护一棵权值线段树，修改和查询时同时操作 $O(\log n)$ 棵树。

- 时间复杂度：单次操作 $O(\log^2 n)$
- 空间复杂度：$O(n\log^2 n)$

### 3.4 可持久化数组

主席树可以作为可持久化数组使用：在版本 $v$ 的基础上修改位置 $x$ 的值，产生新版本。每次修改只新建 $O(\log n)$ 个节点。这是可持久化并查集等高级结构的基础。

### 3.5 变体对比

| 变体 | 建树方式 | 查询方式 | 时间复杂度 | 空间复杂度 |
|------|---------|---------|-----------|-----------|
| 静态区间第 k 小 | 前缀版本 | 双版本差分 | $O(\log n)$ | $O(n\log n)$ |
| 树上路径第 k 小 | DFS 版本 | 四版本差分 | $O(\log n)$ | $O(n\log n)$ |
| 带修改第 k 小 | BIT 管理 | 多版本同步 | $O(\log^2 n)$ | $O(n\log^2 n)$ |
| 可持久化数组 | 标准建树 | 版本回溯 | $O(\log n)$ | $O(n\log n)$ |

### 题目 1：静态区间第 k 小（洛谷 P3834）

> **题意：** 给定长度为 $n$ 的序列，$m$ 次询问，每次查询区间 $[l, r]$ 中第 $k$ 小的元素，要求在线。

**分析**：主席树的裸模板题。先离散化，然后按前缀建 $n$ 个权值线段树版本。查询时在 $T_r$ 和 $T_{l-1}$ 上同步行走，利用左子树计数差值与 $k$ 比较来决定方向。

```
数据结构：主席树（可持久化权值线段树）
支持操作：前缀版本建树 + 双版本差分查询
关键性质：版本差分 cnt[l,r] = T_r - T_{l-1}
时间复杂度：建树 O(n log n)，查询 O(log n)
```

**C++ 代码**：

```cpp
int ls[MAXM], rs[MAXM], cnt[MAXM], tot;
int root[MAXN];

int build(int l, int r) {
    int p = ++tot;
    cnt[p] = 0;
    if (l == r) return p;
    int mid = (l + r) >> 1;
    ls[p] = build(l, mid);
    rs[p] = build(mid + 1, r);
    return p;
}

int update(int prev, int l, int r, int x) {
    int p = ++tot;
    ls[p] = ls[prev]; rs[p] = rs[prev]; cnt[p] = cnt[prev] + 1;
    if (l == r) return p;
    int mid = (l + r) >> 1;
    if (x <= mid) ls[p] = update(ls[prev], l, mid, x);
    else          rs[p] = update(rs[prev], mid + 1, r, x);
    return p;
}

int query(int u, int v, int l, int r, int k) {
    if (l == r) return l;
    int mid = (l + r) >> 1;
    int d = cnt[ls[v]] - cnt[ls[u]];
    if (k <= d) return query(ls[u], ls[v], l, mid, k);
    else        return query(rs[u], rs[v], mid + 1, r, k - d);
}
```

**核心启示**：
- 主席树的核心是"前缀版本 + 差分查询"
- 离散化是必要步骤，将值域压缩到 $[1, m]$

### 题目 2：可持久化数组（洛谷 P3919）

> **题意：** 给定一个长度为 $n$ 的数组，支持两种操作：1）在某个版本基础上修改某位置的值，生成新版本；2）查询某个版本某位置的值。

**分析**：可持久化线段树的基础应用。每个版本对应一棵线段树，修改时只新建从根到叶子路径上的 $O(\log n)$ 个节点。查询直接在对应版本上走到叶子。

```
数据结构：可持久化线段树（非权值）
支持操作：版本修改 + 版本查询
关键性质：每次修改只新建 O(log n) 个节点，共享其余
时间复杂度：每次操作 O(log n)
```

**C++ 代码**：

```cpp
int ls[MAXM], rs[MAXM], val[MAXM], tot;
int root[MAXN];

int build(int l, int r) {
    int p = ++tot;
    if (l == r) { val[p] = a[l]; return p; }
    int mid = (l + r) >> 1;
    ls[p] = build(l, mid);
    rs[p] = build(mid + 1, r);
    return p;
}

int update(int prev, int l, int r, int x, int v) {
    int p = ++tot;
    ls[p] = ls[prev]; rs[p] = rs[prev]; val[p] = val[prev];
    if (l == r) { val[p] = v; return p; }
    int mid = (l + r) >> 1;
    if (x <= mid) ls[p] = update(ls[prev], l, mid, x, v);
    else          rs[p] = update(rs[prev], mid + 1, r, x, v);
    return p;
}

int query(int p, int l, int r, int x) {
    if (l == r) return val[p];
    int mid = (l + r) >> 1;
    if (x <= mid) return query(ls[p], l, mid, x);
    else          return query(rs[p], mid + 1, r, x);
}
```

**核心启示**：
- 可持久化数组是可持久化线段树最直接的应用
- 它是可持久化并查集等高级结构的基础组件

### 题目 3：历史版本查询（洛谷 P3834 变体）

> **题意：** 给定序列，支持在任意历史版本基础上插入元素，并在任意版本上查询区间信息。版本之间形成树形关系。

**分析**：当版本关系不再是线性链而是树形结构时，需要维护"版本树"。每个操作基于某个历史版本创建新版本。核心思路不变——新版本只修改路径上的节点，其余共享。

```
数据结构：可持久化权值线段树（版本树形结构）
支持操作：基于任意版本 update / query
关键性质：版本间共享节点，每次操作新建 O(log n) 个节点
时间复杂度：每次操作 O(log n)
```

**C++ 代码**：

```cpp
int update(int prev, int l, int r, int x, int val) {
    int p = ++tot;
    ls[p] = ls[prev]; rs[p] = rs[prev];
    cnt[p] = cnt[prev];
    if (l == r) { cnt[p] += val; return p; }
    int mid = (l + r) >> 1;
    if (x <= mid) ls[p] = update(ls[prev], l, mid, x, val);
    else          rs[p] = update(rs[prev], mid + 1, r, x, val);
    cnt[p] = cnt[ls[p]] + cnt[rs[p]];
    return p;
}

// 使用：root[new_ver] = update(root[old_ver], 1, m, x, +1);
```

**核心启示**：
- 可持久化的本质是"版本管理"，版本之间可以形成任意 DAG 或树
- 掌握了共享节点与版本树的概念后，可以灵活应对各种变形题

### 题目 4：区间不同数的个数（离线做法参考）

> **题意：** 给定长度为 $n$ 的序列，$m$ 次询问，每次查询区间 $[l, r]$ 中有多少个不同的数。

**分析**：经典做法是离线 + 树状数组，但也可以用主席树在线解决。关键思路：对每个位置 $i$，记录 $a[i]$ 上一次出现的位置 $pre[i]$。建主席树时，版本 $i$ 在位置 $pre[i]$ 处值为 $0$，在位置 $i$ 处值为 $1$。查询 $[l, r]$ 中不同数个数 = 在版本 $r$ 上查询 $[l, r]$ 范围内 $1$ 的个数。

```
数据结构：主席树（按位置建树，非权值）
支持操作：版本建树 + 区间求和
关键性质：pre[i] 标记上次出现位置，当前版本只保留每个数最后出现的位置
时间复杂度：建树 O(n log n)，查询 O(log n)
```

**C++ 代码**：

```cpp
int pre[MAXN]; // a[i] 上次出现的位置

void solve() {
    // 预处理 pre[]
    map<int, int> last;
    for (int i = 1; i <= n; i++) {
        pre[i] = last.count(a[i]) ? last[a[i]] : 0;
        last[a[i]] = i;
    }
    // 建主席树
    root[0] = build(1, n);
    for (int i = 1; i <= n; i++) {
        root[i] = update(root[i-1], 1, n, i, 1);
        if (pre[i])
            root[i] = update(root[i], 1, n, pre[i], -1);
    }
    // 查询 [l, r] 中不同数个数
    // = query_sum(root[r], 1, n, l, r)
}
```

**核心启示**：
- 主席树不仅用于权值线段树，也可以在位置维度上建版本
- pre 数组是处理"不同数"问题的关键预处理

### 题目 5：粟粟的书架（洛谷 P2468）

> **题意：** 给定一个矩阵（或序列），每次询问一个子区间内，至少选多少本书，使得页数之和 $\ge S$。

**分析**：当矩阵退化为一维时，问题变为：区间内从大到小选书，求最少选多少本使总和 $\ge S$。这等价于在主席树上从高权值到低权值贪心。额外维护 sum 数组记录子树权值和。

```
数据结构：主席树（权值线段树 + sum 维护）
支持操作：前缀版本差分 + 从大到小贪心
关键性质：右子树（大值）优先选取，不够再进左子树
时间复杂度：建树 O(n log n)，查询 O(log n)
```

**C++ 代码**：

```cpp
int ls[MAXM], rs[MAXM], cnt[MAXM];
long long sum[MAXM];

int query(int u, int v, int l, int r, long long S) {
    if (l == r) return min((long long)(cnt[v] - cnt[u]),
                           (S + l - 1) / l);
    int mid = (l + r) >> 1;
    long long rsum = sum[rs[v]] - sum[rs[u]];
    int rcnt = cnt[rs[v]] - cnt[rs[u]];
    if (rsum >= S)
        return query(rs[u], rs[v], mid + 1, r, S);
    else
        return rcnt + query(ls[u], ls[v], l, mid, S - rsum);
}
```

**核心启示**：
- 主席树不仅能查第 $k$ 小，还能在权值上做贪心/二分
- 额外维护 sum 数组可以支持"从大到小取"的策略

### 题目 6：任务查询系统（洛谷 P3168）

> **题意：** 有 $n$ 个任务，每个任务在时间 $[s_i, e_i]$ 内有优先级 $p_i$。每次查询时刻 $t$ 时，所有活跃任务中优先级前 $k$ 小的任务的优先级之和。

**分析**：将每个任务拆成差分事件：时刻 $s_i$ 插入 $p_i$，时刻 $e_i+1$ 删除 $p_i$。按时间顺序依次操作，每个时刻对应一个主席树版本。查询某时刻的前 $k$ 小之和。

```
数据结构：主席树（按时间轴建版本）
支持操作：差分事件建版本 + 前 k 小求和
关键性质：时间轴差分 + 主席树版本串联
时间复杂度：建树 O(n log n)，查询 O(log n)
```

**C++ 代码**：

```cpp
long long query(int p, int l, int r, int k) {
    if (l == r) return (long long)l * min(k, cnt[p]);
    int mid = (l + r) >> 1;
    int lc = cnt[ls[p]];
    if (k <= lc) return query(ls[p], l, mid, k);
    else return sum[ls[p]] + query(rs[p], mid + 1, r, k - lc);
}
```

**核心启示**：
- 主席树的版本管理天然适合时间轴上的差分操作
- 配合区间事件拆分可以优雅地处理时间段覆盖问题

### 题目 7：区间排名查询（洛谷 P3834 扩展）

> **题意：** 给定长度为 $n$ 的序列，$m$ 次询问：1）区间 $[l,r]$ 中 $x$ 的排名；2）区间 $[l,r]$ 中排名为 $k$ 的数；3）$x$ 的前驱/后继。

**分析**：在主席树上扩展出多种查询。排名查询 = 在版本差分上查 $< x$ 的元素个数 + 1。前驱 = 先查排名再查排名减一。后继 = 查排名加一。所有操作均基于版本差分。

```
数据结构：主席树
支持操作：排名查询 / 第 k 小 / 前驱 / 后继
关键性质：所有操作基于版本差分的值域查询
时间复杂度：每次查询 O(log n)
```

**C++ 代码**：

```cpp
// 查询区间 [ql,qr] 中 < x 的元素个数
int countLT(int u, int v, int l, int r, int x) {
    if (l == r) return 0;
    int mid = (l + r) >> 1;
    if (x <= mid) return countLT(ls[u], ls[v], l, mid, x);
    int lc = cnt[ls[v]] - cnt[ls[u]];
    return lc + countLT(rs[u], rs[v], mid + 1, r, x);
}

int rank_of(int l, int r, int x) {
    return countLT(root[l-1], root[r], 1, m, x) + 1;
}

// 前驱：排名为 rank_of(x) - 1 的数
// 后继：排名为 rank_of(x+1) 的数
```

**核心启示**：
- 主席树支持的查询不止第 $k$ 小，排名、前驱、后继都可以在同一结构上实现
- 本质上主席树可以看作"区间限定的值域查询工具"

### 题目 8：区间中位数（洛谷 P3834 应用）

> **题意：** 给定序列，$m$ 次查询区间 $[l, r]$ 的中位数（第 $\lceil (r-l+1)/2 \rceil$ 小的元素）。

**分析**：直接使用主席树查询区间第 $k$ 小，其中 $k = \lceil (r-l+1)/2 \rceil$。这是最基本的主席树应用之一。

```
数据结构：主席树
支持操作：区间第 k 小查询（k = 区间长度/2 上取整）
关键性质：中位数 = 第 k 小，k = ⌈len/2⌉
时间复杂度：每次查询 O(log n)
```

**C++ 代码**：

```cpp
int queryMedian(int l, int r) {
    int len = r - l + 1;
    int k = (len + 1) / 2;
    int rank = query(root[l-1], root[r], 1, m, k);
    return b[rank]; // b[] 为离散化的原值数组
}
```

**核心启示**：
- 中位数是第 $k$ 小的特例，主席树可以直接求解
- 离散化后需要映射回原值

### 题目 9：Count on a tree（洛谷 P2633）

> **题意：** 给定一棵 $n$ 个节点的带权树，每次查询路径 $(u, v)$ 上的第 $k$ 小点权。

**分析**：经典的"树上路径第 $k$ 小"问题。对每个节点建一个主席树版本（从根 DFS 而来），利用 LCA 做版本差分。在四棵树上同步行走。

```
数据结构：主席树 + LCA（倍增）
支持操作：DFS 建版本 + 四版本差分查询
关键性质：path(u,v) 差分 = T_u + T_v - T_lca - T_fa(lca)
时间复杂度：预处理 O(n log n)，查询 O(log n)
```

**C++ 代码**：

```cpp
void dfs(int u, int fa) {
    root[u] = update(root[fa], 1, m, val[u]);
    dep[u] = dep[fa] + 1;
    f[u][0] = fa;
    for (int i = 1; i < LOG; i++)
        f[u][i] = f[f[u][i-1]][i-1];
    for (int v : adj[u])
        if (v != fa) dfs(v, u);
}

int query(int u, int v, int lca, int flca, int l, int r, int k) {
    if (l == r) return l;
    int mid = (l + r) >> 1;
    int d = cnt[ls[u]] + cnt[ls[v]] - cnt[ls[lca]] - cnt[ls[flca]];
    if (k <= d)
        return query(ls[u], ls[v], ls[lca], ls[flca], l, mid, k);
    else
        return query(rs[u], rs[v], rs[lca], rs[flca], mid + 1, r, k - d);
}
```

**核心启示**：
- 树上前缀和版本差分是主席树在树形结构上的自然推广
- 公式 $T_u + T_v - T_{lca} - T_{fa(lca)}$ 是核心

### 题目 10：美味（洛谷 P3293）

> **题意：** 给定序列 $a$，每次查询给定 $b, x, l, r$，求 $\max(b \oplus a_i)$，其中 $l \le i \le r$。即区间内与 $b$ 异或最大的值。

**分析**：经典的按位贪心（类似 Trie 上求最大异或），但限制在区间 $[l,r]$ 内。用主席树代替 Trie，从高位到低位贪心选择。每一位判断值域中对应范围是否有元素。

```
数据结构：主席树
支持操作：版本差分 + 按位贪心值域查询
关键性质：每一位判断能否使异或结果为 1
时间复杂度：每次查询 O(log^2 n) 或 O(log n)
```

**C++ 代码**：

```cpp
int countRange(int u, int v, int l, int r, int vl, int vr) {
    if (vl <= l && r <= vr) return cnt[v] - cnt[u];
    int mid = (l + r) >> 1, res = 0;
    if (vl <= mid) res += countRange(ls[u], ls[v], l, mid, vl, vr);
    if (vr > mid)  res += countRange(rs[u], rs[v], mid + 1, r, vl, vr);
    return res;
}

int solve(int l, int r, int b) {
    int ans = 0;
    for (int bit = LOG - 1; bit >= 0; bit--) {
        int want = ((b >> bit) & 1) ^ 1; // 希望该位为 1
        int lo = (ans | (want << bit));
        int hi = lo | ((1 << bit) - 1);
        if (lo <= m && countRange(root[l-1], root[r], 1, m,
                                   lo + 1, min(hi + 1, m)) > 0)
            ans |= (want << bit);
        else
            ans |= (((b >> bit) & 1) << bit);
    }
    return ans ^ b;
}
```

**核心启示**：
- 主席树可以作为"区间限定的值域查询工具"
- 将 Trie 上的贪心推广到区间问题

### 题目 12：区间数颜色（在线做法）

> **题意：** 给定长度为 $n$ 的序列，$m$ 次在线询问区间 $[l, r]$ 中有多少种不同的数。

**分析**：与题目 4 类似，但要求在线。利用 pre 数组（每个位置上次相同值出现的位置），为每个位置建主席树版本。版本 $i$ 表示只保留每个数最后出现的位置。查询 $[l, r]$ 中位置值 $\geq l$ 的个数。

```
数据结构：主席树（按位置维度建版本）
支持操作：在线查询区间不同数个数
关键性质：pre[i] 标记上次出现位置，版本 i 删除 pre[i] 处标记
时间复杂度：建树 O(n log n)，查询 O(log n)
```

**C++ 代码**：

```cpp
void solve() {
    root[0] = build(1, n);
    for (int i = 1; i <= n; i++) {
        if (pre[i])
            root[i] = update(root[i-1], 1, n, pre[i], -1);
        else
            root[i] = root[i-1];
        root[i] = update(root[i], 1, n, i, 1);
    }
    // query [l, r]: 在 root[r] 上查询位置 [l, n] 中 cnt 之和
}

int query_range(int p, int l, int r, int ql, int qr) {
    if (ql <= l && r <= qr) return cnt[p];
    int mid = (l + r) >> 1, res = 0;
    if (ql <= mid) res += query_range(ls[p], l, mid, ql, qr);
    if (qr > mid)  res += query_range(rs[p], mid + 1, r, ql, qr);
    return res;
}
```

**核心启示**：
- 在线数颜色需要将"位置维度"作为线段树下标
- 利用 pre 数组去重是核心技巧

### 题目 13：森林（洛谷 P3302）

> **题意：** 给定一棵带点权的森林，支持两种操作：查询两点路径上的第 $k$ 小点权；连接两棵树中的一条边。

**分析**：树上路径第 $k$ 小用主席树 + LCA 解决。连边操作使用启发式合并（按 size 合并），每次把小树重新接入大树并重建对应版本。

```
数据结构：主席树 + LCA + 启发式合并
支持操作：路径第 k 小 + 动态连边
关键性质：启发式合并保证每个节点最多重建 O(log n) 次
时间复杂度：O(n log^2 n)
```

**C++ 代码**：

```cpp
void merge_tree(int u, int v) {
    if (sz[find(u)] < sz[find(v)]) swap(u, v);
    link(u, v);
    dep[v] = dep[u] + 1;
    fa[v][0] = u;
    rebuild(v, u); // DFS 重建小树的主席树版本和倍增数组
}

void rebuild(int u, int par) {
    root[u] = update(root[par], 1, m, val[u]);
    f[u][0] = par;
    for (int i = 1; i < LOG; i++)
        f[u][i] = f[f[u][i-1]][i-1];
    for (int v : adj[u])
        if (v != par && find(v) == find(u))
            rebuild(v, u);
}
```

**核心启示**：
- 启发式合并保证每个节点最多被重建 $O(\log n)$ 次
- 与主席树结合可处理动态连边的树上问题

### 题目 14：列队（主席树 + 逆序对统计）

**分析**：利用主席树做区间值域统计。对每个被选中的人，用主席树版本差分统计区间内比它大的数的个数，累加得到逆序对数。

```
数据结构：主席树
支持操作：版本差分 + 区间内大于某值的元素个数
关键性质：逆序对贡献 = 对每个位置 i，统计 [l, i-1] 中 > a[i] 的个数
时间复杂度：O(n log n) 分治或 O(n log n) 主席树
```

**C++ 代码**：

```cpp
// 查询区间 [ql,qr] 中 > val 的元素个数
int countGT(int u, int v, int l, int r, int val) {
    if (l == r) return (l > val) ? (cnt[v] - cnt[u]) : 0;
    int mid = (l + r) >> 1;
    if (val >= mid) return countGT(rs[u], rs[v], mid + 1, r, val);
    int rc = cnt[rs[v]] - cnt[rs[u]];
    return rc + countGT(ls[u], ls[v], l, mid, val);
}

long long countInv(int l, int r) {
    long long inv = 0;
    for (int i = l; i <= r; i++)
        inv += countGT(root[l-1], root[i-1], 1, m, a[i]);
    return inv;
}
```

**核心启示**：
- 主席树的区间版本差分可用来高效统计区间中特定值域的元素个数
- 适用于逆序对、排名等计数问题

### 题目 15：可持久化并查集（洛谷 P3402）

> **题意：** 给定 $n$ 个初始独立的元素，支持三种操作：1）合并两个元素所在集合；2）回退到某个历史版本；3）查询两元素是否在同一集合。

**分析**：可持久化并查集 = 可持久化数组 + 按秩合并。用可持久化线段树维护 fa 和 rank 数组。每次合并在对应版本上修改 fa 指针。注意不能用路径压缩（会破坏历史版本），必须用按秩合并保证深度 $O(\log n)$。

```
数据结构：可持久化线段树（模拟数组）
支持操作：版本回退 + 并查集合并 + 查询连通性
关键性质：按秩合并保证深度 O(log n)，不能路径压缩
时间复杂度：每次操作 O(log^2 n)
```

**C++ 代码**：

```cpp
int find(int ver, int x) {
    int fa = query(root[ver], 1, n, x); // 查询版本 ver 中 x 的 fa
    if (fa == x) return x;
    return find(ver, fa);
}

void merge(int ver, int x, int y) {
    int fx = find(ver, x), fy = find(ver, y);
    if (fx == fy) { root[++cur] = root[ver]; return; }
    int rx = query_rank(root[ver], 1, n, fx);
    int ry = query_rank(root[ver], 1, n, fy);
    if (rx < ry) swap(fx, fy), swap(rx, ry);
    // fa[fy] = fx
    root[++cur] = update(root[ver], 1, n, fy, fx);
    if (rx == ry)
        root[cur] = update_rank(root[cur], 1, n, fx, rx + 1);
}
```

**核心启示**：
- 可持久化并查集是"可持久化数组 + 按秩合并"的组合
- 不能用路径压缩是核心限制——因为路径压缩会修改历史版本

### 题目 16：区间 mex + 主席树（区间 mex 在线查询）

> **题意：** 给定长度为 $n$ 的非负整数序列，$m$ 次在线查询区间 $[l, r]$ 的 mex（最小未出现的非负整数）。

**分析**：对每个位置 $i$，记录 $a[i]$ 上一次出现的位置 $pre[i]$。建主席树，版本 $r$ 中维护每个值最后出现的位置。查询区间 $[l, r]$ 的 mex = 在版本 $r$ 上找值域中最后出现位置 $< l$ 的最小值。在主席树上二分查找。

```
数据结构：主席树（权值线段树，维护每个值的最后出现位置的最小值）
支持操作：版本建树 + 线段树上二分
关键性质：mex = 值域中 last_pos < l 的最小值
时间复杂度：建树 O(n log n)，查询 O(log n)
```

**C++ 代码**：

```cpp
// 节点维护子树中最后出现位置的最小值
int mn[MAXM]; // 子树中所有值的 last_pos 的最小值

int build(int l, int r) {
    int p = ++tot;
    mn[p] = 0; // 初始时所有值的 last_pos = 0 (< 任何 l)
    if (l == r) return p;
    int mid = (l + r) >> 1;
    ls[p] = build(l, mid);
    rs[p] = build(mid + 1, r);
    return p;
}

int update(int prev, int l, int r, int x, int pos) {
    int p = ++tot;
    ls[p] = ls[prev]; rs[p] = rs[prev];
    if (l == r) { mn[p] = pos; return p; }
    int mid = (l + r) >> 1;
    if (x <= mid) ls[p] = update(ls[prev], l, mid, x, pos);
    else          rs[p] = update(rs[prev], mid + 1, r, x, pos);
    mn[p] = min(mn[ls[p]], mn[rs[p]]);
    return p;
}

int query(int p, int l, int r, int ql) {
    if (l == r) return l;
    int mid = (l + r) >> 1;
    // 左子树有 last_pos < ql 的值，优先进左子树
    if (mn[ls[p]] < ql) return query(ls[p], l, mid, ql);
    else return query(rs[p], mid + 1, r, ql);
}

void solve() {
    root[0] = build(0, MAXV);
    for (int i = 1; i <= n; i++)
        root[i] = update(root[i-1], 0, MAXV, a[i], i);
    // 查询 [l, r] 的 mex
    // query(root[r], 0, MAXV, l)
}
```

**核心启示**：
- 主席树可以维护"值到位置"的映射，支持在线段树上二分
- mex 问题的关键转化：mex = 值域中最后出现位置 $< l$ 的最小值
- 这展示了主席树在非传统"第 k 小"场景下的强大能力

## 五、常见技巧与思路总结

### 5.1 空间估算

主席树的空间是最易出错的地方。总节点数上界为：

$$\text{节点数} = \text{初始建树} + n \times \lceil\log_2 m\rceil$$

一般开 $40n$ 到 $50n$ 的节点池即可。带修改版本需要 $O(n\log^2 n)$，需开更大。

### 5.2 离散化

当值域过大时（如 $10^9$），必须先离散化：

```cpp
sort(b + 1, b + n + 1);
m = unique(b + 1, b + n + 1) - b - 1;
for (int i = 1; i <= n; i++)
    a[i] = lower_bound(b + 1, b + m + 1, a[i]) - b;
```

### 5.3 动态开点 vs 预先建树

- **预先建空树：** 结构整齐，代码简洁，适合值域确定时使用
- **动态开点：** 省空间，适合值域极大或带修改的场景

### 5.4 避免常见错误

1. **MLE：** 节点池开小了。计算时别忘了初始建树也占空间
2. **WA：** 离散化后忘记映射回原值
3. **RE：** `tot` 溢出节点池。调试时可加 `assert(tot < MAXM)`
4. **TLE：** 带修改时忘记用树状数组管理，复杂度退化

### 5.5 调试方法

可以打印某个版本的线段树来检查正确性：

```cpp
void print(int p, int l, int r, int dep = 0) {
    if (!p) return;
    if (l == r) { if (cnt[p]) printf("%*s[%d]: %d\n", dep*2, "", l, cnt[p]); return; }
    int mid = (l + r) >> 1;
    print(ls[p], l, mid, dep + 1);
    print(rs[p], mid + 1, r, dep + 1);
}
```

## 七、终极总结与归纳

### 7.1 主席树的本质

**主席树 = 权值线段树 + 可持久化（共享节点）+ 版本差分。** 通过为每个前缀维护一个版本，利用差分思想将"区间查询"转化为"两个版本的差"。

核心公式：
- **前缀版本差分：** $cnt_{[l,r]} = T_r - T_{l-1}$
- **树上路径差分：** $cnt_{path(u,v)} = T_u + T_v - T_{lca} - T_{fa(lca)}$
- **空间复杂度：** $O(n\log n)$（静态），$O(n\log^2 n)$（带修改）

### 7.2 核心要点

1. **版本差分是核心**：所有查询归结为两个或多个版本的差
2. **共享节点省空间**：每次只新建路径上 $O(\log n)$ 个节点
3. **离散化是必要步骤**：值域过大时必须压缩
4. **多种查询**：第 k 小、排名、前驱/后继、区间统计均可支持
5. **树上推广**：DFS 建版本 + LCA 差分 = 树上路径查询
6. **带修改**：树状数组套主席树，代价为多一个 $\log$
