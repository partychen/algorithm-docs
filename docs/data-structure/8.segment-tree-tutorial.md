## 一、什么是线段树

线段树（Segment Tree）是一种基于**分治思想**的二叉树数据结构，用于高效处理**区间查询**与**区间修改**问题。它将一个长度为 $n$ 的数组递归地划分为若干子区间，每个节点维护对应区间的聚合信息（如区间和、最大值、最小值等）。

**核心能力：**

| 操作 | 朴素做法 | 线段树 |
|------|---------|--------|
| 单点修改 | $O(1)$ | $O(\log n)$ |
| 区间查询 | $O(n)$ | $O(\log n)$ |
| 区间修改 | $O(n)$ | $O(\log n)$（带懒标记） |

线段树的节点编号通常采用**堆式存储**：对于节点 $p$，左儿子为 $2p$，右儿子为 $2p+1$。因此数组大小一般开到 $4n$。

**适用场景：**
- 需要同时支持**修改**和**查询**的区间问题
- 满足**区间可合并性**（即两个子区间的信息可以合并为父区间的信息）
- 在线回答多次查询

## 二、核心思想与前置知识

### 2.1 建树（Build）

建树的过程是一个递归过程：将区间 $[l, r]$ 从中点 $mid = \lfloor(l+r)/2\rfloor$ 分成 $[l, mid]$ 和 $[mid+1, r]$，递归建树后向上合并信息（pushup）。

```
         [1,4] sum=10
        /          \
    [1,2] sum=3    [3,4] sum=7
    /    \          /    \
[1,1]=1 [2,2]=2 [3,3]=3 [4,4]=4
```

```cpp
void build(int p, int l, int r) {
    if (l == r) { tree[p] = a[l]; return; }
    int mid = (l + r) >> 1;
    build(p * 2, l, mid);
    build(p * 2 + 1, mid + 1, r);
    tree[p] = tree[p * 2] + tree[p * 2 + 1]; // pushup
}
```

**时间复杂度：** $O(n)$，每个叶子节点恰好被访问一次。

### 2.2 单点修改与区间查询

**单点修改**：从根节点出发，根据目标位置与当前区间中点的关系，递归进入左或右子树，到达叶子后修改，回溯时 pushup 更新祖先。

```cpp
void update(int p, int l, int r, int pos, long long val) {
    if (l == r) { tree[p] += val; return; }
    int mid = (l + r) >> 1;
    if (pos <= mid) update(p * 2, l, mid, pos, val);
    else update(p * 2 + 1, mid + 1, r, pos, val);
    tree[p] = tree[p * 2] + tree[p * 2 + 1];
}
```

**区间查询**：当查询区间 $[ql, qr]$ 完全覆盖当前节点区间时直接返回；否则递归查询左右子树中与查询区间有交集的部分，合并结果。

```cpp
long long query(int p, int l, int r, int ql, int qr) {
    if (ql <= l && r <= qr) return tree[p];
    int mid = (l + r) >> 1;
    long long res = 0;
    if (ql <= mid) res += query(p * 2, l, mid, ql, qr);
    if (qr > mid) res += query(p * 2 + 1, mid + 1, r, ql, qr);
    return res;
}
```

### 2.3 懒标记（Lazy Propagation）

当需要**区间修改**时，如果每次都递归到叶子节点，复杂度退化为 $O(n)$。懒标记的核心思想是：**修改到"刚好覆盖"的节点就停下来**，把修改信息暂存在该节点的懒标记中，等到后续需要访问其子节点时再下传（pushdown）。

**懒标记的三个关键操作：**

1. **打标记（addtag）**：给节点附加一个待下传的修改操作
2. **下传标记（pushdown）**：将父节点的懒标记传给两个子节点
3. **上传信息（pushup）**：用子节点信息更新父节点

```cpp
void pushdown(int p, int l, int r) {
    if (lazy[p]) {
        int mid = (l + r) >> 1;
        tree[p*2] += lazy[p] * (mid - l + 1);
        tree[p*2+1] += lazy[p] * (r - mid);
        lazy[p*2] += lazy[p];
        lazy[p*2+1] += lazy[p];
        lazy[p] = 0;
    }
}

void range_update(int p, int l, int r, int ql, int qr, long long val) {
    if (ql <= l && r <= qr) {
        tree[p] += val * (r - l + 1);
        lazy[p] += val;
        return;
    }
    pushdown(p, l, r);
    int mid = (l + r) >> 1;
    if (ql <= mid) range_update(p*2, l, mid, ql, qr, val);
    if (qr > mid) range_update(p*2+1, mid+1, r, ql, qr, val);
    tree[p] = tree[p*2] + tree[p*2+1];
}
```

> **要点：** 在任何需要访问子节点信息的操作（修改、查询）之前，必须先 pushdown。

### 2.4 线段树合并

线段树合并是指将两棵**动态开点**线段树合并为一棵。常用于树上问题（如树上每个节点维护一棵权值线段树，自底向上合并子节点的线段树）。

```cpp
int merge(int p, int q, int l, int r) {
    if (!p || !q) return p + q;
    if (l == r) {
        val[p] += val[q];
        return p;
    }
    int mid = (l + r) >> 1;
    ls[p] = merge(ls[p], ls[q], l, mid);
    rs[p] = merge(rs[p], rs[q], mid + 1, r);
    pushup(p);
    return p;
}
```

**时间复杂度：** 单次合并为 $O(\text{重叠节点数})$，总合并复杂度在很多场景下可以分析为 $O(n \log n)$。

### 2.5 扫描线

扫描线是线段树的一个经典应用，主要用于**矩形面积并**、**矩形周长并**等几何问题。其思想是：将矩形按一个坐标轴排序后，用另一个坐标轴上的线段树维护"被覆盖的长度"。

```
          扫描方向 →
  ┌───────┐
  │  矩形1 │    ┌────────┐
  │       │    │  矩形2  │
  └───────┘    │        │
               └────────┘
  对每条竖线，用线段树维护纵坐标上被覆盖的总长度
```

```cpp
struct ScanNode {
    int cnt;
    double len;
};
ScanNode t[MAXN * 4];

void pushup(int p, int l, int r, vector<double>& ys) {
    if (t[p].cnt > 0)
        t[p].len = ys[r + 1] - ys[l];
    else if (l == r)
        t[p].len = 0;
    else
        t[p].len = t[p*2].len + t[p*2+1].len;
}

void update(int p, int l, int r, int ql, int qr, int val, vector<double>& ys) {
    if (ql <= l && r <= qr) {
        t[p].cnt += val;
        pushup(p, l, r, ys);
        return;
    }
    int mid = (l + r) >> 1;
    if (ql <= mid) update(p*2, l, mid, ql, qr, val, ys);
    if (qr > mid) update(p*2+1, mid+1, r, ql, qr, val, ys);
    pushup(p, l, r, ys);
}
```

## 三、分类与变体详解

| 变体 | 特点 | 典型应用 |
|------|------|---------|
| **基本线段树** | 单点修改 + 区间查询 | 区间和、区间最值 |
| **双懒标记线段树** | 同时维护加法和乘法两种懒标记 | P3373 模板题（区间乘 + 区间加） |
| **动态开点线段树** | 不预先建满整棵树，按需新建节点 | 值域很大（如 $10^9$）、可持久化 |
| **权值线段树** | 以值域为下标，维护每个值出现的次数 | 第 $k$ 大/小、排名查询 |
| **扫描线线段树** | 配合扫描线处理几何问题 | 矩形面积并、矩形周长并 |
| **线段树合并** | 将多棵动态开点线段树合并 | 树上统计、分治合并 |

**选型指南：**
- 区间修改 + 区间查询 → 带懒标记的基本线段树
- 值域大或稀疏 → 动态开点
- 树上每个节点需要独立信息 → 线段树合并
- 矩形相关几何问题 → 扫描线
- 需要同时维护多种运算 → 双懒标记

### 题目 1：区间加 + 区间求和（洛谷 P3372）

> **题意：** 给定长度为 $n$ 的数列，支持两种操作：将区间 $[l,r]$ 内每个数加上 $k$；查询区间 $[l,r]$ 的和。

**分析：**

```
数据结构：线段树 + 加法懒标记
支持操作：区间加、区间求和
关键性质：pushdown 时将懒标记传给子节点，同时更新子节点的 sum 值；修改与查询前都要先 pushdown
时间复杂度：O(n + q log n)
```

**C++ 代码：**

```cpp
long long tree[N*4], lazy[N*4];

void pushdown(int p, int l, int r) {
    if (!lazy[p]) return;
    int mid = (l + r) >> 1;
    tree[p*2] += lazy[p] * (mid - l + 1);
    tree[p*2+1] += lazy[p] * (r - mid);
    lazy[p*2] += lazy[p]; lazy[p*2+1] += lazy[p];
    lazy[p] = 0;
}

void update(int p, int l, int r, int ql, int qr, long long v) {
    if (ql <= l && r <= qr) { tree[p] += v*(r-l+1); lazy[p] += v; return; }
    pushdown(p, l, r); int mid = (l+r) >> 1;
    if (ql <= mid) update(p*2, l, mid, ql, qr, v);
    if (qr > mid) update(p*2+1, mid+1, r, ql, qr, v);
    tree[p] = tree[p*2] + tree[p*2+1];
}

long long query(int p, int l, int r, int ql, int qr) {
    if (ql <= l && r <= qr) return tree[p];
    pushdown(p, l, r); int mid = (l+r) >> 1;
    long long res = 0;
    if (ql <= mid) res += query(p*2, l, mid, ql, qr);
    if (qr > mid) res += query(p*2+1, mid+1, r, ql, qr);
    return res;
}
```

**核心启示：**
- 懒标记是线段树处理区间修改的基石
- pushdown 的时机是"需要访问子节点之前"
- 本题是所有线段树题目的基础，务必完全掌握

### 题目 2：区间乘 + 区间加 + 区间求和（洛谷 P3373）

> **题意：** 给定长度为 $n$ 的数列和模数 $p$，支持三种操作：区间乘以 $k$、区间加上 $k$、查询区间和对 $p$ 取模的结果。

**分析：**

```
数据结构：线段树 + 双懒标记（乘法 mul 初始为 1，加法 add 初始为 0）
支持操作：区间乘、区间加、区间求和
关键性质：下传顺序为先乘后加；统一理解为仿射变换 x → mul·x + add
时间复杂度：O(n + q log n)
```

**C++ 代码：**

```cpp
long long tree[N*4], mul[N*4], add[N*4];
long long mod;

void pushdown(int p, int l, int r) {
    int mid = (l+r) >> 1;
    tree[p*2] = (tree[p*2]*mul[p] + add[p]*(mid-l+1)) % mod;
    tree[p*2+1] = (tree[p*2+1]*mul[p] + add[p]*(r-mid)) % mod;
    mul[p*2] = mul[p*2]*mul[p] % mod;
    mul[p*2+1] = mul[p*2+1]*mul[p] % mod;
    add[p*2] = (add[p*2]*mul[p] + add[p]) % mod;
    add[p*2+1] = (add[p*2+1]*mul[p] + add[p]) % mod;
    mul[p] = 1; add[p] = 0;
}
```

**核心启示：**
- 当存在多种操作时，懒标记的**数学含义**和**下传顺序**至关重要
- 统一理解为仿射变换 $x \mapsto mul \cdot x + add$，可以清晰推导出合并规则

### 题目 3：最大子段和（洛谷 P4513）

> **题意：** 给定长度为 $n$ 的数列，支持单点修改。多次查询区间 $[l,r]$ 内的最大子段和。

**分析：**

```
数据结构：线段树，每个节点维护 sum / lmax / rmax / ans 四个值
支持操作：单点修改、区间最大子段和查询
关键性质：pushup 通过组合左右子节点四个值来计算父节点；ans = max(左.ans, 右.ans, 左.rmax + 右.lmax)
时间复杂度：O(n + q log n)
```

**C++ 代码：**

```cpp
struct Node { long long sum, lmax, rmax, ans; } tree[N*4];

Node merge(Node L, Node R) {
    Node res;
    res.sum = L.sum + R.sum;
    res.lmax = max(L.lmax, L.sum + R.lmax);
    res.rmax = max(R.rmax, R.sum + L.rmax);
    res.ans = max({L.ans, R.ans, L.rmax + R.lmax});
    return res;
}

void update(int p, int l, int r, int pos, long long val) {
    if (l == r) { tree[p] = {val, val, val, val}; return; }
    int mid = (l+r) >> 1;
    if (pos <= mid) update(p*2, l, mid, pos, val);
    else update(p*2+1, mid+1, r, pos, val);
    tree[p] = merge(tree[p*2], tree[p*2+1]);
}
```

**核心启示：**
- **只要信息是可合并的，就能用线段树维护**
- 设计 merge 函数时要确保所有跨越左右子区间的情况都被覆盖
- 本题是理解"自定义 pushup"的绝佳范例

### 题目 4：扫描线求矩形面积并（洛谷 P5490）

> **题意：** 给定 $n$ 个矩形，求它们的面积并（所有矩形覆盖区域的总面积）。

**分析：**

```
数据结构：扫描线 + 线段树维护 cnt[]（覆盖次数）和 len[]（有效长度）
支持操作：区间加减（覆盖/取消覆盖）、全局查询有效覆盖长度
关键性质：cnt 不需要 pushdown，因为只在 cnt=0 时才用子节点信息；面积 = Σ(相邻 x 差 × 当前 y 覆盖长度)
时间复杂度：O(n log n)
```

**C++ 代码：**

```cpp
int cnt[N*4]; double len[N*4];
vector<double> ys;

void pushup(int p, int l, int r) {
    if (cnt[p] > 0) len[p] = ys[r+1] - ys[l];
    else if (l == r) len[p] = 0;
    else len[p] = len[p*2] + len[p*2+1];
}

void update(int p, int l, int r, int ql, int qr, int v) {
    if (ql <= l && r <= qr) { cnt[p] += v; pushup(p, l, r); return; }
    int mid = (l+r) >> 1;
    if (ql <= mid) update(p*2, l, mid, ql, qr, v);
    if (qr > mid) update(p*2+1, mid+1, r, ql, qr, v);
    pushup(p, l, r);
}
```

**核心启示：**
- 扫描线线段树的 `cnt` 标记不需要下传——`cnt > 0` 时当前区间被完整覆盖，`cnt = 0` 时才需要看子节点
- 这是一种特殊的"不下传的标记"

### 题目 5：操作序列乘积（洛谷 P4588）

> **题意：** 初始值 $x = 1$，支持两种操作：将 $x$ 乘以 $m$（记录操作编号）；将 $x$ 除以第 $pos$ 次操作乘的那个 $m$。每次操作后输出 $x \bmod M$。

**分析：**

```
数据结构：线段树维护操作序列上的乘积
支持操作：单点赋值、全局乘积查询
关键性质：乘操作 → 位置 i 赋值 m；除操作 → 位置 pos 赋值 1；根节点值即为当前 x
时间复杂度：O(q log q)
```

**C++ 代码：**

```cpp
long long tree[N*4]; int mod;

void build(int p, int l, int r) {
    tree[p] = 1;
    if (l == r) return;
    int mid = (l+r) >> 1;
    build(p*2, l, mid); build(p*2+1, mid+1, r);
}

void update(int p, int l, int r, int pos, long long val) {
    if (l == r) { tree[p] = val % mod; return; }
    int mid = (l+r) >> 1;
    if (pos <= mid) update(p*2, l, mid, pos, val);
    else update(p*2+1, mid+1, r, pos, val);
    tree[p] = tree[p*2] * tree[p*2+1] % mod;
}
```

**核心启示：**
- 线段树不仅能维护"数组上的信息"，还能维护"操作序列上的信息"
- 将难以逆向的操作（如取模下的除法）转化为线段树上的单点赋值，是一种重要的思维转换

### 题目 6：降雨量判定（洛谷 P2471）

> **题意：** 已知某些年份的降雨量数据（可能不连续），对于查询"$y_1$ 到 $y_2$ 年间，$y_1$ 的降雨量不小于中间所有年份，且 $y_2$ 的降雨量严格小于 $y_1$"是否成立，回答 `true`、`false` 或 `maybe`。

**分析：**

```
数据结构：线段树维护区间最大降雨量
支持操作：区间最大值查询
关键性质：将已知年份离散化后，根据 y1/y2 是否已知、中间是否有未知年份，综合判断三种结果
时间复杂度：O(n + q log n)
```

**C++ 代码：**

```cpp
int tree[N*4];

void build(int p, int l, int r) {
    if (l == r) { tree[p] = rain[l]; return; }
    int mid = (l+r) >> 1;
    build(p*2, l, mid); build(p*2+1, mid+1, r);
    tree[p] = max(tree[p*2], tree[p*2+1]);
}

int query(int p, int l, int r, int ql, int qr) {
    if (ql > qr) return 0;
    if (ql <= l && r <= qr) return tree[p];
    int mid = (l+r) >> 1, res = 0;
    if (ql <= mid) res = max(res, query(p*2, l, mid, ql, qr));
    if (qr > mid) res = max(res, query(p*2+1, mid+1, r, ql, qr));
    return res;
}
```

**核心启示：**
- 线段树本身可能只是工具，真正的难度在于**问题建模与边界分类**
- 这类"已知/未知"交织的问题需要极其细致的逻辑讨论

### 题目 7：李超线段树（洛谷 P4097）

> **题意：** 在平面上动态插入线段，查询 $x = x_0$ 处纵坐标最大的线段编号。

**分析：**

```
数据结构：李超线段树（Li Chao Tree），每个节点存储一条"在该区间中点处最优"的线段
支持操作：插入线段、单点查询最优线段
关键性质：插入时比较新线段与当前线段在中点处的大小，较优者留在当前节点，较劣者递归到可能更优的子区间；查询时从根到叶取路径上最大值
时间复杂度：插入 O(log^2 n)，查询 O(log n)
```

**C++ 代码：**

```cpp
struct Line { double k, b; } lines[N];
int tree[N*4];

double eval(int id, int x) { return lines[id].k * x + lines[id].b; }

void insert(int p, int l, int r, int id) {
    int mid = (l+r) >> 1;
    bool lbetter = eval(id, l) > eval(tree[p], l);
    bool mbetter = eval(id, mid) > eval(tree[p], mid);
    if (mbetter) swap(tree[p], id);
    if (l == r) return;
    if (lbetter != mbetter) insert(p*2, l, mid, id);
    else insert(p*2+1, mid+1, r, id);
}

double query(int p, int l, int r, int x) {
    if (l == r) return eval(tree[p], x);
    int mid = (l+r) >> 1;
    double res = eval(tree[p], x);
    if (x <= mid) return max(res, query(p*2, l, mid, x));
    else return max(res, query(p*2+1, mid+1, r, x));
}
```

**核心启示：**
- 李超线段树利用了"线段在子区间上的单调性"，巧妙地将线段的优劣比较限制在 $O(\log n)$ 次
- 它是线段树思想的一个优美变体

### 题目 8：窗口的星星（洛谷 P1502）

> **题意：** 平面上有 $n$ 颗星星，每颗有坐标 $(x, y)$ 和亮度 $c$。给定窗口大小 $W \times H$（边界上的不算），求窗口能框住的最大亮度和。

**分析：**

```
数据结构：扫描线 + 线段树维护区间最大值
支持操作：区间加、全局最大值查询
关键性质：每颗星星 (x,y,c) 产生矩形 [x, x+W-1] × [y, y+H-1]；问题转化为所有矩形覆盖后某点被覆盖的最大权值和；对 y 坐标离散化，按 x 排序扫描
时间复杂度：O(n log n)
```

**C++ 代码：**

```cpp
long long tree[N*4], lazy[N*4];

void pushdown(int p) {
    if (lazy[p]) {
        tree[p*2] += lazy[p]; lazy[p*2] += lazy[p];
        tree[p*2+1] += lazy[p]; lazy[p*2+1] += lazy[p];
        lazy[p] = 0;
    }
}

void update(int p, int l, int r, int ql, int qr, long long v) {
    if (ql <= l && r <= qr) { tree[p] += v; lazy[p] += v; return; }
    pushdown(p); int mid = (l+r) >> 1;
    if (ql <= mid) update(p*2, l, mid, ql, qr, v);
    if (qr > mid) update(p*2+1, mid+1, r, ql, qr, v);
    tree[p] = max(tree[p*2], tree[p*2+1]);
}
```

**核心启示：**
- 扫描线不仅能求面积并，还能求"最大覆盖权值"
- 关键是将"窗口能框住的星星"转化为"每颗星星产生的影响矩形"

### 题目 9：楼房重建（洛谷 P4198）

> **题意：** 有 $n$ 栋楼，初始高度为 0。每次修改一栋楼的高度后，求从位置 0 能看到多少栋楼。（能看到第 $i$ 栋楼当且仅当 $h_i / i$ 严格大于前面所有楼的 $h_j / j$。）

**分析：**

```
数据结构：线段树，每个节点维护区间最大 slope 和区间可见楼数
支持操作：单点修改、全局查询可见楼数
关键性质：pushup 时左区间贡献直接取 cnt[左]，右区间需递归查询"大于某阈值的前缀最大值个数"（线段树上二分）
时间复杂度：O(n log^2 n)
```

**C++ 代码：**

```cpp
double mx[N*4]; int cnt[N*4];

int calc(int p, int l, int r, double threshold) {
    if (mx[p] <= threshold) return 0;
    if (l == r) return mx[p] > threshold;
    int mid = (l+r) >> 1;
    if (mx[p*2] <= threshold) return calc(p*2+1, mid+1, r, threshold);
    return cnt[p] - cnt[p*2] + calc(p*2, l, mid, threshold);
}

void pushup(int p, int l, int r) {
    int mid = (l+r) >> 1;
    mx[p] = max(mx[p*2], mx[p*2+1]);
    cnt[p] = cnt[p*2] + calc(p*2+1, mid+1, r, mx[p*2]);
}
```

**核心启示：**
- 当 pushup 无法 $O(1)$ 完成时，可以在 pushup 中**递归查询**另一子区间，代价是多一个 $\log$
- 这种"线段树上二分"的技巧非常强大

### 题目 10：区间开方（洛谷 P4145）

> **题意：** 给定长度为 $n$ 的正整数数列，支持两种操作：将区间 $[l,r]$ 内每个数取整数下取整平方根；查询区间 $[l,r]$ 的和。

**分析：**

```
数据结构：线段树维护区间和与区间最大值
支持操作：区间开方（暴力递归到叶子）、区间求和
关键性质：若区间最大值 ≤ 1 则跳过（已全为 0 或 1，开方后不变）；每个数最多经过 O(log log V) 次开方后变为 1；势能分析保证总复杂度
时间复杂度：O(n √(log V) log n)（势能分析）
```

**C++ 代码：**

```cpp
long long tree[N*4], mx[N*4];

void build(int p, int l, int r) {
    if (l == r) { tree[p] = mx[p] = a[l]; return; }
    int mid = (l+r) >> 1;
    build(p*2, l, mid); build(p*2+1, mid+1, r);
    tree[p] = tree[p*2] + tree[p*2+1];
    mx[p] = max(mx[p*2], mx[p*2+1]);
}

void update(int p, int l, int r, int ql, int qr) {
    if (mx[p] <= 1) return;
    if (l == r) {
        tree[p] = mx[p] = (long long)sqrt((double)tree[p]);
        return;
    }
    int mid = (l+r) >> 1;
    if (ql <= mid) update(p*2, l, mid, ql, qr);
    if (qr > mid) update(p*2+1, mid+1, r, ql, qr);
    tree[p] = tree[p*2] + tree[p*2+1];
    mx[p] = max(mx[p*2], mx[p*2+1]);
}
```

**核心启示：**
- 某些操作看似每次 $O(n)$，但通过势能分析可以证明总复杂度低
- "区间最大值 ≤ 1 就跳过"是经典的势能剪枝技巧

### 题目 11：01 序列排序（洛谷 P2824）

> **题意：** 给定一个 $1 \sim n$ 的排列，执行 $m$ 次操作：对区间 $[l,r]$ 升序或降序排序。最后查询位置 $q$ 上的值。

**分析：**

```
数据结构：二分答案 + 01 线段树（区间赋值 + 区间求和）
支持操作：区间赋值（全 0 / 全 1）、区间求和
关键性质：二分答案 mid 后，≥ mid 标 1、< mid 标 0；排序操作变为对 01 序列的区间赋值；排完后查询位置 q 的值决定二分方向
时间复杂度：O(n log^2 n)
```

**C++ 代码：**

```cpp
int tree[N*4], lazy[N*4]; // lazy: -1 无标记, 0 全0, 1 全1

void assign(int p, int l, int r, int ql, int qr, int v) {
    if (ql <= l && r <= qr) { tree[p] = v*(r-l+1); lazy[p] = v; return; }
    pushdown(p, l, r); int mid = (l+r) >> 1;
    if (ql <= mid) assign(p*2, l, mid, ql, qr, v);
    if (qr > mid) assign(p*2+1, mid+1, r, ql, qr, v);
    tree[p] = tree[p*2] + tree[p*2+1];
}

// 升序: 查 1 的个数 cnt，赋值 [l,r-cnt]→0, [r-cnt+1,r]→1
// 降序: 查 1 的个数 cnt，赋值 [l,l+cnt-1]→1, [l+cnt,r]→0
```

**核心启示：**
- "二分答案将值域问题转化为 01 问题"是一种极其强大的技巧
- 01 序列的排序等价于区间赋值，使得线段树能高效处理

### 题目 12：影魔（洛谷 P3722）

> **题意：** 给定排列 $a[1..n]$，对于每对 $(i,j)$（$i < j$），设 $p = \max(a[i+1..j-1])$：若 $p < \min(a[i], a[j])$ 贡献 $p_1$；若 $p$ 介于 $a[i]$ 和 $a[j]$ 之间贡献 $p_2$。多次查询区间 $[l,r]$ 内所有对的总贡献。

**分析：**

```
数据结构：单调栈预处理 + 扫描线 + 线段树（区间加 + 前缀查询）
支持操作：区间加、前缀和查询
关键性质：对每个位置 i 用单调栈求 L[i] 和 R[i]（左右第一个 > a[i] 的位置），将点对贡献转化为二维平面上的矩形加法问题；离线按右端点排序后扫描线回答
时间复杂度：O(n log n)
```

**C++ 代码：**

```cpp
// 单调栈求左右第一个更大值位置
for (int i = 1; i <= n; i++) {
    while (!stk.empty() && a[stk.top()] < a[i]) {
        int j = stk.top(); stk.pop();
        R[j] = i;
    }
    if (!stk.empty()) L[i] = stk.top();
    stk.push(i);
}
// 对每个 i 生成贡献事件，挂在 x 坐标上
// 扫描线从左到右，线段树维护 y 轴上的区间加、前缀和
```

**核心启示：**
- 许多"统计满足条件的点对"问题可以通过单调栈预处理 + 贡献转化 + 扫描线来解决
- 将看似复杂的条件分解为几何上的矩形操作是关键

### 题目 13：雨天的尾巴（洛谷 P4556）

> **题意：** 树上有 $n$ 个节点，$m$ 次操作每次给路径 $(u,v)$ 上的所有节点发放一种物资 $z$。最后询问每个节点收到最多的物资种类。

**分析：**

```
数据结构：树上差分 + 动态开点权值线段树 + 线段树合并
支持操作：单点加减、查询全局最大值及其位置
关键性质：路径修改用树上差分拆解：diff[u]+1, diff[v]+1, diff[lca]-1, diff[fa[lca]]-1；自底向上合并子节点线段树后查询最大值对应种类
时间复杂度：O((n+m) log V)，V 为物资种类数
```

**C++ 代码：**

```cpp
int ls[M], rs[M], mx[M], mxid[M], tot;

void pushup(int p) {
    if (mx[ls[p]] >= mx[rs[p]]) { mx[p] = mx[ls[p]]; mxid[p] = mxid[ls[p]]; }
    else { mx[p] = mx[rs[p]]; mxid[p] = mxid[rs[p]]; }
}

void update(int &p, int l, int r, int pos, int v) {
    if (!p) p = ++tot;
    if (l == r) { mx[p] += v; mxid[p] = l; return; }
    int mid = (l+r) >> 1;
    if (pos <= mid) update(ls[p], l, mid, pos, v);
    else update(rs[p], mid+1, r, pos, v);
    pushup(p);
}

int merge(int p, int q, int l, int r) {
    if (!p || !q) return p + q;
    if (l == r) { mx[p] += mx[q]; return p; }
    int mid = (l+r) >> 1;
    ls[p] = merge(ls[p], ls[q], l, mid);
    rs[p] = merge(rs[p], rs[q], mid+1, r);
    pushup(p); return p;
}
```

**核心启示：**
- 树上差分 + 线段树合并是处理"树上路径修改 + 节点信息统计"的标准范式
- 动态开点保证了空间的可控性

### 题目 14：区间最多颜色（CF 600E）

> **题意：** 给定一棵 $n$ 个节点的有根树，每个节点有颜色。对每个节点 $u$，求子树中出现次数最多的颜色的编号之和（若有多个颜色出现次数相同，全部累加）。

**分析：**

```
数据结构：DSU on tree（树上启发式合并）+ 全局计数数组（也可用线段树合并实现）
支持操作：子树颜色频次统计、最大频次对应颜色编号和
关键性质：DSU on tree 利用重链性质，先处理轻子树（统计后撤销），再处理重子树（保留统计），最后将轻子树重新加入；也可直接用动态开点线段树合并
时间复杂度：O(n log n)（DSU on tree）或 O(n log n)（线段树合并）
```

**C++ 代码：**

```cpp
int col[N], sz[N], son[N];
long long ans[N], sum;
int cnt[N], mx_cnt;

void add(int u, int fa, int val) {
    cnt[col[u]] += val;
    if (cnt[col[u]] > mx_cnt) { mx_cnt = cnt[col[u]]; sum = col[u]; }
    else if (cnt[col[u]] == mx_cnt) sum += col[u];
    for (int v : adj[u]) if (v != fa) add(v, u, val);
}

void dfs(int u, int fa, bool keep) {
    for (int v : adj[u]) if (v != fa && v != son[u]) dfs(v, u, false);
    if (son[u]) dfs(son[u], u, true);
    for (int v : adj[u]) if (v != fa && v != son[u]) add(v, u, 1);
    cnt[col[u]]++;
    if (cnt[col[u]] > mx_cnt) { mx_cnt = cnt[col[u]]; sum = col[u]; }
    else if (cnt[col[u]] == mx_cnt) sum += col[u];
    ans[u] = sum;
    if (!keep) { add(u, fa, -1); cnt[col[u]]--; mx_cnt = 0; sum = 0; }
}
```

**核心启示：**
- DSU on tree（树上启发式合并）是处理子树统计问题的高效方法
- 该思想与线段树合并殊途同归，都利用了"重子树信息保留、轻子树信息重建"的策略

### 题目 15：删数（洛谷 P5324）

> **题意：** 给定长度为 $n$ 的数列，每次操作可以删除一个值等于其所在连续相同段长度的元素。支持单点修改和查询"最少删几次能清空"。

**分析：**

```
数据结构：线段树维护区间最小值和最小值个数
支持操作：区间加 +1/-1（覆盖/取消覆盖）、全局最小值及其个数查询
关键性质：值为 v 出现 cnt[v] 次 → 在数轴上 [v-cnt[v]+1, v] 被覆盖；答案 = 值域 [1,n] 中未被覆盖的位置个数（即最小值为 0 的位置个数）
时间复杂度：O(n + q log n)
```

**C++ 代码：**

```cpp
int mn[N*4], mnc[N*4], lazy[N*4];

void pushup(int p) {
    mn[p] = min(mn[p*2], mn[p*2+1]);
    mnc[p] = 0;
    if (mn[p] == mn[p*2]) mnc[p] += mnc[p*2];
    if (mn[p] == mn[p*2+1]) mnc[p] += mnc[p*2+1];
}

void apply(int p, int v) { mn[p] += v; lazy[p] += v; }

void pushdown(int p) {
    if (lazy[p]) { apply(p*2, lazy[p]); apply(p*2+1, lazy[p]); lazy[p] = 0; }
}
// 答案：mn[1] == 0 时为 mnc[1]，否则为 0
```

**核心启示：**
- 将"删除操作"抽象为值域上的"区间覆盖"，再用线段树维护覆盖信息，是一种高度抽象的建模技巧
- 线段树维护"区间最小值及其个数"是处理覆盖类问题的利器

### 题目 16：语言（洛谷 P5327）

## 五、技巧与进阶

### 5.1 线段树上二分

在线段树上直接二分查找，避免额外的 $O(\log n)$。例如"找到第一个前缀和 $\geq k$ 的位置"：

```cpp
int find(int p, int l, int r, long long k) {
    if (l == r) return l;
    int mid = (l+r) >> 1;
    if (tree[p*2] >= k) return find(p*2, l, mid, k);
    else return find(p*2+1, mid+1, r, k - tree[p*2]);
}
```

### 5.2 动态开点

值域过大时不预先分配所有节点，而是按需创建。空间复杂度从 $O(4V)$ 降为 $O(q \log V)$。

```cpp
int ls[M], rs[M], val[M], tot;
void update(int &p, int l, int r, int pos, int v) {
    if (!p) p = ++tot;
    if (l == r) { val[p] += v; return; }
    int mid = (l+r) >> 1;
    if (pos <= mid) update(ls[p], l, mid, pos, v);
    else update(rs[p], mid+1, r, pos, v);
    val[p] = val[ls[p]] + val[rs[p]];
}
```

### 5.3 标记永久化

在可持久化线段树等场景中，为避免 pushdown 产生新节点，可以让标记不下传，查询时累加路径上的所有标记。

### 5.4 势能分析

某些操作（如区间取模、区间开方）看似每次 $O(n)$，但通过势能分析可以证明总复杂度低。例如区间开方最多使每个元素变 1 约 $O(\log \log V)$ 次后变为 0 或 1 不再改变。

### 5.5 常数优化

- `p << 1` 代替 `p * 2`
- 避免在查询中传递大结构体（使用全局变量或引用）
- 减少不必要的 pushdown（如仅查询时可以只读不写）

## 六、代码框架

以下是一个通用的线段树模板框架，支持区间修改和区间查询：

```cpp
template<typename T>
struct SegTree {
    int n;
    vector<T> tree, lazy;

    SegTree(int _n) : n(_n), tree(4 * _n, 0), lazy(4 * _n, 0) {}

    void pushup(int p) {
        tree[p] = tree[p*2] + tree[p*2+1];
    }

    void apply(int p, int l, int r, T val) {
        tree[p] += val * (r - l + 1);
        lazy[p] += val;
    }

    void pushdown(int p, int l, int r) {
        if (lazy[p]) {
            int mid = (l + r) >> 1;
            apply(p*2, l, mid, lazy[p]);
            apply(p*2+1, mid+1, r, lazy[p]);
            lazy[p] = 0;
        }
    }

    void build(int p, int l, int r, vector<T>& a) {
        if (l == r) { tree[p] = a[l]; return; }
        int mid = (l + r) >> 1;
        build(p*2, l, mid, a);
        build(p*2+1, mid+1, r, a);
        pushup(p);
    }

    void update(int p, int l, int r, int ql, int qr, T val) {
        if (ql <= l && r <= qr) { apply(p, l, r, val); return; }
        pushdown(p, l, r);
        int mid = (l + r) >> 1;
        if (ql <= mid) update(p*2, l, mid, ql, qr, val);
        if (qr > mid) update(p*2+1, mid+1, r, ql, qr, val);
        pushup(p);
    }

    T query(int p, int l, int r, int ql, int qr) {
        if (ql <= l && r <= qr) return tree[p];
        pushdown(p, l, r);
        int mid = (l + r) >> 1;
        T res = 0;
        if (ql <= mid) res += query(p*2, l, mid, ql, qr);
        if (qr > mid) res += query(p*2+1, mid+1, r, ql, qr);
        return res;
    }
};
```

| pushup | 子节点信息合并为父节点，是线段树正确性的基础 |
| pushdown | 懒标记下传，是区间修改高效的关键 |
| 信息可合并性 | 设计线段树的前提：子区间信息能 $O(1)$ 合并 |
| 空间 $4n$ | 堆式存储需要 $4n$ 空间，动态开点则按需分配 |

### 7.2 常见错误与调试

1. **忘记 pushdown**：在任何需要访问子节点的操作之前，必须先下传标记
2. **pushup 遗漏**：修改子节点后忘记更新父节点
3. **数组开小**：线段树数组要开到 $4n$，动态开点要估计最大节点数
4. **边界混淆**：$[l, mid]$ 和 $[mid+1, r]$ 不要写错
5. **懒标记初始值**：乘法标记初始为 1，加法标记初始为 0，区间赋值标记需要特殊的"无效值"
6. **long long 溢出**：取模题目中乘法运算前要注意类型

### 7.3 复杂度速查表

| 操作 | 时间复杂度 |
|------|-----------|
| 建树 | $O(n)$ |
| 单点修改/查询 | $O(\log n)$ |
| 区间修改/查询（带懒标记） | $O(\log n)$ |
| 线段树合并（总计） | $O(n \log n)$ |
| 线段树上二分 | $O(\log n)$ |
| 李超线段树插入 | $O(\log^2 n)$ |

### 7.4 与其他数据结构的对比

| 数据结构 | 优势 | 劣势 |
|---------|------|------|
| **线段树** | 适用性广，支持多种区间操作 | 代码量大，常数较高 |
| **树状数组** | 代码简短，常数小 | 只支持前缀类操作，扩展性有限 |
| **分块** | 实现简单，适合不好用线段树的问题 | 复杂度 $O(\sqrt{n})$，不如 $O(\log n)$ |
| **平衡树** | 支持序列的分裂与合并 | 常数大，调试困难 |
