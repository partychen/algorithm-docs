## 一、什么是树套树

**树套树**是一种将两种树形数据结构嵌套组合的高级技巧，用于处理**多维度查询**问题。其核心思路是：外层用一棵树（如线段树、树状数组）维护一个维度（通常是区间/位置），内层在外层的每个节点上再挂一棵树（如平衡树、主席树）维护另一个维度（通常是值域/权值）。

**典型应用场景：**
- 区间第 $k$ 小 / 区间排名
- 区间内某值的前驱 / 后继
- 带修改的区间统计问题
- 二维偏序、三维偏序等多维查询

**为什么需要树套树？**

| 单一结构 | 能力 | 局限 |
|---------|------|------|
| 线段树 | 区间查询与修改 | 无法高效处理值域上的排名/前驱后继 |
| 平衡树 | 排名、前驱、后继 | 无法高效处理任意区间 |
| 主席树 | 静态区间第 $k$ 小 | 不支持修改 |

将它们嵌套组合后，就能同时在**位置维度**和**值域维度**上高效操作。

## 二、核心思想与前置知识

### 2.1 前置知识

学习树套树之前需要牢固掌握以下内容：

1. **线段树**：区间查询、区间修改、懒标记下传
2. **平衡树（FHQ Treap / Splay）**：插入、删除、排名、第 $k$ 小、前驱、后继
3. **树状数组**：前缀和、单点修改
4. **主席树（可持久化线段树）**：动态开点、版本管理
5. **CDQ 分治 / 整体二分**（作为替代方案）

### 2.2 线段树套平衡树

这是最经典的树套树组合。外层线段树按**下标区间**建树，每个线段树节点维护一棵**平衡树（FHQ Treap）**，存储该区间内所有元素的值。

- **区间排名**：在线段树上拆分为 $O(\log n)$ 个节点，在每棵平衡树上查排名后求和
- **区间第 $k$ 小**：二分答案 + 区间排名验证，总复杂度 $O(\log^3 n)$
- **区间前驱/后继**：在 $O(\log n)$ 棵平衡树上分别查前驱/后继，取最优
- **单点修改**：在线段树路径上的 $O(\log n)$ 棵平衡树中删除旧值、插入新值

**复杂度：** 单次操作 $O(\log^2 n)$ 至 $O(\log^3 n)$，空间 $O(n \log n)$。

### 2.3 树状数组套主席树

当需要**带修改的区间第 $k$ 小**时，可以用树状数组套主席树：

- 外层树状数组管理修改的"差分"关系
- 内层对每个树状数组位置维护一棵动态开点的权值线段树（主席树思想）
- 查询区间第 $k$ 小时，同时维护 $O(\log n)$ 棵树的根，在值域线段树上二分

**复杂度：** 每次操作 $O(\log^2 n)$，空间 $O(n \log^2 n)$（需要动态开点）。

### 2.4 外层线段树 + 内层平衡树的操作流程

以「区间排名查询 + 单点修改」为例：

```
查询 query_rank(l, r, val):
  1. 在外层线段树上，将 [l, r] 分解为 O(log n) 个节点
  2. 对每个节点的平衡树，查询 val 的排名（即小于 val 的元素个数）
  3. 将所有结果求和，再加 1 即为 val 在 [l, r] 中的排名

修改 update(pos, old_val, new_val):
  1. 从线段树根到叶子，经过 O(log n) 个节点
  2. 在每个节点的平衡树中，删除 old_val，插入 new_val
```

### 2.5 C++ 代码骨架

下面给出线段树套 FHQ Treap 的核心骨架：

```cpp
const int MAXN = 50005;
struct TreapNode {
    int l, r, val, key, sz, cnt;
} tr[MAXN * 40];
int tot, root[MAXN * 4];

// Treap 基本操作（省略具体实现）
void treap_insert(int &o, int val);
void treap_delete(int &o, int val);
int treap_rank(int o, int val);       // 小于 val 的个数
int treap_kth(int o, int k);
int treap_pre(int o, int val);
int treap_suc(int o, int val);

// 线段树部分
void seg_update(int o, int l, int r, int pos, int oldv, int newv) {
    treap_delete(root[o], oldv);
    treap_insert(root[o], newv);
    if (l == r) return;
    int mid = (l + r) >> 1;
    if (pos <= mid) seg_update(o*2, l, mid, pos, oldv, newv);
    else seg_update(o*2+1, mid+1, r, pos, oldv, newv);
}

int seg_query_rank(int o, int l, int r, int ql, int qr, int val) {
    if (ql <= l && r <= qr) return treap_rank(root[o], val);
    int mid = (l + r) >> 1, res = 0;
    if (ql <= mid) res += seg_query_rank(o*2, l, mid, ql, qr, val);
    if (qr > mid) res += seg_query_rank(o*2+1, mid+1, r, ql, qr, val);
    return res;
}
```

## 三、分类与变体详解

### 3.1 线段树套 FHQ Treap

- **最经典**的树套树方案，支持区间排名、前驱、后继、第 $k$ 小、单点修改
- 常数较大但功能全面，适用于 P3380 等综合题
- 内层使用无旋 Treap（FHQ Treap）更易编写

### 3.2 线段树套线段树

- 外层按位置建线段树，内层按**值域**建线段树（动态开点）
- 适合需要**区间修改 + 值域查询**的场景（如 P3332 K 大数查询）
- 内层需要动态开点，否则空间爆炸。空间 $O(n \log^2 n)$
- 支持标记下传时需要格外小心：外层的 lazy 标记需要对内层整棵线段树生效

### 3.3 树状数组套主席树（BIT 套主席树）

- 用于**带修改的区间第 $k$ 小**
- 树状数组管理前缀和的"差分修改"，每个位置挂一棵动态开点权值线段树
- 查询时同时操作 $O(\log n)$ 棵树的根，在值域上二分下降
- 实现比线段树套平衡树简洁，但功能相对单一（主要解决第 $k$ 小）

### 3.4 CDQ 分治替代

- 许多树套树能解决的问题（尤其是三维偏序类问题），可以用 **CDQ 分治**替代
- CDQ 分治离线处理，按一个维度排序后分治，内部再用树状数组或归并排序
- 优势：常数小、空间少、无需复杂的嵌套结构
- 劣势：必须**离线**，强制在线时无法使用

### 3.5 整体二分

- 将多次"第 $k$ 小查询"一起处理
- 按答案值二分，用扫描线 + 树状数组统计每个查询的答案落在左半还是右半
- 适合离线批量处理带修改/不带修改的第 $k$ 小问题
- 复杂度 $O(n \log^2 n)$，常数小于树套树

**方案对比：**

| 方案 | 在线 | 功能丰富度 | 常数 | 编码难度 |
|------|------|-----------|------|---------|
| 线段树套 Treap | 是 | 高 | 大 | 高 |
| 线段树套线段树 | 是 | 中 | 中 | 高 |
| BIT 套主席树 | 是 | 低（第 $k$ 小） | 中 | 中 |
| CDQ 分治 | 否 | 中 | 小 | 中 |
| 整体二分 | 否 | 低（第 $k$ 小） | 小 | 低 |

### 题目 1：二逼平衡树（洛谷 P3380）

> 给定长度为 $n$ 的序列，支持 5 种操作：区间查排名、区间查第 $k$ 小、单点修改、区间求前驱、区间求后继。$n, m \le 5 \times 10^4$。

**分析**：这是树套树的**模板题**，完美对应「线段树套平衡树」的所有操作。外层线段树维护区间，每个节点挂一棵 FHQ Treap 维护值的集合。

```
数据结构：外层线段树 + 内层 FHQ Treap
支持操作：区间排名、区间第 k 小、单点修改、区间前驱、区间后继
关键性质：二分答案将"第 k 小"转化为"排名查询"
时间复杂度：O(m log^3 n)
```

**C++ 代码**：

```cpp
// 区间第 k 小：二分 + 区间排名
int query_kth(int ql, int qr, int k) {
    int lo = 0, hi = 1e8, ans = 0;
    while (lo <= hi) {
        int mid = (lo + hi) >> 1;
        if (seg_query_rank(1, 1, n, ql, qr, mid) + 1 <= k)
            ans = mid, lo = mid + 1;
        else hi = mid - 1;
    }
    return ans;
}
// 区间前驱：O(log n) 棵 Treap 上分别查前驱取 max
int query_pre(int o, int l, int r, int ql, int qr, int val) {
    if (ql <= l && r <= qr) return treap_pre(root[o], val);
    int mid = (l + r) >> 1, res = -INF;
    if (ql <= mid) res = max(res, query_pre(o*2,l,mid,ql,qr,val));
    if (qr > mid) res = max(res, query_pre(o*2+1,mid+1,r,ql,qr,val));
    return res;
}
```

**核心启示**：
- P3380 是树套树的标杆题目，掌握此题即掌握线段树套平衡树的全部核心操作
- 二分答案将"第 $k$ 小"转化为"排名查询"是关键技巧
- 内层使用 FHQ Treap 比 Splay 更易编写

### 题目 2：动态区间第 k 小（ZJU 2112 / BZOJ 1901）

> 给定长度为 $n$ 的序列，支持单点修改和查询区间第 $k$ 小。$n, m \le 10^4$。

**分析**：带修改的区间第 $k$ 小是树状数组套主席树的经典应用。树状数组管理差分修改，内层动态开点权值线段树维护值域信息。

```
数据结构：外层树状数组 + 内层动态开点权值线段树
支持操作：单点修改、区间第 k 小
关键性质：查询时同时操作 O(log n) 棵树的根，在值域上二分下降
时间复杂度：O(m log^2 n)
```

**C++ 代码**：

```cpp
int T1[MAXN], T2[MAXN]; // 前缀树状数组中用到的根
int t1[30], t2[30], cnt1, cnt2;

void bit_update(int pos, int val, int d) {
    for (int i = pos; i <= n; i += i & -i)
        modify(T1[i], 1, sz, val, d);
}

int bit_query(int l, int r, int k) {
    cnt1 = cnt2 = 0;
    for (int i = l - 1; i > 0; i -= i & -i) t1[++cnt1] = T1[i];
    for (int i = r; i > 0; i -= i & -i) t2[++cnt2] = T1[i];
    // 加上静态主席树的两个根
    int lroot = root[l - 1], rroot = root[r];
    int lo = 1, hi = sz;
    while (lo < hi) {
        int mid = (lo + hi) >> 1;
        int cnt = 0;
        for (int i = 1; i <= cnt2; i++) cnt += sum[ls[t2[i]]];
        for (int i = 1; i <= cnt1; i++) cnt -= sum[ls[t1[i]]];
        cnt += sum[ls[rroot]] - sum[ls[lroot]];
        if (cnt >= k) {
            for (int i = 1; i <= cnt2; i++) t2[i] = ls[t2[i]];
            for (int i = 1; i <= cnt1; i++) t1[i] = ls[t1[i]];
            rroot = ls[rroot]; lroot = ls[lroot];
            hi = mid;
        } else {
            k -= cnt;
            for (int i = 1; i <= cnt2; i++) t2[i] = rs[t2[i]];
            for (int i = 1; i <= cnt1; i++) t1[i] = rs[t1[i]];
            rroot = rs[rroot]; lroot = rs[lroot];
            lo = mid + 1;
        }
    }
    return lo;
}
```

**核心启示**：
- 树状数组套主席树是处理"带修改区间第 $k$ 小"的标准方案
- 查询时需同时维护 $O(\log n)$ 棵树的根指针，在值域线段树上同步二分下降
- 相比线段树套平衡树，代码更短且常数更小

### 题目 3：K 大数查询（洛谷 P3332）

> 给定 $n$ 个位置（初始为空），支持两种操作：1）在区间 $[l, r]$ 的每个位置插入一个数 $c$；2）查询区间 $[l, r]$ 中所有数的第 $k$ 大。$n, m \le 5 \times 10^4$。

**分析**：注意这里是**区间插入**而非单点修改，且查询第 $k$ 大。外层按**值域**建线段树，内层按**位置**建线段树（动态开点），支持区间加、区间求和。

```
数据结构：外层值域线段树 + 内层位置线段树（动态开点）
支持操作：区间插入、区间第 k 大
关键性质：外层建在值域上、内层建在位置上，角色互换
时间复杂度：O(m log^2 n)
```

**C++ 代码**：

```cpp
// 外层值域线段树上二分查第 k 大
long long query_kth(int o, int vl, int vr, int ql, int qr, long long k) {
    if (vl == vr) return vl;
    int vmid = (vl + vr) >> 1;
    long long rcnt = inner_query(rs[o], 1, n, ql, qr);
    if (rcnt >= k) return query_kth(rs[o], vmid+1, vr, ql, qr, k);
    else return query_kth(ls[o], vl, vmid, ql, qr, k - rcnt);
}
// 插入：在外层值域树上定位 c，对内层位置树做区间 +1
void insert(int o, int vl, int vr, int c, int ql, int qr) {
    inner_add(rt[o], 1, n, ql, qr, 1);
    if (vl == vr) return;
    int vmid = (vl + vr) >> 1;
    if (c <= vmid) insert(ls[o], vl, vmid, c, ql, qr);
    else insert(rs[o], vmid+1, vr, c, ql, qr);
}
```

**核心启示**：
- 当操作从「单点修改」变为「区间修改」时，内外层的角色需要灵活调换
- 本题外层建在值域上、内层建在位置上，是线段树套线段树的经典应用
- 内层必须动态开点，且需要支持懒标记（区间加 + 区间求和）

### 题目 4：排队（洛谷 P1975）

> 给定一个排列，每次交换两个位置的元素，询问每次交换后序列的逆序对数。$n \le 2 \times 10^4$，$m \le 2 \times 10^3$。

**分析**：交换位置 $i$ 和 $j$（$i < j$）时，需统计 $(i, j)$ 之间有多少元素值介于 $a[i]$ 和 $a[j]$ 之间来确定逆序对数的变化量，这就需要树套树的区间排名查询。

```
数据结构：外层线段树 + 内层 FHQ Treap
支持操作：区间排名查询、单点修改
关键性质：逆序对变化量可归结为 O(1) 次区间排名查询
时间复杂度：O(m log^2 n)
```

**C++ 代码**：

```cpp
void solve_swap(int i, int j) {
    if (i > j) swap(i, j);
    int ai = a[i], aj = a[j];
    long long delta = 0;
    if (i + 1 <= j - 1) {
        int cnt_i = seg_query_rank(1,1,n,i+1,j-1, ai);
        int cnt_j = seg_query_rank(1,1,n,i+1,j-1, aj);
        int len = j - i - 1;
        delta += (len - cnt_i) - cnt_i;
        delta += cnt_j - (len - cnt_j);
    }
    delta += (ai < aj) ? 1 : -1;
    ans += delta;
    seg_update(1, 1, n, i, ai, aj);
    seg_update(1, 1, n, j, aj, ai);
    swap(a[i], a[j]);
}
```

**核心启示**：
- 逆序对的动态维护，关键在于分析交换操作对逆序对数的影响可以归结为区间排名查询
- 树套树使得每次交换只需 $O(\log^2 n)$ 的查询和修改

### 题目 5：数颜色（洛谷 P1903）

> 给定长度为 $n$ 的颜色序列，支持两种操作：1）单点修改颜色；2）查询区间 $[l, r]$ 内有多少种不同的颜色。$n, m \le 1.33 \times 10^5$。

**分析**：将问题转化为三维偏序。对每个位置 $i$，记录 $\text{pre}[i]$ 为前一个与 $a[i]$ 相同颜色的位置。查询 $[l, r]$ 内不同颜色数等于 $[l, r]$ 中 $\text{pre}[i] < l$ 的个数。

```
数据结构：外层树状数组 + 内层动态开点权值线段树
支持操作：单点修改 pre 值、区间内满足 pre < l 的计数
关键性质："颜色数"转化为"pre < 左端点的计数"是经典套路
时间复杂度：O(m log^2 n)
```

**C++ 代码**：

```cpp
int bit_root[MAXN];

void bit_update(int pos, int pre_val, int delta) {
    for (int i = pos; i <= n; i += i & -i)
        seg_modify(bit_root[i], 0, n, pre_val, delta);
}

int bit_query(int pos, int lo, int hi) {
    int res = 0;
    for (int i = pos; i > 0; i -= i & -i)
        res += seg_query(bit_root[i], 0, n, lo, hi);
    return res;
}

int solve(int l, int r) {
    return bit_query(r, 0, l - 1) - bit_query(l - 1, 0, l - 1);
}
```

**核心启示**：
- 将"区间不同颜色数"转化为"前驱位置 < 查询左端点的个数"是处理此类问题的经典套路
- 修改时要注意维护 pre 链的一致性

### 题目 6：不勤劳的图书管理员（洛谷 P3759）

> 有 $n$ 本书排成一列，每本书有编号 $a_i$ 和价值 $v_i$。定义"不整洁度"为所有逆序对 $(i,j)$ 的 $v_i + v_j$ 之和。支持交换两本书的位置，每次交换后输出不整洁度。$n, m \le 5 \times 10^4$。

**分析**：不仅要维护逆序对的**数量**，还要维护逆序对的**权值和**。平衡树中每个节点需要额外维护子树权值和。

```
数据结构：外层线段树 + 内层 FHQ Treap（维护子树权值和）
支持操作：区间排名 + 对应权值和查询、单点修改
关键性质：平衡树的分裂/合并需正确 pushup 权值和信息
时间复杂度：O(m log^2 n)
```

**C++ 代码**：

```cpp
struct TreapNode {
    int ls, rs, val, key, sz;
    long long w, sum; // w: 本节点权值, sum: 子树权值和
};

pair<int, long long> treap_rank_sum(int o, int val) {
    if (!o) return {0, 0LL};
    if (val <= tr[o].val) return treap_rank_sum(tr[o].ls, val);
    auto [c, s] = treap_rank_sum(tr[o].rs, val);
    return {sz[tr[o].ls] + 1 + c, sum[tr[o].ls] + tr[o].w + s};
}
```

**核心启示**：
- 逆序对问题附加权值信息时，平衡树需要额外维护子树权值和
- 在 Treap 的分裂/合并过程中正确地 pushup 权值和，体现了树套树在维护复合信息时的灵活性

### 题目 7：三维偏序 CDQ 分治（洛谷 P3810）

> 给定 $n$ 个三元组 $(a_i, b_i, c_i)$，对每个 $i$ 求有多少 $j$ 满足 $a_j \le a_i, b_j \le b_i, c_j \le c_i$。$n \le 10^5$。

**分析**：三维偏序的经典解法是 CDQ 分治：按 $a$ 排序后分治，左右合并时按 $b$ 归并，用树状数组统计 $c$ 维的贡献。这是树套树的离线替代方案。

```
数据结构：CDQ 分治 + 树状数组
支持操作：三维偏序计数
关键性质：CDQ 按 a 分治、按 b 归并、树状数组统计 c
时间复杂度：O(n log^2 n)
```

**C++ 代码**：

```cpp
void cdq(int l, int r) {
    if (l == r) return;
    int mid = (l + r) >> 1;
    cdq(l, mid); cdq(mid + 1, r);
    int i = l, j = mid + 1, k = l;
    while (i <= mid && j <= r) {
        if (q[i].b <= q[j].b) {
            bit_add(q[i].c, q[i].cnt);
            tmp[k++] = q[i++];
        } else {
            q[j].ans += bit_query(q[j].c);
            tmp[k++] = q[j++];
        }
    }
    while (j <= r) { q[j].ans += bit_query(q[j].c); tmp[k++] = q[j++]; }
    // 清空树状数组并完成归并
    for (int t = l; t < i; t++) bit_add(q[t].c, -q[t].cnt);
    while (i <= mid) tmp[k++] = q[i++];
    for (int t = l; t <= r; t++) q[t] = tmp[t];
}
```

**核心启示**：
- CDQ 分治是三维偏序问题的经典离线解法，常数小、空间少
- 与树套树相比，CDQ 分治必须离线，但实现更简洁

### 题目 8：Dynamic Rankings 整体二分（BZOJ 1901 / 洛谷 P2617）

> 给定长度为 $n$ 的序列，支持单点修改和查询区间第 $k$ 小。$n, m \le 10^5$。

**分析**：整体二分将所有询问一起处理。按答案值二分，统计每个查询中值 $\le$ mid 的元素个数，决定查询归入左半还是右半。

```
数据结构：整体二分 + 树状数组
支持操作：单点修改、区间第 k 小（离线）
关键性质：将多次第 k 小查询批量处理，用树状数组辅助统计
时间复杂度：O((n+m) log^2 n)
```

**C++ 代码**：

```cpp
void solve(int ql, int qr, int vl, int vr) {
    if (ql > qr) return;
    if (vl == vr) {
        for (int i = ql; i <= qr; i++)
            if (op[i].type == 2) ans[op[i].id] = vl;
        return;
    }
    int vmid = (vl + vr) >> 1;
    int cl = 0, cr = 0;
    for (int i = ql; i <= qr; i++) {
        if (op[i].type == 1) {
            if (op[i].val <= vmid)
                bit_add(op[i].pos, op[i].delta), lq[++cl] = op[i];
            else rq[++cr] = op[i];
        } else {
            int cnt = bit_query(op[i].r) - bit_query(op[i].l - 1);
            if (cnt >= op[i].k) lq[++cl] = op[i];
            else { op[i].k -= cnt; rq[++cr] = op[i]; }
        }
    }
    // 清空树状数组
    for (int i = 1; i <= cl; i++)
        if (lq[i].type == 1) bit_add(lq[i].pos, -lq[i].delta);
    // 放回并递归
    for (int i = 1; i <= cl; i++) op[ql + i - 1] = lq[i];
    for (int i = 1; i <= cr; i++) op[ql + cl + i - 1] = rq[i];
    solve(ql, ql + cl - 1, vl, vmid);
    solve(ql + cl, qr, vmid + 1, vr);
}
```

**核心启示**：
- 整体二分将多次"第 $k$ 小查询"一起处理，是树套树在离线场景下的高效替代
- 复杂度 $O((n+m) \log^2 n)$，常数远小于树套树

### 题目 10：接水果 整体二分（洛谷 P3242）

> 给定一棵 $n$ 个节点的树，有 $P$ 条路径（"水果"），每条有权值。$Q$ 次询问，每次给一条路径，问包含它的所有"水果"路径中权值第 $k$ 小的。$n \le 4 \times 10^4$，$P, Q \le 2.5 \times 10^4$。

**分析**：将"路径包含"用 DFS 序转化为二维区间的包含关系，再用整体二分 + 扫描线 + 树状数组解决。

```
数据结构：整体二分 + 扫描线 + 树状数组
支持操作：矩形覆盖计数、第 k 小查询
关键性质：树上路径包含关系经 DFS 序转化为二维矩形覆盖
时间复杂度：O(n log^2 n)
```

**C++ 代码**：

```cpp
void solve(int ql, int qr, int vl, int vr,
           vector<Rect>& rects, vector<Query>& qrys) {
    if (ql > qr || vl == vr) {
        for (int i = ql; i <= qr; i++) ans[qrys[i].id] = vl;
        return;
    }
    int vmid = (vl + vr) >> 1;
    // 将权值 <= vmid 的矩形做扫描线加入树状数组
    // 对每个查询点，查询被覆盖次数
    vector<Query> left_q, right_q;
    for (auto &q : qrys) {
        int cnt = bit_query(q.x);
        if (cnt >= q.k) left_q.push_back(q);
        else { q.k -= cnt; right_q.push_back(q); }
    }
    // 撤销树状数组操作，递归两半
    solve(0, left_q.size()-1, vl, vmid, rects, left_q);
    solve(0, right_q.size()-1, vmid+1, vr, rects, right_q);
}
```

**核心启示**：
- 树上路径问题经 DFS 序转化后，往往变成二维几何问题
- 整体二分是树套树在离线场景下的高效替代方案

### 题目 16：Ynoi 2017 由乃的 OJ（洛谷 P4690 / 树链剖分套线段树）

> 给定一棵带权树，每个节点有一个值域 $[0, 2^K-1]$ 内的值和一个操作类型（AND/OR/XOR）。查询从 $u$ 到 $v$ 路径上依次对初始值 $x$ 执行各节点操作后的结果最大值。支持单点修改操作和值。$n, m \le 10^5$，$K \le 64$。

**分析**：路径上的位运算操作可以用线段树维护"输入每一位为 0/1 时输出是什么"的二进制变换信息。树链剖分将路径拆为区间，线段树维护区间变换的合并。求最优初始值时逐位贪心。

```
数据结构：树链剖分 + 线段树（维护位运算变换）
支持操作：路径变换合并、单点修改、逐位贪心求最优输入
关键性质：每个节点的操作等价于对每一位的 0→? 和 1→? 变换，可合并
时间复杂度：O(m K log^2 n)
```

**C++ 代码**：

```cpp
// 维护变换：输入全 0 时输出 z0，输入全 1 时输出 z1
struct Trans {
    unsigned long long z0, z1; // 全 0 输入的输出, 全 1 输入的输出
};

Trans merge(Trans a, Trans b) {
    Trans c;
    // 对每一位：若 a 输出 0，则看 b 的 z0 该位；若 a 输出 1，看 b 的 z1 该位
    c.z0 = (b.z0 & ~a.z0) | (b.z1 & a.z0);
    c.z1 = (b.z0 & ~a.z1) | (b.z1 & a.z1);
    return c;
}

unsigned long long solve(Trans t, unsigned long long lim) {
    unsigned long long res = 0;
    for (int i = K - 1; i >= 0; i--) {
        int b0 = (t.z0 >> i) & 1, b1 = (t.z1 >> i) & 1;
        if (b0 >= b1) {
            res |= ((unsigned long long)b0 << i);
        } else {
            if ((lim >> i) & 1)
                res |= ((unsigned long long)b1 << i), lim -= (1ULL << i);
            else
                res |= ((unsigned long long)b0 << i);
        }
    }
    return res;
}
```

**核心启示**：
- 位运算操作可以抽象为"变换"并用线段树合并
- 树链剖分 + 线段树处理树上路径的位运算合并，逐位贪心求最优值是极难但优美的技巧
- 路径有方向性，需要分别维护正向和反向合并

## 五、实用技巧与调试要点

1. **动态开点是必须的**：内层树（无论是平衡树还是线段树）都必须动态开点，否则空间会达到 $O(n^2)$ 级别。预估节点数一般为 $n \log^2 n$。

2. **FHQ Treap 优于 Splay**：作为内层平衡树，无旋 Treap（FHQ Treap）的分裂/合并操作天然支持"按值分裂"，实现比 Splay 简洁得多，且不依赖父指针。

3. **二分答案求第 $k$ 小**：当内层是平衡树时，无法直接在多棵树上联合求第 $k$ 小，必须二分答案值 $v$，再用"区间排名 $\le k$"来验证，额外多一个 $\log$。

4. **垃圾回收**：由于频繁的插入删除，内层树的节点会大量废弃。编写一个简单的内存池（回收删除节点的编号）可以有效控制空间。

5. **空间预估公式**：
   - 线段树套 Treap：约 $O(n \log n)$ 个 Treap 节点
   - 线段树套线段树：约 $O(n \log^2 n)$ 个节点（双层动态开点）
   - BIT 套主席树：约 $O(n \log^2 n)$ 个节点

6. **常数优化**：树套树常数大，注意以下优化：
   - 避免递归过深，可在内层使用迭代
   - 减少不必要的 `new` / `malloc`，使用数组模拟
   - 对于平衡树操作，尽量复用分裂后的节点

7. **调试建议**：
   - 先单独测试内层数据结构的正确性
   - 用小数据（$n \le 10$）手动验证每次操作后的树结构
   - 对拍：用暴力 $O(n^2)$ 做法生成答案，与树套树结果比较

## 六、通用代码框架

下面给出一个**线段树套 FHQ Treap** 的完整框架，支持五种操作（与 P3380 一致）：

```cpp
// ==================== FHQ Treap ====================
const int POOL = 1500005;
struct FHQ {
    int ls, rs, val, key, sz;
} t[POOL];
int pool_top, recycle_stack[POOL], rec_top;

int new_node(int val) {
    int o = rec_top ? recycle_stack[rec_top--] : ++pool_top;
    t[o] = {0, 0, val, rand(), 1};
    return o;
}
void del_node(int o) { recycle_stack[++rec_top] = o; }
void push_up(int o) { t[o].sz = t[t[o].ls].sz + t[t[o].rs].sz + 1; }

void split(int o, int val, int &x, int &y) {
    if (!o) { x = y = 0; return; }
    if (t[o].val <= val) { x = o; split(t[o].rs, val, t[o].rs, y); }
    else { y = o; split(t[o].ls, val, x, t[o].ls); }
    push_up(o);
}
int merge(int x, int y) {
    if (!x || !y) return x | y;
    if (t[x].key > t[y].key) { t[x].rs = merge(t[x].rs, y); push_up(x); return x; }
    else { t[y].ls = merge(x, t[y].ls); push_up(y); return y; }
}

void fhq_insert(int &root, int val) {
    int x, y; split(root, val, x, y);
    root = merge(merge(x, new_node(val)), y);
}
void fhq_delete(int &root, int val) {
    int x, y, z; split(root, val, x, z); split(x, val - 1, x, y);
    if (y) { int tmp = y; y = merge(t[y].ls, t[y].rs); del_node(tmp); }
    root = merge(merge(x, y), z);
}
int fhq_rank(int root, int val) {
    int x, y; split(root, val - 1, x, y);
    int res = t[x].sz;
    root = merge(x, y);
    return res;
}
int fhq_kth(int o, int k) {
    while (true) {
        int lsz = t[t[o].ls].sz;
        if (k <= lsz) o = t[o].ls;
        else if (k == lsz + 1) return t[o].val;
        else { k -= lsz + 1; o = t[o].rs; }
    }
}

// ==================== 线段树外层 ====================
const int MAXN = 50005;
int seg_root[MAXN * 4];
int a[MAXN], n;

void seg_build(int o, int l, int r) {
    for (int i = l; i <= r; i++) fhq_insert(seg_root[o], a[i]);
    if (l == r) return;
    int mid = (l + r) >> 1;
    seg_build(o*2, l, mid);
    seg_build(o*2+1, mid+1, r);
}

void seg_modify(int o, int l, int r, int pos, int oldv, int newv) {
    fhq_delete(seg_root[o], oldv);
    fhq_insert(seg_root[o], newv);
    if (l == r) return;
    int mid = (l + r) >> 1;
    if (pos <= mid) seg_modify(o*2, l, mid, pos, oldv, newv);
    else seg_modify(o*2+1, mid+1, r, pos, oldv, newv);
}

int seg_rank(int o, int l, int r, int ql, int qr, int val) {
    if (ql <= l && r <= qr) return fhq_rank(seg_root[o], val);
    int mid = (l + r) >> 1, res = 0;
    if (ql <= mid) res += seg_rank(o*2, l, mid, ql, qr, val);
    if (qr > mid)  res += seg_rank(o*2+1, mid+1, r, ql, qr, val);
    return res;
}
```

## 七、终极总结与归纳

### 7.1 树套树的本质

树套树的本质是**用一层树管理一个维度，用另一层树管理另一个维度**，从而在 $O(\log^2 n)$ 到 $O(\log^3 n)$ 的复杂度内处理本来需要 $O(n)$ 暴力的多维查询与修改问题。

### 7.2 何时选择树套树

- 需要**在线处理**多维查询（排名 + 区间、值域 + 位置等）
- 操作种类多（排名、前驱、后继、第 $k$ 小、修改需同时支持）
- 数据规模在 $n \le 5 \times 10^4$ 至 $10^5$ 量级（树套树常数大，更大规模需要考虑替代方案）
- 离线时优先考虑 CDQ 分治或整体二分

1. **入门**：先掌握线段树和平衡树（FHQ Treap）的基本操作
