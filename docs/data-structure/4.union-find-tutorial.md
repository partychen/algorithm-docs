## 一、什么是并查集

并查集（Disjoint Set Union，简称 DSU）是一种用于管理**不相交集合**的数据结构。它高效地支持两种核心操作：

- **查找（Find）**：确定某个元素属于哪个集合（返回集合的代表元素/根节点）
- **合并（Union）**：将两个不同的集合合并为一个

```
初始:  {1} {2} {3} {4} {5}     5 个独立集合

合并(1,2):  {1,2} {3} {4} {5}

合并(3,4):  {1,2} {3,4} {5}

合并(1,3):  {1,2,3,4} {5}

查询(2,4): → 同一集合（True）
查询(2,5): → 不同集合（False）
```

**核心优势：** 均摊时间复杂度为 $O(\alpha(n))$，其中 $\alpha$ 是反阿克曼函数，增长极其缓慢，实际中可视为常数。

**典型应用场景：**

| 场景 | 说明 |
|------|------|
| 连通性判断 | 判断图中两点是否连通 |
| 动态连通分量 | 动态加边，维护连通分量个数 |
| 最小生成树 | Kruskal 算法中判断是否形成环 |
| 等价关系维护 | 传递性等价关系的合并与查询 |
| 带权关系推导 | 维护元素间的相对权值关系 |

## 二、核心思想与前置知识

并查集用**森林**（若干棵树）来表示若干个不相交集合，每棵树代表一个集合，树的根节点即为集合的代表元素。

### 2.1 路径压缩（Path Compression）

在 `Find` 操作时，将查找路径上的所有节点直接连接到根节点，使树趋于扁平化。

```cpp
int fa[N];

void init(int n) { for (int i = 1; i <= n; i++) fa[i] = i; }

int find(int x) {
    if (fa[x] != x) fa[x] = find(fa[x]); // 路径压缩
    return fa[x];
}
```

```
路径压缩前:          路径压缩后:
    1                   1
    |                 / | \
    2               2   3   4
    |
    3
    |
    4
find(4) 后，3 和 4 都直接指向 1
```

### 2.2 按秩合并（Union by Rank）

合并时将**秩小的树**挂到秩大的树下面，避免树退化为链。

```cpp
int fa[N], rnk[N];

void init(int n) {
    for (int i = 1; i <= n; i++) { fa[i] = i; rnk[i] = 0; }
}

void unite(int x, int y) {
    int fx = find(x), fy = find(y);
    if (fx == fy) return;
    if (rnk[fx] < rnk[fy]) swap(fx, fy);
    fa[fy] = fx;
    if (rnk[fx] == rnk[fy]) rnk[fx]++;
}
```

> 路径压缩 + 按秩合并同时使用，保证均摊 $O(\alpha(n))$。实际竞赛中通常只用路径压缩即可。

### 2.3 带权并查集（Weighted DSU）

为每条边赋予**权值**，维护节点与根之间的相对关系（距离、倍数、偏移量等）。路径压缩时需同步累加权值。

```cpp
int fa[N];
long long d[N]; // d[x] = x 到 fa[x] 的权值

int find(int x) {
    if (fa[x] != x) {
        int root = find(fa[x]);
        d[x] += d[fa[x]]; // 路径压缩时累加
        fa[x] = root;
    }
    return fa[x];
}

// 合并：声明 d[x] - d[y] = w
void unite(int x, int y, long long w) {
    int fx = find(x), fy = find(y);
    if (fx == fy) return;
    fa[fx] = fy;
    d[fx] = d[y] - d[x] + w;
}

// 查询 x 与 y 的权值差（需在同一集合）
long long query(int x, int y) { return d[x] - d[y]; }
```

### 2.4 扩展域并查集（Extended Domain DSU）

将每个元素"拆分"为多个虚拟节点，表达**多种关系类型**。$n$ 个元素、$k$ 种关系 → $n \times k$ 个节点。

```cpp
// 以 3 种关系为例（食物链：同类、捕食、被捕食）
// 节点 x 的三个域: x, x+n, x+2n
int fa[3 * N];

void init(int n) { for (int i = 0; i < 3 * n; i++) fa[i] = i; }

// 声明 x 和 y 是同类
void same_kind(int x, int y, int n) {
    unite(x, y); unite(x + n, y + n); unite(x + 2*n, y + 2*n);
}

// 声明 x 吃 y
void x_eat_y(int x, int y, int n) {
    unite(x, y + n); unite(x + n, y + 2*n); unite(x + 2*n, y);
}
```

## 三、分类与变体详解

### 3.1 变体总览

| 变体 | 核心特点 | 典型应用 |
|------|----------|----------|
| **基本并查集** | 路径压缩 + 按秩/按大小合并 | 连通性、Kruskal |
| **带权并查集** | 边上附加权值，维护相对关系 | 银河英雄传说、距离查询 |
| **扩展域并查集** | 拆点为多个域，表达多种关系 | 食物链、关押罪犯 |
| **可持久化并查集** | 结合可持久化数组，支持版本回退 | 历史版本查询 |
| **可回退并查集** | 栈记录操作，支持撤销 | 线段树分治 |

### 3.2 选型指南

- 只需连通性 → **基本并查集**
- 需要量化关系（距离、权值） → **带权并查集**
- 多种互斥/循环关系 → **扩展域并查集**
- 需要历史版本 → **可持久化并查集**（不能用路径压缩，只用按秩合并，单次 $O(\log n)$）
- 需要撤销操作 → **可回退并查集**（栈记录合并信息，同样不用路径压缩）

### 题目 1：并查集模板（洛谷 P3367）

> $n$ 个元素，$m$ 次操作：合并两个元素所在的集合，或查询两个元素是否在同一集合。

**分析**：并查集裸模板题，只需实现 init/find/unite 三件套。

```
数据结构：并查集（路径压缩）
支持操作：合并（unite）、查询（find）
关键性质：路径压缩使均摊复杂度为 O(α(n))
时间复杂度：O(m·α(n))
```

**C++ 代码**：

```cpp
int fa[N];
void init(int n) { for (int i = 1; i <= n; i++) fa[i] = i; }
int find(int x) { return fa[x] == x ? x : fa[x] = find(fa[x]); }
void unite(int x, int y) { fa[find(x)] = find(y); }
bool same(int x, int y) { return find(x) == find(y); }
```

**核心启示**：
- **三行核心代码**：并查集的 find 只需一行递归 + 路径压缩
- **合并方向**：`fa[find(x)] = find(y)`，将 x 的根挂到 y 的根上

### 题目 2：修复公路（洛谷 P1111）

> $n$ 个村庄和 $m$ 条公路，每条有修建时间。按时间加边，求最早全部连通的时刻。

**分析**：将公路按修建时间排序后依次加入并查集，连通分量数降至 1 时即为答案。

```
数据结构：并查集 + 排序
支持操作：按时间顺序合并、计数连通分量
关键性质：排序后贪心加边，连通分量从 n 降至 1
时间复杂度：O(m log m + m·α(n))
```

**C++ 代码**：

```cpp
struct Edge { int u, v, t; } edges[M];

int solve(int n, int m) {
    sort(edges, edges + m, [](auto& a, auto& b) { return a.t < b.t; });
    init(n);
    int cnt = n;
    for (int i = 0; i < m; i++) {
        int fx = find(edges[i].u), fy = find(edges[i].v);
        if (fx != fy) { fa[fx] = fy; cnt--; }
        if (cnt == 1) return edges[i].t;
    }
    return -1;
}
```

**核心启示**：
- **排序 + 并查集**：按某维度排序后逐步加边，是并查集最常见的使用模式
- **连通分量计数**：初始为 $n$，每次成功合并减 1

### 题目 3：奶酪（洛谷 P3958）

> 高度为 $h$ 的奶酪中有 $n$ 个半径为 $r$ 的球形空洞。两空洞相切/相交则可穿行。判断能否从底面到顶面。

**分析**：将空洞看作节点，距离 $\leq 2r$ 则连边。引入底面/顶面两个虚拟节点，判断它们是否连通。

```
数据结构：并查集 + 虚拟节点
支持操作：合并相邻空洞、合并空洞与边界
关键性质：虚拟节点将边界条件转化为普通连通性问题
时间复杂度：O(n²·α(n))
```

**C++ 代码**：

```cpp
int S, T; // 底面、顶面虚拟节点

bool solve(int n, double r, double h, double x[], double y[], double z[]) {
    S = n + 1; T = n + 2;
    init(n + 2);
    for (int i = 1; i <= n; i++) {
        if (z[i] - r <= 0) unite(i, S);
        if (z[i] + r >= h) unite(i, T);
        for (int j = 1; j < i; j++) {
            double d2 = (x[i]-x[j])*(x[i]-x[j]) + (y[i]-y[j])*(y[i]-y[j]) + (z[i]-z[j])*(z[i]-z[j]);
            if (d2 <= 4.0 * r * r) unite(i, j);
        }
    }
    return find(S) == find(T);
}
```

**核心启示**：
- **虚拟节点**：将"与边界相连"转化为"与虚拟节点合并"，是并查集经典建模技巧
- **避免浮点误差**：比较距离平方而非距离，省去 `sqrt`

### 题目 4：程序自动分析（洛谷 P1955）

> 给出 $n$ 个约束：$x_i = x_j$ 或 $x_i \neq x_j$。判断是否能同时满足。变量编号 $\leq 10^9$。

**分析**：先处理所有等式约束（合并），再检查不等式约束（查询是否在同一集合）。编号大需要离散化。

```
数据结构：并查集 + 离散化
支持操作：合并等式对、检查不等式矛盾
关键性质：先等式后不等式——等式建立集合关系，不等式检验矛盾
时间复杂度：O(n log n)
```

**C++ 代码**：

```cpp
vector<int> vals;
int id(int x) { return lower_bound(vals.begin(), vals.end(), x) - vals.begin() + 1; }

bool solve(vector<tuple<int,int,int>>& cons) {
    // 收集所有值并离散化
    sort(vals.begin(), vals.end());
    vals.erase(unique(vals.begin(), vals.end()), vals.end());
    init(vals.size());
    // 先处理等式
    for (auto& [x, y, e] : cons)
        if (e == 1) unite(id(x), id(y));
    // 再检查不等式
    for (auto& [x, y, e] : cons)
        if (e == 0 && find(id(x)) == find(id(y))) return false;
    return true;
}
```

**核心启示**：
- **先等式后不等式**：约束满足问题的经典处理顺序
- **离散化**：编号范围大但数量少时，将大编号映射为连续小编号

### 题目 5：关押罪犯（洛谷 P1525）

> $n$ 个罪犯、$m$ 对仇恨关系（带权值）。分入两个监狱，使同一监狱内的最大冲突值最小。

**分析**：按冲突值从大到小排序。用扩展域并查集（敌人关系）：`unite(a, b+n)` 和 `unite(a+n, b)` 表示 a 和 b 是敌人。第一个无法分离的冲突即为答案。

```
数据结构：扩展域并查集（2 域：自身域 + 敌对域）
支持操作：声明敌对关系（跨域合并）、检查是否已在同一集合
关键性质：贪心从大到小处理 + 扩展域表达二元对立关系
时间复杂度：O(m log m + m·α(n))
```

**C++ 代码**：

```cpp
struct Edge { int u, v, w; } edges[M];

int solve(int n, int m) {
    sort(edges, edges + m, [](auto& a, auto& b) { return a.w > b.w; });
    init(2 * n);
    for (int i = 0; i < m; i++) {
        int a = edges[i].u, b = edges[i].v;
        if (find(a) == find(b)) return edges[i].w; // 已在同一组，无法分离
        unite(a, b + n); unite(a + n, b); // 声明 a、b 是敌人
    }
    return 0;
}
```

**核心启示**：
- **扩展域处理二元对立**：`x` 和 `x+n` 互为敌对域，跨域合并表达敌对关系
- **贪心 + 并查集**：从大到小处理冲突，第一个无法满足的就是答案

### 题目 6：MooTube（洛谷 P4185）

> $n$ 个视频构成一棵树（边带权）。查询：从视频 $v$ 出发，沿权值 $\geq k$ 的边可达多少视频？$q$ 次查询。

**分析**：离线。边按权值从大到小排序，查询也按阈值从大到小排序。双指针加边，查询时取 $v$ 所在集合大小减 1。

```
数据结构：并查集（按大小合并）+ 离线排序
支持操作：按阈值排序双指针加边、查询集合大小
关键性质：边和查询按同一维度排序，双指针避免重复加边
时间复杂度：O((n+q) log(n+q))
```

**C++ 代码**：

```cpp
int sz[N]; // 集合大小

void init(int n) { for (int i = 1; i <= n; i++) { fa[i] = i; sz[i] = 1; } }

void unite(int x, int y) {
    int fx = find(x), fy = find(y);
    if (fx == fy) return;
    fa[fx] = fy; sz[fy] += sz[fx];
}

int ans[Q];

void solve(int n, int q, Edge edges[], Query queries[]) {
    sort(edges, edges + n - 1, [](auto& a, auto& b) { return a.w > b.w; });
    sort(queries, queries + q, [](auto& a, auto& b) { return a.k > b.k; });
    init(n);
    int ptr = 0;
    for (int i = 0; i < q; i++) {
        while (ptr < n - 1 && edges[ptr].w >= queries[i].k)
            { unite(edges[ptr].u, edges[ptr].v); ptr++; }
        ans[queries[i].id] = sz[find(queries[i].v)] - 1;
    }
}
```

**核心启示**：
- **离线排序 + 双指针**：将边和查询按同一维度排序，用指针控制加边进度，是"阈值类查询"的经典范式
- **维护集合大小**：`sz[root]` 在合并时累加，根节点处始终有效

### 题目 7：Kruskal 最小生成树（洛谷 P3366）

> 给定 $n$ 个点 $m$ 条带权边的无向图，求最小生成树的边权之和。

**分析**：Kruskal 算法：边按权值从小到大排序，依次尝试加入。用并查集判断两端是否已连通——若不连通则加入 MST。

```
数据结构：并查集 + 边排序
支持操作：排序后贪心加边、判断是否成环
关键性质：每次选最小的不成环边，贪心正确性由切分定理保证
时间复杂度：O(m log m + m·α(n))
```

**C++ 代码**：

```cpp
struct Edge { int u, v, w; } edges[M];

long long kruskal(int n, int m) {
    sort(edges, edges + m, [](auto& a, auto& b) { return a.w < b.w; });
    init(n);
    long long sum = 0; int cnt = 0;
    for (int i = 0; i < m && cnt < n - 1; i++) {
        int fx = find(edges[i].u), fy = find(edges[i].v);
        if (fx != fy) {
            fa[fx] = fy;
            sum += edges[i].w;
            cnt++;
        }
    }
    return cnt == n - 1 ? sum : -1; // -1 表示不连通
}
```

**核心启示**：
- **Kruskal = 边排序 + 并查集**：并查集在 MST 算法中扮演"判环"的角色
- **选 n-1 条边即完成**：提前终止可以减少不必要的 find 操作

### 题目 8：团伙（洛谷 P1892）

> $n$ 个人，$m$ 条关系（朋友或敌人）。朋友的朋友是朋友，敌人的敌人是朋友。求最终有多少个"团伙"。

**分析**：扩展域并查集。`unite(a, b)` 表示朋友关系，`unite(a, b+n)` 和 `unite(a+n, b)` 表示敌对关系（敌人的敌人自动成为朋友）。

```
数据结构：扩展域并查集（2 域）
支持操作：朋友合并（同域）、敌人合并（跨域）
关键性质：敌人的敌人是朋友 —— 自动通过传递性在扩展域中实现
时间复杂度：O(m·α(n))
```

**C++ 代码**：

```cpp
int solve(int n, int m, tuple<char,int,int> ops[]) {
    init(2 * n);
    for (int i = 0; i < m; i++) {
        auto [type, a, b] = ops[i];
        if (type == 'F') { unite(a, b); }
        else { unite(a, b + n); unite(a + n, b); } // 敌人关系
    }
    int cnt = 0;
    for (int i = 1; i <= n; i++)
        if (find(i) == i) cnt++; // 统计不同的根
    return cnt;
}
```

**核心启示**：
- **传递性自动维护**：扩展域中"敌人的敌人是朋友"通过并查集的传递性自动实现
- **计数连通分量**：统计 `find(i) == i` 的个数（仅统计前 $n$ 个节点）

### 题目 9：食物链（洛谷 P2024）

> 三类动物 $A$ 吃 $B$、$B$ 吃 $C$、$C$ 吃 $A$。给出 $k$ 条描述，判断多少条是假话（与已知事实矛盾）。

**分析**：扩展域并查集经典题。每个动物拆成 3 个域：同类域 $x$、捕食域 $x+n$、天敌域 $x+2n$。

```
数据结构：扩展域并查集（3 域）
支持操作：同类合并（三域分别合并）、捕食合并（跨域合并）
关键性质：三个域编码三种循环关系，每次操作同时维护三组一致性
时间复杂度：O(k·α(n))
```

**C++ 代码**：

```cpp
int solve(int n, int k) {
    init(3 * n);
    int fake = 0;
    for (int i = 0; i < k; i++) {
        int op, x, y; // 读入操作
        if (x > n || y > n) { fake++; continue; }
        if (op == 1) { // x 和 y 同类
            if (find(x) == find(y + n) || find(x) == find(y + 2*n)) { fake++; continue; }
            unite(x, y); unite(x + n, y + n); unite(x + 2*n, y + 2*n);
        } else { // x 吃 y
            if (x == y) { fake++; continue; }
            if (find(x) == find(y) || find(x) == find(y + 2*n)) { fake++; continue; }
            unite(x, y + n); unite(x + n, y + 2*n); unite(x + 2*n, y);
        }
    }
    return fake;
}
```

**核心启示**：
- **三域编码循环关系**：$A \to B \to C \to A$ 的捕食链通过三域合并自动维护
- **矛盾检测**：合并前检查是否已有矛盾关系（如 x 吃 y 但 x 和 y 在同类域中）

### 题目 10：星球大战（洛谷 P1197）

> $n$ 个星球、$m$ 条通道。依次摧毁 $k$ 个星球，每次输出连通分量数。

**分析**：删除对并查集不友好。**时光倒流**：先计算全部摧毁后的状态，然后逆序恢复星球并加回通道。

```
数据结构：并查集 + 逆序处理
支持操作：逆序恢复节点、加回相关边
关键性质：正向删除难 → 逆向添加易
时间复杂度：O((n+m)·α(n))
```

**C++ 代码**：

```cpp
int ans[N];

void solve(int n, int m, int k, int destroy[], vector<int> adj[]) {
    bool dead[N] = {};
    for (int i = 0; i < k; i++) dead[destroy[i]] = true;
    // 初始化：只加入未摧毁节点之间的边
    init(n); int comp = 0;
    for (int i = 0; i < n; i++) if (!dead[i]) comp++;
    for (int u = 0; u < n; u++) if (!dead[u])
        for (int v : adj[u]) if (!dead[v] && u < v) {
            if (find(u) != find(v)) { fa[find(u)] = find(v); comp--; }
        }
    ans[k] = comp;
    // 逆序恢复
    for (int i = k - 1; i >= 0; i--) {
        int u = destroy[i];
        dead[u] = false; comp++;
        for (int v : adj[u]) if (!dead[v]) {
            if (find(u) != find(v)) { fa[find(u)] = find(v); comp--; }
        }
        ans[i] = comp;
    }
}
```

**核心启示**：
- **时光倒流**：并查集只擅长合并不擅长分裂，遇到删除操作就反向处理
- **逆序输出**：正向输出时使用预存的 `ans[]` 数组

### 题目 11：银河英雄传说（洛谷 P1196）

> $n$ 列战舰，每列初始一艘。操作：将第 $i$ 列接到第 $j$ 列末尾。查询：两艘战舰间隔几艘（不同列输出 -1）。

**分析**：带权并查集。`d[x]` 表示 x 到所在列队首的距离。合并时设 `d[fi] = sz[fj]`（i 列接到 j 列末尾）。

```
数据结构：带权并查集
支持操作：带权合并（权值 = 目标集合大小）、带权查询（权值差）
关键性质：d[x] = x 到根的距离，路径压缩时 d[x] += d[fa[x]]
时间复杂度：O(q·α(n))
```

**C++ 代码**：

```cpp
int fa[N], sz[N];
long long d[N]; // d[x] = x 到列首距离

int find(int x) {
    if (fa[x] != x) {
        int root = find(fa[x]);
        d[x] += d[fa[x]];
        fa[x] = root;
    }
    return fa[x];
}

void merge(int a, int b) { // a 列接到 b 列末尾
    int fa_ = find(a), fb = find(b);
    if (fa_ == fb) return;
    d[fa_] = sz[fb];
    sz[fb] += sz[fa_];
    fa[fa_] = fb;
}

int query(int a, int b) {
    if (find(a) != find(b)) return -1;
    return abs(d[a] - d[b]) - 1;
}
```

**核心启示**：
- **路径压缩传递权值**：`d[x] += d[fa[x]]` 是带权并查集的精髓
- **合并时设权**：`d[root_a] = sz[root_b]` 巧妙实现"接到末尾"

### 题目 12：How Many Answers Are Wrong（HDU 3038）

> 给出 $m$ 条信息"区间 $[l, r]$ 的和为 $s$"，按顺序判断有多少条与前面矛盾。

**分析**：带权并查集。将前缀和看作节点，`pre[r] - pre[l-1] = s` 即为权值关系 `d[r] - d[l-1] = s`。合并后若查询发现矛盾（已有权值差 $\neq s$），则为假。

```
数据结构：带权并查集
支持操作：带权合并（前缀和差）、矛盾检测
关键性质：区间和 = 前缀和之差 → 转化为带权并查集的权值差
时间复杂度：O(m·α(n))
```

**C++ 代码**：

```cpp
int solve(int n, int m) {
    init(n + 1); // 前缀和 pre[0..n]
    int fake = 0;
    for (int i = 0; i < m; i++) {
        int l, r, s; // 读入
        l--; // pre[l-1] 到 pre[r]
        int fl = find(l), fr = find(r);
        if (fl == fr) {
            if (d[r] - d[l] != s) fake++; // 矛盾
        } else {
            fa[fl] = fr;
            d[fl] = d[r] - d[l] - s; // 推导权值关系
            // 等价于: d[fl] 使得 d[r] - d[l] = s
        }
    }
    return fake;
}
```

**核心启示**：
- **前缀和 + 带权并查集**：区间和约束自然转化为前缀和节点间的权值关系
- **合并时推导权值**：`d[fl] = d[r] - d[l] - s` 需要仔细推导，画图辅助

### 题目 13：NOI2001 食物链 带权解法（洛谷 P2024）

> 同题目 9，但用带权并查集（权值模 3）解决。

**分析**：用 `d[x]` 表示 x 相对于根的"物种偏移量"（模 3）。0=同类，1=x 吃根，2=根吃 x。合并时通过模运算推导权值。

```
数据结构：带权并查集（权值模 3）
支持操作：带权合并（模 3 运算）、矛盾检测
关键性质：0=同类，1=捕食，2=被捕食，循环关系通过 mod 3 自然表达
时间复杂度：O(k·α(n))
```

**C++ 代码**：

```cpp
int d[N]; // d[x] = x 相对于根的关系（0 同类, 1 x吃根, 2 根吃x）

int find(int x) {
    if (fa[x] != x) {
        int root = find(fa[x]);
        d[x] = (d[x] + d[fa[x]]) % 3;
        fa[x] = root;
    }
    return fa[x];
}

int solve(int n, int k) {
    init(n);
    memset(d, 0, sizeof d);
    int fake = 0;
    for (int i = 0; i < k; i++) {
        int op, x, y;
        if (x > n || y > n) { fake++; continue; }
        int fx = find(x), fy = find(y);
        if (op == 1) { // x 和 y 同类
            if (fx == fy && (d[x] - d[y] + 3) % 3 != 0) { fake++; continue; }
            if (fx != fy) { fa[fx] = fy; d[fx] = (d[y] - d[x] + 3) % 3; }
        } else { // x 吃 y → d[x]-d[y] ≡ 1 (mod 3)
            if (x == y) { fake++; continue; }
            if (fx == fy && (d[x] - d[y] + 3) % 3 != 1) { fake++; continue; }
            if (fx != fy) { fa[fx] = fy; d[fx] = (d[y] - d[x] + 1 + 3) % 3; }
        }
    }
    return fake;
}
```

**核心启示**：
- **权值模 $k$ 表达循环关系**：食物链的三种关系恰好用 mod 3 编码
- **两种解法对比**：扩展域直观但空间 $3n$，带权并查集空间 $n$ 但需要推导权值公式

### 题目 14：Supermarket（POJ 1456）

> $n$ 个商品，每个有利润 $p_i$ 和截止日期 $d_i$。每天最多卖一个商品，求最大利润。

**分析**：贪心 + 并查集。按利润降序排序。用并查集维护"每个时间点最近的空闲时间槽"：`find(d)` 返回 $\leq d$ 的最大空闲时间点。

```
数据结构：并查集（维护空闲时间槽）
支持操作：查找最近空闲时间、标记已占用（合并到前一天）
关键性质：find(d) = d 时表示第 d 天空闲，占用后 fa[d] = d-1
时间复杂度：O(n log n + n·α(D))
```

**C++ 代码**：

```cpp
struct Item { int p, d; } items[N];

int solve(int n) {
    sort(items, items + n, [](auto& a, auto& b) { return a.p > b.p; });
    int maxD = 0;
    for (int i = 0; i < n; i++) maxD = max(maxD, items[i].d);
    for (int i = 0; i <= maxD; i++) fa[i] = i; // init
    int ans = 0;
    for (int i = 0; i < n; i++) {
        int slot = find(items[i].d);
        if (slot > 0) {
            ans += items[i].p;
            fa[slot] = slot - 1; // 标记已占用
        }
    }
    return ans;
}
```

**核心启示**：
- **并查集维护"最近空闲位置"**：`fa[d] = d-1` 使后续查询自动跳到前一个空闲天
- **非典型并查集应用**：这里并查集不是维护集合关系，而是维护"下一个可用资源"

### 题目 15：可持久化并查集（洛谷 P3402）

> 支持三种操作：合并两个元素所在集合、回退到第 $k$ 次操作后的状态、查询两元素是否在同一集合。

**分析**：用可持久化数组（主席树）存储 `fa[]` 和 `rnk[]`，每次操作产生新版本。**不能用路径压缩**（会破坏历史版本），只能按秩合并。

```
数据结构：可持久化并查集（主席树 + 按秩合并）
支持操作：版本化的合并与查询、版本回退
关键性质：不能路径压缩，只用按秩合并，单次 O(log²n)
时间复杂度：O(m log²n)
```

**C++ 代码**：

```cpp
// 可持久化数组（主席树实现）
int ls[M], rs[M], val[M], tot;

int build(int l, int r) {
    int p = ++tot;
    if (l == r) { val[p] = l; return p; } // fa[l] = l
    int mid = (l + r) >> 1;
    ls[p] = build(l, mid); rs[p] = build(mid + 1, r);
    return p;
}

int query(int p, int l, int r, int pos) {
    if (l == r) return val[p];
    int mid = (l + r) >> 1;
    return pos <= mid ? query(ls[p], l, mid, pos) : query(rs[p], mid + 1, r, pos);
}

int update(int p, int l, int r, int pos, int v) {
    int q = ++tot;
    ls[q] = ls[p]; rs[q] = rs[p]; val[q] = val[p];
    if (l == r) { val[q] = v; return q; }
    int mid = (l + r) >> 1;
    if (pos <= mid) ls[q] = update(ls[p], l, mid, pos, v);
    else rs[q] = update(rs[p], mid + 1, r, pos, v);
    return q;
}

// find 不路径压缩
int find(int root, int x, int n) {
    int f = query(root, 1, n, x);
    return f == x ? x : find(root, f, n);
}
```

**核心启示**：
- **可持久化 = 历史版本**：每次修改创建新版本，旧版本保留不变
- **禁用路径压缩**：路径压缩修改了历史节点，会破坏可持久化语义

### 题目 16：线段树分治 + 可回退并查集

> 给定 $n$ 个点，$m$ 条边，每条边有存在时间 $[l_i, r_i]$。$q$ 次查询，每次询问某时刻的连通分量数。离线处理。

**分析**：将每条边的存在时间挂到线段树上（区间挂载），DFS 遍历线段树，进入节点时加边（按秩合并），离开时回退（用栈记录合并信息）。

```
数据结构：线段树分治 + 可回退并查集（按秩合并 + 栈回退）
支持操作：加边（按秩合并）、撤销边（栈弹出恢复）
关键性质：不用路径压缩，用栈记录合并时修改的 fa 和 rnk，撤销时恢复
时间复杂度：O((n+m) log q · log n)
```

**C++ 代码**：

```cpp
int fa[N], rnk[N];
struct Snap { int x, y, rnk_x; }; // 回退快照
vector<Snap> stk;

int find(int x) { while (fa[x] != x) x = fa[x]; return x; } // 不压缩

bool unite(int x, int y) {
    int fx = find(x), fy = find(y);
    if (fx == fy) return false;
    if (rnk[fx] < rnk[fy]) swap(fx, fy);
    stk.push_back({fy, fx, rnk[fx]});
    fa[fy] = fx;
    if (rnk[fx] == rnk[fy]) rnk[fx]++;
    return true;
}

void rollback(int sz) { // 回退到栈大小 sz
    while ((int)stk.size() > sz) {
        auto [x, y, r] = stk.back(); stk.pop_back();
        fa[x] = x; rnk[y] = r;
    }
}

// 线段树分治：DFS 遍历，进入时加边，离开时回退
void solve(int node, int l, int r, vector<pair<int,int>> segs[]) {
    int snap = stk.size();
    for (auto [u, v] : segs[node]) unite(u, v);
    if (l == r) {
        // 回答第 l 个时刻的查询
        // comp = n - (成功合并数)
    } else {
        int mid = (l + r) >> 1;
        solve(node*2, l, mid, segs);
        solve(node*2+1, mid+1, r, segs);
    }
    rollback(snap);
}
```

**核心启示**：
- **线段树分治**：将"边的存在时间"挂到线段树上，DFS 遍历时自然处理时间区间
- **可回退并查集**：用栈记录合并操作，DFS 回溯时恢复，实现"加边-撤边"
- **必须按秩合并**：路径压缩不可撤销，因此只能用按秩合并（$O(\log n)$ 单次）

## 五、常见技巧与思路总结

### 5.1 虚拟节点

将边界条件转化为普通连通性：创建虚拟节点代表"底面"、"超级源"等概念。

### 5.2 时光倒流

并查集只擅长合并。遇到删除操作时，逆序处理将删除变为添加。

### 5.3 离线排序 + 双指针

阈值类查询：将边和查询按同一维度排序，双指针控制加边进度。

### 5.4 先等式后不等式

混合约束：先处理所有等式（合并），再检验不等式（查询矛盾）。

### 5.5 维护集合附加信息

合并时顺带维护 `sz[]`（集合大小）、权值和等。注意只在根节点处有效。

### 5.6 路径压缩 vs 按秩合并

- **大多数题**：只用路径压缩即可
- **需要可持久化或回退**：只用按秩合并，不能路径压缩
- **最优理论复杂度**：两者同时使用 → $O(\alpha(n))$

## 七、终极总结与归纳

### 7.1 并查集的本质

**并查集 = 用森林维护不相交集合，以近乎 $O(1)$ 的代价完成合并与查询。**

它的核心价值在于：用极简的代码（三行 find + 一行 unite）解决大量看似复杂的连通性和等价关系问题。

### 7.2 核心要点

1. **路径压缩**使树扁平化，`fa[x] = find(fa[x])` 一行代码将均摊复杂度降至 $O(\alpha(n))$
2. **扩展域**和**带权并查集**是处理多种关系的两种等价方法——前者直观空间大，后者紧凑需推导
3. **时光倒流**是处理删除操作的通用策略：将删除反转为添加
4. **离线排序 + 双指针**是阈值查询的标准范式
5. **可持久化和可回退**是进阶技能，核心约束是不能用路径压缩
6. **并查集维护"空闲资源"**（如 Supermarket）是非典型但重要的应用
