## 一、什么是珂朵莉树

珂朵莉树（Chtholly Tree），又称 **ODT（Old Driver Tree，老司机树）**，是一种基于 `std::set` 的暴力数据结构，专门用于处理**区间赋值操作**频繁出现的问题。

它的名字来源于动画《末日时在做什么？有没有空？可以来被拯救吗？》中的女主角——珂朵莉，而该数据结构最早由 CF896C 的出题人提出。

**核心特点：**

- 基于 `std::set` 维护若干个**不相交的连续区间**
- 每个区间存储一个统一的值，用三元组 $(l, r, val)$ 表示
- 依赖 `assign`（区间推平）操作来**大幅减少区间数量**
- 在**随机数据**下表现极佳，期望复杂度接近 $O(n \log n)$
- 实现简单、代码短小，非常适合对付数据随机的竞赛题

**适用场景：**

| 条件 | 说明 |
|------|------|
| 区间赋值操作频繁 | `assign` 操作会合并大量区间，保持集合规模小 |
| 数据随机或接近随机 | 随机 `assign` 使得区间数量期望为 $O(\log n)$ |
| 需要暴力遍历区间 | 区间数量少时，暴力枚举所有区间即可 |
| 操作种类繁多 | 任意统计查询都可暴力遍历完成 |

> **警告：** 珂朵莉树在精心构造的数据下可能退化为 $O(n^2)$，因此仅建议在数据随机或题目明确保证时使用。

## 二、核心思想与前置知识

### 2.1 前置知识

- `std::set` 的基本操作：`insert`、`erase`、`lower_bound`、迭代器
- 快速幂（部分题目需要）
- 区间问题的基本思维

### 2.2 区间表示：$(l, r, val)$ 三元组

珂朵莉树将整个序列分成若干个**值相同的连续段**，每一段用一个三元组 $(l, r, val)$ 表示：

- $l$：区间左端点
- $r$：区间右端点
- $val$：该区间内所有位置的统一值

所有区间存储在 `std::set` 中，按左端点 $l$ 排序，且任意两个区间不相交。

```
序列: [1,1,1,3,3,2,2,2,2,5]

珂朵莉树内部表示:
  (1, 3, 1)  (4, 5, 3)  (6, 9, 2)  (10, 10, 5)
```

节点定义：

```cpp
struct Node {
    long long l, r;
    mutable long long val;
    Node(long long l, long long r = -1, long long val = 0)
        : l(l), r(r), val(val) {}
    bool operator<(const Node &o) const { return l < o.l; }
};
set<Node> tree;
typedef set<Node>::iterator IT;
```

> `val` 声明为 `mutable`，是因为 `std::set` 中的元素默认不可修改，但我们需要在不改变排序键（$l$）的情况下修改 $val$。

### 2.3 split 操作（区间拆分）

`split(pos)` 在位置 $pos$ 处将区间切开，返回左端点为 $pos$ 的区间的迭代器。这是珂朵莉树的基础操作。

```cpp
IT split(long long pos) {
    IT it = tree.lower_bound(Node(pos));
    if (it != tree.end() && it->l == pos) return it;
    --it;
    long long l = it->l, r = it->r;
    long long val = it->val;
    tree.erase(it);
    tree.insert(Node(l, pos - 1, val));
    return tree.insert(Node(pos, r, val)).first;
}
```

**过程图示：**

```
split(5) 之前:
  ...(1, 8, val)...

split(5) 之后:
  ...(1, 4, val)  (5, 8, val)...
                  ^--- 返回这个迭代器
```

### 2.4 assign_val 操作（核心：区间推平）

`assign_val(l, r, val)` 是珂朵莉树的**灵魂操作**。它将区间 $[l, r]$ 内的所有值统一赋为 $val$。

执行步骤：

1. 调用 `split(r + 1)` 在 $r+1$ 处切开
2. 调用 `split(l)` 在 $l$ 处切开
3. 删除 $[l, r]$ 范围内的所有旧区间
4. 插入一个新区间 $(l, r, val)$

```cpp
void assign_val(long long l, long long r, long long val) {
    IT itr = split(r + 1), itl = split(l);
    tree.erase(itl, itr);
    tree.insert(Node(l, r, val));
}
```

> **关键：** 必须先 `split(r + 1)` 再 `split(l)`。因为 `split(l)` 可能使 `split(r + 1)` 返回的迭代器失效。

**为什么 assign 是核心？** 每次 `assign` 会把多个区间合并为一个，使得 `set` 中的区间总数持续减少。在随机数据下，`set` 中的区间数量始终维持在 $O(\log n)$ 级别，从而保证后续的暴力遍历操作高效运行。

### 2.5 常用暴力遍历操作

所有非 `assign` 操作都基于同一模式：先 `split` 切出 $[l,r]$，再暴力遍历其中所有区间。

```cpp
// 区间加
void add(long long l, long long r, long long val) {
    IT itr = split(r + 1), itl = split(l);
    for (IT it = itl; it != itr; ++it)
        it->val += val;
}

// 区间求和
long long query_sum(long long l, long long r) {
    IT itr = split(r + 1), itl = split(l);
    long long res = 0;
    for (IT it = itl; it != itr; ++it)
        res += (it->r - it->l + 1) * it->val;
    return res;
}

// 区间最大值
long long query_max(long long l, long long r) {
    IT itr = split(r + 1), itl = split(l);
    long long res = LLONG_MIN;
    for (IT it = itl; it != itr; ++it)
        res = max(res, it->val);
    return res;
}

// 区间第 k 小
long long kth(long long l, long long r, long long k) {
    IT itr = split(r + 1), itl = split(l);
    vector<pair<long long, long long>> vec;
    for (IT it = itl; it != itr; ++it)
        vec.push_back({it->val, it->r - it->l + 1});
    sort(vec.begin(), vec.end());
    for (auto &p : vec) {
        k -= p.second;
        if (k <= 0) return p.first;
    }
    return 0;
}

// 区间幂次和
long long sum_of_pow(long long l, long long r, long long ex, long long mod) {
    IT itr = split(r + 1), itl = split(l);
    long long res = 0;
    for (IT it = itl; it != itr; ++it)
        res = (res + (it->r - it->l + 1) % mod * qpow(it->val, ex, mod)) % mod;
    return res;
}
```

### 2.6 复杂度分析

| 数据特征 | 区间数量 | 单次操作 | 总复杂度 |
|---------|---------|---------|---------|
| 随机数据 | 期望 $O(\log n)$ | $O(\log^2 n)$ | $O(n \log^2 n)$ |
| 最坏情况（无 assign） | $O(n)$ | $O(n)$ | $O(n \cdot m)$ |

**核心结论：** 在随机数据下，每次 `assign` 操作期望删除 $O(\log n)$ 个区间，因此 `set` 中的活跃区间数始终维持在较低水平。所有暴力遍历操作的总代价摊到全局，总复杂度接近 $O(n \log n)$。

## 三、分类与变体详解

珂朵莉树本身结构简单，变体不多，主要在**操作类型**和**使用方式**上有所区分：

### 3.1 基本 ODT：区间赋值 + 暴力遍历

这是最经典、最常用的形式。

| 操作类型 | 实现方式 | 时间复杂度（随机数据） |
|---------|---------|---------------------|
| 区间赋值 | `assign_val` 合并区间 | 均摊 $O(\log n)$ |
| 区间加 | 暴力遍历 $[l, r]$ 内所有区间 | $O(\text{区间数})$ |
| 区间第 $k$ 小 | 取出区间排序 | $O(\text{区间数} \cdot \log)$ |
| 区间求和 | 暴力遍历累加 | $O(\text{区间数})$ |
| 区间幂次和 | 暴力遍历 + 快速幂 | $O(\text{区间数} \cdot \log)$ |

### 3.2 增强 ODT：assign 中维护全局信息

在 `assign_val` 中遍历被删除的区间时，可以顺便维护全局统计量（如总和、不同值个数等），避免额外遍历。

### 3.3 ODT + 其他数据结构

在 ODT 框架上叠加线段树、树状数组等结构，用 ODT 管理区间赋值的合并逻辑，用其他结构维护更复杂的信息。

### 3.4 变体汇总

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| **基本 ODT** | 纯 `set` + `split` + `assign` | 随机数据，标准区间赋值题 |
| **增强 ODT** | assign 中同步维护全局信息 | 需要动态维护统计量 |
| **ODT + 线段树** | ODT 管理区间，线段树辅助查询 | 非随机数据需要保底 |
| **离散化 ODT** | 值域大但操作有限时先离散化 | $n$ 很大但操作数有限 |
| **持久化 ODT** | 将 `set` 替换为可持久化平衡树 | 历史版本查询（极少使用） |

### 题目 1：ODT 模板 — assign_val 操作（模板题）

> 给定长度为 $n$ 的序列，支持区间赋值操作和单点查询。每次赋值将 $[l, r]$ 所有元素设为 $val$，查询第 $pos$ 个位置的值。

**分析**：

```
数据结构：set<Node> 存储不相交区间三元组 (l, r, val)
支持操作：split(pos) 拆分区间 / assign_val(l, r, val) 推平区间 / 单点查询
关键性质：assign 将多个区间合并为一个，保持 set 规模小
时间复杂度：随机数据下均摊 O(log n) 每操作
```

**C++ 代码**：

```cpp
struct Node {
    long long l, r;
    mutable long long val;
    Node(long long l, long long r = -1, long long val = 0)
        : l(l), r(r), val(val) {}
    bool operator<(const Node &o) const { return l < o.l; }
};
set<Node> tree;
typedef set<Node>::iterator IT;

IT split(long long pos) {
    IT it = tree.lower_bound(Node(pos));
    if (it != tree.end() && it->l == pos) return it;
    --it;
    long long l = it->l, r = it->r, val = it->val;
    tree.erase(it);
    tree.insert(Node(l, pos - 1, val));
    return tree.insert(Node(pos, r, val)).first;
}

void assign_val(long long l, long long r, long long val) {
    IT itr = split(r + 1), itl = split(l);
    tree.erase(itl, itr);
    tree.insert(Node(l, r, val));
}

long long point_query(long long pos) {
    IT it = --tree.upper_bound(Node(pos));
    return it->val;
}
```

**核心启示**：

- `split` 和 `assign_val` 是珂朵莉树的两个基石操作，所有后续操作都建立在它们之上
- 必须先 `split(r+1)` 再 `split(l)`，否则迭代器失效
- 单点查询只需 `upper_bound` 后退一步即可定位

### 题目 2：区间赋值 + 区间求和（基础练习）

> 给定长度为 $n$ 的序列，支持两种操作：将 $[l, r]$ 赋值为 $val$；查询 $[l, r]$ 的元素之和。数据随机生成。

**分析**：

```
数据结构：set<Node> 存储不相交区间三元组 (l, r, val)
支持操作：assign_val(l, r, val) / query_sum(l, r)
关键性质：assign 推平后区间数少，暴力遍历求和即可
时间复杂度：随机数据下总复杂度 O(n log n)
```

**C++ 代码**：

```cpp
long long query_sum(long long l, long long r) {
    IT itr = split(r + 1), itl = split(l);
    long long res = 0;
    for (IT it = itl; it != itr; ++it)
        res += (it->r - it->l + 1) * it->val;
    return res;
}
```

**核心启示**：

- 区间求和是最基本的暴力遍历操作，每个区间贡献为 `长度 * 值`
- 只要 `assign` 频繁，区间数量始终很小，暴力求和足够高效

### 题目 3：区间赋值 + 区间最值（基础练习）

> 给定长度为 $n$ 的序列，支持两种操作：将 $[l, r]$ 赋值为 $val$；查询 $[l, r]$ 的最大值。数据随机生成。

**分析**：

```
数据结构：set<Node> 存储不相交区间三元组 (l, r, val)
支持操作：assign_val(l, r, val) / query_max(l, r)
关键性质：assign 推平后遍历取 max 即可
时间复杂度：随机数据下总复杂度 O(n log n)
```

**C++ 代码**：

```cpp
long long query_max(long long l, long long r) {
    IT itr = split(r + 1), itl = split(l);
    long long res = LLONG_MIN;
    for (IT it = itl; it != itr; ++it)
        res = max(res, it->val);
    return res;
}

long long query_min(long long l, long long r) {
    IT itr = split(r + 1), itl = split(l);
    long long res = LLONG_MAX;
    for (IT it = itl; it != itr; ++it)
        res = min(res, it->val);
    return res;
}
```

**核心启示**：

- 区间最值查询和区间求和类似，都是暴力遍历所有区间
- 最大值和最小值可以同时查询，代码几乎相同

### 题目 4：区间赋值 + 单点查询（P5251 第二代图灵机）

> **题意：** 维护一个长度为 $n$ 的序列，支持区间赋值、区间加、查询区间最大值。数据随机生成。

**分析**：

```
数据结构：set<Node> 存储不相交区间三元组 (l, r, val)
支持操作：assign_val / add / query_max
关键性质：区间加也是暴力遍历，给每个区间的 val 加上增量
时间复杂度：随机数据下均摊 O(log n) 每操作
```

```cpp
void add(long long l, long long r, long long val) {

    for (IT it = itl; it != itr; ++it)
```

### 题目 5：CF896C Willem, Chtholly and Seniorious（Codeforces CF896C）

> **题意：** 维护长度为 $n$ 的序列，支持四种操作：区间加、区间赋值、查询区间第 $k$ 小、查询区间幂次和 $\sum a_i^{ex} \bmod p$。**数据随机生成。**

**分析**：

```
数据结构：set<Node> 存储不相交区间三元组 (l, r, val)
支持操作：add / assign_val / kth / sum_of_pow
关键性质：题目明确说明数据随机生成，这是使用 ODT 的信号
时间复杂度：随机数据下期望 O(n log^2 n)
```

这是珂朵莉树的**起源题**，也是 ODT 得名的来源。

**C++ 代码**：

```cpp
long long kth(long long l, long long r, long long k) {
    IT itr = split(r + 1), itl = split(l);
    vector<pair<long long, long long>> vec;
    for (IT it = itl; it != itr; ++it)
        vec.push_back({it->val, it->r - it->l + 1});
    sort(vec.begin(), vec.end());
    for (auto &p : vec) {
        k -= p.second;
        if (k <= 0) return p.first;
    }
    return 0;
}

long long qpow(long long a, long long b, long long mod) {
    long long res = 1; a %= mod;
    for (; b > 0; b >>= 1) {
        if (b & 1) res = res * a % mod;
        a = a * a % mod;
    }
    return res;
}

long long sum_of_pow(long long l, long long r, long long ex, long long mod) {
    IT itr = split(r + 1), itl = split(l);
    long long res = 0;
    for (IT it = itl; it != itr; ++it)
        res = (res + (it->r - it->l + 1) % mod * qpow(it->val, ex, mod)) % mod;
    return res;
}
```

**核心启示**：

- 当题目中出现"随机数据"字样且包含区间赋值操作时，珂朵莉树几乎是最优选择
- 四种截然不同的操作，全部用暴力遍历解决
- 第 $k$ 小：提取所有 `(val, len)` 排序后逐段累减 $k$

### 题目 6：区间赋值 + 区间 k 次方和（幂次和变体）

> **题意：** 给定长度为 $n$ 的序列和模数 $p$，支持两种操作：将 $[l, r]$ 赋值为 $val$；查询 $[l, r]$ 的 $k$ 次方和 $\sum_{i=l}^{r} a_i^k \bmod p$。

**分析**：

```
数据结构：set<Node> 存储不相交区间三元组 (l, r, val)
支持操作：assign_val / sum_of_pow(l, r, k, mod)
关键性质：相同值的区间，k 次方和 = len * val^k，快速幂计算
时间复杂度：随机数据下 O(区间数 * log k) 每查询
```

**C++ 代码**：

```cpp
long long sum_of_pow(long long l, long long r, long long k, long long mod) {
    IT itr = split(r + 1), itl = split(l);
    long long res = 0;
    for (IT it = itl; it != itr; ++it) {
        long long len = (it->r - it->l + 1) % mod;
        res = (res + len * qpow(it->val % mod, k, mod)) % mod;
    }
    return res;
}
```

**核心启示**：

- 珂朵莉树的区间内值相同特性，使得幂次和计算只需对每个区间做一次快速幂
- 注意取模的位置：`len` 和 `val` 都要先对 `mod` 取模再乘

### 题目 7：区间赋值 + 区间不同数个数（去重计数）

> **题意：** 给定长度为 $n$ 的序列，支持两种操作：将 $[l, r]$ 赋值为 $val$；查询 $[l, r]$ 中有多少个不同的值。数据随机。

**分析**：

```
数据结构：set<Node> 存储不相交区间三元组 (l, r, val)
支持操作：assign_val / count_distinct(l, r)
关键性质：遍历区间时用 set/哈希表记录出现过的值
时间复杂度：随机数据下 O(区间数) 每查询
```

**C++ 代码**：

```cpp
long long count_distinct(long long l, long long r) {
    IT itr = split(r + 1), itl = split(l);
    set<long long> vals;
    for (IT it = itl; it != itr; ++it)
        vals.insert(it->val);
    return vals.size();
}
```

**核心启示**：

- 珂朵莉树天然按值相同的连续段存储，统计不同值只需遍历区间去重
- 这种查询在线段树上实现极为困难，但在 ODT 上仅需几行代码

### 题目 8：随机数据优化 — CF915E Physical Education Lessons（Codeforces CF915E）

> **题意：** 数轴上初始有 $[1, n]$ 全为工作日。每次操作将某个区间 $[l, r]$ 设为休息日或工作日，每次操作后输出剩余工作日的天数。$n$ 可达 $10^9$。

**分析**：

```
数据结构：set<Node> 存储不相交区间三元组 (l, r, val)，val 为 0 或 1
支持操作：assign_val（赋 0 或 1）/ 全局求和
关键性质：n 虽大但不需要建出全部区间，ODT 只维护有限个活跃区间
时间复杂度：O(q log q)，q 为操作次数
```

**C++ 代码**：

```cpp
long long total; // 全局维护工作日总数

void assign_val(long long l, long long r, long long val) {
    IT itr = split(r + 1), itl = split(l);
    for (IT it = itl; it != itr; ++it)
        total -= (it->r - it->l + 1) * it->val;
    tree.erase(itl, itr);
    tree.insert(Node(l, r, val));
    total += (r - l + 1) * val;
}
```

**核心启示**：

- 珂朵莉树不要求将整个值域建出来，它只维护"存在的区间"，因此 $n$ 很大也不影响
- 在 `assign` 中同步维护全局统计量（先减旧贡献、再加新贡献），避免额外遍历
- 所有操作都是纯粹的区间赋值，是 ODT 的最佳应用场景

### 题目 9：ODT + 区间 GCD（区间赋值 + GCD 查询）

> **题意：** 给定长度为 $n$ 的序列，支持两种操作：将 $[l, r]$ 赋值为 $val$；查询 $[l, r]$ 所有元素的 GCD。数据随机。

**分析**：

```
数据结构：set<Node> 存储不相交区间三元组 (l, r, val)
支持操作：assign_val / query_gcd(l, r)
关键性质：同一区间内值相同，GCD 贡献就是 val 本身
时间复杂度：随机数据下 O(区间数 * log V) 每查询，V 为值域
```

**C++ 代码**：

```cpp
long long gcd(long long a, long long b) {
    return b ? gcd(b, a % b) : a;

long long query_gcd(long long l, long long r) {
    IT itr = split(r + 1), itl = split(l);
    long long res = 0;
    for (IT it = itl; it != itr; ++it)
        res = gcd(res, it->val);
    return res;
}
```

**核心启示**：

- GCD 具有结合律，可以逐区间累计计算
- ODT 的区间内值相同特性使得无需考虑一个区间内多个不同值的 GCD
- 这类具有"结合律"的查询（GCD、LCM、AND、OR、XOR）都可以用同样的暴力遍历模式

### 题目 10：ODT + 区间排名查询（区间赋值 + 排名）

> **题意：** 给定长度为 $n$ 的序列，支持三种操作：将 $[l, r]$ 赋值为 $val$；查询 $val$ 在 $[l, r]$ 中的排名（比它小的元素个数 +1）；查询 $[l, r]$ 中第 $k$ 小。数据随机。

**分析**：

```
数据结构：set<Node> 存储不相交区间三元组 (l, r, val)
支持操作：assign_val / rank_query / kth_query
关键性质：提取所有区间的 (val, len) 对后排序处理
时间复杂度：随机数据下 O(区间数 * log) 每查询
```

**C++ 代码**：

```cpp
long long rank_query(long long l, long long r, long long val) {
    IT itr = split(r + 1), itl = split(l);
    long long cnt = 0;
    for (IT it = itl; it != itr; ++it)
        if (it->val < val)
            cnt += it->r - it->l + 1;
    return cnt + 1;
}

long long kth(long long l, long long r, long long k) {
    IT itr = split(r + 1), itl = split(l);
    vector<pair<long long, long long>> vec;
    for (IT it = itl; it != itr; ++it)
        vec.push_back({it->val, it->r - it->l + 1});
    sort(vec.begin(), vec.end());
    for (auto &p : vec) {
        k -= p.second;
        if (k <= 0) return p.first;
    }
    return 0;
}
```

**核心启示**：

- 排名查询只需统计值小于 $val$ 的元素总个数，暴力遍历即可
- 第 $k$ 小需要先排序再累减，这和 CF896C 中的做法一致
- 排名和第 $k$ 小是互逆操作，在 ODT 上都很容易实现

### 题目 11：ODT 结合线段树（P5350 序列）

> **题意：** 维护一个长度为 $n$ 的序列，支持多种操作：区间赋值、区间加、区间求和、区间最大/最小值、区间复制到另一段等。

**分析**：

```
数据结构：set<Node> 存储不相交区间三元组 (l, r, val)
支持操作：assign_val / add / query_sum / query_max / query_min / copy_range
关键性质：操作种类多但核心是 assign 推平，区间复制可拆解为取出+赋值
时间复杂度：随机数据下总复杂度 O(n log n)
```

**C++ 代码**：

```cpp
// 区间复制: 将 [l1, r1] 的值复制到 [l2, r2] (长度相同)
void copy_range(long long l1, long long r1, long long l2) {
    IT itr = split(r1 + 1), itl = split(l1);
    vector<pair<long long, long long>> segs;
    for (IT it = itl; it != itr; ++it)
        segs.push_back({it->r - it->l + 1, it->val});
    long long cur = l2;
    for (auto &[len, val] : segs) {
        assign_val(cur, cur + len - 1, val);
        cur += len;
    }
}

long long query_sum(long long l, long long r) {
    IT itr = split(r + 1), itl = split(l);
    long long res = 0;
    for (IT it = itl; it != itr; ++it)
        res += (it->r - it->l + 1) * it->val;
    return res;
}
```

**核心启示**：

- 珂朵莉树的强大之处在于：无论查询操作有多复杂，只要有频繁的区间赋值，就可以把所有查询退化为"暴力遍历少量区间"
- 区间复制操作的实现：先取出源区间的所有段，再逐段对目标区间赋值
- 操作种类多时，ODT 的代码量远少于线段树

### 题目 12：动态区间赋值 + 历史查询（区间赋值 + 版本记录）

> **题意：** 给定长度为 $n$ 的序列，支持区间赋值操作。每次赋值后记录当前版本。给定查询：某个历史版本中位置 $pos$ 的值是多少。

**分析**：

```
数据结构：每个版本用一个 set<Node>，或离线处理用时间戳标记
支持操作：assign_val / 复制版本 / 单点历史查询
关键性质：可以通过离线+时间倒流的方式处理，或者维护操作序列二分查找
时间复杂度：离线 O(q log q)，在线需要可持久化
```

**C++ 代码**：

```cpp
// 离线方案: 记录每次 assign 的时间戳，对于查询二分找最后一次覆盖
struct Assign {
    long long l, r, val;
    int time;
};
vector<Assign> ops;

// 查询版本 t 时位置 pos 的值
long long history_query(int pos, int t) {
    long long res = initial[pos]; // 初始值
    int last_time = 0;
    for (auto &op : ops) {
        if (op.time <= t && op.l <= pos && pos <= op.r && op.time > last_time) {
            res = op.val;
            last_time = op.time;
        }
    }
    return res;
}

// 优化: 对每个位置维护 (时间, 值) 的有序表, 二分查找
// 或: 使用 ODT, 在 assign 时记录时间戳
struct NodeWithTime {
    long long l, r;
    mutable long long val;
    mutable int timestamp;
    NodeWithTime(long long l, long long r = -1, long long val = 0, int t = 0)
        : l(l), r(r), val(val), timestamp(t) {}
    bool operator<(const NodeWithTime &o) const { return l < o.l; }
};
```

**核心启示**：

- 历史查询可以通过在 ODT 节点中增加时间戳字段来实现
- 离线处理时，可以记录所有 assign 操作后按查询倒序处理
- 在线版本需要可持久化，但可持久化 ODT 实现复杂度高，通常考虑其他方案

### 题目 13：ODT 在图论中的应用（区间赋值 + 连通性）

> **题意：** 给定 $n$ 个节点排成一行，初始每个节点颜色不同。支持两种操作：将 $[l, r]$ 的节点颜色都赋为 $c$；查询 $[l, r]$ 中相邻且同色的节点对数。

**分析**：

```
数据结构：set<Node> 存储不相交区间三元组 (l, r, val=颜色)
支持操作：assign_val / query_pairs(l, r)
关键性质：相邻同色对数 = 区间总长度 - 区间段数
时间复杂度：随机数据下 O(区间数) 每操作
```

**C++ 代码**：

```cpp
long long query_pairs(long long l, long long r) {
    IT itr = split(r + 1), itl = split(l);
    long long total_len = r - l + 1;
    long long seg_count = 0;
    for (IT it = itl; it != itr; ++it)
        seg_count++;
    // 相邻同色对数 = 总长度 - 段数
    return total_len - seg_count;
}

// 更复杂的变体: 查询相邻不同色的边界数
long long query_boundaries(long long l, long long r) {
    IT itr = split(r + 1), itl = split(l);
    long long seg_count = 0;
    for (IT it = itl; it != itr; ++it)
        seg_count++;
    return seg_count - 1; // 段数 - 1 = 颜色边界数
}
```

**核心启示**：

- ODT 的区间段直接反映了连通分量的结构
- "相邻同色对数 = 总长 - 段数"是一个常用的计数技巧
- 颜色边界数等图论性质可以直接从 ODT 的段数推导

### 题目 14：区间赋值 + 复杂查询（区间赋值 + 区间多项式求和）

> **题意：** 给定长度为 $n$ 的序列和多项式 $f(x) = \sum_{j=0}^{d} c_j x^j$，支持两种操作：将 $[l, r]$ 赋值为 $val$；查询 $\sum_{i=l}^{r} f(a_i) \bmod p$。

**分析**：

```
数据结构：set<Node> 存储不相交区间三元组 (l, r, val)
支持操作：assign_val / poly_sum(l, r, poly, mod)
关键性质：同一区间内值相同，f(val) 只需计算一次，乘以区间长度
时间复杂度：随机数据下 O(区间数 * d) 每查询
```

**C++ 代码**：

```cpp
long long eval_poly(vector<long long> &c, long long x, long long mod) {
    long long res = 0, pw = 1;
    for (long long ci : c) {
        res = (res + ci % mod * (pw % mod)) % mod;
        pw = pw * (x % mod) % mod;
    }
    return res;
}

long long poly_sum(long long l, long long r, vector<long long> &c, long long mod) {
    IT itr = split(r + 1), itl = split(l);
    long long res = 0;
    for (IT it = itl; it != itr; ++it) {
        long long len = (it->r - it->l + 1) % mod;
        long long fval = eval_poly(c, it->val, mod);
        res = (res + len * fval) % mod;
    }
    return res;
}
```

**核心启示**：

- 任何形如 $\sum f(a_i)$ 的查询，只要 $f$ 可以对单个值计算，ODT 就能暴力处理
- 多项式求值的单次复杂度为 $O(d)$，乘以区间数仍然很小
- 这体现了 ODT 的核心优势：查询的灵活性不受数据结构限制

### 题目 15：ODT 优化 DP（区间赋值优化状态转移）

> **题意：** 给定长度为 $n$ 的序列，每个位置有代价 $c_i$。你可以执行若干次区间赋值操作（将 $[l, r]$ 赋为某个值），每次操作的代价为 $r - l + 1$。目标是使最终序列满足某种单调性条件。求最小总代价。

**分析**：

```
数据结构：set<Node> 存储当前序列的区间表示
支持操作：模拟赋值操作，在 DP 转移中利用 ODT 快速合并状态
关键性质：DP 状态中，连续相同值的段可以用 ODT 压缩表示
时间复杂度：取决于具体 DP 方程，ODT 优化减少了状态数
```

**C++ 代码**：

```cpp
// 示例: 最少操作次数使序列非递减
// dp[i] = 将前 i 个元素变成非递减序列的最小代价
// 利用 ODT 维护当前的值段，合并相邻的可合并段

struct State {
    long long val;
    long long cost;
};

// ODT 优化: 当区间赋值时，自动合并相邻同值段
void merge_adjacent() {
    for (IT it = tree.begin(); it != tree.end(); ) {
        IT nxt = next(it);
        if (nxt != tree.end() && it->val == nxt->val) {
            long long l = it->l, r = nxt->r, val = it->val;
            tree.erase(it);
            tree.erase(nxt);
            it = tree.insert(Node(l, r, val)).first;
        } else {
            ++it;
        }
    }
}

// 在 DP 转移中，利用 ODT 维护的段信息快速计算
// 连续相同值的段只需一次转移，而非逐元素转移
long long dp_with_odt(int n) {
    long long res = 0;
    // 遍历所有段，对每段做一次 DP 转移
    for (IT it = tree.begin(); it != tree.end(); ++it) {
        long long len = it->r - it->l + 1;
        // 段长为 len、值为 val 的转移代价
        res += calc_cost(it->val, len);
    }
    return res;
}
```

**核心启示**：

- ODT 可以作为 DP 状态的压缩表示：连续同值段只需记录一次
- 当 DP 转移涉及区间赋值操作时，ODT 自然地维护了状态的变化
- 合并相邻同值段是优化关键，减少了需要考虑的状态数

### 题目 16：非随机数据下的 ODT 替代方案与复杂度对拍（竞赛实战）

> **题意：** 给定长度为 $n$ 的序列，支持区间赋值和多种查询。数据**不保证随机**，但保证 assign 操作占比超过 50%。要求在 $O(n \sqrt{n})$ 时间内解决。

**分析**：

```
数据结构：ODT + 分块兜底 / 纯 ODT 加区间数监控
支持操作：assign_val + 各种查询 + 退化检测
关键性质：当 set 中区间数过多时，说明 assign 不够频繁，需要切换策略
时间复杂度：ODT 路线期望 O(n log n) / 分块兜底 O(n sqrt(n))
```

**C++ 代码**：

```cpp
const int THRESHOLD = 1000; // 区间数阈值

// 方案一：ODT + 退化检测
bool use_odt = true;

void process_operation(int op, long long l, long long r, long long val) {
    if (use_odt) {
        if ((int)tree.size() > THRESHOLD) {
            // 区间数过多，可能被卡
            // 选择1: 强制执行一次全局 assign
            // 选择2: 切换到分块
            use_odt = false;
            convert_to_block(); // 将 ODT 转为分块
        }
    }
    if (use_odt) {
        // 正常 ODT 操作
        if (op == 1) assign_val(l, r, val);
        else query_odt(l, r);
    } else {
        // 分块兜底
        if (op == 1) block_assign(l, r, val);
        else block_query(l, r);
    }
}

// 方案二：对拍框架
// 暴力解法 (用于验证 ODT 正确性)
long long brute[MAXN];

void brute_assign(long long l, long long r, long long val) {
    for (long long i = l; i <= r; i++) brute[i] = val;
}

long long brute_sum(long long l, long long r) {
    long long res = 0;
    for (long long i = l; i <= r; i++) res += brute[i];
    return res;
}

// 对拍主函数
void stress_test() {
    srand(42);
    int n = 1000, q = 10000;
    // 初始化
    for (int i = 1; i <= n; i++) {
        long long v = rand() % 100;
        tree.insert(Node(i, i, v));
        brute[i] = v;
    }
    for (int i = 0; i < q; i++) {
        int l = rand() % n + 1, r = rand() % n + 1;
        if (l > r) swap(l, r);
        if (rand() % 2 == 0) {
            long long v = rand() % 100;
            assign_val(l, r, v);
            brute_assign(l, r, v);
        } else {
            long long odt_ans = query_sum(l, r);
            long long brute_ans = brute_sum(l, r);
            assert(odt_ans == brute_ans);
        }
    }
}
```

**核心启示**：

- 非随机数据下，ODT 可能被精心构造的数据卡到 $O(n^2)$
- 实战中可通过监控 `tree.size()` 来检测退化，超过阈值时切换备用策略
- 对拍是验证 ODT 正确性的最佳手段：写一个暴力解法，随机生成数据对比
- `assign` 占比是使用 ODT 的关键指标：占比超过 30% 通常安全，低于此需谨慎
- 竞赛中若不确定数据是否随机，可以先写 ODT 提交试探，TLE 再换方案

## 五、解题技巧与注意事项

### 技巧 1：split 的调用顺序

**必须先 `split(r + 1)` 再 `split(l)`。** 如果先 `split(l)`，那么 `split(r + 1)` 可能会将 `l` 所在的区间拆分，导致之前返回的迭代器失效。

```cpp
// 正确写法
IT itr = split(r + 1), itl = split(l);

// 错误写法（可能导致迭代器失效）
IT itl = split(l), itr = split(r + 1);  // 危险!
```

### 技巧 2：识别 ODT 适用题目

以下信号暗示可以使用珂朵莉树：

- 题目明确说明**数据随机生成**
- 操作中包含**区间赋值**
- 操作种类繁多，用传统数据结构难以同时维护
- 题目来源为 Codeforces 且标签含 `*2600` 以上（出题人可能设计了随机数据）

### 技巧 3：初始化

将序列中每个位置初始化为独立区间：

```cpp
for (int i = 1; i <= n; i++)
    tree.insert(Node(i, i, a[i]));
```

或者如果初始值相同：

```cpp
tree.insert(Node(1, n, initial_val));
```

### 技巧 4：在 assign 中维护全局信息

可以在 `assign_val` 中，遍历被删除的区间时顺便维护全局信息（如总和、最大值等），避免额外遍历。参见题目 8 的做法。

### 技巧 5：防止 TLE 的实践经验

- `assign` 操作的比例越高，珂朵莉树越快
- 如果 `assign` 操作占比低于 $30\%$，需要慎重考虑是否使用 ODT
- 极端情况下可以监控 `tree.size()`，超过阈值时切换策略

## 六、解题框架

遇到珂朵莉树类型的题目时，可以按照以下框架思考和编码：

```
Step 1: 判断是否适用
  ├── 是否有区间赋值操作？ → 没有则不适用
  ├── 数据是否随机？       → 不随机则可能被卡
  └── 是否需要复杂的区间查询？ → 是则 ODT 暴力遍历很方便

Step 2: 建立基本结构
  ├── 定义 Node 结构体 (l, r, mutable val)
  ├── 定义 typedef set<Node>::iterator IT
  ├── 实现 split(pos)
  └── 实现 assign_val(l, r, val)

Step 3: 实现各种查询操作
  ├── 区间求和: 遍历累加 (r-l+1)*val
  ├── 区间最值: 遍历取 max/min
  ├── 区间第k小: 取出排序
  ├── 区间幂次和: 遍历 + 快速幂
  ├── 区间不同数: 遍历 + set 去重
  ├── 区间 GCD: 遍历累计 gcd
  └── 其他自定义查询: 都用暴力遍历实现

Step 4: 调试验证
  ├── 写暴力解法对拍
  ├── 监控 tree.size() 观察区间数变化
  └── 边界检查: n=1、l=r、全局 assign 等
```

$$
\text{有区间赋值?} \xrightarrow{\text{是}} \text{数据随机?} \xrightarrow{\text{是}} \text{使用 ODT}
$$

$$
\text{数据随机?} \xrightarrow{\text{否}} \text{选择线段树 + 懒标记}
$$

## 七、总结

### 7.1 珂朵莉树的本质

珂朵莉树并不是一种"高级数据结构"，它的本质是**利用区间赋值操作来减少区间数量，从而使暴力遍历变得高效**。核心在于 `assign` 操作不断将多个区间合并为一个，维持 `set` 中的区间数量处于极低水平。如果没有频繁的区间赋值，它就退化为纯暴力。

### 7.2 核心要点

1. **`split` 和 `assign_val` 是两个基石操作**：`split` 拆分区间，`assign_val` 合并区间。所有查询都基于 `split` 切出范围后暴力遍历
2. **先 `split(r+1)` 再 `split(l)`**：这是最容易犯的错误，顺序反了会导致迭代器失效和运行时错误
3. **`mutable` 修饰 `val`**：因为 `set` 中的元素不可修改，但区间加等操作需要就地修改值
4. **随机数据是使用前提**：在精心构造的数据下 ODT 会退化到 $O(n^2)$，只有数据随机时才能保证 $O(n \log n)$
5. **查询灵活性无限**：任何能对单个值计算的函数 $f$，都可以暴力遍历区间计算 $\sum f(a_i)$
6. **assign 占比决定性能**：assign 操作占比越高，区间越少，性能越好。占比低于 30% 时需要警惕

```
    ↓

    ↓
Phase 3 (进阶): CF896C 全四操作 + CF915E 大值域
    ↓
Phase 4 (提高): 结合 GCD / 排名 / 复制 / 历史查询
    ↓

```
