## 一、什么是可并堆

**可并堆（Mergeable Heap）** 是一种支持高效合并操作的堆数据结构。

普通的二叉堆（如 `std::priority_queue`）支持插入和删除最值，时间复杂度均为 $O(\log n)$，但**合并两个堆**的操作需要 $O(n)$ 的时间。在很多问题中，我们需要频繁地将两个堆合并为一个，这时普通堆就力不从心了。

可并堆在保持堆的基本功能（插入、查询最值、删除最值）的同时，额外支持 **$O(\log n)$ 甚至更优的合并操作**。

| 操作 | 普通二叉堆 | 左偏树 | 配对堆 |
|------|-----------|--------|--------|
| 插入 | $O(\log n)$ | $O(\log n)$ | $O(1)$ |
| 查询最值 | $O(1)$ | $O(1)$ | $O(1)$ |
| 删除最值 | $O(\log n)$ | $O(\log n)$ | $O(\log n)$ 均摊 |
| **合并** | $O(n)$ | $O(\log n)$ | $O(1)$ |

**适用场景：** 需要维护多个优先队列并频繁执行合并的问题，例如并查集 + 堆、贪心中的分组合并等。

## 二、核心思想与前置知识

### 2.1 左偏树（Leftist Tree）— 性质与合并

左偏树是一棵满足**堆性质**的二叉树，同时维护一个额外属性 **dist（距离）**：

- **dist 定义：** 节点 $x$ 的 $\text{dist}(x)$ 为 $x$ 到其子树中最近的空节点的距离。空节点的 dist 为 $-1$，叶节点的 dist 为 $0$。
- **左偏性质：** 对任意节点，左子节点的 dist $\geq$ 右子节点的 dist。即树总是向左"倾斜"。
- **关键推论：** 右链（从根一直走右子节点到空节点的路径）长度为 $O(\log n)$，这保证了合并操作的时间复杂度。

**合并过程（递归）：**

1. 比较两棵树的根节点，选较小（小根堆）的作为新根。
2. 将新根的右子树与另一棵树递归合并。
3. 合并完成后，如果左子树的 dist < 右子树的 dist，交换左右子树。
4. 更新当前节点的 dist = 右子树的 dist + 1。

**插入** = 将单个节点视为一棵树，与原树合并。

**删除最值** = 合并根节点的左右子树。

### 2.2 配对堆（Pairing Heap）简介

配对堆是一种多叉树结构的可并堆，实现更简单，均摊性能优秀：

- **结构：** 每个节点可以有任意多个子节点，满足堆性质。
- **合并：** 比较两个根，将较大的根作为较小根的子节点，$O(1)$。
- **插入：** 新建节点与原堆合并，$O(1)$。
- **删除最值：** 删除根后，将所有子节点**两两配对合并**，再依次合并，均摊 $O(\log n)$。

配对堆虽然理论分析复杂，但实际表现非常优秀，常数小，竞赛中常用。

### 2.3 斜堆（Skew Heap）

斜堆是左偏树的简化版本：合并时**无条件交换**左右子树，不维护 dist。均摊 $O(\log n)$，代码更短，但不支持可持久化。

### 2.4 C++ 左偏树模板代码

```cpp
struct Node {
    int val, ls, rs, dist, fa;
} tr[MAXN];

int find(int x) {
    while (tr[x].fa) x = tr[x].fa;
    return x;
}

int merge(int x, int y) {
    if (!x || !y) return x | y;
    if (tr[x].val > tr[y].val || (tr[x].val == tr[y].val && x > y))
        swap(x, y);
    tr[x].rs = merge(tr[x].rs, y);
    tr[tr[x].rs].fa = x;
    if (tr[tr[x].ls].dist < tr[tr[x].rs].dist)
        swap(tr[x].ls, tr[x].rs);
    tr[x].dist = tr[tr[x].rs].dist + 1;
    return x;
}

void push(int id, int v, int &root) {
    tr[id] = {v, 0, 0, 0, 0};
    root = merge(root, id);
}

int pop(int root) {
    int l = tr[root].ls, r = tr[root].rs;
    tr[l].fa = tr[r].fa = 0;
    tr[root] = {0, 0, 0, 0, 0};
    return merge(l, r);
}
```

## 三、分类与变体详解

### 3.1 左偏树（Leftist Tree）

- 通过维护 dist 保证右链长度 $O(\log n)$。
- 合并、插入、删除均为 $O(\log n)$。
- 实现直观，竞赛中最常用。

### 3.2 斜堆（Skew Heap）

- 左偏树的简化版本：合并时**无条件交换**左右子树，不维护 dist。
- 均摊 $O(\log n)$，代码更短。
- 缺点：不支持可持久化。

### 3.3 配对堆（Pairing Heap）

- 合并 $O(1)$，删除最值均摊 $O(\log n)$。
- 用"左儿子右兄弟"表示法存储多叉树。
- 实际常数小，常用于需要大量合并操作的场景。

### 3.4 可删堆（Deletable Heap）

- 基于两个可并堆实现：一个存所有元素，一个存"待删除"元素。
- 查询最值时，若两个堆堆顶相同，则同时弹出。
- 支持懒删除任意元素，非常实用的技巧。

### 3.5 可持久化左偏树

- 由于 merge 只修改右链上的节点，可以通过复制节点实现持久化。
- 每次 merge 产生 $O(\log n)$ 个新节点，总空间 $O(n \log n)$。
- 注意斜堆不支持可持久化（因为每次合并都交换子树，修改量不可控）。

### 3.6 变体对比

| 变体 | 合并 | 删除最值 | 可持久化 | 实现难度 |
|------|------|---------|---------|---------|
| 左偏树 | $O(\log n)$ | $O(\log n)$ | 支持 | 中等 |
| 斜堆 | $O(\log n)$ 均摊 | $O(\log n)$ 均摊 | 不支持 | 简单 |
| 配对堆 | $O(1)$ | $O(\log n)$ 均摊 | 不支持 | 简单 |
| 可删堆 | $O(\log n)$ | $O(\log n)$ | 视底层 | 中等 |

### 题目 1：左偏树模板（洛谷 P3377）

> **题意：** 给定 $n$ 个数，支持两种操作：1）合并两个数所在的集合；2）查询并删除某个集合中的最小值。

**分析**：这是左偏树的裸模板题。每个数作为一个节点，用左偏树的 fa 指针维护连通性。合并操作 = 找到两个根后 merge，删除最小值 = 弹出堆顶后合并左右子树。

```
数据结构：左偏树（小根堆）+ 路径压缩找根
支持操作：merge / find / pop
关键性质：merge 沿右链递归，左偏性质保证 O(log n)
时间复杂度：O(m log n)
```

**C++ 代码**：

```cpp
struct Node {
    int val, ls, rs, dist, fa;
} tr[MAXN];
bool dead[MAXN];

int find(int x) {
    while (tr[x].fa) x = tr[x].fa;
    return x;
}

int merge(int x, int y) {
    if (!x || !y) return x | y;
    if (tr[x].val > tr[y].val || (tr[x].val == tr[y].val && x > y))
        swap(x, y);
    tr[x].rs = merge(tr[x].rs, y);
    tr[tr[x].rs].fa = x;
    if (tr[tr[x].ls].dist < tr[tr[x].rs].dist)
        swap(tr[x].ls, tr[x].rs);
    tr[x].dist = tr[tr[x].rs].dist + 1;
    return x;
}

// 操作 1：合并 x, y 所在堆
void op_merge(int x, int y) {
    if (dead[x] || dead[y]) return;
    int fx = find(x), fy = find(y);
    if (fx != fy) merge(fx, fy);
}

// 操作 2：查询并删除 x 所在堆的最小值
int op_pop(int x) {
    if (dead[x]) return -1;
    int fx = find(x);
    int ans = tr[fx].val;
    dead[fx] = true;
    int l = tr[fx].ls, r = tr[fx].rs;
    tr[l].fa = tr[r].fa = 0;
    tr[fx].ls = tr[fx].rs = tr[fx].dist = 0;
    merge(l, r);
    return ans;
}
```

**核心启示**：
- 左偏树的 merge 操作是一切的基础，插入和删除都可以转化为合并
- 用 `while (tr[x].fa) x = tr[x].fa` 代替并查集即可找根

### 题目 2：罗马游戏（洛谷 P2713）

> **题意：** $n$ 个士兵各有战斗力，支持两种操作：1）合并两个士兵所在的军团；2）查询某军团中战斗力最小的士兵并将其杀死。

**分析**：与模板题几乎相同，但需要处理"已被杀死的士兵"的查询。对已死亡的士兵直接输出 $0$，否则找到所在堆的根输出最小值后弹出。

```
数据结构：左偏树（小根堆）
支持操作：merge / find / pop
关键性质：dead 数组标记已杀死的士兵，避免重复操作
时间复杂度：O(m log n)
```

**C++ 代码**：

```cpp
bool dead[MAXN];

void solve() {
    char op;
    int x, y;
    while (m--) {
        scanf(" %c", &op);
        if (op == 'M') {
            scanf("%d%d", &x, &y);
            if (dead[x] || dead[y]) continue;
            int fx = find(x), fy = find(y);
            if (fx != fy) merge(fx, fy);
        } else {
            scanf("%d", &x);
            if (dead[x]) { puts("0"); continue; }
            int fx = find(x);
            printf("%d\n", tr[fx].val);
            dead[fx] = true;
            int l = tr[fx].ls, r = tr[fx].rs;
            tr[l].fa = tr[r].fa = 0;
            merge(l, r);
        }
    }
}
```

**核心启示**：
- 左偏树天然适合"并查集 + 优先队列"模型
- 加一个 dead 标记即可处理删除后的无效查询

### 题目 3：数列合并（洛谷 P1631）

> **题意：** 给定两个长度为 $n$ 的有序序列 $a$ 和 $b$，从中各取一个数相加，求前 $n$ 小的和。

**分析**：经典的多路归并问题。将 $a[1]+b[1], a[1]+b[2], \ldots$ 视为 $n$ 条有序链，用小根堆做 $n$ 路归并。初始将 $a[i]+b[1]$ 入堆，每次弹出最小后推入该链的下一个元素。

```
数据结构：小根堆（可用 priority_queue 或左偏树）
支持操作：多路归并——弹出最小，推入后继
关键性质：有序序列保证每路单调递增
时间复杂度：O(n log n)
```

**C++ 代码**：

```cpp
struct Item {
    long long val;
    int i, j;
    bool operator>(const Item& o) const { return val > o.val; }
};

void solve(int n, int a[], int b[]) {
    priority_queue<Item, vector<Item>, greater<Item>> pq;
    for (int i = 1; i <= n; i++)
        pq.push({a[i] + b[1], i, 1});
    for (int t = 0; t < n; t++) {
        auto [val, i, j] = pq.top(); pq.pop();
        printf("%lld ", val);
        if (j < n) pq.push({a[i] + b[j + 1], i, j + 1});
    }
}
```

**核心启示**：
- 多路归并是堆的经典应用模式——弹出最小后推入后继
- 虽然此题用 STL 堆更简洁，但它展示了可并堆的核心思维：多个有序源的合并

### 题目 4：Huffman 树（洛谷 P1090 合并果子）

> **题意：** $n$ 堆果子，每次选两堆合并，代价为两者之和。求把所有果子合成一堆的最小总代价。

**分析**：Huffman 编码思想。每次贪心选最小的两堆合并。用小根堆维护，每次弹出两个最小值、合并后放回。这就是 Huffman 树的构建过程。

```
数据结构：小根堆
支持操作：取两个最小值合并后放回
关键性质：贪心——小的堆被合并次数多，代价总和最小
时间复杂度：O(n log n)
```

**C++ 代码**：

```cpp
void solve(int n, int w[]) {
    priority_queue<long long, vector<long long>, greater<long long>> pq;
    for (int i = 1; i <= n; i++) pq.push(w[i]);
    long long ans = 0;
    while (pq.size() > 1) {
        long long a = pq.top(); pq.pop();
        long long b = pq.top(); pq.pop();
        ans += a + b;
        pq.push(a + b);
    }
    printf("%lld\n", ans);
}
```

**核心启示**：
- Huffman 贪心：每次取最小两个合并，总代价最优
- 可并堆视角：Huffman 树的构建过程本质上是不断合并两个最小堆

### 题目 5：Monkey King（洛谷 P1456）

> **题意：** $n$ 只猴子各有战斗力。每次选两只不同群的猴子打架，各自群中最强的猴子战斗力减半，然后两群合并。输出合并后最强猴子的战斗力。

**分析**：需要一种支持"修改最大值"和"合并"的堆。使用大根左偏树，修改堆顶 = 弹出堆顶 → 修改值 → 重新插入 → 合并两个堆。每次操作 $O(\log n)$。

```
数据结构：大根左偏树
支持操作：pop → 修改值 → push → merge
关键性质：修改堆顶可分解为删除+修改+重新插入
时间复杂度：每次操作 O(log n)
```

**C++ 代码**：

```cpp
int solve(int x, int y) {
    int a = find(x), b = find(y);
    if (a == b) return -1;
    // 弹出两个堆的最大值并减半
    tr[a].val /= 2;
    int na = merge(tr[a].ls, tr[a].rs);
    tr[a].ls = tr[a].rs = tr[a].dist = 0;
    na = merge(na, a);
    tr[b].val /= 2;
    int nb = merge(tr[b].ls, tr[b].rs);
    tr[b].ls = tr[b].rs = tr[b].dist = 0;
    nb = merge(nb, b);
    int root = merge(na, nb);
    return tr[root].val;
}
```

**核心启示**：
- "修改堆顶"操作可以分解为删除 + 修改 + 重新插入
- 左偏树使得这一系列操作仅需 $O(\log n)$

### 题目 6：黑匣子（洛谷 P1801）

> **题意：** 维护一个集合，支持两种操作：1）插入一个数；2）查询当前集合中第 $k$ 小的数（$k$ 每次加一）。

**分析**：用对顶堆思想，维护一个大根堆（前 $k$ 小）和一个小根堆（其余）。插入时比较新元素与大根堆堆顶；查询第 $k$ 小时调整两个堆的大小。可以用左偏树实现大根/小根堆，也可以直接用 STL。

```
数据结构：对顶堆（大根堆 + 小根堆）
支持操作：插入 → 调整平衡 → 取大根堆堆顶
关键性质：大根堆始终维护前 k 小的元素
时间复杂度：每次操作 O(log n)
```

**C++ 代码**：

```cpp
void solve(int n, int m, int a[], int q[]) {
    priority_queue<int> maxH; // 大根堆：前k小
    priority_queue<int, vector<int>, greater<int>> minH; // 小根堆：其余
    int k = 0, j = 1;
    for (int i = 1; i <= m; i++) {
        while (j <= q[i]) {
            maxH.push(a[j]);
            if ((int)maxH.size() > k)
                { minH.push(maxH.top()); maxH.pop(); }
            j++;
        }
        k++;
        while ((int)maxH.size() < k && !minH.empty())
            { maxH.push(minH.top()); minH.pop(); }
        printf("%d\n", maxH.top());
    }
}
```

**核心启示**：
- 对顶堆是动态第 k 小的标准方法
- 左偏树可以替代 STL 堆，但本题 STL 更简洁

### 题目 7：荷马史诗（洛谷 P2168）

> **题意：** 给定 $n$ 个单词的频率，用 $k$ 叉 Huffman 编码，使加权编码总长最小。长度相同时最长编码尽量短。

**分析**：$k$ 叉 Huffman 树。每次取 $k$ 个最小值合并。当 $(n-1) \bmod (k-1) \neq 0$ 时，需补虚拟节点使合法。用小根堆维护，堆中存 (权值, 深度)，按权值优先、深度次优先排序。

```
数据结构：小根堆（优先队列）
支持操作：每次取 k 个最小合并后放回
关键性质：k 叉 Huffman + 补虚节点使 (n-1)%(k-1)==0
时间复杂度：O(n log n)
```

**C++ 代码**：

```cpp
typedef pair<long long, int> pli;

void solve(int n, int k, long long w[]) {
    priority_queue<pli, vector<pli>, greater<pli>> pq;
    // 补虚拟节点
    while ((n - 1) % (k - 1) != 0) { pq.push({0, 0}); n++; }
    for (int i = 1; i <= n; i++) pq.push({w[i], 0});
    long long ans = 0;
    while (pq.size() > 1) {
        long long sum = 0;
        int maxd = 0;
        for (int i = 0; i < k; i++) {
            auto [v, d] = pq.top(); pq.pop();
            sum += v;
            maxd = max(maxd, d);
        }
        ans += sum;
        pq.push({sum, maxd + 1});
    }
    printf("%lld\n%d\n", ans, pq.top().second);
}
```

**核心启示**：
- $k$ 叉 Huffman 是 2 叉 Huffman 的自然推广
- 补虚节点是关键技巧：确保每层恰好合并 $k$ 个

### 题目 8：序列合并（洛谷 P1382）

### 题目 9：派遣（洛谷 P1552）

> **题意：** 一棵有根树，每个节点有费用 $c_i$ 和领导力 $l_i$。选一个节点 $u$ 作为管理者，在 $u$ 的子树中选若干节点，费用之和 $\leq M$，答案为选出节点数 $\times l_u$。求最大答案。

**分析**：对于每个节点 $u$，我们需要在其子树中贪心地选费用最小的节点。用大根堆维护当前选中的节点，合并子树的堆后，若费用超出 $M$ 则弹出费用最大的。DFS 过程中自底向上合并，左偏树完美支持此操作。

```
数据结构：大根左偏树
支持操作：自底向上合并子节点的堆 → 弹出超预算节点
关键性质：贪心——每个子树维护费用最小的节点集合
时间复杂度：O(n log n)
```

**C++ 代码**：

```cpp
int rt[MAXN], sz[MAXN];
long long sum[MAXN];
long long ans;

void dfs(int u) {
    rt[u] = u;
    sz[u] = 1;
    sum[u] = c[u];
    for (int v : son[u]) {
        dfs(v);
        rt[u] = merge(rt[u], rt[v]);
        sz[u] += sz[v];
        sum[u] += sum[v];
    }
    while (sum[u] > M) {
        sum[u] -= tr[rt[u]].val;
        rt[u] = pop(rt[u]);
        sz[u]--;
    }
    ans = max(ans, (long long)sz[u] * l[u]);
}
```

**核心启示**：
- "子树合并 + 贪心弹出"是左偏树在树形结构上的经典应用模式
- DFS 自底向上时合并子节点的堆非常自然

### 题目 10：中位数（洛谷 P1168）

> **题意：** 动态插入 $n$ 个数，每插入奇数个后输出当前中位数。

**分析**：经典的对顶堆问题。用大根堆维护较小一半，小根堆维护较大一半，保持大根堆的大小等于小根堆或多一个。大根堆堆顶即为中位数。可以用左偏树实现两个堆。

```
数据结构：对顶堆（大根左偏树 + 小根左偏树）
支持操作：插入 → 调整平衡 → 取大根堆堆顶
关键性质：大根堆维护前 ⌈n/2⌉ 小的元素
时间复杂度：每次插入 O(log n)
```

**C++ 代码**：

```cpp
int rt1 = 0, sz1 = 0; // 大根堆
int rt2 = 0, sz2 = 0; // 小根堆

void insert(int val) {
    if (sz1 == 0 || val <= tr[rt1].val) {
        // 插入大根堆
        int id = newNode(val);
        rt1 = merge_max(rt1, id);
        sz1++;
    } else {
        int id = newNode(val);
        rt2 = merge_min(rt2, id);
        sz2++;
    }
    // 调整平衡
    while (sz1 > sz2 + 1) {
        int top = rt1;
        rt1 = pop_max(rt1); sz1--;
        rt2 = merge_min(rt2, top); sz2++;
    }
    while (sz2 > sz1) {
        int top = rt2;
        rt2 = pop_min(rt2); sz2--;
        rt1 = merge_max(rt1, top); sz1++;
    }
}

int getMedian() { return tr[rt1].val; }
```

**核心启示**：
- 对顶堆是动态中位数的标准解法
- 用左偏树实现对顶堆可以额外获得合并能力

### 题目 11：棘手的操作（洛谷 P3273）

> **题意：** 给定 $n$ 个数，支持：合并两个集合、给单点加值、给集合整体加值、给全局加值、查询单点/集合最大值/全局最大值。

**分析**：使用大根左偏树维护每个集合，额外用一个全局堆（或 multiset）维护所有集合的堆顶。全局加值用一个偏移量，集合加值用集合内的标记。单点修改需要先删除再插入。

```
数据结构：大根左偏树 + multiset（存所有堆顶）
支持操作：merge / 单点修改 / 集合加值 / 全局加值 / 全局最大查询
关键性质：三层标记分离——全局偏移 + 集合标记 + 节点值
时间复杂度：O(m log n)
```

**C++ 代码**：

```cpp
multiset<int> global_max;
int global_add = 0;
int tag[MAXN];

void add_heap(int rt, int v) {
    tag[rt] += v;
    global_max.erase(global_max.find(tr[rt].val + tag[rt] - v));
    global_max.insert(tr[rt].val + tag[rt]);
}

void add_all(int v) { global_add += v; }

int query_all() { return *global_max.rbegin() + global_add; }
```

**核心启示**：
- 多层标记的管理是难点
- "全局偏移量 + 集合标记 + 节点值"三层分离可以高效处理多粒度修改

### 题目 12：斜堆排序

> **题意：** 使用斜堆实现堆排序。给定 $n$ 个数，用斜堆构建后逐一弹出最小值。

**分析**：斜堆是左偏树的简化版本，合并时无条件交换左右子树，不维护 dist。代码极短，适合快速实现。建堆时将 $n$ 个单节点依次合并，然后反复弹出最小值。

```
数据结构：斜堆（Skew Heap）
支持操作：merge / pop
关键性质：合并时无条件交换左右子树，均摊 O(log n)
时间复杂度：O(n log n)
```

**C++ 代码**：

```cpp
struct Node {
    int val, ls, rs;
} tr[MAXN];

int merge(int x, int y) {
    if (!x || !y) return x | y;
    if (tr[x].val > tr[y].val) swap(x, y);
    tr[x].rs = merge(tr[x].rs, y);
    swap(tr[x].ls, tr[x].rs); // 无条件交换
    return x;
}

int pop(int root) {
    return merge(tr[root].ls, tr[root].rs);
}

void heapSort(int n, int a[]) {
    int root = 0;
    for (int i = 1; i <= n; i++) {
        tr[i] = {a[i], 0, 0};
        root = merge(root, i);
    }
    for (int i = 1; i <= n; i++) {
        printf("%d ", tr[root].val);
        root = pop(root);
    }
}
```

**核心启示**：
- 斜堆比左偏树更简洁：不维护 dist，无条件交换
- 均摊 $O(\log n)$ 的合并足以应对大多数场景

### 题目 13：城池攻占（洛谷 P3261）

> **题意：** 一棵有根树代表城池体系，$m$ 个骑士各从某个叶节点出发向根进攻。每个城池有防御值，骑士战斗力 $\geq$ 防御值才能攻占并继续前进。攻占后骑士战斗力按城池规则变化（加法或乘法）。求每个骑士倒在哪个城池，以及每个城池击败了多少骑士。

**分析**：自底向上 DFS，每个节点维护一个小根堆存储当前存活的骑士。到达节点 $u$ 时，合并所有子节点的堆，然后弹出战斗力 < 防御值的骑士。战斗力的加法/乘法修改需要用**懒标记**下推到整棵堆上。

```
数据结构：带懒标记的左偏树（小根堆）
支持操作：merge with pushdown / 整体 add+mul 标记
关键性质：merge 前 pushdown，支持堆上整体修改
时间复杂度：O((n + m) log m)
```

**C++ 代码**：

```cpp
struct Node {
    long long val, add, mul;
    int ls, rs, dist;
} tr[MAXM];

void pushdown(int x) {
    if (tr[x].mul != 1 || tr[x].add != 0) {
        for (int c : {tr[x].ls, tr[x].rs}) if (c) {
            tr[c].val = tr[c].val * tr[x].mul + tr[x].add;
            tr[c].mul *= tr[x].mul;
            tr[c].add = tr[c].add * tr[x].mul + tr[x].add;
        }
        tr[x].mul = 1; tr[x].add = 0;
    }
}

int merge(int x, int y) {
    if (!x || !y) return x | y;
    pushdown(x); pushdown(y);
    if (tr[x].val > tr[y].val) swap(x, y);
    tr[x].rs = merge(tr[x].rs, y);
    if (tr[tr[x].ls].dist < tr[tr[x].rs].dist)
        swap(tr[x].ls, tr[x].rs);
    tr[x].dist = tr[tr[x].rs].dist + 1;
    return x;
}
```

**核心启示**：
- 左偏树可以支持懒标记（lazy tag），类似线段树的下推机制
- 在 merge 前 pushdown，即可实现堆上整体修改——这是进阶的核心技巧

### 题目 14：Sequence（洛谷 P4331）

> **题意：** 给定序列 $a_1, a_2, \ldots, a_n$，构造非严格递增序列 $b_1 \leq b_2 \leq \cdots \leq b_n$，使得 $\sum |a_i - b_i|$ 最小。

**分析**：经典的"保序回归"问题。最优的 $b_i$ 满足分段为常数，每段取对应 $a$ 值的中位数。使用"左偏树维护大根堆"来动态维护中位数：从左到右扫描，每个位置建一个堆，若当前堆的中位数 > 下一个值，则合并并弹出直到满足单调性。

```
数据结构：大根左偏树
支持操作：合并堆 → 弹出使中位数下降
关键性质：保序回归最优解为分段中位数，用堆维护
时间复杂度：O(n log n)
```

**C++ 代码**：

```cpp
int stk[MAXN], top = 0;
int len[MAXN], rt[MAXN];

for (int i = 1; i <= n; i++) {
    rt[++top] = i; len[top] = 1;
    tr[i].val = a[i];
    while (top > 1 && tr[rt[top-1]].val > tr[rt[top]].val) {
        top--;
        rt[top] = merge(rt[top], rt[top+1]);
        len[top] += len[top+1];
        while (heap_size(rt[top]) * 2 > len[top] + 1)
            rt[top] = pop(rt[top]);
    }
}
// 还原 b 数组：每段 b 值 = 对应堆顶（中位数）
```

**核心启示**：
- 左偏树可以用来高效维护动态中位数
- "保序回归"问题是中位数 + 可并堆的经典组合

### 题目 15：可持久化左偏树（洛谷 P3835 可持久化平衡树变体）

> **题意：** 给定多个版本的可并堆操作，支持在某个历史版本基础上执行插入、删除最值、查询最值、合并操作，要求保留所有历史版本。

**分析**：可持久化左偏树的核心是在 merge 过程中复制路径上的节点而非修改原节点。每次 merge 只涉及右链上 $O(\log n)$ 个节点，因此每次操作新建 $O(\log n)$ 个节点。注意斜堆不支持可持久化。

```
数据结构：可持久化左偏树
支持操作：persistent merge / insert / pop / query
关键性质：merge 时复制节点，空间 O(n log n)
时间复杂度：每次操作 O(log n)，空间 O(n log n)
```

**C++ 代码**：

```cpp
struct Node {
    int val, ls, rs, dist;
} tr[MAXN * 40];
int tot;

int newNode(int v) {
    int p = ++tot;
    tr[p] = {v, 0, 0, 0};
    return p;
}

int copyNode(int x) {
    int p = ++tot;
    tr[p] = tr[x];
    return p;
}

int merge(int x, int y) {
    if (!x || !y) return x ? copyNode(x) : (y ? copyNode(y) : 0);
    if (tr[x].val > tr[y].val) swap(x, y);
    int p = copyNode(x);
    tr[p].rs = merge(tr[p].rs, y);
    if (tr[tr[p].ls].dist < tr[tr[p].rs].dist)
        swap(tr[p].ls, tr[p].rs);
    tr[p].dist = tr[tr[p].rs].dist + 1;
    return p;
}

int pop(int root) {
    return merge(tr[root].ls, tr[root].rs);
}
```

**核心启示**：
- 可持久化的核心是"复制而非修改"——merge 走右链时复制每个经过的节点
- 空间开销为 $O(n \log n)$，需合理估算节点池大小

### 🟣 竞赛级 Level 5：综合与高难度

### 题目 16：配对堆实现 Dijkstra（配对堆高级应用）

> **题意：** 使用配对堆替代 STL 优先队列实现 Dijkstra 最短路算法，支持 decrease-key 操作，实现理论更优的时间复杂度。

**分析**：配对堆支持 $O(1)$ 的 merge 和 insert，以及均摊 $O(\log n)$ 的 delete-min。更重要的是，配对堆支持 decrease-key 操作（将某个节点的键值减小），均摊 $O(\log n)$。这使得 Dijkstra 的总复杂度可以优化。用"左儿子右兄弟"表示法实现配对堆。

```
数据结构：配对堆（Pairing Heap）
支持操作：insert / merge / delete-min / decrease-key
关键性质：左儿子右兄弟表示法，两两配对合并
时间复杂度：Dijkstra 使用配对堆为 O(E + V log V)
```

**C++ 代码**：

```cpp
struct PNode {
    int val, id;
    int child, sibling;
} ph[MAXN];
int phcnt;

int ph_new(int val, int id) {
    ++phcnt;
    ph[phcnt] = {val, id, 0, 0};
    return phcnt;
}

int ph_merge(int a, int b) {
    if (!a || !b) return a | b;
    if (ph[a].val > ph[b].val) swap(a, b);
    ph[b].sibling = ph[a].child;
    ph[a].child = b;
    return a;
}

int ph_two_pass(int x) {
    if (!x || !ph[x].sibling) return x;
    int a = x, b = ph[x].sibling, rest = ph[b].sibling;
    ph[a].sibling = ph[b].sibling = 0;
    return ph_merge(ph_merge(a, b), ph_two_pass(rest));
}

int ph_pop(int root) {
    return ph_two_pass(ph[root].child);
}

// Dijkstra 使用配对堆
void dijkstra(int s, int n) {
    int root = ph_new(0, s);
    dist[s] = 0;
    while (root) {
        int u = ph[root].id;
        root = ph_pop(root);
        if (vis[u]) continue;
        vis[u] = true;
        for (auto [v, w] : adj[u]) {
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                root = ph_merge(root, ph_new(dist[v], v));
            }
        }
    }
}
```

**核心启示**：
- 配对堆的两两配对合并（two-pass）是其高效性的关键
- 在需要 decrease-key 的场景中，配对堆比二叉堆和左偏树更优
- Dijkstra + 配对堆理论复杂度 $O(E + V\log V)$，优于二叉堆的 $O((E+V)\log V)$

## 五、常见技巧与思路总结

### 5.1 万物皆合并

左偏树的插入是"合并一个单节点堆"，删除是"合并左右子树"。只需写好 merge，其他操作自然实现。

### 5.2 懒标记下推

类似线段树，左偏树也可以打懒标记。在 merge/pop 时先 pushdown，即可支持整体加、整体乘等修改。

### 5.3 并查集配合

很多题目同时需要"查找元素所在集合"和"合并集合"。左偏树与并查集天然互补——find 找根，merge 合堆。可以直接用 `while (tr[x].fa) x = tr[x].fa;` 替代并查集。

### 5.4 大根堆 vs 小根堆

只需在 merge 的比较中改变方向（`<` 变 `>`）即可在大根堆和小根堆之间切换。

### 5.5 可删堆技巧

维护两个堆 A（全部元素）和 B（待删除元素），查询时若 A 的堆顶 == B 的堆顶，则同时弹出。适用于需要删除非堆顶元素的场景。

## 六、解题思路框架

```
第一步：识别问题类型
├── 是否需要维护多个优先队列？
├── 是否需要合并操作？
├── 是否涉及树上子树合并？
└── 是否需要堆上整体修改？

第二步：选择数据结构
├── 仅合并 + 查询/删除最值 → 左偏树（最稳妥）
├── 合并操作极多，删除较少 → 配对堆
├── 需要持久化 → 左偏树（不能用斜堆）
└── 需要删除任意元素 → 可删堆（双堆法）

第三步：确定堆的类型
├── 选最小值 → 小根堆
├── 选最大值 → 大根堆
└── 选中位数 → 对顶堆 / 大根堆维护前半段

第四步：确定是否需要懒标记
├── 整体加值 → add 标记
├── 整体乘值 → mul + add 标记
└── 无整体修改 → 不需要标记

第五步：实现与调试
├── 写好 merge（核心）
├── 根据需要加 pushdown
├── find 函数用循环找根
└── 注意初始化和边界
```

## 七、终极总结与归纳

### 7.1 可并堆的本质

**可并堆 = 堆 + 高效合并。** 左偏树通过维护 dist 和左偏性质，保证右链长度为 $O(\log n)$，从而使递归合并的复杂度得到保证。所有其他操作（插入、删除、查找）都可以归结为 merge。

### 7.2 核心要点

1. **merge 是核心**：左偏树的插入、删除、合并全部归结为 merge 操作
2. **左偏性质**：右链长度 $O(\log n)$，保证效率
3. **懒标记**：在 merge 前 pushdown，可以支持堆上整体修改
4. **树上合并**：DFS 自底向上合并子节点的堆，是处理子树选择问题的利器
5. **可持久化**：复制右链节点即可实现版本管理
6. **配对堆**：合并 $O(1)$，两两配对删除，实际常数极小
