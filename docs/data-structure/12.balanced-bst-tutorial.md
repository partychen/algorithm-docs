## 一、什么是平衡树

平衡树（Balanced Binary Search Tree）是一类在保持**二叉搜索树（BST）性质**的同时，通过某种机制维持树高为 $O(\log n)$ 的数据结构。

普通 BST 在最坏情况下退化为链，导致查询、插入、删除的时间复杂度均为 $O(n)$。平衡树通过**旋转、分裂合并、势能均摊**等手段，保证所有操作的时间复杂度始终为 $O(\log n)$。

**平衡树能支持的典型操作：**

| 操作 | 说明 |
|------|------|
| 插入 / 删除 | 动态维护元素集合 |
| 查询排名 | 给定值 $x$，求集合中比 $x$ 小的元素个数 + 1 |
| 查询第 $k$ 小 | 给定排名 $k$，求对应的值 |
| 前驱 / 后继 | 小于 $x$ 的最大值 / 大于 $x$ 的最小值 |
| 区间翻转 / 区间操作 | 维护序列而非集合时的特殊应用 |

## 二、核心思想与前置知识

### 2.1 BST 性质

对于 BST 中任意节点 $u$：

$$
\forall v \in \text{左子树}(u),\ key(v) < key(u) \quad \text{且} \quad \forall w \in \text{右子树}(u),\ key(w) > key(u)
$$

中序遍历 BST 得到的序列是**有序**的，这是所有平衡树操作的基础。

### 2.2 Treap（旋转式）

Treap = Tree + Heap。每个节点同时拥有一个**键值 key**（满足 BST 性质）和一个**随机优先级 priority**（满足堆性质）。

- 插入时：先按 BST 规则插入，然后通过**旋转**上浮以满足堆性质。
- 删除时：将目标节点旋转到叶子位置后直接删除。
- 旋转分为**左旋（zig）**和**右旋（zag）**，单次旋转 $O(1)$。

由于 priority 是随机的，Treap 的期望树高为 $O(\log n)$。

### 2.3 FHQ-Treap（无旋 / 分裂合并式）

FHQ-Treap（又称非旋 Treap）由范浩强提出，用**分裂（split）与合并（merge）**两个操作替代旋转，代码简洁且天然支持**可持久化**。

**核心操作：**

- **split(u, k) → (l, r)**：将以 $u$ 为根的树按照**权值 $k$**（或**大小**）分裂为两棵树，左树所有键 $\le k$，右树所有键 $> k$。
- **merge(l, r) → u**：将两棵树合并为一棵，要求左树所有键 $<$ 右树所有键，按 priority 维护堆性质。

所有高级操作（插入、删除、查排名、查第 $k$ 小、前驱、后继）均可通过 split 和 merge 组合实现。

### 2.4 Splay（伸展操作）

Splay 树的核心是**伸展操作 splay(x)**：将节点 $x$ 旋转到根的位置。通过**双旋（zig-zig / zig-zag）**保证均摊 $O(\log n)$。

Splay 树的优势在于：
- 支持**区间操作**：将区间 $[l, r]$ 对应的子树提取出来统一处理。
- 实际常数较小，适合维护序列类问题。

### 2.5 FHQ-Treap 核心代码模板

**按值分裂（split by value）：**

```cpp
void split(int u, int k, int &l, int &r) {
    if (!u) { l = r = 0; return; }
    if (val[u] <= k) {
        l = u;
        split(ch[u][1], k, ch[u][1], r);
    } else {
        r = u;
        split(ch[u][0], k, l, ch[u][0]);
    }
    pushup(u);
}
```

**合并（merge）：**

```cpp
int merge(int l, int r) {
    if (!l || !r) return l | r;
    if (pri[l] > pri[r]) {
        ch[l][1] = merge(ch[l][1], r);
        pushup(l);
        return l;
    } else {
        ch[r][0] = merge(l, ch[r][0]);
        pushup(r);
        return r;
    }
}
```

**按大小分裂（split by size，用于序列操作）：**

```cpp
void splitBySize(int u, int k, int &l, int &r) {
    if (!u) { l = r = 0; return; }
    pushdown(u);
    int leftSz = sz[ch[u][0]];
    if (leftSz >= k) {
        r = u;
        splitBySize(ch[u][0], k, l, ch[u][0]);
    } else {
        l = u;
        splitBySize(ch[u][1], k - leftSz - 1, ch[u][1], r);
    }
    pushup(u);
}
```

## 三、分类与变体详解

| 平衡树类型 | 平衡方式 | 特点 | 竞赛适用场景 |
|------------|---------|------|-------------|
| **Treap** | 旋转 + 随机堆 | 实现简单，期望 $O(\log n)$ | 基础平衡树题目 |
| **FHQ-Treap** | 分裂合并 + 随机堆 | 无旋，支持可持久化，代码最短 | **竞赛首选**，适用几乎所有场景 |
| **Splay** | 双旋伸展 | 均摊 $O(\log n)$，擅长区间操作 | LCT 底层结构、区间序列维护 |
| **替罪羊树** | 暴力重构 | 当子树不平衡时整棵重建 | 需要严格最坏 $O(\log n)$ 的静态场景 |
| **红黑树** | 旋转 + 染色 | STL `std::set/map` 底层实现 | 一般不手写，竞赛中直接用 STL |

**选型建议：** 竞赛中优先使用 **FHQ-Treap**（代码短、易调试、功能全）；涉及 LCT 时使用 **Splay**；工程中使用红黑树（STL 自带）。

### 3.1 按值 split vs 按大小 split

| split 类型 | 适用场景 | 典型题目 |
|-----------|---------|---------|
| 按值 split | 维护集合（排名、前驱后继） | P3369 普通平衡树 |
| 按大小 split | 维护序列（区间翻转、区间赋值） | P3391 文艺平衡树 |

### 3.2 可持久化 FHQ-Treap

FHQ-Treap 天然支持可持久化：split 和 merge 时对经过路径上的节点**新建副本**（路径复制），不需要旋转操作。每次操作新增 $O(\log n)$ 个节点。

### 3.3 启发式合并

多棵平衡树需要合并时，将较小的树逐节点插入较大的树。每个节点最多被合并 $O(\log n)$ 次，总复杂度 $O(n \log^2 n)$。

## 四、16 道经典例题详解

### 题目 1：普通平衡树（洛谷 P3369）

🟢 入门

> 维护一个多重集合，支持 6 种操作：插入、删除、查排名、查第 $k$ 小、求前驱、求后继。

**分析：**

- **数据结构**：FHQ-Treap（按值 split）
- **支持操作**：插入、删除、排名、第 $k$ 小、前驱、后继
- **关键性质**：所有操作均可由 split + merge 组合完成
- **时间复杂度**：每次操作期望 $O(\log n)$

**C++ 代码：**

```cpp
// FHQ-Treap 插入
void insert(int x) {
    int l, r;
    split(root, x, l, r);
    root = merge(merge(l, newNode(x)), r);
}
// FHQ-Treap 删除（删一个）
void del(int x) {
    int l, mid, r;
    split(root, x - 1, l, mid);
    split(mid, x, mid, r);
    mid = merge(ch[mid][0], ch[mid][1]); // 去掉根
    root = merge(l, merge(mid, r));
}
// 查询第 k 小
int kth(int u, int k) {
    while (true) {
        if (k <= sz[ch[u][0]]) u = ch[u][0];
        else if (k == sz[ch[u][0]] + 1) return val[u];
        else k -= sz[ch[u][0]] + 1, u = ch[u][1];
    }
}
// 查询排名
int getRank(int x) {
    int l, r, res;
    split(root, x - 1, l, r);
    res = sz[l] + 1;
    root = merge(l, r);
    return res;
}
```

**核心启示：**
- FHQ-Treap 用 split/merge 统一处理所有操作，思路清晰
- 这是竞赛中平衡树的最佳入门选择
- 掌握本题即拥有了平衡树的基本工具箱

### 题目 2：普通平衡树（加强版）（洛谷 P6136）

🟢 入门

> 与 P3369 相同的 6 种操作，但数据量更大（$n, m \le 10^6$），强制在线。要求卡常。

**分析：**

- **数据结构**：FHQ-Treap / 旋转 Treap
- **支持操作**：同 P3369
- **关键性质**：需要优化常数，使用内联函数、快读快写、合理的随机数生成
- **时间复杂度**：每次操作期望 $O(\log n)$，但需要极低常数

**C++ 代码：**

```cpp
// 快速随机数生成器
unsigned int sed = 19260817;
int myRand() {
    sed ^= sed << 13;
    sed ^= sed >> 17;
    sed ^= sed << 5;
    return (int)(sed & 0x7fffffff);
}
// 内联 pushup
inline void pushup(int u) {
    sz[u] = sz[ch[u][0]] + sz[ch[u][1]] + 1;
}
// 非递归 kth
int kth(int u, int k) {
    while (true) {
        int ls = sz[ch[u][0]];
        if (k <= ls) u = ch[u][0];
        else if (k == ls + 1) return val[u];
        else k -= ls + 1, u = ch[u][1];
    }
}
```

**核心启示：**
- 大数据量下平衡树需要优化常数，快速随机数、内联函数都很重要
- 强制在线意味着不能离线排序处理，必须用平衡树在线操作
- 本题是检验平衡树模板效率的标准题

### 题目 3：文艺平衡树（洛谷 P3391）

🟡 进阶

> 维护一个长度为 $n$ 的序列，支持区间翻转操作：将区间 $[l, r]$ 内的元素反转。输出最终序列。

**分析：**

- **数据结构**：FHQ-Treap（按大小 split）
- **支持操作**：区间翻转
- **关键性质**：翻转通过交换左右子树 + 懒标记实现
- **时间复杂度**：每次操作 $O(\log n)$

**C++ 代码：**

```cpp
void pushdown(int u) {
    if (rev[u]) {
        swap(ch[u][0], ch[u][1]);
        if (ch[u][0]) rev[ch[u][0]] ^= 1;
        if (ch[u][1]) rev[ch[u][1]] ^= 1;
        rev[u] = 0;
    }
}
void reverse(int l, int r) {
    int a, b, c;
    splitBySize(root, l - 1, a, b);
    splitBySize(b, r - l + 1, b, c);
    rev[b] ^= 1;
    root = merge(a, merge(b, c));
}
```

**核心启示：**
- 平衡树维护序列时，按**大小**分裂而非按值分裂
- 区间操作通过懒标记下传实现，与线段树思想一致
- 这是学会"平衡树维护序列"的必做题

### 题目 4：书架（洛谷 P3850）

🟡 进阶

> 维护一个书架（序列），支持在位置 $p$ 后插入一本书、查询某本书的位置等操作。

**分析：**

- **数据结构**：FHQ-Treap（按大小 split）
- **支持操作**：任意位置插入、位置查询
- **关键性质**：按大小分裂天然支持位置插入

```cpp
void insertAt(int pos, int bookId) {

int queryPos(int u) {
    int res = sz[ch[u][0]] + 1;
    while (fa[u]) {
        if (u == ch[fa[u]][1])
            res += sz[ch[fa[u]][0]] + 1;
        u = fa[u];

    return res;
}
```

**核心启示：**
- 平衡树的「按大小分裂」天然支持在任意位置插入/删除

### 题目 5：滚榜（洛谷 P5338）

🟡 进阶

> 模拟 ICPC 滚榜过程，每次将一支队伍的罚时更新后重新排序，求最终排名。

**分析：**

- **数据结构**：FHQ-Treap（按值 split）
- **支持操作**：删除、插入、动态有序维护
- **关键性质**：复合比较键 (solved, penalty, id) 的三元组排序
- **时间复杂度**：每次操作期望 $O(\log n)$

**C++ 代码：**

```cpp
struct Key {
    int solved, penalty, id;
    bool operator<=(const Key &o) const {
        if (solved != o.solved) return solved >= o.solved;
        if (penalty != o.penalty) return penalty <= o.penalty;
        return id <= o.id;
    }
};
void update(int team, int addSolved, int addPenalty) {
    del(root, {info[team].solved, info[team].penalty, team});
    info[team].solved += addSolved;
    info[team].penalty += addPenalty;
    insert(root, {info[team].solved, info[team].penalty, team});
}
```

**核心启示：**
- 平衡树天然支持动态有序维护，适合模拟排名变化场景
- 复合比较键的设计是正确排序的关键
- 每次更新需要先删后插，而非直接修改

### 题目 6：郁闷的出纳员（洛谷 P1486）

🟠 提高

> 维护员工工资集合，支持：新员工入职（工资 $\ge$ 最低标准才加入）、全体加薪、全体减薪（减薪后低于标准的自动离开）、查询第 $k$ 高工资。

**分析：**

- **数据结构**：FHQ-Treap（按值 split）
- **支持操作**：插入、批量删除、全局偏移、第 $k$ 大
- **关键性质**：全体加薪/减薪用全局偏移量 delta 实现，无需逐节点修改
- **时间复杂度**：每次操作 $O(\log n)$

**C++ 代码：**

```cpp
int delta = 0, minSal, leaveCnt = 0;
void addSalary(int k) { delta += k; }
void subSalary(int k) {
    delta -= k;
    int trash;
    split(root, minSal - delta - 1, trash, root);
    leaveCnt += sz[trash];
    recycle(trash);
}
void hire(int salary) {
    if (salary < minSal) return;
    insert(root, salary - delta);
}
int queryKth(int k) {
    return kth(root, sz[root] - k + 1) + delta;
}
```

**核心启示：**
- 全局增减操作用偏移量代替逐节点修改，$O(1)$ 完成
- 批量删除不合格元素用 split 一步完成
- 插入时减去偏移量，输出时加上偏移量

### 题目 7：报表统计（洛谷 P1110）

🟠 提高

> 维护若干数列，支持在某一数列末尾追加元素，查询相邻元素之差的最小值（MIN_GAP）以及全局任意两元素之差的最小值（MIN_SORT_GAP）。

**分析：**

- **数据结构**：FHQ-Treap / std::set + multiset
- **支持操作**：插入、前驱后继查询、全局最近对维护
- **关键性质**：MIN_SORT_GAP 每次插入时查前驱后继更新；MIN_GAP 用链表和 multiset 维护相邻差
- **时间复杂度**：每次操作 $O(\log n)$

**C++ 代码：**

```cpp
void insertAndUpdate(int x) {
    int l, r;
    split(root, x, l, r);
    int pre = getMax(l);
    int suc = getMin(r);
    if (pre != -INF) minSortGap = min(minSortGap, x - pre);
    if (suc != INF)  minSortGap = min(minSortGap, suc - x);
    root = merge(merge(l, newNode(x)), r);
}
// MIN_GAP: 用 multiset 维护所有相邻差值
// 追加到第 i 段时，更新旧尾与新尾的差值
```

**核心启示：**
- 全局最小差值可通过平衡树的前驱后继在插入时 $O(\log n)$ 维护
- 区分"相邻元素差"与"排序后相邻差"两种不同含义的最小值
- 多种数据结构协同工作是本题的特色

### 题目 8：文本编辑器（洛谷 P4008）

🟠 提高

> 模拟一个文本编辑器，支持光标移动、插入字符串、删除区间、翻转区间等操作。

**分析：**

- **数据结构**：FHQ-Treap / Splay（按大小 split）
- **支持操作**：光标定位、批量插入、区间删除、区间翻转
- **关键性质**：大量插入用批量建树 $O(n)$ 代替逐个插入 $O(n \log n)$；需要内存回收
- **时间复杂度**：批量建树 $O(n)$，其余操作 $O(\log n)$

**C++ 代码：**

```cpp
int build(char *s, int len) {
    int stk[MAXN], top = 0;
    for (int i = 0; i < len; i++) {
        int u = newNode(s[i]);
        int last = 0;
        while (top && pri[stk[top]] > pri[u])
            pushup(last = stk[top--]);
        ch[u][0] = last;
        if (top) ch[stk[top]][1] = u;
        stk[++top] = u;
    }
    while (top) pushup(stk[top--]);
    return stk[1];
}
// 插入字符串：build 后 merge 到对应位置
// 删除区间：splitBySize 取出后回收
```

**核心启示：**
- 大量插入时使用「批量建树 + 合并」而非逐个插入
- 内存池回收（用栈维护空闲节点）对高操作量题目至关重要
- 类笛卡尔树的 $O(n)$ 建树是常用优化手段

### 题目 9：维护数列（洛谷 P2042）

🔴 困难

> 维护一个整数序列，支持：区间插入、区间删除、区间修改（赋值）、区间翻转、区间求和、求全局最大子段和。共 6 种操作。

**分析：**

- **数据结构**：Splay / FHQ-Treap（按大小 split）
- **支持操作**：区间插入/删除/赋值/翻转/求和/最大子段和
- **关键性质**：多懒标记共存，赋值标记优先于翻转标记；最大子段和需维护前缀、后缀、全局三个量
- **时间复杂度**：每次操作 $O(\log n)$

**C++ 代码：**

```cpp
void pushup(int u) {
    int l = ch[u][0], r = ch[u][1];
    sz[u] = sz[l] + sz[r] + 1;
    sum[u] = sum[l] + sum[r] + val[u];
    lmx[u] = max(lmx[l], sum[l] + val[u] + max(0, lmx[r]));
    rmx[u] = max(rmx[r], sum[r] + val[u] + max(0, rmx[l]));
    mmx[u] = max({mmx[l], mmx[r],
        max(0, rmx[l]) + val[u] + max(0, lmx[r])});
}
void pushAssign(int u, int v) {
    if (!u) return;
    val[u] = v; sum[u] = v * sz[u];
    lmx[u] = rmx[u] = mmx[u] = max(v, v * sz[u]);
    tagA[u] = v; hasA[u] = 1; tagR[u] = 0;
}
```

**核心启示：**
- 多懒标记共存时，明确下传优先级是正确性的关键
- 最大子段和需要维护前缀最大、后缀最大和全局最大三个量
- 被称为「平衡树毒瘤题」，是检验平衡树功底的试金石

### 题目 10：数列（洛谷 P2710）

🔴 困难

> 维护一个数列，支持：区间插入、区间删除、区间赋值、区间翻转、区间求和、求最大子段和、光标移动。（P2042 的加强版）

**分析：**

- **数据结构**：Splay / FHQ-Treap（按大小 split）
- **支持操作**：同 P2042 + 光标移动
- **关键性质**：赋值标记覆盖翻转标记；内存池回收；边界哨兵节点
- **时间复杂度**：每次操作 $O(\log n)$

**C++ 代码：**

```cpp
void pushdown(int u) {
    if (hasAssign[u]) {
        assignTo(ch[u][0], tagAssign[u]);
        assignTo(ch[u][1], tagAssign[u]);
        hasAssign[u] = 0;
        tagRev[u] = 0; // 赋值覆盖翻转
    }
    if (tagRev[u]) {
        revNode(ch[u][0]);
        revNode(ch[u][1]);
        tagRev[u] = 0;
    }
}
void revNode(int u) {
    if (!u) return;
    swap(ch[u][0], ch[u][1]);
    swap(lmx[u], rmx[u]); // 翻转后前缀后缀互换
    tagRev[u] ^= 1;
}
```

**核心启示：**
- 毒瘤题的关键在于懒标记的正确组合与下传顺序
- 推荐先写好 P2042 再挑战本题
- 哨兵节点可以避免大量边界讨论

### 题目 11：永无乡（洛谷 P3224）

🟠 提高

> 给定 $n$ 个岛屿，每个岛有一个重要度。支持两种操作：合并两个岛屿所在的联通块、查询某联通块中重要度第 $k$ 小的岛屿编号。

**分析：**

- **数据结构**：并查集 + FHQ-Treap（启发式合并）
- **支持操作**：联通块合并、联通块内第 $k$ 小查询
- **关键性质**：启发式合并保证每个节点最多被合并 $O(\log n)$ 次
- **时间复杂度**：$O(n \log^2 n)$

**C++ 代码：**

```cpp
void mergeTree(int &dst, int src) {
    if (!src) return;
    mergeTree(dst, ch[src][0]);
    mergeTree(dst, ch[src][1]);
    ch[src][0] = ch[src][1] = 0;
    sz[src] = 1;
    int l, r;
    split(dst, val[src], l, r);
    dst = merge(merge(l, src), r);
}
void unite(int a, int b) {
    a = find(a); b = find(b);
    if (a == b) return;
    if (sz[root[a]] < sz[root[b]]) swap(a, b);
    mergeTree(root[a], root[b]);
    fa[b] = a;
}
```

**核心启示：**
- 并查集 + 平衡树启发式合并是处理「动态联通块 + 有序查询」的经典套路
- 启发式合并保证每个节点最多被合并 $O(\log n)$ 次
- 也可使用线段树合并替代

### 题目 12：八邻旁之桥（洛谷 P3644）

🔴 困难

> 在一条河的两岸各有若干地点，需要建最多两座桥，使得所有人从一岸到另一岸的总距离最小。

**分析：**

- **数据结构**：FHQ-Treap（动态中位数维护）
- **支持操作**：动态插入、中位数查询、距离和维护
- **关键性质**：中位数最优性——绝对值之和最小化的最优点是中位数；两桥时枚举分界点
- **时间复杂度**：$O(n \log n)$

**C++ 代码：**

```cpp
void addPoint(int x) {
    insert(root, x);
    n++;
    int mid = kth(root, (n + 1) / 2);
    // 维护 sumL = 中位数左侧元素之和, cntL = 个数
    // 维护 sumR = 中位数右侧元素之和, cntR = 个数
    // 总距离 = mid * cntL - sumL + sumR - mid * cntR
}
long long getCost() {
    int mid = kth(root, (n + 1) / 2);
    return (long long)mid * cntL - sumL + sumR - (long long)mid * cntR;
}
```

**核心启示：**
- 中位数最优性是绝对值求和最小化的经典结论
- 动态中位数用平衡树维护，配合前缀/后缀枚举分界点
- 建模能力是本题的核心考察点

### 题目 13：方伯伯的 OJ（洛谷 P3285）

🔴 困难

> 维护 $n$ 个用户的排名，支持：将用户 $x$ 移到队首/队尾/某用户之前或之后，查询用户 $x$ 的排名，查询排名为 $k$ 的用户。

**分析：**

- **数据结构**：Splay / FHQ-Treap
- **支持操作**：移动到首/尾/指定位置、排名查询、第 $k$ 大查询
- **关键性质**：$n$ 可达 $10^8$，使用区间压缩——一个节点代表一段连续编号，操作时动态分裂

```cpp
void extractUser(int x) {
    int u = locateInterval(root, x);
    int lo = intv[u].l, hi = intv[u].r;
    splay(u);

    root = merge(merge(left, single), right);
}
```

- 值域极大但操作数有限时，使用区间压缩（一个节点代表一段连续区间）

### 题目 14：可持久化文艺平衡树（洛谷 P5055）

🔴 困难

> 维护一个可持久化的序列，支持在历史版本上进行：在指定位置插入元素、删除指定位置元素、区间翻转操作。每次操作生成新版本。

**分析：**

- **数据结构**：可持久化 FHQ-Treap
- **支持操作**：版本化的插入、删除、区间翻转
- **关键性质**：split/merge 时路径复制（新建节点），不修改历史版本；pushdown 时也需要先复制再修改
- **时间复杂度**：每次操作 $O(\log n)$，空间 $O(n \log n)$

**C++ 代码：**

```cpp
int copyNode(int u) {
    int v = ++tot;
    ch[v][0] = ch[v][0 + 0]; // copy from u
    ch[v][1] = ch[u][1];
    val[v] = val[u]; pri[v] = pri[u];
    sz[v] = sz[u]; rev[v] = rev[u];
    return v;
}
void split(int u, int k, int &l, int &r) {
    if (!u) { l = r = 0; return; }
    u = copyNode(u); // 路径复制
    pushdown(u);
    int ls = sz[ch[u][0]];
    if (ls >= k) {
        r = u;
        split(ch[u][0], k, l, ch[u][0]);
    } else {
        l = u;
        split(ch[u][1], k - ls - 1, ch[u][1], r);
    }
    pushup(u);
}
```

**核心启示：**
- FHQ-Treap 天然支持可持久化，只需在 split/merge 进入节点时复制
- pushdown 也必须在复制后的节点上进行，否则修改了历史版本
- 这是可持久化平衡树的模板题

### 题目 15：Splay 区间操作模板（SPOJ-GSS6）

🟣 竞赛

> 维护一个整数序列，支持：在位置 $p$ 后插入元素、删除位置 $p$ 的元素、查询区间 $[l,r]$ 的最大子段和。

**分析：**

- **数据结构**：Splay
- **支持操作**：位置插入、位置删除、区间最大子段和
- **关键性质**：Splay 通过旋转到根来提取区间；维护 lmax/rmax/mmax 三个量
- **时间复杂度**：每次操作均摊 $O(\log n)$

**C++ 代码：**

```cpp
// Splay 提取区间 [l, r]
// 将 l-1 旋转到根，r+1 旋转到根的右子树
// 则 root->right->left 即为 [l, r] 对应子树
void splay(int x, int goal) {
    while (fa[x] != goal) {
        int y = fa[x], z = fa[y];
        if (z != goal)
            (ch[y][0] == x) == (ch[z][0] == y) ?
                rotate(y) : rotate(x);
        rotate(x);
    }
    if (!goal) root = x;
}
int getInterval(int l, int r) {
    int x = kthNode(l - 1), y = kthNode(r + 1);
    splay(x, 0);
    splay(y, x);
    return ch[y][0]; // 区间 [l, r] 的子树根
}
```

**核心启示：**
- Splay 通过将关键节点旋转到根来提取区间，思路与 FHQ-Treap 的 split 不同但效果相同
- 维护最大子段和需要 lmax/rmax/mmax 的合并，与线段树上的做法一致
- Splay 是 LCT 的基础，掌握它才能进一步学习动态树

### 题目 16：送别（洛谷 P5321）

🟣 竞赛

> 给定一棵树，每个节点有权值。支持子树权值排序后查询第 $k$ 小、子树合并等操作。强制在线。

**分析：**

- **数据结构**：FHQ-Treap 启发式合并 / 线段树合并
- **支持操作**：子树合并、子树内第 $k$ 小
- **关键性质**：DFS 过程中从叶子向根合并平衡树；启发式合并保证复杂度
- **时间复杂度**：$O(n \log^2 n)$

**C++ 代码：**

```cpp
void dfs(int u, int fa) {
    root[u] = newNode(w[u]);
    for (int v : adj[u]) {
        if (v == fa) continue;
        dfs(v, u);
        if (sz[root[v]] > sz[root[u]])
            swap(root[u], root[v]);
        mergeTree(root[u], root[v]);
    }
    ans[u] = kth(root[u], query[u]);
}
```

**核心启示：**
- 树上子树信息合并是树上问题的经典模型
- 当涉及有序信息时，平衡树合并（或线段树合并）是核心工具
- 省选/NOI+ 级别题目常将树结构与平衡树交织

## 五、常用技巧与优化

1. **全局偏移量**：全体加减操作不修改节点值，而是维护一个偏移量 delta，查询时加上 delta。适用于"所有元素同时加/减"的场景（如 P1486）。

2. **哨兵节点**：在序列两端各插入一个哨兵节点，避免提取区间时的越界讨论。Splay 中尤为常用。

3. **内存池回收**：大量插入删除时，用栈维护已删除节点的编号，新建节点时优先复用。避免内存超限。

4. **批量建树**：将一段有序数据建成平衡树时，使用类似笛卡尔树的 $O(n)$ 建树法，而非逐个插入的 $O(n \log n)$。

5. **区间压缩**：值域极大时，一个节点代表一段连续区间，操作时动态分裂出需要的具体位置（如 P3285）。

6. **启发式合并**：多棵平衡树需要合并时，将较小的树逐节点插入较大的树。每个节点最多被合并 $O(\log n)$ 次，总复杂度 $O(n \log^2 n)$。

7. **可持久化**：FHQ-Treap 天然支持可持久化——split 和 merge 时对经过的路径新建节点即可，不需要旋转。

8. **懒标记组合**：多种懒标记共存时，明确优先级。典型规则：赋值标记覆盖翻转标记；下传顺序必须一致。

## 六、解题框架

```
Step 1：明确问题类型
  ├─ 维护集合（权值相关）→ 按值 split
  └─ 维护序列（位置相关）→ 按大小 split

Step 2：确定需要支持的操作
  ├─ 基础：插入 / 删除 / 排名 / 第 k 小 / 前驱 / 后继
  ├─ 区间：区间和 / 区间最值 / 区间翻转 / 区间赋值
  └─ 高级：合并多棵树 / 可持久化 / 配合其他数据结构

Step 3：选择平衡树变体
  ├─ FHQ-Treap：默认选择，适合绝大多数场景
  ├─ Splay：需要 LCT 底层或频繁区间操作
  └─ 替罪羊树 / 红黑树：特殊场景

Step 4：设计节点信息与懒标记
  ├─ 节点信息：sz, val, pri, 以及题目要求的附加信息（sum, max, etc.）
  ├─ 懒标记：翻转 / 赋值 / 加减 等
  └─ 明确 pushdown 的下传顺序和 pushup 的合并方式

Step 5：实现核心操作
  ├─ split / merge（FHQ-Treap）
  ├─ 或 rotate / splay（Splay）
  └─ 在此基础上组合出题目要求的操作

Step 6：处理边界与工程细节
  ├─ 哨兵节点
  ├─ 内存回收
  ├─ 初始化（随机种子、空节点的特殊值）
  └─ 调试：中序遍历输出验证 BST 性质
```

## 七、终极总结与归纳

### 7.1 平衡树的本质

**平衡树 = BST + 平衡机制。** 在 BST 的基础上，通过额外的平衡机制（随机优先级、伸展操作、势能分析）将最坏 $O(n)$ 优化为 $O(\log n)$。FHQ-Treap 的 split/merge 范式将所有操作统一为两个基本原子操作，极大简化了思维和编码复杂度。

### 7.2 核心要点

1. **FHQ-Treap 是竞赛首选**：代码短、易调试、功能全，按值 split 维护集合，按大小 split 维护序列
2. **Splay 擅长区间操作和 LCT**：双旋保证均摊复杂度，是 LCT 的底层结构
3. **懒标记的下传优先级必须一致**：赋值 > 翻转，多标记共存时明确规则
4. **全局偏移量**：全局加减操作的 $O(1)$ 技巧
5. **启发式合并**：多棵平衡树合并时的复杂度保证
6. **可持久化**：FHQ-Treap 路径复制天然支持，Splay 不支持

### 7.5 口诀

> ⚖ **平衡树心法口诀**
>
> 平衡根基在 BST，左小右大序有方。
> Treap 随机优先堆，旋转上浮保平衡。
> FHQ 分裂加合并，无旋代码最简洁。
> Splay 双旋到根节，均摊对数不会差。
> 按值分裂维护集，按大小分护序列。
> 翻转赋值打标记，下传顺序要记牢。
> 偏移合并压区间，技巧用好省时间。
> 可持久化路径复制，split merge 万能钥匙。
