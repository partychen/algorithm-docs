## 一、什么是 K-D Tree

**K-D Tree**（k-dimensional tree）是一种用于组织 k 维空间中点集的**二叉搜索树**。它通过交替选择划分维度，将高维空间递归二分，广泛应用于最近邻搜索、范围查询、高维数据检索等场景。

```
二维 K-D Tree 的空间划分过程：

      ┌───────────────────┐
      │         P1        │   第 0 层按 x 划分
      │    ┌────┼────┐    │
      │    │    │ P3 │    │   第 1 层按 y 划分
      │    │ P2 │----│    │
      │    │    │ P4 │    │   第 2 层按 x 划分
      │    └────┼────┘    │
      └───────────────────┘
```

**核心操作与复杂度：**

| 操作 | 期望复杂度 | 最坏复杂度 |
|------|-----------|-----------|
| 建树 | $O(n \log n)$ | $O(n \log n)$ |
| 最近邻查询 | $O(\sqrt{n})$ | $O(n)$ |
| 范围查询 | $O(\sqrt{n})$ | $O(n)$ |
| 动态插入（替罪羊） | $O(\sqrt{n})$ 均摊 | $O(n)$ |

## 二、核心思想与前置知识

### 2.1 建树（交替划分维度）

1. **选择划分维度**：深度 $d$ 的节点选第 $d \bmod k$ 维
2. **选择划分点**：取当前维度的**中位数**（用 `nth_element` $O(n)$ 完成）
3. **递归构建**：左子树 < 中位数，右子树 > 中位数
4. **维护包围盒**：每个节点记录子树中所有点的坐标极值 `mn[]/mx[]`

```cpp
const int N = 200005;
const double alpha = 0.725;
int cur_dim;

struct Point {
    int d[2], val;
    bool operator<(const Point& o) const { return d[cur_dim] < o.d[cur_dim]; }
};

struct Node {
    Point p;
    int mn[2], mx[2], sz, sum, ls, rs;
} tr[N];

int root, tot;
Point tmp[N];

void pushup(int u) {
    int l = tr[u].ls, r = tr[u].rs;
    tr[u].sz = 1; tr[u].sum = tr[u].p.val;
    for (int i = 0; i < 2; i++) {
        tr[u].mn[i] = tr[u].mx[i] = tr[u].p.d[i];
        if (l) tr[u].mn[i] = min(tr[u].mn[i], tr[l].mn[i]),
               tr[u].mx[i] = max(tr[u].mx[i], tr[l].mx[i]);
        if (r) tr[u].mn[i] = min(tr[u].mn[i], tr[r].mn[i]),
               tr[u].mx[i] = max(tr[u].mx[i], tr[r].mx[i]);
    }
    if (l) tr[u].sz += tr[l].sz, tr[u].sum += tr[l].sum;
    if (r) tr[u].sz += tr[r].sz, tr[u].sum += tr[r].sum;
}

int build(int l, int r, int dep) {
    if (l > r) return 0;
    int mid = (l + r) >> 1, u = ++tot;
    cur_dim = dep % 2;
    nth_element(tmp + l, tmp + mid, tmp + r + 1);
    tr[u].p = tmp[mid];
    tr[u].ls = build(l, mid - 1, dep + 1);
    tr[u].rs = build(mid + 1, r, dep + 1);
    pushup(u);
    return u;
}
```

### 2.2 最近邻查询

1. 根据查询点与划分维度的关系，**优先进入更近的子树**
2. 回溯时用**包围盒到查询点的最小距离**（下界）剪枝
3. 若下界 $\geq$ 当前最优距离，则跳过另一侧子树

### 2.3 矩形范围查询

利用包围盒判断三种情况：
- **完全包含**：返回子树 sum
- **完全不交**：返回 0（剪枝）
- **部分相交**：检查当前点 + 递归两子树

### 2.4 替罪羊重构（动态插入）

当 $\max(sz_{left}, sz_{right}) > \alpha \cdot sz_{cur}$ 时，将子树拍扁重建，保证平衡。

```cpp
void flatten(int u, int& cnt) {
    if (!u) return;
    flatten(tr[u].ls, cnt);
    tmp[++cnt] = tr[u].p;
    flatten(tr[u].rs, cnt);
}

void check(int& u, int dep) {
    if (tr[u].ls && tr[tr[u].ls].sz > alpha * tr[u].sz ||
        tr[u].rs && tr[tr[u].rs].sz > alpha * tr[u].sz) {
        int cnt = 0; flatten(u, cnt);
        u = build(1, cnt, dep);
    }
}

void insert(int& u, Point p, int dep) {
    if (!u) { u = ++tot; tr[u].p = p; tr[u].ls = tr[u].rs = 0; pushup(u); return; }
    cur_dim = dep % 2;
    if (p < tr[u].p) insert(tr[u].ls, p, dep + 1);
    else insert(tr[u].rs, p, dep + 1);
    pushup(u); check(u, dep);
}
```

## 三、分类与变体详解

### 3.1 变体总览

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| **静态 K-D Tree** | 一次建树，不修改 | 离线查询 |
| **动态 K-D Tree（替罪羊）** | 支持插入，不平衡时重建 | 在线插入+查询 |
| **带权 K-D Tree** | 维护子树权值和 | 矩形区域权值查询 |
| **高维 K-D Tree** | $k \geq 3$ | 多维偏序（效率随维度下降） |
| **定期重构 K-D Tree** | 每 $\sqrt{n}$ 次操作重建 | 简化替罪羊实现 |

### 3.2 估价函数选择

| 距离度量 | 最近邻估价（下界） | 最远邻估价（上界） |
|---------|------------------|------------------|
| 欧几里得 | $\sum \max(0, \text{mn}_i - q_i, q_i - \text{mx}_i)^2$ | $\sum \max(|q_i - \text{mn}_i|, |q_i - \text{mx}_i|)^2$ |
| 曼哈顿 | $\sum \max(0, \text{mn}_i - q_i, q_i - \text{mx}_i)$ | $\sum \max(|q_i - \text{mn}_i|, |q_i - \text{mx}_i|)$ |

### 3.3 K-D Tree vs 其他空间数据结构

| 数据结构 | 维度适用 | 动态操作 | 查询类型 |
|---------|---------|---------|---------|
| K-D Tree | 2-4 维 | 替罪羊重构 | 最近邻、范围 |
| 二维线段树 | 2 维 | 支持 | 矩形区间操作 |
| CDQ 分治+BIT | 3 维 | 离线 | 偏序问题 |

### 题目 1：平面最近点对（经典问题）

> 给定平面上 $n$ 个点，求距离最近的两个点的欧几里得距离。

**分析**：建 K-D Tree 后对每个点查询最近邻，取全局最小值。也可以用分治法 $O(n \log n)$ 解，K-D Tree 是期望 $O(n \sqrt{n})$ 的替代方案。

```
数据结构：静态 K-D Tree
支持操作：最近邻查询
关键性质：估价函数为点到包围盒的欧几里得距离下界
时间复杂度：期望 O(n√n)
```

**C++ 代码**：

```cpp
long long min_dist_sq(int u, Point& q) { // 点到包围盒的最小距离²
    long long res = 0;
    for (int i = 0; i < 2; i++) {
        if (q.d[i] < tr[u].mn[i]) res += (long long)(tr[u].mn[i] - q.d[i]) * (tr[u].mn[i] - q.d[i]);
        if (q.d[i] > tr[u].mx[i]) res += (long long)(q.d[i] - tr[u].mx[i]) * (q.d[i] - tr[u].mx[i]);
    }
    return res;
}

long long ans;
void query_near(int u, Point& q) {
    if (!u) return;
    long long d = 0;
    for (int i = 0; i < 2; i++)
        d += (long long)(q.d[i] - tr[u].p.d[i]) * (q.d[i] - tr[u].p.d[i]);
    if (d > 0) ans = min(ans, d); // 排除自身
    long long dl = tr[u].ls ? min_dist_sq(tr[u].ls, q) : 1e18;
    long long dr = tr[u].rs ? min_dist_sq(tr[u].rs, q) : 1e18;
    if (dl < dr) {
        if (dl < ans) query_near(tr[u].ls, q);
        if (dr < ans) query_near(tr[u].rs, q);
    } else {
        if (dr < ans) query_near(tr[u].rs, q);
        if (dl < ans) query_near(tr[u].ls, q);
    }
}
```

**核心启示**：
- **优先搜索更近子树**：先进入估价更小的子树，让 `ans` 尽快收紧，增加剪枝效果
- **估价函数是剪枝核心**：下界估价必须正确（不能高估），否则会漏解

### 题目 2：矩形区域计数（基础范围查询）

> 平面上 $n$ 个点，$q$ 次查询，每次给定矩形 $[x_1, y_1, x_2, y_2]$，求矩形内的点数。

**分析**：静态建 K-D Tree，包围盒判断完全包含/完全不交/部分相交。

```
数据结构：静态 K-D Tree
支持操作：矩形范围计数
关键性质：三种情况判断——完全包含返回 sz、完全不交返回 0、部分相交递归
时间复杂度：期望每次 O(√n)
```

**C++ 代码**：

```cpp
int range_count(int u, int x1, int y1, int x2, int y2) {
    if (!u) return 0;
    if (tr[u].mn[0] >= x1 && tr[u].mx[0] <= x2 &&
        tr[u].mn[1] >= y1 && tr[u].mx[1] <= y2) return tr[u].sz;
    if (tr[u].mx[0] < x1 || tr[u].mn[0] > x2 ||
        tr[u].mx[1] < y1 || tr[u].mn[1] > y2) return 0;
    int res = 0;
    int px = tr[u].p.d[0], py = tr[u].p.d[1];
    if (px >= x1 && px <= x2 && py >= y1 && py <= y2) res++;
    res += range_count(tr[u].ls, x1, y1, x2, y2);
    res += range_count(tr[u].rs, x1, y1, x2, y2);
    return res;
}
```

**核心启示**：
- **包围盒三分类**：完全包含/完全不交/部分相交是所有范围查询的固定模式
- **维护 `sz`/`sum`**：根据需要在 pushup 中维护不同的聚合信息

### 题目 3：K 远点对（洛谷 P4357）

> 给定平面上 $n$ 个点，求第 $k$ 远的点对距离。

**分析**：维护大小为 $2k$ 的小根堆（点对 $(i,j)$ 和 $(j,i)$ 都会被计算），对每个点在 K-D Tree 上查询最远点。

```
数据结构：静态 K-D Tree + 小根堆
支持操作：对每个点查最远距离，堆维护前 2k 大
关键性质：最远邻用包围盒最大距离（上界）剪枝
时间复杂度：期望 O(n√n · log k)
```

**C++ 代码**：

```cpp
priority_queue<long long, vector<long long>, greater<long long>> pq;

long long max_dist_sq(int u, Point& q) {
    long long res = 0;
    for (int i = 0; i < 2; i++)
        res += max((long long)(q.d[i]-tr[u].mn[i])*(q.d[i]-tr[u].mn[i]),
                   (long long)(q.d[i]-tr[u].mx[i])*(q.d[i]-tr[u].mx[i]));
    return res;
}

void query_far(int u, Point& q) {
    if (!u) return;
    long long d = 0;
    for (int i = 0; i < 2; i++)
        d += (long long)(q.d[i]-tr[u].p.d[i])*(q.d[i]-tr[u].p.d[i]);
    if (d > pq.top()) { pq.pop(); pq.push(d); }
    long long dl = tr[u].ls ? max_dist_sq(tr[u].ls, q) : -1;
    long long dr = tr[u].rs ? max_dist_sq(tr[u].rs, q) : -1;
    if (dl > dr) {
        if (dl > pq.top()) query_far(tr[u].ls, q);
        if (dr > pq.top()) query_far(tr[u].rs, q);
    } else {
        if (dr > pq.top()) query_far(tr[u].rs, q);
        if (dl > pq.top()) query_far(tr[u].ls, q);
    }
}
// 堆初始化 2k 个 0，最后堆顶即为答案
```

**核心启示**：
- **反转估价方向**：最远邻用最大距离上界剪枝，与最近邻完全对称
- **堆大小 $2k$**：无序点对 $(i,j)$ 和 $(j,i)$ 各计一次

### 题目 4：简单题（洛谷 P4148）

> $n \times n$ 矩阵初始全 0，支持单点加值和矩形求和。强制在线。

**分析**：动态插入带权点 + 矩形范围查询。替罪羊重构维护平衡。

```
数据结构：动态 K-D Tree（替罪羊重构）
支持操作：动态插入带权点、矩形范围求和
关键性质：替罪羊重构保证均摊 O(√n) 每次操作
时间复杂度：均摊 O(√n) 每次操作
```

**C++ 代码**：

```cpp
long long range_sum(int u, int x1, int y1, int x2, int y2) {
    if (!u) return 0;
    if (tr[u].mn[0]>=x1 && tr[u].mx[0]<=x2 &&
        tr[u].mn[1]>=y1 && tr[u].mx[1]<=y2) return tr[u].sum;
    if (tr[u].mx[0]<x1 || tr[u].mn[0]>x2 ||
        tr[u].mx[1]<y1 || tr[u].mn[1]>y2) return 0;
    long long res = 0;
    if (tr[u].p.d[0]>=x1 && tr[u].p.d[0]<=x2 &&
        tr[u].p.d[1]>=y1 && tr[u].p.d[1]<=y2) res += tr[u].p.val;
    res += range_sum(tr[u].ls, x1, y1, x2, y2);
    res += range_sum(tr[u].rs, x1, y1, x2, y2);
    return res;
}
```

**核心启示**：
- **动态 K-D Tree 模板题**：强制在线 + 单点修改 + 矩形查询的标配
- **$\alpha$ 调参**：通常 0.7~0.75，影响重构频率和树高

### 题目 5：天使玩偶（洛谷 P4169）

> 平面上动态插入点，每次查询距某点最近的已有点的曼哈顿距离。强制在线。

**分析**：动态 K-D Tree + 曼哈顿距离最近邻。估价函数改为曼哈顿距离下界。

```
数据结构：动态 K-D Tree
支持操作：动态插入、曼哈顿最近邻查询
关键性质：曼哈顿距离下界 = Σ max(0, mn[i]-q[i], q[i]-mx[i])
时间复杂度：均摊 O(√n) 每次操作
```

**C++ 代码**：

```cpp
long long min_manhattan(int u, Point& q) {
    long long res = 0;
    for (int i = 0; i < 2; i++) {
        if (q.d[i] < tr[u].mn[i]) res += tr[u].mn[i] - q.d[i];
        if (q.d[i] > tr[u].mx[i]) res += q.d[i] - tr[u].mx[i];
    }
    return res;
}

void query_near_manh(int u, Point& q, long long& ans) {
    if (!u) return;
    long long d = abs(q.d[0]-tr[u].p.d[0]) + abs(q.d[1]-tr[u].p.d[1]);
    ans = min(ans, d);
    long long dl = tr[u].ls ? min_manhattan(tr[u].ls, q) : 1e18;
    long long dr = tr[u].rs ? min_manhattan(tr[u].rs, q) : 1e18;
    if (dl < dr) {
        if (dl < ans) query_near_manh(tr[u].ls, q, ans);
        if (dr < ans) query_near_manh(tr[u].rs, q, ans);
    } else {
        if (dr < ans) query_near_manh(tr[u].rs, q, ans);
        if (dl < ans) query_near_manh(tr[u].ls, q, ans);
    }
}
```

**核心启示**：
- **曼哈顿距离估价**：与欧几里得类似，但改为绝对值求和
- **优先搜索更近子树**：对实际运行效率影响巨大

### 题目 7：最大曼哈顿距离（洛谷 P2479）

> 平面上 $n$ 个点，求曼哈顿距离最大的两个点的距离。

**分析**：对每个点在 K-D Tree 上查最远曼哈顿距离。估价函数用包围盒顶点的曼哈顿距离上界。

```
数据结构：静态 K-D Tree
支持操作：最远曼哈顿距离查询
关键性质：曼哈顿距离上界 = Σ max(|q[i]-mn[i]|, |q[i]-mx[i]|)
时间复杂度：期望 O(n√n)
```

**C++ 代码**：

```cpp
long long max_manhattan(int u, Point& q) {
    long long res = 0;
    for (int i = 0; i < 2; i++)
        res += max((long long)abs(q.d[i] - tr[u].mn[i]),
                   (long long)abs(q.d[i] - tr[u].mx[i]));
    return res;
}

void query_max_manh(int u, Point& q, long long& ans) {
    if (!u) return;
    long long d = abs(q.d[0]-tr[u].p.d[0]) + abs(q.d[1]-tr[u].p.d[1]);
    ans = max(ans, d);
    long long dl = tr[u].ls ? max_manhattan(tr[u].ls, q) : -1;
    long long dr = tr[u].rs ? max_manhattan(tr[u].rs, q) : -1;
    if (dl > dr) {
        if (dl > ans) query_max_manh(tr[u].ls, q, ans);
        if (dr > ans) query_max_manh(tr[u].rs, q, ans);
    } else {
        if (dr > ans) query_max_manh(tr[u].rs, q, ans);
        if (dl > ans) query_max_manh(tr[u].ls, q, ans);
    }
}
```

**核心启示**：
- **曼哈顿最远也可以用坐标变换**：$(x,y) \to (x+y, x-y)$ 转切比雪夫后取 max 更快
- **K-D Tree 是通用方案**：不需要特殊变换也能解

### 题目 8：巧克力王国（洛谷 P4475）

> $n$ 个带权点，$m$ 次查询半平面 $ax + by < c$ 内点的权值和。

**分析**：将包围盒 4 个顶点代入 $ax+by$ 求极值——全部 $< c$ 返回 sum，全部 $\geq c$ 返回 0，否则递归。

```
数据结构：静态 K-D Tree
支持操作：半平面范围查询
关键性质：包围盒 4 顶点的 ax+by 极值判断完全包含/不交
时间复杂度：期望 O(√n) 每次查询
```

**C++ 代码**：

```cpp
long long query_half(int u, long long a, long long b, long long c) {
    if (!u) return 0;
    long long v[4] = {
        a*tr[u].mn[0]+b*tr[u].mn[1], a*tr[u].mn[0]+b*tr[u].mx[1],
        a*tr[u].mx[0]+b*tr[u].mn[1], a*tr[u].mx[0]+b*tr[u].mx[1]
    };
    long long vmin = *min_element(v, v+4), vmax = *max_element(v, v+4);
    if (vmax < c) return tr[u].sum;
    if (vmin >= c) return 0;
    long long res = 0;
    if (a*tr[u].p.d[0]+b*tr[u].p.d[1] < c) res += tr[u].p.val;
    res += query_half(tr[u].ls, a, b, c);
    res += query_half(tr[u].rs, a, b, c);
    return res;
}
```

**核心启示**：
- **K-D Tree 不只能查矩形**：任何能用包围盒快速判定的几何条件均可作为剪枝依据
- **半平面查询**：包围盒 4 顶点代入线性函数求极值

### 题目 9：JZPFAR（洛谷 P2093）

> $n$ 个点，$m$ 次查询，每次给定查询点和 $k$，求距查询点第 $k$ 远的点的编号。

**分析**：小根堆维护前 $k$ 远。自定义比较规则处理同距离取编号小的。

```
数据结构：K-D Tree + 小根堆（自定义比较）
支持操作：第 k 远查询
关键性质：同距离优先取编号小 → 自定义堆比较
时间复杂度：期望 O(√n · log k) 每次查询
```

**C++ 代码**：

```cpp
struct Info {
    long long dist; int id;
    bool operator<(const Info& o) const {
        return dist != o.dist ? dist > o.dist : id < o.id;
    }
};
priority_queue<Info> pq; // 自定义小根堆

void query_kth_far(int u, Point& q) {
    if (!u) return;
    long long d = 0;
    for (int i = 0; i < 2; i++)
        d += (long long)(q.d[i]-tr[u].p.d[i])*(q.d[i]-tr[u].p.d[i]);
    Info cur = {d, tr[u].p.d[0]}; // 用点的某属性作 id
    if (pq.top() < cur) { pq.pop(); pq.push(cur); }
    long long dl = tr[u].ls ? max_dist_sq(tr[u].ls, q) : -1;
    long long dr = tr[u].rs ? max_dist_sq(tr[u].rs, q) : -1;
    if (dl > dr) {
        if (dl >= pq.top().dist) query_kth_far(tr[u].ls, q);
        if (dr >= pq.top().dist) query_kth_far(tr[u].rs, q);
    } else {
        if (dr >= pq.top().dist) query_kth_far(tr[u].rs, q);
        if (dl >= pq.top().dist) query_kth_far(tr[u].ls, q);
    }
}
```

**核心启示**：
- **自定义比较处理优先级**：同距离时的 tie-breaking 规则需要在堆的比较函数中明确
- **堆 + K-D Tree 组合通用**：第 k 近/远都可以这样解决

### 题目 10：TATT 四维偏序（洛谷 P3769）

> 四维空间 $n$ 个点有权值，求最长上升子序列（四维都严格递增）的最大权值和。

**分析**：第一维排序消去后，用三维 K-D Tree 维护 DP。对每个点查三维均小于它的最大 DP 值，更新后插入。

```
数据结构：三维动态 K-D Tree
支持操作：三维偏序查最大值、动态插入
关键性质：排序降一维 + K-D Tree 处理剩余三维
时间复杂度：期望 O(n · n^{2/3}) 或更好
```

**C++ 代码**：

```cpp
// 三维 K-D Tree，节点维护子树 dp 最大值
int query_max_dp(int u, int y, int z, int w) {
    if (!u) return 0;
    if (tr[u].mx[0]<y && tr[u].mx[1]<z && tr[u].mx[2]<w) return tr[u].max_dp;
    if (tr[u].mn[0]>=y || tr[u].mn[1]>=z || tr[u].mn[2]>=w) return 0;
    int res = 0;
    if (tr[u].p.d[0]<y && tr[u].p.d[1]<z && tr[u].p.d[2]<w) res = tr[u].dp_val;
    res = max(res, query_max_dp(tr[u].ls, y, z, w));
    res = max(res, query_max_dp(tr[u].rs, y, z, w));
    return res;
}
```

**核心启示**：
- **K-D Tree 处理高维偏序**：虽然理论复杂度不如 CDQ 优秀，但代码简单且竞赛中常能 AC
- **查询条件 = 剪枝条件**：包围盒的 mn/mx 直接对应偏序的判断

### 题目 11：弹跳（洛谷 P5471）

> $n$ 个弹跳装置在平面上，每个装置可花费 $w_i$ 跳到一个矩形区域内的任意装置。求从装置 1 到所有其他装置的最短路。

**分析**：Dijkstra + K-D Tree 批量松弛。取出当前最近点后，在 K-D Tree 上查矩形内所有未访问点并松弛。已访问的点从 K-D Tree 中标记删除。

```
数据结构：K-D Tree + Dijkstra
支持操作：矩形范围内批量松弛、标记删除
关键性质：每个点最多被松弛一次后删除 → 总复杂度可控
时间复杂度：O(n√n log n)
```

**C++ 代码**：

```cpp
void relax(int u, int x1, int y1, int x2, int y2, long long w,
           priority_queue<pair<long long,int>, vector<pair<long long,int>>, greater<>>& pq,
           long long dist[]) {
    if (!u || tr[u].alive == 0) return;
    if (tr[u].mx[0]<x1 || tr[u].mn[0]>x2 || tr[u].mx[1]<y1 || tr[u].mn[1]>y2) return;
    if (tr[u].mn[0]>=x1 && tr[u].mx[0]<=x2 && tr[u].mn[1]>=y1 && tr[u].mx[1]<=y2) {
        collect_and_relax(u, w, pq, dist); // 遍历子树所有存活点
        return;
    }
    auto& p = tr[u].p;
    if (p.d[0]>=x1 && p.d[0]<=x2 && p.d[1]>=y1 && p.d[1]<=y2 && !visited[p.id])
        if (w < dist[p.id]) { dist[p.id] = w; pq.push({w, p.id}); }
    relax(tr[u].ls, x1,y1,x2,y2, w, pq, dist);
    relax(tr[u].rs, x1,y1,x2,y2, w, pq, dist);
}
```

**核心启示**：
- **K-D Tree + Dijkstra**：解决边数爆炸的最短路，矩形批量松弛是核心思路
- **删除已访问点**：保证每个点只被松弛一次

### 题目 12：K-D Tree 优化 DP

**分析**：二维偏序 DP，K-D Tree 维护满足偏序条件的最优 DP 值。当维度较高时（3-4维），K-D Tree 是比分治更简单的选择。

```
数据结构：动态 K-D Tree
支持操作：二维偏序查最值、动态插入
关键性质：DP 转移 = 偏序查询 + 单点插入
时间复杂度：期望 O(n√n)
```

**核心启示**：
- **K-D Tree 优化 DP**：当 DP 转移涉及多维偏序条件时，K-D Tree 是通用工具
- **简单直接**：相比 CDQ+BIT 的多层嵌套，K-D Tree 代码量更少

```
数据结构：三维 K-D Tree
```

### 题目 15：高维 K 近邻（综合）

> 给定 $d$ 维空间（$d \leq 4$）中 $n$ 个点和 $q$ 次查询，每次求查询点的 $k$ 个最近邻。

**分析**：K-D Tree + 大小为 $k$ 的大根堆。维度越高剪枝效率越低，需要更精细的估价函数（如方差最大维度划分）。

```

关键性质：维度选择策略影响效率——可选方差最大的维度划分

```

**核心启示**：
- **方差最大维度划分**：不一定按 `dep % k` 轮换，选方差最大的维度能更好地分割数据
- **维度诅咒**：$d \geq 5$ 时 K-D Tree 几乎退化为暴力

### 题目 16：K-D Tree 在计算几何中的综合应用

> 给定平面上 $n$ 个点和 $m$ 条线段/半平面/圆形查询，求每次查询的某种统计信息。

**分析**：K-D Tree 的包围盒剪枝可以推广到圆形查询（点到包围盒的最近距离 $\leq$ 半径则可能相交）、凸多边形查询等。关键是设计正确的"乐观估价函数"。

```
关键性质：包围盒是 K-D Tree 剪枝的万能接口，适配各种几何条件
时间复杂度：取决于查询形状与数据分布
```

**核心启示**：
- **估价函数的设计**：这是 K-D Tree 最核心的技能。能否高效剪枝取决于估价函数的质量
- **K-D Tree 的本质是搜索+剪枝**：不是精确数据结构，而是启发式加速工具

## 五、常见技巧与思路总结

### 5.1 `nth_element` 加速建树

用 `nth_element` 而非 `sort` 选中位数，建树从 $O(n \log^2 n)$ 降至 $O(n \log n)$。

### 5.2 优先搜索更优子树

查询时先进入估价更优的子树，让答案尽快收紧，增加另一侧的剪枝成功率。

### 5.3 估价函数必须是乐观的

最近邻用下界（不能高估），最远邻用上界（不能低估）。错误的估价会漏解。

### 5.4 替罪羊参数

$\alpha = 0.7 \sim 0.75$。偏大→重构少但树偏斜；偏小→重构频繁但树更平衡。

### 5.5 曼哈顿 ↔ 切比雪夫转换

$(x,y) \to (x+y, x-y)$ 将曼哈顿距离转为切比雪夫距离，有时更方便处理。

## 七、终极总结与归纳

### 7.1 K-D Tree 的本质

**K-D Tree = 交替维度划分的 BST + 包围盒剪枝。** 它本质上是一种搜索加速器，通过包围盒快速排除不可能的区域，将暴力 $O(n)$ 查询降至期望 $O(\sqrt{n})$。

### 7.2 核心要点

1. **交替维度划分 + 中位数**：保证树的平衡性，用 `nth_element` $O(n)$ 选中位数
2. **包围盒是剪枝核心**：所有查询的效率都取决于包围盒能排除多少区域
3. **估价函数必须乐观**：最近邻用下界，最远邻用上界，否则会漏解
4. **优先搜索更优子树**：对实际效率影响巨大，是必须的优化
5. **替罪羊重构**：动态插入时维护平衡，$\alpha \approx 0.725$
6. **维度不宜太高**：$k \leq 4$ 时效果好，$k \geq 5$ 时退化

### 7.5 口诀

> 🗺️ **K-D Tree 心法口诀**
>
> 交替维度分空间，中位数处把树建。

> 维度不过四五层，超了暴力也无妨。
