## 一、什么是猫树

猫树（Cat Tree），也叫**不相交稀疏表（Disjoint Sparse Table）**，是一种专门为**静态区间查询**设计的数据结构。它的核心卖点只有一句话：**对满足结合律的运算，做到 $O(n \log n)$ 预处理、$O(1)$ 查询**。

普通线段树查询一次需要在树上走 $O(\log n)$ 步，每步合并一次；ST 表虽然查询 $O(1)$，但要求运算满足**幂等性**（如 $\min$、$\max$、$\gcd$），不能处理区间和、区间乘、矩阵乘这些非幂等运算。猫树填补了这个空白——只要运算**满足结合律**，它就能做到 $O(1)$ 查询。

猫树的名字来源已不可考，但它的结构确实像猫一样灵巧：在线段树的分治骨架上，每个节点从中点向两侧预处理前缀/后缀信息，查询时只需找到 LCA、合并两半，一步到位。

```text
              [1, 8]              ← 第 0 层
           /          \
       [1, 4]        [5, 8]      ← 第 1 层
       /    \        /    \
    [1,2]  [3,4]  [5,6]  [7,8]  ← 第 2 层

每个节点存储：
  从 mid 向左的后缀累积信息
  从 mid+1 向右的前缀累积信息

查询 [3, 7]：
  LCA = [1, 8]（第 0 层），mid = 4
  答案 = suf[0][3] ⊕ pre[0][7]   ← 一次合并，O(1)
```

| 数据结构 | 预处理 | 单次查询 | 支持修改 | 运算要求 |
|----------|--------|----------|----------|----------|
| 线段树 | $O(n)$ | $O(\log n)$ | ✅ | 结合律 |
| ST 表 | $O(n \log n)$ | $O(1)$ | ❌ | 结合律 + 幂等性 |
| **猫树** | $O(n \log n)$ | $O(1)$ | ❌ | **仅需结合律** |
| 前缀和 | $O(n)$ | $O(1)$ | ❌ | 需有逆运算 |

## 二、核心思想与前置知识

### 2.1 分治结构与层的概念

猫树建立在线段树的分治骨架上，但不关心"树的节点"，而关心**层**。

把序列长度补到 2 的幂 $N$。第 $d$ 层把序列分成若干大小为 $2^{L-d}$ 的块（$L = \log_2 N$）。每个块有一个中点 $mid$，从 $mid$ 向左累积后缀信息，从 $mid+1$ 向右累积前缀信息。

```text
序列: a[1..8]，N=8，L=3

第 0 层（块大小 8）：一整块 [1,8]，mid=4
  suf[0]: ← 从 4 向左累积到 1
  pre[0]: → 从 5 向右累积到 8

第 1 层（块大小 4）：两块 [1,4] [5,8]
  [1,4] mid=2:  suf[1] ← 从 2 向左    pre[1] → 从 3 向右
  [5,8] mid=6:  suf[1] ← 从 6 向左    pre[1] → 从 7 向右

第 2 层（块大小 2）：四块 [1,2] [3,4] [5,6] [7,8]
  每块各有一个 mid，向两侧各累积一个元素
```

总共 $O(\log n)$ 层，每层处理 $O(n)$ 个元素，预处理总复杂度 $O(n \log n)$。

### 2.2 查询的关键：O(1) 找到正确的层

查询 $[l, r]$ 时，需要找到**最深的一层**，使得 $l$ 和 $r$ 分属同一块的左右两半。这等价于找 $l$ 和 $r$ 对应叶节点在线段树上的 LCA。

用堆式编号：叶节点 $l$ 的编号为 $l + N - 1$（1-indexed）。两个叶节点 $x, y$ 的 LCA 深度可以通过**异或**直接算出：

$$d = L - \lfloor \log_2(x \oplus y) \rfloor$$

其中 $x = l + N - 1$，$y = r + N - 1$。预处理对数表后，这一步 $O(1)$。

```cpp
// 预处理对数表
int lg[MAXN * 2];
lg[1] = 0;
for (int i = 2; i < 2 * N; i++)
    lg[i] = lg[i >> 1] + 1;

// 查询时确定层号
int dep = lg[N] - lg[(l + N - 1) ^ (r + N - 1)];
```

### 2.3 建树模板

```cpp
const int MAXN = 1 << 18;
const int LOG = 18;

long long pre[LOG][MAXN + 1]; // pre[d][i]: 第 d 层，从块左半末尾+1 向右累积到 i
long long suf[LOG][MAXN + 1]; // suf[d][i]: 第 d 层，从块右半起点-1 向左累积到 i
int a[MAXN + 1];
int N, L; // N = 补齐后长度, L = log2(N)
int lg[MAXN * 2];

void build() {
    // 补到 2 的幂
    N = 1; L = 0;
    while (N < n) N <<= 1, L++;
    for (int i = n + 1; i <= N; i++) a[i] = 0; // 补零（按运算的单位元）

    // 预处理对数表
    lg[1] = 0;
    for (int i = 2; i < 2 * N; i++) lg[i] = lg[i >> 1] + 1;

    // 逐层建树
    for (int d = 0; d < L; d++) {
        int block = N >> d; // 当前层的块大小
        int half = block >> 1;
        for (int bl = 1; bl <= N; bl += block) {
            int mid = bl + half - 1;
            // 后缀：从 mid 向左
            suf[d][mid] = a[mid];
            for (int i = mid - 1; i >= bl; i--)
                suf[d][i] = suf[d][i + 1] + a[i]; // 替换 + 为具体运算
            // 前缀：从 mid+1 向右
            pre[d][mid + 1] = a[mid + 1];
            for (int i = mid + 2; i <= bl + block - 1; i++)
                pre[d][i] = pre[d][i - 1] + a[i]; // 替换 + 为具体运算
        }
    }
}
```

### 2.4 查询模板

```cpp
long long query(int l, int r) {
    if (l == r) return a[l];
    int d = lg[N] - lg[(l + N - 1) ^ (r + N - 1)];
    return suf[d][l] + pre[d][r]; // 替换 + 为具体合并运算
}
```

整个查询过程：一次异或、一次查表、一次合并。$O(1)$。

### 2.5 正确性直觉

为什么 $suf[d][l]$ 和 $pre[d][r]$ 拼起来就是 $[l, r]$ 的答案？

在第 $d$ 层，$l$ 和 $r$ 属于同一个块 $[bl, bl+block-1]$，中点为 $mid$。由分治结构保证 $l \le mid < r$。于是：

$$[l, r] = [l, mid] \cup [mid+1, r]$$

$suf[d][l]$ 是 $a[l] \oplus a[l+1] \oplus \cdots \oplus a[mid]$（从 $mid$ 向左累积到 $l$），$pre[d][r]$ 是 $a[mid+1] \oplus \cdots \oplus a[r]$（从 $mid+1$ 向右累积到 $r$）。合并即得 $[l, r]$ 的结果。

## 三、与相关数据结构的对比

### 3.1 猫树 vs ST 表

ST 表利用"区间可以重叠覆盖"的性质实现 $O(1)$ 查询，但这要求运算满足幂等性（$x \oplus x = x$）。

猫树利用"区间可以无重叠拆分"的性质实现 $O(1)$ 查询，只需要结合律。

```
ST 表：  [l ......... r]
         [--- 2^k ---]
             [--- 2^k ---]
         两段重叠，但 min/max 重叠无所谓（幂等）

猫树：   [l ......... r]
         [l .. mid][mid+1 .. r]
         两段恰好不重叠，任何满足结合律的运算都行
```

| 维度 | ST 表 | 猫树 |
|------|-------|------|
| 预处理 | $O(n \log n)$ | $O(n \log n)$ |
| 查询 | $O(1)$ | $O(1)$ |
| 运算要求 | 结合律 + 幂等性 | 仅需结合律 |
| 适用运算 | $\min, \max, \gcd, \text{OR}, \text{AND}$ | 加法、乘法、矩阵乘、线性基合并等 |
| 空间 | $O(n \log n)$ | $O(n \log n)$ |

### 3.2 猫树 vs 线段树

线段树功能全面但查询需 $O(\log n)$ 步合并。当合并代价 $M$ 较高时（如矩阵乘 $M = O(k^3)$），查询总代价 $O(M \log n)$ 很大。猫树把查询降到 $O(M)$——只做一次合并。

| 维度 | 线段树 | 猫树 |
|------|--------|------|
| 预处理 | $O(nM)$ | $O(nM \log n)$ |
| 查询 | $O(M \log n)$ | $O(M)$ |
| 修改 | $O(M \log n)$ | ❌ 不支持 |
| 适用场景 | 动态/静态均可 | 仅静态 |

**什么时候猫树更优？**  当查询次数远大于 $n$ 且不需要修改时，猫树的 $O(1)$ 查询优势明显。特别是当 $M$ 很大时（矩阵乘、线性基合并），猫树直接省掉 $\log n$ 倍的合并，收益巨大。

### 3.3 猫树 vs 前缀和

前缀和也能 $O(1)$ 查询，但要求运算有**逆运算**（如减法之于加法、除法之于乘法）。矩阵乘、$\min$、$\max$ 这类运算没有方便的逆运算，前缀和就用不了，猫树可以。

### 3.4 适用场景总结

```
需要区间查询
 ├─ 有修改？
 │   └─ 是：用线段树 / 树状数组
 ├─ 运算有逆运算？（如加法）
 │   └─ 是：前缀和最简单
 ├─ 运算满足幂等性？（如 min/max/gcd）
 │   └─ 是：ST 表即可
 └─ 运算只有结合律？（如矩阵乘、线性基合并）
     └─ 猫树是最佳选择
```

### 🟢 入门级 Level 1：先把基本框架跑通

### 题目 1：静态区间和（模板题）

> 给定长为 $n$ 的数组和 $q$ 次查询，每次查询 $[l, r]$ 的元素之和。$n, q \le 10^5$。

**分析**：这是猫树的 Hello World。虽然区间和用前缀和更简单，但这道题用来理解猫树的建树和查询流程最合适。

```
数据结构：猫树
支持操作：区间求和
关键性质：加法满足结合律
时间复杂度：预处理 O(n log n)，查询 O(1)
```

**C++ 代码**：
```cpp
const int MAXN = 1 << 17;
const int LOG = 17;

long long pre[LOG][MAXN + 1], suf[LOG][MAXN + 1];
int a[MAXN + 1];
int N, L, lg[MAXN * 2];

void build(int n) {
    N = 1; L = 0;
    while (N < n) N <<= 1, L++;
    for (int i = n + 1; i <= N; i++) a[i] = 0;
    lg[1] = 0;
    for (int i = 2; i < 2 * N; i++) lg[i] = lg[i >> 1] + 1;
    for (int d = 0; d < L; d++) {
        int block = N >> d, half = block >> 1;
        for (int bl = 1; bl <= N; bl += block) {
            int mid = bl + half - 1;
            suf[d][mid] = a[mid];
            for (int i = mid - 1; i >= bl; i--)
                suf[d][i] = suf[d][i + 1] + a[i];
            pre[d][mid + 1] = a[mid + 1];
            for (int i = mid + 2; i <= bl + block - 1; i++)
                pre[d][i] = pre[d][i - 1] + a[i];
        }
    }
}

long long query(int l, int r) {
    if (l == r) return a[l];
    int d = lg[N] - lg[(l + N - 1) ^ (r + N - 1)];
    return suf[d][l] + pre[d][r];
}
```

**核心启示**：
- **猫树的骨架只有三步**：补齐长度、逐层从中点向两侧累积、查询时异或定层。
- **区间和用猫树是大材小用**，但它完美展示了工作原理。
- **`lg` 表预处理是 $O(1)$ 查询的关键**，所有猫树题共用这个技巧。

### 题目 2：静态区间最大子段和（洛谷 P4513 简化版）

> 给定长为 $n$ 的数组，$q$ 次查询 $[l, r]$ 内连续子数组的最大和。无修改操作。

**分析**：区间最大子段和需要维护四元组 $(sum, pre_{\max}, suf_{\max}, ans)$，合并操作不满足幂等性，ST 表做不了。线段树可以但查询 $O(\log n)$。猫树 $O(1)$ 查询，每次只做一次四元组合并。

```
数据结构：猫树，信息类型为四元组
支持操作：区间最大子段和查询
关键性质：四元组合并满足结合律但不满足幂等性
时间复杂度：预处理 O(n log n)，查询 O(1)
```

**C++ 代码**：
```cpp
struct Info {
    long long sum, pre, suf, ans;
    Info() : sum(0), pre(0), suf(0), ans(0) {}
    Info(long long v) : sum(v), pre(v), suf(v), ans(v) {}
};
Info merge(const Info& a, const Info& b) {
    Info c;
    c.sum = a.sum + b.sum;
    c.pre = max(a.pre, a.sum + b.pre);
    c.suf = max(b.suf, b.sum + a.suf);
    c.ans = max({a.ans, b.ans, a.suf + b.pre});
    return c;
}

const int MAXN = 1 << 17;
const int LOG = 17;
Info pre_info[LOG][MAXN + 1], suf_info[LOG][MAXN + 1];
Info single[MAXN + 1]; // 单点信息
int N, L, lg[MAXN * 2];

void build(int n, int* a) {
    N = 1; L = 0;
    while (N < n) N <<= 1, L++;
    for (int i = 1; i <= n; i++) single[i] = Info(a[i]);
    for (int i = n + 1; i <= N; i++) single[i] = Info(0);
    lg[1] = 0;
    for (int i = 2; i < 2 * N; i++) lg[i] = lg[i >> 1] + 1;
    for (int d = 0; d < L; d++) {
        int block = N >> d, half = block >> 1;
        for (int bl = 1; bl <= N; bl += block) {
            int mid = bl + half - 1;
            suf_info[d][mid] = single[mid];
            for (int i = mid - 1; i >= bl; i--)
                suf_info[d][i] = merge(single[i], suf_info[d][i + 1]);
            pre_info[d][mid + 1] = single[mid + 1];
            for (int i = mid + 2; i <= bl + block - 1; i++)
                pre_info[d][i] = merge(pre_info[d][i - 1], single[i]);
        }
    }
}

Info query(int l, int r) {
    if (l == r) return single[l];
    int d = lg[N] - lg[(l + N - 1) ^ (r + N - 1)];
    return merge(suf_info[d][l], pre_info[d][r]);
}
// 答案为 query(l, r).ans
```

**核心启示**：
- **把猫树的 `long long` 换成自定义结构体 + 自定义 merge**，就能处理各种复杂信息。
- **最大子段和的四元组合并**满足结合律但不满足幂等性，这正是猫树相比 ST 表的优势所在。
- **模板的通用性**：改 `Info` 和 `merge` 就能适配不同问题，猫树框架本身不变。

### 🟡 进阶级 Level 2：处理更复杂的合并运算

### 题目 3：区间乘积取模

> 给定长为 $n$ 的正整数数组和质数 $p$，$q$ 次查询 $[l, r]$ 内所有元素乘积模 $p$。无修改。$n, q \le 2 \times 10^5$。

**分析**：乘法有结合律，但模意义下的乘法**不满足幂等性**（$a \times a \ne a$），ST 表不适用。前缀积需要模逆元，而 $a_i$ 可能是 $p$ 的倍数（逆元不存在）。猫树直接 $O(1)$ 查询，无需逆元。

```
数据结构：猫树
支持操作：区间乘积模 p
关键性质：模乘满足结合律，但不满足幂等性且不一定有逆元
时间复杂度：预处理 O(n log n)，查询 O(1)
```

**C++ 代码**：
```cpp
const int MAXN = 1 << 18;
const int LOG = 18;

long long pre[LOG][MAXN + 1], suf[LOG][MAXN + 1];
int a[MAXN + 1];
int N, L, lg[MAXN * 2];
long long MOD;

void build(int n) {
    N = 1; L = 0;
    while (N < n) N <<= 1, L++;
    for (int i = n + 1; i <= N; i++) a[i] = 1; // 乘法单位元
    lg[1] = 0;
    for (int i = 2; i < 2 * N; i++) lg[i] = lg[i >> 1] + 1;
    for (int d = 0; d < L; d++) {
        int block = N >> d, half = block >> 1;
        for (int bl = 1; bl <= N; bl += block) {
            int mid = bl + half - 1;
            suf[d][mid] = a[mid] % MOD;
            for (int i = mid - 1; i >= bl; i--)
                suf[d][i] = suf[d][i + 1] * a[i] % MOD;
            pre[d][mid + 1] = a[mid + 1] % MOD;
            for (int i = mid + 2; i <= bl + block - 1; i++)
                pre[d][i] = pre[d][i - 1] * a[i] % MOD;
        }
    }
}

long long query(int l, int r) {
    if (l == r) return a[l] % MOD;
    int d = lg[N] - lg[(l + N - 1) ^ (r + N - 1)];
    return suf[d][l] * pre[d][r] % MOD;
}
```

**核心启示**：
- **补零位置要填运算的单位元**：加法填 0，乘法填 1，min 填 $+\infty$。
- **猫树绕过了"逆元不存在"的问题**，因为它根本不需要逆运算。
- **只改三处就从区间和变成了区间积**：单位元、累积运算、合并运算。

### 题目 4：区间矩阵乘（CF 718C 简化版）

> 给定 $n$ 个 $2 \times 2$ 矩阵，$q$ 次查询 $[l, r]$ 内矩阵的连乘积，结果对 $10^9 + 7$ 取模。无修改。

**分析**：矩阵乘法满足结合律但**不满足交换律**（顺序不能换），更不满足幂等性。单次矩阵乘代价 $O(k^3)$（$k=2$ 时是常数）。线段树查询 $O(\log n)$ 次矩阵乘；猫树只做 $O(1)$ 次矩阵乘。

```
数据结构：猫树，信息类型为 2×2 矩阵
支持操作：区间矩阵连乘
关键性质：矩阵乘满足结合律，合并代价 O(k^3)
时间复杂度：预处理 O(n k^3 log n)，查询 O(k^3)
```

**C++ 代码**：
```cpp
const long long MOD = 1e9 + 7;
struct Mat {
    long long a[2][2];
    Mat() { memset(a, 0, sizeof(a)); }
};
Mat mul(const Mat& A, const Mat& B) {
    Mat C;
    for (int i = 0; i < 2; i++)
        for (int j = 0; j < 2; j++)
            for (int k = 0; k < 2; k++)
                C.a[i][j] = (C.a[i][j] + A.a[i][k] * B.a[k][j]) % MOD;
    return C;
}
Mat identity() { Mat I; I.a[0][0] = I.a[1][1] = 1; return I; }

const int MAXN = 1 << 18;
const int LOG = 18;
Mat pre_m[LOG][MAXN + 1], suf_m[LOG][MAXN + 1];
Mat single_m[MAXN + 1];
int N, L, lg[MAXN * 2];

void build(int n) {
    N = 1; L = 0;
    while (N < n) N <<= 1, L++;
    for (int i = n + 1; i <= N; i++) single_m[i] = identity();
    lg[1] = 0;
    for (int i = 2; i < 2 * N; i++) lg[i] = lg[i >> 1] + 1;
    for (int d = 0; d < L; d++) {
        int block = N >> d, half = block >> 1;
        for (int bl = 1; bl <= N; bl += block) {
            int mid = bl + half - 1;
            suf_m[d][mid] = single_m[mid];
            for (int i = mid - 1; i >= bl; i--)
                suf_m[d][i] = mul(single_m[i], suf_m[d][i + 1]);
            pre_m[d][mid + 1] = single_m[mid + 1];
            for (int i = mid + 2; i <= bl + block - 1; i++)
                pre_m[d][i] = mul(pre_m[d][i - 1], single_m[i]);
        }
    }
}

Mat query(int l, int r) {
    if (l == r) return single_m[l];
    int d = lg[N] - lg[(l + N - 1) ^ (r + N - 1)];
    return mul(suf_m[d][l], pre_m[d][r]);
}
```

**核心启示**：
- **矩阵乘是猫树最经典的发力场景**：合并代价高、无逆运算、非幂等。
- **注意乘法顺序**：后缀是 `single[i] * suf[i+1]`，前缀是 `pre[i-1] * single[i]`，不能反。
- **空间是主要瓶颈**：$O(n \log n)$ 个矩阵，$k$ 大时内存可能吃紧。

### 题目 5：区间仿射变换复合

> 给定 $n$ 个仿射变换 $f_i(x) = a_i x + b_i$（模 $998244353$），$q$ 次查询：给定 $[l, r]$ 和 $x$，求 $f_r(f_{r-1}(\cdots f_l(x)))$。无修改。

**分析**：仿射变换的复合 $f \circ g$ 也是仿射变换：$(a_1, b_1) \circ (a_2, b_2) = (a_1 a_2,\; a_1 b_2 + b_1)$。这满足结合律但不满足幂等性。猫树直接套用。

```
数据结构：猫树，信息类型为仿射变换 (a, b)
支持操作：区间仿射变换复合
关键性质：仿射变换复合满足结合律
时间复杂度：预处理 O(n log n)，查询 O(1)
```

**C++ 代码**：
```cpp
const long long MOD = 998244353;
struct Affine { long long a, b; }; // f(x) = a*x + b

Affine compose(const Affine& f, const Affine& g) {
    // f(g(x)) = f.a * (g.a * x + g.b) + f.b = f.a*g.a*x + f.a*g.b+f.b
    return {f.a * g.a % MOD, (f.a * g.b + f.b) % MOD};
}
Affine identity_af() { return {1, 0}; }

const int MAXN = 1 << 18;
const int LOG = 18;
Affine pre_af[LOG][MAXN + 1], suf_af[LOG][MAXN + 1];
Affine single_af[MAXN + 1];
int N, L, lg[MAXN * 2];

void build(int n) {
    N = 1; L = 0;
    while (N < n) N <<= 1, L++;
    for (int i = n + 1; i <= N; i++) single_af[i] = identity_af();
    lg[1] = 0;
    for (int i = 2; i < 2 * N; i++) lg[i] = lg[i >> 1] + 1;
    for (int d = 0; d < L; d++) {
        int block = N >> d, half = block >> 1;
        for (int bl = 1; bl <= N; bl += block) {
            int mid = bl + half - 1;
            suf_af[d][mid] = single_af[mid];
            for (int i = mid - 1; i >= bl; i--)
                suf_af[d][i] = compose(suf_af[d][i + 1], single_af[i]);
                // 注意：f_l(f_{l+1}(...)) 所以是右边先作用，compose(外层, 内层)
            pre_af[d][mid + 1] = single_af[mid + 1];
            for (int i = mid + 2; i <= bl + block - 1; i++)
                pre_af[d][i] = compose(single_af[i], pre_af[d][i - 1]);
                // f_r(f_{r-1}(...)) 所以新的在外层
        }
    }
}

Affine query(int l, int r) {
    if (l == r) return single_af[l];
    int d = lg[N] - lg[(l + N - 1) ^ (r + N - 1)];
    return compose(suf_af[d][l], pre_af[d][r]);
    // suf 是从 mid 到 l 的复合，pre 是从 mid+1 到 r 的复合
    // 总效果：f_l(...f_mid(f_{mid+1}(...f_r(x))))
    // 但合并时 suf 在外层（先执行的在内层），需要 compose(外, 内) = compose(suf, pre)
}

// 最终求值：Affine f = query(l, r); ans = (f.a * x + f.b) % MOD;
```

**核心启示**：
- **仿射变换复合是"函数查询"的最简单形态**，为后面更复杂的转移函数复合做铺垫。
- **compose 的顺序要想清楚**：谁先作用、谁后作用，搞反了结果全错。
- **猫树天然适合"函数复合"类问题**：每个位置是一个函数，区间查询就是函数的链式复合。

### 题目 6：区间 GCD 与区间和的组合查询

> 给定长为 $n$ 的正整数数组，$q$ 次查询 $[l, r]$ 内的 $\gcd$ 和 区间和。无修改。$n, q \le 5 \times 10^5$。

**分析**：GCD 满足幂等性，可以用 ST 表。区间和有逆运算，可以用前缀和。但如果题目要求在**同一个数据结构**里同时维护这两种信息（比如后续有更复杂的合并），用猫树统一处理最整洁。每个位置存 $(val, sum)$，合并时分别取 $\gcd$ 和 $+$。

```
数据结构：猫树，信息类型为 (gcd, sum) 二元组
支持操作：区间 GCD + 区间和
关键性质：两种运算各自满足结合律
时间复杂度：预处理 O(n log n log V)，查询 O(log V)（gcd 的合并代价）
```

**C++ 代码**：
```cpp
struct Info {
    long long g, s; // gcd 和 sum
};
Info make(long long v) { return {v, v}; }
Info merge(const Info& a, const Info& b) {
    return {__gcd(a.g, b.g), a.s + b.s};
}
Info unit() { return {0, 0}; } // gcd 的单位元是 0

const int MAXN = 1 << 20;
const int LOG = 20;
Info pre_info[LOG][MAXN + 1], suf_info[LOG][MAXN + 1];
Info single[MAXN + 1];
int N, L, lg[MAXN * 2];

void build(int n) {
    N = 1; L = 0;
    while (N < n) N <<= 1, L++;
    for (int i = n + 1; i <= N; i++) single[i] = unit();
    lg[1] = 0;
    for (int i = 2; i < 2 * N; i++) lg[i] = lg[i >> 1] + 1;
    for (int d = 0; d < L; d++) {
        int block = N >> d, half = block >> 1;
        for (int bl = 1; bl <= N; bl += block) {
            int mid = bl + half - 1;
            suf_info[d][mid] = single[mid];
            for (int i = mid - 1; i >= bl; i--)
                suf_info[d][i] = merge(single[i], suf_info[d][i + 1]);
            pre_info[d][mid + 1] = single[mid + 1];
            for (int i = mid + 2; i <= bl + block - 1; i++)
                pre_info[d][i] = merge(pre_info[d][i - 1], single[i]);
        }
    }
}

Info query(int l, int r) {
    if (l == r) return single[l];
    int d = lg[N] - lg[(l + N - 1) ^ (r + N - 1)];
    return merge(suf_info[d][l], pre_info[d][r]);
}
```

**核心启示**：
- **猫树可以"打包"多种信息**，只要合并函数整体满足结合律。
- **这体现了猫树作为"万能静态区间查询"的通用性**。
- **当单独看每种信息都有更简单的做法时，猫树的价值在于统一框架**。

### 题目 7：静态区间线性基查询（洛谷 P4839）

> 给定 $n$ 个整数，$q$ 次查询：$[l, r]$ 内选若干个数异或，能得到的最大值。无修改。$n, q \le 5 \times 10^5$，$a_i < 2^{60}$。

**分析**：这道题是猫树发挥最大优势的经典场景。线性基的合并代价为 $O(\log^2 V)$（$V$ 为值域），但**逐元素插入线性基的代价只有 $O(\log V)$**。

线段树做法：建树 $O(n \log V)$，每次查询在线段树上走 $O(\log n)$ 步，每步合并线性基 $O(\log^2 V)$，总查询 $O(q \log n \log^2 V)$。

猫树做法：建树每层逐元素插入，$O(n \log n \log V)$。查询只做一次线性基合并，$O(\log^2 V)$。当 $q$ 很大时优势巨大。

```
数据结构：猫树，信息类型为线性基
支持操作：区间线性基合并 + 最大异或值查询
关键性质：建树时"逐元素插入"比"合并"快
时间复杂度：预处理 O(n log n log V)，查询 O(log^2 V)
```

**C++ 代码**：
```cpp
const int B = 60;
struct Basis {
    long long b[B];
    Basis() { memset(b, 0, sizeof(b)); }
    void insert(long long x) {
        for (int i = B - 1; i >= 0; i--) {
            if (!(x >> i & 1)) continue;
            if (!b[i]) { b[i] = x; return; }
            x ^= b[i];
        }
    }
    long long max_xor() const {
        long long res = 0;
        for (int i = B - 1; i >= 0; i--)
            res = max(res, res ^ b[i]);
        return res;
    }
};
Basis merge_basis(const Basis& a, const Basis& b) {
    Basis c = a;
    for (int i = B - 1; i >= 0; i--)
        if (b.b[i]) c.insert(b.b[i]);
    return c;
}

const int MAXN = 1 << 20;
const int LOG = 20;
Basis pre_b[LOG][MAXN + 1], suf_b[LOG][MAXN + 1];
long long val[MAXN + 1];
int N, L, lg[MAXN * 2];

void build(int n) {
    N = 1; L = 0;
    while (N < n) N <<= 1, L++;
    lg[1] = 0;
    for (int i = 2; i < 2 * N; i++) lg[i] = lg[i >> 1] + 1;
    for (int d = 0; d < L; d++) {
        int block = N >> d, half = block >> 1;
        for (int bl = 1; bl <= N; bl += block) {
            int mid = bl + half - 1;
            // 后缀：从 mid 向左逐个插入
            suf_b[d][mid] = Basis();
            suf_b[d][mid].insert(val[mid]);
            for (int i = mid - 1; i >= bl; i--) {
                suf_b[d][i] = suf_b[d][i + 1]; // 复制
                suf_b[d][i].insert(val[i]);      // 插入，O(log V)
            }
            // 前缀：从 mid+1 向右逐个插入
            pre_b[d][mid + 1] = Basis();
            pre_b[d][mid + 1].insert(val[mid + 1]);
            for (int i = mid + 2; i <= bl + block - 1; i++) {
                pre_b[d][i] = pre_b[d][i - 1];
                pre_b[d][i].insert(val[i]);
            }
        }
    }
}

long long query(int l, int r) {
    if (l == r) { Basis t; t.insert(val[l]); return t.max_xor(); }
    int d = lg[N] - lg[(l + N - 1) ^ (r + N - 1)];
    Basis res = merge_basis(suf_b[d][l], pre_b[d][r]); // 只做一次合并
    return res.max_xor();
}
```

**核心启示**：
- **"逐元素插入便宜、整体合并昂贵"的信息是猫树的最佳客户**。
- **建树时每层从中点向两侧逐个 insert**，而不是两两合并，这是猫树比线段树快的根本原因。
- **空间是痛点**：$O(n \log n)$ 个线性基，每个 60 个 `long long`，需谨慎估算内存。

### 题目 8：区间线性递推查询

> 给定 $n$ 个二元组 $(a_i, b_i)$，定义递推 $f_0 = x$，$f_i = a_i \cdot f_{i-1} + b_i$。$q$ 次查询：给定 $[l, r]$ 和初始值 $x$，求 $f_r$。无修改。

**分析**：每一步递推就是一个仿射变换 $f_i(x) = a_i x + b_i$。区间 $[l, r]$ 的递推结果就是 $f_r \circ f_{r-1} \circ \cdots \circ f_l$ 作用于 $x$。这和题目 5 完全一样，但这里强调**递推视角**。

实际上，更一般地，如果递推涉及 $k$ 维向量（如斐波那契数列 $k=2$），每步递推对应一个 $k \times k$ 转移矩阵，区间递推就是区间矩阵乘。猫树把查询从 $O(k^3 \log n)$ 降到 $O(k^3)$。

```
数据结构：猫树，信息类型为仿射变换/转移矩阵
支持操作：区间递推求值
关键性质：递推的每一步是函数复合，满足结合律
时间复杂度：预处理 O(n log n)，查询 O(1)（对仿射变换；O(k^3) 对矩阵）
```

**C++ 代码**：
```cpp
// 与题目 5 完全一致，此处展示查询调用方式
// 给定 [l, r] 和初始值 x，求递推结果
long long solve(int l, int r, long long x) {
    Affine f = query(l, r);
    return (f.a % MOD * x % MOD + f.b) % MOD;
}
```

**核心启示**：
- **"递推"和"函数复合"是同一件事**。看到"区间递推"就应该想到猫树。
- **从一维仿射到 $k$ 维矩阵，猫树的模板不变**，只是 `Info` 和 `merge` 变了。
- **猫树的本质是"静态区间函数复合器"**。

### 题目 9：摸鱼（猫树分治 + 分段转移函数复合）

> 实习了 $n$ 天，第 $i$ 天工作量为 $a_i$（可能为负）。有一个 01 序列 $\{b_i\}$，$b_i = 1$ 表示第 $i$ 天是关键日期。$q$ 次查询 $[l, r]$：把 $[a_l, \ldots, a_r]$ 分成若干段，每段**恰好包含一个**关键日期。若一段之和 $\ge 0$ 则算"好段"。求好段的最大数量。若 $[l, r]$ 内无关键日期则输出 0。
>
> $\sum n, \sum q \le 5 \times 10^5$，$|a_i| \le 10^9$。

**分析**：设 $[l, r]$ 内有 $m$ 个关键日期 $p_1 < p_2 < \cdots < p_m$，那么需要选 $m - 1$ 个"切割点"，把序列分成 $m$ 段，第 $i$ 段恰好包含 $p_i$。

切割点只能在相邻关键日期之间选（"间隙"$g_j$ 表示 $p_j$ 和 $p_{j+1}$ 之间的区域）。从左到右贪心：对每个间隙 $g_j$，我们有一组可选的前缀和值（在间隙中不同切割位置对应的前缀和），上一段的"右端"前缀和确定后，本段之和就确定了。排序后用二分选使当前段非负的最优切割点。

核心洞察：每个间隙可以描述为一个**分段转移函数** $T_j$：输入 $cur$（上一段右端的前缀和），输出 $(cnt, cur')$——$cnt$ 表示成功数（0 或 1），$cur'$ 表示选定的切割点前缀和。这个函数的"断点"就是间隙内各切割位置的升序前缀和。

函数复合满足结合律：先经过间隙 $j$、再经过 $j + 1$，等价于先把 $j$ 的输出代入 $j + 1$。用**猫树分治**（离线分治 + 预处理 L_data/R_data）完成区间函数复合，查询只需一次 eval。

```
数据结构：猫树分治，信息类型为分段转移函数
支持操作：区间转移函数复合 + 查询
关键性质：分段函数复合满足结合律，合并后段数 ≤ 左半段数
时间复杂度：预处理 O(n log^2 n)，查询 O(log n)
```

**C++ 代码**：
```cpp
struct Seg { int cnt; long long out; };
struct Func {
    vector<long long> breaks; // 升序断点
    vector<Seg> segs;         // segs.size() = breaks.size() + 1
};

// 构建单个间隙的转移函数
// sv: 间隙内各切割位置的前缀和（已升序）
Func make_func(const vector<long long>& sv) {
    Func f;
    f.breaks = sv;
    f.segs.resize(sv.size() + 1);
    // cur <= sv[0]: 本段和 = sv[0] - cur >= 0，成功，输出 sv[0]
    // ...
    // cur <= sv[k]: 用 sv[k] 切割，成功
    // cur > sv.back(): 无法使本段非负，选 sv[0] 最小化损失
    for (int i = 0; i < (int)sv.size(); i++)
        f.segs[i] = {1, sv[i]};
    f.segs[sv.size()] = {0, sv[0]};
    return f;
}

Seg eval(const Func& f, long long cur) {
    int idx = (int)(lower_bound(f.breaks.begin(), f.breaks.end(), cur)
                    - f.breaks.begin());
    return f.segs[idx];
}

Func compose_func(const Func& L, const Func& R) {
    Func res;
    res.breaks = L.breaks;
    res.segs.resize(L.segs.size());
    for (int k = 0; k < (int)L.segs.size(); k++) {
        Seg ls = L.segs[k];
        Seg rs = eval(R, ls.out);
        res.segs[k] = {ls.cnt + rs.cnt, rs.out};
    }
    return res;
}

// === 猫树分治部分 ===
// gaps[0..G-1]: 每个间隙的转移函数
// 查询 [ql, qr]（间隙编号范围）时，需要复合 gaps[ql..qr]
// 用分治预处理 L_data/R_data，查询时 O(1) 合并

vector<Func> L_data, R_data; // 每层的左后缀/右前缀复合
vector<vector<pair<int,int>>> block_queries; // 每个分治块上的查询

void solve(int bl, int br, vector<Func>& gaps) {
    if (bl > br) return;
    int mid = (bl + br) / 2;
    // 预处理从 mid 向左的后缀复合
    Func cur_f = gaps[mid];
    // L_data[mid] = gaps[mid] 自身
    L_data[mid] = cur_f;
    for (int i = mid - 1; i >= bl; i--) {
        cur_f = compose_func(gaps[i], cur_f);
        L_data[i] = cur_f;
    }
    // 预处理从 mid+1 向右的前缀复合
    if (mid + 1 <= br) {
        cur_f = gaps[mid + 1];
        R_data[mid + 1] = cur_f;
        for (int i = mid + 2; i <= br; i++) {
            cur_f = compose_func(cur_f, gaps[i]);
            R_data[i] = cur_f;
        }
    }
    // 处理跨越 mid 的查询（此处省略查询分发逻辑）
    // 对于查询 [ql, qr] 满足 ql <= mid <= qr:
    //   若 ql == qr == mid: 直接 eval(gaps[mid], init_cur)
    //   若 qr == mid: eval(L_data[ql], init_cur)
    //   若 ql == mid: eval(compose_func(gaps[mid], R_data[qr]), init_cur)（需合并）
    //   一般情况: eval(compose_func(L_data[ql], R_data[qr]), init_cur)
    solve(bl, mid - 1, gaps);
    solve(mid + 1, br, gaps);
}

// 主流程：
// 1. 提取关键日期位置 pos[]，计算间隙内的前缀和数组
// 2. 对每个间隙建 Func
// 3. 查询 [l, r] → 找到 [l, r] 内的关键日期范围 → 转化为间隙范围查询
// 4. 分治 + 合并得到答案
// 5. 特殊处理：最左段和最右段（不完整间隙）的贡献需单独计算
```

**核心启示**：
- **猫树不限于"数值"信息，可以维护"函数"信息**。只要函数复合满足结合律就行。
- **合并后段数 $\le$ 左半段数**——每次 compose 不增加断点，保证空间和时间不爆炸。
- **猫树分治是离线版的猫树**：不预存所有层的前缀/后缀，而是在分治过程中按需构建、处理查询，适合信息体积大（如 `vector` 类型的 Func）的场景。
- **"贪心 + 二分 + 分段函数 + 猫树分治"是这类区间贪心决策问题的通用框架**。

### 题目 10：猫树优化 DP（区间 DP 加速）

> 给定长为 $n$ 的数组，将其分成若干段，每段的代价为段内元素之和的平方。求最小总代价。但附加限制：某些分割点不允许使用。$n \le 10^5$。

**分析**：这是一道经典的区间 DP 优化题。朴素的 DP 转移为：

$$dp[i] = \min_{j < i,\; j \text{ 合法}} \{dp[j] + (S_i - S_j)^2\}$$

展开后 $(S_i - S_j)^2 = S_i^2 - 2S_i S_j + S_j^2$，可以用斜率优化。但如果转移点有特殊限制（如"不连续"），需要维护动态凸包。

另一种思路：如果把转移看作"从 $j$ 到 $i$ 的函数"$f_{j \to i}(dp[j]) = dp[j] + (S_i - S_j)^2$，那区间 DP 可以用分治 + 猫树加速：预处理区间内的转移函数，批量查询。

猫树在这类"DP 转移可以表示为函数复合"的场景中提供了高效的区间函数查询能力，将某些 $O(n^2)$ 的 DP 优化到 $O(n \log^2 n)$。

```
数据结构：猫树 + 分治优化
支持操作：区间转移函数批量查询
关键性质：DP 转移可分段表示为函数
时间复杂度：O(n log^2 n)
```

这类题的具体实现高度依赖问题结构，此处给出思路框架而非完整代码：

```cpp
// 思路框架：
// 1. 将 DP 转移 dp[i] = min(f_j(dp[j])) 中的 f_j 抽象为函数
// 2. 对合法转移点序列建猫树，预处理区间函数的合并
// 3. 分治处理：solve(l, r) 时，用猫树查询 [l, mid] 的函数对 [mid+1, r] 的贡献
// 4. 每次查询 O(1) 从猫树取出合并后的函数，再 O(1) 求值

// 关键：并非所有 DP 都能这样优化，需要转移函数满足结合律
// 典型适用：SMAWK 类、凸优化类、分段线性类
```

**核心启示**：
- **猫树 + 分治是优化某些区间 DP 的利器**，前提是转移可以表示为函数复合。
- **从"数据结构题"到"DP 优化"**，猫树的应用延伸到了算法设计层面。
- **高级应用中，猫树往往不是主角而是配角**——它负责高效的区间函数查询，主角是问题本身的结构。

## 五、常见技巧与思路总结

### 5.1 补齐长度与单位元

猫树要求序列长度是 2 的幂。补齐时填的值必须是**运算的单位元**：

| 运算 | 单位元 | 补齐值 |
|------|--------|--------|
| 加法 | 0 | 0 |
| 乘法 | 1 | 1 |
| 最小值 | $+\infty$ | `INT_MAX` |
| 最大值 | $-\infty$ | `INT_MIN` |
| GCD | 0 | 0 |
| 矩阵乘 | 单位矩阵 | $I$ |
| 仿射变换 | $(1, 0)$ | $f(x) = x$ |

### 5.2 注意 merge 的顺序

对于**不满足交换律**的运算（矩阵乘、函数复合），后缀和前缀的累积方向必须正确：

```cpp
// 后缀（从 mid 向左）：最终效果是 a[l] ⊕ a[l+1] ⊕ ... ⊕ a[mid]
suf[d][mid] = single[mid];
for (int i = mid - 1; i >= bl; i--)
    suf[d][i] = merge(single[i], suf[d][i + 1]);  // 新元素在左

// 前缀（从 mid+1 向右）：最终效果是 a[mid+1] ⊕ ... ⊕ a[r]
pre[d][mid + 1] = single[mid + 1];
for (int i = mid + 2; i <= bl + block - 1; i++)
    pre[d][i] = merge(pre[d][i - 1], single[i]);  // 新元素在右
```

### 5.3 空间优化：层编号复用

所有层共用同一个 $[1, N]$ 的下标空间。不同层的同一位置存的是不同块的信息，互不冲突，因此 `pre[d][i]` 和 `suf[d][i]` 的下标空间天然不重叠（不同层的块划分不同，但同一位置在每层恰好属于一个块）。

### 5.4 什么时候不该用猫树

- **需要修改**：猫树是静态结构，不支持修改。用线段树。
- **运算有逆运算**：前缀和更简单，$O(n)$ 预处理 $O(1)$ 查询。
- **运算满足幂等性**：ST 表同样 $O(n \log n)$ 预处理 $O(1)$ 查询，实现更简洁。
- **$n$ 很小**：猫树的 $O(n \log n)$ 空间和预处理可能不如暴力或线段树划算。

### 5.5 与"猫树分治"的区别

有些题目中提到"猫树分治"，指的是**利用猫树的分治结构来处理离线查询**——不一定预处理完整的前缀/后缀信息，而是在分治过程中逐层处理跨越中点的查询。这本质上是猫树结构的推广应用，核心思想一致。

## 七、终极总结与归纳

### 7.1 猫树的本质

**一句话概括：在线段树的分治骨架上，每个节点预处理从中点向两侧的前缀/后缀信息，查询时通过异或 $O(1)$ 定位层号，一次合并得到答案。**

猫树把线段树查询时"从上到下走 $\log n$ 步，每步合并一次"的模式，变成了"预处理时每层扫一遍 $O(n)$，查询时只合并一次"的模式。本质是**用 $O(\log n)$ 倍的预处理时间换取 $O(\log n)$ 倍的查询加速**。

### 7.2 核心要点

1. **适用条件**：静态序列，运算满足结合律，无需修改。
2. **复杂度**：$O(n \log n)$ 预处理，$O(1)$ 查询（一次合并运算）。
3. **空间**：$O(n \log n)$，每层存一整个序列的前缀/后缀。
4. **查询原理**：异或 + 对数表 → $O(1)$ 找到 LCA 层号 → 合并左后缀和右前缀。
5. **最大优势场景**：合并代价 $M$ 较高时（矩阵 $O(k^3)$、线性基 $O(\log^2 V)$），查询从 $O(M \log n)$ 降到 $O(M)$。
6. **不适用场景**：需要修改、运算有逆运算、运算满足幂等性——这些场景有更好的选择。
