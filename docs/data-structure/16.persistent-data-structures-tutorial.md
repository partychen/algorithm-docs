## 一、什么是可持久化

在普通数据结构中，每次修改操作都会**覆盖**原有数据，历史版本永久丢失。而**可持久化数据结构**的核心目标是：

> **保留所有历史版本**，使得我们能够在任意历史版本上进行查询甚至修改，同时保证时间和空间复杂度可控。

### 直觉理解

想象一个文档编辑器，每次保存都会生成一个新版本，你可以随时回到任意历史版本继续编辑。可持久化数据结构做的就是同样的事情——只不过对象是线段树、并查集、平衡树等。

### 关键手段：路径复制（Path Copying）

- 修改时**不**直接改原节点，而是将从根到修改位置的路径上的节点全部**复制一份**。
- 未被修改的子树与旧版本**共享**，因此每次修改仅新增 $O(\log n)$ 个节点。
- 每个版本拥有独立的根节点，通过根节点即可访问对应版本的完整数据。

```
版本 0 的根 ──► [A] ──► [B] ──► [C]（叶子）
                        │
版本 1 的根 ──► [A'] ──► [B'] ──► [C']（修改后的叶子）
                         │
                    共享未改变的子树
```

**核心特点：**

| 特性 | 说明 |
|------|------|
| 核心思想 | 路径复制（Path Copying） |
| 时间开销 | 每次修改 $O(\log n)$ |
| 空间开销 | 每次修改新增 $O(\log n)$ 个节点 |
| 版本管理 | 通过根节点数组 `root[ver]` 访问任意版本 |
| 前提条件 | 底层结构为树形（线段树、Trie、平衡树等） |

## 二、核心思想与前置知识

### 2.1 路径复制（Path Copying）

路径复制是所有可持久化数据结构的基石。其思路为：

1. 修改某个节点时，将该节点**复制**一份，在副本上进行修改。
2. 该节点的父节点也需要复制，因为它的子指针需要指向新副本。
3. 沿路径向上递归，直到根节点。
4. 新根即代表新版本。

**时间复杂度**：每次修改 $O(\log n)$（树高）

**空间复杂度**：每次修改 $O(\log n)$ 个新节点

### 2.2 可持久化线段树（主席树）

可持久化线段树是最经典的可持久化结构，由主席（HJT）推广，因此又称**主席树**。

核心特征：
- 动态开点线段树 + 路径复制
- 每次修改仅新建 $O(\log n)$ 个节点
- 通过版本根数组 `root[ver]` 访问任意版本

```cpp
// 可持久化线段树模板
const int MAXN = 200005, MAXM = MAXN * 40;
int ls[MAXM], rs[MAXM], val[MAXM], cnt;
int root[MAXN];

int build(int l, int r) {
    int p = ++cnt;
    val[p] = 0;
    if (l == r) return p;
    int mid = (l + r) >> 1;
    ls[p] = build(l, mid);
    rs[p] = build(mid + 1, r);
    return p;
}

int update(int pre, int l, int r, int pos, int v) {
    int p = ++cnt;
    ls[p] = ls[pre]; rs[p] = rs[pre]; val[p] = val[pre];
    if (l == r) { val[p] += v; return p; }
    int mid = (l + r) >> 1;
    if (pos <= mid) ls[p] = update(ls[pre], l, mid, pos, v);
    else rs[p] = update(rs[pre], mid + 1, r, pos, v);
    val[p] = val[ls[p]] + val[rs[p]];
    return p;
}

int query(int u, int v, int l, int r, int k) {
    if (l == r) return l;
    int mid = (l + r) >> 1;
    int d = val[ls[v]] - val[ls[u]];
    if (k <= d) return query(ls[u], ls[v], l, mid, k);
    else return query(rs[u], rs[v], mid + 1, r, k - d);
}
```

### 2.3 可持久化并查集

用**可持久化数组**（即可持久化线段树维护的数组）存储 `fa[]` 和 `rank[]`。

- 每次 `find` / `union` 操作在对应版本的数组上进行。
- 不能使用路径压缩（会破坏历史版本），改用**按秩合并**保证 $O(\log n)$。

### 2.4 可持久化 Trie

可持久化 01-Trie 与主席树思路一脉相承：

- 逐个插入值，每次产生一个新版本的 Trie。
- 利用版本差实现"区间限定"查询。
- 典型应用：求区间内与某个值异或最大的结果。

### 2.5 可持久化平衡树

通常采用**可持久化 FHQ Treap**，因为 FHQ Treap 的 split/merge 操作天然适合路径复制。每次经过节点时复制再修改即可。

### 2.6 部分可持久化 vs 完全可持久化

- **部分可持久化（Partially Persistent）**：只能查询历史版本，新修改只基于最新版本。
- **完全可持久化（Fully Persistent）**：可以在任意历史版本上修改并产生新版本（形成版本树）。

本文讨论的均为**完全可持久化**。

## 三、分类与变体详解

### 3.1 可持久化线段树（主席树）

最经典的可持久化结构。动态开点线段树 + 路径复制，每次修改 $O(\log n)$ 时间空间。典型应用：区间第 $k$ 小、历史版本点查询/区间查询。

### 3.2 可持久化数组

用可持久化线段树维护一维数组的随机访问和单点修改。它是可持久化并查集的基础。

### 3.3 可持久化并查集

可持久化数组 + 按秩合并（禁止路径压缩）。支持版本回退的连通性查询。

### 3.4 可持久化 01-Trie

字典树 + 路径复制，支持"区间内与给定值异或最大/最小"的查询。与主席树同源，利用版本差实现区间限定。

### 3.5 可持久化平衡树

FHQ Treap + 路径复制。支持在任意历史版本上执行插入、删除、排名查询等有序集合操作。

### 3.6 可持久化文艺平衡树

在可持久化 FHQ Treap 上增加懒标记（如翻转标记）。难点：pushDown 时必须**先复制子节点再下传**，否则污染历史版本。

**分类速查表：**

| 类别 | 底层结构 | 典型应用 | 空间复杂度 |
|------|---------|---------|-----------|
| 可持久化线段树 | 动态开点线段树 | 区间第 $k$ 小、历史版本查询 | $O(n \log n)$ |
| 可持久化数组 | 可持久化线段树 | 并查集底层、历史版本随机访问 | $O(n \log n)$ |
| 可持久化并查集 | 可持久化数组 + 按秩合并 | 动态连通性回退 | $O(n \log n)$ |
| 可持久化平衡树 | FHQ Treap + 路径复制 | 历史版本有序集合操作 | $O(n \log n)$ |
| 可持久化 Trie | 字典树 + 路径复制 | 异或极值、历史版本字符串 | $O(n \log V)$ |
| 可持久化文艺平衡树 | FHQ Treap + 懒标记 | 区间翻转的历史版本 | $O(n \log n)$ |

### 题目 1：可持久化数组（洛谷 P3919）

> 给定长度为 $n$ 的数组，支持两种操作：(1) 在某个历史版本上修改某个位置的值，生成新版本；(2) 查询某个历史版本上某个位置的值。

**分析**：

```
数据结构：可持久化线段树维护数组
支持操作：单点修改（路径复制）、单点查询
关键性质：每次修改新建 O(log n) 个节点，其余与旧版本共享
时间复杂度：O((n + m) log n)
```

**C++ 代码**：

```cpp
const int MAXN = 1000006, MAXM = MAXN * 25;
int ls[MAXM], rs[MAXM], val[MAXM], cnt;
int root[MAXN];

int build(int l, int r, int a[]) {
    int p = ++cnt;
    if (l == r) { val[p] = a[l]; return p; }
    int mid = (l + r) >> 1;
    ls[p] = build(l, mid, a);
    rs[p] = build(mid + 1, r, a);
    return p;
}

int update(int pre, int l, int r, int pos, int v) {
    int p = ++cnt;
    ls[p] = ls[pre]; rs[p] = rs[pre]; val[p] = val[pre];
    if (l == r) { val[p] = v; return p; }
    int mid = (l + r) >> 1;
    if (pos <= mid) ls[p] = update(ls[pre], l, mid, pos, v);
    else rs[p] = update(rs[pre], mid + 1, r, pos, v);
    return p;
}

int query(int p, int l, int r, int pos) {
    if (l == r) return val[p];
    int mid = (l + r) >> 1;
    if (pos <= mid) return query(ls[p], l, mid, pos);
    else return query(rs[p], mid + 1, r, pos);
}
```

**核心启示**：
- 可持久化数组是后续所有可持久化结构的基础
- 理解路径复制在线段树上的应用是关键第一步
- 版本管理通过 `root[]` 数组实现，`root[newVer] = root[oldVer]` 即可回退

### 题目 2：主席树模板——区间第 k 小（洛谷 P3834）

> 给定长度为 $n$ 的序列，$m$ 次询问，每次查询区间 $[l, r]$ 中第 $k$ 小的值。

**分析**：

```
数据结构：可持久化权值线段树（主席树）
支持操作：逐个插入元素产生新版本，版本差实现区间查询
关键性质：前缀和思想——版本 r 减去版本 l-1 即为区间 [l, r] 的贡献
时间复杂度：O((n + m) log n)
```

**C++ 代码**：

```cpp
// 离散化后对值域 [1, tot] 建权值线段树
// 依次插入 a[1..n]，root[i] 基于 root[i-1]

int update(int pre, int l, int r, int pos) {
    int p = ++cnt;
    ls[p] = ls[pre]; rs[p] = rs[pre]; val[p] = val[pre] + 1;
    if (l == r) return p;
    int mid = (l + r) >> 1;
    if (pos <= mid) ls[p] = update(ls[pre], l, mid, pos);
    else rs[p] = update(rs[pre], mid + 1, r, pos);
    return p;
}

int query(int u, int v, int l, int r, int k) {
    if (l == r) return l;
    int mid = (l + r) >> 1;
    int d = val[ls[v]] - val[ls[u]];
    if (k <= d) return query(ls[u], ls[v], l, mid, k);
    else return query(rs[u], rs[v], mid + 1, r, k - d);
}
// 调用：query(root[l-1], root[r], 1, tot, k)
```

**核心启示**：
- 主席树的精髓在于"前缀和思想"——版本 $r$ 减去版本 $l-1$ 就是区间 $[l, r]$ 的贡献
- 必须先对值域离散化，否则线段树无法建在值域上
- 这是可持久化线段树最经典、最高频的应用

### 题目 3：版本回退——基于版本操作（CF707D）

> 维护一个集合，支持三种操作：(1) 插入元素；(2) 删除元素；(3) 查询元素是否存在。每次操作可以基于任意历史版本进行。

**分析**：

```
数据结构：可持久化线段树（可持久化数组/01标记）
支持操作：单点赋值（插入/删除对应设0/1）、单点查询
关键性质：版本树结构——每个操作指定基于哪个版本，形成树形版本关系
时间复杂度：O(m log n)
```

**C++ 代码**：

```cpp
// 将元素映射到 [1, MAXVAL] 的位置
// 插入元素 x：在版本 ver 上将位置 x 设为 1
int insert(int pre, int l, int r, int pos) {
    int p = ++cnt;
    ls[p] = ls[pre]; rs[p] = rs[pre]; val[p] = val[pre];
    if (l == r) { val[p] = 1; return p; }
    int mid = (l + r) >> 1;
    if (pos <= mid) ls[p] = insert(ls[pre], l, mid, pos);
    else rs[p] = insert(rs[pre], mid + 1, r, pos);
    return p;
}

// 删除元素 x：在版本 ver 上将位置 x 设为 0
int erase(int pre, int l, int r, int pos) {
    int p = ++cnt;
    ls[p] = ls[pre]; rs[p] = rs[pre]; val[p] = val[pre];
    if (l == r) { val[p] = 0; return p; }
    int mid = (l + r) >> 1;
    if (pos <= mid) ls[p] = erase(ls[pre], l, mid, pos);
    else rs[p] = erase(rs[pre], mid + 1, r, pos);
    return p;
}
```

**核心启示**：
- "回到历史版本"只需令 `root[newVer] = root[oldVer]`，零代价
- 版本形成树形结构时，理解"当前操作基于哪个版本"是关键
- 可持久化天然支持"撤销"操作

### 题目 4：可持久化线段树区间求和（洛谷 P3567 / POI2014 KUR）

> 给定序列和多个历史版本，每个版本经过若干次单点修改。查询某版本中区间 $[l, r]$ 的和。

**分析**：

```
数据结构：可持久化线段树（维护区间和）
支持操作：单点修改产生新版本、区间求和查询
关键性质：pushup 维护子树和，路径复制保留所有历史数据
时间复杂度：O((n + m) log n)
```

**C++ 代码**：

```cpp
long long sum[MAXM];

int update(int pre, int l, int r, int pos, int v) {
    int p = ++cnt;
    ls[p] = ls[pre]; rs[p] = rs[pre]; sum[p] = sum[pre] + v;
    if (l == r) return p;
    int mid = (l + r) >> 1;
    if (pos <= mid) ls[p] = update(ls[pre], l, mid, pos, v);
    else rs[p] = update(rs[pre], mid + 1, r, pos, v);
    sum[p] = sum[ls[p]] + sum[rs[p]];
    return p;
}

long long query(int p, int l, int r, int ql, int qr) {
    if (ql <= l && r <= qr) return sum[p];
    int mid = (l + r) >> 1;
    long long res = 0;
    if (ql <= mid) res += query(ls[p], l, mid, ql, qr);
    if (qr > mid) res += query(rs[p], mid + 1, r, ql, qr);
    return res;
}
```

**核心启示**：
- 可持久化线段树不限于权值线段树，也可以维护区间和等信息
- pushup 的写法和普通线段树完全一致，区别仅在于 update 中做路径复制
- 空间需要预估：节点总数 $\le n \log n + m \log n$

### 题目 5：可持久化并查集（洛谷 P3402）

> 给定 $n$ 个元素，支持三种操作：(1) 合并两个元素所在集合并生成新版本；(2) 回到某个历史版本；(3) 查询两个元素是否在同一集合。

**分析**：

```
数据结构：可持久化线段树维护 fa[] 和 rank[] 两个数组
支持操作：find（沿 fa 跳到根）、union（按秩合并 + 路径复制产生新版本）
关键性质：不能路径压缩！路径压缩会修改多个节点破坏历史版本。按秩合并保证树高 O(log n)
时间复杂度：O(m log^2 n)
```

**C++ 代码**：

```cpp
int findFa(int rt, int l, int r, int pos) {
    if (l == r) return val[rt];
    int mid = (l + r) >> 1;
    if (pos <= mid) return findFa(ls[rt], l, mid, pos);
    return findFa(rs[rt], mid + 1, r, pos);
}

int findRoot(int rt, int x) {
    int fa = findFa(rt, 1, n, x);
    return fa == x ? x : findRoot(rt, fa);
}

int modify(int pre, int l, int r, int pos, int v) {
    int p = ++cnt;
    ls[p] = ls[pre]; rs[p] = rs[pre]; val[p] = val[pre];
    if (l == r) { val[p] = v; return p; }
    int mid = (l + r) >> 1;
    if (pos <= mid) ls[p] = modify(ls[pre], l, mid, pos, v);
    else rs[p] = modify(rs[pre], mid + 1, r, pos, v);
    return p;
}
// 合并时：修改 fa[ry] = rx，若秩相等则 rank[rx]++
```

**核心启示**：
- 可持久化并查集展示了"可持久化数组"的威力——任何基于数组的数据结构都可以通过替换底层数组来实现可持久化
- 按秩合并是唯一合法的优化，路径压缩绝对禁止
- 时间 $O(m \log^2 n)$，一个 $\log$ 来自树高，一个来自线段树查询

### 题目 6：可持久化 Trie——最大异或和（洛谷 P4735）

> 给定初始序列，支持两种操作：(1) 在序列末尾添加一个数；(2) 查询从第 $l$ 个到第 $r$ 个前缀异或值中，选一个与给定值 $x$ 异或后最大的结果。

**分析**：

```
数据结构：可持久化 01-Trie
支持操作：末尾插入新前缀异或值产生新版本、版本差查询区间内最大异或
关键性质：前缀异或和 pre[i] = a[1]^...^a[i]，区间 [l,r] 的异或和 = pre[r]^pre[l-1]
时间复杂度：O((n + m) log V)，V 为值域
```

**C++ 代码**：

```cpp
int ch[MAXM][2], sz[MAXM], cnt;
int root[MAXN];

int insert(int pre, int val, int dep) {
    int p = ++cnt;
    ch[p][0] = ch[pre][0]; ch[p][1] = ch[pre][1];
    sz[p] = sz[pre] + 1;
    if (dep < 0) return p;
    int b = (val >> dep) & 1;
    ch[p][b] = insert(ch[pre][b], val, dep - 1);
    return p;
}

int query(int u, int v, int val, int dep) {
    if (dep < 0) return 0;
    int b = (val >> dep) & 1, want = b ^ 1;
    if (sz[ch[v][want]] - sz[ch[u][want]] > 0)
        return (1 << dep) | query(ch[u][want], ch[v][want], val, dep - 1);
    return query(ch[u][b], ch[v][b], val, dep - 1);
}
```

**核心启示**：
- 可持久化 Trie 与主席树同源——利用版本差实现"区间限定"查询
- 贪心策略：从高位到低位，优先走与目标位相反的分支
- 是区间异或极值问题的标准解法

### 题目 7：可持久化平衡树（洛谷 P3835）

> 在某个历史版本的基础上执行插入、删除、查排名、查第 $k$ 大、查前驱、查后继操作，每次操作生成新版本。

**分析**：

```
数据结构：可持久化 FHQ Treap
支持操作：split/merge 时对经过的节点进行复制，六种操作均由 split + merge 组合实现
关键性质：FHQ Treap 的 split/merge 天然适合路径复制，每次经过 O(log n) 个节点
时间复杂度：O(m log n)
```

**C++ 代码**：

```cpp
struct Node {
    int ls, rs, val, pri, sz;
} tr[MAXM];
int cnt;

int copyNode(int o) {
    int p = ++cnt;
    tr[p] = tr[o];
    return p;
}

void split(int p, int v, int &x, int &y) {
    if (!p) { x = y = 0; return; }
    if (tr[p].val <= v) {
        x = copyNode(p);
        split(tr[x].rs, v, tr[x].rs, y);
        pushUp(x);
    } else {
        y = copyNode(p);
        split(tr[y].ls, v, x, tr[y].ls);
        pushUp(y);
    }
}

int merge(int x, int y) {
    if (!x || !y) return x | y;
    if (tr[x].pri > tr[y].pri) {
        int p = copyNode(x);
        tr[p].rs = merge(tr[p].rs, y);
        pushUp(p); return p;
    } else {
        int p = copyNode(y);
        tr[p].ls = merge(x, tr[p].ls);
        pushUp(p); return p;
    }
}
```

**核心启示**：
- FHQ Treap 天然适合可持久化，因为所有操作仅依赖 split 和 merge
- 每次操作复制 $O(\log n)$ 个节点，空间 $O(m \log n)$
- 六种操作的实现与普通 FHQ Treap 完全一致，区别仅在于 copyNode

### 题目 8：带修区间第 k 小——树状数组套主席树（洛谷 P2617）

> 给定长度为 $n$ 的序列，支持两种操作：(1) 单点修改 $a[p] = v$；(2) 查询区间 $[l, r]$ 中第 $k$ 小的值。

**分析**：

```
数据结构：树状数组套可持久化权值线段树
支持操作：树状数组管理修改的差分关系，每个位置挂动态开点权值线段树
关键性质：查询时同时操作 O(log n) 棵树的根在值域上二分下降
时间复杂度：O(m log^2 n)
```

**C++ 代码**：

```cpp
// 树状数组的每个位置维护一棵动态开点权值线段树
int bit_root[MAXN];
int add_stk[25], add_top;
int sub_stk[25], sub_top;

void bit_update(int x, int pos, int v, int n) {
    for (; x <= n; x += x & -x)
        bit_root[x] = update(bit_root[x], 1, tot, pos, v);
}

int bit_query(int l, int r, int k) {
    add_top = sub_top = 0;
    for (int x = r; x; x -= x & -x) add_stk[++add_top] = bit_root[x];
    for (int x = l - 1; x; x -= x & -x) sub_stk[++sub_top] = bit_root[x];
    // 同时在这些根上做值域二分
    int lo = 1, hi = tot;
    while (lo < hi) {
        int mid = (lo + hi) >> 1, cnt = 0;
        for (int i = 1; i <= add_top; i++) cnt += val[ls[add_stk[i]]];
        for (int i = 1; i <= sub_top; i++) cnt -= val[ls[sub_stk[i]]];
        if (k <= cnt) {
            for (int i = 1; i <= add_top; i++) add_stk[i] = ls[add_stk[i]];
            for (int i = 1; i <= sub_top; i++) sub_stk[i] = ls[sub_stk[i]];
            hi = mid;
        } else {
            k -= cnt;
            for (int i = 1; i <= add_top; i++) add_stk[i] = rs[add_stk[i]];
            for (int i = 1; i <= sub_top; i++) sub_stk[i] = rs[sub_stk[i]];
            lo = mid + 1;
        }
    }
    return lo;
}
```

**核心启示**：
- 树状数组套主席树是"带修区间第 $k$ 小"的标准在线解法
- 修改时沿树状数组路径上的 $O(\log n)$ 棵树分别修改
- 查询时将 $O(\log n)$ 棵树的根同步在值域上二分

### 题目 9：异或粽子（洛谷 P5283）

> 给定长度为 $n$ 的序列，求所有 $\frac{n(n+1)}{2}$ 个区间异或和中，前 $k$ 大的值之和。

**分析**：

```
数据结构：可持久化 01-Trie + 最大堆
支持操作：前缀异或和转化为两数异或，Trie 上贪心找最大异或配对
关键性质：每对 (i,j) 被计算两次，堆大小设为 2k。各 pre[i] 的第 1/2/3/... 大配对依次入堆
时间复杂度：O((n + k) log V)
```

**C++ 代码**：

```cpp
// 对 pre[0..n] 建可持久化 01-Trie
// 对每个 pre[i]，在 Trie 上贪心找与其异或第 t 大的值
long long query_kth_xor(int u, int v, int val, int dep, int k) {
    // 在版本 [u, v] 的 Trie 上找与 val 异或第 k 大
    if (dep < 0) return 0;
    int b = (val >> dep) & 1, want = b ^ 1;
    int cnt_want = sz[ch[v][want]] - sz[ch[u][want]];
    if (k <= cnt_want)
        return (1LL << dep) |
               query_kth_xor(ch[u][want], ch[v][want], val, dep-1, k);
    return query_kth_xor(ch[u][b], ch[v][b], val, dep-1, k - cnt_want);
}

// 堆中存 (异或值, i, 当前是第几大配对)
// 每次弹出后，找该 i 的下一大配对入堆
```

**核心启示**：
- 可持久化 Trie 配合堆贪心可以高效求解 Top-K 异或问题
- 注意每对 $(i, j)$ 被计算两次（$i < j$ 和 $j < i$），答案除以 2
- 将 "找前 k 大" 转化为 "每人先贡献最大值入堆，弹出后放入次大值"

### 题目 10：可持久化文艺平衡树（洛谷 P5055）

> 在某个历史版本的基础上，支持区间插入一段数、区间翻转、区间求和操作，每次操作产生新版本。

**分析**：

```
数据结构：可持久化 FHQ Treap + 翻转懒标记
支持操作：区间翻转、区间插入、区间求和，每次操作产生新版本
关键性质：pushDown 时必须先复制子节点再翻转，否则旧版本被破坏
时间复杂度：O(m log n)，常数较大
```

**C++ 代码**：

```cpp
void pushDown(int p) {
    if (!p || !tr[p].rev) return;
    if (tr[p].ls) { tr[p].ls = copyNode(tr[p].ls); tr[tr[p].ls].rev ^= 1; }
    if (tr[p].rs) { tr[p].rs = copyNode(tr[p].rs); tr[tr[p].rs].rev ^= 1; }
    swap(tr[p].ls, tr[p].rs);
    tr[p].rev = 0;
}

// split 和 merge 与可持久化 FHQ Treap 相同
// 只是在 split/merge 入口处先 pushDown

// 区间翻转: split 出 [l, r] 对应的子树，打翻转标记
void reverse(int &rt, int l, int r) {
    int x, y, z;
    splitByRank(rt, r, x, z);
    splitByRank(x, l - 1, x, y);
    tr[y = copyNode(y)].rev ^= 1;
    rt = merge(merge(x, y), z);
}
```

**核心启示**：
- "可持久化 + 懒标记"的组合中，pushDown 先复制是铁律
- 忘记复制是最常见的 bug，会导致旧版本被静默修改
- 文艺平衡树的可持久化版本常数较大，需要注意空间预分配

### 题目 11：区间第 k 小（带树上路径）——树上主席树（洛谷 P2633）

> 给定一棵 $n$ 个节点的带权树，$m$ 次询问，每次求树上路径 $(u, v)$ 上第 $k$ 小的点权。

**分析**：

```
数据结构：树上可持久化权值线段树
支持操作：DFS 建树——每个节点的版本基于其父节点的版本
关键性质：路径 (u,v) 的信息 = root[u] + root[v] - root[lca] - root[fa[lca]]
时间复杂度：O((n + m) log n)
```

**C++ 代码**：

```cpp
// DFS 中：root[u] = update(root[fa[u]], 1, tot, val_rank[u])
// 查询 (u, v) 路径第 k 小：

int query(int a, int b, int c, int d, int l, int r, int k) {
    // a=root[u], b=root[v], c=root[lca], d=root[fa[lca]]
    if (l == r) return l;
    int mid = (l + r) >> 1;
    int cnt = val[ls[a]] + val[ls[b]] - val[ls[c]] - val[ls[d]];
    if (k <= cnt)
        return query(ls[a], ls[b], ls[c], ls[d], l, mid, k);
    else
        return query(rs[a], rs[b], rs[c], rs[d], mid + 1, r, k - cnt);
}
```

**核心启示**：
- 主席树的"前缀和"思想可以推广到树上——DFS 序中每个节点的版本基于父节点
- 路径 $(u,v)$ 的信息通过容斥原理拆解为四个版本的组合
- 离线 LCA 预处理 + 在线查询是标准组合

```
数据结构：可持久化并查集（可持久化数组 + 按秩合并）
支持操作：边按权值排序逐条加入，每次合并后产生新版本，查询对应版本的连通块数
关键性质：二分找到权值 <= w 的最后一条边对应的版本，该版本的连通块数即为答案
```

```cpp
// 按权值排序边，逐条合并
// edges 排序后，version[i] 表示前 i 条边合并完成后的版本

// 查询时先二分找到 <= w 的最后一条边的版本号 ver
// 答案 = n - ver 次有效合并的数量

int comp[MAXN]; // comp[ver] = 当前版本的连通块数
// 合并 (u, v)：
int ru = findRoot(root_fa[ver], u);
int rv = findRoot(root_fa[ver], v);
```

```cpp
// 连边 (u, v) 时，将较小的连通块重新 DFS

void rebuild(int u, int fa) {

// link(u, v)：
int su = sz_comp[find(u)], sv = sz_comp[find(v)];
```

### 题目 14：可持久化 Trie 上的第 k 大异或（CF840D）

> 给定一棵 $n$ 个节点的带权树，每次查询路径 $(u, v)$ 上所有点权与给定值 $x$ 异或后，第 $k$ 大的结果。

**分析**：

```
数据结构：树上可持久化 01-Trie
支持操作：DFS 建树——每个节点版本基于父节点版本，路径通过容斥原理拆解
关键性质：类似树上主席树的容斥——cnt = sz[a] + sz[b] - sz[c] - sz[d]
时间复杂度：O((n + m) log V)
```

**C++ 代码**：

```cpp
// DFS 中：trie_root[u] = trie_insert(trie_root[fa[u]], val[u], LOG)
// 查询路径 (u, v) 上与 x 异或第 k 大：

int query(int a, int b, int c, int d, int x, int dep, int k) {
    if (dep < 0) return 0;
    int bit = (x >> dep) & 1, want = bit ^ 1;
    int cnt_want = sz[ch[a][want]] + sz[ch[b][want]]
                 - sz[ch[c][want]] - sz[ch[d][want]];
    if (k <= cnt_want)
        return (1 << dep) | query(ch[a][want], ch[b][want],
               ch[c][want], ch[d][want], x, dep - 1, k);
    return query(ch[a][bit], ch[b][bit],
           ch[c][bit], ch[d][bit], x, dep - 1, k - cnt_want);
}
```

**核心启示**：
- 可持久化 Trie 在树上的应用与主席树完全对称——都用 DFS 序 + 容斥
- "第 k 大异或值"通过从高位到低位贪心 + 计数实现
- 这类问题同时考察 Trie、可持久化和树上路径处理三项技能

### 题目 15：线段树合并 + 可持久化（CF600E）

> 给定一棵 $n$ 个节点的树，每个节点有一个颜色。对每个节点，求其子树中出现次数最多的颜色的编号之和（可能有多个）。

**分析**：

```
数据结构：动态开点权值线段树 + 线段树合并
支持操作：自底向上合并子节点的线段树，维护出现次数最大值及对应颜色编号和
关键性质：线段树合并的总节点数 = O(n log n)，每个节点只会参与一次合并
时间复杂度：O(n log n)
```

**C++ 代码**：

```cpp
int mx[MAXM]; long long ans_sum[MAXM];

int merge_tree(int a, int b, int l, int r) {
    if (!a || !b) return a | b;
    if (l == r) {
        val[a] += val[b];
        mx[a] = val[a]; ans_sum[a] = l;
        return a;
    }
    int mid = (l + r) >> 1;
    ls[a] = merge_tree(ls[a], ls[b], l, mid);
    rs[a] = merge_tree(rs[a], rs[b], mid + 1, r);
    // pushup: 维护出现次数最大值和颜色编号和
    if (mx[ls[a]] > mx[rs[a]])
        mx[a] = mx[ls[a]], ans_sum[a] = ans_sum[ls[a]];
    else if (mx[ls[a]] < mx[rs[a]])
        mx[a] = mx[rs[a]], ans_sum[a] = ans_sum[rs[a]];
    else
        mx[a] = mx[ls[a]], ans_sum[a] = ans_sum[ls[a]] + ans_sum[rs[a]];
    return a;
}
```

**核心启示**：
- 线段树合并是"子树统计"类问题的利器，本质是自底向上的信息合并
- 合并过程中只有路径上的节点被访问，非路径节点共享，总复杂度 $O(n \log n)$
- 虽然不是严格意义上的"可持久化"，但共享节点的理念与路径复制同源

### 题目 16：可持久化可并堆——最短路 K 优解（洛谷 P2483）

> 给定有向图，求从 $s$ 到 $t$ 的前 $k$ 短路长度。

**分析**：

```
数据结构：可持久化左偏树（可并堆）
支持操作：反向建最短路树，每个节点的"可选弯路"构成可并堆。合并堆 + A* 搜索求前 k 短路
关键性质：可持久化使每次堆merge不破坏原版本，支持多路同时搜索
时间复杂度：O((n + m) log n + k log k)
```

**C++ 代码**：

```cpp
// 可持久化左偏树核心
struct LeftistNode {
    int ls, rs, dist;
    long long val;
} heap[MAXM];
int heap_cnt;

int heap_merge(int a, int b) {
    if (!a || !b) return a | b;
    if (heap[a].val > heap[b].val) swap(a, b);
    int p = ++heap_cnt;
    heap[p] = heap[a]; // 路径复制
    heap[p].rs = heap_merge(heap[p].rs, b);
    if (heap[heap[p].ls].dist < heap[heap[p].rs].dist)
        swap(heap[p].ls, heap[p].rs);
    heap[p].dist = heap[heap[p].rs].dist + 1;
    return p;
}

// 建最短路树后，沿 DFS 序合并每个节点的非树边偏移量
// 使用最大堆 + 可持久化堆展开求前 k 短路
```

**核心启示**：
- 可持久化可并堆（左偏树）是求 K 短路的经典工具
- 路径复制保证每次 merge 不破坏已有版本
- 这是可持久化思想在堆结构上的经典延伸——函数式编程的体现

## 五、实用技巧与注意事项

### 5.1 内存预分配

可持久化结构最大的开销是空间。每次修改新增 $O(\log n)$ 个节点，所以要预估节点总数：

$$\text{总节点数} \leq n \cdot \log n + m \cdot \log n$$

一般开 $40 \sim 50$ 倍的数组即可。

### 5.2 绝对不要路径压缩

在可持久化并查集中，路径压缩会修改历史版本中的节点，导致所有共享该节点的版本被破坏。**必须使用按秩合并**。

### 5.3 懒标记 + 可持久化 = 先复制再下传

```
普通懒标记：pushDown 直接修改子节点
可持久化懒标记：pushDown 先复制子节点，再在副本上修改
```

遗忘这一步是可持久化文艺平衡树最常见的 bug。

### 5.4 离散化

主席树求区间第 $k$ 小时，必须先对值域**离散化**，否则线段树无法建在值域上。

### 5.5 版本号管理

- 版本从 0 开始（初始版本）
- 每次操作生成的新版本号需要递增
- "回到历史版本"只需令 `root[newVer] = root[oldVer]`

## 六、解题框架

遇到可持久化相关问题时，可按以下框架思考：

```
步骤 1：识别问题是否需要可持久化
  - 是否需要回退到历史版本？
  - 是否需要在不同版本上分别操作？
  - 是否能用"前缀和"思想将区间问题转化为版本差？

步骤 2：选择底层数据结构
  - 单点修改 + 单点/区间查询 → 可持久化线段树
  - 连通性查询 + 版本回退 → 可持久化并查集
  - 有序集合操作 + 版本回退 → 可持久化平衡树
  - 异或相关 + 区间限定 → 可持久化 Trie

步骤 3：实现路径复制
  - 修改操作中：复制路径上的节点
  - 有懒标记：pushDown 时先复制子节点

步骤 4：版本管理
  - root[] 数组记录每个版本的根
  - 确定新版本基于哪个旧版本

步骤 5：空间估算
  - 节点总数 = 初始节点 + 操作次数 × 每次新增节点数
  - 一般开 40~50 倍
```

## 七、终极总结与归纳

### 7.1 可持久化的本质

**可持久化 = 路径复制 + 版本管理。** 每次修改只复制从根到修改点的路径，其余部分与旧版本共享。这保证了每次操作 $O(\log n)$ 的时间和空间开销。它是函数式编程思想在数据结构中的核心体现——"不可变数据 + 共享结构"。

### 7.2 核心要点

1. **路径复制**：修改时复制根到修改点的路径，未改部分与旧版本共享
2. **版本管理**：`root[ver]` 数组记录每个版本的根，回退只需赋值
3. **前缀和思想**：主席树/可持久化 Trie 利用版本差实现区间限定查询
4. **禁止路径压缩**：可持久化并查集必须按秩合并，否则破坏历史版本
5. **懒标记先复制**：pushDown 时必须先 copyNode 再下传，否则污染旧版本
6. **空间预估**：节点总数 = $O((n + m) \log n)$，一般开 40~50 倍数组

### 7.5 口诀

> **可持久化心法口诀**
>
> 路径复制留历史，版本根数组来记。
> 主席前缀差一减，区间第 k 小无敌。
> 并查集禁路径压，按秩合并是唯一。
> Trie 树异或贪心走，高位优先取反位。
> 文艺标记先复制，pushDown 忘了全白费。
> 空间预估四五十，函数式思想贯到底。
