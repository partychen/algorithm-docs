## 一、什么是栈

栈（Stack）是一种**后进先出**（LIFO, Last In First Out）的线性数据结构。你可以把它想象成一摞盘子——只能从最顶端放入或取出。

**核心操作：**

| 操作 | 说明 | 时间复杂度 |
|------|------|-----------|
| push(x) | 将元素放到栈顶 | $O(1)$ |
| pop() | 将栈顶元素移除 | $O(1)$ |
| top() | 查看栈顶元素不移除 | $O(1)$ |
| empty() | 检查栈是否为空 | $O(1)$ |

```
操作示意：

push(1)  push(2)  push(3)  pop()   top()
  |   |   |   |   | 3 |   |   |   |   |
  |   |   | 2 |   | 2 |   | 2 |   | 2 |  → 返回 2
  | 1 |   | 1 |   | 1 |   | 1 |   | 1 |
  +---+   +---+   +---+   +---+   +---+
```

栈的核心价值在于它天然适合处理**"最近相关性"**问题——最近的左括号、最近的更大元素、最近的函数调用帧。

## 二、核心思想与前置知识

### 2.1 基本栈操作

竞赛中通常使用**数组模拟栈**，比 STL `stack` 更快且支持随机访问：

```cpp
int stk[MAXN], top = 0;

void push(int x) { stk[++top] = x; }
void pop() { top--; }
int peek() { return stk[top]; }
bool empty() { return top == 0; }
// 优势：可用 stk[i] 随机访问栈内任意位置
```

### 2.2 单调栈

单调栈是一种特殊的栈，栈内元素始终保持**单调递增**或**单调递减**的顺序。

**核心用途：** 在 $O(n)$ 时间内找到每个元素左边/右边**第一个**比它大（或小）的元素。

```
数组: [2, 1, 4, 3, 5]（单调递增栈，找右边第一个更大元素）

i=0: 栈空，push(0)            栈: [2]
i=1: 1 < 2，push(1)           栈: [2, 1]
i=2: 4 > 1，弹出1，ans[1]=4
     4 > 2，弹出2，ans[0]=4
     push(2)                   栈: [4]
i=3: 3 < 4，push(3)           栈: [4, 3]
i=4: 5 > 3，弹出3，ans[3]=5
     5 > 4，弹出4，ans[2]=5
     push(4)                   栈: [5]

结果: ans = [4, 4, 5, 5, -1]
```

**模板代码：**

```cpp
// 求每个元素右边第一个比它大的元素下标
int ans[N];
int stk[N], top = 0;

void solve(int a[], int n) {
    memset(ans, -1, sizeof ans);
    for (int i = 0; i < n; i++) {
        while (top && a[stk[top]] < a[i]) ans[stk[top--]] = i;
        stk[++top] = i;
    }
}
```

**关键性质：** 每个元素最多入栈一次、出栈一次，总时间复杂度 $O(n)$。

### 2.3 表达式求值

表达式求值使用**两个栈**：一个存放操作数，一个存放运算符。核心规则：当前运算符优先级 $\leq$ 栈顶运算符时，先弹出计算。

```
示例：3 + 2 * 4 - 1

数字栈:  [3] [3,2] [3,2,4] →计算2*4=8→ [3,8] →计算3+8=11→ [11] [11,1] →计算11-1=10→ [10]
运算符栈: []   [+]   [+,*]  →弹出*→     [+]    →弹出+→       []   [-]     →弹出-→       []
```

### 2.4 括号匹配

遇到左括号入栈，遇到右括号时检查栈顶是否匹配。扫描结束后栈必须为空。这是栈 LIFO 特性的直接体现：每个右括号应与**最近的**未匹配左括号配对。

## 三、分类与变体详解

### 3.1 单调栈的方向与类型

| 扫描方向 | 栈单调性（底→顶） | 求解目标 |
|---------|-------------------|---------|
| 从左到右 | 递增（小→大） | 每个元素**右边第一个更大** |
| 从右到左 | 递增（小→大） | 每个元素**左边第一个更大** |
| 从左到右 | 递减（大→小） | 每个元素**右边第一个更小** |
| 从右到左 | 递减（大→小） | 每个元素**左边第一个更小** |

### 3.2 表达式栈

| 表达式类型 | 说明 | 栈的使用 |
|-----------|------|---------|
| 后缀表达式（逆波兰） | 运算符在操作数之后 | 一个数字栈 |
| 中缀表达式 | 正常书写方式 | 数字栈 + 运算符栈 |
| 中缀转后缀（调度场算法） | 转换表达式形式 | 一个运算符栈 |

### 3.3 递归栈模拟

任何递归都可以用栈来模拟。当递归层数很深（$10^6$ 级别）可能导致栈溢出时，可以手动用栈代替：

```cpp
// 栈模拟 DFS
void dfs_stack(int start) {
    int stk[N], top = 0;
    stk[++top] = start;
    while (top) {
        int u = stk[top--];
        if (vis[u]) continue;
        vis[u] = true;
        for (int i = head[u]; ~i; i = ne[i])
            if (!vis[e[i]]) stk[++top] = e[i];
    }
}
```

### 3.4 变体对比总表

| 变体 | 核心思想 | 时间复杂度 | 典型应用 |
|------|---------|-----------|---------|
| 单调栈 | 维护栈内单调性 | $O(n)$ | 下一个更大/小元素 |
| 表达式栈 | 优先级驱动 | $O(n)$ | 四则运算求值 |
| 递归模拟栈 | 手动管理调用帧 | 同递归 | 防栈溢出 |
| 最小栈 | 辅助栈记录最小值 | $O(1)$ 查询最小 | LeetCode 155 |

### 🟢 入门级 Level 1：栈的基础应用

### 题目 1：后缀表达式（洛谷 P1449）

> 给定一个后缀表达式（逆波兰表达式），数字和运算符之间用 `.` 分隔，以 `@` 结尾。求表达式的值。

**分析**：后缀表达式求值是栈最直接的应用。从左到右扫描，遇到数字入栈，遇到运算符弹出两个数计算后将结果压回栈。

```
数据结构：栈
支持操作：入栈数字、弹出两个操作数计算
关键性质：后缀表达式无需考虑优先级，扫描一遍即可求值
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
int stk[N], top = 0;

void solve(const string& s) {
    int num = 0;
    bool reading = false;
    for (char c : s) {
        if (c == '@') break;
        if (c >= '0' && c <= '9') { num = num * 10 + (c - '0'); reading = true; }
        else if (c == '.') {
            if (reading) { stk[++top] = num; num = 0; reading = false; }
        } else {
            int b = stk[top--], a = stk[top--];
            if (c == '+') stk[++top] = a + b;
            else if (c == '-') stk[++top] = a - b;
            else if (c == '*') stk[++top] = a * b;
            else if (c == '/') stk[++top] = a / b;
        }
    }
}
// 答案位置：stk[1]
```

**核心启示**：
- **后缀表达式天然无需优先级**：省去了处理运算符优先级的复杂逻辑
- **弹出顺序**：先弹出的是右操作数 $b$，后弹出的是左操作数 $a$，减法和除法不能交换

### 题目 2：表达式括号匹配（洛谷 P1739）

> 给定一个只包含 `(`、`)` 和其他字符的表达式，判断其中的括号是否匹配。

**分析**：栈的入门级应用。遇到左括号入栈，遇到右括号检查栈顶是否匹配。扫描结束后栈必须为空。

```
数据结构：栈
支持操作：左括号入栈、右括号匹配弹出
关键性质：LIFO 保证右括号总与最近的未匹配左括号配对
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
bool solve(const string& s) {
    int top = 0; // 只需计数，不需存内容
    for (char c : s) {
        if (c == '(') top++;
        else if (c == ')') {
            if (top == 0) return false;
            top--;
        }
    }
    return top == 0;
}
```

**核心启示**：
- **单种括号只需计数**：不需要真正的栈，一个计数器即可（多种括号才需要栈）
- **"最近匹配"**：括号匹配的本质是 LIFO，这是栈最直观的应用场景

### 题目 3：验证栈序列（洛谷 P1165 / LeetCode 946）

> 给定入栈序列和出栈序列，判断出栈序列是否合法。

**分析**：模拟入栈/出栈过程。按入栈顺序依次压入，每次压入后检查栈顶是否等于出栈序列的当前元素，若是则弹出。

```
数据结构：栈（模拟）
支持操作：按序入栈、条件弹出
关键性质：合法出栈序列 ↔ 模拟过程中栈不发生矛盾
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
bool validateStackSequences(vector<int>& pushed, vector<int>& popped) {
    int stk[N], top = 0, j = 0;
    for (int x : pushed) {
        stk[++top] = x;
        while (top && stk[top] == popped[j]) { top--; j++; }
    }
    return top == 0;
}
```

**核心启示**：
- **模拟法验证**：判断序列合法性时，直接模拟操作过程是最可靠的方法
- **贪心弹出**：每次入栈后尽可能弹出，贪心策略保证正确性

### 题目 4：最小栈（LeetCode 155）

> 设计一个栈，支持 push、pop、top 操作，并能在 $O(1)$ 时间内检索到最小元素。

**分析**：用一个辅助栈同步记录当前栈的最小值。每次 push 时，辅助栈压入 `min(x, 辅助栈顶)`。

```
数据结构：双栈（主栈 + 最小值辅助栈）
支持操作：push/pop/top/getMin 均 O(1)
关键性质：辅助栈第 i 个元素 = 主栈前 i 个元素的最小值
时间复杂度：所有操作 O(1)
```

**C++ 代码**：

```cpp
int stk[N], mn[N], top = 0; // mn[i] = min(stk[1..i])

void push(int x) {
    stk[++top] = x;
    mn[top] = (top == 1) ? x : min(mn[top - 1], x);
}
void pop() { top--; }
int getTop() { return stk[top]; }
int getMin() { return mn[top]; }
```

**核心启示**：
- **辅助栈/辅助数组**：额外维护一份"前缀最值"信息，空间换时间的经典策略
- **同步操作**：辅助栈与主栈同步 push/pop，无需额外逻辑

### 🟡 进阶级 Level 2：单调栈入门

### 题目 5：单调栈模板（洛谷 P5788）

> 给定长度为 $n$ 的数列，求每个元素**右边第一个**比它大的元素的下标（不存在输出 0）。

**分析**：单调栈的模板题。从左到右扫描，维护一个栈底到栈顶递减的栈（存下标），当前元素能"解决"所有比它小的栈顶。

```
数据结构：单调栈（栈底到栈顶递减）
支持操作：入栈、弹出并记录答案
关键性质：每个元素至多入栈一次、出栈一次，总 O(n)
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
int stk[N], top = 0;
int ans[N];

void solve(int a[], int n) {
    for (int i = n; i >= 1; i--) {
        while (top && a[stk[top]] <= a[i]) top--;
        ans[i] = top ? stk[top] : 0;
        stk[++top] = i;
    }
}
```

**核心启示**：
- **单调栈模板**：从右往左扫描时，栈维护"右边比当前元素大的候选"，栈顶就是最近的答案
- **存下标比存值更通用**：下标可以反查值，值无法反查下标

### 题目 6：每日温度（LeetCode 739）

> 给定每日温度数组，求每一天需要等几天才能等到更高的温度。如果没有，答为 0。

**分析**：本质就是"右边第一个更大元素的距离"，单调栈的直接应用。

```
数据结构：单调栈（栈底到栈顶递减）
支持操作：入栈、弹出时计算距离差
关键性质：ans[i] = nextGreater[i] - i
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
int ans[N], stk[N], top = 0;

void solve(int temp[], int n) {
    for (int i = 0; i < n; i++) {
        while (top && temp[stk[top]] < temp[i]) {
            ans[stk[top]] = i - stk[top];
            top--;
        }
        stk[++top] = i;
    }
    // 栈中剩余元素的 ans 保持 0（无更高温度）
}
```

**核心启示**：
- **下标做差求距离**：单调栈存下标的好处在此体现——弹出时 `i - stk[top]` 即为等待天数
- **变形灵活**：同一个模板，求"距离"还是求"值"只是弹出时取值方式的区别

### 题目 7：表达式求值（洛谷 P1981）

> 给定一个只包含加法和乘法的非负整数表达式（没有括号），求其值对 10000 取模。

**分析**：乘法优先级高于加法。用栈存储待相加的项：遇到乘法立即与栈顶计算，遇到加法先将数字入栈。最后栈中所有数求和。

```
数据结构：栈
支持操作：数字入栈、乘法立即计算、最终求和
关键性质：高优先级运算立即执行，低优先级运算延后 → 栈中只存待加的项
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MOD = 10000;
int stk[N], top = 0;

void solve() {
    int num;
    char op;
    // 读取第一个数
    stk[++top] = num % MOD;
    while (/* 读取 op 和 num */) {
        num %= MOD;
        if (op == '*') { stk[top] = (long long)stk[top] * num % MOD; }
        else { stk[++top] = num; }
    }
    int ans = 0;
    while (top) ans = (ans + stk[top--]) % MOD;
}
```

**核心启示**：
- **优先级分层处理**：高优先级立即计算，低优先级延后入栈，是栈处理表达式的通用策略
- **只有加乘时的简化**：不需要完整的中缀求值框架，一个栈就够

### 题目 8：下一个更大元素 II（LeetCode 503）

> 给定一个**循环数组**，求每个元素的下一个更大元素。

**分析**：循环数组的经典处理方式：将数组"展开"为两倍长度，即扫描 $[0, 2n)$ 并用 `i % n` 取模。结合单调栈求解。

```
数据结构：单调栈
支持操作：循环展开 + 单调栈弹出
关键性质：将循环数组拉直为 2n 长度，用取模模拟循环
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
int ans[N], stk[N], top = 0;

void solve(int a[], int n) {
    memset(ans, -1, sizeof(int) * n);
    for (int i = 0; i < 2 * n; i++) {
        while (top && a[stk[top]] < a[i % n]) {
            ans[stk[top--]] = a[i % n];
        }
        if (i < n) stk[++top] = i;
    }
}
```

**核心启示**：
- **循环数组展开**：`i % n` 是处理循环的万能技巧，适用于循环队列、循环数组等场景
- **只入栈一轮**：虽然扫描 $2n$ 次，但只有前 $n$ 个元素需要入栈

### 题目 9：柱状图中最大的矩形（LeetCode 84）

> 给定 $n$ 个宽度为 1 的柱子的高度，求柱状图中能勾勒出的最大矩形面积。

**分析**：对每个柱子，找到它**左边和右边第一个比它矮**的柱子位置，以该柱子高度为矩形高度，宽度为左右边界之间的距离。用单调递增栈一次遍历同时得到左右边界。

```
数据结构：单调栈（栈底到栈顶递增）
支持操作：弹出时计算以弹出元素为高度的矩形面积
关键性质：弹出元素 h 时，左边界 = 新栈顶，右边界 = 当前 i
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
int stk[N], top = 0;

long long largestRectangle(int h[], int n) {
    long long ans = 0;
    stk[0] = -1; top = 0; // 哨兵
    for (int i = 0; i <= n; i++) {
        int cur = (i == n) ? 0 : h[i]; // 末尾加高度 0 强制清栈
        while (top && h[stk[top]] > cur) {
            int height = h[stk[top--]];
            int width = i - stk[top] - 1;
            ans = max(ans, (long long)height * width);
        }
        stk[++top] = i;
    }
    return ans;
}
```

**核心启示**：
- **弹出时计算面积**：弹出元素的高度就是矩形高度，左右边界由栈顶和当前位置决定
- **哨兵技巧**：栈底放 -1、末尾加 0，避免大量边界判断
- **经典模型**：柱状图最大矩形是单调栈最重要的应用之一

### 题目 10：接雨水（LeetCode 42）

> 给定 $n$ 个非负整数表示柱子高度，计算下雨后能接多少雨水。

**分析**：单调栈解法（横向累加）。维护单调递减栈，当遇到比栈顶高的柱子时，弹出栈顶形成"凹槽"，用凹槽宽度和深度计算积水。

```
数据结构：单调栈（栈底到栈顶递减）
支持操作：弹出时计算凹槽积水量
关键性质：栈顶弹出后，新栈顶与当前柱子形成"左右墙壁"，中间是水槽
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
int stk[N], top = 0;

int trap(int h[], int n) {
    int ans = 0;
    for (int i = 0; i < n; i++) {
        while (top && h[stk[top]] < h[i]) {
            int bot = h[stk[top--]]; // 凹槽底部
            if (!top) break;         // 左边没有墙
            int w = i - stk[top] - 1;
            int depth = min(h[stk[top]], h[i]) - bot;
            ans += w * depth;
        }
        stk[++top] = i;
    }
    return ans;
}
```

**核心启示**：
- **横向累加 vs 竖向累加**：单调栈按层（横向）计算积水，双指针按列（竖向）计算积水，两种思路等价
- **三元素模型**：弹出的栈顶是凹槽底部，新栈顶是左墙，当前元素是右墙

### 题目 11：去除重复字母（LeetCode 316）

> 给定一个字符串，去除重复字母使得每个字母只出现一次，且结果的字典序最小。

**分析**：贪心 + 单调栈。栈内维护字典序递增的字符。入栈前，若栈顶字符比当前大且后面还会出现，则弹出栈顶。

```
数据结构：单调栈（字典序递增）
支持操作：条件弹出（栈顶更大且后面仍有）、跳过已在栈中的字符
关键性质：贪心选择局部最优，单调栈维护全局字典序最小
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
string removeDuplicateLetters(const string& s) {
    int cnt[26] = {}, inStk[26] = {};
    for (char c : s) cnt[c - 'a']++;
    string stk;
    for (char c : s) {
        cnt[c - 'a']--;
        if (inStk[c - 'a']) continue;
        while (!stk.empty() && stk.back() > c && cnt[stk.back() - 'a'] > 0) {
            inStk[stk.back() - 'a'] = 0;
            stk.pop_back();
        }
        stk += c;
        inStk[c - 'a'] = 1;
    }
    return stk;
}
```

**核心启示**：
- **单调栈 + 贪心**：弹出条件增加了"后面还有"的约束，确保不会丢失必要的字符
- **字符串做栈**：`string` 本身可以当栈用（`push_back` / `pop_back`）

### 题目 12：表达式的转换（洛谷 P1175）

> 将中缀表达式转换为后缀表达式（逆波兰表达式），并输出逐步求值的过程。

**分析**：两步：用调度场算法完成中缀转后缀，再逐步执行后缀表达式并输出中间状态。

```
数据结构：运算符栈（调度场）+ 数字栈（求值）
支持操作：优先级比较、括号匹配、弹出并输出
关键性质：调度场算法按优先级重排运算符，将中缀转为后缀
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
int pri(char op) {
    if (op == '+' || op == '-') return 1;
    if (op == '*' || op == '/') return 2;
    return 0;
}

// 调度场算法核心
void infixToPostfix(const string& expr, vector<string>& post) {
    char ops[N]; int top = 0;
    for (int i = 0; i < (int)expr.size(); i++) {
        if (isdigit(expr[i])) {
            string num;
            while (i < (int)expr.size() && isdigit(expr[i])) num += expr[i++];
            i--; post.push_back(num);
        } else if (expr[i] == '(') {
            ops[++top] = '(';
        } else if (expr[i] == ')') {
            while (ops[top] != '(') { post.push_back(string(1, ops[top--])); }
            top--; // 弹出 '('
        } else {
            while (top && ops[top] != '(' && pri(ops[top]) >= pri(expr[i]))
                post.push_back(string(1, ops[top--]));
            ops[++top] = expr[i];
        }
    }
    while (top) post.push_back(string(1, ops[top--]));
}
```

**核心启示**：
- **调度场算法**：中缀转后缀的标准方法，核心是用栈按优先级重排运算符
- **优先级相等时弹出**：左结合运算符（如 +、-）相同优先级时弹出，保证正确求值顺序

### 题目 13：最大矩形（LeetCode 85）

> 给定一个仅包含 0 和 1 的二维矩阵，找出只包含 1 的最大矩形面积。

**分析**：将每一行视为"地基"，将其上方连续 1 的个数作为"柱子高度"，对每一行用柱状图最大矩形（题目 9）求解。

```
数据结构：单调栈（逐行复用柱状图最大矩形）
支持操作：逐行构建高度数组 + 单调栈求最大矩形
关键性质：heights[j] = 当前行往上连续 1 的个数，遇到 0 则重置为 0
时间复杂度：O(n * m)
```

**C++ 代码**：

```cpp
int h[N]; // 每列的柱子高度

int maximalRectangle(vector<vector<char>>& mat) {
    int n = mat.size(), m = mat[0].size(), ans = 0;
    memset(h, 0, sizeof h);
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++)
            h[j] = (mat[i][j] == '1') ? h[j] + 1 : 0;
        ans = max(ans, (int)largestRectangle(h, m)); // 复用题目 9
    }
    return ans;
}
```

**核心启示**：
- **降维转化**：二维最大矩形 → 对每一行应用一维柱状图最大矩形
- **复合题型**：大问题拆成子问题逐行求解，是矩阵类单调栈题的通用套路

### 题目 14：股票价格跨度（LeetCode 901）

> 设计一个类，每天调用 `next(price)` 返回当前价格连续 $\leq$ price 的天数（包括今天）。

**分析**：本质是找"左边第一个更大元素"的距离。用单调递减栈，栈中存 `(price, span)` 的信息。

```
数据结构：单调栈（栈底到栈顶递减，存价格和跨度）
支持操作：弹出所有 ≤ 当前价格的元素，累加它们的跨度
关键性质：弹出元素的跨度被当前元素"吸收"
时间复杂度：每次调用均摊 O(1)
```

**C++ 代码**：

```cpp
int prices[N], spans[N], top = 0;

int next(int price) {
    int span = 1;
    while (top && prices[top] <= price) {
        span += spans[top--]; // 吸收被弹出元素的跨度
    }
    prices[++top] = price;
    spans[top] = span;
    return span;
}
```

**核心启示**：
- **跨度吸收**：弹出元素时将其覆盖的天数累加到当前元素，避免重复遍历
- **在线处理**：单调栈天然支持在线（每次只需处理新元素），适合流式数据

### 题目 15：子数组的最小值之和（LeetCode 907）

> 给定整数数组，对所有连续子数组求最小值，返回这些最小值的总和。

**分析**：对每个元素 `a[i]`，计算它作为最小值的子数组个数。用单调栈找出 `a[i]` 左右第一个更小元素的位置，则 `a[i]` 作为最小值的子数组数 = `left * right`。

```
数据结构：单调栈（分别求左右边界）
支持操作：求每个元素左边第一个更小、右边第一个更小（或等于）
关键性质：contribution[i] = a[i] * left[i] * right[i]，注意处理相等元素
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MOD = 1e9 + 7;
int L[N], R[N], stk[N], top;

long long sumSubarrayMins(int a[], int n) {
    // 求左边第一个严格更小的位置
    top = 0;
    for (int i = 0; i < n; i++) {
        while (top && a[stk[top]] >= a[i]) top--;
        L[i] = top ? (i - stk[top]) : (i + 1);
        stk[++top] = i;
    }
    // 求右边第一个更小或等于的位置（用 <= 防止重复计数）
    top = 0;
    for (int i = n - 1; i >= 0; i--) {
        while (top && a[stk[top]] > a[i]) top--;
        R[i] = top ? (stk[top] - i) : (n - i);
        stk[++top] = i;
    }
    long long ans = 0;
    for (int i = 0; i < n; i++)
        ans = (ans + (long long)a[i] * L[i] % MOD * R[i]) % MOD;
    return ans;
}
```

**核心启示**：
- **贡献法**：不枚举子数组再求最小值，而是枚举每个元素计算其贡献次数
- **处理相等元素**：左边用"严格小于"、右边用"小于等于"（或反过来），避免重复计数

### 🟣 竞赛级 Level 5：高难度栈应用

### 题目 16：「HNOI2008」玩具装箱（洛谷 P3195）

> 将 $n$ 个玩具装箱，连续若干个一组，每组费用为 $(j - i + \sum_{k=i}^{j} c_k - L)^2$。求总费用最小值。

**分析**：经典斜率优化 DP。设 $f[i]$ 为前 $i$ 个玩具的最小费用，转移方程展开后用单调栈（单调队列）维护下凸包，实现 $O(n)$ 求解。

```
数据结构：单调栈/单调队列（维护下凸包）
支持操作：队尾弹出维护凸性、队首弹出维护最优决策
关键性质：转移方程满足决策单调性，斜率优化将 O(n²) 降为 O(n)
时间复杂度：O(n)
```

**C++ 代码**：

```cpp
long long f[N], s[N]; // s[i] = c[1]+...+c[i]+i（前缀和+偏移）
int q[N], ql, qr;     // 单调队列存决策点

double slope(int j, int k) {
    // (f[j]+s[j]^2 - f[k]-s[k]^2) / (s[j]-s[k])
    return (double)(f[j] + s[j] * s[j] - f[k] - s[k] * s[k]) / (s[j] - s[k]);
}

void solve(int n, long long L) {
    // s[i] = sum(c[1..i]) + i, 实际偏移为 L+1
    ql = qr = 0; q[0] = 0;
    for (int i = 1; i <= n; i++) {
        while (ql < qr && slope(q[ql + 1], q[ql]) <= 2.0 * (s[i] - L - 1)) ql++;
        int j = q[ql];
        f[i] = f[j] + (s[i] - s[j] - L - 1) * (s[i] - s[j] - L - 1);
        while (ql < qr && slope(q[qr], q[qr - 1]) >= slope(i, q[qr])) qr--;
        q[++qr] = i;
    }
}
// 答案位置：f[n]
```

**核心启示**：
- **斜率优化 DP**：当转移方程可以化为 $y = kx + b$ 的形式，用单调栈/队列维护凸包，将 $O(n^2)$ 转移优化为 $O(n)$
- **单调栈的抽象应用**：这里的"单调性"不是值的单调，而是斜率的单调，是更高层次的单调栈思想

## 七、终极总结与归纳

### 7.1 栈的本质

**栈 = 后进先出（LIFO），天然处理"最近相关性"问题。**

从括号匹配到单调栈，从表达式求值到斜率优化，栈的价值在于：用 $O(1)$ 的操作维护一个有序或待处理的状态序列，使得每个元素至多入栈、出栈各一次，总复杂度 $O(n)$。

### 7.2 核心要点

1. **单调栈**是竞赛中最高频的栈应用，"下一个更大/更小元素"是基础模型
2. **哨兵技巧**（栈底放极值、数组末尾加 0）能显著简化边界处理
3. **存下标比存值更通用**：下标可以反查值和计算距离
4. **贡献法 + 单调栈**是"子数组最值之和"类问题的标准解法
5. **表达式求值**的核心是优先级驱动：高优先级立即算，低优先级入栈等待
6. **斜率优化 DP** 是单调栈在 DP 领域的高级应用，将 $O(n^2)$ 降为 $O(n)$
