## 一、什么是构造

构造题（Constructive Algorithms）是算法竞赛中一类极具创造性的题型。它不要求你"求最优解"或"计数"，而是要求你**按照给定约束条件，主动创造出一个满足所有条件的对象**——可以是一个序列、一个排列、一个图、一棵树、一个字符串，甚至一个方案。

构造题的核心特征是：**答案不唯一，自由度高，验证比构造容易得多**。你需要利用数学性质、分类讨论、贪心思维等手段，从看似混乱的条件中找到一条可行路线。

### 构造题 vs 其他题型对比

| 特征 | 构造题 | DP/搜索 | 贪心 |
|------|--------|---------|------|
| 目标 | 给出一组**可行解** | 求**最优解** | 求**最优解** |
| 解的数量 | 通常**多解** | 通常唯一 | 通常唯一 |
| 模板化程度 | **极低** | 中高 | 中 |
| 核心能力 | 观察、归纳、分类 | 状态设计 | 贪心策略证明 |
| 验证难度 | **容易** | 不适用 | 不适用 |

> **一句话理解**：构造题就是"出题人给你限制条件，你当建筑师，造一个满足所有条件的东西出来。"

## 二、核心思想与前置知识

### 2.1 构造题的思维方式

构造题需要两种关键思维：

**逆向思维**：不从"怎么构造"出发，而从"答案需要满足什么性质"倒推。例如"前缀和模 n 互不相同"→ 逆推得出 n 必须为偶数。

**特殊到一般**：先手算 n=1,2,3,4,5 的小数据，观察规律，再归纳出通用构造方法。这是构造题最常用的入手方式。

```
思维链路：
题目条件 → 手算小数据 → 发现规律/不变量 → 猜想构造方法 → 证明正确性
```

### 2.2 常用构造手法

| 手法 | 说明 | 典型场景 |
|------|------|----------|
| **打表找规律** | 暴力枚举小数据，观察输出模式 | 排列构造、数论构造 |
| **对称构造** | 利用对称性简化问题 | 奇偶交替、首尾配对 |
| **递推构造** | 从 n-1 的解推导 n 的解 | 排列、图的增量构造 |
| **分类讨论** | 按 n 的奇偶、质合等分情况处理 | 几乎所有构造题 |
| **贪心构造** | 按某种优先策略逐步搭建解 | 最优化类构造 |
| **差分/前缀和** | 将条件转化为差分数组上的操作 | 区间操作类构造 |

### 2.3 构造题的正确性验证

构造题与其他题型最大的不同在于：**你必须自己验证答案的正确性**。常见方法：

1. **手动验证**：对小数据手算，确认构造结果满足所有约束
2. **数学证明**：利用不变量、归纳法等严格证明构造的正确性
3. **对拍检验**：写暴力检查程序，随机数据验证构造结果

```
正确性检查清单：
1. 是否满足所有约束条件？（逐条对照题意）
2. 是否覆盖所有情况？（边界、特殊值）
3. 无解的判定是否正确？（必要条件 vs 充分条件）
```

## 三、构造的子模型与分类详解

### 3.1 奇偶性构造（利用奇偶性质构造解）

利用数的奇偶性来设计构造方案，是最常见的入门构造手法。

```
核心思路：
- 奇 + 奇 = 偶, 偶 + 偶 = 偶, 奇 + 偶 = 奇
- 奇 × 奇 = 奇, 偶 × 任何 = 偶
- 排列中奇偶数个数固定，利用此性质分配位置
适用信号：题目涉及"奇偶交替"、"和的奇偶性"、"相邻差"
实现要点：按奇偶分组，分别处理后合并
```

```cpp
// 奇偶分离构造模板
void construct_parity(int n, int a[]) {
    int idx = 0;
    for (int i = 2; i <= n; i += 2) a[idx++] = i; // 先放偶数
    for (int i = 1; i <= n; i += 2) a[idx++] = i; // 再放奇数
}
```

### 3.2 排列构造（构造满足特定性质的排列）

要求构造 1~n 的一个排列，使其满足特定前缀和、前缀积、相邻差等约束。

```
核心思路：
- 排列的总和固定为 n*(n+1)/2
- 首尾交替配对（n,1,n-1,2,...）可控制前缀和变化
- 利用逆序对、位置交换调整排列性质
适用信号：题目要求"构造排列"、"前缀和/积满足条件"
实现要点：先确定特殊位置（首、尾），再填充中间
```

```cpp
// 首尾交替排列构造
void construct_zigzag(int n, int a[]) {
    int l = 1, r = n, idx = 0;
    while (l <= r) {
        a[idx++] = r--;
        if (l <= r) a[idx++] = l++;
    }
}
```

### 3.3 数论构造（利用逆元、模运算构造）

利用模运算、逆元、质数性质等数论工具进行构造，常见于要求"模 n 互不相同"的排列构造。

```
核心思路：
- 质数 p 的乘法群 Z_p* 中每个非零元素都有逆元
- 前缀积模 p 互不相同 → 利用逆元递推
- 费马小定理：a^{p-1} ≡ 1 (mod p)，逆元 = a^{p-2} mod p
适用信号：题目涉及"模运算"、"逆元"、"质数判断"
实现要点：先判无解条件（合数），再用逆元递推
```

```cpp
long long qpow(long long a, long long b, long long mod) {
    long long res = 1;
    a %= mod;
    while (b > 0) {
        if (b & 1) res = res * a % mod;
        a = a * a % mod;
        b >>= 1;
    }
    return res;
}
// 逆元：inv(a, p) = qpow(a, p - 2, p)
```

### 3.4 贪心构造（通过贪心策略构造最优解）

当构造题带有"最大化/最小化"目标时，通常需要贪心策略来指导构造过程。

```
核心思路：
- 排序后优先处理"影响最大"的元素
- 每一步的选择都不会让后续更差（贪心正确性）
- 常与博弈论结合：两人轮流选择时，每次选当前最大值
适用信号：题目要求"最多/最少"、"最优策略"、轮流选择
实现要点：排序 + 模拟，注意类型判断（奇偶/颜色）
```

```cpp
void greedy_construct(int n, int a[]) {
    sort(a, a + n, greater<int>()); // 从大到小
    for (int i = 0; i < n; i++) {
        // 按贪心策略分配每个元素
    }
}
```

### 3.5 图论构造（构造满足条件的图/树）

要求构造一个图或树，满足度数、连通性、强连通性等约束。

```
核心思路：
- 竞赛图（完全有向图）中出度均匀 → 强连通子图最多
- 正 n 边形顶点模型：沿顺时针方向连边，控制出度
- 树的构造：考虑叶子节点、直径、度数约束
适用信号：题目涉及"有向图"、"出度/入度"、"连通分量"
实现要点：利用几何对称性（环形排列）简化构造
```

```cpp
// 竞赛图构造：每个点向后 m 个点连有向边
int g[MAXN][MAXN];
void construct_tournament(int n) {
    int m = (n + 1) / 2;
    for (int i = 0; i < n; i++)
        for (int j = 1; j <= m; j++)
            g[i][(i + j) % n] = 1;
}
```

### 3.6 打表找规律（暴力枚举小数据发现规律）

当构造方法不明显时，先用暴力程序枚举小数据的所有解，从中发现规律。

```
核心思路：
- 对 n=1,2,...,10 暴力搜索所有合法解
- 观察解的结构特征（周期性、对称性、递推关系）
- 归纳猜想通用公式，再用数学证明
适用信号：题意简单但构造方法不明显
实现要点：暴力搜索 → 观察输出 → 归纳 → 推广
```

```cpp
// 暴力打表框架
void brute_force(int n) {
    // 枚举所有 1~n 的排列
    vector<int> perm(n);
    iota(perm.begin(), perm.end(), 1);
    do {
        if (check(perm)) { // 验证是否满足条件
            print(perm);
        }
    } while (next_permutation(perm.begin(), perm.end()));
}
```

### 题目 1：简单奇偶构造（CF1constructive-1）

> 给定正整数 n，构造一个长度为 n 的数组，使得所有元素之和为奇数，且每个元素为正整数。

**分析**：和为奇数的最简单方法——放 1 个奇数和 n-1 个偶数。如果 n 个偶数之和一定为偶数，加一个奇数即为奇数。直接构造：第一个元素为 1，其余全为 2 即可。

```
构造策略：
- 第一个位置放 1（奇数）
- 其余位置全放 2（偶数）
- 总和 = 1 + 2*(n-1) = 2n-1（奇数）✓
```

**C++ 代码**：

```cpp
int ans[MAXN];

void solve(int n) {
    ans[0] = 1;
    for (int i = 1; i < n; i++) ans[i] = 2;
}
```

答案位置：`ans[0..n-1]` 即为构造结果

**核心启示**：
- **奇偶性是最基本的构造工具**：一个奇数 + 若干偶数 = 奇数
- **从最简单的情况出发**：不要想复杂，先找最直接的构造方案
- **验证必不可少**：构造完立刻检查是否满足所有约束

### 题目 2：Wonderful Coloring - 1（CF1551B1）

> 给一个只含小写字母的字符串，用红/绿两色染色。要求：同种颜色中字母互不相同，红色数量 = 绿色数量，最大化总染色数。输出红色（或绿色）的字母数量。

**分析**：出现次数 ≥ 2 的字母可以一个染红一个染绿，各贡献 1 对。出现恰好 1 次的字母需要两两配对，每对贡献 1。统计即可。

```
构造策略：
- 统计每个字母出现次数 cnt[c]
- cnt[c] >= 2：贡献 1 对（红1 + 绿1）
- cnt[c] == 1：累加到 singles 计数
- 答案 = 贡献对数 + floor(singles / 2)
```

**C++ 代码**：

```cpp
int cnt[26];

int solve(const char* s, int len) {
    memset(cnt, 0, sizeof(cnt));
    for (int i = 0; i < len; i++) cnt[s[i] - 'a']++;
    int pairs = 0, singles = 0;
    for (int i = 0; i < 26; i++) {
        if (cnt[i] >= 2) pairs++;
        else if (cnt[i] == 1) singles++;
    }
    return pairs + singles / 2;
}
```

答案位置：`solve(s, len)` 返回值

**核心启示**：
- **分类计数是构造的基础**：按出现次数分类处理，各取所需
- **配对思想**：当需要两组相等时，总是考虑如何两两配对
- **贪心与构造结合**：最大化目标时，优先处理确定贡献的部分

### 题目 3：歌唱比赛（P5595 [XR-4]）

> 给定长度为 n 的字符串 s，每位为 'X'/'Y'/'Z'。从最低位到最高位比较两个 n 位数，'X' 表示第一个数当前更大，'Y' 表示第二个更大，'Z' 表示相等。构造两个满足条件的 n 位数，或输出 -1。

**分析**：关键观察——'Z' 只能出现在字符串的连续后缀中。如果 'Z' 出现在中间（后面还有非 'Z'），则无解。合法时，逐位构造：'X' 位让 a=2,b=1；'Y' 位让 a=1,b=2；'Z' 位让 a=b=1。

```
构造策略：
- 合法性检查：'Z' 只能出现在末尾连续段
- 若中间出现 'Z'（其后有非 'Z'）→ 输出 -1
- 逐位赋值：X→(2,1), Y→(1,2), Z→(1,1)
```

**C++ 代码**：

```cpp
int a[MAXN], b[MAXN];

bool solve(const char* s, int n) {
    // 合法性检查：找到最后一个非Z字符的位置
    int last_nonZ = -1;
    for (int i = 0; i < n; i++)
        if (s[i] != 'Z') last_nonZ = i;
    // last_nonZ 之前不能有 Z
    for (int i = 0; i < last_nonZ; i++)
        if (s[i] == 'Z') return false;
    // 构造
    for (int i = 0; i < n; i++) {
        if (s[i] == 'X')      { a[i] = 2; b[i] = 1; }
        else if (s[i] == 'Y') { a[i] = 1; b[i] = 2; }
        else                   { a[i] = 1; b[i] = 1; }
    }
    return true;
}
```

答案位置：`a[0..n-1]` 和 `b[0..n-1]` 即为两个数的各位

**核心启示**：
- **无解判定先行**：先分析什么条件下无解，排除后再构造
- **逐位独立构造**：当各位之间约束独立时，逐位处理最简洁
- **'Z' 的连续性约束**：相等关系只能从末尾开始延伸，体现了数位比较的本质

### 题目 4：Even-Odd Game（CF1472D）

> n 个数排成一列，Alice 和 Bob 轮流选数（Alice 先手）。Alice 选偶数得分，Bob 选奇数得分，选到另一类不得分。判断谁赢。

**分析**：贪心策略——每人都应优先选当前最大的数。如果最大数是自己能得分的类型就得分，否则也要拿走（阻止对手得分）。排序后模拟即可。

```
构造策略：
- 将数组从大到小排序
- Alice 选下标 0,2,4,...（偶数位），Bob 选 1,3,5,...
- Alice 轮：若当前数为偶数则加分
- Bob 轮：若当前数为奇数则加分
- 比较两人总分
```

**C++ 代码**：

```cpp
int a[MAXN];

void solve(int n) {
    sort(a, a + n, greater<int>());
    long long alice = 0, bob = 0;
    for (int i = 0; i < n; i++) {
        if (i % 2 == 0) { // Alice 的回合
            if (a[i] % 2 == 0) alice += a[i];
        } else { // Bob 的回合
            if (a[i] % 2 == 1) bob += a[i];
        }
    }
    // alice > bob → "Alice", alice < bob → "Bob", else → "Tie"
}
```

答案位置：比较 `alice` 和 `bob` 输出结果

**核心启示**：
- **贪心构造的核心：排序**，让决策顺序变得明确
- **阻止对手得分 = 自己得分**：选最大值即使不得分也在阻止对手
- **贪心正确性**：每步选最大值的策略是博弈论中的最优策略

### 题目 5：Product of Three Numbers（CF1294C）

> 给定正整数 n，判断能否将 n 分解为三个不同的 ≥ 2 的正整数之积 a×b×c = n。若能，输出一组解。

**分析**：枚举因子。从 2 到 √n 找第一个因子 a，然后在 n/a 中从 a+1 开始找第二个因子 b，剩余 c = n/(a×b)。检查 c ≥ 2 且三者互不相同。

```
构造策略：
- 枚举 a ∈ [2, √n]，找到 n%a==0 的第一个 a
- 令 rem = n/a，枚举 b ∈ [a+1, √rem]，找到 rem%b==0 的第一个 b
- c = rem/b，检查 c >= 2 且 c != a 且 c != b
- 两层循环都找不到则无解
```

**C++ 代码**：

```cpp
bool solve(int n, int &a, int &b, int &c) {
    for (a = 2; (long long)a * a * a <= n; a++) {
        if (n % a != 0) continue;
        int rem = n / a;
        for (b = a + 1; (long long)b * b <= rem; b++) {
            if (rem % b != 0) continue;
            c = rem / b;
            if (c > b) return true;
        }
    }
    return false;
}
```

答案位置：`a`, `b`, `c` 即为三个因子

**核心启示**：
- **因子枚举是数论构造的基本功**：范围控制在 √n 保证效率
- **贪心取最小因子**：优先取小因子，留给后续更大的搜索空间
- **三数互不相同的判定**：b 从 a+1 开始枚举，c 必须 > b，自然互异

### 题目 6：Social Distance（CF1367C）

> 给定长度为 n 的 01 串和正整数 k，任意两个 1 之间至少间隔 k 个 0。问最多能把多少个 0 变成 1。

**分析**：贪心地从左到右扫描，维护上一个 1 的位置。遇到 0 时，若与上一个 1 的距离 > k，则可以放置新的 1。遇到原有的 1 时更新位置。

```
构造策略：
- 记录上一个 1 的位置 last = -k-1（初始化为极远）
- 从左到右扫描每个位置 i：
  - s[i]=='1'：更新 last=i
  - s[i]=='0' 且 i-last > k：放置 1，ans++，last=i
- 但还需检查右侧 k 范围内是否有原始的 1
```

**C++ 代码**：

```cpp
int solve(int n, int k, char s[]) {
    int ans = 0, last = -k - 1;
    for (int i = 0; i < n; i++) {
        if (s[i] == '1') {
            last = i;
        } else if (i - last > k) {
            // 检查右侧 k 范围内无原始 1
            bool ok = true;
            for (int j = i + 1; j <= min(i + k, n - 1); j++) {
                if (s[j] == '1') { ok = false; i = j - 1; break; }
            }
            if (ok) { ans++; last = i; }
        }
    }
    return ans;
}
```

答案位置：`solve(n, k, s)` 返回值

**核心启示**：
- **贪心放置策略**：尽量早放，因为早放不会让后续更差
- **双向约束处理**：不仅看左侧距离，还需检查右侧是否有冲突
- **模拟是验证贪心的好方法**：对小数据手动模拟确认策略正确性

### 题目 7：Alphabetical Strings（Codeforces CF1547B）

> 定义"字母序字符串"：初始为空，依次插入 a, b, c, ...（按字母序），每次只能插到当前串的**最前面或最后面**。给定字符串 s，判断它是否是字母序字符串。

**分析**：逆向模拟——从最大字母向 'a' 逐个剥离。每步检查当前串的首字符或尾字符是否等于期望字母，若都不等则不合法。用双指针从两端向中间收缩即可。

```
构造策略：
- 串长 n 必须 ≤ 26，且必须恰好包含 a~第 n 个字母各一次
- 从字母 (n-1+'a') 到 'a' 逆序验证
- 每步检查 s[l] 或 s[r] 是否等于当前字母
- 匹配则收缩对应端，不匹配则判 NO
```

**C++ 代码**：

```cpp
bool solve(char s[], int n) {
    if (n > 26) return false;
    int l = 0, r = n - 1;
    for (int c = n - 1; c >= 0; c--) {
        char expected = 'a' + c;
        if (s[l] == expected) l++;
        else if (s[r] == expected) r--;
        else return false;
    }
    return true;
}
```

**核心启示**：
- **逆向模拟是验证构造合法性的利器**：正向构造自由度高，逆向验证约束更强
- **双指针处理首尾操作**：插入只在两端 → 验证也只看两端
- **字符唯一性是隐含约束**：每个字母恰好出现一次，违反即无解

### 题目 8：K-th Not Divisible by n（Codeforces CF1352C）

> 给定正整数 n 和 k，求第 k 个**不能被 n 整除**的正整数。

**分析**：每连续 n 个正整数中恰好有 n−1 个不被 n 整除。因此将 k 个数分成完整的"块"：每块 n−1 个不整除数对应一段长度 n 的区间。用除法算出完整块数 q 和余数 r，直接算出答案。

```
构造策略：
- 每 n 个连续整数含 n-1 个不整除数
- q = (k-1) / (n-1)，r = (k-1) % (n-1)
- 答案 = q * n + r + 1
- 即跳过了 q 个 n 的倍数后，再偏移 r+1
```

**C++ 代码**：

```cpp
long long solve(long long n, long long k) {
    long long q = (k - 1) / (n - 1);
    long long r = (k - 1) % (n - 1);
    return q * n + r + 1;
}
```

**核心启示**：
- **周期性是数学构造的核心工具**：每 n 个数形成一个周期，周期内结构固定
- **除法 + 取余 = 定位**：q 定位到哪个周期，r 定位周期内偏移
- **O(1) 公式优于模拟**：不需要逐个数判断，直接数学计算

### 题目 9：伤痕（P5441 [XR-2]）

> 给定奇数 n，在 n 个点的竞赛图（完全有向图，双向边最多 n 条）中构造邻接矩阵，使得"任取 4 点两两可互达"的方案数最大。

**分析**：最大化强连通四元组数 = 总四元组数 - 非强连通四元组数。非强连通四元组数与各点出度的 C(d_i, 3) 之和有关。由凸函数性质，出度越均匀，非强连通数越少。令每个点出度 = (n-3)/2（单向边部分），用正 n 边形模型构造。

```
构造策略：
- 最大方案数公式：n*(n-3)*(n²+6n-31)/48
- 构造方法：将 n 个点放在正 n 边形顶点上
- 每个点向顺时针后 (n+1)/2 个点连有向边
- 这使得所有点出度相等，最小化非强连通组数
```

**C++ 代码**：

```cpp
int g[MAXN][MAXN];

void solve(int n) {
    long long ans = (long long)n * (n - 3) % 48 == 0 ?
        (long long)n * (n - 3) / 48 * ((long long)(n) * n + 6 * n - 31) :
        (long long)n * (n - 3) * ((long long)n * n + 6 * n - 31) / 48;
    // 输出 ans
    int m = (n + 1) / 2;
    for (int i = 0; i < n; i++)
        for (int j = 1; j <= m; j++)
            g[i][(i + j) % n] = 1;
    // g[i][j] 即邻接矩阵
}
```

答案位置：第一行输出 `ans`，后 n 行输出邻接矩阵 `g[][]`

**核心启示**：
- **凸函数 + 均匀分配**：当目标函数是凸的，让变量尽量均匀可以最小化/最大化
- **正多边形模型**：环形对称结构是构造均匀有向图的利器
- **公式推导先行**：先算出答案公式，再反推构造方法

### 题目 10：Super-Permutation（CF1822D）

> 构造长度为 n 的排列 a（1~n），使得 b[i] = (a[1]+...+a[i]) % n 也恰好是 0~n-1 的一个排列。若不存在输出 -1。

**分析**：n=1 时答案为 [1]。n 为奇数且 n>1 时无解（可证前缀和模 n 必有重复）。n 为偶数时，采用"首尾交替"策略：依次放 n, 1, n-1, 2, n-2, 3, ...，使得相邻前缀和的差交替为大正数和小负数，保证模 n 互不相同。

```
构造策略：
- n = 1 → 输出 [1]
- n 为奇数且 n > 1 → 无解，输出 -1
- n 为偶数 → 首尾交替：n, 1, n-1, 2, n-2, 3, ...
  即 a[奇数位] = 从大到小，a[偶数位] = 从小到大
```

**C++ 代码**：

```cpp
int a[MAXN];

bool solve(int n) {
    if (n == 1) { a[0] = 1; return true; }
    if (n % 2 == 1) return false;
    int l = 1, r = n, idx = 0;
    while (l <= r) {
        a[idx++] = r--;
        if (l <= r) a[idx++] = l++;
    }
    return true;
}
```

答案位置：`a[0..n-1]` 即为构造的排列

**核心启示**：
- **奇偶分类是排列构造的第一步**：n 的奇偶性往往决定有解/无解
- **首尾交替配对**：这是控制前缀和模 n 的经典技巧
- **无解判定要严格**：不能遗漏边界情况（n=1 特判）

### 题目 11：IncDec Sequence（P4552 [Poetize6]）

> 给定长度为 n 的序列，每次操作可选一个区间 [l,r] 全部 +1 或 -1。问最少多少次操作使所有元素相等？最少操作次数下有多少种最终值？

**分析**：转化为差分数组。区间 [l,r] 加 1 等价于 d[l]++ 和 d[r+1]--。目标是让 d[2]~d[n] 全变为 0。正差分之和为 P，负差分绝对值之和为 N，则：最少操作 = max(P, N)，结果种数 = |P - N| + 1。

```
构造策略（差分视角）：
- 差分数组 d[i] = a[i] - a[i-1]（i >= 2）
- 目标：让 d[2..n] 全为 0
- 正差分总和 P = Σ max(d[i], 0)
- 负差分总和 N = Σ |min(d[i], 0)|
- 最少操作 = max(P, N)（正负配对抵消 + 剩余单独处理）
- 最终值种数 = |P - N| + 1
```

**C++ 代码**：

```cpp
long long a[MAXN];

void solve(int n) {
    long long P = 0, N = 0;
    for (int i = 2; i <= n; i++) {
        long long d = a[i] - a[i - 1];
        if (d > 0) P += d;
        else N -= d;
    }
    long long min_ops = max(P, N);
    long long num_results = abs(P - N) + 1;
    // 输出 min_ops 和 num_results
}
```

答案位置：`min_ops` 为最少操作次数，`num_results` 为结果种数

**核心启示**：
- **差分是区间操作的核心工具**：区间加减 ↔ 差分数组的单点修改
- **正负配对抵消**：一次操作可以同时消除一个正差分和一个负差分
- **自由度 = 结果种数**：剩余无法配对的操作产生 a[1] 的变化范围

### 题目 12：Correct Placement（Codeforces CF1472E）

> 有 n 个人，每人有身高 h_i 和体宽 w_i。对每个人 i，找一个人 j，使得（可能旋转 90° 后）j 的两个维度都严格小于 i 的两个维度。输出任意合法 j，或 -1。

**分析**：将每个人规范化为 (min, max)，按第一维排序。对于每个人 i，需要在第一维严格小于自己的那些人中，找第二维最小的。用前缀维护"第一维 < 当前值"的最小第二维及其编号即可。

```
构造策略：
- 规范化：令 a[i] = min(h,w), b[i] = max(h,w)
- 按 a 排序（相同则按 b 排序）
- 维护前缀中 b 的最小值 best_b 及对应原始下标
- 对当前组（a 相同的连续段），查询 best_b < 当前 b 即可
- 处理完当前组后，更新 best_b
```

**C++ 代码**：

```cpp
int ans[MAXN];

void solve(int n, int h[], int w[]) {
    vector<array<int,3>> v(n);
    for (int i = 0; i < n; i++)
        v[i] = {min(h[i],w[i]), max(h[i],w[i]), i};
    sort(v.begin(), v.end());
    int bestB = INT_MAX, bestIdx = -1;
    for (int i = 0; i < n; ) {
        int j = i;
        while (j < n && v[j][0] == v[i][0]) j++;
        for (int k = i; k < j; k++)
            ans[v[k][2]] = (bestB < v[k][1]) ? bestIdx + 1 : -1;
        for (int k = i; k < j; k++)
            if (v[k][1] < bestB) { bestB = v[k][1]; bestIdx = v[k][2]; }
        i = j;
    }
}
```

**核心启示**：
- **规范化降维**：旋转等价 → 统一为 (min, max)，消除旋转自由度
- **排序 + 前缀最值**：排序消除一维约束，前缀最值处理另一维
- **分组处理相同值**：第一维相同时不能互相匹配，需先查询再更新

### 题目 13：Boats Competition（Codeforces CF1399C）

> n 个人（n ≤ 50）体重分别为 w_1,...,w_n。选一个目标和 s，将人两两配对使每对体重之和恰为 s。求最优 s 下的最大配对数。

**分析**：n 很小，直接枚举所有可能的 s（范围 2 到 2·max_w）。对每个 s，排序后用双指针贪心配对，统计最大对数。

```
构造策略：
- 排序数组 w
- 枚举 s ∈ [2, 200]
- 对每个 s，双指针 l=0, r=n-1 贪心配对
  - w[l]+w[r]==s → 配对成功，l++, r--, cnt++
  - w[l]+w[r]<s  → l++
  - w[l]+w[r]>s  → r--
- 答案 = max(cnt) over all s
```

**C++ 代码**：

```cpp
int solve(int n, int w[]) {
    sort(w, w + n);
    int ans = 0;
    for (int s = 2; s <= 200; s++) {
        int cnt = 0, l = 0, r = n - 1;
        while (l < r) {
            if (w[l] + w[r] == s) { cnt++; l++; r--; }
            else if (w[l] + w[r] < s) l++;
            else r--;
        }
        ans = max(ans, cnt);
    }
    return ans;
}
```

**核心启示**：
- **枚举构造参数**：当参数空间小时，暴力枚举所有可能的构造目标
- **双指针贪心配对**：排序后首尾配对是经典的两数之和计数方法
- **O(V·n) 暴力可行**：n ≤ 50，值域 ≤ 100，直接枚举无需优化

### 题目 14：K-Complete Word（Codeforces CF1332C）

> 给定长度为 n 的字符串 s 和正整数 k（k | n）。要求用最少的字符替换使 s 同时是**回文串**且**k-周期串**（s[i]=s[i+k]）。输出最少替换次数。

**分析**：k-周期性使得下标模 k 相同的位置必须相同字符。回文性使得位置 j 和 k−1−j（在周期内）也必须相同。因此位置 j 和 k−1−j 及其所有周期重复构成一个等价类。对每个等价类，统计所有位置的字符频率，保留出现最多的，其余全部替换。

```
构造策略：
- 对周期内每对 (j, k-1-j)，收集所有等价位置的字符
  - j 的等价位置：j, j+k, j+2k, ...
  - k-1-j 的等价位置（若 j ≠ k-1-j）：k-1-j, 2k-1-j, ...
- 统计每个等价类中 26 个字母的频次
- 替换数 = 类中总字符数 - 最大频次
- 答案 = 所有类的替换数之和
```

**C++ 代码**：

```cpp
int solve(int n, int k, char s[]) {
    int ans = 0;
    for (int j = 0; j <= (k - 1) / 2; j++) {
        int cnt[26] = {}, total = 0;
        for (int p = j; p < n; p += k)
            cnt[s[p] - 'a']++, total++;
        if (j != k - 1 - j)
            for (int p = k - 1 - j; p < n; p += k)
                cnt[s[p] - 'a']++, total++;
        ans += total - *max_element(cnt, cnt + 26);
    }
    return ans;
}
```

**核心启示**：
- **周期 + 回文 = 等价类合并**：两种对称性叠加，产生更大的等价类
- **最优替换 = 总数 − 最大频次**：经典的"统一为众数"策略
- **模运算建立等价关系**：位置 i 的等价类由 i%k 和回文对称共同决定

### 题目 15：Mikasa（Codeforces CF1554C）

> 给定非负整数 n 和 m，集合 S = {n⊕0, n⊕1, ..., n⊕m}。求 mex(S)，即不在 S 中的最小非负整数。

**分析**：x ∈ S 等价于存在 k ∈ [0,m] 使 n⊕k = x，即 k = n⊕x ≤ m。因此 x ∉ S 等价于 n⊕x > m。问题转化为：**求最小 x 使得 n⊕x ≥ m+1**。若 n > m，则 x=0 时 n⊕0 = n > m，答案为 0。否则从高位到低位贪心构造 x：每位优先取 0（最小化 x），仅当 n⊕x 的前缀会小于 m+1 时才被迫取 1。

```
构造策略：
- 若 n > m → 答案 0（因为 0 ∉ S）
- 令 t = m+1，从高位到低位贪心：
  - n 的当前位 == t 的当前位 → x 取 0，保持 tight
  - n 的当前位 > t 的当前位 → x 取 0，已超过 t，结束
  - n 的当前位 < t 的当前位 → x 必须取 1（翻转后 = t 位），继续
```

**C++ 代码**：

```cpp
long long solve(long long n, long long m) {
    if (n > m) return 0;
    long long t = m + 1, x = 0;
    for (int b = 29; b >= 0; b--) {
        int nb = (n >> b) & 1, tb = (t >> b) & 1;
        if (nb == tb) continue;
        if (nb > tb) break;
        x |= (1LL << b);
    }
    return x;
}
```

**核心启示**：
- **XOR 性质转化**：x ∈ S ⟺ n⊕x ≤ m，将 mex 问题转为"最小化 x 使 n⊕x ≥ t"
- **逐位贪心构造**：从高位到低位，优先取 0 以最小化 x，必要时取 1
- **tight 约束的二进制贪心**：与数位 DP 思路一致，维护是否仍与目标"齐平"

### 题目 16：Koishi Loves Construction（P3599）

> 构造 1~n 的排列，使得：
> - Task1：前缀和模 n 全部互不相同
> - Task2：前缀积模 n 全部互不相同

**分析**：

**Task1（前缀和模 n）**：n=1 特判。n 为奇数且 n>1 时无解。n 为偶数时，构造 [n, 1, n-2, 3, n-4, 5, ...]，前缀和模 n 依次为 0, 1, n-1, 2, n-2, ... 互不相同。

**Task2（前缀积模 n）**：n=1 或 n=4 特判。n 为合数（非 4）时无解。n 为质数时，利用逆元递推：令第 i 个前缀积模 n 恰为 i，则 a[i] = i × inv(i-1) mod n。

```
构造策略（Task1）：
- n=1 → [1]
- n 为奇数且 n>1 → 无解
- n 为偶数 → [n, 1, n-2, 3, n-4, 5, ...]

构造策略（Task2）：
- n=1 → [1]，n=4 → [1, 3, 2, 4]
- n 为合数且 n≠4 → 无解
- n 为质数 → a[1]=1, a[i]=i*inv(i-1) mod n, a[n]=n
  其中 inv(x) = x^{n-2} mod n（费马小定理）
```

**C++ 代码**：

```cpp
int ans[MAXN];

long long qpow(long long a, long long b, long long mod) {
    long long res = 1; a %= mod;
    while (b > 0) {
        if (b & 1) res = res * a % mod;
        a = a * a % mod; b >>= 1;
    }
    return res;
}

bool is_prime(int n) {
    if (n < 2) return false;
    for (int i = 2; (long long)i * i <= n; i++)
        if (n % i == 0) return false;
    return true;
}

// Task1：前缀和模 n 构造
bool solve_task1(int n) {
    if (n == 1) { ans[0] = 1; return true; }
    if (n % 2 == 1) return false;
    for (int i = 0; i < n; i++)
        ans[i] = (i % 2 == 0) ? n - i : i;
    return true;
}

// Task2：前缀积模 n 构造
bool solve_task2(int n) {
    if (n == 1) { ans[0] = 1; return true; }
    if (n == 4) {
        ans[0] = 1; ans[1] = 3; ans[2] = 2; ans[3] = 4;
        return true;
    }
    if (!is_prime(n)) return false;
    ans[0] = 1;
    long long prod = 1;
    for (int i = 1; i < n - 1; i++) {
        long long inv = qpow(prod, n - 2, n);
        long long val = (long long)(i + 1) * inv % n;
        ans[i] = (int)val;
        prod = prod * val % n;
    }
    ans[n - 1] = n;
    return true;
}
```

答案位置：`ans[0..n-1]` 即为构造的排列

**核心启示**：
- **无解条件的严格分析**：Task1 依赖 n 的奇偶性，Task2 依赖 n 的质合性
- **逆元是数论构造的核心武器**：通过逆元递推精确控制前缀积
- **特殊值特判**：n=4 是 Task2 的唯一合数有解情况，需要单独处理
- **两个子任务体现两种构造范式**：对称配对（Task1） vs 逆元递推（Task2）

### 题目 17：Rectangles（Codeforces CF1028C）

> 给 n 个矩形（用左下角和右上角坐标表示），找一个点使其位于至少 n−1 个矩形内部（含边界），或报告不存在。

**分析**：若所有 n 个矩形的交集非空，交集内任意点即为答案。若交集为空，则"破坏者"一定是某个极端矩形——贡献了最大 x1、最小 x2、最大 y1 或最小 y2 的那个。最多 4 个候选，逐个尝试移除后检查剩余 n−1 个矩形的交集。

```
构造策略：
- 找 4 个候选矩形：max(x1) 来源、min(x2) 来源、max(y1) 来源、min(y2) 来源
- 对每个候选，排除它后计算剩余 n-1 个矩形的交集
- 交集 = [max(x1), min(x2)] × [max(y1), min(y2)]
- 若交集非空，返回其中任意点
- 4 个候选都不行 → 无解
```

**C++ 代码**：

```cpp
struct Rect { int x1, y1, x2, y2; };

bool solve(int n, Rect r[], int &ax, int &ay) {
    int cand[4] = {0, 0, 0, 0};
    for (int i = 1; i < n; i++) {
        if (r[i].x1 > r[cand[0]].x1) cand[0] = i;
        if (r[i].x2 < r[cand[1]].x2) cand[1] = i;
        if (r[i].y1 > r[cand[2]].y1) cand[2] = i;
        if (r[i].y2 < r[cand[3]].y2) cand[3] = i;
    }
    for (int c = 0; c < 4; c++) {
        int lx = INT_MIN, ly = INT_MIN, rx = INT_MAX, ry = INT_MAX;
        for (int i = 0; i < n; i++) {
            if (i == cand[c]) continue;
            lx = max(lx, r[i].x1); rx = min(rx, r[i].x2);
            ly = max(ly, r[i].y1); ry = min(ry, r[i].y2);
        }
        if (lx <= rx && ly <= ry) { ax = lx; ay = ly; return true; }
    }
    return false;
}
```

**核心启示**：
- **交集被破坏 → 极端元素是嫌疑人**：只需检查 4 个候选，而非 n 个
- **"去掉一个"的技巧**：允许排除 1 个时，找到最可能需要排除的那个
- **几何交集 = 四维独立取极值**：矩形交集分解为 x 和 y 两个独立区间交

### 题目 18：Ehab and the Expected XOR Problem（Codeforces CF1174D）

> 给定 n 和 x，构造一个尽量长的数组 a（元素在 [1, 2^n−1]），使得**任何非空连续子数组的异或和都不等于 0 或 x**。

**分析**：子数组 [l,r] 的异或和 = pre[r] ⊕ pre[l−1]（前缀异或）。条件等价于：前缀异或集合 S（含 pre[0]=0）中任意两个元素的异或 ≠ 0 且 ≠ x。即 S 中元素互不相同，且不存在 u,v ∈ S 使得 u⊕v = x。贪心构造 S：从 1 到 2^n−1 扫描，若 v 和 v⊕x 都不在 S 中则加入 v。最后 a[i] = pre[i] ⊕ pre[i+1]。

```
构造策略：
- 前缀异或集 S = {0}，标记 ban[0] = ban[x] = true
- 遍历 v = 1 到 2^n-1：
  - 若 ban[v] = false → 加入 S，标记 ban[v] = ban[v⊕x] = true
- 数组长度 = |S| - 1
- a[i] = S[i] ⊕ S[i+1]（相邻前缀异或差）
```

**C++ 代码**：

```cpp
int a[MAXN];

int solve(int n, int x) {
    int sz = 1 << n;
    vector<bool> ban(sz, false);
    vector<int> pre = {0};
    ban[0] = ban[x] = true;
    for (int v = 1; v < sz; v++) {
        if (ban[v]) continue;
        pre.push_back(v);
        ban[v] = ban[v ^ x] = true;
    }
    int len = (int)pre.size() - 1;
    for (int i = 0; i < len; i++)
        a[i] = pre[i] ^ pre[i + 1];
    return len;
}
```

**核心启示**：
- **前缀异或将子数组问题转为集合问题**：子数组异或 = 两个前缀值的异或
- **配对排斥**：加入 v 则排斥 v⊕x，每次排斥恰好 1 个值，最大化集合大小
- **O(2^n) 贪心达到理论上界**：集合大小约 2^(n−1)，即数组长度约 2^(n−1)−1

## 五、常见技巧与思路总结

### 5.1 先判无解，再搞构造

构造题中，**无解判定往往是第一步**，也是最容易被忽略的一步。

```
无解判定思路：
1. 必要条件分析：从约束条件推导出 n 必须满足的性质
2. 反证法：假设有解，推出矛盾
3. 奇偶分析：总和的奇偶性、排列元素个数的奇偶性
4. 模运算：前缀和/积模 n 的取值范围
```

### 5.2 从小数据打表入手

当没有思路时，暴力枚举 n=1~8 的所有解，观察规律是最实用的方法。

```
打表三步法：
Step 1: 写暴力搜索程序，枚举所有合法构造
Step 2: 输出每个 n 的所有解，寻找共性
Step 3: 归纳通用公式或构造方法
```

### 5.3 对称与配对

许多构造题的解具有对称结构：首尾配对、奇偶交替、环形对称。

```cpp
// 首尾配对：(n, 1), (n-1, 2), (n-2, 3), ...
// 每对之和为 n+1，前缀和变化可控
for (int i = 0; i < n; i++)
    a[i] = (i % 2 == 0) ? n - i / 2 : i / 2 + 1;
```

### 5.4 贪心构造的正确性论证

贪心策略的正确性需要证明"每步的局部最优不会影响全局最优"。

| 论证方法 | 适用场景 |
|----------|----------|
| 交换论证 | 证明任何非贪心解都可以通过交换变得不更差 |
| 归纳法 | 证明贪心选择后，子问题仍有最优解 |
| 反证法 | 假设贪心不最优，推出矛盾 |

### 5.5 差分思维转化问题

区间操作类构造题几乎都可以用差分数组简化：

```
原始问题：区间 [l,r] 加减操作 → 让序列满足条件
差分转化：单点修改 d[l] 和 d[r+1] → 让差分数组满足条件
```

### 5.6 分类讨论要穷尽

构造题的分类讨论是重灾区。常见分类维度：

```
分类维度列表：
- n 的奇偶性（最常见）
- n 是否为质数 / 合数
- n 的特殊值（n=1, 2, 4 等）
- 输入中某些量的正负性
- 字符种类（如 X/Y/Z 的出现模式）
```

## 七、终极总结与归纳

### 7.1 构造的本质

**构造题的本质是"在约束空间中寻找一个可行点"**。

不同于优化问题追求最优解，构造题只需找到**任意一个**满足所有约束的解。这意味着：你不需要搜索整个解空间，只需要利用数学性质和观察力，**直接给出一个确定性的构造方案**。构造题考验的不是算法模板的积累，而是**观察力、归纳力和创造力**。

### 7.2 核心要点

1. **无解判定是第一步**：先分析必要条件，确定哪些情况无解
2. **小数据打表是最实用的入手方式**：从 n=1 开始手算，归纳规律
3. **分类讨论要穷尽**：按奇偶、质合、特殊值等维度全面覆盖
4. **对称性是构造的强大工具**：首尾配对、环形排列、奇偶交替
5. **差分/前缀和转化**：将区间操作问题转化为单点问题
6. **验证不可省略**：构造完毕后，务必检查是否满足所有约束条件
