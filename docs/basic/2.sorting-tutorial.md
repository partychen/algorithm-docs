## 一、什么是排序算法

排序算法是计算机科学中最基础、最重要的算法之一。它的目标是将一组数据按照某种确定的顺序（升序或降序）重新排列。排序不仅本身是高频考点，更是许多高级算法的前置步骤——贪心、二分查找、双指针等技术往往依赖数据的有序性。

在竞赛和日常开发中，我们几乎无时无刻不在使用排序：从简单的数组去重，到复杂的多关键字排名、逆序对计数、归并排序分治，排序思想贯穿算法学习的始终。

掌握排序算法，不仅要会调用 `sort()`，更要理解每种排序的**原理、复杂度、稳定性**，以及何时需要**手写排序**（如归并排序求逆序对）。

### 排序算法总览对比表

| 排序算法 | 平均时间复杂度 | 最坏时间复杂度 | 空间复杂度 | 稳定性 | 适用场景 |
|---------|-------------|-------------|----------|-------|---------|
| 冒泡排序 | O(n²) | O(n²) | O(1) | ✅ 稳定 | 教学，小规模数据 |
| 选择排序 | O(n²) | O(n²) | O(1) | ❌ 不稳定 | 教学，小规模数据 |
| 插入排序 | O(n²) | O(n²) | O(1) | ✅ 稳定 | 近乎有序的数据 |
| 快速排序 | O(n log n) | O(n²) | O(log n) | ❌ 不稳定 | 通用排序，竞赛常用 |
| 归并排序 | O(n log n) | O(n log n) | O(n) | ✅ 稳定 | 求逆序对，稳定排序 |
| 堆排序   | O(n log n) | O(n log n) | O(1) | ❌ 不稳定 | 原地排序，优先队列 |
| 计数排序 | O(n + k) | O(n + k) | O(k) | ✅ 稳定 | 值域小的整数排序 |
| 桶排序   | O(n + k) | O(n²) | O(n + k) | ✅ 稳定 | 均匀分布的数据 |
| 基数排序 | O(d(n+k)) | O(d(n+k)) | O(n + k) | ✅ 稳定 | 多位数整数排序 |

> **竞赛中最常用**：`std::sort`（内省排序，快排+堆排+插排混合），手写归并排序（求逆序对）。

## 二、核心思想与前置知识

### 2.1 排序的本质

排序的本质是**消除逆序对**。对于数组 `a[]`，若存在 `i < j` 但 `a[i] > a[j]`，则 `(i, j)` 构成一个逆序对。排序过程就是不断消除逆序对直到数组有序。

- **O(n²) 排序**：每次操作消除 O(1) 个逆序对（冒泡、插入）
- **O(n log n) 排序**：每次操作消除 O(n) 个逆序对（归并、快排）

### 2.2 稳定性

**稳定排序**：相等元素的相对顺序在排序后保持不变。

```
稳定：  [3a, 1, 3b, 2] -> [1, 2, 3a, 3b]  ✅ 3a 仍在 3b 前面
不稳定：[3a, 1, 3b, 2] -> [1, 2, 3b, 3a]  ❌ 顺序改变
```

多关键字排序时稳定性非常重要：先按次要关键字排序，再按主要关键字**稳定排序**，即可实现多关键字排序。

### 2.3 C++ STL 排序工具

```cpp
// std::sort — 不稳定排序，O(n log n)
sort(a, a + n);
sort(a, a + n, greater<int>());  // 降序

// std::stable_sort — 稳定排序，O(n log n)
stable_sort(a, a + n, cmp);

// 自定义比较函数
bool cmp(const Node &a, const Node &b) {
    if (a.score != b.score) return a.score > b.score;  // 分数降序
    return a.id < b.id;  // 分数相同，编号升序
}
```

### 2.4 排序算法的选择策略

```
数据规模 n <= 5000        -> O(n²) 可接受（插入排序等）
数据规模 n <= 10^6        -> 必须 O(n log n)（快排、归并、堆排）
数据规模 n <= 10^7        -> O(n log n) 或 O(n) 线性排序
需要求逆序对             -> 归并排序（手写）
多关键字且要求稳定       -> stable_sort 或手写归并
值域小                   -> 计数排序
```

## 三、排序算法的子模型与分类详解

### 3.1 比较排序

比较排序通过元素之间的**比较操作**来确定顺序，理论下界为 **O(n log n)**。

#### 3.1.1 冒泡排序（Bubble Sort）

冒泡排序的核心思想是**相邻元素两两比较**，每一轮将当前未排序部分的最大值"冒泡"到末尾。经过 n-1 轮后数组有序。

冒泡排序虽然效率低，但概念直观，是理解排序的最佳入门。可以加入"无交换提前退出"的优化。

- **时间复杂度**：O(n²)（最好 O(n)，加优化时）
- **空间复杂度**：O(1)
- **稳定性**：✅ 稳定

```cpp
void bubbleSort(int a[], int n) {
    for (int i = 0; i < n - 1; i++) {
        bool swapped = false;
        for (int j = 0; j < n - 1 - i; j++) {
            if (a[j] > a[j + 1]) {
                swap(a[j], a[j + 1]);
                swapped = true;
            }
        }
        if (!swapped) break;  // 无交换，已有序
    }
}
```

#### 3.1.2 选择排序（Selection Sort）

选择排序每轮从未排序部分中**选出最小元素**，与未排序部分的首元素交换。思路极其简单，但由于交换可能改变相等元素的相对位置，所以不稳定。

- **时间复杂度**：O(n²)
- **空间复杂度**：O(1)
- **稳定性**：❌ 不稳定

```cpp
void selectionSort(int a[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int minIdx = i;
        for (int j = i + 1; j < n; j++) {
            if (a[j] < a[minIdx]) minIdx = j;
        }
        swap(a[i], a[minIdx]);
    }
}
```

#### 3.1.3 插入排序（Insertion Sort）

插入排序将数组分为"已排序"和"未排序"两部分，每次从未排序部分取出一个元素，在已排序部分中找到合适位置插入。类似于打牌时整理手牌的过程。

对于**近乎有序**的数据，插入排序非常高效（接近 O(n)）。STL 的 `sort` 在小区间会切换到插入排序。

- **时间复杂度**：O(n²)（最好 O(n)）
- **空间复杂度**：O(1)
- **稳定性**：✅ 稳定

```cpp
void insertionSort(int a[], int n) {
    for (int i = 1; i < n; i++) {
        int key = a[i], j = i - 1;
        while (j >= 0 && a[j] > key) {
            a[j + 1] = a[j];
            j--;
        }
        a[j + 1] = key;
    }
}
```

#### 3.1.4 快速排序（Quick Sort）

快速排序是**分治法**的经典应用。选择一个**基准元素（pivot）**，将数组分为小于 pivot 和大于 pivot 两部分，递归排序子数组。平均 O(n log n)，但最坏情况（已排序数组 + 固定选首元素为 pivot）退化为 O(n²)。

竞赛中常用**随机化 pivot** 或 **三数取中**避免最坏情况。C++ STL 的 `sort` 基于内省排序（Introsort），融合了快排、堆排和插排的优点。

- **时间复杂度**：平均 O(n log n)，最坏 O(n²)
- **空间复杂度**：O(log n)（递归栈）
- **稳定性**：❌ 不稳定

```cpp
void quickSort(int a[], int l, int r) {
    if (l >= r) return;
    int pivot = a[l + rand() % (r - l + 1)];  // 随机选 pivot
    int i = l, j = r;
    while (i <= j) {
        while (a[i] < pivot) i++;
        while (a[j] > pivot) j--;
        if (i <= j) {
            swap(a[i], a[j]);
            i++; j--;
        }
    }
    quickSort(a, l, j);
    quickSort(a, i, r);
}
```

#### 3.1.5 归并排序（Merge Sort）

归并排序同样基于**分治法**：将数组从中间分成两半，递归排序后**合并两个有序数组**。合并过程是归并排序的核心，也是求逆序对的关键——当左半部分的元素大于右半部分时，产生逆序对。

归并排序是**唯一常用的 O(n log n) 稳定排序**，在需要稳定性或需要求逆序对时必不可少。

- **时间复杂度**：O(n log n)（稳定）
- **空间复杂度**：O(n)
- **稳定性**：✅ 稳定

```cpp
const int MAXN = 5e5 + 5;
int tmp[MAXN];

void mergeSort(int a[], int l, int r) {
    if (l >= r) return;
    int mid = (l + r) / 2;
    mergeSort(a, l, mid);
    mergeSort(a, mid + 1, r);
    int i = l, j = mid + 1, k = l;
    while (i <= mid && j <= r) {
        if (a[i] <= a[j]) tmp[k++] = a[i++];
        else tmp[k++] = a[j++];
    }
    while (i <= mid) tmp[k++] = a[i++];
    while (j <= r) tmp[k++] = a[j++];
    for (int p = l; p <= r; p++) a[p] = tmp[p];
}
```

#### 3.1.6 堆排序（Heap Sort）

堆排序利用**二叉堆**数据结构：先将数组建成最大堆，然后依次取出堆顶（最大值）放到数组末尾，再调整堆。堆排序是**原地 O(n log n)** 排序，不需要额外空间，但常数较大且不稳定。

- **时间复杂度**：O(n log n)
- **空间复杂度**：O(1)
- **稳定性**：❌ 不稳定

```cpp
void heapify(int a[], int n, int i) {
    int largest = i, l = 2 * i + 1, r = 2 * i + 2;
    if (l < n && a[l] > a[largest]) largest = l;
    if (r < n && a[r] > a[largest]) largest = r;
    if (largest != i) {
        swap(a[i], a[largest]);
        heapify(a, n, largest);
    }
}

void heapSort(int a[], int n) {
    for (int i = n / 2 - 1; i >= 0; i--) heapify(a, n, i);
    for (int i = n - 1; i > 0; i--) {
        swap(a[0], a[i]);
        heapify(a, i, 0);
    }
}
```

### 3.2 非比较排序

非比较排序不通过元素间的比较来确定顺序，而是利用元素的**值域信息**，可以突破 O(n log n) 的下界。

#### 3.2.1 计数排序（Counting Sort）

计数排序统计每个值出现的次数，然后依次输出。适用于**值域较小的非负整数**排序。

- **时间复杂度**：O(n + k)，k 为值域大小
- **空间复杂度**：O(k)
- **稳定性**：✅ 稳定（正确实现时）

```cpp
const int MAXV = 1e6 + 5;
int cnt[MAXV];

void countingSort(int a[], int n, int maxVal) {
    memset(cnt, 0, sizeof(cnt));
    for (int i = 0; i < n; i++) cnt[a[i]]++;
    int idx = 0;
    for (int v = 0; v <= maxVal; v++) {
        while (cnt[v]-- > 0) a[idx++] = v;
    }
}
```

#### 3.2.2 桶排序（Bucket Sort）

桶排序将数据分配到若干个**桶**中，每个桶内部单独排序，最后合并所有桶。当数据**均匀分布**时效率最高。

- **时间复杂度**：平均 O(n + k)，最坏 O(n²)
- **空间复杂度**：O(n + k)
- **稳定性**：✅ 稳定（取决于桶内排序）

```cpp
const int MAXB = 1005;
vector<int> bucket[MAXB];

void bucketSort(int a[], int n, int bucketSize) {
    int minVal = *min_element(a, a + n);
    int maxVal = *max_element(a, a + n);
    int bucketCnt = (maxVal - minVal) / bucketSize + 1;
    for (int i = 0; i < n; i++)
        bucket[(a[i] - minVal) / bucketSize].push_back(a[i]);
    int idx = 0;
    for (int i = 0; i < bucketCnt; i++) {
        sort(bucket[i].begin(), bucket[i].end());
        for (int v : bucket[i]) a[idx++] = v;
    }
}
```

#### 3.2.3 基数排序（Radix Sort）

基数排序从最低位到最高位，对每一位执行一次**稳定的计数排序**。适用于**多位数整数**排序，位数为 d 时复杂度为 O(d(n+k))。

- **时间复杂度**：O(d(n + k))
- **空间复杂度**：O(n + k)
- **稳定性**：✅ 稳定

```cpp
const int MAXN = 1e5 + 5;

void radixSort(int a[], int n) {
    int maxVal = *max_element(a, a + n);
    int out[MAXN];
    for (int exp = 1; maxVal / exp > 0; exp *= 10) {
        int cnt[10] = {};
        for (int i = 0; i < n; i++) cnt[(a[i] / exp) % 10]++;
        for (int i = 1; i < 10; i++) cnt[i] += cnt[i - 1];
        for (int i = n - 1; i >= 0; i--)
            out[--cnt[(a[i] / exp) % 10]] = a[i];
        for (int i = 0; i < n; i++) a[i] = out[i];
    }
}
```

### 3.3 自定义排序

#### 3.3.1 结构体排序

竞赛中经常需要对**结构体数组**按自定义规则排序。通常通过重载 `<` 运算符或编写比较函数实现。

```cpp
struct Student {
    int id, score;
    bool operator<(const Student &o) const {
        if (score != o.score) return score > o.score;  // 分数降序
        return id < o.id;  // 同分则编号升序
    }
};
Student stu[MAXN];
// 使用：sort(stu, stu + n);
```

#### 3.3.2 多关键字排序

多关键字排序的关键在于**比较函数的书写**。常用的方式有：

1. **if-else 链**：逐级比较各关键字
2. **pair/tuple**：利用 STL 的字典序比较
3. **稳定排序多次**：先排次要关键字，再稳定排序主要关键字

```cpp
// 方法1：比较函数
bool cmp(const Node &a, const Node &b) {
    if (a.x != b.x) return a.x < b.x;
    if (a.y != b.y) return a.y > b.y;
    return a.z < b.z;
}

// 方法2：pair 自动字典序
pair<int, int> p[MAXN];
// sort(p, p + n); 自动先比 first，再比 second
```

### 3.4 排序应用

#### 3.4.1 逆序对（归并排序求）

逆序对计数是归并排序的经典应用。在归并过程中，当左半部分的 `a[i] > a[j]`（j 在右半部分）时，左半部分从 i 到 mid 的所有元素都与 a[j] 构成逆序对，贡献 `mid - i + 1` 个逆序对。

```cpp
long long cnt;
int tmp[MAXN];

void mergeCount(int a[], int l, int r) {
    if (l >= r) return;
    int mid = (l + r) / 2;
    mergeCount(a, l, mid);
    mergeCount(a, mid + 1, r);
    int i = l, j = mid + 1, k = l;
    while (i <= mid && j <= r) {
        if (a[i] <= a[j]) tmp[k++] = a[i++];
        else {
            cnt += mid - i + 1;  // 逆序对计数
            tmp[k++] = a[j++];
        }
    }
    while (i <= mid) tmp[k++] = a[i++];
    while (j <= r) tmp[k++] = a[j++];
    for (int p = l; p <= r; p++) a[p] = tmp[p];
}
```

#### 3.4.2 排序后贪心

许多贪心问题的第一步就是排序。排序将问题转化为有序结构，使得贪心策略可以正确执行。常见模式：

- **活动选择**：按结束时间排序
- **区间合并**：按左端点排序
- **任务调度**：按截止时间或权重排序
- **拼数问题**：自定义字符串比较排序

### 3.5 子模型对比总表

| 分类 | 代表算法 | 核心思想 | 竞赛常用度 | 手写频率 |
|------|---------|---------|-----------|---------|
| 比较排序 | 快排、归并、堆排 | 元素间比较 | ⭐⭐⭐⭐⭐ | 归并（逆序对）常手写 |
| 非比较排序 | 计数排序 | 值域统计 | ⭐⭐⭐ | 值域小时常用 |

| 排序应用 | 逆序对、贪心 | 排序为工具 | ⭐⭐⭐⭐ | 逆序对必须手写 |

## 四、16 道经典例题详解

### 🟢 入门级 Level 1：基础排序与去重

### 题目 1：【模板】快速排序（洛谷 P1177）

> 给定 n 个整数，将它们从小到大排序后输出。

**分析**：排序模板题，直接使用 `sort` 即可。关键在于理解 STL 排序的使用方式和复杂度保证。本题也可用于练习手写快速排序。

```
核心操作：对数组进行升序排序
算法选择：std::sort（O(n log n)）
```

**C++ 代码**：

```cpp
const int MAXN = 1e5 + 5;
int a[MAXN];

void solve(int n) {
    sort(a, a + n);
}
```

答案位置：排序后的数组 `a[0..n-1]`

**核心启示**：
- **`sort` 是竞赛中最常用的排序方式**，底层是内省排序，平均 O(n log n)
- **熟练掌握 STL 排序是一切排序题的基础**，后续题目都建立在此之上

---

### 题目 2：明明的随机数（洛谷 P1059）

> 明明生成了 n 个 1 到 1000 之间的随机整数，要求去掉重复数字后排序输出。

**分析**：经典的**排序 + 去重**问题。由于值域仅 1~1000，可以用计数排序（桶）O(n + k) 解决，也可以先排序再去重。计数排序天然完成去重和排序。

```
方法一：计数排序（桶标记）
方法二：sort + unique
核心：值域小 -> 计数排序最优
```

**C++ 代码**：

```cpp
bool exist[1001];

void solve(int n, int a[]) {
    memset(exist, false, sizeof(exist));
    for (int i = 0; i < n; i++) exist[a[i]] = true;
    for (int v = 1; v <= 1000; v++) {
        if (exist[v]) {
            // 输出 v
        }
    }
}
```

答案位置：所有 `exist[v] == true` 的 v，按升序输出

**核心启示**：
- **值域小时优先考虑计数排序/桶**，时间复杂度可降至 O(n + k)
- **排序 + 去重是基础组合技**，`sort + unique` 是通用方案，计数排序是特化方案
- **分析值域范围是选择算法的关键**

### 🟡 进阶级 Level 2：自定义排序与多关键字

### 题目 3：分数线划定（洛谷 P1068）

> n 个考生，每人有报名号和成绩。先按成绩降序排，成绩相同按报名号升序排。取前 m 名的最低分数线（m 为计划录取人数的 150%），所有达到分数线的考生都录取。求分数线和录取人数。

**分析**：典型的**多关键字排序**题。定义比较函数，按成绩降序、报名号升序排序。分数线为第 m 名的成绩（m 取计划数的 150% 向下取整），然后统计所有不低于分数线的考生数。

```
排序规则：成绩降序，同分报名号升序
分数线确定：sorted[m-1].score
录取人数：所有 score >= 分数线的人
```

**C++ 代码**：

```cpp
const int MAXN = 5005;
struct Student {
    int id, score;
};
Student stu[MAXN];

bool cmp(const Student &a, const Student &b) {
    if (a.score != b.score) return a.score > b.score;
    return a.id < b.id;
}

void solve(int n, int planM) {
    int m = (int)(planM * 1.5);
    sort(stu, stu + n, cmp);
    int line = stu[m - 1].score;
    int cnt = 0;
    for (int i = 0; i < n; i++) {
        if (stu[i].score >= line) cnt++;
    }
    // 输出 line 和 cnt
```

**分析**：这是冒泡排序的直接应用。冒泡排序每次比较并交换相邻逆序元素，总交换次数恰好等于**逆序对数**。由于 n ≤ 10000，O(n²) 的冒泡排序直接模拟计数即可。与 P1774 相同的结论（最少交换次数 = 逆序对数），但数据规模允许暴力。

```
核心定理：冒泡排序交换次数 = 逆序对数
数据规模：n ≤ 10000，O(n²) 可接受
与 P1774 对比：大数据用归并 O(n log n)，小数据冒泡直接模拟
```

**C++ 代码**：

```cpp
const int MAXN = 1e4 + 5;
int a[MAXN];

int solve(int n) {
    int cnt = 0;
    for (int i = 0; i < n - 1; i++)
        for (int j = 0; j < n - 1 - i; j++)
            if (a[j] > a[j + 1]) {
                swap(a[j], a[j + 1]);
                cnt++;
            }
    return cnt;
}
```

答案位置：`cnt`（冒泡排序的总交换次数）

**核心启示**：

- **小数据量（n ≤ 10^4）时 O(n²) 完全可行**，无需过度优化
- **理解算法复杂度边界**：n ≤ 10^4 时 n² = 10^8，刚好可过；n ≥ 10^5 则必须 O(n log n)

### 题目 9：瑞士轮（洛谷 P1309）

> 2n 个选手进行 r 轮比赛。每轮按当前分数降序排列（同分编号小的在前），第 1 名与第 2 名比，第 3 名与第 4 名比……赢者加 1 分（实力值大的赢，相同则编号小的赢）。求 r 轮后排名第 q 的选手编号。

**分析**：朴素做法每轮 `sort` 一次，复杂度 O(r * n log n)，可能超时。关键优化：每轮比赛后，赢家和输家各自**已经有序**（因为赛前有序，且配对相邻），因此可以用**归并**将两个有序序列合并为一个有序序列，每轮 O(n)。

这是排序与**归并思想**结合的经典题。总复杂度从 O(r * n log n) 降至 O(n log n + r * n)。

```
核心优化：赢家序列有序 + 输家序列有序 -> 归并 O(n)
初始：sort 一次 O(n log n)
每轮：模拟比赛 + 归并 O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 2e5 + 5;
struct Player {
    int id, score, strength;
};
Player all[MAXN], win[MAXN], lose[MAXN];

bool cmp(const Player &a, const Player &b) {
    if (a.score != b.score) return a.score > b.score;
    return a.id < b.id;
}

void playRound(int n) {
    int wc = 0, lc = 0;
    for (int i = 0; i < 2 * n; i += 2) {
        Player &a = all[i], &b = all[i + 1];
        if (a.strength > b.strength ||
            (a.strength == b.strength && a.id < b.id)) {
            a.score++;
            win[wc++] = a;
            lose[lc++] = b;
        } else {
            b.score++;
            win[wc++] = b;
            lose[lc++] = a;
        }
    }
    int i = 0, j = 0, k = 0;
    while (i < wc && j < lc) {
        if (cmp(win[i], lose[j])) all[k++] = win[i++];
        else all[k++] = lose[j++];
    }
    while (i < wc) all[k++] = win[i++];
    while (j < lc) all[k++] = lose[j++];
}
```

答案位置：r 轮后 `all[q-1].id`

**核心启示**：
- **利用局部有序性可以用归并代替排序**，将 O(n log n) 降为 O(n)
- **竞赛中要敏锐发现"已有序"的性质**，这是优化排序的关键
- **瑞士轮是归并思想在模拟题中的经典应用**

### 题目 10：逆序对（洛谷 P1908）

> 给定一个长度为 n 的序列，求其中逆序对的个数。逆序对定义：`i < j` 且 `a[i] > a[j]`。

**分析**：暴力枚举所有 (i, j) 对是 O(n²)，无法通过大数据。经典做法是**归并排序**：在合并两个有序子数组时，当左半部分 `a[i] > a[j]`（j 在右半部分），说明左半部分从 i 到 mid 的元素都与 a[j] 构成逆序对，贡献 `mid - i + 1`。

```
核心思想：归并排序过程中统计逆序对
逆序对计数：左半元素 > 右半元素时，cnt += mid - i + 1
时间复杂度：O(n log n)
```

**C++ 代码**：

```cpp
const int MAXN = 5e5 + 5;
int a[MAXN], tmp[MAXN];
long long cnt;

void mergeCount(int l, int r) {
    if (l >= r) return;
    int mid = (l + r) / 2;
    mergeCount(l, mid);
    mergeCount(mid + 1, r);
    int i = l, j = mid + 1, k = l;
    while (i <= mid && j <= r) {
        if (a[i] <= a[j]) {
            tmp[k++] = a[i++];
        } else {
            cnt += mid - i + 1;
            tmp[k++] = a[j++];
        }
    }
    while (i <= mid) tmp[k++] = a[i++];
    while (j <= r) tmp[k++] = a[j++];
    for (int p = l; p <= r; p++) a[p] = tmp[p];
}
```

答案位置：`cnt`（调用前置零：`cnt = 0; mergeCount(0, n - 1);`）

**核心启示**：
- **归并排序求逆序对是分治思想的经典应用**，必须熟练掌握
- **关键代码只有一行**：`cnt += mid - i + 1`，理解"左侧剩余元素全部大于右侧当前元素"
- **注意用 `long long`**，逆序对数量可能达到 n(n-1)/2

### 题目 11：三元上升子序列（洛谷 P1637）

> 给定长度为 n 的序列 a[]，求满足 i < j < k 且 a[i] < a[j] < a[k] 的三元组 (i, j, k) 的个数。

**分析**：枚举中间元素 j，答案等于 Σ(left[j] × right[j])，其中 left[j] 为 j 左侧严格小于 a[j] 的元素数，right[j] 为 j 右侧严格大于 a[j] 的元素数。利用**树状数组**维护前缀计数：正序扫描求 left，逆序扫描求 right。这是逆序对思想从"二元"到"三元"的推广。

```
核心思路：枚举中间元素，left[j] × right[j]
树状数组：正扫求左侧小于 a[j] 的个数，逆扫求右侧大于 a[j] 的个数
时间复杂度：O(n log n)（需离散化）
```

**C++ 代码**：

```cpp
const int MAXN = 3e4 + 5;
int a[MAXN], tree[MAXN], n;
long long leftCnt[MAXN], rightCnt[MAXN];

void add(int x, int v) { for (; x <= n; x += x & -x) tree[x] += v; }
int ask(int x) { int s = 0; for (; x; x -= x & -x) s += tree[x]; return s; }

long long solve() {
    memset(tree, 0, sizeof(tree));
    for (int i = 0; i < n; i++) {
        leftCnt[i] = ask(a[i] - 1);
        add(a[i], 1);
    }
    memset(tree, 0, sizeof(tree));
    for (int i = n - 1; i >= 0; i--) {
        rightCnt[i] = ask(n) - ask(a[i]);
        add(a[i], 1);
    }
    long long ans = 0;
    for (int i = 0; i < n; i++) ans += leftCnt[i] * rightCnt[i];
    return ans;
}
```

答案位置：`ans`（所有三元上升子序列的数量）

**核心启示**：
- **"枚举中间元素"是三元组问题的通用策略**，将 O(n³) 降至 O(n × 查询复杂度)
- **树状数组的"前缀计数"功能**可以高效统计某一侧小于/大于某值的元素个数
- **本题是逆序对的推广**：逆序对统计"二元关系"，本题统计"三元关系"

### 题目 12：中位数（洛谷 P1168）

> 依次读入 n 个正整数。每读入一个奇数位置的数后，输出当前已读入的所有数的中位数。

**分析**：经典的**对顶堆**（第 k 大问题的典型方案）。维护大根堆存较小的一半，小根堆存较大的一半，保持大根堆的大小 ≥ 小根堆。每次插入后调整平衡，大根堆的堆顶即为中位数。对顶堆本质上是用两个堆维护一个动态有序集合的"中间位置"。

```
核心结构：对顶堆（大根堆 + 小根堆）
不变量：maxH.size() == minH.size() 或 maxH.size() == minH.size() + 1
中位数：maxH.top()
时间复杂度：O(n log n)
```

**C++ 代码**：

```cpp
priority_queue<int> maxH;
priority_queue<int, vector<int>, greater<int>> minH;

void insert(int x) {
    if (maxH.empty() || x <= maxH.top()) maxH.push(x);
    else minH.push(x);
    while (maxH.size() > minH.size() + 1) {
        minH.push(maxH.top()); maxH.pop();
    }
    while (maxH.size() < minH.size()) {
        maxH.push(minH.top()); minH.pop();
    }
}

int getMedian() { return maxH.top(); }
```

答案位置：每读入奇数位置的元素后调用 `getMedian()` 输出

**核心启示**：
- **对顶堆是动态维护中位数/第 k 大的标准方案**，插入和查询均为 O(log n)
- **两个堆的"平衡调整"是核心操作**，保证大根堆大小 ≥ 小根堆大小且差 ≤ 1
- **对顶堆思想可推广**：维护第 k 大只需调整两个堆的大小比例

### 题目 14：火柴排队（洛谷 P1966）

> 两列火柴，每列 n 根，高度分别为 `a[i]` 和 `b[i]`。只能对**同一列**进行相邻交换。求最少交换次数使得 `sum(a[i] - b[i])^2` 最小。答案对 10^8 - 3 取模。

**分析**：数学分析可知，当 a 和 b 的**相对排名一致**时，`sum(a[i] - b[i])^2` 最小（排序不等式）。具体做法：

1. 对 a 和 b 分别求排名（离散化）
2. 建立映射：令 `c[rank_a[i]] = rank_b[i]`
3. 问题转化为：将 c 数组排成 1, 2, ..., n 的最少相邻交换次数 = c 的逆序对数

```
数学原理：排序不等式 -> 同排名位置配对最优
转化步骤：排名映射 -> 逆序对计数
时间复杂度：O(n log n)
```

**C++ 代码**：

```cpp
const int MAXN = 1e5 + 5;
const int MOD = 1e8 - 3;
int a[MAXN], b[MAXN], c[MAXN], tmp[MAXN];
int rankA[MAXN], rankB[MAXN], idxA[MAXN], idxB[MAXN];
long long ans;

void mergeCount(int l, int r) {
    if (l >= r) return;
    int mid = (l + r) / 2;
    mergeCount(l, mid);
    mergeCount(mid + 1, r);
    int i = l, j = mid + 1, k = l;
    while (i <= mid && j <= r) {
        if (c[i] <= c[j]) {
            tmp[k++] = c[i++];
        } else {
            ans = (ans + mid - i + 1) % MOD;
            tmp[k++] = c[j++];
        }
    }
    while (i <= mid) tmp[k++] = c[i++];
    while (j <= r) tmp[k++] = c[j++];
    for (int p = l; p <= r; p++) c[p] = tmp[p];
}

void solve(int n) {
    // 对 a 排序，记录排名
    for (int i = 0; i < n; i++) idxA[i] = i;
    sort(idxA, idxA + n, [](int x, int y) { return a[x] < a[y]; });
    for (int i = 0; i < n; i++) rankA[idxA[i]] = i;

    // 对 b 排序，记录排名
    for (int i = 0; i < n; i++) idxB[i] = i;
    sort(idxB, idxB + n, [](int x, int y) { return b[x] < b[y]; });
    for (int i = 0; i < n; i++) rankB[idxB[i]] = i;

    // 构造 c：c[rankA[i]] = rankB[i]
    for (int i = 0; i < n; i++) c[rankA[i]] = rankB[i];

    ans = 0;
    mergeCount(0, n - 1);
    ans %= MOD;
}
```

答案位置：`ans`

**核心启示**：
- **排序不等式是将"最小化差的平方和"转化为"排名一致"的数学基础**
- **将两个数组的配对问题转化为单个数组的逆序对问题**，是非常精妙的转化技巧
- **离散化（排名映射）是处理"相对顺序"问题的通用手段**
- **逆序对计数的取模要在累加时进行**，避免溢出

### 题目 15：【模板】普通平衡树（洛谷 P3369）

> 维护一个有序集合，支持六种操作：插入 x、删除 x、查询 x 的排名、查询排名为 k 的数、求 x 的前驱、求 x 的后继。

**分析**：这是**有序集合维护**的模板题，是排序思想的数据结构化。需要支持动态插入删除和顺序查询。竞赛中常用 C++ `pb_ds` 库的 `tree`（策略树）实现，它自带 `order_of_key`（查排名）和 `find_by_order`（查第 k 个）。用 `pair<int,int>` 处理重复元素，第二关键字用时间戳区分。

```
核心结构：pb_ds::tree（红黑树 + 顺序统计）
关键操作：order_of_key（排名查询）、find_by_order（第 k 小）
处理重复：pair<value, timestamp> 保证唯一性
时间复杂度：所有操作 O(log n)
```

**C++ 代码**：

```cpp
typedef tree<pair<int,int>, null_type, less<pair<int,int>>,
    rb_tree_tag, tree_order_statistics_node_update> OSTree;
OSTree tr;
int timer = 0;

void ins(int x)      { tr.insert({x, ++timer}); }
void del(int x)      { tr.erase(tr.lower_bound({x, 0})); }
int rnk(int x)       { return tr.order_of_key({x, 0}) + 1; }
int kth(int k)       { return tr.find_by_order(k - 1)->first; }
int pre(int x)       { return prev(tr.lower_bound({x, 0}))->first; }
int nxt(int x)       { return tr.lower_bound({x + 1, 0})->first; }
```

答案位置：根据操作类型调用对应函数，返回值即为答案

**核心启示**：
- **`pb_ds::tree` 是竞赛中维护有序集合的利器**，代码简洁且功能强大
- **`pair` + 时间戳是处理重复元素的通用技巧**，保证插入的每个元素唯一
- **排序的本质延伸**：从"一次性排序"到"动态维护有序结构"

### 题目 16：排序（洛谷 P1975）

> 给定一个 1~n 的排列，支持操作：交换位置 x 和 y 的元素，每次操作后输出当前序列的逆序对数。

**分析**：初始逆序对数用**树状数组**求解：从左到右扫描，每个元素的贡献 = 已插入元素中大于它的个数。交换 a[x] 和 a[y]（设 x < y）后，只有 x 和 y 之间的元素对逆序对数产生影响。遍历中间元素更新增减量，复杂度 O(n) 每次交换。总复杂度 O(n log n + mn)。

```
初始化：树状数组 O(n log n) 求逆序对
交换更新：遍历 (x, y) 之间的元素计算 delta
核心公式：中间元素在 (min, max) 范围内时贡献 ±2
时间复杂度：O(n log n + mn)
```

**C++ 代码**：

```cpp
const int MAXN = 2e4 + 5;
int a[MAXN], tr[MAXN], n;
long long inv;

void add(int x, int v) { for (; x <= n; x += x & -x) tr[x] += v; }
int ask(int x) { int s = 0; for (; x; x -= x & -x) s += tr[x]; return s; }

void init() {
    inv = 0; memset(tr, 0, sizeof(tr));
    for (int i = 0; i < n; i++) { inv += i - ask(a[i]); add(a[i], 1); }
}

void swapAndQuery(int x, int y) {
    if (x > y) swap(x, y);
    int u = a[x], v = a[y];
    for (int i = x + 1; i < y; i++) {
        if (a[i] > u && a[i] < v) inv += 2;
        else if (a[i] < u && a[i] > v) inv -= 2;
    }
    if (u < v) inv++; else if (u > v) inv--;
    swap(a[x], a[y]);
    // 输出 inv
}
```

答案位置：每次操作后输出 `inv`

**核心启示**：
- **树状数组求逆序对**是归并排序之外的另一种经典方法，更适合需要动态修改的场景
- **交换两元素对逆序对的影响只涉及中间元素**，利用这一性质可 O(n) 更新
- **与归并排序对比**：归并求逆序对是"一次性"，树状数组方案支持动态操作

## 五、常见技巧与思路总结

### 5.1 STL sort 的正确使用

```cpp
// 升序（默认）
sort(a, a + n);
// 降序
sort(a, a + n, greater<int>());
// 自定义比较（严格弱序！）
sort(a, a + n, [](int x, int y) { return x > y; });
```

**注意**：比较函数必须满足**严格弱序**（Strict Weak Ordering），即 `cmp(a, a)` 必须返回 `false`。违反此规则会导致未定义行为（RE 或 WA）。

### 5.2 去重的两种方式

```cpp
// 方法1：sort + unique + erase
sort(a.begin(), a.end());
a.erase(unique(a.begin(), a.end()), a.end());

// 方法2：set 自动去重
set<int> s(a.begin(), a.end());
```

### 5.3 离散化

将大值域映射到小的连续整数范围，常用于树状数组、线段树等数据结构前的预处理。

```cpp
int b[MAXN];  // b 是 a 的副本
void discretize(int a[], int n) {
    copy(a, a + n, b);
    sort(b, b + n);
    int m = unique(b, b + n) - b;
    for (int i = 0; i < n; i++)
        a[i] = lower_bound(b, b + m, a[i]) - b + 1;
}
```

### 5.4 结构体排序的三种写法

```cpp
// 1. 重载 < 运算符
struct Node {
    int x, y;
    bool operator<(const Node &o) const { return x < o.x; }
};

// 2. 外部比较函数
bool cmp(const Node &a, const Node &b) { return a.x < b.x; }

// 3. Lambda 表达式
sort(nodes, nodes + n, [](const Node &a, const Node &b) {
    return a.x < b.x;
});
```

### 5.5 逆序对的两种求法

| 方法 | 时间复杂度 | 实现难度 | 适用场景 |
|------|-----------|---------|---------|
| 归并排序 | O(n log n) | ⭐⭐ | 通用，代码短 |
| 树状数组 | O(n log n) | ⭐⭐⭐ | 需要离散化，但更灵活 |

### 5.6 判断排序题型的信号词

| 信号词 | 对应操作 |
|-------|---------|
| "从大到小/从小到大排列" | 直接 sort |
| "去除重复" | sort + unique 或 计数排序 |
| "多个条件排序" | 自定义 cmp 函数 |
| "拼接最大/最小" | 字符串拼接比较 |
| "逆序对/交换次数" | 归并排序 |
| "第 k 大/小" | nth_element 或 快速选择 |
| "稳定排序" | stable_sort 或 归并排序 |

### 6.3 常见错误与避坑

1. **cmp 不满足严格弱序**：`return a <= b;` ❌ 应为 `return a < b;` ✅
2. **逆序对忘用 long long**：n = 5*10^5 时逆序对最多约 1.25*10^11
3. **归并排序忘拷回原数组**：合并后需要 `for (p = l; p <= r; p++) a[p] = tmp[p];`
4. **自定义排序边界**：比较函数中两个关键字都相同时应返回 `false`
5. **计数排序值域估计错误**：注意负数需要偏移

## 七、终极总结与归纳

### 7.1 排序算法的本质

**排序的本质是消除逆序对，建立有序结构。** 无论是简单的数组排序，还是复杂的逆序对计数、多关键字排名，核心都是对"序"的理解与操作。排序不仅是一种算法，更是一种**思维工具**——它将无序的混沌转化为有序的结构，为后续的二分查找、贪心策略、双指针技巧奠定基础。

### 7.2 核心要点

1. **掌握 STL sort**：90% 的排序题只需要 `sort` + 自定义比较函数
2. **理解稳定性**：多关键字排序时，稳定性决定了是否需要 `stable_sort`
3. **归并排序必须手写**：求逆序对是归并排序的独特应用，无法用 STL 替代
4. **自定义比较函数必须满足严格弱序**：违反此规则是竞赛中最常见的 RE 原因之一
5. **值域分析决定算法选择**：值域小用计数排序，通用场景用 `sort`，求逆序对用归并
6. **逆序对 = 最少相邻交换次数**：这个等价关系是许多困难题的突破口
