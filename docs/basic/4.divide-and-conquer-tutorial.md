## 一、什么是分治

**分治**（Divide and Conquer），顾名思义就是"分而治之"：把一个规模较大的问题拆分成若干个规模较小的子问题，分别求解后再将结果合并，从而得到原问题的解。

分治的本质可以用一句话概括：

> **大事化小，小事化了，最后汇总。**

分治是递归思想最直接的应用。几乎所有分治算法都具有天然的递归结构：不断把问题"掰成两半"，直到足够小可以直接解决，再一层一层地把答案"拼回来"。

### 分治的三个阶段

```
原问题 P(n)
    │
    ├── 分解(Divide)  ──→  子问题 P(n/2), P(n/2)
    │                            │           │
    │                      递归求解      递归求解
    │                            │           │
    │                            ▼           ▼
    └── 合并(Combine) ←── 子解 S1    +   子解 S2
                                │
                                ▼
                           原问题的解 S
```

### 分治与其他算法范式对比

| 算法范式 | 核心思路 | 子问题关系 | 典型代表 |
|----------|----------|------------|----------|
| 分治 | 分解 → 递归求解 → 合并 | 子问题独立、不重叠 | 归并排序、快速幂 |
| 动态规划 | 分解 → 记忆化/递推 → 合并 | 子问题重叠、共享 | 背包、LIS |
| 贪心 | 每步选局部最优 | 无需回溯 | 区间调度、Huffman |
| 搜索/回溯 | 穷举所有可能 | 树形展开 | N 皇后、全排列 |

分治与动态规划最大的区别在于：分治的子问题**互不重叠**，不需要记忆化；而 DP 的子问题**大量重叠**，必须记忆化才能高效。

## 二、核心思想与前置知识

### 2.1 分治三步曲

任何分治算法都包含以下三步：

1. **分解（Divide）**：将原问题分解为若干规模更小的同类子问题。
2. **求解（Conquer）**：递归地解决每个子问题；当子问题足够小时，直接求解（递归基/边界）。
3. **合并（Combine）**：将子问题的解合并为原问题的解。

```
// 分治通用框架
void solve(int l, int r) {
    if (l >= r) return;        // 递归基：问题足够小
    int mid = (l + r) / 2;    // 分解
    solve(l, mid);             // 递归求解左半
    solve(mid + 1, r);         // 递归求解右半
    merge(l, mid, r);          // 合并
}
```

**关键洞察**：分治的难点通常不在"分"，而在"合"——如何高效地将两个子问题的解合并成原问题的解，是每道题的核心。

### 2.2 递归与分治的关系

递归是分治的实现手段，但不是所有递归都是分治：

- **分治要求**：子问题与原问题**同构**（结构相同，规模更小），且子问题之间**独立**。
- **一般递归**：子问题不一定同构，也可能相互依赖。

分治的递归通常是**平衡分割**（如对半分），这样递归树的深度为 `O(log n)`，是分治高效的根本原因。

### 2.3 主定理（Master Theorem）

主定理用于分析分治算法的时间复杂度。设递推关系为：

```
T(n) = a · T(n/b) + O(n^d)

其中：
  a = 子问题个数
  b = 子问题规模缩小倍数
  d = 合并步骤的复杂度指数
```

则复杂度为：

| 条件 | 复杂度 | 直觉理解 |
|------|--------|----------|
| `d > log_b(a)` | `O(n^d)` | 合并主导，每层减少 |
| `d = log_b(a)` | `O(n^d · log n)` | 各层均等，乘 log |
| `d < log_b(a)` | `O(n^(log_b(a)))` | 分裂主导，叶子层决定 |

**经典例子**：

| 算法 | 递推式 | a, b, d | 复杂度 |
|------|--------|---------|--------|
| 归并排序 | `T(n) = 2T(n/2) + O(n)` | 2, 2, 1 | `O(n log n)` |
| 快速幂 | `T(n) = T(n/2) + O(1)` | 1, 2, 0 | `O(log n)` |
| Karatsuba 乘法 | `T(n) = 3T(n/2) + O(n)` | 3, 2, 1 | `O(n^1.585)` |
| 平面最近点对 | `T(n) = 2T(n/2) + O(n log n)` | — | `O(n log² n)` |

### 2.4 分治的适用条件

一个问题适合用分治解决，通常满足以下条件：

1. **可分解性**：问题可以分成若干规模更小的**同类**子问题。
2. **独立性**：子问题之间相互独立，不存在依赖关系。
3. **可合并性**：子问题的解可以高效合并为原问题的解。
4. **递归基明确**：当规模足够小时，问题可以直接求解。

### 2.5 分治的复杂度优势

分治之所以高效，关键在于：

- **减少冗余计算**：每次规模减半，总工作量从 `O(n²)` 降为 `O(n log n)` 甚至 `O(log n)`。
- **递归树浅**：对半分的递归深度只有 `O(log n)` 层。
- **合并高效**：好的合并算法能在线性或近线性时间内完成。

## 三、分治的子模型与分类详解

### 3.1 递归分治（经典分治）

**特征**：将问题按规模对半分（或按某种方式分解），递归求解后直接合并或直接得到答案。

```
适用信号：问题规模可指数级缩减（每次减半）
实现要点：递归 + 分解 + 合并
经典问题：快速幂、快速排序、二分搜索
复杂度：通常 O(log n) 或 O(n log n)
```

**快速幂模板**：

```cpp
long long power(long long a, long long b, long long p) {
    long long res = 1;
    a %= p;
    while (b > 0) {
        if (b & 1) res = res * a % p;
        a = a * a % p;
        b >>= 1;
    }
    return res;
}
```

**快速排序核心（划分 + 递归）**：

```cpp
int a[MAXN];

void qsort(int l, int r) {
    if (l >= r) return;
    int i = l, j = r, pivot = a[(l + r) / 2];
    while (i <= j) {
        while (a[i] < pivot) i++;
        while (a[j] > pivot) j--;
        if (i <= j) swap(a[i++], a[j--]);
    }
    qsort(l, j);
    qsort(i, r);
}
```

### 3.2 归并分治

**特征**：在归并排序的过程中，利用"合并"阶段统计跨越中点的信息。归并排序本身就是分治的经典实现，而许多计数问题可以嵌入归并过程中解决。

```
适用信号：统计跨越左右两半的某种关系（逆序对、跨区间计数等）
实现要点：归并排序框架 + 在合并时统计
经典问题：逆序对、归并排序
复杂度：O(n log n)
```

**归并排序 + 逆序对统计模板**：

```cpp
int a[MAXN], tmp[MAXN];
long long cnt;

void merge_sort(int l, int r) {
    if (l >= r) return;
    int mid = (l + r) / 2;
    merge_sort(l, mid);
    merge_sort(mid + 1, r);
    int i = l, j = mid + 1, k = l;
    while (i <= mid && j <= r) {
        if (a[i] <= a[j]) tmp[k++] = a[i++];
        else { tmp[k++] = a[j++]; cnt += mid - i + 1; }
    }
    while (i <= mid) tmp[k++] = a[i++];
    while (j <= r) tmp[k++] = a[j++];
    for (int p = l; p <= r; p++) a[p] = tmp[p];
}
```

### 3.3 几何分治

**特征**：在平面几何问题中，将点集按某一维坐标排序后对半分，分别处理左右两侧，再处理跨越分界线的部分。

```
适用信号：平面点集上的最优化问题（最近点对、凸包等）
实现要点：按 x 排序 + 分治 + 带状区域处理
经典问题：平面最近点对
复杂度：O(n log n) 或 O(n log² n)
```

**平面最近点对核心思路**：

```cpp
struct Point { double x, y; };
Point pts[MAXN], strip[MAXN];

double closest(int l, int r) {
    if (r - l <= 2) return brute_force(l, r);
    int mid = (l + r) / 2;
    double d = min(closest(l, mid), closest(mid + 1, r));
    // 收集带状区域内的点，按 y 排序后暴力比较
    // 每个点最多比较 7 个邻居
    return d;
}
```

### 3.4 分块（分治思想的延伸）

**特征**：将数组分成 `√n` 个块，每个块内部独立维护。操作时，整块用预处理信息快速回答，边角块暴力处理。虽然不是严格的递归分治，但其"分而治之"的核心思想与分治一脉相承。

```
适用信号：需要同时支持区间修改和区间查询，且不易用线段树/树状数组
实现要点：分块 + 块内排序/预处理 + 散块暴力
经典问题：教主的魔法、区间众数
复杂度：单次操作 O(√n) 或 O(√n · log(√n))
```

**分块核心框架**：

```cpp
const int BLOCK = 320;
int a[MAXN], belong[MAXN];
int tag[MAXN];  // 每块的懒标记

void init(int n) {
    for (int i = 1; i <= n; i++)
        belong[i] = (i - 1) / BLOCK + 1;
}
```

### 3.5 CDQ 分治（进阶）

**特征**：CDQ 分治是一种离线分治技术，常用于处理多维偏序问题。核心思想是：按第一维排序后分治，在合并阶段处理"左半对右半的贡献"，再用其他数据结构处理剩余维度。

```
适用信号：多维偏序（二维/三维）、离线动态问题
实现要点：按时间/第一维排序 + 分治 + 树状数组处理第二维
经典问题：三维偏序、动态逆序对
复杂度：O(n log² n)
```

**CDQ 分治核心框架**（以三维偏序为例）：

```cpp
struct Query { int a, b, c, cnt, ans; };
Query q[MAXN];

void cdq(int l, int r) {
    if (l >= r) return;
    int mid = (l + r) / 2;
    cdq(l, mid);
    cdq(mid + 1, r);
    // 左半按 b 排序，右半按 b 排序
    // 双指针扫描，用树状数组维护 c 维
    // 统计左半对右半的贡献
    merge_by_b(l, mid, r);
}
```

### 模型对比总表

| 模型 | 核心操作 | 分割方式 | 合并方式 | 复杂度 | 代表题 |
|------|----------|----------|----------|--------|--------|
| 递归分治 | 分 + 递归 + 合并 | 规模减半 | 直接合并 | O(log n)~O(n log n) | 快速幂、快速排序 |
| 归并分治 | 归并过程中统计 | 对半分 | 归并合并 | O(n log n) | 逆序对 |
| 几何分治 | 空间分割 | 按坐标分 | 带状区域 | O(n log n)~O(n log² n) | 最近点对 |
| 分块 | √n 分块 | 等大小块 | 整块+散块 | O(√n) | 教主的魔法 |
| CDQ 分治 | 离线偏序处理 | 时间/维度分 | 贡献统计 | O(n log² n) | 三维偏序 |

## 四、15 道经典例题详解

### 🟢 入门级 Level 1：递归分治基础

### 题目 1：【模板】快速幂（洛谷 P1226）

> 给定三个整数 a、b、p，求 `a^b mod p` 的值。其中 b 可达 2^31 - 1。

**分析**：直接连乘 `O(b)` 会超时。利用分治思想：将 `a^b` 拆为 `a^(b/2) * a^(b/2)`（b 为偶数）或 `a * a^(b-1)`（b 为奇数），每次规模减半，复杂度降为 `O(log b)`。这就是"快速幂"——分治思想最纯粹的体现。

```
核心思路：
  a^b = (a^(b/2))^2          当 b 为偶数
  a^b = a * (a^(b/2))^2      当 b 为奇数
  a^0 = 1                    递归基
复杂度：O(log b)
```

**C++ 代码**：

```cpp
long long power(long long a, long long b, long long p) {
    long long res = 1;
    a %= p;
    while (b > 0) {
        if (b & 1) res = res * a % p;
        a = a * a % p;
        b >>= 1;
    }
    return res;
}
// 调用：power(a, b, p)
```

答案位置：`power(a, b, p)` 的返回值

**核心启示**：
- **指数折半是分治的基本操作**：每次将指数减半，是 `O(log n)` 复杂度的根源。
- **位运算替代递归**：迭代版快速幂用 `b & 1` 和 `b >>= 1` 代替递归，更高效且无栈溢出风险。
- **取模的传递性**：每步都取模，避免溢出，`(a * b) % p = ((a % p) * (b % p)) % p`。

### 题目 2：【模板】快速排序（洛谷 P1177）

> 给定 n 个整数，将它们从小到大排序后输出。

**分析**：快速排序是分治的经典应用：选取基准值（pivot），将数组划分为"小于 pivot"和"大于 pivot"两部分，递归排序两部分。平均复杂度 `O(n log n)`。

```
核心思路：
  1. 选取 pivot（通常取中间元素避免最坏情况）
  2. 划分：< pivot 的放左边，> pivot 的放右边
  3. 递归排序左右两部分
复杂度：平均 O(n log n)，最坏 O(n²)
```

**C++ 代码**：

```cpp
const int MAXN = 100005;
int a[MAXN];

void qsort(int l, int r) {
    if (l >= r) return;
    int i = l, j = r, pivot = a[(l + r) / 2];
    while (i <= j) {
        while (a[i] < pivot) i++;
        while (a[j] > pivot) j--;
        if (i <= j) swap(a[i++], a[j--]);
    }
    qsort(l, j);
    qsort(i, r);
}
// 调用：qsort(1, n)
```

答案位置：排序后 `a[1..n]` 即为有序序列

**核心启示**：
- **划分是快排的灵魂**：不同的划分策略（Lomuto、Hoare）决定了常数和稳定性。
- **pivot 选取影响性能**：取中间元素或随机元素可避免有序数据下退化为 `O(n²)`。
- **分治不一定需要合并**：快排的"合并"步骤是空操作——划分完毕后数据已经归位。

### 🟡 基础级 Level 2：分治应用拓展

### 题目 3：幂次方（洛谷 P1010）

> 任意正整数可以用 2 的幂次方表示。规则：`a^b` 写成 `a(b)` 的形式，指数也要递归展开。如 `137 = 2^7 + 2^3 + 2^0`，最终输出 `2(2(2)+2+2(0))+2(2+2(0))+2(0)`。

**分析**：经典的递归分治题。对于数 n，找到最高位的 2 的幂次方 `2^t`，输出对应格式后递归处理剩余部分 `n - 2^t`；对指数 t 也要递归展开。特殊情况：指数为 0 输出 `2(0)`，指数为 1 只输出 `2`，指数为 2 输出 `2(2)`。

```
核心思路：
  1. 找最高位幂次 t，使得 2^t ≤ n
  2. 递归输出 2(f(t))，其中 f(t) 递归展开 t
  3. 若 n - 2^t > 0，输出 "+" 后递归处理剩余
  特殊处理：t=0 → "0"；t=1 → "2"
```

**C++ 代码**：

```cpp
void solve(int n) {
    if (n == 0) { printf("0"); return; }
    bool first = true;
    for (int t = 20; t >= 0; t--) {
        if (!((n >> t) & 1)) continue;
        if (!first) printf("+");
        first = false;
        if (t == 0) printf("2(0)");
        else if (t == 1) printf("2");
        else { printf("2("); solve(t); printf(")"); }
    }
}
// 调用：solve(n)
```

答案位置：直接输出到标准输出

**核心启示**：
- **分治可以递归处理表示本身**：这里不仅"值"被分治，"指数"也被分治，形成嵌套递归。
- **二进制分解是分治的天然载体**：从高位到低位拆解，每一位独立处理。
- **边界特判是递归分治的关键**：0、1、2 三个特殊值必须单独处理。

### 题目 4：逆序对（洛谷 P1908）

> 给定一个长度为 n 的序列，求其中逆序对的个数。逆序对定义：`i < j` 且 `a[i] > a[j]`。

**分析**：暴力枚举所有点对 `O(n²)` 不够快。利用归并排序的分治框架：在合并左右两半时，若左半的元素 `a[i]` 大于右半的 `a[j]`，则 `a[i..mid]` 都与 `a[j]` 构成逆序对，一次性统计 `mid - i + 1` 个。总复杂度 `O(n log n)`。

```
核心思路：
  1. 归并排序框架：分两半 → 递归 → 合并
  2. 合并时：当 a[i] > a[j]（i 在左半，j 在右半）
     贡献 mid - i + 1 个逆序对
复杂度：O(n log n)
```

**C++ 代码**：

```cpp
const int MAXN = 500005;
int a[MAXN], tmp[MAXN];
long long ans;

void merge_sort(int l, int r) {
    if (l >= r) return;
    int mid = (l + r) / 2;
    merge_sort(l, mid);
    merge_sort(mid + 1, r);
    int i = l, j = mid + 1, k = l;
    while (i <= mid && j <= r) {
        if (a[i] <= a[j]) tmp[k++] = a[i++];
        else { tmp[k++] = a[j++]; ans += mid - i + 1; }
    }
    while (i <= mid) tmp[k++] = a[i++];
    while (j <= r) tmp[k++] = a[j++];
    for (int p = l; p <= r; p++) a[p] = tmp[p];
}
// 调用：ans = 0; merge_sort(1, n);
```

答案位置：全局变量 `ans`

**核心启示**：
- **归并分治的精髓在于"合并时统计"**：左右各自有序后，跨越中点的关系可以高效计算。
- **逆序对是归并分治的经典模型**：将 `O(n²)` 暴力优化到 `O(n log n)`。
- **分治思路可推广**：类似地可以统计"顺序对"、"跨区间计数"等。

### 题目 5：地毯填补问题（洛谷 P1228）

> 有一个 `2^k × 2^k` 的棋盘，其中有一个格子被公主占据。要用 L 形地毯（覆盖 3 格）铺满其余所有格子。输出每块地毯的位置和类型。

**分析**：经典的棋盘覆盖分治问题。将棋盘分成四个 `2^(k-1) × 2^(k-1)` 的象限，公主在其中一个象限。在中心交汇处放一块 L 形地毯覆盖其余三个象限各一格，这样每个象限都有一个"已占据"的格子，递归处理四个子棋盘。

```
核心思路：
  1. 将 2^k × 2^k 棋盘分成四个象限
  2. 公主所在象限已有特殊格
  3. 在中心放 L 形地毯，覆盖其余三个象限各一格
  4. 四个象限各自递归
  递归基：2×2 棋盘直接放一块 L 形地毯
复杂度：O(4^k) = O(n²)（n = 2^k 为边长）
```

**C++ 代码**：

```cpp
int tot;

void solve(int x, int y, int size, int px, int py) {
    if (size == 1) return;
    int half = size / 2;
    int id = ++tot;
    int mx = x + half - 1, my = y + half - 1;
    // 左上象限
    if (px <= mx && py <= my) solve(x, y, half, px, py);
    else { /* 标记 (mx, my) */ solve(x, y, half, mx, my); }
    // 右上象限
    if (px <= mx && py > my) solve(x, my + 1, half, px, py);
    else { /* 标记 (mx, my+1) */ solve(x, my + 1, half, mx, my + 1); }
    // 左下象限
    if (px > mx && py <= my) solve(mx + 1, y, half, px, py);
    else { /* 标记 (mx+1, my) */ solve(mx + 1, y, half, mx + 1, my); }
    // 右下象限
    if (px > mx && py > my) solve(mx + 1, my + 1, half, px, py);
    else { /* 标记 (mx+1, my+1) */ solve(mx + 1, my + 1, half, mx + 1, my + 1); }
}
// 调用：tot = 0; solve(1, 1, 1 << k, px, py)
```

答案位置：每次递归输出 L 形地毯的位置和类型

**核心启示**：
- **空间分割是二维分治的核心**：将平面区域不断四等分，是二维分治的标准手法。
- **"制造"子问题的技巧**：通过在中心放 L 形地毯，人为地把四个象限统一为相同的子问题结构。
- **分治不只是数组操作**：棋盘覆盖展示了分治在二维几何上的强大应用。

### 题目 6：[USACO17JAN] Secret Cow Code（洛谷 P3612）

> 给定字符串 s，每次操作将 s 变为 `s + 右循环一位的 s`（最后一个字符移到最前面后拼接）。字符串长度不断翻倍。给定 N（≤ 10^18），求无限操作后第 N 个字符。

**分析**：字符串长度指数增长，无法直接构造。利用分治/递归反推：先将总长度翻倍直到 ≥ N，然后逆向定位——若 N 在后半段，利用循环移位的规则将 N 映射回前半段的位置，不断缩小规模直到落在原始字符串内。

```
核心思路：
  1. 令 len = |s|，不断 len *= 2 直到 len ≥ N
  2. 递归反推：
     若 N ≤ |s|，直接返回 s[N-1]
     若 N = len/2 + 1，返回 solve(len/2)（循环移位的首字符是前半末字符）
     若 N > len/2 + 1，返回 solve(N - len/2 - 1)
复杂度：O(|s| + log N)
```

**C++ 代码**：

```cpp
char s[35];
int slen;

char solve(long long n, long long len) {
    if (n <= slen) return s[n - 1];
    long long half = len / 2;
    if (n == half + 1) return solve(half, half);
    if (n > half + 1) return solve(n - half - 1, half);
    return solve(n, half);
}
// 调用：先倍增 len 到 >= N，然后 solve(N, len)
```

答案位置：`solve(N, len)` 的返回值

**核心启示**：
- **分治可以"反向"使用**：不是构建再分解，而是从大问题反推映射到小问题。
- **倍增结构天然适合分治**：长度翻倍的递推过程，逆向就是规模减半的分治。
- **处理超大数据范围的利器**：`N ≤ 10^18` 无法模拟，但分治每步减半，只需 `O(log N)` 步。

### 题目 7：平面上的最接近点对（洛谷 P1257）

> 给定平面上 n 个点（n ≤ 10^4），求距离最近的一对点的欧几里得距离。

**分析**：暴力 `O(n²)` 虽然对本题数据范围可行，但分治法是更优雅的 `O(n log n)` 或 `O(n log² n)` 解法：按 x 坐标排序后对半分，分别求左右最近距离 d，再在宽度为 2d 的带状区域内处理跨越中线的点对。

```
核心思路：
  1. 按 x 坐标排序
  2. 递归求左半和右半的最近距离 d1, d2，令 d = min(d1, d2)
  3. 收集距中线 < d 的点，按 y 排序
  4. 对每个点只需比较 y 方向上距离 < d 的邻居（最多 7 个）
复杂度：O(n log² n)（若合并时归并 y 可优化为 O(n log n)）
```

**C++ 代码**：

```cpp
const int MAXN = 10005;
struct Point { double x, y; } pts[MAXN], strip[MAXN];

double dist(Point a, Point b) {
    return sqrt((a.x-b.x)*(a.x-b.x) + (a.y-b.y)*(a.y-b.y));
}

bool cmp_y(Point a, Point b) { return a.y < b.y; }

double solve(int l, int r) {
    if (l >= r) return 1e18;
    if (l + 1 == r) return dist(pts[l], pts[r]);
    int mid = (l + r) / 2;
    double mx = pts[mid].x;
    double d = min(solve(l, mid), solve(mid + 1, r));
    int cnt = 0;
    for (int i = l; i <= r; i++)
        if (fabs(pts[i].x - mx) < d) strip[cnt++] = pts[i];
    sort(strip, strip + cnt, cmp_y);
    for (int i = 0; i < cnt; i++)
        for (int j = i + 1; j < cnt && strip[j].y - strip[i].y < d; j++)
            d = min(d, dist(strip[i], strip[j]));
    return d;
}
// 调用前先按 x 排序：sort(pts, pts+n, cmp_x); 然后 solve(0, n-1)
```

答案位置：`solve(0, n-1)` 的返回值（保留 4 位小数输出）

**核心启示**：
- **几何分治的核心是"带状区域"**：跨越中线的点对只需在窄带中查找，每个点最多比较 7 个邻居。
- **排序是分治的前置操作**：x 排序保证分割有效，y 排序保证合并高效。
- **O(n²) 到 O(n log n) 的跨越**：这是分治在计算几何中最经典的加速案例。

### 题目 8：IncDec Sequence（洛谷 P4552）

> 给定长度为 n 的数列，每次操作可以选择一个区间 [l, r] 整体加 1 或减 1。求：(1) 最少操作次数使所有数相等；(2) 最少操作下有多少种最终结果。

**分析**：本题的关键在于"分而治之"的思路——将原问题通过差分转化后分解。建立差分数组 `c[i] = a[i] - a[i-1]`（i ≥ 2），目标是让 `c[2..n]` 全部变为 0。正的差分和负的差分可以"对冲"消除，剩余部分单独处理。这是将问题分解为独立的正负两部分再合并结果的分治思维。

```
核心思路：
  1. 差分数组 c[i] = a[i] - a[i-1]，i = 2..n
  2. 正数之和 pos，负数绝对值之和 neg
  3. 最少操作 = max(pos, neg)（正负对冲）
  4. 结果种类 = |pos - neg| + 1
复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 100005;
long long a[MAXN];

void solve(int n) {
    long long pos = 0, neg = 0;
    for (int i = n; i >= 2; i--) a[i] -= a[i - 1];
    for (int i = 2; i <= n; i++) {
        if (a[i] > 0) pos += a[i];
        else neg += -a[i];
    }
    // 最少操作次数：max(pos, neg)
    // 最终结果种类：abs(pos - neg) + 1
}
// 调用：solve(n)
```

答案位置：`max(pos, neg)` 和 `abs(pos - neg) + 1`

**核心启示**：
- **差分是一种"分解"手段**：将区间操作转化为端点操作，本质是把复杂问题分解为独立的子问题。
- **正负对冲体现分治合并思想**：将差分数组分为正负两组独立处理，再合并得出最优策略。
- **问题转化 + 分解是高阶分治思维**：不一定要递归才叫分治，"大问题拆小、独立求解、合并结果"的思想更本质。

### 题目 9：火柴排队（洛谷 P1966）

> 给定两列各 n 根火柴，高度分别为 a[i] 和 b[i]。可以交换同一列中相邻火柴的位置，求最少交换次数使得 Σ(a[i]-b[i])² 最小，答案对 99999997 取模。

**分析**：由排序不等式，两列按相同相对顺序排列时 Σ(a[i]-b[i])² 最小。将 a、b 分别求出排名，得到一个排列映射，将问题转化为求该排列的逆序对数——即最少相邻交换次数。归并排序 `O(n log n)` 求逆序对，这是分治在排列问题上的经典应用。

```
核心思路：
  1. 排序不等式：同序排列使差的平方和最小
  2. 离散化：分别对 a、b 求排名
  3. 构造排列 c[rank_a[i]] = rank_b[i]
  4. 归并排序求 c 的逆序对数
复杂度：O(n log n)
```

**C++ 代码**：

```cpp
const int MAXN = 100005;
const int MOD = 99999997;
int c[MAXN], tmp[MAXN];
long long ans;

void msort(int l, int r) {
    if (l >= r) return;
    int mid = (l + r) / 2;
    msort(l, mid); msort(mid + 1, r);
    int i = l, j = mid + 1, k = l;
    while (i <= mid && j <= r) {
        if (c[i] <= c[j]) tmp[k++] = c[i++];
        else { tmp[k++] = c[j++]; ans = (ans + mid - i + 1) % MOD; }
    }
    while (i <= mid) tmp[k++] = c[i++];
    while (j <= r) tmp[k++] = c[j++];
    for (int p = l; p <= r; p++) c[p] = tmp[p];
}
// 离散化后构造 c[]，然后 ans = 0; msort(1, n);
```

答案位置：全局变量 `ans`

**核心启示**：
- **排序不等式 + 逆序对是经典组合**：很多"最少交换次数"问题可以转化为逆序对。
- **归并排序不只是排序**：它的合并过程天然适合统计跨区间的逆序关系。
- **离散化是转化的桥梁**：将原始值映射为排名，问题从"值的比较"变为"排列的逆序"。

### 题目 10：合并神犇（洛谷 P2300）

> 给定 n 个神犇排成一排，每个有一个能量值。若相邻两个能量值相同，它们合并为一个，能量翻倍。合并可能引发连锁反应，求最终序列的长度。

**分析**：直接模拟需要反复扫描，效率低下。用分治思想：将序列分为左右两半，分别递归合并，再处理分界处的连锁合并。实际实现时，用栈模拟这一过程——每次入栈时若栈顶与当前值相同则合并（翻倍），不断向上合并直到不同，这正是分治"边合并边处理"的思想体现。

```
核心思路：
  1. 维护一个栈，依次处理每个元素
  2. 入栈时，若栈顶 == 当前值，弹出并翻倍
  3. 翻倍后继续检查新栈顶，连锁合并
  4. 最终栈中元素即为结果
复杂度：O(n)（每个元素最多入栈出栈一次）
```

**C++ 代码**：

```cpp
const int MAXN = 100005;
long long a[MAXN], stk[MAXN];
int top;

void solve(int n) {
    top = 0;
    for (int i = 1; i <= n; i++) {
        long long v = a[i];
        while (top > 0 && stk[top] == v) {
            v <<= 1;
            top--;
        }
        stk[++top] = v;
    }
}
// 结果：stk[1..top] 为合并后的序列，top 为长度
```

答案位置：`top` 为最终序列长度，`stk[1..top]` 为序列内容

**核心启示**：
- **栈是分治合并的线性化实现**：每次入栈都在处理"当前元素与已处理部分的合并"，本质上是分治中"合并"步骤的迭代版本。
- **连锁合并体现递归本质**：一次合并可能触发更多合并，对应分治中子问题解合并后可能引发上层合并。
- **O(n) 的均摊复杂度**：虽然单次可能连续合并多次，但每个元素最多入栈出栈各一次。

void solve(int n) {

### 题目 12：平面最近点对（加强版）（洛谷 P1429）

> 给定平面上 n 个点（n ≤ 200000），求距离最近的一对点的欧几里得距离。

**分析**：数据量达到 20 万，暴力 `O(n²)` 完全无法通过，必须使用 `O(n log n)` 的分治法。与 P1257 思路相同，但需要优化常数：合并时避免每层都 sort（用归并代替），确保总复杂度为严格 `O(n log n)`。

```
核心思路：
  1. 按 x 排序后分治
  2. 递归求解左右最近距离 d
  3. 带状区域收集 + 按 y 排序 + 暴力比较邻居
  4. 优化：归并维护 y 序，避免反复 sort
复杂度：O(n log n)（优化后）或 O(n log² n)（每层 sort）
```

**C++ 代码**：

```cpp
const int MAXN = 200005;
struct Point { double x, y; } pts[MAXN], tmp[MAXN];

double dist(Point a, Point b) {
    return sqrt((a.x-b.x)*(a.x-b.x) + (a.y-b.y)*(a.y-b.y));
}

double solve(int l, int r) {
    if (l >= r) return 1e18;
    if (r - l <= 2) {
        double d = 1e18;
        for (int i = l; i <= r; i++)
            for (int j = i + 1; j <= r; j++)
                d = min(d, dist(pts[i], pts[j]));
        sort(pts + l, pts + r + 1,
             [](Point a, Point b){ return a.y < b.y; });
        return d;
    }
    int mid = (l + r) / 2;
    double mx = pts[mid].x;
    double d = min(solve(l, mid), solve(mid + 1, r));
    // 归并按 y 排序
    int i = l, j = mid + 1, k = 0;
    while (i <= mid && j <= r)
        tmp[k++] = (pts[i].y <= pts[j].y) ? pts[i++] : pts[j++];
    while (i <= mid) tmp[k++] = pts[i++];
    while (j <= r) tmp[k++] = pts[j++];
    for (int p = 0; p < k; p++) pts[l + p] = tmp[p];
    // 带状区域暴力
    k = 0;
    for (int p = l; p <= r; p++)
        if (fabs(pts[p].x - mx) < d) tmp[k++] = pts[p];
    for (int p = 0; p < k; p++)
        for (int q = p + 1; q < k && tmp[q].y - tmp[p].y < d; q++)
            d = min(d, dist(tmp[p], tmp[q]));
    return d;
}
// 调用前按 x 排序，然后 solve(0, n-1)
```

答案位置：`solve(0, n-1)` 的返回值

**核心启示**：
- **归并优化是分治进阶的关键**：将每层 `O(n log n)` 的 sort 替换为 `O(n)` 的归并，总复杂度从 `O(n log² n)` 降为 `O(n log n)`。
- **带状区域的 7 点性质**：已证明在宽度为 2d 的带中，每个点最多只需比较 7 个 y 坐标相邻的点。
- **大数据量下常数优化很重要**：20 万点时，sort 和归并的性能差异显著。

### 题目 13：教主的魔法（洛谷 P2801）

> 给定一个长度为 n（≤ 10^6）的序列，支持两种操作：(1) 区间 [L, R] 所有数加 W；(2) 查询区间 [L, R] 中有多少个数 ≥ C。最多 3000 次操作。

**分析**：经典分块题，体现了分治"大块整体处理、小块暴力处理"的思想。将数组分成 `√n` 个块，每个块内维护排序副本用于二分查找。整块用懒标记处理区间加，散块暴力修改后重新排序。查询时整块二分计数，散块暴力统计。

```
核心思路：
  1. 分成 √n 个块，每块维护排序副本 b[]
  2. 区间加：整块打 tag，散块暴力修改 + 重排序
  3. 区间查询 ≥ C：整块用 lower_bound 计数，散块暴力
复杂度：单次操作 O(√n · log(√n))
```

**C++ 代码**：

```cpp
const int MAXN = 1000005;
const int BLOCK = 1005;
int a[MAXN], b[MAXN];
int tag[BLOCK];
int bl[BLOCK], br[BLOCK], belong[MAXN];
int block_cnt;

void init(int n) {
    int sz = (int)sqrt(n);
    block_cnt = (n - 1) / sz + 1;
    for (int i = 1; i <= block_cnt; i++) {
        bl[i] = (i - 1) * sz + 1;
        br[i] = min(i * sz, n);
    }
    for (int i = 1; i <= n; i++) belong[i] = (i - 1) / sz + 1;
    for (int i = 1; i <= block_cnt; i++)
        sort(b + bl[i], b + br[i] + 1);
}

void rebuild(int id) {
    for (int i = bl[id]; i <= br[id]; i++) b[i] = a[i];
    sort(b + bl[id], b + br[id] + 1);
}

void add(int l, int r, int w) {
    int lb = belong[l], rb = belong[r];
    if (lb == rb) {
        for (int i = l; i <= r; i++) a[i] += w;
        rebuild(lb); return;
    }
    for (int i = l; i <= br[lb]; i++) a[i] += w;
    rebuild(lb);
    for (int i = bl[rb]; i <= r; i++) a[i] += w;
    rebuild(rb);
    for (int i = lb + 1; i < rb; i++) tag[i] += w;
}

int query(int l, int r, int c) {
    int lb = belong[l], rb = belong[r], ans = 0;
    if (lb == rb) {
        for (int i = l; i <= r; i++)
            if (a[i] + tag[lb] >= c) ans++;
        return ans;
    }
    for (int i = l; i <= br[lb]; i++)
        if (a[i] + tag[lb] >= c) ans++;
    for (int i = bl[rb]; i <= r; i++)
        if (a[i] + tag[rb] >= c) ans++;
    for (int i = lb + 1; i < rb; i++) {
        int val = c - tag[i];
        int start = bl[i];
        ans += br[i] - bl[i] + 1
             - (int)(lower_bound(b+start, b+br[i]+1, val) - (b+start));
    }
    return ans;
}
// 调用：init(n) 后执行 add/query 操作
```

答案位置：`query(l, r, c)` 的返回值

**核心启示**：
- **分块是"物理分治"**：虽然不递归，但将数组分成独立的块分别处理，是分治思想的直接体现。
- **整块用数据结构加速，散块暴力**：这是分块的核心模式——"大段批量 + 两端零散"。
- **分块的普适性**：线段树/树状数组难以处理的复杂查询（如区间排名、区间众数），分块往往是退路。

### 题目 14：【模板】三维偏序（陌上花开）（洛谷 P3810）

> 给定 n 个元素，每个元素有三个属性 (a, b, c)。对于每个元素 i，求满足 a[j]≤a[i]、b[j]≤b[i]、c[j]≤c[i] 且 j≠i 的元素 j 的个数 f(i)。输出 f 值为 0, 1, ..., n-1 的元素各有多少个。

**分析**：三维偏序是 CDQ 分治的经典模板。第一维排序消除，第二维用 CDQ 分治处理，第三维用树状数组（BIT）统计。CDQ 分治的核心思想：将序列按时间/第一维分成左右两半，递归处理各自的内部贡献，再计算左半对右半的跨区间贡献——这是"分治统计跨区间信息"的高阶推广。

```
核心思路：
  1. 按 a 排序，去重（相同元素合并计数）
  2. CDQ 分治：按 a 分左右半，递归各自
  3. 合并时：按 b 归并，用 BIT 维护 c 维前缀和
  4. 左半元素按 b 序插入 BIT，右半元素查询 BIT
复杂度：O(n log² n)
```

**C++ 代码**：

```cpp
const int MAXN = 100005;
struct Node { int a, b, c, cnt, ans; } q[MAXN], tmp[MAXN];
int tree[200005], k, ans[MAXN];

void add(int x, int v) { for (; x <= k; x += x&-x) tree[x] += v; }
int ask(int x) { int s = 0; for (; x; x -= x&-x) s += tree[x]; return s; }

void cdq(int l, int r) {
    if (l >= r) return;
    int mid = (l + r) / 2;
    cdq(l, mid); cdq(mid + 1, r);
    int i = l, j = mid + 1, p = l;
    while (i <= mid && j <= r) {
        if (q[i].b <= q[j].b) { add(q[i].c, q[i].cnt); tmp[p++] = q[i++]; }
        else { q[j].ans += ask(q[j].c); tmp[p++] = q[j++]; }
    }
    while (j <= r) { q[j].ans += ask(q[j].c); tmp[p++] = q[j++]; }
    for (int t = l; t < i; t++) add(q[t].c, -q[t].cnt);
    while (i <= mid) tmp[p++] = q[i++];
    for (int t = l; t <= r; t++) q[t] = tmp[t];
}
// 按 a 排序去重后 cdq(1, m)，再将 ans 映射回原数组
```

答案位置：`q[i].ans + q[i].cnt - 1` 为每个元素的 f 值，统计各 f 值的出现次数

**核心启示**：
- **CDQ 分治是多维偏序的通用框架**：每增加一维，嵌套一层分治或数据结构，维度逐步消除。
- **"左对右"的贡献统计**：CDQ 分治只计算左半对右半的贡献，避免重复统计，是归并分治思想的自然推广。
- **归并 + BIT 的配合**：归并处理第二维的有序性，BIT 处理第三维的前缀查询，各司其职。

### 题目 15：[Violet] 天使玩偶 / Angel Beats（洛谷 P4169）

> 平面上初始有 n 个点。依次执行 m 次操作：(1) 在坐标 (x, y) 添加一个新点；(2) 查询离给定点 (x, y) 曼哈顿距离最近的已有点的距离。n, m ≤ 500000。

**分析**：曼哈顿距离 |x1-x2|+|y1-y2| 展开后，对于左下方向的点（x1≤x2, y1≤y2），距离 = (x2+y2)-(x1+y1)，转化为求 x1+y1 的最大值。将四个象限分别处理（坐标翻转），每个方向用 CDQ 分治：按时间分治，左半的添加操作对右半的查询操作产生贡献，用树状数组按 x 坐标维护 x+y 的最大值。

```
核心思路：
  1. 曼哈顿距离拆为四个象限分别求解
  2. 每个象限：CDQ 按时间分治
  3. 左半添加 → 按 x 归并 → BIT 维护 x+y 最大值
  4. 右半查询 → BIT 查询前缀最大值 → 更新答案
  5. 四次 CDQ 取最小值
复杂度：O(n log² n)
```

**C++ 代码**：

```cpp
const int MAXN = 500005;
const int INF = 0x3f3f3f3f;
struct Op { int tp, x, y, ans; } ops[MAXN * 2], tmp[MAXN * 2];
int tree[MAXN * 2], m;

void upd(int x, int v) { for (; x <= m; x += x&-x) tree[x] = max(tree[x], v); }
int qry(int x) { int r = -INF; for (; x; x -= x&-x) r = max(r, tree[x]); return r; }
void clr(int x) { for (; x <= m; x += x&-x) tree[x] = -INF; }

void cdq(int l, int r) {
    if (l >= r) return;
    int mid = (l + r) / 2;
    cdq(l, mid); cdq(mid + 1, r);
    int i = l, j = mid + 1, p = l;
    while (i <= mid && j <= r) {
        if (ops[i].x <= ops[j].x) {
            if (ops[i].tp == 1) upd(ops[i].y, ops[i].x + ops[i].y);
            tmp[p++] = ops[i++];
        } else {
            if (ops[j].tp == 2) {
                int v = qry(ops[j].y);
                if (v > -INF) ops[j].ans = min(ops[j].ans, ops[j].x + ops[j].y - v);
            }
            tmp[p++] = ops[j++];
        }
    }
    while (j <= r) {
        if (ops[j].tp == 2) {
            int v = qry(ops[j].y);
            if (v > -INF) ops[j].ans = min(ops[j].ans, ops[j].x + ops[j].y - v);
        }
        tmp[p++] = ops[j++];
    }
    for (int t = l; t < i; t++) if (ops[t].tp == 1) clr(ops[t].y);
    while (i <= mid) tmp[p++] = ops[i++];
    for (int t = l; t <= r; t++) ops[t] = tmp[t];
}
// 对四个象限分别翻转坐标后各跑一次 cdq
```

答案位置：`ops[i].ans` 为每个查询操作的最近曼哈顿距离

**核心启示**：
- **曼哈顿距离的象限拆分**：绝对值展开为四种情况，每种转化为前缀最值问题，是处理曼哈顿距离的标准技巧。
- **CDQ 分治处理带时间轴的问题**：在线操作离线化后按时间分治，是 CDQ 分治最强大的应用场景之一。
- **多次坐标变换复用同一算法**：四次翻转 + 同一份 CDQ 代码，体现了"分治 + 对称性"的组合威力。

## 五、常见技巧与思路总结

### 5.1 分治的"分"：如何选择分割方式

分治的效率取决于分割方式。常见策略：

| 分割方式 | 适用场景 | 示例 |
|----------|----------|------|
| 对半分（mid = (l+r)/2） | 数组/序列问题 | 归并排序、逆序对 |
| 按坐标分 | 平面几何 | 最近点对 |
| 按二进制位分 | 数值分解 | 快速幂、幂次方 |
| 等大小块（√n） | 区间操作 | 分块 |
| 按时间/维度分 | 偏序问题 | CDQ 分治 |

### 5.2 分治的"合"：常见合并技巧

合并通常是分治的难点，常见模式：

1. **归并合并**：左右各自有序，双指针归并的同时统计跨区间信息。
2. **带状区域**：几何分治中，只处理距离分界线 ≤ d 的点。
3. **无需合并**：快速排序的划分天然归位，不需要额外合并步骤。
4. **贡献统计**：CDQ 分治中，只统计左半对右半的贡献。

### 5.3 递归基的处理

递归基（base case）的正确性直接影响整个分治的正确性：

```
常见递归基：
  - l >= r：单元素或空区间，直接返回
  - r - l <= 2：暴力处理 2-3 个元素
  - n == 0 或 n == 1：直接返回结果
```

**注意**：递归基不要设太大（影响效率），也不能漏掉（导致死循环或错误）。

### 5.4 避免分治退化

快速排序的 pivot 选取不当会退化为 `O(n²)`。常见规避策略：

- **取中间元素**：`pivot = a[(l+r)/2]`
- **随机化**：`pivot = a[rand() % (r-l+1) + l]`
- **三数取中**：取 `a[l]`、`a[mid]`、`a[r]` 的中位数

### 5.5 空间优化

分治可能需要额外空间（如归并排序的 tmp 数组）：

- **全局数组**：提前分配 `tmp[MAXN]`，避免递归中反复 new。
- **原地操作**：快速排序不需要额外空间，是原地分治。
- **共享缓冲区**：分治的不同层可以共用同一块 tmp 空间。

### 5.6 分治与二分的关系

二分搜索可以看作是"单侧分治"：每次分成两半后，只递归处理其中一半：

```
二分搜索：T(n) = T(n/2) + O(1) → O(log n)
归并排序：T(n) = 2T(n/2) + O(n) → O(n log n)
```

二分是分治的特例——只有"分"和"选"，没有"合"。

## 七、终极总结与归纳

### 7.1 分治的本质

> **把一个大问题拆成若干独立的小问题，分别解决后合并——用 O(log n) 层递归换取每层 O(n) 的高效合并。**

分治的核心不在于"分"——几乎任何问题都能分——而在于：
- 分出来的子问题是否**同构**（与原问题结构相同）？
- 子问题之间是否**独立**（没有重叠计算）？
- 合并步骤是否**高效**（不超过线性时间）？

满足这三个条件，分治就能带来指数级的效率提升。

### 7.2 核心要点

1. **分治三步曲**：分解 → 递归求解 → 合并。合并是难点，也是每道题的差异所在。
2. **主定理是复杂度分析的利器**：`T(n) = aT(n/b) + O(n^d)`，比较 `d` 和 `log_b(a)` 即可判断。
3. **归并分治统计跨区间信息**：当需要统计"左右之间的关系"时，归并排序框架是首选。
4. **几何分治靠带状区域**：平面问题按坐标分治，跨分界线的答案在窄带中查找。
5. **分块是分治思想的变体**：不递归，但"大段整体 + 两端暴力"的核心思想与分治一脉相承。
6. **递归反推是分治的逆向应用**：对于倍增构造的结构，从大到小反推位置，每步规模减半。
