## 一、什么是贪心

贪心算法（Greedy Algorithm）是一种在每一步都做出**当前看来最优选择**的策略。它不回溯、不枚举所有方案，而是坚信"每一步的局部最优"能够累积出"全局最优"。

举一个生活中的例子：你去超市购物，手里钱有限，想买尽可能多的东西。贪心策略就是——**每次都挑最便宜的那个放进购物车**。这种"只看眼前、不瞻前顾后"的做法，就是贪心的精髓。

贪心与其他算法策略的对比：

| 策略 | 核心思路 | 是否回溯 | 时间复杂度 | 适用条件 |
|------|----------|----------|------------|----------|
| 贪心 | 每步取局部最优 | 否 | 通常 O(n log n) | 贪心选择性质 + 最优子结构 |
| 动态规划 | 枚举所有子问题 | 否（记忆化） | O(状态数 × 转移) | 最优子结构 + 重叠子问题 |
| 回溯/搜索 | 枚举所有方案 | 是 | 指数级 | 无特殊性质要求 |

贪心的魅力在于：**当它适用时，代码极简、效率极高**；但它的难点在于——**如何判断它是否适用，以及如何证明正确性**。

## 二、核心思想与前置知识

### 2.1 贪心的适用条件

贪心算法能得到最优解，需要同时满足两个性质：

**1. 贪心选择性质（Greedy Choice Property）**

> 通过局部最优选择，能够达到全局最优解。即：存在一个最优解，其第一步选择与贪心策略一致。

**2. 最优子结构（Optimal Substructure）**

> 问题的最优解包含其子问题的最优解。做完第一步贪心选择后，剩余子问题仍然是同类型的最优化问题。

```
贪心适用的信号：
1. 求"最多/最少/最大/最小"的优化问题
2. 每一步选择不影响后续选择的可行域（无后效性）
3. 能找到一个排序标准，使得按此顺序处理一定不劣
4. 问题具有"交换不变性"——交换两个元素的顺序，只影响局部
```

### 2.2 贪心正确性证明方法

贪心题最难的不是写代码，而是**证明策略的正确性**。常见三种证明方法：

**方法一：交换论证法（Exchange Argument）**

最常用的方法。思路：
1. 假设存在一个最优解 OPT，不完全符合贪心策略
2. 找到 OPT 中第一个与贪心不同的位置
3. 证明：将该位置"交换"为贪心的选择后，解不会变差
4. 反复交换，最终将 OPT 变为贪心解，说明贪心解也是最优解

**方法二：数学归纳法**

1. 证明 n=1 时贪心策略成立
2. 假设对规模 k 成立，证明对规模 k+1 也成立

**方法三：反证法**

1. 假设贪心解不是最优
2. 存在更优解，推导出与贪心策略矛盾的结论

```
证明方法选择指南：
┌──────────────────────────────────────────┐
│ 排序型贪心（调度、排列）→ 交换论证法    │
│ 递推型贪心（逐步构造）  → 数学归纳法    │
│ 存在性问题（能否达到）  → 反证法        │
└──────────────────────────────────────────┘
```

### 2.3 贪心 vs 动态规划 对比

很多初学者分不清贪心和 DP。核心区别：

| 维度 | 贪心 | 动态规划 |
|------|------|----------|
| 决策方式 | 每步只取当前最优，不可撤销 | 枚举所有子问题取最优 |
| 子问题 | 只产生一个子问题 | 可能产生多个重叠子问题 |
| 正确性 | 需要额外证明 | 状态转移方程保证正确 |
| 效率 | 通常更快 | 通常更慢但更通用 |
| 典型标志 | "每次选最大/最小" | "方案数 / 所有可能" |

**判断技巧**：如果贪心选择后，子问题的结构不变且规模缩小，大概率可以贪心；如果当前选择会影响后续可选范围的"质量"，可能需要 DP。

## 三、贪心的子模型与分类详解

### 3.1 排序贪心

**核心思想**：将元素按某个关键字排序后，顺序/逆序贪心选取。

排序贪心是最常见的贪心类型。关键在于**找到正确的排序标准**。

```
适用信号：
- 题目给出一组元素，每个有若干属性
- 需要选取/安排顺序使某个目标最优
- 排序后顺序遍历即可得到答案

常见排序标准：
- 按价值/权重排序（背包类）
- 按截止时间排序（调度类）
- 按差值/比值排序（交换类）
```

```cpp
// 排序贪心通用框架
sort(items, items + n, [](const Item& a, const Item& b) {
    return a.key < b.key;  // 关键：确定排序标准
});
int ans = 0;
for (int i = 0; i < n; i++) {
    if (canSelect(items[i])) {
        ans += items[i].value;
        updateState(items[i]);
    }
}
```

### 3.2 区间贪心

**核心思想**：处理区间的选取、覆盖、合并问题，通常按端点排序后贪心。

区间贪心有两大经典问题：

**问题一：区间调度（最多不重叠区间）**
- 按**右端点**从小到大排序
- 每次选择不与已选区间冲突且右端点最小的区间

**问题二：区间覆盖（最少区间覆盖目标段）**
- 按**左端点**从小到大排序
- 每次选择能覆盖当前未覆盖起点且右端点最远的区间

```cpp
// 区间调度：最多不重叠区间
sort(seg, seg + n, [](const Seg& a, const Seg& b) {
    return a.r < b.r;
});
int ans = 0, last = -INF;
for (int i = 0; i < n; i++) {
    if (seg[i].l >= last) {
        ans++;
        last = seg[i].r;
    }
}
```

### 3.3 邻项交换法

**核心思想**：通过比较相邻两个元素交换前后的代价变化，推导最优排列顺序。

这是贪心中**最需要数学推导**的类型，也是竞赛常考点。

```
使用步骤：
1. 假设序列中相邻元素 i, j 的代价分别为 cost(i,j) 和 cost(j,i)
2. 列出 i 在前 vs j 在前的目标函数表达式
3. 化简比较，得到排序的充要条件
4. 证明该比较关系满足传递性（保证排序合法）
```

**经典应用**：国王游戏（P1080）中，设第 i 人左手 `a[i]`、右手 `b[i]`，邻项交换推导得排序关键字为 `a[i] * b[i]` 升序。

```cpp
// 邻项交换法通用比较函数
bool cmp(const Person& x, const Person& y) {
    // 由邻项交换推导得到的比较关系
    return x.a * x.b < y.a * y.b;
}
```

### 3.4 反悔贪心

**核心思想**：允许"撤销"之前的贪心选择，用更优的选择替代，通常借助堆（优先队列）实现。

当普通贪心可能因为"眼前最优"而错过"全局最优"时，反悔贪心允许我们在后续发现更好选项时，撤销之前的决定。

```
反悔贪心的典型流程：
1. 按某种顺序遍历元素
2. 将当前选择放入堆中
3. 遇到新元素时，与堆顶（当前最差的已选元素）比较
4. 如果替换后更优，弹出堆顶、换入新元素
```

```cpp
// 反悔贪心通用框架（小根堆维护已选集合）
priority_queue<int, vector<int>, greater<int>> pq;
int ans = 0;
for (int i = 0; i < n; i++) {
    pq.push(val[i]);
    ans += val[i];
    if ((int)pq.size() > limit) {
        ans -= pq.top();  // 撤销最差选择
        pq.pop();
    }
}
```

### 3.5 数学贪心

**核心思想**：通过不等式推导（如 AM-GM、排序不等式等）证明贪心策略的正确性。

数学贪心将最优化问题转化为数学不等式问题，用严格的数学推导来确定贪心策略。

```
常见数学工具：
- 排序不等式：顺序和 ≥ 乱序和 ≥ 逆序和
- AM-GM 不等式：算术均值 ≥ 几何均值
- 柯西不等式
- 调整法：证明任何偏离贪心策略的方案不会更优
```

### 模型对比总表

| 子模型 | 核心操作 | 常用数据结构 | 典型题目 |
|--------|----------|-------------|----------|
| 排序贪心 | 排序 + 顺序选取 | 数组 | 混合牛奶、纪念品分组 |
| 区间贪心 | 按端点排序 + 选取/覆盖 | 数组 | 线段覆盖 |
| 邻项交换 | 推导比较函数 + 排序 | 排序 | 国王游戏、皇后游戏 |
| 反悔贪心 | 贪心 + 撤销最差选择 | 堆/优先队列 | 梅深不见冬 |
| 数学贪心 | 不等式推导 + 贪心 | 无特殊 | 跳跳！ |

### 🟢 入门级 Level 1：排序贪心基础

### 题目 1：混合牛奶（洛谷 P1208 [USACO1.3]）

> 需要购买 M 单位牛奶，有 N 个奶农，第 i 个奶农可以提供最多 `q[i]` 单位牛奶，单价为 `p[i]`。求最小花费。

**分析**：典型的排序贪心。按单价从低到高排序，优先购买最便宜的牛奶。每次尽量多买，直到需求满足。

```
贪心策略：按单价升序排序
选择规则：从最便宜的开始，能买多少买多少
终止条件：需求量 M 降为 0
```

**C++ 代码**：

```cpp
const int MAXN = 5005;
int p[MAXN], q[MAXN];
int idx[MAXN];

void solve(int n, int m) {
    // 按单价升序排序
    for (int i = 0; i < n; i++) idx[i] = i;
    sort(idx, idx + n, [](int a, int b) {
        return p[a] < p[b];
    });
    long long cost = 0;
    for (int i = 0; i < n && m > 0; i++) {
        int id = idx[i];
        int buy = min(m, q[id]);
        cost += (long long)buy * p[id];
        m -= buy;
    }
    // cost 即为最小花费
}
```

答案位置：`cost`

**核心启示**：
- **排序是贪心的第一步**：确定"什么是当前最优"的标准
- **贪心 + 上界约束**：每个奶农有供应上限，贪心时需注意约束
- **O(n log n) 排序 + O(n) 扫描**是排序贪心的标准复杂度模式

### 题目 2：跳跳！（洛谷 P4995）

> 有 n 块石头，高度各不相同，起点高度为 0。每次从一块石头跳到另一块，消耗体力为高度差的平方。要求跳完所有石头，使总消耗最大。

**分析**：要使每次高度差的平方尽量大，就要让每次跳跃的高度落差尽量大。贪心策略：**高低交替跳**——先跳最高的，再跳最低的，再跳次高的，再跳次低的……利用排序不等式可以证明这种交替策略最优。

```
贪心策略：排序后从两端交替选取（先最高、再最低...）
数学依据：|a-b|² 在 a,b 差值越大时越大
         排序不等式保证高低交替使得差值平方和最大
```

**C++ 代码**：

```cpp
const int MAXN = 305;
int h[MAXN];

long long solve(int n) {
    sort(h, h + n);  // 升序排列
    // 构造跳跃序列：最高、最低、次高、次低...
    int order[MAXN], lo = 0, hi = n - 1, cnt = 0;
    while (lo <= hi) {
        order[cnt++] = h[hi--];  // 先选最高
        if (lo <= hi) order[cnt++] = h[lo++];  // 再选最低
    }
    long long ans = (long long)order[0] * order[0];  // 从地面(0)跳到第一块
    for (int i = 1; i < n; i++) {
        long long d = order[i] - order[i - 1];
        ans += d * d;
    }
    return ans;
}
```

答案位置：`solve(n)` 的返回值

**核心启示**：
- **排序不等式的应用**：高低交替使差值平方和最大，这是数学贪心的典型应用
- **构造最优序列**：不是选子集，而是确定最优排列顺序
- **起点为 0 要特殊处理**：第一步从地面开始跳，先跳最高的石头

### 🟡 进阶级 Level 2：双指针与博弈贪心

### 题目 3：纪念品分组（洛谷 P1094）

> 有 n 件纪念品，价格为 `w[i]`。每组最多放 2 件，且价格之和不超过上限 W。求最少分几组。

**分析**：排序后使用双指针。最贵的和最便宜的能配对就配对，不能则最贵的单独一组。这样可以保证组数最少。

```
贪心策略：排序后双指针，最大配最小
正确性：如果最大和最小都配不上，最大只能单独一组
        如果能配上，配最小的一定不劣于配其他的（留更大空间给后续）
```

**C++ 代码**：

```cpp
const int MAXN = 30005;
int w[MAXN];

int solve(int n, int limit) {
    sort(w, w + n);
    int lo = 0, hi = n - 1, groups = 0;
    while (lo <= hi) {
        if (lo == hi) {
            groups++;
            break;
        }
        if (w[lo] + w[hi] <= limit) {
            lo++;  // 最小的能和最大的配对
        }
        hi--;      // 最大的（无论是否配对）都处理完了
        groups++;
    }
    return groups;
}
```

答案位置：`solve(n, W)` 的返回值

**核心启示**：
- **双指针是排序贪心的常见实现手段**：一头一尾向中间逼近
- **"能配就配，不能就独立"**：这种策略在配对问题中非常通用
- **交换论证**：如果最小的不配最大的而配了中间的，剩余的更难配对，不会更优

### 题目 4：三国游戏（洛谷 P1199）

> N 个武将，任意两人之间有默契值。你和电脑轮流选人（你先选），电脑每次会拆散你能组成的最大默契对。问你能保证获得的最大默契值是多少。

**分析**：博弈 + 贪心。关键洞察：**全场最大默契对一定会被电脑拆散**（你选 A，电脑选 B）。因此你能保证拿到的最大值，是所有武将中"与其他武将默契值的次大值"的最大者。对每个武将找次大默契值，取全局最大即可。

```
贪心策略：对每个武将 i，找它与其他武将默契值的次大值 sub[i]
答案为 max(sub[i])，人类必胜
正确性：最大值对必被电脑盯上拆掉，次大值对可以通过策略保住
```

**C++ 代码**：

```cpp
const int MAXN = 505;
int a[MAXN][MAXN];

int solve(int n) {
    int ans = 0;
    for (int i = 1; i <= n; i++) {
        int max1 = 0, max2 = 0;
        for (int j = 1; j <= n; j++) {
            if (i == j) continue;
            if (a[i][j] > max1) {
                max2 = max1;
                max1 = a[i][j];
            } else if (a[i][j] > max2) {
                max2 = a[i][j];
            }
        }
        ans = max(ans, max2);
    }
    return ans;
    // 输出 "1\n" 和 ans（人类必胜）
}
```

答案位置：输出 `1` 和 `ans`

**核心启示**：
- **博弈中的贪心思维**：对手会阻止你的最优选择，所以要退而求其次
- **"次大值的最大值"**：经典的对抗策略——最好的拿不到，就拿次好中最好的
- **O(N²) 暴力即可**：N ≤ 500，不需要复杂数据结构

### 题目 5：排队接水（洛谷 P1223）

> 有 n 个人排队打水，第 i 个人打水需要 t[i] 时间。确定打水顺序使得所有人的平均等待时间最小。

**分析**：排序贪心。按打水时间升序排列，让最快的人先打水。直觉：打水慢的人排在前面会让后面所有人多等，应把"拖后腿"最少的人放前面。用交换论证法可严格证明。

```
贪心策略：按 t[i] 升序排序
数学依据：第 k 个人的等待时间 = t[1]+t[2]+...+t[k-1]
         总等待时间 = Σ(n-i)*t[排第i个人]，t 小的系数大时总和最小
```

**C++ 代码**：

```cpp
const int MAXN = 1005;
int t[MAXN], idx[MAXN];

double solve(int n) {
    for (int i = 0; i < n; i++) idx[i] = i;
    sort(idx, idx + n, [](int a, int b) {
        return t[a] < t[b];
    });
    long long total = 0, wait = 0;
    for (int i = 0; i < n; i++) {
        total += wait;
        wait += t[idx[i]];
    }
    return (double)total / n;
}
```

答案位置：`solve(n)` 的返回值（总等待时间 `total` 和平均等待时间 `total/n`）

**核心启示**：
- **排序不等式的直接应用**：顺序和 ≥ 乱序和 ≥ 逆序和，耗时短的配大权重（被等次数多）
- **等待时间的数学建模**：第 i 个人的打水时间会被后面 n-i 个人"继承"为等待时间
- **O(n log n) 排序**是瓶颈，扫描仅 O(n)

### 题目 6：合并果子（洛谷 P1090 [NOIP2004]）

> 有 n 堆果子，每堆数量为 a[i]。每次合并两堆，代价为两堆之和。求将所有果子合并为一堆的最小总代价。

**分析**：哈夫曼编码思想。每次选最小的两堆合并，可以保证总代价最小。用优先队列（小根堆）维护当前所有堆。

```
贪心策略：每次取最小的两堆合并
正确性：哈夫曼树——频率小的节点深度大
         等价于：每个果子被计入代价的次数 = 它在合并树中的深度
         总代价 = Σ a[i] * depth[i]，要最小化则小的放深处
```

**C++ 代码**：

```cpp
long long solve(int a[], int n) {
    priority_queue<long long, vector<long long>, greater<long long>> pq;
    for (int i = 0; i < n; i++) pq.push(a[i]);
    long long cost = 0;
    while (pq.size() > 1) {
        long long x = pq.top(); pq.pop();
        long long y = pq.top(); pq.pop();
        cost += x + y;
        pq.push(x + y);
    }
    return cost;
}
```

答案位置：`solve(a, n)` 的返回值

**核心启示**：
- **哈夫曼思想**：最小两个先合并，使最终深度（被叠加次数）最大的是最小元素
- **优先队列是反复取极值的标配数据结构**：O(n log n) 总复杂度
- **从"合并代价"到"树的带权路径长度"的转化**是理解本题的关键

### 题目 7：凌乱的 yyy / 线段覆盖（洛谷 P1803）

> 有 n 个比赛（线段区间），每场比赛有起止时间 [l, r)。同一时间只能参加一场比赛。问最多能参加几场比赛。

**分析**：经典区间调度问题。按**右端点**升序排序，每次选择不与上一场冲突的、结束最早的比赛。

```
贪心策略：按右端点升序排序
选择规则：如果当前比赛的开始时间 ≥ 上一场的结束时间，就选它
正确性（交换论证）：选结束更早的一定不劣——它为后续留出更多时间
```

**C++ 代码**：

```cpp
const int MAXN = 1000005;
struct Seg { int l, r; } seg[MAXN];

int solve(int n) {
    sort(seg, seg + n, [](const Seg& a, const Seg& b) {
        return a.r < b.r;
    });
    int ans = 0, last = -1;
    for (int i = 0; i < n; i++) {
        if (seg[i].l >= last) {
            ans++;
            last = seg[i].r;
        }
    }
    return ans;
}
```

答案位置：`solve(n)` 的返回值

**核心启示**：
- **区间调度的标准策略**：按右端点排序是求最多不重叠区间的固定套路
- **交换论证的典型应用**：选结束更晚的区间，不可能比选结束更早的更优
- **注意开闭区间**：`[l, r)` 的情况下 `l >= last` 而非 `l > last`

### 题目 8：推销员（洛谷 P2672）

> 住户分布在一条直线上，第 i 家距入口 `s[i]` 米，推销疲劳值 `a[i]`。推销员从入口出发，对于 X = 1, 2, ..., N，分别求拜访恰好 X 家住户时的最大总疲劳值（走路疲劳 = 2 × 最远距离 + 推销疲劳之和）。

**分析**：对于每个 X，有两种候选方案：① 选 `a[i]` 最大的前 X 家，补上最远距离；② 选前 X-1 家最大 `a[i]`，第 X 家选"剩余中 `2*s[i]+a[i]` 最大的"。两者取 max。利用前缀/后缀预处理实现 O(n) 查询。

```
贪心策略：
  方案 A: top-X 的 a[] 之和 + 2 * 这 X 家中最远距离
  方案 B: top-(X-1) 的 a[] 之和 + 剩余中 max(2*s[i]+a[i])
  ans[X] = max(方案A, 方案B)
```

**C++ 代码**：

```cpp
const int MAXN = 100005;
int s[MAXN], a[MAXN], idx[MAXN];
long long sum[MAXN];  // a[] 前缀和（按 a 降序排列后）
long long q[MAXN];    // 前 i 个中 2*s 的最大值
long long h[MAXN];    // 后 i 个中 2*s+a 的最大值

void solve(int n) {
    for (int i = 0; i < n; i++) idx[i] = i;
    sort(idx, idx + n, [](int x, int y) { return a[x] > a[y]; });

    for (int i = 1; i <= n; i++) {
        int id = idx[i - 1];
        sum[i] = sum[i - 1] + a[id];
        q[i] = max(q[i - 1], (long long)2 * s[id]);
    }
    h[n + 1] = 0;
    for (int i = n; i >= 1; i--) {
        int id = idx[i - 1];
        h[i] = max(h[i + 1], (long long)2 * s[id] + a[id]);
    }
    for (int x = 1; x <= n; x++) {
        long long optA = sum[x] + q[x];
        long long optB = (x > 1 ? sum[x - 1] + h[x] : h[1]);
        // 输出 max(optA, optB)
    }
}
```

答案位置：对每个 X 输出 `max(optA, optB)`

**核心启示**：
- **多目标贪心的分方案讨论**：当一个目标由多个维度决定时，分别考虑各维度的贡献
- **前缀/后缀预处理**：将 O(n²) 暴力优化到 O(n)
- **"多选一"的反悔思想**：方案 B 本质是"放弃一个小 a，换一个远距离 + a 的综合收益"

### 题目 9：木棍加工（洛谷 P1233）

> n 根木棍，每根有长度 l 和宽度 w。机器加工一根新木棍需要 1 分钟准备时间，但如果下一根的 l 和 w 都不超过当前这根，则无需额外准备。求最少准备时间。

**分析**：最少准备时间 = 最少需要几次"重新启动" = 把木棍分成最少的递减链数。根据 **Dilworth 定理**：最少的递减链数 = 最长递增子序列长度。先按 l 降序（l 相同按 w 降序）排序，然后对 w 序列求最长严格递增子序列（LIS）。

```
Dilworth 定理：最少链覆盖数 = 最长反链长度
本题中：
  - "链"= 一组 l, w 均递减的木棍序列
  - "反链"= 一组两两不存在偏序关系的木棍
  按 l 降序排序后，对 w 求 LIS 即为答案
```

**C++ 代码**：

```cpp
const int MAXN = 5005;
struct Stick { int l, w; } stk[MAXN];
int dp[MAXN];  // dp[i]: 长度为 i+1 的 LIS 末尾最小值

int solve(int n) {
    sort(stk, stk + n, [](const Stick& a, const Stick& b) {
        return a.l != b.l ? a.l > b.l : a.w > b.w;
    });
    int len = 0;
    for (int i = 0; i < n; i++) {
        int pos = lower_bound(dp, dp + len, stk[i].w) - dp;
        dp[pos] = stk[i].w;
        if (pos == len) len++;
    }
    return len;
}
```

答案位置：`solve(n)` 的返回值

**核心启示**：
- **Dilworth 定理是偏序集问题的核心工具**：最少链覆盖 = 最长反链
- **二维偏序问题的标准处理**：一维排序消除，另一维用 LIS / LDS
- **O(n log n) 的 LIS**：用 `lower_bound` 维护单调序列是经典技巧

### 题目 10：导弹拦截（洛谷 P1020）

> 某国有一套导弹拦截系统，能拦截一组高度不升的导弹序列。给出 n 枚导弹的高度序列，求：(1) 一套系统最多拦截多少导弹；(2) 最少需要几套系统拦截所有导弹。

**分析**：问题 1 = 最长不升子序列（LDS），用类似 LIS 的 O(n log n) 贪心 + 二分求解。问题 2 由 **Dilworth 定理**得：最少不升链覆盖数 = 最长严格递增子序列长度（LIS）。

```
Q1: 最长不升子序列 → 维护单调不降序列，用 upper_bound 插入
Q2: Dilworth 定理 → 最少链覆盖 = 最长反链 = LIS
    维护单调递增序列，用 lower_bound 插入
```

**C++ 代码**：

```cpp
const int MAXN = 100005;
int h[MAXN], d[MAXN];

void solve(int n) {
    // Q1: 最长不升子序列（patience sorting 变体）
    int len1 = 0;
    for (int i = 0; i < n; i++) {
        int pos = upper_bound(d, d + len1, h[i], greater<int>()) - d;
        d[pos] = h[i];
        if (pos == len1) len1++;
    }
    // Q2: 最长严格递增子序列
    int len2 = 0;
    for (int i = 0; i < n; i++) {
        int pos = lower_bound(d, d + len2, h[i]) - d;
        d[pos] = h[i];
        if (pos == len2) len2++;
    }
    // len1 = 一套系统最多拦截数, len2 = 最少系统数
}
```

答案位置：`len1`（最多拦截数）和 `len2`（最少系统数）

**核心启示**：
- **Dilworth 定理连接两个看似不同的问题**：最少链划分 = 最长反链
- **O(n log n) 的 LIS/LDS**：贪心 + 二分是处理最长单调子序列的标准技巧
- **注意"不升"与"严格递增"**：`upper_bound` vs `lower_bound` 的选择决定是否允许相等

### 题目 11：雷达安装（洛谷 P1325）

> 海岸线为 x 轴，海中有 n 个岛屿坐标为 (x[i], y[i])。雷达安装在 x 轴上，探测半径为 d。求最少需要多少个雷达才能覆盖所有岛屿，若无法覆盖输出 -1。

**分析**：将每个岛屿转化为 x 轴上的可覆盖区间：以岛屿为圆心、d 为半径的圆与 x 轴的交线段 [x-√(d²-y²), x+√(d²-y²)]。问题转化为**最少点覆盖所有区间** = 区间调度（按右端点排序贪心）。

```
转化：岛屿 (x,y) → 区间 [x - sqrt(d²-y²), x + sqrt(d²-y²)]
前提：y > d 时无解
贪心策略：按区间右端点升序，每次雷达放在右端点
         后续区间若左端点 ≤ 当前雷达位置，则已覆盖
```

**C++ 代码**：

```cpp
const int MAXN = 1005;
struct Seg { double l, r; } seg[MAXN];

int solve(int n, int d, int x[], int y[]) {
    for (int i = 0; i < n; i++) {
        if (y[i] > d) return -1;
        double dx = sqrt((double)d * d - (double)y[i] * y[i]);
        seg[i] = {x[i] - dx, x[i] + dx};
    }
    sort(seg, seg + n, [](const Seg& a, const Seg& b) {
        return a.r < b.r;
    });
    int ans = 1;
    double last = seg[0].r;
    for (int i = 1; i < n; i++) {
        if (seg[i].l > last) {
            ans++;
            last = seg[i].r;
        }
    }
    return ans;
}
```

答案位置：`solve(n, d, x, y)` 的返回值

**核心启示**：
- **几何问题的区间化转化**：将二维覆盖问题降维为一维区间问题是经典技巧
- **区间调度的变体**：最少点覆盖所有区间 ≡ 贪心放置点在右端点
- **注意特判 y > d**：无法覆盖的情况必须提前处理

### 题目 12：国王游戏（洛谷 P1080 [NOIP2012]）

> 国王和 n 位大臣站成一排，每人左手写数字 a，右手写数字 b。第 i 位大臣获得的金币数 = 前面所有人（含国王）左手数字之积 ÷ 自己右手数字（下取整）。重新排列大臣顺序，使获得金币最多的大臣的金币数最小。

**分析**：邻项交换法经典题。设相邻两大臣 i, j，比较 i 在前和 j 在前哪种更优。设前面所有人左手之积为 P：
- i 在前：max(P/b[i], P×a[i]/b[j])
- j 在前：max(P/b[j], P×a[j]/b[i])

化简后得出排序关键字为 `a[i]*b[i]` 升序。注意本题需要**高精度乘法**。

```
邻项交换推导：
  i在前: 两人金币的较大值与 a[i]*b[i] 相关
  j在前: 两人金币的较大值与 a[j]*b[j] 相关
  若 a[i]*b[i] < a[j]*b[j]，则 i 放前面更优
排序标准：按 a[i]*b[i] 升序
特殊要求：乘积可能极大，需要高精度
```

**C++ 代码**：

```cpp
const int MAXN = 1005;
struct Minister { int a, b; } m[MAXN];

// 高精度乘法：大数（vector 存储）× 普通整数
vector<int> multiply(vector<int>& num, int x) {
    int carry = 0;
    for (int i = 0; i < (int)num.size(); i++) {
        long long cur = (long long)num[i] * x + carry;
        num[i] = cur % 10000;
        carry = cur / 10000;
    }
    while (carry) {
        num.push_back(carry % 10000);
        carry /= 10000;
    }
    return num;
}

// 高精度除法：大数 ÷ 普通整数
vector<int> divide(vector<int> num, int x) {
    int rem = 0;
    for (int i = (int)num.size() - 1; i >= 0; i--) {
        long long cur = (long long)rem * 10000 + num[i];
        num[i] = cur / x;
        rem = cur % x;
    }
    while (num.size() > 1 && num.back() == 0) num.pop_back();
    return num;
}

// 高精度比较
bool greater_than(vector<int>& a, vector<int>& b) {
    if (a.size() != b.size()) return a.size() > b.size();
    for (int i = (int)a.size() - 1; i >= 0; i--)
        if (a[i] != b[i]) return a[i] > b[i];
    return false;
}

void solve(int n) {
    // m[0] 为国王，m[1..n] 为大臣
    sort(m + 1, m + n + 1, [](const Minister& x, const Minister& y) {
        return x.a * x.b < y.a * y.b;
    });
    vector<int> product = {m[0].a};  // 前缀积，初始为国王的 a
    vector<int> ans = {0};
    for (int i = 1; i <= n; i++) {
        vector<int> coins = divide(product, m[i].b);
        if (greater_than(coins, ans)) ans = coins;
        multiply(product, m[i].a);
    }
    // ans 即为最小化的最大金币数（高精度输出）
}
```

答案位置：高精度数 `ans`

**核心启示**：
- **邻项交换法的标准流程**：假设相邻交换 → 列式比较 → 化简得排序标准
- **高精度是本题的实现难点**：前缀积可达数百位，必须用大数运算
- **排序标准 `a*b` 升序**：这是一个简洁而优美的结论，但需要严格推导

### 题目 13：皇后游戏（洛谷 P2123）

> 国王游戏的变体。n 位大臣各有 (a[i], b[i])，第 i 位大臣的奖金为：c[1] = a[1]+b[1]，c[i] = max(c[i-1], Σa[1..i]) + b[i]（i ≥ 2）。重新排列使 max(c[i]) 最小。

**分析**：同样用邻项交换法。比较相邻 i, j 交换前后 max(c[i], c[j]) 的大小。经过推导，排序规则为：i 排在 j 前面当且仅当 `min(a[i], b[j]) < min(a[j], b[i])`；若相等则按 `a[i] < a[j]` 为第二关键字。注意该比较函数可能不满足严格弱序，需小心处理。

```
邻项交换推导：
  i 在前: max(S + a[i] + b[i], S + a[i] + a[j] + b[j])
  j 在前: max(S + a[j] + b[j], S + a[j] + a[i] + b[i])
  化简后比较关键在于 min(a[i], b[j]) vs min(a[j], b[i])
排序标准：min(a[i], b[j]) < min(a[j], b[i]) 时 i 在前
注意：需要分类讨论确保传递性
```

**C++ 代码**：

```cpp
const int MAXN = 20005;
struct Minister { long long a, b; } m[MAXN];

bool cmp(const Minister& x, const Minister& y) {
    long long lhs = min(x.a, y.b);
    long long rhs = min(y.a, x.b);
    if (lhs != rhs) return lhs < rhs;
    return x.a < y.a;  // 第二关键字保证严格弱序
}

long long solve(int n) {
    sort(m, m + n, cmp);
    long long sumA = 0, last = 0, ans = 0;
    for (int i = 0; i < n; i++) {
        sumA += m[i].a;
        last = max(last, sumA) + m[i].b;
        ans = max(ans, last);
    }
    return ans;
}
```

答案位置：`solve(n)` 的返回值

**核心启示**：
- **邻项交换法的进阶应用**：推导过程比国王游戏更复杂，涉及 max 和 min 的化简
- **排序比较函数的传递性**：若比较函数不满足严格弱序，sort 行为未定义
- **与流水线调度的联系**：c[i] 的递推式本质上是两台机器的流水线调度问题

### 题目 14：梅深不见冬（洛谷 P5521 [yLOI2019]）

> 一棵 n 个节点的有根树，每个节点需要放 w[i] 朵梅花。规则：所有子节点放完后才能放父节点。可以回收子节点的梅花。DFS 遍历树，确定每个节点的子树遍历顺序，使得过程中树上同时存在的梅花数峰值最小。

**分析**：树上贪心。对每个节点的子节点确定最优遍历顺序。设 f[v] 为子树 v 的峰值花数，子节点按 `f[v] - w[v]` 降序排列遍历最优。直觉：`f[v]-w[v]` 越大表示该子树"临时额外占用"的资源越多，应优先处理以减小全局峰值。

```
贪心策略：
  对每个节点 u 的子节点 v1, v2, ..., vk
  按 f[vi] - w[vi] 降序排列
  DFS 顺序按此排列遍历

递推公式：
  叶子：f[u] = w[u]
  非叶：遍历子节点，维护 sum（已回收的梅花总量）
        f[u] = max(max(f[vi] + sum), total_w + w[u])
```

**C++ 代码**：

```cpp
const int MAXN = 200005;
vector<int> son[MAXN];
long long w[MAXN], f[MAXN];

void dfs(int u) {
    if (son[u].empty()) {
        f[u] = w[u];
        return;
    }
    for (int v : son[u]) dfs(v);
    sort(son[u].begin(), son[u].end(), [](int x, int y) {
        return f[x] - w[x] > f[y] - w[y];
    });
    long long sum = 0;
    f[u] = 0;
    for (int v : son[u]) {
        f[u] = max(f[u], f[v] + sum);
        sum += w[v];
    }
    f[u] = max(f[u], sum + w[u]);
}
```

答案位置：`f[root]`（根节点的峰值花数）

**核心启示**：
- **树上贪心的经典模式**：确定子节点遍历顺序以优化某个目标
- **"净额外消耗"排序**：`f[v]-w[v]` 衡量的是子树 v 除了最终贡献外的临时开销
- **邻项交换法同样适用于树上**：比较两个子节点谁先遍历更优，推导排序标准

### 题目 15：Work Scheduling（洛谷 P2949 [USACO09OPEN]）

> 有 n 项工作，每项有截止时间 d[i] 和报酬 p[i]，每项工作耗时 1 单位。在截止时间前完成可获得报酬。求最大总报酬。

**分析**：反悔贪心。按截止时间升序排序，依次尝试接受每项工作。维护一个小根堆存放已选工作的报酬。若已选工作数超过当前截止时间（时间不够），弹出报酬最小的工作——相当于"反悔"掉收益最低的选择。

```
贪心策略：按 d[i] 升序排序
反悔机制：堆中维护已选工作的报酬
         当 pq.size() > d[i] 时，弹出堆顶（最小报酬）
         等价于：用当前高报酬工作替换之前低报酬工作
```

**C++ 代码**：

```cpp
const int MAXN = 100005;
struct Job { int d, p; } job[MAXN];

long long solve(int n) {
    sort(job, job + n, [](const Job& a, const Job& b) {
        return a.d < b.d;
    });
    priority_queue<int, vector<int>, greater<int>> pq;
    long long ans = 0;
    for (int i = 0; i < n; i++) {
        pq.push(job[i].p);
        ans += job[i].p;
        if ((int)pq.size() > job[i].d) {
            ans -= pq.top();
            pq.pop();
        }
    }
    return ans;
}
```

答案位置：`solve(n)` 的返回值

**核心启示**：
- **反悔贪心的入门模板**：堆维护已选集合，遇到更优选择时替换最差的
- **"先全选再踢"的思路**：每项工作先假设选中，超出容量时踢掉最差的
- **O(n log n)** 排序 + 堆操作，比 DP 更直观高效

### 题目 16：建筑抢修（洛谷 P4053 [JSOI2007]）

> 有 n 座建筑需要抢修，第 i 座需要 t[i] 时间修复，必须在 d[i] 时刻前修完（即开工后连续修 t[i] 时间，完工时刻 ≤ d[i]）。同一时间只能修一座。求最多能修几座建筑。

**分析**：反悔贪心经典题。按截止时间 d[i] 升序排序。维护一个大根堆记录已选建筑的修复时间。若当前建筑可以在截止前完成，直接加入；否则，若当前建筑比堆中最耗时的建筑更快，则替换它（腾出时间）。

```
贪心策略：按 d[i] 升序排序
反悔机制：大根堆维护已选建筑的 t[i]
         若 cur + t[i] > d[i]，检查堆顶 t_max
         若 t[i] < t_max，替换（节省 t_max - t[i] 时间）
```

**C++ 代码**：

```cpp
const int MAXN = 150005;
struct Build { long long t, d; } b[MAXN];

int solve(int n) {
    sort(b, b + n, [](const Build& a, const Build& b) {
        return a.d < b.d;
    });
    priority_queue<long long> pq;
    long long cur = 0;
    for (int i = 0; i < n; i++) {
        if (cur + b[i].t <= b[i].d) {
            cur += b[i].t;
            pq.push(b[i].t);
        } else if (!pq.empty() && pq.top() > b[i].t) {
            cur -= pq.top();
            pq.pop();
            cur += b[i].t;
            pq.push(b[i].t);
        }
    }
    return (int)pq.size();
}
```

答案位置：`solve(n)` 的返回值

**核心启示**：
- **反悔贪心的"替换"思想**：选不了就看能不能换掉一个更差的
- **大根堆维护已选集合的最大耗时**：方便找到最值得替换的目标
- **与 P2949 的区别**：P2949 最大化收益（踢最小报酬），本题最大化数量（踢最大耗时）

### 题目 17：Buy Low Sell High（Codeforces CF865D）

> 已知未来 n 天的股票价格 p[i]。每天可以买入一股、卖出一股或不操作（同一天只能做一种操作）。求最大利润。

**分析**：反悔贪心。维护一个小根堆。遍历每天价格：若堆顶 < 当前价格，可以"卖出"获利（弹出堆顶，差价计入利润），同时将当前价格 push 两次——一次代表真实持有，一次作为"反悔票"（允许将来以更高价重新卖出）。否则仅 push 当前价格。

```
贪心策略：小根堆维护可选的买入价
反悔机制：卖出时 push 当前价格作为反悔票
         未来若有更高价，可弹出反悔票再获利
         效果等价于：低买 → 跳过中间 → 高卖
例子：价格 1, 3, 5
  第2天: 卖出获利 3-1=2, push 3（反悔票）
  第3天: 弹出 3, 获利 5-3=2, 总利润 4 = 实际 5-1
```

**C++ 代码**：

```cpp
long long solve(int p[], int n) {
    priority_queue<int, vector<int>, greater<int>> pq;
    long long ans = 0;
    for (int i = 0; i < n; i++) {
        if (!pq.empty() && pq.top() < p[i]) {
            ans += p[i] - pq.top();
            pq.pop();
            pq.push(p[i]);  // 反悔票
        }
        pq.push(p[i]);
    }
    return ans;
}
```

答案位置：`solve(p, n)` 的返回值

**核心启示**：
- **反悔票机制是反悔贪心的精髓**：通过多 push 一次当前值，允许将来"延长"交易
- **等价性**：低买中卖 + 中买高卖 = 低买高卖（利润叠加正确）
- **O(n log n)** 堆操作，优于 DP 的 O(n²) 或状态机 DP 的复杂建模

## 五、常见技巧与思路总结

### 5.1 排序是贪心的基石

绝大多数贪心题的第一步都是排序。常见排序标准：

| 问题类型 | 排序标准 | 代表题 |
|----------|----------|--------|
| 最小花费购买 | 单价升序 | 混合牛奶 |
| 配对最少组 | 值排序 + 双指针 | 纪念品分组 |
| 最多不重叠 | 右端点升序 | 线段覆盖 |
| 邻项交换类 | 推导得出的比较函数 | 国王游戏 |
| 二维偏序 | 一维降序 + 另一维 LIS | 木棍加工 |

### 5.2 交换论证法的通用模板

```
1. 设最优解为 OPT，贪心解为 G
2. 令 k 为 OPT 和 G 第一个不同的位置
3. 在 OPT 中将位置 k 的元素替换/交换为 G 的选择
4. 证明新解 OPT' 不劣于 OPT
5. 重复步骤 2-4，得到 OPT 最终 = G
```

### 5.3 双指针/首尾逼近

当问题具有"对称性"或"配对性"时，排序后用两个指针从两端向中间靠拢：

```cpp
int lo = 0, hi = n - 1;
while (lo < hi) {
    if (check(a[lo], a[hi])) {
        // 配对成功
        lo++; hi--;
    } else {
        hi--;  // 较大的无法配对，单独处理
    }
}
```

### 5.4 前缀/后缀预处理优化

当暴力贪心是 O(n²) 时，考虑用前缀和、前缀最大值、后缀最大值将查询优化到 O(1)：

```cpp
// 前缀最大值
for (int i = 1; i <= n; i++)
    preMax[i] = max(preMax[i-1], val[i]);
// 后缀最大值
for (int i = n; i >= 1; i--)
    sufMax[i] = max(sufMax[i+1], val[i]);
```

### 5.5 Dilworth 定理与偏序集

**Dilworth 定理**：在有限偏序集中，最少链覆盖数 = 最长反链长度。

应用：当问题要求把元素分成最少的"单调链"时，转化为求最长反向子序列。

### 5.6 高精度运算的处理策略

当贪心排序确定后，计算过程中的数值可能超出 long long 范围时：
- 使用 `vector<int>` 存储万进制大数
- 实现高精度 × 低精度、高精度 ÷ 低精度、高精度比较
- 或使用 Python 等原生支持大整数的语言

## 七、终极总结与归纳

### 7.1 贪心的本质

**贪心的本质是：在正确性可保证的前提下，用最简单的局部决策代替全局搜索。**

贪心不是"拍脑袋"的算法——每一个贪心策略背后，都应该有严格的正确性保证。它之所以高效，是因为在特定问题结构下（贪心选择性质 + 最优子结构），局部最优确实能推导出全局最优，从而省去了搜索/DP 的大量冗余计算。

### 7.2 核心要点

1. **贪心 = 排序 + 选择**：绝大多数贪心的第一步是确定排序标准
2. **正确性证明是灵魂**：交换论证法是最通用的证明工具
3. **邻项交换法是竞赛高频考点**：一定要熟练掌握推导流程
4. **Dilworth 定理连接贪心与 LIS**：二维偏序分链问题的核心工具
5. **反悔贪心扩展了贪心的适用范围**：堆维护可撤销选择
6. **数据范围决定实现细节**：大数可能需要高精度，大 n 需要前缀优化
