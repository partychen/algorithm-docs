## 一、什么是深度优先搜索

**深度优先搜索（DFS）** 是一种用于遍历或搜索树/图结构的基础算法。它的核心策略是：**从起点出发，沿着一条路径尽可能深地探索下去，直到无法继续时再回退（回溯），换一条路继续探索**。就像走迷宫时"一条路走到黑"，碰壁了再折返。

DFS 的执行天然依赖 **栈结构**——要么利用程序的 **递归调用栈**，要么显式维护一个栈。每当访问一个新节点时将其压栈，回退时弹栈。这种"后进先出"的特性决定了 DFS 总是优先深入而非扩展。

DFS 是算法竞赛和面试中出现频率最高的搜索策略之一，几乎所有涉及**排列组合、连通性判断、路径搜索、回溯枚举**的题目都离不开 DFS。掌握 DFS 及其变体，是解决搜索类问题的基石。

### DFS 遍历过程图示

```
        1
       / \
      2   5
     / \   \
    3   4   6

访问顺序：1 → 2 → 3 → (回溯到2) → 4 → (回溯到1) → 5 → 6

栈变化过程：
  [1]
  [1, 2]
  [1, 2, 3]    ← 到达叶子，回溯
  [1, 2]
  [1, 2, 4]    ← 到达叶子，回溯
  [1, 2]
  [1]
  [1, 5]
  [1, 5, 6]    ← 到达叶子，回溯
  [1, 5]
  [1]
  []            ← 搜索结束
```

### DFS vs BFS 对比

| 特性 | DFS（深度优先） | BFS（广度优先） |
|------|----------------|----------------|
| 数据结构 | 栈（递归/显式栈） | 队列 |
| 搜索方式 | 一条路走到底再回溯 | 逐层扩展 |
| 空间复杂度 | O(h)，h 为最大深度 | O(w)，w 为最大宽度 |
| 最短路（无权图） | ❌ 不保证 | ✅ 保证 |
| 适用场景 | 路径搜索、回溯、连通性、排列组合 | 最短路、层序遍历 |
| 实现复杂度 | 递归实现极简洁 | 需要队列，略繁琐 |
| 是否需要回溯 | 常需要 | 通常不需要 |

## 二、核心思想与前置知识

### 2.1 递归与系统栈

DFS 最自然的实现方式是 **递归**。每次递归调用会在系统栈上创建一个新的栈帧，保存局部变量和返回地址。当递归返回时，栈帧自动弹出，状态自然恢复——这正是"回溯"的底层机制。

需要注意的是，系统栈的深度有限（通常几千到几万层）。当搜索深度很大时（如 n > 10^5），可能导致 **栈溢出（Stack Overflow）**，此时需要改用显式栈的迭代写法。

### 2.2 回溯（Backtracking）思想

**回溯** 是 DFS 最核心的伴生思想，三步曲：

1. **做选择**：修改状态（标记访问、加入路径等）
2. **递归**：进入下一层搜索
3. **撤销选择**：恢复状态（取消标记、弹出路径等）

回溯保证搜索树的每条分支被独立探索，互不影响。并非所有 DFS 都需要回溯——如 Flood Fill 中标记后不需要恢复，因为每个节点只访问一次。

### 2.3 DFS 通用框架代码

```cpp
// DFS 通用框架（递归版）
void dfs(状态 state) {
    if (达到终止条件) {
        记录或更新答案;
        return;
    }
    for (auto& next : 所有可能的下一步选择) {
        if (next 不合法 || 已访问) continue;  // 剪枝
        做选择;       // 标记 vis / 修改状态
        dfs(next);    // 递归搜索
        撤销选择;     // 回溯（如果需要）
    }
}
```

这个框架涵盖了绝大多数 DFS 的场景。不同题目的差异在于：**状态如何定义、选择有哪些、终止条件是什么、是否需要回溯**。

### 2.4 时间复杂度分析

DFS 的时间复杂度取决于 **搜索空间的大小**：

| 问题类型 | 搜索空间 | 时间复杂度 |
|---------|---------|-----------|
| 图/网格遍历 | 每个节点访问一次 | O(V + E) 或 O(n × m) |
| 全排列 | n! 种排列 | O(n × n!) |
| 子集枚举 | 2^n 个子集 | O(n × 2^n) |
| N 皇后 | 最坏 n! | O(n!)，实际远小于此 |

**剪枝** 可以大幅减少实际搜索量，但不改变最坏时间复杂度。实际竞赛中，DFS + 有效剪枝往往能在合理时间内通过 n ≤ 20 甚至更大的搜索问题。

## 三、DFS 的子模型与分类详解

### 3.1 排列型 DFS（全排列、N 皇后）

**说明**：从 n 个元素中选出全部或部分，考虑**顺序**，枚举所有排列方案。

**适用信号**：题目要求"所有排列"、"全排列"、"不同顺序"、"逐行/逐位放置"。

```
搜索空间：n!（全排列）或 A(n,k)（部分排列）
关键技巧：vis[] 数组标记已使用元素
剪枝方向：对称性、冲突约束（如 N 皇后的列/对角线）
```

```cpp
int n, path[MAXN];
bool vis[MAXN];

void dfs(int dep) {
    if (dep == n) {
        // 处理一个完整排列 path[0..n-1]
        return;
    }
    for (int i = 0; i < n; i++) {
        if (vis[i]) continue;
        vis[i] = true;
        path[dep] = i;
        dfs(dep + 1);
        vis[i] = false;  // 回溯
    }
}
```

### 3.2 组合型 DFS（子集枚举、组合数）

**说明**：从 n 个元素中选若干个，**不考虑顺序**，枚举所有组合/子集。

**适用信号**：题目要求"所有子集"、"组合"、"选或不选"、"方案数"。

```
搜索空间：2^n（子集）或 C(n,k)（k 元组合）
关键技巧：用 start 参数保证选取顺序递增，避免重复
剪枝方向：元素和超限、剩余元素不足
```

```cpp
vector<int> path;

void dfs(int start, int n) {
    // 当前 path 即为一个合法子集，可在此处理
    for (int i = start; i < n; i++) {
        path.push_back(a[i]);
        dfs(i + 1, n);       // 从 i+1 开始，保证不重复
        path.pop_back();      // 回溯
    }
}
```

### 3.3 连通性 DFS（Flood Fill、连通块计数）

**说明**：在网格或图上，从某点出发，遍历所有相连的同类节点，用于判断连通性、计数连通块、填充区域。

**适用信号**：题目涉及"岛屿"、"连通块"、"区域填充"、"相邻传播"。

```
搜索空间：O(n × m)（网格）或 O(V + E)（图）
关键技巧：方向数组 dx[]/dy[]、原地标记或 vis[][] 数组
特点：标记后不需要回溯（每个节点只访问一次）
```

```cpp
int dx[] = {0, 0, 1, -1};
int dy[] = {1, -1, 0, 0};

void dfs(int x, int y) {
    vis[x][y] = true;
    for (int d = 0; d < 4; d++) {
        int nx = x + dx[d], ny = y + dy[d];
        if (nx >= 0 && nx < n && ny >= 0 && ny < m
            && !vis[nx][ny] && grid[nx][ny] == target)
            dfs(nx, ny);
    }
}
```

### 3.4 图遍历 DFS（有向图/无向图遍历、判环）

**说明**：在一般图上进行 DFS，可用于遍历、判环、求连通分量、拓扑排序辅助等。

**适用信号**：题目涉及"有向图"、"是否有环"、"连通分量"、"欧拉路径"。

```
搜索空间：O(V + E)
关键技巧：邻接表存图、三色标记法判环
有向图判环：访问中（灰色）节点被再次访问 → 有环
```

```cpp
vector<int> adj[MAXN];
int color[MAXN];  // 0=白(未访问) 1=灰(访问中) 2=黑(已完成)

bool hasCycle;

void dfs(int u) {
    color[u] = 1;
    for (int v : adj[u]) {
        if (color[v] == 1) { hasCycle = true; return; }
        if (color[v] == 0) dfs(v);
    }
    color[u] = 2;
}
```

### 3.5 树遍历 DFS（前中后序、树的直径等）

**说明**：在树结构上进行 DFS，利用"递归到底再返回"的特性，实现前/中/后序遍历，或通过返回值传递子树信息。

**适用信号**：题目涉及"二叉树遍历"、"树的深度/直径"、"子树统计"、"LCA"。

```
搜索空间：O(V)（树的节点数）
关键技巧：递归返回值携带子树信息
特点：树无环，无需 vis 数组（不走父节点即可）
```

```cpp
// 树的最大深度
int maxDepth(TreeNode* root) {
    if (!root) return 0;
    return max(maxDepth(root->left), maxDepth(root->right)) + 1;
}

// 树的直径（经过节点数）
int diameter;
int dfs(TreeNode* root) {
    if (!root) return 0;
    int L = dfs(root->left), R = dfs(root->right);
    diameter = max(diameter, L + R);  // 更新直径
    return max(L, R) + 1;
}
```

### 子模型对比总表

| 子模型 | 典型问题 | 搜索空间 | 是否回溯 | 关键技巧 | 时间复杂度 |
|--------|---------|---------|---------|---------|-----------|
| 排列型 | 全排列、N 皇后 | n! | ✅ 是 | `vis[]` 数组、对角线剪枝 | O(n!) |
| 组合型 | 子集、组合数 | 2^n | ✅ 是 | `start` 参数去重 | O(2^n) |
| 连通性 | Flood Fill、岛屿 | n × m | ❌ 否 | 方向数组、原地标记 | O(n × m) |
| 图遍历 | 判环、欧拉路径 | V + E | 视情况 | 三色标记、邻接表 | O(V + E) |
| 树遍历 | 深度、直径、LCA | V | ❌ 否 | 递归返回值 | O(V) |

### 题目 1：岛屿数量（LeetCode 200）

> 给定一个由 `'1'`（陆地）和 `'0'`（水）组成的二维网格，计算岛屿的数量。岛屿由水平或垂直方向上相邻的陆地连接而成，四周被水包围。

**分析**：经典 Flood Fill 入门题。遍历网格，遇到 `'1'` 就启动一次 DFS 把整个连通的陆地全部标记为已访问，每次启动 DFS 就是发现了一个新岛屿。

```
状态定义：dfs(x, y) 表示从 (x,y) 出发标记整块陆地
转移/递归逻辑：向上下左右四个方向扩展
边界条件：越界 或 非陆地 时返回
答案位置：主循环中 DFS 启动的次数
```

**C++ 代码**：

```cpp
const int MAXN = 305;
int n, m;
char grid[MAXN][MAXN];
int dx[] = {0, 0, 1, -1};
int dy[] = {1, -1, 0, 0};

void dfs(int x, int y) {
    grid[x][y] = '0';  // 原地标记，无需 vis 数组
    for (int d = 0; d < 4; d++) {
        int nx = x + dx[d], ny = y + dy[d];
        if (nx >= 0 && nx < n && ny >= 0 && ny < m && grid[nx][ny] == '1')
            dfs(nx, ny);
    }
}

int countIslands() {
    int ans = 0;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            if (grid[i][j] == '1') { ans++; dfs(i, j); }
    return ans;
}
```

答案位置：`countIslands()` 的返回值

**核心启示**：
- **原地标记**：将 `'1'` 改为 `'0'` 代替 `vis` 数组，节省空间
- **连通块计数模式**：外层循环 + 内层 DFS 是统计连通块的经典范式
- **方向数组**：`dx[]/dy[]` 是网格 DFS 的标配，避免写四个 if

### 题目 2：全排列问题（洛谷 P1706）

> 输出自然数 1 到 n 的所有全排列（n ≤ 9），按字典序输出，每个数占 5 个场宽。

**分析**：排列型 DFS 最基础的模板题。用 `vis[]` 数组标记已使用的数字，每层选一个未用过的数放入当前位置，递归到下一层。

```
状态定义：dfs(dep) 表示正在填第 dep 个位置
转移/递归逻辑：枚举 1~n 中未使用的数字填入 path[dep]
边界条件：dep == n+1 时输出当前排列
答案位置：所有到达叶子节点的路径
```

**C++ 代码**：

```cpp
const int MAXN = 12;
int n, path[MAXN];
bool vis[MAXN];

void dfs(int dep) {
    if (dep == n + 1) {
        // 输出 path[1..n]，每个数占 5 场宽
        return;
    }
    for (int i = 1; i <= n; i++) {
        if (vis[i]) continue;
        vis[i] = true;
        path[dep] = i;
        dfs(dep + 1);
        vis[i] = false;  // 回溯：释放数字 i
    }
}
// 调用：dfs(1)
```

答案位置：每次 `dep == n+1` 时的 `path[1..n]`

**核心启示**：
- **vis 数组是排列型 DFS 的核心**：标记"已选/未选"，回溯时必须恢复
- **字典序天然保证**：循环从小到大枚举，无需额外排序
- **排列 vs 组合**：排列型每层从 1 开始枚举，组合型从 `start` 开始

### 题目 3：二叉树的最大深度（LeetCode 104）

> 给定一棵二叉树，求其最大深度（根节点到最远叶节点的最长路径上的节点数）。

**分析**：树遍历 DFS 最经典的入门题。递归地计算左右子树的深度，取较大值加 1。

```
状态定义：dfs(node) 返回以 node 为根的子树的最大深度
转移/递归逻辑：max(左子树深度, 右子树深度) + 1
边界条件：空节点返回 0
答案位置：dfs(root) 的返回值
```

**C++ 代码**：

```cpp
struct TreeNode {
    int val;
    TreeNode *left, *right;
};

int maxDepth(TreeNode* root) {
    if (!root) return 0;
    return max(maxDepth(root->left), maxDepth(root->right)) + 1;
}
```

答案位置：`maxDepth(root)`

**核心启示**：
- **递归返回值**：树形 DFS 的精髓——通过返回值从子树向上传递信息
- **空节点作为 base case**：树递归的最常见边界条件
- **自底向上**：先递归子节点，再用子节点结果计算当前节点——后序遍历思想

### 题目 4：全排列（LeetCode 46）

> 给定一个不含重复数字的数组 `nums`，返回其所有可能的全排列。

**分析**：与 P1706 思路相同，但操作对象是数组元素。用 `vis[]` 标记已选元素，`path` 存储当前排列，回溯时弹出并取消标记。

```
状态定义：dfs() 时 path 中已有若干元素，继续填充
转移/递归逻辑：枚举所有未被选中的 nums[i] 加入 path
边界条件：path.size() == nums.size() 时记录答案
答案位置：所有完整排列收集到 ans 中
```

**C++ 代码**：

```cpp
vector<vector<int>> ans;
vector<int> path;
vector<bool> vis;

void dfs(vector<int>& nums) {
    if (path.size() == nums.size()) {
        ans.push_back(path);
        return;
    }
    for (int i = 0; i < (int)nums.size(); i++) {
        if (vis[i]) continue;
        vis[i] = true;
        path.push_back(nums[i]);
        dfs(nums);
        path.pop_back();
        vis[i] = false;
    }
}
// 初始化：vis.assign(nums.size(), false); dfs(nums);
```

答案位置：`ans` 中收集的所有排列

**核心启示**：
- **回溯三步曲清晰体现**：`push_back` → 递归 → `pop_back`
- **从固定范围到数组元素**：P1706 是 1~n，本题是任意数组，模式完全一致
- **vis 与 path 联动**：两者的"做/撤"必须同步

### 题目 5：单词搜索（LeetCode 79）

> 给定一个 m×n 的字符网格和一个字符串 `word`，判断网格中是否存在一条路径，依次经过相邻格子拼出该单词。每个格子最多使用一次。

**分析**：网格 DFS + 回溯。从每个格子出发尝试匹配 `word` 的第一个字符，然后向四方向递归匹配后续字符。关键在于**修改原数组代替 vis 数组**，回溯时恢复。

```
状态定义：dfs(x, y, k) 表示从 (x,y) 开始匹配 word[k..]
转移/递归逻辑：当前格匹配 word[k] 后，向四方向递归匹配 word[k+1]
边界条件：k == word.size() 匹配成功；越界/不匹配则失败
答案位置：任一起点返回 true 即可
```

**C++ 代码**：

```cpp
int n, m;
int dx[] = {0, 0, 1, -1};
int dy[] = {1, -1, 0, 0};

bool dfs(vector<vector<char>>& board, string& word, int x, int y, int k) {
    if (k == (int)word.size()) return true;
    if (x < 0 || x >= n || y < 0 || y >= m) return false;
    if (board[x][y] != word[k]) return false;
    char tmp = board[x][y];
    board[x][y] = '#';  // 标记已访问
    for (int d = 0; d < 4; d++) {
        if (dfs(board, word, x + dx[d], y + dy[d], k + 1))
            return true;
    }
    board[x][y] = tmp;  // 回溯：恢复原字符
    return false;
}
```

答案位置：枚举所有起点 `(i,j)`，任一 `dfs(board, word, i, j, 0)` 返回 `true` 即可

**核心启示**：
- **修改原数组代替 vis**：省空间且代码简洁，回溯时恢复原值即可
- **提前返回**：一旦找到答案立刻 `return true`，无需遍历所有可能
- **路径型 DFS**：每个节点在同一条路径上只能用一次，必须回溯

### 题目 6：Lake Counting（洛谷 P1596）

> 给定 N×M 的字符矩阵，`'W'` 表示有水。水洼由**八方向相邻**的 `'W'` 组成。求水洼的数量。

**分析**：与题目 1 几乎相同，区别在于 **八连通**（含对角线方向）。方向数组从 4 个扩展到 8 个。

```
状态定义：dfs(x, y) 表示从 (x,y) 出发标记整个水洼
转移/递归逻辑：向八个方向扩展
边界条件：越界 或 非 'W' 时返回
答案位置：主循环中 DFS 启动的次数
```

**C++ 代码**：

```cpp
const int MAXN = 105;
int n, m;
char grid[MAXN][MAXN];
int dx[] = {-1, -1, -1, 0, 0, 1, 1, 1};
int dy[] = {-1, 0, 1, -1, 1, -1, 0, 1};

void dfs(int x, int y) {
    grid[x][y] = '.';
    for (int d = 0; d < 8; d++) {
        int nx = x + dx[d], ny = y + dy[d];
        if (nx >= 0 && nx < n && ny >= 0 && ny < m && grid[nx][ny] == 'W')
            dfs(nx, ny);
    }
}

int countLakes() {
    int ans = 0;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            if (grid[i][j] == 'W') { ans++; dfs(i, j); }
    return ans;
}
```

答案位置：`countLakes()` 的返回值

**核心启示**：
- **四连通 vs 八连通**：唯一区别是方向数组大小（4 → 8）
- **Flood Fill 是连通性 DFS 的核心范式**：题目 1、题目 6 本质相同
- **复用已有思维**：从题目 1 到本题，只需改一行方向数组

### 题目 7：被围绕的区域（LeetCode 130）

> 给定 m×n 的矩阵，包含 `'X'` 和 `'O'`。将所有**被 `'X'` 完全包围的 `'O'` 区域**翻转为 `'X'`，但与边界相连的 `'O'` 不翻转。

**分析**：**逆向思维**——正面判断"是否被包围"很复杂，不如反过来：从边界上的 `'O'` 出发 DFS 标记所有与边界相连的 `'O'`，剩余未标记的 `'O'` 就是被包围的。

```
状态定义：dfs(x, y) 表示标记与边界相连的 'O'
转移/递归逻辑：向四方向扩展，遇到 'O' 则标记为 '#' 并继续
边界条件：越界 或 非 'O' 时返回
答案位置：遍历全图，'#' 恢复为 'O'，未标记的 'O' 改为 'X'
```

**C++ 代码**：

```cpp
int n, m;
int dx[] = {0, 0, 1, -1};
int dy[] = {1, -1, 0, 0};

void dfs(vector<vector<char>>& board, int x, int y) {
    if (x < 0 || x >= n || y < 0 || y >= m) return;
    if (board[x][y] != 'O') return;
    board[x][y] = '#';  // 标记为"与边界连通"
    for (int d = 0; d < 4; d++)
        dfs(board, x + dx[d], y + dy[d]);
}

void solve(vector<vector<char>>& board) {
    n = board.size(); m = board[0].size();
    for (int i = 0; i < n; i++) { dfs(board, i, 0); dfs(board, i, m - 1); }
    for (int j = 0; j < m; j++) { dfs(board, 0, j); dfs(board, n - 1, j); }
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++) {
            if (board[i][j] == '#') board[i][j] = 'O';
            else if (board[i][j] == 'O') board[i][j] = 'X';
        }
}
```

答案位置：`board` 被原地修改

**核心启示**：
- **边界 DFS / 逆向思维**：从"不被包围"的边界出发，比正面判断"被包围"简单得多
- **三态标记**：`'O'` → `'#'`（安全）→ 最后统一还原，是一种常用技巧
- **两轮遍历**：第一轮 DFS 标记安全区，第二轮全图扫描执行翻转

### 题目 8：N 皇后（LeetCode 51）

> 在 n×n 的棋盘上放置 n 个皇后，使得任意两个皇后不能处于同一行、同一列或同一对角线。返回所有不同的解法。

**分析**：逐行放置皇后，每行选一列。用三个布尔数组分别标记列、主对角线（`row-col`）、副对角线（`row+col`）是否被占用，实现 O(1) 冲突检测。

```
状态定义：dfs(row) 表示正在第 row 行放置皇后
转移/递归逻辑：枚举该行每一列，检查列和两条对角线是否冲突
边界条件：row == n 时记录一个合法解
答案位置：所有合法解收集到 ans 中
```

**C++ 代码**：

```cpp
int n;
vector<vector<string>> ans;
vector<string> board;
vector<bool> col, diag, anti;

void dfs(int row) {
    if (row == n) { ans.push_back(board); return; }
    for (int j = 0; j < n; j++) {
        if (col[j] || diag[row - j + n] || anti[row + j]) continue;
        board[row][j] = 'Q';
        col[j] = diag[row - j + n] = anti[row + j] = true;
        dfs(row + 1);
        board[row][j] = '.';
        col[j] = diag[row - j + n] = anti[row + j] = false;
    }
}
// 初始化：board(n, string(n, '.')); col(n), diag(2*n), anti(2*n) 全 false; dfs(0)
```

答案位置：`ans` 中收集的所有棋盘布局

**核心启示**：
- **约束传播**：三个布尔数组实现 O(1) 冲突判断，是 N 皇后的核心优化
- **对角线编码**：主对角线用 `row - col + n`，副对角线用 `row + col`
- **逐行搜索**：行不冲突是天然保证的（每行只放一个），只需检查列和对角线
- **回溯模式**：放置 → 标记 → 递归 → 恢复 → 取消标记

### 题目 9：八皇后（洛谷 P1219）

> 在 n×n 的棋盘上放 n 个皇后（n ≤ 13），输出前 3 组解（每组为每行皇后的列号），以及解的总数。

**分析**：与 LeetCode 51 本质相同，但是竞赛风格：需要输出前 3 组解并统计总数。1-indexed，使用数组而非 vector。

```
状态定义：dfs(row) 表示正在第 row 行放置皇后（1-indexed）
转移/递归逻辑：枚举列 1~n，检查 col/diag/anti 冲突
边界条件：row > n 时计数并可能输出
答案位置：cnt 为总解数，path[1..n] 存储每组解
```

**C++ 代码**：

```cpp
const int MAXN = 15;
int n, cnt;
int path[MAXN];
bool col[MAXN], diag[2 * MAXN], anti[2 * MAXN];

void dfs(int row) {
    if (row > n) {
        cnt++;
        if (cnt <= 3) {
            for (int i = 1; i <= n; i++)
                // 输出 path[i]（每行皇后所在列号）
                ;
        }
        return;
    }
    for (int j = 1; j <= n; j++) {
        if (col[j] || diag[row - j + n] || anti[row + j]) continue;
        path[row] = j;
        col[j] = diag[row - j + n] = anti[row + j] = true;
        dfs(row + 1);
        col[j] = diag[row - j + n] = anti[row + j] = false;
    }
}
// 调用：dfs(1); 最后输出 cnt
```

答案位置：`cnt` 为解的总数

**核心启示**：
- **LeetCode vs 竞赛**：同一问题在不同平台的代码风格差异（STL vs 数组，0-indexed vs 1-indexed）
- **部分输出**：只输出前几组解时，不必停止搜索，继续统计总数
- **竞赛版 N 皇后是面试版的自然延伸**，核心逻辑完全复用

### 题目 10：扫雷游戏（LeetCode 529）

> 给定一个扫雷棋盘，点击一个格子后模拟揭开逻辑：若是地雷 `'M'` 则变为 `'X'`；若是空白 `'E'`，计算周围 8 格的地雷数——若为 0 则标为 `'B'` 并递归揭开邻居，否则标为对应数字。

**分析**：八方向 DFS，但递归条件更复杂：只有当周围地雷数为 0 时才递归扩展。需要先统计再决定是否递归。

```
状态定义：dfs(x, y) 表示揭开格子 (x,y)
转移/递归逻辑：统计周围地雷数 cnt，cnt>0 标数字，cnt==0 标 'B' 并递归八方向
边界条件：越界 或 非 'E' 时返回
答案位置：board 被原地修改
```

**C++ 代码**：

```cpp
int dx[] = {-1, -1, -1, 0, 0, 1, 1, 1};
int dy[] = {-1, 0, 1, -1, 1, -1, 0, 1};

void dfs(vector<vector<char>>& board, int x, int y) {
    int n = board.size(), m = board[0].size();
    int cnt = 0;
    for (int d = 0; d < 8; d++) {
        int nx = x + dx[d], ny = y + dy[d];
        if (nx >= 0 && nx < n && ny >= 0 && ny < m && board[nx][ny] == 'M')
            cnt++;
    }
    if (cnt > 0) {
        board[x][y] = '0' + cnt;
    } else {
        board[x][y] = 'B';
        for (int d = 0; d < 8; d++) {
            int nx = x + dx[d], ny = y + dy[d];
            if (nx >= 0 && nx < n && ny >= 0 && ny < m && board[nx][ny] == 'E')
                dfs(board, nx, ny);
        }
    }
}
// 入口：若 click 位置为 'M' 则改为 'X'；否则调用 dfs(board, click_x, click_y)
```

答案位置：`board` 被原地修改

**核心启示**：
- **条件递归**：不是所有相邻节点都递归，只有满足条件（地雷数为 0）才扩展
- **先统计后决策**：先计算 `cnt`，再根据结果决定标数字还是递归
- **Flood Fill 的变体**：核心模式相同，但扩展条件更复杂

### 题目 11：太平洋大西洋水流问题（LeetCode 417）

> 给定 m×n 的高度矩阵，水可以从高处流向相邻的等高或低处。左边界和上边界连接太平洋，右边界和下边界连接大西洋。求所有既能流向太平洋又能流向大西洋的格子。

**分析**：**逆向 DFS**——正向从每个格子模拟水流太复杂。反过来：从太平洋边界出发"逆流而上"（只走 `>=` 当前高度的邻居），标记所有能到达太平洋的格子；同理对大西洋做一遍。两者交集即为答案。

```
状态定义：dfs(x, y, reach) 表示从 (x,y) 逆流标记可达的格子
转移/递归逻辑：向四方向扩展，邻居高度 >= 当前高度才可走
边界条件：越界 / 已标记 / 高度不够 时返回
答案位置：pac[i][j] && atl[i][j] 同时为 true 的所有 (i,j)
```

**C++ 代码**：

```cpp
int n, m;
int dx[] = {0, 0, 1, -1};
int dy[] = {1, -1, 0, 0};

void dfs(vector<vector<int>>& h, vector<vector<bool>>& reach, int x, int y) {
    reach[x][y] = true;
    for (int d = 0; d < 4; d++) {
        int nx = x + dx[d], ny = y + dy[d];
        if (nx >= 0 && nx < n && ny >= 0 && ny < m
            && !reach[nx][ny] && h[nx][ny] >= h[x][y])
            dfs(h, reach, nx, ny);
    }
}

vector<vector<int>> solve(vector<vector<int>>& h) {
    n = h.size(); m = h[0].size();
    vector<vector<bool>> pac(n, vector<bool>(m, false));
    vector<vector<bool>> atl(n, vector<bool>(m, false));
    for (int i = 0; i < n; i++) { dfs(h, pac, i, 0); dfs(h, atl, i, m - 1); }
    for (int j = 0; j < m; j++) { dfs(h, pac, 0, j); dfs(h, atl, n - 1, j); }
    vector<vector<int>> ans;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            if (pac[i][j] && atl[i][j]) ans.push_back({i, j});
    return ans;
}
```

答案位置：`solve(heights)` 返回的坐标列表

**核心启示**：
- **逆向 DFS 是核心思想**：从终点（海洋边界）反向搜索，化"多源到多汇"为"多源 Flood Fill"
- **两遍 DFS 取交集**：分别标记两个海洋的可达集，交集就是答案
- **复用题目 7 的思路**：题目 7 从边界标记安全区，本题从边界标记可达区，模式一致

### 题目 12：重新安排行程（LeetCode 332）

> 给定一组机票 `[from, to]`，所有机票都属于从 `JFK` 出发的行程。找出一条使用所有机票的行程，若有多条则返回字典序最小的。

**分析**：本质是在有向图中求 **欧拉路径**（经过每条边恰好一次）。使用 **Hierholzer 算法**：维护有序邻接表（multiset 保证字典序），DFS 时不断删除已用边，后序记录节点，最终反转即为答案。

```
状态定义：dfs(u) 表示从节点 u 出发继续使用剩余的边
转移/递归逻辑：每次取字典序最小的邻居，删边后递归，返回后将 u 加入路径
边界条件：节点无剩余边时自然返回
答案位置：route 反转后即为行程
```

**C++ 代码**：

```cpp
map<string, multiset<string>> graph;
vector<string> route;

void dfs(const string& u) {
    while (graph[u].size()) {
        string next = *graph[u].begin();
        graph[u].erase(graph[u].begin());
        dfs(next);
    }
    route.push_back(u);
}
// 构建 graph 后调用 dfs("JFK"); reverse(route.begin(), route.end());
```

答案位置：反转后的 `route`

**核心启示**：
- **Hierholzer 算法**：后序插入 + 反转 = 欧拉路径，是求欧拉回路/路径的经典算法
- **删边而非标记**：`erase` 已用边保证每条边恰好使用一次
- **multiset 保证字典序**：自动排序使得每次贪心选最小邻居
- **DFS 不仅仅是"搜索"**：它还可以用来构造路径，思维要灵活

### 题目 13：数的划分（洛谷 P1025）

> 将正整数 n 分成 k 份，每份不为空（允许相同），求方案总数。例如 n=7, k=3，合法划分有 `1+1+5, 1+2+4, 1+3+3, 2+2+3` 共 4 种。

**分析**：DFS + 剪枝。为避免重复计数，强制各份**不递减**。用 `minVal` 参数保证当前选的数不小于上一个。关键剪枝：当前数的上界为 `remain / (k - parts)`，因为后续每份都不小于当前选的值。

```
状态定义：dfs(remain, parts, minVal) 表示剩余 remain 要分成 k-parts 份，每份 >= minVal
转移/递归逻辑：枚举当前份的值 i，从 minVal 到 remain/(k-parts)
边界条件：parts == k 时检查 remain 是否为 0
答案位置：全局变量 ans 累加
```

**C++ 代码**：

```cpp
int n, k, ans;

void dfs(int remain, int parts, int minVal) {
    if (parts == k) {
        if (remain == 0) ans++;
        return;
    }
    // 上界剪枝：当前值 i 最大为 remain/(k-parts)
    for (int i = minVal; i <= remain / (k - parts); i++) {
        dfs(remain - i, parts + 1, i);
    }
}
// ans = 0; dfs(n, 0, 1);
```

答案位置：`ans`

**核心启示**：
- **不递减约束消除重复**：`minVal` 参数是组合型搜索中避免重复的经典手法
- **上界剪枝**：`remain / (k - parts)` 巧妙地限制了搜索范围，效果显著
- **搜索 vs 动态规划**：本题也可用 DP 解，但 DFS 思路更直观，剪枝后效率足够

### 题目 14：解数独（LeetCode 37）

> 填充一个 9×9 数独的空格，使得每行、每列、每个 3×3 宫内的数字 1-9 各出现一次。保证输入有唯一解。

**分析**：复杂约束回溯。遍历所有空格，对每个空格尝试填入 1-9，用三组布尔数组（行/列/宫）O(1) 检测冲突。找到第一个解即返回。

```
状态定义：dfs() 找到下一个空格并尝试填数
转移/递归逻辑：对空格尝试 1-9，检查 row/col/box 约束
边界条件：所有格子已填满返回 true；某空格无合法数字返回 false
答案位置：board 被原地修改
```

**C++ 代码**：

```cpp
bool row[9][10], col[9][10], box[9][10];

bool dfs(vector<vector<char>>& board) {
    for (int i = 0; i < 9; i++)
        for (int j = 0; j < 9; j++) {
            if (board[i][j] != '.') continue;
            int b = (i / 3) * 3 + j / 3;
            for (int d = 1; d <= 9; d++) {
                if (row[i][d] || col[j][d] || box[b][d]) continue;
                board[i][j] = '0' + d;
                row[i][d] = col[j][d] = box[b][d] = true;
                if (dfs(board)) return true;
                board[i][j] = '.';
                row[i][d] = col[j][d] = box[b][d] = false;
            }
            return false;  // 当前空格无合法数字 → 回溯
        }
    return true;  // 所有格子已填满 → 成功
}
// 先遍历 board 初始化 row/col/box，再调用 dfs(board)
```

答案位置：`board` 被原地修改

**核心启示**：
- **约束数组加速**：`row[i][d]/col[j][d]/box[b][d]` 三组数组实现 O(1) 冲突检测
- **宫编号公式**：`(i/3)*3 + j/3` 将 9 个 3×3 宫映射到 0-8
- **找到即返回**：`if (dfs(...)) return true` 保证找到唯一解后立即终止
- **"无合法数字"剪枝**：遍历 1-9 都不行时 `return false`，这是回溯的关键

### 题目 15：靶形数独（洛谷 P1074）

> 在 9×9 的数独棋盘中，每个格子有一个**权值**（由从外到内的圈数决定：最外圈权 6，次外圈权 7，…，中心格权 10）。在满足数独规则的前提下，使所有格子 `值 × 权值` 之和最大。若无解输出 -1。

**分析**：在解数独的基础上增加权值最大化目标。与题目 14 不同的是：需要遍历**所有**合法解找最优（而非找到一个就停）。关键优化：**按行空格数升序搜索**——空格少的行选择少，能更早剪枝。

```
状态定义：dfs(idx, score) 表示已填到第 idx 个空格，当前得分为 score
转移/递归逻辑：对当前空格尝试 1-9，累加 d * weight(i,j)
边界条件：所有空格填完时更新 ans = max(ans, score)
答案位置：全局变量 ans（初始化为 -1）
```

**C++ 代码**：

```cpp
bool row[9][10], col[9][10], box[9][10];
int board[9][9], ans = -1;

int weight(int i, int j) {
    return min({i, j, 8 - i, 8 - j}) + 6;
}

// blanks[] 存储空格坐标，按所在行的空格数升序排列
int blanks[81], tot;

void dfs(int idx, int score) {
    if (idx == tot) {
        ans = max(ans, score);
        return;
    }
    int i = blanks[idx] / 9, j = blanks[idx] % 9;
    int b = (i / 3) * 3 + j / 3;
    for (int d = 1; d <= 9; d++) {
        if (row[i][d] || col[j][d] || box[b][d]) continue;
        row[i][d] = col[j][d] = box[b][d] = true;
        dfs(idx + 1, score + d * weight(i, j));
        row[i][d] = col[j][d] = box[b][d] = false;
    }
}
```

答案位置：`ans`（若始终为 -1 则无解）

初始化说明：统计 `board` 中已有数字并设置 `row/col/box`；收集空格到 `blanks[]` 并按行空格数排序；计算已有数字的初始得分 `initScore`；调用 `dfs(0, initScore)`。

**核心启示**：
- **搜索顺序优化**：按空格数少的行优先搜索，大幅减少搜索树规模
- **从"求任意解"到"求最优解"**：不能找到一个就停，需遍历所有解取最优
- **极致剪枝组合**：约束数组 + 搜索顺序优化 + 及时回溯，三者缺一不可
- **竞赛题的核心竞争力**：正确性 + 优化 = AC，只有正确没有优化会 TLE

### 题目 16：单词接龙（洛谷 P1019）

> 给定 n（≤20）个单词和一个起始字母，进行单词接龙：相邻两个单词的连接处，后一个单词的前缀必须与前一个单词的后缀重叠（重叠部分合并，且不能完全包含）。每个单词最多使用两次，求能连成的最长龙的长度。

**分析**：预处理每对单词之间的**最短合法重叠长度**（重叠越短龙越长，贪心取最短）。从所有以起始字母开头的单词出发 DFS，每次尝试拼接所有可重叠的后继单词，维护当前总长度并更新全局最大值。每个单词使用次数 ≤ 2，回溯时恢复计数。

```
状态定义：dfs(last, curLen) 表示当前龙尾单词为 last，龙总长度为 curLen
转移/递归逻辑：枚举所有可接续的单词 i（used[i] < 2 且与 last 有合法重叠），拼接后递归
边界条件：无法继续接龙时更新 ans = max(ans, curLen)
答案位置：全局变量 ans
```

**C++ 代码**：

```cpp
string words[25];
int n, ans, used[25];
int ov[25][25];  // ov[i][j] = words[i] 后缀与 words[j] 前缀的最短重叠长度

void preprocess() {
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++) {
            int la = words[i].size(), lb = words[j].size();
            for (int l = 1; l < min(la, lb); l++) {
                if (words[i].substr(la - l) == words[j].substr(0, l)) {
                    ov[i][j] = l;
                    break;  // 取最短重叠，使龙尽量长
                }
            }
        }
}

void dfs(int last, int curLen) {
    ans = max(ans, curLen);
    for (int i = 0; i < n; i++) {
        if (used[i] < 2 && ov[last][i] > 0) {
            used[i]++;
            dfs(i, curLen + (int)words[i].size() - ov[last][i]);
            used[i]--;
        }
    }
}
// preprocess(); 对每个以起始字母开头的单词 s：used[s]=1; dfs(s, words[s].size()); used[s]=0;
```

答案位置：`ans`

**核心启示**：
- **预处理加速**：提前算好所有单词对的重叠长度，避免 DFS 中重复进行字符串匹配
- **最短重叠贪心**：重叠越短，拼出的龙越长，因此取第一个匹配的（最短的）重叠
- **使用次数上限**：`used[i] < 2` 控制每个单词最多用两次，回溯时递减
- **枚举起点**：龙头必须以指定字母开头，需要遍历所有满足条件的单词作为起点

### 题目 17：Scales（洛谷 P5194 [USACO05DEC]）

> 给定 N（≤1000）个砝码，质量递增排列，且从第 3 个起每个砝码质量不小于前两个之和（类斐波那契增长）。天平最大承重为 C（≤2³⁰）。从中选若干砝码放在天平上，总质量不超过 C，求最大可能的总质量。

**分析**：虽然 N 最大 1000，但砝码质量呈斐波那契式增长，不超过 2³⁰ 的有效砝码数仅约 45 个。从大到小逐个决策"选或不选"，配合**前缀和剪枝**：若当前已选质量加上剩余全部砝码仍不超 C，则直接全选。由于有效分支极少，搜索树非常小。

```
状态定义：dfs(idx, sum) 表示考虑第 idx 个砝码（从大到小），当前已选总质量为 sum
转移/递归逻辑：对当前砝码选或不选，选的条件是 sum + w[idx] ≤ C
边界条件：idx < 0 时更新 ans = max(ans, sum)
答案位置：全局变量 ans
```

**C++ 代码**：

```cpp
long long w[1010], pre[1010], C, ans;
int n;

void dfs(int idx, long long sum) {
    if (idx < 0) {
        ans = max(ans, sum);
        return;
    }
    // 剪枝：剩余全部砝码加上也不超 C，直接全选
    if (sum + pre[idx] <= C) {
        ans = max(ans, sum + pre[idx]);
        return;
    }
    // 不选当前砝码
    dfs(idx - 1, sum);
    // 选当前砝码（前提不超 C）
    if (sum + w[idx] <= C)
        dfs(idx - 1, sum + w[idx]);
}
// pre[0]=w[0]; pre[i]=pre[i-1]+w[i]（前缀和）; 调用 dfs(n-1, 0)
```

答案位置：`ans`

**核心启示**：
- **数据特性决定算法**：砝码呈斐波那契增长，有效个数极少（~45），DFS 完全可行
- **前缀和剪枝**：`sum + pre[idx] ≤ C` 时直接全选，一步到位，极大缩减搜索树
- **从大到小搜索**：优先决策大砝码，能更快逼近 C，使剪枝更早生效
- **看似大数据实为小规模**：N ≤ 1000 是烟雾弹，真正的搜索空间由增长速度决定

### 题目 18：【XR-2】奇迹（洛谷 P5440）

> 给定一个 8 位日期字符串（格式 YYYYMMDD），其中部分位为 `?`（未确定）。求有多少种合法填法，同时满足：① 日期真实存在（考虑闰年与每月天数，年份范围 1~9999）；② DD（日）为质数；③ MMDD（月日四位数）为质数；④ YYYYMMDD（完整八位数）为质数。多组查询。

**分析**：逐位 DFS 枚举 `?` 位的取值 0~9，搜索中根据已确定位进行范围剪枝（月份十位 ≤ 1、日十位 ≤ 3 等）。填满 8 位后验证日期合法性和三重质数条件。质数判断使用**欧拉筛**预处理至 10⁸。优化方案：预处理出所有满足全部条件的合法日期（实际数量极少），查询时逐位匹配即可。

```
状态定义：dfs(pos) 表示正在填第 pos 位（0~7）
转移/递归逻辑：若 s[pos] ≠ '?'，直接递归 pos+1；否则枚举 '0'~'9' 填入，带范围剪枝
边界条件：pos == 8 时检查日期合法性 + 三重质数条件，合法则 ans++
答案位置：全局变量 ans（每组查询前清零）
```

**C++ 代码**：

```cpp
bool notPrime[100000010];
int primes[6000010], cnt;
char s[9];
int ans;

void euler_sieve(int lim) {
    notPrime[0] = notPrime[1] = true;
    for (int i = 2; i <= lim; i++) {
        if (!notPrime[i]) primes[cnt++] = i;
        for (int j = 0; j < cnt && (long long)i * primes[j] <= lim; j++) {
            notPrime[i * primes[j]] = true;
            if (i % primes[j] == 0) break;
        }
    }
}

int days[] = {0,31,28,31,30,31,30,31,31,30,31,30,31};
bool isLeap(int y) { return (y%4==0 && y%100!=0) || y%400==0; }

bool validDate(int y, int m, int d) {
    if (y < 1 || m < 1 || m > 12 || d < 1) return false;
    int mx = days[m];
    if (m == 2 && isLeap(y)) mx = 29;
    return d <= mx;
}

void dfs(int pos) {
    if (pos == 8) {
        int y = (s[0]-'0')*1000+(s[1]-'0')*100+(s[2]-'0')*10+(s[3]-'0');
        int m = (s[4]-'0')*10+(s[5]-'0');
        int d = (s[6]-'0')*10+(s[7]-'0');
        if (!validDate(y, m, d)) return;
        int mmdd = m * 100 + d;
        long long full = (long long)y * 10000 + mmdd;
        if (!notPrime[d] && !notPrime[mmdd] && !notPrime[full]) ans++;
        return;
    }
    if (s[pos] != '?') { dfs(pos + 1); return; }
    for (char c = '0'; c <= '9'; c++) {
        if (pos == 0 && c == '0') continue;   // 年份首位不为 0
        if (pos == 4 && c > '1') break;       // 月份十位 ≤ 1
        if (pos == 6 && c > '3') break;       // 日十位 ≤ 3
        s[pos] = c;
        dfs(pos + 1);
    }
    s[pos] = '?';  // 回溯
}
// euler_sieve(99999999); 对每组输入读入 s，ans=0，dfs(0)，输出 ans
```

答案位置：`ans`

**核心启示**：
- **DFS + 数论交叉**：搜索负责枚举，数论（质数判定）负责验证，两者结合是竞赛常见模式
- **欧拉筛预处理**：对 ≤ 10⁸ 的质数一次性筛完，避免每次 O(√n) 判定
- **逐位剪枝**：月份、日期的高位有明确范围约束，尽早排除不合法分支
- **预处理 vs 在线搜索**：预处理所有合法日期后查询匹配是更优策略，展示了"离线思想"

### 题目 19：油滴扩展（洛谷 P1378）

> 在一个矩形框内有 N（≤6）个点。依次在每个点滴一滴油，油滴从该点向四周扩展，直到碰到矩形边界或已有油滴为止（不融合）。不同的滴油顺序会导致不同的覆盖面积。求所有顺序中，矩形剩余未被覆盖面积的最小值（四舍五入取整输出）。

**分析**：N ≤ 6，搜索空间为 6! = 720，可以 DFS 枚举全排列。对每种顺序模拟放置过程：第 i 滴油的扩展半径 r = min（到四条边的距离，到所有已放油滴边缘的距离）。若当前点已被某个油滴覆盖则半径为 0。累加所有油滴面积 πr²，取最大覆盖面积，最终输出矩形面积 − 最大覆盖面积。

```
状态定义：dfs(cnt, totalArea) 表示已放 cnt 个油滴，当前总覆盖面积为 totalArea
转移/递归逻辑：枚举未放的点 i，计算其扩展半径 r，递归 dfs(cnt+1, totalArea + π·r²)
边界条件：cnt == N 时更新 maxArea = max(maxArea, totalArea)
答案位置：round(矩形面积 − maxArea)
```

**C++ 代码**：

```cpp
const double PI = acos(-1.0);
int N;
double px[7], py[7], r[7];
double X1, Y1, X2, Y2;  // 矩形对角坐标
bool used[7];
double maxArea;

double calcR(int i) {
    double res = min({px[i]-X1, X2-px[i], py[i]-Y1, Y2-py[i]});
    for (int j = 0; j < N; j++) {
        if (!used[j] || j == i) continue;
        double dist = sqrt((px[i]-px[j])*(px[i]-px[j])
                         + (py[i]-py[j])*(py[i]-py[j]));
        res = min(res, dist - r[j]);
    }
    return max(res, 0.0);  // 被覆盖则半径为 0
}

void dfs(int cnt, double totalArea) {
    if (cnt == N) {
        maxArea = max(maxArea, totalArea);
        return;
    }
    for (int i = 0; i < N; i++) {
        if (used[i]) continue;
        used[i] = true;
        r[i] = calcR(i);
        dfs(cnt + 1, totalArea + PI * r[i] * r[i]);
        used[i] = false;
        r[i] = 0;
    }
}
// maxArea=0; dfs(0, 0); 输出 (int)round(abs((X2-X1)*(Y2-Y1)) - maxArea)
```

答案位置：`round(矩形面积 − maxArea)`

**核心启示**：
- **排列枚举经典应用**：N ≤ 6 暗示全排列暴搜，6! = 720 完全可承受
- **几何建模能力**：扩展半径需综合考虑边界距离和已有油滴的遮挡，公式推导是关键
- **模拟 + 搜索结合**：每种排列对应一次完整物理模拟，搜索框架套模拟逻辑
- **浮点精度处理**：使用 `double` 计算面积，最终四舍五入输出整数，注意精度陷阱

## 五、常见技巧与思路总结

### 5.1 回溯的"做选择-递归-撤销"三部曲

回溯是 DFS 最常见的配套技巧，模式固定：

```cpp
for (auto& choice : choices) {
    if (!isValid(choice)) continue;
    makeChoice(choice);       // 1. 做选择
    dfs(nextState);           // 2. 递归
    undoChoice(choice);       // 3. 撤销
}
```

常见的"做/撤"对应关系：

| 做选择 | 撤销选择 |
|--------|---------|
| `vis[i] = true` | `vis[i] = false` |
| `path.push_back(x)` | `path.pop_back()` |
| `board[x][y] = '#'` | `board[x][y] = tmp` |
| `col[j] = true` | `col[j] = false` |

### 5.2 visited 数组 vs 修改原数组

两种标记"已访问"的方式各有优劣：

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| `vis[][]` 数组 | 不修改原数据，安全 | 额外空间 | 需要保留原数据时 |
| 修改原数组 | 无额外空间 | 需要回溯恢复 | Flood Fill、路径搜索 |

**经验法则**：Flood Fill 类（不需要回溯）优先原地标记；路径搜索类（需要回溯）用 `vis` 更安全，或修改后记得恢复。

### 5.3 剪枝的基本策略

剪枝是 DFS 效率的生命线，常见策略：

1. **可行性剪枝**：当前状态已经不可能产生合法解，直接返回
   - 例：N 皇后中列/对角线冲突

2. **最优性剪枝**：当前状态的最优可能值已经不如已知最优解，直接返回
   - 例：`currentScore + upperBound <= bestScore` 时剪枝

3. **对称性剪枝**：利用对称性避免搜索等价的状态
   - 例：全排列中跳过重复元素

4. **搜索顺序优化**：优先搜索分支少的节点，更早触发剪枝
   - 例：靶形数独中按空格数排序

```cpp
// 最优性剪枝示例
void dfs(int idx, int curScore) {
    if (curScore + upperBound(idx) <= ans) return;  // 剪枝
    if (idx == n) { ans = max(ans, curScore); return; }
    // ...
}
```

### 5.4 DFS 与递归的栈溢出处理

当搜索深度超过系统栈限制时（通常递归 > 10^4 ~ 10^5 层），会发生栈溢出。应对方法：

1. **改用显式栈的迭代 DFS**：

```cpp
void dfs_iterative(int start) {
    stack<int> stk;
    stk.push(start);
    vis[start] = true;
    while (!stk.empty()) {
        int u = stk.top(); stk.pop();
        for (int v : adj[u]) {
            if (!vis[v]) {
                vis[v] = true;
                stk.push(v);
            }
        }
    }
}
```

2. **增大栈空间**：竞赛中可在编译选项或程序头部设置（平台相关）

### 5.5 DFS 序与时间戳

在图论中，DFS 过程可以为每个节点记录 **发现时间**（`dfn`，进入时）和 **完成时间**（`fin`，离开时），即 DFS 序。

```cpp
int timer = 0;
int dfn[MAXN], fin[MAXN];

void dfs(int u) {
    dfn[u] = ++timer;  // 发现时间
    for (int v : adj[u])
        if (!dfn[v]) dfs(v);
    fin[u] = ++timer;  // 完成时间
}
```

DFS 序的核心应用：
- **判断祖先关系**：u 是 v 的祖先 ⟺ `dfn[u] < dfn[v] && fin[u] > fin[v]`
- **求割点/桥**：Tarjan 算法基于 DFS 序和 `low` 值
- **子树转区间**：将树上子树操作转化为区间操作（配合线段树/树状数组）

## 七、终极总结与归纳

### 7.1 DFS 的本质

**一句话**：DFS 就是"一条路走到黑，碰壁就回头"的系统化搜索策略。

DFS 的本质是对搜索空间的 **树形展开与遍历**。无论原问题是图、网格、排列还是约束满足，DFS 都将其转化为一棵**搜索树**，然后用"深入-回溯"的方式遍历每一条分支。DFS 的强大在于它的 **通用性**——几乎所有搜索问题都可以用 DFS 框架表达，差异仅在于"状态怎么定义"和"选择有哪些"。

### 7.2 核心要点

1. **DFS = 栈 + 回溯**：递归天然利用系统栈，回溯保证分支独立探索
2. **框架统一**：所有 DFS 都遵循"终止判断 → 枚举选择 → 剪枝 → 做选择 → 递归 → 撤销"
3. **剪枝决定效率**：搜索空间往往是指数级的，有效剪枝是 DFS 能否 AC 的关键
4. **子模型要熟记**：排列型、组合型、连通性、图遍历、树遍历，各有固定模式
5. **逆向思维常有奇效**：从终点/边界反向搜索（如题目 7、11），化复杂为简单
6. **搜索顺序是隐藏优化**：优先处理约束强的分支（如靶形数独），可大幅加速
