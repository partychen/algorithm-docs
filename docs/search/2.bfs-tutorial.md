## 一、什么是广度优先搜索

广度优先搜索（BFS）就像向平静的水面投入一颗石子——波纹从中心向四周**一层一层、均匀扩散**。BFS 从起点出发，先访问所有距离为 1 的节点，再访问所有距离为 2 的节点，以此类推。这种"逐层推进"的特性使得 BFS 天然适合求解**最短路径**问题：当它第一次到达目标时，走过的步数一定是最少的。

与 DFS（深度优先搜索）"一条路走到黑再回头"不同，BFS 像是同时派出无数个探索者，它们齐头并进、步调一致。BFS 的核心数据结构是**队列（Queue）**——先进先出保证了层次的正确性。DFS 则依赖**栈**（或递归调用栈），适合探索所有可能路径、判断连通性等场景。

简单来说：**BFS 求"最短"，DFS 求"所有"**。当题目出现"最少步数""最短距离""最少操作次数"等关键词时，BFS 几乎是首选。

**BFS 的波纹扩散过程（ASCII 图示）**：

```
起点 S，目标 E，墙壁 #，空地 .

初始：           第 1 层：         第 2 层：         第 3 层：
. . . . .        . . . . .        . 2 . . .        3 2 . . .
. . # . .        . 1 # . .        2 1 # . .        2 1 # . .
. . # . .   →    . . # . .   →    . 2 # . .   →    3 2 # . .
. . . . .        . . . . .        . . . . .        . 2 . . .
. . . E .        . . . . .        . . . . .        . . . . .
  S=(1,1)         扩散 1 步         扩散 2 步         扩散 3 步
```

**BFS vs DFS 对比表**：

| 维度 | BFS（广度优先） | DFS（深度优先） |
|------|----------------|----------------|
| 数据结构 | 队列（Queue） | 栈 / 递归 |
| 搜索顺序 | 逐层扩展 | 一条路走到底 |
| 最短路保证 | ✅ 边权相同时保证 | ❌ 不保证 |
| 空间复杂度 | O(分支数^深度)，较大 | O(深度)，较小 |
| 典型应用 | 最短路径、层序遍历 | 连通性、拓扑排序、回溯 |
| 是否需要 visited | 几乎总是需要 | 常需要（防止环） |

## 二、核心思想与前置知识

### 2.1 队列数据结构

BFS 的灵魂是**队列（Queue）**。队列遵循 FIFO（先进先出）原则：最先入队的节点最先被处理，保证了搜索的层次顺序。

在 C++ 中，常用 `queue<T>` 或 `deque<T>`：

```cpp
queue<int> q;
q.push(start);      // 入队
int cur = q.front(); // 取队首
q.pop();             // 出队
q.empty();           // 判空
q.size();            // 队列大小
```

### 2.2 层次遍历思想

BFS 的"分层"能力是其最强大的武器。通过在每轮循环开始时记录当前队列大小 `sz = q.size()`，即可精确地处理当前层的所有节点，实现**按层统计、按层决策**。

关键性质：
- 同一层中的所有节点距起点的步数相同
- 第一次到达某节点时的步数即为最短步数
- 入队时标记 visited（而非出队时），避免重复入队

### 2.3 BFS 通用框架代码

```cpp
// BFS 通用骨架（以网格/图搜索为例）
// start: 起点, target: 目标状态

queue<State> q;
visited[start] = true;
q.push(start);
int step = 0;

while (!q.empty()) {
    int sz = q.size();
    while (sz--) {
        State cur = q.front(); q.pop();
        if (cur == target) return step;  // 找到目标
        for (State nxt : expand(cur)) {  // 扩展邻居
            if (!visited[nxt] && isValid(nxt)) {
                visited[nxt] = true;     // 入队时标记
                q.push(nxt);
            }
        }
    }
    step++;  // 层数 +1
}
return -1;  // 无法到达
```

### 2.4 时间/空间复杂度分析

| 场景 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 图（V 顶点，E 边） | O(V + E) | O(V) |
| 网格（n × m） | O(n × m) | O(n × m) |
| 状态图（S 个状态） | O(S × 分支因子) | O(S) |
| 树（N 个节点） | O(N) | O(最宽层节点数) |

BFS 的空间瓶颈在于队列可能同时存储一整层的节点。对于宽度很大的搜索空间（如状态图 BFS），内存是主要限制。

## 三、BFS 的子模型与分类详解

### 3.1 网格 BFS（迷宫最短路）

**简要说明**：在二维网格上搜索最短路径，每个格子是一个状态，相邻格子之间的边权为 1。

**适用信号**：题目给出二维网格/矩阵，要求"最短路径""最少步数"到达某个位置。

```cpp
// 网格 BFS 模板
int dx[] = {0, 0, 1, -1}, dy[] = {1, -1, 0, 0};

int gridBFS(vector<vector<int>>& grid, int sx, int sy, int ex, int ey) {
    int n = grid.size(), m = grid[0].size();
    vector<vector<int>> dist(n, vector<int>(m, -1));
    queue<pair<int,int>> q;
    dist[sx][sy] = 0;
    q.push({sx, sy});
    while (!q.empty()) {
        auto [x, y] = q.front(); q.pop();
        if (x == ex && y == ey) return dist[x][y];
        for (int d = 0; d < 4; d++) {
            int nx = x + dx[d], ny = y + dy[d];
            if (nx >= 0 && nx < n && ny >= 0 && ny < m
                && grid[nx][ny] == 0 && dist[nx][ny] == -1) {
                dist[nx][ny] = dist[x][y] + 1;
                q.push({nx, ny});
            }
        }
    }
    return -1;
}
```

### 3.2 图 BFS（无权图最短路）

**简要说明**：在无权图（或等权图）上求单源最短路。每条边的代价相同，BFS 天然保证最短。

**适用信号**：给出图的邻接关系，求从某点到另一点的最少步数/最短距离。

```cpp
// 图 BFS 模板
vector<int> adj[MAXN];
int dist[MAXN];

void graphBFS(int start, int n) {
    memset(dist, -1, sizeof(dist));
    queue<int> q;
    dist[start] = 0;
    q.push(start);
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int v : adj[u]) {
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
}
```

### 3.3 多源 BFS（多个起点同时扩散）

**简要说明**：多个起点同时开始 BFS，等价于添加一个"超级源点"连接所有实际起点。初始时将所有起点一起入队，距离均设为 0。

**适用信号**："所有 X 同时开始扩散""求每个格子到最近 X 的距离""多个起点同时腐烂/感染"。

```cpp
// 多源 BFS 模板
int dx[] = {0, 0, 1, -1}, dy[] = {1, -1, 0, 0};

void multiSourceBFS(vector<vector<int>>& grid, int n, int m) {
    vector<vector<int>> dist(n, vector<int>(m, -1));
    queue<pair<int,int>> q;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            if (grid[i][j] == SOURCE) {
                dist[i][j] = 0;
                q.push({i, j});
            }
    while (!q.empty()) {
        auto [x, y] = q.front(); q.pop();
        for (int d = 0; d < 4; d++) {
            int nx = x + dx[d], ny = y + dy[d];
            if (nx >= 0 && nx < n && ny >= 0 && ny < m && dist[nx][ny] == -1) {
                dist[nx][ny] = dist[x][y] + 1;
                q.push({nx, ny});
            }
        }
    }
}
```

### 3.4 层次遍历 BFS（树的层序遍历）

**简要说明**：对树按层访问，每层的节点放在一组。通过记录每层的 `q.size()` 实现分层。

**适用信号**："层序遍历""每层的最大值/平均值""最小深度""最左/最右节点"。

```cpp
// 树的层序遍历模板
vector<vector<int>> levelOrder(TreeNode* root) {
    vector<vector<int>> res;
    if (!root) return res;
    queue<TreeNode*> q;
    q.push(root);
    while (!q.empty()) {
        int sz = q.size();
        vector<int> level;
        while (sz--) {
            auto node = q.front(); q.pop();
            level.push_back(node->val);
            if (node->left) q.push(node->left);
            if (node->right) q.push(node->right);
        }
        res.push_back(level);
    }
    return res;
}
```

### 3.5 状态图 BFS（状态压缩、密码锁等）

**简要说明**：将问题的"状态"作为图的节点，状态之间的合法转换作为边。常用字符串或整数编码状态，用 `unordered_set` 判重。

**适用信号**："最少操作次数""从初始状态变换到目标状态""密码锁/拼图/魔方"。

```cpp
// 状态图 BFS 模板
int stateBFS(string start, string target) {
    if (start == target) return 0;
    unordered_set<string> vis;
    queue<string> q;
    q.push(start);
    vis.insert(start);
    int step = 0;
    while (!q.empty()) {
        step++;
        int sz = q.size();
        while (sz--) {
            string cur = q.front(); q.pop();
            for (string nxt : getNextStates(cur)) {
                if (nxt == target) return step;
                if (!vis.count(nxt)) {
                    vis.insert(nxt);
                    q.push(nxt);
                }
            }
        }
    }
    return -1;
}
```

### 3.6 01-BFS（边权只有 0/1 的最短路）

**简要说明**：当图的边权只有 0 和 1 时，使用**双端队列（deque）**代替普通队列：权为 0 的边加入队首，权为 1 的边加入队尾。这样无需 Dijkstra 也能 O(V+E) 求最短路。

**适用信号**："边权 0 或 1""翻转代价为 1，不翻转为 0""经过某种格子代价不同"。

```cpp
// 01-BFS 模板
int dist[MAXN];

void bfs01(int start, int n) {
    memset(dist, 0x3f, sizeof(dist));
    deque<int> dq;
    dist[start] = 0;
    dq.push_back(start);
    while (!dq.empty()) {
        int u = dq.front(); dq.pop_front();
        for (auto [v, w] : adj[u]) {  // w 只取 0 或 1
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                if (w == 0) dq.push_front(v);
                else dq.push_back(v);
            }
        }
    }
}
```

### 模型对比总表

| 模型 | 数据结构 | 状态表示 | 边权 | 典型场景 |
|------|---------|---------|------|---------|
| 网格 BFS | queue | (x, y) | 均为 1 | 迷宫最短路 |
| 图 BFS | queue | 节点编号 | 均为 1 | 无权图最短路 |
| 多源 BFS | queue | (x, y) 或节点 | 均为 1 | 多起点同时扩散 |
| 层次遍历 | queue | TreeNode* | 均为 1 | 树的按层处理 |
| 状态图 BFS | queue | string/int | 均为 1 | 拼图、密码锁 |
| 01-BFS | deque | 节点编号 | 0 或 1 | 混合代价最短路 |

### 题目 1：二叉树的层序遍历（LeetCode 102）

> 给定一棵二叉树，返回其节点值的层序遍历结果，即逐层从左到右访问所有节点。

**分析**：最经典的 BFS 入门题。每次循环处理一整层，用 `q.size()` 控制当前层的节点数。

```
状态定义：每个树节点为一个状态
转移/扩展逻辑：当前节点 → 左子节点、右子节点
边界条件：根节点为空则返回空数组
答案位置：每层收集的节点值组成结果数组
```

**C++ 代码**：

```cpp
vector<vector<int>> levelOrder(TreeNode* root) {
    vector<vector<int>> res;
    if (!root) return res;
    queue<TreeNode*> q;
    q.push(root);
    while (!q.empty()) {
        int sz = q.size();
        vector<int> level;
        while (sz--) {
            auto node = q.front(); q.pop();
            level.push_back(node->val);
            if (node->left) q.push(node->left);
            if (node->right) q.push(node->right);
        }
        res.push_back(level);
    }
    return res;
}
```

**核心启示**：
- **`sz = q.size()` 是分层的关键**：在内层循环开始前记录当前层节点数，避免新入队节点干扰当前层处理
- **BFS 的"模板性"极强**：几乎所有 BFS 题目都是在此骨架上微调扩展逻辑
- **入队时机**：子节点存在才入队，天然处理了空节点的边界

### 题目 2：岛屿数量（LeetCode 200）

> 给定一个 `m × n` 的二维网格，其中 `'1'` 表示陆地，`'0'` 表示水域。计算岛屿的数量。岛屿由水平或垂直相邻的陆地连接而成。

**分析**：遍历网格，遇到未访问的 `'1'` 就启动一次 BFS 将整个连通块标记，计数器 +1。用原地修改 `'1'→'0'` 代替 visited 数组。

```
状态定义：每个格子 (x, y) 为一个状态
转移/扩展逻辑：向上下左右四个方向扩展，目标格子为 '1' 才入队
边界条件：不越界 && 格子值为 '1'
答案位置：BFS 启动次数即为岛屿数
```

**C++ 代码**：

```cpp
int dx[] = {0, 0, 1, -1}, dy[] = {1, -1, 0, 0};

int numIslands(vector<vector<char>>& grid) {
    int m = grid.size(), n = grid[0].size(), ans = 0;
    for (int i = 0; i < m; i++)
        for (int j = 0; j < n; j++)
            if (grid[i][j] == '1') {
                ans++;
                queue<pair<int,int>> q;
                q.push({i, j});
                grid[i][j] = '0';
                while (!q.empty()) {
                    auto [x, y] = q.front(); q.pop();
                    for (int d = 0; d < 4; d++) {
                        int nx = x + dx[d], ny = y + dy[d];
                        if (nx >= 0 && nx < m && ny >= 0 && ny < n
                            && grid[nx][ny] == '1') {
                            grid[nx][ny] = '0';
                            q.push({nx, ny});
                        }
                    }
                }
            }
    return ans;
}
```

**核心启示**：
- **原地标记代替 visited**：将已访问的 `'1'` 改为 `'0'`，省去额外空间
- **"入队即标记"原则**：在 `push` 时就修改状态，而非 `pop` 时，避免同一节点被多次入队
- **BFS 求连通块**是网格类题目的基本功，可迁移到"染色""计数"等问题

### 题目 3：马的遍历（洛谷 P1443）

> 在 `n × m` 的棋盘上，给定一个马的初始位置 `(sx, sy)`，求马到棋盘上每个格子的最少步数。马走"日"字。

**分析**：标准的网格 BFS，与迷宫最短路的唯一区别是方向数组——马有 8 个跳跃方向（日字形）。

```
状态定义：(x, y) 为马当前所在位置
转移/扩展逻辑：8 个日字方向跳跃
边界条件：不越出棋盘 && 未被访问
答案位置：dist[i][j] 即为马到 (i, j) 的最少步数
```

**C++ 代码**：

```cpp
const int MAXN = 405;
int dist[MAXN][MAXN];
int dx[] = {-2,-2,-1,-1,1,1,2,2};
int dy[] = {-1,1,-2,2,-2,2,-1,1};

void solve(int n, int m, int sx, int sy) {
    memset(dist, -1, sizeof(dist));
    queue<pair<int,int>> q;
    dist[sx][sy] = 0;
    q.push({sx, sy});
    while (!q.empty()) {
        auto [x, y] = q.front(); q.pop();
        for (int d = 0; d < 8; d++) {
            int nx = x + dx[d], ny = y + dy[d];
            if (nx >= 1 && nx <= n && ny >= 1 && ny <= m
                && dist[nx][ny] == -1) {
                dist[nx][ny] = dist[x][y] + 1;
                q.push({nx, ny});
            }
        }
    }
}
```

**核心启示**：
- **方向数组是 BFS 模板的唯一变量**：从 4 方向到 8 方向、日字跳跃，只需修改 `dx/dy`
- **-1 做"未访问"标记**：初始化为 -1 既可判断是否访问过，又可作为无法到达的标志
- **单源全图 BFS**：一次 BFS 即可求出起点到所有可达点的最短距离

### 题目 4：01 矩阵（LeetCode 542）

> 给定一个 `m × n` 的 0-1 矩阵 `mat`，求每个格子到最近的 `0` 的曼哈顿距离。

**分析**：典型多源 BFS。将所有值为 `0` 的格子作为起点同时入队，同步向外扩散，每个 `1` 格子第一次被到达时的距离即为答案。

```
状态定义：(x, y) 为格子坐标
转移/扩展逻辑：上下左右四方向扩展
边界条件：所有 0 格子初始距离为 0 并入队
答案位置：dist[i][j] 即为格子 (i,j) 到最近 0 的距离
```

**C++ 代码**：

```cpp
int dx[] = {0, 0, 1, -1}, dy[] = {1, -1, 0, 0};

vector<vector<int>> updateMatrix(vector<vector<int>>& mat) {
    int m = mat.size(), n = mat[0].size();
    vector<vector<int>> dist(m, vector<int>(n, -1));
    queue<pair<int,int>> q;
    for (int i = 0; i < m; i++)
        for (int j = 0; j < n; j++)
            if (mat[i][j] == 0) { dist[i][j] = 0; q.push({i, j}); }
    while (!q.empty()) {
        auto [x, y] = q.front(); q.pop();
        for (int d = 0; d < 4; d++) {
            int nx = x + dx[d], ny = y + dy[d];
            if (nx >= 0 && nx < m && ny >= 0 && ny < n && dist[nx][ny] == -1) {
                dist[nx][ny] = dist[x][y] + 1;
                q.push({nx, ny});
            }
        }
    }
    return dist;
}
```

**核心启示**：
- **"反向思维"是多源 BFS 的钥匙**：不是从每个 1 找最近的 0（暴力），而是从所有 0 同时出发
- **超级源点思想**：多源 BFS 等价于虚拟一个超级源点连接所有实际源点，边权为 0
- **多源 BFS 和单源 BFS 代码几乎一样**，唯一区别是初始入队多个节点

### 题目 5：腐烂的橘子（LeetCode 994）

> 网格中 `0` 表示空，`1` 表示新鲜橘子，`2` 表示腐烂橘子。每分钟，腐烂橘子的四邻会被感染。求所有橘子变腐烂的最少时间，若不可能则返回 -1。

**分析**：多源 BFS 的经典应用。所有初始腐烂橘子同时入队，按层扩散，层数即为时间。最后检查是否还有新鲜橘子。

```
状态定义：(x, y) 为格子坐标
转移/扩展逻辑：腐烂橘子向四邻的新鲜橘子扩散
边界条件：所有值为 2 的格子初始入队；统计新鲜橘子总数
答案位置：BFS 层数即时间；若结束后 fresh > 0 则返回 -1
```

**C++ 代码**：

```cpp
int dx[] = {0, 0, 1, -1}, dy[] = {1, -1, 0, 0};

int orangesRotting(vector<vector<int>>& grid) {
    int m = grid.size(), n = grid[0].size(), fresh = 0, ans = 0;
    queue<pair<int,int>> q;
    for (int i = 0; i < m; i++)
        for (int j = 0; j < n; j++) {
            if (grid[i][j] == 2) q.push({i, j});
            if (grid[i][j] == 1) fresh++;
        }
    while (!q.empty() && fresh > 0) {
        int sz = q.size();
        while (sz--) {
            auto [x, y] = q.front(); q.pop();
            for (int d = 0; d < 4; d++) {
                int nx = x + dx[d], ny = y + dy[d];
                if (nx >= 0 && nx < m && ny >= 0 && ny < n
                    && grid[nx][ny] == 1) {
                    grid[nx][ny] = 2;
                    fresh--;
                    q.push({nx, ny});
                }
            }
        }
        ans++;
    }
    return fresh == 0 ? ans : -1;
}
```

**核心启示**：
- **"同时扩散"模型**：现实中的传染、火灾蔓延等过程都可建模为多源 BFS
- **`fresh` 计数器**：用于判断是否所有目标都被覆盖，比遍历整个网格更高效
- **`while (fresh > 0)` 提前终止**：所有新鲜橘子都腐烂后无需继续 BFS

### 题目 6：二叉树的最小深度（LeetCode 111）

> 给定一棵二叉树，找到从根节点到最近叶子节点的最短路径长度。叶子节点是没有子节点的节点。

**分析**：BFS 层序遍历，遇到第一个叶子节点就返回当前深度。BFS 保证第一个遇到的叶子一定在最浅层。

```
状态定义：树节点作为状态
转移/扩展逻辑：左右子节点
边界条件：空树返回 0
答案位置：第一个叶子节点所在的层数
```

**C++ 代码**：

```cpp
int minDepth(TreeNode* root) {
    if (!root) return 0;
    queue<TreeNode*> q;
    q.push(root);
    int depth = 0;
    while (!q.empty()) {
        depth++;
        int sz = q.size();
        while (sz--) {
            auto node = q.front(); q.pop();
            if (!node->left && !node->right) return depth;
            if (node->left) q.push(node->left);
            if (node->right) q.push(node->right);
        }
    }
    return depth;
}
```

**核心启示**：
- **BFS 天然求"最近/最小"**：相比 DFS 需要遍历所有路径取最小值，BFS 找到即最优
- **叶子节点判定**：`!left && !right`，是树上 BFS 的常见终止条件
- **DFS vs BFS 选择**：当树极度不平衡（一侧很深一侧很浅）时，BFS 远优于 DFS

### 题目 7：填涂颜色（洛谷 P1162）

> 给定 `n × n` 的 0-1 矩阵，由 `1` 围成一个封闭圈。将被 `1` 围住的所有 `0` 改为 `2` 后输出。

**分析**：从矩阵边界的所有 `0` 出发做 BFS，标记所有与边界连通的 `0`。剩余未标记的 `0` 就是被 `1` 包围的区域，输出时改为 `2`。

```
状态定义：(x, y) 为格子坐标
转移/扩展逻辑：四方向扩展，目标格子为 0 且未访问
边界条件：边界上所有值为 0 的格子作为起点
答案位置：未被 BFS 标记的 0 即为被围住的区域
```

**C++ 代码**：

```cpp
const int MAXN = 35;
int a[MAXN][MAXN], vis[MAXN][MAXN];
int dx[] = {0, 0, 1, -1}, dy[] = {1, -1, 0, 0};

void solve(int n) {
    queue<pair<int,int>> q;
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= n; j++)
            if ((i == 1 || i == n || j == 1 || j == n) && a[i][j] == 0) {
                vis[i][j] = 1;
                q.push({i, j});
            }
    while (!q.empty()) {
        auto [x, y] = q.front(); q.pop();
        for (int d = 0; d < 4; d++) {
            int nx = x + dx[d], ny = y + dy[d];
            if (nx >= 1 && nx <= n && ny >= 1 && ny <= n
                && !vis[nx][ny] && a[nx][ny] == 0) {
                vis[nx][ny] = 1;
                q.push({nx, ny});
            }
        }
    }
    // 输出：被围住的 0 变为 2
    // a[i][j]==1 输出 1；vis[i][j]==1 输出 0；否则输出 2
}
```

**核心启示**：
- **"正难则反"策略**：直接找被围住的区域很难，但找"不被围住的"（连通边界的）很容易
- **边界 BFS 是一类经典技巧**：从边界出发标记"自由区域"，取反即为"封闭区域"
- **多源 BFS 的灵活应用**：边界上的多个 `0` 同时作为起点

### 题目 8：单词接龙（LeetCode 127）

> 给定 `beginWord`、`endWord` 和一个单词列表，每次只能改变一个字母，求从 `beginWord` 到 `endWord` 的最短转换序列长度。每个中间词必须在列表中。

**分析**：每个单词是一个状态，能变一个字母互相到达的单词之间有边。对状态图做 BFS 求最短路。枚举当前单词每个位置替换 26 个字母来寻找邻居。

```
状态定义：当前单词字符串
转移/扩展逻辑：改变一个字母得到的新单词，且在字典中
边界条件：beginWord 为起点，字典中无 endWord 则直接返回 0
答案位置：到达 endWord 时的步数
```

**C++ 代码**：

```cpp
int ladderLength(string beginWord, string endWord, vector<string>& wordList) {
    unordered_set<string> dict(wordList.begin(), wordList.end());
    if (!dict.count(endWord)) return 0;
    queue<string> q;
    q.push(beginWord);
    dict.erase(beginWord);
    int step = 1;
    while (!q.empty()) {
        int sz = q.size();
        step++;
        while (sz--) {
            string cur = q.front(); q.pop();
            for (int i = 0; i < (int)cur.size(); i++) {
                char orig = cur[i];
                for (char c = 'a'; c <= 'z'; c++) {
                    cur[i] = c;
                    if (cur == endWord) return step;
                    if (dict.count(cur)) {
                        dict.erase(cur);
                        q.push(cur);
                    }
                }
                cur[i] = orig;
            }
        }
    }
    return 0;
}
```

**核心启示**：
- **隐式图 BFS**：图不需要显式建出来，邻居通过"枚举变换"实时生成
- **从字典中删除代替 visited**：访问过的单词直接从 `dict` 中移除，一举两得
- **枚举 26 字母 vs 枚举字典**：当字典很大时，枚举 `26 × len` 种变换比遍历字典更高效

```
状态定义：4 位数字字符串
转移/扩展逻辑：每个拨轮 +1 或 -1（mod 10）
边界条件："0000" 为起点；死锁状态不可进入
```

```cpp
int openLock(vector<string>& deadends, string target) {
    unordered_set<string> dead(deadends.begin(), deadends.end());
    if (dead.count("0000")) return -1;
    unordered_set<string> vis;
    queue<string> q;
    q.push("0000");
    vis.insert("0000");
    int step = 0;
    while (!q.empty()) {
        int sz = q.size();
        while (sz--) {
            string cur = q.front(); q.pop();
            if (cur == target) return step;
            for (int i = 0; i < 4; i++)
                for (int d : {-1, 1}) {
                    string nxt = cur;
                    nxt[i] = (cur[i] - '0' + d + 10) % 10 + '0';
                    if (!vis.count(nxt) && !dead.count(nxt)) {
                        vis.insert(nxt);
                        q.push(nxt);
                    }
                }
        }
        step++;
    }
    return -1;
}
```

- **状态空间分析**：4 位数字共 10^4 = 10000 种状态，BFS 完全可行
- **死锁 = 障碍**：和网格 BFS 中的墙壁本质一样，都是不可达状态
- **环形取模 `(x + d + 10) % 10`**：处理 0↔9 循环的标准技巧

### 题目 10：二进制矩阵中的最短路径（LeetCode 1091）

> 在 `n × n` 的二进制网格中，找到从左上角 `(0,0)` 到右下角 `(n-1,n-1)` 的最短畅通路径长度。可以沿 8 个方向移动，路径上所有格子值为 `0`。

**分析**：标准网格 BFS，只是方向从 4 扩展到 8。答案是路径上格子数（包含起终点），即步数 + 1。

```
状态定义：(x, y) 为格子坐标
转移/扩展逻辑：8 方向移动，目标格子值为 0
边界条件：起点或终点为 1 则无解

```

```cpp
int dx[] = {-1,-1,-1,0,0,1,1,1}, dy[] = {-1,0,1,-1,1,-1,0,1};

int shortestPathBinaryMatrix(vector<vector<int>>& grid) {
    int n = grid.size();
    if (grid[0][0] || grid[n-1][n-1]) return -1;
    queue<pair<int,int>> q;

                    q.push({nx, ny});
                }
            }
        }
        step++;
    }
    return -1;
}
```

### 题目 11：完全平方数（LeetCode 279）

> 给定正整数 `n`，找到最少数量的完全平方数（如 1, 4, 9, 16, ...）使得它们的和为 `n`。

**分析**：将 0 到 n 视为数轴上的节点，从 0 出发，每步可以跳到 `cur + i²` 的位置。BFS 找到 n 时的层数即为最少平方数个数。

```
状态定义：当前数值 cur（0 到 n）
转移/扩展逻辑：cur → cur + i² （i = 1, 2, ...，且 cur + i² ≤ n）
边界条件：起点为 0，dist[0] = 0
答案位置：dist[n]
```

**C++ 代码**：

```cpp
int numSquares(int n) {
    vector<int> dist(n + 1, -1);
    queue<int> q;
    dist[0] = 0;
    q.push(0);
    while (!q.empty()) {
        int cur = q.front(); q.pop();
        for (int i = 1; cur + i * i <= n; i++) {
            int nxt = cur + i * i;
            if (dist[nxt] == -1) {
                dist[nxt] = dist[cur] + 1;
                if (nxt == n) return dist[nxt];
                q.push(nxt);
            }
        }
    }
    return dist[n];
}
```

**核心启示**：
- **数学问题也能转化为图论**：数轴上的跳跃就是图上的边，BFS 求最短步数
- **隐式图的灵活性**：状态不一定是坐标或字符串，任何"当前值"都可以作为节点
- **提前终止**：到达 n 立刻返回，无需跑完整个 BFS

### 题目 12：滑动谜题（LeetCode 773）

> 在一个 `2 × 3` 的面板上，有数字 `1-5` 和一个空格 `0`。每次可以将 `0` 与相邻数字交换。求从给定状态到目标状态 `[[1,2,3],[4,5,0]]` 的最少移动次数。

**分析**：将 2×3 面板展平为字符串作为状态，预计算每个位置的相邻位置表。用 BFS 搜索从初始状态到目标字符串 `"123450"` 的最短路径。

```
状态定义：6 位字符串（面板展平）
转移/扩展逻辑：找到 '0' 的位置，与相邻位置交换
边界条件：初始面板 → 字符串；目标为 "123450"
答案位置：到达目标字符串的步数
```

**C++ 代码**：

```cpp
int slidingPuzzle(vector<vector<int>>& board) {
    string start, target = "123450";
    for (auto& row : board)
        for (int x : row) start += char(x + '0');
    if (start == target) return 0;
    // 位置 0-5 的邻接表（2×3 网格展平）
    vector<vector<int>> adj = {{1,3},{0,2,4},{1,5},{0,4},{1,3,5},{2,4}};
    unordered_set<string> vis;
    queue<string> q;
    q.push(start);
    vis.insert(start);
    int step = 0;
    while (!q.empty()) {
        step++;
        int sz = q.size();
        while (sz--) {
            string cur = q.front(); q.pop();
            int p = cur.find('0');
            for (int np : adj[p]) {
                string nxt = cur;
                swap(nxt[p], nxt[np]);
                if (nxt == target) return step;
                if (!vis.count(nxt)) { vis.insert(nxt); q.push(nxt); }
            }
        }
    }
    return -1;
}
```

**核心启示**：
- **状态压缩为字符串**：将二维面板序列化，用哈希集合判重，是拼图类问题的通用手法
- **预计算邻接表**：小规模网格的邻接关系可以硬编码，避免每次重新计算
- **状态图规模**：6 个位置的排列数为 6! = 720，BFS 轻松搜完

### 题目 13：公交路线（LeetCode 815）

> 给定若干公交路线 `routes[i]`（每条路线经过若干站点），求从站点 `source` 到站点 `target` 的最少乘坐公交线路数。

**分析**：关键在于**建图方式**：以"公交线路"为节点而非"站点"。先建立站点→线路的映射，然后 BFS 在线路之间跳转。两条线路若共享至少一个站点就相邻。

```
状态定义：当前所在的公交线路编号
转移/扩展逻辑：当前线路的所有站点 → 经过该站的其他线路
边界条件：source 所在的所有线路为起始层；到达包含 target 的线路即终止
答案位置：BFS 层数即为最少乘坐线路数
```

**C++ 代码**：

```cpp
int numBusesToDestination(vector<vector<int>>& routes, int source, int target) {
    if (source == target) return 0;
    int n = routes.size();
    unordered_map<int, vector<int>> stop2routes;
    for (int i = 0; i < n; i++)
        for (int s : routes[i]) stop2routes[s].push_back(i);
    vector<bool> visRoute(n, false);
    unordered_set<int> visStop;
    queue<int> q;
    for (int r : stop2routes[source]) { q.push(r); visRoute[r] = true; }
    int step = 0;
    while (!q.empty()) {
        step++;
        int sz = q.size();
        while (sz--) {
            int r = q.front(); q.pop();
            for (int s : routes[r]) {
                if (s == target) return step;
                if (visStop.count(s)) continue;
                visStop.insert(s);
                for (int nr : stop2routes[s])
                    if (!visRoute[nr]) { visRoute[nr] = true; q.push(nr); }
            }
        }
    }
    return -1;
}
```

**核心启示**：
- **BFS 的节点不一定是"点"**：以线路为节点而非站点，大幅减少搜索空间
- **双层 visited**：同时维护 `visRoute`（线路）和 `visStop`（站点）避免重复遍历
- **图建模能力**是 BFS 进阶的分水岭——同一道题，不同的建模方式复杂度差距巨大

```cpp
            int nx = x + dx[d], ny = y + dy[d];
            if (nx >= 0 && nx < n && ny >= 0 && ny < m
                && grid[nx][ny] != '#' && dist[nx][ny] == -1) {
                dist[nx][ny] = dist[x][y] + 1;
                from_dir[nx][ny] = d;
                q.push({nx, ny});
            }
        }
    }
    if (dist[ex][ey] == -1) return "";
    // 路径还原：从终点回溯到起点
    string path;
    int cx = ex, cy = ey;
    while (cx != sx || cy != sy) {
        int d = from_dir[cx][cy];
        path += dc[d];
        cx -= dx[d];
        cy -= dy[d];
    }
    reverse(path.begin(), path.end());
    return path;
}
```

**核心启示**：
- **路径还原 = 记录来源 + 反向回溯**：BFS 中额外记录 `from_dir`（或 `parent`），终点沿方向回溯到起点
- **反转路径**：回溯从终点到起点，所以需要 `reverse`
- **BFS 路径还原是竞赛高频考点**：不仅要求最短距离，还要输出具体方案

### 题目 16：棋盘（洛谷 P3956） 🟠 Level 3

> 给定一个 `m × m` 的棋盘，其中 `n` 个格子被涂上了颜色（红色或黄色），其余格子无色。从 `(1,1)` 走到 `(m,m)`，每次可向上下左右相邻格子移动。移动到**同色**格子花费 **0** 金币，移动到**异色**格子花费 **1** 金币。还可以花费 **2** 金币对相邻**无色**格子施魔法，将其临时变为当前格子的颜色并走过去，但**魔法不能连续使用**（离开后恢复无色）。求最小花费，无解输出 `-1`。

**分析**：由于移动代价不等（0、1、2），普通 BFS 无法保证最优性，需要用**优先队列 BFS（Dijkstra 思想）**按代价从小到大扩展。关键在于额外维护一个"上一步是否使用了魔法"的状态维度，防止连续施法。每次扩展时分两种情况讨论：目标格有颜色（正常移动）或目标格无色且可施法。

```
状态定义：(x, y, usedMagic) —— 位置 + 上一步是否用了魔法
转移/扩展逻辑：① 邻格有色 → 同色代价 0、异色代价 1；② 邻格无色且上一步未用魔法 → 代价 2 施法
边界条件：不越界 && (1,1) 必须有颜色才能出发
答案位置：第一次从优先队列弹出 (m, m) 时的代价
```

**C++ 代码**：

```cpp
struct Node {
    int x, y, cost, color, magic;
    bool operator>(const Node& o) const { return cost > o.cost; }
};

int dx[] = {0, 0, 1, -1}, dy[] = {1, -1, 0, 0};
int color[105][105];   // -1=无色, 0=红, 1=黄
int dist[105][105][2]; // dist[x][y][是否刚用过魔法]
int m;

int bfs() {
    memset(dist, 0x3f, sizeof(dist));
    priority_queue<Node, vector<Node>, greater<Node>> pq;
    dist[1][1][0] = 0;
    pq.push({1, 1, 0, color[1][1], 0});
    while (!pq.empty()) {
        auto cur = pq.top(); pq.pop();
        if (cur.x == m && cur.y == m) return cur.cost;
        if (cur.cost > dist[cur.x][cur.y][cur.magic]) continue;
        for (int d = 0; d < 4; d++) {
            int nx = cur.x + dx[d], ny = cur.y + dy[d];
            if (nx < 1 || ny < 1 || nx > m || ny > m) continue;
            if (color[nx][ny] != -1) {  // 目标格有颜色
                int nc = cur.cost + (cur.color == color[nx][ny] ? 0 : 1);
                if (nc < dist[nx][ny][0]) {
                    dist[nx][ny][0] = nc;
                    pq.push({nx, ny, nc, color[nx][ny], 0});
                }
            } else if (!cur.magic) {  // 目标格无色 && 上一步没用魔法
                int nc = cur.cost + 2;
                if (nc < dist[nx][ny][1]) {
                    dist[nx][ny][1] = nc;
                    pq.push({nx, ny, nc, cur.color, 1});
                }
            }
        }
    }
    return -1;
}
```

**核心启示**：
- **代价不等 → 优先队列 BFS**：当边权有 0、1、2 等多种取值时，普通队列无法保证出队顺序即最优顺序，必须用小根堆
- **"魔法不可连续" → 状态多一维**：布尔维度 `usedMagic` 防止连续施法，是"约束条件转状态维度"的典型手法
- **弹出即最优 + 跳过劣解**：Dijkstra 式的 `if (cur.cost > dist[...]) continue` 剪枝不可或缺

### 题目 17：字串变换（洛谷 P1032） 🟠 Level 3

> 已知初始字符串 `A` 和目标字符串 `B`，以及若干条变换规则（形如 `a → b`，表示可以在任意位置将子串 `a` 替换为 `b`）。每次可以选择一条规则、在当前串的某个匹配位置执行一次替换。求将 `A` 变为 `B` 的**最少变换步数**；若 **10 步以内**无法完成，输出 `NO ANSWER!`。

**分析**：将每个字符串视为一个"状态节点"，每次替换操作是一条"边"，问题转化为从状态 A 到状态 B 的**最短路径**。BFS 逐层扩展：对当前字符串的每条规则，枚举所有可替换位置生成新状态，用哈希集合去重。10 步限制天然控制了搜索深度。

```
状态定义：当前字符串 s
转移/扩展逻辑：对每条规则，枚举 s 中所有匹配位置，替换后生成新字符串
边界条件：步数 > 10 则剪枝；字符串不在 visited 中才入队
答案位置：第一次出队的目标字符串 B 对应的步数
```

**C++ 代码**：

```cpp
struct State { string s; int step; };

int bfs(string A, string B, vector<pair<string,string>>& rules) {
    queue<State> q;
    unordered_set<string> vis;
    q.push({A, 0});
    vis.insert(A);
    while (!q.empty()) {
        auto [s, step] = q.front(); q.pop();
        if (s == B) return step;
        if (step >= 10) continue;
        for (auto& [from, to] : rules) {
            size_t pos = 0;
            while ((pos = s.find(from, pos)) != string::npos) {
                string ns = s.substr(0, pos) + to
                          + s.substr(pos + from.size());
                if (!vis.count(ns)) {
                    vis.insert(ns);
                    q.push({ns, step + 1});
                }
                pos++;  // 逐位后移，避免遗漏重叠匹配
            }
        }
    }
    return -1;  // NO ANSWER!
}
```

**核心启示**：
- **字符串即状态、替换即转移**：这是"隐式图 BFS"的又一典型——状态不是坐标或数字，而是整个字符串
- **枚举所有匹配位置**：`s.find(from, pos)` 循环查找，`pos++` 避免跳过重叠匹配（如 `aaa` 中找 `aa`）
- **10 步剪枝 + 哈希去重**：没有步数限制时状态空间可能爆炸，题目给定的 10 步上界是天然剪枝条件

### 题目 18：机器人搬重物（洛谷 P1126） 🔴 Level 4

> 在 `n × m` 的网格仓库中，格子值为 `1` 表示障碍，`0` 表示空地。一个**有体积的圆形机器人**在**格点（交叉点）**上移动，它在格点 `(x, y)` 时会占据周围 4 个格子 `(x,y)、(x,y+1)、(x+1,y)、(x+1,y+1)`。机器人每秒可执行一个操作：向当前朝向**前进 1/2/3 步**，或**左转 90°**，或**右转 90°**。给定起点、终点及初始朝向（N/S/E/W），求到达终点的**最少秒数**，无解输出 `-1`。

**分析**：位置 + 方向构成三维状态 `(x, y, dir)`。BFS 每步扩展 5 种操作（前进 1/2/3 步、左转、右转），每种操作耗时 1 秒，属于等权 BFS。核心难点：① 机器人有体积，移动前必须检查目标格点周围 4 个格子无障碍；② 前进多步时**中途每个格点也必须合法**，遇到障碍不能"跳过"。

```
状态定义：(x, y, dir) —— 格点坐标 + 当前朝向（0~3 表示北/东/南/西）
转移/扩展逻辑：① 前进 k=1,2,3 步，逐步检查合法性；② 左转 (dir+3)%4；③ 右转 (dir+1)%4
边界条件：格点 (x,y) 合法范围 [1, n-1]×[1, m-1]；周围 4 格无障碍
答案位置：第一次到达终点格点时的步数（方向不限）
```

**C++ 代码**：

```cpp
int n, m;
int grid[55][55], vis[55][55][4];
int dx[] = {-1, 0, 1, 0};  // 北 东 南 西
int dy[] = {0, 1, 0, -1};

struct Node { int x, y, d, step; };

// 检查机器人能否站在格点 (x, y)
bool canStand(int x, int y) {
    if (x < 1 || y < 1 || x >= n || y >= m) return false;
    return !grid[x][y] && !grid[x][y+1]
        && !grid[x+1][y] && !grid[x+1][y+1];
}

int bfs(int sx, int sy, int sd, int ex, int ey) {
    memset(vis, 0, sizeof(vis));
    queue<Node> q;
    vis[sx][sy][sd] = 1;
    q.push({sx, sy, sd, 0});
    while (!q.empty()) {
        auto [x, y, d, step] = q.front(); q.pop();
        if (x == ex && y == ey) return step;
        // 前进 1~3 步（中途不可穿越障碍）
        for (int k = 1; k <= 3; k++) {
            int nx = x + dx[d] * k, ny = y + dy[d] * k;
            if (!canStand(nx, ny)) break;  // 遇障碍，更远也不可达
            if (!vis[nx][ny][d]) {
                vis[nx][ny][d] = 1;
                q.push({nx, ny, d, step + 1});
            }
        }
        // 左转 90°
        int ld = (d + 3) % 4;
        if (!vis[x][y][ld]) {
            vis[x][y][ld] = 1;
            q.push({x, y, ld, step + 1});
        }
        // 右转 90°
        int rd = (d + 1) % 4;
        if (!vis[x][y][rd]) {
            vis[x][y][rd] = 1;
            q.push({x, y, rd, step + 1});
        }
    }
    return -1;
}
```

**核心启示**：
- **方向作为状态维度**：`vis[x][y][dir]` 三维判重，同一位置不同朝向是完全不同的状态
- **体积约束 → 合法性检查**：机器人不是质点，移动时必须验证占据区域内所有格子均无障碍
- **多步前进的连贯性**：前进 k 步时，中间每一步都必须合法，遇到障碍立即 `break`，不能跳过

## 五、常见技巧与思路总结

### 5.1 BFS 的三种写法

| 写法 | 数据结构 | 适用场景 |
|------|---------|---------|
| 标准 BFS | `queue` | 边权全为 1 的最短路 |
| 01-BFS | `deque` | 边权为 0 或 1 的最短路 |
| 优先队列 BFS | `priority_queue` | 即 Dijkstra，任意非负边权 |

```cpp
// 标准 BFS
queue<int> q;

// 01-BFS：权 0 加队首，权 1 加队尾
deque<int> dq;
if (w == 0) dq.push_front(v);
else dq.push_back(v);

// 优先队列 BFS（Dijkstra）
priority_queue<pair<int,int>, vector<pair<int,int>>, greater<>> pq;
```

### 5.2 多源 BFS 技巧（超级源点）

多源 BFS 的本质是在所有源点之外虚拟一个**超级源点 S'**，S' 到每个实际源点的边权为 0。这样多源问题就转化为从 S' 出发的单源 BFS。

实际代码中不需要真的建超级源点，只需在 BFS 开始前将所有源点入队、距离设为 0：

```cpp
for (auto& src : sources) {
    dist[src] = 0;
    q.push(src);
}
```

### 5.3 状态编码与哈希

| 编码方式 | 适用场景 | 示例 |
|---------|---------|------|
| 字符串 | 棋盘、拼图 | `"123450"` |
| 整数压缩 | 位状态、小范围 | `mask = (1 << n) - 1` |
| `pair/tuple` | 二维/三维坐标 | `{x, y}` |
| 自定义哈希 | 复合状态 | 结构体 + 重载 |

当状态空间较小时（≤ 10^6），用 `unordered_set<string>` 或 `unordered_map` 判重；更大时考虑位压缩或 Zobrist Hash。

### 5.4 01-BFS 与 Dijkstra 的关系

```
边权全为 1  →  标准 BFS（queue），O(V+E)
边权 0 或 1 →  01-BFS（deque），O(V+E)
边权非负   →  Dijkstra（priority_queue），O((V+E)logV)
```

01-BFS 是 Dijkstra 的特化版：当边权只有 0/1 时，双端队列代替优先队列，时间从 O((V+E)logV) 优化到 O(V+E)。关键原理：权 0 的边不增加距离，应优先处理（加入队首）。

### 5.5 路径还原技巧

三种常见的路径还原方式：

```cpp
// 方法 1：记录父节点
int parent[MAXN];
// BFS 中：parent[nxt] = cur;
// 还原：从终点沿 parent 回溯到起点，reverse

// 方法 2：记录来的方向
int from_dir[MAXN][MAXN];
// BFS 中：from_dir[nx][ny] = d;
// 还原：从终点反向移动 dx/dy，还原方向字符

// 方法 3：记录完整路径（空间大，仅小规模适用）
queue<pair<State, vector<State>>> q;  // (当前状态, 路径)
```

## 七、终极总结与归纳

### 7.1 BFS 的本质

**BFS 的本质是"逐层扩展的最短路算法"**。

它利用队列的 FIFO 特性，保证了搜索的层次性——同一层的节点距起点距离相同，越早入队的节点距离越近。这一性质使 BFS 成为求解等权/无权图最短路问题的最优选择。一切 BFS 问题的核心都在于三个要素：**明确状态、定义转移、控制边界**。

### 7.2 核心要点

1. **入队即标记**：在 `push` 时而非 `pop` 时标记 visited，避免重复入队导致 TLE
2. **`sz = q.size()` 分层**：这是层次遍历、计算层数/步数的关键技巧
3. **多源 BFS = 超级源点**：所有起点同时入队，距离为 0，一次 BFS 解决
4. **状态即节点**：BFS 不限于坐标和图节点，任何可编码的状态都能做 BFS
5. **01-BFS 用 deque**：边权 0/1 时，O(V+E) 替代 Dijkstra 的 O((V+E)logV)
6. **路径还原记 parent**：需要输出方案时，额外记录来源信息，终点回溯即可
