## 一、什么是记忆化搜索

记忆化搜索（Memoization Search）是一种将 **DFS（深度优先搜索）** 与 **缓存（备忘录）** 相结合的技术。它的核心思想非常朴素：在递归过程中，如果某个子问题已经计算过，就直接返回缓存中的结果，不再重复计算。

你可以把它理解为"带记忆的递归"。假设你在解一道数学题，过程中需要反复计算 `f(3)` 的值。第一次你认真算出来了，之后每次遇到 `f(3)` 时你就直接看笔记本里记下的答案——这就是记忆化搜索的本质。

从实现上看，记忆化搜索是 **自顶向下** 的动态规划：从大问题出发，递归拆分为子问题，遇到已计算过的子问题就直接返回。与之对应，递推 DP 是 **自底向上** 的，从最小子问题开始逐步推导出大问题的答案。两者殊途同归，解决的是同一类具有 **最优子结构** 和 **重叠子问题** 的问题。

### 递归树中重复子问题被缓存的过程

以计算 `fib(5)` 为例：

```
暴力递归（大量重复）：              记忆化搜索（缓存后剪枝）：

        fib(5)                            fib(5)
       /      \                          /      \
    fib(4)   fib(3)                   fib(4)   fib(3) ✅ 命中缓存
    /    \    /    \                   /    \
 fib(3) fib(2) fib(2) fib(1)      fib(3) fib(2) ✅
  / \    / \                       /    \
f(2) f(1) f(1) f(0)            fib(2) fib(1) ✅
/ \                              / \
f(1) f(0)                    f(1) f(0)

调用次数：15 次                    调用次数：9 次 → O(n)
```

### 记忆化搜索 vs 递推 DP 对比

| 对比维度 | 记忆化搜索（自顶向下） | 递推 DP（自底向上） |
|---------|---------------------|-------------------|
| 实现方式 | 递归 + memo 数组 | 循环填表 |
| 思维方向 | 从大问题拆到小问题 | 从小问题推到大问题 |
| 状态遍历 | 只计算实际需要的状态 | 遍历所有状态 |
| 代码直觉 | 更接近数学递推式 | 需要考虑填表顺序 |
| 栈空间 | 递归深度受限 | 无递归栈开销 |
| 空间优化 | 较难滚动优化 | 容易滚动数组优化 |
| 适用场景 | 状态空间稀疏、转移复杂 | 状态空间稠密、转移规律 |

## 二、核心思想与前置知识

### 2.1 递归与重叠子问题

记忆化搜索能发挥作用的前提是 **重叠子问题（Overlapping Subproblems）**：在递归求解过程中，相同参数的子问题被反复调用。

判断是否存在重叠子问题的简单方法：画出递归树，观察是否有重复节点。如果有，就可以用记忆化搜索优化。

### 2.2 备忘录（memo）数组的设计

备忘录是记忆化搜索的核心数据结构，它记录每个状态的计算结果。设计 memo 需要考虑：

- **维度**：与 DFS 函数的参数一一对应。`dfs(i)` → 一维 `memo[i]`；`dfs(i, j)` → 二维 `memo[i][j]`
- **初始值**：用一个"不可能出现的值"标记未计算状态，最常用 `-1`
- **类型**：状态空间小用数组，状态空间大或稀疏用哈希表（`unordered_map`）

```
常见 memo 初始化方式：
┌──────────────┬──────────────────────┬──────────────────────┐
│ 初始化方式    │ 适用场景              │ 示例                  │
├──────────────┼──────────────────────┼──────────────────────┤
│ memset(,-1)  │ 答案 ≥ 0 的整数问题   │ 方案数、最优值          │
│ memset(,0x3f)│ 求最小值问题          │ 最短路径、最少操作       │
│ bool vis[]   │ 答案可能为 0 或 -1    │ 博弈论、存在性判断       │
│ unordered_map│ 状态空间稀疏或参数大   │ 大范围数位DP            │
└──────────────┴──────────────────────┴──────────────────────┘
```

### 2.3 记忆化搜索通用框架代码（骨架）

```cpp
const int MAXN = 1005;
int memo[MAXN];  // 备忘录，维度与状态参数对应

// 初始化：memset(memo, -1, sizeof(memo));

int dfs(int state) {
    // 1. 边界条件
    if (state == 终止条件) return 边界值;
    // 2. 查缓存
    if (memo[state] != -1) return memo[state];
    // 3. 递归转移
    int res = 初始值;
    for (auto& next : 所有合法转移) {
        res = 最优化(res, dfs(next) + 转移代价);
    }
    // 4. 存缓存并返回
    return memo[state] = res;
}
```

### 2.4 从暴力 DFS 到记忆化搜索的转化三步法

**第一步：写出暴力 DFS**
- 明确函数参数（即"状态"）
- 写出边界条件和递归转移

**第二步：加备忘录**
- 开一个与参数同维度的数组
- 函数入口处检查是否已计算
- 返回前将结果存入备忘录

**第三步：验证正确性**
- 确认每个状态只被计算一次
- 确认边界条件覆盖所有 base case
- 确认 memo 初始值不与合法答案冲突

### 2.5 时间/空间复杂度分析

记忆化搜索的复杂度分析公式：

```
时间复杂度 = 状态总数 × 每个状态的转移代价
空间复杂度 = 状态总数（memo 数组大小） + 递归栈深度
```

**示例**：
- 一维状态 `dfs(i)`，`i ∈ [0, n]`，每个状态 O(1) 转移 → 时间 O(n)，空间 O(n)
- 二维状态 `dfs(i, j)`，`i,j ∈ [0, n]`，每个状态 O(n) 转移 → 时间 O(n³)，空间 O(n²)
- 状态压缩 `dfs(row, mask)`，`mask ∈ [0, 2ⁿ)`，每个状态 O(2ⁿ) 转移 → 时间 O(m·4ⁿ)，空间 O(m·2ⁿ)

## 三、记忆化搜索的子模型与分类详解

### 3.1 一维状态记忆化（斐波那契、爬楼梯）

**简要说明**：状态只有一个参数，通常是当前位置或剩余容量。

```
状态定义：dfs(i) 表示从位置 i 开始/到位置 i 为止的最优值或方案数
转移关键：dfs(i) 由 dfs(i-1)、dfs(i-2) 等少数前驱决定
适用信号：问题可以"一步步走"，每步只影响一个维度
实现要点：memo 为一维数组，memset(memo, -1, sizeof(memo))
```

```cpp
int memo[MAXN];
int dfs(int i) {
    if (i <= 1) return i;
    if (memo[i] != -1) return memo[i];
    return memo[i] = dfs(i - 1) + dfs(i - 2);
}
```

### 3.2 二维状态记忆化（网格路径、背包）

**简要说明**：状态由两个参数决定，如坐标 `(i, j)` 或物品编号和容量 `(i, w)`。

```
状态定义：dfs(i, j) 表示在 (i,j) 位置或前 i 个物品容量 j 下的最优值
转移关键：由相邻格子或选/不选当前物品转移而来
适用信号：网格类问题、背包类问题、双参数约束
实现要点：memo 为二维数组，注意两个维度的范围
```

```cpp
int memo[MAXN][MAXN];
int dfs(int i, int j) {
    if (i == 0 && j == 0) return 1;
    if (i < 0 || j < 0) return 0;
    if (memo[i][j] != -1) return memo[i][j];
    return memo[i][j] = dfs(i - 1, j) + dfs(i, j - 1);
}
```

### 3.3 区间型记忆化（石子合并、矩阵链乘）

**简要说明**：状态为区间 `[l, r]`，在区间内枚举分割点进行转移。

```
状态定义：dfs(l, r) 表示处理区间 [l, r] 的最优值
转移关键：枚举分割点 k ∈ [l, r-1]，合并左右两部分
适用信号：合并操作、区间划分、回文判断
实现要点：区间长度为 1 是 base case，注意环形问题需要断环为链
```

```cpp
int memo[MAXN][MAXN];
int pre[MAXN];  // 前缀和

int dfs(int l, int r) {
    if (l == r) return 0;
    if (memo[l][r] != -1) return memo[l][r];
    int res = 0x3f3f3f3f;
    for (int k = l; k < r; k++)
        res = min(res, dfs(l, k) + dfs(k + 1, r) + pre[r] - pre[l - 1]);
    return memo[l][r] = res;
}
```

### 3.4 树形记忆化（树上 DP）

**简要说明**：在树结构上进行 DFS，每个节点的状态依赖其子树的结果。

```
状态定义：dfs(u, k) 表示以 u 为根的子树中，选 k 个节点的最优值
转移关键：将子树结果通过"树形背包"合并到父节点
适用信号：树结构 + 选择/分配问题
实现要点：先递归子节点，再合并到父节点；虚拟根处理森林
```

```cpp
int dp[MAXN][MAXN];
int val[MAXN];
vector<int> g[MAXN];

void dfs(int u) {
    dp[u][1] = val[u];
    for (int v : g[u]) {
        dfs(v);
        for (int i = m + 1; i >= 1; i--)
            for (int j = 1; j < i; j++)
                dp[u][i] = max(dp[u][i], dp[u][i - j] + dp[v][j]);
    }
}
```

### 3.5 状态压缩记忆化（bitmask + memo）

**简要说明**：用二进制位表示集合或行状态，将组合状态压缩为一个整数。

```
状态定义：dfs(row, mask) 表示处理到第 row 行，上一行状态为 mask 时的最优值
转移关键：枚举当前行的所有合法 mask，检查与上一行的兼容性
适用信号：行数少（≤20）、每行有选/不选、相邻约束
实现要点：预处理合法状态列表，用位运算判断兼容性
```

```cpp
int memo[MAXN][1 << MAXM];
int valid[MAXN]; // valid[i] 表示第 i 行可用位置的 bitmask

int dfs(int row, int prev) {
    if (row == m) return 0;
    if (memo[row][prev] != -1) return memo[row][prev];
    int res = 0;
    for (int cur = valid[row]; cur >= 0; /* 枚举子集 */) {
        if ((cur & (cur << 1)) == 0           // 当前行无相邻
            && (cur & (prev << 1)) == 0       // 左上无冲突
            && (cur & (prev >> 1)) == 0) {    // 右上无冲突
            res = max(res, __builtin_popcount(cur) + dfs(row + 1, cur));
        }
        if (cur == 0) break;
        cur = (cur - 1) & valid[row];
    }
    return memo[row][prev] = res;
}
```

### 模型对比总表

| 模型 | 状态维度 | 典型状态 | 转移方式 | 时间复杂度 | 代表题目 |
|------|---------|---------|---------|-----------|---------|
| 一维状态 | 1D | `dfs(i)` | 前驱递推 | O(n) | 斐波那契、爬楼梯 |
| 二维状态 | 2D | `dfs(i,j)` | 格子/背包 | O(n²)~O(n·W) | 网格路径、01背包 |
| 区间型 | 2D | `dfs(l,r)` | 枚举分割点 | O(n³) | 石子合并、戳气球 |
| 树形 | 树+背包 | `dp[u][k]` | 子树合并 | O(n·m²) | 选课、树上背包 |
| 状态压缩 | 行+bitmask | `dfs(row,mask)` | 枚举合法子集 | O(m·4ⁿ) | 最大学生数、棋盘覆盖 |

### 题目 1：斐波那契数（LeetCode 509）

> 给定 `n`，求第 `n` 个斐波那契数。`F(0)=0, F(1)=1, F(n)=F(n-1)+F(n-2)`。

**分析**：最经典的记忆化入门题。暴力递归是指数级，加上 memo 数组后每个状态只算一次，变为 O(n)。

```
状态定义：dfs(i) = 第 i 个斐波那契数
转移方程：dfs(i) = dfs(i-1) + dfs(i-2)
边界条件：dfs(0) = 0, dfs(1) = 1
答案位置：dfs(n)
```

**C++ 代码**：
```cpp
int memo[31];

int dfs(int n) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];
    return memo[n] = dfs(n - 1) + dfs(n - 2);
}
// memset(memo, -1, sizeof(memo)); dfs(n)
```

**核心启示**：
- **记忆化模板**：这是最小完整的记忆化搜索骨架——边界、查缓存、递归、存缓存
- **从指数到线性**：一个 memo 数组将 O(2ⁿ) 降到 O(n)，这就是记忆化的威力

### 题目 2：爬楼梯（LeetCode 70）

> 每次可以爬 1 或 2 个台阶，问爬到第 `n` 阶有多少种方法。

**分析**：本质就是斐波那契变体。`dfs(i)` 表示爬到第 `i` 阶的方案数，可以从第 `i-1` 阶跨 1 步或第 `i-2` 阶跨 2 步到达。

```
状态定义：dfs(i) = 爬到第 i 阶的方案数
转移方程：dfs(i) = dfs(i-1) + dfs(i-2)
边界条件：dfs(0) = 1, dfs(1) = 1
答案位置：dfs(n)
```

**C++ 代码**：
```cpp
int memo[46];

int dfs(int n) {
    if (n <= 1) return 1;
    if (memo[n] != -1) return memo[n];
    return memo[n] = dfs(n - 1) + dfs(n - 2);
}
// memset(memo, -1, sizeof(memo)); dfs(n)
```

**核心启示**：
- **问题转化能力**：看到"有几种走法"就想到"方案数型 DP"
- **一维记忆化**：状态只有一个参数，memo 就是一维数组

### 题目 3：不同路径（LeetCode 62）

> `m×n` 网格，从左上角到右下角，每次只能向下或向右移动一步，问有多少条不同路径。

**分析**：二维状态记忆化的入门题。`dfs(i, j)` 表示从 `(0,0)` 到 `(i,j)` 的路径数，由上方和左方转移而来。

```
状态定义：dfs(i, j) = 从 (0,0) 到 (i,j) 的路径数
转移方程：dfs(i, j) = dfs(i-1, j) + dfs(i, j-1)
边界条件：dfs(0, j) = 1, dfs(i, 0) = 1
答案位置：dfs(m-1, n-1)
```

**C++ 代码**：
```cpp
int memo[101][101];

int dfs(int i, int j) {
    if (i == 0 || j == 0) return 1;
    if (memo[i][j] != -1) return memo[i][j];
    return memo[i][j] = dfs(i - 1, j) + dfs(i, j - 1);
}
// memset(memo, -1, sizeof(memo)); dfs(m-1, n-1)
```

**核心启示**：
- **二维状态 = 二维 memo**：函数参数有几个，memo 就开几维
- **网格路径模型**：向下/向右 → 只需考虑上方和左方，天然无后效性

### 题目 4：打家劫舍（LeetCode 198）

> 一排房屋，每间有一定金额，不能偷相邻两间，求能偷到的最大金额。

**分析**：经典"选与不选"模型。对于第 `i` 间房屋，要么偷（跳过 `i-1`），要么不偷（考虑 `i-1`）。

```
状态定义：dfs(i) = 从第 i 间到最后一间能偷的最大金额
转移方程：dfs(i) = max(nums[i] + dfs(i+2), dfs(i+1))
边界条件：dfs(n) = 0（越界返回 0）
答案位置：dfs(0)
```

**C++ 代码**：
```cpp
int memo[101];
int nums[101], n;

int dfs(int i) {
    if (i >= n) return 0;
    if (memo[i] != -1) return memo[i];
    return memo[i] = max(nums[i] + dfs(i + 2), dfs(i + 1));
}
// memset(memo, -1, sizeof(memo)); dfs(0)
```

**核心启示**：
- **选与不选**：面对每个元素，决策只有"选"和"不选"两种，是最基本的 DP 决策模型
- **状态方向灵活**：可以从前往后定义 `dfs(i)` 为"前 i 间"，也可以从后往前

### 题目 5：目标和（LeetCode 494）

> 给定整数数组 `nums` 和目标值 `target`，每个数前加 `+` 或 `-`，求使结果等于 `target` 的方案数。

**分析**：对每个数做"加"或"减"的决策，DFS 参数为当前下标和当前累计和。由于累计和可能为负数，用哈希 map 做 memo 或对和做偏移。

```
状态定义：dfs(i, sum) = 从第 i 个数开始，当前累计为 sum 时的方案数
转移方程：dfs(i, sum) = dfs(i+1, sum+nums[i]) + dfs(i+1, sum-nums[i])
边界条件：i == n 时，若 sum == target 返回 1，否则返回 0
答案位置：dfs(0, 0)
```

**C++ 代码**：
```cpp
int nums[21], n, target;
map<pair<int,int>, int> memo;

int dfs(int i, int sum) {
    if (i == n) return sum == target ? 1 : 0;
    auto key = make_pair(i, sum);
    if (memo.count(key)) return memo[key];
    return memo[key] = dfs(i + 1, sum + nums[i])
                     + dfs(i + 1, sum - nums[i]);
}
// memo.clear(); dfs(0, 0)
```

**核心启示**：
- **哈希 memo**：当状态参数范围不方便开数组时（如含负数），用 `map` 做备忘录
- **方案计数型**：转移是累加而非取 max/min，注意边界返回 0 或 1

### 题目 6：采药 / 01 背包（洛谷 P1048）

> `N` 种草药，每种有采集时间 `w[i]` 和价值 `v[i]`，总时间为 `T`，每种最多采一次，求最大价值。

**分析**：标准 01 背包的记忆化写法。`dfs(i, j)` 表示考虑前 `i` 种草药、剩余时间为 `j` 时的最大价值。

```
状态定义：dfs(i, j) = 考虑前 i 种草药，容量为 j 时的最大价值
转移方程：dfs(i, j) = max(dfs(i-1, j), dfs(i-1, j-w[i]) + v[i])  (j >= w[i])
边界条件：i == 0 时返回 0
答案位置：dfs(N, T)
```

**C++ 代码**：
```cpp
const int MAXN = 105, MAXW = 1005;
int memo[MAXN][MAXW];
int w[MAXN], v[MAXN], N, T;

int dfs(int i, int j) {
    if (i == 0) return 0;
    if (memo[i][j] != -1) return memo[i][j];
    int res = dfs(i - 1, j); // 不选第 i 种

        res = max(res, dfs(i - 1, j - w[i]) + v[i]); // 选第 i 种

}
// memset(memo, -1, sizeof(memo)); dfs(N, T)
```

- **背包 = 选与不选 + 容量约束**：在打家劫舍的基础上多了一维"容量"

### 题目 7：零钱兑换（LeetCode 322）

> 给定不同面额的硬币 `coins` 和总金额 `amount`，求凑成总金额所需的最少硬币数。

```

const int INF = 0x3f3f3f3f;

    if (memo[j] != -1) return memo[j];

### 题目 8：让字符串成为回文串的最少插入次数（LeetCode 1312）

> 给定字符串 `s`，每次可以在任意位置插入一个字符，求最少插入多少次能使 `s` 变成回文串。

**分析**：经典区间 DP。`dfs(l, r)` 表示将子串 `s[l..r]` 变为回文的最少插入次数。若两端字符相同则向内收缩，否则选择插入较优的一端。

         若 s[l]!=s[r]，dfs(l,r) = min(dfs(l+1,r), dfs(l,r-1)) + 1
边界条件：l >= r 时返回 0
答案位置：dfs(0, n-1)
```

**C++ 代码**：
```cpp
int memo[501][501];
char s[501];
int n;

int dfs(int l, int r) {

    if (memo[l][r] != -1) return memo[l][r];
    if (s[l] == s[r])
        return memo[l][r] = dfs(l + 1, r - 1);
    return memo[l][r] = min(dfs(l + 1, r), dfs(l, r - 1)) + 1;
}
// memset(memo, -1, sizeof(memo)); dfs(0, n-1)
```

**核心启示**：
- **区间 DP 的两端收缩**：区间问题不一定要枚举分割点，两端字符比较也是常见模式

### 题目 9：石子合并（洛谷 P1880）

> 环形排列的 `n` 堆石子，每次合并相邻两堆，代价为两堆石子数之和，求最小/最大总代价。

**分析**：环形区间 DP，断环为链——将数组复制一遍拼接，对长度为 `2n` 的链做区间 DP，最终在所有长度为 `n` 的区间中取最优。

```
状态定义：dfs_min(l, r) = 合并 [l, r] 的最小代价；dfs_max 同理
转移方程：dfs_min(l, r) = min{dfs_min(l,k) + dfs_min(k+1,r) + pre[r]-pre[l-1]}
边界条件：l == r 时返回 0
答案位置：min/max{ dfs(i, i+n-1) } 对 i ∈ [1, n]
```

**C++ 代码**：
```cpp
const int MAXN = 205, INF = 0x3f3f3f3f;
int f[MAXN][MAXN], g[MAXN][MAXN]; // f:最小 g:最大
int pre[MAXN], n;

int dfs_min(int l, int r) {
    if (l == r) return 0;
    if (f[l][r] != -1) return f[l][r];
    int res = INF;
    for (int k = l; k < r; k++)
        res = min(res, dfs_min(l, k) + dfs_min(k + 1, r) + pre[r] - pre[l - 1]);
    return f[l][r] = res;
}

int dfs_max(int l, int r) {
    if (l == r) return 0;
    if (g[l][r] != -1) return g[l][r];
    int res = 0;
    for (int k = l; k < r; k++)
        res = max(res, dfs_max(l, k) + dfs_max(k + 1, r) + pre[r] - pre[l - 1]);
    return g[l][r] = res;
}
// 断环为链：a[i+n] = a[i]，pre 为长度 2n 的前缀和
// 答案：遍历 i∈[1,n]，取 dfs_min(i, i+n-1) 的最小值和 dfs_max(i, i+n-1) 的最大值
```

**核心启示**：
- **断环为链**：环形问题的标准技巧，将长度 n 的环复制成长度 2n 的链
- **区间 DP 三件套**：枚举区间长度 → 枚举左端点 → 枚举分割点，记忆化让你不用操心枚举顺序

### 题目 10：矩阵中的最长递增路径（LeetCode 329）

> 给定整数矩阵，找出最长递增路径的长度。可以向上下左右四个方向移动，但只能移向严格更大的值。

**分析**：网格记忆化搜索的经典题。由于路径严格递增，不存在环，因此不需要 `vis` 标记访问。`dfs(i, j)` 表示从 `(i, j)` 出发的最长路径长度。

```
状态定义：dfs(i, j) = 从 (i,j) 出发的最长递增路径长度
转移方程：dfs(i, j) = 1 + max{dfs(ni, nj)} 对所有相邻且 matrix[ni][nj] > matrix[i][j]
边界条件：无法继续移动时返回 1
答案位置：max{dfs(i, j)} 对所有 (i, j)
```

**C++ 代码**：
```cpp
int memo[201][201];
int matrix[201][201], rows, cols;
int dx[] = {0, 0, 1, -1}, dy[] = {1, -1, 0, 0};

int dfs(int i, int j) {
    if (memo[i][j]) return memo[i][j];
    memo[i][j] = 1;
    for (int d = 0; d < 4; d++) {
        int ni = i + dx[d], nj = j + dy[d];
        if (ni >= 0 && ni < rows && nj >= 0 && nj < cols
            && matrix[ni][nj] > matrix[i][j])
            memo[i][j] = max(memo[i][j], 1 + dfs(ni, nj));
    }
    return memo[i][j];
}
// memset(memo, 0, sizeof(memo));
// ans = max{dfs(i, j)} 对所有 (i, j)
```

**核心启示**：
- **天然无环 = 无需 vis**：严格递增保证不会走回头路，省去访问标记
- **memo 初始化为 0**：路径长度 ≥ 1，所以 0 表示未计算，无需用 -1

### 题目 11：单词拆分（LeetCode 139）

> 给定字符串 `s` 和单词字典 `wordDict`，判断 `s` 是否可以被拆分为字典中的单词。

**分析**：从位置 `i` 开始，尝试匹配每个单词，若匹配成功则递归检查剩余部分。用 memo 记录每个起始位置的结果。

```
状态定义：dfs(i) = s[i..n-1] 是否能被字典拆分
转移方程：dfs(i) = OR{ dfs(i + len(w)) } 对所有 w ∈ dict 且 s[i..] 以 w 开头
边界条件：i == n 时返回 true
答案位置：dfs(0)
```

**C++ 代码**：
```cpp
int memo[301]; // -1:未访问 0:false 1:true
string s;
vector<string> dict;

int dfs(int i) {
    if (i == (int)s.size()) return 1;
    if (memo[i] != -1) return memo[i];
    for (auto& w : dict) {
        int len = w.size();
        if (i + len <= (int)s.size()
            && s.substr(i, len) == w
            && dfs(i + len))
            return memo[i] = 1;
    }
    return memo[i] = 0;
}
// memset(memo, -1, sizeof(memo)); dfs(0)
```

**核心启示**：
- **布尔型记忆化**：memo 存三种状态（未访问 -1 / false 0 / true 1），区分"没算过"和"算过结果为 false"
- **字符串匹配 + 记忆化**：将字符串拆分问题转化为位置上的 DFS

### 题目 12：戳气球（LeetCode 312）

> `n` 个气球排成一排，每个气球有分数 `nums[i]`。戳破气球 `i` 获得 `nums[i-1] * nums[i] * nums[i+1]` 分（越界视为 1），求最大总分。

**分析**：直接枚举戳的顺序是阶乘级别。关键转化：不考虑"先戳哪个"，而是考虑区间 `(l, r)` 内 **最后戳哪个**。设 `dfs(l, r)` 表示戳破开区间 `(l, r)` 内所有气球的最大得分。

```
状态定义：dfs(l, r) = 戳破开区间 (l, r) 内所有气球获得的最大得分
转移方程：dfs(l, r) = max{ dfs(l,k) + dfs(k,r) + nums[l]*nums[k]*nums[r] }
         其中 k ∈ (l, r)，k 是最后被戳破的气球
边界条件：r - l <= 1 时返回 0（区间内无气球）
答案位置：dfs(0, n+1)，其中 nums[0] = nums[n+1] = 1
```

**C++ 代码**：
```cpp
int memo[502][502];
int nums[502]; // nums[0] = nums[n+1] = 1
int n;

int dfs(int l, int r) {
    if (r - l <= 1) return 0;
    if (memo[l][r] != -1) return memo[l][r];
    int res = 0;
    for (int k = l + 1; k < r; k++)
        res = max(res, dfs(l, k) + dfs(k, r) + nums[l] * nums[k] * nums[r]);
    return memo[l][r] = res;
}
// 原数组前后各加一个 1：nums[0]=1, nums[n+1]=1
// memset(memo, -1, sizeof(memo)); dfs(0, n+1)
```

**核心启示**：
- **"最后操作"思维**：当正向枚举顺序导致状态不独立时，反过来想"最后做什么"
- **开区间技巧**：用开区间 `(l, r)` 而非闭区间，边界处理更简洁

### 题目 13：选课 / 树形 DP（洛谷 P2014）

> `N` 门课程构成森林（有先修关系），选 `M` 门课使总学分最大。选某课必须先选其先修课。

**分析**：树形背包经典题。建虚拟根节点 0 将森林连为树，`dp[u][k]` 表示以 `u` 为根的子树中选 `k` 门课的最大学分。对每个子节点进行背包合并。

```
状态定义：dp[u][k] = 以 u 为根的子树中选 k 门课的最大学分
转移方程：dp[u][i] = max(dp[u][i], dp[u][i-j] + dp[v][j])  对子节点 v
边界条件：dp[u][1] = val[u]（选 u 自己）
答案位置：dp[0][M+1]（虚拟根选 M+1 门 = 实际选 M 门）
```

**C++ 代码**：
```cpp
const int MAXN = 305;
int dp[MAXN][MAXN], val[MAXN];
vector<int> g[MAXN];
int N, M;

void dfs(int u) {
    dp[u][1] = val[u];
    for (int v : g[u]) {
        dfs(v);
        for (int i = M + 1; i >= 1; i--)
            for (int j = 1; j < i; j++)
                dp[u][i] = max(dp[u][i], dp[u][i - j] + dp[v][j]);
    }
}
// 建树：g[pre].push_back(i)，无先修课的连到虚拟根 0
// dfs(0); 答案：dp[0][M+1]
```

**核心启示**：
- **树形背包**：在树上做背包合并，是树形 DP 的核心手法
- **虚拟根节点**：处理森林问题的标准技巧，加一个权值为 0 的虚拟根将森林连为一棵树
- **倒序枚举容量**：合并子树时倒序枚举保证每个子节点只被使用一次（类似 01 背包）

### 题目 15：参加考试的最大学生数（LeetCode 1349）

> `m×n` 的教室座位图，`'.'` 可坐人，`'#'` 是坏椅子。学生不能坐在另一位学生的左边、右边、左上、右上（即能看到答案的位置）。求最多能坐多少学生。

**分析**：行数 `m ≤ 8`，列数 `n ≤ 8`，每行状态可用 bitmask 表示。逐行决策，用 `dfs(row, prev)` 表示处理到第 `row` 行、上一行状态为 `prev` 时的最大学生数。需要检查三类约束：当前行内无相邻、不坐坏椅子、与上一行无左上/右上冲突。

```
状态定义：dfs(row, prev) = 从第 row 行开始，上一行状态为 prev 时的最大学生数
转移方程：dfs(row, prev) = max{ popcount(cur) + dfs(row+1, cur) }
         其中 cur 是当前行的合法状态
合法条件：(cur & avail[row]) == cur         // 不坐坏椅子
         (cur & (cur << 1)) == 0            // 无左右相邻
         (cur & (prev << 1)) == 0           // 无左上冲突
         (cur & (prev >> 1)) == 0           // 无右上冲突
边界条件：row == m 时返回 0
答案位置：dfs(0, 0)
```

**C++ 代码**：
```cpp
int memo[9][1 << 9];
int avail[9]; // avail[i]: 第 i 行可用座位的 bitmask
int m, n;

int dfs(int row, int prev) {
    if (row == m) return 0;
    if (memo[row][prev] != -1) return memo[row][prev];
    int res = 0, full = (1 << n) - 1;
    for (int cur = avail[row]; ; cur = (cur - 1) & avail[row]) {
        if ((cur & (cur << 1)) == 0       // 行内无相邻
            && (cur & (prev << 1)) == 0    // 左上无冲突
            && (cur & (prev >> 1)) == 0) { // 右上无冲突
            res = max(res, __builtin_popcount(cur) + dfs(row + 1, cur));
        }
        if (cur == 0) break;
    }
    return memo[row][prev] = res;
}
// 预处理 avail[i]：遍历每行，'.' 位置置 1
// memset(memo, -1, sizeof(memo)); dfs(0, 0)
```

**核心启示**：
- **状态压缩的本质**：用二进制整数表示集合/行状态，把组合问题转化为整数枚举
- **枚举子集技巧**：`for (int s = mask; ; s = (s-1) & mask)` 可枚举 mask 的所有子集（含空集）
- **位运算判断约束**：左移/右移 + 按位与，一行代码搞定相邻性检测
- **逐行决策模型**：状压 DP 的经典框架——逐行处理，上一行状态作为参数传递

### 题目 16：滑雪（洛谷 P1434 [SHOI2002]）

🟠 Level 3 — 网格记忆化搜索经典

> 给定 `R×C` 的矩阵表示各点高度，从任意点出发，每次向上下左右移动一格，且只能从高处滑向低处（严格递减）。求最长滑雪路径经过的格子数。

**分析**：与 LeetCode 329（最长递增路径）几乎同构，是网格记忆化搜索的入门必做题。由于路径严格递减，不可能形成环，因此无需 `vis` 标记。对每个点计算 `dfs(x, y)` 表示从该点出发的最长路径，取所有点的最大值即为答案。

```
状态定义：dfs(x, y) = 从 (x, y) 出发的最长下降路径长度
转移方程：dfs(x, y) = 1 + max{dfs(nx, ny)}  对所有相邻且 h[nx][ny] < h[x][y]
边界条件：四周无法继续下滑时返回 1
答案位置：max{dfs(x, y)} 对所有 (x, y)
```

**C++ 代码**：
```cpp
int h[105][105], dp[105][105];
int R, C;
int dx[] = {0, 0, 1, -1}, dy[] = {1, -1, 0, 0};

int dfs(int x, int y) {
    if (dp[x][y]) return dp[x][y];
    dp[x][y] = 1;
    for (int d = 0; d < 4; d++) {
        int nx = x + dx[d], ny = y + dy[d];
        if (nx >= 1 && nx <= R && ny >= 1 && ny <= C
            && h[nx][ny] < h[x][y])
            dp[x][y] = max(dp[x][y], dfs(nx, ny) + 1);
    }
    return dp[x][y];
}
// memset(dp, 0, sizeof(dp));
// ans = max{dfs(i, j)} 对所有 1 ≤ i ≤ R, 1 ≤ j ≤ C
```

**核心启示**：
- **严格递减 = 天然无环**：与 LC 329 同理，高度严格递减保证不会走回头路，无需额外 `vis` 数组
- **memo 初始化为 0**：路径长度 ≥ 1，0 表示未计算，比 -1 更简洁
- **全局起点枚举**：答案不是固定起点，需要对所有格子做一次 DFS 取最大值

### 题目 17：游荡的奶牛 Cow Travelling（洛谷 P1535 [USACO08MAR]）

🟠 Level 3 — 三维状态记忆化

> `N×M` 的草地网格，`*` 为障碍（树），`.` 可通行。奶牛从 `(R1, C1)` 出发，每秒必须向上下左右移动一格（不能停留），可走重复格。求**恰好** `T` 步到达 `(R2, C2)` 的路径方案数。（N, M ≤ 100，T ≤ 15）

**分析**：关键在于"恰好 T 步"，需要用步数作为状态的一维。`dfs(x, y, t)` 表示从 `(x, y)` 出发还剩 `t` 步到达终点的方案数。T ≤ 15 保证三维状态空间可控。曼哈顿距离剪枝可进一步加速。

```
状态定义：dfs(x, y, t) = 从 (x, y) 还剩 t 步恰好到达 (R2, C2) 的方案数
转移方程：dfs(x, y, t) = Σ dfs(nx, ny, t-1)  对所有合法相邻格
边界条件：t == 0 时，若 (x, y) == (R2, C2) 返回 1，否则返回 0
答案位置：dfs(R1, C1, T)
```

**C++ 代码**：
```cpp
long long memo[105][105][16];
bool vis[105][105][16];
char grid[105][105];
int N, M, T, R1, C1, R2, C2;
int dx[] = {0, 0, 1, -1}, dy[] = {1, -1, 0, 0};

long long dfs(int x, int y, int t) {
    if (t == 0) return (x == R2 && y == C2) ? 1 : 0;
    if (abs(x - R2) + abs(y - C2) > t) return 0; // 曼哈顿剪枝
    if (vis[x][y][t]) return memo[x][y][t];
    vis[x][y][t] = true;
    long long res = 0;
    for (int d = 0; d < 4; d++) {
        int nx = x + dx[d], ny = y + dy[d];
        if (nx >= 1 && nx <= N && ny >= 1 && ny <= M
            && grid[nx][ny] != '*')
            res += dfs(nx, ny, t - 1);
    }
    return memo[x][y][t] = res;
}
// memset(vis, false, sizeof(vis)); dfs(R1, C1, T)
```

**核心启示**：
- **步数作为状态维度**：恰好 T 步的约束通过增加第三维 `t` 来处理，是"固定步数路径计数"的通用手法
- **曼哈顿距离剪枝**：若剩余步数不够走到终点，直接返回 0，大幅减少无效搜索
- **vis + memo 分离**：由于方案数可能为 0 且 0 是合法答案，用额外 `vis` 数组标记是否已计算

### 题目 18：引水入城（洛谷 P1514 [NOIP2010 提高组]）

🔴 Level 4 — 记忆化搜索 + 区间覆盖

> `N×M` 的网格，每格有一个海拔值。蓄水厂只能建在第 1 行，水从高处流向相邻低处（上下左右）。需要让第 `N` 行所有城市都能获得水。问能否实现全覆盖？若能，输出最少蓄水厂数量；若不能，输出无法覆盖的城市数。

**分析**：关键性质——从第 1 行某点出发，能到达第 `N` 行的城市一定构成一段**连续区间** `[L, R]`（可反证）。因此先对第 1 行每个点做一次记忆化 DFS，求出其覆盖的区间。若所有区间的并不能覆盖 `[1, M]` 则无解；否则问题转化为经典的**最少区间覆盖**贪心。

```
状态定义：对每个第 1 行的点 j，DFS 求出能覆盖第 N 行的区间 [L[j], R[j]]
搜索过程：dfs(x, y) 从 (x, y) 向四方向扩展（只能高→低）
          到达第 N 行时更新对应源点的 L、R
记忆化：vis[x][y] 标记是否访问过，避免重复搜索
最终问题：用 [L[j], R[j]] 最少覆盖 [1, M]（贪心）
```

**C++ 代码**：
```cpp
int h[505][505], N, M;
bool vis[505][505];
int L[505], R[505]; // 第 1 行第 j 列覆盖第 N 行的区间
int dx[] = {0, 0, 1, -1}, dy[] = {1, -1, 0, 0};

void dfs(int x, int y, int src) {
    vis[x][y] = true;
    if (x == N) {
        L[src] = min(L[src], y);
        R[src] = max(R[src], y);
    }
    for (int d = 0; d < 4; d++) {
        int nx = x + dx[d], ny = y + dy[d];
        if (nx >= 1 && nx <= N && ny >= 1 && ny <= M
            && !vis[nx][ny] && h[nx][ny] < h[x][y])
            dfs(nx, ny, src);
    }
}

// 主流程：
// for j in [1, M]: L[j] = M+1, R[j] = 0
// for j in [1, M]: memset(vis, 0, sizeof(vis)); dfs(1, j, j)
// 检查第 N 行是否全被覆盖；若未全覆盖输出 0 和未覆盖数
// 否则对区间 [L[j], R[j]] 做贪心最少覆盖
int greedy() {
    int cnt = 0, cur = 0; // cur: 当前已覆盖到的右端
    while (cur < M) {
        int best = cur;
        for (int j = 1; j <= M; j++)
            if (L[j] <= cur + 1)
                best = max(best, R[j]);
        if (best == cur) return -1; // 无法推进
        cur = best; cnt++;
    }
    return cnt;
}
```

**核心启示**：
- **搜索 + 贪心组合**：DFS 负责求每个源点的覆盖区间，贪心负责最优选取——两阶段思维是本题精髓
- **连续区间性质**：从一个源点能覆盖的最后一行必然是连续区间，这是贪心策略正确性的基石
- **多源 DFS**：对第 1 行每个点分别搜索，每次搜索独立计算覆盖范围

### 题目 19：逛公园（洛谷 P3953 [NOIP2017 提高组]）

🔴 Level 4 — 记忆化搜索 + 最短路

> 有向图 `N` 个点 `M` 条边（边权 ≥ 0），1 号为起点，N 号为终点。设 1→N 最短路长度为 `d`，求长度不超过 `d+K` 的路径条数（对 `P` 取模）。若存在无穷多条合法路径，输出 `-1`。（K ≤ 50）

**分析**：将"路径长度"转化为"比最短路多走了多少"。设 `dis[u]` 为 1→u 的最短路（Dijkstra 预处理），则走到 `u` 时"多花的代价"为 `j = 实际花费 - dis[u]`。`dfs(u, j)` 表示从 `u` 出发、还可以多花 `j` 步到达 `N` 的方案数。若存在"0 权环"可达终点，则方案数无穷，需特判。

```
状态定义：dfs(u, j) = 从 u 出发，可多花 j 的代价到达 N 的方案数
转移方程：dfs(u, j) = Σ dfs(v, j - extra)
         其中 extra = w(u,v) - (dis[v] - dis[u]) ≥ 0
边界条件：u == N 且 j ≥ 0 时贡献 1（不终止，因为可绕路回来）
判无穷：若搜索时某状态 (u, j) 正在栈中又被访问到 → 0 环可达，输出 -1
答案位置：Σ dfs(1, j)  对 j = 0..K（等价于 dfs(1, K) 若终点累计所有 j ≥ 0）
```

**C++ 代码**：
```cpp
const int MAXN = 100005;
int dis[MAXN]; // 1 到各点最短路（Dijkstra）
int memo[MAXN][52]; // memo[u][j]
int inStack[MAXN][52]; // 检测 0 环
bool visited[MAXN][52];
int N, M, K, P;
vector<pair<int,int>> adj[MAXN]; // 邻接表 (v, w)
bool has_inf;

int dfs(int u, int j) {
    if (j < 0) return 0;
    if (inStack[u][j]) { has_inf = true; return 0; }
    if (visited[u][j]) return memo[u][j];
    visited[u][j] = true;
    inStack[u][j] = true;
    int res = (u == N) ? 1 : 0;
    for (auto [v, w] : adj[u]) {
        int extra = w - (dis[v] - dis[u]);
        if (extra < 0 || j - extra < 0) continue;
        res = (res + dfs(v, j - extra)) % P;
        if (has_inf) return 0;
    }
    inStack[u][j] = false;
    return memo[u][j] = res;
}
// Dijkstra 求 dis[]; memset 初始化; ans = dfs(1, K)
// 若 has_inf 则输出 -1，否则输出 ans
```

**核心启示**：
- **"多花代价"分层**：将绝对路径长度转为相对最短路的偏移量，K ≤ 50 保证状态空间可控
- **0 环检测**：在记忆化搜索中用 `inStack` 标记判断是否存在 0 权环导致无穷方案
- **最短路 + DP 结合**：Dijkstra 预处理提供全局信息，记忆化搜索在此基础上做路径计数——是图上 DP 的经典范式

## 五、常见技巧与思路总结

### 5.1 memo 数组初始化：-1 vs 极值 vs 哈希

| 初始化方式 | 代码 | 适用场景 | 注意事项 |
|-----------|------|---------|---------|
| `-1` 初始化 | `memset(memo, -1, sizeof(memo))` | 答案 ≥ 0 的大多数问题 | 最常用，需确认 -1 不是合法答案 |
| `0` 初始化 | `memset(memo, 0, sizeof(memo))` | 答案 ≥ 1 的路径类问题 | 0 表示未计算（如 LeetCode 329） |
| `0x3f` 极值 | `memset(memo, 0x3f, sizeof(memo))` | 求最小值问题 | 每个字节填 0x3f，int 值约 1e9 |
| `bool vis[]` | 额外开 `vis[N]` 标记 | 答案可能为 0 或 -1 | 多一个数组，更通用但占空间 |
| 哈希表 | `map<tuple, int>` | 状态空间稀疏或含负数参数 | 常数大，适合参数范围不确定的题 |

**经验法则**：优先尝试 `-1` 初始化，不行就用 `vis[]` 数组，参数含负数或范围大就用哈希表。

### 5.2 从递推 DP 反推记忆化写法

已有递推 DP 代码时，可以机械地转化为记忆化搜索：

```
递推 DP：                          记忆化搜索：
┌─────────────────────────┐       ┌─────────────────────────┐
│ for (int i = ...) {     │       │ int dfs(int i, int j) { │
│   for (int j = ...) {   │  ──►  │   if (base) return ...;  │
│     dp[i][j] = 转移;    │       │   if (memo) return memo; │
│   }                     │       │   memo = 转移(dfs递归);   │
│ }                       │       │   return memo;            │
│ ans = dp[...][...];     │       │ }                         │
└─────────────────────────┘       └─────────────────────────┘
```

**转化步骤**：
1. 将 `dp[i][j] = ...` 的循环变量变为函数参数
2. 将填表的 base case 变为递归的 base case
3. 将 `dp[依赖项]` 变为 `dfs(依赖项)` 的递归调用
4. 函数入口加 memo 检查，出口存 memo

### 5.3 何时选记忆化而非递推

**优先选记忆化搜索的场景**：
- 状态空间稀疏，大量状态不会被访问（如博弈论、搜索剪枝类）
- 转移方程复杂，填表顺序不容易确定（如区间 DP、树形 DP）
- 需要和 DFS 天然结合（如网格搜索、图上 DP）
- 刚接触一个新问题，用递归思维更容易想清楚状态转移

**优先选递推 DP 的场景**：
- 需要空间优化（滚动数组）
- 状态空间稠密，所有状态都会被计算
- 递归深度过大可能栈溢出（n > 10⁵）
- 需要字典序最优解等特殊输出

### 5.4 记忆化搜索的常见错误

**错误 1：memo 初始值与合法答案冲突**
```cpp
// ❌ 若答案可以为 0，则 memo[i]=0 被误判为"已计算"
memset(memo, 0, sizeof(memo));
if (memo[i] != 0) return memo[i]; // 答案为 0 的状态会被重复计算
```

**错误 2：忘记存 memo**
```cpp
// ❌ 计算了结果但忘记存入备忘录，等于没有记忆化
int dfs(int i) {
    if (memo[i] != -1) return memo[i];
    int res = dfs(i-1) + dfs(i-2);
    return res; // 应该是 return memo[i] = res;
}
```

**错误 3：状态定义不完整**
```cpp
// ❌ 决策还依赖于"上一个选了什么"，但状态里没有这个参数
int dfs(int i) { ... }        // 缺少必要参数
int dfs(int i, int last) { ... } // ✅ 正确
```

**错误 4：递归参数与 memo 维度不匹配**
```cpp
// ❌ 函数有两个参数但 memo 只有一维
int memo[MAXN]; // 应该是 memo[MAXN][MAXW]
int dfs(int i, int w) { ... }
```

### 5.5 空间优化技巧

记忆化搜索的空间优化不如递推 DP 灵活，但仍有一些技巧：

- **哈希表替换数组**：当状态空间巨大但实际访问的状态稀疏时，用 `unordered_map` 代替数组
- **状态压缩**：将多个小参数打包成一个整数作为 key，如 `key = i * W + j`
- **及时释放**：在树形 DP 中，子树处理完后可以释放其 memo 空间（较少用）
- **转为递推**：如果空间实在不够，将记忆化搜索转为递推 DP 后使用滚动数组

## 七、终极总结与归纳

### 7.1 记忆化搜索的本质

**一句话**：记忆化搜索 = DFS + 缓存 = 自顶向下的动态规划。

它是连接"暴力搜索"与"动态规划"的桥梁。你不需要一开始就想出完美的递推填表顺序——先写暴力 DFS，发现重叠子问题后加个 memo 数组，就完成了从搜索到 DP 的优雅过渡。它特别适合状态转移复杂、填表顺序不易确定的问题，也是竞赛中快速解题的利器。

### 7.2 核心要点

1. **状态设计是灵魂**：DFS 函数的参数就是"状态"，参数选对了，问题就解决了一半
2. **memo 初始值要安全**：必须与所有合法答案区分开，`-1` 是最常用的哨兵值
3. **每个状态只算一次**：这是记忆化搜索的核心保证，也是复杂度分析的基础
4. **边界条件要完备**：漏掉 base case 是最常见的 bug 来源
5. **先暴力再优化**：不要一开始就想 DP 公式，先写暴力 DFS，再加 memo
6. **与递推 DP 等价**：记忆化搜索和递推 DP 解决的是同一个问题，可以互相转化
