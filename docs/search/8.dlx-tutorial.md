## 一、什么是 DLX

### 通俗理解

DLX 全称 **Dancing Links X**（舞蹈链），由 Donald Knuth 在 2000 年提出，是求解**精确覆盖问题**（Exact Cover Problem）的高效算法。

想象一块棋盘，你手里有若干不同形状的拼图块，要求**恰好无重叠、无空隙地铺满整块棋盘**——这就是精确覆盖。DLX 用一种极其巧妙的"舞蹈"操作来完成这件事：链表节点被摘除后，只要保留原始指针，就能以 O(1) 的代价**原地恢复**，回溯时就像跳舞一样优雅地"跳回来"。

DLX = **Algorithm X**（回溯搜索框架） + **Dancing Links**（双向十字链表实现）。

### 双向十字链表结构（ASCII Art）

```
       head ←→ C1 ←→ C2 ←→ C3 ←→ C4 ←→ (back to head)
                ↕      ↕      ↕      ↕
               n11 ←→ n12          ←→ n14
                ↕             ↕
               n21    n22 ←→ n23
                ↕      ↕      ↕      ↕
               (C1)   (C2)   (C3)   (C4)   ← 列头环形链

    每个节点有 L/R/U/D 四个方向的双向指针
    行内节点通过 L/R 形成环；列内节点通过 U/D 形成环
    每列有一个列头 (Column Header)，记录该列 1 的个数
```

### Algorithm X 与 Dancing Links 的关系

| 层次 | 内容 | 角色 |
|------|------|------|
| **Algorithm X** | Knuth 的递归回溯策略 | "怎么搜" |
| **Dancing Links** | 双向十字链表 + remove/recover | "怎么存 + 怎么改" |
| **DLX** | 两者结合 | 完整求解器 |

Algorithm X 是搜索策略——每次选一列、尝试该列所有行、递归、回溯。Dancing Links 是它的数据结构引擎——让每次"删除"和"恢复"都在 O(1) 完成，使回溯的代价极低。

## 二、核心思想与前置知识

### 2.1 精确覆盖问题（Exact Cover）

**定义**：给定一个 01 矩阵，选出若干行，使得每一列**恰好有且只有一个 1**。

例如：
```
     C1 C2 C3 C4 C5 C6 C7
R1:   0  0  1  0  1  1  0
R2:   1  0  0  1  0  0  1
R3:   0  1  1  0  0  1  0
R4:   1  0  0  1  0  0  0
R5:   0  1  0  0  0  0  1
R6:   0  0  0  1  1  0  1

选择 R1, R4, R5 → 每列恰好一个 1 ✓
```

### 2.2 Algorithm X 的回溯思想

1. 若矩阵为空（所有列都被覆盖），成功，输出当前解
2. 选择一个列 C（策略：选含 1 最少的列，减小分支因子）
3. 对列 C 中每个含 1 的行 R：
   - 将 R 加入部分解
   - 对 R 中每个 1 所在列 J：删除列 J 以及所有在 J 列含 1 的行
   - 递归求解剩余子矩阵
   - 回溯：恢复所有被删除的行和列
4. 若没有行可选，回溯

### 2.3 双向十字链表的数据结构

每个节点存储 6 个信息：

```
struct Node {
    int L, R, U, D;  // 左右上下四个方向的链接
    int row, col;     // 所在行、列编号
};
```

列头节点额外存储 `size`（该列 1 的个数），用于启发式选择。

所有列头通过 L/R 连成环，加一个超级头节点 `head`。同列节点通过 U/D 连成环，同行节点通过 L/R 连成环。

### 2.4 remove / recover 操作详解

**remove(c)**：将列 c 从列头链中摘除，并删除该列所有 1 所在的整行。

```
remove(c):
    L[R[c]] = L[c]        // 列头从左右链中摘除
    R[L[c]] = R[c]
    for i = D[c] → 回到 c:     // 遍历列 c 的每个节点
        for j = R[i] → 回到 i:  // 遍历该行的每个节点
            U[D[j]] = U[j]      // 从上下链中摘除
            D[U[j]] = D[j]
            size[col[j]]--
```

**recover(c)**：逆序恢复（核心！顺序必须与 remove 严格相反）。

```
recover(c):
    for i = U[c] → 回到 c:      // 逆序遍历
        for j = L[i] → 回到 i:
            size[col[j]]++
            U[D[j]] = j          // 重新链入
            D[U[j]] = j
    L[R[c]] = c
    R[L[c]] = c
```

**为什么能恢复？** 节点被摘除后，它自身的 L/R/U/D 指针并未改变——它还"记得"自己原来在哪里。只要按逆序操作，就能完美恢复，这就是"舞蹈"的精髓。

### 2.5 DLX 完整骨架代码（精确覆盖）

```cpp
const int MAXN = 100010;

// 节点数组，0 号为 head
int L[MAXN], R[MAXN], U[MAXN], D[MAXN];
int col[MAXN], row[MAXN];
int sz[MAXN];       // sz[c] = 列 c 中 1 的个数
int head_node[MAXN]; // head_node[r] = 第 r 行的第一个节点（用于建表）
int idx;            // 节点计数器
int n, m;           // n 行 m 列
int ans[MAXN];      // 保存选中的行
int ans_cnt;

void init(int _m) {
    // 建立 head(0号) + m 个列头(1..m)
    for (int i = 0; i <= _m; i++) {
        L[i] = i - 1; R[i] = i + 1;
        U[i] = D[i] = i;
        sz[i] = 0;
    }
    L[0] = _m; R[_m] = 0;
    idx = _m + 1;
    memset(head_node, 0, sizeof(head_node));
}

void addNode(int r, int c) {
    // 在第 r 行、第 c 列添加一个 1
    col[idx] = c; row[idx] = r; sz[c]++;
    // 插入列 c 的上下链
    U[idx] = U[c]; D[idx] = c;
    D[U[c]] = idx; U[c] = idx;
    // 插入行 r 的左右链
    if (!head_node[r]) {
        head_node[r] = L[idx] = R[idx] = idx;
    } else {
        L[idx] = L[head_node[r]];
        R[idx] = head_node[r];
        R[L[head_node[r]]] = idx;
        L[head_node[r]] = idx;
    }
    idx++;
}

void remove(int c) {
    L[R[c]] = L[c]; R[L[c]] = R[c];
    for (int i = D[c]; i != c; i = D[i])
        for (int j = R[i]; j != i; j = R[j]) {
            U[D[j]] = U[j]; D[U[j]] = D[j];
            sz[col[j]]--;
        }
}

void recover(int c) {
    for (int i = U[c]; i != c; i = U[i])
        for (int j = L[i]; j != i; j = L[j]) {
            sz[col[j]]++;
            U[D[j]] = j; D[U[j]] = j;
        }
    L[R[c]] = c; R[L[c]] = c;
}

bool dance(int dep) {
    if (R[0] == 0) { ans_cnt = dep; return true; }
    // 选含 1 最少的列
    int c = R[0];
    for (int i = R[0]; i != 0; i = R[i])
        if (sz[i] < sz[c]) c = i;
    remove(c);
    for (int i = D[c]; i != c; i = D[i]) {
        ans[dep] = row[i];
        for (int j = R[i]; j != i; j = R[j]) remove(col[j]);
        if (dance(dep + 1)) return true;
        for (int j = L[i]; j != i; j = L[j]) recover(col[j]);
    }
    recover(c);
    return false;
}
```

## 三、DLX 的两大问题模型详解

### 3.1 精确覆盖（Exact Cover）

**定义**：选若干行，使每列**恰好被覆盖一次**。

**建模方法**：
- **列** = 需要满足的约束条件（每个条件必须且只能被满足一次）
- **行** = 一个决策/选择（选这一行意味着同时满足它所覆盖的若干列约束）
- **矩阵中的 1** = 第 r 行的决策覆盖了第 c 列的约束

**模板代码**：见 2.5 节完整骨架。

**典型场景**：数独（每格填一个数、每行每列每宫各数字恰好一次）、N 皇后、拼图铺满棋盘。

### 3.2 重复覆盖（Multiple Cover）

**定义**：选若干行，使每列**至少被覆盖一次**，求最少行数。

**与精确覆盖的差异**：

| 对比项 | 精确覆盖 | 重复覆盖 |
|--------|----------|----------|
| 目标 | 每列恰好 1 次 | 每列至少 1 次 |
| remove 操作 | 删列 + 删关联行 | 仅删列（不删其他行） |
| 搜索策略 | 纯回溯 | IDA*（迭代加深 + 估价剪枝） |
| 典型应用 | 数独、拼图 | 最小集合覆盖、最少操作覆盖 |

**修改要点**：
1. `remove(c)` 只摘除列头，不删除关联行的其他节点
2. 新增估价函数 `h()` 计算还需最少多少步
3. 搜索改为 IDA*，`dep + h() > limit` 时剪枝

**重复覆盖模板代码**：

```cpp
const int MAXN = 100010;
const int MAXM = 510;

int L[MAXN], R[MAXN], U[MAXN], D[MAXN];
int col[MAXN], row[MAXN], sz[MAXM];
int head_node[MAXN];
int idx, m;
bool vis[MAXM];

void init(int _m) {
    m = _m;
    for (int i = 0; i <= m; i++) {
        L[i] = i - 1; R[i] = i + 1;
        U[i] = D[i] = i; sz[i] = 0;
    }
    L[0] = m; R[m] = 0;
    idx = m + 1;
    memset(head_node, 0, sizeof(head_node));
}

void addNode(int r, int c) {
    col[idx] = c; row[idx] = r; sz[c]++;
    U[idx] = U[c]; D[idx] = c;
    D[U[c]] = idx; U[c] = idx;
    if (!head_node[r]) {
        head_node[r] = L[idx] = R[idx] = idx;
    } else {
        L[idx] = L[head_node[r]];
        R[idx] = head_node[r];
        R[L[head_node[r]]] = idx;
        L[head_node[r]] = idx;
    }
    idx++;
}

// 重复覆盖的 remove：仅摘列头
void remove(int c) {
    L[R[c]] = L[c]; R[L[c]] = R[c];
}

void recover(int c) {
    L[R[c]] = c; R[L[c]] = c;
}

// 估价函数：贪心计算还需覆盖的最少步数
int h() {
    int cnt = 0;
    memset(vis, false, sizeof(vis));
    for (int c = R[0]; c != 0; c = R[c]) {
        if (!vis[c]) {
            cnt++;
            vis[c] = true;
            for (int i = D[c]; i != c; i = D[i])
                for (int j = R[i]; j != i; j = R[j])
                    vis[col[j]] = true;
        }
    }
    return cnt;
}

int ans;

void dance(int dep) {
    if (dep + h() >= ans) return;  // 剪枝
    if (R[0] == 0) { ans = dep; return; }
    // 选含 1 最少的列
    int c = R[0];
    for (int i = R[0]; i != 0; i = R[i])
        if (sz[i] < sz[c]) c = i;
    for (int i = D[c]; i != c; i = D[i]) {
        // 删除该行覆盖的所有列
        remove(col[i]);
        for (int j = R[i]; j != i; j = R[j]) remove(col[j]);
        // 删除与已删列冲突的节点
        for (int k = D[col[i]]; k != col[i]; k = D[k])
            if (k != i) { U[D[k]] = U[k]; D[U[k]] = D[k]; }
        for (int j = R[i]; j != i; j = R[j])
            for (int k = D[col[j]]; k != col[j]; k = D[k])
                if (k != j) { U[D[k]] = U[k]; D[U[k]] = D[k]; }

        dance(dep + 1);

        // 恢复
        for (int j = L[i]; j != i; j = L[j])
            for (int k = U[col[j]]; k != col[j]; k = U[k])
                if (k != j) { U[D[k]] = k; D[U[k]] = k; }
        for (int k = U[col[i]]; k != col[i]; k = U[k])
            if (k != i) { U[D[k]] = k; D[U[k]] = k; }
        for (int j = L[i]; j != i; j = L[j]) recover(col[j]);
        recover(col[i]);
    }
}
```

### 3.3 常见建模场景

| 场景 | 行的含义 | 列的含义 | 覆盖类型 |
|------|----------|----------|----------|
| 标准数独 | 在 (r,c) 填数字 d | 324 列：格约束+行数字+列数字+宫数字 | 精确覆盖 |
| N 皇后 | 在 (r,c) 放皇后 | 行+列（必选）+对角线（可选） | 精确覆盖（对角线为次要列） |
| 拼图铺板 | 用形状 p 的某旋转在某位置放置 | 每格 + 每形状各用一次 | 精确覆盖 |
| 最少集合覆盖 | 选集合 S_i | 每个元素至少被覆盖 | 重复覆盖 |

### 精确覆盖 vs 重复覆盖 对比总表

```
┌──────────────┬────────────────────┬──────────────────────┐
│              │   精确覆盖          │   重复覆盖            │
├──────────────┼────────────────────┼──────────────────────┤
│ 目标         │ 每列恰好 1 次       │ 每列至少 1 次         │
│ remove       │ 删列 + 删关联行     │ 仅删列头              │
│ 搜索         │ 纯回溯 dance()     │ IDA* + h() 估价       │
│ 求什么       │ 可行解 / 所有解     │ 最少行数              │
│ 代表题       │ 数独、拼图          │ 最小覆盖、最少操作     │
└──────────────┴────────────────────┴──────────────────────┘
```

## 四、12 道经典例题详解

### 🟢 入门级 Level 1：精确覆盖基础

### 题目 1：精确覆盖入门（自编模板题）

> 给定一个 n×m 的 01 矩阵，选出若干行，使每列恰好有且仅有一个 1。输出任一可行方案（选中行的编号），无解输出 -1。

**分析**：这是最纯粹的精确覆盖问题，直接套用 DLX 模板。建表后调用 `dance(0)` 即可。

```
状态定义：DLX 内部隐式维护"当前哪些列未覆盖"
转移方程：选行 r → 覆盖 r 中所有 1 对应的列 → 递归子问题
边界条件：R[0] == 0 时所有列已覆盖，找到解
答案位置：ans[] 数组存储选中的行编号
```

**C++ 代码**：

```cpp
// 使用 2.5 节的 DLX 骨架代码
// 建表
void solve(int n, int m, vector<vector<int>>& matrix) {
    init(m);
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= m; j++)
            if (matrix[i][j]) addNode(i, j);
    if (dance(0)) {
        // ans[0..ans_cnt-1] 即选中的行
        sort(ans, ans + ans_cnt);
    }
    // 否则无解
}
```

**核心启示**：
- **DLX 是精确覆盖的通用求解器**：只要能建出 01 矩阵，就能直接求解
- **建表复杂度与 1 的个数线性相关**：稀疏矩阵效率极高
- **选列策略至关重要**：选 sz 最小的列能大幅减小搜索树

### 题目 2：精确覆盖问题模板（洛谷 P4929）

> 给定 n 行 m 列的 01 矩阵，找到一组行的集合使得集合中每列恰好包含一个 1。输出选中的行号，无解输出 "No Solution!"。

**分析**：洛谷标准 DLX 模板题。直接读入矩阵、建表、搜索。注意行号从 1 开始。

```
状态定义：同题 1
转移方程：同题 1
边界条件：同题 1
答案位置：ans[] 按升序输出
```

**C++ 代码**：

```cpp
// 使用 2.5 节 DLX 骨架
// 读入后建表并求解
void solve() {
    init(m);
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= m; j++) {
            int x;
            // 假设 x 已读入
            if (x) addNode(i, j);
        }
    if (dance(0)) {
        sort(ans, ans + ans_cnt);
        for (int i = 0; i < ans_cnt; i++)
            printf("%d ", ans[i]);
    } else {
        printf("No Solution!");
    }
}
```

**核心启示**：
- **熟练套用模板是基础**：P4929 是 DLX 的 "Hello World"
- **注意输出格式**：有些题要求行号排序后输出
- **调试技巧**：小矩阵手动验证 remove/recover 是否正确

### 🟡 进阶级 Level 2：经典建模

### 题目 3：9×9 标准数独（经典建模题）

> 给定一个 9×9 的数独，部分格子已填数字（1-9），其余为 0。填满所有格子，使每行、每列、每个 3×3 宫内的数字 1-9 各出现恰好一次。

**分析**：数独是 DLX 最经典的应用。关键在于将数独的 4 类约束转化为精确覆盖的列：

1. **格约束**（81 列）：每个格子恰好填一个数
2. **行-数字约束**（81 列）：每行中 1-9 各出现一次
3. **列-数字约束**（81 列）：每列中 1-9 各出现一次
4. **宫-数字约束**（81 列）：每宫中 1-9 各出现一次

共 324 列。每行代表"在 (r,c) 填数字 d"，共最多 729 行（81 格 × 9 种数字）。

```
状态定义：DLX 矩阵 729 行 × 324 列
建模映射：
  行 = (r, c, d) 表示在 (r,c) 填 d
  列 1~81   : 格约束   → (r-1)*9 + c
  列 82~162 : 行数字   → 81 + (r-1)*9 + d
  列 163~243: 列数字   → 162 + (c-1)*9 + d
  列 244~324: 宫数字   → 243 + (b-1)*9 + d
  其中 b = ((r-1)/3)*3 + (c-1)/3 + 1
答案位置：从 ans[] 反推每格填的数字
```

**C++ 代码**：

```cpp
int board[10][10];  // 输入的数独盘面

void buildSudoku() {
    init(324);
    for (int r = 1; r <= 9; r++)
        for (int c = 1; c <= 9; c++) {
            int lo = 1, hi = 9;
            if (board[r][c]) lo = hi = board[r][c];  // 已填的格只有一种选择
            for (int d = lo; d <= hi; d++) {
                int rowId = (r - 1) * 81 + (c - 1) * 9 + d;
                int b = ((r - 1) / 3) * 3 + (c - 1) / 3 + 1;
                addNode(rowId, (r - 1) * 9 + c);           // 格约束
                addNode(rowId, 81 + (r - 1) * 9 + d);      // 行-数字
                addNode(rowId, 162 + (c - 1) * 9 + d);     // 列-数字
                addNode(rowId, 243 + (b - 1) * 9 + d);     // 宫-数字
            }
        }
    dance(0);
    // 从 ans[] 还原盘面
    for (int i = 0; i < ans_cnt; i++) {
        int v = ans[i] - 1;
        int r = v / 81 + 1, c = (v / 9) % 9 + 1, d = v % 9 + 1;
        board[r][c] = d;
    }
}
```

**核心启示**：
- **数独的 324 列建模是 DLX 经典范式**：4 类约束 × 81 = 324
- **已填格限制选择范围**：只为已填格生成唯一行，减小矩阵规模
- **从 ans[] 反推原始信息**：行编号蕴含 (r, c, d) 三元组

### 题目 4：数独（洛谷 P1784）

> 输入一个 9×9 的数独盘面（0 表示待填），输出完整解。保证有唯一解。

**分析**：与题 3 完全相同的建模方式。本题保证唯一解，DLX 找到第一个解即可返回。竞赛中注意输入格式处理。

```
状态定义：同题 3（729 行 × 324 列精确覆盖）
答案位置：同题 3
```

**C++ 代码**：

### 题目 5：N 皇后问题（DLX 建模）

> 在 N×N 的棋盘上放置 N 个皇后，使得任意两个皇后不在同一行、同一列、同一对角线上。求方案数或输出一组解。

**分析**：N 皇后可以建模为精确覆盖+次要列。行和列是**必须恰好覆盖**的（主要列），对角线是**最多覆盖一次**的（次要列，允许为空）。

- **主要列**：N 个行约束 + N 个列约束 = 2N 列
- **次要列**：(2N-1) 个主对角线 + (2N-1) 个副对角线 = (4N-2) 列
- **行**：在 (r,c) 放皇后 → 覆盖行 r、列 c、主对角线 r-c+N、副对角线 r+c-1

次要列的处理：初始化时不将次要列链入列头环（或标记为可选），使 DLX 不强制覆盖它们。

```
状态定义：DLX 矩阵 N² 行 × (6N-2) 列
主要列：前 2N 列（行+列约束，必须覆盖）
次要列：后 4N-2 列（对角线约束，可选）
答案位置：ans[] 中的行号反推皇后坐标
```

**C++ 代码**：

```cpp
int N;

void buildNQueens() {
    int primary = 2 * N;
    int total = 6 * N - 2;
    init(total);
    // 将次要列从列头环中断开（不强制覆盖）
    for (int i = primary + 1; i <= total; i++) {
        L[R[i]] = L[i]; R[L[i]] = R[i];
        // 保留 U/D 链，只从 L/R 主链摘除
        L[i] = R[i] = i;
    }
    for (int r = 1; r <= N; r++)
        for (int c = 1; c <= N; c++) {
            int rid = (r - 1) * N + c;
            addNode(rid, r);                     // 行约束
            addNode(rid, N + c);                 // 列约束
            addNode(rid, 2 * N + r - c + N);     // 主对角线
            addNode(rid, 4 * N - 1 + r + c - 1); // 副对角线
        }
    dance(0);
}
```

**核心启示**：
- **次要列（secondary column）**：DLX 不仅能处理"恰好覆盖"，还能处理"最多覆盖一次"
- **建模的灵活性**：通过控制列头链的连接方式，实现主要/次要列区分
- **N 皇后用 DLX 更多是建模练习**：实际竞赛中位运算回溯更快

### 题目 6：重复覆盖模板题

**分析**：经典重复覆盖问题，使用 DLX + IDA* 求解。与精确覆盖的关键区别：remove 只删列头不删关联行，搜索用迭代加深 + 估价剪枝。

### 题目 8：拉丁方阵填数

> 在 N×N 的网格中填入 1~N，使得每行每列各数字恰好出现一次（部分格已填好）。输出任意一组解。

**分析**：拉丁方阵是"简化版数独"——没有宫约束。建模为精确覆盖：

- **格约束**（N² 列）：每格恰好填一个数
- **行-数字约束**（N² 列）：每行中 1~N 各出现一次
- **列-数字约束**（N² 列）：每列中 1~N 各出现一次

共 3N² 列，最多 N³ 行。

```
状态定义：DLX 矩阵 N³ 行 × 3N² 列
建模映射：
  行 = (r, c, d) 表示在 (r,c) 填 d
  列 1~N²          : 格约束
  列 N²+1~2N²      : 行-数字约束
  列 2N²+1~3N²     : 列-数字约束
答案位置：ans[] 反推 (r, c, d)
```

**C++ 代码**：

```cpp
int N;
int board[20][20];

void buildLatin() {
    int colCnt = 3 * N * N;
    init(colCnt);
    for (int r = 1; r <= N; r++)
        for (int c = 1; c <= N; c++) {
            int lo = 1, hi = N;
            if (board[r][c]) lo = hi = board[r][c];
            for (int d = lo; d <= hi; d++) {
                int rid = (r - 1) * N * N + (c - 1) * N + d;
                addNode(rid, (r - 1) * N + c);                // 格约束
                addNode(rid, N * N + (r - 1) * N + d);        // 行-数字
                addNode(rid, 2 * N * N + (c - 1) * N + d);    // 列-数字
            }
        }
    dance(0);
    for (int i = 0; i < ans_cnt; i++) {
        int v = ans[i] - 1;
        int r = v / (N * N) + 1;
        int c = (v / N) % N + 1;
        int d = v % N + 1;
        board[r][c] = d;
    }
}
```

**核心启示**：
- **拉丁方阵是数独建模的简化版**：去掉宫约束，只剩 3N² 列
- **建模能力可迁移**：掌握数独建模后，拉丁方阵、数壹等变种信手拈来
- **预填格限制 d 的范围**：一个常用的建模简化技巧

### 题目 10：五连块覆盖（Pentomino Tiling）

> 用 12 种五连块（pentomino）恰好无重叠地铺满一个 6×10 的矩形棋盘。输出任意一组方案。

**分析**：12 种五连块各用一次，铺满 60 格（6×10=60，12×5=60 恰好用完）。列 = 60 格约束 + 12 种块约束 = 72 列。行 = 每种块的每种旋转/翻转在每个合法位置的放置方案。

```
状态定义：DLX 矩阵 ~1500 行 × 72 列
建模映射：
  列 1~60    : 格子约束（每格恰被覆盖一次）
  列 61~72   : 块约束（每种五连块恰用一次）
  行 = (块 p, 形态 t, 起始位置)
答案位置：ans[] 反推每块放在哪里
```

**C++ 代码**：

```cpp
// 12 种五连块的所有旋转/翻转形态（预处理）
// shapes[p] = 该块所有不同形态的坐标偏移列表
vector<vector<pair<int,int>>> shapes[12];
int ROWS = 6, COLS_BOARD = 10;

void buildPentomino() {
    int totalCol = ROWS * COLS_BOARD + 12;  // 72 列
    init(totalCol);
    int rid = 0;
    for (int p = 0; p < 12; p++) {
        for (auto& shape : shapes[p]) {
            for (int r = 0; r < ROWS; r++)
                for (int c = 0; c < COLS_BOARD; c++) {
                    // 检查 shape 以 (r,c) 为锚点是否全部在棋盘内
                    bool ok = true;
                    vector<int> cells;
                    for (auto& [dr, dc] : shape) {
                        int nr = r + dr, nc = c + dc;
                        if (nr < 0 || nr >= ROWS || nc < 0 || nc >= COLS_BOARD)
                            { ok = false; break; }
                        cells.push_back(nr * COLS_BOARD + nc + 1);
                    }
                    if (!ok) continue;
                    rid++;
                    addNode(rid, 60 + p + 1);  // 块约束
                    for (int cell : cells)
                        addNode(rid, cell);      // 格约束
                }
        }
    }
    dance(0);

```

### 题目 11：最少行覆盖（高难重复覆盖题）

> 给定 n 行 m 列的 01 矩阵，选最少的行使每列至少被覆盖一次。若有多组最优解，输出字典序最小的方案。n,m ≤ 100。

**分析**：重复覆盖 + 字典序最小方案。用 IDA* 求出最少行数后，再在搜索中按行号从小到大尝试，找到的第一个方案即为字典序最小。

```
状态定义：DLX 重复覆盖模型
搜索框架：先 IDA* 确定最少行数 k，再 DFS 找字典序最小的 k 行组合
估价函数：h() 贪心独立集下界
答案位置：选中的行号集合
```

**C++ 代码**：

```cpp
// 基于 3.2 节重复覆盖模板
int chosen[110]; // 记录选中的行

bool dfs(int dep, int limit, int lastRow) {
    if (dep + h() > limit) return false;
    if (R[0] == 0) return true;
    int c = R[0];
    for (int i = R[0]; i != 0; i = R[i])
        if (sz[i] < sz[c]) c = i;
    for (int i = D[c]; i != c; i = D[i]) {
        if (row[i] <= lastRow) continue;  // 保证字典序
        chosen[dep] = row[i];
        // remove 该行覆盖的列
        remove(col[i]);
        for (int j = R[i]; j != i; j = R[j]) remove(col[j]);
        if (dfs(dep + 1, limit, row[i])) return true;
        for (int j = L[i]; j != i; j = L[j]) recover(col[j]);
        recover(col[i]);
    }
    return false;
}

void solve() {
    for (int limit = h(); limit <= n; limit++)
        if (dfs(0, limit, 0)) break;
}
```

**核心启示**：
- **字典序约束通过搜索顺序控制**：只尝试行号大于上一选择的行
- **两遍搜索策略**：第一遍求最少行数，第二遍按字典序找方案
- **IDA* 的 limit 递增保证最优性**：找到的第一个解必为最少行

## 五、常见技巧与思路总结

### 5.1 如何将问题建模为 01 矩阵

**通用思路**：
1. 确定**约束条件** → 每个约束对应一列
2. 确定**决策/选择** → 每个可能的选择对应一行
3. 确定**关联关系** → 选择 r 满足约束 c 时，矩阵 [r][c] = 1

```
约束 = 列     选择 = 行     满足关系 = 1
```

**常见映射模式**：

| 问题类型 | 约束（列） | 选择（行） |
|----------|-----------|-----------|
| 数独 | 格、行数字、列数字、宫数字 | (r,c) 填 d |
| N 皇后 | 行、列、（对角线） | (r,c) 放皇后 |
| 拼图铺板 | 格子、零件种类 | 零件某形态放某位置 |
| 集合覆盖 | 每个元素 | 选集合 S_i |

### 5.2 列选择策略（MRV 启发式）

**核心原则**：每次选择含 1 最少的列（Minimum Remaining Values）。

```cpp
int c = R[0];
for (int i = R[0]; i != 0; i = R[i])
    if (sz[i] < sz[c]) c = i;
```

- sz[c] == 0：无行可选，直接回溯，这是最强的剪枝
- sz[c] == 1：唯一选择，无需分支
- sz[c] 越小：分支因子越小，搜索树越小

### 5.3 预处理与优化

1. **预填格限制**：数独中已填的格只生成一行（而非 9 行）
2. **形态去重**：拼图中相同旋转/翻转只保留一次
3. **合法性剪枝**：建表时就排除不合法的放置方案
4. **对称性消除**：某些问题可通过固定第一个选择减半搜索量

### 5.4 DLX 的空间与时间常数优化

**空间优化**：
- 节点数 = 矩阵中 1 的个数 + 列头数 + 1（head）
- 用数组而非动态分配，避免内存碎片

**时间优化**：
- 使用 `memset` 初始化而非逐一赋值
- 行内节点按列号排序可改善缓存命中
- 实测中 DLX 常数极小，通常不需要额外优化

```
空间复杂度：O(矩阵中 1 的个数)
时间复杂度：最坏指数级，但 MRV + 稀疏矩阵使实际效率极高
```

### 5.5 DLX 与其他搜索方法的选择

| 方法 | 适用场景 | 优劣 |
|------|----------|------|
| DLX 精确覆盖 | 约束恰好满足一次 | 最佳选择 |
| DLX 重复覆盖 | 约束至少满足一次 | 需 IDA* 辅助 |
| 位运算回溯 | N 皇后、小规模数独 | 常数更小 |
| SAT 求解器 | 超大规模约束满足 | 建模更通用 |
| 贪心/DP | 具有最优子结构 | 不适合严格覆盖 |

**何时用 DLX**：问题可自然建模为 01 矩阵覆盖，且约束数和选择数在可承受范围内（通常各 ≤ 几千）。

## 七、终极总结与归纳

### 7.1 DLX 的本质

**一句话**：DLX 是通过双向十字链表高效实现的精确覆盖回溯求解器——用"舞蹈般"的删除与恢复，在指数搜索空间中极速剪枝。

展开来说：Algorithm X 提供了搜索策略（选列→选行→递归→回溯），Dancing Links 提供了数据结构引擎（O(1) 删除与恢复）。两者结合，使得即使面对 NP 完全问题，在实际中也能高效求解。核心在于**建模**——能否将问题转化为 01 矩阵的行列覆盖关系，决定了 DLX 能否发挥作用。

### 7.2 核心要点

1. **精确覆盖是 DLX 的灵魂**：每列恰好覆盖一次是最核心的模型
2. **列 = 约束，行 = 选择**：建模的关键是识别约束和决策
3. **MRV 策略是效率关键**：选含 1 最少的列，sz==0 直接回溯
4. **remove/recover 的逆序性**：恢复必须与删除严格逆序
5. **重复覆盖需要 IDA* 辅助**：估价函数 h() 是性能瓶颈
6. **预处理决定建表质量**：旋转去重、合法性过滤、已知格限制
