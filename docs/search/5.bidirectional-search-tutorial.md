## 一、什么是双向搜索

双向搜索是一种**从问题的两端同时出发、在中间会师**的搜索优化策略。它的核心目标是：通过将一个大规模搜索问题拆分成两个小规模搜索，大幅压缩搜索空间。

双向搜索在算法竞赛中有两大应用分支：

- **双向 BFS**：在起点和终点都已知的最短路问题中，从两端同时做 BFS，在中间层相遇即得最优解。
- **Meet in the Middle（折半搜索）**：将 n 个元素拆成两半，分别暴力枚举所有可能，再将两半结果合并。用于子集枚举、方程求解等问题。

### ASCII Art：双向搜索的"会师"过程

```
单向搜索（从起点 → 终点）：

  S ──●──●──●──●──●──●──●──●──●── T
       ╰──── 搜索树层层展开 ────╯
       每层分支 k 个，深度 d → O(k^d) 个节点

双向搜索（从两端 → 中间会师）：

  S ──●──●──●──┐    ┌──●──●──●── T
       正向搜索  │    │  反向搜索
                ╰─🤝─╯
                 会师！
       每侧深度 d/2 → O(2 × k^{d/2}) 个节点
```

### 单向搜索 vs 双向搜索 复杂度对比

| 维度 | 单向搜索 | 双向搜索 |
|------|---------|---------|
| 搜索方向 | 从起点到终点 | 从两端向中间 |
| 搜索深度 | d | d/2 |
| 分支因子 k 时节点数 | O(k^d) | O(2 × k^{d/2}) |
| 典型场景 k=10, d=8 | 10^8 = 1亿 | 2 × 10^4 = 2万 |
| 子集枚举 n=40 | 2^40 ≈ 1万亿 | 2 × 2^20 ≈ 200万 |

一句话：**双向搜索的本质是"开根号"——把指数级复杂度砍半。**

## 二、核心思想与前置知识

### 2.1 搜索空间指数爆炸问题

很多搜索问题的状态空间随深度或元素个数**指数增长**：
- BFS 最短路：分支因子 k，深度 d → O(k^d)
- 子集枚举：n 个元素 → O(2^n)
- 多变量方程：每个变量 m 种取值，n 个变量 → O(m^n)

当 d、n 达到 30~45 左右时，单向搜索会超时。而双向搜索正好将指数减半，使问题可解。

### 2.2 双向 BFS 的原理与框架

**适用条件**：起点和终点都已知，求最短路径/最少步数。

**核心思路**：
1. 维护两个"前沿集合"：`fwd`（正向）和 `bwd`（反向）
2. 每轮选择**较小**的集合进行扩展（减少分支）
3. 扩展时检查新状态是否在对侧已访问过 → 若是，则会师成功

```cpp
// 双向 BFS 通用骨架
unordered_map<State, int> fwd_dist, bwd_dist;
// fwd_dist[s] = 0;  bwd_dist[t] = 0;
// fwd_set = {s};  bwd_set = {t};

int biBFS(State s, State t) {
    if (s == t) return 0;
    unordered_set<State> fwd_set = {s}, bwd_set = {t};
    fwd_dist[s] = 0; bwd_dist[t] = 0;
    int step = 0;
    while (!fwd_set.empty() && !bwd_set.empty()) {
        step++;
        // 总是扩展较小的一侧
        if (fwd_set.size() > bwd_set.size()) {
            swap(fwd_set, bwd_set);
            swap(fwd_dist, bwd_dist);
        }
        unordered_set<State> nxt;
        for (auto& cur : fwd_set) {
            for (auto& nb : getNeighbors(cur)) {
                if (fwd_dist.count(nb)) continue;  // 同侧已访问
                if (bwd_dist.count(nb)) return step + bwd_dist[nb]; // 会师
                fwd_dist[nb] = step;
                nxt.insert(nb);
            }
        }
        fwd_set = nxt;
    }
    return -1; // 不可达
}
```

### 2.3 Meet in the Middle（折半搜索）的原理与框架

**适用条件**：n 个元素（通常 n ≤ 40），需要枚举所有子集或组合，但 2^n 太大。

**核心思路**：
1. 将 n 个元素分成前半部分 A（n/2 个）和后半部分 B（n - n/2 个）
2. 分别枚举 A 和 B 的所有子集（各 O(2^{n/2})）
3. 将 A 的结果存入哈希表或排序数组
4. 遍历 B 的结果，在 A 中查找匹配/互补的部分

```cpp
// Meet in the Middle 通用骨架
vector<long long> leftSums, rightSums;

void enumSubsets(int arr[], int l, int r, long long sum, vector<long long>& res) {
    if (l > r) { res.push_back(sum); return; }
    enumSubsets(arr, l + 1, r, sum, res);           // 不选 arr[l]
    enumSubsets(arr, l + 1, r, sum + arr[l], res);  // 选 arr[l]
}

void solve(int a[], int n, long long target) {
    int mid = n / 2;
    enumSubsets(a, 0, mid - 1, 0, leftSums);
    enumSubsets(a, mid, n - 1, 0, rightSums);
    sort(leftSums.begin(), leftSums.end());
    // 遍历 rightSums，在 leftSums 中二分查找
    for (long long rs : rightSums) {
        // 根据问题类型：查找 target - rs、upper_bound、lower_bound 等
    }
}
```

### 2.4 复杂度分析：O(k^n) → O(k^{n/2})

| 问题类型 | 单向复杂度 | 双向复杂度 | 降幅 |
|---------|-----------|-----------|------|
| BFS（分支 k，深度 d）| O(k^d) | O(k^{d/2}) | 指数减半 |
| 子集枚举（n 个元素）| O(2^n) | O(2^{n/2} × log) | n=40: 万亿→百万 |
| 多变量方程（n 变量，m 取值）| O(m^n) | O(m^{n/2} × log) | 指数减半 |

**核心公式**：将搜索空间从 O(k^n) 降为 O(k^{n/2})，额外代价仅为合并时的排序/查找 O(log)。

## 三、双向搜索的子模型与分类详解

### 3.1 双向 BFS（起终点已知的最短路）

**适用信号**：
- 题目给定明确的**起始状态**和**目标状态**
- 求**最短路径**或**最少步数**
- 状态空间巨大但两端搜索可大幅剪枝

**实现要点**：
- 用两个 `unordered_set` 或 `unordered_map` 维护前沿
- **每轮扩展较小的一侧**，平衡两端搜索树大小
- 会师判定：新生成的状态已被对侧访问过

```cpp
// 双向 BFS 模板 —— 适用于状态为字符串/整数的最短路
unordered_map<string, int> distA, distB;

int biBFS(string src, string dst, unordered_set<string>& dead) {
    if (src == dst) return 0;
    if (dead.count(src) || dead.count(dst)) return -1;
    unordered_set<string> frontA = {src}, frontB = {dst};
    distA[src] = 0; distB[dst] = 0;
    int step = 0;
    while (!frontA.empty() && !frontB.empty()) {
        step++;
        if (frontA.size() > frontB.size()) {
            swap(frontA, frontB);
            swap(distA, distB);
        }
        unordered_set<string> nxt;
        for (auto& s : frontA) {
            for (auto& ns : expand(s)) {
                if (dead.count(ns) || distA.count(ns)) continue;
                if (distB.count(ns)) return step + distB[ns];
                distA[ns] = step;
                nxt.insert(ns);
            }
        }
        frontA = nxt;
    }
    return -1;
}
```

**适用信号总结**：起终点明确 + 最短路 + 状态空间大 → 双向 BFS。

### 3.2 Meet in the Middle - 子集枚举型（折半枚举后合并）

**适用信号**：
- n 个元素（20 ≤ n ≤ 45），需要枚举所有子集
- 求满足某个条件（和 ≤ target / 和 = target / 和 mod m 最大）的方案数或最优解
- 2^n 不可行，但 2^{n/2} 可行

**实现要点**：
- 将元素分两半，分别 DFS/位枚举所有子集和
- 一半排序，另一半遍历时二分查找
- 或用哈希表存一半的所有值和计数

```cpp
// 折半枚举 —— 子集和计数模板
long long a[45];
vector<long long> L, R;

void dfs(int l, int r, long long sum, long long limit, vector<long long>& res) {
    if (sum > limit) return;  // 剪枝
    if (l > r) { res.push_back(sum); return; }
    dfs(l + 1, r, sum, limit, res);
    dfs(l + 1, r, sum + a[l], limit, res);
}

long long solve(int n, long long m) {
    int mid = n / 2;
    dfs(0, mid - 1, 0, m, L);
    dfs(mid, n - 1, 0, m, R);
    sort(L.begin(), L.end());
    long long ans = 0;
    for (long long r : R)
        ans += upper_bound(L.begin(), L.end(), m - r) - L.begin();
    return ans;
}
```

**适用信号总结**：n ≤ 40~45 + 子集枚举 + 计数/最优 → 折半枚举。

### 3.3 Meet in the Middle - 路径拼接型

**适用信号**：
- 多变量方程求解，变量个数小（n ≤ 8~12），但每个变量取值范围大
- 将变量分两组，分别求出所有可能的"半方程值"
- 在两组结果中寻找互补对

**实现要点**：
- 前半 DFS 遍历所有取值组合，将求和结果存入哈希表
- 后半 DFS 遍历，查哈希表中是否存在互补值
- 适合 POJ 1186 方程的解数这类题目

```cpp
// 折半枚举 —— 方程求解模板
int k[10], p[10], n, m;
unordered_map<long long, int> mp;

long long power(int base, int exp) {
    long long res = 1;
    for (int i = 0; i < exp; i++) res *= base;
    return res;
}

void dfsLeft(int idx, int half, long long sum) {
    if (idx == half) { mp[sum]++; return; }
    for (int x = 1; x <= m; x++)
        dfsLeft(idx + 1, half, sum + k[idx] * power(x, p[idx]));
}

long long dfsRight(int idx, long long sum) {
    if (idx == n) return mp.count(-sum) ? mp[-sum] : 0;
    long long res = 0;
    for (int x = 1; x <= m; x++)
        res += dfsRight(idx + 1, sum + k[idx] * power(x, p[idx]));
    return res;
}
// 调用：dfsLeft(0, n/2, 0); ans = dfsRight(n/2, 0);
```

**适用信号总结**：多变量方程 + 变量少取值多 + 求解计数 → 路径拼接型折半。

### 模型对比总表

| 子模型 | 适用场景 | 拆分方式 | 合并方法 | 典型复杂度 |
|-------|---------|---------|---------|-----------|
| 双向 BFS | 起终点已知最短路 | 正向+反向 BFS | 会师判定 | O(k^{d/2}) |
| 折半枚举-子集型 | n≤45 子集和问题 | 前半+后半元素 | 排序+二分 | O(2^{n/2} log) |
| 折半枚举-路径拼接型 | 多变量方程求解 | 前半+后半变量 | 哈希表查找 | O(m^{n/2}) |

### 题目 1：打开转盘锁（LeetCode 752）

> 四个拨轮组成的锁，每个拨轮有 0-9 十个数字，初始为 "0000"。每次只能拨动一个轮一位（+1 或 -1，0/9 循环）。给定死锁列表 deadends 和目标 target，求最少拨动次数到达目标。

**分析**：状态空间 10^4 = 10000，每个状态有 8 个邻居。起点 "0000"，终点 target，典型的双向 BFS 场景。从两端同时搜索，中间会师。

```
状态定义：每个状态是一个 4 位字符串
转移方式：每个位 +1/-1（mod 10），产生 8 个邻居
搜索策略：双向 BFS，较小层优先扩展
终止条件：新状态出现在对侧已访问集合中
```

**C++ 代码**：

```cpp
int ans;

vector<string> getNext(const string& s) {
    vector<string> res;
    for (int i = 0; i < 4; i++) {
        string t = s;
        t[i] = (s[i] - '0' + 1) % 10 + '0'; res.push_back(t);
        t[i] = (s[i] - '0' + 9) % 10 + '0'; res.push_back(t);
    }
    return res;
}

int solve(vector<string>& deadends, string target) {
    unordered_set<string> dead(deadends.begin(), deadends.end());
    if (dead.count("0000") || dead.count(target)) return -1;
    if (target == "0000") return 0;
    unordered_set<string> fwd = {"0000"}, bwd = {target}, visited;
    visited.insert("0000"); visited.insert(target);
    int step = 0;
    while (!fwd.empty() && !bwd.empty()) {
        step++;
        if (fwd.size() > bwd.size()) swap(fwd, bwd);
        unordered_set<string> nxt;
        for (auto& s : fwd)
            for (auto& ns : getNext(s)) {
                if (dead.count(ns)) continue;
                if (bwd.count(ns)) return step;
                if (!visited.count(ns)) { visited.insert(ns); nxt.insert(ns); }
            }
        fwd = nxt;
    }
    return -1;
}
```

**核心启示**：
- **双向 BFS 入门范式**：用两个集合 fwd/bwd 交替扩展，swap 技巧使代码统一
- **较小层优先**：每轮扩展 size 较小的一侧，有效减少总扩展节点数
- **死锁预处理**：提前将 deadends 放入 set，O(1) 判断合法性

### 题目 2：单词接龙（LeetCode 127）

> 给定 beginWord 和 endWord，以及字典 wordList，每次变换一个字母，求从 beginWord 到 endWord 的最短变换序列长度。

**分析**：每个单词是一个状态，相邻状态之间只差一个字母。起终点已知，求最短路径长度，经典双向 BFS。

```
状态定义：字典中的每个单词
转移方式：改变一个字母后的单词如果在字典中则可达
搜索策略：双向 BFS，从 beginWord 和 endWord 同时扩展
答案位置：会师时的 step + 1（序列长度 = 步数 + 1）
```

**C++ 代码**：

```cpp
int solve(string beginWord, string endWord, vector<string>& wordList) {
    unordered_set<string> dict(wordList.begin(), wordList.end());
    if (!dict.count(endWord)) return 0;
    unordered_set<string> fwd = {beginWord}, bwd = {endWord};
    dict.erase(beginWord); dict.erase(endWord);
    int step = 1;
    while (!fwd.empty() && !bwd.empty()) {
        step++;
        if (fwd.size() > bwd.size()) swap(fwd, bwd);
        unordered_set<string> nxt;
        for (auto& w : fwd) {
            string t = w;
            for (int i = 0; i < (int)t.size(); i++) {
                char orig = t[i];
                for (char c = 'a'; c <= 'z'; c++) {
                    t[i] = c;
                    if (bwd.count(t)) return step;
                    if (dict.count(t)) { nxt.insert(t); dict.erase(t); }
                }
                t[i] = orig;
            }
        }
        fwd = nxt;
    }
    return 0;
}
```

**核心启示**：
- **字典去重技巧**：找到后立即从 dict 中删除，既判重又节省空间
- **字母替换枚举**：26×len 种邻居生成方式，比逐对比较更高效
- **双向 BFS 本质**：两侧"夹击"，搜索树高度减半，效率提升显著

### 题目 3：八数码难题（洛谷 P1379）

> 3×3 棋盘上有 1~8 和一个空格（用 0 表示），每次空格与相邻数字交换。给定初始状态，求变换到目标状态 `123804765` 的最少步数。

**分析**：状态总数 9! = 362880，起终点已知，适合双向 BFS。可用整数或字符串编码状态，用 map 记录距离和方向。

```
状态定义：棋盘的 9 位整数表示
转移方式：空格上下左右移动（4 个方向）
搜索策略：双向 BFS，从初始态和目标态同时扩展
答案位置：会师时两端距离之和 + 1
```

**C++ 代码**：

```cpp
const int GOAL = 123804765;
int dx[] = {1, -1, 0, 0}, dy[] = {0, 0, 1, -1};

int toBoard(int num, int b[3][3], int& zx, int& zy) {
    for (int i = 2; i >= 0; i--)
        for (int j = 2; j >= 0; j--) {
            b[i][j] = num % 10; num /= 10;
            if (!b[i][j]) { zx = i; zy = j; }
        }
    return 0;
}

int toNum(int b[3][3]) {
    int r = 0;
    for (int i = 0; i < 3; i++)
        for (int j = 0; j < 3; j++) r = r * 10 + b[i][j];
    return r;
}

int solve(int start) {
    if (start == GOAL) return 0;
    unordered_map<int, int> dist, side;
    queue<int> q;
    q.push(start); q.push(GOAL);
    dist[start] = 0; dist[GOAL] = 0;
    side[start] = 1; side[GOAL] = 2;
    while (!q.empty()) {
        int cur = q.front(); q.pop();
        int b[3][3], zx, zy;
        toBoard(cur, b, zx, zy);
        for (int d = 0; d < 4; d++) {
            int nx = zx + dx[d], ny = zy + dy[d];
            if (nx < 0 || nx >= 3 || ny < 0 || ny >= 3) continue;
            swap(b[zx][zy], b[nx][ny]);
            int nxt = toNum(b);
            if (side.count(nxt) && side[nxt] != side[cur])
                return dist[cur] + dist[nxt] + 1;
            if (!side.count(nxt)) {
                side[nxt] = side[cur];
                dist[nxt] = dist[cur] + 1;
                q.push(nxt);
            }
            swap(b[zx][zy], b[nx][ny]);
        }
    }
    return -1;
}
```

**核心启示**：
- **状态编码**：将 3×3 棋盘编码为 9 位整数，便于哈希和比较
- **方向标记**：用 side 标记状态来自正向(1)或反向(2)，遇到不同侧即会师
- **康托展开可选**：若需进一步优化空间，可用康托展开将状态映射到 0~362879

### 题目 4：单词接龙 II（LeetCode 126）

> 与 LeetCode 127 类似，但需要输出所有最短变换路径。

**分析**：在双向 BFS 的基础上增加路径记录。先用双向 BFS 建立每层的"父节点图"（DAG），再用 DFS 回溯所有路径。关键是 BFS 只记录层级关系，路径还原交给 DFS。

```
状态定义：字典中的每个单词
转移方式：改变一个字母后在字典中可达
搜索策略：双向 BFS 建图 + DFS 回溯路径
答案位置：所有从 beginWord 到 endWord 的最短路径
```

**C++ 代码**：

```cpp
unordered_map<string, vector<string>> parent; // 记录 DAG
vector<vector<string>> ans;
vector<string> path;

bool biBFS(string& src, string& dst, unordered_set<string>& dict) {
    unordered_set<string> fwd = {src}, bwd = {dst};
    bool found = false, reversed = false;
    while (!fwd.empty() && !found) {
        // 从字典中移除已访问的两端节点
        for (auto& w : fwd) dict.erase(w);
        for (auto& w : bwd) dict.erase(w);
        if (fwd.size() > bwd.size()) { swap(fwd, bwd); reversed = !reversed; }
        unordered_set<string> nxt;
        for (auto& w : fwd) {
            string t = w;
            for (int i = 0; i < (int)t.size(); i++) {
                char orig = t[i];
                for (char c = 'a'; c <= 'z'; c++) {
                    t[i] = c;
                    if (bwd.count(t)) { // 会师
                        found = true;
                        reversed ? parent[w].push_back(t) : parent[t].push_back(w);
                    } else if (dict.count(t)) {
                        nxt.insert(t);
                        reversed ? parent[w].push_back(t) : parent[t].push_back(w);
                    }
                }
                t[i] = orig;
            }
        }
        fwd = nxt;
    }
    return found;
}

void dfs(string& cur, string& dst) {
    path.push_back(cur);
    if (cur == dst) { ans.push_back(path); }
    else if (parent.count(cur))
        for (auto& p : parent[cur]) dfs(p, dst);
    path.pop_back();
}
// 调用：biBFS(beginWord, endWord, dict); dfs(endWord, beginWord); 然后反转每条路径
```

**核心启示**：
- **BFS 建图 + DFS 回溯**：BFS 保证最短性，DFS 枚举所有路径，职责分离
- **DAG 方向注意**：根据 reversed 标记决定 parent 的方向，确保图的正确性
- **逐层从字典删除**：每层扩展完毕才删除，保证同层节点可互相到达

### 题目 5：最小基因变化（LeetCode 433）

> 基因序列由 'A','C','G','T' 四个字符组成，长度为 8。给定起始基因 start、目标基因 end 和基因库 bank，每次变换一个字符，求最少变换次数。

**分析**：与 LeetCode 127 结构完全相同，只是字母表从 26 缩小到 4（A/C/G/T），状态空间更小。双向 BFS 即可高效解决。

```
状态定义：长度为 8 的基因字符串
转移方式：改变一个字符为 A/C/G/T 之一，且结果在 bank 中
搜索策略：双向 BFS
答案位置：会师时的步数
```

**C++ 代码**：

```cpp
int solve(string start, string end, vector<string>& bank) {
    unordered_set<string> dict(bank.begin(), bank.end());
    if (!dict.count(end)) return -1;
    if (start == end) return 0;
    unordered_set<string> fwd = {start}, bwd = {end};
    dict.erase(start); dict.erase(end);
    const string genes = "ACGT";
    int step = 0;
    while (!fwd.empty() && !bwd.empty()) {
        step++;
        if (fwd.size() > bwd.size()) swap(fwd, bwd);
        unordered_set<string> nxt;
        for (auto& g : fwd) {
            string t = g;
            for (int i = 0; i < 8; i++) {
                char orig = t[i];
                for (char c : genes) {
                    if (c == orig) continue;
                    t[i] = c;
                    if (bwd.count(t)) return step;
                    if (dict.count(t)) { nxt.insert(t); dict.erase(t); }
                }
                t[i] = orig;
            }
        }
        fwd = nxt;
    }
    return -1;
}
```

**核心启示**：
- **模式复用**：与单词接龙结构相同，只是字母表不同，代码高度相似
- **小字母表优势**：只有 4 种字符，每个位置仅 3 种变换，状态空间更紧凑
- **双向 BFS 通用性**：只要起终点明确 + 求最短路，都可直接套用此模板

### 题目 6：世界冰球锦标赛（洛谷 P4799）

> n 场比赛（n ≤ 40），每场票价 a_i，总预算 m。求有多少种买票方案（可以不买），使得总花费不超过 m。

**分析**：n ≤ 40，2^40 爆炸。折半枚举：将 n 场分两半，分别枚举所有子集和，左半排序后对右半每个和二分查找 ≤ m - right 的个数。

```
状态定义：左半/右半子集和列表
转移方式：每个元素选或不选
合并方法：排序 + upper_bound 二分
答案位置：所有合法 (left + right ≤ m) 的方案数之和
```

**C++ 代码**：

```cpp
long long a[45], m;
vector<long long> L, R;

void dfs(int l, int r, long long sum, vector<long long>& res) {
    if (sum > m) return;
    if (l > r) { res.push_back(sum); return; }
    dfs(l + 1, r, sum, res);
    dfs(l + 1, r, sum + a[l], res);
}

long long solve(int n) {
    int mid = n / 2;
    dfs(0, mid - 1, 0, L);
    dfs(mid, n - 1, 0, R);
    sort(L.begin(), L.end());
    long long ans = 0;
    for (long long r : R)
        ans += upper_bound(L.begin(), L.end(), m - r) - L.begin();
    return ans;
}
```

**核心启示**：
- **折半枚举入门范式**：拆两半 → 分别 DFS → 排序 → 二分合并，四步走
- **剪枝关键**：DFS 中 `sum > m` 立即返回，有效减少枚举量
- **upper_bound 妙用**：查找 ≤ m-r 的元素个数，恰好等于 upper_bound 返回的迭代器与 begin 的距离

### 题目 7：Lights（洛谷 P2962）

> n 盏灯（n ≤ 35）初始全灭，每个开关控制自身和若干相连的灯（按按钮取反），求最少按几次按钮使所有灯点亮。

**分析**：每个开关按 0 或 1 次（按两次等于没按）。n ≤ 35，2^35 ≈ 340 亿不可直接枚举，但折半后 2^18 ≈ 26 万可行。将开关分两半，前半枚举所有方案记录灯的状态和按钮次数，后半枚举后查找互补状态。

```
状态定义：长整型 bitmask 表示 n 盏灯的亮灭
转移方式：按第 i 个开关 → 异或 toggle[i]（i 自身及相连灯的 mask）
合并方法：哈希表存前半状态 → 后半查找互补
答案位置：使最终状态全 1 的最少按钮次数
```

**C++ 代码**：

```cpp
int n;
long long toggle[36]; // toggle[i] = 按第 i 个按钮影响的灯 bitmask
unordered_map<long long, int> mp; // 状态 → 最少按钮次数
int ans;

void dfsLeft(int idx, int half, long long state, int cnt) {
    if (idx == half) {
        if (!mp.count(state) || mp[state] > cnt)
            mp[state] = cnt;
        return;
    }
    dfsLeft(idx + 1, half, state, cnt);                    // 不按
    dfsLeft(idx + 1, half, state ^ toggle[idx], cnt + 1);  // 按
}

void dfsRight(int idx, long long state, int cnt) {
    if (cnt >= ans) return; // 剪枝
    if (idx == n) {
        long long need = ((1LL << n) - 1) ^ state; // 需要前半补全的状态
        if (mp.count(need)) ans = min(ans, cnt + mp[need]);
        return;
    }
    dfsRight(idx + 1, state, cnt);
    dfsRight(idx + 1, state ^ toggle[idx], cnt + 1);
}
// 调用：ans = n + 1; dfsLeft(0, n/2, 0, 0); dfsRight(n/2, 0, 0);
```

**核心启示**：
- **异或建模**：每个开关的效果用 bitmask 异或表示，天然适合折半
- **互补状态查找**：前半达到状态 S，后半需要达到 ALL ^ S，异或互补
- **剪枝优化**：后半 DFS 中 `cnt >= ans` 提前终止，避免无效搜索

### 题目 8：Meet in the Middle（CSES）

> 给定 n 个整数（n ≤ 40）和目标值 x，问有多少种方式选出若干个数使和恰好等于 x。

**分析**：标准折半枚举。与洛谷 P4799 类似，只是条件从"≤ m"变成"= x"。左半排序后对右半用 `equal_range` 或哈希表计数。

```
状态定义：左半/右半子集和列表
转移方式：每个元素选或不选
合并方法：排序 + lower_bound/upper_bound 计数
答案位置：满足 left + right = x 的方案数
```

**C++ 代码**：

```cpp
long long a[45], x;
vector<long long> L, R;

void dfs(int l, int r, long long sum, vector<long long>& res) {
    if (l > r) { res.push_back(sum); return; }
    dfs(l + 1, r, sum, res);
    dfs(l + 1, r, sum + a[l], res);
}

long long solve(int n) {

    for (long long r : R) {
        auto range = equal_range(L.begin(), L.end(), x - r);
        ans += range.second - range.first;
    }
    return ans;
}
```

**核心启示**：
- **equal_range 精准计数**：一次调用获取值域的 [lower, upper) 区间，直接得到计数
- **与 P4799 的对比**：≤ 用 upper_bound，= 用 equal_range，> 用 end - lower_bound
- **折半枚举的通用框架不变**：变的只是合并时的查找条件

### 题目 9：最接近目标值的子序列和（LeetCode 1755）

> 给定整数数组 nums（n ≤ 40）和目标值 goal，选出若干元素使和与 goal 的绝对差最小，返回最小绝对差。

**分析**：n ≤ 40 直接枚举不可行。折半枚举两半所有子集和，左半排序，对右半每个和在左半中二分查找最接近 goal - right 的值。

```
状态定义：左半/右半子集和列表
转移方式：每个元素选或不选
合并方法：排序 + lower_bound 找最接近值
答案位置：min(|left + right - goal|) 对所有组合
```

**C++ 代码**：

```cpp
vector<long long> L, R;
int ans;

void dfs(int nums[], int l, int r, long long sum, vector<long long>& res) {
    if (l > r) { res.push_back(sum); return; }
    dfs(nums, l + 1, r, sum, res);
    dfs(nums, l + 1, r, sum + nums[l], res);
}

int solve(int nums[], int n, int goal) {
    int mid = n / 2;
    dfs(nums, 0, mid - 1, 0, L);
    dfs(nums, mid, n - 1, 0, R);
    sort(L.begin(), L.end());
    ans = INT_MAX;
    for (long long r : R) {
        long long need = goal - r;
        auto it = lower_bound(L.begin(), L.end(), need);
        if (it != L.end()) ans = min(ans, (int)abs(*it + r - goal));
        if (it != L.begin()) { --it; ans = min(ans, (int)abs(*it + r - goal)); }
    }
    return ans;
}
```

**核心启示**：
- **二分逼近**：lower_bound 找到 ≥ need 的位置，前一个元素是 < need 的最大值，两者取最优
- **绝对差最小化**：检查 lower_bound 及其前驱，覆盖"偏大"和"偏小"两种情况
- **折半枚举万能性**：无论是计数、求和还是求最优，框架不变，只改合并逻辑

### 题目 10：方程的解数（POJ 1186）

> 给定方程 k1·x1^p1 + k2·x2^p2 + ... + kn·xn^pn = 0，每个 xi 取值 1~m，求满足方程的解的组数（n ≤ 6，m ≤ 150）。

**分析**：直接枚举 O(m^n) = 150^6 ≈ 10^13，不可行。折半：前 n/2 个变量枚举所有取值组合，将半方程和存入哈希表；后 n/2 个变量枚举，在哈希表中查找互补值。

```
状态定义：前半变量的所有可能求和值
转移方式：每个变量取值 1~m，计算 k_i * x_i^p_i
合并方法：哈希表查找 -sum
答案位置：互补对的计数之和
```

**C++ 代码**：

```cpp
int n, m, k[8], p[8];
unordered_map<long long, int> mp;
long long ans;

long long pw(int base, int exp) {
    long long r = 1;
    for (int i = 0; i < exp; i++) r *= base;
    return r;
}

void dfsL(int idx, int half, long long sum) {
    if (idx == half) { mp[sum]++; return; }
    for (int x = 1; x <= m; x++)
        dfsL(idx + 1, half, sum + (long long)k[idx] * pw(x, p[idx]));
}

void dfsR(int idx, long long sum) {
    if (idx == n) { if (mp.count(-sum)) ans += mp[-sum]; return; }
    for (int x = 1; x <= m; x++)
        dfsR(idx + 1, sum + (long long)k[idx] * pw(x, p[idx]));
}
// 调用：dfsL(0, n/2, 0); dfsR(n/2, 0);
```

**核心启示**：
- **方程分割**：将等式变形为 sumA = -sumB，即查找互补值
- **哈希表计数**：前半存 map[sum]++，后半查 map[-sum]，O(1) 合并
- **自定义幂函数**：避免浮点 pow 的精度问题，循环乘保证整数精确

### 题目 11：Balanced Cow Subsets（洛谷 P3067）

> n 头牛（n ≤ 20），每头牛产奶量 a_i。求有多少个非空子集可被分成两组，使两组产奶量之和相等。

**分析**：每头牛有三种状态：不选、分到 A 组（+a_i）、分到 B 组（-a_i）。总状态 3^n，n=20 时 3^20 ≈ 35 亿，但折半后 3^10 ≈ 6 万可行。用 bitmask 记录选了哪些牛，通过 sum 差值在两半之间匹配。

```
状态定义：(差值 sum, 选取 bitmask) 对
转移方式：每头牛 三种选择 → 三进制枚举
合并方法：哈希表按 sum 值分组，合并 bitmask
答案位置：去重后的非空子集数
```

**C++ 代码**：

```cpp
int n, a[21], ans;
unordered_map<int, vector<int>> mp; // sum → 选取 mask 列表
bool vis[1 << 20]; // 去重：每个子集只计一次

void dfsL(int idx, int half, int sum, int mask) {
    if (idx > half) { mp[sum].push_back(mask); return; }
    dfsL(idx + 1, half, sum, mask);                            // 不选
    dfsL(idx + 1, half, sum + a[idx], mask | (1 << idx));      // A 组
    dfsL(idx + 1, half, sum - a[idx], mask | (1 << idx));      // B 组
}

void dfsR(int idx, int sum, int mask) {
    if (idx > n) {
        if (!mp.count(-sum)) return;
        for (int lm : mp[-sum]) {
            int full = lm | mask;
            if (full && !vis[full]) { vis[full] = true; ans++; }
        }
        return;
    }
    dfsR(idx + 1, sum, mask);
    dfsR(idx + 1, sum + a[idx], mask | (1 << idx));
    dfsR(idx + 1, sum - a[idx], mask | (1 << idx));
}
// 调用：dfsL(1, n/2, 0, 0); dfsR(n/2 + 1, 0, 0);
```

**核心启示**：
- **三进制枚举**：每个元素有三种选择（不选/A 组/B 组），比二进制多一维
- **差值互补**：左半差值 sum 与右半差值 -sum 匹配时，说明两组和相等
- **bitmask 去重**：同一个子集可能被多种分组方式覆盖，vis 数组保证只计一次

### 🟣 挑战级 Level 5：折半搜索与高级数据结构

### 题目 12：Maximum Subsequence（CF 888E）

> 给定 n 个整数（n ≤ 35）和模数 m，选出若干数使其和对 m 取模的值最大。

**分析**：n ≤ 35，折半枚举所有子集和（均 mod m）。关键在于合并：对于右半的每个和 b，需要在左半找到最大的 a 使得 (a + b) mod m 最大。将左半排序，对每个 b 用二分查找 ≤ m-1-b 的最大值（使和不溢出 m）；若不存在则取左半最大值（和溢出后取模）。

```
状态定义：左半/右半所有子集和 mod m
转移方式：每个元素选或不选
合并方法：排序 + 二分查找最优互补
答案位置：max((left + right) % m)
```

**C++ 代码**：

```cpp
long long a[36];
int n, m;
vector<int> L, R;

void dfs(int l, int r, long long sum, vector<int>& res) {
    if (l > r) { res.push_back(sum % m); return; }
    dfs(l + 1, r, sum, res);
    dfs(l + 1, r, sum + a[l], res);
}

int solve() {
    int mid = n / 2;
    dfs(0, mid - 1, 0, L);
    dfs(mid, n - 1, 0, R);
    sort(L.begin(), L.end());
    L.erase(unique(L.begin(), L.end()), L.end());
    int ans = 0;
    for (int b : R) {
        int need = m - 1 - b;
        auto it = upper_bound(L.begin(), L.end(), need);
        if (it != L.begin()) {
            --it;
            ans = max(ans, b + *it); // 不溢出 m，直接相加
        }
        // 溢出情况：取 L 中最大值，和 mod m
        ans = max(ans, (b + L.back()) % m);
    }
    return ans;
}
```

**核心启示**：
- **模运算下的二分**：目标是 (a+b) mod m 最大，需要分"不溢出"和"溢出"两种情况
- **不溢出优先**：先找 a ≤ m-1-b 中最大的（使 a+b < m，值直接为 a+b）
- **溢出兜底**：若所有 a > m-1-b，取最大 a 让 (a+b) mod m 尽可能大
- **去重优化**：对 L 排序后 unique 去重，减少二分查找次数

### 🟡 追加：BFS 状态搜索与双向搜索

### 题目 13：Knights of Ni（洛谷 P5195 / USACO05DEC）

> W×H 的网格地图中，0 为空地、1 为障碍、2 为贝茜起点、3 为骑士位置（终点）、4 为灌木丛。贝茜需要先到达某个灌木丛格子拾取灌木，然后才能前往骑士处。只能上下左右移动，每步花费 1 天。求最少天数。

**分析**：经典的"两端 BFS + 中间汇合"思路。分别从起点和骑士位置各做一次 BFS，得到两张距离表 d1、d2。枚举每个灌木丛节点 p，答案 = min(d1[p] + d2[p])。注意从起点出发的 BFS 需将骑士格视为障碍（未持有灌木不可进入）。也可用三维状态 (x, y, 是否持有灌木) 的单次 BFS 解决，但两端各做一次 BFS 再在中间汇合更贴合"双向搜索"的核心思想。

```
状态定义：d1[x][y] = 起点到 (x,y) 的最短距离（骑士格不可进）
          d2[x][y] = 骑士到 (x,y) 的最短距离（无限制）
转移方式：上下左右四方向移动
合并方法：枚举灌木丛格 p，取 min(d1[p] + d2[p])
答案位置：所有灌木丛中合并距离的最小值
```

**C++ 代码**：

```cpp
const int MAXN = 1005;
int W, H, sx, sy, kx, ky;
int grid[MAXN][MAXN];
int d1[MAXN][MAXN], d2[MAXN][MAXN];
int dx[] = {0, 0, 1, -1}, dy[] = {1, -1, 0, 0};

void bfs(int sx, int sy, int dist[][MAXN], bool blockKnight) {
    memset(dist, -1, MAXN * MAXN * sizeof(int));
    queue<pair<int,int>> q;
    dist[sx][sy] = 0;
    q.push({sx, sy});
    while (!q.empty()) {
        auto [x, y] = q.front(); q.pop();
        for (int d = 0; d < 4; d++) {
            int nx = x + dx[d], ny = y + dy[d];
            if (nx < 1 || ny < 1 || nx > H || ny > W) continue;
            if (grid[nx][ny] == 1 || dist[nx][ny] != -1) continue;
            if (blockKnight && grid[nx][ny] == 3) continue;
            dist[nx][ny] = dist[x][y] + 1;
            q.push({nx, ny});
        }
    }
}

int solve() {
    bfs(sx, sy, d1, true);   // 起点出发，骑士格视为障碍
    bfs(kx, ky, d2, false);  // 骑士出发，无限制
    int ans = INT_MAX;
    for (int i = 1; i <= H; i++)
        for (int j = 1; j <= W; j++)
            if (grid[i][j] == 4 && d1[i][j] != -1 && d2[i][j] != -1)
                ans = min(ans, d1[i][j] + d2[i][j]);
    return ans;
}
```

**核心启示**：
- **两端 BFS 汇合**：从起点和终点各做一次 BFS，在中间节点（灌木丛）合并距离，是双向搜索的经典应用
- **条件性障碍处理**：骑士格只在未持灌木时不可进入，通过 blockKnight 参数控制两次 BFS 的行为差异
- **枚举中转点取最优**：答案 = min(d1[p] + d2[p])，枚举所有合法中转点即灌木丛格子
- **三维状态 BFS 替代方案**：(x, y, 0/1) 表示是否持有灌木，适合状态维度少、转移明确的场景

## 五、常见技巧与思路总结

### 5.1 双向 BFS 的"较小层优先扩展"策略

双向 BFS 中，每轮应扩展当前集合 **size 较小** 的一侧：

```
if (fwd.size() > bwd.size()) swap(fwd, bwd);
```

**为什么**：假设分支因子为 k。若一侧有 10 个节点，另一侧有 1000 个，扩展小侧新增 10k 个，扩展大侧新增 1000k 个。总是扩展小侧可以让两侧大小尽量均衡，总节点数最少。

### 5.2 Meet in the Middle 的合并方法

| 合并方法 | 适用条件 | 复杂度 | 代表题目 |
|---------|---------|--------|---------|
| 排序 + 二分 | 有序性查找（≤、=、最近）| O(N log N) | P4799、LeetCode 1755 |
| 哈希表 | 精确匹配（= 0、互补）| O(N) 期望 | POJ 1186、P2962 |
| 排序 + 双指针 | 对称逼近 | O(N) | 两数之和变体 |
| 排序 + upper_bound/equal_range | 计数问题 | O(N log N) | CSES Meet in the Middle |

**选择原则**：
- 需要 "= target"：哈希表或 equal_range
- 需要 "≤ target" 计数：排序 + upper_bound
- 需要 "最接近 target"：排序 + lower_bound（检查前后两个位置）
- 需要 "mod m 最大"：排序 + 分类讨论二分

### 5.3 如何判断是否适合双向搜索

**信号清单**：

| 信号 | 方法 |
|------|------|
| 起终点已知 + 最短路 + 状态空间大 | 双向 BFS |
| n ≤ 40~45 + 子集枚举/选或不选 | 折半枚举 |
| 多变量方程 + 变量少(≤8) + 取值多 | 折半枚举-路径拼接 |
| 每个元素 2~3 种选择 + n ≤ 20 | 折半枚举-三进制 |
| 题目明确说"n 最多 35~45" | 强烈暗示折半 |

**反信号（不适合双向搜索）**：
- n 很大（> 50）但有 DP 结构 → 用 DP
- 起点已知但终点未知 → 单向 BFS/DFS
- 状态之间有后效性 → 不适合 BFS

### 5.4 状态存储与哈希

双向搜索中，**状态的高效存储和查找**是性能关键：

```
// 字符串状态 → unordered_set<string>
unordered_set<string> visited;

// 整数状态 → unordered_map<int, int>
unordered_map<int, int> dist; // 状态 → 距离

// bitmask 状态 → 数组（最快）
bool vis[1 << 20]; // 状态数 ≤ 2^20 时

// 大整数状态 → 康托展开/自定义哈希
// 例如八数码：9! = 362880 种状态，可用数组
```

**性能排序**：数组 > unordered_map > map > 手写哈希（视场景而定）

## 七、终极总结与归纳

### 7.1 双向搜索的本质

**一句话：双向搜索的本质是"指数减半"——通过从两端搜索或将问题折半，把 O(k^n) 压缩为 O(k^{n/2})。**

这个思想的威力在于：指数级别的优化远超常数优化。当 n 从 40 减到 20 时，搜索空间从万亿级降到百万级，直接从"不可能"变成"轻松过"。双向搜索不改变问题本身，只改变了**搜索的方向和组织方式**。

### 7.2 核心要点

1. **双向 BFS**：起终点已知时，从两端同时搜索在中间会师，搜索深度减半
2. **较小层优先**：每轮扩展 size 较小的集合，平衡两侧搜索树
3. **折半枚举**：n ≤ 40~45 是强信号，拆两半分别暴力，合并时用二分/哈希
4. **合并是灵魂**：排序+二分（有序查找）、哈希表（精确匹配）、双指针（对称逼近）
5. **状态存储决定性能**：小状态用数组，中等用 unordered_map，大状态考虑编码压缩
6. **剪枝不可少**：即使折半后搜索量可控，合理剪枝（如 sum > limit）仍能提速数倍
