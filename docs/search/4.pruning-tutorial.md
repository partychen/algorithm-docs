## 一、什么是搜索剪枝

搜索剪枝，顾名思义，就是在搜索树上"剪掉"那些**不可能产生有效解**的分支。想象你在一棵大树前修剪枝叶——不需要的枝条越早剪掉，树就越精干，搜索效率就越高。

在回溯/DFS 搜索中，我们本质上是在遍历一棵**搜索树**（或搜索图）。暴力枚举会把每一条路都走到底，但很多路径在中途就已经注定不会产生合法解或更优解。**剪枝**就是在这些"注定无效"的节点处提前返回（return），让搜索跳过整棵子树。

一句话：**能不搜的就不搜，能早停的就早停。**

### 搜索树剪枝前后对比

```
【无剪枝的搜索树】                  【有剪枝的搜索树】

         (root)                            (root)
        / | \  \                          / |  \
       A  B  C  D                        A  B   ✂️
      /|  |  |\ |\                      /|  |
     1 2  3  4 5 6 7                   1  ✂️ 3
    /\        |                        /\
   x  y      z                       x  y
  ...       ...                     ...
  
  节点数：15+                        节点数：7
```

### 有剪枝 vs 无剪枝效率对比

| 维度 | 无剪枝（暴力） | 有剪枝 |
|------|---------------|--------|
| 搜索节点数 | 指数级（如 `2^n`、`n!`） | 大幅减少（视剪枝效果） |
| 时间复杂度 | 往往 TLE | 可在时限内通过 |
| 实现复杂度 | 最简单 | 略增加判断逻辑 |
| 正确性 | 保证正确 | **必须保证剪掉的都是无效分支** |
| 典型加速比 | 1x | 10x ~ 10000x+ |

> ⚠️ 剪枝最重要的前提：**剪掉的分支一定不包含有效解**。错误的剪枝会导致漏解！

## 二、核心思想与前置知识

### 2.1 搜索树的概念

搜索问题本质上是在一棵**隐式树**上进行遍历。每个节点代表一个"状态"，每条边代表一次"决策"。

```
         选第1个数
        /    |    \
       1     2     3
      / \    |
    选第2个 选第2个 选第2个
    /  \    |
   2    3   3
   |    |   |
  选第3个...
```

- **根节点**：初始状态（空集/起点）
- **中间节点**：部分解
- **叶子节点**：完整解（或不可行解）
- **搜索过程**：DFS / BFS 遍历这棵树

### 2.2 剪枝的数学本质

设搜索树的总节点数为 `T`，有效解的数量为 `S`。暴力搜索遍历所有 `T` 个节点，而剪枝的目标是只遍历 `T' ≪ T` 个节点，且保证 `S` 个有效解全部被找到。

**剪枝的核心不等式**：

```
设当前节点状态为 state，若能证明：
  以 state 为根的子树中不存在有效解
则：
  直接 return，跳过该子树的所有 T_sub 个节点
```

剪枝效果取决于**判断条件的强度**和**剪枝位置的深度**。越靠近根节点剪枝，跳过的子树越大，效果越好。

### 2.3 剪枝的通用插入位置（框架代码）

```cpp
int ans = INF;  // 或其他初始最优值

void dfs(int depth, int current_state) {
    // ① 可行性剪枝：当前状态已不满足约束
    if (!is_feasible(current_state)) return;
    
    // ② 最优性剪枝：当前状态不可能优于已知最优
    if (lower_bound(current_state) >= ans) return;
    
    // ③ 到达叶子/完整解：更新答案
    if (is_complete(depth)) {
        ans = min(ans, evaluate(current_state));
        return;
    }
    
    // ④ 枚举下一步决策（可先排序以优化搜索顺序）
    for (auto& choice : get_choices(depth, current_state)) {
        // ⑤ 对称性剪枝：跳过等价分支
        if (is_duplicate(choice)) continue;
        
        make_choice(choice);
        dfs(depth + 1, new_state);
        undo_choice(choice);
    }
}
```

> 框架中 ①~⑤ 是五个常见的剪枝插入位置，实际题目中通常只需要其中 1~3 个。

## 三、剪枝策略分类详解

### 3.1 可行性剪枝（Feasibility Pruning）

**核心思想**：当前状态已经违反了题目约束，继续搜索也不可能得到合法解，立即返回。

**适用信号**：题目有硬性约束（总和不超过 X、不能冲突、不能越界等）。

**模板代码**：

```cpp
void dfs(int depth, int sum, int target) {
    if (sum > target) return;  // 可行性剪枝：超限直接返回
    if (depth == n) {
        if (sum == target) ans++;
        return;
    }
    for (int i = start; i <= n; i++) {
        dfs(depth + 1, sum + a[i], target);
    }
}
```

**示例场景**：
- 组合总和中，当前和已超过 target
- 走迷宫中，当前位置越界或撞墙
- N 皇后中，当前列/对角线已被占用

---

### 3.2 最优性剪枝（Optimality Pruning / Bound Pruning）

**核心思想**：当前部分解的代价已经不优于已知最优解，继续搜索只会更差，提前终止。

**适用信号**：求最大/最小值问题，且搜索过程中代价单调变化。

**模板代码**：

```cpp
int ans = INF;

void dfs(int depth, int cost) {
    if (cost >= ans) return;  // 最优性剪枝：当前已不可能更优
    if (depth == n) {
        ans = min(ans, cost);
        return;
    }
    for (int i = 0; i < choices; i++) {
        dfs(depth + 1, cost + w[i]);
    }
}
```

**示例场景**：
- 旅行商问题中，当前路径长度已超最短距离
- 小猫爬山中，已用缆车数 ≥ 当前最优值

---

### 3.3 对称性剪枝（Symmetry Pruning / 去重剪枝）

**核心思想**：若多个分支实际上产生等价解（如排列去重、组合顺序限制），只保留其中一个分支。

**适用信号**：数组有重复元素、求组合（而非排列）、多个等价容器。

**模板代码（排列去重）**：

```cpp
// nums 已排序
bool used[MAXN];

void dfs(int depth) {
    if (depth == n) { record(); return; }
    for (int i = 0; i < n; i++) {
        if (used[i]) continue;
        // 对称性剪枝：相同元素只让第一个未用的先选
        if (i > 0 && nums[i] == nums[i-1] && !used[i-1]) continue;
        used[i] = true;
        dfs(depth + 1);
        used[i] = false;
    }
}
```

**示例场景**：
- 全排列 II（重复元素去重）
- 组合总和 II（同层不选相同值）
- k 个子集划分（等价桶去重）

---

### 3.4 排序优化剪枝（Ordering Pruning）

**核心思想**：通过对候选元素排序，使得搜索优先探索"更有希望"的分支，同时让可行性/最优性剪枝更早触发。

**适用信号**：候选元素有大小关系，且大/小的元素更容易触发剪枝条件。

**模板代码**：

```cpp
// 降序排序 → 大的先放 → 更快超限 → 更早剪枝
sort(nums, nums + n, greater<int>());

void dfs(int depth, int remain) {
    if (remain == 0) { ans++; return; }
    for (int i = start; i < n; i++) {
        if (nums[i] > remain) continue;  // 排序后，后面的更小，无需break
        dfs(depth + 1, remain - nums[i]);
    }
}
```

**升序排序配合 break**：

```cpp
sort(nums, nums + n);

void dfs(int start, int remain) {
    for (int i = start; i < n; i++) {
        if (nums[i] > remain) break;  // 升序，后面更大，全部超限
        dfs(i + 1, remain - nums[i]);
    }
}
```

---

### 3.5 奇偶性剪枝（Parity Pruning）

**核心思想**：利用棋盘染色或距离奇偶性，判断当前状态是否能在指定步数内到达目标。

**适用信号**：网格/棋盘问题，要求恰好走 k 步到达目标。

**核心公式**：

```
从 (sx, sy) 到 (tx, ty)，最短距离 dist = |sx-tx| + |sy-ty|
若剩余步数 remain 满足：
  remain < dist           → 走不到，剪枝
  (remain - dist) % 2 ≠ 0 → 奇偶不对，剪枝（因为绕路必须绕偶数步）
```

**模板代码**：

```cpp
void dfs(int x, int y, int steps) {
    int dist = abs(x - tx) + abs(y - ty);
    int remain = total_steps - steps;
    // 奇偶性剪枝
    if (remain < dist || (remain - dist) % 2 != 0) return;
    if (steps == total_steps) {
        if (x == tx && y == ty) found = true;
        return;
    }
    // 四方向搜索...
}
```

---

### 3.6 上下界估计剪枝（Bound Estimation Pruning）

**核心思想**：对当前状态的"未来最优可能"做一个乐观估计（下界/上界），若即使最乐观也不够好，则剪枝。

**适用信号**：组合数不够、体积/价值的上界估计、A* 中的启发函数。

**模板代码（组合数上界）**：

```cpp
// 从 n 个数中选 k 个，当前已选 cnt 个，从 start 开始
void dfs(int start, int cnt) {
    // 上界剪枝：剩余可选数量不够凑齐 k 个
    if (cnt + (n - start + 1) < k) return;
    if (cnt == k) { record(); return; }
    for (int i = start; i <= n; i++) {
        chosen[cnt] = i;
        dfs(i + 1, cnt + 1);
    }
}
```

**价值上界估计（类似分支定界）**：

```cpp
void dfs(int depth, int value) {
    // 上界剪枝：即使剩余全取最大值也不超过当前最优
    if (value + upper_bound(depth) <= ans) return;
    if (depth == n) { ans = max(ans, value); return; }
    // ...
}
```

### 六种剪枝策略对比总表

| 策略 | 核心判断 | 适用场景 | 典型题目 |
|------|---------|---------|---------|
| 可行性剪枝 | 违反约束 → return | 有硬性限制的问题 | 组合总和、N 皇后 |
| 最优性剪枝 | 当前 ≥ 最优 → return | 求最值问题 | 吃奶酪、小猫爬山 |
| 对称性剪枝 | 等价分支 → skip | 有重复元素/等价容器 | 全排列 II、k 子集 |
| 排序优化 | 排序后更快触发剪枝 | 候选有大小关系 | 组合总和、子集划分 |
| 奇偶性剪枝 | 奇偶不匹配 → return | 网格恰好 k 步问题 | HDU 1010 |
| 上下界估计 | 乐观估计仍不够 → return | 选取/路径上界 | 组合、八数码 A* |

### 题目 1：组合（LeetCode 77）

> 给定 `n` 和 `k`，返回 `[1, n]` 中所有 `k` 个数的组合。

**分析**：经典回溯问题。关键剪枝：如果从当前位置 `start` 到 `n`，剩余可选数量不够凑齐 `k` 个，直接返回。

```
剪枝条件：start + (k - cnt) - 1 > n 时直接 return
即：还需选 (k - cnt) 个，但从 start 到 n 只剩 (n - start + 1) 个
```

**C++ 代码**：

```cpp
int n, k;
vector<int> path;
vector<vector<int>> ans;

void dfs(int start, int cnt) {
    if (cnt + (n - start + 1) < k) return;  // 上界剪枝
    if (cnt == k) {
        ans.push_back(path);
        return;
    }
    for (int i = start; i <= n; i++) {
        path.push_back(i);
        dfs(i + 1, cnt + 1);
        path.pop_back();
    }
}
// 调用：dfs(1, 0)
```

**核心启示**：
- **"剩余不够选"是组合问题最基本的剪枝**，一行代码可减少大量无效递归
- **上界估计**是通用技巧：计算"最乐观情况"是否满足需求
- 此剪枝将复杂度从遍历所有子集降低到只遍历合法的组合路径

### 题目 2：组合总和（LeetCode 39）

> 给定无重复正整数数组 `candidates` 和目标值 `target`，找出所有和为 `target` 的组合（元素可重复选取）。

**分析**：先对数组升序排序。搜索过程中，若当前和加上 `candidates[i]` 已超过 `target`，由于后续元素更大，可直接 break。

```
剪枝条件：sum + candidates[i] > target → break（升序排序后）
```

**C++ 代码**：

```cpp
vector<int> cand;
int target;
vector<int> path;
vector<vector<int>> ans;

void dfs(int start, int sum) {
    if (sum == target) { ans.push_back(path); return; }
    for (int i = start; i < (int)cand.size(); i++) {
        if (sum + cand[i] > target) break;  // 排序+可行性剪枝
        path.push_back(cand[i]);
        dfs(i, sum + cand[i]);  // 可重复选，仍从 i 开始
        path.pop_back();
    }
}
// 前置：sort(cand.begin(), cand.end()); dfs(0, 0)
```

**核心启示**：
- **排序是可行性剪枝的最佳搭档**：升序排序后 break 一次即可跳过所有更大的候选
- **"可重复选取"对应 `dfs(i, ...)`**，"不可重复"对应 `dfs(i+1, ...)`
- 排序 + break 的组合在总和类问题中极为常见

### 题目 3：全排列问题（洛谷 P1706）

> 输出 `1` 到 `n` 的所有全排列，按字典序输出。

**分析**：经典回溯。使用 `used[]` 数组标记已选数字，按顺序枚举保证字典序。无重复元素，故剪枝仅为跳过已选。

```
剪枝条件：used[i] == true → continue（已选过的数字跳过）
```

**C++ 代码**：

```cpp
const int MAXN = 15;
int n, path[MAXN];
bool used[MAXN];

void dfs(int depth) {
    if (depth == n) {
        for (int i = 0; i < n; i++)
            printf("%5d", path[i]);
        puts("");
        return;
    }
    for (int i = 1; i <= n; i++) {
        if (used[i]) continue;  // 基本去重：跳过已选
        used[i] = true;
        path[depth] = i;
        dfs(depth + 1);
        used[i] = false;
    }
}
// 调用：dfs(0)
```

**核心启示**：
- **`used[]` 数组是回溯去重的基石**，几乎所有排列问题都需要它
- 按顺序枚举 `1~n` 天然保证字典序
- 这是后续复杂排列剪枝的出发点，理解它才能理解"排列去重"

### 题目 4：全排列 II（LeetCode 47）

> 给定一个**可包含重复数字**的序列 `nums`，返回所有**不重复的**全排列。

**分析**：先排序，然后在同一层递归中，若当前数字与前一个相同且前一个未被使用，则跳过——这是排列去重的经典条件。

```
剪枝条件：nums[i] == nums[i-1] && !used[i-1] → continue
含义：同层中相同元素只让排在前面的先被选取，避免生成重复排列
```

**C++ 代码**：

```cpp
int n;
vector<int> nums, path;
vector<vector<int>> ans;
bool used[25];

void dfs(int depth) {
    if (depth == n) { ans.push_back(path); return; }
    for (int i = 0; i < n; i++) {
        if (used[i]) continue;
        if (i > 0 && nums[i] == nums[i-1] && !used[i-1]) continue;  // 去重剪枝
        used[i] = true;
        path.push_back(nums[i]);
        dfs(depth + 1);
        path.pop_back();
        used[i] = false;
    }
}
// 前置：sort(nums.begin(), nums.end()); dfs(0)
```

**核心启示**：
- **排列去重公式 `nums[i]==nums[i-1] && !used[i-1]`** 是面试和竞赛的高频考点
- 这里的逻辑是：相同值的元素，强制按**原始顺序**使用，打破对称性
- 必须先排序才能使用此剪枝

### 题目 5：组合总和 II（LeetCode 40）

> 给定候选数组 `candidates`（有重复）和目标值 `target`，找出所有和为 `target` 的组合，每个数字只能用一次。

**分析**：三重剪枝组合拳——排序 + 可行性剪枝（超限 break）+ 同层去重（跳过连续相同值）。

```
剪枝1：sum + candidates[i] > target → break
剪枝2：i > start && candidates[i] == candidates[i-1] → continue（同层去重）
```

**C++ 代码**：

```cpp
vector<int> cand, path;
vector<vector<int>> ans;
int target;

void dfs(int start, int sum) {
    if (sum == target) { ans.push_back(path); return; }
    for (int i = start; i < (int)cand.size(); i++) {
        if (sum + cand[i] > target) break;  // 可行性剪枝
        if (i > start && cand[i] == cand[i-1]) continue;  // 同层去重
        path.push_back(cand[i]);
        dfs(i + 1, sum + cand[i]);
        path.pop_back();
    }
}
// 前置：sort(cand.begin(), cand.end()); dfs(0, 0)
```

**核心启示**：
- **排序 + 可行性 + 去重**三件套是组合类问题的标配
- `i > start`（而非 `i > 0`）确保只在**同一层**去重，不同层可以选相同值
- 理解"同层"和"不同层"的区别是掌握组合去重的关键

### 题目 6：吃奶酪（洛谷 P1433）

> 平面上有 `n` 块奶酪（`n ≤ 15`），从原点出发，求经过所有奶酪的最短路径长度。

**分析**：本质是小规模 TSP。DFS 枚举访问顺序，用最优性剪枝：当前路径长度已超过全局最优解时立即返回。

```
剪枝条件：cur_dist >= ans → return（最优性剪枝）
```

**C++ 代码**：

```cpp
const int MAXN = 20;
int n;
double x[MAXN], y[MAXN], dist[MAXN][MAXN];
double ans = 1e18;
bool vis[MAXN];

double calc(int a, int b) {
    return sqrt((x[a]-x[b])*(x[a]-x[b]) + (y[a]-y[b])*(y[a]-y[b]));
}

void dfs(int cur, int cnt, double d) {
    if (d >= ans) return;  // 最优性剪枝
    if (cnt == n) { ans = min(ans, d); return; }
    for (int i = 1; i <= n; i++) {
        if (vis[i]) continue;
        vis[i] = true;
        dfs(i, cnt + 1, d + dist[cur][i]);
        vis[i] = false;
    }
}
// 前置：预计算 dist[i][j]; 0 号点为原点; dfs(0, 0, 0.0)
```

**核心启示**：
- **最优性剪枝是求最值搜索的必备武器**，一行代码可能将超时变为通过
- 全局变量 `ans` 持续更新，搜索越深信息越多，剪枝越有效
- 对于 `n ≤ 15` 的 TSP，DFS + 最优性剪枝已可通过，更优方案是状态压缩 DP

### 题目 7：Tempter of the Bone（HDU 1010）

> 一个 `N×M` 的迷宫，从起点 `S` 出发，恰好在第 `T` 步到达终点 `D`。判断是否可行。

**分析**：经典的奇偶性剪枝。从当前位置到终点的曼哈顿距离 `dist`，若 `(T - steps - dist)` 为奇数，则不可能恰好到达（因为绕路必须绕偶数步）。

```
剪枝条件：
  remain = T - steps
  dist = |x-tx| + |y-ty|
  若 remain < dist 或 (remain - dist) % 2 != 0 → return
```

**C++ 代码**：

```cpp
int n, m, T;
int sx, sy, tx, ty;
char grid[10][10];
bool found;

void dfs(int x, int y, int steps) {
    if (found) return;
    int remain = T - steps;
    int dist = abs(x - tx) + abs(y - ty);
    if (remain < dist || (remain - dist) % 2 != 0) return;  // 奇偶性剪枝
    if (x == tx && y == ty && steps == T) { found = true; return; }
    int dx[] = {0,0,1,-1}, dy[] = {1,-1,0,0};
    for (int d = 0; d < 4; d++) {
        int nx = x + dx[d], ny = y + dy[d];
        if (nx < 0 || nx >= n || ny < 0 || ny >= m) continue;
        if (grid[nx][ny] == 'X') continue;
        grid[x][y] = 'X';  // 标记已访问
        dfs(nx, ny, steps + 1);
        grid[x][y] = '.';
    }
}
// 调用：found = false; dfs(sx, sy, 0)
```

**核心启示**：
- **奇偶性剪枝是网格恰好 k 步问题的"必杀技"**，不加则必 TLE
- 棋盘染色的本质：黑白格交替走，绕路必然绕偶数步
- 此技巧可推广到所有"恰好到达"类问题

### 题目 8：N 皇后（LeetCode 51）

> 在 `n×n` 棋盘上放置 `n` 个皇后，使它们互不攻击（不同行、列、对角线）。输出所有方案。

**分析**：逐行放置皇后，用三个数组记录列、主对角线、副对角线的占用情况。每次尝试放置时检查三个约束，不满足则跳过（可行性剪枝）。

```
剪枝条件：
  col[j] || diag1[row-j+n] || diag2[row+j] → continue
  列冲突 或 对角线冲突 → 跳过该列
```

**C++ 代码**：

```cpp
const int MAXN = 20;
int n, queen[MAXN];
bool col[MAXN], diag1[2*MAXN], diag2[2*MAXN];
vector<vector<string>> ans;

void dfs(int row) {
    if (row == n) {
        vector<string> board(n, string(n, '.'));
        for (int i = 0; i < n; i++) board[i][queen[i]] = 'Q';
        ans.push_back(board);
        return;
    }
    for (int j = 0; j < n; j++) {
        if (col[j] || diag1[row-j+n] || diag2[row+j]) continue;  // 三维剪枝
        queen[row] = j;
        col[j] = diag1[row-j+n] = diag2[row+j] = true;
        dfs(row + 1);
        col[j] = diag1[row-j+n] = diag2[row+j] = false;
    }
}
// 调用：dfs(0)
```

**核心启示**：
- **用布尔数组 O(1) 判断冲突**，避免每次 O(n) 遍历检查
- 对角线编码技巧：主对角线 `row-col+n`，副对角线 `row+col`
- 逐行搜索天然保证不同行，只需检查列和对角线

### 题目 9：小猫爬山（洛谷 P5194 / 经典问题）

> `n` 只小猫（`n ≤ 18`），每只有重量 `w[i]`。一辆缆车最大载重 `W`。求最少需要几辆缆车。

**分析**：DFS 尝试将每只猫放入已有缆车或新开一辆。两个关键剪枝：①最优性——当前缆车数 ≥ 已知最优，return。②排序优化——先放重猫，更快填满缆车触发剪枝。

```
剪枝1：k >= ans → return（最优性剪枝）
剪枝2：降序排序重量（排序优化）
```

**C++ 代码**：

```cpp
const int MAXN = 20;
int n, W, w[MAXN], cab[MAXN];
int ans;

void dfs(int i, int k) {
    if (k >= ans) return;  // 最优性剪枝
    if (i == n) { ans = k; return; }
    for (int j = 0; j < k; j++) {  // 尝试放入已有缆车
        if (cab[j] + w[i] <= W) {
            cab[j] += w[i];
            dfs(i + 1, k);
            cab[j] -= w[i];
        }
    }
    cab[k] = w[i];  // 新开一辆缆车
    dfs(i + 1, k + 1);
    cab[k] = 0;
}
// 前置：降序排序 w[]; ans = n; dfs(0, 0)
```

**核心启示**：
- **"装箱"类问题用贪心 + DFS + 最优性剪枝**效果极佳
- **先放大的再放小的**是排序优化的经典应用
- `ans = n` 是一个合理的初始上界（每只猫单独一车）

### 题目 10：划分为 k 个相等的子集（LeetCode 698）

> 给定整数数组 `nums` 和整数 `k`，判断是否能将数组分成 `k` 个元素和相等的子集。

**分析**：多种剪枝的组合大戏——①前置检查（总和不整除直接 false）②排序优化（降序排列）③桶容量剪枝（超限 skip）④等价桶去重（空桶等价，只试一个）⑤提前终止（装满 k-1 个即可）。

```
剪枝1：sum % k != 0 → false
剪枝2：nums 降序排序
剪枝3：bucket[j] + nums[i] > target → continue
剪枝4：bucket[j] == bucket[j-1] → continue（等价桶去重）
剪枝5：k == 1 → true（剩余必定合法）
```

**C++ 代码**：

```cpp
int n, k, target;
int nums[20], bucket[20];

bool dfs(int idx) {
    if (idx == n) return true;  // 所有数字分配完毕
    for (int j = 0; j < k; j++) {
        if (bucket[j] + nums[idx] > target) continue;  // 容量剪枝
        if (j > 0 && bucket[j] == bucket[j-1]) continue;  // 等价桶去重
        bucket[j] += nums[idx];
        if (dfs(idx + 1)) return true;
        bucket[j] -= nums[idx];
        if (bucket[j] == 0) break;  // 空桶放不下→后续空桶也不行
    }
    return false;
}
// 前置：降序排序 nums[]; target = sum/k; dfs(0)
```

**核心启示**：
- **多种剪枝组合是高阶搜索题的关键**，任一剪枝缺失都可能 TLE
- **等价桶去重**极易被忽略，但效果惊人（避免 k! 级别重复）
- `bucket[j]==0` 时 break 的含义：第一个空桶都装不下，后面的空桶也一样

### 题目 11：数字三角形（洛谷 P1118）

> 给定 `n` 和 `S`，要求找到 `1~n` 的一个排列，使其构建的杨辉三角形最底层数字加权和恰好等于 `S`。输出字典序最小的排列。

**分析**：第一行排列 `a[1..n]`，最终和为 `∑ C(n-1, i) * a[i+1]`。DFS 枚举排列，利用部分和进行最优性剪枝：若当前已算出的部分和已超过 `S`，或即使剩余位置全取最大值也不够 `S`，均可剪枝。

```
剪枝条件：
  partial_sum > S → return（上界剪枝）
  partial_sum + max_possible < S → return（下界剪枝）
```

**C++ 代码**：

```cpp
const int MAXN = 15;
int n, S, perm[MAXN], C[MAXN];
bool used[MAXN], found;

void dfs(int depth, int sum) {
    if (found) return;
    if (sum > S) return;  // 上界剪枝：部分和已超
    if (depth == n) {
        if (sum == S) {
            found = true;
            for (int i = 0; i < n; i++)
                printf("%d%c", perm[i], i==n-1?'\n':' ');
        }
        return;
    }
    for (int i = 1; i <= n; i++) {
        if (used[i]) continue;
        used[i] = true;
        perm[depth] = i;
        dfs(depth + 1, sum + C[depth] * i);
        used[i] = false;
        if (found) return;
    }
}
// 前置：预计算 C[i] = C(n-1, i)（杨辉三角系数）; dfs(0, 0)
```

**核心启示**：
- **预计算组合数系数**将"模拟过程"转化为"加权求和"，大幅简化问题
- **上界/下界双重剪枝**比单一剪枝效果好得多
- 找字典序最小解时，按 `1~n` 顺序枚举 + 找到立刻返回

### 题目 12：解数独（LeetCode 37）

> 填充一个 9×9 的数独，使每行、每列、每个 3×3 宫内数字 1-9 各出现一次。

**分析**：约束传播 + 回溯剪枝。用三个布尔数组 `row/col/box` 记录已用数字，O(1) 判断合法性。进一步优化：优先填写候选数最少的空格（MRV 启发式）。

```
剪枝条件：
  row[r][d] || col[c][d] || box[r/3*3+c/3][d] → skip
  即：该数字在所在行/列/宫已存在
```

**C++ 代码**：

```cpp
bool row[9][10], col[9][10], box[9][10];
char board[9][9];
bool solved;

void dfs(int pos) {
    // 跳过非空格
    while (pos < 81 && board[pos/9][pos%9] != '.') pos++;
    if (pos == 81) { solved = true; return; }
    int r = pos / 9, c = pos % 9, b = r/3*3 + c/3;
    for (int d = 1; d <= 9; d++) {
        if (row[r][d] || col[c][d] || box[b][d]) continue;  // 约束剪枝
        board[r][c] = '0' + d;
        row[r][d] = col[c][d] = box[b][d] = true;
        dfs(pos + 1);
        if (solved) return;
        row[r][d] = col[c][d] = box[b][d] = false;
        board[r][c] = '.';
    }
}
// 前置：初始化 row/col/box; dfs(0)
```

**核心启示**：
- **布尔数组实现 O(1) 约束检查**是数独类问题的标配
- **MRV 启发式**（选候选最少的格子先填）可以进一步大幅减少搜索量
- 约束传播本质上就是将"可行性剪枝"做到极致

### 题目 13：靶形数独（洛谷 P1074）

> 在标准数独基础上，每个格有得分权重（同心圆越内圈分越高），求填满后的最大总得分。

**分析**：在数独基础上，需要极致的剪枝才能通过。核心策略：①位运算记录行/列/宫可用数字集合②优先填候选数最少的空格（MRV）③已知最大分数的最优性剪枝。

```
剪枝条件：
  avail = ~(row[r] | col[c] | box[b]) & 0x1FF  // 位运算求可用集合
  若 avail == 0 → return（无候选数可填）
  优先选 popcount(avail) 最小的格子
```

**C++ 代码**：

```cpp
int row[9], col[9], box[9];  // 位掩码记录已用数字
int score[9][9];              // 每格的得分权重
int ans = -1;

int box_id(int r, int c) { return r/3*3 + c/3; }

void dfs(int remain, int cur_score) {
    if (remain == 0) { ans = max(ans, cur_score); return; }
    // MRV：找候选最少的空格
    int best_r = -1, best_c = -1, min_cnt = 10;
    for (int r = 0; r < 9; r++)
        for (int c = 0; c < 9; c++)
            if (!((row[r]>>c) & 1) /* 格子为空的判断需根据实际数据结构 */) {
                int avail = (~(row[r]|col[c]|box[box_id(r,c)])) & 0x1FF;
                int cnt = __builtin_popcount(avail);
                if (cnt < min_cnt) { min_cnt = cnt; best_r = r; best_c = c; }
            }
    if (min_cnt == 0) return;  // 无解分支
    int r = best_r, c = best_c, b = box_id(r,c);
    int avail = (~(row[r]|col[c]|box[b])) & 0x1FF;
    for (int bits = avail; bits; bits &= bits-1) {
        int d = __builtin_ctz(bits & -bits);  // 取最低位对应数字
        row[r] |= (1<<d); col[c] |= (1<<d); box[b] |= (1<<d);
        dfs(remain - 1, cur_score + (d+1) * score[r][c]);
        row[r] ^= (1<<d); col[c] ^= (1<<d); box[b] ^= (1<<d);
    }
}
// 前置：初始化 row/col/box 掩码; 计算 remain（空格数）; dfs(remain, init_score)
```

**核心启示**：
- **位运算记录状态**比布尔数组更高效（一次按位或即可合并约束）
- **MRV 是约束满足问题中最强的搜索顺序优化**
- 极致剪枝 = 快速约束检查 + 智能搜索顺序 + 最优性剪枝

### 题目 14：正方形数组的数目（LeetCode 996）

> 给定数组 `A`，求所有排列中相邻元素之和为完全平方数的排列数（去重）。

**分析**：排列回溯 + 两重剪枝：①相邻和不为完全平方数 → 可行性剪枝；②排序后去重 → 对称性剪枝。

```
剪枝1：prev + A[i] 不是完全平方数 → continue
剪枝2：i > 0 && A[i] == A[i-1] && !used[i-1] → continue（去重）
```

**C++ 代码**：

```cpp
int n, ans;
int A[15];
bool used[15];

bool is_square(int x) {
    int r = (int)sqrt(x);
    return r * r == x;
}

void dfs(int depth, int prev) {
    if (depth == n) { ans++; return; }
    for (int i = 0; i < n; i++) {
        if (used[i]) continue;
        if (i > 0 && A[i] == A[i-1] && !used[i-1]) continue;  // 对称性去重
        if (depth > 0 && !is_square(prev + A[i])) continue;  // 可行性剪枝
        used[i] = true;
        dfs(depth + 1, A[i]);
        used[i] = false;
    }
}
// 前置：sort(A, A+n); ans = 0; dfs(0, -1)
```

**核心启示**：
- **对称性剪枝 + 可行性剪枝组合**是处理"约束全排列"问题的经典模式
- 排列去重与题目 4 思路一致，但这里额外加了相邻约束
- 完全平方数判定可以预处理加速，但 `sqrt` 在此规模下已足够

### 题目 15：八数码难题（洛谷 P1379）

> 3×3 的棋盘上有 1-8 的数字和一个空格，通过滑动空格使其恢复到目标状态。求最少步数。

**分析**：经典的 A* / IDA* 问题。核心剪枝：使用**曼哈顿距离作为估价函数**（h），如果 `g + h > 当前深度限制`，则剪枝。IDA* 中用迭代加深控制搜索深度，配合估价函数大幅减少搜索量。

```
估价函数：h = Σ |cur_r[i] - goal_r[i]| + |cur_c[i] - goal_c[i]|（各数字的曼哈顿距离之和）
剪枝条件：g + h > depth_limit → return（IDA* 剪枝）
附加剪枝：不走回头路（排除上一步的逆方向）
```

**C++ 代码**：

```cpp
int goal_r[9] = {2,0,0,0,1,1,1,2,2};  // 数字 0-8 在目标状态的行
int goal_c[9] = {2,0,1,2,0,1,2,0,1};  // 数字 0-8 在目标状态的列
int board[3][3], depth_limit;
bool solved;
int dx[] = {0,0,1,-1}, dy[] = {1,-1,0,0};

int h() {
    int sum = 0;
    for (int i = 0; i < 3; i++)
        for (int j = 0; j < 3; j++)
            if (board[i][j])
                sum += abs(i - goal_r[board[i][j]]) + abs(j - goal_c[board[i][j]]);
    return sum;
}

void dfs(int g, int bx, int by, int last_dir) {
    int est = h();
    if (est == 0) { solved = true; return; }
    if (g + est > depth_limit) return;  // IDA* 剪枝
    for (int d = 0; d < 4; d++) {
        if (d == (last_dir ^ 1) && last_dir >= 0) continue;  // 不走回头路
        int nx = bx + dx[d], ny = by + dy[d];
        if (nx < 0 || nx >= 3 || ny < 0 || ny >= 3) continue;
        swap(board[bx][by], board[nx][ny]);
        dfs(g + 1, nx, ny, d);
        swap(board[bx][by], board[nx][ny]);
        if (solved) return;
    }
}

// IDA* 主循环
void ida_star(int bx, int by) {
    for (depth_limit = h(); depth_limit <= 80; depth_limit++) {
        solved = false;
        dfs(0, bx, by, -1);
        if (solved) break;
    }
}
// depth_limit 即为最少步数
```

**核心启示**：
- **IDA* = 迭代加深 + 估价函数剪枝**，是滑块/华容道类问题的最优算法
- **曼哈顿距离是八数码问题的标准估价函数**，可证明其可容性（不会高估）
- **排除反方向**的剪枝看似简单，却可以减少约 1/4 的搜索量
- IDA* 相比 BFS 节省大量内存，适合状态空间大但解深度有限的问题

### 题目 16：小木棍 ［数据加强版］（洛谷 P1120）

> 乔治有一些同样长的木棍，他把这些木棍随意砍成若干段小木棍（每段长度 ≤ 50）。现在他想把小木棍拼接回原来的样子，但忘记了原始木棍的数量和长度。给出每段小木棍的长度，求原始木棍的最小可能长度。

**分析**：枚举原始长度 `L`（从最大小段到总和），验证所有小段能否恰好拼成 `sum/L` 根长度为 `L` 的木棍。这是一道需要**多种剪枝策略组合**才能通过的极致剪枝经典题。核心剪枝包括：排序优化（降序优先用长段）、对称性去重（同长段跳过）、可行性剪枝（首段/末段失败立即返回）以及搜索起点优化（组合无序性）。

```
枚举策略：L 从 max(a[i]) 到 sum 枚举，且 sum % L == 0
剪枝1（排序优化）：将小段降序排序，优先使用长段减少分支
剪枝2（同长去重）：若当前长度的段失败，跳过后续所有同长段
剪枝3（首段失败）：拼新木棍时第一段就失败 → 该 L 无解，直接返回
剪枝4（末段失败）：恰好能拼满但后续失败 → 换段也无济于事，返回
剪枝5（搜索起点）：同一根木棍内从上次选择位置继续枚举，避免重复组合
```

**C++ 代码**：

```cpp
int a[70], n, target, sum_all;
bool used[70];

bool dfs(int stick, int cur_len, int start) {
    // stick: 已拼好的完整木棍数, cur_len: 当前木棍已拼长度
    if (stick == sum_all / target) return true;          // 全部拼完
    if (cur_len == target) return dfs(stick + 1, 0, 0);  // 当前拼完，开始下根

    for (int i = start; i < n; i++) {                    // 剪枝5：从 start 枚举
        if (used[i]) continue;
        if (cur_len + a[i] > target) continue;           // 可行性剪枝
        if (i > 0 && a[i] == a[i-1] && !used[i-1])
            continue;                                     // 剪枝2：同长去重

        used[i] = true;
        if (dfs(stick, cur_len + a[i], i + 1)) return true;
        used[i] = false;

        if (cur_len == 0) return false;                  // 剪枝3：首段失败
        if (cur_len + a[i] == target) return false;      // 剪枝4：末段失败
    }
    return false;
}

// 主逻辑：
// sort(a, a+n, greater<int>());        // 剪枝1：降序排序
// for (target = a[0]; target <= sum_all; target++) {
//     if (sum_all % target != 0) continue;
//     memset(used, false, sizeof(used));
//     if (dfs(0, 0, 0)) { printf("%d\n", target); break; }
// }
```

**核心启示**：
- **五重剪枝缺一不可**：排序、去重、首段、末段、起点优化共同作用才能通过加强数据
- **"首段失败"剪枝的本质**：如果拼新木棍时最先选的段就走不通，说明这段无论放在哪根木棍里都会被最先尝试，注定失败
- **"末段失败"剪枝的本质**：恰好拼满后续却失败，说明是后面的组合有问题，更换当前段只会更短更不利
- 此题是搜索剪枝综合训练的**必做题**，涵盖了几乎所有常见剪枝思想

### 题目 17：Mayan 游戏（洛谷 P1312）

> 有一个 5 列 7 行的棋盘，某些格子上有带颜色的方块。每一步可以选择一个方块向左或向右移动一格（对方有方块则交换，无方块则移过去并下落）。每步操作后，连续 3 个及以上同色方块（横或竖）同时消除，消除后下落并可链式消除。要求在恰好 `n` 步（n ≤ 5）内消除所有方块，输出字典序最小的操作方案；无解输出 `-1`。

**分析**：步数极少（≤5），可 DFS 暴力枚举所有操作方案，但需精心模拟消除-下落过程，并加入合理剪枝。关键点：①按列优先、行次之、方向（右优先于左）的顺序枚举保证字典序最小；②若一个方块与右邻颜色相同则跳过向左交换（等价操作去重）；③消除后某颜色剩余不足 3 个但不为 0 则必定无解，直接剪枝。

```
搜索顺序：x(列) 从 0→4, y(行) 从 0→6, 方向先右(+1)再左(-1)，保证字典序
剪枝1（等价去重）：交换两同色方块无意义 → 跳过
剪枝2（方向限制）：左移时左边有方块才需尝试（右移已覆盖交换情况）
剪枝3（颜色计数）：消除后某颜色剩余 ∈ [1,2] → 永远无法消除 → 剪枝
剪枝4（空格跳过）：当前位置无方块 → 跳过
```

**C++ 代码**：

```cpp
int g[5][7];  // g[x][y]: 第x列第y行的颜色，0 表示空
struct Move { int x, y, dir; };
Move ans[5];
int n;

void fall() {
    for (int x = 0; x < 5; x++) {
        int ptr = 0;
        for (int y = 0; y < 7; y++)
            if (g[x][y]) g[x][ptr++] = g[x][y];
        while (ptr < 7) g[x][ptr++] = 0;
    }
}

bool clear() {
    bool mark[5][7] = {};
    bool cleared = false;
    for (int x = 0; x < 5; x++)
        for (int y = 0; y < 7; y++) {
            if (!g[x][y]) continue;
            if (x + 2 < 5 && g[x][y] == g[x+1][y] && g[x][y] == g[x+2][y]) {
                mark[x][y] = mark[x+1][y] = mark[x+2][y] = true;
                cleared = true;
            }
            if (y + 2 < 7 && g[x][y] == g[x][y+1] && g[x][y] == g[x][y+2]) {
                mark[x][y] = mark[x][y+1] = mark[x][y+2] = true;
                cleared = true;
            }
        }
    for (int x = 0; x < 5; x++)
        for (int y = 0; y < 7; y++)
            if (mark[x][y]) g[x][y] = 0;
    return cleared;
}

bool empty() {
    for (int x = 0; x < 5; x++)
        if (g[x][0]) return false;
    return true;
}

bool color_check() {
    int cnt[11] = {};
    for (int x = 0; x < 5; x++)
        for (int y = 0; y < 7; y++)
            if (g[x][y]) cnt[g[x][y]]++;
    for (int c = 1; c <= 10; c++)
        if (cnt[c] > 0 && cnt[c] < 3) return false;  // 剪枝3
    return true;
}

bool dfs(int step) {
    if (!color_check()) return false;
    if (step == n) return empty();

    int backup[5][7];
    memcpy(backup, g, sizeof(g));

    for (int x = 0; x < 5; x++)
        for (int y = 0; y < 7; y++) {
            if (!g[x][y]) continue;                        // 剪枝4
            for (int dir = 1; dir >= -1; dir -= 2) {
                int nx = x + dir;
                if (nx < 0 || nx >= 5) continue;
                if (dir == -1 && g[nx][y]) continue;       // 剪枝2：左边有块时由右移处理

                swap(g[x][y], g[nx][y]);
                fall();
                while (clear()) fall();

                ans[step] = {x, y, dir};
                if (dfs(step + 1)) return true;

                memcpy(g, backup, sizeof(g));
            }
        }
    return false;
}
// 若 dfs(0) 为 true，依次输出 ans[0..n-1] 的 (x, y, dir)；否则输出 -1
```

**核心启示**：
- **搜索 + 模拟组合题的核心在于模拟的正确性**：消除→下落→链式消除的循环必须无遗漏
- **字典序保证靠枚举顺序**：按 `(x, y, dir)` 字典序枚举，第一个找到的解即为最小解
- **等价操作去重**是关键剪枝：A 右移与 B 左移等价，只保留一个避免重复搜索
- **颜色计数剪枝**虽简单但效果显著：剩余 1-2 个同色块永远无法消除

## 五、常见技巧与思路总结

### 5.1 排序是剪枝的好搭档

排序本身不是剪枝，但它让剪枝**更早触发**：

```
升序排序 + break：当 nums[i] > remain 时，后面全部更大，一次 break 即可
降序排序 + 优先尝试大值：大值更容易超限 → 更快触发可行性剪枝
```

排序在以下场景中几乎是必须的：
- 组合总和（升序 + break）
- 子集划分（降序 + 大的先放）
- 排列去重（排序后相邻比较）

### 5.2 全局变量记录最优解

```cpp
int ans = INF;  // 全局最优解

void dfs(...) {
    if (cur >= ans) return;  // 最优性剪枝依赖 ans
    if (is_leaf) ans = min(ans, cur);
    // ...
}
```

- 全局 `ans` 随搜索持续收紧
- 搜索越深，`ans` 越精确，剪枝越有效
- 初始值设为合理上界（如 `ans = n`），可以加速首次剪枝

### 5.3 预处理加速判断

将"运行时反复计算"改为"预处理一次查表"：

```
✅ 预计算组合数 C[i] = C(n-1, i)，避免递归中重复算
✅ 预计算距离矩阵 dist[i][j]，避免 sqrt 重复计算
✅ 预计算完全平方数集合 is_sq[x]，避免 sqrt 判断
✅ 预计算可达性/连通性，提前排除孤立状态
```

### 5.4 多种剪枝组合使用

单一剪枝往往不够，真正的威力来自**组合**：

```
题目 10 (LeetCode 698) 使用了 5 种剪枝：
  ① 总和整除检查（前置）
  ② 降序排序（搜索顺序）
  ③ 桶容量检查（可行性）
  ④ 等价桶去重（对称性）
  ⑤ 空桶 break（对称性的变体）
```

经验法则：先加**最容易实现**的剪枝，再逐步添加更复杂的。

### 5.5 剪枝的正确性验证

**剪枝可能引入 bug**。验证方法：

```
步骤 1：先写一个无剪枝的暴力版本（保证正确）
步骤 2：添加剪枝
步骤 3：用小数据对拍——两个版本结果是否一致
步骤 4：边界测试——最小输入、全相同元素、极端数据
```

常见错误：
- 可行性剪枝条件写反（`>` 写成 `>=`）
- 去重条件中 `i > start` 写成 `i > 0`
- 最优性剪枝的初始值设得太小

## 七、终极总结与归纳

### 7.1 搜索剪枝的本质

**一句话**：剪枝就是在搜索树上做"提前判断"——凡是注定走不通或走不优的路，越早放弃越好。

展开来说，搜索的本质是遍历决策树，而剪枝的本质是在遍历过程中利用**约束条件、最优性单调性、对称等价性**等数学性质，证明某些子树不可能包含有效解，从而跳过它们。这不改变算法的正确性，只改变它的效率。

### 7.2 核心要点

1. **可行性剪枝是基础**：违反约束就 return，最容易实现，永远第一个加
2. **最优性剪枝是利器**：对求最值问题效果最显著，依赖全局 `ans` 的持续更新
3. **对称性剪枝防重复**：排序 + 去重条件，排列和组合有不同的去重公式
4. **排序是最佳辅助**：不是剪枝本身，但能让其他剪枝更早触发
5. **估价函数是高阶武器**：IDA* 和 A* 中不可或缺，需要保证"不高估"
6. **多种剪枝组合才是王道**：竞赛中的搜索题往往需要 3-5 种剪枝配合
