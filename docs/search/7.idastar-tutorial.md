## 一、什么是 IDA* 算法

IDA*（Iterative Deepening A*）是一种**迭代加深 + 启发式估价**的搜索算法。它的核心理念非常优美：用 DFS 的空间效率，达到 A* 的最优性保证。

想象你在迷雾中找出口：A* 像是带着地图但背着巨大背包（优先队列占内存），IDDFS 像是空手但完全瞎摸（没有启发式方向）。而 IDA* 则是**轻装上阵但手持指南针**——每一轮设定一个"预算"（阈值），只走"当前代价 + 预估剩余代价"不超过预算的路；走不通就提高预算再来。

### ASCII art：阈值逐渐增大的搜索过程

```
第 1 轮：阈值 threshold = h(start)
         S
        / \
       A   B          ← f(A)=3 ≤ 3 ✓, f(B)=4 > 3 ✗ 剪枝
      / \
     C   D             ← f(C)=4 > 3 ✗, f(D)=3 ≤ 3 ✓（非目标）
    未找到解 → 记录最小超限值 next_threshold = 4

第 2 轮：阈值 threshold = 4
         S
        / \
       A   B           ← f(B)=4 ≤ 4 ✓ 展开
      / \    \
     C   D    E        ← f(C)=4 ✓, f(E)=4 ✓
    /
   G ← 目标！返回解
```

### IDA* vs A* vs IDDFS 核心对比

| 特性 | A* | IDDFS | IDA* |
|------|-----|-------|------|
| 搜索策略 | 最优优先（BFS） | 迭代加深 DFS | 迭代加深 DFS |
| 启发式 | ✅ h(n) | ❌ 无 | ✅ h(n) |
| 空间复杂度 | O(b^d) 指数级 | O(d) 线性 | O(d) 线性 |
| 最优性保证 | ✅（h 可容许） | ✅ | ✅（h 可容许） |
| 重复访问 | 少（有 closed 表）| 多 | 多 |
| 适用场景 | 状态空间可存 | 深度有限 | 状态空间巨大 |

> **一句话总结**：当状态空间大到 A* 存不下、但有好的启发式函数时，IDA* 是最佳选择。

## 二、核心思想与前置知识

### 2.1 迭代加深搜索（IDDFS）思想

迭代加深的核心：**限制 DFS 的深度上限，从小到大逐步放宽**。

```
maxDepth = 0, 1, 2, 3, ...
  对每个 maxDepth：
    从根出发做 DFS，深度 > maxDepth 则剪枝
    若找到解 → 返回
```

看似"重复搜索"浪费时间，但由于搜索树节点数按层指数增长，**最后一层的节点数远超前面所有层之和**，因此总时间仅比单次 BFS 多一个常数因子。

### 2.2 估价函数与阈值控制

IDA* 在 IDDFS 基础上引入估价函数 `f(n) = g(n) + h(n)`：
- **g(n)**：从起点到当前节点的实际代价
- **h(n)**：从当前节点到目标的**启发式估计**（下界）
- **阈值 threshold**：初始为 `h(start)`，每轮更新为本轮所有超限节点中 `f(n)` 的最小值

**可容许性**（Admissibility）：`h(n)` 永远不高估真实代价 → 保证找到最优解。

**一致性**（Consistency）：`h(n) ≤ cost(n, n') + h(n')` → 保证路径上 f 值单调不减，减少冗余搜索。

### 2.3 IDA* 通用框架代码

```cpp
int threshold;

// 返回值：-1 表示找到解，否则返回本次搜索中超出阈值的最小 f 值
int dfs(int g, int last_op) {
    int h = heuristic();
    int f = g + h;
    if (f > threshold) return f;
    if (h == 0) return -1;  // 到达目标

    int next_threshold = INF;
    for (int op = 0; op < OP_COUNT; op++) {
        if (is_reverse(op, last_op)) continue;  // 剪枝：跳过逆操作
        apply(op);
        int res = dfs(g + cost(op), op);
        if (res == -1) return -1;
        next_threshold = min(next_threshold, res);
        undo(op);
    }
    return next_threshold;
}

void ida_star() {
    threshold = heuristic();
    while (true) {
        int res = dfs(0, -1);
        if (res == -1) return;   // 找到解
        threshold = res;         // 更新阈值
    }
}
```

### 2.4 空间复杂度优势分析

| 算法 | 空间复杂度 | 存储内容 |
|------|-----------|---------|
| BFS / A* | O(b^d) | open 表 + closed 表 |
| IDA* | O(d) | 仅当前搜索路径 |

以 15 数码为例：状态总数约 16!/2 ≈ 10^13，A* 根本存不下。IDA* 只需存当前路径（最多几十层），内存开销几乎可忽略。

## 三、IDA* 与相关算法对比详解

### 3.1 IDA* vs A*（何时选哪个）

| 维度 | A* | IDA* |
|------|-----|------|
| 内存 | 需要 open/closed 表，可能爆内存 | 仅用栈空间 O(d) |
| 速度 | 单次扩展，无重复 | 会重复扩展节点 |
| 适用规模 | 状态空间可控（≤10^6） | 状态空间巨大（10^10+） |
| 实现难度 | 需要优先队列 + 哈希去重 | 简单递归 |
| 最优解 | ✅ | ✅（h 可容许） |

**选择策略**：
- 状态空间小、需要快速找解 → **A***
- 状态空间巨大、解的深度可控 → **IDA***
- 边权均为 1 → 两者均可，IDA* 实现更简洁

### 3.2 IDA* vs IDDFS（启发式的价值）

IDDFS 按**深度**限制，IDA* 按 **f = g + h** 限制。启发式函数 h 的价值在于：

```
IDDFS：                     IDA*（有好的 h）：
     S                          S
    /|\                        / \
   A B C  (全展开)            A   B (h 大的被剪)
  /|\ ...                   / \
 盲目搜索每层全部节点        只搜有希望的分支
```

h 越精确，剪枝越狠。理想情况下 h = h*（完美启发式），IDA* 只沿最优路径搜索，无任何浪费。

### 3.3 IDA* 的适用场景

| 场景 | 典型问题 | 关键条件 |
|------|---------|---------|
| 滑动拼图 | 八数码、十五数码 | 曼哈顿距离启发式 |
| 棋盘变换 | 骑士精神、Rotation Game | 差异位计数启发式 |
| 分数拆分 | 埃及分数 | 搜索深度无上界，需迭代加深 |
| 最短操作序列 | Editing a Book、Power Calculus | 每步修复量有限 → 好的下界 |
| 组合优化 | 加法链、排列还原 | BFS 内存爆炸 |

### 对比总表

| | A* | IDDFS | IDA* | 双向 BFS |
|--|-----|-------|------|----------|
| 空间 | 高 | 低 | 低 | 中 |
| 时间 | 最优 | 较慢 | 略慢于 A* | 快 |
| 启发式 | 需要 | 不需要 | 需要 | 不需要 |
| 实现 | 中等 | 简单 | 简单 | 中等 |
| 最佳场景 | 小状态空间 | 深度有限 | 大状态空间 | 双端可达 |

### 题目 1：八数码难题（洛谷 P1379）

> 3×3 方格中有 1~8 和空格，每次将空格与相邻数字交换，求到达目标状态 `123804765`（即标准排列）的最少步数。

**分析**：经典 IDA* 入门题。状态空间 9!/2 ≈ 181440，虽然 BFS 也能做，但 IDA* 更优雅。启发式函数用各数字到目标位置的**曼哈顿距离之和**。

```
状态定义：当前 3×3 棋盘状态
估价函数：h = Σ 每个数字当前位置到目标位置的曼哈顿距离
剪枝条件：g + h > threshold
答案位置：首次 h == 0 时的 g 值
```

**C++ 代码**：

```cpp
int board[3][3];
int goal[9][2] = {{1,1},{0,0},{0,1},{0,2},{1,0},{2,0},{2,1},{2,2},{1,2}};
// goal[v] = 数字 v 在目标状态的 (行, 列)，0 表示空格
int dx[] = {-1, 1, 0, 0}, dy[] = {0, 0, -1, 1};
int threshold;

int heuristic() {
    int h = 0;
    for (int i = 0; i < 3; i++)
        for (int j = 0; j < 3; j++)
            if (board[i][j])
                h += abs(i - goal[board[i][j]][0]) + abs(j - goal[board[i][j]][1]);
    return h;
}

bool dfs(int g, int bx, int by, int last) {
    int h = heuristic();
    if (g + h > threshold) return false;
    if (h == 0) return true;
    for (int d = 0; d < 4; d++) {
        if (d == (last ^ 1) && last >= 0) continue;  // 跳过逆方向
        int nx = bx + dx[d], ny = by + dy[d];
        if (nx < 0 || nx >= 3 || ny < 0 || ny >= 3) continue;
        swap(board[bx][by], board[nx][ny]);
        if (dfs(g + 1, nx, ny, d)) return true;
        swap(board[bx][by], board[nx][ny]);
    }
    return false;
}
// 调用：threshold = heuristic(); 循环 threshold++ 直到 dfs 返回 true
```

**核心启示**：
- **曼哈顿距离**是滑动拼图类问题的"标配"启发式，可容许且紧凑
- **逆操作剪枝**：方向 d 与 last 互逆时跳过（`d ^ 1`），避免来回抖动
- **阈值自然递增**：八数码中每步代价为 1，阈值即为深度上限

### 题目 2：简单网格迷宫（迭代加深基础）

> 给定 N×M 网格迷宫，'S' 起点、'T' 终点、'#' 障碍。求从 S 到 T 的最短路径长度。

**分析**：这道题用 BFS 即可解决，但我们用它来练习 IDA* 的基本框架。启发式函数用**起点/当前点到终点的曼哈顿距离**。

```
状态定义：当前位置 (x, y)
估价函数：h = |x - tx| + |y - ty|
剪枝条件：g + h > threshold
答案位置：首次到达 T 时的 g 值
```

**C++ 代码**：

```cpp
const int MAXN = 105;
char grid[MAXN][MAXN];
int N, M, tx, ty, threshold, ans;
int dx[] = {-1, 1, 0, 0}, dy[] = {0, 0, -1, 1};
bool vis[MAXN][MAXN];

int h(int x, int y) { return abs(x - tx) + abs(y - ty); }

bool dfs(int x, int y, int g) {
    if (g + h(x, y) > threshold) return false;
    if (x == tx && y == ty) { ans = g; return true; }
    vis[x][y] = true;
    for (int d = 0; d < 4; d++) {
        int nx = x + dx[d], ny = y + dy[d];
        if (nx < 0 || nx >= N || ny < 0 || ny >= M) continue;
        if (grid[nx][ny] == '#' || vis[nx][ny]) continue;
        if (dfs(nx, ny, g + 1)) return true;
    }
    vis[x][y] = false;
    return false;
}
// 调用：threshold = h(sx, sy); 循环 threshold++ 直到 dfs 返回 true
```

**核心启示**：
- **vis 数组回溯**：IDA* 本质是 DFS，需要在回溯时清除标记
- **曼哈顿距离下界**：在网格中无权最短路问题中天然可容许
- **对比 BFS**：此题 BFS 更高效，但 IDA* 框架的练习价值在于熟悉模板

### 题目 3：骑士精神（洛谷 P2324 / SCOI2005）

> 5×5 棋盘上有 12 白骑士、12 黑骑士和 1 个空位。骑士按"日"字移动（实际是空位按骑士规则跳）。求最少多少步还原到目标状态，超过 15 步输出 -1。

**分析**：状态空间约 C(25,12)×C(13,12) ≈ 极大，BFS 无法胜任。IDA* 的启发式函数：**当前状态与目标状态不同位置的个数**。每步最多改变 1 个位置的正确性，所以 h 可容许。

```
状态定义：5×5 棋盘状态 + 空位位置
估价函数：h = 与目标不同的格子数（不含空格）
剪枝条件：g + h > threshold（且 threshold ≤ 15）
答案位置：首次 h == 0 时的 g 值，或无解输出 -1
```

**C++ 代码**：

```cpp
int board[5][5], goal[5][5];
int dx[] = {-2,-2,-1,-1,1,1,2,2};
int dy[] = {-1,1,-2,2,-2,2,-1,1};
int threshold;

int heuristic() {
    int cnt = 0;
    for (int i = 0; i < 5; i++)
        for (int j = 0; j < 5; j++)
            if (board[i][j] != goal[i][j]) cnt++;
    return cnt;
}

bool dfs(int g, int bx, int by) {
    int h = heuristic();
    if (g + h > threshold) return false;
    if (h == 0) return true;
    for (int d = 0; d < 8; d++) {
        int nx = bx + dx[d], ny = by + dy[d];
        if (nx < 0 || nx >= 5 || ny < 0 || ny >= 5) continue;
        swap(board[bx][by], board[nx][ny]);
        if (dfs(g + 1, nx, ny)) return true;
        swap(board[bx][by], board[nx][ny]);
    }
    return false;
}

int solve(int bx, int by) {
    for (threshold = 0; threshold <= 15; threshold++)
        if (dfs(0, bx, by)) return threshold;
    return -1;
}
```

**核心启示**：
- **差异位计数**是棋盘置换类问题的通用启发式
- **上限剪枝**：题目给定最多 15 步，天然的迭代加深上界
- **空位视角**：将问题转化为"空位跳跃"而非"骑士移动"，简化状态转移

### 题目 4：十五数码问题（15-Puzzle）

> 4×4 方格中有 1~15 和空格，每次将空格与相邻数字交换。求到达目标状态（1~15 顺序排列，空格在右下角）的最少步数。

**分析**：状态空间 16!/2 ≈ 1.05×10^13，A* 完全无法胜任。IDA* + 曼哈顿距离是经典做法。可进一步使用**线性冲突**增强启发式。

```
状态定义：4×4 棋盘状态
估价函数：h = Σ 曼哈顿距离 + 线性冲突修正
剪枝条件：g + h > threshold
答案位置：首次 h == 0 时的 g 值
```

**C++ 代码**：

```cpp
int board[4][4];
int goal_r[16], goal_c[16];  // 数字 v 的目标行列
int dx[] = {-1,1,0,0}, dy[] = {0,0,-1,1};
int threshold;

void init_goal() {
    for (int v = 1; v <= 15; v++) {
        goal_r[v] = (v - 1) / 4;
        goal_c[v] = (v - 1) % 4;
    }
}

int heuristic() {
    int h = 0;
    for (int i = 0; i < 4; i++)
        for (int j = 0; j < 4; j++)
            if (board[i][j])
                h += abs(i - goal_r[board[i][j]]) + abs(j - goal_c[board[i][j]]);
    return h;
}

bool dfs(int g, int bx, int by, int last) {
    int h = heuristic();
    if (g + h > threshold) return false;
    if (h == 0) return true;
    for (int d = 0; d < 4; d++) {
        if (last >= 0 && d == (last ^ 1)) continue;
        int nx = bx + dx[d], ny = by + dy[d];
        if (nx < 0 || nx >= 4 || ny < 0 || ny >= 4) continue;
        swap(board[bx][by], board[nx][ny]);
        if (dfs(g + 1, nx, ny, d)) return true;
        swap(board[bx][by], board[nx][ny]);
    }
    return false;
}
// 调用：init_goal(); threshold = heuristic(); 循环直到 dfs 返回 true
```

**核心启示**：
- **曼哈顿距离**在大规模滑动拼图中依然高效
- **线性冲突**：同行/同列两个数字目标位置互相阻挡时，至少需额外 2 步，可加入 h 增强
- **增量更新**：移动空格时可只更新被移动数字的曼哈顿距离差值，避免全局重算

### 题目 5：埃及分数问题（洛谷 P1763）

> 将真分数 a/b 拆分为若干互不相同的单位分数之和（1/x₁ + 1/x₂ + ... = a/b），要求项数最少，项数相同时最大分母最小。

**分析**：这是**迭代加深搜索**的经典题目。状态空间理论上无穷（分母可以任意大），没有固定深度上限，必须用迭代加深。IDA* 中的"估价"：剩余分数至少需要 `⌈剩余分数 / (1/当前最小可用分母)⌉` 个单位分数。

```
状态定义：剩余分数 a/b，当前搜索深度 d，上次使用的分母
估价函数：h = 至少还需要多少个单位分数（剩余值 / 最大可用单位分数）
剪枝条件：d + h > maxDepth
答案位置：剩余分数为 0 时记录方案
```

**C++ 代码**：

```cpp
typedef long long ll;
ll ans_arr[25], cur[25];  // 存放当前拆分方案和最优方案
int maxDepth;

ll gcd(ll a, ll b) { return b ? gcd(b, a % b) : a; }

// a/b 为剩余分数，d 为当前深度，from 为分母下界
bool dfs(int d, ll a, ll b, ll from) {
    if (d == maxDepth) {
        if (a == 0) return true;  // 恰好拆完
        return false;
    }
    // 剪枝：剩余 (maxDepth - d) 个分数，每个最大为 1/from
    // 若 (maxDepth - d) * 1/from < a/b，即 b * (maxDepth - d) < a * from
    if ((ll)(maxDepth - d) * b < a * from) return false;
    // 枚举下一个分母 i，从 max(from, b/a 上取整) 开始
    ll start = (b + a - 1) / a;
    if (start < from) start = from;
    for (ll i = start; ; i++) {
        // 剪枝：1/i 太小，剩余步数用完也凑不齐
        if ((ll)(maxDepth - d) * b < a * i) break;
        cur[d] = i;
        ll na = a * i - b, nb = b * i;
        ll g = gcd(abs(na), nb);
        if (dfs(d + 1, na / g, nb / g, i + 1)) return true;
    }
    return false;
}

void solve(ll a, ll b) {
    for (maxDepth = 1; ; maxDepth++)
        if (dfs(0, a, b, 2)) break;
    // 最优方案在 cur[] 中
}
```

**核心启示**：
- **无穷搜索空间**时，迭代加深是唯一可行的完备搜索策略
- **分母下界剪枝**：`1/i < 剩余/剩余步数` 时立即终止枚举
- **方案比较**：题目要求字典序最优时，可在找到解后继续搜索同深度更优方案

### 题目 6：The Rotation Game（HDU 1667 / UVa 1343）

> 井字形棋盘有 24 个格子，值为 1/2/3。8 种操作（A~H）分别沿某行/列循环移位。目标：使中心 8 格全部相同。求最少操作次数及方案。

**分析**：状态空间虽有限但极大。启发式函数：中心 8 格中出现次数最多的数为 maxcnt，则 `h = 8 - maxcnt`（每次操作最多让一个中心格变正确）。

```
状态定义：24 格的值
估价函数：h = 8 - max(cnt[1], cnt[2], cnt[3])（中心 8 格）
剪枝条件：g + h > threshold
答案位置：h == 0 时的操作序列
```

**C++ 代码**：

```cpp
int board[24];
// center[0..7] 为中心 8 格的下标（根据题意预定义）
int center[] = {6, 7, 8, 11, 12, 15, 16, 17};
// op[i][0..6] 为第 i 种操作影响的 7 个格子下标
int op[8][7];  // 预定义 A~H 操作的格子序列
int rev[] = {5, 4, 7, 6, 1, 0, 3, 2};  // A↔F, B↔E, C↔H, D↔G
char path[100];
int threshold;

int heuristic() {
    int cnt[4] = {};
    for (int i = 0; i < 8; i++) cnt[board[center[i]]]++;
    return 8 - max({cnt[1], cnt[2], cnt[3]});
}

void apply_op(int k) {
    int tmp = board[op[k][0]];
    for (int i = 0; i < 6; i++) board[op[k][i]] = board[op[k][i + 1]];
    board[op[k][6]] = tmp;
}

bool dfs(int g, int last) {
    int h = heuristic();
    if (g + h > threshold) return false;
    if (h == 0) { path[g] = '\0'; return true; }
    for (int k = 0; k < 8; k++) {
        if (last >= 0 && k == rev[last]) continue;
        path[g] = 'A' + k;
        apply_op(k);
        if (dfs(g + 1, k)) return true;
        apply_op(rev[k]);  // 撤销 = 执行逆操作
    }
    return false;
}
// 调用：threshold = heuristic(); 循环 threshold++ 直到 dfs 返回 true
```

**核心启示**：
- **操作互逆**：定义 rev[] 映射，通过执行逆操作实现撤销
- **"最多数"启发式**：`8 - maxcnt` 是经典的"差异计数"变体
- **操作编码**：将 A~H 映射为 0~7，配合预定义的格子序列数组处理

### 题目 7：Editing a Book（UVa 11212）

> 给定 1~n 的排列（n ≤ 9），每次可以剪切一段连续子序列并粘贴到另一位置。求最少操作次数使序列变为 1,2,...,n。

**分析**：每次剪切粘贴最多修复 3 个"断点"（即 `a[i]+1 ≠ a[i+1]` 的位置）。所以 `h = ⌈breaks / 3⌉`。枚举所有剪切+粘贴方案进行搜索。

```
状态定义：当前排列
估价函数：h = ⌈不连续断点数 / 3⌉
剪枝条件：g + h > threshold
答案位置：breaks == 0 时的 g 值
```

**C++ 代码**：

```cpp
const int MAXN = 10;
int n;

int breaks(int a[]) {
    int cnt = 0;
    for (int i = 0; i < n - 1; i++)
        if (a[i] + 1 != a[i + 1]) cnt++;
    return cnt;
}

bool dfs(int a[], int g, int maxd) {
    int b = breaks(a);
    if (b == 0) return true;
    if (g + (b + 2) / 3 > maxd) return false;  // 上取整
    // 枚举剪切区间 [l, r) 和粘贴位置 k
    int tmp[MAXN], cur[MAXN];
    for (int l = 0; l < n; l++)
        for (int r = l + 1; r < n; r++) {
            // 剪出 a[l..r-1]，剩余部分拼接后在位置 k 处插入
            int sz = 0;
            for (int i = 0; i < l; i++) tmp[sz++] = a[i];
            for (int i = r; i < n; i++) tmp[sz++] = a[i];
            for (int k = 0; k <= sz; k++) {
                int p = 0;
                for (int i = 0; i < k; i++) cur[p++] = tmp[i];
                for (int i = l; i < r; i++) cur[p++] = a[i];
                for (int i = k; i < sz; i++) cur[p++] = tmp[i];
                if (dfs(cur, g + 1, maxd)) return true;
            }
        }
    return false;
}

int solve(int a[]) {
    for (int maxd = 0; ; maxd++)
        if (dfs(a, 0, maxd)) return maxd;
}
```

**核心启示**：
- **断点修复**是排列操作类 IDA* 问题的通用启发式
- **每步最多修复 3 个断点** → `h = ⌈breaks/3⌉` 是紧凑的可容许估计
- **状态传递**：每步都创建新数组，避免复杂的撤销操作

### 题目 8：Power Calculus（POJ 3134）

> 从 x^1 开始，每步可将已有的两个幂次相加或相减得到新幂次。求最少几步得到 x^n（n ≤ 1000）。

**分析**：这是加法链问题。维护一个已计算的幂次序列 `seq[]`，每步从已有幂次中选两个做加/减。启发式剪枝：当前最大幂次每步最多翻倍，若 `seq[d] × 2^(maxd-d) < n` 则不可能在剩余步数内达到 n。

```
状态定义：已计算幂次集合 seq[0..d]
估价函数：若 seq[d] × 2^(maxd-d) < n 则剪枝
剪枝条件：最大值翻倍仍不够
答案位置：seq[d] == n 时的 d 值
```

**C++ 代码**：

```cpp
int n, seq[50], maxd;

bool dfs(int d) {
    if (seq[d] == n) return true;
    if (d == maxd) return false;
    if (seq[d] << (maxd - d) < n) return false;  // 翻倍剪枝
    for (int i = d; i >= 0; i--) {
        // 加法
        seq[d + 1] = seq[d] + seq[i];
        if (dfs(d + 1)) return true;
        // 减法
        seq[d + 1] = seq[d] - seq[i];
        if (seq[d + 1] > 0 && dfs(d + 1)) return true;
    }
    return false;
}

int solve() {
    seq[0] = 1;
    if (n == 1) return 0;
    for (maxd = 1; ; maxd++)
        if (dfs(0)) return maxd;
}
```

**核心启示**：
- **翻倍上界**是加法链问题的经典剪枝：最大值每步最多倍增
- **倒序枚举**：优先使用较大的幂次，更快接近目标
- **只保留正值**：`seq[d] - seq[i] > 0` 的合法性判断不可省略

### 题目 10：埃及分数深度上限优化（洛谷 P1763 加强版）

> 同题 5，但要求在项数最少的前提下，最大的分母尽可能小；若仍相同，则次大分母尽可能小，以此类推。

**分析**：在题 5 基础上增加更精细的**方案比较**。每找到一个合法拆分，与当前最优方案逐项比较。关键优化：分母上界剪枝更紧。

```
状态定义：同题 5
估价函数：同题 5
剪枝条件：同题 5 + 分母上界 ≤ 剩余分数 × 剩余步数
答案位置：找到所有同深度方案后取最优
```

**C++ 代码**：

```cpp
typedef long long ll;
ll ans[25], cur[25];
int maxDepth;
bool found;

ll gcd(ll a, ll b) { return b ? gcd(b, a % b) : a; }

bool better() {  // cur 是否优于 ans（最大分母更小）
    if (!found) return true;
    for (int i = maxDepth - 1; i >= 0; i--)
        if (cur[i] != ans[i]) return cur[i] < ans[i];
    return false;
}

void dfs(int d, ll a, ll b, ll from) {
    if (d == maxDepth) {
        if (a == 0 && better()) {
            found = true;
            for (int i = 0; i < maxDepth; i++) ans[i] = cur[i];
        }
        return;
    }
    if ((ll)(maxDepth - d) * b < a * from) return;
    ll start = (b + a - 1) / a;
    if (start < from) start = from;
    for (ll i = start; ; i++) {
        if ((ll)(maxDepth - d) * b < a * i) break;
        cur[d] = i;
        ll na = a * i - b, nb = b * i, g = gcd(abs(na), nb);
        dfs(d + 1, na / g, nb / g, i + 1);
    }
}

void solve(ll a, ll b) {
    for (maxDepth = 1; ; maxDepth++) {
        found = false;
        dfs(0, a, b, 2);
        if (found) break;
    }
}
```

**核心启示**：
- **多方案比较**：同一深度可能有多个合法拆分，需遍历所有方案取最优
- **better() 函数**：从最大分母开始逆序比较，体现"最大分母最小"的优先级
- **分母上界收紧**：`(maxDepth - d) / i ≥ a/b` 的变形提供更强剪枝

### 题目 11：骑士精神深度分析（SCOI 变种深度讨论）

> 同题 3 设定，但要求分析不同启发式函数对搜索效率的影响，并实现**增量更新**的高效版本。

**分析**：题 3 中每次调用 heuristic() 都遍历 25 格，时间为 O(25)。优化方向：移动空位时只有 2 个格子状态改变，可增量更新 h 值。

```
状态定义：同题 3
估价函数：h = 不同位计数（增量维护）
优化要点：swap 前后只需检查 2 个格子是否与目标一致
答案位置：同题 3
```

**C++ 代码**：

```cpp
int board[5][5], goal[5][5];
int dx[] = {-2,-2,-1,-1,1,1,2,2};
int dy[] = {-1,1,-2,2,-2,2,-1,1};

int full_h() {
    int cnt = 0;
    for (int i = 0; i < 5; i++)
        for (int j = 0; j < 5; j++)
            if (board[i][j] != goal[i][j]) cnt++;
    return cnt;
}

// 增量更新：交换 (bx,by) 和 (nx,ny) 后 h 的变化量
int delta_h(int bx, int by, int nx, int ny) {
    int d = 0;
    // 交换前：检查 (nx,ny) 处的数字移到 (bx,by) 是否变好/变差
    d -= (board[nx][ny] != goal[nx][ny]);  // 之前在 (nx,ny) 的贡献
    d += (board[nx][ny] != goal[bx][by]);  // 移到 (bx,by) 后的贡献
    // 空格移到 (nx,ny)：空格不计入 h
    d -= (board[bx][by] != goal[bx][by]);  // 之前空格在 (bx,by)
    d += (board[bx][by] != goal[nx][ny]);  // 空格移到 (nx,ny)
    return d;
}

bool dfs(int g, int h, int bx, int by, int threshold) {
    if (g + h > threshold) return false;
    if (h == 0) return true;
    for (int d = 0; d < 8; d++) {
        int nx = bx + dx[d], ny = by + dy[d];
        if (nx < 0 || nx >= 5 || ny < 0 || ny >= 5) continue;
        int dh = delta_h(bx, by, nx, ny);
        swap(board[bx][by], board[nx][ny]);
        if (dfs(g + 1, h + dh, nx, ny, threshold)) return true;
        swap(board[bx][by], board[nx][ny]);
    }
    return false;
}
// 调用：int h = full_h(); for threshold 0..15: dfs(0, h, bx, by, threshold)
```

**核心启示**：
- **增量更新** h 值将每步的启发式计算从 O(N²) 降为 O(1)
- **delta 计算要小心**：空格不计入不匹配数，需要分别处理
- **通用优化思路**：任何 IDA* 问题中 heuristic() 被频繁调用时，都应考虑增量更新

### 题目 12：复杂棋盘变换（UVa 10181 / 15-Puzzle 完整版）

> 给定 4×4 棋盘初始状态，判断是否有解；若有解，输出不超过 45 步的操作序列（U/D/L/R）将其还原为目标状态。

**分析**：综合运用 IDA* 全部技巧：(1) 逆序数奇偶性判断有解性；(2) 曼哈顿距离 + 线性冲突启发式；(3) 逆操作剪枝；(4) 增量更新 h 值。

```
状态定义：4×4 棋盘状态 + 空格位置
有解判断：逆序数 + 空格所在行（从底部数）的奇偶性为偶 → 有解
估价函数：h = 曼哈顿距离 + 线性冲突
剪枝条件：g + h > threshold，逆操作剪枝
答案位置：h == 0 时输出操作路径
```

**C++ 代码**：

```cpp
int board[4][4], gr[16], gc[16];
int dx[] = {-1,1,0,0}, dy[] = {0,0,-1,1};
char dir[] = "UDLR";
char path[50];
int threshold;

void init_goal() {
    for (int v = 1; v <= 15; v++) { gr[v] = (v-1)/4; gc[v] = (v-1)%4; }
}

bool solvable(int bx) {
    int inv = 0, a[16], k = 0;
    for (int i = 0; i < 4; i++)
        for (int j = 0; j < 4; j++)
            if (board[i][j]) a[k++] = board[i][j];
    for (int i = 0; i < k; i++)
        for (int j = i+1; j < k; j++)
            if (a[i] > a[j]) inv++;
    return (inv + (3 - bx)) % 2 == 0;  // 空格从底部数的行号
}

int manhattan() {
    int h = 0;
    for (int i = 0; i < 4; i++)
        for (int j = 0; j < 4; j++)
            if (board[i][j])
                h += abs(i - gr[board[i][j]]) + abs(j - gc[board[i][j]]);
    return h;
}

int linear_conflict() {
    int lc = 0;
    for (int i = 0; i < 4; i++) {
        // 行冲突
        for (int j = 0; j < 4; j++)
            for (int k = j+1; k < 4; k++) {
                int vj = board[i][j], vk = board[i][k];
                if (vj && vk && gr[vj]==i && gr[vk]==i && gc[vj]>gc[vk])
                    lc++;
            }
        // 列冲突
        for (int j = 0; j < 4; j++)
            for (int k = j+1; k < 4; k++) {
                int vj = board[j][i], vk = board[k][i];
                if (vj && vk && gc[vj]==i && gc[vk]==i && gr[vj]>gr[vk])
                    lc++;
            }
    }
    return 2 * lc;
}

bool dfs(int g, int bx, int by, int last) {
    int h = manhattan() + linear_conflict();
    if (g + h > threshold) return false;
    if (h == 0) { path[g] = '\0'; return true; }
    for (int d = 0; d < 4; d++) {
        if (last >= 0 && d == (last ^ 1)) continue;
        int nx = bx + dx[d], ny = by + dy[d];
        if (nx < 0 || nx >= 4 || ny < 0 || ny >= 4) continue;
        path[g] = dir[d];
        swap(board[bx][by], board[nx][ny]);
        if (dfs(g + 1, nx, ny, d)) return true;
        swap(board[bx][by], board[nx][ny]);
    }
    return false;
}
// 调用：init_goal(); if (!solvable(bx)) 输出无解;
// 否则 threshold = manhattan() + linear_conflict(); 循环直到 dfs 返回 true
```

**核心启示**：
- **有解性预判**：逆序数 + 空格行奇偶性，O(n²) 即可判断，避免无效搜索
- **线性冲突增强**：在曼哈顿距离基础上，每对冲突额外 +2，显著减少搜索节点
- **综合技巧**：逆操作剪枝 + 启发式增强 + 有解性预判，三管齐下

### 题目 13：铁盘整理（洛谷 P2534 / AHOI2012）

> n 个半径互不相同的铁盘从上到下堆叠（n ≤ 16），每次可以将最上面的 k 个盘子整体翻转（前缀翻转），使它们的顺序反转。目标是将所有铁盘按半径从小到大排列。求最少翻转次数。

**分析**：经典的前缀翻转排序问题（Pancake Sorting 变体）。n ≤ 16，每步有 n−1 种翻转选择，BFS 状态空间 16! 级别不可接受，适合 IDA*。启发式函数：统计相邻位置数值之差不为 1 的"断点"数（含末尾哨兵 a[n+1] = n+1），每次前缀翻转最多消除 1 个断点，因此 h 是步数的合法下界。先对输入离散化为 1~n 简化判断。

```
状态定义：当前排列 a[1..n]（离散化后）
估价函数：h = 相邻位置差 ≠ 1 的对数（含哨兵 a[n+1]=n+1）
剪枝条件：dep + h > maxdep
答案位置：h == 0 时的 dep 值
```

**C++ 代码**：

```cpp
int n, a[20];

int h() {
    int cnt = 0;
    for (int i = 1; i <= n; i++)
        if (abs(a[i] - a[i + 1]) != 1) cnt++;
    return cnt;
}

bool dfs(int dep, int maxdep, int last) {
    int est = h();
    if (est == 0) return true;
    if (dep + est > maxdep) return false;
    for (int i = 2; i <= n; i++) {
        if (i == last) continue;  // 避免连续相同翻转（自逆操作剪枝）
        reverse(a + 1, a + i + 1);
        if (dfs(dep + 1, maxdep, i)) return true;
        reverse(a + 1, a + i + 1);  // 撤销：前缀翻转的逆是自身
    }
    return false;
}

int solve() {
    // 离散化：将铁盘半径映射为 1~n
    int b[20];
    copy(a + 1, a + n + 1, b + 1);
    sort(b + 1, b + n + 1);
    for (int i = 1; i <= n; i++)
        a[i] = lower_bound(b + 1, b + n + 1, a[i]) - b;
    a[n + 1] = n + 1;  // 哨兵，简化边界判断
    for (int maxdep = h(); ; maxdep++)
        if (dfs(0, maxdep, -1)) return maxdep;
}
```

**核心启示**：
- **断点启发式**：统计相邻不连续的对数，每次前缀翻转最多修复 1 个断点，是经典的不高估下界
- **前缀翻转可逆性**：翻转操作自身即为逆操作（翻转两次等于不动），撤销直接再翻一次
- **离散化简化判断**：将任意大小映射为 1~n，目标统一为升序排列 [1, 2, ..., n]
- **哨兵技巧**：设 a[n+1] = n+1 避免边界特判，使断点计数覆盖末尾位置

## 五、常见技巧与思路总结

### 5.1 启发式函数的设计

设计 h(n) 的核心原则：**可容许（不高估）且尽量紧凑（接近真实值）**。

| 问题类型 | 常用启发式 | 原理 |
|---------|-----------|------|
| 滑动拼图 | 曼哈顿距离 | 每步只能移动一格 |
| 棋盘置换 | 不匹配位数 | 每步最多修复一个位置 |
| 排列操作 | 断点数 / 操作修复量 | 每次操作最多修复 k 个断点 |
| 加法链 | 翻倍上界 | 最大值每步最多倍增 |
| 分数拆分 | 剩余值 / 最大可用分数 | 贪心下界 |

**进阶技巧**：
- **组合多个启发式**：`h = max(h1, h2)` 取最大值，仍然可容许但更紧
- **Pattern Database**：预计算子问题的精确解作为启发式（用于大规模拼图）

### 5.2 迭代加深的阈值更新策略

```
标准策略：threshold = min{ f(n) | f(n) > 当前 threshold }
特殊情况：
  - 边权全为 1 → threshold 直接 +1
  - 边权为整数 → threshold 按整数递增
  - 边权为实数 → 必须记录最小超限 f 值（否则可能死循环）
```

注意：如果启发式太弱（h ≈ 0），阈值增长极慢，IDA* 退化为 IDDFS，性能大幅下降。

### 5.3 避免逆操作的剪枝

```cpp
// 方法 1：方向编码 + 异或
// 0↔1 (上↔下), 2↔3 (左↔右)
if (d == (last ^ 1)) continue;

// 方法 2：预计算逆操作表
int rev[] = {5, 4, 7, 6, 1, 0, 3, 2};
if (k == rev[last]) continue;

// 方法 3：记录上一步移动的对象，禁止立即移回
if (moved_piece == last_moved) continue;
```

逆操作剪枝效果显著：分支因子从 b 降为 b-1（相对减少 `1/b` 的搜索量）。

### 5.4 IDA* 的常见陷阱

| 陷阱 | 症状 | 解决方案 |
|------|------|---------|
| h 不可容许 | 找到非最优解 | 检查 h 是否高估 |
| 忘记撤销操作 | 搜索结果错误 | apply + undo 严格配对 |
| 阈值不更新 | 死循环 | 确保 next_threshold 严格增大 |
| 对称状态重复 | TLE | 添加对称性剪枝 |
| heuristic 重算慢 | TLE | 增量更新 h 值 |

## 七、终极总结与归纳

### 7.1 IDA* 的本质

**一句话**：IDA* 是在 DFS 的空间效率与 A* 的最优保证之间取得完美平衡的搜索算法。

**展开**：它用"迭代加深"解决了 DFS 不保证最优的问题，用"启发式估价"解决了 IDDFS 盲目搜索的低效问题，用"线性空间"解决了 A* 内存爆炸的困境。三者结合，IDA* 成为大状态空间最优路径搜索的首选武器。

### 7.2 核心要点

1. **f(n) = g(n) + h(n)**：当前代价 + 启发式估计 = 总估价，控制搜索边界
2. **阈值迭代**：每轮以最小超限 f 值为新阈值，逐步逼近最优解
3. **可容许性**：h(n) ≤ h*(n) 是最优性的根本保证，设计启发式时必须验证
4. **空间 O(d)**：只存储当前搜索路径，适合 10^10+ 状态空间
5. **逆操作剪枝**：几乎所有 IDA* 题目都需要，简单但有效
6. **增量更新**：高频调用的 heuristic() 应尽量 O(1) 增量计算
