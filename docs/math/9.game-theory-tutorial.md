## 一、什么是博弈论

**博弈论**（Game Theory）研究的是两个或多个理性决策者之间的策略互动问题。在算法竞赛中，博弈论通常指**组合博弈论**（Combinatorial Game Theory），即两名玩家轮流操作，信息完全公开，无随机因素，最终一方无法操作则判负（Normal Play Convention）。

博弈论问题的核心不在于"怎么玩"，而在于"谁必胜"。给定一个初始局面，我们需要判断先手还是后手拥有必胜策略，并在必要时给出具体的获胜操作。这类问题往往需要深刻的数学观察与精巧的构造，是竞赛数学中最优雅的专题之一。

博弈论是数学类算法的重要分支，与数论（GCD/辗转相除）、位运算（异或）等知识密切相关，也是许多高级构造问题的理论基础。

### 博弈论 vs 普通搜索/DP

| 维度 | 博弈论 | 普通搜索/DP |
|------|--------|------------|
| 决策者 | 两人对抗，轮流操作 | 单人最优化 |
| 目标 | 判断先手/后手必胜 | 求最优值/方案数 |
| 核心工具 | 异或、SG 函数、数学结论 | 状态转移、剪枝 |
| 复杂度 | 常有 O(1)/O(n) 数学解 | 通常 O(n^k) |
| 思维方式 | 找规律、归纳、构造 | 枚举、递推 |

## 二、核心思想与前置知识

### 2.1 博弈论基本概念

**必胜态（N-position）**：当前操作的玩家有必胜策略的局面。
**必败态（P-position）**：当前操作的玩家无论怎么走都会输的局面。

核心判定规则：
- 所有终止局面（无法操作）是 **P-position**（必败态）
- 能转移到至少一个 P-position 的局面是 **N-position**（必胜态）
- 所有后继都是 N-position 的局面是 **P-position**（必败态）

> **P/N 判定示意：**
> 终态(P) $\leftarrow$ 能到P的局面(N) $\leftarrow$ 只能到N的局面(P) $\leftarrow$ ...
>
> 例：取石子，每次取 1-3 个
> 石子数:  0   1   2   3   4   5   6   7   8
> 状态:    P   N   N   N   P   N   N   N   P
> 规律:    4的倍数为P（必败），其余为N（必胜）

### 2.2 Sprague-Grundy 定理

**SG 定理**是组合博弈论的核心，将复杂博弈分解为独立子博弈。

**SG 函数定义**：对于一个局面 x，其 SG 值为：

$$SG(x) = \text{mex}\{SG(y) \mid y \text{ 是 } x \text{ 的后继局面}\}$$

其中 $\text{mex}(S)$ 表示不属于集合 $S$ 的最小非负整数。

**SG 定理**：多个独立子博弈的组合博弈，其 SG 值等于各子博弈 SG 值的异或：

$$SG(G_1 + G_2 + \cdots + G_n) = SG(G_1) \oplus SG(G_2) \oplus \cdots \oplus SG(G_n)$$

- 整体 SG 值 $\neq 0$：先手必胜
- 整体 SG 值 $= 0$：后手必胜

```cpp
// SG 函数计算框架
int sg[MAXN];
bool vis[MAXN];

int getSG(int x) {
    if (sg[x] != -1) return sg[x];
    set<int> S;
    for (int y : moves(x))  // 枚举所有后继
        S.insert(getSG(y));
    int res = 0;
    while (S.count(res)) res++;  // mex
    return sg[x] = res;
}
```

### 2.3 Nim 游戏与异或

**Nim 游戏**：n 堆石子，每次从一堆中取任意正数个。先手必胜当且仅当所有堆的异或和不为 0：

$$a_1 \oplus a_2 \oplus \cdots \oplus a_n \neq 0 \Rightarrow \text{先手必胜}$$

这是 SG 定理最经典的应用：每堆石子是一个独立子博弈，$SG(x) = x$。

> **Nim 游戏异或分析：**
> 堆: 3, 5, 7
> 异或: $3 \oplus 5 \oplus 7 = 011 \oplus 101 \oplus 111 = 001 \neq 0$ $\rightarrow$ 先手必胜
> 策略: 从 7 中取 1 个变为 6，使 $3 \oplus 5 \oplus 6 = 0$

## 三、分类与变体详解

### 3.1 经典 Nim 游戏

n 堆石子，每次从任意一堆中取任意正数个石子，取走最后一颗者胜。

> **核心观察：** $SG(x) = x$
> **构造策略：** 先手使异或和变为 0，之后对手每步都会破坏平衡，先手再恢复
> **正确性：** 终态异或和为 0（P态），异或和为 0 时任何操作都使其非 0

### 3.2 巴什博弈（Bash Game）

一堆 n 个石子，每次最多取 m 个（至少取 1 个），取完者胜。

> **核心观察：** $n \% (m+1) = 0$ 时后手必胜，否则先手必胜
> **构造策略：** 先手取 $n \% (m+1)$ 个，之后对手取 $k$ 个先手取 $m+1-k$ 个
> **正确性：** $SG(x) = x \% (m+1)$，周期为 $m+1$

### 3.3 威佐夫博弈（Wythoff's Game）

两堆石子（a, b），每次从一堆取任意个或从两堆取相同个，取完者胜。

> **核心观察：** 必败态为 $(\lfloor k \cdot \varphi \rfloor, \lfloor k \cdot \varphi^2 \rfloor)$，$k=0,1,2,...$，$\varphi = (1+\sqrt{5})/2$
> **构造策略：** 判断 $(\min(a,b), \max(a,b))$ 是否为 Beatty 序列对
> **正确性：** Beatty 定理保证必败态恰好覆盖所有非负整数对

必败态序列（前几项）：(0,0), (1,2), (3,5), (4,7), (6,10), (8,13), ...

### 3.4 阶梯博弈（Staircase Nim）

将石子放在阶梯上（编号 0, 1, 2, ...），每次从第 i 级（i≥1）移任意个到第 i-1 级，无法操作者负。

> **核心观察：** 等价于对奇数位置上的石子做 Nim 游戏
> **构造策略：** 只需关注奇数阶梯上石子的异或和
> **正确性：** 偶数位置的移动不影响奇数位异或和，对手在偶数位的操作可被镜像

### 3.5 Green Hackenbush

在一个根植图上，每次删一条边以及与根不连通的部分，无法操作者负。

> **核心观察：** 树上 Hackenbush 每条链的 SG 值 = 链长
> **构造策略：** 对于树，每个节点的 SG 值 = 所有子树 SG 值异或
> **正确性：** Colon Principle（合并等价定理）

### 3.6 博弈变体对比总表

| 博弈类型 | 局面描述 | 必败条件 | 核心工具 |
|---------|---------|---------|---------|
| 经典 Nim | n 堆取任意 | 异或和 = 0 | 异或 |
| 巴什博弈 | 1 堆取 1~m | n%(m+1)=0 | 取模 |
| 威佐夫 | 2 堆取同或单 | Beatty 序列 | 黄金比例 |
| 阶梯 Nim | 阶梯移动 | 奇数位异或=0 | 奇偶+异或 |
| Hackenbush | 图上删边 | 树上异或=0 | SG+Colon |
| 反 Nim | 取最后者负 | 全为1时异或=0 | 特判+异或 |

### 🟢 入门级 Level 1：Nim 与巴什博弈基础

### 题目 1：Nim 游戏（洛谷 P2197）

> 有 n 堆石子，两人轮流取，每次从一堆中取任意正数个，取完最后一颗者胜。判断先手是否必胜。

**分析**：经典 Nim 游戏，直接用异或判定。

> **核心观察：** 每堆石子 SG 值等于自身，多堆 Nim 的 SG 值为所有堆的异或和
> **构造策略：** 异或和 $\neq 0$ 则先手必胜
> **正确性：** SG 定理 + mex 定义可证 $SG(x) = x$

**C++ 代码**：

```cpp
int n, a[MAXN];

void solve() {
    int xorSum = 0;
    for (int i = 0; i < n; i++)
        xorSum ^= a[i];
    // xorSum != 0 → 先手必胜，否则后手必胜
}
```

**核心启示**：
- **Nim 游戏是博弈论基石**：几乎所有公平组合博弈最终都归结于 Nim
- **异或是博弈论的核心运算**：异或和为 0 的平衡态是关键
- **SG(x)=x 是最基础的 SG 函数**：理解它才能理解更复杂的博弈

### 题目 2：取火柴游戏（洛谷 P1247）

> n 堆火柴，两人轮流取，每次从一堆取任意正数根，取完最后一根者胜。若先手必胜，输出第一步操作。

**分析**：标准 Nim 游戏，需要输出具体策略。先手需要找到一堆使操作后异或和变为 0。

> **核心观察：** 若 $xor \neq 0$，存在某堆 $a[i]$ 使得 $a[i] \oplus xor < a[i]$
> **构造策略：** 从第 $i$ 堆取走 $a[i] - (a[i] \oplus xor)$ 个，使新异或和为 0
> **正确性：** $a[i] \oplus xor < a[i]$ 意味着 $xor$ 的最高位 1 在 $a[i]$ 中也为 1

**C++ 代码**：

```cpp
int n, a[MAXN];

void solve() {
    int xorSum = 0;
    for (int i = 0; i < n; i++)
        xorSum ^= a[i];
    if (xorSum == 0) return;  // 后手必胜
    for (int i = 0; i < n; i++) {
        if ((a[i] ^ xorSum) < a[i]) {
            int take = a[i] - (a[i] ^ xorSum);
            // 从第 i 堆取走 take 个
            a[i] ^= xorSum;
            break;
        }
    }
}
```

**核心启示**：
- **构造必胜策略的通用方法**：找到异或和最高位对应的堆并调整
- **a[i] ^ xor < a[i] 的判定**：xor 的最高位在 a[i] 中为 1 即满足
- **博弈论不仅要判必胜，还要会构造操作**：这在竞赛中非常常见

### 题目 3：取石子游戏（洛谷 P2252）

> 两堆石子（a, b），每次可以从一堆取任意个或从两堆各取相同个，取完者胜。判断先手是否必胜。

**分析**：经典威佐夫博弈。必败态满足黄金比例关系。

> **核心观察：** $(a,b)$ 为必败态当且仅当 $\min(a,b) = \lfloor |a-b| \cdot \varphi \rfloor$，$\varphi = (1+\sqrt{5})/2$
> **构造策略：** 计算差值 $d = |a-b|$，判断 $\lfloor d \cdot \varphi \rfloor$ 是否等于 $\min(a,b)$
> **正确性：** Beatty 定理保证 $\{\lfloor n\varphi \rfloor\}$ 和 $\{\lfloor n\varphi^2 \rfloor\}$ 恰好划分正整数

**C++ 代码**：

```cpp
void solve(int a, int b) {
    if (a > b) swap(a, b);
    int d = b - a;
    double phi = (1.0 + sqrt(5.0)) / 2.0;
    int k = (int)(d * phi);
    // k == a → 必败态（后手必胜），否则先手必胜
}
```

**核心启示**：
- **威佐夫博弈的黄金比例**：$\varphi = \frac{1+\sqrt{5}}{2}$ 是必败态的核心常数
- **浮点精度需要注意**：大数据时可能需要高精度或其他判定方式
- **差值是关键变量**：将二维问题转化为一维判定

### 题目 4：取数游戏 II（洛谷 P1288）

> 环形排列 n 个数，两人轮流取，每次只能取当前剩余数列两端之一，最终谁的数之和大谁赢。

**分析**：这不是标准 Nim，而是一个环形取数博弈。关键观察：n 为偶数时，先手可以保证取到所有奇数位或所有偶数位上的数。

> **核心观察：** $n$ 为偶数时，先手可以选择取所有奇数位或偶数位的数
> **构造策略：** 先手选择奇数位和与偶数位和中较大的一组，按对应策略取数
> **正确性：** 先手第一步选择确定了奇偶性，之后每步保持策略一致即可

**C++ 代码**：

```cpp
int a[MAXN], n;

void solve() {
    int sumOdd = 0, sumEven = 0;
    for (int i = 0; i < n; i++) {
        if (i % 2 == 0) sumEven += a[i];
        else sumOdd += a[i];
    }
    // 先手选 max(sumOdd, sumEven)，必定不输
}
```

**核心启示**：
- **环形博弈的奇偶策略**：先手可以锁定一组位置上的数
- **不是所有博弈都用异或判定**：有些需要构造性思维
- **"先手不输"与"先手必胜"的区别**：此题先手至少平局

### 题目 5：欧几里德的游戏（洛谷 P1290）

> 给定两个正整数 (a, b)，两人轮流操作：从较大数中减去较小数的正整数倍（不能减到负数），将某个数减到 0 者胜。

**分析**：类似辗转相除的博弈。当较大数 ≥ 2 倍较小数时，当前玩家有选择权（可以多减或少减），因此必胜。

> **核心观察：** 若 $a \geq 2b$（假设 $a \geq b$），当前玩家必胜；否则只能减一次，递归判断
> **构造策略：** 大数 $\geq$ 2 倍小数时直接赢，否则执行一步辗转相除后交换先后手
> **正确性：** 有选择权的玩家可以控制局面转入对手的必败态

**C++ 代码**：

```cpp
// 返回 true 表示当前操作者必胜
bool solve(int a, int b) {
    if (a < b) swap(a, b);
    if (b == 0) return false;  // 上一步已经赢了
    if (a >= 2 * b) return true;  // 有选择权，必胜
    return !solve(b, a - b);  // 只能减一次，交换先后手
}
```

**核心启示**：
- **辗转相除与博弈的结合**：GCD 的过程本身就是博弈的行动序列
- **"有选择权即必胜"**：当可以控制减多少时，当前玩家掌握主动
- **递归深度等于辗转相除的步数**：复杂度为 $O(\log(\max(a,b)))$

### 题目 6：巴什博弈（HDU 1846）

> 一堆 n 个石子，每次取 1~m 个，取完者胜。判断先手是否必胜。

**分析**：经典巴什博弈，直接取模判定。

> **核心观察：** $n \% (m+1) \neq 0$ 时先手必胜
> **构造策略：** 先手取 $n \% (m+1)$ 个，之后对手取 $k$ 个先手取 $m+1-k$ 个
> **正确性：** $(m+1)$ 的倍数为必败态，先手可将状态推到 $(m+1)$ 的倍数

**C++ 代码**：

```cpp
void solve(int n, int m) {
    // n % (m + 1) != 0 → 先手必胜
    // 先手首步取 n % (m + 1) 个
}
```

**核心启示**：
- **巴什博弈是 Nim 的单堆特例**：限制了每次最多取 m 个
- **SG(x) = x % (m+1)**：理解 SG 函数的周期性
- **"凑满"策略**：双方每轮合取 m+1 个，先手控制节奏

### 题目 7：Nim 游戏升级——限制取数（POJ 2068）

> n 堆石子，每次从一堆中取 1~m 个。所有堆异或 SG 值判定胜负。

**分析**：每堆的 SG 值不再是 x 本身，而是 x % (m+1)，然后异或所有 SG 值。

> **核心观察：** 限制每次最多取 $m$ 个时，$SG(x) = x \% (m+1)$
> **构造策略：** 计算所有堆 $SG(a[i]) = a[i] \% (m+1)$ 的异或和
> **正确性：** 巴什博弈的 SG 值具有周期 $m+1$，SG 定理保证异或合成

**C++ 代码**：

```cpp
int n, a[MAXN], m;

void solve() {
    int xorSum = 0;
    for (int i = 0; i < n; i++)
        xorSum ^= (a[i] % (m + 1));
    // xorSum != 0 → 先手必胜
}
```

**核心启示**：
- **SG 定理的威力**：将限制取数的博弈转化为标准 Nim
- **SG 值的周期性**：许多博弈的 SG 函数都有周期规律
- **从特殊到一般**：巴什博弈→限制 Nim→任意博弈

### 题目 8：阶梯 Nim（POJ 1704）

> n 个棋子在数轴上，两人轮流选择一个棋子向左移动任意步（不能越过其他棋子），无法移动者负。

**分析**：将棋子间的间隔视为阶梯博弈。从右到左，相邻棋子之间的空隙构成阶梯，移动棋子等价于将石子从一级移到下一级。

> **核心观察：** 将间隔按奇偶分组，奇数位间隔的异或和决定胜负
> **构造策略：** 排序后计算相邻棋子间距，取奇数位间距做 Nim
> **正确性：** 偶数位间隔的变化不影响奇数位异或和，等价于标准阶梯 Nim

**C++ 代码**：

```cpp
int pos[MAXN], n;

void solve() {
    sort(pos, pos + n);
    int xorSum = 0;
    // 从右往左配对，奇数位间隔参与异或
    for (int i = n - 1; i >= 0; i -= 2) {
        int gap = (i > 0) ? pos[i] - pos[i-1] - 1 : pos[i] - 1;
        xorSum ^= gap;
    }
    // xorSum != 0 → 先手必胜
}
```

**核心启示**：
- **棋盘博弈到阶梯 Nim 的转化**：识别移动的本质是间距转移
- **从右往左配对**：奇数位间距参与异或，偶数位间距无关
- **建模能力是关键**：将看似不同的博弈归约到已知模型

### 题目 9：翻硬币游戏（ POJ 2484）

> 环形排列 n 枚硬币，每次翻转 1 枚或相邻 2 枚正面朝上的硬币使其朝下，无法操作者负。

**分析**：环形对称性是关键。当 n ≥ 3 时，先手首步翻中间（如果 n 为奇数翻 1 枚，偶数翻中间 2 枚），之后镜像对手操作。

> **核心观察：** $n = 1$ 或 $n = 2$ 时先手直接胜；$n \geq 3$ 时先手利用对称必胜
> **构造策略：** 先手破坏对称后，利用镜像策略保证最后一步
> **正确性：** 对称操作保证先手比后手多操作一次

**C++ 代码**：

```cpp
void solve(int n) {
    // n >= 1 时先手必胜
    // n == 0 时后手必胜（无操作可做）
}
```

**核心启示**：
- **对称策略是博弈论的强力武器**：先手制造对称，后手被迫打破
- **环形问题优先考虑对称性**：先手占据中心后镜像跟随
- **有时结论极简但证明需要分类讨论**：博弈论的特点

### 题目 10：Multi-SG 游戏（HDOJ 3537）

> 一排硬币，每次翻转一枚正面硬币（变反面），同时可选翻转左边任意一枚。所有硬币反面朝上时无法操作者负。

**分析**：每枚正面硬币是一个独立子博弈，其 SG 值取决于位置。单独翻一枚位置 i 的 SG 值需要通过 mex 递推。

> **核心观察：** 位置 $i$ 的 $SG(i)$ = $i$ 的二进制中最低位的 2 的幂（即 lowbit(i+1) 相关）
> **构造策略：** 所有正面朝上硬币位置的 SG 值异或
> **正确性：** 翻转操作的后继集合决定了 mex 值，可通过打表发现规律

**C++ 代码**：

```cpp
int coins[MAXN], n;  // coins[i] = 1 表示正面

void solve() {
    int xorSum = 0;
    for (int i = 0; i < n; i++) {
        if (coins[i])
            xorSum ^= getSG(i);  // SG 值通过打表或公式计算
    }
    // xorSum != 0 → 先手必胜
}
```

**核心启示**：
- **打表找规律是求 SG 函数的常用手段**：先暴力算小规模，再发现周期或公式
- **翻硬币游戏的通用框架**：每枚硬币独立，SG 值异或
- **lowbit 在 SG 函数中的出现**：位运算与博弈论的深层联系

### 🔴 高级 Level 4：复杂博弈与构造

### 题目 11：Green Hackenbush on Trees（HDU 3590）

> 给定一棵有根树，两人轮流删一条边（同时删去与根不连通的部分），无法操作者负。

**分析**：树上 Green Hackenbush，每个节点的 SG 值等于其所有子边的 SG 值异或。叶子边的 SG 值为 1（链长 1）。

> **核心观察：** 树上每条边 SG 值为 1，合并时异或；等价于每个节点的深度奇偶
> **构造策略：** DFS 计算每个子树的 SG 值（边数的异或）
> **正确性：** Colon Principle -- 可以将一棵子树合并为等价的单条链

**C++ 代码**：

```cpp
vector<int> adj[MAXN];
int sg[MAXN];

void dfs(int u, int fa) {
    sg[u] = 0;
    for (int v : adj[u]) {
        if (v == fa) continue;
        dfs(v, u);
        sg[u] ^= (sg[v] + 1);  // 子树 SG 值 + 1（加上连接边）
    }
}
// sg[root] != 0 → 先手必胜
```

**核心启示**：
- **Colon Principle 是树上博弈的基本工具**：子树可合并为等效链
- **sg[v]+1 表示加上连接边**：每条边贡献 1 到 SG 值
- **树形博弈归结为 DFS**：自底向上合并 SG 值

### 题目 12：反 Nim 游戏（HDU 4480）

> n 堆石子，取最后一颗者**负**（Misère Nim）。判断先手胜负。

**分析**：Misère Nim 与标准 Nim 的区别仅在于结论需要特判"全为 1"的情况。

> **核心观察：** 所有堆都 $\leq 1$ 时，堆数为奇数后手胜（最后一个被迫取的人输）
> **构造策略：** 若存在 $a[i] > 1$，则异或和 $\neq 0$ 先手必胜；若全 $\leq 1$，奇数堆后手胜
> **正确性：** 存在大堆时先手可控制进入全 1 的有利状态

**C++ 代码**：

```cpp
int n, a[MAXN];

void solve() {
    int xorSum = 0, maxVal = 0;
    for (int i = 0; i < n; i++) {
        xorSum ^= a[i];
        maxVal = max(maxVal, a[i]);
    }
    bool win;
    if (maxVal <= 1)
        win = (n % 2 == 0);  // 全是1，偶数堆先手胜
    else
        win = (xorSum != 0);  // 有大堆，异或和≠0先手胜
}
```

**核心启示**：
- **Misère 变体只改变终局判定**：大部分情况与正常 Nim 相同
- **特判"全为 1"的边界**：是 Misère Nim 的唯一不同之处
- **理解正常 Nim 是理解 Misère 的前提**：不要死记结论

### 题目 13：Every-SG 博弈（HDU 3595）

> 多个独立子博弈同时进行，每轮必须在**所有未结束的博弈**中各操作一步。最后一个结束博弈的人根据该博弈的胜负规则判定整体胜负。

**分析**：Every-SG 博弈不能简单异或。关键在于控制每个子博弈的**步数奇偶性**（持续步数最长的博弈决定最终命运）。

> **核心观察：** 最后结束的博弈决定胜负，因此要最大化/最小化博弈持续步数
> **构造策略：** $SG > 0$ 的博弈先手可控制步数奇偶；$SG = 0$ 的博弈后手可控制
> **正确性：** 先手在 $SG > 0$ 的博弈中选择最优持续步数来决定最终奇偶

**C++ 代码**：

```cpp
// Every-SG: 计算每个子博弈的 step（最长/最短结束步数）
int sg[MAXN], step[MAXN];

void calcStep(int u) {
    if (isEnd(u)) { sg[u] = 0; step[u] = 0; return; }
    // sg[u] > 0: step[u] = max{step[v]+1 | v 是后继且 sg[v]=0} 中奇数步
    // sg[u] = 0: step[u] = min{step[v]+1 | v 是后继} 中偶数步
}
// 所有子博弈中 step 最大的为奇数 → 先手必胜
```

**核心启示**：
- **Every-SG 不满足简单异或合成**：需要考虑步数维度
- **最后结束的博弈决定一切**：要控制持续时间
- **SG 值为正时掌握主动**：可以选择拖延或加速

### 🟣 挑战级 Level 5：综合应用与高级理论

### 题目 14：SG 函数打表与周期发现（CF 850C）

> 给定 n 堆石子，每次可从一堆中取 $k^2$ 个（$k = 1, 2, 3, ...$）。判断先手胜负。

**分析**：每堆的 SG 函数需要 mex 计算。先对小范围打表，发现 SG 函数的周期规律。

> **核心观察：** SG 值序列为 0,1,0,2,0,1,0,3,...（通过打表发现周期或规律）
> **构造策略：** 打表 $SG(0..N)$，找到周期 $T$ 后用公式计算大数的 SG 值
> **正确性：** 有限状态的 SG 函数必存在最终周期（Sprague-Grundy 周期性定理）

**C++ 代码**：

```cpp
int sg[MAXN];

void buildSG(int maxn) {
    memset(sg, 0, sizeof(sg));
    for (int i = 0; i <= maxn; i++) {
        set<int> S;
        for (int k = 1; k * k <= i; k++)
            S.insert(sg[i - k * k]);
        int res = 0;
        while (S.count(res)) res++;
        sg[i] = res;
    }
}

void solve() {
    buildSG(1000);  // 打表找规律
    int xorSum = 0;
    for (int i = 0; i < n; i++)
        xorSum ^= sg[a[i]];  // 或用周期公式
    // xorSum != 0 → 先手必胜
}
```

**核心启示**：
- **打表找规律是 SG 函数的核心技能**：竞赛中最实用的方法
- **SG 周期性定理**：有限转移的博弈 SG 函数最终具有周期性
- **先暴力后优化**：打表 → 发现周期 → 公式替代，经典三部曲

### 题目 15：组合博弈综合（CF 768E）

> 给定一棵树，每条边有权值 0 或 1。两人轮流选择一条权为 1 的边删除，使得分成的两个连通块中选一个保留。无法操作者负。最终问题可建模为 Green Hackenbush。

**分析**：这是 Green Hackenbush 在一般图上的应用。对于树的情况，利用 Colon Principle 自底向上计算 SG 值。

> **核心观察：** 权值为 1 的边参与博弈，权值为 0 的边等价于合并端点
> **构造策略：** 缩 0 边后在新树上计算 Hackenbush SG 值
> **正确性：** 0 边删除后两部分合并，不影响博弈；1 边是真正的博弈操作

**C++ 代码**：

```cpp
vector<pair<int,int>> adj[MAXN];  // {邻居, 边权}
int sg[MAXN];

void dfs(int u, int fa) {
    sg[u] = 0;
    for (auto [v, w] : adj[u]) {
        if (v == fa) continue;
        dfs(v, u);
        if (w == 1)
            sg[u] ^= (sg[v] + 1);  // 1 边贡献 SG
        else
            sg[u] ^= sg[v];        // 0 边直接合并
    }
}
// sg[root] != 0 → 先手必胜
```

**核心启示**：
- **Green Hackenbush 是博弈论的高级模型**：适用于图上删边类博弈
- **边权 0/1 的处理**：0 边合并，1 边贡献 SG+1
- **树上博弈的通用框架**：DFS + 异或合并，适应各种变体

## 五、常见技巧与思路总结

### 5.1 异或万能定律

几乎所有公平组合博弈都能归结为 SG 值的异或。当你面对博弈问题时，第一反应应该是"能否拆成独立子博弈并异或"。

> 多堆博弈 $\rightarrow$ 每堆求 SG $\rightarrow$ 异或所有 SG
> $SG = 0$ $\rightarrow$ 后手必胜；$SG \neq 0$ $\rightarrow$ 先手必胜

### 5.2 打表找规律

当无法直接推导 SG 函数时，对小规模数据暴力计算 SG 值，观察规律：

```cpp
// 通用 SG 打表框架
for (int i = 0; i <= N; i++) {
    set<int> S;
    for (auto nxt : transitions(i))
        S.insert(sg[nxt]);
    sg[i] = mex(S);
}
// 打印 sg[0..N] 观察周期或公式
```

### 5.3 对称策略

当博弈具有某种对称性时，先手（或后手）可以利用镜像操作保持对称，从而必胜。典型场景：环形博弈、二维棋盘博弈。

### 5.4 "有选择权即必胜"判定

当某一方在某步操作中有多种选择（可以选择对自己有利的方向），往往意味着该方必胜。典型应用：欧几里德博弈（a ≥ 2b 时有选择权）。

### 5.5 特殊博弈的快速判定

| 博弈类型 | 快速判定公式 |
|---------|------------|
| 经典 Nim | `XOR(a[i]) != 0` |
| 巴什博弈 | `n % (m+1) != 0` |
| 威佐夫 | `min != ⌊|diff|·φ⌋` |
| 阶梯 Nim | `XOR(奇数位) != 0` |
| Misère Nim | 全1特判 + 异或 |

## 七、终极总结与归纳

### 7.1 博弈论的本质

**博弈论的本质是状态空间的必胜/必败二分。** 每个局面要么是 N-position（先手必胜），要么是 P-position（后手必胜），没有第三种可能。SG 定理将复杂博弈分解为独立子博弈的异或合成，使我们能够用简洁的数学工具分析极其复杂的游戏。

### 7.2 核心要点

1. **P/N 判定是基础**：终态为 P，能到 P 的为 N，只能到 N 的为 P
2. **SG 定理是灵魂**：多个独立子博弈 = SG 值的异或
3. **Nim 是万博之源**：几乎所有公平组合博弈都可以归约到 Nim
4. **打表找规律是竞赛利器**：暴力 SG → 发现周期 → 公式化
5. **对称与构造并重**：有时不需要 SG，对称策略直接搞定
6. **变体记清区别**：Misère 仅改终局判定，阶梯仅看奇数位
