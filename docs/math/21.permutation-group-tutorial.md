## 一、什么是置换群

置换是集合到自身的双射。置换群是一组置换在复合运算下构成的群。在算法竞赛中，置换群主要用于解决**本质不同的计数问题**——当存在对称性（旋转、翻转等）时，用 Burnside 引理或 Polya 计数定理消除重复。

| 概念 | 定义 | 应用 |
|------|------|------|
| 置换 | 集合到自身的双射 | 状态变换建模 |
| 置换群 | 置换的集合 + 复合运算 | 对称性描述 |
| Burnside 引理 | 本质不同 = 各置换不动点平均 | 计数去重 |
| Polya 定理 | Burnside 的具体化（按循环计数） | 染色计数 |

## 二、核心思想与前置知识

### 2.1 置换的表示

置换可以用**循环分解**表示。例如 $(1\ 3\ 2)(4\ 5)$ 表示 $1 \to 3 \to 2 \to 1$，$4 \to 5 \to 4$。

### 2.2 Burnside 引理

设群 $G$ 作用在集合 $X$ 上，本质不同的元素数（轨道数）为：

$$|X/G| = \frac{1}{|G|} \sum_{g \in G} |X^g|$$

其中 $X^g = \{x \in X : g(x) = x\}$ 是 $g$ 的不动点集。

### 2.3 Polya 计数定理

用 $k$ 种颜色给 $n$ 个位置染色，本质不同的方案数为：

$$N = \frac{1}{|G|} \sum_{g \in G} k^{c(g)}$$

其中 $c(g)$ 是置换 $g$ 的循环个数。

```cpp
// Polya 计数骨架
long long polya(vector<vector<int>>& perms, int k, long long mod) {
    long long ans = 0;
    for (auto& perm : perms) {
        int cycles = count_cycles(perm);
        ans = (ans + qpow(k, cycles, mod)) % mod;
    }
    return ans * qpow(perms.size(), mod - 2, mod) % mod;
}

int count_cycles(vector<int>& perm) {
    int n = perm.size(), cnt = 0;
    vector<bool> vis(n, false);
    for (int i = 0; i < n; i++) {
        if (vis[i]) continue;
        cnt++;
        for (int j = i; !vis[j]; j = perm[j]) vis[j] = true;
    }
    return cnt;
}
```

### 2.4 旋转群与二面体群

- **旋转群** $C_n$：$n$ 次旋转，置换 $\sigma_k(i) = (i + k) \bmod n$，循环数 = $\gcd(k, n)$
- **二面体群** $D_n$：$n$ 次旋转 + $n$ 次翻转，共 $2n$ 个元素

## 三、分类与变体详解

### 3.1 项链问题（旋转等价）

```
群：旋转群 C_n
置换个数：n
第 k 个置换的循环数：gcd(k, n)
公式：N = (1/n) · ∑_{k=0}^{n-1} c^{gcd(k,n)}
优化：按 gcd 值分组 → N = (1/n) · ∑_{d|n} φ(n/d) · c^d
```

### 3.2 手镯问题（旋转 + 翻转等价）

```
群：二面体群 D_n
置换个数：2n
旋转部分：同项链
翻转部分：n 为奇数 → n 个翻转各有 (n+1)/2 个循环
          n 为偶数 → n/2 个有 n/2+1 个循环，n/2 个有 n/2 个循环
```

### 3.3 Burnside + DP

```
适用场景：不动点计数不是简单的 k^c(g)
方法：对每个置换 g，用 DP 计算 |X^g|
关键：不动点 = 同一循环内所有位置颜色相同 + 满足约束
```

### 3.4 图同构计数

```
群：对称群 S_n 作用在边集上
置换：顶点置换诱导边置换
循环计数：需计算边置换的循环数
难度：需要理解置换的诱导作用
```

### 题目 1：Polya 定理模板（洛谷 P4980）

> 用 $k$ 种颜色给长度为 $n$ 的项链染色，旋转等价视为相同，求方案数模素数 $p$。

**分析**：经典 Polya 项链问题。旋转群 $C_n$ 中第 $i$ 个旋转有 $\gcd(i, n)$ 个循环。

```
核心观察：旋转群 → 循环数 = gcd(i, n)
构造策略：N = (1/n) · ∑_{d|n} φ(n/d) · k^d
正确性：Polya 定理 + 按 gcd 分组优化
```

**C++ 代码**：

```cpp
long long solve(long long n, long long k, long long p) {
    long long ans = 0;
    for (long long d = 1; d * d <= n; d++) {
        if (n % d) continue;
        ans = (ans + euler_phi(n / d) % p * qpow(k, d, p)) % p;
        if (d != n / d)
            ans = (ans + euler_phi(d) % p * qpow(k, n / d, p)) % p;
    }
    return ans * qpow(n, p - 2, p) % p;
}
```

**核心启示**：
- **枚举约数**而非枚举旋转，将 $O(n)$ 优化为 $O(\sqrt{n})$
- $\varphi$ 函数在 Polya 中的自然出现

### 题目 2：手镯染色（经典变体）

> 同题目 1，但旋转和翻转都视为相同（二面体群）。

**分析**：在旋转的基础上加入翻转。翻转的循环数取决于 $n$ 的奇偶。

```
核心观察：二面体群 D_n = 旋转 + 翻转
构造策略：旋转部分同项链；翻转部分分 n 奇偶讨论
正确性：Burnside 引理对 D_n 的展开
```

**C++ 代码**：

```cpp
long long bracelet(long long n, long long k, long long p) {
    long long rot = solve(n, k, p) * n % p; // 旋转部分（未除 n）
    long long flip = 0;
    if (n % 2 == 1) {
        flip = n % p * qpow(k, (n + 1) / 2, p) % p;
    } else {
        flip = (n / 2 % p * qpow(k, n / 2 + 1, p) +
                n / 2 % p * qpow(k, n / 2, p)) % p;
    }
    return (rot + flip) % p * qpow(2 * n % p, p - 2, p) % p;
}
```

**核心启示**：
- 翻转轴的类型取决于 $n$ 的奇偶性
- 群的大小为 $2n$

### 题目 3：正方体染色（经典）

> 用 $k$ 种颜色给正方体的 6 个面染色，旋转等价视为相同。

**分析**：正方体旋转群有 24 个元素。枚举每种旋转的循环数。

```
核心观察：正方体旋转群 |G| = 24
构造策略：分类：恒等(1个,6循环)、面旋转90°/270°(6个,3循环)、面旋转180°(3个,4循环)、棱旋转180°(6个,3循环)、顶点旋转120°/240°(8个,2循环)
正确性：N = (1/24)(k^6 + 6k^3 + 3k^4 + 6k^3 + 8k^2)
```

**C++ 代码**：

```cpp
long long cube_coloring(long long k, long long p) {
    long long ans = (qpow(k, 6, p) + 6 * qpow(k, 3, p) % p +
                     3 * qpow(k, 4, p) % p + 6 * qpow(k, 3, p) % p +
                     8 * qpow(k, 2, p) % p) % p;
    return ans * qpow(24, p - 2, p) % p;
}
```

**核心启示**：
- 枚举旋转群元素并分类是关键
- 正多面体旋转群大小：正四 12、正六/正八 24、正十二/正二十 60

### 题目 4：Cards（洛谷 P1446）

> 给定 $n$ 张牌，用 3 种颜色染色（每种颜色数量固定为 $a, b, c$），给定若干置换构成置换群，求本质不同方案数。

**分析**：不能直接用 Polya（颜色数量有限制），需要 Burnside + DP 计算不动点。

```
核心观察：颜色数量有约束 → 不动点数需用 DP 计算
构造策略：对每个置换 g，同一循环内颜色相同 → 用循环大小做背包 DP
正确性：Burnside 引理 + 背包 DP
```

**C++ 代码**：

```cpp
long long count_fixed(vector<int>& cycle_sizes, int a, int b, int c, long long p) {
    // dp[i][j] = 用 i 个 A 色和 j 个 B 色的方案数
    int m = cycle_sizes.size();
    vector<vector<long long>> dp(a + 1, vector<long long>(b + 1, 0));
    dp[0][0] = 1;
    for (int sz : cycle_sizes) {
        // 这个循环可以染 A(消耗 sz 个 A) 或 B 或 C
        for (int i = a; i >= 0; i--)
            for (int j = b; j >= 0; j--) {
                long long v = 0;
                if (i >= sz) v = (v + dp[i - sz][j]) % p;
                if (j >= sz) v = (v + dp[i][j - sz]) % p;
                // C 色不消耗 A 或 B
                dp[i][j] = v + dp[i][j]; // 保留不选此循环的部分...
            }
    }
    return dp[a][b];
}
```

**核心启示**：
- 当 Polya 的 $k^{c(g)}$ 不适用时，用 **Burnside + DP** 替代
- 循环大小做背包的思想非常重要

### 题目 5：黑白瓷砖（洛谷 P2561）

> 用黑白两色给正六边形网格的单元染色，旋转等价，求方案数。

**分析**：确定正六边形的旋转群（$C_6$ 或 $D_6$），枚举每种旋转的循环分解。

```
核心观察：正六边形旋转群 |C_6| = 6
构造策略：枚举 6 种旋转，对每种旋转手动求循环数
正确性：Polya 定理直接应用
```

**C++ 代码**：

```cpp
// 对于 n 个格子的正六边形，旋转 k*60° 后循环数手动计算
long long solve(int n, long long p) {
    // 具体循环数取决于格子编号与旋转的对应关系
    // 一般做法：编号每个格子，模拟旋转，求循环分解
    long long ans = 0;
    for (int k = 0; k < 6; k++) {
        int cycles = count_cycles_rotation(n, k);
        ans = (ans + qpow(2, cycles, p)) % p;
    }
    return ans * qpow(6, p - 2, p) % p;
}
```

**核心启示**：
- 非标准几何形状需要**手动构建置换**
- 编号 + 模拟旋转是通用方法

### 题目 6：涂色问题（AcWing 3193）

> $n$ 个珠子的项链，$k$ 种颜色，旋转等价，且相邻珠子颜色不同。

**分析**：Burnside + 限制条件。不动点要求同一循环内颜色相同且相邻不同。

```
核心观察：相邻不同 + 同循环同色 → 约束在循环图上
构造策略：对旋转 k 步，不动点数 = 环形图的色多项式 P(G, k) 在循环缩并后的值
正确性：色多项式理论
```

**C++ 代码**：

```cpp
// 环形色多项式：P(C_n, k) = (k-1)^n + (-1)^n * (k-1)
long long necklace_proper(int n, int k, long long p) {
    long long ans = 0;
    for (int d = 1; d * d <= n; d++) {
        if (n % d) continue;
        // 旋转 d 步：n/gcd(d,n) = n/d 个循环，每个长 d
        // 不动点 = 缩并后环的色多项式
        int m = n / d; // 循环个数 = gcd(d, n) ... 这里 d|n
        long long fix = (qpow(k - 1, d, p) + (d % 2 == 0 ? 1 : -1) * (k - 1) % p + p) % p;
        ans = (ans + euler_phi(n / d) % p * fix) % p;
        // 对称处理 n/d
    }
    return ans * qpow(n, p - 2, p) % p;
}
```

**核心启示**：
- Burnside + 色多项式的结合
- 不是所有计数问题都能用简单的 $k^{c(g)}$

### 题目 7：有色图（洛谷 P4128）

> $n$ 个顶点的完全图，用 $k$ 种颜色给边染色，图同构等价，求方案数。

**分析**：群 = $S_n$ 作用在 $\binom{n}{2}$ 条边上。对每个顶点置换，计算诱导的边置换的循环数。

```
核心观察：顶点置换 σ 诱导边置换：边 {u,v} → {σ(u), σ(v)}
构造策略：枚举 σ 的循环型 (1^{a1} 2^{a2} ... n^{an})
         边置换循环数 = ∑_{i<j} gcd(c_i, c_j) + ∑_i ⌊c_i/2⌋
正确性：边置换循环数的公式推导
```

**C++ 代码**：

```cpp
// 枚举循环型，计算贡献
// 循环型 (c1, c2, ..., cm) 的边置换循环数：
// ∑_{i<j} gcd(c_i, c_j) + ∑_i ⌊c_i / 2⌋
// 该循环型的置换个数：n! / ∏(c_i · cnt[c_i]!)
void dfs(int remain, int last, vector<int>& cycles, long long& ans,
         int n, int k, long long p) {
    if (remain == 0) {
        int edge_cycles = 0;
        for (int i = 0; i < cycles.size(); i++) {
            edge_cycles += cycles[i] / 2;
            for (int j = i + 1; j < cycles.size(); j++)
                edge_cycles += __gcd(cycles[i], cycles[j]);
        }
        long long weight = fac[n]; // 该循环型的置换个数
        // ... 除以各循环长度和重复计数
        ans = (ans + weight * qpow(k, edge_cycles, p)) % p;
    }
    for (int c = last; c <= remain; c++) {
        cycles.push_back(c);
        dfs(remain - c, c, cycles, ans, n, k, p);
        cycles.pop_back();
    }
}
```

**核心启示**：
- **图同构计数**是 Polya 的高级应用
- 关键在于边置换循环数的公式

### 题目 9：环形染色进阶（CF 710F）

> 长度为 $n$ 的环，$k$ 种颜色，相邻不同色，旋转等价。$n$ 可达 $10^{18}$。

**分析**：需要高效计算 $\sum_{d \mid n} \varphi(n/d) \cdot f(d)$，其中 $f(d)$ 是长度 $d$ 的环的合法染色数。

```
核心观察：f(d) = (k-1)^d + (-1)^d · (k-1)（环的色多项式）
构造策略：枚举 n 的因子 d，计算 φ(n/d) · f(d)
正确性：Polya 定理 + 色多项式
```

**C++ 代码**：

```cpp
long long solve(long long n, long long k, long long p) {
    long long ans = 0;
    // 枚举 n 的所有因子
    for (long long d = 1; d * d <= n; d++) {
        if (n % d) continue;
        long long f_d = (qpow(k - 1, d, p) + (d % 2 ? p - (k-1) % p : (k-1) % p)) % p;
        ans = (ans + euler_phi(n / d) % p * f_d) % p;
        if (d != n / d) {
            long long f_nd = (qpow(k - 1, n / d, p) + ((n/d) % 2 ? p - (k-1) % p : (k-1) % p)) % p;
            ans = (ans + euler_phi(d) % p * f_nd) % p;
        }
    }
    return ans * qpow(n % p, p - 2, p) % p;
}
```

**核心启示**：
- $n$ 极大时仍可枚举因子（因子个数 $O(\sqrt{n})$）
- 色多项式 + Polya 的结合

### 题目 10：Burnside + 矩阵快速幂（CF 954I）

> 长度 $n$ 的项链，$k$ 种颜色，旋转等价，且满足某些颜色对不能相邻。求方案数。

**分析**：对每个旋转 $d$ 步（$d \mid n$），不动点数需要用矩阵快速幂计算。

```
核心观察：不动点 = 循环内颜色相同且满足邻接约束 → 合法环计数
构造策略：转移矩阵 A 表示相邻颜色约束，不动点数 = tr(A^d)
正确性：矩阵幂的迹 = 所有长度 d 的合法闭路数
```

**C++ 代码**：

```cpp
long long solve(long long n, int k, vector<pair<int,int>>& forbidden, long long p) {
    // 建 k×k 转移矩阵 A
    Matrix A(k, k, 1); // 全 1
    for (auto [u, v] : forbidden) A[u][v] = A[v][u] = 0;
    long long ans = 0;
    for (long long d = 1; d * d <= n; d++) {
        if (n % d) continue;
        Matrix Ad = mat_pow(A, d, p);
        long long tr = trace(Ad, p); // 矩阵迹
        ans = (ans + euler_phi(n / d) % p * tr) % p;
        if (d != n / d) {
            Matrix And = mat_pow(A, n / d, p);
            ans = (ans + euler_phi(d) % p * trace(And, p)) % p;
        }
    }
    return ans * qpow(n % p, p - 2, p) % p;
}
```

**核心启示**：
- **矩阵迹**计算闭路数是优雅的技巧
- Burnside + 矩阵快速幂是处理约束 Polya 的通用方法

### 题目 11：多面体染色（POJ 2409）

> 正 $n$ 边形的旋转和翻转（二面体群），$k$ 种颜色，求本质不同方案。

**分析**：完整的二面体群 $D_n$ Polya 计数。

```
核心观察：D_n 有 n 个旋转 + n 个翻转
构造策略：旋转：∑_{d|n} φ(n/d)·k^d；翻转：分 n 奇偶
正确性：标准 Polya 推导
```

**C++ 代码**：

```cpp
long long dihedral(int n, int k) {
    long long rot_sum = 0;
    for (int d = 1; d <= n; d++)
        if (n % d == 0) rot_sum += euler_phi(n / d) * pow(k, d);
    long long flip_sum = 0;
    if (n % 2 == 1) flip_sum = n * pow(k, (n + 1) / 2);
    else flip_sum = n / 2 * (pow(k, n / 2 + 1) + pow(k, n / 2));
    return (rot_sum + flip_sum) / (2 * n);
}
```

**核心启示**：
- 二面体群是最常见的几何对称群
- 注意整除性（答案保证是整数）

### 题目 12：项链 + 容斥（CSES Necklace）

> $n$ 颗珠子的项链，恰好用 $k$ 种颜色（每种至少用一次），旋转等价。

**分析**：先用 Polya 求"至多 $k$ 种颜色"的方案，再容斥得"恰好 $k$ 种"。

```
核心观察：恰好 k 种 = ∑_{i=0}^{k} (-1)^i · C(k,i) · f(k-i)
构造策略：f(c) = Polya(n, c)，容斥减去缺少某种颜色的情况
正确性：容斥原理 + Polya
```

**C++ 代码**：

```cpp
long long solve(int n, int k, long long p) {
    long long ans = 0;
    for (int i = 0; i <= k; i++) {
        long long polya_val = polya_necklace(n, k - i, p);
        long long sign = (i % 2 == 0) ? 1 : p - 1;
        ans = (ans + sign * C(k, i, p) % p * polya_val) % p;
    }
    return ans;
}
```

**核心启示**：
- **Polya + 容斥**处理"恰好"型计数
- 容斥常与其它计数工具结合

### 🟣 挑战级 Level 5：极限难度

### 题目 13：图的同构计数进阶（OEIS A001349）

> 求 $n$ 个顶点的连通非同构简单图个数。

**分析**：先用 Polya 求所有非同构图（题目 8），再用容斥去掉不连通的。

```
核心观察：连通图计数 = 所有图计数 - 不连通的
构造策略：f(n) = g(n) - ∑_{k=1}^{n-1} C(n-1,k-1)·f(k)·h(n-k)
正确性：容斥：选一个顶点所在连通分量大小为 k
```

**C++ 代码**：

```cpp
// g[n] = n 顶点非同构图总数（Polya）
// f[n] = n 顶点连通非同构图数
void solve(int n) {
    for (int i = 1; i <= n; i++) {
        f[i] = g[i];
        for (int k = 1; k < i; k++)
            f[i] -= C(i - 1, k - 1) * f[k] * g_labeled(i - k); // 需要处理标号/无标号
    }
}
```

**核心启示**：
- 连通图计数 = 总图计数 - 不连通容斥
- 较大 $n$ 需要更高效的算法

### 题目 14：对称群上的 Burnside（CF 建模题）

> 给定棋盘和若干变换，变换构成群，求本质不同的棋盘状态数。

**分析**：通用 Burnside 框架：枚举群中每个元素，计算不动点数。

```
核心观察：给定群 → 直接应用 Burnside
构造策略：群可能由生成元给出 → 先展开群（BFS/DFS）
正确性：Burnside 引理
```

**C++ 代码**：

```cpp
// 给定生成元，展开群的所有元素
vector<vector<int>> expand_group(vector<vector<int>>& gens, int n) {
    set<vector<int>> seen;
    queue<vector<int>> q;
    vector<int> id(n);
    iota(id.begin(), id.end(), 0);
    q.push(id); seen.insert(id);
    while (!q.empty()) {
        auto cur = q.front(); q.pop();
        for (auto& g : gens) {
            vector<int> nxt(n);
            for (int i = 0; i < n; i++) nxt[i] = cur[g[i]];
            if (!seen.count(nxt)) { seen.insert(nxt); q.push(nxt); }
        }
    }
    return vector<vector<int>>(seen.begin(), seen.end());
}
```

**核心启示**：
- 群可能不以显式形式给出，需要从**生成元**展开
- 展开后标准 Burnside

### 题目 15：带权 Polya（加权版 Burnside）

> 用颜色 1..k 给 $n$ 个位置染色，每种颜色有权重 $w_i$，旋转等价，求所有本质不同方案的权重和。

**分析**：带权 Burnside：$\sum_{g \in G} \sum_{x: g(x)=x} \text{weight}(x) / |G|$。

```
核心观察：权重 = 所有循环权重之积 = ∏_j w_{color of cycle j}
构造策略：对置换 g 的 c(g) 个循环，每个循环贡献 ∑w_i → 总贡献 = (∑w_i)^{c(g)}
正确性：加权 Polya 定理
```

**C++ 代码**：

```cpp
long long weighted_polya(long long n, long long w[], int k, long long p) {
    long long W = 0;
    for (int i = 0; i < k; i++) W = (W + w[i]) % p;
    long long ans = 0;
    for (long long d = 1; d * d <= n; d++) {
        if (n % d) continue;
        ans = (ans + euler_phi(n / d) % p * qpow(W, d, p)) % p;
        if (d != n / d)
            ans = (ans + euler_phi(d) % p * qpow(W, n / d, p)) % p;
    }
    return ans * qpow(n % p, p - 2, p) % p;
}
```

**核心启示**：
- 带权 Polya 中 $k^{c(g)}$ 变为 $(\sum w_i)^{c(g)}$
- 生成函数视角：Polya 本质是 Cycle Index 代入颜色权重

## 五、常见技巧与思路总结

### 5.1 按循环型分组

$S_n$ 中同一循环型的置换数 = $\frac{n!}{\prod c_i \cdot \prod (\text{cnt}[c_i])!}$。

### 5.2 枚举因子优化

项链问题中 $\sum_{k=0}^{n-1} f(\gcd(k,n)) = \sum_{d \mid n} \varphi(n/d) \cdot f(d)$。

### 5.3 边置换循环数公式

顶点置换循环型 $(c_1, \ldots, c_m)$ 的边置换循环数 = $\sum_{i<j} \gcd(c_i, c_j) + \sum_i \lfloor c_i/2 \rfloor$。

### 5.4 群的生成与展开

群可能由生成元给出，用 BFS 展开所有元素，注意群大小可能很大。

### 5.5 Burnside vs Polya

Burnside 更通用（不动点可以 DP 计算），Polya 是 Burnside 在无约束染色时的特化。

## 七、终极总结与归纳

### 7.1 置换群的本质（一句话）

置换群通过 Burnside 引理将**对称性下的计数**归结为**不动点的平均**——理解群的结构就能解决"本质不同"的问题。

### 7.2 核心要点

1. **Burnside 引理**：轨道数 = 不动点总数 / 群的大小
2. **Polya 定理**：无约束染色时，不动点数 = $k^{c(g)}$
3. **循环分解**是置换的核心表示
4. **旋转群**循环数 = $\gcd(k, n)$，按因子分组优化
5. **图同构**需要计算边置换循环数
6. **有约束**时 Burnside + DP 或矩阵快速幂
