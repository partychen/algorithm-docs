## 一、什么是最大公约数

**最大公约数**（Greatest Common Divisor，简称 GCD），是指两个或多个整数共有的约数中最大的那个。例如 `gcd(12, 18) = 6`，因为 6 是同时整除 12 和 18 的最大正整数。

GCD 是数论中最基础也最核心的概念之一。它不仅是数学竞赛的常客，更是密码学（RSA）、计算几何（格点计数）、组合数学（分数化简）等领域的底层工具。

> **一句话概括：GCD 是两个数"最大的公共度量单位"，所有公因子都是它的因子。**

### 与相关概念的对比

| 概念 | 定义 | 记号 | 与 GCD 的关系 |
|------|------|------|---------------|
| 最大公约数 GCD | 最大的公共因子 | `gcd(a, b)` | — |
| 最小公倍数 LCM | 最小的公共倍数 | `lcm(a, b)` | `lcm(a,b) = a * b / gcd(a,b)` |
| 互质 | `gcd(a, b) = 1` | `a ⊥ b` | GCD 判定 |
| 欧拉函数 | `[1, n]` 中与 `n` 互质的数的个数 | `φ(n)` | 依赖 GCD 定义 |
| 扩展欧几里得 | 求 `ax + by = gcd(a,b)` 的整数解 | `exgcd` | GCD 的扩展应用 |

## 二、核心思想与前置知识

### 2.1 GCD 的基本性质

GCD 有如下重要性质，务必牢记：

1. **交换律**：`gcd(a, b) = gcd(b, a)`
2. **结合律**：`gcd(a, b, c) = gcd(gcd(a, b), c)`
3. **辗转关系**：`gcd(a, b) = gcd(b, a mod b)`（欧几里得算法的基础）
4. **与 LCM 的乘积关系**：`gcd(a, b) * lcm(a, b) = a * b`（当 `a, b > 0`）
5. **分配律**：`gcd(a*c, b*c) = c * gcd(a, b)`（当 `c > 0`）
6. **互质传递**：若 `gcd(a, b) = 1` 且 `a | bc`，则 `a | c`
7. **线性组合**：`gcd(a, b)` 是 `a` 和 `b` 的最小正线性组合，即必存在整数 `x, y` 使 `ax + by = gcd(a, b)`（裴蜀定理）

### 2.2 欧几里得算法（辗转相除法）

欧几里得算法是求 GCD 的经典方法，基于性质 3：反复用余数替换，直到余数为 0。

```
gcd(48, 18) 的计算过程：

  48 = 2 × 18 + 12
  18 = 1 × 12 + 6
  12 = 2 × 6  + 0
                ↑ 余数为 0，上一步的余数 6 即为 gcd
```

时间复杂度为 `O(log(min(a, b)))`，因为每两步至少把较小数减半。

**核心框架代码**：

```cpp
long long gcd(long long a, long long b) {
    return b == 0 ? a : gcd(b, a % b);
}

long long lcm(long long a, long long b) {
    return a / gcd(a, b) * b;  // 先除后乘防溢出
}
```

### 2.3 裴蜀定理（Bezout's Identity）

> 对于任意整数 `a, b`，必存在整数 `x, y` 使得 `ax + by = gcd(a, b)`。

推论：`ax + by = c` 有整数解的充要条件是 `gcd(a, b) | c`。

这是扩展欧几里得算法的理论基础。

### 2.4 扩展欧几里得算法

在求 `gcd(a, b)` 的同时，求出 `ax + by = gcd(a, b)` 的一组整数解 `(x, y)`。

```cpp
long long exgcd(long long a, long long b, long long &x, long long &y) {
    if (b == 0) { x = 1; y = 0; return a; }
    long long g = exgcd(b, a % b, y, x);
    y -= (a / b) * x;
    return g;
}
```

通解公式：若 `(x0, y0)` 是一组特解，则所有解为：
- `x = x0 + k * (b / gcd(a, b))`
- `y = y0 - k * (a / gcd(a, b))`

## 三、分类与变体详解

### 3.1 欧几里得算法（辗转相除法）

最经典的 GCD 算法，利用 `gcd(a, b) = gcd(b, a % b)` 递归求解。

```
核心观察：a mod b 比 b 小，问题规模持续缩小
构造策略：递归/迭代取模，直到余数为 0
正确性：每一步 gcd 不变，余数递减到 0 时即为答案
时间复杂度：O(log(min(a, b)))
```

```cpp
long long gcd(long long a, long long b) {
    while (b) { a %= b; swap(a, b); }
    return a;
}
```

### 3.2 扩展欧几里得算法（exgcd）

在辗转相除的回溯过程中同步计算线性组合系数。

```
核心观察：gcd(a, b) = gcd(b, a%b)，且线性系数可以递推回传
构造策略：递归到 base case 后逐层回推 x, y
正确性：每层满足 a*x + b*y = gcd(a, b)
```

```cpp
long long exgcd(long long a, long long b, long long &x, long long &y) {
    if (b == 0) { x = 1; y = 0; return a; }
    long long g = exgcd(b, a % b, y, x);
    y -= (a / b) * x;
    return g;
}
```

**典型应用**：解线性同余方程 `ax ≡ c (mod m)`、求模逆元、中国剩余定理。

### 3.3 更相减损术

中国古代《九章算术》中的算法：反复用大数减小数，直到两数相等。

```
核心观察：gcd(a, b) = gcd(a - b, b)（当 a > b）
构造策略：每次用大数减小数
正确性：差与较小数的 GCD 不变
缺点：对差距大的数（如 gcd(10^9, 1)）退化为 O(max(a,b))
```

```cpp
long long gcd_sub(long long a, long long b) {
    while (a != b) {
        if (a > b) a -= b;
        else b -= a;
    }
    return a;
}
```

### 3.4 Stein 算法（二进制 GCD）

利用位运算加速，避免大数取模运算，在处理高精度 GCD 时非常有用。

```
核心观察：
  - gcd(2a, 2b) = 2 * gcd(a, b)
  - gcd(2a, b) = gcd(a, b)  （b 为奇数）
  - gcd(a, b) = gcd(|a-b|, min(a,b))  （a, b 均为奇数）
构造策略：先提出公共 2 因子，再用减法处理奇数
正确性：每轮至少一个操作数减半
时间复杂度：O(log(max(a, b))^2)
```

```cpp
long long stein(long long a, long long b) {
    if (a == 0) return b;
    if (b == 0) return a;
    int shift = __builtin_ctzll(a | b);  // 公共的 2 因子
    a >>= __builtin_ctzll(a);
    do {
        b >>= __builtin_ctzll(b);
        if (a > b) swap(a, b);
        b -= a;
    } while (b);
    return a << shift;
}
```

### 3.5 多个数的 GCD

利用结合律 `gcd(a, b, c) = gcd(gcd(a, b), c)` 逐个合并即可。

```cpp
long long multi_gcd(long long a[], int n) {
    long long g = a[0];
    for (int i = 1; i < n; i++) {
        g = gcd(g, a[i]);
        if (g == 1) return 1;  // 提前终止优化
    }
    return g;
}
```

### 3.6 GCD 相关重要性质与定理

| 性质/定理 | 内容 | 应用场景 |
|-----------|------|----------|
| 裴蜀定理 | `ax + by = c` 有解 ⟺ `gcd(a,b) \| c` | 判断线性方程可解性 |
| GCD-LCM 关系 | `gcd(a,b) * lcm(a,b) = a * b` | 互转计算 |
| 欧拉函数 | `φ(n) = n * ∏(1 - 1/p)` | 与 `n` 互质的数的个数 |
| 莫比乌斯函数 | `∑_{d\|n} μ(d) = [n=1]` | GCD 求和反演 |
| GCD 卷积 | `∑_{gcd(i,j)=d} f(i)g(j)` | 数论函数卷积 |

### 变体对比总表

| 算法 | 时间复杂度 | 适用场景 | 核心操作 |
|------|-----------|----------|----------|
| 欧几里得算法 | `O(log min(a,b))` | 通用 | 取模 |
| 扩展欧几里得 | `O(log min(a,b))` | 需要线性组合系数 | 取模 + 回推系数 |
| 更相减损术 | `O(max(a,b))` 最坏 | 教学/理解原理 | 减法 |
| Stein 算法 | `O(log²max(a,b))` | 高精度 / 避免取模 | 位运算 + 减法 |
| 多数 GCD | `O(n log V)` | 数组求 GCD | 逐个合并 |

### 题目 1：快速 GCD（洛谷 P5435）

> 题意：给定两个长度为 `n` 的数组 `a[]` 和 `b[]`，对每个 `i` 求 `∑_{j=1}^{n} gcd(a[i], b[j]) * j`，对 `998244353` 取模。

**分析**：朴素做法 `O(n²log V)` 会超时。可以对 `b[]` 按因子预处理：枚举每个因子 `d`，记录 `b[j]` 中含因子 `d` 的 `j` 的加权和。然后对每个 `a[i]`，枚举其因子并用莫比乌斯反演或容斥计算贡献。

```
核心观察：gcd(a[i], b[j]) 可以按因子分解后聚合计算
构造策略：预处理 b 数组的因子贡献，对 a[i] 枚举因子求和
正确性：利用 gcd 的因子分解性质，将 O(n²) 转化为 O(n√V)
```

**C++ 代码**：

```cpp
const int MAXV = 1000001;
const int MOD = 998244353;
long long f[MAXV];  // f[d] = sum of j where d | b[j]
int a[MAXV], b[MAXV];

void solve(int n) {
    for (int j = 1; j <= n; j++)
        for (int d = 1; d * d <= b[j]; d++)
            if (b[j] % d == 0) {
                f[d] = (f[d] + (long long)j) % MOD;
                if (d != b[j] / d)
                    f[b[j] / d] = (f[b[j] / d] + (long long)j) % MOD;
            }
    // 对每个 a[i]，枚举因子 d，利用莫比乌斯函数做容斥
    // ans[i] = sum_{d | a[i]} phi(d) * f[d]  （利用 euler 反演）
}
```

答案位置：对每个 `i` 输出 `ans[i] % MOD`

**核心启示**：
- **GCD 求和问题**常用"枚举因子 + 反演"的思路，将二重循环降维
- **预处理因子贡献表**是处理批量 GCD 查询的核心工具
- 当值域 `V` 不大时，按因子聚合的复杂度 `O(n√V)` 远优于暴力

### 题目 2：缘分（洛谷 P5436）

> 题意：给定 `n` 个正整数，求它们的最大公约数。

**分析**：经典的多数 GCD 问题，利用结合律逐个合并即可。

```
核心观察：gcd(a1, a2, ..., an) = gcd(gcd(a1, a2), a3, ..., an)
构造策略：从左到右逐个合并，维护当前 GCD
正确性：GCD 满足结合律，逐个合并等价于一次性求解
```

**C++ 代码**：

```cpp
long long gcd(long long a, long long b) {
    return b == 0 ? a : gcd(b, a % b);
}

void solve(int n, long long a[]) {
    long long g = a[0];
    for (int i = 1; i < n; i++)
        g = gcd(g, a[i]);
    // g 即为答案
}
```

答案位置：变量 `g`

**核心启示**：
- **多数 GCD 直接逐个合并**，代码极简
- 若中途 `g` 已变为 1，可提前终止（因为 `gcd(1, x) = 1`）
- 这是几乎所有 GCD 题的基础操作

### 题目 3：最大公约数和最小公倍数问题（洛谷 P1029）

> 题意：给定 `x` 和 `y`，求满足 `gcd(a, b) = x` 且 `lcm(a, b) = y` 的正整数对 `(a, b)` 的个数（`a ≤ b`）。

**分析**：由 `gcd * lcm = a * b` 以及 `x | y` 可知，设 `a = x * p`，`b = x * q`，则 `p * q = y / x` 且 `gcd(p, q) = 1`。问题转化为求 `y / x` 的互质因子对数。

```
核心观察：a = x*p, b = x*q，其中 p*q = y/x 且 gcd(p, q) = 1
构造策略：枚举 y/x 的因子 p，检查 gcd(p, y/x/p) == 1
正确性：GCD-LCM 乘积公式 + 互质分解
```

**C++ 代码**：

```cpp
long long gcd(long long a, long long b) {
    return b == 0 ? a : gcd(b, a % b);
}

int solve(long long x, long long y) {
    if (y % x != 0) return 0;
    long long m = y / x;
    int cnt = 0;
    for (long long p = 1; p * p <= m; p++)
        if (m % p == 0 && gcd(p, m / p) == 1)
            cnt++;
    return cnt;
}
```

答案位置：`solve(x, y)` 的返回值

**核心启示**：
- **GCD 和 LCM 的乘积关系**是此类题的核心工具
- 将 `a, b` 提取公因子 `x` 后，问题转化为"互质因子对"的计数
- 枚举因子 + 互质判定是经典组合

### 题目 4：GCD 与 LCM（AcWing / 经典题）

> 题意：给定 `n` 个正整数，求它们的 GCD 和 LCM。

**分析**：GCD 逐个合并；LCM 同理，`lcm(a, b) = a / gcd(a, b) * b`，注意先除后乘防溢出。

```
核心观察：GCD 和 LCM 都满足结合律，可逐个合并
构造策略：维护当前 gcd 和 lcm，逐个更新
正确性：结合律保证逐个合并与一次性求解等价
```

**C++ 代码**：

```cpp
long long gcd(long long a, long long b) {
    return b == 0 ? a : gcd(b, a % b);
}

void solve(int n, long long a[]) {
    long long g = a[0], l = a[0];
    for (int i = 1; i < n; i++) {
        g = gcd(g, a[i]);
        l = l / gcd(l, a[i]) * a[i];  // 先除后乘防溢出
    }
    // g 为 GCD，l 为 LCM
}
```

答案位置：变量 `g` 和 `l`

**核心启示**：
- **先除后乘**是 LCM 计算防溢出的重要技巧
- GCD 和 LCM 的配合使用是数论题的基本功
- 多数的 LCM 可能非常大，有时需要高精度或取模

### 题目 5：又是毕业季 II（洛谷 P1414）

> 题意：给定 `n` 个正整数，对于每个 `k`（`1 ≤ k ≤ n`），求从中选 `k` 个数使得它们的 GCD 最大。

**分析**：反向思考——枚举可能的 GCD 值 `d`，统计有多少个数是 `d` 的倍数。如果 `d` 的倍数 `≥ k` 个，则选 `k` 个数可以达到 GCD 至少为 `d`。从大到小枚举 `d`，对每个 `k` 记录最大的可行 `d`。

```
核心观察：如果 d 的倍数有 cnt 个，则选 k ≤ cnt 个数的 GCD 至少为 d
构造策略：枚举因子 d，统计倍数个数 cnt[d]，从大到小更新答案
正确性：选的数都是 d 的倍数，GCD 至少为 d；取最大 d 即为最优
```

**C++ 代码**：

```cpp
const int MAXV = 1000001;
int cnt[MAXV];   // cnt[d] = 数组中 d 的倍数个数
int ans[MAXV];   // ans[k] = 选 k 个数的最大 GCD

void solve(int n, int a[]) {
    int mx = *max_element(a, a + n);
    for (int i = 0; i < n; i++) cnt[a[i]]++;
    // 统计 d 的倍数个数
    for (int d = 1; d <= mx; d++) {
        int tot = 0;
        for (int j = d; j <= mx; j += d)
            tot += cnt[j];
        // tot 个数可被 d 整除 → 选 k ≤ tot 个数 GCD 至少为 d
        for (int k = 1; k <= tot && !ans[k]; k++)
            ans[k] = d;
    }
}
```

答案位置：`ans[k]` 为选 `k` 个数的最大 GCD

**核心启示**：
- **"枚举 GCD 值，统计倍数"**是处理"最大 GCD"问题的经典套路
- 枚举因子并统计倍数的复杂度为 `O(V log V)`（调和级数）
- 从大到小枚举保证贪心的正确性

### 题目 6：互质对数（经典题）

> 题意：给定 `n`，求 `1 ≤ i < j ≤ n` 中满足 `gcd(i, j) = 1` 的有序对数。

**分析**：答案等于 `∑_{i=1}^{n} φ(i)` 再乘 2 减 1（因为 `gcd(1,1)=1` 只算一次）。利用欧拉函数的线性筛即可。

```
核心观察：gcd(i, j) = 1 的对数与欧拉函数直接相关
构造策略：线性筛求欧拉函数，前缀和统计答案
正确性：φ(n) = [1, n-1] 中与 n 互质的数的个数
```

**C++ 代码**：

```cpp
const int MAXN = 10000001;
int phi[MAXN], primes[MAXN], pcnt;
bool not_prime[MAXN];

void euler_sieve(int n) {
    phi[1] = 1;
    for (int i = 2; i <= n; i++) {
        if (!not_prime[i]) { primes[pcnt++] = i; phi[i] = i - 1; }
        for (int j = 0; j < pcnt && (long long)i * primes[j] <= n; j++) {
            not_prime[i * primes[j]] = true;
            if (i % primes[j] == 0) {
                phi[i * primes[j]] = phi[i] * primes[j];
                break;
            }
            phi[i * primes[j]] = phi[i] * (primes[j] - 1);
        }
    }
}
// ans = 2 * (phi[1] + phi[2] + ... + phi[n]) - 1
```

答案位置：`2 * sum(phi[1..n]) - 1`

**核心启示**：
- **欧拉函数与互质计数**是天然对应的
- **线性筛**可以在 `O(n)` 内求出所有 `φ(i)`
- 许多 GCD 求和/计数问题最终都会归结到欧拉函数

### 题目 7：SuperGCD（洛谷 P2152，SDOI2009）

> 题意：给定两个非常大的正整数 `a, b`（可达 $10^{10000}$），求 `gcd(a, b)`。

**分析**：数字太大，无法直接取模。使用 Stein 算法（更相减损术 + 二进制优化）配合高精度。核心思想：提取公共 2 因子后，两个奇数的 GCD 等于它们差的绝对值与较小值的 GCD。

```
核心观察：高精度取模代价高，但高精度减法和除以 2（右移）很快
构造策略：Stein 算法 —— 提取 2 因子 + 奇数做减法
正确性：gcd(2a, 2b) = 2*gcd(a,b); gcd(2a, odd_b) = gcd(a, odd_b)
```

**C++ 代码**：

```cpp
// 高精度 BigInt 需支持：比较、减法、除以2、乘以2、判奇偶
BigInt stein(BigInt a, BigInt b) {
    if (a == 0) return b;
    if (b == 0) return a;
    int shift = 0;
    while (a.is_even() && b.is_even()) {
        a.div2(); b.div2(); shift++;
    }
    while (a.is_even()) a.div2();
    while (b.is_even()) b.div2();
    // 此时 a, b 都是奇数
    while (a != b) {
        if (a < b) swap(a, b);
        a = a - b;
        while (a.is_even()) a.div2();
    }
    return a.lshift(shift);  // 补回公共的 2 因子
}
```

答案位置：`stein(a, b)` 的返回值

**核心启示**：
- **Stein 算法的核心优势**在于只用减法和除以 2，完美避开高精度除法
- 高精度场景下，选择合适的 GCD 算法比硬套欧几里得更重要
- 记住提取公共 2 因子是 Stein 算法的第一步

### 题目 8：Hankson 的趣味题（洛谷 P1072）

> 题意：给定 `a0, a1, b0, b1`，求满足 `gcd(x, a0) = a1` 且 `lcm(x, b0) = b1` 的正整数 `x` 的个数。

**分析**：由条件可知 `a1 | x` 且 `x | b1`。设 `x = a1 * k`，则 `k | (b1 / a1)`，且需满足 `gcd(k, a0/a1) = 1` 和 `lcm(a1*k, b0) = b1`。枚举 `b1` 的因子即可。

```
核心观察：x 必须是 a1 的倍数且是 b1 的因子
构造策略：枚举 b1 的因子 x，逐一验证 gcd 和 lcm 条件
正确性：直接检验充要条件，枚举范围由 b1 的因子个数决定
```

**C++ 代码**：

```cpp
long long gcd(long long a, long long b) {
    return b == 0 ? a : gcd(b, a % b);
}
long long lcm(long long a, long long b) {
    return a / gcd(a, b) * b;
}

int solve(long long a0, long long a1, long long b0, long long b1) {
    if (a0 % a1 != 0 || b1 % b0 != 0 || b1 % a1 != 0) return 0;
    int cnt = 0;
    for (long long x = a1; x * x <= b1 * a1; x += a1) {
        if (b1 % x != 0) continue;
        if (gcd(x, a0) == a1 && lcm(x, b0) == b1) cnt++;
        long long y = b1 / x * a1;  // 配对因子
        if (y != x && b1 % y == 0 && y % a1 == 0)
            if (gcd(y, a0) == a1 && lcm(y, b0) == b1) cnt++;
    }
    return cnt;
}
```

答案位置：`solve(a0, a1, b0, b1)` 的返回值

**核心启示**：
- **GCD 和 LCM 联合约束**时，先推出 `x` 的范围再枚举验证
- 枚举因子时注意配对（`d` 和 `n/d`），避免遗漏
- 前置判断 `a1 | a0`、`b0 | b1`、`a1 | b1` 可快速排除无解情况

### 题目 9：GCD 求和（CSES / 经典数论）

> 题意：求 `∑_{i=1}^{n} ∑_{j=1}^{n} gcd(i, j)`。

**分析**：经典的 GCD 求和问题。利用 `gcd(i,j) = ∑_{d|gcd(i,j)} φ(d)` 进行欧拉反演，交换求和顺序得 `∑_{d=1}^{n} φ(d) * ⌊n/d⌋²`。

```
核心观察：gcd(i,j) = ∑_{d|gcd(i,j)} φ(d)（欧拉函数的莫比乌斯反演）
构造策略：交换求和顺序，按 d 的倍数统计贡献
正确性：每个 d 的贡献为 φ(d) * (d 的倍数对数)
```

**C++ 代码**：

```cpp
const int MAXN = 1000001;
long long phi[MAXN];
// 线性筛求 phi[]（代码见题目 6）

long long solve(int n) {
    long long ans = 0;
    for (int d = 1; d <= n; d++)
        ans += phi[d] * (long long)(n / d) * (n / d);
    return ans;
}
```

答案位置：`solve(n)` 的返回值

**核心启示**：
- **欧拉反演公式** `n = ∑_{d|n} φ(d)` 是 GCD 求和问题的万能钥匙
- 交换求和顺序是将二重循环变为一重的关键手法
- 可进一步用整除分块优化到 `O(√n)`

### 题目 10：GCD 计数（经典题）

> 题意：给定 `n` 和 `k`，求 `1 ≤ i ≤ j ≤ n` 中 `gcd(i, j) = k` 的对数。

**分析**：令 `i = k*a`，`j = k*b`，则问题等价于 `1 ≤ a ≤ b ≤ n/k` 中 `gcd(a, b) = 1` 的对数。利用莫比乌斯函数或欧拉函数即可。

```
核心观察：gcd(i, j) = k 等价于 gcd(i/k, j/k) = 1
构造策略：变量替换后统计互质对，用莫比乌斯反演
正确性：只需对 [1, n/k] 范围做互质对计数
```

**C++ 代码**：

```cpp
const int MAXN = 1000001;
int mu[MAXN], primes[MAXN], pcnt;
bool not_prime[MAXN];

void mobius_sieve(int n) {
    mu[1] = 1;
    for (int i = 2; i <= n; i++) {
        if (!not_prime[i]) { primes[pcnt++] = i; mu[i] = -1; }
        for (int j = 0; j < pcnt && (long long)i * primes[j] <= n; j++) {
            not_prime[i * primes[j]] = true;
            if (i % primes[j] == 0) { mu[i * primes[j]] = 0; break; }
            mu[i * primes[j]] = -mu[i];
        }
    }
}

long long solve(int n, int k) {
    int m = n / k;
    long long ans = 0;
    for (int d = 1; d <= m; d++)
        ans += (long long)mu[d] * (m / d) * (m / d);
    return (ans + m) / 2;  // 无序对
}
```

答案位置：`solve(n, k)` 的返回值

**核心启示**：
- **变量替换**是 GCD 计数问题的第一步：把 `gcd = k` 变成 `gcd = 1`
- **莫比乌斯反演**是互质计数的终极武器
- 整除分块可进一步优化复杂度到 `O(√(n/k))`

### 题目 11：裴蜀定理应用（经典构造题）

> 题意：给定 `n` 个整数 `a1, a2, ..., an`，求 `∑ a[i] * x[i]` 能表示的最小正整数。

**分析**：由裴蜀定理的推广，`n` 个数的线性组合能表示的最小正整数就是它们的 GCD。

```
核心观察：ax + by 能表示的最小正整数 = gcd(a, b)，推广到 n 个数同理
构造策略：求 n 个数的 GCD 即可
正确性：裴蜀定理保证 GCD 可表示，且不可能更小
```

**C++ 代码**：

```cpp
long long gcd(long long a, long long b) {
    return b == 0 ? a : gcd(b, a % b);
}

long long solve(int n, long long a[]) {
    long long g = 0;
    for (int i = 0; i < n; i++)
        g = gcd(g, abs(a[i]));
    return g;
}
```

答案位置：`solve(n, a)` 的返回值

**核心启示**：
- **裴蜀定理是 GCD 与线性组合之间的桥梁**
- `gcd(0, x) = x` 这个性质使得初始化 `g = 0` 再逐个合并非常优雅
- 该结论对负数同样成立，只需取绝对值

### 题目 12：同余方程（AcWing / 经典题）

> 题意：求 `ax ≡ 1 (mod m)` 的最小正整数解。

**分析**：即求 `a` 在模 `m` 意义下的逆元。用扩展欧几里得解 `ax + my = 1`（需 `gcd(a, m) = 1`）。

```
核心观察：ax ≡ 1 (mod m) 等价于 ax + my = 1
构造策略：exgcd 求出 x 后取模调整为最小正整数
正确性：gcd(a, m) = 1 时有唯一解（模 m 意义下）
```

**C++ 代码**：

```cpp
long long exgcd(long long a, long long b, long long &x, long long &y) {
    if (b == 0) { x = 1; y = 0; return a; }
    long long g = exgcd(b, a % b, y, x);
    y -= (a / b) * x;
    return g;
}

long long solve(long long a, long long m) {
    long long x, y;
    exgcd(a, m, x, y);
    return (x % m + m) % m;  // 调整为最小正整数
}
```

答案位置：`solve(a, m)` 的返回值

**核心启示**：
- **扩展欧几里得是求模逆元的基础方法**
- 注意结果可能为负数，需 `(x % m + m) % m` 调整
- 当 `m` 为质数时也可用费马小定理：`a^(m-2) mod m`

### 题目 13：格点计数（CSES / 经典几何）

> 题意：给定两个点 `(0, 0)` 和 `(a, b)`，求线段上除端点外的整数格点个数。

**分析**：答案为 `gcd(a, b) - 1`。这是因为线段 `(0,0)-(a,b)` 上的格点恰好将线段等分为 `gcd(a,b)` 段。

```
核心观察：线段 (0,0)-(a,b) 上的格点数 = gcd(|a|, |b|) - 1（不含端点）
构造策略：直接求 gcd(a, b)
正确性：格点 (ka/g, kb/g) 恰好在线段上，k = 1, ..., g-1
```

```cpp
long long gcd(long long a, long long b) {

long long solve(long long a, long long b) {
    return gcd(abs(a), abs(b)) - 1;
}
```

答案位置：`solve(a, b)` 的返回值

**核心启示**：
- **GCD 在计算几何中的经典应用**：格点计数

- 这是一个"意想不到用 GCD"的好例子

### 题目 14：阶乘之和的 GCD（Codeforces 经典）

> 题意：给定 `n` 个正整数 `a1 < a2 < ... < an`，求 `gcd(a1!, a2!, ..., an!)`。

**分析**：阶乘是递增的，即 `a1! | a2! | ... | an!`。所以 `gcd(a1!, a2!, ..., an!) = a1!`，也就是最小值的阶乘。但如果题目问的是 `gcd(a1! + a2! + ... + an!, ...)` 则需要提取公因子。

```
核心观察：若 a1 ≤ a2 ≤ ... ≤ an，则 a1! | ai! 对所有 i 成立
构造策略：直接输出最小值的阶乘
正确性：所有 ai! 都是 a1! 的倍数，且 a1! 整除自身
```

**C++ 代码**：

```cpp
// a[] 已排序
void solve(int n, int a[]) {
    // gcd(a[0]!, a[1]!, ..., a[n-1]!) = a[0]!
    long long ans = 1;
    for (int i = 1; i <= a[0]; i++)
        ans *= i;  // 若需取模则 ans = ans * i % MOD
    // ans 即为答案
}
```

答案位置：变量 `ans`

**核心启示**：
- **整除关系的传递性**使得多个阶乘的 GCD 极其简单
- 关键是观察到 `a! | b!`（当 `a ≤ b`）
- 变体：如果问 `gcd(C(n,k1), C(n,k2), ...)`，需要用 Kummer 定理

### 题目 15：GCD 区间查询（经典数据结构题）

> 题意：给定长度为 `n` 的数组，`q` 次查询，每次求区间 `[l, r]` 的 GCD。

**分析**：GCD 满足结合律且有"幂等性"（`gcd(a, a) = a`），因此可以用稀疏表（Sparse Table）实现 `O(1)` 查询。

```
核心观察：gcd 满足结合律和幂等性，适合 Sparse Table
构造策略：预处理 st[i][j] = gcd(a[i..i+2^j-1])，查询两段覆盖
正确性：两段有重叠不影响结果（幂等性）
```

**C++ 代码**：

```cpp
const int MAXN = 500005, LOG = 20;
int st[MAXN][LOG], lg[MAXN];

void build(int a[], int n) {
    for (int i = 1; i <= n; i++) st[i][0] = a[i];
    for (int j = 1; (1 << j) <= n; j++)
        for (int i = 1; i + (1 << j) - 1 <= n; i++)
            st[i][j] = __gcd(st[i][j - 1], st[i + (1 << (j - 1))][j - 1]);
    lg[1] = 0;
    for (int i = 2; i <= n; i++) lg[i] = lg[i / 2] + 1;
}

int query(int l, int r) {
    int k = lg[r - l + 1];
    return __gcd(st[l][k], st[r - (1 << k) + 1][k]);
}
```

答案位置：`query(l, r)` 的返回值

**核心启示**：
- **GCD 的幂等性**使它成为 Sparse Table 的完美应用场景
- 预处理 `O(n log n)`，单次查询 `O(1)`
- 线段树也能做，但对于只有查询的问题 Sparse Table 更优

### 题目 16：GCD 等于子数组长度（Codeforces 经典）

> 题意：给定数组 `a[1..n]`，求满足 `gcd(a[l], ..., a[r]) = r - l + 1` 的子数组 `[l, r]` 的个数。

**分析**：固定右端点 `r`，左端点 `l` 从 `r` 往左移动时，`gcd(a[l..r])` 是单调不增的，且不同值最多 `O(log V)` 个。维护每个右端点的 GCD 段集合，逐一检查是否等于区间长度。

```
核心观察：固定右端点，左移左端点时 GCD 单调不增且变化 O(log V) 次
构造策略：维护 (gcd_val, 最远左端点) 的列表，逐步合并
正确性：GCD 每次变化至少除以 2，因此段数 O(log V)
```

**C++ 代码**：

```cpp
int a[200005];

long long solve(int n) {
    long long ans = 0;
    vector<pair<int,int>> segs;  // {gcd_val, leftmost_l}
    for (int r = 1; r <= n; r++) {
        vector<pair<int,int>> nxt;
        nxt.push_back({a[r], r});
        for (auto &[g, l] : segs) {
            int ng = __gcd(g, a[r]);
            if (nxt.back().first == ng) nxt.back().second = l;
            else nxt.push_back({ng, l});
        }
        for (auto &[g, l] : nxt) {
            int target_l = r - g + 1;  // gcd = r - l + 1 → l = r - gcd + 1
            if (target_l >= l && target_l <= r)
                ans++;  // 需进一步验证 target_l 到下一段的边界
        }
        segs = nxt;
    }
    return ans;
}
```

答案位置：`solve(n)` 的返回值

**核心启示**：
- **"固定右端点，维护 GCD 段"**是处理子数组 GCD 问题的核心技巧
- GCD 段的个数为 `O(log V)`，这使得总复杂度为 `O(n log V)`
- 该技巧可推广到子数组 GCD 的最值、计数、求和等问题

### 题目 17：GCD 路径（Codeforces / 树上 GCD）

> 题意：给定一棵 `n` 个节点带边权的树，`q` 次询问两个节点路径上所有边权的 GCD。

**分析**：树上路径查询可用 LCA + 倍增。在倍增数组中同时维护路径 GCD：`gcd_up[u][k]` 表示从 `u` 向上走 `2^k` 步路径上的 GCD。查询时按 LCA 求法合并两段。

```
核心观察：倍增求 LCA 时可同步维护路径上的 GCD
构造策略：gcd_up[u][k] = gcd(gcd_up[u][k-1], gcd_up[fa[u][k-1]][k-1])
正确性：GCD 满足结合律，可区间合并
```

**C++ 代码**：

```cpp
const int MAXN = 200005, LOG = 18;
int fa[MAXN][LOG], gcd_up[MAXN][LOG], dep[MAXN];
vector<pair<int,int>> adj[MAXN];  // {to, weight}

void dfs(int u, int p, int d) {
    fa[u][0] = p; dep[u] = d;
    for (int k = 1; k < LOG; k++) {
        fa[u][k] = fa[fa[u][k - 1]][k - 1];
        gcd_up[u][k] = __gcd(gcd_up[u][k - 1], gcd_up[fa[u][k - 1]][k - 1]);
    }
    for (auto [v, w] : adj[u]) if (v != p) {
        gcd_up[v][0] = w;
        dfs(v, u, d + 1);
    }
}

int query(int u, int v) {
    int g = 0;
    if (dep[u] < dep[v]) swap(u, v);
    int diff = dep[u] - dep[v];
    for (int k = LOG - 1; k >= 0; k--)
        if ((diff >> k) & 1) { g = __gcd(g, gcd_up[u][k]); u = fa[u][k]; }
    if (u == v) return g;
    for (int k = LOG - 1; k >= 0; k--)
        if (fa[u][k] != fa[v][k]) {
            g = __gcd(g, __gcd(gcd_up[u][k], gcd_up[v][k]));
            u = fa[u][k]; v = fa[v][k];
        }
    return __gcd(g, __gcd(gcd_up[u][0], gcd_up[v][0]));
}
```

答案位置：`query(u, v)` 的返回值

**核心启示**：
- **倍增 + GCD**是树上路径 GCD 查询的标配
- GCD 的结合律使它可以像 min/max 一样在倍增中维护
- 此模式可推广到树上路径的任意满足结合律的运算

### 题目 18：GCD 卷积（莫比乌斯反演经典）

> 题意：给定数组 `f[1..n]` 和 `g[1..n]`，求 `h[d] = ∑_{gcd(i,j)=d} f[i] * g[j]`。

**分析**：令 `F[d] = ∑_{d|i} f[i]`，`G[d] = ∑_{d|j} g[j]`，则 `H[d] = F[d] * G[d]`（其中 `H[d] = ∑_{d|gcd(i,j)} f[i]*g[j]`）。再对 `H` 做莫比乌斯反演得到 `h[d] = ∑_{d|k} μ(k/d) * H[k]`。

```
核心观察：GCD 卷积可以通过"倍数求和 → 点乘 → 莫比乌斯反演"三步完成
构造策略：狄利克雷前缀和 + 点乘 + 狄利克雷差分
正确性：莫比乌斯反演保证从 H 精确恢复 h
```

**C++ 代码**：

```cpp
const int MAXN = 1000001;
long long f[MAXN], g[MAXN], F[MAXN], G[MAXN], H[MAXN], h[MAXN];
int mu[MAXN];  // 预处理莫比乌斯函数

void solve(int n) {
    // Step 1: 狄利克雷前缀和
    for (int i = 1; i <= n; i++) F[i] = f[i], G[i] = g[i];
    for (int p : primes)  // 枚举质因子
        for (int j = n / p; j >= 1; j--)
            F[j] += F[j * p], G[j] += G[j * p];
    // Step 2: 点乘
    for (int i = 1; i <= n; i++) H[i] = F[i] * G[i];
    // Step 3: 莫比乌斯反演（狄利克雷差分）
    for (int p : primes)
        for (int j = 1; (long long)j * p <= n; j++)
            H[j] -= H[j * p];
    // H[d] 即为 h[d]
}
```

答案位置：数组 `H[d]` 即为 `h[d]`

**核心启示**：
- **GCD 卷积 = 狄利克雷前缀和 + 点乘 + 莫比乌斯反演**，三步走
- 这是处理形如 `∑_{gcd(i,j)=d}` 类问题的终极框架
- 狄利克雷前缀和的复杂度为 `O(n log log n)`，非常高效

## 五、常见技巧与思路总结

### 5.1 辗转相除的迭代写法

递归版简洁但可能栈溢出（极端情况），迭代版更安全：

```cpp
long long gcd(long long a, long long b) {
    while (b) { a %= b; swap(a, b); }
    return a;
}
```

### 5.2 先除后乘防溢出

计算 `lcm(a, b)` 时，直接 `a * b` 可能溢出。正确做法：

```cpp
long long lcm(long long a, long long b) {
    return a / gcd(a, b) * b;  // 先除再乘
}
```

### 5.3 枚举因子的调和级数复杂度

枚举 `1` 到 `n` 的所有倍数，总次数为 `n/1 + n/2 + ... + n/n = O(n log n)`。这是许多 GCD 题的复杂度来源。

### 5.4 GCD 段的 O(log V) 性质

固定一端，另一端移动时，子区间的 GCD 最多变化 `O(log V)` 次（每次至少减半）。这使得维护所有不同 GCD 值的复杂度可控。

### 5.5 gcd(0, x) = x

这个边界性质看似简单，但在多数 GCD 合并中是关键：初始化 `g = 0` 后逐个合并。

### 5.6 整除分块加速求和

对于 `∑_{d=1}^{n} f(d) * ⌊n/d⌋` 形式的求和，`⌊n/d⌋` 最多只有 `O(√n)` 种不同取值，可以分块计算。

```cpp
long long block_sum(int n) {
    long long ans = 0;
    for (int l = 1, r; l <= n; l = r + 1) {
        r = n / (n / l);
        ans += (sum_f[r] - sum_f[l - 1]) * (n / l);
    }
    return ans;
}
```

## 七、终极总结与归纳

### 7.1 GCD 的本质

GCD 的本质，可以压缩成一句话：

> **GCD 是两个整数的"最大公共度量"，它连接了整除关系、线性组合、互质性、因子分解等数论核心概念。**

几乎所有 GCD 题目都在围绕以下三个问题展开：
- 这些数的最大公共因子是多少？（直接求 GCD）
- 有多少对数满足某个 GCD 条件？（GCD 计数/求和）
- 如何利用 GCD 的性质来构造或优化？（GCD 应用/反演）

### 7.2 核心要点

1. **欧几里得算法是基石**
   - `gcd(a, b) = gcd(b, a % b)`，时间复杂度 `O(log min(a, b))`，几乎所有题的起点。

2. **GCD-LCM 乘积关系要条件反射**
   - `gcd(a,b) * lcm(a,b) = a * b`，GCD 和 LCM 的题常常互相转化。

3. **裴蜀定理是线性组合的判据**
   - `ax + by = c` 有解当且仅当 `gcd(a,b) | c`，扩展欧几里得给出具体解。

4. **欧拉/莫比乌斯反演是 GCD 求和的终极武器**
   - `n = ∑_{d|n} φ(d)`，再配合整除分块可以高效完成。

5. **GCD 段 O(log V) 是子数组 GCD 问题的核心性质**
   - 固定一端移动另一端，GCD 最多变化 `O(log V)` 次。

6. **Stein 算法应对高精度**
   - 只用减法和除以 2，避免高精度取模。
