## 一、什么是线性规划

线性规划（Linear Programming, LP）是在一组线性不等式（约束）下，求一个线性目标函数的最大值或最小值的优化问题。它是运筹学的基石，也是算法竞赛中网络流、博弈论、近似算法等领域的理论基础。

**标准形式**：

$$
\max \; c^T x \quad \text{s.t.} \; Ax \le b, \; x \ge 0
$$

其中 $x \in \mathbb{R}^n$ 是决策变量，$c \in \mathbb{R}^n$ 是目标系数，$A \in \mathbb{R}^{m \times n}$ 是约束矩阵，$b \in \mathbb{R}^m$ 是约束右端。

**几何直觉**：可行域是一个凸多面体，最优解一定出现在某个顶点上。单纯形法就是沿着多面体的边从一个顶点"走"到另一个顶点，直到目标函数不再增大。

## 二、核心思想与前置知识

### 2.1 单纯形法（Simplex Method）

单纯形法是求解 LP 的经典算法。虽然最坏情况指数级，但实际运行中平均时间复杂度为 $O(m \cdot n)$，在竞赛中是最实用的 LP 求解器。

**核心流程**：
1. **初始化**：构造增广矩阵（松弛变量 + 目标行）
2. **选入基变量**（entering）：目标行中系数最大的列
3. **选出基变量**（leaving）：最小比值法确定 pivot 行
4. **旋转操作**（pivot）：高斯消元，更新矩阵
5. **终止判定**：目标行所有系数 $\le 0$，当前解即为最优

```cpp
const int MAXN = 25, MAXM = 10005;
const double eps = 1e-8, INF = 1e18;
double a[MAXM][MAXN]; // a[0] = 目标行, a[1..m] = 约束行, 列0 = 常数列
int n, m; // n 个变量, m 个约束

void pivot(int r, int c) {
    double t = a[r][c]; a[r][c] = 1;
    for (int j = 0; j <= n; j++) a[r][j] /= t;
    for (int i = 0; i <= m; i++) {
        if (i == r || fabs(a[i][c]) < eps) continue;
        t = a[i][c]; a[i][c] = 0;
        for (int j = 0; j <= n; j++) a[i][j] -= t * a[r][j];
    }
}

double simplex() {
    while (true) {
        int c = 0;
        for (int j = 1; j <= n; j++)
            if (a[0][j] > a[0][c]) c = j;
        if (a[0][c] < eps) break; // 所有系数 <= 0，最优
        int r = 0; double mn = INF;
        for (int i = 1; i <= m; i++)
            if (a[i][c] > eps && a[i][0] / a[i][c] < mn)
                mn = a[i][0] / a[i][c], r = i;
        if (!r) return INF; // 无界
        pivot(r, c);
    }
    return a[0][0];
}
```

### 2.2 对偶问题（Dual Problem）

原问题（Primal）与对偶问题（Dual）互为镜像：

| | 原问题 | 对偶问题 |
|---|---|---|
| 目标 | $\max \; c^T x$ | $\min \; b^T y$ |
| 约束 | $Ax \le b, \; x \ge 0$ | $A^T y \ge c, \; y \ge 0$ |

**弱对偶定理**：任何对偶可行解 $\ge$ 任何原可行解（min $\ge$ max）。

**强对偶定理**：若原问题有最优解，则对偶也有，且 $\max c^T x^* = \min b^T y^*$。

**互补松弛条件**：$x_j^*(A^T y^* - c)_j = 0$ 且 $y_i^*(b - Ax^*)_i = 0$。

### 2.3 全幺模矩阵（Totally Unimodular Matrix）

若约束矩阵 $A$ 是全幺模的（每个子方阵的行列式为 $0, \pm 1$），则 LP 的最优解自动为整数。

**关键推论**：网络流的约束矩阵（关联矩阵）是全幺模的，因此网络流问题等价于对应的 LP，且总有整数最优解。

### 2.4 线性规划与网络流

最小费用最大流本质上就是一个特殊的 LP。许多竞赛中看似需要 LP 来建模的问题，因为约束矩阵的全幺模性，可以用网络流高效求解。这是竞赛中最常见的 LP 应用路径：

```
题意 → LP 建模 → 发现全幺模性 → 转化网络流 → 高效求解
```

## 三、分类与变体详解

### 3.1 单纯形法及其变体

| 方法 | 特点 | 竞赛适用性 |
|------|------|------------|
| 原始单纯形法 | 从可行解出发，迭代至最优 | ★★★★★ |
| 对偶单纯形法 | 从对偶可行解出发 | ★★★★ |
| Bland 规则 | 防止退化循环，选最小下标 | ★★★★★ |
| 随机扰动 | 随机选入基变量，避免最坏情况 | ★★★★ |

### 3.2 对偶转化技巧

当原问题约束多、变量少时，取对偶可以减少约束个数，降低单纯形法的实际复杂度。

### 3.3 LP 松弛与整数规划

对于 NP-hard 的整数线性规划（ILP），LP 松弛可以给出下界（最小化问题）或上界（最大化问题），是分支定界法的基础。

### 3.4 LP 在组合优化中的地位

| LP 模型 | 对应组合问题 |
|---------|-------------|
| LP + 全幺模 | 网络流（最大流、费用流） |
| LP 对偶 | 最短路、匹配的互补松弛 |
| LP 松弛 | 集合覆盖、调度问题的近似 |
| LP + 博弈 | 零和博弈的 minimax |

### 🟢 入门级 Level 1：单纯形法基础与简单建模

### 题目 1：标准 LP 求解模板（UOJ 179）

> 给定 $n$ 个变量 $m$ 个约束的标准形式线性规划 $\max c^T x$，$Ax \le b$，$x \ge 0$，求最优解。

**分析**：这是最直接的单纯形法应用。注意初始化时需要加入松弛变量将不等式转为等式，并处理可能的无解（Infeasible）和无界（Unbounded）情况。

```
核心观察：标准形式 LP 直接用单纯形法求解
构造策略：建立增广矩阵 [b | A]，目标行放第 0 行，迭代 pivot 直到最优
正确性：每次 pivot 使目标函数严格递增（非退化时），顶点数有限 → 必终止
```

**C++ 代码**：

```cpp
int id[MAXN + MAXM]; // 记录基变量编号

void pivot(int r, int c) {
    swap(id[n + r], id[c]);
    double t = a[r][c]; a[r][c] = 1;
    for (int j = 0; j <= n; j++) a[r][j] /= t;
    for (int i = 0; i <= m; i++) {
        if (i == r || fabs(a[i][c]) < eps) continue;
        t = a[i][c]; a[i][c] = 0;
        for (int j = 0; j <= n; j++) a[i][j] -= t * a[r][j];
    }
}

double simplex() {
    while (true) {
        int c = 0;
        for (int j = 1; j <= n; j++)
            if (a[0][j] > a[0][c]) c = j;
        if (a[0][c] < eps) return a[0][0];
        int r = 0; double mn = 1e18;
        for (int i = 1; i <= m; i++)
            if (a[i][c] > eps && a[i][0] / a[i][c] < mn)
                mn = a[i][0] / a[i][c], r = i;
        if (!r) return 1e18; // Unbounded
        pivot(r, c);
    }
}
```

**核心启示**：
- **增广矩阵布局**：列 0 存常数 $b$，列 1..n 存系数，行 0 存目标函数系数
- **Pivot 操作**：本质是高斯消元的一步，将第 $c$ 列化为单位向量
- **无界判定**：若入基列所有系数 $\le 0$，说明可以沿该方向无限增加

### 题目 2：两变量 LP 图解法（AcWing 模拟）

> 给定两个变量 $x_1, x_2$ 和若干线性约束，最大化 $c_1 x_1 + c_2 x_2$。数据保证可行域有界。

**分析**：两个变量的 LP 可行域是平面上的凸多边形。最优解在顶点上取到。我们枚举所有约束的交点，检查可行性，取使目标函数最大的顶点即可。

```
核心观察：2D LP 的可行域是凸多边形，最优解在顶点
构造策略：枚举 O(m²) 个交点，逐一检查是否满足全部约束，取最优
正确性：最优解必在顶点 → 枚举所有顶点即可保证正确
```

**C++ 代码**：

```cpp
struct Line { double a, b, c; }; // ax1 + bx2 <= c
Line L[MAXM];
int m;

double solve(double c1, double c2) {
    double ans = -1e18;
    for (int i = 0; i < m; i++)
        for (int j = i + 1; j < m; j++) {
            double det = L[i].a * L[j].b - L[i].b * L[j].a;
            if (fabs(det) < eps) continue;
            double x1 = (L[i].c * L[j].b - L[j].c * L[i].b) / det;
            double x2 = (L[i].a * L[j].c - L[j].a * L[i].c) / det;
            if (x1 < -eps || x2 < -eps) continue;
            bool ok = true;
            for (int k = 0; k < m && ok; k++)
                if (L[k].a * x1 + L[k].b * x2 > L[k].c + eps) ok = false;
            if (ok) ans = max(ans, c1 * x1 + c2 * x2);
        }
    return ans;
}
```

**核心启示**：
- **顶点枚举**：小规模（变量少）时，枚举顶点是最直观的方法
- **数值精度**：使用 eps 容忍浮点误差，避免将可行解误判为不可行
- **复杂度**：$O(m^3)$ 对于两变量足够，但无法扩展到高维

### 题目 3：志愿者招募（洛谷 P3980）

> 有 $n$ 天，第 $i$ 天至少需要 $a_i$ 个志愿者。有 $m$ 种志愿者，第 $j$ 种可在第 $s_j$ 天到第 $t_j$ 天服务，招募费用为 $c_j$。求满足所有天数需求的最小总费用。

**分析**：这是 LP 与网络流等价性的经典例题。直接建 LP：变量 $x_j$ 表示第 $j$ 种招第几人，目标 $\min \sum c_j x_j$，约束第 $i$ 天 $\sum_{j: s_j \le i \le t_j} x_j \ge a_i$。关键洞察：约束矩阵是区间覆盖矩阵，具有全幺模性，可转化为最小费用流。

```
核心观察：约束矩阵为连续 1 的区间结构 → 全幺模 → LP 整数最优解存在
构造策略：建费用流图——链 1→2→...→n+1，每条边容量 INF 费用 0 减去需求差分；每种志愿者 s→t+1 容量 INF 费用 c
正确性：LP 对偶等价于费用流的对偶，强对偶保证最优值相同
```

**C++ 代码**：

```cpp
// MCMF 费用流建图
int S, T;  // 源汇
int a[MAXN]; // 每天需求

void build(int n, int m) {
    S = 0; T = n + 2;
    // 链边: i → i+1
    for (int i = 0; i <= n; i++) {
        int d = a[i + 1] - a[i]; // 差分 (a[0]=a[n+1]=0)
        if (d > 0) addEdge(S, i + 1, d, 0);
        else if (d < 0) addEdge(i + 1, T, -d, 0);
        if (i < n + 1) addEdge(i + 1, i + 2, INF, 0); // 贯穿边
    }
    // 志愿者边
    for (int j = 1; j <= m; j++)
        addEdge(s[j], t[j] + 1, INF, c[j]);
    // 求 S→T 最小费用最大流即为答案
}
```

**核心启示**：
- **LP → 网络流**：当约束矩阵全幺模时，LP 与网络流等价，这是竞赛中最高频的 LP 应用
- **差分建图**：将"每天至少 $a_i$"用差分转化为流量守恒，是费用流建模的标准技巧
- **区间覆盖结构**：只要约束矩阵每行的 1 形成连续区间，就具有全幺模性

### 题目 4：无意识之外的捉迷藏（洛谷 P4232）

> 给定 $n$ 个点的完全图，边权为 $w_{ij}$。A 选一个点 $u$，B 选一个点 $v$（可重合），A 的收益为 $w_{uv}$。A 想最大化期望收益，B 想最小化。求最优混合策略下的期望值。

**分析**：这是一个零和博弈问题。A 选择概率分布 $p$，B 选择概率分布 $q$，博弈值 $V = \max_p \min_q p^T W q$。由 minimax 定理，这等于 $\min_q \max_p p^T W q$。将其转化为 LP：$\max V$，约束 $Wp \ge V \cdot \mathbf{1}$，$\sum p_i = 1$，$p \ge 0$。然后用单纯形法求解。

```
核心观察：零和博弈 → minimax 定理 → LP
构造策略：A 的策略为 LP 变量 p_i，约束为对每个 B 的纯策略 j，∑ w_ij p_i ≥ V
正确性：minimax 定理保证 LP 的最优值等于博弈均衡值
```

**C++ 代码**：

```cpp
double w[MAXN][MAXN];
// 构造 LP: max V
// 令 x_i = p_i / V, 则 max 1/(∑x_i), 即 min ∑x_i
// 约束: ∑ w_ij * x_i >= 1 对所有 j, x_i >= 0
// 标准形式: min c^T x, A^T x >= 1

void buildLP(int n) {
    m = n; // n 个约束 (对应B的每个纯策略)
    // 变量 x_1..x_n
    for (int j = 1; j <= n; j++)      // 约束 j
        for (int i = 1; i <= n; i++)   // 变量 i
            a[j][i] = w[i][j];        // ∑ w_ij x_i >= 1
    for (int j = 1; j <= n; j++) a[j][0] = 1; // 右端
    for (int i = 1; i <= n; i++) a[0][i] = -1; // 目标 min ∑x_i → max -∑x_i
    double opt = -simplex(); // min ∑x_i = 1/V
    printf("%.6f\n", 1.0 / opt);
}
```

**核心启示**：
- **博弈 → LP 转化**：minimax 定理将零和博弈的均衡计算化归为 LP
- **变量替换**：令 $x_i = p_i / V$ 消去非线性，是博弈 LP 的标准技巧
- **对称性**：A、B 的角色互换对应取 LP 对偶

### 题目 5：随机化单纯形（CF 375E）

> 给定一个线性规划问题，约束可能导致单纯形法退化。用随机化策略保证期望多项式时间。

**分析**：退化（degenerate）指多个约束在同一顶点处紧致，pivot 不改变目标值，可能导致循环。随机选择入基变量可以以高概率避免循环，且实际效果极好。

```
核心观察：退化时确定性选列规则可能循环，随机化打破对称性
构造策略：每次从所有正系数列中均匀随机选一个作为入基变量
正确性：随机选择使退化路径出现循环的概率为 0（连续分布下）
```

**C++ 代码**：

```cpp
mt19937 rng(chrono::steady_clock::now().time_since_epoch().count());

double simplex_random() {
    while (true) {
        // 收集所有正系数列
        vector<int> cands;
        for (int j = 1; j <= n; j++)
            if (a[0][j] > eps) cands.push_back(j);
        if (cands.empty()) return a[0][0];
        // 随机选入基变量
        int c = cands[rng() % cands.size()];
        int r = 0; double mn = 1e18;
        for (int i = 1; i <= m; i++)
            if (a[i][c] > eps && a[i][0] / a[i][c] < mn)
                mn = a[i][0] / a[i][c], r = i;
        if (!r) return 1e18;
        pivot(r, c);
    }
}
```

**核心启示**：
- **随机化防退化**：在竞赛中，随机单纯形法是最实用的防退化策略，实现简单
- **Bland 规则的替代**：Bland 规则（选最小下标）也能防止循环，但实际速度可能更慢
- **初始化扰动**：也可以对 $b$ 加微小随机扰动来避免退化

### 题目 6：整数 LP 松弛与近似（CF 1061E）

> 给定一组物品，每个有重量和价值，以及一组约束（如分组限制），求最大总价值。NP-hard 背景下用 LP 松弛获得上界。

**分析**：将 $x_i \in \{0, 1\}$ 松弛为 $0 \le x_i \le 1$，求解 LP 获得上界。LP 最优解中的分数部分提示了"最有争议"的物品，可结合分支定界或四舍五入得到可行整数解。

```
核心观察：ILP 是 NP-hard，LP 松弛给出目标函数上界
构造策略：去掉整数约束 → 求 LP → 上界用于剪枝或近似
正确性：LP 可行域 ⊇ ILP 可行域 → LP 最优值 ≥ ILP 最优值
```

**C++ 代码**：

```cpp
// 分支定界框架中调用 LP 松弛
double lp_relax[MAXN]; // 松弛解

double branch_and_bound(int dep) {
    double ub = simplex(); // LP 松弛上界
    if (ub <= best_int) return ub; // 剪枝
    // 找分数变量
    int frac = -1;
    for (int i = 1; i <= n; i++) {
        double v = lp_relax[i];
        if (v > eps && v < 1 - eps) { frac = i; break; }
    }
    if (frac == -1) { best_int = max(best_int, ub); return ub; }
    // 分支 x[frac] = 0 和 x[frac] = 1
    // 添加约束后递归
    add_constraint(frac, 0); branch_and_bound(dep + 1);
    remove_constraint();
    add_constraint(frac, 1); branch_and_bound(dep + 1);
    remove_constraint();
    return ub;
}
```

**核心启示**：
- **LP 松弛的意义**：提供目标函数的理论界，是分支定界和近似算法的基础
- **分数解的信息**：松弛解中接近 0.5 的变量是"最难决定"的，优先分支
- **竞赛应用**：部分竞赛题可以用 LP 松弛来证明贪心或特殊算法的最优性

### 🟠 提高级 Level 3：对偶应用与 LP 建模

### 题目 7：对偶转化——约束与变量互换（UOJ 模拟赛）

> 有 $10^4$ 个约束、$20$ 个变量的 LP。直接用单纯形法太慢，利用对偶转化后求解。

**分析**：原问题 $m = 10^4, n = 20$，单纯形法的 pivot 操作 $O(mn)$，总迭代次数可能很大。取对偶后变成 $m' = 20, n' = 10^4$，约束数大幅减少，pivot 的行扫描从 $10^4$ 降到 $20$。

```
核心观察：单纯形法主要瓶颈在约束行数 m，取对偶交换 m 和 n
构造策略：原 max c^Tx, Ax≤b → 对偶 min b^Ty, A^Ty≥c → 标准化为 max -b^Ty, -A^Ty≤-c
正确性：强对偶定理保证原问题与对偶最优值相同
```

**C++ 代码**：

```cpp
// 原问题: max c^T x, Ax <= b, x >= 0
// 对偶: min b^T y, A^T y >= c, y >= 0
// 等价: max -b^T y, -A^T y <= -c, y >= 0

void buildDual(double A_orig[][MAXN_ORIG], double b_orig[], double c_orig[],
               int m_orig, int n_orig) {
    n = m_orig; m = n_orig; // 对偶的变量数 = 原约束数
    for (int i = 1; i <= m; i++) {
        a[i][0] = -c_orig[i]; // 右端 = -c
        for (int j = 1; j <= n; j++)
            a[i][j] = -A_orig[j][i]; // 系数 = -A^T
    }
    for (int j = 1; j <= n; j++)
        a[0][j] = -b_orig[j]; // 目标 = -b
}
// 调用 simplex() 后取负得原问题最优值
```

**核心启示**：
- **对偶转化加速**：当 $m \gg n$ 时，取对偶可以大幅加速单纯形法
- **转化方向**：永远用"min变量少的那边作为约束"的形式来求解
- **双重理解**：对偶不仅是数学工具，也是降低复杂度的算法技巧

### 题目 8：LP 证明贪心最优性（CF 802C）

> 给定 $n$ 个任务，每个有截止时间 $d_i$ 和惩罚 $w_i$。每个时刻只能做一个任务（耗时 1）。最小化超期任务的总惩罚。

**分析**：贪心策略是按惩罚降序排，优先安排惩罚大的任务在截止时间前完成。要证明贪心是最优的，可以建 LP 松弛，发现松弛解等于贪心解（因为调度问题的约束矩阵具有全幺模性）。

```
核心观察：调度问题的约束矩阵是区间矩阵 → 全幺模 → LP 松弛 = ILP 最优
构造策略：LP 变量 x_i ∈ {0,1} 表示任务 i 是否按时完成，约束 ∑_{d_i≤t} x_i ≤ t
正确性：全幺模性 → LP 最优解为整数 → 贪心找到该整数解 → 贪心最优
```

**C++ 代码**：

```cpp
struct Task { int d, w; } tasks[MAXN];

long long greedy(int n) {
    sort(tasks, tasks + n, [](const Task &a, const Task &b) {
        return a.w > b.w;
    });
    long long penalty = 0;
    set<int> slots; // 可用时间槽
    for (int t = 1; t <= n; t++) slots.insert(t);
    for (int i = 0; i < n; i++) {
        auto it = slots.upper_bound(tasks[i].d);
        if (it != slots.begin()) {
            --it;
            slots.erase(it); // 安排在截止前最晚的空闲时刻
        } else {
            penalty += tasks[i].w; // 无法按时完成
        }
    }
    return penalty;
}
```

**核心启示**：
- **LP 证明贪心**：建立 LP 松弛并证明全幺模性，是证明贪心算法正确性的标准方法
- **调度问题的结构**：单机调度的约束矩阵天然全幺模，这是调度与 LP 的深层联系
- **逆向思维**：最小化惩罚 = 最大化按时完成的任务总价值

### 题目 9：多商品流 LP 建模（CF 843E）

> 有向图中，$k$ 种商品各有源汇和需求量。每条边有总容量限制。判断是否存在可行的多商品流方案。

**分析**：单商品流可用网络流求解，但多商品共享容量时，不同商品的流互相竞争，必须用 LP 建模。变量 $f_e^i$ 表示商品 $i$ 在边 $e$ 上的流量，约束包括流量守恒和边容量共享。

```
核心观察：多商品流一般不可分解为独立的单商品流 → 必须用 LP
构造策略：变量 f_e^i，约束 (1) 流量守恒 (2) ∑_i f_e^i ≤ cap_e (3) f_e^i ≥ 0
正确性：LP 可行性判定即为多商品流存在性判定
```

**C++ 代码**：

```cpp
// 建 LP 矩阵：仅判断可行性 → 目标函数为 0
// 变量编号: f[i][e] 为第 i 种商品在边 e 上的流

void buildMultiCommodity(int k, int E, int V) {
    n = k * E; // 总变量数
    m = 0;
    // 流量守恒: 对每种商品 i, 每个节点 v
    for (int i = 0; i < k; i++)
        for (int v = 0; v < V; v++) {
            m++;
            for (int e = 0; e < E; e++) {
                int var = i * E + e + 1;
                if (edge[e].from == v) a[m][var] = 1;
                if (edge[e].to == v) a[m][var] = -1;
            }
            a[m][0] = (v == src[i]) ? demand[i] : (v == dst[i] ? -demand[i] : 0);
        }
    // 容量共享: ∑_i f[i][e] <= cap[e]
    for (int e = 0; e < E; e++) {
        m++;
        for (int i = 0; i < k; i++)
            a[m][i * E + e + 1] = 1;
        a[m][0] = cap[e];
    }
}
```

**核心启示**：
- **共享容量**：多商品流的核心难点在于不同商品共享边容量，LP 是标准建模工具
- **变量个数**：$O(k \cdot E)$ 个变量，约束 $O(k \cdot V + E)$，规模较大时需要列生成等技术
- **整数性丧失**：多商品流的约束矩阵一般不全幺模，LP 最优解可能是分数

### 题目 10：最大权闭合子图（洛谷 P2762）

> 有 $n$ 个项目和 $m$ 个设备。项目 $i$ 有收益 $p_i$（可负），完成项目 $i$ 需要集合 $S_i$ 中的所有设备。每个设备有购买成本 $c_j$。选定一组项目使净收益最大。

**分析**：闭合子图：选了某个节点就必须选它所有后继。max 权闭合子图 = 正权之和 - 最小割。LP 视角：变量 $x_i \in [0,1]$ 表示是否选节点 $i$，约束 $x_i \le x_j$（若 $i \to j$），目标 max $\sum w_i x_i$。由全幺模性，LP 解为 0/1。

```
核心观察：闭合子图约束 x_i ≤ x_j 的系数矩阵全幺模 → LP = ILP
构造策略：源 S 连正权点（容量 = 权值），负权点连汇 T（容量 = |权值|），依赖关系为 INF 边
正确性：最大权闭合子图 = ∑正权 - 最小割（LP 对偶 ↔ 最小割）
```

**C++ 代码**：

```cpp
int S, T;
long long sumPos = 0;

void build(int n, int m, int p[], int c[], vector<int> dep[]) {
    S = 0; T = n + m + 1;
    for (int i = 1; i <= n; i++) {
        if (p[i] > 0) {
            addEdge(S, i, p[i]);
            sumPos += p[i];
        } else {
            addEdge(i, T, -p[i]);
        }
        for (int j : dep[i])
            addEdge(i, n + j, INF); // 项目依赖设备
    }
    for (int j = 1; j <= m; j++)
        addEdge(n + j, T, c[j]); // 设备成本
    // 答案 = sumPos - maxflow(S, T)
}
```

**核心启示**：
- **闭合子图 = 最小割**：这是 LP 对偶在图论中最优美的应用之一
- **建图模式**：正权连源、负权连汇、依赖关系为 INF 边，是固定模板
- **LP 揭示本质**：从 LP 角度理解最大权闭合子图，比直接理解最小割更自然

### 题目 11：最优匹配的 LP 对偶（UOJ 80）

> 二分图最大权匹配。用 LP 对偶理论推导 KM 算法的正确性。

**分析**：原 LP：$\max \sum w_{ij} x_{ij}$，约束 $\sum_j x_{ij} \le 1, \sum_i x_{ij} \le 1, x_{ij} \ge 0$。对偶：$\min \sum u_i + \sum v_j$，约束 $u_i + v_j \ge w_{ij}$。KM 算法维护的"顶标"$u_i, v_j$ 就是对偶变量，互补松弛条件要求匹配边满足 $u_i + v_j = w_{ij}$。

```
核心观察：KM 算法就是对偶单纯形法在二分图匹配上的特化
构造策略：维护顶标 u_i, v_j（对偶可行），在相等子图中找完美匹配（互补松弛）
正确性：互补松弛 + 强对偶定理 → 找到的匹配一定是最优的
```

**C++ 代码**：

```cpp
int w[MAXN][MAXN], mt[MAXN];
int u[MAXN], v[MAXN]; // 顶标（对偶变量）
bool S[MAXN], T_vis[MAXN];

bool dfs(int x, int n) {
    S[x] = true;
    for (int y = 1; y <= n; y++) {
        if (T_vis[y]) continue;
        if (u[x] + v[y] == w[x][y]) {
            T_vis[y] = true;
            if (!mt[y] || dfs(mt[y], n)) { mt[y] = x; return true; }
        }
    }
    return false;
}

long long km(int n) {
    memset(mt, 0, sizeof mt);
    for (int i = 1; i <= n; i++) {
        u[i] = *max_element(w[i] + 1, w[i] + n + 1);
        v[i] = 0;
    }
    for (int i = 1; i <= n; i++) {
        while (true) {
            memset(S, 0, sizeof S);
            memset(T_vis, 0, sizeof T_vis);
            if (dfs(i, n)) break;
            int d = INT_MAX;
            for (int x = 1; x <= n; x++) if (S[x])
                for (int y = 1; y <= n; y++) if (!T_vis[y])
                    d = min(d, u[x] + v[y] - w[x][y]);
            for (int x = 1; x <= n; x++) if (S[x]) u[x] -= d;
            for (int y = 1; y <= n; y++) if (T_vis[y]) v[y] += d;
        }
    }
    long long ans = 0;
    for (int j = 1; j <= n; j++) if (mt[j]) ans += w[mt[j]][j];
    return ans;
}
```

**核心启示**：
- **KM = 对偶单纯形**：KM 算法的顶标调整就是对偶变量的更新，相等子图就是互补松弛条件
- **理论指导算法**：LP 对偶理论不仅证明了 KM 的正确性，还指导了算法的设计
- **推广性**：这种"原始-对偶"框架可推广到一般匹配、最短路等问题

### 题目 12：列生成法求大规模 LP（CF 经典）

> 切割问题：有长度为 $L$ 的钢管，需要切出不同长度 $l_1, ..., l_k$ 的短管各 $d_i$ 根。每根长管可以有不同的切割方案。最小化使用长管数量。

**分析**：切割方案数指数级，不能枚举所有列。列生成法：先用少量初始列建 LP，求解后利用对偶值（影子价格）生成新列（新方案），迭代直到无法改进。每次新列的生成是一个背包问题。

```
核心观察：方案数指数级但大部分方案在最优解中不使用 → 只需生成"有用"的列
构造策略：主问题 LP 求解 → 取对偶值 π → 子问题: max ∑π_i a_i, ∑l_i a_i ≤ L → 若 > 1 则加入新列
正确性：对偶值指示了新方案的"边际价值"，当无 reduced cost > 0 时达到最优
```

**C++ 代码**：

```cpp
double pi[MAXK]; // 对偶值（影子价格）

// 子问题: 完全背包，物品价值 = pi[i], 重量 = l[i], 容量 = L
bool generateColumn(int k, int L, int l[]) {
    // dp[w] = 用长度 w 能获得的最大对偶值
    vector<double> dp(L + 1, 0);
    vector<vector<int>> choice(L + 1, vector<int>(k, 0));
    for (int i = 0; i < k; i++)
        for (int w = l[i]; w <= L; w++)
            if (dp[w - l[i]] + pi[i] > dp[w]) {
                dp[w] = dp[w - l[i]] + pi[i];
                choice[w] = choice[w - l[i]];
                choice[w][i]++;
            }
    if (dp[L] > 1 + eps) {
        // 将 choice[L] 作为新方案列加入主问题 LP
        addColumn(choice[L]);
        return true;
    }
    return false;
}
```

**核心启示**：
- **列生成的适用场景**：方案数指数级但最优解只涉及少量方案时，列生成极为有效
- **主-子问题分解**：主问题求 LP 和对偶值，子问题利用对偶值生成新列（通常是背包/最短路）
- **reduced cost**：新列的判定条件是 $\sum \pi_i a_i > 1$（即 reduced cost $> 0$）

### 🟣 挑战级 Level 5：高级理论与竞赛综合

### 题目 13：Bland 规则与退化处理（UOJ 179 加强数据）

> LP 模板题的加强版：数据构造了退化情况（多个约束在同一顶点紧致），确定性最大系数规则可能进入死循环。

**分析**：退化时 pivot 不改变目标值，如果选择规则不当可能在几个基之间循环。Bland 规则规定：入基选最小正下标系数列，出基选最小下标行。这保证了有限步终止。

```
核心观察：退化 → pivot 后目标值不变 → 可能循环
构造策略：Bland 规则：入基选 j 最小的 a[0][j] > 0 列，出基选 i 最小的满足最小比值的行
正确性：Bland 定理证明此规则不会循环（基于字典序论证）
```

**C++ 代码**：

```cpp
double simplex_bland() {
    while (true) {
        // Bland: 选最小下标的正系数列
        int c = -1;
        for (int j = 1; j <= n; j++)
            if (a[0][j] > eps) { c = j; break; }
        if (c == -1) return a[0][0]; // 最优
        // Bland: 选最小下标的满足最小比值的行
        int r = -1; double mn = 1e18;
        for (int i = 1; i <= m; i++)
            if (a[i][c] > eps) {
                double ratio = a[i][0] / a[i][c];
                if (ratio < mn - eps || (fabs(ratio - mn) < eps && i < r))
                    mn = ratio, r = i;
            }
        if (r == -1) return 1e18; // 无界
        pivot(r, c);
    }
}
```

**核心启示**：
- **Bland 规则是理论保证**：它是唯一能在最坏情况下证明不循环的简单规则
- **实际竞赛策略**：结合随机扰动 + Bland 规则，双保险防止退化
- **退化检测**：若连续多次 pivot 目标值不变，说明可能退化，应切换规则

### 题目 14：全幺模矩阵与 LP 整数性（经典理论题）

> 证明：如果约束矩阵 $A$ 是全幺模的，且 $b$ 是整向量，则 LP $\max c^T x, Ax \le b, x \ge 0$ 的每个基可行解都是整数。并应用于网络流证明整数性。

**分析**：基可行解由 $A_B x_B = b$ 确定，$x_B = A_B^{-1} b$。由 Cramer 法则，$x_{B,i} = \det(A_B^{(i)}) / \det(A_B)$。全幺模性 $\Rightarrow$ $|\det(A_B)| = 1$ $\Rightarrow$ $x_{B,i}$ 是整数。

```
核心观察：全幺模 → 基矩阵行列式 = ±1 → 基可行解自动为整数
构造策略：验证约束矩阵的全幺模性（每行每列至多两个非零元素，+1/-1 交替）
正确性：Cramer 法则 + 全幺模定义 → 直接推出整数性
```

**C++ 代码**：

```cpp
// 验证矩阵 A 是否全幺模（检验必要条件：0/±1 矩阵 + 列分划性质）
// Ghouila-Houri 定理：A 全幺模 ⇔ A 是 0/±1 矩阵且
// 每列可分为两组 R1, R2 使每行中 ∑R1 - ∑R2 ∈ {-1, 0, 1}

bool checkTU(int A[][MAXN], int m, int n) {
    // 简化检验：必要条件 — A 全为 0/±1
    for (int i = 0; i < m; i++)
        for (int j = 0; j < n; j++)
            if (A[i][j] != 0 && A[i][j] != 1 && A[i][j] != -1)
                return false;
    // 网络流关联矩阵：每列恰两个非零，一个+1一个-1
    for (int j = 0; j < n; j++) {
        int pos = 0, neg = 0;
        for (int i = 0; i < m; i++) {
            if (A[i][j] == 1) pos++;
            if (A[i][j] == -1) neg++;
        }
        if (pos + neg > 2) return false; // 不满足网络流结构
    }
    return true;
}
```

**核心启示**：
- **全幺模 = 整数性保证**：这是 LP 理论中最深刻的结构定理之一
- **网络流的本质**：网络流问题之所以有整数最优解，根本原因是关联矩阵全幺模
- **判别方法**：直接验证全幺模是 coNP-complete 的，但常见结构（网络矩阵、区间矩阵）有简单判据

### 题目 15：零和博弈的 LP 求解（CF 917D）

> 两人零和博弈，A 有 $n$ 个纯策略，B 有 $m$ 个纯策略。给定收益矩阵 $W$，A 想最大化期望收益，B 想最小化。求博弈均衡值及 A 的最优混合策略。

**分析**：由 von Neumann 的 minimax 定理，$\max_p \min_q p^T W q = \min_q \max_p p^T W q = V$。A 的 LP：$\max V$，约束 $\forall j: \sum_i w_{ij} p_i \ge V$，$\sum p_i = 1$，$p_i \ge 0$。令 $x_i = p_i / V$（假设 $V > 0$），化为标准 LP。

```
核心观察：minimax 定理保证零和博弈有均衡值，可用 LP 求解
构造策略：A 的混合策略 p 作为 LP 变量，B 的每个纯策略 j 产生一个约束
正确性：minimax ↔ LP 对偶，强对偶 ↔ 均衡存在性
```

**C++ 代码**：

```cpp
// 处理 V 可能 ≤ 0 的情况：对 W 加常数 M 使所有元素为正
double W[MAXN][MAXM];
double p[MAXN]; // A 的最优混合策略

double solveGame(int n, int m) {
    double M = 0;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            M = max(M, -W[i][j] + 1);
    // 所有元素加 M 后为正
    // LP: min ∑x_i, s.t. ∑(W[i][j]+M) x_i >= 1 对所有 j, x_i >= 0
    // 构建增广矩阵
    ::n = n; ::m = m;
    for (int j = 1; j <= m; j++) {
        a[j][0] = 1; // 右端 = 1
        for (int i = 1; i <= n; i++)
            a[j][i] = W[i-1][j-1] + M;
    }
    for (int i = 1; i <= n; i++) a[0][i] = -1; // 目标 min ∑x_i → max -∑x_i
    double opt = -simplex(); // opt = 1/V'
    double V_shifted = 1.0 / opt;
    // 恢复策略: p_i = x_i * V_shifted
    for (int i = 1; i <= n; i++) {
        p[i] = 0; // 从基变量中提取
        for (int j = 1; j <= m; j++)
            if (id[n + j] == i) { p[i] = a[j][0] * V_shifted; break; }
    }
    return V_shifted - M; // 减去偏移量
}
```

**核心启示**：
- **偏移技巧**：收益矩阵可能有负值导致 $V \le 0$，加常数偏移保证 $V' > 0$ 后变量替换才合法
- **策略提取**：从单纯形法的基变量中可以直接读出最优策略
- **对偶对称**：B 的最优策略可以从对偶 LP 的解中提取，无需单独求解

## 五、常见技巧与思路总结

### 5.1 退化处理

| 方法 | 实现 | 保证 |
|------|------|------|
| Bland 规则 | 入基/出基都选最小下标 | 理论保证不循环 |
| 随机扰动 | 对 $b$ 加 $\epsilon \cdot \text{rand}$ | 概率不退化 |
| 随机入基 | 从正系数列中随机选 | 期望多项式时间 |
| 字典序法 | 用字典序比较替代比值 | 理论保证不循环 |

### 5.2 对偶转化的时机

- 原问题约束 $m$ 远大于变量 $n$ → 取对偶
- 原问题的对偶有明显组合意义 → 取对偶（如最短路、匹配）
- 需要证明算法最优性 → 用对偶界

### 5.3 LP → 网络流的判别

问自己三个问题：
1. 约束矩阵是否全为 $0, \pm 1$？
2. 每列是否至多两个非零元素？
3. 是否具有网络关联矩阵或区间矩阵的结构？

若三者皆是，大概率可以转化为网络流。

### 5.4 数值精度

- eps 通常取 $10^{-8}$ 到 $10^{-7}$
- pivot 后可能累积误差，必要时重新计算
- 整数 LP 松弛后，若解接近整数（距离 $< 10^{-6}$），可四舍五入

| 信号词 / 题意特征 | 方法 |
|---|---|
| "最大化/最小化 + 线性约束" | 直接 LP（单纯形法） |
| "区间覆盖 + 最小费用" | LP → 费用流（P3980 型） |
| "选子集 + 依赖关系 + 最大权" | 最大权闭合子图 → 最小割 |
| "对手博弈 + 混合策略" | minimax → LP |
| "约束很多变量很少" | 对偶后 LP |
| "匹配 + 权值" | LP 对偶 → KM 算法 |
| "多种资源共享" | 多商品流 LP |
| "切割/装箱 + 方案数巨大" | 列生成 |

### 6.2 解题流程

```
1. 建模：写出变量、目标函数、约束
2. 判型：
   ├─ 全幺模 → 转网络流
   ├─ 博弈型 → minimax → LP
   ├─ 大规模 → 对偶 / 列生成
   └─ 一般型 → 直接单纯形法
3. 实现：选合适的单纯形模板 + 退化处理
4. 验证：检查数值精度 + 对拍
```

## 七、终极总结与归纳

### 7.1 本质

线性规划在凸多面体的顶点上寻找最优解。单纯形法沿多面体的边从顶点到顶点移动，对偶理论揭示问题的深层结构，全幺模性将连续优化与离散优化（网络流）联系起来。

### 7.2 核心要点

1. **单纯形法**：选入基 → 选出基 → pivot → 迭代至最优，竞赛中实际 $O(mn)$
2. **对偶定理**：强对偶 $\max = \min$，互补松弛提供最优性条件
3. **全幺模性**：约束矩阵全幺模 → LP 解自动整数 → 等价网络流
4. **退化处理**：Bland 规则 / 随机化，竞赛中必备
5. **建模能力**：从题意到 LP 模型的转化是核心竞争力

### 7.3 路线图

```
L1 🟢 题1-3：LP 模板 + 图解法 + 志愿者招募（基础）
      ↓
L2 🟡 题4-6：博弈 LP + 随机单纯形 + LP 松弛（进阶）
      ↓
L3 🟠 题7-9：对偶转化 + 贪心证明 + 多商品流（提高）
      ↓
L4 🔴 题10-12：闭合子图 + KM 对偶 + 列生成（困难）
      ↓
L5 🟣 题13-15：Bland 规则 + 全幺模 + 博弈综合（挑战）
```
