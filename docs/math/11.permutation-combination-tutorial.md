## 一、什么是排列组合

排列组合是组合数学（Combinatorics）的基础，研究的是"从 n 个元素中选取 r 个元素"的计数问题。其核心区别在于：**排列关注顺序，组合不关注顺序**。

在算法竞赛中，排列组合是最常见的计数类工具。无论是 DP 计数题、概率期望题，还是博弈论、图论中的计数问题，都离不开排列数 $P(n,r)$、组合数 $C(n,r)$、二项式定理、卢卡斯定理等核心知识。掌握排列组合，等于拥有了解决一切"数数题"的基础武器库。

### 排列组合 vs 其他计数方法对比

| 特征 | 排列组合 | DP 计数 | 容斥原理 | 生成函数 |
|------|----------|---------|----------|----------|
| 适用范围 | 元素选取/排列 | 复杂状态转移计数 | 集合交并计数 | 序列/多项式计数 |
| 公式化程度 | **极高** | 中 | 中高 | 高 |
| 前置知识 | 阶乘、逆元 | 状态设计 | 集合论 | 多项式运算 |
| 典型用途 | 选取、分配、排列 | 方案数、路径数 | "至少一个"型 | 背包计数、分拆数 |

> **一句话理解**：排列组合就是"数清楚有多少种选法和排法"的数学工具，是竞赛计数题的基石。

## 二、核心思想与前置知识

### 2.1 排列数与组合数

**排列数** $P(n,r) = \frac{n!}{(n-r)!}$：从 n 个不同元素中取出 r 个，**有序**排列的方案数。

**组合数** $C(n,r) = \binom{n}{r} = \frac{n!}{r!(n-r)!}$：从 n 个不同元素中取出 r 个，**无序**选取的方案数。

```
基本关系：
P(n,r) = C(n,r) × r!
C(n,r) = C(n, n-r)           （对称性）
C(n,r) = C(n-1,r-1) + C(n-1,r)  （递推/杨辉三角）
```

```cpp
// 预处理阶乘与逆元（模 p 意义下）
const int MAXN = 1e6 + 5;
const long long MOD = 1e9 + 7;
long long fac[MAXN], inv_fac[MAXN];

long long qpow(long long a, long long b, long long mod) {
    long long res = 1; a %= mod;
    for (; b > 0; b >>= 1) {
        if (b & 1) res = res * a % mod;
        a = a * a % mod;
    }
    return res;
}

void init(int n) {
    fac[0] = 1;
    for (int i = 1; i <= n; i++) fac[i] = fac[i - 1] * i % MOD;
    inv_fac[n] = qpow(fac[n], MOD - 2, MOD);
    for (int i = n - 1; i >= 0; i--) inv_fac[i] = inv_fac[i + 1] * (i + 1) % MOD;
}

long long C(int n, int r) {
    if (r < 0 || r > n) return 0;
    return fac[n] % MOD * inv_fac[r] % MOD * inv_fac[n - r] % MOD;
}
```

### 2.2 卢卡斯定理

当模数 p 是**质数**且 n 很大时，直接计算 $C(n,r) \bmod p$ 可能溢出。卢卡斯定理给出分解方案：

$$C(n,r) \equiv C(\lfloor n/p \rfloor, \lfloor r/p \rfloor) \cdot C(n \bmod p, r \bmod p) \pmod{p}$$

递归终止条件：当 $n = 0$ 时返回 1；当 $r > n$ 时返回 0。

```cpp
// Lucas 定理（p 为质数且较小）
long long lucas(long long n, long long r, long long p) {
    if (r == 0) return 1;
    return lucas(n / p, r / p, p) % p * C(n % p, r % p, p) % p;
}
// C(n,r,p)：p 较小时用预处理阶乘；或直接 O(r) 计算
```

### 2.3 二项式定理与组合恒等式

**二项式定理**：

$$(a + b)^n = \sum_{k=0}^{n} \binom{n}{k} a^k b^{n-k}$$

**常用组合恒等式**：

| 恒等式 | 公式 |
|--------|------|
| 对称性 | $\binom{n}{k} = \binom{n}{n-k}$ |
| 吸收恒等式 | $k\binom{n}{k} = n\binom{n-1}{k-1}$ |
| 范德蒙德卷积 | $\sum_{i=0}^{r}\binom{m}{i}\binom{n}{r-i} = \binom{m+n}{r}$ |
| 上指标求和 | $\sum_{i=r}^{n}\binom{i}{r} = \binom{n+1}{r+1}$ |
| 二项式求和 | $\sum_{k=0}^{n}\binom{n}{k} = 2^n$ |
| 交替求和 | $\sum_{k=0}^{n}(-1)^k\binom{n}{k} = 0$ |

### 2.4 多重集排列与组合

**多重集排列（可重排列）**：n 个元素中有 $n_1, n_2, \ldots, n_k$ 个相同的（$\sum n_i = n$），全排列数为：

$$\frac{n!}{n_1! \cdot n_2! \cdot \ldots \cdot n_k!}$$

**多重集组合（隔板法/星与条）**：从 k 种元素中选 n 个（每种无限多），方案数为：

$$\binom{n+k-1}{k-1}$$

### 2.5 生成函数入门

生成函数将组合计数问题转化为多项式运算。序列 $\{a_0, a_1, a_2, \ldots\}$ 的**普通生成函数**（OGF）为：

$$A(x) = \sum_{n=0}^{\infty} a_n x^n$$

其中 $a_n = \binom{n+k-1}{k-1}$（多重集组合数）对应生成函数 $\frac{1}{(1-x)^k}$。

生成函数在竞赛中常用于：将"选取方案数"转化为多项式乘法问题，结合 FFT/NTT 加速求解。

## 三、分类与变体详解

### 3.1 排列数与组合数的直接计算

最基本的场景：直接套用公式计算 $P(n,r)$ 或 $C(n,r)$，涉及阶乘预处理和模逆元。

```
核心观察：当 n, r ≤ 10^6 且模数为大质数时，预处理阶乘 + 逆元即可 O(1) 查询
构造策略：预处理 fac[] 和 inv_fac[]，再由公式直接算
正确性：费马小定理保证逆元存在（p 为质数）
```

### 3.2 卢卡斯定理（大 n 小 p）

当 n 极大（如 $10^{18}$）而模数 p 较小（如 $\leq 10^5$）时，卢卡斯定理将大组合数分解为多个小组合数的乘积。

```
核心观察：C(n,r) mod p 可递归分解为 p 进制各位的组合数之积
构造策略：逐位取 n mod p、r mod p，递归计算
正确性：卢卡斯定理要求 p 为质数
```

```cpp
// 卢卡斯定理完整实现
long long fac_small[MAXP], inv_small[MAXP]; // MAXP 为质数 p 的大小

void init_lucas(int p) {
    fac_small[0] = 1;
    for (int i = 1; i < p; i++) fac_small[i] = fac_small[i - 1] * i % p;
    inv_small[p - 1] = qpow(fac_small[p - 1], p - 2, p);
    for (int i = p - 2; i >= 0; i--) inv_small[i] = inv_small[i + 1] * (i + 1) % p;
}

long long C_small(int n, int r, int p) {
    if (r > n) return 0;
    return fac_small[n] * inv_small[r] % p * inv_small[n - r] % p;
}

long long lucas(long long n, long long r, int p) {
    if (r == 0) return 1;
    return lucas(n / p, r / p, p) % p * C_small(n % p, r % p, p) % p;
}
```

### 3.3 组合恒等式与变换

利用各种组合恒等式化简求和式，是竞赛中组合数学题的核心技巧。常见方法包括：

- **吸收恒等式**：消去求和变量的系数
- **范德蒙德卷积**：将双求和化为单个组合数
- **上指标求和**：处理"前缀组合数之和"

```
核心观察：复杂计数式往往可以通过恒等式化简为封闭形式
构造策略：识别求和式的结构，匹配已知恒等式模式
正确性：每个恒等式都有组合证明或代数证明
```

### 3.4 容斥原理与排列组合

当计数"满足某些条件"的方案数较难直接计算时，用**容斥原理**转化为"总方案数 - 不满足条件的方案数"：

$$|A_1 \cup A_2 \cup \ldots \cup A_n| = \sum|A_i| - \sum|A_i \cap A_j| + \ldots + (-1)^{n+1}|A_1 \cap \ldots \cap A_n|$$

```
核心观察：正难则反，补集计数
构造策略：计算"至少违反 k 个限制"的方案数，用容斥合并
正确性：容斥公式保证不重不漏
```

### 3.5 错排问题与特殊排列

**错排（Derangement）**：n 个元素的全排列中，每个元素都不在原位的排列数 $D_n$：

$$D_n = n!\sum_{k=0}^{n}\frac{(-1)^k}{k!} = (n-1)(D_{n-1} + D_{n-2})$$

其中 $D_0 = 1, D_1 = 0$。

```cpp
// 错排数预处理
long long D[MAXN];
void init_derangement(int n) {
    D[0] = 1; D[1] = 0;
    for (int i = 2; i <= n; i++)
        D[i] = (i - 1) % MOD * (D[i - 1] + D[i - 2]) % MOD;
}
```

### 3.6 生成函数与多项式方法

将计数问题建模为多项式系数的提取问题。典型应用：

- **背包计数**：每种物品对应一个多项式因子，总方案数 = 乘积多项式某项系数
- **分拆数**：整数分拆方案数可用生成函数的欧拉乘积表示
- **Catalan 数**：$C_n = \frac{1}{n+1}\binom{2n}{n}$，其生成函数为 $\frac{1-\sqrt{1-4x}}{2x}$

```
核心观察：方案数 = 多项式乘积的某项系数
构造策略：为每种选择方式写出生成函数，相乘后提取系数
正确性：多项式乘法对应独立选择的方案数相乘
```

### 变体对比总表

| 变体 | 适用场景 | 核心公式/方法 | 时间复杂度 |
|------|----------|---------------|------------|
| 直接计算 | n ≤ 10^6, p 大质数 | 预处理阶乘 + 逆元 | O(n) 预处理, O(1) 查询 |
| 卢卡斯 | n 极大, p 较小质数 | 递归分解 | O(p + log_p(n)) |
| 杨辉三角 | n ≤ 5000, 求多组 C | 递推 C[i][j] | O(n^2) |
| 容斥 | 带限制条件的计数 | 总数 - 违反数 | 视问题而定 |
| 生成函数 | 多项式计数 | FFT/NTT | O(n log n) |
| 错排 | 无固定点排列 | 递推 D[n] | O(n) |

### 🟢 入门级 Level 1：基本组合数计算

### 题目 1：【模板】卢卡斯定理（洛谷 P3807）

> 给定 n, m, p，求 $C(n+m, m) \bmod p$，其中 p 为质数。多组查询。

**分析**：卢卡斯定理的模板题。由于 p 可能较小（$\leq 10^5$），而 n, m 可达 $10^5$，直接预处理 p 以内的阶乘和逆元，然后递归应用卢卡斯定理。

```
核心观察：p 为质数且较小，C(n+m, m) 中 n+m 可能大于 p
构造策略：对 n+m 和 m 做 p 进制分解，逐位计算组合数之积
正确性：卢卡斯定理保证递归分解的正确性
```

**C++ 代码**：

```cpp
long long fac[MAXP], ifac[MAXP];

void init(int p) {
    fac[0] = 1;
    for (int i = 1; i < p; i++) fac[i] = fac[i - 1] * i % p;
    ifac[p - 1] = qpow(fac[p - 1], p - 2, p);
    for (int i = p - 2; i >= 0; i--) ifac[i] = ifac[i + 1] * (i + 1) % p;
}

long long C(int n, int r, int p) {
    if (r > n) return 0;
    return fac[n] * ifac[r] % p * ifac[n - r] % p;
}

long long lucas(long long n, long long r, int p) {
    if (r == 0) return 1;
    return lucas(n / p, r / p, p) * C(n % p, r % p, p) % p;
}
// 调用 init(p); lucas(n + m, m, p)
```

**核心启示**：
- **卢卡斯定理的适用条件**：p 必须是质数，且通常 p 较小
- **预处理范围与 p 相关**：只需预处理 [0, p) 内的阶乘和逆元
- **递归终止**：r == 0 时返回 1，处理好边界

### 题目 2：组合数问题（洛谷 P2822）

> 给定 t 组查询，每次给 n, m，求满足 $k \bmod p = 0$ 的 $\binom{i}{j}$（$1 \leq i \leq n, 0 \leq j \leq \min(i, m)$）的个数，其中 p 在全局给定且 p 为质数。

**分析**：预处理杨辉三角 $C[i][j] \bmod p$，同时维护前缀和。$C[i][j] \bmod p = 0$ 当且仅当 $C[i][j]$ 是 p 的倍数。用二维前缀和预处理，每次 O(1) 回答。

```
核心观察：C(i,j) mod p = 0 即杨辉三角中该位置为 0
构造策略：预处理杨辉三角 mod p + 二维前缀和
正确性：杨辉三角递推保证正确性，前缀和实现 O(1) 查询
```

**C++ 代码**：

```cpp
const int MAXN = 2005;
int c[MAXN][MAXN], s[MAXN][MAXN]; // c: 组合数 mod p, s: 前缀和

void init(int n, int p) {
    for (int i = 0; i <= n; i++) {
        c[i][0] = 1;
        for (int j = 1; j <= i; j++)
            c[i][j] = (c[i - 1][j - 1] + c[i - 1][j]) % p;
    }
    for (int i = 1; i <= n; i++)
        for (int j = 0; j <= i; j++)
            s[i][j] = s[i - 1][j] + s[i][j - 1] - s[i - 1][j - 1] + (c[i][j] == 0);
    // 注意 s 的下标处理，j > i 时 c[i][j] = 0 也要计入
}
// 查询 [1..n][0..m] 中 c[i][j]==0 的个数：s[n][m]
```

**核心启示**：
- **杨辉三角预处理**：当 n 较小（$\leq 2000$）时，O(n^2) 递推是最稳妥的方法
- **二维前缀和**：多组查询时，预处理前缀和将每次查询降至 O(1)
- **模 p 为 0 的判定**：直接在递推时取模，判断是否等于 0

### 题目 3：青原樱（洛谷 P5520 [yLOI2019]）

> n 棵樱花树排成一排，相邻两棵之间可以种花或者不种。给定总花数 m 和树数 n，求在 n-1 个间隔中放 m 朵花（每个间隔最多放 1 朵）的方案数，对 $10^9 + 7$ 取模。

**分析**：这是经典的"从 n-1 个位置中选 m 个"的组合问题，答案就是 $C(n-1, m)$。需要用预处理阶乘和逆元来计算大组合数。

```
核心观察：n-1 个间隔选 m 个放花，经典组合数
构造策略：预处理阶乘和逆元，直接计算 C(n-1, m)
正确性：每个间隔最多放 1 朵 ↔ 不重复选取 ↔ 组合数
```

**C++ 代码**：

```cpp
const long long MOD = 1e9 + 7;
long long fac[MAXN], ifac[MAXN];

// init(n) 预处理阶乘和逆元（见 2.1 节）
// 答案 = C(n - 1, m)
long long ans = fac[n - 1] % MOD * ifac[m] % MOD * ifac[n - 1 - m] % MOD;
```

**核心启示**：
- **问题建模**：将实际问题转化为"从 n-1 个位置中选 m 个"的组合模型
- **可行性判断**：若 m > n-1 则无解，需特判
- **大组合数取模**：阶乘预处理 + 逆元是标准做法

### 题目 4：越狱（洛谷 P3197 [HNOI2008]）

> n 个囚犯关在一排 n 间牢房中，每间有 m 种宗教可选。如果相邻两间牢房的囚犯信同一宗教就会越狱。求有多少种信仰分配方案会导致越狱，答案对 $10^9 + 7$ 取模。

**分析**：正难则反。总方案数为 $m^n$（每个人 m 种选择），不会越狱的方案数为：第一个人 m 种选择，之后每个人必须与前一个不同，有 m-1 种选择，即 $m \times (m-1)^{n-1}$。答案 = $m^n - m \times (m-1)^{n-1}$。

```
核心观察：直接计数"存在相邻相同"很难，转为补集计数
构造策略：总方案 m^n 减去"全部不相邻相同"的方案 m*(m-1)^(n-1)
正确性：补集计数不重不漏
```

**C++ 代码**：

```cpp
const long long MOD = 1e9 + 7;

long long solve(long long n, long long m) {
    long long total = qpow(m, n, MOD);
    long long safe = m % MOD * qpow(m - 1, n - 1, MOD) % MOD;
    return (total - safe + MOD) % MOD;
}
```

**核心启示**：
- **补集计数**：当"满足条件"的方案数难以直接计算时，用"总数 - 不满足条件"
- **独立选择的乘法原理**：每个人独立选宗教，方案数相乘
- **快速幂处理大指数**：$n$ 可达 $10^{12}$，必须用快速幂

### 题目 5：树的计数（洛谷 P2290 [HNOI2004]）

> 已知一棵 n 个节点的有标号无根树的 Prufer 序列（长度 n-2），求对应的无根树数量。即给定每个节点的度数 $d_1, d_2, \ldots, d_n$，求有多少棵有标号无根树满足各节点度数恰好为给定值。

**分析**：由 Prufer 序列与有标号无根树的一一对应关系，节点 i 在 Prufer 序列中出现 $d_i - 1$ 次。问题转化为：长度 n-2 的序列中，数字 i 恰好出现 $d_i - 1$ 次的排列数，即多重集排列数：

$$\frac{(n-2)!}{\prod_{i=1}^{n}(d_i-1)!}$$

前提条件：$\sum(d_i - 1) = n - 2$，且每个 $d_i \geq 1$。

```
核心观察：Prufer 序列中节点 i 出现 d_i - 1 次，问题化为多重集排列
构造策略：验证度数和条件后，直接计算多重集排列数
正确性：Prufer 序列与有标号无根树一一对应
```

**C++ 代码**：

```cpp
// d[i]: 节点 i 的度数，n: 节点数
long long solve(int n, int d[]) {
    long long sum = 0;
    for (int i = 1; i <= n; i++) sum += d[i] - 1;
    if (sum != n - 2) return 0; // 无解
    long long ans = fac[n - 2];
    for (int i = 1; i <= n; i++)
        ans = ans % MOD * ifac[d[i] - 1] % MOD;
    return ans;
}
```

**核心启示**：
- **Prufer 序列**：连接树与序列的桥梁，将树的计数转化为序列排列计数
- **多重集排列公式**：$\frac{n!}{\prod n_i!}$ 是经典的可重排列计数
- **合法性预判**：先验证度数和是否等于 n-2

### 题目 6：组合数求和（洛谷 P1118 [USACO06FEB] 数字三角形）

> 给定 n，将 1~n 的一个排列填入三角形第一行，每行相邻两数相加得到下一行，求使最后一行结果为给定值 S 的字典序最小排列。

**分析**：第一行第 i 个数对最终结果的贡献系数恰好是 $C(n-1, i-1)$。即 $S = \sum_{i=1}^{n} a_i \cdot C(n-1, i-1)$。在已知系数的情况下，按字典序从小到大尝试每个位置填什么数字（DFS + 剪枝）。

```
核心观察：杨辉三角展开后，第 i 个数的权重系数 = C(n-1, i-1)
构造策略：预处理组合数权重，DFS 搜索字典序最小排列
正确性：组合数系数由杨辉三角递推保证
```

**C++ 代码**：

```cpp
int n, S, ans[25];
long long w[25]; // w[i] = C(n-1, i-1)
bool used[25], found;

void dfs(int pos, long long rem) {
    if (found) return;
    if (pos > n) { if (rem == 0) found = true; return; }
    for (int v = 1; v <= n; v++) {
        if (used[v]) continue;
        if (v * w[pos] > rem) continue; // 剪枝
        used[v] = true; ans[pos] = v;
        dfs(pos + 1, rem - v * w[pos]);
        if (found) return;
        used[v] = false;
    }
}
// 预处理 w[i] = C(n-1, i-1)，调用 dfs(1, S)
```

**核心启示**：
- **组合数作为权重**：杨辉三角的层层叠加产生组合数系数
- **搜索 + 剪枝**：在排列组合系数已知的条件下，用搜索求具体方案
- **字典序最小**：从小到大尝试填数，第一个可行解即为答案

### 题目 7：可重集组合数（洛谷 P1771 方程的整数解）

```
核心观察：正整数解 ↔ n 个球放 k-1 个隔板 ↔ C(n-1, k-1)
```

```cpp
// 正整数解：C(n-1, k-1)
// 非负整数解：C(n+k-1, k-1)
```

**核心启示**：

### 题目 9：【XR-4】题（洛谷 P5596）

```
核心观察：组合数前缀和，直接枚举累加
```

```cpp
long long solve(int n, int m) {

}
```

**核心启示**：

### 题目 10：【XR-4】混乱度（洛谷 P5598）

> 给定 n 和 k，定义排列的"混乱度"为排列中满足 $|p_i - i| \geq k$ 的位置个数。求所有 n! 个排列的混乱度之和，答案模 $998244353$。

**分析**：由期望的线性性，将"所有排列的混乱度之和"拆解为每个位置的贡献。对于位置 i，满足 $|p_i - i| \geq k$ 的 $p_i$ 值有若干个（设为 $c_i$ 个），每个值出现在位置 i 的概率为 $\frac{(n-1)!}{n!}$，对应方案数为 $(n-1)!$。

具体地，位置 i 满足 $|v - i| \geq k$ 的合法值 v 的个数 $c_i = n - |[max(1, i-k+1), min(n, i+k-1)]|$，即总数 n 减去区间 $[i-k+1, i+k-1] \cap [1, n]$ 的长度。

答案 = $\sum_{i=1}^{n} c_i \cdot (n-1)!$。

```
核心观察：拆解每个位置的贡献，用 (n-1)! 乘以合法值个数
构造策略：对每个位置 i 计算 |v-i| >= k 的 v 的个数 c_i
正确性：线性性保证拆分正确，乘法原理保证方案数
```

**C++ 代码**：

```cpp
long long solve(int n, int k) {
    long long ans = 0;
    for (int i = 1; i <= n; i++) {
        int lo = max(1, i - k + 1), hi = min(n, i + k - 1);
        int bad = hi - lo + 1; // |v - i| < k 的个数
        int good = n - bad;    // |v - i| >= k 的个数
        ans = (ans + (long long)good % MOD * fac[n - 1]) % MOD;
    }
    return ans;
}
```

**核心启示**：
- **贡献拆分法**：求所有方案的某度量之和，拆解为每个位置/元素的贡献
- **线性性 (Linearity)**：$\sum$ 可以拆开，化整体计数为局部计数
- **区间计算**：$|v - i| < k$ 对应一个连续区间，长度易算

### 题目 11：二项式反演 — 至多与恰好（CSES Counting Sequences）

> 求长度为 n 的序列（每个元素 $\in [1, m]$），恰好包含 k 个不同值的序列数量，对 $10^9 + 7$ 取模。

**分析**：先算"至多用 k 种值"的方案数 $f(k) = \binom{m}{k} \cdot k^n$（从 m 种中选 k 种，再排列），然后用二项式反演得到"恰好 k 种"的方案数 $g(k) = \sum_{i=0}^{k}(-1)^{k-i}\binom{k}{i} \cdot i^n$，最后乘以 $\binom{m}{k}$。

```
核心观察：至多 k 种 → 恰好 k 种，需要二项式反演（容斥）
构造策略：g(k) = Σ(-1)^(k-i) * C(k,i) * i^n，再乘 C(m,k)
正确性：二项式反演保证恰好计数的正确性
```

**C++ 代码**：

```cpp
long long solve(int n, int m, int k) {
    long long ans = 0;
    for (int i = 0; i <= k; i++) {
        long long term = C(k, i) % MOD * qpow(i, n, MOD) % MOD;
        if ((k - i) & 1) ans = (ans - term + MOD) % MOD;
        else ans = (ans + term) % MOD;
    }
    return ans % MOD * C(m, k) % MOD;
}
```

**核心启示**：
- **二项式反演**：从"至多"到"恰好"的标准工具
- **容斥与反演的等价性**：二项式反演本质上是容斥原理在组合数上的推广
- **框架识别**：看到"恰好 k 种/个"，优先考虑容斥或反演

### 题目 13：冒泡排序（洛谷 P4769 [NOI2018]）

> 给定排列 p 的前缀 $q$（长度为 n 的排列的一个长度为 n 的排列），计算满足以下条件的排列数：冒泡排序的交换次数恰好等于排列的逆序对数。更具体地：统计长度为 n 的排列中，其前缀最大值序列字典序不超过给定序列 $q$ 的排列个数。（答案对 $998244353$ 取模）

**分析**：这是 NOI2018 的一道组合计数难题。关键观察：一个排列能在恰好"逆序对数"次冒泡后排好序，等价于排列中不存在长度 $\geq 3$ 的递减子序列（即排列可分解为至多两个递增子序列）。统计这类排列中字典序受限的方案数，需要利用 Catalan 数和 DP 结合。

核心结论：满足条件的排列数与"从 (0,0) 到 (n,n) 的不超过对角线的格路"相关，涉及 Catalan 数的推广和树状数组维护的 DP。

```
核心观察：冒泡排序次数 = 逆序对数 ⟺ 不存在长度 ≥ 3 的递减子序列
构造策略：利用 Catalan 数和格路计数，结合树状数组维护字典序约束
正确性：双射将排列映射到不超过对角线的格路
```

**C++ 代码**：

```cpp
// dp[i] 表示前 i 个位置，前缀最大值满足约束的方案数
// 利用树状数组维护后缀和
long long dp[MAXN], bit[MAXN];
int n;

void update(int x, long long v) {
    for (; x > 0; x -= x & -x) bit[x] = (bit[x] + v) % MOD;
}
long long query(int x) {
    long long s = 0;
    for (; x <= n; x += x & -x) s = (s + bit[x]) % MOD;
    return s;
}

void solve(int q[]) {
    // 离散化 + DP，结合 Catalan 数计算
    // dp[i] 依赖于 q 的前缀最大值约束
    long long ans = 0;
    for (int i = 1; i <= n; i++) {
        long long ways = query(q[i] + 1); // 满足字典序约束的方案
        dp[i] = ways % MOD * C(2 * n - 2 * i, n - i) % MOD;
        // 乘以 Catalan 相关系数
        ans = (ans + dp[i]) % MOD;
        update(q[i], dp[i]);
    }
}
```

**核心启示**：
- **性质等价转化**：冒泡排序次数 = 逆序对数 ⟺ 避免长递减子序列，这是关键观察
- **Catalan 数与格路**：受限排列计数常与 Catalan 数及格路计数相关
- **树状数组优化 DP**：当 DP 转移涉及前缀/后缀查询时，用数据结构加速

### 题目 14：情侣？给我烧了！加强版（洛谷 P4931）

> $2n$ 个人坐成一排（$n$ 对情侣），恰好有 k 对情侣坐在相邻位置的方案数是多少？多组查询，答案对 $998244353$ 取模。

**分析**：经典的错排推广问题。用容斥原理：

设 $f(k)$ = 恰好 k 对情侣相邻的方案数。先计算 $g(k)$ = 至少 k 对情侣相邻的方案数：
- 选 k 对情侣绑定：$C(n, k)$
- 每对内部两人可交换：$2^k$
- 绑定后有 $2n - k$ 个"元素"全排列：$(2n - k)!$

所以 $g(k) = C(n, k) \cdot 2^k \cdot (2n - k)!$。

由容斥反演：$f(k) = \sum_{i=k}^{n} (-1)^{i-k} C(i, k) \cdot g(i)$。

```
核心观察：将情侣对视为绑定/不绑定，用容斥从"至少"推"恰好"
构造策略：g(k) = C(n,k) * 2^k * (2n-k)!，再容斥反演得 f(k)
正确性：容斥原理保证恰好 k 对的准确性
```

**C++ 代码**：

```cpp
// 预处理 fac[], ifac[], pow2[]
long long pow2[MAXN];

void init(int n) {
    // fac, ifac 预处理（同前）
    pow2[0] = 1;
    for (int i = 1; i <= n; i++) pow2[i] = pow2[i - 1] * 2 % MOD;
}

long long g(int n, int k) {
    return C(n, k) % MOD * pow2[k] % MOD * fac[2 * n - k] % MOD;
}

long long f(int n, int k) {
    long long ans = 0;
    for (int i = k; i <= n; i++) {
        long long term = C(i, k) % MOD * g(n, i) % MOD;
        if ((i - k) & 1) ans = (ans - term + MOD) % MOD;
        else ans = (ans + term) % MOD;
    }
    return ans;
}
```

**核心启示**：
- **绑定法**：将"相邻的一对"绑定为一个元素，简化问题
- **"至少"到"恰好"的容斥**：$f(k) = \sum (-1)^{i-k} C(i,k) g(i)$ 是经典套路
- **预处理加速**：2 的幂次和阶乘提前预处理，避免重复计算

### 题目 15：Catalan 数应用 — 合法括号序列（LeetCode 22 变体）

> 求 n 对括号能组成的合法括号序列数量。

**分析**：经典 Catalan 数问题。合法括号序列数 $= C_n = \frac{1}{n+1}\binom{2n}{n}$。

Catalan 数还等于：从 (0,0) 走到 (n,n)，每步 →或↑，不穿越对角线的路径数。

```
核心观察：合法括号序列 ↔ 不穿越对角线的格路 ↔ Catalan 数
构造策略：C_n = C(2n, n) / (n + 1)
正确性：反射原理证明 Catalan 数公式
```

**C++ 代码**：

```cpp
// Catalan(n) = C(2n, n) / (n+1)
long long catalan(int n) {
    return C(2 * n, n) % MOD * qpow(n + 1, MOD - 2, MOD) % MOD;
}
```

**核心启示**：
- **Catalan 数的多种等价形式**：$\frac{1}{n+1}\binom{2n}{n} = \binom{2n}{n} - \binom{2n}{n-1}$
- **反射原理**：不合法路径通过"反射"变换与另一组路径一一对应
- **广泛应用**：括号匹配、栈排序、二叉树计数、多边形三角剖分等

### 题目 16：错排信封问题（经典数论 错排公式）

> n 封信放入 n 个信封，每封信都不在自己对应的信封中，共有多少种放法？

**分析**：经典错排问题。答案 $D_n = (n-1)(D_{n-1} + D_{n-2})$，其中 $D_0 = 1, D_1 = 0$。

也可用容斥公式：$D_n = n! \sum_{k=0}^{n} \frac{(-1)^k}{k!}$。

```
核心观察：错排 = 所有元素都不在原位的排列
构造策略：递推 D[n] = (n-1)(D[n-1] + D[n-2]) 或容斥公式
正确性：容斥原理排除"至少一个在原位"的情况
```

**C++ 代码**：

```cpp
long long D[MAXN];
void init_derangement(int n) {
    D[0] = 1; D[1] = 0;
    for (int i = 2; i <= n; i++)
        D[i] = (long long)(i - 1) % MOD * ((D[i - 1] + D[i - 2]) % MOD) % MOD;
}
// D[n] 即为答案
```

**核心启示**：
- **递推效率**：$D_n = (n-1)(D_{n-1} + D_{n-2})$ 比容斥公式更高效（O(n) 预处理）
- **容斥视角**：$D_n = n! - C(n,1)(n-1)! + C(n,2)(n-2)! - \ldots$ 展开化简即得公式
- **与排列的关系**：$D_n / n! \to 1/e$（当 n 大时错排概率趋近 $1/e \approx 0.368$）

### 题目 17：排列计数 — 有禁位排列（洛谷 P4071 [SDOI2016] 排列计数）

> n 个元素的排列中，恰好有 m 个元素在原位（即 $p_i = i$）的排列个数，答案对 $10^9 + 7$ 取模。多组查询。

**分析**：从 n 个位置中选 m 个作为固定点（$C(n, m)$），剩下 n-m 个元素做错排（$D_{n-m}$）。答案 = $C(n, m) \times D_{n-m}$。

```
核心观察：恰好 m 个固定点 = C(n,m) × 错排 D(n-m)
```

```cpp
// 预处理 fac[], ifac[], D[]（错排数）
long long solve(int n, int m) {

}
```

**核心启示**：

## 五、常见技巧与思路总结

### 5.1 补集计数法（正难则反）

当直接计算"满足条件"的方案困难时，用"总方案 - 不满足条件的方案"。

```
模式：题目要求"至少存在..."/"不全为..."
公式：答案 = 全集 - 补集
例：越狱问题 → m^n - m(m-1)^(n-1)
```

### 5.2 贡献拆分法

求所有方案的"度量之和"时，将总贡献拆解为每个元素/位置的独立贡献。

```
模式：题目要求"所有方案的 XXX 之和"
公式：Σ_方案 f(方案) = Σ_元素 贡献(元素)
例：混乱度问题 → 每个位置贡献 c_i × (n-1)!
```

### 5.3 绑定法

将强关联的元素绑定为一个"超级元素"，简化排列计数。

```
模式：题目涉及"相邻对"/"捆绑"
方法：绑定后全排列，乘以内部排列数
例：情侣问题 → 绑定 k 对后排列 (2n-k)! × 2^k
```

### 5.4 容斥反演："至少"到"恰好"

$$f(k) = \sum_{i=k}^{n} (-1)^{i-k} \binom{i}{k} g(i)$$

其中 $g(i)$ 是"至少 i 个满足条件"的方案数。

```
模式：题目要求"恰好 k 个满足..."
前提：g(k)（至少 k 个）易算
例：恰好 k 对情侣相邻 → 至少 k 对绑定容易算
```

### 5.5 Prufer 序列与树计数

利用 Prufer 序列将有标号树计数转化为序列计数。

```
Cayley 公式：n 个有标号节点的无根树数 = n^(n-2)
节点度数约束：度为 d_i 的树数 = (n-2)! / Π(d_i - 1)!
```

### 5.6 杨辉三角与组合恒等式

利用 $C(n,r) = C(n-1,r-1) + C(n-1,r)$ 递推，以及各种恒等式化简求和式。

```
常用恒等式识别：
- 看到 Σ C(n,i) 形式 → 二项式定理
- 看到 Σ C(m,i)C(n,r-i) → 范德蒙德卷积
- 看到 Σ C(i,r) 从 i=r 到 n → 上指标求和
```

## 七、终极总结与归纳

### 7.1 排列组合的本质

排列组合是**将"有多少种选法/排法"这一问题数学化**的核心工具。它的本质在于：通过阶乘、逆元、递推、容斥等手段，把"数数"变成"算数"。

掌握排列组合，不仅仅是记住 $C(n,r)$ 的公式，更重要的是：

- 识别问题属于哪种计数模型（选取？排列？分配？禁位？）
- 选择合适的计算工具（直接公式？容斥？生成函数？）
- 处理好取模、大数、预处理等工程细节

### 7.2 核心要点

1. **排列关注顺序，组合不关注顺序**：$P(n,r) = C(n,r) \times r!$
2. **预处理阶乘和逆元**：大多数组合数题的基础操作，$O(n)$ 预处理后 $O(1)$ 查询
3. **正难则反**：正向计数困难时，用补集计数（总方案 - 不合法方案）
4. **容斥反演**：从"至少"到"恰好"的标准工具，$f(k) = \sum (-1)^{i-k} C(i,k) g(i)$
5. **卢卡斯定理**：大 n 小 p 时的组合数计算利器
6. **贡献拆分与线性性**：将总贡献拆解为各元素独立贡献之和
