## 一、什么是容斥原理

容斥原理（Inclusion-Exclusion Principle）是组合数学中处理**集合并集计数**的核心工具。基本思想：先加上每个集合的大小，再减去两两交集，加回三三交集……交替进行直到消除所有重复。

$$|A_1 \cup A_2 \cup \cdots \cup A_n| = \sum_{i} |A_i| - \sum_{i<j} |A_i \cap A_j| + \cdots + (-1)^{n+1} |A_1 \cap \cdots \cap A_n|$$

在竞赛中，容斥原理远不止简单的集合计数——它与**二项式反演**、**Min-Max 容斥**、**子集卷积**等高级工具深度结合，是解决"恰好"型计数问题的万能钥匙。

| 变体 | 公式 | 应用场景 |
|------|------|---------|
| 基本容斥 | $\|\bigcup A_i\| = \sum (-1)^{\|S\|+1} \|\bigcap_{i \in S} A_i\|$ | 至少一个条件成立 |
| 补集容斥 | $\|U\| - \|\bigcup A_i\|$ | 所有条件都不成立 |
| 二项式反演 | $f(n) = \sum \binom{n}{k} g(k) \Leftrightarrow g(n) = \sum (-1)^{n-k} \binom{n}{k} f(k)$ | "恰好"与"至少"互转 |
| Min-Max 容斥 | $\max(S) = \sum (-1)^{\|T\|+1} \min(T)$ | 覆盖时间/最大值期望 |

## 二、核心思想与前置知识

### 2.1 基本容斥

设全集 $U$，条件集 $A_1, \ldots, A_m$。"不满足任何条件"的元素数 = $|U| - |\bigcup A_i|$。

当 $m$ 较小（$\le 20$）时，直接 $2^m$ 枚举子集即可。

### 2.2 二项式反演

**第一形式**：$f(n) = \sum_{k=0}^{n} \binom{n}{k} g(k) \Leftrightarrow g(n) = \sum_{k=0}^{n} (-1)^{n-k} \binom{n}{k} f(k)$

**第二形式**：$f(k) = \sum_{i=k}^{n} \binom{i}{k} g(i) \Leftrightarrow g(k) = \sum_{i=k}^{n} (-1)^{i-k} \binom{i}{k} f(i)$

关键应用：设 $f(k)$ = "至少 $k$ 个条件成立的方案数"，$g(k)$ = "恰好 $k$ 个条件成立的方案数"，则两者可通过二项式反演互转。

### 2.3 Min-Max 容斥

$$E[\max(S)] = \sum_{\emptyset \neq T \subseteq S} (-1)^{|T|+1} E[\min(T)]$$

其中 $\min(T) = \min_{i \in T} X_i$。在随机过程中，$E[\min(T)]$ 通常比 $E[\max(S)]$ 容易计算（如通过吸收态高斯消元）。

### 2.4 子集反演与高维前缀和

$$f(S) = \sum_{T \subseteq S} g(T) \Leftrightarrow g(S) = \sum_{T \subseteq S} (-1)^{|S|-|T|} f(T)$$

高维前缀和（SOS DP）：$O(n \cdot 2^n)$ 计算子集和或超集和。

```cpp
// 子集和（高维前缀和）
for (int i = 0; i < n; i++)
    for (int S = 0; S < (1 << n); S++)
        if (S >> i & 1) f[S] += f[S ^ (1 << i)];
```

## 三、分类与变体详解

### 3.1 基本容斥（枚举子集）

> 适用条件：条件数 $m \le 20$
> 方法：枚举 $2^m$ 个子集，交替加减
> 时间复杂度：$O(2^m \cdot \text{单次计算})$

### 3.2 二项式反演

> 适用条件："恰好 $k$ 个" $\leftrightarrow$ "至少 $k$ 个" 互转
> 方法：$f(k) = \sum C(i,k)g(i) \leftrightarrow g(k) = \sum(-1)^{i-k}C(i,k)f(i)$
> 时间复杂度：$O(n^2)$ 或 $O(n \log n)$（NTT 加速）

### 3.3 Min-Max 容斥

> 适用条件：求覆盖时间 / 最大值期望
> 方法：$E[\max] = \sum(-1)^{|T|+1} E[\min_T]$
> 时间复杂度：$O(2^n \cdot \text{单次} E[\min])$

### 3.4 子集卷积

> 适用条件：$h(S) = \sum_{T \subseteq S} f(T) \cdot g(S \setminus T)$
> 方法：按 popcount 分层 + 高维前缀和
> 时间复杂度：$O(n^2 \cdot 2^n)$

### 3.5 变体对比

| 变体 | 条件 | 核心思想 | 典型题 |
|------|------|---------|--------|
| 基本容斥 | $m$ 小 | $2^m$ 枚举 | 硬币购物 |
| 二项式反演 | "恰好" | f↔g 互转 | 染色 |
| Min-Max | max 期望 | 转化为 min | 覆盖时间 |
| 子集卷积 | 子集拆分 | 分层 FWT | DP 优化 |

## 四、15 道经典例题详解

### 🟢 入门级 Level 1：基础容斥

### 题目 1：错排问题（经典）

> 求 $n$ 个元素的错排数 $D_n$（没有元素在原位）。

**分析**：设 $A_i = \{\sigma(i) = i\}$，错排数 = $n! - |A_1 \cup \cdots \cup A_n|$。$|\bigcap_{i \in S} A_i| = (n - |S|)!$。

> **核心观察**：$|S|$ 个不动点的方案数 $= (n-|S|)!$，与 $S$ 的具体内容无关
> **构造策略**：$D_n = n! \cdot \sum_{k=0}^{n} (-1)^k / k!$
> **正确性**：容斥公式 + 二项系数化简

**C++ 代码**：

```cpp
const int MAXN = 1000006;
long long fac[MAXN], inv_fac[MAXN], MOD;

long long derangement(int n) {
    long long ans = 0;
    for (int k = 0; k <= n; k++) {
        long long sign = (k % 2 == 0) ? 1 : MOD - 1;
        ans = (ans + sign * inv_fac[k]) % MOD;
    }
    return ans * fac[n] % MOD;
}
```

**核心启示**：
- **容斥的本质**：将"都不满足"转化为"至少满足 k 个"的交替和
- **错排公式**逼近 $n!/e$，是容斥最经典的入门例题
- $(-1)^k / k!$ 的指数生成函数是 $e^{-x}$

### 题目 2：硬币购物（洛谷 P1450）

> 4 种面值硬币，每种最多 $d_i$ 枚，多组询问凑出金额 $s$ 的方案数。

**分析**：先用完全背包求无限制方案 $f[s]$，再容斥减去超限的：枚举哪些硬币超限（使用 $d_i+1$ 枚以上）。

> **核心观察**：超限 = 先强制用 $d_i+1$ 枚，剩余无限制
> **构造策略**：$\sum_{S \subseteq \{1..4\}} (-1)^{|S|} \cdot f[s - \sum_{i \in S} (d_i+1) \cdot c_i]$
> **正确性**：容斥消除多种超限的重叠

**C++ 代码**：

```cpp
long long f[MAXS]; // 完全背包预处理
int c[4], d[4];

long long query(int s) {
    long long ans = 0;
    for (int mask = 0; mask < 16; mask++) {
        int sign = (__builtin_popcount(mask) % 2 == 0) ? 1 : -1;
        int rem = s;
        for (int i = 0; i < 4; i++)
            if (mask >> i & 1) rem -= (d[i] + 1) * c[i];
        if (rem >= 0) ans += sign * f[rem];
    }
    return ans;
}
```

**核心启示**：
- **有上限背包**的标准容斥解法：无限制方案 - 超限方案
- 条件数只有 4，$2^4 = 16$ 次枚举很快
- 完全背包预处理 $O(S)$，每次询问 $O(2^4)$

### 题目 3：Emiya 家今天的饭（洛谷 P5664）

> $n$ 种烹饪方式，$m$ 种食材，每种方式对每种食材有方案数 $a_{ij}$。选若干菜（每种方式至多用一次），要求每种食材使用次数严格小于总菜数的一半。

**分析**：至多一种食材可能超半。容斥：总方案 - 至少一种食材超半。枚举超半的食材 $j$，做"选/不选 + 是否用食材 $j$"的 DP。

> **核心观察**：至多一种食材超半 $\to$ 枚举哪种超半
> **构造策略**：对每种食材 $j$，DP 统计"$j$ 的使用次数 $>$ 总数$/2$"的方案数
> **正确性**：最多一种食材违规，无需二次容斥

**C++ 代码**：

```cpp
long long row_sum[MAXN]; // row_sum[i] = ∑_j a[i][j]

long long solve(int n, int m, long long a[][MAXM], long long mod) {
    // 总方案 = ∏(1 + row_sum[i]) - 1（至少选一道）
    long long total = 1;
    for (int i = 0; i < n; i++) total = total * (1 + row_sum[i]) % mod;
    total = (total - 1 + mod) % mod;
    // 对每种食材 j，DP: dp[i][diff] = 前 i 种方式中 j 的数量 - 其余数量 = diff
    long long bad = 0;
    for (int j = 0; j < m; j++) {
        // dp[diff + n] = 方案数，diff > 0 表示 j 超半
        // 转移：不选 / 选 j / 选非 j
        // ... O(n²) DP
    }
    return (total - bad % mod + mod) % mod;
}
```

**核心启示**：
- **"至多一个违规"**时不需要二次容斥，只需枚举哪个违规
- DP 的状态设为"差值"（食材 $j$ 的数量 - 其余数量）可以降维
- NOIP 2019 提高组的经典容斥 + DP 题

### 题目 4：卡农（洛谷 P3214）

> 从 $\{1, 2, \ldots, 2^n - 1\}$ 中选 $m$ 个不同的非空子集，组成的多重集的异或和为 0。求方案数。

**分析**：最后一个子集由前 $m-1$ 个决定。但需排除：(1) 最后一个子集为空集；(2) 最后一个子集与之前重复。递推 + 容斥。

> **核心观察**：前 $m-1$ 个自由选，最后一个被确定
> **构造策略**：$f(m) = A(2^n-1, m-1) - f(m-1) \cdot (2^n-1-(m-2)) - f(m-2) \cdot (m-2) \cdot \ldots$
> **正确性**：依次排除空集和重复的情况

**C++ 代码**：

```cpp
long long f[MAXM];
long long pw; // 2^n - 1

void solve(int m, long long mod) {
    // A(pw, k) = pw * (pw-1) * ... * (pw-k+1)
    f[1] = 0; // 1 个子集异或和为 0 → 空集，不合法
    f[2] = 0; // 2 个相同子集 → 不允许重复
    for (int i = 3; i <= m; i++) {
        long long all = A(pw, i - 1, mod); // 前 i-1 个自由选的排列数
        long long bad1 = f[i - 1] * (pw - (i - 2)) % mod; // 最后一个 = 已有某个
        long long bad2 = f[i - 2] * (i - 2) % mod * pw % mod; // 最后一个为空
        f[i] = (all - bad1 - bad2 % mod + 2 * mod) % mod;
    }
    // 答案 = f[m] / m!（无序）
}
```

**核心启示**：
- **递推式容斥**：逐步排除非法情况而非直接枚举子集
- 异或和为 0 的约束决定了最后一个元素
- 排列转组合需除以 $m!$

### 题目 5：成绩比较（洛谷 P3270）

> $n$ 个学生 $m$ 门课，已知第 1 名碾压了 $k$ 个人（每门课都比对方高），且每门课的排名已知。求满足条件的成绩分配方案数。

**分析**：容斥枚举哪些人被碾压。对每门课，用组合数分配名次。

> **核心观察**：碾压 = 所有课都高，容斥处理"恰好碾压 $k$ 人"
> **构造策略**：二项式反演：$f(k)$ = "至少碾压 $k$ 人"，反演得"恰好 $k$ 人"
> **正确性**：二项式反演保证正确性

**C++ 代码**：

```cpp
// g(S) = 1号碾压 S 中所有人的方案数（对每门课独立计算）
long long g(int cnt, int ranks[], int n, int m, long long mod) {
    long long res = C(n - 1, cnt, mod); // 选哪 cnt 人被碾压
    for (int j = 0; j < m; j++) {
        // 第 j 门课：1号排名 r[j]，cnt 人都排在 1号之后
        // 前 r[j]-1 名中选 cnt 人之外的，后面的人随意
        res = res * C(ranks[j] - 1, 0, mod) % mod; // 具体组合数计算
    }
    return res;
}
// 答案 = ∑_{i=k}^{n-1} (-1)^{i-k} · C(i,k) · g(i)
```

**核心启示**：
- **二项式反演**将"恰好"转为"至少"
- 每门课的贡献独立计算，用**乘法原理**合并

### 题目 7：黑暗前的幻想乡（洛谷 P4336）

> 无向图，$n-1$ 家公司各负责一组边。选 $n-1$ 条边构成生成树，每家公司至少贡献一条边。

**分析**：容斥：总生成树数 - 至少一家缺席。枚举缺席公司集合 $S$，用只含非 $S$ 边的图求生成树数（Matrix-Tree 定理）。

> **核心观察**：容斥枚举哪些公司缺席
> **构造策略**：$\sum_{S} (-1)^{|S|} \cdot$ (去掉 $S$ 中公司的所有边后的生成树数)
> **正确性**：Matrix-Tree 定理求生成树数 + 容斥

**C++ 代码**：

```cpp
long long kirchhoff[MAXN][MAXN]; // Kirchhoff 矩阵

long long solve(int n, int m, vector<pair<int,int>> company_edges[], long long mod) {
    long long ans = 0;
    for (int S = 0; S < (1 << (n - 1)); S++) {
        // 建图：只包含不在 S 中的公司的边
        memset(kirchhoff, 0, sizeof(kirchhoff));
        for (int c = 0; c < n - 1; c++) {
            if (S >> c & 1) continue; // 公司 c 缺席
            for (auto [u, v] : company_edges[c]) {
                kirchhoff[u][u]++; kirchhoff[v][v]++;
                kirchhoff[u][v]--; kirchhoff[v][u]--;
            }
        }
        long long trees = det(kirchhoff, n - 1, mod); // (n-1)阶余子式行列式
        int sign = (__builtin_popcount(S) % 2 == 0) ? 1 : -1;
        ans = (ans + sign * trees % mod + mod) % mod;
    }
    return ans;
}
```

**核心启示**：
- **容斥 + Matrix-Tree 定理**是处理"每种边至少用一条"的标准方法
- $n$ 很小（≤ 17）时 $2^{n-1}$ 枚举可行
- Kirchhoff 矩阵的行列式 = 生成树数

### 题目 8：染色（洛谷 P4491）

> $n$ 个位置，$m$ 种颜色，要求恰好有 $c$ 种颜色出现了 $s$ 次，其余颜色任意。

**分析**：二项式反演。设 $f(k)$ = "至少 $k$ 种颜色恰好出现 $s$ 次"的方案数，用反演得"恰好 $c$ 种"。

> **核心观察**："恰好 $c$ 种" $\to$ 二项式反演从"至少 $c$ 种"推导
> **构造策略**：$f(k) = C(m,k) \cdot C(n, s \cdot k) \cdot (s \cdot k)! / (s!)^k \cdot (m-k)^{n-s \cdot k}$
> **正确性**：选 $k$ 种颜色各出现 $s$ 次，其余位置用剩余颜色随意填

**C++ 代码**：

```cpp
long long f[MAXN]; // f[k] = 至少 k 种颜色恰好出现 s 次

void solve(int n, int m, int s, int c, long long mod) {
    for (int k = c; k <= min(m, n / s); k++) {
        f[k] = C(m, k, mod) * C(n, k * s, mod) % mod;
        f[k] = f[k] * fac[k * s] % mod * qpow(inv_fac[s], k, mod) % mod;
        f[k] = f[k] * qpow(m - k, n - k * s, mod) % mod;
    }
    // 二项式反演：g(c) = ∑_{k=c}^{...} (-1)^{k-c} · C(k,c) · f(k)
    long long ans = 0;
    for (int k = c; k <= min(m, n / s); k++) {
        long long sign = (k - c) % 2 == 0 ? 1 : mod - 1;
        ans = (ans + sign * C(k, c, mod) % mod * f[k]) % mod;
    }
    printf("%lld\n", ans);
}
```

**核心启示**：
- **二项式反演**是处理"恰好 $k$ 个满足"问题的黄金模板
- $f(k)$ 的计算需要精确的组合数推导
- NTT 可将反演加速到 $O(n \log n)$

### 题目 9：唱、跳、rap 和篮球（洛谷 P5339）

> $n$ 人排队，4 种类型人数分别为 $a, b, c, d$，不能出现连续 4 人恰好是"唱跳rap篮球"。

**分析**：容斥枚举有多少个"非法四人组"。$k$ 个非法组绑定后，相当于 $k$ 个整体 + 剩余个体的排列。

> **核心观察**：枚举非法组数 $k$，绑定后剩余 $n-3k$ 个位置
> **构造策略**：$f(k) = C(n-3k, k) \cdot$ 多项式系数$(a-k, b-k, c-k, d-k,$ 剩余$)$
> **正确性**：容斥 + 绑定缩点

**C++ 代码**：

```cpp
long long solve(int n, int a, int b, int c, int d, long long mod) {
    long long ans = 0;
    int max_k = min({a, b, c, d, (n - 3) / 4 + 1});
    for (int k = 0; k <= max_k; k++) {
        if (a < k || b < k || c < k || d < k) break;
        long long sign = (k % 2 == 0) ? 1 : mod - 1;
        int tot = n - 3 * k; // 绑定后总元素数
        // C(tot, k) 选放非法组的位置 × 多项式系数
        long long ways = C(tot, k, mod);
        ways = ways * fac[tot - k] % mod; // 剩余排列
        ways = ways * inv_fac[a - k] % mod * inv_fac[b - k] % mod;
        ways = ways * inv_fac[c - k] % mod * inv_fac[d - k] % mod;
        ans = (ans + sign * ways) % mod;
    }
    return ans;
}
```

**核心启示**：
- **连续模式禁止**的标准容斥：绑定非法模式 → 缩点 → 重新排列
- 多种类型的人数限制通过多重排列系数处理

### 题目 10：随机立方体（洛谷 P5400）

> $n \times m \times l$ 立方体，每格填 $[1, V]$ 的随机数。求恰好有 $k$ 个位置是"三维极大值"（每个维度都是该行/列/柱的最大值）的概率。

**分析**：先求"至少 $k$ 个极大值"的概率 $f(k)$，二项式反演得"恰好 $k$ 个"。

> **核心观察**：极大值互不冲突 $\to$ 可选 $k$ 个位置
> **构造策略**：$f(k) = C(\text{合法位置}, k) \cdot P(\text{这} k \text{个都是极大值})$
> **正确性**：二项式反演

**C++ 代码**：

```cpp
long long solve(int n, int m, int l, int k, int V, long long mod) {
    // f(i) = C(n·m·l 中合法极大值位置, i) × 概率
    // 需要推导"i 个位置同时为极大值"的概率
    // 涉及排列和幂次的精确计算
    long long ans = 0;
    for (int i = k; i <= min({n, m, l}); i++) {
        long long sign = (i - k) % 2 == 0 ? 1 : mod - 1;
        long long fi = compute_f(i, n, m, l, V, mod);
        ans = (ans + sign * C(i, k, mod) % mod * fi) % mod;
    }
    return ans;
}
```

**核心启示**：
- **高维问题**的容斥：行/列/柱三个维度的约束
- $f(i)$ 的计算需要精确推导独立性条件
- CTS2019 级别的困难题

### 题目 11：Min-Max 容斥·覆盖时间（经典）

> 无向连通图上从 $s$ 出发随机游走，求访问所有节点的期望步数。

**分析**：覆盖时间 = $E[\max_v T_v]$（$T_v$ = 首次到达 $v$ 的时间）。Min-Max 容斥：$E[\max] = \sum_{T} (-1)^{|T|+1} E[\min_T]$。$E[\min_T]$ 通过吸收态随机游走（高斯消元）求解。

> **核心观察**：覆盖 = 所有点都被访问 = $\max$ 首达时间
> **构造策略**：Min-Max 容斥 $\to$ 对每个子集 $T$ 求 $E[\min_T]$（吸收态消元）
> **正确性**：Min-Max 容斥定理

**C++ 代码**：

```cpp
double solve(int adj[][MAXN], int n, int s) {
    double ans = 0;
    for (int S = 1; S < (1 << n); S++) {
        if (!(S >> s & 1)) continue; // s 必须在集合中
        int sign = (__builtin_popcount(S) % 2 == 1) ? 1 : -1;
        // 求从 s 出发首次到达 S 中任一非 s 节点的期望
        // 建吸收态方程组，高斯消元
        double e_min = absorbing_walk(adj, n, s, S);
        ans += sign * e_min;
    }
    return ans;
}
```

**核心启示**：
- **Min-Max 容斥**是处理"覆盖"问题的核心工具
- $E[\min_T]$ 通过将 $T$ 中节点设为吸收态来求解
- $n$ 很小时（≤ 18）可以 $2^n$ 枚举

### 题目 12：子集卷积（CF 综合）

> 给定 $f, g$ 定义在 $2^n$ 上，求 $h(S) = \sum_{T \subseteq S} f(T) \cdot g(S \setminus T)$。

**分析**：子集卷积需要按 $|T|$ 分层，避免 $T$ 和 $S \setminus T$ 重叠。

> **核心观察**：普通子集和会产生 $T \cap (S \setminus T) \neq \emptyset$ 的错误项
> **构造策略**：$f'[\text{popcount}][S], g'[\text{popcount}][S]$，分层 FWT 后逐层卷积
> **正确性**：popcount 分层保证 $T$ 和 $S \setminus T$ 不交

**C++ 代码**：

```cpp
long long f_hat[MAXN + 1][1 << MAXN], g_hat[MAXN + 1][1 << MAXN];
long long h_hat[MAXN + 1][1 << MAXN];

void subset_conv(long long f[], long long g[], long long h[], int n, long long mod) {
    // 按 popcount 分层
    for (int S = 0; S < (1 << n); S++) {
        int pc = __builtin_popcount(S);
        f_hat[pc][S] = f[S];
        g_hat[pc][S] = g[S];
    }
    // 每层做子集和（FWT OR）
    for (int i = 0; i <= n; i++) {
        fwt_or(f_hat[i], n); fwt_or(g_hat[i], n);
    }
    // 逐层卷积
    for (int i = 0; i <= n; i++)
        for (int S = 0; S < (1 << n); S++) {
            h_hat[i][S] = 0;
            for (int j = 0; j <= i; j++)
                h_hat[i][S] = (h_hat[i][S] + f_hat[j][S] * g_hat[i-j][S]) % mod;
        }
    // 逆 FWT 还原
    for (int i = 0; i <= n; i++) ifwt_or(h_hat[i], n);
    for (int S = 0; S < (1 << n); S++)
        h[S] = h_hat[__builtin_popcount(S)][S];
}
```

**核心启示**：
- **子集卷积** = popcount 分层 + FWT + 多项式卷积
- 复杂度 $O(n^2 \cdot 2^n)$
- 是状压 DP 中**子集拆分**的标准优化

### 题目 13：SOS DP · 高维前缀和（CF 165E）

> 给定数组 $a[0..2^n-1]$，对每个 $S$，求 $\sum_{T \subseteq S} a[T]$。

**分析**：高维前缀和（Sum over Subsets）：枚举每一维做前缀和。

> **核心观察**：逐维累加等价于子集求和
> **构造策略**：for 每一维 $i$: if $S$ 的第 $i$ 位为 1, $f[S]$ += $f[S \oplus (1 \ll i)]$
> **正确性**：类比高维前缀和的每一维独立

**C++ 代码**：

```cpp
long long f[1 << MAXN];

void sos_dp(int n) {
    for (int i = 0; i < n; i++)
        for (int S = 0; S < (1 << n); S++)
            if (S >> i & 1)
                f[S] += f[S ^ (1 << i)];
}
// 逆变换（莫比乌斯反演）：改 += 为 -=
void sos_inv(int n) {
    for (int i = 0; i < n; i++)
        for (int S = 0; S < (1 << n); S++)
            if (S >> i & 1)
                f[S] -= f[S ^ (1 << i)];
}
```

**核心启示**：
- **SOS DP** 是 $O(n \cdot 2^n)$ 的子集求和/莫比乌斯反演
- 它是子集卷积、FWT 的基石
- 逆变换只需将 `+=` 改为 `-=`

### 题目 14：二项式反演 + NTT 加速（CF 综合）

> 给定 $f(0), f(1), \ldots, f(n)$，求 $g(0), g(1), \ldots, g(n)$，其中 $g(k) = \sum_{i=k}^{n} (-1)^{i-k} \binom{i}{k} f(i)$。

**分析**：这是二项式反演的计算。展开后化为卷积形式，用 NTT 加速到 $O(n \log n)$。

> **核心观察**：$g(k) = \sum (-1)^{i-k} C(i,k) f(i)$ 可以化为卷积
> **构造策略**：令 $a[i] = (-1)^i f(i)/i!$, $b[j] = 1/j!$, 则 $g(k)/k!$ = $(a*b)[k]$ 的某种变形
> **正确性**：卷积展开与反演公式一致

**C++ 代码**：

```cpp
void binomial_inversion(long long f[], long long g[], int n, long long mod) {
    // 令 F[i] = (-1)^i * f[i] * inv_fac[i]
    // G[j] = inv_fac[j]
    // 卷积 H = F * G
    // g[k] = H[k] * fac[k] * (-1)^k ... 具体推导
    vector<long long> A(n + 1), B(n + 1);
    for (int i = 0; i <= n; i++) {
        A[i] = (i % 2 == 0 ? 1 : mod - 1) * f[n - i] % mod * inv_fac[n - i] % mod;
        B[i] = inv_fac[i];
    }
    ntt_conv(A, B); // NTT 卷积
    for (int k = 0; k <= n; k++)
        g[k] = A[n - k] * fac[k] % mod;
}
```

**核心启示**：
- 二项式反演可以用 **NTT 加速**到 $O(n \log n)$
- 关键是将反演公式变形为**翻转 + 卷积**的形式
- 这是很多高级计数题的最终优化步骤

### 题目 15：广义容斥 · 莫比乌斯反演综合（CF 综合）

> 给定偏序集上的函数 $f$，求其莫比乌斯反演 $g$。

**分析**：容斥原理的本质是偏序集上的莫比乌斯反演。子集格上 $\mu(S, T) = (-1)^{|T|-|S|}$，因子格上 $\mu = $ 莫比乌斯函数。

> **核心观察**：容斥 = 子集格上的 Mobius 反演
> **构造策略**：一般偏序集用 DP 或矩阵求 Mobius 函数
> **正确性**：Mobius 反演定理

**C++ 代码**：

```cpp
// 子集格上的 Möbius 反演 = SOS DP 的逆
// 因子格上的 Möbius 反演 = 数论 Möbius 函数
// 一般偏序集：DP 或矩阵求逆

// 子集格：已由 SOS 的逆变换实现
// 因子格：
long long g[MAXN]; // g[n] = ∑_{d|n} μ(n/d) · f[d]
void divisor_mobius(long long f[], long long g[], int n) {
    for (int i = 1; i <= n; i++) {
        g[i] = 0;
        for (int d = 1; d * d <= i; d++) {
            if (i % d) continue;
            g[i] += mu[i / d] * f[d];
            if (d != i / d) g[i] += mu[d] * f[i / d];
        }
    }
}
```

**核心启示**：
- **Möbius 反演**统一了容斥原理的各种变体
- 子集格 → SOS DP，因子格 → 数论 μ 函数
- 理解反演的代数本质有助于设计新的容斥策略

## 五、常见技巧与思路总结

### 5.1 容斥方向选择

"至少 $k$ 个"比"恰好 $k$ 个"更容易计算。先求"至少"，再用二项式反演得"恰好"。

### 5.2 绑定缩点

"不能连续出现某模式"→ 把非法模式绑为整体（缩点），容斥枚举非法组数。

### 5.3 Matrix-Tree 配合

"每种边至少用一条构成生成树"→ 容斥枚举缺席的边集 + Matrix-Tree 求生成树数。

### 5.4 SOS DP 加速

子集求和 $O(n \cdot 2^n)$ 代替 $O(3^n)$ 的枚举子集方法。

### 5.5 NTT 加速反演

当反演公式可化为卷积时，$O(n \log n)$ NTT 加速。

## 七、终极总结与归纳

### 7.1 容斥原理的本质（一句话）

容斥原理通过**交替加减**消除重复计数——它是偏序集上 Möbius 反演在子集格上的具体化。

### 7.2 核心要点

1. **基本容斥**：$2^m$ 枚举，适用于条件数小的情况
2. **二项式反演**："恰好"与"至少"的桥梁
3. **Min-Max 容斥**：$E[\max] = \sum (-1)^{|T|+1} E[\min_T]$
4. **SOS DP**：$O(n \cdot 2^n)$ 的子集求和
5. **子集卷积**：popcount 分层 + FWT
6. **NTT 加速**反演：将反演化为卷积
