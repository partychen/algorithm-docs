## 一、什么是中国剩余定理

**中国剩余定理**（Chinese Remainder Theorem，简称 CRT），是数论中一个关于同余方程组求解的经典定理。它描述了当模数两两互素时，一组线性同余方程组有唯一解（在模所有模数之积的意义下）。

> **一句话概括：CRT 告诉我们，如果模数两两互素，那么知道一个数分别除以这些模数的余数，就能唯一确定这个数。**

这一定理最早见于中国古代数学著作《孙子算经》中的"物不知数"问题："今有物不知其数，三三数之剩二，五五数之剩三，七七数之剩二，问物几何？" 答曰：二十三。

CRT 在算法竞赛中应用广泛：大数取模分解、组合数对合数取模（Lucas + CRT）、同余方程组求解等都离不开它。当模数不互素时，需要使用**扩展中国剩余定理（EXCRT）**逐步合并方程。

### 与相关概念的对比

| 概念 | 适用条件 | 核心思想 | 复杂度 |
|------|----------|----------|--------|
| CRT（中国剩余定理） | 模数两两互素 | 构造法直接求解 | $O(n \log M)$ |
| EXCRT（扩展CRT） | 模数任意 | 逐步合并两个方程 | $O(n \log M)$ |
| 扩展欧几里得 exgcd | 单个线性同余方程 | 求 $ax + by = \gcd(a,b)$ | $O(\log \min(a,b))$ |
| Lucas 定理 | 模数为质数 | 组合数对质数取模 | $O(p + \log_p n)$ |
| exLucas（扩展Lucas） | 模数为质数幂之积 | Lucas + CRT 组合 | 视分解而定 |

## 二、核心思想与前置知识

### 2.1 同余与线性同余方程

**同余**：若 $a - b$ 是 $m$ 的倍数，则称 $a$ 与 $b$ 模 $m$ 同余，记作 $a \equiv b \pmod{m}$。

**线性同余方程** $ax \equiv b \pmod{m}$ 有解的充要条件是 $\gcd(a, m) \mid b$。求解可转化为 $ax + my = b$，使用扩展欧几里得算法。

### 2.2 扩展欧几里得算法（前置工具）

CRT 和 EXCRT 都需要用到 exgcd 求解线性同余方程或求模逆元。

```cpp
long long exgcd(long long a, long long b, long long &x, long long &y) {
    if (b == 0) { x = 1; y = 0; return a; }
    long long g = exgcd(b, a % b, y, x);
    y -= (a / b) * x;
    return g;
}

long long mod_inv(long long a, long long m) {
    long long x, y;
    exgcd(a, m, x, y);
    return (x % m + m) % m;
}
```

### 2.3 CRT 的数学表述

给定一组同余方程：

> $x \equiv a_1 \pmod{m_1}$
> $x \equiv a_2 \pmod{m_2}$
> $\dots$
> $x \equiv a_n \pmod{m_n}$

其中 $m_1, m_2, \ldots, m_n$ **两两互素**，令 $M = m_1 \times m_2 \times \cdots \times m_n$，$M_i = M / m_i$，$t_i = M_i^{-1} \pmod{m_i}$（模逆元），则：

> $x \equiv \sum(a_i \times M_i \times t_i) \pmod{M}$

**构造原理**：每个 $M_i \times t_i$ 在模 $m_i$ 下为 1，在模 $m_j\ (j \neq i)$ 下为 0，因此 $a_i \times M_i \times t_i$ 恰好只对第 $i$ 个方程贡献余数 $a_i$。

**CRT 核心框架代码**：

```cpp
long long CRT(int n, long long a[], long long m[]) {
    long long M = 1, ans = 0;
    for (int i = 0; i < n; i++) M *= m[i];
    for (int i = 0; i < n; i++) {
        long long Mi = M / m[i];
        long long ti = mod_inv(Mi, m[i]);
        ans = (ans + a[i] % M * (Mi % M) % M * (ti % M)) % M;
    }
    return (ans % M + M) % M;
}
```

### 2.4 EXCRT 的数学表述

当模数**不两两互素**时，CRT 不再直接适用。EXCRT 的思路是**逐步合并**：每次将两个同余方程合并为一个。

设当前已合并为 $x \equiv r \pmod{\text{lcm}}$，新方程为 $x \equiv a_i \pmod{m_i}$，则需求解：

> $\text{lcm} \times k + r = m_i \times k' + a_i$
> 即 $\text{lcm} \times k \equiv a_i - r \pmod{m_i}$

用 exgcd 求解 $k$，若 $\gcd(\text{lcm}, m_i) \nmid (a_i - r)$ 则无解。否则更新 $r$ 和 $\text{lcm}$。

**EXCRT 核心框架代码**：

```cpp
// 需要 __int128 防溢出，或者自行实现快速乘
typedef long long ll;
typedef __int128 lll;

ll exgcd(ll a, ll b, ll &x, ll &y) {
    if (b == 0) { x = 1; y = 0; return a; }
    ll g = exgcd(b, a % b, y, x);
    y -= (a / b) * x;
    return g;
}

ll EXCRT(int n, ll a[], ll m[]) {
    ll M = m[0], R = a[0];
    for (int i = 1; i < n; i++) {
        ll x, y;
        ll g = exgcd(M, m[i], x, y);
        if ((a[i] - R) % g != 0) return -1; // 无解
        ll t = m[i] / g;
        x = (lll)(a[i] - R) / g % t * x % t;
        R = R + (lll)x * M;
        M = M / g * m[i]; // lcm(M, m[i])
        R = (R % M + M) % M;
    }
    return R;
}
```

## 三、分类与变体详解

### 3.1 经典 CRT（模数两两互素）

最基本的形式，要求所有模数两两互素。通过构造 $M_i \times t_i$ 的"正交基"直接求解。

> **核心观察**：$M_i \times t_i$ 模 $m_i$ 为 1，模其余 $m_j$ 为 0，构成一组"正交基"
> **构造策略**：$x = \sum(a_i \times M_i \times t_i) \mod M$，其中 $t_i$ 为 $M_i$ 的模 $m_i$ 逆元
> **正确性**：每个 $a_i \times M_i \times t_i$ 只影响第 $i$ 个余数，叠加后满足所有方程

```cpp
ll CRT(int n, ll a[], ll m[]) {
    ll M = 1, ans = 0;
    for (int i = 0; i < n; i++) M *= m[i];
    for (int i = 0; i < n; i++) {
        ll Mi = M / m[i];
        ll ti = mod_inv(Mi, m[i]);
        ans = (ans + (__int128)a[i] * Mi % M * ti) % M;
    }
    return (ans % M + M) % M;
}
```

### 3.2 扩展 CRT（EXCRT，模数不互素）

不要求模数互素，采用**逐步合并**策略。每步将两个方程合为一个，核心是用 exgcd 判断可行性并求解。

> **核心观察**：两方程 $x \equiv r_1 \pmod{m_1}$、$x \equiv r_2 \pmod{m_2}$ 可转化为线性同余方程
> **构造策略**：用 exgcd 求解 $m_1 \times k \equiv r_2 - r_1 \pmod{m_2}$，得到新的 $r$ 和 $\text{lcm}(m_1, m_2)$
> **正确性**：合并后的方程等价于原来两个方程的交集，逐步合并 $n$ 个方程即可

```cpp
ll EXCRT(int n, ll a[], ll m[]) {
    ll M = m[0], R = a[0];
    for (int i = 1; i < n; i++) {
        ll x, y, g = exgcd(M, m[i], x, y);
        if ((a[i] - R) % g) return -1;
        ll t = m[i] / g;
        x = (lll)(a[i] - R) / g % t * x % t;
        R += (lll)x * M;
        M = M / g * m[i];
        R = (R % M + M) % M;
    }
    return R;
}
```

### 3.3 CRT 与 Lucas 定理结合（exLucas）

当需要计算 $C(n, m) \bmod P$，而 $P$ 不是质数但可以分解为质数幂之积 $P = p_1^{e_1} \times p_2^{e_2} \times \cdots$ 时：

1. 分别计算 $C(n, m) \bmod p_i^{e_i}$（使用扩展 Lucas）
2. 用 CRT 合并结果

> **核心观察**：$P$ 分解后各质数幂两两互素，满足 CRT 条件
> **构造策略**：分别在小模数下求组合数，再用 CRT 合并
> **正确性**：CRT 保证合并后的解唯一且正确

### 3.4 CRT 在大数问题中的应用

当需要对一个很大的合数模数 $P$ 计算某些值时，若 $P$ 可分解为互素因子之积，就可以在每个小模数下分别计算，最后用 CRT 合并。这在以下场景中尤其有用：

- **大组合数取模**（合数模数）
- **高精度恢复**：用多个小质数分别取模，再 CRT 恢复原始值
- **多项式运算**：NTT 使用不同 NTT 友好质数后 CRT 合并

> **核心观察**：在小模数下运算更高效，CRT 保证可以无损恢复原始值
> **构造策略**：选取若干互素模数，分别计算后 CRT 合并
> **正确性**：只要模数之积大于目标值的范围，CRT 结果即为真实值

### 变体对比总表

| 变体 | 模数要求 | 核心工具 | 典型应用 |
|------|----------|----------|----------|
| CRT | 两两互素 | 模逆元 | 标准同余方程组 |
| EXCRT | 任意 | exgcd 逐步合并 | 模数不互素的同余方程组 |
| exLucas（CRT + Lucas） | $P = \prod p_i^{e_i}$ | 扩展Lucas + CRT | 组合数对合数取模 |
| CRT 恢复大数 | 自选互素模数 | 多模数计算 + CRT | 高精度/多项式合并 |

### 题目 1：韩信点兵（经典问题）

> 有一堆物品，三个三个数余 2，五个五个数余 3，七个七个数余 2。问这堆物品至少有多少个？

**分析**：经典的"物不知数"问题，即孙子定理的原始形态。模数 3, 5, 7 两两互素，直接套用 CRT。

> **核心观察**：模数 3, 5, 7 两两互素，满足 CRT 条件
> **构造策略**：$M = 105$, $M_1=35$, $M_2=21$, $M_3=15$, 分别求逆元后加权求和
> **正确性**：CRT 保证解在 $[0, 105)$ 内唯一

**C++ 代码**：

```cpp
ll a[] = {2, 3, 2};
ll m[] = {3, 5, 7};
ll M = 105;

ll solve() {
    ll ans = 0;
    for (int i = 0; i < 3; i++) {
        ll Mi = M / m[i];
        ll ti = mod_inv(Mi, m[i]);
        ans = (ans + a[i] * Mi % M * ti) % M;
    }
    return (ans % M + M) % M;
}
```

答案位置：`solve()` 返回值，本题为 23

**核心启示**：
- **CRT 的本质是构造正交基**：每个 $M_i \times t_i$ 只在一个模下为 1，其余为 0
- **模数互素是前提**：使用 CRT 前必须验证模数两两互素
- **经典问题学模板**：韩信点兵是 CRT 最好的入门练习

### 题目 2：同余方程组（基础练习）

> 给定 n 个同余方程 $x \equiv a_i \pmod{m_i}$，其中 $m_i$ 两两互素，求最小非负整数解 $x$。

**分析**：直接套用 CRT 模板。注意中间乘法可能溢出，需要使用 `__int128` 或快速乘。

> **核心观察**：模数两两互素，CRT 直接适用
> **构造策略**：计算 $M = \prod m_i$，对每个 $i$ 求 $M_i$ 的逆元，加权求和取模
> **正确性**：CRT 保证 $[0, M)$ 内唯一解

**C++ 代码**：

```cpp
ll a[N], m[N]; // 余数和模数
int n;

ll solve() {
    ll M = 1, ans = 0;
    for (int i = 0; i < n; i++) M *= m[i];
    for (int i = 0; i < n; i++) {
        ll Mi = M / m[i];
        ll ti = mod_inv(Mi, m[i]);
        ans = (ans + (__int128)a[i] * Mi % M * ti) % M;
    }
    return (ans % M + M) % M;
}
```

答案位置：`solve()` 返回值

**核心启示**：
- **防溢出是 CRT 实现的关键**：$M_i \times t_i \times a_i$ 很容易超过 `long long` 范围
- **`__int128` 是竞赛中处理溢出的利器**：比手写快速乘更简洁
- **取模顺序有讲究**：尽量在每步乘法后就取模

### 题目 3：余数求和（基础变形）

> 给定模数 $m_1 = 2, m_2 = 3, m_3 = 5, m_4 = 7, m_5 = 11$ 以及对应余数，求 $[0, 2310)$ 内的唯一解。

**分析**：五个质数两两互素，$M = 2310$。CRT 直接求解，注意乘法用 `__int128` 防溢出。

> **核心观察**：五个质数天然互素，直接 CRT
> **构造策略**：逐个构造 $M_i \times t_i$ 并加权求和
> **正确性**：解在 $[0, 2310)$ 内唯一

```cpp
ll a[5], m[] = {2, 3, 5, 7, 11};
ll M = 2310;

ll solve() {

        ll ti = mod_inv(Mi, m[i]);
        ans = (ans + (__int128)a[i] * Mi % M * ti) % M;
    }
    return (ans % M + M) % M;
}
```

答案位置：`solve()` 返回值

**核心启示**：
- **质数天然互素**：选取质数作为模数是 CRT 的常见套路
- **M 的大小由模数之积决定**：模数越多/越大，M 增长极快
- **CRT 可看作一种"编码-解码"**：余数是编码，CRT 是解码

### 🟡 进阶级 Level 2：EXCRT 模板与标准应用

### 题目 4：扩展中国剩余定理模板（洛谷 P4777）

> 给定 n 组 $(a_i, m_i)$，求满足 $x \equiv a_i \pmod{m_i}$ 的最小非负整数 $x$。模数不一定两两互素。

**分析**：EXCRT 模板题。逐步合并方程，每步用 exgcd 检查是否有解并求出新的余数和模数。注意使用 `__int128` 防止乘法溢出。

> **核心观察**：模数不互素时 CRT 失效，需要逐步合并
> **构造策略**：维护当前合并结果 $(R, M)$，每次与新方程合并，用 exgcd 求解
> **正确性**：合并后 $M = \text{lcm}(\text{旧}M, m_i)$，$R$ 满足所有已合并方程

**C++ 代码**：

```cpp
ll a[N], m[N]; // 余数和模数
int n;
typedef __int128 lll;

ll solve() {
    ll M = m[0], R = a[0];
    for (int i = 1; i < n; i++) {
        ll x, y, g = exgcd(M, m[i], x, y);
        if ((a[i] - R) % g) return -1; // 无解
        ll t = m[i] / g;
        x = (lll)(a[i] - R) / g % t * x % t;
        R += (lll)x * M;
        M = M / g * m[i];
        R = (R % M + M) % M;
    }
    return R;
}
```

答案位置：`solve()` 返回值，-1 表示无解

**核心启示**：
- **EXCRT 的核心是"合并"**：每步将两个方程合并为一个等价方程
- **无解判定**：当 $\gcd(M, m_i)$ 不整除差值时无解
- **`__int128` 必不可少**：合并过程中乘法极易溢出

### 题目 5：猜数字（洛谷 P3868 [TJOI2009]）

> 给定 n 个正整数 $a_1, \ldots, a_n$ 和 n 个正整数 $m_1, \ldots, m_n$，求最小正整数 $x$ 满足对所有 $i$，$x \equiv a_i \pmod{m_i}$。

**分析**：标准 EXCRT 应用。模数不保证互素，直接套用 EXCRT 逐步合并。注意题目求最小**正**整数，若 EXCRT 求出的解为 0 需要加上 lcm。

> **核心观察**：模数不保证互素，需要 EXCRT
> **构造策略**：EXCRT 逐步合并，最后确保结果为正
> **正确性**：EXCRT 合并的正确性由 exgcd 保证

**C++ 代码**：

```cpp
ll a[N], m[N];
int n;
typedef __int128 lll;

ll solve() {
    ll M = m[0], R = ((a[0] % m[0]) + m[0]) % m[0];
    for (int i = 1; i < n; i++) {
        ll ai = ((a[i] % m[i]) + m[i]) % m[i];
        ll x, y, g = exgcd(M, m[i], x, y);
        if ((ai - R) % g) return -1;
        ll t = m[i] / g;
        x = (lll)(ai - R) / g % t * x % t;
        R += (lll)x * M;
        M = M / g * m[i];
        R = (R % M + M) % M;
    }
    return R == 0 ? M : R;
}
```

答案位置：`solve()` 返回值

**核心启示**：
- **余数可能为负数**：输入的 $a_i$ 可能是负数，需要先取模到 $[0, m_i)$ 范围
- **最小正整数 vs 最小非负整数**：如果求出的结果为 0，需要加一个 lcm
- **EXCRT 是 CRT 的严格推广**：模数互素时 EXCRT 同样正确


**核心启示**：
- **实际问题转化为同余方程**：养猪问题看似应用题，本质就是 CRT
- **最小正解与最小非负解**：本题数据保证答案不为 0
- **CRT 的适用性判定**：先检查模数是否两两互素

### 🟠 提高级 Level 3：CRT 与其他定理的结合

### 题目 7：古代猪文（洛谷 P2480 [SDOI2010]）

> 求 $G^{\sum C(n,d)} \bmod 999911659$，其中求和遍历 $n$ 的所有因子 $d$。已知 $999911659$ 是质数。

**分析**：由费马小定理，$G^k \bmod p = G^{k \bmod (p-1)} \bmod p$。关键在于求 $k = \sum C(n,d) \bmod (p-1)$。而 $p-1 = 999911658 = 2 \times 3 \times 4679 \times 35617$，分解为四个互素因子。分别用 Lucas 定理求 $\sum C(n,d) \bmod p_i$，再用 CRT 合并出 $k \bmod (p-1)$。

> **核心观察**：$p-1$ 可分解为互素因子之积，可用 CRT 合并
> **构造策略**：对每个质因子 $p_i$，用 Lucas 求 $\sum C(n,d) \mod p_i$，再 CRT 合并
> **正确性**：CRT 保证在 $[0, p-1)$ 内唯一恢复指数

**C++ 代码**：

```cpp
const ll P = 999911659;
const ll mod = P - 1; // 999911658 = 2 * 3 * 4679 * 35617
ll primes[] = {2, 3, 4679, 35617};

ll Lucas(ll n, ll m, ll p) {
    if (m == 0) return 1;
    if (n % p < m % p) return 0;
    return Lucas(n / p, m / p, p) * C(n % p, m % p, p) % p;
}

ll solve(ll G, ll n) {
    if (G % P == 0) return 0;
    ll a[4]; // 各质因子下的余数
    for (int i = 0; i < 4; i++) {
        a[i] = 0;
        for (ll d = 1; d * d <= n; d++) {
            if (n % d == 0) {
                a[i] = (a[i] + Lucas(n, d, primes[i])) % primes[i];
                if (d != n / d)
                    a[i] = (a[i] + Lucas(n, n / d, primes[i])) % primes[i];
            }
        }
    }
    ll k = CRT(4, a, primes); // CRT 合并
    return qpow(G, k, P);
}
```

答案位置：`solve(G, n)` 返回值

**核心启示**：
- **费马小定理降幂**：$G^k \bmod p$ 只需关心 $k \bmod (p-1)$
- **CRT 与 Lucas 的经典结合**：当 $p-1$ 可分解为小质因子时，分别用 Lucas 再合并
- **枚举因子的技巧**：只需枚举到 $\sqrt{n}$，注意 $d$ 和 $n/d$ 都要统计

### 题目 8：Strange Way to Express Integers（POJ 2891）

> 给定 n 组 $(a_i, m_i)$，求最小非负整数 $x$ 满足 $x \equiv a_i \pmod{m_i}$。若无解输出 -1。

**分析**：EXCRT 经典练习题。与模板题 P4777 完全一致，考察 EXCRT 的标准实现。重点是无解判定和溢出处理。

> **核心观察**：模数不互素，需要 EXCRT 逐步合并
> **构造策略**：标准 EXCRT 流程，每步合并一个方程
> **正确性**：若某步 gcd 不整除差值则无解，否则逐步合并得到正确结果

**C++ 代码**：

```cpp
ll a[N], m[N];
int n;
typedef __int128 lll;

ll solve() {
    ll M = m[0], R = a[0];
    for (int i = 1; i < n; i++) {
        ll x, y, g = exgcd(M, m[i], x, y);
        if ((a[i] - R) % g) return -1;
        ll t = m[i] / g;
        x = (lll)(a[i] - R) / g % t * x % t;
        R += (lll)x * M;
        M = M / g * m[i];
        R = (R % M + M) % M;
    }
    return R;
}
```

答案位置：`solve()` 返回值，-1 表示无解

**核心启示**：
- **EXCRT 的无解情况**：当差值不被 gcd 整除时无解
- **多组数据的工程细节**：注意初始化和边界处理
- **EXCRT 是处理同余方程组的万能工具**：比 CRT 更通用

### 题目 9：组合数对合数取模（exLucas 模板）

> 求 $C(n, m) \bmod P$，其中 $P$ 不一定是质数，但 $P \leq 10^6$。

**分析**：exLucas 问题。将 $P$ 分解为质数幂之积，在每个质数幂下分别计算组合数，最后用 CRT 合并。每个质数幂下的计算涉及阶乘中提取 $p$ 因子、Wilson 定理推广等。

> **核心观察**：$P = \prod p_i^{e_i}$，各质数幂两两互素，可用 CRT 合并
> **构造策略**：对每个 $p_i^{e_i}$ 分别求 $C(n,m) \mod p_i^{e_i}$，再 CRT 合并
> **正确性**：exLucas 在每个质数幂下正确求值，CRT 合并无误

**C++ 代码**：

```cpp
ll fac_mod(ll n, ll p, ll pe) {
    if (n == 0) return 1;
    ll res = 1;
    for (ll i = 2; i <= pe; i++)
        if (i % p) res = res * i % pe;
    res = qpow(res, n / pe, pe);
    for (ll i = 2; i <= n % pe; i++)
        if (i % p) res = res * i % pe;
    return res * fac_mod(n / p, p, pe) % pe;
}

ll C_mod_pe(ll n, ll m, ll p, ll pe) {
    ll fn = fac_mod(n, p, pe), fm = fac_mod(m, p, pe);
    ll fnm = fac_mod(n - m, p, pe);
    ll cnt = 0; // n!中p的幂次 - m!中的 - (n-m)!中的
    for (ll t = p; t <= n; t *= p) cnt += n / t - m / t - (n - m) / t;
    return fn * mod_inv(fm, pe) % pe * mod_inv(fnm, pe) % pe * qpow(p, cnt, pe) % pe;
}

ll exLucas(ll n, ll m, ll P) {
    ll ans = 0, M = P, tmp = P;
    for (ll p = 2; p * p <= tmp; p++) {
        if (tmp % p == 0) {
            ll pe = 1;
            while (tmp % p == 0) { pe *= p; tmp /= p; }
            ll ai = C_mod_pe(n, m, p, pe);
            ll Mi = M / pe;
            ans = (ans + (__int128)ai * Mi % M * mod_inv(Mi, pe)) % M;
        }
    }
    if (tmp > 1) {
        ll ai = C_mod_pe(n, m, tmp, tmp);
        ll Mi = M / tmp;
        ans = (ans + (__int128)ai * Mi % M * mod_inv(Mi, tmp)) % M;
    }
    return ans;
}
```

答案位置：`exLucas(n, m, P)` 返回值

**核心启示**：
- **exLucas 的本质是分解 + CRT**：将困难的合数模问题转化为多个质数幂模问题
- **阶乘分解的技巧**：提取 p 因子后，剩余部分具有周期性（Wilson 定理推广）
- **代码量较大但结构清晰**：分解、单质数幂求值、CRT 合并三步走

### 题目 10：屠龙勇士（洛谷 P4774 [NOI2018]）

> n 头龙，打第 i 头龙选择攻击力为 ATK 的剑，需要恰好打 $a_i$ 点伤害。龙有回血能力，每次回血 $p_i$。求最小攻击次数 $x$ 使得 $\text{ATK} \times x \equiv a_i \pmod{p_i}$ 对所有 $i$ 同时成立。

**分析**：本题是 EXCRT 的变形。每条方程形如 $\text{ATK}_i \times x \equiv a_i \pmod{p_i}$，需要先将其化为标准形式 $x \equiv r_i \pmod{m_i}$，再用 EXCRT 合并。化简时需要用 exgcd 求解线性同余方程，注意处理 gcd 和无解情况。选剑用 multiset 模拟。

> **核心观察**：方程含系数 ATK，需先化为标准同余方程再合并
> **构造策略**：exgcd 化简每个方程得到 $(r_i, m_i)$，EXCRT 逐步合并
> **正确性**：先化简保证每个方程正确，EXCRT 合并保证全局正确

**C++ 代码**：

```cpp
ll a[N], p[N], atk[N]; // 龙的血量、回血量、选择的剑攻击力
int n;
typedef __int128 lll;

ll solve() {
    ll M = 1, R = 0; // 合并结果
    bool first = true;
    for (int i = 0; i < n; i++) {
        // 化简 atk[i] * x ≡ a[i] (mod p[i])
        ll A = atk[i], B = a[i], mod = p[i];
        ll x0, y0, g = exgcd(A, mod, x0, y0);
        if (B % g) { /* 无解处理 */ return -1; }
        ll new_mod = mod / g;
        ll new_r = (lll)(B / g) % new_mod * x0 % new_mod;
        new_r = (new_r % new_mod + new_mod) % new_mod;
        if (first) { M = new_mod; R = new_r; first = false; }
        else {
            // EXCRT 合并 x ≡ R (mod M) 和 x ≡ new_r (mod new_mod)
            ll xx, yy, gg = exgcd(M, new_mod, xx, yy);
            if ((new_r - R) % gg) return -1;
            ll t = new_mod / gg;
            xx = (lll)(new_r - R) / gg % t * xx % t;
            R += (lll)xx * M;
            M = M / gg * new_mod;
            R = (R % M + M) % M;
        }
    }
    return R == 0 ? M : R;
}
```

答案位置：`solve()` 返回值

**核心启示**：
- **带系数的同余方程需要预处理**：$ax \equiv b \pmod{m}$ 先用 exgcd 化简
- **EXCRT 的变形题核心不变**：化简后仍是标准的同余方程组合并
- **NOI 级别题的难点在建模**：把实际问题转化为数学方程是关键

### 题目 11：快乐肥宅（洛谷 P5345 [XR-1]）

> 小 X 有 n 种快乐水，每种快乐水容量为 $v_i$ 毫升。他想配制恰好 $V$ 毫升的混合饮料。每种快乐水有一个限制 $x \equiv a_i \pmod{m_i}$，求满足所有限制的最小正整数 $x$，或判断无解。

**分析**：本质是 EXCRT 求解同余方程组。数据范围较大，需要注意溢出处理。逐步合并 n 个同余方程，若任一步无解则整体无解。

> **核心观察**：$n$ 个同余方程组，模数不互素，使用 EXCRT
> **构造策略**：EXCRT 逐步合并，每步检查可行性
> **正确性**：EXCRT 的逐步合并保证正确性和无解检测

**C++ 代码**：

```cpp
ll a[N], m[N];
int n;
typedef __int128 lll;

ll solve() {
    ll M = m[0], R = a[0] % m[0];
    for (int i = 1; i < n; i++) {
        ll x, y, g = exgcd(M, m[i], x, y);
        ll diff = ((a[i] - R) % m[i] + m[i]) % m[i];
        if (diff % g) return -1;
        ll t = m[i] / g;
        x = (lll)diff / g % t * x % t;
        R += (lll)x * M;
        M = M / g * m[i];
        R = (R % M + M) % M;
    }
    return R == 0 ? M : R;
}
```

答案位置：`solve()` 返回值

**核心启示**：
- **diff 的处理要小心**：`a[i] - R` 可能为负数，取模需要加 `m[i]` 再取模
- **最小正整数处理**：结果为 0 时应返回 M
- **EXCRT 的鲁棒实现**：每步都要保证 R 在合法范围内

### 题目 12：多项式 CRT 合并（NTT 多模数）

> 使用三个 NTT 友好质数（如 998244353, 985661441, 754974721）分别做 NTT 多项式乘法，结果的系数可能超过单个模数范围。用 CRT 合并三个模下的结果恢复真实系数。

**分析**：三个 NTT 质数两两互素，它们的乘积约为 $7 \times 10^{26}$，足以容纳大部分场景。对每个系数位置分别做 CRT 合并。由于乘积超过 `long long` 范围，需要用特殊的合并技巧（先合并两个，再与第三个合并时做 `__int128` 运算）。

> **核心观察**：三个 NTT 质数互素，CRT 合并恢复真实值
> **构造策略**：先合并 $p_1$ 和 $p_2$ 的结果，再与 $p_3$ 合并，使用 `__int128`
> **正确性**：三模数之积足够大，CRT 恢复的值即为精确值

**C++ 代码**：

```cpp
const ll p1 = 998244353, p2 = 985661441, p3 = 754974721;
const ll inv1 = mod_inv(p1, p2); // p1 在 mod p2 下的逆元
const ll inv12 = mod_inv(p1 * p2 % p3, p3); // p1*p2 在 mod p3 下的逆元

ll CRT3(ll a1, ll a2, ll a3, ll MOD) {
    // 先合并 a1, a2
    ll r = ((__int128)(a2 - a1 + p2) % p2 * inv1 % p2 * p1 + a1);
    // 再与 a3 合并
    ll M12 = (__int128)p1 * p2;
    ll k = (__int128)(a3 - r % p3 + p3) % p3 * inv12 % p3;
    ll ans = ((__int128)k * M12 + r) % MOD;
    return (ans + MOD) % MOD;
}
```

答案位置：`CRT3(a1, a2, a3, MOD)` 对每个系数调用

**核心启示**：
- **任意模数 NTT 的核心就是 CRT**：分别在小模数下做 NTT，再合并
- **分步合并避免超大数**：先合并两个，中间结果用 `__int128` 处理
- **NTT 质数的选取**：需要满足 $p = k \times 2^t + 1$ 的形式

### 题目 13：二次剩余与 CRT（综合应用）

> 给定合数 $n = p \times q$（$p, q$ 为不同奇质数），求 $x^2 \equiv a \pmod{n}$ 的所有解。

**分析**：直接在模 $n$ 下求解困难，但可以分别在模 $p$ 和模 $q$ 下求解 $x^2 \equiv a \pmod{p}$ 和 $x^2 \equiv a \pmod{q}$（使用 Cipolla 算法或 Tonelli-Shanks 算法），然后用 CRT 合并。每个质数下有 0 或 2 个解，所以合数下最多有 4 个解。

> **核心观察**：合数模的二次剩余可分解为质数模下分别求解
> **构造策略**：Cipolla 分别求 $x^2 \equiv a \pmod{p}$ 和 $\pmod{q}$，CRT 合并
> **正确性**：CRT 保证 $p$、$q$ 互素时合并无遗漏

**C++ 代码**：

```cpp
// Cipolla 算法求 x² ≡ a (mod p)，返回两个解 {r, p-r}
pair<ll, ll> cipolla(ll a, ll p); // 省略实现

ll p, q; // n = p * q

vector<ll> solve(ll a) {
    auto [r1, r2] = cipolla(a % p, p); // mod p 下的两个解
    auto [s1, s2] = cipolla(a % q, q); // mod q 下的两个解
    vector<ll> ans;
    ll rp[] = {r1, r2}, sq[] = {s1, s2};
    for (int i = 0; i < 2; i++)
        for (int j = 0; j < 2; j++) {
            ll A[] = {rp[i], sq[j]}, M[] = {p, q};
            ans.push_back(CRT(2, A, M));
        }
    sort(ans.begin(), ans.end());
    return ans;
}
```

答案位置：`solve(a)` 返回的解向量

**核心启示**：
- **CRT 是分解合数模问题的通用框架**：分解 → 分别求解 → 合并
- **解的组合爆炸**：$k$ 个质因子各有 2 个解，合并后最多 $2^k$ 个解
- **RSA 密码学中的应用**：知道 $n = p \times q$ 的分解就能用 CRT 加速解密

### 题目 14：CRT 与数位 DP 结合

> 求 $[1, N]$ 中满足"除以 $m_1$ 余 $a_1$ 且除以 $m_2$ 余 $a_2$ 且 $\ldots$ 且除以 $m_k$ 余 $a_k$"的数的个数。其中模数两两互素，$N \leq 10^{18}$。

**分析**：先用 CRT 将 $k$ 个同余条件合并为 $x \equiv r \pmod{M}$（其中 $M = \prod m_i$），然后答案就是 $[1, N]$ 中满足 $x \equiv r \pmod{M}$ 的数的个数。这是一个简单的计数问题：答案为 $\lfloor(N - r) / M\rfloor + [r \geq 1]$（当 $r > 0$ 时）。

> **核心观察**：多个同余条件可用 CRT 合并为一个
> **构造策略**：CRT 合并 → 转化为单个同余条件下的计数问题
> **正确性**：CRT 保证合并等价，计数公式保证正确

**C++ 代码**：

```cpp
ll a[N], m[N];
int k;
ll N_val; // 上界

ll solve() {
    ll M = 1, r = 0;
    // CRT 合并
    for (int i = 0; i < k; i++) M *= m[i];
    for (int i = 0; i < k; i++) {
        ll Mi = M / m[i];
        ll ti = mod_inv(Mi, m[i]);
        r = (r + (__int128)a[i] * Mi % M * ti) % M;
    }
    r = (r % M + M) % M;
    if (r == 0) return N_val / M;
    return r <= N_val ? (N_val - r) / M + 1 : 0;
}
```

答案位置：`solve()` 返回值

**核心启示**：
- **CRT 降维**：将多个条件合并为一个条件，大幅简化问题
- **同余计数的技巧**：$[1, N]$ 中满足 $x \equiv r \pmod{M}$ 的数的个数有直接公式
- **CRT + 计数/DP 是常见组合**：先合并条件再统计

### 题目 15：Broken Robot（Codeforces 24D，CRT + 高斯消元）

> $n \times m$ 的网格，机器人从 $(x, y)$ 出发，每步等概率向左/右/下/不动（边界处调整概率），求到达第 $n$ 行的期望步数。$m \leq 1000$，需要对质数取模或使用高精度。若使用多模数方法，可以在多个质数下分别高斯消元，再用 CRT 合并得到有理数答案。

**分析**：每行的期望值形成一个三对角线性方程组，高斯消元复杂度 $O(m)$。若需要精确有理数结果或在合数模下工作，可选取多个质数模数分别做高斯消元，最后用 CRT 合并。这是 CRT 在数值计算/期望 DP 中的高级应用。

> **核心观察**：三对角方程组在单个模数下可以 $O(m)$ 求解
> **构造策略**：选取 $k$ 个大质数，分别求解，CRT 合并恢复精确值（或对目标模数取余）
> **正确性**：质数之积足够大时 CRT 恢复精确值

**C++ 代码**：

```cpp
// 在模 p 下做三对角高斯消元
ll solve_mod(int x, int y, int n, int m, ll p) {
    // E[i][j] 表示从第 i 行第 j 列出发到达第 n 行的期望步数
    // 从第 n-1 行开始逆推，每行解三对角方程组
    vector<ll> dp(m + 1, 0); // 当前行的期望值
    for (int row = n - 1; row >= x; row--) {
        // 建立三对角方程组并消元（省略具体实现）
        // a[j]*E[j-1] + b[j]*E[j] + c[j]*E[j+1] = d[j]
    }
    return dp[y];
}

// 多模数 CRT 合并
ll primes[] = {998244353, 985661441, 754974721};
ll solve(int x, int y, int n, int m, ll MOD) {
    ll a[3];
    for (int i = 0; i < 3; i++)
        a[i] = solve_mod(x, y, n, m, primes[i]);
    return CRT3(a[0], a[1], a[2], MOD);
}
```

答案位置：`solve(x, y, n, m, MOD)` 返回值

**核心启示**：
- **CRT 是"分模计算 + 合并"的范式**：适用于任何支持模运算的算法
- **多模数高斯消元**：当单模数精度不足时，多模数 + CRT 是通用解决方案
- **CRT 的应用远超同余方程组**：它是一种通用的"分治"思想在模运算中的体现

## 五、常见技巧与思路总结

### 5.1 防溢出——`__int128` 与快速乘

CRT 和 EXCRT 中的乘法极易溢出 `long long`。两种常用解决方案：

```cpp
// 方法一：__int128（推荐，编译器支持时首选）
ll mul = (__int128)a * b % mod;

// 方法二：快速乘（O(log b)，兼容性更好）
ll qmul(ll a, ll b, ll mod) {
    ll res = 0; a %= mod;
    while (b > 0) {
        if (b & 1) res = (res + a) % mod;
        a = (a + a) % mod;
        b >>= 1;
    }
    return res;
}
```

### 5.2 余数归一化

输入的余数可能为负数或超出模数范围，**第一步**就应该归一化到 $[0, m)$ 区间：

```cpp
a[i] = ((a[i] % m[i]) + m[i]) % m[i];
```

### 5.3 带系数同余方程的化简

对于 $a \times x \equiv b \pmod{m}$，先令 $g = \gcd(a, m)$：
- 若 $g \nmid b$，无解
- 否则化简为 $(a/g) \times x \equiv (b/g) \pmod{m/g}$，此时 $a/g$ 与 $m/g$ 互素，可求逆元

### 5.4 判断是否使用 CRT 还是 EXCRT

> 模数两两互素？
> ├── 是 → CRT（构造法，效率更高）
> └── 否 → EXCRT（逐步合并）

### 5.5 CRT 的"分治"思想

CRT 的思想可以推广为：**如果一个问题在多个互素模数下分别求解更容易，就分别求解后合并**。典型应用：
- exLucas：组合数对合数取模
- 多模数 NTT：任意模数多项式乘法
- 高精度恢复：用多个小质数取模后恢复

### 5.6 最小正整数解

EXCRT 求出的是最小非负解。如果题目要求最小**正**整数解，当结果为 0 时需要返回 `M`（所有模数的 lcm）。

## 七、终极总结与归纳

### 7.1 中国剩余定理的本质

**CRT 的本质是"余数编码的唯一性定理"**：当模数两两互素时，一个数被这些模数完全确定。

展开来说：在模 $M = \prod m_i$ 的剩余系中，一个数 $x$ 与它的"余数向量" $(x \bmod m_1, x \bmod m_2, \ldots, x \bmod m_n)$ 一一对应。CRT 提供了从余数向量恢复 $x$ 的构造方法，而 EXCRT 则将这一思想推广到模数不互素的情形——代价是解可能不存在，且 $M$ 变为 lcm。

### 7.2 核心要点

1. **CRT 适用条件**：模数两两互素，构造正交基直接求解
2. **EXCRT 是通用工具**：模数任意时逐步合并，每步用 exgcd
3. **防溢出至关重要**：`__int128` 或快速乘不可省略
4. **无解检测**：EXCRT 中 $\gcd$ 不整除差值时无解
5. **CRT 的思想可推广**：分解 → 分别求解 → 合并，适用于多种场景
6. **前置知识要扎实**：exgcd、模逆元、裴蜀定理是基础
