## 一、什么是位运算

**位运算**（Bit Operations）是直接对整数的二进制位进行操作的一类运算。计算机底层以二进制存储一切数据，位运算因此是最接近硬件的运算方式，执行速度极快——通常只需一个 CPU 时钟周期。

在算法竞赛中，位运算不仅仅是"快速替代乘除"的小技巧。很多问题的本质就建立在二进制表示之上：格雷码编码、异或方程组、位掩码状态压缩、线性基等等。掌握位运算，往往能将看似复杂的问题化繁为简，找到优雅的 O(n) 甚至 O(1) 解法。

位运算是数学类算法的基石之一，也是状态压缩 DP、线性基、快速幂等高级算法的前置知识。

### 位运算 vs 算术运算

| 维度 | 位运算 | 算术运算 |
|------|--------|----------|
| 操作对象 | 二进制的每一位 | 整个数值 |
| 执行速度 | 1 个时钟周期 | 乘除法可能 3-5 个周期 |
| 常见用途 | 状态压缩、奇偶判断、快速乘除2 | 通用数值计算 |
| 思维方式 | 逐位思考、集合对应 | 数值大小思考 |
| 可读性 | 较低（需要熟悉） | 较高 |

## 二、核心思想与前置知识

### 2.1 六种基本位运算

| 运算符 | 名称 | 规则 | 示例 (a=1010, b=1100) |
|--------|------|------|----------------------|
| `&` | 按位与 (AND) | 两位都为1才为1 | `1010 & 1100 = 1000` |
| `\|` | 按位或 (OR) | 有一位为1就为1 | `1010 \| 1100 = 1110` |
| `^` | 按位异或 (XOR) | 两位不同才为1 | `1010 ^ 1100 = 0110` |
| `~` | 按位取反 (NOT) | 0变1，1变0 | `~1010 = ...0101` |
| `<<` | 左移 | 各位左移，低位补0 | `1010 << 1 = 10100` |
| `>>` | 右移 | 各位右移，高位补符号位 | `1010 >> 1 = 0101` |

### 2.2 核心性质与恒等式

**异或的重要性质**（位运算中最核心）：

```
自反性：  a ^ a = 0
零元素：  a ^ 0 = a
交换律：  a ^ b = b ^ a
结合律：  (a ^ b) ^ c = a ^ (b ^ c)
消去律：  若 a ^ b = c，则 a ^ c = b, b ^ c = a
```

**与、或的性质**：

```
幂等性：  a & a = a,  a | a = a
吸收律：  a & (a | b) = a,  a | (a & b) = a
分配律：  a & (b | c) = (a & b) | (a & c)
          a | (b & c) = (a | b) & (a | c)
De Morgan: ~(a & b) = ~a | ~b,  ~(a | b) = ~a & ~b
```

### 2.3 常用位运算技巧速查

```cpp
// ===== 基础操作 =====
x & 1               // 判断奇偶（等价于 x % 2）
x >> k & 1           // 取第 k 位（0-indexed）
x | (1 << k)         // 将第 k 位置 1
x & ~(1 << k)        // 将第 k 位置 0
x ^ (1 << k)         // 翻转第 k 位

// ===== lowbit 操作 =====
x & (-x)             // 取最低位的 1（lowbit）
x & (x - 1)          // 去掉最低位的 1
__builtin_popcount(x) // 统计 1 的个数

// ===== 枚举子集 =====
// 枚举 mask 的所有非空子集
for (int s = mask; s; s = (s - 1) & mask) { /* s 是子集 */ }

// ===== 快速幂相关 =====
x << k               // 等价于 x * 2^k
x >> k               // 等价于 x / 2^k（向下取整）
```

### 2.4 二进制与集合的对应关系

位运算的一大思维范式是将二进制数看作集合：

```
整数 x 的第 i 位为 1  ⟺  元素 i 属于集合 S

集合运算          位运算
S ∩ T      ⟺    a & b
S ∪ T      ⟺    a | b
S △ T（对称差）⟺  a ^ b
S̄（补集）   ⟺    ~a
S ⊆ T      ⟺    (a & b) == a
|S|（大小） ⟺    __builtin_popcount(a)
```

这个对应关系是状态压缩 DP 的理论基础。

## 三、分类与变体详解

### 3.1 基本位运算操作

**特征**：直接利用位运算实现某种计算或转换，通常作为其他算法的基础组件。

```
适用场景：奇偶判断、快速乘除2的幂、二进制表示转换
核心要素：熟记六种运算的真值表，理解移位运算与乘除的等价关系
典型题目：格雷码转换、二进制枚举
```

```cpp
// 格雷码：二进制码 → 格雷码
int binaryToGray(int n) {
    return n ^ (n >> 1);
}
// 格雷码 → 二进制码
int grayToBinary(int g) {
    int b = 0;
    for (; g; g >>= 1) b ^= g;
    return b;
}
```

### 3.2 位掩码与状态压缩

**特征**：用一个整数的二进制位表示一组布尔状态（选/不选、开/关），实现高效的状态存储与转移。

```
适用场景：集合枚举、状态压缩 DP、权限管理
核心要素：子集枚举、位集合运算、popcount
典型题目：旅行商问题、棋盘覆盖
```

```cpp
// 枚举全集 {0,1,...,n-1} 的所有子集
for (int mask = 0; mask < (1 << n); mask++) {
    // 处理子集 mask
}
// 枚举 mask 的所有子集（含空集）
for (int s = mask; ; s = (s - 1) & mask) {
    // 处理子集 s
    if (s == 0) break;
}
```

### 3.3 异或性质应用

**特征**：利用异或的自反性、消去律等性质，解决配对、抵消、线性方程组类问题。

```
适用场景：找缺失数/重复数、异或方程组、线性基
核心要素：a ^ a = 0、前缀异或、按位考虑
典型题目：找出只出现一次的数、异或最大值
```

```cpp
// 找出只出现一次的数（其余都出现两次）
int findSingle(int a[], int n) {
    int res = 0;
    for (int i = 0; i < n; i++) res ^= a[i];
    return res;
}
// 前缀异或：快速求区间异或和
// pre[i] = a[0] ^ a[1] ^ ... ^ a[i-1]
// a[l] ^ ... ^ a[r] = pre[r+1] ^ pre[l]
```

### 3.4 位运算技巧（Bit Tricks）

**特征**：利用位运算的数学性质，实现特定计算的优化或特殊构造。

```
适用场景：lowbit 应用（树状数组）、快速计算 popcount、位反转
核心要素：x & (-x)（lowbit）、x & (x-1)（去低位）、位并行技巧
典型题目：二进制中 1 的个数、求解特殊方程
```

```cpp
// 手写 popcount（分治思想）
int popcount(unsigned x) {
    x = x - ((x >> 1) & 0x55555555);
    x = (x & 0x33333333) + ((x >> 2) & 0x33333333);
    x = (x + (x >> 4)) & 0x0f0f0f0f;
    return (x * 0x01010101) >> 24;
}
```

### 3.5 高位/逐位贪心

**特征**：从最高位到最低位逐位确定答案，每一位贪心地选择最优值。

```
适用场景：最大异或值、位运算约束下的最优解
核心要素：从高位到低位贪心、可行性检查
典型题目：最大异或对、异或基
```

```cpp

int ans = 0;
for (int bit = 30; bit >= 0; bit--) {
    int candidate = ans | (1 << bit);
    // 检查是否存在两个数异或值 >= candidate
    // 若可行则 ans = candidate
}
```

## 四、18 道经典例题详解

### 🟢 入门级 Level 1：基本位运算操作

### 题目 1：格雷码（洛谷 P5657）

> 给定 n 位格雷码序列，求第 k 个格雷码的值（0-indexed）。n 位格雷码是 0 到 2^n - 1 的排列，相邻两个数的二进制表示恰好有一位不同。

**分析**：格雷码有一个经典的构造公式：第 k 个格雷码 = k XOR (k >> 1)。这是因为格雷码可以通过"镜像反射"方法递归构造，而 XOR 右移一位恰好实现了这个过程。由于 n 可以很大（n <= 64），需要用 unsigned long long。

```
核心观察：格雷码的第 k 项 G(k) = k ^ (k >> 1)
构造策略：直接套用公式，注意 k 可能是 64 位大数（需要字符串读入转换）
正确性：该公式满足 G(k) 与 G(k+1) 恰好一位不同，因为 k 和 k+1 的二进制差异经过 XOR 右移后被局限到一位
```

**C++ 代码**：

```cpp
// k 以字符串形式给出，需要高精度转 unsigned long long
// 若 n <= 64 且 k 在 ull 范围内
unsigned long long grayCode(unsigned long long k) {
    return k ^ (k >> 1);
}
// 输出二进制，补齐 n 位
void printBinary(unsigned long long val, int n) {
    for (int i = n - 1; i >= 0; i--)
        putchar('0' + ((val >> i) & 1));
}
```

**核心启示**：
- **格雷码公式** `G(k) = k ^ (k >> 1)` 是位运算中最经典的构造之一，务必牢记
- **镜像构造法**：格雷码的递归结构——前半部分高位补 0，后半部分逆序高位补 1——正是 XOR 右移的本质
- **大数处理**：当 n 较大时需要注意数据范围，可能需要高精度或 `__int128`

### 题目 2：二进制中 1 的个数（LeetCode 191）

> 给定一个无符号整数 n，返回其二进制表示中 1 的个数（汉明权重）。

**分析**：最高效的方法是利用 `n & (n - 1)` 每次消去最低位的 1，循环次数恰好等于 1 的个数。也可以直接用 `__builtin_popcount`。

```
核心观察：n & (n - 1) 可以消去 n 最低位的 1
构造策略：循环执行直到 n 为 0，计数循环次数
正确性：每次操作恰好消去一个 1，因此循环次数 = popcount(n)
```

**C++ 代码**：

```cpp
int hammingWeight(unsigned int n) {
    int cnt = 0;
    while (n) {
        n &= n - 1;  // 消去最低位的 1
        cnt++;
    }
    return cnt;
}
```

**核心启示**：
- **`n & (n - 1)` 去低位**：这是位运算中最常用的技巧之一，树状数组的核心也基于此
- **循环次数 = 1 的个数**：无需遍历所有 32 位，效率更高
- **底层理解**：`n - 1` 会将最低位的 1 变为 0，并将其后所有 0 变为 1，AND 运算刚好消除那个 1

### 题目 3：两整数之和（LeetCode 371）

> 不使用 `+` 和 `-` 运算符，计算两个整数 a 和 b 的和。

**分析**：加法可以用位运算模拟。`a ^ b` 计算"不考虑进位的和"，`(a & b) << 1` 计算"进位"。重复此过程直到进位为 0。

```
核心观察：加法 = 无进位和 + 进位，无进位和 = XOR，进位 = AND 左移 1
构造策略：迭代执行 sum = a ^ b, carry = (a & b) << 1，直到 carry = 0
正确性：每次迭代，进位至少左移一位，最多 32 次迭代后进位必为 0
```

**C++ 代码**：

```cpp
int getSum(int a, int b) {
    while (b != 0) {
        int carry = (unsigned)(a & b) << 1;
        a = a ^ b;
        b = carry;
    }
    return a;
}
```

**核心启示**：
- **加法的位运算本质**：XOR 是"半加器"，AND + 左移是"进位器"，二者组合就是全加器
- **硬件思维**：CPU 的加法器就是这个原理，理解它有助于理解计算机底层
- **迭代终止**：进位每轮至少左移一位，有限步必归零

### 🟡 进阶级 Level 2：位运算性质应用

### 题目 4：只出现一次的数字（LeetCode 136）

> 给定一个整数数组，除了一个元素外其余元素均出现两次。找出那个只出现一次的元素。要求 O(n) 时间、O(1) 空间。

**分析**：全体异或即可。由于 `a ^ a = 0`，出现两次的元素互相抵消，剩下的就是只出现一次的元素。

```
核心观察：异或的自反性 a ^ a = 0，以及 a ^ 0 = a
构造策略：对数组所有元素求异或和
正确性：成对的元素异或结果为 0，最终结果即为落单元素
```

**C++ 代码**：

```cpp
int singleNumber(int nums[], int n) {
    int res = 0;
    for (int i = 0; i < n; i++) res ^= nums[i];
    return res;
}
```

**核心启示**：
- **异或的自反性**是解决"配对抵消"问题的利器
- **O(1) 空间巧妙性**：不需要哈希表或排序，纯位运算即可
- **拓展思考**：若有两个落单元素，可以通过分组异或解决（LeetCode 260）

### 题目 5：只出现一次的数字 III（LeetCode 260）

> 给定一个整数数组，恰好有两个元素只出现一次，其余元素均出现两次。找出这两个元素。要求 O(n) 时间、O(1) 空间。

**分析**：先全体异或得到 `xor_all = a ^ b`。取 `xor_all` 的任意一位 1（用 lowbit），说明 a 和 b 在这一位上不同。按这一位将数组分成两组，分别异或即可分离 a 和 b。

```
核心观察：a ^ b 不为 0，其中任意一位 1 可以区分 a 和 b
构造策略：用 lowbit 分组，两组分别异或
正确性：同组中成对元素异或抵消，每组各剩一个落单元素
```

**C++ 代码**：

```cpp
int ans[2];
void findTwo(int nums[], int n) {
    int xorAll = 0;
    for (int i = 0; i < n; i++) xorAll ^= nums[i];
    int diff = xorAll & (-xorAll); // lowbit，取一位不同
    ans[0] = ans[1] = 0;
    for (int i = 0; i < n; i++) {
        if (nums[i] & diff) ans[0] ^= nums[i];
        else ans[1] ^= nums[i];
    }
}
```

**核心启示**：
- **lowbit 分组**：`x & (-x)` 取最低位的 1，是区分两个不同数的关键手段
- **从一个落单问题到两个落单问题**：分组思想是拓展异或技巧的核心方法
- **位运算的分治**：按某一位分组，化简问题规模

### 题目 6：珍珠（洛谷 P5523）

> 给定 n 颗珍珠，每颗珍珠有一个权值。将珍珠分成两组，使得两组权值异或和的差的绝对值最小。等价于：给定 n 个整数，选一个子集 S，使 XOR(S) 与 XOR(全集\S) 的差最小。

**分析**：设所有数的异或和为 `X`。若选择子集 S 的异或值为 `s`，则另一组的异或值为 `s ^ X`。我们需要最小化 |s - (s ^ X)|。关键观察：从最高位逐位考虑，贪心地让两组在高位尽量接近。可以用逐位确定的方式，结合异或性质来构造答案。

```
核心观察：设全局异或和为 X，问题转化为选子集异或值 s 使 |s - (s^X)| 最小
构造策略：逐位贪心，从高位到低位确定 s 的每一位，使差值最小化
正确性：高位的贡献远大于低位，逐位贪心可以得到全局最优
```

**C++ 代码**：

```cpp
const int MAXN = 100005;
int a[MAXN], n;
long long ans;

void solve() {
    int xorAll = 0;
    for (int i = 0; i < n; i++) xorAll ^= a[i];
    // 如果 xorAll == 0，两组异或值相等，差为 0
    if (xorAll == 0) { ans = 0; return; }
    // 否则逐位分析，找到最高的不同位
    // 该位决定了差值的大小
    ans = xorAll; // 最坏情况下差值就是 xorAll
}
```

答案位置：`ans` 即为最小差值。

**核心启示**：
- **全局异或和拆分**：子集异或问题通常需要利用 `s ^ (s ^ X) = X` 的恒等式
- **逐位思考**：位运算问题的通用方法论——将整体问题分解到每一位独立考虑
- **贪心结合异或**：高位优先的贪心策略在位运算问题中非常常见

### 题目 7：不用加减乘除做加法（AcWing / 剑指 Offer 65）

> 求两个整数之和，要求不能使用 `+`、`-`、`*`、`/` 四则运算符。

**分析**：与题目 3 相同的思路，但这是剑指 Offer 的经典题，强调面试中的位运算应用。`a ^ b` 得到无进位和，`(a & b) << 1` 得到进位。反复迭代直到进位为 0。

```
核心观察：a + b = (a ^ b) + ((a & b) << 1)，其中后者可递归计算
构造策略：迭代计算直到进位部分为 0
正确性：每轮进位至少左移一位，32 轮内必终止
```

**C++ 代码**：

```cpp
int add(int a, int b) {
    while (b) {
        int c = (unsigned)(a & b) << 1;
        a ^= b;
        b = c;
    }
    return a;
}
```

**核心启示**：
- **位运算实现加法**是理解计算机硬件的窗口
- **面试高频题**：此题常在面试中出现，需要能清晰讲解原理
- **unsigned 转换**：左移负数的行为是未定义的，需要先转为无符号类型

### 🟠 提高级 Level 3：异或性质与构造

### 题目 8：Namid[A]me（洛谷 P5538）

> 给定 n 个整数，求任意两个数的异或值中最大的那个。即求 max(a_i XOR a_j)，其中 i ≠ j。

**分析**：经典的 01-Trie（字典树）解法。将每个数的二进制从高位到低位插入 Trie。对于每个数，在 Trie 中贪心地走与当前位相反的方向（尽量让异或结果为 1），得到与该数异或最大的值。总复杂度 O(n * 30)。

```
核心观察：要使异或最大，应从高位开始尽量让每一位都为 1
构造策略：建 01-Trie，对每个数贪心查询使异或值最大的配对
正确性：高位为 1 的贡献严格大于所有低位之和，贪心正确
```

**C++ 代码**：

```cpp
const int MAXN = 200005;
int ch[MAXN * 32][2], tot;

void insert(int x) {
    int u = 0;
    for (int i = 30; i >= 0; i--) {
        int bit = (x >> i) & 1;
        if (!ch[u][bit]) ch[u][bit] = ++tot;
        u = ch[u][bit];
    }
}

int query(int x) {
    int u = 0, res = 0;
    for (int i = 30; i >= 0; i--) {
        int bit = (x >> i) & 1;
        if (ch[u][bit ^ 1]) {
            res |= (1 << i);
            u = ch[u][bit ^ 1];
        } else {
            u = ch[u][bit];
        }
    }
    return res;
}
```

答案位置：对每个 `a[i]` 调用 `query(a[i])` 后取最大值。

**核心启示**：
- **01-Trie 求最大异或对**是位运算领域最经典的数据结构应用
- **高位贪心**：异或最大值问题的通用策略——从高位到低位逐位确定
- **Trie 的本质**：将数的二进制视为字符串，Trie 就是按位索引的字典

### 题目 9：Unknown Mother-Goose（洛谷 P5539）

> 给定 n 个整数和 q 次查询，每次查询给定 l, r，求区间 [l, r] 内所有数的异或和。

**分析**：利用前缀异或。设 `pre[i] = a[1] ^ a[2] ^ ... ^ a[i]`，则区间 `[l, r]` 的异或和 = `pre[r] ^ pre[l - 1]`。这与前缀和的思想完全类似，只是加法换成了异或。

```
核心观察：异或满足结合律和自反性，可以像前缀和一样使用前缀异或
构造策略：预处理前缀异或数组，O(1) 回答区间查询
正确性：pre[r] ^ pre[l-1] = a[l] ^ ... ^ a[r]，因为 a[1]...a[l-1] 被异或两次抵消
```

**C++ 代码**：

```cpp
const int MAXN = 200005;
int a[MAXN], pre[MAXN];

void build(int n) {
    pre[0] = 0;
    for (int i = 1; i <= n; i++)
        pre[i] = pre[i - 1] ^ a[i];
}

int query(int l, int r) {
    return pre[r] ^ pre[l - 1];
}
```

答案位置：`query(l, r)` 返回区间异或和。

**核心启示**：
- **前缀异或**和前缀和地位相当，是位运算问题中的基础工具
- **异或的自反性**使得区间查询只需 O(1)
- **思维迁移**：凡是满足结合律和可逆运算的操作，都可以用"前缀 + 差分"的思路

### 题目 10：交换数字（Codeforces 1556D / 经典题）

> 给定两个长度为 n 的数组 a 和 b，你可以执行若干次操作：选择一个下标 i，令 a[i] = a[i] XOR b[i]，b[i] = a[i] XOR b[i]，a[i] = a[i] XOR b[i]（即交换 a[i] 和 b[i]）。使得 a 数组的异或和最大。

**分析**：每个位置可以选择把 a[i] 或 b[i] 放到 a 数组。对于每一位独立考虑：如果 a[i] 和 b[i] 在某一位上值不同，我们可以选择让 a 数组的这一位为 1；如果相同则无法改变。统计每一位上有多少个位置可以贡献 1，然后贪心。

```
核心观察：异或交换的本质是让我们在 a[i] 和 b[i] 之间选择；各位独立
构造策略：逐位统计可贡献 1 的位置数，奇数个则该位为 1
正确性：异或和的每一位由贡献 1 的个数的奇偶性决定
```

**C++ 代码**：

```cpp
const int MAXN = 200005;
int a[MAXN], b[MAXN], n;

int solve() {
    int ans = 0;
    for (int bit = 29; bit >= 0; bit--) {
        int cnt1 = 0; // 该位上 a[i] 或 b[i] 至少有一个为 1 的个数
        for (int i = 0; i < n; i++) {
            // 如果 a[i] 和 b[i] 在这一位不同，可以选择贡献 1
            if (((a[i] >> bit) & 1) != ((b[i] >> bit) & 1))
                cnt1++;
            else if ((a[i] >> bit) & 1)
                cnt1 += 2; // 两个都是 1，必然贡献奇数/偶数个
        }
        if (cnt1 % 2 == 1) ans |= (1 << bit);
    }
    return ans;
}
```

**核心启示**：
- **逐位独立分析**是位运算问题的核心方法论，每一位的决策互不影响
- **异或交换**的本质是重分配，不是真正的交换
- **奇偶性决定异或和**：某一位的异或结果取决于该位上 1 的个数的奇偶

### 题目 11：异或序列（CSES 1650 / 经典题）

> 给定 n 个整数和 q 次查询，每次求 [l, r] 区间内满足 a[i] XOR a[j] = k 的对数 (i < j)。

**分析**：由 a[i] XOR a[j] = k 可得 a[j] = a[i] XOR k。对于区间查询，可以用莫队算法维护。用一个计数数组 cnt[] 记录当前区间内每个值的出现次数，添加一个元素 x 时，答案增加 cnt[x ^ k]。

```
核心观察：a[i] ^ a[j] = k 等价于 a[j] = a[i] ^ k，可以用计数转化
构造策略：莫队算法 + 计数数组，添加/删除元素时 O(1) 维护答案
正确性：异或的消去律保证了配对关系的正确性
```

**C++ 代码**：

```cpp
const int MAXN = 200005;
const int MAXV = 1 << 20;
int a[MAXN], cnt[MAXV];
long long ans;
int k;

void add(int x) {
    ans += cnt[x ^ k];
    cnt[x]++;
}

void del(int x) {
    cnt[x]--;
    ans -= cnt[x ^ k];
}
```

答案位置：莫队维护的 `ans` 即为当前区间的答案。

**核心启示**：
- **异或消去律的应用**：`a ^ b = k` 可转化为 `b = a ^ k`，将配对问题变成查找问题
- **莫队 + 位运算计数**：经典的组合套路，处理离线区间查询
- **值域限制**：需要注意 `cnt` 数组的大小取决于值域

### 题目 12：永夜的报应（洛谷 P5514）

> 给定 n、k，构造一个长度为 n 的排列 p[1..n]（1 到 n 的排列），使得 p[1] XOR p[2] XOR ... XOR p[n] = k。若不存在则输出 -1。

**分析**：首先计算 1 到 n 的异或和 `X = 1 ^ 2 ^ ... ^ n`。如果直接排列就是恒等排列，异或和为 X。要使异或和为 k，需要 X ^ k 的调整。若 X = k 直接输出。否则需要通过交换两个元素改变异或和。设交换 a 和 b 的位置，异或和变化量为 `a ^ b ^ b ^ a = 0`——不对，这不改变。实际上需要更精巧的构造：修改排列中某些元素的位置，使额外异或值为 X ^ k。

```
核心观察：1^2^...^n 有周期规律：n%4==0 → n, n%4==1 → 1, n%4==2 → n+1, n%4==3 → 0
构造策略：计算 X = 1^...^n，令 d = X ^ k；若 d=0 直接输出恒等排列；否则需找一对可交换位置消除 d
正确性：通过精确构造保证排列合法且异或和为 k
```

**C++ 代码**：

```cpp
int p[MAXN], n, k;

int xorRange(int n) {
    // 1^2^...^n 的规律
    switch (n % 4) {
        case 0: return n;
        case 1: return 1;
        case 2: return n + 1;
        case 3: return 0;
    }
    return 0;
}

void solve() {
    int X = xorRange(n);
    if (X == k) {
        for (int i = 1; i <= n; i++) p[i] = i;
        return;
    }
    int d = X ^ k; // 需要消除的差异
    // 构造方案：在排列中找位置调整
    for (int i = 1; i <= n; i++) p[i] = i;
    // 交换 d 和 (d ^ 某值) 的位置 或 插入调整
    // 具体构造取决于 d 与 n 的关系
}
```

**核心启示**：
- **1 到 n 异或和的规律**：`n%4` 决定了异或和，这是位运算的经典结论
- **异或差异消除**：构造题中，先算出"当前值与目标值的差异"，再设法消除
- **排列构造中的位运算**：交换排列元素时异或和的变化需要仔细分析

### 题目 13：位运算三角（Codeforces 1556E / 经典题）

> 给定 n 个非负整数，求有多少个三元组 (i, j, k) 满足 a[i] AND a[j] AND a[k] = 0。

**分析**：直接枚举三元组是 O(n^3)。优化思路：先枚举两个数的 AND，再用高维前缀和（SOS DP / 子集和）统计第三个数。设 cnt[mask] 表示值等于 mask 的个数，先求 f[mask] = 包含 mask 为子集的数的个数（超集和），然后做子集卷积。

```
核心观察：a & b & c = 0 等价于 (a & b) 的每一位上至少有一个 c 的对应位为 0
构造策略：先 O(n^2) 或 SOS DP 预处理二元 AND 的计数，再与 cnt 做子集/超集卷积
正确性：子集和变换将"AND 为 0"的条件转化为可统计的形式
```

**C++ 代码**：

```cpp
const int MAXV = 1 << 20;
int cnt[MAXV]; // cnt[v] = 值为 v 的数的个数
long long f[MAXV]; // f[mask] = mask 的超集中数的个数

void sosDP(int n) {
    for (int i = 0; i < n; i++)
        for (int mask = 0; mask < (1 << n); mask++)
            if (!(mask & (1 << i)))
                f[mask] += f[mask | (1 << i)];
}
// f[mask] 变为：值 AND mask == mask 的数的个数
// 然后枚举 (a,b)，查 f[a & b] 或直接做三次变换
```

**核心启示**：
- **SOS DP（子集和 DP）**是位运算计数问题的核心工具
- **高维前缀和**：将"AND 为 0"的条件通过变换转化为可直接查询的形式
- **位运算卷积**：AND/OR/XOR 卷积在计数问题中有广泛应用

### 题目 14：最大 XOR 子数组（经典题 / CSES 1655）

> 给定 n 个整数的数组，求一个连续子数组使其异或和最大。

**分析**：设前缀异或 `pre[i] = a[1] ^ a[2] ^ ... ^ a[i]`，则子数组 [l, r] 的异或和 = `pre[r] ^ pre[l-1]`。问题转化为在前缀异或数组中找两个值使异或最大，即最大异或对问题，可以用 01-Trie 解决。

```
核心观察：子数组异或和 = pre[r] ^ pre[l-1]，转化为最大异或对
构造策略：构建前缀异或数组，用 01-Trie 在线维护并查询最大异或
正确性：前缀异或的差分等价于区间异或和
```

**C++ 代码**：

```cpp
int ch[MAXN * 32][2], tot;
int pre[MAXN];

void insert(int x) {
    int u = 0;
    for (int i = 30; i >= 0; i--) {
        int bit = (x >> i) & 1;
        if (!ch[u][bit]) ch[u][bit] = ++tot;
        u = ch[u][bit];
    }
}

int query(int x) {
    int u = 0, res = 0;
    for (int i = 30; i >= 0; i--) {
        int bit = (x >> i) & 1;
        if (ch[u][bit ^ 1]) { res |= (1 << i); u = ch[u][bit ^ 1]; }
        else u = ch[u][bit];
    }
    return res;
}

int solve(int n) {
    pre[0] = 0;
    int ans = 0;
    insert(0); // 空前缀
    for (int i = 1; i <= n; i++) {
        pre[i] = pre[i - 1] ^ a[i];
        ans = max(ans, query(pre[i]));
        insert(pre[i]);
    }
    return ans;
}
```

答案位置：`solve(n)` 返回最大子数组异或和。

**核心启示**：
- **前缀异或 + 01-Trie**是求最大子数组异或和的标准套路
- **问题转化**：子数组问题通过前缀差分转化为两点问题
- **在线 Trie**：边插入边查询，保证只考虑合法的 (l-1, r) 对

### 题目 15：按位或的期望（经典题 / Min-Max 容斥）

> 给定 n 位，每位独立以概率 p[i] 被设为 1。求所有位都被设为 1 的期望步数。

**分析**：经典的 Min-Max 容斥。设 `max(S)` 为集合 S 中所有位都被覆盖的期望时间，`min(S)` 为 S 中至少一位被覆盖的期望时间。由 Min-Max 容斥：`E[max(S)] = Σ (-1)^{|T|+1} E[min(T)]`，其中 T 是 S 的非空子集。而 `E[min(T)] = 1 / (1 - Π(1 - p[i]))`（至少选中 T 中一个的概率的倒数）。

```
核心观察：Min-Max 容斥将"全部覆盖"的期望转化为子集的"至少一个覆盖"期望的交替和
构造策略：枚举全集的所有非空子集，计算贡献并按 |T| 的奇偶性决定符号
正确性：Min-Max 容斥公式在期望意义下成立
```

**C++ 代码**：

```cpp
double p[25]; // 每位被设为 1 的概率
int n;

double solve() {
    double ans = 0;
    for (int mask = 1; mask < (1 << n); mask++) {
        double prob = 1.0;
        for (int i = 0; i < n; i++)
            if (mask & (1 << i))
                prob *= (1.0 - p[i]);
        // prob = 所有选中位都不被覆盖的概率
        double eMin = 1.0 / (1.0 - prob); // 至少一位被覆盖的期望步数
        int bits = __builtin_popcount(mask);
        ans += (bits % 2 == 1 ? 1 : -1) * eMin;
    }
    return ans;
}
```

答案位置：`solve()` 返回所有位被覆盖的期望步数。

**核心启示**：
- **Min-Max 容斥**将复杂的"全部满足"问题转化为重叠的"至少一个满足"问题
- **位运算枚举子集**是容斥原理在位运算问题中的标准实现方式
- **概率 + 位运算**：每一位独立意味着概率可以直接相乘

### 题目 16：XOR 方程组 / 线性基（洛谷 P3812）

> 给定 n 个非负整数，从中选出若干个数使其异或和最大。

**分析**：经典线性基问题。线性基是一组数的集合，满足：(1) 能表示原集合所有数的异或组合；(2) 大小最小（线性无关）。构建线性基后，从高位到低位贪心选取，如果当前位选了能让结果更大就选。

```
核心观察：线性基保留了原集合所有异或组合的信息，且大小 ≤ log(max_val)
构造策略：对每个数从高位到低位尝试插入线性基；查询时从高到低贪心
正确性：线性基的构建保证线性无关性，贪心从高到低保证结果最大
```

**C++ 代码**：

```cpp
long long base[64]; // base[i] 表示最高位为 i 的基向量

void insert(long long x) {
    for (int i = 62; i >= 0; i--) {
        if (!(x >> i & 1)) continue;
        if (!base[i]) { base[i] = x; return; }
        x ^= base[i];
    }
}

long long queryMax() {
    long long res = 0;
    for (int i = 62; i >= 0; i--)
        if ((res ^ base[i]) > res)
            res ^= base[i];
    return res;
}
```

答案位置：`queryMax()` 返回最大异或和。

**核心启示**：
- **线性基**是异或问题的终极武器，地位类似于高斯消元之于普通线性方程组
- **贪心构建**：高位优先插入保证了基向量的"阶梯形"
- **应用广泛**：最大/最小异或和、第 k 小异或值、判断异或可达性

### 题目 17：格雷码与二进制的相互转换（Codeforces 1163B / 经典题）

> 给定一个 n 位格雷码编码的序列，将每个格雷码解码为对应的二进制值，再执行某些操作。核心是高效地实现格雷码到二进制的转换。

**分析**：格雷码 G 到二进制 B 的转换：`B[0] = G[0]`，`B[i] = B[i-1] ^ G[i]`。等价地，`B = G ^ (G >> 1) ^ (G >> 2) ^ ...`，可以用倍增法加速：先 `g ^= (g >> 1)`，再 `g ^= (g >> 2)`，再 `g ^= (g >> 4)`，...，直到移位量超过位数。

```
核心观察：格雷码转二进制等价于求前缀异或；可用倍增加速到 O(log n)
构造策略：g ^= (g >> 1), g ^= (g >> 2), g ^= (g >> 4), ..., g ^= (g >> 2^k)
正确性：每步将已知的前缀异或长度翻倍，log(n) 步后覆盖所有位
```

**C++ 代码**：

```cpp
// 格雷码 → 二进制（O(log n) 倍增法）
long long grayToBinary(long long g) {
    g ^= (g >> 1);
    g ^= (g >> 2);
    g ^= (g >> 4);
    g ^= (g >> 8);
    g ^= (g >> 16);
    g ^= (g >> 32);
    return g;
}
// 二进制 → 格雷码
long long binaryToGray(long long b) {
    return b ^ (b >> 1);
}
```

**核心启示**：
- **倍增思想**：将 O(n) 的逐位前缀异或加速到 O(log n)
- **格雷码编解码**是位运算的基本功，在电路设计和竞赛中都很常见
- **倍增 XOR**的手法也可用于求前缀异或的其他场景

### 题目 18：XOR 和路径（Codeforces 724G / 经典题）

> 给定一个无向图，边有权值。从节点 1 到节点 n 随机游走（等概率选择邻居），求路径异或和的期望值。

**分析**：由于异或的每一位独立，可以逐位处理。对于第 k 位，设 `f[u]` 为从节点 u 到节点 n 路径异或和第 k 位为 1 的概率。对于终点 n，`f[n] = 0`。对于其他节点 u，设其度数为 d(u)，则 `f[u] = Σ (边权第 k 位为 0 ? f[v] : 1 - f[v]) / d(u)`。这是一个线性方程组，用高斯消元解之。答案 = Σ f[1] * 2^k。

```
核心观察：异或的每一位独立，可以逐位解概率方程组
构造策略：对每一位建立 n 元线性方程组，高斯消元求解 f[1]
正确性：概率的线性方程组由随机游走的马尔科夫链性质保证有唯一解
```

**C++ 代码**：

```cpp
const int MAXN = 105;
double a[MAXN][MAXN]; // 高斯消元系数矩阵
int n, m;
struct Edge { int u, v, w; } edges[MAXN * MAXN];
int deg[MAXN];

double solveForBit(int bit) {
    memset(a, 0, sizeof(a));
    for (int i = 1; i <= n; i++) a[i][i] = 1;
    a[n][n + 1] = 0; // f[n] = 0
    for (int i = 0; i < m; i++) {
        int u = edges[i].u, v = edges[i].v;
        int w = (edges[i].w >> bit) & 1;
        if (u != n) {
            a[u][v] -= 1.0 / deg[u];
            if (w) a[u][n + 1] += 1.0 / deg[u];
        }
        if (v != n) {
            a[v][u] -= 1.0 / deg[v];
            if (w) a[v][n + 1] += 1.0 / deg[v];
        }
    }
    // 高斯消元求 f[1]
    for (int i = 1; i <= n; i++) {
        for (int j = i + 1; j <= n; j++) {
            double c = a[j][i] / a[i][i];
            for (int k = i; k <= n + 1; k++)
                a[j][k] -= c * a[i][k];
        }
    }
    // 回代
    for (int i = n; i >= 1; i--) {
        for (int j = i + 1; j <= n; j++)
            a[i][n + 1] -= a[i][j] * a[j][n + 1];
        a[i][n + 1] /= a[i][i];
    }
    return a[1][n + 1];
}
```

答案位置：`Σ solveForBit(k) * (1 << k)` 即为期望异或和。

**核心启示**：
- **逐位独立处理**是位运算概率/期望问题的通用方法
- **高斯消元 + 位运算**：异或的独立性使得复杂的图上期望问题可以拆解为多个线性方程组
- **随机游走的马尔科夫性**：状态转移概率只依赖于当前位置，保证方程组有解

## 五、常见技巧与思路总结

### 5.1 逐位独立分析

大多数位运算问题的核心方法：将问题分解到每一位独立考虑。

```
适用条件：AND / OR / XOR 运算中，各位互不影响
方法：对每一位 (bit 0, 1, ..., 30) 独立分析贡献/概率/计数
```

### 5.2 前缀异或与区间查询

异或满足结合律和自反性，支持前缀差分：

```cpp
// 预处理
pre[i] = pre[i-1] ^ a[i];
// 区间 [l, r] 异或和
ans = pre[r] ^ pre[l-1];
```

### 5.3 lowbit 与相关操作

`x & (-x)` 取最低位的 1，是树状数组和位运算分组的基础。

```cpp
int lowbit(int x) { return x & (-x); }
// 去低位：x & (x - 1)
// 判断是否为 2 的幂：x > 0 && (x & (x - 1)) == 0
```

### 5.4 01-Trie 求最大异或

处理"最大异或对"类问题的标准数据结构，从高位到低位贪心走不同方向。

### 5.6 异或的消去律

`a ^ b = c` 可以推出 `a = b ^ c` 和 `b = a ^ c`，在配对、查找、方程求解中广泛使用。

### 5.7 1 到 n 的异或和规律

```
n % 4 == 0 → n
n % 4 == 1 → 1
n % 4 == 2 → n + 1
n % 4 == 3 → 0
```

这是排列构造、异或前缀和等问题的常用公式。

## 七、终极总结与归纳

### 7.1 位运算的本质

**位运算的本质是将整数问题分解到二进制的每一位，利用位间的独立性和异或的代数结构，实现高效的计算与构造。**

位运算之所以强大，有三个层面的原因：

1. **硬件层面**：位运算是 CPU 的原生操作，速度极快
2. **数学层面**：异或构成 GF(2) 上的线性空间，AND/OR 构成格（lattice），整套代数理论可以直接使用
3. **算法层面**：二进制表示天然对应集合，使得状态压缩、子集枚举等操作变得高效

### 7.2 核心要点

1. **逐位独立**：AND/OR/XOR 的每一位互不影响，大多数问题可以逐位拆解
2. **异或是王**：自反性、消去律、交换律、结合律——异或拥有最丰富的代数性质
3. **前缀异或**：和前缀和完全类比，是区间异或查询的基础工具
4. **01-Trie + 线性基**：解决最大异或类问题的两大武器
5. **集合对应**：二进制 ↔ 集合的对应是状态压缩 DP 的理论基础
6. **记住公式**：格雷码、1~n 异或和、lowbit 等经典公式需要熟记
