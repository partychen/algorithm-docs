## 一、什么是线性基

**线性基**（Linear Basis）是一种用于处理**异或空间**的数学工具。给定一组非负整数，线性基是其中一个极小子集，使得原集合中任意若干个数异或出来的值，都能被这个子集异或出来。

换一种说法：

> **线性基就是在异或运算意义下，一组数的"基底"。它保留了原集合异或可达的全部信息，同时元素个数最少。**

你可以类比线性代数中的概念：线性基就是异或空间的一组基向量。如果把每个数看成一个 60 位的 01 向量，"异或"就是 GF(2)（二元域）上的向量加法，线性基就是这组向量的极大线性无关组。

### 直观理解

> 原始集合: $\{7, 3, 5, 1\}$  (二进制: $111, 011, 101, 001$)
>
> 线性基构造过程:
> 插入 $7\ (111)$: $basis[2] = 7$      (最高位 bit 2)
> 插入 $3\ (011)$: $3$ 的最高位是 bit 1, $basis[1] = 3$      (最高位 bit 1)
> 插入 $5\ (101)$: $5 \oplus basis[2]=7 \to 5 \oplus 7=2\ (010)$
> $2 \oplus basis[1]=3 \to 2 \oplus 3=1\ (001)$, $basis[0] = 1$      (最高位 bit 0)
> 插入 $1\ (001)$: $1 \oplus basis[0]=1 \to 0$  → 线性相关，不插入
>
> 结果线性基: $\{7, 3, 1\}$ — 只用 3 个数就能异或出原集合的所有组合
> 异或可达值: $0, 1, 3, 7, 5(=7 \oplus 3 \oplus 1), 6(=7 \oplus 1), 4(=7 \oplus 3), 2(=3 \oplus 1)$ — 共 $2^3 = 8$ 个

### 与其他工具的对比

| 工具 | 运算域 | 核心操作 | 典型应用 |
|------|--------|----------|----------|
| 高斯消元 | 实数域/模域 | 加减消元 | 解线性方程组 |
| 线性基 | GF(2)（异或） | 异或消元 | 最大异或和、异或方程组 |
| 并查集 | 等价关系 | 合并/查询连通性 | 连通分量、Kruskal |
| 线性基 + 贪心 | GF(2) + 排序 | 先排序再插入 | 带权异或选择 |

## 二、核心思想与前置知识

### 2.1 异或的基本性质

线性基的一切都建立在异或运算的代数性质上：

| 性质 | 公式 | 作用 |
|------|------|------|
| 交换律 | `a ^ b = b ^ a` | 顺序无关 |
| 结合律 | `(a ^ b) ^ c = a ^ (b ^ c)` | 可以任意组合 |
| 自反性 | `a ^ a = 0` | 消去相同元素 |
| 单位元 | `a ^ 0 = a` | 异或 0 不变 |
| 逆元 | 每个元素的逆是自己 | 异或两次恢复 |

关键推论：在异或空间中，**子集的异或和等价于 GF(2) 上的线性组合**，且每个元素要么选（系数 1）要么不选（系数 0）。

### 2.2 线性基的核心性质

线性基满足以下关键性质：

1. **线性基中的元素互相异或不为零**（线性无关）
2. **原集合的任意子集异或值，都可以由线性基中若干元素异或得到**（张成空间相同）
3. **线性基的大小不超过 `log(max_val)`**，即 60 位整数最多 60 个基向量
4. **线性基没有异或和为 0 的非空子集**（等价于线性无关）

### 2.3 线性基的构造算法

构造思路：逐个插入原集合中的数。对于每个数 `x`，从高位到低位扫描，尝试用已有基向量消去 `x` 的最高位。如果消不到 0，就把 `x` 放入对应位的基向量槽位。

```cpp
const int MAXBIT = 62; // 处理 long long 范围
long long basis[MAXBIT + 1]; // basis[i] 表示最高位为第 i 位的基向量

bool insert(long long x) {
    for (int i = MAXBIT; i >= 0; i--) {
        if (!(x >> i & 1)) continue;
        if (!basis[i]) { basis[i] = x; return true; }
        x ^= basis[i];
    }
    return false; // x 可以被现有基线性表示 → 线性相关
}
```

**时间复杂度**：单次插入 `O(log V)`，V 为值域。构造 n 个数的线性基 `O(n log V)`。

### 2.4 查询最大异或和

在线性基上贪心：从高位到低位，如果当前答案异或 `basis[i]` 能变大，就异或上。

```cpp
long long queryMax() {
    long long ans = 0;
    for (int i = MAXBIT; i >= 0; i--)
        ans = max(ans, ans ^ basis[i]);
    return ans;
}
```

### 2.5 查询最小异或和

线性基中最小的非零元素就是最小非零异或和。如果原集合能异或出 0（即有元素插入时线性相关），则最小异或和为 0。

### 2.6 查询第 k 小异或和

先对线性基做一次"化简"（类似行最简形），使得每个基向量只在自己的最高位有 1，其他基向量在该位都是 0。然后把 k 的二进制拆分，对应选取基向量异或即可。

```cpp
long long rb[MAXBIT + 1]; // 化简后的基向量
int cnt; // 线性基大小

void rebuild() {
    cnt = 0;
    for (int i = MAXBIT; i >= 0; i--) {
        if (!basis[i]) continue;
        for (int j = i - 1; j >= 0; j--)
            if (basis[i] >> j & 1) basis[i] ^= basis[j];
        rb[cnt++] = basis[i];
    }
}

long long queryKth(long long k) {
    // 若能异或出 0，需要 k-- (0 是最小值)
    if (k >= (1LL << cnt)) return -1; // 不存在
    long long ans = 0;
    for (int i = 0; i < cnt; i++)
        if (k >> i & 1) ans ^= rb[i];
    return ans;
}
```

## 三、分类与变体详解

### 3.1 基础线性基构造

最基本的形式：给定一组数，构造线性基，然后查询最大/最小异或和。

> **核心观察**：贪心从高位到低位消元
> **构造策略**：每个数尝试插入，成功则占用最高位槽位
> **正确性**：插入前后张成空间不变，即异或可达集合不变

```cpp
// 基础构造 + 最大异或和查询（见 2.3 和 2.4）
```

### 3.2 线性基求第 k 小异或和

将线性基化简为"行最简形"后，k 的二进制位与基向量一一对应。

> **核心观察**：化简后基向量互不干扰，选与不选独立
> **构造策略**：先构造再化简（消去低位冗余 1），$k$ 的第 $i$ 位对应第 $i$ 个基向量
> **正确性**：化简后每个基向量唯一对应一个二进制位，构成双射

```cpp
// 化简 + 第 k 小查询（见 2.6）
```

### 3.3 线性基与贪心

当每个元素有权值时，需要在**最大化权值的同时维护线性无关性**。典型做法：按权值从大到小排序，依次尝试插入线性基，能插入就选。

> **核心观察**：贪心保证优先选高价值元素
> **构造策略**：排序后逐个插入，线性无关就纳入
> **正确性**：拟阵理论保证贪心最优（线性基构成拟阵）

```cpp
// 带权元素按权值降序排列
sort(a + 1, a + n + 1, [](auto& x, auto& y) {
    return x.w > y.w;
});
long long totalW = 0;
for (int i = 1; i <= n; i++)
    if (insert(a[i].v)) totalW += a[i].w;
```

### 3.4 线性基合并

两个线性基合并：将一个线性基的所有元素逐一插入另一个线性基。

> **核心观察**：合并后张成空间 = 两个空间的并
> **构造策略**：遍历一个基的所有非零向量，插入另一个基
> **正确性**：插入操作保持张成空间只增不减

```cpp
void merge(long long b[]) {
    for (int i = MAXBIT; i >= 0; i--)
        if (b[i]) insert(b[i]);
}
```

**复杂度**：`O(log^2 V)`。常用于树上路径问题（倍增/树剖 + 线性基合并）。

### 3.5 前缀线性基（在线/区间查询）

对于区间 `[l, r]` 上的异或最大值查询，可以维护**前缀线性基**：记录每个位置的基向量及其来源位置，查询时只使用来源位置 ≥ l 的基向量。

> **核心观察**：每个基向量记录其来源的最大下标
> **构造策略**：插入第 $i$ 个元素时，优先保留下标更大的基向量（使基向量尽可能"靠右"）
> **正确性**：查询 $[l,r]$ 时，只用下标 $\geq l$ 的基向量，保证选取的元素都在区间内

```cpp
long long basis[MAXBIT + 1];
int pos[MAXBIT + 1]; // pos[i] = basis[i] 对应的最大下标

void insert(long long x, int id) {
    for (int i = MAXBIT; i >= 0; i--) {
        if (!(x >> i & 1)) continue;
        if (!basis[i]) { basis[i] = x; pos[i] = id; return; }
        if (id > pos[i]) {
            swap(x, basis[i]);
            swap(id, pos[i]);
        }
        x ^= basis[i];
    }
}

long long query(int l) { // 查询 [l, 当前位置] 的最大异或和
    long long ans = 0;
    for (int i = MAXBIT; i >= 0; i--)
        if (basis[i] && pos[i] >= l)
            ans = max(ans, ans ^ basis[i]);
    return ans;
}
```

### 3.6 线性基在图论中的应用

经典问题：图上从 s 到 t 的最大异或路径。

> **核心观察**：任意 $s \to t$ 路径的异或值 = 某条固定路径的值 $\oplus$ 若干环的异或值
> **构造策略**：DFS 求出任意一条 $s \to t$ 路径值 $d$，同时收集所有环的异或值建立线性基
> **正确性**：图上任意环都可以异或到路径上，而所有环的异或值构成一个线性空间

### 变体对比总表

| 变体 | 核心修改 | 适用场景 | 复杂度 |
|------|----------|----------|--------|
| 基础构造 | 无 | 单组数最大/最小异或 | O(n log V) |
| k-th 查询 | 化简行最简形 | 第 k 小异或和 | O(log^2 V) |
| 贪心结合 | 排序后插入 | 带权选择 | O(n log n + n log V) |
| 线性基合并 | 逐一插入 | 树上路径、离线分治 | O(log^2 V) |
| 前缀线性基 | 记录来源下标 | 在线区间查询 | O(n log V + q log V) |
| 图论应用 | DFS 找环 + 构造 | 图上最大异或路径 | O((n+m) log V) |

### 题目 1：【模板】线性基（洛谷 P3812）

> 给定 n 个整数，求从中选取若干个数使得异或和最大。

**分析**：线性基最基础的模板题。构造线性基后，贪心从高位到低位取最大异或和。

> **核心观察**：异或和最大 = 在线性基上从高位贪心
> **构造策略**：逐个插入线性基，然后 queryMax
> **正确性**：贪心每一步都在当前最高位取 1，由线性基性质保证不会冲突

**C++ 代码**：

```cpp
const int MAXBIT = 50;
long long basis[MAXBIT + 1];

void insert(long long x) {
    for (int i = MAXBIT; i >= 0; i--) {
        if (!(x >> i & 1)) continue;
        if (!basis[i]) { basis[i] = x; return; }
        x ^= basis[i];
    }
}

long long queryMax() {
    long long ans = 0;
    for (int i = MAXBIT; i >= 0; i--)
        ans = max(ans, ans ^ basis[i]);
    return ans;
}
// 对 n 个数依次 insert，最后输出 queryMax()
```

**核心启示**：
- **线性基模板**是所有异或问题的起点，必须滚瓜烂熟
- **贪心方向**：高位优先，因为高位的贡献一定大于所有低位之和
- **insert 返回值**可以判断线性相关性，后续题目会用到

### 题目 2：彩灯（洛谷 P3857, TJOI2008）

> 有 n 盏彩灯，每个开关控制若干盏灯（用二进制表示）。问所有开关的开关组合能产生多少种不同的灯光状态？

**分析**：每个开关对应一个二进制数，选取若干开关就是求它们的异或子集大小。构造线性基后，答案为 `2^(线性基大小)`。

> **核心观察**：不同异或组合数 = $2^{\text{线性基大小}}$
> **构造策略**：把每个开关的状态作为数插入线性基
> **正确性**：线性基大小 = 维度，异或空间大小 = $2^{\text{维度}}$

**C++ 代码**：

```cpp
const int MAXBIT = 50;
long long basis[MAXBIT + 1];
int sz = 0; // 线性基大小

bool insert(long long x) {
    for (int i = MAXBIT; i >= 0; i--) {
        if (!(x >> i & 1)) continue;
        if (!basis[i]) { basis[i] = x; sz++; return true; }
        x ^= basis[i];
    }
    return false;
}
// 依次插入所有开关状态，答案 = (1LL << sz) % MOD
```

**核心启示**：
- **线性基大小即维度**，异或可达集合的大小恰好是 `2^dim`
- **方案计数**类题目经常归结到线性基大小
- 注意**取模**：`2^60` 已经很大

### 题目 3：XOR（HDU 3949）

> 给定 n 个数，求从中选取若干数使得异或和为第 k 小（非空子集）。

**分析**：线性基化简为行最简形后，k 与基向量形成一一映射。注意：如果原集合能异或出 0（即 n > 线性基大小），则 k 需要减 1（0 是最小值）。

> **核心观察**：化简后的基向量与 $k$ 的二进制位一一对应
> **构造策略**：构造 → 化简（消去多余 1）→ 按 $k$ 的二进制选基向量
> **正确性**：化简后基向量在各自最高位唯一为 1，组合不会冲突

**C++ 代码**：

```cpp
const int MAXBIT = 62;
long long basis[MAXBIT + 1];
long long rb[MAXBIT + 1]; // 化简后
int cnt = 0;
bool hasZero = false; // 能否异或出 0

void rebuild() {
    for (int i = MAXBIT; i >= 0; i--) {
        if (!basis[i]) continue;
        for (int j = i - 1; j >= 0; j--)
            if (basis[i] >> j & 1) basis[i] ^= basis[j];
        rb[cnt++] = basis[i];
    }
}

long long queryKth(long long k) {
    if (hasZero) k--; // 0 是最小值
    if (k <= 0) return 0;
    if (k >= (1LL << cnt)) return -1;
    long long ans = 0;
    for (int i = 0; i < cnt; i++)
        if (k >> i & 1) ans ^= rb[i];
    return ans;
}
// hasZero = (n > cnt); rebuild(); 然后查询
```

**核心启示**：
- **行最简形**是处理第 k 小的关键，它让选择变成独立的 01 决策
- **是否包含 0**：n 个数插入后线性基大小 < n，说明有线性相关，能异或出 0
- 此技巧可推广到"第 k 大"：k-th 大 = (2^cnt - k)-th 小

### 题目 4：元素（洛谷 P4570, BJWC2011）

> 有 n 个矿石，每个有编号（一个整数）和价值。选若干矿石使得编号的异或和非零，且总价值最大。

**分析**：要求选出的子集异或和非零，即选出的元素线性无关。按价值降序排列，依次尝试插入线性基，能插入就选——这就是拟阵上的贪心。

> **核心观察**：线性无关集 + 最大权 = 拟阵贪心
> **构造策略**：按价值降序排列，逐个插入线性基
> **正确性**：线性基构成拟阵，贪心选择最优

**C++ 代码**：

```cpp
struct Ore { long long id; int val; } a[MAXN];

// 按价值降序排序
sort(a + 1, a + n + 1, [](auto& x, auto& y) {
    return x.val > y.val;
});

long long ans = 0;
for (int i = 1; i <= n; i++)
    if (insert(a[i].id)) ans += a[i].val;
// ans 即为最大总价值
```

**核心启示**：
- **拟阵贪心**：线性基中的"线性无关集"构成一个拟阵，所以排序贪心一定最优
- **带权选择**的标准范式：排序 + 依次尝试插入
- 本质和 Kruskal 算法异曲同工：并查集检查连通性 ↔ 线性基检查线性无关性

### 题目 5：新 Nim 游戏（洛谷 P4301, CQOI2013）

> 在经典 Nim 游戏之前，先手可以先拿走若干堆（不能全拿），然后进行标准 Nim 游戏。先手想赢，问先手至少拿走多少石子。

**分析**：先手想赢，需要让剩下的石子堆的 Nim 值（异或和）不为 0，即**剩下的数不能异或出 0**。等价于：剩下的数要线性无关。要最小化拿走的石子数 = 最大化保留的石子数。按石子数降序贪心插入线性基。

> **核心观察**：Nim 必胜 $\iff$ 异或和 $\neq 0$ $\iff$ 保留的数线性无关
> **构造策略**：按石子数降序依次尝试插入，能插入则保留
> **正确性**：拟阵贪心保证保留的总和最大

**C++ 代码**：

```cpp
int a[MAXN]; // 每堆石子数
sort(a + 1, a + n + 1, greater<int>());

long long total = 0, keep = 0;
for (int i = 1; i <= n; i++) {
    total += a[i];
    if (insert(a[i])) keep += a[i];
}
// 答案 = total - keep
```

**核心启示**：
- **Nim 博弈与线性基**的经典结合：异或和为 0 ⟺ 线性相关
- 转化思路：最小化拿走 → 最大化保留 → 拟阵贪心
- 此思路可推广到所有"从集合中删除最少元素使异或和非零"的问题

### 题目 6：Shortest Path Problem?（CF 845G）

> 给定无向图，边有异或权值，求从 1 到 n 的异或最短路（异或和最小的路径）。

**分析**：与最大异或路径类似，先求出 1 到 n 的任意一条路径的异或值 d，然后收集所有环的异或值建线性基。在线性基上贪心让 d 尽量小。

> **核心观察**：任意两条 $s \to t$ 路径的异或差是若干环的异或
> **构造策略**：DFS 求 $dis[v]$，遇到环将环值插入线性基，最后用线性基最小化 $dis[n]$
> **正确性**：路径空间 = 固定路径 $\oplus$ 环空间

**C++ 代码**：

```cpp
long long dis[MAXN]; // dis[v] = 1 到 v 的任意路径异或值
bool vis[MAXN];

void dfs(int u) {
    vis[u] = true;
    for (auto [v, w] : adj[u]) {
        if (!vis[v]) { dis[v] = dis[u] ^ w; dfs(v); }
        else insert(dis[u] ^ dis[v] ^ w); // 环值
    }
}

// dfs(1); 然后用线性基最小化 dis[n]
long long ans = dis[n];
for (int i = MAXBIT; i >= 0; i--)
    ans = min(ans, ans ^ basis[i]);
```

**核心启示**：
- **异或最短路与最长路**本质相同：DFS + 环空间线性基
- 与题目 10（最大 XOR 路径）互为镜像：最大用 max，最小用 min
- 图上异或距离的核心定理：**路径差 = 环的线性组合**

### 题目 7：最大 XOR 和路径（洛谷 P4151, WC2011）

> 给定 n 个点 m 条边的无向图，边有非负权值，求从 1 到 n 的路径中，边权异或和的最大值。可以重复经过边和点。

**分析**：这是线性基在图论中的经典应用。由于可以重复走边（走两次异或抵消），所有 1→n 路径的异或值 = 任意一条 1→n 路径 ⊕ 若干环。用 DFS 找出所有环值建线性基，再最大化。

> **核心观察**：重复走边可抵消 → 路径值 = 固定路径 $\oplus$ 环空间的线性组合
> **构造策略**：DFS 建出异或距离树，遇到返祖边就收集环值入线性基
> **正确性**：所有 $s \to t$ 路径的异或值集合 = $\{d[t] \oplus \text{环空间中任意元素}\}$

**C++ 代码**：

```cpp
long long dis[MAXN];
bool vis[MAXN];

void dfs(int u) {
    vis[u] = true;
    for (auto [v, w] : adj[u]) {
        if (!vis[v]) { dis[v] = dis[u] ^ w; dfs(v); }
        else insert(dis[u] ^ dis[v] ^ w);
    }
}

// dfs(1); 最大化 dis[n]
long long ans = dis[n];
for (int i = MAXBIT; i >= 0; i--)
    ans = max(ans, ans ^ basis[i]);
```

**核心启示**：
- **异或图论的核心定理**：s→t 的所有路径异或值 = 某固定值 ⊕ 环空间
- 环空间 = 所有基本环的异或值的线性张成
- 最大化/最小化只需改贪心方向（max/min）

### 题目 8：幸运数字（洛谷 P3292, SCOI2016）

> 一棵 n 个节点的树，每个节点有权值。多次查询两点路径上选若干节点权值的最大异或和。

**分析**：树上路径问题。用**倍增合并线性基**：预处理每个节点到 2^k 级祖先的路径线性基，查询时将 LCA 路径上的线性基合并。

> **核心观察**：树上路径 = 两段到 LCA 的路径拼接
> **构造策略**：倍增预处理每段的线性基，查询时合并 $O(\log n)$ 段
> **正确性**：合并操作保证张成空间 = 路径上所有值的张成

**C++ 代码**：

```cpp
struct LB { // 线性基
    long long b[MAXBIT + 1];
    void insert(long long x) {
        for (int i = MAXBIT; i >= 0; i--) {
            if (!(x >> i & 1)) continue;
            if (!b[i]) { b[i] = x; return; }
            x ^= b[i];
        }
    }
    void merge(LB& o) {
        for (int i = MAXBIT; i >= 0; i--)
            if (o.b[i]) insert(o.b[i]);
    }
    long long queryMax() {
        long long ans = 0;
        for (int i = MAXBIT; i >= 0; i--)
            ans = max(ans, ans ^ b[i]);
        return ans;
    }
};

LB up[MAXN][LOG]; // up[u][k] = u 到 2^k 级祖先路径的线性基
int fa[MAXN][LOG], dep[MAXN];

// 查询 u→v 路径最大异或和：倍增求 LCA，沿途合并线性基
```

**核心启示**：
- **线性基合并**是处理树上路径异或问题的标准工具
- 倍增 + 线性基合并：预处理 `O(n log n log^2 V)`，单次查询 `O(log n log^2 V)`
- 也可用树链剖分 + 线段树合并线性基来做

### 题目 9：Xor-matic Number（CF 724G）

> 给定 n 个数，求所有非空子集的异或和之和（所有不同异或结果各算一次，乘以出现次数）。

**分析**：构造线性基后，线性基外的 n - cnt 个数每个都可选可不选（不影响异或结果），按位统计贡献。

> **核心观察**：线性基外元素自由选取，贡献系数为 $2^{n-cnt-1}$
> **构造策略**：构造线性基，按位统计异或值中每个 bit 为 1 的方案数
> **正确性**：线性基恰好覆盖所有异或可达值，每个值被 $2^{n-cnt}$ 种子集异或出

**C++ 代码**：

```cpp
int cnt = 0; // 线性基大小
for (int i = 1; i <= n; i++) cnt += insert(a[i]);
int free_cnt = n - cnt; // 自由元素个数

// 化简线性基，按位统计
rebuild();
long long ans = 0, pw = power(2, free_cnt - 1); // 2^(free_cnt - 1)
// 线性基能表示的每个非零值 v，贡献 v * 2^(free_cnt)
// 按位统计：第 j 位为 1 的异或值有 2^(cnt-1) 个
for (int j = 0; j <= MAXBIT; j++) {
    long long bit_cnt = (1LL << (cnt - 1)) % MOD; // 基中第 j 位为 1 的值个数
    ans = (ans + (1LL << j) % MOD * bit_cnt % MOD * power(2, free_cnt) % MOD) % MOD;
}
```

**核心启示**：
- **按位统计贡献**是处理"所有子集异或和的和"的关键
- 自由元素个数 = n - 线性基大小，它们不改变异或可达集合但贡献系数 2^free
- 线性基化简后，恰好一半的异或值在某一位为 1

### 题目 10：Ivan and Burgers（CF 1100F）

> 给定 n 个数，多次查询区间 [l, r] 中选若干数的最大异或和。

**分析**：这是**前缀线性基**的经典应用。离线按右端点排序，维护当前前缀的线性基，每个基向量记录其来源的最大下标。查询 [l, r] 时只使用下标 ≥ l 的基向量。

> **核心观察**：前缀线性基中优先保留右侧元素，查询时按下标过滤
> **构造策略**：离线排序 + 前缀线性基（swap 保持下标最大）
> **正确性**：swap 操作不改变张成空间，只让基向量的来源尽可能靠右

**C++ 代码**：

```cpp
long long basis[MAXBIT + 1];
int pos[MAXBIT + 1];

void insert(long long x, int id) {
    for (int i = MAXBIT; i >= 0; i--) {
        if (!(x >> i & 1)) continue;
        if (!basis[i]) { basis[i] = x; pos[i] = id; return; }
        if (id > pos[i]) { swap(x, basis[i]); swap(id, pos[i]); }
        x ^= basis[i];
    }
}

long long query(int l) {
    long long ans = 0;
    for (int i = MAXBIT; i >= 0; i--)
        if (basis[i] && pos[i] >= l)
            ans = max(ans, ans ^ basis[i]);
    return ans;
}
// 离线：按 r 排序，处理到 r 时依次插入 a[1..r]，回答所有右端点为 r 的查询
```

**核心启示**：
- **前缀线性基**的核心 trick：插入时 swap 让基向量下标尽可能大
- 离线处理使得只需一个线性基而非区间线性基
- 在线做法也可用持久化线性基（但常数较大）

### 题目 11：Xor Tree（CF 1427E）

> 给定 n 个数（叶子权值），构建一棵二叉树使得：每个内部节点的值是其子树叶子的异或和，且树的总代价（所有内部节点值之和）最小。实际上需要删除最少的数使得剩余数构建的树满足条件。

**分析**：按二进制最高位分治建 Trie。在每层，如果某一侧只有一个数（或为 0），就不用删除；否则递归两侧，每次取较小的一侧全部删除。用线性基判断某一侧是否可以通过异或消为 0。

> **核心观察**：按位分治，每层决定保留哪一侧更多的数
> **构造策略**：构建 01-Trie 分治，结合线性基判断消去可能
> **正确性**：最高位分治保证全局最优

**C++ 代码**：

```cpp
int solve(vector<long long>& nums, int bit) {
    if (nums.size() <= 1 || bit < 0) return 0;
    vector<long long> L, R;
    for (auto x : nums)
        ((x >> bit & 1) ? R : L).push_back(x);
    if (L.empty()) return solve(R, bit - 1);
    if (R.empty()) return solve(L, bit - 1);
    // 需要删除一侧，保留较多的一侧
    return min(
        (int)L.size() - 1 + solve(R, bit - 1),
        (int)R.size() - 1 + solve(L, bit - 1)
    );
}
// 答案 = solve(all_nums, MAXBIT)
```

**核心启示**：
- **01-Trie 分治**与线性基的思想相通：按最高位分组
- 保留较大组、删除较小组是一种局部贪心
- 分治的终止条件：只剩 0 或 1 个元素

**分析**：等价于判断目标值是否在给定向量的线性张成中。对操作向量建线性基，判断目标能否被表示。

> **核心观察**：操作等价于 $GF(2)$ 上的线性组合
> **构造策略**：对所有操作向量建线性基，检查目标向量能否被消为 0
> **正确性**：能消为 $0$ $\iff$ 在张成空间中

```cpp
// 判断 x 是否在当前线性基的张成空间中
bool canRepresent(long long x) {
    for (int i = MAXBIT; i >= 0; i--) {
        if (!(x >> i & 1)) continue;
        if (!basis[i]) return false;
        x ^= basis[i];
    }
    return true; // x 被消为 0，在张成空间中
}
```

**核心启示**：
- **判断是否在张成空间中** = 尝试插入，看能否消为 0
- 这是线性基的基础操作之一，很多高级题都会用到
- 可以推广到判断"异或方程是否有解"

### 题目 14：线性基合并（树上路径，洛谷 P4869 类型）

> 给一棵带点权的树，每次查询路径 u→v 上所有点权能异或出的最大值。强制在线。

**分析**：与 P3292 类似，但要求在线。可以用**树链剖分 + 线段树**，每个线段树节点维护对应区间的线性基。查询时合并路径上的 O(log^2 n) 个线段树节点的线性基。

> **核心观察**：线段树每个节点维护区间线性基，查询时合并
> **构造策略**：树链剖分后建线段树，每个节点存线性基
> **正确性**：合并不丢失信息，张成空间只增不减

**C++ 代码**：

```cpp
struct Node {
    LB lb; // 线性基
} seg[MAXN * 4];

void build(int id, int l, int r) {
    if (l == r) { seg[id].lb.insert(val[dfn[l]]); return; }
    int mid = (l + r) / 2;
    build(id * 2, l, mid);
    build(id * 2 + 1, mid + 1, r);
    seg[id].lb = seg[id * 2].lb;
    seg[id].lb.merge(seg[id * 2 + 1].lb);
}

LB query(int id, int l, int r, int ql, int qr) {
    if (ql <= l && r <= qr) return seg[id].lb;
    int mid = (l + r) / 2;
    LB res;
    if (ql <= mid) { LB t = query(id*2, l, mid, ql, qr); res.merge(t); }
    if (qr > mid)  { LB t = query(id*2+1, mid+1, r, ql, qr); res.merge(t); }
    return res;
}
// 树链剖分 + 线段树查询路径线性基
```

**核心启示**：
- **线段树 + 线性基**是在线处理路径查询的标准方案
- 合并复杂度 `O(log^2 V)` 乘到线段树和树剖的 log 上，总查询 `O(log^2 n log^2 V)`
- 空间较大，需注意常数优化

### 题目 16：圣剑护符（洛谷 P5556）

> 给定 n 个数和 m 次操作：区间赋值、区间查询能否异或出某个目标值。

**分析**：用**线段树维护区间线性基**，支持区间赋值（打 lazy tag 时重建线性基）和区间查询（合并后判断目标是否在张成空间中）。

> **核心观察**：线段树 + 线性基 + lazy 标记
> **构造策略**：区间赋值时整段重建线性基，查询时合并区间线性基并检验
> **正确性**：lazy 下传时正确重建保证线性基始终准确

**C++ 代码**：

```cpp
struct Node {
    LB lb;
    long long lazy; // 区间赋值标记
    bool hasLazy;
} seg[MAXN * 4];

void pushDown(int id, int l, int r) {
    if (!seg[id].hasLazy) return;
    int mid = (l + r) / 2;
    auto apply = [&](int child, int sz) {
        seg[child].lb = LB(); // 清空
        seg[child].lb.insert(seg[id].lazy);
        seg[child].lazy = seg[id].lazy;
        seg[child].hasLazy = true;
    };
    apply(id * 2, mid - l + 1);
    apply(id * 2 + 1, r - mid);
    seg[id].hasLazy = false;
}

bool query(int id, int l, int r, int ql, int qr, long long target) {
    // 合并 [ql, qr] 的线性基，判断 target 是否可表示
    LB res = queryLB(id, l, r, ql, qr);
    return res.canRepresent(target);
}
```

- **线段树 + 线性基 + lazy**是数据结构与线性代数的深度结合
- 区间赋值比区间修改更适合线性基（赋值可以直接重建）
- 此类题目对实现能力和常数优化要求很高

## 五、常见技巧与思路总结

### 5.1 线性基的判空与计数

线性基大小（非零基向量个数）= 异或空间的维度。

- 异或可达值个数 = `2^dim`
- 能否异或出 0 = 是否有元素插入失败（线性相关）
- 能否异或出 target = 尝试用线性基消去 target，看能否消为 0

### 5.2 swap 技巧保持下标最大

前缀线性基中，插入新元素时，如果新元素下标更大，就 swap 到基向量位置：

```cpp
if (id > pos[i]) { swap(x, basis[i]); swap(id, pos[i]); }
```

这保证每个位置的基向量来源尽可能靠右，从而支持区间查询。

### 5.3 线性基大小的上界

- 值域 V 以内的线性基大小 ≤ log₂V + 1
- 对于 long long：最多 63 个基向量
- 对于 int：最多 31 个基向量

这意味着线性基占用空间是 O(log V)，合并复杂度是 O(log² V)。

### 5.4 判断线性相关性

`insert` 返回 false ⟺ 该元素可以被已有基线性表示 ⟺ 线性相关。

这在很多问题中是关键判断：
- Nim 游戏：能异或出 0 ⟺ 存在线性相关
- 拟阵贪心：只选线性无关的元素
- 方案计数：自由元素（线性相关）贡献系数 2^k

### 5.5 化简（行最简形）

化简让每个基向量只在自己的最高位有 1：

```cpp
for (int i = MAXBIT; i >= 0; i--)
    for (int j = i - 1; j >= 0; j--)
        if (basis[i] >> j & 1) basis[i] ^= basis[j];
```

化简后的好处：k-th 查询变为二进制拆分，每个基向量独立。

### 5.6 图上异或问题万能公式

对于图上 s→t 的异或路径问题：

1. DFS 求出任意一棵生成树，记录 `dis[v]`
2. 所有非树边 (u,v,w) 产生环值 `dis[u] ^ dis[v] ^ w`
3. 环值建线性基
4. 答案 = 在线性基上优化 `dis[t]`

## 六、优化方法 / 解题思路框架

### 6.1 解题决策流程

> 看到异或 / XOR 关键词
> ├─ 问"最大/最小异或和" → 线性基 + 贪心 queryMax/queryMin
> ├─ 问"第 $k$ 小异或和" → 线性基 + 化简行最简形 + 二进制拆分
> ├─ 问"异或方案数" → 线性基大小 → $2^{dim}$ 或按位统计
> ├─ 问"能否异或出 target" → 用线性基消 target
> ├─ 带权选择 + 线性无关 → 排序 + 拟阵贪心
> ├─ 区间异或最大值 → 前缀线性基 / 线段树 + 线性基
> ├─ 树上路径异或 → 倍增/树剖 + 线性基合并
> └─ 图上异或路径 → DFS 求环 + 线性基

### 6.2 信号词 → 题型判断表

| 信号词/条件 | 对应题型 | 解法 |
|-------------|----------|------|
| "最大异或和/子集" | 基础线性基 | 构造 + queryMax |
| "多少种不同异或值" | 异或空间大小 | 2^(线性基大小) |
| "第 k 小/大异或值" | k-th 查询 | 化简 + 二进制拆分 |
| "选数使异或非零 + 最大权" | 拟阵贪心 | 排序 + 插入 |
| "Nim 游戏 + 改规则" | 博弈 + 线性基 | 异或和 = 0 ⟺ 线性相关 |
| "区间 [l,r] 异或最大" | 前缀线性基 | swap 保最大下标 |
| "树上路径异或" | 线性基合并 | 倍增/树剖 + merge |
| "图上异或路径" | 环空间 | DFS + 环值建基 |
| "所有子集异或和之和" | 按位贡献 | 统计每位为 1 的比例 |

## 七、终极总结与归纳

### 7.1 线性基的本质

**线性基 = GF(2) 上的极大线性无关组。** 它用 O(log V) 的空间精确刻画了一组数在异或运算下能张成的全部空间，是处理异或问题的万能工具。

一切异或子集问题，本质上都是在问：异或空间长什么样？线性基就是描述这个空间的最紧凑方式。

### 7.2 核心要点

1. **构造**：逐个插入，从高位到低位消元，时间 O(n log V)
2. **最大/最小异或和**：在线性基上从高位贪心，O(log V)
3. **第 k 小**：化简为行最简形，k 的二进制与基向量一一对应
4. **带权选择**：排序 + 拟阵贪心，正确性由拟阵理论保证
5. **图上问题**：所有 s→t 路径 = 固定路径 ⊕ 环空间
6. **合并/区间**：线性基可合并（O(log²V)），前缀线性基支持区间查询
