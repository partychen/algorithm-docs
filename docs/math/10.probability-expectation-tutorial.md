## 一、什么是概率与期望

概率描述事件发生的可能性，期望描述随机变量的平均值。在算法竞赛中，**概率 DP** 和**期望 DP** 是最核心的工具——通过状态转移方程计算复杂随机过程的概率或期望。

| 概念 | 定义 | 竞赛应用 |
|------|------|---------|
| 概率 DP | 状态 = 概率值 | 正向推进 |
| 期望 DP | 状态 = 期望值 | 逆向推导 |
| 期望线性性 | $E[X+Y] = E[X]+E[Y]$ | 拆分计算 |
| 全概率公式 | $P(A) = \sum P(A|B_i)P(B_i)$ | 分类讨论 |

## 二、核心思想与前置知识

### 2.1 概率 DP 框架

正向推进：已知初始状态的概率分布，逐步转移到后续状态。

```cpp
// 概率 DP 骨架（正推）
double prob[MAXN]; // prob[s] = 到达状态 s 的概率
prob[init] = 1.0;
for (state s in topological order) {
    for (each transition s → t with probability p) {
        prob[t] += prob[s] * p;
    }
}
```

### 2.2 期望 DP 框架

逆向推导：从终止状态开始，$E[s] = \sum_t p_{s \to t} \cdot (E[t] + \text{cost}_{s \to t})$。

```cpp
// 期望 DP 骨架（逆推）
double E[MAXN]; // E[s] = 从状态 s 到终止状态的期望步数
E[terminal] = 0;
for (state s in reverse topological order) {
    E[s] = 1; // 当前步的代价
    for (each transition s → t with probability p) {
        E[s] += p * E[t];
    }
}
```

### 2.3 期望的线性性

$E[\sum X_i] = \sum E[X_i]$，即使 $X_i$ 之间不独立。这是拆分复杂期望的核心工具。

### 2.4 高斯消元求期望

当状态转移图有环时（如随机游走），期望方程形成线性方程组，需高斯消元求解。

## 三、分类与变体详解

### 3.1 概率 DP（正推）

```
方向：从初始状态推到终止状态
适用：求到达某状态的概率
转移：prob[t] += prob[s] * p(s→t)
```

### 3.2 期望 DP（逆推）

```
方向：从终止状态推到初始状态
适用：求到达终止状态的期望步数/代价
转移：E[s] = cost + ∑ p(s→t) * E[t]
```

### 3.3 期望 DP + 高斯消元

```
适用：状态转移有环（如随机游走）
方法：列方程 E[s] = 1 + ∑ p·E[t]，高斯消元
```

### 3.4 期望线性性拆分

```
适用：期望可以拆分为独立贡献的和
思路：E[总] = ∑ E[每个元素的贡献]
```

### 3.5 树上期望 DP

```
适用：树形结构上的随机过程
方法：换根 DP 或树上高斯消元
```

### 题目 1：红包发红包（洛谷 P5104）

> $n$ 个人依次从总额 $w$ 中随机取红包，每人取 $[0, \text{剩余}]$ 的均匀分布，求第 $k$ 个人的期望金额。

**分析**：由对称性和期望线性性，第 $k$ 个人的期望为 $\frac{w}{2^k}$... 实际上更精确地，每人取走当前剩余的期望一半。

```
核心观察：每次取走剩余的期望 1/2
构造策略：E[第k人] = w · (1/2) · (1/2)^{k-1} = w/2^k... 需要更精确推导
正确性：均匀分布期望为区间中点
```

**C++ 代码**：

```cpp
// 第 k 个人期望 = w / (2^k) · C(something)
// 实际：剩余期望 R_k = w · (1/2)^k，第 k 人期望 = R_{k-1}/2 = w/2^k
double solve(double w, int n, int k) {
    // E[k] = w * (1/2)^k (简化模型)
    return w * pow(0.5, k);
}
```

**核心启示**：
- **均匀分布**的期望 = 区间中点
- 多轮随机过程可以逐步递推

### 题目 2：换教室（洛谷 P1850）

> $n$ 个时段，每段可以选择申请换教室（概率 $k_i$ 成功）或不换。求使总移动距离期望最小的策略。

**分析**：期望 DP，$dp[i][j][0/1]$ = 前 $i$ 个时段用了 $j$ 次申请机会、第 $i$ 段是否申请的最小期望距离。

```
核心观察：决策 DP + 概率加权
构造策略：dp[i][j][换/不换]，转移时考虑成功/失败两种情况
正确性：期望的线性性保证加权转移
```

**C++ 代码**：

```cpp
double dp[MAXN][MAXM][2]; // dp[i][j][申请?]

void solve(int n, int m, double k[], int c[], int d[], int dist[][MAXN]) {
    // 初始化 dp[1][0][0], dp[1][1][1]
    for (int i = 2; i <= n; i++)
        for (int j = 0; j <= min(i, m); j++) {
            // 不申请第 i 段
            dp[i][j][0] = min(dp[i-1][j][0] + dist[c[i-1]][c[i]],
                              dp[i-1][j][1] + k[i-1]*dist[d[i-1]][c[i]] + (1-k[i-1])*dist[c[i-1]][c[i]]);
            // 申请第 i 段（成功概率 k[i]）
            if (j >= 1) {
                // 四种组合：上一步申请/不申请 × 本步成功/失败
                // ... 概率加权
            }
        }
}
```

**核心启示**：
- **决策型期望 DP**：每步有选择，求最优策略的期望
- 转移时按概率加权

### 题目 3：奖励关（洛谷 P2473）

> $n$ 关，每关有若干宝物可选（有前置依赖），每关随机出现一个宝物子集，求最优策略下的最大期望收益。

**分析**：状态压缩 + 期望 DP 逆推。$E[i][S]$ = 第 $i$ 关已获得宝物集合 $S$ 时的最大期望。

```
核心观察：状压 DP + 逆推期望
构造策略：从最后一关逆推，每关枚举可能出现的宝物，贪心选择
正确性：期望 DP 的逆推框架
```

**C++ 代码**：

```cpp
double E[MAXN][1 << MAXK]; // E[关][已获得集合]

void solve(int n, int k) {
    // 从第 n 关逆推到第 1 关
    for (int i = n; i >= 1; i--)
        for (int S = 0; S < (1 << k); S++) {
            E[i][S] = 0;
            for (int j = 0; j < k; j++) {
                // 宝物 j 出现，检查前置条件
                if ((prereq[j] & S) == prereq[j])
                    E[i][S] += max(E[i+1][S], E[i+1][S | (1<<j)] + val[j]);
                else
                    E[i][S] += E[i+1][S];
            }
            E[i][S] /= k; // 每种宝物等概率出现
        }
}
```

**核心启示**：
- **状压 + 期望逆推**是处理小状态空间的标准方法
- 每步取 max 体现**最优策略**

### 题目 4：高速公路（洛谷 P2221）

> 链上随机选区间 $[l, r]$，区间边权和的期望。

**分析**：对每条边计算被选中的概率 × 边权，利用期望线性性求和。

```
核心观察：E[总] = ∑ E[每条边贡献] = ∑ w_i · P(边 i 被选中)
构造策略：边 (i, i+1) 被选中 ⟺ l ≤ i 且 r ≥ i+1，概率 = i·(n-i) / C(n,2)
正确性：期望线性性
```

**C++ 代码**：

```cpp
double solve(int w[], int n) {
    double total = (double)n * (n - 1) / 2; // C(n,2) 种选法
    double ans = 0;
    for (int i = 1; i < n; i++)
        ans += (double)w[i] * i * (n - i) / total;
    return ans;
}
```

**核心启示**：
- **期望线性性**将复杂期望拆为单元素贡献之和
- 每条边被选中概率 = 左端点数 × 右端点数 / 总方案数

### 题目 5：亚瑟王（洛谷 P3239）

> $n$ 张卡牌，$r$ 轮游戏，每轮从第 1 张开始检查，卡 $i$ 以概率 $p_i$ 触发（且仅触发一次），触发造成 $d_i$ 伤害。求总期望伤害。

**分析**：$E = \sum d_i \cdot P(\text{卡}\ i\ \text{被触发})$。关键是求每张卡在 $r$ 轮中被触发的概率。

```
核心观察：E = ∑ d_i · P_i，核心是求 P_i
构造策略：P_i = 1 - (1 - p_i · q_i)^r'，其中 q_i 是"到达卡 i"的概率
正确性：需精确推导"到达概率"的递推
```

**C++ 代码**：

```cpp
double prob[MAXN]; // prob[i] = 卡 i 在所有轮次中被触发的概率
// f[i] = 一轮游戏中到达卡 i 且卡 i 尚未触发的概率
// 这需要更细致的递推...
double solve(int n, int r, double p[], double d[]) {
    // 状态：dp[i] = 卡 i 被触发的概率
    // 需要考虑多轮的影响
    double ans = 0;
    for (int i = 0; i < n; i++) ans += d[i] * prob[i];
    return ans;
}
```

**核心启示**：
- "多轮 + 顺序检查"的模型需要仔细推导到达概率
- 期望线性性始终适用

### 题目 6：分手是祝愿（洛谷 P3750）

> $n$ 盏灯，初始一些亮着。每步随机选一盏操作（翻转它及所有因子灯）。实际上最优 $k$ 步可以全灭，若还剩 $> k$ 盏需要灭时随机操作，$\le k$ 时最优操作。求期望步数。

**分析**：关键观察：从大到小贪心确定哪些灯必须操作（类似线性基）。设需要操作的灯有 $cnt$ 个，期望 = $\sum_{i=k+1}^{cnt} \frac{n}{i}$（从 $cnt$ 个必须操作的灯降到 $k$ 个时的期望）+ $k$。

```
核心观察：必须操作的灯集合唯一，贪心从大到小确定
构造策略：E = k + ∑_{i=k+1}^{cnt} n/i（几何分布期望）
正确性：每步命中一个目标灯的概率 = i/n
```

**C++ 代码**：

```cpp
long long solve(int light[], int n, int k, long long mod) {
    // 从大到小确定需要操作的灯
    int cnt = 0;
    for (int i = n; i >= 1; i--)
        if (light[i]) {
            cnt++;
            for (int j = 1; j * j <= i; j++)
                if (i % j == 0) { light[j] ^= 1; if (j != i / j) light[i/j] ^= 1; }
        }
    if (cnt <= k) return cnt % mod * fac[n] % mod; // 直接最优操作
    long long ans = k;
    for (int i = k + 1; i <= cnt; i++)
        ans = (ans + qpow(i, mod - 2, mod) * n) % mod;
    return ans * fac[n] % mod;
}
```

**核心启示**：
- **贪心确定操作集**+ 随机过程期望分析
- 几何分布期望 $E = 1/p$

### 题目 7：概率充电器（洛谷 P4284）

> 树上每个节点有独立概率 $p_i$ 自发充电，边有概率 $q_e$ 导通。求每个节点被充电的期望个数。

**分析**：对每个节点，它被充电 = 自己充电 OR 通过某条边从邻居传来。用换根 DP。

```
核心观察：P(节点 i 被充电) = 1 - P(自己不充电) · ∏_{边 e} P(不从 e 传来)
构造策略：先求以 1 为根的子树贡献，再换根
正确性：独立事件的乘法原理
```

**C++ 代码**：

```cpp
double p_sub[MAXN]; // 从子树方向不被充电的概率

void dfs1(int u, int fa) { // 求子树贡献
    p_sub[u] = 1 - p[u]; // 自己不充电
    for (auto [v, q] : adj[u]) {
        if (v == fa) continue;
        dfs1(v, u);
        p_sub[u] *= (1 - q + q * p_sub[v]); // 边不导通 + 边导通但子树不充电
    }
}
// 换根 DP 类似处理父方向
```

**核心启示**：
- **树上概率** = 换根 DP + 独立性
- "不发生的概率"比"发生的概率"更容易递推

### 题目 9：地震后的幻想乡（洛谷 P3343）

> $n$ 个点，$m$ 条边，每条边权是 $[0, 1]$ 均匀随机。求最小生成树最大边权的期望。

**分析**：$E[\max] = \int_0^1 P(\max > t) dt = \int_0^1 (1 - P(\text{MST} \max \le t)) dt$。$P(\text{MST} \max \le t) = P(\text{只用} \le t \text{的边连通}) = t^{\text{边数}} \cdot \text{某个系数}$... 实际需要用 Matrix-Tree 定理 + 容斥。

```
核心观察：E[最大边] = ∫(1 - P(连通所需边都 ≤ t)) dt
构造策略：枚举边子集是否 ≤ t，容斥计算连通概率，积分
正确性：概率积分公式 + 容斥
```

**C++ 代码**：

```cpp
// 枚举边子集 S（2^m），判断 S 的边是否使图连通
// P(所有边 ≤ t) = t^|S|，需要容斥
// E = ∫_0^1 (1 - ∑ coef[k] · t^k) dt = 1 - ∑ coef[k]/(k+1)
double solve(int n, int m) {
    // 状压枚举连通子图... n ≤ 10
    double ans = 0;
    for (int S = 0; S < (1 << m); S++) {
        if (!is_spanning(S, n)) continue;
        int sign = (__builtin_popcount(S) - (n-1)) % 2 == 0 ? 1 : -1;
        ans += sign * 1.0 / (__builtin_popcount(S) + 1);
    }
    return 1.0 - ans;
}
```

**核心启示**：
- **连续概率**问题用积分转化
- $n$ 小时可以状压枚举

### 题目 10：随机数生成器（洛谷 P3600）

> 长度 $n$ 的随机序列，每个位置在 $[1, x]$ 均匀随机，求最小值的期望。

**分析**：$E[\min] = \sum_{v=1}^{x} P(\min \ge v) = \sum_{v=1}^{x} ((x - v + 1) / x)^n$。

```
核心观察：E[min] = ∑ P(min ≥ v)（经典公式）
构造策略：P(min ≥ v) = ((x - v + 1) / x)^n
正确性：所有位置 ≥ v 的概率
```

**C++ 代码**：

```cpp
long long solve(int n, int x, long long mod) {
    long long inv_x = qpow(x, mod - 2, mod);
    long long ans = 0;
    for (int v = 1; v <= x; v++)
        ans = (ans + qpow((long long)(x - v + 1) * inv_x % mod, n, mod)) % mod;
    return ans;
}
```

**核心启示**：
- $E[\min] = \sum P(\min \ge v)$ 是离散 min 期望的标准公式
- 连续情况：$E[\min] = \int_0^\infty P(\min > t) dt$

### 题目 13：随机树（洛谷 P3830）

> 随机二叉搜索树，求树高期望 / 叶子深度期望。

**分析**：递归定义概率。$n$ 个节点的随机 BST，根的左子树大小 $k$ 等概率为 $0, 1, \ldots, n-1$。

```
核心观察：BST 的递归结构 → 递推 DP
构造策略：dp[n] = 期望，按左子树大小分类
正确性：随机 BST 的等概率假设
```

**C++ 代码**：

```cpp
double dp[MAXN]; // dp[n] = n 节点随机 BST 的期望深度/高度

void solve(int n) {
    dp[0] = dp[1] = 0;
    for (int i = 2; i <= n; i++) {
        dp[i] = 0;
        for (int k = 0; k < i; k++)
            dp[i] += (dp[k] + dp[i-1-k] + something) / i;
    }
}
```

**核心启示**：
- 随机 BST 的分析与快排分析类似
- 调和级数 $H_n$ 经常出现

### 题目 15：随机游走期望（经典综合）

> 无向连通图上从 $s$ 出发随机游走（每步等概率走向相邻节点），求访问所有节点的期望步数。

**分析**：Min-Max 容斥 + 期望线性性。$E[\max T_i] = \sum_{S \neq \emptyset} (-1)^{|S|+1} E[\min_{i \in S} T_i]$，其中 $T_i$ = 首次到达 $i$ 的时间。

```
核心观察：Min-Max 容斥将覆盖时间转为首达时间的交替和
构造策略：E[min T_S] 通过吸收态高斯消元求解
正确性：Min-Max 容斥公式
```

**C++ 代码**：

```cpp
double cover_time(int adj[][MAXN], int n, int s) {
    double ans = 0;
    for (int S = 1; S < (1 << n); S++) {
        int sign = (__builtin_popcount(S) % 2 == 1) ? 1 : -1;
        // E[min T_S]：从 s 出发首次到达 S 中任一节点的期望
        double e_min = solve_absorbing(adj, n, s, S);
        ans += sign * e_min;
    }
    return ans;
}
```

**核心启示**：
- **Min-Max 容斥**是处理"覆盖"问题的强大工具
- $n$ 很小时可以 $2^n$ 枚举

## 五、常见技巧与思路总结

### 5.1 期望线性性

$E[X_1 + X_2 + \cdots] = E[X_1] + E[X_2] + \cdots$，不需要独立性。

### 5.2 几何分布

成功概率 $p$ 的几何分布，期望次数 = $1/p$。

### 5.3 逆推 vs 正推

期望通常逆推（从终态开始），概率通常正推（从初态开始）。

### 5.4 有环图用高斯消元

当状态转移图有环时，期望方程形成线性方程组。

### 5.5 Min-Max 容斥

$E[\max] = \sum (-1)^{|S|+1} E[\min_S]$，用于覆盖时间等问题。

## 六、优化方法 / 解题思路框架

```
问题类型判断：
  │
  ├── 无环状态转移 → 拓扑序 DP（概率正推/期望逆推）
  ├── 有环 → 高斯消元
  ├── 树形 → 换根 DP
  ├── "所有元素被覆盖" → Min-Max 容斥
  ├── 独立贡献 → 期望线性性拆分
  └── 决策型 → max/min 选择最优
```

## 七、终极总结与归纳

### 7.1 概率与期望的本质（一句话）

概率与期望 DP 的核心是**状态转移**——通过概率加权的递推方程，将随机过程的分析化归为确定性计算。

### 7.2 核心要点

1. **概率正推，期望逆推**是基本方向
2. **期望线性性**是最强大的工具
3. **高斯消元**处理有环的期望方程
4. **Min-Max 容斥**处理覆盖问题
5. **树上换根 DP**处理树形概率
6. **几何分布**的期望 = $1/p$
