## 一、什么是 Manacher 算法

**Manacher 算法**（又称"马拉车算法"）是 1975 年由 Glenn Manacher 提出的一种线性时间算法，专门用于**求解字符串中所有回文子串**的问题。它最经典的应用是在 O(n) 时间内求出一个字符串的**最长回文子串**。

回文串是竞赛中的高频考点。朴素的"枚举中心 + 向两边扩展"做法需要 O(n²)，而 Manacher 利用**已知回文的对称性**来跳过冗余比较，将时间压缩到线性。它的核心思想可以用一句话概括：

> **利用右端点最远的已知回文，通过对称性快速确定新位置的回文半径初值，再尝试扩展。**

除了求最长回文子串，Manacher 还能高效解决回文计数、回文分割、前/后缀最长回文等一系列问题，是字符串算法的重要基础工具。

### Manacher vs 其他回文算法

| 维度 | 中心扩展法 | Manacher | 回文自动机（PAM） | 字符串哈希 |
|------|-----------|----------|------------------|-----------|
| 时间复杂度 | O(n²) | O(n) | O(n) | O(n) 预处理 + O(1) 查询 |
| 空间复杂度 | O(1) | O(n) | O(n·Σ) | O(n) |
| 最长回文子串 | ✅ | ✅ | ✅ | ✅ |
| 所有回文子串计数 | 需 O(n²) | 配合差分 O(n) | ✅ 天然支持 | 需额外处理 |
| 回文分割/DP | 可作 check | 常与 DP 结合 | 可结合 | 可作 check |
| 实现难度 | 极低 | 中等 | 较高 | 中等 |

## 二、核心思想与前置知识

### 2.1 奇偶统一：插入分隔符

回文串有奇数长度（如 `aba`）和偶数长度（如 `abba`）两种，中心扩展时需要分别处理。Manacher 的第一个技巧是**在每个字符之间及首尾插入特殊字符 `#`**，将所有回文统一为奇数长度：

```
原串:    a b b a
变换后:  # a # b # b # a #
```

变换后：
- 原来的奇回文 `a` → `#a#`（长度 3，仍是奇数中心）
- 原来的偶回文 `abba` → `#a#b#b#a#`（长度 9，中心是 `#`）
- **设变换后某位置的回文半径为 p[i]，则对应原串中的回文长度 = p[i] - 1**

### 2.2 p 数组与 mid/mr 优化

定义 `p[i]` 为以位置 i 为中心的最长回文**半径**（含中心本身），即 `t[i-p[i]+1 .. i+p[i]-1]` 是回文。

核心优化变量：
- `mid`：当前右端点最远的回文的**中心**
- `mr`：该回文的**右端点** `mid + p[mid]`

对于新位置 i，利用对称性：
```
如果 i < mr:
    i 关于 mid 的对称点 j = 2*mid - i
    p[i] 至少 = min(p[j], mr - i)   ← 核心跳跃
否则:
    p[i] = 1（只有自身）
然后暴力向两边扩展
最后更新 mid 和 mr
```

ASCII 图示：
```
  [----------- mid 的回文 -----------]
  |         mid         |            mr
  |    j    |    i      |
  |<--p[j]->|    |<--至少这么长-->|
  对称点 j 的回文信息可以"搬运"给 i
```

### 2.3 Manacher 骨架代码

```cpp
const int MAXN = 2000005;
char s[MAXN], t[MAXN]; // s 原串, t 变换后
int p[MAXN];           // 回文半径数组

int manacher(const char* s, int n) {
    // 构造变换串 t
    int len = 0;
    t[len++] = '$'; // 左哨兵
    t[len++] = '#';
    for (int i = 0; i < n; i++) {
        t[len++] = s[i];
        t[len++] = '#';
    }
    t[len++] = '@'; // 右哨兵（防越界）
    t[len] = '\0';

    int mid = 0, mr = 0, ans = 0;
    for (int i = 1; i < len - 1; i++) {
        if (i < mr)
            p[i] = min(p[2 * mid - i], mr - i);
        else
            p[i] = 1;
        // 暴力扩展
        while (t[i + p[i]] == t[i - p[i]]) p[i]++;
        // 更新最远右端点
        if (i + p[i] > mr) {
            mr = i + p[i];
            mid = i;
        }
        ans = max(ans, p[i] - 1); // 原串中回文长度
    }
    return ans;
}
```

### 2.4 复杂度分析

- **时间复杂度**：O(n)。虽然有 while 扩展，但 mr 是单调递增的，每个字符最多被扩展一次。
- **空间复杂度**：O(n)。需要存储变换串 t 和 p 数组。

## 三、分类与变体详解

### 3.1 标准 Manacher（最长回文子串）

最基础的应用：直接取 `max(p[i] - 1)` 即为最长回文子串长度。

```
算法思路：Manacher 求 p 数组，取最大值
关键步骤：插入分隔符 → 利用对称性 + 扩展 → 维护 mid/mr
时间复杂度：O(n)
空间复杂度：O(n)
```

```cpp
// 求最长回文子串长度
int longest = 0;
for (int i = 1; i < len - 1; i++)
    longest = max(longest, p[i] - 1);
```

### 3.2 前/后缀最长回文

求以某个位置为结尾（或开头）的最长回文子串。Manacher 之后，利用 p 数组可推导出每个位置的端点信息：

- 以原串位置 j 结尾的最长回文 = 包含 j 的回文中最长的那个
- 通过差分数组递推：从右往左扫，`pre[i] = max(pre[i], pre[i+2] - 2)` 保证连续性

```
算法思路：Manacher + 端点贡献 + 递推修正
关键步骤：对每个中心 i，将 p[i]-1 贡献到左右端点 → 递推填补空缺
时间复杂度：O(n)
空间复杂度：O(n)
```

```cpp
// pre[i]: 以变换串位置 i 结尾的最长回文长度
// suf[i]: 以变换串位置 i 开头的最长回文长度
int pre[MAXN], suf[MAXN];

// 初始贡献
for (int i = 1; i < len - 1; i++) {
    int L = i - p[i] + 1, R = i + p[i] - 1;
    pre[R] = max(pre[R], p[i] - 1);
    suf[L] = max(suf[L], p[i] - 1);
}
// 递推修正：如果 i 处有长度 k 的回文结尾，i-2 处至少有 k-2 的
for (int i = len - 3; i >= 1; i--)
    pre[i] = max(pre[i], pre[i + 2] - 2);
for (int i = 2; i < len - 1; i++)
    suf[i] = max(suf[i], suf[i - 2] - 2);
```

### 3.3 Manacher + DP 综合

Manacher 常作为预处理工具，与 DP 结合解决更复杂的问题：

- **回文分割**：`dp[i]` = 前 i 个字符的最少回文分割数，用 p 数组 O(1) 判定子串是否回文
- **最长双回文串**：对每个分割点，取左侧最长回文 + 右侧最长回文
- **回文子序列计数**：Manacher 辅助识别回文子串，再做组合计数

```
算法思路：Manacher 预处理 + DP 转移
关键步骤：p 数组 → 端点信息 → DP 状态转移利用回文性质
时间复杂度：O(n) ~ O(n²)（取决于 DP 复杂度）
空间复杂度：O(n)
```

### 3.4 回文计数与拆分

利用 p 数组统计回文子串总数、各长度回文的出现次数：

- 以 i 为中心的回文子串个数 = `⌈p[i]/2⌉`（变换串上）
- 总回文子串数 = `Σ⌈p[i]/2⌉`（对原字符位置求和）
- 配合差分数组可统计"经过位置 x 的回文数量"

```
算法思路：p 数组 → 差分/前缀和 → 统计各类回文信息
关键步骤：将 p[i] 拆解为对各端点的贡献 → 前缀和聚合
时间复杂度：O(n)
空间复杂度：O(n)
```

```cpp
// 统计所有回文子串数量
long long total = 0;
for (int i = 1; i < len - 1; i++) {
    // 变换串上 i 对应的回文子串数（映射回原串）
    total += (p[i]) / 2; // 向下取整：每个 p[i] 贡献 p[i]/2 个原串回文
}
```

### 变体对比总表

| 变体 | 核心额外操作 | 典型题目 | 复杂度 |
|------|------------|---------|--------|
| 最长回文子串 | 直接取 max(p[i]-1) | HDU3068, P3805 | O(n) |
| 前/后缀最长回文 | 端点贡献 + 递推 | P4555, CF1326D2 | O(n) |

## 四、15 道经典例题详解

### 🟢 入门级 Level 1：Manacher 模板直接应用

### 题目 1：最长回文（HDU3068）

> 给定一个字符串（长度 ≤ 110000），求其最长回文子串的长度。

**分析**：纯模板题，直接运行 Manacher 算法，输出 `max(p[i] - 1)` 即可。

```
算法思路：标准 Manacher 求最长回文子串
关键步骤：插入分隔符 → 维护 mid/mr → 取最大 p[i]-1
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 220005;
char s[MAXN], t[MAXN];
int p[MAXN];

void solve(int n) {
    int len = 0;
    t[len++] = '$'; t[len++] = '#';
    for (int i = 0; i < n; i++) t[len++] = s[i], t[len++] = '#';
    t[len++] = '@'; t[len] = '\0';

    int mid = 0, mr = 0, ans = 0;
    for (int i = 1; i < len - 1; i++) {
        p[i] = (i < mr) ? min(p[2 * mid - i], mr - i) : 1;
        while (t[i + p[i]] == t[i - p[i]]) p[i]++;
        if (i + p[i] > mr) mr = i + p[i], mid = i;
        ans = max(ans, p[i] - 1);
    }
    // ans 即为最长回文子串长度
}
```

**核心启示**：
- **Manacher 模板**是所有回文问题的基础，必须做到默写无误
- **哨兵字符 `$` 和 `@`** 避免了边界判断，是一个优雅的工程技巧

### 题目 2：【模板】manacher 算法（洛谷 P3805）

> 给定一个仅由小写字母组成的字符串（长度 ≤ 1.1×10⁷），求最长回文子串的长度。

**分析**：与 HDU3068 相同，但数据规模更大（10⁷ 级别），要求 O(n) 算法。需注意用字符数组而非 string，避免常数过大。

```
算法思路：标准 Manacher 模板，大规模数据
关键步骤：字符数组读入 → Manacher → 取 max(p[i]-1)
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 22000005;
char s[MAXN], t[MAXN];
int p[MAXN];

void solve(int n) {
    int len = 0;
    t[len++] = '$'; t[len++] = '#';
    for (int i = 0; i < n; i++) t[len++] = s[i], t[len++] = '#';

    int mid = 0, mr = 0, ans = 0;
    for (int i = 1; i < len - 1; i++) {
        p[i] = (i < mr) ? min(p[2 * mid - i], mr - i) : 1;

        if (i + p[i] > mr) mr = i + p[i], mid = i;
        ans = max(ans, p[i] - 1);
    }
    // 输出 ans
}
```

**核心启示**：
- **大规模字符串务必用 `char[]` 而非 `string`**，避免 TLE
- **数组大小需开 2n+3**（变换后长度约为 2n+2），P3805 的 n 可达 1.1×10⁷

### 题目 3：最长回文子串（LeetCode 5）

> 给定一个字符串 s（长度 ≤ 1000），返回 s 中最长的回文子串。

**分析**：经典面试题。虽然 O(n²) 的中心扩展法已足够，但 Manacher 是最优解。需要记录最长回文的起始位置以便提取子串。

```
算法思路：Manacher 求最长回文子串，记录起始位置
关键步骤：记录最大 p[i] 对应的中心 → 反推原串起始下标 (i-p[i])/2
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 2005;
char t[MAXN];
int p[MAXN];

// 返回 {起始下标, 长度}
pair<int,int> solve(const char* s, int n) {
    int len = 0;
    t[len++] = '$'; t[len++] = '#';
    for (int i = 0; i < n; i++) t[len++] = s[i], t[len++] = '#';
    t[len++] = '@'; t[len] = '\0';

    int mid = 0, mr = 0, best = 0, center = 0;
    for (int i = 1; i < len - 1; i++) {
        p[i] = (i < mr) ? min(p[2 * mid - i], mr - i) : 1;
        while (t[i + p[i]] == t[i - p[i]]) p[i]++;
        if (i + p[i] > mr) mr = i + p[i], mid = i;
        if (p[i] - 1 > best) {
            best = p[i] - 1;
            center = i;
        }
    }
    int start = (center - best) / 2; // 映射回原串
    return {start, best};
}
```

**核心启示**：
- **变换串下标与原串的映射关系**：原串起始位置 = `(center - p[center] + 1) / 2` 或等价地 `(center - best) / 2`
- **面试中 Manacher 是加分项**，但需要能清晰讲解对称性优化的原理

### 题目 4：回文子串计数（LeetCode 647）

> 给定一个字符串 s（长度 ≤ 1000），返回 s 中回文子串的总数。不同位置的相同字符串视为不同子串。

**分析**：每个中心 i 在变换串上的回文半径为 p[i]，对应原串的回文子串数量为 `⌊p[i]/2⌋`。对所有位置求和即可。

```
算法思路：Manacher 求 p 数组 → 累加每个中心的回文贡献
关键步骤：对每个 i 计算 p[i]/2（下取整）并求和
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 2005;
char t[MAXN];
int p[MAXN];

int countPalindromes(const char* s, int n) {
    int len = 0;
    t[len++] = '$'; t[len++] = '#';
    for (int i = 0; i < n; i++) t[len++] = s[i], t[len++] = '#';
    t[len++] = '@'; t[len] = '\0';

    int mid = 0, mr = 0, ans = 0;
    for (int i = 1; i < len - 1; i++) {
        p[i] = (i < mr) ? min(p[2 * mid - i], mr - i) : 1;
        while (t[i + p[i]] == t[i - p[i]]) p[i]++;
        if (i + p[i] > mr) mr = i + p[i], mid = i;
        ans += p[i] / 2; // 每个中心贡献的回文子串数
    }
    return ans;
}
```

**核心启示**：
- **p[i]/2 的含义**：变换串上半径为 p[i]，包含 p[i]/2 个不同长度的原串回文
- **回文计数是 Manacher 的经典延伸**，也是后续"回文对计数"等高级问题的基础

### 题目 5：最长双回文串（洛谷 P4555）

> 给定一个字符串（长度 ≤ 10⁵），求最长的子串使其恰好由两个非空回文串拼接而成。

**分析**：对每个分割点，需要知道"以该点结尾的最长回文"和"以该点+1 开头的最长回文"。用 Manacher 求出 p 数组后，通过端点贡献 + 递推得到 pre/suf 数组，枚举分割点取最大值。

```
算法思路：Manacher + 端点贡献 + 递推修正 + 枚举分割点
关键步骤：p 数组 → pre[R]/suf[L] 端点贡献 → 递推补全 → max(pre[i]+suf[i+1])
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 200005;
char s[MAXN], t[MAXN];
int p[MAXN], pre[MAXN], suf[MAXN];

void solve(int n) {
    int len = 0;
    t[len++] = '$'; t[len++] = '#';
    for (int i = 0; i < n; i++) t[len++] = s[i], t[len++] = '#';
    t[len++] = '@'; t[len] = '\0';

    int mid = 0, mr = 0;
    for (int i = 1; i < len - 1; i++) {
        p[i] = (i < mr) ? min(p[2 * mid - i], mr - i) : 1;
        while (t[i + p[i]] == t[i - p[i]]) p[i]++;
        if (i + p[i] > mr) mr = i + p[i], mid = i;
    }
    // 端点贡献：以 i 为中心的回文覆盖 [i-p[i]+1, i+p[i]-1]
    for (int i = 1; i < len - 1; i++) {
        int L = i - p[i] + 1, R = i + p[i] - 1;
        pre[R] = max(pre[R], p[i] - 1);
        suf[L] = max(suf[L], p[i] - 1);
    }
    // 递推：若 i 处有长度 k 的回文结尾，i-2 处至少有 k-2
    for (int i = len - 3; i >= 1; i--)
        pre[i] = max(pre[i], pre[i + 2] - 2);
    for (int i = 2; i < len - 1; i++)
        suf[i] = max(suf[i], suf[i - 2] - 2);
    // 枚举分割点（只在 '#' 位置分割才对应原串的整数分割）
    int ans = 0;
    for (int i = 3; i < len - 3; i += 2) // '#' 位置
        if (pre[i] > 0 && suf[i] > 0)
            ans = max(ans, pre[i] + suf[i]);
    // ans 即为最长双回文串长度
}
```

**核心启示**：
- **端点贡献 + 递推修正**是 Manacher 最重要的延伸技巧，适用于所有"前/后缀最长回文"问题
- **分割点只在 `#` 位置**才对应原串中两个回文子串的无缝拼接

### 题目 6：拉拉队排练（洛谷 P1659）

> 给定长度为 n（≤ 10⁶）的字符串和整数 k（≤ 10¹²），求所有**奇数长度**回文子串中，选前 k 大的长度，输出这些长度的乘积模 19930726。若不足 k 个输出 -1。

**分析**：Manacher 求出 p 数组后，统计每种奇数长度的回文出现次数（用桶 cnt[]）。注意一个长为 L 的回文包含长为 L-2 的子回文，需**从大到小递推累加** cnt。最后从最长开始贪心选取，用快速幂计算乘积。

```
算法思路：Manacher + 桶计数 + 递推累加 + 快速幂
关键步骤：p 数组 → cnt[L]++ → 从大到小 cnt[i] += cnt[i+2] → 贪心选取 + 快速幂
时间复杂度：O(n log k)（快速幂 log k）
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 2000005;
const int MOD = 19930726;
char s[MAXN], t[MAXN];
int p[MAXN];
long long cnt[MAXN]; // cnt[i]: 长度为 i 的奇数回文子串数量

long long qpow(long long base, long long exp, long long mod) {
    long long res = 1;
    base %= mod;
    while (exp > 0) {
        if (exp & 1) res = res * base % mod;
        base = base * base % mod;
        exp >>= 1;
    }
    return res;
}

void solve(int n, long long k) {
    int len = 0;
    t[len++] = '$'; t[len++] = '#';
    for (int i = 0; i < n; i++) t[len++] = s[i], t[len++] = '#';
    t[len++] = '@'; t[len] = '\0';

    int mid = 0, mr = 0;
    for (int i = 1; i < len - 1; i++) {
        p[i] = (i < mr) ? min(p[2 * mid - i], mr - i) : 1;
        while (t[i + p[i]] == t[i - p[i]]) p[i]++;
        if (i + p[i] > mr) mr = i + p[i], mid = i;
        int L = p[i] - 1; // 原串回文长度
        if (L > 0 && L % 2 == 1) cnt[L]++;
    }
    // 从大到小递推：长为 L 的奇回文包含长为 L-2 的奇回文
    for (int i = n - (n % 2 == 0 ? 1 : 0); i >= 3; i -= 2)
        cnt[i - 2] += cnt[i];

    long long ans = 1;
    for (int i = n - (n % 2 == 0 ? 1 : 0); i >= 1; i -= 2) {
        if (k <= 0) break;
        if (cnt[i] == 0) continue;
        long long take = min((long long)cnt[i], k);
        ans = ans * qpow(i, take, MOD) % MOD;
        k -= take;
    }
    // k > 0 时输出 -1，否则输出 ans
}
```

**核心启示**：
- **回文包含关系**：长为 L 的回文必包含同中心的长为 L-2 的回文，递推累加是关键
- **快速幂处理大指数乘积**：k 可达 10¹²，必须用快速幂
- **只统计奇数长度**：利用 Manacher 后 p[i]-1 的奇偶性筛选

### 题目 7：双倍回文（洛谷 P4287）

> 给定长度 ≤ 500000 的字符串，求最长"双倍回文"子串的长度。双倍回文要求：长度为 4 的倍数，本身是回文，且前半部分和后半部分也各自是回文。

**分析**：Manacher 求出 p 数组后，枚举变换串上每个中心 i。一个长度为 4m 的双倍回文在变换串上半径为 4m，其前半段中心在 i - m 处。需要判断 p[i-m] 是否覆盖到 i（即 p[i-m] >= 2m+1）。利用单调队列或从内向外扩展优化。

```
算法思路：Manacher + 枚举中心 + 内层回文判定
关键步骤：p 数组 → 枚举中心 i → 检查 i±半径/2 处的 p 值 → 更新答案
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 1000005;
char s[MAXN], t[MAXN];
int p[MAXN];

void solve(int n) {
    int len = 0;
    t[len++] = '$'; t[len++] = '#';
    for (int i = 0; i < n; i++) t[len++] = s[i], t[len++] = '#';
    t[len++] = '@'; t[len] = '\0';

    int mid = 0, mr = 0;
    for (int i = 1; i < len - 1; i++) {
        p[i] = (i < mr) ? min(p[2 * mid - i], mr - i) : 1;
        while (t[i + p[i]] == t[i - p[i]]) p[i]++;
        if (i + p[i] > mr) mr = i + p[i], mid = i;
    }
    int ans = 0;
    // 枚举变换串上的中心（只看原字符位置，即偶数下标）
    for (int i = 1; i < len - 1; i++) {
        if (t[i] == '#') continue; // 双倍回文长度 4 的倍数，中心须在原字符上
        int r = p[i] - 1; // 变换串半径（不含中心）
        // 双倍回文在变换串上长度 4m+1，半径 2m
        // 前半中心在 i - m 处（'#' 位置），需 p[i-m] >= m+1
        // 枚举 m: 从 r/2 向下找最大满足条件的 m（4 的倍数对应偶数 m）
        for (int m = r / 2; m >= 2; m--) {
            if (m % 2 != 0) continue; // 保证原串长度是 4 的倍数
            if (p[i - m] >= m + 1) {
                ans = max(ans, 4 * (m / 2));
                break; // p 值只会更小，找到最大即可
            }
        }
    }
    // ans 即为最长双倍回文长度
}
// 注：上述朴素枚举 m 在最坏情况可能退化，竞赛中常用指针递增优化到均摊 O(n)
```

**核心启示**：
- **双倍回文的判定可分解为两层 Manacher 信息**：外层回文 + 内层半回文
- **变换串下标的奇偶性**直接关联原串的长度奇偶，利用这一点筛选合法中心
- **均摊优化**：实际竞赛中常用"对每个中心维护一个递增指针"使总复杂度 O(n)

### 题目 8：Palindromic characteristics（CF835D）

> 定义 k-回文：1-回文就是普通回文；k-回文（k>1）要求本身是回文且左半部分是 (k-1)-回文。对长度 ≤ 5000 的字符串，输出对每个 k（1 到 n），k-回文子串的个数。

**分析**：先用 DP 判定所有回文子串：`is_pal[l][r]`。再递推每个回文子串的最大 k 值：`d[l][r] = d[l][mid] + 1`（其中 mid = (l+r)/2）。最后从高到低累加统计。

```
算法思路：DP 判定回文 + 递推 k 值 + 从高到低累加
关键步骤：is_pal 判定 → d[l][r] 递推 k 值 → cnt[k] 统计 → 后缀累加
时间复杂度：O(n²)
空间复杂度：O(n²)
```

**C++ 代码**：

```cpp
const int MAXN = 5005;
bool is_pal[MAXN][MAXN];
int d[MAXN][MAXN]; // d[l][r] = 该回文子串的最大 k 值
long long cnt[MAXN];

void solve(const char* s, int n) {
    // 判定回文
    for (int i = n - 1; i >= 0; i--)
        for (int j = i; j < n; j++)
            is_pal[i][j] = (s[i] == s[j]) && (j - i < 2 || is_pal[i + 1][j - 1]);
    // 递推 k 值
    for (int i = 0; i < n; i++)
        for (int j = i; j < n; j++) {
            if (!is_pal[i][j]) { d[i][j] = 0; continue; }
            if (i == j) { d[i][j] = 1; continue; }
            int mid = (i + j - 1) / 2;
            d[i][j] = 1; // 至少是 1-回文
            if (is_pal[i][mid])
                d[i][j] = d[i][mid] + 1;
        }
    // 统计
    for (int i = 0; i < n; i++)
        for (int j = i; j < n; j++)
            if (d[i][j] > 0) cnt[d[i][j]]++;
    // 从高到低累加：k-回文也是 (k-1)-回文
    for (int k = n; k >= 2; k--)
        cnt[k - 1] += cnt[k];
    // cnt[1..n] 即为答案
}
```

**核心启示**：
- **k-回文的递归定义**天然适合 DP，关键是 `d[l][r] = d[l][mid] + 1`
- **从高到低累加**处理"k-回文也是 (k-1)-回文"的包含关系
- 本题 n ≤ 5000，O(n²) 足够；更大数据可用回文自动机

### 题目 9：Longest Regular Bracket Substring（CF5C）

> 给定一个括号序列（长度 ≤ 10⁶），求最长的合法括号子串的长度和这种最长长度的出现次数。

**分析**：虽然本题不是直接的回文问题，但解法中的"利用已知信息加速扩展"思想与 Manacher 一脉相承。用栈匹配括号，标记匹配位置后，求最长连续已匹配段。

```
算法思路：栈匹配括号 + 标记数组 + 扫描最长连续段
关键步骤：栈记录未匹配 '(' 下标 → 匹配时标记两端 → 扫描连续标记段
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 1000005;
bool matched[MAXN]; // matched[i] = true 表示位置 i 参与了合法括号匹配
int stk[MAXN], top;

void solve(const char* s, int n) {
    top = 0;
    for (int i = 0; i < n; i++) {
        if (s[i] == '(') {
            stk[top++] = i;
        } else if (top > 0) {
            matched[stk[--top]] = true;
            matched[i] = true;
        }
    }
    int maxLen = 0, cnt = 0, cur = 0;
    for (int i = 0; i < n; i++) {
        if (matched[i]) {
            cur++;
        } else {
            cur = 0;
        }
        if (cur > maxLen) {
            maxLen = cur;
            cnt = 1;
        } else if (cur == maxLen && maxLen > 0) {
            cnt++;
        }
    }
    // maxLen 和 cnt 即为答案；maxLen=0 时特判输出 "0 1"
}
```

**核心启示**：
- **"已知信息加速"**是一种通用思维：Manacher 用已知回文跳过比较，括号匹配用栈跳过搜索
- **标记 + 扫描最长连续段**是处理"最长合法子串"的通用技巧
- 本题训练的是与 Manacher 相同的"区间扩展/维护"思维

### 题目 10：最少回文分割（LeetCode 132）

> 给定一个字符串 s（长度 ≤ 2000），求将 s 分割为若干回文子串所需的最少切割次数。

**分析**：经典 DP 问题。`dp[i]` = 前 i 个字符最少分割次数。转移时需快速判断 s[j..i] 是否回文。可以用 Manacher 预处理或 O(n²) DP 预处理回文判定表，再做线性 DP。

```
算法思路：回文预处理 + 线性 DP
关键步骤：is_pal[i][j] 预处理 → dp[i] = min(dp[j]+1) 对所有 is_pal[j+1][i] 的 j
时间复杂度：O(n²)
空间复杂度：O(n²)
```

**C++ 代码**：

```cpp
const int MAXN = 2005;
bool is_pal[MAXN][MAXN];
int dp[MAXN]; // dp[i]: s[0..i-1] 的最少分割次数

void solve(const char* s, int n) {
    // 预处理回文
    for (int i = n - 1; i >= 0; i--)
        for (int j = i; j < n; j++)
            is_pal[i][j] = (s[i] == s[j]) && (j - i < 2 || is_pal[i + 1][j - 1]);

    for (int i = 0; i < n; i++) {
        if (is_pal[0][i]) { dp[i] = 0; continue; }
        dp[i] = i; // 最多切 i 刀
        for (int j = 1; j <= i; j++)
            if (is_pal[j][i])
                dp[i] = min(dp[i], dp[j - 1] + 1);
    }
    // dp[n-1] 即为答案
}
```

**核心启示**：
- **回文预处理是许多回文 DP 问题的第一步**，Manacher 或 O(n²) DP 均可
- **分割 DP 的转移**依赖于快速的回文判定，这是 Manacher 与 DP 结合的典型场景
- 进阶优化可用 Manacher + BFS 做到 O(n)

### 题目 11：最短回文前缀（LeetCode 214）

> 给定字符串 s（长度 ≤ 5×10⁴），在 s 的前面添加最少字符使整个字符串成为回文。

**分析**：等价于求 s 的**最长回文前缀**，剩余部分反转后添加到前面。Manacher 后遍历所有中心，找到右端点恰好到达字符串末尾的最长回文即可。也可用 KMP 解决。

```
算法思路：Manacher 求最长回文前缀
关键步骤：Manacher 求 p 数组 → 找满足 i+p[i]-1 == len-2 的最大回文 → 反转剩余部分
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 100005;
char s[MAXN], t[MAXN];
int p[MAXN];

int longestPalPrefix(const char* s, int n) {
    int len = 0;
    t[len++] = '$'; t[len++] = '#';
    for (int i = 0; i < n; i++) t[len++] = s[i], t[len++] = '#';
    t[len++] = '@'; t[len] = '\0';

    int mid = 0, mr = 0, best = 0;
    for (int i = 1; i < len - 1; i++) {
        p[i] = (i < mr) ? min(p[2 * mid - i], mr - i) : 1;
        while (t[i + p[i]] == t[i - p[i]]) p[i]++;
        if (i + p[i] > mr) mr = i + p[i], mid = i;
        // 检查该回文是否从原串位置 0 开始
        int origL = (i - p[i] + 1) / 2; // 映射到原串左端
        if (origL == 0)
            best = max(best, p[i] - 1);
    }
    return best; // 最长回文前缀长度
    // 需添加 s[best..n-1] 的反转到前面
}
```

**核心启示**：
- **回文前缀/后缀**问题的通用思路：Manacher 后筛选端点条件
- **KMP 做法**也是经典替代：构造 `s + "#" + rev(s)` 后求 fail 数组
- 两种方法的时间复杂度相同，但 Manacher 更具通用性

### 题目 12：前缀后缀回文（CF1326D2）

> 给定字符串 s，构造一个回文串 t，使得 t 是 s 的前缀或 s 的后缀的拼接形成的最长回文。具体地：取 s 的一个前缀 a 和一个后缀 c（不重叠），使 a+b+c 是回文，其中 b 是 a、c 之间剩余部分的最长回文前缀或后缀。

**分析**：先贪心匹配 s 的首尾相同字符（双指针），得到已匹配的前后缀长度 k。然后对中间剩余部分 s[k..n-k-1] 用 Manacher 求最长回文前缀和最长回文后缀，取较长者拼接。

```
算法思路：双指针匹配首尾 + Manacher 求中间段最长回文前缀/后缀
关键步骤：首尾匹配 → 中间段 Manacher → 选最长回文前缀或后缀 → 拼接
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 200005;
char t[MAXN];
int p[MAXN];

// 返回 s[0..n-1] 的最长回文前缀长度和最长回文后缀长度
pair<int,int> palPrefixSuffix(const char* s, int n) {
    int len = 0;
    t[len++] = '$'; t[len++] = '#';
    for (int i = 0; i < n; i++) t[len++] = s[i], t[len++] = '#';
    t[len++] = '@'; t[len] = '\0';

    int mid = 0, mr = 0, bestPre = 0, bestSuf = 0;
    for (int i = 1; i < len - 1; i++) {
        p[i] = (i < mr) ? min(p[2 * mid - i], mr - i) : 1;
        while (t[i + p[i]] == t[i - p[i]]) p[i]++;
        if (i + p[i] > mr) mr = i + p[i], mid = i;
        int origL = (i - p[i] + 1) / 2;
        int origR = (i + p[i] - 2) / 2;
        if (origL == 0) bestPre = max(bestPre, p[i] - 1);
        if (origR == n - 1) bestSuf = max(bestSuf, p[i] - 1);
    }
    return {bestPre, bestSuf};
}

// 主函数中：
// 1. 双指针匹配 s[0..k-1] 和 s[n-k..n-1]
// 2. 对 s[k..n-k-1] 调用 palPrefixSuffix
// 3. 拼接 s[0..k-1] + 中间最长回文 + s[n-k..n-1]
```

**核心启示**：
- **首尾匹配 + 中间 Manacher** 是一种分治式的解题思路
- **回文前缀/后缀的求法**在多种题目中反复出现
- 本题是 Codeforces Div.1 D 题，综合了贪心、Manacher、字符串拼接

### 题目 13：Palisection（CF17E）

> 给定长度为 n（≤ 2×10⁶）的字符串，求有多少对回文子串的区间有交集（相交或包含）。答案对 51123987 取模。

**分析**：正难则反。总对数 = C(total, 2)，减去不相交的对数。用 Manacher 计算每个位置有多少回文子串结束（diff 数组），前缀和得到"位置 i 之前结束的回文数"，乘以"位置 i 之后开始的回文数"即为不相交对数。

```
算法思路：Manacher + 差分统计 + 容斥（总对数 - 不相交对数）
关键步骤：p 数组 → 差分求每个位置的 start/end 回文数 → 前缀和 → 容斥
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 4000005;
const int MOD = 51123987;
char s[MAXN], t[MAXN];
int p[MAXN];
long long st[MAXN], ed[MAXN]; // st[i]: 在原串位置 i 开始的回文数; ed[i]: 在 i 结束的

void solve(int n) {
    int len = 0;
    t[len++] = '$'; t[len++] = '#';
    for (int i = 0; i < n; i++) t[len++] = s[i], t[len++] = '#';
    t[len++] = '@'; t[len] = '\0';

    int mid = 0, mr = 0;
    long long total = 0;
    for (int i = 1; i < len - 1; i++) {
        p[i] = (i < mr) ? min(p[2 * mid - i], mr - i) : 1;
        while (t[i + p[i]] == t[i - p[i]]) p[i]++;
        if (i + p[i] > mr) mr = i + p[i], mid = i;
        total = (total + p[i] / 2) % MOD; // 该中心贡献的回文子串数
    }
    // 差分统计每个原串位置的 start/end
    // 变换串中心 i 对应原串区间 [L, R]，L=(i-p[i]+1)/2, R=(i+p[i]-2)/2
    // 半径从 1 到 p[i]，每个半径对应一个回文子串
    // 用差分：st[L]++, st[center+1]-- 等处理
    for (int i = 1; i < len - 1; i++) {
        int oL = (i - p[i] + 1 + 1) / 2; // 最大回文的原串左端
        int oR = (i + p[i] - 1 - 1) / 2; // 最大回文的原串右端
        int oC = (i - 1) / 2;             // 中心的原串近似位置
        // 以 i 为中心的回文子串左端范围 [oL, oC] → st[oL]++, st[oC+1]--
        st[oL]++;
        if (oC + 1 < n) st[oC + 1]--;
        // 右端范围 [oC, oR] → ed[oC]++, ed[oR+1]--
        ed[oC]++;
        if (oR + 1 < n) ed[oR + 1]--;
    }
    // 前缀和还原
    for (int i = 1; i < n; i++) st[i] += st[i - 1], ed[i] += ed[i - 1];

    // 不相交对数：对每个位置 i，ed[i] * (位置 i+1 之后开始的回文总数)
    // 后缀和 suffSt[i] = st[i] + st[i+1] + ... + st[n-1]
    long long suffSt = 0, disjoint = 0;
    for (int i = n - 1; i >= 0; i--) {
        suffSt = (suffSt + st[i]) % MOD;
        if (i > 0)
            disjoint = (disjoint + ed[i - 1] % MOD * (suffSt % MOD)) % MOD;
    }

    long long allPairs = total % MOD * ((total - 1 + MOD) % MOD) % MOD;
    // C(total,2) = total*(total-1)/2，注意模下除法用逆元或提前处理
    // 由于 MOD 是奇数，total*(total-1) 必有一个偶数，可直接除
    allPairs = allPairs % MOD;
    // 需要模逆元求 /2
    long long inv2 = (MOD + 1) / 2;
    allPairs = allPairs * inv2 % MOD;

    long long ans = (allPairs - disjoint % MOD + MOD) % MOD;
    // ans 即为答案
}
```

**核心启示**：
- **正难则反**：直接数相交对很难，转化为"总对数 - 不相交对数"
- **差分数组统计回文端点分布**是 Manacher 的高级应用
- **模运算细节**：MOD = 51123987 不是质数（= 3×17041329），需注意逆元存在性；此处 2 与 MOD 互素故可用 `(MOD+1)/2`

### 题目 14：回文子串统计优化（AcWing 139 / BZOJ 2565）

> 给定一个字符串（长度 ≤ 10⁵），对每个位置 i，求以 i 为右端点的最长回文子串长度。输出所有位置的答案。

**分析**：Manacher 求 p 数组后，每个中心 i 的回文覆盖到右端 i+p[i]-1。对右端点做贡献并递推修正即可。这是"前/后缀最长回文"变体的直接应用。

```
算法思路：Manacher + 右端点贡献 + 递推修正
关键步骤：p 数组 → 对每个中心 i，在右端点记录贡献 → 从右往左递推
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 200005;
char s[MAXN], t[MAXN];
int p[MAXN], ans[MAXN]; // ans[i]: 以原串位置 i 为右端点的最长回文长度

void solve(int n) {
    int len = 0;
    t[len++] = '$'; t[len++] = '#';
    for (int i = 0; i < n; i++) t[len++] = s[i], t[len++] = '#';
    t[len++] = '@'; t[len] = '\0';

    int mid = 0, mr = 0;
    for (int i = 1; i < len - 1; i++) {
        p[i] = (i < mr) ? min(p[2 * mid - i], mr - i) : 1;
        while (t[i + p[i]] == t[i - p[i]]) p[i]++;
        if (i + p[i] > mr) mr = i + p[i], mid = i;
        // 该回文在原串的右端点
        int origR = (i + p[i] - 2) / 2;
        int palLen = p[i] - 1;
        if (palLen > 0)
            ans[origR] = max(ans[origR], palLen);
    }
    // 递推修正：位置 i 有长 k 的回文结尾，位置 i-1 至少有 k-2
    for (int i = n - 2; i >= 0; i--)
        ans[i] = max(ans[i], ans[i + 1] - 2);
    // 保证每个位置至少为 1（单字符回文）
    for (int i = 0; i < n; i++)
        ans[i] = max(ans[i], 1);
}
```

**核心启示**：
- **端点贡献 + 递推修正**是 Manacher 端点类问题的统一范式
- **递推方向**取决于求的是左端点信息还是右端点信息
- 此技巧在 P4555（双回文串）等多题中反复出现

### 🟣 挑战级 Level 5：Manacher + 高级数据结构/综合

### 题目 15：回文串综合应用（CF906E）

> 给定两个长度相同（≤ 5×10⁵）的字符串 s 和 t，可以选若干不相交区间 [l₁,r₁], [l₂,r₂],...，将 s 的这些区间反转，使 s 变成 t。求最少的反转区间数，或判断不可能。

**分析**：将问题转化为：找到 s 和 t 不同的位置，这些位置需要被覆盖。当一个区间被反转时，相当于该区间内的字符做了一次镜像操作。构造差异串 d[i] = (s[i] != t[i])，问题变为用最少的回文区间覆盖所有 d[i]=1 的位置。这需要 Manacher 预处理回文信息 + 贪心/DP。

```
算法思路：差异串构造 + Manacher 预处理 + 区间覆盖 DP
关键步骤：构造差异信息 → 回文区间预处理 → 贪心/DP 求最少覆盖
时间复杂度：O(n log n)
空间复杂度：O(n)
```

```cpp

void solve(int n) {
    for (int i = 0; i < n; i++)
        diff[i] = (s[i] != t2[i]);

    // 检查基本可行性：反转保持字符集不变
    // 构造匹配串进行 Manacher 分析
    // DP 求解最少区间覆盖（具体实现因变体而异）

}

```

**核心启示**：
- **问题转化能力**：将"区间反转"转化为"回文区间覆盖"是关键突破口
- **Manacher 作为预处理工具**，为后续 DP/贪心提供高效的回文判定
- **综合题往往需要多个算法协同**：Manacher + DP + 贪心，考验整合能力

## 五、常见技巧与思路总结

### 5.1 变换串下标与原串的映射

变换串 `$ # a # b # a # @` 中：
- 原串字符 `s[i]` 在变换串中的位置为 `2*i + 2`
- 变换串位置 `j` 对应原串位置 `(j - 2) / 2`（仅当 j 为偶数时对应原字符）
- 回文长度映射：**变换串半径 p[i] 对应原串回文长度 p[i] - 1**

### 5.2 哨兵字符的选择

```cpp
t[0] = '$';        // 左哨兵，不等于任何 s 中的字符和 '#'
t[len-1] = '@';    // 右哨兵
// 作用：while 扩展时自动停止，无需边界判断
```

### 5.3 端点贡献 + 递推修正模式

这是 Manacher 最重要的延伸技巧：

```
1. 对每个中心 i，将 p[i]-1 贡献到对应的左/右端点
2. 从外向内递推：f[i] = max(f[i], f[i±2] - 2)
3. 结果：每个位置获得以该位置为端点的最长回文长度
```

### 5.4 回文数量的差分统计

```
每个中心 i 贡献 p[i]/2 个回文子串
这些回文的左端点范围 [L, C]，右端点范围 [C, R]
用差分数组 O(1) 登记，前缀和 O(n) 还原
```

### 5.5 "正难则反"的容斥思路

当正面计数困难时（如"相交回文对"），转为：

```
目标 = 总量 - 反面
相交对数 = C(total, 2) - 不相交对数
```

不相交对数可通过前缀和高效计算。

## 七、终极总结与归纳

### 7.1 Manacher 的本质

**利用已知回文的对称性，将冗余的字符比较跳过，从而在线性时间内获取所有位置的回文信息。**

它不仅是一个"求最长回文子串"的工具，更是一个**回文信息的全量预处理器**。配合差分、前缀和、DP 等技巧，可以解决几乎所有与"回文子串"相关的问题。

### 7.2 核心要点

1. **分隔符统一奇偶**：插入 `#` 后所有回文变成奇数长度，统一处理
2. **mid/mr 维护**：始终跟踪右端点最远的回文，是 O(n) 的关键
3. **对称性跳跃**：`p[i] = min(p[2*mid-i], mr-i)` 是核心加速
4. **p[i]-1 = 原串回文长度**：变换串与原串的映射关系
5. **端点贡献 + 递推**：将中心信息转化为端点信息的标准手法
6. **容斥与差分**：高级计数问题的两大利器
