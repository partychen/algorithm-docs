## 一、什么是字符串哈希

字符串哈希（String Hashing）是一种将字符串映射为整数的技术。它的核心思想是：把字符串看作某个进制下的"大数"，通过取模运算将其压缩为一个较小的整数（哈希值）。两个字符串相同，则哈希值必然相同；哈希值不同，则字符串必然不同。

这一技术被广泛应用于**字符串匹配、去重、区间比较、最长公共前/后缀（LCP/LCS）** 等场景。与暴力逐字符比较 O(n) 不同，哈希比较只需 O(1)，极大提升了效率。

**字符串哈希 vs 其他算法对比：**

| 特性 | 字符串哈希 | KMP | 后缀数组 | Trie |
|------|-----------|-----|---------|------|
| 单次匹配 | O(n+m) | O(n+m) | O(n log n) | O(m) |
| 子串比较 | O(1) | 不支持 | O(1)（需RMQ） | 不支持 |
| 实现难度 | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐ |
| 出错概率 | 极低（双哈希） | 0 | 0 | 0 |
| 灵活性 | 极高 | 中等 | 高 | 中等 |

## 二、核心思想与前置知识

### 2.1 多项式哈希函数

将字符串 `s[0..n-1]` 视为 B 进制数：

`H(s) = s[0] * B^(n-1) + s[1] * B^(n-2) + ... + s[n-1] * B^0 (mod M)`

其中 B 是基数（base），M 是模数（modulus）。每个字符用其 ASCII 值（或偏移值）作为"数位"。

```
示例：s = "abc"，B = 131
H("abc") = 97 * 131^2 + 98 * 131^1 + 99 * 131^0
         = 97 * 17161 + 98 * 131 + 99
         = 1664617 + 12838 + 99
         = 1677554
```

### 2.2 基数与模数的选择

**基数 B** 的选择原则：
- 必须大于字符集大小（ASCII 可见字符取 B > 128）
- 常用值：`131`、`13331`、`1e9+7`、`1e9+9`
- 取质数可降低冲突概率

**模数 M** 的选择原则：
- 取大质数：`1e9+7`、`1e9+9`、`998244353`
- 自然溢出：利用 `unsigned long long` 的 2^64 自动取模
- **双哈希**：同时取两个不同模数，冲突概率降至 1/M1 × 1/M2 ≈ 10^{-18}

| 方案 | 冲突概率 | 速度 | 安全性 |
|------|---------|------|--------|
| 自然溢出 (ull) | ~1/2^64 | 最快 | 可被卡 |
| 单模 (1e9+7) | ~1/10^9 | 快 | 较弱 |
| 双模 | ~1/10^{18} | 稍慢 | 极强 |

### 2.3 前缀哈希与区间查询

预处理前缀哈希数组 `h[i]` 和幂数组 `pw[i]`，即可 O(1) 提取任意子串哈希：

```cpp
// 前缀哈希预处理骨架
const ull BASE = 131;
ull h[MAXN], pw[MAXN];

void build(const char s[], int n) {
    pw[0] = 1;
    for (int i = 1; i <= n; i++) pw[i] = pw[i - 1] * BASE;
    h[0] = 0;
    for (int i = 1; i <= n; i++) h[i] = h[i - 1] * BASE + s[i];
}

// 获取 s[l..r] 的哈希值（1-indexed）
ull get(int l, int r) {
    return h[r] - h[l - 1] * pw[r - l + 1];
}
```

**原理图解：**

```
字符串:  a   b   c   d   e
下标:    1   2   3   4   5
h[i]:   h1  h2  h3  h4  h5

get(2,4) = h[4] - h[1] * pw[3]
         = Hash("abcd") - Hash("a") * B^3
         = Hash("bcd")
```

### 2.4 时间与空间复杂度

- **预处理**：O(n) 时间、O(n) 空间
- **单次子串哈希查询**：O(1)
- **字符串比较**：O(1)（vs 暴力 O(n)）

## 三、分类与变体详解

### 3.1 单哈希 — 自然溢出

利用 `unsigned long long` 的溢出特性，等价于对 2^64 取模，速度最快但**可被构造数据卡掉**（Codeforces 常见 hack）。

```
算法思路：直接用 ull 存储哈希值，溢出即取模
关键步骤：h[i] = h[i-1] * BASE + s[i]
时间复杂度：O(n) 预处理 / O(1) 查询
空间复杂度：O(n)
```

```cpp
typedef unsigned long long ull;
const ull BASE = 131;
ull h[MAXN], pw[MAXN];

void build(const char s[], int n) {
    pw[0] = 1;
    for (int i = 1; i <= n; i++) pw[i] = pw[i - 1] * BASE;
    for (int i = 1; i <= n; i++) h[i] = h[i - 1] * BASE + s[i];
}

ull get(int l, int r) {
    return h[r] - h[l - 1] * pw[r - l + 1];
}
```

### 3.2 双哈希（Double Hash）

同时维护两组独立的 (base, mod)，返回 `pair<ull, ull>` 作为哈希值。冲突概率极低，竞赛中**防 hack** 的标准做法。

```
算法思路：两组独立哈希同时计算，比较时要求两组都相等
关键步骤：分别维护 h1[i], h2[i]，查询返回 pair
时间复杂度：O(n) 预处理 / O(1) 查询
空间复杂度：O(n)
```

```cpp
const ull B1 = 131, M1 = 1e9 + 7;
const ull B2 = 13331, M2 = 1e9 + 9;
ull h1[MAXN], h2[MAXN], pw1[MAXN], pw2[MAXN];

void build(const char s[], int n) {
    pw1[0] = pw2[0] = 1;
    for (int i = 1; i <= n; i++) {
        pw1[i] = pw1[i - 1] * B1 % M1;
        pw2[i] = pw2[i - 1] * B2 % M2;
    }
    for (int i = 1; i <= n; i++) {
        h1[i] = (h1[i - 1] * B1 + s[i]) % M1;
        h2[i] = (h2[i - 1] * B2 + s[i]) % M2;
    }
}

pair<ull, ull> get(int l, int r) {
    ull v1 = (h1[r] - h1[l - 1] * pw1[r - l + 1] % M1 + M1) % M1;
    ull v2 = (h2[r] - h2[l - 1] * pw2[r - l + 1] % M2 + M2) % M2;
    return {v1, v2};
}
```

### 3.3 前缀哈希 + 区间哈希查询

这是最常用的实战形式——预处理后 O(1) 比较任意两个子串。常见应用：

- **判断两个子串是否相等**：`get(l1, r1) == get(l2, r2)`
- **统计不同子串数**：枚举所有子串的哈希值去重
- **Rabin-Karp 匹配**：滑动窗口逐一比较模式串哈希

```
算法思路：前缀哈希数组支持 O(1) 区间哈希提取
关键步骤：get(l, r) = h[r] - h[l-1] * pw[r-l+1]
时间复杂度：O(n) 预处理 + O(1) 每次查询
空间复杂度：O(n)
```

### 3.4 二分 + 哈希求 LCP

求两个后缀（或任意两个子串）的**最长公共前缀（LCP）**：二分长度 `mid`，用哈希 O(1) 判断前 `mid` 个字符是否相同。

```
算法思路：二分答案 + 哈希判等
关键步骤：二分长度 mid，check get(i, i+mid-1) == get(j, j+mid-1)
时间复杂度：O(log n) 每次 LCP 查询
空间复杂度：O(n) 预处理
```

```cpp
// 求 s[i..] 与 s[j..] 的 LCP 长度
int lcp(int i, int j, int n) {
    int lo = 0, hi = min(n - i + 1, n - j + 1), ans = 0;
    while (lo <= hi) {
        int mid = (lo + hi) / 2;
        if (get(i, i + mid - 1) == get(j, j + mid - 1))
            ans = mid, lo = mid + 1;
        else
            hi = mid - 1;
    }
    return ans;
}
```

**变体对比总表：**

| 变体 | 哈希组数 | 取模方式 | 防 hack | 适用场景 |
|------|---------|---------|---------|---------|
| 自然溢出 | 1 | ull 溢出 | ✗ | 简单题 / 练习 |
| 单模哈希 | 1 | 大质数 | 较弱 | 一般竞赛 |
| 双模哈希 | 2 | 两大质数 | ✓ | 正式比赛 / CF |
| 前缀+区间 | 1-2 | 任意 | 视组数 | 子串比较类 |
| 二分+哈希 | 1-2 | 任意 | 视组数 | LCP / 最长重复 |

### 🟢 入门级 Level 1：基础哈希与去重

### 题目 1：字符串哈希模板（洛谷 P3370）

> 给定 n 个字符串，求其中有多少个不同的字符串。n ≤ 10000，每个字符串长度 ≤ 1500。

**分析**：最基础的字符串哈希模板题。对每个字符串计算哈希值，将所有哈希值排序或放入集合去重，最终统计不同值的个数。

```
算法思路：逐个字符串计算多项式哈希，排序去重统计
关键步骤：hash(s) = Σ s[i]*B^(n-1-i)，排序后统计不同值
时间复杂度：O(n * L + n log n)，L 为平均串长
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const ull BASE = 131;

ull calc(const char s[], int len) {
    ull res = 0;
    for (int i = 0; i < len; i++) res = res * BASE + s[i];
    return res;
}

void solve(int n, char strs[][1501]) {
    vector<ull> v(n);
    for (int i = 0; i < n; i++)
        v[i] = calc(strs[i], strlen(strs[i]));
    sort(v.begin(), v.end());
    int ans = unique(v.begin(), v.end()) - v.begin();
    // ans 即为不同字符串个数
}
```

**核心启示**：
- **哈希即指纹**：将字符串压缩为一个整数，O(1) 比较代替 O(n) 逐字符比较
- **排序去重**是竞赛中统计不同哈希值最简洁的方式，比 `set` 更快
- 本题用自然溢出即可通过，但正式比赛建议双哈希

### 题目 2：Buzzwords（UVA11855）

> 给定一个字符串，对于每个长度 L（从大到小），若存在某个长度为 L 的子串出现至少 2 次，则输出 L。

**分析**：经典的"重复子串检测"问题。对每个长度 L，用滚动哈希计算所有长度为 L 的子串哈希，检查是否有重复。从最大长度开始递减枚举。

```
算法思路：枚举子串长度 L，哈希所有 L 长子串，检查重复
关键步骤：对每个 L，用前缀哈希 O(1) 提取子串哈希，排序或 set 判重
时间复杂度：O(n^2) 最坏（枚举所有长度）
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const ull BASE = 131;
ull h[MAXN], pw[MAXN];

bool hasRepeat(int n, int L) {
    vector<ull> v;
    for (int i = 1; i + L - 1 <= n; i++)
        v.push_back(h[i + L - 1] - h[i - 1] * pw[L]);
    sort(v.begin(), v.end());
    for (int i = 1; i < (int)v.size(); i++)
        if (v[i] == v[i - 1]) return true;
    return false;
}

void solve(const char s[], int n) {
    // build(s, n) 预处理 h[] 和 pw[]
    for (int L = n - 1; L >= 1; L--)
        if (hasRepeat(n, L))
            // 输出 L
            ;
}
```

**核心启示**：
- **枚举长度 + 哈希判重**是"重复子串"类问题的通用套路
- 若只需最长重复子串长度，可用**二分 + 哈希**优化到 O(n log n)
- 排序后相邻比较是最简洁的判重方式

### 题目 3：魔族密码（洛谷 P1481）

> 给定 n 个字符串，求最长的"密码链"：链中每个字符串是下一个字符串的前缀。求链的最大长度。

**分析**：将所有字符串按长度排序，用哈希判断前缀关系。对每个字符串，检查是否存在某个更短的字符串是它的前缀（哈希值匹配），用 DP 求最长链。

```
算法思路：排序 + 哈希判前缀 + DP 求最长链
关键步骤：对每个串计算所有前缀哈希，DP 转移时 O(1) 判断前缀关系
时间复杂度：O(n^2)（枚举转移对）
空间复杂度：O(n * L)
```

**C++ 代码**：

```cpp
const ull BASE = 131;

ull fullHash[MAXN];  // 每个字符串的完整哈希
int len[MAXN], dp[MAXN];

// 计算字符串 s 长度 L 的前缀哈希
ull prefixHash(const char s[], int L) {
    ull res = 0;
    for (int i = 0; i < L; i++) res = res * BASE + s[i];
    return res;
}

void solve(int n, char strs[][MAXN], int idx[]) {
    // idx[] 是按长度排序后的下标
    int ans = 1;
    for (int i = 0; i < n; i++) dp[i] = 1;
    for (int i = 1; i < n; i++)
        for (int j = 0; j < i; j++)
            if (len[idx[j]] < len[idx[i]] &&
                prefixHash(strs[idx[i]], len[idx[j]]) == fullHash[idx[j]])
                dp[i] = max(dp[i], dp[j] + 1);
    for (int i = 0; i < n; i++) ans = max(ans, dp[i]);
    // ans 即为最长链长度
}
```

**核心启示**：
- **前缀判断**可通过"截取前缀哈希与整串哈希比较"在 O(1) 完成
- 哈希将"字符串前缀关系"转化为"数值相等判断"，简化了 DP 转移
- 按长度排序保证了 DP 的无后效性

### 题目 4：朵拉和哈希（洛谷 P4503）

> 给定一个长度为 n 的字符串，问有多少对 (i, j)（i < j）满足 s[i..i+L-1] 与 s[j..j+L-1] 是"字母重排"（即两个子串包含的每种字符出现次数相同）。

**分析**：两个子串互为重排，当且仅当它们的字符频次向量相同。为每个字符随机赋一个权值，子串的"频次哈希"等于区间内所有字符权值之和。用前缀和 O(1) 计算区间频次哈希，相同哈希值的子串即为重排对。

```
算法思路：随机赋权 + 前缀和 → 区间频次哈希，哈希值相同即重排
关键步骤：对 26 个字符各赋随机 ull 权值 w[c]；sum[i] = Σ w[s[k]] (k=1..i)
时间复杂度：O(n) 预处理 + O(n) 枚举
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
ull w[26], sum[MAXN];  // w[c] 为字符 c 的随机权值

void init() {
    mt19937_64 rng(chrono::steady_clock::now().time_since_epoch().count());
    for (int i = 0; i < 26; i++) w[i] = rng();
}

void build(const char s[], int n) {
    sum[0] = 0;
    for (int i = 1; i <= n; i++) sum[i] = sum[i - 1] + w[s[i] - 'a'];
}

ull getSum(int l, int r) { return sum[r] - sum[l - 1]; }

// 统计所有长度为 L 的子串中，频次哈希相同的对数
long long countPairs(int n, int L) {
    map<ull, int> cnt;
    long long ans = 0;
    for (int i = 1; i + L - 1 <= n; i++) {
        ull h = getSum(i, i + L - 1);
        ans += cnt[h];
        cnt[h]++;
    }
    return ans;
}
```

**核心启示**：
- **随机赋权哈希**：不按位置而按字符赋权，实现"顺序无关"的哈希
- 前缀和结构支持 O(1) 区间查询，与经典前缀哈希异曲同工
- 这种"频次哈希"思想可迁移到任何需要"多重集相等判断"的场景

### 题目 5：Watto and Mechanism（CF514C）

> 给定 n 个字符串集合 S（字符集 {a, b, c}），每次查询一个字符串 t，问 S 中是否存在某个字符串与 t 长度相同且恰好有 1 个位置不同。

**分析**：对 S 中每个字符串计算双哈希值存入集合。对于查询串 t，枚举每个位置 i，尝试将 `t[i]` 替换为其他两个字符，计算修改后的哈希值，查询是否在集合中。

```
算法思路：预处理集合哈希 → 枚举修改位 → 查集合
关键步骤：修改位 i 的哈希 = 原哈希 - t[i]*B^(n-1-i) + c*B^(n-1-i)
时间复杂度：O(Σ|s|) 预处理 + O(|t| * 2) 每次查询
空间复杂度：O(n)（哈希集合）
```

**C++ 代码**：

```cpp
const ull B1 = 131, M1 = 1e9 + 7, B2 = 13331, M2 = 1e9 + 9;
set<pair<ull, ull>> S;

pair<ull, ull> calc(const char s[], int n) {
    ull v1 = 0, v2 = 0;
    for (int i = 0; i < n; i++) {
        v1 = (v1 * B1 + s[i]) % M1;
        v2 = (v2 * B2 + s[i]) % M2;
    }
    return {v1, v2};
}

bool query(const char t[], int n) {
    ull v1 = 0, v2 = 0, p1 = 1, p2 = 1;
    // 先算完整哈希和幂次
    for (int i = 0; i < n; i++) {
        v1 = (v1 * B1 + t[i]) % M1;
        v2 = (v2 * B2 + t[i]) % M2;
    }
    // pw1[k], pw2[k] 为预处理的 B^k mod M
    for (int i = 0; i < n; i++) {
        for (char c = 'a'; c <= 'c'; c++) {
            if (c == t[i]) continue;
            ull nv1 = (v1 + (c - t[i]) * pw1[n - 1 - i] % M1 + M1) % M1;
            ull nv2 = (v2 + (c - t[i]) * pw2[n - 1 - i] % M2 + M2) % M2;
            if (S.count({nv1, nv2})) return true;
        }
        // 注意取模时处理负数
    }
    return false;
}
```

**核心启示**：
- **单字符修改的哈希增量**：`new_hash = old_hash + (new_c - old_c) * B^pos`
- 双哈希防冲突在本题尤为关键，CF 上单哈希极易被 hack
- 这是"近似匹配"用哈希解决的典型范式

### 题目 6：Milk Patterns（洛谷 P2852 / USACO06DEC）

> 给定长度为 n 的整数序列，求出现至少 k 次的最长子串长度。n ≤ 20000。

**分析**：二分子串长度 L，对每个 L 计算所有长度为 L 的子串哈希，统计最大出现次数是否 ≥ k。经典的"二分 + 哈希"模板。

```
算法思路：二分子串长度 + 哈希统计出现次数
关键步骤：check(L) = 是否存在某长度 L 子串出现 ≥ k 次
时间复杂度：O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const ull BASE = 131;
ull h[MAXN], pw[MAXN];
int a[MAXN]; // 原始整数序列

ull get(int l, int r) { return h[r] - h[l - 1] * pw[r - l + 1]; }

bool check(int n, int L, int k) {
    map<ull, int> cnt;
    for (int i = 1; i + L - 1 <= n; i++)
        if (++cnt[get(i, i + L - 1)] >= k) return true;
    return false;
}

int solve(int n, int k) {
    pw[0] = 1;
    for (int i = 1; i <= n; i++) pw[i] = pw[i - 1] * BASE;
    for (int i = 1; i <= n; i++) h[i] = h[i - 1] * BASE + a[i];
    int lo = 1, hi = n, ans = 0;
    while (lo <= hi) {
        int mid = (lo + hi) / 2;
        if (check(n, mid, k)) ans = mid, lo = mid + 1;
        else hi = mid - 1;
    }
    return ans;
}
```

**核心启示**：
- **二分 + 哈希**是求"最长满足条件的子串"的黄金组合
- 整数序列也可以用多项式哈希，把每个整数当作一个"字符"
- `check` 函数内用 `map` 或排序统计均可，`unordered_map` 最快

### 题目 7：Compress Words（CF1200E）

> 依次给出 n 个单词，逐个拼接。每次拼接时，去掉"当前结果的后缀"与"新单词前缀"的最长重叠部分。求最终结果。

**分析**：每次拼接新单词 w 时，需要找当前结果串 res 的后缀与 w 的前缀的最长匹配。可以对 w 建前缀哈希，对 res 维护后缀哈希，枚举重叠长度从长到短，用哈希 O(1) 判等。

```
算法思路：维护结果串哈希 + 新单词前缀哈希，枚举重叠长度判等
关键步骤：重叠长度 L ∈ [1, min(|res|, |w|)]，hash(res后缀L) == hash(w前缀L)
时间复杂度：O(Σ|w_i|)
空间复杂度：O(Σ|w_i|)
```

**C++ 代码**：

```cpp
const ull BASE = 131;

// res: 已拼接结果，rh[i]: res 的前缀哈希，pw[i]: BASE^i
ull rh[MAXN], pw[MAXN];
int rlen; // res 当前长度

ull getRes(int l, int r) { return rh[r] - rh[l - 1] * pw[r - l + 1]; }

void merge(const char w[], int wlen) {
    // 计算 w 的前缀哈希
    ull wh = 0;
    int maxOv = min(rlen, wlen), best = 0;
    for (int L = 1; L <= maxOv; L++) {
        wh = wh * BASE + w[L]; // w[1..L] 的哈希
        if (wh == getRes(rlen - L + 1, rlen)) best = L;
    }
    // 将 w[best+1..wlen] 追加到 res 末尾，同步更新 rh[]
    for (int i = best + 1; i <= wlen; i++) {
        rlen++;
        rh[rlen] = rh[rlen - 1] * BASE + w[i];
    }
}
```

**核心启示**：
- **前后缀重叠检测**是字符串拼接/合并的核心问题，哈希提供 O(1) 判等
- 维护结果串的前缀哈希数组，每次追加字符时增量更新
- 也可用 KMP 求 border，但哈希实现更直观

### 题目 8：DNA（洛谷 P3763 / TJOI2017）

> 给定文本串 s0 和模式串 s，求 s0 中有多少个长度为 |s| 的子串，与 s 至多有 3 个位置不同。

**分析**：对每个对齐位置，用"哈希 + 二分"快速跳过匹配部分。从对齐起点出发，二分查找第一个失配位置（用 LCP），跳过后继续匹配，最多跳 3 次。若 3 次失配后剩余部分仍全部匹配则合法。

```
算法思路：枚举对齐位置 → 二分找失配 → 最多允许 3 次失配
关键步骤：每次用 LCP(二分+哈希) 跳过匹配段，失配次数 ≤ 3 则合法
时间复杂度：O(n log n)（每个对齐位置最多 3 次二分）
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const ull BASE = 131;
ull hs[MAXN], ht[MAXN], pw[MAXN]; // s0 和 s 的前缀哈希

ull getS(int l, int r) { return hs[r] - hs[l - 1] * pw[r - l + 1]; }
ull getT(int l, int r) { return ht[r] - ht[l - 1] * pw[r - l + 1]; }

// s0[st..st+m-1] 与 s[1..m] 的失配数是否 ≤ 3
bool check(int st, int m) {
    int mis = 0, pos = 0;
    while (pos < m && mis <= 3) {
        // 二分求 s0[st+pos..] 与 s[pos+1..] 的 LCP
        int lo = 0, hi = m - pos, lcp = 0;
        while (lo <= hi) {
            int mid = (lo + hi) / 2;
            if (mid == 0 || getS(st + pos, st + pos + mid - 1) == getT(pos + 1, pos + mid))
                lcp = mid, lo = mid + 1;
            else hi = mid - 1;
        }
        pos += lcp;
        if (pos < m) { mis++; pos++; } // 跳过失配位置
    }
    return mis <= 3;
}
```

**核心启示**：
- **k 次失配匹配**的通用模板：LCP 跳过 + 失配计数
- 二分 + 哈希求 LCP 是 O(log n)，3 次失配总共 O(log n)
- 该技巧可推广到任意 k 次失配（复杂度 O(k log n) 每位置）

### 题目 9：ANT-Antisymmetry（洛谷 P3501 / POI2010）

> 给定长度为 n 的 01 串，统计有多少个"反对称"子串。反对称定义：翻转后每位取反等于原串（即 s[i] + s[n+1-i] = 1 对所有 i 成立）。

**分析**：反对称子串长度必须为偶数。对于每个可能的中心位置（相邻字符之间），二分找最大半径 r，使得 `s[c-r+1..c]` 翻转取反后等于 `s[c+1..c+r]`。预处理原串和取反翻转串的前缀哈希。

```
算法思路：枚举中心 + 二分最大半径 + 双哈希判等
关键步骤：正串哈希 h[]，反串取反哈希 rh[]，二分半径 r 使区间匹配
时间复杂度：O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const ull BASE = 131;
ull h[MAXN], rh[MAXN], pw[MAXN]; // h: 正串哈希, rh: 取反翻转串哈希
int n;

ull getH(int l, int r)  { return h[r] - h[l - 1] * pw[r - l + 1]; }
ull getRH(int l, int r) { return rh[r] - rh[l - 1] * pw[r - l + 1]; }

// 在翻转串中，原串 s[l..r] 对应的下标为 [n+1-r, n+1-l]
long long solve(const char s[]) {
    // 预处理 h[]（正串）和 rh[]（s 翻转后每位取反的前缀哈希）
    long long ans = 0;
    for (int c = 1; c < n; c++) { // 中心在 c 与 c+1 之间
        int lo = 1, hi = min(c, n - c), best = 0;
        while (lo <= hi) {
            int mid = (lo + hi) / 2;
            // 正串 s[c-mid+1..c] 对应翻转取反串中的 s[c+1..c+mid]
            if (getH(c + 1, c + mid) == getRH(n + 1 - c, n - c + mid))
                best = mid, lo = mid + 1;
            else hi = mid - 1;
        }
        ans += best; // 半径 1..best 都合法
    }
    return ans;
}
```

**核心启示**：
- **翻转 + 变换**类问题：预处理变换后的串的哈希，统一为区间比较
- 枚举中心 + 二分半径是处理"对称/反对称子串计数"的标准框架
- 二分得到最大半径 best 后，半径 1 到 best 的所有偶数长子串都合法

### 题目 10：Password（CF126B）

> 给定字符串 s，求最长的子串 t，使得 t 既是 s 的前缀、又是 s 的后缀、还在 s 的中间（非前缀非后缀位置）出现过。若不存在则输出 "Just a legend"。

**分析**：先用哈希找出所有同时是前缀和后缀的长度 L（即 `get(1, L) == get(n-L+1, n)`）。对这些候选 L，检查长度为 L 的子串是否在中间出现过——将所有子串哈希存入集合，排除首尾后检查。取最大的合法 L。

```
算法思路：枚举前缀=后缀的长度 → 哈希检查中间是否出现
关键步骤：候选 L 满足 get(1,L)==get(n-L+1,n)，中间检查用哈希集合
时间复杂度：O(n log n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const ull BASE = 131;
ull h[MAXN], pw[MAXN];
int n;

ull get(int l, int r) { return h[r] - h[l - 1] * pw[r - l + 1]; }

int solve(const char s[]) {
    // build h[], pw[]
    // 收集所有"中间位置"子串哈希
    set<ull> mid;
    // 中间子串起点 i ∈ [2, n-1]，只要子串不完全占据首尾即可
    // 枚举所有长度 L 的候选
    vector<int> cands;
    for (int L = 1; L < n; L++)
        if (get(1, L) == get(n - L + 1, n))
            cands.push_back(L);

    int ans = -1;
    for (int L : cands) {
        ull target = get(1, L);
        // 检查 s[2..n-1] 中是否存在长度 L 子串哈希等于 target
        for (int i = 2; i + L - 1 <= n - 1; i++)
            if (get(i, i + L - 1) == target) { ans = max(ans, L); break; }
    }
    return ans; // -1 表示不存在
}
```

**核心启示**：
- **前缀 = 后缀**的判断直接用首尾哈希比较
- 中间出现的检查可以用哈希集合预存所有中间子串
- 实际实现中可先筛候选再从大到小检查，找到即停止

### 题目 11：无论怎样神树大人都会删库跑路（洛谷 P5270）

> 给定初始序列和若干小串，依次插入小串到末尾，维护一个长度为 T 的滑动窗口。判断窗口内每种数字的出现频次是否与目标序列相同。统计满足条件的窗口数。

**分析**：这不是普通的字符串哈希，而是"频次多重集哈希"。对每种数字 d 赋随机权值 w[d]，窗口的哈希 = 窗口内所有元素权值之和。滑动窗口加入/移除时 O(1) 更新哈希，与目标哈希比较。

```
算法思路：随机赋权的频次哈希 + 滑动窗口
关键步骤：hash = Σ w[a[i]]（窗口内），与目标 hash 比较
时间复杂度：O(n)（滑动窗口遍历）
空间复杂度：O(n + V)，V 为值域
```

**C++ 代码**：

```cpp
ull w[MAXV]; // 每种数字的随机权值
ull targetHash; // 目标频次哈希

void init(int target[], int T) {
    mt19937_64 rng(42);
    for (int i = 0; i < MAXV; i++) w[i] = rng();
    targetHash = 0;
    for (int i = 0; i < T; i++) targetHash += w[target[i]];
}

int solve(int seq[], int n, int T) {
    ull curHash = 0;
    int ans = 0;
    for (int i = 0; i < n; i++) {
        curHash += w[seq[i]];
        if (i >= T) curHash -= w[seq[i - T]]; // 滑出窗口
        if (i >= T - 1 && curHash == targetHash) ans++;
    }
    return ans;
}
```

**核心启示**：
- **频次哈希**：对值赋随机权值，求和即得"多重集指纹"，顺序无关
- 滑动窗口 + 频次哈希 = O(1) 的"多重集相等"判断
- 该思想与题目 4（朵拉和哈希）一脉相承，是"随机化哈希"的核心应用

### 题目 12：字符串匹配（洛谷 P7114 / NOIP2020）

> 给定字符串 S，统计有多少种方式将 S 拆分为 (A, B, C)，使得 S = (AB)^k · C（k ≥ 1），其中 AB 重复 k 次后接上 C，且 C 中出现奇数次的字符数 ≤ A 中出现奇数次的字符数。

**分析**：枚举 AB 的长度 L（即周期长度），用哈希验证 S 的前 k*L 个字符确实是周期 L 的重复。对于每个合法的 (L, k)，统计 A 的所有可能长度中满足"奇数字符数"条件的个数。预处理前缀的奇数字符计数可用异或技巧。

```
算法思路：枚举周期长度 L → 哈希验证重复 → 枚举 A 长度统计合法拆分
关键步骤：hash(s[1..L]) == hash(s[L+1..2L]) == ... 验证周期性
时间复杂度：O(n log n)（调和级数枚举）
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const ull BASE = 131;
ull h[MAXN], pw[MAXN];
int oddCnt[MAXN]; // oddCnt[i]: s[1..i] 中出现奇数次的字符种数

ull get(int l, int r) { return h[r] - h[l - 1] * pw[r - l + 1]; }

long long solve(const char s[], int n) {
    // 预处理 h[], pw[], oddCnt[]
    // oddCnt 用 bitmask 异或维护: mask ^= (1 << (s[i]-'a'))
    long long ans = 0;
    for (int L = 1; L <= n / 2; L++) {
        // 找最大 k 使得 s[1..L] 重复 k 次
        int k = 1;
        while ((k + 1) * L <= n && get(1, L) == get(k * L + 1, (k + 1) * L))
            k++;
        // 对于每个有效 k'（1..k），C = s[k'*L+1..n]
        // 枚举 A 的长度 a ∈ [1, L-1]，统计满足条件的
        // 需要维护 oddCnt 的前缀信息用树状数组或预处理
        // （此处简化核心逻辑）
        for (int t = 1; t <= k; t++) {
            int cStart = t * L + 1;
            if (cStart > n) break;
            int cOdd = oddCnt[n] - oddCnt[cStart - 1]; // 简化
            // 统计 A 长度 a ∈ [1,L-1] 中 oddCnt[a] >= cOdd 的个数
            // 实际需要树状数组维护
            ans += /* 合法 A 长度个数 */ 0;
        }
    }
    return ans;
}
```

**核心启示**：
- **周期验证**：哈希 O(1) 判断 `s[1..L] == s[kL+1..(k+1)L]`
- 调和级数枚举周期倍数保证总复杂度 O(n log n)
- 字符奇偶性统计可用 bitmask 异或，配合树状数组做区间计数

### 题目 13：Palindromic Partitions（洛谷 P4656 / CEOI2017）

> 给定字符串 s，将其分成最多的段 p1, p2, ..., pk，使得 pi = p_{k+1-i} 对所有 i 成立。求最大的 k。

**分析**：贪心 + 哈希。从两端向中间扫描：令左指针从左往右、右指针从右往左，同时累积哈希值。一旦左侧和右侧的哈希相等，就分出一对段，答案 +2，然后重置继续。若最后中间还有剩余，答案 +1。

```
算法思路：双指针贪心匹配 + 哈希判等
关键步骤：从两端同步扫描，累积哈希，相等即分段
时间复杂度：O(n)
空间复杂度：O(1)（不计输入）
```

**C++ 代码**：

```cpp
const ull BASE = 131;

int solve(const char s[], int n) {
    int ans = 0, lo = 0, hi = n - 1;
    while (lo < hi) {
        ull hL = 0, hR = 0, pw = 1;
        bool found = false;
        int i = lo, j = hi;
        while (i < j) {
            hL = hL * BASE + s[i];
            hR = hR + s[j] * pw;
            pw *= BASE;
            i++; j--;
            if (hL == hR) { ans += 2; lo = i; hi = j; found = true; break; }
        }
        if (!found) break;
    }
    if (lo <= hi) ans++; // 中间剩余部分
    return ans;
}
```

**核心启示**：
- **贪心最短匹配**：越早匹配一对，剩余部分越多，段数越大
- 左哈希正向累积、右哈希反向累积，方向不同但比较结果等价
- 纯 O(n) 的优雅解法，体现了哈希在贪心中的应用

### 题目 14：系统设计（洛谷 P5537 / XR-3）

> 给定有根树（n 个节点）和长度为 m 的序列 a，支持两种操作：(1) 从节点 x 出发，依次按 a[l..r] 指示走向第 a[i] 小的儿子，输出最终到达的节点；(2) 修改 a[t] 的值。

**分析**：将每个节点的"从根到它的路径选择序列"编码为哈希值，建立哈希到节点的映射。查询时，将起点 x 的路径哈希与 a[l..r] 的区间哈希拼接，得到目标哈希，在映射表中查找对应节点。序列 a 用线段树维护区间哈希，支持单点修改。走不通时用二分找最远可达位置。

```
算法思路：路径哈希编码 + 线段树维护区间哈希 + 二分查找
关键步骤：target = H[x] * B^len + hash(a[l..r])，查哈希表得节点
时间复杂度：O((n + q) log n)
空间复杂度：O(n + m)
```

**C++ 代码**：

```cpp
const ull BASE = 131;
ull pathHash[MAXN]; // pathHash[u]: 根到 u 的路径选择序列哈希
ull pw[MAXN];
unordered_map<ull, int> hashToNode;

// 线段树维护序列 a 的区间哈希
struct SegTree {
    ull h[MAXN * 4];
    int sz[MAXN * 4];
    // build, update, query 省略
    ull query(int l, int r); // 返回 a[l..r] 的多项式哈希
} seg;

int query(int x, int l, int r) {
    // 二分找最远可达位置
    int lo = l, hi = r, res = l - 1;
    while (lo <= hi) {
        int mid = (lo + hi) / 2;
        ull target = pathHash[x] * pw[mid - l + 1] + seg.query(l, mid);
        if (hashToNode.count(target))
            res = mid, lo = mid + 1;
        else hi = mid - 1;
    }
    if (res < l) return x;
    ull finalHash = pathHash[x] * pw[res - l + 1] + seg.query(l, res);
    return hashToNode[finalHash];
}
```

**核心启示**：
- **路径编码为哈希**：将树上的路径选择序列视为字符串，哈希后建立双向映射
- 线段树维护序列哈希，支持 O(log n) 单点修改和区间查询
- 二分 + 哈希表查找 = O(log^2 n) 定位最远可达节点

### 题目 15：优秀的拆分（洛谷 P1117 / NOI2016）

> 给定字符串 s，求有多少个子串可以表示为 AABB 的形式（A、B 非空，AA 和 BB 分别是两个相同串的拼接）。

**分析**：预处理 f[i] = 以位置 i 结尾的 AA 形式子串数量，g[i] = 以位置 i 开头的 BB 形式子串数量。答案为 Σ f[i] * g[i+1]。求 f[i] 和 g[i] 可枚举半长 L，对每个 L 设置关键点 1, L+1, 2L+1, ...，用哈希 + 二分求相邻关键点的 LCP 和 LCS，确定以该半长形成 AA 的所有位置区间，差分标记。

```
算法思路：枚举半长 L → 关键点 LCP/LCS → 差分统计 f[]/g[]
关键步骤：LCP/LCS 用二分+哈希 O(log n)，差分标记 O(n/L)
时间复杂度：O(n log^2 n)（调和级数 × 二分）
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const ull BASE = 131;
ull h[MAXN], rh[MAXN], pw[MAXN]; // 正串和反串的前缀哈希
long long f[MAXN], g[MAXN]; // f[i]: 以i结尾的AA数, g[i]: 以i开头的BB数

ull getH(int l, int r)  { return h[r] - h[l - 1] * pw[r - l + 1]; }
ull getRH(int l, int r) { return rh[r] - rh[l - 1] * pw[r - l + 1]; }

int lcpAt(int i, int j, int n) {
    int lo = 0, hi = min(n - i + 1, n - j + 1), ans = 0;
    while (lo <= hi) {
        int mid = (lo + hi) / 2;
        if (mid == 0 || getH(i, i + mid - 1) == getH(j, j + mid - 1))
            ans = mid, lo = mid + 1;
        else hi = mid - 1;
    }
    return ans;
}

// lcs: 反串的 LCP
int lcsAt(int i, int j, int n) {
    int ri = n + 1 - i, rj = n + 1 - j;
    int lo = 0, hi = min(i, j), ans = 0;
    while (lo <= hi) {
        int mid = (lo + hi) / 2;
        if (mid == 0 || getRH(ri - mid + 1, ri) == getRH(rj - mid + 1, rj))
            ans = mid, lo = mid + 1;
        else hi = mid - 1;
    }
    return ans;
}

void solve(int n) {
    // 差分数组 df[], dg[]
    for (int L = 1; L <= n / 2; L++) {
        for (int i = 1; i + L <= n; i += L) {
            int j = i + L;
            int lcp = min(lcpAt(i, j, n), L);
            int lcs = min(lcsAt(i, j, n), L - 1);
            if (lcp + lcs < L) continue;
            // AA 结尾范围 [j + lcp - L, j - 1 + lcs] → 差分更新 f[]
            // BB 开头范围 [i - lcs, i + lcp - L] → 差分更新 g[]
        }
    }
    // 对 f[], g[] 做前缀和还原
    // ans = Σ f[i] * g[i+1]
}
```

**核心启示**：
- **关键点技巧**：每隔 L 设一个检查点，用 LCP+LCS 覆盖所有可能位置
- 调和级数保证总检查点数 O(n log n)，配合二分哈希总复杂度 O(n log^2 n)
- 差分标记是区间统计的利器，将位置级更新降为 O(1)

### 题目 16：Kefa and Watch（CF580E）

> 给定长度为 n 的数字串，支持两种操作：(1) 将区间 [l, r] 的所有字符替换为 c；(2) 查询子串 s[l..r] 是否以某个长度 d 为周期（即 s[l..r-d] == s[l+d..r]）。

**分析**：用线段树维护区间的多项式哈希值。区间赋值操作等价于将整段替换为 `ccc...c`，可以预处理 `c * (B^(len-1) + B^(len-2) + ... + 1)` 的值作为懒标记。查询周期 d 时，比较 `hash(l, r-d)` 和 `hash(l+d, r)` 是否相等。

```
算法思路：线段树维护区间哈希 + 懒标记支持区间赋值
关键步骤：合并时 h = h_left * B^right_len + h_right；赋值时 h = c * 等比数列和
时间复杂度：O(n + q log n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const ull BASE = 131, MOD = 1e9 + 7;
ull pw[MAXN], sumPw[MAXN]; // sumPw[k] = B^0 + B^1 + ... + B^(k-1)

struct Node {
    ull h; int len; int lazy; // lazy = -1 表示无操作
};
Node tree[MAXN * 4];

void pushUp(int u) {
    tree[u].h = (tree[u * 2].h * pw[tree[u * 2 + 1].len] + tree[u * 2 + 1].h) % MOD;
}

void assign(int u, int c) {
    tree[u].h = (ull)c * sumPw[tree[u].len] % MOD;
    tree[u].lazy = c;
}

// pushDown, build, update(l, r, c), query(l, r) 省略

bool checkPeriod(int l, int r, int d) {
    // 查 hash(l, r-d) == hash(l+d, r)
    ull h1 = query(l, r - d);
    ull h2 = query(l + d, r);
    return h1 == h2;
}
```

**核心启示**：
- **线段树 + 哈希**：将哈希的"可拼接性"与线段树的区间操作完美结合
- 区间赋值的哈希值 = 字符值 × 等比数列求和，可 O(1) 计算
- 周期判断转化为两段子串哈希比较，是哈希最优雅的应用之一

## 五、常见技巧与思路总结

### 5.1 双哈希防碰撞

在 Codeforces 等平台，出题者常构造数据 hack 单哈希。**双哈希**（两组独立的 base/mod）几乎不可能碰撞：

```cpp
// 双哈希模板核心
pair<ull, ull> get(int l, int r) {
    return {getH1(l, r), getH2(l, r)};
}
```

碰撞概率：单哈希 ≈ 1/M ≈ 10^{-9}，双哈希 ≈ 1/(M1×M2) ≈ 10^{-18}。

### 5.2 自然溢出的风险

`unsigned long long` 自然溢出等价于模 2^64，但 2^64 不是质数，因此存在系统性弱点（如 Thue-Morse 序列可构造大量碰撞）。比赛中若不确定数据强度，**务必用双哈希**。

### 5.3 随机 base 防 hack

```cpp
mt19937 rng(chrono::steady_clock::now().time_since_epoch().count());
ull BASE = rng() % 900000000 + 100000000; // 随机选取 base
```

随机 base 使对手无法预知哈希函数，进一步提升安全性。

### 5.4 频次哈希（多重集哈希）

当比较的不是"字符串相等"而是"字符频次相同"（即一个是另一个的排列）时：

```cpp
// 为每个字符赋随机权值，区间和 = 频次哈希
ull w[26]; // 随机初始化
ull sum[MAXN]; // 前缀和
ull getFreqHash(int l, int r) { return sum[r] - sum[l - 1]; }
```

### 5.5 二分 + 哈希的万能搭配

当问题包含"最长"、"最大"、"最少次数"等**单调性**关键词时，考虑二分答案 + 哈希验证：

| 问题模式 | 二分对象 | 哈希验证 |
|---------|---------|---------|
| 最长重复子串 | 子串长度 | 是否有重复哈希 |
| 最长公共前缀 | 前缀长度 | 两段哈希是否相等 |
| k 次失配匹配 | 每段匹配长度 | LCP 判断 |
| 最长回文子串 | 半径 | 正反串哈希比较 |

### 5.6 哈希值的增量更新

单字符修改后哈希的快速更新：

```
new_hash = old_hash + (new_char - old_char) * B^pos
```

追加字符：`h = h * B + new_char`
删除首字符：`h = h - first_char * B^(len-1)`

## 七、终极总结与归纳

### 7.1 本质

字符串哈希的本质是**将字符串问题降维为数值问题**——通过多项式映射，将 O(n) 的逐字符比较压缩为 O(1) 的整数比较，以极小的碰撞代价换取巨大的效率提升。

### 7.2 核心要点

1. **多项式哈希公式**：`H(s) = Σ s[i] * B^(n-1-i) mod M`，理解"进制表示"的本质
2. **前缀哈希 + 区间提取**：`get(l, r) = h[r] - h[l-1] * B^(r-l+1)`，O(1) 子串哈希
3. **双哈希防碰撞**：两组独立 (B, M)，碰撞概率降至 10^{-18}
4. **二分 + 哈希**：求 LCP / 最长重复子串 / k 次失配的万能组合
5. **频次哈希**：随机赋权求和，解决"排列等价"和"多重集相等"问题
6. **数据结构 + 哈希**：线段树/树状数组维护哈希，支持区间修改与查询
