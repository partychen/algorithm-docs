## 一、什么是 Trie 树

**Trie 树**（又称"字典树"或"前缀树"，读作 /traɪ/）是一种专门用来**高效存储和检索字符串集合**的树形数据结构。它的核心思想是：将具有公共前缀的字符串共享相同的路径前缀，从而节省空间并加速查找。

假设我们有单词集合 `{cat, car, card, do, dog}`，其 Trie 树结构如下：

```
        (root)
       /      \
      c        d
      |        |
      a        o
     / \       |  \
    t   r      g   (end)
        |
        d
```

每条从根到叶（或标记节点）的路径代表一个完整的字符串。查找、插入一个长度为 L 的字符串，时间复杂度仅为 **O(L)**，与集合总大小无关。

### Trie vs 其他字符串数据结构

| 维度 | 哈希表 | 平衡BST | Trie | AC 自动机 |
|------|--------|---------|------|-----------|
| 单次查找 | O(L) 均摊 | O(L log n) | O(L) | O(L) |
| 前缀查询 | 不支持 | 不支持 | ✅ 天然支持 | ✅ |
| 空间 | O(总字符) | O(总字符) | O(总字符·Σ) | O(总字符·Σ) |
| 支持多模匹配 | ❌ | ❌ | 单模 | ✅ 多模 |
| 实现难度 | 低 | 中 | 低 | 中高 |

## 二、核心思想与前置知识

### 2.1 Trie 的基本结构

Trie 的每个节点代表一个**字符**（严格说是一条边上的字符），从根到某节点的路径拼接起来就是一个前缀。关键要素：

- `ch[u][c]`：节点 u 沿字符 c 走到的子节点编号（0 表示不存在）
- `cnt[u]` 或 `end[u]`：节点 u 是否为某个字符串的结尾（或以此为结尾的字符串个数）
- `sz[u]`：经过节点 u 的字符串个数（用于前缀计数）

### 2.2 Trie 的三大基本操作

```cpp
const int MAXN = 500005;
int ch[MAXN][26], cnt[MAXN], sz[MAXN], tot;

void insert(const char* s) {
    int u = 0;
    for (int i = 0; s[i]; i++) {
        int c = s[i] - 'a';
        if (!ch[u][c]) ch[u][c] = ++tot;
        u = ch[u][c];
        sz[u]++;  // 经过此节点的字符串数
    }
    cnt[u]++;     // 以此结尾的字符串数
}

bool search(const char* s) {
    int u = 0;
    for (int i = 0; s[i]; i++) {
        int c = s[i] - 'a';
        if (!ch[u][c]) return false;
        u = ch[u][c];
    }
    return cnt[u] > 0;
}

int prefixCount(const char* s) {
    int u = 0;
    for (int i = 0; s[i]; i++) {
        int c = s[i] - 'a';
        if (!ch[u][c]) return 0;
        u = ch[u][c];
    }
    return sz[u]; // 以 s 为前缀的字符串数
}
```

### 2.3 复杂度分析

- **时间复杂度**：插入 / 查找 / 前缀查询均为 O(L)，L 为字符串长度
- **空间复杂度**：O(总字符数 × 字符集大小 Σ)。若 Σ=26，每个节点需 26 个指针槽位
- **优化方向**：用链表/map 代替数组可降低空间；01-Trie 中 Σ=2 空间非常紧凑

## 三、分类与变体详解

### 3.1 字符 Trie（标准字典树）

最基础的形态，字符集通常是小写字母（Σ=26）或大小写+数字（Σ=62）。用于字符串存储、查找、前缀统计。

```
数据结构：数组模拟多叉树，ch[MAXN][Σ]
支持操作：插入、查找、前缀计数、删除
关键性质：公共前缀共享路径
时间复杂度：O(L) 每次操作
```

```cpp
// 支持大小写+数字的字符映射（Σ=62）
int getIdx(char c) {
    if (c >= 'A' && c <= 'Z') return c - 'A';
    if (c >= 'a' && c <= 'z') return c - 'a' + 26;
    return c - '0' + 52;
}
```

### 3.2 01-Trie（二进制字典树）

每个数按**二进制位**从高位到低位插入，每层仅两个分支（0/1）。核心应用：**最大异或值查询**——贪心地优先走与当前位相反的分支。

```
数据结构：二叉 Trie，ch[MAXN][2]
支持操作：插入、删除、查询最大异或值
关键性质：高位优先贪心保证异或最大
时间复杂度：O(log V) 每次操作，V 为值域
```

```cpp
const int BIT = 30;
int ch[MAXN * 32][2], num[MAXN * 32], tot;

void insert(int x) {
    int u = 0;
    for (int i = BIT; i >= 0; i--) {
        int b = (x >> i) & 1;
        if (!ch[u][b]) ch[u][b] = ++tot;
        u = ch[u][b];
        num[u]++;
    }
}

int queryMax(int x) {
    int u = 0, res = 0;
    for (int i = BIT; i >= 0; i--) {
        int b = (x >> i) & 1;
        if (ch[u][b ^ 1] && num[ch[u][b ^ 1]]) {
            res |= (1 << i);
            u = ch[u][b ^ 1];
        } else {
            u = ch[u][b];
        }
    }
    return res;
}
```

### 3.3 可持久化 Trie

类似主席树（可持久化线段树），每次插入时只新建 O(log V) 个节点，共享历史版本的其余部分。支持**区间异或最大值查询**：通过版本号差分，在 `[root[l], root[r]]` 之间查询。

```
数据结构：可持久化 01-Trie，按版本号差分
支持操作：插入新版本、区间查询最大异或
关键性质：版本差分 = 区间信息，节点共享历史
时间复杂度：O(log V) 每次操作
```

```cpp
int rt[MAXN]; // 每个版本的根

void insert(int prev, int cur, int x) {
    for (int i = BIT; i >= 0; i--) {
        int b = (x >> i) & 1;
        ch[cur][b ^ 1] = ch[prev][b ^ 1]; // 共享另一分支
        ch[cur][b] = ++tot;
        cur = ch[cur][b]; prev = ch[prev][b];
        sum[cur] = sum[prev] + 1;
    }
}

int query(int u, int v, int x) {
    int res = 0;
    for (int i = BIT; i >= 0; i--) {
        int b = (x >> i) & 1;
        if (sum[ch[v][b ^ 1]] - sum[ch[u][b ^ 1]] > 0) {
            res |= (1 << i);
            u = ch[u][b ^ 1]; v = ch[v][b ^ 1];
        } else {
            u = ch[u][b]; v = ch[v][b];
        }
    }
    return res;
}
```

### 3.4 Trie 图（AC 自动机中的 Trie）

在字符 Trie 上补全所有不存在的边（类似 KMP 的 fail 指针），使得从任何节点沿任何字符都能转移，形成一个有向图。这是 AC 自动机的基础结构，用于多模式串匹配。

```
数据结构：Trie + fail 指针 + 补全转移
支持操作：多模式串匹配、构建 fail 树
关键性质：fail 指针指向最长后缀匹配节点
时间复杂度：构建 O(总字符 × Σ)，匹配 O(文本长)
```

### 变体对比总表

| 变体 | 字符集 | 核心应用 | 空间 | 典型题型 |
|------|--------|---------|------|---------|
| 字符 Trie | Σ=26/62 | 字符串查找/前缀统计 | O(NΣ) | 模板、点名、阅读理解 |
| 01-Trie | Σ=2 | 最大异或值 | O(N·logV) | 异或路径、Multiset |
| 可持久化 Trie | Σ=2 | 区间异或最大值 | O(N·logV) | 最大异或和、异或粽子 |
| Trie 图 | Σ=26 | 多模匹配、DP on Trie | O(NΣ) | AC 自动机系列 |

### 题目 1：【模板】字典树（洛谷 P8306）

> 给定 n 个模式串和 q 个查询串，对每个查询串 t，求有多少个模式串以 t 为前缀。字符含大小写字母和数字（Σ=62）。多组数据。

**分析**：标准 Trie 模板。插入时经过每个节点计数 +1，查询时沿路径走到底，返回该节点的计数值。

```
数据结构：字符 Trie，ch[N][62]，经过计数 sz[u]
支持操作：insert 插入模式串，query 查询前缀计数
关键性质：sz[u] = 经过节点 u 的字符串数 = 以对应前缀开头的模式串数
时间复杂度：O(Σ|s_i| + Σ|t_i|)
```

**C++ 代码**：

```cpp
const int N = 3000005, S = 62;
int ch[N][S], sz[N], tot;

int getId(char c) {
    if (c >= 'A' && c <= 'Z') return c - 'A';
    if (c >= 'a' && c <= 'z') return c - 'a' + 26;
    return c - '0' + 52;
}

void insert(const char* s) {
    int u = 0;
    for (int i = 0; s[i]; i++) {
        int c = getId(s[i]);
        if (!ch[u][c]) ch[u][c] = ++tot;
        u = ch[u][c];
        sz[u]++;
    }
}

int query(const char* s) {
    int u = 0;
    for (int i = 0; s[i]; i++) {
        int c = getId(s[i]);
        if (!ch[u][c]) return 0;
        u = ch[u][c];
    }
    return sz[u];
}
// 多组数据时需循环清空 ch 和 sz（仅清到 tot 范围）
```

**核心启示**：
- **经过计数 vs 结尾计数**：sz[u] 统计经过节点 u 的串数，天然就是"以该前缀开头"的串数
- **大字符集映射**：Σ=62 时需统一映射函数，避免数组越界
- **多组数据清空**：只清 `[0, tot]` 范围，避免 memset 超时

### 题目 2：于是他错误的点名开始了（洛谷 P2580）

> 给定 n 个学生姓名。依次点名 m 次，每次输出：若在名单中且首次点到输出 OK，已点过输出 REPEAT，不在名单输出 WRONG。

**分析**：Trie 存储名单，结尾节点记录状态标记（0=不存在，1=存在未点，2=已点过）。

```
数据结构：字符 Trie，ch[N][26]，结尾状态 flag[u]
支持操作：insert 建名单，query 点名并更新状态
关键性质：flag 三态标记：0/1/2 → WRONG/OK/REPEAT
时间复杂度：O(Σ|s_i| + Σ|t_i|)
```

**C++ 代码**：

```cpp
const int N = 500005;
int ch[N][26], flag[N], tot;

void insert(const char* s) {
    int u = 0;
    for (int i = 0; s[i]; i++) {
        int c = s[i] - 'a';
        if (!ch[u][c]) ch[u][c] = ++tot;
        u = ch[u][c];
    }
    flag[u] = 1; // 标记为存在
}

int query(const char* s) {
    int u = 0;
    for (int i = 0; s[i]; i++) {
        int c = s[i] - 'a';
        if (!ch[u][c]) return 0; // WRONG
        u = ch[u][c];
    }
    if (flag[u] == 0) return 0;      // WRONG
    if (flag[u] == 1) { flag[u] = 2; return 1; } // OK
    return 2;                          // REPEAT
}
```

**核心启示**：
- **状态标记法**：在 Trie 结尾节点附加多态信息，可扩展到各种"查找+修改"场景
- **Trie 优于 map/set**：当字符串数量大、需频繁查找时，Trie 常数更小

### 题目 3：[TJOI2010] 阅读理解（洛谷 P3879）

> 给定 N 篇文章，每篇由若干单词组成。M 次查询，每次给一个单词，输出它在哪些文章中出现过（升序编号）。

**分析**：将所有单词插入 Trie，每个结尾节点用 bitset 记录出现过的文章编号集合。查询时走到结尾节点输出 bitset。

```
数据结构：字符 Trie + bitset<N> appear[u]
支持操作：insert(word, docId)，query(word)
关键性质：bitset 天然支持集合并操作，空间 O(N/64) 每节点
时间复杂度：O(Σ|word| + M·N/64)
```

**C++ 代码**：

```cpp
const int MAXN = 500005, MAXD = 1005;
int ch[MAXN][26], tot;
bitset<MAXD> appear[MAXN];

void insert(const char* s, int docId) {
    int u = 0;
    for (int i = 0; s[i]; i++) {
        int c = s[i] - 'a';
        if (!ch[u][c]) ch[u][c] = ++tot;
        u = ch[u][c];
    }
    appear[u].set(docId);
}

bitset<MAXD> query(const char* s) {
    int u = 0;
    for (int i = 0; s[i]; i++) {
        int c = s[i] - 'a';
        if (!ch[u][c]) return bitset<MAXD>();
        u = ch[u][c];
    }
    return appear[u];
}
```

**核心启示**：
- **Trie 节点附加信息**：结尾节点不仅能存 bool/int，还能存 bitset、vector 等复杂信息
- **bitset 压缩**：N 篇文章用 bitset<N> 存储，空间和遍历效率都优于 vector

### 题目 4：[USACO08DEC] Secret Message（洛谷 P2922）

> M 条二进制信息插入 Trie，N 次查询。对每条查询暗号，统计与它前缀匹配的信息数。匹配长度为两者长度的较小值。

**分析**：将信息插入 01-Trie，每个节点记录经过数 sz 和结尾数 ed。查询时沿暗号路径累加路径上的 ed（信息是暗号的前缀），最后加上终点的 sz - ed（暗号是信息的前缀）。

```
数据结构：01-Trie（二进制串），sz[u] 经过计数，ed[u] 结尾计数
支持操作：insert 插入信息，query 统计匹配数
关键性质：双向前缀关系需分别统计——路径上的 ed（短匹配长）和终点的 sz（长匹配短）
时间复杂度：O(Σ|s_i| + Σ|t_i|)
```

**C++ 代码**：

```cpp
const int N = 500005;
int ch[N][2], sz[N], ed[N], tot;

void insert(int* bits, int len) {
    int u = 0;
    for (int i = 0; i < len; i++) {
        int b = bits[i];
        if (!ch[u][b]) ch[u][b] = ++tot;
        u = ch[u][b];
        sz[u]++;
    }
    ed[u]++;
}

int query(int* bits, int len) {
    int u = 0, res = 0;
    for (int i = 0; i < len; i++) {
        int b = bits[i];
        if (!ch[u][b]) return res;
        u = ch[u][b];
        res += ed[u]; // 信息是暗号的前缀
    }
    res += sz[u] - ed[u]; // 暗号是信息的前缀（不重复计 ed）
    return res;
}
```

**核心启示**：
- **双向前缀统计**：路径上累加 ed（短串完整匹配）+ 终点 sz-ed（长串包含查询串）
- **sz 和 ed 的配合**：两个计数器解决"谁是谁的前缀"的对称问题

### 题目 5：[HNOI2004] L 语言（洛谷 P2292）

> 给定字典 D 含 n 个单词（长度 ≤ 20），m 个无标点文本 T。对每个文本，求能从头拆分为字典单词的最长前缀长度。

**分析**：将字典单词插入 Trie。对文本做 DP：`dp[i]` 表示前 i 个字符能否被完全拆分。枚举每个 dp[j]=true 的位置 j，从 j+1 开始沿 Trie 匹配，遇到单词结尾则 dp[k]=true。答案为最大的 dp[i]=true 的 i。

```
数据结构：字符 Trie + DP 数组 dp[|T|]
支持操作：insert 建字典，DP 扫描文本
关键性质：字典单词长度 ≤ 20，每个位置最多沿 Trie 走 20 步
时间复杂度：O(|T| × max_word_len)
```

**C++ 代码**：

```cpp
const int N = 200005;
int ch[N][26], ed[N], tot;
bool dp[N];

void insert(const char* s) {
    int u = 0;
    for (int i = 0; s[i]; i++) {
        int c = s[i] - 'a';
        if (!ch[u][c]) ch[u][c] = ++tot;
        u = ch[u][c];
    }
    ed[u] = 1;
}

int solve(const char* t, int len) {
    for (int i = 0; i <= len; i++) dp[i] = false;
    dp[0] = true;
    int ans = 0;
    for (int i = 0; i < len; i++) {
        if (!dp[i]) continue;
        int u = 0;
        for (int j = i; j < len; j++) {
            int c = t[j] - 'a';
            if (!ch[u][c]) break;
            u = ch[u][c];
            if (ed[u]) { dp[j + 1] = true; ans = max(ans, j + 1); }
        }
    }
    return ans;
}
```

**核心启示**：
- **Trie + DP 经典组合**：Trie 加速多模式匹配，DP 处理可拆分性
- **单词长度上界优化**：字典单词长度有限（≤20），内层循环最多 20 步，总复杂度可控

### 题目 6：最长异或路径（洛谷 P4551）

> 给定 n 个点的带权树，求两点间路径边权异或和的最大值。

**分析**：令 d[u] = 根到 u 的边权异或和，则 u→v 路径异或 = d[u] ⊕ d[v]（LCA 部分自动抵消）。问题转化为：在 d 数组中找两数异或最大值。用 01-Trie 贪心即可。

```
数据结构：01-Trie，按 30 位二进制插入
支持操作：DFS 求 d[]，insert 全部 d[u]，queryMax 查最大异或
关键性质：树上路径异或 = 前缀异或差；贪心优先走反向位
时间复杂度：O(n log V)
```

**C++ 代码**：

```cpp
const int N = 100005, BIT = 30;
int ch[N * 32][2], tot;
int d[N]; // d[u] = root 到 u 的异或和
struct Edge { int to, w; };
vector<Edge> adj[N];

void dfs(int u, int fa) {
    for (auto& e : adj[u]) if (e.to != fa) {
        d[e.to] = d[u] ^ e.w;
        dfs(e.to, u);
    }
}

void insert(int x) {
    int u = 0;
    for (int i = BIT; i >= 0; i--) {
        int b = (x >> i) & 1;
        if (!ch[u][b]) ch[u][b] = ++tot;
        u = ch[u][b];
    }
}

int queryMax(int x) {
    int u = 0, res = 0;
    for (int i = BIT; i >= 0; i--) {
        int b = (x >> i) & 1;
        if (ch[u][b ^ 1]) { res |= (1 << i); u = ch[u][b ^ 1]; }
        else u = ch[u][b];
    }
    return res;
}
// dfs(1,0); 插入所有 d[u]; 对每个 d[u] 查询 queryMax 取最大值
```

**核心启示**：
- **树上异或路径经典转化**：d[u]⊕d[v] 利用异或自消性，LCA 以上部分自动抵消
- **01-Trie 贪心**：从高位到低位，优先走与当前位相反的方向

### 题目 7：Vasiliy's Multiset（CF706D）

> 维护一个多重集 A（初始含 0），支持：+ x 插入，- x 删除，? x 查询 A 中与 x 异或最大值。

**分析**：01-Trie 支持带计数的插入和删除。每个节点维护 num[u] 表示子树中有效数字个数。查询时只走 num > 0 的分支。

```
数据结构：01-Trie，num[u] 记录子树有效元素数
支持操作：insert (+1)，erase (-1)，queryMax 贪心最大异或
关键性质：num 支持删除操作，确保只走有效分支
时间复杂度：O(q · 30)
```

**C++ 代码**：

```cpp
const int N = 6000005, BIT = 29;
int ch[N][2], num[N], tot;

void update(int x, int v) { // v=+1 插入, v=-1 删除
    int u = 0;
    for (int i = BIT; i >= 0; i--) {
        int b = (x >> i) & 1;
        if (!ch[u][b]) ch[u][b] = ++tot;
        u = ch[u][b];
        num[u] += v;
    }
}

int queryMax(int x) {
    int u = 0, res = 0;
    for (int i = BIT; i >= 0; i--) {
        int b = (x >> i) & 1;
        if (ch[u][b ^ 1] && num[ch[u][b ^ 1]] > 0) {
            res |= (1 << i); u = ch[u][b ^ 1];
        } else {
            u = ch[u][b];
        }
    }
    return res;
}
// 初始 update(0, 1) 插入 0
```

**核心启示**：
- **计数式 Trie 支持删除**：num[u] 记录子树有效元素数，删除只需 -1
- **初始插入 0**：保证集合非空，避免查询异常

### 题目 8：Sausage Maximization（CF282E）

> 长度 n 的数组，选不重叠的前缀和后缀，使两部分异或和的异或最大。前后缀均可为空。

**分析**：令 pre[i] 为前 i 个元素异或和，suf[i] 为后 n-i+1 个元素异或和。将所有 pre[0..n] 插入 01-Trie，从后往前枚举 suf[i]，在 Trie 中查询与 suf[i] 异或最大值。但需保证不重叠：先全部插入 pre，再枚举 suf 时不会冲突（因为 pre[i] 和 suf[i+1] 不重叠）。

```
数据结构：01-Trie，按 63 位二进制插入（值域可达 10^18）
支持操作：插入所有前缀异或和，枚举后缀异或和查询最大
关键性质：前缀异或 pre[i] 与后缀异或 suf[i+1] 天然不重叠
时间复杂度：O(n · 63)
```

**C++ 代码**：

```cpp
const int N = 100005, BIT = 62;
int ch[N * 64][2], tot;
typedef long long ll;

void insert(ll x) {
    int u = 0;
    for (int i = BIT; i >= 0; i--) {
        int b = (x >> i) & 1;
        if (!ch[u][b]) ch[u][b] = ++tot;
        u = ch[u][b];
    }
}

ll queryMax(ll x) {
    int u = 0;
    ll res = 0;
    for (int i = BIT; i >= 0; i--) {
        int b = (x >> i) & 1;
        if (ch[u][b ^ 1]) { res |= (1LL << i); u = ch[u][b ^ 1]; }
        else u = ch[u][b];
    }
    return res;
}
// 插入 pre[0..n]，从后往前枚举 suf[n..0] 查询最大值
```

**核心启示**：
- **前后缀分离策略**：前缀全部插入后枚举后缀，天然保证不重叠
- **注意位宽**：值域大时需用 long long 和 63 位

### 题目 9：[JSOI2009] 电子字典（洛谷 P4407）

> 字典含 n 个单词。m 次查询，若查询串在字典中输出 -1，否则输出与之编辑距离恰好为 1 的字典单词数。编辑操作：增/删/改一个字符。

**分析**：在 Trie 上 DFS 搜索。维护一个 "已使用编辑操作" 标记 used。未使用时，可尝试三种操作分支（跳过查询字符=删、跳 Trie 子节点=增、走不同字符=改）。used=1 后只能精确匹配。用 visited 标记去重。

```
数据结构：字符 Trie + DFS 枚举编辑操作
支持操作：insert 建字典，dfs(u, pos, used) 搜索编辑距离 1
关键性质：编辑距离 1 只需一次操作 → DFS 分支受限，复杂度可控
时间复杂度：O(m × L × 26)，L 为单词最大长度
```

**C++ 代码**：

```cpp
const int N = 200005;
int ch[N][26], ed[N], tot, ans;
bool vis[N]; // 去重标记
char q[25]; int qlen;

void dfs(int u, int pos, int used) {
    if (pos == qlen) {
        if (ed[u] && used == 1 && !vis[u]) { vis[u] = true; ans++; }
        if (!used) // 插入一个字符后结束
            for (int c = 0; c < 26; c++)
                if (ch[u][c] && ed[ch[u][c]] && !vis[ch[u][c]])
                    { vis[ch[u][c]] = true; ans++; }
        return;
    }
    int c = q[pos] - 'a';
    if (ch[u][c]) dfs(ch[u][c], pos + 1, used); // 精确匹配
    if (!used) {
        dfs(u, pos + 1, 1); // 删除 q[pos]
        for (int i = 0; i < 26; i++) if (ch[u][i]) {
            dfs(ch[u][i], pos, 1);                     // 插入 i
            if (i != c) dfs(ch[u][i], pos + 1, 1);     // 替换为 i
        }
    }
}
// 每次查询前 ans=0, 清空 vis; 若 search(q) 则输出 -1
```

**核心启示**：
- **Trie + DFS 枚举编辑操作**：在树上模拟增/删/改三种操作，分支因子受单词长度和字符集大小限制
- **去重必不可少**：不同编辑路径可能到达同一单词，需 vis 标记

### 题目 10：[USACO12DEC] First!（洛谷 P3065）

> 给定 n 个不同字符串，可以重新定义 26 个字母的排序规则。问哪些字符串可能在某种字母序下成为字典序最小的。

**分析**：将所有串插入 Trie。对每个候选串 s，逐层沿 Trie 走，每层要求 s 的字符优先于同层所有兄弟节点的字符。对这些"优先于"关系建有向图（26 个点），若无环（拓扑排序成功）则 s 可行。若路径上经过某个更短串的结尾，则 s 不可行。

```
数据结构：字符 Trie + 字母关系有向图 + 拓扑排序
支持操作：insert 建 Trie，对每个串检查字母偏序是否有环
关键性质：同层兄弟建边 s[i] → sibling，无环则该字母序合法
时间复杂度：O(n × Σ|s_i| × 26)
```

**C++ 代码**：

```cpp
const int N = 300005;
int ch[N][26], ed[N], tot;
bool e[26][26]; int in[26];

bool topoCheck() {
    int cnt = 0;
    queue<int> q;
    for (int i = 0; i < 26; i++) if (!in[i]) q.push(i);
    while (!q.empty()) {
        int u = q.front(); q.pop(); cnt++;
        for (int v = 0; v < 26; v++)
            if (e[u][v] && --in[v] == 0) q.push(v);
    }
    return cnt == 26;
}

bool canBeFirst(const char* s, int len) {
    memset(e, 0, sizeof e);
    memset(in, 0, sizeof in);
    int u = 0;
    for (int i = 0; i < len; i++) {
        if (ed[u]) return false; // 更短的串已在此结尾
        int c = s[i] - 'a';
        for (int j = 0; j < 26; j++)
            if (j != c && ch[u][j] && !e[c][j])
                { e[c][j] = true; in[j]++; }
        u = ch[u][c];
    }
    return topoCheck();
}
```

**核心启示**：
- **Trie + 拓扑排序**：字典序问题 → 字母间偏序关系 → 有向图判环
- **前缀排斥**：若 s 路径上遇到更短串的结尾，s 一定不可能最小

### 题目 11：[SCOI2016] 背单词（洛谷 P3294）

> n 个单词排列，按规则计算花费。若某单词有后缀单词未背则花 n²；无后缀单词花当前位置 x；后缀单词都已背花 x-y（y 为最近后缀位置）。求最小花费。

**分析**：将所有串反转后插入 Trie，后缀关系变为前缀关系。在 Trie 上只保留代表输入单词的节点，重建树。贪心：DFS 时优先处理子树大小小的子树（打水问题），使每对父子编号差最小化。

```
数据结构：反转后的字符 Trie → 重建树 → 贪心 DFS 分配编号
支持操作：insert 反转串，重建树保留单词节点，子树大小排序 + DFS
关键性质：后缀关系反转为前缀 → Trie 上父子关系；子树小的先处理最优
时间复杂度：O(Σ|s_i| log Σ|s_i|)
```

**C++ 代码**：

```cpp
const int N = 510005;
int ch[N][26], ed[N], tot;
vector<int> children[N]; // 重建树
int sz[N], order[N], idx;
long long ans;

void rebuild(int u, int fa) {
    // fa 是最近的"单词祖先"
    if (ed[u]) { children[fa].push_back(u); fa = u; }
    for (int c = 0; c < 26; c++)
        if (ch[u][c]) rebuild(ch[u][c], fa);
}

void calcSize(int u) {
    sz[u] = 1;
    for (int v : children[u]) { calcSize(v); sz[u] += sz[v]; }
}

void assign(int u) {
    order[u] = ++idx;
    sort(children[u].begin(), children[u].end(),
         [](int a, int b) { return sz[a] < sz[b]; });
    for (int v : children[u]) {
        ans += idx + 1 - order[u]; // 子节点编号 - 父节点编号
        assign(v);
    }
}
// rebuild(0,0); calcSize(0); idx=0; assign(0);
```

**核心启示**：
- **后缀问题反转为前缀**：反转字符串后插入 Trie，利用 Trie 天然的前缀父子关系
- **打水问题贪心**：子树小的先安排，使编号差之和最小

### 题目 12：[IOI2008] Type Printer（洛谷 P4683）

> 打印机支持三种操作：尾部加字母、尾部删字母、打印当前串。用最少操作次数以任意顺序打印所有 n 个单词。初始为空串，结束时可留任意状态。

**分析**：将所有单词插入 Trie。在 Trie 上 DFS 等价于"加字母→递归→删字母"。最后访问的路径不需要回溯删除，因此应将最长单词的路径留到最后。贪心：找最长串并标记路径，DFS 时最后走该标记路径。

```
数据结构：字符 Trie + DFS + 贪心标记最长路径
支持操作：insert 建树，标记最长串路径，DFS 输出操作序列
关键性质：最后走的路径不回溯 → 最长路径最后走省最多操作
时间复杂度：O(Σ|s_i|)
```

**C++ 代码**：

```cpp
const int N = 500005;
int ch[N][26], ed[N], tot;
bool onLongest[N]; // 最长串路径标记
vector<char> ops;

void markLongest(const char* s) {
    int u = 0;
    for (int i = 0; s[i]; i++) {
        u = ch[u][s[i] - 'a'];
        onLongest[u] = true;
    }
}

void dfs(int u) {
    if (ed[u]) ops.push_back('P');
    int last = -1;
    for (int c = 0; c < 26; c++) if (ch[u][c]) {
        if (onLongest[ch[u][c]]) { last = c; continue; }
        ops.push_back('a' + c);
        dfs(ch[u][c]);
        ops.push_back('-'); // 回溯
    }
    if (last >= 0) { // 最后走最长路径
        ops.push_back('a' + last);
        dfs(ch[u][last]);
        if (u != 0) ops.push_back('-');
    }
}
// markLongest(longest_word); dfs(0);
```

**核心启示**：
- **DFS 即操作序列**：Trie 上的 DFS 自然对应"添加→打印→回溯"的操作
- **最后走最长路径**：省去最后一段回溯，操作数最优

### 题目 13：最大异或和（洛谷 P4735）

> 序列支持末尾追加元素，查询 `max{a[p]⊕...⊕a[N]⊕x}`，p∈[l,r]。

**分析**：维护前缀异或和 s[i]。查询转化为：在 s[l-1..r-1] 中找 s[p-1] 使 s[p-1]⊕(s[N]⊕x) 最大。用可持久化 Trie 支持区间查询。每个版本的 Trie 对应插入 s[0..i]，通过版本差分实现区间限制。

```
数据结构：可持久化 01-Trie，版本差分
支持操作：insert 新版本，query(root[l-1], root[r], val) 区间最大异或
关键性质：sum[v]-sum[u] > 0 表示区间 [l,r] 内有数经过该分支
时间复杂度：O((N+M) · 24)
```

**C++ 代码**：

```cpp
const int N = 600005, B = 24;
int ch[N * 25][2], sum[N * 25], rt[N], tot;

void insert(int prev, int cur, int x) {
    for (int i = B; i >= 0; i--) {
        int b = (x >> i) & 1;
        ch[cur][b ^ 1] = ch[prev][b ^ 1];
        ch[cur][b] = ++tot;
        cur = ch[cur][b]; prev = ch[prev][b];
        sum[cur] = sum[prev] + 1;
    }
}

int query(int u, int v, int x) {
    int res = 0;
    for (int i = B; i >= 0; i--) {
        int b = (x >> i) & 1;
        if (sum[ch[v][b ^ 1]] - sum[ch[u][b ^ 1]] > 0) {
            res |= (1 << i); u = ch[u][b ^ 1]; v = ch[v][b ^ 1];
        } else {
            u = ch[u][b]; v = ch[v][b];
        }
    }
    return res;
}
// 对每个 s[i] 建新版本：rt[i+1]=++tot; insert(rt[i],rt[i+1],s[i]);
// 查询 query(rt[l-1], rt[r], s[N]^x)
```

**核心启示**：
- **可持久化 Trie 模板**：类似主席树，版本差分实现区间查询
- **前缀异或和 + 可持久化**：动态序列上的区间异或最大值的标准做法

### 题目 14：[十二省联考2019] 异或粽子（洛谷 P5283）

> 长 n 的数组，选 k 个不同区间，使区间异或和之和最大。

**分析**：前缀异或和 s[i]，区间 [l,r] 异或 = s[r]⊕s[l-1]。问题转化为在 n+1 个 s 值中找 k 个最大异或对之和。用可持久化 Trie 支持"查第 t 大异或值"，配合大根堆逐个取最大。因对称性 (i,j) 和 (j,i) 重复，取 2k 次后除以 2。

```
数据结构：可持久化 01-Trie + 大根堆
支持操作：insert 建版本，kthMax 查第 k 大异或，堆维护全局前 k 大
关键性质：对称去重取 2k 次除 2；堆中维护 (异或值, 位置, 第几大)
时间复杂度：O(n log V + k log n log V)
```

**C++ 代码**：

```cpp
// 在可持久化 Trie 上查询与 x 异或的第 k 大
int kthMax(int u, int v, int x, int k) {
    int res = 0;
    for (int i = B; i >= 0; i--) {
        int b = (x >> i) & 1;
        int cnt = sum[ch[v][b ^ 1]] - sum[ch[u][b ^ 1]];
        if (k <= cnt) {
            res |= (1 << i);
            u = ch[u][b ^ 1]; v = ch[v][b ^ 1];
        } else {
            k -= cnt;
            u = ch[u][b]; v = ch[v][b];
        }
    }
    return res;
}

// 堆节点: {xor_val, index, rank}
// 初始: 每个 s[i] 的第 1 大异或值入堆
// 每次取堆顶加入答案，再将该位置的第 rank+1 大入堆
// 取 2k 次，答案除 2
```

**核心启示**：
- **第 k 大异或查询**：可持久化 Trie 中按分支计数选择走向，类似主席树求第 k 大
- **对称性去重**：(i,j) 与 (j,i) 异或值相同，取 2k 次除 2 避免复杂去重

### 题目 15：[Ynoi2010] Fusion tree（洛谷 P6018）

> 树上每个点有权值。操作：1) 所有邻居 +1；2) 单点减 v；3) 查询所有邻居权值异或和。

**分析**：对每个节点维护一棵 01-Trie 存储其所有儿子的权值（父亲单独处理）。全局 +1 操作利用 01-Trie 从低位建树的性质：交换左右子树即模拟最低位进位。单点修改在父节点的 Trie 中先删旧值再插新值。查询为 Trie 根的异或和 ⊕ 父亲权值。

```
数据结构：每节点一棵 01-Trie（低位在上），支持全局 +1
支持操作：insert/erase 单点改，addAll 全局加一（交换子树），xorSum 查询
关键性质：低位建树 + swap(son[0],son[1]) 模拟二进制 +1 进位
时间复杂度：O((n+m) log V)
```

**C++ 代码**：

```cpp
int ch[N][2], xorv[N], cnt[N], tot;

void pushup(int u) {
    xorv[u] = 0;
    if (ch[u][0]) xorv[u] ^= (xorv[ch[u][0]] << 1);
    if (ch[u][1]) xorv[u] ^= ((xorv[ch[u][1]] << 1) | cnt[ch[u][1]]);
}

void addAll(int u) {
    swap(ch[u][0], ch[u][1]);       // 最低位 0→1, 1→0
    if (ch[u][0]) addAll(ch[u][0]); // 原 1-分支产生进位
    pushup(u);
}

void insert(int& u, int x, int dep) {
    if (!u) u = ++tot;
    if (dep > 20) { cnt[u]++; return; }
    int b = (x >> dep) & 1;
    insert(ch[u][b], x, dep + 1);
    pushup(u); cnt[u]++;
}
// erase 类似 insert，cnt-- 并回收空节点
```

**核心启示**：
- **低位建 Trie 支持全局 +1**：swap 左右子树模拟进位，非常精妙
- **邻域操作拆分**：邻居 = 父亲 + 儿子，分别维护避免重叠

## 五、常见技巧与思路总结

### 5.1 字符集大小的选择

- 小写字母：Σ=26，直接 `ch[N][26]`
- 大小写+数字：Σ=62，需映射函数
- 二进制：Σ=2（01-Trie），空间最紧凑
- 动态字符集：用 `map<char,int>` 代替数组，牺牲常数换空间

### 5.2 Trie 上附加信息的技巧

在 Trie 节点上可以附加各种信息，根据题意灵活选择：

| 附加信息 | 典型用途 | 示例题 |
|---------|---------|--------|
| `cnt[u]`（结尾计数） | 完整串匹配 | 点名、字典查找 |
| `sz[u]`（经过计数） | 前缀统计 | P8306 模板 |
| `bitset<N>` | 文章/文档编号集合 | P3879 阅读理解 |
| `num[u]`（子树有效数） | 支持删除的 01-Trie | CF706D |
| `sum[u]`（版本计数） | 可持久化区间查询 | P4735 |

### 5.3 异或问题的 Trie 转化

遇到"最大异或值"问题，几乎都是 01-Trie：
- **静态集合**：直接建 01-Trie，贪心查询
- **带删除**：每个节点维护计数 num，查询时检查 num > 0
- **区间限制**：可持久化 Trie，版本差分
- **树上路径异或**：前缀异或和，转化为全局最大异或对

### 5.4 Trie 与其他算法的经典搭配

- **Trie + DP**：字符串可拆分性（L 语言）、状态转移基于 Trie 路径
- **Trie + 拓扑排序**：字母序约束问题（First!）
- **Trie + 贪心**：排列顺序优化（背单词、Type Printer）
- **Trie + DFS**：编辑距离搜索（电子字典）、操作序列生成
- **Trie + 最短路**：口令 LCP 转 LCA 深度（天才黑客）

### 5.5 01-Trie 全局 +1 技巧

从低位到高位建树时，全局 +1 等价于：
1. 交换根节点的左右子树（最低位 0↔1）
2. 递归对新的 0-子树执行 +1（处理进位）

这个技巧在维护"邻域批量加一 + 异或和查询"时非常有效。

## 六、解题思路框架

遇到 Trie 相关题目时，可按以下流程判断：


## 七、终极总结与归纳

### 7.1 Trie 树的本质

**Trie 是将字符串集合的公共前缀压缩为共享路径的多叉树，使得每个前缀对应唯一节点，从而将字符串操作转化为树上路径操作。**

它的威力不仅在于 O(L) 的查找，更在于树结构为其他算法（DP、贪心、图论、LCA）提供了天然的骨架。01-Trie 将这一思想推广到二进制数值，使"最大异或"问题有了优雅的贪心解法。

### 7.2 核心要点

1. **字符 Trie 的核心操作**：插入、查找、前缀计数，时间复杂度 O(L)
2. **01-Trie 贪心原理**：高位优先，尽量走反向分支，保证异或值最大
3. **可持久化 Trie**：版本差分 = 区间信息，支持区间异或最大查询
4. **节点附加信息**：cnt/sz/bitset/num/sum 等，灵活适配不同题意
5. **Trie 与其他算法的结合**：DP、拓扑排序、贪心、DFS、LCA、最短路
6. **异或的核心性质**：自消性 a⊕a=0，路径异或 = 前缀异或差
