## 一、什么是回文自动机

**回文自动机**（Palindrome Automaton，简称 PAM，又称 Eertree / 回文树）是 2014 年由 Mikhail Rubinchik 提出的数据结构，能在 **O(n)** 时间内处理一个字符串中**所有本质不同的回文子串**。与后缀自动机处理"所有子串"不同，PAM 专注于回文结构——它的每个节点恰好代表一个**本质不同的回文子串**。

PAM 的核心思想是：维护两棵以"奇根"和"偶根"为根的树，通过 **fail 指针**（类似 AC 自动机的失配链接）连接各回文串之间的最长回文后缀关系，实现在线增量构建。每次在字符串末尾添加一个字符，最多新增一个本质不同的回文子串，因此节点总数不超过 **n + 2**。

> **一句话概括**：PAM 将字符串的全部本质不同回文子串组织成一棵树，支持在线构建、O(1) 均摊插入，并天然维护出现次数、回文后缀等信息。

```text
PAM for s = "abba":

     偶根(len=0)        奇根(len=-1)
       /    \              /   \
     "aa"   "bb"         "a"   "b"
      |
    "abba"

节点数 = 5（不含两个根就是 5 个本质不同回文子串：a, b, bb, aa 不存在... 
实际：a, b, bb, abba, 加空串 → 4个本质不同回文子串 + 2 个根）

更准确的 PAM for "abba":
奇根(len=-1) ─a→ "a"(len=1) ─b→ "aba"? 
偶根(len=0)  ─b→ "bb"(len=2)
              ─a→ "aa"? 不存在

实际 "abba" 的本质不同回文子串：a, b, bb, abba
PAM 结构：
  奇根(len=-1) ──a──→ "a"(len=1)
               ──b──→ "b"(len=1)
  偶根(len=0)  ──b──→ "bb"(len=2)  ──a──→ "abba"(len=4)

  fail: "a" → 偶根, "b" → 偶根, "bb" → "b", "abba" → "a"
```

| 维度 | Manacher | PAM（回文自动机） | 字符串哈希 | 后缀自动机 SAM |
|------|----------|------------------|-----------|---------------|
| 时间复杂度 | O(n) | O(n) | O(n) 预处理 | O(n) |
| 空间复杂度 | O(n) | O(n·\|Σ\|) | O(n) | O(n·\|Σ\|) |
| 所有回文子串枚举 | 需后处理 | ✅ 天然支持 | 需额外处理 | 需结合其他算法 |
| 在线构建 | ❌ | ✅ | ❌ | ✅ |
| 回文出现次数 | 需额外处理 | ✅ fail 树直接给出 | ❌ | 需结合回文判定 |
| 回文分割 DP | 可作 check | ✅ series link 优化 | 可作 check | ❌ |
| 实现难度 | 中等 | 中等偏高 | 中等 | 较高 |

## 二、核心思想与前置知识

### 2.1 PAM 的两棵树与节点含义

PAM 包含两个根节点：
- **奇根**（odd root）：`len = -1`，代表长度为 -1 的"虚拟回文"，所有奇数长度回文串的祖先
- **偶根**（even root）：`len = 0`，代表空串回文，所有偶数长度回文串的祖先

每个非根节点代表一个**本质不同的回文子串**。节点 u 通过字符 c 连向节点 v，表示在 u 代表的回文串**两端各加一个字符 c** 得到 v 代表的回文串。

**关键性质**：
1. 一个长度为 n 的字符串，本质不同的回文子串最多 **n** 个
2. 每次插入一个新字符，最多新增 **1** 个本质不同的回文子串
3. PAM 节点总数不超过 **n + 2**（含两个根）

### 2.2 fail 指针与 getfail 过程

**fail[u]** 指向节点 u 代表的回文串的**最长真回文后缀**对应的节点。

**getfail 过程**：给定当前位置 i 和一个节点 u，不断沿 fail 链向上跳，直到 `s[i - len[u] - 1] == s[i]`，即找到一个回文后缀使得两端可以扩展。

```text
插入字符 s[i] 的流程：
1. 从 last（上一次插入后的节点）调用 getfail，找到可扩展的节点 cur
2. 若 cur 没有 s[i] 的转移边 → 新建节点 now：
   - len[now] = len[cur] + 2
   - fail[now] = ch[getfail(fail[cur], i)][s[i]]
   - ch[cur][s[i]] = now
3. last = ch[cur][s[i]]
4. cnt[last]++（该回文串出现次数 +1）
```

### 2.3 cnt 计数与 series link 优化

**cnt[u]**：节点 u 代表的回文子串的出现次数（需最后从子到父沿 fail 链累加）。

**num[u]**：以当前位置结尾的回文子串个数，`num[u] = num[fail[u]] + 1`。

**series link（等差数列优化）**：定义 `dif[u] = len[u] - len[fail[u]]`，当 `dif[u] == dif[fail[u]]` 时，`slink[u] = slink[fail[u]]`，否则 `slink[u] = fail[u]`。series link 将 fail 链上相同 dif 值的节点"跳过"，使回文分割 DP 从 O(n√n) 优化到 O(n log n)。

### 2.4 PAM 骨架代码

```cpp
const int MAXN = 500005;
struct PAM {
    int ch[MAXN][26], fail[MAXN], len[MAXN], cnt[MAXN], num[MAXN];
    int slink[MAXN], dif[MAXN]; // series link 相关
    int s[MAXN], last, tot, n;
    int newNode(int l) {
        len[++tot] = l;
        return tot;
    }
    void init() {
        tot = -1; last = 0; n = 0;
        s[0] = -1;
        newNode(0);  // 偶根 = 0
        newNode(-1); // 奇根 = 1
        fail[0] = 1;
    }
    int getfail(int x) {
        while (s[n - len[x] - 1] != s[n]) x = fail[x];
        return x;
    }
    void add(int c) {
        s[++n] = c;
        int cur = getfail(last);
        if (!ch[cur][c]) {
            int now = newNode(len[cur] + 2);
            fail[now] = ch[getfail(fail[cur])][c];
            ch[cur][c] = now;
            num[now] = num[fail[now]] + 1;
            dif[now] = len[now] - len[fail[now]];
            slink[now] = (dif[now] == dif[fail[now]]) ? slink[fail[now]] : fail[now];
        }
        last = ch[cur][c];
        cnt[last]++;
    }
    void count() { // 从子到父累加出现次数
        for (int i = tot; i >= 2; i--)
            cnt[fail[i]] += cnt[i];
    }
} pam;
```

## 三、分类与变体详解

### 3.1 基础 PAM 构建

最基础的应用：建 PAM，统计本质不同回文子串个数、每个回文子串的出现次数。

```
数据结构：PAM（回文自动机）
支持操作：在线逐字符插入、回文子串枚举、出现次数统计
关键性质：节点数 = 本质不同回文子串数 + 2；每次插入最多新增 1 个节点
时间复杂度：O(n) 构建，O(n) 统计
```

```cpp
// 本质不同回文子串数 = tot - 1（减去两个根）
int distinctPalindromes() {
    return pam.tot - 1;
}
// 出现次数最多的回文子串：count() 后遍历所有节点
```

### 3.2 回文子串计数与出现次数

PAM 天然支持以下统计：

| 统计量 | 方法 |
|--------|------|
| 本质不同回文子串数 | `tot - 1` |
| 每个回文子串出现次数 | `count()` 后 `cnt[u]` |
| 以位置 i 结尾的回文子串数 | `num[last]`（插入第 i 个字符后） |
| 所有回文子串出现次数之和 | `∑ cnt[u]`（u ≥ 2） |
| 最长回文子串 | `max(len[u])`（u ≥ 2） |

```
数据结构：PAM + fail 树累加
支持操作：回文计数、出现次数、最长回文
关键性质：cnt 沿 fail 树从叶到根累加；num 递推 O(1)
时间复杂度：O(n) 构建 + O(n) 统计
```

### 3.3 PAM + DP（最小回文分割等）

利用 **series link** 优化回文分割类 DP。定义 `f[i]` 为前 i 个字符的最优划分代价，转移时枚举以位置 i 结尾的所有回文后缀，借助 series link 将转移从 O(n) 压缩到 O(log n)。

```
数据结构：PAM + series link + DP
支持操作：最小回文分割、回文划分计数
关键性质：fail 链上 dif 相同的连续段形成等差数列，slink 跳过整段
时间复杂度：O(n log n) 或 O(n)
```

```cpp
// 回文分割 DP 核心（series link 优化）
long long f[MAXN], g[MAXN]; // f[i] = 前 i 个字符最优值, g[u] = 辅助值
void solve(int n) {
    f[0] = 0;
    for (int i = 1; i <= n; i++) {
        pam.add(s[i] - 'a');
        for (int u = pam.last; u > 1; u = pam.slink[u]) {
            g[u] = f[i - pam.len[pam.slink[u]] - pam.dif[u]];
            if (pam.dif[u] == pam.dif[pam.fail[u]])
                g[u] = min(g[u], g[pam.fail[u]]);
            f[i] = min(f[i], g[u] + 1);
        }
    }
}
```

### 3.4 双向 PAM / 在线构建

标准 PAM 支持从右端在线插入。双向 PAM（前端 + 后端同时插入）实现较复杂，需维护两个 `last` 指针。在线构建是 PAM 的天然优势：每插入一个字符可立即查询当前串的所有回文信息。

```
数据结构：双向 PAM / 在线 PAM
支持操作：前端或后端插入字符、实时查询回文信息
关键性质：两端 last 指针独立维护；节点总数仍为 O(n)
时间复杂度：O(n) 均摊
```

### 题目 1：回文自动机模板（洛谷 P5496）

> 给定字符串 S，对每个位置 i 输出以 i 结尾的回文子串个数（强制在线，每个字符需解密）。

**分析**：PAM 模板题。每插入一个字符，`num[last]` 即为以当前位置结尾的回文子串个数。注意强制在线的解密操作。

```
数据结构：PAM
支持操作：在线逐字符插入，实时查询以当前位置结尾的回文子串数
关键性质：num[u] = num[fail[u]] + 1，每次插入后 num[last] 即为答案
时间复杂度：O(n)
```

**C++ 代码**：
```cpp
// PAM 结构同骨架代码
// 对每个字符解密后插入
int lastans = 0;
for (int i = 0; i < n; i++) {
    int c = (s[i] - 'a' + lastans) % 26;
    pam.add(c);
    lastans = pam.num[pam.last];
    printf("%d ", lastans);
}
```

**核心启示**：
- **num[last]** 是 PAM 最基本的查询，直接给出以当前位置结尾的回文个数
- **在线构建**是 PAM 的核心优势，天然支持强制在线问题

### 题目 2：Palindrome Degree（CF7D）

> 定义字符串的"回文度"：若 s 不是回文则度为 0；若 s 是回文且其前半部分回文度为 k，则 s 的回文度为 k+1。求字符串每个前缀的回文度之和。

**分析**：对每个前缀判断是否回文，若是则递推其前半部分的回文度。可以用哈希 O(1) 判回文，然后 DP 递推。PAM 也可辅助判定，但此题哈希更直接。

```
数据结构：字符串哈希 + DP
支持操作：前缀回文判定、回文度递推
关键性质：d[i] = d[i/2] + 1（当前缀 s[1..i] 是回文且 i 为偶数时）
时间复杂度：O(n)
```

**C++ 代码**：
```cpp
const long long MOD = 1e9 + 7, BASE = 131;
long long pw[MAXN], h[MAXN], rh[MAXN];
int d[MAXN]; // d[i] = s[1..i] 的回文度
int ans = 0;

// 预处理正向和反向哈希
// h[i] = hash(s[1..i]), rh[i] = hash(s[n..n-i+1])
bool isPalin(int l, int r) {
    // 通过正向与反向哈希 O(1) 判断 s[l..r] 是否回文
    return getHash(l, r) == getRevHash(l, r);
}

for (int i = 1; i <= n; i++) {
    if (isPalin(1, i)) {
        d[i] = 1;
        if (i % 2 == 0) d[i] = d[i / 2] + 1;
    }
    ans += d[i];
}
```

**核心启示**：
- **回文度的递推本质是"回文套回文"**，利用前半部分信息递推
- 简单回文判定问题中，**哈希往往比 PAM 更轻量**；PAM 的优势在多串/计数场景

### 题目 3：Manacher 模板（洛谷 P3805）

> 给定字符串 S，求最长回文子串长度。

**分析**：Manacher 是求最长回文子串的经典线性算法。用 PAM 同样可以解决：建完 PAM 后，`max(len[u])` 即为答案。两种算法时间均为 O(n)，但 PAM 提供的信息更丰富（所有回文子串），Manacher 则更轻量。

```
数据结构：Manacher 算法 / PAM
支持操作：求最长回文子串长度
关键性质：Manacher 利用对称性跳过冗余比较；PAM 的 max(len[u]) 即答案
时间复杂度：O(n)
```

**C++ 代码**：
```cpp
// Manacher 做法
char t[MAXN * 2];
int p[MAXN * 2];

int manacher(const char* s, int n) {
    int len = 0;
    t[0] = '$'; t[++len] = '#';
    for (int i = 0; i < n; i++) { t[++len] = s[i]; t[++len] = '#'; }
    t[++len] = '!';
    int mid = 0, mr = 0, ans = 0;
    for (int i = 1; i < len; i++) {
        p[i] = (i < mr) ? min(p[2 * mid - i], mr - i) : 1;
        while (t[i - p[i]] == t[i + p[i]]) p[i]++;
        if (i + p[i] > mr) { mid = i; mr = i + p[i]; }
        ans = max(ans, p[i] - 1);
    }
    return ans;
}

// PAM 做法：建完后
int pamMaxPalin() {
    int ans = 0;
    for (int i = 2; i <= pam.tot; i++)
        ans = max(ans, pam.len[i]);
    return ans;
}
```

**核心启示**：
- **Manacher 和 PAM 是互补的回文工具**：Manacher 擅长求回文半径数组，PAM 擅长回文子串枚举与计数
- 选用哪种取决于问题需求：仅求最长 → Manacher；需要枚举/计数所有回文 → PAM

### 🟡 基础级 Level 2：回文计数与区间查询

### 题目 4：回文子串（洛谷 P1872）

> 给定字符串 S，求有多少对 (i, j) 满足 i < j 且 s[i..j] 是回文子串（即求回文子串的总数）。

**分析**：建 PAM，调用 `count()` 累加出现次数后，对每个节点 u，`cnt[u]` 表示该回文子串出现次数。所有回文子串总数 = `∑ cnt[u]`（u ≥ 2）。

```
数据结构：PAM + fail 树计数
支持操作：统计所有回文子串出现次数之和
关键性质：count() 将 cnt 沿 fail 树从子到父累加，得精确出现次数
时间复杂度：O(n)
```

**C++ 代码**：
```cpp
// 建 PAM 后
pam.count();
long long ans = 0;
for (int i = 2; i <= pam.tot; i++)
    ans += pam.cnt[i];
// ans 即为回文子串总数
```

**核心启示**：
- **fail 树的 cnt 累加**是 PAM 最基础的统计操作，类似 SAM 的 siz 上传
- 本质不同回文子串数 = `tot - 1`，总出现次数 = `∑ cnt[u]`

### 题目 5：Palindrome pairs（CF159D）

> 给定字符串 S，求有多少对 (l₁, r₁, l₂, r₂) 满足 r₁ < l₂ 且 s[l₁..r₁] 和 s[l₂..r₂] 都是回文子串。

**分析**：先统计以每个位置结尾的回文子串个数 `ed[i]`，以及以每个位置开头的回文子串个数 `st[i]`。答案为 `∑ ed[i] × sufSt[i+1]`，其中 `sufSt[i]` 是 `st[i..n]` 的后缀和。可以用 Manacher 或 PAM 求 `ed` 和 `st`。

```
数据结构：PAM / Manacher + 前后缀和
支持操作：统计每个位置结尾/开头的回文子串数
关键性质：以位置 i 结尾的回文数 = num[last]（PAM 插入到 i 时）
时间复杂度：O(n)
```

**C++ 代码**：
```cpp
long long ed[MAXN], st[MAXN]; // ed[i]: 以 i 结尾的回文数, st[i]: 以 i 开头的回文数

// 正向建 PAM 得 ed[i]
pam.init();
for (int i = 1; i <= n; i++) {
    pam.add(s[i] - 'a');
    ed[i] = pam.num[pam.last];
}
// 反向建 PAM 得 st[i]
pam.init();
for (int i = n; i >= 1; i--) {
    pam.add(s[i] - 'a');
    st[i] = pam.num[pam.last];
}
// 后缀和计算答案
long long sufSt = 0, ans = 0;
for (int i = n; i >= 1; i--) {
    sufSt += st[i];
    if (i > 1) ans += ed[i - 1] * sufSt;
}
```

**核心启示**：
- **正反两次建 PAM** 可以同时获得以每个位置结尾/开头的回文子串信息
- 组合计数常转化为"按分界点拆分"，化乘法原理为前后缀和

### 题目 6：Queries for Number of Palindromes（CF245H）

> 给定字符串 S（|S| ≤ 5000），回答 q 个查询 [l, r]：子串 s[l..r] 中有多少个回文子串。

**分析**：区间 DP 预处理。令 `f[l][r]` 表示 s[l..r] 是否为回文，`dp[l][r]` 表示 s[l..r] 中回文子串个数。利用容斥原理转移：`dp[l][r] = dp[l+1][r] + dp[l][r-1] - dp[l+1][r-1] + f[l][r]`。

```
数据结构：区间 DP
支持操作：O(1) 回答区间回文子串数查询
关键性质：容斥递推 dp[l][r]；f[l][r] = (s[l]==s[r]) && f[l+1][r-1]
时间复杂度：O(n²) 预处理，O(1) 查询
```

**C++ 代码**：
```cpp
const int MAXN = 5005;
int dp[MAXN][MAXN];
bool f[MAXN][MAXN];

void preprocess(const char* s, int n) {
    for (int i = 1; i <= n; i++) {
        f[i][i] = true; dp[i][i] = 1;
        if (i < n) {
            f[i][i + 1] = (s[i] == s[i + 1]);
            dp[i][i + 1] = 2 + f[i][i + 1];
        }
    }
    for (int len = 3; len <= n; len++)
        for (int l = 1; l + len - 1 <= n; l++) {
            int r = l + len - 1;
            f[l][r] = (s[l] == s[r]) && f[l + 1][r - 1];
            dp[l][r] = dp[l + 1][r] + dp[l][r - 1] - dp[l + 1][r - 1] + f[l][r];
        }
}
// 查询 [l, r] → dp[l][r]
```

**核心启示**：
- **区间 DP + 容斥**是小规模区间回文查询的标准方法
- 当 n 较大时（> 5000），需改用 PAM 或 Manacher 配合其他数据结构

### 🟠 进阶级 Level 3：PAM 综合应用

### 题目 7：回文串（洛谷 P3649 / APIO2014）

> 给定字符串 S（|S| ≤ 300000），求 `max(len[u] × cnt[u])`，即所有回文子串中"长度 × 出现次数"的最大值。

**分析**：建 PAM，调用 `count()` 将 cnt 从叶到根累加。然后遍历所有节点，取 `len[u] × cnt[u]` 的最大值。

```
数据结构：PAM + fail 树 cnt 累加
支持操作：求所有回文子串的 len × 出现次数 的最大值
关键性质：count() 后 cnt[u] 为精确出现次数；len[u] 为该回文串长度
时间复杂度：O(n)
```

**C++ 代码**：
```cpp
pam.count();
long long ans = 0;
for (int i = 2; i <= pam.tot; i++)
    ans = max(ans, (long long)pam.len[i] * pam.cnt[i]);
```

**核心启示**：
- **PAM 的 cnt 累加与 SAM 的 siz 上传本质相同**——都是在 fail/parent 树上从子到父汇聚信息
- 此题也可用 SAM 解决（在 parent tree 上统计），PAM 解法更直接

### 题目 8：双倍回文（洛谷 P4287 / SHOI2011）

> 给定字符串 S，求最长的子串 T，使得 T 的长度为 4 的倍数，且 T 可以表示为 ABAB 形式（其中 A 和 B 都是回文串，且 |A| = |B|）。即 T 是一个回文串，T 的前半部分也是回文串。

**分析**：建 PAM。对每个长度为偶数的回文节点 u，检查其是否存在一个长度为 `len[u]/2` 的回文后缀。利用 fail 链上的信息：在 fail 链上找到满足 `len[v] ≤ len[u]/2` 的最长回文后缀 v，若 `len[v] == len[u]/2` 则 u 是双倍回文。可在 BFS/DFS 建树时维护一个"半长度指针" half。

```
数据结构：PAM + fail 树 + 半长度指针
支持操作：判断回文串是否为双倍回文
关键性质：双倍回文 = 长度 4k 的回文，且存在长度 2k 的回文后缀
时间复杂度：O(n)
```

**C++ 代码**：
```cpp
int half[MAXN]; // half[u]: fail 链上 len ≤ len[u]/2 的最深节点
int ans = 0;

// 在 add 中维护 half 指针
void add(int c) {
    s[++n] = c;
    int cur = getfail(last);
    if (!ch[cur][c]) {
        int now = newNode(len[cur] + 2);
        fail[now] = ch[getfail(fail[cur])][c];
        ch[cur][c] = now;
        // 维护 half 指针
        if (len[now] <= 2) {
            half[now] = fail[now];
        } else {
            int h = half[cur];
            while (s[n - len[h] - 1] != s[n] || len[h] + 2 > len[now] / 2)
                h = fail[h];
            half[now] = ch[h][c];
        }
    }
    last = ch[cur][c];
    cnt[last]++;
    // 检查双倍回文
    int u = last;
    if (len[u] % 4 == 0 && len[half[u]] == len[u] / 2)
        ans = max(ans, len[u]);
}
```

**核心启示**：
- **PAM 上维护额外指针**（如 half）是处理"回文套回文"结构的关键技巧
- fail 链的跳跃可以配合长度约束，实现 O(1) 均摊的条件查找

### 题目 9：Palisection（CF17E）

> 给定字符串 S（|S| ≤ 2×10⁶），求有多少对有交集的回文子串（即两个回文子串至少共享一个位置）。

**分析**：正难则反。总对数 = C(total, 2)，其中 total 为回文子串总数。不相交对数 = `∑ ed[i] × sufSt[i+1]`，其中 `ed[i]` 为以 i 结尾的回文子串数，`sufSt` 为以 i 开头的回文数的后缀和。答案 = 总对数 - 不相交对数。用 Manacher 或 PAM 求 ed 和 st。

```
数据结构：Manacher / PAM + 差分
支持操作：统计有交集的回文子串对数
关键性质：正难则反——总对数减去不相交对数
时间复杂度：O(n)
```

**C++ 代码**：
```cpp
long long ed[MAXN], st[MAXN];
const int MOD = 51123987;

// 用 Manacher 求 ed[i] 和 st[i]（差分累加）
// 或用正反两次 PAM

long long total = 0;
for (int i = 1; i <= n; i++) total = (total + ed[i]) % MOD;
long long allPairs = total % MOD * ((total - 1) % MOD) % MOD;
// 注意取模处理
allPairs = (allPairs % MOD + MOD) % MOD;
// 逆元求 C(total,2) 或直接 total*(total-1)/2 mod p

long long disjoint = 0, sufSum = 0;
for (int i = n; i >= 1; i--) {
    sufSum = (sufSum + st[i]) % MOD;
    if (i > 1) disjoint = (disjoint + ed[i - 1] * sufSum) % MOD;
}
long long ans = ((allPairs - disjoint) % MOD + MOD) % MOD;
```

**核心启示**：
- **正难则反**是组合计数的核心思维——算不相交比算相交容易
- 大规模问题需注意取模运算的正确性，尤其是减法取模

### 题目 10：最长双回文串（洛谷 P4555）

> 给定字符串 S，求最长的子串 T，使得 T 可以分成两个非空回文串的拼接。

**分析**：对每个分割点 i，求"以 i 结尾的最长回文前缀"和"以 i+1 开头的最长回文后缀"。用 Manacher 算法：先求出回文半径数组 p[]，然后用差分标记每个位置能作为某个回文的结束/开始位置的最大长度。也可用 PAM 在线求解。

```
数据结构：Manacher + 差分标记
支持操作：求每个分割点两侧最长回文
关键性质：Manacher 的 p[i] 可推出每个位置作为回文端点的最大延伸长度
时间复杂度：O(n)
```

**C++ 代码**：
```cpp
int maxL[MAXN], maxR[MAXN]; // maxL[i]: 以 i 结尾的最长回文长度, maxR[i]: 以 i 开头的

// Manacher 求 p[] 后差分求 maxL 和 maxR
// 在变换串上: 回文 [i-p[i]+1, i+p[i]-1] 对应原串位置
// 利用差分: 每个回文贡献右端点更新 maxL，左端点更新 maxR
// 具体做法：从每个中心向右递推，maxL[r] = max(maxL[r], 对应长度)
// 再对 maxL 做 max(maxL[i], maxL[i-2]+2) 的递推补全

int ans = 0;
for (int i = 1; i < n; i++)
    ans = max(ans, maxL[i] + maxR[i + 1]);
```

**核心启示**：
- **"拼接两个回文"问题的标准思路**是枚举分割点，预处理两侧最长回文
- Manacher 的 p 数组配合差分递推，可以 O(n) 求出每个位置的最长回文前/后缀

### 题目 11：Virus synthesis（洛谷 P4762 / CERC2014）

> 初始空串，两种操作：① 在头或尾添加一个字符（代价 1）；② 将当前串翻转后拼接到头或尾（代价 1，要求结果为回文）。求构造目标串 S 的最少操作数。多组数据。

**分析**：操作②本质上是"通过翻转构造偶数长度回文"。在 PAM 上 DP：`f[u]` 表示构造节点 u 代表的回文串的最少操作数。对偶数长度节点，可以从 `f[fail[u]]` 或 `f[half[u]]` 转移（half 为长度 ≤ len[u]/2 的最长回文后缀）。BFS 按拓扑序更新。最终答案 = `min(f[u] + n - len[u])`。

```
数据结构：PAM + BFS/DP + half 指针
支持操作：最小操作数构造目标回文
关键性质：偶数回文可通过翻转操作用 len/2 + 1 步构造；half 指针跳到 ≤ len/2 的回文后缀
时间复杂度：O(n|Σ|)
```

**C++ 代码**：
```cpp
int f[MAXN]; // f[u] = 构造节点 u 回文串的最小操作数
int half[MAXN];

void solve() {
    // 建 PAM（同题目 8 维护 half 指针）
    // BFS 初始化
    queue<int> q;
    for (int i = 0; i <= pam.tot; i++) f[i] = pam.len[i]; // 最坏情况逐字符添加
    f[0] = 1; // 偶根：翻转一次
    f[1] = 0; // 奇根：无需操作(虚拟)
    for (int c = 0; c < 26; c++) {
        if (pam.ch[0][c]) q.push(pam.ch[0][c]);
        if (pam.ch[1][c]) q.push(pam.ch[1][c]);
    }
    while (!q.empty()) {
        int u = q.front(); q.pop();
        // 从 fail 转移
        f[u] = f[pam.fail[u]] + pam.len[u] - pam.len[pam.fail[u]];
        // 从 half 转移（偶数长度回文的翻转优化）
        if (pam.len[u] % 2 == 0 && pam.len[u] >= 2) {
            int h = half[u];
            f[u] = min(f[u], f[h] + 1 + pam.len[u] / 2 - pam.len[h]);
        }
        for (int c = 0; c < 26; c++)
            if (pam.ch[u][c]) q.push(pam.ch[u][c]);
    }
    int ans = n;
    for (int i = 2; i <= pam.tot; i++)
        ans = min(ans, f[i] + n - pam.len[i]);
    printf("%d\n", ans);
}
```

**核心启示**：
- **PAM 上 BFS/DP** 是处理回文构造问题的利器——每个节点代表一种"中间状态"
- **half 指针**的维护使得翻转操作的转移 O(1) 完成

### 题目 12：拉拉队排练（洛谷 P1659）

> 给定字符串 S（|S| ≤ 10⁶），取 k 个回文子串（可重叠、可相同但位置不同），使得长度之积最大。只能取奇数长度的回文子串。

**分析**：建 PAM，`count()` 后得每个回文子串出现次数。按长度从大到小贪心选取奇数长度回文。用快速幂计算乘积。

```
数据结构：PAM + 贪心 + 快速幂
支持操作：按长度降序选取奇回文子串，最大化乘积
关键性质：贪心——优先选最长的回文；奇数长度回文在 PAM 中即奇根子树下的节点
时间复杂度：O(n log MOD)
```

**C++ 代码**：
```cpp
const int MOD = 19930726;

pam.count();
// 收集所有奇数长度回文节点，按 len 降序排序
vector<pair<int,long long>> v; // (len, cnt)
for (int i = 2; i <= pam.tot; i++)
    if (pam.len[i] % 2 == 1)
        v.push_back({pam.len[i], pam.cnt[i]});
sort(v.rbegin(), v.rend());

long long ans = 1, rem = k;
for (auto& [l, c] : v) {
    if (rem <= 0) break;
    long long use = min((long long)c, rem);
    ans = ans * qpow(l, use, MOD) % MOD;
    rem -= use;
}
if (rem > 0) { puts("-1"); return; }
printf("%lld\n", ans);
```

**核心启示**：
- **PAM 的 cnt 统计 + 贪心**是"选回文子串最优化"类问题的标准模式
- 快速幂处理大量相同长度回文的乘积

### 题目 13：Prefix-Suffix Palindrome（CF1326D2）

> 给定字符串 S，求最长的字符串 T = A + B，其中 A 是 S 的前缀，B 是 S 的后缀，A 和 B 不重叠，且 T 是回文。

**分析**：先贪心匹配 S 的首尾相同字符（设匹配了 k 对）。然后在剩余的中间部分 s[k+1..n-k] 中，找最长的回文前缀或回文后缀，拼接到结果中。用字符串哈希或 Manacher O(n) 完成。

```
数据结构：双指针 + 字符串哈希 / Manacher
支持操作：首尾匹配 + 中间最长回文前/后缀
关键性质：T = s[1..k] + 中间最长回文前缀或后缀 + s[n-k+1..n]
时间复杂度：O(n)
```

**C++ 代码**：
```cpp
string solve(const string& s) {
    int n = s.size(), k = 0;
    while (k < n - 1 - k && s[k] == s[n - 1 - k]) k++;
    // 中间部分 s[k..n-1-k]
    string mid = s.substr(k, n - 2 * k);
    // 找 mid 的最长回文前缀和最长回文后缀
    // 用 Manacher 或哈希
    int mLen = mid.size();
    string bestMid = "";
    // 检查回文前缀
    for (int len = mLen; len >= 1; len--) {
        if (isPalin(mid, 0, len - 1)) {
            bestMid = mid.substr(0, len); break;
        }
    }
    // 检查回文后缀
    for (int len = mLen; len >= 1; len--) {
        if (isPalin(mid, mLen - len, mLen - 1)) {
            if (len > (int)bestMid.size())
                bestMid = mid.substr(mLen - len);
            break;
        }
    }
    return s.substr(0, k) + bestMid + s.substr(n - k, k);
}
// 实际实现用哈希 O(n) 判回文，避免暴力枚举
```

**核心启示**：
- **首尾贪心匹配 + 中间取最长回文**是前后缀回文问题的经典拆解
- 回文判定可用哈希、Manacher 或 PAM 多种工具

### 题目 14：Palindrome Partition（CF932G）

> 给定字符串 S（|S| ≤ 10⁶），将 S 分成 k 段 t₁t₂...tₖ，使得 tᵢ = t_{k+1-i}（对称段相等），且每段都是回文串。求分割方案数（mod 10⁹+7）。

**分析**：构造新串 `w[i] = s[i]·s[n+1-i]`（双字符对），问题转化为求 w 的"回文分割"方案数。在 w 上建 PAM 并利用 series link 优化 DP：`f[i]` 表示前 i 对的方案数，沿 series link 跳跃转移。

```
数据结构：PAM + series link + DP
支持操作：对称回文分割计数
关键性质：series link 跳过 fail 链上 dif 相同的等差段，转移次数 O(log n)
时间复杂度：O(n log n)
```

**C++ 代码**：
```cpp
const int MOD = 1e9 + 7;
long long f[MAXN], g[MAXN];

// 构造新序列 w，建 PAM
// 注意：w 使用 26×26 种"双字符"作为字符集
void solve(int n) {
    f[0] = 1;
    for (int i = 1; i <= n / 2; i++) {
        pam.add(w[i]); // w[i] 编码 (s[i], s[n+1-i])
        f[i] = 0;
        for (int u = pam.last; u > 1; u = pam.slink[u]) {
            g[u] = f[i - pam.len[pam.slink[u]] - pam.dif[u]];
            if (pam.dif[u] == pam.dif[pam.fail[u]])
                g[u] = (g[u] + g[pam.fail[u]]) % MOD;
            f[i] = (f[i] + g[u]) % MOD;
        }
    }
    printf("%lld\n", f[n / 2]);
}
```

**核心启示**：
- **series link 是回文分割 DP 的核心优化工具**——将 O(n²) 暴力优化到 O(n log n)
- "对称分割"类问题常通过构造"双字符"序列转化为标准回文分割

### 题目 15：Reverses（CF906E）

> 给定字符串 s 和 t（|s| = |t| ≤ 5×10⁵），可以选若干不相交的 t 的子串并翻转，使 t 变为 s。求最少翻转次数及方案。

**分析**：构造交错串 `w = s₁t₁s₂t₂...sₙtₙ`。t 的子串 [l, r] 翻转后与 s 对应段匹配，等价于 w 的对应子串是偶数长度回文。问题转化为 w 的最少偶数回文分割。在 w 上建 PAM，利用 series link 优化 DP，长度为 2 的段（sᵢ = tᵢ）不消耗翻转次数。

```
数据结构：PAM + series link + DP + 路径还原
支持操作：最小偶数回文分割
关键性质：交错串上的偶数回文 ↔ 原问题的翻转区间；series link 优化转移
时间复杂度：O(n log n)
```

**C++ 代码**：
```cpp
const int INF = 1e9;
int f[MAXN], g[MAXN], pre[MAXN];

void solve(int n) {
    // 构造交错串 w[1..2n]
    for (int i = 1; i <= n; i++) {
        w[2 * i - 1] = s[i]; // s 的字符
        w[2 * i] = t[i];     // t 的字符
    }
    // 建 PAM
    pam.init();
    for (int i = 1; i <= 2 * n; i++) f[i] = INF;
    f[0] = 0;
    for (int i = 1; i <= 2 * n; i++) {
        pam.add(w[i] - 'a');
        if (i >= 2 && w[i] == w[i - 1] && i % 2 == 0)
            f[i] = min(f[i], f[i - 2]); // 相同字符无需翻转
        for (int u = pam.last; u > 1; u = pam.slink[u]) {
            g[u] = f[i - pam.len[pam.slink[u]] - pam.dif[u]];
            if (pam.dif[u] == pam.dif[pam.fail[u]])
                g[u] = min(g[u], g[pam.fail[u]]);
            if (pam.len[u] % 2 == 0 && g[u] + 1 < f[i]) {
                f[i] = g[u] + 1;
            }
        }
    }
    if (f[2 * n] >= INF) puts("-1");
    else printf("%d\n", f[2 * n]);
    // 路径还原通过 pre[] 回溯
}
```

**核心启示**：
- **交错串构造**是将"两串匹配 + 翻转"转化为回文问题的经典技巧
- **CF906E 是 PAM + series link 的终极应用**，综合了构造转化、回文分割、DP 优化

## 五、常见技巧与思路总结

### 5.1 fail 树的 cnt 累加

PAM 构建过程中每个节点的 cnt 只记录了"作为最长回文后缀被匹配到"的次数。最终需从编号最大的节点到根（按编号递减）累加 `cnt[fail[i]] += cnt[i]`，才能得到每个回文子串的真实出现次数。

### 5.2 正反双建 PAM

许多问题需要同时知道"以每个位置结尾的回文信息"和"以每个位置开头的回文信息"。正向建一次 PAM 得到前者，对反串再建一次得到后者。

### 5.3 half 指针维护

在 add 函数中额外维护 `half[u]`：指向 fail 链上 `len ≤ len[u]/2` 的最深节点。用于双倍回文、Virus synthesis 等需要"回文的一半也是回文"的问题。

### 5.4 series link 等差数列优化

fail 链上连续 dif 值相同的节点长度构成等差数列。`slink[u]` 跳到等差段的第一个节点，配合辅助数组 `g[u]` 可将 DP 转移从 O(n) 降至 O(log n)。

### 5.5 PAM 与其他算法的结合

| 结合方式 | 应用场景 |
|---------|---------|
| PAM + 哈希 | 回文判定辅助、双倍回文验证 |
| PAM + Manacher | 互相验证、结合差分求端点信息 |
| PAM + SAM | 回文串在后缀自动机上的定位 |
| PAM + 线段树 | 区间回文查询、在线维护 |

### 5.6 多组数据时的清空策略

PAM 的清空需注意：不能简单 memset 整个数组（太慢），应只清空用过的节点。推荐在 init 时记录 tot，清空时只清 [0, tot] 范围。

```cpp
void clear() {
    for (int i = 0; i <= tot; i++) {
        memset(ch[i], 0, sizeof(ch[i]));
        fail[i] = len[i] = cnt[i] = num[i] = 0;
    }
}
```

## 六、优化方法 / 解题思路框架

```text
遇到回文相关问题
├── 仅求最长回文子串？
│   └── Manacher 即可（O(n)，最轻量）
├── 需要枚举所有回文子串？
│   └── 建 PAM → 遍历节点即可
├── 需要回文子串出现次数？
│   └── PAM + count()（fail 树 cnt 累加）
├── 回文分割 / 回文划分 DP？
│   └── PAM + series link 优化 DP
├── 回文构造（最少操作数）？
│   └── PAM + BFS/DP（如 Virus synthesis）
├── 两串匹配 + 翻转？
│   └── 交错串 + PAM + series link DP

│   └── 区间 DP O(n²) 预处理

    └── 正反双建 PAM + 前后缀和
```

## 七、终极总结与归纳

### 7.1 回文自动机的本质

**PAM 是将字符串的全部回文结构压缩到 O(n) 个节点的树形索引**。每个节点对应一个本质不同的回文子串，fail 指针构成一棵"回文后缀树"，series link 进一步将 fail 链优化为等差跳跃。这使得几乎所有回文相关问题都能在 O(n) 或 O(n log n) 内解决。

### 7.2 核心要点

1. **两根结构**：奇根(len=-1) 和偶根(len=0) 分别管理奇偶长度回文
2. **getfail 跳跃**：沿 fail 链找可扩展位置，是 PAM 构建的核心操作
3. **cnt 累加**：从子到父沿 fail 链累加，得精确出现次数（类比 SAM 的 siz 上传）
4. **series link**：跳过 fail 链上的等差段，使回文分割 DP 降至 O(n log n)
5. **half 指针**：维护"长度 ≤ len/2 的最长回文后缀"，用于双倍回文、构造类问题
6. **在线性**：PAM 天然支持逐字符在线构建，适合强制在线问题
