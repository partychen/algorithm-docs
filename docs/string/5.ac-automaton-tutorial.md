## 一、什么是 AC 自动机

**AC 自动机**（Aho-Corasick Automaton）是由 Alfred V. Aho 和 Margaret J. Corasick 于 1975 年提出的一种**多模式串匹配**数据结构。它可以看成 **Trie 树与 KMP 算法的结合体**：先把所有模式串插入 Trie，再用类似 KMP 的 fail 指针在 Trie 上做失配跳转，从而在 O(n + m + z) 时间内找出文本串中所有模式串的出现（n 为文本长度，m 为模式串总长，z 为匹配次数）。

AC 自动机的核心思想是：**当某个模式串匹配失败时，利用已匹配的前缀信息，跳转到另一个可能匹配成功的模式串的某个前缀状态继续匹配**——这正是 KMP 思想在多模式场景下的推广。

```text
AC 自动机 = Trie（多模式组织） + fail 指针（失配跳转）

单模式匹配：KMP   ──→ 文本指针不回退，模式指针按 next 跳
多模式匹配：AC 自动机 ──→ 文本指针不回退，Trie 指针按 fail 跳

Trie 示例（模式串 "he", "she", "his", "hers"）:

         root
        / | \
       h  s  ...
      / \  \
     e   i  h
     |   |   \
     r   s    e → fail → (h→e)
     |
     s

fail 指针将 Trie 变成自动机：
  "she" 的 'e' 节点 fail → "he" 的 'e' 节点
  → 匹配到 "she" 时自动发现 "he" 也被匹配
```

| 对比项 | 朴素多模匹配 | AC 自动机 | KMP（单模） | 后缀自动机 |
|--------|------------|----------|------------|-----------|
| 模式串数量 | 多个 | 多个 | 1 个 | 1 个（子串） |
| 预处理 | 无 | O(m) 建 Trie + fail | O(m) 建 next | O(n) 建 SAM |
| 匹配时间 | O(n × m) | O(n + z) | O(n) | O(\|t\|) 判定 |
| 空间 | O(m) | O(m\|Σ\|) | O(m) | O(n\|Σ\|) |
| 适用场景 | 小规模 | 多模匹配 | 单模匹配 | 子串统计 |

## 二、核心思想与前置知识

### 2.1 Trie 建树

将所有模式串逐字符插入 Trie 树。每个模式串的末尾节点打上标记（记录该模式串的编号或长度）。

```cpp
const int MAXN = 2000005; // 所有模式串总长
int ch[MAXN][26], fail[MAXN], val[MAXN], tot;

void insert(char s[], int id) {
    int u = 0;
    for (int i = 1; s[i]; i++) {
        int c = s[i] - 'a';
        if (!ch[u][c]) ch[u][c] = ++tot;
        u = ch[u][c];
    }
    val[u] = id; // 标记模式串编号
}
```

### 2.2 fail 指针（BFS 构建）

fail 指针的含义：节点 u 的 fail 指向 **Trie 中 u 所代表字符串的最长真后缀** 对应的节点。构建方法是 BFS 按层处理：

1. 根的所有儿子的 fail 指向根
2. 对于节点 u 的儿子 v（字符 c）：若 `ch[fail[u]][c]` 存在，则 `fail[v] = ch[fail[u]][c]`；否则沿 fail 链继续跳
3. **Trie 图优化**：若 `ch[u][c]` 不存在，直接令 `ch[u][c] = ch[fail[u]][c]`，将 Trie 补全为 DFA

```cpp
int que[MAXN], hd, tl;

void build() {
    hd = tl = 0;
    for (int c = 0; c < 26; c++)
        if (ch[0][c]) que[tl++] = ch[0][c];
    while (hd < tl) {
        int u = que[hd++];
        for (int c = 0; c < 26; c++) {
            if (ch[u][c]) {
                fail[ch[u][c]] = ch[fail[u]][c];
                que[tl++] = ch[u][c];
            } else {
                ch[u][c] = ch[fail[u]][c]; // Trie 图补边
            }
        }
    }
}
```

> Trie 图补边后，`ch[u][c]` 对任意 u, c 都有值——这是一个完整的 DFA，匹配时无需再沿 fail 链跳转，直接走 `ch` 转移即可。

### 2.3 多模匹配过程

在 Trie 图上逐字符转移，每到一个节点，沿 fail 链（或后缀链接）收集所有匹配的模式串：

```cpp
void query(char t[]) {
    int u = 0;
    for (int i = 1; t[i]; i++) {
        u = ch[u][t[i] - 'a'];
        for (int j = u; j && val[j] != -1; j = fail[j]) {
            if (val[j] > 0) {
                // 模式串 val[j] 在位置 i 处匹配
            }
            val[j] = -1; // 标记已访问，避免重复
        }
    }
}
```

### 2.4 fail 树与拓扑优化

所有 fail 指针构成一棵以根为根的**fail 树**。关键性质：

- **节点 u 是模式串 p 的末尾节点，当且仅当文本匹配到 u 时 p 出现了**
- **节点 v 在 fail 树上是 u 的祖先 ⟺ v 所代表的字符串是 u 所代表字符串的后缀**
- **统计每个模式串的出现次数**：文本匹配时只在到达的节点上 +1，最后在 fail 树上从叶到根做**拓扑排序上传**，累加到所有祖先

```text
fail 树示意（将 fail 指针方向反转）：

        root (0)
       / | \
      h  s  ...
     / \  \
    he  hi  sh
    |   |    \
   her his   she
    |
  hers

统计模式串出现次数的拓扑排序：
  按 BFS 逆序处理，cnt[fail[u]] += cnt[u]
  → 等价于 fail 树上子树和
```

```cpp
// 拓扑排序上传（用 BFS 序逆序即可）
void topoCount() {
    for (int i = tl - 1; i >= 0; i--)
        cnt[fail[que[i]]] += cnt[que[i]];
}
```

## 三、分类与变体详解

### 3.1 基础多模式匹配

最基本的用法：给定若干模式串和一个文本串，判断哪些模式串在文本中出现，或统计总出现次数。

```
数据结构：AC 自动机（Trie + fail 指针）
支持操作：多模式串匹配、出现判定
关键性质：Trie 图保证每步转移 O(1)；沿 fail 链收集所有匹配
时间复杂度：O(m) 建 Trie，O(m) 建 fail，O(n + z) 匹配
```

### 3.2 fail 树 + 拓扑排序

当需要精确统计每个模式串的出现次数时，不在匹配时沿 fail 链暴力跳（可能退化为 O(n√m)），而是在 fail 树上做拓扑排序一次性上传。

```
数据结构：AC 自动机 + fail 树拓扑排序
支持操作：精确统计每个模式串出现次数
关键性质：fail 树上子树和 = 该节点作为后缀被经过的总次数
时间复杂度：O(n + m) 匹配 + 上传
```

```cpp
// 匹配：只在经过的节点上累加，不跳 fail
void match(char t[]) {
    int u = 0;
    for (int i = 1; t[i]; i++) {
        u = ch[u][t[i] - 'a'];
        cnt[u]++;
    }
}
// 拓扑上传后，cnt[end[i]] 即为模式串 i 的出现次数
```

### 3.3 AC 自动机 + DP

将 AC 自动机视为 DFA，状态为自动机节点，DP 在自动机上转移。常见模式：

- **方案计数**：长度为 n 的字符串中，至少包含一个模式串的方案数（正难则反：总方案 − 不含任何模式串的方案）
- **最优选择**：在自动机上走 n 步，最大化/最小化匹配到的模式串权值
- **数位 DP + AC 自动机**：枚举数字串的每一位，在自动机上转移，统计不含/包含某些模式的数字个数

```
数据结构：AC 自动机 DFA + 动态规划
支持操作：方案计数、最优匹配、避免/收集模式串
关键性质：自动机节点即 DP 状态，转移沿 ch[][] 走
时间复杂度：O(n × tot × |Σ|)，tot 为自动机节点数
```

### 3.4 AC 自动机 + 矩阵快速幂

当 DP 的"步数" n 极大（≤ 10⁹）而自动机节点数 tot 较小（≤ 几百）时，将 AC 自动机的转移写成 tot × tot 矩阵，用矩阵快速幂加速。

```
数据结构：AC 自动机 DFA + 矩阵快速幂
支持操作：极大步数的方案计数
关键性质：转移矩阵 A[i][j] = 从状态 i 走一步到状态 j 的方案数
时间复杂度：O(tot³ × log n)
```

| 变体 | 典型问题 | 核心技巧 |
|------|---------|---------|
| 基础多模匹配 | P3808, HDU2222 | 建 Trie + fail + 匹配 |
| fail 树拓扑 | P5357, P3966 | 拓扑上传统计出现次数 |
| AC + DP | P4052, P3311, CF86C | DFA 上 DP 计数 |
| AC + 矩阵幂 | UVA11468 变体 | 矩阵快速幂加速 |
| AC + fail 树离线 | P2414, CF1207G | fail 树 DFS 序 + 树状数组 |
| AC + 栈回退 | P3121 | 栈保存匹配路径 |
| AC + 图论 | P2444 | 自动机上找环 |

## 四、16 道经典例题详解

### 🟢 入门级 Level 1：基础多模匹配

### 题目 1：AC 自动机简单版（洛谷 P3808）

> 给定 n 个模式串和一个文本串，求有多少个模式串在文本串中出现过。

**分析**：AC 自动机的入门模板题。将模式串建 Trie、构建 fail 指针，匹配时沿 fail 链收集所有匹配。由于只需统计出现过的数量（不计重复），可以在访问过的节点上打标记避免重复计数。

```
数据结构：AC 自动机（Trie + fail 指针 + Trie 图）
支持操作：多模式串出现判定
关键性质：Trie 图补边后每步转移 O(1)；沿 fail 链收集后缀匹配
时间复杂度：O(n + m)，n 为文本长度，m 为模式串总长
```

**C++ 代码**：

```cpp
const int MAXN = 1000005;
int ch[MAXN][26], fail[MAXN], val[MAXN], tot;
int que[MAXN];

void insert(char s[]) {
    int u = 0;
    for (int i = 0; s[i]; i++) {
        int c = s[i] - 'a';
        if (!ch[u][c]) ch[u][c] = ++tot;
        u = ch[u][c];
    }
    val[u]++;
}

void build() {
    int hd = 0, tl = 0;
    for (int c = 0; c < 26; c++)
        if (ch[0][c]) que[tl++] = ch[0][c];
    while (hd < tl) {
        int u = que[hd++];
        for (int c = 0; c < 26; c++)
            if (ch[u][c]) fail[ch[u][c]] = ch[fail[u]][c], que[tl++] = ch[u][c];
            else ch[u][c] = ch[fail[u]][c];
    }
}

int query(char t[]) {
    int u = 0, ans = 0;
    for (int i = 0; t[i]; i++) {
        u = ch[u][t[i] - 'a'];
        for (int j = u; j && val[j] != -1; j = fail[j])
            ans += val[j], val[j] = -1;
    }
    return ans;
}
```

**核心启示**：
- **Trie 图补边是 AC 自动机高效匹配的关键**，补边后转移无需循环跳 fail
- **沿 fail 链收集 + 标记已访问**避免重复统计，适用于"出现过几种"类问题

### 题目 2：Keywords Search（HDU2222）

> 给定 n 个模式串和一个文本串，求有多少个模式串在文本串中出现过。多组数据。

**分析**：与 P3808 基本一致，但需要注意多组数据时清空 Trie 的操作。关键点在于 `memset` 的范围只需覆盖 `[0, tot]`，避免全局清空导致超时。

```
数据结构：AC 自动机（Trie + fail 指针）
支持操作：多模式串出现判定（多组数据）
关键性质：每组数据需清空 [0, tot] 范围内的 Trie 节点
时间复杂度：O(n + m) 每组，n 文本长度，m 模式串总长
```

**C++ 代码**：

```cpp
const int MAXN = 500005;
int ch[MAXN][26], fail[MAXN], val[MAXN], tot;
int que[MAXN];

void init() {
    for (int i = 0; i <= tot; i++) {
        memset(ch[i], 0, sizeof(ch[i]));
        fail[i] = val[i] = 0;
    }
    tot = 0;
}

// insert, build 同题目 1
// query: 沿 fail 链收集，val[j] = -1 标记已访问
```

**核心启示**：
- **多组数据时局部清空 `[0, tot]`** 是 AC 自动机题目的常见优化
- HDU2222 是 AC 自动机在 OJ 上最经典的入门题，与 P3808 互为验证

### 🟡 基础级 Level 2：统计出现次数

### 题目 3：AC 自动机加强版（洛谷 P3796）

> 给定 n（≤ 150）个模式串和一个文本串，求出现次数最多的模式串（可能有多个），输出出现次数和对应模式串。

**分析**：需要统计每个模式串的出现次数（而非仅判断是否出现）。匹配时沿 fail 链跳到所有标记节点，对每个模式串累加计数。由于 n ≤ 150 且串较短，暴力跳 fail 链不会超时。

```
数据结构：AC 自动机（Trie + fail 指针）
支持操作：统计每个模式串的出现次数
关键性质：沿 fail 链暴力跳可收集所有后缀匹配的模式串
时间复杂度：O(n × L + |T| × L)，L 为模式串最大长度
```

**C++ 代码**：

```cpp
const int MAXN = 100005;
int ch[MAXN][26], fail[MAXN], val[MAXN], tot;
int cnt[155]; // cnt[i] = 第 i 个模式串出现次数

void query(char t[]) {
    int u = 0;
    for (int i = 0; t[i]; i++) {
        u = ch[u][t[i] - 'a'];
        for (int j = u; j; j = fail[j])
            if (val[j]) cnt[val[j]]++;
    }
    int mx = *max_element(cnt + 1, cnt + n + 1);
    printf("%d\n", mx);
    for (int i = 1; i <= n; i++)
        if (cnt[i] == mx) puts(pat[i]); // pat[i] 为原模式串
}
```

**核心启示**：
- **val 记录模式串编号，cnt 按编号累加**是统计每个模式串出现次数的标准做法
- 当模式串数量和长度有限时，**暴力跳 fail 链**的复杂度可控

### 题目 4：单词（洛谷 P3966 [TJOI2013]）

> 给定 n 个单词，所有单词拼接成一篇文章（相邻单词间无分隔），求每个单词在文章中出现的次数。

**分析**：特殊之处在于文本就是所有模式串的拼接。将 n 个单词插入 Trie 并建 fail 指针后，用拼接后的文本在 AC 自动机上匹配。匹配时只在经过的节点 +1，最后用 fail 树拓扑排序上传，得到每个模式串末尾节点的出现次数。

```
数据结构：AC 自动机 + fail 树拓扑排序
支持操作：统计每个模式串在文章（所有模式串拼接）中的出现次数
关键性质：fail 树上拓扑上传等价于子树求和，避免暴力跳 fail 链
时间复杂度：O(m)，m 为所有单词总长
```

**C++ 代码**：

```cpp
const int MAXN = 2000005;
int ch[MAXN][26], fail[MAXN], cnt[MAXN], endp[205], tot;
int que[MAXN], hd, tl;

// 插入第 i 个单词后 endp[i] = 末尾节点编号
// 匹配阶段：将拼接文本逐字符走自动机，经过节点 cnt[u]++
// 拓扑上传：
void solve() {
    // 匹配
    int u = 0;
    for (int i = 0; text[i]; i++) {
        u = ch[u][text[i] - 'a'];
        cnt[u]++;
    }
    // 拓扑排序（BFS 逆序）上传
    for (int i = tl - 1; i >= 0; i--)
        cnt[fail[que[i]]] += cnt[que[i]];
    // cnt[endp[i]] 即为第 i 个单词的出现次数
}
```

**核心启示**：
- **文本就是模式串拼接**时，"模式串同时也是文本"是本题的核心特点
- **拓扑上传代替暴力跳 fail 链**是从"加强版"到"二次加强版"的关键优化思路

### 题目 5：String Set Queries（CF710F）

> 维护一个字符串集合 S，支持三种操作：① 插入字符串 ② 删除字符串 ③ 给定文本串 T，求 S 中所有串在 T 中出现的总次数。操作总数 ≤ 3×10⁵，所有字符串总长 ≤ 3×10⁵。

**分析**：AC 自动机不支持高效的动态插入/删除。关键观察：**每次查询时重建 AC 自动机**。由于每个字符最多被插入和删除各一次，总重建代价 O(总长) 均摊。查询时在当前 AC 自动机上匹配文本串。

```
数据结构：AC 自动机（按需重建）
支持操作：动态维护模式串集合 + 查询匹配总次数
关键性质：每个字符最多插入/删除各一次，重建总代价均摊 O(总长)
时间复杂度：O(m + n) 均摊每次查询，m 为当前模式串总长，n 为文本长度
```

**C++ 代码**：

```cpp
set<string> S; // 当前活跃的模式串集合
bool dirty;     // 标记是否需要重建

void addStr(const string &s) { S.insert(s); dirty = true; }
void delStr(const string &s) { S.erase(s); dirty = true; }

long long query(char t[]) {
    if (dirty) {
        // 清空 Trie，将 S 中所有串重新 insert + build
        rebuildAC(S);
        dirty = false;
    }
    int u = 0;
    long long ans = 0;
    for (int i = 0; t[i]; i++) {
        u = ch[u][t[i] - 'a'];
        for (int j = u; j; j = fail[j])
            ans += val[j];
    }
    return ans;
}
```

**核心启示**：
- **AC 自动机不支持动态修改 → 按需全量重建**，均摊分析保证总复杂度
- **"暴力重建"思想**在很多离线/半在线数据结构问题中都能奏效

### 🟠 提高级 Level 3：优化与变体

### 题目 6：AC 自动机二次加强版（洛谷 P5357）

> 给定 n 个模式串和一个文本串，求每个模式串在文本串中各出现了多少次。n ≤ 2×10⁵，模式串总长 ≤ 2×10⁵，文本串长度 ≤ 2×10⁶。

**分析**：与 P3796 不同，模式串数量和长度都很大，暴力跳 fail 链会 TLE。正确做法是：匹配时只在经过的节点上 +1（O(|T|)），匹配结束后在 fail 树上做**拓扑排序从叶到根上传 cnt**，使每个节点的 cnt 包含所有后代的贡献。注意可能有重复模式串，需用末尾节点去重。

```
数据结构：AC 自动机 + fail 树拓扑排序
支持操作：精确统计每个模式串出现次数
关键性质：fail 树上 cnt 上传 = 子树和；避免暴力跳 fail 的 O(n√m) 退化
时间复杂度：O(|T| + m)，m 为模式串总长
```

**C++ 代码**：

```cpp
const int MAXN = 2000005;
int ch[MAXN][26], fail[MAXN], cnt[MAXN], endp[MAXN], tot;
int que[MAXN], in_deg[MAXN];

void solve(char t[], int n) {
    // 匹配：只记录经过节点
    int u = 0;
    for (int i = 0; t[i]; i++) {
        u = ch[u][t[i] - 'a'];
        cnt[u]++;
    }
    // fail 树拓扑排序上传
    for (int i = 1; i <= tot; i++) in_deg[fail[i]]++;
    int hd = 0, tl = 0;
    for (int i = 1; i <= tot; i++)
        if (!in_deg[i]) que[tl++] = i;
    while (hd < tl) {
        int u = que[hd++];
        cnt[fail[u]] += cnt[u];
        if (--in_deg[fail[u]] == 0 && fail[u])
            que[tl++] = fail[u];
    }
    for (int i = 1; i <= n; i++) printf("%d\n", cnt[endp[i]]);
}
```

**核心启示**：
- **拓扑排序上传是 AC 自动机统计出现次数的标准优化**，将匹配与统计解耦
- 也可以用 BFS 逆序代替入度拓扑——build 阶段的 BFS 序天然满足"父在子前"

### 题目 7：Censoring Gold（洛谷 P3121 [USACO15FEB]）

> 给定文本串 S 和 n 个模式串。反复删除 S 中最早出现的模式串，删除后剩余部分拼接继续匹配，直到没有模式串出现。输出最终剩余串。

**分析**：经典的"AC 自动机 + 栈"问题。维护一个栈，栈中每个元素保存当前字符和对应的 AC 自动机状态。逐字符入栈并走自动机，当某个节点命中模式串时，弹出该模式串长度个元素，回退到栈顶的自动机状态继续匹配。

```
数据结构：AC 自动机 + 栈（保存字符与状态）
支持操作：删除匹配后继续匹配（类似括号消除）
关键性质：栈保存历史状态，弹出后无需重新匹配；每个字符最多入栈出栈一次
时间复杂度：O(|S| + m)，m 为模式串总长
```

**C++ 代码**：

```cpp
const int MAXN = 100005;
int ch[MAXN][26], fail[MAXN], len_at[MAXN], tot; // len_at[u] = u 处匹配到的模式串长度
int stk_state[MAXN], top; // 栈中保存的自动机状态
char stk_ch[MAXN];        // 栈中保存的字符

void solve(char s[]) {
    top = 0;
    stk_state[0] = 0; // 初始状态
    for (int i = 0; s[i]; i++) {
        int c = s[i] - 'a';
        int u = ch[stk_state[top]][c];
        stk_ch[++top] = s[i];
        stk_state[top] = u;
        if (len_at[u]) top -= len_at[u]; // 弹出匹配串长度
    }
    stk_ch[top + 1] = '\0';
    // stk_ch[1..top] 即为最终结果
}
```

**核心启示**：
- **AC 自动机 + 栈**是"删除后继续匹配"问题的标准模型，与 KMP + 栈（P4824）思路相同
- **栈保存自动机状态**，弹出后从栈顶状态继续，避免从头重新匹配

**分析**：将字典中的单词建 AC 自动机，然后对文本串做 DP。设 `f[i]` 表示前缀 S[1..i] 能否被拼接。对于每个位置 i，若 `f[i]` 为真，从 i+1 开始在 AC 自动机上匹配，匹配到模式串末尾时令 `f[i + len] = true`。由于单词最长 20，每个位置最多匹配 20 步。

```
数据结构：AC 自动机 + 布尔 DP
支持操作：判定前缀能否被字典单词拼接
关键性质：单词长度 ≤ 20，每个可达位置向后最多走 20 步
```

```cpp
const int MAXN = 2000005;

bool f[MAXN];      // f[i] = S[1..i] 能否被拼接

void solve(char s[], int len) {
    memset(f, 0, sizeof(bool) * (len + 1));
    f[0] = true;

    for (int i = 0; i < len; i++) {

            if (isEnd[u]) f[j + 1] = true;

    printf("%d\n", ans);
}
```

**核心启示**：
- **AC 自动机 + DP 拼接判定**是"字典分词"问题的经典方案

### 题目 9：阿狸的打字机（洛谷 P2414 [NOI2011]）

```
数据结构：AC 自动机 fail 树 + DFS 序 + 树状数组
支持操作：离线查询"串 x 在串 y 中出现的次数"
关键性质：fail 树上 x 的子树 DFS 序区间 ∩ y 路径上的节点 → 区间计数
时间复杂度：O(n log n + m)，n 为操作长度，m 为询问数
```

**C++ 代码**：

```cpp
const int MAXN = 100005;

int dfn[MAXN], sz[MAXN], dfn_cnt; // fail 树 DFS 序
int bit[MAXN];                     // 树状数组
vector<pair<int,int>> qry[MAXN];   // qry[y] = {(x, 询问编号)}
int ans[MAXN];

// 在 fail 树上求 DFS 序：dfn[u], sz[u]
// 将询问离线挂在 y 上
// 在 Trie 上重新 DFS（模拟操作序列）：
//   进入节点 u → BIT 上 dfn[u] 位置 +1
//   离开节点 u → BIT 上 dfn[u] 位置 -1

//     ans[id] = BIT.query(dfn[endp[x]] + sz[endp[x]] - 1) - BIT.query(dfn[endp[x]] - 1)
```

**核心启示**：
- **"串 x 在串 y 中出现次数" = fail 树子树内 y 路径节点的计数**，这是 AC 自动机 fail 树最经典的转化

### 题目 10：病毒（洛谷 P2444 [POI2000]）

> 给定 n 个 01 模式串（病毒代码），问是否存在一个无限长的 01 串不包含任何模式串。

**分析**：将模式串建 AC 自动机。标记所有"危险节点"——自身是模式串末尾，或 fail 链上有模式串末尾的节点。问题转化为：**在 AC 自动机的 Trie 图上，从根出发，是否存在一条只经过安全节点的无限长路径**。等价于：安全节点构成的子图中是否存在**从根可达的环**。

```
数据结构：AC 自动机 Trie 图 + 图论（环检测）
支持操作：判定是否存在不含任何模式串的无限长串
关键性质：安全节点子图中有从根可达的环 ⟺ 存在安全无限串
时间复杂度：O(m × |Σ|)，m 为模式串总长
```

**C++ 代码**：

```cpp
const int MAXN = 30005;
int ch[MAXN][2], fail[MAXN], tot;
bool danger[MAXN]; // 危险节点标记
int vis[MAXN];     // 0=未访问, 1=栈中, 2=已离栈

void markDanger() {
    // BFS 建 fail 后，传播危险标记
    for (int i = 0; i < tl; i++) {
        int u = que[i];
        if (danger[fail[u]]) danger[u] = true;
    }
}

bool dfs(int u) {
    vis[u] = 1;
    for (int c = 0; c < 2; c++) {
        int v = ch[u][c];
        if (danger[v]) continue;
        if (vis[v] == 1) return true;  // 找到环
        if (!vis[v] && dfs(v)) return true;
    }
    vis[u] = 2;
    return false;
}
// 调用 dfs(0)，返回 true 则输出 "TAK"，否则 "NIE"
```

**核心启示**：
- **"是否存在不含模式串的无限串" = AC 自动机安全子图上是否有可达环**
- **危险标记需从 fail 链传播**——如果 fail 链上有终止节点，自身也是危险的

### 题目 11：文本生成器（洛谷 P4052 [JSOI2007]）

> 给定 n 个模式串，求长度恰好为 m 的、至少包含一个模式串的字符串数目（对 10007 取模）。字符集 {A,C,G,T}。

**分析**：正难则反。总方案数 4^m，减去"不包含任何模式串"的方案数。建 AC 自动机，标记所有危险节点（模式串末尾及其 fail 链传播），在安全节点上做 DP。设 `dp[i][u]` = 走了 i 步到达安全节点 u 的方案数。

```
数据结构：AC 自动机 DFA + DP（正难则反）
支持操作：计数长度为 m 的至少含一个模式串的字符串数
关键性质：不含模式串 = 只走安全节点；答案 = 4^m − ∑dp[m][u]
时间复杂度：O(m × tot × |Σ|)
```

**C++ 代码**：

```cpp
const int MAXN = 10005, MOD = 10007;
int ch[MAXN][4], fail[MAXN], tot;
bool danger[MAXN];
int dp[105][MAXN]; // dp[步数][节点]

void solve(int m) {
    // 标记危险节点（模式串末尾 + fail 链传播）
    dp[0][0] = 1;
    for (int i = 0; i < m; i++)
        for (int u = 0; u <= tot; u++) {
            if (!dp[i][u] || danger[u]) continue;
            for (int c = 0; c < 4; c++) {
                int v = ch[u][c];
                if (!danger[v])
                    dp[i + 1][v] = (dp[i + 1][v] + dp[i][u]) % MOD;
            }
        }
    int safe = 0;
    for (int u = 0; u <= tot; u++)
        safe = (safe + dp[m][u]) % MOD;
    int total = 1;
    for (int i = 0; i < m; i++) total = total * 4 % MOD;
    printf("%d\n", (total - safe + MOD) % MOD);
}
```

**核心启示**：
- **正难则反**是 AC 自动机计数 DP 的经典策略——"至少包含一个"反转为"全部不包含"
- **AC 自动机上的 DP**：节点 = 状态，字符 = 转移，危险节点 = 不可达

```
支持操作：查询模式串在 Trie 路径串中的出现次数
```

```cpp
// AC 自动机建在所有查询的模式串上

int acState[MAXN]; // Trie 节点 v 对应的 AC 自动机状态

void dfsTrie(int v) {
```

### 题目 14：文本编辑器（洛谷 P5599 [XR-4]）

> 给定一个长度为 n 的文本串和 m 个模式串。可以对文本串执行若干次"选中一段连续子串并替换为任意等长字符串"的操作。求最少修改多少个字符，使得文本串不包含任何模式串。

**分析**：将模式串建 AC 自动机，标记危险节点。问题转化为：在文本串上找到所有匹配到模式串的区间，用最少的修改点覆盖所有这些区间（每修改一个字符可以破坏所有经过该位置的匹配）。这是一个贪心/DP 问题：扫描文本串，维护当前匹配到的最右端点，贪心地选择修改最右端位置。

```
数据结构：AC 自动机 + 贪心覆盖
支持操作：最少修改字符使文本不含任何模式串
关键性质：匹配区间的最少点覆盖 → 贪心取右端点
```

**C++ 代码**：

```cpp
const int MAXN = 200005;
int ch[MAXN][26], fail[MAXN], minLen[MAXN], tot;
// minLen[u] = 到达 u 时能匹配到的最短模式串长度（沿 fail 链取 min）

void solve(char t[], int n) {
    // 匹配 + 贪心
    int u = 0, ans = 0, last_modify = -1;
    for (int i = 0; i < n; i++) {
        u = ch[u][t[i] - 'a'];
        if (minLen[u]) {
            int L = i - minLen[u] + 1; // 匹配区间左端
            if (L > last_modify) {     // 上次修改未覆盖
                ans++;
                last_modify = i;       // 贪心修改最右端
                u = 0;                 // 重置自动机状态
            }
        }
    }
    printf("%d\n", ans);
}
```

**核心启示**：
- **AC 自动机匹配 + 区间覆盖贪心**是"最少修改消除匹配"的标准思路
- **minLen 沿 fail 链传播**确保能检测到最短匹配，贪心修改覆盖范围最大

### 题目 15：Genetic Engineering（CF86C）

> 给定 m 个基因片段（长度 ≤ 10），构造长度恰好为 n 的 DNA 串（字符集 ACGT），要求每个位置都被至少一个基因片段覆盖（即串的每个位置都属于某个基因片段的某次匹配）。求方案数。

**分析**：将基因片段建 AC 自动机。DP 状态需记录：当前位置 i、AC 自动机状态 u、**末尾连续多少个位置已被覆盖**（由于最长片段 ≤ 10，只需记录最后 10 个位置的覆盖情况，可用 bitmask 或简化为"最后未覆盖的距离"）。转移时，若在当前位置匹配到长度为 L 的模式串，则最后 L 个位置都被覆盖。

```
数据结构：AC 自动机 + 状压 DP
支持操作：计数所有位置都被基因片段覆盖的方案数
关键性质：模式串长度 ≤ 10 → 覆盖状态可压缩为 10 位 bitmask
时间复杂度：O(n × tot × 2^L × |Σ|)，L 为最长片段长度
```

**C++ 代码**：

```cpp
const int MAXN = 1005, MAXT = 105;
int ch[MAXT][4], fail[MAXT], tot;
int matchLen[MAXT]; // matchLen[u] 为节点 u 处匹配到的最长模式串长度（含 fail 链）
long long dp[MAXN][MAXT][11];
// dp[i][u][j] = 前 i 位，自动机状态 u，最后 j 个未覆盖位置

void solve(int n) {
    dp[0][0][0] = 1;
    for (int i = 0; i < n; i++)
        for (int u = 0; u <= tot; u++)
            for (int j = 0; j <= 10; j++) {
                if (!dp[i][u][j]) continue;
                for (int c = 0; c < 4; c++) {
                    int v = ch[u][c];
                    int nj = (matchLen[v] > j + 1) ? 0 : j + 1;
                    if (nj <= 10)
                        dp[i + 1][v][nj] += dp[i][u][j];
                }
            }
    long long ans = 0;
    for (int u = 0; u <= tot; u++) ans += dp[n][u][0];
    printf("%lld\n", ans);
}
```

**核心启示**：
- **"每个位置都被覆盖"需要记录覆盖缺口**，模式串长度有限时可压缩状态
- **AC 自动机 + 状压 DP** 是处理"覆盖约束"类多模式问题的有力工具

### 题目 16：Substring（UVA11468）

> 给定 n 个模式串和每个字符的出现概率，随机生成长度为 L 的字符串，求该串不包含任何模式串的概率。

**分析**：将模式串建 AC 自动机，标记危险节点。概率 DP：设 `dp[i][u]` = 走了 i 步到达安全节点 u 的概率。转移时按字符概率加权。最终答案 = ∑dp[L][u]（所有安全节点）。

```
数据结构：AC 自动机 + 概率 DP
支持操作：计算随机串不含任何模式串的概率
关键性质：AC 自动机上概率转移，只走安全节点
时间复杂度：O(L × tot × |Σ|)
```

**C++ 代码**：

```cpp
const int MAXN = 505;
int ch[MAXN][26], fail[MAXN], tot;
bool danger[MAXN];
double prob[26];   // prob[c] = 字符 c 的出现概率
double dp[105][MAXN]; // dp[步数][节点]

double solve(int L) {
    dp[0][0] = 1.0;
    for (int i = 0; i < L; i++)
        for (int u = 0; u <= tot; u++) {
            if (dp[i][u] < 1e-15 || danger[u]) continue;
            for (int c = 0; c < 26; c++) {
                int v = ch[u][c];
                if (!danger[v])
                    dp[i + 1][v] += dp[i][u] * prob[c];
            }
        }
    double ans = 0;
    for (int u = 0; u <= tot; u++)
        if (!danger[u]) ans += dp[L][u];
    return ans;
}
```

**核心启示**：
- **概率 DP 与计数 DP 框架完全一致**，只是把"方案数累加"变成"概率累加"
- **AC 自动机 + 概率 DP** 是处理"随机串匹配概率"的标准模型

## 五、常见技巧与思路总结

### 5.1 Trie 图补边——消除 fail 链跳转

构建 fail 时，若 `ch[u][c]` 不存在则令 `ch[u][c] = ch[fail[u]][c]`。补边后整个 Trie 变成完整 DFA，匹配时每步 O(1)，不再需要循环跳 fail。

### 5.2 危险标记沿 fail 链传播

若节点 u 是某个模式串的末尾，则 fail 链上所有以 u 为后缀的节点也应标记为危险。在 BFS 建 fail 时同步传播：

```cpp
if (isEnd[fail[u]]) isEnd[u] = true; // 或用 val 合并
```

### 5.3 拓扑排序代替暴力跳 fail

统计每个模式串出现次数时，匹配阶段只在经过节点上 +1，匹配后用 fail 树拓扑排序上传——将 O(n × 树高) 优化为 O(n + m)。

### 5.4 fail 树子树和 → DFS 序 + 树状数组

"串 x 在串 y 中出现几次" = y 路径上有多少节点落在 x 末尾节点的 fail 树子树中。离线：Trie 上 DFS 时维护树状数组，进入/离开节点时 +1/−1。

### 5.5 AC 自动机 + 栈回退

"匹配后删除"类问题：用栈保存每个字符和对应的自动机状态。匹配成功时弹出 L 个栈顶元素，从新栈顶的状态继续转移。

### 5.6 正难则反

"至少包含一个模式串的方案数" = 总方案数 − "不含任何模式串的方案数"。后者在 AC 自动机上只走安全节点即可 DP。

## 七、终极总结与归纳

### 7.1 AC 自动机的本质

**AC 自动机的本质是将多个模式串的匹配信息融合到一个确定性有穷自动机中。** Trie 组织了多个模式串的前缀结构，fail 指针提供了失配时的最优跳转（类比 KMP 的 next 数组），Trie 图补边则将整个结构变成 DFA，使得文本的每一步转移都是 O(1)。fail 指针构成的 fail 树进一步揭示了模式串之间的后缀包含关系，是统计和离线查询的核心工具。

### 7.2 核心要点

1. **Trie + fail = AC 自动机**：Trie 组织前缀，fail 连接后缀，两者结合完成多模匹配
2. **Trie 图补边**：`ch[u][c] = ch[fail[u]][c]`（当不存在时），将 Trie 变成 DFA
3. **拓扑排序上传**：统计出现次数时不暴力跳 fail，匹配后在 fail 树上一次性上传
4. **fail 树子树 = 后缀包含关系**：fail 树上 u 的子树中的节点，其代表串都以 u 的代表串为后缀
5. **AC + DP**：自动机节点作为 DP 状态，安全/危险节点控制转移可达性
6. **正难则反**：计数"至少含一个模式串" → 总方案 − 全部不含
