## 一、什么是后缀自动机

**后缀自动机**（Suffix Automaton，简称 SAM）是一种能在 **O(n)** 时间和空间内，压缩表示一个字符串 **所有子串** 信息的有限状态自动机。它是接受字符串 S 的全部子串（且仅接受这些子串）的**最小确定有穷自动机（DFA）**。

SAM 的核心思想是将 **endpos 集合相同的子串** 归为同一个等价类，用一个状态（节点）来表示。这样，一个长度为 n 的字符串最多只有 **2n − 1** 个状态和 **3n − 4** 条转移边，却能表达所有 O(n²) 个子串的信息——这是 SAM 高效的根本原因。

SAM 的每个状态通过 **suffix link（后缀链接）** 连接成一棵树，称为 **parent tree**。这棵树与原串反串的后缀树同构，是 SAM 上进行 DP 和统计的核心结构。

```text
SAM for s = "aab":

DAG（转移边）:           Parent Tree（后缀链接）:
      a       a       b
(0) ──→ (1) ──→ (2) ──→ (3)       (0)
 │        │              ↑         ├── (1) ── len=1, "a"
 │        └──────b───────┘         │    └── (2) ── len=2, "aa"
 └───────────b───────────┘         └── (3) ── len=3, "b","ab","aab"

状态 0：初始态 ε
状态 1：endpos={1,2}，代表 "a"（出现 2 次）
状态 2：endpos={2}，  代表 "aa"（出现 1 次）
状态 3：endpos={3}，  代表 "b","ab","aab"（各出现 1 次）
```

| 特性 | 后缀自动机 SAM | 后缀数组 SA | 后缀树 ST |
|------|---------------|------------|----------|
| 构建时间 | O(n) | O(n log n) 或 O(n) | O(n) |
| 空间 | O(n\|Σ\|) | O(n) | O(n\|Σ\|) |
| 子串判定 | ✅ 在线 O(\|t\|) | 二分 O(\|t\| log n) | O(\|t\|) |
| endpos 信息 | parent tree 直接给出 | 需额外处理 | height 数组 |
| 在线构建 | ✅ | ❌ | 有限支持 |
| 多串扩展 | 广义 SAM | 拼接+分隔符 | 广义后缀树 |

## 二、核心思想与前置知识

### 2.1 endpos 等价类

对于字符串 S 的子串 t，定义 **endpos(t)** 为 t 在 S 中所有出现位置的**结束下标集合**。

**关键性质**：

1. **后缀包含性**：若 a 是 b 的后缀，则 endpos(b) ⊆ endpos(a)；若 a 不是 b 的后缀，则 endpos(a) ∩ endpos(b) = ∅
2. **等价类长度连续**：同一等价类中所有子串的长度构成连续区间 `[minLen, maxLen]`，其中 `minLen = len[link[u]] + 1`，`maxLen = len[u]`
3. **等价类数量线性**：不同的 endpos 等价类最多 **2n − 1** 个

每个 SAM 状态 u 代表一个等价类，包含所有 endpos 相同的子串。该状态的 `len[u]` 存储等价类中**最长子串**的长度。

### 2.2 suffix link 与 parent tree

每个状态 u 的 **suffix link**（记为 `link[u]`）指向：u 所代表的最长子串的、**不属于同一等价类的最长后缀** 所在的状态。

suffix link 构成一棵以初始状态为根的树，称为 **parent tree**。核心性质：

- 从叶到根，endpos 集合**严格递增**（包含关系）
- `endpos(u) ⊂ endpos(link[u])`
- parent tree 与 **S 的反串的后缀树** 同构
- 每个节点 u 新增的子串数 = `len[u] − len[link[u]]`

### 2.3 O(n) 在线构造算法

SAM 采用增量构造：逐个添加字符，每次 O(1) 均摊。添加字符 c 时，核心流程为——创建新状态 `cur`，沿 `last` 的 suffix link 链向上跳，补充缺失的 c 转移；若遇到已有 c 转移的状态，判断是否需要**克隆（clone）**节点以维持最小性。

```cpp
const int MAXN = 2000005;
struct State {
    int len, link, siz;
    int ch[26];
} sam[MAXN];
int sz = 1, last = 1; // 1 号为初始状态

void extend(int c) {
    int cur = ++sz, p = last;
    sam[cur].len = sam[p].len + 1;
    sam[cur].siz = 1;
    for (; p && !sam[p].ch[c]; p = sam[p].link)
        sam[p].ch[c] = cur;
    if (!p) {
        sam[cur].link = 1;
    } else {
        int q = sam[p].ch[c];
        if (sam[q].len == sam[p].len + 1) {
            sam[cur].link = q;
        } else {
            int cl = ++sz;
            sam[cl] = sam[q];
            sam[cl].len = sam[p].len + 1;
            sam[cl].siz = 0;
            for (; p && sam[p].ch[c] == q; p = sam[p].link)
                sam[p].ch[c] = cl;
            sam[q].link = sam[cur].link = cl;
        }
    }
    last = cur;
}
```

> 三种情况：① 跳到根仍无 c 转移 → link 指向根；② 找到转移且 `len[q] == len[p]+1` → 直接连；③ 否则 clone 节点。

### 2.4 基本统计量

**拓扑排序 + siz 上传**：按 `len` 降序处理，将每个状态的 `siz` 累加到其 parent。

```cpp
int ord[MAXN], buc[MAXN];
void topoCount() {
    for (int i = 1; i <= sz; i++) buc[sam[i].len]++;
    for (int i = 1; i <= sz; i++) buc[i] += buc[i - 1];
    for (int i = sz; i >= 1; i--) ord[buc[sam[i].len]--] = i;
    for (int i = sz; i >= 1; i--)
        sam[sam[ord[i]].link].siz += sam[ord[i]].siz;
}
```

| 统计量 | 公式 |
|--------|------|
| 本质不同子串数 | `∑(len[u] − len[link[u]])` |
| 子串 t 的出现次数 | `siz[u]`（t 所在状态 u） |
| 所有子串出现次数之和 | `∑ siz[u] × (len[u] − len[link[u]])` |

## 三、分类与变体详解

### 3.1 基础 SAM 构建与子串统计

最基础的 SAM 应用：构建后直接利用状态信息回答子串相关问题。

```
数据结构：后缀自动机（单串 SAM）
支持操作：子串判定、子串计数、出现次数统计
关键性质：每个状态对应一个 endpos 等价类，len 差给出新增子串数
时间复杂度：O(n) 构建，O(1) 或 O(n) 统计
```

```cpp
// 本质不同子串数
long long distinctSub() {
    long long res = 0;
    for (int i = 2; i <= sz; i++)
        res += sam[i].len - sam[sam[i].link].len;
    return res;
}
```

### 3.2 parent 树与 endpos 集合

parent tree 是 SAM 上做 DP 的核心。常见操作：

- **siz 上传**：拓扑排序后从子到父累加，得到每个状态的出现次数
- **子树贡献统计**：对 parent tree 做树形 DP，统计节点对答案的贡献
- **endpos 集合维护**：在 parent tree 上用线段树合并维护精确的 endpos 集合

```
数据结构：parent tree（suffix link 树）
支持操作：出现次数统计、LCA/LCP 计算、子树贡献 DP
关键性质：parent tree 与反串后缀树同构；endpos 集合自下而上严格递增
时间复杂度：O(n) 基本 DP，O(n log n) 线段树合并
```

### 3.3 广义后缀自动机（多串 SAM）

将多个字符串的子串信息合并到同一个 SAM 中。常见构建方式：每插入新串前将 `last` 重置为初始状态。

```cpp
// 广义 SAM：对每个串 s_i，重置 last 后逐字符 extend
void buildGSAM(vector<string>& strs) {
    for (auto& s : strs) {
        last = 1;
        for (char c : s) extend(c - 'a');
    }
}
```

注意：广义 SAM 的 extend 需要特判 `sam[last].ch[c]` 已存在且 `len` 恰好等于 `len[last]+1` 的情况，此时无需创建新节点，直接跳转即可。

```
数据结构：广义后缀自动机
支持操作：多串子串统计、公共子串查询、多串 endpos 标记
关键性质：节点数仍为 O(∑|s_i|)；需要额外标记每个状态属于哪些串
时间复杂度：O(∑|s_i|) 构建
```

### 3.4 SAM + 线段树合并

当需要精确维护每个状态的 endpos 集合（而非仅知大小）时，在 parent tree 上进行**线段树合并**。

典型流程：
1. 每个原始状态（非 clone）建一棵只含一个叶子的权值线段树
2. 按拓扑序从子到父合并线段树
3. 合并后可查询任意状态的 endpos 集合中是否有元素落在某区间 `[l, r]`

```
数据结构：SAM + 权值线段树（动态开点）
支持操作：endpos 集合区间查询、区间出现判定
关键性质：合并后每个状态的线段树表示其精确 endpos 集合
时间复杂度：O(n log n) 建树 + 合并
```

## 四、16 道经典例题详解

### 🟢 入门级 Level 1：SAM 构建与基础统计

### 题目 1：后缀自动机模板（洛谷 P3804）

> 给定字符串 S，求所有出现次数大于 1 的子串中，`出现次数 × 长度` 的最大值。

**分析**：建 SAM，用拓扑排序将 siz 从子节点上传到 parent。对所有 `siz[u] > 1` 的状态，取 `len[u] × siz[u]` 的最大值。

```
数据结构：SAM + parent tree siz 上传
支持操作：统计每个子串的出现次数
关键性质：siz[u] = endpos(u) 的大小 = 该等价类中子串的出现次数
时间复杂度：O(n)
```

**C++ 代码**：
```cpp
// 建 SAM 后调用 topoCount()
long long ans = 0;
for (int i = 2; i <= sz; i++)
    if (sam[i].siz > 1)
        ans = max(ans, (long long)sam[i].len * sam[i].siz);
```

**核心启示**：
- **拓扑排序上传 siz** 是 SAM 最基础的操作，几乎所有 SAM 题都用到
- **等价类中的最长子串** `len[u]` 贡献最大价值，无需枚举等价类内所有长度

### 🟡 基础级 Level 2：SAM 上的匹配与遍历

### 题目 3：弦论（洛谷 P3975）

> 给定字符串 S 和参数 t、k。`t=0` 时不同位置的相同子串只算一个，`t=1` 时算多个。求字典序第 k 小的子串。

**分析**：在 SAM 的 DAG 上按字典序 DFS。每个状态的"权重"：`t=0` 时为 1，`t=1` 时为 `siz[u]`。预处理每个状态的子树权重和 `sum[u]`，在 DAG 上贪心选择字符。

```
数据结构：SAM DAG + 子树权重 DP
支持操作：第 k 小子串查询
关键性质：DAG 按字符顺序遍历即为字典序；子树 sum 可快速跳过
时间复杂度：O(n|Σ|) 预处理，O(|ans|) 查询
```

**C++ 代码**：
```cpp
long long sum[MAXN]; // sum[u] = u 子树（DAG 意义）的权重和
// t=0 时 wt[u]=1（u≠初始态），t=1 时 wt[u]=siz[u]
// 按 len 降序 DP：sum[u] = wt[u] + ∑ sum[ch[u][c]]

void kth(int u, long long k) {
    if (t == 1) k -= siz[u]; // 当前状态本身贡献 siz[u] 个子串
    else if (u != 1) k--;     // 当前状态贡献 1 个子串
    if (k <= 0) return;
    for (int c = 0; c < 26; c++) {
        int v = sam[u].ch[c];
        if (!v) continue;
        if (k <= sum[v]) { putchar('a' + c); kth(v, k); return; }
        k -= sum[v];
    }
}
// 调用 kth(1, k)
```

**核心启示**：
- **SAM 的 DAG 结构**天然按字典序组织子串，配合子树权重实现 O(|ans|) 查询
- `t=0/1` 的切换只需改变权重定义，框架不变

### 题目 4：生成魔咒（洛谷 P4070）

> 依次向字符串末尾添加字符，每次添加后输出当前本质不同子串数。

**分析**：在线构建 SAM。每次 extend 一个字符后，新增的本质不同子串数恰好是 `len[last] − len[link[last]]`，累加即可。

```
数据结构：在线 SAM
支持操作：动态维护本质不同子串数
关键性质：每次 extend 新增子串数 = len[last] − len[link[last]]
时间复杂度：O(n) 总计
```

**C++ 代码**：
```cpp
// 注意：字符集可能很大，需用 map 代替数组
struct State {
    int len, link, siz;
    map<int, int> ch;
} sam[MAXN];

long long ans = 0;
for (int i = 0; i < n; i++) {
    extend(a[i]);
    ans += sam[last].len - sam[sam[last].link].len;
    printf("%lld\n", ans);
}
```

**核心启示**：
- **SAM 的在线性质**使其天然支持增量式子串统计
- 大字符集时用 `map` 替换数组，时间变为 O(n log |Σ|)

### 题目 5：Cyclical Quest（CF235C）

> 给定文本串 s 和查询串 t，求 t 的所有循环移位在 s 中的出现次数之和（相同循环移位只算一次）。

**分析**：建 s 的 SAM。将 t 倍长为 `t + t`，在 SAM 上做滑动窗口匹配：窗口大小固定为 `|t|`，每次右端加一个字符、左端移除一个字符。当匹配长度达到 `|t|` 时，当前状态的 siz 即为该循环移位的出现次数。用标记去重避免同一状态重复计数。

```
数据结构：SAM + 滑动窗口匹配
支持操作：循环子串匹配与计数
关键性质：滑动窗口维护当前匹配长度，超过 |t| 时沿 link 上跳
时间复杂度：O(|s| + |t|)
```

**C++ 代码**：
```cpp
long long query(char* t, int m) {
    long long res = 0;
    int cur = 1, ml = 0;
    for (int i = 0; i < 2 * m - 1; i++) {
        int c = t[i % m] - 'a';
        while (cur != 1 && !sam[cur].ch[c])
            cur = sam[cur].link, ml = sam[cur].len;
        if (sam[cur].ch[c]) cur = sam[cur].ch[c], ml++;
        if (ml > m) { // 窗口超长，缩短
            while (sam[sam[cur].link].len >= m)
                cur = sam[cur].link;
            ml = m;
        }
        if (ml == m && vis[cur] != qid) {
            vis[cur] = qid; // 去重标记
            res += sam[cur].siz;
        }
    }
    return res;
}
```

**核心启示**：
- **循环串 = 倍长 + 滑动窗口**是处理循环子串的通用技巧
- SAM 上沿 link 上跳相当于缩短匹配长度，对应"移除左端字符"

### 题目 6：最长公共子串（SP1811 LCS）

> 给定两个字符串 s 和 t，求它们的最长公共子串长度。

**分析**：建 s 的 SAM，然后让 t 在 SAM 上逐字符匹配。维护当前匹配长度 `ml`，匹配失败时沿 link 上跳，取全程 `ml` 的最大值。

```
数据结构：SAM + 字符串匹配
支持操作：两串最长公共子串
关键性质：SAM 上匹配失败时沿 link 跳转等价于尝试更短的后缀
时间复杂度：O(|s| + |t|)
```

**C++ 代码**：
```cpp
int lcs(char* t, int m) {
    int cur = 1, ml = 0, ans = 0;
    for (int i = 0; i < m; i++) {
        int c = t[i] - 'a';
        while (cur != 1 && !sam[cur].ch[c])
            cur = sam[cur].link, ml = sam[cur].len;
        if (sam[cur].ch[c]) cur = sam[cur].ch[c], ml++;
        ans = max(ans, ml);
    }
    return ans;
}
```

**核心启示**：
- **SAM 上的匹配过程**是后续所有"在 SAM 上跑另一个串"题目的基础模板
- 失败时沿 link 上跳，本质是在 endpos 等价类的包含链上回退

### 🟠 进阶级 Level 3：parent 树 DP 与多串 SAM

### 题目 7：NSUBSTR（SP8222）

> 给定字符串 S，对每个长度 L（1 ≤ L ≤ |S|），求长度为 L 的子串的最大出现次数。

**分析**：建 SAM，求出每个状态的 `siz`。每个状态 u 表示长度区间 `[len[link[u]]+1, len[u]]` 内的子串都出现了 `siz[u]` 次。用 `f[len[u]] = max(f[len[u]], siz[u])` 收集，再从大到小取后缀最大值 `f[i] = max(f[i], f[i+1])`。

```
数据结构：SAM + parent tree siz 上传
支持操作：按长度统计最大出现次数
关键性质：短子串出现次数 ≥ 长子串出现次数（后缀最大值传播）
时间复杂度：O(n)
```

**C++ 代码**：
```cpp
int f[MAXN];
// topoCount() 后
for (int i = 2; i <= sz; i++)
    f[sam[i].len] = max(f[sam[i].len], sam[i].siz);
for (int i = n - 1; i >= 1; i--)
    f[i] = max(f[i], f[i + 1]);
// f[L] 即长度 L 的子串的最大出现次数
```

**核心启示**：
- **按 len 收集再后缀传播**是 SAM 中"按长度统计"的经典手法
- 短子串出现次数不低于长子串，因此从长到短取 max 保证正确

### 题目 8：差异（洛谷 P4248）

> 给定字符串 S（长 n），T_i 为第 i 个后缀。求 ∑_{1≤i<j≤n} (|T_i| + |T_j| − 2·lcp(T_i, T_j))。

**分析**：第一部分 `∑(|T_i|+|T_j|)` = `(n−1)·n·(n+1)/2`。第二部分需求 `∑ lcp(T_i, T_j)`。在 SAM 的 parent tree 上，两个后缀的 LCP 等于其 LCA 的 `len`。遍历 parent tree，用合并子树大小的方式计算每个节点作为 LCA 的贡献。

```
数据结构：SAM parent tree + 子树合并统计
支持操作：所有后缀对的 LCP 求和
关键性质：parent tree 上 LCA 的 len = 两后缀的 LCP 长度
时间复杂度：O(n)
```

**C++ 代码**：
```cpp
// 建 SAM，topoCount() 后建 parent tree 的邻接表
vector<int> children[MAXN];
for (int i = 2; i <= sz; i++)
    children[sam[i].link].push_back(i);

long long lcpSum = 0;
// DFS：返回子树中后缀数量
int dfs(int u) {
    int tot = (u != 1 && sam[u].siz > 0) ? 0 : 0;
    // 实际 tot 初始 = 0，siz 已经算好
    tot = 0;
    for (int v : children[u]) {
        int sv = dfs(v);
        lcpSum += (long long)sam[u].len * tot * sv;
        tot += sv;
    }
    return tot + sam[u].siz - (u == 1 ? 0 : 0);
}
// 更简洁的方式：拓扑序处理
long long sumLcp = 0;
for (int i = sz; i >= 2; i--) {
    int u = ord[i], fa = sam[u].link;
    // 用类似启发式的方式在父节点处合并统计
}
```

更简洁的实现是利用拓扑序：对每个非根节点 u，在 parent 处贡献 `len[u] × siz[u] × (已合并的 siz 之和)`。

```cpp
long long sumLcp = 0;
// 建邻接表后 DFS
function<long long(int)> solve = [&](int u) -> long long {
    long long s = 0; // 已合并的后缀数
    for (int v : children[u]) {
        long long sv = solve(v);
        sumLcp += (long long)sam[u].len * s * sv;
        s += sv;
    }
    return s + (sam[u].siz > 0 && u != 1 ? sam[u].siz : sam[u].siz);
};
// 答案 = (n-1)*n*(n+1)/2 - 2*sumLcp
```

> 注：这里 `siz` 已含拓扑上传值。在 DFS 中，`s` 累计已访问子树的后缀数，每合并一个新子树时计算跨子树对的贡献。

**核心启示**：
- **parent tree 上的子树合并贡献统计**是 SAM 处理"所有后缀对"问题的标准方法
- 这道题本质等价于后缀数组的 height 数组求 LCP 之和，但 SAM 做法更直观

### 题目 9：Match & Catch（CF427D）

> 给定两个字符串 s 和 t，找一个最短的子串 w，使得 w 在 s 中恰好出现 1 次，在 t 中也恰好出现 1 次。输出 w 的长度。

**分析**：将 s 和 t 用分隔符拼接，建 SAM。对每个状态，分别统计其在 s 和 t 中的出现次数（通过标记原始位置属于哪个串）。找 `cntS[u] == 1 && cntT[u] == 1` 的状态，取 `len[link[u]] + 1` 的最小值（即该状态代表的最短子串）。

```
数据结构：拼接串 SAM + 分串标记
支持操作：两串中各出现恰好一次的最短子串
关键性质：拼接后每个状态可分别维护在各串中的出现次数
时间复杂度：O(|s| + |t|)
```

**C++ 代码**：
```cpp
// 拼接 s + '#' + t 建 SAM，标记每个原始位置属于哪个串
int cntS[MAXN], cntT[MAXN];
// 初始 extend 时：若位置 ≤ |s|-1 则 cntS[cur]=1，否则 cntT[cur]=1
// topoCount 后上传 cntS、cntT

int ans = INT_MAX;
for (int i = 2; i <= sz; i++)
    if (cntS[i] == 1 && cntT[i] == 1)
        ans = min(ans, sam[sam[i].link].len + 1);
// ans == INT_MAX 则无解，输出 -1
```

**核心启示**：
- **拼接 + 分标记**是处理双串/多串 SAM 问题的基本模式
- 最短满足条件的子串长度 = `len[link[u]] + 1`，因为等价类中最短子串在此处

**分析**：建 SAM，拓扑上传 siz。对每个状态 u，若 `siz[u] == k`，则长度区间 `[len[link[u]]+1, len[u]]` 的每个长度各有一个子串恰好出现 k 次。用差分数组记录区间 +1，最后扫描找最大值。

```
关键性质：同一等价类内所有子串出现次数相同

```

```cpp
int diff[MAXN]; // 差分数组
for (int i = 2; i <= sz; i++)

    s += diff[i];

}
```

**核心启示**：
- **差分数组处理等价类内连续长度区间**是 SAM 中"按长度统计"的常见优化

### 题目 11：回文串（洛谷 P3649）

> 给定字符串 S，求所有回文子串中 `出现次数 × 长度` 的最大值。

**分析**：使用**回文自动机（PAM / eertree）**——它与 SAM 共享"自动机 + link 树 + 拓扑统计"的思想体系。PAM 中每个节点表示一个本质不同的回文子串，`cnt[u]` 为其出现次数（需在 fail 树上从子到父上传）。答案为 `max(len[u] × cnt[u])`。

```
数据结构：回文自动机（PAM）+ fail 树统计
支持操作：回文子串出现次数统计
关键性质：PAM 节点数 ≤ n+2；fail 树上传 cnt 类似 SAM parent tree 上传 siz
时间复杂度：O(n)
```

**C++ 代码**：
```cpp
struct PAM {
    int len[MAXN], fail[MAXN], ch[MAXN][26];
    long long cnt[MAXN];
    int last, sz;
    char s[MAXN];
    int cur; // 当前串长度

    int getfail(int x) {
        while (s[cur - len[x] - 1] != s[cur]) x = fail[x];
        return x;
    }
    void extend(int c) {
        s[++cur] = c;
        int p = getfail(last);
        if (!ch[p][c]) {
            int q = ++sz;
            len[q] = len[p] + 2;
            fail[q] = ch[getfail(fail[p])][c];
            ch[p][c] = q;
        }
        last = ch[p][c];
        cnt[last]++;
    }
} pam;
// 构建后从后往前上传 cnt
long long ans = 0;
for (int i = pam.sz; i >= 2; i--) {
    pam.cnt[pam.fail[i]] += pam.cnt[i];
    ans = max(ans, (long long)pam.len[i] * pam.cnt[i]);
}
```

**核心启示**：
- **PAM 与 SAM 同属自动机家族**，共享 link/fail 树统计的方法论
- 回文问题优先考虑 PAM，但 SAM + hash 判回文也可行
- fail 树上传 cnt 与 SAM parent tree 上传 siz 是同一个思想

### 题目 12：Cool Slogans（CF700E）

> 给定字符串 S，求最长序列 t_1, t_2, ..., t_k，其中每个 t_i 是 S 的子串，且 t_i 在 t_{i+1} 中至少出现两次作为子串。求最大的 k。

**分析**：在 SAM 的 parent tree 上 DP。对于节点 u 和其 parent v = link[u]，若 v 代表的串在 u 代表的串中出现至少两次（即 v 的 endpos 集合在 u 的 endpos 范围内至少有两个位置），则 `dp[u] = dp[v] + 1`。判断"出现至少两次"需要用**线段树合并**维护 endpos 集合，查询区间内元素个数。

```
数据结构：SAM + 线段树合并 + parent tree DP
支持操作：endpos 集合区间计数、链式 DP
关键性质：v 在 u 的串中出现 ≥ 2 次 ⟺ v 的 endpos 在 [l, r] 内有 ≥ 2 个元素
时间复杂度：O(n log n)
```

**C++ 代码**：
```cpp
int dp[MAXN], ans = 1;
// seg[u]: 节点 u 的 endpos 线段树根
// query(seg[u], l, r): 查 endpos ∩ [l,r] 的元素个数

void solve(int u) {
    dp[u] = 1;
    // 找到 dp 链上真正的"上一层"节点 p
    int p = sam[u].link;
    // v = p 代表的串长 len[p]，在 u 的 endpos 范围 [len[u], ...] 中
    // 检查 p 的 endpos 中是否有 ≥ 2 个落在 u 的 endpos 范围内
    // 具体：u 的 endpos 范围 [?, ?]，查 seg[p] 在该范围内的计数
    if (p > 1) {
        // 若 p 的串在 u 的串中出现 ≥ 2 次
        dp[u] = dp[p] + 1;
    }
    ans = max(ans, dp[u]);
    for (int v : children[u]) solve(v);
}
```

> 完整实现需要在 parent tree 上维护一个"DP 链祖先"指针，使得判断条件精确匹配。此处给出骨架思路。

**核心启示**：
- **线段树合并 + parent tree DP** 是 SAM 高阶题的标志性组合
- "出现至少 k 次"转化为 endpos 区间计数是核心建模技巧
- parent tree 上的 DP 链需要精心维护"有效祖先"

### 题目 13：熟悉的文章（洛谷 P4022）

> 有 m 个字典串和 q 个查询串。对每个查询串，找最大的 L，使得该串能被划分成若干段，其中来自字典串的子串（长度 ≥ L）占总长度的至少 90%。

**分析**：对所有字典串建广义 SAM。对每个查询串，先在 SAM 上匹配求出 `match[i]`（以位置 i 结尾的最长字典匹配长度）。二分答案 L，check 用单调队列优化 DP：`dp[i] = max(dp[i-1], max_{j ∈ [i-match[i], i-L]} (dp[j] + i - j))`。

```
数据结构：广义 SAM + 二分答案 + 单调队列 DP
支持操作：字典匹配 + 覆盖率最大化
关键性质：match[i] 单调性保证 DP 决策集合滑动
时间复杂度：O((∑|dict| + |query|) log |query|)
```

**C++ 代码**：
```cpp
// 建广义 SAM 后，对查询串 s 求 match 数组
int cur = 1, ml = 0;
for (int i = 0; i < n; i++) {
    int c = s[i] - '0';
    while (cur != 1 && !sam[cur].ch[c])
        cur = sam[cur].link, ml = sam[cur].len;
    if (sam[cur].ch[c]) cur = sam[cur].ch[c], ml++;
    match[i] = ml;
}

// check(L): 单调队列 DP
bool check(int L) {
    deque<int> dq;
    for (int i = 0, j; i < n; i++) {
        dp[i + 1] = dp[i]; // 不选
        j = i - L + 1;
        if (j >= 0) { // 入队
            while (!dq.empty() && dp[dq.back()] - dq.back() <= dp[j] - j)
                dq.pop_back();
            dq.push_back(j);
        }
        while (!dq.empty() && dq.front() < i - match[i])
            dq.pop_front();
        if (!dq.empty())
            dp[i + 1] = max(dp[i + 1], dp[dq.front()] + i + 1 - dq.front());
    }
    return dp[n] * 10 >= n * 9;
}
```

**核心启示**：
- **广义 SAM 统一处理多字典串**，避免逐串匹配
- **match 数组 + 单调队列 DP** 是"最大覆盖"类问题的经典组合
- 二分答案 L 利用了"L 越大越难满足"的单调性

### 题目 16：奥术神杖（洛谷 P5319）

> 给定一个含通配符的字符串和若干模式串（各有权值）。填充通配符后，设模式串在串中出现的权值依次为 v_1, v_2, ...，目标是最大化几何平均值 (v_1·v_2·...·v_m)^{1/m}。

## 五、常见技巧与思路总结

### 5.1 拓扑排序代替 DFS

SAM 的 parent tree 上统计信息时，**按 `len` 桶排序**得到的逆序即为拓扑序，比显式 DFS 更快更简洁：

```cpp
for (int i = sz; i >= 1; i--)
    sam[sam[ord[i]].link].siz += sam[ord[i]].siz;
```

### 5.2 在线增量维护

SAM 的增量构造使得每次 extend 后可以立即获取新增信息（新增子串数 = `len[last] − len[link[last]]`），无需重建。

### 5.3 匹配失败时沿 link 回退

在 SAM 上匹配另一个串时，遇到无转移的情况，沿 suffix link 回退到能继续匹配的最长后缀。这等价于"缩短当前匹配子串"。

### 5.4 拼接与分标记处理多串

双串问题：用特殊分隔符拼接后建 SAM，在 topoCount 时分别维护各串的计数。

### 5.5 差分数组处理等价类长度区间

同一等价类的子串长度是连续的 `[len[link[u]]+1, len[u]]`。按长度统计时，用差分数组 O(1) 标记区间，最后 O(n) 前缀和还原。

### 5.6 clone 节点的特殊处理

clone 出的节点 `siz` 初始为 0（它不对应原串的任何新后缀），但会在 topoCount 中被累加。在某些题目中需要区分"原始节点"和"clone 节点"。

## 六、解题思路框架

### 6.1 遇到 SAM 题时的思考流程

```text
读题
 ├─ 单串还是多串？
 │   ├─ 单串 → 标准 SAM
 │   └─ 多串 → 广义 SAM 或拼接
 ├─ 需要什么信息？
 │   ├─ 子串是否存在 → 在 SAM 上匹配
 │   ├─ 出现次数 → siz（topoCount）
 │   ├─ 不同子串数 → len 差求和
 │   ├─ 第 k 小子串 → DAG 权重 DFS
 │   ├─ endpos 精确集合 → 线段树合并
 │   └─ 后缀间 LCP → parent tree LCA
 ├─ 是否需要在线？
 │   ├─ 是 → 利用 extend 的增量性
 │   └─ 否 → 建完再统计
 └─ 是否需要与其他结构结合？
     ├─ DP → DAG 上 DP 或 parent tree DP
     ├─ 二分 → 二分答案 + SAM check
     ├─ 线段树 → endpos 区间查询
     └─ AC 自动机 → 多模式 + SAM 互补
```

## 七、终极总结与归纳

### 7.1 后缀自动机的本质

**一句话概括：SAM 用 O(n) 个 endpos 等价类状态，压缩表示 O(n²) 个子串的全部信息，同时提供 DAG（子串遍历）和 parent tree（后缀关系 DP）两个视角。**

展开来说，SAM 的威力在于它同时具备两个结构：**转移 DAG**（按字典序枚举子串、匹配子串）和 **suffix link 树**（endpos 的包含关系、LCA = LCP、子树统计）。绝大多数 SAM 题目都是在这两个视角之间切换或联合使用。

### 7.2 核心要点

1. endpos 等价类是 SAM 的灵魂：同一状态的子串共享出现位置集合
2. parent tree = 反串后缀树，是做统计和 DP 的主战场
3. 在线增量构造 + 三种情况处理（无转移/直连/clone）是唯一需要背的模板
4. `len[u] − len[link[u]]` 是每个状态新增的子串数，反复出现在各种统计中
5. 线段树合并维护精确 endpos 集合是 SAM 的高阶核心技术
6. 广义 SAM 处理多串，拼接 + 分标记处理双串，是多串问题的两种基本范式
