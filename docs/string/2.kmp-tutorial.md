## 一、什么是 KMP 算法

KMP（Knuth-Morris-Pratt）算法是一种高效的**单模式串匹配算法**，由 Donald Knuth、Vaughan Pratt 和 James H. Morris 于 1977 年联合发表。它能在 O(n + m) 的时间复杂度内，在文本串中查找模式串的所有出现位置。

与朴素暴力匹配算法不同，KMP 的核心思想是：**当匹配失败时，利用已匹配的信息，避免从头开始比较**。它通过预处理模式串，构建一个"失配指针"数组（通常称为 `next` 数组或前缀函数 `pi`），使得文本串的指针永远不回退，模式串的指针根据 `next` 数组跳转到合适位置继续匹配。

```
暴力匹配：每次失配，文本指针回退，模式指针归零    → O(n × m)
KMP 匹配：失配时，文本指针不动，模式指针按 next 跳转 → O(n + m)
```

| 对比项 | 暴力匹配 | KMP | Rabin-Karp |
|--------|---------|-----|------------|
| 预处理 | 无 | O(m) 构建 next | O(m) 计算哈希 |
| 匹配时间 | O(n × m) | O(n) | O(n) 期望，O(nm) 最坏 |
| 额外空间 | O(1) | O(m) | O(1) |
| 确定性 | 是 | 是 | 否（哈希冲突） |
| 适用场景 | 短串 | 精确匹配 | 多模式/近似匹配 |

## 二、核心思想与前置知识

### 2.1 Border 概念与前缀函数

**Border**：对于字符串 S 的一个前缀 S[1..i]，如果存在一个**真前缀**同时也是**真后缀**，则这个子串称为该前缀的一个 border。

```
字符串：  a b a b a b c
前缀函数：0 0 1 2 3 4 0
          ↑         ↑
       pi[1]=0   pi[6]=4 → "abab" 既是前缀也是后缀
```

**前缀函数 `pi[i]`**（也常记作 `next[i]`）：S[1..i] 的最长 border 长度。

**Border 的传递性**：如果 B 是 S 的 border，B' 是 B 的 border，那么 B' 也是 S 的 border。因此，从 `pi[i]` 出发不断跳转 `pi[pi[i]]`... 直到 0，就能遍历 S[1..i] 的所有 border——这条链称为 **border chain**。

### 2.2 next 数组的构建

构建过程本质上是**模式串自己与自己做 KMP 匹配**：

```cpp
// 构建 next 数组（前缀函数），下标从 1 开始
int nxt[MAXN];
void buildNext(char s[], int m) {
    nxt[1] = 0;
    for (int i = 2, j = 0; i <= m; i++) {
        while (j && s[i] != s[j + 1]) j = nxt[j];
        if (s[i] == s[j + 1]) j++;
        nxt[i] = j;
    }
}
```

### 2.3 KMP 匹配过程

文本串指针 `i` 始终前进，模式串指针 `j` 在失配时按 `next` 跳转：

```cpp
// 在文本串 t 中查找模式串 s 的所有出现位置
// t 长度 n，s 长度 m，下标均从 1 开始
void kmpSearch(char t[], int n, char s[], int m) {
    for (int i = 1, j = 0; i <= n; i++) {
        while (j && t[i] != s[j + 1]) j = nxt[j];
        if (t[i] == s[j + 1]) j++;
        if (j == m) {
            // 匹配成功，起始位置为 i - m + 1
            j = nxt[j]; // 继续寻找下一个匹配
        }
    }
}
```

### 2.4 next 数组的关键性质

| 性质 | 描述 | 常见应用 |
|------|------|---------|
| 最短周期 | `period = i - nxt[i]`，若 `i % period == 0` 则为完美循环 | P4391 |
| Border chain | `nxt[i] → nxt[nxt[i]] → ... → 0` 枚举所有 border | CF126B, P3435 |
| Border 计数 | `cnt[i] = cnt[nxt[i]] + 1` 递推 border 数量 | P2375 |
| 出现次数 | 逆序 `f[nxt[i]] += f[i]` 统计前缀出现次数 | CF432D |

## 三、分类与变体详解

### 3.1 基础 KMP 匹配

最基本的用法：在文本串中查找模式串的所有出现位置，输出匹配位置和 next 数组。

```
算法思路：预处理模式串 next 数组，双指针扫描文本串
关键步骤：失配时 j = nxt[j]，匹配成功时记录位置
时间复杂度：O(n + m)
空间复杂度：O(m)
```

### 3.2 next 数组的性质与应用

#### 最短周期 / 循环节

字符串 S[1..n] 的**最短周期**为 `p = n - nxt[n]`。若 `n % p == 0`，则 S 是 S[1..p] 的完美重复。

```cpp
int period = n - nxt[n];
bool perfect = (n % period == 0);
```

#### Border Chain 遍历

沿 `nxt` 链跳转，可得到所有 border 长度，从长到短：

```cpp
// 枚举 S[1..n] 的所有 border 长度
for (int j = nxt[n]; j; j = nxt[j]) {
    // j 是一个 border 长度
}
```

#### 前缀出现次数统计

利用 "border 是 border 的 border" 的传递性，逆序累加：

```cpp
int f[MAXN]; // f[i] = S[1..i] 作为子串的出现次数
for (int i = 1; i <= n; i++) f[i] = 1;
for (int i = n; i >= 1; i--) f[nxt[i]] += f[i];
```

### 3.3 KMP + DP 综合

当 KMP 与动态规划结合时，常见模式：

- **KMP + 矩阵快速幂**：在长度为 n 的字符串中，不包含某模式串的方案数，通过 KMP 自动机状态转移 + 矩阵幂加速。
- **KMP + 贪心/栈**：删除匹配子串后继续匹配（类似括号消除），用栈保存匹配状态。
- **KMP + 递推统计**：利用 border chain 的性质进行 DP 递推。

```
算法思路：将 KMP 匹配视为自动机，状态转移与 DP 结合
关键步骤：建立 KMP 转移矩阵 trans[j][c]，用 DP 或矩阵幂求解
时间复杂度：视具体问题，通常 O(m² × log n) 或 O(n × m)
空间复杂度：O(m²) 或 O(n)
```

### 3.4 扩展 KMP（Z-Algorithm）简介

Z 函数 `z[i]` 定义为：S[i..n] 与 S[1..n] 的最长公共前缀长度。

```cpp
// Z 函数模板
int z[MAXN];
void buildZ(char s[], int n) {
    z[1] = n;
    for (int i = 2, l = 0, r = 0; i <= n; i++) {
        if (i <= r) z[i] = min(z[i - l + 1], r - i + 1);
        while (i + z[i] <= n && s[z[i] + 1] == s[i + z[i]]) z[i]++;
        if (i + z[i] - 1 > r) l = i, r = i + z[i] - 1;
    }
}
```

| 对比 | KMP (前缀函数) | Z-Algorithm |
|------|---------------|-------------|
| `pi[i]` / `z[i]` | S[1..i] 最长 border | S[i..n] 与 S 的 LCP |
| 视角 | 后缀看前缀 | 后缀看整串 |
| 典型应用 | 模式匹配、周期 | LCP 查询、匹配计数 |

### 题目 1：KMP 字符串匹配（洛谷 P3375）

> 给定文本串 S 和模式串 T，输出 T 在 S 中所有出现的位置，并输出 T 的 next 数组。

**分析**：KMP 模板题，直接应用标准 KMP 算法即可。

```
算法思路：构建模式串 next 数组，KMP 扫描文本串
关键步骤：失配跳转 j = nxt[j]，j == m 时记录匹配位置
时间复杂度：O(n + m)
空间复杂度：O(m)
```

**C++ 代码**：

```cpp
const int MAXN = 1e6 + 5;
int nxt[MAXN];
char s[MAXN], t[MAXN];

void solve(int n, int m) {
    // 构建 next
    for (int i = 2, j = 0; i <= m; i++) {
        while (j && t[i] != t[j + 1]) j = nxt[j];
        if (t[i] == t[j + 1]) j++;
        nxt[i] = j;
    }
    // KMP 匹配
    for (int i = 1, j = 0; i <= n; i++) {
        while (j && s[i] != t[j + 1]) j = nxt[j];
        if (s[i] == t[j + 1]) j++;
        if (j == m) {
            printf("%d\n", i - m + 1);
            j = nxt[j];
        }
    }
    for (int i = 1; i <= m; i++)
        printf("%d%c", nxt[i], " \n"[i == m]);
}
```

**核心启示**：
- **next 数组是 KMP 的灵魂**，所有变体题都建立在对它的理解之上
- **文本指针不回退**是 KMP 线性复杂度的根本保证

### 题目 2：Radio Transmission（洛谷 P4391 [BOI2009]）

> 给定长度为 n 的字符串，它是某个字符串 S 不断重复后截取前 n 个字符得到的。求 S 的最短长度。

**分析**：最短周期即 `n - nxt[n]`。由 next 数组性质，`nxt[n]` 是最长 border，`n - nxt[n]` 就是最短循环节长度（不要求整除）。

```
算法思路：构建 next 数组，利用最短周期公式
关键步骤：ans = n - nxt[n]
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 1e6 + 5;
int nxt[MAXN];
char s[MAXN];

void solve(int n) {
    for (int i = 2, j = 0; i <= n; i++) {
        while (j && s[i] != s[j + 1]) j = nxt[j];
        if (s[i] == s[j + 1]) j++;
        nxt[i] = j;
    }
    printf("%d\n", n - nxt[n]);
}
```

**核心启示**：
- **`n - nxt[n]` 是最短周期**，这是 next 数组最重要的性质之一
- **无需整除判断**——题目允许截断，所以直接输出即可

### 题目 3：Periods of Words（洛谷 P3435 [POI2006]OKR）

> 给定长度为 n 的字符串 S，对每个前缀 S[1..i]，定义其"周期"为最短的 S[1..k]（k < i），使得 S[1..i] 是 S[1..k] 重复若干次后的一个前缀。求所有前缀的最大周期长度之和。

**分析**：每个前缀的最大周期 = `i - (最短 border 长度)`。最短 border 通过沿 nxt 链跳到底获得。为避免重复跳转，使用路径压缩优化。

```
算法思路：对每个前缀，沿 border chain 找最短非零 border，用路径压缩加速
关键步骤：while(nxt[nxt[j]]) j = nxt[j]; 跳到最短 border，累加 i - j
时间复杂度：O(n)（路径压缩均摊）
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 1e6 + 5;
int nxt[MAXN];
char s[MAXN];

void solve(int n) {
    for (int i = 2, j = 0; i <= n; i++) {
        while (j && s[i] != s[j + 1]) j = nxt[j];
        if (s[i] == s[j + 1]) j++;
        nxt[i] = j;
    }
    long long ans = 0;
    for (int i = 2; i <= n; i++) {
        int j = nxt[i];
        while (nxt[j]) j = nxt[j]; // 跳到最短 border
        if (nxt[i]) nxt[i] = j;    // 路径压缩
        ans += i - j;
    }
    printf("%lld\n", ans);
}
```

**核心启示**：
- **Border chain 的路径压缩**是处理 "所有前缀的某种 border 性质" 的经典优化
- **最短 border = border chain 的末端**，通过不断跳 nxt 到底获得

### 题目 4：Censoring（洛谷 P4824 [USACO15FEB] Silver）

> 给定文本串 S 和模式串 T，不断从 S 中删去第一次出现的 T，直到 S 中不再包含 T。输出最终结果。

**分析**：用栈模拟 KMP 匹配。每次匹配成功时弹出栈顶 |T| 个字符，同时恢复之前的匹配状态。关键是栈中保存每个位置对应的 KMP 状态 `j`。

```
算法思路：KMP + 栈，匹配成功则弹出，恢复栈顶的 KMP 状态
关键步骤：栈保存 (字符, 匹配状态 j)，匹配成功弹出 m 个元素
时间复杂度：O(n + m)
空间复杂度：O(n + m)
```

**C++ 代码**：

```cpp
const int MAXN = 1e6 + 5;
int nxt[MAXN], stk[MAXN], top;
int state[MAXN]; // 栈中每个位置的 KMP 状态
char s[MAXN], t[MAXN], res[MAXN];

void solve(int n, int m) {
    for (int i = 2, j = 0; i <= m; i++) {
        while (j && t[i] != t[j + 1]) j = nxt[j];
        if (t[i] == t[j + 1]) j++;
        nxt[i] = j;
    }
    top = 0;
    state[0] = 0;
    for (int i = 1; i <= n; i++) {
        stk[++top] = i;
        int j = state[top - 1];
        while (j && s[i] != t[j + 1]) j = nxt[j];
        if (s[i] == t[j + 1]) j++;
        state[top] = j;
        if (j == m) top -= m; // 弹出匹配的 m 个字符
    }
    for (int i = 1; i <= top; i++) res[i] = s[stk[i]];
    res[top + 1] = '\0';
    printf("%s\n", res + 1);
}
```

**核心启示**：
- **KMP + 栈**是处理"删除后继续匹配"类问题的标准范式
- **保存匹配状态到栈**使得弹出后可以 O(1) 恢复，避免重新匹配

### 题目 5：Password（CF126B）

> 给定字符串 S，找一个最长的子串 T，满足 T 既是 S 的前缀，也是 S 的后缀，同时还在 S 的内部（非前缀/后缀位置）出现过。若不存在输出 "Just a legend"。

**分析**：先用 border chain 找出所有既是前缀又是后缀的长度，再检查这些长度是否在中间出现过。中间出现可通过检查 `nxt[i]` 的最大值（2 ≤ i ≤ n-1）来判断。

```
算法思路：border chain 枚举前后缀长度，检查中间是否出现
关键步骤：记录中间 nxt 最大值，沿 border chain 找满足条件的最长 border
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 1e6 + 5;
int nxt[MAXN];
char s[MAXN];

void solve(int n) {
    for (int i = 2, j = 0; i <= n; i++) {
        while (j && s[i] != s[j + 1]) j = nxt[j];
        if (s[i] == s[j + 1]) j++;
        nxt[i] = j;
    }
    // 标记中间出现过的 border 长度
    int mx = 0;
    for (int i = 2; i < n; i++) mx = max(mx, nxt[i]);
    // 沿 border chain 找答案
    int ans = -1;
    for (int j = nxt[n]; j; j = nxt[j]) {
        if (j <= mx) { ans = j; break; }
    }
    if (ans <= 0) puts("Just a legend");
    else { s[ans + 1] = '\0'; printf("%s\n", s + 1); }
}
```

**核心启示**：
- **"前缀=后缀且中间出现"是 border chain 的经典变种**
- **记录中间 nxt 最大值**可以快速判断某长度的 border 是否在中间出现

### 题目 6：Prefixes and Suffixes（CF432D）

> 给定字符串 S，求所有既是前缀又是后缀的子串长度，以及每个这样的子串在 S 中作为子串出现的次数。

**分析**：既是前缀又是后缀 → 沿 `nxt[n]` 的 border chain 枚举。出现次数 → 逆序递推 `f[nxt[i]] += f[i]`。

```
算法思路：border chain 枚举 + 逆序累加统计出现次数
关键步骤：f[i] 初始为 1，逆序 f[nxt[i]] += f[i]
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 1e5 + 5;
int nxt[MAXN], f[MAXN];
char s[MAXN];

void solve(int n) {
    for (int i = 2, j = 0; i <= n; i++) {
        while (j && s[i] != s[j + 1]) j = nxt[j];
        if (s[i] == s[j + 1]) j++;
        nxt[i] = j;
    }
    for (int i = 1; i <= n; i++) f[i] = 1;
    for (int i = n; i >= 1; i--) f[nxt[i]] += f[i];

    vector<pair<int,int>> borders;
    for (int j = n; j; j = nxt[j])
        borders.push_back({j, f[j]});
    reverse(borders.begin(), borders.end());
    printf("%d\n", (int)borders.size());
    for (auto &p : borders)
        printf("%d %d\n", p.first, p.second);
}
```

**核心启示**：
- **逆序累加 `f[nxt[i]] += f[i]]`** 是统计前缀出现次数的核心技巧
- **Border chain 天然给出所有"前缀=后缀"的长度**

### 🟠 提高级 Level 3：深入 next 性质

### 题目 7：动物园（洛谷 P2375 [NOI2014]）

> 给定字符串 S，对每个前缀 S[1..i]，求 `num[i]`：满足长度不超过 i/2 的 border 个数。输出 ∏(num[i]+1) mod 10⁹+7。

**分析**：先递推 `cnt[i] = cnt[nxt[i]] + 1` 得到所有 border 数量，再对每个 i 沿 nxt 链跳到 ≤ i/2 的位置，利用 cnt 得到 num[i]。跳转过程均摊 O(n)。

```
算法思路：KMP 递推 cnt 数组 + 跳 nxt 链限制长度
关键步骤：cnt[i] = cnt[nxt[i]] + 1；j 从 nxt[i] 跳到 j ≤ i/2
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 1e6 + 5, MOD = 1e9 + 7;
int nxt[MAXN], cnt[MAXN];
char s[MAXN];

long long solve(int n) {
    for (int i = 2, j = 0; i <= n; i++) {
        while (j && s[i] != s[j + 1]) j = nxt[j];
        if (s[i] == s[j + 1]) j++;
        nxt[i] = j;
        cnt[i] = cnt[j] + 1;
    }
    long long ans = 1;
    for (int i = 2, j = 0; i <= n; i++) {
        while (j && s[i] != s[j + 1]) j = nxt[j];
        if (s[i] == s[j + 1]) j++;
        while (j * 2 > i) j = nxt[j];
        ans = ans * (cnt[j] + 1) % MOD;
    }
    return ans;
}
```

**核心启示**：
- **`cnt[i] = cnt[nxt[i]] + 1` 递推所有 border 数**是 NOI 级别的经典技巧
- **限制"不重叠"通过 `j*2 > i` 条件跳转**，均摊仍为 O(n)

### 题目 8：SZA-Template（洛谷 P3426 [POI2005]）

> 给定字符串 S，用一个"印章"反复盖印（可重叠，但重叠处字符必须相同），使得恰好覆盖 S。求最短印章长度。

**分析**：印章必须是 S 的某个前缀，且也是某个 border。用 DP：`f[i]` 表示覆盖 S[1..i] 所需的最短印章长度。如果上一次盖到 `f[nxt[i]]` 的覆盖范围能延伸到 i，则 `f[i] = f[nxt[i]]`，否则 `f[i] = i`。

```
算法思路：KMP + DP，f[i] = f[nxt[i]] 或 i，用桶记录最远覆盖位置
关键步骤：t[f[nxt[i]]] >= i - nxt[i] 时可继承，否则 f[i] = i
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 5e5 + 5;
int nxt[MAXN], f[MAXN], last[MAXN]; // last: 该印章长度最近使用位置
char s[MAXN];

void solve(int n) {
    for (int i = 2, j = 0; i <= n; i++) {
        while (j && s[i] != s[j + 1]) j = nxt[j];
        if (s[i] == s[j + 1]) j++;
        nxt[i] = j;
    }
    f[1] = 1; last[1] = 1;
    for (int i = 2; i <= n; i++) {
        f[i] = i; // 默认整段为印章
        if (nxt[i] && last[f[nxt[i]]] >= i - nxt[i])
            f[i] = f[nxt[i]];
        last[f[i]] = i;
    }
    printf("%d\n", f[n]);
}
```

**核心启示**：
- **印章问题转化为 border + DP 的覆盖判定**
- **"桶记录最远覆盖位置"** 是 O(1) 判定能否延伸的关键

### 题目 9：MUH and Cube Walls（CF471D）

> 给定两个整数序列 a 和 b，问 b 在 a 中出现多少次（"出现"定义为差分相同，即形状匹配而非数值匹配）。

**分析**：将两个序列分别做差分，然后在差分后的 a 上用 KMP 匹配差分后的 b。注意 b 长度为 1 时特殊处理。

```
算法思路：差分转化 + 标准 KMP 匹配
关键步骤：a'[i] = a[i+1] - a[i]，b'[i] = b[i+1] - b[i]，KMP 匹配 a' 和 b'
时间复杂度：O(n + m)
空间复杂度：O(n + m)
```

**C++ 代码**：

```cpp
const int MAXN = 2e5 + 5;
int a[MAXN], b[MAXN], da[MAXN], db[MAXN], nxt[MAXN];

void solve(int n, int m) {
    if (m == 1) { printf("%d\n", n); return; }
    for (int i = 1; i < n; i++) da[i] = a[i + 1] - a[i];
    for (int i = 1; i < m; i++) db[i] = b[i + 1] - b[i];
    int pm = m - 1, pn = n - 1;
    // 对 db 建 next
    for (int i = 2, j = 0; i <= pm; i++) {
        while (j && db[i] != db[j + 1]) j = nxt[j];
        if (db[i] == db[j + 1]) j++;
        nxt[i] = j;
    }
    int ans = 0;
    for (int i = 1, j = 0; i <= pn; i++) {
        while (j && da[i] != db[j + 1]) j = nxt[j];
        if (da[i] == db[j + 1]) j++;
        if (j == pm) { ans++; j = nxt[j]; }
    }
    printf("%d\n", ans);
}
```

**核心启示**：
- **差分转化**将"形状匹配"问题化归为标准 KMP
- **序列长度为 1 的边界处理**容易遗漏

### 题目 10：Compress Words（CF1200E）

> 给 n 个字符串，依次拼接，但相邻两个字符串若有重叠（前一个的后缀 = 后一个的前缀），则合并重叠部分。求最终结果。

**分析**：每次将新字符串与当前结果的后缀做 KMP，求最大重叠长度。为控制复杂度，每次只取当前结果的后 min(|result|, |new|) 个字符参与匹配。

```
算法思路：逐个拼接，KMP 求最大前后缀重叠
关键步骤：构造临时串 new + '#' + tail(result)，求 nxt 末尾值
时间复杂度：O(Σ|wi|)（所有字符串总长）
空间复杂度：O(Σ|wi|)
```

**C++ 代码**：

```cpp
string res;

void merge(const string &w) {
    if (res.empty()) { res = w; return; }
    int len = min(res.size(), w.size());
    string tmp = w + "#" + res.substr(res.size() - len);
    int m = tmp.size();
    vector<int> nxt(m + 1, 0);
    for (int i = 1; i < m; i++) {
        int j = nxt[i];
        while (j && tmp[i] != tmp[j]) j = nxt[j];
        if (tmp[i] == tmp[j]) j++;
        nxt[i + 1] = j;
    }
    int overlap = nxt[m];
    res += w.substr(overlap);
}
// 对 n 个单词逐个调用 merge(w)
```

**核心启示**：
- **"拼接取最大重叠"本质是求两个串的最长公共前后缀**
- **用 `new + '#' + tail` 构造辅助串**避免了在完整 result 上做 KMP

### 题目 11：Count the String（HDU3336）

> 给定字符串 S，求 S 的所有前缀在 S 中作为子串出现的次数之和，结果对 10007 取模。

**分析**：经典的 "前缀出现次数" 统计。令 `f[i] = 1`，逆序 `f[nxt[i]] += f[i]`，最终累加即可。

```
算法思路：KMP + 逆序递推统计前缀出现次数
关键步骤：f[i] = 1，逆序 f[nxt[i]] += f[i]，答案 = Σf[i]
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 2e5 + 5, MOD = 10007;
int nxt[MAXN], f[MAXN];
char s[MAXN];

void solve(int n) {
    for (int i = 2, j = 0; i <= n; i++) {
        while (j && s[i] != s[j + 1]) j = nxt[j];
        if (s[i] == s[j + 1]) j++;
        nxt[i] = j;
    }
    int ans = 0;
    for (int i = 1; i <= n; i++) f[i] = 1;
    for (int i = n; i >= 1; i--) f[nxt[i]] = (f[nxt[i]] + f[i]) % MOD;
    for (int i = 1; i <= n; i++) ans = (ans + f[i]) % MOD;
    printf("%d\n", ans);
}
```

**核心启示**：
- **逆序累加是 "前缀出现次数" 的标准套路**
- **理解 `f[nxt[i]] += f[i]`**：i 出现一次，则 nxt[i] 也出现一次

### 🔴 困难级 Level 4：KMP + DP 综合

### 题目 12：GT 考试（洛谷 P3193 [HNOI2008]）

> 求长度为 n 的数字串中，不包含给定长度为 m 的子串的方案数，结果对 K 取模。（n ≤ 10⁹，m ≤ 20）

**分析**：经典的 KMP 自动机 + 矩阵快速幂。将 KMP 的匹配状态 0..m-1 作为 DP 状态，转移矩阵 `A[i][j]` 表示从状态 i 添加一个数字后到达状态 j 的方案数。用矩阵快速幂计算 Aⁿ。

```
算法思路：KMP 自动机转移 + 矩阵快速幂加速
关键步骤：建立 m×m 转移矩阵 A，答案为 A^n 第 0 行之和
时间复杂度：O(m³ × log n)
空间复杂度：O(m²)
```

**C++ 代码**：

```cpp
const int MAXM = 25;
int nxt[MAXM], MOD;
char t[MAXM];

struct Matrix {
    int a[MAXM][MAXM], sz;
    Matrix(int n) : sz(n) { memset(a, 0, sizeof(a)); }
    Matrix operator*(const Matrix &o) const {
        Matrix res(sz);
        for (int i = 0; i < sz; i++)
            for (int k = 0; k < sz; k++) if (a[i][k])
                for (int j = 0; j < sz; j++)
                    res.a[i][j] = (res.a[i][j] + 1LL * a[i][k] * o.a[k][j]) % MOD;
        return res;
    }
};

void solve(int n, int m) {
    // 建 next
    for (int i = 2, j = 0; i <= m; i++) {
        while (j && t[i] != t[j + 1]) j = nxt[j];
        if (t[i] == t[j + 1]) j++;
        nxt[i] = j;
    }
    // 建转移矩阵
    Matrix A(m);
    for (int i = 0; i < m; i++)
        for (char c = '0'; c <= '9'; c++) {
            int j = i;
            while (j && t[j + 1] != c) j = nxt[j];
            if (t[j + 1] == c) j++;
            if (j < m) A.a[i][j]++;
        }
    // 矩阵快速幂
    Matrix res(m);
    for (int i = 0; i < m; i++) res.a[i][i] = 1; // 单位矩阵
    while (n) {
        if (n & 1) res = res * A;
        A = A * A;
        n >>= 1;
    }
    int ans = 0;
    for (int j = 0; j < m; j++) ans = (ans + res.a[0][j]) % MOD;
    printf("%d\n", ans);
}
```

**核心启示**：
- **KMP 自动机是 AC 自动机的单模式版**，适合与矩阵快速幂结合
- **m 很小 + n 极大 → 矩阵快速幂**是经典信号

### 题目 13：Power Strings（POJ2406）

> 给定字符串 S，求最大的 k 使得 S 可以表示为某个串的 k 次重复。

**分析**：最短周期 `p = n - nxt[n]`，若 `n % p == 0`，则 `k = n / p`；否则 `k = 1`。

```
算法思路：KMP 求最短周期，判断是否完美整除
关键步骤：p = n - nxt[n]，若 n % p == 0 则 k = n/p，否则 k = 1
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 1e6 + 5;
int nxt[MAXN];
char s[MAXN];

void solve(int n) {
    for (int i = 2, j = 0; i <= n; i++) {
        while (j && s[i] != s[j + 1]) j = nxt[j];
        if (s[i] == s[j + 1]) j++;
        nxt[i] = j;
    }
    int p = n - nxt[n];
    printf("%d\n", n % p == 0 ? n / p : 1);
}
```

**核心启示**：
- **完美循环节 = 最短周期且整除**
- **与 P4391 的区别**：P4391 不要求整除，此题要求完美重复

### 题目 14：Period（POJ1961）

> 给定字符串 S，对每个前缀 S[1..i]（i ≥ 2），若存在最短周期 p 使得 `i % p == 0` 且 `i/p ≥ 2`，则输出 i 和 i/p。

**分析**：对每个 i，检查 `p = i - nxt[i]`，若 `i % p == 0` 且 `i/p ≥ 2` 则满足条件。

```
算法思路：枚举每个前缀，检查最短周期性质
关键步骤：p = i - nxt[i]，检查 i % p == 0 && i / p >= 2
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 1e6 + 5;
int nxt[MAXN];
char s[MAXN];

void solve(int n) {
    for (int i = 2, j = 0; i <= n; i++) {
        while (j && s[i] != s[j + 1]) j = nxt[j];
        if (s[i] == s[j + 1]) j++;
        nxt[i] = j;
    }
    for (int i = 2; i <= n; i++) {
        int p = i - nxt[i];
        if (p != i && i % p == 0)
            printf("%d %d\n", i, i / p);
    }
}
```

**核心启示**：
- **逐前缀检查周期性是 next 数组的入门应用**
- **`p != i` 等价于 `nxt[i] > 0`**，确保存在非平凡周期

### 题目 15：Seek the Name, Seek the Fame（POJ2752）

> 给定字符串 S，输出所有既是前缀又是后缀的子串长度（从小到大）。

**分析**：从 `nxt[n]` 沿 border chain 跳到 0，收集所有长度，加上 n 本身，逆序输出。

```
算法思路：沿 border chain 收集所有 border 长度
关键步骤：j = nxt[n] → nxt[nxt[n]] → ... → 0，收集后加 n
时间复杂度：O(n)
空间复杂度：O(n)
```

**C++ 代码**：

```cpp
const int MAXN = 4e5 + 5;
int nxt[MAXN];
char s[MAXN];

void solve(int n) {
    for (int i = 2, j = 0; i <= n; i++) {
        while (j && s[i] != s[j + 1]) j = nxt[j];
        if (s[i] == s[j + 1]) j++;
        nxt[i] = j;
    }
    vector<int> res;
    for (int j = n; j; j = nxt[j]) res.push_back(j);
    reverse(res.begin(), res.end());
    for (int i = 0; i < (int)res.size(); i++)
        printf("%d%c", res[i], " \n"[i + 1 == (int)res.size()]);
}
```

**核心启示**：
- **Border chain 遍历是 "前缀=后缀" 类问题的通用解法**
- **注意输出顺序**：收集时从大到小，输出从小到大需 reverse

### 题目 16：Fibonacci String 周期检测（综合练习）

> 给定两个字符串 s₀ 和 s₁，定义 sₙ = sₙ₋₁ + sₙ₋₂（字符串拼接）。给定字符串 t，求 t 在 sₙ 中出现次数。

**分析**：由于 sₙ 长度呈指数增长，不能直接构造。利用 KMP 匹配后，sₙ 中 t 的出现次数可以通过记录 sₙ₋₁ 和 sₙ₋₂ 拼接处的匹配状态递推。关键是维护每个 sₖ 的"末尾 KMP 状态"和"出现次数"。

```
算法思路：KMP + 递推，维护拼接处匹配状态
关键步骤：cnt[k] = cnt[k-1] + cnt[k-2] + cross(state_k-1, s_k-2)
时间复杂度：O(|t| × n)（n 为 Fibonacci 层数）
空间复杂度：O(|t|)
```

**C++ 代码**：

```cpp
const int MAXM = 1e5 + 5;
int nxt[MAXM];
char t[MAXM];

// 从状态 j 开始匹配文本 txt，返回 {匹配次数, 结束状态}
pair<long long, int> kmpCount(const string &txt, int m, int j) {
    long long cnt = 0;
    for (char c : txt) {
        while (j && c != t[j + 1]) j = nxt[j];
        if (c == t[j + 1]) j++;
        if (j == m) { cnt++; j = nxt[j]; }
    }
    return {cnt, j};
}

// 对 Fibonacci String 递推 cnt 和末尾 KMP 状态
// f[0] = s0, f[1] = s1, f[k] = f[k-1] + f[k-2]
// 维护 (cnt[k], endState[k]) 递推即可
```

**核心启示**：
- **拼接串的 KMP 计数可以通过状态传递递推**，无需真正构造长串
- **Fibonacci 结构 + KMP 状态**是"不可构造长串"问题的经典处理方式

## 五、常见技巧与思路总结

### 5.1 next 数组的多重身份

| 用途 | 公式/做法 |
|------|----------|
| 最长 border | `nxt[i]` 直接就是 |
| 最短周期 | `p = i - nxt[i]` |
| 完美循环判断 | `i % (i - nxt[i]) == 0` |
| 所有 border | 沿 `nxt[i] → nxt[nxt[i]] → ...` |
| 不重叠 border | 跳到 `j ≤ i/2` |

### 5.2 路径压缩加速 border chain

当需要对每个前缀都跳到链底时，直接跳是 O(n²) 最坏。加入路径压缩：

```cpp
for (int i = 2; i <= n; i++) {
    int j = nxt[i];
    while (nxt[j]) j = nxt[j];
    if (nxt[i]) nxt[i] = j; // 压缩
}
```

### 5.3 KMP + 栈处理"删除后继续匹配"

对于"匹配后删除、在删除后的新串中继续匹配"类问题，用栈维护字符和匹配状态，匹配成功弹出 m 个栈顶。

### 5.4 KMP 自动机与矩阵快速幂

当 m 很小（≤ 20-30）、n 极大（≤ 10⁹）时，将 KMP 转移写成矩阵形式，用矩阵快速幂加速。

### 5.5 差分 + KMP 处理"形状匹配"

若匹配条件是"差分相同"（即平移不变），先对序列做差分，再做标准 KMP。

### 5.6 构造辅助串 `B + '#' + A` 求最大重叠

求串 A 的后缀与串 B 的前缀的最长重叠：构造 `B + '#' + suffix(A)`，取 nxt 末尾值。

## 七、终极总结与归纳

### 7.1 KMP 的本质

**KMP 的本质是利用已匹配信息避免重复比较。** `next` 数组编码了模式串自身的结构信息——所有前缀的最长 border。这一信息不仅用于加速匹配，更揭示了字符串的周期性质、前后缀关系和自相似结构，使得大量看似复杂的字符串问题都可以在线性时间内解决。

### 7.2 核心要点

1. **`nxt[i]`**：S[1..i] 的最长真前后缀（border）长度
2. **最短周期**：`p = i - nxt[i]`，完美循环当且仅当 `i % p == 0`
3. **Border chain**：`nxt[i] → nxt[nxt[i]] → ... → 0` 遍历所有 border
4. **出现次数**：逆序 `f[nxt[i]] += f[i]` 统计前缀出现次数
5. **KMP + 栈**：处理"删除后继续匹配"，栈保存匹配状态
6. **KMP + 矩阵幂**：m 小 n 大的方案计数，自动机状态转移
